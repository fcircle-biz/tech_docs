# 図解提案ワークフロー

このワークフローは、指定されたディレクトリ内のHTML技術ガイドを分析し、図解を追加すべき箇所を提案するMarkdownレポートを生成します。

## 入力
- **TargetDirectory**: 分析対象のHTMLガイドが含まれるディレクトリ。

## 手順

1.  **HTMLファイルの検索**:
    - `TargetDirectory`内の`.html`ファイルを検索します。
    - **除外**: `v`の後に数字が続くディレクトリ（例: `v1`, `v2`）は除外します。
    - **コマンド**: `find [TargetDirectory] -name "*.html" -not -path "*/v[0-9]*/*"`

2.  **内容の分析とレポート生成**:
    - 見つかった各HTMLファイルについて:
        - 内容を読み込みます。
        - 図解が有効なセクション（抽象的な概念、複雑なフロー、関係性など）を特定します。
        - **出力形式**: Markdown。
    - **レポート構成**:
        ```markdown
        # 図解の提案

        ## ファイル: [ファイルの相対パス]
        | セクション / コンテキスト | 提案 | 理由 |
        | :--- | :--- | :--- |
        | [見出しまたはテキストの抜粋] | [図解の説明] | [なぜ役立つか] |
        ```

3.  **出力**:
    - 生成されたMarkdownレポートをユーザーに提示します。
