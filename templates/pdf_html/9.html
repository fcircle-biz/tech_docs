サーバーコントロール徹底解説

サーバーコントロールはASP.NET WebFormsの中核要素で、HTMLを直接記述せずにUIコンポーネントを構築できます。 コントロールはサーバー上で処理され、クライアント側のHTMLに変換されます。

主要なサーバーコントロール

標準コントロール - TextBox, Button, Label, CheckBox, DropDownList等
データコントロール - GridView, DetailsView, FormView, Repeater等
検証コントロール - RequiredFieldValidator, RegularExpressionValidator等
ナビゲーションコントロール - Menu, SiteMapPath, TreeView等
<asp:TextBox runat="server" ID="txtName" /> <asp:Button runat="server" ID="btnSubmit" Text="送信" OnClick="btnSubmit_Click" /> <asp:Label runat="server" ID="lblResult" />

ViewStateの仕組み

ViewStateはコントロールの状態をページのライフサイクル間で保持する仕組みです。

ページのHTMLに隠しフィールド(__VIEWSTATE)として保存
ポストバック時にコントロールの状態を復元
暗号化・改ざん検知機能を内蔵

コントロール間の連携例

protected void btnSubmit_Click( object sender, EventArgs e) { // TextBoxの値をLabelに表示 lblResult.Text = "こんにちは、" + txtName.Text + "さん！"; }

開発のポイント

コントロールIDはページ内で一意である必要があります。マスターページ使用時は注意が必要です。
