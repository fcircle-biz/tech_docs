共通仕様

人物情報：名前、年齢、性別コード

性別コード：

  • 2（女性）
  • 9（その他）
  • 上記以外（不明）

実施ルール

  • まず自分で考えてコードを書いてから、回答を確認してください
  • IDEまたはテキストエディタでJavaファイルを作成し、実際にコンパイル・実行してください
  • エラーが出た場合は、エラーメッセージをよく読んで原因を考えてください
1

Personクラス作成
name、age、genderCodeを持つPersonクラスを作成し、コンストラクタで初期化してください。

回答を表示

模範解答:

public class Person {
    String name;
    int age;
    int genderCode;

    public Person(String name, int age, int genderCode) {
        this.name = name;
        this.age = age;
        this.genderCode = genderCode;
    }

    public void display() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("性別コード: " + genderCode);
    }
}

ポイント: クラスはフィールド（属性）とメソッド（振る舞い）を持ちます。コンストラクタはnewでインスタンス生成時に呼ばれ、フィールドを初期化します。thisはインスタンス自身を指します。

2

性別名表示
genderCodeから性別名（「男性」「女性」「その他」「不明」）を取得して表示するメソッドを追加してください。

回答を表示

模範解答:

public class Person {
    String name;
    int age;
    int genderCode;

    public Person(String name, int age, int genderCode) {
        this.name = name;
        this.age = age;
        this.genderCode = genderCode;
    }

    public String getGenderName() {
        switch (genderCode) {
            case 1:
                return "男性";
            case 2:
                return "女性";
            case 9:
                return "その他";
            default:
                return "不明";
        }
    }

    public void display() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("性別: " + getGenderName());
    }
}

ポイント: メソッドは処理をカプセル化します。switch文で複数の条件分岐を簡潔に記述できます。defaultは該当しない場合の処理です。

3

カプセル化
フィールドをprivateに変更し、getterメソッドを作成してください。

回答を表示

模範解答:

public class Person {
    private String name;
    private int age;
    private int genderCode;

    public Person(String name, int age, int genderCode) {
        this.name = name;
        this.age = age;
        this.genderCode = genderCode;
    }

    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    public int getGenderCode() {
        return genderCode;
    }

    public String getGenderName() {
        switch (genderCode) {
            case 1:
                return "男性";
            case 2:
                return "女性";
            case 9:
                return "その他";
            default:
                return "不明";
        }
    }

    public void display() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("性別: " + getGenderName());
    }
}

ポイント: カプセル化はフィールドをprivateにして外部から直接アクセスできないようにし、公開する情報をgetterメソッドで制御します。これにより情報の隠蔽と保護ができます。

4

入力値チェック
年齢が0未満の場合にIllegalArgumentExceptionをスローするようコンストラクタを修正してください。

回答を表示

模範解答:

public class Person {
    private String name;
    private int age;
    private int genderCode;

    public Person(String name, int age, int genderCode) {
        if (age < 0) {
            throw new IllegalArgumentException("年齢は0以上で指定してください");
        }
        this.name = name;
        this.age = age;
        this.genderCode = genderCode;
    }

    // getterメソッド、getGenderName()、display()は省略
}

ポイント: 例外は異常な状態を通知する仕組みです。IllegalArgumentExceptionは不正な引数が渡された時にスローする標準例外です。throwで例外をスローします。

5

例外処理
mainメソッドでtry-catchを使い、例外をキャッチしてエラーメッセージを表示してください。

回答を表示

模範解答:

public class Main {
    public static void main(String[] args) {
        try {
            // 正常なケース
            Person person1 = new Person("田中太郎", 25, 1);
            person1.display();
            System.out.println();

            // 異常なケース（年齢が負）
            Person person2 = new Person("山田花子", -5, 2);
            person2.display();
        } catch (IllegalArgumentException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}

ポイント: try-catchで例外を捕捉し、適切に処理します。e.getMessage()で例外メッセージを取得できます。例外が発生するとcatchブロックに移り、以降の処理はスキップされます。

6

継承
PersonをextendsしたStudentクラスを作成し、学籍番号（studentId）フィールドを追加してください。

回答を表示

模範解答:

public class Student extends Person {
    private String studentId;

    public Student(String name, int age, int genderCode, String studentId) {
        super(name, age, genderCode);
        this.studentId = studentId;
    }

    public String getStudentId() {
        return studentId;
    }
}

ポイント: extendsで継承を実現します。子クラスは親クラスのフィールドとメソッドを引き継ぎます。super()で親クラスのコンストラクタを呼び出します。

7

オーバーライド
Studentクラスでdisplay()メソッドをオーバーライドし、学籍番号も表示するようにしてください。

回答を表示

模範解答:

public class Student extends Person {
    private String studentId;

    public Student(String name, int age, int genderCode, String studentId) {
        super(name, age, genderCode);
        this.studentId = studentId;
    }

    public String getStudentId() {
        return studentId;
    }

    @Override
    public void display() {
        super.display();
        System.out.println("学籍番号: " + studentId);
    }
}

ポイント: @Overrideアノテーションで親クラスのメソッドを上書きします。super.display()で親クラスのメソッドを呼び出せます。これにより共通処理を再利用できます。

8

動作確認
mainメソッドでPersonとStudentのインスタンスを生成し、それぞれの情報を表示してください。

回答を表示

模範解答:

public class Main {
    public static void main(String[] args) {
        try {
            // Personインスタンスの生成と表示
            Person person = new Person("田中太郎", 25, 1);
            System.out.println("=== Person情報 ===");
            person.display();
            System.out.println();

            // Studentインスタンスの生成と表示
            Student student = new Student("山田花子", 20, 2, "S2024001");
            System.out.println("=== Student情報 ===");
            student.display();
        } catch (IllegalArgumentException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}

// 実行結果:
// === Person情報 ===
// 名前: 田中太郎
// 年齢: 25
// 性別: 男性
//
// === Student情報 ===
// 名前: 山田花子
// 年齢: 20
// 性別: 女性
// 学籍番号: S2024001

ポイント: 継承により、StudentはPersonのフィールドとメソッドを持ちつつ、独自のフィールド（学籍番号）とオーバーライドしたメソッドで機能を拡張しています。これがオブジェクト指向の「再利用性」と「拡張性」です。
