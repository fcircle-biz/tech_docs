人物管理クラス
Personオブジェクトを管理するクラス（PersonManager）を作成してください。登録、一覧表示、名前検索の3つの機能を実装してください。

回答を表示

模範解答:

import java.util.ArrayList;
import java.util.List;

public class PersonManager {
    private List personList;

    public PersonManager() {
        this.personList = new ArrayList<>();
    }

    // 登録
    public void register(Person person) {
        personList.add(person);
        System.out.println(person.getName() + "を登録しました。");
    }

    // 一覧表示
    public void displayAll() {
        if (personList.isEmpty()) {
            System.out.println("登録されている人物はいません。");
            return;
        }
        System.out.println("--- 人物一覧 ---");
        for (Person person : personList) {
            person.display();
        }
    }

    // 名前検索
    public Person findByName(String name) {
        for (Person person : personList) {
            if (person.getName().equals(name)) {
                return person;
            }
        }
        return null;
    }
}

ポイント: ListインターフェースとArrayListクラスを使用してPersonオブジェクトを管理します。コレクションの基本的な操作（追加、反復処理、検索）を実装しています。

役割追加
StudentクラスとEmployeeクラスを追加し、PersonManagerで両方を管理できるようにしてください。StudentとEmployeeそれぞれ固有のフィールド（学籍番号、社員番号など）を追加してください。

回答を表示

模範解答:

// Student.java
public class Student extends Person {
    private String studentId;

    public Student(String name, int age, int genderCode, String studentId) {
        super(name, age, genderCode);
        this.studentId = studentId;
    }

    public String getStudentId() {
        return studentId;
    }

    @Override
    public void display() {
        super.display();
        System.out.println("学籍番号: " + studentId);
    }
}

// Employee.java
public class Employee extends Person {
    private String employeeId;
    private String department;

    public Employee(String name, int age, int genderCode, String employeeId, String department) {
        super(name, age, genderCode);
        this.employeeId = employeeId;
        this.department = department;
    }

    public String getEmployeeId() {
        return employeeId;
    }

    public String getDepartment() {
        return department;
    }

    @Override
    public void display() {
        super.display();
        System.out.println("社員番号: " + employeeId);
        System.out.println("所属部署: " + department);
    }
}

ポイント: 継承を活用することで、PersonManagerはPersonクラス（基底クラス）の型でStudent、Employeeの両方を管理できます。これがポリモーフィズムの基本です。

性別コード変換
性別コードから性別名への変換処理を一元化してください。GenderCodeConverterクラスを作成し、各Personクラスでこれを利用するように修正してください。

模範解答:

// GenderCodeConverter.java
public class GenderCodeConverter {

    public static String toGenderName(int genderCode) {
        switch (genderCode) {
            case 1:
                return "男性";
            case 2:
                return "女性";
            case 9:
                return "その他";
            default:
                return "不明";
        }
    }
}

// Person.java（修正版）
public class Person {
    private String name;
    private int age;
    private int genderCode;

    public Person(String name, int age, int genderCode) {
        this.name = name;
        if (age < 0) {
            throw new IllegalArgumentException("年齢は0以上である必要があります。");
        }
        this.age = age;
        this.genderCode = genderCode;
    }

    public void display() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("性別: " + GenderCodeConverter.toGenderName(genderCode));
    }

    // getterメソッド
    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    public int getGenderCode() {
        return genderCode;
    }
}

ポイント: 変換ロジックを専用のクラスに集約することで、コードの重複を防ぎ、変更時の修正箇所を1箇所に限定できます。これは単一責任の原則（SRP）に従った設計です。

独自例外
PersonManagerの名前検索で人物が見つからなかった場合に、独自例外（PersonNotFoundException）をスローするように変更してください。

模範解答:

// PersonNotFoundException.java
public class PersonNotFoundException extends Exception {
    public PersonNotFoundException(String message) {
        super(message);
    }
}

// PersonManager.java（修正版）
public class PersonManager {
    private List personList;

    public PersonManager() {
        this.personList = new ArrayList<>();
    }

    public void register(Person person) {
        personList.add(person);
        System.out.println(person.getName() + "を登録しました。");
    }

    public void displayAll() {
        if (personList.isEmpty()) {
            System.out.println("登録されている人物はいません。");
            return;
        }
        System.out.println("--- 人物一覧 ---");
        for (Person person : personList) {
            person.display();
            System.out.println("---");
        }
    }

    public Person findByName(String name) throws PersonNotFoundException {
        for (Person person : personList) {
            if (person.getName().equals(name)) {
                return person;
            }
        }
        throw new PersonNotFoundException(name + "という名前の人物は見つかりませんでした。");
    }
}

// 使用例
public class Main {
    public static void main(String[] args) {
        PersonManager manager = new PersonManager();

        manager.register(new Person("田中太郎", 30, 1));
        manager.register(new Student("佐藤花子", 22, 2, "S12345"));

        try {
            Person found = manager.findByName("田中太郎");
            System.out.println("検索結果:");
            found.display();

            Person notFound = manager.findByName("山田太郎");
        } catch (PersonNotFoundException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}

ポイント: 独自例外クラスを作成することで、エラーの種類を明確にし、呼び出し側で適切な処理を行えるようにします。検査例外として定義することで、呼び出し側に例外処理を強制できます。

コレクション選択
PersonManagerの内部実装で、ListではなくMapを使う設計も考えられます。それぞれのメリット・デメリットを説明し、どちらが適切か理由とともに述べてください。

回答を表示

解答例:

List実装（現在の方式）

  • メリット:
  • - 登録順序を保持できる
  • - 同名の人物を複数登録できる
  • - 実装がシンプル
  • デメリット:
  • - 名前検索が線形探索（O(n)）で遅い

Map実装（名前をキーとする）

  • メリット:
  • - 名前検索が高速（O(1)）
  • - 名前の重複チェックが容易
  • デメリット:
  • - 同名の人物を登録できない（キーが重複）
  • - 実装がやや複雑

推奨設計:

人物管理の要件として「同名の人物を区別する必要がある」「一意の識別子（ID）が存在する」という前提であれば、ID（学籍番号、社員番号など）をキーとしたMapを使う設計が適切です。名前はあくまで検索条件の一つとして扱い、複数ヒットする可能性を考慮すべきです。

// Map実装例（IDベース）
public class PersonManager {
    private Map personMap;  // キー: ID

    public PersonManager() {
        this.personMap = new HashMap<>();
    }

    public void register(String id, Person person) {
        if (personMap.containsKey(id)) {
            System.out.println("エラー: ID " + id + " は既に登録されています。");
            return;
        }
        personMap.put(id, person);
        System.out.println(person.getName() + "（ID: " + id + "）を登録しました。");
    }

    public Person findById(String id) throws PersonNotFoundException {
        Person person = personMap.get(id);
        if (person == null) {
            throw new PersonNotFoundException("ID " + id + " の人物は見つかりませんでした。");
        }
        return person;
    }

    public List findByName(String name) {
        List results = new ArrayList<>();
        for (Person person : personMap.values()) {
            if (person.getName().equals(name)) {
                results.add(person);
            }
        }
        return results;
    }
}

ポイント: コレクションの選択は、データの特性や操作の頻度によって決まります。一意性が保証されるキーが存在し、高速な検索が必要ならMap、順序保持や重複許容が必要ならListを選択します。
