拡張前提設計
Teacher（教員）とAdmin（管理者）の役割を追加できるように、既存設計を見直してください。Student / Employee と同じようにPersonを継承する想定です。

回答を表示

設計方針:

  • • Personクラスを抽象クラスにし、共通処理をテンプレートメソッドパターンで定義
  • • 役割ごとの固有処理は抽象メソッドとして子クラスで実装
  • • 管理クラスでは役割の型ではなくPerson型のListで保持
  • • 検索条件として「役割」を追加（instanceof で判定）
// 抽象クラス化
public abstract class Person {
    private String name;
    private int age;
    private int genderCode;

    // 共通処理
    public final void displayInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("性別: " + getGenderName());
        displayRoleInfo(); // 役割固有情報
    }

    // 子クラスで実装
    protected abstract void displayRoleInfo();
}

// 拡張例: Teacher
public class Teacher extends Person {
    private String subject; // 担当科目

    @Override
    protected void displayRoleInfo() {
        System.out.println("担当科目: " + subject);
    }
}

ポイント: Open-Closed原則（拡張に開いて、修正に閉じる）を意識します。新しい役割を追加するときに既存コードを変更しない設計を目指します。

検索条件追加
以下の検索条件を追加してください：
• 年齢範囲（minAge 〜 maxAge）
• 役割（Student / Employee / Teacher / Admin）

回答を表示

模範解答:

public class PersonManager {
    private List persons = new ArrayList<>();

    // 年齢範囲で検索
    public List searchByAgeRange(int minAge, int maxAge) {
        List result = new ArrayList<>();
        for (Person p : persons) {
            if (p.getAge() >= minAge && p.getAge() <= maxAge) {
                result.add(p);
            }
        }
        return result;
    }

    // 役割で検索
    public List searchByRole(Class roleClass) {
        List result = new ArrayList<>();
        for (Person p : persons) {
            if (roleClass.isInstance(p)) {
                result.add(p);
            }
        }
        return result;
    }

    // 複合検索（メソッドチェーン可能にする設計も検討）
    public List search(Integer minAge, Integer maxAge,
                               Class roleClass) {
        List result = persons;

        // 年齢範囲フィルタ
        if (minAge != null || maxAge != null) {
            result = filterByAge(result, minAge, maxAge);
        }

        // 役割フィルタ
        if (roleClass != null) {
            result = filterByRole(result, roleClass);
        }

        return result;
    }
}

ポイント: 検索条件をnull許容にすることで、条件の組み合わせに柔軟に対応できます。より高度な設計としては、Specificationパターンの適用も検討できます。

// 表示専用クラス（Single Responsibility Principle）
public class PersonDisplayer {
    public void display(Person person) {
        System.out.println("=== 人物情報 ===");
        System.out.println("名前: " + person.getName());
        System.out.println("年齢: " + person.getAge());
        System.out.println("性別: " + GenderConverter.toName(person.getGenderCode()));

        // 役割固有情報
        displayRoleSpecificInfo(person);
    }

    private void displayRoleSpecificInfo(Person person) {
        if (person instanceof Student) {
            Student s = (Student) person;
            System.out.println("学籍番号: " + s.getStudentId());
        } else if (person instanceof Employee) {
            Employee e = (Employee) person;
            System.out.println("社員番号: " + e.getEmployeeId());
        }
        // 以降、Teacher / Adminも同様
    }

    public void displayList(List persons) {
        for (int i = 0; i < persons.size(); i++) {
            System.out.println((i + 1) + "人目:");
            display(persons.get(i));
            System.out.println();
        }
    }
}

// Personクラスからは表示処理を削除
public abstract class Person {
    private String name;
    private int age;
    private int genderCode;

    // getter/setterのみ、表示処理は持たない
}

ポイント: 単一責任の原則（SRP）に従い、表示ロジックをPersonから分離します。これにより表示形式の変更がPersonクラスに影響しなくなります。

表示形式切り替え
画面表示とCSV出力を切り替えられるように設計してください。Strategyパターンまたはインターフェース実装で実現します。

回答を表示

模範解答:

// 出力形式のインターフェース
public interface PersonFormatter {
    String format(Person person);
    String formatList(List persons);
}

// コンソール出力形式
public class ConsoleFormatter implements PersonFormatter {
    @Override
    public String format(Person person) {
        StringBuilder sb = new StringBuilder();
        sb.append("名前: ").append(person.getName()).append("\n");
        sb.append("年齢: ").append(person.getAge()).append("\n");
        sb.append("性別: ").append(GenderConverter.toName(person.getGenderCode())).append("\n");
        return sb.toString();
    }

    @Override
    public String formatList(List persons) {
        StringBuilder sb = new StringBuilder();
        for (Person p : persons) {
            sb.append(format(p)).append("\n");
        }
        return sb.toString();
    }
}

// CSV出力形式
public class CsvFormatter implements PersonFormatter {
    @Override
    public String format(Person person) {
        return String.format("%s,%d,%d",
            person.getName(),
            person.getAge(),
            person.getGenderCode());
    }

    @Override
    public String formatList(List persons) {
        StringBuilder sb = new StringBuilder();
        sb.append("名前,年齢,性別コード\n"); // ヘッダー
        for (Person p : persons) {
            sb.append(format(p)).append("\n");
        }
        return sb.toString();
    }
}

// 使用例
PersonFormatter formatter = new CsvFormatter(); // 切り替え可能
String result = formatter.formatList(persons);
System.out.println(result);

ポイント: Strategyパターンにより、出力形式の切り替えをクライアント側で自由に行えます。新しい形式（JSON、XMLなど）の追加も容易です。

Stream API
一覧表示・検索処理をStream APIを使って実装し直してください。従来のforループと比較して可読性やメンテナンス性を評価します。

回答を表示

模範解答:

import java.util.stream.Collectors;

public class PersonManager {
    private List persons = new ArrayList<>();

    // 名前で検索（部分一致）
    public List searchByName(String keyword) {
        return persons.stream()
                .filter(p -> p.getName().contains(keyword))
                .collect(Collectors.toList());
    }

    // 年齢範囲で検索
    public List searchByAgeRange(int minAge, int maxAge) {
        return persons.stream()
                .filter(p -> p.getAge() >= minAge && p.getAge() <= maxAge)
                .collect(Collectors.toList());
    }

    // 役割で検索
    public List searchByRole(Class roleClass) {
        return persons.stream()
                .filter(roleClass::isInstance)
                .collect(Collectors.toList());
    }

    // 複合検索
    public List search(String keyword, Integer minAge, Integer maxAge,
                               Class roleClass) {
        return persons.stream()
                .filter(p -> keyword == null || p.getName().contains(keyword))
                .filter(p -> minAge == null || p.getAge() >= minAge)
                .filter(p -> maxAge == null || p.getAge() <= maxAge)
                .filter(p -> roleClass == null || roleClass.isInstance(p))
                .collect(Collectors.toList());
    }

    // 年齢の平均を計算
    public double getAverageAge() {
        return persons.stream()
                .mapToInt(Person::getAge)
                .average()
                .orElse(0.0);
    }

    // 名前のリストを取得
    public List getAllNames() {
        return persons.stream()
                .map(Person::getName)
                .sorted()
                .collect(Collectors.toList());
    }
}

ポイント: Stream APIにより、宣言的で読みやすいコードになります。filter / map / collectなどのメソッドチェーンで処理を組み立てます。ラムダ式やメソッド参照も活用しましょう。
