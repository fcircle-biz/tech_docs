タスクを登録・編集しよう - フォームの作成

新しいタスクを登録する画面と、既存のタスクを編集する画面を作成します。

このステップで実装すること

  • フォームウィジェットを使ってデータ入力画面を作る
  • 新規タスク登録画面を作成する
  • 既存タスク編集画面を作成する
  • Input Parameterを使って画面間でデータを受け渡す
  • 一覧画面から編集画面へのリンクを作成する

前提条件

  • Step 4でTaskエンティティを作成済み
  • Step 5でタスク一覧画面を作成済み
  • アグリゲートの使い方を理解している

6.1 フォームとは？

フォームは、ユーザーがデータを入力・編集するための画面部品です。 Excelでいうと「入力用のセル」、Wordでいうと「入力欄」に相当します。

OutSystemsでは、Formウィジェットを使うことで、簡単にデータ入力画面を作成できます。 タスク名や説明を入力する欄、保存ボタン、キャンセルボタンなどをまとめて配置できます。

フォームの役割

新規作成: 空のフォームでデータを入力 → 保存ボタンで新しいデータが登録される

編集: 既存データが入ったフォームで内容を変更 → 保存ボタンで更新される

6.2 タスク詳細画面の作成

新規登録と編集の両方に使える「TaskDetail」画面を作成します。 この画面は、タスクIDが渡されたら編集モード、渡されなかったら新規作成モードとして動作します。

実装 6-1: TaskDetail画面を作成する

新規登録・編集の両方に対応した画面を作成します。

1

新しいスクリーンを作成

Interface タブ → MainFlow を右クリック → 「Add Screen」を選択

  • テンプレート: Emptyを選択
  • Screen name: TaskDetail
2

Input Parameterを追加

TaskDetail画面で、Input Parameterを追加します（これは画面に渡す値の受け皿です）:

  • 画面左のElement Treeで「TaskDetail」を選択
  • 右側のPropertiesパネルで「Input Parameters」の「+」をクリック
  • Name: TaskId
  • Data Type: Task Identifier
  • Is Mandatory: No（チェックを外す）
Is Mandatory を No にする理由: 新規作成時はTaskIdが存在しない（渡されない）ため、省略可能にする必要があります。
3

アグリゲートでタスクデータを取得

編集時に既存データを表示するため、TaskIdをもとにデータを取得します:

  • 画面左のElement Treeで「TaskDetail」を右クリック → 「Fetch Data from Database」
  • Dataタブの「Entities」→「Database」→「Task」をAggregateにドラッグ
  • Aggregateの名前をGetTaskByIdに変更
  • Filterを追加: Task.Id = TaskId

6.3 フォームウィジェットの配置

TaskDetail画面にフォームウィジェットを配置して、タスク名と説明を入力できるようにします。

実装 6-2: フォームを配置する

データ入力欄と保存ボタンを配置します。

1

Formウィジェットを配置

Toolbox → Widgets → Form → FormをTaskDetail画面にドラッグ

Formの名前をTaskFormに変更

2

タイトル入力欄を追加

Toolbox → Widgets → Input → InputをTaskFormの中にドラッグ

  • Variable: 新規作成でTaskTitle
  • Prompt: 「タスク名を入力してください」

Inputの前にLabelウィジェットを配置して、「タスク名」と表示させましょう。

3

説明入力欄を追加

Toolbox → Widgets → Input → Text AreaをTaskFormの中にドラッグ

  • Variable: 新規作成でTaskDescription
  • Prompt: 「詳細を入力してください」

Text Areaの前にもLabelウィジェットを配置して、「説明」と表示させましょう。

4

保存ボタンを配置

Toolbox → Widgets → Button → ButtonをTaskFormの中にドラッグ

  • Text: 「保存」
  • Name: SaveButton
5

キャンセルボタンを配置

もう1つButtonをTaskFormの中にドラッグ

  • Text: 「キャンセル」
  • Name: CancelButton
  • On Click: 新しいアクションを作成（後で設定）

6.4 保存処理の実装

保存ボタンをクリックしたときの処理を作成します。 OutSystemsには「CreateOrUpdate」というアクションがあり、新規作成と更新の両方を自動的に判定してくれます。

実装 6-3: 保存アクションを作成する

データをデータベースに保存する処理を作成します。

1

保存ボタンのOn Clickアクションを作成

SaveButtonを選択 → PropertiesのOn Clickで「New Client Action」を選択

アクション名をSaveTaskに変更

2

CreateOrUpdateTaskアクションを配置

Dataタブ → Entities → Database → Taskを展開

CreateOrUpdateTaskをフローにドラッグ

これが新規作成・更新を自動判定してくれるアクションです。

3

Sourceパラメータを設定

CreateOrUpdateTaskのSourceパラメータをクリックして、以下を設定:

Expression 
If(TaskId = NullIdentifier(),
    Task{
        Title: TaskTitle,
        Description: TaskDescription,
        IsCompleted: False,
        CreatedDate: CurrDateTime()
    },
    Task{
        Id: TaskId,
        Title: TaskTitle,
        Description: TaskDescription
    }
)
説明: TaskIdが空（新規作成）の場合は全項目を設定し、TaskIdがある（編集）の場合はIDと変更項目だけを設定します。
4

成功メッセージと一覧画面への遷移

Toolbox → Logic → Message → Feedback Messageをフローにドラッグ

  • Message: 「タスクを保存しました」
  • Type: Success

次に、Toolbox → Logic → Destination → Destinationをフローにドラッグ

  • Destination: Home（タスク一覧画面）

キャンセルボタンの処理

CancelButtonのOn Clickには、単純にDestinationでHome画面に戻る処理を設定しましょう。保存せずに一覧に戻ります。

6.5 一覧画面からのリンク作成

一覧画面（Home）から新規作成画面・編集画面へ移動できるようにします。

実装 6-4: 一覧画面にボタンを追加する

新規作成ボタンと編集ボタンを配置します。

1

新規作成ボタンを配置

Home画面にButtonウィジェットを配置

  • Text: 「新規作成」
  • On Click: DestinationでTaskDetailを選択
  • TaskIdパラメータ: NullIdentifier()（空のIDを渡す）
2

各タスクに編集ボタンを追加

タスクのリスト（ListRecords）内の各行に、編集用のButtonまたはLinkを配置

  • Text: 「編集」
  • On Click: DestinationでTaskDetailを選択
  • TaskIdパラメータ: GetTasks.List.Current.Task.Id（現在の行のTaskId）

6.6 編集時の初期値設定

編集モードで画面を開いたとき、既存データがフォームに表示されるようにします。

実装 6-5: 初期値を設定する

画面表示時に既存データをフォームにセットします。

1

OnReadyアクションを作成

画面左のElement Treeで「TaskDetail」を選択 → PropertiesのOn Readyで「New Client Action」

これは画面表示時に自動実行される処理です。

2

条件分岐を追加

Toolbox → Logic → If をフローにドラッグ

  • Condition: TaskId <> NullIdentifier()
  • 意味: TaskIdが渡されている（編集モード）かを判定
3

Assignで初期値を設定

Ifの「True」側に、Toolbox → Logic → Assignをドラッグ

Assignで以下の代入を設定:

  • TaskTitle = GetTaskById.List.Current.Task.Title
  • TaskDescription = GetTaskById.List.Current.Task.Description

これで、編集モードで開いたときに既存のタスク名と説明が表示されます。

6.7 動作確認

アプリを公開して動作を確認

1-Click Publishでアプリを公開し、以下の動作を確認しましょう。

確認すべきポイント

  • 一覧画面に「新規作成」ボタンが表示されている
  • 「新規作成」をクリックすると、空のフォームが表示される
  • タスク名と説明を入力して「保存」すると、データが登録される
  • 保存後、成功メッセージが表示され、一覧画面に戻る
  • 各タスクに「編集」ボタンが表示されている
  • 「編集」をクリックすると、既存データが入ったフォームが表示される
  • 内容を変更して「保存」すると、データが更新される
  • 「キャンセル」をクリックすると、保存せずに一覧画面に戻る

トラブルシューティング

編集画面でデータが表示されない

原因: OnReadyアクションでの初期値設定が正しくない、またはアグリゲートのFilterが正しくない可能性があります。

解決方法:

  • GetTaskByIdアグリゲートのFilterがTask.Id = TaskIdになっているか確認
  • OnReadyアクションでAssignが正しく設定されているか確認
  • 編集ボタンのDestinationでTaskIdパラメータにGetTasks.List.Current.Task.Idを渡しているか確認
保存ボタンを押してもデータが保存されない

原因: CreateOrUpdateTaskアクションのSourceパラメータが正しく設定されていない可能性があります。

解決方法:

  • SaveTaskアクションのフローを確認（CreateOrUpdateTask → Feedback Message → Destinationの順）
  • CreateOrUpdateTaskのSourceパラメータのIf文が正しいか確認
  • TaskTitle、TaskDescription変数がInputウィジェットと正しく紐付いているか確認
新規作成ボタンをクリックするとエラーが出る

原因: Input ParameterのTaskIdがMandatory（必須）になっている可能性があります。

解決方法:

  • TaskDetail画面のTaskId Input Parameterの「Is Mandatory」をNoに設定
  • 新規作成ボタンのDestinationでTaskIdにNullIdentifier()を渡しているか確認

ステップ6のまとめ

このステップでは、フォームを使ったデータ入力画面を作成しました。

  • Formウィジェット: データ入力欄をまとめて管理できる便利な部品
  • Input Parameter: 画面間でデータを受け渡す仕組み（TaskIdを渡して編集対象を指定）
  • CreateOrUpdateアクション: 新規作成と更新を自動判定してくれる便利なアクション
  • OnReadyアクション: 画面表示時に自動実行される処理（初期値設定に使用）

次のステップでは、タスクを削除する機能と、完了/未完了を切り替える機能を追加します。 アプリの基本機能がすべて揃います。
