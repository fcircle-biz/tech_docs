データを保存する場所を作ろう - エンティティの設計

タスクのデータを保存するための「エンティティ」(データベースのテーブル)を作成します。どんな情報を保存するか設計しましょう。

このステップで実装すること

  • エンティティ(データベースのテーブル)とは何かを理解する
  • タスクエンティティを作成する
  • アトリビュート(属性)を追加する(Title, Description, IsCompleted, CreatedDate)
  • データ型を適切に選択する

前提条件

  • ステップ1〜3を完了していること
  • タスク管理アプリケーションのモジュールが開かれていること

4.1 エンティティとは？

アプリケーションを作るとき、ユーザーが入力したデータをどこかに保存する必要があります。 そのための「データの入れ物」がエンティティです。

Excelの表で考えてみよう

エンティティは、Excelの「表」のようなものです。

  • エンティティ = Excelの「シート」や「テーブル」全体
  • アトリビュート(属性) = Excelの「列」(タイトル、説明、完了フラグなど)
  • レコード = Excelの「行」(1つのタスクのデータ)

たとえば、タスク管理アプリでは「Task」というエンティティを作り、そこに各タスクの情報を保存します。

エンティティ データを保存する入れ物 Task (タスクエンティティ)
アトリビュート エンティティに保存する項目 Title, Description, IsCompleted
データ型 項目に保存できるデータの種類 Text (文字)、Boolean (はい/いいえ)など
レコード エンティティに保存された1つのデータ 「買い物に行く」というタスク1件分

4.2 タスクエンティティの設計

タスク管理アプリで保存したい情報を整理しましょう。各タスクについて、以下の情報を保存します。

項目名(日本語) 英語名 データ型 説明
ID Id Long Integer 各タスクを区別するための番号(自動で付与)
タスク名 Title Text (50) タスクの名前(必須、50文字まで)
説明 Description Text (500) タスクの詳しい説明(任意、500文字まで)
完了フラグ IsCompleted Boolean 完了したかどうか(初期値：False)
作成日時 CreatedDate Date Time タスクを作成した日時(自動で設定)

データ型の選び方

  • Text: 文字を保存する(名前、説明など)
  • Boolean: はい/いいえ、True/Falseを保存する(完了/未完了など)
  • Date Time: 日付と時刻を保存する(作成日時など)
  • Long Integer: 数字を保存する(ID、個数など)

4.3 Taskエンティティの作成

Service Studioを使って、実際にTaskエンティティを作成しましょう。

実装 4-1: Taskエンティティの作成

データを保存するエンティティを作成します。

1

Service Studioでモジュールを開く

Service Studioを起動し、これまで作成してきたタスク管理アプリのモジュールを開いてください。

2

Dataタブを開く

画面左側のツリー表示で、「Data」タブをクリックします。 ここがデータを扱うエリアです。

3

Database配下のEntitiesを右クリック

Dataタブの中にある「Database」を展開し、「Entities」フォルダを右クリックします。

表示されるメニューから「Add Entity」を選択してください。

4

エンティティ名を入力

新しいエンティティが作成され、名前を入力できる状態になります。

エンティティ名 
Task

と入力して、Enterキーを押してください。

5

エンティティが作成されたことを確認

Dataタブの「Entities」配下に「Task」エンティティが表示されます。 このエンティティには、デフォルトで「Id」というアトリビュートが自動的に作成されます。

注意: Idは自動生成される

「Id」というアトリビュートは、OutSystemsが自動的に追加します。 これは各レコード(各タスク)を区別するためのユニークな番号で、システムが自動で番号を割り振ってくれます。 削除したり変更したりしないでください。

4.4 アトリビュート(属性)の追加

Taskエンティティに、必要なアトリビュートを追加していきましょう。

実装 4-2: アトリビュートの追加

Taskエンティティに4つのアトリビュートを追加します。

1

Title アトリビュートを追加

Dataタブで「Task」エンティティをダブルクリックして開きます。 画面中央にエンティティの詳細が表示されます。

Attributesエリアで右クリックし、「Add Attribute」を選択します。

以下のように入力してください:

  • Name: Title
  • Data Type: Text
  • Length: 50
  • Is Mandatory: チェックを入れる (必須項目にする)
2

Description アトリビュートを追加

同じ手順で2つ目のアトリビュートを追加します。

  • Name: Description
  • Data Type: Text
  • Length: 500
  • Is Mandatory: チェックを入れない (任意項目)
3

IsCompleted アトリビュートを追加

完了/未完了を表すBoolean型のアトリビュートを追加します。

  • Name: IsCompleted
  • Data Type: Boolean
  • Default Value: False (新しいタスクは未完了)
4

CreatedDate アトリビュートを追加

タスクが作成された日時を保存するアトリビュートを追加します。

  • Name: CreatedDate
  • Data Type: Date Time
  • Default Value: CurrDateTime() (現在の日時を自動設定)
5

全アトリビュートを確認

Taskエンティティに以下の5つのアトリビュートが表示されていることを確認してください:

  • Id (Long Integer) - 自動生成
  • Title (Text 50) - 必須
  • Description (Text 500) - 任意
  • IsCompleted (Boolean) - デフォルト: False
  • CreatedDate (Date Time) - デフォルト: CurrDateTime()

デフォルト値(初期値)とは？

デフォルト値は、新しいレコードを作成したときに自動的に設定される値です。 たとえば、IsCompletedのデフォルト値をFalseにしておくと、新しいタスクは自動的に「未完了」になります。 CurrDateTime()は「現在の日時」を返す関数で、タスク作成時の日時を自動記録できます。

4.5 エンティティの公開

エンティティを作成したら、データベースに反映させるために公開(Publish)します。

実装 4-3: エンティティの公開

作成したエンティティをデータベースに反映させます。

1

1-Click Publishを実行

Service Studioのツールバーにある「1-Click Publish」ボタンをクリックします。 (青い丸に上向き矢印のアイコン、または「Publish」と書かれたボタンです)

キーボードショートカット: F5

2

公開プロセスを待つ

Service Studioが自動的にアプリをビルドし、クラウド環境に公開します。 下部のログウィンドウに進捗が表示されます。 「Done!」と表示されれば成功です。

3

データベースを確認

公開が完了すると、OutSystemsのクラウド環境のデータベースに「Task」テーブルが作成されます。 これでタスクのデータを保存する準備ができました。

4.6 動作確認

エンティティの確認

Taskエンティティが正しく作成できているか確認しましょう。

確認すべきポイント

  • Dataタブの「Entities」配下に「Task」エンティティが表示されている
  • Taskエンティティに5つのアトリビュート(Id, Title, Description, IsCompleted, CreatedDate)が含まれている
  • 各アトリビュートのデータ型が正しく設定されている
  • 1-Click Publishが成功し、「Done!」と表示されている

トラブルシューティング

「Add Entity」が見つからない

原因: 間違った場所を右クリックしている可能性があります。

解決方法:

  • Dataタブが開いていることを確認してください
  • 「Database」→「Entities」フォルダを右クリックしてください
  • エンティティ名ではなく、「Entities」というフォルダ自体を右クリックしてください
アトリビュートが追加できない

原因: エンティティが開かれていない、または正しいエリアをクリックしていない可能性があります。

解決方法:

  • Taskエンティティをダブルクリックして、詳細画面を開いてください
  • 「Attributes」と書かれたエリアを右クリックしてください
  • メニューから「Add Attribute」を選択してください
Publishがエラーになる

原因: 必須項目が設定されていない、またはデータ型が不正な可能性があります。

解決方法:

  • エラーメッセージを確認してください(画面下部のログウィンドウ)
  • Titleアトリビュートの「Is Mandatory」にチェックが入っているか確認してください
  • 各アトリビュートのデータ型が正しいか確認してください
  • それでも解決しない場合は、エンティティを削除して最初から作り直してみてください
CurrDateTime()と入力してもエラーになる

原因: データ型がDate Timeになっていない、またはスペルミスの可能性があります。

解決方法:

  • CreatedDateのData TypeがDate Timeになっているか確認してください
  • CurrDateTime()は正確に入力してください(大文字小文字を区別します)
  • Default Valueの欄に入力してください(他の欄ではありません)

ステップ完了チェック

以下の項目が完了していることを確認してください:
