データを保存する場所を作ろう - エンティティの設計

タスクのデータを保存するための「エンティティ」（データベースのテーブル）を作成します。どんな情報を保存するか設計しましょう。

このステップで学ぶこと

  • エンティティとは何か（データを保存する入れ物）
  • アトリビュート（属性）とは何か（保存する項目）
  • タスクエンティティの設計（Id、Title、Description、IsCompleted、CreatedDate）
  • データ型の選び方（文字、数字、日付、はい/いいえ など）
  • エンティティの作成手順

4.1 エンティティとは？

エンティティとは、データを保存するための「入れ物」です。 Excelでいうところの「シート」や「テーブル」に相当します。 OutSystemsでは、エンティティを作成することで、アプリケーションで扱うデータを保存・管理できるようになります。

データベースとエンティティの関係

OutSystemsでは、エンティティを作成すると、自動的にデータベースにテーブルが作成されます。 プログラミング言語でSQLを書く必要はありません。視覚的な操作だけで、データベースを構築できます。

身近な例で理解しよう

Excelで「タスク管理表」を作る場面を想像してください：

  • シート名: 「タスク」（これがエンティティ）
  • 列見出し: 「タスク名」「説明」「完了」「作成日」（これがアトリビュート）
  • データ行: 実際のタスクの内容（これがレコード）

4.2 アトリビュート（属性）とは？

アトリビュートは、エンティティに保存する個々の項目です。 Excelの「列」に相当します。例えば、タスク管理アプリでは、以下のような項目が必要です：

項目名（日本語） 英語名 データ型 説明
ID Id 自動番号 各タスクを区別するための番号（自動で付与）
タスク名 Title テキスト タスクの名前（必須、50文字まで）
説明 Description 長いテキスト タスクの詳しい説明（任意、500文字まで）
完了フラグ IsCompleted はい/いいえ 完了したかどうか（初期値：いいえ）
作成日時 CreatedDate 日時 タスクを作成した日時（自動で設定）

データ型とは？

データ型は、保存するデータの種類を指定します。OutSystemsでは、以下のような基本的なデータ型が用意されています：

  • Text: 文字列（短いテキスト）
  • Long Text: 長い文字列（長いテキスト）
  • Integer: 整数（1, 2, 3...）
  • Boolean: はい/いいえ（True/False）
  • Date Time: 日付と時刻

4.3 タスクエンティティの作成

実装 4-1: Taskエンティティの作成

Service Studioで新しいエンティティを作成し、タスクデータを保存できるようにします。

1

Service Studioを開く

前のステップで作成したアプリケーションをService Studioで開きます。 左側のツールボックスから「Data」タブをクリックします。

2

新しいエンティティを追加

画面左側の「Data」タブ内で、「Database」フォルダを右クリックし、 「Add Entity」を選択します。

Database → 右クリック → Add Entity

3

エンティティ名を設定

新しく作成されたエンティティの名前をTaskに変更します。

エンティティ名: Task

4

アトリビュートを追加

Taskエンティティをダブルクリックして、詳細画面を開きます。 デフォルトでIdアトリビュートが作成されていますが、 さらに以下のアトリビュートを追加します：

アトリビュート 1: Title

  • Name: Title
  • Data Type: Text
  • Length: 50
  • Is Mandatory: Yes（チェックを入れる）

アトリビュート 2: Description

  • Name: Description
  • Data Type: Text (2000)
  • Is Mandatory: No

アトリビュート 3: IsCompleted

  • Name: IsCompleted
  • Data Type: Boolean
  • Default Value: False
  • Is Mandatory: Yes

アトリビュート 4: CreatedDate

  • Name: CreatedDate
  • Data Type: Date Time
  • Default Value: CurrDateTime()（現在日時を返す関数）
  • Is Mandatory: Yes
5

保存してPublish

エンティティの設定が完了したら、Ctrl + Sで保存し、 「1-Click Publish」ボタンをクリックして変更を反映します。

ヒント: Is Mandatoryとは？

「Is Mandatory」にチェックを入れると、そのアトリビュートは必須項目になります。 つまり、データを保存する際に必ず値を入力しなければなりません。 タスク名（Title）は必須、説明（Description）は任意としています。

4.4 作成したエンティティの確認

確認方法

  1. 1 Service Studioの左側「Data」タブ内の「Database」フォルダを開く
  2. 2 「Task」エンティティが表示されていることを確認
  3. 3 Taskエンティティをダブルクリックして、5つのアトリビュート（Id、Title、Description、IsCompleted、CreatedDate）が存在することを確認

期待される結果

Taskエンティティの詳細画面で、以下の5つのアトリビュートが表示されていれば成功です：

  • Id（自動番号）
  • Title（テキスト、必須）
  • Description（長いテキスト、任意）
  • IsCompleted（はい/いいえ、デフォルトFalse）
  • CreatedDate（日時、デフォルトCurrDateTime()）

4.5 データ型の選び方

データ型 使う場面 例
Text 短い文字列を保存したいとき 名前、タイトル、メールアドレス
Text (2000) 長い文字列を保存したいとき 説明、コメント、本文
Integer 整数を保存したいとき 年齢、個数、順序番号
Decimal 小数を保存したいとき 金額、割合、評価スコア
Boolean はい/いいえ、オン/オフを保存したいとき 完了フラグ、有効/無効、公開/非公開
Date Time 日付と時刻を保存したいとき 作成日時、更新日時、予定日時
Date 日付のみを保存したいとき 誕生日、期限日

データ型選びのコツ

迷ったら、より大きめのデータ型を選んでおくのが安全です。 例えば、短い文字列でも後から長くなる可能性があるなら、最初から Text (2000) にしておくと良いでしょう。

トラブルシューティング

エンティティが作成できない

原因: 操作権限がない、または既に同じ名前のエンティティが存在する

解決方法:

  • Personal Environmentに正しくログインしているか確認
  • 既存のエンティティと名前が重複していないか確認
  • Service Studioを再起動してみる
アトリビュートの設定がわからない

原因: どの項目をどう設定すればいいか迷っている

解決方法:

  • 上記の表を参考に、各アトリビュートの設定を確認
  • 「Is Mandatory」は、必須項目にしたい場合のみチェック
  • 「Default Value」は、デフォルトで設定したい値がある場合のみ入力
  • まずは基本設定だけでOK、後から変更も可能です
Publishでエラーが出る

原因: エンティティやアトリビュートの設定に問題がある

解決方法:

  • エラーメッセージを確認し、指摘されている箇所を修正
  • アトリビュート名に日本語やスペースを使っていないか確認（英語で入力）
  • Is Mandatoryをチェックしているのに、Default Valueが設定されていない場合は設定する
