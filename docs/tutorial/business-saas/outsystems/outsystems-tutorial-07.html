タスクを編集・削除しよう - 更新と削除機能

登録済みのタスクを編集したり、不要なタスクを削除したりする機能を追加します。

このステップで学ぶこと

  • 編集画面への遷移方法
  • パラメータ（データの受け渡し）の使い方
  • 既存データの読み込みと表示
  • データの更新処理
  • 削除機能の実装
  • 確認ダイアログの表示
  • 完了/未完了の切り替え機能

7.1 パラメータとデータの受け渡し

タスクを編集するには、「どのタスクを編集するか」という情報を一覧画面から編集画面へ渡す必要があります。 OutSystemsでは、パラメータ（Input Parameter）を使って画面間でデータを受け渡します。

パラメータとは？

画面やアクションに情報を渡すための「入れ物」です。 例えば、「タスクID = 5」という情報を渡すことで、編集画面では「ID:5のタスク」を表示できます。

  • Input Parameter（入力パラメータ）: 外部から受け取る値
  • データ型: パラメータには型があります（Integer、Text、Task Identifierなど）

7.2 編集画面の作成

実装 7-1: TaskDetail画面にInput Parameterを追加

既存のTaskDetail画面（Step 6で作成）にInput Parameterを追加して、タスクIDを受け取れるようにします。

1

TaskDetail画面を開く

Service StudioのInterface タブで、TaskDetail画面をダブルクリックして開きます。

2

Input Parameterを追加

画面のプロパティパネルで「Input Parameters」を展開し、右クリックして「Add Input Parameter」を選択します。

  • Name: TaskId
  • Data Type: Task Identifier
  • Is Mandatory: No（新規作成時はIDがないため）
3

Aggregateを追加してタスクを取得

画面のPreparationセクションで、新しいAggregateを追加します。

  • Name: GetTaskById
  • Taskエンティティを追加
  • フィルター追加: Task.Id = TaskId（入力パラメータ）
4

フォームのデータソースを変更

Formウィジェットのプロパティで、Data Sourceを GetTaskById.List.Current に変更します。 これで、取得したタスクのデータがフォームに自動的に表示されます。

ヒント

TaskIdがNullOrEmpty（空）の場合は新規作成、値がある場合は編集という動作になります。 この仕組みで、同じ画面を新規作成と編集の両方で使えます。

7.3 一覧画面から編集画面へのリンク

実装 7-2: 編集ボタンの追加と画面遷移

Home画面のタスク一覧に編集ボタンを追加し、クリックしたときにTaskDetail画面へ遷移するようにします。

1

Home画面を開く

タスク一覧を表示しているHome画面を開きます。

2

Linkウィジェットを追加

ListのItem内（各タスク行）に、Linkウィジェットをドラッグ&ドロップします。

  • Label: 「編集」
  • Icon: fas fa-edit（オプション）
3

リンク先を設定

Linkのプロパティで、On Click → Destination を TaskDetail に設定します。

4

TaskIdパラメータを渡す

Destinationを設定すると、TaskIdパラメータが表示されます。 ここに GetTasks.List.Current.Task.Id をバインドします。

これで、クリックしたタスクのIDが編集画面に渡されます。

動作確認

  1. 1 1-Click Publishでアプリを公開
  2. 2 タスク一覧で「編集」リンクをクリック
  3. 3 編集画面に遷移し、既存のタスクデータがフォームに表示されることを確認

期待される結果

編集画面のフォームに、選択したタスクの既存データ（タイトル、説明など）が自動的に入力された状態で表示されます。

7.4 削除機能の実装

実装 7-3: タスク削除アクションの作成

タスクを削除するボタンとアクションを作成します。削除前に確認ダイアログを表示することで、誤削除を防ぎます。

1

削除ボタンを追加

Home画面のタスク一覧に、削除用のButtonまたはLinkを追加します。

  • Label: 「削除」
  • Icon: fas fa-trash（オプション）
  • Style: 危険な操作なので赤系の色を推奨
2

Client Actionを作成

削除ボタンのOn Clickで、新しいClient Actionを作成します。

  • Name: DeleteTaskAction
3

確認ダイアログを追加

アクションフローに「Confirm」ロジックを追加します（Logicタブ → Confirmation）。

  • Message: 「本当にこのタスクを削除しますか？」
4

DeleteTaskアクションを追加

確認ダイアログの「Yes」分岐に、DeleteTaskアクション（Server Action）を追加します。

  • Source: GetTasks.List.Current.Task.Id
5

Refreshでリストを更新

削除後、Refresh Data → GetTasks を追加して、一覧を最新状態に更新します。

6

成功メッセージを表示

Messageロジックを追加して、「タスクを削除しました」というメッセージを表示します。

注意

削除は元に戻せない操作です。確認ダイアログを必ず表示するようにしましょう。 本番システムでは、「論理削除」（データを実際には削除せず、削除フラグを立てる）という手法もよく使われます。

7.5 完了/未完了の切り替え機能

実装 7-4: チェックボックスで完了状態を切り替え

タスク一覧にチェックボックスを追加し、クリックするだけで完了/未完了を切り替えられるようにします。

1

Checkboxウィジェットを追加

タスク一覧のList内に、Checkboxウィジェットをドラッグ&ドロップします。

  • Variable: GetTasks.List.Current.Task.IsCompleted
2

On Change Actionを作成

CheckboxのOn Changeイベントで、新しいClient Actionを作成します。

  • Name: ToggleTaskCompletionAction
3

CreateOrUpdateTaskを実行

アクションフローに、CreateOrUpdateTask（Server Action）を追加します。

  • Source: GetTasks.List.Current.Task

これで、チェックボックスの変更がデータベースに即座に反映されます。

4

（オプション）完了タスクのスタイル変更

完了済みタスクを視覚的に区別するため、Ifウィジェットを使って条件付きスタイルを適用できます。

  • 条件: GetTasks.List.Current.Task.IsCompleted = True
  • True時: グレーアウト、取り消し線などのスタイル

動作確認

  1. 1 1-Click Publishでアプリを公開
  2. 2 タスク一覧でチェックボックスをクリック
  3. 3 完了/未完了が切り替わることを確認（スタイルも変わるか確認）
  4. 4 「削除」ボタンをクリック → 確認ダイアログが表示されることを確認
  5. 5 「Yes」をクリック → タスクが削除され、一覧から消えることを確認
  6. 6 「編集」リンクをクリック → 編集画面に遷移し、データが表示されることを確認

期待される結果

タスクの編集、削除、完了切り替えがすべて正常に動作し、 ユーザーがタスクを自由に管理できる状態になります。

トラブルシューティング

編集画面でデータが表示されない

原因: TaskIdパラメータが正しく渡されていない、またはAggregateのフィルターが間違っている

解決方法:

  • 編集リンクのDestinationでTaskIdパラメータが設定されているか確認
  • GetTaskByIdのフィルター条件を確認（Task.Id = TaskId）
  • FormのData Sourceが GetTaskById.List.Current に設定されているか確認
  • Debuggerで実行時のTaskIdの値を確認
削除しても一覧から消えない

原因: DeleteTask後にRefreshが実行されていない

解決方法:

  • DeleteTaskアクションの後に Refresh Data → GetTasks を追加
  • アクションフローの接続順序を確認
  • ページをリロードして最新状態を確認
チェックボックスの変更が保存されない

原因: On Change Actionが正しく設定されていない、またはCreateOrUpdateが実行されていない

解決方法:

  • CheckboxのOn Changeイベントが設定されているか確認
  • アクションフローにCreateOrUpdateTaskが含まれているか確認
  • Sourceに正しいTaskオブジェクトが渡されているか確認
  • TrueChangeログを確認してエラーがないか調査
確認ダイアログが表示されない

原因: Confirmロジックが正しく配置されていない

解決方法:

  • アクションフローの最初にConfirmロジックが配置されているか確認
  • Messageプロパティに表示メッセージが設定されているか確認
  • DeleteTaskが「Yes」分岐に接続されているか確認
