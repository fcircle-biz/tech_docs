タスクを削除しよう - 削除と完了機能

不要なタスクを削除する機能と、タスクの完了/未完了を切り替える機能を追加します。

このステップで実装すること

  • タスクを削除する機能を実装する
  • チェックボックスでタスクの完了/未完了を切り替える
  • 完了済みタスクの見た目を変更する（オプション）

前提条件

  • ステップ6までの内容が完了していること
  • タスク一覧画面が表示できること
  • タスクの登録・編集ができること

7.1 削除機能とは？

アプリケーションでは、不要になったデータを削除する機能が必要です。 タスク管理アプリでも、完了したタスクや間違って登録したタスクを削除できるようにします。

ただし、削除は「元に戻せない操作」です。そのため、ユーザーが誤って削除してしまわないように、 削除前に確認メッセージを表示する必要があります。

削除機能の基本

削除の流れ: 削除ボタンをクリック → データベースから削除 → 一覧画面を更新

重要ポイント: 削除は取り消せない操作です。実際のアプリでは確認ダイアログの追加を検討しましょう。

7.2 削除ボタンの追加

まず、タスク一覧画面の各タスクに削除ボタンを追加します。

実装 7-1: 削除ボタンの配置

タスク一覧の各タスクに削除ボタンを追加します。

1

Home画面を開く

Service Studioで「Home」画面をダブルクリックして開きます。

2

リスト内のタスク表示部分を見つける

画面の中で、タスクの一覧を表示している「List」ウィジェットを探します。 その中に、タスクのタイトルと編集ボタンが配置されているはずです。

3

削除ボタンを追加

ツールボックスから「Button」ウィジェットをドラッグして、編集ボタンの隣に配置します。

  • Textプロパティ: 「削除」
  • Nameプロパティ: 「DeleteButton」

削除ボタンの配置のポイント

削除ボタンは、編集ボタンの隣に配置するのが一般的です。 ボタンを横に並べるには、「Container」ウィジェットで囲み、「Display: Flex」を設定すると良いでしょう。

7.3 削除アクションの作成

次に、削除ボタンが押されたときの処理（削除アクション）を作成します。

実装 7-2: 削除アクションの実装

削除ボタンのOnClickイベントにアクションを設定します。

1

削除ボタンをダブルクリック

先ほど追加した「DeleteButton」をダブルクリックします。 自動的にOnClickイベントのアクションフローが開きます。

2

削除アクションを追加

「Data」タブから「DeleteTask」をドラッグしてフローに配置します。

  • Id: GetTasks.List.Current.Task.Id
3

一覧を再読み込み

削除後、画面に表示されているタスク一覧を更新するため、「Refresh Data」ノードを追加します。

  • Data Source: 「GetTasks」を選択
4

成功メッセージを表示（オプション）

削除が成功したことをユーザーに知らせるため、「Message」ノードを追加します。

  • Message: 「タスクを削除しました」
  • Type: 「Success」

注意: 削除は元に戻せない

一度削除したデータは復元できません。実際のアプリでは確認ダイアログ（Popup等）の追加を検討しましょう。 また、本格的なシステムでは「論理削除」（削除フラグを立てるだけで、実際には削除しない）を使うこともあります。

7.4 完了/未完了の切り替え機能

次に、タスクの完了/未完了を切り替える機能を追加します。 チェックボックスを使って、タスクが完了したかどうかを簡単に切り替えられるようにします。

実装 7-3: チェックボックスで完了状態を切り替え

タスク一覧にチェックボックスを追加し、完了/未完了を切り替えられるようにします。

1

既存のチェックボックスを削除

エンティティをリストにドラッグした際に自動生成されたIsCompletedのチェックボックスを選択し、Deleteキーで削除します。

2

新しいチェックボックスを追加

ツールボックスから「Checkbox」ウィジェットをドラッグして、タスクのタイトルの前に配置します。

  • Variable: GetTasks.List.Current.Task.IsCompleted
3

OnChangeイベントを設定

チェックボックスをダブルクリックして、「On Change」イベントのアクションフローを開きます。

4

タスクを更新

「CreateOrUpdateTask」をドラッグして、チェックボックスの変更をデータベースに保存します。

  • Source: GetTasks.List.Current.Task
5

一覧を再読み込み

「Refresh Data」ノードを追加して、一覧画面を更新します。

  • Data Source: 「GetTasks」

チェックボックスのポイント

チェックボックスの「Variable」プロパティにIsCompletedをバインドすることで、 チェックボックスの状態とデータベースの値が自動的に連動します。 チェックを入れると「True」、外すと「False」になります。

7.5 完了タスクのスタイル変更（オプション）

完了したタスクと未完了のタスクを見分けやすくするため、完了タスクの見た目を変更しましょう。 例えば、文字をグレーにしたり、取り消し線を引いたりすると効果的です。

実装 7-4: 完了タスクの見た目を変更

完了したタスクを視覚的に区別できるようにします。

1

タスクタイトルのウィジェットを選択

タスクのタイトルを表示している「Expression」または「Text」ウィジェットをクリックします。

2

条件付きスタイルを設定

プロパティパネルの「Style Classes」を動的に設定します。

  • 「Style Classes」の右にある式ボタン（fxマーク）をクリック
  • 条件式を入力: "task-title " + If(GetTasks.List.Current.Task.IsCompleted, "completed-task", "")
3

CSSスタイルを追加

Home画面を選択し、プロパティパネルの「Style Sheet」欄をダブルクリックしてCSSを編集します。

CSS 
/* タスクタイトルのスタイル */
.task-title {
    font-size: 1.2rem; /* 大きめの文字 */
    font-weight: bold; /* 太字 */
}

/* 完了したタスクのスタイル */
.completed-task {
    color: #999; /* グレーの文字色 */
    text-decoration: line-through; /* 取り消し線 */
}

条件付きスタイルの仕組み

If式を使うことで、「もしIsCompletedがTrueなら"completed-task"というクラスを適用し、Falseなら何も適用しない」という動作を実現できます。 この方法は様々な場面で活用できるので、覚えておくと便利です。

7.6 動作確認

アプリの動作確認

すべての実装が完了したら、1-Click Publishでアプリを公開し、動作を確認しましょう。

確認すべきポイント

  • タスク一覧に削除ボタンが表示される
  • 削除ボタンを押すとタスクが削除され、一覧から消える
  • チェックボックスが表示され、クリックできる
  • チェックを入れるとタスクが完了状態になる
  • 完了したタスクの見た目が変わる（スタイル変更を実装した場合）
  • チェックを外すと未完了状態に戻る

トラブルシューティング

削除ボタンを押してもタスクが削除されない

原因: DeleteアクションのIdが正しく設定されていない可能性があります。

解決方法:

  • DeleteTaskのIdプロパティを確認してください
  • 「GetTasks.List.Current.Task.Id」のように、現在のタスクのIdを指定しているか確認
チェックボックスをクリックしても変更されない

原因: CheckboxのVariableプロパティが正しくバインドされていない、またはOnChangeアクションが設定されていない可能性があります。

解決方法:

  • CheckboxのVariableプロパティが「GetTasks.List.Current.Task.IsCompleted」になっているか確認
  • OnChangeイベントにCreateOrUpdateアクションが設定されているか確認
  • Refresh Dataでタスク一覧を再読み込みしているか確認
完了タスクのスタイルが変わらない

原因: CSSスタイルが正しく適用されていない、または条件式が間違っている可能性があります。

解決方法:

  • CSSタブに「.completed-task」のスタイルが記述されているか確認
  • タイトルウィジェットのStyle Classesに条件式が設定されているか確認
  • 条件式のIf関数の構文が正しいか確認（カンマの位置、引用符など）
  • アプリを再公開してキャッシュをクリアしてみる
削除後、一覧が更新されない

原因: Refresh Dataノードが抜けている、または正しいアグリゲートが指定されていない可能性があります。

解決方法:

  • 削除アクションの後にRefresh Dataノードが配置されているか確認
  • Refresh DataのData Sourceが「GetTasks」になっているか確認
  • ブラウザをリロードして、データが削除されているか確認
