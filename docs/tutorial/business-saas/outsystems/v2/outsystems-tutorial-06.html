タスクを編集しよう - 編集画面の作成

既存のタスクを編集できる画面を作成します。タスク一覧から編集画面への遷移と、変更内容の保存方法を学びます。

このステップで学ぶこと

  • 編集画面の作成方法
  • 画面間でのデータ受け渡し（Input Parameter）
  • 既存データの読み込みと表示
  • フォームウィジェットの使い方
  • CreateOrUpdate（データの更新処理）の使い方
  • タスク一覧から編集画面へのリンク

6.1 編集機能とは？

このステップの完成イメージ

このステップでは、タスク一覧から選択したタスクを編集できる画面を作成します。

編集機能は、既存のデータを変更するための仕組みです。 一覧画面で編集したいデータを選び、詳細画面で内容を変更して保存する、という流れが一般的です。

編集機能に必要な要素

編集機能を実装するには、以下の要素が必要です：

  • データの識別: どのタスクを編集するかを特定する（TaskId）
  • 既存データの読み込み: データベースから現在の値を取得
  • 編集フォーム: ユーザーが内容を変更できる画面
  • 更新処理: 変更した内容をデータベースに反映

6.2 タスク編集画面の作成

実装 6-1: 編集画面の作成とInput Parameterの設定

タスクを編集するための画面を作成し、どのタスクを編集するかを受け取るためのInput Parameterを設定します。

1

Interface タブを開く

Service Studioの左側のツリーで「Interface」タブをクリックします。

2

MainFlow を右クリック

「UI Flows」→「MainFlow」を右クリックし、「Add Screen」を選択します。

3

Empty テンプレートを選択

スクリーンテンプレート選択画面で「Empty」を選択します。

4

スクリーン名を設定

スクリーン名を「TaskDetail」と入力し、「Create Screen」をクリックします。

5

Input Parameterを追加

編集するタスクを識別するためのパラメータを追加します：

  1. Widget Tree（画面右下）で「TaskDetail」スクリーンを右クリック
  2. 「Add Input Parameter」を選択
  3. 名前を「TaskId」と入力
  4. プロパティパネルでData Typeを「Task Identifier」に設定

Input Parameterとは？

Input Parameterは、画面に渡される値のことです。 「どのタスクを編集するか」を一覧画面から編集画面に伝えるために使います。 URLの一部として渡されます（例: TaskDetail?TaskId=1）。

6.3 既存データの読み込みとフォーム配置

実装 6-2: Aggregateで既存データを取得

編集画面を開いたとき、Input ParameterのTaskIdを使って既存のタスクデータを読み込みます。

1

TaskエンティティをMainContentにドラッグ

データを取得してフォームを自動生成します：

  1. 画面右上の「Data」タブをクリック
  2. 「Entities」→「Database」→「Task」を見つける
  3. 「Task」をTaskDetailスクリーンのMainContentエリアにドラッグ&ドロップ
2

Formを選択

「List」と「Form」の選択画面が表示されたら、「Form」を選択して「OK」をクリックします。

3

Aggregateにフィルターを設定

自動作成された「GetTaskById」Aggregateを編集して、特定のタスクだけを取得するようにします：

  1. Widget TreeまたはInterfaceタブで「GetTaskById」をダブルクリック
  2. Aggregateエディタが開きます
  3. 右側の「Filters」をクリック
  4. 「Add Filter」をクリック
  5. 以下のフィルター条件を入力：
Filter条件 
Task.Id = TaskId

これにより、Input Parameterで渡されたTaskIdと一致するタスクだけが取得されます。

4

自動生成された内容を確認

ドラッグ&ドロップにより、以下が自動的に作成されます：

  • Title（入力欄）
  • Description（入力欄）
  • Is Completed（チェックボックス）
  • Created Date（日付入力欄）
  • Saveボタン
5

不要なフィールドを削除

「Is Completed」と「Created Date」は編集画面では不要なので、削除します：

  1. 「Is Completed」のラベルとチェックボックスを選択（Ctrlキーを押しながらクリックで複数選択可）
  2. Deleteキーを押して削除
  3. 同様に「Created Date」のラベルと入力欄も削除

削除後、フォームには「Title」「Description」「Save」ボタンだけが残ります。

ポイント

エンティティをドラッグ&ドロップするだけで、データ取得用のAggregateとフォームが自動生成されます。 フィルターを設定することで、編集対象のタスクだけを表示できます。

6.4 入力フィールドのカスタマイズ

自動生成されたフォームを、より使いやすくカスタマイズします。

実装 6-3: ラベルと入力欄の調整

1

ラベルを日本語化

各入力フィールドのラベル（見出し）をクリックして、以下のように変更します：

  • 「Title」→「タスク名」
  • 「Description」→「説明」
2

入力欄のプレースホルダーを設定

各Inputウィジェットを選択し、プロパティパネルで「Prompt」（プレースホルダー）を設定します：

  • Titleの入力欄：「例: 資料作成」
  • Descriptionの入力欄：「タスクの詳細を入力してください」
3

Descriptionを複数行入力に変更（オプション）

この手順は省略可能です。1行入力のままでも問題ありません。

複数行入力にしたい場合は、以下の手順で行います：

  1. Description入力欄（Input_Description）を選択してDeleteキーで削除
  2. 左側のツールボックスから「Text Area」ウィジェットを探す
  3. 「Text Area」をDescriptionラベルの下にドラッグ&ドロップ
  4. プロパティパネルでVariableを「GetTaskById.List.Current.Task.Description」に設定
4

ボタンのテキストを変更

フォーム内の送信ボタンを選択し、「Text」プロパティを「保存」に変更します。

6.5 更新処理の実装

ボタンにエラーが表示されていますか？

「保存」ボタンにエラー（On Click must be set）が表示されているのは正常です。 OutSystemsでは、Buttonには必ずクリック時の処理（On Click）を設定する必要があります。 この手順でアクションを設定するとエラーは解消されます。

「保存」ボタンがクリックされたときの更新処理を実装します。 編集した内容をデータベースに保存し、一覧画面に戻ります。

実装 6-4: 保存ボタンのアクション設定

1

保存ボタンをダブルクリック

フォーム内の「保存」ボタンをダブルクリックすると、アクションフローエディタが開きます。

2

CreateOrUpdateTaskを追加

変更内容をデータベースに保存するため、「CreateOrUpdateTask」をフローに追加します。

CreateOrUpdateTaskの場所：

  1. 画面右上の「Data」タブをクリック
  2. 「Entities」→「Database」→「Task」を展開
  3. Taskの下にある「CreateOrUpdateTask」を見つける
  4. 「CreateOrUpdateTask」を「Start」と「End」の間にドラッグ&ドロップ

Sourceパラメータの設定：

ドロップすると、プロパティパネルに「Source」パラメータが表示されます。以下を設定してください：

Source設定 
Source: GetTaskById.List.Current.Task

Sourceのドロップダウンをクリックして「GetTaskById」→「List」→「Current」→「Task」を選択します。

Security Warning が表示されますか？
「You're exposing a database operation in the client side...」という警告が表示されますが、 学習目的では無視して問題ありません。 これは「本番環境ではServer Actionを経由すべき」というベストプラクティスの警告です。 アプリは正常に動作します。

3

成功メッセージを表示

ツールボックスから「Message」をドラッグし、「Text」プロパティに以下を設定します：

メッセージ 
"タスクを更新しました"

「Message Type」は「Success」に設定します。

4

Homeスクリーンへ遷移

ツールボックスから「Destination」をドラッグし、遷移先として「Home」スクリーンを選択します。 これにより、更新後にタスク一覧画面に戻ります。

CreateOrUpdateとは？

「CreateOrUpdate」は、レコードのIdに値があれば更新（Update）、 なければ新規作成（Create）を自動的に判断してくれる便利な処理です。 今回は既存のTaskIdを持つデータを編集しているので、更新処理が実行されます。

6.6 入力チェック（バリデーション）

タスク名が空のまま保存されないように、入力チェックを追加します。

実装 6-5: 必須入力チェック

1

Titleを必須項目に設定

TaskDetailスクリーンに戻り、Titleの入力欄（Input_Title）を選択します。

  1. Widget TreeまたはキャンバスでTitle入力欄をクリック
  2. 右側のプロパティパネルで「Mandatory」を探す
  3. 「Mandatory」を「True」に変更

これだけで、空の場合にデフォルトのエラーメッセージが自動表示されます。

2

バリデーションの動作確認

Mandatoryを「True」に設定すると、以下の動作になります：

  • タスク名が空のまま「保存」ボタンを押すと、エラーメッセージが表示される
  • 保存処理が実行されない（Form1.Valid? がFalseになる）
  • 入力欄が赤くハイライトされる

6.7 タスク一覧から編集画面へのリンク追加

タスク一覧画面（Home）の各タスクに、編集画面（TaskDetail）へ遷移するリンクを追加します。

実装 6-6: 編集リンクの追加

1

Homeスクリーンを開く

InterfaceツリーからHomeスクリーンをダブルクリックして開きます。

2

タスクリスト内にLinkを追加

タスク一覧の各行に編集リンクを追加します：

  1. ツールボックスから「Link」をドラッグ
  2. タスクリスト内（各タスクの行）にドロップ
3

リンクのテキストを設定

追加したLinkを選択し、「Text」プロパティを「編集」に設定します。

4

遷移先とパラメータを設定

リンクのプロパティで遷移先とTaskIdを設定します：

  1. 「On Click」プロパティで「Navigate」を選択
  2. 「Destination」で「TaskDetail」を選択
  3. 「TaskId」パラメータに「GetTasks.List.Current.Task.Id」を設定

「GetTasks.List.Current.Task.Id」は、現在の行のタスクのIdを意味します。 これにより、クリックしたタスクの編集画面に遷移します。

ポイント

リストの中でLinkを使うと、各行のデータ（この場合はTask.Id）を パラメータとして次の画面に渡すことができます。 これが「どのタスクを編集するか」を伝える仕組みです。

6.8 動作確認

動作確認の手順

  1. 1 1-Click Publishでアプリを公開します
  2. 2 Homeスクリーンを開き、タスク一覧が表示されることを確認
  3. 3 任意のタスクの「編集」リンクをクリック
  4. 4 TaskDetail画面に遷移し、既存のタスク情報が表示されることを確認
  5. 5 タスク名を空にして「保存」ボタンを押し、エラーメッセージが表示されることを確認
  6. 6 タスク名や説明を変更して「保存」ボタンをクリック
  7. 7 成功メッセージが表示され、Homeスクリーンに戻ることを確認
  8. 8 変更した内容がタスク一覧に反映されていることを確認

期待される結果

  • 編集画面に既存のタスク情報が表示される
  • タスク名が空の場合、エラーメッセージが表示される
  • 変更を保存すると、成功メッセージが表示される
  • 自動的にHomeスクリーンに戻る
  • 変更した内容が一覧に反映されている

トラブルシューティング

「保存」ボタンを押しても何も起きない

原因: アクションが正しく設定されていない、またはフローが完成していない

解決方法:

  • ボタンのアクションフローを再確認する
  • CreateOrUpdateTaskが正しく配置されているか確認
  • すべてのフロー要素が接続されているか確認
  • エラーパネル（画面下部）に警告やエラーがないか確認
データが保存されない

原因: Formのデータソースが正しく設定されていない、またはバリデーションエラー

解決方法:

  • Formウィジェットのプロパティでデータソースを確認
  • 必須項目が入力されているか確認
  • CreateOrUpdateTaskのSourceパラメータが正しいか確認
  • Service Studioのデバッガーを使って値を確認
保存後にHomeスクリーンに戻らない

原因: Destinationが設定されていない、または誤った遷移先が指定されている

解決方法:

  • アクションフローの最後にDestinationがあるか確認
  • Destinationの遷移先が「Home」になっているか確認
  • Destinationの前にEndが配置されていないか確認
エラーメッセージが表示されない

原因: 入力欄のMandatoryプロパティが設定されていない

解決方法:

  • Title入力欄を選択し、Mandatoryプロパティを確認
  • Validation Messageが設定されているか確認
  • フォームのValidationが有効になっているか確認
