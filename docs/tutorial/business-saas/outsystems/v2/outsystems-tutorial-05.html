5.1 アグリゲートとは？

前のステップで作成した「Task」エンティティには、タスクのデータが保存されています。 このデータを画面に表示するには、まずアグリゲート（Aggregate）を使ってデータベースからデータを取り出す必要があります。

アグリゲートとは

アグリゲートは、データベースからデータを「取り出す」ための機能です。 OutSystemsでは、複雑なSQLクエリを書く代わりに、視覚的にデータの取り出し方を設定できます。

  • 目的: エンティティに保存されたデータを読み取る
  • 特徴: ドラッグ&ドロップでデータの絞り込みや並び替えができる
  • 結果: 画面で使えるデータのリストとして提供される

5.2 アグリゲートの作成

実装 5-1: タスク一覧を取得するアグリゲートの作成

まず、画面でタスクのデータを取得するためのアグリゲートを作成します。

1

ホーム画面を開く

Service Studioで、前のステップで作成した「Home」スクリーンをダブルクリックして開きます。

2

データタブを確認

画面右側のプロパティエリアの上部にある「Data」タブをクリックします。 ここでアグリゲートを作成・管理できます。

3

Taskエンティティを画面にドラッグ&ドロップ

これが最も簡単な方法です。以下の手順で行います：

  1. 画面右上にある「Data」タブをクリック（「Processes」「Interface」「Logic」「Data」と並んでいるタブ）
  2. 「Entities」→「Database」フォルダを展開
  3. 「Task」エンティティを見つける
  4. 「Task」を画面中央のキャンバス（「タスク一覧」テキストの下あたり）にドラッグ&ドロップ
  5. 「Select data」ダイアログが表示されたら、「Choose layout view」で「List」を選択して「Select」ボタンをクリック

ListとTableの違い：

  • List：カード形式で縦に並べて表示（モバイル向き、柔軟なデザイン）
  • Table：表形式で行・列に整理して表示（Excel風、データ一覧向き）

このチュートリアルでは「List」を使用します。

自動作成されるもの：
ドラッグ&ドロップするだけで、以下が自動的に作成されます：

  • アグリゲート（GetTasks）：データベースからTaskを取得
  • Listウィジェット：タスクを繰り返し表示
  • 表示項目：Title、Description等の表示用ウィジェット
4

作成されたアグリゲートを確認

ドラッグ&ドロップ後、自動的に「GetTasks」というアグリゲートが作成されます。

アグリゲートの確認場所：

  1. 画面右上の「Interface」タブをクリック
  2. 「UI Flows」→「MainFlow」→「Home」を展開
  3. Homeの下に「GetTasks」というアグリゲートが表示されます

名前を変更したい場合は、「GetTasks」をクリックして選択し、右側のプロパティパネルで「Name」を編集します（例: GetAllTasks）。

5

並び替え設定（オプション）

タスクを作成日時の新しい順に表示したい場合は、以下の手順で設定します：

  1. Widget Treeで「GetTasks」をダブルクリックしてアグリゲートエディタを開く
  2. 「Sort」タブをクリック
  3. 「Add Sort」をクリック
  4. 「Task.CreatedDate」を選択し、「Descending」（降順）を選択

ヒント

アグリゲートには、フィルター（Filter）を追加することで、特定の条件に合うデータだけを取得できます。 例えば「完了していないタスクだけ」を表示したい場合などに使います。

5.3 自動作成された内容の確認

Taskエンティティをドラッグ&ドロップして「List」を選択すると、以下が自動的に作成されます。 画面に表示されている内容を確認しましょう。

自動作成された要素

  • GetTasks（アグリゲート）：データベースからTaskデータを取得
  • Listウィジェット：タスクを繰り返し表示する枠組み
  • Title：タスクのタイトルを表示
  • Description：タスクの説明を表示
  • IsCompleted：完了状態をチェックマークで表示
  • CreatedDate：作成日時を表示

ポイント

OutSystemsの便利な機能により、エンティティをドラッグ&ドロップするだけで、 データ取得（アグリゲート）から表示（List）まで自動的に設定されます。 手動でウィジェットを配置してバインディングを設定する手間が省けます。

5.4 テストデータの作成と確認

実装 5-3: テストデータの追加

一覧表示を確認するために、いくつかのサンプルタスクをデータベースに追加します。

1

Dataタブを開く

Service Studioの左側のツールボックスで「Data」タブをクリックし、 「Entities」フォルダの中にある「Task」を右クリックします。

2

「View Data」を選択

右クリックメニューから「View or Edit Data」を選択します。 データベースの内容を直接見て編集できる画面が開きます。

3

新しい行を追加

画面下部の「Add」ボタンをクリックして、新しいタスクのレコードを追加します。 以下のようなテストデータを入力してみましょう：

  • タスク1: Title: "プロジェクト計画書を作成", Description: "Q1のプロジェクト計画をまとめる", IsCompleted: False
  • タスク2: Title: "チームミーティング資料準備", Description: "週次ミーティング用のスライドを作成", IsCompleted: False
  • タスク3: Title: "顧客へのメール返信", Description: "見積もり依頼への回答を送る", IsCompleted: True
4

保存して閉じる

すべてのテストデータを入力したら、画面を閉じます。変更は自動的に保存されます。

5.5 動作確認

アプリケーションを実行

  1. 1 Service Studioで「1-Click Publish」ボタンをクリックして、アプリを公開します
  2. 2 公開が完了したら、「Open in Browser」をクリックして、ブラウザでアプリを開きます
  3. 3 ホーム画面に、先ほど追加したテストデータのタスクが一覧で表示されることを確認します

期待される結果

  • 画面に3件のタスクが表示される
  • 各タスクのタイトルと説明が正しく表示されている
  • 作成日時の新しい順（設定した場合）に並んでいる

5.6 発展的な内容（オプション）

このセクションはオプションです。 基本的なタスク一覧表示ができていれば、次のステップに進んでも問題ありません。 興味がある方は、後で試してみてください。

OutSystemsでは、表示をさらにカスタマイズできます。以下は将来的に試せる内容です：

  • スタイルの適用：Style Classesを使って見た目を変更
  • 完了状態の表示：完了したタスクをグレーアウト表示
  • フィルター：未完了タスクだけを表示
  • 並び替え：作成日時や完了状態で並び替え

これらの機能は、基本操作に慣れてから試すことをお勧めします。

トラブルシューティング

タスクが1件も表示されない

原因: テストデータが登録されていない、またはアグリゲートの設定が正しくない

解決方法:

  • 「View or Edit Data」でTaskエンティティにデータが存在するか確認
  • アグリゲートのSourceが正しくTaskエンティティを参照しているか確認
  • ListウィジェットのSourceプロパティが「GetAllTasks.List」になっているか確認
  • フィルター設定で誤ってすべてのデータが除外されていないか確認
タスクのタイトルや説明が表示されない

原因: TextウィジェットのバインディングExpressionが正しく設定されていない

解決方法:

  • Textウィジェットを選択し、プロパティの「Expression」を確認
  • 「GetAllTasks.List.Current.Task.Title」のように、正しいパスが設定されているか確認
  • テストデータにTitle、Descriptionの値が入力されているか確認
データの並び順が期待通りではない

原因: アグリゲートのSort設定が正しくない

解決方法:

  • アグリゲートを開き、Sortセクションを確認
  • CreatedDateで並び替えたい場合は、「CreatedDate」を選択し、「Descending」（降順）または「Ascending」（昇順）を設定
  • 1-Click Publishで再公開してから確認
