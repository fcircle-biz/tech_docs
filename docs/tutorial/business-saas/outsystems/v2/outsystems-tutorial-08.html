処理の流れを制御しよう - アクションとロジック

ボタンを押したときの処理をより詳しく設定します。条件によって処理を変える方法を学びます。

このステップで学ぶこと

  • クライアントアクションとサーバーアクションの違い

  • メッセージの表示（成功、エラー、警告）

8.1 アクションとは？

OutSystemsでは、ボタンをクリックしたときなどに実行される「処理のまとまり」をアクションと呼びます。 アクションは、ユーザーの操作に応じて、データを保存したり、画面を切り替えたり、メッセージを表示したりします。

アクションの種類

OutSystemsには大きく2種類のアクションがあります。

  • クライアントアクション: ユーザーのブラウザ上で実行される処理（画面の操作、簡単な計算など）
  • サーバーアクション: サーバー側で実行される処理（データベース操作、複雑な計算など）

8.2 フローチャート形式でのロジック作成

OutSystemsのアクションは、フローチャート（流れ図）形式で作成します。 プログラムのコードを書く代わりに、視覚的に処理の流れを組み立てることができます。

フロー内の主な要素

  • Start/End: フローの開始と終了
  • Assign: 変数に値を代入する
  • If: 条件によって処理を分岐する
  • Message: メッセージを表示する
  • Refresh Data: 画面のデータを再読み込みする
flowchart TD A([Start]) --> B[Assign: 変数に値を設定] B --> C{If: 条件をチェック} C -- 条件が真 --> D[Message: 成功メッセージ] C -- 条件が偽 --> E[Message: エラーメッセージ] D --> F[Refresh Data: データ再読み込み] E --> G([End]) F --> G

8.3 条件分岐（If）の使い方

実装 8-1: タスク名の入力チェック

タスクを保存する前に、タスク名が空でないかチェックする処理を追加します。 空の場合はエラーメッセージを表示し、保存処理を中断します。

1

タスク登録画面を開く

Service Studioで、タスク登録画面（TaskDetail）を開きます。

2

保存ボタンのアクションを開く

保存ボタンをダブルクリックして、そのアクションフローを開きます。 現在は、すぐにCreateOrUpdateが実行されているはずです。

3

Ifノードを追加

ツールボックスから「If」をドラッグして、CreateOrUpdateの前に配置します。

4

条件を設定

Ifノードを選択し、プロパティの「Condition」に以下を入力します：

条件式 
Task.Title = "" or Task.Title = NullTextIdentifier()

これは「タスク名が空文字、またはNull（空の値）」という意味です。

5

エラーメッセージを追加

Ifの「True」分岐（条件が真の場合）に、「Message」ノードをドラッグします。 メッセージの内容を「タスク名を入力してください」と設定し、Typeを「Error」にします。

6

フローを完成させる

Ifの「False」分岐（条件が偽、つまりタスク名が入力されている場合）に、CreateOrUpdateを配置します。

flowchart TD A([Start: SaveButtonClick]) --> B{If: タスク名が空?} B -- True: 空の場合 --> C[Message: エラー
タスク名を入力してください] B -- False: 入力されている --> D[CreateOrUpdate Task] C --> E([End]) D --> F[Message: 成功
タスクを保存しました] F --> G[Refresh: GetTasks] G --> H([End])

ヒント

条件式では、「=」は等しいかを判定、「or」は「または」を意味します。 「タスク名が空、または、Nullである」という2つの条件のいずれかに該当する場合、エラーメッセージが表示されます。

8.4 完了タスクの表示制御

実装 8-2: 完了済みタスクの色を変える

タスク一覧で、完了済みのタスクは背景色をグレーにして、視覚的にわかりやすくします。 条件によって表示を変える方法を学びます。

1

タスク一覧画面を開く

Service Studioで、タスク一覧画面（Home）を開きます。

2

Listウィジェット内のContainerを選択

タスク一覧のListウィジェット内にあるContainer（タスクの行全体を囲んでいる部品）を選択します。

3

Style Classesに条件式を追加

プロパティの「Style Classes」に、以下のように入力します：

Style Classes 
If(GetTasks.List.Current.Task.IsCompleted, "background-gray", "background-white")

これは「もしタスクが完了済みなら、グレーの背景。そうでなければ白の背景」という意味です。

4

スタイルクラスを定義

「background-gray」と「background-white」というCSSクラスをあらかじめ定義しておくか、 OutSystemsのThemeに組み込まれているスタイルを利用します。

ヒント

If関数は、OutSystemsの式の中で使える便利な関数です。 「If(条件, 真の場合の値, 偽の場合の値)」という形式で、条件によって異なる値を返すことができます。

8.5 メッセージの表示

ユーザーに処理の結果をわかりやすく伝えるために、メッセージを表示します。 OutSystemsには、成功、エラー、警告など、さまざまな種類のメッセージがあります。

成功（Success）

処理が正常に完了したことを知らせます。

エラー（Error）

処理が失敗したことを知らせます。

警告（Warning）

注意が必要なことを知らせます。

メッセージノードの設定例

  • Message: 「タスクを保存しました」
  • Type: Success（または Error、Warning）
  • Display: Popup（ポップアップで表示）

8.6 画面の再読み込み（Refresh Data）

データを保存したり、削除したりした後は、画面に表示されているデータを最新の状態に更新する必要があります。 これを「Refresh（リフレッシュ）」と呼びます。

実装 8-3: データの再読み込み

タスクを保存した後、タスク一覧のデータを再読み込みして、新しいタスクが一覧に表示されるようにします。

1

保存アクションを開く

タスク保存ボタンのアクションフローを開きます。

2

Refresh Dataノードを追加

CreateOrUpdateの後に、「Refresh Data」ノードをドラッグします。

3

再読み込みするデータを選択

Refresh Dataノードのプロパティで、「Data Source」に「GetTasks」（タスク一覧を取得するアグリゲート）を選択します。

重要ポイント

Refresh Dataを行わないと、データベースには保存されているのに、画面には反映されない状態になります。 データを変更した後は、必ずRefreshを忘れないようにしましょう。

8.7 処理の順序の考え方

アクションフローでは、ノードが上から下へ、左から右へと順番に実行されます。 処理の順序を正しく設定しないと、期待通りに動作しないことがあります。

flowchart TD A([Start: SaveTask]) --> B{If: 入力チェック} B -- エラーあり --> C[Message: エラー表示] C --> Z([End]) B -- エラーなし --> D[CreateOrUpdate: データ保存] D --> E[Message: 成功表示] E --> F[Refresh Data: 一覧再読み込み] F --> G[Destination: 一覧画面へ遷移] G --> Z style B fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style D fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style F fill:#d1fae5,stroke:#10b981,stroke-width:2px

処理の順序の基本ルール

  1. 入力チェックは最初に行う（データを保存する前）
  2. データ保存が成功してから、成功メッセージを表示する
  3. メッセージを表示してから、データを再読み込みする
  4. すべての処理が完了してから、画面遷移する

動作確認

アプリを公開して、以下の動作を確認してみましょう。

  1. 1 タスク名を空のまま保存ボタンを押すと、エラーメッセージが表示されること
  2. 2 タスク名を入力して保存すると、成功メッセージが表示されること
  3. 3 保存後、タスク一覧に新しいタスクが表示されること
  4. 4 完了済みのタスクは、一覧でグレーの背景になっていること

期待される結果

条件によって処理が変わり、適切なメッセージが表示され、データが正しく更新されることを確認できれば成功です。

トラブルシューティング

エラーメッセージが表示されない

原因: Messageノードが正しく配置されていない、または条件式が間違っている

解決方法:

  • Ifノードの「True」分岐にMessageノードが接続されているか確認
  • 条件式が正しいか確認（Task.Title = "" or Task.Title = NullTextIdentifier()）
  • MessageノードのTypeが「Error」に設定されているか確認
保存後に一覧が更新されない

原因: Refresh Dataノードが配置されていない、または正しく設定されていない

解決方法:

  • CreateOrUpdateの後にRefresh Dataノードが配置されているか確認
  • Refresh DataのData Sourceに「GetTasks」が選択されているか確認
  • フローの接続が正しいか確認
完了タスクの色が変わらない

原因: Style Classesの条件式が間違っている、またはCSSクラスが定義されていない

解決方法:

  • If関数の構文が正しいか確認（If(条件, 真の値, 偽の値)）
  • IsCompletedフィールドが正しく参照されているか確認
  • background-grayやbackground-whiteのCSSクラスが定義されているか確認
