# OutSystems入門チュートリアル - はじめてのタスク管理アプリ開発

プログラミング未経験者でも安心して始められる、OutSystems 11（O11）を使った実践的なチュートリアルです。ローコード開発プラットフォームの特徴を活かし、視覚的な操作を中心に「タスク管理アプリ」を一から作り上げていきます。

## はじめに

### このチュートリアルについて

「アプリを作ってみたいけど、プログラミングは難しそう...」

そんな風に思っていませんか？

このチュートリアルでは、**プログラミングの知識がなくても**、視覚的な操作だけでWebアプリを作る方法を学びます。使用するのは「OutSystems」というローコード開発プラットフォーム。世界中の企業で使われている本格的なツールですが、初心者でも直感的に操作できるよう設計されています。

PowerPointでスライドを作るような感覚で、実際に動くアプリケーションを作れるようになります。

### 作成するアプリケーション

このチュートリアルでは、シンプルで実用的な**「タスク管理アプリ」**を作成します。

**完成イメージ:**
- タスクの一覧を見る
- 新しいタスクを登録する
- タスクの内容を編集する
- 完了したタスクを削除する
- タスクの完了/未完了を切り替える

日常的に使えるアプリを作りながら、OutSystemsの基本操作を自然に身につけていきます。

## ローコード開発とは？

### コードを書かずにアプリを作る

「プログラミング」という言葉を聞くと、英語のような文字がたくさん並んだ画面を想像するかもしれません。たとえば、こんな感じです：

```javascript
// 従来のプログラミングの例（見るだけで大丈夫です）
function addTask(taskName) {
    const task = {
        name: taskName,
        completed: false,
        createdAt: new Date()
    };
    database.tasks.insert(task);
    return task;
}
```

このようなコードを書けるようになるには、通常は何年もの学習が必要です。

**ローコード開発**では、このようなコードをほとんど書きません。代わりに、画面上で部品を並べたり、設定を選んだりするだけでアプリを作れます。

### たとえばこんな感じ

**従来の方法:**
1. プログラミング言語を学ぶ（数ヶ月〜数年）
2. コードを1行1行書く
3. エラーが出たらコードを読み直して修正

**ローコードの方法:**
1. 部品を画面にドラッグ&ドロップ
2. 設定項目を選ぶ
3. ボタンを押してアプリを公開

まるでPowerPointでスライドを作るように、視覚的な操作でアプリが作れるのです。

### OutSystemsを選ぶ理由

OutSystemsは、世界中で使われているローコード開発プラットフォームです：

| 特徴 | 説明 |
|------|------|
| **大企業でも採用** | トヨタ、デロイト、シーメンスなど多くの大企業で利用されています |
| **本格的な機能** | 業務で使える高機能なアプリケーションが作れます |
| **無料で始められる** | 個人学習用の無料環境が用意されています |
| **初心者にやさしい** | 視覚的なインターフェースで直感的に操作できます |
| **学習リソースが豊富** | 公式の教材やコミュニティが充実しています |

## よくある質問（FAQ）

### Q: プログラミング経験がなくても本当にできますか？

**A:** はい、できます。このチュートリアルはプログラミング未経験者を対象に作られています。PowerPointでスライドを作るような感覚で操作できます。専門用語は必ず説明しますので、安心して進めてください。

### Q: OutSystemsは無料で使えますか？

**A:** はい、Personal Environmentという無料環境が提供されています。学習や個人的な開発には十分な機能が使えます。有料版との主な違いは、利用者数やデータ量の制限、本格的なビジネス利用向けの機能などです。

### Q: OutSystems 11（O11）とODCの違いは何ですか？

**A:** このチュートリアルで使用するのはOutSystems 11（O11）です。ODC（OutSystems Developer Cloud）は新しいクラウドネイティブ版ですが、O11はより成熟しており、学習リソースも豊富です。初心者にはO11から始めることをお勧めします。

### Q: どのくらいの時間がかかりますか？

**A:** 全10ステップで約12時間を想定しています。1日1〜2時間ずつ進めれば、1〜2週間で完了できます。急がず、理解しながら進めることをおすすめします。

### Q: 作ったアプリは実際に使えますか？

**A:** はい、使えます。公開したアプリにはURLが発行され、ブラウザからアクセスできます。友人や家族と共有して使うこともできます（Personal Environmentには同時利用者数などの制限があります）。

### Q: エラーが出て先に進めません

**A:** Service Studioはエラーの箇所を赤く表示してくれます。まずはエラーメッセージを読んでみてください。それでも解決しない場合は、前のステップを確認するか、新しいアプリで最初からやり直すのも有効です。

### Q: macOSでも使えますか？

**A:** Service Studioは現在Windows版のみ提供されています。macOSの場合は、Parallels DesktopやBoot Campなどを使ってWindows環境を用意するか、ブラウザベースのODC Studio（OutSystems Developer Cloud）を使用する方法があります。

### Q: スマートフォン用のアプリも作れますか？

**A:** はい、OutSystemsではスマートフォン用のアプリも作成できます。このチュートリアルではWeb版を作成しますが、基本操作を覚えた後にモバイルアプリ開発にも挑戦できます。

### Q: 各ステップの順番を飛ばしても大丈夫ですか？

**A:** 基本的には順番通りに進めることをお勧めします。特にStep 4（エンティティの作成）を飛ばすと、Step 5以降でデータを表示・操作できなくなります。各ステップは前のステップの内容を前提としています。
