# OutSystems入門チュートリアル - はじめてのタスク管理アプリ開発

プログラミング未経験者でも安心して始められる、OutSystemsを使った実践的なチュートリアルです。ローコード開発プラットフォームの特徴を活かし、視覚的な操作を中心に「タスク管理アプリ」を一から作り上げていきます。

## チュートリアル概要

### このチュートリアルで作るもの

シンプルで実用的な「タスク管理アプリ」を作成します。

**完成イメージ:**
- タスクの一覧を見る
- 新しいタスクを登録する
- タスクの内容を編集する
- 完了したタスクを削除する
- タスクの完了/未完了を切り替える

日常的に使えるアプリを作りながら、OutSystemsの基本操作を自然に身につけていきます。

### 学習目標

このチュートリアルを完了すると、以下のことができるようになります：

- **ローコード開発の理解** - プログラムコードを書かずにアプリを作る方法がわかる
- **OutSystemsの基本操作** - Service Studioの画面構成と基本的な使い方をマスターする
- **画面の作り方** - スクリーン（画面）にボタンやテキストなどの部品を配置できる
- **データの保存** - エンティティ（データベース）を使ってデータを保存・管理できる
- **一覧表示** - 保存したデータを画面に一覧で表示できる
- **フォーム作成** - ユーザーがデータを入力・編集できる画面を作れる
- **処理の実装** - ボタンを押したときの動作（ロジック）を設定できる
- **アプリの公開** - 作ったアプリをインターネット上で使える状態にできる

### 対象読者

- **パソコンの基本操作ができる方**（ファイル管理、ブラウザ操作、文字入力）
- **プログラミング経験は不要です** - むしろ未経験の方を想定しています
- **Webアプリを作ってみたい方** - 自分のアイデアをカタチにしたい方
- **業務改善に興味がある方** - 仕事で使えるツールを自分で作りたい方
- **ExcelやPowerPointを使ったことがある方** - 同じような感覚で操作できます

### 特徴

- **完全初心者対応**: 専門用語は使わず、使う場合は必ず丁寧に説明します
- **視覚的な学習**: 画面のどこをクリックするか、何を入力するかを明確に示します
- **段階的な構築**: 少しずつ機能を追加して、最終的に完成したアプリになります
- **実践重視**: 実際に動くアプリを作りながら学べます
- **すぐに試せる**: 無料の環境で今日から始められます

## ローコード開発とは？

### 従来の開発との違い

アプリケーションを作るには、通常「プログラミング」という作業が必要です。プログラミングでは、コンピュータに命令を出すための「コード」と呼ばれる文字列を書いていきます。

```
// 従来のプログラミングの例（参考）
function addTask(taskName) {
    const task = {
        name: taskName,
        completed: false,
        createdAt: new Date()
    };
    database.tasks.insert(task);
    return task;
}
```

このようなコードを書くには専門的な知識と経験が必要で、習得に何年もかかることがあります。

**ローコード開発**では、このようなコードをほとんど書く必要がありません。代わりに、画面上で部品をドラッグ&ドロップしたり、設定項目を選んだりすることで、アプリケーションを組み立てていきます。

PowerPointでスライドを作るように、Excelで表を作るように、視覚的な操作でアプリが作れるのがローコード開発の特徴です。

### OutSystemsの特徴

OutSystemsは世界的に広く使われているローコード開発プラットフォームです：

- **大企業でも採用**: トヨタ、デロイト、シーメンスなど多くの大企業で利用
- **豊富な機能**: 業務システムに必要な機能がすぐに使える状態で揃っている
- **学習しやすい**: 初心者向けの教材やコミュニティが充実
- **日本語対応**: 管理画面やドキュメントの一部が日本語化

## 作成するアプリケーションの詳細

### アプリケーション名: My Task Manager

#### 主な機能

| 機能 | 説明 | 実装するStep |
|------|------|--------------|
| タスク一覧表示 | 登録されているタスクを一覧で表示 | Step 5 |
| タスク登録 | 新しいタスクを追加 | Step 6 |
| タスク編集 | 既存のタスクの内容を変更 | Step 7 |

| 完了切り替え | タスクの完了/未完了を切り替え | Step 7 |

#### データ構造（エンティティ設計）

**Taskエンティティ:**

| 項目名 | 英語名 | データ型 | 説明 |
|--------|--------|----------|------|
| ID | Id | 自動番号 | 各タスクを区別するための番号（自動で付与） |
| タスク名 | Title | テキスト | タスクの名前（必須、50文字まで） |
| 説明 | Description | 長いテキスト | タスクの詳しい説明（任意、500文字まで） |
| 完了フラグ | IsCompleted | はい/いいえ | 完了したかどうか（初期値：いいえ） |
| 作成日時 | CreatedDate | 日時 | タスクを作成した日時（自動で設定） |

#### 画面構成

1. **タスク一覧画面（Home）**
   - タスクの一覧をリスト形式で表示
   - 各タスクに編集・削除ボタン
   - 完了/未完了の切り替えチェックボックス
   - 「新規作成」ボタン

2. **タスク登録・編集画面（TaskDetail）**
   - タスク名の入力欄
   - 説明の入力欄
   - 保存ボタン
   - キャンセルボタン

## 学習の進め方

### おすすめの学習方法

1. **まず手を動かす**
   - 説明を読むだけでなく、実際にService Studioを操作してください
   - 「見る」より「やる」方が何倍も身につきます

2. **エラーを恐れない**
   - エラーが出ても大丈夫です。Service Studioは親切にエラーの場所を教えてくれます
   - エラーを直す経験が、理解を深めます

3. **分からなくても進む**
   - 100%理解できなくても、まずは手順通りに進めてみてください
   - 全体像が見えると、後から理解できることも多いです

4. **復習する**
   - 一度作ったものを、説明を見ずにもう一度作ってみると効果的です

### 困ったときは

1. **エラーメッセージを読む**
   - Service Studioは、何が問題かを赤い文字や下線で教えてくれます

2. **前のステップに戻る**
   - 手順が抜けていないか、確認してみてください

3. **最初からやり直す**
   - どうしても解決しない場合は、新しいアプリを作り直すのも一つの方法です

4. **コミュニティに質問**
   - OutSystemsには日本語のコミュニティもあります

## よくある質問（FAQ）

### Q: プログラミング経験がなくても本当にできますか？

**A:** はい、できます。このチュートリアルはプログラミング未経験者を対象に作られています。PowerPointでスライドを作るような感覚で操作できます。専門用語は必ず説明しますので、安心して進めてください。

### Q: OutSystemsは無料で使えますか？

**A:** はい、Personal Environmentという無料環境が提供されています。学習や個人的な開発には十分な機能が使えます。有料版との主な違いは、利用者数やデータ量の制限、本格的なビジネス利用向けの機能などです。

### Q: どのくらいの時間がかかりますか？

**A:** 全10ステップで約12時間を想定しています。1日1〜2時間ずつ進めれば、1〜2週間で完了できます。急がず、理解しながら進めることをおすすめします。

### Q: 作ったアプリは実際に使えますか？

**A:** はい、使えます。公開したアプリにはURLが発行され、ブラウザからアクセスできます。友人や家族と共有して使うこともできます（Personal Environmentには同時利用者数などの制限があります）。

### Q: エラーが出て先に進めません

**A:** Service Studioはエラーの箇所を赤く表示してくれます。まずはエラーメッセージを読んでみてください。それでも解決しない場合は、前のステップを確認するか、新しいアプリで最初からやり直すのも有効です。

### Q: macOSでも使えますか？

**A:** Service Studioは現在Windows版のみ提供されています。macOSの場合は、Parallels DesktopやBoot Campなどを使ってWindows環境を用意するか、ブラウザベースのODC Studio（OutSystems Developer Cloud）を使用する方法があります。

### Q: スマートフォン用のアプリも作れますか？

**A:** はい、OutSystemsではスマートフォン用のアプリも作成できます。このチュートリアルではWeb版を作成しますが、基本操作を覚えた後にモバイルアプリ開発にも挑戦できます。

## カラーテーマ設定

このチュートリアルでは、学習に集中できる落ち着いた緑系（emerald）のカラーテーマを使用します：

```javascript
primary: {
    50: '#ecfdf5',
    100: '#d1fae5',
    200: '#a7f3d0',
    300: '#6ee7b7',
    400: '#34d399',
    500: '#10b981',  // メインカラー
    600: '#059669',
    700: '#047857',
    800: '#065f46',
    900: '#064e3b',
}
```

アイコン: `fas fa-tasks` または `fas fa-cube`
