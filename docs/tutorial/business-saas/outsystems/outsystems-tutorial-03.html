画面をデザインしよう - スクリーンとウィジェット

タスク管理アプリのメイン画面を作成します。ボタン、テキスト、コンテナなどの「ウィジェット」（部品）を配置して、見た目を整えていきます。

このステップで実装すること

  • ウィジェット（部品）の種類と使い方を理解する
  • ドラッグ&ドロップでウィジェットを画面に配置する
  • プロパティ（設定項目）を変更して見た目を調整する
  • タスク管理アプリのメイン画面のレイアウトを作成する

前提条件

  • ステップ2で作成したアプリケーションが開いている状態
  • Service Studioの基本操作（ツールボックス、プロパティ）に慣れている

3.1 ウィジェットとは？

OutSystemsでは、画面を構成する部品のことを「ウィジェット」と呼びます。 これはPowerPointの図形やExcelのセルのようなもので、画面にドラッグ&ドロップで配置するだけで使えます。

ウィジェット 役割 例え
Container 部品をまとめる箱 PowerPointのグループ化
Text 文字を表示 PowerPointのテキストボックス
Button クリックできるボタン Webサイトの「送信」ボタン
Input 文字を入力する欄 検索窓や入力フォーム
Image 画像を表示 PowerPointの画像挿入

ウィジェットはどこにある?

Service Studioの左側にある「Toolbox」（ツールボックス）タブを開くと、使用できるウィジェットの一覧が表示されます。 カテゴリごとに分類されているので、目的のウィジェットを探しやすくなっています。

3.2 ウィジェットの配置方法

ウィジェットを画面に配置する方法は、PowerPointで図形を配置するのと同じくらい簡単です。 ツールボックスから目的のウィジェットを選んで、画面にドラッグ&ドロップするだけです。

実装 3-1: Containerウィジェットを配置する

まずは基本となるContainerウィジェットを配置してみましょう。Containerは他のウィジェットをまとめる「箱」の役割をします。

1

Toolboxタブを開く

Service Studioの左側にある「Toolbox」タブをクリックして、ツールボックスを開きます。

2

Containerを探す

ツールボックス内の「Widgets」→「Containers」を展開し、「Container」を見つけます。

3

ドラッグ&ドロップ

「Container」を画面中央の編集エリア（キャンバス）にドラッグ&ドロップします。

灰色の枠が表示されたら配置成功です！

4

プロパティを確認

配置したContainerをクリックすると、右側の「Properties」（プロパティ）タブに設定項目が表示されます。 ここで名前や見た目を変更できます。

初心者がつまずきやすいポイント

ドラッグ中に表示される青い線に注目してください。この線は「ここに配置されます」という目印です。 思った場所に配置されない場合は、青い線が正しい位置に表示されているか確認しましょう。

3.3 プロパティで見た目を変える

配置したウィジェットは、「Properties」（プロパティ）で見た目や動作を変更できます。 これはPowerPointで図形の色やサイズを変えるのと同じ感覚です。

実装 3-2: Textウィジェットで見出しを作る

Textウィジェットを使って、アプリのタイトル「タスク管理」という見出しを作ります。

1

Textウィジェットを配置

ツールボックスから「Widgets」→「Display」→「Text」を、先ほど配置したContainerの中にドラッグ&ドロップします。

2

テキストを変更

配置されたTextウィジェットをダブルクリックして、表示されているテキストを「タスク管理」に変更します。

3

プロパティでスタイルを変更

Textウィジェットを選択した状態で、右側の「Properties」タブを確認します。

  • Style Classes: 「Heading1」と入力（大きな見出しになります）
4

確認

画面上で「タスク管理」という文字が大きく表示されていれば成功です！

プロパティ 用途 設定例
Name ウィジェットの識別名 TaskListContainer
Style Classes 見た目のスタイル Heading1, Card, Button-Primary
Visible 表示/非表示 True / False

実装 3-3: タスク管理画面の基本レイアウト

画面を3つのエリアに分けて、それぞれにウィジェットを配置します。

1

見出しエリアを作成

ステップ2で作成したHomeScreenを開きます。画面に「Container」を配置し、その中に「Text」ウィジェットを配置します。

  • ContainerのName: HeaderContainer
  • Textの内容: 「タスク管理」
  • TextのStyle Classes: Heading1
2

タスク表示エリアを作成

HeaderContainerの下に、もう1つ「Container」を配置し、その中に「Text」を配置します。

  • ContainerのName: TaskListContainer
  • Textの内容: 「タスク一覧がここに表示されます」
  • ContainerのStyle Classes: Card（カード風の見た目になります）
3

ボタンエリアを作成

TaskListContainerの下に「Container」を配置し、その中に「Button」ウィジェットを配置します。

  • ContainerのName: ButtonContainer
  • ButtonのText: 「新しいタスクを追加」
  • ButtonのStyle Classes: btn btn-primary（目立つ色のボタンになります）
4

ボタンのクリックイベントを設定

ボタンにはクリック時の動作を設定する必要があります。設定しないとエラーになり、パブリッシュできません。

  • Buttonを選択し、右側のプロパティパネルで「Events」セクションを探す
  • On Clickの「Handler」ドロップダウンをクリック
  • 「New Client Action」を選択（自動的に空のアクションが作成されます）
補足: この段階ではボタンを押しても何も起きませんが、エラーは解消されます。実際の処理は後のステップで追加します。
5

配置を調整

各Containerの間隔が狭い場合は、Containerを選択してStyle Classesに「margin-top-base」などを追加すると、上部に余白が追加されます。

つまずきポイント: Containerの入れ子構造

「Containerの中にTextを入れる」という表現に戸惑うかもしれません。Service Studioでは、ウィジェットを別のウィジェットの中にドラッグすると、 自動的に「親子関係」ができます。画面左下の「Widget Tree」（ウィジェットツリー）を見ると、階層構造が確認できます。

3.5 スタイルクラスで見た目を整える

OutSystemsには、あらかじめ用意された「スタイルクラス」があります。 これを使うことで、CSSの知識がなくても、プロフェッショナルな見た目を簡単に実現できます。

スタイルクラス 用途
Heading1, Heading2, Heading3 大見出し、中見出し、小見出し
Card カード風の枠で囲む
btn btn-primary 目立つ色のボタン（プライマリカラー）
margin-top-base 上部に標準的な余白を追加
padding-base 内側に標準的な余白を追加

スタイルクラスは複数指定できる

Style Classesには、スペース区切りで複数のクラスを指定できます。 例: Card margin-top-base padding-base とすると、カード風の見た目で、上部と内側に余白が追加されます。

3.6 動作確認

画面の確認

作成した画面をパブリッシュして、ブラウザで確認しましょう。

1 画面上部の「1-Click Publish」ボタンをクリック
2 公開が完了したら「Open in Browser」をクリック
3 OutSystemsアカウントでログインして画面を確認

期待される結果

  • 「タスク管理」という大きな見出しが表示されている
  • カード風の枠の中に「タスク一覧がここに表示されます」というテキストがある
  • 「新しいタスクを追加」ボタンが目立つ色で表示されている
  • 各要素が適切な間隔で配置されている

完成イメージ

トラブルシューティング

ウィジェットがうまく配置できない

原因: ドラッグ先の位置が正しくない可能性があります。

解決方法:

  • ドラッグ中に表示される青い線の位置を確認してください
  • Container内に配置したい場合は、Containerの枠内にドロップしてください
  • Widget Tree（左下）で階層構造を確認し、ドラッグ&ドロップで移動できます
スタイルクラスを指定しても見た目が変わらない

原因: スタイルクラス名のスペルミス、または適用対象のウィジェットが違う可能性があります。

解決方法:

  • スタイルクラス名が正しく入力されているか確認してください（大文字・小文字も区別されます）
  • 複数のクラスを指定する場合は、スペースで区切ってください
  • 一度Publishして、ブラウザでリロードしてから確認してください
ウィジェットが重なって表示される

原因: ウィジェットの配置順序や、Containerの使い方に問題がある可能性があります。

解決方法:

  • Widget Treeで階層構造を確認してください
  • 各ウィジェットは別々のContainerに入れるか、縦に並ぶように配置してください
  • margin-top-baseなどのスタイルクラスで間隔を調整してください
