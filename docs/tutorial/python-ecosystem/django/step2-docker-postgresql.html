2.1 Docker Desktop for Windowsのインストール

コンテナ技術を使用してPostgreSQLデータベースを管理するため、Docker Desktopをセットアップします。

実装 2-1: Docker Desktop for Windowsのセットアップ

Docker公式サイトからDocker Desktopをダウンロードし、適切にインストールします。

手順
  1. ダウンロード: https://www.docker.com/products/docker-desktop からWindows版をダウンロード
  2. システム要件確認:
    • Windows 10 64-bit: Pro, Enterprise, Education (Build 19041以降)
    • WSL 2機能が有効
    • BIOS/UEFIでHardware Virtualizationが有効
  3. インストール実行:
    • 管理者権限でインストーラーを実行
    • 「Use WSL 2 instead of Hyper-V」を選択
    • 「Add shortcut to desktop」にチェック
  4. 再起動: インストール完了後、システムを再起動
  5. Docker Desktop起動: デスクトップのアイコンからDocker Desktopを起動
動作確認

コマンドプロンプトまたはPowerShellで以下を実行:

docker --version
docker-compose --version
期待される結果
Docker version 24.x.x, build xxxxxxx
Docker Compose version v2.x.x
トラブルシューティング

WSL 2が無効の場合:

  1. 管理者権限でPowerShellを開く
  2. wsl --installを実行
  3. システムを再起動
  4. Docker Desktopを再起動

2.2 WSL2の設定と最適化

Docker DesktopのパフォーマンスとWSL2の統合を最適化します。

実装 2-2: WSL2環境の最適化設定

WSL2のメモリ使用量を制限し、Docker環境を最適化します。

手順
  1. WSL設定ファイル作成: ユーザーホームディレクトリに.wslconfigを作成 
    # %USERPROFILE%\.wslconfig ファイルを作成
[wsl2]
memory=4GB
processors=2
swap=1GB
localhostForwarding=true
  2. WSL再起動: 
    wsl --shutdown
  3. Docker Desktop設定: Docker Desktop > Settings > Resources > WSL Integration で設定確認
  4. Ubuntu/DebianディストリビューションでWSL統合を有効化
動作確認

WSL2の状態を確認:

wsl --status
wsl -l -v
期待される結果

WSL2がデフォルトバージョンとして設定され、適切なメモリ制限が適用されている

2.3 PostgreSQLとpgAdminのDocker Compose設定

PostgreSQLデータベースとpgAdmin管理ツールをコンテナで管理するためのCompose設定を作成します。

実装 2-3: docker-compose.ymlファイルの作成

PostgreSQLとpgAdminを同時に起動できるDocker Compose設定を作成します。

手順
  1. Dockerディレクトリ作成: 
    cd C:\django-tutorial
mkdir docker
cd docker
  2. docker-compose.yml作成: 
    # docker-compose.yml
version: '3.8'

services:
  # PostgreSQLデータベース
  postgres:
    image: postgres:16
    container_name: django_postgres
    environment:
      POSTGRES_DB: django_db
      POSTGRES_USER: django_user
      POSTGRES_PASSWORD: django_password
      POSTGRES_HOST_AUTH_METHOD: md5
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./init-scripts:/docker-entrypoint-initdb.d
    restart: unless-stopped
    networks:
      - django_network

  # pgAdmin管理ツール
  pgadmin:
    image: dpage/pgadmin4:latest
    container_name: django_pgadmin
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@example.com
      PGADMIN_DEFAULT_PASSWORD: admin123
      PGADMIN_CONFIG_SERVER_MODE: 'False'
    ports:
      - "5050:80"
    volumes:
      - pgadmin_data:/var/lib/pgadmin
    depends_on:
      - postgres
    restart: unless-stopped
    networks:
      - django_network

# ボリューム定義
volumes:
  postgres_data:
    driver: local
  pgadmin_data:
    driver: local

# ネットワーク定義
networks:
  django_network:
    driver: bridge
  3. 初期化スクリプトディレクトリ作成: 
    mkdir init-scripts
動作確認

docker-compose.ymlファイルの構文をチェック:

docker-compose config
期待される結果

エラーなしでDocker Compose設定が表示される

2.4 PostgreSQLとpgAdminコンテナの起動

作成したDocker Compose設定を使用してデータベース環境を起動します。

実装 2-4: データベース環境の起動と確認

Docker Composeを使用してPostgreSQLとpgAdminを起動し、接続を確認します。

手順
  1. コンテナ起動: 
    # dockerディレクトリで実行
docker-compose up -d
  2. コンテナ状態確認: 
    docker-compose ps
  3. ログ確認: 
    # PostgreSQLのログ
docker-compose logs postgres

# pgAdminのログ
docker-compose logs pgadmin
  4. ポート確認: 
    netstat -an | findstr :5432
netstat -an | findstr :5050
動作確認

PostgreSQLに直接接続してテスト:

# PostgreSQLコンテナに接続
docker exec -it django_postgres psql -U django_user -d django_db

# データベース一覧表示
\l

# 接続終了
\q
期待される結果
  • 両方のコンテナが「Up」状態で動作
  • PostgreSQLのポート5432がリッスン状態
  • pgAdminのポート5050がリッスン状態
  • PostgreSQLにpsqlで接続可能
トラブルシューティング

ポート競合エラーの場合:

# 使用中のポートを確認
netstat -ano | findstr :5432
# プロセスを終了するかポート番号を変更

2.5 pgAdmin4による管理画面アクセス

ブラウザからpgAdminにアクセスし、PostgreSQLデータベースの管理画面を設定します。

実装 2-5: pgAdmin Webインターフェースの設定

ブラウザベースのpgAdmin管理画面を設定し、PostgreSQLサーバーに接続します。

手順
  1. pgAdminアクセス: ブラウザで http://localhost:5050 にアクセス
  2. ログイン:
    • Email: admin@example.com
    • Password: admin123
  3. サーバー追加:
    • 左側パネルで「Servers」を右クリック → Create → Server...
    • Generalタブ:
      • Name: Django PostgreSQL
    • Connectionタブ:
      • Host name/address: postgres
      • Port: 5432
      • Maintenance database: django_db
      • Username: django_user
      • Password: django_password
      • Save password: チェック
  4. 接続テスト: 「Save」をクリックして接続確認
動作確認

pgAdminで以下が確認できることをチェック:

  • 「Django PostgreSQL」サーバーがサーバーリストに表示
  • 「django_db」データベースにアクセス可能
  • テーブル、ビュー、関数などのオブジェクトツリーが表示
期待される結果

pgAdminからPostgreSQLデータベースに正常に接続でき、データベースオブジェクトが確認できる

2.6 Django用データベース設定の準備

次のステップでDjangoからPostgreSQLに接続するための設定ファイルを準備します。

実装 2-6: 環境変数ファイルの作成

データベース接続情報を環境変数で管理するための.envファイルを作成します。

手順
  1. .envファイル作成: プロジェクトルートに作成 
    cd C:\django-tutorial
  2. 環境変数設定: 
    # .env
# Django設定
DEBUG=True
SECRET_KEY=your-secret-key-here-change-in-production

# データベース設定
DB_ENGINE=django.db.backends.postgresql
DB_NAME=django_db
DB_USER=django_user
DB_PASSWORD=django_password
DB_HOST=localhost
DB_PORT=5432

# 国際化設定
LANGUAGE_CODE=ja
TIME_ZONE=Asia/Tokyo
USE_I18N=True
USE_TZ=True
  3. .gitignoreファイル更新: 
    # .gitignore に追加
.env
docker/postgres_data/
docker/pgadmin_data/
動作確認

環境変数ファイルが作成されていることを確認:

dir /B *.env
期待される結果

.envファイルが作成され、データベース接続情報が適切に設定されている

Docker管理用コマンド（参考）
# コンテナ起動
docker-compose up -d

# コンテナ停止
docker-compose down

# コンテナ状態確認
docker-compose ps

# ログ確認
docker-compose logs -f postgres
docker-compose logs -f pgadmin

# データベースバックアップ（参考）
docker exec django_postgres pg_dump -U django_user django_db > backup.sql

# データベースリストア（参考）
docker exec -i django_postgres psql -U django_user django_db < backup.sql
ステップ完了チェック

以下の項目が完了していることを確認してください：

  • ☐ Docker Desktop for Windowsが正常にインストール・起動している
  • ☐ WSL2が適切に設定されている
  • ☐ docker-compose.ymlファイルが作成されている
  • ☐ PostgreSQLコンテナが正常に起動している
  • ☐ pgAdminコンテナが正常に起動している
  • ☐ pgAdminからPostgreSQLデータベースに接続できる
  • ☐ .envファイルにデータベース設定が記録されている
