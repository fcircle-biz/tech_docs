1.1 Python 3.12のインストール

Django開発には最新のPython（3.12推奨）を使用します。Windows環境でのPythonインストールから始めましょう。

実装 1-1: Python 3.12のダウンロードとインストール

Python公式サイトから最新版をダウンロードし、適切にインストールします。

手順
  1. ダウンロード: Python公式サイト（https://www.python.org/）からWindows用Python 3.12をダウンロード
  2. インストーラー実行: ダウンロードしたファイルを管理者として実行
    重要: インストール時に「Add Python to PATH」に必ずチェックを入れてください
  3. カスタムインストール選択: 「Customize installation」を選択
  4. オプション機能: すべてのオプションにチェック（pip、tcl/tk、Python test suite等）
  5. 詳細オプション:
    • 「Install for all users」にチェック
    • 「Add Python to environment variables」にチェック
    • 「Precompile standard library」にチェック
動作確認

コマンドプロンプトを開いて以下のコマンドを実行します：

python --version
pip --version
期待される結果
Python 3.12.x
pip 23.x.x from C:\Program Files\Python312\lib\site-packages\pip (python 3.12)
トラブルシューティング

よくあるエラー: 'python' is not recognized as an internal or external command

解決方法:

  1. Pythonを再インストールし、「Add Python to PATH」にチェック
  2. 環境変数PATHに手動でPythonのパスを追加
  3. コマンドプロンプトを再起動

1.2 Visual Studio Codeのセットアップ

効率的なDjango開発のために、Visual Studio Codeと必要な拡張機能をセットアップします。

実装 1-2: VS Codeと拡張機能のインストール

Django開発に最適化されたIDE環境を構築します。

手順
  1. VS Codeインストール: https://code.visualstudio.com/ からダウンロードしてインストール
  2. 拡張機能のインストール: VS Codeを開き、拡張機能パネル（Ctrl+Shift+X）から以下をインストール:
    • Python (Microsoft) - Python開発サポート
    • Django (Baptiste Darthenay) - Djangoテンプレート支援
    • Python Docstring Generator - ドキュメント生成
    • GitLens - Git強化機能
    • SQLite Viewer - データベース確認
  3. Pythonインタープリターの設定: Ctrl+Shift+P → "Python: Select Interpreter" → Python 3.12を選択
動作確認

VS Codeでpythonファイルを作成して確認:

# test.py
print("Hello, Django Development!")
期待される結果

F5キーでプログラムが実行され、ターミナルに "Hello, Django Development!" が表示される

1.3 仮想環境（venv）の作成と管理

プロジェクトごとに独立したPython環境を作成し、依存関係を管理します。

実装 1-3: Django専用仮想環境の作成

プロジェクト専用の仮想環境を作成し、依存関係を分離します。

手順
  1. プロジェクトディレクトリ作成: 
    mkdir C:\django-tutorial
cd C:\django-tutorial
  2. 仮想環境作成: 
    python -m venv venv
  3. 仮想環境の有効化: 
    # コマンドプロンプトの場合
venv\Scripts\activate

# PowerShellの場合
venv\Scripts\Activate.ps1
  4. pipのアップグレード: 
    python -m pip install --upgrade pip
動作確認

仮想環境が正しく有効化されていることを確認:

where python
pip list
期待される結果

プロンプトの前に (venv) が表示され、pythonのパスが仮想環境内を指している

(venv) C:\django-tutorial>where python
C:\django-tutorial\venv\Scripts\python.exe
トラブルシューティング

PowerShellでExecutionPolicy エラー:

# 管理者権限でPowerShellを起動して実行
Set-ExecutionPolicy RemoteSigned

1.4 Djangoとその他必要パッケージのインストール

仮想環境にDjangoとデータベース関連パッケージをインストールします。

実装 1-4: Djangoと関連パッケージのインストール

Django 5.0とPostgreSQL接続に必要なパッケージをインストールします。

手順
  1. Djangoインストール: 
    pip install django==5.0
  2. PostgreSQL接続パッケージ: 
    pip install psycopg2-binary
  3. 環境変数管理パッケージ: 
    pip install python-decouple
  4. 開発用パッケージ: 
    pip install django-debug-toolbar
動作確認

インストールされたパッケージを確認:

pip list
django-admin --version
期待される結果
5.0.x

Djangoのバージョンが表示され、必要なパッケージがすべてインストールされている

1.5 Djangoプロジェクトの作成とHello World

最初のDjangoプロジェクトを作成し、動作確認を行います。

実装 1-5: 初回Djangoプロジェクト作成

django-adminコマンドを使用してプロジェクトを初期化し、開発サーバーを起動します。

手順
  1. Djangoプロジェクト作成: 
    django-admin startproject myproject .
  2. プロジェクト構造確認: 
    dir /B
  3. 開発サーバー起動: 
    python manage.py runserver
  4. ブラウザで確認: http://127.0.0.1:8000/ にアクセス
動作確認

ブラウザで以下のようなDjangoのウェルカムページが表示されることを確認:

期待される結果

「The install worked successfully! Congratulations!」メッセージが表示され、Djangoロケットのアイコンが表示される

トラブルシューティング

ポート8000が使用中の場合:

インストールした依存関係を記録し、環境の再現性を確保します。

実装 1-6: requirements.txtの作成

プロジェクトに必要なパッケージのリストを作成し、バージョン管理します。

手順
  1. 依存関係の書き出し: 
    pip freeze > requirements.txt
  2. requirements.txtの確認: 
    type requirements.txt
  3. .gitignoreファイル作成: 
    # .gitignore
venv/
*.pyc
__pycache__/
db.sqlite3
.env
.vscode/

Windows環境特有の注意点
  • パス区切り文字: バックスラッシュ（\）を使用
  • 権限問題: 必要に応じて管理者権限で実行
  • ファイアウォール: Django開発サーバーのポート許可が必要な場合がある
  • 文字エンコーディング: UTF-8を推奨、VS Codeで設定
  • 改行コード: LF推奨、Gitで自動変換設定
