AWS Amplifyでデプロイ

作成したNext.jsアプリをAWS Amplifyを使ってインターネットに公開します。GitHubと連携して、コードをプッシュすると自動的にデプロイされる仕組みを構築します。

このステップで実装すること

  • GitHubリポジトリの作成とコードのプッシュ
  • AWS Amplify Hostingアプリの作成
  • 環境変数の設定（API Gateway URL）
  • 自動デプロイの動作確認とアプリ公開

前提条件

  • ステップ8までの完了（フロントエンドとAPIが連携済み）
  • GitHubアカウント（無料アカウントでOK）
  • Gitがインストール済み

9.1 AWS Amplifyとは何か

AWS Amplifyは、フロントエンドアプリケーションを簡単にデプロイ・ホスティングできるAWSのサービスです。GitHubと連携して、コードをプッシュするだけで自動的にビルド・デプロイされます。

AWS Amplifyの特徴

  • • GitHubと連携して自動デプロイ（CI/CD）
  • • HTTPSが自動的に設定される
  • • グローバルCDNで高速配信
  • • プレビュー環境の自動生成
  • • 無料枠で月1000ビルド分まで無料

9.2 GitHubリポジトリの作成

実装 9-1: GitHubリポジトリの作成とコードのプッシュ

フロントエンドのコードをGitHubにプッシュします。

1

GitHubで新規リポジトリを作成

GitHubにログインし、「New repository」をクリックします。

  • • Repository name: aws-serverless-todo
  • • Description: AWS Serverless Todo App
  • • Public/Private: Public（またはPrivate）
  • • 「Create repository」をクリック
2

.gitignoreファイルの確認

frontend/フォルダに.gitignoreファイルがあることを確認します。Next.jsプロジェクトを作成した時点で自動生成されているはずです。

.gitignore 
# dependencies
/node_modules
/.pnp
.pnp.js

# testing
/coverage

# next.js
/.next/
/out/

# production
/build

# misc
.DS_Store
*.pem

# debug
npm-debug.log*
yarn-debug.log*
yarn-error.log*

# local env files
.env*.local

# vercel
.vercel

# typescript
*.tsbuildinfo
next-env.d.ts
3

Gitリポジトリの初期化

ターミナルでfrontend/フォルダに移動し、Gitリポジトリを初期化します。

ターミナル 
cd frontend

# Gitリポジトリの初期化
git init

# ファイルを全てステージング
git add .

# 初回コミット
git commit -m "Initial commit"
4

リモートリポジトリの設定とプッシュ

GitHubリポジトリのURLを設定し、コードをプッシュします。

ターミナル 
# リモートリポジトリの追加（あなたのGitHubユーザー名に置き換えてください）
git remote add origin https://github.com/YOUR_USERNAME/aws-serverless-todo.git

# ブランチ名をmainに変更
git branch -M main

# GitHubにプッシュ
git push -u origin main

プッシュ時にGitHubのユーザー名とパスワード（またはパーソナルアクセストークン）の入力を求められます。

9.3 AWS Amplify Hostingの設定

実装 9-2: Amplifyアプリの作成

AWSマネジメントコンソールでAmplifyアプリを作成し、GitHubと連携します。

1

Amplifyコンソールを開く

AWSマネジメントコンソールで「Amplify」と検索し、AWS Amplifyを開きます。

2

新しいアプリの作成

「新しいアプリをホスト」→「GitHub」を選択し、「続行」をクリックします。

3

GitHubとの接続

「GitHubを認証」をクリックし、GitHubアカウントで認証します。認証後、以下を設定します。

  • • リポジトリ: aws-serverless-todo
  • • ブランチ: main
  • • 「次へ」をクリック
4

ビルド設定の確認

Amplifyが自動的にNext.jsを検出し、ビルド設定を生成します。以下の内容が表示されることを確認します。

amplify.yml（自動生成） 
version: 1
frontend:
  phases:
    preBuild:
      commands:
        - npm ci
    build:
      commands:
        - npm run build
  artifacts:
    baseDirectory: .next
    files:
      - '**/*'
  cache:
    paths:
      - node_modules/**/*

デフォルトの設定で問題ありません。「次へ」をクリックします。

5

アプリの作成と初回デプロイ

設定内容を確認し、「保存してデプロイ」をクリックします。初回デプロイが開始されます。

初回デプロイには5〜10分程度かかります。ビルドの進行状況はリアルタイムで確認できます。

9.4 環境変数の設定

API Gateway URLをAmplifyの環境変数として設定します。これにより、デプロイされたアプリがバックエンドAPIにアクセスできるようになります。

実装 9-3: 環境変数の追加

AmplifyアプリにAPI Gateway URLを環境変数として追加します。

1

API Gateway URLの確認

API Gatewayコンソールを開き、TodoAPIのステージURLを確認します（ステップ5で作成したもの）。

例: https://abc123xyz.execute-api.ap-northeast-1.amazonaws.com/prod

2

Amplify環境変数の設定

Amplifyコンソールで、左メニューから「ホスティング環境」→「環境変数」を選択し、「変数を追加」をクリックします。

  • • 変数: NEXT_PUBLIC_API_URL
  • • 値: API GatewayのステージURL
  • • 「保存」をクリック
3

再デプロイ

環境変数を反映させるため、再デプロイを実行します。左メニューから「デプロイ」を選択し、「アプリを再デプロイ」をクリックします。

環境変数の変更は再デプロイしないと反映されません。

ローカルとAmplifyの環境変数

ローカル開発では.env.localファイルで環境変数を管理していましたが、Amplifyでは以下の流れになります：

  • • Amplifyコンソールで設定した環境変数がビルド時に自動的に注入される
  • • .env.localファイルはGitにプッシュされない（.gitignoreで除外）
  • • 本番環境とローカル環境で異なるAPI URLを使える

9.5 デプロイの確認とアプリ公開

公開URLでアプリにアクセス

Amplifyが生成した公開URLにアクセスし、アプリが正しく動作することを確認します。

  1. 1. Amplifyコンソールの「デプロイ」タブで、デプロイが成功したことを確認します（緑色のチェックマーク）。
  2. 2. 画面上部に表示されているURLをクリックします。
    https://main.xxxxxxxxxxxxx.amplifyapp.com
  3. 3. Todoアプリが表示され、タスクの追加・編集・削除ができることを確認します。

期待される結果

  • • アプリがHTTPSで公開されている
  • • タスクの追加・編集・削除が正常に動作する
  • • ブラウザの開発者ツールでエラーが表示されない
  • • スマートフォンからもアクセスできる

デプロイログの確認

Amplifyの「デプロイ」タブで各フェーズ（Provision、Build、Deploy）のログを確認できます。エラーが発生した場合は、ここでログを確認してください。

9.6 自動デプロイの動作確認

コードを変更してGitHubにプッシュすると、自動的にデプロイされる仕組みを確認します。

実装 9-4: コード変更と自動デプロイの確認

簡単な変更を加えて、自動デプロイが動作することを確認します。

1

ページタイトルを変更

app/page.tsxを開き、タイトルを変更します。

app/page.tsx（変更部分）

  Todo リスト

// 変更後

  AWS Todo リスト 🚀

2

変更をコミット・プッシュ

変更をGitにコミットし、GitHubにプッシュします。

ターミナル 
# 変更をステージング
git add .

# コミット
git commit -m "Update page title"

# GitHubにプッシュ
git push origin main
3

Amplifyで自動デプロイを確認

Amplifyコンソールの「デプロイ」タブを開き、新しいデプロイが自動的に開始されることを確認します。

デプロイ完了後、公開URLにアクセスすると、タイトルが変更されているはずです。

9.7 カスタムドメインの設定（オプション）

独自ドメイン（例: mytodo.com）を持っている場合、Amplifyアプリにカスタムドメインを設定できます。

カスタムドメイン設定の手順概要

  1. Amplifyコンソールで「ドメイン管理」を開く
  2. 「ドメインを追加」をクリックし、所有しているドメインを入力
  3. DNS設定でCNAMEレコードを追加（ドメインレジストラーで設定）
  4. SSL証明書が自動的に発行される（数分〜数時間）
  5. カスタムドメインでアクセス可能になる

注意: カスタムドメインは必須ではありません。Amplifyが提供するURLでも問題なく利用できます。

トラブルシューティング

ビルドが失敗する

原因: 依存関係のエラー、またはビルドコマンドの問題

解決方法:

  • • Amplifyのビルドログで詳細なエラーメッセージを確認
  • • ローカルでnpm run buildが成功するか確認
  • • package.jsonが正しくコミットされているか確認
  • • Node.jsのバージョンを確認（Amplifyは最新のLTSを使用）
デプロイされたアプリでAPIが呼び出せない

原因: 環境変数が正しく設定されていない、またはCORSエラー

解決方法:

  • • Amplifyの環境変数でNEXT_PUBLIC_API_URLが正しく設定されているか確認
  • • API Gatewayで正しいCORS設定がされているか確認
  • • ブラウザの開発者ツールでネットワークタブを確認
  • • 環境変数を変更した後、必ず再デプロイする
GitHubとの接続ができない

原因: GitHubの認証エラー、またはリポジトリへのアクセス権限がない

解決方法:

  • • GitHubで再度認証を行う
  • • リポジトリがPublicになっているか確認
  • • Privateリポジトリの場合、AmplifyにGitHubアプリの権限を付与
  • • ブランチ名が正しいか確認（main/masterの違いに注意）
ページが真っ白になる

原因: JavaScriptエラー、またはルーティングの問題

解決方法:

  • • ブラウザの開発者ツールでコンソールエラーを確認
  • • ローカル環境でnpm run build && npm startを実行してビルド版を確認
  • • Next.js App Routerの構造が正しいか確認
  • • Amplifyのビルド設定でbaseDirectoryが正しいか確認
