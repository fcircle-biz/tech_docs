import { useEffect } from 'react';

type ToastProps = {
  message: string;
  type: 'success' | 'error' | 'info';
  onClose: () => void;
};

export default function Toast({ message, type, onClose }: ToastProps) {
  // 3秒後に自動的に閉じる

10.5 AWSリソースの削除

学習が終了したら、不要な課金を防ぐために、使用したAWSリソースを削除します。

注意事項

リソースを削除すると、保存されているデータも完全に消去されます。必要なデータは事前にバックアップしてください。

リソース削除の推奨順序

実装 10-3: AWSリソースの削除

使用したAWSリソースを順番に削除していきます。

1

Amplifyアプリの削除

AWS Amplifyコンソールを開きます。

  • • 該当のアプリを選択
  • • 「アクション」→「アプリを削除」をクリック
  • • アプリ名を入力して削除を確定
2

API Gatewayの削除

API Gatewayコンソールを開きます。

  • • 「TodoAPI」を選択
  • • 「アクション」→「APIを削除」をクリック
  • • API名を入力して削除を確定
3

Lambda関数の削除

Lambdaコンソールを開きます。

  • • getTodos を選択し削除
  • • createTodo を選択し削除
  • • updateTodo を選択し削除
  • • deleteTodo を選択し削除
4

DynamoDBテーブルの削除

DynamoDBコンソールを開きます。

  • • 「todos」テーブルを選択
  • • 「テーブルを削除」をクリック
  • • 確認チェックボックスにチェックを入れて削除
5

IAMロールの削除

IAMコンソールを開き、「ロール」を選択します。

  • • Lambda関数用に作成したロールを検索
  • • 各ロールを選択し、「ロールの削除」をクリック

Lambda関数を削除すると、関連するロールは自動的に削除されることが多いです。

6

CloudWatchログの削除

CloudWatchコンソールを開き、「ロググループ」を選択します。

  • • /aws/lambda/ で始まるロググループを検索
  • • 各ロググループを選択し、「アクション」→「削除」をクリック

AWS公式の無料利用枠ページで、各サービスの無料範囲を確認する

不要なリソースの削除

テスト用のリソースは使い終わったらすぐに削除する習慣をつける

Cost Explorerの活用

月に一度、AWS Cost Explorerで使用状況と料金を確認する

AWS CLIでリソースを確認

以下のコマンドで、各サービスのリソースが削除されたことを確認できます。

ターミナル 
# Lambda関数の一覧確認
aws lambda list-functions --query "Functions[?starts_with(FunctionName, 'getTodos') || starts_with(FunctionName, 'createTodo')].FunctionName"

# DynamoDBテーブルの一覧確認
aws dynamodb list-tables --query "TableNames[?contains(@, 'todos')]"

# API Gatewayの一覧確認
aws apigateway get-rest-apis --query "items[?name=='TodoAPI'].name"

トラブルシューティング

リソースが削除できない

原因: 依存関係があるリソースが残っている

解決方法:

  • • 推奨順序(Amplify → API Gateway → Lambda → DynamoDB)で削除する
  • • エラーメッセージを確認し、依存先のリソースを先に削除する
  • • Lambda関数を削除してから、IAMロールを削除する
削除後も料金が発生している

原因: CloudWatchログが残っている、または削除したリソースの料金締め日がまだ来ていない

解決方法:

  • • CloudWatchコンソールでロググループをすべて削除する
  • • AWS Cost Explorerで詳細な料金内訳を確認する
  • • 削除直後の料金は、前月の使用分の可能性があるため、翌月も確認する
トーストメッセージが表示されない

原因: Toastコンポーネントのインポート忘れ、またはステート管理の問題

解決方法:

  • • import Toast from '@/components/Toast'; が含まれているか確認
  • • ブラウザの開発者ツールでJavaScriptエラーを確認
  • • setToast が正しく呼び出されているか確認
