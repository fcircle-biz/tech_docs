開発環境の準備

ローカル環境に必要なソフトウェアをインストールし、開発を始める準備をします。

このステップで実装すること

  • Visual Studio Codeのインストールと基本設定
  • Node.js（LTS版）のインストールとバージョン確認
  • AWS CLIのインストールと設定（IAM認証情報の登録）
  • Gitのインストールと初期設定（ユーザー名・メールアドレス）

前提条件

  • ステップ1でAWSアカウントが作成済み
  • 管理者権限を持つIAMユーザーが作成済み
  • パソコンへのソフトウェアインストール権限がある

2.2 Visual Studio Codeのインストール

Visual Studio Code（VS Code）は、Microsoft社が提供する無料のコードエディタです。軽量で高機能、拡張機能が豊富なため、世界中の開発者に利用されています。

実装 2-1: VS Codeのインストール

公式サイトから最新版をダウンロードしてインストールします。

1

公式サイトにアクセス

ブラウザで以下のURLを開き、「Download」ボタンをクリックします。

https://code.visualstudio.com/
2

インストーラーをダウンロード

お使いのOSに合わせたインストーラーが自動的に選択されます。

  • • Windows: VSCodeUserSetup-x64-X.XX.X.exe
  • • macOS: VSCode-darwin-universal.zip
  • • Linux: .deb または .rpm パッケージ
3

インストールを実行

ダウンロードしたファイルをダブルクリックし、インストールウィザードに従います。

Windowsの場合、「PATHへの追加」にチェックを入れることをおすすめします。

4

VS Codeを起動して確認

インストール完了後、VS Codeを起動します。ようこそ画面が表示されればOKです。

推奨拡張機能（後でインストール可）

  • • Japanese Language Pack: VS Codeを日本語化
  • • Python: Python開発支援（Lambda関数用）
  • • AWS Toolkit: AWS操作の補助
  • • ES7+ React/Redux/React-Native snippets: React開発支援

2.3 Node.jsのインストール

Node.jsは、JavaScriptをブラウザ外で実行できる環境です。Next.jsアプリケーションを動かすために必要です。LTS版（長期サポート版）をインストールします。

実装 2-2: Node.js LTS版のインストール

公式サイトからLTS版（推奨版）をダウンロードしてインストールします。

1

公式サイトにアクセス

ブラウザで以下のURLを開き、「LTS」と書かれたボタンをクリックします。

https://nodejs.org/

必ず「LTS」版をダウンロードしてください（安定版で長期サポートあり）。

2

インストーラーを実行

ダウンロードしたインストーラーをダブルクリックし、ウィザードに従ってインストールします。基本的にすべてデフォルト設定でOKです。

3

インストール確認

コマンドプロンプト（Windows）またはターミナル（macOS/Linux）を開き、以下のコマンドを実行します。

bash 
node --version
npm --version

バージョン番号が表示されれば成功です（例: v18.17.0）。

npmとは？

npm（Node Package Manager）は、Node.jsと一緒にインストールされるパッケージ管理ツールです。Next.jsなどのライブラリをインストールする際に使用します。

2.4 AWS CLIのインストールと設定

AWS CLI（Command Line Interface）は、コマンドラインからAWSサービスを操作するためのツールです。ブラウザのAWSコンソールよりも素早く操作できます。

実装 2-3: AWS CLIのインストール

公式サイトから最新版のAWS CLIをダウンロードしてインストールします。

1

公式ドキュメントにアクセス

ブラウザで以下のURLを開き、お使いのOSに合わせたインストール方法を確認します。

https://docs.aws.amazon.com/ja_jp/cli/latest/userguide/getting-started-install.html
2

インストール（Windows）

以下のリンクからMSIインストーラーをダウンロードし、実行します。

AWSCLIV2.msi
3

インストール（macOS）

ターミナルで以下のコマンドを実行します。

bash 
curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
sudo installer -pkg AWSCLIV2.pkg -target /
4

インストール（Linux）

ターミナルで以下のコマンドを実行します。

bash 
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
5

インストール確認

コマンドプロンプトまたはターミナルを開き直して、以下のコマンドを実行します。

bash 
aws --version

バージョン番号が表示されればOKです（例: aws-cli/2.13.0）。

実装 2-4: AWS CLIの認証設定

IAMユーザーのアクセスキーを作成し、AWS CLIに設定します。

1

IAMユーザーのアクセスキーを作成

AWSマネジメントコンソールにIAMユーザーでログインし、以下の手順を実行します。

  • • IAMコンソールを開く
  • • 「ユーザー」→ 自分のIAMユーザー名をクリック
  • • 「セキュリティ認証情報」タブを選択
  • • 「アクセスキーを作成」をクリック
  • • 使用例: 「コマンドラインインターフェイス（CLI）」を選択
  • • タグは空白でOK、「アクセスキーを作成」をクリック
2

アクセスキー情報を控える

作成されたアクセスキー情報をメモ帳などにコピーします。

  • • アクセスキーID: AKIA から始まる20桁の文字列
  • • シークレットアクセスキー: 40桁のランダムな文字列

シークレットアクセスキーはこの画面でしか確認できません。必ず控えてください。

3

AWS CLIに認証情報を設定

コマンドプロンプトまたはターミナルで以下のコマンドを実行します。

bash 
aws configure

以下の情報を入力します（括弧内は入力例です）。

入力例 
AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: ap-northeast-1
Default output format [None]: json

リージョンは東京リージョンの ap-northeast-1 を推奨します。

4

設定確認

以下のコマンドでAWSアカウント情報を取得できることを確認します。

bash 
aws sts get-caller-identity

アカウントIDやユーザーARNが表示されれば設定成功です。

2.5 Gitのインストールと設定

Gitは、コードのバージョン管理システムです。後のステップでGitHubにコードをアップロードし、AWS Amplifyと連携する際に使用します。

実装 2-5: Gitのインストール

公式サイトから最新版のGitをダウンロードしてインストールします。

1

公式サイトにアクセス

ブラウザで以下のURLを開き、「Download」ボタンをクリックします。

https://git-scm.com/
2

インストーラーを実行

ダウンロードしたインストーラーをダブルクリックし、ウィザードに従ってインストールします。基本的にすべてデフォルト設定でOKです。

Windowsの場合、「Git Bash」がインストールされます（Unix風のコマンドラインツール）。

3

インストール確認

コマンドプロンプトまたはターミナルを開き直して、以下のコマンドを実行します。

bash 
git --version

バージョン番号が表示されればOKです（例: git version 2.42.0）。

実装 2-6: Gitの初期設定

Gitにユーザー名とメールアドレスを登録します。コミット履歴に記録されます。

1

ユーザー名を設定

以下のコマンドを実行します（YourNameの部分を自分の名前に変更）。

bash 
git config --global user.name "YourName"
2

メールアドレスを設定

以下のコマンドを実行します（your.email@example.comの部分を自分のメールアドレスに変更）。

bash 
git config --global user.email "your.email@example.com"
3

設定確認

以下のコマンドで設定内容を確認します。

bash 
git config --list

user.name と user.email が表示されればOKです。

2.6 動作確認

すべてのツールが動作することを確認

インストールしたすべてのソフトウェアが正しく動作することを確認します。

  1. 1.

    コマンドプロンプトまたはターミナルを開き、以下のコマンドをすべて実行します。

    bash 
    # Node.jsとnpmのバージョン確認
node --version
npm --version

# AWS CLIのバージョン確認
aws --version

# Gitのバージョン確認
git --version

# AWS CLIの認証確認
aws sts get-caller-identity
  2. 2. Visual Studio Codeを起動し、正常に動作することを確認します。

期待される結果

  • • すべてのコマンドでバージョン番号が表示される
  • • aws sts get-caller-identity でアカウント情報が表示される
  • • VS Codeが起動し、編集画面が表示される

トラブルシューティング

「コマンドが見つかりません」と表示される

原因: PATHが通っていない、またはターミナルを開き直していない

解決方法:

  • • コマンドプロンプトまたはターミナルを一度閉じて、再度開く
  • • Windowsの場合、PCを再起動してみる
  • • ソフトウェアを再インストールし、「PATHに追加」オプションを有効にする
AWS CLIで「Unable to locate credentials」エラー

原因: AWS CLIの認証情報が設定されていない

解決方法:

  • • aws configure を再実行
  • • アクセスキーIDとシークレットアクセスキーを正確に入力する
  • • 入力時にスペースが入らないよう注意する
Node.jsのバージョンが古い

原因: インストールしたバージョンが古い

解決方法:

  • • Node.js公式サイトから最新のLTS版をダウンロード
  • • 既存のNode.jsをアンインストールしてから再インストール
  • • バージョン18.17以上であることを確認
VS Codeが日本語化されていない

原因: 日本語パックがインストールされていない

解決方法:

  • • VS Codeを起動
  • • 左側のアイコンから「Extensions」（拡張機能）を開く
  • • 「Japanese Language Pack for Visual Studio Code」を検索してインストール
  • • VS Codeを再起動
