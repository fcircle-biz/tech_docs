フロントエンドとAPIの連携

フロントエンドからAPI Gatewayを呼び出し、データの取得・操作を行います。

このステップで実装すること

  • 環境変数の設定（APIエンドポイント）
  • fetch APIを使ったHTTPリクエストの実装
  • useEffectでのデータ取得とローディング処理
  • CRUD操作（作成・読取・更新・削除）のフロント実装
  • エラーハンドリングとユーザーへのフィードバック

前提条件

  • Step 7でNext.jsプロジェクトが作成されている
  • API GatewayのエンドポイントURLをメモしている
  • Lambda関数とDynamoDBが正常に動作している

8.1 フロント・バック連携の仕組み

フロントエンド（ブラウザで動くReactアプリ）とバックエンド（AWS上のAPI）を連携させることで、ユーザーの操作がデータベースに反映される仕組みを実現します。

sequenceDiagram participant U as ユーザー participant F as フロントエンド
Next.js participant A as API Gateway participant L as Lambda関数 participant D as DynamoDB U->>F: タスクを追加 F->>A: POST /todos A->>L: createTodo実行 L->>D: データ保存 D-->>L: 保存完了 L-->>A: レスポンス A-->>F: 作成されたタスク F-->>U: 画面を更新

フロント・バック連携の流れ

HTTPメソッドとCRUD操作の対応

  • • GET → 読取（Read）- タスク一覧を取得
  • • POST → 作成（Create）- 新しいタスクを追加
  • • PUT → 更新（Update）- タスクの内容を変更
  • • DELETE → 削除（Delete）- タスクを削除

8.2 環境変数の設定

APIエンドポイントURLをコード内にハードコーディングせず、環境変数として管理します。これにより開発環境と本番環境で異なるAPIを簡単に切り替えられます。

APIエンドポイントを環境変数として設定します。

1

.env.localファイルを作成

8.3 タスク一覧取得の実装

ページが読み込まれたタイミングで、DynamoDBからタスク一覧を取得して画面に表示します。

実装 8-2: データ取得機能の追加

page.tsxにタスク一覧を取得する処理を追加します。

1

page.tsxに状態管理を追加

タスク一覧、ローディング状態、エラー状態を管理するためのstateを追加します。

frontend/app/page.tsx 
'use client';

import { useState, useEffect } from 'react';

// Todoの型定義
interface Todo {
  id: string;
  title: string;
  completed: boolean;
  createdAt: string;
  updatedAt?: string;
}

export default function Home() {
  // 状態管理
  const [todos, setTodos] = useState<Todo[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  // APIエンドポイント
  const API_URL = process.env.NEXT_PUBLIC_API_URL;

  // ページ読み込み時にタスク一覧を取得
  useEffect(() => {
    fetchTodos();
  }, []);

  // タスク一覧取得関数
  const fetchTodos = async () => {
    try {
      setLoading(true);
      setError(null);

      const response = await fetch(`${API_URL}/todos`);

      if (!response.ok) {
        throw new Error('タスクの取得に失敗しました');
      }

      const data = await response.json();
      setTodos(data);
    } catch (err) {
      setError(err instanceof Error ? err.message : '不明なエラー');
      console.error('エラー:', err);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="min-h-screen bg-gray-50 py-8">
      <div className="max-w-4xl mx-auto px-4">
        <h1 className="text-3xl font-bold text-gray-900 mb-8">
          Todoリスト
        </h1>

        {/* ローディング表示 */}
        {loading && (
          <div className="text-center py-8">
            <div className="animate-spin rounded-full h-12 w-12 border-b-2 border-blue-500 mx-auto"></div>
            <p className="mt-4 text-gray-600">読み込み中...</p>
          </div>
        )}

        {/* エラー表示 */}
        {error && (
          <div className="bg-red-50 border border-red-200 rounded-lg p-4 mb-4">
            <p className="text-red-800">{error}</p>
          </div>
        )}

        {/* タスク一覧表示 */}
        {!loading && !error && (
          <div className="bg-white rounded-lg shadow">
            {todos.length === 0 ? (
              <p className="p-8 text-center text-gray-500">
                タスクがまだありません
              </p>
            ) : (
              <ul className="divide-y divide-gray-200">
                {todos.map((todo) => (
                  <li key={todo.id} className="p-4">
                    <p className="text-gray-900">{todo.title}</p>
                    <p className="text-sm text-gray-500">
                      {todo.completed ? '完了' : '未完了'}
                    </p>
                  </li>
                ))}
              </ul>
            )}
          </div>
        )}
      </div>
    </div>
  );
}
2

コードの解説

  • • useState: タスク一覧、ローディング、エラーの状態を管理
  • • useEffect: ページ読み込み時にfetchTodos()を実行
  • • fetchTodos: fetch APIでGETリクエストを送信
  • • try-catch: エラーハンドリングでユーザーにメッセージを表示
  • • finally: 成功・失敗問わずローディングを終了

タスク追加関数を追加

frontend/app/page.tsx（追加部分） 
  // タスク追加関数
  const addTodo = async (title: string) => {
    try {
      setError(null);

      const response = await fetch(`${API_URL}/todos`, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({ title }),
      });

      if (!response.ok) {
        throw new Error('タスクの追加に失敗しました');
      }

      // 一覧を再取得して画面を更新
      await fetchTodos();
    } catch (err) {
      setError(err instanceof Error ? err.message : '不明なエラー');
      console.error('エラー:', err);
    }
  };

  // フォーム送信ハンドラー
  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    const formData = new FormData(e.currentTarget);
    const title = formData.get('title') as string;

    if (title.trim()) {
      addTodo(title);
      e.currentTarget.reset(); // フォームをクリア
    }
  };

更新・削除関数を追加

addTodo関数の後に、以下の関数を追加します。

frontend/app/page.tsx（追加部分） 
  // タスク完了状態の切り替え
  const toggleTodo = async (id: string, completed: boolean) => {
    try {
      setError(null);

      const response = await fetch(`${API_URL}/todos/${id}`, {
        method: 'PUT',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({ completed: !completed }),
      });

      if (!response.ok) {
        throw new Error('タスクの更新に失敗しました');
      }

      await fetchTodos();
    } catch (err) {
      setError(err instanceof Error ? err.message : '不明なエラー');
      console.error('エラー:', err);
    }
  };

  // タスク削除
  const deleteTodo = async (id: string) => {
    if (!confirm('このタスクを削除しますか?')) {
      return;
    }

    try {
      setError(null);

      const response = await fetch(`${API_URL}/todos/${id}`, {
        method: 'DELETE',
      });

      if (!response.ok) {
        throw new Error('タスクの削除に失敗しました');
      }

      await fetchTodos();
    } catch (err) {
      setError(err instanceof Error ? err.message : '不明なエラー');
      console.error('エラー:', err);
    }
  };

開発者ツールでの確認

ブラウザの開発者ツール（F12）→「ネットワーク」タブで、APIリクエストとレスポンスを確認できます。正常に通信できているか確認しましょう。

トラブルシューティング

CORSエラーが出る

エラーメッセージ: "Access to fetch has been blocked by CORS policy"

原因: API GatewayのCORS設定が不完全

解決方法:

  • • API GatewayコンソールでCORSを有効化
  • • 各メソッド（GET, POST, PUT, DELETE）でCORSを有効化
  • • APIを再デプロイ
タスクが追加されない

原因: Lambda関数のエラー、または権限不足

解決方法:

  • • CloudWatch Logsでエラーログを確認
  • • LambdaのIAMロールにDynamoDB書き込み権限があるか確認
  • • Lambda関数を直接テストして動作確認
環境変数が読み込まれない

原因: 開発サーバーの再起動忘れ

解決方法:

  • • .env.local作成後、必ず開発サーバーを再起動（Ctrl+C → npm run dev）
  • • NEXT_PUBLIC_プレフィックスが付いているか確認
  • • console.log(process.env.NEXT_PUBLIC_API_URL)でURLを確認
ページをリロードするとデータが消える

原因: データがDynamoDBに保存されていない可能性

解決方法:

  • • DynamoDBコンソールでデータが保存されているか確認
  • • Lambda関数のログでエラーがないか確認
  • • APIエンドポイントが正しいか確認
