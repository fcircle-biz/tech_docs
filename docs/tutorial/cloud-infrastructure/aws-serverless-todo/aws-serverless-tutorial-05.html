API Gatewayの設定

Lambda関数をインターネットから呼び出せるように、API Gatewayを設定します。REST APIを作成し、ブラウザからアクセスできるWebエンドポイントを公開します。

このステップで実装すること

  • REST APIの作成とリソース設定
  • HTTPメソッド(GET, POST, PUT, DELETE)の定義
  • Lambda関数との統合設定
  • CORS(Cross-Origin Resource Sharing)の設定
  • APIのデプロイとエンドポイントURL取得

前提条件

  • ステップ4でLambda関数が作成済み
  • AWSマネジメントコンソールにログイン可能
  • REST APIの基本概念を理解している(このステップで説明)

5.1 API Gatewayとは何か

API Gatewayは、Lambda関数やその他のAWSサービスをHTTP/HTTPSエンドポイントとして公開するマネージドサービスです。いわば「お店の受付窓口」のような役割を果たします。

API Gatewayの役割

  • • HTTPリクエストをLambda関数に転送
  • • 認証・認可の管理
  • • リクエスト/レスポンスの変換
  • • レート制限(DDoS対策)
  • • APIのバージョン管理

API Gatewayがリクエストを振り分ける仕組み

REST APIとは

REST(REpresentational State Transfer)は、Webシステムで広く使われるAPI設計のルールです。HTTPメソッドとURLを組み合わせて、データの操作を表現します。

HTTPメソッド 操作 例
GET データの取得 タスク一覧を取得
POST データの新規作成 新しいタスクを作成
PUT データの更新 タスクを更新
DELETE データの削除 タスクを削除

5.2 REST APIの作成

実装 5-1: REST APIの新規作成

API Gatewayで新しいREST APIを作成します。

1

API Gatewayコンソールを開く

AWSマネジメントコンソールで「API Gateway」と検索し、API Gatewayのダッシュボードを開きます。

2

REST APIを選択

「APIを作成」ボタンをクリックし、「REST API」(プライベートではない方)の「構築」をクリックします。

「HTTP API」ではなく「REST API」を選択してください。REST APIの方が機能が豊富です。

3

API設定の入力

以下の情報を入力します。

  • • プロトコル: REST
  • • 新しいAPIの作成: 新しいAPI
  • • API名: TodoAPI
  • • 説明: Todoアプリ用のREST API(任意)
  • • エンドポイントタイプ: リージョン
4

APIの作成

「APIの作成」をクリックします。API Gatewayのリソース画面が表示されます。

5.4 CORS(Cross-Origin Resource Sharing)の設定

CORSは、異なるドメイン(オリジン)からのHTTPリクエストを許可するセキュリティ機能です。フロントエンド(Amplify)からAPI Gateway(異なるドメイン)へのアクセスを可能にするため、CORS設定が必須です。

CORSが必要な理由

ブラウザは、セキュリティのため、異なるドメイン間の通信を制限します(同一オリジンポリシー)。
例えば、https://your-app.amplifyapp.com(フロントエンド)からhttps://api-id.execute-api.us-east-1.amazonaws.com(API Gateway)にアクセスする際、CORSが必要です。

実装 5-4: CORSの有効化

各リソースでCORSを有効にし、ブラウザからのアクセスを許可します。

1

/todosリソースのCORS設定

/todosリソースを選択し、「アクション」→「CORSの有効化」をクリックします。

以下の設定を確認します:

  • • Access-Control-Allow-Origin: *(すべてのドメインを許可)
  • • Access-Control-Allow-Headers: デフォルト値のまま
  • • Access-Control-Allow-Methods: GET, POST, OPTIONS

「CORSを有効にして既存のCORSヘッダーを置換」をクリックします。

2

/todos/{id}リソースのCORS設定

同様に/todos/{id}リソースで「アクション」→「CORSの有効化」を実行します。

  • • Access-Control-Allow-Methods: PUT, DELETE, OPTIONS

「CORSを有効にして既存のCORSヘッダーを置換」をクリックします。

本番環境では

Access-Control-Allow-Origin: *はすべてのドメインを許可するため、本番環境では特定のドメインに制限することが推奨されます。
例: https://your-app.amplifyapp.com

5.5 APIのデプロイ

API Gatewayで設定した変更は、デプロイするまで反映されません。本番環境で使えるようにするため、APIをデプロイします。

実装 5-5: APIを本番環境にデプロイ

作成したAPIを本番ステージにデプロイし、エンドポイントURLを取得します。

1

APIのデプロイ

リソースツリーでルート「/」を選択し、「アクション」→「APIのデプロイ」をクリックします。

  • • デプロイされるステージ: [新しいステージ]
  • • ステージ名: prod
  • • ステージの説明: 本番環境(任意)

「デプロイ」をクリックします。

2

エンドポイントURLの確認

デプロイが完了すると、ステージエディタ画面が表示されます。画面上部に「URLの呼び出し」が表示されます。

例: https://abc123def.execute-api.us-east-1.amazonaws.com/prod

このURLをコピーして、メモ帳などに保存しておきます。ステップ8で使用します。

3

エンドポイントの完全なURLを確認

各APIエンドポイントのURLは以下のようになります。

  • • GET タスク一覧: {ベースURL}/todos
  • • POST タスク作成: {ベースURL}/todos
  • • PUT タスク更新: {ベースURL}/todos/{id}
  • • DELETE タスク削除: {ベースURL}/todos/{id}

トラブルシューティング

デプロイ後もAPIが動作しない

原因: デプロイが正しく完了していない、またはキャッシュが残っている

解決方法:

  • • 「アクション」→「APIのデプロイ」を再度実行
  • • ブラウザのキャッシュをクリア
  • • エンドポイントURLが正しいか確認
CORSエラーが発生する

原因: CORS設定が不完全、またはデプロイされていない

解決方法:

  • • 各リソースで「CORSの有効化」を再実行
  • • OPTIONSメソッドが自動作成されているか確認
  • • APIを再デプロイ
  • • ブラウザの開発者ツールでエラー内容を確認
Lambda統合でエラーが出る

原因: Lambda関数名が間違っている、または権限が不足している

解決方法:

  • • Lambda関数名のスペルを確認
  • • Lambda関数のリージョンがAPI Gatewayと同じか確認
  • • 統合設定画面で「Lambda関数にアクセス許可を追加する」ダイアログが表示されたら「OK」をクリック
