4.1 AWS Lambdaとは何か

AWS Lambda（ラムダ）は、サーバーの管理を一切せずにコードを実行できる「サーバーレスコンピューティングサービス」です。コードをアップロードするだけで、AWSが自動的にリソースを用意して実行してくれます。

AWS Lambdaの特徴

  • • サーバーの用意・管理が不要
  • • 必要な時だけ実行され、使った分だけ課金
  • • 自動的にスケール（アクセスが増えても対応可能）
  • • 多数のプログラミング言語に対応（Python、Node.js、Javaなど）
flowchart TD A([イベント発生
API呼び出し、スケジュール等]) --> B[Lambda関数が起動] B --> C[コード実行
処理を行う] C --> D[DynamoDB等にアクセス
データ操作] D --> E([レスポンスを返す]) E --> F[Lambda関数終了]

Lambda関数の実行フロー

従来のサーバーとの違い

項目 従来のサーバー AWS Lambda
サーバー管理 必要（OS更新、セキュリティ対応など） 不要（AWSが管理）
起動時間 24時間稼働 必要な時だけ起動
料金体系 稼働時間に応じて課金 実行時間に応じて課金（使わなければ無料）
スケーリング 手動で台数を増減 自動でスケール

4.2 最初のLambda関数を作成する

実装 4-1: Hello World Lambda関数の作成

まずはシンプルな「Hello World」を返すLambda関数を作成し、基本操作を学びます。

1

Lambdaコンソールを開く

AWSマネジメントコンソールにログインし、検索ボックスで「Lambda」と入力して、Lambdaコンソールを開きます。

2

関数の作成を開始

「関数の作成」ボタンをクリックし、以下の設定を行います。

  • • オプション: 「一から作成」を選択
  • • 関数名: HelloWorldFunction
  • • ランタイム: Python 3.12 を選択
  • • アーキテクチャ: x86_64 のまま
3

実行ロールの設定

「アクセス権限」セクションで、以下を選択します。

  • • 実行ロール: 「基本的な Lambda アクセス権限で新しいロールを作成」を選択

IAMロールは、Lambda関数がAWSの他のサービスにアクセスするための権限を定義します。ここでは自動的に基本的なロールが作成されます。

4

関数を作成

画面下部の「関数の作成」ボタンをクリックします。数秒で関数が作成されます。

4.3 Lambda関数のコードを理解する

関数が作成されると、自動的にサンプルコードが表示されます。このコードの仕組みを理解しましょう。

デフォルトのコード（自動生成）

lambda_function.py 
import json

def lambda_handler(event, context):
    # TODO implement
    return {
        'statusCode': 200,
        'body': json.dumps('Hello from Lambda!')
    }

コードの仕組み

各部分の説明

1. lambda_handler 関数

Lambda関数のエントリーポイント（実行開始地点）です。この関数名は設定で変更可能ですが、デフォルトでは lambda_handler が使われます。

2. event パラメータ

Lambda関数を呼び出した際に渡されるデータ（入力情報）が格納されています。API Gatewayから呼び出される場合、リクエストの内容がここに入ります。

3. context パラメータ

実行環境に関する情報（関数名、実行時間の残り時間、メモリサイズなど）が格納されています。

4. return 値

Lambda関数の実行結果を返します。statusCode にHTTPステータスコード、body にレスポンス内容を設定します。

コードをカスタマイズしてみる

以下のコードに書き換えて、もう少し詳しい情報を返すようにしてみましょう。

lambda_function.py（カスタマイズ版） 
import json
import datetime

def lambda_handler(event, context):
    # 現在時刻を取得
    current_time = datetime.datetime.now().isoformat()

    # レスポンスメッセージを作成
    message = {
        'message': 'Hello from Lambda!',
        'timestamp': current_time,
        'function_name': context.function_name,
        'memory_limit': context.memory_limit_in_mb
    }

    # レスポンスを返す
    return {
        'statusCode': 200,
        'body': json.dumps(message, ensure_ascii=False)
    }

実装 4-2: コードの編集とデプロイ

Lambda関数のコードを編集して、変更を適用します。

1

コードエディタで編集

Lambda関数の詳細画面で「コード」タブを開き、上記のカスタマイズ版コードをコピーして貼り付けます。

2

変更をデプロイ

「Deploy」をクリックしないと変更が反映されないので注意してください。

4.5 CloudWatch Logsでログを確認する

Lambda関数が実行されると、自動的にログがCloudWatch Logsに記録されます。エラーが発生した時やデバッグ時に非常に役立ちます。

実装 4-4: ログの確認とprintデバッグ

CloudWatch Logsでログを確認し、print文でデバッグする方法を学びます。

1

print文を追加

Lambda関数のコードにprint文を追加してみます。

lambda_function.py 
import json
import datetime

def lambda_handler(event, context):
    # デバッグ用のログ出力
    print("Lambda関数が開始されました")
    print(f"受信したイベント: {event}")

    # 現在時刻を取得
    current_time = datetime.datetime.now().isoformat()
    print(f"現在時刻: {current_time}")

    # レスポンスメッセージを作成
    message = {
        'message': 'Hello from Lambda!',
        'timestamp': current_time,
        'function_name': context.function_name,
        'memory_limit': context.memory_limit_in_mb
    }

    print(f"レスポンス: {message}")

    # レスポンスを返す
    return {
        'statusCode': 200,
        'body': json.dumps(message, ensure_ascii=False)
    }
2

デプロイして再実行

3

CloudWatch Logsを開く

4

ログストリームを確認

4.6 IAMロールと権限の基本

Lambda関数が他のAWSサービス（DynamoDB、S3など）にアクセスするには、適切な権限を持つIAMロールが必要です。

flowchart LR A[Lambda関数] --> B[IAMロール
実行ロール] B --> C[ポリシー
権限設定] C --> D[DynamoDB] C --> E[S3] C --> F[CloudWatch Logs]

Lambda関数とIAMロールの関係

自動作成されたロールの確認

実装 4-5: IAMロールの確認

Lambda関数に自動作成されたIAMロールを確認します。

1

設定タブを開く

Lambda関数の詳細画面で「設定」タブをクリックし、左サイドバーから「アクセス権限」を選択します。

2

ロール名を確認

「実行ロール」セクションに、自動作成されたロール名が表示されます（例: HelloWorldFunction-role-xxxxx）。

3

ポリシーを確認

ロール名のリンクをクリックすると、IAMコンソールが開きます。デフォルトで以下のポリシーがアタッチされています。

  • • AWSLambdaBasicExecutionRole: CloudWatch Logsへのログ出力権限

DynamoDBアクセス権限の追加（次のステップで使用）

次のステップでDynamoDBにアクセスするため、今のうちに権限を追加しておきます。

実装 4-6: DynamoDBアクセス権限の追加

Lambda関数がDynamoDBにアクセスできるように権限を追加します。

1

IAMロールのページを開く

Lambda関数の「設定 > アクセス権限」から、ロール名のリンクをクリックしてIAMコンソールを開きます。

2

ポリシーをアタッチ

「許可を追加」→「ポリシーをアタッチ」をクリックし、検索ボックスで「DynamoDB」と入力します。

3

AmazonDynamoDBFullAccessを選択

AmazonDynamoDBFullAccess にチェックを入れて、「許可を追加」をクリックします。

本番環境では、必要最小限の権限のみを付与する「最小権限の原則」に従うことが推奨されますが、学習用途ではFullAccessを使用します。

Lambda関数の動作確認

  2. 2. 実行結果が「succeeded」になることを確認します。
  3. 3. CloudWatch Logsでprint文の出力が確認できることを確認します。

トラブルシューティング

テスト実行でエラーが出る

原因: コードに文法エラーがある、またはDeployを忘れている

解決方法:

  • • コードを再確認し、インデントやカンマの抜けがないかチェック
  • • 「Deploy」ボタンをクリックして変更を保存
  • • CloudWatch Logsでエラーの詳細を確認
CloudWatch Logsが表示されない

原因: Lambda関数が一度も実行されていない、またはIAMロールにCloudWatch Logsへの書き込み権限がない

解決方法:

  • • まずテストを実行してから、ログを確認
  • • IAMロールに「AWSLambdaBasicExecutionRole」ポリシーが含まれているか確認
  • • 数分待ってから再度確認（ログ反映に時間がかかる場合がある）
IAMロールにポリシーをアタッチできない

原因: IAMユーザーに十分な権限がない

解決方法:

  • • 管理者権限のあるIAMユーザーでログインしているか確認
  • • ルートユーザーでログインして、IAMユーザーに「IAMFullAccess」ポリシーを追加
