Lambda関数でCRUD操作を実装

DynamoDBと連携して、タスクの作成・読取・更新・削除（CRUD）を行うLambda関数を実装します。

このステップで実装すること

  • boto3を使ったDynamoDBへのデータ追加（put_item）
  • タスク一覧取得（scan）とタスク詳細取得（get_item）
  • タスク更新（update_item）と削除（delete_item）の実装
  • エラーハンドリングとバリデーション処理

前提条件

  • DynamoDBテーブル（todos）が作成されている（Step 3）
  • Lambda関数の基礎を理解している（Step 4）
  • API Gatewayの設定が完了している（Step 5）

6.1 CRUD操作とは

CRUD（クラッド）とは、データベースに対する4つの基本操作の頭文字を取ったものです。Webアプリケーションの多くは、このCRUD操作でデータを管理しています。

CRUD操作とは

  • • Create（作成）: 新しいデータを追加する
  • • Read（読取）: データを取得して表示する
  • • Update（更新）: 既存のデータを変更する
  • • Delete（削除）: データを削除する

CRUD操作の全体像

Todoアプリでの具体例

  • Create: 「牛乳を買う」という新しいタスクを追加
  • Read: 保存されているタスク一覧を画面に表示
  • Update: 「牛乳を買う」を「牛乳とパンを買う」に変更
  • Delete: 完了したタスクを削除

6.2 boto3（AWS SDK for Python）

boto3は、PythonからAWSサービスを操作するための公式ライブラリです。Lambda関数からDynamoDBを操作するために使用します。

boto3でできること

  • • DynamoDBにデータを追加・取得・更新・削除
  • • S3にファイルをアップロード・ダウンロード
  • • SNSでメール通知を送信
  • • その他、200以上のAWSサービスを操作可能
boto3の基本的な使い方 
# boto3のインポート
import boto3

# DynamoDBのリソースを取得
dynamodb = boto3.resource('dynamodb')

# テーブルを指定
table = dynamodb.Table('todos')

# これで、tableを使ってデータ操作ができます

6.3 タスク一覧取得（Read）の実装

実装 6-1: getTodos関数の作成

DynamoDBからすべてのタスクを取得する関数を作成します。

1

Lambda関数を作成

Lambdaコンソールで「関数の作成」をクリックし、以下を設定します。

  • • 関数名: getTodos
  • • ランタイム: Python 3.12
  • • 実行ロール: 既存のロール（Step 4で作成したロール）を使用
2

関数コードを実装

以下のコードをコードエディタに貼り付けます。

getTodos.py 
import json
import boto3
from decimal import Decimal

# DynamoDBリソースの取得
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('todos')

# Decimalを通常のfloat/intに変換するヘルパー関数
def decimal_default(obj):
    if isinstance(obj, Decimal):
        return int(obj) if obj % 1 == 0 else float(obj)
    raise TypeError

def lambda_handler(event, context):
    try:
        # DynamoDBからすべてのタスクを取得
        response = table.scan()
        items = response['Items']

        # createdAtで降順ソート（新しいタスクが上に）
        items.sort(key=lambda x: x.get('createdAt', ''), reverse=True)

        # レスポンスを返す
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps(items, default=decimal_default, ensure_ascii=False)
        }

    except Exception as e:
        print(f"エラー: {str(e)}")
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'タスクの取得に失敗しました'
            }, ensure_ascii=False)
        }

コード解説

  • table.scan(): テーブル内のすべてのデータを取得します（小規模なアプリ向け）
  • decimal_default: DynamoDBのDecimal型をJSONに変換可能な型に変換します
  • sort(): createdAt（作成日時）で降順ソートし、最新のタスクが先頭に来るようにします
  • Access-Control-Allow-Origin: CORSヘッダーを設定し、ブラウザからのアクセスを許可します

6.4 タスク作成（Create）の実装

実装 6-2: createTodo関数の作成

新しいタスクをDynamoDBに追加する関数を作成します。

1

Lambda関数を作成

関数名を createTodo として新しい関数を作成します。

2

関数コードを実装

以下のコードをコードエディタに貼り付けます。

createTodo.py 
import json
import boto3
import uuid
from datetime import datetime

# DynamoDBリソースの取得
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('todos')

def lambda_handler(event, context):
    try:
        # リクエストボディを解析
        body = json.loads(event['body'])
        title = body.get('title', '').strip()

        # バリデーション: タイトルが空でないか確認
        if not title:
            return {
                'statusCode': 400,
                'headers': {
                    'Content-Type': 'application/json',
                    'Access-Control-Allow-Origin': '*'
                },
                'body': json.dumps({
                    'error': 'タイトルは必須です'
                }, ensure_ascii=False)
            }

        # バリデーション: タイトルの長さを確認（1〜200文字）
        if len(title) > 200:
            return {
                'statusCode': 400,
                'headers': {
                    'Content-Type': 'application/json',
                    'Access-Control-Allow-Origin': '*'
                },
                'body': json.dumps({
                    'error': 'タイトルは200文字以内で入力してください'
                }, ensure_ascii=False)
            }

        # 新しいタスクの作成
        todo = {
            'id': str(uuid.uuid4()),  # 一意のIDを生成
            'title': title,
            'completed': False,
            'createdAt': datetime.now().isoformat(),
            'updatedAt': datetime.now().isoformat()
        }

        # DynamoDBに保存
        table.put_item(Item=todo)

        # 成功レスポンスを返す
        return {
            'statusCode': 201,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps(todo, ensure_ascii=False)
        }

    except Exception as e:
        print(f"エラー: {str(e)}")
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'タスクの作成に失敗しました'
            }, ensure_ascii=False)
        }

コード解説

  • uuid.uuid4(): 一意のID（UUID）を生成します。例: "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
  • datetime.now().isoformat(): 現在日時をISO 8601形式で取得します。例: "2025-01-24T10:30:00"
  • table.put_item(): DynamoDBにデータを追加します
  • バリデーション: タイトルの存在チェックと文字数制限をチェックしています
  • statusCode: 201: 新規作成成功を示すHTTPステータスコード

6.5 タスク更新（Update）の実装

実装 6-3: updateTodo関数の作成

既存のタスクのタイトルや完了状態を更新する関数を作成します。

1

Lambda関数を作成

関数名を updateTodo として新しい関数を作成します。

2

関数コードを実装

以下のコードをコードエディタに貼り付けます。

updateTodo.py 
import json
import boto3
from datetime import datetime
from decimal import Decimal

# DynamoDBリソースの取得
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('todos')

def decimal_default(obj):
    if isinstance(obj, Decimal):
        return int(obj) if obj % 1 == 0 else float(obj)
    raise TypeError

def lambda_handler(event, context):
    try:
        # パスパラメータからIDを取得
        todo_id = event['pathParameters']['id']

        # リクエストボディを解析
        body = json.loads(event['body'])

        # 更新する属性を動的に構築
        update_expression = "SET updatedAt = :updatedAt"
        expression_values = {
            ':updatedAt': datetime.now().isoformat()
        }

        # タイトルが送られてきた場合
        if 'title' in body:
            title = body['title'].strip()
            if not title:
                return {
                    'statusCode': 400,
                    'headers': {
                        'Content-Type': 'application/json',
                        'Access-Control-Allow-Origin': '*'
                    },
                    'body': json.dumps({
                        'error': 'タイトルは必須です'
                    }, ensure_ascii=False)
                }
            if len(title) > 200:
                return {
                    'statusCode': 400,
                    'headers': {
                        'Content-Type': 'application/json',
                        'Access-Control-Allow-Origin': '*'
                    },
                    'body': json.dumps({
                        'error': 'タイトルは200文字以内で入力してください'
                    }, ensure_ascii=False)
                }
            update_expression += ", title = :title"
            expression_values[':title'] = title

        # 完了状態が送られてきた場合
        if 'completed' in body:
            update_expression += ", completed = :completed"
            expression_values[':completed'] = bool(body['completed'])

        # DynamoDBを更新
        response = table.update_item(
            Key={'id': todo_id},
            UpdateExpression=update_expression,
            ExpressionAttributeValues=expression_values,
            ReturnValues='ALL_NEW'
        )

        # 更新後のアイテムを返す
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps(response['Attributes'], default=decimal_default, ensure_ascii=False)
        }

    except Exception as e:
        print(f"エラー: {str(e)}")
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'タスクの更新に失敗しました'
            }, ensure_ascii=False)
        }

API Gatewayと統合

「PUT /todos/{id}」メソッドとこの関数を統合し、APIをデプロイします。

コード解説

  • pathParameters['id']: URLから対象タスクのIDを取得します（例: /todos/abc-123）
  • UpdateExpression: 更新する属性を指定するSQL風の構文
  • ExpressionAttributeValues: UpdateExpressionで使う値を定義
  • 動的な更新: titleまたはcompletedのどちらか、または両方を更新できます
  • ReturnValues='ALL_NEW': 更新後の完全なアイテムを返します

実装 6-4: deleteTodo関数の作成

指定されたタスクをDynamoDBから削除する関数を作成します。

1

Lambda関数を作成

関数名を deleteTodo として新しい関数を作成します。

2

関数コードを実装

以下のコードをコードエディタに貼り付けます。

deleteTodo.py 
import json
import boto3

# DynamoDBリソースの取得
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('todos')

def lambda_handler(event, context):
    try:
        # パスパラメータからIDを取得
        todo_id = event['pathParameters']['id']

        # DynamoDBから削除
        table.delete_item(
            Key={'id': todo_id}
        )

        # 成功レスポンスを返す
        return {
            'statusCode': 204,  # No Content（削除成功）
            'headers': {
                'Access-Control-Allow-Origin': '*'
            }
        }

    except Exception as e:
        print(f"エラー: {str(e)}")
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'
            },
            'body': json.dumps({
                'error': 'タスクの削除に失敗しました'
            }, ensure_ascii=False)
        }
3

API Gatewayと統合

「DELETE /todos/{id}」メソッドとこの関数を統合し、APIをデプロイします。

コード解説

  • table.delete_item(): 指定されたキーのアイテムを削除します
  • Key={'id': todo_id}: 削除対象のタスクをIDで指定
  • statusCode: 204: 削除成功を示すHTTPステータスコード（No Content）
  • 注意: 存在しないIDを指定してもエラーにはならず、204が返ります

トラブルシューティング

Lambda関数実行時に「AccessDeniedException」が発生

原因: Lambda関数のIAMロールにDynamoDBへのアクセス権限がない

解決方法:

  • • IAMコンソールでLambda関数のロールを開く
  • • 「ポリシーをアタッチ」をクリック
  • • 「AmazonDynamoDBFullAccess」ポリシーを検索して追加
  • • Lambda関数を再実行
「ResourceNotFoundException」が発生

原因: テーブル名が間違っている、またはテーブルが存在しない

解決方法:

  • • DynamoDBコンソールで「todos」テーブルが存在するか確認
  • • Lambda関数のコード内のテーブル名が「todos」であることを確認
  • • リージョンが一致しているか確認（東京リージョン: ap-northeast-1）
API呼び出しで「Internal Server Error」が返ってくる

原因: Lambda関数内でエラーが発生している

解決方法:

  • • CloudWatch Logsで該当Lambda関数のログを確認
  • • エラーメッセージから原因を特定
  • • 「KeyError」の場合: 必要なキーが存在しないため、条件分岐を追加
  • • 「JSONDecodeError」の場合: リクエストボディのJSON形式が不正
curlで「CORS policy」エラーが出る

原因: CORSヘッダーが正しく設定されていない

解決方法:

  • • Lambda関数のレスポンスに「Access-Control-Allow-Origin」ヘッダーが含まれているか確認
  • • API GatewayでCORSが有効になっているか確認
  • • APIを再デプロイ
