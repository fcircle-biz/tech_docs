AWSアカウント作成と初期設定

AWSを使い始めるための準備をします。アカウントの作成から、安全に使うための設定まで行います。

このステップで実装すること

  • AWSアカウントの作成とメールアドレス認証
  • ルートユーザーの多要素認証（MFA）設定
  • 請求アラートの設定（予想外の課金を防ぐ）
  • 日常利用するIAMユーザーの作成

前提条件

  • 有効なメールアドレス（GmailなどのフリーメールでOK）
  • 有効なクレジットカード（無料枠内なら課金なし）
  • スマートフォン（MFA設定用）

1.1 AWSとは何か

AWS（Amazon Web Services）は、Amazonが提供するクラウドコンピューティングサービスです。サーバー、ストレージ、データベースなど、アプリケーション開発に必要なインフラをインターネット経由で利用できます。

AWSの特徴

  • • 初期費用不要、使った分だけ課金
  • • 世界中のデータセンターを利用可能
  • • 必要な時に必要なだけリソースを拡張可能
  • • 200以上のサービスから選択できる

1.2 AWSアカウントの作成

実装 1-1: AWSアカウントの作成

AWSの公式サイトから無料アカウントを作成します。

1

AWS公式サイトにアクセス

ブラウザで以下のURLを開き、「無料でサインアップ」をクリックします。

https://aws.amazon.com/jp/
2

メールアドレスとパスワードを設定

以下の情報を入力します。

  • • ルートユーザーのメールアドレス: 今後ログインに使用
  • • AWSアカウント名: 任意の名前（例: my-aws-account）
  • • パスワード: 大文字、小文字、数字、記号を含む強固なパスワード
3

連絡先情報の入力

氏名、住所、電話番号を入力します。個人用の場合は「パーソナル」を選択します。

4

お支払い情報の入力

クレジットカード情報を入力します。無料枠内の使用であれば課金されません。

本人確認のため、1ドルの仮課金が行われますが、すぐに返金されます。

5

本人確認（電話番号認証）

スマートフォンの電話番号を入力し、SMSまたは音声通話で送られてくる確認コードを入力します。

6

サポートプランの選択

個人学習用の場合は「ベーシックサポート - 無料」を選択します。

1.3 多要素認証（MFA）の設定

MFAは、パスワードに加えて認証アプリから生成される6桁のコードを使う二段階認証です。アカウントのセキュリティを大幅に向上させます。

実装 1-2: MFAデバイスの設定

スマートフォンに認証アプリをインストールし、AWSアカウントと連携します。

1

認証アプリのインストール

スマートフォンに以下のいずれかのアプリをインストールします。

  • • Google Authenticator（推奨）
  • • Microsoft Authenticator
  • • Authy
2

AWSマネジメントコンソールにログイン

作成したメールアドレスとパスワードでログインします。

3

IAMダッシュボードを開く

右上のアカウント名をクリック → 「セキュリティ認証情報」を選択

4

MFAデバイスの割り当て

「多要素認証（MFA）」セクションで「MFAデバイスの割り当て」をクリックし、「仮想MFAデバイス」を選択します。

5

QRコードをスキャン

表示されたQRコードを認証アプリでスキャンし、連続する2つのMFAコードを入力します。

1.4 請求アラートの設定

予想外の課金を防ぐため、一定金額を超えたらメール通知が届くように設定します。

実装 1-3: 請求アラートの作成

CloudWatchアラームを使って請求額の監視を設定します。

1

請求アラートの有効化

「請求情報とコスト管理」→「請求設定」→「請求アラートを受け取る」にチェックを入れます。

2

CloudWatchでアラームを作成

リージョンを「バージニア北部（us-east-1）」に変更してから、CloudWatchコンソールを開きます。

請求メトリクスはバージニア北部リージョンでのみ利用可能です。

3

アラーム条件の設定

「アラーム」→「アラームの作成」をクリックし、以下を設定します。

  • • メトリクス: 請求 > 合計推定料金
  • • 条件: 以上
  • • しきい値: 5（USD）
4

通知先の設定

SNSトピックを新規作成し、通知先のメールアドレスを登録します。

1.5 IAMユーザーの作成

ルートユーザーは強力な権限を持つため、日常的な作業には使いません。代わりに、IAMユーザーを作成して使用します。

実装 1-4: 管理者権限を持つIAMユーザーの作成

日常的に使用する管理者権限のIAMユーザーを作成します。

1

IAMコンソールを開く

サービス検索で「IAM」と入力し、IAMダッシュボードを開きます。

2

ユーザーの追加

「ユーザー」→「ユーザーを追加」をクリックし、ユーザー名を入力します（例: admin-user）。

3

アクセス権限の設定

「ポリシーを直接アタッチ」を選択し、AdministratorAccess を選択します。

4

コンソールアクセスの有効化

「コンソールパスワード」を自動生成し、「パスワードのリセットが必要」にチェックを入れます。

5

認証情報のダウンロード

作成完了後、認証情報（ユーザー名、パスワード、サインインURL）をCSVでダウンロードし、安全な場所に保存します。

この情報は後から確認できないため、必ずダウンロードしてください。

1.6 動作確認

IAMユーザーでログイン確認

作成したIAMユーザーでログインできることを確認します。

  1. 1. 一度ログアウトし、ダウンロードしたCSVに記載されたサインインURLにアクセスします。
  2. 2. IAMユーザー名とパスワードを入力してログインします。
  3. 3. 初回ログイン時にパスワード変更を求められるので、新しいパスワードを設定します。

期待される結果

IAMユーザーでAWSマネジメントコンソールにログインでき、右上に「ユーザー名 @ アカウントID」が表示されます。

トラブルシューティング

クレジットカード情報が受け付けられない

原因: カード情報の入力ミス、またはカード会社側のブロック

解決方法:

  • • カード番号、有効期限、CVV番号を再確認
  • • 別のクレジットカードを試す
  • • カード会社に連絡してAWSからの請求が許可されているか確認
MFAデバイスが設定できない

原因: 認証アプリの時刻がずれている

解決方法:

  • • スマートフォンの時刻設定を「自動」にする
  • • 認証アプリを再起動する
  • • QRコードを再度スキャンしてやり直す
IAMユーザーでログインできない

原因: サインインURLが間違っている、またはユーザー名・パスワードが間違っている

解決方法:

  • • ダウンロードしたCSVファイルを開いて情報を再確認
  • • サインインURLが正しいか確認（アカウントIDが含まれている）
  • • ルートユーザーでログインし、IAMユーザーのパスワードをリセット
