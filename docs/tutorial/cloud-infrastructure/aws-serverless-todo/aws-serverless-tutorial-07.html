ユーザーが操作する画面（UI）をNext.jsで作成します。モダンなReactフレームワークで、美しく使いやすいTodoアプリを構築しましょう。

7.1 Next.jsとは何か

Next.jsは、Reactをベースにした強力なフロントエンドフレームワークです。ページ遷移の高速化、サーバーサイドレンダリング、画像最適化など、現代的なWebアプリケーション開発に必要な機能が最初から組み込まれています。

Next.jsの特徴

  • • Reactをベースにした直感的なコンポーネント開発
  • • App Routerによるファイルベースのルーティング
  • • Tailwind CSSとの相性が抜群
  • • 開発サーバーによる自動リロード（Hot Reload）

7.4 タスク入力フォーム（TodoForm）の実装

新しいタスクを入力するためのフォームコンポーネントを作成します。ユーザーがタスクのタイトルを入力し、「追加」ボタンをクリックすると、新しいタスクが作成されます。

実装 7-3: TodoForm.tsxの作成

components/TodoForm.tsxファイルを作成し、以下のコードを記述します。

typescript 
// components/TodoForm.tsx
'use client';

import { useState } from 'react';

interface TodoFormProps {
  onAdd: (title: string) => void;  // タスク追加時のコールバック関数
}

export default function TodoForm({ onAdd }: TodoFormProps) {
  // 入力フィールドの値を管理するstate
  const [title, setTitle] = useState('');

  // フォーム送信時の処理
  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();  // ページリロードを防ぐ

    // 空白のみの入力を防ぐ
    if (title.trim() === '') {
      alert('タスクを入力してください');
      return;
    }

    // 親コンポーネントにタイトルを渡す
    onAdd(title);

    // 入力フィールドをクリア
    setTitle('');
  };

  return (
    <form onSubmit={handleSubmit} className="mb-6">
      <div className="flex gap-2">
        <input
          type="text"
          value={title}
          onChange={(e) => setTitle(e.target.value)}
          placeholder="新しいタスクを入力..."
          className="flex-1 px-4 py-3 border border-slate-300 rounded-lg
                     focus:outline-none focus:ring-2 focus:ring-emerald-500
                     focus:border-transparent"
        />
        <button
          type="submit"
          className="px-6 py-3 bg-emerald-500 text-white rounded-lg
                     hover:bg-emerald-600 transition-colors font-medium
                     shadow-sm hover:shadow-md"
        >
          追加
        </button>
      </div>
    </form>
  );
}

コードの解説

  • • 'use client': クライアントサイドで動作するコンポーネントであることを宣言
  • • useState: 入力値を管理するReactフック
  • • handleSubmit: フォーム送信時の処理（バリデーション + コールバック実行）
  • • onAdd: 親コンポーネントから渡される関数（実際のタスク追加処理）

7.5 タスク1件表示（TodoItem）の実装

タスク1件を表示するコンポーネントを作成します。チェックボックス、タイトル、編集ボタン、削除ボタンを含みます。

実装 7-4: TodoItem.tsxの作成

components/TodoItem.tsxファイルを作成し、以下のコードを記述します。

typescript 
// components/TodoItem.tsx
'use client';

import { useState } from 'react';

interface Todo {
  id: string;
  title: string;
  completed: boolean;
}

interface TodoItemProps {
  todo: Todo;
  onToggle: (id: string) => void;    // 完了/未完了を切り替え
  onDelete: (id: string) => void;    // タスク削除
  onUpdate: (id: string, title: string) => void;  // タスク更新
}

export default function TodoItem({ todo, onToggle, onDelete, onUpdate }: TodoItemProps) {
  const [isEditing, setIsEditing] = useState(false);  // 編集モードかどうか
  const [editTitle, setEditTitle] = useState(todo.title);  // 編集中のタイトル

  // 編集を確定
  const handleUpdate = () => {
    if (editTitle.trim() === '') {
      alert('タスクを入力してください');
      return;
    }
    onUpdate(todo.id, editTitle);
    setIsEditing(false);
  };

  // 編集をキャンセル
  const handleCancel = () => {
    setEditTitle(todo.title);
    setIsEditing(false);
  };

  return (
    <div className="flex items-center gap-3 p-4 bg-white border border-slate-200
                    rounded-lg hover:shadow-md transition-shadow">
      {/* チェックボックス */}
      <input
        type="checkbox"
        checked={todo.completed}
        onChange={() => onToggle(todo.id)}
        className="w-5 h-5 text-emerald-500 rounded focus:ring-emerald-500
                   cursor-pointer"
      />

      {/* タイトル表示/編集 */}
      {isEditing ? (
        <input
          type="text"
          value={editTitle}
          onChange={(e) => setEditTitle(e.target.value)}
          className="flex-1 px-3 py-1 border border-emerald-300 rounded
                     focus:outline-none focus:ring-2 focus:ring-emerald-500"
          autoFocus
        />
      ) : (
        <span
          className={`flex-1 ${
            todo.completed
              ? 'line-through text-slate-400'
              : 'text-slate-700'
          }`}
        >
          {todo.title}
        </span>
      )}

      {/* ボタン群 */}
      <div className="flex gap-2">
        {isEditing ? (
          <>
            <button
              onClick={handleUpdate}
              className="px-3 py-1 bg-emerald-500 text-white text-sm rounded
                         hover:bg-emerald-600 transition-colors"
            >
              保存
            </button>
            <button
              onClick={handleCancel}
              className="px-3 py-1 bg-slate-300 text-slate-700 text-sm rounded
                         hover:bg-slate-400 transition-colors"
            >
              キャンセル
            </button>
          </>
        ) : (
          <>
            <button
              onClick={() => setIsEditing(true)}
              className="px-3 py-1 bg-blue-500 text-white text-sm rounded
                         hover:bg-blue-600 transition-colors"
            >
              編集
            </button>
            <button
              onClick={() => {
                if (confirm('このタスクを削除しますか？')) {
                  onDelete(todo.id);
                }
              }}
              className="px-3 py-1 bg-red-500 text-white text-sm rounded
                         hover:bg-red-600 transition-colors"
            >
              削除
            </button>
          </>
        )}
      </div>
    </div>
  );
}

コードの解説

  • • isEditing: 編集モードとビューモードの切り替えを管理
  • • line-through: 完了済みタスクに取り消し線を表示
  • • confirm(): 削除前に確認ダイアログを表示

7.6 タスク一覧表示（TodoList）の実装

実装 7-5: TodoList.tsxの作成

components/TodoList.tsxファイルを作成し、以下のコードを記述します。

typescript 
// components/TodoList.tsx
'use client';

import TodoItem from './TodoItem';

interface Todo {
  id: string;
  title: string;
  completed: boolean;
}

interface TodoListProps {
  todos: Todo[];
  onToggle: (id: string) => void;
  onDelete: (id: string) => void;
  onUpdate: (id: string, title: string) => void;
}

export default function TodoList({ todos, onToggle, onDelete, onUpdate }: TodoListProps) {
  // タスクが0件の場合
  if (todos.length === 0) {
    return (
      <div className="text-center py-12 text-slate-400">
        <i className="fas fa-clipboard-list text-5xl mb-4"></i>
        <p>タスクがありません</p>
      </div>
    );
  }

  return (
    <div className="space-y-3">
      {todos.map((todo) => (
        <TodoItem
          key={todo.id}
          todo={todo}
          onToggle={onToggle}
          onDelete={onDelete}
          onUpdate={onUpdate}
        />
      ))}
    </div>
  );
}

コードの解説

  • • todos.map(): タスク配列をループして各タスクを表示
  • • key={todo.id}: React が各要素を識別するための一意のキー
  • • タスクが0件の場合は、空の状態を表示

7.7 フィルターボタン（FilterButtons）の実装

実装 7-6: FilterButtons.tsxの作成

components/FilterButtons.tsxファイルを作成し、以下のコードを記述します。

typescript 
// components/FilterButtons.tsx
'use client';

type Filter = 'all' | 'active' | 'completed';

interface FilterButtonsProps {
  currentFilter: Filter;
  onFilterChange: (filter: Filter) => void;
}

export default function FilterButtons({ currentFilter, onFilterChange }: FilterButtonsProps) {
  const filters: { value: Filter; label: string }[] = [
    { value: 'all', label: 'すべて' },
    { value: 'active', label: '未完了' },
    { value: 'completed', label: '完了済み' }
  ];

  return (
    <div className="flex gap-2 mb-6">
      {filters.map((filter) => (
        <button
          key={filter.value}
          onClick={() => onFilterChange(filter.value)}
          className={`px-4 py-2 rounded-lg font-medium transition-colors ${
            currentFilter === filter.value
              ? 'bg-emerald-500 text-white shadow-sm'
              : 'bg-white text-slate-600 border border-slate-300 hover:bg-slate-50'
          }`}
        >
          {filter.label}
        </button>
      ))}
    </div>
  );
}

コードの解説

  • • Filter型: 'all' | 'active' | 'completed' の3つの値のみを許可
  • • 選択中のフィルターは緑色でハイライト表示

7.8 メインページ（page.tsx）の更新

実装 7-7: app/page.tsxの更新

app/page.tsxファイルを開き、以下のコードに置き換えます。このステップでは、まだAPIとは連携せず、ローカルのstate（useState）でタスクを管理します。

typescript 
// app/page.tsx
'use client';

import { useState } from 'react';
import TodoForm from '@/components/TodoForm';
import TodoList from '@/components/TodoList';
import FilterButtons from '@/components/FilterButtons';

// タスクの型定義
interface Todo {
  id: string;
  title: string;
  completed: boolean;
}

// フィルターの型
type Filter = 'all' | 'active' | 'completed';

export default function Home() {
  // タスク一覧を管理するstate
  const [todos, setTodos] = useState<Todo[]>([]);

  // 現在のフィルターを管理するstate
  const [filter, setFilter] = useState<Filter>('all');

  // タスク追加（仮実装）
  const handleAddTodo = (title: string) => {
    const newTodo: Todo = {
      id: Date.now().toString(),  // 仮のID生成
      title,
      completed: false
    };
    setTodos([...todos, newTodo]);
  };

  // タスク完了/未完了の切り替え（仮実装）
  const handleToggleTodo = (id: string) => {
    setTodos(
      todos.map((todo) =>
        todo.id === id ? { ...todo, completed: !todo.completed } : todo
      )
    );
  };

  // タスク削除（仮実装）
  const handleDeleteTodo = (id: string) => {
    setTodos(todos.filter((todo) => todo.id !== id));
  };

  // タスク更新（仮実装）
  const handleUpdateTodo = (id: string, title: string) => {
    setTodos(
      todos.map((todo) =>
        todo.id === id ? { ...todo, title } : todo
      )
    );
  };

  // フィルター適用
  const filteredTodos = todos.filter((todo) => {
    if (filter === 'active') return !todo.completed;
    if (filter === 'completed') return todo.completed;
    return true;  // 'all' の場合
  });

  return (
    <div className="min-h-screen bg-gradient-to-br from-emerald-50 to-teal-50 py-8">
      <div className="max-w-3xl mx-auto px-4">
        {/* ヘッダー */}
        <header className="text-center mb-8">
          <h1 className="text-4xl font-bold text-slate-800 mb-2">
            📝 Todoリスト
          </h1>
          <p className="text-slate-600">タスクを管理して効率アップ！</p>
        </header>

        {/* メインカード */}
        <div className="bg-white rounded-2xl shadow-xl p-6">
          {/* タスク追加フォーム */}
          <TodoForm onAdd={handleAddTodo} />

          {/* フィルターボタン */}
          <FilterButtons
            currentFilter={filter}
            onFilterChange={setFilter}
          />

          {/* タスク一覧 */}
          <TodoList
            todos={filteredTodos}
            onToggle={handleToggleTodo}
            onDelete={handleDeleteTodo}
            onUpdate={handleUpdateTodo}
          />
        </div>

        {/* 統計情報 */}
        <div className="mt-6 text-center text-slate-600 text-sm">
          全{todos.length}件
          {' | '}
          未完了{todos.filter((t) => !t.completed).length}件
          {' | '}
          完了{todos.filter((t) => t.completed).length}件
        </div>
      </div>
    </div>
  );
}

コードの解説

  • • useState<Todo[]>([]): 空の配列でタスクを初期化
  • • Date.now().toString(): 現在時刻をID代わりに使用（仮実装）
  • • filteredTodos: 選択中のフィルターに応じてタスクを絞り込み
  • • Step 8でAPIと連携する際に、これらの関数を書き換えます

トラブルシューティング

npm run devでエラーが出る

原因: Node.jsのバージョンが古い、または依存パッケージのインストールが不完全

解決方法:

  • • Node.jsのバージョンを確認: node --version （18.17以上が必要）
  • • 依存パッケージを再インストール: npm install
  • • node_modulesフォルダを削除して再インストール: rm -rf node_modules && npm install
コンポーネントが表示されない

原因: ファイルパスが間違っている、またはimport文が正しくない

解決方法:

  • • ファイル名が正しいか確認（TodoForm.tsx、大文字小文字も一致）
  • • componentsフォルダがプロジェクトルートにあるか確認
  • • import文のパスが正しいか確認: @/components/...
  • • ブラウザの開発者ツール（F12）でエラーメッセージを確認
スタイルが適用されない

原因: Tailwind CSSが正しく設定されていない

解決方法:

  • • プロジェクト作成時にTailwind CSSを「Yes」にしたか確認
  • • 開発サーバーを再起動: Ctrl+Cで停止 → npm run dev
  • • ブラウザのキャッシュをクリア（Ctrl+Shift+R または Cmd+Shift+R）
