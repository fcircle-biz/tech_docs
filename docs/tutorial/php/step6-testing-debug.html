6.1 ブラウザでのアクセス確認

開発したユーザー管理システムをブラウザで確認し、正常に動作することを確認します。

実習 6-1: アプリケーションの起動確認

XAMPPのApacheとPostgreSQLが起動していることを確認し、ブラウザでアプリケーションにアクセスします。

手順
  1. XAMPPコントロールパネルでApacheが緑色（起動中）であることを確認
  2. PostgreSQLサービスが起動していることを確認
  3. ブラウザで以下のURLにアクセス：
    http://localhost/user-management/users/list.php
  4. ユーザー管理システムの一覧ページが表示されることを確認
アクセスできない場合の対処法
  • Apache未起動: XAMPPコントロールパネルでApacheを開始
  • フォルダがない: htdocs内にuser-managementフォルダがあるか確認
  • ファイルがない: index.phpファイルが存在するか確認
  • データベース接続エラー: PostgreSQLが起動しているか確認

6.2 全機能の動作テスト

ユーザー管理システムの各機能を順番にテストし、正常に動作することを確認します。

実習 6-2: 機能テストの実行

以下のチェックリストに従って、全ての機能をテストしてください。

📝 機能テストチェックリスト

以下の項目を順番に実行し、問題がないかチェックしてください：

1. ユーザー登録機能
  • トップページから「新規登録」リンクをクリック
  • ユーザー名、メールアドレス、フルネーム、生年月日を入力
  • 「登録する」ボタンをクリック
  • 「ユーザーを登録しました」メッセージが表示される
2. ユーザー一覧機能
  • トップページから「ユーザー一覧」リンクをクリック
  • 登録したユーザーがテーブルに表示される
  • ユーザー名、メール、フルネーム、登録日が正しく表示される
  • 「詳細」ボタンが各行に表示される
3. ユーザー詳細機能
  • 一覧から「詳細」ボタンをクリック
  • ユーザーの詳細情報が表示される
  • 「編集」と「削除」ボタンが表示される
  • 「一覧に戻る」ボタンで一覧ページに戻れる
4. ユーザー編集機能
  • 詳細ページから「編集」ボタンをクリック
  • 現在の情報がフォームに入力済みで表示される
  • 情報を変更して「更新する」ボタンをクリック
  • 「ユーザー情報を更新しました」メッセージが表示される
5. ユーザー削除機能
  • 詳細ページから「削除」ボタンをクリック
  • 削除確認ページが表示される
  • 削除対象のユーザー情報が表示される
  • 「削除する」ボタンをクリックし、確認ダイアログで「OK」
  • 一覧ページに戻り、ユーザーが削除されている
✅ 全てのチェックが完了した場合

おめでとうございます！ユーザー管理システムが正常に動作しています。

6.3 基本的なエラー確認

よくあるエラーとその対処法を確認し、問題が発生した際の解決方法を学習します。

実習 6-3: エラー確認と対処

以下のエラーが発生した場合の対処法を確認してください。

よくあるエラーと対処法
🚫 「データベースに接続できません」エラー

原因: PostgreSQLサービスが停止している

対処法:

  1. PostgreSQLサービスを起動する
  2. config/database.phpの接続情報を確認する
  3. データベース名、ユーザー名、パスワードが正しいか確認
🚫 「ページが見つかりません」エラー

原因: ファイルパスが間違っている

対処法:

  1. URLが正しいか確認（http://localhost/user-management/users/...）
  2. ファイルが正しい場所に保存されているか確認
  3. Apacheが起動しているか確認
🚫 「テーブルが見つかりません」エラー

原因: データベーステーブルが作成されていない

対処法:

  1. pgAdminでデータベースに接続
  2. usersテーブルが存在するか確認
  3. 存在しない場合は、step2のSQL文を実行してテーブルを作成
💡 エラー確認のコツ
  • ブラウザの開発者ツール（F12キー）でエラー詳細を確認
  • PHPエラーメッセージを注意深く読む
  • 手順を一つずつ確認して、どの段階でエラーが発生するかを特定
  • 設定ファイル（database.php等）の内容を確認
理解度確認クイズ
  1. PHPでエラーメッセージをログに出力する関数は？
  2. PHPでisset()関数とempty()関数の違いは？
  3. PHPでvar_dump()関数の用途は？
  4. PHPでexitとdieの違いは？
  5. PHPでmicrotime()関数の用途は？
  6. PHPでmemory_get_usage()関数の用途は？
解答を確認
  1. error_log()
  2. isset()は変数が定義されているかチェック、empty()は空かどうかチェック
  3. 変数の内容とデータ型を詳細に出力
  4. 機能的には同じ（処理を終了する）
  5. 現在時刻をマイクロ秒まで取得
  6. 現在のメモリ使用量を取得
