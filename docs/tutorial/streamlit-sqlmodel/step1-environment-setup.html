環境構築とDockerセットアップ

  • PostgreSQLコンテナの起動と接続確認

1.1 前提条件の確認

このチュートリアルを開始する前に、以下の要件が満たされていることを確認してください：

必要なシステム要件
  • OS: Windows 10/11 (64-bit)
  • RAM: 8GB以上推奨
  • HDD/SSD: 10GB以上の空き容量
  • CPU: 仮想化技術対応（Hyper-V、WSL2）

1.2 Docker Desktop for Windowsのインストール

Dockerは、アプリケーションをコンテナとして実行するためのプラットフォームです。PostgreSQLデータベースを簡単に起動するために使用します。

実習 1-1: Docker Desktopのインストール

Docker Desktop for Windowsをインストールし、WSL2バックエンドを設定します。

手順
  1. Docker Desktop for Windowsを公式サイトからダウンロード
  2. インストーラーを管理者権限で実行
  3. 「Use WSL 2 instead of Hyper-V」オプションを選択
  4. インストール完了後、システムを再起動
  5. Docker Desktopを起動し、初期設定を完了
インストール確認

PowerShellまたはコマンドプロンプトで以下のコマンドを実行：

docker --version
docker compose --version
期待される結果
Docker version 24.0.6, build ed223bc
Docker Compose version v2.21.0-desktop.1

1.3 WSL2の有効化と設定

WSL2（Windows Subsystem for Linux 2）を有効化することで、Dockerのパフォーマンスを向上させます。

実習 1-2: WSL2の設定

Windows機能を有効化し、WSL2をセットアップします。

手順
  1. 管理者権限でPowerShellを開く
  2. WSL機能を有効化：
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
  1. システムを再起動
  2. WSL2をデフォルトバージョンに設定：
wsl --set-default-version 2
確認方法
wsl --list --verbose

1.4 PostgreSQLコンテナの起動

DockerでPostgreSQLデータベースを起動し、アプリケーションで使用します。

実習 1-3: PostgreSQLコンテナの作成

Docker Composeを使用してPostgreSQLを起動します。

手順
  1. プロジェクト用のディレクトリを作成：
mkdir streamlit-user-management
cd streamlit-user-management
  1. docker-compose.ymlファイルを作成：
version: '3.8'

services:
  postgres:
    image: postgres:15
    container_name: streamlit_postgres
    restart: always
    environment:
      POSTGRES_DB: user_management
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: password123
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./docker/postgres/init.sql:/docker-entrypoint-initdb.d/init.sql

volumes:
  postgres_data:
  1. PostgreSQL初期化用のディレクトリとファイルを作成：
mkdir -p docker/postgres
  1. docker/postgres/init.sqlファイルを作成：
-- データベース初期化スクリプト
CREATE SCHEMA IF NOT EXISTS public;

-- タイムゾーンの設定
SET timezone = 'Asia/Tokyo';

-- データベース情報の確認
SELECT version();
SELECT current_database();
SELECT current_user;
  1. PostgreSQLコンテナを起動：
docker compose up -d
接続確認
docker compose exec postgres psql -U admin -d user_management -c "SELECT version();"
期待される結果

PostgreSQLのバージョン情報が表示されれば成功です。

1.5 Python仮想環境の構築

プロジェクト専用のPython環境を作成し、必要なパッケージをインストールします。

実習 1-4: Python仮想環境の作成

venvを使用してプロジェクト専用の環境を構築します。

手順
  1. Python仮想環境を作成：
python -m venv streamlit_env
  1. 仮想環境を有効化：
# Windows Command Prompt
streamlit_env\Scripts\activate

# PowerShell
streamlit_env\Scripts\Activate.ps1
  1. pipを最新版にアップデート：
python -m pip install --upgrade pip
  1. requirements.txtファイルを作成：
streamlit==1.28.1
sqlmodel==0.0.14
psycopg2-binary==2.9.7
pandas==2.1.1
plotly==5.17.0
python-dotenv==1.0.0
  1. パッケージをインストール：
pip install -r requirements.txt
インストール確認
pip list

1.7 環境構築の確認

すべての設定が正しく行われているかを確認します。

実習 1-6: 環境確認テスト

基本的なStreamlitアプリケーションを作成して動作確認を行います。

手順
  1. app/main.pyファイルを作成：
import streamlit as st
import psycopg2
from dotenv import load_dotenv
import os

# 環境変数を読み込み
load_dotenv()

def test_database_connection():
    """データベース接続をテスト"""
    try:
        conn = psycopg2.connect(
            host="localhost",
            port="5432",
            database=os.getenv("POSTGRES_DB"),
            user=os.getenv("POSTGRES_USER"),
            password=os.getenv("POSTGRES_PASSWORD")
        )
        cursor = conn.cursor()
        cursor.execute("SELECT version();")
        version = cursor.fetchone()
        cursor.close()
        conn.close()
        return True, version[0]
    except Exception as e:
        return False, str(e)

def main():
    st.title("🚀 Streamlit + SQLModel 環境確認")
    st.markdown("---")
    
    # 環境情報の表示
    st.subheader("環境情報")
    col1, col2 = st.columns(2)
    
    with col1:
        st.metric("Python Version", f"{st.session_state.get('python_version', 'Unknown')}")
        st.metric("Streamlit Version", st.__version__)
    
    with col2:
        import sqlmodel
        st.metric("SQLModel Version", sqlmodel.__version__)
        import pandas as pd
        st.metric("Pandas Version", pd.__version__)
    
    # データベース接続テスト
    st.subheader("データベース接続テスト")
    if st.button("接続テスト実行"):
        with st.spinner("データベースに接続中..."):
            success, result = test_database_connection()
            
            if success:
                st.success("✅ データベース接続成功！")
                st.info(f"PostgreSQL Version: {result}")
            else:
                st.error("❌ データベース接続失敗")
                st.error(f"エラー: {result}")
    
    # 次のステップ
    st.markdown("---")
    st.subheader("🎯 次のステップ")
    st.markdown("""
    環境構築が完了しました！次は以下の手順で進みます：
    
    1. **SQLModel**を使用したデータベース設計
    2. **ユーザーモデル**の作成
    3. **CRUD操作**の実装
    4. **Streamlit UI**の構築
    """)

if __name__ == "__main__":
    import sys
    st.session_state['python_version'] = f"{sys.version_info.major}.{sys.version_info.minor}.{sys.version_info.micro}"
    main()
  1. Streamlitアプリケーションを起動：
streamlit run app/main.py
  1. ブラウザで http://localhost:8501 にアクセス
  2. 「接続テスト実行」ボタンをクリックして動作確認
期待される結果
  • Streamlitアプリケーションが正常に起動
  • 環境情報が正しく表示
  • データベース接続テストが成功

1.8 トラブルシューティング

よくある問題と解決方法
  • Docker Desktop起動エラー: WSL2が正しく有効化されているか確認
  • ポート5432使用中エラー: 既存のPostgreSQLサービスを停止
  • 仮想環境エラー: Python PATH設定を確認
  • パッケージインストールエラー: pipを最新版にアップデート
理解度確認クイズ
  1. Dockerコンテナを起動するコマンドは何ですか？
  2. Python仮想環境を有効化するコマンドは何ですか？
  3. PostgreSQLの接続を確認するために使用したポート番号は？
  4. .envファイルの目的は何ですか？
  5. requirements.txtファイルに記載されているパッケージのうち、ORMライブラリはどれですか？
解答例
  1. docker compose up -d
  2. streamlit_env\Scripts\activate（Windows）
  3. 5432
  4. 環境変数（データベース接続情報など）を安全に管理するため
  5. SQLModel
