1.1 Node.js環境の確認とインストール

Reactアプリケーション開発には、Node.jsとnpmが必要です。まず、現在の環境を確認します。

実装 1-1: Node.js環境の確認

ターミナル（コマンドプロンプト）で以下のコマンドを実行して、Node.jsとnpmがインストールされているか確認します。

手順
  1. Node.jsバージョン確認: 
    node --version
  2. npmバージョン確認: 
    npm --version
期待される結果

Node.js: v18.0以上（推奨: v20.x LTS）

npm: 9.0以上

# 例:
$ node --version
v20.10.0

$ npm --version
10.2.3
トラブルシューティング

Node.jsがインストールされていない場合:

  1. Node.js公式サイトから最新のLTSバージョンをダウンロード
  2. インストーラーを実行してデフォルト設定でインストール
  3. インストール後、ターミナルを再起動して再確認

1.2 Viteを使用したReactプロジェクトの作成

Viteは、高速な開発サーバーとビルドツールを提供する次世代フロントエンドツールです。Reactプロジェクトを作成します。

実装 1-2: Reactプロジェクトの作成

Viteを使用して、新しいReactプロジェクトを作成します。

手順
  1. プロジェクト作成: 
    # Viteを使用してReactプロジェクトを作成
npm create vite@latest user-management-frontend -- --template react

# または対話式で作成する場合
npm create vite@latest

# プロジェクト名: user-management-frontend
# フレームワーク: React
# バリアント: JavaScript
  2. プロジェクトディレクトリに移動: 
    cd user-management-frontend
  3. 依存関係のインストール: 
    npm install
動作確認

開発サーバーを起動して、Reactアプリケーションが正常に動作することを確認します：

npm run dev
期待される結果

ブラウザで http://localhost:5173 を開くと、Vite + Reactのウェルカムページが表示されます。

  VITE v5.0.0  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help

1.3 プロジェクト構造の理解

作成されたReactプロジェクトのファイル構成を確認します。

プロジェクト構成
user-management-frontend/
├── node_modules/          # 依存パッケージ
├── public/                # 静的ファイル
│   └── vite.svg
├── src/                   # ソースコード
│   ├── assets/           # 画像などのアセット
│   │   └── react.svg
│   ├── App.css           # Appコンポーネントのスタイル
│   ├── App.jsx           # ルートコンポーネント
│   ├── index.css         # グローバルスタイル
│   └── main.jsx          # エントリーポイント
├── .eslintrc.cjs         # ESLint設定
├── .gitignore            # Git除外設定
├── index.html            # HTMLテンプレート
├── package.json          # プロジェクト設定と依存関係
├── vite.config.js        # Vite設定
└── README.md             # プロジェクト説明

主要ファイルの役割

  • index.html: アプリケーションのエントリーポイントHTML
  • src/main.jsx: JavaScriptのエントリーポイント、Reactアプリをマウント
  • src/App.jsx: ルートコンポーネント、アプリケーションの起点
  • vite.config.js: Viteのビルド設定
  • package.json: 依存パッケージとスクリプトの定義

1.5 ESLintとPrettierの設定

コード品質を保つため、ESLintとPrettierを設定します。

実装 1-4: Prettierの設定
手順
  1. Prettierの設定ファイル作成:

    プロジェクトルートに .prettierrc ファイルを作成します。

    {
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5",
  "printWidth": 80,
  "arrowParens": "always"
}
  2. VS Codeのフォーマット設定:

    VS Codeの設定（settings.json）に以下を追加します。

    {
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}

1.6 JSXの基本とReactコンポーネント

JSX（JavaScript XML）は、JavaScriptの中にHTML風の構文を書くことができるReactの特徴的な記法です。

実装 1-5: 初めてのReactコンポーネント作成
手順
  1. App.jsxを編集:

    src/App.jsx を以下のように編集します。

    import { useState } from 'react';
import './App.css';

function App() {
  const [count, setCount] = useState(0);

  return (
    <div className="App">
      <header className="App-header">
        <h1>React + Vite ユーザー管理システム</h1>
        <p>ステップ1: 開発環境構築完了！</p>

        <div className="card">
          <button onClick={() => setCount((count) => count + 1)}>
            カウント: {count}
          </button>
          <p>
            Edit <code>src/App.jsx</code> and save to test HMR
          </p>
        </div>

        <p className="read-the-docs">
          Click on the Vite and React logos to learn more
        </p>
      </header>
    </div>
  );
}

export default App;
動作確認

開発サーバーが起動している状態でファイルを保存すると、ブラウザが自動的に更新されます（Hot Module Replacement）。

期待される結果

ブラウザに「React + Vite ユーザー管理システム」というタイトルとカウントボタンが表示され、ボタンをクリックするとカウントが増えます。

1.7 JSXの基本概念

JSXの特徴

flowchart LR A["JavaScript
コード"] --> B["JSX
(HTML風の構文)"] B --> C["Babel
(トランスパイル)"] C --> D["React.createElement()
(JavaScript)"] D --> E["DOM
(表示)"]

JSXの重要ルール

  • 1つのルート要素: コンポーネントは必ず1つのルート要素を返す必要があります 
    // OK: 1つのdivで囲む
return (
  <div>
    <h1>Title</h1>
    <p>Content</p>
  </div>
);

// OK: Fragmentを使用
return (
  <>
    <h1>Title</h1>
    <p>Content</p>
  </>
);

// NG: 複数のルート要素
return (
  <h1>Title</h1>
  <p>Content</p>
);
  • className属性: HTMLの class ではなく className を使用 
    <div className="container">内容</div>
  • 中括弧で JavaScript式を埋め込み: 
    const name = "太郎";
return <h1>こんにちは、{name}さん！</h1>;
  • 自己閉じタグ: 子要素がない場合は自己閉じタグを使用 
    <img src="logo.png" alt="Logo" />
<input type="text" />
