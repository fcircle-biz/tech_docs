8.1 NextAuth.jsのセットアップ

実装 8-1: NextAuth.jsのインストールと基本設定
手順
  1. NextAuth.jsのインストール: 
    pnpm add next-auth @auth/prisma-adapter bcrypt
pnpm add -D @types/bcrypt
  2. 環境変数の設定: .envに追加 
    # .env
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=your-secret-key-here
  3. 認証APIルートの作成: app/api/auth/[...nextauth]/route.tsを作成 
    // app/api/auth/[...nextauth]/route.ts
import NextAuth from 'next-auth'
import { PrismaAdapter } from '@auth/prisma-adapter'
import CredentialsProvider from 'next-auth/providers/credentials'
import { prisma } from '@/lib/prisma'
import bcrypt from 'bcrypt'

const handler = NextAuth({
  adapter: PrismaAdapter(prisma),
  providers: [
    CredentialsProvider({
      name: 'Credentials',
      credentials: {
        email: { label: "Email", type: "email" },
        password: { label: "Password", type: "password" }
      },
      async authorize(credentials) {
        if (!credentials?.email || !credentials?.password) {
          return null
        }

        // ユーザー認証ロジック
        return null // 実際の実装はセキュリティ要件に応じて
      }
    })
  ],
  session: {
    strategy: 'jwt',
  },
  pages: {
    signIn: '/login',
  },
})

export { handler as GET, handler as POST }
期待される結果

✓ NextAuth.jsが正しくインストールされる

✓ 認証APIルートが動作する

✓ 環境変数が正しく設定される

8.2 ミドルウェアによるルート保護

実装 8-2: 認証ミドルウェアの設定
手順
  1. ミドルウェアの作成: middleware.tsを作成 
    // middleware.ts
import { withAuth } from 'next-auth/middleware'

export default withAuth({
  pages: {
    signIn: '/login',
  },
})

export const config = {
  matcher: ['/users/:path*', '/dashboard/:path*'],
}
期待される結果

✓ 未認証ユーザーが保護されたルートにアクセスできない

✓ ログインページにリダイレクトされる

セキュリティベストプラクティス
  • パスワードハッシング: bcryptを使用してパスワードを安全に保存
  • 環境変数: 秘密情報は必ず環境変数で管理
  • HTTPS: 本番環境では必ずHTTPSを使用
  • CSRF対策: NextAuth.jsが自動的にCSRF保護を提供
  • Rate Limiting: APIエンドポイントにレート制限を実装
