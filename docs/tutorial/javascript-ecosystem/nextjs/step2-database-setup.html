2.1 Docker Composeファイルの作成

PostgreSQLデータベースとAdminer管理UIをDockerコンテナで構築します。

実装 2-1: docker-compose.ymlの作成

プロジェクトのルートディレクトリにDocker Compose設定ファイルを作成します。

手順
  1. docker-compose.ymlファイルの作成: プロジェクトルートに以下の内容で作成 
    version: '3.8'

services:
  postgres:
    image: postgres:16-alpine
    container_name: nextjs-postgres
    environment:
      POSTGRES_USER: nextuser
      POSTGRES_PASSWORD: nextpass
      POSTGRES_DB: userdb
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U nextuser"]
      interval: 10s
      timeout: 5s
      retries: 5

  adminer:
    image: adminer:latest
    container_name: nextjs-adminer
    ports:
      - "8080:8080"
    depends_on:
      - postgres

volumes:
  postgres_data:
  2. Dockerコンテナの起動: ターミナルで以下を実行 
    # プロジェクトルートで実行
docker-compose up -d

# コンテナの状態確認
docker-compose ps
動作確認

Adminerにアクセスしてデータベース接続を確認します。

# ブラウザで以下にアクセス
http://localhost:8080

# ログイン情報
# サーバー: postgres
# ユーザー名: nextuser
# パスワード: nextpass
# データベース: userdb
期待される結果

✓ Adminerのログイン画面が表示される

✓ 正しい情報でログインできる

✓ userdbデータベースが存在する

トラブルシューティング

解決方法: 既存のPostgreSQLを停止するか、docker-compose.ymlでポート番号を変更（例: "5433:5432"）

2.3 Prismaスキーマ定義とマイグレーション

ユーザーテーブルのスキーマを定義し、データベースに反映します。

実装 2-3: スキーマの定義

Prismaスキーマファイルにユーザーモデルを定義します。

手順
  1. schema.prismaの編集: prisma/schema.prismaを以下のように編集 
    // prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(cuid())
  username  String   @unique
  email     String   @unique
  fullName  String
  bio       String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("users")
}
  2. マイグレーションの作成と実行: スキーマをデータベースに適用 
    # マイグレーションファイルの生成と適用
pnpm dlx prisma migrate dev --name init

# マイグレーション履歴の確認
pnpm dlx prisma migrate status
  3. Prisma Clientの生成: 型定義を生成 
    # Prisma Clientの再生成
pnpm dlx prisma generate
動作確認

データベースにテーブルが作成されたことを確認します。

2.4 Prisma Clientのセットアップ

Next.jsアプリケーションからPrismaを使用するためのクライアントを設定します。

実装 2-4: Prismaクライアントのインスタンス化

開発環境でのホットリロード対応を含めたPrismaクライアントを作成します。

手順
  1. libディレクトリの作成: プロジェクトルートにlibフォルダを作成 
    # ディレクトリ作成
mkdir lib
  2. prisma.tsファイルの作成: lib/prisma.tsを以下の内容で作成 
    // lib/prisma.ts
import { PrismaClient } from '@prisma/client'

const globalForPrisma = global as unknown as {
  prisma: PrismaClient | undefined
}

export const prisma =
  globalForPrisma.prisma ??
  new PrismaClient({
    log: process.env.NODE_ENV === 'development' ? ['query', 'error', 'warn'] : ['error'],
  })

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma
}
コードの説明
  • グローバル変数の使用: 開発環境でのホットリロード時に複数のPrismaClientインスタンスが作成されるのを防ぐ
  • ログ設定: 開発環境ではクエリログを出力し、本番環境ではエラーログのみ
  • シングルトンパターン: アプリケーション全体で1つのPrismaClientインスタンスを共有
期待される結果

✓ lib/prisma.tsファイルが作成される

✓ TypeScriptの型チェックがエラーなく通る

✓ 他のファイルからインポートして使用できる
