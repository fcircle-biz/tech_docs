3.1 Server ComponentsとClient Componentsの理解

Next.js 14 App Routerでは、デフォルトで全てのコンポーネントがServer Componentsとして動作します。

flowchart TB A[コンポーネント] --> B{インタラクティブ性
必要?} B -->|不要| C[Server Component
デフォルト] B -->|必要| D[Client Component
'use client'] C --> E[サーバーでレンダリング] C --> F[データベース直接アクセス] C --> G[小さいバンドルサイズ] D --> H[ブラウザでレンダリング] D --> I[useState/useEffect使用可] D --> J[イベントハンドラー]
実装 3-1: Server Componentの作成

データベースから直接データを取得するServer Componentを作成します。

手順
  1. ユーザー一覧ページの作成: app/users/page.tsxを作成 
    // app/users/page.tsx
import { prisma } from '@/lib/prisma'

// Server Componentはasync関数にできる
export default async function UsersPage() {
  // サーバーサイドで直接データベースにアクセス
  const users = await prisma.user.findMany({
    orderBy: {
      createdAt: 'desc'
    }
  })

  return (
    <div className="container mx-auto p-8">
      <h1 className="text-3xl font-bold mb-6">ユーザー一覧</h1>
      <div className="grid gap-4">
        {users.map((user) => (
          <div key={user.id} className="border p-4 rounded-lg">
            <h2 className="text-xl font-semibold">{user.fullName}</h2>
            <p className="text-gray-600">@{user.username}</p>
            <p className="text-sm text-gray-500">{user.email}</p>
          </div>
        ))}
      </div>
    </div>
  )
}

# ブラウザで http://localhost:3000/users にアクセス
期待される結果

✓ ユーザー一覧が表示される

✓ データベースから直接データが取得されている

✓ JavaScriptバンドルにPrismaクライアントが含まれていない

Server Componentの利点
  • 直接データアクセス: APIルートを経由せずデータベースに直接アクセス可能
  • セキュリティ: 機密情報がクライアントに送信されない
  • パフォーマンス: クライアントのJavaScriptバンドルサイズが小さい
  • SEO: サーバーレンダリングによりSEOに有利

実装 3-2: インタラクティブなボタンコンポーネント

'use client'ディレクティブを使用してClient Componentを作成します。

手順
  1. componentsディレクトリの作成: app/components/ディレクトリを作成 
    mkdir -p app/components
  2. DeleteButtonコンポーネントの作成: app/components/DeleteButton.tsxを作成 
    // app/components/DeleteButton.tsx
'use client'

import { useState } from 'react'

interface DeleteButtonProps {
  userId: string
  userName: string
}

export function DeleteButton({ userId, userName }: DeleteButtonProps) {
  const [isDeleting, setIsDeleting] = useState(false)

  const handleDelete = async () => {
    if (!confirm(`本当に ${userName} を削除しますか?`)) {
      return
    }

    setIsDeleting(true)
    try {
      // 後でServer Actionを実装
      console.log('削除処理:', userId)
      alert('削除しました（デモ）')
    } catch (error) {
      console.error('削除エラー:', error)
      alert('削除に失敗しました')
    } finally {
      setIsDeleting(false)
    }
  }

  return (
    <button
      onClick={handleDelete}
      disabled={isDeleting}
      className="px-4 py-2 bg-red-500 text-white rounded hover:bg-red-600 disabled:opacity-50"
    >
      {isDeleting ? '削除中...' : '削除'}
    </button>
  )
}

実装 3-4: エラーページの作成

手順
  1. エラー境界の作成: app/users/error.tsxを作成 
    // app/users/error.tsx
'use client'

import { useEffect } from 'react'

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  useEffect(() => {
    console.error('エラーが発生しました:', error)
  }, [error])

  return (
    <div className="container mx-auto p-8">
      <div className="bg-red-50 border border-red-200 rounded-lg p-6">
        <h2 className="text-2xl font-bold text-red-800 mb-4">エラーが発生しました</h2>
        <p className="text-red-600 mb-4">{error.message}</p>
        <button
          onClick={reset}
          className="px-4 py-2 bg-red-600 text-white rounded hover:bg-red-700"
        >
          再試行
        </button>
      </div>
    </div>
  )
}

    // app/users/[id]/page.tsx
import { prisma } from '@/lib/prisma'
import { notFound } from 'next/navigation'

export default async function UserDetailPage({
  params,
}: {
  params: { id: string }
}) {
  const user = await prisma.user.findUnique({
    where: { id: params.id }
  })

  if (!user) {
    notFound()
  }

  return (
    <div className="container mx-auto p-8">
      <h1 className="text-3xl font-bold mb-6">{user.fullName}</h1>
      <div className="bg-white rounded-lg shadow p-6">
        <p><strong>ユーザー名:</strong> @{user.username}</p>
        <p><strong>メールアドレス:</strong> {user.email}</p>
        {user.bio && <p><strong>自己紹介:</strong> {user.bio}</p>}
      </div>
    </div>
  )
}
