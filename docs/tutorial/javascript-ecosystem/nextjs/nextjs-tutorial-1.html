ステップ1: 環境構築とNext.jsプロジェクト作成

開発環境の準備とプロジェクト初期化

このステップで実装すること
  • Node.jsのインストールと環境確認
  • Visual Studio Codeのセットアップと拡張機能の導入
  • Docker Desktopのインストールと動作確認
  • create-next-appによるプロジェクト作成
  • TypeScript、Tailwind CSS、ESLint設定
  • プロジェクト構造の理解
  • 開発サーバーの起動とHello World確認
前提条件
  • ターミナル/コマンドプロンプトの基本的な操作ができること
  • パソコン(Windows/Mac/Linux)上で管理者権限でソフトウェアをインストール可能なこと
  • インターネット接続が安定していること

1.1 Node.jsのインストール

Next.jsはNode.js上で動作するため、まずNode.jsをインストールします。Node.js 18.17以上が必要です。

実装 1-1: Node.jsのインストール

Node.jsの公式ウェブサイトから最新LTSバージョンをダウンロードしてインストールします。

手順（Windows）
  1. 公式ウェブサイトへアクセス: https://nodejs.org/ にアクセスします。
  2. LTSバージョンをダウンロード: 「LTS」ボタンをクリックして、インストーラーをダウンロードします。
  3. インストーラーを実行: ダウンロードしたファイルをダブルクリックして実行します。
  4. デフォルト設定で進める: インストールウィザードに従い、ほぼデフォルト設定でOKをクリックします。
手順（macOS）
  1. Homebrewを使用: ターミナルで以下のコマンドを実行します。 
    brew install node@20
手順（Linux - Ubuntu/Debian）
  1. NodeSource リポジトリを追加: 
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
  2. Node.jsをインストール: 
    sudo apt-get install -y nodejs
動作確認

インストール後、ターミナルで以下のコマンドを実行してバージョンを確認します：

node --version  # v20.x.x 以上
npm --version   # 10.x.x 以上
期待される結果

Node.jsのバージョン20以上、npmのバージョン10以上が表示されればインストール成功です。

v20.10.0
10.2.4
トラブルシューティング

問題: 「node: command not found」と表示される

解決策: ターミナルを再起動するか、新しいターミナルウィンドウを開いてください。インストール後の環境設定が反映されます。

1.2 Visual Studio Codeのセットアップ

コード編集用のエディタとして、Visual Studio Code（VS Code）を使用します。

実装 1-2: VS Codeのインストール

VS Codeをインストールし、Next.js開発に必要な拡張機能を追加します。

手順
  1. VS Codeをダウンロード: https://code.visualstudio.com/ から自分のOSに対応したバージョンをダウンロードします。
  2. インストール: インストーラーを実行して、デフォルト設定でインストールします。
  3. 拡張機能を追加: VS Codeを開き、左サイドバーの拡張機能アイコン（四角形）をクリックします。
推奨する拡張機能

検索ボックスで以下の拡張機能を検索して「Install」ボタンをクリックします：

- ES7+ React/Redux/React-Native snippets (by dsznajder.es7-react-js-snippets)
- Prettier - Code formatter (by Prettier)
- ESLint (by Microsoft)
- Tailwind CSS IntelliSense (by Bradleys)
- Prisma (by Prisma)
- JavaScript (ES6) code snippets (by charalampos karalis)
動作確認

VS Codeを再起動して、すべての拡張機能がインストールされていることを確認します。

期待される結果

VS Codeで .js/.tsx ファイルを開くと、シンタックスハイライトとコード補完が機能します。

1.3 Docker Desktopのインストール

PostgreSQLはDockerコンテナで実行するため、Docker Desktopをインストールします。

実装 1-3: Docker Desktopのインストール

Docker Desktopをダウンロードしてインストールします。

手順
  1. Docker Desktopをダウンロード: https://www.docker.com/products/docker-desktop から自分のOSに対応したバージョンをダウンロードします。
  2. インストール実行: インストーラーを実行して、デフォルト設定でインストールします。
  3. Docker Desktopを起動: インストール完了後、Docker Desktopアプリケーションを起動します（初回起動時は数分かかります）。
動作確認

ターミナルで以下のコマンドを実行して、Dockerが正常に動作していることを確認します：

docker --version
docker run hello-world
期待される結果

Docker バージョン情報が表示され、hello-world コンテナが正常に実行されます。

Docker version 24.0.0, build abc1234
Hello from Docker!

1.4 Next.jsプロジェクトの作成

create-next-appを使用して、新しいNext.jsプロジェクトを初期化します。

実装 1-4: プロジェクトの初期化

create-next-appコマンドでプロジェクトを生成します。

手順
  1. プロジェクトを作成するディレクトリに移動: ターミナルで、プロジェクトを保存したいディレクトリに移動します。
  2. create-next-appを実行: 
    npx create-next-app@latest user-management-app
  3. プロンプトに応答: 以下のプロンプトが表示されたら、各項目に対して以下のように選択します： 
    ✔ Would you like to use TypeScript? … Yes
✔ Would you like to use ESLint? … Yes
✔ Would you like to use Tailwind CSS? … Yes
✔ Would you like to use `src/` directory? … Yes
✔ Would you like to use App Router? … Yes
✔ Would you like to customize the default import alias? … No
動作確認

プロジェクト作成完了後、プロジェクトディレクトリに移動して開発サーバーを起動します：

cd user-management-app
npm run dev
期待される結果

ターミナルに以下のメッセージが表示されます。ブラウザで http://localhost:3000 にアクセスすると、Next.jsのウェルカムページが表示されます。

▲ Next.js 14.0.0
- Local:        http://localhost:3000

✓ Ready in 2.5s
トラブルシューティング

問題: ポート3000が既に使用されている

解決策: 別のポートで起動します： 

PORT=3001 npm run dev

1.5 プロジェクト構造の理解

作成されたプロジェクトの構成を確認します。

現在のプロジェクト構成
user-management-app/
├── .next/                      # Next.jsビルド出力（自動生成）
├── node_modules/              # 依存パッケージ
├── public/                    # 静的ファイル
│   └── images/
├── src/
│   ├── app/                   # App Router（ルーティング）
│   │   ├── globals.css        # グローバルスタイル
│   │   ├── layout.tsx         # ルートレイアウト
│   │   └── page.tsx           # ホームページ
│   └── components/            # 再利用可能なコンポーネント
├── .env.local                 # 環境変数（ローカル）
├── .eslintrc.json            # ESLint設定
├── .gitignore                # Git除外設定
├── next.config.js            # Next.js設定
├── package.json              # npmパッケージ設定
├── postcss.config.js         # PostCSS設定
├── tailwind.config.ts        # Tailwind CSS設定
└── tsconfig.json             # TypeScript設定

1.6 最初のコンポーネント修正

デフォルトのホームページを簡単なHello Worldに変更します。

実装 1-5: ホームページの修正

src/app/page.tsx を編集して、シンプルなホームページを作成します。

手順
  1. ファイルを開く: VS Codeで src/app/page.tsx を開きます。
  2. コンテンツをクリア: 既存の内容をすべて削除します。
  3. 新しいコードを入力: 
    export default function Home() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-center bg-gray-100">
      <div className="text-center">
        <h1 className="text-4xl font-bold text-gray-900 mb-4">
          ユーザー管理システム
        </h1>
        <p className="text-xl text-gray-600">
          Next.js + Prisma + PostgreSQL実践チュートリアル
        </p>
      </div>
    </main>
  );
}
  4. ファイルを保存: Ctrl+S（またはCmd+S）でファイルを保存します。
動作確認

ブラウザを自動リロード（ホットリロード）して、変更が反映されます。http://localhost:3000 でホームページが表示されていることを確認します。

期待される結果

ブラウザに「ユーザー管理システム」というタイトルと説明文が表示されます。
