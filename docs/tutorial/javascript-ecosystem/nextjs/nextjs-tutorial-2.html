2.1 App Routerの概要

Next.js 13以降は「App Router」が推奨されています。Pages Routerよりも柔軟で、Server Componentsなどの新しい機能が使えます。

flowchart TD A["App Router
Next.js 13+"] --> B["ファイルベースルーティング"] A --> C["Server Components"] A --> D["Server Actions"] B --> E["src/app/page.tsx"] B --> F["src/app/users/page.tsx"] B --> G["src/app/users/[id]/page.tsx"] C --> H["デフォルトでServer Component"] D --> I["フォーム処理を簡潔に"]

2.2 基本的なルーティング

App Routerではファイルの配置がURLルートに直結します。

実装 2-1: 基本的なページルートの作成

users一覧ページと詳細ページのルートを作成します。

手順
  1. usersディレクトリを作成: src/app の直下に users というディレクトリを作成します。 
    mkdir -p src/app/users
  2. ユーザー一覧ページを作成: src/app/users/page.tsx を作成します。 
    // src/app/users/page.tsx
export default function UsersPage() {
  return (
    <div className="container mx-auto py-10">
      <h1 className="text-3xl font-bold mb-4">ユーザー一覧</h1>
      <p className="text-gray-600">このページにはユーザー一覧が表示されます。</p>
    </div>
  );
}
  3. 動的ルート用ディレクトリを作成: users の中に [id] ディレクトリを作成します。 
    mkdir -p src/app/users/\[id\]
  4. 詳細ページを作成: src/app/users/[id]/page.tsx を作成します。 
    // src/app/users/[id]/page.tsx
type Props = {
  params: {
    id: string;
  };
};

export default function UserDetailPage({ params }: Props) {
  return (
    <div className="container mx-auto py-10">
      <h1 className="text-3xl font-bold mb-4">ユーザー詳細</h1>
      <p className="text-gray-600">ユーザーID: {params.id}</p>
    </div>
  );
}
動作確認

開発サーバーが起動している状態で、以下のURLにアクセスします：

http://localhost:3000/users        # ユーザー一覧ページ
http://localhost:3000/users/123    # ID 123 のユーザー詳細ページ
期待される結果

各ページが正常に表示され、詳細ページではURLのパラメータ（ID）が正しく取得されます。

2.3 Server Components vs Client Components

Next.jsではデフォルトがServer Componentです。Client Componentは 'use client' ディレクティブで明示的に指定します。

実装 2-2: Client Componentの作成

インタラクティブな機能が必要な場合、Client Componentを使用します。

手順
  1. Client Component を作成: src/app/components/counter.tsx を作成します。 
    // src/app/components/counter.tsx
'use client'; // クライアントコンポーネント指定

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div className="p-4 border rounded">
      <p className="text-lg font-semibold">カウント: {count}</p>
      <button
        onClick={() => setCount(count + 1)}
        className="mt-2 px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600"
      >
        +1
      </button>
    </div>
  );
}
  2. Server Component でClient Componentを使用: ホームページに Counter を追加します。 
    // src/app/page.tsx
import Counter from '@/app/components/counter';

export default function Home() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-center bg-gray-100">
      <div className="text-center bg-white p-8 rounded shadow">
        <h1 className="text-4xl font-bold text-gray-900 mb-4">
          ユーザー管理システム
        </h1>
        <Counter />
      </div>
    </main>
  );
}
動作確認

ブラウザで http://localhost:3000 にアクセスし、カウンターボタンをクリックしてカウントが増加することを確認します。

期待される結果

ボタンをクリックするたびにカウント値が1ずつ増加します。これはClient ComponentのuseStateフックが機能している証拠です。

2.4 レイアウトの構造

layout.tsx ファイルはそのディレクトリ以下のすべてのページに適用されるレイアウトを定義します。

実装 2-3: 階層的なレイアウトの作成

ユーザー管理セクション用の専用レイアウトを作成します。

手順

export const metadata = {

  description: 'ユーザー一覧・詳細ページ',
};

export default function UsersLayout({
  children,

      </div>

/users および /users/[id] ページにアクセスして、レイアウトが適用されていることを確認します。

期待される結果

両ページ上部に「ユーザー管理」というナビゲーションが表示されます。

2.6 エラーハンドリング

error.tsx ファイルでエラーをハンドルし、ユーザーフレンドリーなエラーページを表示できます。

実装 2-5: エラーページの作成

グローバルエラーハンドラーを実装します。

手順
  1. エラーページを作成: src/app/error.tsx を作成します。 
    // src/app/error.tsx
'use client';

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string };
  reset: () => void;
}) {
  return (
    <div className="flex min-h-screen items-center justify-center bg-gray-100">
      <div className="text-center bg-white p-8 rounded shadow">
        <h1 className="text-3xl font-bold text-red-600 mb-4">エラーが発生しました</h1>
        <p className="text-gray-600 mb-4">{error.message}</p>
        <button
          onClick={() => reset()}
          className="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600"
        >
          もう一度試す
        </button>
      </div>
    </div>
  );
}
  2. 404 Not Foundページを作成: src/app/not-found.tsx を作成します。 
    // src/app/not-found.tsx
import Link from 'next/link';

export default function NotFound() {
  return (
    <div className="flex min-h-screen items-center justify-center bg-gray-100">
      <div className="text-center bg-white p-8 rounded shadow">
        <h1 className="text-5xl font-bold text-gray-900 mb-2">404</h1>
        <p className="text-xl text-gray-600 mb-4">ページが見つかりません</p>
        <Link href="/" className="text-blue-500 hover:underline">
          ホームへ戻る
        </Link>
      </div>
    </div>
  );
}
動作確認

http://localhost:3000/undefined-page にアクセスして、404ページが表示されることを確認します。

期待される結果

カスタムの404ページが表示されます。
