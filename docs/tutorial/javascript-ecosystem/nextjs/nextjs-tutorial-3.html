ステップ3: データベース設計とPrisma設定

前提条件

3.1 Docker ComposeによるPostgreSQL環境構築

Docker Composeを使用して、PostgreSQLとpgAdmin4をコンテナで実行します。

実装 3-1: docker-compose.ymlの作成

プロジェクトルートに docker-compose.yml ファイルを作成します。

手順
  1. docker-compose.yml を作成: プロジェクトルート（user-management-app）に以下の内容で作成します。 
    version: '3.8'

services:
  postgres:
    image: postgres:16-alpine
    container_name: nextjs_postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: userdb
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  pgadmin:
    image: dpage/pgadmin4
    container_name: nextjs_pgadmin
    restart: unless-stopped
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@example.com
      PGADMIN_DEFAULT_PASSWORD: admin
    ports:
      - "5050:80"
    depends_on:
      - postgres

volumes:
  postgres_data:
  2. Dockerコンテナを起動: ターミナルで以下のコマンドを実行します。 
    docker-compose up -d
  3. コンテナの状態を確認: 
    docker-compose ps
動作確認

ブラウザで以下のURLにアクセスします：

http://localhost:5050  # pgAdmin4
期待される結果

pgAdmin4のログイン画面が表示されます。ユーザー名：admin@example.com、パスワード：admin でログインできます。

トラブルシューティング

問題: ポート5432/5050が既に使用されている

解決策: docker-compose.ymlで別のポート番号を指定してください。例：5432を5433に変更するなど。

3.2 Prismaのインストール

PrismaはTypeScript対応のORM（Object-Relational Mapping）ツールです。

実装 3-2: Prismaの初期化

Prismaをプロジェクトにインストールし、初期化します。

手順
  1. Prismaをインストール: 
    npm install @prisma/client
npm install -D prisma
  2. Prismaを初期化: PostgreSQL用に初期化します。 
    npx prisma init --datasource-provider postgresql
期待される結果

プロジェクトルートに prisma/ ディレクトリと .env.local ファイルが生成されます。

3.3 環境変数の設定

.env.local ファイルに PostgreSQL の接続情報を設定します。

実装 3-3: .env.local の設定

データベース接続文字列を環境変数に設定します。

手順
  1. .env.local ファイルを編集: 以下の内容に更新します。 
    # .env.local
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/userdb"
動作確認

接続テストを実行：

npx prisma db push
期待される結果

「Successfully created databases」メッセージが表示されます。

3.4 Prisma Schemaの定義

Prisma Schemaでデータベースのテーブル構造を定義します。

実装 3-4: Userモデルの定義

prisma/schema.prisma を編集してユーザーテーブルを定義します。

手順
  1. schema.prisma を編集: prisma/schema.prisma の内容を以下に更新します。 
    // prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(cuid())
  name      String
  email     String   @unique
  password  String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("users")
}
動作確認

Prisma マイグレーション実行：

npx prisma migrate dev --name init
期待される結果

「Your database is now in sync with your schema」メッセージが表示され、prisma/migrations/ ディレクトリにマイグレーションファイルが生成されます。

3.5 Prisma Clientの使用

Prisma Clientを使用してデータベースにアクセスするユーティリティ関数を作成します。

実装 3-5: Prismaクライアント用ユーティリティの作成

src/lib/prisma.ts を作成してPrismaクライアントをシングルトンパターンで管理します。

手順
  1. src/lib ディレクトリを作成: 
    mkdir -p src/lib
  2. src/lib/prisma.ts を作成: 
    // src/lib/prisma.ts
import { PrismaClient } from '@prisma/client';

const globalForPrisma = global as unknown as { prisma: PrismaClient };

export const prisma =
  globalForPrisma.prisma ||
  new PrismaClient({
    log: ['query'],
  });

if (process.env.NODE_ENV !== 'production')
  globalForPrisma.prisma = prisma;
動作確認

Next.jsサーバーが起動していることを確認します。

期待される結果

エラーが表示されず、プロジェクトがビルドされます。

3.6 Prisma Studioでデータ管理

Prisma Studioは GUI でデータベースを管理できるツールです。

Prisma Studioの起動
npx prisma studio

http://localhost:5555 でPrisma Studioが開きます。ここからユーザーデータを視覚的に管理できます。
