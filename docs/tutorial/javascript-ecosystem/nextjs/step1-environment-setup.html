1.1 Node.js環境の準備

Next.js開発には、Node.js 18.17以上が必要です。Node.jsのバージョン管理にはnvm（Node Version Manager）の使用を推奨します。

実装 1-1: Node.jsのインストールと確認

Node.jsをインストールし、バージョンを確認します。

手順
  1. Node.jsバージョン確認: 既存のNode.jsバージョンを確認 
    node --version
# 推奨: v20.x以上
  2. Node.jsインストール: 必要に応じて公式サイトからインストール

    公式サイト: https://nodejs.org/

    LTS（Long Term Support）版をダウンロードしてインストールします。

  3. インストール確認: Node.jsとnpmが正常にインストールされたか確認 
    node --version
npm --version
期待される結果
v20.11.0
10.2.4

Node.js 18.17以上、npm 9以上が表示されれば成功です。

1.2 pnpmのインストール

pnpmは高速で効率的なパッケージマネージャーです。Next.jsプロジェクトでの使用を推奨します。

実装 1-2: pnpmのインストール
手順
  1. pnpmインストール: npmを使用してグローバルにインストール 
    npm install -g pnpm
  2. バージョン確認: pnpmが正常にインストールされたか確認 
    pnpm --version
期待される結果
9.12.0

pnpm 8以上が表示されれば成功です。

1.3 Next.jsプロジェクトの作成

create-next-appを使用して、Next.js 14のプロジェクトを作成します。

実装 1-3: Next.jsプロジェクトの作成

TypeScript、Tailwind CSS、App Routerを使用したNext.jsプロジェクトを作成します。

手順
  1. プロジェクト作成: create-next-appコマンドを実行 
    pnpm create next-app@latest user-management-app
  2. 設定オプション: 対話形式で以下を選択 
    ✔ Would you like to use TypeScript? … Yes
✔ Would you like to use ESLint? … Yes
✔ Would you like to use Tailwind CSS? … Yes
✔ Would you like to use `src/` directory? … No
✔ Would you like to use App Router? (recommended) … Yes
✔ Would you like to customize the default import alias? … No
  3. ディレクトリ移動: プロジェクトディレクトリへ移動 
    cd user-management-app
動作確認

プロジェクトが正常に作成されたことを確認します。

ls -la
期待される結果

以下のファイル・ディレクトリが作成されていれば成功です：

  • app/ - App Routerディレクトリ
  • public/ - 静的ファイル
  • package.json - 依存関係
  • next.config.js - Next.js設定
  • tailwind.config.ts - Tailwind設定
  • tsconfig.json - TypeScript設定

ブラウザでNext.jsのウェルカムページが表示されれば成功です！

  • Next.jsのロゴとドキュメントリンクが表示

1.8 App Routerの基本概念

Next.js 14のApp Routerは、従来のPages Routerとは異なる新しいルーティングシステムです。

flowchart LR A["app/"] --> B["page.tsx
(/)"] A --> C["about/"] C --> D["page.tsx
(/about)"] A --> E["users/"] E --> F["page.tsx
(/users)"] E --> G["[id]/"] G --> H["page.tsx
(/users/:id)"] style A fill:#667eea,color:#fff style B fill:#4ade80,color:#000 style D fill:#4ade80,color:#000 style F fill:#4ade80,color:#000 style H fill:#4ade80,color:#000
App Routerの重要な特徴
ファイル 役割
page.tsx ルートのUIを定義。URLパスに対応
layout.tsx 複数ページで共有するレイアウト
loading.tsx ローディング状態のUI
error.tsx エラー時のUI
not-found.tsx 404エラーページ
[id]/ 動的ルート（パラメータ）
実装 1-7: ルートレイアウトのカスタマイズ

app/layout.tsxをカスタマイズして、全ページ共通のナビゲーションを追加します。

手順

app/layout.tsxを以下のように修正：

import type { Metadata } from 'next'
import { Inter } from 'next/font/google'
import './globals.css'
import Link from 'next/link'

const inter = Inter({ subsets: ['latin'] })

export const metadata: Metadata = {
  title: 'User Management App',
  description: 'Next.js 14 Tutorial Application',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="ja">
      <body className={inter.className}>
        <nav className="bg-purple-600 text-white p-4">
          <div className="container mx-auto flex justify-between items-center">
            <Link href="/" className="text-xl font-bold">
              User Management
            </Link>
            <div className="space-x-4">
              <Link href="/" className="hover:text-purple-200">
                Home
              </Link>
              <Link href="/about" className="hover:text-purple-200">
                About
              </Link>
            </div>
          </div>
        </nav>
        {children}
      </body>
    </html>
  )
}
期待される結果

✓ 全ページに紫色のナビゲーションバーが表示

✓ HomeとAboutのリンクが機能

✓ ページ遷移が高速（クライアントサイドナビゲーション）

import type { Config } from 'tailwindcss'

const config: Config = {
  content: [

    './components/**/*.{js,ts,jsx,tsx,mdx}',

    extend: {
      colors: {

          50: '#f5f3ff',
          100: '#ede9fe',

          600: '#7c3aed',
          700: '#6d28d9',

          900: '#4c1d95',

export default config
