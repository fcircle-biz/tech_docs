4.1 SQLModelとORMの基本

SQLModel（エスキューエル モデル）は、FastAPIの作者が開発した、Pythonでデータベースとやりとりするためのライブラリです。FastAPIとの相性が良く、ORM（オブジェクト関係マッピング）という仕組みを使って、SQL文を直接書かなくてもPythonのコードでデータベース操作ができます。

ORMの仕組み

flowchart LR A["Pythonコード
todo = Todo
title='買い物'"] --> B["ORM
SQLModel"] B --> C["SQL文に変換
INSERT INTO todos"] C --> D["PostgreSQL
データベース"]

ORMを使うメリット

  • • SQL文を覚えなくても、Pythonのコードだけで書ける
  • • データベースを変更してもコードの変更が少ない
  • • タイプミスや文法ミスが減る

4.2 データベース接続設定の作成

実装 4-1: database.py の作成

PostgreSQLに接続するための設定ファイルを作成します。

1

backendフォルダで database.py を作成

VS Codeでbackendフォルダを開き、新しいファイル database.py を作成します。

2

データベース接続コードを記述

以下のコードを入力します。

backend/database.py 
from sqlmodel import create_engine, Session

# データベース接続URL
# 形式: postgresql://ユーザー名:パスワード@ホスト:ポート/データベース名
DATABASE_URL = "postgresql://todo_user:todo_password@localhost:5432/todo_db"

# データベースエンジンの作成
# エンジンは、データベースとの接続を管理する
engine = create_engine(DATABASE_URL, echo=True)

# データベースセッションを取得する関数
def get_db():
    """
    データベースセッションを取得し、使用後に自動でクローズする
    """
    with Session(engine) as session:
        yield session  # セッションを返す

コードの解説

DATABASE_URL: データベースへの接続情報。ユーザー名、パスワード、ホスト、ポート、データベース名を含みます。
create_engine: データベースエンジンを作成。これがデータベースとの接続を管理します。
Session: データベースとやりとりする際に使う「セッション」。with文で自動的に開閉されます。
get_db: セッションを取得して、使い終わったら自動でクローズする関数。

backendフォルダに新しいファイル models.py を作成します。

2

Todoモデルを定義

backend/models.py 
from sqlmodel import SQLModel, Field
from datetime import datetime
from typing import Optional

class Todo(SQLModel, table=True):
    """
    Todoテーブルを表すモデルクラス
    データベースの「todos」テーブルと対応する
    """
    __tablename__ = "todos"  # データベースのテーブル名

    # カラム定義
    id: Optional[int] = Field(default=None, primary_key=True)  # ID（主キー）
    title: str = Field(max_length=200)  # タイトル（必須、最大200文字）
    completed: bool = Field(default=False)  # 完了状態（デフォルトはFalse=未完了）
    created_at: datetime = Field(default_factory=datetime.now)  # 作成日時
    updated_at: datetime = Field(default_factory=datetime.now)  # 更新日時

from pydantic import BaseModel, Field
from datetime import datetime
from typing import Optional

class TodoBase(BaseModel):
    """
    Todoの基本情報
    """
    title: str = Field(..., min_length=1, max_length=200)  # タイトル（必須、1〜200文字）
    completed: bool = False  # 完了状態（デフォルトはFalse）

class TodoCreate(TodoBase):
    """
    Todo作成時のリクエスト
    """
    pass  # TodoBaseと同じ内容

class TodoUpdate(BaseModel):
    """
    Todo更新時のリクエスト
    すべてのフィールドをオプションにする
    """
    title: Optional[str] = Field(None, min_length=1, max_length=200)  # タイトル（任意、1〜200文字）
    completed: Optional[bool] = None

class TodoResponse(TodoBase):
    """
    APIレスポンス用のTodo
    """
    id: int  # ID
    created_at: datetime  # 作成日時
    updated_at: datetime  # 更新日時

    class Config:
        # ORMモード: SQLModelのモデルから自動変換できるようにする
        from_attributes = True

4.5 タスク取得処理の実装

実装 4-4: crud.py の作成（タスク一覧取得機能）

データベースからタスクを取得する処理を実装します。

1

crud.py を作成

backend/crud.py 
from sqlmodel import Session, select
from models import Todo
from typing import Optional
from datetime import datetime
import schemas

def get_todos(db: Session, status: Optional[str] = None):
    """
    タスク一覧を取得する

    Args:
        db: データベースセッション
        status: フィルター条件（"all", "active", "completed"）

    Returns:
        Todoのリスト
    """
    # 基本クエリ: todosテーブルからすべてを取得
    statement = select(Todo)

    # フィルター処理
    if status == "active":
        # 未完了（completed=False）のみ
        statement = statement.where(Todo.completed == False)
    elif status == "completed":
        # 完了済み（completed=True）のみ
        statement = statement.where(Todo.completed == True)
    # status=="all" または None の場合はフィルターなし（すべて）

    # 作成日時の新しい順で並べ替えて返す
    statement = statement.order_by(Todo.created_at.desc())
    return db.exec(statement).all()

未完了のみ取得"] B -->|"completed"| E["completed=True
完了済みのみ取得"] C --> F["作成日時の降順で並び替え"] D --> F E --> F F --> G["タスク一覧を返す"]

4.6 タスク一覧APIエンドポイントの作成

実装 4-5: main.py にタスク一覧取得APIを追加

main.pyを編集し、タスク一覧を取得するAPIエンドポイントを追加します。

1

main.py を以下の内容に更新

backend/main.py 
from fastapi import FastAPI, Depends, HTTPException
from sqlmodel import Session, SQLModel
from typing import List, Optional
from database import get_db, engine
from models import Todo
import schemas
import crud

# データベースのテーブルを作成（既に存在する場合は何もしない）
SQLModel.metadata.create_all(engine)

# FastAPIアプリケーションの作成
app = FastAPI(
    title="Todo API",
    description="Next.js + FastAPI Todoアプリのバックエンド",
    version="1.0.0"
)

# ルートエンドポイント
@app.get("/")
def read_root():
    """
    ルートパス - APIの動作確認用
    """
    return {"message": "Todo API is running!"}

# タスク一覧取得API
@app.get("/api/todos", response_model=List[schemas.TodoResponse])
def get_todos_api(
    status: Optional[str] = None,
    db: Session = Depends(get_db)
):
    """
    タスク一覧を取得

    Args:
        status: フィルター条件（all, active, completed）
        db: データベースセッション

    Returns:
        タスクのリスト
    """
    todos = crud.get_todos(db, status=status)
    return todos

トラブルシューティング

「ModuleNotFoundError: No module named 'sqlmodel'」

原因: SQLModelがインストールされていません。

解決方法: 仮想環境を有効化してからインストールしてください。

コマンドプロンプト 
cd backend
venv\Scripts\activate
pip install sqlmodel psycopg2-binary
「could not connect to server」

原因: PostgreSQLコンテナが起動していません。

解決方法:

コマンドプロンプト 
docker-compose up -d
空のリスト [] が返される

原因: データベースにデータが入っていません。

解決方法: pgAdminでテストデータを追加してください。

「relation "todos" does not exist」

原因: todosテーブルが作成されていません。

解決方法: ステップ2で作成したCREATE TABLE文をpgAdminで実行してください。
