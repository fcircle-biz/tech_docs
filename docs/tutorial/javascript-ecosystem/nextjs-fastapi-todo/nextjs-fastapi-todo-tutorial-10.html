フィルター機能の実装

「すべて」「未完了」「完了済み」でタスクを絞り込むフィルター機能を実装し、Todoアプリの基本機能を完成させます。

このステップで実装すること

  • 「すべて」「未完了」「完了済み」でタスクを絞り込み
  • クエリパラメータを使ったAPI呼び出し
  • ボタンの選択状態の表示

前提条件

  • タスクの追加機能が実装済み（Step 8）
  • タスクの完了切替・削除機能が実装済み（Step 9）
  • バックエンドがクエリパラメータをサポート

10.1 フィルター機能の仕組み

フィルター機能は、APIにクエリパラメータを追加することで、サーバー側でデータを絞り込みます。これにより、必要なデータだけを取得できます。 

flowchart LR
    A[ユーザー] -->|フィルター選択| B[フロントエンド]
    B -->|"GET /api/todos?status=all"| C[バックエンド]
    B -->|"GET /api/todos?status=active"| C
    B -->|"GET /api/todos?status=completed"| C
    C -->|絞り込んだデータ| D[データベース]
    D -->|結果を返す| C
    C -->|JSON| B
    B -->|画面に表示| A

クエリパラメータとは

URLの末尾に「?」で始まる追加情報のことです。例: ?status=active は「未完了のみ取得」を指定します。

frontend/components/Filter.tsx 
type FilterProps = {
  currentFilter: 'all' | 'active' | 'completed'
  onFilterChange: (filter: 'all' | 'active' | 'completed') => void
}

export default function Filter({ currentFilter, onFilterChange }: FilterProps) {
  const filters: { value: 'all' | 'active' | 'completed'; label: string }[] = [
    { value: 'all', label: 'すべて' },
    { value: 'active', label: '未完了' },
    { value: 'completed', label: '完了済み' },
  ]

  return (
    <div className="flex gap-2 mb-6">
      {filters.map((filter) => (
        <button
          key={filter.value}
          onClick={() => onFilterChange(filter.value)}
          className={`px-4 py-2 rounded-lg font-medium transition-colors ${
            currentFilter === filter.value
              ? 'bg-emerald-500 text-white'
              : 'bg-white text-slate-700 hover:bg-slate-100 border border-slate-300'
          }`}
        >
          {filter.label}
        </button>
      ))}
    </div>
  )
}

コードの解説

  • currentFilter: 現在選択されているフィルター状態
  • onFilterChange: フィルターが変更されたときに呼ばれる関数
  • className条件分岐: 選択中のボタンは緑色、それ以外は白色

frontend/app/page.tsx 
'use client'

import { useState, useEffect } from 'react'
import TodoList from '@/components/TodoList'
import TodoForm from '@/components/TodoForm'
import Filter from '@/components/Filter'

const API_BASE_URL = 'http://localhost:8000/api'

type Todo = {
  id: number
  title: string
  completed: boolean
  created_at: string
  updated_at: string
}

export default function Home() {
  const [todos, setTodos] = useState<Todo[]>([])
  const [filter, setFilter] = useState<'all' | 'active' | 'completed'>('all')
  const [loading, setLoading] = useState(false)

  const fetchTodos = async () => {
    setLoading(true)
    try {
      const response = await fetch(`${API_BASE_URL}/todos?status=${filter}`)
      if (!response.ok) {
        throw new Error('タスク一覧の取得に失敗しました')
      }
      const data = await response.json()
      setTodos(data)
    } catch (error) {
      console.error('エラー:', error)
      alert('タスク一覧の取得に失敗しました')
    } finally {
      setLoading(false)
    }
  }

  useEffect(() => {
    fetchTodos()
  }, [filter])

  return (
    <div className="min-h-screen bg-gradient-to-br from-blue-50 to-indigo-100 py-8 px-4">
      <div className="max-w-2xl mx-auto">
        <h1 className="text-3xl font-bold text-center text-slate-800 mb-8">
          Todoリスト
        </h1>

        <TodoForm onAdd={fetchTodos} />

        <Filter currentFilter={filter} onFilterChange={setFilter} />

        {loading ? (
          <div className="text-center py-8">
            <div className="inline-block animate-spin rounded-full h-8 w-8 border-4 border-emerald-500 border-t-transparent"></div>
            <p className="text-slate-600 mt-2">読み込み中...</p>
          </div>
        ) : (
          <TodoList todos={todos} onUpdate={fetchTodos} />
        )}
      </div>
    </div>
  )
}

重要なポイント

  • filter状態: 現在のフィルター状態を保持（all / active / completed）
  • useEffect依存配列: filterが変わると自動的にfetchTodosが実行される
  • クエリパラメータ: ?status=${filter} でバックエンドに絞り込み条件を送信

トラブルシューティング

フィルターボタンをクリックしても何も起こらない

原因: useEffectの依存配列にfilterが含まれていない、またはonFilterChangeが正しく渡されていません。

解決方法:

  1. page.tsxのuseEffectの依存配列に [filter] が含まれているか確認
  2. FilterコンポーネントにonFilterChangeが正しく渡されているか確認
  3. ブラウザのコンソール（F12キー）でエラーがないか確認
フィルターを変更してもタスクが絞り込まれない

原因: バックエンドのフィルター機能が正しく動作していません。

解決方法:

  1. http://localhost:8000/api/todos?status=active に直接アクセスして、未完了のタスクだけが返ってくるか確認
  2. FastAPIのmain.pyでクエリパラメータを正しく処理しているか確認
  3. FastAPIを再起動してみる
選択中のボタンの色が変わらない

原因: currentFilterの値が正しく渡されていないか、条件分岐が間違っています。

解決方法:

  1. Filter.tsxのclassNameの条件分岐（currentFilter === filter.value）が正しいか確認
  2. page.tsxからcurrentFilter={filter}が渡されているか確認
  3. ブラウザの開発者ツールでHTML要素を確認し、正しいクラスが適用されているか確認
