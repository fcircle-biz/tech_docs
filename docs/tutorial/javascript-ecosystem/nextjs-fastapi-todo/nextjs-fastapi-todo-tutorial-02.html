DockerでPostgreSQLデータベースを構築

Dockerを使って、データを保存するためのPostgreSQLデータベースを立ち上げます。

このステップで実装すること

  • プロジェクトフォルダの作成
  • docker-compose.ymlファイルの作成
  • PostgreSQLとpgAdminコンテナの起動
  • pgAdminでデータベースに接続
  • Todoテーブルの作成

前提条件

  • ステップ1の環境構築が完了していること
  • Docker Desktopが起動していること

2.1 Dockerとは何か

Dockerは、アプリケーションを「コンテナ」という箱の中で実行する技術です。コンテナを使うと、どのパソコンでも同じ環境を簡単に作ることができます。

Dockerを使う利点

  • • PostgreSQLを手動でインストールする必要がない
  • • コマンド一つでデータベースを起動・停止できる
  • • 環境を汚さず、削除も簡単
  • • チーム全員が同じ環境を使える

2.2 プロジェクトフォルダの作成

まず、Todoアプリのファイルを保存するフォルダを作成します。

実装 2-1: プロジェクトフォルダの作成

1

作業フォルダを作成

コマンドプロンプトを開き、以下のコマンドを実行します。

コマンドプロンプト 
# ホームディレクトリに移動
cd %USERPROFILE%

# todo-appフォルダを作成
mkdir todo-app

# 作成したフォルダに移動
cd todo-app
2

VS Codeでフォルダを開く

作成したフォルダをVS Codeで開きます。

コマンドプロンプト 
code .

VS Codeが起動し、todo-app フォルダが開きます。

2.3 docker-compose.ymlファイルの作成

docker-compose.ymlファイルは、Dockerコンテナの設定を記述するファイルです。このファイルを作成することで、PostgreSQLとpgAdminを一度に起動できます。

実装 2-2: docker-compose.ymlの作成

1

新しいファイルを作成

VS Codeで docker-compose.yml という名前の新しいファイルを作成します。

左側のエクスプローラーで、「新しいファイル」アイコンをクリックし、ファイル名を「docker-compose.yml」と入力してください。

2

設定を記述

以下の内容をコピーして、docker-compose.ymlファイルに貼り付けてください。

docker-compose.yml 
version: '3.8'

services:
  # PostgreSQLデータベース
  postgres:
    image: postgres:16
    container_name: todo_postgres
    environment:
      POSTGRES_USER: todo_user          # データベースのユーザー名
      POSTGRES_PASSWORD: todo_password  # データベースのパスワード
      POSTGRES_DB: todo_db              # データベース名
    ports:
      - "5432:5432"                     # ポート番号（外部:内部）
    volumes:
      - postgres_data:/var/lib/postgresql/data  # データを保存する場所

  # pgAdmin（データベース管理ツール）
  pgadmin:
    image: dpage/pgadmin4
    container_name: todo_pgadmin
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@example.com  # pgAdminのログインメール
      PGADMIN_DEFAULT_PASSWORD: admin           # pgAdminのログインパスワード
    ports:
      - "5050:80"                               # ポート番号（外部:内部）
    depends_on:
      - postgres

# データを永続化するためのボリューム
volumes:
  postgres_data:
3

ファイルを保存

Ctrl + S を押して、ファイルを保存してください。

設定の解説

postgres:

PostgreSQLデータベースのコンテナです。ポート5432で接続できます。

pgadmin:

データベースを視覚的に管理できるツールです。ブラウザで http://localhost:5050 にアクセスして使います。

volumes:

データを保存する場所です。コンテナを削除してもデータは残ります。

2.4 Dockerコンテナの起動

作成したdocker-compose.ymlを使って、PostgreSQLとpgAdminを起動します。

実装 2-3: Dockerコンテナの起動

1

Docker Desktopが起動していることを確認

タスクバーにDockerのアイコンがあり、「Docker Desktop is running」と表示されていればOKです。

2

コンテナを起動

VS Codeのターミナルを開き（Ctrl + `）、以下のコマンドを実行します。

ターミナル 
docker-compose up -d

-d オプションは、バックグラウンドで起動することを意味します。

3

起動を確認

以下のコマンドで、コンテナが起動していることを確認します。

ターミナル 
docker-compose ps

todo_postgres と todo_pgadmin の状態が「Up」になっていればOKです。

よく使うDockerコマンド

docker-compose up -d コンテナを起動（バックグラウンド）
docker-compose down コンテナを停止・削除
docker-compose ps コンテナの状態を確認
docker-compose logs コンテナのログを表示

2.5 pgAdminでデータベースに接続

pgAdminを使って、PostgreSQLデータベースに接続します。pgAdminはブラウザで動作するデータベース管理ツールです。

実装 2-4: pgAdminでデータベースに接続

1

pgAdminを開く

ブラウザで以下のURLを開きます。

URL: http://localhost:5050

2

ログイン

docker-compose.ymlに設定した情報でログインします。

  • Email: admin@example.com
  • Password: admin
3

サーバーを追加

左側のメニューで「Servers」を右クリック → 「Register」 → 「Server」を選択します。

General タブで以下を入力：

  • Name: Todo Database（任意の名前）

Connection タブで以下を入力：

  • Host name/address: postgres
  • Port: 5432
  • Maintenance database: todo_db
  • Username: todo_user
  • Password: todo_password

「Save」をクリックして保存します。

4

接続を確認

左側のツリーで「Todo Database」→「Databases」→「todo_db」が表示されればOKです。

注意

Host名は「localhost」ではなく「postgres」を指定してください。これはDockerコンテナ間の接続のためです。

2.6 Todoテーブルの作成

データを保存するためのテーブルを作成します。テーブルとは、データを整理して保存する「表」のことです。

Todoテーブルの設計

カラム名 型 説明
id SERIAL タスクの一意なID（自動採番）
title VARCHAR(200) タスクのタイトル（最大200文字）
completed BOOLEAN 完了フラグ（true/false）
created_at TIMESTAMP 作成日時
updated_at TIMESTAMP 更新日時

実装 2-5: テーブルを作成するSQLを実行

1

Query Toolを開く

pgAdminで「Todo Database」→「Databases」→「todo_db」を右クリックし、「Query Tool」を選択します。

2

SQLを実行

以下のSQLをコピーして、Query Toolに貼り付けて実行（▶ボタンをクリック）します。

SQL 
-- Todoテーブルを作成
CREATE TABLE todos (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    completed BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
3

テーブル作成を確認

左側のツリーを更新し、「Schemas」→「public」→「Tables」に「todos」テーブルが表示されればOKです。

4

（オプション）テストデータを挿入

動作確認のため、サンプルデータを追加してみましょう。

SQL 
-- サンプルデータを追加
INSERT INTO todos (title, completed) VALUES
('牛乳を買う', FALSE),
('レポートを提出', TRUE),
('部屋を掃除する', FALSE);

2.7 データベースの動作確認

データが正しく保存されているか確認

Query Toolで以下のSQLを実行し、データを取得してみましょう。

SQL 
-- すべてのTodoを取得
SELECT * FROM todos;

期待される結果

トラブルシューティング

コンテナが起動しない

原因: ポート5432や5050が既に使われている可能性があります。

解決方法:

  1. Docker Desktopを再起動する
  2. 他のPostgreSQLやアプリケーションを終了する
  3. docker-compose.ymlのポート番号を変更する（例: 5432 → 15432）
pgAdminにログインできない

原因: pgAdminコンテナが起動していないか、ログイン情報が間違っています。

解決方法:

  1. docker-compose ps でコンテナの状態を確認
  2. Email: admin@example.com、Password: admin を正確に入力
  3. コンテナを再起動: docker-compose restart
データベースに接続できない

原因: 接続情報が間違っています。

解決方法:

  1. Host name: postgres を指定（localhostではない）
  2. Username: todo_user、Password: todo_password を正確に入力
  3. Port: 5432 が正しいことを確認
テーブル作成時にエラーが出る

原因: 既にテーブルが存在しているか、SQL文にミスがあります。

解決方法:

  1. テーブルを削除してから再作成: DROP TABLE IF EXISTS todos;
  2. SQL文をもう一度コピー&ペーストして実行
  3. エラーメッセージを確認し、該当箇所を修正
