フロントエンドとバックエンドの連携

Next.jsからFastAPIを呼び出して、実際にデータを操作できるようにします。

このステップで実装すること

  • タスク一覧をAPIから取得して表示
  • 新規タスクの追加機能
  • タスクの完了/未完了切り替え
  • タスクの編集と削除機能
  • フィルター機能（すべて/未完了/完了済み）

前提条件

  • FastAPIバックエンドが実装済み
  • Next.jsフロントエンドが実装済み
  • PostgreSQLデータベースが起動中

7.1 フロントエンドとバックエンドの連携の仕組み

Next.js（フロントエンド）とFastAPI（バックエンド）は、HTTPリクエストを使って通信します。フロントエンドがAPIにリクエストを送り、バックエンドがデータベースとやり取りして結果を返します。 

sequenceDiagram
    participant User as ユーザー
    participant Next as Next.js
フロントエンド
localhost:3000
    participant FastAPI as FastAPI
バックエンド
localhost:8000
    participant DB as PostgreSQL
データベース

    User->>Next: タスク一覧を見たい
    Next->>FastAPI: GET /api/todos
    FastAPI->>DB: SELECT * FROM todos
    DB-->>FastAPI: タスク一覧データ
    FastAPI-->>Next: JSON形式で返す
    Next-->>User: 画面に表示

    User->>Next: 新しいタスクを追加
    Next->>FastAPI: POST /api/todos
    FastAPI->>DB: INSERT INTO todos
    DB-->>FastAPI: 保存完了
    FastAPI-->>Next: 作成したタスクを返す
    Next-->>User: 画面に追加表示

ポイント

フロントエンド（Next.js）はユーザーが見る「お店の店頭」で、バックエンド（FastAPI）は裏で処理する「バックヤード」です。両者がHTTP通信でやり取りすることで、アプリケーションが動作します。

7.2 CORS設定（バックエンド側）

異なるポート（Next.js: 3000番、FastAPI: 8000番）間で通信するには、CORS（Cross-Origin Resource Sharing）設定が必要です。これを設定しないと、ブラウザがセキュリティのためにリクエストをブロックします。

実装 7-1: FastAPIにCORS設定を追加

バックエンドの main.py にCORS設定を追加します。

1

main.pyを開く

VS Codeで backend/main.py を開きます。

2

CORSミドルウェアをインポート

ファイルの先頭に以下のインポートを追加します。

backend/main.py 
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware  # 追加
3

CORS設定を追加

FastAPIインスタンス作成後に、以下のCORS設定を追加します。

backend/main.py 
app = FastAPI()

# CORS設定
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],  # Next.jsのURL
    allow_credentials=True,
    allow_methods=["*"],  # すべてのHTTPメソッドを許可
    allow_headers=["*"],  # すべてのHTTPヘッダーを許可
)
4

FastAPIサーバーを再起動

設定を反映させるため、サーバーを再起動します。

コマンドプロンプト 
# Ctrl+C でサーバーを停止してから
cd backend
venv\Scripts\activate
uvicorn main:app --reload

7.3 タスク一覧の取得と表示

まずは、バックエンドからタスク一覧を取得して、画面に表示する機能を実装します。

実装 7-2: タスク一覧取得機能の実装

1

page.tsxを開く

VS Codeで frontend/app/page.tsx を開きます。

2

状態管理とAPI呼び出しを追加

タスク一覧を保持する状態と、APIから取得する関数を追加します。

frontend/app/page.tsx 
'use client'

import { useState, useEffect } from 'react'
import TodoList from '@/components/TodoList'
import TodoForm from '@/components/TodoForm'
import Filter from '@/components/Filter'

// APIのベースURL
const API_BASE_URL = 'http://localhost:8000/api'

// Todoの型定義
type Todo = {
  id: number
  title: string
  completed: boolean
  created_at: string
  updated_at: string
}

export default function Home() {
  const [todos, setTodos] = useState<Todo[]>([])
  const [filter, setFilter] = useState<'all' | 'active' | 'completed'>('all')
  const [loading, setLoading] = useState(false)

  // タスク一覧を取得する関数
  const fetchTodos = async () => {
    setLoading(true)
    try {
      const response = await fetch(`${API_BASE_URL}/todos?status=${filter}`)
      if (!response.ok) {
        throw new Error('タスク一覧の取得に失敗しました')
      }
      const data = await response.json()
      setTodos(data)
    } catch (error) {
      console.error('エラー:', error)
      alert('タスク一覧の取得に失敗しました')
    } finally {
      setLoading(false)
    }
  }

  // 初回とフィルター変更時にタスク一覧を取得
  useEffect(() => {
    fetchTodos()
  }, [filter])

  return (
    <div className="min-h-screen bg-gradient-to-br from-blue-50 to-indigo-100 py-8 px-4">
      <div className="max-w-2xl mx-auto">
        <h1 className="text-3xl font-bold text-center text-slate-800 mb-8">
          Todoリスト
        </h1>

        {/* タスク追加フォーム */}
        <TodoForm onAdd={fetchTodos} />

        {/* フィルター */}
        <Filter currentFilter={filter} onFilterChange={setFilter} />

        {/* ローディング表示 */}
        {loading && (
          <div className="text-center py-8">
            <div className="inline-block animate-spin rounded-full h-8 w-8 border-4 border-emerald-500 border-t-transparent"></div>
            <p className="text-slate-600 mt-2">読み込み中...</p>
          </div>
        )}

        {/* タスク一覧 */}
        {!loading && (
          <TodoList todos={todos} onUpdate={fetchTodos} />
        )}
      </div>
    </div>
  )
}

7.4 タスクの追加機能

新しいタスクをフォームから入力して、バックエンドのAPIに送信する機能を実装します。

実装 7-3: TodoFormコンポーネントの実装

1

TodoForm.tsxを開く

VS Codeで frontend/components/TodoForm.tsx を開きます。

2

タスク追加処理を実装

フォーム送信時にAPIにPOSTリクエストを送る処理を追加します。

frontend/components/TodoForm.tsx 
'use client'

import { useState } from 'react'

const API_BASE_URL = 'http://localhost:8000/api'

type TodoFormProps = {
  onAdd: () => void
}

export default function TodoForm({ onAdd }: TodoFormProps) {
  const [title, setTitle] = useState('')
  const [submitting, setSubmitting] = useState(false)

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault()

    if (!title.trim()) {
      alert('タスクを入力してください')
      return
    }

    setSubmitting(true)
    try {
      const response = await fetch(`${API_BASE_URL}/todos`, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({ title: title.trim() }),
      })

      if (!response.ok) {
        throw new Error('タスクの追加に失敗しました')
      }

      setTitle('') // フォームをクリア
      onAdd() // 一覧を再取得
    } catch (error) {
      console.error('エラー:', error)
      alert('タスクの追加に失敗しました')
    } finally {
      setSubmitting(false)
    }
  }

  return (
    <form onSubmit={handleSubmit} className="mb-6">
      <div className="flex gap-2">
        <input
          type="text"
          value={title}
          onChange={(e) => setTitle(e.target.value)}
          placeholder="新しいタスクを入力..."
          className="flex-1 px-4 py-2 border border-slate-300 rounded-lg
                     focus:outline-none focus:ring-2 focus:ring-emerald-500"
          disabled={submitting}
        />
        <button
          type="submit"
          disabled={submitting}
          className="px-6 py-2 bg-emerald-500 text-white rounded-lg
                     hover:bg-emerald-600 disabled:bg-slate-300
                     transition-colors font-medium"
        >
          {submitting ? '追加中...' : '追加'}
        </button>
      </div>
    </form>
  )
}

2

更新・削除機能を実装

タスクの完了切り替えと削除の処理を実装します。

frontend/components/TodoItem.tsx 
'use client'

const API_BASE_URL = 'http://localhost:8000/api'

type Todo = {
  id: number
  title: string
  completed: boolean
  created_at: string
  updated_at: string
}

type TodoItemProps = {
  todo: Todo
  onUpdate: () => void
}

export default function TodoItem({ todo, onUpdate }: TodoItemProps) {
  // 完了/未完了を切り替える
  const handleToggle = async () => {
    try {
      const response = await fetch(`${API_BASE_URL}/todos/${todo.id}`, {
        method: 'PUT',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          title: todo.title,
          completed: !todo.completed,
        }),
      })

      if (!response.ok) {
        throw new Error('タスクの更新に失敗しました')
      }

      onUpdate() // 一覧を再取得
    } catch (error) {
      console.error('エラー:', error)
      alert('タスクの更新に失敗しました')
    }
  }

  // タスクを削除する
  const handleDelete = async () => {
    if (!confirm('このタスクを削除しますか?')) {
      return
    }

    try {
      const response = await fetch(`${API_BASE_URL}/todos/${todo.id}`, {
        method: 'DELETE',
      })

      if (!response.ok) {
        throw new Error('タスクの削除に失敗しました')
      }

      onUpdate() // 一覧を再取得
    } catch (error) {
      console.error('エラー:', error)
      alert('タスクの削除に失敗しました')
    }
  }

  return (
    <div className="flex items-center gap-3 p-4 bg-white rounded-lg shadow-sm
                    border border-slate-200 hover:shadow-md transition-shadow">
      {/* チェックボックス */}
      <input
        type="checkbox"
        checked={todo.completed}
        onChange={handleToggle}
        className="w-5 h-5 rounded border-slate-300 text-emerald-500
                   focus:ring-emerald-500 cursor-pointer"
      />

      {/* タスクタイトル */}
      <span
        className={`flex-1 ${
          todo.completed
            ? 'line-through text-slate-400'
            : 'text-slate-800'
        }`}
      >
        {todo.title}
      </span>

      {/* 削除ボタン */}
      <button
        onClick={handleDelete}
        className="text-red-500 hover:text-red-700 transition-colors"
        title="削除"
      >
        <i className="fas fa-trash"></i>
      </button>
    </div>
  )
}

7.6 フィルター機能の実装

「すべて」「未完了」「完了済み」でタスクを絞り込むフィルター機能を実装します。

実装 7-5: Filterコンポーネントの実装

1

Filter.tsxを開く

VS Codeで frontend/components/Filter.tsx を開きます。

2

フィルターボタンを実装

3つのフィルターボタンを実装します。

frontend/components/Filter.tsx 
type FilterProps = {
  currentFilter: 'all' | 'active' | 'completed'
  onFilterChange: (filter: 'all' | 'active' | 'completed') => void
}

export default function Filter({ currentFilter, onFilterChange }: FilterProps) {
  const filters: { value: 'all' | 'active' | 'completed'; label: string }[] = [
    { value: 'all', label: 'すべて' },
    { value: 'active', label: '未完了' },
    { value: 'completed', label: '完了済み' },
  ]

  return (
    <div className="flex gap-2 mb-6">
      {filters.map((filter) => (
        <button
          key={filter.value}
          onClick={() => onFilterChange(filter.value)}
          className={`px-4 py-2 rounded-lg font-medium transition-colors ${
            currentFilter === filter.value
              ? 'bg-emerald-500 text-white'
              : 'bg-white text-slate-700 hover:bg-slate-100 border border-slate-300'
          }`}
        >
          {filter.label}
        </button>
      ))}
    </div>
  )
}

トラブルシューティング

CORSエラーが発生する

原因: CORS設定が正しくない、またはバックエンドが起動していません。

解決方法:

  1. FastAPIのmain.pyにCORSミドルウェアが追加されているか確認
  2. allow_originsに "http://localhost:3000" が含まれているか確認
  3. FastAPIサーバーを再起動
タスクが表示されない

原因: APIからデータが取得できていない、またはデータベースにデータがありません。

解決方法:

  1. ブラウザのコンソール（F12キー）でエラーメッセージを確認
  2. http://localhost:8000/api/todos にアクセスしてデータが取得できるか確認
  3. データベースが起動しているか確認（docker-compose ps）
タスクの追加・更新・削除ができない

原因: API呼び出しが失敗しているか、バックエンドのエンドポイントに問題があります。

解決方法:

  1. ブラウザのコンソール（F12キー）でエラーメッセージを確認
  2. FastAPIのSwagger UI（http://localhost:8000/docs）で各エンドポイントが動作するか確認
  3. fetchのURL（http://localhost:8000/api/todos）が正しいか確認
画面が真っ白になる

原因: Next.jsでエラーが発生しています。

解決方法:

  1. VS Codeのターミナルでエラーメッセージを確認
  2. ブラウザのコンソール（F12キー）でエラーを確認
  3. Next.jsサーバーを再起動（Ctrl+C → npm run dev）
