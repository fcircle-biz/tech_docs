5.1 CRUD操作とは

CRUDとは、データベース操作の基本となる4つの操作の頭文字を取ったものです。Webアプリケーション開発で必ず必要になる概念です。

CRUD操作の4つの基本

Create（作成）

新しいデータを追加する操作

HTTPメソッド: POST

Read（読取）

データを取得・表示する操作

HTTPメソッド: GET

Update（更新）

既存のデータを変更する操作

HTTPメソッド: PUT / PATCH

Delete（削除）

データを削除する操作

HTTPメソッド: DELETE

CRUD操作のフロー図

flowchart LR A[クライアント] -->|POST /api/todos| B[Create
タスク作成] A -->|GET /api/todos| C[Read
タスク取得] A -->|PUT /api/todos/:id| D[Update
タスク更新] A -->|DELETE /api/todos/:id| E[Delete
タスク削除] B --> F[(データベース)] C --> F D --> F E --> F

5.2 タスク作成API（POST）の実装

新しいタスクをデータベースに追加するAPIを実装します。POSTリクエストを受け取り、バリデーション後にデータを保存します。

実装 5-1: タスク作成エンドポイント

crud.py にタスク作成関数を追加します。

1

crud.py にタスク作成関数を追加

backend/crud.py を開き、以下の関数を追加します。

backend/crud.py（関数を追加） 
def create_todo(db: Session, todo: schemas.TodoCreate):
    """
    新しいタスクを作成する

    Args:
        db: データベースセッション
        todo: 作成するタスクの情報

    Returns:
        作成されたタスク
    """
    # タイトルの長さチェック（バリデーション）
    if len(todo.title) > 200:
        raise ValueError("タイトルは200文字以内で入力してください")

    # データベースモデルのインスタンスを作成
    db_todo = Todo(
        title=todo.title,
        completed=False  # 新規作成時は常に未完了
    )

    # データベースに追加
    db.add(db_todo)
    db.commit()  # コミット（実際に保存）
    db.refresh(db_todo)  # IDなど自動生成された値を取得

    return db_todo

main.py にPOSTエンドポイントを追加

backend/main.py にタスク作成エンドポイントを追加します。

backend/main.py（追加） 
@app.post("/api/todos", response_model=schemas.TodoResponse)
def create_new_todo(
    todo: schemas.TodoCreate,
    db: Session = Depends(get_db)
):
    """
    新しいタスクを作成するエンドポイント

    Args:
        todo: 作成するタスクの情報（titleを含む）
        db: データベースセッション

    Returns:
        作成されたタスク

    Raises:
        HTTPException: バリデーションエラー
    """
    try:
        return crud.create_todo(db=db, todo=todo)
    except ValueError as e:
        # バリデーションエラーの場合は400エラーを返す
        raise HTTPException(status_code=400, detail=str(e))

5.3 タスク更新API（PUT）の実装

既存のタスクを更新するAPIを実装します。タイトルの編集や完了/未完了の切り替えに使用します。

実装 5-2: タスク更新エンドポイント

1

crud.py にタスク更新関数を追加

backend/crud.py（追加） 
def update_todo(db: Session, todo_id: int, todo_update: schemas.TodoUpdate):
    """
    既存のタスクを更新する

    Args:
        db: データベースセッション
        todo_id: 更新するタスクのID
        todo_update: 更新する内容

    Returns:
        更新されたタスク、見つからない場合はNone
    """
    # IDでタスクを検索
    db_todo = db.get(Todo, todo_id)

    if db_todo is None:
        return None

    # タイトルが指定されていれば更新
    if todo_update.title is not None:
        if len(todo_update.title) > 200:
            raise ValueError("タイトルは200文字以内で入力してください")
        db_todo.title = todo_update.title

    # completedが指定されていれば更新
    if todo_update.completed is not None:
        db_todo.completed = todo_update.completed

    # 更新日時を現在時刻に設定
    db_todo.updated_at = datetime.now()

    db.commit()
    db.refresh(db_todo)

    return db_todo
2

main.py にPUTエンドポイントを追加

backend/main.py（追加） 
@app.put("/api/todos/{todo_id}", response_model=schemas.TodoResponse)
def update_existing_todo(
    todo_id: int,
    todo_update: schemas.TodoUpdate,
    db: Session = Depends(get_db)
):
    """
    既存のタスクを更新するエンドポイント

    Args:
        todo_id: 更新するタスクのID
        todo_update: 更新内容
        db: データベースセッション

    Returns:
        更新されたタスク

    Raises:
        HTTPException: タスクが見つからない、またはバリデーションエラー
    """
    try:
        db_todo = crud.update_todo(db=db, todo_id=todo_id, todo_update=todo_update)
        if db_todo is None:
            raise HTTPException(status_code=404, detail="指定されたタスクは存在しません")
        return db_todo
    except ValueError as e:
        raise HTTPException(status_code=400, detail=str(e))

5.4 タスク削除API（DELETE）の実装

タスクをデータベースから削除するAPIを実装します。完了したタスクや不要になったタスクを消去できるようにします。

実装 5-3: タスク削除エンドポイント

1

crud.py にタスク削除関数を追加

backend/crud.py（追加） 
def delete_todo(db: Session, todo_id: int):
    """
    タスクを削除する

    Args:
        db: データベースセッション
        todo_id: 削除するタスクのID

    Returns:
        削除されたタスク、見つからない場合はNone
    """
    # IDでタスクを検索
    db_todo = db.get(Todo, todo_id)

    if db_todo is None:
        return None

    # データベースから削除
    db.delete(db_todo)
    db.commit()

    return db_todo
2

main.py にDELETEエンドポイントを追加

backend/main.py（追加） 
@app.delete("/api/todos/{todo_id}", response_model=schemas.TodoResponse)
def delete_existing_todo(
    todo_id: int,
    db: Session = Depends(get_db)
):
    """
    タスクを削除するエンドポイント

    Args:
        todo_id: 削除するタスクのID
        db: データベースセッション

    Returns:
        削除されたタスク

    Raises:
        HTTPException: タスクが見つからない
    """

    if db_todo is None:
        raise HTTPException(status_code=404, detail="指定されたタスクは存在しません")

トラブルシューティング

「422 Unprocessable Entity」エラーが出る

原因: リクエストボディのバリデーションエラーです。

解決方法:

  • titleが空文字になっていないか確認
  • titleが200文字を超えていないか確認
  • JSONの形式が正しいか確認
「404 Not Found」エラーが出る

原因: 指定したIDのタスクが存在しません。

解決方法:

  • GETエンドポイントで存在するタスクのIDを確認
  • 正しいIDを指定しているか確認
  • すでに削除されたタスクを操作していないか確認
データベースに保存されない

原因: db.commit()が呼ばれていない、またはトランザクションエラーです。

解決方法:

  • crud.py の各関数でdb.commit()が呼ばれているか確認
  • PostgreSQLコンテナが起動しているか確認
  • database.py の接続情報が正しいか確認
