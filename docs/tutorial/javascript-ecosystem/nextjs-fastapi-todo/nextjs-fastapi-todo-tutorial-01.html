1.1 Visual Studio Codeのインストール

Visual Studio Code（VS Code）は、コードを書くための無料のエディタです。多くの開発者に使われており、便利な拡張機能が豊富にあります。

実装 1-1: VS Codeのインストールと拡張機能の追加

VS Codeをインストールし、開発に便利な拡張機能を追加します。

1

VS Codeをダウンロード

公式サイトから最新版をダウンロードします。

URL: https://code.visualstudio.com/

「Download for Windows」ボタンをクリックして、インストーラーをダウンロードしてください。

2

インストーラーを実行

ダウンロードした VSCodeSetup.exe を実行し、画面の指示に従ってインストールします。

インストール時の推奨設定：

  • 「PATHへの追加」をチェック
  • 「コンテキストメニューに追加」をチェック
3

推奨拡張機能をインストール

VS Codeを起動し、左側のメニューから拡張機能アイコン（四角が4つ並んだアイコン）をクリックして、以下の拡張機能を検索してインストールします。

  • Japanese Language Pack - VS Codeの日本語化
  • Python - Python開発サポート
  • ES7+ React/Redux/React-Native snippets - React開発支援
  • Tailwind CSS IntelliSense - Tailwind CSS補完
  • Docker - Docker操作支援

1.2 Node.jsのインストール

Node.jsは、JavaScriptをブラウザ外で実行するための環境です。Next.jsを動かすために必要です。

実装 1-2: Node.jsのインストールと動作確認

1

Node.jsをダウンロード

公式サイトから「LTS版」をダウンロードします。LTSは長期サポート版で安定しています。

URL: https://nodejs.org/
「LTS」と書かれたボタンをクリックしてダウンロードしてください。

2

インストーラーを実行

ダウンロードしたファイルを実行し、画面の指示に従ってインストールします。すべてデフォルト設定でOKです。

3

動作確認

コマンドプロンプトを開いて、以下のコマンドを実行します。

コマンドプロンプト 
node --version
npm --version

1.3 Pythonのインストール

PythonはFastAPIを動かすために必要です。バックエンド（サーバー側）の開発に使用します。

実装 1-3: Pythonのインストールと動作確認

1

Pythonをダウンロード

公式サイトから最新版（Python 3.10以上）をダウンロードします。

URL: https://www.python.org/downloads/
「Download Python 3.x.x」ボタンをクリックしてダウンロードしてください。

2

インストーラーを実行

インストーラーを起動したら、必ず「Add Python to PATH」にチェックを入れてください。これを忘れるとコマンドが使えなくなります。

重要

インストール画面の一番下にある「Add Python to PATH」に必ずチェックを入れてください。

3

動作確認

コマンドプロンプトを開いて、以下のコマンドを実行します。

コマンドプロンプト 
python --version
pip --version

1.4 Docker Desktopのインストール

Dockerは、PostgreSQLデータベースを簡単に起動・管理するために使用します。コンテナという技術を使って、環境を手軽に構築できます。

実装 1-4: Docker Desktopのインストールと起動確認

1

Docker Desktopをダウンロード

公式サイトからWindows版をダウンロードします。

URL: https://www.docker.com/products/docker-desktop/
「Download for Windows」ボタンをクリックしてダウンロードしてください。

2

インストーラーを実行

インストール中にWSL 2のインストールが求められた場合は「OK」をクリックします。インストール完了後、PCの再起動が必要です。

ヒント

WSL 2は「Windows Subsystem for Linux 2」の略で、Windows上でLinux環境を動かすための仕組みです。Dockerがこれを使用します。

3

Docker Desktopを起動

再起動後、Docker Desktopを起動します。初回起動時はチュートリアルが表示されますが、スキップして構いません。

4

動作確認

コマンドプロンプトで以下のコマンドを実行します。

コマンドプロンプト 
docker --version
docker-compose --version

1.5 環境セットアップの動作確認

すべてのツールが正しくインストールされたか確認

コマンドプロンプトで以下のコマンドをすべて実行して、バージョンが表示されることを確認してください。

コマンドプロンプト 
node --version
npm --version
python --version
pip --version
docker --version
docker-compose --version

期待される結果 

v20.x.x
10.x.x
Python 3.10.x （または3.11.x、3.12.xなど）
pip 23.x.x
Docker version 24.x.x
Docker Compose version v2.x.x

トラブルシューティング

「コマンドが見つかりません」と表示される

原因: ソフトウェアが正しくインストールされていないか、PATHが設定されていません。

解決方法:

  1. パソコンを再起動してみる
  2. 再起動後も同じエラーが出る場合は、ソフトウェアをアンインストールして再度インストール
  3. Pythonの場合は「Add Python to PATH」にチェックを入れてインストールし直す
Dockerが起動しない

原因: WSL 2が有効になっていない、または仮想化が無効になっています。

解決方法:

  1. Windowsの設定 → アプリ → オプション機能 → Windowsのその他の機能
  2. 「仮想マシン プラットフォーム」と「Linux用Windowsサブシステム」にチェック
  3. PCを再起動
バージョン番号が違う

原因: インストールした時期によってバージョンが異なります。

解決方法: 基本的に問題ありません。最小バージョン要件（Node.js 18.17以上、Python 3.10以上）を満たしていればOKです。
