FastAPIバックエンドの基礎

PythonのFastAPIを使って、データを処理するバックエンド（API）を作成します。

このステップで実装すること

  • FastAPIプロジェクトの作成とディレクトリ構成の理解
  • Python仮想環境（venv）の作成と必要パッケージのインストール
  • 最初のAPIエンドポイント（Hello World）を作成
  • Swagger UIでAPIをブラウザからテスト

前提条件

  • ステップ1（開発環境の準備）が完了していること
  • ステップ2（PostgreSQLデータベース構築）が完了していること
  • Pythonとpipが正常に動作していること

3.1 APIとは何か

API（Application Programming Interface）は、ソフトウェア同士がやり取りするための「窓口」です。Todoアプリでは、フロントエンド（Next.js）がバックエンド（FastAPI）にデータを要求したり、送信したりします。

フロントエンドとバックエンドのやりとり

sequenceDiagram participant F as フロントエンド
【Next.js】 participant B as バックエンド
【FastAPI】 participant D as データベース
【PostgreSQL】 F->>B: タスク一覧を取得したい B->>D: データベースからタスクを検索 D-->>B: タスクデータを返す B-->>F: タスク一覧を返す F->>F: 画面に一覧を表示

APIのメリット

  • 分離できる: フロントエンドとバックエンドを別々に開発できる
  • 再利用できる: 同じAPIを複数のアプリから使える（Webアプリ、スマホアプリなど）
  • セキュアにできる: データベースへの直接アクセスを防げる

3.3 Python仮想環境（venv）の作成

Pythonの仮想環境は、プロジェクトごとに独立したパッケージ環境を作る仕組みです。他のプロジェクトと干渉しないため、安全に開発できます。

実装 3-2: 仮想環境の作成と有効化

1

VS Codeでターミナルを開く

VS Code上でターミナルを開きます。

メニューバー → ターミナル → 新しいターミナル または Ctrl + `（バッククォート）

2

backendフォルダに移動

ターミナルで以下のコマンドを実行して、backendフォルダに移動します。

ターミナル 
cd backend
3

仮想環境を作成

venv という名前で仮想環境を作成します。

ターミナル 
python -m venv venv

これで backend/venv フォルダが作成されます。

4

仮想環境を有効化

仮想環境を有効化します。

ターミナル（Windows） 
venv\Scripts\activate

成功すると、プロンプトの先頭に (venv) と表示されます。

PowerShellでエラーが出る場合

「スクリプトの実行が無効です」というエラーが出た場合は、PowerShellを管理者権限で開き、以下のコマンドを実行してください。

PowerShell（管理者） 
Set-ExecutionPolicy RemoteSigned

3.4 必要なパッケージのインストール

FastAPIとその他必要なパッケージをインストールします。

実装 3-3: requirements.txtの作成とパッケージインストール

1

requirements.txtを作成

backend フォルダ内に requirements.txt ファイルを作成し、以下の内容を記述します。

backend/requirements.txt 
fastapi==0.104.1
uvicorn[standard]==0.24.0
sqlmodel==0.0.22
psycopg2-binary==2.9.9

各パッケージの役割：

  • fastapi: FastAPIフレームワーク本体
  • uvicorn: FastAPIを動かすためのサーバー
  • sqlmodel: データベース操作用ライブラリ（FastAPI作者が開発）
  • psycopg2-binary: PostgreSQL接続用ライブラリ
2

パッケージをインストール

仮想環境が有効化されている状態で、以下のコマンドを実行します。

ターミナル 
pip install -r requirements.txt

インストールには数分かかる場合があります。

3.5 最初のAPIエンドポイント作成

シンプルな「Hello World」APIを作成して、FastAPIが正しく動作することを確認します。

実装 3-4: main.pyの作成とHello Worldエンドポイント

1

main.pyを作成

backend フォルダ内に main.py ファイルを作成し、以下のコードを記述します。

backend/main.py 
# FastAPIをインポート
from fastapi import FastAPI

# FastAPIアプリケーションのインスタンスを作成
app = FastAPI()

# ルートエンドポイント（/）
@app.get("/")
def read_root():
    """
    ルートパスにアクセスしたときのレスポンス
    """
    return {"message": "Hello World"}

# API一覧取得用エンドポイント（/api/todos）
@app.get("/api/todos")
def get_todos():
    """
    タスク一覧を取得するAPI（現在はダミーデータを返す）
    """
    # ダミーデータ
    return [
        {"id": 1, "title": "牛乳を買う", "completed": False},
        {"id": 2, "title": "レポートを提出", "completed": True},
        {"id": 3, "title": "部屋を掃除する", "completed": False}
    ]
2

コードの説明

  • @app.get("/") - GETリクエストを受け付けるルートパス
  • @app.get("/api/todos") - タスク一覧を返すAPI
  • return {...} - JSON形式でデータを返す

3.6 FastAPIサーバーの起動とSwagger UI

作成したAPIサーバーを起動し、ブラウザからテストします。

実装 3-5: サーバー起動とAPI動作確認

1

サーバーを起動

ターミナルで以下のコマンドを実行します。

ターミナル（backendフォルダ内） 
uvicorn main:app --reload

オプションの意味:

  • main:app - main.pyのappを実行
  • --reload - コード変更時に自動でサーバーを再起動
2

起動確認

以下のようなメッセージが表示されれば成功です。 

INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
3

ブラウザでAPIにアクセス

ブラウザで以下のURLにアクセスします。

ルートパス:

http://localhost:8000

→ {"message": "Hello World"} と表示されればOK

タスク一覧API:

http://localhost:8000/api/todos

→ ダミーのタスク一覧が表示されればOK

4

Swagger UIでテスト

FastAPIには自動的にAPI仕様書（Swagger UI）が生成されます。ブラウザで以下のURLにアクセスしてください。

http://localhost:8000/docs

ここから各APIを試すことができます。「Try it out」ボタンを押して、「Execute」を押すと実行できます。

Swagger UIとは？

Swagger UIは、APIの仕様を視覚的に確認し、ブラウザから直接APIをテストできるツールです。FastAPIはこれを自動生成してくれるので、開発がとても楽になります。

3.7 動作確認

APIが正しく動作することを確認

以下の項目を確認してください：

  • ターミナルで uvicorn main:app --reload を実行してサーバーが起動
  • http://localhost:8000 で {"message": "Hello World"} が表示される
  • http://localhost:8000/api/todos でタスクのダミーデータが表示される
  • http://localhost:8000/docs でSwagger UIが開く

期待される画面（Swagger UI）

Swagger UIでは、以下のようなAPI一覧が表示されます：

  • GET / - ルートエンドポイント
  • GET /api/todos - タスク一覧取得

各APIをクリックして「Try it out」→「Execute」でテストできます。

トラブルシューティング

「ModuleNotFoundError: No module named 'fastapi'」と表示される

原因: 仮想環境が有効化されていない、またはパッケージがインストールされていません。

解決方法:

  1. 仮想環境を有効化: venv\Scripts\activate
  2. パッケージをインストール: pip install -r requirements.txt
「Address already in use」と表示される

原因: ポート8000が既に使用されています。

解決方法:

  1. 別のポートで起動: uvicorn main:app --reload --port 8001
  2. または、既に起動しているサーバーを停止（Ctrl+C）してから再度起動
ブラウザで「このサイトにアクセスできません」と表示される

原因: サーバーが起動していません。

解決方法:

  1. ターミナルでサーバーが起動しているか確認
  2. エラーメッセージが出ていないか確認
  3. URLが正しいか確認（http://localhost:8000）
