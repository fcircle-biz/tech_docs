タスク追加機能

新しいタスクを入力して追加できる機能を実装します。

このステップで実装すること

  • 新しいタスクを入力して追加できるようにする
  • POSTリクエストでバックエンドにデータを送信
  • 追加後に一覧を再取得して更新

8.1 タスク追加の流れ

ユーザーがフォームに入力したタスクをPOSTリクエストでバックエンドに送信し、データベースに保存します。保存が成功したら、タスク一覧を再取得して画面を更新します。 

sequenceDiagram
    participant User as ユーザー
    participant Next as Next.js
フロントエンド
localhost:3000
    participant FastAPI as FastAPI
バックエンド
localhost:8000
    participant DB as PostgreSQL
データベース

    User->>Next: タスクを入力して「追加」をクリック
    Next->>Next: 入力チェック（空文字でないか）
    Next->>FastAPI: POST /api/todos
（title: "牛乳を買う"）
    FastAPI->>DB: INSERT INTO todos
    DB-->>FastAPI: 保存完了（新規タスクID返却）
    FastAPI-->>Next: 作成したタスクを返す（JSON）
    Next->>Next: フォームをクリア
    Next->>FastAPI: GET /api/todos
（一覧再取得）
    FastAPI->>DB: SELECT * FROM todos
    DB-->>FastAPI: 全タスク取得
    FastAPI-->>Next: タスク一覧を返す（JSON）
    Next-->>User: 更新された一覧を表示

ポイント

タスクを追加したら、画面に反映するために一覧を再取得します。これにより、データベースに保存された最新の状態が画面に表示されます。

8.2 TodoFormコンポーネントの作成

タスクを入力するフォームを作成し、POSTリクエストでバックエンドに送信する機能を実装します。

実装 8-1: TodoFormコンポーネントの実装

1

TodoForm.tsxを開く

VS Codeで frontend/components/TodoForm.tsx を開きます。

2

フォームとPOST処理を実装

以下のコードを実装します。フォーム送信時にAPIにPOSTリクエストを送り、成功したら一覧を再取得します。

frontend/components/TodoForm.tsx 
'use client'

import { useState } from 'react'

const API_BASE_URL = 'http://localhost:8000/api'

type TodoFormProps = {
  onAdd: () => void  // タスク追加後に呼び出すコールバック
}

export default function TodoForm({ onAdd }: TodoFormProps) {
  const [title, setTitle] = useState('')  // 入力されたタスクタイトル
  const [submitting, setSubmitting] = useState(false)  // 送信中かどうか

  // フォーム送信時の処理
  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault()  // ページリロードを防ぐ

    // 入力チェック
    if (!title.trim()) {
      alert('タスクを入力してください')
      return
    }

    setSubmitting(true)
    try {
      // POSTリクエストでタスクを追加
      const response = await fetch(`${API_BASE_URL}/todos`, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({ title: title.trim() }),
      })

      if (!response.ok) {
        throw new Error('タスクの追加に失敗しました')
      }

      setTitle('')  // フォームをクリア
      onAdd()  // 一覧を再取得
    } catch (error) {
      console.error('エラー:', error)
      alert('タスクの追加に失敗しました')
    } finally {
      setSubmitting(false)
    }
  }

  return (
    <form onSubmit={handleSubmit} className="mb-6">
      <div className="flex gap-2">
        <input
          type="text"
          value={title}
          onChange={(e) => setTitle(e.target.value)}
          placeholder="新しいタスクを入力..."
          className="flex-1 px-4 py-2 border border-slate-300 rounded-lg
                     focus:outline-none focus:ring-2 focus:ring-emerald-500
                     bg-white text-slate-800"
          disabled={submitting}
        />
        <button
          type="submit"
          disabled={submitting}
          className="px-6 py-2 bg-emerald-500 text-white rounded-lg
                     hover:bg-emerald-600 disabled:bg-slate-300
                     transition-colors font-medium"
        >
          {submitting ? '追加中...' : '追加'}
        </button>
      </div>
    </form>
  )
}

コードのポイント

  • useState: 入力値（title）と送信中フラグ（submitting）の状態管理
  • handleSubmit: フォーム送信時にPOSTリクエストを送信
  • 入力チェック: 空文字の場合はアラートを表示して送信しない
  • onAdd(): 追加成功後にページ側で一覧を再取得する
  • disabled: 送信中はボタンと入力欄を無効化（二重送信防止）

1

page.tsxを開く

2

TodoFormを組み込む

以下のコードに更新します。TodoFormコンポーネントを追加し、onAddプロパティにfetchTodos関数を渡します。

frontend/app/page.tsx 
'use client'

import { useState, useEffect } from 'react'
import TodoList from '@/components/TodoList'
import TodoForm from '@/components/TodoForm'

const API_BASE_URL = 'http://localhost:8000/api'

type Todo = {
  id: number
  title: string
  completed: boolean
  created_at: string
  updated_at: string
}

export default function Home() {
  const [todos, setTodos] = useState<Todo[]>([])
  const [loading, setLoading] = useState(true)

  // タスク一覧を取得する関数
  const fetchTodos = async () => {
    try {
      const response = await fetch(`${API_BASE_URL}/todos`)
      if (!response.ok) {
        throw new Error('タスク一覧の取得に失敗しました')
      }
      const data = await response.json()
      setTodos(data)
    } catch (error) {
      console.error('エラー:', error)
      alert('タスク一覧の取得に失敗しました')
    } finally {
      setLoading(false)
    }
  }

  // 初回レンダリング時にタスク一覧を取得
  useEffect(() => {
    fetchTodos()
  }, [])

  return (
    <div className="min-h-screen bg-gradient-to-br from-blue-50 to-indigo-100 py-8 px-4">
      <div className="max-w-2xl mx-auto">
        <h1 className="text-3xl font-bold text-center text-slate-800 mb-8">
          Todoリスト
        </h1>

        {/* タスク追加フォーム */}
        <TodoForm onAdd={fetchTodos} />

        {/* ローディング表示 */}
        {loading ? (
          <div className="text-center py-8">
            <div className="inline-block animate-spin rounded-full h-8 w-8 border-4 border-emerald-500 border-t-transparent"></div>
            <p className="text-slate-600 mt-2">読み込み中...</p>
          </div>
        ) : (
          <TodoList todos={todos} />
        )}
      </div>
    </div>
  )
}

コードのポイント

  • fetchTodos: タスク一覧を取得する関数（TodoFormのonAddに渡す）
  • <TodoForm onAdd={fetchTodos} />: タスク追加後にfetchTodosが呼ばれる
  • loading: 初回読み込み中はローディング表示

8.4 動作確認

タスク追加機能の動作確認

フロントエンドとバックエンドが起動していることを確認してから、ブラウザで動作を確認します。

1. 開発サーバーが起動していることを確認

  • バックエンド: http://localhost:8000
  • フロントエンド: http://localhost:3000

2. ブラウザで確認

http://localhost:3000 を開いて以下を確認します。

確認項目

  • ✓ 入力欄に「牛乳を買う」と入力して「追加」ボタンをクリック
  • ✓ タスクが一覧に追加される
  • ✓ 入力欄がクリアされる
  • ✓ 空文字で追加しようとするとアラートが表示される
  • ✓ 追加中は「追加中...」と表示され、ボタンが無効化される
  • ✓ ページをリロードしても追加したタスクが表示される

トラブルシューティング

「追加」ボタンをクリックしても反応しない

原因: バックエンドが起動していない、またはCORS設定が正しくありません。

解決方法:

  1. FastAPIサーバーが起動しているか確認（http://localhost:8000/docs）
  2. ブラウザのコンソール（F12キー）でエラーメッセージを確認
  3. main.pyにCORS設定が追加されているか確認
タスクを追加しても一覧に表示されない

原因: onAdd()が呼ばれていない、またはfetchTodosに問題があります。

解決方法:

  1. TodoFormコンポーネントでonAdd()が呼ばれているか確認
  2. page.tsxのfetchTodos関数が正しく実装されているか確認
  3. ブラウザをリロードして一覧が更新されるか確認（手動で確認）
「タスクの追加に失敗しました」と表示される

原因: バックエンドのPOSTエンドポイントに問題があります。

解決方法:

  1. Swagger UI（http://localhost:8000/docs）でPOST /api/todosが動作するか確認
  2. リクエストボディが正しいJSON形式か確認
  3. FastAPIのターミナルでエラーログを確認
