タスクの完了切替・削除機能

タスクの完了/未完了をチェックボックスで切り替えたり、タスクを削除できるようにします。

このステップで実装すること

  • タスクの完了/未完了をチェックボックスで切り替え
  • タスクを削除できるようにする
  • PUT/DELETEリクエストの使い方を学ぶ

9.1 更新・削除の流れ

タスクの更新（PUT）と削除（DELETE）のHTTPリクエストの流れを理解しましょう。 

sequenceDiagram
    participant User as ユーザー
    participant Next as Next.js
フロントエンド
    participant FastAPI as FastAPI
バックエンド
    participant DB as PostgreSQL
データベース

    User->>Next: チェックボックスをクリック
    Next->>FastAPI: PUT /api/todos/{id}
完了状態を更新
    FastAPI->>DB: UPDATE todos SET completed=...
    DB-->>FastAPI: 更新完了
    FastAPI-->>Next: 更新されたタスクを返す
    Next-->>User: 画面を更新

    User->>Next: 削除ボタンをクリック
    Next->>Next: 確認ダイアログを表示
    User->>Next: 削除を確定
    Next->>FastAPI: DELETE /api/todos/{id}
    FastAPI->>DB: DELETE FROM todos WHERE id=...
    DB-->>FastAPI: 削除完了
    FastAPI-->>Next: 成功レスポンス
    Next-->>User: 一覧から削除

ポイント

PUTリクエストは「更新」、DELETEリクエストは「削除」に使います。どちらもタスクのID（/api/todos/{id}）を指定して、特定のタスクを操作します。

9.2 TodoItemコンポーネントの作成

個別のタスク表示を担当するTodoItemコンポーネントを作成し、完了切り替えと削除機能を実装します。

実装 9-1: TodoItemコンポーネントの作成

1

TodoItem.tsxを開く

VS Codeで frontend/components/TodoItem.tsx を開きます（Step 6で作成済み）。

2

完了切替・削除機能を実装

以下のコードでTodoItem.tsxを書き換えます。

frontend/components/TodoItem.tsx 
'use client'

const API_BASE_URL = 'http://localhost:8000/api'

type Todo = {
  id: number
  title: string
  completed: boolean
  created_at: string
  updated_at: string
}

type TodoItemProps = {
  todo: Todo
  onUpdate: () => void
}

export default function TodoItem({ todo, onUpdate }: TodoItemProps) {
  const handleToggle = async () => {
    try {
      const response = await fetch(`${API_BASE_URL}/todos/${todo.id}`, {
        method: 'PUT',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          title: todo.title,
          completed: !todo.completed,
        }),
      })

      if (!response.ok) {
        throw new Error('タスクの更新に失敗しました')
      }

      onUpdate()
    } catch (error) {
      console.error('エラー:', error)
      alert('タスクの更新に失敗しました')
    }
  }

  const handleDelete = async () => {
    if (!confirm('このタスクを削除しますか？')) {
      return
    }

    try {
      const response = await fetch(`${API_BASE_URL}/todos/${todo.id}`, {
        method: 'DELETE',
      })

      if (!response.ok) {
        throw new Error('タスクの削除に失敗しました')
      }

      onUpdate()
    } catch (error) {
      console.error('エラー:', error)
      alert('タスクの削除に失敗しました')
    }
  }

  return (
    <div className="flex items-center gap-3 p-4 bg-white rounded-lg shadow-sm
                    border border-slate-200 hover:shadow-md transition-shadow">
      <input
        type="checkbox"
        checked={todo.completed}
        onChange={handleToggle}
        className="w-5 h-5 rounded border-slate-300 text-emerald-500
                   focus:ring-emerald-500 cursor-pointer"
      />
      <span
        className={`flex-1 ${
          todo.completed
            ? 'line-through text-slate-400'
            : 'text-slate-800'
        }`}
      >
        {todo.title}
      </span>
      <button
        onClick={handleDelete}
        className="text-red-500 hover:text-red-700 transition-colors"
        title="削除"
      >
        <i className="fas fa-trash"></i>
      </button>
    </div>
  )
}

コードの説明

handleToggle関数: チェックボックスがクリックされたとき、PUTリクエストで完了状態を反転させます。

handleDelete関数: 削除ボタンがクリックされたとき、確認ダイアログを表示してからDELETEリクエストを送ります。

onUpdate(): APIリクエスト成功後、親コンポーネントに通知して一覧を再取得させます。

9.3 TodoListコンポーネントの更新

TodoListコンポーネントを更新して、TodoItemを使用するようにします。

実装 9-2: TodoListコンポーネントの更新

1

TodoList.tsxを開く

VS Codeで frontend/components/TodoList.tsx を開きます。

2

TodoItemを使用するよう更新

以下のコードでTodoList.tsxを書き換えます。

frontend/components/TodoList.tsx 
import TodoItem from './TodoItem'

type Todo = {
  id: number
  title: string
  completed: boolean
  created_at: string
  updated_at: string
}

type TodoListProps = {
  todos: Todo[]
  onUpdate: () => void
}

export default function TodoList({ todos, onUpdate }: TodoListProps) {
  if (todos.length === 0) {
    return (
      <div className="text-center py-8 text-slate-500">
        タスクがありません
      </div>
    )
  }

  return (
    <div className="space-y-3">
      {todos.map((todo) => (
        <TodoItem key={todo.id} todo={todo} onUpdate={onUpdate} />
      ))}
    </div>
  )
}

9.4 page.tsxの更新（確認用）

念のため、page.tsxが正しく設定されているか確認します（Step 8で既に実装済みのはずです）。

参考情報

page.tsxは、fetchTodos関数をTodoListのonUpdateプロパティに渡しています。これにより、タスクの更新・削除後に一覧が自動的に再取得されます。

page.tsx の全体コード（クリックで展開）
frontend/app/page.tsx 
'use client'

import { useState, useEffect } from 'react'
import TodoList from '@/components/TodoList'
import TodoForm from '@/components/TodoForm'

const API_BASE_URL = 'http://localhost:8000/api'

type Todo = {
  id: number
  title: string
  completed: boolean
  created_at: string
  updated_at: string
}

export default function Home() {
  const [todos, setTodos] = useState<Todo[]>([])
  const [loading, setLoading] = useState(true)

  const fetchTodos = async () => {
    try {
      const response = await fetch(`${API_BASE_URL}/todos`)
      if (!response.ok) {
        throw new Error('タスク一覧の取得に失敗しました')
      }
      const data = await response.json()
      setTodos(data)
    } catch (error) {
      console.error('エラー:', error)
      alert('タスク一覧の取得に失敗しました')
    } finally {
      setLoading(false)
    }
  }

  useEffect(() => {
    fetchTodos()
  }, [])

  return (
    <div className="min-h-screen bg-gradient-to-br from-blue-50 to-indigo-100 py-8 px-4">
      <div className="max-w-2xl mx-auto">
        <h1 className="text-3xl font-bold text-center text-slate-800 mb-8">
          Todoリスト
        </h1>

        <TodoForm onAdd={fetchTodos} />

        {loading ? (
          <div className="text-center py-8">
            <div className="inline-block animate-spin rounded-full h-8 w-8 border-4 border-emerald-500 border-t-transparent"></div>
            <p className="text-slate-600 mt-2">読み込み中...</p>
          </div>
        ) : (
          <TodoList todos={todos} onUpdate={fetchTodos} />
        )}
      </div>
    </div>
  )
}

9.5 動作確認

完了切替・削除機能の動作確認

以下の手順で、実装した機能が正しく動作するか確認します。

1. サーバーを起動

PostgreSQL、FastAPI、Next.jsの3つが起動していることを確認します。

  • ✓ PostgreSQL: docker-compose up -d
  • ✓ FastAPI: uvicorn main:app --reload
  • ✓ Next.js: npm run dev

2. ブラウザで確認

ブラウザで http://localhost:3000 を開きます。

確認項目

  • ✓ タスクのチェックボックスをクリックすると、完了/未完了が切り替わる
  • ✓ 完了したタスクは取り消し線が付き、グレー表示になる
  • ✓ 削除ボタン（ゴミ箱アイコン）をクリックすると確認ダイアログが表示される
  • ✓ 削除を確定すると、タスクが一覧から消える
  • ✓ ページをリロードしても、変更が保存されている

トラブルシューティング

チェックボックスをクリックしても反応しない

原因: PUTリクエストが正しく送信されていないか、APIエンドポイントに問題があります。

解決方法:

  1. ブラウザのコンソール（F12キー）でエラーメッセージを確認
  2. FastAPIのSwagger UI（http://localhost:8000/docs）でPUT /api/todos/{id} が動作するか確認
  3. TodoItemコンポーネントのhandleToggle関数にconsole.logを追加して、実行されているか確認
削除ボタンをクリックしても削除されない

原因: DELETEリクエストが正しく送信されていないか、APIエンドポイントに問題があります。

解決方法:

  1. ブラウザのコンソール（F12キー）でエラーメッセージを確認
  2. FastAPIのSwagger UI（http://localhost:8000/docs）でDELETE /api/todos/{id} が動作するか確認
  3. 確認ダイアログで「キャンセル」を押していないか確認
更新・削除後に一覧が更新されない

原因: onUpdate()が正しく呼び出されていないか、fetchTodos関数に問題があります。

解決方法:

  1. TodoItemのhandleToggleとhandleDeleteで、成功後にonUpdate()を呼び出しているか確認
  2. page.tsxでfetchTodosをTodoListのonUpdateプロパティに渡しているか確認
  3. ページを手動でリロード（F5キー）して、データベースに変更が反映されているか確認
「405 Method Not Allowed」エラーが出る

原因: FastAPIのエンドポイントがPUT/DELETEメソッドに対応していません。

解決方法:

  1. backend/main.pyで@app.put("/api/todos/{todo_id}")と@app.delete("/api/todos/{todo_id}")が定義されているか確認（Step 5で実装済みのはず）
  2. FastAPIサーバーを再起動
