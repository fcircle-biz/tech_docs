8.1 ユーザー編集機能の実装

既存のユーザー情報を更新する機能を実装します。

実習 8-1: 編集コントローラーの作成

ユーザー編集用のコントローラーメソッドを作成します。

UserEditController.java
@Controller
@RequestMapping("/users")
public class UserEditController {
    
    private final UserService userService;
    
    @Autowired
    public UserEditController(UserService userService) {
        this.userService = userService;
    }
    
    // 編集フォーム表示
    @GetMapping("/{id}/edit")
    public String showEditForm(@PathVariable Long id, Model model) {
        Optional userOptional = userService.findUserById(id);
        
        if (userOptional.isEmpty()) {
            model.addAttribute("errorMessage", "ユーザーが見つかりません");
            return "users/error";
        }
        
        User user = userOptional.get();
        UserForm userForm = new UserForm();
        userForm.setName(user.getName());
        userForm.setEmail(user.getEmail());
        
        model.addAttribute("userForm", userForm);
        model.addAttribute("user", user);
        return "users/edit";
    }
    
    // 更新処理
    @PostMapping("/{id}")
    public String updateUser(@PathVariable Long id,
                           @Valid @ModelAttribute UserForm userForm,
                           BindingResult bindingResult,
                           Model model,
                           RedirectAttributes redirectAttributes) {
        
        if (bindingResult.hasErrors()) {
            Optional userOptional = userService.findUserById(id);
            model.addAttribute("user", userOptional.orElse(null));
            return "users/edit";
        }
        
        try {
            User updatedUser = userService.updateUser(id, userForm);
            redirectAttributes.addFlashAttribute("successMessage", 
                "ユーザー情報を更新しました");
            return "redirect:/users/" + updatedUser.getId();
            
        } catch (Exception e) {
            model.addAttribute("errorMessage", "更新中にエラーが発生しました: " + e.getMessage());
            return "users/edit";
        }
    }
    
    // 削除処理
    @PostMapping("/{id}/delete")
    public String deleteUser(@PathVariable Long id, 
                           RedirectAttributes redirectAttributes) {
        try {
            userService.deleteUser(id);
            redirectAttributes.addFlashAttribute("successMessage", "ユーザーを削除しました");
        } catch (Exception e) {
            redirectAttributes.addFlashAttribute("errorMessage", "削除中にエラーが発生しました");
        }
        
        return "redirect:/users";
    }
}

8.2 サービス層の更新メソッド

ユーザー情報の更新処理をサービス層に実装します。

実習 8-2: 更新メソッドの実装

UserServiceに更新メソッドを追加します。

UserService.java（更新メソッド追加）
// ユーザー情報を更新
public User updateUser(Long id, UserForm userForm) {
    Optional userOptional = userRepository.findById(id);
    
    if (userOptional.isEmpty()) {
        throw new RuntimeException("ユーザーが見つかりません: ID " + id);
    }
    
    User user = userOptional.get();
    
    // メールアドレスの重複チェック（自分以外）
    if (!user.getEmail().equals(userForm.getEmail()) && 
        userRepository.existsByEmail(userForm.getEmail())) {
        throw new RuntimeException("このメールアドレスは既に使用されています");
    }
    
    // 情報を更新
    user.setName(userForm.getName());
    user.setEmail(userForm.getEmail());
    
    // パスワードが入力されている場合のみ更新
    if (userForm.getPassword() != null && !userForm.getPassword().isEmpty()) {
        user.setPassword(userForm.getPassword());
    }
    
    return userRepository.save(user);
}

8.4 削除確認機能

ユーザーを誤って削除しないよう、確認ダイアログを実装します。

実習 8-4: 削除確認ダイアログ

JavaScript を使用した削除確認機能を実装します。

削除確認JavaScript
function confirmDelete(userId, userName) {
    if (confirm(`本当に「${userName}」さんを削除しますか？\nこの操作は取り消せません。`)) {
        // フォームを作成して削除リクエストを送信
        const form = document.createElement('form');
        form.method = 'POST';
        form.action = `/users/${userId}/delete`;
        
        document.body.appendChild(form);
        form.submit();
    }
}

// 削除ボタンにイベントリスナーを追加
document.addEventListener('DOMContentLoaded', function() {
    const deleteButtons = document.querySelectorAll('.btn-delete');
    
    deleteButtons.forEach(button => {
        button.addEventListener('click', function(e) {
            e.preventDefault();
            const userId = this.dataset.userId;
            const userName = this.dataset.userName;
            confirmDelete(userId, userName);
        });
    });
});

8.5 トランザクション管理

データの整合性を保つためのトランザクション管理を実装します。

実習 8-5: トランザクション設定

@Transactionalアノテーションを適切に使用します。

トランザクション管理の実装
@Service
@Transactional
public class UserService {
    
    // 読み取り専用トランザクション
    @Transactional(readOnly = true)
    public Optional findUserById(Long id) {
        return userRepository.findById(id);
    }
    
    // 更新トランザクション
    @Transactional
    public User updateUser(Long id, UserForm userForm) {
        // 更新処理（前述の実装）
        return userRepository.save(user);
    }
    
    // 削除トランザクション
    @Transactional
    public void deleteUser(Long id) {
        if (!userRepository.existsById(id)) {
            throw new RuntimeException("ユーザーが見つかりません: ID " + id);
        }
        userRepository.deleteById(id);
    }
}
トラブルシューティング
  • 更新が反映されない場合: @Transactionalアノテーションが正しく設定されているか確認してください。
  • 楽観的ロックエラー: 同時編集を防ぐためバージョン管理の実装を検討してください。
  • 削除確認が動作しない場合: JavaScriptのイベントリスナーが正しく設定されているか確認してください。
  • フォーム送信エラー: HTMLのmethod属性とaction属性が正しく設定されているか確認してください。
理解度確認クイズ
  1. @Transactionalアノテーションの役割は何ですか？
  2. 楽観的ロックと悲観的ロックの違いは何ですか？
  3. ソフトデリートの利点は何ですか？
  4. パスワード更新時に注意すべき点は何ですか？
  5. 削除確認ダイアログが必要な理由は何ですか？
解答例
  1. データベース操作をトランザクション単位で管理し、整合性を保つ
  2. 楽観的ロックは更新時にチェック、悲観的ロックは読み取り時にロック
  3. データの完全削除を避け、復旧可能性を保つ
  4. 空白の場合は更新しない、ハッシュ化、確認入力の実装
  5. 誤操作による重要データの削除を防ぐため
