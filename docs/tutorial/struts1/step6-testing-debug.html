6.1 ブラウザでの動作確認

開発したStruts アプリケーションをブラウザで実際に動作させ、各機能が正常に動作することを確認します。

実習 6-1: ブラウザでの動作確認手順

ブラウザを使用してアプリケーションの各機能をテストし、正常に動作することを確認します。

基本的なブラウザテスト手順
  1. アプリケーションの起動:
    • EclipseでプロジェクトをTomcatサーバーで実行
    • ブラウザで http://localhost:8080/struts-learning/ にアクセス
  2. ユーザー登録機能の確認:
    • 「新規ユーザー登録」リンクをクリック
    • 必要な情報を入力して登録ボタンをクリック
    • 成功メッセージが表示されることを確認
  3. ユーザー一覧機能の確認:
    • 登録したユーザーが一覧に表示されることを確認
    • 詳細ボタンをクリックして詳細画面に遷移することを確認
  4. バリデーション機能の確認:
    • 必須項目を空にして登録を試行
    • 適切なエラーメッセージが表示されることを確認
    • 不正な形式のメールアドレスでテスト
  5. CRUD操作の確認:
    • ユーザー情報の編集機能をテスト
    • ユーザーの削除機能をテスト
    • 操作後の画面遷移を確認
テスト実行時の注意点
  • データベースに接続されていることを確認
  • Tomcatサーバーが正常に起動していることを確認
  • ブラウザのキャッシュを適宜クリア
  • 異なるブラウザでも動作確認を実施
理解度確認クイズ
  1. Strutsアプリケーションをブラウザで動作確認する際の基本的なURLパターンはどのようなものですか？
  2. ユーザー登録機能をテストする際に確認すべき項目を3つ挙げてください。
  3. バリデーション機能をテストする際に試すべき入力パターンを説明してください。
  4. アプリケーションが正常に動作しない場合の確認ポイントを挙げてください。
解答を確認
  1. http://localhost:8080/[プロジェクト名]/[アクション名].do
  2. フォーム入力、送信処理、成功メッセージ表示
  3. 必須項目の空値、不正な形式の入力、文字数制限超過
  4. サーバー起動状態、データベース接続、ログファイル確認
