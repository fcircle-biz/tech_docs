第1章: 環境構築とプロジェクト作成（Windows初心者向け）

Struts 1.x開発環境の構築とプロジェクト作成

👋 初心者の方へ

この章では、WindowsパソコンでEclipse IDEを使ってStruts開発環境を作ります。プログラミングが初めての方でも大丈夫！一つ一つ丁寧に説明していきます。

専門用語も分かりやすく解説するので、安心して進めてください。

🎯 この章で学ぶこと
  • Java開発環境（JDK、Eclipse、Tomcat）をWindowsパソコンにインストール
  • Eclipse IDEの基本的な使い方と設定方法
  • Maven（マーベン）というビルドツールの使い方
  • Struts 1.x プロジェクトをゼロから作成する手順
  • 初めてのStrutsアプリを作って動かしてみる
  • Webアプリ開発の流れを体験する

メモ: 分からない単語が出てきたら、遠慰なく質問してください！

1.1 💻 Windowsでの開発環境準備

Web開発が初めての方でも大丈夫！WindowsパソコンでStrutsアプリを作るためのソフトを一つずつインストールしていきましょう。

📝 インストールするソフト一覧

以下のソフトを順番にインストールします：

🔸 1. JDK 8（Java開発キット）

「JDK」とは？ Javaでプログラムを作るための基本ツールです。StrutsもJavaで作られているので必須です。

  • 推奨: OpenJDK 8 （無料で使えます）
  • ダウンロード先: Eclipse Adoptium

🌐 2. Apache Tomcat 9.0（Webサーバー）

「Tomcat」とは？ 作ったWebアプリを動かすためのサーバーソフトです。

  • ダウンロード先: Apache Tomcat 9.0
  • 選ぶファイル: Windows用の32-bit/64-bit Windows Service Installer

📝 3. Eclipse IDE（開発ツール）

「Eclipse」とは？ Javaのプログラムを書くためのエディターツールです。自動補完やエラーチェック機能が便利！

  • 推奨: Eclipse IDE for Enterprise Java and Web Developers
  • ダウンロード先: Eclipse公式サイト

🔧 4. Maven 3.6+（ビルドツール）

「Maven」とは？ Javaプロジェクトのビルド（編集）やライブラリ管理を自動化してくれるツールです。

  • ダウンロード先: Apache Maven
  • 選ぶファイル: Binary zip archive

💾 5. PostgreSQL 12+（データベース）

「PostgreSQL」とは？ データを保存するためのデータベースソフトです。後の章で使用します。

  • ダウンロード先: PostgreSQL Windows版
  • 注意: インストール時に設定するパスワードを覚えておいてください！
💻 実習 1-1: WindowsでJDKとEclipseのインストール

初心者向けに、Windowsでのインストール手順を詳しく説明します。

🔸 ステップ 1: JDK 8のインストール
  1. Eclipse Adoptiumにアクセス
  2. 「Temurin 8 (LTS)」を選択
  3. 「Operating System: Windows」を選択
  4. 「Architecture: x64」を選択（またはx86で64ビットの場合）
  5. ダウンロードした.msiファイルをダブルクリック
  6. インストーラーの指示に従ってインストール（全てデフォルトでOK）
🔸 ステップ 2: Eclipse IDEのインストール
  1. Eclipse公式サイトにアクセス
  2. 「Eclipse IDE for Enterprise Java and Web Developers」をダウンロード
  3. ダウンロードした.exeファイルを実行
  4. インストール先を選択（例: C:\\eclipse）
  5. インストール完了後、Eclipseを起動
  6. ワークスペースを選択（例: C:\\workspace）
🔸 ステップ 3: TomcatのインストールとEclipse連携
  1. Tomcat 9.0からWindows Service Installerをダウンロード
  2. .exeファイルを実行してインストール
  3. Eclipseで 「Window」 → 「Show View」 → 「Servers」 を選択
  4. Serversビューで右クリック → 「New」 → 「Server」
  5. 「Apache」 → 「Tomcat v9.0 Server」を選択
  6. Tomcatのインストールパスを指定（例: C:\\Program Files\\Apache Software Foundation\\Tomcat 9.0）
📋 動作確認方法

JDKの確認:

  1. Windowsキー + R で「cmd」と入力してEnter
  2. 「java -version」と入力してEnter
  3. 「openjdk version "1.8.0_xxx"」と表示されるOK

Eclipse + Tomcatの確認:

  1. EclipseのServersビューでTomcatサーバーを右クリック
  2. 「Start」をクリック
  3. ブラウザで http://localhost:8080 にアクセス
  4. Tomcatのウェルカムページが表示されるOK！
⚠️ よくあるトラブルと解決方法
  • ポート8080が使われているエラー: 他のアプリが8080を使用中。PCを再起動するか、Tomcatのポートを変更。
  • JAVA_HOMEが設定されていないエラー: システム環境変数を確認。通常はJDKインストーラーが自動設定。
  • Eclipseが起動しない: JDKが正しくインストールされているか確認。

1.2 📝 EclipseでMavenプロジェクト作成

コマンドラインを使わずに、Eclipseのグラフィカルな画面で簡単にStrutsプロジェクトを作成しましょう！

💡 Mavenって何？

Mavenは、Javaプロジェクトを管理してくれる便利なツールです。以下のことを自動でやってくれます：

  • フォルダ構造の自動作成
  • 必要なライブラリの自動ダウンロード
  • プロジェクトのビルド（コンパイル）
💻 実習 1-2: EclipseでMavenプロジェクト作成

Eclipseのメニューから簡単にMavenプロジェクトを作成します。コマンドラインは使いません！

📋 ステップバイステップ手順
  1. Eclipseを起動
  2. メニューから 「File」 → 「New」 → 「Other...」 をクリック
  3. 「Maven」 フォルダを展開し、「Maven Project」を選択
  4. 「Next」をクリック
  5. 「Create a simple project」のチェックを外して「Next」
  6. Archetype選択画面で以下を入力：
    • Filter: 「webapp」と入力
    • 「maven-archetype-webapp」を選択
  7. 「Next」をクリック
  8. プロジェクト情報入力画面で以下を入力：
    • Group Id: com.example.struts
    • Artifact Id: struts-tutorial
    • Version: 1.0-SNAPSHOT (デフォルトのまま)
    • Package: com.example.struts
  9. 「Finish」をクリック
📝 用語解説
  • Group Id: プロジェクトの組織名（例: 会社名）
  • Artifact Id: プロジェクト名（フォルダ名にもなります）
  • Package: Javaクラスのパッケージ名
📋 作成されるフォルダ構造
struts-tutorial/
├── pom.xml                     ← Maven設定ファイル
└── src/
    └── main/
        ├── java/                 ← Javaソースコード置き場
        ├── resources/            ← 設定ファイル置き場
        └── webapp/               ← Webコンテンツ置き場
            ├── WEB-INF/
            │   └── web.xml       ← Webアプリ設定
            └── index.jsp         ← トップページ
⚠️ もしエラーが出たら...
  • 「archetypeが見つからない」エラー: インターネット接続を確認。Eclipseが初回起動時はarchetypeのダウンロードに時間がかかります。
  • プロジェクトが異なるフォルダに作成される: Eclipseのワークスペース設定を確認。

1.3 📦 Struts 1.xライブラリの設定

作成したプロジェクトにStrutsを使うためのライブラリを追加します。Eclipseで簡単に編集できます！

💡 ライブラリって何？

ライブラリは、他の人が作った便利な機能を自分のプロジェクトで使えるようにしたものです。Strutsのライブラリを追加することで、Strutsの機能が使えるようになります。

📄 実習 1-3: Eclipseでpom.xml編集

Eclipseのエディターでpom.xmlファイルを編集し、Strutsのライブラリを追加します。

📋 手順
  1. EclipseのProject Explorerで「struts-tutorial」プロジェクトを展開
  2. pom.xmlファイルをダブルクリックで開く
  3. 以下の内容で全てを置き換えします：
📄 新しいpom.xmlの内容
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/maven-v4_0_0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example.struts</groupId>
    <artifactId>struts-tutorial</artifactId>
    <packaging>war</packaging>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>8</maven.compiler.source>
        <maven.compiler.target>8</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencies>
        <!-- Struts 1.x -->
        <dependency>
            <groupId>org.apache.struts</groupId>
            <artifactId>struts-core</artifactId>
            <version>1.3.10</version>
        </dependency>
        
        <dependency>
            <groupId>org.apache.struts</groupId>
            <artifactId>struts-taglib</artifactId>
            <version>1.3.10</version>
        </dependency>
        
        <!-- Servlet/JSP API -->
        <dependency>
            <groupId>javax.servlet</groupId>
            <artifactId>javax.servlet-api</artifactId>
            <version>4.0.1</version>
            <scope>provided</scope>
        </dependency>
        
        <dependency>
            <groupId>javax.servlet.jsp</groupId>
            <artifactId>javax.servlet.jsp-api</artifactId>
            <version>2.3.3</version>
            <scope>provided</scope>
        </dependency>

        <!-- JSTL -->
        <dependency>
            <groupId>javax.servlet</groupId>
            <artifactId>jstl</artifactId>
            <version>1.2</version>
        </dependency>
    </dependencies>

    <build>
        <finalName>struts-tutorial</finalName>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <version>3.8.1</version>
            </plugin>
        </plugins>
    </build>
</project>
🔄 Eclipseでライブラリダウンロード
  1. pom.xmlを保存したら、Project Explorerでプロジェクトを右クリック
  2. 「Maven」 → 「Reload Projects」をクリック
  3. 「Maven Dependencies」フォルダが自動で作成され、Strutsのライブラリがダウンロードされます
🔍 ダウンロードされたライブラリの確認

Project Explorerで「Maven Dependencies」フォルダを展開し、以下のライブラリがあることを確認：

  • struts-core-1.3.10.jar
  • struts-taglib-1.3.10.jar
  • javax.servlet-api-4.0.1.jar
  • jstl-1.2.jar
⚠️ もしエラーが出たら...
  • ダウンロードが始まらない: インターネット接続を確認。企業環境ではプロキシ設定が必要な場合あり。
  • バージョンエラー: pom.xmlの内容を再度確認。コピーアンドペーストで間違いが無いかチェック。

1.4 🎉 初めてのStrutsアプリ作成

いよいよ初めてのStrutsアプリケーションを作ってみましょう！「Hello World」と表示するシンプルなアプリです。

💡 Strutsアプリの仕組み

Strutsアプリは以下のファイルが連携して動きます：

  1. web.xml: Webアプリの基本設定
  2. struts-config.xml: Strutsの設定
  3. Actionクラス: ビジネスロジックを処理
  4. JSPファイル: 画面表示
📋 実習 1-4: EclipseでHello Worldアプリ作成

Eclipseを使って、ファイルを一つずつ作成していきます。初心者でも大丈夫！

🔸 ステップ 1: web.xmlの更新

Webアプリの基本設定をします。Eclipseで以下の手順で行います：

  1. src/main/webapp/WEB-INF/web.xmlをダブルクリックで開く
  2. 全ての内容を以下で置き換え：
<!DOCTYPE web-app PUBLIC
  "-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN"
  "http://java.sun.com/dtd/web-app_2_3.dtd">
<web-app>
    <display-name>Struts Tutorial</display-name>
    
    <servlet>
        <servlet-name>action</servlet-name>
        <servlet-class>org.apache.struts.action.ActionServlet</servlet-class>
        <init-param>
            <param-name>config</param-name>
            <param-value>/WEB-INF/struts-config.xml</param-value>
        </init-param>
        <load-on-startup>1</load-on-startup>
    </servlet>
    
    <servlet-mapping>
        <servlet-name>action</servlet-name>
        <url-pattern>*.do</url-pattern>
    </servlet-mapping>
    
    <welcome-file-list>
        <welcome-file>index.jsp</welcome-file>
    </welcome-file-list>
</web-app>
🔸 ステップ 2: struts-config.xmlの作成

Strutsの設定ファイルを作成します。Eclipseで以下の手順で行います：

  1. src/main/webapp/WEB-INFフォルダを右クリック
  2. 「New」 → 「File」を選択
  3. ファイル名に「struts-config.xml」と入力
  4. 「Finish」をクリック
  5. 作成されたファイルに以下の内容を貼り付け：
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE struts-config PUBLIC
  "-//Apache Software Foundation//DTD Struts Configuration 1.3//EN"
  "http://struts.apache.org/dtds/struts-config_1_3.dtd">

<struts-config>
    <action-mappings>
        <action path="/hello"
                type="com.example.struts.action.HelloAction"
                scope="request">
            <forward name="success" path="/hello.jsp"/>
        </action>
    </action-mappings>
</struts-config>
🔸 ステップ 3: HelloAction.javaの作成

ビジネスロジックを処理するJavaクラスを作成します。Eclipseで以下の手順で行います：

  1. src/main/javaフォルダを右クリック
  2. 「New」 → 「Package」を選択
  3. パッケージ名に「com.example.struts.action」と入力
  4. 作成されたパッケージを右クリック → 「New」 → 「Class」
  5. クラス名に「HelloAction」と入力
  6. 「Finish」をクリック
  7. 作成されたファイルの内容を以下で置き換え：
package com.example.struts.action;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import org.apache.struts.action.Action;
import org.apache.struts.action.ActionForm;
import org.apache.struts.action.ActionForward;
import org.apache.struts.action.ActionMapping;

public class HelloAction extends Action {
    
    @Override
    public ActionForward execute(ActionMapping mapping,
                                ActionForm form,
                                HttpServletRequest request,
                                HttpServletResponse response) {
        
        request.setAttribute("message", "Hello, Struts 1.x World!");
        return mapping.findForward("success");
    }
}
🔸 ステップ 4: hello.jspの作成

画面表示用のJSPファイルを作成します。Eclipseで以下の手順で行います：

  1. src/main/webappフォルダを右クリック
  2. 「New」 → 「File」を選択
  3. ファイル名に「hello.jsp」と入力
  4. 「Finish」をクリック
  5. 作成されたファイルに以下の内容を貼り付け：
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ taglib uri="http://struts.apache.org/tags-html" prefix="html" %>
<html>
<head>
    <title>Hello Struts</title>
</head>
<body>
    <h1>${message}</h1>
    <p>現在の時刻: <%= new java.util.Date() %></p>
</body>
</html>
🔸 ステップ 5: Eclipseでビルドと実行

作成したアプリを実際に動かしてみましょう！Eclipseで簡単に実行できます。

  1. プロジェクトを右クリック → 「Run As」 → 「Run on Server」
  2. 「Tomcat v9.0 Server」を選択し、「Next」
  3. 「Add All」ボタンでプロジェクトを追加
  4. 「Finish」をクリック
  5. Eclipseの内蔵ブラウザまたは外部ブラウザで以下にアクセス:
🌐 アクセスURL

http://localhost:8080/struts-tutorial/hello.do

メモ: 「.do」で終わるのがStrutsの特徴です！

🎉 成功した場合の表示

ブラウザに以下が表示されれOK！

  • 「Hello, Struts 1.x World!」
  • 現在の時刻（例: 2024-01-15 10:30:45）
⚠️ うまく表示されない場合
  • 404エラー: URLを再度確認。「.do」を忘れていないかチェック。
  • コンパイルエラー: Javaファイルの内容を再度確認。EclipseのProblemsビューでエラーを確認。
  • Tomcatが起動しない: ServersビューでTomcatの状態を確認。再起動してみる。
⚠️ 注意点
  • JDK 8を使用することを推奨（新しいバージョンでは互換性問題が発生する可能性）
  • Struts 1.xはメンテナンス終了しているため、新規開発での使用は推奨されません
  • Eclipseでのホットデプロイを有効にしておくと開発効率が向上します
理解度確認クイズ
  1. Struts 1.x アプリケーションで必須の設定ファイルは何ですか？
  2. ActionServletのURL パターンマッピングで一般的に使用される拡張子は何ですか？
  3. Actionクラスでビジネスロジックを実装するメソッドの名前は何ですか？
  4. struts-config.xmlでアクションマッピングを定義するタグ名は何ですか？
  5. Mavenでプロジェクトをビルドするコマンドは何ですか？
解答を確認
  1. web.xml と struts-config.xml
  2. .do
  3. execute
  4. <action-mappings> および <action>
  5. mvn clean package
