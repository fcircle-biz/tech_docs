第1章: 開発環境構築とプロジェクト初期化

Struts 1.x開発環境の構築

章の概要

Struts 1.xアプリケーション開発に必要な環境を効率的に構築します。Pleiades Eclipse All-in-Oneを使用して統合開発環境を構築し、struts-blank.warテンプレートを利用したプロジェクト初期化手法を学習します。

本手法は新卒研修やレガシーシステムの保守業務に必要な実践的スキルを提供します。

学習目標
  • Pleiades Eclipse All-in-Oneを使用した統合開発環境の構築
  • struts-blank.warテンプレートを活用したプロジェクト初期化
  • Eclipse IDEによるWARファイルインポート手法
  • Tomcatアプリケーションサーバーでのデプロイメント
  • Strutsアプリケーションの動作確認とデバッグ
  • Webアプリケーション開発の基本ワークフロー

1.1 Pleiades Eclipse開発環境の構築

Pleiades Eclipse All-in-OneはJava開発環境とアプリケーションサーバーが統合されたパッケージで、企業での開発業務に必要なツールチェーンを効率的に構築できます。

Pleiades Eclipse All-in-Oneの構成要素
  • IDE: Eclipse IDE for Enterprise Java Developers (日本語化済み)
  • アプリケーションサーバー: Apache Tomcat 9.x
  • Java実行環境: OpenJDK 17 (LTS版)
  • ビルドツール: 手動ビルド（Eclipseビルド機能使用）
  • インストール方式: アーカイブ展開形式
⚠️ 事前準備
  • ハードディスク容量: 最低5GB以上の空き容量
  • メモリ: 8GB以上推奨（4GBでも動作可能）
  • Windows: Windows 10/11 (64bit)
  • 権限: ファイル展開とフォルダ作成権限
💻 実習 1-1: Pleiades Eclipse All in Oneのダウンロードと設定

たった3ステップで完了！すべてが一つになったPleiadesパッケージを使います。

🔸 ステップ 1: Pleiades Eclipse All in Oneのダウンロード
  1. Pleiades All in One ダウンロードページにアクセス
  2. 「Eclipse 2023-12 Java Full Edition」を選択
  3. Windows x86_64版をダウンロード
  4. ファイルサイズは約2GB（時間に余裕を持ってダウンロード）
ステップ 2: アーカイブの展開と配置
  1. ダウンロードした.zipファイルを右クリック
  2. 「すべて展開...」を選択
  3. 展開先フォルダを指定（例: C:\pleiades）
  4. 注意: パスに日本語や空白が含まれないフォルダを選択
  5. 展開完了まで待機（5-10分程度）
ステップ 3: IDEの起動と初期設定
  1. 展開されたフォルダ内のeclipseフォルダを開く
  2. eclipse.exeをダブルクリックして起動
  3. ワークスペースの場所を聞かれたらデフォルトのまま「起動」をクリック
  4. Eclipseが日本語で起動することを確認
  5. 「ファイル」→「新規」→「その他」でメニューが日本語表示されることを確認
環境確認手順

Tomcatサーバー設定の確認:

  1. Eclipse下部の「サーバー」ビューを確認
  2. 何も表示されていない場合は、右クリック→「新規」→「サーバー」
  3. 「Apache」→「Tomcat v9.0 サーバー」を選択
  4. 「次へ」をクリック後、Tomcatの場所が自動設定されていることを確認
  5. 「完了」をクリック

JDKの確認:

  1. 「ウィンドウ」→「設定」を開く
  2. 「Java」→「インストール済みのJRE」を選択
  3. JDK 17が設定されていることを確認
トラブルシューティング
  • 起動失敗: パスに非アスキー文字が含まれていないか確認。必要に応じて管理者権限で実行
  • メモリ不足: eclipse.iniの-Xmxパラメータを調整 (例: -Xmx1024m)
  • 文字エンコーディング: ワークスペース設定でUTF-8を指定

1.2 📦 struts-blank.warの入手

Strutsプロジェクトを手早く作成するために、struts-blank.war（空のStrutsプロジェクトテンプレート）を使用します。これなら設定で迷うことがありません！

💡 struts-blank.warって何？

struts-blank.warは、Strutsの基本設定がすべて完了した「空のプロジェクト」です。以下が含まれています：

  • 必要なStrutsライブラリ（jarファイル）
  • web.xmlの基本設定
  • struts-config.xmlのひな形
  • 基本的なフォルダ構造

これを使えば面倒な設定作業をスキップして、すぐにStrutsアプリ開発に集中できます！

実習 1-2: プロジェクトテンプレートの入手

Apache Struts公式リポジトリからstruts-blank.warテンプレートを取得します。

テンプレート取得手順
  1. Apache Struts公式サイトにアクセス： https://archive.apache.org/dist/struts/binaries/
  2. struts-1.3.10-all.zipをクリックしてダウンロード
  3. ダウンロードしたstruts-1.3.10-all.zipを適当なフォルダに解凍
  4. 解凍されたフォルダ内のwebappsフォルダを開く
  5. struts-blank.warファイルを見つける
  6. このファイルを分かりやすい場所（例: デスクトップ）にコピー
アーカイブ構成の確認

struts-1.3.10-all.zipを解凍すると以下のような構成になっています：

struts-1.3.10/
├── lib/                      ← Strutsライブラリファイル
├── webapps/
│   ├── struts-blank.war     ← これを使います！
│   ├── struts-examples.war
│   └── struts-mailreader.war
└── docs/                    ← ドキュメント
WARファイルの構成確認 (参考)

struts-blank.warの中身を確認したい場合は、.zip形式で開くことができます：

  1. struts-blank.warを右クリック
  2. 「プログラムから開く」→「エクスプローラー」
  3. 中身を確認（削除や変更はしないでください）
取得時の注意事項
  • ファイルサイズ: アーカイブは約15MB
  • セキュリティ: ダウンロード後はウイルススキャンを実行
  • ファイル管理: テンプレートは適切なディレクトリに保存

1.3 📂 プロジェクト作成とインポート

struts-blank.warを使ってEclipseにStrutsプロジェクトを作成します。とても簡単で、設定ファイルも自動で用意されます！

Eclipse WARインポート機能

Eclipse IDEはWARファイルをプロジェクトとして統合する機能を提供しています。struts-blank.warテンプレートをインポートすることで、必要な設定が事前構成された状態で開発を開始できます。

実習 1-3: WARファイルインポートとプロジェクト構成

struts-blank.warテンプレートをEclipseプロジェクトとしてインポートし、開発環境を構築します。

インポート手順
  1. Eclipseのメニューから「ファイル」→「インポート」を選択
  2. 「Web」フォルダを展開し、「WARファイル」を選択
  3. 「次へ」をクリック
  4. WARファイル欄で「参照」をクリック
  5. 先ほど保存したstruts-blank.warを選択
  6. プロジェクト名を「struts-tutorial」に変更
  7. ターゲットランタイムで「Tomcat v9.0 Server」を選択
  8. 「完了」をクリック
プロジェクト構造の確認

インポートが成功すると、以下のような構造でプロジェクトが作成されます：

struts-tutorial/
├── src/
│   └── main/
│       └── webapp/
│           ├── WEB-INF/
│           │   ├── web.xml          ← Webアプリ設定
│           │   ├── struts-config.xml ← Struts設定
│           │   └── lib/             ← Strutsライブラリ
│           ├── index.jsp            ← トップページ
│           └── Welcome.do           ← サンプルアクション
└── WebContent/ (同じ内容)
インポート結果の検証

以下の要素が確認できればインポート成功です：

  • プロジェクトエクスプローラーに「struts-tutorial」プロジェクト
  • WEB-INF/libフォルダ内にStrutsのJARファイル
  • WEB-INF/struts-config.xmlファイル
  • Welcome.jspなどのサンプルファイル
インポート時の問題対応
  • WARインポートオプションが非表示: Eclipse for Enterprise Java Developersの使用を確認
  • ランタイム選択不可: Tomcatサーバーの事前設定が必要
  • プロジェクトエラー: リフレッシュ操作で解決可能

1.4 アプリケーションのデプロイと動作確認

struts-blank.warテンプレートにはサンプルアプリケーションが含まれています。Tomcatサーバーでのデプロイを実行し、Strutsフレームワークの動作を確認します。

テンプレートに含まれるサンプルコンポーネント

テンプレートには以下のStrutsコンポーネントが事前実装されています：

  • Welcome.do: サンプルアクション
  • Welcome.jsp: ウェルカムページ
  • WelcomeAction.java: サンプルアクションクラス
  • struts-config.xml: 基本設定済み

これらのコンポーネントの連携を通じて、Strutsフレームワークの基本的なMVCアーキテクチャを理解します。

実習 1-4: Tomcatサーバーでのデプロイと動作検証

プロジェクトをTomcatアプリケーションサーバーで実行し、Strutsフレームワークの動作を検証します。

🔸 ステップ 1: プロジェクトの確認
  1. プロジェクトエクスプローラーで「struts-tutorial」プロジェクトを展開
  2. WebContentフォルダを確認
  3. WEB-INF/struts-config.xmlをダブルクリックで開いて内容を確認
  4. Welcome.jspファイルがあることを確認
🔸 ステップ 2: Tomcatサーバーでの実行
  1. プロジェクトを右クリック → 「実行」 → 「サーバーで実行」
  2. 「Tomcat v9.0 Server」を選択し、「次へ」
  3. プロジェクトが右側に表示されることを確認
  4. 「完了」をクリック
  5. Tomcatサーバーが起動するまで待機
🔸 ステップ 3: ブラウザでの動作確認
  1. Eclipse内蔵ブラウザが自動で開きます
  2. または外部ブラウザで以下にアクセス:
🌐 アクセスURL
  • トップページ: http://localhost:8080/struts-tutorial/
  • ウェルカムアクション: http://localhost:8080/struts-tutorial/Welcome.do

メモ: 「.do」で終わるのがStrutsアクションの特徴です！

🎉 成功した場合の表示

以下のページが表示されればStruts環境の構築成功です：

  • 「Welcome!」のメッセージ
  • Strutsのロゴと基本的な説明
  • サンプルリンクが動作する
🔸 ステップ 4: 簡単なカスタマイズ

サンプルページを少し変更してみましょう：

  1. Welcome.jspをダブルクリックで開く
  2. 日本語文字化け対策: ファイルの先頭行に以下のディレクティブを追加
    <%@ page contentType="text/html; charset=UTF-8" %>
  3. 日本語言語設定: HTMLタグに日本語属性を追加
    <html> → <html lang="ja">
  4. タイトル部分を見つけて「私の最初のStrutsアプリ」に変更
  5. Ctrl+Sで保存
  6. ブラウザでF5を押してリロード
  7. 変更が反映されることを確認
⚠️ トラブルシューティング
  • 404エラー: プロジェクトが正しくデプロイされているかサーバービューで確認
  • 500エラー: Tomcatのログを確認（コンソールビュー）
  • アクセスできない: ポート8080が他のアプリで使用されていないか確認
  • Tomcat起動エラー: サーバーを削除して再作成してみる
開発環境に関する重要な注意事項
  • Javaバージョン互換性: JDK 17とStruts 1.xの組み合わせでの動作を確認済み
  • フレームワークのEOL状態: Struts 1.xはメンテナンス終了しており、本番運用では非推奨
  • ホットデプロイ機能: ファイル保存時の自動リロードで開発効率が向上
  • パス設定: 非アスキー文字やスペースを含むパスは問題の原因となる可能性
  • テンプレートの意義: struts-blank.warは学習・研修目的の参考実装
理解度検証
  1. Pleiades Eclipse All-in-Oneパッケージに含まれる主要コンポーネントを3つ挙げてください。
  2. struts-blank.warテンプレートを使用する技術的メリットを説明してください。
  3. Struts 1.xアプリケーションで使用される標準的なURLパターンを答えてください。
  4. Eclipse IDEでWARファイルインポートを実行する手順を簡潔に説明してください。
  5. Strutsアプリケーションの動作確認でアクセスすべきURLと、その意味を説明してください。
解答例
  1. Eclipse IDE、Apache Tomcat、OpenJDK (JRE)
  2. 必要なライブラリ、設定ファイル、フォルダ構造が事前構成され、開発効率が向上
  3. *.doパターン（ActionServletへのマッピング）
  4. ファイルメニュー → インポート → Web → WARファイルを選択
  5. http://localhost:8080/struts-tutorial/Welcome.do - Welcomeアクションの実行でMVCフローを確認
