ステップ5: 動作確認

AIチャットアプリケーションの動作確認とテスト

このステップで実施すること
  • アプリケーションの起動 - Gradle bootRunでアプリケーションを起動
  • チャット機能のテスト - AIとの対話を実際に試す
  • バリデーション確認 - 入力エラーの挙動を確認
  • 処理中メッセージ確認 - ローディング表示のテスト
  • 基本的なトラブルシューティング - よくある問題と対処法

所要時間： 約1時間

前提条件
  • ステップ1〜4が完了していること
  • Ollamaコンテナが起動していること
  • Qwen3:1.7bモデルがダウンロード済みであること

5.1 アプリケーションの起動

Spring Bootアプリケーションを起動して、チャット画面にアクセスします。

実施 5-1: アプリケーション起動
手順
  1. Ollamaコンテナの確認: 
    # Ollamaコンテナの起動確認
docker ps | grep ollama

# 起動していない場合は起動
docker run -d -p 11434:11434 --name ollama ollama/ollama

# Qwen3モデルのダウンロード（初回のみ）
docker exec ollama ollama pull qwen3:1.7b
  2. Spring Bootアプリケーションの起動: 
    # Windows (コマンドプロンプト)
cd ai-chat
gradlew bootRun

# Windows (PowerShell)
cd ai-chat
.\gradlew bootRun

# macOS / Linux
cd ai-chat
./gradlew bootRun
  3. 起動ログの確認:

    以下のようなログが表示されれば起動成功です：

    Started AiChatApplication in X.XXX seconds (process running for X.XXX)
  4. ブラウザでアクセス:

    ブラウザで以下のURLにアクセスします：

    http://localhost:8080/chat
期待される結果
  • チャット画面が正しく表示される
  • 「AI Chat」というヘッダーが表示される
  • 「Powered by Qwen3:1.7b」のバッジが表示される
  • テキストエリアと送信ボタンが表示される

5.2 チャット機能のテスト

実際にAIとの対話をテストして、正常に動作することを確認します。

実施 5-2: 基本的なチャット機能のテスト
テストシナリオ
  1. 簡単な質問をする:

    テキストエリアに以下を入力して送信：

    こんにちは！自己紹介してください。

    期待される動作：

    • 送信ボタンを押すと「AIが回答を生成しています...」メッセージが表示される
    • 送信ボタンが「処理中...」に変わり、無効化される
    • 数秒〜数分後、AI応答が表示される
    • ユーザーの質問が青い枠で表示される
    • AIの回答がオレンジの枠で表示される
  2. 技術的な質問をする: 
    Javaの主な特徴を3つ教えてください。

    期待される動作：

    • AIが日本語で技術的な回答を返す
    • 箇条書きや段落が正しく表示される（改行が保持される）
  3. 長文の質問をする: 
    Spring Bootフレームワークの利点と、従来のSpring Frameworkとの違いについて詳しく説明してください。

    期待される動作：

    • 長い回答が正しく表示される
    • スクロールバーが表示される（500px以上の場合）

5.3 バリデーションエラーの確認

入力バリデーションが正しく動作することを確認します。

実施 5-3: バリデーションテスト
テストケース
  1. 空白で送信:

    テキストエリアに何も入力せず送信ボタンをクリック

    期待される動作：

    • 赤い枠がテキストエリアに表示される
    • 「メッセージを入力してください」というエラーメッセージが表示される
    • 処理中メッセージは表示されない
    • AIへのリクエストは送信されない
  2. 空白文字のみで送信:

    スペースやタブのみを入力して送信

    期待される動作：

    • 同様にバリデーションエラーが表示される
    • @NotBlankは空白のみの入力も禁止するため

5.4 基本的なトラブルシューティング

よくある問題とその対処法を確認します。

よくある問題と対処法
問題1: Ollamaに接続できない

エラーメッセージ：

エラーが発生しました: Connection refused
Ollamaが起動しているか確認してください。

対処法：

  1. Ollamaコンテナが起動しているか確認： 
    docker ps | grep ollama
  2. 起動していない場合は起動： 
    docker start ollama
  3. ポート11434が使用可能か確認： 
    curl http://localhost:11434/api/tags
問題2: モデルが見つからない

エラーメッセージ：

model 'qwen3:1.7b' not found

対処法：

  1. モデルをダウンロード： 
    docker exec ollama ollama pull qwen3:1.7b
  2. ダウンロード済みモデルを確認： 
    docker exec ollama ollama list
問題3: AIの応答が遅い

症状： 応答に数分以上かかる

対処法：

  • 初回リクエストはモデルのロードに時間がかかります（正常動作）
  • 2回目以降は高速になります
  • CPUのみで動作している場合、数分かかることがあります
  • qwen3:1.7bは軽量モデルですが、さらに軽量なモデルが必要な場合は他のモデルの使用を検討してください
問題4: ポート8080が既に使用中

エラーメッセージ：

Web server failed to start. Port 8080 was already in use.

対処法：

  1. application.ymlでポート番号を変更： 
    server:
  port: 8081
  2. または、既存のプロセスを停止する
問題5: 日本語の回答品質が低い

対処法：

  • システムプロンプトで「必ず日本語で回答してください」と明示
  • ChatController.javaのシステムプロンプトを調整
  • より大きなモデル（qwen3:7b）の使用を検討（メモリ要件: 8GB以上）
