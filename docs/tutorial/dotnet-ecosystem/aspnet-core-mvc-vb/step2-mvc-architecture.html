2.1 MVCパターンとは

MVC（Model-View-Controller）は、ユーザーインターフェースを持つアプリケーションを3つの相互接続された要素に分割するアーキテクチャパターンです。

Model（モデル）
  • データの構造を定義
  • ビジネスロジック
  • データベースとの連携
  • データの検証
View（ビュー）
  • ユーザーインターフェース
  • データの表示
  • HTML生成
  • Razor構文
Controller（コントローラー）
  • 処理の制御
  • ユーザー入力の処理
  • ModelとViewの連携
  • リクエスト処理
MVCの処理フロー
ユーザー
リクエスト
→
Controller
処理制御
↓
Model
データ処理
→
View
HTML生成

2.2 コントローラーの作成

実際にコントローラーを作成して、MVCの動作を理解しましょう。

実習 2-1: 基本的なコントローラーの作成

新しいコントローラーを作成し、基本的なアクションメソッドを実装します。

手順
  1. 1ソリューションエクスプローラーで「Controllers」フォルダを右クリック
  2. 2「追加」→「コントローラー」を選択
  3. 3「空のMVCコントローラー」を選択し、「追加」をクリック
  4. 4コントローラー名を「DemoController」と入力し、「追加」をクリック
  5. 5以下のコードを追加:
DemoController.vb
Imports Microsoft.AspNetCore.Mvc

Public Class DemoController
    Inherits Controller

    ' インデックスアクション
    Public Function Index() As IActionResult
        ViewData("Message") = "こんにちは、ASP.NET Core MVC!"
        ViewData("CurrentDate") = DateTime.Now.ToString("yyyy年MM月dd日 HH:mm:ss")
        Return View()
    End Function

    ' パラメータ付きアクション
    Public Function Greeting(name As String) As IActionResult
        If String.IsNullOrEmpty(name) Then
            name = "ゲスト"
        End If
        
        ViewBag.Name = name
        ViewBag.Time = DateTime.Now.ToString("HH:mm")
        
        Return View()
    End Function

    ' JSON形式でデータを返すアクション
    Public Function GetData() As IActionResult
        Dim data = New With {
            .Message = "JSON データの例",
            .Timestamp = DateTime.Now,
            .Status = "Success"
        }
        
        Return Json(data)
    End Function

    ' リダイレクトの例
    Public Function GoToHome() As IActionResult
        Return RedirectToAction("Index", "Home")
    End Function
End Class

2.4 ルーティングシステム

ASP.NET Core MVCのルーティングシステムは、URLをコントローラーのアクションにマッピングする仕組みです。

デフォルトルーティング規則

ASP.NET Core MVCは以下のデフォルトルーティング規則を使用します：

/[Controller]/[Action]/[id?]

例:
/Demo/Index           → DemoController.Index()
/Demo/Greeting/太郎   → DemoController.Greeting("太郎")
/Home/Privacy         → HomeController.Privacy()

2.5 Web Forms との比較

Web Forms開発者がMVCに移行する際の主な違いを理解しましょう。

Web Forms vs ASP.NET Core MVC 比較表
概念 Web Forms ASP.NET Core MVC 移行のポイント
ページ構造 .aspx + コードビハインド Controller + View イベント駆動からアクション駆動へ
状態管理 ViewState、ViewData ViewData、ViewBag、TempData ステートレス設計が基本
ポストバック 自動的にサーバーへ送信 明示的なHTTP動詞（GET/POST） RESTful設計の採用
URL構造 物理ファイルパス ルーティング規則 SEOフレンドリーなURL
テスト テストが困難 単体テストが容易 依存性注入の活用

2.6 Razor構文の基礎

VB.NETでのRazor構文の基本的な使い方を学習します。

VB.NET Razor構文の基本
@' コメント
@* 複数行コメント *@

@' 変数の出力
@ViewBag.Name
@ViewData("Message")

@' 条件分岐
@If User.Identity.IsAuthenticated Then
    @<p>ログイン済み: @User.Identity.Name</p>
Else
    @<p>未ログイン</p>
End If

@' ループ処理
@For Each item In ViewBag.Items
    @<li>@item</li>
Next

@' コードブロック
@Code
    Dim message As String = "Hello World"
    Dim currentTime As DateTime = DateTime.Now
End Code

@' HTML補助メソッド
@Html.ActionLink("リンクテキスト", "Action", "Controller")
@Url.Action("Action", "Controller")
