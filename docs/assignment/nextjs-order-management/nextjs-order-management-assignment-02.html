顧客管理機能

顧客情報（名前、メール、電話番号）を管理する機能を実装します。Todoアプリで学んだCRUD操作の復習として最適です。

このステップで実装すること

  • Customerモデルの作成（名前、メール、電話番号、作成日時、更新日時）
  • 顧客CRUD API実装（一覧取得、詳細取得、登録、更新、削除）
  • 顧客一覧画面（検索、削除機能付き）
  • 顧客登録画面（フォーム入力）
  • 顧客編集画面（既存データの編集）

参照する設計資料

このステップでは、以下の設計資料を参照して実装を進めてください：

システム仕様書 - 顧客管理機能、顧客API API設計、バリデーションルールを確認
データベース設計 - customersテーブル テーブル定義とフィールド仕様
実装ガイド - Step 2 顧客管理機能の実装手順

実装のヒント

バックエンド実装のヒント

  • • モデル定義: database.md の customersテーブル定義を参考に SQLModel クラスを作成
  • • API実装: README.md の「顧客API」セクションを参照し、5つのエンドポイントを実装
  • • バリデーション: 名前は必須（1〜100文字）、メールは形式チェック、電話番号は10〜15文字
  • • 検索機能: クエリパラメータ `search` で名前の部分一致検索を実装
  • • 削除処理: 現時点では単純削除（Step 6で注文履歴チェックを追加）

フロントエンド実装のヒント

  • • 一覧画面: useEffect でAPIを呼び出し、useState で顧客リストを管理、テーブルで表示
  • • 検索機能: 検索ボックスの入力値をクエリパラメータとして送信
  • • フォーム: useState で入力値を管理、送信後に一覧画面へリダイレクト
  • • 共通コンポーネント: CustomerForm.tsx を作成し、登録・編集で共通化
  • • 参考: Todoアプリで作った一覧・追加機能を拡張する形で実装

よくあるミス

  • • モデルで created_at/updated_at を自動設定する設定を忘れる
  • • APIでJSONレスポンスの形式が仕様書と異なる
  • • フォームでバリデーションエラーが表示されない
  • • 一覧画面で削除後にデータが再取得されない
