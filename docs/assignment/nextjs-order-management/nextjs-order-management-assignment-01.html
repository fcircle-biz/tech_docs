環境構築

プロジェクトのセットアップとDocker環境の構築を行います。

このステップで実装すること

  • プロジェクトフォルダの作成
  • Docker ComposeでPostgreSQLを起動
  • FastAPIバックエンドの環境構築
  • Next.jsフロントエンドの環境構築

実装ガイド - Step 1 環境構築の詳細な手順

実装のヒント

プロジェクト構造

  • • フォルダ構成: ルートフォルダに backend/ と frontend/ を作成
  • • Docker Compose: ルートに docker-compose.yml を配置
  • • 参考: Todoアプリで作ったプロジェクト構造と同じパターンです

PostgreSQL設定

  • • ポート: 5432（デフォルト）を使用
  • • データベース名: order_management
  • • 初期ユーザー: postgres / postgres（開発環境用）
  • • 確認方法: Docker Desktop で postgres コンテナが起動していることを確認

FastAPIバックエンド

  • • 仮想環境: python -m venv venv で作成
  • • 必要パッケージ: fastapi, uvicorn, sqlmodel, psycopg2-binary
  • • 起動確認: http://localhost:8000/docs でSwagger UIが開けばOK
  • • ポート: 8000（デフォルト）を使用

Next.jsフロントエンド

  • • 作成コマンド: npx create-next-app@latest frontend
  • • オプション: TypeScript=Yes, Tailwind CSS=Yes, App Router=Yes
  • • 起動確認: http://localhost:3000 でNext.jsのウェルカムページが表示されればOK
  • • ポート: 3000（デフォルト）を使用

よくあるミス

  • • Docker Desktopが起動していない状態で docker-compose up すると失敗します
  • • ポート5432が既に使われている場合、PostgreSQLが起動できません
  • • Python仮想環境を activate せずにパッケージをインストールするとグローバルにインストールされます
  • • Node.js のバージョンが古いと create-next-app が失敗することがあります（v18以上推奨）
