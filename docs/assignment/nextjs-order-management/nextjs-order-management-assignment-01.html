環境構築

プロジェクトの基盤を構築し、開発環境を整えます。Docker ComposeでPostgreSQLを起動し、FastAPIバックエンドとNext.jsフロントエンドの開発環境を整えます。

このステップで実装すること

  • Docker Compose設定（PostgreSQL）
  • FastAPIプロジェクトの初期セットアップ（SQLModel設定）
  • Next.jsプロジェクトの初期セットアップ（TypeScript、Tailwind CSS）
  • 共通ファイルの作成（レイアウト、ナビゲーション、環境変数）

実装のヒント

Docker Compose設定のヒント

  • • PostgreSQL: ポート5432、データボリューム設定を忘れずに
  • • 環境変数: POSTGRES_USER、POSTGRES_PASSWORD、POSTGRES_DBを設定
  • • ヘルスチェック: pg_isreadyコマンドでDB起動完了を確認

バックエンド実装のヒント

  • • プロジェクト構成: app/ フォルダにmain.py、database.py、modelsフォルダ、routersフォルダを作成
  • • SQLModel設定: database.pyでDB接続とセッション管理を実装
  • • CORS設定: フロントエンド（localhost:3000）からのアクセスを許可

フロントエンド実装のヒント

  • • Next.js App Router: app/ディレクトリ構成を使用
  • • レイアウト: app/layout.tsxに共通ヘッダー・ナビゲーションを配置
  • • 環境変数: .env.localでバックエンドAPIのURLを設定
  • • Tailwind CSS: モック画面と同じスタイルを目指す

よくあるミス

  • • Docker Composeで環境変数の設定漏れ（特にDATABASE_URL）
  • • フロントエンドとバックエンドのポート番号の間違い
  • • PostgreSQLのデータボリュームを設定せず、コンテナ再起動でデータが消える
  • • CORSの設定漏れでフロントエンドからAPIにアクセスできない
