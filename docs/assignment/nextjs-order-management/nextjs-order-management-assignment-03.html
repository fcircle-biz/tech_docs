商品管理機能

商品情報（名前、価格、カテゴリ、販売状態）を管理する機能を実装します。顧客管理とほぼ同じパターンですが、カテゴリフィルタや販売状態切替などの追加機能があります。

このステップで実装すること

  • Productモデルの作成（名前、価格、カテゴリ、販売状態）
  • 商品CRUD API（一覧、登録、更新、削除）
  • 商品一覧画面（カテゴリフィルタ、販売状態表示）
  • 商品登録/編集画面（価格バリデーション、カテゴリ選択）

データベース設計 - productsテーブル テーブル定義、カラム仕様

実装のヒント

バックエンド実装のヒント

  • • モデル定義: database.md の productsテーブル定義を参考に SQLModel クラスを作成
  • • API実装: README.md の「商品API」セクションを参照
  • • 価格バリデーション: Field(ge=1, le=999999) で範囲チェック
  • • カテゴリフィルタ: クエリパラメータ category で絞り込み（food/drink/other）
  • • 販売状態フィルタ: クエリパラメータ available で販売中のみ表示可能に

フロントエンド実装のヒント

  • • 一覧画面: useEffect でAPIを呼び出し、テーブルで表示
  • • カテゴリフィルタ: ボタンまたはタブで切り替え（全て/食品/飲料/その他）
  • • 価格表示: toLocaleString() で3桁カンマ区切り（例: ¥1,000）
  • • 販売状態切替: トグルボタンでAPIをPUT、即座に反映
  • • フォーム: useState で入力値を管理、送信後にリダイレクト
  • • 参考: 顧客管理機能（Step 2）と同じパターンで実装可能

よくあるミス

  • • 価格を文字列のまま送信（numberに変換が必要）
  • • カテゴリの値を英語（food/drink/other）にせず、日本語で送ってしまう
  • • 販売状態（is_available）をtrueでなく"true"（文字列）で送ってしまう
  • • 価格の範囲チェック漏れ（1〜999999の範囲外を許容してしまう）
