画面をつなげる - ナビゲーションとパラメータ

複数の画面を作って相互に移動する方法を学び、リンクやボタンで画面遷移し、データを受け渡す方法を習得しましょう。

この章で学ぶこと

  • 画面遷移とナビゲーションの基本概念を理解する
  • リンクとボタンを使って画面を移動する方法を学ぶ
  • パラメータを使って画面間でデータを受け渡す方法を習得する
  • メニューを作成してアプリケーションのナビゲーションを構築する

7.1 画面遷移とナビゲーションとは

実際のアプリケーションでは、複数の画面を行き来しながら操作することが一般的です。 例えば、商品一覧画面から詳細画面に移動したり、入力フォームから確認画面に進んだりといった動きです。

ナビゲーションの基本

OutSystemsでは、Link（リンク）やButton（ボタン）を使って、 ユーザーを別の画面に移動させることができます。これを画面遷移（ナビゲーション）と呼びます。

画面遷移の種類

種類 説明 使用例
Linkによる遷移 クリック可能なテキストやアイコンで画面移動 メニュー、一覧項目
Buttonによる遷移 ボタンをクリックして画面移動 登録完了後の遷移、次へボタン
パラメータ付き遷移 データを渡しながら画面移動 IDを渡して詳細表示

画面遷移のイメージ

Webサイトでリンクをクリックすると別のページに移動するのと同じように、 OutSystemsでもLink や Button を使って、画面から画面へ移動できます。

7.2 リンクとボタンによる画面遷移

OutSystemsでは、Linkウィジェット（リンク）とButtonウィジェット（ボタン）を使って、簡単に画面遷移を実装できます。

Linkウィジェットによる遷移

Linkは、クリックすると別の画面に移動できるテキストやアイコンです。 メニューや一覧画面でよく使われます。

Linkの設定手順

  1. 1 ツールボックスから Link を画面にドラッグ＆ドロップ
  2. 2 プロパティパネルで Destination（遷移先）を設定
  3. 3 リンクのテキスト（表示名）を編集

Buttonによる遷移

Buttonは、クリックするとアクションを実行できるボタンです。 画面遷移だけでなく、データの保存や削除など、さまざまな処理に使えます。

Buttonの設定手順

  1. 1 ツールボックスから Button を画面にドラッグ＆ドロップ
  2. 2 ボタンの OnClick アクションを作成または選択
  3. 3 アクションフローに Destination（Navigate To）を追加

実習 7-1: Linkで画面遷移を作成

2つの画面を作成し、Linkで相互に移動できるようにしましょう。

手順

  1. 1 Service Studioで新しい画面を2つ作成します（例：「HomeScreen」と「AboutScreen」）
  2. 2 HomeScreenに Linkウィジェットを配置し、テキストに「Aboutページへ」と入力します
  3. 3 プロパティパネルで Destination に「AboutScreen」を設定します
  4. 4 AboutScreenにも同様に「ホームへ戻る」Linkを作成し、HomeScreenへの遷移を設定します
  5. 5 1-Click Publish でアプリを公開し、ブラウザで画面間の移動を確認します

7.3 パラメータによるデータの受け渡し

画面を移動する際、前の画面から次の画面に情報を渡したい場合があります。 例えば、商品一覧で選んだ商品のIDを詳細画面に渡して、その商品の情報を表示するといったケースです。

Input Parameterとは

Input Parameter（入力パラメータ）は、画面が受け取ることができる情報の「入れ物」です。 遷移元の画面からデータを渡すと、遷移先の画面でそのデータを使うことができます。

データ取得| C[(データベース)] C -->|商品情報を返す| B style A fill:#dbeafe,stroke:#3b82f6 style B fill:#dcfce7,stroke:#22c55e style C fill:#fef3c7,stroke:#f59e0b

Input Parameterの設定方法

受け取る側（遷移先画面）の設定

  1. 1 遷移先の画面を選択し、プロパティパネルを表示
  2. 2 Input Parameters タブで「Add Input Parameter」をクリック
  3. 3 パラメータ名（例：ProductId）とデータ型（例：Long Integer）を設定

渡す側（遷移元画面）の設定

  1. 1 遷移元のLinkまたはButtonを選択
  2. 2 Destinationで遷移先画面を設定すると、パラメータ入力欄が表示される
  3. 3 パラメータ欄に渡したい値（例：商品ID）を入力

パラメータのイメージ

郵便配達員が手紙を届けるとき、宛先（どの画面に）と手紙の中身（どんなデータを）を指定するのと同じです。 Input Parameterは「手紙の中身を受け取る郵便受け」のようなものです。

実習 7-2: パラメータで商品詳細画面を作成

商品IDをパラメータで渡して、詳細画面で表示してみましょう。

手順

  1. 1 新しい画面「ProductDetailScreen」を作成します
  2. 2 ProductDetailScreenを選択し、Input Parametersタブで「ProductId」(Long Integer型)を追加します
  3. 3 ProductDetailScreen に Textウィジェットを配置し、Expressionで「ProductId」と入力します
  4. 4 一覧画面にLinkを配置し、Destination を「ProductDetailScreen」に設定します
  5. 5 Linkのプロパティで ProductId パラメータに「123」と入力します
  6. 6 1-Click Publish で公開し、リンクをクリックして詳細画面で「123」が表示されることを確認します

詳細画面で渡したProductIdが表示されていれば成功です！

応用：複数のパラメータ

Input Parameterは複数追加できます。例えば、ProductIdとCategoryIdを同時に渡すことも可能です。

7.4 メニューの作成

アプリケーションに複数の画面がある場合、メニューを作成すると、 ユーザーがどの画面にもすぐにアクセスできるようになります。

メニューの種類

ヘッダーメニュー

画面の上部に常に表示されるメニューです。主要な画面へのリンクを配置します。

サイドメニュー

画面の左側または右側に表示されるメニューです。多くの項目を整理して配置できます。

シンプルなメニューの作り方

メニュー作成の基本手順

  1. 1 Containerウィジェットを画面の上部または左側に配置します
  2. 2 Container内に複数のLinkを縦または横に並べます
  3. 3 各Linkの Destination に対応する画面を設定します
  4. 4 スタイル（色、間隔、配置）を調整して見やすくします

共通メニューの作成（Block機能）

すべての画面に同じメニューを表示したい場合、Block（ブロック）を使うと便利です。 Blockは、複数の画面で再利用できる「部品」のようなものです。

Block を使ったメニューの作り方

  1. 1 エレメントツリーで「UI Flows」を右クリック→「Add Block」を選択
  2. 2 Block名を「MenuBlock」などと入力して作成
  3. 3 Block内にメニュー用のLinkを配置します
  4. 4 各画面で、ツールボックスから作成したBlockをドラッグ＆ドロップ

Blockのメリット

メニューの内容を変更したいとき、Blockを1回編集するだけで、 すべての画面のメニューが自動的に更新されます。とても効率的です！

実習 7-3: 共通メニューを作成

Blockを使って、すべての画面で使える共通メニューを作りましょう。

手順

  1. 1 エレメントツリーで「UI Flows」→「MainFlow」を右クリック→「Add Block」を選択
  2. 2 Block名に「HeaderMenu」と入力して作成します
  3. 3 HeaderMenuブロック内にContainerを配置し、その中に3つのLinkを横並びで配置します
  4. 4 各Linkに「ホーム」「商品一覧」「お問い合わせ」などのテキストを入力し、Destinationを設定
  5. 5 各画面の上部に、ツールボックスから「HeaderMenu」Blockをドラッグ＆ドロップ
  6. 6 1-Click Publish で公開し、すべての画面で同じメニューが表示されることを確認します

各画面でメニューが表示され、リンクが機能していれば成功です！

理解度確認クイズ

  1. Q1. 画面遷移を実装するために使えるウィジェットを2つ挙げてください。

    回答を表示

    1. Link（リンク） - クリック可能なテキストやアイコン
    2. Button（ボタン） - ボタンをクリックして遷移

  2. Q2. 画面間でデータを受け渡すために使う機能は何ですか？

    回答を表示

    Input Parameter（入力パラメータ）です。 遷移先の画面にInput Parameterを定義し、遷移元から値を渡すことでデータの受け渡しが可能になります。

  3. Q3. 複数の画面で同じメニューを表示するために使う機能は何ですか？

    回答を表示

    Block（ブロック）です。 Blockを作成してメニューを配置すれば、1回の編集ですべての画面のメニューを更新できます。

  4. Q4. Linkの Destination プロパティは何を設定するものですか？

    回答を表示

    遷移先の画面を設定します。 Linkをクリックしたときに、どの画面に移動するかを指定するプロパティです。

第7章のまとめ

  • 画面遷移は、LinkやButtonを使ってユーザーを別の画面に移動させる機能です
  • Linkはクリック可能なテキスト、Buttonはアクション実行用のボタンで、どちらも画面遷移に使えます
  • Input Parameterを使うと、画面間でデータを受け渡すことができ、動的な画面表示が可能になります
  • Blockを使えば、共通メニューなどの再利用可能な部品を作成でき、効率的に開発できます
  • ナビゲーションを適切に設計することで、ユーザーにとって使いやすいアプリケーションを作ることができます
