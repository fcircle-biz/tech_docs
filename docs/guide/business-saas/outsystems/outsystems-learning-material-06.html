第6章

処理の流れを制御する - 条件分岐とループ

「もし〜なら」という条件によって処理を変える方法と、同じ処理を繰り返す方法を学び、視覚的なフローチャートで処理の流れを理解しましょう。

この章で学ぶこと

  • 条件分岐（If文）とは何か、どのような場面で使うのかを理解する
  • Service StudioでIf文を実装して、条件によって処理を切り替える方法を習得する
  • ループ（For Each）を使って、同じ処理を繰り返す方法を学ぶ
  • 条件分岐とループを組み合わせた実践的な処理フローを作成する

6.1 条件分岐って何だろう？

日常生活の中で、私たちは常に「もし〜なら」という判断をしています。 例えば、「もし雨が降っているなら傘を持っていく、そうでないなら持っていかない」というような判断です。

アプリケーションでも同じように、条件によって処理を変える必要があります。 これを条件分岐（Conditional Branching）といいます。

条件分岐の具体例

ECサイトでの利用例

「購入金額が5,000円以上なら送料無料、5,000円未満なら送料500円」という処理を実装できます。

会員管理システム

「ユーザーがログインしているなら会員ページを表示、していないならログイン画面を表示」という処理を実装できます。

ポイント計算

「会員ランクがゴールドなら2倍ポイント、シルバーなら1.5倍、通常なら1倍」という処理を実装できます。

エラー処理

「データが存在するなら表示、存在しないならエラーメッセージを表示」という処理を実装できます。

条件分岐は、ひし形（◇）で表されます。ここで「はい」か「いいえ」に分かれて、 それぞれ異なる処理が実行されます。

6.2 If文の実装 - Service Studioでの作成方法

OutSystemsでは、Ifという要素を使って条件分岐を実装します。 アクションフロー（Server ActionやClient Action）の中に配置することで、視覚的に処理の流れを確認できます。

If文の基本構造

Ifノードの3つの要素

Condition（条件式）

チェックしたい条件を設定します。例: TotalAmount >= 5000

True（条件が真の場合）

条件が「はい」のときに実行する処理をつなぎます。

False（条件が偽の場合）

条件が「いいえ」のときに実行する処理をつなぎます。

実習 6-1: 送料計算のIf文を作成する

購入金額によって送料を計算する条件分岐を実装しましょう。

手順

  1. 1

    Server Actionを作成

    Logicタブで右クリック → 「Add Server Action」を選択し、名前を「CalculateShippingFee」とします。

  2. 2

    入力パラメータを追加

    Input Parametersで「TotalAmount」（Decimal型）を追加します。

  3. 3

    出力パラメータを追加

    Output Parametersで「ShippingFee」（Decimal型）を追加します。

  4. 4

    Ifノードを配置

    ツールボックスから「If」をドラッグして、StartとEndの間に配置します。

  5. 5

    条件式を設定

    Ifノードを選択し、Conditionプロパティに TotalAmount >= 5000 と入力します。

  6. 6

    Trueブランチに処理を追加

    「True」から「Assign」ノードをつなぎ、ShippingFee = 0 を設定します（送料無料）。

  7. 7

    Falseブランチに処理を追加

    「False」から「Assign」ノードをつなぎ、ShippingFee = 500 を設定します（送料500円）。

  8. 8

    Endノードにつなぐ

    両方のAssignノードからEndノードに矢印をつなぎます。

完成したら「1-Click Publish」で公開し、画面から呼び出してテストしてみましょう。

複数条件の分岐（Else If）

「5,000円以上」「3,000円以上」「それ以外」のように、3つ以上の分岐が必要な場合は、 Ifを連結することで実現できます。

ヒント

複数条件を扱う場合は、上から順に条件をチェックするように設計します。 最初の条件が満たされたら、その処理が実行され、以降の条件はチェックされません。

6.3 ループ（繰り返し）って何だろう？

同じ処理を何度も繰り返す必要がある場合、ループ（Loop）または繰り返しという仕組みを使います。

例えば、「顧客リストの全員にメールを送る」という処理では、 リストの1件目から順番に「メール送信」という同じ処理を繰り返すことになります。

ループの具体例

リスト処理

100件の商品データを1件ずつ読み込んで、画面に表示する処理を100回繰り返します。

集計処理

売上データを1件ずつ読み込んで、合計金額を計算する処理を繰り返します。

一括通知

顧客リストの全員に対して、通知メールを送信する処理を繰り返します。

データフィルタリング

全データを1件ずつチェックして、条件に合うものだけを抽出する処理を繰り返します。

ループの基本的な流れ

ある?} C -->|はい| D[1件のデータを
処理] D --> C C -->|いいえ| E[ループ終了] E --> F[次の処理へ] style A fill:#f1f5f9,stroke:#64748b style B fill:#e0e7ff,stroke:#6366f1 style C fill:#fef3c7,stroke:#f59e0b style D fill:#dbeafe,stroke:#3b82f6 style E fill:#dcfce7,stroke:#22c55e style F fill:#f1f5f9,stroke:#64748b

ポイント

ループ処理では、「次のデータがあるか？」をチェックして、ある限り処理を繰り返します。 データがなくなったらループを終了します。

6.4 For Eachの実装 - Service Studioでの作成方法

OutSystemsでは、For Eachという要素を使ってループ処理を実装します。 リストの各要素に対して、同じ処理を繰り返し実行できます。

For Eachの基本構造

For Eachノードの要素

Record List（対象リスト）

繰り返し処理の対象となるリストを指定します。例: GetProducts.List

Current（現在の要素）

ループの中で、今処理している1件のデータにアクセスできます。 例: ForEach.Current.Product.Name

Cycle（ループの本体）

各要素に対して実行する処理を、この中につなぎます。

実習 6-2: 商品リストの合計金額を計算する

For Eachを使って、商品リストの合計金額を計算しましょう。

手順

  1. 1

    Server Actionを作成

    名前を「CalculateTotalPrice」とします。

  2. 2

    入力パラメータを追加

    「ProductList」（Product List型）を追加します。

  3. 3

    出力パラメータを追加

    「TotalPrice」（Decimal型）を追加します。

  4. 4

    初期化処理を追加

    「Assign」ノードを配置し、TotalPrice = 0 を設定します。

  5. 5

    For Eachノードを配置

    ツールボックスから「For Each」をドラッグして配置します。

  6. 6

    Record Listを設定

    For EachノードのRecord Listプロパティに ProductList を設定します。

  7. 7

    ループ内の処理を追加

    「Cycle」から「Assign」ノードをつなぎ、 TotalPrice = TotalPrice + ForEach.Current.Product.Price を設定します。

  8. 8

    Endノードにつなぐ

    For EachノードからEndノードに矢印をつなぎます。

これで、商品リストの全商品の価格を合計する処理が完成しました！

ループで合計値を計算するときは、必ずループの前に0で初期化してください。 初期化しないと、前回の値が残ったままになってしまいます。

6.5 条件分岐とループの組み合わせ

実際のアプリケーション開発では、ループの中で条件分岐を使うケースがよくあります。 例えば、「リストの中から特定の条件に合うものだけを処理する」といった場合です。

実践例: 割引対象商品の合計金額を計算

商品リストから、価格が1,000円以上の商品のみを対象に、合計金額を計算する処理を作成します。

実習 6-3: 条件付き合計計算

For Eachの中でIf文を使い、条件に合う商品だけを集計しましょう。

手順

  1. 1

    Server Actionを作成

    名前を「CalculateExpensiveProductsTotal」とします。

  2. 2

    パラメータを追加

    Input: 「ProductList」、Output: 「TotalPrice」を追加します。

  3. 3

    初期化とFor Eachを配置

    TotalPriceを0で初期化し、For Eachノードを配置します。

  4. 4

    ループ内にIfを追加

    For Eachの「Cycle」から「If」ノードをつなぎます。

  5. 5

    条件を設定

    Ifの条件に ForEach.Current.Product.Price >= 1000 を設定します。

  6. 6

    Trueブランチに処理を追加

    「True」から「Assign」をつなぎ、 TotalPrice = TotalPrice + ForEach.Current.Product.Price を設定します。

  7. 7

    完成

    Falseブランチは何もしないので、そのままFor Eachに戻ります。

1,000円以上の商品だけが合計に加算される処理が完成しました！

重要ポイント

ループの中で条件分岐を使うことで、「リストの中から特定の条件に合うものだけを処理する」 という実用的な処理が実現できます。これはフィルタリングとも呼ばれ、非常によく使われるパターンです。

理解度確認クイズ

  1. Q1. 条件分岐（If文）は、どのような場合に使いますか？

    回答を表示

    条件によって処理を変えたい場合に使います。 例えば、「購入金額が5,000円以上なら送料無料、それ以外なら送料500円」のように、 状況に応じて異なる処理を実行する必要がある場合に条件分岐を使用します。

  2. Q2. OutSystemsのIfノードには、どのような要素がありますか？

    回答を表示

    3つの要素があります：
    1. Condition（条件式） - チェックしたい条件を設定
    2. True（真） - 条件が「はい」のときに実行する処理
    3. False（偽） - 条件が「いいえ」のときに実行する処理

  3. Q3. ループ（For Each）は、どのような場合に使いますか？

    回答を表示

    同じ処理を繰り返し実行したい場合に使います。 特に、リストの各要素に対して同じ処理を行う場合に便利です。 例えば、「商品リストの全商品の価格を合計する」「顧客リストの全員にメールを送る」などの処理で使用します。

  4. Q4. For Eachで合計値を計算するとき、なぜループの前に0で初期化する必要がありますか？

    回答を表示

    前回の値が残ったままにならないようにするためです。 初期化しないと、変数に前回実行時の値が残っており、その値に加算されてしまいます。 正しい合計値を得るには、必ずループの前に合計変数を0にリセットする必要があります。

第6章のまとめ

  • 条件分岐（If文）は、条件によって処理を切り替える仕組みで、「もし〜なら」という判断を実装できます
  • OutSystemsのIfノードは、Condition（条件式）、True（真）、False（偽）の3つの要素で構成されています
  • ループ（For Each）は、同じ処理を繰り返し実行する仕組みで、リストの各要素に対して処理を行えます
  • ループで合計値を計算する場合は、必ずループの前に0で初期化する必要があります
  • 条件分岐とループを組み合わせることで、「特定の条件に合うデータだけを処理する」という実用的な処理が実現できます
