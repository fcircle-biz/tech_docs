外部と連携する - REST APIとIntegration

他のシステムやサービスとデータをやり取りする方法を学びます。REST APIの基本概念と、OutSystemsでの実装方法を分かりやすく解説します。

この章で学ぶこと

  • REST APIとは何か、外部連携の基本概念を理解する
  • OutSystemsで外部のREST APIを利用する方法（Consume REST API）を学ぶ
  • OutSystemsで自分のAPIを公開する方法（Expose REST API）を学ぶ
  • APIのセキュリティとエラーハンドリングの基本を理解する

11.1 REST APIって何だろう？

スマートフォンのアプリで天気を調べたり、通販サイトで商品を検索したりするとき、 裏側では別のシステムと情報をやり取りしています。この「システム同士の情報交換」を可能にするのが API（Application Programming Interface）です。

APIの日常的な例え

レストランに例えると

レストランで注文するときを想像してください。
お客さん（アプリ）がメニューを見て、ウェイター（API）に注文を伝えます。 ウェイターは厨房（外部システム）に注文を伝え、料理ができたらお客さんに届けます。
お客さんは厨房の中身を知らなくても、ウェイターを通じて料理を受け取れます。 これがAPIの仕組みです。

REST APIとは

REST API（レスト・エーピーアイ）は、 インターネットを通じてシステム同士が情報をやり取りするためのルールです。 「REST」は Representational State Transfer の略で、シンプルで分かりやすい方法でデータを送受信できます。

sequenceDiagram participant App as OutSystemsアプリ participant API as 外部API
（天気情報サービス） participant DB as 外部データベース App->>API: 1. 東京の天気を教えて API->>DB: 2. データを取得 DB-->>API: 3. 東京: 晴れ、25度 API-->>App: 4. JSONで返信 Note over App: 5. 画面に表示

REST APIの基本的な操作

メソッド 意味 例
GET データを取得する 顧客情報を取得
POST 新しいデータを作成する 新規顧客を登録
PUT 既存データを更新する 顧客情報を変更
DELETE データを削除する 顧客情報を削除

なぜREST APIが便利なのか

例えば、天気予報サービスは日本中の気象データを持っています。 REST APIを使えば、あなたのOutSystemsアプリから天気情報を簡単に取得できます。 自分で気象データを集めたり、データベースを用意したりする必要がありません。

11.2 外部連携が必要な場面

OutSystemsアプリを作るとき、どんな場面で外部のシステムと連携する必要があるのでしょうか。 実際によくあるケースを見てみましょう。

天気情報の取得

天気予報APIから、各地の天気や気温を取得してアプリに表示します。

決済処理

Stripe、PayPalなどの決済サービスと連携して、クレジットカード決済を実装します。

地図・位置情報

Google Maps APIで地図を表示したり、住所から緯度経度を取得したりします。

メール・SMS送信

SendGrid、Twilioなどのサービスと連携して、メールやSMSを自動送信します。

既存システム連携

社内の既存システム（在庫管理、会計システムなど）のデータを取得・更新します。

AI・機械学習

GoogleのVision APIなどを使って、画像認識や自然言語処理を実装します。

11.3 OutSystemsで外部APIを利用する（Consume REST API）

OutSystemsでは、Consume REST APIという機能を使って、 外部のREST APIを簡単に利用できます。Service Studio上で設定するだけで、外部サービスのデータを取得できます。

実習 11-1: 天気予報APIを利用する

無料の天気予報API（OpenWeatherMap）を使って、現在の天気を取得してみましょう。

準備: APIキーの取得

  1. 1 OpenWeatherMap にアクセスし、無料アカウントを作成します
  2. 2 マイページから「API keys」を開き、APIキーをコピーします（後で使用します）

手順: Service Studio での設定

  1. 1

    REST APIを追加

    Service Studioの「Logic」タブで、「Integrations」→「REST」を右クリックし、「Consume REST API」を選択します。

  2. 2

    APIの設定

    Name: 「OpenWeatherMap」、Base URL: 「https://api.openweathermap.org/data/2.5」と入力します。

  3. 3

    メソッドを追加

    「OpenWeatherMap」を右クリックし、「Add Method」を選択。 Name: 「GetCurrentWeather」、URL Path: 「/weather」と入力します。

  4. 4

    パラメータを設定

    メソッドの「Input Parameters」で、以下を追加します：
    - q（Text型）: 都市名（例: Tokyo）
    - appid（Text型）: APIキー
    - units（Text型）: metric（摂氏温度）

  5. 5

    画面から呼び出す

    画面のアクションで「Server Action」を作成し、GetCurrentWeatherメソッドを呼び出します。 取得したデータ（気温、天気など）を画面に表示します。

完成したら「1-Click Publish」して、ブラウザで確認してみましょう。 東京の現在の気温や天気が表示されたら成功です！

データの形式（JSON）

REST APIでやり取りするデータは、通常JSON（ジェイソン）という形式で記述されます。 JSONはシンプルで読みやすいテキスト形式です。

天気APIからのレスポンス例（JSON） 
{
  "name": "Tokyo",
  "main": {
    "temp": 25.3,
    "humidity": 60
  },
  "weather": [
    {
      "description": "晴れ"
    }
  ]
}

OutSystemsが自動で変換してくれる

OutSystemsは、このJSON形式のデータを自動的にアプリで使えるデータ構造（Structure）に変換してくれます。 JSONの詳しい知識がなくても、プロパティから簡単にアクセスできます。

11.4 OutSystemsで自分のAPIを公開する（Expose REST API）

OutSystemsでは、自分のアプリのデータを外部から利用できるように、 REST APIとして公開することもできます。 これにより、モバイルアプリや他のシステムから、あなたのOutSystemsアプリのデータにアクセスできるようになります。

実習 11-2: 顧客リストAPIを公開する

顧客エンティティのデータを取得できるREST APIを作成し、外部から利用できるようにしましょう。

手順

  1. 1

    REST APIを作成

    「Logic」タブで「Integrations」→「REST」を右クリックし、「Expose REST API」を選択します。 Name: 「CustomerAPI」と入力します。

  2. 2

    メソッドを追加

    「CustomerAPI」を右クリックし、「Add Method」を選択。 Name: 「GetCustomers」、HTTP Method: 「GET」を選択します。

  3. 3

    データを取得するロジックを作成

    GetCustomersメソッドをダブルクリックしてアクションフローを開きます。 Aggregateを追加し、Customerエンティティからデータを取得します。

  4. 4

    レスポンスを設定

    AggregateのOutputをメソッドのOutput Parameter（CustomerList）に割り当てます。 これで、取得した顧客リストがJSON形式で返されます。

  5. 5

    公開して確認

    「1-Click Publish」で公開します。 Service Studioで「CustomerAPI」を右クリックし、「Open Documentation」を選択すると、 APIのURLとドキュメントが表示されます。

APIのURL例

https://あなたの環境.outsystemscloud.com/CustomerAPI/rest/v1/GetCustomers

このURLにブラウザでアクセスすると、顧客データがJSON形式で表示されます。

11.5 セキュリティとエラーハンドリング

APIを公開したり利用したりするときは、セキュリティとエラー対策が重要です。 安全に外部連携を行うための基本的なポイントを押さえましょう。

APIのセキュリティ対策

APIキーの管理

外部サービスのAPIキーは、Site Propertiesに保存します。 コードに直接書き込まず、設定値として管理することで、セキュリティを高めます。

認証の追加

公開したAPIには認証を設定しましょう。 OutSystemsでは「Basic Authentication」や「Internal Network Only」などの認証方式を簡単に設定できます。

HTTPSの使用

OutSystemsのPersonal Environmentは自動的にHTTPSが有効になっています。 データが暗号化されるため、安全にやり取りできます。

エラーハンドリング

外部APIを利用するとき、ネットワークエラーやAPIのメンテナンスなどで失敗することがあります。 エラーが発生したときの対処方法を実装しましょう。

エラーハンドリングの実装例（擬似コード） 
// アクションフローで Try-Catch パターンを使用

Try（成功時の処理）:
  - REST APIメソッドを呼び出す
  - 取得したデータを画面に表示

Exception（エラー時の処理）:
  - エラーメッセージをログに記録
  - ユーザーに分かりやすいメッセージを表示
    例: 「天気情報の取得に失敗しました。しばらくしてから再度お試しください。」
  - デフォルトデータを表示（あれば）

注意: タイムアウト設定

REST APIメソッドのプロパティでTimeout（タイムアウト）を適切に設定しましょう。 デフォルトは100秒ですが、レスポンスが遅いAPIの場合は延長し、 速いAPIなら短縮してユーザー体験を向上させます。

ステータスコードの理解

コード 意味 対処方法
200 OK 成功 データを正常に取得できた
400 Bad Request リクエスト不正 パラメータの確認が必要
401 Unauthorized 認証エラー APIキーが間違っている
404 Not Found 存在しない URLやエンドポイントの確認
500 Server Error サーバーエラー 外部サービスの問題、時間をおいて再試行

ヒント: テストツールの活用

APIの動作を確認するには、Postmanやcurlなどのツールが便利です。 OutSystemsで実装する前に、これらのツールでAPIの挙動を確認すると開発がスムーズになります。

トラブルシューティング

APIを呼び出すとタイムアウトエラーが発生する

原因: APIのレスポンスが遅い、またはタイムアウト設定が短すぎる

解決方法: REST APIメソッドのプロパティで「Timeout」を延長します（例: 100秒→300秒）。 また、ネットワーク接続も確認してください。

401 Unauthorizedエラーが発生する

原因: APIキーが間違っている、または認証設定が不正

解決方法: APIキーを再確認します。Site Propertiesに正しく保存されているか、 パラメータ名が正しいか（appid、api_key など）をチェックしてください。

JSONのパースエラーが発生する

原因: APIから返されるJSON構造とOutSystemsの定義が一致しない

解決方法: 「Test」ボタンで実際のレスポンスを確認し、「Response」タブでStructureを自動生成します。 APIドキュメントと照らし合わせて、必要な項目が含まれているか確認してください。

CORSエラーが発生する（ブラウザから直接APIを呼び出す場合）

原因: ブラウザのセキュリティ制限により、外部APIへの直接アクセスがブロックされている

解決方法: OutSystemsでは、REST API呼び出しはサーバー側（Server Action）で行うことを推奨します。 Client Actionから直接呼び出すのではなく、Server Actionを経由することでCORS問題を回避できます。

理解度確認クイズ

  1. Q1. REST APIの4つの基本メソッド（GET、POST、PUT、DELETE）それぞれの役割を説明してください。

    回答を表示

    GET: データを取得する（例: 顧客情報を取得）
    POST: 新しいデータを作成する（例: 新規顧客を登録）
    PUT: 既存データを更新する（例: 顧客情報を変更）
    DELETE: データを削除する（例: 顧客情報を削除）

  2. Q2. OutSystemsで外部のREST APIを利用するには、どの機能を使いますか？

    回答を表示

    Consume REST API（REST APIを消費する）機能を使います。 Service Studioの「Logic」タブ→「Integrations」→「REST」から設定できます。

  3. Q3. APIキーをセキュアに管理するために、OutSystemsではどこに保存すべきですか？

    回答を表示

    Site Properties（サイトプロパティ）に保存します。 コードに直接書き込まず、設定値として管理することでセキュリティを高めます。 環境ごとに異なるAPIキーを設定することもできます。

  4. Q4. HTTPステータスコード「404」は何を意味しますか？

    回答を表示

    404 Not Found（見つからない）を意味します。 リクエストしたリソース（URL、エンドポイント）が存在しない場合に返されます。 URLやパスの確認が必要です。

第11章のまとめ

  • REST APIは、システム同士がインターネット経由でデータをやり取りする仕組みで、GET、POST、PUT、DELETEの基本メソッドがあります
  • Consume REST APIを使うと、外部サービス（天気予報、決済、地図など）のAPIを簡単に利用できます
  • Expose REST APIを使うと、自分のOutSystemsアプリのデータを外部に公開し、他のシステムから利用できるようにできます
  • APIキーはSite Propertiesに保存し、認証やHTTPSを設定することでセキュリティを確保します
  • エラーハンドリングとタイムアウト設定により、APIが失敗したときでもユーザーに適切なメッセージを表示できます
