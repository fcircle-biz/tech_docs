第10章

見た目を整える - CSSとテーマ

アプリケーションのデザインをカスタマイズする方法を学び、テーマの適用や色・フォント変更、レスポンシブデザインの基本を理解しましょう。

この章で学ぶこと

  • OutSystemsのテーマシステムとCSSの基礎を理解する
  • アプリケーションの色やフォント、スタイルをカスタマイズする方法を習得する
  • レスポンシブデザインの基本とスマホ対応の実装方法を学ぶ
  • テーマエディタを使った実践的なデザインカスタマイズを体験する

10.1 OutSystemsのテーマシステムとは

これまでの章では、アプリケーションの機能を作ってきました。 しかし、ユーザーにとって「見た目」も非常に重要です。 美しくて使いやすいデザインは、アプリケーションの価値を大きく高めます。

CSSとは何か

CSS（Cascading Style Sheets）は、 Webページの「見た目」を制御する言語です。 文字の色、大きさ、配置、余白、背景色など、デザインに関するすべてを設定できます。

HTMLとCSSの役割分担

HTML
文章の構造と内容を定義
（例: ここはタイトル、ここは本文、ここはボタン）
CSS
HTMLで定義した要素の見た目を装飾
（例: タイトルは青色で大きく、ボタンは角を丸く）

OutSystemsのテーマシステム

OutSystemsでは、CSSを直接書かなくても、テーマという仕組みでデザインを管理できます。 テーマには、アプリケーション全体で使う色、フォント、スタイルなどがまとめられています。

テーマのメリット

テーマを使えば、一箇所の設定を変更するだけで、アプリ全体の色やデザインを統一的に変更できます。 例えば、会社のブランドカラーを変更したい場合、テーマの色設定を変えるだけで全画面に反映されます。

10.2 色をカスタマイズする

アプリケーションの印象を大きく左右するのが「色」です。 OutSystemsでは、テーマエディタを使って簡単に色をカスタマイズできます。

色の指定方法

CSSでは、色を指定する方法がいくつかあります：

指定方法 例 説明
色名 red, blue 英語の色名で指定（簡単だが選択肢が少ない）
16進数 #FF5733 最も一般的な方法、細かい色指定が可能
RGB rgb(255, 87, 51) 赤・緑・青の強さを0-255で指定
RGBA rgba(255, 87, 51, 0.5) RGBに透明度（0-1）を追加

カラーパレットの選び方

プロのデザイナーは、メインカラー、アクセントカラー、背景色の3色を軸にデザインを組み立てます。 OutSystemsのテーマでも、Primary Color（主要色）、Background Color（背景色）などを設定できます。

実習 10-1: テーマの色を変更する

アプリケーションのメインカラーを変更して、デザインの印象を変えてみましょう。

手順

  1. 1 Service Studioで、左側のツリーから「UI Flows」→「Themes」→あなたのテーマ（例: MyAppTheme）を開きます
  2. 2 画面上部の「Edit Theme」ボタンをクリックして、テーマエディタを開きます
  3. 3 「Colors」タブを選択し、「Primary Color」をクリックします
  4. 4 カラーピッカーで好きな色を選ぶか、16進数カラーコード（例: #3B82F6）を入力します
  5. 5 「Apply」をクリックして変更を適用し、テーマエディタを閉じます
  6. 6 「1-Click Publish」で公開し、ブラウザで確認します。ボタンやヘッダーの色が変わっていることを確認しましょう

ヒント: Primary Colorを変更すると、ボタン、リンク、アクティブな要素など、 アプリ全体の主要な色が自動的に変わります。

10.3 フォントとテキストスタイルを設定する

文字のフォント（書体）や大きさも、アプリケーションの印象を大きく変える要素です。 読みやすさと美しさのバランスを考えて設定しましょう。

主なフォント設定

Font Family（フォントファミリー）

文字の書体を指定します。代表的なフォント：

  • ゴシック体（sans-serif）: 現代的で読みやすい（Webアプリに最適）
  • 明朝体（serif）: 伝統的で格式高い印象
  • 等幅フォント（monospace）: コード表示に使用

Font Size（フォントサイズ）

文字の大きさを指定します。単位はpx（ピクセル）や rem（相対サイズ）を使います。
一般的な本文サイズ: 14px～16px

Font Weight（太さ）

文字の太さを指定します。 normal（通常）、 bold（太字）、 または数値（100～900）で細かく指定できます。

Line Height（行間）

行と行の間隔を指定します。広めの行間（1.5～1.8）は読みやすさを向上させます。

Google Fontsの活用

OutSystemsでは、Google Fontsという無料のフォントサービスを簡単に使えます。 数百種類の美しいフォントから選べます。

例: 「Noto Sans JP」は日本語に最適化されたゴシック体フォントで、多くのWebアプリで使われています。

フォント選びの注意点

装飾的すぎるフォントは読みにくくなります。 ビジネスアプリでは、読みやすさを最優先してシンプルなフォントを選びましょう。

10.4 レスポンシブデザイン - スマホ対応

現代では、ユーザーはパソコンだけでなく、スマートフォンやタブレットからもアプリを使います。 レスポンシブデザインとは、 画面サイズに応じて自動的にレイアウトを調整するデザイン手法です。

OutSystemsのレスポンシブ機能

OutSystemsでは、以下の機能を使ってレスポンシブデザインを実現できます：

Grid System（グリッドシステム）

画面を12分割のグリッドに分け、カラム幅を指定することで、画面サイズに応じて自動的にレイアウトが調整されます。

Device Detection（デバイス判定）

現在のデバイスがスマホかPCかを判定して、表示内容を切り替えることができます。

Visibility Classes（表示切替）

特定のデバイスでのみ表示・非表示を切り替えるCSSクラスを使えます（例: phone-only, tablet-only）。

Flexible Layout（柔軟なレイアウト）

コンテナの幅を固定値ではなくパーセントで指定することで、画面サイズに応じて伸縮します。

実習 10-2: スマホ表示を確認する

作成したアプリがスマートフォンでどう表示されるか確認しましょう。

手順

  1. 1 アプリを「1-Click Publish」で公開し、ブラウザで開きます
  2. 2 ブラウザで F12 キーを押して、開発者ツールを開きます
  3. 3 開発者ツールの左上にある「デバイスツールバー切り替え」アイコン（スマホとタブレットのアイコン）をクリックします
  4. 4 上部のドロップダウンから「iPhone 12」や「iPad」などのデバイスを選択します
  5. 5 レイアウトが画面サイズに応じて変化することを確認しましょう

OutSystemsのReactive Web Appは、デフォルトでレスポンシブ対応されているため、 基本的なスマホ表示は自動的に最適化されます。

モバイルファーストの考え方

近年は「モバイルファースト」という設計思想が主流です。 これは、最初にスマホ画面でのデザインを考え、その後PCサイズに拡張するという考え方です。 スマホで使いやすいアプリは、PCでも使いやすくなります。

10.5 カスタムCSSクラスでさらにカスタマイズ

テーマエディタだけでは実現できない細かいデザイン調整が必要な場合、 カスタムCSSクラスを追加することで、より自由度の高いカスタマイズができます。

CSSクラスの仕組み

CSSクラスとは、特定のスタイル（見た目の設定）に名前をつけたものです。 HTMLの要素にクラス名を指定することで、そのスタイルが適用されます。

CSS クラスの例 
/* カスタムボタンのスタイル定義 */
.my-custom-button {
    background-color: #3B82F6;  /* 背景色を青に */
    color: white;                /* 文字色を白に */
    padding: 10px 20px;          /* 内側の余白 */
    border-radius: 8px;          /* 角を丸く */
    font-weight: bold;           /* 文字を太字に */
}

/* マウスを載せたときの変化 */
.my-custom-button:hover {
    background-color: #2563EB;  /* 少し濃い青に */
}

OutSystemsでCSSクラスを追加する方法

1 テーマにCSSを追加

テーマを開き、「Style Sheet」タブでカスタムCSSを記述します。

2 ウィジェットにクラスを指定

ボタンやテキストなどのウィジェットを選択し、プロパティの「Style Classes」に作成したクラス名を入力します。

3 公開して確認

「1-Click Publish」で公開し、ブラウザで見た目を確認します。

よく使うCSSプロパティ

background-color: 背景色
color: 文字色
padding: 内側の余白
margin: 外側の余白
border: 枠線
border-radius: 角の丸み
box-shadow: 影
transition: アニメーション

CSS追加時の注意点

カスタムCSSを追加しすぎると、テーマとの整合性が取れなくなったり、 他の画面に予期しない影響が出る場合があります。 最初はテーマの設定で対応できないか検討してから、CSSを追加しましょう。

理解度確認クイズ

  1. Q1. CSSとは何を制御する言語ですか？

    回答を表示

    Webページの「見た目」を制御する言語です。 文字の色、大きさ、配置、余白、背景色など、デザインに関するすべての設定ができます。 HTMLが「構造と内容」を定義するのに対し、CSSは「装飾」を担当します。

  2. Q2. OutSystemsのテーマを使うメリットは何ですか？

    回答を表示

    一箇所の設定を変更するだけで、アプリ全体の色やデザインを統一的に変更できることです。 例えば、Primary Colorを変更すれば、すべてのボタンやリンクの色が自動的に変わります。 これにより、デザインの一貫性を保ちやすくなります。

  3. Q3. レスポンシブデザインとは何ですか？

    回答を表示

    画面サイズに応じて自動的にレイアウトを調整するデザイン手法です。 パソコン、タブレット、スマートフォンなど、異なるデバイスで最適な表示を実現します。 OutSystemsのReactive Web Appは、標準でレスポンシブデザインに対応しています。

  4. Q4. 色を16進数で指定する場合、どのような形式で記述しますか？

    回答を表示

    #（ハッシュ）に続けて6桁の16進数で記述します。 例: #3B82F6（青色）、 #FF5733（オレンジ色）。 最初の2桁が赤、次の2桁が緑、最後の2桁が青の強さを表します。

第10章のまとめ

  • CSSはWebページの見た目を制御する言語で、OutSystemsではテーマシステムで簡単にデザインを管理できます
  • 色のカスタマイズでは、16進数やRGBで色を指定し、テーマエディタで一括変更することで、アプリ全体のデザインを統一できます
  • フォント設定では、Font Family（書体）、Font Size（サイズ）、Font Weight（太さ）、Line Height（行間）を調整して、読みやすさを向上させます
  • レスポンシブデザインにより、パソコン、タブレット、スマホなど、あらゆるデバイスで最適な表示を実現できます
  • カスタムCSSクラスを追加することで、テーマだけでは実現できない細かいデザイン調整が可能になります
