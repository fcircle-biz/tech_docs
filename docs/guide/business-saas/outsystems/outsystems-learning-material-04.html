第4章

データを表示する - アグリゲートとリスト

データベースからデータを取り出して画面に表示する方法を学びます。アグリゲート（データの取得）とリストウィジェットを使って、一覧表示や検索機能を実装します。

この章で学ぶこと

  • アグリゲート（Aggregate）を使ったデータの取得方法を理解する
  • リストウィジェット（List Widget）でデータを一覧表示する方法を学ぶ
  • フィルタリング（Filter）と検索機能を実装する
  • ソート（並び替え）とページング（分割表示）の基本を理解する

4.1 アグリゲートとは - データを取り出す仕組み

前章（第3章）では、エンティティを使ってデータベースにデータを保存する方法を学びました。 今度は、保存したデータを取り出して画面に表示する方法を見ていきましょう。

アグリゲートの役割

アグリゲート（Aggregate）は、データベースからデータを取り出すための仕組みです。 OutSystemsでは、SQLクエリ（データベースへの問い合わせ文）を書かなくても、視覚的な操作でデータを取得できます。

アグリゲートのイメージ

アグリゲートは、図書館の検索システムのようなものです。 本棚（データベース）から特定の本（データ）を探して取り出してくれます。 「著者名が〇〇の本」「出版年が2020年以降の本」といった条件を指定できます。

従来の開発方法との比較

従来のプログラミングでは、データを取得するためにSQLクエリというコードを書く必要がありました。 OutSystemsなら、ドラッグ＆ドロップだけでデータを取得できます。

従来の方法（SQL） 

-- データを取得するクエリを書く
SELECT Id, Name, Email, Phone
FROM Contact
WHERE IsActive = 1
ORDER BY Name ASC;

構文を覚える必要があり、記述ミスでエラーになりやすい

OutSystemsの方法

画面にアグリゲートを追加

エンティティをドラッグ＆ドロップ

必要に応じてフィルタやソートを設定

視覚的な操作だけで、コードを書かずにデータを取得できる

アグリゲートの構成要素

要素 説明 例
ソース（Source） データの取得元となるエンティティ Contactエンティティ
フィルタ（Filter） データを絞り込む条件 IsActive = True
ソート（Sort） データの並び順 名前の昇順
アトリビュート 取得する項目（列） Name、Email、Phone

4.2 リストウィジェット - データを一覧表示する

アグリゲートで取得したデータを画面に表示するには、リストウィジェット（List Widget）を使います。 リストウィジェットは、取得したデータを繰り返し表示する部品です。

リストウィジェットの種類

Table（テーブル）

データを表形式で表示します。Excelの表のようなレイアウトです。

用途: 顧客一覧、商品一覧、売上データなど

List（リスト）

データを自由なレイアウトで繰り返し表示します。カード形式など。

用途: ニュース記事、商品カード、ブログ投稿など

実習 4-1: 連絡先一覧を表示する

第3章で作成した「Contact」エンティティのデータを一覧表示してみましょう。

手順

  1. 1

    新しい画面を作成

    Service Studioで「MainFlow」を右クリック → 「Add Screen」を選択 → 「Blank」テンプレートで「ContactList」という名前の画面を作成します。

  2. 2

    アグリゲートを追加

    画面を開いた状態で、左側のツールボックスから「Aggregate」を画面にドラッグ＆ドロップします。 すると、画面の上部（Preparationエリア）にアグリゲートが追加されます。

  3. 3

    エンティティを設定

    アグリゲートをダブルクリックして開きます。左側のデータタブから「Contact」エンティティを、アグリゲートの中央エリアにドラッグ＆ドロップします。 これで、Contactエンティティの全データを取得する設定が完了です。

  4. 4

    Tableウィジェットを配置

    画面デザインに戻り、ツールボックスから「Table」ウィジェットを画面にドラッグ＆ドロップします。 右側のプロパティで「Source」に先ほど作成したアグリゲート（GetContacts.List）を選択します。

  5. 5

    列を設定

    Tableの各セルに、表示したいアトリビュートをドラッグします。 例: Name、Email、Phone の3列を作成し、それぞれのセルにアトリビュートを配置します。

  6. 6

    公開して確認

    「1-Click Publish」をクリックして公開し、「Open in Browser」でブラウザで確認します。 Contactエンティティに保存されているデータが一覧表示されます。

データがまだ無い場合は、第3章で学んだフォームを使ってデータを追加してから確認しましょう。

重要ポイント

アグリゲートは画面が読み込まれるタイミング（Preparation）で自動的に実行されます。 そのため、画面を開くだけでデータが表示されます。リロードするたびに最新のデータを取得します。

4.3 フィルタリングと検索 - データを絞り込む

データが増えてくると、全件を表示するのではなく条件に合うデータだけを表示したくなります。 これを実現するのがフィルタ（Filter）機能です。

フィルタの種類

固定フィルタ

アグリゲートに直接設定する、常に適用されるフィルタです。
例: 「IsActive = True」（有効な連絡先のみ表示）

動的フィルタ（検索）

ユーザーが入力した検索キーワードに応じて動的に変わるフィルタです。
例: 検索ボックスに入力した名前で絞り込む

フィルタの書き方

アグリゲートのフィルタは、条件式で記述します。プログラミングの知識は不要で、簡単な式で書けます。

条件 書き方 意味
等しい Contact.IsActive = True IsActiveがTrueのデータ
等しくない Contact.Status <> "削除済み" Statusが「削除済み」以外
部分一致 Contact.Name like "%" + SearchKeyword + "%" Nameに検索キーワードを含む
大小比較 Contact.Age >= 20 Ageが20以上

実習 4-2: 名前で検索できるようにする

連絡先一覧に検索ボックスを追加して、名前で絞り込めるようにしましょう。

手順

  1. 1

    変数を追加

    画面（ContactList）を選択し、右クリック → 「Add Local Variable」を選択します。 変数名を「SearchKeyword」、データ型を「Text」に設定します。

  2. 2

    検索ボックスを配置

    Tableウィジェットの上に「Input」ウィジェットをドラッグ＆ドロップします。 プロパティで「Variable」に先ほど作成した「SearchKeyword」を設定します。

  3. 3

    検索ボタンを配置

    検索ボックスの横に「Button」ウィジェットを配置し、ラベルを「検索」に変更します。

  4. 4

    アクションを追加

    ボタンをダブルクリックして、アクションフローを開きます。 ツールボックスから「Refresh Data」をドラッグし、アグリゲート（GetContacts）を選択します。

  5. 5

    フィルタを設定

    アグリゲート（GetContacts）を開き、「Filters」タブで「Add Filter」をクリックします。 条件に以下を入力します：
    Contact.Name like "%" + SearchKeyword + "%"

  6. 6

    テスト

    公開してブラウザで確認します。検索ボックスに名前の一部を入力して「検索」をクリックすると、該当する連絡先だけが表示されます。

ヒント

「%」は「ワイルドカード」と呼ばれ、「何でもOK」という意味です。 「"%" + SearchKeyword + "%"」は「前後に何があってもOK、途中にSearchKeywordを含む」という条件です。

4.4 ソート機能 - データを並び替える

データを見やすくするために、特定の順序で並び替えたいことがあります。 これを実現するのがソート（Sort）機能です。

ソートの種類

昇順（Ascending）

小さい値から大きい値へ並べます。
例: A → B → C、1 → 2 → 3

降順（Descending）

大きい値から小さい値へ並べます。
例: Z → Y → X、9 → 8 → 7

ソートの設定方法

アグリゲートにソート条件を追加することで、取得したデータを指定した順序で並べることができます。

ソート設定の手順

  1. 1 アグリゲートを開く
  2. 2 「Sorting」タブを選択
  3. 3 並び替えたいアトリビュート（例: Name）をクリック
  4. 4 昇順（▲）または降順（▼）を選択
ソート対象 設定例 結果
名前（Name） 昇順（▲） あ → か → さ（五十音順）
作成日時（CreatedOn） 降順（▼） 新しい順に表示
価格（Price） 昇順（▲） 安い順に表示

複数のソート条件

複数のアトリビュートでソートすることもできます。
例: 「カテゴリの昇順」→「価格の昇順」とすると、同じカテゴリ内で価格が安い順に並びます。

4.5 ページング - データを分割表示する

データが何百件、何千件もあると、全部を一度に表示するのは大変です。 ページング（Pagination）を使うと、データを複数ページに分けて表示できます。

ページングとは

ページングのイメージ

Googleの検索結果のように、「1ページ目に10件、2ページ目に次の10件...」と分けて表示します。 画面の下に「前へ」「次へ」「1, 2, 3...」というナビゲーションが表示されます。

ページングの設定

OutSystemsでは、Tableウィジェットに簡単にページング機能を追加できます。

ページング設定の手順

  1. 1 Tableウィジェットを選択
  2. 2 右側のプロパティで「Show More」ドロップダウンを開く
  3. 3 「Pagination」の「Enabled」を「Yes」に変更
  4. 4 「Page Size」に1ページあたりの件数を設定（例: 10）

ページングのメリット

  • • 画面の読み込みが速くなる
  • • スクロールが少なくなり見やすい
  • • サーバーへの負荷が軽減される

設定時の注意点

  • • Page Sizeは5〜20件が一般的
  • • データが少ない場合は不要
  • • モバイルでは件数を少なめに

パフォーマンス向上のヒント

OutSystemsのアグリゲートは、ページングを設定すると自動的に必要なデータだけを取得します。 例えば、1ページ目を表示するときは最初の10件だけを取得するため、データが大量にあっても高速に動作します。

動作確認チェックリスト

以下の機能が正しく動作するか確認しましょう。

  • 連絡先一覧が表示される（Name、Email、Phoneの列がある）
  • 検索ボックスに名前を入力して「検索」をクリックすると、該当する連絡先だけが表示される
  • 検索ボックスを空にして検索すると、全件が表示される
  • ソート設定（例: 名前の昇順）が正しく適用されている
  • ページングを有効にした場合、「前へ」「次へ」ボタンが表示され、ページ移動できる

理解度確認クイズ

  1. Q1. アグリゲートとは何ですか？

    回答を表示

    アグリゲート（Aggregate）は、データベースからデータを取り出すための仕組みです。 SQLクエリを書かなくても、視覚的な操作でデータを取得できます。

  2. Q2. リストウィジェットにはどんな種類がありますか？代表的な2つを挙げてください。

    回答を表示

    1. Table（テーブル）: データを表形式で表示します。
    2. List（リスト）: データを自由なレイアウトで繰り返し表示します。

  3. Q3. 「Contact.Name like "%" + SearchKeyword + "%"」というフィルタ条件の意味を説明してください。

    回答を表示

    「Name」に「SearchKeyword」の内容を含むデータを取得する、という意味です。
    「%」はワイルドカード（何でもOK）なので、前後に何があってもマッチします。
    例: SearchKeywordが「田中」の場合、「田中太郎」「山田中」などがヒットします。

  4. Q4. ページングのメリットを2つ挙げてください。

    回答を表示

    1. 画面の読み込みが速くなる: 必要なデータだけを取得するため。
    2. スクロールが少なくなり見やすい: 1ページの件数を絞ることで、ユーザーが情報を探しやすくなります。

第4章のまとめ

  • アグリゲートを使うと、SQLを書かずに視覚的な操作だけでデータベースからデータを取得できます
  • リストウィジェット（TableやList）でアグリゲートのデータを画面に表示できます
  • フィルタを使うと、条件に合うデータだけを絞り込んで表示できます。検索機能の実装に使います
  • ソートでデータを昇順・降順に並び替え、ページングでデータを分割表示できます
  • これらの機能を組み合わせることで、使いやすい一覧画面を作成できます
