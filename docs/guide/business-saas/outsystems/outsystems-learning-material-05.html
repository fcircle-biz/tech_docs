第5章

ユーザーの入力を受け取る - フォームとアクション

ユーザーがデータを入力・編集できるフォームの作り方を学び、入力されたデータを保存する処理（アクション）を実装しましょう。

この章で学ぶこと

  • フォームとは何か、どのように使うのかを理解する
  • テキスト入力、選択リスト、チェックボックスなど様々な入力ウィジェットの使い方を習得する
  • データバインディングの概念と、入力フィールドとデータを紐づける方法を理解する
  • アクション（処理）を作成してデータを保存する方法を学ぶ

5.1 フォームって何だろう？

普段、Webサイトで会員登録をしたり、アンケートに答えたりするとき、 名前や住所、メールアドレスなどを入力欄に入力しますよね。 この入力欄が集まったものをフォーム（Form）と呼びます。

フォームの役割

フォームは、ユーザーからアプリケーションへ情報を渡すための窓口です。 紙の申込書に記入する感覚と同じで、必要な情報を入力してもらい、「送信」ボタンを押すことでシステムにデータが保存されます。

フォームの基本構成

テキスト・数値・日付等] A --> C[選択肢
ドロップダウン・ラジオボタン等] A --> D[チェックボックス
複数選択可能] A --> E[送信ボタン
データを保存] style A fill:#f1f5f9,stroke:#64748b style B fill:#dbeafe,stroke:#3b82f6 style C fill:#e0e7ff,stroke:#6366f1 style D fill:#f3e8ff,stroke:#a855f7 style E fill:#dcfce7,stroke:#22c55e

フォームでできること

新規登録

顧客情報、商品情報など、新しいデータをシステムに追加できます。

編集・更新

既存のデータを表示して、内容を修正・更新できます。

検索・絞り込み

検索条件を入力して、データを絞り込み表示できます。

申請・送信

申込書や問い合わせなど、情報を送信できます。

5.2 入力ウィジェットの種類

OutSystemsには、ユーザーから情報を入力してもらうための様々なウィジェット（部品）が用意されています。 入力したい情報の種類に応じて、適切なウィジェットを選びましょう。

ウィジェット 用途 例
Input 1行のテキスト入力 名前、メールアドレス、電話番号
Text Area 複数行のテキスト入力 コメント、備考、説明文
Dropdown 選択リスト（1つ選択） 都道府県、性別、年齢層
Radio Button ラジオボタン（1つ選択） はい/いいえ、配送方法
Checkbox チェックボックス（複数選択可） 興味のある分野、利用規約への同意
Date Picker 日付選択 生年月日、予約日
Switch / Toggle オン/オフの切り替え 通知の有効/無効、公開/非公開

ウィジェット選びのコツ

選択肢が5個以下ならラジオボタン、6個以上ならドロップダウンが見やすいです。 また、複数選択が必要ならチェックボックス、1つだけ選ぶならラジオボタンかドロップダウンを使います。

5.3 フォームを画面に配置しよう

OutSystemsでは、Formウィジェットを使うことで、 入力フィールドをまとめて管理し、データの保存処理を簡単に実装できます。

実習 5-1: お問い合わせフォームを作る

名前とメッセージを入力できる簡単なお問い合わせフォームを作成してみましょう。

手順

  1. 1

    新しい画面を作成

    Service Studio で新しいスクリーン「ContactForm」を追加します。

  2. 2

    Formウィジェットを配置

    ツールボックスから「Form」ウィジェットをキャンバスにドラッグ＆ドロップします。 Formの中には、入力フィールドやボタンを配置していきます。

  3. 3

    名前入力フィールドを追加

    Formの中に「Input」ウィジェットを配置します。 プロパティで「Variable」に「Name」という変数名を設定し、「Prompt」に「お名前」と入力します。 Promptは入力欄の上に表示されるラベルです。

  4. 4

    メッセージ入力フィールドを追加

    「Text Area」ウィジェットを配置します。 プロパティで「Variable」に「Message」、「Prompt」に「お問い合わせ内容」と設定します。 Text Areaは複数行の長文を入力できます。

  5. 5

    送信ボタンを追加

    「Button」ウィジェットを配置し、テキストを「送信」に変更します。 このボタンをクリックすると、入力されたデータを保存する処理が実行されます（次のステップで設定）。

  6. 6

    表示確認

    「1-Click Publish」して、ブラウザで画面を開きます。 名前とメッセージの入力欄、送信ボタンが表示されていることを確認してください。

まだ送信ボタンを押してもデータは保存されません。次のセクションで保存処理を追加します。

フォームの構成

5.4 データバインディング - 入力とデータを紐づける

データバインディング（Data Binding）とは、 画面上の入力フィールドと、アプリが管理するデータ（変数やエンティティ）を結びつける仕組みです。

データバインディングとは

ユーザーが入力フィールドに入力した内容が、自動的に変数やデータベースのフィールドに格納される仕組みです。 OutSystemsでは、Inputウィジェットの「Variable」プロパティを設定することで、簡単にデータバインディングができます。

バインディングの流れ

flowchart LR A[ユーザーが
入力欄に入力] --> B[Input ウィジェット
Variable設定済み] B --> C[変数に
自動格納] C --> D[アクション実行時
データベースに保存] style A fill:#dbeafe,stroke:#3b82f6 style B fill:#e0e7ff,stroke:#6366f1 style C fill:#f3e8ff,stroke:#a855f7 style D fill:#dcfce7,stroke:#22c55e

変数の種類

ローカル変数（Local Variable）

その画面だけで使える変数です。フォームの入力内容を一時的に保存するのに使います。 画面を閉じると内容は消えます。

エンティティの属性（Entity Attribute）

データベースに保存されるデータです。フォームの入力内容を直接エンティティにバインドすることもできます。 保存すると、次回アクセス時も内容が残ります。

使い分けのポイント

新規登録フォームの場合は、まずローカル変数に入力内容を格納し、 送信ボタンが押されたときにエンティティに保存するのが一般的です。 編集フォームの場合は、既存のエンティティデータを直接バインドします。

5.5 アクション - データを保存する処理を作ろう

アクション（Action）とは、 「ボタンがクリックされたらデータを保存する」「フォームが送信されたら確認メッセージを表示する」といった、 処理の流れを定義するものです。

アクションの種類

クライアントアクション

ユーザーのブラウザ上で実行される処理です。画面の表示切り替えや簡単な計算などに使います。

用途: 入力チェック、画面遷移、メッセージ表示

サーバーアクション

サーバー（クラウド）上で実行される処理です。データベースへの保存や複雑な計算などに使います。

用途: データ保存、外部API呼び出し、複雑な処理

実習 5-2: 送信ボタンにアクションを設定する

先ほど作成したお問い合わせフォームに、データ保存処理を追加しましょう。

手順

  1. 1

    エンティティを作成

    まず、お問い合わせデータを保存するエンティティ「ContactMessage」を作成します。 アトリビュート（項目）として「Name（Text）」「Message（Text）」「CreatedAt（DateTime）」を追加します。

  2. 2

    送信ボタンのアクションを開く

    ContactForm画面で、送信ボタンを選択します。 プロパティの「On Click」の横にある「+」アイコンをクリックして、新しいクライアントアクションを作成します。 アクション名を「SubmitContact」とします。

  3. 3

    データ保存ロジックを追加

    アクションフローのキャンバスが表示されます。 ツールボックスから「CreateContactMessage」（エンティティを作成すると自動生成されます）をドラッグして、 StartとEndの間に配置します。

  4. 4

    データをマッピング

    CreateContactMessageを選択し、プロパティで「Source」の各項目に値を設定します。 「Name」には入力フィールドの変数「Name」を、「Message」には「Message」変数を設定します。 「CreatedAt」には「CurrDateTime()」関数を使って現在日時を設定します。

  5. 5

    完了メッセージを表示

    「Message」ウィジェットをアクションフローに追加し、 「お問い合わせを受け付けました。ありがとうございます。」というメッセージを設定します。

  6. 6

    公開して動作確認

    「1-Click Publish」して、ブラウザでフォームに名前とメッセージを入力し、送信ボタンをクリックします。 完了メッセージが表示され、データがContactMessageエンティティに保存されていることを確認します。

完成です！

これでユーザーからの入力を受け取り、データベースに保存できるフォームが完成しました。

5.6 既存データを編集するフォーム

新規登録だけでなく、既存のデータを編集・更新するフォームも重要です。 編集フォームでは、まず既存データを読み込んで表示し、ユーザーが変更した内容を保存します。

編集フォームの流れ

検索] --> B[Aggregateで
データ取得] B --> C[フォームに
既存データを表示] C --> D[ユーザーが
編集] D --> E[Updateアクションで
データ更新] style A fill:#dbeafe,stroke:#3b82f6 style B fill:#e0e7ff,stroke:#6366f1 style C fill:#f3e8ff,stroke:#a855f7 style D fill:#fef3c7,stroke:#f59e0b style E fill:#dcfce7,stroke:#22c55e

編集フォームのポイント

編集フォームでは、Input Parameterを使ってIDを受け取り、 そのIDに対応するデータをAggregateで取得します。 取得したデータをフォームにバインドすることで、既存の値が自動的に入力欄に表示されます。

CreateとUpdateの違い

操作 用途 アクション
Create 新しいレコードを追加 CreateContactMessage
Update 既存レコードを更新 UpdateContactMessage
Delete レコードを削除 DeleteContactMessage

ヒント

OutSystemsでは、エンティティを作成するとCreate / Update / Deleteのアクションが自動生成されます。 これらをアクションフローにドラッグするだけで、データベース操作ができます。

理解度確認クイズ

  1. Q1. フォームの役割を説明してください。

    回答を表示

    フォームは、ユーザーからアプリケーションへ情報を渡すための窓口です。 名前や住所、メッセージなどをユーザーが入力し、送信ボタンを押すことでシステムにデータが保存されます。

  2. Q2. 選択肢が10個ある場合、ラジオボタンとドロップダウンのどちらを使うべきですか？

    回答を表示

    ドロップダウンを使うべきです。 選択肢が6個以上の場合は、ドロップダウンの方が画面をスッキリさせることができます。 ラジオボタンは選択肢が5個以下の場合に適しています。

  3. Q3. データバインディングとは何ですか？

    回答を表示

    画面上の入力フィールドと、アプリが管理するデータ（変数やエンティティ）を結びつける仕組みです。 ユーザーが入力した内容が自動的に変数やデータベースのフィールドに格納されます。 OutSystemsでは、InputウィジェットのVariableプロパティを設定することで実現します。

  4. Q4. クライアントアクションとサーバーアクションの違いを説明してください。

    回答を表示

    クライアントアクションは、ユーザーのブラウザ上で実行される処理で、画面の表示切り替えや簡単な計算などに使います。
    サーバーアクションは、サーバー（クラウド）上で実行される処理で、データベースへの保存や複雑な処理などに使います。

第5章のまとめ

  • フォームは、ユーザーからの入力を受け取るための仕組みで、Input、Text Area、Dropdown、Checkboxなど様々な入力ウィジェットを組み合わせて作成します
  • データバインディングにより、入力フィールドと変数・エンティティを紐づけることで、ユーザーの入力内容を自動的に格納できます
  • アクションを使って、ボタンクリック時にデータを保存する処理を実装できます。CreateアクションとUpdateアクションを使い分けましょう
  • クライアントアクションはブラウザ上で、サーバーアクションはサーバー上で実行され、用途に応じて使い分けることが重要です
  • フォームを使うことで、ユーザーとアプリケーションの双方向のやり取りが実現でき、実用的なアプリケーションが作成できます
