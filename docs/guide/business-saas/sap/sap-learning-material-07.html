7.1 SAP Fioriとは

SAP Fioriは、SAPの次世代ユーザーインターフェース（UX）デザインシステムです。従来のSAP GUIに代わる、モダンで直感的なWebベースのインターフェースを提供します。

Fioriの5つのデザイン原則
  • Role-based（役割ベース）: ユーザーの役割に応じた必要最小限の機能のみ提供
  • Responsive（レスポンシブ）: デスクトップ、タブレット、スマホで同じアプリが動作
  • Simple（シンプル）: 3クリックルール - 主要タスクは3クリック以内で完了
  • Coherent（一貫性）: すべてのFioriアプリで統一されたデザイン
  • Adaptive（適応的）: ユーザーの好みや状況に応じて最適化

7.2 Fioriアプリケーションタイプ

Fioriアプリは、用途に応じて3つの主要タイプに分類されます。

1. Transactional Apps（トランザクション型）

日常業務のトランザクション処理を実行するアプリ。データの作成・更新・削除が可能。

  • 例: 発注書作成、受注登録、経費精算申請
  • 特徴: ワークフロー対応、承認機能、データ更新
2. Analytical Apps（分析型）

ビジネスデータを可視化・分析するアプリ。ダッシュボードやレポート機能を提供。

  • 例: 売上分析、在庫レベル監視、KPIダッシュボード
  • 特徴: チャート表示、ドリルダウン、リアルタイム集計
3. Fact Sheet Apps（ファクトシート型）

特定のビジネスオブジェクトの詳細情報を表示するアプリ。

  • 例: 顧客情報カード、製品詳細、従業員プロフィール
  • 特徴: マスタデータ表示、関連情報のナビゲーション

7.3 SAPUI5フレームワーク

SAPUI5は、Fioriアプリケーション開発のためのJavaScriptフレームワークです。HTML5、CSS3、JavaScriptをベースにしています。

SAPUI5の主要機能
  • MVC アーキテクチャ: Model-View-Controller パターンでコード整理
  • データバインディング: UIとデータモデルの自動同期
  • 豊富なUIコントロール: ボタン、テーブル、フォームなど100以上のコンポーネント
  • ルーティング: SPA（Single Page Application）のナビゲーション
  • 国際化（i18n）: 多言語対応
  • テーマ: SAP Blueなどの標準テーマ

SAPUI5のMVCアーキテクチャ

XML/HTML - UI定義] C[Controller
JavaScript - ビジネスロジック] M[Model

簡単なFioriアプリの構造

// manifest.json - アプリケーション設定
{
  "sap.app": {
    "id": "com.example.myapp",
    "type": "application",
    "title": "My Fiori App"
  },
  "sap.ui5": {
    "rootView": {
      "viewName": "com.example.myapp.view.App",
      "type": "XML"
    }
  }
}

<!-- View (XML) - UI定義 -->
<mvc:View
  controllerName="com.example.myapp.controller.App"
  xmlns:mvc="sap.ui.core.mvc"
  xmlns="sap.m">

  <Page title="製品一覧">
    <List items="{/Products}">
      <StandardListItem
        title="{ProductName}"
        description="{Category}"
        info="{UnitPrice} 円" />
    </List>
  </Page>

</mvc:View>

// Controller (JavaScript) - ビジネスロジック
sap.ui.define([
  "sap/ui/core/mvc/Controller",
  "sap/ui/model/json/JSONModel"
], function (Controller, JSONModel) {
  "use strict";

  return Controller.extend("com.example.myapp.controller.App", {
    onInit: function () {
      // データモデルの作成
      var oModel = new JSONModel({
        Products: [
          { ProductName: "ノートPC", Category: "電子機器", UnitPrice: 120000 },
          { ProductName: "マウス", Category: "周辺機器", UnitPrice: 2000 }
        ]
      });
      this.getView().setModel(oModel);
    }
  });
});

7.4 Fiori Launchpad

Fiori Launchpadは、すべてのFioriアプリへの統一されたエントリーポイントです。ユーザーの役割に応じてカスタマイズされたタイル形式でアプリが表示されます。

Launchpadの主要機能
  • タイル: アプリを起動するためのタイル。動的タイルはKPI情報をリアルタイム表示
  • グループ: 関連するアプリをグループ化（例: 購買、販売、財務）
  • カタログ: 利用可能なアプリの一覧とアサインメント
  • パーソナライゼーション: ユーザーごとにタイル配置をカスタマイズ
  • 検索: アプリとデータの統合検索

7.5 SAP Business Application Studio（BAS）

SAP BASは、クラウドベースの統合開発環境（IDE）です。Fioriアプリの開発に最適化されています。

BASの主要機能
  • クラウドIDE: ブラウザで開発、インストール不要
  • テンプレート: Fiori Elementsなどのプロジェクトテンプレート
  • プレビュー機能: ローカルサーバーで即座に動作確認
  • Git統合: バージョン管理とCI/CD連携
  • CDS開発: CAP（Cloud Application Programming Model）サポート

7.6 ODataサービスとの連携

FioriアプリはバックエンドのSAPシステムとOData（Open Data Protocol）を使って通信します。ODataはRESTful APIの標準規格です。

sequenceDiagram participant F as Fiori App participant O as OData Service participant G as Gateway participant H as HANA Database F->>O: GET /Products O->>G: BAPI/RFC呼び出し G->>H: SELECT * FROM PRODUCTS H-->>G: データセット G-->>O: 結果（ABAP構造体） O-->>F: JSON/XML形式で返却
ODataの主要操作
  • GET: データ取得（例: /Products）
  • POST: データ作成（例: 新規受注登録）
  • PUT/PATCH: データ更新
  • DELETE: データ削除
  • $filter: 条件フィルタ（例: /Products?$filter=Price gt 1000）
  • $expand: 関連エンティティの展開（例: /Orders?$expand=OrderItems）

ODataモデルの使用例

// ODataモデルの定義
var oModel = new sap.ui.model.odata.v2.ODataModel("/sap/opu/odata/sap/ZPRODUCT_SRV/");
this.getView().setModel(oModel);

// データの読み取り
oModel.read("/ProductSet", {
  success: function(oData) {
    console.log("製品データ取得成功:", oData.results);
  },
  error: function(oError) {
    console.error("エラー:", oError);
  }
});

// データの作成
oModel.create("/ProductSet", {
  ProductID: "P001",
  ProductName: "新製品",
  Price: 15000
}, {
  success: function() {
    sap.m.MessageToast.show("製品を作成しました");
  }
});

7.7 Fiori Elements

Fiori Elementsは、標準的なUIパターンを自動生成するフレームワークです。コーディング量を大幅に削減できます。

Fiori Elementsのテンプレート
  • List Report: 一覧表示とフィルタ機能
  • Object Page: 詳細表示ページ
  • Worklist: タスク一覧とアクション
  • Overview Page: カード形式のダッシュボード
  • Analytical List Page: チャートと一覧の組み合わせ
理解度確認演習 7-1: Fioriデザイン原則の適用
シナリオ

従来のSAP GUIで10画面遷移が必要だった「購買依頼承認」プロセスをFioriアプリとして再設計します。Fioriの5つのデザイン原則をどのように適用すべきか考えてください。

解答例
  • Role-based: 承認者には承認・却下ボタンのみ表示、詳細編集機能は非表示
  • Responsive: スマホで移動中でも承認可能、外出先からタブレットでアクセス
  • Simple: 3クリックで完了:
    1. 購買依頼一覧から選択
    2. 詳細確認
    3. 承認ボタンをクリック
  • Coherent: 他の承認アプリと同じUIパターン（承認フロー統一）
  • Adaptive: 承認履歴から優先度の高い依頼を自動的に上位表示
理解度確認クイズ
  1. SAP Fioriの「3クリックルール」とは何ですか？なぜ重要ですか？
  2. Transactional Apps、Analytical Apps、Fact Sheet Appsの違いを説明してください。
  3. SAPUI5のMVCアーキテクチャにおける各コンポーネントの役割は何ですか？
  4. Fiori LaunchpadとSAP GUIの主な違いを3つ挙げてください。
  5. ODataの$filterと$expandクエリオプションの使い方を説明してください。
解答のヒント

1. 主要タスクは3クリック以内で完了すべき。ユーザー効率向上のため

2. Transactional=データ更新、Analytical=分析・可視化、FactSheet=詳細表示

3. Model=データ、View=UI、Controller=ビジネスロジック

4. Webベース、タイル形式、レスポンシブデザイン vs デスクトップアプリ、メニュー形式、固定サイズ

5. $filter=条件絞り込み、$expand=関連データを一度に取得
