第2章: SAPアーキテクチャとナビゲーション基礎

SAPシステムの構造と操作方法を理解する

この章で学ぶこと

2.1 SAP 3層アーキテクチャ

SAPシステムは3層アーキテクチャ（3-tier architecture）を採用しており、これにより高い拡張性とパフォーマンスを実現しています。

flowchart TB subgraph Layer1[プレゼンテーション層] A[SAP GUI] B[Webブラウザ - Fiori] C[モバイルアプリ] end subgraph Layer2[アプリケーション層] D[アプリケーションサーバー
ABAP インスタンス] E[Java インスタンス] F[ビジネスロジック処理] end subgraph Layer3[データベース層] G[SAP HANA Database] H[データ永続化] end A --> D B --> D C --> D D --> G E --> G
各層の役割
  • プレゼンテーション層: ユーザーインターフェース（SAP GUI、Webブラウザ）を提供。実際のビジネスロジックは持たず、画面表示と入力のみ担当
  • アプリケーション層: ビジネスロジックの実行、ABAPプログラムの処理、トランザクション管理を担当
  • データベース層: すべてのビジネスデータを永続化。SAP HANAを使用することでリアルタイム分析も可能

2.2 クライアント（Mandant）とマルチテナント構造

SAPの特徴的な概念の一つがクライアント（Mandant）です。1つのSAPシステム内に複数の独立した企業データを持つことができます。

クライアントの特徴
  • データの独立性: 各クライアントのデータは完全に分離され、他のクライアントから見えない
  • 共通設定の共有: プログラムやカスタマイジング設定の一部はクライアント横断で共有可能
  • 典型的な使い方:
    • クライアント 000, 001: システム管理用（SAP標準設定）
    • クライアント 100: 開発環境
    • クライアント 200: テスト環境
    • クライアント 300: 本番環境
重要な注意事項

ログイン時に間違ったクライアントを選択すると、全く異なるデータにアクセスすることになります。必ず正しいクライアント番号を確認してからログインしましょう。

2.3 SAP GUIの基本操作

SAP GUI（Graphical User Interface）は、SAPシステムにアクセスするための主要なクライアントソフトウェアです。WindowsやMac、Linuxにインストールして使用します。

2.3.1 SAP GUIのログイン画面

ログインに必要な情報
  1. クライアント（Client）: 3桁の数字（例: 100, 200, 300）
  2. ユーザー名（User）: システム管理者から付与されたID
  3. パスワード（Password）: 初回ログイン時は変更が必要
  4. ログオン言語（Language）: JA（日本語）、EN（英語）など

2.3.2 SAP Easy Accessメニュー

ログイン後、最初に表示されるのがSAP Easy Accessメニューです。ここから各種機能にアクセスできます。

flowchart TD A[SAP Easy Access] --> B[お気に入り] A --> C[SAPメニュー] C --> D[会計] C --> E[ロジスティクス] C --> F[人事] C --> G[ツール] D --> H[財務会計 - FI] D --> I[管理会計 - CO] E --> J[購買 - MM] E --> K[販売 - SD]

2.4 トランザクションコード（Tコード）

トランザクションコード（通称: Tコード）は、SAPの特定の機能や画面に素早くアクセスするための4文字前後のショートカットコードです。熟練ユーザーはメニューをたどるのではなく、Tコードを直接入力して操作します。

よく使われるトランザクションコード
Tコード 機能 用途
SE38 ABAPエディタ プログラム開発
SE80 Object Navigator 統合開発環境
VA01 受注登録 販売管理
ME21N 発注書登録 購買管理
FB50 仕訳入力 財務会計
MM03 品目マスタ表示 在庫管理
/n 現在の画面を終了 ナビゲーション
/nex ログアウト システム終了
Tコードの入力方法

画面上部のコマンドフィールド（テキスト入力欄）に直接入力してEnterキーを押します。例: /nVA01 で新しいセッションで受注登録画面を開く

2.5 画面要素とデータ入力の基本

SAP画面には共通の構成要素があり、これらを理解することで効率的な操作が可能になります。

SAP画面の主要構成要素
  • メニューバー: システム、編集、移動、お気に入り、ヘルプなどのメニュー
  • コマンドフィールド: Tコード入力用のテキストボックス
  • アイコンバー（標準ツールバー）: 保存、戻る、終了などの共通ボタン
  • タイトルバー: 現在のトランザクション名を表示
  • アプリケーションツールバー: 画面固有の機能ボタン
  • ワークエリア: データ入力やテーブル表示のメインエリア
  • ステータスバー: メッセージやシステム情報を表示

2.5.1 便利なファンクションキー

覚えておくべきファンクションキー
キー 機能
F1 フィールドヘルプ - カーソル位置のフィールド説明を表示
F4 値ヘルプ - 入力可能な値の一覧を表示（サーチヘルプ）
F3 戻る - 前の画面に戻る
F8 実行 - プログラムやレポートを実行
Ctrl + S 保存 - データを保存
Ctrl + F 検索 - 画面内のテキスト検索

2.6 お気に入りとユーザーメニューのカスタマイズ

頻繁に使用するトランザクションはお気に入りに登録することで、アクセス効率を大幅に向上できます。

実習 2-1: お気に入りの登録
手順
  1. SAP Easy Accessメニューで、登録したいトランザクションを右クリック
  2. 「お気に入りに追加」を選択
  3. お気に入りフォルダを作成し、分類して整理
  4. 次回からは「お気に入り」タブから素早くアクセス可能
推奨フォルダ構成例
  • マスタデータ管理（MM03, XD03, FK03など）
  • 日常業務（VA01, ME21N, FB50など）
  • レポート・照会（VA05, ME2Nなど）
  • 開発ツール（SE38, SE80, SE11など）
理解度確認クイズ
  1. SAP 3層アーキテクチャの各層の名前と役割を説明してください。
  2. クライアント（Mandant）とは何ですか？その目的を説明してください。
  3. トランザクションコード「/n」と「/nex」の違いは何ですか？
  4. F4キーとF1キーの機能の違いを説明してください。
  5. SAP GUIログイン時に必要な4つの情報を挙げてください。
解答のヒント

1. プレゼンテーション層（UI）、アプリケーション層（ビジネスロジック）、データベース層（データ永続化）

2. 1つのSAPシステム内に複数の独立した企業データを持つための仕組み

3. /nは現在の画面を終了、/nexはシステムからログアウト

4. F4は入力可能値一覧、F1はフィールド説明

5. クライアント番号、ユーザー名、パスワード、ログオン言語
