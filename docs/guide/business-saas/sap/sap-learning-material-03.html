第3章: 主要モジュール概要（FI/CO、MM/SD、PP、HCM）

SAPモジュールの全体像と相互連携

この章で学ぶこと
  • FI（財務会計）モジュール: 総勘定元帳、買掛金・売掛金管理
  • CO（管理会計）モジュール: コストセンタ会計、利益センタ会計
  • MM（購買在庫管理）モジュール: 購買依頼、発注、在庫管理
  • SD（販売管理）モジュール: 受注処理、出荷、請求
  • PP（生産管理）モジュール: 生産計画、MRP、製造指図
  • HCM（人事管理）モジュール: 人事マスタ、組織管理、給与計算
  • モジュール間の統合フローの理解

3.1 SAPモジュール全体像

SAPシステムは機能別モジュールに分かれており、各モジュールが企業の特定業務領域を担当します。すべてのモジュールは1つのデータベース上で統合されており、リアルタイムでデータ連携します。

flowchart TB subgraph Finance[財務会計領域] FI[FI - 財務会計] CO[CO - 管理会計] TR[TR - 資金管理] end subgraph Logistics[ロジスティクス領域] MM[MM - 購買在庫] SD[SD - 販売管理] PP[PP - 生産管理] WM[WM - 倉庫管理] end subgraph HR[人事領域] HCM[HCM - 人事管理] PA[PA - 人事管理] PY[PY - 給与計算] end MM --> FI SD --> FI PP --> CO HCM --> FI

3.2 FI（財務会計）モジュール

FI（Financial Accounting）は、企業の外部向け財務報告を管理するモジュールです。会計基準や法規制に準拠した財務諸表を作成します。

FIモジュールの主要サブモジュール
  • FI-GL（総勘定元帳）: すべての会計取引を記録する中心的帳簿。貸借対照表、損益計算書の作成
  • FI-AR（売掛金管理）: 得意先（顧客）への請求書発行、入金管理、債権管理
  • FI-AP（買掛金管理）: 仕入先への支払管理、債務管理
  • FI-AA（固定資産管理）: 固定資産の取得、減価償却、除却管理
  • FI-BL（銀行管理）: 銀行口座管理、電子バンキング、キャッシュマネジメント

3.3 CO（管理会計）モジュール

CO（Controlling）は、社内の経営管理のための情報を提供するモジュールです。部門別損益、製品別原価、プロジェクト収益性などを分析します。

COモジュールの主要サブモジュール
  • CO-OM（間接費管理）: コストセンタ会計、内部指図管理
  • CO-PC（製品原価管理）: 製品別の標準原価設定、原価計算
  • CO-PA（収益性分析）: 製品別、顧客別、地域別の収益性分析
  • CO-PCA（利益センタ会計）: 事業部別、拠点別の損益管理

コストセンタ会計の流れ

コストセンタ（部門）ごとに費用を集計し、予算管理と実績比較を行います。

flowchart TD A[経費発生] --> B[FI伝票転記] B --> C[CO コストセンタへ配賦] C --> D[部門別コスト集計] D --> E[予実比較レポート]

3.4 MM（購買在庫管理）モジュール

MM（Materials Management）は、資材の購買から在庫管理までを統合的に管理するモジュールです。

MMモジュールの主要機能
  • 購買依頼（PR）: 現場からの購入リクエスト
  • 購買発注（PO）: 仕入先への正式発注
  • 入庫処理: 納品された資材の受領登録
  • 請求書照合: 発注-入庫-請求書の3-way matching
  • 在庫管理: 倉庫別、ロット別の在庫数量管理
  • 棚卸: 実地棚卸と帳簿在庫の照合

3.5 SD（販売管理）モジュール

SD（Sales and Distribution）は、見積から受注、出荷、請求までの販売プロセスを管理します。

SDモジュールの主要機能
  • 見積管理: 顧客への見積書作成
  • 受注管理: 販売注文の登録と在庫引当
  • 出荷処理: 出荷伝票作成とピッキング
  • 請求処理: 出荷実績に基づく請求書発行
  • 顧客マスタ管理: 得意先情報、価格条件管理
  • 与信管理: 顧客ごとの与信限度額チェック

3.6 PP（生産管理）モジュール

PP（Production Planning）は、生産計画から製造実行、原価管理までを統合管理します。

PPモジュールの主要機能
  • 需要管理: 販売計画に基づく生産所要量計算
  • MRP（資材所要量計画）: 必要な資材の自動発注提案
  • 製造指図: 生産オーダーの発行と進捗管理
  • キャパシティ計画: 生産設備の稼働計画
  • BOM（部品表）管理: 製品の構成部品定義
  • 作業手順管理: 工程別の作業時間と作業順序

3.7 HCM（人事管理）モジュール

HCM（Human Capital Management）は、従業員情報、組織構造、給与計算、勤怠管理を統合的に管理します。

HCMモジュールの主要機能
  • PA（人事管理）: 従業員マスタ、組織管理、人事イベント
  • OM（組織管理）: 組織図、職位、職務定義
  • PY（給与計算）: 月次給与計算、賞与計算、年末調整
  • TM（タイムマネジメント）: 勤怠管理、休暇管理
  • PA-BN（福利厚生）: 保険、年金、福利厚生プラン

3.8 モジュール間統合の実例

SAPの真価は、各モジュールが独立して動作するのではなく、リアルタイムで統合されている点にあります。

実例: 受注から入金までの統合フロー
シナリオ

顧客Aから製品Xの注文を受け、出荷して代金を回収するまでの流れ

sequenceDiagram participant SD as SD モジュール participant MM as MM モジュール participant PP as PP モジュール participant FI as FI モジュール participant CO as CO モジュール SD->>MM: 受注登録 - 在庫引当 MM-->>SD: 在庫確認OK SD->>MM: 出荷処理 MM->>FI: 売上伝票自動転記 SD->>FI: 請求書発行 FI->>FI: 売掛金計上 FI->>CO: 収益データ連携 CO->>CO: 製品別利益分析
統合のメリット
  • データの二重入力が不要
  • リアルタイムで在庫・財務状況が更新
  • 部門を跨いだ情報共有が即座に可能
  • 経営レポートが自動生成される
理解度確認クイズ
  1. FI（財務会計）とCO（管理会計）の違いを説明してください。
  2. MM モジュールの「3-way matching」とは何ですか？
  3. SD モジュールにおけるO2Cプロセスの各ステップを順番に挙げてください。
  4. PP モジュールのMRPが行う主な処理を説明してください。
  5. なぜSAPのモジュール統合がビジネスに価値をもたらすのか、具体例を挙げて説明してください。
解答のヒント

1. FIは外部報告用（財務諸表）、COは内部管理用（部門別損益、製品原価）

2. 購買発注、入庫伝票、請求書の3つを照合する仕組み

3. 見積 → 受注 → 出荷 → 請求 → 入金

4. 生産計画に基づき、必要な資材の所要量を計算し、自動発注提案を行う

5. 例: 受注と同時に在庫が引き当てられ、出荷時に自動で売上計上、リアルタイムで利益分析可能
