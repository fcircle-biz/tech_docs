12.1 業務自動化ツール開発の流れ

12.2 実践例1：請求書自動作成ツール

顧客データから請求書を自動生成するツールを作成します。ユーザーフォームで必要な情報を入力し、テンプレートに自動入力してPDF出力まで行います。

ツールの仕様

  • 入力: 顧客名、請求日、商品リスト（品名・数量・単価）
  • 処理: 請求書テンプレートに自動入力、小計・消費税・合計を自動計算
  • 出力: 請求書シートの作成、PDF形式で保存

実習 12-1: 請求書自動作成マクロを作る

請求書テンプレートにデータを自動入力し、計算・PDF出力するマクロを実装します。

コード例

請求書作成マクロ 
Sub 請求書作成()
    ' シート設定
    Dim wsTemplate As Worksheet
    Dim wsInvoice As Worksheet

    ' テンプレートシートをコピー
    Sheets("請求書テンプレート").Copy After:=Sheets(Sheets.Count)
    Set wsInvoice = ActiveSheet
    wsInvoice.Name = "請求書_" & Format(Date, "yyyymmdd")

    ' 顧客情報入力
    wsInvoice.Range("B5").Value = "株式会社サンプル様"
    wsInvoice.Range("B8").Value = Format(Date, "yyyy年mm月dd日")

    ' 商品データ入力
    wsInvoice.Range("B12").Value = "商品A"
    wsInvoice.Range("C12").Value = 10
    wsInvoice.Range("D12").Value = 5000
    ' 小計自動計算
    wsInvoice.Range("E12").Formula = "=C12*D12"

    ' 合計計算
    Dim subtotal As Long
    subtotal = wsInvoice.Range("E12").Value
    wsInvoice.Range("E20").Value = subtotal
    wsInvoice.Range("E21").Value = subtotal * 0.1
    wsInvoice.Range("E22").Value = subtotal * 1.1

    MsgBox "請求書を作成しました！", vbInformation
End Sub

実務では、このコードをユーザーフォームと組み合わせ、顧客リストから選択できるようにすると便利です。

PDF出力の注意点

PDF出力にはExportAsFixedFormatメソッドを使用します。ファイル名や保存先パスは変数で管理し、ユーザーが選択できるようにすると良いでしょう。

12.3 実践例2：売上集計レポート生成ツール

日々の売上データから月次レポートを自動生成するツールを作成します。データ集計、ピボットテーブル作成、グラフ生成を自動化します。

ツールの仕様

  • 入力: 売上データシート（日付・商品・金額・担当者）
  • 処理: 商品別・担当者別の集計、月次推移の計算、ピボットテーブル作成
  • 出力: 集計レポートシート、売上推移グラフ

実習 12-2: 売上集計マクロを作る

売上データを商品別に集計し、グラフを自動生成するマクロを実装します。

コード例

売上集計マクロ 
Sub 売上集計レポート作成()
    ' シート設定
    Dim wsData As Worksheet
    Dim wsReport As Worksheet
    Set wsData = Sheets("売上データ")

    ' レポートシート作成
    Sheets.Add After:=Sheets(Sheets.Count)
    Set wsReport = ActiveSheet
    wsReport.Name = "売上レポート"

    ' 見出し作成
    wsReport.Range("A1").Value = "商品名"
    wsReport.Range("B1").Value = "売上合計"

    ' 商品別集計（辞書オブジェクトを使った方法）
    Dim lastRow As Long
    lastRow = wsData.Cells(Rows.Count, 1).End(xlUp).Row

    ' 簡易集計（商品A～Cを例に）
    Dim i As Long, row As Long
    row = 2

    wsReport.Range("A2").Value = "商品A"
    wsReport.Range("B2").Formula = "=SUMIF(売上データ!B:B,A2,売上データ!C:C)"

    MsgBox "売上レポートを作成しました！", vbInformation
End Sub

より高度な集計には、Dictionary オブジェクトやピボットテーブルのVBA操作を活用できます。

グラフ自動生成の追加

集計結果をグラフ化することで、視覚的に分かりやすいレポートになります。

グラフ作成コード例 
Sub グラフ作成()
    ' グラフオブジェクトを追加
    Dim cht As ChartObject
    Set cht = ActiveSheet.ChartObjects.Add(300, 50, 400, 250)

    ' グラフの種類を縦棒グラフに設定
    cht.Chart.ChartType = xlColumnClustered

    ' データ範囲を設定
    cht.Chart.SetSourceData Range("A1:B10")

    ' グラフタイトル
    cht.Chart.HasTitle = True
    cht.Chart.ChartTitle.Text = "商品別売上"
End Sub

実習 12-3: 在庫アラート機能を作る

在庫が指定数以下になった商品を検出し、警告を表示するマクロを実装します。

コード例

在庫アラートマクロ 
Sub 在庫アラート()
    ' 在庫シートを取得
    Dim ws As Worksheet
    Set ws = Sheets("在庫リスト")

    ' 警告メッセージ用変数
    Dim alertMsg As String
    Dim alertCount As Integer
    alertCount = 0

    ' 最終行を取得
    Dim lastRow As Long
    lastRow = ws.Cells(Rows.Count, 1).End(xlUp).Row

    ' 在庫チェック（閾値: 10個）
    Dim i As Long
    For i = 2 To lastRow
        If ws.Cells(i, 3).Value <= 10 Then
            alertMsg = alertMsg & ws.Cells(i, 1).Value & _
                       "（在庫: " & ws.Cells(i, 3).Value & "個）" & vbCrLf
            alertCount = alertCount + 1
            ' セルに警告色を設定
            ws.Cells(i, 3).Interior.Color = RGB(255, 200, 200)
        End If
    Next i

    ' 警告表示
    If alertCount > 0 Then
        MsgBox "【在庫警告】" & vbCrLf & vbCrLf & alertMsg, vbExclamation
    Else
        MsgBox "すべての商品の在庫は十分です。", vbInformation
    End If
End Sub

このマクロをワークブックを開いたときに自動実行するには、Workbook_Openイベントを利用します。

12.5 ツール完成後の運用とメンテナンス

自動化ツールは作って終わりではありません。実際に業務で使いながら改善し、安定した運用を続けることが重要です。

ドキュメント化

使い方マニュアル、仕様書、コメントを充実させ、他の人でも使える・メンテナンスできる状態にする

バージョン管理

定期的にバックアップを取り、更新履歴を記録。大きな変更を加える前は必ず別バージョンとして保存

フィードバック収集

実際に使うユーザーから意見を聞き、使いにくい点や追加してほしい機能を把握して改善

定期見直し

業務の変化に応じて機能を見直し、不要な処理は削除、新しい要件には柔軟に対応

長期運用のポイント

  • エラーログの記録: エラーが発生したらログファイルに記録し、原因分析に活用
  • テストデータの準備: 更新後は必ずテストデータで動作確認してから本番利用
  • ユーザー教育: 定期的に使い方講習を行い、正しい運用方法を周知
  • セキュリティ対策: マクロのパスワード保護、重要データの暗号化も検討
