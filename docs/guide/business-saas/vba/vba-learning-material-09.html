エラー処理で安全なプログラムを作る

On Error文によるエラー処理の実装、エラーの種類と対処方法、ユーザーフレンドリーなエラーメッセージの表示方法を習得します。

この章で学ぶこと

  • エラーとは何か、エラー処理の重要性を理解する
  • On Error文を使ったエラー処理の基本を習得する
  • Errオブジェクトでエラー情報を取得する方法を学ぶ
  • ユーザーフレンドリーなエラーメッセージを実装できる

9.1 エラーとは何か？なぜエラー処理が必要なのか

エラー（実行時エラー）とは、プログラムを実行している途中で予期しない問題が発生し、処理を続けられなくなった状態のことです。エラーが発生すると、VBAは処理を中断し、エラーメッセージを表示します。

エラー処理をしないとどうなる？

  • 処理が突然停止: ユーザーに不親切なエラーメッセージが表示され、マクロが強制終了
  • データ破損のリスク: 途中まで処理したデータが中途半端な状態で残る
  • 原因不明: どこで何が起きたのか分からず、デバッグが困難
  • 信頼性低下: ツールの品質が低く、業務で安心して使えない

よくあるエラーの例

ファイルが見つからない

指定したExcelファイルやCSVファイルが存在しない、またはパスが間違っている

データ型の不一致

数値を期待している変数に文字列が入っている、またはその逆

範囲外のアクセス

配列のサイズを超えた添字を指定した、存在しないシート名を参照した

ゼロ除算エラー

数値をゼロで割ろうとした（計算不能）

9.2 On Error文の基本

VBAでエラー処理を行うには、On Error文を使います。On Error文には主に3つの形式があります。

On Errorの3つの形式

On Error Resume Next

エラーが発生しても無視して次の行に進みます。

用途: エラーが起きても問題ない処理（例: シートの削除で、既に削除済みでもOK）

On Error GoTo ラベル

エラーが発生したら指定したラベルにジャンプし、エラー処理を実行します。

用途: 最も一般的。エラー情報を取得してユーザーに通知したい場合に使う

On Error GoTo 0

エラー処理を無効化し、デフォルトのVBAエラーハンドラに戻します。

用途: エラー処理を一時的に解除したい場合（デバッグ時など）

基本的なエラー処理の構造

On Error GoTo ラベル の基本構造 
Sub エラー処理の基本()
    ' エラー処理の開始（エラーが起きたらErrorHandlerへ）
    On Error GoTo ErrorHandler

    ' 通常の処理（ここでエラーが起きる可能性がある）
    ' 例: 存在しないファイルを開こうとする
    Workbooks.Open "C:\存在しないファイル.xlsx"

    ' 正常終了時はエラー処理をスキップして終了
    Exit Sub

ErrorHandler:
    ' エラーが発生した時にここが実行される
    MsgBox "エラーが発生しました: " & Err.Description
End Sub

9.3 Errオブジェクトでエラー情報を取得する

エラーが発生すると、VBAはErrオブジェクトにエラー情報を格納します。このオブジェクトのプロパティを使って、エラーの詳細を取得できます。

Errオブジェクトの主要プロパティ

プロパティ 説明 例
Err.Number エラー番号（エラーの種類を識別） 53 (ファイルが見つかりません)
Err.Description エラーの説明文 "ファイルが見つかりません。"
Err.Source エラーが発生したオブジェクトやプロジェクト名 "VBAProject"

Errオブジェクトの主要メソッド

Err.Clear

Errオブジェクトの内容をクリアし、エラー状態をリセットします。

Err.Raise

意図的にエラーを発生させます（独自のエラーをテストする場合などに使用）。

Errオブジェクトの使用例 
Sub エラー情報の取得例()
    On Error GoTo ErrorHandler

    ' エラーを起こす処理（例: ゼロ除算）
    Dim result As Double
    result = 10 / 0  ' ここでエラー発生

    Exit Sub

ErrorHandler:
    ' エラー情報を詳しく表示
    MsgBox "エラー番号: " & Err.Number & vbCrLf & _
           "エラー内容: " & Err.Description & vbCrLf & _
           "発生元: " & Err.Source
End Sub

9.4 エラー処理のパターンと実践例

実務でよく使われるエラー処理のパターンを学びましょう。

パターン1: ファイル存在チェック

ファイルを開く際のエラー処理 
Sub ファイルを開く処理()
    On Error GoTo ErrorHandler

    Dim filePath As String
    filePath = "C:\MyData\売上データ.xlsx"

    ' ファイルを開く（存在しない場合エラー）
    Workbooks.Open filePath
    MsgBox "ファイルを開きました"

    Exit Sub

ErrorHandler:
    If Err.Number = 53 Then  ' ファイルが見つからない
        MsgBox "ファイルが見つかりません: " & filePath
    Else
        MsgBox "予期しないエラー: " & Err.Description
    End If
End Sub

パターン2: データ型チェック

数値変換時のエラー処理 
Sub 数値変換処理()
    On Error GoTo ErrorHandler

    Dim userInput As String
    Dim numValue As Double

    userInput = Range("A1").Value
    numValue = CDbl(userInput)  ' 文字列を数値に変換

    MsgBox "変換成功: " & numValue
    Exit Sub

ErrorHandler:
    If Err.Number = 13 Then  ' 型が一致しません
        MsgBox "数値を入力してください（入力値: " & userInput & "）"
    Else
        MsgBox "エラー: " & Err.Description
    End If
End Sub

パターン3: 複数エラー対応

シート操作での複数エラー対応 
Sub シート追加処理()
    On Error GoTo ErrorHandler

    Dim sheetName As String
    sheetName = "集計結果"

    ' シートを追加（既に存在する場合はエラー）
    Worksheets.Add
    ActiveSheet.Name = sheetName

    MsgBox "シート「" & sheetName & "」を作成しました"
    Exit Sub

ErrorHandler:
    Select Case Err.Number
        Case 9  ' インデックスが有効範囲にありません
            MsgBox "シート名が無効です"
        Case 1004  ' 同じ名前のシートが既に存在
            MsgBox "シート「" & sheetName & "」は既に存在します"
        Case Else
            MsgBox "エラー: " & Err.Description
    End Select
End Sub

9.5 ユーザーフレンドリーなエラーメッセージ

エラーが発生した時、ユーザーが理解しやすく、次に何をすればよいか分かるメッセージを表示することが重要です。

悪いエラーメッセージ vs 良いエラーメッセージ

悪い例

"エラー番号: 53"

問題点: エラー番号だけ表示されても、ユーザーは何が起きたのか、どうすればいいのか分からない

良い例

"ファイルが見つかりませんでした。
パスを確認してください: C:\売上.xlsx"

良い点: 何が起きたか、どのファイルで問題が起きたか、何をすればいいかが明確

良いエラーメッセージのポイント

  • 何が起きたかを分かりやすく説明する
  • どこで問題が起きたか（ファイル名、シート名など）を示す
  • 次に何をすればいいかを提案する
  • 専門用語やエラー番号だけでなく、日常的な言葉で説明する

実習 9-1: ユーザーフレンドリーなエラー処理を実装する

CSVファイルを読み込むマクロに、丁寧なエラー処理を実装してみましょう。

実習コード 
Sub CSVファイル読込()
    On Error GoTo ErrorHandler

    Dim filePath As String
    filePath = "C:\MyData\顧客リスト.csv"

    ' CSVファイルを開く
    Workbooks.Open filePath
    MsgBox "ファイルを正常に読み込みました", vbInformation

    Exit Sub

ErrorHandler:
    ' エラー内容に応じて分かりやすいメッセージを表示
    Select Case Err.Number
        Case 53  ' ファイルが見つかりません
            MsgBox "ファイルが見つかりませんでした。" & vbCrLf & _
                   "以下のパスを確認してください:" & vbCrLf & _
                   filePath, vbExclamation, "ファイル読込エラー"

        Case 1004  ' ファイルを開けません
            MsgBox "ファイルを開くことができませんでした。" & vbCrLf & _
                   "他のアプリケーションで開いていないか確認してください。", _
                   vbExclamation, "ファイル読込エラー"

        Case Else
            MsgBox "予期しないエラーが発生しました。" & vbCrLf & _
                   "システム管理者に連絡してください。" & vbCrLf & vbCrLf & _
                   "エラー詳細: " & Err.Description, _
                   vbCritical, "エラー"
    End Select
End Sub

このコードでは、エラーの種類に応じて具体的で分かりやすいメッセージを表示しています。ユーザーが次に何をすべきか明確です。

トラブルシューティング

Q1. エラー処理が動作しない

原因: On Error GoTo文より前でエラーが発生している
対策: On Error文は必ずSubの最初に配置しましょう

Q2. エラーメッセージが2回表示される

原因: ErrorHandlerラベルの後にExit Subがない
対策: ErrorHandlerラベルの直前に必ずExit Subを記述しましょう

Q3. エラー番号が分からない

対策: エラー発生時にイミディエイトウィンドウでDebug.Print Err.Numberを実行するか、MsgBox Err.Numberで確認しましょう

理解度確認クイズ

  1. Q1. エラー処理を実装しないとどのような問題が起きますか？3つ挙げてください。

    回答を表示

    ① 処理が突然停止し、ユーザーに不親切なエラーメッセージが表示される
    ② 途中まで処理したデータが中途半端な状態で残り、データ破損のリスクがある
    ③ エラーの原因が不明で、デバッグが困難になる

  2. Q2. On Error Resume NextとOn Error GoTo ラベルの違いは何ですか？

    回答を表示

    On Error Resume Next: エラーが発生しても無視して次の行に進む
    On Error GoTo ラベル: エラーが発生したら指定したラベルにジャンプし、エラー処理を実行する（より詳細な制御が可能）

  3. Q3. ErrオブジェクトのプロパティErr.NumberとErr.Descriptionは何を表していますか？

    回答を表示

    Err.Number: エラー番号（エラーの種類を識別するための数値）
    Err.Description: エラーの説明文（エラーの内容を示すテキスト）

  4. Q4. ユーザーフレンドリーなエラーメッセージに必要な要素を3つ挙げてください。

    回答を表示

    ① 何が起きたかを分かりやすく説明する
    ② どこで問題が起きたか（ファイル名、シート名など）を示す
    ③ 次に何をすればいいかを提案する
