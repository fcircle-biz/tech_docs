第3章

変数とデータ型の基礎

プログラミングの基本となる変数の概念、データ型の種類と使い分け、変数の宣言と代入方法を実例を通じて習得します。

この章で学ぶこと

  • 変数とは何か、なぜ必要なのかを理解する
  • 変数の宣言と代入の基本文法を習得する
  • VBAの主要データ型（String、Integer、Long、Doubleなど）を理解する
  • 用途に応じた適切なデータ型の選び方を身につける

3.1 変数とは何か？

変数とは、データを一時的に保存しておく「箱」のようなものです。この箱には名前（変数名）をつけて、プログラムの中で何度も使うことができます。

日常生活での例え

変数は、引き出しやラベル付きの箱に似ています。例えば：

  • 「商品名」という箱に「ノートPC」というデータを入れる
  • 「価格」という箱に「98000」という数値を入れる
  • 「在庫数」という箱に「15」という数値を入れる

このように、箱（変数）に名前をつけて、中身（データ）を出し入れするイメージです。

変数のイメージ図

変数のメリット

再利用できる

一度保存したデータを何度でも使い回せます

値を変更できる

プログラムの途中で箱の中身を入れ替えられます

わかりやすい

意味のある名前をつけることでコードが読みやすくなります

計算に使える

数値を保存して計算処理に活用できます

3.2 変数の宣言と代入

変数を使うには、まず宣言（箱を用意）し、次に代入（箱に値を入れる）します。

変数の宣言

Dimキーワードを使って変数を宣言します。

変数宣言の基本文法 
Dim 変数名 As データ型

各要素の説明

  • Dim: 変数を宣言するキーワード（「Dimension＝次元・領域」の略）
  • 変数名: 自分で決める変数の名前（英数字で意味がわかる名前にする）
  • As: データ型を指定するキーワード
  • データ型: 変数に入れるデータの種類（次のセクションで詳しく説明）

変数への代入

=演算子を使って、変数に値を代入します。

変数の宣言と代入の例 
Sub 変数の基本()
    ' 変数を宣言
    Dim shouhinName As String
    Dim kakaku As Long

    ' 変数に値を代入
    shouhinName = "ノートPC"
    kakaku = 98000

    ' セルに出力
    Range("A1").Value = shouhinName
    Range("B1").Value = kakaku
End Sub

ヒント

変数名は英数字で、意味が分かる名前にしましょう。 日本語のローマ字表記でもOKです（例: shouhinName, kakaku）。

3.3 VBAの主要データ型

データ型とは、変数に入れるデータの「種類」です。数値、文字列、日付など、用途に応じて適切なデータ型を選びます。

主要なデータ型一覧

データ型 用途 扱える値の範囲 例
String 文字列（テキスト） 約20億文字まで "こんにちは", "A001"
Integer 整数（小さい範囲） -32,768 ～ 32,767 100, -50
Long 整数（大きい範囲） 約-21億 ～ 21億 1000000, -123456
Double 小数点付き数値 非常に広い範囲 3.14, 98.5, -0.001
Boolean 真偽値（はい/いいえ） True または False True, False
Date 日付・時刻 1900/1/1 ～ 9999/12/31 #2025/12/7#
Variant どんな型でもOK 何でも格納可能 汎用的に使える
各データ型の使用例 
Sub データ型の例()
    Dim msg As String           ' 文字列型
    Dim age As Integer          ' 整数型（小）
    Dim population As Long      ' 整数型（大）
    Dim price As Double         ' 小数型
    Dim isFinished As Boolean   ' 真偽値型
    Dim today As Date           ' 日付型

    msg = "VBAを学習中"
    age = 25
    population = 1000000
    price = 1980.5
    isFinished = True
    today = #12/7/2025#
End Sub

3.4 データ型の選び方

適切なデータ型を選ぶことで、メモリを効率的に使い、エラーを防げます。以下の基準で選びましょう。

文字や文章を扱う場合

→ String を使用

例: 名前、住所、商品名、コメント

整数（小数点なし）を扱う場合

小さい数（-32,768 ～ 32,767）→ Integer
大きい数（それ以上）→ Long

例: 個数、行番号、ID番号、売上金額

小数点を含む数値を扱う場合

→ Double を使用

例: 消費税率、身長・体重、平均値

はい/いいえ、真/偽を扱う場合

→ Boolean を使用

例: チェック済みフラグ、処理完了フラグ

日付や時刻を扱う場合

→ Date を使用

例: 注文日、締切日、現在時刻

重要ポイント

迷ったらVariant型でもOKですが、明確な型を指定した方が、 後からコードを読むときにわかりやすく、バグも防げます。整数なら基本的にLong型を使うのが無難です。

3.5 実習：変数を使ってみよう

実習 3-1: 商品情報を変数で管理する

商品名、単価、個数を変数に格納し、合計金額を計算してExcelに出力します。

手順

  1. 1 VBEを起動（Alt + F11）し、標準モジュールを挿入
  2. 2 以下のコードを入力
実習コード 
Sub 商品情報計算()
    ' 変数の宣言
    Dim shouhinName As String
    Dim tanka As Long
    Dim kosu As Integer
    Dim goukei As Long

    ' 値の代入
    shouhinName = "ワイヤレスマウス"
    tanka = 1980
    kosu = 5

    ' 合計金額を計算
    goukei = tanka * kosu

    ' Excelに出力
    Range("A1").Value = "商品名"
    Range("B1").Value = shouhinName
    Range("A2").Value = "単価"
    Range("B2").Value = tanka
    Range("A3").Value = "個数"
    Range("B3").Value = kosu
    Range("A4").Value = "合計金額"
    Range("B4").Value = goukei
End Sub
  1. 3 F5キーでマクロを実行
  2. 4 Excelシートに計算結果が表示されることを確認

A1～B4に商品情報と合計金額（9900円）が表示されれば成功です！

3.6 よくあるエラーと対処法

トラブルシューティング

型が一致しません（Type Mismatch）

原因: 数値型の変数に文字列を入れようとした、またはその逆

例: Dim num As Integer → num = "こんにちは"

解決方法: 変数の型と代入する値の型を一致させる

変数が定義されていません

原因: 宣言していない変数を使おうとした、または変数名のスペルミス

例: Dim kakakuと宣言したのに、price = 1000と書いた

解決方法: 変数名のスペルを確認し、必ずDimで宣言してから使う

オーバーフロー（Overflow）

原因: Integerに32,767を超える値を代入しようとした

例: Dim num As Integer → num = 50000

解決方法: 大きな整数にはLong型を使う

エラーを防ぐコツ

  • 変数は使う前に必ずDimで宣言する
  • 変数名は意味が分かる名前にする（aではなくageなど）
  • 整数には基本的にLong型を使う
  • 文字列は""（ダブルクォート）で囲む

理解度確認クイズ

  1. Q1. 変数とは何ですか？簡潔に説明してください。

    回答を表示

    変数とは、データを一時的に保存しておく「箱」のようなものです。名前をつけて、プログラムの中で何度も使うことができます。

  2. Q2. 変数を宣言するキーワードは何ですか？

    回答を表示

    Dimキーワードを使います。例: Dim 変数名 As データ型

  3. Q3. 文字列を扱う場合、どのデータ型を使いますか？

    回答を表示

    String型を使います。例: Dim name As String

  4. Q4. 大きな整数（例: 1000000）を扱う場合、IntegerとLongのどちらが適切ですか？

    回答を表示

    Long型が適切です。Integer型は-32,768～32,767の範囲しか扱えないため、大きな数値にはLong型を使います。

  5. Q5. 次のコードでエラーが起きます。原因は何でしょうか？
    Dim age As Integer
    age = "25歳"

    回答を表示

    Integer型は数値専用なのに、文字列（"25歳"）を代入しようとしているため「型が一致しません」エラーが発生します。 数値を入れるならage = 25、文字列ならDim age As Stringと宣言すべきです。
