プロシージャ（Sub/Function）で再利用可能なコードを作る

SubプロシージャとFunctionプロシージャの違いと使い分け、引数と戻り値の活用、モジュール化によるコードの整理方法を学びます。

この章で学ぶこと

  • SubプロシージャとFunctionプロシージャの違いを理解する
  • 引数（パラメータ）を使って、柔軟なプロシージャを作成できる
  • Functionで戻り値を返し、計算結果を取得できる
  • モジュール化によるコードの整理と再利用の重要性を学ぶ

8.1 プロシージャとは？なぜ必要なのか

プロシージャとは、一連の処理をまとめて名前を付けた、再利用可能なコードのブロックです。今まで作成してきた「マクロ」も、実はSubプロシージャの一種です。

プロシージャを使う理由

  • コードの再利用: 同じ処理を何度も書く必要がなくなる
  • 保守性の向上: 修正が一箇所で済み、バグが減る
  • 可読性の向上: コードの意図が明確になり、理解しやすくなる
  • 分業が可能: 機能ごとに分割して開発できる

8.2 Subプロシージャの基本

Subプロシージャは、一連の処理を実行しますが、戻り値は返しません。今まで作成してきたマクロはすべてSubプロシージャです。

Subプロシージャの構文

Subプロシージャの基本構文 
Sub プロシージャ名()
    ' ここに処理を記述
End Sub

実例：挨拶メッセージを表示するSubプロシージャ

シンプルなSubプロシージャ 
Sub 挨拶表示()
    ' メッセージボックスに挨拶を表示
    MsgBox "こんにちは！VBAプロシージャの学習中です。"
End Sub

Subプロシージャの特徴

  • 戻り値なし: 計算結果を返さず、処理だけを実行
  • マクロとして実行可能: F5キーやマクロ一覧から直接実行できる
  • 他のプロシージャから呼び出せる: Callステートメントで呼び出し

8.3 Functionプロシージャの基本

Functionプロシージャは、処理を実行して結果を戻り値として返すプロシージャです。計算や判定結果を受け取りたい場合に使用します。

Functionプロシージャの構文

Functionプロシージャの基本構文 
Function 関数名() As データ型
    ' ここに処理を記述
    関数名 = 戻り値
End Function

実例：税込価格を計算するFunction

税込価格計算Function 
Function 税込価格(税抜き価格 As Double) As Double
    ' 税率10%で税込価格を計算
    税込価格 = 税抜き価格 * 1.1
End Function

' 使用例
Sub 価格計算テスト()
    Dim 結果 As Double
    結果 = 税込価格(1000)
    MsgBox "税込価格: " & 結果 & "円"  ' 1100円と表示
End Sub

SubとFunctionの使い分け

種類 戻り値 用途
Sub なし 処理の実行、データの更新、画面表示
Function あり 計算結果の取得、条件判定の結果取得

8.4 引数（パラメータ）を使いこなす

引数は、プロシージャに渡すデータです。引数を使うことで、同じプロシージャを異なるデータで繰り返し使うことができます。

引数の基本

引数を持つSubプロシージャ 
Sub 挨拶(名前 As String)
    ' 引数で受け取った名前を使って挨拶
    MsgBox "こんにちは、" & 名前 & "さん！"
End Sub

' 使用例
Sub テスト()
    Call 挨拶("田中")   ' "こんにちは、田中さん！"と表示
    Call 挨拶("佐藤")   ' "こんにちは、佐藤さん！"と表示
End Sub

複数の引数を持つプロシージャ

複数の引数を使った実例 
Function 割引価格(定価 As Double, 割引率 As Double) As Double
    ' 定価と割引率から割引後の価格を計算
    割引価格 = 定価 * (1 - 割引率)
End Function

' 使用例
Sub 割引計算テスト()
    Dim 結果 As Double
    結果 = 割引価格(5000, 0.2)  ' 5000円の20%割引
    MsgBox "割引後価格: " & 結果 & "円"  ' 4000円と表示
End Sub

引数のポイント

  • 型を明示: 引数にはデータ型を指定する（String, Integer, Doubleなど）
  • カンマ区切り: 複数の引数はカンマで区切る
  • 順序を守る: 呼び出し時は、引数の順序を正しく指定する

8.5 Functionの戻り値を活用する

Functionプロシージャの戻り値は、計算結果や判定結果を呼び出し元に返すための重要な仕組みです。

戻り値の設定方法

戻り値の例 
Function 合計(数値1 As Integer, 数値2 As Integer) As Integer
    ' 2つの数値の合計を返す
    合計 = 数値1 + 数値2
End Function

' 使用例
Sub 合計テスト()
    Dim 答え As Integer
    答え = 合計(50, 30)
    MsgBox "合計: " & 答え  ' 80と表示
End Sub

Boolean型の戻り値で条件判定

判定結果を返すFunction 
Function 合格判定(点数 As Integer) As Boolean
    ' 60点以上なら合格
    If 点数 >= 60 Then
        合格判定 = True
    Else
        合格判定 = False
    End If
End Function

' 使用例
Sub 判定テスト()
    If 合格判定(75) Then
        MsgBox "合格です！"
    Else
        MsgBox "不合格です"
    End If
End Sub

8.6 モジュール化でコードを整理する

モジュール化とは、プログラムを機能ごとに小さなプロシージャに分割することです。これにより、コードが読みやすく、保守しやすくなります。

再利用性

同じ処理を複数の場所で使える。一度作れば何度でも呼び出せる

保守性

修正が一箇所で済む。バグ修正や機能改善が簡単

可読性

プロシージャ名で処理内容が分かる。コードの意図が明確

テスト容易性

小さな単位でテストできる。問題箇所の特定が簡単

モジュール化の実例

モジュール化されたコード例 
' メインの処理
Sub 売上集計()
    Call データ読み込み
    Call 集計計算
    Call レポート出力
End Sub

' データ読み込み処理
Sub データ読み込み()
    ' CSVファイルからデータを読み込む
End Sub

' 集計計算処理
Sub 集計計算()
    ' 売上を集計する
End Sub

' レポート出力処理
Sub レポート出力()
    ' 結果をシートに出力する
End Sub

8.7 プロシージャの呼び出し方

作成したプロシージャは、他のプロシージャから呼び出して使用します。

Subプロシージャの呼び出し

Subプロシージャの呼び出し方法 
' 方法1: Callステートメントを使う（推奨）
Call プロシージャ名(引数1, 引数2)

' 方法2: プロシージャ名のみ（Callなし）
プロシージャ名 引数1, 引数2

' 実例
Sub メイン()
    Call 挨拶("山田")
    ' または
    挨拶 "山田"
End Sub

Functionプロシージャの呼び出し

Functionプロシージャの呼び出し方法 
' 戻り値を変数に格納
変数 = 関数名(引数1, 引数2)

' 実例
Sub メイン()
    Dim 結果 As Double
    結果 = 税込価格(1000)  ' Functionの戻り値を受け取る
    MsgBox 結果
End Sub

ヒント

Callステートメントを使うと、「これはプロシージャの呼び出しだ」ということが一目で分かり、コードが読みやすくなります。

実習 8-1: BMI計算Functionを作成する

身長と体重からBMIを計算し、判定結果を返すプロシージャを作成します。

手順

  1. 1 標準モジュールに以下のFunctionを作成
BMI計算Function 
Function BMI計算(身長 As Double, 体重 As Double) As Double
    ' BMI = 体重kg ÷ （身長m × 身長m）
    BMI計算 = 体重 / ((身長 / 100) ^ 2)
End Function

Function BMI判定(BMI As Double) As String
    ' BMI値から判定結果を返す
    If BMI < 18.5 Then
        BMI判定 = "やせ"
    ElseIf BMI < 25 Then
        BMI判定 = "標準"
    Else
        BMI判定 = "肥満"
    End If
End Function

Sub BMIテスト()
    Dim 身長 As Double, 体重 As Double
    Dim BMI値 As Double, 判定結果 As String

    身長 = 170  ' cm
    体重 = 65   ' kg

    BMI値 = BMI計算(身長, 体重)
    判定結果 = BMI判定(BMI値)

    MsgBox "BMI: " & Format(BMI値, "0.0") & vbCrLf & _
           "判定: " & 判定結果
End Sub
  1. 2 BMIテストSubプロシージャを実行して動作確認
  2. 3 異なる身長・体重で何度かテストしてみる

BMI計算と判定を別々のFunctionに分けることで、それぞれ独立してテストでき、再利用もしやすくなります！

理解度確認クイズ

  1. Q1. SubプロシージャとFunctionプロシージャの最大の違いは何ですか？

    回答を表示

    最大の違いは戻り値の有無です。Subプロシージャは戻り値を返しませんが、Functionプロシージャは処理結果を戻り値として返すことができます。

  2. Q2. 引数を使うメリットを2つ挙げてください。

    回答を表示

    ① 同じプロシージャを異なるデータで繰り返し使える（再利用性）
    ② プロシージャを汎用的に作成でき、柔軟性が高まる

  3. Q3. モジュール化の主なメリットを3つ挙げてください。

    回答を表示

    ① 再利用性：同じ処理を複数箇所で使える
    ② 保守性：修正が一箇所で済む
    ③ 可読性：プロシージャ名で処理内容が分かり、コードが理解しやすい

  4. Q4. Functionの戻り値を設定するには、どのように記述しますか？

    回答を表示

    関数名 = 戻り値の形式で記述します。例: 合計 = 数値1 + 数値2
