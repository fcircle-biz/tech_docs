7.1 配列とは？なぜ必要なのか

これまで学んできた変数は、1つの値だけを格納するものでした。しかし実務では、複数の関連するデータをまとめて扱いたい場面が頻繁に発生します。ここで登場するのが配列（Array）です。

配列とは

配列は、同じ種類のデータを連続して保存できる入れ物です。1つの変数名で複数の値を管理でき、インデックス（番号）で各要素にアクセスできます。

7.2 配列の宣言と基本操作

配列には静的配列（要素数が固定）と動的配列（要素数を変更可能）の2種類があります。まずは基本的な静的配列から見ていきましょう。

静的配列の宣言

静的配列の宣言例 
Sub 配列の基本()
    ' 5つの要素を持つ配列を宣言（インデックス0～4）
    Dim scores(4) As Integer

    ' 各要素に値を代入
    scores(0) = 85
    scores(1) = 92
    scores(2) = 78
    scores(3) = 88
    scores(4) = 95

    ' 配列の値を表示
    MsgBox "1人目の点数: " & scores(0)  ' 85
    MsgBox "3人目の点数: " & scores(2)  ' 78
End Sub

重要ポイント

  • インデックスは0から始まる: scores(4)は5つ目の要素ではなく、0～4の5個の要素
  • 要素数は固定: 宣言後にサイズを変更できない
  • データ型を指定: As Integerなど、配列全体のデータ型を統一

配列の要素数を調べる

UBound関数とLBound関数 
Sub 配列のサイズ確認()
    Dim names(2) As String
    names(0) = "田中"
    names(1) = "佐藤"
    names(2) = "鈴木"

    ' 配列の下限と上限を取得
    MsgBox "下限: " & LBound(names)  ' 0
    MsgBox "上限: " & UBound(names)  ' 2

    ' 要素数を計算
    Dim count As Integer
    count = UBound(names) - LBound(names) + 1
    MsgBox "要素数: " & count  ' 3
End Sub

関数の説明

  • LBound(配列): 配列の最小インデックスを返す（通常は0）
  • UBound(配列): 配列の最大インデックスを返す

ループで配列を処理する

For文で配列を処理 
Sub 配列をループ処理()
    Dim scores(4) As Integer
    scores(0) = 85: scores(1) = 92: scores(2) = 78
    scores(3) = 88: scores(4) = 95

    ' 全要素を表示
    Dim i As Integer
    For i = 0 To UBound(scores)
        Debug.Print "受験者" & (i + 1) & "の点数: " & scores(i)
    Next i

    ' 合計と平均を計算
    Dim total As Integer
    total = 0
    For i = 0 To UBound(scores)
        total = total + scores(i)
    Next i
    MsgBox "平均点: " & (total / (UBound(scores) + 1))
End Sub

7.3 動的配列の活用

動的配列は、実行時に要素数を変更できる柔軟な配列です。データ量が事前に分からない場合や、処理中に配列サイズを拡張したい場合に便利です。

動的配列の宣言とReDim

動的配列の基本 
Sub 動的配列の基本()
    ' サイズを指定せずに宣言
    Dim data() As String

    ' 実行時にサイズを設定（インデックス0～2の3要素）
    ReDim data(2)
    data(0) = "東京"
    data(1) = "大阪"
    data(2) = "名古屋"

    ' サイズを変更（0～4の5要素に拡張）
    ReDim data(4)
    data(3) = "福岡"
    data(4) = "札幌"

    ' 注意: ReDimすると既存データが消える
    MsgBox data(0)  ' 空欄になる！
End Sub

つまずきポイント

ReDimでサイズを変更すると、既存のデータはすべて消えます。データを保持したい場合はReDim Preserveを使います。

ReDim Preserveでデータを保持

ReDim Preserveの使用例 
Sub データを保持して拡張()
    Dim cities() As String
    ReDim cities(2)
    cities(0) = "東京"
    cities(1) = "大阪"
    cities(2) = "名古屋"

    ' Preserveキーワードで既存データを保持しながら拡張
    ReDim Preserve cities(4)
    cities(3) = "福岡"
    cities(4) = "札幌"

    ' データが保持されている
    MsgBox cities(0)  ' 東京（データが残っている）
    MsgBox cities(4)  ' 札幌
End Sub

ヒント

ReDim Preserveは処理負荷が高いため、頻繁に実行すると遅くなります。あらかじめ余裕を持ったサイズで宣言するか、配列拡張の頻度を減らす工夫をしましょう。

7.4 2次元配列でExcelデータを扱う

2次元配列は、行と列を持つ表形式のデータを扱うのに最適です。Excelのセル範囲をそのまま配列に読み込むことで、大量データを高速処理できます。

2次元配列の宣言

2次元配列の基本 
Sub 二次元配列の基本()
    ' 3行2列の配列を宣言（行0～2、列0～1）
    Dim table(2, 1) As Variant

    ' データを代入
    table(0, 0) = "田中"
    table(0, 1) = 85
    table(1, 0) = "佐藤"
    table(1, 1) = 92
    table(2, 0) = "鈴木"
    table(2, 1) = 78

    ' データを表示
    MsgBox table(0, 0) & "さんの点数: " & table(0, 1)
End Sub

重要ポイント

2次元配列のアクセスは配列名(行, 列)の形式です。table(0, 1)は「0行目の1列目」を意味します。

Excelセル範囲を配列に読み込む

セル範囲を配列に一括で読み込むことで、処理速度が劇的に向上します。

セル範囲を配列に読み込む 
Sub セル範囲を配列で処理()
    ' A1:B5のセル範囲を配列に読み込む
    Dim data As Variant
    data = Range("A1:B5").Value

    ' 配列のインデックスは1から始まる（注意！）
    ' data(1, 1) = A1セルの値
    ' data(1, 2) = B1セルの値

    ' 配列の内容を変更
    Dim i As Long
    For i = 1 To UBound(data, 1)  ' 行数分ループ
        data(i, 2) = data(i, 2) * 1.1  ' B列の値を10%増加
    Next i

    ' 配列を一括でセル範囲に書き戻す
    Range("A1:B5").Value = data
End Sub

注意

Range.Valueで取得した配列はインデックスが1から始まります。通常の配列宣言（0始まり）と混同しないよう注意してください。

2次元配列のループ処理

2次元配列の全要素を処理 
Sub 二次元配列ループ()
    Dim data As Variant
    data = Range("A1:C10").Value  ' 10行3列のデータ

    Dim row As Long, col As Long

    ' 行ごとにループ
    For row = 1 To UBound(data, 1)
        ' 列ごとにループ
        For col = 1 To UBound(data, 2)
            Debug.Print "(" & row & ", " & col & ") = " & data(row, col)
        Next col
    Next row
End Sub

7.5 コレクションオブジェクトの基礎

コレクションは、配列と似ていますが、より柔軟なデータ管理ができるオブジェクトです。要素の追加・削除が容易で、キー（名前）でアクセスできる点が特徴です。

配列とコレクションの違い

項目 配列 コレクション
サイズ変更 ReDim Preserveが必要 Add/Removeで自由に変更
アクセス方法 インデックス（番号）のみ インデックスまたはキー（名前）
用途 大量データの高速処理 動的なデータ管理、キーによる検索

コレクションの基本操作

コレクションの作成と操作 
Sub コレクションの基本()
    ' コレクションを作成
    Dim students As New Collection

    ' 要素を追加（値, キー）
    students.Add "田中太郎", "student1"
    students.Add "佐藤花子", "student2"
    students.Add "鈴木次郎", "student3"

    ' インデックスでアクセス（1から始まる）
    MsgBox students(1)  ' 田中太郎

    ' キーでアクセス
    MsgBox students("student2")  ' 佐藤花子

    ' 要素数を取得
    MsgBox "学生数: " & students.Count  ' 3
End Sub

コレクションのループ処理

For Eachでコレクションを処理 
Sub コレクションのループ()
    Dim cities As New Collection
    cities.Add "東京"
    cities.Add "大阪"
    cities.Add "名古屋"

    ' For Eachで全要素を処理
    Dim city As Variant
    For Each city In cities
        Debug.Print city
    Next city

    ' 要素を削除（インデックスまたはキーで指定）
    cities.Remove 2  ' 2番目の要素（大阪）を削除

    MsgBox "削除後の要素数: " & cities.Count  ' 2
End Sub

ヒント

コレクションはオブジェクトやカスタムデータ型も格納できます。複雑なデータ構造を扱う際に便利です。

7.6 実習：配列を使った成績管理

実習 7-1: 配列を使った成績集計プログラム

A列に学生名、B列に点数が入力されているデータを配列に読み込み、平均点・最高点・最低点を計算するプログラムを作成します。

前提条件

  • セルA1に「氏名」、B1に「点数」のヘッダーがある
  • A2以降に学生名、B2以降に点数が入力されている

手順

  1. 1 サンプルデータを作成（A1:B6に氏名と点数）
  2. 2 VBEで新しいモジュールを作成
  3. 3 以下のコードを入力
成績集計プログラム 
Sub 成績集計()
    ' データ範囲を配列に読み込む
    Dim data As Variant
    data = Range("A2:B6").Value  ' A2～B6を読み込み

    ' 集計用変数
    Dim total As Long, maxScore As Long, minScore As Long
    total = 0
    maxScore = data(1, 2)  ' 最初の点数で初期化
    minScore = data(1, 2)

    ' ループで集計
    Dim i As Long
    For i = 1 To UBound(data, 1)
        Dim score As Long
        score = data(i, 2)  ' B列の点数

        total = total + score

        If score > maxScore Then maxScore = score
        If score < minScore Then minScore = score
    Next i

    ' 結果を表示
    Dim average As Double
    average = total / UBound(data, 1)

    MsgBox "平均点: " & average & vbCrLf & _
           "最高点: " & maxScore & vbCrLf & _
           "最低点: " & minScore
End Sub

配列を使うことで、セルに直接アクセスするより高速に処理できます！

発展課題：コレクションで成績管理

コレクションを使って、学生名をキーとして点数を管理し、特定の学生の点数を検索できるプログラムに改良してみましょう。

理解度確認クイズ

  1. Q1. 配列を使うメリットを2つ挙げてください。

    回答を表示

    ① 関連するデータをまとめて管理でき、変数宣言が簡潔になる
    ② ループ処理と組み合わせることで、大量データを効率的に処理できる
    （他にも、セル範囲を配列に読み込むと処理が高速化される、などの回答も正解）

  2. Q2. ReDimとReDim Preserveの違いは何ですか？

    回答を表示

    ReDimは配列のサイズを変更しますが、既存のデータは削除されます。ReDim Preserveは既存のデータを保持したままサイズを変更できます。

  3. Q3. Range("A1:B5").Valueで取得した配列のインデックスは何から始まりますか？

    回答を表示

    1から始まります。通常の配列宣言（0始まり）と異なるため注意が必要です。

  4. Q4. コレクションのAddメソッドで要素を追加する際、キーを指定するメリットは何ですか？

    回答を表示

    キーを指定すると、インデックス番号ではなく名前で要素にアクセスできます。これにより、データの意味が明確になり、コードの可読性が向上します。
