DAO/ADOによるデータベース操作

DAO（Data Access Objects）とADO（ActiveX Data Objects）を使ったレコードセット操作、データの検索・追加・更新・削除（CRUD操作）の実装方法を習得します。

この章で学ぶこと

  • DAOとADOの違いと使い分けを理解する
  • レコードセットオブジェクトを使ってデータを操作する方法を習得する
  • CRUD操作（検索・追加・更新・削除）を実装する
  • トランザクション処理の基本を理解する

7.1 DAOとADOの概要

Access VBAでデータベースを操作する方法として、DAO（Data Access Objects）とADO（ActiveX Data Objects）の2つのライブラリがあります。

どちらもデータベースのテーブルやクエリからデータを取得したり、追加・更新・削除する機能を提供しますが、特徴や得意分野が異なります。

DAO（Data Access Objects）

Accessデータベース専用のライブラリで、高速かつ軽量です。Accessファイル（.accdb）のみを扱う場合に最適です。

おすすめの使いどころ

  • • Accessデータベースのみ使用
  • • 高速なデータ操作が必要
  • • シンプルなCRUD操作

ADO（ActiveX Data Objects）

複数のデータベースに対応した汎用ライブラリ。Access以外（SQL Server、MySQL等）にも接続可能です。

おすすめの使いどころ

  • • 複数種類のDB接続が必要
  • • SQL Serverとの連携
  • • 将来的な拡張性を確保

この章での方針

本章では、Accessデータベースで最も一般的に使われるDAOを中心に学びます。ADOについても基本的な使い方を紹介します。

7.2 DAOの参照設定

DAOを使用する前に、参照設定を行う必要があります。これはDAOライブラリを有効化する作業です。

DAO参照設定の手順

  1. 1 Visual Basic Editor（VBE）を開く（Alt + F11）
  2. 2 メニューバーから「ツール」→「参照設定」をクリック
  3. 3 一覧から「Microsoft DAO 3.6 Object Library」を探してチェックを入れる
  4. 4 「OK」ボタンをクリックして完了

バージョン番号について

Accessのバージョンによって「DAO 3.6」や「DAO 12.0」など番号が異なる場合があります。最も新しいバージョンを選択してください。

7.3 DAOでデータを取得する（SELECT）

DAOの基本はレコードセットオブジェクトです。レコードセットは、テーブルやクエリから取得した複数のレコード（行）を扱うためのオブジェクトです。

基本的なコード例

VBA 
Sub 顧客データ取得()
    Dim db As DAO.Database
    Dim rs As DAO.Recordset
    Dim strSQL As String

    ' カレントデータベースを取得
    Set db = CurrentDb

    ' SQL文を作成
    strSQL = "SELECT 顧客ID, 顧客名, メールアドレス FROM T_顧客"

    ' レコードセットを開く
    Set rs = db.OpenRecordset(strSQL)

    ' レコードが存在するかチェック
    If Not rs.EOF Then
        ' 先頭レコードに移動
        rs.MoveFirst

        ' 全レコードをループ処理
        Do Until rs.EOF
            ' フィールド値を取得して表示
            Debug.Print rs!顧客ID, rs!顧客名, rs!メールアドレス

            ' 次のレコードへ移動
            rs.MoveNext
        Loop
    End If

    ' オブジェクトを閉じる
    rs.Close
    Set rs = Nothing
    Set db = Nothing
End Sub

コードの解説

  • CurrentDb：現在開いているAccessデータベースを取得
  • OpenRecordset：SQL文を実行してレコードセットを取得
  • EOF：End of File（最終レコードに到達したか判定）
  • MoveNext：次のレコードへカーソルを移動
  • rs!フィールド名：フィールドの値を取得

7.4 データの追加（INSERT）

DAOでテーブルに新しいレコードを追加するには、AddNewメソッドとUpdateメソッドを使います。

VBA 
Sub 顧客データ追加()
    Dim db As DAO.Database
    Dim rs As DAO.Recordset

    Set db = CurrentDb

    ' テーブルをレコードセットとして開く
    Set rs = db.OpenRecordset("T_顧客", dbOpenDynaset)

    ' 新規レコードを追加
    rs.AddNew
    rs!顧客名 = "山田太郎"
    rs!メールアドレス = "yamada@example.com"
    rs!電話番号 = "03-1234-5678"
    rs.Update  ' 確定

    rs.Close
    Set rs = Nothing
    Set db = Nothing

    MsgBox "顧客データを追加しました", vbInformation
End Sub

ポイント

  • dbOpenDynaset：編集可能なレコードセットを開く
  • AddNew：新規レコードの追加を開始
  • Update：変更を確定してテーブルに保存
  • Updateを忘れると追加されないので注意！

7.5 データの更新（UPDATE）

既存レコードを更新するには、Editメソッドを使ってから値を変更し、Updateで確定します。

VBA 
Sub 顧客メール更新()
    Dim db As DAO.Database
    Dim rs As DAO.Recordset
    Dim strSQL As String

    Set db = CurrentDb

    ' 特定の顧客を検索
    strSQL = "SELECT * FROM T_顧客 WHERE 顧客名 = '山田太郎'"
    Set rs = db.OpenRecordset(strSQL, dbOpenDynaset)

    ' レコードが見つかったら更新
    If Not rs.EOF Then
        rs.Edit  ' 編集開始
        rs!メールアドレス = "yamada_new@example.com"
        rs.Update  ' 確定

        MsgBox "メールアドレスを更新しました", vbInformation
    Else
        MsgBox "顧客が見つかりません", vbExclamation
    End If

    rs.Close
    Set rs = Nothing
    Set db = Nothing
End Sub

注意

Editを呼ばずに直接フィールドを変更しても、Updateを実行するとエラーになります。必ずEdit → 値変更 → Updateの順序を守ってください。

7.6 データの削除（DELETE）

レコードを削除するには、Deleteメソッドを使います。削除は元に戻せないため、慎重に実行してください。

VBA 
Sub 顧客データ削除()
    Dim db As DAO.Database
    Dim rs As DAO.Recordset
    Dim strSQL As String
    Dim intResult As Integer

    Set db = CurrentDb

    ' 削除確認
    intResult = MsgBox("本当に削除しますか？", vbYesNo + vbQuestion)
    If intResult = vbNo Then Exit Sub

    ' 削除対象を検索
    strSQL = "SELECT * FROM T_顧客 WHERE 顧客名 = '山田太郎'"
    Set rs = db.OpenRecordset(strSQL, dbOpenDynaset)

    If Not rs.EOF Then
        rs.Delete  ' 削除実行（Updateは不要）
        MsgBox "顧客データを削除しました", vbInformation
    Else
        MsgBox "顧客が見つかりません", vbExclamation
    End If

    rs.Close
    Set rs = Nothing
    Set db = Nothing
End Sub

削除時の注意点

  • • Deleteの後にUpdateは不要（即座に削除される）
  • • 削除前にユーザーに確認メッセージを表示する
  • • 関連テーブルがある場合は外部キー制約に注意
  • • 削除ではなく「削除フラグ」を立てる設計も検討する

7.7 ADOによるデータベース操作

ADOを使う場合は、DAOと同様に参照設定が必要です（Microsoft ActiveX Data Objects 6.1 Library）。基本的な使い方はDAOとほぼ同じです。

ADOでのデータ取得例

VBA 
Sub ADO顧客データ取得()
    Dim cn As ADODB.Connection
    Dim rs As ADODB.Recordset
    Dim strSQL As String

    ' コネクション作成
    Set cn = New ADODB.Connection
    cn.Open "Provider=Microsoft.ACE.OLEDB.12.0;Data Source=" & CurrentProject.Path & "\MyDB.accdb"

    ' SQL文を作成
    strSQL = "SELECT 顧客ID, 顧客名 FROM T_顧客"

    ' レコードセット作成
    Set rs = New ADODB.Recordset
    rs.Open strSQL, cn

    ' データを表示
    Do Until rs.EOF
        Debug.Print rs!顧客ID, rs!顧客名
        rs.MoveNext
    Loop

    ' クローズ
    rs.Close
    cn.Close
    Set rs = Nothing
    Set cn = Nothing
End Sub

DAOとADOの違い

項目 DAO ADO
接続 CurrentDb Connection.Open
対象DB Access専用 複数DB対応
速度 高速 やや低速
汎用性 低い 高い

7.8 トランザクション処理

トランザクションとは、複数のデータベース操作をひとまとまりとして扱う仕組みです。途中でエラーが発生した場合、すべての変更を取り消す（ロールバック）ことができます。

VBA 
Sub トランザクション例()
    Dim db As DAO.Database
    Dim rs As DAO.Recordset

    Set db = CurrentDb

    On Error GoTo ErrorHandler

    ' トランザクション開始
    db.BeginTrans

    ' 複数の操作を実行
    Set rs = db.OpenRecordset("T_顧客", dbOpenDynaset)
    rs.AddNew
    rs!顧客名 = "鈴木一郎"
    rs.Update
    rs.Close

    Set rs = db.OpenRecordset("T_売上", dbOpenDynaset)
    rs.AddNew
    rs!顧客ID = 1
    rs!金額 = 10000
    rs.Update
    rs.Close

    ' 全て成功したのでコミット
    db.CommitTrans
    MsgBox "トランザクション完了", vbInformation

    Set db = Nothing
    Exit Sub

ErrorHandler:
    ' エラー発生時はロールバック
    db.Rollback
    MsgBox "エラーが発生したため、すべての変更を取り消しました", vbCritical
    Set db = Nothing
End Sub

トランザクションが必要な場面

  • • 複数テーブルへの関連データ登録（顧客＋注文＋明細など）
  • • 在庫数の引き落としと売上記録の同時実行
  • • データの整合性を保ちたい重要な処理

理解度確認クイズ

  1. Q1. DAOとADOの主な違いを説明してください。

    回答を表示

    DAOはAccess専用で高速・軽量、ADOは複数種類のデータベースに対応可能で汎用性が高い。Access単独の場合はDAOが推奨される。

  2. Q2. レコードセットで新規レコードを追加する際に使う2つのメソッドは何ですか？

    回答を表示

    AddNew（新規レコードの追加を開始）、Update（変更を確定）

  3. Q3. 既存レコードを更新する場合、どのメソッドを使いますか？

    回答を表示

    Edit（編集開始）→ フィールド値を変更 → Update（確定）の順序で実行する

  4. Q4. トランザクション処理で使う3つのメソッドを答えてください。

    回答を表示

    BeginTrans（トランザクション開始）、CommitTrans（確定）、Rollback（取り消し）
