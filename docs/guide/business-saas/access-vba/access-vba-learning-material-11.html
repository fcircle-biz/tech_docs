エラー処理とデバッグ技法

On Error文によるエラー処理の実装、エラーの種類と対処方法、デバッグツールの活用、ログ出力によるトラブルシューティング手法を学びます。

この章で学ぶこと

  • エラー処理の基本（On Error文の使い方）
  • VBAで発生する主なエラーの種類と対処方法
  • デバッグツール（ブレークポイント、イミディエイトウィンドウ、ステップ実行）の活用
  • ログファイルを使ったトラブルシューティング手法

11.1 エラー処理の重要性

VBAプログラムを実務で使う際、エラー処理は必須のスキルです。エラーが発生したときに適切に処理しないと、プログラムが突然停止してデータが失われたり、ユーザーが混乱してしまいます。

エラー処理を実装することで、以下のメリットが得られます：

エラー処理のメリット

  • • プログラムが予期せず停止するのを防ぐ
  • • ユーザーにわかりやすいエラーメッセージを表示できる
  • • エラーの原因を特定しやすくなる
  • • データの破損やロールバック漏れを防ぐ
  • • 運用保守が容易になる

11.2 On Error文の基本

VBAでエラー処理を行うには、On Error文を使用します。主に3つの使い方があります。

On Error Resume Next

エラーが発生しても無視して次の行を実行します。簡単ですが、エラーを見逃すリスクがあるため注意が必要です。

VBA 
Sub Example1()
    On Error Resume Next

    ' ファイルが存在しなくてもエラーで止まらない
    Kill "C:\存在しないファイル.txt"

    ' 処理継続
    MsgBox "処理が完了しました"
End Sub

On Error GoTo ラベル

エラーが発生したら指定したラベルにジャンプします。最も推奨される方法です。

VBA 
Sub Example2()
    On Error GoTo ErrorHandler

    ' エラーが発生する可能性のある処理
    Dim num As Integer
    num = 10 / 0  ' ゼロ除算エラー

    Exit Sub

ErrorHandler:
    MsgBox "エラーが発生しました: " & Err.Description
End Sub

On Error GoTo 0

エラー処理を解除し、VBAのデフォルトのエラー処理（実行時エラーダイアログ表示）に戻します。

VBA 
Sub Example3()
    On Error Resume Next
    ' エラーを無視する処理

    On Error GoTo 0  ' エラー処理を解除
    ' この後はエラーが発生すると標準ダイアログ表示
End Sub

11.3 Errオブジェクトの活用

エラーが発生したときの詳細情報は、Errオブジェクトに格納されます。主なプロパティを理解しましょう。

プロパティ 説明
Err.Number エラー番号（整数値）
Err.Description エラーの説明文
Err.Source エラーの発生元
Err.Clear エラー情報をクリアするメソッド
VBA - Errオブジェクトの活用例 
Sub DetailedErrorHandling()
    On Error GoTo ErrorHandler

    ' エラーが発生する処理
    Dim rs As DAO.Recordset
    Set rs = CurrentDb.OpenRecordset("存在しないテーブル")

    Exit Sub

ErrorHandler:
    Dim msg As String
    msg = "エラーが発生しました" & vbCrLf
    msg = msg & "番号: " & Err.Number & vbCrLf
    msg = msg & "説明: " & Err.Description & vbCrLf
    msg = msg & "発生元: " & Err.Source

    MsgBox msg, vbCritical, "エラー"
    Err.Clear  ' エラー情報をクリア
End Sub

11.4 よくあるエラーと対処法

エラー番号 3021: レコードセットが空

原因：データが存在しないレコードセットにアクセスしようとした

対処：EOF、BOFプロパティで確認してからアクセス 

If Not rs.EOF Then
    ' レコードが存在する場合の処理
    txtName.Value = rs!氏名
End If

エラー番号 3078: テーブルが見つかりません

原因：存在しないテーブル名やタイプミス

対処：テーブル名のスペルミスを確認、または存在確認を実装

エラー番号 13: 型が一致しません

原因：文字列を数値型変数に代入しようとした、など

対処：IsNumeric関数で事前チェック、またはCInt、CDbl等で型変換 

If IsNumeric(txtAge.Value) Then
    intAge = CInt(txtAge.Value)
Else
    MsgBox "数値を入力してください"
End If

エラー番号 91: オブジェクト変数またはWithブロック変数が設定されていません

原因：Set文でオブジェクトを代入していない

対処：Is Nothing でチェック、またはSet文を追加 

Dim rs As DAO.Recordset
Set rs = CurrentDb.OpenRecordset("顧客")
If Not rs Is Nothing Then
    ' 処理
End If

11.5 デバッグツールの活用

VBE（Visual Basic Editor）には強力なデバッグツールが用意されています。これらを使いこなすことで、エラーの原因を素早く特定できます。

ブレークポイント

コード行をクリックして設定。実行がその行で一時停止します。

F9キー または コード行の左余白をクリック

ステップ実行

コードを1行ずつ実行して、変数の値の変化を確認できます。

F8キー（ステップイン）

イミディエイトウィンドウ

変数の値を確認したり、簡単なテストを実行できます。

Ctrl + G（表示）、? 変数名 で値確認

ウォッチウィンドウ

特定の変数を監視し、値が変わるたびに確認できます。

「表示」→「ウォッチウィンドウ」

デバッグのコツ

エラーが発生したら、まずブレークポイントを設定してステップ実行し、変数の値がどのように変化するか追跡しましょう。イミディエイトウィンドウで ? 変数名 と入力すると、その時点での値を確認できます。

11.6 ログ出力による保守性向上

実務では、エラーが発生したときの状況をログファイルに記録することで、後からトラブルシューティングがしやすくなります。

VBA - エラーログ出力の実装例 
Sub WriteErrorLog(strMessage As String)
    Dim fso As Object
    Dim logFile As Object
    Dim logPath As String

    ' ログファイルパス（Accessと同じフォルダ）
    logPath = CurrentProject.Path & "\error.log"

    Set fso = CreateObject("Scripting.FileSystemObject")

    ' ログファイルを開く（なければ新規作成）
    Set logFile = fso.OpenTextFile(logPath, 8, True)

    ' ログ書き込み
    logFile.WriteLine Now & " - " & strMessage

    logFile.Close
    Set logFile = Nothing
    Set fso = Nothing
End Sub
VBA - ログ出力を含むエラー処理の実装例 
Sub SampleWithLogging()
    On Error GoTo ErrorHandler

    ' 何らかの処理
    Dim rs As DAO.Recordset
    Set rs = CurrentDb.OpenRecordset("顧客")

    ' 処理成功
    WriteErrorLog "顧客データの処理が完了しました"

    Exit Sub

ErrorHandler:
    ' エラー情報をログに記録
    Dim errMsg As String
    errMsg = "エラー番号: " & Err.Number & _
             " / 説明: " & Err.Description
    WriteErrorLog errMsg

    ' ユーザーにもエラー通知
    MsgBox "エラーが発生しました。詳細はログファイルを確認してください。", _
           vbCritical, "エラー"
End Sub

よくあるトラブルと解決法

Q. On Error GoToを設定したのにエラーハンドラが動かない

A. ラベル名のスペルミスや、Exit Sub の記述漏れを確認しましょう。また、On Error GoTo 0 で解除されていないかチェック

Q. デバッグ実行が途中で止まってしまう

A. 無限ループやブレークポイントが複数設定されていないか確認。Ctrl + Break で強制停止できます

Q. ログファイルが作成されない

A. CurrentProject.Path のフォルダに書き込み権限があるか確認。権限がない場合は別のフォルダ（例: Tempフォルダ）を指定

理解度確認クイズ

  1. Q1. On Error Resume Next と On Error GoTo ラベル の違いは何ですか？どちらが推奨されますか？

    回答を表示

    Resume Nextはエラーを無視して次行を実行しますが、エラーを見逃すリスクがあります。GoTo ラベルは指定した場所でエラー処理を行えるため、より安全で推奨されます。

  2. Q2. Errオブジェクトの主要なプロパティを3つ挙げてください。

    回答を表示

    Err.Number（エラー番号）、Err.Description（エラーの説明）、Err.Source（エラーの発生元）

  3. Q3. VBEでブレークポイントを設定するキーは何ですか？

    回答を表示

    F9キー（またはコード行の左余白をクリック）

  4. Q4. イミディエイトウィンドウで変数の値を確認する方法は？

    回答を表示

    Ctrl + G でイミディエイトウィンドウを開き、「? 変数名」と入力してEnterキーを押す
