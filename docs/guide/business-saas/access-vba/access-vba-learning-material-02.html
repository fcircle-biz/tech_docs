Accessデータベースの基礎知識

テーブルの作成とフィールド設計、データ型の選択、主キーと外部キーの概念、リレーションシップの設定方法を実践的に学び、データベース設計の基礎を固めます。

この章で学ぶこと

  • テーブルの作成方法とフィールド設計の基本を習得する
  • データ型の種類と適切な選択方法を理解する
  • 主キーと外部キーの概念と設定方法を学ぶ
  • テーブル間のリレーションシップを設定する方法を習得する

2.1 テーブルとは何か？

テーブル（Table）は、Accessデータベースの最も基本的な構成要素で、データを保存する場所です。Excelのワークシートに似ていますが、より厳格なデータ構造を持ちます。

テーブルは行（レコード）と列（フィールド）で構成されています：

全12316;行・データ全12317;"] B --> B1["顧客ID"] B --> B2["顧客名"] B --> B3["住所"] C --> C1["データ1行目"] C --> C2["データ2行目"] style A fill:#d1fae5,stroke:#10b981,stroke-width:2px style B fill:#dbeafe,stroke:#3b82f6,stroke-width:1px style C fill:#fef3c7,stroke:#f59e0b,stroke-width:1px

フィールド（Field）- 列

データの項目を表します。例えば「顧客ID」「顧客名」「電話番号」「住所」など。
ポイント：フィールドには必ずデータ型を設定する必要があります。

レコード（Record）- 行

1件分のデータを表します。例えば「顧客ID=001、顧客名=山田太郎、電話番号=03-1234-5678、住所=東京都」。
ポイント：1つのテーブルに何千、何万件のレコードを保存できます。

2.2 テーブルの作成方法

Accessでテーブルを作成する基本手順を見てみましょう。ここでは「顧客マスタ」テーブルを例に説明します。

テーブル作成の手順

  1. 1 Accessを開き、データベースファイル（.accdb）を開く、または新規作成します
  2. 2 リボンメニューの「作成」タブをクリック
  3. 3 「テーブルデザイン」ボタンをクリック
  4. 4 フィールド名とデータ型を設定していきます
  5. 5 保存時にテーブル名（例：「T_顧客マスタ」）を入力して完成

命名規則のヒント

テーブル名には接頭辞として「T_」を付けると、後からクエリやフォームと区別しやすくなります。
例：T_顧客マスタ、T_商品マスタ、T_売上明細

2.3 データ型の種類と選び方

フィールドを作成する際、必ずデータ型（Data Type）を設定する必要があります。データ型は、そのフィールドに保存できるデータの種類を決めるものです。

データ型 説明 使用例
短いテキスト 255文字以内の文字列 顧客名、住所、電話番号
長いテキスト 65,535文字までの長文 備考、説明、メモ
数値型 整数または小数 年齢、数量、在庫数
日付/時刻型 日付・時刻データ 生年月日、登録日、更新日時
通貨型 金額データ（小数点以下4桁） 単価、金額、売上
Yes/No型 真偽値（True/False） フラグ、有効/無効
オートナンバー型 自動採番される連番 顧客ID、商品ID（主キー）

データ型選択のポイント

  • • 郵便番号や電話番号は「短いテキスト」 - ハイフンが入るため数値型は使えません
  • • 金額は「通貨型」 - 小数点の計算誤差を防ぎます
  • • IDには「オートナンバー型」 - 重複しない一意の値が自動生成されます

2.4 主キー（Primary Key）の設定

主キー（Primary Key）は、テーブル内の各レコードを一意に識別するためのフィールドです。重複した値が登録されず、必ず値が入ります（NULL不可）。

全12316;主キー全12317;"] A --> C["顧客名"] A --> D["住所"] B --> E["重複不可
必須入力"] C --> F["重複可能"] D --> F style A fill:#d1fae5,stroke:#10b981,stroke-width:2px style B fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style E fill:#fecaca,stroke:#ef4444,stroke-width:1px

主キーの設定方法

  1. 1 テーブルデザインビューを開く
  2. 2 主キーにしたいフィールド（例：顧客ID）の行を選択
  3. 3 リボンメニューの「主キー」ボタン（鍵マーク）をクリック
  4. 4 フィールド名の左に鍵マークが表示されれば設定完了

主キーの重要性

すべてのテーブルには必ず主キーを設定してください。主キーがないと、後でレコードを特定・更新・削除する際に問題が発生します。

2.5 外部キーとリレーションシップ

外部キー（Foreign Key）は、別のテーブルの主キーを参照するフィールドです。これにより、テーブル間にリレーションシップ（関連性）を設定できます。

graph LR A["T_顧客マスタ"] -->|1対多| B["T_売上明細"] A1["顧客ID 全12316;主キー全12317;"] -.参照される.- B1["顧客ID 全12316;外部キー全12317;"] style A fill:#d1fae5,stroke:#10b981,stroke-width:2px style B fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style A1 fill:#fef3c7,stroke:#f59e0b,stroke-width:1px style B1 fill:#e9d5ff,stroke:#a855f7,stroke-width:1px

リレーションシップの種類

  • 1対多（One-to-Many） - 最も一般的。1つの顧客に対して複数の売上データが紐づく
  • 1対1（One-to-One） - 1つのレコードに対して1つだけ関連。顧客と顧客詳細情報など
  • 多対多（Many-to-Many） - 中間テーブルを使って実現。商品と発注の関係など

リレーションシップの設定手順

  1. 1 リボンメニュー「データベースツール」→「リレーションシップ」をクリック
  2. 2 関連付けたいテーブルを追加（例：T_顧客マスタ、T_売上明細）
  3. 3 T_顧客マスタの「顧客ID」をドラッグして、T_売上明細の「顧客ID」にドロップ
  4. 4 「参照整合性を適用する」にチェックを入れて「作成」をクリック

参照整合性とは？

参照整合性を適用すると、存在しない顧客IDの売上データを登録できなくなります。これによりデータの整合性が保たれ、孤立したデータが発生しません。

2.6 実習：顧客管理テーブルの設計

ここまで学んだ知識を使って、実際に顧客管理用のテーブルを設計してみましょう。

実習：T_顧客マスタテーブルの作成

以下のフィールドを持つテーブルを作成してください：

フィールド名 データ型 備考
顧客ID オートナンバー型 主キー
顧客名 短いテキスト 必須
フリガナ 短いテキスト -
郵便番号 短いテキスト 7桁（ハイフンなし）
住所 短いテキスト -
電話番号 短いテキスト ハイフン含む
メールアドレス 短いテキスト -
登録日 日付/時刻型 既定値 Date()

実習のポイント

  • • 「顧客ID」を主キーに設定することを忘れずに
  • • 郵便番号と電話番号は「短いテキスト」型を選択（数値型ではない）
  • • 「登録日」フィールドの既定値に「Date()」と設定すると、自動で今日の日付が入ります

理解度確認クイズ

  1. Q1. テーブルにおける「レコード」と「フィールド」の違いを説明してください。

    回答を表示

    レコードは「行」で1件分のデータ全体を表し、フィールドは「列」でデータの項目（顧客名、住所など）を表します。

  2. Q2. 電話番号を保存するフィールドに適切なデータ型は何ですか？また、その理由を答えてください。

    回答を表示

    「短いテキスト」型。理由：電話番号にはハイフン（-）が含まれるため、数値型では保存できないため。

  3. Q3. 主キーの役割を2つ挙げてください。

    回答を表示

    ① テーブル内の各レコードを一意に識別する、② 重複した値の登録を防ぐ（その他、外部キーからの参照先となる、なども正解）

  4. Q4. 参照整合性を適用する目的は何ですか？

    回答を表示

    存在しない親レコード（例：顧客ID）を参照する子レコード（例：売上データ）の登録を防ぎ、データの整合性を保つため。
