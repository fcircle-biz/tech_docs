第8章

SQLとVBAの連携

VBAからSQLクエリを実行する方法、パラメータクエリの活用、動的SQLの生成、トランザクション処理の実装を学び、効率的なデータ操作を実現します。

この章で学ぶこと

  • VBAからSQLクエリを実行する基本的な方法を理解する
  • パラメータクエリを使って安全で効率的なデータ操作を実装する
  • 動的SQLを生成して条件に応じたクエリを実行する
  • トランザクション処理でデータの整合性を保証する方法を習得する

8.1 VBAからSQLを実行する基本

SQL（Structured Query Language）は、データベースを操作するための標準的な言語です。VBAからSQLを実行することで、より柔軟で強力なデータ操作が可能になります。

VBAからSQLを実行する主な方法は以下の3つです：

1. DoCmd.RunSQL

アクションクエリ（INSERT、UPDATE、DELETE）を実行するシンプルな方法。戻り値は取得できません。

VBAコード 
' 顧客テーブルから東京都のレコードを削除
DoCmd.SetWarnings False  ' 確認ダイアログを非表示
DoCmd.RunSQL "DELETE FROM T_顧客 WHERE 都道府県 = '東京都'"
DoCmd.SetWarnings True   ' 確認ダイアログを再表示

2. DAO.Database.Execute

DAOを使ってSQLを実行。影響を受けた行数を取得できます。

VBAコード 
Dim db As DAO.Database
Set db = CurrentDb

' 売上テーブルの価格を10%値上げ
db.Execute "UPDATE T_売上 SET 単価 = 単価 * 1.1", dbFailOnError

MsgBox "更新完了: " & db.RecordsAffected & "件", vbInformation

3. DAO.Database.OpenRecordset

SELECTクエリを実行し、結果をレコードセットとして取得。データを読み取る際に使用します。

VBAコード 
Dim db As DAO.Database
Dim rs As DAO.Recordset

Set db = CurrentDb
Set rs = db.OpenRecordset("SELECT * FROM T_顧客 WHERE 売上金額 > 100000")

' データを処理
Do Until rs.EOF
    Debug.Print rs!顧客名, rs!売上金額
    rs.MoveNext
Loop

注意

DoCmd.RunSQLは、実行前に必ずDoCmd.SetWarnings Falseで確認ダイアログを無効にし、実行後にTrueで再有効化してください。

8.2 パラメータクエリの活用

パラメータクエリは、実行時に値を指定してクエリを実行する方法です。SQLインジェクション攻撃を防ぎ、安全で効率的なデータ操作を実現します。

パラメータクエリの作成と実行

VBAコード 
Sub パラメータクエリ実行()
    Dim db As DAO.Database
    Dim qdf As DAO.QueryDef
    Dim rs As DAO.Recordset

    Set db = CurrentDb

    ' パラメータクエリを作成（保存しない一時クエリ）
    Set qdf = db.CreateQueryDef("")
    qdf.SQL = "SELECT * FROM T_顧客 WHERE 都道府県 = [pref] AND 売上金額 > [amount]"

    ' パラメータに値を設定
    qdf.Parameters("pref").Value = "東京都"
    qdf.Parameters("amount").Value = 100000

    ' クエリ実行
    Set rs = qdf.OpenRecordset()

    ' 結果を処理
    Do Until rs.EOF
        Debug.Print rs!顧客名, rs!都道府県, rs!売上金額
        rs.MoveNext
    Loop

    rs.Close
    Set rs = Nothing
    Set qdf = Nothing
    Set db = Nothing
End Sub

パラメータクエリのメリット

  • • 安全性: SQLインジェクション攻撃を防ぐ
  • • 再利用性: クエリを再利用して異なる値で実行
  • • パフォーマンス: クエリプランが最適化される
  • • 可読性: SQLが読みやすく保守しやすい

8.3 動的SQLの生成

動的SQLは、実行時の条件に応じてSQL文を文字列として組み立てる方法です。複雑な検索条件や可変的なクエリに対応できます。

実習: 検索条件に応じたSQL生成

VBAコード 
Sub 動的検索クエリ()
    Dim strSQL As String
    Dim strWhere As String
    Dim rs As DAO.Recordset

    ' 基本SELECT文
    strSQL = "SELECT * FROM T_顧客"

    ' WHERE句を動的に組み立て
    If Me.txt都道府県 <> "" Then
        strWhere = strWhere & " AND 都道府県 = '" & Me.txt都道府県 & "'"
    End If

    If Me.txt最低売上 <> "" Then
        strWhere = strWhere & " AND 売上金額 >= " & Me.txt最低売上
    End If

    If Me.chk会員のみ = True Then
        strWhere = strWhere & " AND 会員フラグ = True"
    End If

    ' WHERE句が存在する場合、先頭の" AND "を除去して追加
    If strWhere <> "" Then
        strSQL = strSQL & " WHERE " & Mid(strWhere, 6)
    End If

    ' ORDER BY句を追加
    strSQL = strSQL & " ORDER BY 売上金額 DESC"

    ' クエリ実行
    Set rs = CurrentDb.OpenRecordset(strSQL)

    ' 結果を表示
    Debug.Print "実行SQL: " & strSQL
    Debug.Print "検索結果: " & rs.RecordCount & "件"

    rs.Close
    Set rs = Nothing
End Sub

動的SQL使用時の注意点

  • • 文字列値には必ずシングルクォート（'）で囲む
  • • 日付型は # で囲む: #2024/01/01#
  • • 特殊文字のエスケープ: シングルクォートは2つ重ねる（''）
  • • 可能な限りパラメータクエリを優先して、SQLインジェクションを防ぐ

8.4 トランザクション処理

トランザクション処理は、複数のSQL操作を一つの単位として扱い、すべて成功するか、すべて失敗するかを保証する仕組みです。データの整合性を保つために重要です。

実習: トランザクション処理の実装

VBAコード 
Sub 注文処理WithTransaction()
    Dim db As DAO.Database
    Dim strSQL As String

    On Error GoTo ErrorHandler

    Set db = CurrentDb

    ' トランザクション開始
    db.BeginTrans

    ' 1. 在庫テーブルから商品を減らす
    strSQL = "UPDATE T_在庫 SET 在庫数 = 在庫数 - 5 WHERE 商品ID = 101"
    db.Execute strSQL, dbFailOnError

    ' 2. 売上テーブルに注文を追加
    strSQL = "INSERT INTO T_売上 (顧客ID, 商品ID, 数量, 売上日) " & _
             "VALUES (201, 101, 5, #" & Date & "#)"
    db.Execute strSQL, dbFailOnError

    ' 3. 顧客の累計購入額を更新
    strSQL = "UPDATE T_顧客 SET 累計購入額 = 累計購入額 + 50000 WHERE 顧客ID = 201"
    db.Execute strSQL, dbFailOnError

    ' すべて成功したらコミット
    db.CommitTrans
    MsgBox "注文処理が完了しました", vbInformation

    Exit Sub

ErrorHandler:
    ' エラーが発生したらロールバック
    db.Rollback
    MsgBox "エラーが発生しました。すべての処理を取り消しました。" & vbCrLf & _
           Err.Description, vbCritical
End Sub

BeginTrans

トランザクションを開始します。この時点から変更は一時的なものとなり、CommitTrans または Rollback が実行されるまで確定しません。

CommitTrans

トランザクション内のすべての変更を確定します。データベースに永続的に保存されます。

Rollback

トランザクション内のすべての変更を取り消します。エラーが発生した場合に使用し、データベースを元の状態に戻します。

8.5 SQL最適化のヒント

VBAからSQLを実行する際、パフォーマンスを向上させるためのベストプラクティスを紹介します。

インデックスを活用

WHERE句やJOIN句で使用するフィールドにはインデックスを設定すると検索が高速化されます。

主キー、外部キー、頻繁に検索するフィールド

必要なフィールドのみ取得

SELECT * ではなく、必要なフィールドのみを指定することでデータ転送量を削減します。

SELECT 顧客名, 売上金額

複数操作をまとめる

レコードセットで1件ずつ更新するより、UPDATE文で一括更新する方が高速です。

可能な限り一括処理を優先

適切なデータ型を使用

テキスト型よりも数値型、日付型の方が比較演算が高速です。適切なデータ型を選択しましょう。

Long, Double, Date/Time型を優先

ヒント

複雑なクエリの場合、SQLビューで直接クエリを作成・テストしてから、VBAにコピーすると効率的です。Accessのクエリデザイナーを活用しましょう。

実習課題

  1. 課題1: 顧客テーブルから特定の都道府県の顧客を検索するパラメータクエリを作成してください。
  2. 課題2: 検索条件（都道府県、最低売上金額、会員フラグ）を動的に組み立てる検索フォームを作成してください。
  3. 課題3: 商品の発注処理を実装してください。在庫減少、売上登録、顧客の累計更新をトランザクションで処理します。

理解度確認クイズ

  1. Q1. VBAからアクションクエリ（INSERT、UPDATE、DELETE）を実行する最もシンプルな方法は何ですか？

    回答を表示

    DoCmd.RunSQL（ただし、DoCmd.SetWarnings False/Trueで確認ダイアログの制御が必要）

  2. Q2. パラメータクエリを使用する主なメリットを3つ挙げてください。

    回答を表示

    ① SQLインジェクション攻撃の防止（安全性）、② クエリの再利用性、③ クエリプランの最適化（パフォーマンス）、④ 可読性・保守性の向上

  3. Q3. トランザクション処理で使用する3つのメソッド（開始、確定、取り消し）は何ですか？

    回答を表示

    BeginTrans（開始）、CommitTrans（確定）、Rollback（取り消し）

  4. Q4. 動的SQLで文字列値を埋め込む際、どのように囲む必要がありますか？

    回答を表示

    シングルクォート（'）で囲む。例: "WHERE 都道府県 = '東京都'"
    日付型は # で囲む。例: "#2024/01/01#"

よくあるトラブルと対処法

「パラメータが少なすぎます」エラー

原因: SQLに含まれるパラメータ名が間違っている、または指定されていない
対処: qdf.Parameters のパラメータ名がSQL内の [パラメータ名] と一致しているか確認

「データ型が一致しません」エラー

原因: 文字列値をシングルクォートで囲んでいない、または数値を文字列として扱っている
対処: データ型に応じて適切にクォートを使用（文字列='値'、数値=値、日付=#値#）

トランザクション中にエラーが出ても変更が残る

原因: エラーハンドラで Rollback を呼んでいない
対処: On Error GoTo で必ずエラーハンドラを設置し、Rollback を実行
