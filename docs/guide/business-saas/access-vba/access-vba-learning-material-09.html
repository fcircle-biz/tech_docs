第9章

フォームとイベント処理

Accessフォームのイベントモデル、コントロールの操作、入力検証、フォーム間のデータ連携を学び、ユーザーフレンドリーなインターフェースを作成します。

この章で学ぶこと

  • フォームイベントの種類と発生タイミングを理解する
  • テキストボックス、コンボボックス、ボタンなどのコントロール操作をマスターする
  • 入力検証（バリデーション）を実装し、データの品質を保つ
  • フォーム間でデータを受け渡し、連携動作を実現する

9.1 フォームイベントの基礎

フォームイベントとは、ユーザーがフォームを開いたり、ボタンをクリックしたり、データを入力したりしたときに自動的に発生する「きっかけ」のことです。VBAではこのイベントに対して処理を記述することで、インタラクティブなアプリケーションを作成できます。

主要なフォームイベント

  • • Form_Load - フォームが開かれたときに発生
  • • Form_Current - レコードが移動したときに発生
  • • Form_BeforeUpdate - レコードが保存される前に発生
  • • Form_AfterUpdate - レコードが保存された後に発生
  • • Form_Close - フォームが閉じられるときに発生
VBA 
' フォーム読み込み時の初期化処理
Private Sub Form_Load()
    ' フォームが開かれたときに自動実行される
    Me.txtDate.Value = Date    ' 今日の日付を自動入力
    Me.cmbStatus.Value = "未処理"   ' ステータスを初期化
    MsgBox "フォームを開きました", vbInformation
End Sub

' レコード保存前の検証処理
Private Sub Form_BeforeUpdate(Cancel As Integer)
    ' 必須項目チェック
    If IsNull(Me.txtName.Value) Then
        MsgBox "氏名を入力してください", vbExclamation
        Cancel = True   ' 保存をキャンセル
    End If
End Sub

9.2 コントロールイベントの活用

フォーム上の各コントロール（テキストボックス、ボタン、コンボボックスなど）にも独自のイベントがあります。これらを組み合わせることで、ユーザーの操作に応じた動的な処理を実装できます。

ボタンのClickイベント

ボタンがクリックされたときに処理を実行します。最もよく使われるイベントです。

テキストのChangeイベント

テキストボックスの値が変更されるたびに発生します。リアルタイム検証に便利です。

コンボボックスのAfterUpdate

ドロップダウンで値を選択した後に発生します。選択に応じた連動処理に使います。

GotFocus / LostFocus

コントロールにフォーカスが当たった/離れたときに発生します。入力ガイドの表示に便利です。

VBA - ボタンクリックイベント 
' ボタンクリックイベント
Private Sub btnSave_Click()
    ' 保存ボタンがクリックされたときの処理
    If MsgBox("データを保存しますか?", vbYesNo) = vbYes Then
        DoCmd.RunCommand acCmdSaveRecord
        MsgBox "保存しました", vbInformation
    End If
End Sub

' テキスト変更イベント
Private Sub txtZipCode_Change()
    ' 郵便番号が7桁入力されたら自動で住所検索
    If Len(Me.txtZipCode.Value) = 7 Then
        Call SearchAddress(Me.txtZipCode.Value)
    End If
End Sub

9.3 コントロールの操作

VBAからフォーム上のコントロールを操作することで、値の取得・設定、表示/非表示の切り替え、有効/無効の制御など、柔軟なUIを実現できます。

VBA - コントロールの基本操作 
' 値の取得と設定
Private Sub SetControlValues()
    ' テキストボックスに値を設定
    Me.txtCustomerName.Value = "山田太郎"

    ' コンボボックスの選択項目を設定
    Me.cmbCategory.Value = "商品A"

    ' チェックボックスをオン/オフ
    Me.chkActive.Value = True

    ' 値を取得して変数に格納
    Dim customerName As String
    customerName = Me.txtCustomerName.Value
End Sub

' コントロールの表示制御
Private Sub ToggleControls()
    ' コントロールを非表示にする
    Me.txtDiscountRate.Visible = False

    ' コントロールを無効化（グレーアウト）
    Me.btnDelete.Enabled = False

    ' ラベルの色を変更
    Me.lblWarning.ForeColor = vbRed
End Sub

ヒント: Meキーワード

Meは「このフォーム自身」を指すキーワードです。Me.txtNameと書くことで、そのフォーム上のtxtNameというコントロールを参照できます。

9.4 入力検証（バリデーション）の実装

入力検証は、ユーザーが不正なデータを入力しないようにチェックする仕組みです。データベースの品質を保つために非常に重要です。

VBA - 入力検証の実装 
' レコード保存前の検証処理
Private Sub Form_BeforeUpdate(Cancel As Integer)
    ' 必須項目のチェック
    If IsNull(Me.txtCustomerName) Or Me.txtCustomerName = "" Then
        MsgBox "顧客名を入力してください", vbExclamation
        Cancel = True   ' 保存をキャンセル
        Me.txtCustomerName.SetFocus   ' カーソルを移動
        Exit Sub
    End If

    ' 郵便番号の形式チェック（7桁の数字）
    If Not IsNull(Me.txtZipCode) Then
        If Len(Me.txtZipCode) <> 7 Or Not IsNumeric(Me.txtZipCode) Then
            MsgBox "郵便番号は7桁の数字で入力してください", vbExclamation
            Cancel = True
            Exit Sub
        End If
    End If

    ' 日付の妥当性チェック
    If Me.txtStartDate > Me.txtEndDate Then
        MsgBox "終了日は開始日以降を指定してください", vbExclamation
        Cancel = True
        Exit Sub
    End If
End Sub

注意: Cancel引数の使い方

Cancel = Trueを設定すると、保存がキャンセルされます。検証エラーがある場合は必ずこれを設定してください。

9.5 フォーム間のデータ連携

複数のフォームを連携させることで、より複雑な業務フローを実現できます。例えば、顧客一覧フォームから顧客詳細フォームを開いて編集する、といった操作です。

sequenceDiagram participant U as ユーザー participant F1 as 顧客一覧フォーム participant F2 as 顧客詳細フォーム participant DB as データベース U->>F1: レコードを選択 U->>F1: 詳細ボタンクリック F1->>F2: OpenArgs で顧客IDを渡す F2->>DB: 顧客IDでレコード検索 DB-->>F2: 顧客データ取得 F2-->>U: 詳細画面表示 U->>F2: データ編集・保存 F2->>DB: 更新実行 F2->>F1: Requery でリスト更新 F1-->>U: 一覧画面に戻る
VBA - フォーム間のデータ受け渡し 
' 【一覧フォーム】詳細フォームを開く
Private Sub btnOpenDetail_Click()
    ' 選択中の顧客IDを取得
    Dim customerID As Long
    customerID = Me.CustomerID.Value

    ' 詳細フォームを開き、顧客IDを渡す
    DoCmd.OpenForm "F_CustomerDetail", _
                   acNormal, , , , , _
                   OpenArgs:=customerID
End Sub

' 【詳細フォーム】渡されたIDでデータを表示
Private Sub Form_Load()
    ' OpenArgsで渡された顧客IDを取得
    If Not IsNull(Me.OpenArgs) Then
        Dim customerID As Long
        customerID = CLng(Me.OpenArgs)

        ' 該当する顧客のレコードを表示
        Me.RecordsetClone.FindFirst "CustomerID = " & customerID
        If Not Me.RecordsetClone.NoMatch Then
            Me.Bookmark = Me.RecordsetClone.Bookmark
        End If
    End If
End Sub

9.6 コンボボックスの動的な値設定

コンボボックス（ドロップダウンリスト）の選択肢をVBAで動的に設定することで、条件に応じた絞り込みや連動選択を実現できます。

VBA - コンボボックスの動的設定 
' カテゴリーコンボボックスの選択に応じて商品コンボを更新
Private Sub cmbCategory_AfterUpdate()
    Dim selectedCategory As String
    selectedCategory = Me.cmbCategory.Value

    ' 商品コンボボックスのRowSourceを動的に変更
    Me.cmbProduct.RowSource = _
        "SELECT ProductName FROM T_Products " & _
        "WHERE Category = '" & selectedCategory & "' " & _
        "ORDER BY ProductName"

    ' コンボボックスを再クエリして最新状態に
    Me.cmbProduct.Requery
    Me.cmbProduct.Value = Null   ' 選択をクリア
End Sub

' フォーム読み込み時にコンボボックスを初期化
Private Sub Form_Load()
    ' カテゴリーコンボボックスの設定
    Me.cmbCategory.RowSource = _
        "SELECT DISTINCT Category FROM T_Products ORDER BY Category"
    Me.cmbCategory.Requery
End Sub

9.7 サブフォームの操作

サブフォームは、メインフォームの中に埋め込まれた別のフォームです。例えば、顧客詳細フォーム（メイン）の中に注文履歴フォーム（サブ）を表示するような使い方をします。

VBA - サブフォームへのアクセス 
' メインフォームからサブフォームのコントロールを参照
Private Sub btnUpdateSubform_Click()
    ' サブフォームコントロールの参照
    Dim subForm As Form
    Set subForm = Me.SubformControl.Form

    ' サブフォームのテキストボックスに値を設定
    subForm.txtNote.Value = "自動入力されました"

    ' サブフォームを再クエリして最新データを表示
    Me.SubformControl.Requery
End Sub

' サブフォームの合計値をメインフォームに表示
Private Sub Form_Current()
    Dim totalAmount As Currency

    ' サブフォームの合計を取得
    totalAmount = Nz(Me.SubformControl.Form.Controls("txtTotal"), 0)

    ' メインフォームのラベルに表示
    Me.lblTotalDisplay.Caption = "合計: " & Format(totalAmount, "#,##0円")
End Sub

理解度確認クイズ

  1. Q1. フォームが開かれたときに自動実行されるイベントは何ですか？

    回答を表示

    Form_Load イベント

  2. Q2. レコード保存前に入力検証を行いたい場合、どのイベントを使いますか？

    回答を表示

    Form_BeforeUpdate イベント。検証エラーがある場合は Cancel = True で保存をキャンセルします。

  3. Q3. 別のフォームを開くときにデータを渡すには、何という引数を使いますか？

    回答を表示

    OpenArgs 引数。受け取り側のフォームでは Me.OpenArgs で値を取得できます。

  4. Q4. コンボボックスの選択肢を更新するには何というメソッドを使いますか？

    回答を表示

    Requery メソッド（例: Me.cmbCategory.Requery）

よくあるトラブルと対処法

イベントが実行されない

原因: イベント名のスペルミス、またはプロシージャが Private Sub で定義されていない
対処法: VBEのコードウィンドウ上部のドロップダウンから正しいイベントを選択して自動生成する

Me.コントロール名 でエラーが出る

原因: コントロール名が間違っているか、フォームモジュール以外で Me を使っている
対処法: フォームのデザインビューでコントロールの「名前」プロパティを確認する

Cancel = True が効かない

原因: BeforeUpdate 以外のイベントで使っている
対処法: Cancel 引数が使えるイベント（BeforeUpdate, BeforeDelConfirm など）で使用する
