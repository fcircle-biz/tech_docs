第3章

VBA開発環境とはじめてのプログラム

Visual Basic Editor（VBE）の詳しい使い方、プロジェクトとモジュールの概念、はじめてのVBAプログラム作成、デバッグツールの基本を学びます。

この章で学ぶこと

  • Visual Basic Editor（VBE）の詳細な画面構成と各ウィンドウの役割を理解する
  • プロジェクトとモジュールの概念を理解し、標準モジュールを作成できる
  • はじめてのVBAプログラム（Subプロシージャ）を書いて実行する
  • デバッグツール（ブレークポイント、ステップ実行、イミディエイトウィンドウ）の基本操作を習得する

3.1 Visual Basic Editorの画面構成

第1章で簡単に紹介したVBEについて、より詳しく見ていきましょう。VBEは複数のウィンドウで構成されており、それぞれに重要な役割があります。

プロジェクトエクスプローラー（左上）

データベースファイル内のすべてのオブジェクトが階層構造で表示されます。フォームやレポート、モジュールなどを選択してコードを編集します。

表示される主な項目：

  • • Microsoft Access クラスオブジェクト: フォーム、レポート
  • • 標準モジュール: 汎用的なプログラムコードを保存
  • • クラスモジュール: 独自のオブジェクトを作成

プロパティウィンドウ（左下）

選択したオブジェクトの詳細設定を表示・変更できます。フォームコントロールの色、サイズ、動作などを調整する際に使用します。

コードウィンドウ（右側・中央）

最も重要なウィンドウです。ここにVBAコードを記述します。構文の色分け、自動補完機能があり、コードが書きやすくなっています。

イミディエイトウィンドウ（下部）

コードのテストやデバッグに使用します。Debug.Printで出力した内容がここに表示されます。

表示方法： メニューバー「表示」→「イミディエイトウィンドウ」または Ctrl + G

3.2 プロジェクトとモジュールの概念

VBAでは、コードを「モジュール」という単位で管理します。モジュールは、関連するコードをまとめて保存する箱のようなものです。

標準モジュール

どこからでも呼び出せる汎用的なプログラムを保存します。データベース全体で共通して使う処理を記述します。

用途例： データ検証、集計処理、共通関数

フォームモジュール

特定のフォームに関連付けられたコードを保存します。ボタンクリックやデータ更新などのイベント処理を記述します。

用途例： ボタンクリックイベント、フォーム読み込み処理

レポートモジュール

特定のレポートに関連付けられたコードを保存します。レポート生成時の集計や書式設定を記述します。

用途例： レポート開く時の集計処理、ページヘッダー制御

3.3 標準モジュールの作成

それでは、実際に標準モジュールを作成してみましょう。ここに最初のVBAコードを書いていきます。

実習：標準モジュールを作成する

  1. 1 Accessを開き、Alt + F11でVBEを起動
  2. 2 メニューバーから「挿入」→「標準モジュール」をクリック
  3. 3 プロジェクトエクスプローラーに「Module1」が追加され、コードウィンドウが開きます
  4. 4 モジュール名は右クリック→「プロパティ」から変更可能（例：「共通処理」など）

3.4 はじめてのVBAプログラム

それでは、最初のVBAプログラムを書いてみましょう。メッセージボックスを表示する簡単なプログラムです。

Module1 - はじめてのプログラム 
Sub はじめてのプログラム()
    ' メッセージボックスを表示
    MsgBox "こんにちは！Access VBAの世界へようこそ！"
End Sub

コードの説明

  • • Sub はじめてのプログラム(): プロシージャの開始宣言
  • • ' コメント: アポストロフィで始まる行はコメント（メモ書き）
  • • MsgBox: メッセージボックスを表示する命令
  • • End Sub: プロシージャの終了

プログラムを実行する

コードを入力したら、実行してみましょう。3つの方法があります：

方法1： コード内にカーソルを置いて F5 キーを押す
方法2： メニューバー「実行」→「Sub/ユーザーフォームの実行」をクリック
方法3： ツールバーの再生ボタン（▶）をクリック

実行すると「こんにちは！Access VBAの世界へようこそ！」というメッセージが表示されます

    ' ここに処理を書く

プロシージャ名の命名ルール

  • • 日本語・英数字が使える（推奨：日本語で分かりやすく）
  • • 最初の1文字は必ず文字（数字はNG）
  • • スペースは使えない（アンダースコア_は使用可）
  • • VBAの予約語（Sub, If, Loopなど）は使えない

3.6 デバッグツールの基本

プログラムが正しく動作しているか確認するため、VBEには強力なデバッグツールが用意されています。基本的な使い方を押さえましょう。

ブレークポイント

コードの実行を一時停止したい行をクリックすると、赤い丸印がつきます。実行がその行で止まります。

設定方法： コード行の左端をクリック、または F9 キー

ステップ実行

コードを1行ずつ実行して、動作を確認できます。プログラムの流れを理解するのに役立ちます。

F8キー： ステップイン（1行ずつ実行、関数の中にも入る）
Shift + F8： ステップオーバー（1行ずつ実行、関数は飛ばす）

イミディエイトウィンドウ

プログラムの途中経過や変数の値を確認できます。デバッグの強力な味方です。

Debug.Printの使用例 
Sub デバッグテスト()
    Dim 名前 As String
    名前 = "山田太郎"
    Debug.Print "名前: " & 名前  ' イミディエイトウィンドウに表示
End Sub

3.7 実習：複数のメッセージを表示する

学んだ知識を使って、もう少し実践的なプログラムを書いてみましょう。

実習課題

以下の仕様を満たすプログラムを作成してください：

  1. 1. 「ようこそ」というメッセージを表示
  2. 2. 「Access VBAの学習を開始します」というメッセージを表示
  3. 3. イミディエイトウィンドウに「プログラム終了」と表示
解答例 
Sub 学習開始メッセージ()
    ' 1つ目のメッセージ
    MsgBox "ようこそ"

    ' 2つ目のメッセージ
    MsgBox "Access VBAの学習を開始します"

    ' イミディエイトウィンドウに出力
    Debug.Print "プログラム終了"
End Sub

ポイント

  • • 命令は上から順番に実行される
  • • MsgBoxは実行のたびにダイアログが表示される
  • • Debug.Printはイミディエイトウィンドウ（Ctrl+G）で確認
  • • コメント（'で始まる行）は実行されない

理解度確認クイズ

  1. Q1. VBEの4つの主要ウィンドウを答えてください。

    回答を表示

    プロジェクトエクスプローラー、プロパティウィンドウ、コードウィンドウ、イミディエイトウィンドウ

  2. Q2. 標準モジュールの役割は何ですか？

    回答を表示

    データベース全体で共通して使える汎用的なプログラムコードを保存する場所

  3. Q3. Subプロシージャを実行する3つの方法を挙げてください。

    回答を表示

    ① F5キーを押す、② メニュー「実行」→「Sub/ユーザーフォームの実行」、③ ツールバーの再生ボタン（▶）をクリック

  4. Q4. Debug.Printで出力した内容はどこに表示されますか？また、そのウィンドウを開くショートカットキーは何ですか？

    回答を表示

    イミディエイトウィンドウに表示される。ショートカットキーは Ctrl + G
