第4章

変数とデータ型の基礎

プログラミングの基本となる変数の概念、VBAで使用するデータ型の種類と使い分け、変数の宣言と代入方法を実例を通じて習得します。

この章で学ぶこと

  • 変数とは何か、なぜ必要なのかを理解する
  • VBAで使用する主なデータ型（Integer、String、Dateなど）を学ぶ
  • 変数の宣言（Dim文）と代入の方法を習得する
  • Option Explicit文によるバグ防止のベストプラクティスを理解する

4.1 変数とは何か？

変数とは、プログラムの中でデータを一時的に保存しておくための「箱」のようなものです。例えば、顧客の名前や売上金額などを一時的に記憶しておき、後で計算や表示に使うことができます。

graph LR A["変数名: customerName"] --> B["値: 山田太郎"] C["変数名: totalSales"] --> D["値: 150000"] style A fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style B fill:#d1fae5,stroke:#10b981,stroke-width:2px style C fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style D fill:#d1fae5,stroke:#10b981,stroke-width:2px

変数を使うことで、プログラムの中で値を何度も使い回したり、計算結果を保存したりすることができます。変数がないと、毎回同じデータを直接書かなければならず、プログラムが複雑になってしまいます。

変数を使うメリット

  • • データを再利用できる（同じ値を何度も書く必要がない）
  • • プログラムの可読性が向上する（意味のある名前をつけられる）
  • • 値の変更が簡単になる（1箇所変えれば全体に反映）
  • • 計算結果を保存して後で使える

4.2 VBAのデータ型

変数には「データ型」というものがあり、どのような種類のデータを保存するかを決めます。VBAには様々なデータ型がありますが、ここでは実務でよく使う主要な型を紹介します。

Integer（整数型）

範囲：-32,768 ～ 32,767の整数
用途：商品個数、年齢、順番など、小さな整数を扱う場合

整数型の例 
Dim age As Integer
age = 25

Dim quantity As Integer
quantity = 100

Long（長整数型）

範囲：-2,147,483,648 ～ 2,147,483,647の整数
用途：売上金額、ID番号、大きな数値を扱う場合

長整数型の例 
Dim totalSales As Long
totalSales = 15000000

Dim customerId As Long
customerId = 123456789

Double（倍精度浮動小数点数型）

範囲：小数点を含む非常に大きな数値
用途：価格、税率、割合など、小数を扱う場合

倍精度浮動小数点数型の例 
Dim price As Double
price = 1980.5

Dim taxRate As Double
taxRate = 0.1

String（文字列型）

範囲：文字列（テキスト）
用途：名前、住所、メッセージなど、文字を扱う場合

文字列型の例 
Dim customerName As String
customerName = "山田太郎"

Dim address As String
address = "東京都渋谷区"

Date（日付型）

範囲：日付と時刻
用途：生年月日、受注日、期限など、日時を扱う場合

日付型の例 
Dim birthDate As Date
birthDate = #1990/5/15#

Dim orderDate As Date
orderDate = Date ' 今日の日付

Boolean（ブール型）

範囲：True（真）または False（偽）のみ
用途：フラグ、スイッチ、条件判定の結果など

ブール型の例 
Dim isActive As Boolean
isActive = True

Dim isPaid As Boolean
isPaid = False

4.3 変数の宣言と代入

変数を使うには、まず「宣言」（変数を定義すること）してから「代入」（値を格納すること）する必要があります。VBAではDim文を使って変数を宣言します。

変数宣言の基本構文

基本構文 
Dim 変数名 As データ型
変数宣言と代入の具体例 
Sub VariableExample()
    ' 変数の宣言
    Dim customerName As String
    Dim age As Integer
    Dim totalPrice As Double

    ' 変数への代入
    customerName = "山田太郎"
    age = 35
    totalPrice = 15800.5

    ' 変数の値を使ってメッセージ表示
    MsgBox "顧客名: " & customerName & vbCrLf & _
           "年齢: " & age & vbCrLf & _
           "合計金額: " & totalPrice & "円"
End Sub

変数名の命名ルール

  • • 先頭は文字でなければならない（数字や記号で始めることはできない）
  • • スペースや記号（-、+など）は使えない（アンダースコア_は使える）
  • • 意味のある名前をつける（a、b、cより customerName、totalSalesなど）
  • • 大文字・小文字は区別されない（CustomerName と customername は同じ）

実習: 変数を使った簡単な計算プログラム

商品の単価と数量から、消費税込みの合計金額を計算するプログラムを作成してみましょう。

  1. 1 Visual Basic Editor（Alt + F11）を開く
  2. 2 「挿入」→「標準モジュール」で新しいモジュールを作成
  3. 3 以下のコードを入力
消費税計算プログラム 
Sub CalculateTotalPrice()
    ' 変数の宣言
    Dim unitPrice As Double      ' 単価
    Dim quantity As Integer       ' 数量
    Dim subtotal As Double        ' 小計
    Dim taxRate As Double         ' 税率
    Dim tax As Double             ' 消費税額
    Dim totalPrice As Double      ' 合計金額

    ' 値の代入
    unitPrice = 1500
    quantity = 3
    taxRate = 0.1

    ' 計算
    subtotal = unitPrice * quantity
    tax = subtotal * taxRate
    totalPrice = subtotal + tax

    ' 結果の表示
    MsgBox "単価: " & unitPrice & "円" & vbCrLf & _
           "数量: " & quantity & "個" & vbCrLf & _
           "小計: " & subtotal & "円" & vbCrLf & _
           "消費税: " & tax & "円" & vbCrLf & _
           "合計: " & totalPrice & "円"
End Sub
  1. 4 F5キーを押してプログラムを実行

4.4 Option Explicit文によるバグ防止

VBAでは、変数を宣言せずに使うこともできますが、これはバグの原因になります。例えば、変数名のスペルミスに気づかずにプログラムが誤動作する可能性があります。

宣言なしで変数を使った場合の問題例

悪い例（スペルミスに気づかない） 
Sub BadExample()
    customerName = "山田太郎"
    ' スペルミス！（customerName → custmerName）
    MsgBox custmerName ' 何も表示されない
End Sub

このような問題を防ぐため、モジュールの先頭にOption Explicit文を書くことが推奨されます。

Option Explicitを使った安全なコード

良い例（宣言を強制） 
Option Explicit ' モジュールの先頭に記述

Sub GoodExample()
    Dim customerName As String
    customerName = "山田太郎"
    ' スペルミスがあるとエラーになる
    MsgBox custmerName ' コンパイルエラー！
End Sub

Option Explicitを自動挿入する設定

VBEのメニューから「ツール」→「オプション」→「編集」タブ→「変数の宣言を強制する」にチェックを入れると、新しいモジュールに自動的にOption Explicitが挿入されます。

4.5 変数のスコープ（有効範囲）

変数には「スコープ」（どこまで使えるか）という概念があります。Dimで宣言した変数は、プロシージャ内でのみ有効です（ローカル変数）。

スコープの例 
Option Explicit

' モジュールレベル変数（このモジュール内のどこでも使える）
Private companyName As String

Sub Procedure1()
    ' ローカル変数（このプロシージャ内でのみ使える）
    Dim localVar As String
    localVar = "ローカル"
    companyName = "株式会社サンプル"

    MsgBox localVar       ' OK
    MsgBox companyName    ' OK
End Sub

Sub Procedure2()
    MsgBox localVar       ' エラー！localVarはProcedure1内でしか使えない
    MsgBox companyName    ' OK（モジュールレベル変数）
End Sub

初心者へのアドバイス

最初のうちは、Dimでローカル変数を使うことをお勧めします。グローバル変数は便利ですが、管理が難しくなりバグの原因になりやすいためです。

理解度確認クイズ

  1. Q1. 変数とは何ですか？簡潔に説明してください。

    回答を表示

    プログラムの中でデータを一時的に保存しておくための「箱」のようなもの。値を記憶して、後で計算や表示に使うことができる。

  2. Q2. 以下のデータを保存するのに適したデータ型を答えてください。
    （a）顧客の氏名、（b）商品の価格、（c）在庫数

    回答を表示

    （a）String、（b）Double、（c）Integer または Long

  3. Q3. 変数を宣言するときに使うキーワードは何ですか？

    回答を表示

    Dim（ローカル変数の場合）

  4. Q4. Option Explicit文を使う目的は何ですか？

    回答を表示

    変数の宣言を強制することで、スペルミスなどによるバグを防止するため。
