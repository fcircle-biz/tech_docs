第1章: Django入門と環境構築

Djangoフレームワークの世界へようこそ

この章で学ぶこと
  • Djangoとは何か、なぜWebアプリ開発に選ばれるのかを理解する
  • DjangoのMVT（Model-View-Template）アーキテクチャの基本概念を把握する
  • Python環境にDjangoをインストールし、開発環境を整備する
  • 初めてのDjangoプロジェクトを作成し、開発サーバーを起動する
  • Django管理サイトの初期設定と基本的な使い方を学ぶ

1.1 Djangoとは何か

Djangoは、Python言語で書かれた高水準なWebアプリケーションフレームワークです。2003年にローレンス・ジャーナル・ワールド新聞社で開始され、2005年にオープンソースとして公開されました。

Djangoの名前は、ジャズギタリストのDjango Reinhardtに由来しています。このフレームワークが目指すのは、「完璧主義者のためのWebフレームワーク」というコンセプトです。

Djangoの主な特徴

  • 包括的なフレームワーク: Webアプリケーション開発に必要な機能が最初から含まれている
  • DRY原則: "Don't Repeat Yourself" - コードの重複を避け、再利用性を重視
  • セキュリティ重視: CSRF、XSS、SQLインジェクション対策が標準で組み込まれている
  • スケーラビリティ: 小規模から大規模なWebサイトまで対応可能
  • 豊富なドキュメント: 詳細で分かりやすい公式ドキュメントが充実

Djangoが使われている有名なWebサイト

Djangoは世界中の多くの有名なWebサイトで使用されています：

  • Instagram: 写真・動画共有SNS
  • Spotify: 音楽ストリーミングサービス
  • YouTube: 一部の機能でDjangoを使用
  • Pinterest: 画像共有SNS
  • Mozilla: Firefoxブラウザの開発元

1.2 MVTアーキテクチャの理解

DjangoはMVT（Model-View-Template）アーキテクチャパターンを採用しています。これは一般的なMVC（Model-View-Controller）パターンをDjango流にアレンジしたものです。

graph TB A["ユーザーリクエスト
(HTTPリクエスト)"] --> B["URL設定
(urls.py)"] B --> C["View
(views.py)"] C --> D["Model
(models.py)"] D --> E["データベース
(SQLite/PostgreSQL等)"] E --> D D --> C C --> F["Template
(.html)"] F --> G["HTTPレスポンス
(HTMLページ)"] G --> H["ユーザーのブラウザ"]

各コンポーネントの役割

  • Model（モデル）: データベースとのやりとりを担当。データの構造と操作ロジックを定義
  • View（ビュー）: ユーザーからのリクエストを受け取り、処理の制御を行う。ビジネスロジックを実装
  • Template（テンプレート）: ユーザーに表示されるHTMLの構造を定義。動的なデータの表示を制御

1.3 開発環境の構築

Djangoでの開発を始めるために、Python環境の準備からDjangoのインストールまでを段階的に進めていきます。

前提条件の確認

重要な準備事項
  • Python 3.8以降がインストールされていること
  • pip（Pythonパッケージマネージャー）が利用可能であること
  • テキストエディタ（VS Code、PyCharm、Sublime Text等）が設定済みであること

まず、現在のPythonバージョンを確認しましょう：

# Pythonバージョンの確認
python --version

# 出力例: Python 3.9.7

仮想環境の作成（推奨）

プロジェクトごとに独立したPython環境を作成することで、パッケージの競合を防ぎ、プロジェクト管理を容易にします。 

# 仮想環境の作成
python -m venv django_env

# 仮想環境のアクティベート（Windows）
django_env\Scripts\activate

# 仮想環境のアクティベート（macOS/Linux）
source django_env/bin/activate

Djangoのインストール

仮想環境を有効化した状態で、Djangoをインストールします：

# Djangoの最新版をインストール
pip install django

# 特定のバージョンをインストールする場合
pip install django==4.2.0

# インストール確認
django-admin --version
実習 1-1: 初めてのDjangoプロジェクト作成

実際にDjangoプロジェクトを作成し、開発サーバーを起動してみましょう。

手順
  1. 新しいディレクトリを作成し、そこでDjangoプロジェクトを開始する
  2. プロジェクトの基本構造を確認する
  3. 開発サーバーを起動し、ブラウザでアクセスする
実行コマンド
# プロジェクト用ディレクトリの作成
mkdir my_django_project
cd my_django_project

# Djangoプロジェクトの作成
django-admin startproject mysite

# プロジェクトディレクトリに移動
cd mysite

# 開発サーバーの起動
python manage.py runserver
期待される結果

開発サーバーが起動し、ブラウザで http://127.0.0.1:8000/ にアクセスすると、 Djangoのウェルカムページ（ロケットのイラスト）が表示されます。

1.4 Djangoプロジェクトの構造理解

作成したDjangoプロジェクトの構造を詳しく見てみましょう。各ファイルの役割を理解することが重要です。 

mysite/                  # プロジェクトルートディレクトリ
├── manage.py            # Django管理コマンド用スクリプト
└── mysite/              # プロジェクト設定ディレクトリ
    ├── __init__.py      # Pythonパッケージ識別ファイル
    ├── settings.py      # プロジェクト設定ファイル
    ├── urls.py          # URL設定ファイル
    ├── wsgi.py          # 本番サーバー用設定
    └── asgi.py          # 非同期サーバー用設定

重要ファイルの詳細説明

  • manage.py: サーバー起動、マイグレーション実行など、様々な管理コマンドを実行するためのスクリプト
  • settings.py: データベース設定、タイムゾーン、デバッグモードなど、プロジェクト全体の設定を管理
  • urls.py: URLパターンとビュー関数の対応を定義するファイル
  • wsgi.py: Apache、Nginxなどの本番Webサーバーとの連携用
  • asgi.py: WebSocketや非同期処理に対応するための設定

1.5 Django管理サイトの設定

Djangoの大きな魅力の一つが、標準で提供される管理サイト機能です。データの閲覧・編集・削除を簡単なWeb画面で行うことができます。

データベースの初期化

管理サイトを使用するために、まずデータベースを初期化します：

# データベースマイグレーション（初回）
python manage.py migrate

# 出力例:
# Operations to perform:
#   Apply all migrations: admin, auth, contenttypes, sessions
# Running migrations:
#   Applying contenttypes.0001_initial... OK
#   Applying auth.0001_initial... OK
#   Applying admin.0001_initial... OK
#   Applying sessions.0001_initial... OK

管理者ユーザーの作成

管理サイトにログインするための管理者ユーザーを作成します：

# スーパーユーザーの作成
python manage.py createsuperuser

# ユーザー名、メールアドレス、パスワードを入力
# Username: admin
# Email address: admin@example.com
# Password: ********
# Password (again): ********
実習 1-2: Django管理サイトの利用

作成したスーパーユーザーでDjango管理サイトにログインし、基本的な操作を体験しましょう。

手順
  1. 開発サーバーを起動する
  2. ブラウザで管理サイトURL（/admin/）にアクセスする
  3. 作成したスーパーユーザー情報でログインする
  4. ユーザーとグループの管理画面を探索する
実行手順
# 開発サーバーの起動
python manage.py runserver

# ブラウザで以下にアクセス:
# http://127.0.0.1:8000/admin/
期待される結果

管理サイトのログイン画面が表示され、ログイン後は「Users」と「Groups」の管理項目が表示されます。 ここから新しいユーザーの作成や既存ユーザーの編集が可能です。

1.6 開発サイクルの理解

Django開発では、以下のような基本的な開発サイクルを繰り返します：

graph LR A["要件定義"] --> B["モデル設計"] B --> C["マイグレーション"] C --> D["ビュー作成"] D --> E["テンプレート作成"] E --> F["URL設定"] F --> G["テスト実行"] G --> H["デバッグ・改善"] H --> A

Djangoプロジェクトとアプリケーションの違い

Djangoでは「プロジェクト」と「アプリケーション」という概念を区別します：

  • プロジェクト: Webサイト全体の設定やURLルーティングを管理する最上位の単位
  • アプリケーション: 特定の機能を実装するモジュール（ブログ、ユーザー認証、商品管理など）

次章では、アプリケーションの作成とモデルの定義方法を学習します。

理解度確認クイズ
  1. DjangoのMVTアーキテクチャにおいて、データベースとの操作を担当するコンポーネントはどれですか？
    • a) View
    • b) Template
    • c) Model
    • d) URL設定
  2. Djangoプロジェクトを作成するコマンドは何ですか？
    • a) django-admin createproject
    • b) django-admin startproject
    • c) python manage.py startproject
    • d) pip install django
  3. Django開発サーバーが標準で使用するポート番号は何番ですか？
    • a) 3000
    • b) 5000
    • c) 8000
    • d) 8080
  4. Django管理サイトにアクセスするためのデフォルトURLパスは何ですか？
    • a) /admin/
    • b) /manage/
    • c) /dashboard/
    • d) /control/
  5. Djangoで仮想環境を使用する主な理由は何ですか？
    • a) 実行速度の向上
    • b) パッケージ競合の防止
    • c) セキュリティの強化
    • d) メモリ使用量の削減
解答を見る

1. c) Model - モデルがデータベース操作を担当
2. b) django-admin startproject - 新規プロジェクト作成コマンド
3. c) 8000 - Django開発サーバーの標準ポート
4. a) /admin/ - 管理サイトのデフォルトパス
5. b) パッケージ競合の防止 - プロジェクトごとの独立した環境構築
