1.1 FastAPIとは何か

FastAPIは、現代のPythonを使用して高性能なWeb APIを構築するためのモダンなWebフレームワークです。

FastAPIが生まれた背景

従来、Pythonで高性能なAPI開発を行うには、複数のライブラリを組み合わせ、複雑な設定を行う必要がありました。FastAPIは、このような課題を解決するために2018年にSebastián Ramírez氏によって開発されました。

graph TB A["Python Web開発の課題"] --> B["設定の複雑さ"] A --> C["パフォーマンスの問題"] A --> D["型安全性の欠如"] A --> E["ドキュメンテーションの手間"] F["FastAPIの解決策"] --> G["ゼロ設定で開始"] F --> H["高速な実行速度"] F --> I["型ヒントによる安全性"] F --> J["自動ドキュメント生成"]

FastAPIの核となる哲学

  • 高速性: Node.jsやGo言語に匹敵するパフォーマンス
  • 型安全性: Python 3.6+の型ヒントを活用した堅牢な開発
  • 自動化: ドキュメント生成、バリデーション、シリアライゼーションの自動化
  • 直感性: Pythonらしい読みやすく、書きやすいコード
  • 標準準拠: OpenAPI、JSON Schema等の標準規格への完全対応

1.2 FastAPIの主要な特徴とメリット

1. 圧倒的な高速性

FastAPIは内部でStarlette（ASGIフレームワーク）とPydantic（データバリデーションライブラリ）を使用することで、Node.jsやGoに匹敵するパフォーマンスを実現しています。

(バリデーション)"] D --> E["ビジネスロジック"] E --> F["Pydantic
(シリアライゼーション)"] F --> G["HTTPレスポンス"]

2. 自動ドキュメント生成

コードを書くだけで、SwaggerUIとReDocによる対話式のAPIドキュメントが自動的に生成されます。これにより、フロントエンド開発者やAPIクライアント開発者との連携が格段に向上します。

3. 型安全性によるコード品質向上

Python 3.6以降の型ヒント機能を積極的に活用することで、開発時のエラーを早期に発見でき、IDEの支援も充実します。

実世界での活用例
  • マイクロサービス: 軽量で高速なため、マイクロサービスアーキテクチャに最適
  • 機械学習API: データサイエンスチームのPythonモデルを簡単にAPI化
  • IoTバックエンド: 高スループットが要求されるIoTデータ収集システム
  • 内部ツール: 企業内の業務自動化ツールのAPIバックエンド

1.4 開発環境の構築

前提条件の確認

FastAPIを使用するためには、Python 3.8以降が必要です。まず現在のPythonバージョンを確認しましょう。

実習 1-1: Python環境の確認

ターミナルまたはコマンドプロンプトでPythonのバージョンを確認します。

手順
  1. ターミナル（Windowsの場合はコマンドプロンプトまたはPowerShell）を開きます
  2. 以下のコマンドを実行してPythonのバージョンを確認します
実行例
# Pythonバージョンの確認
python --version
# または
python3 --version

# pipのバージョンも確認
pip --version
期待される結果

Python 3.8.0 以降のバージョンが表示されることを確認してください。古いバージョンの場合は、Python公式サイトから最新版をインストールしてください。

仮想環境の作成

Pythonプロジェクトでは、プロジェクトごとに独立した仮想環境を作成することがベストプラクティスです。これにより、プロジェクト間でのパッケージの競合を防ぐことができます。

実習 1-2: 仮想環境の作成とアクティベート

venvを使用してプロジェクト専用の仮想環境を作成します。

手順
  1. プロジェクト用のディレクトリを作成します
  2. そのディレクトリに移動します
  3. 仮想環境を作成し、アクティベートします
実行例
# プロジェクトディレクトリの作成
mkdir my-fastapi-project
cd my-fastapi-project

# 仮想環境の作成
python -m venv fastapi-env

# 仮想環境のアクティベート
# Windows の場合
fastapi-env\Scripts\activate

# macOS/Linux の場合
source fastapi-env/bin/activate
期待される結果

プロンプトの先頭に (fastapi-env) が表示され、仮想環境がアクティブになったことが確認できます。

1.5 FastAPIのインストール

FastAPIは2つのインストール方法があります：

  • 基本インストール: FastAPI本体のみ
  • 完全インストール: サーバー（Uvicorn）やその他の推奨パッケージも含む
実習 1-3: FastAPIのインストール

開発に必要なパッケージをすべてインストールします。

手順
  1. 仮想環境がアクティベートされていることを確認します
  2. pipを最新版にアップデートします
  3. FastAPIとUvicornをインストールします
実行例
# pipのアップデート
pip install --upgrade pip

# FastAPIと開発用サーバーのインストール
pip install "fastapi[all]" uvicorn[standard]

# インストール確認
pip list | grep fastapi
pip list | grep uvicorn
期待される結果

FastAPIとUvicorn、および関連パッケージ（Pydantic、Starlette等）がインストールされます。

インストールオプションの説明
  • fastapi[all]: すべての推奨依存関係を含む完全インストール
  • uvicorn[standard]: 高性能なASGIサーバーUvicornの標準版
  • これにより、開発に必要なすべてのツールが一度にインストールされます

1.6 最初のFastAPIアプリケーションを作成

それでは、実際にFastAPIアプリケーションを作成してみましょう。まずはシンプルな "Hello World" APIから始めます。

実習 1-4: Hello World APIの作成

最も基本的なFastAPIアプリケーションを作成します。

手順
  1. プロジェクトディレクトリに main.py ファイルを作成します
  2. 基本的なFastAPIコードを記述します
  3. 開発サーバーを起動します
  4. ブラウザでAPIにアクセスします
実装例（main.py）
# fastapi をインポート
from fastapi import FastAPI

# FastAPIアプリケーションのインスタンスを作成
app = FastAPI()

# ルートエンドポイントの定義
@app.get("/")
def read_root():
    """
    ルートパス（/）にアクセスした時の処理
    """
    return {"message": "Hello World"}

# 健康チェック用エンドポイント
@app.get("/health")
def health_check():
    """
    アプリケーションの健康状態をチェック
    """
    return {"status": "healthy"}
サーバーの起動
# 開発サーバーを起動
uvicorn main:app --reload

# --reload オプションによりコード変更時に自動リロードされます
期待される結果

サーバーが起動し、http://127.0.0.1:8000 でAPIにアクセスできるようになります。

1.7 自動生成されるAPIドキュメント

FastAPIの最大の特徴の一つは、コードから自動的にAPIドキュメントが生成されることです。

実習 1-5: 自動ドキュメントの確認

自動生成されるSwagger UIとReDocを確認します。

手順
  1. サーバーが起動していることを確認します
  2. ブラウザで以下のURLにアクセスします
アクセスURL
  • Swagger UI: http://127.0.0.1:8000/docs
  • ReDoc: http://127.0.0.1:8000/redoc
  • OpenAPI Schema: http://127.0.0.1:8000/openapi.json
期待される結果

対話式のAPIドキュメントが表示され、ブラウザから直接APIをテストできます。

自動ドキュメントの詳細
  • Swagger UI: 対話式でAPIを試すことができる
  • ReDoc: 読みやすい形式でAPIドキュメントを表示
  • OpenAPI Schema: 機械読み取り可能なAPI仕様（JSON形式）
  • これらはすべてコードから自動生成されるため、ドキュメントの更新忘れがありません

1.8 効率的な開発ワークフロー

FastAPI開発での効率的なワークフローを理解しましょう。

graph TD A["コード変更"] --> B["自動リロード（--reload）"] B --> C["API動作確認"] C --> D["Swagger UIでテスト"] D --> E["問題があるか？"] E -->|Yes| F["デバッグ・修正"] E -->|No| G["次の機能実装"] F --> A G --> A

推奨開発環境の設定

  • エディタ: VS Code、PyCharm等のPython対応IDE
  • 拡張機能: Python、Pylance（VS Codeの場合）
  • 型チェック: mypyやPylanceによる静的型チェック
  • コードフォーマット: BlackやPrettierでコード整形
実習 1-6: 開発環境の最適化

開発効率を向上させるための追加ツールをインストールします。

手順
  1. 開発に便利なツールをインストールします
  2. 設定ファイルを作成します
実行例
# 開発用ツールのインストール
pip install black isort mypy

# requirements.txtの作成（依存関係の管理）
pip freeze > requirements.txt
期待される結果

コード品質向上のためのツールが使用でき、プロジェクトの依存関係が管理できるようになります。

理解度確認クイズ
  1. FastAPIの主要な特徴を3つ挙げてください。
    解答例

    1. 高速性（Node.js、Goに匹敵するパフォーマンス）
    2. 自動ドキュメント生成（Swagger UI、ReDoc）
    3. 型安全性（Python型ヒントの活用）

  2. FastAPIで自動生成されるドキュメントにアクセスするデフォルトのパスは何ですか？
    解答例

    Swagger UI: /docs
    ReDoc: /redoc
    OpenAPI Schema: /openapi.json

  3. Uvicornで開発サーバーを起動する際、コード変更時に自動リロードを有効にするオプションは何ですか？
    解答例

    --reloadオプション
    例: uvicorn main:app --reload

  4. なぜPython開発では仮想環境を使用することが推奨されるのですか？
    解答例

    プロジェクトごとに独立した環境を作ることで：
    - パッケージバージョンの競合を防ぐ
    - システムPythonを汚さない
    - プロジェクトの依存関係を明確にする
    - 環境の再現性を確保する
