1.1 Streamlitとは何か

Streamlitは、Pythonでデータサイエンス・機械学習アプリケーションを簡単に作成・共有できるオープンソースのWebアプリケーションフレームワークです。

Streamlitの主な特徴
  • シンプルな構文: PythonコードだけでWebアプリを作成
  • リアルタイム更新: コード変更が即座にアプリに反映
  • 豊富なコンポーネント: グラフ、表、入力要素を簡単に配置
  • データサイエンス特化: pandas、NumPy、matplotlibなどとの連携
  • 簡単デプロイ: Streamlit Cloudで簡単に公開

他のWebフレームワークとの比較

フレームワーク 学習コスト 開発速度 データサイエンス向け 用途
Streamlit 低 高 ◎ プロトタイプ、ダッシュボード
Flask 中 中 ○ 一般的なWebアプリ
Django 高 低 △ 大規模Webアプリ

1.2 環境構築とインストール

前提条件の確認

必要な環境
  • Python 3.8以降
  • pip（Pythonパッケージマネージャー）
  • テキストエディタ（VS Code、PyCharm等）
  • Webブラウザ

Pythonバージョンの確認

# ターミナル/コマンドプロンプトで実行
python --version

# または
python3 --version

Streamlitのインストール

# pipを使用してインストール
pip install streamlit

# または仮想環境内で
pip install streamlit

# 特定のバージョンを指定する場合
pip install streamlit==1.28.0
仮想環境の使用を推奨

プロジェクトごとに独立した環境を作成することで、パッケージの競合を避けることができます。

# 仮想環境の作成
python -m venv streamlit_env

# 仮想環境の有効化（Windows）
streamlit_env\Scripts\activate

# 仮想環境の有効化（Mac/Linux）
source streamlit_env/bin/activate

1.3 最初のStreamlitアプリケーション

実習 1-1: Hello World アプリを作成

Streamlitの基本的な仕組みを理解するため、最初のアプリケーションを作成します。

手順
  1. 新しいフォルダ streamlit_tutorial を作成
  2. app.py ファイルを作成
  3. 以下のコードを入力
  4. アプリケーションを実行
app.py のコード
import streamlit as st

# ページの設定
st.set_page_config(
    page_title="私の最初のStreamlitアプリ",
    page_icon="🎉",
    layout="wide"
)

# タイトルの表示
st.title("🎉 Streamlitへようこそ！")

# ヘッダーの表示
st.header("これは私の最初のWebアプリです")

# テキストの表示
st.write("Streamlitを使えば、Pythonだけで簡単にWebアプリが作れます！")

# マークダウンの表示
st.markdown("""
### Streamlitでできること
- データの可視化
- インタラクティブなダッシュボード
- 機械学習モデルのデモ
- レポートの自動生成
""")

# 成功メッセージ
st.success("環境構築が完了しました！🎊")
実行方法
# ターミナルでプロジェクトフォルダに移動
cd streamlit_tutorial

# Streamlitアプリを実行
streamlit run app.py
期待される結果

ブラウザが自動的に開き、以下のような画面が表示されます：

  • タイトル「Streamlitへようこそ！」
  • ヘッダーとテキスト
  • マークダウンで記述されたリスト
  • 緑色の成功メッセージ

1.4 Streamlitの基本的な仕組み

Streamlitアプリの実行サイクル

Streamlitアプリは上から下へ順次実行される仕組みです：

  1. コード実行: Pythonスクリプトを上から下へ実行
  2. UI更新: 実行結果をブラウザに表示
  3. インタラクション待機: ユーザーのアクション（ボタンクリック等）を待機
  4. 再実行: アクションが発生すると、スクリプト全体が再実行

主要なStreamlitコマンド

コマンド 用途 例
st.title() ページタイトルの表示 st.title("アプリタイトル")
st.header() ヘッダーの表示 st.header("セクションタイトル")
st.write() 汎用的なテキスト表示 st.write("テキスト")
st.markdown() マークダウン形式のテキスト st.markdown("**太字**")
st.set_page_config() ページの設定 st.set_page_config(page_title="タイトル")
実習 1-2: 基本コマンドの練習

様々なStreamlitコマンドを使って、テキスト表示の違いを確認します。

新しいファイル text_demo.py を作成
import streamlit as st

# ページ設定
st.set_page_config(page_title="テキスト表示デモ", page_icon="📝")

# 様々なテキスト表示方法
st.title("📝 テキスト表示のデモ")

st.header("これはヘッダーです")
st.subheader("これはサブヘッダーです")

st.write("st.write() は最も汎用的な表示コマンドです")
st.text("st.text() は等幅フォントでテキストを表示します")

# マークダウンの例
st.markdown("""
### マークダウンの例
- **太字**
- *斜体*
- `コード`
- [リンク](https://streamlit.io)

1. 番号付きリスト
2. 項目2
3. 項目3
""")

# コードブロック
st.code("""
def hello_streamlit():
    print("Hello, Streamlit!")
    return "アプリ作成完了"
""", language="python")

# 情報メッセージ
st.info("これは情報メッセージです")
st.success("これは成功メッセージです")
st.warning("これは警告メッセージです")
st.error("これはエラーメッセージです")
実行と確認
streamlit run text_demo.py
期待される結果

異なるテキスト表示方法と、カラフルなメッセージボックスが表示されます。

注意: Hot Reload機能

Streamlitは自動リロード機能があります。ファイルを保存すると、ブラウザ右上に「Rerun」ボタンが表示されます。クリックするとアプリが更新されます。

理解度確認クイズ
  1. Streamlitの特徴として正しくないものはどれですか？
    • a) PythonコードだけでWebアプリを作成できる
    • b) HTMLとCSSの深い知識が必要
    • c) リアルタイムでアプリが更新される
    • d) データサイエンス向けのコンポーネントが豊富
    解答を見る

    答え: b) StreamlitはPythonだけで開発でき、HTMLやCSSの深い知識は不要です。

  2. Streamlitアプリを実行するコマンドは何ですか？
    • a) python app.py
    • b) streamlit app.py
    • c) streamlit run app.py
    • d) python -m streamlit app.py
    解答を見る

    答え: c) streamlit run app.py がStreamlitアプリを実行する正しいコマンドです。

  3. st.write()とst.text()の違いは何ですか？
    解答を見る

    答え: st.write()は汎用的な表示コマンドで、データ型に応じて適切な形式で表示します。st.text()は等幅フォントでプレーンテキストとして表示します。

  4. Streamlitアプリの実行サイクルを順番に並べてください
    • ① インタラクション待機
    • ② コード実行
    • ③ UI更新
    • ④ 再実行
    解答を見る

    答え: ② → ③ → ① → ④ の順番です。コード実行 → UI更新 → インタラクション待機 → 再実行のサイクルです。

🔧 トラブルシューティングガイド

解決策:

  • Pythonのバージョンを確認（Python 3.7以上が必要）
  • pipをアップデート: pip install --upgrade pip
  • 仮想環境を使用: python -m venv streamlit_env
  • ネットワーク制限がある場合: pip install streamlit --user

解決策:

  • 代替コマンドを使用: python -m streamlit run app.py
  • PATHにPythonのScriptsディレクトリを追加
  • 仮想環境をアクティベートしているか確認
  • Streamlitが正しくインストールされているか確認: pip list | grep streamlit

解決策:

  • ターミナルに表示されたURLを確認（通常: http://localhost:8501）
  • ファイアウォールがポート8501をブロックしていないか確認
  • 別のポートを指定: streamlit run app.py --server.port 8502
  • ブラウザのキャッシュをクリア

解決策:

  • ブラウザでページを手動リロード（F5）
