7.1 トランザクションの基本

トランザクションは、データベースの一貫性を保つための重要な概念です。ACID特性（Atomicity, Consistency, Isolation, Durability）を理解し、適切なトランザクション管理を学習します。

実習 7-1: トランザクションの実装
実装例
# トランザクションの基本的な使い方
with SessionLocal() as session:
    try:
        # 複数の操作を一つのトランザクションで実行
        user = User(name='田中', email='tanaka@example.com')
        session.add(user)
        
        order = Order(user_id=user.id, amount=1000)
        session.add(order)
        
        # 全て成功した場合のみコミット
        session.commit()
        print('トランザクション成功')
        
    except Exception as e:
        # エラー発生時はロールバック
        session.rollback()
        print(f'エラー発生: {e}')
        raise

7.2 エラーハンドリング

SQLAlchemyでの一般的なエラータイプとその対処方法を学習します。

from sqlalchemy.exc import (
    IntegrityError,      # 制約違反
    NoResultFound,       # 結果が見つからない
    MultipleResultsFound, # 複数の結果が見つかった
    SQLAlchemyError      # 一般的なSQLAlchemyエラー
)

try:
    # 重複キーの挿入を試行
    user = User(email='existing@example.com')
    session.add(user)
    session.commit()
except IntegrityError as e:
    session.rollback()
    print(f'一意制約違反: {e}')
トランザクション使用時の注意点
  • デッドロックの回避と適切なタイムアウト設定
  • 長時間実行されるトランザクションの回避
  • 適切な分離レベルの設定
  • リトライ機構の実装
理解度確認クイズ
  1. ACID特性の4つの要素を説明し、それぞれがなぜ重要か述べてください。
  2. デッドロックが発生するシナリオとその対策を説明してください。
  3. IntegrityErrorが発生する主な原因を3つ挙げ、それぞれの対処方法を説明してください。
