1.1 SQLModelの概要

SQLModelはSebastian Ramirez（FastAPIの作者）によって開発された、PythonのORM（Object-Relational Mapping）ライブラリです。SQLAlchemyとPydanticの長所を組み合わせており、以下の特徴があります：

  • 型安全性: Python型ヒントを活用した完全な型安全性
  • Pydantic統合: 自動的なデータ検証とシリアライゼーション
  • FastAPI連携: シームレスなFastAPI統合
  • SQLAlchemy Core: 成熟したSQLAlchemyエンジンを基盤とする安定性
  • コード重複の排除: データベースモデルとAPIスキーマの統一

1.2 従来の課題とSQLModelの解決策

従来のPython Web開発では、データベースモデル（SQLAlchemy）とAPIスキーマ（Pydantic）を別々に定義する必要がありました。SQLModelはこの課題を解決します：

従来のアプローチ
# SQLAlchemyモデル
class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True)
    name = Column(String)
    email = Column(String)

# Pydanticスキーマ
class UserCreate(BaseModel):
    name: str
    email: str

class UserResponse(BaseModel):
    id: int
    name: str
    email: str
SQLModelアプローチ
# 統一されたモデル
class User(SQLModel, table=True):
    id: Optional[int] = Field(primary_key=True)
    name: str
    email: str

# 追加のスキーマクラスは不要！
# このクラスがデータベースモデルとAPIスキーマの両方として機能

1.4 最初のSQLModelモデル

実習 1-2: 基本的なモデル定義

シンプルなユーザーモデルを作成し、SQLModelの基本的な使い方を学びます。

手順
  1. main.pyファイルを作成
  2. SQLModelをインポート
  3. Userモデルクラスを定義
  4. データベースエンジンとテーブル作成
実行例 (main.py)
from typing import Optional
from sqlmodel import SQLModel, Field, create_engine, Session

# モデル定義
class User(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    name: str = Field(max_length=100)
    email: str = Field(unique=True, max_length=255)
    age: Optional[int] = Field(default=None, ge=0, le=150)

# データベース設定
DATABASE_URL = "sqlite:///./test.db"
engine = create_engine(DATABASE_URL, echo=True)

def create_db_and_tables():
    SQLModel.metadata.create_all(engine)

if __name__ == "__main__":
    create_db_and_tables()
    print("データベースとテーブルが作成されました！")
期待される結果

SQLiteデータベースファイル（test.db）が作成され、usersテーブルが生成されます。

1.5 基本的なデータ操作

実習 1-3: データの作成と読み取り

SQLModelを使って基本的なCRUD操作の「Create」と「Read」を実装します。

手順
  1. Userインスタンスを作成
  2. データベースセッションを使ってデータを保存
  3. 保存されたデータを読み取り
  4. 結果を確認
実行例 (crud_basic.py)
from main import User, engine
from sqlmodel import Session, select

def create_user():
    # ユーザー作成
    user = User(name="田中太郎", email="tanaka@example.com", age=30)
    
    # データベースに保存
    with Session(engine) as session:
        session.add(user)
        session.commit()
        session.refresh(user)
        print(f"ユーザーが作成されました: ID {user.id}")
        return user

def read_users():
    # 全ユーザーを取得
    with Session(engine) as session:
        statement = select(User)
        users = session.exec(statement).all()
        for user in users:
            print(f"ID: {user.id}, 名前: {user.name}, メール: {user.email}, 年齢: {user.age}")
        return users

if __name__ == "__main__":
    # ユーザー作成
    create_user()
    
    # ユーザー一覧表示
    read_users()
期待される結果
ユーザーが作成されました: ID 1
ID: 1, 名前: 田中太郎, メール: tanaka@example.com, 年齢: 30

1.6 型安全性の確認

SQLModelの最大の特徴である型安全性を確認しましょう。IDE（VS Code、PyCharmなど）では、以下のような型チェックとオートコンプリートが機能します：

# 型安全性の例
user = User(name="山田花子", email="yamada@example.com", age=25)

# IDEでオートコンプリートが効く
print(user.name)    # str型として認識
print(user.age)     # Optional[int]型として認識

# 型エラーの検出
user.age = "thirty"  # mypy/pylanceが型エラーを検出
注意事項
  • SQLModelはまだ開発中のライブラリです（0.x版）
  • 本番環境での使用前に十分なテストを実施してください
  • SQLAlchemyの知識があるとより深く理解できます
理解度確認クイズ
  1. SQLModelが解決する主な課題は何ですか？
  2. SQLModel, table=Trueの意味は何ですか？
  3. Field()パラメータのprimary_key=Trueの役割は何ですか？
  4. データベースセッション（Session）を使う理由は何ですか？
  5. SQLModelの型安全性がもたらすメリットを3つ挙げてください。
解答例
  1. データベースモデルとAPIスキーマの重複定義を解決し、型安全性の確保
  2. このクラスがデータベーステーブルに対応することを示す
  3. このフィールドがテーブルの主キーであることを定義
  4. データベースとの接続を管理し、トランザクションを適切に処理するため
  5. コンパイル時の型チェック、IDEでのオートコンプリート、実行時エラーの減少
