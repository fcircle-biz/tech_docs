第11章

クラスとオブジェクト指向の基礎を学ぼう

C#の特徴であるオブジェクト指向プログラミングの基本概念を学びます。クラス、オブジェクト、プロパティ、コンストラクタについて、身近な例を使って分かりやすく解説します。

この章で学ぶこと

  • オブジェクト指向プログラミングとは何かを理解する
  • クラスとオブジェクトの関係を理解し、クラスを定義できる
  • プロパティを使ってデータを安全に管理する方法を学ぶ
  • コンストラクタでオブジェクトを初期化する方法を理解する

11.1 オブジェクト指向プログラミングとは

オブジェクト指向プログラミング（Object-Oriented Programming, OOP）は、 現実世界の「もの（オブジェクト）」を基準にプログラムを組み立てる考え方です。

身近な例で理解しよう

例えば、「犬」について考えてみましょう。
- データ: 名前、年齢、犬種
- 行動: 吠える、走る、食べる
オブジェクト指向では、これらをまとめて「犬クラス」として定義します。

オブジェクト指向のメリット

コードの再利用

一度作ったクラスを何度でも使い回せるため、効率よくプログラムを作成できます。

データの保護

クラス内部のデータを隠蔽し、必要な操作だけを外部に公開できます。

保守性の向上

関連するデータと処理がまとまっているため、修正や拡張がしやすくなります。

理解しやすさ

現実世界の「もの」として考えられるため、プログラムの構造が理解しやすくなります。

11.2 クラスとオブジェクトの基本

クラス（Class）は、オブジェクトの「設計図」や「ひな型」です。
オブジェクト（Object）は、クラスを基に作られた「実体」です。

たい焼きの例えで理解しよう

flowchart LR A[たい焼きの型] -->|焼く| B[たい焼き1] A -->|焼く| C[たい焼き2] A -->|焼く| D[たい焼き3] style A fill:#ddd6fe,stroke:#7c3aed,stroke-width:2px style B fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style C fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style D fill:#fef3c7,stroke:#f59e0b,stroke-width:2px

クラス（設計図） = たい焼きの型
オブジェクト（実体） = 実際に焼かれたたい焼き
1つの型から、何個でもたい焼きを作れます。同じように、1つのクラスから複数のオブジェクトを生成できます。

クラスの定義

C#でクラスを定義する基本構文は以下の通りです。

Dog.cs（クラス定義の例） 
// クラスの定義
class Dog
{
    // フィールド（データ）
    public string Name;
    public int Age;

    // メソッド（行動）
    public void Bark()
    {
        Console.WriteLine($"{Name}が吠えています: ワンワン！");
    }
}

オブジェクトの生成と使用

クラスからオブジェクトを生成するには、new キーワードを使います。

Program.cs（オブジェクトの生成） 
class Program
{
    static void Main(string[] args)
    {
        // オブジェクトの生成
        Dog myDog = new Dog();

        // データの設定
        myDog.Name = "ポチ";
        myDog.Age = 3;

        // メソッドの呼び出し
        myDog.Bark();
        // 出力: ポチが吠えています: ワンワン！
    }
}

重要ポイント

同じクラスから複数のオブジェクトを作れます。それぞれのオブジェクトは独立したデータを持ちます。

複数のオブジェクト生成 
// 複数のDogオブジェクトを生成
Dog dog1 = new Dog();
dog1.Name = "ポチ";
dog1.Age = 3;

Dog dog2 = new Dog();
dog2.Name = "タロウ";
dog2.Age = 5;

dog1.Bark(); // ポチが吠えています: ワンワン！
dog2.Bark(); // タロウが吠えています: ワンワン！

11.3 プロパティでデータを安全に管理する

プロパティ（Property）は、クラスのデータへのアクセスを制御する仕組みです。 直接フィールドを公開するのではなく、プロパティを使うことでデータを安全に扱えます。

フィールドの問題点

先ほどの例では、フィールド（Name, Age）をpublicで公開していました。 しかし、これには問題があります。

問題のある例 
Dog myDog = new Dog();
myDog.Age = -5;  // 負の値を設定できてしまう！（現実ではありえない）

プロパティの定義

プロパティを使えば、値の検証やルールを適用できます。

Dog.cs（プロパティを使った改良版） 
class Dog
{
    // プライベートフィールド（外部から直接アクセスできない）
    private string name;
    private int age;

    // プロパティ（Name）
    public string Name
    {
        get { return name; }        // 値を取得
        set { name = value; }       // 値を設定
    }

    // プロパティ（Age）- 検証付き
    public int Age
    {
        get { return age; }
        set
        {
            if (value >= 0)  // 0以上の値のみ許可
            {
                age = value;
            }
            else
            {
                Console.WriteLine("年齢は0以上を指定してください");
            }
        }
    }

    public void Bark()
    {
        Console.WriteLine($"{Name}が吠えています: ワンワン！");
    }
}

自動プロパティ（簡潔な書き方）

検証が不要な場合、自動プロパティを使うとコードを簡潔に書けます。

自動プロパティの例 
class Dog
{
    // 自動プロパティ（簡潔）
    public string Name { get; set; }
    public int Age { get; set; }

    public void Bark()
    {
        Console.WriteLine($"{Name}が吠えています: ワンワン！");
    }
}

プロパティ使い分けのヒント

- 値の検証が必要な場合: 完全なプロパティ（get/setを明示）を使用
- 単純にデータを保持するだけ: 自動プロパティを使用

11.4 コンストラクタでオブジェクトを初期化する

コンストラクタ（Constructor）は、オブジェクトが作られるときに自動的に呼ばれる特別なメソッドです。 初期設定を行うために使います。

コンストラクタの定義

Dog.cs（コンストラクタ付き） 
class Dog
{
    public string Name { get; set; }
    public int Age { get; set; }

    // コンストラクタ（クラス名と同じ名前のメソッド）
    public Dog(string name, int age)
    {
        Name = name;
        Age = age;
        Console.WriteLine($"{Name}（{Age}歳）が生まれました！");
    }

    public void Bark()
    {
        Console.WriteLine($"{Name}が吠えています: ワンワン！");
    }
}

コンストラクタを使ったオブジェクト生成

Program.cs（コンストラクタの使用） 
class Program
{
    static void Main(string[] args)
    {
        // コンストラクタで初期値を設定
        Dog myDog = new Dog("ポチ", 3);
        // 出力: ポチ（3歳）が生まれました！

        myDog.Bark();
        // 出力: ポチが吠えています: ワンワン！
    }
}

コンストラクタの特徴

  • メソッド名はクラス名と同じにする
  • 戻り値の型を書かない（voidも不要）
  • オブジェクト生成時（new）に自動的に呼ばれる
  • 複数のコンストラクタを定義できる（オーバーロード）

複数のコンストラクタ（オーバーロード）

複数のコンストラクタ例 
class Dog
{
    public string Name { get; set; }
    public int Age { get; set; }

    // コンストラクタ1: 引数なし
    public Dog()
    {
        Name = "名無し";
        Age = 0;
    }

    // コンストラクタ2: 引数あり
    public Dog(string name, int age)
    {
        Name = name;
        Age = age;
    }

    public void Bark()
    {
        Console.WriteLine($"{Name}が吠えています: ワンワン！");
    }
}

実習 11-1: Personクラスを作成しよう

人を表すPersonクラスを作成し、プロパティとコンストラクタを実装してみましょう。

手順

  1. 1 Personクラスを作成し、Name（名前）とAge（年齢）のプロパティを定義
  2. 2 コンストラクタで初期値を設定できるようにする
  3. 3 SayHello()メソッドを作成し、「こんにちは、〇〇です。△歳です。」と表示する
  4. 4 Main()メソッドでPersonオブジェクトを2つ作成し、それぞれSayHello()を呼び出す
解答例を表示 
class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    // コンストラクタ
    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }

    // 挨拶メソッド
    public void SayHello()
    {
        Console.WriteLine($"こんにちは、{Name}です。{Age}歳です。");
    }
}

class Program
{
    static void Main(string[] args)
    {
        // Personオブジェクトを2つ作成
        Person person1 = new Person("田中太郎", 25);
        Person person2 = new Person("鈴木花子", 30);

        // 挨拶メソッドを呼び出し
        person1.SayHello();
        person2.SayHello();
    }
}

11.5 クラス設計の考え方

良いクラス設計をするには、現実世界の「もの」を観察し、その特徴と行動を整理することが大切です。

flowchart TD A[現実世界のもの] --> B{何を表現したい？} B --> C[データ: プロパティ] B --> D[行動: メソッド] C --> E[クラス設計] D --> E E --> F[オブジェクト生成] style A fill:#f5f3ff,stroke:#7c3aed style B fill:#ddd6fe,stroke:#7c3aed style C fill:#fef3c7,stroke:#f59e0b style D fill:#fef3c7,stroke:#f59e0b style E fill:#a7f3d0,stroke:#10b981 style F fill:#bfdbfe,stroke:#3b82f6

クラス設計の例：Carクラス

現実世界のもの: 車

データ（プロパティ）

  • • メーカー（Maker）
  • • 車種（Model）
  • • 色（Color）
  • • 速度（Speed）

行動（メソッド）

  • • 走る（Drive）
  • • 止まる（Stop）
  • • クラクションを鳴らす（Honk）
Car.cs（設計例） 
class Car
{
    // プロパティ（データ）
    public string Maker { get; set; }
    public string Model { get; set; }
    public string Color { get; set; }
    public int Speed { get; set; }

    // コンストラクタ
    public Car(string maker, string model, string color)
    {
        Maker = maker;
        Model = model;
        Color = color;
        Speed = 0;  // 最初は停止状態
    }

    // メソッド（行動）
    public void Drive(int speed)
    {
        Speed = speed;
        Console.WriteLine($"{Color}の{Maker} {Model}が{Speed}km/hで走っています");
    }

    public void Stop()
    {
        Speed = 0;
        Console.WriteLine($"{Color}の{Maker} {Model}が停止しました");
    }

    public void Honk()
    {
        Console.WriteLine("プップー！");
    }
}

理解度確認クイズ

  1. Q1. クラスとオブジェクトの違いを説明してください。

    回答を表示

    クラスは「設計図」や「ひな型」で、オブジェクトはその設計図から作られた「実体」です。
    例：たい焼きの型（クラス）と実際に焼かれたたい焼き（オブジェクト）

  2. Q2. プロパティを使う利点は何ですか？

    回答を表示

    プロパティを使うと、値を設定・取得する際にルールや検証を適用できます。
    例：年齢に負の値を設定できないようにする、など。
    データを安全に管理できるのが大きな利点です。

  3. Q3. コンストラクタの役割は何ですか？

    回答を表示

    コンストラクタは、オブジェクトが作られるときに自動的に呼ばれる特別なメソッドです。
    主な役割は、オブジェクトの初期設定（プロパティの初期値設定など）を行うことです。

  4. Q4. 以下のコードの実行結果を答えてください。

    Dog myDog = new Dog("ポチ", 5);
myDog.Bark();
    回答を表示

    実行結果:
    ポチ（5歳）が生まれました！
    ポチが吠えています: ワンワン！
