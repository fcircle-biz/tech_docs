第12章

例外処理とエラーへの対応を学ぼう

try-catch文を使った例外処理の基本を学びます。プログラムの実行中に発生するエラーに適切に対応し、堅牢なプログラムを作成する方法を理解します。

この章で学ぶこと

  • エラーと例外の違い、例外が発生する理由を理解する
  • try-catch-finally文の基本構文と使い方を学ぶ
  • よく発生する例外の種類とその対処法を理解する
  • 適切な例外処理を書いて、エラーに強いプログラムを作成する

12.1 エラーと例外について理解しよう

プログラムを実行していると、予期しない問題が発生することがあります。これをエラーと呼びます。 C#では、実行中に発生する問題を例外（Exception）として扱います。

エラーと例外の違い

プログラミングにおける「エラー」には、大きく分けて2種類あります。

コンパイルエラー

プログラムを実行する前に見つかる文法の間違いです。

例: セミコロン忘れ、スペルミス、型の不一致など

実行時エラー（例外）

プログラム実行中に発生する予期しない問題です。

例: ゼロ除算、存在しないファイルの読み込み、nullへのアクセスなど

例外とは何か

例外（Exception）は、プログラム実行中に発生した異常な状態を表すオブジェクトです。
例外が発生すると、プログラムは通常の処理を中断し、例外を処理する仕組み（例外処理）に制御を移します。

例外が発生する典型的な状況

ゼロで割り算をする

数値を0で割ると DivideByZeroException が発生

存在しないファイルを開こうとする

指定したファイルが見つからないと FileNotFoundException が発生

nullのオブジェクトを使おうとする

nullのプロパティやメソッドにアクセスすると NullReferenceException が発生

文字列を数値に変換できない

「abc」を数値に変換しようとすると FormatException が発生

flowchart TD A[プログラム実行開始] --> B{処理実行} B -->|正常| C[次の処理へ] B -->|異常発生| D[例外が発生] D --> E{例外処理はあるか？} E -->|ある| F[例外処理を実行] F --> G[プログラム継続] E -->|ない| H[プログラム強制終了]

12.2 try-catch文の基本

例外が発生してもプログラムが強制終了しないように、try-catch文を使って例外を処理します。

try-catch文の基本構文

C# 
try
{
    // 例外が発生する可能性のある処理
    // この中で問題が起きると、catchブロックに移動する
}
catch (例外の型 変数名)
{
    // 例外が発生したときに実行する処理
    // エラーメッセージの表示や代替処理を書く
}

try-catchの動作の流れ

  1. tryブロック内の処理を順番に実行します
  2. 例外が発生すると、その時点で処理を中断し、catchブロックに移動します
  3. catchブロックで例外を処理します
  4. catchブロックの実行後、プログラムは次の処理を続けます

具体例: ゼロ除算の例外処理

C# 
using System;

class Program
{
    static void Main()
    {
        try
        {
            // ゼロ除算が発生する処理
            int a = 10;
            int b = 0;
            int result = a / b;  // ここで例外が発生
            Console.WriteLine(result);  // この行は実行されない
        }
        catch (DivideByZeroException ex)
        {
            // 例外が発生したときの処理
            Console.WriteLine("エラー: ゼロで割ることはできません");
        }

        Console.WriteLine("プログラムは継続します");
    }
}

実行結果

エラー: ゼロで割ることはできません
プログラムは継続します

例外が発生しても、プログラムは強制終了せず、catchブロックで適切に処理されました。

12.3 複数のcatchブロックとfinally

複数のcatchブロックを使う

異なる種類の例外に対して、それぞれ異なる処理を行いたい場合、複数のcatchブロックを書くことができます。

C# 
using System;

class Program
{
    static void Main()
    {
        try
        {
            Console.Write("数値を入力: ");
            string input = Console.ReadLine();
            int number = int.Parse(input);  // 文字列を数値に変換
            int result = 100 / number;  // 割り算を実行
            Console.WriteLine($"結果: {result}");
        }
        catch (FormatException)
        {
            // 数値に変換できない文字列が入力された場合
            Console.WriteLine("エラー: 数値を入力してください");
        }
        catch (DivideByZeroException)
        {
            // ゼロで割ろうとした場合
            Console.WriteLine("エラー: ゼロでは割れません");
        }
    }
}

finallyブロック - 必ず実行される処理

finallyブロックは、例外が発生してもしなくても必ず実行される処理を書く場所です。 ファイルを閉じたり、データベース接続を切断したりする後処理によく使われます。

C# 
try
{
    // 例外が発生する可能性のある処理
    Console.WriteLine("処理を開始します");
    int result = 10 / 0;  // 例外発生
}
catch (DivideByZeroException)
{
    // 例外処理
    Console.WriteLine("エラーが発生しました");
}
finally
{
    // 例外の有無にかかわらず必ず実行される
    Console.WriteLine("処理を終了します");
}

実行結果

処理を開始します
エラーが発生しました
処理を終了します

例外が発生したにもかかわらず、finallyブロックは必ず実行されました。

flowchart TD A[try開始] --> B{処理実行} B -->|例外なし| C[tryブロック完了] B -->|例外発生| D[catchブロック実行] C --> E[finallyブロック実行] D --> E E --> F[次の処理へ]

12.4 よくある例外の種類と対処法

C#では、さまざまな種類の例外が定義されています。代表的な例外とその対処法を理解しましょう。

DivideByZeroException

発生原因: 整数をゼロで割ろうとしたとき

対処法: 割り算の前に、割る数がゼロでないか確認する

NullReferenceException

発生原因: nullのオブジェクトにアクセスしようとしたとき

対処法: オブジェクトを使う前に、nullでないか確認する（null条件演算子?.を使うのも有効）

FormatException

発生原因: 文字列を数値に変換できなかったとき

対処法: int.TryParse()を使って、変換可能か事前に確認する

IndexOutOfRangeException

発生原因: 配列やリストの範囲外にアクセスしようとしたとき

対処法: インデックスが配列の長さ以内かを確認する

FileNotFoundException

発生原因: 指定したファイルが存在しないとき

対処法: File.Exists()でファイルの存在を確認してから処理する

12.5 実践的な例外処理のパターン

パターン1: ユーザー入力の検証

ユーザーからの入力を受け取るときは、想定外の値が入力される可能性があります。 TryParseメソッドを使うと、例外を発生させずに安全に変換できます。

C# 
using System;

class Program
{
    static void Main()
    {
        Console.Write("年齢を入力してください: ");
        string input = Console.ReadLine();

        // TryParseを使った安全な変換
        if (int.TryParse(input, out int age))
        {
            if (age >= 0 && age <= 120)
            {
                Console.WriteLine($"あなたは{age}歳です");
            }
            else
            {
                Console.WriteLine("年齢は0から120の範囲で入力してください");
            }
        }
        else
        {
            Console.WriteLine("数値を入力してください");
        }
    }
}

パターン2: 一般的な例外のキャッチ

予期しない例外をすべてキャッチするには、Exception型でcatchします。 ただし、具体的な例外を先に処理し、最後に一般的な例外をキャッチする順序が重要です。

C# 
try
{
    // 何らかの処理
}
catch (DivideByZeroException ex)
{
    // 具体的な例外を先に処理
    Console.WriteLine("ゼロ除算エラー");
}
catch (FormatException ex)
{
    // 別の具体的な例外を処理
    Console.WriteLine("形式エラー");
}
catch (Exception ex)
{
    // 最後に一般的な例外をキャッチ
    Console.WriteLine($"予期しないエラー: {ex.Message}");
}

注意点

catchブロックの順序は重要です。具体的な例外を先に、一般的な例外を後に書きます。
一般的な例外（Exception）を最初に書いてしまうと、後の具体的な例外が実行されなくなります。

パターン3: 例外メッセージの活用

例外オブジェクトには、エラーの詳細情報が含まれています。ex.Messageプロパティでエラーメッセージを取得できます。

C# 
try
{
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[10]);  // 範囲外アクセス
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine("配列の範囲外アクセスエラー");
    Console.WriteLine($"詳細: {ex.Message}");
}

実習: 安全な計算機プログラム

ユーザーから2つの数値を入力してもらい、割り算を行うプログラムを作成しましょう。 以下の例外処理を実装してください：

  1. 数値以外が入力されたときのエラー処理（FormatException）
  2. ゼロで割ろうとしたときのエラー処理（DivideByZeroException）
  3. 予期しないエラーへの対応（Exception）
サンプルコードを表示
C# 
using System;

class Program
{
    static void Main()
    {
        try
        {
            Console.Write("1つ目の数値: ");
            int num1 = int.Parse(Console.ReadLine());

            Console.Write("2つ目の数値: ");
            int num2 = int.Parse(Console.ReadLine());

            int result = num1 / num2;
            Console.WriteLine($"結果: {result}");
        }
        catch (FormatException)
        {
            Console.WriteLine("エラー: 数値を入力してください");
        }
        catch (DivideByZeroException)
        {
            Console.WriteLine("エラー: ゼロでは割れません");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"予期しないエラー: {ex.Message}");
        }
        finally
        {
            Console.WriteLine("計算処理を終了しました");
        }
    }
}

12.6 例外をスローする

プログラムの中で異常な状態を検出したとき、自分で例外を発生させることができます。 これを例外をスロー（throw）すると言います。

throw文の基本

C# 
using System;

class Program
{
    static void Main()
    {
        try
        {
            SetAge(-5);  // 負の年齢を設定しようとする
        }
        catch (ArgumentException ex)
        {
            Console.WriteLine($"エラー: {ex.Message}");
        }
    }

    static void SetAge(int age)
    {
        // 年齢が負の場合は例外をスローする
        if (age < 0)
        {
            throw new ArgumentException("年齢は0以上でなければなりません");
        }
        Console.WriteLine($"年齢を{age}に設定しました");
    }
}

throw文を使う場面

- 関数に渡された引数が不正な場合
- オブジェクトが無効な状態の場合
- これ以上処理を続けられない状況の場合
throw文を使うことで、呼び出し元に「問題が発生した」ことを明確に伝えられます。

12.7 例外処理のベストプラクティス

例外処理を適切に使うことで、エラーに強い堅牢なプログラムを作成できます。 以下のポイントを意識しましょう。

1. 予測可能なエラーは事前にチェックする

例外処理に頼りすぎず、if文で事前にチェックできることはチェックしましょう。 例: ゼロ除算の前にif (divisor != 0)で確認する

2. 具体的な例外をキャッチする

すべてをExceptionでキャッチするのではなく、想定される具体的な例外をそれぞれ処理しましょう。 エラーの原因が明確になり、適切な対処ができます。

3. ユーザーにわかりやすいメッセージを表示する

例外メッセージをそのまま表示するのではなく、ユーザーにとって分かりやすい言葉で説明しましょう。 例: 「System.DivideByZeroException」→「ゼロで割ることはできません」

4. 例外を無視しない

catchブロックを空にして例外を無視することは避けましょう。 少なくともログに記録するか、ユーザーに通知するべきです。

理解度確認クイズ

  1. Q1. 例外とは何ですか？簡単に説明してください。

    回答を表示

    例外とは、プログラム実行中に発生した異常な状態を表すオブジェクトです。 例外が発生すると、通常の処理が中断され、例外処理に制御が移ります。

  2. Q2. try-catch文の役割は何ですか？

    回答を表示

    try-catch文は、例外が発生してもプログラムが強制終了しないように、例外を適切に処理する仕組みです。
    tryブロックで例外が発生する可能性のある処理を実行し、catchブロックで例外が発生したときの処理を書きます。

  3. Q3. finallyブロックはいつ実行されますか？

    回答を表示

    finallyブロックは、例外が発生してもしなくても必ず実行されます。
    ファイルを閉じたり、リソースを解放したりする後処理に使われます。

  4. Q4. よくある例外を3つ挙げてください。

    回答を表示

    - DivideByZeroException（ゼロ除算）
    - NullReferenceException（nullへのアクセス）
    - FormatException（文字列変換エラー）
    （他にもIndexOutOfRangeException、FileNotFoundExceptionなども正解です）

この章のまとめ

  • 例外はプログラム実行中に発生する異常な状態を表すオブジェクト
  • try-catch文を使うと、例外が発生してもプログラムを継続できる
  • finallyブロックは例外の有無に関わらず必ず実行される
  • 具体的な例外を先にキャッチし、最後に一般的な例外をキャッチする
  • throw文を使って自分で例外を発生させることができる
