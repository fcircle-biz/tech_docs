13.4 データ構造の設計（クラス作成）

収支データを管理するために、Transactionクラスを作成します。このクラスには、日付、カテゴリ、金額、メモの情報を保存します。

重要ポイント

「Transaction」は「取引」という意味です。収入と支出の両方を表すデータとして使います。

C# 
// 収支データを表すクラス
public class Transaction
{
    // 日付
    public DateTime Date { get; set; }
    // カテゴリ（食費、交通費など）
    public string Category { get; set; }
    // 金額（プラスは収入、マイナスは支出）
    public int Amount { get; set; }
    // メモ
    public string Memo { get; set; }

    // コンストラクタ
    public Transaction(DateTime date, string category, int amount, string memo)
    {
        Date = date;
        Category = category;
        Amount = amount;
        Memo = memo;
    }
}

13.6 収支登録機能

ユーザーから日付、カテゴリ、金額、メモを入力してもらい、Listに追加する機能を作成します。

C# 
// 収支を登録するメソッド
static void AddTransaction(List<Transaction> transactions)
{
    Console.WriteLine("\n--- 収支登録 ---");

    // 日付入力
    Console.Write("日付 (yyyy/MM/dd): ");
    DateTime date = DateTime.Parse(Console.ReadLine());

    // カテゴリ入力
    Console.Write("カテゴリ (例: 食費、交通費): ");
    string category = Console.ReadLine();

    // 金額入力
    Console.Write("金額 (収入はプラス、支出はマイナス): ");
    int amount = int.Parse(Console.ReadLine());

    // メモ入力
    Console.Write("メモ: ");
    string memo = Console.ReadLine();

    // Transactionオブジェクトを作成してリストに追加
    Transaction transaction = new Transaction(date, category, amount, memo);
    transactions.Add(transaction);

    Console.WriteLine("✓ 登録しました！\n");
}

13.7 収支一覧表示機能

登録された収支データを一覧で表示するメソッドを作成します。

C# 
// 収支一覧を表示するメソッド
static void ShowTransactions(List<Transaction> transactions)
{
    Console.WriteLine("\n--- 収支一覧 ---");

    if (transactions.Count == 0)
    {
        Console.WriteLine("まだデータがありません。\n");
        return;
    }

    // ヘッダー表示
    Console.WriteLine("日付       | カテゴリ   | 金額      | メモ");
    Console.WriteLine("---------------------------------------------");

    // 各収支データを表示
    foreach (Transaction t in transactions)
    {
        Console.WriteLine($"{t.Date:yyyy/MM/dd} | {t.Category,-10} | {t.Amount,8}円 | {t.Memo}");
    }
    Console.WriteLine();
}

13.8 収支集計機能

収入の合計、支出の合計、残高を計算して表示するメソッドを作成します。

C# 
// 収支を集計するメソッド
static void ShowSummary(List<Transaction> transactions)
{
    Console.WriteLine("\n--- 収支集計 ---");

    if (transactions.Count == 0)
    {
        Console.WriteLine("まだデータがありません。\n");
        return;
    }

    int totalIncome = 0;   // 収入合計
    int totalExpense = 0;  // 支出合計

    // 各取引を集計
    foreach (Transaction t in transactions)
    {
        if (t.Amount > 0)
        {
            totalIncome += t.Amount;
        }
        else
        {
            totalExpense += t.Amount;
        }
    }

    int balance = totalIncome + totalExpense;  // 残高

    Console.WriteLine($"収入合計: {totalIncome}円");
    Console.WriteLine($"支出合計: {totalExpense}円");
    Console.WriteLine($"残高: {balance}円\n");
}

13.9 エラー処理の実装

ユーザーが間違った入力をした場合にプログラムが停止しないように、try-catch文を使ってエラー処理を追加します。

C# 
// エラー処理を追加した収支登録メソッド
static void AddTransaction(List<Transaction> transactions)
{
    Console.WriteLine("\n--- 収支登録 ---");

    try
    {
        Console.Write("日付 (yyyy/MM/dd): ");
        DateTime date = DateTime.Parse(Console.ReadLine());

        Console.Write("カテゴリ (例: 食費、交通費): ");
        string category = Console.ReadLine();

        Console.Write("金額 (収入はプラス、支出はマイナス): ");
        int amount = int.Parse(Console.ReadLine());

        Console.Write("メモ: ");
        string memo = Console.ReadLine();

        Transaction transaction = new Transaction(date, category, amount, memo);
        transactions.Add(transaction);

        Console.WriteLine("✓ 登録しました！\n");
    }
    catch (FormatException)
    {
        Console.WriteLine("✗ 入力形式が正しくありません。\n");
    }
}

理解度確認クイズ

  1. Q1. Transactionクラスのプロパティを4つ挙げてください。

    回答を表示

    1. Date（日付）
    2. Category（カテゴリ）
    3. Amount（金額）
    4. Memo（メモ）

  2. Q2. 家計簿アプリで収支データを保存するために使ったコレクションの型は何ですか？

    回答を表示

    List<Transaction>
    （Transactionオブジェクトのリスト）

  3. Q3. 収支集計メソッドで、収入と支出を区別する方法を説明してください。

    回答を表示

    Amountプロパティの値が0より大きければ収入、0以下なら支出として扱います。
    if文で「t.Amount > 0」という条件で判定しています。

  4. Q4. なぜtry-catch文を使ってエラー処理を追加したのですか？

    回答を表示

    ユーザーが日付や金額を間違った形式で入力した場合に、プログラムが停止（クラッシュ）しないようにするためです。
    エラーが発生してもメッセージを表示して継続できるようになります。
