はじめてのプログラムを作ろう

「Hello World」プログラムを実際に作成し、プログラムの入力から実行までの流れを体験します。C#の基本的な構文とエラーが出たときの対処法も合わせて学びます。

この章で学ぶこと

  • 新しいC#コンソールアプリケーションプロジェクトの作成方法
  • 「Hello World」プログラムを実際に書いて実行する
  • C#プログラムの基本構造（namespace、class、Mainメソッド）を理解する
  • エラーが発生したときの対処法を学ぶ

4.1 新しいプロジェクトを作成しよう

まずは、Visual Studioで新しいC#プロジェクトを作成します。 第3章で学んだ操作を復習しながら進めましょう。

実習 4-1: プロジェクトを作成する

最初のC#プログラムを作成するためのプロジェクトを準備します。

手順

  1. 1 Visual Studioを起動し、「新しいプロジェクトの作成」をクリック
  2. 2 検索ボックスに「コンソール」と入力し、「コンソール アプリ (.NET)」を選択して「次へ」
  3. 3 プロジェクト名に「HelloWorld」、場所は任意のフォルダ（例: C:\Users\自分の名前\Documents）を指定して「次へ」
  4. 4 フレームワークは「.NET 8.0」を選択し、「作成」をクリック

ヒント

コンソールアプリとは、黒い画面（コマンドプロンプト）で動くプログラムのことです。
見た目はシンプルですが、プログラミングの基本を学ぶには最適です。

4.2 Hello Worldプログラムを書こう

プロジェクトが作成されると、自動的にコードが表示されます。 「Program.cs」というファイルに、すでにシンプルなプログラムが書かれています。

初期状態のコード

プロジェクトを作成すると、以下のようなコードが表示されます。

Program.cs 
// コメント: プログラムの開始地点
Console.WriteLine("Hello, World!");

重要ポイント

これは「トップレベルステートメント」と呼ばれる、C# 9.0以降の新しい書き方です。
初心者にとって余計なコードを書かなくて済むため、学習しやすくなっています。

コードの意味を理解しよう

このシンプルなコードには、重要な要素が含まれています。

Console.WriteLine()

画面に文字を表示するための命令です。
カッコ( )の中に書いた文字が、コンソールに表示されます。

例: Console.WriteLine("こんにちは"); → 画面に「こんにちは」と表示

"Hello, World!"（文字列）

ダブルクォーテーション（"）で囲まれた部分は「文字列」と呼ばれます。
プログラムで扱う文字のことです。

注意: 文字列は必ず " " で囲む必要があります

セミコロン（;）

C#では、ほとんどの文の最後にセミコロン（;）を付けます。
これは「この命令はここで終わり」という意味です。

忘れると「エラー」になるので注意！

4.3 プログラムを実行しよう

コードが書けたら、実際に実行してみましょう。

実習 4-2: プログラムを実行する

初めてのプログラムを実行して、結果を確認します。

手順

  1. 1 画面上部のツールバーから、緑色の「▶ HelloWorld」ボタンをクリック（または F5 キーを押す）
  2. 2 黒い画面（コンソール）が開き、「Hello, World!」と表示されます
  3. 3 コンソールは自動的に閉じます（または Enter キーを押すと閉じます）

動作確認

プログラムを実行すると、コンソールに以下のように表示されます。

期待される結果 

Hello, World!

4.4 プログラムを改造してみよう

「Hello, World!」が表示できたら、次は自分の好きなメッセージを表示してみましょう。

実習 4-3: メッセージを変更する

表示する文字列を変更して、プログラムを改造してみます。

手順

  1. 1 Program.cs の「Hello, World!」の部分を「こんにちは、C#の世界へようこそ!」に変更
  2. 2 次の行に Console.WriteLine("プログラミングは楽しい!"); を追加
  3. 3 保存（Ctrl + S）してから、再度実行（F5）
Program.cs（改造版） 
// メッセージを2行表示するプログラム
Console.WriteLine("こんにちは、C#の世界へようこそ!");
Console.WriteLine("プログラミングは楽しい!");

ヒント

Console.WriteLine() を複数回書くと、その回数だけメッセージが表示されます。
コードは上から順番に実行されるので、書いた順番に表示されます。

4.5 エラーに対処しよう

プログラミングでは、エラーは避けられません。 エラーメッセージを読んで、修正する力を身につけましょう。

よくあるエラーと対処法

トラブルシューティング

エラー: セミコロン（;）が抜けている

原因: 文の最後にセミコロンを付け忘れた

エラーメッセージ例: CS1002: ; が必要です

解決方法: エラーが出ている行の最後に ; を追加します

エラー: ダブルクォーテーション（"）が閉じていない

原因: 文字列の開始 " はあるが、終了 " がない

エラーメッセージ例: CS1010: 改行がリテラル内にあります

解決方法: 文字列の最後に " を追加します

エラー: スペルミス（Console を Consol と書いた）

原因: 命令の名前を間違えて入力した

エラーメッセージ例: CS0103: 名前 'Consol' は現在のコンテキスト内に存在しません

解決方法: 正しいスペル（Console）に修正します。C#では大文字・小文字も区別されます

エラー: カッコ( )が閉じていない

原因: 開きカッコ ( はあるが、閉じカッコ ) がない

エラーメッセージ例: CS1026: ) が必要です

解決方法: 開きカッコと閉じカッコの数が一致するように修正します

重要ポイント

エラーは「間違っている場所」と「何が問題か」を教えてくれる親切なメッセージです。
エラーメッセージをよく読めば、ほとんどの問題は自分で解決できるようになります。

4.6 プログラムの基本構造を理解しよう（補足）

これまで使ってきた「トップレベルステートメント」は、初心者向けの簡略版です。
従来の書き方も知っておくと、他の教材やサンプルコードを読むときに役立ちます。

従来の書き方

従来のC#プログラムは、以下のような構造になっています。

Program.cs（従来の書き方） 
namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, World!");
        }
    }
}

namespace（名前空間）

プログラムのコードを整理するための「フォルダ」のようなものです。
複数のクラスを分類して管理できます。

class（クラス）

プログラムの「設計図」です。
ここにプログラムの処理（メソッド）をまとめます。

Main メソッド（エントリーポイント）

プログラムが実行されるとき、最初に実行される場所です。
「プログラムの入口」とも呼ばれます。

ヒント

最初のうちは「トップレベルステートメント」で十分です。
将来的にクラスやオブジェクト指向を学ぶときに、従来の書き方を詳しく理解すればOKです。

理解度確認クイズ

  1. Q1. 画面に文字を表示するための命令は何ですか?

    回答を表示

    Console.WriteLine() です。
    カッコの中に表示したい文字列を書きます。

  2. Q2. C#のプログラムで、ほとんどの文の最後に付ける記号は何ですか?

    回答を表示

    セミコロン（;）です。
    これを付けないとエラーになります。

  3. Q3. 文字列を表す場合、どの記号で囲む必要がありますか?

    回答を表示

    ダブルクォーテーション（"）で囲みます。
    例: "こんにちは"

  4. Q4. エラーメッセージが表示されたとき、どうすればよいですか?

    回答を表示

    エラーメッセージをよく読んで、どこに問題があるかを確認します。
    セミコロンの抜け、スペルミス、カッコの閉じ忘れなどをチェックしましょう。
