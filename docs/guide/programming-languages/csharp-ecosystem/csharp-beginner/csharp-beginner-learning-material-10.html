メソッドで処理をまとめよう

メソッド（関数）の作成方法を学びます。引数と戻り値の概念を理解し、処理を再利用可能な形にまとめる技術を身につけます。

この章で学ぶこと

  • メソッドとは何か、なぜ必要なのかを理解する
  • メソッドの基本的な書き方とルールを学ぶ
  • 引数（パラメータ）と戻り値の概念を理解する
  • 実際にメソッドを作成して処理を再利用できるようになる

10.1 メソッドとは何か

メソッドとは、複数の処理をひとまとまりにして名前をつけたものです。 他のプログラミング言語では「関数（function）」と呼ばれることもあります。

日常生活で例えると

メソッドは、料理の「レシピ」のようなものです。
例えば「カレーを作る」という名前のレシピ（メソッド）には、「野菜を切る」「肉を炒める」「ルーを入れる」といった手順がまとまっています。
一度レシピを作っておけば、何度でも同じ手順でカレーを作れます。

なぜメソッドが必要なのか

プログラムを書いていると、同じような処理を何度も書くことがあります。 メソッドを使うと、以下のようなメリットがあります。

コードの再利用

一度書いた処理を何度も使い回せます。同じコードを何度も書く必要がありません。

読みやすさの向上

処理に名前をつけることで、プログラムが何をしているか分かりやすくなります。

修正が簡単

メソッドの中身を修正すれば、そのメソッドを使っている全ての場所に反映されます。

分割統治

複雑な処理を小さな単位に分けることで、プログラム全体を理解しやすくなります。

メソッドの基本的な流れ

flowchart LR A[メソッドを呼び出す] --> B[メソッド内の処理が実行される] B --> C[結果が返される（戻り値がある場合）] C --> D[呼び出し元に戻る]

10.2 メソッドの基本的な書き方

まずは、最もシンプルな形のメソッドから学びましょう。

Program.cs - 基本的なメソッドの例 
using System;

class Program
{
    static void Main()
    {
        // メソッドを呼び出す
        SayHello();
        SayHello(); // 何度でも呼び出せる
        SayHello();
    }

    // メソッドの定義（引数なし、戻り値なし）
    static void SayHello()
    {
        Console.WriteLine("こんにちは！");
    }
}

実行結果

こんにちは！
こんにちは！
こんにちは！

メソッドの構造を理解しよう

メソッドの各部分の意味

static 現時点では「おまじない」として覚えてください。クラスを学ぶと意味が分かります。
void 戻り値がないことを表します。「何も返さない」という意味です。
SayHello メソッドの名前です。処理内容が分かる名前をつけます。
() 引数（パラメータ）を書く場所です。今は空なので「引数なし」という意味です。
{} 中かっこの中に、実際の処理を書きます。

重要ポイント

メソッド名は必ず大文字で始めるのがC#の慣習です（例：SayHello, CalculateSum）。
また、メソッドを呼び出すときは必ずかっこ「()」をつけることを忘れないでください。

10.3 引数（パラメータ）を受け取るメソッド

引数（ひきすう）は、メソッドに渡す「材料」のようなものです。 メソッドに情報を渡すことで、より柔軟な処理ができるようになります。

Program.cs - 引数を使ったメソッド 
using System;

class Program
{
    static void Main()
    {
        // 引数に異なる値を渡して呼び出す
        SayHelloTo("太郎");
        SayHelloTo("花子");
        SayHelloTo("次郎");
    }

    // 引数を1つ受け取るメソッド
    static void SayHelloTo(string name)
    {
        Console.WriteLine($"こんにちは、{name}さん！");
    }
}

実行結果

こんにちは、太郎さん！
こんにちは、花子さん！
こんにちは、次郎さん！

複数の引数を受け取る

メソッドは複数の引数を受け取ることもできます。引数はカンマ「,」で区切って指定します。

Program.cs - 複数の引数 
using System;

class Program
{
    static void Main()
    {
        // 2つの引数を渡す
        ShowProfile("太郎", 25);
        ShowProfile("花子", 30);
    }

    // 2つの引数を受け取るメソッド
    static void ShowProfile(string name, int age)
    {
        Console.WriteLine($"名前: {name}");
        Console.WriteLine($"年齢: {age}歳");
        Console.WriteLine(); // 空行
    }
}

ヒント

引数を渡すときは、定義したときと同じ順番、同じ型で渡す必要があります。
例えば「ShowProfile(25, "太郎")」とすると、型が合わずにエラーになります。

10.4 戻り値を返すメソッド

戻り値（もどりち）は、メソッドが処理した結果を呼び出し元に返すものです。 計算機能などでよく使われます。

Program.cs - 戻り値を返すメソッド 
using System;

class Program
{
    static void Main()
    {
        // メソッドの戻り値を変数に代入
        int result = Add(5, 3);
        Console.WriteLine($"5 + 3 = {result}");

        // 直接使うこともできる
        Console.WriteLine($"10 + 20 = {Add(10, 20)}");
    }

    // int型の戻り値を返すメソッド
    static int Add(int a, int b)
    {
        int sum = a + b;
        return sum; // 結果を返す
    }
}

returnキーワードの役割

returnは、メソッドから値を返すためのキーワードです。

  • returnを実行すると、その時点でメソッドが終了します
  • returnの後ろに書いた値が、呼び出し元に返されます
  • 戻り値の型は、メソッド定義の型（この例ではint）と一致する必要があります

いろいろな戻り値の例

Program.cs - さまざまな戻り値 
using System;

class Program
{
    static void Main()
    {
        // int型の戻り値
        int total = Multiply(4, 7);
        Console.WriteLine($"4 × 7 = {total}");

        // bool型の戻り値
        bool isAdult = IsAdult(25);
        Console.WriteLine($"25歳は成人？ {isAdult}");

        // string型の戻り値
        string greeting = GetGreeting("午前");
        Console.WriteLine(greeting);
    }

    // int型を返すメソッド
    static int Multiply(int x, int y)
    {
        return x * y;
    }

    // bool型を返すメソッド
    static bool IsAdult(int age)
    {
        return age >= 18; // 18歳以上ならtrue
    }

    // string型を返すメソッド
    static string GetGreeting(string timeOfDay)
    {
        if (timeOfDay == "午前")
            return "おはようございます！";
        else if (timeOfDay == "午後")
            return "こんにちは！";
        else
            return "こんばんは！";
    }
}

10.5 メソッドを使う理由とメリット

実際のプログラムでメソッドがどれだけ便利かを、具体例で見てみましょう。

悪い例：メソッドを使わない場合

Program.cs - 同じ処理を繰り返し書いている 
using System;

class Program
{
    static void Main()
    {
        // 太郎の消費税込み金額計算
        int price1 = 1000;
        int taxIncluded1 = price1 + (price1 * 10 / 100);
        Console.WriteLine($"太郎: {taxIncluded1}円");

        // 花子の消費税込み金額計算
        int price2 = 1500;
        int taxIncluded2 = price2 + (price2 * 10 / 100);
        Console.WriteLine($"花子: {taxIncluded2}円");

        // 次郎の消費税込み金額計算
        int price3 = 800;
        int taxIncluded3 = price3 + (price3 * 10 / 100);
        Console.WriteLine($"次郎: {taxIncluded3}円");
    }
}

良い例：メソッドを使った場合

Program.cs - メソッドで処理をまとめる 
using System;

class Program
{
    static void Main()
    {
        // メソッドを使えばシンプルに書ける
        Console.WriteLine($"太郎: {CalculateTax(1000)}円");
        Console.WriteLine($"花子: {CalculateTax(1500)}円");
        Console.WriteLine($"次郎: {CalculateTax(800)}円");
    }

    // 消費税込み金額を計算するメソッド
    static int CalculateTax(int price)
    {
        return price + (price * 10 / 100);
    }
}

メソッドを使うメリット

  • • コードが短くて読みやすい
  • • 消費税率が変わったとき、メソッドの中だけ修正すればOK
  • • 計算ミスがあっても、1箇所直せば全体に反映される
  • • 他のプログラムでも同じメソッドを使い回せる

メソッドの設計のコツ

分かりやすい名前をつける

メソッド名を見ただけで、何をするメソッドか分かるようにしましょう。
例：Calculate（計算する）、Show（表示する）、Check（確認する）

1つのことだけをする

1つのメソッドは1つの役割だけを持たせましょう。
複数の処理をする場合は、メソッドを分割します。

適切な長さにする

メソッドが長すぎると理解しにくくなります。
目安として、画面に収まる程度（20〜30行以内）が理想的です。

コメントを書く

メソッドが何をするのか、引数は何を表すのかをコメントで説明すると、後で見返したときに分かりやすくなります。

実習課題

以下の課題に挑戦して、メソッドの理解を深めましょう。

  1. 課題1: 2つの整数を受け取り、大きい方を返すメソッド「GetMax」を作成してください。
  2. 課題2: 名前と点数を受け取り、「〇〇さんの点数は△△点です」と表示するメソッド「ShowScore」を作成してください。
  3. 課題3: 整数を受け取り、その数が偶数ならtrue、奇数ならfalseを返すメソッド「IsEven」を作成してください。

理解度確認クイズ

  1. Q1. メソッドを使うメリットを3つ挙げてください。

    回答を表示

    1. コードの再利用：同じ処理を何度も書かずに済む
    2. 読みやすさの向上：処理に名前がつくので分かりやすい
    3. 修正が簡単：メソッドの中身を変えれば全体に反映される
    （他にも「分割統治」なども正解です）

  2. Q2. 引数と戻り値の違いを説明してください。

    回答を表示

    引数は、メソッドに渡す「材料」のようなもので、メソッドを呼び出すときに値を渡します。
    戻り値は、メソッドが処理した「結果」を呼び出し元に返すものです。
    引数は入力、戻り値は出力と考えると分かりやすいです。

  3. Q3. 以下のメソッド定義で間違っている部分を指摘してください。
    static int Calculate()

    回答を表示

    文法的には正しいですが、戻り値の型が「int」なのに、メソッド名が「Calculate（計算する）」だけでは、何を計算するのか分かりにくいです。
    より良い名前は「CalculateSum（合計を計算）」や「CalculateTax（税金を計算）」など、具体的な名前をつけるべきです。
    また、引数がないので、何を計算するのか不明確です。

  4. Q4. 「void」とはどういう意味ですか？

    回答を表示

    「void」は「何も返さない」という意味です。
    戻り値がないメソッド（例：画面に表示するだけで結果を返さないメソッド）の場合に使います。
    voidのメソッドでは、return文は不要です（書いてもOKですが、値は返せません）。

この章のまとめ

  • メソッドは、処理をまとめて名前をつけたもので、コードの再利用や読みやすさを向上させる
  • 引数は、メソッドに渡す「材料」で、メソッドをより柔軟にする
  • 戻り値は、メソッドが処理した「結果」を呼び出し元に返すもの
  • メソッドを使うことで、保守性・可読性・再利用性が大幅に向上する
