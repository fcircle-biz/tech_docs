第5章

変数とデータ型を理解しよう

データを一時的に保存する「変数」の概念と、数値や文字列などのデータ型について学びます。int、string、boolなど、C#でよく使うデータ型を実際のプログラムで使ってみます。

この章で学ぶこと

  • 変数とは何か、なぜ必要なのかを理解する
  • 変数の宣言方法と代入の仕組みを学ぶ
  • int、double、string、boolなど主要なデータ型を理解する
  • 実際のプログラムで変数を使ってみる

5.1 変数とは何か

プログラミングでは、計算結果やユーザーが入力した情報など、さまざまなデータを一時的に保存する必要があります。 このデータを保存するための「箱」のような存在が変数です。

変数を箱にたとえると

変数は「名前のついた箱」です。箱にはラベル（変数名）が付いていて、 中にはデータ（値）を入れることができます。
必要なときにその箱を開けて、中のデータを取り出したり、新しいデータに入れ替えたりできます。

flowchart LR A[変数名: age] --> B[箱の中身: 25] C[変数名: name] --> D[箱の中身: 太郎] E[変数名: isStudent] --> F[箱の中身: true]

なぜ変数が必要なのか

もし変数がなければ、計算結果をすぐに画面に表示するか、そのまま捨てるしかありません。 変数があることで、以下のようなことができるようになります：

  • 計算結果を保存しておいて、後で使う
  • ユーザーの入力を受け取って保存する
  • データを加工・変更する
  • 複雑な計算を段階的に行う

5.2 変数の宣言と代入

C#で変数を使うには、まず変数を宣言し、その後値を代入します。

変数の宣言

変数の宣言とは、「これからこの名前の箱を使います」とコンピュータに伝えることです。

変数の宣言（基本形） 
// データ型 変数名;
int age;         // 整数を保存する変数ageを宣言
string name;     // 文字列を保存する変数nameを宣言
bool isActive;   // 真偽値を保存する変数isActiveを宣言

値の代入

宣言した変数に値を入れることを代入といいます。 代入には=（イコール記号）を使います。

値の代入 
// 変数名 = 値;
age = 25;                    // 変数ageに25を代入
name = "太郎";               // 変数nameに「太郎」を代入
isActive = true;             // 変数isActiveにtrueを代入

重要ポイント

プログラミングでの=は「等しい」という意味ではなく、 「右側の値を左側の変数に入れる」という命令です。
例：age = 25は「ageは25と等しい」ではなく、 「ageに25を代入する」という意味です。

宣言と代入を同時に行う

実際には、宣言と代入を1行でまとめて書くことが多いです。これを初期化といいます。

宣言と代入を同時に行う（初期化） 
// データ型 変数名 = 値;
int age = 25;
string name = "太郎";
bool isActive = true;

5.3 主要なデータ型

C#では、変数に保存するデータの種類（型）を明示する必要があります。 これをデータ型といいます。ここでは、特によく使う4つのデータ型を学びます。

int型（整数型）

小数点を含まない整数を扱うためのデータ型です。

例：

int age = 25;
int score = 100;
int temperature = -5;

範囲：約-21億 ～ 21億

double型（浮動小数点型）

小数点を含む数値を扱うためのデータ型です。

例：

double price = 1500.5;
double pi = 3.14159;
double weight = 65.3;

精度：約15～17桁の有効数字

string型（文字列型）

文字や文章を扱うためのデータ型です。必ずダブルクォーテーション"で囲みます。

例：

string name = "山田太郎";
string message = "こんにちは";
string email = "test@example.com";

注意：数字も""で囲むと文字列として扱われます

bool型（真偽値型）

「はい/いいえ」「真/偽」を表すデータ型です。true（真）かfalse（偽）の2つの値しかありません。

例：

bool isStudent = true;
bool hasLicense = false;
bool isRaining = true;

用途：条件判定やフラグ管理に使用

データ型の使い分け早見表

データ型 保存できるデータ 使用例
int 整数 年齢、個数、スコア
double 小数 価格、身長、体重
string 文字列 名前、住所、メッセージ
bool true/false 判定結果、フラグ

5.4 データ型の注意点

C#では、変数に宣言した型と異なる種類のデータを代入しようとすると、エラーになります。

型の不一致エラー

以下のコードはエラーになります。

エラーになる例 
// エラー例1：int型の変数に文字列を代入
int age = "25";  // エラー！int型には数値を入れるべき

// エラー例2：string型の変数に数値を代入
string name = 100;  // エラー！string型には文字列を入れるべき

// エラー例3：bool型の変数に数値を代入
bool isActive = 1;  // エラー！bool型にはtrueかfalseのみ

正しい書き方

データ型と値の種類を一致させましょう。

正しい例 
// 正しい例
int age = 25;              // int型には整数
string name = "太郎";      // string型には文字列（""で囲む）
bool isActive = true;      // bool型にはtrueかfalse

数値と文字列の違い

初心者がよく間違えるのが、数値と文字列としての数字の違いです。

数値と文字列の違い 
int number = 100;        // これは数値（計算できる）
string text = "100";     // これは文字列（計算できない）

// numberは計算できる
int result = number + 50;  // 150になる

// textは計算できない（文字として連結される）
// string combined = text + 50;  // "10050"という文字列になる

5.5 変数の命名規則

変数名は自由につけられますが、いくつかのルールと推奨される書き方があります。

必ず守るべきルール

  • 数字から始めてはいけない：1age ❌
  • 空白を含めてはいけない：user name ❌
  • 特殊文字は使えない：user-name、user@name ❌
  • C#の予約語は使えない：int、class、if ❌

推奨される書き方（キャメルケース）

C#では、変数名はキャメルケース（camelCase）で書くのが一般的です。 最初の単語は小文字、2つ目以降の単語の頭文字を大文字にします。

良い変数名の例 
// 良い例（キャメルケース）
string userName = "太郎";
int studentAge = 20;
double productPrice = 1500.5;
bool isActive = true;

// 分かりやすい変数名
int examScore = 85;
string emailAddress = "test@example.com";
double totalAmount = 5000.0;

変数名の付け方のコツ

- 意味が分かる名前をつける：xよりageの方が良い
- 英語を使う：日本語はエラーの原因になりやすい
- 省略しすぎない：nmよりnameの方が分かりやすい

5.6 実習：変数を使ったプログラム

実習 5-1: 自己紹介プログラム

複数の変数を使って、自己紹介情報を表示するプログラムを作成しましょう。

手順

  1. 1 Visual Studioで新しいコンソールアプリプロジェクトを作成
  2. 2 以下のコードを入力してください
Program.cs 
using System;

class Program
{
    static void Main()
    {
        // 変数の宣言と初期化
        string name = "山田太郎";
        int age = 20;
        double height = 170.5;
        bool isStudent = true;

        // 自己紹介を表示
        Console.WriteLine("=== 自己紹介 ===");
        Console.WriteLine("名前: " + name);
        Console.WriteLine("年齢: " + age + "歳");
        Console.WriteLine("身長: " + height + "cm");
        Console.WriteLine("学生: " + isStudent);
    }
}
  1. 3 F5キーを押してプログラムを実行
  2. 4 変数の値を自分の情報に変更して、再度実行してみましょう

動作確認

プログラムを実行すると、以下のような出力が表示されるはずです。

期待される結果 

=== 自己紹介 ===
名前: 山田太郎
年齢: 20歳
身長: 170.5cm
学生: True

理解度確認クイズ

  1. Q1. 変数とは何ですか？簡単に説明してください。

    回答を表示

    変数とは、データを一時的に保存するための「名前のついた箱」のようなものです。 必要なときにデータを取り出したり、新しいデータに入れ替えたりできます。

  2. Q2. 以下のうち、正しい変数の宣言はどれですか？

    A) int 123age = 20;

    B) string name = "太郎";

    C) bool is active = true;

    回答を表示

    正解：B
    A) 変数名は数字から始められません
    B) 正しい宣言です
    C) 変数名に空白を含めることはできません（isActiveとすべき）

  3. Q3. 小数を保存したいとき、使うべきデータ型はどれですか？

    回答を表示

    正解：double型
    小数点を含む数値を扱うには、double型を使います。 int型は整数のみで、小数は扱えません。

  4. Q4. 次のコードの間違いを指摘してください。
    int price = "1000";

    回答を表示

    間違い：int型の変数に文字列を代入しようとしている
    正しい書き方：int price = 1000;
    int型には数値を代入します。ダブルクォーテーションで囲むと文字列になってしまいます。

この章のまとめ

  • 変数はデータを保存するための「名前のついた箱」
  • 変数は宣言してから代入して使う（初期化で同時も可能）
  • 主要なデータ型：int（整数）、double（小数）、string（文字列）、bool（真偽値）
  • 変数名はキャメルケースで、意味が分かる名前をつける
