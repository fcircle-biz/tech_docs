配列とリストでデータを管理しよう

複数のデータをまとめて扱う配列とListの基本を学びます。要素の追加・削除・検索など、実践的なデータ操作を体験します。

この章で学ぶこと

  • 配列とは何か、どのように使うかを理解する
  • Listクラスの基本操作（追加、削除、検索）を習得する
  • 配列とListの違いと使い分けを理解する
  • foreachを使った配列・Listの反復処理をマスターする

9.1 配列とは何か

これまで変数を使ってデータを保存してきましたが、1つの変数には1つの値しか入れられませんでした。 配列を使うと、同じ種類のデータを複数まとめて管理できます。

日常生活での例え

配列は「ロッカー」のようなものです。
ロッカーには複数の引き出しがあり、それぞれに番号（0, 1, 2...）が振られています。
番号を指定することで、特定の引き出しの中身を取り出したり、入れたりできます。

配列の宣言と初期化

配列を使うには、まず宣言と初期化が必要です。

配列の宣言と初期化 
// 方法1: サイズを指定して宣言
int[] scores = new int[5];  // 5つの要素を持つ配列

// 方法2: 初期値を指定して宣言
int[] numbers = { 10, 20, 30, 40, 50 };

// 方法3: newを使った初期化
string[] fruits = new string[] { "りんご", "バナナ", "みかん" };

重要ポイント

配列のインデックス（番号）は0から始まります。
5つの要素がある配列のインデックスは 0, 1, 2, 3, 4 となります。

配列の要素にアクセスする

配列の読み書き 
int[] scores = { 85, 92, 78, 88, 95 };

// 要素の読み取り
Console.WriteLine(scores[0]);  // 出力: 85
Console.WriteLine(scores[2]);  // 出力: 78

// 要素の書き換え
scores[1] = 100;  // 2番目の要素を100に変更
Console.WriteLine(scores[1]);  // 出力: 100

// 配列の長さを取得
Console.WriteLine($"配列の長さ: {scores.Length}");  // 出力: 5

9.2 Listとは何か

配列は便利ですが、作成後にサイズを変更できないという制限があります。 Listを使うと、自由に要素を追加・削除できます。

配列とListの違い

配列

  • ✓ サイズが固定
  • ✓ シンプルで高速
  • ✗ 要素の追加・削除が難しい

List

  • ✓ サイズが可変
  • ✓ 要素の追加・削除が簡単
  • ✗ 配列より少し遅い

Listの宣言と初期化

Listの基本 
using System.Collections.Generic;  // 必須

// 空のListを作成
List<int> numbers = new List<int>();

// 初期値を持つListを作成
List<string> fruits = new List<string>
{
    "りんご", "バナナ", "みかん"
};

注意

Listを使うには、ファイルの先頭にusing System.Collections.Generic;を書く必要があります。

9.3 Listの基本操作

要素の追加

Addメソッドで要素を追加 
List<string> tasks = new List<string>();

// 要素を追加
tasks.Add("宿題をする");
tasks.Add("買い物に行く");
tasks.Add("掃除をする");

Console.WriteLine($"タスク数: {tasks.Count}");  // 出力: 3

要素の削除

要素を削除する方法 
List<string> fruits = new List<string>
{ "りんご", "バナナ", "みかん", "ぶどう" };

// 値を指定して削除
fruits.Remove("バナナ");  // "バナナ"を削除

// インデックスを指定して削除
fruits.RemoveAt(0);  // 最初の要素を削除

// すべて削除
fruits.Clear();

要素の検索

要素を探す方法 
List<string> colors = new List<string>
{ "赤", "青", "緑", "黄色" };

// 要素が存在するか確認
if (colors.Contains("青"))
{
    Console.WriteLine("青が見つかりました");
}

// 要素のインデックスを取得
int index = colors.IndexOf("緑");
Console.WriteLine($"緑のインデックス: {index}");  // 出力: 2

実習：買い物リストを作ろう

Listを使って買い物リストを管理するプログラムを作ってみましょう。

Program.cs 
using System;
using System.Collections.Generic;

List<string> shoppingList = new List<string>();

// アイテムを追加
shoppingList.Add("牛乳");
shoppingList.Add("卵");
shoppingList.Add("パン");

// 一覧を表示
Console.WriteLine("=== 買い物リスト ===");
for (int i = 0; i < shoppingList.Count; i++)
{
    Console.WriteLine($"{i + 1}. {shoppingList[i]}");
}

実行結果:
=== 買い物リスト ===
1. 牛乳
2. 卵
3. パン

9.4 foreachで配列・Listを処理する

配列やListのすべての要素を処理する場合、foreachを使うと簡潔に書けます。

foreachの基本 
// 配列の場合
int[] scores = { 85, 92, 78, 88, 95 };
foreach (int score in scores)
{
    Console.WriteLine($"点数: {score}");
}

// Listの場合
List<string> fruits = new List<string> { "りんご", "バナナ", "みかん" };
foreach (string fruit in fruits)
{
    Console.WriteLine($"果物: {fruit}");
}

forとforeachの使い分け

foreach: すべての要素を順番に処理するだけ（シンプル）
for: インデックスが必要な場合、特定の範囲だけ処理する場合

9.5 実践例：成績管理プログラム

これまでの知識を総合して、成績管理プログラムを作ってみましょう。

成績管理プログラム 
using System;
using System.Collections.Generic;

List<int> scores = new List<int> { 85, 92, 78, 88, 95 };

// 合計点を計算
int total = 0;
foreach (int score in scores)
{
    total += score;
}

// 平均点を計算
double average = (double)total / scores.Count;

Console.WriteLine($"合計点: {total}点");
Console.WriteLine($"平均点: {average:F1}点");

動作確認

上記のコードを実行すると、次のように表示されます：

合計点: 438点
平均点: 87.6点

よくあるエラーと対処法

IndexOutOfRangeException

原因: 配列の範囲外のインデックスにアクセスしています。
対処: インデックスは0から始まり、配列の長さ-1までです。

ArgumentOutOfRangeException

原因: Listの範囲外のインデックスにアクセスしています。
対処: Countプロパティで要素数を確認しましょう。

型が見つかりません（List）

原因: using System.Collections.Generic;が抜けています。
対処: ファイルの先頭に追加してください。

理解度確認クイズ

  1. Q1. 配列のインデックスは何番から始まりますか？

    回答を表示

    0番から始まります。5つの要素がある配列のインデックスは 0, 1, 2, 3, 4 となります。

  2. Q2. 配列とListの主な違いは何ですか？

    回答を表示

    配列はサイズが固定で作成後に変更できません。
    Listはサイズが可変で、要素の追加・削除が自由にできます。

  3. Q3. Listに要素を追加するメソッドは何ですか？

    回答を表示

    Add()メソッドを使います。
    例: list.Add("新しい要素");

  4. Q4. foreachループの利点は何ですか？

    回答を表示

    配列やListのすべての要素を順番に処理する際、forループよりシンプルに書けます。
    インデックスを管理する必要がないため、コードが読みやすくなります。

この章のまとめ

  • 配列は同じ種類のデータを複数まとめて管理できる（サイズ固定）
  • インデックスは0から始まり、配列の長さ-1まで
  • Listはサイズが可変で、Add、Remove、Containsなどのメソッドで操作できる
  • foreachを使うと配列やListのすべての要素を簡単に処理できる
