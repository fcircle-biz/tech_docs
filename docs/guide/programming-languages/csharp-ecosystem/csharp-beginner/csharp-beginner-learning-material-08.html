第8章

繰り返し処理をマスターしよう

for文、while文、do-while文、foreach文を使った繰り返し処理を学びます。同じ処理を効率的に繰り返す方法と、無限ループを避けるためのポイントを理解します。

この章で学ぶこと

  • 繰り返し処理（ループ）とは何か、なぜ必要なのかを理解する
  • for文、while文、do-while文、foreach文の書き方と使い分け
  • ループ制御（break、continue）の使い方を習得する
  • 無限ループを回避し、安全なループを書く方法を学ぶ

8.1 繰り返し処理（ループ）とは

繰り返し処理（ループ）とは、同じ処理を何度も繰り返し実行する仕組みです。 プログラミングでは、手作業では面倒な反復作業を簡単に実現できます。

日常生活の繰り返し処理

「階段を10段登る」という動作は、「1段登る」を10回繰り返す処理です。
「洗濯物を10枚干す」も、「1枚干す」を10回繰り返す処理と考えられます。
プログラムでも同様に、「1回分の処理」を何度も繰り返すことで複雑な処理を実現します。

繰り返し処理が必要な場面

  • 1から100までの数字を表示したい
  • 配列の全要素に対して同じ処理をしたい
  • ユーザーが「終了」と入力するまで処理を続けたい
  • ファイルの内容を1行ずつ読み込んで処理したい

これらの処理をコードで1つずつ書いていたら大変ですが、繰り返し処理を使えば数行で実現できます。

flowchart TD A[処理開始] --> B{繰り返し条件を満たす？} B -->|はい| C[処理を実行] C --> B B -->|いいえ| D[処理終了]

8.2 for文：回数が決まっている繰り返し

for文は、繰り返す回数があらかじめ決まっている場合に使います。 「1から10まで表示する」「配列の全要素を処理する」など、回数が明確な場面で活躍します。

for文の基本構文

C# 
// for文の基本形
for (初期化; 条件式; 増減式)
{
    // 繰り返し実行する処理
}

// 具体例：1から5まで表示
for (int i = 1; i <= 5; i++)
{
    Console.WriteLine(i);
}

for文の3つの要素

1. 初期化（int i = 1）

ループ開始時に1回だけ実行されます。カウンター変数を初期化します。

2. 条件式（i <= 5）

この条件がtrueの間、繰り返しが続きます。falseになったら終了します。

3. 増減式（i++）

1回のループが終わるたびに実行されます。通常はカウンターを増やします。

実習：九九の表を作成しよう

実習：2の段を表示する

for文を使って、2の段（2×1から2×9）を表示するプログラムを作成してみましょう。

C# 
// 2の段を表示するプログラム
Console.WriteLine("2の段");
for (int i = 1; i <= 9; i++)
{
    int result = 2 * i;
    Console.WriteLine($"2 × {i} = {result}");
}

ネストしたfor文（入れ子のループ）

for文の中にさらにfor文を書くことをネスト（入れ子）と言います。 二重ループを使うと、九九の表全体を作成できます。

C# 
// 九九の表全体を作成（二重ループ）
for (int i = 1; i <= 9; i++)
{
    for (int j = 1; j <= 9; j++)
    {
        Console.Write($"{i * j}\t");
    }
    Console.WriteLine(); // 改行
}

8.3 while文：条件が満たされる間繰り返す

while文は、条件がtrueの間、繰り返しを続ける仕組みです。 繰り返し回数が事前にわからない場合に適しています。

while文の基本構文

C# 
// while文の基本形
while (条件式)
{
    // 繰り返し実行する処理
}

// 具体例：1から5まで表示
int i = 1;
while (i <= 5)
{
    Console.WriteLine(i);
    i++; // カウンターを増やす
}

while文とfor文の違い

for文：繰り返し回数が決まっている場合に便利
while文：「終了」と入力されるまで、など条件が動的に変わる場合に便利

実習：ユーザー入力を繰り返し受け付ける

実習：終了と入力されるまで繰り返す

ユーザーが「終了」と入力するまで、入力を受け付け続けるプログラムを作成します。

C# 
// 「終了」と入力されるまで繰り返す
string input = "";
while (input != "終了")
{
    Console.Write("文字を入力してください（終了で終わり）: ");
    input = Console.ReadLine();
    Console.WriteLine($"入力された文字: {input}");
}
Console.WriteLine("プログラムを終了します。");

8.4 do-while文：最低1回は実行する繰り返し

do-while文は、while文と似ていますが、最低1回は必ず処理を実行します。 条件チェックが後にあるため、条件が最初からfalseでも1回は実行されます。

do-while文の基本構文

C# 
// do-while文の基本形
do
{
    // 繰り返し実行する処理
} while (条件式);

// 具体例：1から5まで表示
int i = 1;
do
{
    Console.WriteLine(i);
    i++;
} while (i <= 5);

while文

条件を最初にチェックしてから処理を実行。条件が最初からfalseなら1回も実行されない。

do-while文

処理を実行してから最後に条件をチェック。最低1回は必ず実行される。

do-while文が便利な場面

メニュー選択やデータ入力など、「最低1回は必ずユーザーに選択肢を見せたい」場合に適しています。

C# 
// 簡単なメニュー選択プログラム
int choice;
do
{
    Console.WriteLine("=== メニュー ===");
    Console.WriteLine("1. 開始");
    Console.WriteLine("2. 設定");
    Console.WriteLine("3. 終了");
    Console.Write("選択してください: ");
    choice = int.Parse(Console.ReadLine());
} while (choice < 1 || choice > 3);
Console.WriteLine($"選択された項目: {choice}");

8.5 foreach文：配列やリストの全要素を処理する

foreach文は、配列やリストなどのコレクションの全要素を順番に処理する専用の繰り返し文です。 インデックス番号を気にせずに、要素を直接扱えるのが特徴です。

foreach文の基本構文

C# 
// foreach文の基本形
foreach (型 変数名 in コレクション)
{
    // 繰り返し実行する処理
}

// 具体例：配列の全要素を表示
string[] fruits = { "りんご", "バナナ", "オレンジ" };
foreach (string fruit in fruits)
{
    Console.WriteLine(fruit);
}

foreach文の利点

・インデックス番号を管理する必要がない
・コードがシンプルで読みやすい
・範囲外エラー（IndexOutOfRangeException）が起きない

実習：合計点を計算する

実習：テストの合計点と平均点を計算

foreach文を使って、テストの点数配列から合計点と平均点を求めましょう。

C# 
// テストの点数を配列に格納
int[] scores = { 85, 92, 78, 90, 88 };

// 合計点を計算
int total = 0;
foreach (int score in scores)
{
    total += score;
}

// 平均点を計算
double average = (double)total / scores.Length;

Console.WriteLine($"合計点: {total}");
Console.WriteLine($"平均点: {average}");

8.6 ループ制御：breakとcontinue

ループの途中で処理を終了したり、一部の処理をスキップしたりしたい場合に使う命令がbreakとcontinueです。

break：ループを途中で終了

breakは、ループを即座に終了して、ループの外に抜けます。

C# 
// 1から10まで表示するが、5で終了
for (int i = 1; i <= 10; i++)
{
    if (i == 5)
    {
        break; // iが5になったらループを終了
    }
    Console.WriteLine(i);
}
// 出力結果: 1, 2, 3, 4

continue：今回の処理をスキップ

continueは、その回の処理をスキップして、次のループに進みます。

C# 
// 1から10まで表示するが、5はスキップ
for (int i = 1; i <= 10; i++)
{
    if (i == 5)
    {
        continue; // iが5の時はスキップ
    }
    Console.WriteLine(i);
}
// 出力結果: 1, 2, 3, 4, 6, 7, 8, 9, 10

break

ループを完全に終了する。ループの外に抜けます。

continue

今回の処理をスキップして、次のループへ進みます。

8.7 無限ループの回避と安全なループ

無限ループとは、終了条件が満たされず、永遠に繰り返し続けてしまう状態です。 プログラムがフリーズしたり、応答しなくなる原因になります。

危険な無限ループの例

C# (悪い例) 
// 悪い例：カウンターが増えないため無限ループになる
int i = 1;
while (i <= 10)
{
    Console.WriteLine(i);
    // i++; を忘れている！
}

無限ループを防ぐチェックポイント

1. カウンターが正しく更新されているか確認

while文やfor文では、必ずカウンター変数が増減するようにします。

2. 終了条件が必ず満たされるか確認

条件式が必ずfalseになる瞬間があるかをチェックします。

3. ユーザー入力に依存する場合は脱出手段を用意

ユーザーが「終了」などのキーワードを入力すれば終了できるようにします。

無限ループから脱出する方法

もし無限ループに陥ってしまったら、Visual StudioでCtrl + Cを押すか、
デバッグメニューから「デバッグの停止」を選択してプログラムを強制終了できます。

理解度確認クイズ

  1. Q1. for文の3つの要素（初期化、条件式、増減式）をそれぞれ説明してください。

    回答を表示

    初期化：ループ開始時に1回だけ実行され、カウンター変数を初期化します（例: int i = 0）。
    条件式：この条件がtrueの間、繰り返しが続きます（例: i < 10）。
    増減式：1回のループが終わるたびに実行され、通常はカウンターを増減します（例: i++）。

  2. Q2. while文とdo-while文の違いは何ですか？

    回答を表示

    while文：条件を最初にチェックしてから処理を実行します。条件が最初からfalseなら1回も実行されません。
    do-while文：処理を実行してから最後に条件をチェックします。条件に関わらず最低1回は必ず実行されます。

  3. Q3. breakとcontinueの違いを説明してください。

    回答を表示

    break：ループを即座に完全に終了し、ループの外に抜けます。
    continue：その回の処理をスキップして、次のループに進みます。ループ自体は続きます。

  4. Q4. 無限ループを避けるために注意すべきポイントを2つ挙げてください。

    回答を表示

    1. カウンター変数が正しく更新されているか確認する（i++などの増減式を忘れない）
    2. 終了条件が必ず満たされるようにする（条件式が必ずfalseになるタイミングがあるか確認）
    （他にも「ユーザー入力に脱出手段を用意する」なども正解です）

この章のまとめ

  • 繰り返し処理（ループ）は、同じ処理を効率的に何度も実行する仕組み
  • for文は回数が決まっている繰り返し、while文は条件が満たされる間の繰り返しに使う
  • do-while文は最低1回は必ず実行し、foreach文は配列やリストの全要素を処理する
  • breakでループを終了、continueで今回の処理をスキップできる
  • 無限ループを避けるには、カウンターの更新と終了条件を必ず確認する
