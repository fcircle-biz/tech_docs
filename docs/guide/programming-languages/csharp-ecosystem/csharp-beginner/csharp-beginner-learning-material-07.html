第7章

条件分岐で処理を分けよう

if文とswitch文を使って、条件によって異なる処理を実行する方法を学びます。比較演算子と論理演算子を組み合わせた複雑な条件判断も練習します。

この章で学ぶこと

  • if文の基本構文と使い方を理解する
  • else、else ifを使った複数条件の分岐を学ぶ
  • switch文の構文と使い方を理解する
  • 比較演算子（==, !=, <, >, <=, >=）を使いこなす
  • 論理演算子（&&, ||, !）を組み合わせた複雑な条件判断を作成する

7.1 条件分岐とは

条件分岐とは、ある条件が満たされているかどうかによって、実行する処理を変える仕組みです。 日常生活でも、私たちは常に条件分岐を行っています。

日常生活の条件分岐の例

「もし雨が降っていたら傘を持っていく。そうでなければ傘は持っていかない」
この判断も条件分岐です。「雨が降っている」という条件によって、行動（傘を持つ/持たない）が変わります。

プログラムでの条件分岐

プログラムでも同じように、ある条件に応じて処理を変えることができます。 条件分岐を使うことで、より柔軟で賢いプログラムを作ることができます。

flowchart TD A[処理開始] --> B{条件判断
年齢 >= 20歳?} B -->|はい true| C[成人です] B -->|いいえ false| D[未成年です] C --> E[処理終了] D --> E

重要ポイント

条件分岐では、条件がtrue（真、正しい）かfalse（偽、正しくない）かによって処理が分かれます。 このtrue/falseの値を真偽値（ブール値）と呼びます。

7.2 if文の基本

if文は、条件分岐を行うための最も基本的な構文です。 「もし〜ならば…する」という形で処理を記述します。

if文の基本構文

C# 
// if文の基本構文
if (条件式)
{
    // 条件がtrueのときに実行される処理
}

具体例：年齢チェック

C# 
int age = 25;

// もし年齢が20以上ならば
if (age >= 20)
{
    Console.WriteLine("成人です");
}
// 実行結果: 成人です

この例では、age >= 20という条件式がtrueなので、 中かっこ{}の中の処理が実行されます。

ヒント：中かっこの省略について

実行する処理が1行だけの場合、中かっこ{}を省略できますが、 省略しないことを強く推奨します。中かっこがあることで、どこまでがif文の範囲かが明確になり、バグを防げます。

7.3 比較演算子

条件式を書くときには、比較演算子を使います。 比較演算子は、2つの値を比較してtrue/falseを返します。

主な比較演算子

演算子 意味 例 結果
== 等しい 5 == 5 true
!= 等しくない 5 != 3 true
< 小さい（未満） 3 < 5 true
> 大きい（超える） 5 > 3 true
<= 以下（小さいか等しい） 5 <= 5 true
>= 以上（大きいか等しい） 5 >= 5 true

注意：「==」と「=」の違い

==は「等しいかどうかを比較する」演算子です。
=は「値を代入する」演算子です。
if文では必ず「==」を使います。間違えると正しく動作しません。

比較演算子の使用例

C# 
int score = 85;

// 80点以上なら合格
if (score >= 80)
{
    Console.WriteLine("合格です！");
}

// 点数がちょうど100点なら満点
if (score == 100)
{
    Console.WriteLine("満点です！");
}

// 点数が60点未満なら不合格
if (score < 60)
{
    Console.WriteLine("不合格です");
}
// 実行結果: 合格です！

7.4 else文とelse if文

if文だけでは「条件がtrueのときだけ処理する」ことしかできません。 else文とelse if文を使うことで、より複雑な条件分岐ができます。

else文：条件がfalseのときの処理

C# 
int age = 18;

// もし20歳以上なら
if (age >= 20)
{
    Console.WriteLine("成人です");
}
else  // そうでなければ（20歳未満なら）
{
    Console.WriteLine("未成年です");
}
// 実行結果: 未成年です
flowchart TD A[年齢を確認] --> B{age >= 20?} B -->|true| C[成人です] B -->|false| D[未成年です] C --> E[処理終了] D --> E

else if文：複数の条件を順番にチェック

3つ以上の分岐が必要な場合は、else if文を使います。 上から順番に条件をチェックし、最初にtrueになった条件の処理だけが実行されます。

C# 
int score = 75;

if (score >= 90)
{
    Console.WriteLine("評価：A（優秀）");
}
else if (score >= 80)
{
    Console.WriteLine("評価：B（良好）");
}
else if (score >= 60)
{
    Console.WriteLine("評価：C（合格）");
}
else
{
    Console.WriteLine("評価：D（不合格）");
}
// 実行結果: 評価：C（合格）
flowchart TD A[点数チェック開始] --> B{score >= 90?} B -->|true| C[評価：A] B -->|false| D{score >= 80?} D -->|true| E[評価：B] D -->|false| F{score >= 60?} F -->|true| G[評価：C] F -->|false| H[評価：D] C --> I[処理終了] E --> I G --> I H --> I

重要ポイント

if - else if - else の構造では、いずれか1つの処理だけが実行されます。 最初の条件がtrueになった時点で、残りの条件はチェックされません。

7.5 論理演算子で複雑な条件を作る

論理演算子を使うと、複数の条件を組み合わせて、より複雑な条件判断ができます。

主な論理演算子

演算子 意味 説明
&& かつ（AND） 両方の条件がtrueのときにtrueになる
|| または（OR） どちらか一方でもtrueならtrueになる
! 〜でない（NOT） 条件を反転させる（true→false、false→true）

&&（かつ）演算子の例

C# 
int age = 25;
bool hasLicense = true;

// 年齢が18歳以上かつ免許を持っている場合
if (age >= 18 && hasLicense)
{
    Console.WriteLine("運転できます");
}
else
{
    Console.WriteLine("運転できません");
}
// 実行結果: 運転できます

||（または）演算子の例

C# 
int age = 15;
bool isWithParent = true;

// 18歳以上または保護者同伴の場合
if (age >= 18 || isWithParent)
{
    Console.WriteLine("入場できます");
}
else
{
    Console.WriteLine("入場できません");
}
// 実行結果: 入場できます

!（〜でない）演算子の例

C# 
bool isRaining = false;

// 雨が降っていない場合
if (!isRaining)
{
    Console.WriteLine("傘は不要です");
}
else
{
    Console.WriteLine("傘を持っていきましょう");
}
// 実行結果: 傘は不要です

ヒント：論理演算子の優先順位

複数の論理演算子を組み合わせるときは、優先順位に注意しましょう。
! → && → || の順に優先されます。
複雑な条件では、かっこ ( ) を使って明示的に順序を指定することを推奨します。

7.6 switch文で値による分岐を行う

switch文は、1つの変数の値によって処理を分岐させる構文です。 if - else if を何度も書くより、switch文の方がシンプルで読みやすくなる場合があります。

switch文の基本構文

C# 
switch (変数)
{
    case 値1:
        // 変数が値1と等しい場合の処理
        break;
    case 値2:
        // 変数が値2と等しい場合の処理
        break;
    default:
        // どのcaseにも該当しない場合の処理
        break;
}

具体例：曜日による処理分岐

C# 
int dayOfWeek = 3; // 1=月曜、2=火曜、3=水曜、...

switch (dayOfWeek)
{
    case 1:
        Console.WriteLine("月曜日です");
        break;
    case 2:
        Console.WriteLine("火曜日です");
        break;
    case 3:
        Console.WriteLine("水曜日です");
        break;
    case 6:
    case 7:
        // 複数のcaseをまとめることも可能
        Console.WriteLine("週末です");
        break;
    default:
        Console.WriteLine("平日です");
        break;
}
// 実行結果: 水曜日です

注意：breakを忘れずに

各caseの最後には必ずbreakを書きます。breakがないと、次のcaseの処理も実行されてしまいます（フォールスルー）。 C#では、breakなしでフォールスルーするとコンパイルエラーになるため、忘れずに記述しましょう。

if文とswitch文の使い分け

if文を使う場合

・範囲の比較（>, <, >=, <=）
・複雑な条件（論理演算子の組み合わせ）
・真偽値（bool型）の判定

switch文を使う場合

・特定の値との一致判定（==）
・選択肢が3つ以上ある場合
・整数や文字列の値による分岐

実習：成績判定プログラムを作ろう

ユーザーに点数を入力させ、以下の基準で成績を判定するプログラムを作成してください。

判定基準：

  • ・90点以上：「優秀です！」
  • ・80点以上90点未満：「良好です」
  • ・60点以上80点未満：「合格です」
  • ・60点未満：「不合格です。もう一度頑張りましょう」
解答例を表示
C# 
// 点数を入力
Console.Write("点数を入力してください: ");
int score = int.Parse(Console.ReadLine());

// 成績判定
if (score >= 90)
{
    Console.WriteLine("優秀です！");
}
else if (score >= 80)
{
    Console.WriteLine("良好です");
}
else if (score >= 60)
{
    Console.WriteLine("合格です");
}
else
{
    Console.WriteLine("不合格です。もう一度頑張りましょう");
}

理解度確認クイズ

  1. Q1. if文で条件式がfalseだった場合、どうなりますか？

    回答を表示

    if文の中かっこ{}内の処理はスキップされ、実行されません。 else文がある場合は、else文の処理が実行されます。

  2. Q2. 次のコードの実行結果は何ですか？
    int x = 5; if (x > 3 && x < 10) { Console.WriteLine("範囲内"); }

    回答を表示

    「範囲内」と表示されます。
    x（5）は3より大きく（true）、かつ10より小さい（true）ので、 &&演算子によって全体の条件がtrueになります。

  3. Q3. switch文で各caseの最後に書く必要があるものは何ですか？

    回答を表示

    break;（ブレイク文）です。
    breakがないと次のcaseの処理も実行されてしまうため、必ず記述します。

  4. Q4. 「==」と「=」の違いを説明してください。

    回答を表示

    ==：等しいかどうかを比較する演算子（比較演算子）。結果はtrue/false。
    =：値を代入する演算子（代入演算子）。左辺の変数に右辺の値を格納する。

この章のまとめ

  • 条件分岐は、条件によって実行する処理を変える仕組み
  • if文は基本の条件分岐構文。else、else ifを組み合わせて複雑な分岐も可能
  • 比較演算子（==, !=, <, >, <=, >=）で値を比較してtrue/falseを得る
  • 論理演算子（&&, ||, !）で複数の条件を組み合わせられる
  • switch文は特定の値による分岐に適している。各caseの最後にbreakが必要
