14.1 プロジェクト概要

14.2 データベース設計

まず、アプリケーションで使用するデータベースとテーブルを設計します。 ToDoリストアプリケーションには、ユーザー情報とタスク情報の2つのテーブルが必要です。

テーブル設計

erDiagram USERS ||--o{ TASKS : "所有する" USERS { int id PK varchar username varchar password datetime created_at } TASKS { int id PK int user_id FK varchar title text description tinyint is_completed datetime created_at datetime updated_at }

データベースとテーブルの作成

phpMyAdminまたはMySQLコマンドラインで、以下のSQLを実行してデータベースとテーブルを作成します。

setup.sql 
-- データベースの作成
CREATE DATABASE todo_app CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE todo_app;

-- ユーザーテーブルの作成
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- タスクテーブルの作成
CREATE TABLE tasks (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    title VARCHAR(100) NOT NULL,
    description TEXT,
    is_completed TINYINT(1) DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

外部キー制約について

FOREIGN KEYとON DELETE CASCADEを設定することで、ユーザーが削除されたときに、そのユーザーに紐づくタスクも自動的に削除されます。

14.4 共通機能の実装

まず、アプリケーション全体で使用する共通機能を実装します。

データベース接続設定

config/database.php 
<?php
// データベース接続情報
define('DB_HOST', 'localhost');
define('DB_NAME', 'todo_app');
define('DB_USER', 'root');
define('DB_PASS', '');

// PDOでデータベースに接続
function getDbConnection() {
    try {
        $pdo = new PDO(
            "mysql:host=" . DB_HOST . ";dbname=" . DB_NAME . ";charset=utf8mb4",
            DB_USER,
            DB_PASS
        );
        // エラーモード設定
        $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
        return $pdo;
    } catch (PDOException $e) {
        die("データベース接続エラー: " . $e->getMessage());
    }
}

共通関数の作成

includes/functions.php 
<?php
// XSS対策：HTMLエスケープ
function h($str) {
    return htmlspecialchars($str, ENT_QUOTES, 'UTF-8');
}

// リダイレクト関数
function redirect($url) {
    header("Location: " . $url);
    exit;
}

// CSRFトークン生成
function generateCsrfToken() {
    if (!isset($_SESSION['csrf_token'])) {
        $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
    }
    return $_SESSION['csrf_token'];
}

// CSRFトークン検証
function verifyCsrfToken($token) {
    return isset($_SESSION['csrf_token']) && hash_equals($_SESSION['csrf_token'], $token);
}

セッション管理

includes/session.php 
<?php
// セッション開始
session_start();

// ログインチェック関数
function isLoggedIn() {
    return isset($_SESSION['user_id']);
}

// ログインユーザーID取得
function getLoggedInUserId() {
    return $_SESSION['user_id'] ?? null;
}

// ログイン必須ページのチェック
function requireLogin() {
    if (!isLoggedIn()) {
        redirect('login.php');
    }
}

14.5 ユーザー認証機能の実装

ユーザー登録とログイン機能を実装します。

ユーザー登録ページ

public/register.php 
<?php
require_once '../config/database.php';
require_once '../includes/session.php';
require_once '../includes/functions.php';

$error = '';

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $username = trim($_POST['username']);
    $password = $_POST['password'];

    // バリデーション
    if (empty($username) || empty($password)) {
        $error = 'ユーザー名とパスワードを入力してください';
    } elseif (strlen($password) < 6) {
        $error = 'パスワードは6文字以上で入力してください';
    } else {
        $pdo = getDbConnection();
        $stmt = $pdo->prepare("SELECT id FROM users WHERE username = ?");
        $stmt->execute([$username]);

        if ($stmt->fetch()) {
            $error = 'このユーザー名は既に使用されています';
        } else {
            // パスワードをハッシュ化して保存
            $hashed_password = password_hash($password, PASSWORD_DEFAULT);
            $stmt = $pdo->prepare("INSERT INTO users (username, password) VALUES (?, ?)");
            $stmt->execute([$username, $hashed_password]);

            redirect('login.php?registered=1');
        }
    }
}
?>
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>ユーザー登録 - ToDoアプリ</title>
    <link rel="stylesheet" href="css/style.css">
</head>
<body>
    <div class="container">
        <h1>ユーザー登録</h1>
        <?php if ($error): ?>
            <p class="error"><?= h($error) ?></p>
        <?php endif; ?>
        <form method="POST">
            <label>ユーザー名: <input type="text" name="username" required></label>
            <label>パスワード: <input type="password" name="password" required></label>
            <button type="submit">登録</button>
        </form>
        <p><a href="login.php">ログインはこちら</a></p>
    </div>
</body>

ログインページ（抜粋）

public/login.php（処理部分のみ） 
<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $username = trim($_POST['username']);
    $password = $_POST['password'];

    $pdo = getDbConnection();
    $stmt = $pdo->prepare("SELECT id, password FROM users WHERE username = ?");
    $stmt->execute([$username]);
    $user = $stmt->fetch(PDO::FETCH_ASSOC);

    if ($user && password_verify($password, $user['password'])) {
        // ログイン成功
        $_SESSION['user_id'] = $user['id'];
        redirect('index.php');
    } else {
        $error = 'ユーザー名またはパスワードが間違っています';
    }
}
?>

セキュリティのポイント

password_hash()とpassword_verify()を使用して、パスワードを安全にハッシュ化して保存し、検証しています。 平文でパスワードを保存することは絶対に避けてください。

14.6 タスク管理機能の実装

タスクのCRUD操作（作成、読み取り、更新、削除）を実装します。

タスク一覧表示（index.php）

public/index.php（処理部分） 
<?php
require_once '../config/database.php';
require_once '../includes/session.php';
require_once '../includes/functions.php';

// ログインチェック
requireLogin();

$user_id = getLoggedInUserId();
$pdo = getDbConnection();

// タスク一覧を取得
$stmt = $pdo->prepare("SELECT * FROM tasks WHERE user_id = ? ORDER BY created_at DESC");
$stmt->execute([$user_id]);
$tasks = $stmt->fetchAll(PDO::FETCH_ASSOC);
?>

タスク追加処理

public/add_task.php 
<?php
require_once '../config/database.php';
require_once '../includes/session.php';
require_once '../includes/functions.php';

requireLogin();

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // CSRFトークン検証
    if (!verifyCsrfToken($_POST['csrf_token'])) {
        die('不正なリクエストです');
    }

    $title = trim($_POST['title']);
    $description = trim($_POST['description']);
    $user_id = getLoggedInUserId();

    if (!empty($title)) {
        $pdo = getDbConnection();
        $stmt = $pdo->prepare("INSERT INTO tasks (user_id, title, description) VALUES (?, ?, ?)");
        $stmt->execute([$user_id, $title, $description]);
    }
}

redirect('index.php');

タスク削除処理

public/delete_task.php 
<?php
require_once '../config/database.php';
require_once '../includes/session.php';
require_once '../includes/functions.php';

requireLogin();

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // CSRFトークン検証
    if (!verifyCsrfToken($_POST['csrf_token'])) {
        die('不正なリクエストです');
    }

    $task_id = (int)$_POST['task_id'];
    $user_id = getLoggedInUserId();

    // 自分のタスクかチェック
    $pdo = getDbConnection();
    $stmt = $pdo->prepare("DELETE FROM tasks WHERE id = ? AND user_id = ?");
    $stmt->execute([$task_id, $user_id]);
}

redirect('index.php');

14.8 デプロイの基礎

作成したアプリケーションを公開するには、Webホスティングサービスにデプロイ（配置）する必要があります。

デプロイの流れ

flowchart LR A[ローカル開発] --> B[コードの最終確認] B --> C[ホスティングサービス選定] C --> D[ファイルアップロード] D --> E[データベース設定] E --> F[動作確認] F --> G[公開完了]

推奨ホスティングサービス

  • XFREE（エックスフリー）: 無料でPHP/MySQLが使える日本のサービス
  • さくらのレンタルサーバ: 低価格で高機能なレンタルサーバー
  • エックスサーバー: 国内シェアNo.1の安定したサーバー

デプロイ時の注意点

データベース接続情報（config/database.php）は、デプロイ先のサーバーに合わせて変更が必要です。 また、エラー表示設定を本番環境用に変更し、詳細なエラーメッセージがユーザーに表示されないようにしましょう。

理解度確認クイズ

  1. Q1. CRUD操作とは何を指しますか？ それぞれの頭文字が表す操作を答えてください。

    回答を表示

    C: Create（作成）、R: Read（読み取り）、U: Update（更新）、D: Delete（削除）。 データベース操作の基本的な4つの機能を表します。

  2. Q2. パスワードをデータベースに保存する際に使用すべきPHP関数は何ですか？

    回答を表示

    password_hash()関数を使用します。これにより、パスワードを安全にハッシュ化して保存できます。 検証時にはpassword_verify()を使用します。

  3. Q3. ユーザーが入力したデータをHTMLに出力する際、XSS対策として何をすべきですか？

    回答を表示

    htmlspecialchars()関数を使ってHTMLエスケープ処理を行います。 これにより、悪意のあるスクリプトが実行されるのを防ぎます。

  4. Q4. CSRF攻撃を防ぐために実装すべき対策を説明してください。

    回答を表示

    CSRFトークンを生成してセッションに保存し、フォーム送信時にそのトークンを検証します。 トークンが一致しない場合は不正なリクエストとして処理を拒否します。
