第7章

フォーム処理とHTTP通信

HTMLフォームからのデータ受信、GET/POSTメソッドの違い、フォームバリデーションの基礎を学びます。

この章で学ぶこと

  • HTMLフォームの基本構造を理解する
  • GETメソッドとPOSTメソッドの違いを理解する
  • $_GET、$_POSTでフォームデータを受け取る方法を習得する
  • フォームバリデーションの基礎を学ぶ
  • セキュリティの基本（XSS対策）を理解する

7.1 HTMLフォームの基本

HTMLフォームは、ユーザーからデータを入力してもらうための仕組みです。 お問い合わせフォーム、ログインフォーム、検索ボックスなど、Webサイトでよく見かける入力欄は、すべてHTMLフォームで作られています。

フォームの基本構造

HTMLフォームは <form> タグで定義します。 フォームには、データの送信先（action属性）と送信方法（method属性）を指定します。

basic_form.html 
<!-- 基本的なHTMLフォーム -->
<form action="receive.php" method="post">
    <!-- テキスト入力欄 -->
    <label for="name">名前:</label>
    <input type="text" id="name" name="name">

    <!-- メール入力欄 -->
    <label for="email">メール:</label>
    <input type="email" id="email" name="email">

    <!-- 送信ボタン -->
    <button type="submit">送信</button>
</form>

フォームの主要な属性

action: フォームデータの送信先（PHPファイルのパス）
method: 送信方法（getまたはpost）
name: 各入力欄の識別名（PHP側で受け取る際に使用）

主な入力要素

要素 用途 記述例
text 1行テキスト入力 <input type="text" name="username">
email メールアドレス入力 <input type="email" name="email">
password パスワード入力（隠し文字） <input type="password" name="pass">
textarea 複数行テキスト入力 <textarea name="message"></textarea>
radio ラジオボタン（単一選択） <input type="radio" name="gender" value="male">
checkbox チェックボックス（複数選択） <input type="checkbox" name="hobby[]" value="sports">
select ドロップダウンリスト <select name="country"><option>日本</option></select>

7.2 GETメソッドとPOSTメソッドの違い

フォームデータをサーバーに送信する方法には、GETとPOSTの2つがあります。 それぞれ特徴と適切な使い分けがあります。

flowchart LR subgraph GET[GETメソッド] G1["URLにデータを付加"] G2["example.php?name=太郎"] end subgraph POST[POSTメソッド] P1["HTTPボディにデータを格納"] P2["URLには表示されない"] end Browser[ブラウザ] --> GET Browser --> POST GET --> Server[Webサーバー] POST --> Server

GETメソッド

GETは、データをURLの末尾に付加して送信する方法です。 送信されたデータはブラウザのアドレスバーに表示されます。

get_form.html 
<!-- GETメソッドを使用するフォーム -->
<form action="search.php" method="get">
    <label>検索キーワード:</label>
    <input type="text" name="keyword">
    <button type="submit">検索</button>
</form>

<!-- 送信すると以下のようなURLになります -->
<!-- search.php?keyword=PHP -->
search.php 
<?php
// GETで送信されたデータを受け取る
$keyword = $_GET['keyword'] ?? '';

echo "検索キーワード: " . $keyword;
// 検索キーワード: PHP
?>

POSTメソッド

POSTは、データをHTTPリクエストのボディ（本文）に格納して送信する方法です。 データはURLに表示されないため、パスワードなどの機密情報を送信する際に使用します。

post_form.html 
<!-- POSTメソッドを使用するフォーム -->
<form action="login.php" method="post">
    <label>ユーザー名:</label>
    <input type="text" name="username">

    <label>パスワード:</label>
    <input type="password" name="password">

    <button type="submit">ログイン</button>
</form>
login.php 
<?php
// POSTで送信されたデータを受け取る
$username = $_POST['username'] ?? '';
$password = $_POST['password'] ?? '';

echo "ユーザー名: " . $username;
// パスワードは画面に表示しない
?>

GET vs POST の比較

項目 GET POST
データの表示 URLに表示される URLに表示されない
データ量の制限 あり（約2000文字） なし（サーバー設定次第）
ブックマーク 可能 不可
セキュリティ 低い（履歴に残る） 高い（履歴に残らない）
適した用途 検索、ページ遷移 ログイン、データ登録

使い分けの目安

GET: 検索機能、ページネーション、共有可能なURLを作りたい場合
POST: ログイン、会員登録、データの作成・更新・削除を行う場合

7.3 フォームバリデーション（入力検証）

フォームバリデーションは、ユーザーが入力したデータが正しい形式かどうかをチェックする処理です。 不正なデータや空のデータが送信されるのを防ぎ、エラーメッセージを表示してユーザーに修正を促します。

基本的なバリデーション

以下は、よく使用されるバリデーションのパターンです。

validate.php 
<?php
// フォームデータを受け取る
$name = $_POST['name'] ?? '';
$email = $_POST['email'] ?? '';
$age = $_POST['age'] ?? '';

// エラーメッセージを格納する配列
$errors = [];

// 1. 必須チェック
if (empty($name)) {
    $errors[] = "名前は必須です";
}

// 2. メールアドレスの形式チェック
if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
    $errors[] = "メールアドレスの形式が正しくありません";
}

// 3. 数値チェック
if (!is_numeric($age)) {
    $errors[] = "年齢は数値で入力してください";
}

// エラーがある場合
if (!empty($errors)) {
    foreach ($errors as $error) {
        echo "<p>エラー: {$error}</p>";
    }
} else {
    echo "入力データは正しいです！";
}
?>

便利なバリデーション関数

チェック内容 関数・方法
空かどうか empty($value)
数値かどうか is_numeric($value)
メール形式 filter_var($value, FILTER_VALIDATE_EMAIL)
URL形式 filter_var($value, FILTER_VALIDATE_URL)
文字数チェック mb_strlen($value) <= 100
正規表現マッチ preg_match('/^[0-9]+$/', $value)

7.4 セキュリティ対策の基礎（XSS対策）

フォームから受け取ったデータをそのまま表示すると、XSS（クロスサイトスクリプティング）という攻撃を受ける可能性があります。 必ずエスケープ処理を行いましょう。

XSS攻撃とは

悪意のあるユーザーがフォームに <script> タグなどを入力し、 他のユーザーのブラウザで不正なスクリプトを実行させる攻撃手法です。

危険な例（エスケープなし）

dangerous.php（❌ 危険） 
<?php
// ❌ 危険: そのまま表示
$comment = $_POST['comment'];
echo "<p>" . $comment . "</p>";

// もし $comment が "<script>alert('攻撃')</script>" だったら
// スクリプトが実行されてしまう！
?>

安全な例（エスケープあり）

safe.php（✅ 安全） 
<?php
// ✅ 安全: htmlspecialchars()でエスケープ
$comment = $_POST['comment'] ?? '';
$safe_comment = htmlspecialchars($comment, ENT_QUOTES, 'UTF-8');
echo "<p>" . $safe_comment . "</p>";

// <script> は &lt;script&gt; に変換され、
// ただのテキストとして表示される
?>

htmlspecialchars()の必須引数

第1引数: エスケープする文字列
第2引数: ENT_QUOTES（シングルクォートもエスケープ）
第3引数: 'UTF-8'（文字コード）

実習 7-1: お問い合わせフォームの作成

HTMLフォームとPHP処理を組み合わせて、簡単なお問い合わせフォームを作成しましょう。

手順

  1. 1 contact_form.htmlを作成し、名前、メール、メッセージの入力欄を持つフォームを作成
  2. 2 contact_receive.phpを作成し、POSTで受け取ったデータを表示
  3. 3 必須チェックとメール形式チェックのバリデーションを追加
  4. 4 htmlspecialchars()でエスケープ処理を実装

動作確認

ブラウザでcontact_form.htmlを開き、フォームに情報を入力して送信してください。

期待される結果 

お問い合わせを受け付けました

名前: 山田太郎
メール: yamada@example.com
メッセージ: こんにちは、お問い合わせです。

トラブルシューティング

$_POSTが空になる

原因: フォームのmethod属性がgetになっている、またはinputにname属性がない

解決方法: <form method="post">になっているか、各inputにname="..."が設定されているか確認してください

Notice: Undefined index エラー

原因: 存在しないキーにアクセスしている

解決方法: null合体演算子を使用して$value = $_POST['key'] ?? '';のように書く

日本語が文字化けする

原因: 文字コードの設定が不適切

解決方法: HTMLファイルに<meta charset="UTF-8">を追加し、PHPファイルの先頭にheader('Content-Type: text/html; charset=UTF-8');を記述

理解度確認クイズ

  1. Q1. GETメソッドとPOSTメソッドの主な違いを説明してください。

    回答を表示

    GET: データがURLに表示され、ブックマーク可能。データ量に制限があり、検索や共有可能なリンクに適している。
    POST: データがURLに表示されず、セキュリティが高い。データ量に制限がなく、ログインや会員登録など機密情報の送信に適している。

  2. Q2. フォームで送信されたデータをPHPで受け取るには、どのようなコードを書きますか？

    回答を表示

    GETメソッドの場合: $value = $_GET['name'] ?? '';
    POSTメソッドの場合: $value = $_POST['name'] ?? '';
    ※ ?? はnull合体演算子で、キーが存在しない場合のデフォルト値を設定できます。

  3. Q3. XSS攻撃を防ぐために使用する関数は何ですか？

    回答を表示

    htmlspecialchars() 関数を使用します。
    推奨される使い方: htmlspecialchars($value, ENT_QUOTES, 'UTF-8')
    この関数は、HTMLの特殊文字をエスケープして、スクリプトがそのまま実行されるのを防ぎます。

  4. Q4. フォームバリデーションはなぜ必要ですか？

    回答を表示

    フォームバリデーションは以下の理由で必要です：
    1. 不正なデータの送信を防ぐ
    2. ユーザーの入力ミスを早期に発見し、適切なエラーメッセージを表示する
    3. データベースに不正なデータが保存されるのを防ぐ
    4. セキュリティリスクを軽減する

第7章のまとめ

  • HTMLフォームは、<form>タグで定義し、action（送信先）とmethod（送信方法）を指定します。
  • GETメソッドはデータをURLに表示し、POSTメソッドはURLに表示しません。用途に応じて使い分けます。
  • PHPでフォームデータを受け取るには、$_GETまたは$_POSTスーパーグローバル変数を使用します。
  • フォームバリデーションにより、必須チェック、形式チェック、文字数チェックなどを行い、不正なデータの送信を防ぎます。
  • XSS攻撃を防ぐために、ユーザー入力を表示する際は必ずhtmlspecialchars()でエスケープ処理を行います。
