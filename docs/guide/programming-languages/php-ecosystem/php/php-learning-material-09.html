セッション管理とCookie

セッションの仕組み、ユーザー情報の保持、Cookieの活用、ログイン機能の基本実装を学びます。

この章で学ぶこと

  • セッションとCookieの仕組みを理解する
  • セッションを使った情報の保持方法を習得する
  • Cookieの設定と取得方法を学ぶ
  • 簡単なログイン機能を実装する

9.1 セッションとは

セッションとは、Webサイトにアクセスしたユーザーの情報を一時的に保持する仕組みです。 HTTPプロトコルはステートレス（状態を保持しない）なので、セッションを使うことで複数のページにまたがって情報を共有できるようになります。

なぜセッションが必要？

例えばオンラインショッピングで、カートに商品を入れた後に別のページに移動しても、 カートの中身を覚えておく必要があります。セッションを使うことで、このような情報を保持できます。

セッションの仕組み

sequenceDiagram participant U as ユーザー（ブラウザ） participant S as Webサーバー participant F as セッションファイル U->>S: 1. 初回アクセス S->>F: 2. セッションID生成＆ファイル作成 S->>U: 3. セッションIDをCookieで送信 U->>S: 4. 2回目以降のアクセス（SessionID付き） S->>F: 5. SessionIDでデータ取得 S->>U: 6. 個別化されたページを表示

PHPでは、サーバー上に一意のセッションIDを生成し、そのIDに紐づくデータをサーバー内に保存します。 セッションIDはCookieとしてブラウザに送られ、次回のアクセス時にそのIDを元にデータを取り出します。

9.2 セッションの基本操作

PHPでセッションを使うには、session_start()関数でセッションを開始します。 その後、$_SESSIONというスーパーグローバル変数を使ってデータの保存と取得を行います。

セッションの開始

session_start.php 
<?php
// セッションの開始（全てのセッション処理の前に必要）
session_start();

// セッションにデータを保存
$_SESSION['username'] = '田中太郎';
$_SESSION['user_id'] = 123;

echo 'セッションを開始し、データを保存しました。';
?>

重要な注意点

session_start()は、HTMLやテキストを出力する前に呼び出す必要があります。 出力後に呼び出すとエラーが発生します。

セッションデータの取得

get_session.php 
<?php
session_start();

// セッションデータの取得
if (isset($_SESSION['username'])) {
    echo 'ようこそ、' . htmlspecialchars($_SESSION['username']) . 'さん！';
} else {
    echo 'ログインしていません。';
}
?>

セッションデータの削除

session_destroy.php 
<?php
session_start();

// 特定のセッション変数を削除
unset($_SESSION['username']);

// 全てのセッション変数を削除
$_SESSION = [];

// セッションを完全に破棄（ログアウト時など）
session_destroy();

echo 'セッションを破棄しました。';
?>

9.3 Cookieとは

Cookie（クッキー）は、Webサーバーがブラウザに保存させる小さなデータファイルです。 セッションと異なり、データはサーバーではなくユーザーのブラウザに保存されます。

セッションとCookieの違い

項目 セッション Cookie
保存場所 サーバー ブラウザ（クライアント）
有効期限 ブラウザを閉じるまで 任意に設定可能
セキュリティ 高い（サーバー内） 低い（ユーザーが見れる）
保存容量 サーバー容量次第 約4KB程度
用途 ログイン状態、カート情報 ユーザー設定、訪問履歴

Cookieの設定

set_cookie.php 
<?php
// Cookieの設定（出力前に行う）
// setcookie(名前, 値, 有効期限, パス, ドメイン, HTTPS, HTTPOnly)

// 1時間後に期限切れとなるCookieを設定
setcookie('user_name', '田中太郎', time() + 3600, '/');

// 30日間保持されるCookieを設定
setcookie('theme', 'dark', time() + (86400 * 30), '/');

echo 'Cookieを設定しました。';
?>

Cookieの取得

get_cookie.php 
<?php
// Cookieの取得
if (isset($_COOKIE['user_name'])) {
    echo 'ようこそ、' . htmlspecialchars($_COOKIE['user_name']) . 'さん！';
} else {
    echo 'Cookieが設定されていません。';
}

// テーマ設定の取得
$theme = isset($_COOKIE['theme']) ? $_COOKIE['theme'] : 'light';
echo '現在のテーマ: ' . htmlspecialchars($theme);
?>

Cookieの削除

delete_cookie.php 
<?php
// Cookieを削除（有効期限を過去にする）
setcookie('user_name', '', time() - 3600, '/');

echo 'Cookieを削除しました。';
?>

9.4 簡単なログイン機能の実装

セッションを使って、簡単なログイン機能を実装してみましょう。 実際のシステムではパスワードのハッシュ化など、より高度なセキュリティ対策が必要ですが、 ここでは基本的な流れを学びます。

ログインフォーム（login.html）

login.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>ログイン</title>
</head>
<body>
    <h1>ログイン</h1>
    <form action="login.php" method="post">
        <label>ユーザー名: <input type="text" name="username" required></label><br>
        <label>パスワード: <input type="password" name="password" required></label><br>
        <button type="submit">ログイン</button>
    </form>
</body>
</html>

ログイン処理（login.php）

login.php 
<?php
session_start();

// ダミーの認証情報（実際はデータベースから取得）
$valid_username = 'admin';
$valid_password = 'password123';

// フォームから送信されたデータを取得
$username = $_POST['username'] ?? '';
$password = $_POST['password'] ?? '';

// 認証チェック
if ($username === $valid_username && $password === $valid_password) {
    // ログイン成功
    $_SESSION['logged_in'] = true;
    $_SESSION['username'] = $username;
    header('Location: welcome.php');
    exit;
} else {
    // ログイン失敗
    echo 'ユーザー名またはパスワードが間違っています。';
    echo '<br><a href="login.html">戻る</a>';
}
?>

ログイン後のページ（welcome.php）

welcome.php 
<?php
session_start();

// ログインチェック
if (!isset($_SESSION['logged_in']) || $_SESSION['logged_in'] !== true) {
    // 未ログインの場合はログインページへリダイレクト
    header('Location: login.html');
    exit;
}

// ログイン済みの場合の処理
$username = htmlspecialchars($_SESSION['username']);
?>
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>マイページ</title>
</head>
<body>
    <h1>ようこそ、<?php echo $username; ?>さん！</h1>
    <p>これはログイン後のページです。</p>
    <a href="logout.php">ログアウト</a>
</body>
</html>

ログアウト処理（logout.php）

logout.php 
<?php
session_start();

// セッション変数を全て削除
$_SESSION = [];

// セッションを破棄
session_destroy();

// Cookieも削除（セッションIDのCookie）
if (isset($_COOKIE[session_name()])) {
    setcookie(session_name(), '', time() - 3600, '/');
}

echo 'ログアウトしました。';
echo '<br><a href="login.html">ログインページへ</a>';
?>

実習課題

課題1: セッションカウンター

ページを訪問するたびにカウントが増えるセッションカウンターを作成してください。

ヒント: $_SESSION['count'] を使って訪問回数を記録します。

課題2: テーマ切り替え機能

Cookieを使って、ライトモードとダークモードを切り替えられる機能を実装してください。

ヒント: Cookieで選択したテーマを保存し、次回訪問時に反映させます。

課題3: 複数ユーザー対応ログイン

配列を使って複数のユーザー情報（ユーザー名とパスワード）を管理し、 どのユーザーでもログインできる仕組みを作成してください。

ヒント: 連想配列でユーザー情報を管理します。

9.5 セキュリティ上の注意点

セッションとCookieを扱う際には、セキュリティに十分注意する必要があります。

セッションハイジャック対策

セッションIDを盗まれると、なりすましログインが可能になります。 

// ログイン成功時にセッションIDを再生成
session_regenerate_id(true);

HTTPS通信の使用

通信が暗号化されていないと、セッションIDやCookieが盗聴される危険があります。

  • 本番環境では必ずHTTPSを使用
  • Cookie設定でsecure属性を有効化

XSS対策

セッションやCookieから取得したデータを出力する際は、必ずエスケープ処理を行います。 

// 安全な出力
echo htmlspecialchars($_SESSION['username']);

タイムアウト設定

長時間放置されたセッションは自動的に無効化する仕組みを設けます。 

// 30分間操作がなければログアウト
if (time() - $_SESSION['last_activity'] > 1800) {
    session_destroy();
}

パスワードの取り扱い

実際のシステムでは、パスワードは必ずpassword_hash()でハッシュ化して保存し、 password_verify()で検証します。平文での保存は絶対に避けてください。

理解度確認クイズ

  1. Q1. セッションとCookieの最も大きな違いは何ですか？

    回答を表示

    保存場所の違いです。セッションはサーバー側にデータを保存し、Cookieはクライアント（ブラウザ）側にデータを保存します。 そのため、セキュリティ面ではセッションの方が安全で、重要な情報はセッションで管理すべきです。

  2. Q2. session_start()をHTMLタグの前に記述する必要があるのはなぜですか？

    回答を表示

    session_start()はHTTPヘッダーを送信する処理だからです。HTTPヘッダーはHTMLコンテンツより先に送信される必要があり、 HTMLタグやテキストを出力した後にヘッダーを送ろうとするとエラーが発生します。

  3. Q3. セッションハイジャック対策として有効な方法を1つ挙げてください。

    回答を表示

    ログイン成功時にセッションIDを再生成する（session_regenerate_id(true)を使用）ことが有効です。 他にも、HTTPS通信の使用、タイムアウト設定、IPアドレスやユーザーエージェントの確認なども有効な対策です。

  4. Q4. Cookieを削除するにはどうすればよいですか？

    回答を表示

    setcookie()関数で有効期限を過去の時刻に設定します。 例: setcookie('name', '', time() - 3600, '/');
    これにより、ブラウザが該当のCookieを削除します。

第9章のまとめ

  • セッションは、サーバー側にユーザー情報を保存し、複数のページ間で情報を共有する仕組みです。
  • セッションを使うにはsession_start()を出力前に呼び出し、$_SESSION配列でデータを管理します。
  • Cookieは、ブラウザ側にデータを保存する仕組みで、有効期限を自由に設定できます。
  • Cookieはセキュリティ面でセッションより劣るため、重要な情報はセッションで管理します。
  • ログイン機能では、認証後にセッションにユーザー情報を保存し、各ページでログイン状態をチェックします。
  • セキュリティ対策として、セッションIDの再生成、HTTPS通信、XSS対策、タイムアウト設定が重要です。
