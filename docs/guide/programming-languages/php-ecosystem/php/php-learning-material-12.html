オブジェクト指向プログラミング入門

クラスとオブジェクトの概念、プロパティとメソッド、継承の基礎など、OOPの基本的な考え方を理解します。

この章で学ぶこと

  • オブジェクト指向プログラミング（OOP）の基本概念を理解する
  • クラスとオブジェクトの違いを理解し、定義・使用できる
  • プロパティとメソッドの使い方を習得する
  • アクセス修飾子とカプセル化の意義を学ぶ
  • 継承の基礎を理解し、コードの再利用ができる

12.1 オブジェクト指向プログラミングとは

オブジェクト指向プログラミング（Object-Oriented Programming: OOP）は、 プログラムを「オブジェクト」という単位で組み立てる考え方です。 現実世界のモノや概念をプログラムで表現し、それらを組み合わせてシステムを構築します。

例えば、「車」をプログラムで表現する場合を考えてみましょう。 車には「色」「速度」といった属性（データ）があり、 「走る」「止まる」といった動作（機能）があります。 OOPでは、これらをまとめて1つの「オブジェクト」として扱います。

なぜOOPが必要なのか？

複雑なプログラムを作る時、すべてを手続き型（順番に処理を書く方法）で書くと、 コードが長く複雑になり、管理が大変になります。OOPを使うことで、 プログラムを整理しやすく、再利用しやすく、保守しやすくできます。

OOPの3つの主要な特徴

カプセル化

データと処理をまとめて、外部から不要な操作をできないようにする仕組み。

継承

既存のクラスの機能を引き継いで、新しいクラスを作る仕組み。コードの再利用が可能に。

ポリモーフィズム

同じ名前のメソッドでも、オブジェクトによって異なる動作をさせられる仕組み。

12.2 クラスとオブジェクトの基本

OOPを理解する上で最も重要な概念がクラスとオブジェクトです。

クラスとは

クラスは、オブジェクトの「設計図」や「型」のようなものです。 クラスには、オブジェクトが持つ「データ（プロパティ）」と「機能（メソッド）」を定義します。

オブジェクトとは

オブジェクトは、クラスという設計図から作られた「実体」です。 1つのクラスから複数のオブジェクトを作ることができます。

クラスの定義

PHPでクラスを定義するには、classキーワードを使います。

Car.php 
<?php
// Carクラスの定義
class Car {
    // プロパティ（属性）
    public $color;  // 車の色
    public $speed;  // 現在の速度

    // メソッド（動作）
    public function accelerate() {
        $this->speed += 10;  // 速度を10増やす
        echo "加速しました！現在の速度: {$this->speed}km/h\n";
    }

    public function brake() {
        $this->speed = 0;  // 速度を0にする
        echo "停車しました。\n";
    }
}
?>

重要な用語

  • プロパティ: クラスが持つ変数（データ）
  • メソッド: クラスが持つ関数（動作）
  • $this: 自分自身のオブジェクトを指すキーワード

12.3 オブジェクトの生成と使用

クラスからオブジェクトを作るには、newキーワードを使います。

use_car.php 
<?php
// Carクラスを使う
require_once 'Car.php';

// オブジェクトの生成（インスタンス化）
$myCar = new Car();

// プロパティに値を設定
$myCar->color = "赤";
$myCar->speed = 0;

// プロパティの値を表示
echo "私の車の色: {$myCar->color}\n";
echo "現在の速度: {$myCar->speed}km/h\n\n";

// メソッドを呼び出す
$myCar->accelerate();  // 出力: 加速しました！現在の速度: 10km/h
$myCar->accelerate();  // 出力: 加速しました！現在の速度: 20km/h
$myCar->brake();       // 出力: 停車しました。

// 別のオブジェクトを作成
$yourCar = new Car();
$yourCar->color = "青";
$yourCar->speed = 50;

echo "\nあなたの車の色: {$yourCar->color}\n";
echo "現在の速度: {$yourCar->speed}km/h\n";
?>

オブジェクトへのアクセス方法

オブジェクトのプロパティやメソッドにアクセスするには、->演算子（アロー演算子）を使います。
例: $myCar->color、$myCar->accelerate()

12.4 コンストラクタ

コンストラクタは、オブジェクトが生成されたときに自動的に実行される特別なメソッドです。 主に、プロパティの初期値を設定するために使います。

Car_v2.php 
<?php
class Car {
    public $color;
    public $speed;

    // コンストラクタ（__constructという名前の特別なメソッド）
    public function __construct($color) {
        $this->color = $color;
        $this->speed = 0;  // 最初は停止状態
        echo "{$color}の車が作られました！\n";
    }

    public function accelerate() {
        $this->speed += 10;
        echo "加速しました！現在の速度: {$this->speed}km/h\n";
    }

    public function brake() {
        $this->speed = 0;
        echo "停車しました。\n";
    }
}

// コンストラクタに引数を渡してオブジェクトを生成
$myCar = new Car("赤");     // 出力: 赤の車が作られました！
$yourCar = new Car("青");   // 出力: 青の車が作られました！

$myCar->accelerate();       // 出力: 加速しました！現在の速度: 10km/h
?>

コンストラクタの利点

  • オブジェクト生成時に必要な初期設定を自動で行える
  • 必須のプロパティ値を強制的に設定させられる
  • 初期化処理をまとめて書けるので、コードがすっきりする

12.5 アクセス修飾子とカプセル化

アクセス修飾子は、プロパティやメソッドへのアクセスを制限するキーワードです。 これにより、カプセル化（データを保護し、不正な操作を防ぐ）を実現できます。

3種類のアクセス修飾子

修飾子 アクセス可能範囲 用途
public どこからでもアクセス可能 外部に公開したいメソッド
private 同じクラス内のみ 内部でのみ使うプロパティ・メソッド
protected 同じクラスと継承先クラス 継承先にも使わせたいもの
Car_v3.php 
<?php
class Car {
    // プロパティをprivateにして外部から直接変更できないようにする
    private $color;
    private $speed;

    public function __construct($color) {
        $this->color = $color;
        $this->speed = 0;
    }

    // ゲッター：プロパティの値を取得するメソッド
    public function getColor() {
        return $this->color;
    }

    public function getSpeed() {
        return $this->speed;
    }

    // セッター：プロパティの値を設定するメソッド（検証付き）
    public function setColor($color) {
        // 許可された色のみ設定可能
        $allowedColors = ['赤', '青', '白', '黒'];
        if (in_array($color, $allowedColors)) {
            $this->color = $color;
            echo "色を{$color}に変更しました。\n";
        } else {
            echo "エラー: その色は選択できません。\n";
        }
    }

    public function accelerate() {
        $this->speed += 10;
        echo "現在の速度: {$this->speed}km/h\n";
    }
}

// 使用例
$myCar = new Car("赤");
echo "車の色: " . $myCar->getColor() . "\n";  // 赤

$myCar->setColor("青");    // 色を青に変更しました。
$myCar->setColor("緑");    // エラー: その色は選択できません。

// 直接アクセスしようとするとエラー
// $myCar->color = "緑";  // Fatal error!
?>

なぜカプセル化が重要なのか

プロパティをpublicにすると、外部から不正な値を設定される可能性があります。 privateにして、ゲッター/セッターを通じてアクセスすることで、 値の検証やエラー処理を一箇所で管理できます。

12.6 継承の基礎

継承は、既存のクラスの機能を引き継いで、新しいクラスを作る仕組みです。 これにより、コードの重複を減らし、再利用性を高めることができます。

継承の関係

（乗り物）
プロパティ: $speed
メソッド: move()"] B["子クラス: Car
（車）

（自転車）

継承の実装

継承するには、extendsキーワードを使います。

Vehicle.php 
<?php
// 親クラス（基底クラス）
class Vehicle {
    protected $speed;  // 継承先でも使えるようにprotected

    public function __construct() {
        $this->speed = 0;
    }

    public function move() {
        echo "移動中... 速度: {$this->speed}km/h\n";
    }

    public function getSpeed() {
        return $this->speed;
    }
}

// 子クラス（派生クラス） - Vehicleを継承
class Car extends Vehicle {
    private $color;

    public function __construct($color) {
        parent::__construct();  // 親のコンストラクタを呼び出し
        $this->color = $color;
    }

    // 子クラス独自のメソッド
    public function accelerate() {
        $this->speed += 20;  // 親のプロパティにアクセス可能
        echo "車が加速！ 速度: {$this->speed}km/h\n";
    }
}

// 別の子クラス
class Bicycle extends Vehicle {
    public function pedal() {
        $this->speed += 5;
        echo "自転車をこいだ！ 速度: {$this->speed}km/h\n";
    }
}

// 使用例
$car = new Car("赤");
$car->accelerate();  // 車が加速！ 速度: 20km/h
$car->move();        // 移動中... 速度: 20km/h

$bike = new Bicycle();
$bike->pedal();      // 自転車をこいだ！ 速度: 5km/h
$bike->move();       // 移動中... 速度: 5km/h
?>

継承のメリット

  • 共通の処理を親クラスにまとめられる（コードの重複を避けられる）
  • 親クラスの修正が子クラスにも自動的に反映される
  • プログラムの構造が整理され、理解しやすくなる

メソッドのオーバーライド

子クラスで親クラスのメソッドを上書き（再定義）することをオーバーライドといいます。

override_example.php 
<?php
class Vehicle {
    protected $speed = 0;

    public function move() {
        echo "移動中... 速度: {$this->speed}km/h\n";
    }
}

class Car extends Vehicle {
    // 親のmove()メソッドをオーバーライド
    public function move() {
        echo "車が道路を走行中... 速度: {$this->speed}km/h\n";
    }
}

$vehicle = new Vehicle();
$vehicle->move();  // 移動中... 速度: 0km/h

$car = new Car();
$car->move();      // 車が道路を走行中... 速度: 0km/h
?>

実習 12-1: Userクラスの作成

簡単なUserクラスを作成し、オブジェクト指向の基本を実践してみましょう。

手順

  1. 1 User.phpを作成し、以下の仕様でUserクラスを定義する
    - プロパティ: $name（名前）、$email（メールアドレス）
    - コンストラクタ: 名前とメールアドレスを受け取る
    - メソッド: introduce()（自己紹介文を出力）
  2. 2 プロパティをprivateにし、ゲッター・セッターを作成する
  3. 3 Userクラスを継承したAdminUserクラスを作成し、deleteUser()メソッドを追加する
  4. 4 複数のオブジェクトを作成し、各メソッドを実行してみる
解答例を表示 
<?php
class User {
    private $name;
    private $email;

    public function __construct($name, $email) {
        $this->name = $name;
        $this->email = $email;
    }

    public function getName() {
        return $this->name;
    }

    public function getEmail() {
        return $this->email;
    }

    public function setName($name) {
        $this->name = $name;
    }

    public function setEmail($email) {
        $this->email = $email;
    }

    public function introduce() {
        echo "こんにちは、{$this->name}です。\n";
        echo "メール: {$this->email}\n";
    }
}

class AdminUser extends User {
    public function deleteUser($username) {
        echo "[管理者権限] {$username}を削除しました。\n";
    }
}

// 使用例
$user1 = new User("田中太郎", "tanaka@example.com");
$user1->introduce();

$admin = new AdminUser("管理者", "admin@example.com");
$admin->introduce();
$admin->deleteUser("田中太郎");
?>

理解度確認クイズ

  1. Q1. クラスとオブジェクトの違いを説明してください。

    回答を表示

    クラスは、オブジェクトの「設計図」や「型」です。データ（プロパティ）と機能（メソッド）の定義を含みます。
    オブジェクトは、クラスから作られた「実体」です。1つのクラスから複数のオブジェクトを作ることができます。

  2. Q2. コンストラクタは何のために使いますか？

    回答を表示

    コンストラクタは、オブジェクトが生成されたときに自動的に実行される特別なメソッドです。 主にプロパティの初期値を設定したり、オブジェクト生成時に必要な初期化処理を行ったりするために使います。

  3. Q3. 3種類のアクセス修飾子（public、private、protected）の違いを説明してください。

    回答を表示

    public: どこからでもアクセス可能。外部に公開したいメソッドに使う。
    private: 同じクラス内のみアクセス可能。内部でのみ使うプロパティ・メソッドに使う。
    protected: 同じクラスと継承先クラスからアクセス可能。継承先にも使わせたいものに使う。

  4. Q4. 継承のメリットを2つ挙げてください。

    回答を表示

    1. コードの重複を避けられる（共通の処理を親クラスにまとめられる）
    2. コードの再利用性が高まる（既存のクラスの機能を引き継げる）
    ※その他、保守性の向上、プログラムの構造が整理されるなども該当します。

第12章のまとめ

  • オブジェクト指向プログラミング（OOP）は、プログラムを「オブジェクト」という単位で組み立てる考え方です。
  • クラスは設計図、オブジェクトはその実体です。newキーワードでオブジェクトを生成します。
  • プロパティはクラスが持つデータ、メソッドはクラスが持つ機能です。
  • コンストラクタは、オブジェクト生成時に自動実行される特別なメソッドです。
  • アクセス修飾子（public/private/protected）でカプセル化を実現し、データを保護します。
  • 継承を使うと、既存クラスの機能を引き継いで新しいクラスを作れます（extendsキーワード）。
