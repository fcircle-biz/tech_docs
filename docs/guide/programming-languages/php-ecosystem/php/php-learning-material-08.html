ファイル操作

ファイルの読み書き、CSVファイルの処理、ディレクトリ操作など、ファイルシステムを扱う方法を習得します。

この章で学ぶこと

  • ファイルの読み込み・書き込みの基本を理解する
  • CSVファイルの処理方法を習得する
  • ディレクトリ操作の基本を学ぶ
  • ファイル操作のエラーハンドリングを理解する

8.1 ファイル操作の基本

PHPでは、テキストファイルやバイナリファイルを読み書きするための豊富な関数が用意されています。 ファイル操作は、ログの記録、データの永続化、外部システムとの連携など、様々な場面で活用されます。

ファイル操作の基本的な流れ

flowchart TD A([開始]) --> B[ファイルを開く] B --> C{成功?} C -- はい --> D[ファイルを読む/書く] C -- いいえ --> E[エラー処理] D --> F[ファイルを閉じる] E --> G([終了]) F --> G

ファイル操作の3ステップ

1. ファイルを開く (fopen)
2. 読み書きする (fread, fwrite等)
3. ファイルを閉じる (fclose)
この流れを守ることで、安全にファイルを扱うことができます。

8.2 ファイルの読み込み

PHPには、ファイルを読み込むための複数の方法があります。 ファイルのサイズや用途に応じて、最適な方法を選択することが重要です。

方法1: file_get_contents（ファイル全体を一度に読み込む）

read_all.php 
<?php
// ファイル全体を文字列として読み込む
$content = file_get_contents('data.txt');

if ($content !== false) {
    echo "ファイルの内容:\n";
    echo $content;
} else {
    echo "ファイルの読み込みに失敗しました";
}
?>

方法2: file（1行ずつ配列に格納）

read_lines.php 
<?php
// ファイルを1行ずつ配列に読み込む
$lines = file('data.txt');

if ($lines !== false) {
    foreach ($lines as $number => $line) {
        // 行番号は0から始まる
        echo "行" . ($number + 1) . ": " . $line;
    }
}
?>

方法3: fopen/fgets（大きなファイルに適した方法）

read_stream.php 
<?php
// ファイルを開く（rは読み込み専用モード）
$file = fopen('data.txt', 'r');

if ($file) {
    // ファイルの終わりまで1行ずつ読み込む
    while (($line = fgets($file)) !== false) {
        echo $line;
    }
    // ファイルを閉じる
    fclose($file);
}
?>
関数 特徴 適した場面
file_get_contents 全体を一度に読み込む 小さなファイル
file 行ごとに配列に格納 行単位で処理したい場合
fopen/fgets 1行ずつ順次読み込む 大きなファイル（メモリ節約）

8.3 ファイルの書き込み

ファイルへの書き込みは、ログの記録、設定ファイルの保存、データのエクスポートなど、様々な場面で使われます。

方法1: file_put_contents（ファイル全体を書き込む）

write_all.php 
<?php
$content = "これはテストファイルです\n";
$content .= "2行目のテキストです\n";

// ファイルに書き込む（既存の内容は上書きされる）
$result = file_put_contents('output.txt', $content);

if ($result !== false) {
    echo "書き込み成功: {$result}バイト";
} else {
    echo "書き込み失敗";
}
?>

方法2: 追記モード（既存の内容を保持）

append.php 
<?php
$log = date('Y-m-d H:i:s') . " - アクセスログ\n";

// FILE_APPENDフラグで追記
file_put_contents('log.txt', $log, FILE_APPEND);

echo "ログを追記しました";
?>

方法3: fopen/fwrite（細かい制御が必要な場合）

write_stream.php 
<?php
// wは書き込み専用モード（既存の内容は削除される）
$file = fopen('output.txt', 'w');

if ($file) {
    fwrite($file, "1行目\n");
    fwrite($file, "2行目\n");
    fclose($file);
    echo "書き込み成功";
}
?>
モード 説明
r 読み込み専用
w 書き込み専用（既存の内容は削除）
a 追記モード（既存の内容を保持）
r+ 読み書き両用

8.4 CSVファイルの処理

CSV（Comma-Separated Values）は、データを扱う際に頻繁に使われるファイル形式です。 ExcelやGoogleスプレッドシートのデータをCSV形式でエクスポートし、PHPで処理することができます。

CSVファイルの読み込み

read_csv.php 
<?php
$file = fopen('data.csv', 'r');

if ($file) {
    // 1行目（ヘッダー行）を読み込む
    $header = fgetcsv($file);
    echo "列名: " . implode(', ', $header) . "\n\n";

    // データ行を読み込む
    while (($row = fgetcsv($file)) !== false) {
        echo "名前: {$row[0]}, 年齢: {$row[1]}\n";
    }

    fclose($file);
}
?>

CSVファイルの書き込み

write_csv.php 
<?php
$file = fopen('export.csv', 'w');

if ($file) {
    // ヘッダー行を書き込む
    $header = ['名前', '年齢', '職業'];
    fputcsv($file, $header);

    // データ行を書き込む
    $data = [
        ['田中太郎', 30, 'エンジニア'],
        ['佐藤花子', 25, 'デザイナー'],
        ['鈴木一郎', 35, 'マネージャー']
    ];

    foreach ($data as $row) {
        fputcsv($file, $row);
    }

    fclose($file);
    echo "CSVファイルを作成しました";
}
?>

fgetcsv/fputcsvの利点

これらの関数は、カンマやダブルクォートを含むデータを自動的に正しく処理してくれます。 自分で文字列を分割する必要がないため、安全で確実です。

8.5 ディレクトリ操作

ファイルの管理には、ディレクトリ（フォルダ）の操作も重要です。 ディレクトリの作成、削除、ファイル一覧の取得などの操作を学びます。

ディレクトリの作成と削除

directory_basic.php 
<?php
// ディレクトリを作成
if (!file_exists('uploads')) {
    mkdir('uploads', 0755);
    echo "ディレクトリを作成しました\n";
}

// ディレクトリの存在確認
if (is_dir('uploads')) {
    echo "uploadsディレクトリが存在します\n";
}

// 空のディレクトリを削除
// rmdir('uploads');
?>

ディレクトリ内のファイル一覧を取得

list_files.php 
<?php
// scandir: ファイル一覧を配列で取得
$files = scandir('.');

echo "現在のディレクトリの内容:\n";
foreach ($files as $file) {
    // .と..（カレント/親ディレクトリ）をスキップ
    if ($file !== '.' && $file !== '..') {
        if (is_file($file)) {
            echo "📄 {$file}\n";
        } elseif (is_dir($file)) {
            echo "📁 {$file}\n";
        }
    }
}
?>

ファイル情報の取得

file_info.php 
<?php
$filename = 'data.txt';

if (file_exists($filename)) {
    // ファイルサイズ（バイト）
    $size = filesize($filename);
    echo "サイズ: {$size}バイト\n";

    // 最終更新日時
    $mtime = filemtime($filename);
    echo "更新日時: " . date('Y-m-d H:i:s', $mtime) . "\n";

    // ファイルの種類
    echo "ファイル: " . (is_file($filename) ? 'はい' : 'いいえ') . "\n";
}
?>

8.6 エラーハンドリングとセキュリティ

ファイル操作では、様々なエラーが発生する可能性があります。 安全で堅牢なプログラムを作るために、適切なエラーハンドリングが必要です。

エラーハンドリングの実装

error_handling.php 
<?php
$filename = 'data.txt';

// ファイルの存在確認
if (!file_exists($filename)) {
    die("エラー: ファイルが見つかりません");
}

// 読み込み権限の確認
if (!is_readable($filename)) {
    die("エラー: ファイルを読み込めません");
}

// ファイルを安全に読み込む
$content = @file_get_contents($filename);
if ($content === false) {
    die("エラー: ファイルの読み込みに失敗しました");
}

echo "読み込み成功";
?>

セキュリティ上の注意点

1. ユーザー入力を直接ファイル名に使わない: パストラバーサル攻撃のリスク
2. アップロードファイルの検証: ファイルタイプと拡張子をチェック
3. 権限設定: ファイルやディレクトリに適切な権限を設定
4. 機密情報の保護: パスワードなどはファイルに平文で保存しない

安全なファイル名の処理

safe_filename.php 
<?php
// ユーザーからのファイル名入力を受け取る（仮定）
$userInput = $_GET['file'] ?? '';

// 危険な文字を除去
$safeName = basename($userInput);

// さらに、許可されたディレクトリ内のみアクセス
$allowedDir = 'uploads/';
$fullPath = $allowedDir . $safeName;

// パストラバーサル攻撃を防ぐ
$realPath = realpath($fullPath);
if ($realPath && strpos($realPath, realpath($allowedDir)) === 0) {
    // 安全なパスの場合のみ処理
    echo "安全なファイルパス: {$realPath}";
}
?>

実習 8-1: 簡単なアクセスログシステム

訪問者のアクセス情報を記録するシンプルなログシステムを作成します。

手順

  1. 1 access_log.phpを作成し、現在の日時とアクセス元IPアドレスを取得する
  2. 2 file_put_contentsを使って、ログ情報をaccess.logファイルに追記する
  3. 3 view_log.phpを作成し、ログファイルの内容を読み込んで表示する
  4. 4 ブラウザで複数回アクセスして、ログが正しく記録されることを確認する

理解度確認クイズ

  1. Q1. ファイルを開いて操作した後、必ず行うべき処理は何ですか？

    回答を表示

    fclose()を使ってファイルを閉じる必要があります。ファイルを開いたままにすると、リソースが解放されず、他のプロセスがファイルにアクセスできなくなる可能性があります。

  2. Q2. 既存のファイルに新しい内容を追加（上書きしない）したい場合、どのモードを使用しますか？

    回答を表示

    追記モード（'a'）を使用します。fopenで'a'モードを指定するか、file_put_contentsでFILE_APPENDフラグを使用します。これにより、既存の内容を保持したまま、ファイルの末尾に新しい内容を追加できます。

  3. Q3. CSVファイルを読み込む際に使用する関数は何ですか？

    回答を表示

    fgetcsv()関数を使用します。この関数は、CSV形式の1行を読み込んで配列として返します。カンマやダブルクォートを含むデータも正しく処理してくれます。

  4. Q4. ユーザーから受け取ったファイル名を使用する際、セキュリティ上注意すべきことは何ですか？

    回答を表示

    パストラバーサル攻撃を防ぐため、basename()で危険な文字（../など）を除去し、realpath()で実際のパスを確認してから、許可されたディレクトリ内にあることをチェックする必要があります。ユーザー入力を直接ファイルパスに使用してはいけません。

第8章のまとめ

  • ファイル操作は開く→読み書き→閉じるの3ステップで行います。
  • ファイルの読み込みには、file_get_contents、file、fopen/fgetsなど、用途に応じた方法があります。
  • ファイルへの書き込みはfile_put_contentsが簡単で、追記にはFILE_APPENDフラグを使います。
  • CSVファイルはfgetcsv/fputcsvを使うことで、安全かつ簡単に処理できます。
  • ディレクトリ操作にはmkdir、scandir、is_dirなどの関数を使用します。
  • ファイル操作では、エラーハンドリングとセキュリティ対策が重要です。
