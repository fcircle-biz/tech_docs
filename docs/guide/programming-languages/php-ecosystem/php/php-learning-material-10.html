データベース連携の基礎

MySQLの基礎、PDOを使用したデータベース接続、基本的なCRUD操作の実装方法を習得します。

この章で学ぶこと

  • データベースとは何かを理解する
  • MySQLの基本操作を知る
  • PDOを使ったデータベース接続方法を習得する
  • CRUD操作（作成・読取・更新・削除）の実装方法を学ぶ

10.1 データベースとは

データベースとは、大量のデータを効率的に保存・管理・検索するための仕組みです。 Excelのように表形式でデータを整理し、必要な情報を素早く取り出すことができます。

データベースが必要な理由

  • 大量のデータを管理: 何千、何万件ものデータも効率的に扱える
  • データの永続化: サーバーを再起動してもデータが消えない
  • 高速な検索: 条件に合うデータを素早く見つけられる
  • 複数ユーザーの同時アクセス: 多くの人が同時に利用できる

データベースの構造

graph TB A[データベース] --> B[テーブル1: users
ユーザー情報] A --> C[テーブル2: posts
投稿情報] A --> D[テーブル3: comments
コメント情報] B --> B1[id, name, email, password] C --> C1[id, user_id, title, content] D --> D1[id, post_id, user_id, comment]

データベースとファイルの違い

ファイル（テキストファイル、CSVなど）でもデータを保存できますが、データベースは検索が高速で、複数ユーザーでの同時編集にも対応しています。

10.2 MySQLの基礎

MySQL（マイエスキューエル）は、世界中で広く使われている無料のデータベース管理システム（DBMS）です。 PHPとの相性が良く、XAMPP/MAMPにも最初から含まれています。

基本用語

用語 説明 例
データベース 複数のテーブルをまとめたもの my_website
テーブル データを保存する表（Excelのシートに似ています） users, posts
カラム（列） データの項目名 id, name, email
レコード（行） 1件のデータ id=1, name=太郎

データベースとテーブルの作成

まずは、データベースとテーブルを作成します。以下のSQL文をphpMyAdminで実行してください。

SQL 
-- データベースの作成
CREATE DATABASE my_database;

-- データベースの選択
USE my_database;

-- テーブルの作成
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

主キー（PRIMARY KEY）とは

主キーは、各レコードを一意に識別するための項目です。通常は自動で番号が振られる「id」列を使います。 AUTO_INCREMENTを指定すると、データ挿入時に自動で番号が付与されます。

10.3 PDOによるデータベース接続

PDO（PHP Data Objects）は、PHPからデータベースに接続するための標準的な方法です。 MySQLだけでなく、PostgreSQLやSQLiteなど、様々なデータベースに対応しています。

データベース接続の基本

db_connect.php 
<?php
// データベース接続情報
$host = 'localhost';
$dbname = 'my_database';
$user = 'root';
$password = '';

try {
    // PDOオブジェクトの作成
    $pdo = new PDO("mysql:host=$host;dbname=$dbname;charset=utf8", $user, $password);

    // エラーモードを例外に設定
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

    echo "データベース接続成功！";
} catch (PDOException $e) {
    echo "接続エラー: " . $e->getMessage();
}
?>

接続情報の管理

データベースのパスワードなどの機密情報は、別ファイルに分けて管理し、Gitリポジトリにアップロードしないようにしましょう。 本番環境では、環境変数を使うことが推奨されます。

10.4 データの挿入（Create）

CRUD操作の「C（Create）」は、新しいデータをデータベースに挿入する操作です。 SQLのINSERT文を使います。

基本的なデータ挿入

insert_user.php 
<?php
// データベース接続（上記のdb_connect.phpを読み込む）
require 'db_connect.php';

// 挿入するデータ
$name = "山田太郎";
$email = "yamada@example.com";

try {
    // SQL文の準備（プリペアドステートメント）
    $sql = "INSERT INTO users (name, email) VALUES (:name, :email)";
    $stmt = $pdo->prepare($sql);

    // 値をバインド
    $stmt->bindParam(':name', $name);
    $stmt->bindParam(':email', $email);

    // SQL文の実行
    $stmt->execute();

    echo "データを挿入しました！";
    echo "挿入されたID: " . $pdo->lastInsertId();
} catch (PDOException $e) {
    echo "エラー: " . $e->getMessage();
}
?>

プリペアドステートメントの重要性

:nameのようなプレースホルダーを使い、後から値を結び付ける方式を「プリペアドステートメント」と呼びます。 この方式は、SQLインジェクション攻撃を防ぐために必須です。

10.5 データの取得（Read）

CRUD操作の「R（Read）」は、データベースからデータを取得する操作です。 SQLのSELECT文を使います。

全データの取得

select_all.php 
<?php
require 'db_connect.php';

try {
    // SQL文の実行
    $sql = "SELECT * FROM users";
    $stmt = $pdo->query($sql);

    // 全レコードを取得
    $users = $stmt->fetchAll(PDO::FETCH_ASSOC);

    // データの表示
    foreach ($users as $user) {
        echo "ID: " . $user['id'] . "<br>";
        echo "名前: " . $user['name'] . "<br>";
        echo "メール: " . $user['email'] . "<br><hr>";
    }
} catch (PDOException $e) {
    echo "エラー: " . $e->getMessage();
}
?>

条件付きでデータを取得

select_by_id.php 
<?php
require 'db_connect.php';

$user_id = 1; // 取得したいユーザーのID

try {
    $sql = "SELECT * FROM users WHERE id = :id";
    $stmt = $pdo->prepare($sql);
    $stmt->bindParam(':id', $user_id, PDO::PARAM_INT);
    $stmt->execute();

    // 1件のみ取得
    $user = $stmt->fetch(PDO::FETCH_ASSOC);

    if ($user) {
        echo "名前: " . $user['name'];
    } else {
        echo "ユーザーが見つかりません";
    }
} catch (PDOException $e) {
    echo "エラー: " . $e->getMessage();
}
?>

fetchとfetchAllの違い

fetch()は1件のレコードを取得、fetchAll()は全レコードを配列で取得します。 IDで検索する場合など、1件だけ取得したい場合はfetch()を使いましょう。

10.6 データの更新（Update）

CRUD操作の「U（Update）」は、既存のデータを更新する操作です。 SQLのUPDATE文を使います。

update_user.php 
<?php
require 'db_connect.php';

// 更新するデータ
$user_id = 1;
$new_name = "鈴木一郎";
$new_email = "suzuki@example.com";

try {
    $sql = "UPDATE users SET name = :name, email = :email WHERE id = :id";
    $stmt = $pdo->prepare($sql);

    $stmt->bindParam(':name', $new_name);
    $stmt->bindParam(':email', $new_email);
    $stmt->bindParam(':id', $user_id, PDO::PARAM_INT);

    $stmt->execute();

    // 更新された行数を確認
    $count = $stmt->rowCount();
    echo "{$count}件のデータを更新しました";
} catch (PDOException $e) {
    echo "エラー: " . $e->getMessage();
}
?>

WHERE句を忘れずに

UPDATE文でWHERE句を忘れると、テーブル内の全データが更新されてしまいます。 必ず条件を指定しましょう。

10.7 データの削除（Delete）

CRUD操作の「D（Delete）」は、データを削除する操作です。 SQLのDELETE文を使います。

delete_user.php 
<?php
require 'db_connect.php';

$user_id = 1; // 削除するユーザーのID

try {
    $sql = "DELETE FROM users WHERE id = :id";
    $stmt = $pdo->prepare($sql);
    $stmt->bindParam(':id', $user_id, PDO::PARAM_INT);
    $stmt->execute();

    $count = $stmt->rowCount();

    if ($count > 0) {
        echo "{$count}件のデータを削除しました";
    } else {
        echo "削除対象のデータが見つかりません";
    }
} catch (PDOException $e) {
    echo "エラー: " . $e->getMessage();
}
?>

削除は慎重に

削除したデータは復元できません。本番環境では、実際に削除するのではなく、「削除フラグ」を立てる方式が推奨されます。 また、DELETE文でもWHERE句を忘れると全データが削除されるため注意が必要です。

10.8 プリペアドステートメント（セキュリティ対策）

プリペアドステートメントは、SQLインジェクション攻撃を防ぐための必須技術です。 ユーザーからの入力を直接SQL文に埋め込むと、悪意のあるコードが実行されるリスクがあります。

危険な書き方（絶対NG）

悪い例 
<?php
// ❌ 危険！SQLインジェクションの脆弱性あり
$user_id = $_GET['id'];
$sql = "SELECT * FROM users WHERE id = $user_id";
$result = $pdo->query($sql);
?>

安全な書き方

良い例 
<?php
// ✅ 安全！プリペアドステートメントを使用
$user_id = $_GET['id'];
$sql = "SELECT * FROM users WHERE id = :id";
$stmt = $pdo->prepare($sql);
$stmt->bindParam(':id', $user_id, PDO::PARAM_INT);
$stmt->execute();
$user = $stmt->fetch(PDO::FETCH_ASSOC);
?>

SQLインジェクション攻撃の例

sequenceDiagram participant 攻撃者 participant Webサイト participant データベース 攻撃者->>Webサイト: 悪意のある入力
id = "1 OR 1=1" Webサイト->>データベース: SELECT * FROM users
WHERE id = 1 OR 1=1 Note over データベース: 条件が常に真になり
全ユーザー情報が流出 データベース->>Webサイト: 全ユーザーデータ Webサイト->>攻撃者: 全ユーザーデータを表示

プリペアドステートメントの仕組み

プリペアドステートメントでは、SQL文の構造とデータが分離されます。 プレースホルダー（:nameなど）に後から値を結び付けることで、悪意のあるコードの実行を防ぎます。

実習 10-1: 簡単な会員管理システムを作る

データベースを使った簡単な会員管理システムを作成しましょう。

手順

  1. 1 上記のSQL文でテーブルを作成
  2. 2 データベース接続ファイル（db_connect.php）を作成
  3. 3 会員登録フォーム（register.php）を作成し、データを挿入
  4. 4 会員一覧ページ（list.php）を作成し、全データを表示
  5. 5 会員情報の更新・削除機能を追加

理解度確認クイズ

  1. Q1. データベースとファイルの主な違いを3つ挙げてください。

    回答を表示

    1. 大量のデータを効率的に管理できる
    2. 高速な検索機能を備えている
    3. 複数ユーザーが同時にアクセスできる
    ※その他、データの永続化、トランザクション機能などもあります。

  2. Q2. CRUD操作とは何ですか？それぞれのSQL文を答えてください。

    回答を表示

    CRUD操作とは、データベースの基本操作です。
    - C（Create）: 作成 → INSERT文
    - R（Read）: 読取 → SELECT文
    - U（Update）: 更新 → UPDATE文
    - D（Delete）: 削除 → DELETE文

  3. Q3. プリペアドステートメントを使う理由は何ですか？

    回答を表示

    SQLインジェクション攻撃を防ぐためです。 ユーザーからの入力を直接SQL文に埋め込むと、悪意のあるコードが実行されるリスクがあります。 プリペアドステートメントでは、SQL文の構造とデータが分離されるため、安全です。

  4. Q4. fetch()とfetchAll()の違いは何ですか？

    回答を表示

    - fetch(): 1件のレコードを連想配列で取得します。
    - fetchAll(): 全レコードを配列の配列で取得します。
    IDで検索する場合など、1件だけ取得したい場合はfetch()、複数件取得したい場合はfetchAll()を使います。

第10章のまとめ

  • データベースは、大量のデータを効率的に保存・管理・検索するための仕組みです。
  • MySQLは、世界中で広く使われている無料のデータベース管理システムで、PHPとの相性が良いです。
  • PDOを使うことで、様々なデータベースに統一的な方法でアクセスできます。
  • CRUD操作（Create, Read, Update, Delete）は、データベースの基本操作です。
  • プリペアドステートメントを使うことで、SQLインジェクション攻撃を防ぎ、安全なアプリケーションを作ることができます。
