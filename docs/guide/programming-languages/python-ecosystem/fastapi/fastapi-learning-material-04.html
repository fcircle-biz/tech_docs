パラメータと検証

パスパラメータ、クエリパラメータ、リクエストボディの処理方法を学び、FastAPIの強力なバリデーション機能を活用してデータの整合性を保つ方法を習得します。

  • カスタムバリデータによる独自のデータ検証ロジックの実装

4.1 FastAPIのパラメータとは

FastAPIでは、クライアントからデータを受け取る方法として主に3種類のパラメータがあります。それぞれの用途と特徴を理解することで、適切なAPIを設計できます。

3種類のパラメータ

パスパラメータ

URL内に埋め込まれる値。リソースの識別に使用されます。
例: /users/123

クエリパラメータ

URLの?以降に付与される値。フィルタやソート条件に使用されます。
例: ?page=1&size=20

リクエストボディ

POST/PUT等で送信されるJSONデータ。複雑なデータ構造を送信できます。
例: {"name": "太郎"}

4.2 パスパラメータの基本

パスパラメータは、URL内に埋め込まれる値で、リソースを特定するために使用されます。FastAPIは型ヒントを使って自動的に型変換とバリデーションを行います。

基本的な使い方

main.py 
from fastapi import FastAPI

app = FastAPI()

# パスパラメータの基本（文字列型）
@app.get("/items/{item_id}")
def read_item(item_id: str):
    """アイテムIDを文字列で受け取る"""
    return {"item_id": item_id}

# パスパラメータ（整数型への自動変換）
@app.get("/users/{user_id}")
def read_user(user_id: int):
    """ユーザーIDを整数で受け取る"""
    return {"user_id": user_id}

自動型変換とバリデーション

user_id: int と指定すると、FastAPIは自動的に文字列を整数に変換します。変換できない場合（例: /users/abc）はエラーを返します。

Pathを使った高度なバリデーション

Path クラスを使用すると、パスパラメータに詳細な制約を設定できます。

main.py 
from fastapi import FastAPI, Path

app = FastAPI()

@app.get("/items/{item_id}")
def read_item(
    item_id: int = Path(
        ...,  # 必須パラメータ
        title="アイテムID",
        description="取得するアイテムのID",
        ge=1,  # 1以上
        le=1000  # 1000以下
    )
):
    """IDが1から1000の範囲内であることを保証"""
    return {"item_id": item_id}

主なPathパラメータ

  • ge (greater than or equal): 指定値以上
  • gt (greater than): 指定値より大きい
  • le (less than or equal): 指定値以下
  • lt (less than): 指定値より小さい
  • title: APIドキュメントに表示されるタイトル
  • description: APIドキュメントに表示される説明

4.3 クエリパラメータの活用

クエリパラメータは、検索条件、ページネーション、ソート順序など、リソースのフィルタリングや取得条件を指定するために使用されます。

基本的な使い方

main.py 
from fastapi import FastAPI

app = FastAPI()

# オプションのクエリパラメータ
@app.get("/items/")
def read_items(skip: int = 0, limit: int = 10):
    """
    スキップ件数と取得件数をクエリパラメータで指定
    例: /items/?skip=0&limit=10
    """
    return {"skip": skip, "limit": limit}

# 必須クエリパラメータ（デフォルト値なし）
@app.get("/search/")
def search_items(q: str):
    """検索キーワードは必須"""
    return {"query": q}

デフォルト値の有無で判定

デフォルト値を設定しないパラメータは必須になります。q: str のように書くと、クライアントは必ず値を指定する必要があります。

Queryを使った詳細な設定

Query クラスを使用すると、クエリパラメータに詳細な制約やメタデータを設定できます。

main.py 
from fastapi import FastAPI, Query
from typing import Optional

app = FastAPI()

@app.get("/items/")
def read_items(
    q: Optional[str] = Query(
        None,  # デフォルト値（None = オプショナル）
        min_length=3,  # 最小文字数
        max_length=50,  # 最大文字数
        regex="^[a-zA-Z0-9]+$",  # 正規表現
        title="検索キーワード",
        description="アイテムを検索するキーワード"
    ),
    limit: int = Query(10, ge=1, le=100)
):
    """クエリパラメータに詳細な制約を設定"""
    return {"q": q, "limit": limit}

複数値の受け取り

main.py 
from fastapi import FastAPI, Query
from typing import List

app = FastAPI()

@app.get("/items/")
def read_items(tags: List[str] = Query([])):
    """
    複数のタグを受け取る
    例: /items/?tags=python&tags=fastapi&tags=web
    """
    return {"tags": tags}

4.4 リクエストボディとバリデーション

POSTやPUTなどのHTTPメソッドでは、リクエストボディを使ってより複雑なデータを送信できます。PydanticモデルとFieldを組み合わせることで、強力なバリデーションを実現します。

Fieldを使った詳細なバリデーション

main.py 
from fastapi import FastAPI
from pydantic import BaseModel, Field

app = FastAPI()

class Item(BaseModel):
    name: str = Field(
        ...,  # 必須フィールド
        min_length=1,
        max_length=100,
        description="商品名"
    )
    price: float = Field(
        ...,
        gt=0,  # 0より大きい
        le=1000000,  # 100万以下
        description="価格（円）"
    )
    quantity: int = Field(1, ge=1, description="数量")

@app.post("/items/")
def create_item(item: Item):
    """バリデーション付きアイテム作成"""
    return item

4.5 カスタムバリデータの実装

標準のバリデーションでは対応できない複雑なロジックには、Pydanticのバリデータ機能を使ってカスタムバリデーションを実装できます。

main.py 
from fastapi import FastAPI
from pydantic import BaseModel, validator, Field

app = FastAPI()

class User(BaseModel):
    username: str = Field(..., min_length=3, max_length=20)
    email: str
    age: int = Field(..., ge=0, le=150)

    @validator('email')
    def email_must_be_valid(cls, v):
        """メールアドレスの形式チェック"""
        if '@' not in v:
            raise ValueError('有効なメールアドレスを入力してください')
        return v

    @validator('username')
    def username_alphanumeric(cls, v):
        """ユーザー名は英数字のみ"""
        if not v.isalnum():
            raise ValueError('英数字のみ使用できます')
        return v

@app.post("/users/")
def create_user(user: User):
    return user

バリデータの実行順序

バリデータは、まず型チェックが行われ、次にField制約、最後にカスタムバリデータの順で実行されます。いずれかのステップで失敗すると、以降の処理はスキップされます。

実習 4-1: 商品検索APIの作成

パスパラメータとクエリパラメータを組み合わせた商品検索APIを作成しましょう。

仕様

  1. 1 エンドポイント: GET /categories/{category_id}/products
  2. 2 パスパラメータ: category_idは1以上の整数
  3. 3 クエリパラメータ: min_price（最小価格、デフォルト0）、max_price（最大価格、オプション）、limit（取得件数、デフォルト20）

実装例

from fastapi import FastAPI, Path, Query
from typing import Optional

app = FastAPI()

@app.get("/categories/{category_id}/products")
def search_products(
    category_id: int = Path(..., ge=1, description="カテゴリID"),
    min_price: float = Query(0, ge=0, description="最小価格"),
    max_price: Optional[float] = Query(None, ge=0),
    limit: int = Query(20, ge=1, le=100)
):
    return {
        "category_id": category_id,
        "min_price": min_price,
        "max_price": max_price,
        "limit": limit
    }

理解度確認クイズ

  1. Q1. パスパラメータ、クエリパラメータ、リクエストボディの適切な使い分けを説明してください。

    回答を表示

    パスパラメータ: リソースの識別に使用（例: /users/123）
    クエリパラメータ: フィルタリング、ページネーション、ソート条件に使用（例: ?page=1&sort=asc）
    リクエストボディ: 複雑なデータ構造や大量のデータを送信する場合に使用（例: POSTでの新規作成）

  2. Q2. ge、gt、le、lt の意味と使い分けを説明してください。

    回答を表示

    ge (greater than or equal): 指定値以上（例: age=18なら18歳以上）
    gt (greater than): 指定値より大きい（例: price>0なら0円より高い）
    le (less than or equal): 指定値以下（例: score≤100なら100点以下）
    lt (less than): 指定値より小さい（例: temperature<100なら100度未満）

  3. Q3. Pydanticのカスタムバリデータで、バリデーションエラーを発生させるにはどうすればよいですか？

    回答を表示

    ValueError を発生させます。例: raise ValueError('エラーメッセージ')
    このエラーメッセージはクライアントに422ステータスコードと共に返されます。

  4. Q4. クエリパラメータをオプショナルにする方法を2つ挙げてください。

    回答を表示

    1. デフォルト値を設定する: q: str = "default"
    2. Optional型とNoneを使う: q: Optional[str] = None
    どちらの方法でも、クライアントがパラメータを指定しなかった場合にエラーにならずに処理が続行されます。
