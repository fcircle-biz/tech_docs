FastAPI入門とセットアップ

FastAPIの概要と特徴を理解し、開発環境を構築して最初のAPIエンドポイントを作成します。自動生成されるAPIドキュメントの活用方法についても学習します。

  • Python仮想環境の作成とFastAPIのインストール方法
  • 最初のAPIエンドポイントの作成と動作確認
  • Swagger UIとReDocの使い方と活用方法

1.1 FastAPIとは

FastAPIは、Pythonで高速かつ現代的なWeb APIを構築するためのフレームワークです。2018年に登場し、型ヒント（Type Hints）を活用した開発体験と高いパフォーマンスで急速に人気を集めています。

FastAPIの名前の由来

"Fast" は高速なパフォーマンスと開発速度の両方を意味しています。Node.jsやGoに匹敵する実行速度を持ちながら、開発者の生産性も高いことが特徴です。

1.2 FastAPIの主な特徴

高速なパフォーマンス

Starlette（ASGI）とPydanticをベースに構築され、Node.jsやGoに匹敵する実行速度を実現します。非同期処理によりI/O待機時間を効率化します。

型ヒントベースの開発

Python 3.6+の型ヒントを活用し、エディタの補完機能が充実。型チェックによりバグを早期に発見でき、コードの可読性も向上します。

自動APIドキュメント生成

コードから自動的にSwagger UIとReDocが生成されます。追加の設定不要で、常に最新のAPI仕様書が利用できます。

自動データ検証

Pydanticによるデータ検証が自動実行されます。不正なデータは明確なエラーメッセージと共に拒否され、セキュリティも向上します。

1.3 開発環境のセットアップ

前提条件の確認

FastAPIを使用するには、Python 3.8以上が必要です。まず、インストールされているPythonのバージョンを確認しましょう。

ターミナル 
# Pythonバージョンの確認
python --version
# または
python3 --version

# 出力例: Python 3.11.5

仮想環境の作成

プロジェクト専用の仮想環境を作成することで、他のプロジェクトと依存関係を分離できます。

ターミナル 
# プロジェクトディレクトリの作成と移動
mkdir fastapi-tutorial
cd fastapi-tutorial

# 仮想環境の作成
python -m venv venv

# 仮想環境の有効化
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

仮想環境が有効化されたら

ターミナルのプロンプトに (venv) という表示が追加されます。これが仮想環境がアクティブな状態の目印です。

FastAPIのインストール

FastAPIと、開発用サーバーのUvicornをインストールします。

ターミナル 
# FastAPIとUvicornのインストール
pip install fastapi uvicorn[standard]

# インストール確認
pip list | grep fastapi

Uvicornとは

Uvicornは、ASGIアプリケーション用の高速なWebサーバーです。FastAPIはASGI（Asynchronous Server Gateway Interface）をサポートしており、Uvicornを通じて非同期処理の恩恵を受けられます。

1.4 最初のAPIエンドポイントの作成

それでは、最もシンプルなFastAPIアプリケーションを作成してみましょう。

main.py 
# FastAPIライブラリをインポート
from fastapi import FastAPI

# FastAPIアプリケーションのインスタンスを作成
app = FastAPI()

# ルートパス("/")へのGETリクエストを処理するエンドポイント
@app.get("/")
def read_root():
    """
    ルートエンドポイント
    シンプルなJSON形式のメッセージを返す
    """
    return {"message": "Hello World"}

# "/items/{item_id}"へのGETリクエストを処理するエンドポイント
@app.get("/items/{item_id}")
def read_item(item_id: int, q: str = None):
    """
    アイテム取得エンドポイント

    パラメータ:
        item_id: アイテムID（パスパラメータ）
        q: 検索クエリ（クエリパラメータ、オプション）
    """
    return {"item_id": item_id, "q": q}

コードの解説

1. FastAPIインスタンスの作成

app = FastAPI() でアプリケーションのインスタンスを作成します。このオブジェクトがすべてのAPIエンドポイントを管理します。

2. デコレータによるルーティング

@app.get("/") というデコレータで、HTTPのGETメソッドに対応するエンドポイントを定義します。POSTやPUTなど他のHTTPメソッドも同様に定義できます。

3. 型ヒントの活用

関数の引数に型を指定することで、FastAPIが自動的にデータ検証とドキュメント生成を行います。item_id: int は整数型を期待することを示しています。

アプリケーションの実行

ターミナル 
# Uvicornでアプリケーションを起動
uvicorn main:app --reload

# 出力例:
# INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
# INFO:     Started reloader process
# INFO:     Started server process
# INFO:     Waiting for application startup.
# INFO:     Application startup complete.

--reloadオプション

--reload オプションを付けると、コードを変更したときに自動的にサーバーが再起動されます。開発時に便利ですが、本番環境では使用しないでください。

動作確認

ブラウザで以下のURLにアクセスして、APIの動作を確認しましょう。

ルートエンドポイント:

http://127.0.0.1:8000/

期待される結果: {"message": "Hello World"}

アイテムエンドポイント:

http://127.0.0.1:8000/items/5?q=test

期待される結果: {"item_id": 5, "q": "test"}

1.5 自動ドキュメントの活用

FastAPIの最大の特徴の一つが、自動生成されるインタラクティブなAPIドキュメントです。追加の設定なしで、2種類のドキュメントが利用できます。

Swagger UI（/docs）

http://127.0.0.1:8000/docs にアクセスすると、Swagger UIベースのインタラクティブなドキュメントが表示されます。

APIのテスト機能

"Try it out" ボタンで、ブラウザから直接APIをテストできます。パラメータを入力して実行すると、レスポンスが即座に表示されます。

リクエスト例の自動生成

各エンドポイントのリクエスト形式が自動的にドキュメント化され、curlコマンドやHTTPリクエストの例も確認できます。

ReDoc（/redoc）

http://127.0.0.1:8000/redoc にアクセスすると、ReDocベースの読みやすいドキュメントが表示されます。

ReDocの特徴

  • 検索機能による素早いエンドポイントの発見
  • 階層的で見やすいレイアウト
  • 印刷やPDF出力に適した形式
  • 外部チームへの共有に最適

実習 1-1: 挨拶APIの作成

名前を受け取って挨拶メッセージを返すエンドポイントを作成しましょう。

手順

  1. 1 main.pyに新しいエンドポイント /greet/{name} を追加します
  2. 2 パスパラメータで名前を受け取り、挨拶メッセージを返す関数を実装します
  3. 3 ブラウザと自動ドキュメントの両方で動作を確認します

実装例

@app.get("/greet/{name}")
def greet(name: str):
    """
    名前を受け取って挨拶を返すエンドポイント
    """
    return {"message": f"こんにちは、{name}さん！"}

期待される結果

http://127.0.0.1:8000/greet/太郎 にアクセスすると: 

{"message": "こんにちは、太郎さん！"}

理解度確認クイズ

  1. Q1. FastAPIが他のPythonフレームワークと比較して優れている点を3つ挙げてください。

    回答を表示

    1. 高速なパフォーマンス: 非同期処理により、Node.jsやGoに匹敵する速度を実現
    2. 自動ドキュメント生成: OpenAPIに準拠したSwagger UIとReDocが自動生成される
    3. 型ヒントベースの開発: Python 3.6+の型ヒントを活用し、自動検証とエディタサポートが充実

  2. Q2. uvicorn main:app --reload コマンドの --reload オプションの役割は何ですか？

    回答を表示

    コードに変更があったときに自動的にサーバーを再起動するオプションです。開発中は便利ですが、本番環境では使用すべきではありません。本番環境では安定性が優先されるため、意図しない再起動を避ける必要があります。

  3. Q3. FastAPIで利用できる2種類の自動生成ドキュメントのURLとそれぞれの特徴を説明してください。

    回答を表示

    1. Swagger UI（/docs）: インタラクティブなテスト機能があり、ブラウザ上でAPIを実際に試すことができます。開発時のデバッグに最適です。
    2. ReDoc（/redoc）: 読みやすいレイアウトで、検索機能や印刷に適した形式です。外部チームへのAPI仕様の共有に最適です。

  4. Q4. 型ヒント（Type Hints）を使用することで、FastAPIが自動的に提供してくれる機能は何ですか？

    回答を表示

    1. データ検証: 型に合わないデータが送られてきた場合、自動的にエラーを返します
    2. 自動ドキュメント生成: 型情報がAPIドキュメントに反映されます
    3. エディタサポート: IDEやエディタでの補完機能が充実し、開発効率が向上します

第1章のまとめ

FastAPIは高速、型安全、自動ドキュメント生成が特徴のモダンなPython Webフレームワークです

開発環境は仮想環境を作成し、pipでFastAPIとUvicornをインストールすることで構築できます

デコレータと型ヒントを使うことで、シンプルにAPIエンドポイントを定義できます

Swagger UI（/docs）とReDoc（/redoc）により、コードから自動的にAPIドキュメントが生成されます
