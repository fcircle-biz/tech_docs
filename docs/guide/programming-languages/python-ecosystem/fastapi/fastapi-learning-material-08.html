非同期処理とバックグラウンドタスク

async/awaitの活用、非同期データベース操作、バックグラウンドタスクの実装、WebSocketの基本、並行処理のパフォーマンス最適化を学習します。

  • 並行処理によるAPIパフォーマンスの最適化手法

8.1 非同期処理の基礎

非同期処理（Asynchronous Processing）は、I/O待機時間を効率化し、同時に複数のリクエストを処理できるようにする仕組みです。FastAPIはPythonのasync/await構文を活用して、高パフォーマンスなAPIを実現します。

同期処理 vs 非同期処理の比較

sequenceDiagram participant Client as クライアント participant API as APIサーバー participant DB as データベース participant Ext as 外部API Note over Client,Ext: 同期処理（順次実行） Client->>API: リクエスト1 API->>DB: クエリ実行 Note right of DB: 待機中... DB-->>API: 結果 API->>Ext: 外部API呼び出し Note right of Ext: 待機中... Ext-->>API: 結果 API-->>Client: レスポンス1 Note over Client,Ext: 非同期処理（並行実行） Client->>API: リクエスト1 API->>DB: クエリ実行 API->>Ext: 外部API呼び出し Note right of API: 他の処理を実行可能 DB-->>API: 結果 Ext-->>API: 結果 API-->>Client: レスポンス1

非同期処理のメリット

データベースクエリや外部APIの呼び出しなど、I/O待機時間がある処理を非同期化することで、その待機時間に別のリクエストを処理できます。これにより、同じリソースでより多くのリクエストを処理可能になります。

async/await構文の基本

Pythonの非同期処理は、asyncとawaitキーワードで制御します。

非同期関数の定義と使用 
from fastapi import FastAPI
import asyncio
import time

app = FastAPI()

# 同期処理の例
def sync_slow_operation():
    """同期的に2秒待機"""
    time.sleep(2)
    return {"result": "完了（同期）"}

# 非同期処理の例
async def async_slow_operation():
    """非同期的に2秒待機"""
    await asyncio.sleep(2)
    return {"result": "完了（非同期）"}

# 非同期エンドポイント
@app.get("/async")
async def async_endpoint():
    """非同期処理を使ったエンドポイント"""
    result = await async_slow_operation()
    return result

async キーワード

関数定義の前に async を付けると、その関数は非同期関数（コルーチン）になります。非同期関数内では await を使えます。

await キーワード

await は非同期関数の完了を待ちます。待機中、他の処理を実行できるため、ブロッキングが発生しません。

8.2 非同期データベース操作

データベース操作を非同期化することで、クエリ実行中に他のリクエストを処理できます。encode/databasesやSQLAlchemyの非同期エンジンを使用します。

encode/databasesの導入

ターミナル 
# 非同期データベースライブラリのインストール
pip install databases[asyncpg] asyncpg
database.py 
from databases import Database

# データベース接続URL
DATABASE_URL = "postgresql://user:password@localhost/dbname"

# データベースインスタンスの作成
database = Database(DATABASE_URL)

# 起動時にデータベースに接続
async def connect_db():
    """データベースに接続"""
    await database.connect()
    print("データベースに接続しました")

# 終了時にデータベースを切断
async def disconnect_db():
    """データベースから切断"""
    await database.disconnect()
    print("データベースから切断しました")
main.py 
from fastapi import FastAPI
from database import database, connect_db, disconnect_db

app = FastAPI()

# アプリケーション起動時にDB接続
@app.on_event("startup")
async def startup():
    await connect_db()

# アプリケーション終了時にDB切断
@app.on_event("shutdown")
async def shutdown():
    await disconnect_db()

# 非同期データベースクエリ
@app.get("/users/{user_id}")
async def get_user(user_id: int):
    """ユーザー情報を非同期で取得"""
    query = "SELECT * FROM users WHERE id = :user_id"
    user = await database.fetch_one(query=query, values={"user_id": user_id})
    return user

データベース操作メソッド

  • fetch_one(): 単一レコードを取得
  • fetch_all(): 複数レコードを取得
  • execute(): INSERT/UPDATE/DELETE実行

8.3 バックグラウンドタスク

レスポンス返却後に実行したい処理（メール送信、ログ記録、データ加工など）は、BackgroundTasksを使ってバックグラウンドで実行できます。

バックグラウンドタスクの実行フロー

sequenceDiagram participant Client as クライアント participant API as APIエンドポイント participant BG as バックグラウンド
タスク participant Mail as メールサーバー Client->>API: ユーザー登録リクエスト API->>API: ユーザーをDBに保存 API->>BG: メール送信タスクを登録 API-->>Client: レスポンス返却
（即座） Note over Client: ユーザーは
待たされない BG->>Mail: ウェルカムメール送信 Mail-->>BG: 送信完了

BackgroundTasksの使い方

main.py 
from fastapi import FastAPI, BackgroundTasks
import time

app = FastAPI()

# バックグラウンドで実行する関数
def send_email(email: str, message: str):
    """
    メール送信処理（重い処理のシミュレーション）
    """
    print(f"メール送信開始: {email}")
    time.sleep(5)  # メール送信に5秒かかると仮定
    print(f"メール送信完了: {email} - {message}")

# バックグラウンドタスクを使用するエンドポイント
@app.post("/send-notification/")
async def send_notification(
    email: str,
    background_tasks: BackgroundTasks
):
    """
    通知を送信するエンドポイント
    レスポンスはすぐに返し、メール送信は後で実行
    """
    # バックグラウンドタスクに追加
    background_tasks.add_task(send_email, email, "登録ありがとうございます！")

    # すぐにレスポンスを返す
    return {"message": "通知を送信します"}

BackgroundTasksの利点

  • • レスポンスタイムの短縮: ユーザーは重い処理の完了を待たなくて良い
  • • シンプルな実装: 複雑なキューシステムが不要
  • • 軽量な処理に最適: メール送信、ログ記録などに向いている

複数のバックグラウンドタスク

複数タスクの実行例 
def write_log(message: str):
    """ログファイルに書き込み"""
    with open("log.txt", "a") as f:
        f.write(f"{message}\n")

def update_statistics():
    """統計情報を更新"""
    print("統計情報を更新しています...")
    time.sleep(2)
    print("統計情報の更新が完了しました")

@app.post("/register/")
async def register_user(
    username: str,
    email: str,
    background_tasks: BackgroundTasks
):
    """複数のバックグラウンドタスクを実行"""
    # タスク1: メール送信
    background_tasks.add_task(send_email, email, "登録完了")
    # タスク2: ログ記録
    background_tasks.add_task(write_log, f"新規登録: {username}")
    # タスク3: 統計更新
    background_tasks.add_task(update_statistics)

    return {"message": f"{username}が登録されました"}

8.4 WebSocketの基本

WebSocketは、サーバーとクライアント間で双方向のリアルタイム通信を実現するプロトコルです。チャットアプリケーション、リアルタイム通知、ライブダッシュボードなどに使用されます。

HTTP通信とWebSocket通信の違い

sequenceDiagram participant Client as クライアント participant Server as サーバー Note over Client,Server: HTTP通信（リクエスト/レスポンス） Client->>Server: リクエスト1 Server-->>Client: レスポンス1 Client->>Server: リクエスト2 Server-->>Client: レスポンス2 Note over Client: 毎回接続が必要 Note over Client,Server: WebSocket通信（双方向・永続接続） Client->>Server: 接続確立 Server-->>Client: 接続完了 Server->>Client: メッセージ送信 Client->>Server: メッセージ送信 Server->>Client: メッセージ送信 Note over Client,Server: 接続は維持される

シンプルなWebSocketエンドポイント

main.py 
from fastapi import FastAPI, WebSocket, WebSocketDisconnect

app = FastAPI()

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    """
    WebSocketエンドポイント
    クライアントからのメッセージをエコーバック
    """
    # 接続を受け入れる
    await websocket.accept()

    try:
        while True:
            # クライアントからのメッセージを待機
            data = await websocket.receive_text()
            # メッセージを送り返す
            await websocket.send_text(f"受信: {data}")
    except WebSocketDisconnect:
        print("クライアントが切断しました")

チャットルームの実装例

chat.py 
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from typing import List

app = FastAPI()

# 接続中のクライアントを管理
class ConnectionManager:
    def __init__(self):
        # アクティブな接続のリスト
        self.active_connections: List[WebSocket] = []

    async def connect(self, websocket: WebSocket):
        """新しい接続を追加"""
        await websocket.accept()
        self.active_connections.append(websocket)

    def disconnect(self, websocket: WebSocket):
        """接続を削除"""
        self.active_connections.remove(websocket)

    async def broadcast(self, message: str):
        """全クライアントにメッセージを送信"""
        for connection in self.active_connections:
            await connection.send_text(message)

manager = ConnectionManager()

@app.websocket("/ws/chat")
async def chat_endpoint(websocket: WebSocket):
    """チャットルームのWebSocketエンドポイント"""
    await manager.connect(websocket)

    try:
        while True:
            # メッセージを受信
            data = await websocket.receive_text()
            # 全クライアントにブロードキャスト
            await manager.broadcast(f"メッセージ: {data}")
    except WebSocketDisconnect:
        manager.disconnect(websocket)
        await manager.broadcast("ユーザーが退出しました")

8.5 パフォーマンス最適化のベストプラクティス

非同期処理を効果的に活用し、APIのパフォーマンスを最大化するためのベストプラクティスを紹介します。

I/O処理の非同期化

データベースクエリ、外部API呼び出し、ファイル読み書きなど、待機時間が発生する処理は必ず非同期化しましょう。

コネクションプール

データベース接続は再利用可能なプールで管理し、接続の確立・切断のオーバーヘッドを削減します。

並行処理の活用

複数の独立した非同期処理はasyncio.gather()で並行実行し、合計待機時間を短縮します。

重い処理の分離

画像処理やデータ集計などCPU集約的な処理は、BackgroundTasksやCeleryなどで分離し、APIの応答性を保ちます。

並行処理の実装例

並行実行でパフォーマンス向上 
import asyncio
from fastapi import FastAPI

app = FastAPI()

async def fetch_user_data(user_id: int):
    """ユーザーデータを取得（仮想）"""
    await asyncio.sleep(1)  # DB問い合わせをシミュレート
    return {"user_id": user_id, "name": "田中太郎"}

async def fetch_order_history(user_id: int):
    """注文履歴を取得（仮想）"""
    await asyncio.sleep(1)  # DB問い合わせをシミュレート
    return [{"order_id": 1, "total": 3000}]

async def fetch_recommendations(user_id: int):
    """おすすめ商品を取得（仮想）"""
    await asyncio.sleep(1)  # 外部API呼び出しをシミュレート
    return [{"product_id": 101, "name": "商品A"}]

# 逐次実行版（遅い）
@app.get("/dashboard/sequential/{user_id}")
async def dashboard_sequential(user_id: int):
    """逐次実行: 合計3秒かかる"""
    user = await fetch_user_data(user_id)
    orders = await fetch_order_history(user_id)
    recommendations = await fetch_recommendations(user_id)

    return {
        "user": user,
        "orders": orders,
        "recommendations": recommendations
    }

# 並行実行版（速い）
@app.get("/dashboard/concurrent/{user_id}")
async def dashboard_concurrent(user_id: int):
    """並行実行: 約1秒で完了"""
    # 3つの処理を同時に実行
    user, orders, recommendations = await asyncio.gather(
        fetch_user_data(user_id),
        fetch_order_history(user_id),
        fetch_recommendations(user_id)
    )

    return {
        "user": user,
        "orders": orders,
        "recommendations": recommendations
    }

パフォーマンスの比較

逐次実行では1秒 + 1秒 + 1秒 = 3秒かかりますが、並行実行では3つの処理が同時に実行されるため、最も遅い処理の1秒で完了します。これにより、レスポンスタイムが大幅に短縮されます。

実習 8-1: バックグラウンドタスクの実装

ユーザー登録後にウェルカムメールを送信するエンドポイントを作成し、バックグラウンドタスクで処理しましょう。

手順

  1. 1 メール送信をシミュレートする関数を作成します（5秒の待機処理）
  2. 2 /registerエンドポイントを作成し、BackgroundTasksを使用します
  3. 3 レスポンスがすぐに返ることを確認し、ターミナルでメール送信ログを確認します

期待される動作

  • • エンドポイントにリクエストを送ると、即座にレスポンスが返る
  • • 5秒後にターミナルに「メール送信完了」のログが出力される
  • • ユーザーは5秒待たされない

動作確認 8-1: WebSocketのテスト

簡単なWebSocketエコーサーバーを作成し、ブラウザのコンソールからテストしてみましょう。

テスト手順

  1. 1 FastAPIアプリを起動します
  2. 2 ブラウザの開発者ツールを開き、コンソールに以下のコードを入力します 
// WebSocket接続を確立
const ws = new WebSocket("ws://localhost:8000/ws");

// 接続確立時
ws.onopen = () => {
    console.log("接続しました");
    ws.send("こんにちは！");
};

// メッセージ受信時
ws.onmessage = (event) => {
    console.log("受信:", event.data);
};

コンソールに「受信: こんにちは！」と表示されれば成功です。

理解度確認クイズ

  1. Q1. 非同期処理のメリットを3つ挙げ、それぞれ説明してください。

    回答を表示

    1. スループットの向上: I/O待機中に他のリクエストを処理できるため、同時処理数が増加します
    2. リソース効率の改善: ブロッキングが減り、少ないスレッド/プロセスで多くのリクエストを処理できます
    3. レスポンスタイムの短縮: 並行処理により、複数の独立した処理を同時実行できます

  2. Q2. BackgroundTasksを使用すべき処理と、使用すべきでない処理を説明してください。

    回答を表示

    使用すべき処理: メール送信、ログ記録、簡単な統計更新など、軽量で失敗しても再試行が容易な処理
    使用すべきでない処理: 重いデータ処理、長時間かかる処理、確実な実行が必要な処理。これらはCeleryなどの専用タスクキューを使用すべきです

  3. Q3. WebSocketとHTTPの違いを説明し、WebSocketが適している用途を挙げてください。

    回答を表示

    違い: HTTPはリクエスト/レスポンス型で毎回接続を確立しますが、WebSocketは双方向の永続接続を維持し、サーバーからもクライアントへ自由に送信できます
    適した用途: チャットアプリ、リアルタイム通知、ライブダッシュボード、オンラインゲーム、共同編集ツールなど

  4. Q4. asyncio.gather()を使用する利点と、使用例を説明してください。

    回答を表示

    利点: 複数の非同期処理を並行実行し、すべての処理が完了するのを待ちます。逐次実行と比べて合計待機時間を大幅に短縮できます
    使用例: ユーザーデータ、注文履歴、おすすめ商品を3つのデータベースから取得する場合、gather()を使えば3秒→1秒に短縮できます

トラブルシューティング

非同期関数内で同期処理を使ってしまう

time.sleep()や同期ライブラリをasync関数内で使うと、非同期のメリットが失われます。

解決策:

非同期版を使用: asyncio.sleep()、httpx.AsyncClientなど

WebSocketで接続が切れる

タイムアウトやネットワークエラーで接続が切断されることがあります。

解決策:

WebSocketDisconnect例外をキャッチし、適切に処理します。定期的なping/pongで接続を維持することも有効です。

BackgroundTasksが実行されない

エンドポイントがエラーを返すと、BackgroundTasksは実行されません。

解決策:

レスポンスが正常に返されることを確認します。重要な処理はCeleryなどの専用キューを使用することを検討してください。

第8章のまとめ

async/awaitを使った非同期処理により、I/O待機時間を有効活用し、高いスループットを実現できます

encode/databasesやSQLAlchemyの非同期エンジンで、データベース操作を非同期化できます

BackgroundTasksを使用すると、レスポンス返却後に処理を実行し、ユーザー体験を向上できます

WebSocketにより、サーバーとクライアント間でリアルタイム双方向通信が可能になります

asyncio.gather()による並行処理で、複数の非同期処理を同時実行し、レスポンスタイムを短縮できます
