データベースの統合（SQLAlchemy）

SQLAlchemyを使用したデータベース統合について学習します。ORMモデルの定義、CRUDオペレーション、トランザクション管理、マイグレーション方法を実践的に習得します。

この章で学ぶこと

  • SQLAlchemyの基本概念とFastAPIとの統合方法
  • データベース接続の設定とセッション管理
  • ORMモデルの定義とテーブルマッピング
  • CRUDオペレーション（作成・読取・更新・削除）の実装
  • Alembicによるデータベースマイグレーション

5.1 SQLAlchemyとは

SQLAlchemyは、Pythonで最も広く使われているORM（Object-Relational Mapping）ライブラリです。データベースのテーブルをPythonのクラスとして扱い、SQLを直接書かずにデータベース操作ができます。

SQLAlchemyの役割

flowchart LR A[FastAPI アプリ] --> B[SQLAlchemy
ORM] B --> C{データベース} C --> D[(SQLite)] C --> E[(PostgreSQL)] C --> F[(MySQL)] style B fill:#dbeafe,stroke:#3b82f6,stroke-width:3px

ORM機能

データベースのテーブルをPythonクラスとしてマッピング。SQLを書かずにPythonコードでデータベース操作が可能です。

データベース非依存

SQLite、PostgreSQL、MySQLなど複数のデータベースに対応。接続文字列を変えるだけで切り替え可能です。

セッション管理

データベース接続とトランザクションを効率的に管理。接続プールによりパフォーマンスを最適化します。

マイグレーション

Alembicと連携してデータベーススキーマの変更履歴を管理。バージョン管理された安全なスキーマ変更が可能です。

ORMのメリット

ORMを使用することで、SQLインジェクション対策やデータ型の自動変換などのセキュリティと安全性が向上します。また、コードの可読性と保守性も大幅に改善されます。

5.2 データベース接続の設定

必要なパッケージのインストール

SQLAlchemyと、今回はSQLiteを使用するため必要なパッケージをインストールします。

ターミナル 
# SQLAlchemyのインストール
pip install sqlalchemy

# PostgreSQL使用の場合
pip install psycopg2-binary

データベース設定ファイルの作成

データベース接続を管理する database.py ファイルを作成します。

database.py 
# SQLAlchemyのコアモジュールをインポート
from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker

# データベース接続URL（SQLiteの例）
SQLALCHEMY_DATABASE_URL = "sqlite:///./test.db"

# データベースエンジンの作成
engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    connect_args={"check_same_thread": False}  # SQLiteのみ必要
)

# セッションローカルクラスの作成
SessionLocal = sessionmaker(
    autocommit=False,
    autoflush=False,
    bind=engine
)

# ORMモデルのベースクラス
Base = declarative_base()

PostgreSQLの接続URL

PostgreSQLを使用する場合の接続URL:
postgresql://user:password@localhost/dbname

データベースセッションの依存性注入

FastAPIのDependency Injectionを使用して、エンドポイントでデータベースセッションを利用できるようにします。

database.py（続き） 
# データベースセッション取得用の依存性関数
def get_db():
    """
    データベースセッションを提供し、
    リクエスト終了時に自動的にクローズする
    """
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

5.3 ORMモデルの定義

データベースのテーブル構造を表すORMモデルを定義します。ここではユーザー情報を管理するシンプルなモデルを作成します。

models.py 
# SQLAlchemyのカラム型とデータベース設定をインポート
from sqlalchemy import Boolean, Column, Integer, String
from database import Base

# Userテーブルに対応するORMモデル
class User(Base):
    __tablename__ = "users"  # テーブル名を指定

    # カラム定義
    id = Column(Integer, primary_key=True, index=True)
    email = Column(String, unique=True, index=True)
    username = Column(String, index=True)
    hashed_password = Column(String)
    is_active = Column(Boolean, default=True)

Column型の種類

  • Integer: 整数型
  • String: 文字列型
  • Boolean: 真偽値型
  • DateTime: 日時型
  • Float: 浮動小数点型

カラムオプション

  • primary_key=True: 主キーとして設定
  • unique=True: 一意制約を追加
  • index=True: インデックスを作成（検索高速化）
  • default=value: デフォルト値を設定

Pydanticスキーマの定義

APIのリクエスト/レスポンス用のPydanticモデルを定義します。

schemas.py 
from pydantic import BaseModel, EmailStr

# ユーザー作成時のリクエストモデル
class UserCreate(BaseModel):
    email: EmailStr
    username: str
    password: str

# ユーザー情報のレスポンスモデル
class UserResponse(BaseModel):
    id: int
    email: str
    username: str
    is_active: bool

    class Config:
        from_attributes = True  # ORMモードを有効化

ORMモードとは

from_attributes = True を設定することで、SQLAlchemyのORMオブジェクトを直接Pydanticモデルに変換できます。

テーブルの作成

定義したモデルからデータベーステーブルを作成します。

main.py 
from fastapi import FastAPI
from database import engine, Base
import models

# テーブルを作成（すでに存在する場合はスキップ）
Base.metadata.create_all(bind=engine)

app = FastAPI()

5.4 CRUDオペレーション

CRUDとは、Create（作成）、Read（読取）、Update（更新）、Delete（削除）の頭文字です。これらの基本的なデータベース操作を実装します。

CRUD操作の流れ

flowchart TD A[クライアント] --> B{HTTPメソッド} B -->|POST| C[Create
新規作成] B -->|GET| D[Read
読取] B -->|PUT/PATCH| E[Update
更新] B -->|DELETE| F[Delete
削除] C --> G[(データベース)] D --> G E --> G F --> G style G fill:#dbeafe,stroke:#3b82f6,stroke-width:2px

Create（作成）

main.py 
from fastapi import Depends
from sqlalchemy.orm import Session
from database import get_db
import models, schemas

@app.post("/users/", response_model=schemas.UserResponse)
def create_user(user: schemas.UserCreate, db: Session = Depends(get_db)):
    """新規ユーザーを作成"""
    # ORMモデルのインスタンスを作成
    db_user = models.User(
        email=user.email,
        username=user.username,
        hashed_password=user.password  # 実際はハッシュ化が必要
    )
    db.add(db_user)  # セッションに追加
    db.commit()      # データベースにコミット
    db.refresh(db_user)  # オブジェクトをリフレッシュ
    return db_user

Read（読取）

main.py 
@app.get("/users/{user_id}", response_model=schemas.UserResponse)
def get_user(user_id: int, db: Session = Depends(get_db)):
    """IDでユーザーを取得"""
    user = db.query(models.User).filter(models.User.id == user_id).first()
    if user is None:
        raise HTTPException(status_code=404, detail="User not found")
    return user

@app.get("/users/", response_model=list[schemas.UserResponse])
def get_users(skip: int = 0, limit: int = 10, db: Session = Depends(get_db)):
    """ユーザーのリストを取得"""
    users = db.query(models.User).offset(skip).limit(limit).all()
    return users

Update（更新）

main.py 
@app.put("/users/{user_id}", response_model=schemas.UserResponse)
def update_user(user_id: int, user: schemas.UserCreate,
                db: Session = Depends(get_db)):
    """ユーザー情報を更新"""
    db_user = db.query(models.User).filter(models.User.id == user_id).first()
    if db_user is None:
        raise HTTPException(status_code=404, detail="User not found")

    # 属性を更新
    db_user.username = user.username
    db_user.email = user.email
    db.commit()
    db.refresh(db_user)
    return db_user

Delete（削除）

main.py 
@app.delete("/users/{user_id}")
def delete_user(user_id: int, db: Session = Depends(get_db)):
    """ユーザーを削除"""
    db_user = db.query(models.User).filter(models.User.id == user_id).first()
    if db_user is None:
        raise HTTPException(status_code=404, detail="User not found")

    db.delete(db_user)
    db.commit()
    return {"message": "User deleted successfully"}

クエリメソッド

query(): クエリを開始
filter(): 条件でフィルタリング
first(): 最初の1件を取得
all(): すべての結果を取得

5.5 トランザクション管理

トランザクションは、複数のデータベース操作を1つの単位としてまとめて実行する仕組みです。すべての操作が成功するか、すべて失敗するかのどちらかになります。

main.py 
@app.post("/users/bulk/")
def create_multiple_users(users: list[schemas.UserCreate],
                         db: Session = Depends(get_db)):
    """複数のユーザーを一度に作成（トランザクション）"""
    try:
        # 複数のユーザーを追加
        for user_data in users:
            db_user = models.User(
                email=user_data.email,
                username=user_data.username,
                hashed_password=user_data.password
            )
            db.add(db_user)

        db.commit()  # すべて成功したらコミット
        return {"message": f"{len(users)} users created"}
    except Exception as e:
        db.rollback()  # エラーが発生したらロールバック
        raise HTTPException(status_code=400, detail=str(e))

トランザクションの重要性

複数の操作を行う場合、途中でエラーが発生するとデータの整合性が失われる可能性があります。トランザクションを使用することで、すべての操作が成功するか、何も変更されないかを保証できます。

5.6 データベースマイグレーション（Alembic）

Alembicは、SQLAlchemyのマイグレーションツールです。データベーススキーマの変更履歴を管理し、バージョン管理された方法でスキーマを更新できます。

Alembicのインストールと初期化

ターミナル 
# Alembicのインストール
pip install alembic

# Alembicの初期化
alembic init alembic

マイグレーションファイルの作成

ターミナル 
# 新しいマイグレーションファイルを自動生成
alembic revision --autogenerate -m "Create users table"

# マイグレーションを適用
alembic upgrade head

# マイグレーションを1つ戻す
alembic downgrade -1

マイグレーションのメリット

  • スキーマ変更の履歴を管理できる
  • チーム開発でスキーマ変更を共有できる
  • 本番環境への安全なデプロイが可能
  • 変更を元に戻すロールバック機能

実習 5-1: 商品管理APIの作成

商品情報を管理するCRUD APIを実装しましょう。

手順

  2. 2 schemas.pyにProductCreate, ProductResponseスキーマを定義
  3. 3 main.pyに商品のCRUDエンドポイントを実装
  4. 4 Swagger UIで動作確認

必要なエンドポイント

  • POST /products/ 新規商品登録
  • GET /products/ 商品一覧取得

  • PUT /products/{id} 商品情報更新
  • DELETE /products/{id} 商品削除

理解度確認クイズ

  1. Q1. ORMの主なメリットを3つ挙げてください。

    回答を表示

    1. SQLを直接書かずに済む: Pythonのコードでデータベース操作が可能
    2. データベース非依存: 接続文字列を変えるだけで異なるDBに切り替え可能
    3. セキュリティ向上: SQLインジェクション対策が自動的に行われる

  2. Q2. db.commit() と db.rollback() の違いは何ですか？

    回答を表示

    commit(): セッション内の変更をデータベースに確定させます。
    rollback(): セッション内の変更を破棄し、前回のコミット時点に戻します。エラーが発生した際にデータの整合性を保つために使用します。

  3. Q3. Pydanticモデルで from_attributes = True を設定する理由は何ですか？

    回答を表示

    SQLAlchemyのORMオブジェクト（属性アクセス形式）を直接Pydanticモデルに変換できるようにするためです。これにより、データベースから取得したオブジェクトをレスポンスモデルとして返す際の変換が簡単になります。

  4. Q4. Alembicを使用するメリットは何ですか？

    回答を表示

    データベーススキーマの変更履歴を管理でき、チーム開発でスキーマ変更を共有しやすくなります。また、変更を段階的に適用したり、問題があれば元に戻したり（ロールバック）することができます。本番環境への安全なデプロイも可能になります。

SQLAlchemyを使用することで、Pythonコードでデータベースをオブジェクトとして扱えます

ORMモデルとPydanticスキーマを分離することで、データベース層とAPI層を明確に区別できます

CRUDオペレーションを実装することで、基本的なデータベース操作を網羅できます

トランザクション管理により、データの整合性を保証できます

Alembicを使用することで、データベーススキーマのバージョン管理が可能になります
