Djangoとは - Webフレームワークの基礎理解

DjangoとWebフレームワークの概念を理解し、MVTアーキテクチャの基本と開発環境のセットアップ方法を学びます。

この章で学ぶこと

  • Djangoとは何か、Webフレームワークの役割と必要性
  • MVT（Model-View-Template）アーキテクチャの基本概念
  • Djangoの特徴と利点、他のフレームワークとの違い
  • 開発環境のセットアップと動作確認の方法

1.1 Djangoとは

Django（ジャンゴ）は、Pythonで書かれた高機能なWebアプリケーションフレームワークです。2005年にリリースされ、現在では世界中の多くの企業や開発者に利用されています。InstagramやSpotifyなどの大規模サービスでも採用されており、信頼性と実績のあるフレームワークです。

Djangoは「The web framework for perfectionists with deadlines（締め切りのある完璧主義者のためのWebフレームワーク）」というキャッチフレーズのもと、開発効率と品質の両立を目指して設計されています。豊富な機能が標準で提供されているため、開発者は重複したコードを書く必要がなく、ビジネスロジックの実装に集中できます。

Webフレームワークとは？

Webフレームワークは、Webアプリケーションの開発を効率化するための「土台」や「枠組み」のことです。HTTPリクエストの処理、データベース操作、セキュリティ対策など、Webアプリケーション開発で共通して必要な機能があらかじめ用意されており、開発者はこれらの基盤を活用してアプリケーション固有の機能に集中できます。

1.2 Djangoの主な特徴

Djangoには、Web開発を効率化するための多くの特徴があります。ここでは、特に重要な5つの特徴を紹介します。

Batteries Included

認証、管理画面、フォーム処理、ORMなど、Web開発に必要な機能がすべて標準搭載されています。

高いセキュリティ

SQLインジェクション、XSS、CSRF対策など、主要なセキュリティ脅威への対策が組み込まれています。

スケーラブル

小規模から大規模まで対応可能。Instagram、Spotifyなど月間数億ユーザーのサービスで採用実績があります。

DRY原則

Don't Repeat Yourself（同じことを繰り返さない）の哲学で、重複コードを最小限に抑え、保守性を高めています。

1.3 MVTアーキテクチャの理解

DjangoはMVT（Model-View-Template）アーキテクチャというデザインパターンを採用しています。これは、アプリケーションを3つの役割に分離することで、コードの保守性と拡張性を高める設計手法です。

Model（モデル）- データ層

データベースとのやり取りを担当します。データの構造（テーブル設計）や、データの保存・取得・更新・削除などの操作を定義します。

例: ブログ記事の情報（タイトル、本文、投稿日時など）をデータベースに保存・取得する処理

View（ビュー）- ロジック層

ユーザーのリクエストを受け取り、必要な処理を行い、レスポンスを返す「コントローラー」の役割を担います。Modelからデータを取得し、Templateに渡します。

例: 記事一覧ページにアクセスされたら、Modelから記事データを取得して、Templateに渡す処理

Template（テンプレート）- 表示層

ユーザーに表示するHTMLの見た目を定義します。Viewから受け取ったデータを埋め込んで、動的なWebページを生成します。

例: 記事データを受け取り、タイトルや本文を含むHTMLページを生成する処理

MVCとMVTの違い

他のフレームワーク（Ruby on RailsやLaravelなど）で使われるMVC（Model-View-Controller）パターンと、DjangoのMVTは類似していますが、名称と役割が少し異なります。

MVC（一般的なフレームワーク）

  • • Model: データ層
  • • View: 表示層（HTML生成）
  • • Controller: ロジック層（制御）

MVT（Django）

  • • Model: データ層
  • • View: ロジック層（制御）
  • • Template: 表示層（HTML生成）

実質的には同じ役割分担ですが、DjangoではViewがロジック（制御）を担い、Templateが表示を担当する点が特徴です。

1.4 開発環境のセットアップ

Djangoを使い始めるための環境を整えましょう。ここでは、Python環境の確認からDjangoのインストール、最初のプロジェクト作成までの手順を説明します。

ステップ1: Python環境の確認

まず、Pythonがインストールされていることを確認します。ターミナル（Windowsの場合はコマンドプロンプトまたはPowerShell）を開き、以下のコマンドを実行してください。

ターミナル 
# Pythonのバージョン確認
python --version
# または
python3 --version

期待される結果

Python 3.8.10 のように、Python 3.8以上のバージョンが表示されればOKです。

ステップ2: 仮想環境の作成

Pythonの仮想環境を作成します。仮想環境を使うことで、プロジェクトごとに独立したパッケージ環境を構築でき、他のプロジェクトとの依存関係の競合を防げます。

ターミナル 
# 作業ディレクトリを作成して移動
mkdir myproject
cd myproject

# 仮想環境を作成
python -m venv venv

# 仮想環境を有効化
# Windowsの場合:
venv\Scripts\activate
# macOS/Linuxの場合:
source venv/bin/activate

仮想環境が有効化されると

ターミナルのプロンプトの先頭に(venv)と表示されます。これは仮想環境内で作業していることを示します。

ステップ3: Djangoのインストール

仮想環境内にDjangoをインストールします。pipコマンドを使用して、最新の安定版Djangoをインストールします。

ターミナル 
# Djangoをインストール
pip install django

# インストール確認
django-admin --version

ステップ4: 最初のプロジェクト作成

Djangoプロジェクトを作成します。django-admin startprojectコマンドを使用して、プロジェクトの雛形を生成します。

ターミナル 
# プロジェクトを作成（「.」は現在のディレクトリに作成する指定）
django-admin startproject config .

# ディレクトリ構造を確認
ls -la  # Windows: dir

作成されるファイル構成 

myproject/
├── venv/              # 仮想環境（作成済み）
├── config/            # プロジェクト設定ディレクトリ
│   ├── __init__.py    # Pythonパッケージであることを示す
│   ├── settings.py    # プロジェクト全体の設定
│   ├── urls.py        # URLルーティング設定
│   ├── asgi.py        # ASGI（非同期）サーバー設定
│   └── wsgi.py        # WSGI（同期）サーバー設定
└── manage.py          # Django管理コマンド

ステップ5: 開発サーバーの起動

Djangoの開発サーバーを起動して、動作を確認します。

ターミナル 
# 開発サーバーを起動
python manage.py runserver

動作確認

ブラウザで http://127.0.0.1:8000/ にアクセスしてください。Djangoのウェルカムページが表示されれば成功です！

表示される内容：

  • 「The install worked successfully! Congratulations!」というメッセージ
  • 緑色のロケットのイラスト
  • 「You are seeing this page because DEBUG=True is in your settings file...」という説明

サーバーの停止方法

開発サーバーを停止するには、ターミナルで Ctrl + C を押してください。

理解度確認クイズ

  1. Q1. Djangoの「Batteries Included」という特徴は何を意味しますか？

    回答を表示

    Web開発に必要な機能（認証、管理画面、フォーム処理、ORMなど）が標準で提供されているという意味です。開発者は追加のライブラリをインストールすることなく、すぐに本格的なWebアプリケーション開発を始められます。

  2. Q2. MVTアーキテクチャにおいて、ユーザーのリクエストを受け取り、必要な処理を行う役割を担うのはどれですか？

    回答を表示

    View（ビュー）です。Viewはリクエストを受け取り、必要に応じてModelからデータを取得し、処理結果をTemplateに渡してレスポンスを返す「コントローラー」の役割を担います。

  3. Q3. Pythonの仮想環境を使う主な理由は何ですか？

    回答を表示

    プロジェクトごとに独立したパッケージ環境を構築し、他のプロジェクトとの依存関係の競合を防ぐためです。異なるプロジェクトで異なるバージョンのパッケージを使用する場合でも、仮想環境を分けることで互いに影響を与えずに開発できます。

第1章のまとめ

  • Djangoは「Batteries Included」の哲学で、Web開発に必要な機能が標準搭載されている高機能なPythonフレームワークです。
  • MVTアーキテクチャにより、Model（データ）、View（ロジック）、Template（表示）の役割が明確に分離され、保守性と拡張性が向上します。
  • 仮想環境を使うことで、プロジェクトごとに独立した開発環境を構築できます。
  • django-admin startprojectコマンドでプロジェクトの雛形を作成し、python manage.py runserverで開発サーバーを起動できます。
