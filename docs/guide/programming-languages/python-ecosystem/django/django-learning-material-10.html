認証システム - ユーザー管理の実装

Djangoの認証システムを活用して、ログイン・ログアウト機能、ユーザー登録、パスワード管理、アクセス制限の実装方法を学びます。

この章で学ぶこと

  • Djangoの認証システムの概要とUserモデルの理解
  • ログイン・ログアウト機能の実装とセッション管理
  • ユーザー登録フォームの作成と新規ユーザーの追加
  • パスワード変更・リセット機能の実装
  • ログイン必須ページの作成とアクセス制限の設定

10.1 Django認証システムの概要

Django認証システム（django.contrib.auth）は、Webアプリケーションでのユーザー認証とアクセス制御を実現するための強力な機能群です。ユーザー管理、パスワード暗号化、セッション管理、パーミッション管理など、認証に必要な機能がすべて組み込まれており、すぐに実用的な認証機能を実装できます。

認証システムはsettings.pyのINSTALLED_APPSに'django.contrib.auth'として最初から含まれているため、追加のインストールは不要です。

Userモデルの基本構造

Djangoの認証システムの中核となるUserモデルは、ユーザー情報を格納するデータベーステーブルです。以下のフィールドが標準で用意されています。

Userモデルの主要フィールド

フィールド 説明 必須
username ユーザー名（ログインID） 必須
password パスワード（ハッシュ化済み） 必須
email メールアドレス 任意
first_name 名 任意
last_name 姓 任意
is_staff 管理画面へのアクセス権限 デフォルトFalse
is_active アカウントの有効/無効 デフォルトTrue
is_superuser スーパーユーザー権限 デフォルトFalse
date_joined 登録日時 自動設定
last_login 最終ログイン日時 自動更新

パスワードのセキュリティ

Djangoはパスワードを平文で保存せず、PBKDF2アルゴリズムを使用してハッシュ化します。これにより、データベースが漏洩した場合でも元のパスワードを復元できない強固なセキュリティが確保されます。

10.2 ログイン・ログアウト機能の実装

Djangoにはdjango.contrib.authモジュールに、ログイン・ログアウトの処理を簡単に実装できる関数が用意されています。ここでは、これらを使ってログイン機能を実装する方法を学びます。

ログイン処理の流れ

sequenceDiagram participant U as ユーザー participant V as ログインビュー participant A as authenticate関数 participant L as login関数 participant S as セッション U->>V: ユーザー名とパスワードを送信 V->>A: 認証情報を渡す A->>A: パスワードをハッシュ化して比較 alt 認証成功 A->>V: Userオブジェクトを返す V->>L: ログイン処理を実行 L->>S: セッションを作成 S->>U: ログイン成功（セッションID付与） else 認証失敗 A->>V: Noneを返す V->>U: エラーメッセージを表示 end

ステップ1: URLパターンの定義

まず、アプリケーションのurls.pyにログイン・ログアウト用のURLパターンを定義します。

accounts/urls.py 
from django.urls import path
from . import views

app_name = 'accounts'

urlpatterns = [
    # ログイン・ログアウト
    path('login/', views.login_view, name='login'),
    path('logout/', views.logout_view, name='logout'),
]

ステップ2: ログインビューの実装

views.pyにログイン処理を実装します。authenticate()で認証し、login()でセッションを作成します。

accounts/views.py 
from django.shortcuts import render, redirect
from django.contrib.auth import authenticate, login, logout
from django.contrib import messages

def login_view(request):
    # POSTリクエストの場合、ログイン処理を実行
    if request.method == 'POST':
        username = request.POST['username']
        password = request.POST['password']

        # ユーザー認証
        user = authenticate(request, username=username, password=password)

        if user is not None:
            # 認証成功：ログイン処理
            login(request, user)
            messages.success(request, 'ログインしました')
            return redirect('home')  # ホームページへリダイレクト
        else:
            # 認証失敗：エラーメッセージ
            messages.error(request, 'ユーザー名またはパスワードが正しくありません')

    # GETリクエストの場合、ログインフォームを表示
    return render(request, 'accounts/login.html')

ステップ3: ログアウトビューの実装

ログアウト処理はlogout()関数を呼ぶだけで完了します。

accounts/views.py（続き） 
def logout_view(request):
    # ログアウト処理
    logout(request)
    messages.info(request, 'ログアウトしました')
    return redirect('accounts:login')  # ログインページへリダイレクト

ステップ4: ログインテンプレートの作成

ログインフォームを表示するテンプレートを作成します。

templates/accounts/login.html 
<h2>ログイン</h2>

<!-- メッセージ表示 -->
{% if messages %}
  {% for message in messages %}
    <div class="alert">{{ message }}</div>
  {% endfor %}
{% endif %}

<!-- ログインフォーム -->
<form method="post">
  {% csrf_token %}
  <div>
    <label for="username">ユーザー名:</label>
    <input type="text" id="username" name="username" required>
  </div>
  <div>
    <label for="password">パスワード:</label>
    <input type="password" id="password" name="password" required>
  </div>

主要な認証関数

  • authenticate(): ユーザー名とパスワードで認証、成功時にUserオブジェクトを返す
  • login(): セッションを作成し、ユーザーをログイン状態にする
  • logout(): セッションを削除し、ログアウト状態にする

10.3 ユーザー登録機能の実装

新規ユーザーをアプリケーションに登録する機能を実装します。DjangoのUserCreationFormを使うことで、簡単にユーザー登録フォームを作成できます。

ステップ1: ユーザー登録フォームの作成

forms.pyにユーザー登録用のフォームを作成します。メールアドレスフィールドを追加してカスタマイズします。

accounts/forms.py 
from django import forms
from django.contrib.auth.forms import UserCreationForm
from django.contrib.auth.models import User

class SignUpForm(UserCreationForm):
    # メールアドレスフィールドを追加
    email = forms.EmailField(max_length=254, required=True)

    class Meta:
        model = User
        fields = ('username', 'email', 'password1', 'password2')

ステップ2: ユーザー登録ビューの実装

フォームを処理して新規ユーザーを作成し、自動的にログインさせます。

accounts/views.py（続き） 
from .forms import SignUpForm

def signup_view(request):
    if request.method == 'POST':
        form = SignUpForm(request.POST)
        if form.is_valid():
            # ユーザーを作成
            user = form.save()
            # 登録後、自動的にログイン
            login(request, user)
            messages.success(request, 'アカウントが作成されました')
            return redirect('home')
    else:
        form = SignUpForm()

    return render(request, 'accounts/signup.html', {'form': form})

ステップ3: 登録フォームテンプレートの作成

ユーザー登録フォームを表示するテンプレートを作成します。

<form method="post">
  {% csrf_token %}
  {{ form.as_p }}
  <button type="submit">登録</button>
</form>

UserCreationFormの特徴

UserCreationFormには、パスワードの確認入力フィールドと、パスワード強度のバリデーションが標準で実装されています。ユーザー名の重複チェックも自動的に行われます。

10.4 パスワード管理機能の実装

ログイン中のユーザーがパスワードを変更したり、パスワードを忘れた場合にリセットできる機能を実装します。Djangoには、これらの機能を簡単に実装できるビューが用意されています。

パスワード変更機能

ログイン中のユーザーが現在のパスワードを入力して、新しいパスワードに変更できる機能です。

accounts/urls.py（更新） 
from django.contrib.auth import views as auth_views

urlpatterns = [
    path('login/', views.login_view, name='login'),
    path('logout/', views.logout_view, name='logout'),
    path('signup/', views.signup_view, name='signup'),

    # パスワード変更
    path('password_change/', auth_views.PasswordChangeView.as_view(
        template_name='accounts/password_change.html',
        success_url='/accounts/password_change/done/'
    ), name='password_change'),

    path('password_change/done/', auth_views.PasswordChangeDoneView.as_view(
        template_name='accounts/password_change_done.html'
    ), name='password_change_done'),
]

パスワードリセット機能

パスワードを忘れたユーザーがメールアドレスを入力し、リセットリンクを受け取って新しいパスワードを設定できる機能です。

accounts/urls.py（続き） 
    # パスワードリセット
    path('password_reset/', auth_views.PasswordResetView.as_view(
        template_name='accounts/password_reset.html'
    ), name='password_reset'),

    path('password_reset/done/', auth_views.PasswordResetDoneView.as_view(
        template_name='accounts/password_reset_done.html'
    ), name='password_reset_done'),

    path('reset/<uidb64>/<token>/', auth_views.PasswordResetConfirmView.as_view(
        template_name='accounts/password_reset_confirm.html'
    ), name='password_reset_confirm'),

    path('reset/done/', auth_views.PasswordResetCompleteView.as_view(
        template_name='accounts/password_reset_complete.html'
    ), name='password_reset_complete'),
]

メール送信の設定が必要

パスワードリセット機能を動作させるには、settings.pyでメールサーバーの設定を行う必要があります。開発環境では、コンソールにメールを表示する設定が便利です。

settings.py（開発環境用メール設定）

10.5 アクセス制限とパーミッション

特定のページをログインユーザーのみに限定したり、権限に応じてアクセスを制御する方法を学びます。Djangoには、ビューやテンプレートで使えるアクセス制限の仕組みが用意されています。

関数ビューでのアクセス制限

@login_requiredデコレータを使用すると、ログインしていないユーザーをログインページにリダイレクトできます。

views.py（関数ビューの例） 
from django.contrib.auth.decorators import login_required

@login_required
def my_profile_view(request):
    # ログインユーザーのプロフィールを表示
    return render(request, 'accounts/profile.html', {
        'user': request.user
    })

クラスベースビューでのアクセス制限

クラスベースビューではLoginRequiredMixinを使います。

views.py（クラスベースビューの例） 
from django.contrib.auth.mixins import LoginRequiredMixin
from django.views.generic import TemplateView

class ProfileView(LoginRequiredMixin, TemplateView):
    template_name = 'accounts/profile.html'

    # ログインページのURL（デフォルトは /accounts/login/）
    login_url = '/accounts/login/'

テンプレート内での認証状態確認

テンプレート内で、ログイン状態に応じて表示内容を切り替えることができます。

base.html（テンプレートの例） 
<nav>
  {% if user.is_authenticated %}
    <!-- ログイン中のユーザーに表示 -->
    <p>ようこそ、{{ user.username }}さん</p>
    <a href="{% url 'accounts:profile' %}">プロフィール</a>
    <a href="{% url 'accounts:logout' %}">ログアウト</a>
  {% else %}
    <!-- ログインしていないユーザーに表示 -->
    <a href="{% url 'accounts:login' %}">ログイン</a>
    <a href="{% url 'accounts:signup' %}">新規登録</a>
  {% endif %}
</nav>

権限に基づくアクセス制限

特定の権限を持つユーザーのみにアクセスを許可することもできます。

views.py（権限チェックの例） 
from django.contrib.auth.decorators import permission_required

# 「blog.add_post」権限を持つユーザーのみアクセス可能
@permission_required('blog.add_post')
def create_post_view(request):
    return render(request, 'blog/create_post.html')

ログインページのURL設定

settings.pyでLOGIN_URLを設定すると、ログインが必要な場合のリダイレクト先を指定できます。

LOGIN_URL = '/accounts/login/'

アクセス制限方法の比較

方法 用途 使用例
@login_required ログイン必須（関数ビュー） プロフィールページ
LoginRequiredMixin ログイン必須（クラスベース） マイページ一覧
@permission_required 特定の権限が必要 記事投稿ページ
user.is_staff 管理者のみ 管理画面
user.is_superuser スーパーユーザーのみ システム設定

理解度確認クイズ

  1. Q1. authenticate()関数とlogin()関数の違いは何ですか？

    回答を表示

    authenticate()は、ユーザー名とパスワードが正しいかを確認し、認証成功時にUserオブジェクトを返します。login()は、認証済みのUserオブジェクトを受け取り、セッションを作成してユーザーをログイン状態にします。認証と実際のログイン処理は別のステップです。

  2. Q2. Djangoはパスワードをどのように保存していますか？

    回答を表示

    パスワードは平文で保存されず、PBKDF2アルゴリズムを使用してハッシュ化されます。ハッシュ化されたパスワードは元に戻せないため、データベースが漏洩した場合でも元のパスワードを復元できません。ログイン時は、入力されたパスワードをハッシュ化して保存されたハッシュと比較します。

  3. Q3. @login_requiredデコレータを使わずに、ビュー内でログイン状態を確認する方法は？

    回答を表示

    request.user.is_authenticatedで確認できます。ログインしている場合はTrue、していない場合はFalseを返します。例: if not request.user.is_authenticated: return redirect('login')

よくあるトラブルと対処法

問題: ログイン後にホームページにリダイレクトされない

原因: LOGIN_REDIRECT_URLが設定されていない可能性があります。

対処法:

# settings.py
LOGIN_REDIRECT_URL = '/' # ログイン後のリダイレクト先

問題: CSRF検証エラーが発生する

原因: フォームに{% csrf_token %}が含まれていない。

対処法:

すべてのPOSTフォームに{% csrf_token %}を追加してください。

問題: パスワードリセットメールが送信されない

原因: メールバックエンドが設定されていない。

対処法:

開発環境ではEMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'を設定し、ターミナルでメール内容を確認してください。

第10章のまとめ

  • Django認証システムは、ユーザー管理、パスワード暗号化、セッション管理など、認証に必要な機能をすべて標準で提供しています。
  • authenticate()とlogin()関数を使用して、ユーザー認証とセッション作成を実装できます。ログアウトはlogout()関数で行います。
  • UserCreationFormを使用することで、ユーザー登録フォームを簡単に作成でき、パスワード強度チェックや重複確認も自動的に行われます。
  • パスワード変更・リセット機能は、Djangoの組み込みビューを使用することで簡単に実装できます。
  • @login_requiredデコレータやLoginRequiredMixinを使用して、ログインが必要なページを簡単に保護できます。
  • テンプレートではuser.is_authenticatedを使用して、ログイン状態に応じた表示切り替えができます。
