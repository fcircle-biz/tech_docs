第4章

ビュー - リクエストを処理してレスポンスを返す

関数ビューとクラスベースビュー、HttpRequestとHttpResponseオブジェクト、コンテキストデータの渡し方を学びます。

この章で学ぶこと

  • ビューの役割とMVTアーキテクチャにおける位置づけ
  • 関数ベースビュー（FBV）の書き方と実装パターン
  • HttpRequestとHttpResponseオブジェクトの扱い方
  • テンプレートへのコンテキストデータの渡し方
  • クラスベースビュー（CBV）の基礎概念

4.1 ビューとは

ビュー（View）は、DjangoのMVTアーキテクチャにおいて「ロジック層」を担当する重要な要素です。ユーザーからのHTTPリクエストを受け取り、必要な処理を行い、HTTPレスポンスを返すという、Webアプリケーションの中心的な役割を果たします。

他のフレームワークでは「Controller（コントローラー）」と呼ばれることが多い部分ですが、Djangoではこれを「View」と呼びます。第1章で学んだように、Djangoでは「Template」が表示層を担当するため、名称が異なりますが役割は同じです。

ビューの主な責務

  • • リクエストの受付: URLから渡されたパラメータを取得
  • • ビジネスロジックの実行: 必要な処理や計算を実行
  • • データの取得: Modelを通じてデータベースから情報を取得
  • • データの加工: テンプレートで表示しやすい形に整形
  • • レスポンスの生成: HTMLページやJSONデータを返却

4.2 関数ベースビュー（Function-Based View: FBV）

関数ベースビューは、Pythonの通常の関数としてビューを定義する最もシンプルな方法です。初心者にとって理解しやすく、小規模なビューや単純な処理に適しています。

基本的な関数ビューの構造

関数ビューは、HttpRequestオブジェクトを受け取り、HttpResponseオブジェクトを返す関数です。

blog/views.py 
# 最もシンプルなビュー
from django.http import HttpResponse

def hello_view(request):
    # requestオブジェクトを受け取る
    # HttpResponseオブジェクトを返す
    return HttpResponse("Hello, Django!")

テンプレートをレンダリングするビュー

実際のWebアプリケーションでは、HTMLテンプレートをレンダリングしてレスポンスを返すことが一般的です。render()ショートカット関数を使用します。

blog/views.py 
# テンプレートをレンダリングするビュー
from django.shortcuts import render

def article_list(request):
    # テンプレートをレンダリングしてレスポンスを返す
    return render(request, 'blog/article_list.html')

URLパラメータを受け取るビュー

URLから渡されたパラメータを関数の引数として受け取ることができます。URLルーティングの設定と連携して動作します。

blog/views.py 
# URLパラメータを受け取るビュー
from django.shortcuts import render

def article_detail(request, article_id):
    # URLから渡された記事IDを受け取る
    # 例: /blog/5/ → article_id = 5
    context = {
        'article_id': article_id
    }
    return render(request, 'blog/article_detail.html', context)

render()関数の役割

render(request, template_name, context)は、テンプレートをレンダリングしてHttpResponseを返すショートカット関数です。内部では、テンプレートの読み込み、コンテキストデータの埋め込み、HttpResponseオブジェクトの生成を自動的に行います。

4.3 HttpRequestとHttpResponseオブジェクト

DjangoのビューはHTTPプロトコルに基づいて動作します。HttpRequestオブジェクトにはクライアントからの情報が、HttpResponseオブジェクトにはサーバーからの応答情報が含まれます。

HttpRequestオブジェクト

ビュー関数の第1引数として必ず渡されるオブジェクトで、クライアントからのリクエスト情報にアクセスできます。

主なHttpRequest属性

属性 説明 例
request.method HTTPメソッド 'GET', 'POST', 'PUT'
request.GET GETパラメータ ?page=2 → {'page': '2'}
request.POST POSTデータ フォーム送信データ
request.path URLパス '/blog/5/'
request.user 現在のユーザー User オブジェクト
request.session セッションデータ 辞書型オブジェクト
HttpRequestの使用例 
# リクエスト情報を使用するビュー
def search_view(request):
    # GETパラメータを取得
    query = request.GET.get('q', '')  # デフォルト値は空文字

    # メソッドをチェック
    if request.method == 'POST':
        # POSTデータを処理
        keyword = request.POST.get('keyword')

    # ユーザー情報を確認
    if request.user.is_authenticated:
        username = request.user.username

    return render(request, 'search.html')

HttpResponseオブジェクト

ビュー関数が返すオブジェクトで、クライアントへのレスポンス内容を定義します。通常はrender()などのショートカット関数を使いますが、直接作成することもできます。

HttpResponseの例 
from django.http import HttpResponse, JsonResponse
from django.shortcuts import redirect

# テキストレスポンス
def text_view(request):
    return HttpResponse("Plain text response")

# JSONレスポンス
def json_view(request):
    data = {'status': 'success', 'count': 10}
    return JsonResponse(data)

# リダイレクト
def redirect_view(request):
    return redirect('/blog/')

4.4 コンテキストデータの渡し方

コンテキスト（Context）は、ビューからテンプレートにデータを渡すための辞書オブジェクトです。テンプレート内で変数として使用できるデータを設定します。

基本的なコンテキストの渡し方

blog/views.py 
# 単純なコンテキストデータ
def greeting_view(request):
    context = {
        'name': '山田太郎',
        'message': 'こんにちは',
        'year': 2025
    }
    return render(request, 'greeting.html', context)
templates/greeting.html 
<!-- テンプレート内でコンテキスト変数を使用 -->
<h1>{{ message }}、{{ name }}さん！</h1>
<p>今年は{{ year }}年です。</p>

モデルデータを渡す

データベースから取得したモデルオブジェクトをコンテキストに含めることで、テンプレートで表示できます。

blog/views.py 
# モデルデータをコンテキストに渡す
from .models import Article

def article_list_view(request):
    # データベースから記事を全件取得
    articles = Article.objects.all()

    context = {
        'articles': articles,
        'total_count': articles.count()
    }
    return render(request, 'blog/article_list.html', context)
templates/blog/article_list.html 
<!-- テンプレートでモデルデータを表示 -->
<h1>記事一覧 (全{{ total_count }}件)</h1>
<ul>
  {% for article in articles %}
    <li>{{ article.title }} - {{ article.author }}</li>
  {% endfor %}
</ul>

重要ポイント

コンテキストのキー名がテンプレート内の変数名になります。わかりやすく、一貫性のある命名を心がけましょう。複数の単語を組み合わせる場合は、スネークケース（article_list）を使用するのが一般的です。

4.5 クラスベースビュー（Class-Based View: CBV）入門

クラスベースビュー（CBV）は、Pythonのクラスを使ってビューを定義する方法です。関数ベースビューよりも再利用性が高く、DjangoはCRUD操作などの一般的なパターンを提供しています。

FBV vs CBVの比較

関数ベースビュー（FBV）

  • シンプルで理解しやすい
  • 小規模な処理に適している
  • コードの再利用が難しい
  • 条件分岐が複雑になりがち

クラスベースビュー（CBV）

  • コードの再利用性が高い
  • 標準的な処理が簡潔に書ける
  • 学習コストが高い
  • 内部動作が見えにくい

シンプルなクラスベースビューの例

blog/views.py 
# クラスベースビューの基本
from django.views import View
from django.shortcuts import render

class ArticleListView(View):
    # GETリクエストの処理
    def get(self, request):
        return render(request, 'blog/article_list.html')

    # POSTリクエストの処理
    def post(self, request):
        # POSTデータの処理
        return render(request, 'blog/success.html')
blog/urls.py 
# URLパターンでCBVを使用
from django.urls import path
from .views import ArticleListView

urlpatterns = [
    # as_view()メソッドで関数に変換
    path('articles/', ArticleListView.as_view(), name='article_list'),
]

汎用ビュー（Generic Views）

Djangoは、よく使われる処理パターンを実装した汎用ビュークラスを提供しています。これらを継承することで、少ないコードで高機能なビューを作成できます。

汎用ビューの例（ListView） 
# ListViewを使った記事一覧ビュー
from django.views.generic import ListView
from .models import Article

class ArticleListView(ListView):
    model = Article  # 使用するモデル
    template_name = 'blog/article_list.html'
    context_object_name = 'articles'  # テンプレート内の変数名
    paginate_by = 10  # ページネーション（1ページ10件）

学習のヒント

初心者は、まず関数ベースビュー（FBV）で基本を理解することをお勧めします。ビューの動作原理を理解してから、クラスベースビュー（CBV）に進むと、より深く理解できます。

実習 4-1: 簡単なビューを作成してみよう

実際にビューを作成して、リクエストとレスポンスの流れを体験しましょう。

手順

  1. 1 blog/views.pyに、現在時刻を表示する関数ビューを作成する
  2. 2 コンテキストに現在時刻を含め、テンプレートに渡す
  3. 3 URLパターンを設定し、ブラウザでアクセスして確認する

理解度確認クイズ

  1. Q1. ビュー関数が必ず受け取る第1引数は何ですか？また、何を返す必要がありますか？

    回答を表示

    第1引数はHttpRequestオブジェクト（通常はrequestという名前）です。戻り値としてHttpResponseオブジェクトを返す必要があります。render()関数などは内部でHttpResponseを生成して返してくれます。

  2. Q2. テンプレートにデータを渡すために使用する辞書オブジェクトを何と呼びますか？

    回答を表示

    コンテキスト（Context）と呼びます。render()関数の第3引数として渡し、そのキーがテンプレート内で変数名として使用できます。

  3. Q3. 関数ベースビュー（FBV）とクラスベースビュー（CBV）の主な違いは何ですか？

    回答を表示

    FBVは通常の関数として定義し、シンプルで理解しやすいのが特徴です。CBVはPythonクラスとして定義し、コードの再利用性が高く、継承やミックスインで機能を拡張できます。初心者はFBVから学び、慣れてからCBVに進むのが推奨されます。

第4章のまとめ

  • ビューは、MVTアーキテクチャのロジック層を担当し、リクエストを受け取りレスポンスを返す「コントローラー」の役割を果たします。
  • 関数ベースビュー（FBV）は、シンプルで理解しやすく、初心者の学習に適しています。HttpRequestを受け取り、HttpResponseを返します。
  • HttpRequestオブジェクトにはメソッド、GETパラメータ、POSTデータ、ユーザー情報など、クライアントからの情報が含まれます。
  • コンテキストは、ビューからテンプレートにデータを渡す辞書オブジェクトで、テンプレート内で変数として使用できます。
  • クラスベースビュー（CBV）は、コードの再利用性が高く、Djangoの汎用ビューを使うことで効率的な開発ができます。
