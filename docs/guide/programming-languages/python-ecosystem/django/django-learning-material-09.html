静的ファイルの管理 - CSS/JavaScript/画像を扱う

静的ファイルの設定と配信、テンプレートでの静的ファイル参照、メディアファイルのアップロード処理を学びます。

この章で学ぶこと

  • 静的ファイルとメディアファイルの違いと役割
  • settings.pyでの静的ファイル設定（STATIC_URL、STATIC_ROOT、STATICFILES_DIRS）
  • テンプレートでの {% static %} タグの使い方
  • メディアファイルのアップロードとMEDIA_ROOT、MEDIA_URLの設定

9.1 静的ファイルとは

Webアプリケーションには、プログラムによって動的に生成されるコンテンツだけでなく、CSS、JavaScript、画像ファイルなどの静的ファイル（Static Files）が必要です。Djangoでは、これらのファイルを効率的に管理・配信するための仕組みが用意されています。

静的ファイルは、開発時と本番環境で扱い方が異なります。開発時はDjangoの開発サーバーが自動的に配信しますが、本番環境ではNginxやApacheなどのWebサーバーから配信するのが一般的です。

静的ファイルとメディアファイルの違い

静的ファイル（Static Files）

開発者が作成する、アプリケーション固有のファイル

  • CSS（スタイルシート）
  • JavaScript（スクリプト）
  • 画像（ロゴ、アイコン等）

メディアファイル（Media Files）

ユーザーがアップロードするファイル

  • プロフィール画像
  • 記事の添付ファイル
  • ユーザーコンテンツ

静的ファイルの配信フロー

flowchart TB subgraph Dev[開発環境] A1[Djangoアプリ/static/] --> B1[開発サーバー] B1 --> C1[自動配信] end subgraph Prod[本番環境] A2[各アプリ/static/] --> B2[collectstatic] B2 --> D2[STATIC_ROOT] D2 --> E2[Nginx/Apache] E2 --> F2[ユーザーに配信] end style Dev fill:#e0f2fe,stroke:#0ea5e9 style Prod fill:#fef3c7,stroke:#f59e0b

開発環境では各アプリのstaticフォルダから直接配信されますが、本番環境ではcollectstaticコマンドで全ての静的ファイルを一箇所に集約し、Webサーバーから配信します。

9.2 静的ファイルの設定

静的ファイルを使用するために、settings.pyで必要な設定を行います。Djangoプロジェクトを作成すると、いくつかの設定はデフォルトで用意されていますが、追加の設定が必要な場合もあります。

主要な設定項目

STATIC_URL

静的ファイルにアクセスするためのURLパス。通常は /static/ が使われます。

例: STATIC_URL = '/static/'
テンプレートから /static/css/style.css でアクセス可能になります。

STATICFILES_DIRS

プロジェクト全体で共通して使う静的ファイルの保存場所。各アプリのstatic/フォルダに加えて、追加で読み込むディレクトリを指定します。

例: STATICFILES_DIRS = [BASE_DIR / 'static']
プロジェクトルートのstatic/フォルダも静的ファイルの検索対象になります。

STATIC_ROOT

本番環境でcollectstaticコマンドを実行した際に、全ての静的ファイルが集約されるディレクトリ。開発環境では通常使用しません。

例: STATIC_ROOT = BASE_DIR / 'staticfiles'
本番環境では、このフォルダをWebサーバーから配信します。

settings.pyの設定例

config/settings.py 
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

# 静的ファイルのURL（ブラウザからアクセスするパス）
STATIC_URL = '/static/'

# 開発時に追加で読み込むディレクトリ
STATICFILES_DIRS = [
    BASE_DIR / 'static',  # プロジェクトルートのstaticフォルダ
]

# 本番環境で collectstatic が集約するディレクトリ
STATIC_ROOT = BASE_DIR / 'staticfiles'

ディレクトリ構成のポイント

各アプリのmyapp/static/myapp/のように、アプリ名のサブディレクトリを作ることで、他のアプリとファイル名が衝突するのを防げます。

9.3 テンプレートで静的ファイルを参照する

テンプレートで静的ファイルを読み込むには、{% static %}テンプレートタグを使用します。このタグは、STATIC_URLを自動的に付与してくれるため、設定変更時にテンプレートを修正する必要がありません。

実習 9-1: 静的ファイルを使ったページ作成

CSSと画像を含むHTMLページを作成し、静的ファイルの読み込みを実践します。

手順

  1. 1 アプリ内に静的ファイル用のディレクトリを作成します
  2. 2 CSSファイルと画像ファイルを配置します
  3. 3 テンプレートで {% static %} タグを使って読み込みます

ステップ1: ディレクトリ構造の作成

作成するディレクトリ構造 

myapp/
├── static/
│   └── myapp/              # アプリ名でネームスペース分け
│       ├── css/
│       │   └── style.css   # スタイルシート
│       ├── js/
│       │   └── script.js   # JavaScriptファイル
│       └── images/
│           └── logo.png    # 画像ファイル
├── templates/
│   └── myapp/
│       └── index.html
└── views.py

ステップ2: CSSファイルの作成

myapp/static/myapp/css/style.css 
/* 基本スタイル */
body {
    font-family: sans-serif;
    margin: 0;
    padding: 20px;
}

.container {
    max-width: 800px;
    margin: 0 auto;
}

h1 {
    color: #2563eb;
}

ステップ3: テンプレートでの読み込み

myapp/templates/myapp/index.html 
{% load static %}
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>静的ファイルのサンプル</title>
    <!-- CSSファイルの読み込み -->
    <link rel="stylesheet" href="{% static 'myapp/css/style.css' %}">
</head>
<body>
    <div class="container">
        <h1>静的ファイルの読み込み</h1>
        <!-- 画像の表示 -->
        <img src="{% static 'myapp/images/logo.png' %}" alt="ロゴ">
    </div>
    <!-- JavaScriptファイルの読み込み -->
    <script src="{% static 'myapp/js/script.js' %}"></script>
</body>
</html>

重要なポイント

テンプレートの先頭で{% load static %}を必ず記述してください。これを忘れると、{% static %}タグが使えません。

9.4 メディアファイルのアップロード処理

メディアファイルは、ユーザーがアップロードするファイル（プロフィール画像、記事の添付ファイルなど）を指します。これらは静的ファイルとは別の設定が必要です。

メディアファイルの設定

config/settings.py 
# メディアファイルのURL
MEDIA_URL = '/media/'

# メディアファイルの保存先
MEDIA_ROOT = BASE_DIR / 'media'

開発環境でのURL設定

開発環境でメディアファイルを配信するには、URLconfに設定を追加する必要があります。本番環境では、Webサーバー（NginxやApache）から直接配信するため、この設定は不要です。

config/urls.py 
from django.contrib import admin
from django.urls import path, include
from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')),
]

# 開発環境のみメディアファイルの配信設定を追加
if settings.DEBUG:
    urlpatterns += static(
        settings.MEDIA_URL,
        document_root=settings.MEDIA_ROOT
    )

ファイルアップロードの実装例

モデルにImageFieldやFileFieldを定義することで、ファイルアップロード機能を簡単に実装できます。

myapp/models.py 
from django.db import models

class Profile(models.Model):
    name = models.CharField(max_length=100)
    # 画像ファイルのフィールド
    avatar = models.ImageField(upload_to='avatars/')
    # 通常のファイルフィールド
    resume = models.FileField(upload_to='resumes/', blank=True)

    def __str__(self):
        return self.name

Pillowライブラリのインストール

ImageFieldを使用する場合は、Pillowライブラリが必要です。pip install Pillowでインストールしてください。

フォームでのファイルアップロード

myapp/views.py 
from django.shortcuts import render, redirect
from .models import Profile
from .forms import ProfileForm

def upload_profile(request):
    if request.method == 'POST':
        form = ProfileForm(request.POST, request.FILES)
        if form.is_valid():
            form.save()
            return redirect('profile_list')
    else:
        form = ProfileForm()
    return render(request, 'myapp/upload.html', {'form': form})
myapp/templates/myapp/upload.html 
<!-- ファイルアップロードには enctype="multipart/form-data" が必須 -->
<form method="post" enctype="multipart/form-data">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">アップロード</button>
</form>

enctype属性を忘れずに

ファイルアップロードを含むフォームには、必ずenctype="multipart/form-data"を指定してください。これがないとファイルがアップロードされません。

アップロードされたファイルの表示

テンプレート内でのメディアファイル表示 
{% for profile in profiles %}
    <div>
        <h3>{{ profile.name }}</h3>
        {% if profile.avatar %}
            <img src="{{ profile.avatar.url }}" alt="Avatar">
        {% endif %}
    </div>
{% endfor %}

9.5 本番環境での静的ファイル配信

本番環境では、Djangoの開発サーバーではなく、NginxやApacheなどのWebサーバーから静的ファイルを配信するのが一般的です。そのため、collectstaticコマンドで全ての静的ファイルを一箇所に集約します。

collectstaticコマンドの実行

このコマンドは、各アプリのstatic/フォルダやSTATICFILES_DIRSで指定したディレクトリから、全ての静的ファイルをSTATIC_ROOTにコピーします。

ターミナル 
# 静的ファイルを収集
python manage.py collectstatic

# 確認メッセージが表示されたら yes を入力

本番環境での設定の流れ

1

settings.pyでSTATIC_ROOTを設定

例: STATIC_ROOT = '/var/www/myproject/static/'

2

collectstaticコマンドを実行

全ての静的ファイルがSTATIC_ROOTにコピーされます

3

Webサーバー（Nginx/Apache）の設定

STATIC_ROOTをドキュメントルートとして配信

Nginx設定例（参考）

/etc/nginx/sites-available/myproject 
server {
    listen 80;
    server_name example.com;

    # 静的ファイルの配信
    location /static/ {
        alias /var/www/myproject/static/;
    }

    # メディアファイルの配信
    location /media/ {
        alias /var/www/myproject/media/;
    }

    # Djangoアプリケーションへのプロキシ
    location / {
        proxy_pass http://127.0.0.1:8000;
    }
}

WhiteNoiseの活用

WhiteNoiseというパッケージを使うと、Webサーバーの設定なしでDjangoから静的ファイルを効率的に配信できます。小規模なアプリケーションでは便利な選択肢です。

理解度確認クイズ

  1. Q1. 静的ファイルとメディアファイルの違いは何ですか？

    回答を表示

    静的ファイルは開発者が作成するCSS、JavaScript、画像などのアプリケーション固有のファイルです。一方、メディアファイルはユーザーがアップロードするファイル（プロフィール画像、記事の添付ファイルなど）を指します。それぞれ異なる設定（STATIC_URL/MEDIA_URL、STATIC_ROOT/MEDIA_ROOT）で管理されます。

  2. Q2. テンプレートで静的ファイルを読み込むために必要な2つのステップは何ですか？

    回答を表示

    1つ目は、テンプレートの先頭で{% load static %}を記述すること。2つ目は、ファイルのパスを指定する際に{% static 'path/to/file' %}タグを使用することです。

  3. Q3. ファイルアップロードを含むHTMLフォームに必須の属性は何ですか？

    回答を表示

    enctype="multipart/form-data"属性が必須です。この属性がないと、ファイルデータがサーバーに送信されません。また、ビュー側でrequest.FILESを処理する必要があります。

第9章のまとめ

  • 静的ファイルは開発者が作成するCSS/JS/画像、メディアファイルはユーザーがアップロードするファイルです。
  • settings.pyでSTATIC_URL、STATICFILES_DIRS、STATIC_ROOTを設定することで、静的ファイルの管理ができます。
  • テンプレートでは{% static %}タグを使って静的ファイルを読み込みます。
  • ファイルアップロードにはImageField/FileFieldとenctype="multipart/form-data"が必要です。
  • 本番環境ではcollectstaticコマンドで静的ファイルを集約し、Webサーバーから配信します。
