第8章

フォーム - ユーザー入力を処理する

Djangoフォームの基礎、フォームフィールドとバリデーション、ModelFormによる効率的な開発、CSRF対策の実装を習得します。

この章で学ぶこと

  • Djangoフォームの基礎と役割、手動フォームとの違い
  • フォームフィールドの種類とバリデーション機能の実装
  • ModelFormを使った効率的なフォーム開発
  • CSRF攻撃からアプリケーションを守る方法

8.1 Djangoフォームとは

Djangoフォームは、HTMLフォームの生成、ユーザー入力の検証（バリデーション）、データのクリーニング（正規化）を自動化する強力な仕組みです。HTMLのフォームタグを手書きする代わりに、Pythonクラスでフォームを定義することで、セキュアで保守性の高いフォーム処理が実現できます。

Djangoフォームを使用すると、XSS（クロスサイトスクリプティング）やCSRF（クロスサイトリクエストフォージェリ）といった一般的なセキュリティ脅威への対策が自動的に組み込まれます。また、エラーメッセージの表示やフィールドの再入力なども簡単に実装できます。

Djangoフォームの処理フロー

手動フォーム vs Djangoフォーム

❌ 手動フォーム（非推奨）

  • • HTML/CSSを直接記述する必要がある
  • • バリデーションを自分で実装
  • • CSRF対策を手動で設定
  • • エラーハンドリングが煩雑
  • • コードの重複が発生しやすい

✅ Djangoフォーム（推奨）

  • • Pythonクラスでフォームを定義
  • • バリデーションが自動化
  • • CSRF対策が自動適用
  • • エラーメッセージが自動生成
  • • DRY原則に基づいた再利用可能なコード

8.2 基本的なフォームの作成

Djangoフォームはforms.Formクラスを継承して作成します。各フィールドは、フォームフィールドクラス（CharField、EmailFieldなど）のインスタンスとして定義します。

フォームクラスの定義

アプリケーション内にforms.pyファイルを作成し、フォームクラスを定義します。

myapp/forms.py 
from django import forms

class ContactForm(forms.Form):
    # 名前フィールド（必須、最大100文字）
    name = forms.CharField(
        max_length=100,
        label='お名前',
        widget=forms.TextInput(attrs={'class': 'form-control'})
    )

    # メールアドレスフィールド（必須、メール形式検証）
    email = forms.EmailField(
        label='メールアドレス',
        widget=forms.EmailInput(attrs={'class': 'form-control'})
    )

    # 件名フィールド（必須）
    subject = forms.CharField(max_length=200, label='件名')

    # メッセージフィールド（必須、複数行入力）
    message = forms.CharField(
        label='お問い合わせ内容',
        widget=forms.Textarea(attrs={'rows': 5})
    )

ビューでのフォーム処理

ビューでフォームをインスタンス化し、GETリクエストでは空のフォームを表示、POSTリクエストでは送信されたデータを検証します。

myapp/views.py 
from django.shortcuts import render, redirect
from .forms import ContactForm

def contact_view(request):
    # POSTリクエストの場合
    if request.method == 'POST':
        form = ContactForm(request.POST)  # 送信データでフォームを初期化
        if form.is_valid():  # バリデーション実行
            # 検証済みデータを取得
            name = form.cleaned_data['name']
            email = form.cleaned_data['email']
            # ... データベース保存やメール送信などの処理 ...
            return redirect('success')  # 成功ページへリダイレクト
    # GETリクエストの場合
    else:
        form = ContactForm()  # 空のフォームを作成

    return render(request, 'contact.html', {'form': form})

テンプレートでのフォーム表示

テンプレートでは{{ form }}変数を使ってフォームを表示します。CSRF対策のための{% csrf_token %}タグも必須です。

templates/contact.html 
<form method="post">
    {% csrf_token %}  <!-- CSRF対策トークン（必須） -->
    {{ form.as_p }}  <!-- フォームを段落タグで整形して出力 -->
    <button type="submit">送信</button>
</form>

フォームのレンダリング方法

{{ form.as_p }}: 各フィールドを<p>タグで囲んで出力
{{ form.as_table }}: <tr>タグで出力（<table>タグは手動で追加）
{{ form.as_ul }}: <li>タグで出力（<ul>タグは手動で追加）

8.3 フォームフィールドとバリデーション

Djangoは多様なフォームフィールドを提供しており、それぞれに適したバリデーション機能が組み込まれています。また、カスタムバリデーションも簡単に追加できます。

主なフォームフィールド

CharField

テキスト入力。max_length、min_lengthで文字数制限可能。

EmailField

メールアドレス入力。自動的にメール形式を検証。

IntegerField

整数入力。min_value、max_valueで範囲制限可能。

BooleanField

チェックボックス。True/Falseの選択。

DateField

日付入力。input_formatsで入力形式指定可能。

ChoiceField

選択肢から1つ選択。choicesで選択肢を定義。

カスタムバリデーションの追加

独自のバリデーションルールを追加するには、clean_<フィールド名>()メソッドを定義します。

カスタムバリデーション例 
from django import forms
from django.core.exceptions import ValidationError

class RegistrationForm(forms.Form):
    username = forms.CharField(max_length=50)
    age = forms.IntegerField()
    password = forms.CharField(widget=forms.PasswordInput)

    # usernameフィールドのカスタムバリデーション
    def clean_username(self):
        username = self.cleaned_data['username']
        # 禁止ワードチェック
        if 'admin' in username.lower():
            raise ValidationError('ユーザー名に"admin"は使用できません')
        return username

    # ageフィールドのカスタムバリデーション
    def clean_age(self):
        age = self.cleaned_data['age']
        # 年齢制限チェック
        if age < 18:
            raise ValidationError('18歳以上である必要があります')
        return age

バリデーションの流れ

1. is_valid()が呼ばれる
2. 各フィールドの型チェック（CharField、IntegerFieldなど）
3. カスタムバリデーション（clean_<フィールド名>メソッド）
4. すべて成功するとcleaned_dataに検証済みデータが格納される

8.4 ModelFormの活用

ModelFormは、Djangoモデルと連携したフォームを自動生成する機能です。モデルの定義からフォームフィールドを自動的に作成し、データベースへの保存も簡単に行えます。これにより、コードの重複を大幅に削減できます。

ModelFormの基本

まず、モデルを定義します。

myapp/models.py 
from django.db import models

class Article(models.Model):
    # 記事タイトル
    title = models.CharField(max_length=200)
    # 記事本文
    content = models.TextField()
    # 投稿日時（自動設定）
    created_at = models.DateTimeField(auto_now_add=True)
    # 公開/非公開フラグ
    is_published = models.BooleanField(default=False)

    def __str__(self):
        return self.title

次に、このモデルに対応するModelFormを作成します。

myapp/forms.py 
from django import forms
from .models import Article

class ArticleForm(forms.ModelForm):
    class Meta:
        model = Article  # 対応するモデル
        fields = ['title', 'content', 'is_published']  # 表示するフィールド
        # または fields = '__all__' で全フィールド

        # フィールドラベルのカスタマイズ
        labels = {
            'title': '記事タイトル',
            'content': '記事本文',
            'is_published': '公開する'
        }

        # ウィジェットのカスタマイズ
        widgets = {
            'content': forms.Textarea(attrs={'rows': 10}),
        }

ModelFormを使ったビュー

ModelFormを使うと、データベースへの保存がsave()メソッド1つで完了します。

myapp/views.py 
from django.shortcuts import render, redirect
from .forms import ArticleForm

def create_article(request):
    if request.method == 'POST':
        form = ArticleForm(request.POST)
        if form.is_valid():
            form.save()  # データベースに自動保存
            return redirect('article_list')
    else:
        form = ArticleForm()

    return render(request, 'create_article.html', {'form': form})

既存データの編集

ModelFormにinstance引数を渡すことで、既存データの編集ができます。

編集ビュー例 
from django.shortcuts import render, redirect, get_object_or_404
from .models import Article
from .forms import ArticleForm

def edit_article(request, pk):
    # 編集対象の記事を取得
    article = get_object_or_404(Article, pk=pk)

    if request.method == 'POST':
        # 既存データを初期値として設定
        form = ArticleForm(request.POST, instance=article)
        if form.is_valid():
            form.save()  # 既存データを更新
            return redirect('article_detail', pk=pk)
    else:
        form = ArticleForm(instance=article)

    return render(request, 'edit_article.html', {'form': form})

ModelFormの利点

• モデルの変更が自動的にフォームに反映される
• フィールドの型に応じた適切なウィジェットが自動選択される
• バリデーションがモデルの制約と連動する
• save()メソッドでデータベース操作が簡単

8.5 CSRF対策の実装

CSRF（Cross-Site Request Forgery：クロスサイトリクエストフォージェリ）は、悪意のあるWebサイトがユーザーの意図しないリクエストを送信させる攻撃です。Djangoは標準でCSRF対策機能を提供しており、簡単に実装できます。

CSRF攻撃のシナリオ

sequenceDiagram participant U as ユーザー participant B as 銀行サイト participant A as 悪意のあるサイト U->>B: 1. ログイン B-->>U: セッション確立 U->>A: 2. 悪意のあるサイトを訪問 A->>B: 3. ユーザーのセッションを悪用して送金リクエスト B->>B: 4. 正規のセッションと判断して処理 B-->>A: 送金完了 Note over U,B: ユーザーは気づかないうちに送金されてしまう

DjangoのCSRF対策

Djangoでは、フォームに{% csrf_token %}タグを含めるだけでCSRF対策が有効になります。

templates/form.html 
<form method="post" action="{% url 'submit_form' %}">
    {% csrf_token %}  <!-- CSRFトークンを埋め込む（必須） -->
    {{ form.as_p }}
    <button type="submit">送信</button>
</form>

CSRF保護の仕組み

  1. 1. トークン生成： Djangoがフォーム表示時にランダムなCSRFトークンを生成
  2. 2. トークン埋め込み： トークンがフォームのhiddenフィールドとして埋め込まれる
  3. 3. トークン送信： フォーム送信時にトークンも一緒に送信される
  4. 4. トークン検証： サーバー側でトークンの正当性を検証
  5. 5. リクエスト処理： トークンが正しい場合のみリクエストを処理

CSRF対策の設定確認

settings.pyで、CSRFミドルウェアが有効になっていることを確認します（デフォルトで有効）。

settings.py 
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',  # CSRF保護（重要）
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'django.middleware.clickjacking.XFrameOptionsMiddleware',
]

CSRF保護の重要性

POSTリクエストを受け付けるすべてのフォームで{% csrf_token %}を必ず使用してください。これを省略すると403 Forbiddenエラーが発生し、フォーム送信ができません。

実習 8-1: お問い合わせフォームを作成する

Djangoフォームを使って、名前、メールアドレス、メッセージを入力できるお問い合わせフォームを作成しましょう。

手順

  1. 1. forms.pyを作成

    ContactFormクラスを定義し、name、email、subjectフィールドを追加

  2. 2. ビューを作成

    GET/POSTリクエストを処理するcontact_view関数を実装

  3. 3. テンプレートを作成

    フォームを表示するHTMLを作成（csrf_tokenを忘れずに）

  4. 4. URLを設定

    urls.pyにパスを追加してアクセスできるようにする

  5. 5. 動作確認

    ブラウザでアクセスし、フォーム送信が成功することを確認

💡 確認ポイント

  • • フォームが正しく表示されるか
  • • バリデーションエラーが適切に表示されるか
  • • CSRF保護が有効になっているか

理解度確認クイズ

  1. Q1. Djangoフォームを使用する主な利点を3つ挙げてください。

    回答を表示

    1. 自動バリデーション：フィールドの型や制約に応じた検証が自動的に行われる
    2. セキュリティ：CSRF対策やXSS対策が組み込まれている
    3. コードの再利用性：DRY原則に基づき、重複コードを削減できる

  2. Q2. ModelFormと通常のFormの違いは何ですか？

    回答を表示

    ModelFormは、Djangoモデルと直接連携し、モデルの定義からフォームフィールドを自動生成します。save()メソッドでデータベースへの保存が簡単に行えます。一方、通常のFormは、モデルと関係なく独立して定義され、データベース操作は手動で実装する必要があります。

  3. Q3. CSRF攻撃から保護するために、テンプレートで必ず記述しなければならないタグは何ですか？

    回答を表示

    {% csrf_token %}タグです。このタグを<form>タグ内に記述することで、DjangoがCSRFトークンを生成し、リクエストの正当性を検証します。

第8章のまとめ

  • Djangoフォームは、HTMLフォームの生成、バリデーション、セキュリティ対策を自動化する強力な仕組みです。
  • forms.Formクラスを継承してフォームを定義し、様々なフィールドタイプとカスタムバリデーションを設定できます。
  • ModelFormを使用すると、モデルと連携したフォームを簡単に作成でき、データベース操作が効率化されます。
  • CSRF対策は{% csrf_token %}タグで実装でき、悪意のあるリクエストから保護します。
