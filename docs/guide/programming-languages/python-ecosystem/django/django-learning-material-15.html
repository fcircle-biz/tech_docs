まず、ブログアプリケーションに必要なデータ構造を設計します。主要なモデルは「記事（Post）」「カテゴリ（Category）」「コメント（Comment）」の3つです。

モデルの実装

blog/models.pyに以下のモデルを定義します。

blog/models.py 
from django.db import models
from django.contrib.auth.models import User
from django.utils.text import slugify

# カテゴリモデル
class Category(models.Model):
    name = models.CharField(max_length=100, unique=True)
    slug = models.SlugField(max_length=100, unique=True)
    description = models.TextField(blank=True)

    def save(self, *args, **kwargs):
        if not self.slug:
            self.slug = slugify(self.name)
        super().save(*args, **kwargs)

    def __str__(self):
        return self.name

# 記事モデル
class Post(models.Model):
    STATUS_CHOICES = [
        ('draft', '下書き'),
        ('published', '公開'),
    ]

    title = models.CharField(max_length=200)
    slug = models.SlugField(max_length=200, unique=True)

slugフィールドについて

slugは、URLに使用するための短い識別子です。日本語タイトルを自動的にローマ字化し、URL-フレンドリーな形式に変換します。

15.4 クラスベースビューの実装

Djangoのクラスベースビュー（CBV）を使用して、効率的に記事のCRUD機能を実装します。汎用ビューを活用することで、少ないコードで高機能なビューを作成できます。

blog/views.py 
from django.views.generic import ListView, DetailView, CreateView, UpdateView
from django.contrib.auth.mixins import LoginRequiredMixin
from django.urls import reverse_lazy
from .models import Post, Category

# 記事一覧ビュー
class PostListView(ListView):
    model = Post
    template_name = 'blog/post_list.html'
    context_object_name = 'posts'
    paginate_by = 10

    def get_queryset(self):
        # 公開済みの記事のみ、新しい順に取得
        return Post.objects.filter(
            status='published'
        ).order_by('-published_at')

15.6 フォームとバリデーション

ModelFormを使用して、記事投稿・編集フォームを実装します。カスタムバリデーションでタイトルの重複チェックなどを追加します。

blog/forms.py 
from django import forms
from .models import Post, Comment

class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'content', 'category', 'status']
        widgets = {
            'title': forms.TextInput(attrs={'class': 'form-control'}),
            'content': forms.Textarea(attrs={'class': 'form-control', 'rows': 10}),
            'category': forms.Select(attrs={'class': 'form-select'}),
            'status': forms.Select(attrs={'class': 'form-select'}),
        }

    def clean_title(self):
        title = self.cleaned_data.get('title')
        # 禁止ワードチェック（例）
        if '広告' in title:
            raise forms.ValidationError('タイトルに禁止ワードが含まれています。')
        return title

15.7 認証機能の統合

Djangoの認証システムを統合し、ログインユーザーのみが記事を投稿・編集できるようにします。

blog/views.py（認証追加） 
from django.contrib.auth.mixins import LoginRequiredMixin, UserPassesTestMixin

# 記事作成ビュー（ログイン必須）
class PostCreateView(LoginRequiredMixin, CreateView):
    model = Post
    form_class = PostForm
    template_name = 'blog/post_form.html'

    def form_valid(self, form):
        # 投稿者を自動設定
        form.instance.author = self.request.user
        return super().form_valid(form)

# 記事編集ビュー（作成者のみ編集可能）
class PostUpdateView(LoginRequiredMixin, UserPassesTestMixin, UpdateView):
    model = Post
    form_class = PostForm

    def test_func(self):
        post = self.get_object()
        return self.request.user == post.author

重要ポイント

LoginRequiredMixinはログイン必須、UserPassesTestMixinはカスタム条件チェックに使用します。Mixinは左側から順に評価されます。

15.9 パフォーマンス最適化

N+1問題を解決し、ページネーションとキャッシュを導入してパフォーマンスを改善します。

N+1問題の解決

select_relatedとprefetch_relatedを使用して、関連データを効率的に取得します。

キャッシュの活用

記事一覧やサイドバーなど、頻繁に変更されないデータをキャッシュします。

最適化されたクエリ 
# N+1問題を解決
posts = Post.objects.select_related('author', 'category')\
                    .prefetch_related('comments')\
                    .filter(status='published')\
                    .order_by('-published_at')

# ページネーション追加
from django.core.paginator import Paginator
paginator = Paginator(posts, 10)  # 1ページ10件
page_obj = paginator.get_page(page_number)

理解度確認クイズ

  1. Q1. N+1問題を解決するために使用するメソッドは何ですか？

    回答を表示

    select_relatedとprefetch_relatedを使用します。select_relatedは外部キー関係、prefetch_relatedは多対多や逆参照関係に対して効率的にデータを取得します。

  2. Q2. ログインユーザーのみがアクセスできるビューを作るために使用するMixinは何ですか？

    回答を表示

    LoginRequiredMixinを使用します。クラスベースビューの継承リストに追加することで、ログインしていないユーザーをログインページにリダイレクトできます。

  3. Q3. 本番環境にデプロイする際、settings.pyのDEBUGは何に設定すべきですか？

    回答を表示

    DEBUG = Falseに設定します。DEBUGをTrueのままにすると、エラー時に詳細な情報が表示され、セキュリティリスクとなります。本番環境では必ずFalseにしてください。
