Django Admin - 管理画面を活用する

管理画面の設定とカスタマイズ、モデルの登録、管理者ユーザーの作成、データの追加・編集・削除操作を学びます。

この章で学ぶこと

  • Django Adminの概要と利点を理解する
  • スーパーユーザーの作成とログイン方法を習得する
  • モデルを管理画面に登録する方法を学ぶ
  • リスト表示やフィールド表示のカスタマイズ方法を理解する
  • 管理画面でのデータ追加・編集・削除操作を実践する

7.1 Django Adminとは - 管理画面の役割と利点

Django Adminは、Djangoに標準で組み込まれている強力な管理画面（CMS: Content Management System）機能です。Webアプリケーション開発において、管理者がデータベースのデータを管理するための画面を作成するのは時間がかかる作業ですが、Django Adminを使えば、わずか数行のコードで高機能な管理画面を自動生成できます。

Django Adminは本番環境でも実際に使える品質を持ち、認証機能、データの追加・編集・削除、検索、フィルタリング、ページネーションなど、管理画面に必要な機能が最初から備わっています。これにより、開発者はバックエンドの管理画面を作り込む時間を節約し、フロントエンドのユーザー向け機能の開発に集中できます。

高速な開発

モデルを登録するだけで、CRUD（作成・読取・更新・削除）機能を持つ管理画面が自動生成されます。

組み込みセキュリティ

認証・認可機能が標準搭載。ユーザー権限管理やCSRF対策も自動で適用されます。

柔軟なカスタマイズ

表示項目、検索、フィルタ、並び替えなど、細かくカスタマイズ可能。独自の管理画面機能も追加できます。

本番環境対応

Instagram、Mozilla、NASAなど大規模サービスでも実際に使われている実績と信頼性があります。

Django Adminはどんな時に使う？

  • サイト管理者がブログ記事や商品情報を登録・編集する場合
  • 運用チームがユーザーデータや注文データを確認・修正する場合
  • 開発中にテストデータを簡単に作成・管理したい場合
  • マスターデータ（カテゴリ、タグ、設定値など）をメンテナンスする場合

7.2 管理サイトの有効化とアクセス

Django Adminは、デフォルトで有効化されています。新しいプロジェクトを作成すると、settings.pyのINSTALLED_APPSにdjango.contrib.adminが含まれており、URLルーティングも設定済みです。確認してみましょう。

設定ファイルの確認

config/settings.pyを開き、以下の項目が含まれていることを確認します。

config/settings.py 
INSTALLED_APPS = [
    'django.contrib.admin',  # 管理画面アプリ
    'django.contrib.auth',   # 認証システム
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    # あなたのアプリ
]

URLルーティングの確認

config/urls.pyを開き、管理画面のURLパターンが登録されていることを確認します。

config/urls.py 
from django.contrib import admin
from django.urls import path

urlpatterns = [
    path('admin/', admin.site.urls),  # 管理画面のURL
]

管理画面にアクセスしてみよう

開発サーバーを起動して、ブラウザで http://127.0.0.1:8000/admin/ にアクセスしてください。ログイン画面が表示されます。

まだログインできません

ログインするためには、管理者ユーザー（スーパーユーザー）を作成する必要があります。次のセクションで作成方法を学びます。

7.3 スーパーユーザーの作成

管理画面にログインするには、管理者権限を持つユーザー（スーパーユーザー）が必要です。スーパーユーザーは、すべてのデータとユーザーを管理できる最高権限を持つアカウントです。

スーパーユーザーの作成手順

ターミナルで以下のコマンドを実行します。いくつかの質問に答えるだけで、スーパーユーザーを作成できます。

ターミナル 
# スーパーユーザー作成コマンド
python manage.py createsuperuser

実行すると、以下のような質問が表示されます。順番に入力していきましょう。

対話形式の入力例 

Username (leave blank to use 'your_name'): admin
Email address: admin@example.com
Password: ********
Password (again): ********
Superuser created successfully.

Username（ユーザー名）

ログイン時に使用するユーザー名です。半角英数字で入力します（例: admin）。

Email address（メールアドレス）

メールアドレスを入力します。開発環境では実在しないアドレスでも問題ありません（例: admin@example.com）。

Password（パスワード）

パスワードを入力します。セキュリティのため、入力内容は画面に表示されません。2回同じパスワードを入力します。

開発環境でのパスワード: 簡単なパスワード（例: admin123）を設定すると警告が出ますが、y を入力すればそのまま作成できます。本番環境では必ず強固なパスワードを設定してください。

管理画面にログインしよう

  1. ブラウザで http://127.0.0.1:8000/admin/ にアクセス
  2. 先ほど作成したユーザー名とパスワードを入力
  3. 「Log in」ボタンをクリック

ログインに成功すると、Django管理画面のダッシュボードが表示されます！

7.4 モデルを管理画面に登録する

管理画面にログインしても、最初はグループとユーザーしか表示されません。自分で作成したモデル（例えば、第6章で作成したBookモデル）を管理画面で操作するには、モデルを「登録」する必要があります。

基本的な登録方法

アプリケーションフォルダ内のadmin.pyファイルを編集します。ここでは、第6章で作成したBookモデルを例に説明します。

myapp/admin.py 
from django.contrib import admin
from .models import Book

# Bookモデルを管理画面に登録
admin.site.register(Book)

たったこれだけです！ブラウザを再読み込みすると、管理画面に「Books」というセクションが表示され、データの追加・編集・削除ができるようになります。

登録の仕組み

  • admin.site.register(Book) でBookモデルを管理サイトに登録
  • Django Adminが自動的にモデルのフィールド情報を読み取り、適切な入力フォームを生成
  • CharField、TextField、DateField など、フィールドタイプに応じた入力欄が自動で配置される

複数のモデルを登録する

複数のモデルがある場合は、それぞれインポートして登録します。

myapp/admin.py 
from django.contrib import admin
from .models import Book, Author, Publisher

# 複数のモデルを登録
admin.site.register(Book)
admin.site.register(Author)
admin.site.register(Publisher)

7.5 管理画面のカスタマイズ基礎

基本的な登録だけでも十分便利ですが、管理画面をカスタマイズすることで、さらに使いやすくできます。よく使うカスタマイズ方法を紹介します。

リスト表示のカスタマイズ

デフォルトでは、モデルの__str__メソッドの値だけが一覧に表示されますが、list_displayを使うと、複数のフィールドを表示できます。

myapp/admin.py 
from django.contrib import admin
from .models import Book

# 管理画面のカスタマイズ用クラス
class BookAdmin(admin.ModelAdmin):
    # 一覧画面で表示するフィールドを指定
    list_display = ('title', 'author', 'published_date', 'price')

# カスタマイズクラスと一緒に登録
admin.site.register(Book, BookAdmin)

これで、一覧画面にタイトル、著者、出版日、価格が表形式で表示されるようになります。

検索機能を追加する

search_fieldsを指定すると、管理画面に検索ボックスが追加されます。

myapp/admin.py 
class BookAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'published_date', 'price')
    # タイトルと著者名で検索可能に
    search_fields = ('title', 'author')

admin.site.register(Book, BookAdmin)

フィルタ機能を追加する

list_filterを使うと、右側にフィルタサイドバーが表示され、特定の条件でデータを絞り込めます。

myapp/admin.py 
class BookAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'published_date', 'price')
    search_fields = ('title', 'author')
    # 出版日でフィルタリング可能に
    list_filter = ('published_date',)

admin.site.register(Book, BookAdmin)

デコレーター形式での登録

@admin.registerデコレーターを使うと、より簡潔に書けます。こちらの書き方も一般的です。

myapp/admin.py 
from django.contrib import admin
from .models import Book

# デコレーターを使った登録方法
@admin.register(Book)
class BookAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'published_date', 'price')
    search_fields = ('title', 'author')
    list_filter = ('published_date',)

よく使うカスタマイズオプション

list_display

一覧画面で表示するフィールドのリスト

search_fields

検索対象のフィールドのリスト

list_filter

フィルタリング可能なフィールドのリスト

ordering

デフォルトの並び順（例: ('-published_date',) で新しい順）

list_per_page

1ページあたりの表示件数（デフォルト: 100）

7.6 実習：管理画面でデータを操作する

実習 7-1: 管理画面でデータを追加・編集・削除する

実際に管理画面を使って、Bookモデルのデータを操作してみましょう。管理画面の基本操作をマスターします。

手順

  1. 1

    モデルの準備: 第6章で作成したBookモデルを使用します。まだ作成していない場合は、以下のモデルを作成してマイグレーションを実行してください。

    myapp/models.py 
    from django.db import models

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)
    published_date = models.DateField()
    price = models.IntegerField()

    def __str__(self):
        return self.title
  2. 2

    管理画面に登録: myapp/admin.py を編集して、Bookモデルを登録します。

    myapp/admin.py 
    from django.contrib import admin
from .models import Book

@admin.register(Book)
class BookAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'published_date', 'price')
    search_fields = ('title', 'author')
    list_filter = ('published_date',)
  3. 3 管理画面にアクセス: ブラウザで http://127.0.0.1:8000/admin/ にアクセスしてログインします。
  4. 4 データを追加: 「Books」セクションの「Add」ボタンをクリックし、以下のような書籍データを3冊追加してください。

追加するデータ例：

  • • タイトル: Python入門, 著者: 山田太郎, 出版日: 2023-01-15, 価格: 2800
  • • タイトル: Django実践ガイド, 著者: 佐藤花子, 出版日: 2023-06-20, 価格: 3200
  • • タイトル: Webアプリ開発の基礎, 著者: 鈴木次郎, 出版日: 2022-11-10, 価格: 2500
  1. 5 データを編集: 一覧画面で任意の書籍をクリックし、価格を変更して保存してみましょう。
  2. 6 検索機能を試す: 検索ボックスに著者名の一部（例: 「山田」）を入力して検索してみましょう。
  3. 7 フィルタ機能を試す: 右側のフィルタバーで出版日を選択し、特定の年のデータだけを表示してみましょう。
  4. 8 データを削除: チェックボックスで書籍を選択し、「Action」ドロップダウンから「Delete selected books」を選んで削除してみましょう（確認画面が表示されます）。

確認ポイント

  • 管理画面で書籍データを追加できましたか？
  • 一覧画面にタイトル、著者、出版日、価格が表示されていますか？
  • 検索ボックスで著者名やタイトルを検索できましたか？
  • フィルタで出版日を絞り込むことができましたか？
  • データの編集と削除が正常に動作しましたか?

トラブルシューティング

管理画面に「Books」が表示されない

原因: モデルが管理画面に登録されていない、またはマイグレーションが実行されていない可能性があります。

解決方法:

  • admin.py でモデルが登録されているか確認
  • python manage.py makemigrations と python manage.py migrate を実行
  • 開発サーバーを再起動してブラウザを再読み込み
パスワードを忘れてしまった

解決方法: changepassword コマンドでパスワードを変更できます。

python manage.py changepassword admin

または、新しいスーパーユーザーを作成することもできます。

管理画面のスタイルが崩れている

原因: 静的ファイルが正しく読み込まれていない可能性があります。

解決方法:

  • settings.py で django.contrib.staticfiles が INSTALLED_APPS に含まれているか確認
  • python manage.py collectstatic を実行（本番環境の場合）
  • 開発環境では通常、自動で静的ファイルが配信されるので、ブラウザのキャッシュをクリアしてみてください

理解度確認クイズ

  1. Q1. Django Adminの主な利点を3つ挙げてください。

    回答を表示

    主な利点：
    1. 高速な開発: モデルを登録するだけで管理画面が自動生成される
    2. セキュリティ: 認証・認可機能が標準搭載されている
    3. カスタマイズ性: 検索、フィルタ、表示項目などを柔軟にカスタマイズできる
    これ以外にも、本番環境で使える品質や豊富な機能が標準搭載されていることも大きな利点です。

  2. Q2. スーパーユーザーを作成するコマンドは何ですか？

    回答を表示

    python manage.py createsuperuser
    このコマンドを実行すると、ユーザー名、メールアドレス、パスワードを対話形式で入力してスーパーユーザーを作成できます。

  3. Q3. list_display、search_fields、list_filterは、それぞれどのような機能を提供しますか？

    回答を表示

    • list_display: 管理画面の一覧ページで表示するフィールドを指定します。複数のフィールドを表形式で表示できます。
    • search_fields: 検索機能を追加し、指定したフィールドで検索できるようにします。検索ボックスが画面上部に表示されます。
    • list_filter: フィルタリング機能を追加し、指定したフィールドで絞り込みができるようにします。フィルタサイドバーが右側に表示されます。

第7章のまとめ

  • Django Adminは、標準搭載の強力な管理画面機能で、モデルを登録するだけで高機能な管理画面が自動生成されます。
  • スーパーユーザーはpython manage.py createsuperuserコマンドで作成し、管理画面へのアクセス権限を持ちます。
  • admin.pyでモデルを登録することで、管理画面でデータのCRUD操作が可能になります。
  • ModelAdminクラスを使うことで、list_display、search_fields、list_filterなどのオプションで管理画面をカスタマイズできます。
  • 管理画面は開発中のデータ管理だけでなく、本番環境でもコンテンツ管理システムとして実際に利用できる品質を持っています。
