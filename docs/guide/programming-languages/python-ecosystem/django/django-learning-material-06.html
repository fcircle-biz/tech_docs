第6章

モデル - データベースとの架け橋

モデルの定義とフィールドタイプ、マイグレーションの仕組み、データベース操作の基礎、リレーションシップの設定を理解します。

この章で学ぶこと

  • Djangoモデルの役割とORMの基本概念
  • モデルの定義方法と主要なフィールドタイプの使い分け
  • マイグレーションの仕組みとデータベースへの反映手順
  • データベース操作の基礎（CRUD操作）とリレーションシップの設定

6.1 Djangoモデルとは

モデル（Model）は、DjangoのMVTアーキテクチャにおいてデータ層を担当する重要なコンポーネントです。モデルは、アプリケーションで扱うデータの構造を定義し、データベースとのやり取りを抽象化します。

DjangoはORM（Object-Relational Mapping）という技術を使用しており、Pythonのクラスとしてデータ構造を定義すると、自動的にデータベースのテーブルとして作成されます。これにより、SQLを直接書くことなく、Pythonのコードだけでデータベース操作が可能になります。

ORMの利点

flowchart LR A[Pythonコード] -->|ORM| B[SQL文] B --> C[(データベース)] C -->|結果| B B -->|Pythonオブジェクト| A style A fill:#dbeafe,stroke:#3b82f6 style B fill:#fef3c7,stroke:#f59e0b style C fill:#dcfce7,stroke:#10b981

データベース非依存

PostgreSQL、MySQL、SQLiteなど、異なるデータベースでも同じコードで動作します。

Pythonで操作

SQLを書かずに、Pythonのメソッドでデータベース操作が可能です。

安全性向上

SQLインジェクション攻撃などのセキュリティリスクを自動的に防ぎます。

保守性向上

データ構造の変更をPythonコードで管理でき、チーム開発でも扱いやすくなります。

6.2 モデルの定義とフィールドタイプ

モデルは、models.pyファイルに定義します。Djangoのmodels.Modelクラスを継承したクラスとして作成し、フィールドを定義していきます。

基本的なモデルの定義

ブログの記事を表すモデルを例に、基本的な定義方法を見てみましょう。

blog/models.py 
from django.db import models

# 記事モデルの定義
class Article(models.Model):
    # タイトル（最大200文字）
    title = models.CharField(max_length=200)
    # 本文（長いテキスト）
    content = models.TextField()
    # 公開日時（自動設定）
    published_date = models.DateTimeField(auto_now_add=True)
    # 更新日時（自動更新）
    updated_date = models.DateTimeField(auto_now=True)
    # 公開状態（デフォルト：下書き）
    is_published = models.BooleanField(default=False)

モデル定義のポイント

クラス名は単数形で定義します（Article、User、Productなど）。Djangoは自動的に複数形のテーブル名を生成します（articles、users、productsなど）。

主要なフィールドタイプ

Djangoには、様々なデータ型に対応したフィールドタイプが用意されています。

フィールドタイプ 用途 例
CharField 短いテキスト タイトル、名前
TextField 長いテキスト 本文、説明
IntegerField 整数 年齢、数量
DecimalField 小数（金額向け） 価格、金額
BooleanField 真偽値 公開フラグ
DateTimeField 日時 作成日時
DateField 日付のみ 誕生日
EmailField メールアドレス 連絡先メール
URLField URL Webサイト
ImageField 画像ファイル プロフィール画像

フィールドオプション

各フィールドには、動作を制御するためのオプションを指定できます。

models.py 
class Product(models.Model):
    # null=True: データベースでNULL許可
    # blank=True: フォームで空欄許可
    name = models.CharField(max_length=100)
    description = models.TextField(blank=True)
    price = models.DecimalField(
        max_digits=10,  # 全体の桁数
        decimal_places=2  # 小数点以下の桁数
    )
    # default: デフォルト値を指定
    stock = models.IntegerField(default=0)
    # unique=True: 一意制約（重複不可）
    code = models.CharField(max_length=50, unique=True)

主要なフィールドオプション

  • null=True データベースレベルでNULL（空）を許可
  • blank=True フォームでの入力を必須ではなくする
  • default フィールドのデフォルト値を設定
  • unique=True 値の重複を許可しない（一意制約）
  • choices 選択肢を制限する

6.3 マイグレーションの仕組み

マイグレーション（Migration）は、モデルの変更をデータベースに反映するための仕組みです。モデルを定義したり変更したりした後、マイグレーションを実行することで、データベースのテーブル構造を更新できます。

ステップ1: モデルを定義する

まず、アプリのmodels.pyにモデルを定義します。

blog/models.py 
from django.db import models

class Article(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    published_date = models.DateTimeField(auto_now_add=True)

ステップ2: マイグレーションファイルを作成

makemigrationsコマンドを実行して、モデルの変更を検出し、マイグレーションファイルを生成します。

ターミナル 
# マイグレーションファイルを作成
python manage.py makemigrations

# 特定のアプリのみ
python manage.py makemigrations blog

成功すると

blog/migrations/0001_initial.pyのようなマイグレーションファイルが作成されます。

ステップ3: データベースに反映する

migrateコマンドで、マイグレーションをデータベースに適用します。

ターミナル 
# すべてのマイグレーションを実行
python manage.py migrate

# 特定のアプリのみ
python manage.py migrate blog

マイグレーションの確認

python manage.py showmigrationsコマンドで、どのマイグレーションが適用済みか確認できます。

よくあるマイグレーションのケース

1

新しいフィールドを追加する場合

既存のモデルに新しいフィールドを追加する際は、defaultまたはnull=Trueを指定する必要があります。

理由: 既存のレコードに対して、新しいフィールドにどんな値を入れるかを決める必要があるためです。
2

フィールド名を変更する場合

マイグレーション時にDjangoが変更を検出し、名前変更かどうか確認してきます。

注意: データを保持したまま名前を変更できますが、慎重に操作しましょう。
3

フィールドを削除する場合

フィールドを削除すると、そのカラムのデータもすべて削除されます。

警告: 本番環境では必ずバックアップを取ってから実行してください。

6.4 データベース操作の基礎（CRUD）

Djangoのモデルを使って、データベースの基本操作であるCRUD（Create、Read、Update、Delete）を実行する方法を学びます。DjangoのORMを使えば、Pythonのコードだけでこれらの操作が可能です。

Create（作成）- データを追加する

新しいレコードを作成するには、モデルのインスタンスを作成してsave()メソッドを呼び出します。

Python Shell 
# 方法1: インスタンスを作成してsave()
article = Article(
    title="Django入門",
    content="Djangoの基本を学びます"
)
article.save()  # データベースに保存

# 方法2: create()で一度に作成と保存
article = Article.objects.create(
    title="Python入門",
    content="Pythonの基礎を学びます"
)

Read（読み取り）- データを取得する

データベースからデータを取得するには、objectsマネージャーを使います。

Python Shell 
# すべての記事を取得
articles = Article.objects.all()

# 条件で絞り込み（filter）
published = Article.objects.filter(is_published=True)

# 1件だけ取得（get）
article = Article.objects.get(id=1)

# 特定の条件で絞り込み（複数条件）
recent = Article.objects.filter(
    is_published=True,
    title__contains="Django"  # タイトルに"Django"を含む
)

filterとgetの違い

  • filter(): 複数件を取得（QuerySetを返す）
  • get(): 1件だけを取得（モデルインスタンスを返す）

Update（更新）- データを変更する

既存のレコードを更新するには、オブジェクトの属性を変更してsave()します。

Python Shell 
# 方法1: 個別のオブジェクトを更新
article = Article.objects.get(id=1)
article.title = "Django完全入門"
article.save()  # 変更を保存

# 方法2: 複数のオブジェクトを一括更新
Article.objects.filter(
    is_published=False
).update(is_published=True)

Delete（削除）- データを削除する

レコードを削除するには、delete()メソッドを使用します。

Python Shell 
# 1件削除
article = Article.objects.get(id=1)
article.delete()

# 条件に一致するものを一括削除
Article.objects.filter(
    is_published=False
).delete()

# すべて削除（注意！）
Article.objects.all().delete()

削除には注意！

delete()メソッドは、一度実行すると元に戻せません。本番環境では特に慎重に使用してください。

6.5 リレーションシップの設定

実際のアプリケーションでは、複数のモデル間に関係性（リレーションシップ）を持たせることが多くあります。Djangoでは、ForeignKey、OneToOneField、ManyToManyFieldの3種類のリレーションシップを定義できます。

1対多（ForeignKey）

最もよく使われるリレーションシップです。1つのモデルが複数の別モデルに関連付けられる場合に使用します。

models.py 
# 著者モデル
class Author(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField()

# 記事モデル（著者に紐づく）
class Article(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    # 外部キー: 1人の著者が複数の記事を持つ
    author = models.ForeignKey(
        Author,
        on_delete=models.CASCADE  # 著者削除時に記事も削除
    )

on_deleteオプション

関連するオブジェクトが削除されたときの動作を指定します。

  • CASCADE 親が削除されると、関連する子も削除
  • PROTECT 子が存在する場合、親の削除を防ぐ
  • SET_NULL 親が削除されると、外部キーをNULLに設定
  • SET_DEFAULT 親が削除されると、デフォルト値を設定
使用例 
# 著者を作成
author = Author.objects.create(
    name="田中太郎",
    email="tanaka@example.com"
)

# 記事を作成（著者に紐づける）
article = Article.objects.create(
    title="Django入門",
    content="...",
    author=author
)

# 著者から記事を取得（逆参照）
articles = author.article_set.all()

1対1（OneToOneField）

2つのモデルが1対1の関係を持つ場合に使用します。ユーザーとプロフィールの関係などが典型例です。

models.py 
from django.contrib.auth.models import User

# ユーザープロフィールモデル
class Profile(models.Model):
    user = models.OneToOneField(
        User,
        on_delete=models.CASCADE
    )
    bio = models.TextField(blank=True)
    birth_date = models.DateField(null=True, blank=True)
    website = models.URLField(blank=True)

多対多（ManyToManyField）

複数のモデルが相互に複数の関係を持つ場合に使用します。記事とタグの関係などが典型例です。

models.py 
# タグモデル
class Tag(models.Model):
    name = models.CharField(max_length=50, unique=True)

# 記事モデル（複数のタグを持つ）
class Article(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    # 多対多: 記事は複数のタグを持ち、タグも複数の記事を持つ
    tags = models.ManyToManyField(Tag)
使用例 
# タグを作成
tag1 = Tag.objects.create(name="Python")
tag2 = Tag.objects.create(name="Django")

# 記事を作成
article = Article.objects.create(
    title="Django入門",
    content="..."
)

# タグを追加
article.tags.add(tag1, tag2)

# タグから記事を取得
articles = tag1.article_set.all()

実習 6-1: ブログモデルを作成してみよう

これまで学んだ知識を使って、ブログアプリケーションのモデルを作成し、基本的なCRUD操作を実践してみましょう。

手順

  1. 1 プロジェクトとblogアプリを作成する
  2. 2 Author、Category、Articleの3つのモデルを定義する
  3. 3 マイグレーションを実行してデータベースに反映する
  4. 4 Django Shellでデータを作成・取得・更新・削除してみる

理解度確認クイズ

  1. Q1. Djangoのモデルにおいて、短いテキスト（例：タイトル）を保存するのに適したフィールドタイプはどれですか？

    回答を表示

    CharFieldです。CharFieldはmax_lengthパラメータで最大文字数を指定する必要があります。長いテキストにはTextFieldを使用します。

  2. Q2. makemigrationsコマンドとmigrateコマンドの違いを説明してください。

    回答を表示

    makemigrationsは、モデルの変更を検出してマイグレーションファイルを生成するコマンドです。migrateは、作成されたマイグレーションファイルを実際にデータベースに適用して、テーブル構造を更新するコマンドです。両方のコマンドを順番に実行することで、モデルの変更がデータベースに反映されます。

  3. Q3. ForeignKeyのon_delete=models.CASCADEは、どのような動作を意味しますか？

    回答を表示

    親となるオブジェクト（参照先）が削除されたとき、それに紐づく子オブジェクト（参照元）も自動的に削除されるという動作です。例えば、著者（Author）が削除されると、その著者が書いたすべての記事（Article）も削除されます。

第6章のまとめ

  • Djangoモデルは、データベースとのやり取りを抽象化するORM機能を提供し、Pythonコードだけでデータベース操作が可能です。
  • フィールドタイプは、CharField、TextField、IntegerField、DateTimeFieldなど、データの種類に応じて適切に選択します。
  • マイグレーションは、makemigrationsでファイルを生成し、migrateでデータベースに反映する2ステップの操作です。
  • CRUD操作は、create()、filter()/get()、update()/save()、delete()メソッドで実行できます。
  • リレーションシップは、ForeignKey（1対多）、OneToOneField（1対1）、ManyToManyField（多対多）で定義します。
