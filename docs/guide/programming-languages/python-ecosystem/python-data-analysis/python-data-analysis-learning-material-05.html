データの読み込みと書き出し - 外部データとの連携

CSV、Excel、JSONなど、様々な形式のファイルからデータを読み込み、加工後に書き出す方法を学びます。実務で扱う多様なデータソースへの対応力を身につけましょう。

この章で学ぶこと

  • read_csv / to_csv を使ったCSVファイルの読み込みと書き出し
  • read_excel / to_excel を使ったExcelファイルの読み書き
  • read_json / to_json を使ったJSONデータの読み書き
  • エンコーディングの指定と文字化け対策
  • 読み込み時の便利なオプション設定（区切り文字・スキップ行・列指定など）

5.1 外部データとの連携とは

実際の業務でデータ分析を行う場合、まず「外部のデータを読み込む」ところから始まります。CSVファイル、Excelシート、JSONデータなど、データはさまざまな形式で存在しています。pandasはこれらを統一された方法で扱える強力な機能を備えています。

また、分析が終わったら結果を再びファイルに書き出して、他の人と共有したりシステムに受け渡したりすることも重要です。この章では「読み込み」と「書き出し」の両方をセットで学びます。

pandasの「変換センター」としての役割

pandasのDataFrameは、さまざまな形式のデータを受け取って統一的に扱える「変換センター」と考えると理解しやすいです。読み込む形式が違っても、DataFrame に変換してしまえば、その後の操作はすべて同じように行えます。

5.2 CSVファイルの読み込みと書き出し

CSVファイル（Comma-Separated Values）は最もよく使われるデータ形式のひとつです。カンマ（,）で区切られたテキストファイルで、Excelでも開けるシンプルな形式です。pandasでは pd.read_csv() で読み込み、df.to_csv() で書き出します。

CSVファイルの読み込み基本

read_csv_basic.py 
import pandas as pd

# 最もシンプルな読み込み（ファイルパスを渡すだけ）
df = pd.read_csv('sales_data.csv')

# 読み込んだデータの確認
print(df.head())   # 先頭5行を表示
print(df.shape)    # (行数, 列数) を表示

CSVファイルの例（sales_data.csv）

日付,商品名,売上,数量
2024-01-01,商品A,5000,10
2024-01-01,商品B,3200,8
2024-01-02,商品A,4500,9
2024-01-02,商品C,8000,16

よく使う読み込みオプション

read_csv_options.py 
import pandas as pd

# エンコーディングを指定（日本語CSVの場合）
df = pd.read_csv('japanese_data.csv', encoding='utf-8')

# Shift-JIS（Windows Excel形式）の場合
df = pd.read_csv('windows_data.csv', encoding='shift-jis')

# 特定の列だけを読み込む
df = pd.read_csv('data.csv', usecols=['日付', '売上'])

# 最初の数行をスキップする（説明文が含まれる場合）
df = pd.read_csv('data.csv', skiprows=2)

# インデックス列を指定する
df = pd.read_csv('data.csv', index_col='日付')

CSVへの書き出し

to_csv_example.py 
import pandas as pd

df = pd.read_csv('sales_data.csv', encoding='utf-8')

# 基本的な書き出し（インデックスなし）
df.to_csv('output.csv', index=False)

# 日本語を含むファイルを書き出す場合
df.to_csv('output_jp.csv', index=False, encoding='utf-8-sig')
# utf-8-sig: BOM付きUTF-8。Excelで開いても文字化けしない

index=False を忘れずに

デフォルトでは index=True のため、DataFrameのインデックス（0, 1, 2...）も列として書き出されます。不要な列が増えてしまうので、通常は index=False を指定することを習慣にしましょう。

read_csv の主なオプション一覧

オプション 型 説明・例
encoding str 'utf-8'、'shift-jis'、'cp932' など文字コードを指定
sep / delimiter str 区切り文字の指定。タブ区切りなら sep='\t'
header int / None ヘッダー行の行番号。header=None でヘッダーなし
usecols list 読み込む列名のリスト。usecols=['A', 'B']
skiprows int / list スキップする行数または行番号のリスト
nrows int 読み込む行数の上限。大容量ファイルの先頭確認に便利
index_col str / int インデックスとして使う列名または列番号
dtype dict 列ごとのデータ型を指定。dtype={'id': str}
na_values list / str 欠損値として扱う値を指定。na_values=['-', 'N/A']

5.3 Excelファイルの読み込みと書き出し

業務データはExcel形式（.xlsx）で管理されていることが非常に多いです。pandasは pd.read_excel() でExcelファイルを直接読み込むことができます。

openpyxlのインストールが必要

Excelファイルを扱うには openpyxl ライブラリが必要です。事前にインストールしておきましょう。

pip install openpyxl
read_excel_example.py 
import pandas as pd

# 基本的な読み込み（最初のシートを読み込む）
df = pd.read_excel('sales.xlsx')

# シート名を指定して読み込む
df_jan = pd.read_excel('sales.xlsx', sheet_name='1月')
df_feb = pd.read_excel('sales.xlsx', sheet_name='2月')

# シート番号で指定する（0始まり）
df_first = pd.read_excel('sales.xlsx', sheet_name=0)

# 全シートを辞書として読み込む
all_sheets = pd.read_excel('sales.xlsx', sheet_name=None)
# all_sheets は {'シート名': DataFrame} の辞書になる
print(all_sheets.keys())  # シート名の一覧
to_excel_example.py 
import pandas as pd

df = pd.read_csv('sales_data.csv', encoding='utf-8')

# 基本的な書き出し（シート名を指定）
df.to_excel('output.xlsx', sheet_name='売上データ', index=False)

# 複数のDataFrameを別シートに書き出す
df1 = df[df['カテゴリ'] == 'A']
df2 = df[df['カテゴリ'] == 'B']

with pd.ExcelWriter('multi_sheet.xlsx') as writer:
    df1.to_excel(writer, sheet_name='カテゴリA', index=False)
    df2.to_excel(writer, sheet_name='カテゴリB', index=False)

ExcelWriterで複数シート出力

pd.ExcelWriter を使うと、1つのExcelファイルに複数のシートをまとめて書き出せます。with文を使うことで、書き込み後にファイルが自動的に閉じられるので安心です。

5.4 JSONデータの読み込みと書き出し

JSON（JavaScript Object Notation）は、WebのAPIからデータを取得するときによく使われる形式です。pandasは pd.read_json() でJSONを読み込み、DataFrameに変換できます。

よく見るJSONの形式

形式1: レコードの配列（最も一般的）

records形式 
[
  {"名前": "田中", "売上": 500},
  {"名前": "鈴木", "売上": 320},
  {"名前": "佐藤", "売上": 780}
]

形式2: 列ごとの配列

columns形式 
{
  "名前": ["田中", "鈴木", "佐藤"],
  "売上": [500, 320, 780]
}
json_example.py 
import pandas as pd

# JSONファイルからの読み込み（レコード配列形式）
df = pd.read_json('data.json')

# orient オプションで形式を明示的に指定
df_records = pd.read_json('records.json', orient='records')
df_columns = pd.read_json('columns.json', orient='columns')

# JSONに書き出す
df.to_json('output.json', orient='records', force_ascii=False)
# force_ascii=False: 日本語をそのまま出力（エスケープしない）
# indent=2: 人間が読みやすい整形形式で出力
df.to_json('pretty_output.json', orient='records',
           force_ascii=False, indent=2)

Web APIのレスポンスを読み込む場合

APIからJSONレスポンスを受け取った場合は、pd.read_json() ではなく pd.DataFrame(data) や pd.json_normalize(data) を使うことが多いです。ネストした複雑なJSONは json_normalize が特に便利です。

5.5 エンコーディング（文字コード）を理解する

日本語のデータを扱うとき、「文字化け」に悩まされることがあります。これはファイルの「文字コード（エンコーディング）」の指定が合っていないために起きます。エンコーディングとは、文字をコンピュータが扱えるバイト列に変換するルールのことです。

UTF-8

世界標準の文字コード。Mac・Linux・Webで広く使われます。

encoding='utf-8'
新規作成ファイルの標準

UTF-8 BOM付き

ExcelでCSVを開いても文字化けしないUTF-8の変種。

encoding='utf-8-sig'
Excel連携時の書き出しに使用

Shift-JIS / CP932

Windows日本語環境の旧来の文字コード。古いシステムで多い。

encoding='shift-jis'
encoding='cp932'
古いWindowsファイルの読み込み
encoding_tips.py 
import pandas as pd

# 文字化けした場合は charset-normalizer で自動判定
# pip install charset-normalizer でインストール
import charset_normalizer

with open('unknown_encoding.csv', 'rb') as f:
    result = charset_normalizer.detect(f.read())
    print(result)  # {'encoding': 'SHIFT_JIS', 'confidence': 0.99}

# 判定されたエンコーディングで読み込む
df = pd.read_csv('unknown_encoding.csv',
                 encoding=result['encoding'])

文字化けしたら errors オプションを試す

エンコーディングが分からない場合や混在している場合、errors='ignore' または errors='replace' を指定すると読み込みエラーを回避できます。ただし文字が欠損する可能性があるため、あくまで応急処置として使いましょう。

5.6 便利な読み込みテクニック

大きなファイルや特殊な形式のファイルを効率よく扱うための実践的なテクニックを紹介します。

実習 5-1: CSVファイルを読み込んで別形式で書き出す

実際にサンプルデータを作成してCSVに保存し、それを読み込んでExcel形式とJSON形式で書き出してみましょう。

手順

  1. 1 Jupyter Notebookを開き、pandasをインポートする
  2. 2 DataFrameを作成し、to_csv('sample.csv', index=False, encoding='utf-8-sig') でCSVに保存する
  3. 3 保存したCSVを pd.read_csv('sample.csv') で再度読み込む
  4. 4 to_excel('sample.xlsx', index=False) でExcel形式に書き出す
  5. 5 to_json('sample.json', orient='records', force_ascii=False, indent=2) でJSON形式に書き出す
practice_5_1.py 
import pandas as pd

# サンプルデータを作成
data = {
    '商品名': ['りんご', 'みかん', 'ぶどう', 'バナナ'],
    '価格': [150, 80, 300, 120],
    '在庫数': [100, 200, 50, 150]
}
df = pd.DataFrame(data)

# CSV に書き出す（Excelで開いても文字化けしない形式）
df.to_csv('sample.csv', index=False, encoding='utf-8-sig')
print("CSV書き出し完了")

# CSV を読み込む
df_loaded = pd.read_csv('sample.csv')
print(df_loaded)

# Excel に書き出す
df_loaded.to_excel('sample.xlsx', sheet_name='商品一覧', index=False)
print("Excel書き出し完了")

# JSON に書き出す
df_loaded.to_json('sample.json', orient='records',
                  force_ascii=False, indent=2)
print("JSON書き出し完了")

成功の目印

「CSV書き出し完了」「Excel書き出し完了」「JSON書き出し完了」が表示され、同じフォルダに 3つのファイルが作成されること。Excelファイルをダブルクリックで開いて日本語が正しく表示されること。

5.7 この章のまとめ

学んだこと

pd.read_csv() と df.to_csv() でCSVファイルの読み書きができる
pd.read_excel() と df.to_excel() でExcelファイルを扱える（openpyxlが必要）
pd.read_json() と df.to_json() でJSONデータを読み書きできる
encoding='utf-8-sig' を使うとExcelで開いても文字化けしない
usecols・skiprows・nrows など読み込みオプションで柔軟にデータを取り込める
to_csv() は index=False を指定しないと不要な行番号が書き出される

理解度確認クイズ

  1. Q1. Excelで開いても文字化けしないようにCSVを書き出すには、encoding にどのような値を指定すればよいですか？

    回答を表示

    encoding='utf-8-sig' を指定します。これはBOM付きUTF-8と呼ばれる形式で、Excelが自動的に日本語と認識できるようになります。通常のUTF-8 (utf-8) ではExcelで開くと文字化けすることがあります。

  2. Q2. pd.read_csv('data.csv', nrows=100) はどのような動作をしますか？

    回答を表示

    CSVファイルの先頭100行だけを読み込みます。大容量ファイルを全部読み込むと時間とメモリがかかりすぎる場合に、まず最初の数行だけを確認するために使います。

  3. Q3. Excelファイルの複数シートにそれぞれDataFrameを書き出すには、どのような方法を使いますか？

    回答を表示

    pd.ExcelWriter を with文 とともに使います。

    with pd.ExcelWriter('output.xlsx') as writer:
    df1.to_excel(writer, sheet_name='シート1', index=False)
    df2.to_excel(writer, sheet_name='シート2', index=False)

よくあるトラブルと解決方法

UnicodeDecodeError: 'utf-8' codec can't decode byte ...

原因: ファイルの文字コードがUTF-8ではない（Shift-JISやCP932の可能性が高い）。

解決方法: encoding='shift-jis' または encoding='cp932' を試してみる。どちらかで読み込めることが多い。

ModuleNotFoundError: Missing optional dependency 'openpyxl'

原因: Excelファイルを扱うのに必要なopenpyxlがインストールされていない。

解決方法: ターミナルで pip install openpyxl を実行する。

CSVに書き出したら列が1つ余分に増えた（Unnamed: 0 が出る）

原因: to_csv() でデフォルトの index=True のままDataFrameのインデックスが列として書き出されてしまった。

解決方法: 書き出しの際に df.to_csv('output.csv', index=False) と指定する。すでに読み込んだファイルに Unnamed: 0 がある場合は df.drop(columns=['Unnamed: 0']) で削除できる。

FileNotFoundError: No such file or directory

原因: 指定したファイルパスにファイルが存在しない。

解決方法: Jupyter Notebookの作業フォルダとファイルの保存場所を確認する。import os; print(os.getcwd()) で現在の作業フォルダを確認できる。ファイルパスは絶対パスで指定するか、ファイルをJupyterと同じフォルダに置く。
