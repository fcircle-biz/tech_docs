16.1 大規模データを効率的に扱う

データ量が増えると、pandasの標準的な読み込みではメモリが不足したり、処理に時間がかかることがあります。ここでは、大規模データを扱うための代表的なテクニックを紹介します。

データ量とメモリの関係

テクニック1: dtypeの最適化

pandasはデフォルトで int64 や float64 を使いますが、値の範囲が小さいなら int32 や float32 に変えることでメモリを節約できます。

dtype_optimization.py 
import pandas as pd

# 読み込み前にdtypeを指定してメモリを節約
df = pd.read_csv('large_data.csv', dtype={
    'age':    'int8',      # 0〜127の整数なので int8 で十分
    'score':  'float32',   # float64 の半分のメモリで済む
    'status': 'category',  # 繰り返す文字列はcategory型が効率的
})

# メモリ使用量を確認
print(df.memory_usage(deep=True).sum() / 1024**2, "MB")

テクニック2: チャンク読み込み

ファイルを一度に全部読み込まず、少量ずつ（チャンク）読み込んで処理することで、大きなファイルもメモリを節約しながら処理できます。

chunk_reading.py 
import pandas as pd

# chunksize で1万行ずつ読み込む
total_sales = 0

for chunk in pd.read_csv('large_sales.csv', chunksize=10000):
    # チャンクごとに集計（全部メモリに乗せない）
    total_sales += chunk['amount'].sum()

print(f"総売上: {total_sales:,}円")

Dask

pandasと同じAPIで大規模データを並列処理できるライブラリ。既存のpandasコードをほぼそのまま活用できます。

pip install dask

Polars

Rustで書かれた超高速なDataFrameライブラリ。pandasより数倍〜数十倍速く動作することがあります。

pip install polars

まず試すべきこと

大規模データに直面したとき、最初は「dtype最適化」と「チャンク読み込み」を試してください。多くの場合、これだけで実用的な速度・メモリに収まります。それでも足りない場合にDaskやPolarsへの移行を検討しましょう。

16.2 Seabornによる統計的可視化

Seabornはmatplotlibを土台にした統計特化の可視化ライブラリです。コードを少なく書いて、見栄えのよいグラフを作れることが特徴です。データの分布や変数間の関係を視覚的に探索する「探索的データ分析（EDA）」で特に威力を発揮します。

比較項目 matplotlib Seaborn
コード量 多い（細かい設定が必要） 少ない（デフォルトが綺麗）
DataFrame連携 手動で列を指定 列名を直接指定できる
統計グラフ 自分で計算が必要 自動で信頼区間なども表示
カスタマイズ 自由度が高い matplotlibと組み合わせ可能
seaborn_intro.py 
import seaborn as sns
import matplotlib.pyplot as plt

# Seaborn 組み込みのサンプルデータを使用
tips = sns.load_dataset('tips')   # レストランのチップデータ

# 箱ひげ図: 曜日別・チップ金額の分布
sns.boxplot(data=tips, x='day', y='tip', hue='sex')
plt.title('曜日別チップ分布（性別比較）')
plt.show()

# ヒートマップ: 相関行列の可視化
corr = tips.select_dtypes(include='number').corr()
sns.heatmap(corr, annot=True, cmap='coolwarm', fmt='.2f')
plt.title('数値列の相関行列')
plt.show()

pairplot

全数値列の組み合わせの散布図を一括生成。変数間の関係性を素早く俯瞰できる。

sns.pairplot(df)

violinplot

箱ひげ図にデータの密度分布を重ねた図。分布の形状まで把握できる。

sns.violinplot(data=df)

heatmap

行列データを色で表現。相関行列やピボットテーブルの可視化に最適。

sns.heatmap(corr, annot=True)
seaborn_pairplot.py 
import seaborn as sns
import matplotlib.pyplot as plt

# アイリスデータセット（花のサイズ計測データ）
iris = sns.load_dataset('iris')

# pairplot: 数値列の全組み合わせを一括可視化
# hue='species' で品種ごとに色分け
sns.pairplot(iris, hue='species', corner=True)
plt.suptitle('アイリスデータ - 変数間の関係', y=1.02)
plt.tight_layout()
plt.show()

Seabornのインストール

pip install seaborn でインストールできます。matplotlibより少ないコードでプロフェッショナルなグラフを生成できるため、EDA（探索的データ分析）では積極的に活用しましょう。

16.3 Web APIとスクレイピングによるデータ収集

Web APIを使ったデータ取得

Web APIとは、Webサービスがデータを提供するための窓口です。多くのサービス（気象データ・株価・SNSなど）がAPIを公開しており、リクエストを送るとJSONやCSV形式でデータが返ってきます。

web_api_example.py 
import requests
import pandas as pd

# Open-Meteo（無料の天気APIの例）
# 実際のAPIエンドポイントはサービスごとに異なります
url = "https://api.example.com/weather"
params = {
    'latitude':  35.68,   # 緯度（東京）
    'longitude': 139.69,  # 経度（東京）
    'daily': 'temperature_2m_max',
}

response = requests.get(url, params=params)  # GETリクエスト送信
data = response.json()                       # JSONをPython辞書に変換

# DataFrameに変換して分析
df = pd.DataFrame(data['daily'])
print(df.head())

Webスクレイピングの概要

スクレイピングは、Webページの HTML を解析してデータを取得する手法です。APIが提供されていない場合に使いますが、利用規約の確認とサーバーへの過度な負荷をかけないことが最低限のマナーです。

スクレイピングの注意事項

スクレイピングを行う際は必ずサイトの利用規約と robots.txt を確認してください。禁止されているサイトへのスクレイピングは法的問題になる場合があります。また、短時間に大量のリクエストを送るとサーバーに負荷をかけるため、適切な待機時間を設けましょう。

scraping_example.py 
import requests
from bs4 import BeautifulSoup  # pip install beautifulsoup4
import pandas as pd
import time

url = "https://example.com/data-page"  # 利用規約確認済みのサイト

response = requests.get(url)
soup = BeautifulSoup(response.text, 'html.parser')

# テーブル要素を取得してDataFrameに変換
table = soup.find('table')
df = pd.read_html(str(table))[0]  # HTMLテーブルを直接読み込み
print(df.head())

time.sleep(1)  # サーバーへの負荷を避けるため1秒待機

実習 16-1: Seabornで探索的データ分析を行う

Seabornに組み込まれている「tips」データセットを使って、チップ金額に影響する要因を探索的に分析しましょう。

手順

  1. 1 pip install seaborn でSeabornをインストールする
  2. 2 sns.load_dataset('tips') でデータを読み込み、info() と describe() で概要を確認する
  3. 3 pairplot でチップ金額と他の数値列の関係を可視化する
  4. 4 boxplot を使い、曜日・時間帯・喫煙有無によるチップ分布の違いを確認する

成功の目印

pairplotと複数のboxplotが表示され、「チップ金額に最も影響する要因」について自分なりの仮説が立てられる

理解度確認クイズ

  1. Q1. pandasで大規模なCSVを読み込む際、メモリを節約するための代表的な手法を2つ答えてください。

    回答を表示
    • dtype最適化: read_csv(dtype={...}) で列のデータ型を明示的に指定し、メモリ消費を削減する（例: int64 → int8、category型の活用）。
    • チャンク読み込み: read_csv(chunksize=10000) で分割して読み込み、集計しながら処理する。

  2. Q2. Seabornとmatplotlibはどのような関係にありますか？また、Seabornが得意なグラフの例を1つ挙げてください。

    回答を表示

    Seabornはmatplotlibをベースにしたライブラリです。matplotlibの機能を内部で使いながら、統計グラフを少ないコードで美しく描画できます。得意なグラフの例: pairplot（複数変数の関係を一括表示）、heatmap（相関行列の可視化）、boxplot / violinplot（カテゴリ別の分布比較）など。

  3. Q3. Webスクレイピングを行う際に最初に確認すべきことを答えてください。

    回答を表示

    対象サイトの利用規約と robots.txt を確認することです。スクレイピングを禁止しているサイトへの実施は法的問題に発展する場合があります。また、サーバーに過度な負荷をかけないよう、リクエスト間に適切な待機時間（sleep）を設けることも重要です。

よくあるトラブルと解決方法

MemoryError: メモリ不足でCSVが読み込めない

原因: CSVが大きすぎてRAMに収まらない。

解決方法: dtype オプションで列の型を最適化するか、chunksize オプションで分割読み込みを行う。それでも足りない場合はDaskやPolarsへの移行を検討する。

requests.ConnectionError: URLへの接続が失敗する

原因: ネットワーク接続エラー、APIのURLが誤っている、またはAPIキーが必要。

解決方法: URLをブラウザで直接開いて確認する。APIキーが必要なサービスは headers や params に認証情報を含める。

Seabornグラフの日本語が文字化けする

原因: matplotlibのデフォルトフォントが日本語に対応していない。

解決方法: import japanize_matplotlib（pip install japanize-matplotlib）を使うか、plt.rcParams['font.family'] = 'IPAexGothic' などで日本語フォントを設定する。
