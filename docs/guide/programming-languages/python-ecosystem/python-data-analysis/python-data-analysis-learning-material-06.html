データの確認と基本統計 - データを俯瞰する力

読み込んだデータの全体像を素早く把握するためのメソッドを学びましょう。データの形状・型・欠損値を確認し、基本統計量を算出する力は、あらゆるデータ分析の出発点となる重要スキルです。

この章で学ぶこと

  • head / tail / shape / info / describe でデータ全体を素早く把握する方法
  • 各列のデータ型（dtype）を確認し、意図した型かどうか検証する
  • 欠損値（NaN）の存在を検出し、どの列・何件に欠損があるかを把握する
  • 平均・中央値・標準偏差など基本統計量の意味と読み方を理解する
  • value_counts を使ってカテゴリ列の出現頻度を集計する

6.1 なぜデータを「まず確認する」のか

データ分析の現場では、外部から受け取ったデータをいきなり集計・可視化することは禁物です。行数・列数・データ型・欠損値の有無を最初に確認しないまま分析を進めると、間違った結論を導く可能性があります。

この「データを俯瞰する」フェーズは、医師が患者を診察するときの「問診・視診」に相当します。治療（分析）を始める前に、まず患者（データ）の全体状態を把握するのです。

データ確認の標準フロー

flowchart TD A["データ読み込み\n（read_csv など）"] --> B["形状確認\nshape / info"] B --> C["先頭・末尾確認\nhead / tail"] C --> D["データ型確認\ndtypes"] D --> E["欠損値確認\nisnull / info"] E --> F["基本統計量確認\ndescribe"] F --> G["カテゴリ集計\nvalue_counts"] G --> H["前処理・分析へ進む"] style A fill:#dbeafe,stroke:#3b82f6,color:#1e40af style H fill:#d1fae5,stroke:#10b981,color:#064e3b

EDA（探索的データ分析）とは

データの全体像を把握する作業を EDA（Exploratory Data Analysis：探索的データ分析） と呼びます。分析の仮説を立てる前に、データの特徴・分布・異常値・欠損などを「探索」するフェーズです。本章で学ぶメソッドは、EDAの基本ツールセットです。

6.2 サンプルデータの準備

この章では、架空の「売上データ」を使って学習します。実際の業務データをイメージした、わざと欠損値や複数のデータ型が混在するデータを用意します。

import pandas as pd
import numpy as np

# 架空の売上データを作成
data = {

    '商品名':   ['ノートPC', 'マウス', 'キーボード', 'ノートPC',
                 'モニター', 'マウス', 'キーボード', 'モニター'],
    'カテゴリ': ['PC本体', '周辺機器', '周辺機器', 'PC本体',
                 '周辺機器', '周辺機器', '周辺機器', '周辺機器'],
    '売上金額': [128000, 3500, 8800, 142000, 45000, None, 9200, 52000],
    '数量':     [1, 2, 1, 1, 1, 3, 2, 1],
    '担当者':   ['田中', '鈴木', '田中', '佐藤', '鈴木', '田中', None, '佐藤'],
}

df = pd.DataFrame(data)
print(df)

NaN とは

NaN は "Not a Number" の略で、pandasが「値がない（欠損している）」ことを表すために使う特別な値です。数値列に欠損があると NaN、文字列列に欠損があると None または NaN が表示されます。

6.3 head / tail でデータを目視確認する

大きなデータを全件表示するのは非効率です。head() と tail() を使うと、先頭・末尾の数件だけを確認できます。

head_tail.ipynb 
# 先頭3行を表示（デフォルトは5行）
df.head(3)

# 末尾3行を表示
df.tail(3)

# 行数・列数を確認（行数, 列数）のタプル
print(df.shape)       # (8, 6)

# 全行数・全列数を個別に取得
print(f"行数: {df.shape[0]}")   # 8
print(f"列数: {df.shape[1]}")   # 6

head() - 先頭を確認

列名・データの種類・全体的な形式を把握するために使います。読み込み直後に必ず実行する習慣をつけましょう。

tail() - 末尾を確認

ファイルの末尾に不要な行（フッター・空行）が含まれていないかを確認するのに役立ちます。

shape は属性、head() はメソッド

df.shape は括弧なし（属性）、df.head() は括弧あり（メソッド）です。pandasではこの2種類が混在しているので注意しましょう。

6.4 info() でデータ型と欠損値を一覧確認する

info() はDataFrameの構造を一覧表示する最も便利なメソッドです。行数・列名・各列の欠損なし件数・データ型・メモリ使用量を一度に確認できます。

info_check.ipynb 
# DataFrameの構造情報を表示
df.info()

出力例

<class 'pandas.core.frame.DataFrame'>
RangeIndex: 8 entries, 0 to 7
Data columns (total 6 columns):
 #   Column  Non-Null Count  Dtype
---  ------  --------------  -----
 0   日付      8 non-null      object
 1   商品名     8 non-null      object
 2   カテゴリ    8 non-null      object
 3   売上金額    7 non-null      float64
 4   数量      8 non-null      int64
 5   担当者     7 non-null      object
dtypes: float64(1), int64(1), object(4)
memory usage: 512.0+ bytes

info() 出力の読み方

項目 サンプルの値 意味・注目点
RangeIndex: 8 entries 8行 データの総行数
Non-Null Count 7 non-null 欠損なし件数。8行中7なら1件欠損
Dtype: object 文字列扱い 日付列が object なら型変換が必要かも
Dtype: float64 浮動小数点 欠損があると int が float に昇格する

欠損値の発見法：Non-Null Count を確認

全行数（RangeIndex の件数）と各列の Non-Null Count を比較します。数が少ない列には欠損値があります。「売上金額: 7 non-null」＝8行中1件欠損という読み方です。

6.5 欠損値（NaN）を詳しく調べる

info() で欠損の存在を確認したら、次は isnull() や isna() を使ってより詳しく調べます。

missing_values.ipynb 
# 各セルが欠損かどうかを True/False で表示
df.isnull()

# 列ごとの欠損件数を合計
print(df.isnull().sum())

# 欠損値の割合（%）を確認
print(df.isnull().mean() * 100)

欠損件数の出力例

日付      0
商品名     0
カテゴリ    0
売上金額    1
数量      0
担当者     1
dtype: int64

欠損値を確認するメソッド一覧

df.isnull()

各セルが欠損かどうかを True/False の DataFrame で返す

df.isnull().sum()

列ごとの欠損件数を整数で返す（最もよく使う）

df.isnull().mean()

列ごとの欠損割合（0〜1）を返す。×100 で % に換算

df.notnull()

isnull() の逆。欠損でないセルを True で返す

isnull() と isna() は同じ

pandas では isnull() と isna() はまったく同じ動作をします。どちらを使っても構いませんが、コードの中で統一することをお勧めします。

6.6 describe() で基本統計量を確認する

describe() は、数値列に対して基本統計量を一覧表示します。データの「形」を数字で把握するための最重要メソッドです。

describe_check.ipynb 
# 数値列の基本統計量を表示
print(df.describe())

# 文字列列も含めたい場合
print(df.describe(include='all'))

出力例

         売上金額       数量
count      7.000000  8.000000
mean   55500.000000  1.500000
std    51985.097...  0.755929
min     3500.000000  1.000000
25%     9000.000000  1.000000
50%    45000.000000  1.000000
75%    90000.000000  2.000000
max   142000.000000  3.000000

describe() の各統計量の意味

統計量 意味 読み取れること
count 欠損を除いた件数 8行中7件 = 1件欠損と分かる
mean 平均値 全体の「重心」。外れ値に敏感
std 標準偏差 データのばらつき度合い。大きいほどバラバラ
min 最小値 異常に小さな値がないか確認
25% / 50% / 75% 四分位数 50%が中央値。データの分布形状を把握
max 最大値 異常に大きな値（外れ値）がないか確認

平均値 vs 中央値：どちらを信じるべきか

flowchart LR A["データに外れ値あり\n例: 1000円×9件 + 100万円×1件"] --> B{"平均 vs 中央値"} B -->|"mean（平均）"| C["約10万円\n外れ値に引っ張られる"] B -->|"50%（中央値）"| D["1000円\n外れ値の影響を受けない"] C --> E["外れ値があると\n実態を反映しない"] D --> F["外れ値があっても\n「典型的な値」を示す"] style C fill:#fee2e2,stroke:#ef4444,color:#7f1d1d style D fill:#d1fae5,stroke:#10b981,color:#064e3b

平均と中央値が大きく離れているときは、データに外れ値や偏りがある可能性があります。

statistics.ipynb 
# 個別に統計量を取得する方法
col = df['売上金額']

print(f"平均値:   {col.mean():.0f}円")        # 平均
print(f"中央値:   {col.median():.0f}円")      # 中央値（50%）
print(f"標準偏差: {col.std():.0f}円")         # ばらつき
print(f"最小値:   {col.min():.0f}円")         # 最小
print(f"最大値:   {col.max():.0f}円")         # 最大
print(f"合計:     {col.sum():.0f}円")         # 合計

6.7 dtypes でデータ型を確認する

dtypes で各列のデータ型を確認できます。データ型が意図した通りでない場合（日付が文字列として読み込まれたなど）は、後の分析でエラーが起きる原因になります。

dtypes_check.ipynb 
# 全列のデータ型を確認
print(df.dtypes)

# 特定の列のデータ型だけ確認
print(df['日付'].dtype)      # object（文字列）

# 列ごとにデータ型で絞り込む
numeric_cols  = df.select_dtypes(include='number').columns
string_cols   = df.select_dtypes(include='object').columns
print("数値列:", list(numeric_cols))
print("文字列列:", list(string_cols))

数値型

  • int64: 整数（欠損なし）
  • float64: 浮動小数点・欠損あり整数

文字列型

  • object: 文字列・混在データ
  • string: pandas 専用文字列型

日付型

  • datetime64: 日付時刻型
  • 文字列で読まれた場合は変換が必要

6.8 value_counts でカテゴリ列を集計する

文字列（カテゴリ）列に対しては、value_counts() を使って各値の出現回数を確認します。どのカテゴリが多いか、データが偏っていないかを素早く把握できます。

value_counts.ipynb 
# カテゴリ列の出現頻度を降順で表示
print(df['カテゴリ'].value_counts())

# 割合（%）で表示
print(df['カテゴリ'].value_counts(normalize=True) * 100)

# 担当者ごとの件数（欠損値も含める）
print(df['担当者'].value_counts(dropna=False))

出力例（カテゴリ列）

カテゴリ
周辺機器    6
PC本体     2
Name: count, dtype: int64

---（割合）---
カテゴリ
周辺機器    75.0
PC本体     25.0
Name: proportion, dtype: float64

dropna=False で欠損値の個数も確認できる

デフォルトでは欠損値（NaN）はカウントから除外されます。dropna=False を指定すると、欠損値も NaN として件数に含まれます。欠損の存在を見落とさないために活用しましょう。

6.9 まとめ：初回確認チェックリスト

新しいデータを受け取ったとき、分析を始める前に必ず実行する「初回確認チェックリスト」をまとめます。このパターンを習慣化することで、データの問題を早期に発見できます。

initial_check.ipynb 
import pandas as pd

df = pd.read_csv('data.csv')   # 任意のデータを読み込む

# ① 形状確認（行数・列数）
print("▼ 形状")
print(df.shape)

# ② 先頭5行で内容確認
print("\n▼ 先頭5行")
print(df.head())

# ③ データ型・欠損を一覧確認
print("\n▼ 構造情報")
df.info()

# ④ 欠損件数の確認
print("\n▼ 欠損件数")
print(df.isnull().sum())

# ⑤ 数値列の統計量
print("\n▼ 基本統計量")
print(df.describe())

# ⑥ カテゴリ列の出現頻度
for col in df.select_dtypes(include='object').columns:
    print(f"\n▼ {col} の頻度")
    print(df[col].value_counts())

データ確認で見るべき5つのポイント

行数・列数は期待通りか（shape）
データ型は意図した型か（dtypes / info）
欠損値はどの列に何件あるか（isnull().sum()）
数値の範囲に異常値がないか（describe の min/max）
カテゴリ値の種類と分布は想定通りか（value_counts）

実習 6-1: 架空の売上データで初回確認を実施する

セクション 6.2 で作成したサンプルデータに対して、学んだメソッドをすべて実行し、データの問題点を洗い出しましょう。

手順

  1. 1 セクション 6.2 のコードを実行してサンプルデータ df を作成する
  2. 2 df.shape、df.head()、df.info() を順番に実行する
  3. 3 df.isnull().sum() を実行して欠損列と件数を確認する
  4. 4 df.describe() を実行して売上金額の平均・最大・最小を読み取る
  5. 5 df['商品名'].value_counts() でどの商品が何回登場するかを確認する

成功の目印

欠損が「売上金額: 1件」「担当者: 1件」と分かり、日付列の dtype が object（文字列）であることが確認できる

理解度確認クイズ

  1. Q1. DataFrameの行数・列数を確認するには何を使いますか？

    回答を表示

    df.shape を使います。(行数, 列数) のタプルを返します。括弧なしの属性なので df.shape であることに注意しましょう（df.shape() は誤りです）。

  2. Q2. df.info() の出力で「欠損値」を確認するにはどこを見ますか？

    回答を表示

    Non-Null Count（欠損なし件数）の列を見ます。全行数（RangeIndex の件数）と比較して、数が少ない列に欠損があります。たとえば全8行で「7 non-null」と表示されていれば、1件の欠損があることを示します。

  3. Q3. describe() で表示される「50%」は何を意味しますか？

    回答を表示

    中央値（メジアン）を意味します。データを小さい順に並べたときに、ちょうど真ん中に来る値です。平均値（mean）と中央値（50%）が大きく乖離している場合は、データに外れ値（極端に大きい・小さい値）が含まれている可能性があります。

よくあるトラブルと解決方法

数値のはずなのに dtype が object になっている

原因: CSVの中に「¥1,000」のようなカンマ付き数値や「-」「N/A」などの文字が混在していると、pandasが文字列（object）として読み込みます。

解決方法: pd.to_numeric(df['列名'], errors='coerce') で数値に変換する。変換できない値は NaN になります。

整数列なのに float64 になっている

原因: pandasの整数型（int64）は欠損値（NaN）を扱えないため、欠損が1件でもあると自動的に float64 に変換されます。

解決方法: 欠損を fillna() で埋めてから astype(int) で変換する。または pandas の nullable integer 型 Int64（大文字I）を使う。

describe() に特定の列が表示されない

原因: describe() はデフォルトで数値列のみを対象にします。文字列列（object）は除外されます。

解決方法: df.describe(include='all') で全列を含める。または文字列列には value_counts() を使う。
