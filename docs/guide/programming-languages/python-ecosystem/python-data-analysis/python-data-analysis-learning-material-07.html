データの選択とフィルタリング - 必要なデータを取り出す

条件を指定してDataFrameから必要な行・列を取り出す方法を学びます。複数条件の組み合わせ、文字列検索、特定の値による絞り込みなど、データ分析で毎日使うフィルタリング技術を徹底的に習得します。

この章で学ぶこと

  • 条件式を使ったブールインデックスによるフィルタリングの仕組み
  • AND（＆）・OR（|）・NOT（~）を使った複数条件の組み合わせ方
  • isin / between / str.contains などの便利メソッド
  • queryメソッドを使ったSQL風の直感的な絞り込み
  • sort_values / nlargest / nsmallest によるソートと順位付け

7.1 フィルタリングとは何か

データ分析では、数千〜数百万行のデータから「特定の条件を満たす行だけ」を取り出す作業が非常に頻繁に発生します。これをフィルタリングと呼びます。

たとえば「売上が100万円以上の取引だけ見たい」「東京の顧客だけ分析したい」「先月のデータだけ抽出したい」といった場面で使います。Excelの「フィルター」機能に相当しますが、pandasではコードで記述することで再現性と自動化が実現できます。

フィルタリングの概念

この章で使用するサンプルデータ

以降の例では、架空の商品売上データ（商品名・カテゴリ・売上・在庫・地域）を使って説明します。コードを実際に動かすときは、各自のJupyter Notebookで同じデータを作成してから試してください。

サンプルデータの作成 
import pandas as pd
import numpy as np

# この章で使うサンプルデータを作成
data = {
    '商品名': ['りんご', 'バナナ', 'みかん', 'ぶどう', 'いちご',
               'メロン', 'スイカ', 'もも', 'なし', 'キウイ'],
    'カテゴリ': ['果物A', '果物B', '果物A', '果物B', '果物A',
                '高級果物', '高級果物', '高級果物', '果物A', '果物B'],
    '売上': [120, 85, 95, 210, 180, 380, 450, 290, 75, 110],
    '在庫': [50, 120, 80, 30, 45, 15, 10, 25, 90, 70],
    '地域': ['東京', '大阪', '東京', '名古屋', '大阪',
             '東京', '名古屋', '大阪', '東京', '大阪']
}
df = pd.DataFrame(data)
print(df)

7.2 ブールインデックスによる基本フィルタリング

pandasのフィルタリングの核心はブールインデックスという仕組みです。条件式を評価すると True / False の列（Series）が得られ、それをDataFrameに渡すと True の行だけが抽出されます。

ブールインデックスの仕組み

STEP 1: 条件式を書く

df['売上'] >= 200

STEP 2: True/False の列ができる

0    False
1    False
2    False
3    True
4    False
...

STEP 3: True の行だけ抽出

df[条件式]
基本的なフィルタリング 
# 売上が200以上の行を抽出
high_sales = df[df['売上'] >= 200]
print(high_sales)

# 東京の行だけ抽出
tokyo = df[df['地域'] == '東京']
print(tokyo)

# 在庫が30以下の行（在庫切れ間近）
low_stock = df[df['在庫'] <= 30]
print(low_stock)

売上200以上の出力例

   商品名   カテゴリ  売上  在庫   地域
3   ぶどう  果物B   210   30  名古屋
4  いちご  果物A   180   45   大阪
5  メロン  高級果物  380   15   東京
6  スイカ  高級果物  450   10  名古屋
7   もも  高級果物  290   25   大阪

比較演算子のまとめ

使える比較演算子: ==（等しい）, !=（等しくない）, >=（以上）, <=（以下）, >（より大きい）, <（より小さい）

7.3 複数条件の組み合わせ（AND / OR / NOT）

実務では「売上が高くて、かつ在庫が少ない商品」のように、複数の条件を組み合わせる必要があります。pandasでは &（AND）・|（OR）・~（NOT）を使います。

演算子 意味 使い方 Pythonの and/or は使えない
& AND（両方が True） (条件A) & (条件B) and キーワードは使えない
| OR（どちらかが True） (条件A) | (条件B) or キーワードは使えない
~ NOT（条件の反転） ~(条件A) not キーワードは使えない

各条件を必ず括弧 () で囲む

演算子の優先順位の問題で、各条件を括弧で囲まないとエラーになります。
NG: df[df['売上'] >= 200 & df['在庫'] <= 30]
OK: df[(df['売上'] >= 200) & (df['在庫'] <= 30)]

複数条件のフィルタリング 
# AND: 売上200以上 かつ 在庫30以下（要注目商品）
result_and = df[(df['売上'] >= 200) & (df['在庫'] <= 30)]
print("AND条件（売上高 かつ 在庫少）:")
print(result_and)

# OR: 東京 または 大阪 の商品
result_or = df[(df['地域'] == '東京') | (df['地域'] == '大阪')]
print("\nOR条件（東京 または 大阪）:")
print(result_or)

# NOT: 高級果物以外のカテゴリ
result_not = df[~(df['カテゴリ'] == '高級果物')]
print("\nNOT条件（高級果物以外）:")
print(result_not)

3つ以上の条件を組み合わせる

条件が増えてもパターンは同じです。複雑になってきたら変数に条件を分けて書くと読みやすくなります。

3条件の組み合わせ（読みやすい書き方） 
# 条件を変数に分けると読みやすくなる
cond_sales   = df['売上'] >= 150         # 売上150以上
cond_stock   = df['在庫'] <= 50          # 在庫50以下
cond_region  = df['地域'] == '東京'      # 東京のみ

# 3条件をANDで組み合わせ
result = df[cond_sales & cond_stock & cond_region]
print(result)

7.4 便利なフィルタリングメソッド

pandasには、よく使うフィルタリングパターンをシンプルに書けるメソッドが用意されています。

isin() リストの中のどれかに一致する行を取得

「東京または大阪または名古屋」のように複数の値のいずれかに一致するかを調べます。OR条件を複数並べる代わりに使えます。

isin() の使い方 
# 東京・大阪のどちらか（OR条件の簡潔な書き方）
target_regions = ['東京', '大阪']
result = df[df['地域'].isin(target_regions)]
print(result)

# 特定カテゴリ以外を除外（~ で反転）
exclude = ['高級果物']
result_ex = df[~df['カテゴリ'].isin(exclude)]
print(result_ex)

between() 範囲内（以上・以下）の行を取得

数値の範囲条件を簡潔に書けます。両端の値も含まれます（閉区間）。

between() の使い方 
# 売上100以上300以下の商品
result = df[df['売上'].between(100, 300)]
print(result)

# 上と同じ意味（betweenを使わない書き方）
result2 = df[(df['売上'] >= 100) & (df['売上'] <= 300)]

str.contains() 文字列が含まれる行を取得

文字列列に対して部分一致検索ができます。正規表現も使えます。大文字小文字を区別しない場合は case=False を指定します。

str.contains() の使い方 
# 商品名に「も」が含まれるもの
result = df[df['商品名'].str.contains('も')]
print(result)

# カテゴリに「果物」が含まれるもの
fruit = df[df['カテゴリ'].str.contains('果物')]
print(fruit)

# na=False: NaN（欠損値）があってもエラーにしない
result2 = df[df['商品名'].str.contains('ん', na=False)]

strアクセサの他のメソッド

str.startswith('東')（先頭が一致）, str.endswith('京')（末尾が一致）, str.len()（文字数）など、多くのメソッドが使えます。

7.5 queryメソッド - SQL風の直感的な絞り込み

query() メソッドを使うと、SQLのWHERE句のように文字列で条件を記述できます。コードが読みやすくなり、特に条件が複雑なときに重宝します。

query() の使い方 
# 基本的な使い方（文字列で条件を書く）
result = df.query('売上 >= 200')
print(result)

# AND条件（and または & が使える）
result2 = df.query('売上 >= 200 and 在庫 <= 30')

# OR条件
result3 = df.query('地域 == "東京" or 地域 == "大阪"')

# in を使ったリスト一致（isin と同等）
result4 = df.query('地域 in ["東京", "大阪"]')

# 変数を埋め込む場合は @ を使う
threshold = 150
result5 = df.query('売上 >= @threshold')

ブールインデックス vs query() の比較

ブールインデックス

df[(df['売上'] >= 200) & (df['在庫'] <= 30)]
  • ・より汎用的で柔軟
  • ・複雑な条件にも対応
  • ・括弧の記述が煩雑になりがち

query() メソッド

df.query('売上 >= 200 and 在庫 <= 30')
  • ・コードが読みやすい
  • ・SQLになじみがある人に直感的
  • ・列名にスペースがあると使いにくい

7.6 データのソートと順位付け

フィルタリングと合わせてよく使うのがソート（並び替え）です。「売上の高い順に並べる」「名前のアルファベット順に並べる」といった操作ができます。

sort_values() - 値でソート

sort_values() の使い方 
# 売上の昇順（小さい順）
df_asc = df.sort_values('売上')
print(df_asc)

# 売上の降順（大きい順）
df_desc = df.sort_values('売上', ascending=False)
print(df_desc)

# 複数列でソート（地域の昇順 → 売上の降順）
df_multi = df.sort_values(['地域', '売上'], ascending=[True, False])
print(df_multi)

nlargest() / nsmallest() - 上位N件・下位N件の取得

「売上トップ3」「在庫が最も少ない5商品」のような「上位・下位N件」を素早く取得できるメソッドです。

nlargest() / nsmallest() の使い方 
# 売上トップ3の商品
top3 = df.nlargest(3, '売上')
print("売上トップ3:")
print(top3)

# 在庫が最も少ない3商品（在庫切れリスクが高い）
low3 = df.nsmallest(3, '在庫')
print("\n在庫最少3商品:")
print(low3)

売上トップ3の出力例

   商品名   カテゴリ  売上  在庫   地域
6  スイカ  高級果物  450   10  名古屋
5  メロン  高級果物  380   15   東京
7   もも  高級果物  290   25   大阪

rank() - 順位をつける

rank() で順位列を追加 
# 売上の順位列を追加（1位が最高売上）
df['売上順位'] = df['売上'].rank(ascending=False).astype(int)
print(df[['商品名', '売上', '売上順位']].sort_values('売上順位'))

7.7 フィルタリングとソートの組み合わせ

実際のデータ分析では、フィルタリングとソートを組み合わせて使うことがほとんどです。メソッドを連続して呼び出すメソッドチェーンという書き方で、スッキリと記述できます。

フィルタリング + ソートの組み合わせ 
# 東京の商品を売上降順で取得
result = (
    df[df['地域'] == '東京']   # フィルタリング
    .sort_values('売上', ascending=False)  # ソート
    .reset_index(drop=True)    # インデックスをリセット
)
print(result)

# 売上100〜300の商品を在庫少ない順に並べ、上位3件を取得
result2 = (
    df[df['売上'].between(100, 300)]
    .sort_values('在庫')
    .head(3)
)
print(result2)

reset_index(drop=True) とは

フィルタリングすると元のインデックス番号（0, 3, 5...など）がそのまま残ります。reset_index(drop=True) を使うと 0, 1, 2... のきれいなインデックスに振り直せます。drop=True を付けないと、元のインデックスが列として追加されてしまうので注意してください。

7.9 この章のまとめ

学んだこと

ブールインデックスは条件式がTrue/Falseの列を返し、Trueの行だけが抽出される仕組み
複数条件は &（AND）, |（OR）, ~（NOT）で組み合わせ、各条件は必ず ( ) で囲む
isin() はリスト一致、between() は範囲、str.contains() は文字列検索に使う
query() はSQL風の文字列で条件を記述でき、コードの可読性が上がる
sort_values() で並び替え、nlargest()/nsmallest() で上位・下位N件を素早く取得できる
フィルタリング後は reset_index(drop=True) でインデックスを整えるのが良い習慣

理解度確認クイズ

  1. Q1. df[(df['年齢'] >= 20) and (df['年齢'] <= 30)] と書いたらエラーになりました。なぜでしょうか？正しい書き方も示してください。

    回答を表示

    pandasのブールインデックスでは Python の and キーワードは使えません。&（アンパサンド）を使う必要があります。また、各条件を括弧で囲むことも必須です。

    正しい書き方:

    df[(df['年齢'] >= 20) & (df['年齢'] <= 30)]

    または between() を使う:

    df[df['年齢'].between(20, 30)]

  2. Q2. 「都市」列が「東京」「大阪」「京都」のいずれかである行を抽出するために、OR条件を使わず1行で書く方法を示してください。

    回答を表示

    isin() メソッドを使います。
    df[df['都市'].isin(['東京', '大阪', '京都'])]
    OR条件で書くよりも簡潔で、値が増えてもリストに追加するだけで対応できます。

  3. Q3. sort_values() で売上の降順に並べた後、上位5件だけを取得する方法を2通り答えてください。

    回答を表示

    方法1: sort_values() + head()

    df.sort_values('売上', ascending=False).head(5)

    方法2: nlargest() を使う（より簡潔）

    df.nlargest(5, '売上')

よくあるエラーと解決方法

ValueError: The truth value of a Series is ambiguous

原因: 複数条件で and / or キーワードを使っている。

解決方法: and を & に、or を | に変更し、各条件を括弧 () で囲む。

フィルタリング後に元のインデックス番号が残ってしまう

原因: フィルタリングは元のインデックスを保持します（意図した動作）。

解決方法: .reset_index(drop=True) を末尾につける。drop=True を付けないと元のインデックスが列として残るので注意。

str.contains() で TypeError: expected string, got float

原因: 対象列に NaN（欠損値）が含まれており、文字列として処理できない。

解決方法: str.contains('キーワード', na=False) のように na=False を付ける。NaN はFalseとして扱われる。

SettingWithCopyWarning が表示される

原因: フィルタリングで得た部分DataFrameに対して値を変更しようとしている。元のDataFrameに反映されないことがある。

解決方法: フィルタリング後に .copy() を付けて明示的にコピーを作成してから変更する。例: subset = df[df['売上'] > 100].copy()
