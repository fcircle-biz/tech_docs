グループ化と集計 - データを要約する力

groupbyを使ったデータの集計方法を学びます。カテゴリ別の合計・平均・件数の算出、複数の集計関数の同時適用など、データを要約するための強力な手法を習得します。

この章で学ぶこと

  • groupbyの基本的な仕組みと「分割 → 適用 → 結合」の考え方
  • 集計関数（sum, mean, count, agg）を使ったカテゴリ別の集計
  • 複数列を使ったグループ化と階層的な集計
  • pivot_tableとcrosstabを使ったクロス集計

10.1 groupbyとは - 「分割・適用・結合」の考え方

データ分析では「カテゴリ別に集計したい」という場面が非常に多く出てきます。たとえば「商品カテゴリ別の売上合計」「都道府県別の平均年収」「月別の注文件数」などです。このような集計を行うのが groupby です。

groupbyの動作は「分割（Split）→ 適用（Apply）→ 結合（Combine）」の3ステップで考えると理解しやすいです。

Split-Apply-Combine（分割・適用・結合）の仕組み

flowchart LR A["元のDataFrame\n（全データ）"] --> B{"Split\nグループキーで分割"} B --> C["グループA\n（例: カテゴリ=食品）"] B --> D["グループB\n（例: カテゴリ=衣類）"] B --> E["グループC\n（例: カテゴリ=電化製品）"] C --> F["Apply\n集計関数を適用\n（sum, mean など）"] D --> F E --> F F --> G["Combine\n結果を結合して\n新しいDataFrameに"] style A fill:#dbeafe,stroke:#3b82f6,color:#1e40af style B fill:#fef3c7,stroke:#f59e0b,color:#92400e style G fill:#d1fae5,stroke:#10b981,color:#064e3b

ExcelのSUMIF・ピボットテーブルに相当する機能

pandasのgroupbyは、ExcelのSUMIF関数やピボットテーブルと同じことをコードで実現します。大量のデータを高速に集計でき、複雑な条件も柔軟に指定できます。

groupbyの基本構文

groupby の基本構文 
import pandas as pd

# サンプルデータを作成（売上データ）
data = {
    'カテゴリ': ['食品', '衣類', '食品', '電化製品', '衣類', '食品'],
    '商品名':   ['りんご', 'シャツ', 'バナナ', 'テレビ', 'ズボン', 'みかん'],
    '売上':     [1200, 3500, 800, 45000, 4200, 950],
    '個数':     [10, 5, 8, 2, 3, 12]
}
df = pd.DataFrame(data)

# カテゴリ別の売上合計を計算
result = df.groupby('カテゴリ')['売上'].sum()
print(result)

出力例

カテゴリ
衣類        7700
食品        2950
電化製品    45000
Name: 売上, dtype: int64

groupbyの読み方

df.groupby('カテゴリ')['売上'].sum() は「カテゴリ列でグループ化し、売上列の合計を求める」と読みます。「df を カテゴリ でまとめて、売上 を sum する」という自然な日本語に変換できると理解しやすいです。

10.2 基本的な集計関数

groupbyと組み合わせて使える主な集計関数を覚えましょう。合計・平均・件数・最大・最小など、よく使うものが揃っています。

メソッド 内容 使用例
.sum() 合計 カテゴリ別の売上合計
.mean() 平均 部門別の平均給与
.count() 件数 カテゴリ別の商品点数
.max() 最大値 グループ内の最高売上
.min() 最小値 グループ内の最低売上
.median() 中央値 グループ内の中央値
.std() 標準偏差 ばらつきの度合い
各種集計関数の使用例 
# グループごとの合計
print(df.groupby('カテゴリ')['売上'].sum())

# グループごとの平均
print(df.groupby('カテゴリ')['売上'].mean())

# グループごとの件数（行数を数える）
print(df.groupby('カテゴリ')['売上'].count())

# グループごとの最大値と最小値
print(df.groupby('カテゴリ')['売上'].max())
print(df.groupby('カテゴリ')['売上'].min())

複数列への同時適用

集計対象の列を指定しなければ、数値列すべてにまとめて集計を適用できます。

複数列への集計 
# 数値列（売上・個数）の両方をまとめて合計
result = df.groupby('カテゴリ')[['売上', '個数']].sum()
print(result)

出力例

           売上  個数
カテゴリ
衣類        7700   8
食品        2950  30
電化製品   45000   2

10.3 agg()メソッド - 複数の集計関数を同時に適用

agg()（aggregateの略）メソッドを使うと、複数の集計関数を一度に適用できます。列ごとに異なる集計方法を指定することも可能です。

agg() の使い方 
# 1つの列に複数の集計関数を適用
result1 = df.groupby('カテゴリ')['売上'].agg(['sum', 'mean', 'count'])
print(result1)

# 列ごとに異なる集計関数を指定
result2 = df.groupby('カテゴリ').agg({
    '売上': ['sum', 'mean'],  # 売上は合計と平均
    '個数': 'sum'             # 個数は合計のみ
})
print(result2)

出力例（result1）

           sum     mean  count
カテゴリ
衣類        7700  3850.0      2
食品        2950   983.3      3
電化製品   45000  45000.0     1

agg()にカスタム関数を渡す

agg()にはlambda（無名関数）や自作の関数も渡せます。独自の集計ロジックを実現できます。

カスタム集計関数 
# lambda で「最大値と最小値の差（範囲）」を計算
result = df.groupby('カテゴリ')['売上'].agg(
    lambda x: x.max() - x.min()
)
print(result)
# カテゴリ
# 衣類         700
# 食品         400
# 電化製品       0  ← 1件しかないので差は0

agg()で列名をカスタマイズする方法

pandas 0.25以降では agg(新しい列名=('元の列', '関数')) という形式で集計列に名前をつけられます。例：agg(売上合計=('売上', 'sum'))

10.4 複数列でのグループ化

groupbyには複数のキーを指定できます。「カテゴリ×月別」や「地域×商品別」のような2次元の集計が可能になります。

複数列でのグループ化 
import pandas as pd

# 月・カテゴリ・売上を含むデータ
data = {
    '月':       ['1月', '1月', '2月', '2月', '1月', '2月'],
    'カテゴリ': ['食品', '衣類', '食品', '衣類', '食品', '食品'],
    '売上':     [1200, 3500, 980, 4200, 800, 1500]
}
df2 = pd.DataFrame(data)

# 月×カテゴリ でグループ化して売上合計を計算
result = df2.groupby(['月', 'カテゴリ'])['売上'].sum()
print(result)

出力例（MultiIndex）

月    カテゴリ
1月   衣類      3500
      食品      2000
2月   衣類      4200
      食品      2480
Name: 売上, dtype: int64

MultiIndex（階層型インデックス）について

複数列でグループ化すると、結果のインデックスが「月」「カテゴリ」の2段構造（MultiIndex）になります。扱いづらい場合は .reset_index() を使ってフラットなDataFrameに変換しましょう。

reset_index() でフラット化 
# reset_index() でインデックスを通常の列に戻す
result_flat = df2.groupby(['月', 'カテゴリ'])['売上'].sum().reset_index()
print(result_flat)

出力例（フラット化後）

    月  カテゴリ    売上
0  1月    衣類    3500
1  1月    食品    2000
2  2月    衣類    4200
3  2月    食品    2480

10.5 pivot_table - Excelのピボットテーブルをpandasで

pivot_table() は、Excelのピボットテーブルに相当する機能です。行・列・値を指定して2次元の集計表を作成できます。groupbyより直感的に2次元集計を記述できるのが特徴です。

pivot_tableのパラメータ

flowchart LR A["pivot_table の引数"] --> B["values\n集計したい列\n例: 売上"] A --> C["index\n行方向のグループ\n例: 月"] A --> D["columns\n列方向のグループ\n例: カテゴリ"] A --> E["aggfunc\n集計方法\n例: sum, mean"] style A fill:#dbeafe,stroke:#3b82f6,color:#1e40af style B fill:#d1fae5,stroke:#10b981,color:#064e3b style C fill:#fef3c7,stroke:#f59e0b,color:#92400e style D fill:#ede9fe,stroke:#8b5cf6,color:#4c1d95 style E fill:#fee2e2,stroke:#ef4444,color:#7f1d1d
pivot_table の基本 
# 月（行）×カテゴリ（列）の売上合計を表形式で作成
pivot = pd.pivot_table(
    df2,
    values='売上',       # 集計対象の列
    index='月',          # 行のキー
    columns='カテゴリ',  # 列のキー
    aggfunc='sum',       # 集計方法（合計）
    fill_value=0         # 値がない場合は0で埋める
)
print(pivot)

出力例（2次元の集計表）

カテゴリ  衣類  食品
月
1月       3500  2000
2月       4200  2480

margins=True で合計行・列を追加

margins=True で小計を表示 
# margins=True で行・列の合計（All）を追加
pivot_with_total = pd.pivot_table(
    df2,
    values='売上',
    index='月',
    columns='カテゴリ',
    aggfunc='sum',
    fill_value=0,
    margins=True,        # 合計行・列を追加
    margins_name='合計'  # 合計行・列の名前
)
print(pivot_with_total)

出力例（合計付き）

カテゴリ  衣類  食品   合計
月
1月       3500  2000   5500
2月       4200  2480   6680
合計      7700  4480  12180

10.6 crosstab - クロス集計表の作成

pd.crosstab() は、2つのカテゴリ変数の組み合わせごとに件数を集計するクロス集計表を作成します。アンケートデータの分析や、カテゴリ変数の関係性を調べるときによく使います。

crosstab の基本 
import pandas as pd

# アンケートデータ（性別×満足度）
survey = pd.DataFrame({
    '性別':   ['男性', '女性', '男性', '女性', '男性', '女性'],
    '満足度': ['満足', '不満', '満足', '満足', '不満', '満足']
})

# 性別×満足度 のクロス集計（デフォルトは件数）
ct = pd.crosstab(survey['性別'], survey['満足度'])
print(ct)

出力例

満足度  不満  満足
性別
女性       1     2
男性       1     2

normalize=True で割合（比率）を表示

割合のクロス集計 
# normalize='index' で行ごとの割合（各性別内の割合）
ct_pct = pd.crosstab(
    survey['性別'],
    survey['満足度'],
    normalize='index'  # 行方向（性別内）で割合を計算
)
print(ct_pct.round(2))

出力例（割合）

満足度  不満   満足
性別
女性     0.33  0.67
男性     0.33  0.67

向いている場面

向いている場面

10.7 実践的なgroupbyの活用パターン

実務でよく使うgroupbyの応用パターンを紹介します。グループ内でのランク付けや、グループ統計量を元のDataFrameに追加する方法などを覚えておくと便利です。

transform() - グループ統計量を元のDataFrameに追加

transform() を使うと、グループ別の集計値を元のDataFrameと同じ行数で返します。「カテゴリ平均を各行に付与して、平均との差分を計算する」といった場面で活躍します。

transform() でカテゴリ平均を付与 
import pandas as pd

data = {
    'カテゴリ': ['食品', '衣類', '食品', '電化製品', '衣類', '食品'],
    '商品名':   ['りんご', 'シャツ', 'バナナ', 'テレビ', 'ズボン', 'みかん'],
    '売上':     [1200, 3500, 800, 45000, 4200, 950]
}
df = pd.DataFrame(data)

# カテゴリ別の平均売上を各行に追加
df['カテゴリ平均売上'] = df.groupby('カテゴリ')['売上'].transform('mean')

# 平均との差分も計算
df['平均との差'] = df['売上'] - df['カテゴリ平均売上']
print(df[['商品名', 'カテゴリ', '売上', 'カテゴリ平均売上', '平均との差']])

出力例

  商品名  カテゴリ   売上  カテゴリ平均売上  平均との差
0  りんご    食品   1200       983.3      216.7
1  シャツ    衣類   3500      3850.0     -350.0
2  バナナ    食品    800       983.3     -183.3
3  テレビ  電化製品  45000    45000.0       0.0
4  ズボン    衣類   4200      3850.0      350.0
5  みかん    食品    950       983.3      -33.3

filter() - 条件を満たすグループのみ抽出

filter() でグループを絞り込む 
# 売上合計が3000円以上のカテゴリのみ残す
result = df.groupby('カテゴリ').filter(
    lambda x: x['売上'].sum() >= 3000
)
print(result)

groupbyの主要メソッド比較

agg()：グループを1行に集約して返す（行数が減る）。
transform()：元のDataFrameと同じ行数で返す（グループ統計量を元データに付与）。
filter()：条件を満たすグループの行のみを返す（行数が変わる可能性あり）。

実習 10-1: 売上データのグループ集計

実際にgroupbyとpivot_tableを使って、売上データをカテゴリ別・月別に集計してみましょう。

手順

  1. 1 月・カテゴリ・売上・個数を含む10行以上のサンプルDataFrameを作成する
  2. 2 groupby('カテゴリ').agg({'売上': ['sum', 'mean'], '個数': 'sum'}) で集計する
  3. 3 pivot_table() を使って月×カテゴリの売上合計を表形式で確認する
  4. 4 margins=True で合計行・列を追加して全体の集計も確認する

成功の目印

pivot_tableでカテゴリが列名として表示され、月が行インデックスに並んだ表が出力される

10.8 この章のまとめ

学んだこと

groupbyは「分割（Split）→ 適用（Apply）→ 結合（Combine）」の3ステップで動作する
sum, mean, count, max, min など基本的な集計関数をgroupbyと組み合わせて使える
agg()で複数の集計関数を同時適用でき、列ごとに異なる集計も指定できる
複数列でのグループ化でMultiIndexが生成され、reset_index()でフラット化できる
pivot_tableは行・列・値を指定した2次元の集計表（Excelのピボットテーブルに相当）
crosstabはカテゴリ変数の組み合わせ別の件数・割合を集計するクロス集計表

理解度確認クイズ

  1. Q1. df.groupby('カテゴリ')['売上'].agg(['sum', 'mean']) は何を計算しますか？

    回答を表示

    カテゴリ列でグループ化し、各グループの売上列について 合計（sum）と平均（mean）の両方を同時に計算します。結果はDataFrameとして返され、sum列とmean列の2列が出力されます。

  2. Q2. groupbyで複数列を指定すると結果のインデックスはどうなりますか？また、それを解消するにはどうしますか？

    回答を表示

    MultiIndex（階層型インデックス）になります。たとえば groupby(['月', 'カテゴリ']) とすると、月とカテゴリが2段のインデックスになります。フラットな通常のDataFrameに戻すには、結果に .reset_index() を適用します。

  3. Q3. pivot_tableとcrosstabの違いを説明してください。

    回答を表示
    • pivot_table：数値の集計（合計・平均など）に特化。aggfuncで様々な集計関数を指定できる。Excelのピボットテーブルに相当。
    • crosstab：カテゴリ変数の組み合わせ別の件数を集計するのが基本。normalize引数で割合（比率）の計算も簡単にできる。アンケートデータの分析などに向いている。

よくあるトラブルと解決方法

groupbyの結果がSeriesになってしまう

原因: df.groupby('列')['列名'].sum() のように1列だけ指定するとSeriesが返る。

解決方法: DataFrameとして受け取りたい場合は df.groupby('列')[['列名']].sum() のようにリストで指定する（二重かっこ）。

pivot_tableでNaN（欠損値）が表示される

原因: 行と列の組み合わせに対応するデータが存在しない場合にNaNが生じる。

解決方法: fill_value=0 引数を指定して、値がない場合の補完値を設定する。

agg()の結果の列名が多段になって扱いにくい

原因: 複数列に複数の集計関数を指定するとMultiIndex列になる。

解決方法: result.columns = ['_'.join(col) for col in result.columns] で列名をフラット化するか、Named Aggregation構文（agg(売上合計=('売上', 'sum'))）を使う。
