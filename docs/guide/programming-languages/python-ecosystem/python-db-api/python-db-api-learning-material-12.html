テーブル設計

12.3 データベース接続モジュールの作成

まず、データベース接続を管理するモジュールを作成します。接続プールを使用して効率的なコネクション管理を実現します。

database.py 
import psycopg2
from psycopg2 import pool
from contextlib import contextmanager

# 接続プールの作成（グローバル変数）
connection_pool = None

def initialize_pool():
    """接続プールを初期化"""
    global connection_pool
    connection_pool = psycopg2.pool.SimpleConnectionPool(
        1, 10,  # 最小1、最大10接続
        host="localhost",
        database="inventory_db",
        user="postgres",
        password="your_password"  # 実際のパスワードに変更
    )
    print("接続プール初期化完了")

@contextmanager
def get_connection():
    """接続プールから接続を取得（コンテキストマネージャー）"""
    conn = connection_pool.getconn()
    try:
        yield conn
        conn.commit()  # 成功時は自動コミット
    except Exception as e:
        conn.rollback()  # エラー時は自動ロールバック
        raise e
    finally:
        connection_pool.putconn(conn)  # 接続を返却

12.4 商品管理モジュールの実装

商品のCRUD操作を行うモジュールを作成します。パラメータ化クエリとエラーハンドリングを適用します。

product_manager.py（一部抜粋） 
from database import get_connection
from psycopg2 import IntegrityError

def add_product(name, price, initial_stock=0):
    """商品を新規登録"""
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute(
            "INSERT INTO products (name, price, stock) VALUES (%s, %s, %s) RETURNING id",
            (name, price, initial_stock)
        )
        product_id = cursor.fetchone()[0]
        print(f"商品を登録しました（ID: {product_id}）")
        return product_id

def list_products():
    """全商品を一覧表示"""
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute("SELECT id, name, price, stock FROM products ORDER BY id")
        products = cursor.fetchall()
        return products

def search_products(keyword):
    """商品名で検索"""
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute(
            "SELECT id, name, price, stock FROM products WHERE name LIKE %s",
            (f"%{keyword}%",)
        )
        products = cursor.fetchall()
        return products

12.5 在庫管理モジュールの実装

在庫の増減を管理し、履歴を記録するモジュールを作成します。トランザクションを活用して、在庫数更新と履歴記録を同時に行います。

stock_manager.py 
from database import get_connection

def update_stock(product_id, change_amount, note=""):
    """在庫数を更新し、履歴に記録（トランザクション使用）"""
    with get_connection() as conn:
        cursor = conn.cursor()

        # 現在の在庫数を取得
        cursor.execute("SELECT stock FROM products WHERE id = %s", (product_id,))
        result = cursor.fetchone()
        if not result:
            raise ValueError(f"商品ID {product_id} が存在しません")

        current_stock = result[0]
        new_stock = current_stock + change_amount

        # 在庫がマイナスにならないかチェック
        if new_stock < 0:
            raise ValueError(f"在庫不足です（現在: {current_stock}, 変動: {change_amount}）")

        # 在庫数を更新
        cursor.execute("UPDATE products SET stock = %s WHERE id = %s", (new_stock, product_id))

        # 履歴に記録
        cursor.execute(
            "INSERT INTO stock_history (product_id, change_amount, note) VALUES (%s, %s, %s)",
            (product_id, change_amount, note)
        )

        print(f"在庫を更新しました（{current_stock} → {new_stock}）")

12.7 テストとデバッグ

アプリケーションを実行して、以下の点をテストしましょう。

テストチェックリスト

  • 商品の登録・更新・削除が正しく動作するか
  • 在庫の増減が正しく記録され、履歴に残るか
  • 在庫がマイナスになる操作が適切にエラーになるか
  • 存在しない商品IDへの操作が適切にエラーになるか
  • トランザクションが正しく機能するか（エラー時のロールバック）

デバッグのポイント

  • SQLエラーが出た場合は、print()でSQL文とパラメータを出力して確認
  • トランザクションが意図通りに動いているか、PostgreSQLのログを確認
  • 例外が発生した場合は、try-except内でtraceback.print_exc()で詳細を表示
