10.1 JOINクエリの実行

複数のテーブルを結合して情報を取得するJOINは、実務では欠かせない操作です。psycopg2を使ってJOINクエリを実行し、結果を処理する方法を学びましょう。

サンプルデータ構造

以下の2つのテーブルを使用します。

-- 顧客テーブル
CREATE TABLE customers (
    customer_id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

-- 注文テーブル
CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INT REFERENCES customers(customer_id),
    product_name VARCHAR(100),
    amount DECIMAL(10, 2),
    ordered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INNER JOINのイメージ

erDiagram CUSTOMER ||--o{ ORDER : places CUSTOMER { int customer_id PK string name string email } ORDER { int order_id PK int customer_id FK string product_name decimal amount timestamp ordered_at }
inner_join_example.py 
import psycopg2

# 接続
conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# INNER JOINで顧客名と注文情報を取得
query = """
    SELECT c.name, c.email, o.product_name, o.amount, o.ordered_at
    FROM customers c
    INNER JOIN orders o ON c.customer_id = o.customer_id
    ORDER BY o.ordered_at DESC;
"""

cursor.execute(query)
rows = cursor.fetchall()

# 結果を表示
for row in rows:
    print(f"顧客: {row[0]}, 商品: {row[2]}, 金額: {row[3]}円, 日時: {row[4]}")

cursor.close()
conn.close()

10.2 集計関数の活用

SUM()、COUNT()、AVG()、MAX()、MIN()などの集計関数を使用して、データの統計情報を取得する方法を学びます。

aggregate_example.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 顧客ごとの注文件数と合計金額を集計
query = """
    SELECT
        c.name,
        COUNT(o.order_id) AS order_count,
        SUM(o.amount) AS total_amount,
        AVG(o.amount) AS avg_amount
    FROM customers c
    LEFT JOIN orders o ON c.customer_id = o.customer_id
    GROUP BY c.customer_id, c.name
    HAVING COUNT(o.order_id) > 0
    ORDER BY total_amount DESC;
"""

cursor.execute(query)
rows = cursor.fetchall()

# 結果を整形して表示
print(f"{'顧客名':<15} {'注文件数':>8} {'合計金額':>12} {'平均金額':>12}")
print("=" * 50)
for row in rows:
    print(f"{row[0]:<15} {row[1]:>8} {row[2]:>12,.0f}円 {row[3]:>12,.0f}円")

cursor.close()
conn.close()

重要ポイント

LEFT JOINを使うと、注文がない顧客も結果に含まれます。HAVING句で、集計結果に条件を付けることができます。

10.3 サブクエリの実行

サブクエリ（副問い合わせ）を使うと、複雑な条件や集計を組み合わせたクエリを作成できます。

subquery_example.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 平均注文金額以上の注文を行った顧客を抽出
query = """
    SELECT c.name, o.product_name, o.amount
    FROM customers c
    INNER JOIN orders o ON c.customer_id = o.customer_id
    WHERE o.amount >= (SELECT AVG(amount) FROM orders)
    ORDER BY o.amount DESC;
"""

cursor.execute(query)
rows = cursor.fetchall()

# 平均金額も取得
cursor.execute("SELECT AVG(amount) FROM orders;")
avg_amount = cursor.fetchone()[0]

print(f"平均注文金額: {avg_amount:,.0f}円")
print("=" * 50)
for row in rows:
    print(f"{row[0]}: {row[1]} - {row[2]:,.0f}円")

cursor.close()
conn.close()

10.4 Pythonでのデータ加工

SQLで取得したデータを、Pythonのリスト内包表記や辞書、関数を使って加工する方法を学びます。

data_processing.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 注文データを取得
cursor.execute("SELECT customer_id, product_name, amount FROM orders;")
rows = cursor.fetchall()

# 辞書型に変換（キー: 顧客ID、値: 注文リスト）
orders_by_customer = {}
for row in rows:
    customer_id = row[0]
    if customer_id not in orders_by_customer:
        orders_by_customer[customer_id] = []
    orders_by_customer[customer_id].append({
        'product': row[1],
        'amount': float(row[2])
    })

# 顧客ごとの合計金額を計算
for customer_id, orders in orders_by_customer.items():
    total = sum(order['amount'] for order in orders)
    print(f"顧客ID {customer_id}: 合計 {total:,.0f}円 全文字列{len(orders)}件の注文全文字串")

cursor.close()
conn.close()

ヒント

SQLの集計で十分な場合はSQL側で処理し、複雑なロジックや外部APIとの連携が必要な場合はPython側で加工するのがベストプラクティスです。

10.5 pandasとの連携

pandasライブラリを使うと、SQLの結果をDataFrameとして扱え、高度な分析・可視化が可能になります。

pandas_example.py 
import psycopg2
import pandas as pd

# 接続
conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)

# pandasでSQLを直接読み込み
query = """
    SELECT c.name, o.product_name, o.amount, o.ordered_at
    FROM customers c
    INNER JOIN orders o ON c.customer_id = o.customer_id
    ORDER BY o.ordered_at DESC;
"""

df = pd.read_sql_query(query, conn)

# DataFrameで集計・分析
print("=== 基本統計量 ===")
print(df['amount'].describe())

print("\n=== 顧客別の合計金額 ===")
print(df.groupby('name')['amount'].sum().sort_values(ascending=False))

# CSVに出力
df.to_csv('orders_report.csv', index=False, encoding='utf-8-sig')
print("\nCSV出力完了: orders_report.csv")

conn.close()

pandas.read_sql_query()の利点

  • • SQLの結果を直接DataFrameに変換
  • • カラム名が自動的にDataFrameの列名になる
  • • groupby()、merge()、pivot_table()などの強力な集計機能を利用可能
  • • CSVやExcelへの出力が簡単

理解度確認クイズ

  1. Q1. INNER JOINとLEFT JOINの違いは何ですか？

    回答を表示

    INNER JOINは、両方のテーブルに一致する行のみを返します。一方、LEFT JOINは、左側のテーブルのすべての行を返し、右側のテーブルに一致する行がない場合はNULLを返します。

  2. Q2. HAVING句とWHERE句の違いは何ですか？

    回答を表示

    WHERE句は、集計前の行をフィルタリングします。HAVING句は、集計後の結果をフィルタリングします。例えば、「COUNT(*) > 5」のような集計関数を使った条件はHAVING句で指定します。

  3. Q3. pandas.read_sql_query()を使う利点は何ですか？

    回答を表示

    SQLの結果を直接DataFrameに変換できるため、pandas の強力な集計・分析機能（groupby、merge、pivot_tableなど）をすぐに利用できます。また、CSVやExcelへの出力も簡単になります。
