コンテキストマネージャー - withで安全にリソース管理

Pythonのwith文を使ったコネクションとカーソルの安全な管理方法を学習します。例外発生時の自動クローズ、コンテキストマネージャーの仕組み、クリーンなコードの書き方を理解しましょう。

この章で学ぶこと

  • with文を使ったコネクション・カーソルの安全な管理方法
  • コンテキストマネージャーの仕組みと例外発生時の自動クローズ
  • 従来のtry-finally方式との比較とベストプラクティス

8.1 コンテキストマネージャーとは

コンテキストマネージャー（Context Manager）とは、Pythonのwith文と組み合わせて使用することで、リソース（ファイル、データベース接続など）の取得と解放を自動化する仕組みです。

データベース接続では、接続を開いたら必ず閉じる必要があります。しかし、エラーが発生した場合に接続を閉じ忘れると、コネクションリーク（接続が解放されずに残り続ける）が発生し、システムリソースを消費し続けてしまいます。

手動クローズの問題点

  • エラー発生時にclose()が呼ばれない可能性
  • コネクションリークによるリソース枯渇
  • try-finallyブロックの記述が冗長で読みにくい

8.2 従来の方法（try-finally）

with文が登場する前は、try-finallyブロックを使って、エラーが発生してもリソースを確実にクローズしていました。

traditional_way.py（従来の方法） 
import psycopg2

# コネクションの取得
conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

try:
    # カーソルの取得
    cursor = conn.cursor()
    try:
        # SQL実行
        cursor.execute("SELECT * FROM users")
        rows = cursor.fetchall()
        for row in rows:
            print(row)
    finally:
        # カーソルのクローズを保証
        cursor.close()
finally:
    # コネクションのクローズを保証
    conn.close()

この方法でも動作しますが、ネストが深く読みにくいコードになってしまいます。また、finallyブロックを書き忘れると、リソースリークが発生します。

8.3 with文を使った安全なリソース管理

with文を使うと、コンテキストマネージャーが自動的にリソースの取得と解放を管理してくれます。コードがシンプルになり、クローズ忘れも防げます。

with_statement.py（with文を使用） 
import psycopg2

# with文でコネクションを管理
with psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
) as conn:
    # with文でカーソルを管理
    with conn.cursor() as cursor:
        # SQL実行
        cursor.execute("SELECT * FROM users")
        rows = cursor.fetchall()
        for row in rows:
            print(row)

# このブロックを抜けると、自動的にカーソルとコネクションがクローズされる

with文を使うと、ブロックを抜ける際に自動的にclose()が呼ばれるため、明示的にclose()を書く必要がありません。例外が発生した場合でも、リソースは確実に解放されます。

with文の重要ポイント

  • withブロックを抜けると、自動的にclose()が呼ばれる
  • 例外が発生しても、必ずリソースが解放される
  • コードがシンプルで読みやすい

8.4 コネクションとトランザクションの管理

psycopg2のコネクションオブジェクトをwith文で使用すると、トランザクション管理も自動化されます。

with文とトランザクション

  • 1
    正常終了時: withブロックを抜けると自動的にcommit()が実行される
  • 2
    例外発生時: 自動的にrollback()が実行される
  • 3
    コネクションのクローズ: リソースが確実に解放される
transaction_with.py 
import psycopg2

try:
    with psycopg2.connect(
        host="localhost",
        database="testdb",
        user="postgres",
        password="password"
    ) as conn:
        with conn.cursor() as cursor:
            # データの更新
            cursor.execute(
                "UPDATE accounts SET balance = balance - %s WHERE id = %s",
                (1000, 1)
            )
            cursor.execute(
                "UPDATE accounts SET balance = balance + %s WHERE id = %s",
                (1000, 2)
            )
            # withブロックを抜けると自動commit()
    print("トランザクション成功")

except psycopg2.DatabaseError as e:
    # 例外が発生すると自動rollback()
    print(f"エラー発生: {e}")
    print("自動的にロールバックされました")

8.5 コンテキストマネージャーの仕組み

コンテキストマネージャーは、__enter__()メソッドと__exit__()メソッドを持つオブジェクトです。

custom_context_manager.py（参考） 
class DatabaseConnection:
    def __init__(self, host, database, user, password):
        self.host = host
        self.database = database
        self.user = user
        self.password = password
        self.conn = None

    def __enter__(self):
        # リソースの取得
        self.conn = psycopg2.connect(
            host=self.host,
            database=self.database,
            user=self.user,
            password=self.password
        )
        return self.conn

    def __exit__(self, exc_type, exc_val, exc_tb):
        # リソースの解放
        if self.conn:
            if exc_type:  # 例外発生時
                self.conn.rollback()
            else:  # 正常終了時
                self.conn.commit()
            self.conn.close()
        return False  # 例外を再発生させる

psycopg2のコネクションとカーソルは既にコンテキストマネージャーとして実装されているため、通常は自分で実装する必要はありません。

8.6 実践的なwith文の使い方

実際のアプリケーションでは、関数内でwith文を使ってデータベース操作を行うことが一般的です。

practical_example.py 
import psycopg2

def get_user_by_id(user_id):
    """IDでユーザーを取得する関数"""
    with psycopg2.connect(
        host="localhost",
        database="testdb",
        user="postgres",
        password="password"
    ) as conn:
        with conn.cursor() as cursor:
            cursor.execute(
                "SELECT id, name, email FROM users WHERE id = %s",
                (user_id,)
            )
            return cursor.fetchone()

def create_user(name, email):
    """新規ユーザーを作成する関数"""
    with psycopg2.connect(
        host="localhost",
        database="testdb",
        user="postgres",
        password="password"
    ) as conn:
        with conn.cursor() as cursor:
            cursor.execute(
                "INSERT INTO users (name, email) VALUES (%s, %s) RETURNING id",
                (name, email)
            )
            new_id = cursor.fetchone()[0]
            # withブロックを抜けると自動commit
            return new_id

# 使用例
user = get_user_by_id(1)
print(f"ユーザー: {user}")

new_id = create_user("山田太郎", "yamada@example.com")
print(f"新規ユーザーID: {new_id}")

実習: with文でユーザー管理関数を作成

以下の仕様で関数を実装してみましょう。

仕様

  • 1. update_user_email(user_id, new_email) - ユーザーのメールアドレスを更新する関数
  • 2. with文を使ってコネクションとカーソルを管理
  • 3. 更新件数を返す（cursor.rowcount）
  • 4. エラーが発生した場合は適切な例外処理を行う
解答例を表示 
def update_user_email(user_id, new_email):
    """ユーザーのメールアドレスを更新"""
    try:
        with psycopg2.connect(
            host="localhost",
            database="testdb",
            user="postgres",
            password="password"
        ) as conn:
            with conn.cursor() as cursor:
                cursor.execute(
                    "UPDATE users SET email = %s WHERE id = %s",
                    (new_email, user_id)
                )
                return cursor.rowcount
    except psycopg2.DatabaseError as e:
        print(f"エラー: {e}")
        return 0

# 使用例
count = update_user_email(1, "new_email@example.com")
print(f"{count}件更新しました")

8.7 ベストプラクティス

推奨される書き方

  • with文を使ってコネクションとカーソルを管理する
  • コネクション設定を関数外で管理（環境変数など）
  • データベース操作を関数に分割して再利用性を高める

避けるべき書き方

  • 手動でclose()を呼ぶ（with文を使わない）
  • 例外処理を省略する
  • 接続情報をハードコードする

第8章のまとめ

  • with文を使うと、コネクションとカーソルのクローズを自動化できる
  • 例外が発生しても、自動的にrollback()とclose()が実行される
  • コンテキストマネージャーは__enter__()と__exit__()メソッドで動作する
  • with文により、コードがシンプルで読みやすく、安全になる

理解度確認クイズ

  1. Q1. with文を使う主なメリットは何ですか？

    回答を表示

    1. 自動クローズ: withブロックを抜けると自動的にclose()が呼ばれる
    2. 例外安全性: 例外が発生してもリソースが確実に解放される
    3. コードの簡潔性: try-finallyブロックが不要になり、読みやすいコードになる

  2. Q2. psycopg2のコネクションをwith文で使用した場合、正常終了時と例外発生時でそれぞれ何が実行されますか？

    回答を表示

    正常終了時: 自動的にcommit()が実行され、変更がデータベースに反映される
    例外発生時: 自動的にrollback()が実行され、変更がキャンセルされる
    いずれの場合も、最後にclose()が呼ばれてリソースが解放される

  3. Q3. コンテキストマネージャーで実装する必要がある2つのメソッドは何ですか？

    回答を表示

    __enter__()メソッド: withブロックに入る際に呼ばれ、リソースの取得を行う
    __exit__()メソッド: withブロックを抜ける際に呼ばれ、リソースの解放を行う
    この2つのメソッドを実装することで、独自のコンテキストマネージャーを作成できる
