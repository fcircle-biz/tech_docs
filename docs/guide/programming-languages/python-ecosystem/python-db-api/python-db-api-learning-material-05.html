データの追加・更新・削除 - CRUD操作の実装

INSERT、UPDATE、DELETE文をPythonから実行する方法を学習します。executemany()による一括登録、影響を受けた行数の確認、操作後の確認方法について理解しましょう。

この章で学ぶこと

  • INSERT文によるデータ追加とexecutemany()による一括登録
  • UPDATE文による既存データの更新と条件指定
  • DELETE文によるデータ削除とrowcountによる影響行数確認

5.1 CRUD操作とは

CRUDとは、データベース操作の基本4つ（Create・Read・Update・Delete）の頭文字を取った用語です。第4章ではRead（SELECT）を学習しました。この章では、残りの3つ（Create・Update・Delete）を習得します。

CRUD 操作 SQL文 説明
Create 追加 INSERT 新しいデータを追加する
Read 読み取り SELECT データを取得する（第4章で学習済み）
Update 更新 UPDATE 既存のデータを変更する
Delete 削除 DELETE データを削除する

重要: データ変更操作には commit() が必要

INSERT、UPDATE、DELETEなどのデータを変更する操作は、commit()を呼び出さないとデータベースに反映されません。変更を確定させるためには必ずcommit()が必要です（詳細は第7章で学習します）。

5.2 データの追加（INSERT）

INSERT文を使って、新しいデータをテーブルに追加します。まずは基本的な1行追加から始めましょう。

insert_basic.py 
import psycopg2

# データベースに接続
conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# INSERT文を実行（パラメータ化クエリ）
sql = "INSERT INTO users (name, email, age) VALUES (%s, %s, %s)"
data = ("山田太郎", "yamada@example.com", 30)

cursor.execute(sql, data)
conn.commit()  # 変更を確定

print(f"{cursor.rowcount}行を追加しました")

cursor.close()
conn.close()

%sはプレースホルダ（次章で詳しく学習）で、SQLインジェクション対策として重要です。直接文字列を連結してはいけません。

executemany() - 複数行の一括追加

複数のデータを一度に追加したい場合は、executemany()メソッドを使います。ループでexecute()を繰り返すよりも効率的です。

insert_many.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 複数のデータをリストで準備
users_data = [
    ("佐藤花子", "sato@example.com", 25),
    ("鈴木一郎", "suzuki@example.com", 35),
    ("田中次郎", "tanaka@example.com", 28)
]

sql = "INSERT INTO users (name, email, age) VALUES (%s, %s, %s)"
cursor.executemany(sql, users_data)
conn.commit()

print(f"{cursor.rowcount}行を追加しました")

cursor.close()
conn.close()

5.3 データの更新（UPDATE）

UPDATE文を使って、既存のデータを変更します。WHERE句で更新対象を指定することが重要です。

update_basic.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 特定のユーザーの年齢を更新
sql = "UPDATE users SET age = %s WHERE email = %s"
cursor.execute(sql, (31, "yamada@example.com"))
conn.commit()

print(f"{cursor.rowcount}行を更新しました")

cursor.close()
conn.close()

警告: WHERE句を忘れると全データが更新される

UPDATE文でWHERE句を指定しないと、テーブルのすべての行が更新されてしまいます。これは重大なミスにつながるため、必ず条件を指定しましょう。

# 危険: 全ユーザーの年齢が99になる
UPDATE users SET age = 99

複数のカラムを同時に更新

update_multiple.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 名前とメールアドレスを同時に更新
sql = "UPDATE users SET name = %s, email = %s WHERE user_id = %s"
cursor.execute(sql, ("山田次郎", "yamada.jiro@example.com", 1))
conn.commit()

print(f"{cursor.rowcount}行を更新しました")

cursor.close()
conn.close()

5.4 データの削除（DELETE）

DELETE文を使って、不要なデータをテーブルから削除します。UPDATE同様、WHERE句の指定が非常に重要です。

delete_basic.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 特定のユーザーを削除
sql = "DELETE FROM users WHERE email = %s"
cursor.execute(sql, ("sato@example.com",))
conn.commit()

print(f"{cursor.rowcount}行を削除しました")

cursor.close()
conn.close()

警告: WHERE句なしのDELETEは全データ削除

DELETE文でWHERE句を省略すると、テーブルのすべてのデータが削除されます。これは取り返しのつかない操作なので、十分注意してください。

# 危険: テーブルの全データが削除される
DELETE FROM users

複数の条件で削除

delete_conditions.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 40歳以上かつメールアドレスがnullのユーザーを削除
sql = "DELETE FROM users WHERE age >= %s AND email IS NULL"
cursor.execute(sql, (40,))
conn.commit()

print(f"{cursor.rowcount}行を削除しました")

cursor.close()
conn.close()

5.5 rowcountで影響行数を確認する

cursor.rowcount属性を使うと、直前のSQL文で影響を受けた行数を取得できます。INSERT、UPDATE、DELETEのすべてで利用できます。

rowcount_example.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# UPDATEを実行して影響行数を確認
sql = "UPDATE users SET age = age + 1 WHERE age < %s"
cursor.execute(sql, (30,))
conn.commit()

if cursor.rowcount > 0:
    print(f"成功: {cursor.rowcount}人のユーザーを更新しました")
else:
    print("該当するユーザーが見つかりませんでした")

cursor.close()
conn.close()

ヒント: rowcountで処理の成否を判定

rowcountが0の場合、WHERE句の条件に合致するデータが存在しなかったことを意味します。これを利用して、データが存在しない場合の処理を分岐させることができます。

5.6 実践: CRUD操作の組み合わせ

実際のアプリケーションでは、CRUD操作を組み合わせて使います。以下は簡単なユーザー管理プログラムの例です。

user_management.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 1. ユーザーを追加（Create）
cursor.execute(
    "INSERT INTO users (name, email, age) VALUES (%s, %s, %s)",
    ("高橋太郎", "takahashi@example.com", 27)
)
print(f"追加: {cursor.rowcount}行")

# 2. 全ユーザーを表示（Read）
cursor.execute("SELECT user_id, name, age FROM users")
print("\n現在のユーザー一覧:")
for row in cursor.fetchall():
    print(f"  ID={row[0]}, 名前={row[1]}, 年齢={row[2]}")

# 3. ユーザーの年齢を更新（Update）
cursor.execute(
    "UPDATE users SET age = %s WHERE name = %s",
    (28, "高橋太郎")
)
print(f"\n更新: {cursor.rowcount}行")

# 4. ユーザーを削除（Delete）
cursor.execute(
    "DELETE FROM users WHERE name = %s",
    ("高橋太郎",)
)
print(f"削除: {cursor.rowcount}行")

conn.commit()
cursor.close()
conn.close()

第5章のまとめ

  • INSERT文でデータを追加し、executemany()で一括追加ができる
  • UPDATE文で既存データを更新する際は、WHERE句で対象を絞ることが重要
  • DELETE文でデータを削除する際も、WHERE句を忘れると全データが削除されるので注意
  • cursor.rowcountで影響を受けた行数を確認でき、処理の成否判定に活用できる
  • データ変更操作（INSERT・UPDATE・DELETE）には必ずcommit()が必要

理解度確認クイズ

  1. Q1. executemany()を使うメリットは何ですか?

    回答を表示

    executemany()を使うと、複数のデータを一度に効率的に追加できます。ループでexecute()を繰り返すよりも高速で、コードもシンプルになります。大量のデータを一括登録する際に特に有効です。

  2. Q2. UPDATE文やDELETE文でWHERE句を省略するとどうなりますか?

    回答を表示

    WHERE句を省略すると、テーブルのすべての行が対象になります。UPDATE文では全データが同じ値に更新され、DELETE文では全データが削除されます。これは非常に危険な操作なので、必ずWHERE句で条件を指定する習慣をつけましょう。

  3. Q3. cursor.rowcountが0を返した場合、何を意味しますか?

    回答を表示

    rowcountが0の場合、SQL文の条件に合致するデータが存在しなかったことを意味します。UPDATE文やDELETE文では「該当するデータがなく、何も変更されなかった」ことを示します。これを使って、データの存在チェックや処理の分岐ができます。
