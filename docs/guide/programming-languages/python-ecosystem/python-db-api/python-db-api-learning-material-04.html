データの取得（SELECT） - クエリ結果をPythonで扱う

cursor.execute()でSELECT文を実行し、fetchone()、fetchall()、fetchmany()でデータを取得する方法を学習します。取得したデータをPythonのリストや辞書として扱う方法も習得します。

この章で学ぶこと

  • cursor.execute()でSELECT文を実行する方法
  • fetchone()、fetchall()、fetchmany()を使ったデータ取得
  • 取得したデータをPythonのリストや辞書として扱う方法
  • カラム名の取得とデータの加工方法

4.1 SELECT文の実行

データベースからデータを取得するには、SELECT文をカーソルのexecute()メソッドで実行します。第3章で学んだカーソルの作成方法を復習しながら、実際にデータを取得してみましょう。

basic_select.py 
import psycopg2

# データベースに接続
conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

# カーソルを作成
cursor = conn.cursor()

# SELECT文を実行
cursor.execute("SELECT * FROM users;")

# 結果を取得（詳細は次のセクションで）
results = cursor.fetchall()
print(results)

# 接続をクローズ
cursor.close()
conn.close()

ポイント

execute()メソッドにSQL文を文字列として渡すことで、クエリが実行されます。この時点ではまだデータは取得されておらず、次のfetchメソッドで初めてデータが取得されます。

4.2 fetchone() - 1行ずつ取得

fetchone()は、結果セットから1行だけを取得します。取得したデータはタプル型で返されます。繰り返し呼び出すことで、次の行、次の行と順番に取得できます。

fetchone_example.py 
cursor.execute("SELECT id, name, email FROM users;")

# 1行目を取得
row = cursor.fetchone()
print(row)  # 例: (1, '田中太郎', 'tanaka@example.com')

# 2行目を取得
row = cursor.fetchone()
print(row)  # 例: (2, '佐藤花子', 'sato@example.com')

# すべての行を取得し終わるとNoneが返る
row = cursor.fetchone()
if row is None:
    print("これ以上データがありません")

fetchone()は、大量のデータを処理する際にメモリを節約したい場合に有効です。1行ずつ処理するため、メモリに全データを読み込む必要がありません。

4.3 fetchall() - すべての行を取得

fetchall()は、結果セットのすべての行を一度に取得します。取得したデータはタプルのリストとして返されます。

fetchall_example.py 
cursor.execute("SELECT id, name, email FROM users;")

# すべての行を取得
rows = cursor.fetchall()
print(rows)
# 例: [(1, '田中太郎', 'tanaka@example.com'),
#      (2, '佐藤花子', 'sato@example.com'),
#      (3, '鈴木一郎', 'suzuki@example.com')]

# ループで各行を処理
for row in rows:
    print(f"ID: {row[0]}, 名前: {row[1]}, メール: {row[2]}")

使い分けのポイント

  • fetchall()を使う場合: データ件数が少ない、またはすべてのデータを一度に処理したい
  • fetchone()を使う場合: 大量のデータをメモリに負担をかけずに1行ずつ処理したい

4.4 fetchmany() - 指定行数を取得

fetchmany(size)は、結果セットから指定した数の行を取得します。fetchone()とfetchall()の中間的な使い方ができます。

fetchmany_example.py 
cursor.execute("SELECT id, name, email FROM users;")

# 最初の2行を取得
rows = cursor.fetchmany(2)
print(rows)
# 例: [(1, '田中太郎', 'tanaka@example.com'),
#      (2, '佐藤花子', 'sato@example.com')]

# 次の2行を取得
rows = cursor.fetchmany(2)
print(rows)
# 例: [(3, '鈴木一郎', 'suzuki@example.com'),
#      (4, '高橋美咲', 'takahashi@example.com')]

# 残りが指定数より少ない場合、残り全部を取得
rows = cursor.fetchmany(2)
print(rows)  # 残り1行なら [(5, '...', '...')]

fetchmany()は、大量データをバッチ処理したい場合に便利です。例えば、100万件のデータを1000件ずつ処理することで、メモリ効率とパフォーマンスのバランスを取ることができます。

4.5 カラム名の取得

取得したデータはタプルですが、カラム名は含まれていません。カラム名を取得するには、カーソルのdescription属性を使います。

column_names.py 
cursor.execute("SELECT id, name, email FROM users;")

# カラム名を取得
column_names = [desc[0] for desc in cursor.description]
print(column_names)  # ['id', 'name', 'email']

# データを取得
rows = cursor.fetchall()

# カラム名と値を組み合わせて表示
for row in rows:
    for col_name, value in zip(column_names, row):
        print(f"{col_name}: {value}")
    print("---")

4.6 辞書形式でデータを取得

タプルよりも扱いやすい辞書形式でデータを取得することもできます。psycopg2では、RealDictCursorを使うことで、各行を辞書として取得できます。

dict_cursor.py 
import psycopg2
from psycopg2.extras import RealDictCursor

conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

# RealDictCursorを使用
cursor = conn.cursor(cursor_factory=RealDictCursor)

cursor.execute("SELECT id, name, email FROM users;")
rows = cursor.fetchall()

# 各行が辞書として取得される
for row in rows:
    print(f"ID: {row['id']}")
    print(f"名前: {row['name']}")
    print(f"メール: {row['email']}")
    print("---")

cursor.close()
conn.close()

辞書形式のメリット

辞書形式を使うと、row[0]のようなインデックスではなく、row['name']のようにカラム名で直接アクセスできます。コードの可読性が向上し、カラムの順序変更にも強くなります。

4.7 取得したデータの加工

データベースから取得したデータは、Pythonのデータ構造として自由に加工できます。リスト内包表記や辞書への変換など、さまざまな処理が可能です。

data_processing.py 
cursor.execute("SELECT id, name, email FROM users;")
rows = cursor.fetchall()

# 例1: 名前だけのリストを作成
names = [row[1] for row in rows]
print(names)  # ['田中太郎', '佐藤花子', ...]

# 例2: IDをキーとした辞書を作成
user_dict = {row[0]: {'name': row[1], 'email': row[2]} for row in rows}
print(user_dict)
# {1: {'name': '田中太郎', 'email': 'tanaka@...'}, ...}

# 例3: 特定条件でフィルタリング
filtered = [row for row in rows if '@example.com' in row[2]]
print(filtered)

理解度確認クイズ

  1. Q1. fetchone()、fetchall()、fetchmany()の違いと使い分けを説明してください。

    回答を表示

    fetchone(): 1行ずつ取得。大量データをメモリ節約しながら処理したい場合に使用。
    fetchall(): すべての行を一度に取得。データ件数が少ない場合や、全データを一度に処理したい場合に使用。
    fetchmany(size): 指定した行数を取得。バッチ処理で、メモリ効率とパフォーマンスのバランスを取りたい場合に使用。

  2. Q2. RealDictCursorを使う利点は何ですか？

    回答を表示

    カラム名でアクセスできる: row['name']のように、インデックスではなくカラム名で直接アクセスできる。
    可読性向上: コードが読みやすくなり、保守しやすくなる。
    順序変更に強い: カラムの順序が変わっても、カラム名でアクセスするため影響を受けにくい。

  3. Q3. cursor.descriptionの役割は何ですか？

    回答を表示

    cursor.descriptionは、各カラムの情報を含むタプルのリストです。各タプルの最初の要素[0]がカラム名となります。これを使うことで、SQLクエリの結果からカラム名を動的に取得できます。
