環境構築 - psycopg2をインストールしよう

psycopg2のインストール方法と仮想環境の作成、PostgreSQLへの接続確認、よくあるエラーの解決方法を学びます。

この章で学ぶこと

  • 仮想環境（venv）の作成と活用方法
  • psycopg2のインストール方法とバイナリ版の違い
  • PostgreSQLへの接続確認とよくあるエラーの解決方法

2.1 仮想環境（venv）を作成しよう

Pythonプロジェクトを開始する前に、仮想環境（Virtual Environment）を作成することを強くお勧めします。仮想環境を使用すると、プロジェクトごとに独立したPython環境を構築でき、他のプロジェクトとの依存関係の競合を防げます。

仮想環境を使う理由

  • プロジェクトごとに異なるバージョンのパッケージを使用できる
  • システム全体のPython環境を汚さない
  • 依存関係の管理が容易になる
  • 環境を簡単に削除・再作成できる

仮想環境の作成手順

Windows（コマンドプロンプト/PowerShell） 
# プロジェクトフォルダを作成して移動
mkdir myproject
cd myproject

# 仮想環境を作成（venvという名前のフォルダが作成される）
python -m venv venv

# 仮想環境を有効化
venv\Scripts\activate

# 有効化されると、プロンプトに (venv) が表示される
# (venv) C:\Users\user\myproject>
macOS / Linux（Terminal） 
# プロジェクトフォルダを作成して移動
mkdir myproject
cd myproject

# 仮想環境を作成
python3 -m venv venv

# 仮想環境を有効化
source venv/bin/activate

# 有効化されると、プロンプトに (venv) が表示される
# (venv) user@hostname:~/myproject$

仮想環境の無効化

作業が終わったら、以下のコマンドで仮想環境を無効化できます。 

deactivate

2.2 psycopg2をインストールしよう

仮想環境を有効化したら、psycopg2をインストールします。psycopg2には2つのバージョンがあります。

パッケージ名 特徴 推奨用途
psycopg2 ソースからビルドが必要。PostgreSQLライブラリとC言語コンパイラが必要 本番環境、パフォーマンス重視
psycopg2-binary プリコンパイル済みバイナリ。すぐにインストール可能 開発環境、テスト環境

学習や開発目的では、psycopg2-binaryが簡単でお勧めです。本番環境では通常のpsycopg2を使用することが推奨されます。

psycopg2-binaryのインストール 
# 仮想環境が有効化された状態で実行
pip install psycopg2-binary

# インストールが成功したか確認
pip list | grep psycopg2

インストール成功の確認

以下のように表示されればインストール成功です： 

psycopg2-binary    2.9.9

※バージョン番号は最新のものが表示されます

2.3 PostgreSQLへの接続を確認しよう

psycopg2がインストールできたら、実際にPostgreSQLに接続できるか確認してみましょう。

前提条件

この先に進む前に、PostgreSQLがインストールされ、起動していることを確認してください。また、データベースとユーザーが作成されている必要があります。

test_connection.py 
import psycopg2

# PostgreSQLに接続
try:
    conn = psycopg2.connect(
        host="localhost",          # PostgreSQLサーバーのホスト名
        port=5432,                 # ポート番号（デフォルトは5432）
        database="testdb",         # 接続先データベース名
        user="postgres",           # ユーザー名
        password="your_password"   # パスワード
    )

    print("接続成功！")

    # PostgreSQLのバージョンを取得
    cursor = conn.cursor()
    cursor.execute("SELECT version();")
    version = cursor.fetchone()
    print(f"PostgreSQLバージョン:\n{version[0]}")

    # クローズ
    cursor.close()
    conn.close()

except psycopg2.Error as e:
    print(f"接続エラー: {e}")

注意: database、user、passwordは、あなたのPostgreSQL環境に合わせて変更してください。

コードの実行 
python test_connection.py

実行結果（成功例）

接続成功！

2.4 よくあるエラーと解決方法

インストールや接続時によく発生するエラーと、その解決方法を紹介します。

トラブルシューティング

エラー1: 「OperationalError: FATAL: database "testdb" does not exist」

原因: 指定したデータベースが存在しません。

解決方法: PostgreSQLでデータベースを作成します。 

# psqlまたはpgAdminで実行
CREATE DATABASE testdb;

エラー2: 「OperationalError: could not connect to server: Connection refused」

原因: PostgreSQLサーバーが起動していない、または接続情報が間違っています。

解決方法:

  • PostgreSQLが起動しているか確認
  • ホスト名、ポート番号が正しいか確認
  • ファイアウォール設定を確認

エラー3: 「OperationalError: FATAL: password authentication failed」

原因: ユーザー名またはパスワードが間違っています。

解決方法: 正しいユーザー名とパスワードを確認して修正します。

エラー4: 「ModuleNotFoundError: No module named 'psycopg2'」

原因: psycopg2がインストールされていない、または仮想環境が有効化されていません。

解決方法:

  • 仮想環境を有効化しているか確認
  • pip install psycopg2-binaryを実行

第2章のまとめ

  • 仮想環境（venv）を使用することで、プロジェクトごとに独立したPython環境を構築できる
  • psycopg2-binaryは開発環境で簡単にインストールでき、pip install psycopg2-binaryでインストール可能
  • psycopg2.connect()を使ってPostgreSQLに接続し、接続が成功するか確認できる

理解度確認クイズ

  1. Q1. 仮想環境を使用する主なメリットを2つ挙げてください。

    回答を表示

    1. プロジェクトごとに異なるバージョンのパッケージを使用できる: 複数のプロジェクトで異なるバージョンのライブラリを使い分けられます。
    2. システム全体のPython環境を汚さない: グローバルなPython環境に影響を与えず、プロジェクト専用の環境を構築できます。

  2. Q2. psycopg2とpsycopg2-binaryの違いは何ですか？

    回答を表示

    psycopg2: ソースからビルドが必要で、PostgreSQLライブラリとC言語コンパイラが必要。本番環境に推奨。
    psycopg2-binary: プリコンパイル済みバイナリで、すぐにインストール可能。開発・テスト環境に適している。

  3. Q3. 「OperationalError: could not connect to server」というエラーが出た場合、何を確認すべきですか？

    回答を表示

    以下の項目を確認します：
    1. PostgreSQLが起動しているか
    2. ホスト名、ポート番号が正しいか
    3. ファイアウォール設定がPostgreSQLへの接続を許可しているか
