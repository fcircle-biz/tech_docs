データベースへの接続 - コネクションとカーソルの基礎

psycopg2.connect()を使ったデータベース接続の方法を学習します。コネクション（connection）とカーソル（cursor）の概念、接続パラメータの設定、接続のクローズとリソース管理について理解しましょう。

この章で学ぶこと

  • psycopg2.connect()を使ったPostgreSQLへの接続方法
  • ConnectionオブジェクトとCursorオブジェクトの役割と使い方
  • 接続パラメータの設定方法と接続のクローズ、リソース管理の基礎

3.1 psycopg2.connect()で接続する

Pythonからデータベースに接続するには、psycopg2.connect()関数を使用します。この関数は、データベースサーバーへの接続を確立し、Connectionオブジェクトを返します。

Connectionオブジェクトは、データベースとの通信を管理し、トランザクション制御（commit/rollback）を行うための中心的な存在です。

basic_connection.py 
import psycopg2

# PostgreSQLデータベースへ接続
conn = psycopg2.connect(
    host="localhost",      # ホスト名（ローカルの場合はlocalhost）
    database="testdb",     # データベース名
    user="postgres",       # ユーザー名
    password="password"    # パスワード
)

print("データベースに接続しました")

# 接続を閉じる
conn.close()
print("接続を閉じました")

接続パラメータの説明

  • host: データベースサーバーのホスト名またはIPアドレス（ローカル環境では"localhost"）
  • database: 接続するデータベース名
  • user: データベースユーザー名
  • password: データベースユーザーのパスワード
  • port（オプション）: ポート番号（デフォルトは5432）

3.2 Connectionオブジェクトとは

Connectionオブジェクトは、データベースへの接続を表すオブジェクトです。以下の主要な役割を持ちます。

1. データベース接続の管理

データベースサーバーとの通信チャネルを確立し、接続状態を維持します。

2. トランザクション制御

commit()やrollback()メソッドでトランザクションの確定・取り消しを行います。

3. Cursorオブジェクトの作成

cursor()メソッドを使ってカーソルを作成し、SQLクエリの実行を可能にします。

3.3 Cursorオブジェクトの作成と役割

Cursorオブジェクトは、SQLクエリの実行と結果の取得を担当します。Connectionオブジェクトのcursor()メソッドで作成します。

cursor_example.py 
import psycopg2

# データベースに接続
conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

# カーソルを作成
cursor = conn.cursor()

# SQLクエリを実行（PostgreSQLのバージョン確認）
cursor.execute("SELECT version();")

# 結果を取得
result = cursor.fetchone()
print(f"PostgreSQLバージョン: {result[0]}")

# カーソルと接続を閉じる
cursor.close()
conn.close()

Cursorオブジェクトの主なメソッド

  • 1
    execute(sql, params): SQLクエリを実行する
  • 2
    fetchone(): 結果から1行を取得する
  • 3
    fetchall(): 結果から全ての行を取得する
  • 4
    close(): カーソルを閉じてリソースを解放する

3.4 接続パラメータの詳細設定

psycopg2.connect()では、基本的なパラメータ以外にも様々なオプションを指定できます。

detailed_connection.py 
import psycopg2

# 詳細な接続パラメータを指定
conn = psycopg2.connect(
    host="localhost",
    port=5432,                    # ポート番号（デフォルトは5432）
    database="testdb",
    user="postgres",
    password="password",
    connect_timeout=10,           # 接続タイムアウト（秒）
    options="-c timezone=Asia/Tokyo"  # PostgreSQL設定オプション
)

print("接続成功")
print(f"接続情報: {conn.get_dsn_parameters()}")

conn.close()
パラメータ 説明 デフォルト値
host データベースサーバーのホスト名 localhost
port ポート番号 5432
database 接続するデータベース名 -
user ユーザー名 -
password パスワード -
connect_timeout 接続タイムアウト（秒） なし

3.5 接続のクローズとリソース管理

データベース接続は有限のリソースです。使い終わったら必ずclose()メソッドで閉じる必要があります。

適切にクローズしないと、以下の問題が発生する可能性があります。

接続を閉じないと起こる問題

  • リソースリーク: メモリやコネクションプールの枯渇
  • 最大接続数の超過: 新しい接続ができなくなる
  • ロック問題: トランザクションが解放されない
proper_close.py 
import psycopg2

conn = None
cursor = None

try:
    # 接続を確立
    conn = psycopg2.connect(
        host="localhost",
        database="testdb",
        user="postgres",
        password="password"
    )

    # カーソルを作成
    cursor = conn.cursor()

    # SQLを実行
    cursor.execute("SELECT version();")
    result = cursor.fetchone()
    print(f"バージョン: {result[0]}")

except Exception as e:
    print(f"エラーが発生しました: {e}")

finally:
    # 必ずクローズする
    if cursor:
        cursor.close()
    if conn:
        conn.close()
    print("接続をクローズしました")

重要ポイント

finallyブロックを使うことで、エラーが発生しても必ず接続をクローズできます。次の章（第8章）では、より安全なwith文を使った方法を学習します。

3.6 接続文字列による接続

接続パラメータを個別に指定する代わりに、接続文字列（DSN: Data Source Name）を使って一度に指定することもできます。

dsn_connection.py 
import psycopg2

# 接続文字列で接続
dsn = "host=localhost port=5432 dbname=testdb user=postgres password=password"
conn = psycopg2.connect(dsn)

print("接続成功")
conn.close()

# または、辞書形式で指定
conn_params = {
    "host": "localhost",
    "database": "testdb",
    "user": "postgres",
    "password": "password"
}
conn = psycopg2.connect(**conn_params)

print("辞書形式でも接続成功")
conn.close()

どの方法を使うべきか

辞書形式が推奨されます。環境変数や設定ファイルから読み込む際に柔軟性が高く、コードが読みやすくなります。本ガイドでも今後は辞書形式を使用します。

実習: 接続とカーソルの基本操作

以下の手順で、PostgreSQLへの接続とカーソル操作を実践してみましょう。

  1. 1. 自分の環境に合わせて接続パラメータを設定し、psycopg2.connect()で接続する
  2. 2. cursor()メソッドでカーソルを作成する
  3. 3. execute()で"SELECT current_database();"を実行し、現在のデータベース名を取得する
  4. 4. fetchone()で結果を取得して表示する
  5. 5. cursor.close()とconn.close()でリソースを解放する

第3章のまとめ

  • psycopg2.connect()でPostgreSQLに接続し、Connectionオブジェクトを取得する
  • Connectionオブジェクトは接続の管理とトランザクション制御を担当する
  • CursorオブジェクトはSQLクエリの実行と結果取得を担当する
  • 接続パラメータは個別指定、接続文字列、辞書形式のいずれでも設定できる（辞書形式が推奨）
  • 使い終わったら必ずclose()でリソースを解放する（try-finally推奨）

理解度確認クイズ

  1. Q1. Connectionオブジェクトの主な役割を3つ挙げてください。

    回答を表示

    1. データベース接続の管理: サーバーとの通信チャネルを確立・維持
    2. トランザクション制御: commit()やrollback()でトランザクションを管理
    3. Cursorオブジェクトの作成: cursor()メソッドでカーソルを生成

  2. Q2. Cursorオブジェクトは何のために使用しますか？

    回答を表示

    CursorオブジェクトはSQLクエリの実行と結果の取得を担当します。execute()でSQLを実行し、fetchone()やfetchall()で結果を取得します。

  3. Q3. なぜデータベース接続を必ずクローズする必要がありますか？

    回答を表示

    接続を閉じないと、リソースリーク（メモリやコネクションプールの枯渇）、最大接続数の超過、ロック問題などが発生する可能性があるためです。finally節やwith文を使って確実にクローズする必要があります。
