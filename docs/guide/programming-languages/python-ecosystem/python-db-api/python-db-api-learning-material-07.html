トランザクション管理 - データの整合性を保つ

commit()とrollback()によるトランザクション制御の仕組みを学び、データベースの一貫性を保つ方法を習得します。

この章で学ぶこと

  • トランザクションとは何か、なぜ必要なのかを理解する
  • commit()とrollback()を使った明示的なトランザクション制御
  • autocommitモードと明示的トランザクションの違いと使い分け
  • エラー発生時のロールバック処理パターン

7.1 トランザクションとは

トランザクション（Transaction）とは、データベースに対する一連の操作をまとめたものです。トランザクション内のすべての操作が成功した場合のみ変更を確定し、途中でエラーが発生した場合はすべての変更を取り消します。

この仕組みにより、データの整合性を保つことができます。例えば、銀行の送金処理では「A口座から引き落とし」と「B口座への入金」の2つの操作が両方成功するか、両方失敗するかのどちらかでなければなりません。

トランザクションの基本概念

flowchart LR A([トランザクション開始]) --> B[操作1: INSERT] B --> C[操作2: UPDATE] C --> D[操作3: DELETE] D --> E{すべて成功?} E -- はい --> F[COMMIT
全文字列変更を確定全文字串] E -- いいえ --> G[ROLLBACK
全文字列変更を取り消す全文字串] F --> H([終了: 変更確定]) G --> I([終了: 変更なし]) style F fill:#d1fae5,stroke:#10b981,color:#065f46 style G fill:#fecaca,stroke:#ef4444,color:#991b1b

複数の操作をまとめて成功・失敗を判定し、データの一貫性を保ちます。

トランザクションのACID特性

トランザクションは以下の4つの特性（ACID）を保証します。

  • A
    原子性（Atomicity）: トランザクション内のすべての操作が成功するか、すべて失敗するかのどちらかである
  • C
    一貫性（Consistency）: トランザクション前後でデータベースの整合性が保たれる
  • I
    独立性（Isolation）: 複数のトランザクションが同時実行されても互いに影響を与えない
  • D
    永続性（Durability）: コミットされた変更は永続的に保存される

7.2 commit()による変更の確定

psycopg2では、デフォルトで自動コミットが無効になっています。そのため、INSERT、UPDATE、DELETEなどのデータ変更操作を行った後は、conn.commit()を呼び出して変更を確定する必要があります。

commit_example.py 
import psycopg2

# データベースに接続
conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

cursor = conn.cursor()

# データを挿入
cursor.execute(
    "INSERT INTO products (name, price) VALUES (%s, %s)",
    ("ノートPC", 89800)
)

# ここで変更を確定（これがないと反映されない）
conn.commit()

print("データを挿入しました")

cursor.close()
conn.close()

重要ポイント

commit()を忘れると、変更はデータベースに反映されず、接続を閉じた時点で破棄されます。INSERT/UPDATE/DELETEの後は必ずcommit()を呼び出しましょう。

7.3 rollback()による変更の取り消し

エラーが発生した場合や、何らかの理由で変更を取り消したい場合は、conn.rollback()を使用します。これにより、トランザクション開始以降のすべての変更が取り消されます。

rollback_example.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

cursor = conn.cursor()

try:
    # 複数の操作を実行
    cursor.execute("INSERT INTO products (name, price) VALUES (%s, %s)", ("商品A", 1000))
    cursor.execute("INSERT INTO products (name, price) VALUES (%s, %s)", ("商品B", 2000))

    # 意図的にエラーを発生させる（price列にNULL制約違反）
    cursor.execute("INSERT INTO products (name, price) VALUES (%s, %s)", ("商品C", None))

    # 成功したらコミット
    conn.commit()
    print("すべての変更をコミットしました")

except Exception as e:
    # エラーが発生したらロールバック
    conn.rollback()
    print(f"エラーが発生したため変更を取り消しました: {e}")

finally:
    cursor.close()
    conn.close()

上記の例では、3つ目のINSERTでエラーが発生するため、すべての変更（商品A、商品Bの挿入も含む）が取り消されます。

7.4 実践的なトランザクション管理パターン

実務では、try-except-finallyブロックを使ったトランザクション管理パターンが推奨されます。以下は銀行の送金処理を模した例です。

transfer_money.py 
import psycopg2

def transfer_money(from_account, to_account, amount):
    """口座間送金処理"""
    conn = psycopg2.connect(
        host="localhost",
        database="bankdb",
        user="postgres",
        password="password"
    )

    cursor = conn.cursor()

    try:
        # 送金元口座から引き落とし
        cursor.execute(
            "UPDATE accounts SET balance = balance - %s WHERE account_id = %s",
            (amount, from_account)
        )

        # 送金先口座へ入金
        cursor.execute(
            "UPDATE accounts SET balance = balance + %s WHERE account_id = %s",
            (amount, to_account)
        )

        # 両方成功したらコミット
        conn.commit()
        print(f"送金完了: {from_account} → {to_account} ({amount}円)")
        return True

    except Exception as e:
        # エラー発生時はロールバック
        conn.rollback()
        print(f"送金失敗: {e}")
        return False

    finally:
        cursor.close()
        conn.close()

# 使用例
transfer_money("A001", "B002", 50000)

トランザクション設計のヒント

トランザクション内では、ビジネスロジック上「同時に成功すべき操作」をまとめます。上記の例では、引き落としと入金の両方が成功するか、両方が失敗するかのどちらかになります。

7.5 autocommitモード

psycopg2では、autocommitモードを有効にすることもできます。このモードでは、各SQL文が実行されるたびに自動的にコミットされます。

autocommit_example.py 
import psycopg2

conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

# autocommitモードを有効化
conn.autocommit = True

cursor = conn.cursor()

# この操作は即座にコミットされる
cursor.execute("INSERT INTO logs (message) VALUES (%s)", ("ログメッセージ",))

print("autocommitモードで挿入しました（commit不要）")

cursor.close()
conn.close()

明示的トランザクション（推奨）

  • 複数の操作をまとめて管理できる
  • エラー発生時に簡単にロールバック可能
  • 業務処理の境界が明確

autocommitモード（限定的）

  • 単一操作の場合に便利
  • DDL文（CREATE、ALTER等）に適している
  • 複数操作の一貫性が保証されない

注意

autocommitモードでは、エラー発生時に変更を取り消すことができません。複数の操作をまとめて管理する必要がある場合は、明示的なトランザクション管理を使用してください。

7.6 トランザクション分離レベル（発展）

PostgreSQLでは、トランザクションの分離レベルを設定することができます。これは、複数のトランザクションが同時実行される際の動作を制御します。

分離レベル 説明 用途
READ UNCOMMITTED コミットされていない変更も読み取る PostgreSQLでは未サポート
READ COMMITTED コミット済みの変更のみ読み取る（デフォルト） 一般的なWebアプリケーション
REPEATABLE READ トランザクション中は同じデータを読み取る レポート生成、バッチ処理
SERIALIZABLE 完全な直列化（最も厳格） 金融取引など高い一貫性が必要な処理
isolation_level.py 
import psycopg2
from psycopg2 import extensions

conn = psycopg2.connect(
    host="localhost",
    database="testdb",
    user="postgres",
    password="password"
)

# 分離レベルをREPEATABLE READに設定
conn.set_isolation_level(extensions.ISOLATION_LEVEL_REPEATABLE_READ)

cursor = conn.cursor()

# トランザクション内の処理
cursor.execute("SELECT * FROM products")
# ... 処理 ...

conn.commit()
cursor.close()
conn.close()

初心者は、まずデフォルトのREAD COMMITTEDを使用することをお勧めします。より高度な要件がある場合に、他の分離レベルを検討しましょう。

実習 7-1: トランザクション管理を実装しよう

在庫管理システムを想定し、「商品の受注」と「在庫の減少」をトランザクションで管理する関数を作成しましょう。

手順

  1. 1 ordersテーブル（注文）とproductsテーブル（在庫）を用意する
  2. 2 create_order(product_id, quantity)関数を作成し、INSERT（注文）とUPDATE（在庫減少）をトランザクション内で実行
  3. 3 在庫が不足している場合はrollback()を実行し、エラーメッセージを表示
  4. 4 正常に処理できた場合はcommit()を実行し、成功メッセージを表示

第7章のまとめ

  • トランザクションは、複数の操作をまとめて成功・失敗を管理する仕組みで、ACID特性を保証する
  • conn.commit()で変更を確定し、conn.rollback()で変更を取り消す
  • psycopg2ではデフォルトで自動コミットが無効なので、明示的にcommit()を呼び出す必要がある
  • autocommitモードは単一操作に便利だが、複雑な処理には明示的なトランザクション管理が推奨される
  • トランザクション分離レベルで、同時実行時の動作を制御できる（デフォルトはREAD COMMITTED）

理解度確認クイズ

  1. Q1. トランザクションのACID特性のうち、「原子性（Atomicity）」とは何ですか？

    回答を表示

    原子性とは、トランザクション内のすべての操作が成功するか、すべて失敗するかのどちらかであるという特性です。一部だけ成功して一部が失敗するという中途半端な状態にはなりません。例えば、銀行の送金処理では「引き落とし」と「入金」の両方が成功するか、両方が失敗するかのどちらかになります。

  2. Q2. psycopg2で変更をデータベースに確定するメソッドは何ですか？

    回答を表示

    conn.commit()メソッドを使用します。psycopg2ではデフォルトで自動コミットが無効なので、INSERT、UPDATE、DELETEなどのデータ変更操作を行った後は、必ずcommit()を呼び出す必要があります。

  3. Q3. エラーが発生した場合に、トランザクション内のすべての変更を取り消すメソッドは何ですか？

    回答を表示

    conn.rollback()メソッドを使用します。これにより、トランザクション開始以降のすべての変更が取り消され、データベースは変更前の状態に戻ります。エラーハンドリングのexceptブロック内でrollback()を呼び出すのが一般的なパターンです。

  4. Q4. autocommitモードが適している処理と適していない処理を1つずつ挙げてください。

    回答を表示

    適している処理: 単一のINSERT操作や、DDL文（CREATE TABLE、ALTER TABLEなど）
    適していない処理: 複数の操作をまとめて管理する必要がある処理（例: 送金処理、在庫管理など）。autocommitモードでは、エラー発生時に一部の変更だけが確定されてしまい、データの整合性が保たれない可能性があります。
