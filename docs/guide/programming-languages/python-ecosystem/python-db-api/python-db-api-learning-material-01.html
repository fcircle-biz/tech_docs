Python DB-APIとは - データベース接続の標準仕様を知ろう

Python DB-API（PEP 249）の概要と、psycopg2を使ったPostgreSQL連携の基礎を学びます。なぜ標準仕様が必要なのか、どのように活用するのかを理解しましょう。

この章で学ぶこと

  • Python DB-API 2.0とは何か、その目的と標準仕様の概要
  • psycopg2ライブラリの特徴とPostgreSQL連携のメリット
  • 他のデータベースアダプタとの関係と移植性の重要性

1.1 Python DB-APIとは何か

Python DB-API（Database API）とは、Pythonからデータベースにアクセスするための標準インターフェース仕様です。正式名称は「PEP 249 - Python Database API Specification v2.0」で、Pythonの改善提案（PEP: Python Enhancement Proposal）として定義されています。

この仕様により、異なるデータベース（PostgreSQL、MySQL、Oracle、SQLiteなど）に対して、共通のインターフェースでアクセスできるようになります。つまり、データベースの種類を変更しても、Pythonコードの基本構造をほとんど変えずに済むのです。

1.2 なぜDB-API仕様が必要なのか

DB-API仕様がない場合、各データベースごとに異なるライブラリと異なるコード書き方を覚える必要があります。これでは以下のような問題が発生します。

標準仕様がない場合の問題

  • データベースごとに全く異なるコードを書く必要がある
  • 移植性が低く、DBの切り替えが困難
  • 学習コストが高い

DB-API仕様のメリット

  • 共通のインターフェースで複数DBに対応
  • コードの移植性が高い
  • 学習した知識を他のDBでも活用できる

1.3 psycopg2とは

psycopg2は、PythonからPostgreSQLデータベースに接続するための最も人気のあるライブラリです。DB-API 2.0仕様に完全準拠しており、以下の特徴があります。

psycopg2の主な特徴

  • 1
    DB-API 2.0準拠: 標準仕様に従っているため、他のDBアダプタと共通の書き方ができる
  • 2
    高速・効率的: C言語で実装されたコア部分により高いパフォーマンスを実現
  • 3
    PostgreSQL固有機能への対応: JSON型、配列型、トランザクション分離レベルなどPostgreSQL特有の機能をサポート
  • 4
    スレッドセーフ: マルチスレッド環境でも安全に使用可能
simple_example.py 
import psycopg2

# PostgreSQLに接続（詳細は第3章で解説）
conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)

# カーソルを作成してSQLを実行
cursor = conn.cursor()
cursor.execute("SELECT version();")
version = cursor.fetchone()
print(f"PostgreSQLバージョン: {version[0]}")

# 接続をクローズ
cursor.close()
conn.close()

このコードの詳細は次の章以降で学びますが、DB-APIの基本的な流れ（接続 → カーソル作成 → SQL実行 → 切断）を理解しておきましょう。

1.4 他のデータベースアダプタとの関係

DB-API 2.0仕様に準拠したライブラリは他にも多数存在します。以下は主要なデータベースとそのPythonアダプタの対応表です。

データベース Pythonアダプタ インストールコマンド
PostgreSQL psycopg2 pip install psycopg2
MySQL mysql-connector-python pip install mysql-connector-python
SQLite sqlite3（標準ライブラリ） インストール不要
Oracle cx_Oracle pip install cx_Oracle
Microsoft SQL Server pyodbc pip install pyodbc

これらすべてのライブラリがDB-API 2.0に準拠しているため、基本的なコード構造（connect、cursor、execute、fetchなど）は共通です。データベースを変更する場合も、接続部分を書き換えるだけで済む場合が多いのです。

1.5 DB-API 2.0の主要な概念

DB-API 2.0では、以下の主要な概念を理解することが重要です。

Connection（コネクション）

データベースとの接続を表すオブジェクト。トランザクションの開始・コミット・ロールバックを管理します。

Cursor（カーソル）

SQLクエリの実行と結果の取得を行うオブジェクト。Connectionから作成します。

Transaction（トランザクション）

データベース操作の一連の処理をまとめたもの。すべて成功するか、すべて失敗するかのどちらか（ACID特性）。

第1章のまとめ

  • Python DB-API 2.0は、Pythonから各種データベースにアクセスするための標準インターフェース仕様（PEP 249）である
  • psycopg2は、PostgreSQL用のDB-API準拠ライブラリで、高速かつPostgreSQL固有機能もサポートしている
  • DB-API準拠により、異なるデータベース間でのコード移植性が高まり、学習コストも削減される
  • Connection（接続）とCursor（カーソル）がDB-APIの中心概念である

理解度確認クイズ

  1. Q1. Python DB-API 2.0の主な目的は何ですか？

    回答を表示

    異なるデータベース（PostgreSQL、MySQL、Oracleなど）に対して、共通のインターフェースでアクセスできるようにすることが主な目的です。これにより、コードの移植性が高まり、データベースを変更する際のコスト削減や学習効率の向上が図れます。

  2. Q2. psycopg2の特徴を3つ挙げてください。

    回答を表示

    1. DB-API 2.0準拠: 標準仕様に従っているため、他のDBアダプタと共通の書き方ができる
    2. 高速・効率的: C言語で実装されたコア部分により高いパフォーマンスを実現
    3. PostgreSQL固有機能への対応: JSON型、配列型、トランザクション分離レベルなどをサポート

  3. Q3. Connectionオブジェクトの役割は何ですか？

    回答を表示

    データベースとの接続を表すオブジェクトで、トランザクションの管理（開始・コミット・ロールバック）を行います。Connectionからカーソルを作成し、SQLクエリを実行します。
