エラーハンドリング - 例外処理でエラーに対応する

psycopg2が発生させる各種例外の種類と対処方法を学びます。try-except文を使った堅牢なエラーハンドリングパターンを習得しましょう。

この章で学ぶこと

  • psycopg2の例外階層とそれぞれの意味を理解する
  • try-except文を使った適切なエラーハンドリングの方法
  • 実践的なエラーハンドリングパターンとリトライロジック

9.1 psycopg2の例外階層

psycopg2は、DB-API 2.0の仕様に従った例外階層を提供しています。これにより、発生したエラーの種類を正確に判別し、適切に対処できます。

すべての例外はErrorクラスを継承し、データベース関連エラーはDatabaseErrorを継承します。

重要なポイント

より具体的な例外から先にcatchし、一般的な例外は後で処理します。例えば、IntegrityErrorをcatchした後に、より一般的なDatabaseErrorをcatchします。

9.2 主要な例外の種類と発生原因

psycopg2でよく遭遇する例外について、その発生原因と対処方法を理解しましょう。

例外名 発生原因 対処方法
OperationalError 接続失敗、認証エラー、サーバー停止 接続情報確認、サーバー状態確認、リトライ処理
ProgrammingError SQL構文エラー、存在しないテーブル参照 SQL文の修正、スキーマ確認
IntegrityError 主キー重複、外部キー制約違反、NOT NULL制約違反 データ検証、ユーザーへのエラーメッセージ表示
DataError データ型不一致、数値範囲外 入力データの検証、型変換処理
InternalError トランザクション不整合、DB内部エラー ロールバック、接続再確立

9.3 基本的なエラーハンドリング

Pythonのtry-except文を使って、データベース操作中の例外を適切に処理しましょう。

basic_error_handling.py 
import psycopg2
from psycopg2 import Error

try:
    # データベースに接続
    conn = psycopg2.connect(
        host="localhost",
        database="mydb",
        user="postgres",
        password="password"
    )
    cursor = conn.cursor()

    # SQL実行
    cursor.execute("SELECT * FROM users")

except Error as e:
    # すべてのpsycopg2例外をキャッチ
    print(f"エラーが発生しました: {e}")

finally:
    # 必ずリソースをクローズ
    if 'cursor' in locals():
        cursor.close()
    if 'conn' in locals():
        conn.close()

重要なポイント

  • finally節でリソースを必ずクローズする
  • 変数が存在するかチェックしてからclose()を呼ぶ（初期化前のエラーに対処）

9.4 具体的な例外ごとの処理

複数のexcept節を使って、例外の種類ごとに異なる処理を行うことができます。

specific_error_handling.py 
import psycopg2
from psycopg2 import OperationalError, ProgrammingError, IntegrityError

try:
    conn = psycopg2.connect(
        host="localhost",
        database="mydb",
        user="postgres",
        password="password"
    )
    cursor = conn.cursor()

    # ユーザー挿入（主キー重複の可能性）
    cursor.execute(
        "INSERT INTO users (id, name) VALUES (%s, %s)",
        (1, "Alice")
    )
    conn.commit()

except OperationalError as e:
    print(f"接続エラー: {e}")
    print("データベースサーバーが起動しているか確認してください")

except ProgrammingError as e:
    print(f"SQL構文エラー: {e}")
    print("SQL文を確認してください")

except IntegrityError as e:
    print(f"整合性制約違反: {e}")
    print("重複したIDや不正なデータがないか確認してください")
    if conn:
        conn.rollback()  # ロールバックして状態を戻す

except Exception as e:
    print(f"予期しないエラー: {e}")
    if conn:
        conn.rollback()

finally:
    if 'cursor' in locals():
        cursor.close()
    if 'conn' in locals():
        conn.close()

9.5 withステートメントとエラーハンドリングの組み合わせ

第8章で学んだwithステートメントとtry-exceptを組み合わせることで、より安全で読みやすいコードが書けます。

with_error_handling.py 
import psycopg2
from psycopg2 import IntegrityError, OperationalError

try:
    # withで接続を管理（自動クローズ）
    with psycopg2.connect(
        host="localhost",
        database="mydb",
        user="postgres",
        password="password"
    ) as conn:
        with conn.cursor() as cursor:
            # ユーザー追加
            cursor.execute(
                "INSERT INTO users (name, email) VALUES (%s, %s)",
                ("Bob", "bob@example.com")
            )
            # withの終了時に自動的にcommit

    print("ユーザーを正常に追加しました")

except IntegrityError as e:
    print(f"重複エラー: {e}")

except OperationalError as e:
    print(f"接続エラー: {e}")

# withが自動的にクローズするのでfinally不要

9.6 実践的なエラーハンドリングパターン

実務でよく使うエラーハンドリングパターンを紹介します。

リトライロジック

ネットワークの一時的な問題などに対応するため、失敗時に再試行する処理を実装します。

retry_logic.py 
import psycopg2
from psycopg2 import OperationalError
import time

def connect_with_retry(max_retries=3, delay=2):
    """リトライロジック付き接続関数"""
    for attempt in range(max_retries):
        try:
            conn = psycopg2.connect(
                host="localhost",
                database="mydb",
                user="postgres",
                password="password"
            )
            print("接続成功")
            return conn
        except OperationalError as e:
            print(f"接続失敗 ({attempt + 1}/{max_retries}): {e}")
            if attempt < max_retries - 1:
                print(f"{delay}秒後に再試行...")
                time.sleep(delay)
            else:
                print("最大リトライ回数に達しました")
                raise

# 使用例
conn = connect_with_retry()

トランザクションのセーフポイント

一部の処理が失敗しても、他の処理を継続できるようにする方法です。

savepoint_example.py 
import psycopg2
from psycopg2 import IntegrityError

conn = psycopg2.connect(
    host="localhost",
    database="mydb",
    user="postgres",
    password="password"
)
cursor = conn.cursor()

# 複数ユーザーを追加（失敗したものはスキップ）
users = [
    ("Alice", "alice@example.com"),
    ("Bob", "bob@example.com"),  # 重複の可能性
    ("Charlie", "charlie@example.com")
]

for name, email in users:
    try:
        cursor.execute(
            "INSERT INTO users (name, email) VALUES (%s, %s)",
            (name, email)
        )
        conn.commit()
        print(f"{name} を追加しました")
    except IntegrityError as e:
        conn.rollback()  # 失敗した操作のみロールバック
        print(f"{name} の追加に失敗: {e}")

cursor.close()
conn.close()

9.7 エラーメッセージの詳細取得

例外オブジェクトから詳細情報を取得して、ログ出力やデバッグに活用できます。

error_details.py 
import psycopg2
from psycopg2 import Error

try:
    conn = psycopg2.connect(
        host="localhost",
        database="nonexistent_db",  # 存在しないDB
        user="postgres",
        password="password"
    )
except Error as e:
    # 例外の詳細情報を取得
    print(f"エラータイプ: {type(e).__name__}")
    print(f"エラーメッセージ: {e}")

    # pgcodeで詳細なエラーコードを取得
    if hasattr(e, 'pgcode'):
        print(f"PostgreSQLエラーコード: {e.pgcode}")

    # 診断情報を取得
    if hasattr(e, 'diag'):
        print(f"SQLステート: {e.diag.sqlstate}")
        print(f"メッセージ詳細: {e.diag.message_detail}")

理解度確認クイズ

  1. Q1. IntegrityErrorが発生するのはどのような場合ですか？

    回答を表示

    主キー重複、外部キー制約違反、NOT NULL制約違反など、データベースの整合性制約に違反した場合に発生します。例えば、すでに存在するIDで新しいレコードを挿入しようとした場合などです。

  2. Q2. finally節の役割は何ですか？

    回答を表示

    例外が発生してもしなくても必ず実行されるブロックです。データベース接続やカーソルのクローズなど、リソースの解放処理を記述します。これにより、エラーが発生してもリソースリークを防ぐことができます。

  3. Q3. 複数のexcept節を書く際の順序のルールは何ですか？

    回答を表示

    より具体的な例外から先に記述し、一般的な例外は後に記述します。例えば、IntegrityError → DatabaseError → Exceptionの順です。これは、Pythonが上から順に例外をチェックし、最初にマッチしたexcept節が実行されるためです。
