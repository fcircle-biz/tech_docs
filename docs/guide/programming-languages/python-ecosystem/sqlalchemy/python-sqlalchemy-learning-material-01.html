1.1 SQLAlchemyとは

SQLAlchemyは、Pythonで最も人気の高いデータベースライブラリの一つです。「Python SQL toolkit and Object Relational Mapping (ORM) library」として知られ、データベース操作を効率的かつ安全に行うための包括的なツールセットを提供します。

SQLAlchemyの主な特徴：

  • 柔軟性: 低レベルなSQL操作から高レベルなORM操作まで対応
  • データベース独立性: PostgreSQL、MySQL、SQLite、Oracle等の主要なデータベースをサポート
  • パフォーマンス: 効率的なクエリ実行とコネクション管理
  • セキュリティ: SQLインジェクション対策を含む安全なクエリ構築

1.2 SQLAlchemy CoreとORMの違い

SQLAlchemyは大きく2つのアプローチを提供しています：

SQLAlchemy Core
  • SQLに近い低レベルなアプローチ
  • データベーススキーマとクエリを直接制御
  • 高いパフォーマンスと柔軟性
  • 複雑なクエリに適している
SQLAlchemy ORM
  • オブジェクト指向な高レベルアプローチ
  • Pythonクラスとデータベーステーブルをマッピング
  • 開発効率が高い
  • CRUDアプリケーションに適している

1.4 基本的なデータベース接続

SQLAlchemyでは、Engineオブジェクトを使ってデータベースに接続します。

接続文字列の構成

データベース接続文字列は以下の形式で構成されます：

# 基本的な形式
# dialect+driver://username:password@host:port/database

# SQLiteの例
engine = create_engine('sqlite:///example.db')  # ファイルベース
engine = create_engine('sqlite:///:memory:')   # メモリ内

# PostgreSQLの例
engine = create_engine('postgresql://user:password@localhost:5432/dbname')

# MySQLの例
engine = create_engine('mysql+pymysql://user:password@localhost:3306/dbname')
実習 1-2: SQLiteデータベース接続とテーブル作成

SQLiteデータベースファイルを作成し、基本的なテーブル操作を実行します。

手順
  1. SQLiteデータベースファイルを作成する
  2. テーブルを作成する
  3. データを挿入・取得する
実装例
# basic_connection.py
from sqlalchemy import create_engine, text
import os

# データベースファイルの作成
DATABASE_URL = 'sqlite:///learning.db'
engine = create_engine(DATABASE_URL)

print(f"Database engine created: {engine}")
print(f"Database URL: {DATABASE_URL}")

# 接続テストとテーブル作成
with engine.connect() as connection:
    # テーブル作成
    connection.execute(text("""
        CREATE TABLE IF NOT EXISTS users (
            id INTEGER PRIMARY KEY,
            name TEXT NOT NULL,
            email TEXT UNIQUE NOT NULL,
            created_at DATETIME DEFAULT CURRENT_TIMESTAMP
        )
    """))
    
    # テストデータの挿入
    connection.execute(text("""
        INSERT OR IGNORE INTO users (name, email) 
        VALUES ('Alice', 'alice@example.com')
    """))
    
    connection.execute(text("""
        INSERT OR IGNORE INTO users (name, email) 
        VALUES ('Bob', 'bob@example.com')
    """))
    
    # トランザクションのコミット
    connection.commit()
    
    # データの取得
    result = connection.execute(text("SELECT * FROM users"))
    users = result.fetchall()
    
    print("\n--- Users Table Contents ---")
    for user in users:
        print(f"ID: {user[0]}, Name: {user[1]}, Email: {user[2]}, Created: {user[3]}")

# データベースファイルの確認
if os.path.exists('learning.db'):
    file_size = os.path.getsize('learning.db')
    print(f"\nDatabase file created successfully: learning.db ({file_size} bytes)")

1.5 SQLAlchemyの基本構成要素

SQLAlchemyの主要なコンポーネントを理解しましょう：

Engine

データベースとの接続を管理するオブジェクトです。

from sqlalchemy import create_engine
engine = create_engine('sqlite:///example.db')
Connection

実際のデータベース接続を表すオブジェクトです。

with engine.connect() as conn:
    result = conn.execute(text("SELECT 1"))
MetaData

データベーススキーマの情報を保持するオブジェクトです。

from sqlalchemy import MetaData
metadata = MetaData()
Table

データベーステーブルを表現するオブジェクトです。

from sqlalchemy import Table, Column, Integer
users = Table('users', metadata,
    Column('id', Integer, primary_key=True))
重要な注意点
  • バージョン差異: SQLAlchemy 2.0では文法が大きく変更されています。最新の2.0記法を使用することを推奨します。
  • 接続管理: 接続は適切にクローズする必要があります。context manager（with文）の使用を推奨します。
  • SQLインジェクション対策: 生のSQL文字列を使用する際は、必ずtext()でラップしてください。
理解度確認クイズ
  1. SQLAlchemy CoreとORMの主な違いは何ですか？どのような場面で使い分けますか？
  2. 以下のコードの問題点を指摘し、修正してください： 
    engine = create_engine('sqlite:///test.db')
connection = engine.connect()
result = connection.execute("SELECT * FROM users WHERE name = '" + user_input + "'")
print(result.fetchall())
  3. SQLiteを使用する利点と、本番環境での考慮事項を3つずつ挙げてください。
  4. create_engine()の接続文字列でPostgreSQLに接続する場合の形式を書いてください。
  5. なぜwith文を使った接続管理が推奨されるのか説明してください。
解答例

1. Core: SQL中心の低レベルAPI、高いパフォーマンス、複雑なクエリに適する。ORM: オブジェクト指向の高レベルAPI、開発効率が高い、CRUDアプリケーションに適する。

2. SQLインジェクション脆弱性、接続が適切にクローズされていない。修正例：

with engine.connect() as connection:
    result = connection.execute(text("SELECT * FROM users WHERE name = :name"), {"name": user_input})

3. 利点：軽量、設定不要、ファイルベース。考慮事項：同時接続制限、本格的なセキュリティ機能なし、レプリケーション未対応。

4. 'postgresql://username:password@hostname:5432/database_name'

5. 自動的な接続のクローズ、例外発生時の適切なリソース解放、メモリリークの防止のため。
