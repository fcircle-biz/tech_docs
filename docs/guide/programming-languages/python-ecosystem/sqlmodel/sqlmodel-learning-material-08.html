実践！TODOアプリケーションの開発

これまでに学んだ知識を総動員して、実用的なTODOリストアプリケーションを作成します。ユーザー管理、タスク管理、カテゴリー分けなど、実際のアプリケーションに必要な機能を実装します。

この章で学ぶこと

  • 実用的なアプリケーションの要件定義から設計までの流れを理解する
  • ユーザー、タスク、カテゴリーの3つのモデルを設計・実装する
  • 複数のテーブル間のリレーションを実際のアプリで活用する
  • CRUD操作を組み合わせて完全なアプリケーションを構築する

8.1 TODOアプリの要件定義と設計

実際のアプリケーション開発では、いきなりコードを書き始めるのではなく、 何を作りたいのかを明確にすることが大切です。 今回作成するTODOアプリの要件を整理してみましょう。

アプリケーションの基本機能

ユーザー管理

  • • ユーザー登録
  • • ユーザー情報表示
  • • ユーザー一覧

タスク管理

  • • タスク作成
  • • タスク完了/未完了
  • • タスク削除

カテゴリー

  • • カテゴリー作成
  • • タスク分類
  • • カテゴリー別表示

データモデルの関連図

これらの機能を実現するために、3つのテーブルを作成し、それぞれの関連を定義します。

erDiagram USER ||--o{ TASK : "作成する" CATEGORY ||--o{ TASK : "分類する" USER { int id PK string name string email datetime created_at } CATEGORY { int id PK string name string color datetime created_at } TASK { int id PK string title string description bool is_completed int user_id FK int category_id FK datetime created_at datetime updated_at }

リレーションの意味

USER ||--o{ TASK: 1人のユーザーは複数のタスクを作成できます（1対多）
CATEGORY ||--o{ TASK: 1つのカテゴリーには複数のタスクが属します（1対多）

8.2 データモデルの設計と作成

設計が決まったら、SQLModelでデータモデルを定義していきます。 まずはプロジェクトフォルダを作成し、ファイル構成を整えましょう。

プロジェクト構成

ファイル構成 
todo_app/
├── models.py      # データモデル定義
├── database.py    # データベース接続設定
├── main.py        # メインプログラム
└── todo.db        # データベースファイル（自動作成）

データベース接続設定

database.py 
# データベース接続を管理するファイル
from sqlmodel import create_engine, SQLModel

# データベースファイルのパス（SQLiteを使用）
DATABASE_URL = "sqlite:///todo.db"

# エンジンを作成（エコー=Trueでデバッグ用にSQL文を表示）
engine = create_engine(DATABASE_URL, echo=True)

def create_db_and_tables():
    """すべてのテーブルを作成する関数"""
    SQLModel.metadata.create_all(engine)

モデル定義の全体像

models.py 
# 3つのモデルを定義
from sqlmodel import SQLModel, Field, Relationship
from typing import Optional, List
from datetime import datetime

# ユーザーモデル
class User(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    name: str = Field(max_length=50)
    email: str = Field(max_length=100, unique=True)
    created_at: datetime = Field(default_factory=datetime.now)

    # リレーション: このユーザーが作成したタスク一覧
    tasks: List["Task"] = Relationship(back_populates="user")

# カテゴリーモデル
class Category(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    name: str = Field(max_length=30)
    color: str = Field(max_length=7, default="#3b82f6")
    created_at: datetime = Field(default_factory=datetime.now)

    # リレーション: このカテゴリーに属するタスク一覧
    tasks: List["Task"] = Relationship(back_populates="category")

8.3 タスクモデルの実装

タスクモデルは、ユーザーとカテゴリーの両方との関連を持つ中心的な存在です。 外部キーを使って、他のテーブルとの関連を定義します。

models.py（タスクモデル追加） 
# タスクモデル（上記のUserとCategoryの後に追加）
class Task(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    title: str = Field(max_length=100)
    description: Optional[str] = Field(default=None, max_length=500)
    is_completed: bool = Field(default=False)

    # 外部キー: このタスクを作成したユーザーのID
    user_id: int = Field(foreign_key="user.id")

    # 外部キー: このタスクが属するカテゴリーのID
    category_id: int = Field(foreign_key="category.id")

    created_at: datetime = Field(default_factory=datetime.now)
    updated_at: datetime = Field(default_factory=datetime.now)

    # リレーション: このタスクの作成者
    user: User = Relationship(back_populates="tasks")

    # リレーション: このタスクのカテゴリー
    category: Category = Relationship(back_populates="tasks")

外部キーとRelationshipの役割

外部キー（user_id, category_id）: データベースレベルで他のテーブルとの関連を保証
Relationship: Pythonコード上でオブジェクトとして簡単にアクセスできるようにする

8.4 アプリケーションのメイン処理

モデル定義ができたら、実際にデータを操作する機能を実装します。 ユーザー登録からタスク作成まで、一連の流れを作っていきましょう。

初期設定とデータ準備

main.py（基本構造） 
# TODOアプリのメインプログラム
from sqlmodel import Session, select
from database import engine, create_db_and_tables
from models import User, Category, Task

def init_app():
    """アプリケーションの初期化"""
    print("TODOアプリケーションを起動します...")
    create_db_and_tables()
    print("データベースを準備しました！")

def create_sample_data():
    """サンプルデータを作成"""
    with Session(engine) as session:
        # ユーザーを作成
        user = User(name="山田太郎", email="yamada@example.com")
        session.add(user)
        session.commit()
        session.refresh(user)

        # カテゴリーを作成
        work = Category(name="仕事", color="#ef4444")
        personal = Category(name="個人", color="#3b82f6")
        session.add_all([work, personal])
        session.commit()

        print(f"ユーザー '{user.name}' を作成しました")
        print(f"カテゴリー '{work.name}', '{personal.name}' を作成しました")

タスク作成機能

main.py（タスク作成関数） 
def create_task(user_id: int, category_id: int, title: str, description: str = None):
    """新しいタスクを作成"""
    with Session(engine) as session:
        task = Task(
            title=title,
            description=description,
            user_id=user_id,
            category_id=category_id
        )
        session.add(task)
        session.commit()
        session.refresh(task)
        print(f"タスク '{task.title}' を作成しました")
        return task

8.5 タスク一覧表示機能

作成したタスクを見やすく表示する機能を実装します。 リレーションを活用して、タスクと一緒にユーザー名やカテゴリー名も表示します。

main.py（タスク一覧表示） 
def show_all_tasks():
    """すべてのタスクを表示"""
    with Session(engine) as session:
        # すべてのタスクを取得
        tasks = session.exec(select(Task)).all()

        if not tasks:
            print("タスクがありません")
            return

        print("\n=== タスク一覧 ===")
        for task in tasks:
            status = "✓" if task.is_completed else "○"
            print(f"{status} [{task.category.name}] {task.title}")
            print(f"   作成者: {task.user.name}")
            if task.description:
                print(f"   詳細: {task.description}")
            print()

def show_user_tasks(user_id: int):
    """特定ユーザーのタスクを表示"""
    with Session(engine) as session:
        user = session.get(User, user_id)
        if not user:
            print("ユーザーが見つかりません")
            return

        print(f"\n=== {user.name} のタスク一覧 ===")
        for task in user.tasks:
            status = "完了" if task.is_completed else "未完了"
            print(f"[{status}] {task.title} ({task.category.name})")

8.6 タスク完了と削除機能

タスクの状態を更新したり、不要なタスクを削除したりする機能を追加します。

main.py（更新・削除機能） 
def complete_task(task_id: int):
    """タスクを完了にする"""
    with Session(engine) as session:
        task = session.get(Task, task_id)
        if not task:
            print("タスクが見つかりません")
            return

        task.is_completed = True
        task.updated_at = datetime.now()
        session.add(task)
        session.commit()
        print(f"タスク '{task.title}' を完了にしました")

def delete_task(task_id: int):
    """タスクを削除する"""
    with Session(engine) as session:
        task = session.get(Task, task_id)
        if not task:
            print("タスクが見つかりません")
            return

        title = task.title
        session.delete(task)
        session.commit()
        print(f"タスク '{title}' を削除しました")

database.py, models.py, main.py を作成し、上記のコードをコピー

8.7 応用機能の追加アイデア

基本機能ができたら、さらに便利な機能を追加してみましょう。 ここでは、実際のアプリケーションでよく使われる機能のアイデアを紹介します。

期限機能

タスクに期限（due_date）フィールドを追加し、期限切れタスクを検索できるようにする

due_date: Optional[date] = None

優先度設定

タスクに優先度（高・中・低）を設定し、優先度順にソートして表示する

priority: int = Field(default=0)

検索機能

タイトルやキーワードでタスクを検索できる機能を追加する

Task.title.contains("会議")

統計情報

完了率、カテゴリー別タスク数など、統計情報を表示する

count(), group_by()

よくある問題と解決方法

外部キー制約エラー

原因: 存在しないuser_idやcategory_idを指定している
解決: session.get()で事前に存在確認をする

タスクにアクセスできない

原因: セッション外でRelationshipにアクセスしている
解決: with Session()ブロック内でアクセスする

データが更新されない

原因: session.commit()を忘れている
解決: 変更後に必ずcommit()を呼ぶ

理解度確認クイズ

  1. Q1. TODOアプリで使った3つのモデルとその関係を説明してください。

    回答を表示

    User（ユーザー）: タスクの作成者を管理
    Category（カテゴリー）: タスクの分類を管理
    Task（タスク）: 実際のTODO項目を管理
    関係: 1人のユーザーは複数のタスクを作成でき、1つのカテゴリーには複数のタスクが属します（両方とも1対多の関係）

  2. Q2. タスクモデルで外部キーを2つ定義した理由は何ですか？

    回答を表示

    タスクは「誰が作成したか」（user_id）と「どのカテゴリーに属するか」（category_id）の2つの情報を持つ必要があります。 外部キーを使うことで、データベースレベルでこれらの関連性を保証し、存在しないユーザーやカテゴリーへの参照を防ぐことができます。

  3. Q3. Relationshipを使うメリットは何ですか？

    回答を表示

    task.user.nameのように、関連するオブジェクトに直接アクセスできます。 外部キーだけの場合はIDしか取得できませんが、Relationshipを使えば、自動的に関連データを取得してPythonオブジェクトとして扱えるため、コードが簡潔で読みやすくなります。

  4. Q4. 実際のアプリケーション開発で最初にやるべきことは何ですか？

    回答を表示

    要件定義と設計です。いきなりコードを書くのではなく、「どんな機能が必要か」「データをどう管理するか」を明確にすることが重要です。 ER図などを使ってデータモデルの関連を可視化すると、実装がスムーズに進みます。
