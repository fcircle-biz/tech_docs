第5章

データの検索と絞り込み

条件を指定してデータを検索する方法、並び替え、件数制限など、実用的なデータ操作を学びます。

この章で学ぶこと

  • SQLModelでデータを検索するための基本的なselect文の書き方を理解する
  • where句を使って特定の条件に合うデータだけを取得する方法を習得する
  • order_byで結果を並び替える方法、limitで件数を制限する方法を学ぶ
  • 複数の条件を組み合わせた実用的な検索クエリを実装できるようになる

5.1 なぜ検索と絞り込みが必要なのか

前章では、データベースに保存されたすべてのデータを取得する方法を学びました。 しかし、実際のアプリケーションでは「すべて」ではなく「条件に合うデータだけ」を取得したい場面がほとんどです。

実際の検索例

ECサイトでは「価格が1000円以下の商品」、ユーザー管理では「20歳以上のユーザー」、 ブログシステムでは「最新10件の記事」など、条件を指定したデータ取得が必要です。

5.2 select文の基本的な書き方

SQLModelでデータを検索するには、select関数を使います。 これは「データベースからデータを選択する」ための基本的な命令です。

最もシンプルな検索

まず、第4章で学んだ全件取得の復習から始めましょう。

基本的なselect文 
from sqlmodel import Session, select
from models import User  # Userモデルがあると仮定

# データベースに接続
with Session(engine) as session:
    # select文を作成
    statement = select(User)

    # 実行して結果を取得
    users = session.exec(statement).all()

    # 結果を表示
    for user in users:
        print(f"{user.name} ({user.age}歳)")

select文の基本構造

1. select(モデル): 何のデータを取得するか指定
2. session.exec(statement): 実際に検索を実行
3. .all(): すべての結果をリストで取得

5.3 where句で条件を指定する

where句を使うと、「この条件に合うデータだけ」という絞り込みができます。 SQLModelでは、whereメソッドを使って条件を追加します。

シンプルな条件検索

まずは「年齢が20歳以上のユーザー」を検索してみましょう。

where句の基本例 
from sqlmodel import Session, select
from models import User

with Session(engine) as session:
    # 年齢が20歳以上のユーザーを検索
    statement = select(User).where(User.age >= 20)

    # 実行
    users = session.exec(statement).all()

    for user in users:
        print(f"{user.name} ({user.age}歳)")

# 出力例:
# 田中太郎 (25歳)
# 佐藤花子 (30歳)

比較演算子の使い方

== 等しい
!= 等しくない
> より大きい
>= 以上
< より小さい
<= 以下

複数の条件を組み合わせる

「AかつB」「AまたはB」のような複雑な条件も指定できます。

複数条件の組み合わせ 
from sqlmodel import Session, select, and_, or_
from models import User

with Session(engine) as session:
    # 「年齢が20歳以上かつ30歳未満」のユーザー
    statement = select(User).where(
        and_(User.age >= 20, User.age < 30)
    )
    users = session.exec(statement).all()

    # または、複数のwhereを繋げても同じ（AND条件）
    statement = select(User).where(User.age >= 20).where(User.age < 30)
    users = session.exec(statement).all()

    # 「年齢が20歳未満または40歳以上」のユーザー
    statement = select(User).where(
        or_(User.age < 20, User.age >= 40)
    )
    users = session.exec(statement).all()

5.4 order_byで結果を並び替える

検索結果を特定の順番で取得したい場合、order_byを使います。 「価格の安い順」「新しい順」など、用途に応じて並び替えができます。

昇順・降順の並び替え 
from sqlmodel import Session, select, col
from models import Product  # Productモデルがあると仮定

with Session(engine) as session:
    # 価格の安い順（昇順）
    statement = select(Product).order_by(Product.price)
    products = session.exec(statement).all()

    # 価格の高い順（降順）
    statement = select(Product).order_by(col(Product.price).desc())
    products = session.exec(statement).all()

    # 複数の並び順を指定（価格の安い順、同じ価格なら名前順）
    statement = select(Product).order_by(
        Product.price,
        Product.name
    )
    products = session.exec(statement).all()

昇順と降順

昇順（ASC）: 小さい値から大きい値へ（デフォルト）
降順（DESC）: 大きい値から小さい値へ（.desc()を使う）

5.5 limitで取得件数を制限する

検索結果が大量にある場合、「最初の10件だけ」のように件数を制限したいことがあります。 limitを使うと、取得するデータの数を指定できます。

件数制限の例 
from sqlmodel import Session, select, col
from models import Article  # Articleモデルがあると仮定

with Session(engine) as session:
    # 最新10件の記事を取得
    statement = select(Article).order_by(
        col(Article.created_at).desc()
    ).limit(10)
    articles = session.exec(statement).all()

    # offsetと組み合わせてページネーション
    # 11件目から20件目を取得（2ページ目）
    statement = select(Article).order_by(
        col(Article.created_at).desc()
    ).offset(10).limit(10)
    articles = session.exec(statement).all()

ページネーションとは

大量のデータを「ページ1、ページ2...」のように分割して表示する仕組みです。 limitで1ページの件数を、offsetで何件目から取得するかを指定します。

5.6 実用的な検索の組み合わせ

ここまで学んだwhere、order_by、limitを組み合わせて、実際のアプリケーションで使えるような検索を実装してみましょう。

例1: ECサイトの商品検索

商品検索の実装 
from sqlmodel import Session, select, and_, col
from models import Product

with Session(engine) as session:
    # 「価格が1000円以下で、在庫がある商品」を
    # 「価格の安い順」で「最大20件」取得
    statement = select(Product).where(
        and_(
            Product.price <= 1000,
            Product.stock > 0
        )
    ).order_by(Product.price).limit(20)

    products = session.exec(statement).all()

    for product in products:
        print(f"{product.name}: {product.price}円 (在庫: {product.stock})")

# 出力例:
# ノート: 300円 (在庫: 50)
# ボールペン: 500円 (在庫: 100)
# 消しゴム: 800円 (在庫: 30)

例2: ブログ記事の検索

ブログ記事検索の実装 
from sqlmodel import Session, select, col
from models import Article

with Session(engine) as session:
    # 「公開済み」の記事を「新しい順」で「10件」取得
    statement = select(Article).where(
        Article.is_published == True
    ).order_by(
        col(Article.created_at).desc()
    ).limit(10)

    articles = session.exec(statement).all()

    for article in articles:
        print(f"{article.title} ({article.created_at})")

例3: ユーザー検索（部分一致）

名前の一部で検索したい場合は、like演算子を使います。

部分一致検索 
from sqlmodel import Session, select, col
from models import User

with Session(engine) as session:
    # 名前に「田中」を含むユーザーを検索
    search_keyword = "田中"
    statement = select(User).where(
        col(User.name).like(f"%{search_keyword}%")
    )

    users = session.exec(statement).all()

    for user in users:
        print(user.name)

# 出力例:
# 田中太郎
# 田中花子

実習 5-1: 商品検索機能の実装

学んだ検索機能を使って、実際に商品データベースを検索してみましょう。

手順

  1. 1 以下のようなProductモデルを作成し、テストデータを5件以上登録してください
  2. 2 「価格が500円以上1500円以下」の商品を検索するコードを書いてください
  3. 3 結果を「価格の高い順」で並び替えて表示してください
  4. 4 最大3件のみ表示するように制限してください
サンプル回答を表示 
from sqlmodel import Field, SQLModel, Session, create_engine, select, and_, col

class Product(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    name: str
    price: int
    category: str

engine = create_engine("sqlite:///products.db")
SQLModel.metadata.create_all(engine)

with Session(engine) as session:
    # 検索実行
    statement = select(Product).where(
        and_(Product.price >= 500, Product.price <= 1500)
    ).order_by(col(Product.price).desc()).limit(3)

    products = session.exec(statement).all()

    for product in products:
        print(f"{product.name}: {product.price}円")

理解度確認クイズ

  1. Q1. where句を使う目的は何ですか？

    回答を表示

    特定の条件に合うデータだけを絞り込んで取得するためです。 すべてのデータではなく、「年齢が20歳以上」「価格が1000円以下」などの条件に一致するデータのみを取得できます。

  2. Q2. 「価格の高い順」で並び替えるにはどうすればよいですか？

    回答を表示

    order_by(col(Product.price).desc())を使います。 desc()は「降順（Descending）」を意味し、大きい値から小さい値へと並び替えます。

  3. Q3. limitとoffsetの違いは何ですか？

    回答を表示

    limit: 取得するデータの件数を制限します（例: 10件だけ取得）
    offset: 何件目からデータを取得するかを指定します（例: 11件目から取得）
    両方を組み合わせることで、ページネーション（ページ分割）を実装できます。

  4. Q4. 複数の条件を「かつ」で結合するにはどうすればよいですか？

    回答を表示

    and_(条件1, 条件2)を使うか、 複数のwhere()を繋げることで実現できます。
    例: where(and_(User.age >= 20, User.age < 30))

トラブルシューティング

検索結果が空になってしまう

原因: where条件に合うデータがデータベースに存在していない可能性があります。

解決方法: まず条件なしでselect(モデル).all()を実行し、実際にどんなデータがあるか確認しましょう。その後、条件を緩めて徐々に絞り込んでいくと、どの条件で結果が変わるかわかります。

and_やor_がインポートできない

原因: インポート文にand_やor_が含まれていません。

解決方法: ファイルの先頭で from sqlmodel import and_, or_ を追加してください。

降順にならない（desc()が効かない）

原因: desc()を使うには、colでカラムをラップする必要があります。

解決方法: from sqlmodel import colをインポートし、col(Product.price).desc()のように書いてください。
