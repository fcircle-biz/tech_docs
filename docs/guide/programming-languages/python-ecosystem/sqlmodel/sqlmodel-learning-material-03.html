3.1 モデルとは何か

前章では、SQLModelをインストールして最初のプログラムを動かしました。 この章では、SQLModelの核となる「モデル」という概念を学びます。

モデルとは、データベースに保存するデータの構造を定義したPythonのクラスです。 現実世界の「ユーザー」「商品」「注文」といった概念を、コードで表現する設計図のようなものです。

モデルの役割

モデルは「データベースのテーブル定義」と「Pythonで扱うデータ型」の両方を同時に表現します。 これにより、データベース設計とプログラミングを統一的に行えます。

3.2 最初のモデルを定義する

まずは、シンプルな「ユーザー」モデルを作ってみましょう。 ユーザーは、ID、名前、メールアドレスを持つとします。

user_model.py 
from sqlmodel import SQLModel, Field
from typing import Optional

# ユーザーモデルの定義
class User(SQLModel, table=True):
    """ユーザー情報を保存するモデル"""

    id: Optional[int] = Field(default=None, primary_key=True)
    name: str  # ユーザー名
    email: str  # メールアドレス

コードの解説

class User(SQLModel, table=True)

SQLModelを継承し、table=Trueを指定することで、 このクラスがデータベーステーブルに対応することを示します。

id: Optional[int] = Field(default=None, primary_key=True)

primary_key=TrueでIDを主キーに指定します。 Optional[int]は、最初はNoneで後からデータベースが自動採番することを意味します。

name: str

型ヒントでstrを指定することで、文字列型のフィールドであることを宣言します。 データベースにはVARCHAR型として保存されます。

3.3 基本的なデータ型

SQLModelでは、Pythonの型ヒントを使ってデータ型を指定します。 以下の基本的なデータ型がよく使われます。

Python型 DB型 用途 例
str VARCHAR 文字列 名前、メールアドレス
int INTEGER 整数 年齢、在庫数
float FLOAT 小数 価格、評価スコア
bool BOOLEAN 真偽値 有効/無効、公開/非公開
datetime DATETIME 日時 作成日時、更新日時
date DATE 日付 誕生日、配送日

より複雑な商品モデルの例

複数のデータ型を使った、商品モデルの例を見てみましょう。

product_model.py 
from sqlmodel import SQLModel, Field
from typing import Optional
from datetime import datetime

class Product(SQLModel, table=True):
    """商品情報を保存するモデル"""

    id: Optional[int] = Field(default=None, primary_key=True)
    name: str  # 商品名
    price: float  # 価格（小数点を含む）
    stock: int  # 在庫数
    is_available: bool = True  # 販売可能かどうか
    created_at: datetime = Field(default_factory=datetime.now)  # 登録日時

default_factoryの使い方

default_factory=datetime.nowは、データが作成されるたびに現在時刻を自動的に設定します。 これにより、登録日時を自動記録できます。

3.4 Fieldオプションでデータに制約を設定する

Field()関数を使うと、データに様々な制約を設定できます。 これにより、不正なデータの登録を防ぐことができます。

主なFieldオプション

  • primary_key=True：主キー（一意のID）として設定
  • index=True：検索を高速化するインデックスを作成
  • unique=True：重複を許さない（例：メールアドレス）
  • nullable=False：必須項目（空欄不可）
  • default=値：デフォルト値を設定
  • max_length=数値：文字列の最大長を制限

制約を使った改良版ユーザーモデル

user_improved.py 
from sqlmodel import SQLModel, Field
from typing import Optional

class User(SQLModel, table=True):
    """改良版ユーザーモデル"""

    id: Optional[int] = Field(default=None, primary_key=True)
    name: str = Field(max_length=100)  # 名前は100文字まで
    email: str = Field(unique=True, index=True)  # メールは重複不可、検索高速化
    age: Optional[int] = Field(default=None, ge=0, le=150)  # 年齢は0〜150の範囲
    is_active: bool = Field(default=True)  # デフォルトで有効

数値範囲の制約

ge=0は「0以上」、le=150は「150以下」を意味します。 これにより、年齢に不正な値（マイナスや非現実的な値）が入るのを防げます。

実習 3-1: 書籍管理モデルを作成しよう

図書館の書籍を管理するためのBookモデルを作成してみましょう。

必要なフィールド

  1. 1 id：主キー（整数）
  2. 2 title：タイトル（文字列、最大200文字）
  3. 3 author：著者名（文字列、最大100文字）
  4. 4 isbn：ISBN番号（文字列、重複不可）
  5. 5 published_year：出版年（整数、1000〜現在）
  6. 6 is_available：貸出可能かどうか（真偽値、デフォルトTrue）
解答例を表示 
from sqlmodel import SQLModel, Field
from typing import Optional

class Book(SQLModel, table=True):
    """書籍情報を保存するモデル"""

    id: Optional[int] = Field(default=None, primary_key=True)
    title: str = Field(max_length=200)
    author: str = Field(max_length=100)
    isbn: str = Field(unique=True, index=True)
    published_year: int = Field(ge=1000, le=2100)
    is_available: bool = Field(default=True)

3.5 Optional型の理解

これまでのコードでOptional[int]という記述が何度も登場しました。 これは「値があってもなくても良い（NoneでもOK）」という意味です。

str, int, bool] A --> C[任意フィールド
Optional型] B --> D[値が必ず必要
例: name: str] C --> E[値がNoneでもOK
例: age: Optional型] E --> F[データ作成時に
値を指定しなくてもOK] style A fill:#f59e0b,stroke:#d97706,stroke-width:2px,color:#fff style B fill:#ef4444,stroke:#dc2626,stroke-width:2px,color:#fff style C fill:#10b981,stroke:#047857,stroke-width:2px,color:#fff

使い分けの例

optional_example.py 
from sqlmodel import SQLModel, Field
from typing import Optional

class Profile(SQLModel, table=True):
    """ユーザープロフィール"""

    id: Optional[int] = Field(default=None, primary_key=True)

    # 必須項目（値が必ず必要）
    username: str  # ユーザー名は必須

    # 任意項目（値がなくてもOK）
    bio: Optional[str] = None  # 自己紹介は任意
    website: Optional[str] = None  # Webサイトは任意
    age: Optional[int] = None  # 年齢は任意

注意：主キーは常にOptional型

主キー（id）は、データベースが自動的に値を割り当てるため、常にOptional[int]にします。 作成時にはNone、保存後にデータベースが自動で番号を付けます。

理解度確認クイズ

  1. Q1. モデルとは何ですか？簡潔に説明してください。

    回答を表示

    モデルとは、データベースに保存するデータの構造を定義したPythonのクラスです。 現実世界の概念（ユーザー、商品など）をコードで表現する設計図のような役割を果たします。

  2. Q2. Python型strはデータベースではどの型になりますか？

    回答を表示

    VARCHAR型になります。文字列を可変長で保存するデータベースの型です。

  3. Q3. メールアドレスのように重複を許したくないフィールドには、どのFieldオプションを使いますか？

    回答を表示

    unique=Trueを使います。これにより、同じ値が2つ以上登録されるのを防ぐことができます。 また、index=Trueを併用すると、検索も高速化できます。

  4. Q4. Optional[int]とintの違いは何ですか？

    回答を表示

    int：値が必ず必要（必須フィールド）
    Optional[int]：値がNoneでもOK（任意フィールド）
    主キー（id）やデフォルト値がないフィールドには、Optional型を使います。
