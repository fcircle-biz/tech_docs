2.1 Pythonのバージョン確認

SQLModelを使うには、Python 3.8以上が必要です。 まずは、お使いのPCにPythonがインストールされているか、バージョンは何かを確認しましょう。

ターミナル / コマンドプロンプト 
# Pythonのバージョンを確認
python --version

# または python3 コマンドの場合
python3 --version

期待される結果

Python 3.11.5

Python 3.8以上のバージョンが表示されればOKです。

WindowsとMac/Linuxでのコマンドの違い

Windows: python --version
Mac/Linux: python3 --version
お使いの環境に合わせてコマンドを使い分けてください。

2.3 SQLModelのインストール

仮想環境を有効化したら、次にSQLModelをインストールします。 Pythonのライブラリはpipというツールを使ってインストールします。

ターミナル（仮想環境有効化済み） 
# SQLModelをインストール
pip install sqlmodel

インストール完了の確認

インストールが成功すると、SQLModelと一緒に依存ライブラリ（SQLAlchemy、Pydanticなど）も自動的にインストールされます。

インストールの確認

正しくインストールされたか確認するには、Pythonのインタラクティブシェルで試してみましょう。

ターミナル 
# Pythonインタラクティブシェルを起動
python

# またはMac/Linuxの場合
python3
Pythonシェル内で実行 
# SQLModelをインポート
import sqlmodel
print(sqlmodel.__version__)

# 終了するには
exit()

ヒント

エラーが出なければ、インストール成功です。バージョン番号（例: 0.0.14）が表示されます。

2.4 最初のプログラムを作成する

それでは、SQLModelを使った最初のプログラムを作成しましょう。 シンプルな「ユーザー」モデルを定義し、データベースに保存してみます。

実習 2-2: 最初のSQLModelプログラム

シンプルなユーザーモデルを作成し、データベースに保存します。

手順

  1. 1
    main.pyというファイルを作成し、以下のコードを入力
main.py 
# 必要なモジュールをインポート
from sqlmodel import Field, SQLModel, create_engine

# ユーザーモデルを定義
class User(SQLModel, table=True):
    id: int = Field(primary_key=True)
    name: str
    age: int

# データベースに接続
engine = create_engine("sqlite:///database.db")
SQLModel.metadata.create_all(engine)

print("データベースが作成されました！")

コードの説明

  • from sqlmodel import ...

    SQLModelから必要な機能をインポートしています

  • class User(SQLModel, table=True):

    Userという名前のテーブル（データの型）を定義しています

  • id: int = Field(primary_key=True)

    各ユーザーを識別するための番号（主キー）

  • name: str

    ユーザーの名前（文字列型）

  • create_engine("sqlite:///database.db")

    SQLiteデータベースに接続（ファイル名は database.db）

  • SQLModel.metadata.create_all(engine)

    定義したテーブルを実際にデータベースに作成します

プログラムの実行

ターミナル 
# プログラムを実行
python main.py

動作確認

期待される結果 

データベースが作成されました！

プロジェクトフォルダにdatabase.dbというファイルが作成されているはずです。

2.5 データベースファイルの確認

プログラムを実行すると、database.dbというファイルが作成されます。 このファイルがSQLiteデータベースの実体です。

現在のプロジェクト構成 

sqlmodel-tutorial/
├── venv/              ← 仮想環境フォルダ
├── main.py            ← 今回作成したプログラム
└── database.db        ← 自動生成されたデータベースファイル

データベースの中身を確認する方法

データベースファイルの中身を確認するには、専用のツールを使います。 初心者にはオンラインツール「SQLite Online」がおすすめです。

オンラインツール

SQLite Online

ブラウザから使える無料ツール。database.dbをアップロードして確認できます。

デスクトップツール

DB Browser for SQLite

無料のデスクトップアプリ。視覚的にデータを確認・編集できます。

確認ポイント

データベースの中にuserという名前のテーブルが作成されていれば成功です。 ただし、まだデータは1件も保存していないので、テーブルは空の状態です。

2.6 トラブルシューティング

よくあるエラーと対処法

ModuleNotFoundError: No module named 'sqlmodel'

原因: SQLModelがインストールされていないか、仮想環境が有効化されていません。

解決方法:

  1. 仮想環境が有効化されているか確認（プロンプトに(venv)が表示されているか）
  2. pip install sqlmodelを再度実行
python: command not found

原因: Pythonがインストールされていないか、PATHが通っていません。

解決方法:

  1. Python公式サイトから最新版をダウンロードしてインストール
  2. Mac/Linuxの場合はpython3コマンドを使用
venv\Scripts\activate: cannot be loaded because running scripts is disabled

原因: Windows PowerShellでスクリプト実行が無効になっています。

解決方法:

  1. PowerShellを管理者権限で開く
  2. Set-ExecutionPolicy RemoteSignedを実行
  3. またはコマンドプロンプト（cmd）を使用
database.dbファイルが作成されない

原因: プログラムにエラーがあるか、別のフォルダで実行しています。

解決方法:

  1. エラーメッセージがないか確認
  2. lsまたはdirコマンドで現在のフォルダを確認
  3. main.pyと同じフォルダで実行されているか確認

理解度確認クイズ

  1. Q1. 仮想環境を作成する理由は何ですか？

    回答を表示

    プロジェクトごとに独立したPython環境を作ることで、異なるバージョンのライブラリを必要とするプロジェクト間での競合を防ぐためです。 これにより、あるプロジェクトで使うライブラリのバージョンが、他のプロジェクトに影響を与えることがなくなります。

  2. Q2. create_engine("sqlite:///database.db")の役割は何ですか？

    回答を表示

    SQLiteデータベースへの接続を確立します。 "sqlite:///database.db"は、現在のフォルダにdatabase.dbという名前のSQLiteデータベースファイルを作成（または接続）することを意味します。

  3. Q3. SQLModel.metadata.create_all(engine)は何をしていますか？

    回答を表示

    定義したすべてのモデル（テーブル）を実際にデータベースに作成します。 この場合、Userクラスで定義したuserテーブルがデータベース内に作成されます。

  4. Q4. 仮想環境が有効化されているかどうかは、どうやって確認できますか？

    回答を表示

    ターミナルのプロンプトの先頭に(venv)が表示されるかどうかで確認できます。 これが表示されていれば、仮想環境が有効化されています。
