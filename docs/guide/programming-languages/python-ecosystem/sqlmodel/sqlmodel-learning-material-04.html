データの保存と取得（CRUD操作の基本）

データの新規作成（Create）、読み取り（Read）、更新（Update）、削除（Delete）という基本的な操作を、実際のコードを書きながら習得します。

この章で学ぶこと

  • CRUD操作の意味と、データベース操作の基本的な流れを理解する
  • SQLModelで新しいデータを作成してデータベースに保存する方法を習得する
  • 保存されたデータを取得・表示する方法を学ぶ
  • 既存のデータを更新・削除する手順を理解する

4.1 CRUD操作とは

CRUD（クラッド）とは、データベースを使うアプリケーションで必ず行う4つの基本操作の頭文字を取った言葉です。

操作 意味 具体例
Create（作成） 新しいデータを追加する 新規ユーザー登録
Read（読み取り） データを取得して表示する ユーザー一覧を表示
Update（更新） 既存のデータを変更する プロフィール情報を編集
Delete（削除） データを削除する アカウントを削除

CRUD操作の重要性

SNS、ECサイト、業務システムなど、ほぼすべてのアプリケーションはCRUD操作を組み合わせて作られています。 この4つの操作をマスターすれば、実用的なアプリケーションが作れるようになります。

SQLModelでのCRUD操作の流れ

SQLModelでは、Sessionという仕組みを使ってデータベースとやり取りします。 Sessionは、データベースへの「接続窓口」のようなものです。

4.2 Create - データの新規作成

新しいデータを作成してデータベースに保存するには、以下の3つのステップを踏みます。

flowchart TD A([1. モデルのインスタンスを作成]) --> B([2. Sessionに追加]) B --> C([3. コミットして保存]) style A fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style B fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style C fill:#dbeafe,stroke:#3b82f6,stroke-width:2px

基本的なCreate操作

まず、ユーザーモデルを定義して、新しいユーザーを作成してみましょう。

user_model.py 
from sqlmodel import SQLModel, Field, create_engine, Session

# ユーザーモデルの定義
class User(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    name: str
    email: str
    age: int

# データベース接続
engine = create_engine("sqlite:///database.db")
SQLModel.metadata.create_all(engine)

# 新しいユーザーを作成
new_user = User(name="田中太郎", email="tanaka@example.com", age=25)

Sessionを使ってデータを保存

作成したユーザーオブジェクトをデータベースに保存します。

create_user.py 
# Sessionを使ってデータベースに保存
with Session(engine) as session:
    # ユーザーをSessionに追加
    session.add(new_user)

    # コミット（確定）してデータベースに保存
    session.commit()

    # 保存後、IDが自動で割り当てられる
    print(f"ユーザーが保存されました！ ID: {new_user.id}")
# 実行結果: ユーザーが保存されました！ ID: 1

重要ポイント

commit()を呼び出すまで、データはデータベースに保存されません。 commit()は「この変更を確定してください」という意味です。

複数のデータを一度に保存

複数のユーザーをまとめて作成することもできます。

create_multiple_users.py 
with Session(engine) as session:
    # 複数のユーザーを作成
    users = [
        User(name="佐藤花子", email="sato@example.com", age=30),
        User(name="鈴木一郎", email="suzuki@example.com", age=28),
        User(name="高橋美咲", email="takahashi@example.com", age=22)
    ]

    # まとめて追加
    session.add_all(users)
    session.commit()

    print(f"{len(users)}人のユーザーを保存しました！")

4.3 Read - データの読み取り

保存したデータを取得する方法はいくつかあります。ここでは最も基本的な方法を学びます。

すべてのデータを取得

select()を使って、テーブル内のすべてのデータを取得できます。

read_all_users.py 
from sqlmodel import select

with Session(engine) as session:
    # すべてのユーザーを取得
    statement = select(User)
    users = session.exec(statement).all()

    # 取得したユーザーを表示
    for user in users:
        print(f"ID: {user.id}, 名前: {user.name}, メール: {user.email}")

# 実行結果:
# ID: 1, 名前: 田中太郎, メール: tanaka@example.com
# ID: 2, 名前: 佐藤花子, メール: sato@example.com

IDを指定してデータを取得

特定のIDのデータだけを取得したい場合は、get()を使います。

read_user_by_id.py 
with Session(engine) as session:
    # ID=1のユーザーを取得
    user = session.get(User, 1)

    if user:
        print(f"見つかりました: {user.name}")
    else:
        print("ユーザーが見つかりませんでした")

# 実行結果: 見つかりました: 田中太郎

条件を指定してデータを取得

where()を使って、条件に合うデータだけを取得できます。

read_user_by_condition.py 
with Session(engine) as session:
    # 年齢が25歳以上のユーザーを取得
    statement = select(User).where(User.age >= 25)
    users = session.exec(statement).all()

    print(f"{len(users)}人のユーザーが見つかりました")
    for user in users:
        print(f"- {user.name} ({user.age}歳)")

# 実行結果:
# 2人のユーザーが見つかりました
# - 田中太郎 (25歳)
# - 佐藤花子 (30歳)

ヒント

.all()はすべての結果をリストで返します。 1件だけ取得したい場合は.first()を使います。

4.4 Update - データの更新

既存のデータを変更するには、まずデータを取得してから、値を変更してコミットします。

基本的なUpdate操作

取得したオブジェクトの属性を変更するだけで、更新ができます。

update_user.py 
with Session(engine) as session:
    # ID=1のユーザーを取得
    user = session.get(User, 1)

    if user:
        # メールアドレスを変更
        user.email = "tanaka_new@example.com"

        # コミットして変更を保存
        session.commit()
        print(f"{user.name}のメールアドレスを更新しました")
    else:
        print("ユーザーが見つかりませんでした")
flowchart LR A([1. データを取得]) --> B([2. 値を変更]) B --> C([3. コミット]) style A fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style B fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style C fill:#dbeafe,stroke:#3b82f6,stroke-width:2px

複数の項目を更新

一度に複数の項目を更新することもできます。

update_multiple_fields.py 
with Session(engine) as session:
    user = session.get(User, 1)

    if user:
        # 複数の項目を更新
        user.name = "田中太郎（更新）"
        user.age = 26
        user.email = "tanaka_updated@example.com"

        session.commit()
        print("ユーザー情報を更新しました")

注意

commit()を忘れると、変更がデータベースに保存されません。 必ずcommit()を呼び出しましょう。

4.5 Delete - データの削除

不要になったデータを削除するには、delete()を使います。

基本的なDelete操作

まずデータを取得してから、session.delete()で削除します。

delete_user.py 
with Session(engine) as session:
    # 削除したいユーザーを取得
    user = session.get(User, 1)

    if user:
        # ユーザーを削除
        session.delete(user)
        session.commit()
        print(f"{user.name}を削除しました")
    else:
        print("ユーザーが見つかりませんでした")

削除は慎重に

削除したデータは元に戻せません。実行する前に本当に削除して良いか確認しましょう。 実際のアプリケーションでは、削除前に確認画面を表示することが一般的です。

論理削除という方法

実際のシステムでは、データを完全に削除せず、「削除済みフラグ」を立てる方法がよく使われます。 これを論理削除と呼びます。

logical_delete_example.py 
# 論理削除用のモデル例
class User(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    name: str
    email: str
    is_deleted: bool = False  # 削除フラグ

# 論理削除の実行
with Session(engine) as session:
    user = session.get(User, 1)
    if user:
        user.is_deleted = True  # フラグを立てるだけ
        session.commit()
        print("ユーザーを論理削除しました")

実習 4-1: CRUD操作を実践する

商品管理システムのデータベースを作成し、CRUD操作をすべて実行してみましょう。

手順

  1. 1 Productモデルを作成（name、price、stock）
  2. 2 3つの商品を作成して保存（Create）
  3. 3 すべての商品を取得して表示（Read）
  4. 4 1つの商品の価格を変更（Update）
  5. 5 1つの商品を削除（Delete）
ヒントを表示

Productモデルの定義例：

class Product(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    name: str
    price: int
    stock: int

理解度確認クイズ

  1. Q1. CRUD操作の4つの意味を説明してください。

    回答を表示

    Create（作成）: 新しいデータを追加する
    Read（読み取り）: データを取得して表示する
    Update（更新）: 既存のデータを変更する
    Delete（削除）: データを削除する

  2. Q2. commit()を呼び出すタイミングはいつですか？

    回答を表示

    データの作成、更新、削除を行った後に呼び出します。 commit()を呼び出すことで、変更がデータベースに確定されます。 commit()を忘れると、変更は保存されません。

  3. Q3. すべてのデータを取得するには、どのメソッドを使いますか？

    回答を表示

    select()とall()を組み合わせます。
    例: session.exec(select(User)).all()
    select()でクエリを作成し、exec()で実行、all()ですべての結果をリストで取得します。

  4. Q4. 論理削除と物理削除の違いは何ですか？

    回答を表示

    物理削除: データベースからデータを完全に削除する（元に戻せない）
    論理削除: データは残したまま、「削除済みフラグ」を立てて見えなくする（復元可能）
    実際のシステムでは、データを残しておきたい場合が多いため、論理削除がよく使われます。

トラブルシューティング

データが保存されない

原因: commit()を呼び出していない可能性があります。

解決方法: session.add()の後に、必ずsession.commit()を呼び出してください。

「No such table」エラーが出る

原因: テーブルが作成されていません。

解決方法: SQLModel.metadata.create_all(engine)を実行して、テーブルを作成してください。

取得したデータがNoneになる

原因: 指定したIDのデータが存在しないか、条件に合うデータがありません。

解決方法: データ取得後にif文でNoneチェックを行い、データが存在しない場合の処理を追加してください。

この章のまとめ

  • CRUDは、Create（作成）、Read（読み取り）、Update（更新）、Delete（削除）の4つの基本操作です
  • Sessionを使ってデータベースとやり取りし、commit()で変更を確定します
  • Createは、モデルのインスタンスを作成し、session.add()とcommit()で保存します
  • Readは、select()とexec()を組み合わせてデータを取得します
  • Updateは、データを取得して値を変更し、commit()で保存します
  • Deleteは、session.delete()とcommit()でデータを削除します
