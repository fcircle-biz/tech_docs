プラグインの活用 - pytestの機能を拡張する

pytest-cov（カバレッジ計測）、pytest-xdist（並列実行）、pytest-randomly（ランダム実行順）、pytest-mock（モックの簡略化）、pytest-html（HTMLレポート生成）など、実務で役立つ主要プラグインの導入と活用方法を学びます。

この章で学ぶこと

  • pytestのプラグインエコシステムの全体像と、プラグインがどのように機能を拡張するかを理解する
  • pytest-covをインストールしてコードカバレッジを計測し、レポートを生成できる
  • pytest-xdistを使ってテストを複数CPUコアで並列実行し、実行時間を短縮できる
  • pytest-mockを使ってモックをよりシンプルに記述できる
  • pytest-htmlでHTMLレポートを生成し、テスト結果をチームと共有できる

11.1 pytestのプラグインエコシステム

pytestは「コア機能 + プラグイン」という設計になっています。コア機能だけでも十分に使えますが、プラグインを追加することで、カバレッジ計測・並列実行・HTMLレポートなど、実務に欠かせない機能を簡単に追加できます。

プラグインはpipでインストールするだけで自動的に認識されます。設定ファイルで有効/無効を切り替えることもできます。現在、1,000個以上のプラグインがPyPIに公開されています。

プラグイン名 主な機能 実務での用途
pytest-cov コードカバレッジ計測・レポート生成 テストの網羅率を定量的に把握する
pytest-xdist 並列テスト実行・分散テスト テストスイートが大きくなったときの時間短縮
pytest-mock mocker フィクスチャによるモック簡略化 モックのセットアップコードを削減する

プラグインはどこで見つけるか

PyPI で「pytest-」で検索するか、pytest公式のプラグインリストを参照してください。ダウンロード数や最終更新日を見て、活発にメンテナンスされているプラグインを選びましょう。

11.2 pytest-cov - カバレッジ計測の基本

pytest-covは、テスト実行中にどのコードが実行されたかを記録し、カバレッジレポートを生成するプラグインです。「テストがどれくらいのコードをカバーしているか」を数値で把握できます。

pytest-covは内部的にcoverage.pyというライブラリを使用しています。coverage.pyを直接使うより、pytestと統合された形でシームレスに使えるのがメリットです。

インストールと基本的な使い方

インストール 
# pytest-cov のインストール
pip install pytest-cov
基本的なカバレッジ計測コマンド 
# mypackage ディレクトリのカバレッジを計測してテスト実行
pytest --cov=mypackage

# ターミナルに詳細レポートを表示（未カバー行も表示）
pytest --cov=mypackage --cov-report=term-missing

# HTMLレポートを生成（htmlcov/フォルダに出力）
pytest --cov=mypackage --cov-report=html

# 複数の形式を同時に出力
pytest --cov=mypackage --cov-report=term-missing --cov-report=html

カバレッジレポートの見方

--cov-report=term-missingを付けると、ターミナルに以下のようなレポートが表示されます。

カバレッジレポートの出力例

---------- coverage: platform linux, python 3.11 ----------
Name                    Stmts   Miss  Cover   Missing
-----------------------------------------------------
mypackage/__init__.py       2      0   100%
mypackage/calculator.py    20      3    85%   15, 28-29
mypackage/utils.py         12      0   100%
-----------------------------------------------------
TOTAL                      34      3    91%

Stmts: 総ステートメント数（実行可能な行数）

Miss: テストで実行されなかった行数

Cover: カバレッジ率（Stmts - Miss) / Stmts

Missing: 実行されなかった行番号

calculator.py（テスト対象） 
def calculate(operation, a, b):
    """四則演算を行う関数"""
    if operation == "add":
        return a + b
    elif operation == "subtract":
        return a - b
    elif operation == "multiply":    # ← テストされていない場合
        return a * b                  # ← Missing: 15行目
    elif operation == "divide":      # ← テストされていない場合
        if b == 0:                    # ← Missing: 28行目
            raise ZeroDivisionError   # ← Missing: 29行目
        return a / b
    else:
        raise ValueError(f"Unknown operation: {operation}")

カバレッジ最小値の設定（失敗条件）

--cov-fail-underオプションを使うと、カバレッジが指定した値を下回った場合にテストを失敗扱いにできます。CI/CDで品質ゲートとして活用します。

カバレッジ最小値の設定 
# カバレッジが80%未満の場合はテスト失敗にする
pytest --cov=mypackage --cov-fail-under=80

カバレッジ100%が目標ではない

カバレッジ率を高くすることは大切ですが、100%を目指す必要はありません。カバレッジが高くても、テストの内容が不適切であれば品質は保証されません。一般的には70〜90%を目標とし、重要なビジネスロジックを重点的にカバーする戦略が実務では取られます。

11.3 pytest-xdist - テストの並列実行

pytest-xdistは、テストを複数のCPUコアやプロセスに分散して実行するプラグインです。テストスイートが大きくなると実行に数分〜数十分かかることがありますが、pytest-xdistを使うと大幅に短縮できます。

並列実行の仕組み

flowchart LR subgraph seq["通常の実行（順次）"] S1["test_a"] --> S2["test_b"] --> S3["test_c"] --> S4["test_d"] end subgraph par["pytest-xdist（並列）"] W1["ワーカー1: test_a, test_c"] W2["ワーカー2: test_b, test_d"] end seq -->|"時間: 合計"| par style seq fill:#fee2e2,stroke:#ef4444 style par fill:#d1fae5,stroke:#10b981
インストールと基本的な使い方 
# pytest-xdist のインストール
pip install pytest-xdist

# 2つのワーカーで並列実行
pytest -n 2

# CPUコア数に合わせて自動的にワーカー数を決定（推奨）
pytest -n auto

# カバレッジと組み合わせた並列実行
pytest -n auto --cov=mypackage

テストの独立性が必要

pytest-xdistで並列実行するには、各テストが独立している必要があります。グローバル変数・共有ファイル・共有データベースなど、テスト間で状態を共有していると、並列実行時に競合が発生して不安定なテスト（フレイキーテスト）になります。並列実行がうまくいかない場合は、テストの独立性を見直しましょう。

テスト配分モード

--distオプションで、テストをワーカーにどう配分するかを指定できます。

--dist=load

デフォルト。ワーカーが空いたら次のテストを割り当てる（負荷分散）。

--dist=no

並列化を無効にする。デバッグ時に役立つ。

--dist=each

全ワーカーに全テストを実行させる。環境差異の検証用。

11.4 pytest-mock - モックをシンプルに書く

pytest-mockは、unittest.mockの機能をmockerフィクスチャとして提供するプラグインです。第10章で学んだモックをより簡潔に書けるようになります。

最大のメリットは、with patch(...)のコンテキストマネージャを使わなくても、フィクスチャとして受け取るだけで自動的にモックのクリーンアップが行われる点です。

unittest.mock vs pytest-mock の比較

インストール 
pip install pytest-mock

unittest.mock を直接使う場合

unittest.mock版 
from unittest.mock import patch

def test_外部API呼び出しをモック():
    # with 文でコンテキスト管理が必要
    with patch("myapp.service.requests.get") as mock_get:
        mock_get.return_value.json.return_value = {
            "status": "ok"
        }
        result = call_api()
    assert result == "ok"

pytest-mock を使う場合

pytest-mock版 
# mocker フィクスチャを引数として受け取るだけ
def test_外部API呼び出しをモック(mocker):
    # with 文不要でスッキリ書ける
    mock_get = mocker.patch("myapp.service.requests.get")
    mock_get.return_value.json.return_value = {
        "status": "ok"
    }
    result = call_api()
    assert result == "ok"

mocker フィクスチャの主要メソッド

mocker の主要メソッド一覧 
def test_mockerのメソッド例(mocker):
    # 関数・メソッドをモックに差し替え
    mock_func = mocker.patch("module.function_name")

    # オブジェクトの属性をモックに差し替え
    mock_attr = mocker.patch.object(SomeClass, "method_name")

    # 辞書の値を一時的に変更
    mocker.patch.dict(some_dict, {"key": "new_value"})

    # MagicMock を直接作成（パッチなしで使いたい場合）
    mock_obj = mocker.MagicMock()
    mock_obj.some_method.return_value = 42

    # spy: 実際の関数を呼びながら呼び出し回数も記録
    spy = mocker.spy(SomeClass, "real_method")
    SomeClass().real_method()
    assert spy.call_count == 1

自動クリーンアップが最大のメリット

pytest-mockのmockerフィクスチャは、テスト終了後に自動でモックをリストアします。うっかりpatch.stop()を呼び忘れてモックが残り続ける、という問題が起きません。

11.5 pytest-html - HTMLレポートの生成

pytest-htmlは、テスト結果をビジュアルなHTMLレポートとして出力するプラグインです。テスト結果をチームメンバーや非エンジニアのステークホルダーと共有する際に役立ちます。

インストールと使い方 
# pytest-html のインストール
pip install pytest-html

# report.html という名前でレポートを生成
pytest --html=report.html

# CSSを HTML ファイルに埋め込む（単一ファイルで共有しやすい）
pytest --html=report.html --self-contained-html

# カバレッジと組み合わせた使用
pytest --cov=mypackage --html=report.html --self-contained-html

サマリーセクション

  • ・ 総テスト数、成功・失敗・スキップ数
  • ・ 実行時間の合計
  • ・ 実行日時と環境情報

テスト詳細セクション

  • ・ 各テストの結果（PASSED/FAILED/SKIPPED）
  • ・ 失敗テストのエラーメッセージ
  • ・ 各テストの実行時間

pytest.ini でデフォルト設定を固定する

毎回オプションを入力するのが手間な場合は、pytest.iniやpyproject.tomlにデフォルトオプションを記述できます。設定ファイルについては第13章で詳しく学びます。

11.6 pytest-randomly - テスト実行順のランダム化

pytest-randomlyは、テストの実行順をランダムにするプラグインです。インストールするだけで自動的に有効になります。

なぜランダム化が必要なのでしょうか。テストは「どの順番で実行しても同じ結果になる」べきですが、実際には前のテストが設定した状態に後のテストが依存してしまうことがあります（テスト間の依存関係）。ランダム化することで、こうした隠れた依存関係を早期に発見できます。

インストールと使い方 
# pytest-randomly のインストール（インストールするだけで自動的に有効）
pip install pytest-randomly

# 通常通りテストを実行（自動でランダム化される）
pytest

# シードを固定して再現性を確保（特定の順番を再現したい場合）
pytest -p randomly --randomly-seed=12345

# ランダム化を無効にして通常の順序で実行
pytest -p no:randomly

pytest-randomly の出力例

Using --randomly-seed=54321  ← 今回のシード値が表示される
collected 10 items

tests/test_calculator.py::test_multiply PASSED
tests/test_user.py::test_create_user PASSED
tests/test_calculator.py::test_add PASSED
...

シード値を記録しておくと、同じ実行順を再現できます。バグが特定の順番でのみ発生する場合に調査しやすくなります。

11.7 プラグインの組み合わせと設定ファイルへの記述

実務では複数のプラグインを組み合わせて使います。毎回コマンドラインオプションを入力するのは大変なので、pyproject.tomlやpytest.iniにデフォルト設定を記述しておくのが一般的です。

pyproject.toml（プラグイン設定例） 
[tool.pytest.ini_options]
# テストディレクトリの指定
testpaths = ["tests"]

# デフォルトオプション（毎回入力しなくてよい）
addopts = [
    "--cov=mypackage",          # カバレッジ計測
    "--cov-report=term-missing",# ターミナルに詳細表示
    "--cov-report=html",        # HTMLレポートも生成
    "--cov-fail-under=80",      # 80%未満で失敗
    "-v",                       # 詳細モード
]

開発環境のセットアップ（一括インストール）

プロジェクトで使うプラグインをrequirements-dev.txtまたはpyproject.tomlに記述しておくと、チームメンバーが同じ環境をすぐに再現できます。

requirements-dev.txt（開発用依存関係）

pytest-mock>=3.0
pytest-html>=4.0
pytest-randomly>=3.0
一括インストールコマンド 
pip install -r requirements-dev.txt

実習 11-1: プラグインを導入してレポートを確認しよう

第10章で作成したモックのテストコードに、pytest-covとpytest-htmlを追加してレポートを生成します。

手順

  1. 1 pip install pytest-cov pytest-html pytest-mockを実行してインストールする
  2. 2 既存のテストでpytest --cov=. --cov-report=term-missingを実行し、カバレッジレポートを確認する
  3. 3 pytest --html=report.html --self-contained-htmlを実行し、生成されたHTMLをブラウザで開いて確認する
  4. 4 カバレッジレポートで「Missing」となっている行に対するテストを追加し、カバレッジ率を上げてみる

ポイント

  • • カバレッジ率を上げることよりも、「何がテストされていないか」を把握することが重要です
  • • HTMLレポートを見ると、失敗テストの詳細がわかりやすく表示されます

第12章: テストカバレッジ - テストの網羅率を可視化する

理解度確認クイズ

  1. Q1. pytest-covを使ってカバレッジを計測しながらテストを実行するコマンドを書いてください（対象パッケージはmyappとします）。

    回答を表示

    pytest --cov=myapp です。さらに詳細なレポートを見るには --cov-report=term-missing を追加します。

  2. Q2. pytest-xdistでCPUコア数に合わせた自動並列実行をするオプションは何ですか？

    回答を表示

    -n auto です。pytest -n autoのように使います。

  3. Q3. pytest-mockのmockerフィクスチャの最大のメリットは何ですか？

    回答を表示

    テスト終了後にモックが自動的にリストア（クリーンアップ）されることです。with patch()構文やpatch.stop()を書かなくてよくなります。

  4. Q4. pytest-randomlyでテスト実行順をランダム化する目的は何ですか？

    回答を表示

    テスト間の隠れた依存関係（実行順への依存）を検出するためです。テストはどの順番で実行しても同じ結果になるべきですが、前のテストが設定した状態に依存するテストは、ランダム化により失敗として現れます。

トラブルシューティング

pytest-xdist 実行時にテストが不安定（フレイキー）になる

原因: テスト間で状態（グローバル変数・ファイル・データベース）を共有しているため、並列実行時に競合が発生している。

解決方法: まず-n 1（シングルワーカー）で実行してテストが通るか確認する。通る場合は並列化による競合が原因。フィクスチャのスコープを見直し、各テストが独立したリソースを使うようにする。

pytest-cov のカバレッジが 0% になる

原因: --cov=に指定したパッケージ名やパスが正しくない。

解決方法: --cov=.（カレントディレクトリ全体）を試す。または正確なパッケージディレクトリ名を指定する。ディレクトリ構造をlsで確認してから再試行する。

pytest-html でレポートが生成されるが CSS が当たっていない

原因: CSSファイルが別ファイルとして出力されているため、HTMLだけをコピーしてもスタイルが当たらない。

解決方法: --self-contained-htmlオプションを追加する。これでCSSがHTMLに埋め込まれ、単一ファイルで共有できるようになる。
