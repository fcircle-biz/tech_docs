pytestの導入と最初のテスト

pytestのインストール方法、基本的なテストの書き方、テストの実行方法を学びます。assert文を使ったシンプルなテストケースの作成から、テスト結果の読み方まで、pytestを使い始めるための基礎を身につけます。

この章で学ぶこと

  • pytestのインストール方法と仮想環境の正しい使い方を理解する
  • pytestが自動でテストを見つける「テスト収集」のルール（命名規則）を説明できる
  • assert文を使った基本的なテストを自分で書ける
  • テスト実行結果（PASSED / FAILED）の見方と失敗レポートの読み方を理解する
  • よく使うpytestコマンドオプションを使いこなせる

2.1 pytestのインストール

pytestはPythonの外部パッケージです。pipを使って簡単にインストールできます。まず仮想環境を作ってからインストールすることが推奨されます。

なぜ仮想環境が必要なのか

仮想環境（venv）はプロジェクトごとに独立したPython環境を作る仕組みです。「プロジェクトAはpytest 7、プロジェクトBはpytest 8」のように、バージョンが異なるパッケージを共存させることができます。グローバル環境を汚染せずに開発できるため、チーム開発でも必須の習慣です。

インストール手順

1 仮想環境を作成する

ターミナル 
# プロジェクトフォルダを作成して移動
mkdir my-pytest-project
cd my-pytest-project

# 仮想環境を作成（.venv という名前のフォルダが作られる）
python -m venv .venv

2 仮想環境を有効化する

ターミナル（OS別） 
# macOS / Linux の場合
source .venv/bin/activate

# Windows（コマンドプロンプト）の場合
.venv\Scripts\activate.bat

# Windows（PowerShell）の場合
.venv\Scripts\Activate.ps1

# 有効化されると、プロンプトの先頭に (.venv) と表示される

3 pytestをインストールする

ターミナル 
# pytestをインストール
pip install pytest

# インストールを確認（バージョンが表示されればOK）
pytest --version

動作確認

以下のコマンドを実行して、pytestが正しくインストールされているか確認しましょう。

$ pytest --version
pytest 8.x.x

バージョン番号が表示されれば成功です。「command not found」と表示された場合は、仮想環境が有効化されているか確認してください。

2.2 テスト収集のルール - pytestが自動でテストを見つける仕組み

pytestは「どのファイルのどの関数がテストか」を自動的に判断します。この仕組みをテスト収集（Test Discovery）と呼びます。特定の命名規則に従うだけで、設定不要でテストを実行できます。

pytestのテスト収集ルール

flowchart TD A["pytest コマンド実行"] --> B["ファイル検索
test_*.py または *_test.py"] B --> C["クラス検索
Test で始まるクラス名"] B --> D["関数検索
test_ で始まる関数名"] C --> E["メソッド検索
test_ で始まるメソッド名"] D --> F["テストとして実行"] E --> F style A fill:#dbeafe,stroke:#3b82f6 style F fill:#d1fae5,stroke:#10b981
対象 ルール 例
ファイル名 test_で始まる、または_testで終わる test_calculator.py
calculator_test.py
関数名 test_で始まる def test_add():
def test_divide_by_zero():
クラス名 Testで始まる（__init__なし） class TestCalculator:
class TestUserAuth:
メソッド名 test_で始まる def test_add(self):
def test_empty_list(self):

注意：命名規則を守らないとテストが実行されない

check_add()やverify_login()という名前の関数は、pytestに収集されません。必ずtest_で始まる名前をつけてください。

2.3 最初のテストを書いてみよう

実際にテスト対象のコードとテストコードを書いて、pytestを実行してみましょう。ここでは「足し算を行う関数」をテストする例で進めます。

今回作成するファイル構成 

my-pytest-project/
├── calculator.py      ← テスト対象のコード
└── test_calculator.py ← テストコード（test_ で始まる）

1 テスト対象のコードを作成する（calculator.py）

calculator.py 
def add(a, b):
    """2つの数値を足し算して返す関数"""
    return a + b

def subtract(a, b):
    """2つの数値を引き算して返す関数"""
    return a - b

def multiply(a, b):
    """2つの数値を掛け算して返す関数"""
    return a * b

def divide(a, b):
    """2つの数値を割り算して返す関数（ゼロ除算はエラー）"""
    if b == 0:
        raise ValueError("ゼロで割ることはできません")
    return a / b

2 テストコードを作成する（test_calculator.py）

test_calculator.py 
import pytest
from calculator import add, subtract, multiply, divide

# ---- 足し算のテスト ----
def test_add_正の数():
    """正の数同士の足し算が正しいことを確認"""
    assert add(2, 3) == 5

def test_add_負の数():
    """負の数を含む足し算が正しいことを確認"""
    assert add(-1, 1) == 0

def test_add_ゼロ():
    """ゼロを足してもそのままであることを確認"""
    assert add(0, 5) == 5

# ---- 割り算のテスト ----
def test_divide_正常系():
    """通常の割り算が正しいことを確認"""
    assert divide(10, 2) == 5.0

def test_divide_ゼロ除算は例外を発生させる():
    """ゼロで割ったときに ValueError が発生することを確認"""
    with pytest.raises(ValueError):
        divide(10, 0)

テストコードの構造を理解しよう

1

import pytest

pytestライブラリを読み込みます。pytest.raisesなどの機能を使う場合に必要です。

2

assert add(2, 3) == 5

assert文がテストの核心です。「左辺 == 右辺」が真ならパス、偽なら失敗（AssertionError）が発生します。

3

with pytest.raises(ValueError):

「このコードが指定した例外を発生させること」を検証します。例外が発生しなければテスト失敗です。

2.4 テストを実行する

ファイルが作成できたら、いよいよpytestを実行しましょう。基本はpytestコマンドを実行するだけです。

ターミナル

pytest

# 特定のファイルだけ実行
pytest test_calculator.py

# 詳細な出力で実行（各テスト名が表示される）
pytest -v

test_add_正の数のように日本語を含めることで、-vオプションで実行したときに「何をテストしているか」が一目でわかります。テストの可読性が大幅に向上するため、日本語チームでは特に有効です。

2.5 テスト失敗レポートの読み方

pytestの大きな特徴の一つが、テスト失敗時の詳細なエラーレポートです。「何が期待されて、何が実際に返ってきたのか」を明確に表示してくれます。

試しに、add(2, 3)の結果が6であると書いた（わざと間違えた）テストを実行してみましょう。

わざと失敗するテスト例 
def test_add_わざと失敗():
    # 2 + 3 = 5 なのに 6 と書いている（間違い）
    assert add(2, 3) == 6

失敗時のpytestの出力

========================= test session starts ==========================
collected 1 item

test_calculator.py F                                              [100%]

============================== FAILURES ===============================
______________________ test_add_わざと失敗 ______________________

    def test_add_わざと失敗():
        # 2 + 3 = 5 なのに 6 と書いている（間違い）
>       assert add(2, 3) == 6    ← エラーが発生した行を示す
E       AssertionError: assert 5 == 6
E                      ↑実際の値  ↑期待した値
E        +  where 5 = add(2, 3)

========================= 1 failed in 0.12s ==========================

エラーレポートの各部分の意味

F

テスト失敗を表す記号（.が成功、Fが失敗、Eがエラー）

>

AssertionErrorが発生した行を指し示す矢印

E AssertionError

実際の値と期待値の比較が表示される。assert 5 == 6は「実際の値（5）が期待値（6）と一致しなかった」ことを意味する

重要ポイント：失敗レポートこそpytestの真価

unittestではAssertionErrorとしか表示されませんが、pytestは「実際に何が返ってきたか」「どの行で失敗したか」を詳細に教えてくれます。この情報量の差がデバッグ時間を大幅に短縮してくれます。

2.6 よく使うpytestオプション

pytestにはさまざまなコマンドラインオプションがあります。よく使うものを覚えておきましょう。

-v 詳細表示

各テストの名前と結果（PASSED/FAILED）を一行ずつ表示します。何がパスしたかを確認するときに便利。

pytest -v
-s print出力を表示

テストコード内のprint()の出力を表示します。デバッグ時に役立ちます。

pytest -s
-k "キーワード" 絞り込み実行

テスト名に指定したキーワードを含むものだけ実行します。特定のテストだけ確認したいときに便利。

pytest -k "divide"
-x 最初の失敗で停止

最初のテストが失敗した時点で実行を中断します。エラーの原因を一つひとつ修正したいときに有効。

pytest -x
--tb=short エラー出力を短縮

失敗時のトレースバックを短く表示します。多数のテストが失敗しているときに全体像を把握しやすくなります。

pytest --tb=short
--no-header ヘッダー非表示

実行環境情報（OS・Pythonバージョン等）のヘッダーを非表示にします。出力をすっきりさせたいときに使います。

pytest --no-header

オプションは組み合わせて使える

よく使う組み合わせ例 
# 詳細表示 + 最初の失敗で停止（開発中によく使う）
pytest -v -x

# divideに関係するテストだけ詳細表示
pytest -v -k "divide"

# printも表示しながら詳細実行
pytest -v -s

2.7 assert文で書けるさまざまな検証

pytestのassert文はPython標準の文法なので、Pythonで「真」と判定される式なら何でも書けます。よく使うパターンを覚えましょう。

assert文の様々なパターン 
def test_様々なassertパターン():
    # 等値比較
    assert 1 + 1 == 2

    # 不等値比較
    assert 1 != 2

    # リストの要素チェック
    result = [1, 2, 3]
    assert 2 in result         # 要素が含まれるか
    assert len(result) == 3    # 長さの確認

    # 真偽値チェック
    assert True
    assert not False

    # 文字列チェック
    msg = "Hello, World!"
    assert msg.startswith("Hello")
    assert "World" in msg

    # 型チェック
    assert isinstance(42, int)
    assert isinstance("abc", str)

失敗時のメッセージをカスタマイズする

assert文にカンマ区切りでメッセージを追加すると、失敗時に表示されます。

カスタムエラーメッセージ 
def test_カスタムメッセージ付きassert():
    result = add(2, 3)
    # 失敗したとき "2 + 3 の計算結果が正しくありません" と表示される
    assert result == 5, f"2 + 3 の計算結果が正しくありません: {result}"

実習 2-1: 計算機のテストを書いて実行しよう

calculator.pyのsubtract関数とmultiply関数に対するテストを追加し、実際に実行して確認します。

手順

  1. 1 test_calculator.pyにsubtractのテストを3つ追加する（正常系、負の結果になるケース、ゼロのケース）
  2. 2 multiplyのテストも3つ追加する（正の数同士、負の数を含む、ゼロを掛けるケース）
  3. 3 pytest -vで実行して、すべてPASSEDになることを確認する
  4. 4 わざと間違った期待値（例: assert subtract(5, 3) == 99）を書いて、失敗レポートを確認してみる

ヒント：subtract のテスト例

  • • subtract(10, 3) == 7（正常系）
  • • subtract(3, 10) == -7（負の結果）
  • • subtract(5, 5) == 0（同じ数値）

トラブルシューティング

pytest: command not found

原因: 仮想環境が有効化されていない、またはpytestがインストールされていない

解決方法: source .venv/bin/activate（macOS/Linux）または.venv\Scripts\activate（Windows）で仮想環境を有効化してからpip install pytestを実行する

no tests ran（テストが1件も実行されない）

原因: ファイル名や関数名が命名規則に従っていない

解決方法: ファイル名がtest_で始まるか_test.pyで終わるか確認する。テスト関数名がtest_で始まっているか確認する

ModuleNotFoundError: No module named 'calculator'

原因: calculator.pyがテストファイルと同じフォルダにない

解決方法: calculator.pyとtest_calculator.pyが同じディレクトリに存在するか確認する

第3章: unittest入門と比較 - 標準ライブラリのテストフレームワーク

理解度確認クイズ

  1. Q1. pytestがテストとして自動収集するファイル名のパターンを2つ答えてください。

    回答を表示

    test_*.py（test_で始まるファイル）と*_test.py（_testで終わるファイル）の2パターンです。例: test_calculator.py、calculator_test.py

  2. Q2. テスト実行時に.（ドット）とFはそれぞれ何を意味しますか？

    回答を表示

    .（ドット）= テスト成功（PASSED）、F = テスト失敗（FAILED）を意味します。ドットが並ぶほど多くのテストがパスしているということです。

  3. Q3. 「divide(10, 0)がValueErrorを発生させること」をpytestでテストするコードを書いてください。

    回答を表示

    with pytest.raises(ValueError): divide(10, 0)と書きます。pytest.raises()コンテキストマネージャの中でコードを実行し、指定した例外が発生することを検証します。

  4. Q4. 「divideという文字を含むテストだけを実行する」コマンドを書いてください。

    回答を表示

    pytest -k "divide"です。-kオプションはテスト名のキーワード検索で、テスト関数名や定数名にそのキーワードが含まれるテストのみ実行されます。
