第11章

エラーへの対処法 - つまずきポイントの攻略

初心者がよく遭遇するエラーメッセージの読み方と解決方法を学びます。エラーは敵ではなく、問題解決のヒントであることを理解し、デバッグ力を身につけます。

この章で学ぶこと

  • エラーとは何か、なぜエラーが発生するのかを理解する
  • 初心者がよく遭遇するエラーの種類と原因を学ぶ
  • エラーメッセージの読み方と解決のヒントの見つけ方を習得する
  • 効果的なデバッグ手法とエラー解決のステップを身につける

11.1 エラーとは何か？ - エラーは敵ではない

プログラミング初心者にとって、エラーメッセージは恐怖の対象かもしれません。 しかし、エラーは実は「プログラムのここがおかしいよ」と教えてくれる親切なメッセージなのです。

エラーを前向きに捉える

エラーは「失敗」ではなく「学習のチャンス」です。プロのプログラマーでも毎日何度もエラーに遭遇します。 大切なのは、エラーメッセージを読んで原因を理解し、修正する力を身につけることです。 エラーと向き合うことで、プログラミングスキルは確実に向上します。

エラー発生から解決までの流れ

graph LR A[プログラム実行] --> B{エラー発生？} B -->|YES| C[エラーメッセージを読む] B -->|NO| H[正常終了] C --> D[エラーの種類を特定] D --> E[原因箇所を見つける] E --> F[コードを修正] F --> A

エラーの良い点

問題の場所を教えてくれる、何が間違っているか説明してくれる、 プログラムが誤動作する前に止めてくれる

エラーを無視すると

間違った結果が出る、プログラムが予期せず停止する、 原因不明のバグが残り続ける

11.2 初心者がよく遭遇するエラー - 具体例と解決法

Pythonで初心者が遭遇しやすい代表的なエラーと、その解決方法を見ていきましょう。 それぞれのエラーの「原因」と「解決策」を理解することが重要です。

1

SyntaxError - 文法エラー

Pythonの文法ルールに違反しているときに発生します。最も頻繁に遭遇するエラーです。

❌ エラーが発生するコード 
# コロン(:)の付け忘れ
if age >= 20
    print("成人です")

# カッコの閉じ忘れ
print("Hello, World!"

# インデントの間違い
def greet():
print("こんにちは")  # インデントがない
✅ 修正後のコード 
# コロン(:)を追加
if age >= 20:
    print("成人です")

# カッコを正しく閉じる
print("Hello, World!")

# インデントを正しく設定
def greet():
    print("こんにちは")  # スペース4つでインデント

解決のヒント: エラーメッセージに表示される行番号の前後を確認しましょう。 コロン、カッコ、インデントが正しいか注意深くチェックしてください。

2

NameError - 名前エラー

定義されていない変数や関数を使おうとしたときに発生します。

❌ エラーが発生するコード 
# 変数を定義せずに使用
print(message)  # messageが定義されていない

# スペルミス
name = "太郎"
print(nmae)  # nameのつもりがnmaeになっている
✅ 修正後のコード 
# 変数を先に定義
message = "こんにちは"
print(message)

# スペルを正しく修正
name = "太郎"
print(name)

解決のヒント: 変数名や関数名のスペルミス、大文字小文字の違いを確認しましょう。 Pythonは大文字小文字を区別します（nameとNameは別物）。

3

TypeError - 型エラー

データ型が合わない操作をしようとしたときに発生します。

❌ エラーが発生するコード 
# 文字列と数値を直接結合
age = 20
print("私は" + age + "歳です")  # エラー

# 文字列をlenに渡さない
length = len(123)  # 数値に長さはない
✅ 修正後のコード 
# 数値を文字列に変換
age = 20
print("私は" + str(age) + "歳です")

# またはf文字列を使用
print(f"私は{age}歳です")

# 文字列の長さを取得
text = "こんにちは"
length = len(text)  # OK

解決のヒント: 変数のデータ型を意識しましょう。 必要に応じてstr()、int()、float()で型変換してください。

4

IndexError - インデックスエラー

リストやタプルで存在しないインデックスにアクセスしようとしたときに発生します。

❌ エラーが発生するコード 
# リストの範囲外にアクセス
fruits = ["りんご", "バナナ", "みかん"]
print(fruits[3])  # インデックスは0,1,2まで

# 空のリストにアクセス
empty_list = []
print(empty_list[0])  # 要素がない
✅ 修正後のコード 
# インデックスを確認
fruits = ["りんご", "バナナ", "みかん"]
print(fruits[2])  # みかん（最後の要素）

# 要素数を確認してからアクセス
if len(fruits) > 3:
    print(fruits[3])
else:
    print("その位置に要素はありません")

解決のヒント: リストのインデックスは0から始まることを忘れずに。 len()でリストの長さを確認してからアクセスしましょう。

5

KeyError - キーエラー

辞書に存在しないキーを使って値を取得しようとしたときに発生します。

❌ エラーが発生するコード 
# 存在しないキーにアクセス
user = {"name": "太郎", "age": 20}
print(user["email"])  # emailキーは存在しない
✅ 修正後のコード 
# getメソッドを使用（安全）
user = {"name": "太郎", "age": 20}
email = user.get("email", "未登録")  # デフォルト値を指定
print(email)  # 未登録

# キーの存在を確認してからアクセス
if "email" in user:
    print(user["email"])
else:
    print("メールアドレスは未登録です")

解決のヒント: get()メソッドを使うか、in演算子でキーの存在を確認してからアクセスしましょう。

11.3 エラーメッセージの読み方 - 宝の地図を解読する

エラーメッセージは、問題解決への「宝の地図」です。正しく読み解けば、 エラーの原因と修正箇所がすぐに分かります。

典型的なエラーメッセージの例 
Traceback (most recent call last):
  File "sample.py", line 5, in <module>
    result = 10 / 0
ZeroDivisionError: division by zero

エラーメッセージの構成要素

graph TB A[エラーメッセージ] --> B[①Traceback情報] A --> C[②ファイル名と行番号] A --> D[③問題のコード] A --> E[④エラーの種類と説明] B --> B1["実行経路（どのように
エラーに至ったか）"] C --> C1["File "sample.py", line 5
（どこでエラーが発生したか）"] D --> D1["result = 10 / 0
（問題のコード行）"] E --> E1["ZeroDivisionError:
division by zero
（何が問題か）"]

①ファイル名と行番号

File "sample.py", line 5
どのファイルの何行目でエラーが発生したかを示します。まずここを確認しましょう。

②エラーの種類

ZeroDivisionError
どんな種類のエラーかを示します。これが問題の分類です。

③問題のコード

result = 10 / 0
実際に問題が発生したコード行が表示されます。

④エラーの説明

division by zero
何が問題なのかを簡潔に説明してくれます（英語が多い）。

エラーメッセージの読み方のコツ

  1. 下から読む: エラーの種類と説明（最下行）から読み始める
  2. 行番号を確認: どのファイルの何行目かを特定する
  3. コードを見る: 該当するコード行を確認する
  4. 英語を翻訳: わからない単語はGoogle翻訳やDeepLで調べる
  5. 検索する: エラー名で検索すれば、解決策が見つかる

11.4 デバッグの基本手法 - 問題を見つけて修正する

デバッグとは、プログラムのバグ（虫=エラー）を取り除く作業のことです。 効果的なデバッグ手法を身につければ、エラー解決のスピードが格段に上がります。

1

print()デバッグ - 最も基本的な方法

変数の値や実行状況をprint()で表示して、どこで問題が起きているか特定します。

print()でデバッグ 
# 変数の値を確認
numbers = [1, 2, 3, 4, 5]
total = 0

for num in numbers:
    print(f"現在の数値: {num}")  # デバッグ用print
    print(f"加算前の合計: {total}")  # デバッグ用print
    total += num
    print(f"加算後の合計: {total}")  # デバッグ用print
    print("---")

print(f"最終結果: {total}")

ポイント: 怪しい箇所の前後にprint()を仕込んで、変数の値や実行順序を確認しましょう。

2

コメントアウトで範囲を絞る

問題のある行を特定するため、一部のコードを一時的に無効化（コメントアウト）します。

コメントアウトで問題を特定 
# エラーがどこで起きているか分からない場合
def calculate():
    a = 10
    b = 20
    # c = a / 0  # この行をコメントアウトして確認
    d = a + b
    return d

result = calculate()
print(result)  # これが実行されるか確認

ポイント: 疑わしいコードを段階的にコメントアウトして、 どこでエラーが消えるかを確認します。

3

小さく分割して実行

複雑な処理を小さな単位に分けて、1つずつ動作確認します。

小さく分割して確認 
# 複雑な処理を段階的に確認
data = [1, 2, 3, 4, 5]

# ステップ1: リストをフィルタ
filtered = [x for x in data if x > 2]
print("フィルタ結果:", filtered)  # [3, 4, 5]

# ステップ2: 各要素を2倍
doubled = [x * 2 for x in filtered]
print("2倍にした結果:", doubled)  # [6, 8, 10]

# ステップ3: 合計を計算
total = sum(doubled)
print("合計:", total)  # 24

ポイント: 長いコードを一度に書かず、少しずつ書いて動作確認を繰り返しましょう。

4

type()で変数の型を確認

変数が期待した型かどうかをtype()で確認します。

型の確認 
# 変数の型を確認
age = "20"  # 文字列になっている
print(type(age))  # <class 'str'>

# 型変換が必要
age = int(age)
print(type(age))  # <class 'int'>

# 複雑なデータ構造の確認
data = {"name": "太郎", "scores": [80, 90, 85]}
print(type(data))  # <class 'dict'>
print(type(data["scores"]))  # <class 'list'>

ポイント: TypeErrorが出たら、まずtype()で変数の型を確認しましょう。

困った時の解決ステップ

  1. 1
    エラーメッセージを読む
    落ち着いてエラーの種類と行番号を確認しましょう
  2. 2
    該当箇所のコードを確認
    スペルミス、インデント、コロンやカッコの付け忘れがないかチェック
  3. 3
    Google検索
    「Python エラー名」で検索すると、同じ問題の解決策が見つかります
  4. 4
    シンプルなコードで試す
    問題を切り分けて、最小限のコードで再現できるか試しましょう
  5. 5
    休憩してから見直す
    行き詰まったら一度離れましょう。新鮮な目で見ると原因が見つかることも

実践練習 - エラーを見つけて修正しよう

以下のコードにはエラーがあります。エラーメッセージを読んで、問題を特定し、修正してみましょう。

練習問題1

# このコードを修正してください
def greet(name)
    print("こんにちは、" + name + "さん")

greet("太郎")
ヒントを表示

SyntaxError: 関数定義の行末にコロン(:)が必要です

練習問題2

# このコードを修正してください
numbers = [10, 20, 30]
print(numbers[3])
ヒントを表示

IndexError: リストのインデックスは0, 1, 2までしかありません。3を2に変更しましょう。

練習問題3

# このコードを修正してください
age = "25"
next_age = age + 1
print("来年は" + next_age + "歳です")
ヒントを表示

TypeError: 文字列と数値を足すことはできません。age = int("25")と型変換するか、 f文字列を使いましょう。

理解度確認クイズ

  1. Q1. エラーメッセージの役割は何ですか？

    回答を表示

    エラーメッセージは、プログラムのどこに問題があるのか、何が原因なのかを教えてくれる親切なメッセージです。 ファイル名、行番号、エラーの種類、問題の説明を提供してくれます。

  2. Q2. NameErrorはどんなときに発生しますか？

    回答を表示

    定義されていない変数や関数を使おうとしたときに発生します。 よくある原因は、変数名のスペルミス、大文字小文字の間違い、変数を定義する前に使用した場合などです。

  3. Q3. デバッグで最も基本的な手法は何ですか？

    回答を表示

    print()デバッグです。怪しい箇所の前後にprint()を配置して、変数の値や実行状況を確認することで、 問題の箇所を特定できます。

  4. Q4. エラーで行き詰まったときの対処法を3つ挙げてください。

    回答を表示

    ① エラーメッセージと行番号を確認する ② 「Python エラー名」でGoogle検索する ③ 一度休憩して新鮮な目で見直す （他にも：シンプルなコードで切り分ける、コメントアウトで範囲を絞るなど）

第11章のまとめ

  • エラーは敵ではなく、問題箇所を教えてくれる親切なメッセージである
  • 初心者がよく遭遇するのはSyntaxError、NameError、TypeError、IndexError、KeyErrorなど
  • エラーメッセージは下から読み、ファイル名・行番号・エラーの種類・説明を確認する
  • print()デバッグ、コメントアウト、小さく分割などの手法でエラーを特定・修正できる
