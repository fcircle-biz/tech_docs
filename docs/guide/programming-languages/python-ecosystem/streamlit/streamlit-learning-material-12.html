12.1 アプリ設計の基本ステップ

プログラミング初心者がよくつまずくのは、「何から作り始めればいいかわからない」という点です。アプリ開発には、基本的な流れがあります。

アプリ開発の5ステップ

flowchart TD A[1. 何を作るか決める] --> B[2. 必要な機能を洗い出す] B --> C[3. 画面レイアウトを考える] C --> D[4. 小さく作って動かす] D --> E[5. 機能を追加していく]
  1. 1
    何を作るか決める

    目的を明確にします。「体重管理のためにBMI計算アプリが欲しい」など。

  2. 2
    必要な機能を洗い出す

    何ができればいいか、機能を箇条書きにします。

  3. 3
    画面レイアウトを考える

    紙やホワイトボードに、ざっくりと画面構成を描いてみます。

  4. 4
    小さく作って動かす

    最小限の機能だけを実装して、まず動かしてみます。

  5. 5
    機能を追加していく

    動くものができたら、少しずつ機能を追加していきます。

最初から完璧を目指さない

プロのプログラマーでも、最初から完璧なアプリは作りません。まず小さく作って動かし、少しずつ改良していくのが基本です。

実習 12-1: BMI計算アプリを作ろう

BMI（Body Mass Index）は、身長と体重から肥満度を判定する国際的な指標です。計算式と判定ロジックを組み合わせて、実用的なアプリを作ります。

機能の洗い出し

  • 身長（cm）と体重（kg）を入力できる
  • 「計算」ボタンを押すとBMI値を表示
  • BMI値に応じて判定結果（低体重・普通・肥満など）を表示
  • 判定結果の色を変える（視覚的にわかりやすく）

完成イメージ

import streamlit as st

# タイトルと説明
st.title("🏋️ BMI計算アプリ")
st.write("身長と体重を入力して、あなたのBMIを計算します。")

# 入力フォーム
col1, col2 = st.columns(2)
with col1:
    height = st.number_input("身長（cm）", min_value=100.0, max_value=250.0, value=170.0, step=0.1)
with col2:
    weight = st.number_input("体重（kg）", min_value=30.0, max_value=200.0, value=60.0, step=0.1)

# 計算ボタン
if st.button("BMIを計算", type="primary"):
    # BMI計算（身長をメートルに変換）
    height_m = height / 100
    bmi = weight / (height_m ** 2)

    # 判定ロジック
    if bmi < 18.5:
        category = "低体重"
        color = "blue"
    elif bmi < 25:
        category = "普通体重"
        color = "green"
    elif bmi < 30:
        category = "肥満（1度）"
        color = "orange"
    else:
        category = "肥満（2度以上）"
        color = "red"

    # 結果表示
    st.success(f"あなたのBMI: **{bmi:.1f}**")

    # 判定結果を色付きで表示
    if color == "blue":
        st.info(f"判定: {category}")
    elif color == "green":
        st.success(f"判定: {category}")
    elif color == "orange":
        st.warning(f"判定: {category}")
    else:
        st.error(f"判定: {category}")

実行手順

  1. 1 上記コードを bmi_app.py として保存
  2. 2 コマンドラインで streamlit run bmi_app.py を実行
  3. 3 ブラウザで開き、身長・体重を入力して「BMIを計算」ボタンをクリック

コードの解説: height_m ** 2 は「height_mの2乗」を意味します。また、if-elif-else で複数の条件分岐を組み合わせることで、BMI値に応じた判定を実現しています。

12.2 エラー処理の基礎

実用的なアプリでは、ユーザーが間違った入力をすることを想定しておく必要があります。エラーを適切に処理することで、使いやすいアプリになります。

よくあるエラーケース

エラーケース 対処方法
数値が入力されていない デフォルト値を設定しておく
0で割り算してしまう if文で事前にチェック
範囲外の値が入力される min_value/max_valueで制限
ファイルが選択されていない is Noneチェックを追加

エラー処理の例

実習 12-2: シンプル家計簿アプリ

収入と支出を記録し、残高を表示するシンプルな家計簿アプリを作成します。session_stateを使ってデータを保持する実践例です。

完成イメージ

import streamlit as st
import pandas as pd
from datetime import datetime

# タイトル
st.title("💰 シンプル家計簿アプリ")

# session_stateの初期化
if "transactions" not in st.session_state:
    st.session_state.transactions = []

# 入力フォーム
st.subheader("取引を記録")
col1, col2, col3 = st.columns(3)

with col1:
    date = st.date_input("日付", datetime.now())
with col2:
    category = st.selectbox("種類", ["収入", "支出"])
with col3:
    amount = st.number_input("金額（円）", min_value=0, step=100)

description = st.text_input("内容")

# 記録ボタン
if st.button("記録する", type="primary"):
    if amount > 0 and description:
        # 支出の場合はマイナスにする
        value = amount if category == "収入" else -amount
        st.session_state.transactions.append({
            "日付": date,
            "種類": category,
            "内容": description,
            "金額": value
        })
        st.success(f"{category} {amount}円 を記録しました")
    else:
        st.warning("金額と内容を入力してください")

# 残高表示
if st.session_state.transactions:
    st.subheader("残高サマリー")
    total = sum(t["金額"] for t in st.session_state.transactions)
    income = sum(t["金額"] for t in st.session_state.transactions if t["金額"] > 0)
    expense = abs(sum(t["金額"] for t in st.session_state.transactions if t["金額"] < 0))

    col1, col2, col3 = st.columns(3)
    col1.metric("総収入", f"¥{income:,}")
    col2.metric("総支出", f"¥{expense:,}")
    col3.metric("残高", f"¥{total:,}")

    # 取引履歴
    st.subheader("取引履歴")
    df = pd.DataFrame(st.session_state.transactions)
    st.dataframe(df, use_container_width=True)
else:
    st.info("まだ取引が記録されていません")

機能のポイント: session_stateでデータを保持し、リスト内包表記で収入・支出を集計しています。さらに、pandasでDataFrameに変換して見やすく表示しています。

実習 12-3: 三択クイズアプリ

選択肢から答えを選ぶクイズアプリを作成します。状態管理とボタン処理を組み合わせた実践例です。

完成イメージ

import streamlit as st

# タイトル
st.title("📝 Pythonクイズアプリ")

# クイズデータ
quizzes = [
    {
        "question": "Pythonで文字を表示する関数は？",
        "choices": ["display()", "print()", "show()"],
        "answer": 1
    },
    {
        "question": "変数に値を代入する記号は？",
        "choices": ["==", "=", "->"],
        "answer": 1
    },
    {
        "question": "条件分岐に使うのは？",
        "choices": ["for文", "if文", "def文"],
        "answer": 1
    }
]

# session_stateの初期化
if "current_quiz" not in st.session_state:
    st.session_state.current_quiz = 0
if "score" not in st.session_state:
    st.session_state.score = 0
if "answered" not in st.session_state:
    st.session_state.answered = False

# 進捗表示
st.progress((st.session_state.current_quiz) / len(quizzes))
st.write(f"問題 {st.session_state.current_quiz + 1} / {len(quizzes)}")

# クイズ表示
if st.session_state.current_quiz < len(quizzes):
    quiz = quizzes[st.session_state.current_quiz]
    st.subheader(quiz["question"])

    # 選択肢ボタン
    for i, choice in enumerate(quiz["choices"]):
        if st.button(choice, key=f"choice_{i}", disabled=st.session_state.answered):
            st.session_state.answered = True
            if i == quiz["answer"]:
                st.session_state.score += 1
                st.success("正解です！")
            else:
                st.error(f"不正解です。正解は「{quiz['choices'][quiz['answer']]}」でした。")

    # 次へボタン
    if st.session_state.answered:
        if st.button("次の問題へ"):
            st.session_state.current_quiz += 1
            st.session_state.answered = False
            st.rerun()
else:
    # 結果表示
    st.subheader("クイズ終了！")
    st.write(f"あなたのスコア: {st.session_state.score} / {len(quizzes)}")

    if st.button("もう一度挑戦"):
        st.session_state.current_quiz = 0
        st.session_state.score = 0
        st.session_state.answered = False
        st.rerun()

応用のアイデア: クイズデータをCSVファイルから読み込むようにしたり、正答率に応じてメッセージを変えたりすることで、さらに高度なアプリに発展できます。

12.4 アプリをさらに良くする改良ポイント

基本的なアプリが動いたら、以下のような改良を加えることで、より使いやすく実用的なアプリに仕上げられます。

改良アイデア

改良項目 具体例
デザイン改善 色を変える、アイコンを追加、カラムで横並び
データ永続化 CSVファイルに保存して次回も読み込める

データ永続化の例（CSV保存）

CSVに保存する例 
import streamlit as st
import pandas as pd

# データをCSVに保存
if st.button("データを保存"):
    df = pd.DataFrame(st.session_state.transactions)
    df.to_csv("transactions.csv", index=False)
    st.success("データを保存しました")

# CSVからデータを読み込み
if st.button("データを読み込み"):
    df = pd.read_csv("transactions.csv")
    st.session_state.transactions = df.to_dict("records")
    st.success("データを読み込みました")

理解度確認クイズ

  1. Q1. アプリ開発で最初にやるべきことは何ですか？

    回答を表示

    正解：何を作るか（目的）を決める
    いきなりコードを書き始めるのではなく、まず「何を作りたいのか」を明確にすることが重要です。目的が決まれば、必要な機能も自然と洗い出せます。

  2. Q2. エラー処理が重要な理由は何ですか？

    回答を表示

    正解：ユーザーの間違った入力に対応し、使いやすいアプリにするため
    実際のアプリでは、想定外の入力をするユーザーが必ずいます。エラー処理を適切に行うことで、アプリが突然止まることなく、ユーザーに優しいメッセージを表示できます。

  3. Q3. アプリを改良するときのコツは何ですか？

    回答を表示

    正解：小さく作って動かし、少しずつ機能を追加していく
    最初から完璧を目指すのではなく、まず最小限の機能で動くものを作り、それを少しずつ改良していく方が成功しやすいです。これを「段階的開発」と呼びます。
