第11章

データベース連携 - JDBC・DAOパターン

データベースとの連携方法とDAOパターンを学び、データの永続化ができるようになります。

この章で学ぶこと

  • JDBCを使ってデータベースに接続できる
  • DAOパターンを理解し実装できる
  • CRUD操作を実装できる
  • SQLインジェクション対策ができる

11.1 JDBCの基礎

JDBCとは

JDBC（Java Database Connectivity）は、Javaアプリケーションからデータベースにアクセスするための標準APIです。異なるデータベース製品（MySQL、PostgreSQL、Oracle等）に対して統一されたインターフェースで操作できます。 

flowchart TB
    A[Servlet/JSPアプリケーション] -->|JDBC API| B[JDBCドライバ]
    B -->|データベース固有プロトコル| C[(MySQL)]
    B -->|データベース固有プロトコル| D[(PostgreSQL)]
    B -->|データベース固有プロトコル| E[(Oracle)]

JDBCドライバの設定

データベースに接続するには、対応するJDBCドライバ（JARファイル）が必要です。これをWebアプリケーションのWEB-INF/libディレクトリに配置します。

JDBCドライバの配置場所 

myapp/
├── WEB-INF/
│   ├── lib/
│   │   └── mysql-connector-j-8.x.x.jar  ← JDBCドライバ
│   ├── classes/
│   │   └── com/example/
│   └── web.xml

データベース接続の基本

データベースに接続するには、DriverManager.getConnection()メソッドを使用します。接続URLにはデータベースの種類、サーバー、データベース名を指定します。

データベース接続の基本例 
// データベース接続情報
String url = "jdbc:mysql://localhost:3306/mydb";
String user = "root";
String password = "password";

// データベースに接続
Connection conn = DriverManager.getConnection(url, user, password);

// 処理を実行...

// 接続を閉じる
conn.close();

try-with-resourcesの使用

Connectionなどのリソースは、try-with-resources構文を使うことで自動的にクローズされます。確実なリソース解放のため、必ずこの構文を使用しましょう。

try-with-resourcesの使用例 
// try-with-resourcesで自動クローズ
try (Connection conn = DriverManager.getConnection(url, user, password)) {
    // 処理を実行
} catch (SQLException e) {
    e.printStackTrace();
}
// この時点で自動的にconn.close()が呼ばれる

11.2 SQLの実行

StatementとPreparedStatementの違い

JDBCではSQL文を実行するために、StatementまたはPreparedStatementを使用します。セキュリティと性能の観点から、PreparedStatementの使用が強く推奨されます。

特徴 Statement PreparedStatement
SQL文 文字列結合で作成 プレースホルダー使用
SQLインジェクション 脆弱性あり 安全
性能 毎回コンパイル 再利用可能

SQLインジェクションの危険性

Statementで文字列結合を使うと、悪意のあるユーザーがSQL文を改ざんし、データを盗んだり削除したりできます。必ずPreparedStatementを使用してください。

SELECTの実行とResultSet処理

SELECT文を実行するには、PreparedStatementのexecuteQuery()メソッドを使います。結果はResultSetオブジェクトで返されます。

SELECT文の実行例 
String sql = "SELECT id, name, email FROM users WHERE id = ?";

try (PreparedStatement pstmt = conn.prepareStatement(sql)) {
    // プレースホルダーに値を設定（インデックスは1から）
    pstmt.setInt(1, userId);

    // SELECTを実行
    try (ResultSet rs = pstmt.executeQuery()) {
        // 結果を1行ずつ処理
        while (rs.next()) {
            int id = rs.getInt("id");
            String name = rs.getString("name");
            String email = rs.getString("email");
            System.out.println(id + ": " + name + " (" + email + ")");
        }
    }
}

INSERT/UPDATE/DELETEの実行

データの追加、更新、削除にはexecuteUpdate()メソッドを使います。戻り値は影響を受けた行数です。

INSERT/UPDATE/DELETEの実行例 
// INSERT
String insertSql = "INSERT INTO users (name, email) VALUES (?, ?)";
try (PreparedStatement pstmt = conn.prepareStatement(insertSql)) {
    pstmt.setString(1, "田中太郎");
    pstmt.setString(2, "tanaka@example.com");
    int count = pstmt.executeUpdate();
    System.out.println(count + "件追加されました");
}

// UPDATE
String updateSql = "UPDATE users SET email = ? WHERE id = ?";
try (PreparedStatement pstmt = conn.prepareStatement(updateSql)) {
    pstmt.setString(1, "newemail@example.com");
    pstmt.setInt(2, 123);
    int count = pstmt.executeUpdate();
    System.out.println(count + "件更新されました");
}

11.3 DAOパターン

DAOパターンとは

DAO（Data Access Object）パターンは、データベースアクセスのロジックをビジネスロジックから分離するデザインパターンです。これにより、保守性とテスタビリティが向上します。 

flowchart TB
    A[Servlet
Controller] -->|ビジネスロジック| B[Service層]
    B -->|データ操作依頼| C[DAO
Data Access Object]
    C -->|SQL実行| D[(Database)]
    C -->|DTOで返却| B
    B -->|DTOで返却| A
    A -->|表示| E[JSP
View]

DTOクラスの作成

DTO（Data Transfer Object）は、データベースのレコードをJavaオブジェクトとして表現するクラスです。JavaBeansの規約に従い、プロパティとgetter/setterを持ちます。

User.java（DTOクラス） 
package com.example.dto;

public class User {
    private int id;
    private String name;
    private String email;

    // コンストラクタ
    public User() {}

    public User(int id, String name, String email) {
        this.id = id;
        this.name = name;
        this.email = email;
    }

    // getter/setter
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
}

DAOインターフェースの定義

まず、DAOが提供するメソッドをインターフェースで定義します。これにより、実装を切り替えやすくなります（例: MySQLからPostgreSQLへの移行）。

UserDao.java（インターフェース） 
package com.example.dao;

import com.example.dto.User;
import java.util.List;

public interface UserDao {
    List<User> findAll();           // 全件取得
    User findById(int id);           // ID検索
    void insert(User user);          // 新規登録
    void update(User user);          // 更新
    void delete(int id);             // 削除
}

DAO実装クラスの作成

インターフェースを実装し、実際のデータベースアクセスロジックを記述します。

UserDaoImpl.java（実装クラス） 
package com.example.dao;

import com.example.dto.User;
import java.sql.*;
import java.util.ArrayList;
import java.util.List;

public class UserDaoImpl implements UserDao {
    private String url = "jdbc:mysql://localhost:3306/mydb";
    private String user = "root";
    private String password = "password";

    @Override
    public List<User> findAll() {
        List<User> users = new ArrayList<>();
        String sql = "SELECT id, name, email FROM users";

        try (Connection conn = DriverManager.getConnection(url, user, password);
             PreparedStatement pstmt = conn.prepareStatement(sql);
             ResultSet rs = pstmt.executeQuery()) {

            while (rs.next()) {
                users.add(new User(
                    rs.getInt("id"),
                    rs.getString("name"),
                    rs.getString("email")
                ));
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
        return users;
    }
}

11.4 Webアプリケーションでの実装

コネクションプールの概念

毎回データベース接続を作成・破棄すると性能が低下します。コネクションプールを使うと、接続を再利用でき、性能が向上します。本番環境では必須の技術です。 

flowchart LR
    A[Servlet] -->|接続を借りる| B[コネクション
プール]
    B -->|再利用| C[(Database)]
    A -->|接続を返却| B

コネクションプールの設定

Tomcatではcontext.xmlでコネクションプールを設定できます。詳細は公式ドキュメントを参照してください。

Servletからの利用例

ServletでDAOを使用してデータベースからデータを取得し、JSPに表示する例です。

UserListServlet.java 
@WebServlet("/users")
public class UserListServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request,
                         HttpServletResponse response)
            throws ServletException, IOException {

        // DAOを使ってユーザー一覧を取得
        UserDao userDao = new UserDaoImpl();
        List<User> users = userDao.findAll();

        // リクエスト属性に設定
        request.setAttribute("users", users);

        // JSPにフォワード
        RequestDispatcher dispatcher =
            request.getRequestDispatcher("/WEB-INF/views/userList.jsp");
        dispatcher.forward(request, response);
    }
}

トランザクション管理の基礎

複数のSQL文を1つの処理単位として扱いたい場合、トランザクションを使用します。エラー時には全ての処理をロールバック（取り消し）できます。

トランザクション管理の例 
try (Connection conn = DriverManager.getConnection(url, user, password)) {
    // 自動コミットを無効化
    conn.setAutoCommit(false);

    try {
        // 複数のSQL実行
        // ... (INSERT、UPDATE等の処理)

        // 成功したらコミット
        conn.commit();
    } catch (SQLException e) {
        // エラー時はロールバック
        conn.rollback();
        throw e;
    }
}

実習 11-1: ユーザー管理機能の実装

DAOパターンを使ったユーザー一覧表示機能を実装します。

手順

  1. 1 データベースにusersテーブルを作成し、テストデータを挿入
  2. 2 User.java（DTOクラス）を作成
  3. 3 UserDao.javaとUserDaoImpl.javaを実装
  4. 4 UserListServlet.javaでDAOを呼び出し
  5. 5 userList.jsp（EL/JSTLを使用）でユーザー一覧を表示

期待される結果

http://localhost:8080/myapp/usersにアクセスすると、データベースに登録されているユーザーの一覧が表形式で表示されます。

トラブルシューティング

ClassNotFoundException: com.mysql.cj.jdbc.Driver

原因: JDBCドライバのJARファイルが見つかりません。

解決方法: JARファイルがWEB-INF/libディレクトリに配置されているか確認し、Tomcatを再起動してください。

SQLException: Access denied for user

原因: データベースのユーザー名またはパスワードが間違っています。

解決方法: 接続情報（URL、ユーザー名、パスワード）を確認し、データベースサーバーに正しくアクセスできるか確認してください。

リソースリーク - Connectionが閉じられない

原因: Connection、PreparedStatement、ResultSetを明示的にクローズしていません。

解決方法: 必ずtry-with-resources構文を使用してください。リソースは自動的にクローズされます。

理解度確認クイズ

  1. Q1. PreparedStatementをStatementより優先して使うべき理由は何ですか？

    回答を表示

    PreparedStatementはプレースホルダーを使用するため、SQLインジェクション攻撃を防げます。また、SQL文が再利用されるため性能も向上します。セキュリティと性能の両面で優れています。

  2. Q2. DAOパターンを使う利点は何ですか？

    回答を表示

    データベースアクセスのロジックをビジネスロジックから分離できます。これにより、保守性が向上し、データベースの変更があってもビジネスロジックへの影響を最小限に抑えられます。また、テストも容易になります。

  3. Q3. try-with-resourcesを使用する目的は何ですか？

    回答を表示

    Connection、PreparedStatement、ResultSetなどのリソースを確実にクローズするためです。try-with-resourcesを使うと、例外が発生してもリソースが自動的にクローズされ、リソースリークを防げます。

この章のまとめ

  • JDBCを使うことで、Javaアプリケーションからデータベースにアクセスできます
  • PreparedStatementはSQLインジェクション対策として必須です
  • DAOパターンでデータアクセスロジックを分離し、保守性を向上できます
  • try-with-resourcesでリソースを確実にクローズし、リソースリークを防ぎます
  • コネクションプールとトランザクション管理は実務で重要な技術です
