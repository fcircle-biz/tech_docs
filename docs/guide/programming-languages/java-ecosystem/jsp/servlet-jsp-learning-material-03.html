リクエストとレスポンス - データの受け渡し

HTTPリクエストからデータを取得し、適切なレスポンスを返す方法を学びます。

この章で学ぶこと

  • リクエストパラメータを取得できる
  • GETとPOSTの違いを理解し使い分けられる
  • 適切なレスポンスを生成できる
  • 文字エンコーディングを正しく設定できる

3.1 HttpServletRequestの活用

リクエストパラメータの取得

HttpServletRequestオブジェクトを使うと、ブラウザから送信されたデータ（パラメータ）を取得できます。フォームの入力値やURLのクエリ文字列を受け取る基本的な方法を学びましょう。

主なメソッド

  • getParameter(String name): 指定した名前のパラメータ値を取得（単一値）
  • getParameterValues(String name): 指定した名前のパラメータ値を配列で取得（複数値）
  • getParameterNames(): すべてのパラメータ名を列挙
パラメータ取得の基本例 
// 単一パラメータの取得
String name = request.getParameter("name");

// パラメータがnullの場合の対処
if (name == null || name.isEmpty()) {
    name = "ゲスト";
}

// 複数選択可能な値の取得（checkbox等）
String[] hobbies = request.getParameterValues("hobby");

注意点

getParameter()は、パラメータが存在しない場合にnullを返します。必ずnullチェックを行い、NullPointerExceptionを防ぎましょう。

リクエスト属性の設定と取得

リクエスト属性を使うと、Servlet内で一時的にデータを保持し、JSPに渡すことができます。これはServletとJSPの連携で重要な役割を果たします。

リクエスト属性の使用例 
// 属性の設定
request.setAttribute("userName", "山田太郎");
request.setAttribute("age", 25);

// 属性の取得（型キャストが必要）
String userName = (String) request.getAttribute("userName");
Integer age = (Integer) request.getAttribute("age");

3.2 GETリクエストの処理

GETメソッドの特徴

GETメソッドは主にデータの取得に使用されます。パラメータはURLのクエリ文字列として送信され、ブラウザの履歴やブックマークに残ります。 

sequenceDiagram
    participant B as ブラウザ
    participant S as Servlet
    B->>S: GET /search?keyword=Java&category=book
    S->>S: getParameter("keyword") → "Java"
    S->>S: getParameter("category") → "book"
    S->>S: 検索処理を実行
    S->>B: HTMLレスポンス（検索結果）

GETの使用場面

検索機能、ページネーション、フィルタリング、詳細表示など、データを取得して表示するだけの操作に適しています。URLにパラメータが含まれるため、結果をブックマークやシェアできます。

検索フォームの実装例

実習 3-1: 検索機能の実装

GETメソッドを使った簡単な検索機能を作成します。

HTML（検索フォーム）

search.html 
<!-- GETメソッドで送信 -->
<form action="search" method="get">
    <input type="text" name="keyword" placeholder="検索キーワード">
    <button type="submit">検索</button>
</form>

Servlet実装

SearchServlet.java 
@WebServlet("/search")
public class SearchServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request,
                        HttpServletResponse response)
            throws ServletException, IOException {

        // パラメータ取得
        String keyword = request.getParameter("keyword");

        // レスポンス設定
        response.setContentType("text/html; charset=UTF-8");
        PrintWriter out = response.getWriter();

        out.println("<h2>検索結果</h2>");
        if (keyword != null && !keyword.isEmpty()) {
            out.println("<p>「" + keyword + "」の検索結果</p>");
        } else {
            out.println("<p>キーワードを入力してください</p>");
        }
    }
}

3.3 POSTリクエストの処理

POSTメソッドの特徴

POSTメソッドは主にデータの送信や更新に使用されます。パラメータはリクエストボディに含まれるため、URLには表示されません。大量のデータや機密情報の送信に適しています。

GETとPOSTの比較

項目 GET POST
パラメータの場所 URLのクエリ文字列 リクエストボディ
データ量 制限あり（約2KB） 制限なし
ブックマーク 可能 不可
セキュリティ URL に表示される URL に表示されない
用途 データ取得・表示 データ送信・更新

doPost()メソッドの実装

POSTリクエスト処理の例 
@WebServlet("/register")
public class RegisterServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request,
                         HttpServletResponse response)
            throws ServletException, IOException {

        // 文字エンコーディング設定（最初に必ず行う）
        request.setCharacterEncoding("UTF-8");

        // パラメータ取得
        String name = request.getParameter("name");
        String email = request.getParameter("email");

        // レスポンス設定
        response.setContentType("text/html; charset=UTF-8");
        PrintWriter out = response.getWriter();

        out.println("<h2>登録完了</h2>");
        out.println("<p>名前: " + name + "</p>");
        out.println("<p>メール: " + email + "</p>");
    }
}

重要ポイント

request.setCharacterEncoding("UTF-8")は、getParameter()を呼び出す前に必ず実行してください。日本語の文字化けを防ぐために非常に重要です。

3.4 HttpServletResponseの活用

レスポンスの基本設定

HttpServletResponseオブジェクトを使って、クライアントに返すレスポンスの内容やヘッダーを設定します。

主なメソッド

  • setContentType(String type): コンテンツタイプと文字エンコーディングを設定
  • getWriter(): 文字データを出力するためのPrintWriterを取得
  • getOutputStream(): バイナリデータを出力するためのStreamを取得
  • sendRedirect(String location): 別のURLへリダイレクト
レスポンス設定の例 
// HTMLレスポンスの場合
response.setContentType("text/html; charset=UTF-8");
PrintWriter out = response.getWriter();
out.println("<h1>こんにちは</h1>");

// JSONレスポンスの場合
response.setContentType("application/json; charset=UTF-8");
PrintWriter out = response.getWriter();
out.println("{\"message\": \"成功\"}");

// リダイレクト
response.sendRedirect("complete.html");

リダイレクト処理

sendRedirect()メソッドを使うと、クライアントを別のURLに転送できます。フォーム送信後の完了画面への遷移などで使用します。 

sequenceDiagram
    participant B as ブラウザ
    participant S1 as FormServlet
    participant S2 as complete.html
    B->>S1: POST /form（データ送信）
    S1->>S1: データ処理
    S1->>B: 302 Redirect: /complete.html
    B->>S2: GET /complete.html
    S2->>B: 完了画面HTML

3.5 文字エンコーディング対策

文字化けの原因と対策

日本語を扱うWebアプリケーションでは、文字エンコーディングの設定が非常に重要です。設定が不適切だと文字化けが発生します。

文字化け対策チェックリスト

  • リクエスト: request.setCharacterEncoding("UTF-8")をgetParameter()の前に実行
  • レスポンス: response.setContentType("text/html; charset=UTF-8")を設定
  • ファイル: JavaソースコードをUTF-8で保存
  • HTML: <meta charset="UTF-8">を記述
文字化け対策を施したServletの例 
protected void doPost(HttpServletRequest request,
                     HttpServletResponse response)
        throws ServletException, IOException {

    // 1. リクエストのエンコーディング設定（最初に）
    request.setCharacterEncoding("UTF-8");

    // 2. パラメータ取得
    String message = request.getParameter("message");

    // 3. レスポンスのエンコーディング設定
    response.setContentType("text/html; charset=UTF-8");

    // 4. 出力
    PrintWriter out = response.getWriter();
    out.println("<h2>" + message + "</h2>");
}

トラブルシューティング

問題: 日本語が「???」や「・・・」になる

原因: request.setCharacterEncoding()の設定漏れ

解決: getParameter()を呼ぶ前に必ず設定する

問題: ブラウザに表示される日本語が文字化け

原因: response.setContentType()の設定漏れ

解決: "charset=UTF-8"を明示的に指定する

2つの数値を受け取り、計算結果を表示する簡易電卓を作成します。

  • • 計算結果をHTMLで表示する
  • • 数値が入力されていない場合はエラーメッセージを表示

ヒント

理解度確認クイズ

  1. Q1. GETとPOSTのどちらを使うべきか？「パスワードを含むログインフォーム」の場合

    回答を表示

    POSTを使うべきです。パスワードなどの機密情報はURLに表示されるGETメソッドでは送信せず、リクエストボディに含まれるPOSTメソッドで送信します。

  2. Q2. request.getParameter("name")がnullを返すのはどんな場合ですか？

    回答を表示

    パラメータ名"name"がリクエストに含まれていない場合にnullを返します。フォームで未入力の場合や、そもそもその名前のパラメータが送信されていない場合です。

  3. Q3. 日本語の文字化けを防ぐために、どのような順序で設定すべきですか？

    回答を表示

    (1) request.setCharacterEncoding("UTF-8")を最初に設定 → (2) request.getParameter()でパラメータ取得 → (3) response.setContentType("text/html; charset=UTF-8")を設定 → (4) response.getWriter()で出力、という順序です。
