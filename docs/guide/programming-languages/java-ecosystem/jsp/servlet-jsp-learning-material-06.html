ホーム Servlet・JSP 第6章
第6章

フォーム処理とデータの受け渡し

HTMLフォームとServlet/JSPの連携によるデータ処理を学びます。

この章で学ぶこと

  • 各種HTMLフォーム要素を理解する
  • フォームデータをServlet/JSPで受け取れる
  • 入力値の検証ができる
  • ユーザーへの適切なフィードバックができる

6.1 HTMLフォームの要素

form要素の基本属性

HTMLフォームは、ユーザーからデータを受け取るための基本的な手段です。form要素には重要な属性があります。

  • action: フォームデータの送信先URL（Servlet等）
  • method: HTTPメソッド（GET/POST）
  • enctype: データのエンコーディング形式（ファイルアップロード時は multipart/form-data）
基本的なフォーム 
<!-- ユーザー登録フォーム -->
<form action="registerServlet" method="post">
    <label>名前: <input type="text" name="username"></label>
    <label>パスワード: <input type="password" name="password"></label>
    <button type="submit">登録</button>
</form>

テキスト入力系の要素

ユーザーからテキスト情報を受け取るための要素には、様々な種類があります。

テキスト入力の種類 
<!-- 1行テキスト -->
<input type="text" name="username" placeholder="ユーザー名">

<!-- パスワード（非表示） -->
<input type="password" name="password">

<!-- メールアドレス -->
<input type="email" name="email">

<!-- 複数行テキスト -->
<textarea name="comment" rows="5" cols="40"></textarea>

選択系の要素

ラジオボタン、チェックボックス、プルダウンメニューで選択肢から値を選ばせることができます。

選択要素の例 
<!-- ラジオボタン（単一選択） -->
<input type="radio" name="gender" value="male"> 男性
<input type="radio" name="gender" value="female"> 女性

<!-- チェックボックス（複数選択可） -->
<input type="checkbox" name="hobby" value="sports"> スポーツ
<input type="checkbox" name="hobby" value="reading"> 読書
<input type="checkbox" name="hobby" value="music"> 音楽

<!-- プルダウン -->
<select name="country">
    <option value="jp">日本</option>
    <option value="us">アメリカ</option>
    <option value="uk">イギリス</option>
</select>

name属性の重要性

各入力要素のname属性は、サーバー側でパラメータを取得する際のキーとなります。必ず設定しましょう。

6.2 フォームデータの処理

単一値の取得と処理

テキスト入力やラジオボタンなど、単一の値を持つパラメータはgetParameter()で取得します。

RegisterServlet.java - 単一値の取得 
@WebServlet("/register")
public class RegisterServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request,
                         HttpServletResponse response)
            throws ServletException, IOException {

        // 文字エンコーディング設定（最初に！）
        request.setCharacterEncoding("UTF-8");

        // パラメータ取得
        String username = request.getParameter("username");
        String email = request.getParameter("email");
        String gender = request.getParameter("gender");

        // nullチェックと処理
        if (username != null && !username.isEmpty()) {
            // データベース登録など
        }
    }
}

複数選択値の処理

チェックボックスや複数選択可能なselectは、getParameterValues()で配列として取得します。

複数値の取得 
// チェックボックスの値を配列で取得
String[] hobbies = request.getParameterValues("hobby");

if (hobbies != null) {
    for (String hobby : hobbies) {
        System.out.println("趣味: " + hobby);
    }
} else {
    // 何もチェックされていない場合
    System.out.println("趣味が選択されていません");
}

つまずきポイント

チェックボックスが1つもチェックされていない場合、パラメータ自体が送信されません。getParameterValues()はnullを返すため、必ずnullチェックを行いましょう。

未入力・未選択時の挙動

フォーム要素の種類によって、未入力・未選択時の挙動が異なります。

  • テキスト入力: 空文字（""）が送信される
  • ラジオボタン: 何も選択していない場合、パラメータなし（null）
  • チェックボックス: チェックなしの項目はパラメータなし（null）
  • select: 何も選択していない場合は空文字または最初のoptionの値

6.3 入力値の検証（バリデーション）

検証の重要性

ユーザーからの入力値は必ず検証（バリデーション）する必要があります。不正なデータを防ぎ、システムの安定性とセキュリティを保つためです。 

flowchart TD
    A[フォーム送信] --> B{入力値チェック}
    B -->|OK| C[データ登録処理]
    B -->|NG| D[エラー表示]
    C --> E[完了画面]
    D --> F[入力フォーム再表示]

基本的な検証パターン

必須チェック、形式チェック、範囲チェックなど、目的に応じた検証を行います。

バリデーションの例 
// エラーメッセージを格納するリスト
List<String> errors = new ArrayList<>();

// 必須チェック
if (username == null || username.isEmpty()) {
    errors.add("ユーザー名を入力してください");
}

// 長さチェック
if (username != null && username.length() < 3) {
    errors.add("ユーザー名は3文字以上で入力してください");
}

// メールアドレス形式チェック
if (!email.matches("^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$")) {
    errors.add("メールアドレスの形式が正しくありません");
}

// 数値範囲チェック
int age = Integer.parseInt(ageStr);
if (age < 0 || age > 150) {
    errors.add("年齢は0〜150の範囲で入力してください");
}

エラーメッセージの表示

検証エラーが発生した場合、ユーザーにわかりやすくエラーメッセージを表示します。

エラー処理とフォワード 
// エラーがある場合
if (!errors.isEmpty()) {
    // エラーメッセージをリクエスト属性に設定
    request.setAttribute("errors", errors);

    // 入力値も保持（再表示用）
    request.setAttribute("username", username);
    request.setAttribute("email", email);

    // 入力フォームに戻る
    RequestDispatcher dispatcher =
        request.getRequestDispatcher("register.jsp");
    dispatcher.forward(request, response);
    return;
}

// エラーがなければ処理を続行
// ...データベース登録など

実習 6-1: バリデーション機能の実装

ユーザー登録フォームにバリデーションを追加します。

手順

  1. 1 register.jspにユーザー名、メール、年齢の入力フォームを作成
  2. 2 RegisterServletで必須チェック、形式チェック、範囲チェックを実装
  3. 3 エラーがある場合、エラーメッセージをJSPで表示
  4. 4 正常時は完了ページを表示

期待される結果

入力ミスがある場合はエラーメッセージが表示され、正常な入力の場合は登録完了ページが表示される。

6.4 フォームの再表示

入力値の保持（Sticky Form）

バリデーションエラー時に、ユーザーが入力した値を保持して再表示することで、ユーザビリティが向上します。

JSPでの入力値保持 
<%@ page contentType="text/html; charset=UTF-8" %>

<!-- エラーメッセージの表示 -->
<% if (request.getAttribute("errors") != null) { %>
    <div class="error-message">
        <ul>
        <% for (String error : (List<String>)request.getAttribute("errors")) { %>
            <li><%= error %></li>
        <% } %>
        </ul>
    </div>
<% } %>

<!-- 入力値を保持して再表示 -->
<form action="register" method="post">
    <label>ユーザー名:
        <input type="text" name="username"
               value="<%= request.getAttribute("username") != null ?
                       request.getAttribute("username") : "" %>">
    </label>
    <label>メール:
        <input type="email" name="email"
               value="<%= request.getAttribute("email") != null ?
                       request.getAttribute("email") : "" %>">
    </label>
    <button type="submit">登録</button>
</form>

確認画面の実装

重要な情報の登録時には、確認画面を挟むことで入力ミスを防ぎます。 

flowchart LR
    A[入力画面] -->|送信| B{バリデーション}
    B -->|NG| A
    B -->|OK| C[確認画面]
    C -->|戻る| A
    C -->|確定| D[登録処理]
    D --> E[完了画面]
確認画面の実装 
// RegisterServlet.java
protected void doPost(HttpServletRequest request,
                     HttpServletResponse response)
        throws ServletException, IOException {

    String action = request.getParameter("action");

    if ("confirm".equals(action)) {
        // 入力→確認画面への遷移
        // バリデーション実施
        // 問題なければconfirm.jspへ
    } else if ("register".equals(action)) {
        // 確認→登録処理
        // データベース登録など
        // 完了画面へ
    }
}

重要ポイント

確認画面でhiddenフィールドを使って入力値を保持する方法もありますが、セキュリティ上の理由からセッションに保存する方が安全です。

エラー箇所のハイライト

エラーがあった項目を視覚的に強調することで、ユーザーが修正すべき箇所をすぐに認識できます。

エラー箇所の強調 
<!-- CSSクラスでエラー項目を強調 -->
<label>ユーザー名:
    <input type="text" name="username"
           class="<%= request.getAttribute("usernameError") != null ?
                   "error-input" : "" %>"
           value="<%= request.getAttribute("username") != null ?
                   request.getAttribute("username") : "" %>">
</label>
<% if (request.getAttribute("usernameError") != null) { %>
    <span class="error-text">
        <%= request.getAttribute("usernameError") %>
    </span>
<% } %>

理解度確認クイズ

  1. Q1. チェックボックスで複数の値を取得するには、どのメソッドを使いますか？

    回答を表示

    getParameterValues()メソッドを使います。これはString配列を返すため、複数の値を一度に取得できます。getParameter()では最初の1つしか取得できません。

  2. Q2. バリデーションエラー時に入力値を保持するには、どのような方法がありますか？

    回答を表示

    リクエスト属性に入力値を設定し、フォワードでJSPに渡します。JSP側ではvalue属性に<%=...%>でその値を出力することで、入力欄に値を再表示できます。

  3. Q3. なぜフォームのmethod属性にPOSTを使うことが推奨される場面がありますか？

    回答を表示

    POSTはパラメータがURLに表示されないため、機密情報（パスワード等）の送信に適しています。また、GETと違いデータ量の制限がないため、大量のデータ送信にも対応できます。

トラブルシューティング

チェックボックスの値が取得できない

原因: チェックされていない項目はパラメータとして送信されません

解決方法: getParameterValues()の戻り値がnullかどうかをチェックしてから処理を行います。必須の場合はバリデーションでエラーにします。

エラー後に入力値が消える

原因: リクエスト属性に入力値を設定していない、またはリダイレクトを使用している

解決方法: バリデーションエラー時は、入力値をリクエスト属性に設定し、フォワード（sendRedirectではなくforward）でJSPに戻します。

日本語入力が文字化けする

原因: 文字エンコーディングの設定が行われていない

解決方法: doPost()の最初の行でrequest.setCharacterEncoding("UTF-8")を実行します。getParameter()の前に設定しないと効果がありません。

この章のまとめ

  • HTMLフォームには様々な入力要素があり、それぞれ用途に応じて使い分けます
  • 単一値はgetParameter()、複数値はgetParameterValues()で取得します
  • 入力値は必ずバリデーションを行い、不正なデータを防ぎます
  • エラー時は入力値を保持して再表示することで、ユーザビリティが向上します
