Webアプリケーション開発の世界へ

Webアプリケーションの仕組みを理解し、開発環境をセットアップして最初のプログラムを動かすところまでを学びます。

1.1 Webアプリケーションとは

クライアント・サーバーモデル

Webアプリケーションは、クライアント（ブラウザ）とサーバーが通信して動作するシステムです。 ユーザーがブラウザで操作すると、その要求（リクエスト）がサーバーに送られ、サーバーが処理した結果（レスポンス）がブラウザに返されて画面に表示されます。 

flowchart LR
    A[ユーザー
ブラウザ] -->|1. リクエスト
URL入力/ボタンクリック| B[Webサーバー
Tomcat]
    B -->|2. レスポンス
HTML/データ| A
    B -->|処理| C[(データベース)]

静的Webサイトと動的Webアプリケーションの違い

静的Webサイトは、あらかじめ作成されたHTMLファイルをそのまま表示するだけです。 一方、動的Webアプリケーションは、ユーザーの入力やデータベースの情報に応じて、その場でHTMLを生成します。

静的Webサイト

  • • HTMLファイルをそのまま表示
  • • 全ユーザーに同じ内容
  • • 例: 会社案内、ブログ記事

動的Webアプリケーション

  • • プログラムでHTMLを生成
  • • ユーザーごとに異なる内容
  • • 例: ECサイト、SNS、業務システム

ServletとJSPの役割と関係性

ServletとJSPは、Java言語でWebアプリケーションを開発するための技術です。

  • S
    Servlet（サーブレット）

    Javaクラスとして書かれたプログラム。リクエストを受け取り、処理してレスポンスを返す。 ビジネスロジック（計算や判断）に向いている。

  • J
    JSP（JavaServer Pages）

    HTMLの中にJavaコードを埋め込んだファイル。画面の表示（View）に向いている。 実行時にServletに変換される。

役割分担のイメージ

Servletで「データ処理」、JSPで「画面表示」を分担することで、 保守しやすいアプリケーションが作れます。

1.2 HTTPプロトコルの基礎

HTTP（HyperText Transfer Protocol）は、ブラウザとサーバーが通信するときの「決まりごと」です。 この決まりに沿って、リクエストとレスポンスのやり取りが行われます。

HTTPリクエストとレスポンスの構造

sequenceDiagram
    participant Browser as ブラウザ
    participant Server as Webサーバー
    Browser->>Server: HTTPリクエスト
(メソッド, URL, ヘッダー, ボディ)
    Server->>Browser: HTTPレスポンス
(ステータスコード, ヘッダー, ボディ)

HTTPメソッド

HTTPメソッドは「何をするか」を表します。主に使うのは以下の4つです。

メソッド 用途 例
GET データの取得・表示 ページ表示、検索
POST データの送信・登録 フォーム送信、ログイン
PUT データの更新 プロフィール更新
DELETE データの削除 アカウント削除

ステータスコード

レスポンスには、処理結果を表す3桁の数値（ステータスコード）が含まれます。

200 OK（成功）

リクエストが正常に処理された

404 Not Found

指定されたページが見つからない

500 Internal Server Error

サーバー内でエラーが発生

302 Found（リダイレクト）

別のURLに転送

1.3 開発環境のセットアップ

必要なソフトウェア

Servlet/JSP開発には、以下の3つのソフトウェアが必要です。

JDK (Java Development Kit)

Javaプログラムの開発と実行に必要。JDK 17以降を推奨。

Apache Tomcat

ServletとJSPを動かすサーバー（サーブレットコンテナ）。Tomcat 10.x以降を推奨。

IDE (統合開発環境)

Eclipse、IntelliJ IDEA Communityなど。コーディングを効率化。

Tomcatのインストールと起動

Windows - 起動コマンド 
# Tomcatのbinフォルダに移動
cd C:\apache-tomcat-10.1.XX\bin

# Tomcat起動
startup.bat
macOS/Linux - 起動コマンド 
# Tomcatのbinフォルダに移動
cd /opt/tomcat/bin

# 実行権限を付与（初回のみ）
chmod +x *.sh

# Tomcat起動
./startup.sh

起動後、ブラウザで http://localhost:8080 にアクセスすると、 Tomcatのウェルカムページが表示されます。

動作確認

Tomcatの起動に成功すると、コンソールに「Server startup in XXX milliseconds」と表示されます。

1.4 Webアプリケーションの構造

ディレクトリ構成

Tomcatで動作するWebアプリケーションは、以下のような標準的なディレクトリ構造を持ちます。

標準ディレクトリ構成 

myapp/                    ← アプリケーションルート
├── index.html           ← 静的HTMLファイル
├── css/                 ← CSSファイル
├── js/                  ← JavaScriptファイル
├── images/              ← 画像ファイル
└── WEB-INF/             ← 外部から直接アクセス不可（重要）
    ├── web.xml          ← デプロイメント記述子（設定ファイル）
    ├── classes/         ← コンパイル済みクラスファイル (.class)
    │   └── com/
    │       └── example/
    │           └── MyServlet.class
    └── lib/             ← 外部ライブラリ (.jar)

重要ポイント

WEB-INFフォルダ内のファイルは、ブラウザから直接アクセスできません。 クラスファイルや設定ファイルなど、外部に公開したくないファイルを配置します。

web.xmlの役割

web.xmlは、Webアプリケーションの設定を記述するXMLファイルです（デプロイメント記述子）。 Servlet 3.0以降はアノテーション（@WebServlet等）で代替できますが、複雑な設定では今も使われます。

web.xml 
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="https://jakarta.ee/xml/ns/jakartaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="https://jakarta.ee/xml/ns/jakartaee
         https://jakarta.ee/xml/ns/jakartaee/web-app_6_0.xsd"
         version="6.0">

    <!-- アプリケーション名 -->
    <display-name>MyWebApp</display-name>

    <!-- ウェルカムファイルの設定 -->
    <welcome-file-list>
        <welcome-file>index.html</welcome-file>
        <welcome-file>index.jsp</welcome-file>
    </welcome-file-list>

</web-app>

つまずきやすいポイント

Tomcatが起動しない

原因: ポート8080が既に使用されている、またはJAVA_HOMEが設定されていない

解決方法:

  • 他のアプリケーションでポート8080を使っていないか確認
  • 環境変数JAVA_HOMEが正しく設定されているか確認
  • コマンドプロンプトで java -version を実行して確認
404エラーが発生する

原因: URLとファイル配置場所が一致していない

解決方法:

  • URLのスペルミスを確認
  • ファイルがwebapps/アプリ名/ 以下の正しい場所に配置されているか確認
  • Tomcatを再起動してみる
日本語が文字化けする

原因: ファイルのエンコーディングがUTF-8になっていない

解決方法:

  • IDEでファイルのエンコーディングをUTF-8に設定
  • HTMLファイルに <meta charset="UTF-8"> を記述

実習 1-1: Tomcatを起動して動作確認

Apache Tomcatをインストールし、起動してブラウザでアクセスできることを確認します。

手順

  1. 1 Apache TomcatをダウンロードしてインストールまたはZIPを解凍
  2. 2 コマンドラインまたはターミナルからTomcatを起動
  3. 3 ブラウザで http://localhost:8080 にアクセス
  4. 4 Tomcatのウェルカムページが表示されることを確認

期待される結果

ブラウザにTomcatの猫のロゴとウェルカムメッセージが表示されます。

理解度確認クイズ

  1. Q1. 静的Webサイトと動的Webアプリケーションの違いは何ですか？

    回答を表示

    静的Webサイトはあらかじめ作成されたHTMLファイルをそのまま表示しますが、 動的Webアプリケーションはユーザーの入力やデータベースの情報に応じてその場でHTMLを生成します。

  2. Q2. HTTPステータスコード「404」は何を意味しますか？

    回答を表示

    「Not Found」を意味し、指定されたページやリソースがサーバー上に見つからないことを示します。

  3. Q3. WEB-INFフォルダの役割は何ですか？

    回答を表示

    WEB-INFフォルダ内のファイルはブラウザから直接アクセスできません。 クラスファイル（.class）、ライブラリ（.jar）、設定ファイル（web.xml）など、 外部に公開したくないファイルを配置する場所です。
