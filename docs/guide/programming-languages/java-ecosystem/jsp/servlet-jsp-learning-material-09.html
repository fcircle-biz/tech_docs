第9章

JavaBeansとスコープ - データの構造化

JavaBeansを使ってデータを構造化し、4つのスコープを適切に使い分ける方法を学びます。

この章で学ぶこと

  • JavaBeansを作成できる
  • JSPのuseBeanアクションを使える
  • 4つのスコープの違いを理解し使い分けられる
  • 適切なスコープにデータを格納できる

9.1 JavaBeansとは

JavaBeans規約とは

JavaBeansは、再利用可能なJavaコンポーネントを作成するための規約です。主に以下の3つのルールを守る必要があります。

  • 引数なしコンストラクタ: デフォルトコンストラクタが必要
  • プロパティ: privateフィールド + public getter/setter
  • Serializable: シリアライズ可能（推奨）

なぜJavaBeansを使うのか

JavaBeansを使用することで、複数のデータをまとめて扱え、コードの可読性と保守性が向上します。JSPとServlet間でのデータ受け渡しが簡潔になります。

オブジェクト| C[JSP]

基本的なJavaBeansの例

UserBean.java 
package com.example.bean;

import java.io.Serializable;

// JavaBeans規約に従ったクラス
public class UserBean implements Serializable {
    // private フィールド
    private String name;
    private String email;
    private int age;

    // 引数なしコンストラクタ（必須）
    public UserBean() {
    }

    // getter/setter
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

重要ポイント

getter/setterの命名規則は厳密に守る必要があります。プロパティ名がnameの場合、getName()とsetName()という名前でなければなりません。

9.2 JSPでのJavaBeans使用

<jsp:useBean> - Beanの取得・生成

useBeanアクションは、指定されたスコープからBeanを取得するか、存在しなければ新しく生成します。

useBean基本構文 
<jsp:useBean id="user" class="com.example.bean.UserBean" scope="request" />
  • id: Bean変数の名前
  • class: Beanクラスの完全修飾名
  • scope: Bean格納スコープ（page/request/session/application）

<jsp:setProperty> - プロパティの設定

setPropertyの使い方 
<!-- 個別設定 -->
<jsp:setProperty name="user" property="name" value="山田太郎" />
<jsp:setProperty name="user" property="age" value="30" />

<!-- リクエストパラメータから自動設定 -->
<jsp:setProperty name="user" property="name" param="userName" />

<!-- 全プロパティ自動設定（パラメータ名とプロパティ名が一致） -->
<jsp:setProperty name="user" property="*" />

<jsp:getProperty> - プロパティの取得

getPropertyの使い方 
<p>名前: <jsp:getProperty name="user" property="name" /></p>
<p>年齢: <jsp:getProperty name="user" property="age" />歳</p>

ヒント

property="*" を使用すると、リクエストパラメータ名とプロパティ名が一致する全てのプロパティに自動的に値が設定されます。フォーム処理で便利です。

実践例: フォームからBeanへ

form.html 
<form action="result.jsp" method="post">
    <input type="text" name="name" placeholder="名前">
    <input type="email" name="email" placeholder="メール">
    <input type="number" name="age" placeholder="年齢">
    <button type="submit">送信</button>
</form>
result.jsp 
<%@ page contentType="text/html; charset=UTF-8" %>
<!-- Beanの生成と自動プロパティ設定 -->
<jsp:useBean id="user" class="com.example.bean.UserBean" scope="page" />
<jsp:setProperty name="user" property="*" />

<h1>登録情報</h1>
<p>名前: <jsp:getProperty name="user" property="name" /></p>
<p>メール: <jsp:getProperty name="user" property="email" /></p>
<p>年齢: <jsp:getProperty name="user" property="age" />歳</p>

9.3 4つのスコープ

スコープとは

スコープは、データが有効な範囲と期間を定義します。ServletとJSPでは4つのスコープがあり、それぞれ異なる用途に使用されます。

1. page スコープ

現在のJSPページ内でのみ有効。ページを離れると破棄されます。

  • 有効範囲: 現在のJSPページのみ
  • 対応オブジェクト: pageContext
  • 用途: ページ内の一時的なデータ

2. request スコープ

1つのHTTPリクエスト処理中（フォワード先でも有効）。レスポンス送信後に破棄されます。

  • 有効範囲: 1回のリクエスト処理全体
  • 対応オブジェクト: HttpServletRequest
  • 用途: Servlet→JSPのデータ受け渡し

3. session スコープ

同一ユーザーのセッション期間中有効。ログアウトまたはタイムアウトで破棄されます。

  • 有効範囲: セッションが継続する間
  • 対応オブジェクト: HttpSession
  • 用途: ログイン情報、カート情報

4. application スコープ

Webアプリケーション全体で共有。サーバー停止まで有効です。

  • 有効範囲: アプリケーション全体
  • 対応オブジェクト: ServletContext
  • 用途: 設定情報、全ユーザー共通カウンタ

スコープ別のコード例

各スコープの使用例 
<!-- page スコープ（デフォルト） -->
<jsp:useBean id="pageData" class="com.example.bean.DataBean" scope="page" />

<!-- request スコープ -->
<jsp:useBean id="reqData" class="com.example.bean.DataBean" scope="request" />

<!-- session スコープ -->
<jsp:useBean id="sessionData" class="com.example.bean.DataBean" scope="session" />

<!-- application スコープ -->
<jsp:useBean id="appData" class="com.example.bean.DataBean" scope="application" />

9.4 スコープの使い分け

スコープ選択の指針

適切なスコープを選択することで、メモリ効率が向上し、不具合を防げます。

実習 9-1: スコープ動作確認

各スコープの有効範囲を確認します。

手順

  1. 1 CounterBeanクラスを作成（countプロパティとincrementメソッド）
  2. 2 scope-test.jspを作成し、4つのスコープでCounterBeanを生成
  3. 3 各スコープのカウントを1ずつ増やして表示
  4. 4 ページをリロードして、各スコープのカウントがどう変化するか確認

期待される結果

リロードするたびにsessionとapplicationのカウントは増え続け、pageとrequestは常に1です。

実用的なスコープ使用例

LoginServlet.java（session使用例） 
// ログイン処理でユーザー情報をsessionに格納
UserBean user = new UserBean();
user.setName("山田太郎");
user.setEmail("yamada@example.com");

// sessionに保存
request.getSession().setAttribute("loginUser", user);

// JSPへフォワード
RequestDispatcher rd = request.getRequestDispatcher("mypage.jsp");
rd.forward(request, response);
mypage.jsp 
<%@ page contentType="text/html; charset=UTF-8" %>
<jsp:useBean id="loginUser" class="com.example.bean.UserBean" scope="session" />

<h1>マイページ</h1>
<p>ようこそ、<jsp:getProperty name="loginUser" property="name" />さん</p>
<p>メール: <jsp:getProperty name="loginUser" property="email" /></p>

pageContextによるスコープ操作

pageContextオブジェクトを使うと、任意のスコープにプログラムで属性を設定・取得できます。

pageContextの使用例 
// スコープを指定して属性を設定
pageContext.setAttribute("data", value, PageContext.REQUEST_SCOPE);
pageContext.setAttribute("data", value, PageContext.SESSION_SCOPE);
pageContext.setAttribute("data", value, PageContext.APPLICATION_SCOPE);

// スコープを指定して属性を取得
String data = (String) pageContext.getAttribute("data", PageContext.SESSION_SCOPE);

つまずきポイント

sessionやapplicationスコープは複数のユーザーやリクエストで共有されるため、大量のデータを格納するとメモリを圧迫します。必要最小限のデータのみ格納しましょう。

理解度確認クイズ

  1. Q1. JavaBeansの必須要件を3つ答えてください。

    回答を表示

    1) 引数なしのデフォルトコンストラクタがあること、2) フィールドはprivateで、public getter/setterでアクセスすること、3) Serializableインターフェースを実装することが推奨される。

  2. Q2. ログイン情報を保持する場合、どのスコープを使うべきですか？その理由も答えてください。

    回答を表示

    sessionスコープを使います。理由は、ログイン情報はユーザーごとに異なり、複数のページやリクエストにまたがって有効にする必要があるためです。

  3. Q3. <jsp:setProperty property="*" /> が行う処理を説明してください。

    回答を表示

    リクエストパラメータ名とJavaBeansのプロパティ名が一致するすべてのプロパティに、自動的に値を設定します。フォームの入力値をBeanに一括設定する際に便利です。

この章のまとめ

  • JavaBeansは再利用可能なデータコンポーネントで、引数なしコンストラクタとgetter/setterが必須です
  • JSPではuseBean、setProperty、getPropertyアクションでJavaBeansを簡単に扱えます
  • 4つのスコープ（page、request、session、application）はデータの有効範囲が異なります
  • 適切なスコープ選択により、メモリ効率と機能性を両立できます
