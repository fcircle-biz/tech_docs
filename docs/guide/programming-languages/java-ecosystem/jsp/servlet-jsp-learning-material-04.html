はじめてのJSP - HTMLにJavaを埋め込む

JSPの基本を学び、HTMLテンプレートの中でJavaコードを実行できるようになります。

この章で学ぶこと

  • JSPの概念とServletとの関係を理解する
  • JSPの基本構文を理解する
  • 動的なHTMLページを生成できる
  • JSPのライフサイクルを理解する

4.1 JSPとは何か

JSPの概要と特徴

JSP（JavaServer Pages）は、HTMLの中にJavaコードを埋め込んで動的なWebページを生成する技術です。HTMLを書きながらプログラムを組み込めるため、画面表示に特化したコードを簡潔に書くことができます。

  • HTMLベース: HTMLの中にJavaコードを埋め込む形式
  • 見た目重視: デザイナーとの協業がしやすい
  • 自動変換: 初回アクセス時にServletに変換される
  • 再利用可能: ファイルのインクルードで部品化できる

JSPとServletの関係

JSPファイルは、最初にアクセスされた時にServletのJavaコードに自動変換され、コンパイルされます。つまり、JSPは「HTMLっぽく書けるServlet」と言えます。 

graph LR
    A[JSPファイル
hello.jsp] -->|自動変換| B[Javaソース
hello_jsp.java]
    B -->|コンパイル| C[Servletクラス
hello_jsp.class]
    C -->|実行| D[HTMLレスポンス]

生成されたServletの確認方法

Tomcatの場合、work/Catalina/localhost/[アプリ名]/org/apache/jsp/ ディレクトリに、JSPから変換されたJavaファイルとclassファイルが生成されます。一度確認してみると理解が深まります。

JSPを使う利点

ServletでHTMLを出力する場合、すべてのHTML要素をprintln()で書く必要がありますが、JSPならHTMLを直接書けるため、画面表示のコードが非常に読みやすくなります。

Servletでの出力（読みにくい） 
// Servletでは全てJavaコードで書く必要がある
out.println("<html>");
out.println("<head><title>Hello</title></head>");
out.println("<body>");
out.println("<h1>Hello, " + name + "!</h1>");
out.println("</body>");
out.println("</html>");
JSPでの出力（読みやすい） 
<!-- JSPではHTMLを直接書ける -->
<html>
<head><title>Hello</title></head>
<body>
    <h1>Hello, <%= name %>!</h1>
</body>
</html>

4.2 JSPの基本構文

スクリプトレット <% ... %>

スクリプトレットは、Javaコードを実行するための基本的な構文です。<% と %> の間にJavaコードを書きます。

スクリプトレットの例 
<%
    // Javaコードを実行
    String message = "こんにちは";
    int count = 3;
%>

式 <%= ... %>

式は、変数や計算結果を直接HTMLに出力するための構文です。<%= と %> の間に式を書きます。セミコロンは不要です。

式の例 
<p>メッセージ: <%= message %></p>
<p>計算結果: <%= 10 + 20 %></p>
<p>現在時刻: <%= new java.util.Date() %></p>

宣言 <%! ... %>

宣言は、メソッドやクラス変数を定義するための構文です。通常はあまり使いません。

宣言の例 
<%!
    // メソッドの定義
    public int add(int a, int b) {
        return a + b;
    }
%>

コメント <%-- ... --%>

JSP専用のコメントです。HTMLコメント（<!-- -->）と異なり、クライアントに送信されません。

コメントの違い 
<%-- JSPコメント: クライアントに送信されない --%>
<!-- HTMLコメント: ソースに表示される -->

重要ポイント

式 <%= %> はセミコロン不要、スクリプトレット <% %> はセミコロン必要です。この違いを間違えるとエラーになります。

4.3 JSPでの制御構文

if文を使った条件分岐

スクリプトレットの中でif文を使い、条件に応じてHTMLを出力できます。

if文の例 
<%
    int score = 85;
%>
<% if (score >= 80) { %>
    <p style="color: blue;">優秀です！</p>
<% } else { %>
    <p style="color: red;">もう少し頑張りましょう。</p>
<% } %>

for文を使った繰り返し

for文を使って、リストの表示や繰り返し処理ができます。

for文の例 
<ul>
<% for (int i = 1; i <= 5; i++) { %>
    <li>項目 <%= i %></li>
<% } %>
</ul>

HTMLとJavaコードの組み合わせ方

JSPではJavaコードとHTMLを自由に組み合わせられますが、可読性を保つことが重要です。

実習 4-1: 現在時刻を表示するJSP

最初のJSPファイルを作成し、現在の日時を表示してみましょう。

time.jsp 
<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page import="java.time.*" %>
<html>
<head>
    <title>現在時刻</title>
</head>
<body>
    <h1>現在時刻表示</h1>
    <p>現在時刻: <%= LocalDateTime.now() %></p>
</body>
</html>

確認方法

1. webappsフォルダにtime.jspを配置
2. ブラウザでhttp://localhost:8080/myapp/time.jspにアクセス
3. 現在の日時が表示されれば成功

4.4 JSPのライフサイクル

JSPの処理フェーズ

JSPは、初回アクセス時に3つのフェーズを経てHTMLを生成します。 

graph LR
    A[1. 変換フェーズ
JSP→Java] --> B[2. コンパイルフェーズ
Java→class]
    B --> C[3. 実行フェーズ
HTMLレスポンス生成]
  • 変換フェーズ: JSPファイルがServletのJavaソースコードに変換される
  • コンパイルフェーズ: Javaソースコードがclassファイルにコンパイルされる
  • 実行フェーズ: Servletとして実行され、HTMLレスポンスが生成される

初回アクセス時の遅延

JSPは初回アクセス時に変換・コンパイルが行われるため、最初のアクセスは遅くなります。2回目以降はコンパイル済みのServletが実行されるため高速です。

対策方法

本番環境では、デプロイ時に全てのJSPを事前コンパイルする「プリコンパイル」を行うことで、初回アクセスでも高速に動作させることができます。

JSPの更新検知

JSPファイルを更新すると、Tomcatが自動的に変更を検知し、次回アクセス時に再コンパイルします。これにより、開発中は修正がすぐに反映されます。

つまずきポイント

JSPファイルを更新したのに変更が反映されない場合は、以下を試してください：
- ブラウザのキャッシュをクリア（スーパーリロード）
- Tomcatのworkディレクトリを削除して再起動
- ファイルの保存を確認

実習 4-2: 掛け算表（九九）の生成

for文を使って九九の表を生成するJSPを作成してみましょう。

kuku.jsp 
<%@ page contentType="text/html; charset=UTF-8" %>
<html>
<head>
    <title>九九の表</title>
</head>
<body>
    <h1>九九の表</h1>
    <table border="1">
    <% for (int i = 1; i <= 9; i++) { %>
        <tr>
        <% for (int j = 1; j <= 9; j++) { %>
            <td><%= i %> × <%= j %> = <%= i * j %></td>
        <% } %>
        </tr>
    <% } %>
    </table>
</body>
</html>

期待される結果

9×9のテーブルに九九の計算結果が表示されます。

理解度確認クイズ

  1. Q1. JSPファイルは初回アクセス時にどのように処理されますか？

    回答を表示

    JSPファイルは初回アクセス時に、Servletのjavaファイルに変換され、コンパイルされてclassファイルになります。その後、Servletとして実行されてHTMLレスポンスが生成されます。2回目以降はコンパイル済みのServletが実行されます。

  2. Q2. <%= %> と <% %> の違いは何ですか？

    回答を表示

    <%= %> は式で、値を直接HTMLに出力します（セミコロン不要）。<% %> はスクリプトレットで、Javaコードを実行します（セミコロン必要）。式は out.print() に変換され、スクリプトレットはそのままJavaコードとして実行されます。

  3. Q3. JSPとServletはどのような関係ですか？

    回答を表示

    JSPは最終的にServletに変換されて実行されます。つまり、JSPは「HTMLっぽく書けるServlet」です。Servletは処理やロジックに特化し、JSPは画面表示に特化しており、両者を組み合わせて使うことで効率的なWeb開発ができます。

この章のまとめ

  • JSPはHTMLの中にJavaコードを埋め込んで動的なページを生成する技術です
  • JSPは初回アクセス時にServletに変換・コンパイルされて実行されます
  • スクリプトレット <% %> でコード実行、式 <%= %> で値出力ができます
  • if文やfor文を使って動的にHTMLを生成できます
