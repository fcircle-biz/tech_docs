12.1 セキュリティ対策

XSS（クロスサイトスクリプティング）対策

XSSは、悪意のあるスクリプトをWebページに埋め込む攻撃です。ユーザー入力をそのまま出力せず、HTMLエスケープ処理を行うことで防止します。

危険なコード例

<%= request.getParameter("name") %> ← 悪意のあるスクリプトがそのまま実行される可能性

XSS対策 - JSTLのc:outを使用 
<!-- JSTLのc:outタグで自動エスケープ -->
<c:out value="${param.name}" />

<!-- デフォルト値も指定可能 -->
<c:out value="${param.name}" default="名無し" />

SQLインジェクション対策の徹底

SQLインジェクションは、悪意のあるSQL文を注入してデータベースを不正に操作する攻撃です。必ずPreparedStatementを使用します。

UserDAO.java - 安全なSQL実行 
// ユーザー検索メソッド（安全な実装）
public User findByEmail(String email) {
    String sql = "SELECT * FROM users WHERE email = ?";
    try (PreparedStatement ps = conn.prepareStatement(sql)) {
        ps.setString(1, email); // プレースホルダーに値をバインド
        ResultSet rs = ps.executeQuery();
        // 結果処理...
    }
}

CSRF対策の概念

CSRF（クロスサイトリクエストフォージェリ）は、ユーザーが意図しないリクエストを実行させる攻撃です。トークンを使って正規のフォームからの送信かを検証します。 

sequenceDiagram
    participant U as ユーザー
    participant W as Webアプリ
    participant A as 攻撃サイト

    U->>W: ログイン
    W->>U: セッション確立 + CSRFトークン発行
    U->>W: フォーム表示リクエスト
    W->>U: フォーム + CSRFトークン埋め込み
    Note over U,W: 正規の流れ
    U->>W: フォーム送信（トークン付き）
    W->>W: トークン検証 ✓
    W->>U: 処理成功

    Note over U,A: 攻撃の試み
    A->>W: 不正リクエスト（トークンなし/不正）
    W->>W: トークン検証 ✗
    W->>A: 403 Forbidden

パスワードの安全な扱い

パスワードは平文で保存せず、必ずハッシュ化（一方向暗号化）します。BCryptなどのアルゴリズムを使用します。

PasswordUtil.java - BCryptを使用 
import org.mindrot.jbcrypt.BCrypt;

public class PasswordUtil {
    // パスワードのハッシュ化
    public static String hashPassword(String password) {
        return BCrypt.hashpw(password, BCrypt.gensalt());
    }

    // パスワードの検証
    public static boolean checkPassword(String password, String hashed) {
        return BCrypt.checkpw(password, hashed);
    }
}

共通機能の実装（フィルター）

Filterを使って、文字エンコーディング設定や認証チェックなど、複数のServletで共通する処理を一箇所にまとめられます。

AuthFilter.java - 認証チェックフィルター 
@WebFilter("/*")
public class AuthFilter implements Filter {
    public void doFilter(ServletRequest req, ServletResponse res,
                        FilterChain chain) {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;

        // ログインページは認証不要
        String path = request.getRequestURI();
        if (path.endsWith("login.jsp") || path.endsWith("LoginServlet")) {
            chain.doFilter(req, res);
            return;
        }

        // セッションチェック
        HttpSession session = request.getSession(false);
        if (session == null || session.getAttribute("user") == null) {
            response.sendRedirect("login.jsp");
        } else {
            chain.doFilter(req, res);
        }
    }
}

ログ出力の基礎

System.out.printlnではなく、ログフレームワーク（SLF4J + Logback等）を使用します。

TaskService.java - ログ出力例 
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class TaskService {
    private static final Logger logger =
        LoggerFactory.getLogger(TaskService.class);

    public Task createTask(Task task) {
        logger.info("タスク作成開始: {}", task.getTitle());
        try {
            taskDAO.insert(task);
            logger.info("タスク作成成功: ID={}", task.getId());
            return task;
        } catch (Exception e) {
            logger.error("タスク作成失敗", e);
            throw new RuntimeException("タスク作成に失敗しました", e);
        }
    }
}

データベース設計

usersテーブルとtasksテーブルの2つのテーブルを作成します。

schema.sql - テーブル定義 
-- ユーザーテーブル
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- タスクテーブル
CREATE TABLE tasks (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT 'TODO',
    priority VARCHAR(10) DEFAULT 'MEDIUM',
    due_date DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

12.4 次のステップへ

Servlet/JSPの位置づけ

Servlet/JSPはJava Webアプリケーションの基礎技術です。現在は多くの現場でSpring Bootなどのフレームワークが使われていますが、基礎を学んだことでフレームワークの仕組みも理解しやすくなります。

重要ポイント

Servlet/JSPで学んだMVCパターン、セッション管理、データベース連携の概念は、どのフレームワークでも共通の基礎知識です。

Spring Framework/Spring Bootへの移行

Spring Bootは、Servlet/JSPの概念を基盤としながら、より効率的で保守性の高い開発を実現するフレームワークです。

Spring Bootの特徴

  • 設定の簡素化: アノテーションベースで設定が簡単
  • DIコンテナ: 依存性注入で疎結合な設計が可能
  • 組み込みサーバー: Tomcatが内蔵され、デプロイが容易
  • RESTful API: @RestControllerで簡単にAPI開発
  • Spring Data JPA: DAOを自動生成

Thymeleafなどのテンプレートエンジン

JSPに代わり、ThymeleafなどのモダンなテンプレートエンジンがSpring Bootで推奨されています。HTMLの自然な記述が可能です。

REST API開発への発展

最近のWebアプリケーションでは、バックエンドはREST APIとして実装し、フロントエンドはReact/Vue.jsなどのSPAフレームワークで構築する構成が主流です。

理解度確認クイズ

  1. Q1. XSS対策として適切な方法はどれですか？

    回答を表示

    JSTLの<c:out>タグを使ってHTMLエスケープを行います。これにより、ユーザー入力に含まれるスクリプトタグなどが無害化され、XSS攻撃を防止できます。

  2. Q2. パスワードを保存する際の正しい方法は？

    回答を表示

    BCryptなどのハッシュ化アルゴリズムを使って一方向暗号化します。平文やBase64エンコードでの保存は危険です。ハッシュ化により、データベースが漏洩しても元のパスワードは復元できません。

  3. Q3. Spring BootがServlet/JSPと比べて優れている点は？

    回答を表示

    設定の簡素化（アノテーションベース）、DIコンテナによる疎結合な設計、組み込みサーバーによる簡単なデプロイ、Spring Data JPAによるDAO自動生成など、開発効率と保守性が大幅に向上します。
