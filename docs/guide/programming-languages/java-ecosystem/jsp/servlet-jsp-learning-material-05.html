ホーム Servlet・JSP 第5章
第5章

JSPの構文をマスターする

ディレクティブ、アクションタグ、暗黙オブジェクトを学び、JSPを使いこなせるようになります。

この章で学ぶこと

  • pageディレクティブでページ属性を設定できる
  • includeディレクティブでファイルを組み込める
  • 標準アクションタグを使用できる
  • 暗黙オブジェクトを活用できる

5.1 pageディレクティブ

基本構文

pageディレクティブは、JSPページ全体に関する設定を行うためのディレクティブです。ページのエンコーディング、インポートするクラス、エラーページの指定などを行います。

pageディレクティブの基本構文 
<%@ page 属性名="値" 属性名="値" ... %>

<!-- 具体例 -->
<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page import="java.util.Date, java.util.List" %>
<%@ page errorPage="error.jsp" %>

主なpage属性

pageディレクティブで指定できる主な属性は以下の通りです。

  • contentType: MIMEタイプと文字エンコーディング
  • import: Javaクラスのインポート（複数可）
  • errorPage: エラー発生時の転送先ページ
  • isErrorPage: エラーページとして使用するか（true/false）
  • session: セッションを使用するか（デフォルト: true）
pageディレクティブの実例 
<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page import="java.util.Date" %>
<%@ page import="java.text.SimpleDateFormat" %>
<%@ page errorPage="error.jsp" %>
<!DOCTYPE html>
<html>
<head>
    <title>日時表示</title>
</head>
<body>
    <h1>現在の日時</h1>
    <%
        // インポートしたクラスを使用
        Date now = new Date();
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy年MM月dd日 HH:mm:ss");
    %>
    <p><%= sdf.format(now) %></p>
</body>
</html>

ヒント：複数importの記述方法

import属性は、複数のクラスをカンマ区切りで指定できます。また、複数のpageディレクティブに分けて記述することもできます。

5.2 includeディレクティブ

静的インクルードの仕組み

includeディレクティブは、他のファイルの内容を現在のJSPページに静的に組み込むためのディレクティブです。変換時（コンパイル時）にファイルが結合されます。

includeディレクティブの基本構文 
<%@ include file="相対パス" %>

<!-- 具体例 -->
<%@ include file="header.jsp" %>
<%@ include file="/common/footer.jsp" %>

共通ヘッダー・フッターの作成

Webサイトの複数ページで共通のヘッダーやフッターを使用する場合、includeディレクティブを使うと便利です。

実習 5-1: 共通ヘッダー・フッターの実装

includeディレクティブを使って共通部品を作成します。

header.jsp（共通ヘッダー）

<%@ page contentType="text/html; charset=UTF-8" %>
<header style="background: #333; color: white; padding: 20px;">
    <h1>私のWebサイト</h1>
    <nav>
        <a href="index.jsp" style="color: white;">ホーム</a> |
        <a href="about.jsp" style="color: white;">概要</a>
    </nav>
</header>

footer.jsp（共通フッター）

<footer style="background: #333; color: white; padding: 10px; text-align: center;">
    <p>© 2025 My Website. All rights reserved.</p>
</footer>

index.jsp（メインページ）

<%@ page contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<head>
    <title>ホーム</title>
</head>
<body>
    <%@ include file="header.jsp" %>

    <main style="padding: 20px;">
        <h2>ようこそ！</h2>
        <p>これはメインコンテンツです。</p>
    </main>

    <%@ include file="footer.jsp" %>
</body>
</html>

重要ポイント

includeディレクティブは変換時に結合されるため、インクルードされるファイルが変更された場合、インクルード元のJSPも再変換する必要があります。

5.3 標準アクションタグ

動的インクルード：<jsp:include>

jsp:includeアクションは、実行時（リクエスト処理時）に別のJSPやServletを呼び出してその結果を組み込みます。includeディレクティブとの違いは、実行タイミングです。

jsp:includeの使用例 
<!-- 基本的な使い方 -->
<jsp:include page="menu.jsp" />

<!-- パラメータを渡す場合 -->
<jsp:include page="display.jsp">
    <jsp:param name="title" value="お知らせ" />
    <jsp:param name="category" value="news" />
</jsp:include>

フォワード：<jsp:forward>

jsp:forwardアクションは、処理を別のJSPやServletに転送します。転送後、元のページには戻りません。

jsp:forwardの使用例 
<%@ page contentType="text/html; charset=UTF-8" %>
<%
    // ログイン状態をチェック
    String user = (String)session.getAttribute("user");
    if (user == null) {
        // 未ログインの場合はログインページへ転送
%>
        <jsp:forward page="login.jsp" />
<%
    }
%>
<!DOCTYPE html>
<html>
<body>
    <h1>ようこそ、<%= user %>さん！</h1>
</body>
</html>

静的インクルードとの違いと使い分け

includeディレクティブとjsp:includeアクションの違いを理解して、適切に使い分けましょう。 

graph TB
    subgraph 静的インクルード
        A1[includeディレクティブ] -->|変換時に結合| B1[1つのServletに変換]
    end

    subgraph 動的インクルード
        A2["jsp:includeアクション"] -->|実行時に呼び出し| B2[別々のServletとして実行]
    end

使い分けの指針

  • includeディレクティブ: 固定的な共通部品（ヘッダー、フッター）
  • jsp:include: 動的に変わる内容、パラメータを渡したい場合

5.4 JSP暗黙オブジェクト

暗黙オブジェクトとは

JSPでは、明示的に宣言しなくても使用できる組み込みオブジェクトが用意されています。これらを「暗黙オブジェクト」と呼びます。

主な暗黙オブジェクト

  • request: HttpServletRequest - リクエスト情報
  • response: HttpServletResponse - レスポンス情報
  • out: JspWriter - 出力ストリーム
  • session: HttpSession - セッション管理
  • application: ServletContext - アプリケーション全体の情報
  • config: ServletConfig - Servlet設定情報
  • pageContext: PageContext - ページコンテキスト
  • page: Object - 現在のServletインスタンス
  • exception: Throwable - 例外オブジェクト（エラーページのみ）

よく使う暗黙オブジェクトの例

暗黙オブジェクトの使用例 
<%@ page contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<body>
    <h1>暗黙オブジェクトの例</h1>

    <!-- requestオブジェクト -->
    <p>リクエストメソッド: <%= request.getMethod() %></p>
    <p>リクエストURI: <%= request.getRequestURI() %></p>

    <!-- sessionオブジェクト -->
    <p>セッションID: <%= session.getId() %></p>

    <!-- applicationオブジェクト -->
    <p>サーバー情報: <%= application.getServerInfo() %></p>

    <!-- outオブジェクト -->
    <%
        out.println("<p>outオブジェクトで出力</p>");
    %>
</body>
</html>

実習 5-2: エラーページの作成

pageディレクティブとexception暗黙オブジェクトを使ってエラーページを作成します。

error.jsp（エラーページ）

<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page isErrorPage="true" %>
<!DOCTYPE html>
<html>
<head>
    <title>エラー発生</title>
</head>
<body>
    <h1>エラーが発生しました</h1>
    <p>申し訳ございません。処理中にエラーが発生しました。</p>
    <div style="border: 1px solid #ccc; padding: 10px; background: #f0f0f0;">
        <h3>エラー詳細:</h3>
        <p><%= exception.getMessage() %></p>
    </div>
    <p><a href="index.jsp">トップページへ戻る</a></p>
</body>
</html>

test.jsp（テストページ）

<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page errorPage="error.jsp" %>
<!DOCTYPE html>
<html>
<body>
    <h1>エラーページのテスト</h1>
    <%
        // 意図的にエラーを発生させる
        int result = 10 / 0;  // ゼロ除算
    %>
</body>
</html>

動作確認

test.jspにアクセスすると、エラーが発生してerror.jspに転送され、エラーメッセージが表示されます。

理解度確認クイズ

  1. Q1. includeディレクティブとjsp:includeアクションの違いは何ですか？

    回答を表示

    includeディレクティブは変換時（コンパイル時）にファイルを結合する静的インクルードです。一方、jsp:includeアクションは実行時（リクエスト処理時）に他のページを呼び出す動的インクルードで、パラメータを渡すこともできます。

  2. Q2. pageディレクティブのerrorPage属性とisErrorPage属性の関係を説明してください。

    回答を表示

    errorPage属性は、エラーが発生したときに転送先となるページを指定します。isErrorPage="true"は、そのページがエラーページであることを示し、exception暗黙オブジェクトを使用できるようになります。

  3. Q3. JSPの暗黙オブジェクトで、リクエストパラメータを取得するのはどれですか？

    回答を表示

    requestオブジェクト（HttpServletRequest）を使用します。request.getParameter("パラメータ名")でリクエストパラメータを取得できます。

トラブルシューティング

インクルードファイルが見つからない

原因: ファイルパスの指定が間違っている

解決策: 相対パスと絶対パスの違いを確認しましょう。"/"で始まるパスはアプリケーションルートからの絶対パス、それ以外は現在のファイルからの相対パスです。

文字化けが発生する

原因: pageディレクティブでのエンコーディング指定漏れ

解決策: 各JSPファイルの先頭に必ず <%@ page contentType="text/html; charset=UTF-8" %> を記述しましょう。

暗黙オブジェクトが使えない

原因: 使用できないスコープで呼び出している

解決策: exceptionオブジェクトはisErrorPage="true"のページでのみ使用可能です。sessionオブジェクトはsession="false"のページでは使用できません。

  • pageディレクティブでエンコーディング、インポート、エラーページを設定できます
  • includeディレクティブで共通部品を静的に組み込めます
  • jsp:includeとjsp:forwardで動的なページ制御ができます
  • request、response、session、applicationなどの暗黙オブジェクトを活用できます
