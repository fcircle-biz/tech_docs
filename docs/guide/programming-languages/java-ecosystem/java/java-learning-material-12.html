第12章

エラーに対処する - 例外処理

プログラム実行中に起きるエラー(例外)に対処する方法を学び、エラーが起きてもプログラムが止まらないようにする技術を身につけましょう。

この章で学ぶこと

  • 例外とは何か、なぜエラー対処が必要なのかを理解する
  • try-catch文を使って例外をキャッチし、適切に対処する方法を学ぶ
  • finally句を使って必ず実行したい処理を書く方法を学ぶ
  • 複数の例外をキャッチする方法と、例外をスローする方法を理解する

12.1 例外って何だろう?

プログラムを実行していると、時々予期しないことが起きます。 例えば、ファイルが見つからない、数値に変換できない文字列を受け取った、0で割り算をしようとしたなど...。 このような予期しないエラーや異常な状況のことを、Javaでは例外(Exception)と呼びます。

例外とは?

プログラム実行中に起こる「予期しない問題」のことです。 例外が発生すると、何も対処しなければプログラムはその時点で停止してしまいます。

例外が起きる例

例外処理を使わないと、小さなエラー1つでプログラム全体が止まってしまいます。 しかし、例外処理を使えば、エラーが起きてもプログラムを続けたり、エラーメッセージをユーザーに分かりやすく伝えたりできます。

12.2 try-catch文でエラーを捕まえる

例外に対処するには、try-catch文を使います。 「try」は「試す」、「catch」は「捕まえる」という意味です。

try-catch文の基本構文 

try {
    // 例外が起きるかもしれない処理
} catch (例外の型 変数名) {
    // 例外が起きたときの対処
}

たとえ話: 料理のレシピ

例外処理なし

「卵を割る」→ 卵が腐っていたらそこで料理を諦める(プログラム停止)

例外処理あり

「卵を割る(try)」→ 腐っていたら(catch)別の卵を使う、または卵なしレシピに変更する

実際の例: 数値変換

ユーザーが入力した文字列を数値に変換しようとするとき、数値に変換できない文字列が来るとエラーになります。

ExceptionDemo.java 
public class ExceptionDemo {
    public static void main(String[] args) {
        String input = "abc"; // 数値に変換できない文字列

        try {
            // 文字列を整数に変換しようとする
            int number = Integer.parseInt(input);
            System.out.println("変換成功: " + number);
        } catch (NumberFormatException e) {
            // 数値に変換できなかったときの処理
            System.out.println("エラー: 数値に変換できません");
        }

        System.out.println("プログラムは続きます");
    }
}

実行結果

エラー: 数値に変換できません
プログラムは続きます

例外が発生しても、catch句で対処したため、プログラムは止まらずに続きました！

ポイント

  • ・tryブロック: 例外が起きるかもしれない処理を書く
  • ・catchブロック: 例外が起きたときの対処を書く
  • ・NumberFormatException: 数値変換に失敗したときに発生する例外
  • ・例外が起きなければcatch句はスキップされる

12.3 例外オブジェクトから情報を得る

catch句でキャッチした例外は「オブジェクト」として受け取ります。 このオブジェクトには、エラーの詳細情報が含まれています。

ExceptionInfo.java 
public class ExceptionInfo {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // 0で割り算 → 例外発生
        } catch (ArithmeticException e) {
            // 例外の詳細情報を表示
            System.out.println("エラーメッセージ: " + e.getMessage());
            System.out.println("例外の型: " + e.getClass().getName());
        }
    }
}
メソッド 説明
getMessage() エラーの簡潔なメッセージを取得
printStackTrace() エラーの詳細な追跡情報を表示(デバッグ用)
getClass().getName() 例外の型名を取得

12.4 必ず実行したい処理 - finally句

例外が起きても起きなくても、必ず実行したい処理がある場合は、 finally句を使います。

try-catch-finally文の構文 

try {
    // 例外が起きるかもしれない処理
} catch (例外の型 変数名) {
    // 例外が起きたときの対処
} finally {
    // 必ず実行される処理
}

finally句の実行フロー

flowchart TD A([tryブロック開始]) --> B{例外発生?} B -- No --> C[tryブロック最後まで実行] B -- Yes --> D[catchブロック実行] C --> E[finallyブロック実行] D --> E E --> F([終了]) style A fill:#fff7ed,stroke:#f97316 style E fill:#dcfce7,stroke:#22c55e style F fill:#f1f5f9,stroke:#64748b

finallyの使用例: ファイル操作

FinallyDemo.java 
public class FinallyDemo {
    public static void main(String[] args) {
        System.out.println("処理開始");

        try {
            System.out.println("tryブロック実行");
            int result = 10 / 0; // 例外発生
            System.out.println("この行は実行されない");
        } catch (ArithmeticException e) {
            System.out.println("catchブロック: エラーを捕捉");
        } finally {
            System.out.println("finallyブロック: 必ず実行");
        }

        System.out.println("処理終了");
    }
}

実行結果

処理開始
tryブロック実行
catchブロック: エラーを捕捉
finallyブロック: 必ず実行
処理終了

finallyの使いどころ

  • ・ファイルを開いたら、必ず閉じる
  • ・データベース接続を開いたら、必ず切断する
  • ・リソース(メモリなど)を確保したら、必ず解放する

12.5 複数の例外をキャッチする

プログラムによっては、異なる種類の例外が起きる可能性があります。 複数のcatch句を書くことで、それぞれの例外に対して異なる対処ができます。

MultipleCatch.java 
public class MultipleCatch {
    public static void main(String[] args) {
        String[] numbers = {"10", "20", "abc"};

        for (String str : numbers) {
            try {
                // 配列の要素を数値に変換して表示
                int num = Integer.parseInt(str);
                System.out.println("変換成功: " + num);
                int result = 100 / num; // 0だと例外発生
                System.out.println("計算結果: " + result);

            } catch (NumberFormatException e) {
                // 数値変換エラー
                System.out.println("数値に変換できません: " + str);

            } catch (ArithmeticException e) {
                // 算術エラー
                System.out.println("0で割ることはできません");
            }
        }
    }
}

複数例外を1つのcatch句で処理

Java 7以降では、複数の例外を1つのcatch句でまとめて処理することもできます。

MultiCatchSingle.java 
public class MultiCatchSingle {
    public static void main(String[] args) {
        try {
            // 何らかの処理
        } catch (NumberFormatException | ArithmeticException e) {
            // 2つの例外をまとめて処理
            System.out.println("数値エラーまたは計算エラー: " + e.getMessage());
        }
    }
}

12.6 例外を投げる - throw

プログラムの中で意図的に例外を発生させたい場合は、throwキーワードを使います。 これは「例外を投げる(スローする)」と表現します。

ThrowDemo.java 
public class ThrowDemo {
    // 年齢をチェックするメソッド
    public static void checkAge(int age) {
        if (age < 0) {
            // 例外を投げる
            throw new IllegalArgumentException("年齢は0以上でなければなりません");
        }
        System.out.println("年齢: " + age + "歳");
    }

    public static void main(String[] args) {
        try {
            checkAge(25);  // 正常
            checkAge(-5);  // 例外発生
        } catch (IllegalArgumentException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}

throwの使いどころ

  • ・無効なデータが渡されたとき
  • ・処理が続行できない状況になったとき
  • ・他のメソッドに例外を伝えたいとき

12.7 よく使う例外の種類

Javaには様々な種類の例外が用意されています。代表的なものを知っておくと、エラーメッセージを見たときに何が起きているか分かります。

例外名 発生する状況
NullPointerException nullのオブジェクトのメソッドを呼んだ
ArrayIndexOutOfBoundsException 配列の範囲外にアクセスした
ArithmeticException 算術エラー(0で割り算など)
NumberFormatException 文字列を数値に変換できない
IllegalArgumentException メソッドに不正な引数が渡された
FileNotFoundException 指定されたファイルが見つからない

実習 12-1: 電卓プログラムに例外処理を追加しよう

ユーザーから2つの数値を受け取って計算を行う簡単な電卓プログラムを作成し、例外処理を実装します。

手順

  1. 1 新しいファイル「Calculator.java」を作成
  2. 2 以下のコードを入力:
Calculator.java 
import java.util.Scanner;

public class Calculator {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        try {
            // 数値入力
            System.out.print("1つ目の数値を入力: ");
            String input1 = scanner.nextLine();
            int num1 = Integer.parseInt(input1);

            System.out.print("2つ目の数値を入力: ");
            String input2 = scanner.nextLine();
            int num2 = Integer.parseInt(input2);

            // 計算実行
            int result = num1 / num2;
            System.out.println("結果: " + num1 + " ÷ " + num2 + " = " + result);

        } catch (NumberFormatException e) {
            System.out.println("エラー: 数値を入力してください");
        } catch (ArithmeticException e) {
            System.out.println("エラー: 0で割ることはできません");
        } finally {
            scanner.close();
            System.out.println("計算プログラムを終了します");
        }
    }
}
  1. 3 プログラムを実行し、様々な入力を試してみましょう

試してほしいこと

  • ・正常な数値(例: 10と2)を入力
  • ・文字列(例: abcとdef)を入力
  • ・2つ目の数値として0を入力

理解度確認クイズ

  1. Q1. 例外とは何ですか?

    回答を表示

    プログラム実行中に起こる予期しないエラーや異常な状況のことです。 例外が発生すると、何も対処しなければプログラムはその時点で停止してしまいます。

  2. Q2. try-catch文のtryブロックとcatchブロックはそれぞれ何をするためのものですか?

    回答を表示

    tryブロック: 例外が起きるかもしれない処理を書く場所。
    catchブロック: 例外が起きたときの対処を書く場所。例外が起きなければスキップされます。

  3. Q3. finally句はいつ実行されますか?

    回答を表示

    例外が起きても起きなくても、必ず実行されます。 ファイルを閉じる、リソースを解放するなど、必ず実行したい処理を書くのに使います。

  4. Q4. throwキーワードは何のために使いますか?

    回答を表示

    プログラムの中で意図的に例外を発生させる(スローする)ために使います。 無効なデータが渡されたときや、処理が続行できない状況になったときに使用します。

12.8 この章のまとめ

  • 例外はプログラム実行中に起きる予期しないエラーや異常な状況のこと。
  • try-catch文を使うことで、例外が起きてもプログラムを続けることができる。
  • catch句でキャッチした例外オブジェクトから、エラーの詳細情報を取得できる。
  • finally句は例外の有無に関わらず必ず実行される。リソースの解放などに使う。
  • 複数のcatch句を書くことで、異なる種類の例外に対して異なる対処ができる。
  • throwキーワードを使って、意図的に例外を発生させることができる。
