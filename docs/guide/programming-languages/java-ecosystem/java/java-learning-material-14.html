14.1 コレクションって何だろう？

第5章で学んだ配列は、複数のデータをまとめて管理する便利な仕組みでした。 しかし、配列には「一度作ったらサイズを変更できない」という制限があります。

一方、コレクション（Collection）は、配列よりも柔軟にデータを管理できる仕組みです。 要素の追加・削除が自由にでき、便利なメソッドもたくさん用意されています。

コレクションフレームワーク

Javaには「コレクションフレームワーク」という、データを管理するための道具がたくさん用意されています。 この章では、その中でも特によく使われる3つ（ArrayList、HashMap、HashSet）を学びます。

14.2 ArrayList - サイズが変わるリスト

ArrayListは、要素を順番に並べて管理するコレクションです。 配列と似ていますが、要素の追加・削除が自由にできる点が大きく異なります。

ArrayListの基本操作

ArrayListBasic.java 
import java.util.ArrayList;

public class ArrayListBasic {
    public static void main(String[] args) {
        // String型のArrayListを作成
        ArrayList<String> fruits = new ArrayList<>();

        // 要素を追加
        fruits.add("りんご");
        fruits.add("バナナ");
        fruits.add("みかん");

        // 要素数を表示
        System.out.println("果物の数: " + fruits.size());

        // すべての要素を表示
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}

ポイント

  • ・<String>：中に入れる要素の型を指定（ジェネリクス）
  • ・add()：要素を追加するメソッド
  • ・size()：要素数を取得するメソッド

よく使うArrayListのメソッド

メソッド 説明 使用例
add(要素) リストの末尾に要素を追加 list.add("項目")
get(インデックス) 指定位置の要素を取得 list.get(0)
remove(インデックス) 指定位置の要素を削除 list.remove(1)
size() 要素数を取得 list.size()
contains(要素) 要素が含まれているか確認 list.contains("項目")
clear() すべての要素を削除 list.clear()

実習 14-1: ArrayListで買い物リストを作ろう

ArrayListを使って、買い物リストを管理するプログラムを作成します。

ShoppingList.java 
import java.util.ArrayList;

public class ShoppingList {
    public static void main(String[] args) {
        // 買い物リストを作成
        ArrayList<String> list = new ArrayList<>();

        // アイテムを追加
        list.add("牛乳");
        list.add("パン");
        list.add("卵");
        System.out.println("追加後: " + list);

        // 特定のアイテムを削除
        list.remove("パン");
        System.out.println("削除後: " + list);

        // アイテムが含まれるか確認
        if (list.contains("牛乳")) {
            System.out.println("牛乳は買い物リストにあります");
        }
    }
}

期待される結果

追加後: [牛乳, パン, 卵]
削除後: [牛乳, 卵]
牛乳は買い物リストにあります

14.3 HashMap - キーと値のペアで管理

HashMapは、「キー」と「値」のペアでデータを管理するコレクションです。 辞書のように、キーを指定すると対応する値を取り出せます。

たとえ話: 辞書で単語を引く

英和辞典

  • • apple → りんご
  • • banana → バナナ
  • • cat → ネコ

英単語（キー）で日本語（値）を探す

HashMap

  • • "apple" → "りんご"
  • • "banana" → "バナナ"
  • • "cat" → "ネコ"

キーで値を高速検索

HashMapの基本操作

HashMapBasic.java 
import java.util.HashMap;

public class HashMapBasic {
    public static void main(String[] args) {
        // String型のキーとInteger型の値を持つHashMap
        HashMap<String, Integer> scores = new HashMap<>();

        // データを追加（キー、値）
        scores.put("太郎", 85);
        scores.put("花子", 92);
        scores.put("次郎", 78);

        // キーを指定して値を取得
        int score = scores.get("花子");
        System.out.println("花子の点数: " + score);

        // すべてのキーと値を表示
        for (String name : scores.keySet()) {
            System.out.println(name + ": " + scores.get(name));
        }
    }
}

ポイント

  • ・<String, Integer>：キーの型と値の型を指定
  • ・put(キー, 値)：データを追加または更新
  • ・get(キー)：キーに対応する値を取得
  • ・keySet()：すべてのキーを取得

よく使うHashMapのメソッド

メソッド 説明 使用例
put(キー, 値) キーと値のペアを追加 map.put("A", 100)
get(キー) キーに対応する値を取得 map.get("A")
remove(キー) 指定キーのペアを削除 map.remove("A")
containsKey(キー) キーが存在するか確認 map.containsKey("A")
size() ペアの数を取得 map.size()
keySet() すべてのキーを取得 map.keySet()

実習 14-2: HashMapで電話帳を作ろう

HashMapを使って、名前と電話番号を管理する簡単な電話帳を作成します。

PhoneBook.java 
import java.util.HashMap;

public class PhoneBook {
    public static void main(String[] args) {
        // 電話帳を作成
        HashMap<String, String> phoneBook = new HashMap<>();

        // 連絡先を追加
        phoneBook.put("山田太郎", "090-1234-5678");
        phoneBook.put("佐藤花子", "080-9876-5432");
        phoneBook.put("鈴木次郎", "070-1111-2222");

        // 特定の人の電話番号を検索
        String name = "佐藤花子";
        if (phoneBook.containsKey(name)) {
            System.out.println(name + "の電話番号: " + phoneBook.get(name));
        }

        // 全員の連絡先を表示
        System.out.println("\n全員の連絡先:");
        for (String person : phoneBook.keySet()) {
            System.out.println(person + ": " + phoneBook.get(person));
        }
    }
}

期待される結果

佐藤花子の電話番号: 080-9876-5432

全員の連絡先:
山田太郎: 090-1234-5678
佐藤花子: 080-9876-5432
鈴木次郎: 070-1111-2222

14.4 HashSet - 重複しない要素の集合

HashSetは、重複しない要素の集合を管理するコレクションです。 同じ値を何度追加しても、1つしか保存されません。

たとえ話: 参加者リスト

イベントの参加者リストを作るとき、同じ人が何度申し込んでも1人としてカウントしたい場合にHashSetが便利です。

参加申し込み

  • 1. 太郎さんが申し込み → リストに追加
  • 2. 花子さんが申し込み → リストに追加
  • 3. 太郎さんがもう一度申し込み → すでにいるので追加されない

結果: 参加者は2人（太郎、花子）

HashSetの基本操作

HashSetBasic.java 
import java.util.HashSet;

public class HashSetBasic {
    public static void main(String[] args) {
        // String型のHashSetを作成
        HashSet<String> tags = new HashSet<>();

        // タグを追加
        tags.add("Java");
        tags.add("プログラミング");
        tags.add("初心者");
        tags.add("Java");  // 重複 - 追加されない

        // 要素数を表示
        System.out.println("タグの数: " + tags.size());

        // すべての要素を表示
        for (String tag : tags) {
            System.out.println(tag);
        }
    }
}

注意

HashSetは要素の順番を保証しません。追加した順番とは異なる順序で要素が取り出される場合があります。 順番が重要な場合はArrayListを使いましょう。

実習 14-3: HashSetで重複を除去しよう

HashSetを使って、配列から重複する要素を取り除くプログラムを作成します。

RemoveDuplicates.java 
import java.util.HashSet;

public class RemoveDuplicates {
    public static void main(String[] args) {
        // 重複がある配列
        String[] words = {"Java", "Python", "Java", "C++", "Python", "Ruby"};

        // HashSetで重複を除去
        HashSet<String> uniqueWords = new HashSet<>();
        for (String word : words) {
            uniqueWords.add(word);
        }

        // 結果を表示
        System.out.println("元の配列: " + words.length + "個");
        System.out.println("重複除去後: " + uniqueWords.size() + "個");
        System.out.println("ユニークな言語: " + uniqueWords);
    }
}

期待される結果

元の配列: 6個
重複除去後: 4個
ユニークな言語: [Java, Ruby, C++, Python]

14.6 ジェネリクス - 型の安全性

コレクションを使うときに見た<String>や<Integer>のような記法を ジェネリクス（Generics）といいます。

ジェネリクスを使うことで、「このコレクションにはこの型のデータしか入れない」と宣言でき、 間違った型のデータを入れようとするとコンパイル時にエラーが出るようになります。

❌ ジェネリクスなし（非推奨）

ArrayList list = new ArrayList();
list.add("文字列");
list.add(123);  // 違う型も入る
// 取り出すときにエラーの可能性

✅ ジェネリクスあり（推奨）

ArrayList<String> list = new ArrayList<>();
list.add("文字列");
list.add(123);  // コンパイルエラー
// 型が保証されて安全

ポイント

ジェネリクスは「型の安全性」を高める仕組みです。 コレクションを使うときは必ず型を指定しましょう。

理解度確認クイズ

  1. Q1. 配列とArrayListの大きな違いは何ですか？

    回答を表示

    配列はサイズが固定で変更できませんが、ArrayListはサイズが可変で、要素の追加・削除が自由にできます。 また、ArrayListは便利なメソッド（add、remove、containsなど）が豊富に用意されています。

  2. Q2. HashMapはどのような場面で使うと便利ですか？

    回答を表示

    「キー」を使って「値」を素早く検索したい場面で便利です。 例：辞書（単語→意味）、電話帳（名前→電話番号）、設定値（項目名→値）など。

  3. Q3. HashSetの最大の特徴は何ですか？

    回答を表示

    重複を許さない点です。同じ値を何度追加しても、1つしか保存されません。 重複を除去したい場合や、ユニークな値のみを管理したい場合に便利です。

  4. Q4. ジェネリクス（<String>など）を使う理由は何ですか？

    回答を表示

    型の安全性を高めるためです。コレクションに入れるデータの型を明示することで、 間違った型のデータを入れようとするとコンパイル時にエラーが出るため、バグを防げます。

14.7 この章のまとめ

  • コレクションは配列よりも柔軟にデータを管理できる仕組み。
  • ArrayListは順番を保持し、重複を許可する可変長のリスト。
  • HashMapはキーと値のペアでデータを管理し、高速検索が可能。
  • HashSetは重複しない要素の集合を管理する。
  • ジェネリクスを使うことで型の安全性が保証される。
  • 目的に応じてコレクションを適切に使い分けることが重要。
