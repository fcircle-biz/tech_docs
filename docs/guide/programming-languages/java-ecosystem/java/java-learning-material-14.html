ラムダ式と関数型プログラミング

Java 8で導入されたラムダ式について学びます。関数型インターフェース、メソッド参照、Stream APIの基本的な使い方を理解し、より簡潔で読みやすいコードの書き方を習得します。

この章で学ぶこと

  • ラムダ式の基本構文と使い方
  • 関数型インターフェースの概念と標準インターフェース
  • メソッド参照による簡潔な記述方法
  • Stream APIを使ったコレクション操作の基礎

14.1 ラムダ式とは何か

ラムダ式（Lambda Expression）は、メソッドを簡潔に記述するための構文です。Java 8から導入され、関数型プログラミングの考え方を取り入れることで、コードをより読みやすく、保守しやすくすることができます。

従来のJavaでは、メソッドを渡すために匿名クラスを使う必要がありましたが、ラムダ式を使うことで、同じ処理をはるかに短いコードで記述できます。

従来の記述方法（匿名クラス）

AnonymousExample.java 
// リストをソートする従来の方法
List<String> names = Arrays.asList("太郎", "花子", "次郎");

Collections.sort(names, new Comparator<String>() {
    @Override
    public int compare(String a, String b) {
        return a.compareTo(b);
    }
});

ラムダ式を使った記述

LambdaExample.java 
// ラムダ式を使った簡潔な記述
List<String> names = Arrays.asList("太郎", "花子", "次郎");

Collections.sort(names, (a, b) -> a.compareTo(b));

ラムダ式のメリット

ラムダ式を使うと、匿名クラスで10行近く必要だったコードが1行で書けます。これにより、コードの可読性が大幅に向上し、開発効率も上がります。

14.2 ラムダ式の基本構文

ラムダ式の基本的な形式は以下の通りです。 

// 基本構文
(引数リスト) -> { 処理本体 }

// 引数が1つで処理が1行の場合は省略可能
引数 -> 処理式

ラムダ式の記述パターン

LambdaSyntax.java 
// パターン1: 引数なし
() -> System.out.println("Hello");

// パターン2: 引数1つ（括弧省略可）
x -> x * x

// パターン3: 引数複数
(x, y) -> x + y

// パターン4: 複数行の処理（ブロック必要）
(x, y) -> {
    int result = x + y;
    return result;
}

14.3 関数型インターフェース

ラムダ式は、関数型インターフェースと呼ばれる特別なインターフェースの実装として扱われます。関数型インターフェースとは、抽象メソッドを1つだけ持つインターフェースのことです。

関数型インターフェースの定義

Calculator.java 
// 関数型インターフェースの例
@FunctionalInterface
interface Calculator {
    // 抽象メソッドは1つだけ
    int calculate(int a, int b);
}

// ラムダ式で実装
Calculator add = (a, b) -> a + b;
Calculator multiply = (a, b) -> a * b;

System.out.println(add.calculate(5, 3));      // 8
System.out.println(multiply.calculate(5, 3)); // 15

標準関数型インターフェース

Javaには、よく使われる関数型インターフェースがjava.util.functionパッケージに用意されています。

  • Function<T, R>

    引数を受け取って結果を返す（T → R）

  • Predicate<T>

    引数を受け取ってboolean（真偽値）を返す

  • Consumer<T>

    引数を受け取って処理を実行（戻り値なし）

  • Supplier<T>

    引数なしで値を生成して返す

FunctionalInterfaces.java 
import java.util.function.*;

// Function: 文字列長を返す
Function<String, Integer> length = s -> s.length();
System.out.println(length.apply("Hello")); // 5

// Predicate: 偶数判定
Predicate<Integer> isEven = n -> n % 2 == 0;
System.out.println(isEven.test(4)); // true

// Consumer: 表示処理
Consumer<String> print = s -> System.out.println(s);
print.accept("こんにちは");

// Supplier: 乱数生成
Supplier<Double> random = () -> Math.random();
System.out.println(random.get());

14.4 メソッド参照

メソッド参照（Method Reference）は、既存のメソッドをラムダ式の代わりに使うためのさらに簡潔な記法です。::演算子を使って記述します。

メソッド参照の種類

MethodReference.java 
// 1. 静的メソッド参照
Function<String, Integer> parseInt = Integer::parseInt;
System.out.println(parseInt.apply("123")); // 123

// 2. インスタンスメソッド参照
String str = "Hello";
Supplier<Integer> length = str::length;
System.out.println(length.get()); // 5

// 3. 特定型のインスタンスメソッド参照
Function<String, String> upper = String::toUpperCase;
System.out.println(upper.apply("hello")); // HELLO

// 4. コンストラクタ参照
Supplier<ArrayList<String>> listSupplier = ArrayList::new;
List<String> list = listSupplier.get();

重要ポイント

メソッド参照は、ラムダ式が既存のメソッドをそのまま呼び出すだけの場合に使用します。例えば、x -> System.out.println(x)はSystem.out::printlnと書けます。

14.5 Stream APIの基礎

Stream APIは、コレクションを関数型スタイルで処理するための強力な機能です。ラムダ式と組み合わせることで、データの変換、フィルタリング、集計などを簡潔に記述できます。

Streamの基本操作

StreamBasics.java 
import java.util.*;
import java.util.stream.*;

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5, 6);

// filter: 条件に合う要素だけを抽出
List<Integer> evens = numbers.stream()
    .filter(n -> n % 2 == 0)  // 偶数のみ
    .collect(Collectors.toList());
System.out.println(evens); // [2, 4, 6]

// map: 各要素を変換
List<Integer> squared = numbers.stream()
    .map(n -> n * n)  // 各要素を2乗
    .collect(Collectors.toList());
System.out.println(squared); // [1, 4, 9, 16, 25, 36]

よく使うStream操作

StreamOperations.java 
List<String> names = Arrays.asList("太郎", "花子", "次郎", "美咲");

// forEach: 各要素に処理を適用
names.stream().forEach(System.out::println);

// count: 要素数をカウント
long count = names.stream().count();
System.out.println("名前の数: " + count); // 4

// sorted: ソート
List<String> sorted = names.stream()
    .sorted()
    .collect(Collectors.toList());

// limit: 最初のN個を取得
List<String> first2 = names.stream()
    .limit(2)
    .collect(Collectors.toList());
System.out.println(first2); // [太郎, 花子]

14.6 実践的なStream活用例

Stream APIを使った実践的なデータ処理の例を見てみましょう。

データのフィルタリングと変換

StreamPractice.java 
class Product {
    String name;
    int price;

    Product(String name, int price) {
        this.name = name;
        this.price = price;
    }
}

List<Product> products = Arrays.asList(
    new Product("りんご", 150),
    new Product("バナナ", 100),
    new Product("オレンジ", 200)
);

// 価格が150円以上の商品名を取得
List<String> expensiveProducts = products.stream()
    .filter(p -> p.price >= 150)  // フィルタリング
    .map(p -> p.name)              // 変換
    .collect(Collectors.toList());

System.out.println(expensiveProducts); // [りんご, オレンジ]

集計処理

StreamAggregation.java 
List<Integer> scores = Arrays.asList(85, 92, 78, 95, 88);

// 合計
int sum = scores.stream().mapToInt(n -> n).sum();
System.out.println("合計点: " + sum);

// 平均
double average = scores.stream()
    .mapToInt(n -> n)
    .average()
    .orElse(0.0);
System.out.println("平均点: " + average);

// 最大値
int max = scores.stream()
    .max(Integer::compareTo)
    .orElse(0);
System.out.println("最高点: " + max);

実習 14-1: Stream APIで学生データを処理しよう

学生のリストから、Stream APIを使って様々な集計処理を実装してみましょう。

課題

学生クラス（名前と点数を持つ）のリストから、以下の処理を実装してください。

  1. 80点以上の学生の名前を取得
  2. 全学生の平均点を計算
  3. 点数の高い順に並べ替え

実装例

import java.util.*;
import java.util.stream.*;

class Student {
    String name;
    int score;

    Student(String name, int score) {
        this.name = name;
        this.score = score;
    }
}

List<Student> students = Arrays.asList(
    new Student("太郎", 85),
    new Student("花子", 92),
    new Student("次郎", 78),
    new Student("美咲", 95)
);

// 80点以上の学生名
List<String> topStudents = students.stream()
    .filter(s -> s.score >= 80)
    .map(s -> s.name)
    .collect(Collectors.toList());

// 平均点
double avg = students.stream()
    .mapToInt(s -> s.score)
    .average()
    .orElse(0.0);

// 点数順ソート
List<Student> sorted = students.stream()
    .sorted((a, b) -> b.score - a.score)
    .collect(Collectors.toList());

理解度確認クイズ

  1. Q1. ラムダ式を使える条件となるインターフェースの特徴は何ですか？

    回答を表示

    関数型インターフェース（@FunctionalInterface）であること。つまり、抽象メソッドを1つだけ持つインターフェースです。この条件を満たすインターフェースに対してのみ、ラムダ式で実装を提供できます。

  2. Q2. 次のラムダ式と等価なメソッド参照はどれですか？
    x -> System.out.println(x)

    回答を表示

    System.out::println
    メソッド参照は、ラムダ式が既存のメソッドをそのまま呼び出すだけの場合に使える簡潔な記法です。

  3. Q3. Stream APIの処理は「中間操作」と「終端操作」に分かれます。filterとcollectはそれぞれどちらですか？

    回答を表示

    filter: 中間操作（別のStreamを返す）
    collect: 終端操作（最終結果を返す）
    中間操作は複数チェーンでき、終端操作でストリームは消費されます。

  4. Q4. 標準関数型インターフェースのうち、引数を受け取って真偽値を返すものはどれですか？

    回答を表示

    Predicate<T>
    Predicateは主に条件判定に使われ、filter操作などでよく利用されます。例: n -> n > 10

この章のまとめ

  • ラムダ式は、メソッドを簡潔に記述するための構文で、匿名クラスを置き換えられる
  • 関数型インターフェースは抽象メソッドを1つだけ持ち、ラムダ式で実装できる
  • Predicate、Function、Consumer、Supplierなどの標準関数型インターフェースが用意されている
  • メソッド参照（::演算子）を使うと、既存メソッドをさらに簡潔に記述できる
  • Stream APIを使うと、コレクションのフィルタリング、変換、集計を関数型スタイルで記述できる
