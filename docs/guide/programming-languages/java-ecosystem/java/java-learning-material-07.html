第7章

メソッド - 処理をまとめて再利用

一連の処理を「メソッド」としてまとめ、再利用可能なコードを書く方法を学びます。メソッドを使うことで、プログラムの可読性、保守性、再利用性が大幅に向上します。

この章で学ぶこと

  • メソッドの概念と利点を理解する
  • メソッドの定義と呼び出しができる
  • 引数と戻り値の仕組みを理解する
  • メソッドのオーバーロードを理解する

7.1 メソッドとは何か

メソッド（method）は、特定の処理をひとまとめにした「処理の塊」です。プログラミングにおいて「関数」と呼ばれることもあります。
メソッドを使うことで、同じコードを何度も書く必要がなくなり、プログラムがシンプルで読みやすくなります。

メソッドを使う3つの利点

再利用性

一度作成したメソッドを何度でも呼び出して使える

可読性

処理に名前を付けることで、何をしているか分かりやすくなる

保守性

修正が必要な場合、メソッド内を変更するだけで済む

mainメソッドの役割
これまで使ってきた public static void main(String[] args) も実はメソッドです。 mainメソッドは「プログラムの入り口」として特別な役割を持ち、プログラム実行時に最初に呼び出されます。

メソッドの基本構造

メソッドの構文 
// メソッドの基本構造
修飾子 戻り値の型 メソッド名(引数の型 引数名, ...) {
    // 処理内容
    return 戻り値;  // 戻り値がある場合
}

7.2 メソッドの定義

最もシンプルなメソッド（引数なし、戻り値なし）

まずは、引数も戻り値もない最もシンプルなメソッドから見ていきましょう。

SimpleMethod.java 
public class SimpleMethod {
    public static void main(String[] args) {
        // メソッドの呼び出し
        greet();
        greet();  // 何度でも呼び出せる
    }

    // メソッドの定義
    public static void greet() {
        System.out.println("こんにちは！");
        System.out.println("Java学習へようこそ！");
    }
}

実行結果:

こんにちは！
Java学習へようこそ！
こんにちは！
Java学習へようこそ！

メソッドの構成要素

修飾子

public static

この段階では「おまじない」として覚えてOK。mainメソッドから呼び出すメソッドにはstaticを付けます。

戻り値の型

void

戻り値がない場合はvoidを指定します。

メソッド名

greet

処理内容を表す名前を付けます。先頭は小文字、複数の単語は2語目から大文字（キャメルケース）が慣例です。

メソッド名の命名規則

  • 動詞で始めるのが一般的（例: calculate, print, get）
  • 処理内容が分かる名前を付ける
  • 予約語（if、for、whileなど）は使えない

7.3 引数（パラメータ）

引数（ひきすう、argument / parameter）は、メソッドに渡す「入力データ」です。
引数を使うことで、同じメソッドでも異なるデータを処理できるようになります。

引数を持つメソッドの例

MethodWithParameter.java 
public class MethodWithParameter {
    public static void main(String[] args) {
        // 引数を指定してメソッドを呼び出す
        greetPerson("太郎");
        greetPerson("花子");
        greetPerson("次郎");
    }

    // 引数を受け取るメソッド
    public static void greetPerson(String name) {
        System.out.println("こんにちは、" + name + "さん！");
    }
}

実行結果:

こんにちは、太郎さん！
こんにちは、花子さん！
こんにちは、次郎さん！

仮引数と実引数

用語 説明 例
仮引数
（パラメータ）		 メソッド定義時に宣言する変数 String name
実引数
（引数）		 メソッド呼び出し時に渡す実際の値 "太郎"

複数の引数を持つメソッド

メソッドは複数の引数を受け取ることができます。引数はカンマ,で区切ります。

MultipleParameters.java 
public class MultipleParameters {
    public static void main(String[] args) {
        // 複数の引数を渡す
        printProfile("山田太郎", 25, "東京都");
        printProfile("佐藤花子", 30, "大阪府");
    }

    // 複数の引数を受け取るメソッド
    public static void printProfile(String name, int age, String address) {
        System.out.println("===== プロフィール =====");
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("住所: " + address);
        System.out.println();
    }
}

値渡しと参照渡し

Javaでは、プリミティブ型（int, doubleなど）は値渡し、参照型（String, 配列など）は参照渡しになります。 プリミティブ型の場合、メソッド内で引数の値を変更しても呼び出し元には影響しません。

7.4 戻り値（リターン値）

戻り値（もどりち、return value）は、メソッドが処理結果として呼び出し元に返す値です。
戻り値を使うことで、メソッドで計算した結果を受け取って利用できます。

戻り値を持つメソッドの例

MethodWithReturn.java 
public class MethodWithReturn {
    public static void main(String[] args) {
        // メソッドの戻り値を変数で受け取る
        int result1 = add(10, 20);
        System.out.println("10 + 20 = " + result1);

        // 直接表示することもできる
        System.out.println("5 + 15 = " + add(5, 15));

        // 戻り値を使った計算
        int result2 = add(3, 7) * 2;
        System.out.println("(3 + 7) * 2 = " + result2);
    }

    // int型の戻り値を持つメソッド
    public static int add(int a, int b) {
        int sum = a + b;
        return sum;  // 計算結果を返す
    }
}

実行結果:

10 + 20 = 30
5 + 15 = 20
(3 + 7) * 2 = 20

return文の使い方

ポイント1: 戻り値の型と一致させる

メソッドの戻り値の型と、return文で返す値の型は一致している必要があります。

// OK: int型を返すメソッドでint型を返す
public static int getNumber() {
    return 42;
}

// NG: int型を返すメソッドでString型を返す（コンパイルエラー）
public static int getNumber() {
    return "42";  // エラー！
}

ポイント2: return文の後ろは実行されない

return文が実行されると、メソッドの処理はそこで終了します。

public static int sample() {
    return 100;
    System.out.println("実行されない");  // 到達不能コード（エラー）
}

様々な戻り値の型

VariousReturnTypes.java 
public class VariousReturnTypes {
    public static void main(String[] args) {
        double avg = calculateAverage(80, 90, 70);
        System.out.println("平均点: " + avg);

        String greeting = getGreeting("太郎");
        System.out.println(greeting);

        boolean result = isAdult(20);
        System.out.println("成人ですか？ " + result);
    }

    // double型を返すメソッド
    public static double calculateAverage(int a, int b, int c) {
        return (a + b + c) / 3.0;
    }

    // String型を返すメソッド
    public static String getGreeting(String name) {
        return "こんにちは、" + name + "さん！";
    }

    // boolean型を返すメソッド
    public static boolean isAdult(int age) {
        return age >= 20;
    }
}

7.5 メソッドのオーバーロード

オーバーロード（overload、多重定義）とは、同じ名前で引数の型や個数が異なる複数のメソッドを定義することです。
これにより、状況に応じて適切なメソッドが自動的に選択されます。

オーバーロードの例

MethodOverload.java 
public class MethodOverload {
    public static void main(String[] args) {
        // 引数の個数や型に応じて適切なメソッドが呼ばれる
        System.out.println(add(10, 20));           // int版が呼ばれる
        System.out.println(add(3.5, 2.8));         // double版が呼ばれる
        System.out.println(add(1, 2, 3));          // 3つの引数版が呼ばれる
    }

    // 2つのint型を足すメソッド
    public static int add(int a, int b) {
        System.out.println("int版のaddが呼ばれました");
        return a + b;
    }

    // 2つのdouble型を足すメソッド
    public static double add(double a, double b) {
        System.out.println("double版のaddが呼ばれました");
        return a + b;
    }

    // 3つのint型を足すメソッド
    public static int add(int a, int b, int c) {
        System.out.println("3つの引数版のaddが呼ばれました");
        return a + b + c;
    }
}

実行結果:

int版のaddが呼ばれました
30
double版のaddが呼ばれました
6.3
3つの引数版のaddが呼ばれました
6

オーバーロードの判定条件

Javaは以下の条件でメソッドを区別します：

区別される（オーバーロード可能）

  • 引数の個数が異なる
  • 引数の型が異なる
  • 引数の順序が異なる

区別されない（エラー）

  • 戻り値の型だけが異なる
  • 引数名だけが異なる

実際のオーバーロード例
System.out.println()メソッドもオーバーロードされています。 int、double、String、booleanなど、様々な型の値を渡せるのはオーバーロードのおかげです。

つまずきやすいポイント

引数の順序と型の一致

メソッド呼び出し時、引数の順序と型は定義と完全に一致している必要があります。 

// メソッド定義
public static void printInfo(String name, int age) { ... }

// OK
printInfo("太郎", 20);

// NG: 順序が逆（コンパイルエラー）
printInfo(20, "太郎");

戻り値の受け取り忘れ

戻り値を持つメソッドを呼び出す際、戻り値を変数で受け取るか、直接使用する必要があります。 

// メソッド定義
public static int calculate(int a, int b) {
    return a + b;
}

// OK: 戻り値を変数で受け取る
int result = calculate(10, 20);

// OK: 戻り値を直接使用
System.out.println(calculate(10, 20));

// 無意味だが、エラーではない（戻り値を捨てている）
calculate(10, 20);

スコープの混乱

メソッド内で宣言された変数（ローカル変数）は、そのメソッド内でのみ有効です。 

public static void methodA() {
    int x = 10;  // methodA内でのみ有効
}

public static void methodB() {
    System.out.println(x);  // エラー！xは未定義
}

実習 7-1: 2つの数値の最大値を返すメソッド

2つの整数を受け取り、大きい方の値を返すメソッドmaxを作成しましょう。

手順

  1. 1 int型の引数を2つ受け取るメソッドmaxを定義します
  2. 2 if文を使って2つの値を比較し、大きい方をreturnします
  3. 3 mainメソッドから様々な値で呼び出して動作を確認します

実装例

Exercise7_1.java 
public class Exercise7_1 {
    public static void main(String[] args) {
        System.out.println("10と20の最大値: " + max(10, 20));
        System.out.println("50と30の最大値: " + max(50, 30));
        System.out.println("15と15の最大値: " + max(15, 15));
    }

    // 2つの数値の最大値を返すメソッド
    public static int max(int a, int b) {
        if (a > b) {
            return a;
        } else {
            return b;
        }
        // または三項演算子で: return (a > b) ? a : b;
    }
}

期待される結果

10と20の最大値: 20
50と30の最大値: 50
15と15の最大値: 15

実習 7-2: 配列の合計と平均を計算するメソッド

配列を受け取り、その合計値を返すメソッドと、平均値を返すメソッドを作成しましょう。

実装例

Exercise7_2.java 
public class Exercise7_2 {
    public static void main(String[] args) {
        int[] scores = {85, 90, 78, 92, 88};

        int total = sum(scores);
        double avg = average(scores);

        System.out.println("点数: " + java.util.Arrays.toString(scores));
        System.out.println("合計: " + total);
        System.out.println("平均: " + avg);
    }

    // 配列の合計を計算するメソッド
    public static int sum(int[] array) {
        int total = 0;
        for (int value : array) {
            total += value;
        }
        return total;
    }

    // 配列の平均を計算するメソッド
    public static double average(int[] array) {
        return (double) sum(array) / array.length;
        // sumメソッドを再利用している
    }
}

期待される結果

点数: [85, 90, 78, 92, 88]
合計: 433
平均: 86.6

理解度確認クイズ

  1. Q1. メソッドを使う3つの主な利点を説明してください。

    回答を表示

    1. 再利用性: 一度作成したメソッドを何度でも呼び出して使えるため、同じコードを繰り返し書く必要がありません。

    2. 可読性: 処理に名前を付けることで、プログラムが何をしているか分かりやすくなります。

    3. 保守性: 修正が必要な場合、メソッド内を変更するだけで、呼び出し側を変更する必要がありません。

  2. Q2. 次のコードの問題点を指摘してください。

    public static String getMessage() {
    return 100;
}
    回答を表示

    メソッドの戻り値の型がStringなのに、return文でint型の値（100）を返そうとしています。 戻り値の型と実際に返す値の型は一致している必要があります。正しくはreturn "100";のようにString型で返すか、 メソッドの戻り値の型をintに変更する必要があります。

  3. Q3. メソッドのオーバーロードとは何ですか？また、どのような条件で区別されますか？

    回答を表示

    オーバーロードとは、同じ名前で引数の型や個数が異なる複数のメソッドを定義することです（多重定義）。

    区別される条件:

    • 引数の個数が異なる
    • 引数の型が異なる
    • 引数の順序が異なる

    注意: 戻り値の型だけが異なる場合や、引数名だけが異なる場合は区別されません（エラーになります）。

  4. Q4. 次のメソッド呼び出しで、どのメソッドが実行されるか説明してください。

    public static void print(int x) {
    System.out.println("int: " + x);
}
public static void print(double x) {
    System.out.println("double: " + x);
}
public static void print(String x) {
    System.out.println("String: " + x);
}

// 呼び出し
print(10);
print(5.5);
print("Hello");
    回答を表示
    • print(10) → int型のメソッドが呼ばれ、「int: 10」と表示されます
    • print(5.5) → double型のメソッドが呼ばれ、「double: 5.5」と表示されます
    • print("Hello") → String型のメソッドが呼ばれ、「String: Hello」と表示されます

    Javaは引数の型を見て、最も適合するメソッドを自動的に選択します。
