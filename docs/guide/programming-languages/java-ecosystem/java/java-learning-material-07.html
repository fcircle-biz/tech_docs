第7章

メソッドの基礎

処理をまとめて再利用するためのメソッドについて学びます。メソッドの定義、呼び出し、引数と戻り値、オーバーロードなど、構造化プログラミングの基本を理解します。

この章で学ぶこと

  • メソッドとは何か、なぜメソッドを使うのかを理解する
  • メソッドの定義方法と呼び出し方を習得する
  • 引数（パラメータ）と戻り値（返り値）の役割を理解する
  • メソッドのオーバーロード（多重定義）を使いこなす

7.1 メソッドとは何か

メソッドとは、特定の処理をまとめて名前をつけたものです。「関数」とも呼ばれます。これまで使ってきたSystem.out.println()やMath.sqrt()もすべてメソッドです。

メソッドを使う主な目的は以下の通りです。

  • 再利用性: 同じ処理を何度も書かずに済む
  • 可読性: 処理を意味のある名前でまとめることで、コードが読みやすくなる
  • 保守性: 修正が必要なとき、1箇所を変更するだけで全体に反映される
  • モジュール化: 複雑なプログラムを小さな部品に分割できる

メソッドの利点

例えば、「消費税を計算する」処理が10箇所で必要な場合、メソッドを使えば1回の定義で済みます。税率が変わったときも、メソッド内の1箇所を修正するだけで全体に反映されます。

7.2 メソッドの定義と呼び出し

メソッドを定義するには、以下の構文を使用します。

メソッド定義の構文 
アクセス修飾子 戻り値の型 メソッド名(引数リスト) {
    // 処理内容
    return 戻り値; // 戻り値がある場合
}

具体例: 挨拶を表示するメソッド

MethodExample1.java 
public class MethodExample1 {
    public static void main(String[] args) {
        // メソッドの呼び出し
        greet();
        greet();
        greet();
    }

    // メソッドの定義
    public static void greet() {
        System.out.println("こんにちは！");
    }
}

このプログラムを実行すると、「こんにちは！」が3回表示されます。

メソッド定義の各部分の意味

  • public

    アクセス修飾子。このメソッドがどこからでも呼び出せることを示します。

  • static

    クラスメソッドであることを示します（オブジェクト指向の章で詳しく学びます）。

  • void

    戻り値がないことを示します。処理だけを実行するメソッドに使います。

  • greet

    メソッド名。処理内容が分かりやすい名前をつけましょう。

  • ()

    引数リスト。この例では引数がないため空です。

7.3 引数を持つメソッド

引数（パラメータ）は、メソッドに渡すデータです。引数を使うことで、同じメソッドでも異なる値で処理を実行できます。

MethodExample2.java 
public class MethodExample2 {
    public static void main(String[] args) {
        // 引数を渡してメソッドを呼び出す
        greetPerson("太郎");
        greetPerson("花子");
        greetPerson("次郎");
    }

    // 引数を受け取るメソッド
    public static void greetPerson(String name) {
        System.out.println("こんにちは、" + name + "さん！");
    }
}

実行結果: 

こんにちは、太郎さん！
こんにちは、花子さん！
こんにちは、次郎さん！

複数の引数を持つメソッド

複数の引数を受け取ることもできます。引数はカンマで区切ります。

MultipleParameters.java 
public class MultipleParameters {
    public static void main(String[] args) {
        // 複数の引数を渡す
        introduce("太郎", 25);
        introduce("花子", 30);
    }

    // 複数の引数を受け取るメソッド
    public static void introduce(String name, int age) {
        System.out.println(name + "さんは" + age + "歳です。");
    }
}

重要ポイント

引数の型と順序は、メソッド定義と呼び出しで完全に一致させる必要があります。introduce("太郎", 25)は正しいですが、introduce(25, "太郎")はエラーになります。

7.4 戻り値を持つメソッド

戻り値（返り値）は、メソッドが処理した結果を呼び出し元に返すデータです。return文を使って値を返します。

ReturnExample.java 
public class ReturnExample {
    public static void main(String[] args) {
        // 戻り値を受け取って変数に格納
        int result1 = add(10, 20);
        int result2 = add(5, 15);

        System.out.println("10 + 20 = " + result1);
        System.out.println("5 + 15 = " + result2);
    }

    // 戻り値を返すメソッド
    public static int add(int a, int b) {
        int sum = a + b;
        return sum; // 計算結果を返す
    }
}

戻り値の型

メソッド定義のvoidの部分が戻り値の型を示します。以下のような型を使えます。

  • int 整数を返す
  • double 小数を返す
  • String 文字列を返す
  • boolean 真偽値を返す
  • void 何も返さない

実用例: 消費税計算メソッド 
public class TaxCalculator {
    public static void main(String[] args) {
        double price = 1000;
        double taxIncluded = addTax(price);

        System.out.println("税抜価格: " + price + "円");
        System.out.println("税込価格: " + taxIncluded + "円");
    }

    // 消費税を加えた価格を返すメソッド
    public static double addTax(double price) {
        double taxRate = 0.1; // 消費税率10%
        return price * (1 + taxRate);
    }
}

7.5 メソッドのオーバーロード

オーバーロード（多重定義）とは、同じ名前のメソッドを複数定義することです。ただし、引数の型や個数が異なる必要があります。

OverloadExample.java 
public class OverloadExample {
    public static void main(String[] args) {
        // 同じ名前のメソッドを異なる引数で呼び出す
        System.out.println(add(10, 20));           // int版
        System.out.println(add(10.5, 20.3));       // double版
        System.out.println(add(10, 20, 30));       // 3つの引数版
    }

    // 整数2つを足すメソッド
    public static int add(int a, int b) {
        return a + b;
    }

    // 小数2つを足すメソッド
    public static double add(double a, double b) {
        return a + b;
    }

    // 整数3つを足すメソッド
    public static int add(int a, int b, int c) {
        return a + b + c;
    }
}

オーバーロードの条件

メソッドをオーバーロードするには、以下のいずれかが異なる必要があります。

  • 引数の個数: add(int a, int b) と add(int a, int b, int c)
  • 引数の型: add(int a, int b) と add(double a, double b)
  • 引数の順序: show(int a, String b) と show(String a, int b)

注意

戻り値の型だけが異なる場合はオーバーロードできません。int add(int a, int b)とdouble add(int a, int b)は同じシグネチャとみなされ、エラーになります。

7.6 メソッドのスコープと変数

スコープ（有効範囲）とは、変数が使える範囲のことです。メソッド内で宣言した変数は、そのメソッド内でのみ使用できます。

ScopeExample.java 
public class ScopeExample {
    public static void main(String[] args) {
        int x = 10; // mainメソッドのローカル変数

        printNumber();

        // 以下はエラー: yはprintNumber内でのみ有効
        // System.out.println(y);
    }

    public static void printNumber() {
        int y = 20; // printNumberメソッドのローカル変数
        System.out.println("yの値: " + y);

        // 以下はエラー: xはmainメソッド内でのみ有効
        // System.out.println(x);
    }
}

ローカル変数とは

メソッド内で宣言された変数を「ローカル変数」と呼びます。ローカル変数は、そのメソッドの実行が終わると破棄されます。別のメソッドで同じ名前の変数を使っても、互いに影響しません。

実習 7-1: 平均値計算メソッドを作ろう

複数の数値の平均値を計算するメソッドを作成し、メソッドの基本を実践してみましょう。

課題

3つの整数を受け取り、その平均値（double型）を返すメソッドaverageを作成してください。

手順

  1. 1 新しいJavaクラス「AverageCalculator.java」を作成します
  2. 2 3つのint型引数を受け取り、double型の平均値を返すメソッドを定義します
  3. 3 mainメソッドから複数のパターンで呼び出し、結果を表示します

実装例

public class AverageCalculator {
    public static void main(String[] args) {
        // メソッドを呼び出して平均値を計算
        double avg1 = average(80, 90, 100);
        double avg2 = average(60, 70, 80);

        System.out.println("平均1: " + avg1);
        System.out.println("平均2: " + avg2);
    }

    // 3つの数値の平均を計算するメソッド
    public static double average(int a, int b, int c) {
        double sum = a + b + c;
        return sum / 3.0; // 3.0で割って小数結果を得る
    }
}

期待される結果

平均1: 90.0
平均2: 70.0

理解度確認クイズ

  1. Q1. メソッドを使う主な利点を3つ挙げてください。

    回答を表示

    1. 再利用性: 同じ処理を何度も書かずに済む
    2. 可読性: 処理を意味のある名前でまとめてコードを読みやすくする
    3. 保守性: 修正が必要なとき、1箇所の変更で全体に反映される

  2. Q2. 以下のメソッド定義で、各部分の意味を説明してください。
    public static int add(int a, int b)

    回答を表示

    - public: どこからでも呼び出せる
    - static: クラスメソッド
    - int: 戻り値の型（整数を返す）
    - add: メソッド名
    - (int a, int b): 引数リスト（int型の引数を2つ受け取る）

  3. Q3. メソッドのオーバーロードとは何ですか？また、オーバーロードの条件を説明してください。

    回答を表示

    オーバーロードは同じ名前のメソッドを複数定義することです。条件として、以下のいずれかが異なる必要があります：
    1. 引数の個数
    2. 引数の型
    3. 引数の順序
    ※戻り値の型だけが異なる場合はオーバーロードできません。

  4. Q4. 戻り値がないメソッドにはどの型を指定しますか？

    回答を表示

    voidを指定します。voidは「何も返さない」ことを意味し、return文を省略できます（処理だけを実行するメソッドに使用）。

この章のまとめ

  • メソッドは処理をまとめて再利用可能にする仕組みで、再利用性・可読性・保守性を高める
  • メソッド定義には「アクセス修飾子 戻り値の型 メソッド名(引数リスト)」の形式を使う
  • 引数はメソッドに渡すデータで、戻り値はメソッドが処理した結果を返すデータ
  • オーバーロードを使うと、同じ名前で引数が異なるメソッドを複数定義できる
  • メソッド内で宣言した変数（ローカル変数）は、そのメソッド内でのみ有効
