メソッド - 処理をまとめて再利用

一連の処理を「メソッド」としてまとめ、再利用可能なコードを書く方法を学びます。

この章で学ぶこと

  • メソッドの概念と利点を理解する
  • メソッドの定義と呼び出しができる
  • 引数と戻り値の仕組みを理解する
  • メソッドのオーバーロードを理解する

7.1 メソッドとは何か

メソッドとは、特定の処理をひとまとめにして名前を付けたものです。料理のレシピに例えると、「オムレツの作り方」という一連の手順を「オムレツを作る」というメソッドとしてまとめるようなものです。

これまで学習してきた`main`メソッドも、実はメソッドの一種です。プログラムを実行すると、Javaは最初に`main`メソッドを探して実行します。

メソッドを使う利点

メソッドの3つの利点

  • 再利用性: 一度書いたコードを何度でも呼び出せる
  • 可読性: 処理に名前を付けることで、プログラムの意図が分かりやすくなる
  • 保守性: 修正が必要な時は、メソッド1箇所を直せば全体に反映される

メソッドの基本構造

メソッドの基本構造 
// メソッドの基本構造
修飾子 戻り値の型 メソッド名(引数リスト) {
    // メソッドの処理内容
    return 戻り値;  // 戻り値がある場合
}

7.2 メソッドの定義と呼び出し

最もシンプルな形のメソッドは、引数も戻り値もない「voidメソッド」です。voidは「何もない」という意味で、このメソッドは処理を実行するだけで、呼び出し元に値を返しません。

SimpleMethod.java 
public class SimpleMethod {
    public static void main(String[] args) {
        // メソッドを呼び出す
        sayHello();
        sayHello();  // 何度でも呼び出せる
        sayGoodbye();
    }

    // メソッドの定義: 挨拶を表示する
    public static void sayHello() {
        System.out.println("こんにちは!");
    }

    // メソッドの定義: さよならを表示する
    public static void sayGoodbye() {
        System.out.println("さようなら!");
    }
}

staticキーワードの意味

現段階では、メソッドに`static`を付けることで、`main`メソッドから直接呼び出せるようになると理解してください。staticの詳細は、第8章「クラスとオブジェクト」で学びます。

メソッド名の命名規則

メソッド名のルール

  • 最初の文字は小文字で始める（キャメルケース）
  • 複数の単語を組み合わせる場合、2単語目以降は大文字で始める（例: calculateSum, getUserName）
  • 動詞で始めることが推奨される（例: get, set, calculate, show）
  • 処理内容が分かりやすい名前を付ける

7.3 引数（パラメータ）

引数とは、メソッドに渡すデータのことです。引数を使うことで、同じメソッドでも異なるデータを処理できるようになります。

graph LR A[呼び出し側] -->|引数を渡す| B[メソッド] B -->|処理を実行| C[結果] style A fill:#fed7aa,stroke:#f97316 style B fill:#bfdbfe,stroke:#3b82f6 style C fill:#a7f3d0,stroke:#10b981
MethodWithParameter.java 
public class MethodWithParameter {
    public static void main(String[] args) {
        // 引数を渡してメソッドを呼び出す
        greet("太郎");
        greet("花子");

        // 複数の引数を持つメソッド
        introduce("鈴木", 25);
    }

    // 1つの引数を持つメソッド
    public static void greet(String name) {
        System.out.println("こんにちは、" + name + "さん!");
    }

    // 複数の引数を持つメソッド
    public static void introduce(String name, int age) {
        System.out.println("私は" + name + "です。");
        System.out.println(age + "歳です。");
    }
}

仮引数と実引数

用語の整理

  • 仮引数（パラメータ）: メソッド定義時の括弧内の変数（例: `String name`）
  • 実引数（アーギュメント）: メソッド呼び出し時に渡す具体的な値（例: `"太郎"`）

値渡しと参照渡し

Javaでは、基本データ型（int、doubleなど）は「値渡し」、参照型（配列、Stringなど）は「参照渡し」で引数が渡されます。

ValueAndReference.java 
public class ValueAndReference {
    public static void main(String[] args) {
        // 値渡しの例（基本データ型）
        int number = 10;
        changeValue(number);
        System.out.println("値渡し後: " + number);  // 10のまま変わらない

        // 参照渡しの例（配列）
        int[] numbers = {10, 20, 30};
        changeArray(numbers);
        System.out.println("参照渡し後: " + numbers[0]);  // 100に変わる
    }

    // 基本データ型は値のコピーが渡される
    public static void changeValue(int n) {
        n = 100;  // コピーを変更しても元の変数には影響しない
    }

    // 配列は参照が渡される
    public static void changeArray(int[] arr) {
        arr[0] = 100;  // 同じ配列を参照しているため元の配列が変更される
    }
}

7.4 戻り値（リターン値）

戻り値とは、メソッドが処理結果を呼び出し元に返す値のことです。メソッドが計算した結果や、判定結果などを返すために使用します。

graph LR A[呼び出し側] -->|引数: 10, 20| B[addメソッド] B -->|戻り値: 30| A style A fill:#fed7aa,stroke:#f97316 style B fill:#bfdbfe,stroke:#3b82f6
ReturnValue.java 
public class ReturnValue {
    public static void main(String[] args) {
        // 戻り値を受け取って使用
        int sum = add(10, 20);
        System.out.println("合計: " + sum);

        // 戻り値を直接使用
        System.out.println("最大値: " + max(15, 25));

        // 戻り値を使った計算の連鎖
        int result = multiply(add(5, 3), 2);
        System.out.println("結果: " + result);  // (5+3)*2 = 16
    }

    // 2つの整数を足し算して結果を返す
    public static int add(int a, int b) {
        int sum = a + b;
        return sum;  // 計算結果を返す
    }

    // 2つの整数の最大値を返す
    public static int max(int a, int b) {
        if (a > b) {
            return a;
        } else {
            return b;
        }
    }

    // 2つの整数を掛け算して結果を返す
    public static int multiply(int a, int b) {
        return a * b;  // 式を直接返すこともできる
    }
}

returnの重要なルール

  • • return文を実行すると、その時点でメソッドが終了する
  • • 戻り値の型は、メソッド宣言の戻り値型と一致している必要がある
  • • voidメソッド以外は、必ずreturn文で値を返す必要がある

戻り値の型の種類

様々な戻り値の型 
public class ReturnTypes {
    // int型を返す
    public static int getAge() {
        return 25;
    }

    // double型を返す
    public static double calculateAverage(int[] numbers) {
        int sum = 0;
        for (int n : numbers) {
            sum += n;
        }
        return (double) sum / numbers.length;
    }

    // boolean型を返す（真偽値を返す）
    public static boolean isAdult(int age) {
        return age >= 20;
    }

    // String型を返す
    public static String getMessage() {
        return "こんにちは";
    }

    // 配列を返す
    public static int[] createArray(int size) {
        return new int[size];
    }
}

7.5 メソッドのオーバーロード

オーバーロードとは、同じ名前のメソッドを複数定義することです。ただし、引数の数や型が異なっている必要があります。これにより、同じ処理を異なるデータ型で実行できるようになります。

MethodOverload.java 
public class MethodOverload {
    public static void main(String[] args) {
        // 同じ名前のメソッドを異なる引数で呼び出す
        System.out.println(add(10, 20));           // int版を呼び出す
        System.out.println(add(1.5, 2.5));         // double版を呼び出す
        System.out.println(add(5, 10, 15));        // 3つの引数版を呼び出す

        greet("太郎");                              // 1引数版
        greet("花子", "おはようございます");        // 2引数版
    }

    // int型の2つの数値を足し算
    public static int add(int a, int b) {
        System.out.println("int版が呼ばれました");
        return a + b;
    }

    // double型の2つの数値を足し算
    public static double add(double a, double b) {
        System.out.println("double版が呼ばれました");
        return a + b;
    }

    // 3つの整数を足し算
    public static int add(int a, int b, int c) {
        System.out.println("3引数版が呼ばれました");
        return a + b + c;
    }

    // 1つの引数で挨拶
    public static void greet(String name) {
        System.out.println("こんにちは、" + name + "さん!");
    }

    // 2つの引数でカスタマイズした挨拶
    public static void greet(String name, String message) {
        System.out.println(message + "、" + name + "さん!");
    }
}

オーバーロードの条件

  • • メソッド名は同じでなければならない
  • • 引数の数、型、順序のいずれかが異なっている必要がある
  • • 戻り値の型だけが異なる場合はオーバーロードにならない（コンパイルエラー）

Javaの標準メソッドもオーバーロード

実は、これまで使ってきた`System.out.println()`もオーバーロードされています。int、double、String、booleanなど、様々な型のデータを表示できるのは、それぞれの型に対応したprintlnメソッドが用意されているからです。

printlnのオーバーロード例 
// これらは全て異なるprintlnメソッドが呼ばれている
System.out.println(100);          // int版
System.out.println(3.14);         // double版
System.out.println("Hello");      // String版
System.out.println(true);         // boolean版

実習 7-1: 配列の合計と平均を計算するメソッド

配列を受け取り、その合計値と平均値を計算するメソッドを作成しましょう。

手順

  1. 1 配列を受け取り、合計値を返す`calculateSum`メソッドを作成する
  2. 2 配列を受け取り、平均値を返す`calculateAverage`メソッドを作成する
  3. 3 mainメソッドから呼び出して、結果を表示する

実装例

ArrayCalculation.java 
public class ArrayCalculation {
    public static void main(String[] args) {
        int[] scores = {85, 90, 78, 92, 88};

        int sum = calculateSum(scores);
        double average = calculateAverage(scores);

        System.out.println("合計: " + sum);
        System.out.println("平均: " + average);
    }

    // 配列の合計を計算
    public static int calculateSum(int[] numbers) {
        int sum = 0;
        for (int n : numbers) {
            sum += n;
        }
        return sum;
    }

    // 配列の平均を計算
    public static double calculateAverage(int[] numbers) {
        int sum = calculateSum(numbers);  // 既存のメソッドを再利用
        return (double) sum / numbers.length;
    }
}

期待される結果

合計: 433
平均: 86.6

実習 7-2: オーバーロードを使った挨拶メソッド

引数の数によって異なる挨拶をする、オーバーロードされたメソッドを作成しましょう。

実装例

GreetingOverload.java 
public class GreetingOverload {
    public static void main(String[] args) {
        greet();                           // 引数なし版
        greet("太郎");                     // 名前のみ版
        greet("花子", 25);                 // 名前と年齢版
    }

    // 引数なしバージョン
    public static void greet() {
        System.out.println("こんにちは!");
    }

    // 名前のみバージョン
    public static void greet(String name) {
        System.out.println("こんにちは、" + name + "さん!");
    }

    // 名前と年齢バージョン
    public static void greet(String name, int age) {
        System.out.println("こんにちは、" + name + "さん!");
        System.out.println(age + "歳ですね。");
    }
}

理解度確認クイズ

  1. Q1. メソッドを使う3つの利点を説明してください。

    回答を表示

    再利用性: 一度書いたコードを何度でも呼び出せる
    可読性: 処理に名前を付けることで、プログラムの意図が分かりやすくなる
    保守性: 修正が必要な時は、メソッド1箇所を直せば全体に反映される

  2. Q2. 次のメソッド定義で間違っている点を指摘してください。

    public static int calculate() {
    int result = 10 + 20;
    // return文がない
}
    回答を表示

    戻り値の型が`int`と宣言されているのに、return文がありません。戻り値がある場合は、必ず`return 結果;`の形で値を返す必要があります。正しくは、最後に`return result;`を追加します。

  3. Q3. メソッドのオーバーロードとは何か、簡単に説明してください。

    回答を表示

    オーバーロードとは、同じ名前のメソッドを、引数の数や型を変えて複数定義することです。これにより、同じ処理を異なるデータ型で実行できるようになります。ただし、引数の数、型、順序のいずれかが異なっている必要があります。

  4. Q4. 次のコードで何が出力されますか？

    public static void main(String[] args) {
    int x = 10;
    changeValue(x);
    System.out.println(x);
}

public static void changeValue(int n) {
    n = 100;
}
    回答を表示

    出力は`10`です。Javaでは基本データ型は値渡しなので、メソッド内で引数の値を変更しても、元の変数には影響しません。changeValueメソッドで変更されるのは、xのコピーであり、元のxは変わりません。

本章のまとめ

  • メソッドは処理をまとめて名前を付けたもので、再利用性・可読性・保守性を向上させる
  • 引数を使うことで、メソッドに異なるデータを渡して処理できる
  • 戻り値を使うことで、メソッドの処理結果を呼び出し元に返すことができる
  • オーバーロードを使うと、同じ名前で引数の異なるメソッドを複数定義できる
  • 基本データ型は値渡し、参照型（配列など）は参照渡しで引数が渡される
