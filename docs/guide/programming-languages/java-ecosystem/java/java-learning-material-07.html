設計図からモノを作る - クラスとオブジェクトの基本

Javaの核心である「オブジェクト指向」の入り口として、クラス（設計図）とオブジェクト（実体）の概念を学びます。たい焼きの型と実際のたい焼きのたとえで理解を深めましょう。

この章で学ぶこと

  • 「クラス」と「オブジェクト」の違いと関係を理解する
  • 設計図からインスタンスを生成する仕組みを学ぶ
  • 実際にクラスを定義し、オブジェクトを作成できるようになる

7.1 オブジェクト指向って何だろう？

これまで変数、条件分岐、ループ、配列、メソッドなど、プログラミングの基本的な要素を学んできました。 ここからは、Javaの最大の特徴であるオブジェクト指向プログラミング（OOP: Object-Oriented Programming）の世界に入ります。

オブジェクト指向とは？

現実世界の「モノ」（オブジェクト）を、データ（状態）と機能（動作）をひとまとめにして表現するプログラミングの考え方です。 たとえば「車」なら、データは「色、速度、燃料」、機能は「走る、止まる、曲がる」といった具合です。

なぜオブジェクト指向が必要なの？

プログラムが大きく複雑になると、すべてを一つの流れで書くのは大変です。 オブジェクト指向では、プログラムを「モノ」の集まりとして考えることで、以下のメリットがあります。

分かりやすい

現実世界の概念とプログラムの構造が近いため、理解しやすく設計しやすい。

再利用できる

一度作った「モノ」を他のプログラムでも使い回すことができる。

保守しやすい

変更や修正が特定の「モノ」に限定されるため、影響範囲が明確になる。

7.2 クラスとオブジェクトの違い

オブジェクト指向プログラミングの核心となる概念がクラスとオブジェクトです。 この2つの関係を理解することが、Javaをマスターする第一歩です。

たとえ話: たい焼きの型とたい焼き

クラス = たい焼きの型

たい焼きの形、大きさ、尾びれや背びれの位置などが決められた設計図です。 型そのものは食べられません。

  • ・一つの型から何個でもたい焼きを作れる
  • ・同じ型から作ったたい焼きは同じ形

オブジェクト = 実際のたい焼き

型を使って焼いた実物です。 これは実際に手に取って食べることができます。

  • ・中身はあんこ、カスタード、チョコなど
  • ・温度や焼き加減は個別に違う

ポイント

  • ・クラス：データと機能の「設計図」。抽象的な定義。
  • ・オブジェクト：クラスから作られた「実体」。具体的な値を持つ。
  • ・オブジェクトを作ることを「インスタンス化」と呼ぶ。
  • ・一つのクラスから複数のオブジェクト（インスタンス）を作れる。

7.3 クラスの基本構造

それでは、実際にクラスを定義してみましょう。 まずは最もシンプルな形から始めます。

最もシンプルなクラス 
// クラスの定義
public class Dog {
    // ここにクラスの中身を書く
}

public class Dog

「Dog」という名前のクラスを宣言しています。 publicは他のクラスからもこのクラスを使えるという意味です。
※ クラス名は最初の文字を大文字にするのがルール（Dogとか、Carとか、Personとか）。

{ }（波カッコ）

クラスの「中身」を書く場所を示します。 この中にデータ（変数）や機能（メソッド）を定義していきます。

データを持つクラス

クラスには「データ」を持たせることができます。 これをフィールド（field）または属性と呼びます。

Dog.java 
public class Dog {
    // フィールド（データ）
    String name;    // 犬の名前
    int age;        // 犬の年齢
    String breed;   // 犬の品種
}

フィールドとは？

クラスの中で宣言された変数のことです。 このクラスから作られるすべてのオブジェクトが、それぞれ自分専用のフィールドを持ちます。 たとえば「ポチ」という犬と「タロ」という犬は、それぞれ別のnameフィールドを持ちます。

7.4 オブジェクトを作ってみよう

クラスという設計図ができたら、次はそれを使って実際のオブジェクト（インスタンス）を作ります。 オブジェクトを作るにはnew演算子を使います。

Main.java 
public class Main {
    public static void main(String[] args) {
        // Dogクラスからオブジェクトを作成
        Dog myDog = new Dog();

        System.out.println("犬のオブジェクトを作成しました！");
    }
}

Dog myDog

Dog型の変数myDogを宣言しています。 intやStringと同じように、自分で作ったクラスも「型」として使えます。

new Dog()

new演算子でDogクラスから新しいオブジェクト（インスタンス）を作成しています。 「たい焼きの型で新しいたい焼きを焼く」のと同じイメージです。

7.5 オブジェクトのデータを操作する

作ったオブジェクトのフィールドに値を設定したり、取り出したりするにはドット演算子（.）を使います。

Main.java（フィールドアクセス） 
public class Main {
    public static void main(String[] args) {
        // Dogオブジェクトを作成
        Dog myDog = new Dog();

        // フィールドに値を設定
        myDog.name = "ポチ";
        myDog.age = 3;
        myDog.breed = "柴犬";

        // フィールドの値を取り出して表示
        System.out.println("名前: " + myDog.name);
        System.out.println("年齢: " + myDog.age + "歳");
        System.out.println("品種: " + myDog.breed);
    }
}

実行結果

名前: ポチ
年齢: 3歳
品種: 柴犬

ドット演算子の使い方

オブジェクト名.フィールド名 の形式で、 オブジェクトが持つデータにアクセスできます。 代入も参照も同じようにドット演算子を使います。

実習 7-1: 複数の犬オブジェクトを作ろう

同じDogクラスから複数の異なる犬オブジェクトを作成してみましょう。

手順

  1. 1 Dogクラスを作成（name、age、breedフィールドを持つ）
  2. 2 Mainクラスで3匹の異なる犬オブジェクトを作成
  3. 3 それぞれに異なる名前、年齢、品種を設定
  4. 4 3匹の情報を画面に表示
Main.java（解答例） 
public class Main {
    public static void main(String[] args) {
        // 1匹目の犬
        Dog dog1 = new Dog();
        dog1.name = "ポチ";
        dog1.age = 3;
        dog1.breed = "柴犬";

        // 2匹目の犬
        Dog dog2 = new Dog();
        dog2.name = "ハチ";
        dog2.age = 5;
        dog2.breed = "秋田犬";

        // 3匹目の犬
        Dog dog3 = new Dog();
        dog3.name = "マロン";
        dog3.age = 2;
        dog3.breed = "トイプードル";

        // 情報を表示
        System.out.println("=== 犬の情報 ===");
        System.out.println("1匹目: " + dog1.name +
                           " (" + dog1.breed + ", " + dog1.age + "歳)");
        System.out.println("2匹目: " + dog2.name +
                           " (" + dog2.breed + ", " + dog2.age + "歳)");
        System.out.println("3匹目: " + dog3.name +
                           " (" + dog3.breed + ", " + dog3.age + "歳)");

7.6 クラスにメソッドを追加する

クラスには「データ」だけでなく「動作」も持たせることができます。 クラスの中に定義するメソッドを使って、オブジェクトに機能を持たせましょう。

Dog.java（メソッド付き） 
public class Dog {
    // フィールド
    String name;
    int age;
    String breed;

    // メソッド：鳴く
    void bark() {
        System.out.println(name + "「ワンワン！」");
    }

    // メソッド：自己紹介
    void introduce() {
        System.out.println("名前は" + name + "です。");
        System.out.println(age + "歳の" + breed + "です。");
    }
}
Main.java（メソッド呼び出し） 
public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.name = "ポチ";
        myDog.age = 3;
        myDog.breed = "柴犬";

        // メソッドを呼び出す
        myDog.bark();        // 鳴く
        myDog.introduce();   // 自己紹介
    }
}

実行結果

ポチ「ワンワン！」
名前はポチです。
3歳の柴犬です。

メソッドの中でフィールドを使う

クラスのメソッドの中では、同じクラスのフィールドを直接使うことができます。 nameやageをそのまま参照できるのがポイントです。

7.7 実践例：書籍管理クラス

もう少し実用的な例として、書籍を管理するクラスを作ってみましょう。

Book.java 
public class Book {
    // フィールド
    String title;      // タイトル
    String author;     // 著者
    int pages;         // ページ数
    boolean isRented;  // 貸出中かどうか

    // メソッド：貸し出し
    void rent() {
        if (isRented) {
            System.out.println(title + "は貸出中です。");
        } else {
            isRented = true;
            System.out.println(title + "を貸し出しました。");
        }
    }

    // メソッド：返却
    void returnBook() {
        if (isRented) {
            isRented = false;
            System.out.println(title + "を返却しました。");
        } else {
            System.out.println(title + "は貸し出されていません。");
        }
    }

    // メソッド：書籍情報表示
    void showInfo() {
        System.out.println("=== 書籍情報 ===");
        System.out.println("タイトル: " + title);
        System.out.println("著者: " + author);
        System.out.println("ページ数: " + pages);
        System.out.println("状態: " + (isRented ? "貸出中" : "貸出可能"));
    }
}
Main.java 
public class Main {
    public static void main(String[] args) {
        // 書籍オブジェクトを作成
        Book book1 = new Book();
        book1.title = "Javaプログラミング入門";
        book1.author = "山田太郎";
        book1.pages = 350;
        book1.isRented = false;

        // 書籍情報を表示
        book1.showInfo();

        // 貸し出し
        book1.rent();

        // 再度貸し出しを試みる
        book1.rent();

        // 返却
        book1.returnBook();

        // 再度情報表示
        book1.showInfo();
    }
}

実行結果

著者: 山田太郎
ページ数: 350
状態: 貸出可能

Javaプログラミング入門は貸出中です。
Javaプログラミング入門を返却しました。

理解度確認クイズ

  1. Q1. クラスとオブジェクトの違いを説明してください。

    回答を表示

    クラスは「設計図」で、データと機能を定義した抽象的なものです。 オブジェクト（インスタンス）は、クラスから作られた「実体」で、具体的な値を持ちます。 たい焼きの例では、クラスは「型」、オブジェクトは「実際のたい焼き」に相当します。

  2. Q2. オブジェクトを作成するにはどの演算子を使いますか？

    回答を表示

    new演算子を使います。 例: Dog myDog = new Dog(); この文で、Dogクラスから新しいオブジェクトを作成し、変数myDogに代入しています。

  3. Q3. フィールドとは何ですか？

    回答を表示

    クラスの中で宣言された変数のことです。オブジェクトが持つデータ（状態）を表します。 例えば、Dogクラスのname、ageなどがフィールドです。 各オブジェクトは自分専用のフィールドを持ちます。

  4. Q4. 一つのクラスから複数のオブジェクトを作ることはできますか？

    回答を表示

    できます。一つのクラス（設計図）から、いくつでもオブジェクト（実体）を作ることができます。 それぞれのオブジェクトは独立していて、異なる値を持つことができます。 たい焼きの型一つから、何個でもたい焼きを作れるのと同じです。

7.8 この章のまとめ

  • オブジェクト指向は、現実世界の「モノ」をデータと機能のまとまりとして表現する考え方。
  • クラスは設計図、オブジェクト（インスタンス）は実体。
  • フィールドはクラスが持つデータ（状態）を表す変数。
  • オブジェクトはnew演算子で作成し、ドット演算子でフィールドやメソッドにアクセスする。
  • クラスにメソッドを追加することで、オブジェクトに「動作」を持たせることができる。
  • 一つのクラスから複数のオブジェクトを作成でき、それぞれが独立した値を持つ。
