抽象クラスとインターフェース

抽象クラスとインターフェースを使った高度な抽象化について学びます。契約による設計や、実装の柔軟性を高める技法を理解します。

この章で学ぶこと

  • 抽象クラスの概念と実装方法
  • インターフェースによる契約プログラミング
  • 抽象クラスとインターフェースの使い分け
  • 複数インターフェースの実装とデフォルトメソッド

11.1 抽象クラスとは

抽象クラス（abstract class）は、インスタンスを直接作成できないクラスです。抽象クラスは共通の動作を定義しつつ、具体的な実装を子クラスに委ねることができます。これにより、設計レベルでの「枠組み」を定義し、詳細な実装は継承先で決定するというアプローチが可能になります。

抽象クラスの特徴

  • 抽象メソッド: 実装を持たないメソッドを定義できる（子クラスで必ず実装が必要）
  • 具体メソッド: 通常のメソッドも定義でき、共通処理を実装できる
  • フィールド: インスタンス変数を持つことができる
  • インスタンス化不可: 抽象クラス自体のインスタンスは作成できない

抽象クラスの定義例

Shape.java（抽象クラス） 
// 抽象クラス：図形の基本クラス
public abstract class Shape {
    protected double x; // 位置（X座標）
    protected double y; // 位置（Y座標）

    // コンストラクタ
    public Shape(double x, double y) {
        this.x = x;
        this.y = y;
    }

    // 具体メソッド（共通処理）
    public void moveTo(double newX, double newY) {
        this.x = newX;
        this.y = newY;
    }

    // 抽象メソッド（子クラスで実装必須）
    public abstract void draw();
    public abstract double getArea();
}

抽象クラスを継承する具体クラス

Circle.java 
// 抽象クラスを継承する具体クラス
public class Circle extends Shape {
    private double radius; // 半径

    public Circle(double x, double y, double radius) {
        super(x, y); // 親クラスのコンストラクタ呼び出し
        this.radius = radius;
    }

    // 抽象メソッドを実装
    @Override
    public void draw() {
        System.out.println("円を描画: 中心(" + x + ", " + y + "), 半径=" + radius);
    }

    @Override
    public double getArea() {
        return Math.PI * radius * radius; // 円の面積
    }
}

重要ポイント

抽象クラスを継承した具体クラスは、すべての抽象メソッドを実装する必要があります。実装しない場合、そのクラス自体も抽象クラスとして宣言しなければなりません。

11.2 インターフェースとは

インターフェース（interface）は、クラスが実装すべきメソッドの「契約」を定義する仕組みです。インターフェースは実装を持たず、「何をすべきか」だけを定義します。Javaでは複数のインターフェースを実装できるため、多重継承に近い柔軟な設計が可能になります。

インターフェースの特徴

  • すべてのメソッドは暗黙的にpublic abstract（実装を持たない）
  • すべてのフィールドは暗黙的にpublic static final（定数）
  • クラスは複数のインターフェースを実装できる（implements）
  • Java 8以降、defaultメソッドとstaticメソッドを定義可能
classDiagram class Drawable { <> +draw() void } class Movable { <> +moveTo(x, y) void } class GameObject { +name String +draw() void +moveTo(x, y) void } Drawable <|.. GameObject Movable <|.. GameObject

インターフェースの定義例

Drawable.java（インターフェース） 
// インターフェース：描画可能なオブジェクトの契約
public interface Drawable {
    // 抽象メソッド（暗黙的にpublic abstract）
    void draw();
    void setColor(String color);
}
Movable.java（インターフェース） 
// インターフェース：移動可能なオブジェクトの契約
public interface Movable {
    void moveTo(double x, double y);
    double getSpeed();
}

複数インターフェースの実装

Player.java 
// 複数のインターフェースを実装するクラス
public class Player implements Drawable, Movable {
    private double x, y;
    private String color;
    private double speed;

    public Player(double x, double y, double speed) {
        this.x = x;
        this.y = y;
        this.speed = speed;
        this.color = "blue";
    }

    // Drawableインターフェースの実装
    @Override
    public void draw() {
        System.out.println("プレイヤーを描画: (" + x + ", " + y + "), 色=" + color);
    }

    @Override
    public void setColor(String color) {
        this.color = color;
    }

    // Movableインターフェースの実装
    @Override
    public void moveTo(double newX, double newY) {
        this.x = newX;
        this.y = newY;
    }

    @Override
    public double getSpeed() {
        return speed;
    }
}

11.3 抽象クラスとインターフェースの使い分け

抽象クラスとインターフェースは似ていますが、それぞれ異なる用途があります。適切に使い分けることで、柔軟で保守性の高い設計が可能になります。

比較表

項目 抽象クラス インターフェース
継承/実装 extends（単一継承） implements（複数実装可）
フィールド インスタンス変数を持てる 定数のみ（public static final）
メソッド 抽象・具体メソッド両方可 抽象・default・staticメソッド可
コンストラクタ 持てる 持てない
用途 共通の状態・動作を持つ継承階層 能力・契約の定義

使い分けの指針

抽象クラスを使うべき場合

  • 関連するクラス間で共通の状態（フィールド）を持つ
  • 共通の具体的な実装を提供したい
  • 「is-a」関係（〜は〜の一種）が明確

インターフェースを使うべき場合

  • 無関係なクラスに共通の能力を持たせたい
  • 複数の契約を同時に実装したい
  • 「can-do」関係（〜できる）を表現したい

設計のヒント

「動物」のような概念は抽象クラス（状態を持つ）、「飛べる」「泳げる」のような能力はインターフェースで表現すると、自然で拡張しやすい設計になります。

11.4 デフォルトメソッドと静的メソッド

Java 8以降、インターフェースにデフォルトメソッドと静的メソッドを定義できるようになりました。これにより、既存のインターフェースに新しいメソッドを追加しても、実装クラスを変更せずに済む場合があります。

デフォルトメソッド

デフォルトメソッドは、インターフェース内に実装を持つメソッドです。実装クラスはこのメソッドをオーバーライドしなくても使用できます。

Logger.java（デフォルトメソッド付き） 
public interface Logger {
    // 抽象メソッド
    void log(String message);

    // デフォルトメソッド（実装を持つ）
    default void logWithTimestamp(String message) {
        String timestamp = java.time.LocalDateTime.now().toString();
        log("[" + timestamp + "] " + message);
    }

    // デフォルトメソッド
    default void logError(String error) {
        log("ERROR: " + error);
    }
}

静的メソッド

インターフェースに静的メソッドを定義できます。これはユーティリティメソッドとして利用されます。

MathUtils.java（静的メソッド付き） 
public interface MathUtils {
    // 静的メソッド
    static int max(int a, int b) {
        return (a > b) ? a : b;
    }

    static double average(double... numbers) {
        double sum = 0;
        for (double n : numbers) {
            sum += n;
        }
        return sum / numbers.length;
    }
}

実装例

ConsoleLogger.java 
public class ConsoleLogger implements Logger {
    // 必須の抽象メソッドのみ実装
    @Override
    public void log(String message) {
        System.out.println(message);
    }

    // デフォルトメソッドは自動的に利用可能
    // 必要に応じてオーバーライドも可能
}

// 使用例
ConsoleLogger logger = new ConsoleLogger();
logger.log("通常ログ");
logger.logWithTimestamp("タイムスタンプ付きログ"); // デフォルトメソッド
logger.logError("エラーメッセージ"); // デフォルトメソッド

実習 11-1: 支払いシステムの設計

抽象クラスとインターフェースを組み合わせて、柔軟な支払いシステムを設計しましょう。

課題

以下の要件を満たす支払いシステムを実装してください。

  • Payableインターフェース（支払い可能な契約）を定義
  • Paymentクラス（抽象クラス）を定義し、共通処理を実装
  • CreditCardPaymentとBankTransferPaymentを具体クラスとして実装

実装例

// インターフェース
interface Payable {
    boolean pay(double amount);
    String getPaymentType();
}

// 抽象クラス
abstract class Payment implements Payable {
    protected String accountInfo;

    public Payment(String accountInfo) {
        this.accountInfo = accountInfo;
    }

    // 共通処理
    protected void logPayment(double amount) {
        System.out.println("支払い記録: " + amount + "円");
    }
}

// 具体クラス1
class CreditCardPayment extends Payment {
    public CreditCardPayment(String cardNumber) {
        super(cardNumber);
    }

    @Override
    public boolean pay(double amount) {
        System.out.println("クレジットカード支払い: " + amount + "円");
        logPayment(amount);
        return true;
    }

    @Override
    public String getPaymentType() {
        return "クレジットカード";
    }
}

理解度確認クイズ

  1. Q1. 抽象クラスとインターフェースの違いとして正しいものはどれですか？

    回答を表示

    抽象クラスは単一継承のみ可能で、インスタンス変数を持つことができます。一方、インターフェースは複数実装可能で、定数のみを持つことができます。抽象クラスはコンストラクタを持てますが、インターフェースは持てません。

  2. Q2. インターフェースのデフォルトメソッドとは何ですか？

    回答を表示

    デフォルトメソッドは、Java 8以降でインターフェース内に実装を持つメソッドです。実装クラスはこのメソッドをオーバーライドしなくても使用でき、既存のインターフェースに新機能を追加する際に便利です。

  3. Q3. 抽象クラスに抽象メソッドがない場合、インスタンス化できますか？

    回答を表示

    いいえ、できません。抽象クラスは抽象メソッドの有無に関わらず、abstractキーワードで宣言されている限りインスタンス化できません。抽象クラスは継承して使うことを前提としています。

  4. Q4. 一つのクラスが複数のインターフェースを実装する利点は何ですか？

    回答を表示

    複数のインターフェースを実装することで、一つのクラスに異なる複数の「能力」や「契約」を持たせることができます。これにより、柔軟で拡張性の高い設計が可能になり、多重継承に近い機能を実現できます。

この章のまとめ

  • 抽象クラスはインスタンス化できないクラスで、抽象メソッドと具体メソッドを持つことができる
  • インターフェースは実装すべきメソッドの契約を定義し、複数実装が可能
  • 抽象クラスは「is-a」関係、インターフェースは「can-do」関係を表現するのに適している
  • デフォルトメソッドにより、インターフェースに実装を持つメソッドを定義できる
  • 抽象クラスとインターフェースを組み合わせることで、柔軟で拡張性の高い設計が可能
