第11章

コレクションフレームワーク

Javaが提供する豊富なデータ構造を学び、配列よりも柔軟で強力なデータ管理方法を習得します

この章で学ぶこと

  • コレクションフレームワークの全体像と利点を理解する
  • List、Set、Mapの違いと使い分けを学ぶ
  • 各コレクションの主要なメソッドを使いこなす
  • イテレータを使った反復処理ができる

11.1 コレクションフレームワークとは

コレクションフレームワークは、複数のデータを効率的に管理するためのJavaの統一的な仕組みです。配列の限界を克服し、より柔軟で強力なデータ管理を可能にします。

配列との違い

配列は固定サイズで変更できませんが、コレクションは動的にサイズが変わります。また、コレクションは要素の追加・削除・検索などの便利なメソッドを標準で提供しています。

特徴 配列 コレクション
サイズ 固定 可変
型 プリミティブ型も可 オブジェクトのみ
便利メソッド 少ない 豊富
使いやすさ シンプル 高機能

主要なインターフェース

コレクションフレームワークは、以下の主要なインターフェースから構成されています。

ジェネリクスの基礎

コレクションはジェネリクス（型パラメータ）を使用して、扱うデータの型を明示します。これにより、型安全性が向上し、コンパイル時に型エラーを発見できます。

GenericExample.java 
// ジェネリクスを使った型指定
ArrayList<String> names = new ArrayList<>(); // 文字列のリスト
names.add("太郎");    // OK
names.add("花子");    // OK
// names.add(123);   // コンパイルエラー！整数は追加できない

// ジェネリクスを使わない場合（非推奨）
ArrayList list = new ArrayList(); // 警告が出る
list.add("文字列");
list.add(123);      // 何でも入るが、型安全ではない
String s = (String) list.get(0); // キャストが必要

11.2 Listインターフェース

Listは、順序を保持し、重複を許可するコレクションです。配列と似ていますが、サイズが動的に変更でき、便利なメソッドが豊富です。

ArrayList - 動的配列

ArrayListは、内部で配列を使用した可変長リストの実装です。インデックスによる高速なアクセスが可能です。

ArrayListExample.java 
import java.util.ArrayList;

public class ArrayListExample {
    public static void main(String[] args) {
        // ArrayListの作成
        ArrayList<String> fruits = new ArrayList<>();

        // 要素の追加
        fruits.add("りんご");
        fruits.add("バナナ");
        fruits.add("みかん");
        System.out.println(fruits); // [りんご, バナナ, みかん]

        // 要素の取得（インデックス指定）
        String first = fruits.get(0);
        System.out.println("最初の果物: " + first); // りんご

        // 要素数の取得
        System.out.println("要素数: " + fruits.size()); // 3

        // 要素の存在確認
        boolean hasBanana = fruits.contains("バナナ");
        System.out.println("バナナあり: " + hasBanana); // true

        // 要素の削除
        fruits.remove("バナナ");
        fruits.remove(0); // インデックスで削除
        System.out.println(fruits); // [みかん]
    }
}

LinkedList - 連結リスト

LinkedListは、各要素が次の要素への参照を持つ連結リスト構造です。要素の挿入・削除が頻繁な場合に有利です。

LinkedListExample.java 
import java.util.LinkedList;

public class LinkedListExample {
    public static void main(String[] args) {
        LinkedList<String> queue = new LinkedList<>();

        // 先頭と末尾への追加
        queue.addFirst("最初");
        queue.addLast("最後");
        queue.add("中間");
        System.out.println(queue); // [最初, 最後, 中間]

        // 先頭と末尾の取得・削除
        String first = queue.removeFirst();
        String last = queue.removeLast();
        System.out.println("削除: " + first + ", " + last);
        System.out.println(queue); // [中間]
    }
}

ArrayListとLinkedListの使い分け

  • ArrayList: インデックスアクセスが多い場合（例: ランダムアクセス）
  • LinkedList: 先頭や途中の要素の挿入・削除が多い場合
  • 迷ったらArrayListを選ぶのが一般的

Listの便利なメソッド

ListMethods.java 
import java.util.*;

public class ListMethods {
    public static void main(String[] args) {
        List<Integer> numbers = new ArrayList<>();
        numbers.add(10);
        numbers.add(20);
        numbers.add(30);
        numbers.add(20); // 重複OK

        // 要素の検索
        int index = numbers.indexOf(20);
        System.out.println("20の位置: " + index); // 1

        // 部分リストの取得
        List<Integer> sub = numbers.subList(1, 3);
        System.out.println("部分リスト: " + sub); // [20, 30]

        // リストのソート
        Collections.sort(numbers);
        System.out.println("ソート後: " + numbers); // [10, 20, 20, 30]

        // リストの反転
        Collections.reverse(numbers);
        System.out.println("反転後: " + numbers); // [30, 20, 20, 10]
    }
}

11.3 Setインターフェース

Setは、重複を許さないコレクションです。数学的な集合の概念に対応しており、要素の一意性を保証します。

HashSet - 高速な集合

HashSetは、ハッシュテーブルを使用した高速なSetの実装です。要素の順序は保証されません。

HashSetExample.java 
import java.util.HashSet;
import java.util.Set;

public class HashSetExample {
    public static void main(String[] args) {
        Set<String> tags = new HashSet<>();

        // 要素の追加（重複は無視される）
        tags.add("Java");
        tags.add("Python");
        tags.add("Java");  // 重複：追加されない
        tags.add("Ruby");

        System.out.println(tags); // [Java, Python, Ruby] 順不同
        System.out.println("要素数: " + tags.size()); // 3

        // 要素の存在確認
        if (tags.contains("Java")) {
            System.out.println("Javaタグが存在します");
        }

        // 要素の削除
        tags.remove("Python");
        System.out.println(tags);
    }
}

TreeSet - ソートされた集合

TreeSetは、要素を自動的にソートして格納するSetです。常に昇順で要素を保持します。

TreeSetExample.java 
import java.util.TreeSet;

public class TreeSetExample {
    public static void main(String[] args) {
        TreeSet<Integer> scores = new TreeSet<>();

        // 要素の追加（自動でソートされる）
        scores.add(85);
        scores.add(92);
        scores.add(78);
        scores.add(92); // 重複：追加されない

        System.out.println(scores); // [78, 85, 92] 昇順

        // 最小値・最大値の取得
        System.out.println("最低点: " + scores.first()); // 78
        System.out.println("最高点: " + scores.last());  // 92
    }
}

集合演算

SetOperations.java 
import java.util.*;

public class SetOperations {
    public static void main(String[] args) {
        Set<String> set1 = new HashSet<>(Arrays.asList("A", "B", "C"));
        Set<String> set2 = new HashSet<>(Arrays.asList("B", "C", "D"));

        // 和集合（union）
        Set<String> union = new HashSet<>(set1);
        union.addAll(set2);
        System.out.println("和集合: " + union); // [A, B, C, D]

        // 積集合（intersection）
        Set<String> intersection = new HashSet<>(set1);
        intersection.retainAll(set2);
        System.out.println("積集合: " + intersection); // [B, C]

        // 差集合（difference）
        Set<String> difference = new HashSet<>(set1);
        difference.removeAll(set2);
        System.out.println("差集合: " + difference); // [A]
    }
}

11.4 Mapインターフェース

Mapは、キーと値のペアを格納するコレクションです。キーは重複できず、各キーは1つの値にマッピングされます。辞書やハッシュテーブルとも呼ばれます。

HashMap - 高速なマップ

HashMapは、ハッシュテーブルを使用した高速なMapの実装です。キーによる高速な検索が可能です。

HashMapExample.java 
import java.util.HashMap;
import java.util.Map;

public class HashMapExample {
    public static void main(String[] args) {
        // 学生IDと名前のマップ
        Map<Integer, String> students = new HashMap<>();

        // キーと値の追加
        students.put(101, "田中太郎");
        students.put(102, "佐藤花子");
        students.put(103, "鈴木一郎");

        // 値の取得
        String name = students.get(102);
        System.out.println("ID 102: " + name); // 佐藤花子

        // キーの存在確認
        if (students.containsKey(101)) {
            System.out.println("ID 101が存在します");
        }

        // 値の存在確認
        if (students.containsValue("田中太郎")) {
            System.out.println("田中太郎が在籍しています");
        }

        // 要素数
        System.out.println("学生数: " + students.size()); // 3

        // 要素の削除
        students.remove(103);
        System.out.println(students);
    }
}

TreeMap - ソートされたマップ

TreeMapは、キーを自動的にソートして格納するMapです。キーの順序が重要な場合に使用します。

TreeMapExample.java 
import java.util.TreeMap;

public class TreeMapExample {
    public static void main(String[] args) {
        TreeMap<String, Integer> scores = new TreeMap<>();

        scores.put("Bob", 85);
        scores.put("Alice", 92);
        scores.put("Charlie", 78);

        // キーが自動的にソートされる
        System.out.println(scores);
        // {Alice=92, Bob=85, Charlie=78}
    }
}

Mapの反復処理

MapIteration.java 
import java.util.*;

public class MapIteration {
    public static void main(String[] args) {
        Map<String, Integer> fruits = new HashMap<>();
        fruits.put("りんご", 150);
        fruits.put("バナナ", 80);
        fruits.put("みかん", 100);

        // キーのセットで反復
        System.out.println("=== キーで反復 ===");
        for (String key : fruits.keySet()) {
            System.out.println(key + ": " + fruits.get(key) + "円");
        }

        // 値のコレクションで反復
        System.out.println("\n=== 値で反復 ===");
        for (Integer value : fruits.values()) {
            System.out.println(value + "円");
        }

        // エントリセットで反復（推奨）
        System.out.println("\n=== エントリで反復 ===");
        for (Map.Entry<String, Integer> entry : fruits.entrySet()) {
            System.out.println(entry.getKey() + ": " +
                             entry.getValue() + "円");
        }
    }
}

11.5 コレクションの操作

イテレータ（Iterator）

Iteratorは、コレクションの要素を順番に処理するための統一的な方法を提供します。

IteratorExample.java 
import java.util.*;

public class IteratorExample {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("A");
        list.add("B");
        list.add("C");

        // Iteratorを使った反復
        Iterator<String> it = list.iterator();
        while (it.hasNext()) {
            String element = it.next();
            System.out.println(element);

            // 反復中に安全に削除できる
            if (element.equals("B")) {
                it.remove();
            }
        }
        System.out.println(list); // [A, C]
    }
}

拡張for文

拡張for文（for-each）は、コレクションの要素を簡潔に反復処理できます。

ForEachExample.java 
import java.util.*;

public class ForEachExample {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

        // 拡張for文で反復
        for (Integer num : numbers) {
            System.out.println(num);
        }

        // Setでも同様に使える
        Set<String> set = new HashSet<>(Arrays.asList("X", "Y", "Z"));
        for (String s : set) {
            System.out.println(s);
        }
    }
}

Collectionsユーティリティクラス

Collectionsクラスは、コレクションを操作するための便利な静的メソッドを提供します。

CollectionsUtil.java 
import java.util.*;

public class CollectionsUtil {
    public static void main(String[] args) {
        List<Integer> list = new ArrayList<>(
            Arrays.asList(3, 1, 4, 1, 5, 9, 2, 6));

        // ソート
        Collections.sort(list);
        System.out.println("ソート: " + list);

        // 逆順
        Collections.reverse(list);
        System.out.println("逆順: " + list);

        // シャッフル
        Collections.shuffle(list);
        System.out.println("シャッフル: " + list);

        // 最大値・最小値
        int max = Collections.max(list);
        int min = Collections.min(list);
        System.out.println("最大: " + max + ", 最小: " + min);

        // 指定値で埋める
        Collections.fill(list, 0);
        System.out.println("fill: " + list);
    }
}

不変コレクション

Java 9以降では、List.of()、Set.of()、Map.of()で変更不可能なコレクションを簡単に作成できます。

ImmutableCollections.java 
import java.util.*;

public class ImmutableCollections {
    public static void main(String[] args) {
        // 不変リスト
        List<String> immutableList = List.of("A", "B", "C");
        // immutableList.add("D"); // UnsupportedOperationException

        // 不変セット
        Set<Integer> immutableSet = Set.of(1, 2, 3);

        // 不変マップ
        Map<String, Integer> immutableMap = Map.of(
            "One", 1,
            "Two", 2,
            "Three", 3
        );

        System.out.println(immutableList);
        System.out.println(immutableSet);
        System.out.println(immutableMap);
    }
}

実習 11-1: 単語の出現回数カウント

文字列に含まれる単語の出現回数をHashMapを使ってカウントするプログラムを作成しましょう。

手順

  1. 1 テキストを空白で分割して単語の配列を作成
  2. 2 HashMap<String, Integer>を使って単語をキー、出現回数を値として格納
  3. 3 結果を表示する

実装例

import java.util.*;

public class WordCounter {
    public static void main(String[] args) {
        String text = "Java is great Java is powerful";
        String[] words = text.split(" ");

        Map<String, Integer> wordCount = new HashMap<>();

        for (String word : words) {
            wordCount.put(word, wordCount.getOrDefault(word, 0) + 1);
        }

        for (Map.Entry<String, Integer> entry : wordCount.entrySet()) {
            System.out.println(entry.getKey() + ": " +
                             entry.getValue() + "回");
        }
    }
}

実習 11-2: 重複除去プログラム

リストから重複を除去してSetに変換するプログラムを作成しましょう。

実装例

import java.util.*;

public class RemoveDuplicates {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(
            1, 2, 3, 2, 4, 1, 5, 3, 6, 4);

        System.out.println("元のリスト: " + numbers);

        // Setに変換して重複除去
        Set<Integer> uniqueNumbers = new HashSet<>(numbers);
        System.out.println("重複除去後: " + uniqueNumbers);

        // 再度リストに戻す（必要に応じて）
        List<Integer> result = new ArrayList<>(uniqueNumbers);
        Collections.sort(result);
        System.out.println("ソート済み: " + result);
    }
}

理解度確認クイズ

  1. Q1. ListとSetの最大の違いは何ですか？

    回答を表示

    回答: Listは重複を許可し順序を保持しますが、Setは重複を許さず、要素の一意性を保証します。また、Listはインデックスによる要素アクセスができますが、Setはできません。

  2. Q2. HashMapとTreeMapの違いを説明してください。

    回答を表示

    回答: HashMapは順序を保証せず高速なアクセスを提供します。TreeMapはキーを自動的にソートして格納し、常に順序を維持しますが、HashMapより若干遅くなります。

  3. Q3. ジェネリクスを使う利点は何ですか？

    回答を表示

    回答: ジェネリクスを使うことで、コンパイル時に型安全性が保証され、実行時の型エラーを防げます。また、型キャストが不要になり、コードがより読みやすくなります。

  4. Q4. ArrayListとLinkedListの使い分けの基準を説明してください。

    回答を表示

    回答: ArrayListはインデックスによるランダムアクセスが頻繁な場合に適しています。LinkedListは要素の挿入・削除が頻繁に行われる場合（特に先頭や中間）に適しています。一般的にはArrayListが推奨されます。

実践のヒント

  • 迷ったらArrayList: 特別な理由がない限りArrayListを選ぶのが一般的です
  • 重複排除にはSet: 重複を許さないデータにはSetを使いましょう
  • キーで検索するならMap: IDや名前で検索する場合はMapが最適です
  • インターフェース型で宣言: List<String> list = new ArrayList<>();のように左辺はインターフェース型にすると柔軟性が高まります
