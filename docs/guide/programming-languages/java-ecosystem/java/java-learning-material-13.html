コレクションフレームワーク

Javaの強力なデータ構造であるコレクションフレームワークについて学び、List、Set、Mapの使い方と使い分けを理解します。

この章で学ぶこと

  • コレクションフレームワークとは何か、その利点を理解する
  • List（ArrayList、LinkedList）の特性と使い分け
  • Set（HashSet、TreeSet）の特性と重複排除の仕組み
  • Map（HashMap、TreeMap）のキーと値のペア管理
  • イテレータによるコレクションの走査方法

13.1 コレクションフレームワークとは

コレクションフレームワークとは、複数のデータをまとめて扱うための統一されたインターフェースとクラス群です。配列と似ていますが、より柔軟で強力な機能を持っています。

配列は固定長でサイズ変更ができませんが、コレクションフレームワークは動的にサイズが変わり、データの追加・削除・検索などの操作が簡単にできます。

コレクションフレームワークの主な利点

  • 動的サイズ: 要素数に応じて自動的にサイズが拡張・縮小
  • 豊富な操作メソッド: 追加・削除・検索・ソートなど便利なメソッドが用意されている
  • 型安全性: ジェネリクスを使って型を指定でき、実行時エラーを防げる
  • 統一されたインターフェース: 共通のインターフェースを持つため、異なる実装を同じように扱える

ジェネリクスとは

ジェネリクスは、コレクションに格納する要素の型を指定する仕組みです。例えばList<String>と書くことで、文字列のみを格納するリストを作成できます。これにより、誤った型のデータを追加するミスをコンパイル時に検出できます。

13.2 List（リスト）

Listは順序を持つコレクションで、同じ要素を複数回格納できます（重複可）。配列に似ていますが、サイズが動的に変わります。

ArrayList（最も一般的なList実装）

ArrayListは内部的に配列を使ってデータを管理します。インデックスによる高速アクセスが得意です。

ArrayListの基本的な使い方 
import java.util.ArrayList;
import java.util.List;

public class ArrayListExample {
    public static void main(String[] args) {
        // 文字列を格納するArrayListの作成
        List<String> fruits = new ArrayList<>();

        // 要素の追加
        fruits.add("りんご");
        fruits.add("バナナ");
        fruits.add("オレンジ");

        // インデックスによるアクセス
        System.out.println("最初の果物: " + fruits.get(0));

        // サイズの確認
        System.out.println("果物の数: " + fruits.size());

        // 全要素の表示
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}

LinkedList（挿入・削除が得意）

LinkedListは内部的にノードをリンクでつないでデータを管理します。先頭や途中への挿入・削除が高速です。

LinkedListの使用例 
import java.util.LinkedList;

public class LinkedListExample {
    public static void main(String[] args) {
        LinkedList<String> list = new LinkedList<>();

        // 末尾に追加
        list.add("最後");

        // 先頭に追加
        list.addFirst("最初");

        // 途中に追加
        list.add(1, "真ん中");

        System.out.println(list);  // [最初, 真ん中, 最後]
    }
}

使い分けのポイント

  • ArrayList: インデックスによるアクセスが多い場合（一般的な用途）
  • LinkedList: 先頭や途中への挿入・削除が頻繁にある場合

Listの主なメソッド

メソッド 説明
add(E element) 末尾に要素を追加
add(int index, E element) 指定位置に要素を挿入
get(int index) 指定位置の要素を取得
remove(int index) 指定位置の要素を削除
size() 要素数を返す
contains(Object o) 要素が含まれているか確認

13.3 Set（セット）

Setは重複を許さないコレクションです。同じ要素を複数回追加しようとしても、1つだけが保持されます。

HashSet（高速な検索）

HashSetはハッシュテーブルを使って要素を管理します。要素の追加・削除・検索が高速ですが、順序は保証されません。

HashSetの使用例 
import java.util.HashSet;
import java.util.Set;

public class HashSetExample {
    public static void main(String[] args) {
        Set<String> colors = new HashSet<>();

        // 要素の追加
        colors.add("赤");
        colors.add("青");
        colors.add("緑");
        colors.add("赤");  // 重複は無視される

        System.out.println("色の数: " + colors.size());  // 3

        // 含まれているか確認
        if (colors.contains("青")) {
            System.out.println("青が含まれています");
        }
    }
}

TreeSet（ソートされた順序）

TreeSetは要素を自動的にソートして保持します。順序が必要な場合に使用します。

TreeSetの使用例 
import java.util.TreeSet;

public class TreeSetExample {
    public static void main(String[] args) {
        TreeSet<Integer> numbers = new TreeSet<>();

        // 順不同で追加
        numbers.add(5);
        numbers.add(1);
        numbers.add(3);

        // 自動的にソートされて出力
        System.out.println(numbers);  // [1, 3, 5]
    }
}

重複判定の注意

Setの重複判定は、オブジェクトのequals()メソッドとhashCode()メソッドを使って行われます。独自クラスをSetに格納する場合は、これらのメソッドを適切にオーバーライドする必要があります。

13.4 Map（マップ）

Mapはキーと値のペアを管理するコレクションです。各キーは一意である必要があり、キーを使って対応する値を高速に取得できます。

HashMap（最も一般的なMap実装）

HashMapはハッシュテーブルを使ってキーと値を管理します。辞書や電話帳のような用途に最適です。

HashMapの使用例 
import java.util.HashMap;
import java.util.Map;

public class HashMapExample {
    public static void main(String[] args) {
        // 商品名と価格を管理するMap
        Map<String, Integer> prices = new HashMap<>();

        // キーと値のペアを追加
        prices.put("りんご", 100);
        prices.put("バナナ", 80);
        prices.put("オレンジ", 120);

        // キーで値を取得
        System.out.println("りんごの価格: " + prices.get("りんご") + "円");

        // 全てのエントリを表示
        for (Map.Entry<String, Integer> entry : prices.entrySet()) {
            System.out.println(entry.getKey() + ": " + entry.getValue() + "円");
        }
    }
}

TreeMap（ソートされたMap）

TreeMapはキーを自動的にソートして保持します。キーの順序が重要な場合に使用します。

13.5 イテレータによるコレクションの走査

イテレータ（Iterator）は、コレクションの要素を順番に処理するための統一された方法を提供します。

イテレータの使用例 
import java.util.*;

public class IteratorExample {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>();
        names.add("太郎");
        names.add("花子");
        names.add("次郎");

        // イテレータを取得
        Iterator<String> iterator = names.iterator();

        // 要素を順番に処理
        while (iterator.hasNext()) {
            String name = iterator.next();
            System.out.println(name);
        }
    }
}

拡張forループ（より簡潔な方法）

通常は、イテレータを直接使うよりも拡張forループ（for-each）を使う方が簡潔です。

拡張forループの使用例 
// List、Set、Mapのvalues()など、
// あらゆるコレクションで使用可能
for (String name : names) {
    System.out.println(name);
}

// Map全体の走査
Map<String, Integer> scores = new HashMap<>();
scores.put("太郎", 85);
scores.put("花子", 92);

for (Map.Entry<String, Integer> entry : scores.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}

実習 13-1: 成績管理システムを作ろう

コレクションフレームワークを活用して、生徒の成績を管理するプログラムを作成しましょう。

要件

  • • Mapを使って、生徒名をキー、成績をバリューとして管理する
  • • 複数の生徒の成績を登録できる
  • • 全生徒の成績一覧を表示できる
  • • 平均点を計算して表示できる

実装例

import java.util.*;

public class GradeManager {
    public static void main(String[] args) {
        // 成績を管理するMap
        Map<String, Integer> grades = new HashMap<>();

        // 成績の登録
        grades.put("太郎", 85);
        grades.put("花子", 92);
        grades.put("次郎", 78);

        // 全生徒の成績を表示
        System.out.println("=== 成績一覧 ===");
        for (Map.Entry<String, Integer> entry : grades.entrySet()) {
            System.out.println(entry.getKey() + ": " + entry.getValue() + "点");
        }

        // 平均点を計算
        int total = 0;
        for (int score : grades.values()) {
            total += score;
        }
        double average = (double) total / grades.size();
        System.out.println("\n平均点: " + average + "点");
    }
}

チャレンジ: TreeMapを使って、生徒名を五十音順にソートして表示してみましょう。また、最高点と最低点を求める機能も追加してみてください。

理解度確認クイズ

  1. Q1. ArrayListとLinkedListの主な違いは何ですか？

    回答を表示

    ArrayList: 内部的に配列を使用。インデックスによるアクセスが高速。
    LinkedList: ノードをリンクでつなぐ構造。先頭や途中への挿入・削除が高速。
    一般的にはArrayListを使い、頻繁な挿入・削除がある場合はLinkedListを検討します。

  2. Q2. SetとListの最も大きな違いは何ですか？

    回答を表示

    Set: 重複を許さない（同じ要素は1つだけ）
    List: 重複を許す（同じ要素を複数回追加できる）
    また、Listは順序を保持しますが、HashSetは順序を保証しません。

  3. Q3. Mapでキーと値のペアを追加するメソッドは何ですか？

    回答を表示

    put(K key, V value)
    例: map.put("りんご", 100) でキー「りんご」、値100のペアを追加します。

  4. Q4. ジェネリクスを使うことの利点は何ですか？

    回答を表示

    1. 型安全性: コンパイル時に型チェックが行われ、実行時エラーを防げる
    2. キャストが不要: 要素取得時に型キャストする必要がない
    3. コードの可読性向上: どの型が格納されているか明確になる

この章のまとめ

  • コレクションフレームワークは、動的サイズで豊富な操作メソッドを持つデータ構造群
  • Listは順序を保持し重複可能、ArrayListとLinkedListは用途に応じて使い分ける
  • Setは重複を許さず、HashSetは高速、TreeSetは自動ソート機能を持つ
  • Mapはキーと値のペアを管理し、辞書的な用途に最適
  • イテレータや拡張forループで、統一的にコレクションを走査できる
  • ジェネリクスを使うことで型安全性が高まり、実行時エラーを防げる
