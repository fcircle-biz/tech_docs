第3章

演算子と式

算術演算、比較演算、論理演算など、プログラムで値を計算したり判定したりするための演算子について学びます。式の評価順序や型変換についても理解を深めます。

この章で学ぶこと

  • 算術演算子（+、-、*、/、%）の使い方
  • 比較演算子（==、!=、<、>など）による条件判定
  • 論理演算子（&&、||、!）による複雑な条件の組み合わせ
  • 演算子の優先順位と式の評価順序、型変換の仕組み

3.1 算術演算子

算術演算子は、数値の計算を行うための演算子です。加算、減算、乗算、除算など、数学でよく使う計算をJavaで実行できます。

基本的な算術演算子

演算子 名称 例 結果
+ 加算（足し算） 10 + 3 13
- 減算（引き算） 10 - 3 7
* 乗算（掛け算） 10 * 3 30
/ 除算（割り算） 10 / 3 3（整数除算）
% 剰余（余り） 10 % 3 1
ArithmeticExample.java 
public class ArithmeticExample {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;

        // 加算
        System.out.println("10 + 3 = " + (a + b));  // 13
        // 減算
        System.out.println("10 - 3 = " + (a - b));  // 7
        // 乗算
        System.out.println("10 * 3 = " + (a * b));  // 30
        // 除算（整数）
        System.out.println("10 / 3 = " + (a / b));  // 3
        // 剰余
        System.out.println("10 % 3 = " + (a % b));  // 1
    }
}

整数除算の注意点

整数同士の除算では、小数部分が切り捨てられます。10 / 3は3.333...ではなく3になります。小数を含む結果が必要な場合は、どちらかをdouble型にキャストするか、最初からdouble型で計算してください。

インクリメント・デクリメント演算子

値を1増やす（インクリメント）、1減らす（デクリメント）演算子は、カウンタ変数などでよく使われます。

IncrementExample.java 
public class IncrementExample {
    public static void main(String[] args) {
        int count = 5;

        // インクリメント（1増やす）
        count++;  // count = count + 1 と同じ
        System.out.println("count++ → " + count);  // 6

        // デクリメント（1減らす）
        count--;  // count = count - 1 と同じ
        System.out.println("count-- → " + count);  // 5
    }
}

3.2 比較演算子

比較演算子は、2つの値を比較して、その結果を真（true）または偽（false）のboolean値として返します。if文などの条件分岐で頻繁に使用されます。

演算子 名称 例 結果
== 等しい 5 == 5 true
!= 等しくない 5 != 3 true
< より小さい 3 < 5 true
> より大きい 5 > 3 true
<= 以下 5 <= 5 true
>= 以上 5 >= 3 true
ComparisonExample.java 
public class ComparisonExample {
    public static void main(String[] args) {
        int age = 20;

        // 比較演算の結果はboolean型
        boolean isAdult = age >= 20;
        System.out.println("成人? " + isAdult);  // true

        boolean isTeenager = age >= 13 && age <= 19;
        System.out.println("ティーンエイジャー? " + isTeenager);  // false
    }
}

==と=の違い

==は「等しいかどうか」を比較する演算子、=は値を代入する演算子です。if (x = 5)と書くとエラーになります。正しくはif (x == 5)です。

3.3 論理演算子

論理演算子は、複数の条件を組み合わせて判定する際に使います。「AかつB」「AまたはB」「Aでない」といった論理的な組み合わせを表現できます。

演算子 名称 説明 例
&& 論理積（AND） 両方がtrueの時にtrue true && true → true
|| 論理和（OR） 少なくとも一方がtrueならtrue true || false → true
! 論理否定（NOT） trueとfalseを反転 !true → false
LogicalExample.java 
public class LogicalExample {
    public static void main(String[] args) {
        int age = 25;
        boolean hasLicense = true;

        // ANDの例：両方の条件を満たす必要がある
        boolean canDrive = age >= 18 && hasLicense;
        System.out.println("運転可能? " + canDrive);  // true

        // ORの例：どちらか一方を満たせばよい
        boolean isDiscount = age < 18 || age >= 65;
        System.out.println("割引対象? " + isDiscount);  // false

        // NOTの例：条件を反転
        boolean isNotAdult = !(age >= 20);
        System.out.println("未成年? " + isNotAdult);  // false
    }
}

論理演算の動作イメージ

flowchart LR A[条件A: true] --> AND{"AND演算 &&"} B[条件B: false] --> AND AND --> R1[結果: false] A2[条件A: true] --> OR{"OR演算 ||"} B2[条件B: false] --> OR OR --> R2[結果: true] A3[条件: true] --> NOT{"NOT演算 !"} NOT --> R3[結果: false]

短絡評価

&&では左側がfalseなら右側は評価されません。||では左側がtrueなら右側は評価されません。この性質を「短絡評価」といい、パフォーマンス向上やエラー回避に役立ちます。

3.4 演算子の優先順位

複数の演算子を含む式では、演算が実行される順序（優先順位）が決まっています。数学と同じように、掛け算・割り算は足し算・引き算より先に計算されます。

主な演算子の優先順位（高い順）

優先度 演算子 説明
1（最高） ++, -- インクリメント、デクリメント
2 ! 論理否定
3 *, /, % 乗算、除算、剰余
4 +, - 加算、減算
5 <, >, <=, >= 比較演算子
6 ==, != 等価演算子
7 && 論理積
8 || 論理和
9（最低） = 代入
PriorityExample.java 
public class PriorityExample {
    public static void main(String[] args) {
        // 乗算が先に実行される
        int result1 = 5 + 3 * 2;
        System.out.println("5 + 3 * 2 = " + result1);  // 11（3*2=6が先、次に5+6）

        // カッコで優先順位を変更
        int result2 = (5 + 3) * 2;
        System.out.println("(5 + 3) * 2 = " + result2);  // 16

        // 複雑な式の評価
        boolean check = 10 > 5 && 3 < 7;  // trueかつtrue = true
        System.out.println("10 > 5 && 3 < 7 = " + check);
    }
}

カッコを使って読みやすく

優先順位を覚えるのは大変です。迷ったときは()を使って明示的に順序を指定しましょう。コードの可読性も向上します。

3.5 型変換（キャスト）

異なるデータ型同士で演算を行う場合、型変換が発生します。型変換には自動的に行われる「暗黙の型変換」と、明示的に指定する「明示的な型変換（キャスト）」があります。

暗黙の型変換

小さい型から大きい型への変換は自動的に行われます（情報が失われないため）。

ImplicitCast.java 
public class ImplicitCast {
    public static void main(String[] args) {
        int intValue = 100;
        // int型からdouble型へ自動変換
        double doubleValue = intValue;
        System.out.println("int → double: " + doubleValue);  // 100.0

        // 整数と小数の演算では、自動的にdoubleになる
        double result = 10 / 4.0;
        System.out.println("10 / 4.0 = " + result);  // 2.5
    }
}

明示的な型変換（キャスト）

大きい型から小さい型へ変換する場合は、明示的にキャストが必要です。情報が失われる可能性があるため、プログラマが意図的に指定する必要があります。

ExplicitCast.java 
public class ExplicitCast {
    public static void main(String[] args) {
        double pi = 3.14159;
        // double型からint型へ明示的にキャスト（小数部分は切り捨て）
        int intPi = (int) pi;
        System.out.println("(int)3.14159 = " + intPi);  // 3

        // 整数除算で小数を得るには、どちらかをdoubleにキャスト
        int a = 10;
        int b = 3;
        double division = (double) a / b;
        System.out.println("10 / 3 = " + division);  // 3.333...
    }
}

キャストによるデータ損失

double型からint型へキャストすると、小数部分は切り捨てられます。また、大きな値を小さい型にキャストすると、オーバーフローが発生する可能性があります。キャストは慎重に使いましょう。

実習 3-1: 消費税計算プログラムを作ろう

演算子を使って、商品価格から消費税込みの金額を計算するプログラムを作成しましょう。

課題

商品価格（税抜）を入力し、消費税率10%を加算した税込価格を計算して表示してください。

手順

  1. 1 税抜価格を変数に格納します（例: int price = 1000;）
  2. 2 消費税率を定数で定義します（double TAX_RATE = 0.10;）
  3. 3 税込価格を計算して表示します

実装例

public class TaxCalculator {
    public static void main(String[] args) {
        int price = 1000;  // 税抜価格
        double taxRate = 0.10;  // 消費税率10%

        // 税込価格を計算
        double totalPrice = price * (1 + taxRate);

        System.out.println("税抜価格: " + price + "円");
        System.out.println("消費税率: " + (taxRate * 100) + "%");
        System.out.println("税込価格: " + totalPrice + "円");
    }
}

期待される結果

税抜価格: 1000円
消費税率: 10.0%
税込価格: 1100.0円

チャレンジ: 税込価格をint型で表示するようにキャストしてみましょう。

理解度確認クイズ

  1. Q1. 次のコードの実行結果はいくつですか？ int result = 10 % 3;

    回答を表示

    答え: 1
    %は剰余演算子で、10を3で割った余りは1です。

  2. Q2. 論理演算 true && false の結果は？

    回答を表示

    答え: false
    &&（AND）は両方がtrueの時だけtrueになります。一方がfalseなので結果はfalseです。

  3. Q3. 5 + 3 * 2 の計算結果は？

    回答を表示

    答え: 11
    乗算（*）が加算（+）より優先されます。まず3 * 2 = 6が計算され、次に5 + 6 = 11となります。

  4. Q4. int型の変数をdouble型に変換する際、明示的なキャストは必要ですか？

    回答を表示

    答え: 不要
    小さい型から大きい型への変換は暗黙的に行われます（情報損失がないため）。逆にdouble型からint型への変換には明示的なキャストが必要です。

この章のまとめ

  • 算術演算子（+、-、*、/、%）で四則演算や剰余計算ができる
  • 比較演算子（==、!=、<、>など）で値の大小や等価を判定できる
  • 論理演算子（&&、||、!）で複数の条件を組み合わせられる
  • 演算子には優先順位があり、カッコで明示的に順序を制御できる
  • 型変換には暗黙的なものと明示的（キャスト）なものがあり、適切に使い分ける必要がある
