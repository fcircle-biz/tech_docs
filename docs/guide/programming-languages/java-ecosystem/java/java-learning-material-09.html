第9章

継承とポリモーフィズム - クラスの関係性

クラス間の関係性を表現する「継承」と、柔軟なプログラム設計を可能にする「ポリモーフィズム」を学びます。

この章で学ぶこと

  • 継承の概念と使い方を理解する
  • メソッドのオーバーライドができる
  • ポリモーフィズムの仕組みを理解する
  • 抽象クラスとインターフェースの基礎を学ぶ

9.1 継承とは

継承（Inheritance）は、既存のクラスの機能を引き継いで新しいクラスを作成する仕組みです。継承を使うと、共通する機能を一度だけ定義し、複数のクラスで再利用できます。

継承は「is-a関係」（AはBの一種である）を表現します。例えば、「犬は動物の一種である」という関係を、継承を使ってコードで表現できます。

継承の用語

  • スーパークラス（親クラス）: 機能を提供する元となるクラス
  • サブクラス（子クラス）: 機能を引き継ぐクラス
  • extends: 継承を表すキーワード
classDiagram

        -String name
        +Animal(String name)

        +void sleep()

Animal.java 
// 親クラス（スーパークラス）
public class Animal {
    // フィールド
    protected String name;  // protectedは子クラスからアクセス可能

    // コンストラクタ
    public Animal(String name) {
        this.name = name;
    }

    // メソッド
    public void eat() {
        System.out.println(name + "が食事をしています");
    }

    public void sleep() {
        System.out.println(name + "が眠っています");
    }
}
Dog.java 
// 子クラス（サブクラス）
public class Dog extends Animal {
    // コンストラクタ
    public Dog(String name) {
        super(name);  // 親クラスのコンストラクタを呼び出す
    }

    // 独自のメソッド
    public void bark() {
        System.out.println(name + "がワンワン吠えています");
    }
}

// 使用例
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog("ポチ");
        dog.eat();    // 親クラスのメソッド
        dog.sleep();  // 親クラスのメソッド
        dog.bark();   // 自分のメソッド
    }
}

実行結果：

ポチが食事をしています
ポチが眠っています
ポチがワンワン吠えています

9.2 継承の実装

継承を実装する際には、superキーワードを使って親クラスのコンストラクタやメソッドにアクセスします。

重要なポイント

  • 子クラスのコンストラクタの最初の行で、必ず親クラスのコンストラクタを呼び出す必要があります
  • super()の呼び出しを省略すると、引数なしの親クラスのコンストラクタが自動的に呼ばれます
  • Javaでは単一継承のみがサポートされます（1つの親クラスからしか継承できません）
Shape.java 
// 親クラス: 図形
public class Shape {
    protected String color;

    public Shape(String color) {
        this.color = color;
    }

    public void displayInfo() {
        System.out.println("色: " + color);
    }
}

// 子クラス: 円
public class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);  // 親クラスのコンストラクタを呼び出す
        this.radius = radius;
    }

    public double calculateArea() {
        return Math.PI * radius * radius;
    }

    @Override
    public void displayInfo() {
        super.displayInfo();  // 親クラスのメソッドを呼び出す
        System.out.println("半径: " + radius);
        System.out.println("面積: " + calculateArea());
    }
}

// 使用例
public class Main {
    public static void main(String[] args) {
        Circle circle = new Circle("赤", 5.0);
        circle.displayInfo();
    }
}

9.3 メソッドのオーバーライド

オーバーライド（Override）は、親クラスで定義されたメソッドを子クラスで書き換えることです。同じメソッド名と引数を持つメソッドを子クラスで定義することで、親クラスのメソッドの動作を変更できます。

@Overrideアノテーション

@Overrideアノテーションを付けることで、コンパイラがオーバーライドのミス（メソッド名や引数の間違い）を検出してくれます。必須ではありませんが、付けることを強く推奨します。

OverrideExample.java 
// 親クラス
class Vehicle {
    public void start() {
        System.out.println("乗り物が動き始めます");
    }

    public void stop() {
        System.out.println("乗り物が停止します");
    }
}

// 子クラス: 車
class Car extends Vehicle {
    @Override
    public void start() {
        System.out.println("エンジンをかけて車が発進します");
    }
    // stopメソッドはオーバーライドしない（親クラスのまま）
}

// 子クラス: 自転車
class Bicycle extends Vehicle {
    @Override
    public void start() {
        System.out.println("ペダルをこいで自転車が進みます");
    }

    @Override
    public void stop() {
        System.out.println("ブレーキをかけて自転車が止まります");
    }
}

// 使用例
public class Main {
    public static void main(String[] args) {
        Vehicle vehicle = new Vehicle();
        vehicle.start();
        vehicle.stop();
        System.out.println();

        Car car = new Car();
        car.start();      // オーバーライドされたメソッド
        car.stop();       // 親クラスのメソッド
        System.out.println();

        Bicycle bicycle = new Bicycle();
        bicycle.start();  // オーバーライドされたメソッド
        bicycle.stop();   // オーバーライドされたメソッド
    }
}

つまずきやすいポイント：オーバーロードとオーバーライドの違い

オーバーライド（Override）: 親クラスのメソッドを子クラスで書き換える

  • メソッド名、引数、戻り値が同じ
  • 継承関係が必要
  • 実行時に呼ばれるメソッドが決まる

オーバーロード（Overload）: 同じ名前で引数が異なるメソッドを複数定義

  • メソッド名が同じ、引数が異なる
  • 継承関係は不要（同じクラス内でも可能）
  • コンパイル時に呼ばれるメソッドが決まる

9.4 ポリモーフィズム

ポリモーフィズム（多態性）は、親クラス型の変数で子クラスのオブジェクトを扱える仕組みです。これにより、異なる種類のオブジェクトを統一的に扱うことができます。

ポリモーフィズムの利点

  • 異なるクラスのオブジェクトを同じ方法で扱える
  • コードの柔軟性と拡張性が向上する
  • 新しい子クラスを追加しても既存のコードを変更する必要がない
PolymorphismExample.java 
// 親クラス
class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public void makeSound() {
        System.out.println("何か音を出します");
    }
}

// 子クラス1: 犬
class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    public void makeSound() {
        System.out.println(name + ": ワンワン！");
    }
}

// 子クラス2: 猫
class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    @Override
    public void makeSound() {
        System.out.println(name + ": ニャー！");
    }
}

// 子クラス3: 鳥
class Bird extends Animal {
    public Bird(String name) {
        super(name);
    }

    @Override
    public void makeSound() {
        System.out.println(name + ": チュンチュン！");
    }
}

public class Main {
    public static void main(String[] args) {
        // ポリモーフィズム: 親クラス型の配列で子クラスのオブジェクトを管理
        Animal[] animals = new Animal[3];
        animals[0] = new Dog("ポチ");
        animals[1] = new Cat("タマ");
        animals[2] = new Bird("ピー");

        // すべての動物に同じメソッドを呼び出す
        // 実際には各クラスのオーバーライドされたメソッドが呼ばれる
        for (Animal animal : animals) {
            animal.makeSound();  // 動的バインディング
        }
    }
}

実行結果：

ポチ: ワンワン！
タマ: ニャー！
ピー: チュンチュン！

動的バインディング

ポリモーフィズムでは、変数の型ではなく実際のオブジェクトの型に基づいてメソッドが呼び出されます。これを動的バインディングと呼びます。コンパイル時ではなく実行時にどのメソッドを呼ぶかが決定されます。

9.5 抽象クラスとインターフェース

継承をさらに活用するために、抽象クラスとインターフェースという仕組みがあります。これらは「共通の型」を定義し、複数のクラスに共通の振る舞いを強制する役割を持ちます。

抽象クラス（Abstract Class）

抽象クラスは、abstractキーワードで定義されたクラスで、インスタンス化できません。抽象メソッド（実装を持たないメソッド）を含むことができ、子クラスで必ず実装する必要があります。

AbstractClassExample.java 
// 抽象クラス
abstract class Shape {
    protected String color;

    public Shape(String color) {
        this.color = color;
    }

    // 通常のメソッド（実装あり）
    public void displayColor() {
        System.out.println("色: " + color);
    }

    // 抽象メソッド（実装なし）
    public abstract double calculateArea();
    public abstract double calculatePerimeter();
}

// 具象クラス: 円
class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }

    @Override
    public double calculatePerimeter() {
        return 2 * Math.PI * radius;
    }
}

// 具象クラス: 長方形
class Rectangle extends Shape {
    private double width;
    private double height;

    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }

    @Override
    public double calculateArea() {
        return width * height;
    }

    @Override
    public double calculatePerimeter() {
        return 2 * (width + height);
    }
}

public class Main {
    public static void main(String[] args) {
        // Shape shape = new Shape("赤");  // エラー: 抽象クラスはインスタンス化できない

        Shape circle = new Circle("赤", 5.0);
        circle.displayColor();
        System.out.println("面積: " + circle.calculateArea());
        System.out.println("周囲の長さ: " + circle.calculatePerimeter());
        System.out.println();

        Shape rectangle = new Rectangle("青", 4.0, 6.0);
        rectangle.displayColor();
        System.out.println("面積: " + rectangle.calculateArea());
        System.out.println("周囲の長さ: " + rectangle.calculatePerimeter());
    }
}

インターフェース（Interface）

インターフェースは、クラスが実装すべきメソッドの「契約」を定義します。複数のインターフェースを実装できるため、多重継承のような柔軟性が得られます。

抽象クラスとインターフェースの違い

項目 抽象クラス インターフェース
継承 1つのみ 複数可能
フィールド 持てる 定数のみ
実装メソッド 持てる デフォルトメソッドのみ
用途 共通の基底クラス 共通の機能の契約
InterfaceExample.java 
// インターフェース: 飛べる
interface Flyable {
    void fly();  // 抽象メソッド（publicは省略可能）
    void land();
}

// インターフェース: 泳げる
interface Swimmable {
    void swim();
}

// 鳥クラス: Flyableを実装
class Bird implements Flyable {
    private String name;

    public Bird(String name) {
        this.name = name;
    }

    @Override
    public void fly() {
        System.out.println(name + "が空を飛んでいます");
    }

    @Override
    public void land() {
        System.out.println(name + "が着地しました");
    }
}

// カモクラス: FlyableとSwimmableの両方を実装
class Duck implements Flyable, Swimmable {
    private String name;

    public Duck(String name) {
        this.name = name;
    }

    @Override
    public void fly() {
        System.out.println(name + "が空を飛んでいます");
    }

    @Override
    public void land() {
        System.out.println(name + "が水面に着水しました");
    }

    @Override
    public void swim() {
        System.out.println(name + "が泳いでいます");
    }
}

public class Main {
    public static void main(String[] args) {
        Bird bird = new Bird("スズメ");
        bird.fly();
        bird.land();
        System.out.println();

        Duck duck = new Duck("アヒル");
        duck.fly();
        duck.swim();
        duck.land();
        System.out.println();

        // ポリモーフィズム: インターフェース型の変数で扱う
        Flyable[] flyables = {bird, duck};
        for (Flyable flyable : flyables) {
            flyable.fly();
        }
    }
}

実習 9-1: 動物園プログラムの作成

継承とポリモーフィズムを使って、様々な動物を管理する動物園プログラムを作成しましょう。

手順

  1. 1 Animalクラスを作成し、name、ageフィールドとeatメソッド、sleepメソッドを定義する
  2. 2 Lion、Elephant、Monkeyの各クラスを作成し、Animalクラスを継承する
  3. 3 各動物クラスで独自のメソッド（例: roar()、trumpet()、swing()）を追加する
  4. 4 Animal配列を使って複数の動物をまとめて管理し、全ての動物にeatメソッドを呼び出す

実装例

Animal.java 
// Animalクラス
class Animal {
    protected String name;
    protected int age;

    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void eat() {
        System.out.println(name + "が食事をしています");
    }

    public void sleep() {
        System.out.println(name + "が眠っています");
    }
}

// 使用例
Animal[] animals = new Animal[3];
animals[0] = new Lion("レオ", 5);
animals[1] = new Elephant("ジャンボ", 10);
animals[2] = new Monkey("モンキチ", 3);

for (Animal animal : animals) {
    animal.eat();
}

理解度確認クイズ

  1. Q1. 継承を使う主な理由は何ですか？複数の理由を挙げてください。

    回答を表示

    継承を使う主な理由：

    • コードの再利用: 共通する機能を親クラスで一度定義し、複数の子クラスで再利用できる
    • 階層的な関係の表現: 「is-a関係」を明確にコードで表現できる
    • 保守性の向上: 共通機能の変更が一箇所で済む
    • ポリモーフィズム: 親クラス型で複数の子クラスを統一的に扱える

  2. Q2. オーバーライドとオーバーロードの違いを説明してください。

    回答を表示

    オーバーライド（Override）

    • 親クラスのメソッドを子クラスで書き換える
    • メソッド名、引数、戻り値の型が同じ
    • 継承関係が必要
    • 実行時に呼ばれるメソッドが決まる（動的バインディング）

    オーバーロード（Overload）

    • 同じ名前で引数が異なるメソッドを複数定義
    • メソッド名は同じだが、引数の数や型が異なる
    • 同じクラス内でも可能（継承不要）
    • コンパイル時に呼ばれるメソッドが決まる

  3. Q3. ポリモーフィズムを使うと、どのような利点がありますか？

    回答を表示

    ポリモーフィズムの利点：

    • 統一的な処理: 異なるクラスのオブジェクトを同じ方法で扱える
    • 柔軟性: 新しい子クラスを追加しても既存のコードを変更する必要がない
    • 保守性: コードの重複を減らし、変更箇所を最小限にできる
    • 拡張性: 親クラス型の変数やメソッド引数を使うことで、将来追加される子クラスにも対応できる

  4. Q4. 抽象クラスとインターフェースの使い分けの基準を説明してください。

    回答を表示

    抽象クラスを使う場合

    • 共通の状態（フィールド）を持つ場合
    • 共通の実装を持つメソッドがある場合
    • is-a関係が明確な場合（例: 犬は動物である）

    インターフェースを使う場合

    • 複数の無関係なクラスに共通の機能を持たせたい場合
    • 多重継承のような柔軟性が必要な場合
    • 「〜できる」という能力を表現する場合（例: 飛べる、泳げる）

この章のまとめ

  • 継承は、既存のクラスの機能を引き継いで新しいクラスを作成する仕組みで、コードの再利用性を高める
  • オーバーライドにより、親クラスのメソッドを子クラスで書き換えて、独自の動作を実装できる
  • ポリモーフィズムにより、親クラス型の変数で子クラスのオブジェクトを扱い、統一的な処理が可能になる
  • 抽象クラスは共通の基底クラスを定義し、子クラスで実装を強制する
  • インターフェースは複数のクラスに共通の機能の契約を定義し、多重継承のような柔軟性を提供する
