第9章

継承とポリモーフィズム

クラス間の関係性を表現する「継承」と、柔軟なプログラム設計を可能にする「ポリモーフィズム」を学びます。

この章で学ぶこと

  • 継承の概念と「is-a関係」を理解し、実装できる
  • メソッドのオーバーライドとsuperキーワードを使いこなせる
  • ポリモーフィズム（多態性）の仕組みと利点を理解する
  • 抽象クラスとインターフェースの基本を学び、使い分けができる

9.1 継承とは何か

継承（Inheritance）は、オブジェクト指向プログラミングの重要な概念の一つで、既存のクラスの機能を受け継いで新しいクラスを作成する仕組みです。これにより、コードの再利用性が高まり、プログラムの保守性が向上します。

継承の基本概念

継承では、元となるクラスを親クラス（スーパークラス、基底クラス）、新しく作成するクラスを子クラス（サブクラス、派生クラス）と呼びます。子クラスは親クラスのフィールドとメソッドを引き継ぎ、さらに独自の機能を追加できます。

is-a関係とは

継承が適切に使用されるべき場面は、「is-a（〜は〜である）」の関係が成り立つときです。例えば、「犬は動物である（Dog is an Animal）」という関係が成り立つ場合、DogクラスはAnimalクラスを継承するのが適切です。

継承の利点

  • コードの再利用: 共通機能を親クラスにまとめることで、重複コードを削減できます
  • 保守性の向上: 共通機能の修正は親クラスだけで済み、すべての子クラスに自動的に反映されます
  • 拡張性: 既存のクラスを変更せずに機能を追加できます
  • 階層構造の表現: 現実世界の分類を自然にモデル化できます

9.2 継承の実装

Javaではextendsキーワードを使用して継承を実装します。基本的な構文を見てみましょう。

Animal.java 
// 親クラス（スーパークラス）
public class Animal {
    // フィールド（属性）
    protected String name;  // protectedは子クラスからアクセス可能
    protected int age;

    // コンストラクタ
    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // メソッド（振る舞い）
    public void eat() {
        System.out.println(name + "が食事をしています");
    }

    public void sleep() {
        System.out.println(name + "が眠っています");
    }

    public void makeSound() {
        System.out.println(name + "が何か音を立てています");
    }
}
Dog.java 
// 子クラス（サブクラス） - extendsで継承
public class Dog extends Animal {
    // 子クラス独自のフィールド
    private String breed;  // 犬種

    // コンストラクタ
    public Dog(String name, int age, String breed) {
        super(name, age);  // 親クラスのコンストラクタを呼び出す
        this.breed = breed;
    }

    // 子クラス独自のメソッド
    public void fetch() {
        System.out.println(name + "がボールを取ってきます");
    }

    // getter
    public String getBreed() {
        return breed;
    }
}

superキーワード

superキーワードは、親クラスのメンバーにアクセスするために使用します。主に2つの用途があります：

  • 親クラスのコンストラクタ呼び出し: super(引数)で親クラスの初期化を行います（コンストラクタの最初の行に記述）
  • 親クラスのメソッド呼び出し: super.メソッド名()で親クラスのメソッドを実行できます
InheritanceExample.java 
public class InheritanceExample {
    public static void main(String[] args) {
        // 子クラスのオブジェクトを作成
        Dog myDog = new Dog("ポチ", 3, "柴犬");

        // 親クラスから継承したメソッドを使用
        myDog.eat();      // Animalクラスから継承
        myDog.sleep();    // Animalクラスから継承

        // 子クラス独自のメソッドを使用
        myDog.fetch();    // Dogクラス固有

        System.out.println("犬種: " + myDog.getBreed());
    }
}

// 実行結果:
// ポチが食事をしています
// ポチが眠っています
// ポチがボールを取ってきます
// 犬種: 柴犬

protectedアクセス修飾子

protectedは、同じパッケージ内のクラスと、パッケージが異なる場合でも子クラスからアクセスできるアクセス修飾子です。継承関係でフィールドやメソッドを共有したい場合によく使用されます。

9.3 メソッドのオーバーライド

メソッドのオーバーライド（上書き）は、子クラスで親クラスのメソッドを再定義し、独自の振る舞いを実装する機能です。これにより、親クラスの基本動作を保ちながら、子クラス特有の動作に変更できます。

Cat.java 
public class Cat extends Animal {

    public Cat(String name, int age) {
        super(name, age);
    }

    // メソッドのオーバーライド - 親クラスのメソッドを上書き
    @Override  // オーバーライドを明示するアノテーション（推奨）
    public void makeSound() {
        System.out.println(name + "がニャーと鳴きます");
    }

    // 猫特有のメソッド
    public void scratch() {
        System.out.println(name + "が爪を研いでいます");
    }
}
Bird.java 
public class Bird extends Animal {

    public Bird(String name, int age) {
        super(name, age);
    }

    // メソッドのオーバーライド
    @Override
    public void makeSound() {
        System.out.println(name + "がチュンチュンと鳴きます");
    }

    // 鳥特有のメソッド
    public void fly() {
        System.out.println(name + "が飛んでいます");
    }
}

@Overrideアノテーション

@Overrideアノテーションは、メソッドが親クラスのメソッドを正しくオーバーライドしていることをコンパイラに伝えます。これにより、メソッド名のスペルミスや引数の間違いをコンパイル時に検出できます。

オーバーライドのルール

  • • メソッド名、引数の型と数、戻り値の型が親クラスと同じである必要があります
  • • アクセス修飾子は親クラスと同じか、より緩い必要があります（private → protected → public）
  • • 親クラスのprivateメソッドはオーバーライドできません
  • • staticメソッドはオーバーライドできません
  • • finalメソッドはオーバーライドできません

オーバーロードとオーバーライドの違い

比較項目 オーバーロード オーバーライド
概念 同じメソッド名で引数が異なる 親クラスのメソッドを再定義
発生場所 同じクラス内 親クラスと子クラス間
引数 必ず異なる 必ず同じ
戻り値 異なってもよい 同じ（または共変戻り値型）
決定時期 コンパイル時（静的バインディング） 実行時（動的バインディング）

9.4 ポリモーフィズム（多態性）

ポリモーフィズム（Polymorphism、多態性）は、同じ操作を異なる型のオブジェクトに対して実行できる仕組みです。親クラス型の変数で子クラスのオブジェクトを扱うことで、柔軟で拡張性の高いプログラムが書けます。

PolymorphismExample.java 
public class PolymorphismExample {
    public static void main(String[] args) {
        // 親クラス型の変数で子クラスのオブジェクトを参照
        Animal animal1 = new Dog("ポチ", 3, "柴犬");
        Animal animal2 = new Cat("タマ", 2);
        Animal animal3 = new Bird("ピーちゃん", 1);

        // 配列にまとめて管理できる
        Animal[] animals = {animal1, animal2, animal3};

        // 同じメソッド呼び出しでも、実際のオブジェクトに応じた動作
        for (Animal animal : animals) {
            animal.makeSound();  // 各動物に応じた鳴き声
        }
    }
}

// 実行結果:
// ポチが何か音を立てています（Dogクラスでオーバーライドされていない場合）
// タマがニャーと鳴きます
// ピーちゃんがチュンチュンと鳴きます

動的バインディング

ポリモーフィズムでは、どのメソッドが呼び出されるかはコンパイル時ではなく実行時に決定されます。これを動的バインディング（動的束縛）と呼びます。親クラス型の変数で子クラスのオブジェクトを参照している場合、実際のオブジェクトの型に基づいてメソッドが選択されます。

ポリモーフィズムの利点

  • 柔軟性: 親クラス型で統一的に扱えるため、新しい子クラスを追加しても既存のコードを変更する必要がありません
  • 保守性: 共通インターフェースを通じて操作できるため、コードの変更範囲が限定されます
  • 拡張性: 新しい動物クラスを追加しても、既存の処理ロジックが自動的に適用されます
AnimalManager.java 
// ポリモーフィズムを活用した動物管理クラス
public class AnimalManager {

    // 親クラス型の引数で、すべての子クラスを受け入れ可能
    public static void feedAnimal(Animal animal) {
        System.out.println("--- " + animal.name + "の世話 ---");
        animal.eat();
        animal.sleep();
        animal.makeSound();
        System.out.println();
    }

    public static void main(String[] args) {
        Dog dog = new Dog("ポチ", 3, "柴犬");
        Cat cat = new Cat("タマ", 2);
        Bird bird = new Bird("ピーちゃん", 1);

        // 同じメソッドで異なる動物を扱える
        feedAnimal(dog);
        feedAnimal(cat);
        feedAnimal(bird);
    }
}

9.5 抽象クラス

抽象クラス（Abstract Class）は、直接インスタンス化できないクラスで、子クラスで必ず実装すべきメソッドを定義するための仕組みです。abstractキーワードを使用して宣言します。

Shape.java 
// 抽象クラス - 図形の基底クラス
public abstract class Shape {
    protected String name;

    public Shape(String name) {
        this.name = name;
    }

    // 抽象メソッド - 子クラスで必ず実装が必要
    public abstract double calculateArea();
    public abstract double calculatePerimeter();

    // 具象メソッド - 通常のメソッドも定義できる
    public void displayInfo() {
        System.out.println("図形: " + name);
        System.out.println("面積: " + calculateArea());
        System.out.println("周囲の長さ: " + calculatePerimeter());
    }
}
Circle.java 
// 抽象クラスを継承した具象クラス
public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        super("円");
        this.radius = radius;
    }

    // 抽象メソッドを実装
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }

    @Override
    public double calculatePerimeter() {
        return 2 * Math.PI * radius;
    }
}
Rectangle.java 
public class Rectangle extends Shape {
    private double width;
    private double height;

    public Rectangle(double width, double height) {
        super("長方形");
        this.width = width;
        this.height = height;
    }

    @Override
    public double calculateArea() {
        return width * height;
    }

    @Override
    public double calculatePerimeter() {
        return 2 * (width + height);
    }
}

抽象クラスの特徴

  • • abstractキーワードで宣言される
  • • 抽象メソッド（実装を持たないメソッド）を含むことができる
  • • 通常のメソッド（具象メソッド）も含むことができる
  • • 直接インスタンス化できない（new Shape()はエラー）
  • • 子クラスは抽象メソッドをすべて実装する必要がある
  • • フィールドやコンストラクタを持つことができる

9.6 インターフェース

インターフェース（Interface）は、クラスが実装すべき機能（メソッド）を定義する「契約」のようなものです。Javaでは単一継承しかできませんが、インターフェースは複数実装できるため、より柔軟な設計が可能です。

Flyable.java 
// インターフェース - 「飛べる」という能力を定義
public interface Flyable {
    // すべてのメソッドは抽象メソッド（public abstractが省略されている）
    void fly();
    void land();

    // Java 8以降: defaultメソッド（デフォルト実装）を持てる
    default void takeOff() {
        System.out.println("離陸します");
    }
}
Swimmable.java 
// インターフェース - 「泳げる」という能力を定義
public interface Swimmable {
    void swim();
}
Duck.java 
// 複数のインターフェースを実装
public class Duck extends Animal implements Flyable, Swimmable {

    public Duck(String name, int age) {
        super(name, age);
    }

    @Override
    public void makeSound() {
        System.out.println(name + "がガーガーと鳴きます");
    }

    // Flyableインターフェースの実装
    @Override
    public void fly() {
        System.out.println(name + "が飛んでいます");
    }

    @Override
    public void land() {
        System.out.println(name + "が着地しました");
    }

    // Swimmableインターフェースの実装
    @Override
    public void swim() {
        System.out.println(name + "が泳いでいます");
    }
}
Penguin.java 
// ペンギン - 鳥だけど飛べない（Swimmableのみ実装）
public class Penguin extends Animal implements Swimmable {

    public Penguin(String name, int age) {
        super(name, age);
    }

    @Override
    public void makeSound() {
        System.out.println(name + "がクワッと鳴きます");
    }

    @Override
    public void swim() {
        System.out.println(name + "が水中を泳いでいます");
    }
}

抽象クラスとインターフェースの使い分け

比較項目 抽象クラス インターフェース
継承/実装 1つだけ継承可能 複数実装可能
フィールド 任意のフィールドを持てる 定数のみ（public static final）
メソッド 抽象・具象メソッド両方可 抽象メソッド、defaultメソッド
コンストラクタ 持てる 持てない
アクセス修飾子 任意 すべてpublic
使用場面 is-a関係、共通実装を提供 能力（〜できる）を表現

選択のガイドライン

抽象クラスを選ぶ場合:

  • • 共通の状態（フィールド）を持つ必要がある
  • • 共通の実装を提供したい
  • • 明確な「is-a関係」がある

インターフェースを選ぶ場合:

  • • 複数の機能を組み合わせたい
  • • 関係のないクラスに同じ能力を付与したい
  • • APIとして公開する契約を定義したい

実習 9-1: 動物園管理システム

継承とポリモーフィズムを活用して、動物園の動物を管理するシステムを作成しましょう。

実習 9-2: 図形計算システム

抽象クラスを使用して、様々な図形の面積と周囲の長さを計算するプログラムを作成しましょう。

要件

  1. 1 抽象クラスShapeを作成し、抽象メソッド（calculateArea, calculatePerimeter）を定義
  2. 2 Circle、Rectangle、Triangleクラスを作成し、それぞれの計算式を実装
  3. 3 Shape型の配列にすべての図形を格納し、ループで一括処理

  • • 長方形の面積: 幅 × 高さ
  • • 長方形の周囲: 2 × (幅 + 高さ)

理解度確認クイズ

  1. Q1. 継承が適切に使用されるべき場面を表す概念は何ですか？

    回答を表示

    is-a関係（〜は〜である）です。例えば、「犬は動物である（Dog is an Animal）」のように、子クラスが親クラスの一種であることを明確に表現できる場合に継承を使用します。has-a関係（〜を持つ）の場合は、継承ではなくコンポジション（フィールドとして持つ）を使用します。

  2. Q2. メソッドのオーバーライドとオーバーロードの違いを説明してください。

    回答を表示

    オーバーライド: 親クラスのメソッドを子クラスで再定義すること。メソッド名・引数・戻り値が同じで、動作を変更します。実行時に決定されます（動的バインディング）。

    オーバーロード: 同じクラス内で同じ名前のメソッドを引数の型や数を変えて複数定義すること。コンパイル時に決定されます（静的バインディング）。

  3. Q3. ポリモーフィズムの利点を3つ挙げてください。

    回答を表示

    1. 柔軟性: 親クラス型で統一的に扱えるため、新しい子クラスを追加しても既存のコードを変更する必要がありません

    2. 保守性: 共通インターフェースを通じて操作できるため、コードの変更範囲が限定され、保守が容易になります

    3. 拡張性: 新しいクラスを追加しても、既存の処理ロジックが自動的に適用されるため、拡張が簡単です

  4. Q4. 抽象クラスとインターフェースの主な違いは何ですか？

    回答を表示

    主な違い:
    • 抽象クラスは1つだけ継承できますが、インターフェースは複数実装できます
    • 抽象クラスは任意のフィールドを持てますが、インターフェースは定数のみです
    • 抽象クラスはコンストラクタを持てますが、インターフェースは持てません
    • 抽象クラスは具象メソッドを持てますが、インターフェースは基本的に抽象メソッドのみです（Java 8以降はdefaultメソッドも可）

    使い分け: is-a関係で共通実装を提供したい場合は抽象クラス、能力（〜できる）を表現したい場合はインターフェースを使用します。

  5. Q5. superキーワードの2つの主な用途は何ですか？

    回答を表示

    1. 親クラスのコンストラクタ呼び出し: super(引数)を子クラスのコンストラクタの最初の行で使用して、親クラスの初期化を行います

    2. 親クラスのメソッド呼び出し: super.メソッド名()で親クラスのメソッドを明示的に呼び出すことができます。オーバーライドした子クラスのメソッド内から親クラスのメソッドを実行したい場合などに使用します。
