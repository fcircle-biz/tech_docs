カプセル化とアクセス制御

オブジェクト指向の重要原則であるカプセル化について学びます。アクセス修飾子（public、private、protected）の使い分けや、getter/setterによる適切な情報隠蔽を理解します。

この章で学ぶこと

  • カプセル化の概念と必要性を理解する
  • アクセス修飾子（public、private、protected）の使い分け
  • getter/setterメソッドによる適切なデータアクセス
  • 情報隠蔽による安全で保守性の高い設計

9.1 カプセル化とは

カプセル化（Encapsulation）は、オブジェクト指向プログラミングの基本原則の一つで、データ（フィールド）とそれを操作するメソッドを一つのクラスにまとめ、外部から直接アクセスできないように保護することを指します。

カプセル化により、クラス内部の実装詳細を隠し、外部には必要なインターフェースだけを公開することで、以下のメリットが得られます。

  • データの保護: 外部からの不正な値の代入を防ぐ
  • 保守性の向上: 内部実装を変更しても外部に影響しない
  • データの整合性: 値の検証ロジックを一元管理できる
  • 可読性の向上: クラスの使い方が明確になる

カプセル化のたとえ

ATM（現金自動預払機）を思い浮かべてください。利用者は画面とボタンを通じてのみ操作でき、内部の現金や機械に直接触れることはできません。これがカプセル化です。公開インターフェース（画面・ボタン）を通じてのみ、内部データ（現金）にアクセスできます。

9.2 アクセス修飾子の種類と使い分け

Javaでは、アクセス修飾子（Access Modifier）を使ってクラス、フィールド、メソッドのアクセス範囲を制御します。主なアクセス修飾子は4つあります。

修飾子 同じクラス 同じパッケージ サブクラス すべて
public
protected
（修飾子なし）
private

各アクセス修飾子の説明

  • public - 公開

    どこからでもアクセス可能。外部に公開するメソッドやクラスに使用。

  • protected - 継承用保護

    同じパッケージとサブクラスからアクセス可能。継承を前提としたメソッドやフィールドに使用。

  • （修飾子なし）- パッケージプライベート

    同じパッケージ内からのみアクセス可能。パッケージ内部での使用に限定。

  • private - 非公開

    同じクラス内からのみアクセス可能。フィールドや内部ヘルパーメソッドに使用。

カプセル化のベストプラクティス

  • • フィールドは基本的にprivateにする
  • • 必要に応じてpublicなgetter/setterを提供する
  • • 公開する必要のないメソッドはprivateにする
  • • クラスはpublicか修飾子なしにする

9.3 カプセル化の実装例

カプセル化を適用しない場合と適用した場合の違いを比較してみましょう。

カプセル化なし（悪い例）

BankAccount.java（カプセル化なし） 
public class BankAccount {
    // フィールドが公開されている（危険！）
    public String accountNumber;
    public String owner;
    public int balance;
}
使用例（問題のあるコード） 
BankAccount account = new BankAccount();
account.accountNumber = "123-456";
account.owner = "山田太郎";
account.balance = 10000;

// 問題：外部から直接残高を変更できてしまう
account.balance = -5000;  // マイナス残高が設定できてしまう
account.balance = 999999999;  // 検証なしに巨額を設定できてしまう

カプセル化あり（良い例）

BankAccount.java（カプセル化適用） 
public class BankAccount {
    // フィールドをprivateで保護
    private String accountNumber;
    private String owner;
    private int balance;

    // コンストラクタ
    public BankAccount(String accountNumber, String owner) {
        this.accountNumber = accountNumber;
        this.owner = owner;
        this.balance = 0;  // 初期残高は0
    }

    // 残高を取得（getter）
    public int getBalance() {
        return balance;
    }

    // 入金メソッド（検証ロジック付き）
    public void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
        } else {
            System.out.println("入金額は正の値でなければなりません");
        }
    }

    // 出金メソッド（検証ロジック付き）
    public void withdraw(int amount) {
        if (amount <= 0) {
            System.out.println("出金額は正の値でなければなりません");
        } else if (amount > balance) {
            System.out.println("残高不足です");
        } else {
            balance -= amount;
        }
    }
}
使用例（安全なコード） 
BankAccount account = new BankAccount("123-456", "山田太郎");

// 安全な入金・出金
account.deposit(10000);
System.out.println("残高: " + account.getBalance());  // 10000

account.withdraw(3000);
System.out.println("残高: " + account.getBalance());  // 7000

// 不正な操作は防止される
account.deposit(-5000);  // エラーメッセージ表示、残高は変わらない
account.withdraw(10000);  // 残高不足エラー、残高は変わらない

9.4 getter/setterメソッド

getter（ゲッター）とsetter（セッター）は、privateフィールドにアクセスするための公開メソッドです。これらを使うことで、フィールドへのアクセスを制御できます。

getter/setterの命名規則

  • getter: get + フィールド名（先頭大文字） 例: getName()
  • setter: set + フィールド名（先頭大文字） 例: setName(String name)
  • boolean型のgetter: is + フィールド名（先頭大文字） 例: isActive()
Person.java（getter/setter実装例） 
public class Person {
    // privateフィールド
    private String name;
    private int age;
    private boolean active;

    // nameのgetter
    public String getName() {
        return name;
    }

    // nameのsetter（検証付き）
    public void setName(String name) {
        if (name != null && !name.isEmpty()) {
            this.name = name;
        } else {
            System.out.println("名前を空にはできません");
        }
    }

    // ageのgetter
    public int getAge() {
        return age;
    }

    // ageのsetter（検証付き）
    public void setAge(int age) {
        if (age >= 0 && age <= 150) {
            this.age = age;
        } else {
            System.out.println("年齢は0〜150の範囲で設定してください");
        }
    }

    // activeのgetter（boolean型はis〜）
    public boolean isActive() {
        return active;
    }

    // activeのsetter
    public void setActive(boolean active) {
        this.active = active;
    }
}

IDEの自動生成機能

IntelliJ IDEAやEclipseなどのIDEでは、フィールドを選択して右クリックメニューから「Generate → Getter and Setter」を選ぶことで、getter/setterを自動生成できます。手作業で書く必要はありません。

読み取り専用フィールド

setterを提供せず、getterのみを提供することで、読み取り専用のフィールドを実現できます。

読み取り専用フィールドの例 
public class Product {
    private String productId;  // 読み取り専用
    private String name;
    private int price;

    public Product(String productId, String name, int price) {
        this.productId = productId;
        this.name = name;
        this.price = price;
    }

    // productIdはgetterのみ（setterなし）
    public String getProductId() {
        return productId;
    }

    // nameはgetter/setter両方
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

9.5 カプセル化の実践例

カプセル化を活用した実践的なクラス設計を見てみましょう。

Employee.java（社員クラス） 
public class Employee {
    // privateフィールド
    private String employeeId;
    private String name;
    private int salary;
    private String department;

    // コンストラクタ
    public Employee(String employeeId, String name, int salary) {
        this.employeeId = employeeId;
        this.name = name;
        setSalary(salary);  // setterを使って検証
    }

    // getters
    public String getEmployeeId() {
        return employeeId;
    }

    public String getName() {
        return name;
    }

    public int getSalary() {
        return salary;
    }

    public String getDepartment() {
        return department;
    }

    // setters（検証ロジック付き）
    public void setName(String name) {
        if (name != null && name.length() >= 2) {
            this.name = name;
        }
    }

    public void setSalary(int salary) {
        if (salary >= 0) {
            this.salary = salary;
        }
    }

    public void setDepartment(String department) {
        this.department = department;
    }

    // ビジネスロジック
    public void raiseSalary(int amount) {
        if (amount > 0) {
            this.salary += amount;
            System.out.println("昇給しました: +" + amount + "円");
        }
    }

    public void displayInfo() {
        System.out.println("社員ID: " + employeeId);
        System.out.println("名前: " + name);
        System.out.println("給与: " + salary + "円");
        System.out.println("部署: " + department);
    }
}
使用例 
public class Main {
    public static void main(String[] args) {
        // 社員インスタンス作成
        Employee emp = new Employee("E001", "佐藤花子", 300000);
        emp.setDepartment("営業部");

        // 情報表示
        emp.displayInfo();

        // 昇給
        emp.raiseSalary(50000);

        // 更新後の給与確認
        System.out.println("新しい給与: " + emp.getSalary());
    }
}

実習 9-1: 商品管理クラスを作ろう

カプセル化を適用した商品管理クラスを作成し、データの整合性を保つ仕組みを実装してみましょう。

課題

以下の仕様でProductクラスを作成してください。

  • privateフィールド: productId（商品ID）、name（商品名）、price（価格）、stock（在庫数）
  • priceは0以上の値のみ設定可能
  • stockは0以上の値のみ設定可能
  • addStock(int amount)メソッド: 在庫を追加（正の値のみ）
  • sellProduct(int quantity)メソッド: 商品を販売（在庫チェック付き）

実装例

public class Product {
    private String productId;
    private String name;
    private int price;
    private int stock;

    public Product(String productId, String name, int price, int stock) {
        this.productId = productId;
        this.name = name;
        setPrice(price);
        setStock(stock);
    }

    // getters
    public String getProductId() { return productId; }
    public String getName() { return name; }
    public int getPrice() { return price; }
    public int getStock() { return stock; }

    // setters
    public void setPrice(int price) {
        if (price >= 0) this.price = price;
    }

    public void setStock(int stock) {
        if (stock >= 0) this.stock = stock;
    }

    // 在庫追加
    public void addStock(int amount) {
        if (amount > 0) {
            this.stock += amount;
            System.out.println(amount + "個追加しました");
        }
    }

    // 商品販売
    public void sellProduct(int quantity) {
        if (quantity <= 0) {
            System.out.println("販売数は正の値でなければなりません");
        } else if (quantity > stock) {
            System.out.println("在庫不足です");
        } else {
            stock -= quantity;
            System.out.println(quantity + "個販売しました");
        }
    }
}

理解度確認クイズ

  1. Q1. カプセル化の主な目的は何ですか？

    回答を表示

    データ（フィールド）を外部から直接アクセスできないように保護し、公開メソッド経由でのみアクセスできるようにすることで、データの整合性を保ち、保守性を向上させることが目的です。

  2. Q2. フィールドに通常使用すべきアクセス修飾子はどれですか？

    回答を表示

    privateです。フィールドはprivateで保護し、必要に応じてpublicなgetter/setterメソッドを提供することがベストプラクティスです。

  3. Q3. setterメソッド内で検証ロジックを実装する利点は何ですか？

    回答を表示

    不正な値（マイナスの金額、空の文字列など）がフィールドに設定されるのを防ぎ、データの整合性を保つことができます。すべてのアクセスがsetterを通るため、検証ロジックを一箇所で管理できます。

  4. Q4. boolean型のフィールドのgetterメソッド名の命名規則は何ですか？

    回答を表示

    is + フィールド名（先頭大文字）です。例: boolean activeフィールドならisActive()というメソッド名になります。

この章のまとめ

  • カプセル化はデータと処理をまとめて外部から保護するオブジェクト指向の基本原則
  • アクセス修飾子（public、protected、修飾子なし、private）でアクセス範囲を制御
  • フィールドはprivateにし、getter/setterメソッドで公開することが基本
  • setterメソッドで検証ロジックを実装し、データの整合性を保つ
  • カプセル化により保守性・安全性・可読性が向上する
