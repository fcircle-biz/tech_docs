データを守る - カプセル化とアクセス制御

オブジェクトの中身を外部から守る「カプセル化」と、アクセス修飾子（public、private）について学びます。ATMの仕組みをたとえに、なぜ情報を隠すことが大切かを理解しましょう。

この章で学ぶこと

  • カプセル化とは何か、なぜ必要なのかを理解する
  • アクセス修飾子（public、private）の使い方と違いを学ぶ
  • ゲッターとセッターを使って安全にデータを操作する方法を習得する

9.1 カプセル化（Encapsulation）って何だろう？

前の章で、クラスの中にはフィールド（データを保存する変数）とメソッド（動作・機能）があることを学びました。 今回は、これらを「外部から守る」ための重要な考え方であるカプセル化について学びます。

カプセル化とは？

オブジェクトの内部データ（フィールド）を外部から直接触らせず、 メソッドを通じてのみアクセスできるようにする仕組みのことです。 カプセル（capsule：カプセル薬）のように、中身を包んで守るイメージです。

なぜカプセル化が必要なの？

たとえ話: ATMの仕組み

❌ カプセル化なし（危険）

ATMの中身が丸見えで、お金を入れている金庫に直接触れる状態。

  • ・誰でも勝手にお金を増減できる
  • ・不正な金額が設定される
  • ・セキュリティが保てない

✅ カプセル化あり（安全）

ATMは「引き出す」「預ける」というボタン（メソッド）だけを公開し、中身の金庫には直接触れない。

  • ・正しい手順でしか操作できない
  • ・不正な金額は設定できない
  • ・安全に管理できる

プログラムでも同じです。クラスのフィールド（データ）を外部から直接変更できてしまうと、 思わぬ不正なデータが入り込んでバグの原因になります。 そこで、フィールドは隠して（private）、メソッドを通じてのみアクセスできるようにする（public）のがカプセル化です。

9.2 アクセス修飾子 - public と private

カプセル化を実現するために、Javaではアクセス修飾子（Access Modifier）というキーワードを使います。 これは「誰がどこからアクセスできるか」を制御するための仕組みです。

修飾子 意味 アクセス範囲 使い分け
public 公開 どこからでもアクセス可能 外部に公開するメソッド
private 非公開 同じクラス内からのみアクセス可能 守りたいフィールド
protected 保護された 同じパッケージ+継承先 継承で使う（後の章で学習）
（なし） パッケージ 同じパッケージ内のみ あまり使わない

重要な原則

  • ・フィールドは基本的にprivateにして外部から守る
  • ・メソッドは必要に応じてpublicにして外部に公開する
  • ・これがカプセル化の基本パターンです

9.3 カプセル化を実践してみよう

具体的なコード例を見てみましょう。まずはカプセル化していない（良くない）例から。

❌ カプセル化していない例（良くない）

BadPerson.java 
public class BadPerson {
    public String name;  // publicで外部から直接アクセス可能
    public int age;      // publicで外部から直接アクセス可能
}
Main.java（使う側） 
public class Main {
    public static void main(String[] args) {
        BadPerson person = new BadPerson();
        person.name = "田中太郎";  // 直接変更できる
        person.age = 25;

        // 問題: 不正な値も入れられてしまう
        person.age = -10;  // マイナスの年齢！？
        System.out.println(person.name + "は" + person.age + "歳");
    }
}

何が問題？

  • ・外部から直接フィールドを変更できるため、不正な値（マイナスの年齢など）が入る
  • ・値のチェック（検証）ができない
  • ・後からフィールド名を変えると、使っている全てのコードを修正する必要がある

✅ カプセル化した例（正しい）

GoodPerson.java 
public class GoodPerson {
    private String name;  // privateで外部から直接アクセス不可
    private int age;      // privateで外部から直接アクセス不可

    // ゲッター: nameを取得するメソッド
    public String getName() {
        return name;
    }

    // セッター: nameを設定するメソッド
    public void setName(String name) {
        this.name = name;
    }

    // ゲッター: ageを取得するメソッド
    public int getAge() {
        return age;
    }

    // セッター: ageを設定するメソッド（値チェック付き）
    public void setAge(int age) {
        if (age < 0) {
            System.out.println("年齢は0以上である必要があります");
            return;  // 不正な値は設定しない
        }
        this.age = age;
    }
}
Main.java（使う側） 
public class Main {
    public static void main(String[] args) {
        GoodPerson person = new GoodPerson();

        // メソッド経由で設定
        person.setName("田中太郎");
        person.setAge(25);

        // メソッド経由で取得
        System.out.println(person.getName() + "は" + person.getAge() + "歳");

        // 不正な値を設定しようとすると...
        person.setAge(-10);  // エラーメッセージが出て設定されない
    }
}

良いポイント

  • ・フィールドはprivateなので外部から直接触れない
  • ・setAgeメソッドで値のチェック（検証）ができる
  • ・不正な値は設定されないため、安全

9.4 ゲッターとセッター（Getter/Setter）

カプセル化でよく使われるのがゲッター（Getter）とセッター（Setter）です。

ゲッター（Getter）

privateフィールドの値を取得（get）するためのメソッド。 通常、get + フィールド名という命名規則に従います。

セッター（Setter）

privateフィールドに値を設定（set）するためのメソッド。 通常、set + フィールド名という命名規則に従います。

命名規則のパターン

フィールド名 ゲッター名 セッター名
name getName() setName(String name)
age getAge() setAge(int age)
price getPrice() setPrice(int price)

注意

必ずしもすべてのフィールドにゲッター/セッターを付ける必要はありません。
必要なものだけ公開し、不要なものは作らないことで、より安全な設計になります。

実習 9-1: BankAccountクラスでカプセル化を実践

銀行口座を表すBankAccountクラスを作り、カプセル化を実践してみましょう。

手順

  1. 1 BankAccount.javaファイルを作成
  2. 2 以下のコードを入力：
BankAccount.java 
public class BankAccount {
    private String accountNumber;  // 口座番号（非公開）
    private int balance;           // 残高（非公開）

    // コンストラクタ
    public BankAccount(String accountNumber) {
        this.accountNumber = accountNumber;
        this.balance = 0;  // 最初は残高0
    }

    // 残高を取得（ゲッター）
    public int getBalance() {
        return balance;
    }

    // 入金メソッド
    public void deposit(int amount) {
        if (amount <= 0) {
            System.out.println("入金額は0より大きい必要があります");
            return;
        }
        balance += amount;
        System.out.println(amount + "円入金しました");
    }

    // 出金メソッド
    public void withdraw(int amount) {
        if (amount <= 0) {
            System.out.println("出金額は0より大きい必要があります");
            return;
        }
        if (amount > balance) {
            System.out.println("残高不足です");
            return;
        }
        balance -= amount;
        System.out.println(amount + "円出金しました");
    }
}
  1. 3 Main.javaで使ってみる：
Main.java 
public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount("123-456");

        // 入金
        account.deposit(10000);
        System.out.println("残高: " + account.getBalance() + "円");

        // 出金
        account.withdraw(3000);
        System.out.println("残高: " + account.getBalance() + "円");

        // 不正な出金を試みる
        account.withdraw(20000);  // 残高不足
    }
}

期待される結果

10000円入金しました
残高: 10000円
3000円出金しました
残高: 7000円
残高不足です

balanceフィールドはprivateなので外部から直接変更できず、deposit/withdrawメソッドを通じてのみ操作できます。これにより安全に残高管理ができます。

9.5 カプセル化のメリット

1. データの保護

不正な値が設定されるのを防ぎ、オブジェクトの状態を常に正しく保てます。

2. 値の検証

セッターメソッド内でチェック処理を書けば、不正な値を弾けます。

3. 保守性の向上

フィールド名や内部実装を変更しても、メソッド名を変えなければ外部コードに影響しません。

4. 情報隠蔽

クラスの内部構造を隠すことで、使う側はメソッドだけを気にすればよくなります。

理解度確認クイズ

  1. Q1. カプセル化とは何ですか？

    回答を表示

    オブジェクトの内部データ（フィールド）を外部から直接触らせず、 メソッドを通じてのみアクセスできるようにする仕組みのことです。 これにより、データを守り、不正な値が設定されるのを防ぎます。

  2. Q2. privateとpublicの違いは何ですか？

    回答を表示

    private: 同じクラス内からのみアクセス可能。外部からは見えない。
    public: どこからでもアクセス可能。外部に公開する。
    基本的に、フィールドはprivate、公開するメソッドはpublicにします。

  3. Q3. ゲッターとセッターの役割は何ですか？

    回答を表示

    ゲッター: privateフィールドの値を取得（get）するためのpublicメソッド。
    セッター: privateフィールドに値を設定（set）するためのpublicメソッド。
    セッター内で値のチェック（検証）を行うことで、不正な値を弾けます。

  4. Q4. なぜフィールドを直接publicにせず、メソッド経由でアクセスするのですか？

    回答を表示

    メソッド経由にすることで、以下のメリットがあります：
    ・不正な値が設定されるのを防げる（値の検証ができる）
    ・内部実装を変更しても外部コードに影響しない
    ・データの安全性が保たれる

9.6 この章のまとめ

  • カプセル化は、オブジェクトの内部データを外部から守る重要な概念。
  • privateは同じクラス内からのみアクセス可能、publicはどこからでもアクセス可能。
  • フィールドはprivate、公開するメソッドはpublicにするのが基本。
  • ゲッター（getXxx）でフィールドを取得、セッター（setXxx）でフィールドを設定する。
  • セッター内で値の検証を行うことで、不正な値を防げる。
  • カプセル化により、データの保護・保守性の向上・情報隠蔽が実現できる。
