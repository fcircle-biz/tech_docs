1.1 Javaとは何か

Java（ジャバ）は、1995年にサン・マイクロシステムズ（現Oracle）によって開発されたオブジェクト指向プログラミング言語です。現在では世界中で最も広く使用されているプログラミング言語の一つとなっています。

なぜJavaが人気なのでしょうか？

Javaの人気の秘密は、その設計哲学にあります。「Write Once, Run Anywhere（一度書けばどこでも動く）」というスローガンのとおり、Javaで書かれたプログラムは、異なるオペレーティングシステム（Windows、Mac、Linux など）で同じように動作します。

Javaの主要な特徴

  • プラットフォーム独立性：一度コンパイルしたコードは、Java仮想マシン（JVM）がインストールされた任意の環境で実行可能
  • オブジェクト指向：現実世界の概念をプログラムで表現しやすく、大規模なソフトウェア開発に適している
  • 自動メモリ管理：ガベージコレクションにより、メモリリークを防ぎプログラムの安定性を向上
  • 強力な標準ライブラリ：データベース接続、ネットワーク通信、ファイル操作など幅広い機能を標準で提供
  • セキュリティ：型安全性やセキュリティマネージャーによる安全なプログラム実行

1.2 Javaの実行環境について

Javaプログラムが動作する仕組みを理解することは非常に重要です。他の多くのプログラミング言語とは異なる特徴的な実行方式を採用しているからです。

従来のプログラム実行方式との違い

従来のプログラミング言語（C言語など）では、ソースコードを直接機械語（0と1の組み合わせ）に変換（コンパイル）し、特定のオペレーティングシステム用の実行ファイルを作成します。しかし、この方式では異なるOSで動かすためには、それぞれのOS用に個別にコンパイルし直す必要がありました。

graph LR A["Javaソースコード
(.java)"] --> B["Javaコンパイラ
(javac)"] B --> C["バイトコード
(.class)"] C --> D["Java仮想マシン
(JVM)"] D --> E["実行結果"] F["JVM for Windows"] --> D G["JVM for Mac"] --> D H["JVM for Linux"] --> D

Java仮想マシン（JVM）の役割

Javaでは、ソースコードを中間形式である「バイトコード」に変換します。このバイトコードを実際に実行するのが Java仮想マシン（JVM: Java Virtual Machine）です。JVMは各OS用に用意されており、同じバイトコードをどのOSでも実行できるようにしています。

重要なポイント

Java仮想マシンは「仮想的な」コンピュータです。実際のコンピュータ上で動作する、Java専用の実行環境と考えることができます。これにより、Javaプログラムは「JVMという共通の環境」で動作するため、実際のOS（Windows、Mac、Linux）に依存せずに同じように実行できるのです。

1.3 JDKのインストールと環境設定

Javaプログラムを開発するためには、JDK（Java Development Kit）が必要です。JDKには、Javaプログラムを書く、コンパイルする、実行するために必要なすべてのツールが含まれています。

JDK、JRE、JVMの関係を理解しよう

Java関連の用語はよく混同されますが、それぞれ異なる役割を持っています：

graph TB A[JDK - Java開発キット] --> B[開発ツール群] A --> C[JRE - Java実行環境] C --> D[JVM - Java仮想マシン] C --> E[標準ライブラリ] B --> F[javac - コンパイラ] B --> G[java - 実行ツール] B --> H[javadoc - ドキュメント生成]
  • JVM（Java Virtual Machine）：Javaバイトコードを実行する仮想マシン
  • JRE（Java Runtime Environment）：JVM + 標準ライブラリ（Javaプログラムを実行するのに必要）
  • JDK（Java Development Kit）：JRE + 開発ツール（Javaプログラムを開発するのに必要）
実習 1-1: JDKの確認とインストール

まず、コンピュータにJavaがインストールされているかを確認し、必要に応じてインストールを行います。

手順
  1. コマンドプロンプト（Windows）またはターミナル（Mac/Linux）を開く
  2. 以下のコマンドを入力してJavaの確認を行う
Javaのバージョン確認コマンド
# Javaランタイム（実行環境）のバージョン確認
java -version

# Java開発キット（コンパイラ）のバージョン確認  
javac -version
期待される結果

正常にJDKがインストールされている場合、以下のような出力が表示されます：

java version "11.0.12" 2021-07-20 LTS
Java(TM) SE Runtime Environment (build 11.0.12+8-LTS-237)
Java HotSpot(TM) 64-Bit Server VM (build 11.0.12+8-LTS-237, mixed mode)

javac 11.0.12
コマンドが認識されない場合

「'java' は内部コマンドまたは外部コマンド...として認識されていません」のようなエラーが出る場合は、JDKがインストールされていないか、環境変数（PATH）が正しく設定されていません。Oracle公式サイトから最新のJDKをダウンロードしてインストールしてください。

1.4 初めてのJavaプログラム

プログラミング学習の第一歩は、必ず「Hello World」プログラムから始まります。これは、画面に「Hello World」という文字列を表示するだけの簡単なプログラムですが、プログラミング言語の基本的な構造と実行手順を理解するのに最適です。

なぜ「Hello World」から始めるのか？

「Hello World」プログラムを作成することで、以下の重要な要素を確認できます：

  • 開発環境が正しくセットアップされている
  • ソースコードの作成方法
  • コンパイルの手順
  • プログラムの実行方法
  • 基本的なプログラム構造
実習 1-2: 初めてのJavaプログラム作成

テキストエディタを使用してJavaプログラムを作成し、コンパイルと実行を行います。

手順
  1. デスクトップに「JavaStudy」というフォルダを作成
  2. テキストエディタ（メモ帳、Visual Studio Code等）を開く
  3. 以下のコードを正確に入力
  4. ファイル名を「HelloWorld.java」として保存（重要：拡張子は.java）
HelloWorld.javaのソースコード
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello World!");
        System.out.println("私の初めてのJavaプログラム");
    }
}
コンパイルと実行
  1. コマンドプロンプト/ターミナルを開く
  2. 「JavaStudy」フォルダに移動
  3. コンパイルコマンドを実行
  4. 実行コマンドを実行
# フォルダに移動（Windowsの例）
cd C:\Users\（ユーザー名）\Desktop\JavaStudy

# Javaファイルをコンパイル
javac HelloWorld.java

# コンパイル済みプログラムを実行
java HelloWorld
期待される結果
Hello World!
私の初めてのJavaプログラム

プログラムの構造を理解しよう

作成したプログラムの各部分にはそれぞれ重要な意味があります：

public class HelloWorld {           // クラス宣言
    public static void main(String[] args) {  // メインメソッド
        System.out.println("Hello World!");   // 出力処理
    }
}
  • public class HelloWorld：クラスの宣言。Javaではすべてのコードがクラスの中に書かれる
  • public static void main(String[] args)：プログラムの実行開始点（エントリーポイント）
  • System.out.println()：画面に文字列を表示するメソッド
  • { }：波括弧（ブレース）は処理のまとまりを表す
重要な規則
  • クラス名とファイル名は必ず同じにする（HelloWorld.java → class HelloWorld）
  • 大文字と小文字を正確に区別する（Java は大文字小文字を区別する言語）
  • セミコロン（;）で文の終わりを示す
  • 文字列はダブルクォート（""）で囲む

1.5 統合開発環境（IDE）の紹介

これまでテキストエディタとコマンドラインでプログラムを作成しましたが、実際の開発では統合開発環境（IDE: Integrated Development Environment）を使用することが一般的です。

IDEを使うメリット

IDEは「プログラム作成のための専用ソフトウェア」で、以下のような機能を提供します：

  • コード補完：入力途中でメソッド名や変数名を自動提案
  • 構文エラー検出：コンパイル前にエラーを指摘
  • デバッグ機能：プログラムの動作を詳細に調査
  • プロジェクト管理：複数のファイルを効率的に管理
  • 実行環境統合：コンパイルと実行をボタン一つで実行

主要なJava IDE

Eclipse

無料で高機能。豊富なプラグインが利用可能で、企業でも広く使用されている。

IntelliJ IDEA

非常に高機能で使いやすい。無料のCommunity版と有料のUltimate版がある。

Visual Studio Code

軽量で拡張性が高い。Java拡張パックをインストールすることでJava開発に対応。

理解度確認クイズ
  1. Javaの「Write Once, Run Anywhere」という特徴を実現している仕組みは何ですか？
    • a) 高速なコンパイラ
    • b) Java仮想マシン（JVM）
    • c) 統合開発環境（IDE）
    • d) オブジェクト指向プログラミング
  2. Javaソースファイルの拡張子は何ですか？
    • a) .class
    • b) .jar
    • c) .java
    • d) .jdk
  3. JDK、JRE、JVMの関係について正しい説明はどれですか？
    • a) JDK ⊃ JRE ⊃ JVM
    • b) JVM ⊃ JRE ⊃ JDK
    • c) JRE ⊃ JDK ⊃ JVM
    • d) これらは独立したツール
  4. public static void main(String[] args) の役割は何ですか？
    • a) クラスを定義する
    • b) プログラムの実行開始点を示す
    • c) 変数を宣言する
    • d) コメントを書く
  5. 「HelloWorld.java」というファイル名でクラスを作成する時、クラス名として正しいものはどれですか？
    • a) helloworld
    • b) HELLOWORLD
    • c) HelloWorld
    • d) hello_world
解答を表示

解答:

  1. b) Java仮想マシン（JVM） - JVMがバイトコードを各OS上で実行
  2. c) .java - Javaソースファイルの標準拡張子
  3. a) JDK ⊃ JRE ⊃ JVM - JDKはJREを含み、JREはJVMを含む
  4. b) プログラムの実行開始点を示す - メインメソッドはエントリーポイント
  5. c) HelloWorld - ファイル名とクラス名は大文字小文字も含めて完全に一致する必要がある
