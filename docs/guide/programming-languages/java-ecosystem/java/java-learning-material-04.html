第4章

条件分岐 - プログラムに判断させる

プログラムが状況に応じて異なる動作をする「条件分岐」の仕組みを学びます。

この章で学ぶこと

  • if文を使った基本的な条件分岐ができる
  • if-else文、if-else if-else文で複数の条件を処理できる
  • switch文の使い方と適用場面を理解する
  • 条件分岐のネスト（入れ子）を理解する

4.1 条件分岐とは何か

条件分岐とは、プログラムが特定の条件に応じて異なる処理を実行する仕組みです。日常生活でも「もし雨が降っていたら傘を持つ」「そうでなければ持たない」というように、状況に応じて行動を変えます。プログラムでも同様の判断をさせることができます。

条件分岐が必要な理由

条件分岐がないと、プログラムは常に同じ処理しか実行できません。条件分岐を使うことで：

  • ユーザーの入力に応じた処理ができる
  • データの内容によって処理を変えられる
  • エラーが発生した時の対応ができる
  • 複雑なビジネスロジックを実装できる

4.2 if文の基本 - 単純な条件判断

if文は、最も基本的な条件分岐の構文です。「もし〜ならば、〜を実行する」という判断をプログラムに与えます。

if文の構文

if (条件式) {
    // 条件がtrueの時に実行される処理
}
BasicIfExample.java 
public class BasicIfExample {
    public static void main(String[] args) {
        int age = 20;

        // 基本的なif文
        if (age >= 18) {
            System.out.println("あなたは成人です");
        }

        // 条件が満たされない場合は何も実行されない
        if (age < 18) {
            System.out.println("あなたは未成年です");  // 実行されない
        }

        // 複数の文を実行できる
        int score = 85;
        if (score >= 80) {
            System.out.println("優秀な成績です！");
            System.out.println("あなたの得点: " + score);
        }
    }
}

ブロック { } の重要性

if文の後には必ずブロック{ }を付けることをお勧めします。
ブロックを省略すると、if文の直後の1文だけが条件付きで実行されますが、意図しないバグの原因になります。

// 悪い例（ブロックなし）
if (age >= 18)
    System.out.println("成人です");
    System.out.println("投票できます"); // 常に実行される！

4.3 if-else文 - 2つの選択肢

if-else文は、条件がtrueの場合とfalseの場合で異なる処理を実行します。「もし〜ならば〜、そうでなければ〜」という判断です。

if-else文の構文

if (条件式) {
    // 条件がtrueの時の処理
} else {
    // 条件がfalseの時の処理
}
IfElseExample.java 
public class IfElseExample {
    public static void main(String[] args) {
        int age = 15;

        // if-else文の基本
        if (age >= 18) {
            System.out.println("あなたは成人です");
        } else {
            System.out.println("あなたは未成年です");
        }

        // 偶数・奇数の判定
        int number = 7;
        if (number % 2 == 0) {
            System.out.println(number + "は偶数です");
        } else {
            System.out.println(number + "は奇数です");
        }

        // 合格・不合格の判定
        int score = 55;
        if (score >= 60) {
            System.out.println("合格です！");
        } else {
            System.out.println("不合格です。次回頑張りましょう。");
        }
    }
}

4.4 if-else if-else文 - 複数の条件判断

3つ以上の選択肢がある場合は、else if を使って複数の条件を順番にチェックできます。上から順番に条件が評価され、最初にtrueになった条件のブロックだけが実行されます。

if-else if-else文の構文

if (条件式1) {
    // 条件式1がtrueの時の処理
} else if (条件式2) {
    // 条件式2がtrueの時の処理
} else {
    // すべての条件がfalseの時の処理
}
GradeEvaluation.java 
public class GradeEvaluation {
    public static void main(String[] args) {
        int score = 75;

        // 成績の評価（A/B/C/D/F）
        if (score >= 90) {
            System.out.println("評価: A（優秀）");
        } else if (score >= 80) {
            System.out.println("評価: B（良好）");
        } else if (score >= 70) {
            System.out.println("評価: C（普通）");
        } else if (score >= 60) {
            System.out.println("評価: D（合格）");
        } else {
            System.out.println("評価: F（不合格）");
        }

        // 年齢区分による料金設定
        int age = 25;
        int price;

        if (age < 6) {
            price = 0;  // 幼児無料
        } else if (age < 18) {
            price = 500;  // 子供料金
        } else if (age < 65) {
            price = 1000;  // 大人料金
        } else {
            price = 700;  // シニア料金
        }

        System.out.println("年齢: " + age + "歳");
        System.out.println("料金: " + price + "円");
    }
}

条件の順序が重要

else if文は上から順番に評価されるため、条件の順序が大切です。
より厳しい条件を先に書くのが一般的です。例えば、score >= 90 を先に、score >= 60 を後に書きます。

4.5 switch文 - 値による分岐

switch文は、1つの変数の値に応じて複数の処理を切り替える際に便利です。特定の値と完全に一致するかを判定します。

switch文の構文

switch (変数) {
    case 値1:
        // 値1と一致した時の処理
        break;
    case 値2:
        // 値2と一致した時の処理
        break;
    default:
        // どのcaseにも一致しない時の処理
}
SwitchExample.java 
public class SwitchExample {
    public static void main(String[] args) {
        // 曜日の表示
        int dayOfWeek = 3;  // 1=月曜、2=火曜、...、7=日曜

        switch (dayOfWeek) {
            case 1:
                System.out.println("月曜日");
                break;
            case 2:
                System.out.println("火曜日");
                break;
            case 3:
                System.out.println("水曜日");
                break;
            case 4:
                System.out.println("木曜日");
                break;
            case 5:
                System.out.println("金曜日");
                break;
            case 6:
                System.out.println("土曜日");
                break;
            case 7:
                System.out.println("日曜日");
                break;
            default:
                System.out.println("不正な値です");
        }

        // 文字による判定
        char grade = 'B';

        switch (grade) {
            case 'A':
                System.out.println("優秀です！");
                break;
            case 'B':
                System.out.println("良好です");
                break;
            case 'C':
                System.out.println("普通です");
                break;
            case 'D':
                System.out.println("もう少し頑張りましょう");
                break;
            case 'F':
                System.out.println("不合格です");
                break;
            default:
                System.out.println("不正な評価です");
        }
    }
}

breakを忘れずに！

各caseの最後にはbreak;を書くのを忘れないでください。
breakがないと、次のcaseの処理も続けて実行されてしまいます（フォールスルー）。

これは意図的に使うこともありますが、初心者のうちは必ずbreakを付けましょう。

switch式（Java 14以降）

Java 14以降では、より簡潔なswitch式も使えます。

SwitchExpressionExample.java 
public class SwitchExpressionExample {
    public static void main(String[] args) {
        int dayOfWeek = 3;

        // 新しいswitch式（Java 14以降）
        String dayName = switch (dayOfWeek) {
            case 1 -> "月曜日";
            case 2 -> "火曜日";
            case 3 -> "水曜日";
            case 4 -> "木曜日";
            case 5 -> "金曜日";
            case 6 -> "土曜日";
            case 7 -> "日曜日";
            default -> "不正な値";
        };

        System.out.println(dayName);
    }
}

4.6 条件分岐のネスト（入れ子）

if文の中にさらにif文を書くことを「ネスト（入れ子）」といいます。複雑な条件判断が必要な時に使いますが、ネストが深くなりすぎると読みにくくなるので注意が必要です。

NestedIfExample.java 
public class NestedIfExample {
    public static void main(String[] args) {
        int age = 25;
        boolean hasLicense = true;

        // ネストしたif文
        if (age >= 18) {
            System.out.println("年齢は18歳以上です");

            if (hasLicense) {
                System.out.println("運転免許を持っています");
                System.out.println("→ 車を運転できます");
            } else {
                System.out.println("運転免許を持っていません");
                System.out.println("→ 車を運転できません");
            }
        } else {
            System.out.println("年齢が18歳未満です");
            System.out.println("→ 車を運転できません");
        }

        // より読みやすい書き方（論理演算子を使う）
        if (age >= 18 && hasLicense) {
            System.out.println("車を運転できます");
        } else {
            System.out.println("車を運転できません");
        }
    }
}

読みやすいコードを書くヒント

  • • ネストは2〜3段階までに抑える
  • • 論理演算子（&&、||）で複数条件をまとめられないか検討する
  • • 条件が複雑になったらメソッドに分割する（後の章で学習）
  • • インデント（字下げ）を正しく使って構造を明確にする

実習 4-1: 成績評価プログラム

テストの点数を入力し、A〜Fの5段階で成績を評価するプログラムを作成しましょう。

評価基準

  • 90点以上: A（優秀）
  • 80点以上: B（良好）
  • 70点以上: C（普通）
  • 60点以上: D（合格）
  • 60点未満: F（不合格）

実行例

ScoreGrading.java 
public class ScoreGrading {
    public static void main(String[] args) {
        int score = 85;

        System.out.println("あなたの点数: " + score);

        if (score >= 90) {
            System.out.println("評価: A（優秀）");
        } else if (score >= 80) {
            System.out.println("評価: B（良好）");
        } else if (score >= 70) {
            System.out.println("評価: C（普通）");
        } else if (score >= 60) {
            System.out.println("評価: D（合格）");
        } else {
            System.out.println("評価: F（不合格）");
        }
    }
}

実習 4-2: じゃんけんゲーム判定

プレイヤーとコンピュータの手を比較して、じゃんけんの勝敗を判定するプログラムを作成しましょう。

前提

  • 0: グー、1: チョキ、2: パー

実行例

RockPaperScissors.java 
public class RockPaperScissors {
    public static void main(String[] args) {
        int player = 2;    // プレイヤーの手（2=パー）
        int computer = 0;  // コンピュータの手（0=グー）

        System.out.println("プレイヤー: " + getHandName(player));
        System.out.println("コンピュータ: " + getHandName(computer));

        // 勝敗判定
        if (player == computer) {
            System.out.println("結果: 引き分け");
        } else if ((player == 0 && computer == 1) ||
                   (player == 1 && computer == 2) ||
                   (player == 2 && computer == 0)) {
            System.out.println("結果: プレイヤーの勝ち！");
        } else {
            System.out.println("結果: コンピュータの勝ち");
        }
    }

    // 手の名前を返すヘルパーメソッド
    public static String getHandName(int hand) {
        switch (hand) {
            case 0: return "グー";
            case 1: return "チョキ";
            case 2: return "パー";
            default: return "不正な値";
        }
    }
}

理解度確認クイズ

  1. Q1. 次のコードで出力される内容は？
    int x = 10; if (x > 5) { System.out.println("大きい"); }

    回答を表示
    答え: 大きい
    x（10）は5より大きいため、条件がtrueになり「大きい」と出力されます。

  2. Q2. if-else if-else文で、複数の条件がtrueの場合、どの処理が実行されますか？

    回答を表示
    答え: 最初にtrueになった条件の処理だけが実行されます
    上から順番に評価され、最初にtrueになった時点で残りの条件は評価されません。

  3. Q3. switch文のbreakを忘れるとどうなりますか？

    回答を表示
    答え: 次のcaseの処理も続けて実行されてしまいます（フォールスルー）
    意図しない動作になるため、基本的には各caseの最後にbreakを書きます。

  4. Q4. if-else文とswitch文、どちらを使うべきでしょうか？

    回答を表示
    答え: 状況によって使い分けます
    • 特定の値との完全一致を判定 → switch
    • 範囲や複雑な条件の判定 → if-else
    • 2〜3個の選択肢 → if-else
    • 多数の選択肢（値ベース） → switch

  5. Q5. ネスト（入れ子）が深くなりすぎた時の対処法は？

    回答を表示
    いくつかの対処法があります：
    • 論理演算子（&&、||）を使って条件をまとめる
    • 早期リターン（次の章で学習）を使う
    • 処理をメソッドに分割する
    • ネストは2〜3段階までに抑えるのが理想

この章のまとめ

  • if文: 条件がtrueの時だけ処理を実行します
  • if-else文: 条件がtrueとfalseで異なる処理を実行します
  • if-else if-else文: 3つ以上の選択肢から1つを選びます
  • switch文: 特定の値との一致で処理を分岐します（breakを忘れずに）
  • ネスト: if文の中にif文を書けますが、深くなりすぎないように注意
  • 条件の順序: else ifでは上から順に評価されるため順序が重要です
