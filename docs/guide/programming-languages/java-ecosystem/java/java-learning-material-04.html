条件分岐とプログラムの流れ

if文やswitch文を使った条件分岐について学び、プログラムが状況に応じて異なる処理を実行する仕組みを理解します。

この章で学ぶこと

  • 条件分岐とは何か、なぜ必要なのかを理解する
  • if文、else文、else if文の使い方と違いを理解する
  • ネストした条件分岐と論理演算子の組み合わせ
  • switch文による多分岐処理とパターンマッチング

4.1 条件分岐とは何か

プログラムは通常、上から下へ順番に処理を実行していきます。しかし、実際のアプリケーションでは「もし〜ならば〜する」という条件に応じた処理が必要になります。このように、条件によって実行する処理を変えることを条件分岐といいます。

例えば、ログインシステムでは「入力されたパスワードが正しければログイン成功、間違っていればエラーメッセージ表示」というように、条件に応じて処理を分岐させます。

flowchart TD A([プログラム開始]) --> B{条件判定} B -->|条件が真| C[処理A実行] B -->|条件が偽| D[処理B実行] C --> E([続きの処理]) D --> E

日常生活の条件分岐

「雨が降っていたら傘を持つ、降っていなければ持たない」という判断も条件分岐の一種です。プログラムでも同様に、状況に応じた適切な処理を選択します。

4.2 if文の基本

if文は、最も基本的な条件分岐の構文です。条件式が真（true）の場合のみ、波括弧 {} 内の処理を実行します。

if文の基本構文

if文の構文 
if (条件式) {
    // 条件が真のときに実行される処理
}

具体例

IfExample.java 
public class IfExample {
    public static void main(String[] args) {
        int score = 85;

        // 80点以上なら合格メッセージを表示
        if (score >= 80) {
            System.out.println("合格です！");
        }

        System.out.println("判定終了");
    }
}

実行結果

合格です！
判定終了

4.3 if-else文 - 二者択一の処理

条件が真の場合と偽の場合でそれぞれ異なる処理を実行したい場合は、elseを使います。

if-else文の構文

if-else文の構文 
if (条件式) {
    // 条件が真のときの処理
} else {
    // 条件が偽のときの処理
}

具体例

IfElseExample.java 
public class IfElseExample {
    public static void main(String[] args) {
        int age = 17;

        // 18歳以上か未満かで処理を分岐
        if (age >= 18) {
            System.out.println("成人です");
        } else {
            System.out.println("未成年です");
        }
    }
}
flowchart TD A([開始]) --> B{age >= 18?} B -->|true| C["出力: 成人です"] B -->|false| D["出力: 未成年です"] C --> E([終了]) D --> E

4.4 else if文 - 複数条件の分岐

3つ以上の条件を判定したい場合は、else ifを使って複数の条件を順番にチェックします。

else if文の構文

else if文の構文 
if (条件1) {
    // 条件1が真のときの処理
} else if (条件2) {
    // 条件2が真のときの処理
} else if (条件3) {
    // 条件3が真のときの処理
} else {
    // すべての条件が偽のときの処理
}

具体例：成績判定

GradeExample.java 
public class GradeExample {
    public static void main(String[] args) {
        int score = 75;

        // 成績を5段階で評価
        if (score >= 90) {
            System.out.println("評価: A（優秀）");
        } else if (score >= 80) {
            System.out.println("評価: B（良好）");
        } else if (score >= 70) {
            System.out.println("評価: C（普通）");
        } else if (score >= 60) {
            System.out.println("評価: D（要努力）");
        } else {
            System.out.println("評価: F（不合格）");
        }
    }
}

else ifの評価順序

条件は上から順番にチェックされ、最初に真となった条件のブロックだけが実行されます。それ以降の条件は評価されません。

4.5 ネストした条件分岐

if文の中にさらにif文を書くことをネスト（入れ子）といいます。複雑な条件判定が必要な場合に使用しますが、深くネストしすぎるとコードが読みにくくなるため注意が必要です。

NestedIfExample.java 
public class NestedIfExample {
    public static void main(String[] args) {
        int age = 25;
        boolean hasLicense = true;

        // 年齢と免許の有無を両方チェック
        if (age >= 18) {
            if (hasLicense) {
                System.out.println("運転可能です");
            } else {
                System.out.println("免許を取得してください");
            }
        } else {
            System.out.println("18歳未満は運転不可");
        }
    }
}

論理演算子での簡潔化

上記のネストは、論理演算子ANDを使って if (age >= 18 && hasLicense) と簡潔に書くこともできます。

4.6 switch文による多分岐処理

switch文は、一つの変数の値によって複数の処理に分岐する場合に便利です。複数のelse ifを連ねるよりも読みやすくなることがあります。

switch文の基本構文

switch文の構文 
switch (変数) {
    case 値1:
        // 値1に一致したときの処理
        break;
    case 値2:
        // 値2に一致したときの処理
        break;
    default:
        // どの値にも一致しなかったときの処理
}

具体例：曜日判定

SwitchExample.java 
public class SwitchExample {
    public static void main(String[] args) {
        int dayOfWeek = 3; // 1=月曜, 2=火曜, ...

        switch (dayOfWeek) {
            case 1:
                System.out.println("月曜日");
                break;
            case 2:
                System.out.println("火曜日");
                break;
            case 3:
                System.out.println("水曜日");
                break;
            case 6:
            case 7:
                System.out.println("週末");
                break;
            default:
                System.out.println("平日");
        }
    }
}

breakを忘れずに

各caseの最後には必ずbreak;を書きましょう。書かないと、次のcaseの処理も実行されてしまいます（フォールスルー）。

Java 14以降の新しいswitch式

Java 14以降では、より簡潔に書けるswitch式が利用できます。

SwitchExpressionExample.java 
public class SwitchExpressionExample {
    public static void main(String[] args) {
        int dayOfWeek = 3;

        // switch式（breakが不要）
        String dayName = switch (dayOfWeek) {
            case 1 -> "月曜日";
            case 2 -> "火曜日";
            case 3 -> "水曜日";
            case 6, 7 -> "週末";
            default -> "平日";
        };

        System.out.println(dayName);
    }
}

実習 4-1: BMI判定プログラムを作ろう

身長と体重からBMI（肥満度指数）を計算し、判定結果を表示するプログラムを作成します。

課題

BMI = 体重(kg) ÷ (身長(m) × 身長(m)) を計算し、以下の基準で判定してください。

  • • BMI < 18.5: 低体重
  • • 18.5 ≦ BMI < 25: 普通体重
  • • 25 ≦ BMI < 30: 肥満（1度）
  • • BMI ≧ 30: 肥満（2度以上）

実装例

public class BmiCalculator {
    public static void main(String[] args) {
        double height = 1.70; // 身長(m)
        double weight = 65.0; // 体重(kg)

        // BMI計算
        double bmi = weight / (height * height);
        System.out.println("BMI: " + bmi);

        // 判定
        if (bmi < 18.5) {
            System.out.println("判定: 低体重");
        } else if (bmi < 25) {
            System.out.println("判定: 普通体重");
        } else if (bmi < 30) {
            System.out.println("判定: 肥満（1度）");
        } else {
            System.out.println("判定: 肥満（2度以上）");
        }
    }
}

チャレンジ: 身長をcm単位で入力し、プログラム内でm単位に変換してみましょう。

実習 4-2: 季節判定プログラムを作ろう

月の数値を入力し、switch文を使って該当する季節を表示するプログラムを作成します。

実装例

public class SeasonChecker {
    public static void main(String[] args) {
        int month = 7;

        switch (month) {
            case 12, 1, 2 -> System.out.println("冬");
            case 3, 4, 5 -> System.out.println("春");
            case 6, 7, 8 -> System.out.println("夏");
            case 9, 10, 11 -> System.out.println("秋");
            default -> System.out.println("無効な月");
        }
    }
}

理解度確認クイズ

  1. Q1. if文とswitch文、どちらを使うべきかの判断基準は何ですか？

    回答を表示

    一般的に、一つの変数の値による多分岐の場合はswitch文、複雑な条件式や範囲判定が必要な場合はif文が適しています。switch文は整数やString、enumなど特定の型の完全一致判定に向いています。

  2. Q2. else ifとネストしたifの違いは何ですか？

    回答を表示

    else ifは同じレベルの複数条件を順番にチェックし、最初に一致した条件だけを実行します。ネストしたifは、外側の条件が真の場合にさらに内側の条件をチェックする、階層的な判定に使います。

  3. Q3. switch文でbreakを省略するとどうなりますか？

    回答を表示

    breakを省略すると、該当するcaseの処理を実行した後、次のcaseの処理も続けて実行されます（フォールスルー）。意図的に使う場合もありますが、多くの場合はバグの原因になります。

  4. Q4. 次のコードの出力結果は何ですか？
    int x = 10; if (x > 5) System.out.println("A"); else System.out.println("B");

    回答を表示

    「A」が出力されます。xは10なので、条件式 x > 5 は真となり、if側の処理が実行されます。

この章のまとめ

  • 条件分岐は、条件に応じて異なる処理を実行する仕組みで、プログラムの柔軟性を高める
  • if文は条件が真のときのみ処理を実行し、elseで偽の場合の処理を追加できる
  • else ifを使うことで、複数の条件を順番にチェックできる
  • switch文は、一つの変数の値による多分岐処理に適している
  • ネストした条件分岐や論理演算子を使うことで、複雑な条件判定が可能
