第4章

条件分岐 - プログラムに判断させる

プログラムが状況に応じて異なる動作をする「条件分岐」の仕組みを学びます。

この章で学ぶこと

  • if文を使った基本的な条件分岐ができる
  • if-else文、if-else if-else文で複数の条件を処理できる
  • switch文の使い方と適用場面を理解する
  • 条件分岐のネスト（入れ子）を理解する

4.1 if文の基本

プログラミングにおいて、「もし〜ならば」という条件に応じて処理を変えたいことがよくあります。 例えば、「もし点数が60点以上ならば『合格』と表示する」といった処理です。 このような条件判断を行うために、if文を使用します。

if文の構文

if文の基本的な書き方は以下の通りです：

if文の構文 
if (条件式) {
    // 条件式がtrueのときに実行される処理
}

条件式の結果がtrueの場合、ブロック{}内の処理が実行されます。 falseの場合は何も実行されずに次の処理へ進みます。

条件式について

条件式には、第3章で学んだ比較演算子（==、!=、<、>、<=、>=）や 論理演算子（&&、||、!）を使用して、 結果がboolean型（trueまたはfalse）となる式を書きます。

具体例：点数判定プログラム

ScoreCheck.java 
public class ScoreCheck {
    public static void main(String[] args) {
        int score = 75;  // 点数

        // 点数が60以上なら「合格」と表示
        if (score >= 60) {
            System.out.println("合格です！");
        }

        System.out.println("プログラムを終了します。");
    }
}

このプログラムでは、scoreが60以上の場合に「合格です！」と表示されます。 そして、条件に関わらず最後に「プログラムを終了します。」が表示されます。

つまずきポイント：ブロック { } の省略

実行する文が1つだけの場合、ブロック{}を省略することができます。 しかし、初心者のうちは必ず { } を付ける習慣をつけましょう。

❌ 悪い例（省略すると混乱しやすい）：

if (score >= 60)
    System.out.println("合格");
    System.out.println("おめでとう");  // これは常に実行される！

✅ 良い例（{ } を明示）：

if (score >= 60) {
    System.out.println("合格");
    System.out.println("おめでとう");  // これも条件内の処理
}

4.2 if-else文

「もし〜ならば...、そうでなければ...」というように、2つの選択肢から1つを選ぶ場合はif-else文を使います。

if-else文の構文 
if (条件式) {
    // 条件式がtrueのときに実行される処理
} else {
    // 条件式がfalseのときに実行される処理
}

具体例：合格・不合格判定

PassOrFail.java 
public class PassOrFail {
    public static void main(String[] args) {
        int score = 45;  // 点数

        // 点数が60以上なら合格、それ以外は不合格
        if (score >= 60) {
            System.out.println("合格です！");
            System.out.println("よく頑張りました。");
        } else {
            System.out.println("不合格です。");
            System.out.println("次回また挑戦しましょう。");
        }
    }
}

実行結果（score = 45の場合）

不合格です。
次回また挑戦しましょう。

条件分岐の流れ図

if-else文の処理の流れを図で見てみましょう： 

graph TD
    A[プログラム開始] --> B{score >= 60?}
    B -->|true| C[合格を表示]
    B -->|false| D[不合格を表示]
    C --> E[プログラム終了]
    D --> E

4.3 if-else if-else文

3つ以上の選択肢がある場合は、if-else if-else文を使います。 複数の条件を順番にチェックし、最初にtrueになった条件のブロックが実行されます。

if-else if-else文の構文 
if (条件式1) {
    // 条件式1がtrueのときに実行
} else if (条件式2) {
    // 条件式1がfalseで、条件式2がtrueのときに実行
} else if (条件式3) {
    // 条件式1と2がfalseで、条件式3がtrueのときに実行
} else {
    // すべての条件式がfalseのときに実行
}

具体例：成績評価プログラム（A/B/C/D/F）

GradeEvaluator.java 
public class GradeEvaluator {
    public static void main(String[] args) {
        int score = 78;  // 点数
        String grade;    // 評価

        // 点数に応じて評価を決定
        if (score >= 90) {
            grade = "A";
            System.out.println("素晴らしい！");
        } else if (score >= 80) {
            grade = "B";
            System.out.println("良くできました。");
        } else if (score >= 70) {
            grade = "C";
            System.out.println("合格レベルです。");
        } else if (score >= 60) {
            grade = "D";
            System.out.println("ギリギリ合格です。");
        } else {
            grade = "F";
            System.out.println("不合格です。");
        }

        System.out.println("評価: " + grade);
    }
}

実行結果（score = 78の場合）

合格レベルです。
評価: C

重要：条件の順序

if-else if-else文では、条件は上から順番にチェックされ、最初にtrueになった条件のブロックだけが実行されます。 そのため、条件の順序が非常に重要です。この例では、高い点数から順に判定しているため、正しく動作します。

つまずきポイント：条件の順序ミス

条件を逆の順序で書くと、意図した動作にならないことがあります：

❌ 悪い例：

if (score >= 60) {  // 先に緩い条件
    grade = "D";
} else if (score >= 70) {  // この条件は実行されない！
    grade = "C";
}

この場合、78点でも最初の条件(score >= 60)に当てはまるため、評価は「D」になってしまいます。

4.4 switch文

switch文は、ある変数の値によって処理を分岐させる場合に便利な構文です。 特に、値が整数や文字列などの固定値である場合に、if-else if文よりも読みやすくなります。

switch文の構文（従来の書き方） 
switch (変数) {
    case 値1:
        // 変数が値1の場合に実行
        break;
    case 値2:
        // 変数が値2の場合に実行
        break;
    case 値3:
        // 変数が値3の場合に実行
        break;
    default:
        // どのcaseにも当てはまらない場合に実行
        break;
}

具体例：曜日の判定プログラム

DayOfWeek.java 
public class DayOfWeek {
    public static void main(String[] args) {
        int day = 3;  // 曜日（1=月曜、2=火曜、...、7=日曜）
        String dayName;

        switch (day) {
            case 1:
                dayName = "月曜日";
                break;
            case 2:
                dayName = "火曜日";
                break;
            case 3:
                dayName = "水曜日";
                break;
            case 4:
                dayName = "木曜日";
                break;
            case 5:
                dayName = "金曜日";
                break;
            case 6:
                dayName = "土曜日";
                break;
            case 7:
                dayName = "日曜日";
                break;
            default:
                dayName = "不正な値";
                break;
        }

        System.out.println("今日は" + dayName + "です。");
    }
}

実行結果（day = 3の場合）

今日は水曜日です。

つまずきポイント：breakの忘れ（フォールスルー）

breakを書き忘れると、次のcaseの処理まで続けて実行されてしまいます。これをフォールスルーと呼びます。

❌ 悪い例：

switch (day) {
    case 1:
        dayName = "月曜日";
        // break忘れ！
    case 2:
        dayName = "火曜日";  // day=1でもここが実行される
        break;
}

この場合、day = 1でも「火曜日」と表示されてしまいます。 必ず各caseの最後にbreak;を書きましょう。

switch式（Java 14以降の新しい書き方）

Java 14以降では、より簡潔に書けるswitch式が使えるようになりました：

DayOfWeekModern.java（switch式） 
public class DayOfWeekModern {
    public static void main(String[] args) {
        int day = 3;

        // switch式を使った書き方（breakが不要）
        String dayName = switch (day) {
            case 1 -> "月曜日";
            case 2 -> "火曜日";
            case 3 -> "水曜日";
            case 4 -> "木曜日";
            case 5 -> "金曜日";
            case 6 -> "土曜日";
            case 7 -> "日曜日";
            default -> "不正な値";
        };

        System.out.println("今日は" + dayName + "です。");
    }
}

switch式の利点

  • breakが不要（フォールスルーの心配がない）
  • コードが簡潔になる
  • 値を直接返すことができる

if文とswitch文の使い分け

条件 推奨される構文
値が整数・文字列などの固定値 switch文
範囲の比較（>=、<など） if文
複雑な論理式（&&、||など） if文
2つの選択肢 if-else文

4.5 条件分岐のネスト（入れ子）

if文の中にさらにif文を書くことをネスト（入れ子）といいます。 複雑な条件判断が必要な場合に使用しますが、深くネストしすぎるとコードが読みにくくなるので注意が必要です。

具体例：年齢に応じた料金計算

TicketPrice.java 
public class TicketPrice {
    public static void main(String[] args) {
        int age = 25;
        boolean isStudent = false;  // 学生かどうか
        int price;

        // 年齢と学生かどうかで料金を決定
        if (age < 12) {
            price = 500;  // 子供料金
            System.out.println("子供料金が適用されます。");
        } else if (age < 65) {
            // 大人の場合、さらに学生かどうかで判定
            if (isStudent) {
                price = 1000;  // 学生料金
                System.out.println("学生料金が適用されます。");
            } else {
                price = 1500;  // 大人料金
                System.out.println("大人料金が適用されます。");
            }
        } else {
            price = 800;  // シニア料金
            System.out.println("シニア料金が適用されます。");
        }

        System.out.println("料金: " + price + "円");
    }
}

実行結果（age = 25, isStudent = falseの場合）

大人料金が適用されます。
料金: 1500円

ヒント：論理演算子を使って簡潔に書く

ネストが深くなる場合は、論理演算子（&&、||）を使うと簡潔に書けることがあります： 

if (age >= 12 && age < 65 && isStudent) {
    price = 1000;  // 学生料金
    System.out.println("学生料金が適用されます。");
}

読みやすいコードを書くためのヒント

  • • ネストは2〜3階層までに抑える
  • • 論理演算子を活用して条件をまとめる
  • • 複雑な条件は変数に代入して名前を付ける
  • • インデント（字下げ）を正しく使って見やすくする

実習 4-1: じゃんけんゲームの判定ロジック

じゃんけんの勝敗を判定するプログラムを作成しましょう。 自分と相手の手（0=グー、1=チョキ、2=パー）に応じて、勝ち・負け・あいこを表示します。

手順

  1. 1 RockPaperScissors.javaというファイルを作成します。
  2. 2 自分の手をmyHand、相手の手をopponentHandという変数に格納します（0=グー、1=チョキ、2=パー）。
  3. 3 if文を使って、勝ち・負け・あいこを判定し、結果を表示します。

ヒント

じゃんけんのルール：

  • グー(0) vs チョキ(1) → グーの勝ち
  • チョキ(1) vs パー(2) → チョキの勝ち
  • パー(2) vs グー(0) → パーの勝ち
  • 同じ手ならあいこ

実行例

自分: グー
相手: チョキ
結果: あなたの勝ち！

実習 4-2: BMI判定プログラム

第3章で作成したBMI計算プログラムを拡張し、BMI値に応じて肥満度を判定するプログラムを作成しましょう。

手順

  1. 1 身長（m）と体重（kg）からBMIを計算します（BMI = 体重 ÷ (身長 × 身長)）。
  2. 2 BMI値に応じて、以下のように判定します：
  • • BMI < 18.5 → 「低体重」
  • • 18.5 ≤ BMI < 25 → 「標準体重」
  • • 25 ≤ BMI < 30 → 「肥満（1度）」
  • • 30 ≤ BMI → 「肥満（2度以上）」

実行例

身長: 1.70m
体重: 65.0kg
BMI: 22.49
判定: 標準体重

理解度確認クイズ

  1. Q1. 以下のコードで、x = 5のとき、何が表示されますか？

    if (x > 10) {
    System.out.println("A");
} else if (x > 3) {
    System.out.println("B");
} else {
    System.out.println("C");
}
    回答を表示

    答え：B
    最初の条件x > 10はfalse（5は10より大きくない）なので、次の条件x > 3をチェックします。 これはtrue（5は3より大きい）なので、「B」と表示されます。

  2. Q2. switch文でbreak;を書き忘れるとどうなりますか？

    回答を表示

    答え：フォールスルーが発生する
    break;を書き忘れると、マッチしたcaseから次のcaseの処理まで続けて実行されてしまいます（フォールスルー）。 意図的にこの動作を使う場合もありますが、通常は各caseの最後にbreak;を書く必要があります。

  3. Q3. 以下のif文のブロック{ }を省略した場合、どの行が条件内の処理として実行されますか？

    if (score >= 60)
    System.out.println("合格");
    System.out.println("おめでとう");
    回答を表示

    答え：最初のprintlnだけ
    ブロック{ }を省略した場合、if文に続く最初の1文だけが条件の対象となります。 そのため、「合格」だけが条件によって表示され、「おめでとう」はscoreの値に関わらず常に表示されます。 このような混乱を避けるため、常にブロック { } を書く習慣をつけましょう。

  4. Q4. if文とswitch文の使い分けについて、範囲の比較（例：score >= 60）をする場合、どちらが適していますか？

    回答を表示

    答え：if文が適している
    switch文は値が整数や文字列などの固定値の場合に適しています。 範囲の比較（>=、<など）や複雑な論理式（&&、||など）を使う場合は、if文を使うのが適切です。

この章のまとめ

  • if文は条件が真のときだけ処理を実行する
  • if-else文は「〜ならば...、そうでなければ...」の2択を表現する
  • if-else if-else文は3つ以上の選択肢を順番にチェックする
  • switch文は値が固定値の場合に読みやすい分岐ができる（breakを忘れずに）
  • 条件分岐のネストを使うと複雑な判断ができるが、深くしすぎないよう注意
  • ブロック{ }は省略せず、常に書く習慣をつけよう
