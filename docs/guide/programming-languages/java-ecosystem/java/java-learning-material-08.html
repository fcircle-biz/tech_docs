第8章

クラスとオブジェクト - オブジェクト指向の基礎

Javaの核心である「オブジェクト指向プログラミング」の基本概念を学び、クラスとオブジェクトを作成できるようになります。

この章で学ぶこと

  • オブジェクト指向プログラミングの基本概念を理解する
  • クラスの定義ができる
  • オブジェクト(インスタンス)を生成して使用できる
  • フィールドとメソッドの関係を理解する

8.1 オブジェクト指向とは

オブジェクト指向プログラミング（OOP: Object-Oriented Programming）は、現実世界の「モノ」をプログラムで表現する考え方です。例えば、「犬」という存在を考えてみましょう。

現実世界のモデル化

犬には以下のような特徴があります：

  • 属性（データ）: 名前、年齢、色、体重など
  • 振る舞い（機能）: 鳴く、走る、食べるなど

オブジェクト指向では、これらを「フィールド（属性）」と「メソッド（振る舞い）」として、ひとつの「クラス」にまとめます。

クラスとオブジェクトの関係

クラスは「設計図」、オブジェクトは「実体」です。たとえば、「犬」という設計図（クラス）から、「ポチ」「コロ」「タロウ」という具体的な犬（オブジェクト）を作り出すイメージです。

オブジェクト指向の3大要素

カプセル化

データと処理をひとまとめにし、外部から不正なアクセスを防ぐ

継承

既存のクラスを元に新しいクラスを作成し、機能を受け継ぐ

ポリモーフィズム

同じ操作でも、対象によって異なる動作をする柔軟性

この章では、まず「カプセル化」を中心に学びます。「継承」と「ポリモーフィズム」は第9章で詳しく学習します。

8.2 クラスの定義

クラスは「フィールド（属性）」と「メソッド（振る舞い）」で構成されます。まずは最もシンプルなクラスから見てみましょう。

Dog.java 
// 犬クラスの定義
public class Dog {
    // フィールド（属性・データ）
    String name;    // 名前
    int age;        // 年齢
    String color;   // 色

    // メソッド（振る舞い・機能）
    public void bark() {
        System.out.println(name + "が「ワンワン!」と鳴きました");
    }

    public void run() {
        System.out.println(name + "が走っています");
    }

    public void showInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("色: " + color);
    }
}

クラス定義のルール

  • • クラス名は大文字で始める（例: Dog, Student, BankAccount）
  • • 1ファイルにつき1つのpublicクラスを定義（ファイル名とクラス名を一致させる）
  • • フィールドはクラス内に直接宣言する
  • • メソッドはフィールドにアクセスできる

8.3 オブジェクトの生成と使用

クラス（設計図）を定義したら、実際に使える「オブジェクト（実体）」を作成します。これを「インスタンス化」と呼びます。

DogTest.java 
public class DogTest {
    public static void main(String[] args) {
        // オブジェクトの生成（インスタンス化）
        Dog pochi = new Dog();  // ポチという犬を作成

        // フィールドに値を設定
        pochi.name = "ポチ";
        pochi.age = 3;
        pochi.color = "茶色";

        // メソッドを呼び出す
        pochi.bark();      // ポチが「ワンワン!」と鳴きました
        pochi.run();       // ポチが走っています
        pochi.showInfo();  // 情報を表示

        System.out.println("---");

        // 2つ目のオブジェクトを作成
        Dog koro = new Dog();
        koro.name = "コロ";
        koro.age = 5;
        koro.color = "白";

        koro.bark();
        koro.showInfo();
    }
}

オブジェクト生成の流れ

オブジェクト生成の構文

クラス名 変数名 = new クラス名();
  • • クラス名 変数名: 参照変数の宣言（オブジェクトを指す変数）
  • • new: オブジェクトを新規作成する演算子
  • • クラス名(): コンストラクタの呼び出し（次のセクションで詳しく解説）

フィールドとメソッドへのアクセス

ドット演算子（.）を使って、オブジェクトのフィールドやメソッドにアクセスします。

アクセスの例 
// フィールドへのアクセス
pochi.name = "ポチ";      // フィールドに値を設定
String dogName = pochi.name;  // フィールドの値を取得

// メソッドの呼び出し
pochi.bark();  // メソッドを実行

8.4 コンストラクタ

コンストラクタは、オブジェクト生成時に自動的に呼ばれる特別なメソッドです。主にフィールドの初期化に使用します。

Dog.java（コンストラクタ追加版） 
public class Dog {
    String name;
    int age;
    String color;

    // デフォルトコンストラクタ（引数なし）
    public Dog() {
        name = "名無し";
        age = 0;
        color = "不明";
        System.out.println("犬オブジェクトが作成されました");
    }

    // 引数付きコンストラクタ
    public Dog(String name, int age, String color) {
        this.name = name;
        this.age = age;
        this.color = color;
        System.out.println(name + "が作成されました");
    }

    // コンストラクタのオーバーロード
    public Dog(String name) {
        this.name = name;
        this.age = 0;
        this.color = "不明";
    }

    public void bark() {
        System.out.println(name + "が「ワンワン!」と鳴きました");
    }

    public void showInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("色: " + color);
    }
}

コンストラクタの特徴

  • • メソッド名はクラス名と同じにする
  • • 戻り値の型を書かない（voidも不要）
  • • オブジェクト生成時（new）に自動的に呼ばれる
  • • オーバーロードできる（引数の異なる複数のコンストラクタ）
  • • 明示的にコンストラクタを書かない場合、引数なしのデフォルトコンストラクタが自動生成される

thisキーワード

`this`は「このオブジェクト自身」を指すキーワードです。フィールド名と引数名が同じ場合に、区別するために使用します。

thisの使用例 
public Dog(String name, int age, String color) {
    // this.name はフィールド、name は引数
    this.name = name;
    this.age = age;
    this.color = color;
}

// thisなしだと、すべて引数として扱われてしまう（間違い）
public Dog(String name, int age, String color) {
    name = name;  // これではフィールドに代入されない!
    age = age;
    color = color;
}
コンストラクタの使用例 
public class DogTest2 {
    public static void main(String[] args) {
        // デフォルトコンストラクタを使用
        Dog dog1 = new Dog();
        dog1.showInfo();

        System.out.println("---");

        // 引数付きコンストラクタを使用
        Dog dog2 = new Dog("ポチ", 3, "茶色");
        dog2.showInfo();

        System.out.println("---");

        // 名前のみのコンストラクタを使用
        Dog dog3 = new Dog("コロ");
        dog3.showInfo();
    }
}

8.5 アクセス修飾子とカプセル化

カプセル化とは、オブジェクトの内部データを外部から直接アクセスできないようにし、安全なメソッドを通じてのみ操作できるようにする仕組みです。

アクセス修飾子の種類

修飾子 同じクラス 同じパッケージ サブクラス すべて
public ○ ○ ○ ○
protected ○ ○ ○ ×
（なし） ○ ○ × ×
private ○ × × ×

カプセル化の実装

フィールドを`private`にして外部から直接アクセスできないようにし、`public`なgetter/setterメソッドを通じてアクセスします。

BankAccount.java 
public class BankAccount {
    // フィールドをprivateにして外部から直接アクセスできないようにする
    private String accountNumber;  // 口座番号
    private String ownerName;      // 口座名義
    private int balance;           // 残高

    // コンストラクタ
    public BankAccount(String accountNumber, String ownerName) {
        this.accountNumber = accountNumber;
        this.ownerName = ownerName;
        this.balance = 0;  // 初期残高は0
    }

    // getter: 残高を取得（読み取り専用）
    public int getBalance() {
        return balance;
    }

    // getter: 口座名義を取得
    public String getOwnerName() {
        return ownerName;
    }

    // 入金メソッド
    public void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println(amount + "円を入金しました");
            System.out.println("現在の残高: " + balance + "円");
        } else {
            System.out.println("入金額は正の値である必要があります");
        }
    }

    // 出金メソッド
    public void withdraw(int amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println(amount + "円を出金しました");
            System.out.println("現在の残高: " + balance + "円");
        } else if (amount > balance) {
            System.out.println("残高不足です");
        } else {
            System.out.println("出金額は正の値である必要があります");
        }
    }

    // 口座情報を表示
    public void showInfo() {
        System.out.println("口座番号: " + accountNumber);
        System.out.println("名義: " + ownerName);
        System.out.println("残高: " + balance + "円");
    }
}

カプセル化の利点

  • • データの不正な変更を防げる（残高をマイナスにできない、など）
  • • クラス内部の実装を変更しても、外部のコードに影響しない
  • • 必要なチェック処理を必ずメソッド内で実行できる
BankAccountTest.java 
public class BankAccountTest {
    public static void main(String[] args) {
        // 銀行口座オブジェクトを作成
        BankAccount account = new BankAccount("123-456", "山田太郎");

        // メソッドを通じて操作
        account.deposit(10000);   // 10000円入金
        account.withdraw(3000);   // 3000円出金
        account.withdraw(10000);  // 残高不足でエラー

        System.out.println("---");
        account.showInfo();

        // 以下はコンパイルエラー（privateフィールドに直接アクセスできない）
        // account.balance = -1000;  // エラー: balanceはprivate
    }
}

getter/setterメソッドの命名規則

JavaBeansの規約

  • • getter: `get` + フィールド名（先頭大文字） 例: `getName()`, `getAge()`
  • • setter: `set` + フィールド名（先頭大文字） 例: `setName(String name)`
  • • boolean型の場合、getterは`is`で始めることもある 例: `isAdult()`

実習 8-1: 学生クラスの作成

学生を表すStudentクラスを作成し、カプセル化を実践しましょう。

要件

  • フィールド: name（名前）, studentId（学籍番号）, score（点数）をprivateで定義
  • コンストラクタ: name と studentId を引数に取る
  • getter/setterメソッドを作成
  • 成績判定メソッド: getGrade()（A/B/C/D/Fを返す）

実装例

Student.java 
public class Student {
    private String name;
    private String studentId;
    private int score;

    // コンストラクタ
    public Student(String name, String studentId) {
        this.name = name;
        this.studentId = studentId;
        this.score = 0;
    }

    // getter/setter
    public String getName() {
        return name;
    }

    public String getStudentId() {
        return studentId;
    }

    public int getScore() {
        return score;
    }

    public void setScore(int score) {
        if (score >= 0 && score <= 100) {
            this.score = score;
        } else {
            System.out.println("点数は0-100の範囲で設定してください");
        }
    }

    // 成績判定
    public String getGrade() {
        if (score >= 90) return "A";
        else if (score >= 80) return "B";
        else if (score >= 70) return "C";
        else if (score >= 60) return "D";
        else return "F";
    }

    // 情報表示
    public void showInfo() {
        System.out.println("学籍番号: " + studentId);
        System.out.println("名前: " + name);
        System.out.println("点数: " + score);
        System.out.println("成績: " + getGrade());
    }
}

テストプログラム

StudentTest.java 
public class StudentTest {
    public static void main(String[] args) {
        Student student1 = new Student("山田太郎", "S12345");
        student1.setScore(85);
        student1.showInfo();

        System.out.println("---");

        Student student2 = new Student("佐藤花子", "S12346");
        student2.setScore(92);
        student2.showInfo();
    }
}

理解度確認クイズ

  1. Q1. クラスとオブジェクトの関係を説明してください。

    回答を表示

    クラスは「設計図」、オブジェクトは「実体」です。クラスには属性（フィールド）と振る舞い（メソッド）が定義されており、new演算子を使ってクラスからオブジェクト（インスタンス）を作成します。1つのクラスから複数のオブジェクトを作成できます。

  2. Q2. コンストラクタの役割と、通常のメソッドとの違いを説明してください。

    回答を表示

    コンストラクタはオブジェクト生成時に自動的に呼ばれ、主にフィールドの初期化を行います。通常のメソッドとの違いは、①メソッド名がクラス名と同じ、②戻り値の型を書かない、③オブジェクト生成時に自動的に呼ばれる、という点です。

  3. Q3. thisキーワードはどのような時に使用しますか？

    回答を表示

    thisは「このオブジェクト自身」を指します。主にコンストラクタやsetterメソッドで、フィールド名と引数名が同じ場合に区別するために使用します。例えば、`this.name = name;`とすることで、左辺がフィールド、右辺が引数であることを明示できます。

  4. Q4. カプセル化とは何か、なぜ重要なのか説明してください。

    回答を表示

    カプセル化とは、フィールドをprivateにして外部から直接アクセスできないようにし、publicなメソッド（getter/setter）を通じてのみ操作できるようにすることです。これにより、①データの不正な変更を防げる、②クラス内部の実装変更が外部に影響しない、③必要なチェック処理を必ずメソッド内で実行できる、という利点があります。

本章のまとめ

  • オブジェクト指向は現実世界の「モノ」をプログラムで表現する考え方
  • クラスは設計図、オブジェクトは実体。new演算子でインスタンス化する
  • クラスはフィールド（属性）とメソッド（振る舞い）で構成される
  • コンストラクタはオブジェクト生成時に自動的に呼ばれ、フィールドを初期化する
  • カプセル化によりデータを保護し、安全なアクセスを実現できる
  • アクセス修飾子（public, private, protected）でアクセス制御を行う
