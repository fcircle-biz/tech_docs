第8章

クラスとオブジェクト - オブジェクト指向の基礎

Javaの核心である「オブジェクト指向プログラミング」の基本概念を学び、クラスとオブジェクトを作成できるようになります。

この章で学ぶこと

  • オブジェクト指向プログラミングの基本概念を理解する
  • クラスの定義ができる
  • オブジェクト（インスタンス）を生成して使用できる
  • フィールドとメソッドの関係を理解する

8.1 オブジェクト指向とは

オブジェクト指向プログラミング（OOP: Object-Oriented Programming）は、プログラムを「オブジェクト」という単位で構成する考え方です。現実世界のものごとをモデル化し、それをプログラムで表現するアプローチです。

現実世界のモデル化

例えば「犬」を考えてみましょう。犬には「名前」「年齢」「犬種」などの属性があり、「吠える」「走る」「食べる」などの動作ができます。オブジェクト指向では、これらをプログラムで表現します。

クラスとオブジェクトの関係

  • クラス：オブジェクトの「設計図」や「ひな型」。犬という種類全体を定義するもの。
  • オブジェクト（インスタンス）：クラスから作られた「実体」。具体的な一匹の犬（例：ポチ、タロウ）。

オブジェクト指向の3大要素（概要）

カプセル化

データとメソッドをひとまとめにし、外部から直接アクセスできないようにする

継承

既存のクラスの機能を引き継いで新しいクラスを作る（第9章で学習）

ポリモーフィズム

同じメソッド名で異なる動作を実現する（第9章で学習）

手続き型との違い

これまで学んできた「手続き型プログラミング」では、処理の手順を順番に記述していました。オブジェクト指向では、「データ」と「そのデータを操作する処理」を一体化させることで、より現実世界に近い自然な形でプログラムを設計できます。

8.2 クラスの定義

クラスは、フィールド（属性）とメソッド（振る舞い）で構成されます。

クラスの基本構文

Dog.java 
// クラスの定義
public class Dog {
    // フィールド（属性、メンバ変数）
    String name;    // 犬の名前
    int age;        // 犬の年齢
    String breed;   // 犬種

    // メソッド（振る舞い）
    void bark() {
        System.out.println(name + "「ワンワン！」");
    }

    void run() {
        System.out.println(name + "が走っています。");
    }

    void showInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("犬種: " + breed);
    }
}

命名規則の重要性

  • • クラス名は大文字で始める（PascalCase）: Dog, BankAccount, StudentInfo
  • • ファイル名はクラス名と同じにする：Dog.java
  • • フィールド名・メソッド名は小文字で始める（camelCase）: name, showInfo

フィールドとメソッドの違い

項目 フィールド メソッド
役割 データを保持する（属性） 処理を実行する（動作）
例 name, age, breed bark(), run(), showInfo()
アクセス方法 変数として読み書き ()を付けて呼び出す

1ファイル1クラスの原則

Javaでは、通常1つのファイルに1つのpublicクラスを定義します。ファイル名はクラス名と一致させる必要があります（Dog.java には Dog クラスを記述）。

8.3 オブジェクトの生成と使用

クラスはあくまで「設計図」です。実際に使うためには、new演算子を使ってオブジェクト（インスタンス）を作成する必要があります。

new演算子によるインスタンス化

Main.java 
public class Main {
    public static void main(String[] args) {
        // オブジェクトの生成（インスタンス化）
        Dog pochi = new Dog();

        // フィールドに値を設定
        pochi.name = "ポチ";
        pochi.age = 3;
        pochi.breed = "柴犬";

        // メソッドの呼び出し
        pochi.showInfo();
        pochi.bark();
        pochi.run();
    }
}

実行結果:

名前: ポチ
年齢: 3歳
犬種: 柴犬
ポチ「ワンワン！」
ポチが走っています。

参照変数の概念

Dog pochi = new Dog(); という文では：

  • Dog pochi：Dog型の参照変数 pochi を宣言
  • new Dog()：メモリ上に新しいDogオブジェクトを作成
  • pochi = ...：作成したオブジェクトへの参照を pochi に代入
graph LR
    A["参照変数 pochi"] -->|参照| B["メモリ上の\nDogオブジェクト\n(name, age, breed)"]

ドット演算子によるアクセス

オブジェクトのフィールドやメソッドにアクセスするには、ドット演算子（.）を使います。

例: ドット演算子の使用 
// フィールドへのアクセス
pochi.name = "ポチ";          // 値の設定
String dogName = pochi.name;  // 値の取得

// メソッドの呼び出し
pochi.bark();   // メソッド実行

複数のオブジェクトを作成する

同じクラスから複数のオブジェクトを作成できます。それぞれ独立したデータを持ちます。

複数オブジェクトの作成 
// 1匹目の犬
Dog pochi = new Dog();
pochi.name = "ポチ";
pochi.age = 3;
pochi.breed = "柴犬";

// 2匹目の犬
Dog taro = new Dog();
taro.name = "タロウ";
taro.age = 5;
taro.breed = "ゴールデンレトリバー";

// それぞれ独立したデータを持つ
pochi.bark();  // ポチ「ワンワン！」
taro.bark();   // タロウ「ワンワン！」

8.4 コンストラクタ

コンストラクタは、オブジェクト生成時に自動的に実行される特別なメソッドです。主にフィールドの初期化に使われます。

コンストラクタの特徴

  • メソッド名がクラス名と同じ
  • 戻り値の型を書かない（voidも書かない）
  • new演算子でオブジェクトを生成するときに自動実行される
  • オブジェクトの初期化を担当する

デフォルトコンストラクタ

コンストラクタを明示的に定義しない場合、Javaコンパイラが自動的に引数なしのコンストラクタを用意してくれます。これをデフォルトコンストラクタと呼びます。

引数付きコンストラクタ

Dog.java（コンストラクタ追加版） 
public class Dog {
    String name;
    int age;
    String breed;

    // コンストラクタ（引数付き）
    public Dog(String name, int age, String breed) {
        // フィールドの初期化
        this.name = name;
        this.age = age;
        this.breed = breed;
    }

    void bark() {
        System.out.println(name + "「ワンワン！」");
    }

    void showInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("犬種: " + breed);
    }
}
Main.java（コンストラクタを使用） 
public class Main {
    public static void main(String[] args) {
        // コンストラクタで初期化しながらオブジェクト生成
        Dog pochi = new Dog("ポチ", 3, "柴犬");
        Dog taro = new Dog("タロウ", 5, "ゴールデンレトリバー");

        // すでに初期化されているのでそのまま使える
        pochi.showInfo();
        taro.bark();
    }
}

thisキーワード

thisキーワードは、「このオブジェクト自身」を指します。引数名とフィールド名が同じ場合に区別するために使います。

thisの使い分け

this.name → フィールド（オブジェクトのデータ）
name → ローカル変数または引数

コンストラクタのオーバーロード

メソッドと同様に、コンストラクタも複数定義できます。

コンストラクタのオーバーロード 
public class Dog {
    String name;
    int age;
    String breed;

    // 引数なしコンストラクタ
    public Dog() {
        this.name = "名前未設定";
        this.age = 0;
        this.breed = "雑種";
    }

    // 引数1つのコンストラクタ
    public Dog(String name) {
        this.name = name;
        this.age = 0;
        this.breed = "雑種";
    }

    // 引数3つのコンストラクタ
    public Dog(String name, int age, String breed) {
        this.name = name;
        this.age = age;
        this.breed = breed;
    }

    // メソッド省略...
}
様々なコンストラクタの使用例 
Dog dog1 = new Dog();                          // 引数なし
Dog dog2 = new Dog("ポチ");                    // 引数1つ
Dog dog3 = new Dog("タロウ", 5, "柴犬");       // 引数3つ

8.5 アクセス修飾子とカプセル化

アクセス修飾子は、クラスのメンバー（フィールドやメソッド）へのアクセス範囲を制御するキーワードです。これにより、データを適切に保護する「カプセル化」を実現できます。

4つのアクセス修飾子

修飾子 同じクラス内 同じパッケージ内 サブクラス すべて
public ○ ○ ○ ○
protected ○ ○ ○ ×
なし（デフォルト） ○ ○ × ×
private ○ × × ×

カプセル化の重要性

カプセル化とは、フィールドをprivateにして外部から直接アクセスできないようにし、publicなメソッドを通じて安全にアクセスする仕組みです。

カプセル化しない場合の問題

dog.age = -5;
フィールドが直接変更できると、不正な値（マイナスの年齢など）を設定される危険があります。

getter/setterメソッド

フィールドをprivateにした場合、値の取得・設定には専用のメソッドを用意します。

BankAccount.java（カプセル化の例） 
public class BankAccount {
    // フィールドはprivateで隠蔽
    private String accountNumber;  // 口座番号
    private String owner;          // 口座名義
    private int balance;           // 残高

    // コンストラクタ
    public BankAccount(String accountNumber, String owner) {
        this.accountNumber = accountNumber;
        this.owner = owner;
        this.balance = 0;  // 初期残高は0
    }

    // getter: 残高を取得
    public int getBalance() {
        return balance;
    }

    // getter: 口座名義を取得
    public String getOwner() {
        return owner;
    }

    // 入金メソッド（値の検証付き）
    public void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println(amount + "円を入金しました。");
        } else {
            System.out.println("入金額は正の数である必要があります。");
        }
    }

    // 出金メソッド（値の検証付き）
    public void withdraw(int amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println(amount + "円を出金しました。");
        } else if (amount > balance) {
            System.out.println("残高不足です。");
        } else {
            System.out.println("出金額は正の数である必要があります。");
        }
    }

    // 残高照会メソッド
    public void showBalance() {
        System.out.println("口座名義: " + owner);
        System.out.println("現在の残高: " + balance + "円");
    }
}
Main.java（使用例） 
public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount("123-456", "山田太郎");

        account.deposit(10000);   // 10000円を入金
        account.withdraw(3000);   // 3000円を出金
        account.showBalance();    // 残高照会

        // 不正な操作を試みる
        account.withdraw(-500);    // エラー: 出金額は正の数である必要があります
        account.withdraw(20000);   // エラー: 残高不足です

        // 直接フィールドにアクセスしようとするとコンパイルエラー
        // account.balance = 1000000;  // コンパイルエラー！
    }
}

カプセル化の利点

  • • データの保護: 不正な値を防ぐ
  • • 安全なアクセス: メソッドで値を検証できる
  • • 保守性の向上: 内部実装を変更しても外部に影響しない

学生の情報を管理するStudentクラスを作成し、カプセル化を実践します。

つまずきやすいポイント

クラスとオブジェクトの混同

「クラス」は設計図、「オブジェクト」は実体です。クラスだけではプログラムは動きません。必ずnew演算子でオブジェクトを作成する必要があります。

Dog.bark(); // NG: クラス名では呼べない

Dog pochi = new Dog(); pochi.bark(); // OK

参照とnullの理解

参照変数がnull（何も参照していない状態）のままメソッドを呼ぶとNullPointerExceptionが発生します。

Dog dog = null; dog.bark(); // エラー！

Dog dog = new Dog(); dog.bark(); // OK

thisキーワードの役割

引数名とフィールド名が同じとき、thisを付けないとローカル変数（引数）を指してしまいます。

name = name; // NG: 自分自身に代入している

this.name = name; // OK: フィールドに代入

理解度確認クイズ

  1. Q1. クラスとオブジェクトの関係を正しく説明しているのはどれですか？

    回答を表示

    回答: クラスは「設計図」、オブジェクトは「その設計図から作られた実体（インスタンス）」です。1つのクラスから複数のオブジェクトを作成でき、それぞれ独立したデータを持ちます。

  2. Q2. 次のコードで何が出力されますか？

    Dog dog1 = new Dog();
dog1.name = "ポチ";
Dog dog2 = dog1;
dog2.name = "タロウ";
System.out.println(dog1.name);
    回答を表示

    回答: 「タロウ」が出力されます。Dog dog2 = dog1;は、dog2とdog1が同じオブジェクトを参照することを意味します。そのため、dog2で名前を変更すると、dog1から見ても変更されています（参照の共有）。

  3. Q3. privateフィールドにアクセスするための正しい方法は？

    回答を表示

    回答: getter/setterメソッドを使います。privateフィールドは外部から直接アクセスできないため、publicなメソッドを通じて値を取得・設定します。これにより、値の検証や制御ができ、データを安全に扱えます。

  4. Q4. コンストラクタの特徴として正しくないものはどれですか？
    a) クラス名と同じ名前
    b) 戻り値の型がvoid
    c) オブジェクト生成時に自動実行される
    d) オーバーロードできる

    回答を表示

    回答: b) が誤りです。コンストラクタには戻り値の型を書きません（voidも書かない）。コンストラクタは名前がクラス名と同じで、オブジェクト生成時に自動実行され、オーバーロードも可能です。
