クラスとオブジェクトの基礎

Javaの中核概念であるクラスとオブジェクトについて学びます。フィールド、メソッド、コンストラクタ、インスタンス生成など、オブジェクト指向プログラミングの基礎を理解します。

この章で学ぶこと

  • オブジェクト指向プログラミングの基本概念
  • クラスとオブジェクト（インスタンス）の関係と作成方法
  • フィールド（属性）とメソッド（振る舞い）の定義
  • コンストラクタによるオブジェクトの初期化

8.1 オブジェクト指向プログラミングとは

これまで学んできた変数、条件分岐、繰り返し、配列、メソッドなどは、プログラミングの基本的な構成要素です。しかし、規模が大きく複雑なプログラムを作る場合、これらだけでは管理が難しくなります。そこで登場するのがオブジェクト指向プログラミング（OOP: Object-Oriented Programming）です。

オブジェクト指向プログラミングは、現実世界の「もの（オブジェクト）」をプログラムで表現する考え方です。例えば、自動車、銀行口座、社員情報などを、データ（属性）と処理（振る舞い）を持つひとつのまとまりとして扱います。

オブジェクト指向の3つの主要概念

  • カプセル化: データと処理をひとつにまとめ、外部から不適切なアクセスを防ぐ
  • 継承: 既存のクラスを基に新しいクラスを作成し、コードを再利用する
  • ポリモーフィズム（多態性）: 同じ名前のメソッドが異なる振る舞いをする

この章では、オブジェクト指向の基礎であるクラスとオブジェクト（インスタンス）について理解していきます。

8.2 クラスとオブジェクト（インスタンス）

クラスは、オブジェクトの「設計図」または「型」です。設計図にはそのオブジェクトが持つべきデータ（フィールド）と処理（メソッド）が定義されています。

オブジェクト（インスタンス）は、クラスの設計図をもとに実際に作られた「もの」です。クラスという設計図から複数のオブジェクトを作成できます。

クラスとオブジェクトの比喩

クラスを「たい焼きの型」、オブジェクトを「実際のたい焼き」に例えることができます。同じ型（クラス）から複数のたい焼き（オブジェクト）を作ることができ、それぞれのたい焼きは異なる中身（データ）を持つことができます。

8.3 クラスの定義

クラスを定義する基本的な構文を学びましょう。ここでは「Car（車）」クラスを例に説明します。

Car.java 
public class Car {
    // フィールド（属性・データ）
    String color;      // 車の色
    int speed;         // 現在の速度
    String model;      // 車種名

    // メソッド（振る舞い・処理）
    void accelerate() {
        // 加速する
        speed += 10;
        System.out.println("加速しました。現在の速度: " + speed + "km/h");
    }

    void brake() {
        // ブレーキをかける
        speed -= 10;
        if (speed < 0) speed = 0;
        System.out.println("減速しました。現在の速度: " + speed + "km/h");
    }
}

クラスの構成要素

  • フィールド（Field）

    オブジェクトが持つデータ（属性・状態）を表す変数。例: color, speed, model

  • メソッド（Method）

    オブジェクトが行う処理（振る舞い・機能）を表す。例: accelerate(), brake()

8.4 オブジェクトの生成とフィールドへのアクセス

クラスの設計図ができたら、new演算子を使ってオブジェクト（インスタンス）を生成します。

Main.java 
public class Main {
    public static void main(String[] args) {
        // Carクラスからオブジェクトを生成
        Car myCar = new Car();

        // フィールドに値を設定
        myCar.color = "赤";
        myCar.model = "セダン";
        myCar.speed = 0;

        // フィールドの値を表示
        System.out.println("車種: " + myCar.model);
        System.out.println("色: " + myCar.color);
        System.out.println("速度: " + myCar.speed + "km/h");
    }
}

オブジェクト生成の構文

クラス名 変数名 = new クラス名();
  • Car myCar

    Car型の変数myCarを宣言（参照変数）

  • new Car()

    new演算子でCarオブジェクトを生成し、メモリに確保

  • myCar.color

    ドット演算子（.）を使ってオブジェクトのフィールドにアクセス

重要ポイント

同じクラスから複数のオブジェクトを生成できます。それぞれのオブジェクトは独立したフィールドの値を持ちます。

8.5 メソッドの呼び出し

オブジェクトが持つメソッドを呼び出すには、ドット演算子を使います。

Main.java 
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.color = "赤";
        myCar.model = "セダン";
        myCar.speed = 0;

        // メソッドを呼び出す
        myCar.accelerate();  // 加速
        myCar.accelerate();  // さらに加速
        myCar.brake();       // ブレーキ
    }
}

実行結果

加速しました。現在の速度: 10km/h
加速しました。現在の速度: 20km/h
減速しました。現在の速度: 10km/h

8.6 コンストラクタ

コンストラクタは、オブジェクトが生成されるときに自動的に呼び出される特別なメソッドです。オブジェクトの初期化（フィールドに初期値を設定する）を行います。

コンストラクタの特徴

  • クラス名と同じ名前を持つ
  • 戻り値の型を書かない（void も書かない）
  • newでオブジェクトを生成したときに自動的に呼ばれる
Car.java（コンストラクタ追加版） 
public class Car {
    String color;
    int speed;
    String model;

    // コンストラクタ（引数あり）
    public Car(String modelName, String carColor) {
        model = modelName;
        color = carColor;
        speed = 0;  // 初期速度は0
        System.out.println(model + "(" + color + ")を生成しました");
    }

    void accelerate() {
        speed += 10;
        System.out.println("加速しました。現在の速度: " + speed + "km/h");
    }

    void brake() {
        speed -= 10;
        if (speed < 0) speed = 0;
        System.out.println("減速しました。現在の速度: " + speed + "km/h");
    }
}

コンストラクタを使ったオブジェクト生成

Main.java 
public class Main {
    public static void main(String[] args) {
        // コンストラクタで初期化
        Car car1 = new Car("セダン", "赤");
        Car car2 = new Car("SUV", "青");

        car1.accelerate();
        car2.accelerate();
    }
}

実行結果

セダン(赤)を生成しました
SUV(青)を生成しました
加速しました。現在の速度: 10km/h
加速しました。現在の速度: 10km/h

デフォルトコンストラクタ

コンストラクタを定義しない場合、Javaが自動的に引数なしの「デフォルトコンストラクタ」を用意します。しかし、引数ありのコンストラクタを定義した場合、デフォルトコンストラクタは自動生成されなくなります。

8.7 thisキーワード

thisキーワードは「このオブジェクト自身」を指します。特にコンストラクタやメソッドの引数名とフィールド名が同じ場合に使います。

Car.java（thisキーワード使用） 
public class Car {
    String color;
    int speed;
    String model;

    // thisを使ってフィールドと引数を区別
    public Car(String model, String color) {
        this.model = model;  // this.modelはフィールド
        this.color = color;  // modelとcolorは引数
        this.speed = 0;
        System.out.println(this.model + "を生成しました");
    }

    void showInfo() {
        // メソッド内でもthisを使える
        System.out.println("車種: " + this.model);
        System.out.println("色: " + this.color);
        System.out.println("速度: " + this.speed + "km/h");
    }
}

thisの利点

thisを使うと、フィールドと引数の名前を同じにできるため、コードがわかりやすくなります。また、明示的に「このオブジェクトのフィールド」であることを示せます。

実習 8-1: Bookクラスを作ろう

書籍を表すBookクラスを作成し、オブジェクト指向の基礎を実践しましょう。

要件

  • フィールド: title（タイトル）、author（著者）、price（価格）
  • コンストラクタ: 3つのフィールドを初期化
  • メソッド: showInfo()で書籍情報を表示

実装例

public class Book {
    String title;
    String author;
    int price;

    // コンストラクタ
    public Book(String title, String author, int price) {
        this.title = title;
        this.author = author;
        this.price = price;
    }

    // 書籍情報を表示するメソッド
    void showInfo() {
        System.out.println("タイトル: " + title);
        System.out.println("著者: " + author);
        System.out.println("価格: " + price + "円");
        System.out.println("---");
    }
}

// 使用例
public class Main {
    public static void main(String[] args) {
        Book book1 = new Book("Javaの基本", "山田太郎", 2500);
        Book book2 = new Book("プログラミング入門", "佐藤花子", 3000);

        book1.showInfo();
        book2.showInfo();
    }
}

期待される結果

著者: 山田太郎
価格: 2500円
---
タイトル: プログラミング入門
著者: 佐藤花子
価格: 3000円
---

理解度確認クイズ

  1. Q1. クラスとオブジェクト（インスタンス）の違いを説明してください。

    回答を表示

    クラスはオブジェクトの「設計図」または「型」であり、フィールドとメソッドを定義します。オブジェクト（インスタンス）は、クラスをもとにnew演算子で実際に生成された「もの」で、独立したデータを持ちます。同じクラスから複数のオブジェクトを作成できます。

  2. Q2. コンストラクタの役割と特徴を3つ述べてください。

    回答を表示

    役割: オブジェクトの初期化を行う
    特徴1: クラス名と同じ名前を持つ
    特徴2: 戻り値の型を書かない
    特徴3: newでオブジェクトを生成したときに自動的に呼ばれる

  3. Q3. thisキーワードの用途を説明してください。

    回答を表示

    thisは「このオブジェクト自身」を指すキーワードです。主に以下の用途で使います：
    1. コンストラクタやメソッドの引数名とフィールド名が同じ場合に、フィールドを明示的に区別する（this.field = field）
    2. 自オブジェクトのメソッドやフィールドにアクセスする際に、明示的に示す

  4. Q4. フィールドとメソッドの違いを説明してください。

    回答を表示

    フィールド: オブジェクトが持つデータ（属性・状態）を表す変数。例: color, speed, model
    メソッド: オブジェクトが行う処理（振る舞い・機能）を表す。例: accelerate(), showInfo()
    フィールドは「状態」、メソッドは「動作」を表現します。

この章のまとめ

  • オブジェクト指向プログラミングは、データと処理をまとめて管理する手法
  • クラスは設計図、オブジェクト（インスタンス）は実際に生成されたもの
  • フィールドはデータ（属性）、メソッドは処理（振る舞い）を表す
  • new演算子でオブジェクトを生成し、ドット演算子でフィールドやメソッドにアクセス
  • コンストラクタはオブジェクト生成時に自動的に呼ばれる初期化用のメソッド
  • thisキーワードは自オブジェクトを指し、フィールドと引数の区別に使える
