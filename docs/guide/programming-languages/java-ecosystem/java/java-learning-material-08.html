オブジェクトの中身を理解する - フィールドとメソッド

クラスの中に定義する「フィールド」（データを保存する場所）と「メソッド」（動作・機能）について詳しく学び、コンストラクタの役割も理解しましょう。

この章で学ぶこと

  • フィールドとは何か、どのようにオブジェクトのデータを保存するかを理解する
  • メソッドとは何か、オブジェクトの動作をどのように定義するかを理解する
  • コンストラクタの役割と、オブジェクトの初期化方法を学ぶ

8.1 フィールド - オブジェクトの「データ置き場」

前章でクラスとオブジェクトの基本を学びましたが、この章ではオブジェクトの内部構造にもっと深く入り込みます。 まず、フィールド（field）について学びましょう。

フィールドとは？

フィールドは、クラスの中に定義する変数のことです。 各オブジェクトが持つ「データ」や「状態」を保存する場所として使われます。 たとえば、Carクラスなら「色」「車種名」「走行距離」などがフィールドになります。

フィールドの定義方法

フィールドは、クラスの中（メソッドの外）に変数として定義します。

Car.java 
public class Car {
    // フィールド（クラスの中、メソッドの外に定義）
    String color;       // 車の色
    String name;        // 車種名
    int mileage;        // 走行距離（km）

    // メソッドはこの後で定義
}

フィールドとオブジェクトの関係

classDiagram class Car { +String color +String name +int mileage } note for Car "各Carオブジェクトは\nこれらのデータを持つ"

フィールドへのアクセス

オブジェクトを作成すると、そのオブジェクトごとにフィールドが用意されます。 フィールドにアクセスするには、オブジェクト名.フィールド名の形式を使います。

Main.java 
public class Main {
    public static void main(String[] args) {
        // Carオブジェクトを作成
        Car myCar = new Car();

        // フィールドに値を設定
        myCar.color = "赤";
        myCar.name = "スポーツカー";
        myCar.mileage = 5000;

        // フィールドの値を表示
        System.out.println("車の色: " + myCar.color);
        System.out.println("車種: " + myCar.name);
        System.out.println("走行距離: " + myCar.mileage + "km");
    }
}

ポイント

  • ・各オブジェクトは独立したフィールドを持つため、別のCarオブジェクトには影響しません
  • ・フィールドは初期値を設定しなければデフォルト値が入ります（数値は0、文字列はnull）

8.2 メソッド - オブジェクトの「動作」を定義する

フィールドがデータを保存するのに対し、メソッド（method）はオブジェクトの「動作」や「機能」を定義します。 第6章でメソッドの基本を学びましたが、ここではクラスの中でどのように使うかを学びます。

メソッドとは？

メソッドは、オブジェクトが「何をするか」を定義した処理のまとまりです。 たとえば、Carクラスなら「走る」「止まる」「情報を表示する」などの動作をメソッドで定義します。

メソッドの定義方法

Car.java（フィールドとメソッド） 
public class Car {
    // フィールド
    String color;
    String name;
    int mileage;

    // メソッド: 車を走らせる
    void drive(int km) {
        mileage = mileage + km;  // 走行距離を加算
        System.out.println(km + "km走りました");
    }

    // メソッド: 車の情報を表示する
    void showInfo() {
        System.out.println("=== 車の情報 ===");
        System.out.println("色: " + color);
        System.out.println("車種: " + name);
        System.out.println("走行距離: " + mileage + "km");
    }
}

メソッドの呼び出し

メソッドを呼び出すには、オブジェクト名.メソッド名()の形式を使います。

Main.java 
public class Main {
    public static void main(String[] args) {
        // Carオブジェクトを作成
        Car myCar = new Car();
        myCar.color = "青";
        myCar.name = "セダン";
        myCar.mileage = 1000;

        // メソッドを呼び出す
        myCar.showInfo();        // 情報を表示
        myCar.drive(50);         // 50km走る
        myCar.showInfo();        // もう一度情報を表示
    }
}

走行距離: 1000km
50km走りました

実習 8-1: Dogクラスを作ろう

フィールドとメソッドを持つDogクラスを作成し、オブジェクトを操作してみましょう。

仕様

  • フィールド: String name（名前）、int age（年齢）
  • メソッド: void bark()（吠える）、void showInfo()（情報表示）
Dog.java 
public class Dog {
    // フィールド
    String name;
    int age;

    // メソッド: 吠える
    void bark() {
        System.out.println(name + "がワンワン！と吠えました");
    }

    // メソッド: 情報を表示
    void showInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
    }
}
Main.java 
public class Main {
    public static void main(String[] args) {
        // Dogオブジェクトを作成
        Dog myDog = new Dog();
        myDog.name = "ポチ";
        myDog.age = 3;

        // メソッドを呼び出す
        myDog.showInfo();
        myDog.bark();
    }
}

期待される結果

名前: ポチ
年齢: 3歳
ポチがワンワン！と吠えました

8.3 コンストラクタ - オブジェクトの初期化

これまでは、オブジェクトを作った後に一つ一つフィールドに値を設定していました。 もっと便利な方法として、コンストラクタ（constructor）というものがあります。

コンストラクタとは？

コンストラクタは、オブジェクトを作成（new）するときに自動的に実行される特別なメソッドです。 オブジェクトの初期設定（フィールドに初期値を設定する）をするために使います。

コンストラクタの定義方法

コンストラクタは、クラス名と同じ名前で、戻り値の型を書かないメソッドとして定義します。

Dog.java（コンストラクタ付き） 
public class Dog {
    // フィールド
    String name;
    int age;

    // コンストラクタ（クラス名と同じ、戻り値の型を書かない）
    Dog(String n, int a) {
        name = n;   // 引数で受け取った値をフィールドに設定
        age = a;
        System.out.println(name + "が生まれました！");
    }

    // メソッド
    void bark() {
        System.out.println(name + "がワンワン！と吠えました");
    }

    void showInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
    }
}

コンストラクタの使い方

Main.java 
public class Main {
    public static void main(String[] args) {
        // newの時に引数を渡す（コンストラクタが呼ばれる）
        Dog dog1 = new Dog("ポチ", 3);
        Dog dog2 = new Dog("タロー", 5);

        dog1.showInfo();
        dog2.showInfo();
    }
}

実行結果

ポチが生まれました！
タローが生まれました！
名前: ポチ
年齢: 3歳
名前: タロー
年齢: 5歳

コンストラクタのメリット

  • ・オブジェクトを作るときに一度に初期値を設定できる
  • ・フィールドの設定し忘れを防げる
  • ・コードがシンプルで読みやすくなる

デフォルトコンストラクタ

コンストラクタを一つも書かなかった場合、Javaが自動的に引数なしのコンストラクタ（デフォルトコンストラクタ）を用意してくれます。 しかし、引数ありのコンストラクタを1つでも定義すると、デフォルトコンストラクタは自動生成されなくなります。

注意

引数ありのコンストラクタを定義した後で、new Dog()のように引数なしで呼び出すとエラーになります。 両方使いたい場合は、引数なしのコンストラクタも明示的に定義する必要があります。

8.4 thisキーワード - 自分自身を指す

コンストラクタやメソッドの引数名とフィールド名が同じになることがよくあります。 そのような場合、thisキーワードを使って区別します。

thisキーワードとは？

thisは「このオブジェクト自身」を指すキーワードです。 this.nameと書くと、「このオブジェクトのnameフィールド」という意味になります。

Dog.java（thisを使った例） 
public class Dog {
    // フィールド
    String name;
    int age;

    // コンストラクタ（引数名とフィールド名が同じ）
    Dog(String name, int age) {
        this.name = name;  // this.nameはフィールド、nameは引数
        this.age = age;    // this.ageはフィールド、ageは引数
        System.out.println(this.name + "が生まれました！");
    }

    void showInfo() {
        System.out.println("名前: " + this.name);
        System.out.println("年齢: " + this.age + "歳");
    }
}

ポイント

thisを使うことで、引数名とフィールド名を同じにでき、コードが読みやすくなります。 実際の開発現場でも、thisを使ったコンストラクタが一般的です。

実習 8-2: Bookクラスを作ろう

コンストラクタとthisキーワードを使ってBookクラスを作成しましょう。

仕様

  • フィールド: String title（タイトル）、String author（著者）、int price（価格）
  • コンストラクタ: 3つの引数を受け取り、thisを使ってフィールドに設定
  • メソッド: void showInfo()（本の情報を表示）
Book.java 
public class Book {
    // フィールド
    String title;
    String author;
    int price;

    // コンストラクタ
    Book(String title, String author, int price) {
        this.title = title;
        this.author = author;
        this.price = price;
    }

    // メソッド
    void showInfo() {
        System.out.println("=== 本の情報 ===");
        System.out.println("タイトル: " + this.title);
        System.out.println("著者: " + this.author);
        System.out.println("価格: " + this.price + "円");
    }
}
Main.java 
public class Main {
    public static void main(String[] args) {
        // Bookオブジェクトを作成
        Book book1 = new Book("Java入門", "山田太郎", 2500);
        Book book2 = new Book("Python基礎", "佐藤花子", 2800);

        book1.showInfo();
        System.out.println();
        book2.showInfo();
    }
}

理解度確認クイズ

  1. Q1. フィールドとは何ですか？

    回答を表示

    フィールドは、クラスの中に定義する変数のことで、各オブジェクトが持つデータや状態を保存する場所です。 たとえば、Carクラスなら「色」「車種名」「走行距離」などがフィールドになります。

  2. Q2. メソッドとフィールドの違いは何ですか？

    回答を表示

    フィールドはオブジェクトの「データ」を保存する場所で、メソッドはオブジェクトの「動作」を定義する処理のまとまりです。 フィールドは「何を持っているか」、メソッドは「何をするか」を表します。

  3. Q3. コンストラクタとは何ですか？いつ実行されますか？

    回答を表示

    コンストラクタは、オブジェクトを作成（new）するときに自動的に実行される特別なメソッドです。 クラス名と同じ名前で定義し、オブジェクトの初期設定（フィールドに初期値を設定する）を行います。

  4. Q4. thisキーワードは何のために使いますか？

    回答を表示

    thisキーワードは「このオブジェクト自身」を指します。 特に、コンストラクタやメソッドの引数名とフィールド名が同じ場合、 this.nameのように書くことでフィールドと引数を区別できます。

8.5 この章のまとめ

  • フィールドは、クラスの中に定義する変数で、オブジェクトのデータを保存する。
  • メソッドは、オブジェクトの動作や機能を定義する処理のまとまり。
  • フィールドとメソッドを組み合わせることで、意味のあるオブジェクトを作成できる。
  • コンストラクタは、オブジェクト作成時に自動的に実行され、初期化を行う。
  • thisキーワードを使うことで、フィールドと引数を明確に区別できる。
  • コンストラクタを使えば、オブジェクト作成時に必要な情報を一度に設定できる。
