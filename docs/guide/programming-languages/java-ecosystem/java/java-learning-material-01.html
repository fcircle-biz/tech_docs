第1章

Javaの世界へようこそ - 環境構築と最初の一歩

Javaとは何か、なぜ世界中で使われているのかを理解し、開発環境のセットアップから最初のプログラム「Hello World」を作成・実行するまでを学びます。

この章で学ぶこと

  • Javaプログラミング言語の特徴と用途を理解する
  • JDKのインストールと環境変数の設定ができる
  • テキストエディタまたはIDEでJavaプログラムを作成できる
  • コンパイルと実行の仕組みを理解し、Hello Worldプログラムを動かせる

1.1 Javaとは何か

Java（ジャバ）は、1995年にサン・マイクロシステムズ（現在はOracle社）によって開発されたプログラミング言語です。現在では世界中で広く使われており、企業の業務システム、Androidアプリ、Webアプリケーションなど、様々な分野で活躍しています。

「Write Once, Run Anywhere」の理念

Javaの最大の特徴は「一度書けば、どこでも動く（Write Once, Run Anywhere）」という理念です。これは、Javaで書かれたプログラムが、Windows、Mac、Linuxなど、異なるOS（オペレーティングシステム）上でも同じように動作することを意味します。

なぜどこでも動くのか？

一般的なプログラミング言語は、各OS専用のプログラムに変換（コンパイル）されるため、Windowsで作ったプログラムはMacでは動きません。しかしJavaは、「Java仮想マシン（JVM）」という特別なソフトウェアの上で動作します。JVMがあればどのOSでも動くため、Javaプログラムは環境に依存せず実行できます。

JDK、JRE、JVMの違い

Javaを学習する上で、以下の3つの用語を理解しておくことが重要です。

JVM（Java Virtual Machine - Java仮想マシン）

Javaプログラムを実際に実行するソフトウェア。バイトコード（.classファイル）を解釈して、各OSで動作するように変換します。

JRE（Java Runtime Environment - Java実行環境）

JVMと、Javaプログラムを動かすために必要な基本的なライブラリ（便利な部品集）をまとめたもの。Javaプログラムを「実行する」だけなら、JREがあれば十分です。

JDK（Java Development Kit - Java開発キット）

JREに加えて、Javaプログラムを「開発する」ための道具（コンパイラ、デバッガなど）が含まれています。Javaでプログラムを作る場合は、JDKをインストールします。

覚え方のヒント

JDK ⊃ JRE ⊃ JVM という包含関係を覚えましょう。JDKには全てが含まれており、プログラムを作る人（開発者）はJDKをインストールします。

Javaが使われている分野

Javaは以下のような様々な分野で活用されています。

  • 企業の業務システム

    銀行、保険、製造業など、多くの企業の基幹システムがJavaで構築されています。

  • Androidアプリ開発

    Androidスマートフォンのアプリは、主にJavaまたはKotlin（Javaの仲間）で開発されます。

  • Webアプリケーション

    Spring Boot、Jakarta EE（旧Java EE）などのフレームワークを使い、ECサイトやSNSなどのWebサービスを開発できます。

  • 組み込みシステム・IoT

    家電製品や産業機器の制御プログラムにもJavaが使われることがあります。

1.2 開発環境のセットアップ

Javaプログラムを作成・実行するには、まずJDKをインストールする必要があります。ここでは、Windows・Macそれぞれの環境でのインストール手順を説明します。

手順1: JDKのダウンロード

Oracle公式サイトまたはAdoptium（旧AdoptOpenJDK）などのディストリビューションから、JDK 17以降をダウンロードします。初学者にはAdoptiumが推奨されます。

ダウンロード先

  • • Adoptium: https://adoptium.net/
  • • Oracle JDK: https://www.oracle.com/java/technologies/downloads/

手順2: JDKのインストール（Windows）

  1. 1 ダウンロードした.msiファイルを実行します。
  2. 2 インストーラの指示に従い、「次へ」をクリックして進めます。
  3. 3 インストール先のフォルダを確認し、デフォルトのまま（例: C:\Program Files\Eclipse Adoptium\jdk-17.0.x）でOKです。
  4. 4 インストールが完了したら「完了」をクリックします。

手順3: JDKのインストール（Mac）

  1. 1 ダウンロードした.pkgファイルをダブルクリックします。
  2. 2 インストーラの指示に従い、「続ける」をクリックして進めます。
  3. 3 ライセンス同意後、インストール先のディスクを選択し、「インストール」をクリックします。
  4. 4 管理者パスワードを入力し、インストールを完了します。

手順4: 環境変数の設定（Windows）

Javaコマンドをどこからでも実行できるよう、環境変数を設定します。

  1. 1 「コントロールパネル」→「システム」→「システムの詳細設定」→「環境変数」を開きます。
  2. 2 「システム環境変数」の「新規」をクリックし、以下を追加します。
    変数名: JAVA_HOME
    変数値: JDKのインストールパス（例: C:\Program Files\Eclipse Adoptium\jdk-17.0.x）
  3. 3 「Path」変数を選択し、「編集」をクリック。「新規」で %JAVA_HOME%\bin を追加します。
  4. 4 「OK」を押して全てのウィンドウを閉じます。

Macの環境変数設定

Macの場合、ほとんどのインストーラが自動的に環境変数を設定してくれますが、手動で設定する場合は、ターミナルで以下を実行します。 

echo 'export JAVA_HOME=$(/usr/libexec/java_home)' >> ~/.zshrc
source ~/.zshrc

手順5: インストールの確認

正しくインストールされたか、コマンドプロンプト（Windowsの場合）またはターミナル（Macの場合）で確認しましょう。

ターミナル / コマンドプロンプト 
# Javaのバージョン確認
java -version

# Javaコンパイラのバージョン確認
javac -version

以下のような出力が表示されれば、インストール成功です。 

openjdk version "17.0.8" 2023-07-18
OpenJDK Runtime Environment Temurin-17.0.8+7 (build 17.0.8+7)
OpenJDK 64-Bit Server VM Temurin-17.0.8+7 (build 17.0.8+7, mixed mode)

手順6: IDEの選択と基本設定（推奨：VS Code）

Javaプログラムはメモ帳でも書けますが、IDE（統合開発環境）を使うと効率的です。初心者には軽量で使いやすいVS Code（Visual Studio Code）がおすすめです。

  1. 1 VS Code公式サイトからダウンロードし、インストールします。
  2. 2 VS Codeを開き、左側の「拡張機能」アイコンをクリックします。
  3. 3 検索ボックスに「Extension Pack for Java」と入力し、インストールします（Microsoft提供）。
  4. 4 これでJavaの開発環境が整いました！

1.3 最初のプログラム - Hello World

それでは、プログラミングの定番である「Hello World」プログラムを作成してみましょう。このプログラムは、画面に「Hello, World!」というメッセージを表示するだけのシンプルなものです。

ソースファイルの作成

まず、適当なフォルダ（例: C:\java_practiceまたは~/java_practice）を作成し、その中に以下の内容でHelloWorld.javaというファイルを作成します。

HelloWorld.java 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

重要な注意点

ファイル名は必ずHelloWorld.javaとし、クラス名（public class HelloWorldのHelloWorld）と完全に一致させてください。大文字・小文字も区別されます。Javaでは、ファイル名とクラス名が一致していないとエラーになります。

プログラムの基本構造を理解しよう

上記のプログラムの各部分の意味を見ていきましょう。

public class HelloWorld

クラスの宣言です。Javaのプログラムは必ず「クラス」という単位で構成されます。HelloWorldはクラスの名前です。

public static void main(String[] args)

mainメソッドです。Javaプログラムが起動すると、最初にこのmainメソッドが呼び出されます。プログラムのスタート地点と考えてください。

System.out.println("Hello, World!");

画面に文字を表示する命令です。printlnは「print line（行を印刷）」の略で、指定した文字列を画面に表示し、改行します。文字列は"（ダブルクォーテーション）で囲みます。命令の最後には;（セミコロン）を付けます。

{ } （波括弧）

ブロックの範囲を示します。クラスやメソッドの内容は{と}で囲みます。開始の{と終了の}は必ず対応している必要があります。

コンパイルと実行の流れ

Javaプログラムは、以下の2つのステップで実行されます。

HelloWorld.java] -->|javac コマンド
コンパイル| B[バイトコード
HelloWorld.class] B -->|java コマンド
実行| C[結果出力
Hello, World!] style A fill:#fff7ed,stroke:#ea580c style B fill:#fef3c7,stroke:#f59e0b style C fill:#d1fae5,stroke:#10b981

ステップ1: コンパイル（javacコマンド）

人間が読めるJavaソースコード（.javaファイル）を、JVMが理解できるバイトコード（.classファイル）に変換します。

コマンドプロンプト / ターミナル 
# HelloWorld.javaがあるフォルダに移動
cd C:\java_practice  # Windowsの場合
# cd ~/java_practice  # Macの場合

# コンパイル実行
javac HelloWorld.java

エラーが出なければ、同じフォルダにHelloWorld.classというファイルが生成されます。

ステップ2: 実行（javaコマンド）

生成されたバイトコードをJVMで実行します。

コマンドプロンプト / ターミナル 
# 実行（.classは付けない、クラス名のみ）
java HelloWorld

正しく実行されると、以下のように表示されます。 

Hello, World!

ヒント

コンパイル時はjavac HelloWorld.javaのように.javaを付ける、実行時はjava HelloWorldのように.classは付けない（クラス名だけ）という点に注意しましょう。

よくあるエラーとその対処法

トラブルシューティング

エラー: javac: command not found

原因: 環境変数のPATH設定が正しくありません。

解決策: 環境変数の設定を見直し、%JAVA_HOME%\bin（Windows）または$JAVA_HOME/bin（Mac）がPATHに含まれているか確認してください。

エラー: class HelloWorld is public, should be declared in a file named HelloWorld.java

原因: ファイル名とクラス名が一致していません。

解決策: ファイル名をHelloWorld.java（大文字・小文字を含め完全一致）に修正してください。

エラー: Error: Could not find or load main class HelloWorld

原因: .classファイルが生成されていない、または実行時のディレクトリが間違っています。

解決策: まずjavac HelloWorld.javaでコンパイルし、.classファイルが生成されていることを確認してから、同じディレクトリでjava HelloWorldを実行してください。

日本語が文字化けする

原因: ソースファイルの文字コードがUTF-8になっていません。

解決策: エディタでファイルをUTF-8形式で保存し直してください。VS Codeでは右下のステータスバーから文字コードを変更できます。

理解度確認クイズ

  1. Q1. Javaの「Write Once, Run Anywhere」とはどういう意味ですか？

    回答を表示

    一度Javaプログラムを書けば、Java仮想マシン（JVM）が動作するどのOS（Windows、Mac、Linuxなど）でも、同じプログラムを再コンパイルせずに実行できるという意味です。これは、JavaがOS固有のコードではなく、JVM上で動作するバイトコードにコンパイルされるためです。

  2. Q2. JDK、JRE、JVMの違いを簡単に説明してください。

    回答を表示

    JVM: Javaプログラムを実行するための仮想マシン。
    JRE: JVM + Javaプログラムの実行に必要なライブラリ。Javaプログラムを実行するだけならこれで十分。
    JDK: JRE + 開発ツール（コンパイラ、デバッガなど）。Javaプログラムを開発するにはJDKが必要。

  3. Q3. Javaプログラムをコンパイルするコマンドと、実行するコマンドは何ですか？

    回答を表示

    コンパイル: javac ファイル名.java（例: javac HelloWorld.java）
    実行: java クラス名（例: java HelloWorld、.classは付けない）

  4. Q4. Javaのソースファイルとクラス名の関係について、正しいものはどれですか？

    • A. ファイル名とクラス名は異なっていても良い
    • B. ファイル名は小文字、クラス名は大文字で始める必要がある
    • C. publicクラスの場合、ファイル名とクラス名は完全に一致する必要がある
    • D. ファイル名に.classを付ける必要がある
    回答を表示

    正解: C
    publicクラスの場合、ファイル名とクラス名は大文字・小文字を含めて完全に一致している必要があります。例えば、public class HelloWorldであれば、ファイル名はHelloWorld.javaでなければなりません。

  5. Q5. System.out.println("Hello");の役割は何ですか？

    回答を表示

    画面（標準出力）に「Hello」という文字列を表示し、その後改行します。printlnは「print line」の略で、出力後に自動的に改行が行われます。

この章のまとめ

  • Javaは「Write Once, Run Anywhere」の理念を持ち、JVMによって様々なOS上で動作する
  • JDKには開発に必要な全てのツールが含まれており、開発者はJDKをインストールする
  • Javaプログラムは.javaファイルをjavacでコンパイルし、javaコマンドで実行する
  • publicクラスのファイル名とクラス名は完全に一致している必要がある
  • mainメソッドがプログラムのエントリーポイント（開始地点）となる
