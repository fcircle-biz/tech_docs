第1章

Javaの世界へようこそ - 環境構築と最初の一歩

Javaとは何か、なぜ世界中で使われているのかを理解し、開発環境のセットアップから最初のプログラム「Hello World」を作成・実行するまでを学びます。

この章で学ぶこと

  • Javaプログラミング言語の特徴と用途を理解する
  • JDKのインストールと環境変数の設定ができる
  • テキストエディタまたはIDEでJavaプログラムを作成できる
  • コンパイルと実行の仕組みを理解し、Hello Worldプログラムを動かせる

1.1 Javaとは何か

Javaの歴史と「Write Once, Run Anywhere」の理念

Javaは1995年にSun Microsystems（現在はOracle Corporation）によって公開されたプログラミング言語です。当初は家電製品に組み込むための言語として開発されましたが、インターネットの普及とともにWebアプリケーション開発で広く使われるようになりました。

Javaの最大の特徴は「Write Once, Run Anywhere（一度書けば、どこでも動く）」という理念です。これは、Javaプログラムを一度書けば、Windows、macOS、Linuxなど異なるオペレーティングシステム上で同じプログラムが動作することを意味します。

なぜ「どこでも動く」のか？

Javaプログラムは、JVM（Java仮想マシン）という特殊なソフトウェア上で動作します。各OS用のJVMが存在するため、同じJavaプログラムがどのOS上でも動作できるのです。これは、他の多くのプログラミング言語にはない大きな利点です。

JVM（Java仮想マシン）の役割と仕組み

JVM（Java Virtual Machine）は、Javaプログラムを実行するための仮想的なコンピュータです。Javaプログラムは人間が読めるソースコードから「バイトコード」という中間言語に変換され、このバイトコードをJVMが解釈・実行します。

JDK、JRE、JVMの違い

Java開発では、以下の3つの用語がよく出てきます。それぞれの違いを理解しましょう。

用語 正式名称 説明
JVM Java Virtual Machine Javaプログラムを実行するための仮想マシン
JRE Java Runtime Environment Javaプログラムを実行するための環境（JVM + 標準ライブラリ）
JDK Java Development Kit Java開発に必要なツール一式（JRE + コンパイラ + デバッガなど）

開発にはJDKが必要

Javaプログラムを作成するには、コンパイラなどの開発ツールが含まれるJDKが必要です。JREだけではプログラムを実行できても、新しくプログラムを作成することはできません。

Javaが使われている分野

Javaは世界中で広く使われており、さまざまな分野で活躍しています。

Webアプリケーション

企業の業務システムやECサイトなど、大規模なWebアプリケーションで広く使われています（Spring Boot、Strutsなど）。

Android アプリ

Androidスマートフォンのアプリ開発には、長年Javaが使われてきました（現在はKotlinも人気）。

業務システム

銀行、保険、製造業などの基幹システムで、安定性と信頼性が重視される場面で活躍しています。

クラウド・ビッグデータ

Hadoop、Spark、Kafkaなど、大規模データ処理のフレームワークの多くがJavaで書かれています。

1.2 開発環境のセットアップ

JDKのダウンロードとインストール

Java開発を始めるには、まずJDK（Java Development Kit）をインストールする必要があります。Oracle JDKやOpenJDKなど複数の選択肢がありますが、ここではOracle JDKのインストール手順を説明します。

JDKダウンロード手順

  1. 1 Oracle公式サイト（oracle.com/java/）にアクセス
  2. 2 「Java SE」のダウンロードページへ移動
  3. 3 最新のLTS（Long Term Support）版を選択（例：JDK 17、JDK 21）
  4. 4 使用しているOS（Windows、macOS、Linux）に合ったインストーラをダウンロード
  5. 5 ダウンロードしたインストーラを実行し、画面の指示に従ってインストール

環境変数の設定

JDKをインストールした後、コマンドラインからjavaやjavacコマンドを使えるようにするため、環境変数を設定する必要があります。

Windows での環境変数設定

  1. 1. 「システム」の「システムの詳細設定」を開く
  2. 2. 「環境変数」ボタンをクリック
  3. 3. 「システム環境変数」の「新規」をクリック
  4. 4. 変数名: JAVA_HOME
  5. 5. 変数値: JDKのインストールパス（例: C:\Program Files\Java\jdk-17）
  6. 6. 「Path」変数を編集し、%JAVA_HOME%\bin を追加

macOS での環境変数設定

ターミナルを開き、以下のコマンドを実行します（使用しているシェルに応じて設定ファイルが異なります）。

~/.zshrc または ~/.bash_profile 
# JDKのパスを設定（インストール場所に応じて変更）
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home
export PATH=$JAVA_HOME/bin:$PATH

インストールの確認方法

JDKが正しくインストールされ、環境変数が設定されているかを確認しましょう。

ターミナル/コマンドプロンプト 
# Javaランタイムのバージョン確認
java -version

# Javaコンパイラのバージョン確認
javac -version

正しくインストールされていれば、以下のような出力が表示されます。

java version "17.0.8" 2023-07-18 LTS
Java(TM) SE Runtime Environment (build 17.0.8+9-LTS-211)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.8+9-LTS-211, mixed mode, sharing)

1.3 最初のプログラム - Hello World

ソースファイル（.java）の作成

それでは、初めてのJavaプログラムを作成しましょう。まず、HelloWorld.javaという名前のファイルを作成します。

重要：ファイル名の命名規則

Javaでは、ファイル名とクラス名が完全に一致している必要があります。また、大文字・小文字も区別されます。クラス名がHelloWorldなら、ファイル名もHelloWorld.javaでなければなりません。

プログラムの基本構造

以下は、画面に「Hello, World!」と表示する最もシンプルなJavaプログラムです。

HelloWorld.java 
// クラスの定義
public class HelloWorld {
    // mainメソッド（プログラムの開始点）
    public static void main(String[] args) {
        // 画面に文字列を表示
        System.out.println("Hello, World!");
    }
}

このプログラムの各部分を詳しく見ていきましょう。

public class HelloWorld

クラスの定義です。Javaではすべてのコードがクラスの中に書かれます。publicは「公開」を意味し、他の場所からもこのクラスにアクセスできることを示します。

public static void main(String[] args)

mainメソッドは、Javaプログラムの「入り口」です。プログラムを実行すると、JVMは最初にこのmainメソッドを探して実行します。この書き方は決まった形式なので、最初は暗記してしまいましょう。

System.out.println("...")

画面（標準出力）に文字列を表示するための命令です。printlnは「print line（行を印刷）」の略で、表示後に改行します。文字列はダブルクォーテーション（"）で囲みます。

// コメント

//で始まる行はコメントで、プログラムの動作には影響しません。コードの説明を書くために使います。

Javaプログラムを実行するには、「コンパイル」と「実行」の2つのステップが必要です。

実行手順

1. コンパイル

javac HelloWorld.java

これにより、HelloWorld.classファイル（バイトコード）が生成されます。

2. 実行

java HelloWorld

実行時は.classを付けません。クラス名のみを指定します。

3. 出力結果

Hello, World!

よくあるエラーとその対処法

エラー: javac: command not found

原因: JDKがインストールされていないか、環境変数の設定が間違っています。

対処: JDKのインストールと環境変数（JAVA_HOME、PATH）の設定を確認してください。

エラー: class HelloWorld is public, should be declared in a file named HelloWorld.java

原因: ファイル名とクラス名が一致していません。

対処: クラス名がHelloWorldならば、ファイル名もHelloWorld.javaにしてください。

エラー: ';' expected

原因: 文の最後にセミコロン（;）が抜けています。

対処: Javaの文は必ずセミコロンで終わります。エラーメッセージが示す行を確認してください。

エラー: 文字コードの問題（日本語が文字化け）

原因: ソースコードの文字コードがUTF-8になっていません。

対処: エディタでファイルの文字コードをUTF-8に設定してください。コンパイル時に-encoding UTF-8オプションを追加することもできます。

理解度確認クイズ

  1. Q1. JVM、JRE、JDKの違いを説明してください。

    回答を表示
    • JVM: Javaプログラムを実行するための仮想マシン
    • JRE: Javaプログラムを実行するための環境（JVM + 標準ライブラリ）
    • JDK: Java開発に必要なツール一式（JRE + コンパイラ + デバッガなど）
    • 開発にはJDKが必要で、実行だけならJREで十分です。

  2. Q2. Javaプログラムのコンパイルと実行のコマンドを説明してください。

    回答を表示

    コンパイル:

    javac ファイル名.java

    これにより、.classファイル（バイトコード）が生成されます。

    実行:

    java クラス名

    実行時は.classを付けず、クラス名のみを指定します。

  3. Q3. Javaプログラムのファイル名とクラス名には、どのような関係がありますか？

    回答を表示

    Javaでは、publicクラスのファイル名とクラス名が完全に一致している必要があります。大文字・小文字も区別されます。

    例: クラス名がHelloWorldなら、ファイル名もHelloWorld.javaでなければなりません。

  4. Q4. mainメソッドの役割は何ですか？

    回答を表示

    mainメソッドは、Javaプログラムの実行開始点（エントリーポイント）です。

    JVMがプログラムを実行する際、最初にmainメソッドを探して実行します。そのため、実行可能なJavaプログラムには必ずmainメソッドが必要です。
