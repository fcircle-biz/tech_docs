第1章

Javaの世界へようこそ - 環境構築と最初の一歩

Javaとは何か、なぜ世界中で使われているのかを理解し、開発環境のセットアップから最初のプログラム「Hello World」を作成・実行するまでを学びます。

この章で学ぶこと

  • Javaプログラミング言語の特徴と用途を理解する
  • JDKのインストールと環境変数の設定ができる
  • テキストエディタまたはIDEでJavaプログラムを作成できる
  • コンパイルと実行の仕組みを理解し、Hello Worldプログラムを動かせる

1.1 Javaとは何か

Javaの歴史と理念

Javaは1995年にサン・マイクロシステムズ（現在はオラクル社）によって開発されたプログラミング言語です。 Javaの最大の特徴は「Write Once, Run Anywhere」（一度書けば、どこでも動く）という理念です。 これは、一度書いたJavaプログラムが、Windows、Mac、Linux など、異なるOS上で変更なしに動作することを意味します。

JVM（Java仮想マシン）の役割

なぜJavaプログラムはどこでも動くのでしょうか？その秘密はJVM（Java Virtual Machine）にあります。 JVMは、Javaプログラムを実行するための仮想的なコンピュータです。各OS向けにJVMが用意されており、JVMがJavaプログラムを理解して実行してくれます。

Javaソースコードはコンパイルされてバイトコードになり、各OS上のJVMで実行されます。

JDK、JRE、JVMの違い

Javaを学習する上で、よく出てくる3つの用語があります。それぞれの違いを理解しましょう。

JVM

Java Virtual Machine
Javaプログラムを実行するための仮想マシン。各OS用に用意されている。

JRE

Java Runtime Environment
JVMとJavaの実行に必要なライブラリをセットにしたもの。プログラムを実行するだけなら十分。

JDK

Java Development Kit
JREに加え、コンパイラなどの開発ツールを含む。プログラムを作成するには必須。

ポイント

プログラミングを学習するにはJDKをインストールする必要があります。JDKにはJREとJVMが含まれているため、これだけでOKです。

Javaが使われている分野

Javaは非常に幅広い分野で使われています。いくつか代表的な例を見てみましょう。

Webアプリケーション

銀行システム、ECサイト、企業の基幹システムなど、大規模なWebアプリケーションで多用されています。

Androidアプリ

Androidスマートフォンのアプリ開発にJavaが使われています（最近はKotlinも人気）。

業務システム

企業の在庫管理、販売管理、会計システムなど、ビジネスを支えるシステムで広く採用されています。

クラウドサービス

Amazon、Twitter、LinkedInなどの大規模サービスでもJavaが使われています。

1.2 開発環境のセットアップ

JDKのダウンロードとインストール

Java開発を始めるには、まずJDKをインストールする必要があります。ここでは推奨される手順を説明します。

JDKのインストール手順

  1. 1

    Oracle公式サイトにアクセス

    Oracle Java Downloads にアクセスし、最新のJDK（Java 17以降推奨）をダウンロードします。

  2. 2

    OSに合わせたインストーラーを選択

    Windows: x64 Installer (.exe)
    Mac: ARM64 DMG Installer (Apple Silicon) または x64 DMG Installer (Intel)
    Linux: x64 Compressed Archive (.tar.gz)

  3. 3

    インストーラーを実行

    ダウンロードしたファイルを実行し、画面の指示に従ってインストールします。 デフォルト設定のままで問題ありません。

環境変数の設定

JDKをインストールした後、どこからでもJavaコマンドを実行できるように環境変数を設定します。

Windows での環境変数設定

  1. 1. 「スタート」→「システム」→「システムの詳細設定」→「環境変数」をクリック
  2. 2. 「システム環境変数」の「新規」をクリック
  3. 3. 変数名: JAVA_HOME
  4. 4. 変数値: JDKのインストールパス（例: C:\Program Files\Java\jdk-17）
  5. 5. 「Path」変数を選択し、「編集」をクリック
  6. 6. 「新規」をクリックし、%JAVA_HOME%\bin を追加

Mac / Linux での環境変数設定

ターミナルを開き、ホームディレクトリの .bash_profile または .zshrc に以下を追加します。

~/.zshrc または ~/.bash_profile 
# Java環境変数の設定
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home
export PATH=$JAVA_HOME/bin:$PATH

設定後、source ~/.zshrc で設定を反映させます。

インストールの確認

環境変数の設定が完了したら、コマンドライン（ターミナル）でJavaが正しくインストールされているか確認しましょう。

ターミナル / コマンドプロンプト 
# Javaのバージョン確認
java -version

# コンパイラのバージョン確認
javac -version

正しくインストールされていれば、以下のような出力が表示されます。

$ java -version
java version "17.0.9" 2023-10-17 LTS
Java(TM) SE Runtime Environment (build 17.0.9+11-LTS-201)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.9+11-LTS-201, mixed mode, sharing)
$ javac -version
javac 17.0.9

つまずきポイント

java や javac コマンドが認識されない場合は、環境変数の設定が正しくない可能性があります。 PATHに %JAVA_HOME%\bin が追加されているか、 ターミナルを再起動したか確認しましょう。

IDEの選択と基本設定

Javaプログラムはメモ帳などのテキストエディタでも書けますが、IDE（統合開発環境）を使うと効率的に開発できます。 初心者におすすめのIDEは以下の通りです。

VS Code

軽量で拡張性が高い。Extension Pack for Java をインストールすれば快適に開発できる。

初心者向け

IntelliJ IDEA

Java開発に特化した高機能IDE。Community版は無料で使える。コード補完が優秀。

中級者向け

Eclipse

老舗のJava IDE。無料で高機能。企業でも広く使われている。

業務向け

推奨：VS Code + Extension Pack for Java

この教材では、初心者にも扱いやすい VS Code を推奨します。 VS Codeのインストール後、拡張機能から「Extension Pack for Java」をインストールすれば、 快適なJava開発環境が整います。

1.3 最初のプログラム - Hello World

環境が整ったら、さっそく最初のJavaプログラムを作成してみましょう。 プログラミングの世界では、伝統的に「Hello World」と表示するプログラムから始めます。

ソースファイルの作成

まず、作業用のフォルダを作成し、その中に HelloWorld.java というファイルを作成します。

HelloWorld.java 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

プログラムの構造を理解する

たった4行のプログラムですが、Javaの基本的な構造が詰まっています。1行ずつ見ていきましょう。

public class HelloWorld

クラスの定義：Javaのプログラムは「クラス」という単位で構成されます。 HelloWorld がクラス名です。 ファイル名とクラス名は必ず一致させる必要があります。

public static void main(String[] args)

mainメソッド：Javaプログラムが実行されるときに、最初に呼び出される特別なメソッドです。 この書き方は決まり文句として覚えましょう。すべてのJavaプログラムには必ずmainメソッドが必要です。

System.out.println("Hello, World!");

出力命令：画面に文字を表示するための命令です。 "Hello, World!" の部分を変えると、表示する内容を変更できます。 文の最後には必ずセミコロン（;）をつけます。

{ } 波括弧

ブロック：処理のまとまりを示します。クラスの中身、メソッドの中身を波括弧で囲みます。 開き括弧 { と閉じ括弧 } は必ずペアになります。

コンパイルと実行

Javaプログラムを実行するには、2つのステップが必要です。

コンパイルと実行の流れ

ソースファイル] -->|javac

バイトコード] B -->|java

実行手順

  1. 1

    コマンドライン/ターミナルを開く

    HelloWorld.javaを保存したフォルダに移動します。

  2. 2

    コンパイル

    javac HelloWorld.java

    エラーがなければ、HelloWorld.class ファイルが生成されます。

  3. 3

    実行

    java HelloWorld

    注意：.class は付けません。クラス名だけを指定します。

成功すると、画面に「Hello, World!」と表示されます。

$ javac HelloWorld.java
$ java HelloWorld
Hello, World!

よくあるエラーと対処法

エラー: クラス名とファイル名が一致していない

error: class HelloWorld is public, should be declared in a file named HelloWorld.java

原因：ファイル名が HelloWorld.java になっていない。
対処：ファイル名をクラス名と完全に一致させる（大文字・小文字も含む）。

エラー: セミコロンがない

error: ';' expected

原因：文の最後にセミコロン（;）が付いていない。
対処：エラーメッセージに表示された行番号の文末に ; を追加する。

エラー: mainメソッドが見つからない

Error: Main method not found in class HelloWorld

原因：mainメソッドのスペルや書き方が間違っている。
対処：public static void main(String[] args) を正確に書く。

エラー: 日本語が文字化けする

原因：ファイルの文字コードがUTF-8になっていない。
対処：エディタの設定で文字コードを「UTF-8」に変更してファイルを保存し直す。

実習 1-1: 自分の名前を表示するプログラムを作成

Hello Worldプログラムを応用して、自分の名前を表示するプログラムを作成しましょう。

手順

  1. 1 MyName.java という新しいファイルを作成
  2. 2 クラス名を MyName にする
  3. 3 自分の名前を表示する println 文を書く
  4. 4 コンパイルして実行する

コード例

public class MyName {
    public static void main(String[] args) {
        System.out.println("私の名前は 山田太郎 です");
    }
}

期待される結果

私の名前は 山田太郎 です

println を複数回使って、 複数行のメッセージを表示してみましょう。

コード例

実習 1-3: エラーを体験してみよう

意図的にエラーを発生させて、エラーメッセージを読む練習をしましょう。

試してみよう

  1. 1. セミコロンを削除してコンパイルしてみる
  2. 2. クラス名を変えずにファイル名を変更してコンパイルしてみる
  3. 3. main を Main に変えて実行してみる

学習のコツ：エラーメッセージは怖いものではありません。 何が間違っているかを教えてくれる「ヒント」です。エラーを恐れず、積極的に読む習慣をつけましょう。

理解度確認クイズ

  1. Q1. Javaの「Write Once, Run Anywhere」とはどういう意味ですか?

    回答を表示

    一度書いたJavaプログラムが、Windows、Mac、Linuxなど、異なるOS上で変更なしに動作することを意味します。 これはJVM（Java仮想マシン）が各OSに合わせてプログラムを実行してくれるためです。

  2. Q2. JDK、JRE、JVMの違いを説明してください。

    回答を表示

    JVM: Javaプログラムを実行する仮想マシン
    JRE: JVMとJavaの実行に必要なライブラリのセット
    JDK: JREに加え、コンパイラなどの開発ツールを含む開発キット
    プログラミングを行うにはJDKが必要です。

  3. Q3. Javaプログラムの実行には、コンパイルと実行の2つのステップが必要です。それぞれどのコマンドを使いますか?

    回答を表示

    コンパイル: javac ファイル名.java
    実行: java クラス名
    注意：実行時は .class を付けず、クラス名だけを指定します。

  4. Q4. なぜJavaではファイル名とクラス名を一致させる必要があるのですか?

    回答を表示

    Javaコンパイラは、publicクラスがあるファイルを見つけるために、 クラス名とファイル名が一致していることを要求します。 これにより、大規模なプロジェクトでもファイルを簡単に見つけることができます。

  5. Q5. mainメソッドはなぜ必要なのですか?

    回答を表示

    mainメソッドは、Javaプログラムが実行されるときの「エントリーポイント（開始地点）」です。 JVMは、プログラムを実行する際、必ず最初にmainメソッドを探して実行を開始します。 mainメソッドがないと、JVMはプログラムをどこから開始すればよいか分からず、エラーになります。

この章のまとめ

  • Javaは「Write Once, Run Anywhere」の理念のもと、JVMによって異なるOS上で動作する
  • プログラミングにはJDKが必要で、環境変数の設定が重要
  • Javaプログラムはクラスで構成され、mainメソッドから実行が始まる
  • ファイル名とクラス名は必ず一致させる必要がある
  • javacでコンパイル、javaで実行する2ステップの流れを理解する
  • エラーメッセージは敵ではなく、問題解決のヒント
