Javaとプログラミングの基本

Javaとは何か、プログラミングとは何かを理解し、Java開発環境の構築から初めてのプログラム「Hello World」を実行します。

この章で学ぶこと

  • プログラミングとは何か、Javaとは何かを理解する
  • Java開発環境（JDK・IDE）のインストールと設定方法
  • 初めてのJavaプログラム「Hello World」の作成と実行
  • Javaプログラムの基本構造とコンパイル・実行の仕組み

1.1 プログラミングとは何か

プログラミングとは、コンピュータに対して「何をするべきか」を指示するための命令を書くことです。私たちが日常的に使っているスマートフォンのアプリ、Webサイト、ゲーム、業務システムなど、あらゆるソフトウェアはプログラムによって動いています。

コンピュータは人間の言葉を直接理解できないため、プログラミング言語という特別な言語を使ってコンピュータと対話します。プログラミング言語には様々な種類がありますが、Javaはその中でも世界中で広く使われている主要な言語の一つです。

プログラミング言語の役割

プログラミング言語は、人間が理解しやすい形式で命令を記述し、それをコンピュータが実行できる機械語に変換するための橋渡し役です。Javaは「コンパイル」という処理を通じて、人間が書いたコードを機械が理解できる形に変換します。

1.2 Javaとは何か

Javaは1995年にSun Microsystems社（現Oracle社）によって開発されたプログラミング言語です。「Write Once, Run Anywhere」（一度書けばどこでも動く）というコンセプトのもと設計されており、Windows、macOS、Linuxなど、異なるOSでも同じプログラムを実行できることが大きな特徴です。

Javaの主な特徴

  • プラットフォーム非依存: Java仮想マシン（JVM）上で動作するため、OSに依存しない
  • オブジェクト指向: 複雑なプログラムを整理しやすい設計手法を採用
  • 堅牢性と安全性: エラー処理やメモリ管理が厳格で、安全なプログラムを書きやすい
  • 豊富なライブラリ: データベース接続、ネットワーク通信、GUIなど様々な機能がすぐに使える
  • 広範な利用実績: エンタープライズシステム、Androidアプリ、Webアプリなど幅広い分野で活用

1.3 Java開発環境のセットアップ

Javaプログラムを開発するには、以下の2つの準備が必要です。

1. JDK（Java Development Kit）のインストール

JDKは、Javaプログラムを書いて実行するための基本ツールセットです。コンパイラ（javac）、実行環境（JVM）、標準ライブラリなどが含まれています。

JDKダウンロード手順

  1. Oracle公式サイト（oracle.com/java）またはAdoptium（adoptium.net）にアクセス
  2. JDK 17以上（LTS版推奨）を選択してダウンロード
  3. インストーラーを実行し、画面の指示に従ってインストール
  4. 環境変数JAVA_HOMEとPATHを設定（インストーラーが自動設定する場合もあり）

インストールが完了したら、コマンドラインで以下のコマンドを実行して確認しましょう。

ターミナル / コマンドプロンプト 
java -version

以下のようなバージョン情報が表示されればインストール成功です。 

java version "17.0.9" 2023-10-17 LTS
Java(TM) SE Runtime Environment (build 17.0.9+11-LTS-201)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.9+11-LTS-201, mixed mode)

2. IDE（統合開発環境）のインストール

IDEは、コードの記述、実行、デバッグを効率的に行うためのツールです。初心者には以下のIDEがおすすめです。

  • IntelliJ IDEA Community Edition（推奨）

    無料で高機能、初心者から上級者まで幅広く利用されている

  • Eclipse

    老舗のJava IDE、プラグインが豊富

  • Visual Studio Code（VS Code）

    軽量で拡張性が高い、Java拡張パックのインストールが必要

1.4 初めてのJavaプログラム「Hello World」

プログラミング学習の最初のステップとして、「Hello World」と画面に表示する簡単なプログラムを作成しましょう。

ステップ1: プロジェクトの作成

IDEで新しいJavaプロジェクトを作成します。プロジェクト名は「HelloWorld」などシンプルな名前にしましょう。

ステップ2: Javaクラスの作成

プロジェクト内に「HelloWorld.java」という名前のファイルを作成し、以下のコードを入力します。

HelloWorld.java 
public class HelloWorld {
    public static void main(String[] args) {
        // 画面に「Hello, World!」と表示
        System.out.println("Hello, World!");
    }
}

コードの説明

  • public class HelloWorld

    「HelloWorld」という名前のクラスを定義しています。クラスはプログラムの基本単位です。

  • public static void main(String[] args)

    プログラムの開始地点となる特別なメソッド（mainメソッド）です。Javaプログラムは必ずここから実行されます。

  • System.out.println("Hello, World!");

    画面に文字列を表示する命令です。printlnは「print line」の略で、文字を表示した後に改行します。

  • // コメント

    「//」で始まる行はコメントで、プログラムの動作には影響しません。メモや説明を書くために使います。

重要な注意点

  • • ファイル名とクラス名は完全に一致させる必要があります（HelloWorld.java と class HelloWorld）
  • • Javaは大文字・小文字を区別します（HelloWorldとhelloworldは別のもの）
  • • 各命令の最後には「;」（セミコロン）が必要です
  • • 全角スペースは使わず、半角スペースやタブを使用します

ステップ3: プログラムの実行

IDEの実行ボタン（通常は緑色の三角ボタン）をクリックするか、右クリックメニューから「Run」を選択してプログラムを実行します。

コンソール（出力画面）に以下のように表示されれば成功です。 

Hello, World!

1.5 Javaプログラムの実行の仕組み

Javaプログラムが実行されるまでには、以下の2つのステップがあります。

ステップ1: コンパイル

私たちが書いた「HelloWorld.java」ファイル（ソースコード）は、人間が読める形式です。これをコンピュータが理解できる形式に変換する処理をコンパイルといいます。コンパイルによって「HelloWorld.class」というバイトコードファイルが生成されます。

コンパイルコマンド（手動実行の場合） 
javac HelloWorld.java

ステップ2: 実行

生成されたバイトコードをJava仮想マシン（JVM）が読み込んで実行します。JVMは各OSに合わせた形式で動作するため、同じバイトコードがWindows、macOS、Linuxのどれでも実行できます。

実行コマンド（手動実行の場合） 
java HelloWorld

IDEを使う利点

IDEを使うと、コンパイルと実行を自動で行ってくれるため、上記のコマンドを手動で入力する必要はありません。ボタン一つで簡単にプログラムを実行できます。

実習 1-1: 挨拶プログラムを作ろう

Hello Worldの仕組みを理解したら、自分の名前で挨拶するプログラムを作成してみましょう。

課題

「こんにちは、私の名前は[あなたの名前]です。」と表示するプログラムを作成してください。

手順

  1. 1 新しいJavaクラス「Greeting.java」を作成します
  2. 2 HelloWorldのコードを参考に、System.out.printlnを使って挨拶文を表示します
  3. 3 プログラムを実行して、挨拶が正しく表示されることを確認します

実装例

public class Greeting {
    public static void main(String[] args) {
        // あなたの名前に書き換えてください
        System.out.println("こんにちは、私の名前は太郎です。");
    }
}

期待される結果

こんにちは、私の名前は太郎です。

チャレンジ: 複数のSystem.out.printlnを使って、複数行の挨拶文を表示してみましょう。

理解度確認クイズ

  1. Q1. Javaプログラムで必ず必要な「mainメソッド」の正しい記述はどれですか？

    回答を表示

    public static void main(String[] args)
    これはJavaプログラムのエントリーポイント（開始地点）です。この記述は決まり文句として覚えましょう。

  2. Q2. Javaのソースファイルの拡張子は何ですか？また、コンパイル後に生成されるファイルの拡張子は何ですか？

    回答を表示

    ソースファイル: .java
    コンパイル後: .class
    .javaファイルは人間が読み書きするコード、.classファイルはJVMが実行するバイトコードです。

  3. Q3. 画面に文字を表示するために使う命令は何ですか？

    回答を表示

    System.out.println() または System.out.print()
    printlnは表示後に改行、printは改行しません。

  4. Q4. Javaのクラス名とファイル名の関係について正しいものはどれですか？

    回答を表示

    クラス名とファイル名は完全に一致させる必要があります。
    例: public class HelloWorld → ファイル名は HelloWorld.java
