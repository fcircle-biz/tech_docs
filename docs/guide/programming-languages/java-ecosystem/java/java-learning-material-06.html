処理をまとめる - メソッド

よく使う処理をまとめて名前をつける「メソッド」について学びます。レシピのように一度書けば何度でも使える便利な仕組みを解説します。

この章で学ぶこと

  • メソッドとは何か、なぜ使うのかを理解する
  • 引数（パラメータ）と戻り値の仕組みを学ぶ
  • 自分でメソッドを作成して呼び出せるようになる

6.1 メソッドって何だろう？

プログラムを書いていると、「同じような処理を何度も書いている」ことに気づくことがあります。 例えば、「あいさつを表示する処理」を10回書く必要があるとき、毎回同じコードを書くのは面倒ですよね。

メソッドとは？

処理のまとまりに名前をつけて、何度でも呼び出せるようにしたものです。 料理のレシピのように、一度作っておけば何度でも同じ料理を作れるイメージです。

たとえ話: 料理のレシピ

レシピなし

カレーを作るたびに、手順を1から考えなければならない。 同じ料理でも毎回違う出来になってしまう。

レシピあり（メソッド）

一度レシピを作っておけば、「カレーを作る」と言うだけで 同じ手順で同じ品質の料理が作れる。

メソッドを使うメリット

再利用できる

一度書いたコードを何度でも使い回せる。 同じ処理を何度も書く必要がなくなる。

読みやすくなる

処理に名前がつくので、何をしているか分かりやすい。 プログラム全体が整理される。

修正が楽

処理を変更したいとき、メソッドの中身を直すだけで すべての呼び出し箇所に反映される。

6.2 実はもうメソッドを使っている

実は、これまでもメソッドを使っていました。 System.out.println()や mainもメソッドです。

すでに使っているメソッドの例 
public class Example {
    // mainメソッド - プログラムの開始地点
    public static void main(String[] args) {

        // printlnメソッド - 文字を画面に表示する
        System.out.println("Hello");

        String text = "Java";
        // lengthメソッド - 文字列の長さを返す
        int len = text.length();

        // toUpperCaseメソッド - 大文字に変換する
        String upper = text.toUpperCase();
    }
}

メソッドの呼び出し方

メソッド名()の形で呼び出します。 括弧の中に渡すデータを「引数（ひきすう）」と呼びます。

6.3 メソッドを作ってみよう

自分でメソッドを作る方法を学びましょう。まずは最もシンプルな形から始めます。

最もシンプルなメソッド

シンプルなメソッドの例 
public class MethodDemo {
    public static void main(String[] args) {
        // メソッドを呼び出す
        sayHello();
        sayHello();
        sayHello();
    }

    // あいさつを表示するメソッドを定義
    public static void sayHello() {
        System.out.println("こんにちは！");
        System.out.println("今日も良い天気ですね。");
    }
}

6.4 引数（パラメータ）を使う

メソッドに引数（ひきすう、parameter）を渡すことで、 メソッドの動作を変えることができます。

引数のイメージ

自動販売機にお金を入れるようなもの。 「100円」を入れれば100円の商品が出てくるし、「150円」を入れれば150円の商品が買える。 同じ機械でも、入れるものによって結果が変わります。

引数を使ったメソッド 
public class GreetingDemo {
    public static void main(String[] args) {
        // 異なる名前で呼び出す
        greet("田中");
        greet("佐藤");
        greet("鈴木");
    }

    // 名前を受け取ってあいさつするメソッド
    public static void greet(String name) {
        System.out.println(name + "さん、こんにちは！");
    }
}

// 出力:
// 田中さん、こんにちは！
// 佐藤さん、こんにちは！
// 鈴木さん、こんにちは！

複数の引数を使う

複数の引数 
public class MultiParamDemo {
    public static void main(String[] args) {
        showProfile("田中太郎", 25);
        showProfile("佐藤花子", 30);
    }

    // 複数の引数を受け取るメソッド
    public static void showProfile(String name, int age) {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("---");
    }
}

引数の順番に注意！

呼び出すときは、メソッド定義と同じ順番・同じ型で渡す必要があります。 showProfile("田中", 25)はOKですが、 showProfile(25, "田中")はエラーになります。

6.5 戻り値を返す

メソッドは処理の結果を返すことができます。 この返される値を戻り値（もどりち、return value）と呼びます。

戻り値のイメージ

電卓に「5 + 3」と入力すると「8」という結果が返ってくるようなもの。 計算という処理をして、その結果を返してくれます。

戻り値のあるメソッド 
public class ReturnDemo {
    public static void main(String[] args) {
        // 戻り値を変数で受け取る
        int result = add(5, 3);
        System.out.println("5 + 3 = " + result);  // 5 + 3 = 8

        // 直接使うこともできる
        System.out.println("10 + 20 = " + add(10, 20));  // 10 + 20 = 30
    }

    // 足し算の結果を返すメソッド
    public static int add(int a, int b) {
        int sum = a + b;
        return sum;  // 計算結果を返す
    }
}

戻り値のポイント

  • ・戻り値がある場合はvoidの代わりに戻り値の型を書く
  • ・return文で値を返す
  • ・returnが実行されると、メソッドはそこで終了する

様々な戻り値の型

様々な戻り値の例 
// int型を返す
public static int multiply(int a, int b) {
    return a * b;
}

// double型を返す
public static double average(int a, int b) {
    return (a + b) / 2.0;
}

// boolean型を返す
public static boolean isAdult(int age) {
    return age >= 18;
}

// String型を返す
public static String getGrade(int score) {
    if (score >= 80) {
        return "優";
    } else if (score >= 70) {
        return "良";
    } else if (score >= 60) {
        return "可";
    } else {
        return "不可";
    }
}

実習 6-1: 計算機メソッドを作ろう

四則演算を行うメソッドを作成し、計算機プログラムを作ります。

Calculator.java 
public class Calculator {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;

        System.out.println("=== 計算機プログラム ===");
        System.out.println(a + " + " + b + " = " + add(a, b));
        System.out.println(a + " - " + b + " = " + subtract(a, b));
        System.out.println(a + " × " + b + " = " + multiply(a, b));
        System.out.println(a + " ÷ " + b + " = " + divide(a, b));
    }

    // 足し算
    public static int add(int x, int y) {
        return x + y;
    }

    // 引き算
    public static int subtract(int x, int y) {
        return x - y;
    }

    // 掛け算
    public static int multiply(int x, int y) {
        return x * y;
    }

    // 割り算（小数点以下も表示）
    public static double divide(int x, int y) {
        return (double) x / y;
    }
}

6.6 メソッドの活用例

配列と組み合わせた実践的なメソッドの例を見てみましょう。

配列を使ったメソッド 
public class ArrayMethods {
    public static void main(String[] args) {
        int[] scores = {85, 72, 90, 65, 78};

        System.out.println("配列の合計: " + sum(scores));
        System.out.println("配列の平均: " + average(scores));
        System.out.println("最大値: " + max(scores));
        System.out.println("最小値: " + min(scores));
    }

    // 配列の合計を求める
    public static int sum(int[] numbers) {
        int total = 0;
        for (int num : numbers) {
            total += num;
        }
        return total;
    }

    // 配列の平均を求める
    public static double average(int[] numbers) {
        return (double) sum(numbers) / numbers.length;
    }

    // 配列の最大値を求める
    public static int max(int[] numbers) {
        int maxValue = numbers[0];
        for (int num : numbers) {
            if (num > maxValue) {
                maxValue = num;
            }
        }
        return maxValue;
    }

    // 配列の最小値を求める
    public static int min(int[] numbers) {
        int minValue = numbers[0];
        for (int num : numbers) {
            if (num < minValue) {
                minValue = num;
            }
        }
        return minValue;
    }
}

ポイント: メソッドからメソッドを呼び出す

averageメソッドの中でsumメソッドを呼び出しています。 このように、メソッドは他のメソッドを利用することができます。

理解度確認クイズ

  1. Q1. メソッドを使う3つのメリットは何ですか？

    回答を表示

    1. 再利用できる：同じコードを何度でも使い回せる
    2. 読みやすくなる：処理に名前がつき、コードが整理される
    3. 修正が楽：一箇所直せば全ての呼び出し箇所に反映

  2. Q2. 「void」は何を意味しますか？

    回答を表示

    「何も返さない」という意味です。 処理を実行するだけで、戻り値がないメソッドに使用します。

  3. Q3. 引数と戻り値の違いは何ですか？

    回答を表示

    引数：メソッドに渡す入力データ（呼び出す側→メソッド）
    戻り値：メソッドが処理して返す結果データ（メソッド→呼び出す側）

  4. Q4. return文は何をしますか？

    回答を表示

    1. 指定した値を呼び出し元に返す
    2. メソッドの実行を終了する（return以降の処理は実行されない）

6.7 この章のまとめ

  • メソッドは、処理をまとめて名前をつけ、再利用できるようにしたもの。
  • 引数でメソッドにデータを渡し、戻り値で結果を受け取る。
  • voidは戻り値がないことを意味する。
  • returnで値を返し、メソッドを終了する。
  • メソッドを使うと、コードの再利用性、可読性、保守性が向上する。
