第6章

配列 - 複数のデータをまとめて管理

同じ種類のデータを複数まとめて扱う「配列」の概念と使い方を学びます。配列を使うことで、大量のデータを効率的に管理できるようになります。

この章で学ぶこと

  • 配列の概念と必要性を理解する
  • 配列の宣言、生成、初期化ができる
  • 配列の要素にアクセスし操作できる
  • 拡張for文（for-each）を使える
  • 多次元配列の基本を理解する

6.1 配列とは何か

これまで学んできた変数は、1つの値しか保存できませんでした。しかし、実際のプログラミングでは、たくさんのデータをまとめて扱いたい場面が多くあります。例えば、クラスの生徒全員の点数を管理したり、1週間分の気温データを処理したりする場合です。

配列（Array）とは、同じ型のデータを複数まとめて格納できる入れ物です。配列を使うと、大量のデータを効率的に管理できます。

配列の概念図

通常の変数（1つの値しか保存できない）:

score
85

配列（複数の値を保存できる）:

scores[0]
85
scores[1]
92
scores[2]
78
scores[3]
88
scores[4]
95

配列の重要な特徴

  • 同じ型のデータの集まり：1つの配列には同じデータ型の値しか格納できません（全てint型、全てString型など）
  • 固定サイズ：配列を作成するとき、その大きさ（要素数）を決める必要があり、後から変更できません
  • インデックス（添え字）でアクセス：各要素には0から始まる番号（インデックス）が振られ、この番号でアクセスします
  • length プロパティ：配列の要素数は 配列名.length で取得できます

つまずきやすいポイント：インデックスは0から始まる

配列のインデックス（添え字）は0から始まります。つまり、5つの要素を持つ配列の場合、インデックスは0, 1, 2, 3, 4となります。最後の要素は 配列.length - 1 でアクセスします。これを忘れて 配列[5] にアクセスすると、ArrayIndexOutOfBoundsException（配列の範囲外エラー）が発生します。

6.2 配列の作成と初期化

配列を使うには、まず「宣言」して「生成」する必要があります。配列の作成方法はいくつかあります。

方法1：宣言と生成を分ける

ArrayDeclaration.java 
// 配列の宣言（まだ実体はない）
int[] scores;  // int型の配列を宣言（推奨）
// または int scores[]; でもOK（C言語スタイル）

// 配列の生成（メモリを確保）
scores = new int[5];  // 5つの要素を持つint型配列を生成

// これで scores[0], scores[1], ..., scores[4] が使えるようになる
// 初期値は自動的に0になる（int型の場合）
System.out.println(scores[0]);  // 出力: 0

方法2：宣言と生成を同時に行う

ArrayCreation.java 
// 宣言と生成を1行で
int[] scores = new int[5];

// 様々なデータ型の配列
double[] temperatures = new double[7];  // 7つの要素（初期値: 0.0）
String[] names = new String[3];  // 3つの要素（初期値: null）
boolean[] flags = new boolean[10];  // 10の要素（初期値: false）

方法3：初期化子を使う（値を直接指定）

ArrayInitializer.java 
// 初期値を指定して配列を作成
int[] scores = {85, 92, 78, 88, 95};  // 5つの要素を持つ配列
// 要素数は自動的に5になる

// 文字列の配列
String[] fruits = {"りんご", "バナナ", "オレンジ"};

// この書き方もOK（new を使った初期化子）
int[] numbers = new int[] {10, 20, 30, 40, 50};

System.out.println(scores.length);  // 出力: 5
System.out.println(fruits.length);  // 出力: 3

配列のデフォルト値

new で配列を生成すると、各要素は自動的にデフォルト値（初期値）で初期化されます。

データ型 デフォルト値
整数型（byte, short, int, long） 0
浮動小数点型（float, double） 0.0
文字型（char） '\u0000'（ヌル文字）
論理型（boolean） false
参照型（String, オブジェクトなど） null

6.3 配列の操作

配列を作成したら、要素にアクセスして値を読み書きできます。インデックス（添え字）を使って特定の要素を指定します。

要素へのアクセスと代入

ArrayAccess.java 
public class ArrayAccess {
    public static void main(String[] args) {
        // 配列の作成
        int[] scores = new int[5];

        // 要素に値を代入
        scores[0] = 85;  // 最初の要素
        scores[1] = 92;
        scores[2] = 78;
        scores[3] = 88;
        scores[4] = 95;  // 最後の要素

        // 要素の読み取り
        System.out.println("1番目の点数: " + scores[0]);  // 出力: 85
        System.out.println("3番目の点数: " + scores[2]);  // 出力: 78

        // 要素の更新
        scores[2] = 80;  // 3番目の要素を78から80に変更
        System.out.println("更新後の3番目: " + scores[2]);  // 出力: 80

        // 配列の長さを取得
        System.out.println("配列の要素数: " + scores.length);  // 出力: 5
    }
}

配列のエラーを理解する

ArrayError.java 
public class ArrayError {
    public static void main(String[] args) {
        int[] numbers = new int[5];  // 要素数5の配列（インデックスは0〜4）

        // OK: 有効な範囲内のアクセス
        numbers[0] = 10;
        numbers[4] = 50;  // 最後の要素（インデックス4）

        // NG: 範囲外アクセス（エラー発生）
        // numbers[5] = 60;  // ArrayIndexOutOfBoundsException!
        // numbers[-1] = 5;  // ArrayIndexOutOfBoundsException!

        // 安全なアクセス方法：length を使う
        int lastIndex = numbers.length - 1;  // 4
        numbers[lastIndex] = 100;  // OK

        System.out.println("最後の要素: " + numbers[lastIndex]);
    }
}

つまずきやすいポイント：配列の範囲外アクセス

配列の範囲外にアクセスすると ArrayIndexOutOfBoundsException という実行時エラーが発生します。要素数5の配列の場合：

  • 有効なインデックス：0, 1, 2, 3, 4
  • 無効なインデックス：5以上、または負の数
  • 最後の要素は常に 配列.length - 1 でアクセス

6.4 配列とループの組み合わせ

配列の真価は、ループと組み合わせて使うときに発揮されます。配列の全要素を処理するには、for文が非常に便利です。

通常のfor文で配列を処理

ArrayLoop.java 
public class ArrayLoop {
    public static void main(String[] args) {
        int[] scores = {85, 92, 78, 88, 95};

        // 全要素を表示
        System.out.println("=== 全ての点数 ===");
        for (int i = 0; i < scores.length; i++) {
            System.out.println((i + 1) + "番目: " + scores[i] + "点");
        }

        // 合計を計算
        int sum = 0;
        for (int i = 0; i < scores.length; i++) {
            sum += scores[i];
        }
        System.out.println("\n合計点: " + sum + "点");

        // 平均を計算
        double average = (double) sum / scores.length;
        System.out.println("平均点: " + average + "点");
    }
}

// 出力:
// === 全ての点数 ===
// 1番目: 85点
// 2番目: 92点
// 3番目: 78点
// 4番目: 88点
// 5番目: 95点
//
// 合計点: 438点
// 平均点: 87.6点

拡張for文（for-each文）

拡張for文（for-eachループ）は、配列の全要素を順番に処理するための簡潔な書き方です。インデックスを使わずに直接要素にアクセスできます。

EnhancedForLoop.java 
public class EnhancedForLoop {
    public static void main(String[] args) {
        String[] fruits = {"りんご", "バナナ", "オレンジ", "ぶどう"};

        // 拡張for文の構文: for (要素の型 変数名 : 配列名)
        System.out.println("=== 果物リスト ===");
        for (String fruit : fruits) {
            System.out.println(fruit);
        }

        // 数値配列での使用例
        int[] numbers = {10, 20, 30, 40, 50};
        int total = 0;

        for (int num : numbers) {
            total += num;
        }
        System.out.println("\n合計: " + total);  // 出力: 150
    }
}

// 出力:
// === 果物リスト ===
// りんご
// バナナ
// オレンジ
// ぶどう
//
// 合計: 150

通常のfor文と拡張for文の使い分け

  • 拡張for文を使う場合：全要素を順番に読み取るだけの場合（簡潔で読みやすい）
  • 通常のfor文を使う場合：インデックスが必要な場合、要素を書き換える場合、途中からアクセスする場合

配列の検索と最大値・最小値

ArraySearch.java 
public class ArraySearch {
    public static void main(String[] args) {
        int[] scores = {85, 92, 78, 88, 95, 82};

        // 最大値を探す
        int max = scores[0];  // 最初の要素を初期値とする
        for (int i = 1; i < scores.length; i++) {
            if (scores[i] > max) {
                max = scores[i];
            }
        }
        System.out.println("最高点: " + max + "点");  // 出力: 95点

        // 最小値を探す
        int min = scores[0];
        for (int score : scores) {
            if (score < min) {
                min = score;
            }
        }
        System.out.println("最低点: " + min + "点");  // 出力: 78点

        // 特定の値を検索
        int target = 88;
        boolean found = false;
        int foundIndex = -1;

        for (int i = 0; i < scores.length; i++) {
            if (scores[i] == target) {
                found = true;
                foundIndex = i;
                break;  // 見つかったらループを抜ける
            }
        }

        if (found) {
            System.out.println(target + "点は" + foundIndex + "番目にあります");
        } else {
            System.out.println(target + "点は見つかりませんでした");
        }
    }
}

6.5 多次元配列

これまで学んだ配列は1次元配列（一列に並んだデータ）でした。しかし、多次元配列を使うと、表形式のデータ（2次元配列）や、さらに複雑な構造（3次元以上）を扱えます。

2次元配列の概念

2次元配列は「配列の配列」です。行と列を持つ表のようなデータを表現できます。

2次元配列のイメージ

列0 列1 列2
行0 10 20 30
行1 40 50 60
行2 70 80 90

アクセス例：array[1][2] → 60（行1、列2の要素）

2次元配列の作成と使用

TwoDimensionalArray.java 
public class TwoDimensionalArray {
    public static void main(String[] args) {
        // 2次元配列の宣言と生成（3行4列）
        int[][] matrix = new int[3][4];

        // 要素への代入
        matrix[0][0] = 10;
        matrix[0][1] = 20;
        matrix[1][2] = 50;

        // 初期化子を使った作成
        int[][] scores = {
            {85, 92, 78},  // 1人目の3科目の点数
            {88, 95, 82},  // 2人目の3科目の点数
            {90, 87, 93}   // 3人目の3科目の点数
        };

        // 要素へのアクセス
        System.out.println("1人目の2科目目: " + scores[0][1]);  // 92
        System.out.println("3人目の1科目目: " + scores[2][0]);  // 90

        // 全要素を表示（2重ループ）
        System.out.println("\n=== 全員の点数 ===");
        for (int i = 0; i < scores.length; i++) {  // 行（人）
            System.out.print((i + 1) + "人目: ");
            for (int j = 0; j < scores[i].length; j++) {  // 列（科目）
                System.out.print(scores[i][j] + "点 ");
            }
            System.out.println();
        }

        // 拡張for文での処理
        System.out.println("\n=== 各人の合計点 ===");
        int personNum = 1;
        for (int[] personScores : scores) {
            int sum = 0;
            for (int score : personScores) {
                sum += score;
            }
            System.out.println(personNum + "人目の合計: " + sum + "点");
            personNum++;
        }
    }
}

// 出力:
// 1人目の2科目目: 92
// 3人目の1科目目: 90
//
// === 全員の点数 ===
// 1人目: 85点 92点 78点
// 2人目: 88点 95点 82点
// 3人目: 90点 87点 93点
//
// === 各人の合計点 ===
// 1人目の合計: 255点
// 2人目の合計: 265点
// 3人目の合計: 270点

2次元配列のサイズ取得

  • 配列.length → 行数（外側の配列の要素数）
  • 配列[行番号].length → その行の列数

実習 6-1: テストの点数を配列で管理

5人の生徒のテスト点数を配列に格納し、合計点、平均点、最高点、最低点を計算するプログラムを作成しましょう。

手順

  1. 1 5つの要素を持つint型配列を作成し、点数を格納する
  2. 2 for文を使って全ての点数を表示する
  3. 3 合計点と平均点を計算して表示する
  4. 4 最高点と最低点を探して表示する

実装例

public class ScoreAnalysis {
    public static void main(String[] args) {
        // 点数データ
        int[] scores = {85, 92, 78, 88, 95};

        // 全ての点数を表示
        System.out.println("=== 点数一覧 ===");
        for (int i = 0; i < scores.length; i++) {
            System.out.println((i + 1) + "人目: " + scores[i] + "点");
        }

        // 合計と平均を計算
        int sum = 0;
        for (int score : scores) {
            sum += score;
        }
        double average = (double) sum / scores.length;

        // 最高点と最低点を探す
        int max = scores[0];
        int min = scores[0];
        for (int score : scores) {
            if (score > max) max = score;
            if (score < min) min = score;
        }

        // 結果を表示
        System.out.println("\n=== 統計結果 ===");
        System.out.println("合計点: " + sum + "点");
        System.out.println("平均点: " + average + "点");
        System.out.println("最高点: " + max + "点");
        System.out.println("最低点: " + min + "点");
    }
}

1人目: 85点

3人目: 78点
4人目: 88点
5人目: 95点

=== 統計結果 ===

理解度確認クイズ

  1. Q1. 要素数10の配列を作成した場合、有効なインデックスの範囲は何から何までですか？

    回答を表示

    0から9までです。配列のインデックスは0から始まるため、要素数が10の場合、有効なインデックスは0, 1, 2, ..., 9となります。最後の要素は 配列.length - 1（つまり 10 - 1 = 9）でアクセスします。10以上や負の数でアクセスすると ArrayIndexOutOfBoundsException が発生します。

  2. Q2. 次のコードで、配列 numbers の全要素の合計を計算するには、空欄に何を入れるべきですか？
    int[] numbers = {10, 20, 30, 40, 50};
    int sum = 0;
    for (int i = 0; i < ______; i++) { sum += numbers[i]; }

    回答を表示

    numbers.length です。配列の要素数は 配列名.length で取得できます。この場合、numbers.length は5なので、ループは i が 0, 1, 2, 3, 4 のときに実行され、全ての要素にアクセスできます。

  3. Q3. 拡張for文（for-each）の利点と制約を説明してください。どのような場面で使うべきですか？

    回答を表示

    利点：シンプルで読みやすく、インデックスを意識する必要がないため、範囲外エラーのリスクが減ります。
    制約：インデックス番号が取得できず、要素の書き換えができない（プリミティブ型の場合）。また、配列の途中から処理を開始したり、逆順で処理することはできません。
    使うべき場面：配列の全要素を順番に読み取るだけの場合に最適です。例：合計計算、全要素の表示など。

  4. Q4. new int[5] で配列を作成した場合、各要素の初期値は何ですか？また、new String[3] の場合はどうなりますか？

    回答を表示

    new int[5] の場合、各要素の初期値は 0 です（整数型のデフォルト値）。
    new String[3] の場合、各要素の初期値は null です（参照型のデフォルト値）。
    プリミティブ型は数値の0や false で初期化され、参照型は null で初期化されます。

  5. Q5. 2次元配列 int[][] data = new int[3][4]; において、data.length と data[0].length の値はそれぞれいくつですか？

    回答を表示

    data.length の値は 3 です（行数、外側の配列の要素数）。
    data[0].length の値は 4 です（0行目の列数）。
    2次元配列は「配列の配列」なので、外側の長さが行数、内側の長さが列数を表します。
