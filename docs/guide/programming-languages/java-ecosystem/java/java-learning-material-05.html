データをまとめて管理する - 配列

複数のデータをまとめて管理する「配列」について学びます。ロッカーに物を入れるたとえを使って、配列の概念と操作方法を分かりやすく解説します。

この章で学ぶこと

  • 配列とは何か、なぜ必要なのかを理解する
  • 配列の宣言、初期化、要素へのアクセス方法を学ぶ
  • ループを使った配列の効率的な処理方法を習得する

5.1 配列って何だろう？

これまで学んだ変数は、1つの値しか保存できませんでした。 でも、例えばクラス全員のテストの点数を管理したい場合、30人分の変数を作るのは大変ですよね。

変数だけで管理しようとすると...

int score1 = 85;
int score2 = 72;
int score3 = 90;
// ... 30個の変数を作る必要がある
int score30 = 68;

これでは管理が大変すぎます！

そこで登場するのが配列（array：アレイ）です。 配列を使えば、同じ種類のデータをまとめて1つの名前で管理できます。

配列とは？

同じ型のデータを複数まとめて管理できる入れ物です。 データには番号（インデックス）が付いていて、番号を指定することで個々のデータにアクセスできます。

配列をロッカーでイメージしよう

たとえ話: 学校のロッカー

普通の変数

1つの箱に1つの物しか入れられない。
30人分なら30個の箱を別々に用意する必要がある。

配列

番号付きのロッカーのように、1つの名前で複数の「部屋」を管理。
「ロッカー0番」「ロッカー1番」...と番号で指定してアクセス。

5.2 配列の宣言と初期化

配列を使うには、まず「どんな型のデータを」「いくつ入れられるか」を決めて作成する必要があります。

方法1: サイズを指定して作成

配列の宣言（サイズ指定） 
// 基本の書き方
型名[] 配列名 = new 型名[要素数];

// 例: int型を5個入れられる配列
int[] scores = new int[5];

ポイント

  • ・int[] で「int型の配列」という意味
  • ・new int[5] で「5個分の領域を確保」
  • ・作成直後は全ての要素が0で初期化される（int型の場合）

方法2: 初期値を指定して作成

最初から入れたいデータが決まっている場合は、初期値を指定して作成できます。

配列の宣言（初期値指定） 
// 初期値を指定して作成
int[] scores = {85, 72, 90, 65, 78};

// 文字列の配列
String[] names = {"田中", "佐藤", "鈴木"};

// double型の配列
double[] prices = {100.0, 250.5, 89.9};
データ型 配列宣言例 初期化時のデフォルト値
int int[] nums = new int[5]; 0
double double[] vals = new double[3]; 0.0
boolean boolean[] flags = new boolean[2]; false
String String[] strs = new String[4]; null

5.3 配列の要素にアクセスする

配列の各データ（要素）にはインデックス（index：添え字）という番号が付いています。 この番号を使って、特定の要素を取り出したり、値を変更したりします。

重要: インデックスは0から始まる！

人間は「1番目、2番目...」と数えますが、Javaの配列は0番目から始まります。 これはプログラミングでよくある約束事です。

要素へのアクセス 
int[] scores = {85, 72, 90, 65, 78};

// 値を取得（読み取り）
System.out.println(scores[0]);  // 85 （最初の要素）
System.out.println(scores[2]);  // 90 （3番目の要素）
System.out.println(scores[4]);  // 78 （最後の要素）

// 値を変更（書き込み）
scores[1] = 80;  // 2番目の要素を80に変更
System.out.println(scores[1]);  // 80

配列の長さを取得する

配列の要素数はlengthプロパティで取得できます。

配列の長さ 
int[] scores = {85, 72, 90, 65, 78};

System.out.println(scores.length);  // 5

// 最後の要素にアクセスする方法
System.out.println(scores[scores.length - 1]);  // 78

よくあるエラー: ArrayIndexOutOfBoundsException

存在しないインデックスにアクセスしようとすると、このエラーが発生します。 

int[] scores = {85, 72, 90};  // インデックスは0〜2

// エラーになる例
System.out.println(scores[3]);  // エラー！インデックス3は存在しない
System.out.println(scores[-1]); // エラー！負の数は使えない

5.4 ループで配列を処理する

配列の真価が発揮されるのは、ループと組み合わせたときです。 前章で学んだforループを使えば、配列の全要素を効率的に処理できます。

forループで配列を処理

forループで全要素を表示 
int[] scores = {85, 72, 90, 65, 78};

// 全要素を順番に表示
for (int i = 0; i < scores.length; i++) {
    System.out.println("scores[" + i + "] = " + scores[i]);
}

// 出力結果:
// scores[0] = 85
// scores[1] = 72
// scores[2] = 90
// scores[3] = 65
// scores[4] = 78

拡張forループ（for-each文）

Javaには配列を簡単に処理できる拡張forループ（for-each文）があります。 インデックスを使わずに、全要素を順番に取り出せます。

拡張forループ 
int[] scores = {85, 72, 90, 65, 78};

// 拡張forループ（for-each文）
for (int score : scores) {
    System.out.println(score);
}

// 読み方: 「scores配列の各要素をscoreに入れて繰り返す」
種類 使い分け
通常のforループ インデックスが必要なとき、要素を変更したいとき
拡張forループ 単純に全要素を順番に処理したいとき（読み取り専用向き）

実習 5-1: 配列の合計と平均を計算しよう

テストの点数を配列で管理し、合計点と平均点を計算するプログラムを作成します。

ScoreCalculator.java 
public class ScoreCalculator {
    public static void main(String[] args) {
        // テストの点数を配列で管理
        int[] scores = {85, 72, 90, 65, 78};

        // 合計を計算
        int sum = 0;
        for (int score : scores) {
            sum = sum + score;  // または sum += score;
        }

        // 平均を計算
        double average = (double) sum / scores.length;

        // 結果を表示
        System.out.println("点数一覧:");
        for (int i = 0; i < scores.length; i++) {
            System.out.println("  " + (i + 1) + "人目: " + scores[i] + "点");
        }
        System.out.println("合計: " + sum + "点");
        System.out.println("平均: " + average + "点");
    }
}

期待される結果

点数一覧:
  1人目: 85点
  2人目: 72点
  3人目: 90点
  4人目: 65点
  5人目: 78点
合計: 390点
平均: 78.0点

5.5 配列の便利な使い方

最大値・最小値を見つける

最大値・最小値 
int[] scores = {85, 72, 90, 65, 78};

// 最大値を見つける
int max = scores[0];  // 最初の要素を仮の最大値とする
for (int score : scores) {
    if (score > max) {
        max = score;  // より大きい値が見つかったら更新
    }
}
System.out.println("最高点: " + max);  // 90

// 最小値を見つける
int min = scores[0];
for (int score : scores) {
    if (score < min) {
        min = score;
    }
}
System.out.println("最低点: " + min);  // 65

特定の値を検索する

検索 
String[] names = {"田中", "佐藤", "鈴木", "高橋", "伊藤"};

// 「鈴木」を探す
String target = "鈴木";
int foundIndex = -1;  // 見つからなかった場合は-1

for (int i = 0; i < names.length; i++) {
    if (names[i].equals(target)) {
        foundIndex = i;
        break;  // 見つかったらループを抜ける
    }
}

if (foundIndex != -1) {
    System.out.println(target + "さんは " + foundIndex + " 番目にいます");
} else {
    System.out.println(target + "さんは見つかりませんでした");
}

実習 5-2: 成績判定プログラムを作ろう

点数に応じて成績（優・良・可・不可）を判定し、一覧表示するプログラムを作成します。

GradeChecker.java 
public class GradeChecker {
    public static void main(String[] args) {
        int[] scores = {85, 72, 90, 55, 78, 42, 68};
        String[] names = {"田中", "佐藤", "鈴木", "高橋", "伊藤", "山田", "中村"};

        System.out.println("=== 成績一覧 ===");

        for (int i = 0; i < scores.length; i++) {
            String grade;

            // 点数に応じて成績を判定
            if (scores[i] >= 80) {
                grade = "優";
            } else if (scores[i] >= 70) {
                grade = "良";
            } else if (scores[i] >= 60) {
                grade = "可";
            } else {
                grade = "不可";
            }

            System.out.println(names[i] + "さん: " + scores[i] + "点 → " + grade);
        }
    }
}

期待される結果

=== 成績一覧 ===
田中さん: 85点 → 優
佐藤さん: 72点 → 良
鈴木さん: 90点 → 優
高橋さん: 55点 → 不可
伊藤さん: 78点 → 良
山田さん: 42点 → 不可
中村さん: 68点 → 可

5.6 多次元配列（発展）

配列の中に配列を入れることもできます。これを多次元配列（2次元配列など）と呼びます。 表形式のデータを扱うときに便利です。

2次元配列のイメージ

1次元配列が「一列に並んだロッカー」だとすると、 2次元配列は「縦横に並んだ表」や「マンションの部屋」のようなイメージです。

2次元配列 
// 2次元配列の宣言と初期化
// 3人の生徒の、3科目の点数を管理
int[][] scores = {
    {80, 75, 90},  // 1人目の国語、数学、英語
    {70, 85, 80},  // 2人目
    {90, 65, 75}   // 3人目
};

// 要素へのアクセス [行][列]
System.out.println(scores[0][0]);  // 80 (1人目の国語)
System.out.println(scores[1][2]);  // 80 (2人目の英語)

// 2重ループで全要素を処理
for (int i = 0; i < scores.length; i++) {
    System.out.print((i + 1) + "人目: ");
    for (int j = 0; j < scores[i].length; j++) {
        System.out.print(scores[i][j] + " ");
    }
    System.out.println();
}

※ 2次元配列は少し難しい内容なので、今は「こういうものがある」と知っておく程度で大丈夫です。 実際に必要になったときに詳しく学びましょう。

理解度確認クイズ

  1. Q1. 配列のインデックスは何番から始まりますか？

    回答を表示

    0番から始まります。 例えば要素数5の配列のインデックスは0〜4です。

  2. Q2. int[] nums = {10, 20, 30}; のとき、nums.length の値は？

    回答を表示

    3です。 lengthは配列の要素数を返します。

  3. Q3. 拡張forループの利点は何ですか？

    回答を表示

    インデックスを管理する必要がなく、シンプルに全要素を順番に処理できます。 配列の範囲外アクセスのミスも起きにくくなります。

  4. Q4. ArrayIndexOutOfBoundsExceptionはどんなときに発生しますか？

    回答を表示

    存在しないインデックス（範囲外）にアクセスしようとしたときに発生します。 例：要素数3の配列でarray[3]やarray[-1]にアクセス

5.7 この章のまとめ

  • 配列は、同じ型の複数のデータをまとめて管理できる入れ物。
  • インデックス（添え字）は0から始まるので注意。
  • 配列名.lengthで要素数を取得できる。
  • forループと組み合わせると、配列の全要素を効率的に処理できる。
  • 拡張forループ（for-each文）で、よりシンプルに配列を処理できる。
  • 範囲外のインデックスにアクセスするとArrayIndexOutOfBoundsExceptionエラーが発生する。
