第10章

例外処理とJava APIの活用

エラーに強いプログラムを作るための「例外処理」と、実践的なプログラミングに必要なJava標準APIの使い方を学びます。

この章で学ぶこと

  • 例外処理の仕組みを理解し、try-catch-finally構文を正しく使える
  • 例外の発生と伝播、独自例外の作成方法を理解する
  • 主要なJava標準API（String、Math、ArrayList、Scanner）を活用できる
  • 実践的なプログラムを作成し、これまでの知識を統合できる

10.1 例外とは何か

例外（Exception）は、プログラム実行中に発生する予期しない問題やエラーを表現する仕組みです。適切に例外を処理することで、プログラムが突然終了することを防ぎ、エラー発生時にも適切な対応ができるようになります。

例外が発生する場面

  • 配列の範囲外アクセス: 存在しないインデックスにアクセス（ArrayIndexOutOfBoundsException）
  • null参照: nullオブジェクトのメソッド呼び出し（NullPointerException）
  • 数値変換エラー: 文字列を数値に変換できない（NumberFormatException）
  • ファイル操作エラー: 存在しないファイルへのアクセス（FileNotFoundException）
  • 算術エラー: ゼロによる除算（ArithmeticException）

例外の階層構造

Javaのすべての例外は、Throwableクラスを頂点とする階層構造になっています。

検査例外と非検査例外

検査例外（Checked Exception）:

  • • コンパイル時にチェックされる例外
  • • 必ずtry-catchで処理するか、throwsで宣言する必要がある
  • • 例: IOException, SQLException, FileNotFoundException

非検査例外（Unchecked Exception）:

  • • RuntimeExceptionとそのサブクラス
  • • コンパイル時にチェックされない
  • • プログラムのバグによって発生することが多い
  • • 例: NullPointerException, ArrayIndexOutOfBoundsException

10.2 例外処理の基本 - try-catch-finally

例外処理の基本は、try-catch-finally構文です。tryブロック内で例外が発生する可能性のあるコードを実行し、catchブロックで例外を捕捉して処理します。

基本的な構文

BasicExceptionHandling.java 
public class BasicExceptionHandling {
    public static void main(String[] args) {
        try {
            // 例外が発生する可能性のあるコード
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[5]);  // 範囲外アクセス

        } catch (ArrayIndexOutOfBoundsException e) {
            // 例外が発生した場合の処理
            System.out.println("エラー: 配列の範囲外にアクセスしました");
            System.out.println("エラーメッセージ: " + e.getMessage());
        }

        System.out.println("プログラムは正常に続行されます");
    }
}

// 実行結果:
// エラー: 配列の範囲外にアクセスしました
// エラーメッセージ: Index 5 out of bounds for length 3
// プログラムは正常に続行されます

複数のcatchブロック

複数の種類の例外を処理したい場合、複数のcatchブロックを記述できます。より具体的な例外から順に記述する必要があります。

MultipleCatch.java 
import java.util.Scanner;

public class MultipleCatch {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        try {
            System.out.print("整数を入力してください: ");
            String input = scanner.nextLine();

            // 文字列を整数に変換
            int number = Integer.parseInt(input);

            // 計算を実行
            int result = 100 / number;
            System.out.println("100 ÷ " + number + " = " + result);

        } catch (NumberFormatException e) {
            // 数値変換エラー
            System.out.println("エラー: 整数を入力してください");

        } catch (ArithmeticException e) {
            // 算術エラー（ゼロ除算）
            System.out.println("エラー: ゼロでは割り算できません");

        } catch (Exception e) {
            // その他すべての例外
            System.out.println("予期しないエラーが発生しました");
            e.printStackTrace();  // スタックトレースを表示
        }

        scanner.close();
    }
}

finallyブロック

finallyブロックは、例外が発生してもしなくても必ず実行されます。リソースのクリーンアップ（ファイルのクローズなど）によく使用されます。

FinallyExample.java 
import java.util.Scanner;

public class FinallyExample {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        try {
            System.out.print("数値を入力: ");
            int number = scanner.nextInt();
            System.out.println("入力された数値: " + number);

        } catch (Exception e) {
            System.out.println("エラーが発生しました");

        } finally {
            // 必ず実行される（リソースのクリーンアップ）
            System.out.println("Scannerをクローズします");
            scanner.close();
        }

        System.out.println("プログラム終了");
    }
}

try-with-resources構文（Java 7以降）

Java 7以降では、try-with-resources構文を使うことで、リソースの自動クローズが可能です。finallyブロックでのクローズ処理を書く必要がなくなります。 

// try-with-resources構文
try (Scanner scanner = new Scanner(System.in)) {
    int number = scanner.nextInt();
    System.out.println("入力: " + number);
}  // scannerは自動的にクローズされる

10.3 例外の発生と伝播

プログラムで意図的に例外を発生させたり、呼び出し元に例外を伝播させたりする方法を学びます。

throwによる例外の発生

throwキーワードを使用して、意図的に例外を発生させることができます。

BankAccount.java 
public class BankAccount {
    private double balance;

    public BankAccount(double initialBalance) {
        this.balance = initialBalance;
    }

    // お金を引き出すメソッド
    public void withdraw(double amount) {
        if (amount <= 0) {
            // 不正な金額の場合、例外を発生させる
            throw new IllegalArgumentException("引き出し金額は正の数でなければなりません");
        }

        if (amount > balance) {
            // 残高不足の場合、例外を発生させる
            throw new IllegalStateException("残高不足です。現在の残高: " + balance);
        }

        balance -= amount;
        System.out.println(amount + "円を引き出しました。残高: " + balance);
    }

    public double getBalance() {
        return balance;
    }
}

throwsによる例外の宣言

throwsキーワードを使用して、メソッドが特定の例外を投げる可能性があることを宣言します。検査例外の場合は必須です。

FileProcessor.java 
import java.io.*;

public class FileProcessor {

    // 検査例外をthrowsで宣言
    public String readFile(String filename) throws IOException {
        BufferedReader reader = new BufferedReader(new FileReader(filename));
        StringBuilder content = new StringBuilder();
        String line;

        while ((line = reader.readLine()) != null) {
            content.append(line).append("\n");
        }

        reader.close();
        return content.toString();
    }

    public static void main(String[] args) {
        FileProcessor processor = new FileProcessor();

        try {
            // 呼び出し元で例外を処理
            String content = processor.readFile("example.txt");
            System.out.println(content);

        } catch (FileNotFoundException e) {
            System.out.println("ファイルが見つかりません: " + e.getMessage());

        } catch (IOException e) {
            System.out.println("ファイル読み込みエラー: " + e.getMessage());
        }
    }
}

独自例外クラスの作成

業務ロジック固有のエラーを表現するために、独自の例外クラスを作成できます。

InsufficientBalanceException.java 
// 独自の例外クラス
public class InsufficientBalanceException extends Exception {
    private double balance;
    private double requestedAmount;

    public InsufficientBalanceException(double balance, double requestedAmount) {
        super("残高不足: 残高=" + balance + "、要求額=" + requestedAmount);
        this.balance = balance;
        this.requestedAmount = requestedAmount;
    }

    public double getBalance() {
        return balance;
    }

    public double getRequestedAmount() {
        return requestedAmount;
    }

    public double getShortage() {
        return requestedAmount - balance;
    }
}

// 使用例
public class BankAccount2 {
    private double balance;

    public BankAccount2(double initialBalance) {
        this.balance = initialBalance;
    }

    public void withdraw(double amount) throws InsufficientBalanceException {
        if (amount > balance) {
            throw new InsufficientBalanceException(balance, amount);
        }
        balance -= amount;
        System.out.println(amount + "円を引き出しました");
    }

    public static void main(String[] args) {
        BankAccount2 account = new BankAccount2(1000);

        try {
            account.withdraw(1500);

        } catch (InsufficientBalanceException e) {
            System.out.println("エラー: " + e.getMessage());
            System.out.println("不足額: " + e.getShortage() + "円");
        }
    }
}

10.4 主要なJava API

Java標準ライブラリには、実践的なプログラミングに必要な多くのクラスが用意されています。ここでは特に重要なAPIを紹介します。

Stringクラスの主要メソッド

StringMethods.java 
public class StringMethods {
    public static void main(String[] args) {
        String text = "  Hello, Java World!  ";

        // 長さの取得
        System.out.println("長さ: " + text.length());

        // 前後の空白を削除
        System.out.println("trim: " + text.trim());

        // 大文字・小文字変換
        System.out.println("大文字: " + text.toUpperCase());
        System.out.println("小文字: " + text.toLowerCase());

        // 部分文字列の取得
        System.out.println("substring: " + text.substring(2, 7));

        // 文字列の置換
        System.out.println("replace: " + text.replace("Java", "Python"));

        // 文字列の分割
        String[] words = text.trim().split(" ");
        System.out.println("分割: " + java.util.Arrays.toString(words));

        // 文字列の検索
        System.out.println("contains: " + text.contains("Java"));
        System.out.println("indexOf: " + text.indexOf("Java"));
        System.out.println("startsWith: " + text.trim().startsWith("Hello"));

        // 文字列の比較
        System.out.println("equals: " + text.trim().equals("Hello, Java World!"));
        System.out.println("equalsIgnoreCase: " + text.trim().equalsIgnoreCase("hello, java world!"));
    }
}

Mathクラスの主要メソッド

MathMethods.java 
public class MathMethods {
    public static void main(String[] args) {
        // 絶対値
        System.out.println("abs(-10): " + Math.abs(-10));

        // 最大値・最小値
        System.out.println("max(5, 10): " + Math.max(5, 10));
        System.out.println("min(5, 10): " + Math.min(5, 10));

        // べき乗と平方根
        System.out.println("pow(2, 3): " + Math.pow(2, 3));  // 2の3乗
        System.out.println("sqrt(16): " + Math.sqrt(16));    // 平方根

        // 四捨五入
        System.out.println("round(3.7): " + Math.round(3.7));
        System.out.println("ceil(3.2): " + Math.ceil(3.2));   // 切り上げ
        System.out.println("floor(3.8): " + Math.floor(3.8)); // 切り捨て

        // 乱数生成（0.0以上1.0未満）
        System.out.println("random: " + Math.random());

        // 1〜10の乱数を生成
        int randomNumber = (int)(Math.random() * 10) + 1;
        System.out.println("1〜10の乱数: " + randomNumber);

        // 定数
        System.out.println("円周率 PI: " + Math.PI);
        System.out.println("自然対数の底 E: " + Math.E);
    }
}

ArrayListクラス（可変長配列）

ArrayListExample.java 
import java.util.ArrayList;

public class ArrayListExample {
    public static void main(String[] args) {
        // ArrayListの作成（型パラメータで要素の型を指定）
        ArrayList<String> fruits = new ArrayList<>();

        // 要素の追加
        fruits.add("りんご");
        fruits.add("バナナ");
        fruits.add("オレンジ");
        System.out.println("fruits: " + fruits);

        // 要素の取得
        System.out.println("0番目: " + fruits.get(0));

        // 要素数の取得
        System.out.println("要素数: " + fruits.size());

        // 要素の削除
        fruits.remove("バナナ");  // 値で削除
        fruits.remove(0);         // インデックスで削除
        System.out.println("削除後: " + fruits);

        // 要素の存在確認
        System.out.println("contains: " + fruits.contains("オレンジ"));

        // すべての要素を走査
        for (String fruit : fruits) {
            System.out.println("- " + fruit);
        }

        // リストのクリア
        fruits.clear();
        System.out.println("クリア後: " + fruits.isEmpty());
    }
}

理解度確認クイズ

  1. Q1. 検査例外と非検査例外の違いを説明してください。

    回答を表示

    検査例外（Checked Exception）: コンパイル時にチェックされる例外で、必ずtry-catchで処理するか、throwsで宣言する必要があります。IOException、SQLExceptionなどが該当します。

    非検査例外（Unchecked Exception）: RuntimeExceptionとそのサブクラスで、コンパイル時にチェックされません。プログラムのバグによって発生することが多く、NullPointerException、ArrayIndexOutOfBoundsExceptionなどが該当します。

  2. Q2. finally節の役割と、どのような場合に実行されるかを説明してください。

    回答を表示

    finally節は、例外が発生してもしなくても必ず実行される処理を記述する場所です。主にリソースのクリーンアップ（ファイルのクローズ、データベース接続のクローズなど）に使用されます。

    try節で例外が発生しない場合でも、例外が発生してcatch節で処理された場合でも、finally節は必ず実行されます。ただし、System.exit()でプログラムが強制終了された場合や、無限ループなどで処理が完了しない場合は実行されません。

  3. Q3. throwとthrowsの違いを説明してください。

    回答を表示

    throw: メソッド内で例外を意図的に発生させるためのキーワードです。throw new Exception()のように使用します。

    throws: メソッドシグネチャに記述し、そのメソッドが特定の例外を投げる可能性があることを宣言するキーワードです。public void method() throws IOExceptionのように使用します。検査例外の場合は必ず宣言が必要です。

  4. Q4. 配列とArrayListの主な違いを3つ挙げてください。

    回答を表示

    1. サイズ: 配列はサイズが固定ですが、ArrayListは要素の追加・削除によってサイズが動的に変わります

    2. 要素の型: 配列はプリミティブ型と参照型の両方を扱えますが、ArrayListは参照型のみです（プリミティブ型はラッパークラスを使用）

    3. メソッド: 配列はlengthプロパティのみですが、ArrayListはadd、remove、get、sizeなど豊富なメソッドを提供します

  5. Q5. try-with-resources構文の利点を説明してください。

    回答を表示

    try-with-resources構文（Java 7以降）の利点：

    1. 自動クローズ: AutoCloseableインターフェースを実装したリソースを自動的にクローズしてくれるため、finally節でのクローズ処理が不要になります

    2. コードの簡潔性: リソース管理のコードが簡潔になり、可読性が向上します

    3. エラー処理の改善: リソースのクローズ時に発生した例外も適切に処理され、元の例外が隠蔽されません

    使用例: try (Scanner sc = new Scanner(System.in)) { ... }

  • 検査例外は必ず処理が必要、非検査例外はプログラムのバグとして扱う
