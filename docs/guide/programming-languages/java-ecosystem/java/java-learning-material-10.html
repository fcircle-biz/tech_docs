継承とポリモーフィズム

クラスの継承、メソッドのオーバーライド、ポリモーフィズム（多態性）について学びます。コードの再利用性を高め、柔軟な設計を実現する方法を理解します。

この章で学ぶこと

  • 継承の概念とis-a関係の理解
  • スーパークラスとサブクラスの定義方法
  • メソッドのオーバーライドと@Overrideアノテーション
  • ポリモーフィズム（多態性）の活用方法

10.1 継承とは何か

継承（Inheritance）は、既存のクラス（スーパークラス・親クラス）の機能を引き継いで、新しいクラス（サブクラス・子クラス）を作成する仕組みです。継承を使うことで、共通する機能を再利用しながら、それぞれのクラスに特化した機能を追加できます。

継承はis-a関係を表現します。例えば、「犬は動物である」「猫は動物である」という関係を、「Dogクラス」と「Catクラス」が「Animalクラス」を継承する、という形でコードに表現します。

classDiagram Animal <|-- Dog Animal <|-- Cat class Animal { -String name +eat() +sleep() } class Dog { +bark() } class Cat { +meow() }

継承のメリット

継承を使うことで、共通の機能を一箇所にまとめて記述し、複数のクラスで再利用できます。コードの重複を避け、保守性を高めることができます。

10.2 継承の基本構文

Javaではextendsキーワードを使ってクラスを継承します。サブクラスは、スーパークラスのpublicおよびprotectedなフィールドとメソッドを引き継ぎます。

スーパークラスの定義

Animal.java 
public class Animal {
    // フィールド
    protected String name;

    // コンストラクタ
    public Animal(String name) {
        this.name = name;
    }

    // メソッド
    public void eat() {
        System.out.println(name + "が食事しています");
    }

    public void sleep() {
        System.out.println(name + "が眠っています");
    }
}

サブクラスの定義

Dog.java 
public class Dog extends Animal {
    // コンストラクタ
    public Dog(String name) {
        super(name);  // スーパークラスのコンストラクタを呼び出す
    }

    // Dogクラス独自のメソッド
    public void bark() {
        System.out.println(name + "が吠えています: ワンワン!");
    }
}

superキーワード

super(name)は、スーパークラスのコンストラクタを呼び出します。サブクラスのコンストラクタの最初の行で呼び出す必要があります。

継承したクラスの使用

Main.java 
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog("ポチ");

        // スーパークラスから継承したメソッド
        dog.eat();    // ポチが食事しています
        dog.sleep();  // ポチが眠っています

        // Dogクラス独自のメソッド
        dog.bark();   // ポチが吠えています: ワンワン!
    }
}

10.3 メソッドのオーバーライド

オーバーライド（Override）は、サブクラスでスーパークラスのメソッドを再定義する機能です。同じ名前とシグネチャ（引数の型・順序・個数）のメソッドを定義することで、サブクラス独自の実装に置き換えることができます。

Cat.java 
public class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    // Animalクラスのeat()メソッドをオーバーライド
    @Override
    public void eat() {
        System.out.println(name + "が魚を食べています");
    }

    // Catクラス独自のメソッド
    public void meow() {
        System.out.println(name + "が鳴いています: ニャー!");
    }
}

@Overrideアノテーション

@Overrideアノテーションは必須ではありませんが、付けることでコンパイラがオーバーライドが正しく行われているかをチェックしてくれます。メソッド名や引数を間違えた場合にエラーを検出できます。

オーバーライドの動作確認

Main.java 
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog("ポチ");
        Cat cat = new Cat("タマ");

        // Dogのeat()はAnimalクラスの実装
        dog.eat();  // ポチが食事しています

        // Catのeat()はオーバーライドされた実装
        cat.eat();  // タマが魚を食べています
    }
}

10.4 ポリモーフィズム（多態性）

ポリモーフィズム（Polymorphism）は、スーパークラス型の変数でサブクラスのオブジェクトを参照できる機能です。これにより、同じメソッド呼び出しで異なる実装を実行することができます。

ポリモーフィズムの例

PolymorphismDemo.java 
public class PolymorphismDemo {
    public static void main(String[] args) {
        // スーパークラス型の変数でサブクラスを参照
        Animal animal1 = new Dog("ポチ");
        Animal animal2 = new Cat("タマ");

        // 同じeat()メソッド呼び出しで異なる動作
        animal1.eat();  // ポチが食事しています
        animal2.eat();  // タマが魚を食べています

        // 配列でまとめて扱うことも可能
        Animal[] animals = {
            new Dog("シロ"),
            new Cat("ミケ"),
            new Dog("クロ")
        };

        for (Animal animal : animals) {
            animal.eat();
        }
    }
}

ポリモーフィズムの利点

ポリモーフィズムを使うと、異なる種類のオブジェクトを統一的に扱えるようになり、コードの柔軟性と拡張性が向上します。新しいサブクラスを追加しても、既存のコードを変更する必要がありません。

10.5 型キャストとinstanceof演算子

ポリモーフィズムでスーパークラス型の変数でサブクラスを参照している場合、サブクラス固有のメソッドを呼び出すにはダウンキャスト（型変換）が必要です。その際、instanceof演算子で型を確認してから安全にキャストします。

CastingDemo.java 
public class CastingDemo {
    public static void main(String[] args) {
        Animal animal = new Dog("ポチ");

        // instanceof演算子で型を確認
        if (animal instanceof Dog) {
            // ダウンキャスト
            Dog dog = (Dog) animal;
            dog.bark();  // ポチが吠えています: ワンワン!
        }

        // instanceof演算子でCatではないことを確認
        if (animal instanceof Cat) {
            Cat cat = (Cat) animal;
            cat.meow();  // 実行されない
        }
    }
}

注意: ClassCastException

instanceof演算子で型を確認せずにキャストすると、実行時にClassCastExceptionが発生する可能性があります。必ず型チェックを行ってからキャストしましょう。

10.6 継承の注意点

1. Javaは単一継承のみサポート

Javaでは、1つのクラスは1つのスーパークラスしか継承できません（単一継承）。複数のクラスから機能を引き継ぎたい場合は、次章で学ぶインターフェースを使用します。 

// これはエラー: 複数のクラスを継承できない
// public class MyClass extends ClassA, ClassB { }

// 正しい方法
public class MyClass extends ClassA {
    // 1つのクラスのみ継承可能
}

2. 継承の乱用に注意

継承は強力な機能ですが、深い継承階層は複雑性を増し、コードの理解を難しくします。is-a関係が明確な場合のみ継承を使い、機能の共有が目的の場合はコンポジション（has-a関係）を検討しましょう。

継承 vs コンポジション

  • • 継承（is-a関係）: 「犬は動物である」→ Dog extends Animal
  • • コンポジション（has-a関係）: 「車はエンジンを持つ」→ Car has Engine

実習 10-1: 図形クラスの継承

継承とポリモーフィズムを使って、図形を表すクラス階層を作成しましょう。

課題

Shapeスーパークラスを継承して、Circle（円）とRectangle（長方形）クラスを作成し、面積を計算するプログラムを実装してください。

仕様

  1. 1. Shapeクラス: 抽象的な図形を表す
  2. 2. getArea()メソッド: 面積を返す（サブクラスでオーバーライド）
  3. 3. Circleクラス: 半径を持ち、面積 = π × 半径²
  4. 4. Rectangleクラス: 幅と高さを持ち、面積 = 幅 × 高さ

実装例

// Shape.java
public class Shape {
    public double getArea() {
        return 0.0;
    }
}

// Circle.java
public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }
}

// Rectangle.java
public class Rectangle extends Shape {
    private double width;
    private double height;

    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }

    @Override
    public double getArea() {
        return width * height;
    }
}

動作確認

public class Main {
    public static void main(String[] args) {
        // ポリモーフィズムの活用
        Shape[] shapes = {
            new Circle(5.0),
            new Rectangle(4.0, 6.0)
        };

        for (Shape shape : shapes) {
            System.out.printf("面積: %.2f\n", shape.getArea());
        }
    }
}

理解度確認クイズ

  1. Q1. Javaでクラスを継承するために使うキーワードは何ですか？

    回答を表示

    extendsキーワードを使います。
    例: public class Dog extends Animal { }

  2. Q2. メソッドのオーバーライドとは何ですか？また、オーバーライド時に付けるべきアノテーションは何ですか？

    回答を表示

    オーバーライドは、サブクラスでスーパークラスのメソッドを再定義することです。
    @Overrideアノテーションを付けることで、コンパイラがオーバーライドの正確性をチェックしてくれます。

  3. Q3. ポリモーフィズムを使うと、どのような利点がありますか？

    回答を表示

    異なる種類のオブジェクトを統一的に扱えるため、コードの柔軟性と拡張性が向上します。新しいサブクラスを追加しても既存のコードを変更する必要がありません。

  4. Q4. オブジェクトの型を確認するために使う演算子は何ですか？

    回答を表示

    instanceof演算子を使います。
    例: if (animal instanceof Dog) { ... }
    型キャストの前に型を確認することで、ClassCastExceptionを防げます。

  • extendsキーワードでクラスを継承し、superでスーパークラスのコンストラクタを呼び出す
  • メソッドのオーバーライドで、サブクラス独自の実装を提供できる（@Override推奨）
  • ポリモーフィズムにより、スーパークラス型の変数で異なるサブクラスを統一的に扱える
  • instanceof演算子で型を確認してから安全に型キャストする
