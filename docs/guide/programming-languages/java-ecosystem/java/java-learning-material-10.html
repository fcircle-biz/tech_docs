クラスの親子関係 - 継承

既存のクラスの機能を引き継いで新しいクラスを作る「継承」について学び、コードの再利用を理解します。

この章で学ぶこと

  • 継承の仕組みと、なぜコードの再利用が重要かを理解する
  • extendsキーワードを使って親クラスと子クラスを作成する
  • superキーワードで親クラスのメソッドやコンストラクタを呼び出す
  • メソッドのオーバーライド（上書き）を理解する

10.1 継承って何だろう？

これまでに学んだクラスとオブジェクトの概念を思い出してください。 クラスは「設計図」、オブジェクトは「実体」でした。 では、似たような設計図を複数作りたいとき、毎回ゼロから書くのは大変ですよね。

そこで登場するのが継承（inheritance：インヘリタンス）です。 継承とは、既存のクラスの機能（フィールドとメソッド）を引き継いで、 新しいクラスを作る仕組みのことです。

たとえ話: 家族の特徴を受け継ぐ

現実世界

あなたは親から特徴を受け継いでいます。
目の色、髪の色、性格の一部など...親の特徴を持ちながら、 自分独自の特徴も持っています。

プログラミング世界

子クラスは親クラスの機能を受け継ぎます。
親クラスのフィールド・メソッドを持ちながら、 子クラス独自の機能も追加できます。

継承の用語

  • ・親クラス（スーパークラス、基底クラス）：機能を提供する元のクラス
  • ・子クラス（サブクラス、派生クラス）：機能を受け継ぐ新しいクラス
  • ・extends：継承を行うためのキーワード

10.2 継承の基本構文

継承を使うには、extendsキーワードを使います。

継承の基本構文 
// 親クラス
class 親クラス名 {
    // フィールドとメソッド
}

// 子クラス（親クラスを継承）
class 子クラス名 extends 親クラス名 {
    // 親の機能を引き継ぎ、さらに独自機能を追加
}

具体例: 動物クラスを継承する

「動物」という親クラスから、「犬」「猫」という子クラスを作ってみましょう。

継承の関係図

classDiagram Animal <|-- Dog Animal <|-- Cat class Animal { +String name +void eat() } class Dog { +void bark() } class Cat { +void meow() }
Animal.java（親クラス） 
// 親クラス: 動物
class Animal {
    String name;  // すべての動物は名前を持つ

    // すべての動物が食べる
    void eat() {
        System.out.println(name + "が食事をしています");
    }
}
Dog.java（子クラス） 
// 子クラス: 犬（動物を継承）
class Dog extends Animal {
    // 犬独自の機能: 吠える
    void bark() {
        System.out.println(name + "がワンワン！と吠えています");
    }
}
Cat.java（子クラス） 
// 子クラス: 猫（動物を継承）
class Cat extends Animal {
    // 猫独自の機能: 鳴く
    void meow() {
        System.out.println(name + "がニャーと鳴いています");
    }
}

ポイント

  • ・DogクラスとCatクラスはnameフィールドを持たなくても、親クラスから継承している
  • ・同様にeat()メソッドも使える
  • ・それぞれが独自のメソッド（bark、meow）を追加している

実習 10-1: 継承を使ってみよう

動物クラスを継承して犬・猫クラスを作り、実際に動かしてみましょう。

InheritanceDemo.java 
// 親クラス: 動物
class Animal {
    String name;

    void eat() {
        System.out.println(name + "が食事をしています");
    }
}

// 子クラス: 犬
class Dog extends Animal {
    void bark() {
        System.out.println(name + "がワンワン！と吠えています");
    }
}

// 子クラス: 猫
class Cat extends Animal {
    void meow() {
        System.out.println(name + "がニャーと鳴いています");
    }
}

// メインクラス
public class InheritanceDemo {
    public static void main(String[] args) {
        // 犬オブジェクトを作成
        Dog myDog = new Dog();
        myDog.name = "ポチ";
        myDog.eat();   // 親クラスのメソッド
        myDog.bark();  // 子クラスのメソッド

        System.out.println();

        // 猫オブジェクトを作成
        Cat myCat = new Cat();
        myCat.name = "タマ";
        myCat.eat();   // 親クラスのメソッド
        myCat.meow();  // 子クラスのメソッド
    }
}

10.3 コンストラクタと継承

継承したクラスでコンストラクタを使う場合、親クラスのコンストラクタも考慮する必要があります。 子クラスのコンストラクタでは、親クラスのコンストラクタを呼び出すために super()というキーワードを使います。

superキーワード

super()は親クラスのコンストラクタを呼び出します。
子クラスのコンストラクタの最初の行で使う必要があります。

コンストラクタを持つ継承の例 
// 親クラス
class Animal {
    String name;

    // 親クラスのコンストラクタ
    Animal(String name) {
        this.name = name;
        System.out.println("動物が作られました: " + name);
    }
}

// 子クラス
class Dog extends Animal {
    String breed;  // 犬種

    // 子クラスのコンストラクタ
    Dog(String name, String breed) {
        super(name);  // 親クラスのコンストラクタを呼ぶ
        this.breed = breed;
        System.out.println("犬種: " + breed);
    }
}

ポイント

  • ・子クラスのコンストラクタでsuper()を使って親のコンストラクタを呼ぶ
  • ・super()は子クラスのコンストラクタの最初に書く
  • ・親クラスの初期化が必要なフィールドは、親のコンストラクタで初期化する

10.4 メソッドのオーバーライド（上書き）

子クラスでは、親クラスのメソッドをオーバーライド（上書き）することができます。 これにより、親クラスと同じ名前のメソッドを、子クラス独自の動作に変更できます。

たとえ話: レシピのアレンジ

親クラスのレシピ

基本のカレーのレシピ
「野菜を炒めて、水を加えて、ルーを溶かす」

子クラスのアレンジ

辛口カレー用にアレンジ
「野菜を炒めて、水を加えて、ルーを溶かし、唐辛子を追加」

メソッドのオーバーライドの例 
// 親クラス
class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    // 親クラスのメソッド
    void makeSound() {
        System.out.println(name + "が音を出しています");
    }
}

// 子クラス
class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    // メソッドをオーバーライド（上書き）
    @Override
    void makeSound() {
        System.out.println(name + "がワンワン！と吠えます");
    }
}

@Overrideアノテーション

メソッドの前に@Overrideを付けると、「このメソッドは親クラスのメソッドを上書きしている」ことを明示できます。 これにより、間違いを防ぎ、コードの意図が分かりやすくなります。

実習 10-2: オーバーライドを使ってみよう

親クラスのメソッドをオーバーライドして、動物ごとに異なる鳴き声を実装しましょう。

OverrideDemo.java 
// 親クラス
class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    void makeSound() {
        System.out.println(name + "が音を出しています");
    }
}

// 子クラス: 犬
class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + "がワンワン！");
    }
}

// 子クラス: 猫
class Cat extends Animal {
    Cat(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + "がニャー");
    }
}

public class OverrideDemo {
    public static void main(String[] args) {
        Animal animal = new Animal("動物");
        Dog dog = new Dog("ポチ");
        Cat cat = new Cat("タマ");

        animal.makeSound();  // 親クラス版
        dog.makeSound();     // 犬版（オーバーライド）
        cat.makeSound();     // 猫版（オーバーライド）
    }
}

期待される結果

動物が音を出しています
ポチがワンワン！
タマがニャー

10.5 親クラスのメソッドを呼び出す - super

オーバーライドしたメソッドの中から、親クラスの元のメソッドを呼び出したい場合があります。 その場合はsuper.メソッド名()を使います。

super.メソッド名()の例 
class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    void introduce() {
        System.out.println("私は動物です。名前は" + name);
    }
}

class Dog extends Animal {
    String breed;

    Dog(String name, String breed) {
        super(name);
        this.breed = breed;
    }

    @Override
    void introduce() {
        super.introduce();  // 親クラスの処理を呼ぶ
        System.out.println("犬種は" + breed + "です");
    }
}

ポイント

  • ・super.introduce()で親クラスの処理を実行できる
  • ・親の処理に「追加」したい場合に便利
  • ・完全に置き換えたい場合は、superを使わずに新しい処理を書く

10.6 継承とアクセス修飾子 - protected

第9章で学んだアクセス修飾子を思い出してください。 publicは全公開、privateは自クラスのみでした。 継承では、protectedという新しい修飾子がよく使われます。

アクセス修飾子 自クラス 子クラス 他クラス
public ○ ○ ○
protected ○ ○ ×
private ○ × ×

protectedの使いどころ

protectedは「子クラスには使わせたいけど、外部には非公開にしたい」ときに使います。 継承を前提とした設計で便利です。

protectedの使用例 
class Animal {
    protected String name;  // 子クラスからアクセス可能

    protected void sleep() {  // 子クラスから呼び出し可能
        System.out.println(name + "が眠っています");
    }
}

class Dog extends Animal {
    void rest() {
        // protectedなので子クラスから使える
        this.name = "ポチ";
        sleep();
    }
}

動作確認: 総合的な継承の例

これまで学んだ継承の機能をすべて使った総合的なプログラムを実行してみましょう。

ComprehensiveInheritance.java 
class Animal {
    protected String name;
    protected int age;

    Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void introduce() {
        System.out.println("名前: " + name + ", 年齢: " + age + "歳");
    }

    void makeSound() {
        System.out.println("何か音を出しています");
    }
}

class Dog extends Animal {
    private String breed;

    Dog(String name, int age, String breed) {
        super(name, age);  // 親クラスのコンストラクタ呼び出し
        this.breed = breed;
    }

    @Override
    void introduce() {
        super.introduce();  // 親の処理を実行
        System.out.println("犬種: " + breed);
    }

    @Override
    void makeSound() {
        System.out.println(name + "がワンワン！");
    }

    void fetch() {
        System.out.println(name + "がボールを取ってきました");
    }
}

public class ComprehensiveInheritance {
    public static void main(String[] args) {
        Dog myDog = new Dog("ポチ", 3, "柴犬");

        myDog.introduce();  // オーバーライドされた版
        myDog.makeSound();  // オーバーライドされた版
        myDog.fetch();      // 子クラス独自のメソッド
    }
}

犬種: 柴犬
ポチがワンワン！
ポチがボールを取ってきました

理解度確認クイズ

  1. Q1. 継承を行うために使うキーワードは何ですか？

    回答を表示

    extendsキーワードです。
    例: class Dog extends Animal

  2. Q2. 子クラスのコンストラクタで親クラスのコンストラクタを呼び出すには？

    回答を表示

    super()を使います。
    子クラスのコンストラクタの最初の行に書きます。

  3. Q3. メソッドのオーバーライドとは何ですか？

    回答を表示

    親クラスのメソッドと同じ名前のメソッドを子クラスで定義し、親のメソッドを上書き（置き換え）することです。 @Overrideアノテーションを付けて明示的に示すのが推奨されます。

  4. Q4. protectedアクセス修飾子の特徴は？

    回答を表示

    自クラスと子クラスからはアクセスできますが、他のクラスからはアクセスできません。 継承を前提とした設計でよく使われます。

10.7 この章のまとめ

  • 継承は既存クラスの機能を引き継いで新しいクラスを作る仕組み。
  • extendsキーワードで継承を行う。子クラスは親クラスのフィールド・メソッドを使える。
  • super()で親クラスのコンストラクタを呼び出す（最初の行に書く）。
  • オーバーライドで親クラスのメソッドを子クラス独自の動作に上書きできる。
  • super.メソッド名()で親クラスのメソッドを呼び出せる。
  • protected修飾子は自クラスと子クラスからアクセス可能。
