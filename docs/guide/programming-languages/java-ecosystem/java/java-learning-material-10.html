例外処理とJava APIの活用 - 堅牢なプログラムを書く

エラーに強いプログラムを作るための「例外処理」と、実践的なプログラミングに必要なJava標準APIの使い方を学びます。

この章で学ぶこと

  • 例外処理の仕組みを理解し実装できる
  • try-catch-finally構文を正しく使える
  • 主要なJava標準APIを活用できる
  • 実践的なプログラムを作成できる

10.1 例外とは何か

プログラムを実行中に予期しないエラーが発生することがあります。ファイルが見つからない、数値以外の文字列を数値に変換しようとする、配列の範囲外にアクセスするなど、様々な原因でエラーは発生します。Javaでは、このようなプログラム実行時のエラーを「例外（Exception）」として扱います。

例外の階層構造

    A --> C[Exception]

    D --> I[ArithmeticException]

検査例外と非検査例外

検査例外（Checked Exception）: コンパイル時にチェックされ、必ずtry-catchで処理するか、throwsで宣言する必要があります（例: IOException、SQLException）。

非検査例外（Unchecked Exception）: RuntimeExceptionとそのサブクラス。プログラミングミスによるもので、必ずしもtry-catchで処理する必要はありません（例: NullPointerException、ArrayIndexOutOfBoundsException）。

例外が発生しない場合の問題

NoExceptionHandling.java 
public class NoExceptionHandling {
    public static void main(String[] args) {
        // ユーザー入力を想定（実際は数値以外が入力される可能性がある）
        String input = "abc";

        // エラー発生！プログラムが異常終了する
        int number = Integer.parseInt(input);
        System.out.println("変換結果: " + number);

        // この行は実行されない
        System.out.println("プログラム終了");
    }
}
// 実行結果:
// Exception in thread "main" java.lang.NumberFormatException: For input string: "abc"

10.2 例外処理の基本 - try-catch-finally

例外処理を行うには、try-catch構文を使います。例外が発生する可能性のあるコードをtryブロックで囲み、例外が発生した場合の処理をcatchブロックに記述します。

基本的なtry-catch構文

BasicExceptionHandling.java 
public class BasicExceptionHandling {
    public static void main(String[] args) {
        String input = "abc";

        try {
            // 例外が発生する可能性のある処理
            int number = Integer.parseInt(input);
            System.out.println("変換結果: " + number);
        } catch (NumberFormatException e) {
            // 例外が発生した場合の処理
            System.out.println("エラー: 数値に変換できません");
            System.out.println("入力値: " + input);
        }

        // プログラムは正常に続行される
        System.out.println("プログラム終了");
    }
}
// 実行結果:
// エラー: 数値に変換できません
// 入力値: abc
// プログラム終了

複数のcatchブロック

MultipleCatch.java 
public class MultipleCatch {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30};

        try {
            String input = "5";
            int divisor = Integer.parseInt(input);  // NumberFormatExceptionの可能性
            int result = numbers[5] / divisor;      // ArrayIndexOutOfBoundsExceptionの可能性
            System.out.println("結果: " + result);
        } catch (NumberFormatException e) {
            System.out.println("エラー: 数値形式が不正です");
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("エラー: 配列の範囲外です");
        } catch (ArithmeticException e) {
            System.out.println("エラー: 0で割ることはできません");
        } catch (Exception e) {
            // すべての例外をキャッチ（最も一般的な例外は最後に記述）
            System.out.println("エラー: 予期しないエラーが発生しました");
        }
    }
}

finally節 - 必ず実行される処理

finallyブロックは、例外が発生してもしなくても必ず実行されます。ファイルやデータベース接続のクローズなど、リソースの解放処理に使用されます。

FinallyExample.java 
public class FinallyExample {
    public static void main(String[] args) {
        try {
            System.out.println("処理開始");
            int result = 10 / 0;  // ArithmeticException発生
            System.out.println("結果: " + result);
        } catch (ArithmeticException e) {
            System.out.println("エラー: 0で割ることはできません");
        } finally {
            // 例外が発生してもこのブロックは必ず実行される
            System.out.println("リソースをクローズします");
        }

        System.out.println("プログラム終了");
    }
}
// 実行結果:
// 処理開始
// エラー: 0で割ることはできません
// リソースをクローズします
// プログラム終了

つまずきやすいポイント

  • catchの順序: より具体的な例外クラスを先に記述し、一般的な例外クラス（Exceptionなど）は最後に配置します。
  • 例外の握りつぶし: catchブロックで何も処理しないと、エラーを検知できなくなります。最低限ログ出力やエラーメッセージを表示しましょう。

10.3 例外の発生と伝播

メソッドから例外をスローする（投げる）ことで、呼び出し元に例外処理を委譲できます。

throwによる例外のスロー

ThrowExample.java 
public class ThrowExample {
    // 年齢をチェックするメソッド
    public static void checkAge(int age) {
        if (age < 0) {
            // 意図的に例外を発生させる
            throw new IllegalArgumentException("年齢は0以上である必要があります");
        }
        if (age > 120) {
            throw new IllegalArgumentException("年齢が不正です");
        }
        System.out.println("年齢チェック成功: " + age + "歳");
    }

    public static void main(String[] args) {
        try {
            checkAge(25);   // 正常
            checkAge(-5);   // 例外発生
        } catch (IllegalArgumentException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}
// 実行結果:
// 年齢チェック成功: 25歳
// エラー: 年齢は0以上である必要があります

throwsによる例外の宣言

ThrowsExample.java 
import java.io.*;

public class ThrowsExample {
    // このメソッドはIOExceptionをスローする可能性があることを宣言
    public static String readFile(String filename) throws IOException {
        BufferedReader reader = new BufferedReader(new FileReader(filename));
        String line = reader.readLine();
        reader.close();
        return line;
    }

    public static void main(String[] args) {
        try {
            // 呼び出し元で例外処理を行う
            String content = readFile("data.txt");
            System.out.println("ファイル内容: " + content);
        } catch (IOException e) {
            System.out.println("ファイル読み込みエラー: " + e.getMessage());
        }
    }
}

try-with-resources構文

Java 7以降では、AutoCloseableインターフェースを実装したリソース（ファイル、データベース接続など）を自動的にクローズするtry-with-resources構文が使えます。

TryWithResources.java 
import java.io.*;

public class TryWithResources {
    public static void main(String[] args) {
        // try-with-resources: リソースが自動的にクローズされる
        try (BufferedReader reader = new BufferedReader(new FileReader("data.txt"))) {
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
            }
            // reader.close()を明示的に呼ぶ必要なし
        } catch (IOException e) {
            System.out.println("ファイル処理エラー: " + e.getMessage());
        }
    }
}

10.4 主要なJava API

Javaには豊富な標準API（Application Programming Interface）が用意されています。これらを活用することで、効率的にプログラムを作成できます。

Stringクラスの主要メソッド

StringMethods.java 
public class StringMethods {
    public static void main(String[] args) {
        String text = "  Hello World  ";

        // 長さを取得
        System.out.println("長さ: " + text.length());  // 15

        // 前後の空白を削除
        System.out.println("trim: " + text.trim());  // "Hello World"

        // 大文字・小文字変換
        System.out.println("upper: " + text.toUpperCase());  // "  HELLO WORLD  "
        System.out.println("lower: " + text.toLowerCase());  // "  hello world  "

        // 部分文字列の取得
        System.out.println("substring: " + text.substring(2, 7));  // "Hello"

        // 文字列の置換
        System.out.println("replace: " + text.replace("World", "Java"));  // "  Hello Java  "

        // 文字列の検索
        System.out.println("indexOf: " + text.indexOf("World"));  // 8
        System.out.println("contains: " + text.contains("Hello"));  // true

        // 文字列の分割
        String csv = "apple,banana,orange";
        String[] fruits = csv.split(",");
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}

Mathクラス - 数学計算

MathExample.java 
public class MathExample {
    public static void main(String[] args) {
        // 絶対値
        System.out.println("abs: " + Math.abs(-5));  // 5

        // 最大値・最小値
        System.out.println("max: " + Math.max(10, 20));  // 20
        System.out.println("min: " + Math.min(10, 20));  // 10

        // べき乗と平方根
        System.out.println("pow: " + Math.pow(2, 3));  // 8.0 (2の3乗)
        System.out.println("sqrt: " + Math.sqrt(16));  // 4.0 (16の平方根)

        // 四捨五入・切り上げ・切り捨て
        System.out.println("round: " + Math.round(3.7));  // 4
        System.out.println("ceil: " + Math.ceil(3.2));   // 4.0 (切り上げ)
        System.out.println("floor: " + Math.floor(3.8)); // 3.0 (切り捨て)

        // 乱数生成（0.0以上1.0未満）
        System.out.println("random: " + Math.random());

        // 1から10のランダムな整数
        int random = (int)(Math.random() * 10) + 1;
        System.out.println("1-10のランダム値: " + random);
    }
}

ArrayListクラス - 可変長配列

ArrayListExample.java 
import java.util.ArrayList;

public class ArrayListExample {
    public static void main(String[] args) {
        // ArrayListの作成（ジェネリクスで型を指定）
        ArrayList<String> names = new ArrayList<>();

        // 要素の追加
        names.add("太郎");
        names.add("花子");
        names.add("次郎");

        // 要素数の取得
        System.out.println("要素数: " + names.size());  // 3

        // 要素の取得
        System.out.println("0番目: " + names.get(0));  // 太郎

        // 要素の変更
        names.set(1, "美咲");

        // 要素の削除
        names.remove(2);

        // すべての要素を表示
        for (String name : names) {
            System.out.println(name);
        }

        // 要素の存在確認
        System.out.println("太郎がいる: " + names.contains("太郎"));  // true
    }
}

Scannerクラス - ユーザー入力

ScannerExample.java 
import java.util.Scanner;

public class ScannerExample {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        try {
            System.out.print("名前を入力してください: ");
            String name = scanner.nextLine();

            System.out.print("年齢を入力してください: ");
            int age = scanner.nextInt();

            System.out.println("こんにちは、" + name + "さん（" + age + "歳）");

            System.out.println("入力エラー: " + e.getMessage());

        }
    }
}

日付時刻API - LocalDate, LocalTime

DateTimeExample.java 
import java.time.*;
import java.time.format.DateTimeFormatter;

public class DateTimeExample {
    public static void main(String[] args) {
        // 現在の日付
        LocalDate today = LocalDate.now();
        System.out.println("今日の日付: " + today);

        // 日付の作成
        LocalDate birthday = LocalDate.of(2000, 1, 1);
        System.out.println("誕生日: " + birthday);

        // 日付の計算
        LocalDate nextWeek = today.plusDays(7);
        System.out.println("1週間後: " + nextWeek);

        // 現在の時刻
        LocalTime now = LocalTime.now();
        System.out.println("現在時刻: " + now);

        // 日付のフォーマット
        DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy年MM月dd日");
        String formatted = today.format(formatter);
        System.out.println("フォーマット後: " + formatted);
    }
}

実習 10-1: ユーザー入力検証プログラム

ユーザーから数値を入力してもらい、適切な例外処理を行う電卓プログラムを作成します。

仕様

  1. 1 2つの数値をユーザーから入力してもらう
  2. 2 演算（加算、減算、乗算、除算）を選択してもらう
  3. 3 計算結果を表示する
  4. 4 不正な入力やゼロ除算などのエラーを適切に処理する

実装例

Calculator.java 
import java.util.Scanner;

public class Calculator {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        try {
            System.out.print("1つ目の数値: ");
            double num1 = scanner.nextDouble();

            System.out.print("2つ目の数値: ");
            double num2 = scanner.nextDouble();

            System.out.print("演算子 (+, -, *, /): ");
            String operator = scanner.next();

            double result = calculate(num1, num2, operator);
            System.out.println("結果: " + result);

        } catch (Exception e) {
            System.out.println("エラー: " + e.getMessage());
        } finally {
            scanner.close();
        }
    }

    public static double calculate(double a, double b, String op) {
        switch (op) {
            case "+": return a + b;
            case "-": return a - b;
            case "*": return a * b;
            case "/":
                if (b == 0) {
                    throw new ArithmeticException("ゼロで割ることはできません");
                }
                return a / b;
            default:
                throw new IllegalArgumentException("不正な演算子: " + op);
        }
    }
}

理解度確認クイズ

  1. Q1. 検査例外と非検査例外の違いは何ですか？

    回答を表示

    検査例外（Checked Exception）はコンパイル時にチェックされ、必ずtry-catchで処理するか、throwsで宣言する必要があります（例: IOException）。非検査例外（Unchecked Exception）はRuntimeExceptionとそのサブクラスで、プログラミングミスによるもので、必ずしも処理する必要はありません（例: NullPointerException）。

  2. Q2. finally節はどのような場合に使用しますか？

    回答を表示

    finally節は、例外が発生してもしなくても必ず実行される処理を記述します。主にファイルやデータベース接続のクローズ、リソースの解放などに使用されます。

  3. Q3. try-with-resources構文の利点は何ですか？

    回答を表示

    try-with-resources構文を使用すると、AutoCloseableインターフェースを実装したリソース（ファイル、データベース接続など）が自動的にクローズされます。finally節でclose()メソッドを明示的に呼ぶ必要がなく、コードが簡潔になり、リソースリークを防げます。

  4. Q4. 配列とArrayListの主な違いは何ですか？

    回答を表示

    配列は固定長で、作成時にサイズを指定する必要があります。一方、ArrayListは可変長で、要素の追加・削除が自由にでき、サイズが自動的に調整されます。また、ArrayListは豊富なメソッド（add, remove, containsなど）を提供しています。
