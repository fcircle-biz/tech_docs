第5章

プリペアドステートメント（PreparedStatement）の活用

SQLインジェクション対策と処理速度向上のためのPreparedStatementの使い方を詳しく学びます。プレースホルダ（?）の使い方、パラメータのバインド方法、Statementとの違いを理解します。

この章で学ぶこと

  • PreparedStatementとStatementの違いを理解する
  • SQLインジェクション攻撃とその対策方法を学ぶ
  • プレースホルダ（?）を使ったパラメータバインド方法を習得する
  • PreparedStatementのパフォーマンス上のメリットを理解する

5.1 PreparedStatementとは？

PreparedStatementは、SQL文を事前にコンパイルして実行するためのJDBCインターフェースです。第3章で学んだStatementの上位版で、より安全で効率的にSQL文を実行できます。

PreparedStatementを使う主な理由は以下の2つです：

1. セキュリティ対策

SQLインジェクション攻撃を防ぐことができます。ユーザー入力をSQL文に安全に埋め込めます。

2. パフォーマンス向上

SQL文が事前にコンパイルされるため、同じSQL文を繰り返し実行する場合に処理が高速になります。

重要ポイント

実務では原則としてPreparedStatementを使用します。Statementは特殊な理由がない限り使いません。セキュリティと性能の両面で優れているためです。

5.2 SQLインジェクション攻撃とは？

SQLインジェクションは、悪意のあるSQL文を注入することで、データベースを不正に操作する攻撃手法です。Statementを使った文字列連結でSQL文を作成すると、この攻撃を受けやすくなります。

危険な例（Statementを使った場合）

以下のコードは、ユーザーが入力した名前でデータベースを検索していますが、悪意のある入力に対して脆弱です。

BadExample.java（セキュリティ上危険） 
// ユーザーが入力した名前
String userName = request.getParameter("name");

// 文字列連結でSQL文を作成（危険！）
String sql = "SELECT * FROM users WHERE name = '" + userName + "'";
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery(sql);

攻撃者が名前欄に山田' OR '1'='1と入力すると、実行されるSQL文は以下のようになります： 

SELECT * FROM users WHERE name = '山田' OR '1'='1'

'1'='1'は常に真なので、WHERE句の条件が無効化され、すべてのユーザー情報が取得されてしまいます。これがSQLインジェクション攻撃です。

5.3 PreparedStatementでSQLインジェクションを防ぐ

PreparedStatementを使うと、SQL文の構造とデータを明確に分離できるため、SQLインジェクション攻撃を防げます。

プレースホルダ（?）を使ってパラメータの位置を指定し、後から値をバインド（設定）します。

GoodExample.java（安全） 
// ユーザーが入力した名前
String userName = request.getParameter("name");

// プレースホルダ（?）を使ったSQL文
String sql = "SELECT * FROM users WHERE name = ?";
PreparedStatement pstmt = conn.prepareStatement(sql);

// パラメータをバインド（1番目の?にuserNameを設定）
pstmt.setString(1, userName);

// SQL実行
ResultSet rs = pstmt.executeQuery();

このコードでは、攻撃者が山田' OR '1'='1と入力しても、その文字列全体が単なる文字列データとして扱われ、SQL文の構造は変化しません。

安全な理由

PreparedStatementは、SQL文の構造が最初に決まり、後から値が入るため、'やORなどの特殊な文字が含まれていても、それらはSQL文として解釈されず、ただの文字データとして扱われます。

5.4 プレースホルダとパラメータバインド

PreparedStatementでは、?（プレースホルダ）を使ってパラメータの位置を指定します。その後、setXxx()メソッドで値をバインド（設定）します。

主なsetXxx()メソッド

メソッド 用途 例
setString(int, String) 文字列型データ（VARCHAR, TEXT） pstmt.setString(1, "山田太郎")
setInt(int, int) 整数型データ（INT） pstmt.setInt(2, 30)
setDouble(int, double) 浮動小数点数（DOUBLE, FLOAT） pstmt.setDouble(3, 1500.50)
setDate(int, Date) 日付型データ（DATE） pstmt.setDate(4, sqlDate)
setBoolean(int, boolean) 真偽値（BOOLEAN） pstmt.setBoolean(5, true)

プレースホルダの番号

プレースホルダの番号は1から始まります（0ではありません）。SQL文の左から順に、1番目、2番目、3番目...となります。

MultipleParameters.java 
// 複数のパラメータを持つSQL文
String sql = "INSERT INTO employees (name, age, salary) VALUES (?, ?, ?)";
PreparedStatement pstmt = conn.prepareStatement(sql);

// 1番目のプレースホルダに名前をセット
pstmt.setString(1, "佐藤花子");

// 2番目のプレースホルダに年齢をセット
pstmt.setInt(2, 28);

// 3番目のプレースホルダに給与をセット
pstmt.setDouble(3, 350000.00);

// SQL実行
int rows = pstmt.executeUpdate();
System.out.println(rows + "件登録しました");

5.5 StatementとPreparedStatementの比較

StatementとPreparedStatementの違いを具体例で比較してみましょう。

Statement（非推奨） 

String name = "山田太郎";
int age = 30;

String sql = "SELECT * FROM users "
    + "WHERE name = '" + name + "' "
    + "AND age = " + age;

Statement stmt =
    conn.createStatement();
ResultSet rs =
    stmt.executeQuery(sql);
  • SQLインジェクションの危険
  • 文字列連結が複雑で読みにくい
  • 毎回SQLがコンパイルされる

PreparedStatement（推奨） 

String name = "山田太郎";
int age = 30;

String sql = "SELECT * FROM users "
    + "WHERE name = ? AND age = ?";

PreparedStatement pstmt =
    conn.prepareStatement(sql);
pstmt.setString(1, name);
pstmt.setInt(2, age);
ResultSet rs =
    pstmt.executeQuery();
  • SQLインジェクション対策済み
  • コードが読みやすい
  • SQL文が再利用され高速

実習 5-1: PreparedStatementでデータ検索

ユーザーが入力した年齢以上の従業員を検索するプログラムを作成します。

手順

  1. 1 Scanner等でユーザーに年齢を入力してもらう
  2. 2 PreparedStatementでSELECT文を作成（WHERE age >= ?）
  3. 3 setInt()で年齢をバインド
  4. 4 executeQuery()で実行し、結果を表示
SearchByAge.java 
Scanner scanner = new Scanner(System.in);
System.out.print("年齢を入力: ");
int inputAge = scanner.nextInt();

String sql = "SELECT * FROM employees WHERE age >= ?";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setInt(1, inputAge);

ResultSet rs = pstmt.executeQuery();
while (rs.next()) {
    System.out.println(rs.getString("name") + " : " + rs.getInt("age") + "歳");
}

5.6 PreparedStatementのパフォーマンス上のメリット

PreparedStatementは、SQL文を事前にコンパイル（プリコンパイル）するため、同じSQL文を繰り返し実行する場合に高速になります。

大量のデータを登録する場合、PreparedStatementを使うと処理時間が大幅に短縮されます。

パフォーマンスのコツ

同じSQL文を繰り返し実行する場合は、PreparedStatementを1回だけprepareし、パラメータだけを変えて繰り返し実行すると効率的です。

BatchInsert.java（効率的な繰り返し実行） 
// 1回だけprepareする
String sql = "INSERT INTO products (name, price) VALUES (?, ?)";
PreparedStatement pstmt = conn.prepareStatement(sql);

// 複数のデータを登録（パラメータだけ変更して繰り返し実行）
pstmt.setString(1, "商品A");
pstmt.setInt(2, 1000);
pstmt.executeUpdate();

pstmt.setString(1, "商品B");
pstmt.setInt(2, 2000);
pstmt.executeUpdate();

pstmt.setString(1, "商品C");
pstmt.setInt(2, 3000);
pstmt.executeUpdate();

理解度確認クイズ

  1. Q1. PreparedStatementを使う主なメリットを2つ挙げてください。

    回答を表示

    1. セキュリティ対策: SQLインジェクション攻撃を防ぐことができる
    2. パフォーマンス向上: SQL文が事前にコンパイルされるため、同じSQL文を繰り返し実行する場合に高速

  2. Q2. SQLインジェクション攻撃とは何ですか？また、なぜPreparedStatementを使うと防げるのでしょうか？

    回答を表示

    SQLインジェクション攻撃は、悪意のあるSQL文を注入することで、データベースを不正に操作する攻撃手法です。PreparedStatementはSQL文の構造とデータを明確に分離し、パラメータをプレースホルダ（?）で指定するため、ユーザー入力がSQL文として解釈されず、ただの文字データとして扱われます。

  3. Q3. プレースホルダ（?）の番号は何番から始まりますか？また、以下のSQL文で年齢をバインドするコードを書いてください。
    SELECT * FROM users WHERE name = ? AND age = ?

    回答を表示

    プレースホルダの番号は1番から始まります（0ではありません）。

    pstmt.setString(1, "山田太郎"); // 1番目の?に名前をバインド
    pstmt.setInt(2, 30); // 2番目の?に年齢をバインド

  4. Q4. 実務ではStatementとPreparedStatementのどちらを使うべきですか？その理由も説明してください。

    回答を表示

    PreparedStatementを使うべきです。理由は、SQLインジェクション対策とパフォーマンス向上の両面で優れているためです。Statementは特殊な理由がない限り使用しません。

この章のまとめ

  • PreparedStatementは、SQL文を事前にコンパイルして実行するためのインターフェース
  • SQLインジェクション攻撃を防ぐため、実務では原則としてPreparedStatementを使用する
  • プレースホルダ（?）でパラメータ位置を指定し、setXxx()メソッドで値をバインド
  • プレースホルダの番号は1から始まる
  • PreparedStatementは同じSQL文を繰り返し実行する場合にパフォーマンスが向上する
