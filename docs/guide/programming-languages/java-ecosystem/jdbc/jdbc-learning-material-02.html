データベース接続の基本

DriverManagerを使用したデータベース接続の基礎を学び、Connection、Statement、ResultSetの3つの主要オブジェクトの役割と使い方を習得します。

この章で学ぶこと

  • JDBC URLの構造と接続文字列の作成方法
  • DriverManagerを使用した基本的な接続の確立
  • Connection、Statement、ResultSetの役割と関係
  • データベースリソースの適切なクローズ処理

2.1 JDBC接続の仕組み

JDBCを使用してデータベースに接続するには、いくつかの重要なコンポーネントが必要です。まず、データベースとの接続を確立するための「橋渡し役」としてJDBCドライバが機能します。このドライバは、Javaプログラムとデータベース固有の通信プロトコルを変換する役割を担います。

JDBCドライバとは

JDBCドライバは、各データベースベンダーが提供する.jar形式のライブラリです。PostgreSQLにはpostgresql-*.jar、MySQLにはmysql-connector-java-*.jarといった具合に、使用するデータベースに応じた専用のドライバが必要になります。

2.2 JDBC URLの構造

データベースに接続するためには、JDBC URL（接続文字列）が必要です。これはデータベースの場所と接続方法を指定する特殊な文字列で、決められた形式に従って記述します。 

jdbc:[サブプロトコル]://[ホスト名]:[ポート番号]/[データベース名]?[パラメータ]

各構成要素の意味は以下の通りです：

要素 説明 例
jdbc JDBC接続を示す固定のプレフィックス jdbc
サブプロトコル データベースの種類を示す識別子 postgresql, mysql, oracle
ホスト名 データベースサーバーのアドレス localhost, 192.168.1.10
ポート番号 データベースサーバーの待ち受けポート 5432 (PostgreSQL), 3306 (MySQL)
データベース名 接続先のデータベース名 mydb, testdb
パラメータ 接続オプション（任意） useSSL=false, serverTimezone=JST

主要データベースのJDBC URL例

各DBのJDBC URL 
// PostgreSQL
String url = "jdbc:postgresql://localhost:5432/mydb";

// MySQL
String url = "jdbc:mysql://localhost:3306/mydb?serverTimezone=JST";

// Oracle
String url = "jdbc:oracle:thin:@localhost:1521:orcl";

// SQL Server
String url = "jdbc:sqlserver://localhost:1433;databaseName=mydb";

2.3 基本的な接続方法

実際にデータベースに接続するコードを見てみましょう。最も基本的な方法はDriverManager.getConnection()メソッドを使用する方法です。

DatabaseConnection.java 
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class DatabaseConnection {
    public static void main(String[] args) {
        // データベース接続情報
        String url = "jdbc:postgresql://localhost:5432/testdb";
        String user = "dbuser";
        String password = "password123";

        // Connection変数を宣言（try外で宣言してfinallyで使えるようにする）
        Connection conn = null;

        try {
            // データベースに接続
            conn = DriverManager.getConnection(url, user, password);
            System.out.println("データベース接続成功！");

            // ここでデータベース操作を行う

        } catch (SQLException e) {
            System.err.println("接続エラー: " + e.getMessage());
            e.printStackTrace();
        } finally {
            // 接続を閉じる（リソース解放）
            if (conn != null) {
                try {
                    conn.close();
                    System.out.println("接続を閉じました。");
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

重要：リソースのクローズ

データベース接続は貴重なリソースです。使用後は必ずclose()メソッドで閉じる必要があります。finallyブロックで閉じることで、例外が発生した場合でも確実にクローズ処理が実行されます。

2.4 JDBCの主要オブジェクト

JDBCでデータベース操作を行う際、3つの主要なインターフェースが中心的な役割を果たします。それぞれの役割と関係を理解することが重要です。 

classDiagram

    class Statement {
        +executeQuery(sql) ResultSet
        +executeUpdate(sql) int
        +close()
    }
    class ResultSet {
        +next() boolean
        +getString(column) String
        +getInt(column) int
        +close()
    }

    Connection --> Statement : 作成
    Statement --> ResultSet : 生成

1. Connection（接続オブジェクト）

データベースとの接続そのものを表すオブジェクトです。このオブジェクトを通じてSQL文を実行するためのStatementを作成したり、トランザクション制御を行ったりします。

主なメソッド

  • createStatement() : SQL文を実行するStatementオブジェクトを作成
  • prepareStatement(sql) : パラメータ化されたPreparedStatementを作成
  • close() : データベース接続を閉じる

2. Statement（SQL実行オブジェクト）

SQL文をデータベースに送信し、実行するためのオブジェクトです。SELECT文の実行にはexecuteQuery()、INSERT/UPDATE/DELETE文の実行にはexecuteUpdate()を使用します。

主なメソッド

  • executeQuery(sql) : SELECT文を実行し、ResultSetを返す
  • executeUpdate(sql) : INSERT/UPDATE/DELETE文を実行し、影響を受けた行数を返す
  • close() : Statementを閉じる

3. ResultSet（結果セットオブジェクト）

SELECT文の実行結果を保持するオブジェクトです。表形式のデータを1行ずつ処理できる「カーソル」の役割を果たします。next()メソッドで次の行に移動し、各種getXxxメソッドでデータを取得します。

主なメソッド

  • next() : 次の行に移動（データがあればtrue、なければfalse）
  • getString(列名/列番号) : 文字列データを取得
  • getInt(列名/列番号) : 整数データを取得
  • close() : ResultSetを閉じる

2.5 完全な接続とクエリ実行の例

ここまで学んだ知識を統合して、実際にデータベースに接続してデータを取得する完全なプログラムを作成してみましょう。

SimpleQuery.java 
import java.sql.*;

public class SimpleQuery {
    public static void main(String[] args) {
        // 接続情報
        String url = "jdbc:postgresql://localhost:5432/testdb";
        String user = "dbuser";
        String password = "password123";

        Connection conn = null;
        Statement stmt = null;
        ResultSet rs = null;

        try {
            // 1. データベースに接続
            conn = DriverManager.getConnection(url, user, password);
            System.out.println("接続成功");

            // 2. Statementオブジェクトを作成
            stmt = conn.createStatement();

            // 3. SQL文を実行
            String sql = "SELECT id, name, age FROM users";
            rs = stmt.executeQuery(sql);

            // 4. 結果を処理
            System.out.println("--- ユーザー一覧 ---");
            while (rs.next()) {
                int id = rs.getInt("id");
                String name = rs.getString("name");
                int age = rs.getInt("age");

                System.out.printf("ID: %d, 名前: %s, 年齢: %d%n",
                                  id, name, age);
            }

        } catch (SQLException e) {
            System.err.println("エラー: " + e.getMessage());
            e.printStackTrace();
        } finally {
            // 5. リソースをクローズ（作成の逆順で閉じる）
            try {
                if (rs != null) rs.close();
                if (stmt != null) stmt.close();
                if (conn != null) conn.close();
                System.out.println("リソースを解放しました");
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }
    }
}

コードのポイント

  • • リソースは作成の逆順で閉じる（ResultSet → Statement → Connection）
  • • rs.next()は最初にカーソルを1行目に移動させる役割も持つ
  • • カラム名または列番号（1から始まる）でデータを取得できる
  • • 例外処理とリソース解放を忘れずに実装する

実習 2-1: データベース接続を確認する

データベースに接続し、接続が成功したかどうかを確認するプログラムを作成します。接続メタデータも表示してデータベース情報を取得してみましょう。

手順

  1. 1 Eclipseで新しいJavaプロジェクトを作成し、JDBCドライバをビルドパスに追加します
  2. 2 使用するデータベース（PostgreSQL、MySQLなど）のJDBC URLを環境に合わせて設定します
  3. 3 以下のコードを参考に、接続確認プログラムを作成します
  4. 4 プログラムを実行し、接続が成功することを確認します

実装コード

import java.sql.*;

public class ConnectionTest {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/testdb";
        String user = "dbuser";
        String password = "password123";

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            System.out.println("接続成功！");

            // データベースメタデータを取得
            DatabaseMetaData meta = conn.getMetaData();
            System.out.println("データベース製品名: " + meta.getDatabaseProductName());
            System.out.println("データベースバージョン: " + meta.getDatabaseProductVersion());
            System.out.println("ドライバ名: " + meta.getDriverName());
            System.out.println("JDBC URL: " + meta.getURL());

        } catch (SQLException e) {
            System.err.println("接続失敗: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

期待される実行結果

接続成功！
データベース製品名: PostgreSQL
データベースバージョン: 14.5
ドライバ名: PostgreSQL JDBC Driver
JDBC URL: jdbc:postgresql://localhost:5432/testdb

補足：この実習ではtry-with-resources構文を使用しています。この構文では自動的にリソースがクローズされるため、明示的なclose()呼び出しやfinallyブロックが不要になります。

理解度確認クイズ

  1. Q1. JDBC URLの構造について、正しいものはどれですか？

    回答を表示

    正解：jdbc:[サブプロトコル]://[ホスト名]:[ポート]/[DB名]

    JDBC URLは必ず「jdbc:」で始まり、その後にデータベースの種類を示すサブプロトコル（postgresql、mysqlなど）、ホスト名、ポート番号、データベース名が続きます。この形式を正確に記述しないと接続できません。

  2. Q2. Connection、Statement、ResultSetの正しいクローズ順序はどれですか？

    回答を表示

    正解：ResultSet → Statement → Connection

    リソースは作成した順序の逆でクローズします。これは依存関係のあるリソースを安全に解放するための原則です。ResultSetはStatementに依存し、StatementはConnectionに依存しているため、この順序でクローズする必要があります。

  3. Q3. executeQuery()メソッドとexecuteUpdate()メソッドの違いは何ですか？

    回答を表示

    executeQuery()：SELECT文など、結果セットを返すクエリに使用します。戻り値はResultSetオブジェクトです。

    executeUpdate()：INSERT、UPDATE、DELETE文など、データを変更するクエリに使用します。戻り値は影響を受けた行数（int型）です。

    用途に応じて適切なメソッドを選択することが重要です。

  4. Q4. なぜデータベース接続は必ずクローズする必要があるのですか？

    回答を表示

    データベース接続は限られた貴重なリソースです。接続を閉じないと以下の問題が発生します：

    1. メモリリーク：使用されていない接続がメモリを占有し続ける
    2. 接続数の枯渇：データベースサーバーが受け入れられる最大接続数に達し、新しい接続ができなくなる
    3. パフォーマンス低下：不要な接続が維持されることでシステム全体のパフォーマンスが低下する

    finallyブロックやtry-with-resources構文を使用して、必ず接続をクローズするようにしましょう。
