開発環境の準備と初めての接続

JDBCドライバのセットアップ方法、H2データベースのインストール、初めてのデータベース接続プログラムの作成を行います。Connection、DriverManager、接続URLの書き方を実践的に学びます。

この章で学ぶこと

  • JDBCドライバ（H2データベース用）のセットアップ方法を理解する
  • H2データベースのインストールと起動方法を学ぶ
  • DriverManagerとConnectionオブジェクトの使い方を理解する
  • 初めてのデータベース接続プログラムを作成・実行する

2.1 開発環境の全体像

JDBCプログラミングを始めるには、Java開発環境とデータベース、そしてJDBCドライバの3つが必要です。この章では、初心者に最適なH2データベースを使って環境構築を行います。

H2データベースとは？

H2は、Javaで作られた軽量・高速なデータベースです。インストールが簡単で、Javaプログラムに組み込んで使えるため、学習に最適です。本番環境でも使われることがあります。

2.2 H2データベースとJDBCドライバのセットアップ

H2データベースは、1つのJARファイルをダウンロードするだけで、データベース本体とJDBCドライバの両方が手に入ります。

実習 2-1: H2データベースのダウンロードとインストール

H2データベースをダウンロードして、Javaプロジェクトで使えるようにします。

手順

  1. 1 H2公式サイト（https://www.h2database.com/）にアクセスし、「Download」ページから最新版の「Platform-Independent Zip」をダウンロードします。
  2. 2 ダウンロードしたZIPファイルを解凍します。h2/bin/h2-x.x.xxx.jarというファイルが含まれています。
  3. 3 プロジェクトフォルダにlibという名前のフォルダを作成し、h2-x.x.xxx.jarをコピーします。
  4. 4 使用するIDEでJARファイルをプロジェクトのビルドパス（クラスパス）に追加します。

プロジェクト構成例 

jdbc-study/
├── lib/
│   └── h2-2.2.224.jar    ← JDBCドライバ
└── src/
    └── FirstConnection.java

IDE別のクラスパス設定

  • Eclipse: プロジェクト右クリック → Build Path → Configure Build Path → Libraries → Add External JARs
  • IntelliJ IDEA: File → Project Structure → Modules → Dependencies → + → JARs or directories
  • VS Code: Java Projects → Referenced Libraries → + ボタンでJARファイルを選択

データベースに接続するには、接続URLが必要です。接続URLは、「どのデータベースに」「どうやって」接続するかを指定する文字列です。

H2データベースの接続URL例

  • jdbc:h2:~/test - ホームディレクトリにtestという名前のDBを作成（組み込みモード）
  • jdbc:h2:mem:testdb - メモリ上にtestdbという名前のDBを作成（メモリモード、プログラム終了で消える）
  • jdbc:h2:tcp://localhost/~/test - サーバーモードでlocalhost上のtestに接続

この章では、組み込みモード（jdbc:h2:~/test）を使います。これは、Javaプログラムと同じプロセス内でデータベースが動作する最もシンプルな方法です。

2.4 初めてのデータベース接続プログラム

それでは、実際にJavaプログラムからH2データベースに接続してみましょう。最初のプログラムでは、接続に成功したら「接続成功！」と表示するだけのシンプルなコードを作成します。

実習 2-2: 初めての接続プログラムを作成

DriverManagerを使ってH2データベースに接続し、接続が成功したことを確認します。

FirstConnection.java 
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class FirstConnection {
    public static void main(String[] args) {
        // 接続URL、ユーザー名、パスワード
        String url = "jdbc:h2:~/test";
        String user = "sa";  // H2のデフォルトユーザー
        String password = ""; // H2のデフォルトパスワードは空文字

        Connection conn = null;
        try {
            // データベースに接続
            conn = DriverManager.getConnection(url, user, password);
            System.out.println("データベース接続成功！");
        } catch (SQLException e) {
            System.out.println("接続エラー: " + e.getMessage());
        }
    }
}

コードのポイント

  • DriverManager.getConnection() - データベースに接続し、Connectionオブジェクトを取得
  • Connection - データベースとの接続を表すオブジェクト（この章ではまだ使わない）
  • SQLException - データベース関連のエラーをキャッチするための例外クラス

動作確認

プログラムを実行してみましょう。成功すると、以下のメッセージが表示されます。

初回実行時の注意

初めて実行すると、ホームディレクトリにtest.mv.dbというファイルが自動的に作成されます。これがH2データベースの実体です。2回目以降は既存のデータベースに接続します。

2.5 接続を閉じる重要性

データベースに接続したら、必ず接続を閉じる必要があります。閉じ忘れると、リソースのリーク（無駄な消費）が発生し、データベースが新しい接続を受け付けられなくなる可能性があります。

FirstConnection.java（接続を閉じる版） 
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class FirstConnection {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/test";
        String user = "sa";
        String password = "";

        Connection conn = null;
        try {
            conn = DriverManager.getConnection(url, user, password);
            System.out.println("データベース接続成功！");
        } catch (SQLException e) {
            System.out.println("接続エラー: " + e.getMessage());
        } finally {
            // 接続を閉じる（必ず実行される）
            if (conn != null) {
                try {
                    conn.close();
                    System.out.println("接続を閉じました");
                } catch (SQLException e) {
                    System.out.println("接続のクローズに失敗: " + e.getMessage());
                }
            }
        }
    }
}

finallyブロックの役割

finallyブロックは、try-catchブロックの処理が成功しても失敗しても必ず実行されるブロックです。接続のクローズ処理は、ここに書くことで確実に実行されます。

トラブルシューティング

ClassNotFoundException: org.h2.Driver

原因: JDBCドライバ（h2-x.x.xxx.jar）がクラスパスに追加されていません。

解決方法: IDEの設定でJARファイルをプロジェクトのビルドパスに追加してください。

SQLException: Database may be already in use

原因: 同じデータベースファイルに別のプロセスが既に接続しています。

解決方法: 他のJavaプログラムやH2コンソールを終了してから再度実行してください。

接続URLの~（チルダ）が認識されない

原因: Windowsでは~（ホームディレクトリ）が使えない場合があります。

解決方法: 絶対パスを使用してください（例: jdbc:h2:C:/Users/yourname/test）

理解度確認クイズ

  1. Q1. JDBCプログラミングに必要な3つの要素を挙げてください。

    回答を表示

    1. Java開発環境（JDK）
    2. データベース（例: H2、MySQL、PostgreSQLなど）
    3. JDBCドライバ（データベースごとに異なる）

  2. Q2. JDBC接続URL「jdbc:h2:~/test」の各部分は何を意味していますか？

    回答を表示

    jdbc: JDBCプロトコルを使用することを示す
    h2: H2データベースに接続することを示す
    ~/test: ホームディレクトリにある「test」という名前のデータベースに接続することを示す

  3. Q3. データベース接続を取得するために使用するクラスとメソッドは何ですか？

    回答を表示

    DriverManagerクラスのgetConnection()メソッドを使用します。
    例: Connection conn = DriverManager.getConnection(url, user, password);

  4. Q4. データベース接続を閉じる処理はどのブロックに書くべきですか？その理由も説明してください。

    回答を表示

    finallyブロックに書くべきです。
    理由: finallyブロックは、処理が成功しても失敗しても必ず実行されるため、接続を確実に閉じることができます。これにより、リソースリーク（接続の閉じ忘れ）を防ぐことができます。

この章のまとめ

  • JDBC開発にはJDK、データベース、JDBCドライバの3つが必要
  • H2データベースは1つのJARファイルで導入でき、初心者の学習に最適
  • 接続URLは「jdbc:データベース種類:データベース位置」の形式で記述する
  • DriverManager.getConnection()でデータベース接続を取得する
  • 接続はfinallyブロックで必ず閉じる（conn.close()）
