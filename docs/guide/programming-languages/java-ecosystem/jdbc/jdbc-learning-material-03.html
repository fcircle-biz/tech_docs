データの検索（SELECT文）をマスターする

Statement、PreparedStatementを使ったSELECT文の実行、ResultSetを使った検索結果の取得と処理方法を学びます。WHERE句を使った条件検索、複数件データの取得方法を実践します。

この章で学ぶこと

  • StatementとPreparedStatementの使い分けとSELECT文の実行方法を理解する
  • ResultSetを使った検索結果の取得と処理の基本を習得する
  • WHERE句を使った条件検索の実装方法を身につける
  • 複数件データのループ処理と安全なリソース管理を実践する

3.1 SELECT文とは？ データベースからデータを取り出す

SELECT文は、データベースからデータを検索・取得するためのSQL文です。第2章で接続を確立した後、実際にデータを取り出すのがこの章のテーマです。

SELECT文は、データベース操作の中で最も頻繁に使われます。商品一覧の表示、顧客情報の検索、注文履歴の確認など、あらゆる場面でSELECT文が活躍します。

SELECT文の基本構文

全件取得: SELECT * FROM テーブル名

特定カラムのみ: SELECT カラム1, カラム2 FROM テーブル名

条件付き検索: SELECT * FROM テーブル名 WHERE 条件

3.2 Statementを使った基本的なSELECT

まず、最もシンプルなStatementを使ったSELECT文の実行方法を学びましょう。Statementは、固定のSQL文を実行するときに使います。

SelectExample.java 
import java.sql.*;

public class SelectExample {
    public static void main(String[] args) {
        // 接続URL（H2データベース）
        String url = "jdbc:h2:~/testdb";
        String user = "sa";
        String password = "";

        try (Connection conn = DriverManager.getConnection(url, user, password);
             Statement stmt = conn.createStatement()) {

            // SQL文を作成
            String sql = "SELECT id, name, age FROM users";

            // SQL文を実行してResultSetを取得
            ResultSet rs = stmt.executeQuery(sql);

            // 結果をループで処理
            while (rs.next()) {
                // 各カラムの値を取得
                int id = rs.getInt("id");
                String name = rs.getString("name");
                int age = rs.getInt("age");

                System.out.println(id + " | " + name + " | " + age);
            }

            rs.close();  // ResultSetを閉じる

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

コードのポイント

  • executeQuery(): SELECT文を実行し、ResultSetを返す
  • rs.next(): 次の行に移動し、行があればtrueを返す
  • rs.getInt("カラム名"): 指定カラムの整数値を取得
  • rs.getString("カラム名"): 指定カラムの文字列を取得

主なResultSetメソッド

  • boolean next(): 次の行に移動（行があればtrue）
  • int getInt(String columnName): 整数値を取得
  • String getString(String columnName): 文字列を取得
  • double getDouble(String columnName): 小数値を取得
  • Date getDate(String columnName): 日付を取得

3.4 WHERE句を使った条件検索

全件取得だけでなく、条件に合うデータだけを検索するWHERE句の使い方を学びましょう。実務では、特定の条件に合うデータのみを取得することがほとんどです。

WhereExample.java 
import java.sql.*;

public class WhereExample {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/testdb";

        try (Connection conn = DriverManager.getConnection(url, "sa", "");
             Statement stmt = conn.createStatement()) {

            // WHERE句で年齢が30以上のユーザーを検索
            String sql = "SELECT id, name, age FROM users WHERE age >= 30";

            ResultSet rs = stmt.executeQuery(sql);

            System.out.println("年齢30歳以上のユーザー:");
            while (rs.next()) {
                int id = rs.getInt("id");
                String name = rs.getString("name");
                int age = rs.getInt("age");

                System.out.println(id + " | " + name + " | " + age + "歳");
            }

            rs.close();

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

注意: Statementでの条件検索のリスク

上記のコードは固定の条件ですが、ユーザー入力を直接SQL文に埋め込むとSQLインジェクションという重大なセキュリティ問題が発生します。次のセクションで安全な方法（PreparedStatement）を学びます。

3.5 PreparedStatementを使った安全な検索

ユーザーからの入力値を条件に使う場合、PreparedStatementを使うのが安全です。プレースホルダ（?）を使って、値を安全に埋め込むことができます。

PreparedSelectExample.java 
import java.sql.*;

public class PreparedSelectExample {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/testdb";

        // ユーザーからの入力を想定（本来はキーボード入力など）
        int minAge = 25;

        try (Connection conn = DriverManager.getConnection(url, "sa", "")) {

            // プレースホルダ（?）を使ったSQL文
            String sql = "SELECT id, name, age FROM users WHERE age >= ?";
            PreparedStatement pstmt = conn.prepareStatement(sql);

            // プレースホルダに値をセット
            pstmt.setInt(1, minAge);  // 1番目の?にminAgeをセット

            ResultSet rs = pstmt.executeQuery();

            System.out.println("年齢" + minAge + "歳以上のユーザー:");
            while (rs.next()) {
                int id = rs.getInt("id");
                String name = rs.getString("name");
                int age = rs.getInt("age");

                System.out.println(id + " | " + name + " | " + age + "歳");
            }

            rs.close();
            pstmt.close();

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

重要ポイント

  • ?をプレースホルダとして使用
  • setInt(番号, 値)で値をセット（番号は1から始まる）
  • 文字列はsetString()、日付はsetDate()を使う
  • PreparedStatementはSQLインジェクション対策になる

3.6 複数条件のWHERE句とLIKE検索

実務では、複数の条件を組み合わせた検索や、あいまい検索（LIKE）を使うことがよくあります。

MultiConditionExample.java 
import java.sql.*;

public class MultiConditionExample {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/testdb";

        try (Connection conn = DriverManager.getConnection(url, "sa", "")) {

            // 複数条件（AND）とLIKE検索
            String sql = "SELECT id, name, age FROM users " +
                         "WHERE age >= ? AND name LIKE ?";
            PreparedStatement pstmt = conn.prepareStatement(sql);

            pstmt.setInt(1, 20);          // 年齢20歳以上
            pstmt.setString(2, "%田%");    // 名前に「田」を含む

            ResultSet rs = pstmt.executeQuery();

            while (rs.next()) {
                System.out.println(rs.getInt("id") + " | " +
                                   rs.getString("name") + " | " +
                                   rs.getInt("age") + "歳");
            }

            rs.close();
            pstmt.close();

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

LIKE検索のワイルドカード

  • %: 0文字以上の任意の文字列
  • _: 任意の1文字
  • 例: %田% → 「田中」「山田」などにマッチ
  • 例: 田_ → 「田中」「田口」などにマッチ

実習 3-1: 条件検索プログラムを作成しよう

PreparedStatementを使って、ユーザーが指定した年齢以上のユーザーを検索するプログラムを作成します。

手順

  1. 1 データベースに接続し、PreparedStatementを作成する
  2. 2 SQL文にWHERE age >= ?を含め、プレースホルダを使う
  3. 3 setInt()で年齢をセットし、executeQuery()を実行
  4. 4 ResultSetをwhile文でループし、結果を画面に表示する
  5. 5 try-with-resourcesを使って、リソースを確実に解放する

理解度確認クイズ

  1. Q1. StatementとPreparedStatementの違いは何ですか？PreparedStatementを使う主な理由を説明してください。

    回答を表示

    Statementは固定のSQL文を実行するのに対し、PreparedStatementはプレースホルダ（?）を使って動的に値をセットできます。PreparedStatementを使う主な理由は、SQLインジェクション対策とパフォーマンス向上（SQL文の再利用による高速化）です。

  2. Q2. ResultSetのnext()メソッドは何をするメソッドですか？また、戻り値の意味を説明してください。

    回答を表示

    next()メソッドは、ResultSetのカーソルを次の行に移動させます。戻り値はboolean型で、次の行が存在すればtrue、存在しなければfalseを返します。そのため、while (rs.next())というループで全行を処理できます。

  3. Q3. PreparedStatementでsetString(1, "%田%")を実行した場合、SQL文のどこに値がセットされますか？

    回答を表示

    SQL文の1番目のプレースホルダ（?）に値がセットされます。プレースホルダの番号は1から始まる点に注意してください（0ではありません）。

  4. Q4. ResultSetから整数値を取得するメソッドと、文字列を取得するメソッドの名前をそれぞれ答えてください。

    回答を表示

    整数値: getInt("カラム名") または getInt(カラム番号)
    文字列: getString("カラム名") または getString(カラム番号)

この章のまとめ

  • SELECT文はデータベースからデータを検索・取得するためのSQL文
  • executeQuery()メソッドでSELECT文を実行し、ResultSetを取得する
  • ResultSetは検索結果を保持し、next()で行を移動、getStringやgetIntで値を取得
  • PreparedStatementを使うと、プレースホルダ（?）で安全に条件値をセットできる
  • WHERE句で条件検索、LIKE演算子であいまい検索が可能
