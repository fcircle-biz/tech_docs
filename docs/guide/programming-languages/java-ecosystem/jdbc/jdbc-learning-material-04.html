データの登録・更新・削除（INSERT/UPDATE/DELETE文）

データベースへの新規データ登録（INSERT）、既存データの更新（UPDATE）、データの削除（DELETE）をJDBCで実行する方法を学びます。executeUpdate()メソッドの使い方と戻り値の意味を理解します。

この章で学ぶこと

  • executeUpdate()メソッドを使ったデータ操作（INSERT/UPDATE/DELETE）の基本
  • INSERT文によるデータ登録と、戻り値（影響行数）の確認方法
  • UPDATE文によるデータ更新とWHERE句を使った条件指定
  • DELETE文によるデータ削除の安全な実行方法と注意点

4.1 executeUpdate()メソッドの役割

前章ではSELECT文を使ってデータを検索し、ResultSetで結果を取得する方法を学びました。今回は、データベースの内容を変更する操作（INSERT、UPDATE、DELETE）について学習します。

これらのSQL文は、検索とは異なり「データを取得する」のではなく「データを変更する」ため、executeUpdate()メソッドを使用します。

重要ポイント

  • executeQuery(): SELECT文で使用。ResultSetオブジェクトを返す
  • executeUpdate(): INSERT/UPDATE/DELETE文で使用。影響を受けた行数（int型）を返す
  • 戻り値の行数で、操作が成功したかどうかを判断できる

4.2 INSERT文：新しいデータを登録する

INSERT文は、データベースのテーブルに新しいデータ（行）を追加するためのSQL文です。例えば、新しい顧客情報や商品情報をデータベースに登録するときに使います。

INSERT文の基本構文 
-- 基本構文
INSERT INTO テーブル名 (列1, 列2, 列3, ...)
VALUES (値1, 値2, 値3, ...);

-- 例: 顧客テーブルに新規顧客を登録
INSERT INTO customers (name, email, phone)
VALUES ('山田太郎', 'yamada@example.com', '090-1234-5678');

JDBCでINSERT文を実行する場合、StatementまたはPreparedStatementを使用します。ここでは、まずStatementを使った基本的な例を見てみましょう。

InsertExample.java 
import java.sql.*;

public class InsertExample {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/test";
        String user = "sa";
        String password = "";

        try (Connection conn = DriverManager.getConnection(url, user, password);
             Statement stmt = conn.createStatement()) {

            // INSERT文を定義
            String sql = "INSERT INTO customers (name, email, phone) " +
                         "VALUES ('山田太郎', 'yamada@example.com', '090-1234-5678')";

            // executeUpdate()でINSERT実行
            int rowsAffected = stmt.executeUpdate(sql);

            System.out.println(rowsAffected + "件のデータを登録しました");

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

初心者向けヒント

executeUpdate()の戻り値は、影響を受けた行数です。INSERT文の場合、成功すれば通常1が返ります（1件追加されたという意味）。0が返ってきた場合は、何らかの理由で登録されなかったことを意味します。

注意

上記のコードでは、SQL文に直接値を埋め込んでいます。これは学習用の簡単な例ですが、実務ではSQLインジェクションという攻撃に対して脆弱です。次章で学ぶPreparedStatementを使うことで、安全にデータを扱えます。

4.3 UPDATE文：既存のデータを更新する

UPDATE文は、データベース内の既存のデータを変更するためのSQL文です。例えば、顧客の電話番号が変わった場合や、商品の価格を変更したい場合に使います。

UPDATE文の基本構文 
-- 基本構文
UPDATE テーブル名
SET 列1 = 値1, 列2 = 値2, ...
WHERE 条件;

-- 例: ID=1の顧客の電話番号を更新
UPDATE customers
SET phone = '090-9999-8888'
WHERE id = 1;

非常に重要な注意点

WHERE句を忘れると、テーブル内のすべてのデータが更新されてしまいます。例えば、上記のWHERE句を書き忘れると、全顧客の電話番号が同じ値に書き換わってしまいます。UPDATE文を書く際は、必ずWHERE句を確認してください。

UpdateExample.java 
import java.sql.*;

public class UpdateExample {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/test";
        String user = "sa";
        String password = "";

        try (Connection conn = DriverManager.getConnection(url, user, password);
             Statement stmt = conn.createStatement()) {

            // UPDATE文を定義（WHERE句で対象を限定）
            String sql = "UPDATE customers SET phone = '090-9999-8888' WHERE id = 1";

            // executeUpdate()でUPDATE実行
            int rowsAffected = stmt.executeUpdate(sql);

            System.out.println(rowsAffected + "件のデータを更新しました");

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

戻り値の活用

UPDATE文の戻り値も、影響を受けた行数です。0が返った場合は、WHERE句の条件に一致するデータが存在しなかった（更新対象がなかった）ことを意味します。これを利用して、「更新対象が見つからなかった」というエラー処理を実装できます。

4.4 DELETE文：データを削除する

DELETE文は、データベースから不要になったデータを削除するためのSQL文です。例えば、退会した顧客情報を削除する場合などに使います。

DELETE文の基本構文 
-- 基本構文
DELETE FROM テーブル名
WHERE 条件;

-- 例: ID=5の顧客を削除
DELETE FROM customers
WHERE id = 5;

DELETE文の危険性

WHERE句を忘れると、テーブル内のすべてのデータが削除されてしまいます。削除されたデータは元に戻せないため、DELETE文は特に慎重に扱う必要があります。

実務では、本当に削除するのではなく「削除フラグ」を立てる方法（論理削除）がよく使われます。例えば、deletedという列を用意し、削除時にはUPDATE文でdeleted = trueに変更します。

DeleteExample.java 
import java.sql.*;

public class DeleteExample {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/test";
        String user = "sa";
        String password = "";

        try (Connection conn = DriverManager.getConnection(url, user, password);
             Statement stmt = conn.createStatement()) {

            // DELETE文を定義（WHERE句で対象を限定）
            String sql = "DELETE FROM customers WHERE id = 5";

            // executeUpdate()でDELETE実行
            int rowsAffected = stmt.executeUpdate(sql);

            System.out.println(rowsAffected + "件のデータを削除しました");

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

削除前の確認処理

実務では、削除する前にSELECT文で該当データが存在するか確認し、ユーザーに「本当に削除しますか？」と確認メッセージを表示するのが一般的です。また、削除後は戻り値が1以上であることを確認して、正しく削除されたかチェックします。

4.5 executeUpdate()の戻り値の活用

executeUpdate()メソッドの戻り値（影響を受けた行数）は、処理が成功したかどうかを判断する重要な情報です。実務では、この戻り値を使ってエラーハンドリングを行います。

戻り値を使ったエラーハンドリング例 
// UPDATE文で特定のIDのデータを更新
String sql = "UPDATE customers SET phone = '090-1111-2222' WHERE id = 100";
int rowsAffected = stmt.executeUpdate(sql);

if (rowsAffected > 0) {
    System.out.println("更新に成功しました");
} else {
    System.out.println("更新対象のデータが見つかりませんでした");
}

executeUpdate()の戻り値の意味

戻り値 意味
1以上 指定した行数のデータが変更された（成功）
0 条件に一致するデータがなかった（更新・削除対象なし）

ベストプラクティス

INSERT/UPDATE/DELETE文を実行したら、必ず戻り値を確認する習慣をつけましょう。これにより、意図しない動作（例：更新対象が見つからなかった、削除対象がなかった）を早期に検出できます。

実習 4-1: CRUD操作を実装してみよう

データベースの基本操作（CRUD: Create, Read, Update, Delete）を実際に動かして理解を深めましょう。

準備

  1. 1 H2データベースに接続し、以下のSQL文でテーブルを作成してください： CREATE TABLE products (id INT PRIMARY KEY, name VARCHAR(100), price INT)
  2. 2 以下のデータ操作をJDBCで実装してみましょう：
    - INSERT: 商品データを3件登録
    - SELECT: 全商品を検索して表示
    - UPDATE: ID=2の商品の価格を変更
    - DELETE: ID=3の商品を削除
    - SELECT: 削除後の全商品を再度表示して確認
  3. 3 各操作の後に、executeUpdate()の戻り値を確認して画面に表示してください

トラブルシューティング

executeUpdate()が0を返す

原因: WHERE句の条件に一致するデータが存在しないため、UPDATE/DELETE対象が見つかりませんでした。

解決方法: まずSELECT文で該当データが存在するか確認してください。また、WHERE句の条件（ID、名前など）が正しいか再確認しましょう。

全件が更新/削除されてしまった

原因: UPDATE文またはDELETE文でWHERE句を書き忘れたため、テーブル内の全データが対象になりました。

解決方法: UPDATE/DELETE文を書く際は、必ず最初にWHERE句を記述してから実行する癖をつけましょう。可能であれば、実行前にSELECT文で対象データを確認することをお勧めします。

SQLSyntaxErrorException: SQL構文エラー

原因: SQL文の構文に誤りがあります（例：VALUESのスペルミス、カンマの位置間違いなど）。

解決方法: エラーメッセージに表示されるSQL文をよく読み、構文が正しいか確認してください。可能であれば、H2のコンソール画面でSQL文を単体で実行してみると、エラーの原因が分かりやすいです。

理解度確認クイズ

  1. Q1. executeUpdate()メソッドの戻り値は何を表していますか？

    回答を表示

    影響を受けた行数（int型）を返します。INSERT/UPDATE/DELETE文によって、実際に追加、更新、または削除された行の数です。処理が成功したかどうかを判断するために使用します。

  2. Q2. UPDATE文やDELETE文でWHERE句を省略すると、どうなりますか？

    回答を表示

    テーブル内のすべてのデータが更新または削除されてしまいます。これは意図しない動作になることが多く、データを失う危険性があるため、UPDATE/DELETE文では必ずWHERE句を記述することが重要です。

  3. Q3. executeUpdate()とexecuteQuery()の違いは何ですか？

    回答を表示

    executeQuery(): SELECT文用。ResultSetオブジェクトを返す（検索結果を取得）
    executeUpdate(): INSERT/UPDATE/DELETE文用。影響を受けた行数（int型）を返す（データを変更）

  4. Q4. 実務でDELETE文の代わりに「論理削除」を使う理由は何ですか？

    回答を表示

    DELETE文で物理的に削除すると、データを完全に失ってしまい、復元できません。論理削除（削除フラグを立てる方法）を使えば、データを残しつつ「削除された状態」として扱えるため、誤削除の際に復元できます。また、削除履歴を保持できるため、監査や分析にも役立ちます。

この章のまとめ

  • executeUpdate()は、INSERT/UPDATE/DELETE文を実行し、影響を受けた行数を返す
  • INSERT文は新規データを登録、戻り値が1なら成功（1件追加）
  • UPDATE文は既存データを変更、WHERE句を忘れると全件更新されるので要注意
  • DELETE文はデータを削除、WHERE句を忘れると全件削除されるため特に注意が必要
  • 戻り値が0の場合、条件に一致するデータがなかったことを意味する（エラーハンドリングに活用）
  • 実務では論理削除（削除フラグ）を使うことで、データの復元可能性を保つことが多い
