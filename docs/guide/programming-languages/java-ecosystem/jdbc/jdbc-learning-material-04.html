第4章

データの登録・更新・削除

INSERT、UPDATE、DELETE文を使用したデータ操作を学び、executeUpdate()メソッドの使い方と操作結果の確認方法を習得します。

この章で学ぶこと

  • INSERT文を使用した新規データの登録方法
  • UPDATE文を使用した既存データの更新方法
  • DELETE文を使用したデータの削除方法
  • executeUpdate()メソッドの戻り値と操作結果の確認方法

4.1 executeUpdate()メソッドの理解

前章ではSELECT文を使用してデータを取得する方法を学びました。この章では、データベースのデータを変更する操作（INSERT、UPDATE、DELETE）について学びます。これらの操作にはexecuteUpdate()メソッドを使用します。

executeQuery()とexecuteUpdate()の違い

  • executeQuery(): SELECT文を実行し、ResultSetオブジェクトを返す
  • executeUpdate(): INSERT、UPDATE、DELETE文を実行し、影響を受けた行数を返す

executeUpdate()メソッドはint型の戻り値を持ち、この値によって何行のデータが変更されたかを知ることができます。これにより、操作が成功したかどうかを確認できます。

4.2 INSERT文によるデータ登録

INSERT文は、データベースのテーブルに新しいレコード（行）を追加するために使用します。JDBCでは、Statementオブジェクトを使ってINSERT文を実行できます。

INSERT文の基本構文

INSERT文の基本構文は次の通りです：

SQL構文 
INSERT INTO テーブル名 (カラム1, カラム2, カラム3, ...)
VALUES (値1, 値2, 値3, ...);

実装例：ユーザー情報の登録

usersテーブルに新しいユーザー情報を登録するプログラムを作成してみましょう。

InsertExample.java 
import java.sql.*;

public class InsertExample {
    public static void main(String[] args) {
        // データベース接続情報
        String url = "jdbc:postgresql://localhost:5432/mydb";
        String user = "postgres";
        String password = "password";

        // 登録するデータ
        String name = "田中太郎";
        String email = "tanaka@example.com";
        int age = 28;

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            // INSERT文を作成
            String sql = "INSERT INTO users (name, email, age) VALUES ('"
                        + name + "', '" + email + "', " + age + ")";

            // Statementオブジェクトを作成
            Statement stmt = conn.createStatement();

            // INSERT文を実行
            int rowsInserted = stmt.executeUpdate(sql);

            // 実行結果を確認
            if (rowsInserted > 0) {
                System.out.println(rowsInserted + "件のデータを登録しました。");
            }

            stmt.close();
        } catch (SQLException e) {
            System.out.println("データベースエラー: " + e.getMessage());
        }
    }
}

重要な注意点

このコード例はStatementの基本的な使い方を示すためのものです。実際の開発では、SQLインジェクション攻撃を防ぐため、次章で学ぶPreparedStatementを使用することが強く推奨されます。

実行結果の確認

executeUpdate()メソッドの戻り値を確認することで、データが正常に登録されたかを判断できます：

実行結果：

1件のデータを登録しました。

4.3 UPDATE文によるデータ更新

UPDATE文は、既に存在するデータの値を変更するために使用します。WHERE句を使用して、更新対象のレコードを指定することが重要です。

UPDATE文の基本構文

SQL構文 
UPDATE テーブル名
SET カラム1 = 値1, カラム2 = 値2, ...
WHERE 条件;

WHERE句を忘れずに！

WHERE句を指定しない場合、テーブル内の全てのレコードが更新されてしまいます。必ず更新対象を特定する条件を指定してください。

実装例：メールアドレスの更新

UpdateExample.java 
import java.sql.*;

public class UpdateExample {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/mydb";
        String user = "postgres";
        String password = "password";

        // 更新するデータ
        int userId = 1;
        String newEmail = "tanaka.new@example.com";

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            // UPDATE文を作成
            String sql = "UPDATE users SET email = '" + newEmail
                        + "' WHERE id = " + userId;

            // Statementオブジェクトを作成
            Statement stmt = conn.createStatement();

            // UPDATE文を実行
            int rowsUpdated = stmt.executeUpdate(sql);

            // 実行結果を確認
            if (rowsUpdated > 0) {
                System.out.println(rowsUpdated + "件のデータを更新しました。");
            } else {
                System.out.println("指定されたIDのユーザーは存在しません。");
            }

            stmt.close();
        } catch (SQLException e) {
            System.out.println("データベースエラー: " + e.getMessage());
        }
    }
}

複数カラムの同時更新

カンマで区切ることで、複数のカラムを一度に更新できます：

複数カラム更新例 
// 名前とメールアドレスを同時に更新
String sql = "UPDATE users SET name = '田中二郎', email = 'tanaka.jiro@example.com' " +
             "WHERE id = " + userId;

4.4 DELETE文によるデータ削除

DELETE文は、テーブルからレコードを削除するために使用します。UPDATEと同様に、WHERE句を使用して削除対象を特定することが非常に重要です。

DELETE文の基本構文

SQL構文 
DELETE FROM テーブル名
WHERE 条件;

データの削除は取り消せません

DELETE文で削除されたデータは復元できません。WHERE句を忘れると全てのデータが削除されます。削除前に必ず確認しましょう。

実装例：ユーザーの削除

DeleteExample.java 
import java.sql.*;

public class DeleteExample {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/mydb";
        String user = "postgres";
        String password = "password";

        // 削除対象のユーザーID
        int userId = 5;

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            // DELETE文を作成
            String sql = "DELETE FROM users WHERE id = " + userId;

            // Statementオブジェクトを作成
            Statement stmt = conn.createStatement();

            // DELETE文を実行
            int rowsDeleted = stmt.executeUpdate(sql);

            // 実行結果を確認
            if (rowsDeleted > 0) {
                System.out.println(rowsDeleted + "件のデータを削除しました。");
            } else {
                System.out.println("指定されたIDのユーザーは存在しません。");
            }

            stmt.close();
        } catch (SQLException e) {
            System.out.println("データベースエラー: " + e.getMessage());
        }
    }
}

条件を満たす複数レコードの削除

WHERE句の条件に複数のレコードが該当する場合、それら全てが削除されます：

複数レコード削除例 
// 30歳未満のユーザーを全て削除
String sql = "DELETE FROM users WHERE age < 30";
int rowsDeleted = stmt.executeUpdate(sql);
System.out.println(rowsDeleted + "件のデータを削除しました。");

4.5 戻り値を活用した処理制御

executeUpdate()メソッドの戻り値を活用することで、データベース操作の結果に応じた適切な処理を行うことができます。

戻り値の意味

  • 正の整数: その数だけのレコードが影響を受けた（成功）
  • 0: 条件に一致するレコードが存在しなかった（影響なし）
  • 例外発生: SQL文の構文エラーや接続エラーなどの問題が発生

実装例：戻り値に基づく条件分岐

UpdateWithValidation.java 
import java.sql.*;
import java.util.Scanner;

public class UpdateWithValidation {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/mydb";
        String user = "postgres";
        String password = "password";

        Scanner scanner = new Scanner(System.in);
        System.out.print("更新するユーザーIDを入力: ");
        int userId = scanner.nextInt();
        scanner.nextLine(); // 改行を消費

        System.out.print("新しいメールアドレスを入力: ");
        String newEmail = scanner.nextLine();

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            String sql = "UPDATE users SET email = '" + newEmail
                        + "' WHERE id = " + userId;

            Statement stmt = conn.createStatement();
            int rowsUpdated = stmt.executeUpdate(sql);

            // 戻り値に基づいて処理を分岐
            if (rowsUpdated > 0) {
                System.out.println("✓ データの更新に成功しました。");
                System.out.println("  更新件数: " + rowsUpdated + "件");
            } else {
                System.out.println("✗ 指定されたIDのユーザーが見つかりません。");
                System.out.println("  IDを確認してください。");
            }

            stmt.close();
        } catch (SQLException e) {
            System.out.println("✗ データベースエラーが発生しました。");
            System.out.println("  詳細: " + e.getMessage());
        }

        scanner.close();
    }
}

バッチ処理での活用

複数のデータを連続して登録する場合、戻り値を累計することで全体の処理結果を把握できます：

BatchInsert.java 
// 複数ユーザーの一括登録
String[][] userData = {
    {"山田花子", "yamada@example.com", "25"},
    {"佐藤次郎", "sato@example.com", "32"},
    {"鈴木三郎", "suzuki@example.com", "28"}
};

int totalInserted = 0;

for (String[] data : userData) {
    String sql = "INSERT INTO users (name, email, age) VALUES ('"
                + data[0] + "', '" + data[1] + "', " + data[2] + ")";

    int result = stmt.executeUpdate(sql);
    totalInserted += result;
}

System.out.println("合計 " + totalInserted + "件のデータを登録しました。");

実習 4-1: ユーザー管理プログラムの作成

INSERT、UPDATE、DELETEの3つの操作を組み合わせて、簡単なユーザー管理プログラムを作成してください。

要件

  1. 1 メニューを表示し、ユーザーに操作を選択させる（1: 登録、2: 更新、3: 削除、4: 終了）
  2. 2 各操作で必要な情報をユーザーから入力で受け取る
  3. 3 executeUpdate()の戻り値を使って、操作結果を分かりやすく表示する
  4. 4 終了が選択されるまで、繰り返しメニューを表示する

ヒント

  • • while文を使って、メニューをループ表示します
  • • switch文で操作を分岐させると読みやすいコードになります
  • • Scannerクラスを使用してユーザー入力を受け取ります
  • • try-catch文で例外処理を適切に行いましょう

実行例

1. ユーザー登録
2. ユーザー情報更新
3. ユーザー削除
4. 終了
操作を選択してください: 1

名前を入力: 山田太郎
メールアドレスを入力: yamada@example.com

理解度確認クイズ

  1. Q1. executeUpdate()メソッドの戻り値は何を表していますか？

    回答を表示

    影響を受けた行数（int型）
    INSERT、UPDATE、DELETE文によって影響を受けたレコードの件数を返します。例えば、1件のデータを登録した場合は1が返され、3件のデータを削除した場合は3が返されます。条件に該当するデータが存在しない場合は0が返されます。

  2. Q2. UPDATE文やDELETE文でWHERE句を指定しないとどうなりますか？

    回答を表示

    テーブル内の全てのレコードが対象になります
    WHERE句がない場合、条件による絞り込みが行われないため、テーブル内の全てのレコードが更新または削除されてしまいます。これは通常、意図しない動作であり、データの損失につながる可能性があります。必ずWHERE句で対象を限定しましょう。

  3. Q3. executeQuery()とexecuteUpdate()の違いを説明してください。

    回答を表示

    用途と戻り値が異なります
    • executeQuery(): SELECT文を実行し、ResultSetオブジェクトを返す。データの取得に使用。
    • executeUpdate(): INSERT、UPDATE、DELETE文を実行し、影響を受けた行数（int型）を返す。データの変更に使用。
    データを読み取る場合はexecuteQuery()、データを変更する場合はexecuteUpdate()を使用します。

  4. Q4. 次のコードの問題点を指摘してください：
    String sql = "DELETE FROM users";

    回答を表示

    WHERE句がありません
    このSQL文はWHERE句による条件指定がないため、usersテーブルの全てのレコードを削除してしまいます。通常、特定のレコードのみを削除したい場合がほとんどなので、WHERE句を追加して削除対象を限定する必要があります。例：
    DELETE FROM users WHERE id = 5;

  5. Q5. INSERT文で複数のレコードを一度に登録する方法を考えてください。

    回答を表示

    主に2つの方法があります
    1. ループを使って複数回executeUpdate()を実行：配列やListに登録したいデータを格納し、for文やfor-each文でループしながら1件ずつINSERTする方法。理解しやすく、初心者向け。
    2. VALUES句に複数の値セットを指定：SQL文で複数のVALUES句をカンマで区切って指定する方法。例：INSERT INTO users (name, email) VALUES ('田中', 'tanaka@example.com'), ('山田', 'yamada@example.com');
    また、次の章で学ぶPreparedStatementにはバッチ処理機能があり、より効率的に複数データを登録できます。

この章のまとめ

  • executeUpdate()メソッドは、INSERT、UPDATE、DELETE文を実行し、影響を受けた行数を返す
  • INSERT文で新しいレコードを登録できる。VALUES句で登録する値を指定する
  • UPDATE文で既存のレコードを更新できる。SET句で更新内容、WHERE句で対象を指定する
  • DELETE文でレコードを削除できる。WHERE句で削除対象を限定することが重要
  • UPDATE文とDELETE文ではWHERE句を忘れると全レコードが対象になるため注意が必要
  • 戻り値を確認することで、操作が成功したかどうかを判断できる
