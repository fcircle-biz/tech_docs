第1章

JDBCの概要と環境構築

JDBCとは何か、なぜJDBCが必要なのかを理解し、開発環境のセットアップを行います。JDBCアーキテクチャの基本構造とドライバの役割についても解説します。

この章で学ぶこと

  • JDBCの役割とデータベース接続の仕組みを理解する
  • JDBCアーキテクチャとドライバの種類を把握する
  • 開発環境のセットアップとJDBCドライバの導入方法を習得する

1.1 JDBCとは

JDBC（Java Database Connectivity）は、Javaプログラムからリレーショナルデータベースに接続し、SQL文を実行してデータを操作するための標準APIです。JDBCを使用することで、異なるデータベース製品（PostgreSQL、MySQL、Oracleなど）に対して、ほぼ同じコードでアクセスできます。

なぜJDBCが必要なのか？

Javaアプリケーションでデータを永続化（保存）するには、データベースが不可欠です。JDBCは、Javaとデータベースの橋渡しをする役割を果たし、標準化されたインターフェースを提供することで、開発者が効率的にデータベース操作を実装できるようにします。

1.2 JDBCアーキテクチャとドライバ

JDBCアーキテクチャは、主に以下の3つの層で構成されています。

3層構造

  1. 1
    JDBC API層： Javaアプリケーションが使用する標準インターフェース（java.sql パッケージ）
  2. 2
    JDBCドライバマネージャ： 適切なドライバを選択し、接続を管理
  3. 3
    JDBCドライバ： データベース固有の実装を提供（各データベースベンダーが提供）

JDBCドライバの種類

現在の実務では、Type 4ドライバ（Pure Javaドライバ）が主流です。これは100% Javaで実装されたドライバで、プラットフォームに依存せず、直接データベースと通信します。

ドライバ種類 説明 使用状況
Type 1 JDBC-ODBCブリッジ 非推奨（廃止）
Type 2 ネイティブAPIドライバ ほぼ使用されない
Type 3 ネットワークプロトコル 限定的
Type 4 Pure Javaドライバ 主流（推奨）

1.3 開発環境のセットアップ

JDBC開発を始めるには、以下の環境を準備する必要があります。

1. JDKのインストール

Java Development Kit（JDK）11以上をインストールします。Oracle JDKまたはOpenJDKのいずれかを選択できます。

バージョン確認（ターミナル） 
# Javaバージョンの確認
java -version

# コンパイラバージョンの確認
javac -version

2. IDEのインストール

以下のいずれかのIDEを推奨します：

  • Eclipse IDE for Enterprise Java Developers - 無料でJava EE開発に特化
  • IntelliJ IDEA - Community Edition（無料）またはUltimate Edition
  • Visual Studio Code - 軽量で拡張機能が豊富

3. データベースのインストール

本教材ではPostgreSQLを推奨しますが、MySQLでも学習可能です。

PostgreSQLのインストール手順

  1. 1. 公式サイト（https://www.postgresql.org/download/）からインストーラをダウンロード
  2. 2. インストーラを実行し、デフォルト設定で進める
  3. 3. ポート番号はデフォルト（5432）を使用
  4. 4. スーパーユーザー（postgres）のパスワードを設定
  5. 5. インストール完了後、pgAdmin4で接続確認

4. JDBCドライバの取得

使用するデータベースに対応したJDBCドライバをダウンロードします。

主要データベースのJDBCドライバ

  • PostgreSQL： https://jdbc.postgresql.org/
  • MySQL： https://dev.mysql.com/downloads/connector/j/

1.4 JDBCドライバの導入方法

JDBCドライバをプロジェクトに組み込む方法は、主に2通りあります。

方法1：JARファイルを直接追加（従来の方法）

  1. 1 JDBCドライバのJARファイルをダウンロード（例：postgresql-42.6.0.jar）
  2. 2 プロジェクト内にlibフォルダを作成し、JARファイルを配置
  3. 3 IDE上でビルドパスにJARファイルを追加

方法2：Mavenを使用（推奨）

Mavenを使用すると、依存関係を自動的に管理できます。pom.xmlに以下の設定を追加します。

pom.xml（PostgreSQLの場合） 
<dependencies>
    <!-- PostgreSQL JDBCドライバ -->
    <dependency>
        <groupId>org.postgresql</groupId>
        <artifactId>postgresql</artifactId>
        <version>42.6.0</version>
    </dependency>
</dependencies>

Mavenを使用するメリット

  • 依存関係の自動解決とバージョン管理
  • プロジェクト共有時の環境構築が簡単
  • セキュリティアップデートの適用が容易

実習 1-1: 環境確認プログラムの作成

JDBCドライバが正しく導入されているかを確認する簡単なプログラムを作成しましょう。

手順

  1. 1 新しいJavaプロジェクトを作成し、JDBCドライバをビルドパスに追加
  2. 2 以下のコードを入力し、実行してみましょう

実行例

JdbcDriverTest.java 
public class JdbcDriverTest {
    public static void main(String[] args) {
        try {
            // JDBCドライバのクラスをロード
            Class.forName("org.postgresql.Driver");
            System.out.println("JDBCドライバが正常にロードされました！");
        } catch (ClassNotFoundException e) {
            System.err.println("JDBCドライバが見つかりません: " + e.getMessage());
        }
    }
}

期待される結果

JDBCドライバが正常にロードされました！

注意：MySQLを使用する場合は、org.postgresql.Driverをcom.mysql.cj.jdbc.Driverに変更してください。

理解度確認クイズ

  1. Q1. JDBCの正式名称は何ですか？また、その役割を簡潔に説明してください。

    回答を表示

    回答：Java Database Connectivity。Javaプログラムからデータベースに接続し、SQL文を実行してデータを操作するための標準APIです。異なるデータベース製品に対してほぼ同じコードでアクセスできる統一されたインターフェースを提供します。

  2. Q2. 現在主流のJDBCドライバの種類は何ですか？その特徴も答えてください。

    回答を表示

    回答：Type 4ドライバ（Pure Javaドライバ）。100% Javaで実装されており、プラットフォームに依存せず、直接データベースと通信できるのが特徴です。

  3. Q3. JDBCドライバをプロジェクトに導入する方法を2つ挙げ、それぞれの特徴を説明してください。

    回答を表示

    回答：
    1. JARファイルを直接追加：ドライバのJARファイルをダウンロードしてプロジェクトのビルドパスに追加する従来の方法。シンプルだが、バージョン管理が手動になる。
    2. Mavenを使用：pom.xmlに依存関係を記述することで自動的にドライバを取得する方法。依存関係の自動解決、バージョン管理が容易で、プロジェクト共有時の環境構築も簡単になる（推奨）。

この章のまとめ

  • JDBCは、Javaプログラムからデータベースにアクセスするための標準APIである
  • Type 4ドライバ（Pure Javaドライバ）が現在の主流で、プラットフォーム非依存が特徴
  • 開発環境にはJDK、IDE、データベース、JDBCドライバが必要
  • JDBCドライバの導入にはMavenを使用する方法が推奨される
