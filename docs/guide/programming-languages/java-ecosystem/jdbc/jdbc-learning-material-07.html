実践的なDAO（Data Access Object）パターン

データベース操作を整理するDAOパターンの設計と実装を学びます。データベース処理とビジネスロジックの分離、再利用可能なDAOクラスの作成方法を実践します。

この章で学ぶこと

  • DAOパターンとは何か、その目的とメリットを理解する
  • DAOクラスの設計方針と実装方法を習得する
  • DAO・DTO・Entityクラスの役割分担を理解する
  • 実践的なDAOパターンの実装例を通じて保守性の高いコードを学ぶ

7.1 DAOパターンとは？

DAO（Data Access Object）パターンは、データベースアクセス処理を専門のクラスに分離する設計パターンです。ビジネスロジック（業務処理）とデータベース処理を切り離すことで、コードの見通しが良くなり、保守性が向上します。

これまでの章では、メインメソッドの中にSQL実行コードを直接書いていました。しかし、実務のアプリケーションでは、複数の画面や機能から同じようなデータベース操作を行うことが多く、コードの重複が発生します。DAOパターンを使えば、データベース操作を一箇所にまとめ、再利用しやすくなります。

1. 責任の分離

ビジネスロジックとデータベース処理を分けることで、それぞれの役割が明確になり、コードが読みやすくなります。

2. 再利用性

同じデータベース操作を複数の場所で使い回せるため、コードの重複がなくなります。

3. 保守性の向上

データベース操作の変更が必要な場合、DAOクラスのみを修正すればよく、影響範囲が限定されます。

4. テストしやすさ

データベース処理を切り出すことで、単体テストやモック（模擬オブジェクト）を使ったテストがしやすくなります。

7.2 DAOパターンの全体像

DAOパターンでは、通常3つの要素が登場します：DAOクラス、DTOクラス（またはEntityクラス）、そしてビジネスロジッククラスです。それぞれの役割を理解しましょう。

用語解説

  • DAO（Data Access Object）: データベース操作を担当するクラス（例：UserDAO、OrderDAO）
  • DTO（Data Transfer Object） / Entity: データベースのテーブル1行分のデータを格納するクラス（例：User、Order）
  • ビジネスロジック: 業務処理を担当するクラスやメソッド（例：ユーザー登録処理、注文処理）

7.3 DTOクラス（Entityクラス）の作成

まずは、データベースのテーブル1行分のデータを表すクラスを作成します。ここでは、usersテーブルのデータを扱うUserクラスを作ってみましょう。

usersテーブルの構造 

CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL,
    age INT
);
User.java 
// DTO（Data Transfer Object）クラス
// データベースの1行分のデータを保持する
public class User {
    // フィールド（テーブルのカラムに対応）
    private int id;
    private String name;
    private String email;
    private int age;

    // コンストラクタ
    public User() {}

    public User(int id, String name, String email, int age) {
        this.id = id;
        this.name = name;
        this.email = email;
        this.age = age;
    }

    // ゲッター・セッター（省略形）
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

ポイント

DTOクラスは「データの入れ物」です。テーブルの各カラムに対応するフィールドと、それらを取得・設定するためのゲッター・セッターを用意します。データベース操作のロジックは含みません。

7.4 DAOクラスの設計と実装

次に、データベース操作を担当するUserDAOクラスを作成します。一般的に、DAOクラスには以下のようなメソッドを用意します：

  • findAll(): 全件検索
  • findById(int id): ID指定で1件検索
  • insert(User user): 新規登録
  • update(User user): 更新
  • delete(int id): 削除
UserDAO.java 
import java.sql.*;
import java.util.ArrayList;
import java.util.List;

// データベース操作を担当するDAOクラス
public class UserDAO {
    // データベース接続情報
    private static final String URL = "jdbc:h2:~/testdb";
    private static final String USER = "sa";
    private static final String PASS = "";

    // 全件検索
    public List<User> findAll() {
        List<User> users = new ArrayList<>();
        String sql = "SELECT * FROM users";

        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             Statement stmt = conn.createStatement();
             ResultSet rs = stmt.executeQuery(sql)) {

            while (rs.next()) {
                // ResultSetからUserオブジェクトを生成
                User user = new User(
                    rs.getInt("id"),
                    rs.getString("name"),
                    rs.getString("email"),
                    rs.getInt("age")
                );
                users.add(user);
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
        return users;
    }

    // ID指定で1件検索
    public User findById(int id) {
        String sql = "SELECT * FROM users WHERE id = ?";

        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement pstmt = conn.prepareStatement(sql)) {

            pstmt.setInt(1, id);
            ResultSet rs = pstmt.executeQuery();

            if (rs.next()) {
                return new User(
                    rs.getInt("id"),
                    rs.getString("name"),
                    rs.getString("email"),
                    rs.getInt("age")
                );
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
        return null; // 見つからなかった場合
    }
}

7.5 INSERT・UPDATE・DELETEメソッドの実装

検索メソッドに続いて、登録・更新・削除のメソッドも実装しましょう。これらをUserDAOクラスに追加します。

UserDAO.java（続き） 
    // 新規登録
    public boolean insert(User user) {
        String sql = "INSERT INTO users (name, email, age) VALUES (?, ?, ?)";

        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement pstmt = conn.prepareStatement(sql)) {

            pstmt.setString(1, user.getName());
            pstmt.setString(2, user.getEmail());
            pstmt.setInt(3, user.getAge());

            int rows = pstmt.executeUpdate();
            return rows > 0; // 成功したらtrue
        } catch (SQLException e) {
            e.printStackTrace();
            return false;
        }
    }

    // 更新
    public boolean update(User user) {
        String sql = "UPDATE users SET name = ?, email = ?, age = ? WHERE id = ?";

        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement pstmt = conn.prepareStatement(sql)) {

            pstmt.setString(1, user.getName());
            pstmt.setString(2, user.getEmail());
            pstmt.setInt(3, user.getAge());
            pstmt.setInt(4, user.getId());

            int rows = pstmt.executeUpdate();
            return rows > 0;
        } catch (SQLException e) {
            e.printStackTrace();
            return false;
        }
    }

    // 削除
    public boolean delete(int id) {
        String sql = "DELETE FROM users WHERE id = ?";

        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement pstmt = conn.prepareStatement(sql)) {

            pstmt.setInt(1, id);

            int rows = pstmt.executeUpdate();
            return rows > 0;
        } catch (SQLException e) {
            e.printStackTrace();
            return false;
        }
    }

重要ポイント

戻り値はboolean型で、成功したらtrue、失敗したらfalseを返すようにしています。呼び出し側で結果を判定しやすくなります。

7.6 DAOクラスの利用方法

DAOクラスができたので、実際に使ってみましょう。メインプログラムからDAOクラスを呼び出してデータベース操作を行います。

Main.java 
import java.util.List;

public class Main {
    public static void main(String[] args) {
        // DAOインスタンスを生成
        UserDAO dao = new UserDAO();

        // 1. 全件検索
        System.out.println("=== 全件検索 ===");
        List<User> users = dao.findAll();
        for (User u : users) {
            System.out.println(u.getId() + ": " + u.getName());
        }

        // 2. 新規登録
        System.out.println("\n=== 新規登録 ===");
        User newUser = new User();
        newUser.setName("田中太郎");
        newUser.setEmail("tanaka@example.com");
        newUser.setAge(28);

        boolean success = dao.insert(newUser);
        if (success) {
            System.out.println("登録成功！");
        }

        // 3. ID指定検索
        System.out.println("\n=== ID=1のユーザー検索 ===");
        User user = dao.findById(1);
        if (user != null) {
            System.out.println("名前: " + user.getName());
            System.out.println("メール: " + user.getEmail());
        }

        // 4. 更新
        System.out.println("\n=== 更新 ===");
        user.setAge(30); // 年齢を変更
        dao.update(user);
        System.out.println("更新完了");

        // 5. 削除
        System.out.println("\n=== 削除 ===");
        dao.delete(2);
        System.out.println("削除完了");
    }
}

ポイント

メインプログラムからは、SQL文やConnectionの詳細を意識する必要がありません。dao.findAll()、dao.insert(user)のように、わかりやすいメソッド名で操作できます。これがDAOパターンの大きなメリットです。

7.7 接続情報の外部化（応用）

現在のDAOクラスでは、データベース接続情報（URL、ユーザー名、パスワード）がハードコーディングされています。実務では、これらをプロパティファイルや環境変数から読み込むことが一般的です。

db.properties（設定ファイル例） 

db.url=jdbc:h2:~/testdb
db.user=sa
db.password=
DBConfig.java（設定読み込みクラス例） 
import java.io.*;
import java.util.Properties;

// データベース接続情報を管理するクラス
public class DBConfig {
    private static Properties props = new Properties();

    static {
        try (InputStream in = new FileInputStream("db.properties")) {
            props.load(in); // プロパティファイルを読み込み
        } catch (IOException e) {
            e.printStackTrace();
        }
    }

    public static String getUrl() {
        return props.getProperty("db.url");
    }

    public static String getUser() {
        return props.getProperty("db.user");
    }

    public static String getPassword() {
        return props.getProperty("db.password");
    }
}

DAOクラスでは、DBConfig.getUrl()のように呼び出すことで、接続情報を取得できます。設定変更時にソースコードを修正する必要がなくなり、保守性が向上します。

7.8 実践：DAOパターンの演習

実習 7-1: 商品管理DAOの作成

商品管理システムを想定し、ProductDTO と ProductDAO を作成してみましょう。

手順

  1. 1 productsテーブルを作成（id, name, price, stock）
  2. 2 Productクラス（DTO）を作成
  3. 3 ProductDAOクラスを作成し、findAll()、findById()、insert()メソッドを実装
  4. 4 メインプログラムから商品一覧表示と新規登録を実行

理解度確認クイズ

  1. Q1. DAOパターンを使う主なメリットを3つ挙げてください。

    回答を表示

    1. 責任の分離: ビジネスロジックとデータベース処理を分離できる
    2. 再利用性: 同じデータベース操作を複数の箇所で使い回せる
    3. 保守性の向上: 変更の影響範囲が限定され、修正がしやすい

  2. Q2. DTO（Data Transfer Object）クラスとDAOクラスの違いは何ですか？

    回答を表示

    DTO（Entityクラス）: データベースの1行分のデータを保持する「データの入れ物」。フィールドとゲッター・セッターのみを持つ。
    DAOクラス: データベース操作（検索、登録、更新、削除）を担当するクラス。SQL実行やConnectionの管理を行う。

  3. Q3. UserDAO.findById(int id)メソッドが、該当するユーザーが見つからなかった場合に返す値は何ですか？

    回答を表示

    nullを返します。呼び出し側では、戻り値がnullかどうかをチェックして、データが見つかったかどうかを判定します。

  4. Q4. データベース接続情報をハードコーディングせず、外部化する方法として適切なものは何ですか？

    回答を表示

    プロパティファイル（db.propertiesなど）に接続情報を記載し、JavaのPropertiesクラスで読み込む方法が一般的です。また、環境変数やYAML/JSON形式の設定ファイルを使う方法もあります。

この章のまとめ

  • DAOパターンは、データベース操作を専門のクラスに分離する設計パターン
  • DTOクラスはデータの入れ物、DAOクラスはデータベース操作を担当
  • DAOクラスにはfindAll()、findById()、insert()、update()、delete()などのメソッドを用意
  • データベース接続情報は外部ファイルから読み込むことで保守性が向上
  • DAOパターンにより、コードの再利用性・保守性・テストのしやすさが大幅に向上
