第7章

トランザクション管理

複数のデータベース操作を一つの単位として管理するトランザクションの概念を学び、データの整合性を保つ方法を習得します。

この章で学ぶこと

  • トランザクションの概念とACID特性の理解
  • 自動コミットモードの無効化とcommit/rollbackの使い方
  • 複数の操作を含むトランザクションの実装方法
  • エラー発生時のロールバック処理の実践

7.1 トランザクションとは

トランザクションとは、データベースに対する一連の操作をひとまとまりの単位として扱う仕組みです。 複数のSQL文を「全て成功するか、全て失敗するか」という原則で管理することで、データの整合性を保ちます。

トランザクションの例

銀行の振込処理を例に考えてみましょう。「A口座から10,000円を引き落とす」と「B口座に10,000円を振り込む」という2つの操作があります。 もし1つ目の操作だけが成功して2つ目が失敗すると、お金が消えてしまいます。トランザクションを使えば、両方成功するか、両方失敗するかのどちらかになります。

flowchart TD Start([トランザクション開始]) --> Op1[操作1: A口座から引き落とし] Op1 --> Op2[操作2: B口座に入金] Op2 --> Check{全ての操作が
成功したか?} Check -->|はい| Commit[コミット: 変更を確定] Check -->|いいえ| Rollback[ロールバック: 変更を取り消し] Commit --> End([処理完了]) Rollback --> End

7.2 ACID特性

トランザクションはACIDという4つの特性を満たす必要があります。これらはデータベースの信頼性を保証する重要な概念です。

A

Atomicity (原子性)

トランザクション内の全ての操作が成功するか、全て失敗するかのどちらかになります。中途半端な状態は許されません。

C

Consistency (一貫性)

トランザクション実行前後でデータベースの整合性が保たれます。業務ルールや制約に違反しない状態を維持します。

I

Isolation (独立性)

複数のトランザクションが同時に実行されても、お互いに影響を与えません。各トランザクションは独立して動作します。

D

Durability (永続性)

コミットされたトランザクションの結果は、システム障害が発生しても失われません。データは永続的に保存されます。

7.3 自動コミットモードの制御

JDBCの接続はデフォルトで自動コミットモードが有効になっています。これは各SQL文が実行されるたびに自動的にコミットされる状態です。 複数の操作を1つのトランザクションとして扱うには、自動コミットモードを無効にする必要があります。

TransactionExample.java 
import java.sql.*;

public class TransactionExample {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/testdb";
        String user = "postgres";
        String password = "password";

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            // 自動コミットモードを無効化
            conn.setAutoCommit(false);
            System.out.println("トランザクション開始");

            // SQL文を実行（後述の例で使用）

            // 全ての操作が成功したらコミット
            conn.commit();
            System.out.println("トランザクションをコミットしました");

        } catch (SQLException e) {
            System.err.println("エラーが発生しました: " + e.getMessage());
        }
    }
}

重要

setAutoCommit(false)を呼び出した後は、必ずcommit()またはrollback()を明示的に呼び出す必要があります。 これを忘れると、変更がデータベースに反映されないか、ロックが残り続ける可能性があります。

7.4 コミットとロールバック

トランザクションを完了させるには、コミット(commit)またはロールバック(rollback)のいずれかを実行します。

コミット (commit)

トランザクション内の全ての変更をデータベースに確定します。コミット後は変更を取り消すことができません。

conn.commit();

ロールバック (rollback)

トランザクション内の全ての変更を取り消し、開始前の状態に戻します。エラーが発生した場合に使用します。

conn.rollback();
BankTransferExample.java 
import java.sql.*;

public class BankTransferExample {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/bankdb";
        String user = "postgres";
        String password = "password";

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            // 自動コミット無効化
            conn.setAutoCommit(false);

            try {
                // 振込元口座から引き落とし
                String withdrawSql = "UPDATE accounts SET balance = balance - ? WHERE account_id = ?";
                try (PreparedStatement pstmt = conn.prepareStatement(withdrawSql)) {
                    pstmt.setInt(1, 10000);  // 金額
                    pstmt.setInt(2, 1);      // 口座ID
                    pstmt.executeUpdate();
                    System.out.println("口座1から10,000円を引き落としました");
                }

                // 振込先口座に入金
                String depositSql = "UPDATE accounts SET balance = balance + ? WHERE account_id = ?";
                try (PreparedStatement pstmt = conn.prepareStatement(depositSql)) {
                    pstmt.setInt(1, 10000);  // 金額
                    pstmt.setInt(2, 2);      // 口座ID
                    pstmt.executeUpdate();
                    System.out.println("口座2に10,000円を入金しました");
                }

                // 全て成功したらコミット
                conn.commit();
                System.out.println("振込処理が完了しました");

            } catch (SQLException e) {
                // エラーが発生したらロールバック
                conn.rollback();
                System.err.println("エラーが発生したため、処理を取り消しました: " + e.getMessage());
            }

        } catch (SQLException e) {
            System.err.println("データベース接続エラー: " + e.getMessage());
        }
    }
}

7.5 セーブポイントの活用

セーブポイント(Savepoint)を使用すると、トランザクション内の特定の位置まで部分的にロールバックすることができます。 複雑な処理で一部だけをやり直したい場合に便利です。

SavepointExample.java 
import java.sql.*;

public class SavepointExample {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/testdb";

        try (Connection conn = DriverManager.getConnection(url, "postgres", "password")) {
            conn.setAutoCommit(false);

            try (PreparedStatement pstmt = conn.prepareStatement(
                    "INSERT INTO products (name, price) VALUES (?, ?)")) {

                // 商品1を登録
                pstmt.setString(1, "商品A");
                pstmt.setInt(2, 1000);
                pstmt.executeUpdate();
                System.out.println("商品Aを登録しました");

                // セーブポイント作成
                Savepoint savepoint1 = conn.setSavepoint("savepoint1");

                // 商品2を登録
                pstmt.setString(1, "商品B");
                pstmt.setInt(2, 2000);
                pstmt.executeUpdate();
                System.out.println("商品Bを登録しました");

                // 商品Bの登録に問題があった場合
                // セーブポイントまでロールバック
                conn.rollback(savepoint1);
                System.out.println("商品Bの登録を取り消しました");

                // 商品Aはそのまま残る
                conn.commit();
                System.out.println("トランザクション完了");

            } catch (SQLException e) {
                conn.rollback();
                System.err.println("エラー: " + e.getMessage());
            }

        } catch (SQLException e) {
            System.err.println("接続エラー: " + e.getMessage());
        }
    }
}

セーブポイントの使い方

  • conn.setSavepoint("名前"): セーブポイントを作成
  • conn.rollback(savepoint): 指定したセーブポイントまでロールバック
  • conn.releaseSavepoint(savepoint): セーブポイントを解放

実習 7-1: 在庫管理システムのトランザクション実装

商品の注文処理で、在庫を減らす処理と注文履歴を記録する処理を1つのトランザクションとして実装しましょう。

前提条件

以下のテーブルがすでに作成されているものとします。

-- 商品テーブル
CREATE TABLE products (
    product_id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    stock INT  -- 在庫数
);

-- 注文履歴テーブル
CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    product_id INT,
    quantity INT,
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

手順

  1. 1 自動コミットを無効にしてトランザクションを開始します
  2. 2 商品テーブルの在庫を減らすUPDATE文を実行します
  3. 3 注文履歴テーブルに記録を追加するINSERT文を実行します
  4. 4 両方の操作が成功したらcommit、失敗したらrollbackします

実装例

実装例を表示 
import java.sql.*;

public class InventoryTransaction {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/shopdb";
        int productId = 1;   // 商品ID
        int quantity = 5;    // 注文数

        try (Connection conn = DriverManager.getConnection(url, "postgres", "password")) {
            conn.setAutoCommit(false);

            try {
                // 在庫を減らす
                String updateSql = "UPDATE products SET stock = stock - ? WHERE product_id = ?";
                try (PreparedStatement pstmt = conn.prepareStatement(updateSql)) {
                    pstmt.setInt(1, quantity);
                    pstmt.setInt(2, productId);
                    int updated = pstmt.executeUpdate();

                    if (updated == 0) {
                        throw new SQLException("商品が見つかりません");
                    }
                    System.out.println("在庫を" + quantity + "個減らしました");
                }

                // 注文履歴を記録
                String insertSql = "INSERT INTO orders (product_id, quantity) VALUES (?, ?)";
                try (PreparedStatement pstmt = conn.prepareStatement(insertSql)) {
                    pstmt.setInt(1, productId);
                    pstmt.setInt(2, quantity);
                    pstmt.executeUpdate();
                    System.out.println("注文履歴を記録しました");
                }

                conn.commit();
                System.out.println("注文処理が完了しました");

            } catch (SQLException e) {
                conn.rollback();
                System.err.println("エラーが発生したため処理を取り消しました: " + e.getMessage());
            }

        } catch (SQLException e) {
            System.err.println("データベース接続エラー: " + e.getMessage());
        }
    }
}

チャレンジ課題

在庫数が注文数より少ない場合は、エラーを発生させてロールバックするように改良してみましょう。

ヒント: WHERE句にAND stock >= ?の条件を追加し、affected rowsが0かどうかチェックします。

7.6 トランザクション管理のベストプラクティス

トランザクションは短く保つ

トランザクションが長いと、データベースのロックが長時間保持され、他の処理がブロックされる可能性があります。 必要最小限の操作だけをトランザクション内に含めましょう。

必ずtry-catchを使う

例外が発生した場合に確実にロールバックするため、try-catchブロック内でトランザクション処理を行い、 catch句でrollbackを呼び出すようにします。

deadlock(デッドロック)に注意

複数のトランザクションが互いにロックを待ち合う状態をデッドロックと呼びます。 テーブルやレコードへのアクセス順序を統一することで、デッドロックの発生を減らせます。

finally句でリセット

処理終了後は自動コミットモードを元に戻すことを推奨します。これにより、接続がプールで再利用される際の問題を防げます。

finally { conn.setAutoCommit(true); }

理解度確認クイズ

  1. Q1. トランザクションのACID特性のうち、「全ての操作が成功するか、全て失敗するか」を保証する特性は何ですか?

    回答を表示

    Atomicity (原子性)
    原子性は、トランザクション内の操作が「全か無か(all or nothing)」の原則に従うことを保証します。 1つの操作でも失敗すると、全ての操作がロールバックされ、開始前の状態に戻ります。

  2. Q2. 自動コミットモードを無効にするために使用するメソッドは何ですか?

    回答を表示

    conn.setAutoCommit(false);
    このメソッドを呼び出すと、各SQL文が自動的にコミットされなくなります。 その後、明示的にcommit()またはrollback()を呼び出す必要があります。

  3. Q3. トランザクション内で一部の操作だけをロールバックしたい場合に使用する機能は何ですか?

    回答を表示

    セーブポイント (Savepoint)
    Savepoint sp = conn.setSavepoint("名前");でセーブポイントを作成し、 conn.rollback(sp);で指定したセーブポイントまで部分的にロールバックできます。 これにより、トランザクション全体をやり直すことなく、一部の操作だけを取り消せます。

  4. Q4. 次のうち、トランザクション管理で推奨される実装方法はどれですか?

    • A. トランザクションはできるだけ長くして、まとめて処理する
    • B. エラーが発生してもコミットを試みる
    • C. try-catchブロック内で処理し、catch句でrollbackを呼び出す
    • D. 自動コミットモードのまま使う
    回答を表示

    C. try-catchブロック内で処理し、catch句でrollbackを呼び出す
    これが推奨される実装方法です。エラーが発生した場合に確実にロールバックでき、データの整合性を保てます。 A は誤りです(トランザクションは短く保つべき)。B, D も不適切です。
