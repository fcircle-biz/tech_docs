第2章

はじめてのJUnitテスト - 環境構築と最初の一歩

JUnitを使うための環境を整え、人生で初めてのテストコードを書いて実行します。Mavenプロジェクトの作成方法、JUnitの導入、テストの実行方法を画面を見ながら一つ一つ丁寧に解説します。

この章で学ぶこと

  • VSCodeでMavenプロジェクトを作成する方法を習得する
  • JUnit 5をプロジェクトに導入する方法を理解する
  • はじめてのテストコードを書いて実行する
  • テスト結果の見方を理解する

2.1 環境構築の準備

JUnitを使うには、まず開発環境を整える必要があります。この章では、VSCodeとMavenを使ってJUnitプロジェクトを作ります。

必要なツールの確認

以下のツールがインストールされていることを確認してください。

Visual Studio Code（VSCode）

無料のコードエディタ。プログラムを書くための専用ソフトです。

Extension Pack for Java

VSCodeでJava開発をするための拡張機能パックです。

JDK 17以上

Javaでプログラムを作るために必要な道具一式です。

Mavenとは？

Maven（メイヴン）は、Javaプロジェクトを管理するためのツールです。 必要なライブラリ（JUnitなど）を自動でダウンロードしてくれる便利な道具だと考えてください。

2.2 Mavenプロジェクトの作成

VSCodeでMavenプロジェクトを作成しましょう。

実習：Mavenプロジェクトを作成しよう

  1. 1. VSCodeを起動

    VSCodeを開きます。

  2. 2. コマンドパレットを開く

    Ctrl+Shift+P（Mac: Cmd+Shift+P）を押します。

  3. 3. Mavenプロジェクト作成

    「Java: Create Java Project」と入力して選択します。

  4. 4. Maven を選択

    プロジェクトタイプから「Maven」を選びます。

  5. 5. テンプレートを選択

    「maven-archetype-quickstart」を選びます。

  6. 6. バージョンを選択

    最新バージョン（例: 1.4）を選びます。

  7. 7. Group ID を入力

    「com.example」と入力します（プロジェクトの分類名のようなものです）。

  8. 8. Artifact ID を入力

    「junit-demo」と入力します（プロジェクト名です）。

  9. 9. 保存場所を選択

    プロジェクトを保存する場所を選びます。

プロジェクト構造の確認

プロジェクトが作成されると、以下のような構造になっています。 

junit-demo/
├── pom.xml              ← プロジェクト設定ファイル
└── src/
    ├── main/
    │   └── java/        ← プログラム本体を置く場所
    │       └── com/
    │           └── example/
    │               └── App.java
    └── test/
        └── java/        ← テストコードを置く場所
            └── com/
                └── example/
                    └── AppTest.java

ポイント

Mavenプロジェクトでは、プログラム本体（src/main/java）とテストコード（src/test/java）を分けて管理します。 これにより、コードが整理されて分かりやすくなります。

2.3 JUnit 5の導入

プロジェクトにJUnit 5を追加しましょう。pom.xmlというファイルを編集します。

pom.xmlの編集

プロジェクトのルートにあるpom.xmlを開き、以下の内容に更新します。

pom.xml 
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>junit-demo</artifactId>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
    </properties>

    <dependencies>
        <!-- JUnit 5 -->
        <dependency>
            <groupId>org.junit.jupiter</groupId>
            <artifactId>junit-jupiter</artifactId>
            <version>5.10.0</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>

pom.xmlの役割

pom.xmlは、プロジェクトで使うライブラリを定義するファイルです。 ここに「JUnit 5を使います」と書くと、Mavenが自動的にダウンロードしてくれます。

依存関係の解決

pom.xmlを保存すると、VSCodeが自動的にJUnit 5をダウンロードします。少し待ちましょう。

2.4 最初のテストコードを書く

いよいよ、人生で初めてのテストコードを書いてみましょう！

シンプルな計算クラスを作る

まず、テスト対象となる簡単な計算クラスを作ります。 src/main/java/com/example/Calculator.javaを作成します。

Calculator.java 
package com.example;

/**
 * 簡単な計算機クラス
 */
public class Calculator {
    /**
     * 2つの数を足す
     */
    public int add(int a, int b) {
        return a + b;
    }

    /**
     * 2つの数を引く
     */
    public int subtract(int a, int b) {
        return a - b;
    }
}

テストクラスを作る

次に、このCalculatorクラスをテストするコードを書きます。 src/test/java/com/example/CalculatorTest.javaを作成します。

CalculatorTest.java 
package com.example;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

/**
 * Calculatorクラスのテスト
 */
public class CalculatorTest {

    @Test
    public void testAdd() {
        // テスト対象のクラスを作る
        Calculator calculator = new Calculator();

        // 実際に計算してみる
        int result = calculator.add(2, 3);

        // 結果が期待通りか確認する
        assertEquals(5, result);
    }
}

コードの解説

  • @Test - 「これはテストメソッドです」という印（アノテーション）
  • assertEquals(期待値, 実際の値) - 2つの値が等しいか確認する
  • テストメソッド名はtestで始めるのが慣例です
flowchart LR A[Calculatorクラス] -->|add 2, 3| B[計算実行] B --> C[結果: 5] C --> D{assertEquals} D -->|期待値5と一致| E[テスト成功] D -->|一致しない| F[テスト失敗]

2.5 テストを実行する

テストコードを書いたら、実際に実行してみましょう。

動作確認：テストを実行しよう

  1. 方法1: VSCodeのテストエクスプローラーを使う

    左側のサイドバーから「テスト」アイコンをクリックし、「Run Test」ボタンを押します。

  2. 方法2: コードエディタから直接実行

    テストメソッドの上に表示される「Run Test」リンクをクリックします。

  3. 方法3: コマンドで実行

    ターミナルで以下のコマンドを実行します。

    mvn test

テスト結果の見方

テストが成功すると、緑色のチェックマークが表示されます。

成功時の表示例

✓ testAdd (12 ms)

Tests run: 1, Failures: 0, Errors: 0, Skipped: 0

おめでとうございます！

これであなたは人生で初めてのJUnitテストを書いて実行することができました。 テストが成功したということは、Calculatorクラスのaddメソッドが正しく動いていることを確認できたということです。

失敗するテストを試してみる

わざと間違った期待値を指定して、テストが失敗する様子も見てみましょう。

わざと失敗させる例 
@Test
public void testAddWrongExpectation() {
    Calculator calculator = new Calculator();
    int result = calculator.add(2, 3);

    // わざと間違った期待値を指定
    assertEquals(6, result); // 実際は5だが、6を期待している
}

このテストを実行すると、以下のようなエラーメッセージが表示されます。

失敗時の表示例

✗ testAddWrongExpectation (5 ms)
  expected: <6> but was: <5>

エラーメッセージの読み方

「expected: <6> but was: <5>」は、「期待していたのは6だけど、実際は5だった」という意味です。 このようにJUnitは、何が間違っていたのかを教えてくれます。

この章のまとめ

  • VSCodeとMavenを使ってJUnitプロジェクトを作成できる
  • pom.xmlにJUnit 5の依存関係を追加することで、JUnitを使えるようになる
  • @Testアノテーションを付けたメソッドがテストとして実行される
  • assertEquals(期待値, 実際の値)で結果を検証できる
  • テストが成功すると緑色、失敗すると赤色で表示される

理解度確認クイズ

  1. Q1. Mavenプロジェクトで、プログラム本体とテストコードはそれぞれどこに配置しますか？

    回答を表示
    • プログラム本体： src/main/java
    • テストコード： src/test/java

  2. Q2. pom.xmlファイルの役割は何ですか？

    回答を表示
    pom.xmlは、プロジェクトで使うライブラリ（依存関係）を定義するファイルです。 ここに使いたいライブラリを記述すると、Mavenが自動的にダウンロードしてくれます。

  3. Q3. @Testアノテーションの役割は何ですか？

    回答を表示
    @Testアノテーションは、「このメソッドはテストメソッドです」とJUnitに伝えるための印です。 @Testが付いているメソッドは、JUnitが自動的に見つけて実行してくれます。

  4. Q4. assertEquals(5, result) の意味を説明してください。

    回答を表示
    「resultの値が5と等しいことを確認する」という意味です。 第1引数が期待値、第2引数が実際の値です。両者が一致すればテスト成功、一致しなければ失敗となります。
