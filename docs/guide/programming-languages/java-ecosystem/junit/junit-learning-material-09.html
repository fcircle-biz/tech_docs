9.1 電卓アプリの概要

この章では、これまで学んだJUnitの知識を総動員して、実際の電卓アプリケーションをテストします。 電卓アプリは、足し算・引き算・掛け算・割り算という基本的な四則演算を提供するシンプルなクラスです。

実践的な学習のポイント

実際の開発現場では、既存のコードに対してテストを書く機会が多くあります。 この章では、完成したクラスに対してテストを書く実践的な演習を行います。

電卓クラスの仕様

Calculatorクラスは以下の4つのメソッドを提供します：

add(a, b)

2つの数値の足し算を行います
例：add(2, 3) → 5

subtract(a, b)

2つの数値の引き算を行います
例：subtract(5, 2) → 3

multiply(a, b)

2つの数値の掛け算を行います
例：multiply(4, 3) → 12

divide(a, b)

2つの数値の割り算を行います
例：divide(10, 2) → 5

9.2 Calculatorクラスの実装

まず、テスト対象となるCalculatorクラスを作成しましょう。

Calculator.java 
public class Calculator {

    // 足し算メソッド
    public double add(double a, double b) {
        return a + b;
    }

    // 引き算メソッド
    public double subtract(double a, double b) {
        return a - b;
    }

    // 掛け算メソッド
    public double multiply(double a, double b) {
        return a * b;
    }

    // 割り算メソッド（0で割るとエラー）
    public double divide(double a, double b) {
        if (b == 0) {
            throw new ArithmeticException("0で割ることはできません");
        }
        return a / b;
    }
}

重要なポイント

  • すべてのメソッドはdouble型を使用（小数点の計算に対応）
  • divideメソッドは0除算のチェックを実装
  • 0で割ろうとするとArithmeticExceptionをスロー

9.3 テストクラスのセットアップ

次に、Calculatorクラスをテストするためのテストクラスを作成します。

CalculatorTest.java 
import org.junit.jupiter.api.*;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        // 各テストの前に新しいCalculatorインスタンスを作成
        calculator = new Calculator();
    }

    // テストメソッドはこの後に追加していきます
}

@BeforeEachの活用

各テストメソッドの実行前に新しいCalculatorインスタンスを作成することで、 テスト間の独立性を保ちます。これは第5章で学んだライフサイクル管理の実践です。

9.4 足し算のテスト

まずは足し算メソッドのテストから始めましょう。様々なケースをテストすることが重要です。

足し算のテストメソッド 
@Test
@DisplayName("足し算：正の数同士")
void testAddPositiveNumbers() {
    double result = calculator.add(2.0, 3.0);
    assertEquals(5.0, result);
}

@Test
@DisplayName("足し算：負の数を含む")
void testAddWithNegativeNumbers() {
    double result = calculator.add(-5.0, 3.0);
    assertEquals(-2.0, result);
}

@Test
@DisplayName("足し算：小数点を含む")
void testAddWithDecimals() {
    double result = calculator.add(2.5, 3.7);
    assertEquals(6.2, result, 0.0001);  // 小数点誤差を許容
}

@Test
@DisplayName("足し算：0を含む")
void testAddWithZero() {
    double result = calculator.add(5.0, 0.0);
    assertEquals(5.0, result);
}

小数点の比較について

小数点を含む計算では、コンピュータの内部表現の関係で微小な誤差が生じることがあります。 assertEquals(expected, actual, delta)の第3引数で許容誤差を指定できます。

テストケースの設計思想

考慮すべきテストパターン：

✓ 正の数同士： 一般的なケース
✓ 負の数を含む： 符号が異なる場合
✓ 小数点を含む： 精度が必要な場合
✓ 0を含む： 境界値テスト

9.6 掛け算のテスト

掛け算では、0を掛けた場合や負の数を掛けた場合など、特徴的なケースをテストします。

掛け算のテストメソッド 
@Test
@DisplayName("掛け算：正の数同士")
void testMultiplyPositiveNumbers() {
    double result = calculator.multiply(4.0, 3.0);
    assertEquals(12.0, result);
}

@Test
@DisplayName("掛け算：0を掛ける")
void testMultiplyByZero() {
    double result = calculator.multiply(5.0, 0.0);
    assertEquals(0.0, result);
}

@Test
@DisplayName("掛け算：負の数を含む")
void testMultiplyWithNegative() {
    double result = calculator.multiply(-3.0, 4.0);
    assertEquals(-12.0, result);
}

@Test
@DisplayName("掛け算：負の数同士")
void testMultiplyNegativeNumbers() {
    double result = calculator.multiply(-3.0, -4.0);
    assertEquals(12.0, result);  // 負×負=正
}

掛け算の特性

掛け算には「0を掛けると0」「負×負=正」といった数学的な特性があります。 これらの特性が正しく実装されているかを確認することが重要です。

9.7 割り算のテスト（正常系）

割り算は、正常に割り切れる場合と、0で割ろうとしてエラーになる場合の両方をテストする必要があります。 まずは正常系からテストしましょう。

割り算の正常系テスト 
@Test
@DisplayName("割り算：正の数同士")
void testDividePositiveNumbers() {
    double result = calculator.divide(10.0, 2.0);
    assertEquals(5.0, result);
}

@Test
@DisplayName("割り算：割り切れない場合")
void testDivideNotExact() {
    double result = calculator.divide(10.0, 3.0);
    assertEquals(3.333333, result, 0.0001);
}

@Test
@DisplayName("割り算：負の数を含む")
void testDivideWithNegative() {
    double result = calculator.divide(-10.0, 2.0);
    assertEquals(-5.0, result);
}

9.8 割り算のテスト（異常系）

0で割ろうとした場合、正しくエラーが発生することを確認します。これは第7章で学んだ例外テストの実践です。

割り算の異常系テスト 
@Test
@DisplayName("割り算：0で割るとエラー")
void testDivideByZero() {
    ArithmeticException exception = assertThrows(
        ArithmeticException.class,
        () -> calculator.divide(10.0, 0.0)
    );
    assertEquals("0で割ることはできません", exception.getMessage());
}

異常系テストの重要性

プログラムは「正しく動く」だけでなく、「正しくエラーを出す」ことも重要です。 異常系のテストを書くことで、ユーザーが間違った操作をしたときに、 プログラムが適切にエラーを処理できることを確認できます。

理解度確認クイズ

  1. Q1. 正常系と異常系のテストをそれぞれ説明してください。

    回答を表示

    正常系：プログラムが正しく動作する場合のテスト。例えば電卓の足し算が正しい結果を返すことを確認します。

    異常系：プログラムが正しくエラーを出す場合のテスト。例えば0で割ったときに適切な例外がスローされることを確認します。

  2. Q2. なぜ@BeforeEachで毎回新しいCalculatorインスタンスを作成するのですか？

    回答を表示
    各テストメソッドが独立して実行されることを保証するためです。 もし同じインスタンスを使い回すと、前のテストの影響を受けてしまう可能性があります。 毎回新しいインスタンスを作ることで、テストの独立性と再現性が確保されます。

  3. Q3. 小数点の計算結果を検証する際、assertEquals(expected, actual, delta)のdelta引数を使う理由は何ですか？

    回答を表示
    コンピュータは小数点を内部的に2進数で表現するため、計算結果に微小な誤差が生じることがあります。 delta引数を使うことで「この範囲内の誤差なら正しいとみなす」という許容範囲を設定できます。 例えば0.0001を指定すると、期待値との差が0.0001以内なら成功と判定されます。

  4. Q4. 電卓の掛け算メソッドに対して、どのようなテストケースを考えるべきですか？4つ挙げてください。

    回答を表示
    1. 正の数同士の掛け算（一般的なケース）
    2. 0を掛ける場合（結果が0になることを確認）
    3. 負の数を含む掛け算（符号が変わることを確認）
    4. 負の数同士の掛け算（負×負=正を確認）
