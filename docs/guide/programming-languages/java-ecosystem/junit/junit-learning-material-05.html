第5章

テストの準備と後片付け - ライフサイクルを理解しよう

@BeforeEach、@AfterEach、@BeforeAll、@AfterAllアノテーションを使った前処理・後処理について学びます。料理で例えると、調理前の材料準備と調理後の片付けのようなものです。テストの効率化と確実な実行環境の整備方法を解説します。

この章で学ぶこと

  • テストのライフサイクル（生まれてから終わるまでの流れ）を理解する
  • @BeforeEachと@AfterEachアノテーションの使い方を学ぶ
  • @BeforeAllと@AfterAllアノテーションの使い方を学ぶ
  • 効率的なテストコードの書き方を身につける

5.1 テストのライフサイクルとは？

テストを実行するときには、実は「テストを実行する」だけでなく、その前後にいろいろな処理が必要になることがあります。

例えば、データベースを使うテストなら「データベースに接続する」という準備が必要ですし、テストが終わったら「データベースを切断する」という後片付けが必要です。

料理で例えると

料理を作るとき、まず材料を準備しますよね（野菜を洗う、包丁を用意する）。 そして料理を作り、最後に使った道具を洗って片付けます。 テストも同じで、「準備→実行→片付け」という流れがあるのです。

5.2 各テストの前後に実行する - @BeforeEachと@AfterEach

@BeforeEachと@AfterEachは、各テストメソッドの前後に実行されるメソッドに付けるアノテーションです。

@BeforeEach - 準備処理

各テストを実行する前に毎回実行される処理です。テストで使うオブジェクトの初期化などを行います。

CalculatorTest.java - @BeforeEachの基本 
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    // テストで使う電卓オブジェクト
    private Calculator calculator;

    // 各テストの前に実行される
    @BeforeEach
    void setUp() {
        // 新しい電卓を用意（準備）
        calculator = new Calculator();
        System.out.println("電卓を準備しました");
    }

    @Test
    void testAdd() {
        // calculatorはsetUp()で準備済み
        assertEquals(5, calculator.add(2, 3));
    }

    @Test
    void testSubtract() {
        // こちらのテストでもcalculatorは準備済み
        assertEquals(1, calculator.subtract(3, 2));
    }
}

実行の流れ

  1. setUp()を実行（calculatorを準備）
  2. testAdd()を実行
  3. setUp()を実行（新しいcalculatorを準備）
  4. testSubtract()を実行

各テストは毎回まっさらな状態から始まります！

@AfterEach - 後片付け処理

各テストを実行した後に毎回実行される処理です。使ったリソースの解放などを行います。

FileTest.java - @AfterEachの基本 
import org.junit.jupiter.api.*;
import java.io.*;

class FileTest {
    private File testFile;

    @BeforeEach
    void setUp() throws IOException {
        // テスト用のファイルを作成
        testFile = new File("test.txt");
        testFile.createNewFile();
        System.out.println("テストファイルを作成");
    }

    @AfterEach
    void tearDown() {
        // テストで使ったファイルを削除
        if (testFile.exists()) {
            testFile.delete();
            System.out.println("テストファイルを削除");
        }
    }

    @Test
    void testFileExists() {
        // ファイルが存在することを確認
        assertTrue(testFile.exists());
    }

    @Test
    void testFileWrite() throws IOException {
        // ファイルに書き込みできることを確認
        // ... テストコード ...
    }
}

5.3 全テストの前後に実行する - @BeforeAllと@AfterAll

@BeforeAllと@AfterAllは、全テストの実行前と実行後に1回だけ実行されるメソッドに付けるアノテーションです。

@BeforeAll - 全体の準備

すべてのテストを実行する前に1回だけ実行されます。データベース接続の確立など、重い処理を1回だけ行いたいときに使います。

重要な注意点

@BeforeAllと@AfterAllを付けたメソッドはstaticメソッドにする必要があります。

DatabaseTest.java - @BeforeAllと@AfterAll 
import org.junit.jupiter.api.*;

class DatabaseTest {
    // すべてのテストで共有するデータベース接続
    private static Connection dbConnection;

    // 全テストの前に1回だけ実行
    @BeforeAll
    static void initAll() {
        System.out.println("データベースに接続");
        // データベース接続を確立（例）
        // dbConnection = ...
    }

    // 全テストの後に1回だけ実行
    @AfterAll
    static void tearDownAll() {
        System.out.println("データベース接続を閉じる");
        // データベース接続を閉じる
        // dbConnection.close();
    }

    @BeforeEach
    void setUp() {
        System.out.println("テストデータを準備");
    }

    @AfterEach
    void tearDown() {
        System.out.println("テストデータを削除");
    }

    @Test
    void testQuery1() {
        System.out.println("テスト1実行");
        // テストコード
    }

    @Test
    void testQuery2() {
        System.out.println("テスト2実行");
        // テストコード
    }
}

実行順序：

データベースに接続      ← @BeforeAll（1回）
テストデータを準備       ← @BeforeEach
テスト1実行
テストデータを削除       ← @AfterEach
テストデータを準備       ← @BeforeEach
テスト2実行
テストデータを削除       ← @AfterEach
データベース接続を閉じる ← @AfterAll（1回）

5.4 いつどれを使えばいい？

4つのアノテーションの使い分けを理解しましょう。

@BeforeEach - 各テストの前

テストごとに新しいオブジェクトが必要な場合に使います。

例：電卓オブジェクト、リストの初期化、テストデータの準備

@AfterEach - 各テストの後

テストで使ったリソースを片付ける場合に使います。

例：ファイルの削除、一時データのクリーンアップ

@BeforeAll - 全体の開始前

時間がかかる初期化を1回だけ行いたい場合に使います。

例：データベース接続、設定ファイルの読み込み

@AfterAll - 全体の終了後

全テスト終了後に1回だけ後片付けしたい場合に使います。

例：データベース接続の切断、共有リソースの解放

5.5 実習 - ライフサイクルを体験しよう

実習：買い物カゴのテスト

買い物カゴ（ShoppingCart）クラスをテストしてみましょう。

Step 1: テスト対象クラスを作成

// ShoppingCart.java
import java.util.ArrayList;
import java.util.List;

public class ShoppingCart {
    private List<String> items;

    public ShoppingCart() {
        this.items = new ArrayList<>();
    }

    // 商品を追加
    public void addItem(String item) {
        items.add(item);
    }

    // カゴの中身の数
    public int getItemCount() {
        return items.size();
    }

    // カゴを空にする
    public void clear() {
        items.clear();
    }
}

Step 2: テストクラスを作成

// ShoppingCartTest.java
import org.junit.jupiter.api.*;
import static org.junit.jupiter.api.Assertions.*;

class ShoppingCartTest {
    private ShoppingCart cart;

    @BeforeAll
    static void initAll() {
        System.out.println("テスト開始");
    }

    @BeforeEach
    void setUp() {
        // 各テストの前に新しいカゴを用意
        cart = new ShoppingCart();
        System.out.println("新しいカゴを準備");
    }

    @AfterEach
    void tearDown() {
        System.out.println("カゴをクリア");
        cart = null;
    }

    @AfterAll
    static void tearDownAll() {
        System.out.println("全テスト完了");
    }

    @Test
    void testAddItem() {
        cart.addItem("りんご");
        assertEquals(1, cart.getItemCount());
    }

    @Test
    void testAddMultipleItems() {
        cart.addItem("りんご");
        cart.addItem("バナナ");
        assertEquals(2, cart.getItemCount());
    }
}

この章のまとめ

  • テストには準備→実行→片付けというライフサイクルがある
  • @BeforeEachは各テストの前に実行される（オブジェクトの初期化など）
  • @AfterEachは各テストの後に実行される（リソースの解放など）
  • @BeforeAllは全テストの前に1回だけ実行される（staticメソッド）
  • @AfterAllは全テストの後に1回だけ実行される（staticメソッド）
  • ライフサイクルを活用することで、効率的で確実なテストコードが書ける

理解度確認クイズ

  1. Q1. @BeforeEachと@BeforeAllの違いは何ですか？

    回答を表示

    @BeforeEach：各テストメソッドの前に毎回実行される。テストごとに新しいオブジェクトを準備したい場合に使う。

    @BeforeAll：全テストの最初に1回だけ実行される。staticメソッドにする必要がある。重い初期化処理を1回だけ行いたい場合に使う。

  2. Q2. @BeforeAllと@AfterAllを付けたメソッドには、どんな制約がありますか？

    回答を表示
    staticメソッドにする必要があります。これは、テストクラスのインスタンスが作られる前や、すべてのインスタンスが終了した後に実行されるためです。

  3. Q3. 次のテストクラスを実行したとき、コンソールにはどのような順序でメッセージが表示されますか？

    @BeforeAll static void init() { print("A"); }
@BeforeEach void setUp() { print("B"); }
@Test void test1() { print("C"); }
@Test void test2() { print("D"); }
@AfterEach void tearDown() { print("E"); }
@AfterAll static void cleanup() { print("F"); }
    回答を表示
    A → B → C → E → B → D → E → F

    最初に@BeforeAll(A)、次に@BeforeEach(B)→test1(C)→@AfterEach(E)、 その後@BeforeEach(B)→test2(D)→@AfterEach(E)、最後に@AfterAll(F)が実行されます。

  4. Q4. 各テストで使うArrayListを毎回空の状態から始めたい場合、どのアノテーションを使いますか？

    回答を表示
    @BeforeEachを使います。各テストの前に新しいArrayListを作成することで、テスト間の影響を防ぎ、常にまっさらな状態でテストを開始できます。
