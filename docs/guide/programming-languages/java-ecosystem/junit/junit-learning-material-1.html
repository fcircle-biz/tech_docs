第1章: JUnit基礎とセットアップ

JUnit 5を使った単体テストの基礎

この章で学ぶこと
  • JUnitとは何か、なぜ単体テストが重要なのかを理解する
  • JUnit 5の特徴と構成を理解する
  • MavenやGradleプロジェクトでJUnit 5をセットアップする方法
  • 最初の簡単なテストケースを作成し、実行する
  • テストクラスとテストメソッドの基本的な構造を理解する

1.1 JUnitとは何か

JUnitは、Javaプログラムの単体テストを行うためのフレームワークです。単体テストとは、プログラムの最小単位（メソッドやクラス）が期待通りに動作するかを検証することです。

flowchart TD A["プログラム開発"] --> B["コード作成"] B --> C["JUnitテスト作成"] C --> D["テスト実行"] D --> E{テスト結果} E -->|成功| F["品質の高いコード"] E -->|失敗| G["バグ発見・修正"] G --> B F --> H["安心してリリース"]

なぜ単体テストが重要なのか？

  • バグの早期発見：プログラム開発の早い段階でバグを発見できます
  • 安心感のあるリファクタリング：コードを改善する際、既存機能が壊れていないか確認できます
  • 仕様書としての役割：テストコードを見れば、そのメソッドがどのように動作すべきかが分かります
  • チーム開発での安全性：他の開発者がコードを変更しても、テストが通れば安心です

1.2 JUnit 5の特徴

JUnit 5は、2017年にリリースされたJUnitの最新バージョンです。以前のJUnit 4と比べて、以下の特徴があります：

flowchart LR A["JUnit 5プラットフォーム"] --> B["JUnit Jupiter"] A --> C["JUnit Vintage"] B --> D["新しいテスト記述方式"] B --> E["拡張モデル"] C --> F["JUnit 3/4との互換性"]

JUnit 5の主な改善点

  • Java 8以降対応：ラムダ式や新しい言語機能を活用
  • モジュラー構造：必要な部分だけを使用可能
  • 豊富なアサーション：様々な検証方法を提供
  • 動的テスト：実行時にテストケースを生成可能
  • パラメータ化テスト：複数のデータでテストを実行

1.3 プロジェクトのセットアップ

JUnit 5を使用するためには、プロジェクトにJUnitライブラリを追加する必要があります。MavenとGradleの両方の方法を説明します。

1.3.1 Mavenプロジェクトでの設定

Mavenを使用している場合は、pom.xmlファイルに以下の依存関係を追加します：

<dependencies>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter</artifactId>
        <version>5.10.0</version>
        <scope>test</scope>
    </dependency>
</dependencies>

<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-surefire-plugin</artifactId>
            <version>3.1.2</version>
        </plugin>
    </plugins>
</build>

1.3.2 Gradleプロジェクトでの設定

Gradleを使用している場合は、build.gradleファイルに以下を追加します：

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}

test {
    useJUnitPlatform()
}
重要な注意事項

JUnit 5を使用する場合は、必ずtest { useJUnitPlatform() }をGradleに追加してください。これがないとテストが正しく実行されません。

1.4 最初のテストを作成してみよう

それでは、実際にJUnitを使って最初のテストを作成してみましょう。まず、テスト対象となる簡単なクラスを作成します。

実習 1-1: 電卓クラスとテストの作成

簡単な電卓機能を持つクラスを作成し、そのテストを書いてみます。

手順1: テスト対象クラスの作成

まず、src/main/javaにCalculator.javaを作成します：

public class Calculator {
    
    public int add(int a, int b) {
        return a + b;
    }
    
    public int subtract(int a, int b) {
        return a - b;
    }
    
    public int multiply(int a, int b) {
        return a * b;
    }
    
    public double divide(int a, int b) {
        if (b == 0) {
            throw new IllegalArgumentException("ゼロで割ることはできません");
        }
        return (double) a / b;
    }
}
手順2: テストクラスの作成

次に、src/test/javaにCalculatorTest.javaを作成します：

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    
    @Test
    void 足し算のテスト() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
    
    @Test
    void 引き算のテスト() {
        Calculator calculator = new Calculator();
        int result = calculator.subtract(5, 3);
        assertEquals(2, result);
    }
}
手順3: テストの実行

IDEで右クリックして「テストの実行」を選択するか、以下のコマンドでテストを実行します：

# Mavenの場合
mvn test

# Gradleの場合
./gradlew test
期待される結果

テストが成功すると、以下のような結果が表示されます：

  • 2つのテストが実行される
  • どちらもPASSED（成功）となる
  • 「足し算のテスト」では 2 + 3 = 5 が検証される
  • 「引き算のテスト」では 5 - 3 = 2 が検証される

1.5 テストクラスの基本構造

JUnit 5のテストクラスには、いくつかの重要な要素があります：

flowchart TD A["テストクラス"] --> B["@Test アノテーション"] A --> C["テストメソッド"] A --> D["アサーション"] B --> E["このメソッドがテストであることを示す"] C --> F["実際のテスト処理を記述"] D --> G["期待値と実際の値を比較"]

1.5.1 基本的なルール

  • クラス名：テスト対象クラス名 + "Test" （例：Calculator → CalculatorTest）
  • メソッド名：日本語でも英語でもOK、分かりやすい名前を付ける
  • @Test アノテーション：テストメソッドには必須
  • public修飾子：JUnit 5では不要（package-private でOK）
  • staticメソッド：通常のテストメソッドはstaticにしない

1.5.2 テストの基本パターン

良いテストには「AAA パターン」という基本構造があります：

@Test
void テストメソッド名() {
    // Arrange（準備）：テストデータとオブジェクトを準備
    Calculator calculator = new Calculator();
    int a = 10;
    int b = 5;
    
    // Act（実行）：テスト対象のメソッドを実行
    int result = calculator.add(a, b);
    
    // Assert（検証）：期待値と実際の結果を比較
    assertEquals(15, result);
}
AAA パターンとは
  • Arrange（準備）：テストに必要なオブジェクトやデータを準備
  • Act（実行）：テスト対象のメソッドを実行
  • Assert（検証）：結果が期待通りかを確認

この構造にすることで、テストが読みやすく、理解しやすくなります。

1.6 IDEでのテスト実行

多くのIDEでは、JUnitテストを簡単に実行できる機能が提供されています：

IntelliJ IDEAの場合

  • テストクラスまたはメソッドの横の緑の▶ボタンをクリック
  • 右クリックメニューから「Run テスト名」を選択
  • Ctrl + Shift + F10 でカーソル位置のテストを実行

Eclipseの場合

  • テストクラスを右クリック → 「Run As」 → 「JUnit Test」
  • Alt + Shift + X, T でJUnitテストを実行

VS Codeの場合

  • Java Test Runnerプラグインをインストール
  • テストメソッドの上に表示される「Run Test」をクリック
実習 1-2: 失敗するテストを作成してみよう

わざと失敗するテストを作成して、JUnitがどのように失敗を報告するかを確認します。

手順

以下のテストメソッドをCalculatorTest.javaに追加してください：

@Test
void わざと失敗するテスト() {
    Calculator calculator = new Calculator();
    int result = calculator.add(2, 3);
    assertEquals(6, result);  // わざと間違った期待値を設定
}
期待される結果

このテストを実行すると、以下のようなエラーメッセージが表示されます：

org.opentest4j.AssertionFailedError: 
Expected :6
Actual   :5

このメッセージから、期待していた値（6）と実際の値（5）が異なることが分かります。

理解度確認クイズ
  1. JUnitの主な目的は何ですか？
    • a) プログラムの実行速度を向上させる
    • b) コードの単体テストを自動化する
    • c) データベースに接続する
    • d) ユーザーインターフェースを作成する
  2. JUnit 5でテストメソッドを示すアノテーションは何ですか？
    • a) @TestCase
    • b) @TestMethod
    • c) @Test
    • d) @JUnit
  3. AAAパターンの「Act」は何を意味しますか？
    • a) テストデータを準備すること
    • b) テスト対象のメソッドを実行すること
    • c) 結果を検証すること
    • d) テストを終了すること
解答を見る

1. b) コードの単体テストを自動化する

2. c) @Test

3. b) テスト対象のメソッドを実行すること
