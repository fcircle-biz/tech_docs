テストって何だろう？ - ソフトウェアテストの世界へようこそ

プログラムを「テストする」とはどういうことかを学びます。なぜテストが必要なのか、手動テストと自動テストの違い、JUnitとは何かを身近な例を使って解説します。

この章で学ぶこと

  • テストとは何か、なぜ必要なのかを理解する
  • 手動テストと自動テストの違いを知る
  • JUnitとは何か、どんなツールなのかを理解する
  • 単体テストの基本概念を身につける

1.1 テストとは何か？

プログラムを書いた後、「ちゃんと動くかな？」と確認しますよね。それがテストです。

レシピを見ながら料理を作ったとき、味見をして「レシピ通りの味になったかな？」と確認するのと同じです。 プログラムでも、作った機能が「期待通りに動くか」を確認する必要があります。

料理で例えると

カレーを作ったら味見をしますよね。「塩が足りないかな？」「辛さはちょうどいい？」と確認します。 プログラムのテストも同じで、「計算結果は正しいかな？」「エラーは出ないかな？」と確認するのです。

1.2 なぜテストが必要なのか？

「動いたから大丈夫でしょ？」と思うかもしれませんが、以下のような理由でテストは必要不可欠です。

1. バグを見つける

プログラムには必ずバグ（不具合）があります。テストをすることで、バグを早く見つけて修正できます。

2. 安心して修正できる

プログラムを修正したとき、「他の部分が壊れてないかな？」と心配になります。 テストがあれば、すぐに確認できるので安心です。

3. ドキュメント代わりになる

テストコードを読むと「このメソッドはこういう使い方をするんだ」と分かります。 説明書のような役割も果たすのです。

1.3 手動テストと自動テスト

テストには大きく分けて2つの方法があります。

手動テスト

人間が実際にプログラムを動かして、目で見て確認する方法です。

例：計算機プログラムの場合

  1. プログラムを実行する
  2. 「2 + 3」と入力する
  3. 結果が「5」と表示されることを目で確認する
  4. 次は「10 - 4」と入力する...

手動テストの問題点

  • 時間がかかる（毎回同じことを繰り返す必要がある）
  • ミスしやすい（確認を忘れたり、見落としたり）
  • 疲れる（同じ作業を何度もするのは大変）

自動テスト

プログラムでテストを書いて、自動的に確認する方法です。JUnitを使うのがこの方法です。

例：計算機プログラムの場合

  1. テストコードを書く（「2 + 3 = 5」を確認するコード）
  2. ボタン1つでテストを実行
  3. 自動的に結果を確認して、合格/不合格を表示

自動テストのメリット

  • 速い（コンピュータが一瞬で確認）
  • 正確（ミスがない）
  • 何度でも実行できる（疲れない）
  • 修正のたびに簡単に再テストできる

1.4 JUnitとは何か？

JUnit（ジェイユニット）は、Javaプログラムの自動テストを簡単に書くためのテスティングフレームワークです。

フレームワークとは「枠組み」のことで、テストを書くための便利な道具一式だと考えてください。

JUnitの特徴

  • • 無料で使える - オープンソースのツールです
  • • 世界中で使われている - Java開発では標準的なツールです
  • • 簡単に書ける - シンプルな書き方でテストを作れます
  • • IDEと連携 - EclipseやIntelliJ IDEAから簡単に実行できます

JUnitのバージョン

JUnitには主に2つのバージョンがあります。

JUnit 4（旧バージョン）

古いプロジェクトで使われていることが多いですが、新しいプロジェクトでは推奨されません。

JUnit 5（最新版）

推奨

現在の標準バージョンです。このガイドではJUnit 5を使います。 別名「JUnit Jupiter」とも呼ばれます。

1.5 単体テスト（ユニットテスト）とは？

JUnitで書くテストは単体テスト（Unit Test）と呼ばれます。

単体テストとは、プログラムの小さな部品（メソッドやクラス）が正しく動くかを確認するテストです。

自動車で例えると

自動車を作るとき、エンジン、ブレーキ、ハンドルなど、それぞれの部品が正しく動くかを個別にテストしますよね。 これが単体テストです。全部組み立ててから初めてテストするのではなく、部品ごとに確認するのです。

flowchart TB A[プログラム全体] --> B[クラスA] A --> C[クラスB] A --> D[クラスC] B --> B1[メソッド1] B --> B2[メソッド2] C --> C1[メソッド3] C --> C2[メソッド4] D --> D1[メソッド5] B1 -.単体テストで確認.-> T1[テスト1] B2 -.単体テストで確認.-> T2[テスト2] C1 -.単体テストで確認.-> T3[テスト3]

単体テストのメリット

問題を早く発見できる

小さな部品ごとにテストするので、どこに問題があるかすぐに分かります。

安心して修正できる

テストがあれば、修正後もすぐに動作を確認できます。

コードの品質が上がる

テストを書くことで、「テストしやすい綺麗なコード」を書く習慣がつきます。

この章のまとめ

  • テストとは、プログラムが期待通りに動くか確認すること
  • テストがあると、バグを早く発見でき、安心して修正できる
  • 手動テストは時間がかかり疲れるが、自動テストは速く正確
  • JUnitはJavaの自動テストを書くための便利なツール（フレームワーク）
  • 単体テストは、プログラムの小さな部品ごとに動作を確認するテスト
