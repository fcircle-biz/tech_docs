テストを整理整頓しよう - テストクラスの構成

複数のテストをどのように整理するかを学びます。テストクラスの命名規則、テストメソッドの書き方、@Testアノテーションの意味を解説します。

この章で学ぶこと

  • テストクラスの命名規則と構成方法を理解する
  • @Testアノテーションの意味と使い方を理解する
  • テストメソッドの書き方とベストプラクティスを身につける
  • 分かりやすいテストコードの整理方法を学ぶ

4.1 なぜテストを整理する必要があるのか？

プログラムが大きくなると、テストの数もどんどん増えていきます。 整理されていないテストは、後で見返したときに「このテストは何を確認しているんだっけ？」と分からなくなってしまいます。

本棚で例えると

本をバラバラに置いておくと、読みたい本を探すのが大変ですよね。 「小説」「漫画」「技術書」とジャンル別に整理しておけば、すぐに見つけられます。 テストも同じで、きちんと整理しておくことで、後で見返したときに理解しやすくなります。

4.2 テストクラスの命名規則

JUnitのテストは、テストクラスという単位で整理します。 テストクラスには、テスト対象のクラスと関連性が分かる名前をつけます。

命名規則のパターン

  • テスト対象のクラス名 + "Test"
  • 例: Calculatorクラスのテストなら → CalculatorTest
  • 例: UserServiceクラスのテストなら → UserServiceTest

具体例：計算機クラスの場合

テスト対象のクラス

// Calculator.java
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

テストクラス（命名規則に従う）

// CalculatorTest.java
public class CalculatorTest {
    // テストメソッドをここに書く
}

4.3 @Testアノテーション - テストメソッドの目印

テストクラスの中には、複数のテストメソッドを書きます。 テストメソッドには必ず@Testというアノテーションをつけます。

アノテーションとは？

アノテーションは、コードに「目印」をつけるための記号です。@から始まります。 @Testをつけることで「このメソッドはテストだよ」とJUnitに伝えます。 

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class CalculatorTest {

    @Test  // ← これがアノテーション
    public void testAdd() {
        Calculator calc = new Calculator();
        int result = calc.add(2, 3);
        assertEquals(5, result);
    }
}

注意：@Testを忘れると

@Testアノテーションを忘れると、そのメソッドは普通のメソッドとして扱われ、テストとして実行されません。 必ず@Testをつけましょう！

4.4 テストメソッドの命名規則

テストメソッドの名前は、何をテストしているのかが分かる名前にします。

良い例：分かりやすい名前

@Test
public void testAddPositiveNumbers() {
    // 正の数の足し算をテスト
}

@Test
public void testAddNegativeNumbers() {
    // 負の数の足し算をテスト
}

@Test
public void testDivideByZero() {
    // 0で割るとエラーになることをテスト
}

悪い例：曖昧な名前

@Test
public void test1() {
    // 何のテストか分からない
}

@Test
public void test2() {
    // 何のテストか分からない
}

@Test
public void testCalc() {
    // 何を計算しているか不明
}

命名のコツ

  • "test"で始める（testAdd、testDivideなど）
  • テストする内容を具体的に書く
  • 英語でも日本語でもOK（プロジェクトのルールに従う）
  • 分かりやすさ優先で、長くなっても構わない

4.5 テストクラスの基本構成

テストクラスには、基本的に以下の要素が含まれます。 

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

// テストクラス（publicクラス）
public class CalculatorTest {

    // テストメソッド1
    @Test
    public void testAddPositiveNumbers() {
        // 1. 準備：テスト対象を作る
        Calculator calc = new Calculator();

        // 2. 実行：テストしたい処理を実行
        int result = calc.add(2, 3);

        // 3. 検証：結果が正しいか確認
        assertEquals(5, result);
    }

    // テストメソッド2
    @Test
    public void testSubtract() {
        Calculator calc = new Calculator();
        int result = calc.subtract(10, 4);
        assertEquals(6, result);
    }
}

実習：整理されたテストクラスを作ろう

以下の手順で、整理されたテストクラスを作成してみましょう。

手順1：Stringクラスのテストクラスを作る

  1. StringUtilsTestという名前のクラスを作成
  2. Javaの文字列操作をテストする準備をする

手順2：適切な名前のテストメソッドを追加

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class StringUtilsTest {

    @Test
    public void testStringLength() {
        // 文字列の長さが正しいか
        String text = "Hello";
        assertEquals(5, text.length());
    }

    @Test
    public void testStringUpperCase() {
        // 大文字変換が正しいか
        String text = "hello";
        assertEquals("HELLO", text.toUpperCase());
    }

    @Test
    public void testStringContains() {
        // 部分文字列を含むか
        String text = "Hello World";
        assertTrue(text.contains("World"));
    }
}

ポイント

  • 各テストメソッドは1つのことだけをテストする
  • メソッド名から何をテストしているか分かる
  • 準備→実行→検証の順番で書く

4.6 テストの実行順序について

JUnitでは、テストメソッドの実行順序は保証されません。 どのテストから実行されるか分からないので、各テストは独立して動くように書く必要があります。

重要：テストの独立性

あるテストが他のテストの結果に依存してはいけません。 例えば、「test1が実行された後にtest2を実行する」という前提でテストを書くと、 順序が変わったときに失敗してしまいます。

たとえ話

学校のテストで、問題1を解かないと問題2が解けない、というテストは困りますよね。 どの問題からでも解けるのが良いテストです。JUnitのテストも同じで、 どのテストからでも実行できるように独立させておくことが大切です。

この章のまとめ

  • テストを整理することで、後から見返したときに理解しやすくなる
  • テストクラスの名前は「テスト対象クラス名 + Test」とする（例：CalculatorTest）
  • テストメソッドには必ず@Testアノテーションをつける
  • テストメソッドの名前は、何をテストしているか分かる具体的な名前にする
  • テストは準備→実行→検証の3段階で書く
  • 各テストは独立して動くように書く（実行順序に依存しない）

理解度確認クイズ

  1. Q1. UserServiceというクラスのテストクラスを作るとき、適切な名前はどれですか？

    回答を表示

    正解：UserServiceTest

    テスト対象のクラス名に「Test」をつけるのが命名規則です。

  2. Q2. テストメソッドに必ずつけるアノテーションは何ですか？

    回答を表示

    @Testアノテーションです。これをつけることでJUnitがテストとして認識します。

  3. Q3. テストメソッドの3つの段階を順番に答えてください。

    回答を表示
    1. 準備 - テスト対象のオブジェクトを作る
    2. 実行 - テストしたい処理を実行する
    3. 検証 - 結果が正しいか確認する（アサーション）

  4. Q4. 各テストメソッドは独立して動く必要があります。なぜですか？

    回答を表示
    JUnitではテストの実行順序が保証されないためです。 どの順番で実行されても正しく動くように、各テストは独立している必要があります。
