第3章

期待通りか確かめよう - アサーションの基本

テストで「結果が正しいかどうか」を判定するアサーション（assertion：主張・断言）について学びます。assertEquals、assertTrue、assertFalseなど基本的なアサーションの使い方を、電卓の計算結果を確認するたとえで理解します。

この章で学ぶこと

  • アサーション（assertion）の概念を理解する
  • assertEquals を使った値の比較方法
  • assertTrue と assertFalse の使い分け
  • assertNull と assertNotNull の活用

3.1 アサーション（assertion）とは？

アサーション（assertion）は、「〜であるはずだ」と主張・断言することです。

テストでは、「この計算結果は5になるはずだ」「この値はnullではないはずだ」といった期待値を宣言します。 プログラムの実際の結果が期待値と一致すればテストは成功、異なればテストは失敗します。

電卓で例えると

電卓に「2 + 3」と入力したとき、あなたは「結果は5になるはずだ」と期待しますよね。 実際に5と表示されれば「正しい」、4や6が表示されれば「間違っている」と判断します。 テストのアサーションも同じで、「結果は〜であるべき」と宣言して、実際の結果と比較するのです。

flowchart LR A[プログラム実行] --> B[実際の結果] C[期待値] --> D{アサーション
比較} B --> D D -->|一致| E[テスト成功] D -->|不一致| F[テスト失敗]

3.2 assertEquals - 値が等しいか確認する

最もよく使うアサーションがassertEqualsです。「期待値」と「実際の値」が等しいかを確認します。

基本構文

基本形 
// assertEquals(期待値, 実際の値)
assertEquals(期待値, 実際の値);

実例：計算機のテスト

CalculatorTest.java 
import static org.junit.jupiter.api.Assertions.*;
import org.junit.jupiter.api.Test;

class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calc = new Calculator();
        // 2 + 3 の結果は 5 になるはずだ
        assertEquals(5, calc.add(2, 3));
    }

    @Test
    void testSubtract() {
        Calculator calc = new Calculator();
        // 10 - 4 の結果は 6 になるはずだ
        assertEquals(6, calc.subtract(10, 4));
    }
}

重要ポイント

assertEqualsの第1引数は期待値（expected）、第2引数は実際の値（actual）です。 この順番を守ることで、テストが失敗したときのメッセージが分かりやすくなります。

失敗時のメッセージをカスタマイズ

第3引数にメッセージを追加すると、テスト失敗時により分かりやすい情報を表示できます。

メッセージ付きアサーション 
@Test
void testMultiply() {
    Calculator calc = new Calculator();
    assertEquals(15, calc.multiply(3, 5),
                 "3 × 5 の計算結果が正しくありません");
}

3.3 assertTrue と assertFalse - 真偽値を確認する

条件がtrue（真）であるか、false（偽）であるかを確認するアサーションです。

assertTrue - trueであることを確認

assertTrue 例 
@Test
void testIsPositive() {
    Calculator calc = new Calculator();
    // 5 は正の数なので true になるはずだ
    assertTrue(calc.isPositive(5));
}

@Test
void testGreaterThan() {
    // 10 は 5 より大きいので true になるはずだ
    assertTrue(10 > 5);
}

assertFalse - falseであることを確認

assertFalse 例 
@Test
void testIsNegative() {
    Calculator calc = new Calculator();
    // 5 は負の数ではないので false になるはずだ
    assertFalse(calc.isNegative(5));
}

@Test
void testIsEmpty() {
    String text = "Hello";
    // "Hello" は空文字列ではないので false になるはずだ
    assertFalse(text.isEmpty());
}
flowchart TD A{条件を評価} -->|true| B[assertTrue で確認] A -->|false| C[assertFalse で確認] B --> D[テスト成功] C --> D

3.4 assertNull と assertNotNull - null かどうか確認する

オブジェクトがnull（何も参照していない状態）かどうかを確認するアサーションです。

assertNull - null であることを確認

assertNull 例 
@Test
void testFindUserNotFound() {
    UserService service = new UserService();
    // 存在しないIDで検索すると null が返るはずだ
    User user = service.findById(999);
    assertNull(user, "存在しないユーザーはnullを返す");
}

assertNotNull - null でないことを確認

assertNotNull 例 
@Test
void testFindUserExists() {
    UserService service = new UserService();
    // ID=1 のユーザーは存在するので null ではないはずだ
    User user = service.findById(1);
    assertNotNull(user, "ユーザーが見つかること");
}

箱で例えると

オブジェクトを「箱」だと考えてください。
assertNull：箱の中身が空っぽか確認する
assertNotNull：箱に何か入っているか確認する

3.5 その他の便利なアサーション

JUnit 5には、他にも便利なアサーションメソッドがあります。

assertSame / assertNotSame

2つのオブジェクトが同じインスタンスかどうかを確認します。 equalsではなく、メモリ上の同一性（==）をチェックします。 

String a = new String("test");
String b = a;  // 同じインスタンスを参照
assertSame(a, b);  // 成功

assertArrayEquals

2つの配列の内容が同じかどうかを確認します。 

int[] expected = {1, 2, 3};
int[] actual = {1, 2, 3};
assertArrayEquals(expected, actual);

assertAll

複数のアサーションをまとめて実行します。 1つ失敗してもすべてのアサーションが実行されるため、問題を一度に把握できます。 

@Test
void testUser() {
    User user = new User("田中", 25);
    assertAll("ユーザー情報",
        () -> assertEquals("田中", user.getName()),
        () -> assertEquals(25, user.getAge()),
        () -> assertNotNull(user.getId())
    );
}

実習 3-1: 文字列操作のテストを書こう

文字列を操作する StringUtil クラスのテストを作成し、様々なアサーションを使ってみましょう。

手順

  1. 1 StringUtil クラスを作成し、isEmpty、toUpperCase、concat メソッドを実装する
  2. 2 StringUtilTest クラスを作成し、各メソッドのテストを書く
  3. 3 assertEquals、assertTrue、assertFalse、assertNull を使い分ける
  4. 4 テストを実行し、すべて成功することを確認する

この章のまとめ

  • アサーションは「〜であるはずだ」と期待値を宣言し、実際の結果と比較するもの
  • assertEqualsは期待値と実際の値が等しいかを確認する（最もよく使う）
  • assertTrue/assertFalseは条件がtrue/falseかを確認する
  • assertNull/assertNotNullはオブジェクトがnullかどうかを確認する
  • アサーションにはメッセージを追加できる（テスト失敗時に分かりやすくなる）

理解度確認クイズ

  1. Q1. assertEquals の第1引数と第2引数には何を指定しますか？

    回答を表示
    第1引数：期待値（expected）、第2引数：実際の値（actual）。 この順番を守ることで、テスト失敗時のエラーメッセージが分かりやすくなります。

  2. Q2. assertTrue と assertFalse の違いは何ですか？

    回答を表示
    assertTrue：条件がtrueであることを確認する
    assertFalse：条件がfalseであることを確認する
    どちらも真偽値を検証しますが、期待する結果が逆になります。

  3. Q3. 次のテストは成功しますか？失敗しますか？

    String text = null;
assertNull(text);
    回答を表示
    成功します。
    text は null なので、assertNull の条件を満たしています。

  4. Q4. assertAll の利点は何ですか？

    回答を表示
    複数のアサーションをまとめて実行し、1つ失敗してもすべてのアサーションが実行されます。 これにより、1回のテスト実行で複数の問題を一度に発見できます。
