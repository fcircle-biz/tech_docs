エラーもテストしよう - 例外テストの書き方

「正しくエラーになるか」をテストする方法を学びます。0で割り算したときにエラーが出ることを確認するなど、異常系のテストの重要性と書き方を解説します。

この章で学ぶこと

  • 例外とは何か、どのような場面で発生するかを理解する
  • assertThrowsを使って例外をテストする方法を習得する
  • 例外のメッセージを検証する方法を学ぶ
  • 正常系と異常系の両方をテストする重要性を理解する

7.1 例外（Exception）とは？

プログラムを実行中に発生する異常な事態を例外（Exception）といいます。

例えば、0で割り算しようとしたり、存在しないファイルを開こうとしたりすると、プログラムはエラーを起こして止まってしまいます。 このエラーが「例外」です。

電車で例えると

電車が正常に走っているのが「正常系」、線路に障害物があって非常停止するのが「例外」です。 電車の安全装置が正しく作動するかテストするように、プログラムも「正しくエラーになるか」をテストする必要があります。

よくある例外の例

ArithmeticException

0で割り算しようとしたときに発生

int result = 10 / 0;

NullPointerException

null（何もない）のオブジェクトにアクセスしようとしたときに発生

String str = null; int len = str.length();

IllegalArgumentException

メソッドに不正な引数が渡されたときに発生

setAge(-5); // 年齢に負の数は不正
flowchart LR A[プログラム実行] --> B{正常?} B -->|正常| C[処理続行] B -->|異常| D[例外発生] D --> E[エラーメッセージ表示] E --> F[プログラム停止]

7.2 なぜ例外をテストするのか？

「エラーが出るのは良くないことでは？」と思うかもしれませんが、適切にエラーを出すことは大切な機能です。

例外テストが重要な理由

  • • 不正な操作を防ぐ - 0で割り算など、してはいけない操作を検知できる
  • • 分かりやすいエラーを出す - 何が問題かユーザーに伝えられる
  • • プログラムの安全性を高める - 異常な状態でも正しく対処できる

正常系と異常系

テストには2種類あります。

正常系テスト

正しい入力で正しい結果が返ることを確認

例: add(2, 3) が 5 を返す

異常系テスト

不正な入力で正しくエラーが出ることを確認

例: divide(10, 0) が例外を投げる

注意

異常系テストを書かないと、プログラムが不正な入力でどう動くか分かりません。 「正常に動く」だけでなく「正しくエラーになる」こともテストしましょう。

7.3 assertThrows - 例外をテストする

JUnitで例外をテストするにはassertThrowsメソッドを使います。

「このコードを実行したら、この例外が発生するはず」という期待を表現します。

基本構文

Java 
assertThrows(例外クラス.class, () -> {
    // 例外が発生するはずのコード
});

具体例：0で割る計算

簡単な電卓クラスで0除算をテストしてみましょう。

Calculator.java - テスト対象のクラス 
public class Calculator {
    public int divide(int a, int b) {
        if (b == 0) {
            // 0で割ろうとしたら例外を投げる
            throw new ArithmeticException("0で割ることはできません");
        }
        return a / b;
    }
}
CalculatorTest.java - 例外テスト 
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    @Test
    void divide_0で割るとArithmeticExceptionが発生() {
        Calculator calc = new Calculator();

        // 10を0で割ろうとすると例外が発生するはず
        assertThrows(ArithmeticException.class, () -> {
            calc.divide(10, 0);
        });
    }
}

コードの読み方

  1. ArithmeticException.class - 発生するはずの例外クラス
  2. () -> { ... } - 例外が発生するはずのコード（ラムダ式）
  3. このテストは、divide(10, 0) が ArithmeticException を投げれば合格
  4. 例外が発生しなかった場合や、違う例外が発生した場合は不合格
flowchart LR A["calc.divide(10, 0)"] --> B{"ArithmeticException
発生?"} B -->|はい| C["テスト合格"] B -->|いいえ| D["テスト失敗"]

7.4 例外メッセージを検証する

例外が発生するだけでなく、エラーメッセージの内容も確認すると、より詳細なテストができます。

例外メッセージを検証するテスト 
@Test
void divide_0で割ると適切なメッセージが表示される() {
    Calculator calc = new Calculator();

    // 例外をキャッチして検証
    Exception exception = assertThrows(
        ArithmeticException.class,
        () -> calc.divide(10, 0)
    );

    // メッセージの内容を確認
    assertEquals("0で割ることはできません", exception.getMessage());
}

ポイント

  • assertThrows は発生した例外オブジェクトを返します
  • exception.getMessage() でエラーメッセージを取得できます
  • メッセージを確認することで、「なぜエラーになったか」が正しく伝わるかテストできます

メッセージの部分一致を確認する

完全一致ではなく、メッセージに特定の文字列が含まれているか確認することもできます。

部分一致でメッセージを検証 
@Test
void divide_0で割るとメッセージに0が含まれる() {
    Calculator calc = new Calculator();

    Exception exception = assertThrows(
        ArithmeticException.class,
        () -> calc.divide(10, 0)
    );

    // メッセージに「0」という文字が含まれているか確認
    assertTrue(exception.getMessage().contains("0"));
}

7.5 複数の異常系をテストする

実際のアプリケーションでは、様々な異常系をテストする必要があります。

例：年齢を設定するクラス

Person.java 
public class Person {
    private int age;

    public void setAge(int age) {
        // 負の数はダメ
        if (age < 0) {
            throw new IllegalArgumentException(
                "年齢は0以上である必要があります"
            );
        }
        // 150歳以上もダメ
        if (age > 150) {
            throw new IllegalArgumentException(
                "年齢は150以下である必要があります"
            );
        }
        this.age = age;
    }

    public int getAge() {
        return age;
    }
}
PersonTest.java - 複数の異常系テスト 
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class PersonTest {

    @Test
    void setAge_負の値で例外が発生() {
        Person person = new Person();

        assertThrows(IllegalArgumentException.class, () -> {
            person.setAge(-1);
        });
    }

    @Test
    void setAge_151以上で例外が発生() {
        Person person = new Person();

        assertThrows(IllegalArgumentException.class, () -> {
            person.setAge(151);
        });
    }

    @Test
    void setAge_正常な値は例外なし() {
        Person person = new Person();

        // これらは例外が発生しないはず（正常系）
        person.setAge(0);   // 問題なし
        person.setAge(30);  // 問題なし
        person.setAge(150); // 問題なし

        assertEquals(150, person.getAge());
    }
}

テストのポイント

  • • 境界値をテストする - 0, -1, 150, 151など、境界付近の値を重点的にテスト
  • • 正常系も忘れずに - 例外が発生すべきでないケースもテスト
  • • テストは1つ1つ独立 - 各テストメソッドで1つの観点を検証
flowchart TB A[setAge呼び出し] --> B{年齢の値} B -->|負の数| C[例外発生] B -->|0-150| D[正常設定] B -->|151以上| E[例外発生] C --> F[IllegalArgumentException] E --> F D --> G[問題なし]

実習：例外テストを書いてみよう

文字列を整数に変換するメソッドの例外テストを書いてみましょう。

課題

以下のStringConverterクラスのtoIntegerメソッドをテストしてください。

StringConverter.java 
public class StringConverter {
    public int toInteger(String str) {
        if (str == null) {
            throw new IllegalArgumentException(
                "nullは変換できません"
            );
        }
        return Integer.parseInt(str);
    }
}

書くべきテスト

  1. nullを渡すとIllegalArgumentExceptionが発生する
  2. 数字でない文字列を渡すとNumberFormatExceptionが発生する
  3. 正しい数字の文字列は正常に変換される（正常系）
解答例を見る
StringConverterTest.java 
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class StringConverterTest {

    @Test
    void toInteger_nullで例外発生() {
        StringConverter converter = new StringConverter();

        assertThrows(IllegalArgumentException.class,
            () -> converter.toInteger(null)
        );
    }

    @Test
    void toInteger_数字以外で例外発生() {
        StringConverter converter = new StringConverter();

        assertThrows(NumberFormatException.class,
            () -> converter.toInteger("abc")
        );
    }

    @Test
    void toInteger_正常な文字列は変換成功() {
        StringConverter converter = new StringConverter();

        assertEquals(123, converter.toInteger("123"));
        assertEquals(-456, converter.toInteger("-456"));
    }
}

この章のまとめ

  • 例外は、プログラム実行中に発生する異常な事態
  • 例外をテストすることで、正しくエラーになるかを確認できる
  • assertThrowsを使って、特定の例外が発生することを検証
  • 例外のメッセージも検証することで、より詳細なテストが可能
  • 正常系と異常系の両方をテストすることが重要

理解度確認クイズ

  1. Q1. 例外テストが重要な理由を3つ挙げてください。

    回答を表示
    1. 不正な操作を防ぐことができる
    2. 分かりやすいエラーメッセージを提供できる
    3. プログラムの安全性を高めることができる

  2. Q2. assertThrowsメソッドの役割は何ですか？

    回答を表示
    指定したコードを実行したときに、期待した例外が発生することを検証します。 例外が発生しない場合や、違う種類の例外が発生した場合はテストが失敗します。

  3. Q3. 正常系テストと異常系テストの違いを説明してください。

    回答を表示

    正常系テスト：正しい入力で正しい結果が返ることを確認するテスト。

    異常系テスト：不正な入力で正しくエラーが出ることを確認するテスト。

  4. Q4. 例外メッセージを検証する意義は何ですか？

    回答を表示
    エラーメッセージの内容を確認することで、「なぜエラーになったか」がユーザーに正しく伝わるかをテストできます。 適切なメッセージがあれば、問題の原因を素早く特定できます。
