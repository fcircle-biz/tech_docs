フォーム処理とバリデーション

フォームからのデータ受け取りと入力検証を学びます。@ModelAttributeによるフォームデータの受け取り、Bean Validationアノテーション（@NotBlank、@Size、@Email、@Min、@Max）、BindingResultを使ったエラー処理、Thymeleafでのエラーメッセージ表示を実装します。

この章で学ぶこと

  • @ModelAttributeを使ったフォームデータのバインディング方法を理解する
  • Bean Validationアノテーションによる入力検証の実装方法を習得する
  • BindingResultでバリデーションエラーを制御する方法を学ぶ
  • Thymeleafテンプレートでエラーメッセージを表示する手法を実装する

7.1 フォーム処理の基礎

Webアプリケーションでは、ユーザーが入力したデータをサーバーで受け取り、処理する必要があります。Spring MVCでは@ModelAttributeを使うことで、HTMLフォームから送信されたデータをJavaオブジェクトに自動的にバインド（紐付け）できます。

従来のサーブレットとの違い

サーブレットではrequest.getParameter("name")で一つずつパラメータを取得していましたが、Spring MVCではフォームクラスを定義するだけで、すべてのフィールドを自動的にバインドできます。

フォームデータのフロー

graph LR A[ユーザー入力
HTMLフォーム] --> B[HTTP POST] B --> C[Controllerメソッド
@ModelAttribute] C --> D[フォームクラス
自動バインド] D --> E[バリデーション
入力検証] E -->|OK| F[ビジネスロジック
Serviceへ] E -->|NG| G[エラー表示
フォーム再表示]

フォームクラスの定義

まず、フォームから送信されるデータを受け取るためのクラスを作成します。フィールド名はHTMLフォームのname属性と一致させます。

UserForm.java 
package com.example.demo.form;

public class UserForm {
    private String username;
    private String email;
    private Integer age;

    // getter/setter（省略可能: Lombokの@Data推奨）
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    public Integer getAge() { return age; }
    public void setAge(Integer age) { this.age = age; }
}

7.2 @ModelAttributeの使い方

Controllerメソッドの引数に@ModelAttributeを付けることで、フォームデータが自動的にJavaオブジェクトにバインドされます。

UserController.java 
@Controller
@RequestMapping("/users")
public class UserController {

    @GetMapping("/register")
    public String showRegisterForm(Model model) {
        model.addAttribute("userForm", new UserForm());
        return "user/register";
    }

    @PostMapping("/register")
    public String registerUser(@ModelAttribute UserForm form, Model model) {
        // フォームデータが自動的にformオブジェクトにバインドされる
        System.out.println("Username: " + form.getUsername());
        System.out.println("Email: " + form.getEmail());
        System.out.println("Age: " + form.getAge());

        model.addAttribute("message", "登録が完了しました");
        return "user/complete";
    }
}

重要ポイント

GETリクエストでは空のフォームオブジェクトをモデルに追加し、POSTリクエストでは@ModelAttributeで自動バインドされたオブジェクトを受け取ります。これがSpring MVCの標準的なフォーム処理パターンです。

7.3 Bean Validationによる入力検証

ユーザー入力はサーバー側で必ず検証する必要があります。Spring BootではBean Validation（Jakarta Bean Validation）をサポートしており、アノテーションを使って宣言的に検証ルールを定義できます。

依存関係の追加

pom.xmlに以下の依存関係を追加します（Spring Boot Starterを使っている場合は通常含まれています）。

pom.xml 
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>

主要なバリデーションアノテーション

アノテーション 用途 使用例
@NotBlank null、空文字、空白のみを許可しない ユーザー名、メールアドレス
@NotNull nullを許可しない 年齢、チェックボックス
@Size(min=, max=) 文字列長、コレクション要素数の範囲 パスワード（8〜20文字）
@Email メールアドレス形式をチェック メールアドレス
@Min(value), @Max(value) 数値の最小・最大値 年齢（0〜150）
@Pattern(regexp=) 正規表現パターンマッチング 電話番号、郵便番号

フォームクラスにバリデーション追加

UserForm.java（バリデーション追加版） 
package com.example.demo.form;

import jakarta.validation.constraints.*;

public class UserForm {

    @NotBlank(message = "ユーザー名は必須です")
    @Size(min = 3, max = 20, message = "ユーザー名は3〜20文字で入力してください")
    private String username;

    @NotBlank(message = "メールアドレスは必須です")
    @Email(message = "メールアドレスの形式が正しくありません")
    private String email;

    @NotNull(message = "年齢は必須です")
    @Min(value = 0, message = "年齢は0以上で入力してください")
    @Max(value = 150, message = "年齢は150以下で入力してください")
    private Integer age;

    // getter/setter
}

7.4 BindingResultでエラー処理

@ValidアノテーションとBindingResultを組み合わせることで、バリデーションエラーを制御できます。

UserController.java（エラー処理追加版） 
@Controller
@RequestMapping("/users")
public class UserController {

    @PostMapping("/register")
    public String registerUser(
            @Valid @ModelAttribute UserForm form,
            BindingResult result,
            Model model) {

        // バリデーションエラーがある場合
        if (result.hasErrors()) {
            // エラーがあればフォーム再表示
            return "user/register";
        }

        // エラーがなければ処理続行
        // ここでServiceを呼び出してDB保存など
        model.addAttribute("message", "登録が完了しました");
        return "user/complete";
    }
}

注意事項

BindingResultは必ず@ModelAttributeの直後に配置してください。順番が違うとバリデーションエラーが発生します。

7.5 Thymeleafでエラーメッセージ表示

Thymeleafではフォームバインディングとエラー表示のための専用属性が用意されています。

register.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>ユーザー登録</title>
</head>
<body>
    <h1>ユーザー登録</h1>
    <form th:action="@{/users/register}" th:object="${userForm}" method="post">
        <div>
            <label>ユーザー名:</label>
            <input type="text" th:field="*{username}" />
            <!-- エラーメッセージ表示 -->
            <span th:if="${#fields.hasErrors('username')}"
                  th:errors="*{username}"
                  style="color: red;"></span>
        </div>

        <div>
            <label>メールアドレス:</label>
            <input type="email" th:field="*{email}" />
            <span th:if="${#fields.hasErrors('email')}"
                  th:errors="*{email}"
                  style="color: red;"></span>
        </div>

        <div>
            <label>年齢:</label>
            <input type="number" th:field="*{age}" />
            <span th:if="${#fields.hasErrors('age')}"
                  th:errors="*{age}"
                  style="color: red;"></span>
        </div>

        <button type="submit">登録</button>
    </form>
</body>
</html>

Thymeleafのフォーム関連属性

属性 用途
th:object フォーム全体にバインドするオブジェクトを指定
th:field フィールドをバインド（id、name、valueを自動生成）
th:errors 指定フィールドのエラーメッセージを表示
#fields.hasErrors() エラーの有無をチェック

7.6 カスタムバリデーション

標準のアノテーションで対応できない複雑な検証ルールは、独自のバリデーションアノテーションを作成できます。

@ValidUsername.java（カスタムアノテーション） 
@Target({ElementType.FIELD})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = UsernameValidator.class)
public @interface ValidUsername {
    String message() default "ユーザー名に使用できない文字が含まれています";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}

// バリデーターの実装
public class UsernameValidator
        implements ConstraintValidator<ValidUsername, String> {

    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null) return true; // nullチェックは@NotBlankに任せる
        // 英数字とアンダースコアのみ許可
        return value.matches("^[a-zA-Z0-9_]+$");
    }
}

理解度確認クイズ

  1. Q1. フォームデータをJavaオブジェクトに自動バインドするためのアノテーションは何ですか？

    回答を表示

    @ModelAttribute
    Controllerメソッドの引数に@ModelAttributeを付けることで、HTTPリクエストパラメータが自動的にJavaオブジェクトのフィールドにバインドされます。

  2. Q2. @NotBlankと@NotNullの違いを説明してください。

    回答を表示

    @NotBlank: null、空文字、空白のみを許可しない（文字列専用）
    @NotNull: nullのみ許可しない（すべての型で使用可能）
    文字列の場合、通常は@NotBlankを使用します。

  3. Q3. BindingResultをなぜ@ModelAttributeの直後に配置する必要がありますか？

    回答を表示

    理由: BindingResultは直前の@ModelAttributeに対するバリデーション結果を保持するためです。順番が違うと、どのオブジェクトのバリデーション結果かを識別できず、エラーが発生します。

第7章のまとめ

  • @ModelAttributeを使うことで、フォームデータを自動的にJavaオブジェクトにバインドできます。
  • Bean Validationアノテーション（@NotBlank、@Size、@Email、@Min、@Max）で宣言的に検証ルールを定義できます。
  • @Validとバインディングは、エラー制御のために必ず@ModelAttributeの直後に配置します。
  • Thymeleafのth:object、th:field、th:errorsを使ってフォームとエラーメッセージを実装できます。
