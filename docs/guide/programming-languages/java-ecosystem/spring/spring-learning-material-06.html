Thymeleafテンプレートエンジン

モダンなビューテンプレートエンジンであるThymeleafの使い方を学びます。th:text、th:each、th:if、th:hrefなど主要な属性の使い方、フォームの作成とデータバインディング（th:object、th:field）、レイアウトの共通化、JSPとの比較を通じてThymeleafの利点を理解します。

この章で学ぶこと

  • Thymeleafの基本属性（th:text、th:each、th:if、th:href）を使いこなす
  • th:objectとth:fieldを使ったフォームデータバインディングを実装する
  • レイアウトの共通化によりビューのメンテナンス性を向上させる
  • JSPとの比較を通じてThymeleafの利点を理解する

6.1 Thymeleafとは

Thymeleafは、Spring Bootで推奨されているモダンなサーバーサイドテンプレートエンジンです。HTMLファイルをそのまま開いても正常に表示される「Natural Templates」の思想で設計されており、デザイナーとプログラマの協業を容易にします。

テンプレートエンジンとは

HTMLテンプレートに動的データを埋め込んでHTMLを生成する仕組みです。Thymeleafは、通常のHTMLファイルに専用の属性を追加する形でロジックを記述します。

Thymeleaf vs JSP

Thymeleafの3つの利点

Natural Templates

HTMLとして直接ブラウザで開けるため、デザイン確認が容易

Spring統合

Spring MVCとシームレスに連携し、フォームバインディングが簡単

XSS対策

デフォルトでHTMLエスケープされるため、セキュアなHTMLを生成

6.2 基本属性の使い方

Thymeleafの主要な属性を学び、動的なHTMLを生成する方法を理解しましょう。

th:text - テキスト出力

変数の値をHTMLテキストとして安全に出力します。HTMLエスケープが自動適用されます。

UserController.java 
// コントローラー
@GetMapping("/user")
public String showUser(Model model) {
    model.addAttribute("username", "太郎");
    model.addAttribute("email", "taro@example.com");
    return "user";
}
user.html 
<!-- Thymeleafテンプレート -->
<h1 th:text="${username}">サンプル太郎</h1>
<p th:text="${email}">sample@example.com</p>

<!-- 実行時には以下に変換される -->
<h1>太郎</h1>
<p>taro@example.com</p>

th:each - 繰り返し処理

リストやコレクションの要素をループ処理し、HTMLを繰り返し生成します。

ProductController.java 
// リストをModelに追加
@GetMapping("/products")
public String listProducts(Model model) {
    List<String> products = List.of("ノートPC", "マウス", "キーボード");
    model.addAttribute("products", products);
    return "products";
}
products.html 
<ul>
    <li th:each="product : ${products}" th:text="${product}">サンプル商品</li>
</ul>

<!-- 実行時には以下に変換される -->
<ul>
    <li>ノートPC</li>
    <li>マウス</li>
    <li>キーボード</li>
</ul>

th:if / th:unless - 条件分岐

条件に応じて要素の表示/非表示を切り替えます。

condition.html 
<!-- ログイン済みの場合のみ表示 -->
<p th:if="${isLoggedIn}">ようこそ、<span th:text="${username}"></span>さん</p>

<!-- ログインしていない場合のみ表示 -->
<p th:unless="${isLoggedIn}">ログインしてください</p>

th:href - リンクURL生成

動的にURLを生成し、パラメータを安全にエンコードします。

link.html 
<!-- 静的URL -->
<a th:href="@{/home}">ホーム</a>

<!-- パラメータ付きURL -->
<a th:href="@{/user(id=${userId})}">ユーザー詳細</a>

<!-- 実行時: /user?id=123 に変換される -->

重要ポイント

Thymeleafの属性はすべて「th:」で始まります。HTMLタグのデフォルト値を記述しておくことで、ブラウザで直接開いてもレイアウト確認ができます。

6.3 フォームデータバインディング

th:objectとth:fieldを使って、フォームとJavaオブジェクトを簡単にバインディングできます。

フォームオブジェクト（DTO）の定義

UserForm.java 
// フォーム用クラス
public class UserForm {
    private String username;
    private String email;
    private Integer age;

    // ゲッター・セッター省略
}

コントローラーでフォームを準備

UserController.java 
@Controller
public class UserController {

    // フォーム表示
    @GetMapping("/user/new")
    public String showForm(Model model) {
        model.addAttribute("userForm", new UserForm());
        return "user-form";
    }

    // フォーム送信処理
    @PostMapping("/user/create")
    public String createUser(@ModelAttribute UserForm userForm) {
        // userFormに入力値が自動バインディングされる
        System.out.println(userForm.getUsername());
        return "redirect:/user/list";
    }
}

Thymeleafテンプレート

user-form.html 
<form th:action="@{/user/create}" th:object="${userForm}" method="post">
    <div>
        <label>ユーザー名</label>
        <input type="text" th:field="*{username}" />
    </div>
    <div>
        <label>メールアドレス</label>
        <input type="email" th:field="*{email}" />
    </div>
    <div>
        <label>年齢</label>
        <input type="number" th:field="*{age}" />
    </div>
    <button type="submit">登録</button>
</form>

th:fieldの便利な機能

th:fieldは、name属性、id属性、value属性を自動生成します。例えば、th:field="*{username}"はname="username"、id="username"、value="${userForm.username}"に展開されます。

6.4 レイアウトの共通化

Thymeleaf Layout Dialectを使うと、ヘッダーやフッターなどの共通部分を別ファイルに分離し、メンテナンス性を向上できます。

依存関係の追加

pom.xml 
<dependency>
    <groupId>nz.net.ultraq.thymeleaf</groupId>
    <artifactId>thymeleaf-layout-dialect</artifactId>
</dependency>

共通レイアウトファイル

layout.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org"
      xmlns:layout="http://www.ultraq.net.nz/thymeleaf/layout">
<head>
    <title layout:title-pattern="$CONTENT_TITLE - MyApp">MyApp</title>
</head>
<body>
    <header>共通ヘッダー</header>
    <main layout:fragment="content"></main>
    <footer>共通フッター</footer>
</body>
</html>

個別ページ

home.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org"
      xmlns:layout="http://www.ultraq.net.nz/thymeleaf/layout"
      layout:decorate="~{layout}">
<head>
    <title>ホーム</title>
</head>
<body>
    <main layout:fragment="content">
        <h1>ようこそ</h1>
        <p>ここはホームページです</p>
    </main>
</body>
</html>

レイアウト共通化のメリット

ヘッダー・フッターの変更が1箇所で済み、各ページは固有のコンテンツのみを記述すればよくなります。DRY原則（Don't Repeat Yourself）を実現できます。

6.5 JSPとの比較

従来のJSPと比較することで、Thymeleafの利点をより深く理解しましょう。

観点 JSP Thymeleaf
ファイル形式 .jsp（特殊形式） .html（標準HTML）
プレビュー サーバー起動必須 ブラウザで直接開ける
変数出力 ${username} th:text="${username}"
繰り返し <c:forEach>タグ th:each属性
条件分岐 <c:if>タグ th:if属性
Spring統合 spring:formタグ必要 th:object/th:fieldで簡単

同じ処理の記述例

JSP版 
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
<c:forEach items="${products}" var="product">
    <p>${product.name}</p>
</c:forEach>
Thymeleaf版 
<p th:each="product : ${products}" th:text="${product.name}">サンプル商品</p>

Spring BootではThymeleafが推奨される理由

Spring Bootの組込Tomcatでは、JSPのコンパイルに制約があります。一方、Thymeleafは組込サーバーでも問題なく動作し、開発効率が高いため、Spring Boot公式でも推奨されています。

実習 6-1: 商品一覧ページをThymeleafで作成

Thymeleafを使って商品一覧を表示するページを作成しましょう。

手順

  1. 1 Productクラスを作成（id、name、priceフィールド）
  2. 2 ProductControllerで商品リストをModelに追加
  3. 3 product-list.htmlでth:eachを使って商品をテーブル表示
  4. 4 th:ifを使って在庫あり/なしを条件分岐で表示

理解度確認クイズ

  1. Q1. Thymeleafで変数の値をテキストとして出力する属性は何ですか？

    回答を表示

    th:text
    th:text="${変数名}"の形式で記述します。HTMLエスケープが自動適用され、XSS攻撃を防ぎます。

  2. Q2. フォームオブジェクトとフィールドをバインディングするために使う属性の組み合わせを答えてください。

    回答を表示

    th:objectとth:field
    <form th:object="${userForm}">でフォーム全体にオブジェクトを指定し、<input th:field="*{username}" />で各フィールドをバインディングします。

  3. Q3. ThymeleafがJSPより優れている点を3つ挙げてください。

    回答を表示

    1. Natural Templates: ブラウザで直接HTMLとして開ける
    2. Spring Boot統合: 組込サーバーで問題なく動作
    3. HTMLベース: 標準HTMLなのでデザイナーとの協業が容易

第6章のまとめ

  • Thymeleafは標準HTMLベースのテンプレートエンジンで、Natural Templatesの思想で設計されています。
  • th:text、th:each、th:if、th:hrefなどの基本属性でHTMLを動的生成できます。
  • th:objectとth:fieldを使えば、フォームとJavaオブジェクトを簡単にバインディングできます。
  • Thymeleaf Layout Dialectでヘッダー・フッターを共通化し、メンテナンス性を向上できます。
  • JSPと比較し、ブラウザでのプレビュー可能性とSpring Boot統合の優位性を持ちます。
