第5章

Spring MVCの基礎

Spring MVCを使ったWebアプリケーション開発の基礎を学びます。DispatcherServletの役割、@Controllerと@RequestMappingによるルーティング、@GetMappingと@PostMappingの使い分け、Modelオブジェクトを使ったビューへのデータ渡しを理解し、JSP/サーブレットとの違いを実感します。

この章で学ぶこと

  • Spring MVCの全体アーキテクチャとDispatcherServletの役割を理解する
  • @Controllerと@RequestMappingを使ったルーティングの仕組みを習得する
  • @GetMappingと@PostMappingの適切な使い分けを学ぶ
  • Modelオブジェクトを使ってコントローラーからビューへデータを渡す方法を実装する

5.1 Spring MVCとは

Spring MVCは、Spring Frameworkが提供するWebアプリケーション開発のためのMVC（Model-View-Controller）フレームワークです。HTTPリクエストの受付からレスポンスの返却までを、MVCパターンに沿って整然と実装できます。

MVCパターンとは

アプリケーションをModel（データ）、View（表示）、Controller（制御）の3つに分離する設計パターンです。責務を明確に分けることで、保守性とテスト容易性が向上します。

Spring MVCのリクエスト処理フロー

sequenceDiagram participant Browser as ブラウザ participant DS as DispatcherServlet participant HC as HandlerMapping participant Controller as Controller participant Service as Service層 participant View as View
(HTML/JSP) Browser->>DS: HTTPリクエスト
(GET /users) DS->>HC: 適切なControllerを検索 HC-->>DS: Controllerメソッドを返却 DS->>Controller: メソッド実行 Controller->>Service: ビジネスロジック呼び出し Service-->>Controller: データ返却 Controller->>DS: ModelとViewNameを返却 DS->>View: Viewを解決し、
Modelデータを渡す View-->>DS: HTMLレンダリング DS-->>Browser: HTTPレスポンス

JSP/サーブレット開発では、web.xmlでのマッピング設定やServletクラスの実装が必要でしたが、Spring MVCではアノテーションベースで簡潔に記述できます。

5.2 DispatcherServletの役割

DispatcherServletは、Spring MVCのフロントコントローラーとして、すべてのHTTPリクエストを一元的に受け取り、適切なControllerに処理を振り分けます。

DispatcherServletの中核機能

graph LR Request[HTTPリクエスト] --> DS[DispatcherServlet] DS --> HM[HandlerMapping
URLとControllerの
マッピング] DS --> HA[HandlerAdapter
Controller実行] DS --> VR[ViewResolver
View名からHTMLへ変換] DS --> Response[HTTPレスポンス]

重要ポイント

Spring Bootを使用すると、DispatcherServletの設定は自動で行われます。開発者はControllerの実装に集中できます。

5.3 @Controllerと@RequestMapping

@ControllerアノテーションでクラスをSpring MVCのコントローラーとして登録し、@RequestMappingでURLとメソッドをマッピングします。

基本的なControllerの実装

HelloController.java 
package com.example.demo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller  // Controllerとして認識
public class HelloController {

    @RequestMapping("/hello")  // /helloにアクセスされたら実行
    public String hello() {
        return "hello";  // hello.htmlを返す
    }
}

ビュー名の解決

メソッドが"hello"を返すと、ViewResolverがsrc/main/resources/templates/hello.htmlを自動的に検索します。

5.4 @GetMappingと@PostMapping

HTTPメソッド（GET、POST）を明示的に指定するため、@GetMappingと@PostMappingを使用します。

アノテーション HTTPメソッド 用途
@GetMapping GET データ取得・ページ表示
@PostMapping POST データ送信・登録・更新

GET/POSTの使い分け例

UserController.java 
@Controller
@RequestMapping("/users")
public class UserController {

    // ユーザー一覧ページを表示（GET）
    @GetMapping
    public String list() {
        return "users/list";
    }

    // ユーザー登録フォームを表示（GET）
    @GetMapping("/new")
    public String showForm() {
        return "users/form";
    }

    // ユーザー登録処理（POST）
    @PostMapping
    public String create() {
        // 登録処理
        return "redirect:/users";
    }
}

5.5 Modelを使ったデータ渡し

コントローラーからビュー（HTML）へデータを渡すには、Modelオブジェクトを使用します。

Modelへのデータ追加

GreetingController.java 
@Controller
public class GreetingController {

    @GetMapping("/greeting")
    public String greeting(Model model) {
        // Modelに属性を追加
        model.addAttribute("name", "田中太郎");
        model.addAttribute("message", "こんにちは");
        return "greeting";  // greeting.htmlへ
    }
}

ビュー側での受け取り（Thymeleaf）

greeting.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>挨拶ページ</title>
</head>
<body>
    <h1 th:text="${message}"></h1>
    <p th:text="${name} + 'さん'"></p>
</body>
</html>

重要ポイント

Modelに追加した属性は、Thymeleafテンプレート内で${属性名}の形式でアクセスできます。

実習: Spring MVCで簡単なWebアプリケーションを作ろう

目標

現在時刻を表示するWebアプリケーションを作成します。

手順

  1. 1. Controllerクラスの作成
    ClockController.java 
    package com.example.demo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

@Controller
public class ClockController {

    @GetMapping("/clock")
    public String showClock(Model model) {
        // 現在時刻を取得
        LocalDateTime now = LocalDateTime.now();
        String formatted = now.format(
            DateTimeFormatter.ofPattern("yyyy年MM月dd日 HH:mm:ss")
        );

        // Modelに追加
        model.addAttribute("currentTime", formatted);
        return "clock";
    }
}
  2. 2. ビューテンプレートの作成

    src/main/resources/templates/clock.htmlを作成します。

    clock.html 
    <!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>時計アプリ</title>
</head>
<body>
    <h1>現在時刻</h1>
    <p th:text="${currentTime}"></p>
</body>
</html>
  3. 3. アプリケーションを起動してアクセス

    ブラウザでhttp://localhost:8080/clockにアクセスします。

5.6 サーブレットとの比較

従来のサーブレット開発と比較し、Spring MVCの利点を確認しましょう。

従来のサーブレット実装

HelloServlet.java（サーブレット） 
@WebServlet("/hello")
public class HelloServlet extends HttpServlet {
    protected void doGet(HttpServletRequest req,
                        HttpServletResponse res)
            throws ServletException, IOException {

        req.setAttribute("name", "田中太郎");
        RequestDispatcher dispatcher =
            req.getRequestDispatcher("/WEB-INF/hello.jsp");
        dispatcher.forward(req, res);
    }
}

Spring MVCの実装

HelloController.java（Spring MVC） 
@Controller
public class HelloController {

    @GetMapping("/hello")
    public String hello(Model model) {
        model.addAttribute("name", "田中太郎");
        return "hello";
    }
}

Spring MVCの優位性

RequestDispatcherの手動操作が不要で、アノテーションとModelだけで簡潔に記述できます。テストも容易です。

理解度確認クイズ

  1. Q1. Spring MVCですべてのHTTPリクエストを一元管理し、適切なControllerに処理を振り分ける役割を持つコンポーネントは何ですか？

    回答を表示

    DispatcherServlet
    DispatcherServletはフロントコントローラーとして、リクエストのルーティング、Controllerの実行、Viewの解決を担当します。

  2. Q2. GETリクエストとPOSTリクエストの使い分けについて説明してください。

    回答を表示

    GET：データ取得やページ表示に使用（べき等性あり）
    POST：データの登録・更新・削除に使用（べき等性なし）
    GETは安全な操作、POSTは状態を変更する操作に使い分けます。

  3. Q3. ControllerからViewへデータを渡すために使用するオブジェクトは何ですか？

    回答を表示

    Modelオブジェクト
    ModelにaddAttribute()でデータを追加すると、ビューテンプレートで${属性名}として参照できます。

第5章のまとめ

  • Spring MVCは、DispatcherServletがフロントコントローラーとして機能し、リクエストを適切なControllerに振り分けます。
  • @Controllerと@RequestMapping（@GetMapping/@PostMapping）で、アノテーションベースの簡潔なルーティングを実現できます。
  • GETはデータ取得、POSTはデータ送信・変更操作に使い分けます。
  • Modelオブジェクトを使って、ControllerからViewへデータを渡すことができます。
