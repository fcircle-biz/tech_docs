第8章

Spring Data JDBC入門

Spring Data JDBCを使ったデータアクセス層の実装を学びます。従来のJDBCコードとの比較、application.ymlでのデータソース設定、エンティティクラスの定義（@Table、@Id）、CrudRepositoryを継承したリポジトリの作成、基本的なCRUD操作の実装方法を理解します。

この章で学ぶこと

  • Spring Data JDBCとは何か、従来のJDBCとの違いを理解する
  • application.ymlでデータソース（DB接続情報）を設定できる
  • エンティティクラスに@Tableと@Idアノテーションを付けてテーブルとマッピングする
  • CrudRepositoryを継承したリポジトリで基本的なCRUD操作を実装する

8.1 Spring Data JDBCとは

Spring Data JDBCは、データベースアクセスを簡素化するSpring Dataプロジェクトの一部です。従来のJDBCでは、Connection、PreparedStatement、ResultSetの手動管理が必要でしたが、Spring Data JDBCはこれらのボイラープレートコードを削減し、開発者がビジネスロジックに集中できるようにします。

Spring Data JDBCとSpring Data JPAの違い

Spring Data JPAはORMとして動作し、オブジェクトとテーブルの複雑なマッピングを自動化します。Spring Data JDBCはよりシンプルで、JDBCに近い直接的なデータアクセスを提供します。本ガイドでは、JDBCの知識を活かせるSpring Data JDBCを採用しています。

従来のJDBC vs Spring Data JDBC

Spring Data JDBCは、リポジトリパターンを採用しており、データアクセス層をインターフェースとして定義するだけで、基本的なCRUD操作（Create、Read、Update、Delete）が自動的に提供されます。

8.2 データソースの設定（application.yml）

従来のJDBCでは、DriverManager.getConnection()でDB接続情報をコード内に記述していましたが、Spring Bootではapplication.yml（またはapplication.properties）に外部化します。

src/main/resources/application.yml 
# データソース設定
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/mydb
    username: postgres
    password: password
    driver-class-name: org.postgresql.Driver

# SQL実行ログの表示（開発時のみ）
logging:
  level:
    org.springframework.jdbc.core: DEBUG

重要ポイント

設定を外部化することで、環境ごと（開発、テスト、本番）に異なるDB接続先を簡単に切り替えられます。パスワードなどの機密情報も環境変数で管理できます。

依存関係の追加（pom.xml）

Spring Data JDBCとPostgreSQLドライバーの依存関係をpom.xmlに追加します。

pom.xml（依存関係の追加） 
<dependencies>
    <!-- Spring Data JDBC -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jdbc</artifactId>
    </dependency>

    <!-- PostgreSQL JDBC ドライバー -->
    <dependency>
        <groupId>org.postgresql</groupId>
        <artifactId>postgresql</artifactId>
        <scope>runtime</scope>
    </dependency>
</dependencies>

8.3 エンティティクラスの定義

データベースのテーブルに対応するJavaクラスをエンティティクラスと呼びます。Spring Data JDBCでは、@Tableと@Idアノテーションを使ってテーブルとのマッピングを定義します。

テーブル定義（PostgreSQL）

まず、以下のようなusersテーブルがあると仮定します。

users テーブルのDDL 
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) NOT NULL UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

エンティティクラスの実装

User.java（エンティティクラス） 
package com.example.demo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.relational.core.mapping.Table;
import java.time.LocalDateTime;

@Table("users") // テーブル名を指定
public class User {

    @Id // 主キーを指定
    private Long id;

    private String name;
    private String email;
    private LocalDateTime createdAt;

    // コンストラクタ、ゲッター、セッター（省略可能）
}

アノテーションの役割

  • @Table("users"): このクラスがusersテーブルに対応することを示す
  • @Id: idフィールドが主キーであることを示す（自動採番される）

8.4 リポジトリの作成（CrudRepository）

リポジトリは、データアクセス層のインターフェースです。CrudRepositoryを継承するだけで、基本的なCRUD操作のメソッドが自動的に提供されます。

UserRepository.java 
package com.example.demo.repository;

import com.example.demo.entity.User;
import org.springframework.data.repository.CrudRepository;

// CrudRepositoryを継承するだけで基本CRUD操作が使える
public interface UserRepository extends CrudRepository<User, Long> {
    // メソッド定義は不要（自動提供される）
}

CrudRepositoryが提供するメソッド

メソッド 説明
save(S entity) エンティティを保存（INSERT/UPDATE）
findById(ID id) IDで1件取得（SELECT）
findAll() 全件取得（SELECT）
deleteById(ID id) IDで1件削除（DELETE）
count() レコード数を取得（COUNT）

重要ポイント

インターフェースを定義するだけで、Spring Data JDBCがプロキシオブジェクトを生成し、SQL実行をすべて自動化してくれます。開発者はSQLを書く必要がありません。

8.5 基本的なCRUD操作の実装

リポジトリをサービス層で利用して、基本的なCRUD操作を実装します。

1. Create（データ登録）

UserService.java（登録処理） 
@Service
public class UserService {
    private final UserRepository userRepository;

    @Autowired
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    // ユーザー登録
    public User createUser(String name, String email) {
        User user = new User();
        user.setName(name);
        user.setEmail(email);
        return userRepository.save(user); // INSERT実行
    }
}

2. Read（データ取得）

UserService.java（取得処理） 
// ID指定で1件取得
public User findUserById(Long id) {
    return userRepository.findById(id)
        .orElseThrow(() -> new RuntimeException("User not found"));
}

// 全件取得
public List<User> findAllUsers() {
    Iterable<User> users = userRepository.findAll();
    return StreamSupport.stream(users.spliterator(), false)
                        .collect(Collectors.toList());
}

3. Update（データ更新）

UserService.java（更新処理） 
// ユーザー情報更新
public User updateUser(Long id, String name, String email) {
    User user = findUserById(id); // 既存データ取得
    user.setName(name);
    user.setEmail(email);
    return userRepository.save(user); // UPDATE実行
}

4. Delete（データ削除）

UserService.java（削除処理） 
// ユーザー削除
public void deleteUser(Long id) {
    userRepository.deleteById(id); // DELETE実行
}

動作確認

以下のCommandLineRunnerで動作確認してください。

DemoApplication.java

public class DemoApplication implements CommandLineRunner {

理解度確認クイズ

  1. Q1. エンティティクラスで主キーを示すアノテーションは何ですか？

    回答を表示

    @Id
    主キーフィールドに@Idアノテーションを付けることで、Spring Data JDBCがそのフィールドを主キーとして認識します。

  2. Q2. CrudRepositoryを継承したインターフェースで、新規データを保存するメソッド名は何ですか？

    回答を表示

    save(S entity)
    saveメソッドは、エンティティのIDがnullの場合はINSERT、すでに存在する場合はUPDATEを自動実行します。

  3. Q3. 従来のJDBCと比較して、Spring Data JDBCの主な利点を2つ挙げてください。

    回答を表示

    1. ボイラープレートコードの削減：Connection、PreparedStatement、ResultSetの手動管理が不要
    2. リポジトリパターン：インターフェース定義だけで基本CRUD操作が自動提供される

第8章のまとめ

  • Spring Data JDBCは、従来のJDBCの手動管理を自動化し、リポジトリパターンでデータアクセス層を実装します。
  • application.ymlでデータソース設定を外部化し、環境ごとの接続先を柔軟に管理できます。
  • エンティティクラスに@Tableと@Idアノテーションを付けてテーブルとマッピングします。
  • CrudRepositoryを継承したリポジトリで、save、findById、findAll、deleteByIdなどの基本CRUD操作が自動提供されます。
