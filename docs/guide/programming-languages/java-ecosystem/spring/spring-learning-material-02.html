開発環境の構築とSpring Boot入門

Spring Boot開発環境を構築し、最初のアプリケーションを動かします。Spring Tool Suite（STS）のセットアップ、Spring Initializrを使ったプロジェクト生成、pom.xmlの構造理解、Hello Worldアプリケーションの作成と実行までを体験します。

この章で学ぶこと

  • Spring Boot開発に必要なJDK、IDE（Spring Tool Suite）の準備と設定を行う
  • Spring Initializrを使ったプロジェクト生成の手順と依存関係の選択方法を理解する
  • pom.xml（Mavenビルド設定ファイル）の構造と役割を理解する
  • 最初のSpring Bootアプリケーション（Hello World）を作成し、組込Tomcatで動作確認する

2.1 JDKのインストールと確認

Spring Boot開発を始めるには、まずJDK（Java Development Kit）をインストールする必要があります。Spring Boot 3.x系ではJDK 17以上が必須となります。

JDKのバージョン選択

LTS（Long Term Support）版であるJDK 17または21の使用を推奨します。Oracle JDK、Eclipse Temurin（旧AdoptOpenJDK）、Amazon Correttoなど、ディストリビューションはどれでも構いません。

JDKインストール確認手順

ターミナル（WindowsではコマンドプロンプトまたはPowerShell）を開き、以下のコマンドでJavaバージョンを確認します。

ターミナル 
# Javaバージョン確認
java -version

# 期待される出力例（JDK 17の場合）
# openjdk version "17.0.9" 2023-10-17
# OpenJDK Runtime Environment Temurin-17.0.9+9 (build 17.0.9+9)
# OpenJDK 64-Bit Server VM Temurin-17.0.9+9 (build 17.0.9+9, mixed mode)

バージョン番号が17以上であれば、次のステップに進めます。インストールされていない場合は、Eclipse Temurinなどからダウンロードしてインストールしてください。

2.2 Spring Tool Suite（STS）のセットアップ

Spring Tool Suite（STS）は、Eclipseベースの統合開発環境で、Spring Boot開発に最適化されたツールセットを提供します。IntelliJ IDEA（Ultimate版またはCommunity版）でも開発可能ですが、本ガイドではSTSを使用します。

STSのダウンロードとインストール

  1. 1. ダウンロード
    Spring Tools公式サイトから、使用OSに対応したSTSをダウンロードします。
  2. 2. インストール
    ダウンロードしたファイルを解凍し、任意のフォルダに配置します（インストーラーは不要）。
  3. 3. 起動
    SpringToolSuite4.exe（Windows）またはSpringToolSuite4（Mac/Linux）を実行します。
  4. 4. ワークスペース設定
    初回起動時にワークスペース（プロジェクトを保存するフォルダ）を指定します。

重要ポイント

STSにはSpring Boot Dashboard、依存関係の自動補完、application.ymlのシンタックスハイライトなど、Spring開発に便利な機能が最初から組み込まれています。

2.3 Spring Initializrでプロジェクト生成

Spring Initializrは、Spring Bootプロジェクトの雛形を自動生成するWebツールです。依存関係の選択やビルドツールの設定を視覚的に行えます。

実習: 最初のSpring Bootプロジェクトを生成する

  1. 1. Spring Initializrにアクセス
    https://start.spring.io/ をブラウザで開きます。
  2. 2. プロジェクト設定を入力
    • • Project: Maven
    • • Language: Java
    • • Spring Boot: 3.2.x（最新の安定版）
    • • Group: com.example
    • • Artifact: demo
    • • Package name: com.example.demo
    • • Java: 17
  3. 3. 依存関係を追加
    「ADD DEPENDENCIES」ボタンをクリックし、以下を追加：
    • • Spring Web - RESTful API、Spring MVCを含む
    • • Spring Boot DevTools - ホットリロード機能
  4. 4. GENERATEボタンをクリック
    zipファイルがダウンロードされます。
  5. 5. プロジェクトをインポート
    zipファイルを解凍し、STSで「File → Import → Existing Maven Projects」から開きます。

Spring Initializrの役割

Spring Initializrは、pom.xml（Maven）やbuild.gradle（Gradle）の依存関係定義、ディレクトリ構造、メインクラスを自動生成します。これにより、開発者は設定ファイルの記述に時間を取られず、すぐにコーディングを始められます。

2.4 プロジェクト構造とpom.xmlの理解

Spring Initializrで生成されたプロジェクトは、以下の構造を持ちます。

プロジェクト構造

demo/
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/example/demo/
│   │   │       └── DemoApplication.java  ← メインクラス
│   │   └── resources/
│   │       ├── application.properties    ← 設定ファイル
│   │       ├── static/                   ← 静的ファイル（CSS、JS）
│   │       └── templates/                ← Thymeleafテンプレート
│   └── test/                             ← テストコード
│       └── java/
│           └── com/example/demo/
│               └── DemoApplicationTests.java
├── pom.xml                               ← Mavenビルド設定
└── mvnw, mvnw.cmd                        ← Maven Wrapper（Maven不要）

pom.xmlの構造

pom.xmlは、Mavenプロジェクトの設定ファイルであり、依存関係、ビルドプラグイン、プロジェクトのメタ情報を記述します。

pom.xml（抜粋） 
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0">
    <!-- Spring Bootの親POM（バージョン管理） -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.2.0</version>
    </parent>

    <!-- プロジェクト情報 -->
    <groupId>com.example</groupId>
    <artifactId>demo</artifactId>
    <version>0.0.1-SNAPSHOT</version>

    <!-- 依存関係（Spring Webを使用） -->
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
    </dependencies>
</project>

spring-boot-starter-*の意味

Spring Bootの「Starter」は、関連するライブラリをまとめたパッケージです。例えばspring-boot-starter-webには、Spring MVC、組込Tomcat、JSON変換ライブラリ（Jackson）などが含まれます。

2.5 Hello Worldアプリケーションの作成

最初のSpring Bootアプリケーションとして、Webブラウザに「Hello, Spring Boot!」と表示するシンプルなコントローラーを作成します。

実習: HelloControllerを作成する

com.example.demoパッケージにHelloController.javaを作成します。

HelloController.java 
package com.example.demo;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

// @RestController: RESTful APIのコントローラーであることを示す
@RestController
public class HelloController {

    // @GetMapping: HTTP GETリクエストを処理する
    @GetMapping("/")
    public String hello() {
        return "Hello, Spring Boot!"; // 文字列をそのままレスポンスとして返す
    }
}

アプリケーションの起動

STSのSpring Boot Dashboardまたはメインクラス（DemoApplication.java）を右クリックし、「Run As → Spring Boot App」を選択します。

2.6 コードの解説とSpring Bootの仕組み

@RestControllerアノテーション

@RestControllerは、このクラスがRESTful APIのエンドポイントを提供するコントローラーであることを示します。Spring MVCの@Controllerと異なり、メソッドの戻り値をそのままHTTPレスポンスボディとして返します。

@GetMappingアノテーション

@GetMapping("/")は、HTTP GETリクエストの/パスへのアクセスを、このメソッドで処理することを宣言します。サーブレットのdoGet()メソッドに相当しますが、アノテーションベースで簡潔に記述できます。

組込Tomcatの仕組み

従来のJava Web開発では、TomcatやJettyなどのサーブレットコンテナを別途インストールし、WARファイルをデプロイする必要がありました。Spring Bootは組込Tomcatをjarファイルに含めることで、java -jarコマンド一つでWebアプリケーションを起動できます。

Spring Bootアプリケーションの起動プロセス

@SpringBootApplicationの役割

メインクラスに付与された@SpringBootApplicationは、以下の3つのアノテーションを統合したものです：
1. @Configuration（Bean定義クラス）
2. @EnableAutoConfiguration（自動設定有効化）
3. @ComponentScan（コンポーネント自動検出）

理解度確認クイズ

  1. Q1. Spring Bootプロジェクトを自動生成するWebツールの名前は何ですか？

    回答を表示

    Spring Initializr
    https://start.spring.io/ でアクセスでき、依存関係の選択、ビルドツールの設定、パッケージ名の指定などを視覚的に行えます。

  2. Q2. pom.xmlでSpring Bootのライブラリをまとめて導入する仕組みを何と呼びますか？

    回答を表示

    Starter（spring-boot-starter-*）
    例えばspring-boot-starter-webには、Spring MVC、組込Tomcat、Jacksonなど、Web開発に必要なライブラリがすべて含まれます。

  3. Q3. @RestControllerと@Controllerの違いを説明してください。

    回答を表示

    @RestController: メソッドの戻り値をそのままHTTPレスポンスボディとして返す（JSON APIなどに使用）
    @Controller: メソッドの戻り値をビュー名として解釈し、テンプレートエンジン（ThymeleafなどでHTMLを生成）

第2章のまとめ

  • Spring Boot開発にはJDK 17以上が必須です。Spring Tool Suite（STS）を使うと、Spring開発に最適化された環境が手に入ります。
  • Spring Initializrを使えば、依存関係の選択からプロジェクト生成まで数分で完了します。
  • pom.xmlには、親POM、プロジェクト情報、依存関係（spring-boot-starter-*）が記述されます。
  • @RestControllerと@GetMappingで、シンプルなWebアプリケーションをすぐに作成できます。組込Tomcatにより、外部サーバーのインストールは不要です。
