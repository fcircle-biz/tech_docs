第2章: Spring Bootの導入と開発環境構築

Spring Boot開発環境の構築とアプリケーション作成

この章で学ぶこと
  • Spring Bootが従来のSpring開発の何を解決するかを理解する
  • Eclipse IDEとSpring Tool Suite（STS）の導入と設定方法
  • Spring Initializrを使った効率的なプロジェクト作成方法
  • MavenとGradleのビルドツールの基本概念
  • 最初のSpring Bootアプリケーションの作成と実行

2.1 Spring Bootが解決する課題

Spring Bootは、Spring Frameworkの「設定の複雑さ」という大きな問題を解決するために開発されたプロジェクトです。従来のSpring開発では、アプリケーションを動作させるまでに膨大な設定作業が必要でした。

従来のSpring開発の課題

Spring Bootの主要機能

Spring Bootが提供する価値
  • 自動設定（Auto Configuration）：クラスパスの内容に基づいて、Spring Beanを自動的に設定
  • スターター依存関係：目的別にまとめられた依存関係セットで、必要なライブラリを一括導入
  • 内蔵サーバー：Tomcat、Jetty、Undertowなどのサーバーをアプリケーションに組み込み
  • 本番対応機能：メトリクス、ヘルスチェック、設定外部化などの運用機能
  • 設定の外部化：プロファイルによる環境別設定管理

2.2 開発環境の準備

必要なソフトウェア

Spring Boot開発を始めるために、以下のソフトウェアが必要です：

必須ソフトウェア
  • JDK 17以上：Java Development Kit
  • IDE：Eclipse、IntelliJ IDEA、VS Code等
  • ビルドツール：Maven または Gradle
推奨ソフトウェア
  • Spring Tool Suite (STS)：Eclipseベースの統合開発環境
  • Git：バージョン管理システム
  • データベース：PostgreSQL、MySQL等

JDK（Java Development Kit）のインストール

Spring Boot 3.x系では、Java 17以上が必要です。Oracle JDK、OpenJDK、Amazon Corretto、Eclipse Temurin等、複数の選択肢があります。

バージョン確認の重要性

Spring Bootのバージョンによって、必要なJavaバージョンが異なります：

  • Spring Boot 2.x：Java 8以上
  • Spring Boot 3.x：Java 17以上（推奨：Java 21）

Eclipse と Spring Tool Suite (STS) の導入

Spring Tool Suite（STS）は、Spring開発に最適化されたEclipseベースの統合開発環境です。Spring Boot開発に必要なプラグインやテンプレートがあらかじめ組み込まれています。

2.3 Spring Initializrを使ったプロジェクト作成

Spring Initializr（https://start.spring.io/）は、Spring Bootプロジェクトを効率的に作成できるWebサービスです。必要な依存関係を選択するだけで、適切に設定されたプロジェクトテンプレートを生成できます。

Spring Initializrでプロジェクト作成（Webブラウザ版）

実習 2-1: Spring Initializrでプロジェクト作成
手順
  1. https://start.spring.io/ にアクセス
  2. 以下の設定を選択：
    • Project: Maven
    • Language: Java
    • Spring Boot: 3.2.x（最新の安定版）
    • Group: com.example
    • Artifact: demo
    • Name: demo
    • Package name: com.example.demo
    • Packaging: Jar
    • Java: 17
  3. Dependencies（依存関係）を追加：
    • Spring Web：Webアプリケーション機能
    • Spring Boot DevTools：開発時の自動再起動機能
  4. 「GENERATE」ボタンをクリックしてZIPファイルをダウンロード
  5. ダウンロードしたZIPファイルを解凍し、IDEでインポート

Eclipse STSでのプロジェクト作成

Eclipse STSでは、IDE内から直接Spring Initializrを利用できます。

実習 2-2: Eclipse STSでのプロジェクト作成
手順
  1. Eclipse STSを起動
  2. File → New → Spring Starter Project を選択
  3. プロジェクト情報を入力（実習2-1と同じ設定）
  4. Next をクリックし、依存関係を選択
  5. Finish をクリックしてプロジェクト作成完了
期待される結果

プロジェクトエクスプローラーにSpring Bootプロジェクトが表示され、自動的に依存関係がダウンロードされます。

2.4 Spring Bootプロジェクトの構造

Spring Initializrで作成されたプロジェクトは、Maven/Gradleの標準的なディレクトリ構造に従っています。

demo/
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/
│   │   │       └── example/
│   │   │           └── demo/
│   │   │               └── DemoApplication.java
│   │   └── resources/
│   │       ├── static/          # 静的ファイル（CSS, JS, 画像等）
│   │       ├── templates/       # テンプレートファイル（HTML等）
│   │       └── application.properties  # 設定ファイル
│   └── test/
│       └── java/
│           └── com/
│               └── example/
│                   └── demo/
│                       └── DemoApplicationTests.java
├── pom.xml                      # Maven設定ファイル
└── README.md

重要なファイルの役割

1. DemoApplication.java（メインクラス）

Spring Bootアプリケーションのエントリーポイントです。

package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }

}
@SpringBootApplicationアノテーションの役割

このアノテーションは、実際には3つのアノテーションの組み合わせです：

  • @Configuration：Javaベースの設定クラスであることを示す
  • @EnableAutoConfiguration：Spring Bootの自動設定を有効化
  • @ComponentScan：同じパッケージ以下のコンポーネントを自動検出
2. pom.xml（Maven設定ファイル）

プロジェクトの依存関係とビルド設定を定義します。

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0">
    <modelVersion>4.0.0</modelVersion>
    
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.2.0</version>
        <relativePath/>
    </parent>
    
    <groupId>com.example</groupId>
    <artifactId>demo</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    
    <properties>
        <java.version>17</java.version>
    </properties>
    
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-devtools</artifactId>
            <scope>runtime</scope>
            <optional>true</optional>
        </dependency>
    </dependencies>
    
    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
3. application.properties

アプリケーションの設定を外部化するためのファイルです。

# サーバーポート設定
server.port=8080

# アプリケーション名
spring.application.name=demo

# ログレベル設定
logging.level.com.example.demo=DEBUG

2.5 ビルドツール：MavenとGradleの比較

Spring Bootプロジェクトでは、MavenとGradleという2つの主要なビルドツールから選択できます。

Maven
特徴
  • XML形式の設定ファイル（pom.xml）
  • 標準化されたプロジェクト構造
  • 豊富なプラグインエコシステム
  • Javaプロジェクトでは最も一般的
適している場面
  • チームでの標準化を重視
  • 既存のMavenプロジェクトとの統合
  • 設定よりも規約を重視
Gradle
特徴
  • Groovy/Kotlin DSL形式の設定
  • 柔軟性が高い
  • ビルドパフォーマンスが良い
  • Android開発では標準
適している場面
  • カスタムビルドロジックが必要
  • マルチプロジェクト構成
  • ビルド高速化が必要

Gradleの設定ファイル例

参考として、同じプロジェクトのGradle版設定ファイル（build.gradle）を示します：

plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
    id 'io.spring.dependency-management' version '1.1.4'
}

group = 'com.example'
version = '0.0.1-SNAPSHOT'
java {
    sourceCompatibility = '17'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    developmentOnly 'org.springframework.boot:spring-boot-devtools'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}

tasks.named('test') {
    useJUnitPlatform()
}

2.6 最初のSpring Bootアプリケーション実行

実習 2-3: Hello World アプリケーションの作成

まず、単純なRESTエンドポイントを作成してSpring Bootアプリケーションの基本動作を確認しましょう。

手順1: コントローラークラスの作成

com.example.demo パッケージに HelloController.java を作成：

package com.example.demo;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    
    @GetMapping("/")
    public String home() {
        return "Hello, Spring Boot!";
    }
    
    @GetMapping("/hello")
    public String hello() {
        return "Hello, World!";
    }
    
    @GetMapping("/info")
    public String info() {
        return "これは Spring Boot アプリケーションです。";
    }
}
手順2: アプリケーションの実行
  1. IDE で DemoApplication.java を右クリック
  2. Run As → Java Application を選択
  3. コンソールに起動ログが表示されることを確認
手順3: 動作確認

ブラウザで以下のURLにアクセス：

  • http://localhost:8080/
  • http://localhost:8080/hello
  • http://localhost:8080/info
期待される結果

各URLで対応するメッセージが表示されます。また、コンソールにはアクセスログが出力されます。

Spring Bootアプリケーションの起動プロセス

Spring Bootアプリケーションが起動する際の内部的な流れを理解しましょう。

sequenceDiagram participant Main as main()メソッド participant SpringApp as SpringApplication participant AutoConfig as AutoConfiguration participant Context as ApplicationContext participant Server as 内蔵サーバー Main->>SpringApp: SpringApplication.run() SpringApp->>AutoConfig: クラスパススキャン AutoConfig->>AutoConfig: 自動設定の決定 AutoConfig->>Context: Bean定義の登録 Context->>Context: Bean インスタンス化 SpringApp->>Server: サーバー起動 Server->>Main: アプリケーション起動完了
起動時に表示される重要なログ
  • Banner：Spring Bootのロゴとバージョン情報
  • Active Profiles：有効なプロファイル設定
  • Auto Configuration Report：適用された自動設定
  • Server Port：Webサーバーの起動ポート
  • Context Initialization：ApplicationContextの初期化情報

2.7 開発効率を向上させるDevTools

Spring Boot DevToolsは、開発時の生産性を向上させる機能を提供します。

DevToolsの主要機能

自動再起動（Automatic Restart）

クラスパス上のファイルが変更されると、アプリケーションを自動的に再起動します。

ライブリロード（Live Reload）

静的リソース（CSS、JavaScript、HTML）の変更をブラウザに自動反映します。

プロパティのデフォルト設定

開発に適した設定（テンプレートキャッシュ無効化等）を自動適用します。

リモートデバッグサポート

リモート環境でのアプリケーション更新をサポートします。

本番環境での注意

DevToolsは自動的に本番環境で無効化されますが、念のため本番用JARには含めないように注意してください。Mavenでは<optional>true</optional>、GradleではdevelopmentOnlyを使用します。

理解度確認クイズ
  1. Spring Bootの主要な利点を3つ挙げてください。
    解答を見る

    1. 自動設定による設定作業の簡素化
    2. スターター依存関係による依存管理の簡素化
    3. 内蔵サーバーによるデプロイメントの簡素化

  2. @SpringBootApplicationアノテーションが含む3つのアノテーションは何ですか？
    解答を見る

    1. @Configuration
    2. @EnableAutoConfiguration
    3. @ComponentScan

  3. Spring Initializrを使用するメリットは何ですか？
    解答を見る

    ・適切な依存関係が設定されたプロジェクトテンプレートを自動生成
    ・最新のSpring Bootバージョンに対応
    ・Maven/Gradleの設定ファイルが適切に生成される
    ・プロジェクト構造が標準化される

  4. MavenとGradleの主な違いを説明してください。
    解答を見る

    Maven：XML形式、設定重視、Javaエコシステムで標準的
    Gradle：Groovy/Kotlin DSL、柔軟性重視、ビルド性能が良い

  5. Spring Boot DevToolsの自動再起動機能はどのような場合に動作しますか？
    解答を見る

    クラスパス上のファイル（Javaソースファイル、リソースファイル）が変更された時に自動的にアプリケーションを再起動します。ただし、静的リソース（CSS、JSなど）の変更では再起動せず、ライブリロードのみが動作します。

2.8 まとめ

この章では、Spring Boot開発環境の構築とアプリケーションの作成方法について学習しました。

重要なポイント
  • Spring Bootは従来のSpring開発の複雑さを解決するフレームワーク
  • Spring Initializrを使用することで、効率的にプロジェクトを作成できる
  • @SpringBootApplicationアノテーション一つで、多くの設定が自動化される
  • Maven/Gradleのビルドツールによって依存関係とビルドプロセスが管理される
  • DevToolsにより、開発時の生産性を大幅に向上できる
  • 内蔵サーバーにより、複雑なデプロイメント設定なしでアプリケーションを実行できる

次の章では、Spring Bootの核心機能である依存性注入（DI）について詳しく学習します。DIは、Spring Frameworkの最も重要な概念の一つであり、保守性の高いアプリケーション開発の基盤となります。
