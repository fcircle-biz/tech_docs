Beanの管理とコンポーネントスキャン

Spring IoCコンテナでのBean管理を深く理解します。ステレオタイプアノテーション（@Component、@Service、@Repository、@Controller）の役割と使い分け、Beanのスコープ（singleton、prototype）、@Configurationと@Beanによる明示的なBean定義、コンポーネントスキャンの仕組みを学びます。

この章で学ぶこと

  • ステレオタイプアノテーション（@Component、@Service、@Repository、@Controller）の役割と使い分けを理解する
  • Beanのスコープ（singleton、prototype）の違いと適切な使い方を学ぶ
  • @Configurationと@Beanによる明示的なBean定義の方法を習得する
  • コンポーネントスキャンの仕組みとbasePackagesの指定方法を理解する

4.1 Beanとは何か

第3章でDIコンテナを学びましたが、そこで管理される「Bean」とは一体何でしょうか。Bean（ビーン）とは、Spring IoCコンテナが管理するオブジェクトのことです。Springアプリケーションで使用するクラスをBeanとして登録することで、DIコンテナが自動的に生成・管理してくれます。

Beanの由来

Beanという名称は、JavaBeansという仕様に由来します。JavaBeansは再利用可能なJavaコンポーネントの規約で、SpringでもこのコンポーネントをBeanと呼んでいます。

従来のJavaプログラミングでは、new演算子を使ってオブジェクトを生成し、開発者が手動で管理していました。しかしSpringでは、Beanとして登録されたクラスのインスタンス生成・ライフサイクル管理・依存性注入をすべてIoCコンテナが担当します。

Spring IoCコンテナのBean管理

graph LR A[アプリケーション起動] --> B[コンポーネントスキャン] B --> C[Bean定義を検出] C --> D[IoCコンテナに登録] D --> E[Bean生成・初期化] E --> F[依存性注入] F --> G[アプリケーション実行]

重要ポイント

Springアプリケーションでは、ほぼすべてのクラスをBeanとして登録します。コントローラー、サービス、リポジトリなど、レイヤーごとに適切なアノテーションを付けることで、IoCコンテナが自動管理してくれます。

4.2 ステレオタイプアノテーション

SpringでBeanを登録する方法の一つが、ステレオタイプアノテーションです。クラスに付けるアノテーションで、そのクラスの役割をSpringコンテナに伝えます。主なステレオタイプアノテーションは以下の4つです。

アノテーション 役割 使用場所
@Component 汎用的なBeanを定義 どのレイヤーにも属さないクラス
@Service ビジネスロジックを担当 サービス層クラス
@Repository データアクセスを担当 リポジトリ層クラス
@Controller HTTPリクエストを処理 コントローラー層クラス

1. @Component - 汎用Bean

最も基本的なステレオタイプアノテーションです。どのレイヤーにも属さない汎用的なクラスに使用します。実は、他の3つのアノテーション（@Service、@Repository、@Controller）は、内部的に@Componentを継承しています。

EmailSender.java（@Component例） 
// 汎用的なメール送信クラス
@Component
public class EmailSender {
    public void send(String to, String subject) {
        // メール送信処理
        System.out.println("送信先: " + to);
        System.out.println("件名: " + subject);
    }
}

2. @Service - ビジネスロジック層

ビジネスロジックを実装するクラスに使用します。コントローラーから呼び出され、リポジトリを使ってデータアクセスを行い、ビジネスルールを適用します。

UserService.java（@Service例） 
// ユーザー管理のビジネスロジック
@Service
public class UserService {
    @Autowired
    private UserRepository repository;

    public User register(String name, int age) {
        // ビジネスルール: 18歳未満は登録不可
        if (age < 18) {
            throw new IllegalArgumentException("18歳以上が必要");
        }
        User user = new User(name, age);
        return repository.save(user);
    }
}

3. @Repository - データアクセス層

データベースとの接続・操作を担当するクラスに使用します。@Repositoryを付けると、Spring Data JDBCがデータアクセス機能を自動提供してくれます（第8章で詳しく学習）。

UserRepository.java（@Repository例） 
// データアクセス層（Spring Data JDBCの例）
@Repository
public interface UserRepository extends CrudRepository {
    // findById、save、deleteAllなどのメソッドが自動提供される
    List findByName(String name);
}

4. @Controller - プレゼンテーション層

HTTPリクエストを受け付け、ビジネスロジックを呼び出し、レスポンスを返すクラスに使用します。Spring MVCの中核となるアノテーションです（第5章で詳しく学習）。

UserController.java（@Controller例） 
// HTTPリクエストを処理
@Controller
public class UserController {
    @Autowired
    private UserService service;

    @GetMapping("/users/{id}")
    public String showUser(@PathVariable Long id, Model model) {
        User user = service.findById(id);
        model.addAttribute("user", user);
        return "user-detail"; // ビューテンプレート名
    }
}

4.3 Beanのスコープ

Beanにはスコープ（Scope）という概念があります。スコープとは、Beanのインスタンスがいつ生成され、どのように共有されるかを定義するものです。主なスコープは以下の2つです。

1. singleton（デフォルト）

singletonスコープでは、Beanのインスタンスがアプリケーション全体で1つだけ生成されます。すべての注入先で同じインスタンスが使い回されます。Springのデフォルトスコープです。

SingletonBean.java（singleton例） 
// @Scopeを省略するとsingletonになる
@Component
public class CounterService {
    private int count = 0;

    public void increment() {
        count++;
    }

    public int getCount() {
        return count; // 全インスタンスで共有される
    }
}

2. prototype

prototypeスコープでは、Beanが注入されるたびに新しいインスタンスが生成されます。状態を保持しないステートレスなBeanには不要ですが、リクエストごとに異なる状態を持つ必要がある場合に使用します。

PrototypeBean.java（prototype例） 
import org.springframework.context.annotation.Scope;

// 注入のたびに新しいインスタンスを生成
@Component
@Scope("prototype")
public class RequestContext {
    private String requestId;
    private LocalDateTime timestamp;

    public RequestContext() {
        this.timestamp = LocalDateTime.now();
    }
}

注意事項

ほとんどの場合、singletonスコープで十分です。prototypeスコープを使うと、メモリ管理やパフォーマンスに影響する可能性があるため、本当に必要な場合のみ使用してください。

4.4 @Configurationと@Beanによる明示的なBean定義

ステレオタイプアノテーション（@Component、@Service等）以外にも、Beanを定義する方法があります。それが@Configurationと@Beanを使った明示的な定義です。

なぜ明示的な定義が必要なのか？

サードパーティのライブラリクラス（自分で@Componentを付けられない）や、複雑な初期化が必要なBeanを登録する場合に、@Configurationと@Beanを使用します。

@Configuration - 設定クラスの定義

@Configurationを付けたクラスは、Bean定義を行う設定クラスとして扱われます。このクラス内で、@Beanアノテーションを付けたメソッドを定義することで、そのメソッドの戻り値がBeanとして登録されます。

AppConfig.java（@Configuration例） 
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import java.time.format.DateTimeFormatter;

// Bean定義を行う設定クラス
@Configuration
public class AppConfig {

    // DateTimeFormatterをBeanとして登録
    @Bean
    public DateTimeFormatter dateFormatter() {
        return DateTimeFormatter.ofPattern("yyyy/MM/dd HH:mm:ss");
    }

    // カスタムサービスをBeanとして登録
    @Bean
    public ReportService reportService() {
        return new ReportService("日次レポート");
    }
}

上記の例では、dateFormatter()メソッドとreportService()メソッドの戻り値がBeanとして登録されます。他のクラスで@Autowiredすることで、これらのBeanを注入できます。

LogService.java（Bean注入例） 
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

@Service
public class LogService {
    @Autowired
    private DateTimeFormatter dateFormatter; // 設定クラスで定義したBean

    public void log(String message) {
        String timestamp = dateFormatter.format(LocalDateTime.now());
        System.out.println("[" + timestamp + "] " + message);
    }
}

使い分けのポイント

自分で作成するクラスには@Component系アノテーション、サードパーティライブラリや複雑な初期化が必要なクラスには@Configurationと@Beanを使います。

4.5 コンポーネントスキャンの仕組み

@Componentなどのステレオタイプアノテーションを付けただけでは、Springはそのクラスを見つけることができません。Springはコンポーネントスキャン（Component Scan）という機能で、指定されたパッケージ配下のクラスを自動検索し、Bean定義を見つけます。

Spring Bootの自動スキャン

Spring Bootを使用している場合、@SpringBootApplicationアノテーションが付いたメインクラスのパッケージとその配下のパッケージが自動的にスキャン対象になります。

Application.java（メインクラス） 
package com.example.myapp; // このパッケージがスキャンのルート

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication // 内部で@ComponentScanを含む
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
com.example.myapp/               ← @SpringBootApplicationがあるパッケージ（スキャン対象）
├── Application.java
├── controller/                  ← スキャン対象
│   └── UserController.java
├── service/                     ← スキャン対象
│   └── UserService.java
└── repository/                  ← スキャン対象
    └── UserRepository.java

com.other.library/               ← スキャン対象外（別パッケージ）
└── SomeClass.java

@ComponentScanによる明示的な指定

別パッケージのクラスをスキャン対象に含めたい場合、@ComponentScanアノテーションでbasePackagesを明示的に指定できます。

Application.java（@ComponentScan例） 
package com.example.myapp;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.ComponentScan;

@SpringBootApplication
@ComponentScan(basePackages = {
    "com.example.myapp",    // デフォルトのスキャン対象
    "com.other.library"     // 追加でスキャン対象に含める
})
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

コンポーネントスキャンの流れ

graph TD A[アプリケーション起動] --> B[@SpringBootApplication検出] B --> C[basePackages決定] C --> D[パッケージ配下をスキャン] D --> E[@Component系アノテーション検出] E --> F[Bean定義としてIoCコンテナに登録] F --> G[依存性注入実行]

スキャン対象外のクラスは？

コンポーネントスキャンの対象外パッケージにあるクラスをBeanとして使いたい場合は、@Configurationクラスで@Beanアノテーションを使って明示的に登録します。

ハンズオン実習: ステレオタイプアノテーションの使い分け

レイヤードアーキテクチャを意識して、適切なステレオタイプアノテーションを付けたBean定義を作成しましょう。

手順

  1. com.example.demo.model.Productクラスを作成（id、name、priceフィールド）
  2. com.example.demo.repository.ProductRepositoryインターフェースを作成し、@Repositoryアノテーションを付ける
  3. com.example.demo.service.ProductServiceクラスを作成し、@Serviceアノテーションを付ける
  4. ProductServiceにProductRepositoryを@Autowiredで注入する
  5. com.example.demo.controller.ProductControllerクラスを作成し、@Controllerアノテーションを付ける
  6. ProductControllerにProductServiceを@Autowiredで注入する
  7. アプリケーションを起動し、DIコンテナが正しくBeanを注入していることを確認する
実装例を表示
Product.java 
package com.example.demo.model;

public class Product {
    private Long id;
    private String name;
    private int price;

    // コンストラクタ、ゲッター、セッター省略
}
ProductRepository.java 
package com.example.demo.repository;

import org.springframework.stereotype.Repository;
import com.example.demo.model.Product;

@Repository // データアクセス層
public interface ProductRepository {
    Product findById(Long id);
    void save(Product product);
}
ProductService.java 
package com.example.demo.service;

import org.springframework.stereotype.Service;
import org.springframework.beans.factory.annotation.Autowired;
import com.example.demo.repository.ProductRepository;
import com.example.demo.model.Product;

@Service // ビジネスロジック層
public class ProductService {
    @Autowired
    private ProductRepository repository;

    public Product getProduct(Long id) {
        return repository.findById(id);
    }
}

理解度確認クイズ

  1. Q1. @Service、@Repository、@Controllerは内部的にどのアノテーションを継承していますか？

    回答を表示

    @Component
    @Service、@Repository、@Controllerは、すべて@Componentを継承した特殊化されたアノテーションです。役割を明確にするために使い分けられます。

  2. Q2. Beanのデフォルトスコープは何ですか？また、そのスコープの特徴を説明してください。

    回答を表示

    singleton
    singletonスコープでは、Beanのインスタンスがアプリケーション全体で1つだけ生成され、すべての注入先で同じインスタンスが共有されます。

  3. Q3. @Configurationと@Beanを使う場合と、@Componentを使う場合の使い分け基準を説明してください。

    回答を表示

    使い分け基準：
    ・自分で作成するクラス → @Component系アノテーションを使用
    ・サードパーティのライブラリクラスや、複雑な初期化が必要なクラス → @Configurationと@Beanを使用
    @Componentは自分のソースコードに直接付けられますが、外部ライブラリには付けられないため、@Beanメソッドで定義します。

第4章のまとめ

  • BeanとはSpring IoCコンテナが管理するオブジェクトで、生成・ライフサイクル・依存性注入を自動化します。
  • ステレオタイプアノテーション（@Component、@Service、@Repository、@Controller）でレイヤーごとの役割を明示します。
  • Beanのスコープはsingletonがデフォルトで、アプリケーション全体で1つのインスタンスが共有されます。
  • @Configurationと@Beanで、サードパーティライブラリや複雑な初期化が必要なBeanを明示的に定義できます。
  • コンポーネントスキャンにより、@SpringBootApplicationのパッケージ配下が自動的にスキャンされ、Bean定義が検出されます。
