第13章

ユーザー管理とロールベースアクセス制御

データベースと連携したユーザー認証とロールベースの認可を実装します。UserDetailsServiceの実装、データベースからのユーザー情報取得、ロールによるアクセス制御（ROLE_USER、ROLE_ADMIN）、Thymeleafでの認証情報表示（sec:authorize）を学びます。

この章で学ぶこと

  • UserDetailsServiceインターフェースを実装し、データベースからユーザー情報を取得する
  • ユーザーテーブルとロールテーブルの設計とリレーション構築
  • ロールベースのアクセス制御（ROLE_USER、ROLE_ADMIN）の実装
  • Thymeleaf Spring Security拡張を使った認証情報の表示と条件分岐

13.1 データベースを使ったユーザー認証の必要性

第12章では、固定ユーザー（メモリ内認証）を使ったSpring Securityの基本を学びました。しかし、実際のアプリケーションでは、ユーザー登録機能やパスワード変更機能が必要となるため、ユーザー情報をデータベースに保存する必要があります。

UserDetailsServiceとは

Spring Securityがユーザー情報を取得するためのインターフェースです。loadUserByUsername()メソッドを実装し、データベースからユーザー情報を取得してUserDetailsオブジェクトとして返します。

データベース認証のフロー

DB-->>UserDetails: ユーザー情報（パスワードハッシュ含む） UserDetails-->>Spring: UserDetailsオブジェクト返却 Spring->>Spring: パスワード照合（BCrypt） alt 認証成功 Spring-->>User: ログイン成功 else 認証失敗 Spring-->>User: エラーメッセージ表示 end

13.2 ユーザーテーブルとロールテーブルの設計

ロールベースアクセス制御（RBAC: Role-Based Access Control）を実装するために、ユーザーテーブルとロールテーブルを設計します。1人のユーザーが複数のロールを持つことができる多対多のリレーションを構築します。

テーブル設計

erDiagram users ||--o{ user_roles : "1:N" roles ||--o{ user_roles : "1:N" users { bigint id PK varchar username "ユーザー名（一意）" varchar password "パスワードハッシュ" boolean enabled "有効フラグ" } roles { bigint id PK varchar name "ロール名（ROLE_USER, ROLE_ADMIN）" } user_roles { bigint user_id FK bigint role_id FK }

テーブル作成SQL

schema.sql 
-- ユーザーテーブル
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(100) NOT NULL, -- BCryptハッシュ値
    enabled BOOLEAN DEFAULT TRUE
);

-- ロールテーブル
CREATE TABLE roles (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(50) UNIQUE NOT NULL -- ROLE_USER, ROLE_ADMIN等
);

-- ユーザー・ロール中間テーブル
CREATE TABLE user_roles (
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    PRIMARY KEY (user_id, role_id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (role_id) REFERENCES roles(id)
);

-- 初期ロールデータ
INSERT INTO roles (name) VALUES ('ROLE_USER');
INSERT INTO roles (name) VALUES ('ROLE_ADMIN');

ロール名の命名規則

Spring Securityでは、ロール名は慣例として「ROLE_」プレフィックスで始めます。hasRole("USER")と指定すると、内部で「ROLE_USER」として処理されます。

import com.example.demo.entity.User;
import org.springframework.data.jdbc.repository.query.Query;
import org.springframework.data.repository.CrudRepository;
import java.util.Optional;

public interface UserRepository extends CrudRepository<User, Long> {

    Optional<User> findByUsername(String username);

    @Query("SELECT r.name FROM roles r " +
           "JOIN user_roles ur ON r.id = ur.role_id " +
           "WHERE ur.user_id = :userId")
    List<String> findRolesByUserId(Long userId);
}

13.4 UserDetailsServiceの実装

Spring SecurityのUserDetailsServiceインターフェースを実装し、データベースからユーザー情報を取得します。

CustomUserDetailsService.java 
package com.example.demo.service;

import com.example.demo.entity.User;
import com.example.demo.repository.UserRepository;
import org.springframework.security.core.userdetails.*;
import org.springframework.security.core.authority.SimpleGrantedAuthority;
import org.springframework.stereotype.Service;
import java.util.List;
import java.util.stream.Collectors;

@Service
public class CustomUserDetailsService implements UserDetailsService {

    private final UserRepository userRepository;

    public CustomUserDetailsService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    @Override
    public UserDetails loadUserByUsername(String username)
            throws UsernameNotFoundException {

        // 1. データベースからユーザー取得
        User user = userRepository.findByUsername(username)
            .orElseThrow(() ->
                new UsernameNotFoundException("User not found: " + username));

        // 2. ユーザーに紐づくロール取得
        List<String> roleNames = userRepository.findRolesByUserId(user.getId());

        // 3. ロール名をGrantedAuthorityに変換
        List<SimpleGrantedAuthority> authorities = roleNames.stream()
            .map(SimpleGrantedAuthority::new)
            .collect(Collectors.toList());

        // 4. UserDetailsオブジェクトとして返却
        return org.springframework.security.core.userdetails.User
            .withUsername(user.getUsername())
            .password(user.getPassword()) // ハッシュ値
            .authorities(authorities)
            .disabled(!user.isEnabled())
            .build();
    }
}

GrantedAuthorityとは

Spring Securityで権限を表すインターフェースです。SimpleGrantedAuthorityは、文字列（"ROLE_USER"など）を権限として扱う実装クラスです。

13.5 SecurityConfigの更新

第12章で作成したSecurityConfigを、データベース認証を使用するように更新します。また、ロールベースのアクセス制御を設定します。

SecurityConfig.java 
package com.example.demo.config;

import com.example.demo.service.CustomUserDetailsService;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.web.SecurityFilterChain;

@Configuration
public class SecurityConfig {

    private final CustomUserDetailsService userDetailsService;

    public SecurityConfig(CustomUserDetailsService userDetailsService) {
        this.userDetailsService = userDetailsService;
    }

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http)
            throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .requestMatchers("/user/**").hasAnyRole("USER", "ADMIN")
                .requestMatchers("/", "/login", "/css/**").permitAll()
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login")
                .defaultSuccessUrl("/dashboard", true)
                .permitAll()
            )
            .logout(logout -> logout
                .logoutSuccessUrl("/")
                .permitAll()
            )
            .userDetailsService(userDetailsService); // カスタムUserDetailsService設定

        return http.build();
    }

    @Bean
    public BCryptPasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}

ロールベースの制御

hasRole("ADMIN")は「ROLE_ADMIN」権限を持つユーザーのみ許可します。hasAnyRole("USER", "ADMIN")は「ROLE_USER」または「ROLE_ADMIN」のいずれかを持つユーザーを許可します。

13.6 Thymeleafでの認証情報表示

Thymeleaf Spring Security拡張を使用すると、ビューでログインユーザー情報や権限に応じた表示制御ができます。

依存関係の追加

pom.xml 
<dependency>
    <groupId>org.thymeleaf.extras</groupId>
    <artifactId>thymeleaf-extras-springsecurity6</artifactId>
</dependency>

認証情報の表示

dashboard.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org"
      xmlns:sec="http://www.thymeleaf.org/extras/spring-security">
<head>
    <title>ダッシュボード</title>
</head>
<body>
    <h1>ダッシュボード</h1>

    <!-- ログインユーザー名表示 -->
    <p>ようこそ、<span sec:authentication="name"></span> さん</p>

    <!-- ロール別表示 -->
    <div sec:authorize="hasRole('ADMIN')">
        <h2>管理者専用メニュー</h2>
        <a href="/admin/users">ユーザー管理</a>
    </div>

    <div sec:authorize="hasRole('USER')">
        <h2>一般ユーザーメニュー</h2>
        <a href="/user/profile">プロフィール</a>
    </div>

    <!-- 未認証ユーザーには表示しない -->
    <div sec:authorize="isAuthenticated()">
        <a href="/logout">ログアウト</a>
    </div>
</body>
</html>

sec:authorize属性の主な使い方

属性値 説明
isAuthenticated() 認証済みユーザーのみ表示
isAnonymous() 未認証ユーザーのみ表示
hasRole('ADMIN') ROLE_ADMIN権限を持つユーザーのみ表示
hasAnyRole('USER', 'ADMIN') いずれかのロールを持つユーザーに表示

13.7 ユーザー登録機能の実装

新規ユーザーをデータベースに登録する機能を実装します。パスワードはBCryptでハッシュ化し、デフォルトでROLE_USERロールを付与します。

ユーザー登録サービス

UserService.java 
package com.example.demo.service;

import com.example.demo.entity.User;
import com.example.demo.repository.UserRepository;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
public class UserService {

    private final UserRepository userRepository;
    private final PasswordEncoder passwordEncoder;

    public UserService(UserRepository userRepository,
                       PasswordEncoder passwordEncoder) {
        this.userRepository = userRepository;
        this.passwordEncoder = passwordEncoder;
    }

    @Transactional
    public void registerUser(String username, String rawPassword) {
        // パスワードをハッシュ化
        String encodedPassword = passwordEncoder.encode(rawPassword);

        User user = new User();
        user.setUsername(username);
        user.setPassword(encodedPassword);
        user.setEnabled(true);

        userRepository.save(user);

        // デフォルトでROLE_USERを付与（JdbcTemplateでINSERT）
        // 詳細は省略
    }
}

ハンズオン実習

  1. 1. データベーステーブル作成
    users、roles、user_rolesテーブルを作成し、初期ロールデータをINSERTします。
  2. 2. エンティティとリポジトリ作成
    User、Roleエンティティ、UserRepositoryを作成します。
  3. 3. CustomUserDetailsService実装
    loadUserByUsername()メソッドを実装し、データベースからユーザー情報を取得します。
  4. 4. SecurityConfig更新
    ロールベースのアクセス制御を設定し、CustomUserDetailsServiceを登録します。
  5. 5. テストユーザー登録
    管理者（ROLE_ADMIN）と一般ユーザー（ROLE_USER）をINSERTし、ログイン動作を確認します。

動作確認

  • ROLE_USERユーザーでログインし、/user/**にアクセスできることを確認
  • ROLE_USERユーザーで/admin/**にアクセスすると403エラーになることを確認
  • ROLE_ADMINユーザーでログインし、すべてのページにアクセスできることを確認
  • Thymeleafで認証情報が正しく表示されることを確認

理解度確認クイズ

  1. Q1. UserDetailsServiceインターフェースで実装する必要があるメソッド名は何ですか？

    回答を表示

    loadUserByUsername(String username)
    このメソッドは、ユーザー名を受け取り、データベースからユーザー情報を取得してUserDetailsオブジェクトとして返します。ユーザーが見つからない場合はUsernameNotFoundExceptionをスローします。

  2. Q2. Spring Securityでロール名に必要なプレフィックスは何ですか？また、hasRole("USER")はどのロール名として処理されますか？

    回答を表示

    プレフィックスは「ROLE_」です。hasRole("USER")は内部で「ROLE_USER」として処理されます。データベースにロールを保存する際は「ROLE_USER」のように保存します。

  3. Q3. Thymeleafで認証済みユーザーのみに要素を表示するには、どの属性を使いますか？

    回答を表示

    sec:authorize="isAuthenticated()"
    この属性を使うことで、ログイン済みユーザーにのみ要素を表示できます。逆に未認証ユーザーのみに表示したい場合は、sec:authorize="isAnonymous()"を使用します。

第13章のまとめ

  • UserDetailsServiceを実装し、データベースからユーザー情報を取得する認証機能を構築できます。
  • users、roles、user_rolesテーブルを設計し、多対多のリレーションでロールを管理します。
  • hasRole()、hasAnyRole()を使い、URLごとにアクセス可能なロールを制御できます。
  • Thymeleafのsec:authorize属性を使い、ロールに応じてビューの表示内容を動的に切り替えられます。
