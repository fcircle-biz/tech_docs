データベース連携 - JDBCの知識を活かす

DAOパターンを使ってStrutsアプリケーションでCRUD操作を実装する方法を学びましょう。

この章で学ぶこと

  • Action、Service、DAOの役割分担を理解できる
  • StrutsアプリケーションでJDBCを使用できる
  • 一覧表示、詳細表示機能を実装できる
  • データの登録・更新・削除機能を実装できる

10.1 データベース連携の基本構成

Strutsアプリケーションでデータベースを扱う場合、階層的なアーキテクチャを採用することが重要です。 一般的には、Action → Service → DAO → Databaseという3層構造で設計します。

JDBCガイドで学んだ知識を活かし、各層の責任を明確に分けることで、保守性の高いアプリケーションを構築できます。

Action

リクエスト制御を担当。ActionFormから入力値を受け取り、Serviceを呼び出し、結果をJSPに渡します。

例: ProductListAction

Service

ビジネスロジックを担当。複数のDAO呼び出しや業務ルールの実装、トランザクション管理を行います。

例: ProductService

DAO

データアクセスを担当。JDBC APIを使ってSQL実行、ResultSetからエンティティへの変換を行います。

例: ProductDao

なぜ3層に分けるのか？

データベースの種類を変更する場合、DAO層だけを書き換えれば済みます。 また、複数のActionから同じServiceを再利用でき、コードの重複を防げます。

10.2 DAOパターンの実装

DAO（Data Access Object）パターンは、データベース操作をカプセル化し、 ビジネスロジックから分離する設計パターンです。JDBCガイドで学んだSQL実行やトランザクション制御の知識をここで活用します。

エンティティクラスの作成

データベースのテーブルに対応するエンティティクラスを作成します。 今回は「商品（Product）」を例にします。

Product.java - エンティティクラス 
package com.example.entity;

import java.math.BigDecimal;
import java.util.Date;

/**
 * 商品エンティティクラス
 * テーブル: products
 */
public class Product {
    private Integer productId;    // 商品ID（主キー）
    private String productName;   // 商品名
    private BigDecimal price;     // 価格
    private Integer stock;        // 在庫数
    private Date createdAt;       // 登録日時

    // デフォルトコンストラクタ
    public Product() {}

    // getter/setter（抜粋）
    public Integer getProductId() { return productId; }
    public void setProductId(Integer productId) {
        this.productId = productId;
    }

    public String getProductName() { return productName; }
    public void setProductName(String productName) {
        this.productName = productName;
    }
    // ... 他のgetter/setterも同様に実装
}

DAOインターフェースの定義

CRUD操作のメソッドを定義したインターフェースを作成します。

ProductDao.java - DAOインターフェース 
package com.example.dao;

import com.example.entity.Product;
import java.util.List;

/**
 * 商品DAOインターフェース
 */
public interface ProductDao {
    // 全件取得
    List<Product> findAll() throws Exception;

    // IDで検索
    Product findById(Integer productId) throws Exception;

    // 新規登録
    void insert(Product product) throws Exception;

    // 更新
    void update(Product product) throws Exception;

    // 削除
    void delete(Integer productId) throws Exception;
}

DAO実装クラス

JDBCを使ってSQL実行する実装クラスを作成します。

ProductDaoImpl.java - DAO実装クラス（全件取得） 
package com.example.dao.impl;

import com.example.dao.ProductDao;
import com.example.entity.Product;
import java.sql.*;
import java.util.ArrayList;
import java.util.List;

public class ProductDaoImpl implements ProductDao {

    @Override
    public List<Product> findAll() throws Exception {
        List<Product> products = new ArrayList<>();
        String sql = "SELECT * FROM products ORDER BY product_id";

        try (Connection conn = getConnection();
             PreparedStatement pstmt = conn.prepareStatement(sql);
             ResultSet rs = pstmt.executeQuery()) {

            while (rs.next()) {
                products.add(mapResultSetToProduct(rs));
            }
        }
        return products;
    }

    // ResultSetからProductオブジェクトへ変換
    private Product mapResultSetToProduct(ResultSet rs)
            throws SQLException {
        Product product = new Product();
        product.setProductId(rs.getInt("product_id"));
        product.setProductName(rs.getString("product_name"));
        product.setPrice(rs.getBigDecimal("price"));
        product.setStock(rs.getInt("stock"));
        product.setCreatedAt(rs.getTimestamp("created_at"));
        return product;
    }
    // ... その他のメソッド実装
}

10.3 Serviceクラスの実装

Serviceクラスは、ビジネスロジックを実装し、DAOを呼び出す役割を持ちます。 複数のDAO操作をまとめたり、トランザクション境界を定義したりします。

ProductService.java - Serviceクラス 
package com.example.service;

import com.example.dao.ProductDao;
import com.example.dao.impl.ProductDaoImpl;
import com.example.entity.Product;
import java.util.List;

/**
 * 商品サービスクラス
 * ビジネスロジックを実装
 */
public class ProductService {
    private ProductDao productDao;

    public ProductService() {
        this.productDao = new ProductDaoImpl();
    }

    // 商品一覧取得
    public List<Product> getAllProducts() throws Exception {
        return productDao.findAll();
    }

    // 商品登録（在庫チェック付き）
    public void registerProduct(Product product) throws Exception {
        // ビジネスルール: 在庫は0以上
        if (product.getStock() < 0) {
            throw new IllegalArgumentException("在庫数は0以上");
        }
        productDao.insert(product);
    }

    // 商品更新
    public void updateProduct(Product product) throws Exception {
        productDao.update(product);
    }
    // ... その他のメソッド
}

Serviceクラスの役割

ServiceクラスはActionから直接DAOを呼ばせず、ビジネスルール（例：在庫チェック、価格計算）を集約します。 これにより、複数のActionで同じロジックを再利用できます。

10.4 Actionクラスでの利用

Actionクラスでは、Serviceクラスを呼び出して処理を行い、結果をJSPに渡します。

ProductListAction.java - 商品一覧Action 
package com.example.action;

import com.example.service.ProductService;
import com.example.entity.Product;
import org.apache.struts.action.*;
import javax.servlet.http.*;
import java.util.List;

public class ProductListAction extends Action {

    @Override
    public ActionForward execute(ActionMapping mapping,
                                 ActionForm form,
                                 HttpServletRequest request,
                                 HttpServletResponse response) {
        try {
            // Serviceクラスを呼び出し
            ProductService service = new ProductService();
            List<Product> products = service.getAllProducts();

            // リクエストスコープにデータを保存
            request.setAttribute("products", products);

            // 一覧画面へフォワード
            return mapping.findForward("success");

        } catch (Exception e) {
            e.printStackTrace();
            return mapping.findForward("error");
        }
    }
}

10.6 CRUD操作の実装

Create（登録）、Read（参照）、Update（更新）、Delete（削除）の各機能を実装します。

Create（登録）

新規データ登録

  • • ActionForm で入力受取
  • • INSERT文を実行
  • • 登録成功メッセージ表示

Read（参照）

データ取得・表示

  • • SELECT文で取得
  • • 一覧/詳細画面表示
  • • 検索条件指定可能

Update（更新）

既存データ更新

  • • 編集画面で初期値表示
  • • UPDATE文を実行
  • • WHERE句でID指定

Delete（削除）

データ削除

  • • 削除確認画面表示
  • • DELETE文を実行
  • • トランザクション管理

登録機能の実装例

ProductDaoImpl.java - insert メソッド 
@Override
public void insert(Product product) throws Exception {
    String sql = "INSERT INTO products " +
                 "(product_name, price, stock) VALUES (?, ?, ?)";

    try (Connection conn = getConnection();
         PreparedStatement pstmt = conn.prepareStatement(sql)) {

        // プレースホルダーに値をセット
        pstmt.setString(1, product.getProductName());
        pstmt.setBigDecimal(2, product.getPrice());
        pstmt.setInt(3, product.getStock());

        // SQL実行
        pstmt.executeUpdate();
    }
}

課題1: 商品検索機能の実装

課題2: 商品削除機能の実装

商品IDを指定して削除する機能を実装してください。

課題3: 在庫更新機能の実装

商品の在庫数を更新する機能を実装してください。

DAO: 在庫数のみ更新するupdateStock メソッドを実装

よくあるトラブルと対処法

Q. NullPointerExceptionが発生する

原因: DAOやServiceのインスタンスが正しく生成されていない、 またはResultSetからデータ取得時にnullチェックが不足している。
対策: コンストラクタでのインスタンス生成を確認、ResultSetの列名が正しいか確認。

Q. SQLExceptionが発生する

原因: SQL構文エラー、テーブル名やカラム名の誤り、接続情報の誤り。
対策: エラーメッセージからSQL文を確認、データベースツールで直接SQL実行して確認。

Q. データが表示されない

原因: リクエストスコープへのセット名とJSPのname属性が一致していない。
対策: request.setAttribute("products", ...) と <logic:iterate name="products"> が一致しているか確認。

理解度確認クイズ

  1. Q1. DAOパターンの主な役割は何ですか？

    回答を表示

    データアクセスロジックをカプセル化することです。 ビジネスロジックからデータベース操作を分離し、SQL実行やResultSetの変換処理を隠蔽します。 これにより、データベースの種類を変更してもビジネスロジックへの影響を最小限にできます。

  2. Q2. Action、Service、DAOの3層に分ける利点は何ですか？

    回答を表示

    関心の分離（Separation of Concerns）が実現できます。
    • 各層が独立しているため、変更の影響範囲が限定される
    • 複数のActionから同じServiceを再利用できる
    • テスト時に各層を独立してテストできる
    • データベース変更時はDAOだけを修正すれば良い

  3. Q3. Serviceクラスでビジネスルールをチェックする理由は何ですか？

    回答を表示

    ビジネスロジックを一箇所に集約するためです。 例えば「在庫数は0以上」というルールをAction層に書くと、複数のActionで同じチェック処理を重複して書くことになります。 Serviceに集約することで、ルール変更時に一箇所を修正するだけで済みます。

第10章のまとめ

  • StrutsアプリケーションではAction → Service → DAOの3層アーキテクチャを採用する
  • DAOパターンはデータアクセスロジックをカプセル化し、ビジネスロジックから分離する
  • Serviceクラスはビジネスルールを実装し、複数のDAO操作をまとめる役割を持つ
  • CRUD操作（Create/Read/Update/Delete）をDAOインターフェースとして定義し、実装クラスでJDBCを使って実現する
  • JDBCガイドで学んだPreparedStatement、ResultSet、トランザクションの知識を活用する
