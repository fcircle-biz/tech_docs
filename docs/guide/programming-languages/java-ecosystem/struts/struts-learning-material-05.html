struts-config.xml - 設定ファイルの理解

Strutsアプリケーションの中心となる設定ファイルの構造と役割を理解しましょう。

この章で学ぶこと

  • struts-config.xmlの基本構造を理解できる
  • form-beansでActionFormを定義できる
  • action-mappingsでURLとActionを紐付けられる
  • forwardで遷移先を設定できる

5.1 struts-config.xmlとは何か

struts-config.xmlは、Strutsアプリケーションの「設計図」にあたる重要な設定ファイルです。 このファイルに、ActionFormやAction、画面遷移などのすべての設定を記述します。

Strutsは起動時にこのファイルを読み込み、アプリケーションの動作を理解します。 逆に言えば、設定ファイルが正しくないと、アプリケーションは動きません。

設定ファイルの役割

例えば、「/login.do というURLにアクセスされたら、LoginActionクラスを実行して、成功時はmain.jspに遷移する」といったルールを すべてstruts-config.xmlに記述します。

struts-config.xmlの配置場所 

myapp/
├── src/
│   └── main/
│       ├── java/           # Javaソースコード
│       └── webapp/
│           ├── WEB-INF/
│           │   ├── web.xml             # サーブレット設定
│           │   ├── struts-config.xml   ← ここに配置
│           │   └── lib/                # JARファイル
│           └── (JSPファイル)

5.2 struts-config.xmlの基本構造

struts-config.xmlは、いくつかの主要なセクションで構成されています。 それぞれのセクションには明確な役割があります。

struts-config.xml（基本構造） 
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE struts-config PUBLIC
  "-//Apache Software Foundation//DTD Struts Configuration 1.3//EN"
  "http://struts.apache.org/dtds/struts-config_1_3.dtd">

<struts-config>

  <!-- 1. ActionFormの定義 -->
  <form-beans>
    <!-- フォームBeanを定義 -->
  </form-beans>

  <!-- 2. 共通の遷移先定義 -->
  <global-forwards>
    <!-- 共通で使う画面遷移を定義 -->
  </global-forwards>

  <!-- 3. URLとActionの紐付け -->
  <action-mappings>
    <!-- リクエストパスとActionクラスを紐付け -->
  </action-mappings>

  <!-- 4. メッセージリソース -->
  <message-resources parameter="ApplicationResources"/>

</struts-config>

form-beans

フォームデータを受け取るActionFormクラスを定義します。名前とクラスパスを指定します。

global-forwards

アプリケーション全体で共通に使う遷移先を定義します。エラーページやホーム画面などに使います。

action-mappings

URLパスとActionクラスを紐付けます。Strutsアプリケーションの心臓部です。

message-resources

エラーメッセージなどの文字列リソースを管理するプロパティファイルを指定します。

5.3 form-beans要素 - ActionFormの定義

form-beansセクションでは、HTMLフォームのデータを受け取るActionFormクラスを定義します。 各フォームに名前（論理名）とJavaクラスを紐付けます。

form-beansの定義例 
<form-beans>
  <!-- ログインフォーム -->
  <form-bean name="loginForm"
             type="com.example.form.LoginForm"/>

  <!-- ユーザー登録フォーム -->
  <form-bean name="userForm"
             type="com.example.form.UserForm"/>

  <!-- 動的フォーム (DynaActionForm) -->
  <form-bean name="searchForm"
             type="org.apache.struts.action.DynaActionForm">
    <form-property name="keyword" type="java.lang.String"/>
    <form-property name="category" type="java.lang.String"/>
  </form-bean>
</form-beans>

form-bean要素の主な属性

  • name: フォームの論理名（action要素から参照されます）
  • type: ActionFormクラスの完全修飾名（パッケージ名を含む）
  • form-property: DynaActionFormの場合、プロパティを動的に定義

重要ポイント

name属性はaction要素のname属性と対応します。この名前を使ってActionとActionFormを紐付けます。

5.4 action-mappings要素 - URLとActionの紐付け

action-mappingsは、Strutsアプリケーションの最も重要なセクションです。 ブラウザからのリクエストURL（例: /login.do）をどのActionクラスで処理するか、成功時・失敗時にどのページに遷移するかを定義します。

action-mappingsの定義例 
<action-mappings>
  <!-- ログイン処理 -->
  <action path="/login"
          type="com.example.action.LoginAction"
          name="loginForm"
          scope="request"
          validate="true"
          input="/login.jsp">
    <forward name="success" path="/main.jsp"/>
    <forward name="failure" path="/login.jsp"/>
  </action>

  <!-- ユーザー一覧表示 -->
  <action path="/userList"
          type="com.example.action.UserListAction">
    <forward name="success" path="/userList.jsp"/>
  </action>
</action-mappings>

action要素の主な属性

  • path: リクエストパス（例: /login → /login.do でアクセス）
  • type: 実行するActionクラスの完全修飾名
  • name: 使用するActionFormの論理名（form-beansで定義した名前）
  • scope: ActionFormのスコープ（request または session）
  • validate: 入力検証を行うか（true/false）
  • input: 検証エラー時に戻るページ

forward要素 - 遷移先の定義

action要素の中に定義するforward要素は、Actionの処理結果に応じた遷移先を指定します。

forward要素の例 
<action path="/login" type="com.example.action.LoginAction" ...>
  <!-- 成功時: メイン画面へ -->
  <forward name="success" path="/main.jsp"/>

  <!-- 失敗時: ログイン画面に戻る -->
  <forward name="failure" path="/login.jsp"/>

  <!-- 別のActionへリダイレクト -->
  <forward name="dashboard" path="/dashboard.do" redirect="true"/>
</action>

Actionクラスからの呼び出し方

Actionクラスでは、mapping.findForward("success")のように、 forward要素のname属性を指定して遷移先を取得します。

5.5 global-forwards要素 - 共通遷移先の定義

global-forwardsは、アプリケーション全体で共通に使う遷移先を定義するセクションです。 例えば、エラーページ、ホーム画面、ログアウト処理など、複数のActionから使われる遷移先をここに定義します。

global-forwardsの定義例 
<global-forwards>
  <!-- ホーム画面 -->
  <forward name="home" path="/index.jsp"/>

  <!-- エラーページ -->
  <forward name="error" path="/error.jsp"/>

  <!-- ログイン画面（リダイレクト） -->
  <forward name="loginPage" path="/login.do" redirect="true"/>

  <!-- システムエラーページ -->
  <forward name="systemError" path="/systemError.jsp"/>
</global-forwards>

グローバルとローカルの優先順位

  • action要素内のforwardが優先されます
  • action要素内にない場合、global-forwardsを検索します
  • どこからでも使う共通遷移先はglobal-forwardsに定義しましょう

実習 5-1: struts-config.xmlの作成

簡単なログイン機能のstruts-config.xmlを作成してみましょう。

手順

  1. 1 WEB-INFフォルダにstruts-config.xmlを新規作成
  2. 2 form-beansセクションにloginFormを定義（LoginForm.classを指定）
  3. 3 global-forwardsセクションにhomeとerrorを定義
  4. 4 action-mappingsセクションに/loginアクションを定義（success、failureのforward付き）
  5. 5 Tomcatを起動して、設定ファイルが正しく読み込まれるか確認

動作確認

Tomcatの起動ログを確認します。エラーが出ていなければ、設定ファイルが正しく読み込まれています。

期待される結果 

INFO: Loading application resources from resource ApplicationResources
INFO: Initializing Struts ActionServlet
INFO: Processing action config path=/login
INFO: Struts ActionServlet initialized successfully

トラブルシューティング

設定ファイルが読み込まれない（404エラー）

原因: struts-config.xmlの配置場所が間違っています。

解決方法: WEB-INF/struts-config.xml に配置されているか確認してください。webappフォルダ直下ではありません。

SAXParseException（XML構文エラー）

原因: XMLの記述に誤りがあります（閉じタグ忘れ、属性のクォート忘れなど）。

解決方法: エラーメッセージの行番号を確認し、該当箇所のXML構文を修正してください。IDEのXML検証機能も活用しましょう。

ClassNotFoundException（クラスが見つからない）

原因: form-beanやactionのtype属性で指定したクラスが存在しない、またはパッケージ名が間違っています。

解決方法: クラス名の完全修飾名（パッケージ名.クラス名）が正しいか確認してください。大文字小文字も区別されます。

理解度確認クイズ

  1. Q1. struts-config.xmlはどこに配置しますか？

    回答を表示

    WEB-INF/struts-config.xml に配置します。webappフォルダ直下ではなく、WEB-INFフォルダの中です。

  2. Q2. form-beans要素のname属性は、どこから参照されますか？

    回答を表示

    action-mappings要素のaction要素から、name属性で参照されます。 例えば、form-beanで定義した"loginForm"は、actionのname="loginForm"で指定します。

  3. Q3. global-forwardsとaction内のforwardの優先順位はどちらが高いですか？

    回答を表示

    action内のforwardが優先されます。 Strutsは、まずaction要素内のforwardを検索し、見つからなければglobal-forwardsを検索します。

第5章のまとめ

  • struts-config.xmlはStrutsアプリケーションの設計図で、WEB-INFフォルダに配置する
  • form-beansでActionFormクラスを定義し、論理名とクラスを紐付ける
  • action-mappingsでリクエストパスとActionクラスを紐付け、forward要素で遷移先を定義する
  • global-forwardsでアプリケーション全体で共通に使う遷移先を定義する
  • 設定ファイルの記述ミスはエラーログで確認し、XML構文やクラスパスを正しく記述することが重要
