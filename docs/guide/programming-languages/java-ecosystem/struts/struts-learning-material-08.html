メッセージ管理とエラーハンドリング

プロパティファイルによるメッセージ管理と、global-exceptionsを使った例外処理を習得します。

この章で学ぶこと

  • MessageResourcesでメッセージを一元管理できる
  • プロパティファイルにメッセージを定義できる
  • global-exceptionsで例外を処理できる
  • 適切なエラーページに遷移させられる

8.1 メッセージ管理の基礎

Webアプリケーションでは、エラーメッセージ、確認メッセージ、ラベル表示など、多くのメッセージを扱います。 これらをJavaコード内にハードコーディングすると、以下の問題が発生します：

  • メッセージを変更するたびにコードを修正し、再コンパイルが必要
  • 同じメッセージが複数箇所に散らばり、管理が困難
  • 多言語対応（国際化）が非常に難しい

メッセージ外部化のメリット

MessageResourcesは、メッセージをプロパティファイルで管理する仕組みです。 コードとメッセージを分離することで、保守性と国際化対応が大幅に向上します。

Strutsでは、ApplicationResources.propertiesという プロパティファイルにメッセージを定義し、JSPやActionクラスから参照します。

8.2 プロパティファイルの使い方

プロパティファイルはキーと値のペアでメッセージを定義します。

ApplicationResources.propertiesの作成

src/main/resourcesフォルダに配置します。

ApplicationResources.properties 
# エラーメッセージ
errors.required={0}は必須入力です。
errors.invalid={0}の形式が正しくありません。
errors.maxlength={0}は{1}文字以内で入力してください。

# フィールドラベル
label.username=ユーザー名
label.email=メールアドレス
label.password=パスワード

# 共通メッセージ
message.login.success=ログインに成功しました。
message.register.complete=会員登録が完了しました。
message.delete.confirm=本当に削除しますか？

プレースホルダーの使い方

{0}、{1}などは プレースホルダーで、実行時に動的な値を埋め込めます。
例: errors.required={0}は必須入力です。
→ 「ユーザー名は必須入力です。」と表示できます。

struts-config.xmlでの設定

プロパティファイルをStrutsに認識させる設定を追加します。

struts-config.xml（抜粋） 
<struts-config>
    <!-- MessageResources設定 -->
    <message-resources parameter="ApplicationResources" />
</struts-config>

8.3 JSPでのメッセージ表示

定義したメッセージをJSPで表示する方法を学びます。

bean:messageタグの使い方

<bean:message>タグで メッセージキーに対応する値を表示します。

login.jsp 
<%@ taglib uri="/tags/struts-bean" prefix="bean" %>

<h1><bean:message key="label.username"/></h1>
<!-- 「ユーザー名」と表示 -->

<p><bean:message key="message.login.success"/></p>
<!-- 「ログインに成功しました。」と表示 -->

html:errorsタグでエラー表示

バリデーションエラーをまとめて表示する専用タグです。

register.jsp 
<%@ taglib uri="/tags/struts-html" prefix="html" %>

<!-- 全てのエラーメッセージを表示 -->
<html:errors/>

<!-- 特定フィールドのエラーのみ表示 -->
<html:errors property="username"/>

8.4 例外ハンドリングの仕組み

Actionクラスでの処理中に例外が発生した場合、どう対処すべきでしょうか？ Strutsではglobal-exceptionsを使って、 例外ごとに適切なエラーページへ遷移させることができます。

例外処理が必要な場面

データベース例外

接続エラー、SQL構文エラー、データ不整合など、DB操作で発生する例外を適切に処理

ビジネスロジック例外

在庫不足、権限エラーなど、業務ルール違反時の例外を適切にハンドリング

global-exceptionsの設定

struts-config.xmlで 例外とエラーページの対応を定義します。

struts-config.xml（抜粋） 
<struts-config>
    <!-- グローバル例外設定 -->
    <global-exceptions>
        <!-- データベース例外 -->
        <exception
            type="java.sql.SQLException"
            key="error.database"
            path="/error.jsp"/>

        <!-- 業務例外 -->
        <exception
            type="com.example.BusinessException"
            key="error.business"
            path="/businessError.jsp"/>

        <!-- その他の例外 -->
        <exception
            type="java.lang.Exception"
            key="error.system"
            path="/systemError.jsp"/>
    </global-exceptions>
</struts-config>

exception要素の属性

属性 説明
type 捕捉する例外クラスの完全修飾名
key エラーメッセージのプロパティキー
path 遷移先のエラーページパス

プロパティファイルにエラーメッセージ追加

ApplicationResources.properties 
# 例外メッセージ
error.database=データベースエラーが発生しました。管理者に連絡してください。
error.business=業務エラー: {0}
error.system=システムエラーが発生しました。

8.5 エラーページの作成と遷移

例外が発生した際に表示するエラーページを作成します。

エラーページの例

error.jsp 
<%@ page contentType="text/html;charset=UTF-8" isErrorPage="true" %>
<%@ taglib uri="/tags/struts-bean" prefix="bean" %>
<%@ taglib uri="/tags/struts-html" prefix="html" %>

<html>
<head>
    <title>エラー</title>
</head>
<body>
    <h1>エラーが発生しました</h1>

    <!-- エラーメッセージ表示 -->
    <html:errors/>

    <p>
        <a href="<html:rewrite page="/index.jsp"/>">
            トップページに戻る
        </a>
    </p>
</body>
</html>

8.6 実習: メッセージ管理とエラーハンドリング

実習 8-1: メッセージプロパティとエラーページの実装

プロパティファイルでメッセージを管理し、例外発生時にエラーページへ遷移する仕組みを実装します。

手順

  1. 1 src/main/resources/ApplicationResources.propertiesを作成し、 エラーメッセージとラベルを定義
  2. 2 struts-config.xmlに <message-resources>と <global-exceptions>を設定
  3. 3 error.jspを作成し、 <html:errors/>でエラー表示
  4. 4 意図的に例外をスローするActionを作成し、エラーページへの遷移を確認

サンプルコード: 例外をスローするAction

TestExceptionAction.java 
package com.example.action;

import javax.servlet.http.*;
import org.apache.struts.action.*;

// 例外をスローしてエラーハンドリングをテスト
public class TestExceptionAction extends Action {

    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response) throws Exception {

        // 意図的に例外をスロー
        throw new Exception("テスト例外が発生しました");
    }
}

動作確認

  1. Tomcatを起動し、http://localhost:8080/yourapp/testException.doにアクセス
  2. 例外がスローされ、error.jspに遷移することを確認
  3. エラーメッセージが正しく表示されることを確認

トラブルシューティング

メッセージが表示されず、キーがそのまま表示される

原因: プロパティファイルが正しく読み込まれていない、またはキー名が間違っている

解決方法:

  • struts-config.xmlの<message-resources>設定を確認
  • プロパティファイルがsrc/main/resourcesに配置されているか確認
  • キー名のスペルミスをチェック
例外が発生してもエラーページに遷移しない

原因: global-exceptionsの設定が正しくない

解決方法:

  • type属性の例外クラスが完全修飾名になっているか確認
  • path属性のエラーページパスが正しいか確認
  • 例外の継承順序を考慮（具体的な例外を先に、汎用的な例外を後に定義）

理解度確認クイズ

  1. Q1. メッセージをプロパティファイルで管理する主なメリットを2つ挙げてください。

    回答を表示

    1. 保守性の向上 - メッセージを変更してもJavaコードの再コンパイルが不要
    2. 国際化対応 - 言語別のプロパティファイルを用意することで多言語対応が容易

  2. Q2. JSPでプロパティファイルのメッセージを表示するタグは何ですか？

    回答を表示

    <bean:message key="メッセージキー"/>タグです。 key属性でプロパティファイルのキーを指定します。

  3. Q3. global-exceptionsで例外ごとに異なるエラーページに遷移させるには、どの属性を使いますか？

    回答を表示

    path属性です。 例: <exception type="..." path="/error.jsp"/>

第8章のまとめ

  • MessageResourcesを使ってメッセージをプロパティファイルで一元管理できる
  • ApplicationResources.propertiesでキーと値のペアでメッセージを定義し、プレースホルダーで動的な値を埋め込める
  • JSPでは<bean:message>タグと<html:errors>タグでメッセージを表示
  • global-exceptionsで例外の種類ごとに適切なエラーページへ遷移させられる
  • エラーページではisErrorPage="true"を指定し、例外情報を活用できる
