Actionクラスの基礎 - リクエスト処理の要

MVCのController役を担うActionクラスの作成方法と処理の仕組みを理解しましょう。

この章で学ぶこと

  • Actionクラスを継承してコントローラーを作成できる
  • executeメソッドの引数と戻り値を理解できる
  • ActionForwardで画面遷移を制御できる

3.1 Actionクラスとは

Actionクラスは、StrutsアプリケーションにおいてController（制御）の役割を担う中心的なコンポーネントです。 ユーザーからのリクエストを受け取り、ビジネスロジックを実行して、適切な画面（View）に遷移させる責任があります。

Actionクラスの役割

  • ユーザーのリクエストを受け取る
  • フォームデータ（ActionForm）を受け取る
  • ビジネスロジックを実行する
  • ModelとやりとりしてDB操作などを行う
  • 次に表示する画面を決定する

MVC内での位置づけ

Controller（制御）

ユーザーからの入力を処理し、ModelとViewをつなぐ橋渡し役

Model（データ・ロジック）を呼び出す
View（画面）を選択して表示

ServletとActionの違い

ServletはHTTPリクエスト/レスポンスを直接扱いますが、Actionクラスはフレームワークが用意した 抽象化されたインターフェースを使うため、より簡潔にコードが書けます。

3.2 Actionクラスの基本構造

Actionクラスを作成するには、org.apache.struts.action.Actionクラスを継承し、 executeメソッドをオーバーライドします。

HelloAction.java - 最もシンプルなActionクラス 
package com.example.action;

import org.apache.struts.action.Action;
import org.apache.struts.action.ActionForm;
import org.apache.struts.action.ActionForward;
import org.apache.struts.action.ActionMapping;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

/**
 * Actionクラスの基本形
 */
public class HelloAction extends Action {

    @Override
    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response
    ) throws Exception {

        // 1. 処理を実行（ここでは簡単なメッセージ設定）
        String message = "Hello, Struts!";
        request.setAttribute("message", message);

        // 2. 遷移先を返す
        return mapping.findForward("success");
    }
}

必須のポイント

  • Actionクラスを継承する
  • executeメソッドをオーバーライド
  • ActionForwardオブジェクトを返す
  • 例外はthrows Exceptionで宣言

命名規則

  • クラス名は〇〇Actionとする
  • 例: LoginAction, SearchAction
  • パッケージはaction配下に配置
  • 1クラス1責任の原則を守る

3.3 executeメソッドの4つの引数

executeメソッドは4つの引数を受け取ります。それぞれの役割を理解することが重要です。

1

ActionMapping mapping

struts-config.xmlに定義された設定情報へのアクセスを提供します。 遷移先の検索や、このActionに関する設定を取得できます。

主な使い方: mapping.findForward("success")

2

ActionForm form

HTMLフォームから送信されたデータを保持するオブジェクトです。 第4章で詳しく学びますが、ここではユーザー入力値の受け皿と理解してください。

主な使い方: LoginForm loginForm = (LoginForm) form;

3

HttpServletRequest request

ServletでおなじみのHTTPリクエストオブジェクトです。 リクエストパラメータの取得、セッション操作、属性の設定などができます。

主な使い方: request.setAttribute("data", value);

4

HttpServletResponse response

HTTPレスポンスオブジェクトです。 通常はStrutsが自動的に処理するため、直接使用することは稀ですが、特殊な場合に利用します。

特殊な用途: ファイルダウンロード、JSONレスポンスなど

覚えておきたいポイント

最も使うのはmapping（遷移先の決定）とrequest（データの受け渡し）です。 formは第4章、responseは応用編で詳しく学びます。

3.4 ActionForwardによる画面遷移

ActionForwardは、処理完了後にどの画面（JSP）に遷移するかを表すオブジェクトです。 executeメソッドは必ずActionForwardオブジェクトを返す必要があります。

SearchAction.java - ActionForwardの使用例 
public class SearchAction extends Action {

    @Override
    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response
    ) throws Exception {

        // 検索キーワードを取得
        String keyword = request.getParameter("keyword");

        // 検索処理を実行（例）
        if (keyword == null || keyword.trim().isEmpty()) {
            // 入力エラー時はinput画面に戻る
            return mapping.findForward("input");
        }

        // 検索結果を取得（実際はDB検索など）
        List<String> results = performSearch(keyword);

        // 結果をrequestスコープに格納
        request.setAttribute("results", results);

        // 成功時は結果表示画面へ
        return mapping.findForward("success");
    }
}

success

処理が正常に完了した場合の遷移先

failure

ビジネスロジック上のエラーが発生した場合

input

入力エラーで元のフォーム画面に戻る場合

3.5 ActionMappingの役割

ActionMappingは、struts-config.xml内の<action>要素に対応するオブジェクトです。 設定情報を保持し、遷移先の検索などを行います。

設定ファイルとの連携

ActionMappingは設定ファイルの内容を読み込んだオブジェクトです。 つまり、設定ファイルを変更すれば、Javaコードを変更せずに遷移先を変更できるのが大きなメリットです。

3.6 実装例：ログイン処理

実際のログイン処理を例に、Actionクラスの実装を見てみましょう。

LoginAction.java - ログイン処理の実装例 
package com.example.action;

import org.apache.struts.action.*;
import javax.servlet.http.*;

/**
 * ログイン処理を行うActionクラス
 */
public class LoginAction extends Action {

    @Override
    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response
    ) throws Exception {

        // 1. リクエストパラメータから入力値を取得
        String userId = request.getParameter("userId");
        String password = request.getParameter("password");

        // 2. 入力チェック（簡易版）
        if (userId == null || password == null) {
            return mapping.findForward("input");
        }

        // 3. 認証処理（実際はDBと照合）
        boolean isAuthenticated = authenticate(userId, password);

        // 4. 認証結果に応じて遷移先を変更
        if (isAuthenticated) {
            // セッションにユーザーIDを保存
            HttpSession session = request.getSession();
            session.setAttribute("userId", userId);

            // 成功画面へ
            return mapping.findForward("success");
        } else {
            // エラーメッセージを設定
            request.setAttribute("errorMessage", "ユーザーIDまたはパスワードが正しくありません");

            // 失敗画面へ
            return mapping.findForward("failure");
        }
    }

    /**
     * 認証処理（ダミー実装）
     */
    private boolean authenticate(String userId, String password) {
        // 実際はDB問い合わせ
        return "admin".equals(userId) && "password".equals(password);
    }
}

処理の流れ

  1. リクエストパラメータから入力値を取得
  2. 入力値の妥当性をチェック
  3. ビジネスロジック（認証処理）を実行
  4. 結果に応じてセッションにデータを保存
  5. 適切な遷移先（ActionForward）を返す

実習：簡単なActionクラスを作ってみよう

以下の仕様でActionクラスを実装してみましょう：

仕様

  • クラス名: GreetingAction
  • リクエストパラメータ「name」を受け取る
  • 「こんにちは、〇〇さん！」というメッセージを作成
  • メッセージをrequestスコープに「greeting」という名前で格納
  • 「success」という名前のforwardに遷移
実装例を表示 
public class GreetingAction extends Action {
    @Override
    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response
    ) throws Exception {

        String name = request.getParameter("name");
        String greeting = "こんにちは、" + name + "さん！";
        request.setAttribute("greeting", greeting);

        return mapping.findForward("success");
    }
}

理解度確認クイズ

  1. Q1. Actionクラスで必ずオーバーライドする必要があるメソッドの名前は何ですか？

    回答を表示

    executeメソッドです。このメソッドがリクエスト処理の中心となり、 必ずActionForwardオブジェクトを返す必要があります。

  2. Q2. executeメソッドの戻り値の型は何ですか？また、それは何を表していますか？

    回答を表示

    ActionForward型です。これは処理完了後にどの画面（JSP）に遷移するかを表すオブジェクトで、 mapping.findForward("forward名")で取得します。

  3. Q3. ActionMappingオブジェクトの主な役割を説明してください。

    回答を表示

    ActionMappingはstruts-config.xmlの設定情報を保持し、 遷移先（ActionForward）の検索などを行います。 設定ファイルとJavaコードをつなぐ重要な役割を果たします。

第3章のまとめ

  • ActionクラスはMVCのController役で、org.apache.struts.action.Actionを継承して作成する
  • executeメソッドは4つの引数（mapping, form, request, response）を受け取り、ActionForwardを返す
  • ActionForwardは遷移先を表すオブジェクトで、mapping.findForward()で取得する
  • ActionMappingはstruts-config.xmlの設定情報を保持し、設定ファイルとコードをつなぐ
  • 処理結果に応じて適切なActionForward（success, failure, inputなど）を返すことで画面遷移を制御できる
