Strutsタグライブラリ入門 - JSPとの連携

JSPでStrutsタグライブラリを使って効率的に画面を構築する方法を学びます。

この章で学ぶこと

  • Strutsタグライブラリをtaglibディレクティブで宣言できる
  • html:タグでフォーム要素を作成できる
  • bean:タグでデータを表示できる
  • logic:タグで条件分岐・繰り返し処理ができる

6.1 Strutsタグライブラリとは

タグライブラリ（Tag Library）とは、JSPで独自のタグを使えるようにする仕組みです。 Strutsでは、フォーム処理やデータ表示を効率化するための専用タグライブラリが用意されています。

通常のHTMLタグやJavaのスクリプトレット（<% ... %>）を使うより、 Strutsタグライブラリを使うことで、以下のメリットがあります：

  • 可読性の向上 - Javaコードを書かずに処理を記述できる
  • 自動連携 - ActionFormと自動的にバインディングされる
  • エラーハンドリング - バリデーションエラーの表示が簡単
  • 保守性 - ビジネスロジックと表示ロジックが分離される

html:

HTMLタグライブラリ。フォーム要素（input、select、textareaなど）を生成します。

例: <html:text>

bean:

Beanタグライブラリ。スコープ内のJavaBeansからデータを取得・表示します。

例: <bean:write>

logic:

Logicタグライブラリ。条件分岐や繰り返し処理を記述します。

例: <logic:iterate>

6.2 taglib宣言の方法

Strutsタグライブラリを使用するには、JSPファイルの先頭で<%@ taglib %>ディレクティブを宣言します。

userForm.jsp 
<%@ page contentType="text/html;charset=UTF-8" %>
<%-- Strutsタグライブラリの宣言 --%>
<%@ taglib uri="/tags/struts-html" prefix="html" %>
<%@ taglib uri="/tags/struts-bean" prefix="bean" %>
<%@ taglib uri="/tags/struts-logic" prefix="logic" %>

<!DOCTYPE html>
<html>
<head>
    <title>ユーザー登録</title>
</head>
<body>
    <!-- ここからStrutsタグを使用できる -->
</body>
</html>

prefix属性について

prefix="html"は、タグの接頭辞を指定します。これにより<html:form>のように使えます。 慣例として、html、bean、logicという接頭辞が使われますが、任意の名前に変更可能です。

web.xmlでのTLD設定

Strutsタグライブラリを認識させるため、web.xmlにTLD（Tag Library Descriptor）の位置を定義します。

WEB-INF/web.xml 
<web-app>
    <!-- Strutsタグライブラリの定義 -->
    <taglib>
        <taglib-uri>/tags/struts-html</taglib-uri>
        <taglib-location>
            /WEB-INF/struts-html.tld
        </taglib-location>
    </taglib>
    <taglib>
        <taglib-uri>/tags/struts-bean</taglib-uri>
        <taglib-location>
            /WEB-INF/struts-bean.tld
        </taglib-location>
    </taglib>
    <taglib>
        <taglib-uri>/tags/struts-logic</taglib-uri>
        <taglib-location>
            /WEB-INF/struts-logic.tld
        </taglib-location>
    </taglib>
</web-app>

TLDファイルの配置

struts-html.tld、struts-bean.tld、struts-logic.tldの各ファイルは、 StrutsライブラリのJARに含まれています。通常はWEB-INF/ディレクトリに配置します。

6.3 html:タグの使い方（フォーム要素）

html:タグライブラリは、HTMLフォーム要素をStrutsと連携させるために使用します。 ActionFormのプロパティと自動的にバインディングされるため、データの受け渡しが簡単になります。

主要なhtml:タグ

タグ 説明 対応HTML
<html:form> フォームの開始タグ <form>
<html:text> テキスト入力フィールド <input type="text">
<html:password> パスワード入力フィールド <input type="password">
<html:textarea> 複数行テキスト入力 <textarea>
<html:checkbox> チェックボックス <input type="checkbox">
<html:select> プルダウンメニュー <select>
<html:submit> 送信ボタン <input type="submit">

html:formタグの基本

<html:form>タグは、Actionへの送信先を指定し、ActionFormと自動連携します。

JSPでの記述 
<html:form action="/register">
    <!-- フォームの内容 -->
    <html:text property="username" />
    <html:submit value="登録" />
</html:form>

実例: ユーザー登録フォーム

以下は、ユーザー登録画面のJSPでhtml:タグを使った例です。

register.jsp 
<%@ taglib uri="/tags/struts-html" prefix="html" %>

<html:form action="/register">
    <table>
        <tr>
            <td>ユーザー名:</td>
            <td>
                <!-- property属性にActionFormのフィールド名を指定 -->
                <html:text property="username" size="20" />
            </td>
        </tr>
        <tr>
            <td>パスワード:</td>
            <td>
                <html:password property="password" size="20" />
            </td>
        </tr>
        <tr>
            <td>自己紹介:</td>
            <td>
                <html:textarea property="bio" rows="5" cols="40" />
            </td>
        </tr>
        <tr>
            <td colspan="2">
                <html:submit value="登録" />
            </td>
        </tr>
    </table>
</html:form>

自動バインディングのメリット

property="username"と指定すると、Strutsが自動的にActionForm.getUsername()と setUsername()を呼び出します。これにより、手動でリクエストパラメータを取得する必要がありません。

6.4 bean:タグの使い方（データ表示）

bean:タグライブラリは、スコープ内のJavaBeansからデータを取得・表示するために使用します。 <bean:write>タグが最もよく使われます。

主要なbean:タグ

タグ 説明
<bean:write> Beanのプロパティ値を出力
<bean:define> 変数を定義してスコープに格納
<bean:message> プロパティファイルからメッセージを取得

bean:writeタグの基本

<bean:write>タグは、スコープ内のBeanから値を取り出して画面に表示します。

基本的な使い方 
<!-- リクエストスコープの"user"というBeanから"username"を取得 -->
<bean:write name="user" property="username" />

<!-- セッションスコープから取得 -->
<bean:write name="user" property="email" scope="session" />

実例: ユーザー情報表示

Actionでリクエストスコープに設定したBeanを、JSPで表示する例です。

UserAction.java 
public class UserAction extends Action {
    public ActionForward execute(...) {
        // ユーザー情報を取得
        User user = new User();
        user.setUsername("山田太郎");
        user.setEmail("yamada@example.com");

        // リクエストスコープに設定
        request.setAttribute("user", user);

        return mapping.findForward("success");
    }
}
userDetail.jsp 
<%@ taglib uri="/tags/struts-bean" prefix="bean" %>

<h2>ユーザー詳細</h2>
<table>
    <tr>
        <td>ユーザー名:</td>
        <td><bean:write name="user" property="username" /></td>
    </tr>
    <tr>
        <td>メールアドレス:</td>
        <td><bean:write name="user" property="email" /></td>
    </tr>
</table>

スクリプトレットとの比較

従来のJSPでは<%= user.getUsername() %>のように書いていましたが、 <bean:write>を使うことで、Javaコードを書かずにデータを表示できます。

6.5 logic:タグの使い方（条件分岐・繰り返し）

logic:タグライブラリは、JSPで条件分岐や繰り返し処理を記述するために使用します。 Javaのif文やfor文をタグとして記述できます。

主要なlogic:タグ

タグ 説明 対応Java構文
<logic:iterate> コレクションの繰り返し処理 forループ
<logic:present> 変数が存在する場合に実行 if (obj != null)
<logic:notPresent> 変数が存在しない場合に実行 if (obj == null)
<logic:equal> 値が等しい場合に実行 if (value.equals(...))
<logic:notEqual> 値が等しくない場合に実行 if (!value.equals(...))

logic:iterateタグ（繰り返し処理）

<logic:iterate>タグは、リストや配列の要素を1つずつ処理します。

userList.jsp 
<%@ taglib uri="/tags/struts-logic" prefix="logic" %>
<%@ taglib uri="/tags/struts-bean" prefix="bean" %>

<h2>ユーザー一覧</h2>
<table border="1">
    <tr>
        <th>ID</th>
        <th>ユーザー名</th>
        <th>メールアドレス</th>
    </tr>
    <!-- "userList"というリストを繰り返し処理 -->
    <logic:iterate id="user" name="userList">
        <tr>
            <td><bean:write name="user" property="id" /></td>
            <td><bean:write name="user" property="username" /></td>
            <td><bean:write name="user" property="email" /></td>
        </tr>
    </logic:iterate>
</table>

logic:presentタグ（条件分岐）

<logic:present>タグは、変数が存在する場合のみ中身を実行します。

条件分岐の例 
<%-- ユーザーがログインしている場合 --%>
<logic:present name="loginUser" scope="session">
    <p>ようこそ、<bean:write name="loginUser" property="username" />さん</p>
</logic:present>

<%-- ユーザーがログインしていない場合 --%>
<logic:notPresent name="loginUser" scope="session">
    <p>ログインしてください</p>
</logic:notPresent>

logic:equalタグ（値の比較）

<logic:equal>タグは、値が一致する場合のみ処理を実行します。

値の比較例 
<%-- ユーザーのロールが"admin"の場合 --%>
<logic:equal name="user" property="role" value="admin">
    <p>管理者メニュー</p>
    <a href="/admin/users">ユーザー管理</a>
</logic:equal>

<%-- ユーザーのロールが"admin"ではない場合 --%>
<logic:notEqual name="user" property="role" value="admin">
    <p>一般ユーザーメニュー</p>
</logic:notEqual>

logic:タグの利点

従来のJSPでは<% if (user != null) { %>のようにスクリプトレットを書いていましたが、 logic:タグを使うことで、Javaコードを書かずに条件分岐や繰り返し処理を記述できます。

実習 6-1: タグライブラリを使った商品一覧表示

商品一覧をStrutsタグライブラリを使って表示するJSPを作成します。

手順

  1. 1 Product.javaを作成（id、name、priceフィールド）
  2. 2 ProductAction.javaで商品リストをリクエストスコープに設定
  3. 3 productList.jspで<logic:iterate>と<bean:write>を使って商品一覧を表示
  4. 4 在庫が0の商品には<logic:equal>で「在庫切れ」と表示

ヒント

ArrayList<Product>を作成し、request.setAttribute("products", productList)でリクエストスコープに設定します。 JSPでは<logic:iterate id="product" name="products">で繰り返し処理できます。

理解度確認クイズ

  1. Q1. Strutsタグライブラリを使用するために、JSPファイルの先頭で行う必要のある記述は何ですか？

    回答を表示

    <%@ taglib %>ディレクティブを使ってタグライブラリを宣言します。
    例：<%@ taglib uri="/tags/struts-html" prefix="html" %>

  2. Q2. ActionFormのプロパティ「email」と自動的にバインディングされるhtml:タグの記述はどれですか？

    回答を表示

    <html:text property="email" />
    property属性にActionFormのフィールド名を指定することで、自動的にgetEmail()とsetEmail()が呼び出されます。

  3. Q3. リクエストスコープの「productList」というListを繰り返し処理するlogic:タグの記述例を示してください。

    回答を表示

    <logic:iterate id="product" name="productList">
      <!-- 繰り返し処理の内容 -->
    </logic:iterate>
    id属性で各要素を受け取る変数名を指定し、name属性でリスト名を指定します。

第6章のまとめ

  • StrutsタグライブラリはJSPでフォーム処理やデータ表示を効率化するための専用タグ群
  • <%@ taglib %>ディレクティブでタグライブラリを宣言し、web.xmlでTLDを設定する
  • html:タグはフォーム要素を生成し、ActionFormと自動的にバインディングされる
  • bean:タグはスコープ内のJavaBeansからデータを取得・表示する（特にbean:write）
  • logic:タグは条件分岐（logic:present、logic:equal）や繰り返し処理（logic:iterate）を記述する
  • タグライブラリを使うことで、JSPからJavaコード（スクリプトレット）を排除し、可読性と保守性が向上する
