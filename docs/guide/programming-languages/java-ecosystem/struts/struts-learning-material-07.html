フォーム処理とバリデーション機能

ユーザー入力の検証とエラーハンドリングを学び、安全で使いやすいWebアプリケーションを構築しましょう。

この章で学ぶこと

  • ActionFormのvalidateメソッドで入力検証ができる
  • validation.xmlで検証ルールを定義できる
  • ActionErrors/ActionMessagesでエラーを管理できる
  • html:errorsタグでエラーを表示できる

7.1 入力検証（バリデーション）の必要性

Webアプリケーションでは、ユーザーが入力したデータをそのまま信用してはいけません。 誤った入力やセキュリティ上の脅威から、システムを守る必要があります。

検証が必要な理由

  • データの整合性 - 不正なデータがDBに入るのを防ぐ
  • セキュリティ - SQLインジェクションやXSS攻撃を防ぐ
  • ユーザビリティ - 即座にエラーを指摘し修正を促す
  • 業務ルール遵守 - 必須項目や形式チェック

検証の種類

  • クライアント側検証（JavaScript） 即座にフィードバック、通信量削減。ただし迂回可能
  • サーバー側検証（必須） Strutsで実装。最終防衛ライン。絶対に省略不可
  • DB制約 NOT NULL、UNIQUE、外部キー制約など

重要ポイント

クライアント側の検証は補助的なものです。必ずサーバー側でも検証してください。 ユーザーはブラウザのデベロッパーツールでJavaScriptを無効化したり、直接HTTPリクエストを送信できます。

7.2 ActionFormのvalidateメソッド

ActionFormクラスにはvalidate()メソッドが用意されており、 このメソッドをオーバーライドすることで手動で入力検証を実装できます。

validateメソッドの実装例

ユーザー登録フォームで、必須チェックと形式チェックを行う例です。

UserForm.java 
package com.example.form;

import org.apache.struts.action.*;
import javax.servlet.http.HttpServletRequest;

public class UserForm extends ActionForm {
    private String username;  // ユーザー名
    private String email;     // メールアドレス
    private String password;  // パスワード

    // getter/setter省略

    @Override
    public ActionErrors validate(
        ActionMapping mapping,
        HttpServletRequest request) {

        ActionErrors errors = new ActionErrors();

        // 1. ユーザー名の必須チェック
        if (username == null || username.trim().isEmpty()) {
            errors.add("username",
                new ActionMessage("error.username.required"));
        }

        // 2. メールアドレスの必須チェックと形式チェック
        if (email == null || email.trim().isEmpty()) {
            errors.add("email",
                new ActionMessage("error.email.required"));
        } else if (!email.matches("^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$")) {
            errors.add("email",
                new ActionMessage("error.email.invalid"));
        }

        // 3. パスワードの長さチェック
        if (password == null || password.length() < 8) {
            errors.add("password",
                new ActionMessage("error.password.minlength"));
        }

        return errors;  // エラーがなければ空のActionErrors
    }
}

ActionErrorsの使い方

  • errors.add(プロパティ名, ActionMessage) でエラーを追加
  • プロパティ名は、どのフィールドのエラーかを示す（例: "username", "email"）
  • ActionMessageにはメッセージキーを指定（後述のプロパティファイルで定義）
  • エラーがない場合は空のActionErrorsを返す（nullは不可）

メッセージリソースの定義

エラーメッセージはApplicationResources.propertiesに定義します。

ApplicationResources.properties 
# ユーザー名エラー
error.username.required=ユーザー名を入力してください。

# メールアドレスエラー
error.email.required=メールアドレスを入力してください。
error.email.invalid=メールアドレスの形式が正しくありません。

# パスワードエラー
error.password.minlength=パスワードは8文字以上で入力してください。

7.3 html:errorsタグでエラー表示

JSP側では<html:errors>タグを使ってエラーメッセージを表示します。

全体エラーの表示

register.jsp 
<%@ page contentType="text/html;charset=UTF-8" %>
<%@ taglib uri="http://struts.apache.org/tags-html" prefix="html" %>
<html>
<head>
    <title>ユーザー登録</title>
    <style>
        .error { color: red; font-weight: bold; }
    </style>
</head>
<body>
    <h1>ユーザー登録</h1>

    <!-- 全エラーを一覧表示 -->
    <html:errors />

    <html:form action="/register">
        ユーザー名: <html:text property="username" /><br/>
        メール: <html:text property="email" /><br/>
        パスワード: <html:password property="password" /><br/>
        <html:submit value="登録" />
    </html:form>
</body>
</html>

フィールドごとのエラー表示

property属性を指定すると、特定フィールドのエラーだけを表示できます。

register.jsp（フィールドごと） 
<html:form action="/register">
    ユーザー名: <html:text property="username" />
    <html:errors property="username" /><br/>

    メール: <html:text property="email" />
    <html:errors property="email" /><br/>

    パスワード: <html:password property="password" />
    <html:errors property="password" /><br/>

    <html:submit value="登録" />
</html:form>

ヒント

フィールドごとの表示を使うと、ユーザーはどの入力欄に問題があるかすぐに分かります。 ユーザビリティ向上のため、フィールドごとの表示を推奨します。

実習 7-1: validateメソッドの実装

商品登録フォームに入力検証を追加してみましょう。

手順

  2. 2 ApplicationResources.propertiesにエラーメッセージを定義
  3. 3 JSPでhtml:errorsタグを使ってエラー表示
  4. 4 わざとエラーになる値を入力してエラーメッセージが表示されることを確認

7.4 Validatorフレームワーク

validateメソッドで手動検証を書くのは柔軟ですが、同じような検証コードが増えると保守性が下がります。 Validatorフレームワークを使うと、XMLで宣言的に検証ルールを定義できます。

Validatorのメリット

  • Javaコードを書かずに検証ルールを定義できる
  • よく使う検証（必須、長さ、形式など）が最初から用意されている
  • クライアント側のJavaScript検証も自動生成可能
  • 検証ルールの一元管理で保守性向上

Validatorのセットアップ

Validatorを使用するには、以下のファイルが必要です。

必要なファイル 

WEB-INF/
├── validator-rules.xml       ← 標準検証ルール定義（ライブラリ付属）
├── validation.xml            ← アプリ固有の検証ルール
└── struts-config.xml         ← Validatorプラグイン設定

struts-config.xmlにプラグイン追加

struts-config.xml 
<struts-config>
    <!-- form-beans, action-mappings などの設定... -->

    <!-- Validatorプラグイン設定 -->
    <plug-in className="org.apache.struts.validator.ValidatorPlugIn">
        <set-property
            property="pathnames"
            value="/WEB-INF/validator-rules.xml,
                   /WEB-INF/validation.xml"/>
    </plug-in>
</struts-config>

validation.xmlで検証ルール定義

validation.xml 
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE form-validation PUBLIC
  "-//Apache Software Foundation//DTD Commons Validator Rules Configuration 1.1//EN"
  "http://jakarta.apache.org/commons/dtds/validator_1_1.dtd">

<form-validation>
    <formset>
        <!-- UserFormの検証ルール -->
        <form name="userForm">
            <!-- ユーザー名: 必須 -->
            <field property="username" depends="required">
                <arg key="label.username"/>
            </field>

            <!-- メール: 必須＋形式チェック -->
            <field property="email" depends="required,email">
                <arg key="label.email"/>
            </field>

            <!-- パスワード: 必須＋最小長 -->
            <field property="password" depends="required,minlength">
                <arg0 key="label.password"/>
                <arg1 name="minlength" key="${var:minlength}" resource="false"/>
                <var>
                    <var-name>minlength</var-name>
                    <var-value>8</var-value>
                </var>
            </field>
        </form>
    </formset>
</form-validation>

主な属性

  • name: ActionFormのform-beans名と一致させる
  • property: 検証対象のプロパティ名
  • depends: 使用する検証ルール（カンマ区切りで複数指定可）
  • arg: エラーメッセージのプレースホルダ

よく使う検証ルール

  • required: 必須入力
  • minlength/maxlength: 長さ制限
  • email: メール形式チェック
  • integer/float: 数値チェック
  • date: 日付形式チェック
  • mask: 正規表現パターンマッチ

ValidatorFormまたはValidatorActionFormを継承

Validatorを使用する場合、ActionFormの代わりにValidatorFormを継承します。

UserForm.java（Validator使用版） 
package com.example.form;

import org.apache.struts.validator.ValidatorForm;

public class UserForm extends ValidatorForm {
    private String username;
    private String email;
    private String password;

    // getter/setterのみ
    // validateメソッドは不要（XMLで定義）

    public String getUsername() { return username; }
    public void setUsername(String username) {
        this.username = username;
    }

    public String getEmail() { return email; }
    public void setEmail(String email) {
        this.email = email;
    }

    public String getPassword() { return password; }
    public void setPassword(String password) {
        this.password = password;
    }
}

注意

ValidatorFormを使う場合、手動でvalidateメソッドをオーバーライドする必要はありません。 XMLの設定だけで検証が実行されます。ただし、複雑な業務ルール検証は手動実装が必要な場合もあります。

7.5 ActionMessagesで成功メッセージ

エラーだけでなく、成功メッセージも表示したい場合は ActionMessagesを使います。

Actionクラスでメッセージを追加

RegisterAction.java 
public class RegisterAction extends Action {
    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response) throws Exception {

        UserForm userForm = (UserForm) form;

        // ユーザー登録処理（省略）

        // 成功メッセージを追加
        ActionMessages messages = new ActionMessages();
        messages.add(ActionMessages.GLOBAL_MESSAGE,
            new ActionMessage("message.register.success",
                userForm.getUsername()));
        saveMessages(request, messages);

        return mapping.findForward("success");
    }
}

JSPでhtml:messagesタグ

success.jsp 
<%@ taglib uri="http://struts.apache.org/tags-html" prefix="html" %>
<html>
<head>
    <style>
        .message { color: green; font-weight: bold; }
    </style>
</head>
<body>
    <h1>登録完了</h1>

    <!-- 成功メッセージ表示 -->
    <html:messages id="msg" message="true">
        <div class="message">
            <bean:write name="msg"/>
        </div>
    </html:messages>

    <a href="index.jsp">トップへ</a>
</body>
</html>

html:errors（エラー用）

  • <html:errors /> で使用
  • ActionErrorsオブジェクト
  • 赤色など目立つ色で表示

html:messages（成功メッセージ用）

  • <html:messages message="true" /> で使用
  • ActionMessagesオブジェクト
  • 緑色など肯定的な色で表示

トラブルシューティング

エラーメッセージが表示されない

原因: ApplicationResources.propertiesのメッセージキーが間違っている

解決方法: validation.xmlやActionMessageで指定したキーとプロパティファイルのキーが一致しているか確認してください。

Validatorが動作しない

原因: struts-config.xmlのプラグイン設定が正しくない、またはFormがValidatorFormを継承していない

解決方法: ValidatorPlugInの設定とActionFormの継承クラスを確認してください。

日本語のエラーメッセージが文字化けする

原因: プロパティファイルがUTF-8で保存されていない

解決方法: ApplicationResources.propertiesをISO-8859-1エンコードで保存するか、native2asciiコマンドでエスケープしてください。またはStruts 1.3以降でUTF-8対応の設定を追加します。

理解度確認クイズ

  1. Q1. ActionFormのvalidateメソッドが返すオブジェクトの型は何ですか？

    回答を表示

    ActionErrorsです。エラーがない場合でも空のActionErrorsオブジェクトを返す必要があります（nullは不可）。

  2. Q2. Validatorフレームワークで必須入力チェックを行う場合、dependsに指定する値は何ですか？

    回答を表示

    requiredです。validation.xmlでdepends="required"と指定します。

  3. Q3. JSPでフィールドごとにエラーメッセージを表示したい場合、html:errorsタグに指定する属性は何ですか？

    回答を表示

    property属性です。例: <html:errors property="username" />とすると、usernameフィールドのエラーだけが表示されます。

第7章のまとめ

  • 入力検証はセキュリティとデータ整合性のために必須。必ずサーバー側で実装する
  • ActionFormのvalidateメソッドで手動検証が可能。ActionErrorsにエラーを追加して返す
  • html:errorsタグでエラーメッセージを表示。property属性でフィールドごとの表示が可能
  • Validatorフレームワークを使うとXMLで宣言的に検証ルールを定義でき、コード量を削減できる
  • ActionMessagesを使うと成功メッセージも表示できる。html:messagesタグで表示
