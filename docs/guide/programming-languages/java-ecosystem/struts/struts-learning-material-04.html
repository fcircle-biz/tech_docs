第4章

ActionFormの基礎 - フォームデータの管理

HTMLフォームから送信されたデータを受け取るActionFormの作成方法を学びます。

この章で学ぶこと

  • ActionFormを継承してフォームBeanを作成できる
  • フォームの入力値をActionFormで受け取れる
  • DynaActionFormの使い方を理解できる

4.1 ActionFormとは何か

ActionFormは、HTMLフォームから送信されたデータを受け取るためのJavaBeansクラスです。 Strutsでは、ユーザーが入力した値を直接Actionクラスで受け取るのではなく、ActionFormという専用のクラスを経由します。

これにより、以下のメリットがあります：

  • データの一時保管 - フォーム入力値をオブジェクトとして管理できる
  • バリデーション - 入力値の検証をActionFormで実施できる
  • 再利用性 - 同じフォームを複数のActionで使い回せる
  • 型安全性 - 文字列だけでなく、数値や日付型でデータを扱える

JavaBeansとは？

JavaBeansは、プロパティ（フィールド）に対してgetter/setterメソッドを持つJavaクラスの規約です。 Strutsはこの規約に従うことで、自動的にフォームデータをActionFormのプロパティにセットします。

4.2 ActionFormの作成方法

ActionFormクラスを作成するには、org.apache.struts.action.ActionFormを継承します。 例として、ログインフォームのActionFormを作成してみましょう。

LoginForm.java 
package com.example.form;

import org.apache.struts.action.ActionForm;

// ActionFormを継承してフォームBeanを作成
public class LoginForm extends ActionForm {

    // プロパティ（フィールド）を定義
    private String username;  // ユーザー名
    private String password;  // パスワード

    // getter/setterメソッド（必須）
    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }
}

ActionFormの基本ルール

  • • ActionFormを継承する
  • • プロパティはprivateで定義
  • • 各プロパティにgetter/setterを用意
  • • 引数なしのコンストラクタが必要（省略可）

命名規約のポイント

  • • プロパティ名とHTMLのname属性を一致させる
  • • getter: get + プロパティ名（先頭大文字）
  • • setter: set + プロパティ名（先頭大文字）
  • • boolean型はis + プロパティ名も可

4.3 HTMLフォームとの連携

ActionFormのプロパティとHTMLのinput要素のname属性を一致させることで、自動的にデータがマッピングされます。

login.jsp 
<%@ taglib uri="http://struts.apache.org/tags-html" prefix="html" %>

<html:form action="/login">
    <!-- name="username"がLoginFormのusernameプロパティに対応 -->
    <html:text property="username" />

    <!-- name="password"がLoginFormのpasswordプロパティに対応 -->
    <html:password property="password" />

    <html:submit value="ログイン" />
</html:form>

重要ポイント

HTMLのname属性とActionFormのプロパティ名は完全一致させる必要があります。 大文字・小文字も区別されるため、注意しましょう。

4.4 resetメソッドの理解

ActionFormにはreset()メソッドがあり、フォームの初期化処理を記述できます。 特にチェックボックスの扱いで重要になります。

UserForm.java 
package com.example.form;

import javax.servlet.http.HttpServletRequest;
import org.apache.struts.action.ActionForm;
import org.apache.struts.action.ActionMapping;

public class UserForm extends ActionForm {

    private String name;
    private boolean subscribe;  // メール購読の有無

    // リクエスト処理前に呼ばれる初期化メソッド
    @Override
    public void reset(ActionMapping mapping, HttpServletRequest request) {
        // チェックボックスは未選択時にパラメータが送信されないため
        // falseで初期化する必要がある
        this.subscribe = false;
    }

    // getter/setter
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public boolean isSubscribe() { return subscribe; }
    public void setSubscribe(boolean subscribe) { this.subscribe = subscribe; }
}

チェックボックスの注意点

HTMLのチェックボックスは、チェックされていないとパラメータが送信されません。 そのため、resetメソッドでfalseに初期化しないと、前回の値が残り続けてしまいます。

4.5 DynaActionForm - 設定ファイルでフォームを定義

DynaActionFormは、Javaクラスを作成せずに、 struts-config.xmlの設定だけでActionFormを定義できる仕組みです。 シンプルなフォームの場合、コード量を削減できます。

struts-config.xml 
<form-beans>
    <!-- DynaActionFormを使った定義 -->
    <form-bean name="loginForm"
               type="org.apache.struts.action.DynaActionForm">
        <!-- プロパティを設定ファイルで定義 -->
        <form-property name="username" type="java.lang.String" />
        <form-property name="password" type="java.lang.String" />
    </form-bean>
</form-beans>

DynaActionFormの利点

  • • Javaクラスを書かなくて良い
  • • シンプルなフォームで開発が速い
  • • 設定を見ればプロパティが分かる

DynaActionFormの欠点

  • • 複雑な初期化処理ができない
  • • IDEの補完が効かない
  • • 大規模フォームには不向き

使い分けの目安

単純なフォーム（2〜3項目程度）→ DynaActionFormを検討
複雑なフォーム（バリデーション、初期化ロジックあり）→ 通常のActionFormを使う

実習 4-1: 会員登録フォームの作成

会員登録用のActionFormを作成し、HTMLフォームとの連携を確認しましょう。

手順

  1. 1 RegisterForm.javaを作成し、以下のプロパティを定義： name、 email、 age（int型）
  2. 2 各プロパティにgetter/setterメソッドを実装
  3. 3 resetメソッドをオーバーライドし、age=0で初期化
  4. 4 ActionでRegisterFormを受け取り、各値をログ出力して確認

確認ポイント： フォームに入力した値が正しくActionFormに格納され、Actionで参照できることを確認しましょう。

理解度確認クイズ

  1. Q1. ActionFormのプロパティ名とHTMLフォームのname属性の関係について正しいものはどれですか？

    回答を表示

    完全一致させる必要があります。 HTMLのname="username"は、 ActionFormのusernameプロパティ（setUsername/getUsernameメソッド）に対応します。 大文字・小文字も区別されます。

  2. Q2. resetメソッドはいつ呼ばれますか？また、なぜチェックボックスの初期化が必要ですか？

    回答を表示

    resetメソッドはリクエスト処理の前に呼ばれます。 チェックボックスは未選択時にHTTPパラメータが送信されないため、 resetメソッドでfalseに初期化しないと、前回チェックした値が残り続けてしまいます。

  3. Q3. DynaActionFormの主な利点と欠点をそれぞれ1つ挙げてください。

    回答を表示

    利点： Javaクラスを作成せずに、設定ファイル（struts-config.xml）だけでActionFormを定義できる。
    欠点： 複雑な初期化処理やバリデーションロジックが書けない、IDEの補完機能が効かない。

第4章のまとめ

  • ActionFormはHTMLフォームのデータを受け取る専用のJavaBeansクラス
  • プロパティはprivateで定義し、public getter/setterメソッドを用意する
  • HTMLのname属性とActionFormのプロパティ名を完全一致させることで自動マッピングされる
  • resetメソッドはリクエスト前に呼ばれる初期化処理で、チェックボックスの扱いに重要
  • DynaActionFormは設定ファイルでフォームを定義でき、シンプルなケースで便利
