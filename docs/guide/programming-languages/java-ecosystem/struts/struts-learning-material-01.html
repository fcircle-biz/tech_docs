1.1 フレームワークとは何か

Webアプリケーションを開発する際、毎回ゼロからすべてのコードを書くのは非効率的です。 フレームワーク（Framework）とは、アプリケーション開発の「枠組み」を提供するソフトウェアです。

例えば、家を建てる時に基礎や柱の部分がすでに用意されていて、自分は間取りや内装を考えるだけで済むような状態です。 フレームワークを使うことで、以下のメリットがあります：

  • 開発スピード向上 - よく使う機能が最初から用意されている
  • 品質の安定 - テスト済みのコードを使えるため、バグが減る
  • コードの統一性 - チーム開発でコードの書き方が揃う
  • 保守性の向上 - 決まった構造なので、後から見直しやすい

1.2 MVCパターンの基礎

MVC（Model-View-Controller）パターンは、アプリケーションを3つの役割に分けて設計する方法です。 これにより、コードの見通しが良くなり、変更に強いシステムが作れます。

Model

データとビジネスロジックを担当。データベース操作や計算処理など、アプリケーションの核心部分を扱います。

例: 商品情報の取得、在庫計算

View

画面表示を担当。ユーザーに見せる部分で、HTMLやJSPで作成します。データの表示だけに専念します。

例: 商品一覧画面、詳細表示

Controller

処理の制御を担当。ユーザーの操作を受け取り、ModelとViewをつなぐ交通整理役です。

例: 検索リクエストの処理、画面遷移

なぜ分けるのか？

例えば、「商品一覧」をスマホでも見られるようにしたい場合、MVCならViewだけをスマホ用に作り直せばOK。 ModelとControllerは変更不要です。これが「関心の分離」のメリットです。

1.3 Apache Struts 1.xとは

Apache Strutsは、2000年代初頭に登場したJavaのWebアプリケーションフレームワークです。 MVCパターンをJavaのServlet/JSP技術で実現した先駆的な存在で、多くの企業システムで採用されました。

Struts 1.xの特徴

  • 設定ファイル（XML）でアプリケーションを制御
  • Actionクラスで処理を実装
  • ActionFormでフォームデータを管理
  • Strutsタグライブラリで画面を構築

重要な注意事項

Struts 1.xは2013年にEOL（End of Life）を迎えています。 新規開発での使用は推奨されません。

主要コンポーネント

  • ActionServlet: すべてのリクエストを受け取る入口
  • struts-config.xml: 設定を記述したファイル
  • ActionForm: フォームデータを保持するBean
  • Action: 処理を実装するController

処理の流れ（簡略版）

  1. ユーザーが画面で操作
  2. ActionServletがリクエスト受信
  3. 設定ファイルで対応するActionを特定
  4. Actionが処理を実行
  5. JSP（View）で結果を表示

理解度確認クイズ

  1. Q1. MVCパターンの「V」は何の略で、どのような役割を持ちますか？

    回答を表示

    View（ビュー）の略です。画面表示を担当し、ユーザーにデータを見せる役割を持ちます。 Strutsでは主にJSP（JavaServer Pages）で実装します。

  2. Q2. Strutsで全てのリクエストを最初に受け取るコンポーネントの名前は何ですか？

    回答を表示

    ActionServletです。すべてのHTTPリクエストを受け取り、設定ファイル（struts-config.xml）を参照して 適切なActionクラスに処理を振り分けます。

  3. Q3. フレームワークを使う主なメリットを2つ挙げてください。

    回答を表示

    例：
    1. 開発スピードの向上 - よく使う機能が最初から用意されているため、ゼロから実装する必要がありません。
    2. コードの統一性 - 決まった書き方に従うため、チーム開発でコードの品質が揃います。

第1章のまとめ

  • フレームワークはアプリケーション開発の枠組みを提供し、開発効率と品質を向上させる
  • MVCパターンはModel（データ）、View（表示）、Controller（制御）の3つに役割を分ける設計手法
  • Struts 1.xはMVCパターンを実現するJava Webフレームワークで、ActionServlet、Action、ActionForm、JSPで構成される
  • リクエスト処理はActionServlet → 設定ファイル参照 → Action実行 → JSP表示の流れで進む
  • Struts 1.xはEOL済みのため、学習目的やレガシーシステムの保守に活用する
