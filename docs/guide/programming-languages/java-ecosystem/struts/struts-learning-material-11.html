国際化対応（i18n）の基礎

多言語対応アプリケーションの実装方法を学び、リソースバンドルとロケール切り替えを理解しましょう。

この章で学ぶこと

  • 国際化（i18n）の概念を理解できる
  • 言語別のプロパティファイルを作成できる
  • ロケール切り替え機能を実装できる
  • bean:messageタグでメッセージを表示できる

11.1 国際化（i18n）とは

国際化（Internationalization、i18n）とは、アプリケーションを複数の言語や地域に対応させることです。 「i18n」という略語は、Internationalizationの「i」と「n」の間に18文字あることに由来します。

国際化対応により、日本語版・英語版・中国語版などを別々に開発する必要がなくなり、1つのソースコードで多言語対応が可能になります。

国際化のメリット

  • ソースコードに言語依存の文字列を書かない
  • 言語追加時にコード変更が不要
  • 翻訳者がプログラムを触らずに作業可能

Strutsでの実現方法

  • 言語別のプロパティファイルを用意
  • bean:messageタグでメッセージ表示
  • ロケールに応じて自動切り替え

ロケールとは？

ロケール（Locale）は、言語や地域の設定を表します。例えば「ja_JP」（日本語・日本）、「en_US」（英語・アメリカ）、「zh_CN」（中国語・中国）などがあります。 Strutsはユーザーのロケール設定に応じて、適切なメッセージファイルを自動的に選択します。

11.2 リソースバンドルの基礎

リソースバンドル（Resource Bundle）は、言語別のメッセージを管理する仕組みです。 Strutsでは、プロパティファイルにメッセージを定義し、キーを使ってメッセージを取得します。

プロパティファイルの命名規則 

ApplicationResources.properties         ← デフォルト（英語）
ApplicationResources_ja.properties      ← 日本語
ApplicationResources_en.properties      ← 英語（明示的）
ApplicationResources_zh_CN.properties   ← 中国語（簡体字）
ApplicationResources_fr.properties      ← フランス語

ファイル名の末尾に「_ロケール」を付けることで、言語ごとのファイルを区別します。 ロケールが見つからない場合は、デフォルトのファイル（サフィックスなし）が使用されます。

実習 11-1: プロパティファイルの作成

日本語版と英語版のプロパティファイルを作成します。

手順1: デフォルトプロパティファイルの作成

src/main/resources/ に ApplicationResources.properties を作成します。

ApplicationResources.properties 
# デフォルトメッセージ（英語）
app.title=Task Management System
app.welcome=Welcome to Task Manager!
menu.home=Home
menu.tasks=Tasks
menu.logout=Logout
error.required={0} is required.
error.invalid={0} is invalid.

手順2: 日本語プロパティファイルの作成

同じフォルダに ApplicationResources_ja.properties を作成します。

ApplicationResources_ja.properties 
# 日本語メッセージ
app.title=タスク管理システム
app.welcome=タスクマネージャーへようこそ！
menu.home=ホーム
menu.tasks=タスク一覧
menu.logout=ログアウト
error.required={0}は必須項目です。
error.invalid={0}の形式が正しくありません。

文字コードに注意

プロパティファイルは通常「ISO-8859-1」エンコーディングで保存する必要があります。 日本語を含む場合は、native2asciiツールでUnicodeエスケープするか、Eclipseなどの「Properties Editor」プラグインを使用してください。 または、struts.propertiesで「struts.i18n.encoding=UTF-8」を設定すればUTF-8で直接記述できます。

11.3 struts-config.xmlでのリソース設定

作成したプロパティファイルをStrutsで使用するため、struts-config.xmlに設定を追加します。

WEB-INF/struts-config.xml 
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE struts-config PUBLIC
    "-//Apache Software Foundation//DTD Struts Configuration 1.3//EN"
    "http://struts.apache.org/dtds/struts-config_1_3.dtd">

<struts-config>
    <!-- ...その他の設定... -->

    <!-- メッセージリソースの設定 -->
    <message-resources
        parameter="ApplicationResources"
        null="false" />

</struts-config>

message-resources要素の属性

  • parameter - プロパティファイルのベース名（パス.拡張子なし）
  • null - キーが見つからない場合の動作（false=エラー、true=???key???と表示）

11.4 bean:messageタグでメッセージを表示する

JSPでbean:messageタグを使用すると、プロパティファイルのメッセージを表示できます。

index.jsp 
<%@ taglib uri="/tags/struts-bean" prefix="bean" %>
<%@ taglib uri="/tags/struts-html" prefix="html" %>
<html:html>
<head>
    <!-- タイトルをリソースから取得 -->
    <title><bean:message key="app.title" /></title>
</head>
<body>
    <h1><bean:message key="app.welcome" /></h1>
    <nav>
        <a href="home.do"><bean:message key="menu.home" /></a> |
        <a href="tasks.do"><bean:message key="menu.tasks" /></a> |
        <a href="logout.do"><bean:message key="menu.logout" /></a>
    </nav>
</body>
</html:html>

ユーザーのブラウザの言語設定が日本語なら日本語メッセージが、英語なら英語メッセージが表示されます。

パラメータ付きメッセージ

メッセージに動的な値を埋め込む場合は、プレースホルダー（{0}、{1}など）を使用します。

error.jsp 
<!-- プレースホルダーに値を渡す -->
<bean:message key="error.required" arg0="ユーザー名" />
<!-- 結果: 「ユーザー名は必須項目です。」 -->

<bean:message key="error.invalid" arg0="メールアドレス" />
<!-- 結果: 「メールアドレスの形式が正しくありません。」 -->

11.5 ロケールの切り替え機能を実装する

ユーザーが言語を自由に切り替えられるよう、ロケール変更機能を実装します。

ロケール切り替えの流れ

flowchart LR A([ユーザー]) -->|言語選択| B[ChangeLocaleAction] B -->|新しいLocale設定| C[セッションに保存] C -->|次回リクエスト| D[Strutsがロケール読取] D -->|適切なファイル選択| E[メッセージ表示] style B fill:#fed7aa,stroke:#f97316,stroke-width:2px style C fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style E fill:#a7f3d0,stroke:#10b981,stroke-width:2px

実習 11-2: ロケール切り替え機能の実装

手順1: ChangeLocaleActionの作成

ChangeLocaleAction.java 
package com.example.action;

import java.util.Locale;
import javax.servlet.http.*;
import org.apache.struts.Globals;
import org.apache.struts.action.*;

public class ChangeLocaleAction extends Action {

    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response
    ) throws Exception {

        // リクエストパラメータから言語コードを取得
        String language = request.getParameter("lang");

        // Localeオブジェクトを作成
        Locale locale = new Locale(language);

        // セッションにロケールを保存
        HttpSession session = request.getSession();
        session.setAttribute(Globals.LOCALE_KEY, locale);

        // 元のページにリダイレクト
        return mapping.findForward("success");
    }
}

手順2: struts-config.xmlに設定追加

struts-config.xml（抜粋） 
<action-mappings>
    <action
        path="/changeLocale"
        type="com.example.action.ChangeLocaleAction">
        <forward name="success" path="/index.jsp" />
    </action>
</action-mappings>

手順3: 言語切り替えリンクをJSPに追加

index.jsp（言語切り替え部分） 
<div class="language-selector">
    <a href="changeLocale.do?lang=ja">日本語</a> |
    <a href="changeLocale.do?lang=en">English</a> |
    <a href="changeLocale.do?lang=zh">中文</a>
</div>

重要ポイント

Globals.LOCALE_KEY は Struts が標準で使用するセッション属性のキーです。 このキーでロケールを保存することで、Strutsが自動的にロケールを認識し、適切なメッセージファイルを選択します。

11.6 動作確認

動作確認

アプリケーションを起動し、言語切り替えが正しく動作することを確認します。

確認手順

  1. 1 Tomcatを起動し、http://localhost:8080/yourapp/ にアクセス
  2. 2 デフォルトでは英語メッセージが表示される（ブラウザ設定による）
  3. 3 「日本語」リンクをクリックすると、すべてのメッセージが日本語に切り替わる
  4. 4 「English」リンクで英語に戻る

期待される結果 

日本語選択時:
タイトル: タスク管理システム
メッセージ: タスクマネージャーへようこそ！

English選択時:
Title: Task Management System
Message: Welcome to Task Manager!

11.7 実践的なテクニック

フォールバックの仕組み

Javaのリソースバンドルには、言語ファイルが見つからない場合のフォールバック機能があります。

リソースファイルの検索順序 

例: ユーザーのロケールが「zh_CN」（中国語・中国）の場合

1. ApplicationResources_zh_CN.properties  ← 最優先
2. ApplicationResources_zh.properties     ← 国のみ省略
3. ApplicationResources.properties        ← デフォルト

この順序で検索し、最初に見つかったファイルを使用します。 これにより、すべての言語でファイルを用意しなくても、デフォルトの言語で表示できます。

メッセージのモジュール化

大規模なアプリケーションでは、機能ごとにプロパティファイルを分けると管理しやすくなります。

struts-config.xml（複数リソース） 
<!-- 共通メッセージ -->
<message-resources
    parameter="ApplicationResources"
    key="common"
    null="false" />

<!-- タスク機能専用メッセージ -->
<message-resources
    parameter="TaskResources"
    key="task"
    null="false" />

JSPではbundle属性でリソースを指定します。

JSPでのモジュール指定 
<!-- 共通リソースから取得 -->
<bean:message key="app.title" bundle="common" />

<!-- タスクリソースから取得 -->
<bean:message key="task.list.title" bundle="task" />

11.8 トラブルシューティング

よくあるトラブルと解決方法

メッセージが「???key???」と表示される

原因: プロパティファイルにキーが存在しない、またはstruts-config.xmlの設定が間違っている

解決方法:

  • プロパティファイルにキーが正しく定義されているか確認
  • struts-config.xmlのparameter属性がファイル名（拡張子なし）と一致しているか確認
  • プロパティファイルがクラスパス上にあるか確認（通常はsrc/main/resources/）
日本語が文字化けする

原因: プロパティファイルのエンコーディングが正しくない

解決方法:

  • native2asciiツールでUnicodeエスケープする
  • Eclipseの「Properties Editor」プラグインを使用する
  • web.xmlでフィルタを設定してUTF-8対応にする
言語を切り替えても変わらない

原因: セッションにロケールが正しく保存されていない

解決方法:

  • ChangeLocaleActionでGlobals.LOCALE_KEYを使用しているか確認
  • セッションが正しく生成されているか確認（request.getSession()が呼ばれているか）
  • ブラウザのキャッシュをクリアして再試行

理解度確認クイズ

  1. Q1. 国際化（i18n）の「i18n」という略語は何に由来しますか？

    回答を表示

    Internationalizationという単語の「i」と「n」の間に18文字あることに由来します。 同様に、l10n（Localization）は「l」と「n」の間に10文字あることから名付けられています。

  2. Q2. 日本語版のプロパティファイル名は何になりますか？（ベース名がApplicationResourcesの場合）

    回答を表示

    ApplicationResources_ja.propertiesです。 ベース名の末尾に「_ロケールコード」を付加します。日本語のロケールコードは「ja」です。

  3. Q3. Strutsでセッションにロケールを保存する際に使用する標準のキーは何ですか？

    回答を表示

    Globals.LOCALE_KEYです。 このキーを使用することで、Strutsが自動的にロケールを認識し、適切なメッセージファイルを選択します。

第11章のまとめ

  • 国際化（i18n）により、1つのソースコードで多言語対応アプリケーションを構築できる
  • プロパティファイルに言語別のメッセージを定義し、ロケールに応じて切り替える
  • bean:messageタグでJSPからメッセージを表示し、パラメータを埋め込むこともできる
  • Globals.LOCALE_KEYを使ってセッションにロケールを保存することで、ユーザーごとの言語設定を維持できる
  • リソースファイルの検索はフォールバック機能により、存在しないロケールの場合もデフォルトで表示される
