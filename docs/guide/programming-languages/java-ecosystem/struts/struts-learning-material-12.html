CREATE TABLE tasks (
    task_id INT PRIMARY KEY AUTO_INCREMENT,

UserDAO.java（抜粋） 
package com.example.task.dao;

public class UserDAO {
    // ログイン認証
    public User authenticate(String username, String password) {
        Connection conn = null;
        try {
            conn = DBUtil.getConnection();
            String sql = "SELECT * FROM users WHERE username = ? AND password = ?";
            PreparedStatement stmt = conn.prepareStatement(sql);
            stmt.setString(1, username);
            stmt.setString(2, password);
            ResultSet rs = stmt.executeQuery();

            if (rs.next()) {
                User user = new User();
                user.setUserId(rs.getInt("user_id"));
                user.setUsername(rs.getString("username"));
                // 他のフィールドも設定
                return user;
            }
        } finally {
            DBUtil.close(conn);
        }
        return null;  // 認証失敗
    }
}

package com.example.task.form;

public class LoginForm extends ActionForm {
    private String username;
    private String password;

    // validate メソッド（入力検証）
    public ActionErrors validate(...) {
        ActionErrors errors = new ActionErrors();
        if (username == null || username.isEmpty()) {
            errors.add("username",
                new ActionMessage("error.username.required"));
        }
        if (password == null || password.isEmpty()) {
            errors.add("password",
                new ActionMessage("error.password.required"));
        }
        return errors;
    }

    // getter/setter
}

ステップ4: Actionクラスの作成

Controller役を担うActionクラスを実装します。

LoginAction.java（抜粋） 
package com.example.task.action;

public class LoginAction extends Action {
    public ActionForward execute(...) {
        LoginForm form = (LoginForm) actionForm;
        UserDAO dao = new UserDAO();

        // 認証処理
        User user = dao.authenticate(form.getUsername(), form.getPassword());

        if (user != null) {
            // セッションにユーザー情報を保存
            request.getSession().setAttribute("loginUser", user);
            return mapping.findForward("success");
        } else {
            // エラーメッセージを設定
            ActionErrors errors = new ActionErrors();
            errors.add("login", new ActionMessage("error.login.failed"));
            saveErrors(request, errors);
            return mapping.findForward("failure");
        }
    }
}
TaskListAction.java（抜粋） 
package com.example.task.action;

public class TaskListAction extends Action {
    public ActionForward execute(...) {
        // セッションからログインユーザー取得
        User user = (User) request.getSession().getAttribute("loginUser");

        if (user == null) {
            // 未ログインの場合はログイン画面へ
            return mapping.findForward("login");
        }

        // タスク一覧取得
        TaskDAO dao = new TaskDAO();
        List<Task> tasks = dao.findByUserId(user.getUserId());
        request.setAttribute("tasks", tasks);

        return mapping.findForward("success");
    }
}

動作確認手順

  1. 1 Tomcatを起動し、アプリケーションをデプロイ
  2. 2 ブラウザで http://localhost:8080/TaskManagementApp/pages/login.jsp にアクセス
  3. 3 ユーザー名: testuser、パスワード: password123 でログイン
  4. 4 タスク一覧画面が表示されることを確認
  5. 5 タスクの作成、編集、削除を試す

期待される動作

  • ログイン認証が正しく動作する
  • ログイン後、ユーザー専用のタスク一覧が表示される
  • タスクの新規作成、更新、削除ができる
  • バリデーションエラーが適切に表示される
  • ログアウト後、セッションがクリアされる

トラブルシューティング

ログイン後にNullPointerException

原因: セッションへのユーザー情報保存が正しく行われていない

解決方法: LoginActionでセッション保存のコード（request.getSession().setAttribute("loginUser", user)）が実装されているか確認してください。

データベース接続エラー

原因: DB接続情報の設定ミス、またはMySQLが起動していない

解決方法: DBUtilクラスの接続URLやユーザー名・パスワードを確認してください。またMySQLサーバーが起動しているか確認してください。

ActionForward not found

原因: struts-config.xmlのforwardタグの設定ミス

解決方法: Actionクラスでreturnしているforward名と、struts-config.xmlの<forward name="...">が一致しているか確認してください。

理解度確認クイズ

  1. Q1. このタスク管理アプリケーションで使われている設計パターンの名前と、その3つの構成要素を答えてください。

    回答を表示

    MVCパターンです。3つの構成要素は：
    1. Model（DAOとEntity） - データベース操作とビジネスロジック
    2. View（JSP） - 画面表示
    3. Controller（Action） - リクエスト処理と画面遷移の制御

  2. Q2. ログイン認証に成功した後、ユーザー情報をどこに保存していますか？その理由も説明してください。

    回答を表示

    セッション（HttpSession）に保存しています。
    理由：ユーザーがログアウトするまで、複数のリクエストをまたいでログイン状態を維持する必要があるためです。 リクエストスコープではページ遷移のたびに情報が消えてしまいます。

  3. Q3. ActionFormのvalidateメソッドの役割は何ですか？

    回答を表示

    フォームから送信されたデータの入力検証（バリデーション）を行います。 必須チェック、文字数チェック、形式チェックなどを実装し、エラーがある場合はActionErrorsオブジェクトに エラー情報を追加して返します。エラーがあると、Actionクラスは実行されずに入力画面に戻ります。
