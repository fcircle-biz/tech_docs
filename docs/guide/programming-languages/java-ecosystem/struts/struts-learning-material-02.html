開発環境の準備と最初のStrutsアプリケーション

実際にStrutsプロジェクトを作成し、「Hello Struts」アプリケーションを動かしてみましょう。

この章で学ぶこと

  • Struts 1.xのライブラリをプロジェクトに追加できる
  • web.xmlにActionServletを設定できる
  • 簡単なStrutsアプリケーションを動作させられる

2.1 前提条件の確認

Struts 1.xアプリケーションを開発する前に、以下の環境が整っていることを確認しましょう。

必須環境

  • JDK 8以降
    Javaコンパイラと実行環境
  • Apache Tomcat 8.x/9.x
    サーブレットコンテナ
  • Eclipse/IntelliJ IDEA
    統合開発環境（IDE）

前提知識

  • Java言語の基本構文
  • Servlet/JSPの基礎
  • XMLの基本構造
  • HTMLフォームの基本

環境確認コマンド

コマンドラインで以下を実行して、JDKがインストールされていることを確認してください：

環境確認 
# JDKバージョン確認
java -version

# Javaコンパイラバージョン確認
javac -version

# 環境変数JAVA_HOMEの確認（Windows）
echo %JAVA_HOME%

# 環境変数JAVA_HOMEの確認（Mac/Linux）
echo $JAVA_HOME

2.2 Strutsライブラリの準備

Struts 1.xを使用するには、必要なJARファイルをプロジェクトに追加する必要があります。

Struts 1.xのダウンロード

Struts 1.xは公式サポートが終了しています。アーカイブサイトや既存のプロジェクトから ライブラリを取得してください。本ガイドではStruts 1.3.10を使用します。

必要なJARファイル

  • struts-core-1.3.10.jar - Strutsのコア機能
  • struts-taglib-1.3.10.jar - Strutsタグライブラリ
  • commons-logging-1.2.jar - ログ出力
  • commons-beanutils-1.9.4.jar - JavaBeanユーティリティ
  • commons-digester-2.1.jar - XML解析
  • commons-validator-1.6.jar - バリデーション機能

2.3 Webプロジェクトの作成

EclipseまたはIntelliJ IDEAで、Dynamic Web Projectを作成します。

実習 2-1: プロジェクト作成（Eclipse）

Eclipseで新しいWebプロジェクトを作成します。

手順

  1. 1 File → New → Dynamic Web Project を選択
  2. 2 Project name: HelloStrutsApp と入力
  3. 3 Target runtime: Apache Tomcat を選択
  4. 4 Finish をクリック
  5. 5 WebContent/WEB-INF フォルダ内に lib フォルダを作成
  6. 6 Struts関連のJARファイルを lib フォルダにコピー

2.4 web.xmlの設定

web.xmlは、Webアプリケーション全体の設定ファイルです。 ここにStrutsのActionServletを登録します。

ActionServletとは？

ActionServletは、すべてのHTTPリクエストを最初に受け取るStrutsのコア部品です。 リクエストURLを見て、どのActionクラスを呼び出すかを判断します。

WebContent/WEB-INF/web.xml 
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
                             http://xmlns.jcp.org/xml/ns/javaee/web-app_3_1.xsd"
         version="3.1">

    <!-- ActionServletの登録 -->
    <servlet>
        <servlet-name>action</servlet-name>
        <servlet-class>org.apache.struts.action.ActionServlet</servlet-class>
        <!-- Struts設定ファイルの場所を指定 -->
        <init-param>
            <param-name>config</param-name>
            <param-value>/WEB-INF/struts-config.xml</param-value>
        </init-param>
        <load-on-startup>1</load-on-startup>
    </servlet>

    <!-- ActionServletへのマッピング（*.doで終わるURLをすべてActionServletに渡す） -->
    <servlet-mapping>
        <servlet-name>action</servlet-name>
        <url-pattern>*.do</url-pattern>
    </servlet-mapping>

    <!-- ウェルカムファイルリスト -->
    <welcome-file-list>
        <welcome-file>index.jsp</welcome-file>
    </welcome-file-list>

</web-app>

設定の意味

  • <servlet>: ActionServletを登録
  • config: struts-config.xmlのパスを指定
  • <url-pattern>: *.doで終わるURLをStrutsで処理
  • load-on-startup: アプリ起動時にServletを初期化

URLパターンの例

  • ✅ /login.do → ActionServletが処理
  • ✅ /search.do → ActionServletが処理
  • ❌ /index.jsp → 直接JSPを表示
  • ❌ /style.css → 静的ファイル

2.5 struts-config.xmlの作成

struts-config.xmlは、Strutsアプリケーションの動作を制御する中心的な設定ファイルです。 URLとActionクラスの対応、画面遷移の設定などをここに記述します。

WebContent/WEB-INF/struts-config.xml 
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE struts-config PUBLIC
    "-//Apache Software Foundation//DTD Struts Configuration 1.3//EN"
    "http://struts.apache.org/dtds/struts-config_1_3.dtd">

<struts-config>
    <!-- ActionFormの定義（後の章で使用） -->
    <form-beans>
    </form-beans>

    <!-- アクションマッピング（URLとActionクラスの対応） -->
    <action-mappings>
        <!-- /hello.do へのアクセスでHelloActionを実行 -->
        <action path="/hello"
                type="com.example.action.HelloAction">
            <!-- 処理成功時はhello.jspへ遷移 -->
            <forward name="success" path="/hello.jsp"/>
        </action>
    </action-mappings>

</struts-config>

設定ファイルの構造

  • <form-beans>: フォームデータを扱うBeanの定義（第4章で学習）
  • <action-mappings>: URLとActionクラスの対応関係
  • <forward>: 処理後の遷移先を定義

HelloAction.java の作成

まず、Controllerの役割を担うActionクラスを作成します。

src/com/example/action/HelloAction.java 
package com.example.action;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import org.apache.struts.action.Action;
import org.apache.struts.action.ActionForm;
import org.apache.struts.action.ActionForward;
import org.apache.struts.action.ActionMapping;

public class HelloAction extends Action {
    // executeメソッドでリクエストを処理
    @Override
    public ActionForward execute(ActionMapping mapping, ActionForm form,
                                 HttpServletRequest request, HttpServletResponse response)
                                 throws Exception {
        // メッセージを生成してリクエストスコープに保存
        String message = "Hello Struts! ようこそStrutsの世界へ！";
        request.setAttribute("message", message);

        // "success"という名前のforwardで画面遷移
        return mapping.findForward("success");
    }
}

executeメソッドの引数

  • mapping: struts-config.xmlの設定情報
  • form: フォームデータ（今回はnull）
  • request: HTTPリクエストオブジェクト
  • response: HTTPレスポンスオブジェクト

処理の流れ

  1. メッセージ文字列を作成
  2. request.setAttributeでデータを保存
  3. mapping.findForwardで遷移先を取得
  4. ActionForwardを返して画面に遷移

hello.jsp の作成

次に、View（画面表示）を担当するJSPファイルを作成します。

WebContent/hello.jsp 
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Hello Struts</title>
    <style>
        body { font-family: sans-serif; padding: 20px; }
        .message { color: #f97316; font-size: 24px; font-weight: bold; }
    </style>
</head>
<body>
    <h1>Struts 1.x 初めてのアプリケーション</h1>
    <!-- Actionからリクエストスコープにセットされたメッセージを表示 -->
    <p class="message"><%= request.getAttribute("message") %></p>
    <p>この画面は、HelloActionから遷移してきました。</p>
    <a href="index.jsp">トップページに戻る</a>
</body>
</html>

トラブルシューティング

HTTP Status 404 - /hello.do が見つからない

原因: web.xmlのActionServletマッピング、またはstruts-config.xmlのアクション定義が正しくない可能性があります。

解決方法:

  • web.xmlで <url-pattern>*.do</url-pattern> が設定されているか確認
  • struts-config.xmlで <action path="/hello" ...> が定義されているか確認
  • URLは /hello.do（.doを付ける）でアクセスしているか確認
ClassNotFoundException: HelloAction

原因: Actionクラスのパッケージ名やクラス名が間違っている、またはコンパイルされていません。

解決方法:

  • struts-config.xmlの type属性が正しいパッケージ名.クラス名になっているか確認
  • HelloAction.javaがコンパイルされているか確認（Eclipseの場合は自動コンパイル）
  • WEB-INF/classes フォルダにクラスファイルが配置されているか確認
メッセージが表示されない（nullと表示される）

原因: ActionクラスでsetAttributeしたキー名と、JSPでgetAttributeするキー名が一致していません。

解決方法:

  • HelloActionで request.setAttribute("message", ...) と設定
  • hello.jspで request.getAttribute("message") と同じキー名で取得
  • キー名のスペルミスに注意（大文字小文字も区別されます）
SAXParseException: struts-config.xmlの解析エラー

原因: struts-config.xmlの文法エラー（XMLの閉じタグ忘れ、属性値のクォート忘れなど）があります。

解決方法:

  • XMLの開始タグと終了タグが正しく対応しているか確認
  • 属性値が " " で囲まれているか確認
  • 特殊文字（< > &）がエスケープされているか確認
  • EclipseのXMLエディタでエラー箇所を確認

理解度確認クイズ

  1. Q1. web.xmlに設定する <url-pattern>*.do</url-pattern> の意味は何ですか？

    回答を表示

    「.do」で終わるURLすべてをActionServletに渡す設定です。 例えば、/login.do、/search.doなどのリクエストが自動的にActionServletで処理されます。

  2. Q2. HelloActionのexecuteメソッドが返す「ActionForward」は何のために使われますか？

    回答を表示

    ActionForwardは、処理後にどの画面（JSP）に遷移するかを指定するために使われます。 mapping.findForward("success")で、struts-config.xmlに定義された"success"という名前のforwardを探し、 対応するJSPに遷移します。

  3. Q3. Strutsライブラリ（JARファイル）はどこに配置する必要がありますか？

    回答を表示

    WebContent/WEB-INF/lib フォルダに配置します。 このフォルダに配置されたJARファイルは、Webアプリケーション実行時に自動的に読み込まれます。

第2章のまとめ

  • Struts開発にはJDK、Tomcat、Eclipse/IntelliJが必要
  • StrutsライブラリJARをWEB-INF/libフォルダに配置する
  • web.xmlにActionServletを登録し、*.doパターンでマッピング
  • struts-config.xmlでURLとActionクラスの対応、遷移先を定義
  • Actionクラスでexecuteメソッドを実装し、処理とActionForwardを返す
  • JSPでrequest.getAttribute()を使ってActionから渡されたデータを表示
