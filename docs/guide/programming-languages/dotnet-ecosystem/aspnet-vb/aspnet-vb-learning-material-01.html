第1章: ASP.NET Web Forms入門と開発環境構築

ASP.NET Web Formsの世界への扉

この章で学ぶこと

  • コンソールアプリケーションとWebアプリケーションの根本的な違いを理解する
  • Visual Studioでの開発環境構築手順を習得する

1.1 ASP.NET Web Formsとは何か

ASP.NET Web Formsは、マイクロソフトが開発したWebアプリケーション開発フレームワークです。VB.NETの知識を活かしながら、Web上で動作する動的なアプリケーションを開発することができます。

ASP.NET Web Formsの歴史と位置づけ

2002年に登場したASP.NET Web Formsは、デスクトップアプリケーション開発者が馴染み深い「イベントドリブンプログラミング」をWeb開発にも持ち込んだ画期的なフレームワークでした。これにより、VB.NETの知識を持つ開発者は比較的容易にWebアプリケーション開発に移行できるようになりました。

sequenceDiagram participant Browser as ブラウザ participant IIS as IISサーバー participant ASPX as ASPXページ participant CodeBehind as VB.NETコード Browser->>IIS: HTTPリクエスト IIS->>ASPX: ページ要求 ASPX->>CodeBehind: イベント処理 CodeBehind->>ASPX: データ処理結果 ASPX->>IIS: HTMLレスポンス生成 IIS->>Browser: HTMLレスポンス

従来のWebアプリケーション開発との違い

従来のWebアプリケーション開発では、HTMLとサーバーサイドスクリプトを直接組み合わせる必要がありました。しかし、ASP.NET Web Formsでは以下の特徴があります：

  • イベントドリブンモデル：ボタンクリックなどのユーザー操作をイベントとして処理
  • サーバーコントロール：HTMLを直接記述せずに、VB.NETのコントロール感覚でUI構築
  • ページライフサイクル：ページの読み込みから表示まで一連の処理フローが自動化
  • 状態管理：Webの「ステートレス」な性質を意識せずに開発可能

1.2 コンソールアプリケーションとWebアプリケーションの本質的な違い

VB.NETでコンソールアプリケーションやWindows Formsアプリケーションを開発した経験がある方も、Webアプリケーションには独特の特徴があることを理解する必要があります。

graph TD A["コンソール/Windows Forms
アプリケーション"] --> B["単一マシンで実行"] A --> C["継続的なメモリ状態"] A --> D["リアルタイム処理"] E["Webアプリケーション"] --> F["複数クライアント対応"] E --> G["リクエスト・レスポンス"] E --> H["ステートレス通信"]

実行環境の違い

コンソール/Windows Forms：
  • プログラムは一つのマシン上で実行される
  • メモリ内の変数や状態は実行中常に保持される
  • ユーザーの操作に即座に反応する
Webアプリケーション：
  • サーバーとクライアント（ブラウザ）が分離している
  • 一つのリクエストが終了すると、その処理での変数や状態は破棄される
  • 複数のユーザーが同時にアクセスする可能性がある

1.3 Visual Studio開発環境の構築

ASP.NET Web Formsの開発には、Visual Studioが最適な開発環境です。以下の手順で環境を構築します。

必要なソフトウェア

  • Visual Studio 2019以上（Community版で十分）
  • .NET Framework 4.7.2以上 または .NET 6/8
  • IIS Express（Visual Studioに含まれる）
  • SQL Server Express LocalDB（データベースを使用する場合）
実習 1-1: 開発環境の確認

Visual Studioが正しくインストールされているか確認し、Web開発に必要なワークロードがインストールされているかチェックしましょう。

手順
  1. Visual Studioを起動
  2. 「新しいプロジェクトを作成」を選択
  3. プロジェクトテンプレートで「ASP.NET Web アプリケーション (.NET Framework)」が選択可能か確認
  4. もし表示されない場合は、Visual Studio Installerから「ASP.NETとWeb開発」ワークロードをインストール

1.5 Web Formsページの基本構造

Web Formsページは、.aspxファイル（マークアップ部分）と.aspx.vbファイル（コードビハインド部分）の組み合わせで構成されます。

.aspxファイル（マークアップ）

HTML要素とサーバーコントロールを組み合わせて、ページの外観を定義します。

<%@ Page Language="VB" AutoEventWireup="false" CodeBehind="Default.aspx.vb" Inherits="MyFirstWebApp.Default" %>

<!DOCTYPE html>
<html>
<head runat="server">
    <title>私の最初のWebページ</title>
</head>
<body>
    <form id="form1" runat="server">
        <div>
            <h1>Hello, ASP.NET Web Forms!</h1>
            <asp:Label ID="lblMessage" runat="server" Text="ここにメッセージが表示されます"></asp:Label>
            <br /><br />
            <asp:Button ID="btnClick" runat="server" Text="クリックしてください" />
        </div>
    </form>
</body>
</html>

.aspx.vbファイル（コードビハインド）

ページのイベント処理やビジネスロジックをVB.NETで記述します。

Public Partial Class Default
    Inherits System.Web.UI.Page

    Protected Sub Page_Load(ByVal sender As Object, ByVal e As System.EventArgs) Handles Me.Load
        ' ページが読み込まれた時に実行される
        If Not Page.IsPostBack Then
            lblMessage.Text = "ページが初回読み込みされました"
        End If
    End Sub

    Protected Sub btnClick_Click(sender As Object, e As EventArgs) Handles btnClick.Click
        ' ボタンがクリックされた時に実行される
        lblMessage.Text = "ボタンがクリックされました！現在時刻: " & DateTime.Now.ToString()
    End Sub
End Class
重要なポイント
  • runat="server"：この属性により、HTML要素がサーバーサイドで制御可能になります
  • Page.IsPostBack：ページが初回読み込みかどうかを判断する重要なプロパティ
  • Handles句：VB.NETの従来のイベント処理構文をそのまま使用可能

1.7 Web.configファイルの基本理解

Web.configファイルは、ASP.NET Webアプリケーションの設定を管理する重要なXMLファイルです。データベース接続文字列、セキュリティ設定、コンパイル設定などがここに記述されます。

<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <appSettings>
    <add key="AppName" value="MyFirstWebApp" />
    <add key="Version" value="1.0.0" />
  </appSettings>
  
  <system.web>
    <compilation debug="true" targetFramework="4.7.2" />
    <httpRuntime targetFramework="4.7.2" />
  </system.web>
</configuration>
理解度確認クイズ
  1. ASP.NET Web Formsの特徴として正しいものはどれですか？
    • a) HTMLを直接記述する必要がある
    • b) イベントドリブンプログラミングモデルを採用している
    • c) サーバーサイドでの処理ができない
    • d) VB.NETの知識を活用できない
  2. コンソールアプリケーションとWebアプリケーションの主な違いは何ですか？
  3. ASPXページにおいて「runat="server"」属性の役割は何ですか？
  4. Page.IsPostBackプロパティは何を判断するために使用しますか？
解答例：
  1. b) イベントドリブンプログラミングモデルを採用している
  2. コンソールアプリケーションは単一マシンで実行され状態を継続的に保持するが、Webアプリケーションはリクエスト・レスポンス形式で複数クライアントに対応し、基本的にステートレス
  3. HTML要素をサーバーサイドで制御可能にし、VB.NETコードから操作できるようにする
  4. ページが初回読み込みか、それとも何らかの操作による再読み込み（PostBack）かを判断する

1.8 まとめ

この章では、ASP.NET Web Formsの基本概念と開発環境構築について学習しました。重要なポイントを再確認しましょう：

  • ASP.NET Web Formsは、VB.NETの知識を活かしてWebアプリケーション開発ができるフレームワーク
  • コンソールアプリケーションとは実行環境や状態管理の考え方が大きく異なる
  • .aspxファイル（マークアップ）と.aspx.vbファイル（コードビハインド）の分離により、デザインとロジックを明確に分けられる
  • Visual Studioの強力なデバッグ機能により、効率的な開発が可能
