第4章: 配列とコレクション

複数データを効率的に管理する

この章で学ぶこと
  • 配列の概念と一次元・多次元配列の使い方を理解する
  • 動的配列の特徴と操作方法を学ぶ
  • ArrayListとList(Of T)の違いと使い分けを習得する
  • Dictionaryを使った键値ペアデータの管理方法を学ぶ
  • コレクションの反復処理とLINQ基礎を理解する
  • 適切なデータ構造の選択基準を身につける

4.1 配列とは

配列は、同じデータ型の複数の値を一つの変数名で管理するデータ構造です。学校のクラスの生徒名簿や、月の売上データなど、関連する複数のデータをまとめて扱う際に非常に有用です。

配列を使用する利点

  • 効率的なデータ管理: 複数の関連データを一つの名前で管理
  • ループ処理との相性: For文などで簡単に全要素を処理
  • メモリ効率: 連続したメモリ領域に格納される
  • インデックスアクセス: 位置番号で直接要素にアクセス可能

4.2 一次元配列

一次元配列は、データを一列に並べた最もシンプルな配列です。インデックス（添え字）は0から始まり、要素数-1までの番号で各要素にアクセスできます。

一次元配列の宣言と初期化

' 配列の宣言（サイズ指定）
Dim numbers(4) As Integer  ' 5つの要素を持つ整数配列（インデックス0-4）

' 宣言と同時に初期化
Dim fruits() As String = {"りんご", "バナナ", "オレンジ", "ぶどう"}

' 要素への代入
numbers(0) = 10
numbers(1) = 20
numbers(2) = 30

' 要素の取得
Dim firstFruit As String = fruits(0)  ' "りんご"
Dim thirdNumber As Integer = numbers(2)  ' 30
実習 4-1: 学生テスト点数の管理

配列を使って5人の学生のテスト点数を管理し、統計情報を計算するプログラムを作成しましょう。

サンプルコード
Module Program
    Sub Main(args As String())
        ' 5人の学生の点数配列
        Dim scores() As Integer = {85, 78, 92, 67, 88}
        Dim studentNames() As String = {"田中", "佐藤", "鈴木", "高橋", "伊藤"}
        
        Console.WriteLine("=== 学生テスト結果 ===")
        
        ' 各学生の点数表示
        For i As Integer = 0 To scores.Length - 1
            Console.WriteLine($"{studentNames(i)}: {scores(i)}点")
        Next
        
        ' 統計計算
        Dim total As Integer = 0
        Dim maxScore As Integer = scores(0)
        Dim minScore As Integer = scores(0)
        Dim maxStudent As String = studentNames(0)
        Dim minStudent As String = studentNames(0)
        
        For i As Integer = 0 To scores.Length - 1
            total += scores(i)
            
            If scores(i) > maxScore Then
                maxScore = scores(i)
                maxStudent = studentNames(i)
            End If
            
            If scores(i) < minScore Then
                minScore = scores(i)
                minStudent = studentNames(i)
            End If
        Next
        
        Dim average As Double = total / scores.Length
        
        Console.WriteLine()
        Console.WriteLine("=== 統計結果 ===")
        Console.WriteLine($"合計点: {total}点")
        Console.WriteLine($"平均点: {average:F1}点")
        Console.WriteLine($"最高点: {maxStudent} {maxScore}点")
        Console.WriteLine($"最低点: {minStudent} {minScore}点")
        
        Console.ReadKey()
    End Sub
End Module

4.3 多次元配列

多次元配列は、表やマトリックスのような二次元以上のデータ構造を表現できます。座席表、月別売上データ、ゲームの盤面など、行と列で構成されるデータを扱う際に使用します。

二次元配列の使い方

' 二次元配列の宣言（3行4列）
Dim matrix(2, 3) As Integer

' 宣言と初期化
Dim salesData(,) As Integer = {{100, 200, 150},   ' 1月の3つの商品売上
                               {120, 180, 200},   ' 2月の売上
                               {90, 220, 170}}    ' 3月の売上

' 要素への代入
matrix(0, 0) = 10  ' 1行1列目
matrix(1, 2) = 25  ' 2行3列目

' 要素の取得
Dim januaryProduct1 As Integer = salesData(0, 0)  ' 1月の商品1の売上

4.4 動的配列とReDim

動的配列は、プログラム実行中にサイズを変更できる配列です。最初にサイズが決まらない場合や、データの追加・削除が頻繁な場合に使用します。 

Module Program
    Sub Main(args As String())
        ' 動的配列の宣言（サイズ未定）
        Dim dynamicArray() As String
        
        ' 初期サイズを設定
        ReDim dynamicArray(2)  ' 3つの要素
        dynamicArray(0) = "初期データ1"
        dynamicArray(1) = "初期データ2"
        dynamicArray(2) = "初期データ3"
        
        Console.WriteLine("初期状態:")
        For i As Integer = 0 To UBound(dynamicArray)
            Console.WriteLine($"  {i}: {dynamicArray(i)}")
        Next
        
        ' サイズを拡張（データを保持）
        ReDim Preserve dynamicArray(4)  ' 5つの要素に拡張
        dynamicArray(3) = "追加データ1"
        dynamicArray(4) = "追加データ2"
        
        Console.WriteLine()
        Console.WriteLine("拡張後:")
        For i As Integer = 0 To UBound(dynamicArray)
            Console.WriteLine($"  {i}: {dynamicArray(i)}")
        Next
        
        Console.ReadKey()
    End Sub
End Module

4.5 ArrayList

ArrayListは、.NET Frameworkで提供される動的なコレクションです。サイズが自動的に調整され、異なる型のデータも格納できますが、型安全性がない点に注意が必要です。 

Imports System.Collections

Module Program
    Sub Main(args As String())
        ' ArrayListの作成
        Dim arrayList As New ArrayList()
        
        ' 要素の追加
        arrayList.Add("文字列")
        arrayList.Add(123)
        arrayList.Add(True)
        arrayList.Add(#1/1/2025#)
        
        Console.WriteLine("ArrayList の内容:")
        For i As Integer = 0 To arrayList.Count - 1
            Console.WriteLine($"  インデックス {i}: {arrayList(i)} ({arrayList(i).GetType().Name})")
        Next
        
        ' 要素の挿入と削除
        arrayList.Insert(1, "挿入された要素")  ' インデックス1に挿入
        arrayList.RemoveAt(3)  ' インデックス3の要素を削除
        
        Console.WriteLine()
        Console.WriteLine("変更後:")
        For Each item As Object In arrayList
            Console.WriteLine($"  値: {item}")
        Next
        
        Console.ReadKey()
    End Sub
End Module

4.6 List(Of T) - 型安全なリスト

List(Of T)は、型安全性を保ちながら動的にサイズが変更できるジェネリックコレクションです。現代的なVB.NET開発では、ArrayListよりもList(Of T)の使用が推奨されています。

List(Of T)の主な利点

  • 型安全性: コンパイル時に型チェックが行われる
  • パフォーマンス: ボクシング/アンボクシングが不要
  • IntelliSense: IDEでメソッドが適切に表示される
  • 豊富なメソッド: 検索、ソート、フィルタリング機能
実習 4-2: 商品在庫管理システム

List(Of T)を使って商品在庫を管理するシステムを作成しましょう。

サンプルコード
Module Program
    Sub Main(args As String())
        ' 商品名のリスト
        Dim products As New List(Of String)
        Dim quantities As New List(Of Integer)
        Dim prices As New List(Of Decimal)
        
        ' 初期在庫データ
        products.AddRange({"ノートPC", "マウス", "キーボード", "モニター"})
        quantities.AddRange({5, 20, 15, 8})
        prices.AddRange({80000D, 2500D, 3500D, 25000D})
        
        Console.WriteLine("=== 在庫管理システム ===")
        ShowInventory(products, quantities, prices)
        
        ' 商品の追加
        Console.WriteLine()
        Console.WriteLine("新商品を追加します...")
        products.Add("ヘッドセット")
        quantities.Add(12)
        prices.Add(8500D)
        
        ' 在庫の更新（マウスを10個販売）
        Dim mouseIndex As Integer = products.IndexOf("マウス")
        If mouseIndex >= 0 Then
            quantities(mouseIndex) -= 10
            Console.WriteLine("マウスを10個販売しました。")
        End If
        
        Console.WriteLine()
        Console.WriteLine("=== 更新後の在庫 ===")
        ShowInventory(products, quantities, prices)
        
        ' 在庫不足商品の検索
        Console.WriteLine()
        Console.WriteLine("=== 在庫不足警告（10個以下）===")
        For i As Integer = 0 To products.Count - 1
            If quantities(i) <= 10 Then
                Console.WriteLine($"⚠ {products(i)}: 残り{quantities(i)}個")
            End If
        Next
        
        Console.ReadKey()
    End Sub
    
    Sub ShowInventory(products As List(Of String), quantities As List(Of Integer), prices As List(Of Decimal))
        Console.WriteLine("商品名".PadRight(12) & "数量".PadRight(8) & "単価")
        Console.WriteLine("".PadRight(30, "-"c))
        
        For i As Integer = 0 To products.Count - 1
            Console.WriteLine($"{products(i).PadRight(12)}{quantities(i).ToString().PadRight(8)}{prices(i):C}")
        Next
    End Sub
End Module

4.7 Dictionary(Of TKey, TValue)

Dictionaryは、キーと値のペアを格納するコレクションです。辞書のように、キーを指定して対応する値を効率的に検索できます。ユーザー情報管理、設定値の保存、キャッシュなどに活用されます。 

Module Program
    Sub Main(args As String())
        ' Dictionaryの作成（キー: String, 値: Integer）
        Dim studentScores As New Dictionary(Of String, Integer)
        
        ' データの追加
        studentScores.Add("田中太郎", 85)
        studentScores.Add("佐藤花子", 92)
        studentScores.Add("鈴木一郎", 78)
        studentScores("高橋美穂") = 88  ' 別の追加方法
        
        ' データの取得
        Console.WriteLine("=== 学生成績 ===")
        For Each kvp As KeyValuePair(Of String, Integer) In studentScores
            Console.WriteLine($"{kvp.Key}: {kvp.Value}点")
        Next
        
        ' 特定の学生の成績を検索
        Console.WriteLine()
        Dim searchName As String = "佐藤花子"
        If studentScores.ContainsKey(searchName) Then
            Console.WriteLine($"{searchName}の成績: {studentScores(searchName)}点")
        Else
            Console.WriteLine($"{searchName}は見つかりませんでした。")
        End If
        
        ' 成績の更新
        studentScores("田中太郎") = 90  ' 田中太郎の成績を更新
        Console.WriteLine($"田中太郎の更新された成績: {studentScores("田中太郎")}点")
        
        ' 統計情報
        Console.WriteLine()
        Console.WriteLine("=== 統計情報 ===")
        Console.WriteLine($"学生数: {studentScores.Count}人")
        Console.WriteLine($"平均点: {studentScores.Values.Average():F1}点")
        Console.WriteLine($"最高点: {studentScores.Values.Max()}点")
        Console.WriteLine($"最低点: {studentScores.Values.Min()}点")
        
        Console.ReadKey()
    End Sub
End Module

4.8 For Each文によるコレクション反復

For Each文は、配列やコレクションのすべての要素を簡潔に処理するためのループ構文です。インデックスを気にすることなく、要素に直接アクセスできます。 

Module Program
    Sub Main(args As String())
        ' 配列でのFor Each
        Dim colors() As String = {"赤", "青", "緑", "黄色", "紫"}
        
        Console.WriteLine("色の一覧:")
        For Each color As String In colors
            Console.WriteLine($"  - {color}")
        Next
        
        ' List(Of T)でのFor Each
        Dim numbers As New List(Of Integer) From {1, 2, 3, 4, 5}
        
        Console.WriteLine()
        Console.WriteLine("数値の二乗:")
        For Each number As Integer In numbers
            Console.WriteLine($"  {number}の二乗 = {number * number}")
        Next
        
        ' DictionaryでのFor Each
        Dim capitals As New Dictionary(Of String, String) From {
            {"日本", "東京"},
            {"アメリカ", "ワシントンD.C."},
            {"フランス", "パリ"},
            {"ドイツ", "ベルリン"}
        }
        
        Console.WriteLine()
        Console.WriteLine("各国の首都:")
        For Each kvp As KeyValuePair(Of String, String) In capitals
            Console.WriteLine($"  {kvp.Key}の首都: {kvp.Value}")
        Next
        
        Console.ReadKey()
    End Sub
End Module
理解度確認クイズ
  1. 配列のインデックス（添え字）は何から始まりますか？
    a) 1
    b) 0
    c) -1
    d) 任意の数値
  2. 動的配列のサイズを変更する際、元のデータを保持するキーワードは？
    a) ReDim
    b) ReDim Preserve
    c) Resize
    d) Expand
  3. 型安全性の観点で推奨されるのはどちらですか？
    a) ArrayList
    b) List(Of T)
    c) Array
    d) どれも同じ
  4. キーと値のペアを格納するコレクションは？
    a) Array
    b) List
    c) Dictionary
    d) ArrayList
  5. For Each文の利点は？
    a) 実行速度が速い
    b) インデックスを意識しなくてよい
    c) メモリ使用量が少ない
    d) デバッグしやすい
解答を表示
  1. b) 0 - VB.NETの配列インデックスは0から始まります。
  2. b) ReDim Preserve - Preserveキーワードで既存データを保持しながらサイズ変更できます。
  3. b) List(Of T) - ジェネリック型により型安全性が確保されます。
  4. c) Dictionary - Dictionary(Of TKey, TValue)でキー・値ペアを管理できます。
  5. b) インデックスを意識しなくてよい - 要素に直接アクセスでき、インデックス管理が不要です。
