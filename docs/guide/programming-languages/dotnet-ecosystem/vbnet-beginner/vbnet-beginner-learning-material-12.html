12.1 これまでの学習の振り返り

第1章から第11章まで、プログラミングの基礎からWindowsフォームアプリケーションの開発まで、幅広い内容を学んできました。最終章では、これまでの知識を総動員して電卓アプリを作成します。まずは、学んできた内容を振り返りましょう。

学習内容の総まとめ

1 変数とデータ型（第5章）

+、-、*、/などの算術演算子を使った計算方法を学びました。電卓の核となる四則演算に活用します。

12.3 実習: 電卓アプリの作成

それでは、実際に電卓アプリを作成していきましょう。段階的に進めていくので、一つずつ確実に理解しながら実装してください。

実習 12-1: プロジェクトとフォームの作成

新しいプロジェクトを作成し、電卓の画面を設計します。

手順

  1. 1 Visual Studioを起動し、「新しいプロジェクトの作成」をクリック
  2. 2 「Windows フォーム アプリケーション (.NET Framework)」を選択し、「次へ」
  3. 3 プロジェクト名を「CalculatorApp」、保存場所を適切な場所に設定し、「作成」
  4. 4 フォームのタイトルを「電卓アプリ」に変更（プロパティウィンドウのTextプロパティ）
  5. 5 フォームのサイズを適切な大きさに調整（幅300、高さ400程度）

実習 12-2: コントロールの配置

ツールボックスからテキストボックスとボタンを配置します。

配置するコントロール

  1. 1 TextBox（名前: txtDisplay、テキスト: 0、TextAlign: Right、Font: 18pt）
  2. 2 数字ボタン 0〜9（名前: btn0〜btn9、テキスト: 0〜9、Font: 14pt）
  3. 3 演算子ボタン（名前: btnPlus, btnMinus, btnMultiply, btnDivide、テキスト: +, -, ×, ÷）
  4. 4 イコールボタン（名前: btnEqual、テキスト: =）
  5. 5 クリアボタン（名前: btnClear、テキスト: C）

注意

ボタンの配置は、画面設計図を参考に整列させてください。見た目が整っていると、使いやすいアプリになります。

12.4 コードの実装

画面のデザインができたら、次はコードを書いていきます。電卓の動作を実現するために、変数の宣言、イベント処理、計算ロジックを実装します。

ステップ1: 変数の宣言

フォームレベル変数として、計算に必要な情報を保存する変数を宣言します。

Form1.vb（変数宣言部分） 
Public Class Form1
    ' フォームレベル変数の宣言
    Private firstNumber As Double = 0      ' 第1の数値
    Private secondNumber As Double = 0     ' 第2の数値
    Private operatorType As String = ""    ' 演算子の種類 (+, -, *, /)
    Private isOperatorClicked As Boolean = False  ' 演算子がクリックされたか

    ' 以下にイベント処理を記述
End Class

ステップ2: 数字ボタンのクリックイベント

数字ボタン（0〜9）がクリックされたときの処理を実装します。すべての数字ボタンに同じイベントハンドラを使用できます。

Form1.vb（数字ボタンイベント） 
' 数字ボタンのクリックイベント（0〜9すべて共通）
Private Sub NumberButton_Click(sender As Object, e As EventArgs) _
    Handles btn0.Click, btn1.Click, btn2.Click, btn3.Click, btn4.Click, _
            btn5.Click, btn6.Click, btn7.Click, btn8.Click, btn9.Click

    Dim btn As Button = CType(sender, Button)  ' クリックされたボタンを取得

    ' 演算子クリック直後の場合、表示をクリア
    If isOperatorClicked Then
        txtDisplay.Text = ""
        isOperatorClicked = False
    End If

    ' 表示が"0"の場合は置き換え、それ以外は追加
    If txtDisplay.Text = "0" Then
        txtDisplay.Text = btn.Text
    Else
        txtDisplay.Text &= btn.Text
    End If
End Sub

ステップ3: 演算子ボタンのクリックイベント

+、-、×、÷の演算子ボタンがクリックされたときの処理を実装します。

Form1.vb（演算子ボタンイベント） 
' 演算子ボタンのクリックイベント
Private Sub OperatorButton_Click(sender As Object, e As EventArgs) _
    Handles btnPlus.Click, btnMinus.Click, btnMultiply.Click, btnDivide.Click

    Dim btn As Button = CType(sender, Button)

    ' 現在の表示を第1の数値として保存
    firstNumber = Double.Parse(txtDisplay.Text)

    ' 演算子の種類を保存
    operatorType = btn.Text

    ' フラグを立てる
    isOperatorClicked = True
End Sub

ステップ4: イコールボタンのクリックイベント

=ボタンがクリックされたときに、実際の計算を実行します。

Form1.vb（イコールボタンイベント） 
' イコールボタンのクリックイベント
Private Sub btnEqual_Click(sender As Object, e As EventArgs) Handles btnEqual.Click
    ' 第2の数値を取得
    secondNumber = Double.Parse(txtDisplay.Text)

    Dim result As Double = 0

    ' 演算子に応じて計算を実行
    Select Case operatorType
        Case "+"
            result = firstNumber + secondNumber
        Case "-"
            result = firstNumber - secondNumber
        Case "×"
            result = firstNumber * secondNumber
        Case "÷"
            If secondNumber <> 0 Then
                result = firstNumber / secondNumber
            Else
                MessageBox.Show("0で割ることはできません", "エラー")
                Return
            End If
    End Select

    ' 結果を表示
    txtDisplay.Text = result.ToString()

    ' 変数をリセット
    operatorType = ""
    isOperatorClicked = False
End Sub

ステップ5: クリアボタンのクリックイベント

Cボタンで、すべての変数と表示をリセットします。

Form1.vb（クリアボタンイベント） 
' クリアボタンのクリックイベント
Private Sub btnClear_Click(sender As Object, e As EventArgs) Handles btnClear.Click
    ' すべてリセット
    txtDisplay.Text = "0"
    firstNumber = 0
    secondNumber = 0
    operatorType = ""
    isOperatorClicked = False
End Sub

重要ポイント

複数のボタンに同じイベントハンドラを割り当てる場合、「Handles btn1.Click, btn2.Click, ...」のようにカンマ区切りで記述できます。これにより、コードの重複を避けられます。

これまでのステップをすべて組み合わせた、完成版のコードです。コピーして使用できます。

        If txtDisplay.Text = "0" Then

            Case "-"

        isOperatorClicked = False
    End Sub

    ' クリアボタンのクリックイベント
    Private Sub btnClear_Click(sender As Object, e As EventArgs) Handles btnClear.Click

12.7 トラブルシューティング

うまく動かない場合は、以下の点を確認してください。

トラブルシューティング

数字ボタンをクリックしても表示されない

原因: イベントハンドラが正しく設定されていない可能性があります。

解決方法: ボタンをダブルクリックしてイベントハンドラを作成し、NumberButton_Clickメソッドの「Handles」部分に追加されているか確認してください。

計算結果が正しくない

原因: 演算子の判定が正しくない、またはデータ型が間違っている可能性があります。

解決方法: Select Caseの演算子記号（"+"、"-"、"×"、"÷"）が、ボタンのTextプロパティと一致しているか確認してください。また、変数がDouble型で宣言されているか確認してください。

"入力文字列の形式が正しくありません"エラーが出る

原因: 空文字や数値に変換できない文字列をDouble.Parseしようとしています。

解決方法: txtDisplay.Textが空でないこと、または数値に変換可能な文字列であることを確認してください。必要に応じてTry-Catch構文でエラー処理を追加できます。

理解度確認クイズ

  1. Q1. 電卓アプリで、演算子ボタンがクリックされたときに保存する情報は何ですか？

    回答を表示

    「第1の数値（firstNumber）」と「演算子の種類（operatorType）」を保存します。これにより、=ボタンが押されたときに、どの演算を実行するかが分かります。

  2. Q2. 複数のボタンに同じイベントハンドラを割り当てるには、どうすればよいですか？

    回答を表示

    Handles句でカンマ区切りで複数のボタンを指定します。
    例: Handles btn1.Click, btn2.Click, btn3.Click

  3. Q3. 0で割り算をしようとしたときにエラーを防ぐには、どのような処理が必要ですか？

    回答を表示

    割り算を実行する前に、If文で「secondNumber <> 0」をチェックします。0の場合は、MessageBox.Showでエラーメッセージを表示し、処理を中断します（Return）。
