条件分岐で処理を分けよう

If文とSelect Case文を使って、条件によって異なる処理を実行する方法を学びます。比較演算子と論理演算子を組み合わせた複雑な条件判断も練習します。

この章で学ぶこと

  • 条件分岐の概念を理解し、日常生活での判断と関連づける
  • 比較演算子（>, <, =, >=, <=, <>）の使い方をマスターする
  • If～Then～End If、If～Then～Else～End If、If～ElseIf文を使い分ける
  • 論理演算子（And, Or, Not）を使った複雑な条件判断ができる
  • Select Case文で複数分岐を効率的に記述する

7.1 条件分岐とは何か

私たちは日常生活で、無意識のうちにたくさんの判断をしています。たとえば、朝起きて窓の外を見たとき、「雨が降っている」なら「傘を持って行く」、「雨が降っていない」なら「傘はいらない」と判断します。

プログラミングでも同じように、条件によって処理を分けることができます。これを条件分岐（または分岐処理）と呼びます。

ポイント

条件分岐は、「もし〜なら、〜する」という考え方です。プログラムは上から順番に実行されますが、条件によって実行される処理を変えることができます。

7.2 比較演算子を使った条件式

条件分岐では、「雨が降っている?」のような判断を条件式として書きます。条件式には、2つの値を比べる比較演算子を使います。

VB.NETの比較演算子

演算子 意味 例 結果
= 等しい 5 = 5 True（真）
<> 等しくない 5 <> 3 True（真）
> より大きい 10 > 5 True（真）
< より小さい 3 < 10 True（真）
>= 以上（等しいか大きい） 5 >= 5 True（真）
<= 以下（等しいか小さい） 3 <= 5 True（真）

つまずきポイント

VB.NETでは、等しいかどうかを調べるときも = を使います。代入の = と同じ記号ですが、文脈によって使い分けられます。また、「等しくない」は <> です（!= ではありません）。

7.3 If～Then～End Ifの基本

最もシンプルな条件分岐は、If～Then～End If文です。「もし条件が成り立つなら、この処理を実行する」という意味になります。

If文の基本構文

If 条件式 Then
    ' 条件が成り立つときに実行する処理
End If
例: 気温が30度以上なら警告を表示 
' 気温を変数に格納
Dim temperature As Integer = 35

' 条件判断: 気温が30度以上か?
If temperature >= 30 Then
    Console.WriteLine("暑いです！熱中症に注意してください。")
End If

Console.WriteLine("プログラム終了")

実行結果

暑いです！熱中症に注意してください。
プログラム終了

気温が35度なので、条件 temperature >= 30 が成り立ち（True）、If文の中の処理が実行されます。

ポイント

If文は必ず End If で終わります。If と End If の間に書いた処理が、条件が成り立つときだけ実行されます。

7.4 If～Then～Else～End Ifで2方向分岐

「条件が成り立つとき」と「成り立たないとき」の両方で違う処理をしたい場合は、Elseを使います。

If～Else文の構文

If 条件式 Then
    ' 条件が成り立つときの処理
Else
    ' 条件が成り立たないときの処理
End If
例: 年齢で大人か子どもかを判定 
' 年齢を変数に格納
Dim age As Integer = 15

' 条件判断: 年齢が18歳以上か?
If age >= 18 Then
    Console.WriteLine("あなたは大人です。")
Else
    Console.WriteLine("あなたは未成年です。")
End If

重要ポイント

Else を使うと、必ずどちらか一方の処理が実行されます。条件が成り立てば If の中、成り立たなければ Else の中が実行されます。

7.5 If～ElseIf～Else～End Ifで複数分岐

3つ以上の分岐が必要な場合は、ElseIfを使います。たとえば、成績を「優・良・可・不可」の4段階で判定する場合などに便利です。

ElseIfを使った構文

If 条件式1 Then
    ' 条件1が成り立つときの処理
ElseIf 条件式2 Then
    ' 条件1が成り立たず、条件2が成り立つときの処理
ElseIf 条件式3 Then
    ' 条件1,2が成り立たず、条件3が成り立つときの処理
Else
    ' すべての条件が成り立たないときの処理
End If
例: テストの点数で成績を判定 
' テストの点数
Dim score As Integer = 75

' 複数条件で成績判定
If score >= 80 Then
    Console.WriteLine("評価: 優")
ElseIf score >= 70 Then
    Console.WriteLine("評価: 良")
ElseIf score >= 60 Then
    Console.WriteLine("評価: 可")
Else
    Console.WriteLine("評価: 不可")
End If

実行結果（score = 75の場合）

評価: 良

75点は「70以上80未満」なので、2番目の ElseIf の条件が成り立ち、「良」と判定されます。

ポイント

ElseIf は、上から順番に条件を確認します。最初に成り立った条件の処理だけが実行され、その後の ElseIf や Else は無視されます。

7.6 論理演算子で複雑な条件を作る

「気温が30度以上、かつ、湿度が80%以上なら警告」のように、複数の条件を組み合わせたい場合は論理演算子を使います。

VB.NETの論理演算子

And（かつ）

すべての条件が成り立つときだけ True になります。

条件1 And 条件2    ' 両方とも True なら True

Or（または）

どちらか1つでも成り立てば True になります。

条件1 Or 条件2     ' どちらか1つでも True なら True

Not（否定）

条件の結果を反転します。True → False、False → True

Not 条件          ' True なら False に、False なら True になる
例: And演算子を使った条件 
' 気温と湿度の変数
Dim temperature As Integer = 32
Dim humidity As Integer = 85

' 両方の条件が成り立つか判定
If temperature >= 30 And humidity >= 80 Then
    Console.WriteLine("警告: 高温多湿です！")
Else
    Console.WriteLine("快適です。")
End If
例: Or演算子を使った条件 
' 曜日の変数（0=日曜、6=土曜）
Dim dayOfWeek As Integer = 6

' 土曜日または日曜日かを判定
If dayOfWeek = 0 Or dayOfWeek = 6 Then
    Console.WriteLine("今日は休日です。")
Else
    Console.WriteLine("今日は平日です。")
End If

重要ポイント

And は「すべてが成り立つ」、Or は「どれか1つでも成り立つ」という意味です。日常会話の「〜かつ〜」「〜または〜」と同じです。

7.7 Select Case文で多方向分岐

1つの変数の値によって処理を分けたい場合、ElseIf を何度も使うより、Select Case文を使った方が読みやすくなります。

Select Case文の構文

Select Case 変数
    Case 値1
        ' 変数が値1のときの処理
    Case 値2
        ' 変数が値2のときの処理
    Case 値3
        ' 変数が値3のときの処理
    Case Else
        ' どの値にも当てはまらないときの処理
End Select
例: 月の番号から季節を判定 
' 月の番号（1～12）
Dim month As Integer = 7

' 月の値によって季節を判定
Select Case month
    Case 3, 4, 5
        Console.WriteLine("春です。")
    Case 6, 7, 8
        Console.WriteLine("夏です。")
    Case 9, 10, 11
        Console.WriteLine("秋です。")
    Case 12, 1, 2
        Console.WriteLine("冬です。")
    Case Else
        Console.WriteLine("不正な月の値です。")
End Select

実行結果（month = 7の場合）

夏です。

月が7なので、Case 6, 7, 8 に一致し、「夏です。」と表示されます。

ヒント

Select Case では、Case 6, 7, 8 のようにカンマで区切って複数の値を指定できます。また、Case 1 To 10 のように範囲指定も可能です。

7.8 ネストした条件分岐

If文の中に、さらに別のIf文を入れることもできます。これをネスト（入れ子）と呼びます。複雑な条件判断に使われますが、深くなりすぎると読みにくくなるので注意が必要です。

例: 年齢と学生かどうかで料金を判定 
' 年齢と学生フラグ
Dim age As Integer = 20
Dim isStudent As Boolean = True

' ネストした条件分岐
If age >= 18 Then
    ' 大人の場合
    If isStudent Then
        Console.WriteLine("料金: 学生割引 800円")
    Else
        Console.WriteLine("料金: 大人 1000円")
    End If
Else
    ' 未成年の場合
    Console.WriteLine("料金: 子ども 500円")
End If

注意

ネストが深くなりすぎると、コードが読みづらくなります。3段階以上のネストが必要な場合は、論理演算子（And、Or）を使ったり、プロシージャ（関数）に分割したりして、シンプルにできないか検討しましょう。

実習 7-1: 成績判定プログラム

ユーザーからテストの点数を入力してもらい、以下の基準で成績を判定するプログラムを作成しましょう。

判定基準

  • • 90点以上: 「優」
  • • 80点以上90点未満: 「良」
  • • 70点以上80点未満: 「可」
  • • 60点以上70点未満: 「可」
  • • 60点未満: 「不可」

手順

  1. 1 新しいコンソールアプリプロジェクトを作成します
  2. 2 ユーザーに点数の入力を促すメッセージを表示します
  3. 3 入力された点数を整数型変数に格納します
  4. 4 If～ElseIf文を使って、点数に応じた評価を表示します

理解度確認クイズ

  1. Q1. 次のコードで「合格です」と表示されるのは、score が何点以上のときですか？

    If score >= 60 Then
    Console.WriteLine("合格です")
End If
    回答を表示

    60点以上のとき。>= は「以上（等しいか大きい）」という意味なので、60点ちょうども含まれます。

  2. Q2. And演算子とOr演算子の違いを説明してください。

    回答を表示

    And: すべての条件が成り立つときだけTrueになります（「〜かつ〜」）。
    Or: どれか1つでも条件が成り立てばTrueになります（「〜または〜」）。

  3. Q3. Select Case文とIf～ElseIf文は、どちらを使うべきですか？

    回答を表示

    1つの変数の値によって処理を分ける場合は、Select Case文の方が読みやすくなります。複雑な条件式（And、Orなど）を使う場合や、異なる変数を組み合わせる場合は、If～ElseIf文が適しています。

トラブルシューティング

条件が成り立っているはずなのに、If文の中が実行されない

原因: 条件式の書き方が間違っている可能性があります。特に、等しいかどうかの判定は = を使います（== ではありません）。

解決方法: 条件式を確認し、比較演算子が正しいか、変数名にタイプミスがないかチェックしましょう。

「End If」がありませんというエラーが出る

原因: If文を End If で閉じ忘れています。

解決方法: すべての If に対応する End If があるか確認してください。Visual Studioのインデント（字下げ）を見ると、対応関係が分かりやすくなります。

ElseIfの条件が期待通りに動作しない

原因: ElseIfは上から順番に判定され、最初に成り立った条件だけが実行されます。条件の順序が適切でない可能性があります。

解決方法: 条件の順序を見直しましょう。たとえば、範囲判定では、広い範囲より狭い範囲を先に書く必要があります。

第7章のまとめ

  • 条件分岐は、条件によって実行する処理を変える仕組みです。日常生活の判断と同じ考え方です。
  • 比較演算子（=, <>, >, <, >=, <=）を使って、条件式を作ります。
  • If～Then～End If は最もシンプルな条件分岐、Else で2方向、ElseIf で3方向以上の分岐ができます。
  • 論理演算子（And, Or, Not）を使うと、複数の条件を組み合わせた複雑な判断ができます。
  • Select Case文は、1つの変数の値によって多方向分岐するときに読みやすいコードが書けます。
