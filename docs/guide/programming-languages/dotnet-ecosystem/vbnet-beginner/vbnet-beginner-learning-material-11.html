Windowsフォームアプリケーションを作ろう

コンソールアプリからGUIアプリへステップアップします。フォームにボタンやテキストボックスを配置し、クリックイベントに応答するプログラムを作成します。

この章で学ぶこと

  • Windowsフォームアプリケーションとコンソールアプリケーションの違いを理解する
  • フォームデザイナーを使ってボタンやテキストボックスなどのコントロールを配置する
  • イベント（クリック、入力など）とイベント処理の概念を理解する
  • ボタンクリック時に処理を実行する簡単なGUIアプリを作成する

11.1 Windowsフォームアプリケーションとは

これまで学習してきたコンソールアプリケーションは、黒い画面（コマンドプロンプト）に文字を表示し、キーボードから入力を受け取るプログラムでした。一方、Windowsフォームアプリケーションは、ボタンやテキストボックスなど視覚的な部品を持つ、いわゆる「普通のソフト」のようなプログラムです。

コンソールアプリとGUIアプリの違い

コンソールアプリ

  • • 黒い画面に文字を表示
  • • キーボードのみで操作
  • • 順番に処理が進む
  • • 見た目はシンプル

Windowsフォームアプリ（GUI）

  • • ウィンドウにボタンやテキストボックスを表示
  • • マウスとキーボードで操作
  • • イベント（クリックなど）で処理が動く
  • • 直感的で使いやすい

ポイント

GUIは「Graphical User Interface（グラフィカル・ユーザー・インターフェース）」の略で、図形やボタンなどの視覚的な要素で操作できるインターフェースのことです。ExcelやWord、Webブラウザなど、普段使うほとんどのソフトがGUIアプリです。

11.2 フォームの構成要素（コントロール）

Windowsフォームアプリケーションは、フォーム（ウィンドウ）の上にコントロール（部品）を配置して作ります。コントロールには、ボタン、テキストボックス、ラベルなど、様々な種類があります。

主なコントロールの種類

Button（ボタン）

クリックすると処理を実行します。「実行」「キャンセル」などのボタンに使います。

TextBox（テキストボックス）

ユーザーが文字や数値を入力するための枠です。名前や数値の入力に使います。

Label（ラベル）

説明文や結果を表示します。「名前：」などのラベルや、計算結果の表示に使います。

11.3 Windowsフォームプロジェクトの作成

それでは、Visual StudioでWindowsフォームアプリケーションのプロジェクトを作成してみましょう。これまで作成してきたコンソールアプリとは、作成時のテンプレート選択が異なります。

実習 11-1: Windowsフォームプロジェクトの作成

初めてのWindowsフォームアプリケーションプロジェクトを作成します。

手順

  1. 1 Visual Studioを起動し、「新しいプロジェクトの作成」をクリック
  2. 2 テンプレート一覧から「Windows フォーム アプリケーション（.NET Framework）」を選択
  3. 3 言語は「Visual Basic」を選択
  4. 4 プロジェクト名を「MyFirstFormApp」と入力し、場所を選択
  5. 5 「作成」ボタンをクリック

プロジェクトが作成されると、中央にフォームデザイナーという、グレーのウィンドウが表示されます。これが、あなたのアプリケーションのウィンドウです。このウィンドウに、ボタンやテキストボックスを配置していきます。

重要ポイント

フォームデザイナーは「WYSIWYG（ウィジウィグ）」と呼ばれる仕組みで、「見たままが得られる（What You See Is What You Get）」という意味です。デザイナー上で配置した通りに、実際のアプリケーションが動きます。

11.4 コントロールの配置

フォームにボタンやテキストボックスを配置してみましょう。Visual Studioの左側にツールボックスというパネルがあります。ここから、配置したいコントロールをドラッグ＆ドロップでフォームに追加できます。

実習 11-2: ボタンとラベルの配置

フォームにボタンとラベルを配置し、簡単な画面を作成します。

手順

  1. 1 左側の「ツールボックス」をクリックして展開（見つからない場合は、メニューから「表示」→「ツールボックス」）
  2. 2 ツールボックス内の「Button」をフォームにドラッグ＆ドロップ
  3. 3 同様に「Label」もフォームに配置
  4. 4 配置したボタンやラベルは、マウスでドラッグして位置を調整できます

ヒント

ツールボックスが見つからない場合は、キーボードの「Ctrl + Alt + X」を押すと表示されます。また、コントロールをダブルクリックすると、フォームの中央に自動配置されます。

11.5 プロパティの設定

配置したコントロールには、プロパティ（属性）があります。プロパティを変更することで、ボタンに表示する文字や、サイズ、色などを変更できます。プロパティは、Visual Studioの右側にあるプロパティウィンドウで設定します。

主なプロパティ

Name（名前）

コントロールを識別するための名前です。コードから操作するときに使います。

Text（テキスト）

ボタンやラベルに表示される文字です。「クリック」「実行」などを設定します。

Size（サイズ）

コントロールの幅と高さをピクセル単位で設定します。

Location（位置）

フォーム上のどこに配置するかを、X座標とY座標で設定します。

実習 11-3: プロパティの変更

配置したボタンとラベルのプロパティを変更して、見た目を整えます。

手順

  1. 1 フォーム上のボタンをクリックして選択
  2. 2 右側のプロパティウィンドウで「Name」を「btnClick」に変更
  3. 3 「Text」プロパティを「クリックしてね」に変更
  4. 4 次に、ラベルを選択し、「Name」を「lblMessage」に変更
  5. 5 ラベルの「Text」プロパティを空白（何も入力しない）にする

重要ポイント

コントロールの名前（Name）には、わかりやすい名前をつける習慣をつけましょう。ボタンは「btn～」、ラベルは「lbl～」、テキストボックスは「txt～」のように接頭辞をつけると、後でコードを読むときに分かりやすくなります。

11.6 イベントとイベント処理

Windowsフォームアプリケーションは、イベント駆動型のプログラムです。ユーザーがボタンをクリックしたり、テキストボックスに文字を入力したりすると、イベント（出来事）が発生します。プログラマーは、このイベントに対して「何をするか」を書きます。これをイベント処理と呼びます。

主なイベントの種類

Click（クリック）

ボタンやラベルがクリックされたときに発生します。最もよく使うイベントです。

TextChanged（テキスト変更）

テキストボックスの内容が変わったときに発生します。

Load（読み込み）

フォームが最初に表示されるときに発生します。初期設定に使います。

補足

コンソールアプリは「上から順番に」処理が進みましたが、GUIアプリは「ユーザーの操作を待って」処理が動きます。この違いが、イベント駆動型プログラミングの特徴です。

11.7 ボタンクリックイベントの実装

それでは、ボタンがクリックされたときに、ラベルにメッセージを表示するプログラムを作りましょう。イベントハンドラ（イベント処理のコード）を作成します。

実習 11-4: Clickイベントの実装

ボタンをクリックしたときに、ラベルに「こんにちは！」と表示するプログラムを作ります。

手順

  1. 1 フォームデザイナーで、ボタン（btnClick）をダブルクリック
  2. 2 コードエディタが開き、Clickイベントのハンドラが自動生成されます
  3. 3 以下のコードを入力します
Form1.vb 
' ボタンクリック時の処理
Private Sub btnClick_Click(sender As Object, e As EventArgs) Handles btnClick.Click
    ' ラベルにメッセージを表示
    lblMessage.Text = "こんにちは！"
End Sub

コードの説明

Private Sub btnClick_Click(...)

btnClickボタンのClickイベントが発生したときに実行されるプロシージャです。

Handles btnClick.Click

このプロシージャが、btnClickのClickイベントを処理することを宣言しています。

lblMessage.Text = "こんにちは！"

lblMessageというラベルのTextプロパティに「こんにちは！」を設定します。これでラベルに文字が表示されます。

動作確認

プログラムを実行して、ボタンをクリックしてみましょう。

  1. 1. F5キーを押してプログラムを実行
  2. 2. フォームが表示されたら、「クリックしてね」ボタンをクリック
  3. 3. ラベルに「こんにちは！」と表示されることを確認

期待される結果 

ボタンをクリックすると、その下のラベルに
「こんにちは！」というメッセージが表示される。

重要ポイント

イベントハンドラは、デザイナーでコントロールをダブルクリックすることで自動生成されます。Visual Studioが必要なコードのひな形を作ってくれるので、初心者でも安心して作業できます。

11.8 テキストボックスを使った応用

次は、テキストボックスを追加して、ユーザーが入力した名前を表示するプログラムを作ってみましょう。これにより、ユーザーとの対話的なアプリケーションが作れるようになります。

実習 11-5: 名前を表示するアプリ

テキストボックスに入力した名前を、ボタンクリック時にラベルに表示します。

手順

  1. 1 ツールボックスから「TextBox」をフォームに配置
  2. 2 テキストボックスの「Name」を「txtName」に変更
  3. 3 既存のラベル（lblMessage）の上に、新しいラベルを配置
  4. 4 新しいラベルの「Text」を「お名前を入力してください：」に設定
  5. 5 ボタン（btnClick）のClickイベントのコードを以下に書き換え
Form1.vb（書き換え後） 
' ボタンクリック時の処理
Private Sub btnClick_Click(sender As Object, e As EventArgs) Handles btnClick.Click
    ' テキストボックスから名前を取得
    Dim name As String = txtName.Text

    ' ラベルに挨拶を表示
    lblMessage.Text = name & "さん、こんにちは！"
End Sub

コードの説明

Dim name As String = txtName.Text

テキストボックスに入力された文字を、name変数に格納します。txtName.Textで、テキストボックスの内容を取得できます。

lblMessage.Text = name & "さん、こんにちは！"

入力された名前と「さん、こんにちは！」を連結して、ラベルに表示します。

動作確認

プログラムを実行して、名前を入力してみましょう。

  1. 1. F5キーを押してプログラムを実行
  2. 2. テキストボックスに自分の名前を入力（例：「太郎」）
  3. 3. ボタンをクリック
  4. 4. ラベルに「太郎さん、こんにちは！」と表示されることを確認

期待される結果 

テキストボックスに「太郎」と入力してボタンをクリックすると、
ラベルに「太郎さん、こんにちは！」と表示される。

11.9 プロパティとメソッドの違い

コントロールには、プロパティとメソッドがあります。どちらもコントロールを操作するために使いますが、役割が異なります。

プロパティとメソッドの違い

プロパティ

コントロールの「状態」や「見た目」を設定・取得します。

例： Text、Size、BackColorなど

' プロパティの使用例
lblMessage.Text = "こんにちは"
Dim msg As String = lblMessage.Text

メソッド

コントロールに「動作」を実行させます。

例： Show、Hide、Focusなど

' メソッドの使用例
txtName.Clear()  ' テキストボックスをクリア
txtName.Focus()  ' カーソルを移動

覚え方のヒント

プロパティは「形容詞」、メソッドは「動詞」のイメージです。「テキストが"こんにちは"である」（プロパティ）、「テキストボックスをクリアする」（メソッド）のように考えると分かりやすいです。

理解度確認クイズ

  1. Q1. GUIアプリとコンソールアプリの違いを説明してください。

    回答を表示

    コンソールアプリは黒い画面に文字を表示し、キーボードのみで操作します。GUIアプリは、ボタンやテキストボックスなどの視覚的な部品を持ち、マウスとキーボードで操作できます。GUIアプリはイベント駆動型で、ユーザーの操作に応じて処理が実行されます。

  2. Q2. イベント駆動型プログラミングとは何ですか？

    回答を表示

    ユーザーの操作（ボタンクリック、テキスト入力など）によって発生する「イベント」に応じて、プログラムが動くプログラミング方式です。コンソールアプリのように「上から順番に」実行されるのではなく、「ユーザーの操作を待って」処理が実行されます。

  3. Q3. プロパティとメソッドの違いは何ですか？

    回答を表示

    プロパティは、コントロールの「状態」や「見た目」を設定・取得するもので、Text、Size、Colorなどがあります。メソッドは、コントロールに「動作」を実行させるもので、Clear、Show、Hideなどがあります。プロパティは「形容詞」、メソッドは「動詞」のイメージです。

  4. Q4. コントロールのNameプロパティに「btn～」「lbl～」のような接頭辞をつける理由は何ですか？

    回答を表示

    コードを読むときに、そのコントロールが何であるかを一目で判別できるようにするためです。btnで始まればボタン、lblで始まればラベル、txtで始まればテキストボックスだと分かります。これにより、コードの可読性が向上し、メンテナンスしやすくなります。

第11章のまとめ

  • Windowsフォームアプリケーションは、ボタンやテキストボックスなどの視覚的な部品を持つGUIアプリです。
  • フォームデザイナーを使って、ドラッグ＆ドロップで簡単にコントロールを配置できます。
  • イベント駆動型プログラミングでは、ユーザーの操作（クリックなど）に応じて処理が実行されます。
  • プロパティでコントロールの見た目や状態を設定し、メソッドで動作を実行させます。
  • イベントハンドラをダブルクリックで簡単に作成でき、その中に処理を記述します。
