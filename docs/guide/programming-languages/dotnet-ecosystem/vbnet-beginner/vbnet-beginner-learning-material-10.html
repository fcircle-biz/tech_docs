プロシージャで処理をまとめよう

SubプロシージャとFunctionプロシージャの作成方法を学びます。引数と戻り値の概念を理解し、処理を再利用可能な形にまとめる技術を身につけます。

この章で学ぶこと

  • プロシージャとは何か、どのような役割を持つのかを理解する
  • Subプロシージャの作り方と使い方を学ぶ
  • Functionプロシージャの作り方と戻り値の概念を理解する
  • 引数を使ってプロシージャに値を渡す方法を習得する
  • プロシージャを使うことで、コードの再利用性と可読性を高める

10.1 プロシージャとは何か

プログラムを書いていると、同じような処理を何度も書くことがあります。たとえば、「あいさつメッセージを表示する」処理を何度も書くのは大変ですし、コードが長くなって読みづらくなります。

プロシージャ（Procedure）とは、一連の処理をまとめて名前を付けたものです。料理のレシピのように、「この処理」と呼べば、何度でも同じ処理を実行できます。

flowchart LR A[プログラムのメイン処理] --> B[プロシージャを呼び出す] B --> C[プロシージャが実行される] C --> D[処理が完了して戻る] D --> A style A fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style B fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style C fill:#fce7f3,stroke:#ec4899,stroke-width:2px style D fill:#d1fae5,stroke:#10b981,stroke-width:2px

プロシージャのメリット

再利用性

一度書いた処理を何度でも使える。同じコードを繰り返し書く必要がない。

可読性

プロシージャ名を見れば、何をしているか分かりやすい。コードが整理される。

保守性

修正が必要なとき、プロシージャの中だけを直せば良い。あちこち修正する必要がない。

ポイント

プロシージャは、料理のレシピや辞書の見出し語のようなものです。「カレーの作り方」というレシピがあれば、何度でもカレーを作れますよね。同じように、プロシージャを作れば、同じ処理を何度でも実行できます。

10.2 VB.NETの2種類のプロシージャ

VB.NETには、大きく分けて2種類のプロシージャがあります。

Subプロシージャ

処理を実行するだけで、値を返さないプロシージャです。「何かをする」ことに使います。

使用例

メッセージを表示する、データを保存する、画面をクリアするなど

Functionプロシージャ

処理を実行して、その結果を値として返すプロシージャです。「何かを計算して答えを返す」ことに使います。

使用例

合計を計算する、文字列を加工する、判定結果を返すなど

10.3 Subプロシージャの基本

まずは、一番シンプルなSubプロシージャから学びましょう。Subプロシージャは、Subで始まり、End Subで終わります。

基本の書き方

Subプロシージャの構文 
Sub プロシージャ名()
    ' ここに処理を書く
End Sub

実例：あいさつメッセージを表示する

Greeting.vb 
Module Module1
    Sub Main()
        ' プロシージャを呼び出す
        ShowGreeting()
        ShowGreeting()  ' 何度でも呼び出せる
    End Sub

    ' あいさつを表示するプロシージャ
    Sub ShowGreeting()
        Console.WriteLine("こんにちは！")
        Console.WriteLine("VB.NETへようこそ！")
    End Sub
End Module

実行結果

こんにちは！
VB.NETへようこそ！
こんにちは！
VB.NETへようこそ！

重要ポイント

プロシージャは「定義する」部分と「呼び出す」部分に分かれます。Sub ShowGreeting()が定義、ShowGreeting()が呼び出しです。

10.4 引数を使ったSubプロシージャ

プロシージャに値を渡すことができます。この渡す値のことを引数（ひきすう、パラメータ）と呼びます。引数を使えば、プロシージャをより柔軟に使えます。

引数とは

引数は、プロシージャに渡す「材料」のようなものです。料理のレシピに「材料：玉ねぎ1個、にんじん1本」と書いてあるように、プロシージャにも「どんな値を受け取るか」を指定できます。

引数付きプロシージャの書き方

引数付きSubプロシージャの構文 
Sub プロシージャ名(引数名 As データ型)
    ' 引数を使った処理を書く
End Sub

実例：名前を受け取ってあいさつする

PersonalGreeting.vb 
Module Module1
    Sub Main()
        ' 引数に値を渡してプロシージャを呼び出す
        ShowPersonalGreeting("田中さん")
        ShowPersonalGreeting("佐藤さん")
    End Sub

    ' 名前を受け取ってあいさつするプロシージャ
    Sub ShowPersonalGreeting(name As String)
        Console.WriteLine("こんにちは、" & name & "！")
    End Sub
End Module

実行結果

こんにちは、田中さん！
こんにちは、佐藤さん！

複数の引数を使う

カンマで区切れば、複数の引数を受け取ることができます。

MultipleArgs.vb 
Module Module1
    Sub Main()
        ShowProfile("田中太郎", 25)
        ShowProfile("佐藤花子", 30)
    End Sub

    ' 名前と年齢を受け取って表示するプロシージャ
    Sub ShowProfile(name As String, age As Integer)
        Console.WriteLine("名前: " & name)
        Console.WriteLine("年齢: " & age & "歳")
        Console.WriteLine("---")
    End Sub
End Module

10.5 Functionプロシージャの基本

Functionプロシージャは、処理を実行して結果の値を返すことができます。計算結果や判定結果を受け取りたいときに使います。

戻り値とは

戻り値は、プロシージャから呼び出し元に返す「答え」のようなものです。自動販売機にお金を入れると飲み物が出てくるように、Functionに値を渡すと、計算結果が返ってきます。

基本の書き方

Functionプロシージャの構文 
Function プロシージャ名(引数名 As データ型) As 戻り値のデータ型
    ' 処理を書く
    Return 戻り値
End Function

実例：2つの数を足し算する

AddNumbers.vb 
Module Module1
    Sub Main()
        ' Functionを呼び出して、戻り値を受け取る
        Dim result As Integer = Add(10, 20)
        Console.WriteLine("10 + 20 = " & result)

        ' 直接表示することもできる
        Console.WriteLine("5 + 3 = " & Add(5, 3))
    End Sub

    ' 2つの数を足し算して結果を返すFunction
    Function Add(a As Integer, b As Integer) As Integer
        Return a + b
    End Function
End Module

実行結果

10 + 20 = 30
5 + 3 = 8

重要ポイント

Return文で値を返します。また、As Integerのように、返す値のデータ型を必ず指定します。

10.6 Functionプロシージャの実践例

Functionプロシージャの実用的な例を見ていきましょう。

例1: 合格判定プログラム

PassJudge.vb 
Module Module1
    Sub Main()
        Dim score As Integer = 75

        If IsPass(score) Then
            Console.WriteLine("合格です！")
        Else
            Console.WriteLine("不合格です。")
        End If
    End Sub

    ' 60点以上なら合格と判定するFunction
    Function IsPass(score As Integer) As Boolean
        Return score >= 60
    End Function
End Module

例2: 消費税込み金額を計算する

TaxCalculator.vb 
Module Module1
    Sub Main()
        Dim price As Integer = 1000
        Dim total As Integer = AddTax(price)

        Console.WriteLine("本体価格: " & price & "円")
        Console.WriteLine("税込価格: " & total & "円")
    End Sub

    ' 消費税10%を加算した金額を返すFunction
    Function AddTax(price As Integer) As Integer
        Return CInt(price * 1.1)
    End Function
End Module

例3: 配列の合計を求める

ArraySum.vb 
Module Module1
    Sub Main()
        Dim scores() As Integer = {85, 70, 92, 88, 76}
        Dim total As Integer = GetSum(scores)

        Console.WriteLine("合計点: " & total)
        Console.WriteLine("平均点: " & total / scores.Length)
    End Sub

    ' 配列の合計を計算して返すFunction
    Function GetSum(arr() As Integer) As Integer
        Dim sum As Integer = 0
        For Each num In arr
            sum += num
        Next
        Return sum
    End Function
End Module

実習 10-1: プロシージャを作ってみよう

以下のプロシージャを作成してみましょう。

課題

  1. 1 2つの数の大きい方を返すFunctionプロシージャ GetMax を作成してください。
  2. 2 文字列を受け取って、3回繰り返して表示するSubプロシージャ Repeat3Times を作成してください。
  3. 3 作成したプロシージャを呼び出して、正しく動作することを確認してください。

理解度確認クイズ

  1. Q1. プロシージャを使う3つのメリットを挙げてください。

    回答を表示

    1. 再利用性: 同じ処理を何度でも使える
    2. 可読性: コードが整理されて分かりやすくなる
    3. 保守性: 修正が必要なとき、プロシージャの中だけを直せば良い

  2. Q2. SubプロシージャとFunctionプロシージャの違いは何ですか？

    回答を表示

    Subプロシージャは処理を実行するだけで値を返しませんが、Functionプロシージャは処理を実行して結果の値を返します。FunctionはReturn文で値を返し、戻り値のデータ型を指定する必要があります。

  3. Q3. 引数とは何ですか？

    回答を表示

    引数（パラメータ）は、プロシージャに渡す値のことです。プロシージャを呼び出すときに値を渡すことで、プロシージャの動作を柔軟に変えることができます。料理のレシピでいう「材料」に相当します。

  4. Q4. 次のコードの間違いを指摘してください。

    Function Multiply(a As Integer, b As Integer)
    Dim result As Integer = a * b
End Function
    回答を表示

    2つの間違いがあります。
    1. 戻り値のデータ型が指定されていない（As Integerを追加）
    2. Return文がない（Return resultを追加）

第10章のまとめ

  • プロシージャは、処理をまとめて名前を付けたもので、再利用性・可読性・保守性が向上します。
  • Subプロシージャは処理を実行するだけ、Functionプロシージャは値を返します。
  • 引数を使えば、プロシージャに値を渡して柔軟に処理できます。
  • FunctionプロシージャはReturn文で値を返し、戻り値のデータ型を指定します。
  • プロシージャを活用することで、見通しの良い、メンテナンスしやすいコードが書けます。
