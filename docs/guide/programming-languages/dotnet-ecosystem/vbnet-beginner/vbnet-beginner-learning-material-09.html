配列でデータをまとめて管理しよう

複数のデータをまとめて扱う配列の基本を学びます。配列の宣言、要素へのアクセス、For Eachを使った繰り返し処理を実践します。

この章で学ぶこと

  • 配列とは何か、なぜ必要なのかを理解する
  • 配列の宣言と初期化の方法を学ぶ
  • 配列要素へのアクセス方法（インデックス）を理解する
  • For文とFor Each文を使った配列の処理を実践する

9.1 配列とは何か

これまで、変数を使って1つのデータを保存してきました。しかし、たとえば「5人の学生のテストの点数を保存したい」という場合、どうすればよいでしょうか？

通常の変数を使った場合

' 5人分の点数を別々の変数に保存
Dim score1 As Integer = 85
Dim score2 As Integer = 92
Dim score3 As Integer = 78
Dim score4 As Integer = 88
Dim score5 As Integer = 95

' 平均を計算するには、全部足す必要がある
Dim average As Double = (score1 + score2 + score3 + score4 + score5) / 5

この方法では、変数が5個も必要になり、平均を計算するコードも長くなってしまいます。もし100人分の点数を扱うなら、変数を100個も作らなければなりません。

このような問題を解決するのが配列（Array）です。配列を使うと、複数のデータを1つのまとまりとして管理できます。

重要ポイント

配列は、同じ種類のデータ（すべて整数、すべて文字列など）を、1つの箱にまとめて管理できる仕組みです。各データには「インデックス」という番号が付けられます。

9.2 配列の宣言と初期化

配列を使うには、まず配列を宣言する必要があります。配列の宣言方法はいくつかあります。

方法1: 要素数を指定して宣言

' 整数を5個入れる配列を宣言
Dim scores(4) As Integer
' 注意: 配列のサイズは「要素数 - 1」で指定します
' scores(4)は、0から4まで、つまり5個の要素を持つ配列です

' 各要素に値を代入
scores(0) = 85
scores(1) = 92
scores(2) = 78
scores(3) = 88
scores(4) = 95

注意

配列のインデックスは0から始まります。「Dim scores(4)」と宣言すると、インデックス0、1、2、3、4の5個の要素ができます。最初の要素はscores(0)で、最後の要素はscores(4)です。

方法2: 初期値を指定して宣言

' 初期値を指定して配列を作成（推奨）
Dim scores() As Integer = {85, 92, 78, 88, 95}
' この方法なら、要素数は自動的に5になります

' 文字列の配列の例
Dim names() As String = {"田中", "佐藤", "鈴木", "高橋", "伊藤"}

' 実数の配列の例
Dim prices() As Double = {1500.5, 2300.0, 899.9, 4500.0}

ヒント

初期値を指定する方法（方法2）の方が、コードが短く読みやすいので、最初から値が分かっている場合はこちらを使いましょう。

配列名「scores」は、5つの要素の先頭を指しています。各要素には0から始まるインデックス（添字）でアクセスできます。

9.3 配列要素へのアクセス

配列の各要素にアクセスするには、インデックス（添字）を使います。インデックスは配列名の後ろに括弧()で囲んで指定します。 

Dim scores() As Integer = {85, 92, 78, 88, 95}

' 要素の読み取り
Console.WriteLine("1人目の点数: " & scores(0))  ' 85
Console.WriteLine("3人目の点数: " & scores(2))  ' 78

' 要素の変更
scores(1) = 100  ' 2人目の点数を100点に変更
Console.WriteLine("変更後の2人目の点数: " & scores(1))  ' 100

' 変数をインデックスとして使う
Dim index As Integer = 4
Console.WriteLine("5人目の点数: " & scores(index))  ' 95

実行結果

1人目の点数: 85
3人目の点数: 78
変更後の2人目の点数: 100
5人目の点数: 95

よくあるエラー

存在しないインデックスにアクセスしようとすると、「IndexOutOfRangeException」というエラーが発生します。たとえば、要素数が5の配列に対してscores(5)やscores(10)にアクセスするとエラーになります。

9.4 配列の長さを取得する（Length）

配列に何個の要素が入っているかを知りたい場合は、Lengthプロパティを使います。 

Dim scores() As Integer = {85, 92, 78, 88, 95}

' 配列の要素数を取得
Dim count As Integer = scores.Length
Console.WriteLine("要素数: " & count)  ' 5

' 最後の要素にアクセス（Lengthを使う）
Dim lastScore As Integer = scores(scores.Length - 1)
Console.WriteLine("最後の点数: " & lastScore)  ' 95

ポイント

インデックスは0から始まるので、最後の要素のインデックスは「Length - 1」になります。要素数が5なら、インデックスは0～4です。

9.5 For文で配列を処理する

配列の全要素に対して同じ処理を行いたい場合、For文と組み合わせると便利です。

例1: 配列の全要素を表示

Dim scores() As Integer = {85, 92, 78, 88, 95}

Console.WriteLine("全員の点数を表示:")
For i As Integer = 0 To scores.Length - 1
    Console.WriteLine((i + 1) & "人目: " & scores(i) & "点")
Next

実行結果

全員の点数を表示:
1人目: 85点
2人目: 92点
3人目: 78点
4人目: 88点
5人目: 95点

例2: 配列の合計と平均を計算

Dim scores() As Integer = {85, 92, 78, 88, 95}

' 合計を計算
Dim total As Integer = 0
For i As Integer = 0 To scores.Length - 1
    total = total + scores(i)
Next

' 平均を計算
Dim average As Double = total / scores.Length

Console.WriteLine("合計点: " & total)
Console.WriteLine("平均点: " & average)

実行結果

合計点: 438
平均点: 87.6

9.6 For Each文で配列を処理する

配列の全要素を順番に処理したいだけなら、For Each文を使うとより簡潔に書けます。For Each文は、配列の要素を先頭から順番に取り出して処理します。

For Each文の基本構文

For Each 変数名 As データ型 In 配列名
    ' 繰り返し実行する処理
Next

例1: For Eachで全要素を表示

Dim scores() As Integer = {85, 92, 78, 88, 95}

Console.WriteLine("全員の点数を表示:")
For Each score As Integer In scores
    Console.WriteLine(score & "点")
Next

実行結果

全員の点数を表示:
85点
92点
78点
88点
95点

例2: For Eachで合計を計算

Dim scores() As Integer = {85, 92, 78, 88, 95}

' 合計を計算
Dim total As Integer = 0
For Each score As Integer In scores
    total = total + score
Next

Console.WriteLine("合計点: " & total)

For文とFor Each文の使い分け

For文を使う場合

  • • インデックスが必要な場合
  • • 配列の要素を変更する場合
  • • 特定の範囲だけ処理したい場合

For Each文を使う場合

  • • 全要素を読み取るだけの場合
  • • インデックスが不要な場合
  • • コードを簡潔に書きたい場合

重要ポイント

For Each文は配列の要素を「読み取る」ことしかできません。要素の値を変更したい場合は、For文を使う必要があります。

実習：配列を使った成績管理プログラム

5人の学生の点数を配列に保存し、合計、平均、最高点、最低点を計算するプログラムを作成しましょう。

プログラムの仕様

  • 5人分の点数を配列に保存する
  • 全員の点数を表示する
  • 合計点と平均点を計算して表示する
  • 最高点と最低点を見つけて表示する 
Module Module1
    Sub Main()
        ' 5人分の点数を配列に保存
        Dim scores() As Integer = {85, 92, 78, 88, 95}

        ' タイトル表示
        Console.WriteLine("=== 成績管理システム ===")
        Console.WriteLine()

        ' 全員の点数を表示
        Console.WriteLine("【全員の点数】")
        For i As Integer = 0 To scores.Length - 1
            Console.WriteLine((i + 1) & "人目: " & scores(i) & "点")
        Next
        Console.WriteLine()

        ' 合計と平均を計算
        Dim total As Integer = 0
        For Each score As Integer In scores
            total = total + score
        Next
        Dim average As Double = total / scores.Length

        Console.WriteLine("【統計情報】")
        Console.WriteLine("合計点: " & total & "点")
        Console.WriteLine("平均点: " & average & "点")

        ' 最高点を見つける
        Dim maxScore As Integer = scores(0)
        For Each score As Integer In scores
            If score > maxScore Then
                maxScore = score
            End If
        Next

        ' 最低点を見つける
        Dim minScore As Integer = scores(0)
        For Each score As Integer In scores
            If score < minScore Then
                minScore = score
            End If
        Next

        Console.WriteLine("最高点: " & maxScore & "点")
        Console.WriteLine("最低点: " & minScore & "点")

        Console.ReadLine()
    End Sub
End Module

実行結果

9.7 文字列配列の活用

配列には整数だけでなく、文字列など他のデータ型も格納できます。文字列配列は、名前のリストや選択肢の管理などに便利です。 

Module Module1
    Sub Main()
        ' 曜日の配列
        Dim weekdays() As String = {"月曜日", "火曜日", "水曜日", "木曜日", "金曜日", "土曜日", "日曜日"}

        Console.WriteLine("今週の予定:")
        For i As Integer = 0 To weekdays.Length - 1
            Console.WriteLine((i + 1) & "日目: " & weekdays(i))
        Next
        Console.WriteLine()

        ' 名前のリスト
        Dim members() As String = {"田中さん", "佐藤さん", "鈴木さん"}

        Console.WriteLine("メンバー一覧:")
        For Each member As String In members
            Console.WriteLine("- " & member)
        Next

        Console.ReadLine()
    End Sub
End Module

実行結果

2日目: 火曜日
3日目: 水曜日
4日目: 木曜日
5日目: 金曜日
6日目: 土曜日
7日目: 日曜日

- 田中さん
- 佐藤さん
- 鈴木さん

よくあるトラブルと対処法

❌ IndexOutOfRangeException が発生する

原因: 配列の範囲外のインデックスにアクセスしようとしています。

対処: インデックスが0以上、かつLength未満であることを確認してください。For文のループ条件を「0 To scores.Length - 1」にしましょう。

❌ 配列の要素を変更したのに反映されない（For Each使用時）

原因: For Each文で取得した変数は、配列要素のコピーです。

対処: 配列の要素を変更したい場合は、For文を使ってインデックス経由でアクセスしてください。

❌ 配列のサイズを間違える

原因: Dim scores(5)と宣言すると、インデックス0～5の6個の要素ができます。

対処: 5個の要素が欲しい場合は「Dim scores(4)」と宣言するか、初期値を指定する方法を使いましょう。

理解度確認クイズ

  1. Q1. 配列とは何ですか？なぜ配列が必要なのですか？

    回答を表示

    配列は、同じ種類のデータを1つのまとまりとして管理できる仕組みです。複数の変数を個別に宣言する必要がなく、ループを使って効率的に処理できるため、大量のデータを扱う場合に便利です。

  2. Q2. 次のコードの実行結果を答えてください。

    Dim numbers() As Integer = {10, 20, 30}
    Console.WriteLine(numbers(1))
    回答を表示

    実行結果: 20
    配列のインデックスは0から始まるため、numbers(1)は2番目の要素である20を指します。

  3. Q3. For文とFor Each文の使い分けはどうすればよいですか？

    回答を表示

    For文は、インデックスが必要な場合や配列の要素を変更したい場合に使います。For Each文は、配列の全要素を順番に読み取るだけの場合に使い、コードが簡潔になります。For Each文では要素の変更はできません。

第9章のまとめ

  • 配列は、同じ種類のデータをまとめて管理できる仕組みです。各要素にはインデックス（0から始まる番号）でアクセスします。
  • 配列の宣言には、要素数を指定する方法と、初期値を指定する方法があります。初期値を指定する方が簡潔です。
  • 配列の要素数はLengthプロパティで取得できます。最後の要素のインデックスは「Length - 1」です。
  • For文は配列の要素を変更する場合、For Each文は全要素を読み取る場合に適しています。
