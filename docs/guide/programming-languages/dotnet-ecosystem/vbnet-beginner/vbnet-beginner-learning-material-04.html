第4章

はじめてのプログラムを作ろう

「Hello World」プログラムを実際に作成し、プログラムの入力から実行までの流れを体験します。エラーが出たときの対処法も合わせて学びます。

この章で学ぶこと

  • はじめてのプログラム「Hello World」を作成する流れを理解する
  • コンソールアプリケーションプロジェクトの作成方法を学ぶ
  • VB.NETコードの入力と実行方法を体験する
  • エラーが発生したときの対処法を身につける

4.1 「Hello World」プログラムとは

プログラミングを学び始めるとき、最初に作るプログラムは「Hello World」と呼ばれる簡単なプログラムです。これは、画面に「Hello World」という文字を表示するだけの、とてもシンプルなプログラムです。

なぜこのような簡単なプログラムから始めるのでしょうか？理由は次のとおりです。

開発環境の確認

Visual Studioが正しくインストールされているか、プログラムを実行できる状態になっているかを確認できます。

基本的な流れの理解

プログラムを書く → 実行する → 結果を確認する、という基本的な流れを体験できます。

達成感を味わう

自分が書いたコードが実際に動くのを見ることで、プログラミング学習への第一歩を踏み出した実感が得られます。

構文の理解

プログラムの基本的な構造（モジュール、プロシージャ、命令文など）を実際のコードで理解できます。

ポイント

「Hello World」は単純に見えますが、プログラミングの基本がすべて詰まっています。これから学ぶ複雑なプログラムも、基本的な流れは同じです。

4.2 コンソールアプリケーションプロジェクトを作成する

それでは、実際にプログラムを作成してみましょう。まずは、Visual Studioで新しいプロジェクトを作成します。コンソールアプリケーションとは、黒い画面（コマンドプロンプト）に文字を表示するタイプのプログラムです。

実習 4-1: コンソールアプリケーションプロジェクトの作成

Visual Studioで新しいプロジェクトを作成し、Hello Worldプログラムの土台を準備します。

手順

  1. 1 Visual Studio 2022を起動します。
  2. 2 「新しいプロジェクトの作成」をクリックします。
  3. 3 検索ボックスに「コンソール」と入力し、「コンソール アプリ (.NET Framework)」を選択します。言語が「Visual Basic」になっていることを確認してください。
  4. 4 「次へ」をクリックします。
  5. 5 プロジェクト名に「HelloWorld」と入力し、保存先フォルダを確認します（デフォルトのままで構いません）。
  6. 6 「作成」をクリックします。
  7. 7 フレームワークは「.NET Framework 4.8」（または最新バージョン）を選択し、「作成」をクリックします。

注意

「コンソール アプリ」と「コンソール アプリ (.NET Framework)」は異なります。必ず「.NET Framework」が付いているものを選択してください。また、言語が「Visual Basic」になっていることも確認しましょう。

4.3 Hello Worldプログラムを書く

プロジェクトが作成されると、Visual Studioのエディタ画面に、いくつかのコードが自動的に表示されています。これは、VB.NETのプログラムの基本的な構造です。この中に、私たちのコードを書き加えていきます。

自動生成されるコードの説明

プロジェクト作成直後、Module1.vbファイルには以下のようなコードが表示されています。

Module1.vb（初期状態） 
Module Module1

    Sub Main()

    End Sub

End Module
Module Module1: プログラムのまとまりを表す「モジュール」の開始
Sub Main(): プログラムの実行開始地点（メインプロシージャ）
End Sub: プロシージャの終了
End Module: モジュールの終了

実習 4-2: Hello Worldコードの入力

Sub Main()とEnd Subの間に、「Hello World」と表示するコードを追加します。

手順

  1. 1 Sub Main() と End Sub の間にカーソルを置きます。
  2. 2 以下のコードを入力します（スペースや大文字・小文字に注意してください）。
Module1.vb（完成版） 
Module Module1

    Sub Main()
        ' Hello Worldと表示する
        Console.WriteLine("Hello World")
    End Sub

End Module

コードの読み方

' Hello Worldと表示する

アポストロフィ（'）で始まる行はコメントです。プログラムの動作には影響せず、コードの説明を書くために使います。

Console.WriteLine("Hello World")

コンソール画面に文字を表示する命令です。Consoleはコンソール画面を表し、WriteLineは「1行書いて改行する」という意味です。

("Hello World")

ダブルクォーテーション（"）で囲んだ部分は文字列です。表示したい文字をここに書きます。

重要ポイント

VB.NETでは、大文字と小文字は区別されません（console.writeline と書いても動きます）。ただし、読みやすさのため、一般的な書き方に従うことをお勧めします。

4.4 プログラムを実行する

コードを書いたら、実際にプログラムを実行してみましょう。Visual Studioでは、簡単な操作でプログラムを実行できます。

実習 4-3: プログラムの実行

作成したプログラムを実行し、結果を確認します。

実行方法

  1. 1 キーボードで F5 キーを押す、または
  2. 2 画面上部の緑色の「▶ 開始」ボタンをクリックします。

動作確認

プログラムが正しく動作すると、黒いコンソール画面が表示され、以下のように表示されます。

期待される結果 

Hello World

ヒント

コンソール画面がすぐに閉じてしまう場合は、Console.ReadLine() を最後の行（End Sub の直前）に追加してください。これにより、Enterキーを押すまで画面が表示されたままになります。

おめでとうございます！これで、あなたは初めてのプログラムを作成し、実行することができました。この「コードを書く → 実行する → 結果を確認する」という流れは、プログラミングの基本です。

4.5 エラーが出たときの対処法

プログラミングをしていると、必ずエラーに遭遇します。エラーは怖いものではありません。エラーメッセージは、「ここが間違っていますよ」と教えてくれる親切なメッセージです。

エラーの種類

構文エラー（Syntax Error）

コードの書き方が間違っている場合に発生します。Visual Studioでは、構文エラーのある行に赤い波線が表示されます。

例: ダブルクォーテーションの閉じ忘れ、スペルミスなど

実行時エラー（Runtime Error）

プログラムを実行した際に発生するエラーです。コードは正しく書けていますが、実行時に問題が発生する場合に起こります。

例: 0で割り算をする、存在しないファイルを開くなど

よくあるエラーと対処法

「Console」に赤い波線が表示される

原因: 「Console」のスペルが間違っているか、大文字・小文字が違う可能性があります。

解決方法: 「Console」のスペルを確認してください。VB.NETでは大文字・小文字は関係ありませんが、正しいスペル（Console）で入力する必要があります。

「End Sub が必要です」というエラーが出る

原因: Sub Main() を開始したのに、End Sub で閉じていません。

解決方法: Sub Main() の後に、必ず End Sub を書いてください。Visual Studioでは、Sub Main() と入力してEnterを押すと、自動的に End Sub が追加されます。

ダブルクォーテーションに関するエラーが出る

原因: 文字列を囲むダブルクォーテーション（"）が閉じられていません。

解決方法: "Hello World" のように、文字列の前後に必ずダブルクォーテーションを付けてください。開始の " と終了の " が対になっているか確認しましょう。

実行しても何も表示されない

原因: Console.WriteLine の行が、Sub Main() と End Sub の間に書かれていないか、コメント行になっている可能性があります。

解決方法: コードが Sub Main() と End Sub の間に書かれているか確認してください。また、行頭にアポストロフィ（'）がないか確認しましょう（あるとコメントとして無視されます）。

重要ポイント

エラーが出たら、まずエラーメッセージをよく読みましょう。何行目でエラーが起きているか、何が問題なのかを教えてくれます。また、赤い波線が表示されている箇所にマウスカーソルを合わせると、詳細な説明が表示されます。

4.6 コードを書くときのルール

プログラムを正しく動かすために、いくつかのルールがあります。これらを守ることで、エラーを減らし、読みやすいコードを書くことができます。

インデント（字下げ）

Sub と End Sub の間のコードは、Tabキーまたはスペースでインデント（字下げ）します。これにより、どこからどこまでが Sub の中かが一目でわかります。

ペアになるキーワード

Sub には End Sub、Module には End Module のように、開始と終了がペアになっています。必ず対になるように書きましょう。

コメントの活用

アポストロフィ（'）で始まる行はコメントです。コードの説明を書くことで、後で見返したときに理解しやすくなります。

文字列はダブルクォーテーションで囲む

表示したい文字（文字列）は、必ず "～" のようにダブルクォーテーションで囲みます。これにより、コンピュータが「これは表示する文字だ」と認識します。

良いコードの例 
Module Module1

    Sub Main()
        ' プログラムの開始を知らせるメッセージ
        Console.WriteLine("プログラムを開始します")

        ' Hello Worldを表示
        Console.WriteLine("Hello World")

        ' プログラムの終了を知らせるメッセージ
        Console.WriteLine("プログラムを終了します")
    End Sub

End Module

このように、コメントを追加し、適切にインデントを付けることで、読みやすく理解しやすいコードになります。

理解度確認クイズ

  1. Q1. Console.WriteLine("Hello World") の役割は何ですか？

    回答を表示

    コンソール画面に「Hello World」という文字列を表示し、改行する命令です。Consoleはコンソール画面を、WriteLineは「1行書いて改行」を意味します。

  2. Q2. アポストロフィ（'）で始まる行は何と呼ばれますか？

    回答を表示

    コメント（注釈）と呼ばれます。プログラムの動作には影響せず、コードの説明を書くために使います。' で始まる行は、プログラム実行時に無視されます。

  3. Q3. Sub Main() と End Sub の間に書くコードは、どうする必要がありますか？

    回答を表示

    インデント（字下げ）する必要があります。Tabキーまたはスペースでインデントすることで、どこからどこまでがSubプロシージャの中かが分かりやすくなります。Visual Studioでは自動的にインデントされることが多いです。

  4. Q4. エラーが出たとき、まず何を確認すべきですか？

    回答を表示

    エラーメッセージをよく読み、どの行でエラーが発生しているか、何が問題なのかを確認します。また、赤い波線が表示されている箇所にマウスカーソルを合わせると、詳細な説明が表示されます。スペルミス、ダブルクォーテーションの閉じ忘れ、End Subの書き忘れなどがよくある原因です。

第4章のまとめ

  • 「Hello World」プログラムは、プログラミング学習の第一歩として、開発環境の確認と基本的な流れの理解に役立ちます。
  • コンソールアプリケーションプロジェクトを作成し、Sub Main()の中にコードを書くことでプログラムを作成できます。
  • Console.WriteLine("文字列") で、コンソール画面に文字を表示できます。
  • エラーは怖くありません。エラーメッセージを読み、原因を特定して修正することで、プログラミングスキルが向上します。
  • インデント、コメント、ペアになるキーワードなど、コードを書くときのルールを守ることで、読みやすいコードが書けます。
