Visual Studioの基本操作を覚えよう

Visual Studioの画面構成、各パネルの役割、プロジェクトの作成方法を学びます。開発環境に慣れることで、今後のプログラミング学習がスムーズになります。

この章で学ぶこと

  • Visual Studioの起動方法と画面構成を理解する
  • 各ウィンドウやパネルの役割を知る
  • 新しいプロジェクトの作成手順を実践する
  • 開発に便利なショートカットキーを覚える

3.1 Visual Studioを起動しよう

前章でインストールしたVisual Studioを起動してみましょう。Windowsのスタートメニューから「Visual Studio 2022」を探してクリックします。

起動手順

  1. 1 Windowsのスタートボタンをクリック
  2. 2 「Visual Studio 2022」を検索するか、アプリ一覧から探す
  3. 3 「Visual Studio 2022」をクリックして起動

初回起動時は、サインインを求められる場合があります。サインインは任意ですが、Microsoftアカウントでサインインすると、設定が同期されて便利です。「後で行う」を選択してスキップすることもできます。

ヒント

初回起動時は、設定のロードに時間がかかることがあります。焦らず、画面が表示されるまで待ちましょう。

3.2 スタート画面の見方

Visual Studioを起動すると、最初に「スタート画面」が表示されます。この画面から新しいプロジェクトを作成したり、既存のプロジェクトを開いたりできます。

スタート画面の主な項目

新しいプロジェクトの作成

新しいプログラムのプロジェクトを作成します。最もよく使う機能です。

プロジェクトまたはソリューションを開く

以前に作成したプロジェクトを開きます。

リポジトリのクローン

Git（バージョン管理システム）からプロジェクトをコピーします。後の章で学びます。

最近使ったプロジェクト

最近開いたプロジェクトの履歴が表示されます。頻繁に使うプロジェクトにすばやくアクセスできます。

3.3 新しいプロジェクトを作成しよう

それでは、実際に新しいプロジェクトを作成してみましょう。今回は、VB.NETの基本を学ぶための「コンソールアプリケーション」を作成します。

実習 3-1: コンソールアプリケーションプロジェクトの作成

実際にVisual Studioで新しいプロジェクトを作成してみましょう。手順を一つずつ確認しながら進めてください。

手順

  1. 1 スタート画面で「新しいプロジェクトの作成」をクリック
  2. 2 プロジェクトテンプレート選択画面で、検索ボックスに「VB コンソール」と入力
  3. 3 「コンソールアプリ（Visual Basic）」を選択して、「次へ」をクリック
  4. 4 プロジェクト名に「MyFirstProject」と入力（任意の名前でもOK）
  5. 5 保存場所は、デフォルトのまま（C:\Users\ユーザー名\source\repos）でOK
  6. 6 「作成」ボタンをクリック

重要ポイント

プロジェクト名は、半角英数字で付けることをお勧めします。日本語名も使えますが、一部のツールで問題が起こることがあります。

メニューバー

画面上部にある「ファイル」「編集」「表示」などのメニュー。すべての機能にアクセスできます。

ツールバー

よく使う操作（保存、実行、デバッグなど）を素早く実行できるボタン群です。

ソリューションエクスプローラー

プロジェクトに含まれるファイルやフォルダを階層構造で表示します。通常、画面の右側にあります。ファイルをダブルクリックすると、コードエディタで開けます。

コードエディタ

画面中央の、プログラムコードを書く場所です。文字を色分けして表示してくれたり、入力補完をしてくれたりする便利な機能があります。

プロパティウィンドウ

選択したファイルやコントロールの設定を表示・変更できます。通常、画面の右下にあります。

エラー一覧ウィンドウ

コードに間違いがあると、ここにエラーや警告が表示されます。画面下部にあります。エラーをダブルクリックすると、該当する行にジャンプできます。

出力ウィンドウ

プログラムの実行結果やビルド（コンパイル）のメッセージが表示されます。画面下部にあり、エラー一覧とタブで切り替えられます。

ヒント

各ウィンドウは、ドラッグして位置を変更したり、閉じたりできます。閉じてしまったウィンドウは、メニューバーの「表示」から再度表示できます。

3.5 ソリューションとプロジェクトの違い

Visual Studioでは、「ソリューション」と「プロジェクト」という2つの単位でプログラムを管理しています。初心者にとって分かりにくい概念ですが、とても重要なので理解しておきましょう。

プロジェクト

1つのアプリケーションやライブラリを作るための単位です。コードファイル、設定ファイル、画像などのリソースをまとめたものです。最終的に、1つの.exeファイルや.dllファイルになります。

ソリューション

複数のプロジェクトをまとめる「入れ物」のようなものです。関連する複数のプロジェクトを1つのソリューションに含めることができます。初心者のうちは、1つのソリューションに1つのプロジェクトだけ入れることが多いです。

重要ポイント

初心者のうちは、「1つのソリューション = 1つのプロジェクト」と考えておけば大丈夫です。プロジェクトを作成すると、自動的にソリューションも作られます。

3.6 ウィンドウのレイアウトをカスタマイズしよう

Visual Studioの各ウィンドウは、自分の好みに合わせて位置やサイズを変更できます。使いやすいレイアウトを見つけて、作業効率を上げましょう。

ウィンドウ操作の基本

移動

ウィンドウのタイトルバーをドラッグすると、好きな位置に移動できます。ドラッグ中に画面の端や中央に近づけると、青い半透明の領域が表示され、そこにドッキング（固定）できます。

サイズ変更

ウィンドウの境界線にマウスを合わせると、カーソルが矢印に変わります。その状態でドラッグすると、サイズを変更できます。

自動非表示

ウィンドウのタイトルバーにある「ピン」アイコンをクリックすると、自動非表示になります。使わないときは画面の端に収納され、マウスを合わせると表示されます。

閉じる

ウィンドウの「×」ボタンで閉じることができます。閉じたウィンドウは、メニューバーの「表示」から再表示できます。

注意

ウィンドウレイアウトを変更しすぎて、元に戻せなくなってしまった場合は、メニューバーの「ウィンドウ」→「ウィンドウレイアウトのリセット」で初期状態に戻せます。

3.7 覚えておきたいショートカットキー

Visual Studioには、キーボードだけで操作できる「ショートカットキー」がたくさんあります。よく使うものを覚えておくと、作業がぐっと速くなります。

初心者が最初に覚えたいショートカットキー

操作 ショートカットキー 説明
保存 Ctrl + S 編集中のファイルを保存します。こまめに保存する習慣をつけましょう。
すべて保存 Ctrl + Shift + S 開いているすべてのファイルを保存します。
元に戻す Ctrl + Z 直前の操作を元に戻します。
やり直し Ctrl + Y 元に戻した操作をやり直します。
コピー Ctrl + C 選択したテキストをコピーします。
貼り付け Ctrl + V コピーしたテキストを貼り付けます。
切り取り Ctrl + X 選択したテキストを切り取ります。
コメント化 Ctrl + K, Ctrl + C 選択した行をコメントアウトします。
コメント解除 Ctrl + K, Ctrl + U コメントアウトを解除します。
ビルド Ctrl + Shift + B プロジェクトをビルド（コンパイル）します。
実行 F5 プログラムをデバッグモードで実行します。
デバッグなしで実行 Ctrl + F5 プログラムをデバッグなしで実行します。

重要ポイント

最初は「Ctrl + S（保存）」と「F5（実行）」だけでも覚えておけば十分です。使っていくうちに、自然と他のショートカットも覚えていきます。

理解度確認クイズ

  1. Q1. Visual Studioで新しいプロジェクトを作成するとき、最初にクリックするボタンは何ですか?

    回答を表示

    スタート画面の「新しいプロジェクトの作成」ボタンをクリックします。

  2. Q2. プログラムコードを書く画面の名前は何ですか?

    回答を表示

    コードエディタです。画面中央に配置され、文字の色分けや入力補完などの便利な機能があります。

  3. Q3. ソリューションとプロジェクトの違いを簡単に説明してください。

    回答を表示

    プロジェクトは1つのアプリケーションを作るための単位で、ソリューションは複数のプロジェクトをまとめる「入れ物」です。初心者のうちは、1つのソリューションに1つのプロジェクトが入っていることが多いです。

  4. Q4. ファイルを保存するショートカットキーは何ですか?

    回答を表示

    Ctrl + S です。こまめに保存する習慣をつけましょう。
