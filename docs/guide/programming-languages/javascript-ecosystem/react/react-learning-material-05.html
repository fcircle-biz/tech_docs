5.1 mapメソッドによるリストレンダリング

Reactでは、JavaScriptの配列メソッドmap()を使用して、配列データをJSX要素のリストに変換します。これは、Reactで動的なリストを作成する標準的な方法です。

実習 5-1: 基本的なリストレンダリング
シンプルなリストの表示
function FruitList() {
  const fruits = ["りんご", "バナナ", "オレンジ", "ぶどう"];

  return (
    <ul>
      {fruits.map((fruit, index) => (
        <li key={index}>{fruit}</li>
      ))}
    </ul>
  );
}
オブジェクト配列のレンダリング
function UserList() {
  const users = [
    { id: 1, name: "山田太郎", age: 25 },
    { id: 2, name: "佐藤花子", age: 30 },
    { id: 3, name: "鈴木一郎", age: 28 }
  ];

  return (
    <div>
      {users.map(user => (
        <div key={user.id}>
          <h3>{user.name}</h3>
          <p>年齢: {user.age}歳</p>
        </div>
      ))}
    </div>
  );
}

5.2 keyプロパティの重要性

Reactは、リスト内の各要素を識別するためにkeyプロパティを使用します。keyは、どの要素が変更・追加・削除されたかをReactが効率的に判断するために必要です。

keyを指定しないとどうなる？

keyを指定しないと、コンソールに警告が表示されます。さらに、リストの更新時にパフォーマンスの問題や予期しないバグが発生する可能性があります。

keyの役割

  • 要素の識別：Reactがどの要素が変更されたかを特定
  • 効率的な更新：変更された要素のみを更新
  • コンポーネントのState保持：要素の順序が変わってもStateを維持

5.3 適切なkeyの選択

実習 5-2: 正しいkeyの使用
✅ 推奨：一意なIDをkeyに使用
const products = [
  { id: "p1", name: "ノートパソコン" },
  { id: "p2", name: "マウス" },
  { id: "p3", name: "キーボード" }
];

return products.map(product => (
  <div key={product.id}>{product.name}</div>
));
⚠️ 避けるべき：インデックスをkeyに使用
// 要素の順序が変わらない場合のみ使用可能
return items.map((item, index) => (
  <div key={index}>{item}</div>
));
インデックスをkeyに使うべきでないケース
  • リストの順序が変わる可能性がある
  • 要素が追加・削除される
  • リストがソートされる

これらの場合、一意なIDを使用してください。

5.4 ToDoリストの実装 - 追加・削除

実習 5-3: 動的なToDoリストの作成
import { useState } from 'react';

function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: "買い物に行く", completed: false },
    { id: 2, text: "レポートを書く", completed: false }
  ]);
  const [inputText, setInputText] = useState("");

  // ToDoの追加
  const addTodo = () => {
    if (inputText.trim()) {
      const newTodo = {
        id: Date.now(),  // 簡易的な一意なID生成
        text: inputText,
        completed: false
      };
      setTodos([...todos, newTodo]);
      setInputText("");
    }
  };

  // ToDoの削除
  const deleteTodo = (id) => {
    setTodos(todos.filter(todo => todo.id !== id));
  };

  // 完了状態の切り替え
  const toggleTodo = (id) => {
    setTodos(todos.map(todo =>
      todo.id === id
        ? { ...todo, completed: !todo.completed }
        : todo
    ));
  };

  return (
    <div>
      <h2>ToDoリスト</h2>
      <div>
        <input
          value={inputText}
          onChange={(e) => setInputText(e.target.value)}
          placeholder="新しいToDoを入力"
        />
        <button onClick={addTodo}>追加</button>
      </div>
      <ul>
        {todos.map(todo => (
          <li key={todo.id}>
            <input
              type="checkbox"
              checked={todo.completed}
              onChange={() => toggleTodo(todo.id)}
            />
            <span style={{
              textDecoration: todo.completed ? 'line-through' : 'none'
            }}>
              {todo.text}
            </span>
            <button onClick={() => deleteTodo(todo.id)}>削除</button>
          </li>
        ))}
      </ul>
    </div>
  );
}

5.5 フィルタリングとソート

実習 5-4: フィルタリング機能の追加
function FilteredList() {
  const [items] = useState([
    { id: 1, name: "りんご", category: "果物", price: 200 },
    { id: 2, name: "にんじん", category: "野菜", price: 150 },
    { id: 3, name: "バナナ", category: "果物", price: 100 }
  ]);
  const [filter, setFilter] = useState("全て");

  // フィルタリング
  const filteredItems = items.filter(item =>
    filter === "全て" || item.category === filter
  );

  // 価格でソート
  const sortedItems = [...filteredItems].sort((a, b) =>
    a.price - b.price
  );

  return (
    <div>
      <select value={filter} onChange={(e) => setFilter(e.target.value)}>
        <option value="全て">全て</option>
        <option value="果物">果物</option>
        <option value="野菜">野菜</option>
      </select>
      <ul>
        {sortedItems.map(item => (
          <li key={item.id}>
            {item.name} - {item.price}円 ({item.category})
          </li>
        ))}
      </ul>
    </div>
  );
}
