1.1 Reactとは何か - 設計思想と哲学

Reactは、Meta（旧Facebook）が開発したJavaScriptライブラリで、ユーザーインターフェース（UI）を構築するために設計されています。2013年にオープンソース化されて以来、世界中の開発者に支持され、現代のWebアプリケーション開発において最も人気のあるライブラリの1つとなっています。

Reactの核となる設計思想

宣言的（Declarative）

「何をしたいか」を記述するだけで、「どのように実現するか」の詳細はReactが処理します。これにより、コードが読みやすく、予測可能で、デバッグしやすくなります。

コンポーネントベース（Component-Based）

UIを独立した再利用可能なコンポーネントに分割し、それらを組み合わせて複雑なUIを構築します。各コンポーネントは自身の状態を管理し、独立してテスト可能です。

一度学習すれば、どこでも書ける（Learn Once, Write Anywhere）

Reactで学んだ知識は、Web、モバイル（React Native）、デスクトップアプリ、さらにはVR（React 360）など、様々なプラットフォームで活用できます。

なぜReactが選ばれるのか

  • 高いパフォーマンス: 仮想DOMにより、必要最小限のDOM操作で高速なUI更新を実現
  • 豊富なエコシステム: React Router、Redux、TanStack Queryなど、充実したライブラリ
  • 大規模コミュニティ: 活発なコミュニティによる継続的なサポートと改善
  • 企業での採用実績: Meta、Netflix、Airbnb、Uberなど、大手企業での採用

1.2 コンポーネントベース開発の利点

コンポーネントは、Reactアプリケーションの基本的な構成要素です。UIを小さく再利用可能な部品に分割することで、開発効率と保守性が大幅に向上します。

コンポーネント分割の例

コンポーネント化のメリット

再利用性

一度作成したコンポーネントを複数の場所で再利用できます。例えばButtonコンポーネントを作成すれば、アプリ全体で統一されたボタンを使用できます。

保守性

各コンポーネントが独立しているため、変更の影響範囲が限定的です。バグの特定も容易になります。

テスト容易性

小さく独立したコンポーネントは、単体テストが書きやすく、品質を保ちやすくなります。

チーム開発

コンポーネント単位で作業を分担できるため、複数人での並行開発がスムーズになります。

Button.tsx（再利用可能なButtonコンポーネント例） 
// 再利用可能なButtonコンポーネント
interface ButtonProps {
  children: React.ReactNode;
  onClick?: () => void;
  variant?: 'primary' | 'secondary';
}

function Button({ children, onClick, variant = 'primary' }: ButtonProps) {
  const baseStyle = 'px-4 py-2 rounded font-medium transition-colors';
  const variantStyle = variant === 'primary'
    ? 'bg-blue-500 text-white hover:bg-blue-600'
    : 'bg-gray-200 text-gray-800 hover:bg-gray-300';

  return (
    <button className={`${baseStyle} ${variantStyle}`} onClick={onClick}>
      {children}
    </button>
  );
}

// アプリ内の様々な場所で再利用
function App() {
  return (
    <div>
      <Button onClick={() => alert('保存しました')}>保存</Button>
      <Button variant="secondary">キャンセル</Button>
    </div>
  );
}

1.3 仮想DOMとReactファイバー

Reactが高速である理由の1つが、仮想DOM（Virtual DOM）の仕組みです。実際のDOM操作はコストが高いため、Reactは仮想的なDOM表現をメモリ上に保持し、差分のみを実際のDOMに反映します。

仮想DOMの動作フロー

Diffing"] C --> D["変更箇所のみ
実DOMに反映
Reconciliation"] D --> E["画面更新"]

Reactファイバー（React Fiber）

React 16以降では、ファイバーアーキテクチャが導入され、レンダリングプロセスがより洗練されました。ファイバーにより、次のような機能が実現されています：

  • インクリメンタルレンダリング: レンダリング作業を小さな単位に分割し、優先度に基づいて実行
  • 作業の中断と再開: 高優先度のタスク（ユーザー操作など）を優先処理
  • 並行モード（Concurrent Mode）: 複数のレンダリング作業を並行して処理

DOMとは？

DOM（Document Object Model）は、HTML文書をツリー構造として表現したものです。JavaScriptからHTML要素を操作する際に使用しますが、DOMの直接操作は処理コストが高いため、Reactは仮想DOMを使って最適化しています。

1.4 宣言的UIの概念

Reactの最大の特徴の1つが、宣言的（Declarative）なUI記述です。命令的（Imperative）なアプローチとの違いを理解することで、Reactの設計思想がより明確になります。

命令的 vs 宣言的

命令的アプローチ（従来のDOM操作）

「どのように実現するか」を詳細に記述 

// 命令的：手順を逐一指定
const button = document.getElementById('btn');
button.addEventListener('click', () => {
  const counter = document.getElementById('count');
  const current = parseInt(counter.innerText);
  counter.innerText = current + 1;
});

宣言的アプローチ（React）

「何を表示したいか」だけを記述 

// 宣言的：望む状態を記述
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

宣言的UIのメリット

  • コードの可読性向上: 「現在の状態で何を表示するか」が一目で分かる
  • 予測可能性: 状態が同じなら、常に同じUIが生成される
  • デバッグの容易さ: 問題の原因が状態にあるか、レンダリングロジックにあるか特定しやすい

主要ライブラリの役割

React Router

SPA（Single Page Application）でのページ遷移を管理。ブラウザの履歴APIと統合し、URL駆動のナビゲーションを実現します。

TanStack Query（旧React Query）

サーバーからのデータ取得、キャッシング、同期を自動化。複雑な非同期データ管理をシンプルに実装できます。

UIコンポーネントライブラリ

Material-UI、Chakra UI、Ant Designなど、事前に用意されたUIコンポーネントを活用して開発を加速。

1.6 SPAとMPAの違い

Reactは主にSPA（Single Page Application）の構築に使用されます。従来のMPA（Multi Page Application）との違いを理解しておきましょう。

SPA（Single Page Application）

1つのHTMLページで構成され、JavaScriptで動的にコンテンツを更新

メリット

  • 高速なページ遷移
  • スムーズなユーザー体験
  • 状態の保持が容易

デメリット

  • 初回読み込みが遅い
  • SEO対策に工夫が必要

MPA（Multi Page Application）

複数のHTMLページで構成され、サーバーが各ページをレンダリング

メリット

  • SEOに強い
  • 初回表示が速い
  • シンプルな構造

デメリット

  • ページ遷移が遅い
  • 状態管理が複雑

ハイブリッドアプローチ

Next.jsなどのフレームワークを使えば、SPAとMPAの良いとこ取りが可能です。SSR（サーバーサイドレンダリング）やSSG（静的サイト生成）により、初回表示の高速化とSEO対策を両立できます。

実習 1-1: はじめてのReactコンポーネント

Viteで作成したプロジェクトで、シンプルなカウンターコンポーネントを作成してみましょう。

手順

  1. 1 src/App.tsx を開く
  2. 2 以下のコードに置き換える
  3. 3 保存してブラウザで動作を確認

実装コード

import { useState } from 'react';
import './App.css';

function App() {
  // 状態の定義
  const [count, setCount] = useState(0);

  // イベントハンドラー
  const handleIncrement = () => {
    setCount(count + 1);
  };

  const handleDecrement = () => {
    setCount(count - 1);
  };

  const handleReset = () => {
    setCount(0);
  };

  return (
    <div className="App">
      <h1>Reactカウンターアプリ</h1>
      <div className="card">
        <h2>カウント: {count}</h2>
        <div style={{ display: 'flex', gap: '10px', justifyContent: 'center' }}>
          <button onClick={handleIncrement}>+1</button>
          <button onClick={handleDecrement}>-1</button>
          <button onClick={handleReset}>リセット</button>
        </div>
      </div>
    </div>
  );
}

export default App;

期待される結果

ブラウザに3つのボタン（+1、-1、リセット）とカウント表示が現れます。各ボタンをクリックすると、カウントが増減・リセットされます。

理解度確認クイズ

  1. Q1. Reactの設計思想として正しくないものはどれですか？

    回答を表示

    回答: 正解は「命令的（Imperative）」です。Reactは宣言的（Declarative）なUI記述を採用しています。

  2. Q2. 仮想DOMの主な利点は何ですか？

    回答を表示

    回答: 仮想DOMは、変更があった部分だけを実際のDOMに反映することで、DOM操作のコストを最小化し、パフォーマンスを向上させます。

  3. Q3. SPAとMPAの最も大きな違いは何ですか？

    回答を表示

    回答: SPAは1つのHTMLページで構成され、JavaScriptで動的にコンテンツを更新します。MPAは複数のHTMLページで構成され、ページ遷移のたびにサーバーから新しいページを取得します。

  4. Q4. useStateフックは何をするためのものですか？

    回答を表示

    回答: useStateは、関数コンポーネント内で状態（state）を管理するためのReact Hookです。状態が変更されると、コンポーネントが再レンダリングされます。
