第4章: イベント処理とユーザーインタラクション

インタラクティブなUIを作る - イベント処理

この章で学ぶこと
  • Reactのイベントハンドリングの基礎を理解する
  • onClick、onChange、onSubmitなど主要なイベントの使い方を学ぶ
  • イベントハンドラーの定義方法を習得する
  • イベントオブジェクトと合成イベント（SyntheticEvent）を理解する
  • フォーム要素の制御（制御されたコンポーネント）を学ぶ

4.1 Reactのイベントハンドリング

Reactでは、HTMLのイベント処理と似た方法でユーザーの操作に応答できますが、いくつか重要な違いがあります。

HTMLのイベント処理との違い

  • キャメルケース：イベント名はキャメルケースで記述（例：onclick → onClick）
  • 関数を渡す：文字列ではなく、関数を渡す
  • preventDefaultの呼び出し：return falseでは無効化できない
// HTML の書き方
<button onclick="handleClick()">クリック</button>

// React の書き方
<button onClick={handleClick}>クリック</button>

4.2 onClick - クリックイベント

実習 4-1: クリックイベントの実装

ボタンクリックに応答するシンプルなコンポーネントを作成します。

基本的なonClick
import { useState } from 'react';

function ClickCounter() {
  const [count, setCount] = useState(0);

  // イベントハンドラー関数
  const handleClick = () => {
    setCount(count + 1);
  };

  return (
    <div>
      <p>クリック回数: {count}</p>
      <button onClick={handleClick}>クリック</button>
    </div>
  );
}
インライン関数での記述
function ClickCounter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>クリック回数: {count}</p>
      {/* インラインでアロー関数を使用 */}
      <button onClick={() => setCount(count + 1)}>
        クリック
      </button>
    </div>
  );
}
よくある間違い：関数の即時実行
// ❌ 間違い：レンダリング時に即座に実行される
<button onClick={handleClick()}>クリック</button>

// ✅ 正しい：関数の参照を渡す
<button onClick={handleClick}>クリック</button>

// ✅ 正しい：引数を渡したい場合はアロー関数で囲む
<button onClick={() => handleClick(param)}>クリック</button>

4.3 onChange - 入力変更イベント

onChangeイベントは、入力フィールドの値が変更されたときに発火します。Reactでは、この イベントを使って「制御されたコンポーネント」を実装します。

制御されたコンポーネントとは

フォーム要素の値をReactのStateで管理するコンポーネントのことを「制御されたコンポーネント」と呼びます。入力フィールドの値は常にStateと同期され、Reactが「信頼できる唯一の情報源（Single Source of Truth）」となります。

実習 4-2: 制御されたコンポーネントの作成

入力フィールドの値をStateで管理します。

基本的な入力制御
import { useState } from 'react';

function NameInput() {
  const [name, setName] = useState("");

  const handleChange = (event) => {
    // event.target.value で入力値を取得
    setName(event.target.value);
  };

  return (
    <div>
      <input
        type="text"
        value={name}  {/* State と同期 */}
        onChange={handleChange}
        placeholder="名前を入力"
      />
      <p>入力された名前: {name}</p>
    </div>
  );
}
簡潔な書き方
function NameInput() {
  const [name, setName] = useState("");

  return (
    <div>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="名前を入力"
      />
      <p>入力された名前: {name}</p>
    </div>
  );
}
flowchart LR A["ユーザーが入力"] --> B["onChange イベント発火"] B --> C["State 更新
(setName)"] C --> D["コンポーネント再レンダリング"] D --> E["入力フィールドの
value が更新"]

4.4 複数の入力フィールドの管理

フォームに複数の入力フィールドがある場合、それぞれに個別のStateとハンドラーを作成するのは非効率的です。1つのハンドラーで複数のフィールドを管理する方法を学びます。

実習 4-3: 複数フィールドの効率的な管理

1つのオブジェクトStateと汎用的なハンドラーで複数のフィールドを管理します。

name属性を活用した汎用ハンドラー
import { useState } from 'react';

function UserForm() {
  const [formData, setFormData] = useState({
    username: "",
    email: "",
    age: ""
  });

  // 汎用的な変更ハンドラー
  const handleChange = (event) => {
    const { name, value } = event.target;
    setFormData({
      ...formData,
      [name]: value  // 計算されたプロパティ名
    });
  };

  const handleSubmit = (event) => {
    event.preventDefault();  // フォームの送信を防ぐ
    console.log("送信データ:", formData);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        name="username"
        value={formData.username}
        onChange={handleChange}
        placeholder="ユーザー名"
      />
      <input
        type="email"
        name="email"
        value={formData.email}
        onChange={handleChange}
        placeholder="メールアドレス"
      />
      <input
        type="number"
        name="age"
        value={formData.age}
        onChange={handleChange}
        placeholder="年齢"
      />
      <button type="submit">送信</button>
    </form>
  );
}
重要なポイント
  • name属性とStateのプロパティ名を一致させる
  • 計算されたプロパティ名[name]を使用して動的にプロパティにアクセス
  • スプレッド構文で既存のデータを保持しつつ、特定のプロパティのみ更新

4.5 onSubmit - フォーム送信イベント

フォームの送信を処理するには、onSubmitイベントを使用します。デフォルトの送信動作を防ぐために、event.preventDefault()を呼び出す必要があります。

function LoginForm() {
  const [credentials, setCredentials] = useState({
    username: "",
    password: ""
  });

  const handleSubmit = (event) => {
    event.preventDefault();  // ページリロードを防ぐ

    // フォームの検証
    if (credentials.username && credentials.password) {
      console.log("ログイン試行:", credentials);
      // ここでAPIリクエストなどを実行
    } else {
      alert("すべてのフィールドを入力してください");
    }
  };

  const handleChange = (e) => {
    const { name, value } = e.target;
    setCredentials({ ...credentials, [name]: value });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="username"
        value={credentials.username}
        onChange={handleChange}
        placeholder="ユーザー名"
      />
      <input
        type="password"
        name="password"
        value={credentials.password}
        onChange={handleChange}
        placeholder="パスワード"
      />
      <button type="submit">ログイン</button>
    </form>
  );
}

4.6 イベントオブジェクトと合成イベント（SyntheticEvent）

Reactのイベントハンドラーには、ブラウザのネイティブイベントをラップした「合成イベント（SyntheticEvent）」オブジェクトが渡されます。これにより、ブラウザ間の互換性が保証されます。

イベントオブジェクトの主なプロパティとメソッド

  • event.target：イベントが発生した要素
  • event.target.value：入力フィールドの値
  • event.target.name：要素のname属性
  • event.preventDefault()：デフォルトの動作を防ぐ
  • event.stopPropagation()：イベントの伝播を停止
  • event.type：イベントの種類（"click"、"change"等）
function EventExample() {
  const handleEvent = (event) => {
    console.log("イベント種類:", event.type);
    console.log("対象要素:", event.target);
    console.log("入力値:", event.target.value);

    // デフォルトの動作を防ぐ
    event.preventDefault();
  };

  return (
    <form onSubmit={handleEvent}>
      <input onChange={handleEvent} />
      <button type="submit">送信</button>
    </form>
  );
}

4.7 様々なイベントタイプ

Reactでは、多くの種類のイベントをハンドリングできます。以下は頻繁に使用されるイベントの例です。

function EventDemo() {
  return (
    <div>
      {/* マウスイベント */}
      <button onClick={() => console.log("クリック")}>
        onClick
      </button>
      <button onDoubleClick={() => console.log("ダブルクリック")}>
        onDoubleClick
      </button>
      <div onMouseEnter={() => console.log("マウス侵入")}>
        onMouseEnter
      </div>

      {/* キーボードイベント */}
      <input
        onKeyDown={(e) => console.log("キー押下:", e.key)}
        onKeyUp={(e) => console.log("キー離す:", e.key)}
        placeholder="キーボードイベント"
      />

      {/* フォーカスイベント */}
      <input
        onFocus={() => console.log("フォーカス取得")}
        onBlur={() => console.log("フォーカス喪失")}
        placeholder="フォーカスイベント"
      />

      {/* スクロールイベント */}
      <div
        onScroll={() => console.log("スクロール")}
        style={{ height: 100, overflow: 'auto' }}
      >
        長いコンテンツ...
      </div>
    </div>
  );
}
理解度確認クイズ
  1. ReactのイベントハンドリングとHTMLのイベント処理の主な違いを2つ挙げてください。
  2. 「制御されたコンポーネント」とは何ですか？
  3. なぜevent.preventDefault()を使用する必要があるのですか？
  4. 複数の入力フィールドを1つのハンドラーで管理するための重要な要素は何ですか？
  5. <button onClick={handleClick()}>と<button onClick={handleClick}>の違いは何ですか？
  6. 合成イベント（SyntheticEvent）とは何ですか？
解答例
  1. ①イベント名はキャメルケース（onClickなど） ②文字列ではなく関数を渡す
  2. フォーム要素の値をReactのStateで管理するコンポーネント。入力値は常にStateと同期される
  3. フォーム送信などのデフォルトのブラウザ動作（ページリロードなど）を防ぐため
  4. 各入力要素にname属性を設定し、計算されたプロパティ名を使用してStateを動的に更新する
  5. 前者はレンダリング時に即座に実行される（間違い）。後者は関数の参照を渡すため、クリック時に実行される（正しい）
  6. ブラウザのネイティブイベントをラップしたReact独自のイベントオブジェクト。ブラウザ間の互換性を保証する
