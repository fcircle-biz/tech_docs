6.1 Reactのイベントシステム

ReactはブラウザのネイティブイベントをラップしたSyntheticEvent（合成イベント）システムを提供しています。これにより、ブラウザ間の差異を吸収し、一貫したイベント処理が可能になります。

Reactイベントの特徴

合成イベント

ブラウザのネイティブイベントをラップし、すべてのブラウザで同じインターフェースを提供します。イベントプーリングにより、パフォーマンスも最適化されています。

イベントデリゲーション

React 17以降、イベントはルートDOM要素にアタッチされ、効率的なイベント処理が実現されています。

基本的なイベント処理 
// 基本的なクリックイベント
function Button() {
  const handleClick = () => {
    console.log('ボタンがクリックされました');
  };

  return (
    <button onClick={handleClick}>
      クリック
    </button>
  );
}

// イベントオブジェクトの受け取り
function Input() {
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    console.log('入力値:', e.target.value);
  };

  return <input onChange={handleChange} />;
}

キャメルケース記法

ReactではイベントハンドラーはキャメルケースでFellowup onClick、onChange）。HTMLの小文字記法（onclick）とは異なります。

6.2 合成イベントとネイティブイベント

React の SyntheticEvent は、ブラウザのネイティブイベントと同じインターフェースを持ちながら、クロスブラウザの互換性を保証します。

flowchart TB User["ユーザー操作
クリック・入力など"] --> Native["ブラウザ
ネイティブイベント"] Native --> Synthetic["React
SyntheticEvent"] Synthetic --> Handler["イベントハンドラー
実行"] Handler --> State["状態更新"] State --> Render["再レンダリング"]

SyntheticEventのプロパティ

SyntheticEventの主要プロパティ 
function EventExample() {
  const handleEvent = (e: React.MouseEvent<HTMLButtonElement>) => {
    // イベント情報の取得
    console.log('イベント種類:', e.type);
    console.log('ターゲット要素:', e.currentTarget);
    console.log('クリック位置 X:', e.clientX);
    console.log('クリック位置 Y:', e.clientY);

    // イベントの伝播を停止
    e.stopPropagation();

    // デフォルト動作を防止
    e.preventDefault();

    // ネイティブイベントへのアクセス
    const nativeEvent = e.nativeEvent;
  };

  return <button onClick={handleEvent}>イベント詳細</button>;
}

イベントプーリングの廃止

React 17以降、イベントプーリングは廃止されました。非同期処理でイベントオブジェクトを使用する場合、e.persist()を呼ぶ必要はありません。

6.3 イベントハンドラーの型定義

TypeScriptでReactを使用する際、イベントハンドラーの型を正しく定義することで、型安全性と開発体験が向上します。

主要なイベント型

イベント型の定義例 
import React from 'react';

function EventTypeExamples() {
  // マウスイベント
  const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
    console.log('クリック座標:', e.clientX, e.clientY);
  };

  // キーボードイベント
  const handleKeyDown = (e: React.KeyboardEvent<HTMLInputElement>) => {
    if (e.key === 'Enter') {
      console.log('Enterキーが押されました');
    }
  };

  // フォームイベント
  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    console.log('フォーム送信');
  };

  // 入力変更イベント
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    console.log('入力値:', e.target.value);
  };

  return (<div>...</div>);
}
イベント型 用途 主要プロパティ
MouseEvent クリック、ホバー clientX, clientY, button
KeyboardEvent キー入力 key, keyCode, ctrlKey
ChangeEvent 入力値変更 target.value
FormEvent フォーム送信 preventDefault()
FocusEvent フォーカス管理 relatedTarget

6.4 制御コンポーネントと非制御コンポーネント

Reactのフォームには、制御コンポーネント（Controlled Components）と非制御コンポーネント（Uncontrolled Components）という2つのアプローチがあります。

制御コンポーネント vs 非制御コンポーネント

制御コンポーネント

フォームデータをReactの状態で管理。値の変更をReactが完全に制御します。

メリット

  • 値の制御が容易
  • リアルタイムバリデーション
  • 値の変換・整形が可能

非制御コンポーネント

DOMが直接フォームデータを管理。ref経由で値にアクセスします。

メリット

  • コード量が少ない
  • 既存コードとの統合が容易
  • ファイルアップロード向き
制御コンポーネントの実装 
function ControlledForm() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log({ name, email });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="名前"
      />
      <input
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="メール"
      />
      <button type="submit">送信</button>
    </form>
  );
}
非制御コンポーネントの実装 
function UncontrolledForm() {
  const nameRef = useRef<HTMLInputElement>(null);
  const emailRef = useRef<HTMLInputElement>(null);

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log({
      name: nameRef.current?.value,
      email: emailRef.current?.value,
    });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={nameRef} defaultValue="" placeholder="名前" />
      <input ref={emailRef} defaultValue="" placeholder="メール" />
      <button type="submit">送信</button>
    </form>
  );
}

推奨: 制御コンポーネント

ほとんどの場合、制御コンポーネントの使用が推奨されます。バリデーション、条件付き表示、リアルタイムフィードバックなど、より柔軟な実装が可能です。

6.5 フォームバリデーション戦略

フォームバリデーションには、複数のタイミングとアプローチがあります。ユーザー体験と実装の複雑さのバランスを考慮して選択します。

バリデーションのタイミング

flowchart LR Input["ユーザー入力"] --> OnChange["onChange
リアルタイム"] Input --> OnBlur["onBlur
フォーカス喪失時"] Input --> OnSubmit["onSubmit
送信時"] OnChange --> UX1["即座フィードバック
良いUX"] OnBlur --> UX2["入力完了後
バランス型"] OnSubmit --> UX3["送信時のみ
シンプル"]
基本的なバリデーション実装 
function ValidatedForm() {
  const [email, setEmail] = useState('');
  const [errors, setErrors] = useState<Record<string, string>>({});

  // バリデーション関数
  const validateEmail = (value: string): string | null => {
    if (!value) return '必須項目です';
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!emailRegex.test(value)) return '有効なメールアドレスを入力してください';
    return null;
  };

  // onBlurでのバリデーション
  const handleBlur = () => {
    const error = validateEmail(email);
    setErrors(prev => ({
      ...prev,
      email: error || '',
    }));
  };

  return (
    <div>
      <input
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        onBlur={handleBlur}
      />
      {errors.email && <span>{errors.email}</span>}
    </div>
  );
}

6.6 React Hook Formの実装

React Hook Formは、パフォーマンスが高く、使いやすいフォームライブラリです。非制御コンポーネントを基本としながら、柔軟なバリデーションと優れた開発体験を提供します。

インストール 
# React Hook Formのインストール
npm install react-hook-form
React Hook Formの基本的な使用 
import { useForm } from 'react-hook-form';

// フォームデータの型定義
interface FormData {
  username: string;
  email: string;
  age: number;
}

function HookFormExample() {
  const {
    register,
    handleSubmit,
    formState: { errors },
  } = useForm<FormData>();

  const onSubmit = (data: FormData) => {
    console.log('送信データ:', data);
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input
        {...register('username', { required: 'ユーザー名必須' })}
      />
      {errors.username && <span>{errors.username.message}</span>}

      <input
        {...register('email', {
          required: 'メールアドレス必須',
          pattern: {
            value: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
            message: '有効なメールアドレス形式',
          },
        })}
      />
      {errors.email && <span>{errors.email.message}</span>}

      <button type="submit">送信</button>
    </form>
  );
}

React Hook Formの主要機能

register

フォームフィールドを登録し、バリデーションルールを設定します。

handleSubmit

フォーム送信時にバリデーションを実行し、成功時にコールバックを呼び出します。

watch

フィールド値の変更を監視し、リアルタイムで値を取得できます。

6.7 Zodを使ったスキーマバリデーション

Zodは、TypeScript向けの型安全なスキーマ宣言・バリデーションライブラリです。React Hook Formと組み合わせることで、強力な型安全性を実現できます。

インストール 
# ZodとReact Hook Formの統合ライブラリ
npm install zod @hookform/resolvers
Zodスキーマでのバリデーション 
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';

// Zodスキーマ定義
const schema = z.object({
  username: z.string().min(3, 'ユーザー名は3文字以上'),
  email: z.string().email('有効なメール形式'),
  age: z.number().min(18, '18歳以上'),
  password: z.string().min(8, 'パスワードは8文字以上'),
  confirmPassword: z.string(),
}).refine(data => data.password === data.confirmPassword, {
  message: 'パスワードが一致しません',
  path: ['confirmPassword'],
});

// スキーマから型を推論
type FormData = z.infer<typeof schema>;

function ZodFormExample() {
  const { register, handleSubmit, formState: { errors } } =
    useForm<FormData>({
      resolver: zodResolver(schema),
    });

  const onSubmit = (data: FormData) => {
    console.log('送信データ:', data);
  };

  return <form onSubmit={handleSubmit(onSubmit)}>...</form>;
}

Zodの利点

スキーマから型が自動推論され、バリデーションロジックと型定義が一元化されます。複雑なバリデーション（相互依存、条件付きなど）も簡潔に記述できます。

6.8 複雑なフォーム状態管理

動的フィールド、ウィザード形式、条件付き表示など、複雑なフォームの実装パターンを理解しましょう。

動的フィールド配列

useFieldArrayによる動的フィールド 
import { useForm, useFieldArray } from 'react-hook-form';

interface FormData {
  users: { name: string; email: string }[];
}

function DynamicForm() {
  const { register, control, handleSubmit } = useForm<FormData>({
    defaultValues: {
      users: [{ name: '', email: '' }],
    },
  });

  const { fields, append, remove } = useFieldArray({
    control,
    name: 'users',
  });

  return (
    <form onSubmit={handleSubmit(data => console.log(data))}>
      {fields.map((field, index) => (
        <div key={field.id}>
          <input {...register(`users.${index}.name`)} />
          <input {...register(`users.${index}.email`)} />
          <button type="button" onClick={() => remove(index)}>
            削除
          </button>
        </div>
      ))}
      <button
        type="button"
        onClick={() => append({ name: '', email: '' })}
      >
        追加
      </button>
      <button type="submit">送信</button>
    </form>
  );
}

条件付きバリデーション

条件付きフィールド表示 
import { useForm } from 'react-hook-form';

function ConditionalForm() {
  const { register, watch, handleSubmit } = useForm();
  const userType = watch('userType');

  return (
    <form onSubmit={handleSubmit(data => console.log(data))}>
      <select {...register('userType')}>
        <option value="individual">個人</option>
        <option value="company">法人</option>
      </select>

      {userType === 'company' && (
        <input
          {...register('companyName', { required: '会社名必須' })}
          placeholder="会社名"
        />
      )}

      <button type="submit">送信</button>
    </form>
  );
}

理解度確認クイズ

  1. Q1. React の SyntheticEvent の主な利点は何ですか？

    回答を表示

    回答: ブラウザ間の差異を吸収し、すべてのブラウザで一貫したイベント処理インターフェースを提供します。また、イベントプーリングによりパフォーマンスが最適化されています（React 17以降は廃止）。

  2. Q2. 制御コンポーネントと非制御コンポーネントの主な違いは何ですか？

    回答を表示

    回答: 制御コンポーネントはReactの状態でフォームデータを管理し、非制御コンポーネントはDOM自身がデータを管理します。制御コンポーネントはリアルタイムバリデーションや値の変換が容易ですが、コード量が増えます。

  3. Q3. React Hook Form を使用する主なメリットは何ですか？

    回答を表示

    回答: 非制御コンポーネントベースで再レンダリングを最小化し、高パフォーマンスです。また、バリデーションの統合が容易で、TypeScript との相性も良く、コード量を削減できます。

  4. Q4. Zod を React Hook Form と組み合わせる利点は何ですか？

    回答を表示

    回答: スキーマから型が自動推論され、バリデーションロジックと型定義が一元化されます。複雑な条件付きバリデーションも簡潔に記述でき、型安全性が大幅に向上します。
