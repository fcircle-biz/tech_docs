2.1 JSX/TSX構文の基礎

JSX（JavaScript XML）は、JavaScriptの構文拡張であり、UIの構造を宣言的に記述できます。TypeScriptを使用する場合はTSX（TypeScript XML）として扱われ、型の恩恵を受けながら開発できます。

JSXの基本ルール

JSXはJavaScriptではない

JSXはHTML風の記法ですが、実際にはJavaScriptにトランスパイルされます。例えば<div>Hello</div>は、React.createElement('div', null, 'Hello')に変換されます。

JSX基本例 
// 単一のルート要素が必要
function Greeting() {
  return (
    <div>
      <h1>こんにちは</h1>
      <p>Reactへようこそ</p>
    </div>
  );
}

// フラグメントを使用（余分なdivを避ける）
function GreetingFragment() {
  return (
    <>
      <h1>こんにちは</h1>
      <p>Reactへようこそ</p>
    </>
  );
}

JSX内でのJavaScript式

JSX内での式の埋め込み 
function UserInfo() {
  const user = { name: '太郎', age: 25 };
  const isAdult = user.age >= 20;

  return (
    <div>
      {/* 変数の埋め込み */}
      <h2>{user.name}さん</h2>

      {/* 式の評価 */}
      <p>年齢: {user.age}歳</p>

      {/* 条件付きレンダリング（三項演算子） */}
      <p>{isAdult ? '成人' : '未成年'}</p>

      {/* 条件付きレンダリング（論理AND） */}
      {isAdult && <button>成人向けコンテンツ</button>}
    </div>
  );
}

JSXのよくある間違い

1. classの代わりにclassNameを使う
2. forの代わりにhtmlForを使う
3. style属性はオブジェクトで指定
4. コメントは{/* */}形式

2.2 TypeScriptでのコンポーネント型定義

TypeScriptを使うことで、Propsの型を明示的に定義し、開発時のエラーを早期に発見できます。型安全性はReactアプリケーションの品質向上に大きく貢献します。

基本的なProps型定義

Props型定義の基本 
// interface を使用した型定義
interface UserCardProps {
  name: string;
  age: number;
  email: string;
  isActive?: boolean; // オプショナル
}

function UserCard({ name, age, email, isActive = true }: UserCardProps) {
  return (
    <div className="user-card">
      <h3>{name}</h3>
      <p>年齢: {age}</p>
      <p>メール: {email}</p>
      {isActive && <span className="badge">アクティブ</span>}
    </div>
  );
}

// 使用例
<UserCard name="太郎" age={25} email="taro@example.com" />

interfaceとtypeの使い分け

interface

拡張性に優れ、同名のinterfaceは自動的にマージされます。 

interface User {
  name: string;
}

interface User {
  age: number; // マージされる
}

type

ユニオン型、交差型など柔軟な型表現が可能。 

type Status = 'active' | 'inactive';

type UserWithStatus = User & {
  status: Status;
};

推奨ガイドライン

コンポーネントのPropsにはinterfaceを使用し、ユニオン型や複雑な型にはtypeを使用するのが一般的です。

2.3 FC型とchildren propの型定義

React.FC（FunctionComponent）型を使うことで、コンポーネントの型を明示的に定義できます。また、children propの型定義方法も理解しておきましょう。

FC型の使い方

FC型を使ったコンポーネント定義 
import { FC } from 'react';

// FC型を使用した定義
interface GreetingProps {
  name: string;
  greeting?: string;
}

const Greeting: FC<GreetingProps> = ({ name, greeting = 'こんにちは' }) => {
  return (
    <div>
      <h2>{greeting}、{name}さん！</h2>
    </div>
  );
};

// FC型を使わない定義（推奨）
function GreetingSimple({ name, greeting = 'こんにちは' }: GreetingProps) {
  return (
    <div>
      <h2>{greeting}、{name}さん！</h2>
    </div>
  );
}

FC型の注意点

React 18以降、FC型はchildren propを暗黙的に含まなくなりました。明示的にchildren propを定義する必要があります。また、最近のベストプラクティスでは、FC型を使わずに通常の関数として定義することが推奨されています。

children propの型定義

children propの様々な型定義 
import { ReactNode, ReactElement } from 'react';

// ReactNode型：最も柔軟（文字列、数値、要素など）
interface ContainerProps {
  children: ReactNode;
}

function Container({ children }: ContainerProps) {
  return <div className="container">{children}</div>;
}

// ReactElement型：React要素のみ
interface CardProps {
  children: ReactElement;
}

// 特定のコンポーネントのみ許可
interface ListProps {
  children: ReactElement<ListItemProps> | ReactElement<ListItemProps>[];
}

// 関数型children（Render Props）
interface RenderProps {
  children: (data: User) => ReactNode;
}

function DataProvider({ children }: RenderProps) {
  const user = { name: '太郎', age: 25 };
  return <div>{children(user)}</div>;
}

2.4 ジェネリクスを使った汎用コンポーネント

TypeScriptのジェネリクスを活用することで、様々な型に対応できる汎用的なコンポーネントを作成できます。再利用性の高いコンポーネント設計に役立ちます。

ジェネリックリストコンポーネント

汎用的なリストコンポーネント 
// ジェネリック型パラメータTを使用
interface ListProps<T> {
  items: T[];
  renderItem: (item: T) => ReactNode;
  keyExtractor: (item: T) => string | number;
}

function List<T>({ items, renderItem, keyExtractor }: ListProps<T>) {
  return (
    <ul>
      {items.map((item) => (
        <li key={keyExtractor(item)}>{renderItem(item)}</li>
      ))}
    </ul>
  );
}

// 使用例1: ユーザーリスト
interface User {
  id: number;
  name: string;
}

const users: User[] = [
  { id: 1, name: '太郎' },
  { id: 2, name: '花子' }
];

<List
  items={users}
  renderItem={(user) => <span>{user.name}</span>}
  keyExtractor={(user) => user.id}
/>

// 使用例2: 商品リスト
interface Product {
  id: string;
  title: string;
  price: number;
}

const products: Product[] = [
  { id: 'p1', title: 'ノートPC', price: 120000 }
];

<List
  items={products}
  renderItem={(product) => (
    <div>
      <h3>{product.title}</h3>
      <p>¥{product.price.toLocaleString()}</p>
    </div>
  )}
  keyExtractor={(product) => product.id}
/>

ジェネリック型制約

型制約を使った安全な設計 
// idプロパティを持つ型に制約
interface WithId {
  id: string | number;
}

interface SelectProps<T extends WithId> {
  items: T[];
  value: T['id'];
  onChange: (id: T['id']) => void;
  labelKey: keyof T;
}

function Select<T extends WithId>({
  items,
  value,
  onChange,
  labelKey
}: SelectProps<T>) {
  return (
    <select value={value} onChange={(e) => onChange(e.target.value)}>
      {items.map((item) => (
        <option key={item.id} value={item.id}>
          {String(item[labelKey])}
        </option>
      ))}
    </select>
  );
}

2.5 型安全なprops設計のベストプラクティス

型安全なprops設計により、開発時のエラーを削減し、コンポーネントの使いやすさを向上させることができます。

識別可能なユニオン型

条件付きpropsの型定義 
// 識別プロパティを使った型安全な設計
type ButtonProps =
  | {
      variant: 'link';
      href: string;
      onClick?: never; // リンクの場合はonClickは不要
    }
  | {
      variant: 'button';
      onClick: () => void;
      href?: never; // ボタンの場合はhrefは不要
    };

function Button(props: ButtonProps) {
  if (props.variant === 'link') {
    // この分岐内では props.href が使用可能
    return <a href={props.href}>リンク</a>;
  }

  // この分岐内では props.onClick が使用可能
  return <button onClick={props.onClick}>ボタン</button>;
}

// 使用例
<Button variant="link" href="/about" />
<Button variant="button" onClick={() => alert('clicked')} />

デフォルト値の型安全な設定

デフォルト値パターン 
interface AlertProps {
  message: string;
  type?: 'info' | 'warning' | 'error' | 'success';
  dismissible?: boolean;
}

// パターン1: 分割代入でデフォルト値設定
function Alert({
  message,
  type = 'info',
  dismissible = false
}: AlertProps) {
  return (
    <div className={`alert alert-${type}`}>
      {message}
      {dismissible && <button>×</button>}
    </div>
  );
}

// パターン2: デフォルトPropsを別に定義
const defaultProps = {
  type: 'info' as const,
  dismissible: false
};

type AlertPropsWithDefaults = AlertProps & typeof defaultProps;

function AlertWithDefaults(props: AlertProps) {
  const mergedProps = { ...defaultProps, ...props };
  // mergedProps は AlertPropsWithDefaults 型
  return <div>{mergedProps.message}</div>;
}

2.6 React.ReactElement vs JSX.Element

Reactの型システムには、似ているが異なる複数の型が存在します。それぞれの違いと使い分けを理解しましょう。

flowchart TD ReactNode["ReactNode
最も広い型"] ReactElement["ReactElement
React要素"] JSXElement["JSX.Element
JSX構文の結果"] ReactChild["ReactChild
単一の子要素"] ReactNode -->|含む| ReactElement ReactNode -->|含む| ReactChild ReactNode -->|含む| String["string/number/boolean/null"] ReactElement -->|特殊化| JSXElement

各型の詳細

ReactNode

レンダリング可能なすべての値を含む最も広い型。文字列、数値、React要素、配列、null、undefinedなど。childrenプロパティに推奨。

ReactElement

React.createElement()やJSXの結果として生成される要素オブジェクト。ジェネリック型パラメータで特定のコンポーネント型を指定可能。

JSX.Element

ReactElement<any, any>のエイリアス。JSX構文の結果の型として使用されます。

型の使い分け例 
import { ReactNode, ReactElement } from 'react';

// ReactNode: 柔軟なchildren
interface FlexibleProps {
  children: ReactNode; // 文字列、数値、要素すべてOK
}

// ReactElement: React要素のみ
interface StrictProps {
  children: ReactElement; // React要素のみ
}

// JSX.Element: コンポーネントの戻り値
function MyComponent(): JSX.Element {
  return <div>Hello</div>;
}

// 特定のコンポーネント型を制約
interface SpecificProps {
  icon: ReactElement<typeof IconComponent>;
}

実習 2-1: 型安全なカードコンポーネント作成

TypeScriptの型定義を活用して、再利用可能なカードコンポーネントを作成しましょう。

要件

  • title（必須）、description（オプション）を受け取る
  • children propでカスタムコンテンツを表示可能
  • variant（'default' | 'primary' | 'success'）でスタイル変更

実装コード

import { ReactNode } from 'react';

interface CardProps {
  title: string;
  description?: string;
  variant?: 'default' | 'primary' | 'success';
  children?: ReactNode;
}

function Card({
  title,
  description,
  variant = 'default',
  children
}: CardProps) {
  // バリアントに応じたクラス名を決定
  const variantClasses = {
    default: 'border-slate-300 bg-white',
    primary: 'border-blue-500 bg-blue-50',
    success: 'border-emerald-500 bg-emerald-50'
  };

  return (
    <div className={`border rounded-lg p-4 ${variantClasses[variant]}`}>
      <h3 className="text-lg font-semibold mb-2">{title}</h3>
      {description && <p className="text-slate-600 mb-3">{description}</p>}
      {children && <div className="mt-4">{children}</div>}
    </div>
  );
}

// 使用例
function App() {
  return (
    <div>
      <Card
        title="デフォルトカード"
        description="これは説明文です"
      />

      <Card
        title="プライマリカード"
        variant="primary"
      >
        <button>アクション</button>
      </Card>

      <Card
        title="成功カード"
        variant="success"
        description="操作が成功しました"
      />
    </div>
  );
}

export default App;

理解度確認クイズ

  1. Q1. JSXで複数の要素を返す際、必ず必要なものは何ですか？

    回答を表示

    回答: 単一のルート要素が必要です。<div>で囲むか、<></>（フラグメント）を使用します。

  2. Q2. interfaceとtypeの最も大きな違いは何ですか？

    回答を表示

    回答: interfaceは同名のものが自動的にマージされ拡張性に優れます。typeはユニオン型や交差型など柔軟な型表現が可能です。

  3. Q3. children propの型として最も柔軟なものはどれですか？

    回答を表示

    回答: ReactNode型が最も柔軟です。文字列、数値、React要素、配列などすべてのレンダリング可能な値を含みます。

  4. Q4. ジェネリクスを使うメリットは何ですか？

    回答を表示

    回答: 型パラメータを使って、様々な型に対応できる汎用的なコンポーネントを作成できます。コードの再利用性と型安全性を両立できます。
