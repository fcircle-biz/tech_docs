9.1 パフォーマンス最適化の重要性

Reactアプリケーションのパフォーマンスは、ユーザー体験に直結します。不要な再レンダリングの削減、効率的なコード分割、最適化された依存関係の管理により、アプリケーションの応答性を向上させることができます。

パフォーマンス問題の主な原因

flowchart TD A["パフォーマンス低下"] --> B["不要な再レンダリング"] A --> C["重い計算処理"] A --> D["大きなバンドルサイズ"] A --> E["長いリスト表示"] B --> B1["propsの変更検出ミス"] B --> B2["状態管理の問題"] C --> C1["メモ化の不足"] C --> C2["同期的な重い処理"] D --> D1["コード分割の不足"] D --> D2["不要な依存関係"] E --> E1["仮想化の不使用"] E --> E2["DOM要素が多すぎる"]

最適化戦略の全体像

  • メモ化: React.memo、useMemo、useCallbackによる再計算の削減
  • コード分割: 必要なコードだけを読み込むlazy loading
  • 仮想化: 大量のリストを効率的に表示するreact-window
  • バンドル最適化: 不要な依存関係の削減とTree Shaking

早すぎる最適化に注意

「早すぎる最適化は諸悪の根源」という言葉があります。まずは計測してボトルネックを特定してから最適化を行いましょう。React DevToolsプロファイラーを使用して、実際に問題となっている箇所を見つけることが重要です。

9.2 React.memo - コンポーネントのメモ化

React.memoは、コンポーネントのpropsが変更されていない場合、再レンダリングをスキップする高階コンポーネントです。パフォーマンス最適化の基本的な手法です。

React.memoの基本

React.memoの使用例 
// 最適化前：親の再レンダリングで毎回再レンダリングされる
function ExpensiveComponent({ value }: { value: number }) {
  console.log('ExpensiveComponent rendered');

  // 重い計算処理
  const result = heavyCalculation(value);

  return <div>Result: {result}</div>;
}

// 最適化後：propsが同じなら再レンダリングをスキップ
const MemoizedComponent = React.memo(function ExpensiveComponent({
  value
}: {
  value: number
}) {
  console.log('MemoizedComponent rendered');
  const result = heavyCalculation(value);
  return <div>Result: {result}</div>;
});

カスタム比較関数

デフォルトでは浅い比較が行われますが、カスタム比較関数を提供することで、より細かい制御が可能です。

カスタム比較関数の例 
interface UserCardProps {
  user: {
    id: string;
    name: string;
    age: number;
  };
}

// カスタム比較：IDが同じなら再レンダリングしない
const UserCard = React.memo(
  ({ user }: UserCardProps) => {
    return (
      <div>
        <h3>{user.name}</h3>
        <p>Age: {user.age}</p>
      </div>
    );
  },
  (prevProps, nextProps) => {
    // trueを返すと再レンダリングをスキップ
    return prevProps.user.id === nextProps.user.id;
  }
);

React.memoの注意点

関数やオブジェクトをpropsとして渡す場合、親コンポーネントが再レンダリングされるたびに新しい参照が作成されるため、React.memoの効果がなくなります。useCallbackやuseMemoと組み合わせて使用しましょう。

9.3 useMemo - 計算結果のメモ化

useMemoは、重い計算の結果をメモ化し、依存配列の値が変わらない限り再計算をスキップするフックです。

useMemoの基本

useMemoの使用例 
function ProductList({ products, searchTerm }: Props) {
  // 最適化前：毎回フィルタリングが実行される
  // const filtered = products.filter(p =>
  //   p.name.includes(searchTerm)
  // );

  // 最適化後：productsまたはsearchTermが変わった時だけ実行
  const filteredProducts = useMemo(() => {
    console.log('Filtering products...');
    return products.filter(product =>
      product.name.toLowerCase().includes(searchTerm.toLowerCase())
    );
  }, [products, searchTerm]);

  return (
    <ul>
      {filteredProducts.map(product => (
        <li key={product.id}>{product.name}</li>
      ))}
    </ul>
  );
}

useMemoの適切な使用タイミング

使うべき場面

  • • 計算コストが高い処理
  • • 大量のデータのフィルタリング/ソート
  • • 子コンポーネントへ渡すオブジェクト
  • • 依存配列の要素が頻繁に変わらない

不要な場面

  • • 単純な計算（足し算など）
  • • プリミティブ値の比較
  • • 依存配列が頻繁に変わる
  • • メモ化のコストが高い場合
useMemoの実践例 
function DataAnalysis({ data }: { data: number[] }) {
  // 統計情報の計算をメモ化
  const stats = useMemo(() => {
    const sum = data.reduce((a, b) => a + b, 0);
    const avg = sum / data.length;
    const sorted = [...data].sort((a, b) => a - b);
    const median = sorted[Math.floor(sorted.length / 2)];

    return { sum, avg, median };
  }, [data]);

  return (
    <div>
      <p>合計: {stats.sum}</p>
      <p>平均: {stats.avg.toFixed(2)}</p>
      <p>中央値: {stats.median}</p>
    </div>
  );
}

9.4 useCallback - 関数のメモ化

useCallbackは、関数をメモ化し、依存配列の値が変わらない限り同じ関数インスタンスを返すフックです。React.memoと組み合わせて使用することが多いです。

useCallbackの基本

useCallbackの使用例 
// 子コンポーネント（React.memoでメモ化）
const TodoItem = React.memo(({
  todo,
  onToggle
}: {
  todo: Todo;
  onToggle: (id: string) => void;
}) => {
  console.log('TodoItem rendered:', todo.id);
  return (
    <li>
      <input
        type="checkbox"
        checked={todo.completed}
        onChange={() => onToggle(todo.id)}
      />
      {todo.text}
    </li>
  );
});

// 親コンポーネント
function TodoList() {
  const [todos, setTodos] = useState<Todo[]>([]);

  // 最適化前：毎回新しい関数が作成される
  // const handleToggle = (id: string) => { ... };

  // 最適化後：関数インスタンスが再利用される
  const handleToggle = useCallback((id: string) => {
    setTodos(prev => prev.map(todo =>
      todo.id === id ? { ...todo, completed: !todo.completed } : todo
    ));
  }, []); // todosを依存配列に入れない（関数型更新を使用）

  return (
    <ul>
      {todos.map(todo => (
        <TodoItem key={todo.id} todo={todo} onToggle={handleToggle} />
      ))}
    </ul>
  );
}

同じ関数参照を返す"] C --> C1["計算結果をメモ化
値を返す"] B1 --> B2["例: イベントハンドラー
コールバック関数"] C1 --> C2["例: フィルタリング結果
計算値"]
useCallbackとuseMemoの比較 
function Example({ items }: { items: Item[] }) {
  // useCallback：関数をメモ化
  const handleClick = useCallback((id: string) => {
    console.log('Clicked:', id);
  }, []);

  // useMemo：計算結果をメモ化
  const expensiveValue = useMemo(() => {
    return items.reduce((sum, item) => sum + item.price, 0);
  }, [items]);

  // 実はこれと同じ
  // const handleClick = useMemo(() => {
  //   return (id: string) => console.log('Clicked:', id);
  // }, []);

  return <div>Total: {expensiveValue}</div>;
}

9.5 React DevToolsプロファイラー

React DevToolsのプロファイラーは、コンポーネントのレンダリング性能を可視化し、ボトルネックを特定するための強力なツールです。

プロファイラーの使い方

1 記録の開始

React DevToolsを開き、Profilerタブで青い録画ボタンをクリックします。

2 操作の実行

パフォーマンスを測定したい操作（クリック、入力など）を実行します。

3 記録の停止と分析

停止ボタンをクリックし、フレームグラフで各コンポーネントのレンダリング時間を確認します。

プロファイラーで確認すべき項目

  • レンダリング時間: 各コンポーネントの処理時間（黄色が濃いほど遅い）
  • 再レンダリング回数: 不要な再レンダリングが発生していないか
  • 再レンダリングの理由: propsやstateのどの変更が原因か
  • コミット時間: 実際のDOM更新にかかった時間

プロファイラーのヒント

プロダクションビルドでプロファイリングを行う場合は、react-dom/profilingを使用してビルドしましょう。開発モードよりも正確な測定が可能です。

9.7 コード分割とlazy loading

コード分割は、アプリケーションのコードを複数のバンドルに分割し、必要な時だけ読み込む技術です。初回読み込み時間を大幅に短縮できます。

React.lazyとSuspenseの基本

基本的なコード分割 
import { lazy, Suspense } from 'react';

// 最適化前：初回読み込み時にすべてのコンポーネントを読み込む
// import Dashboard from './Dashboard';
// import Settings from './Settings';

// 最適化後：必要な時だけ読み込む
const Dashboard = lazy(() => import('./Dashboard'));
const Settings = lazy(() => import('./Settings'));

function App() {
  return (
    <div>
      <Suspense fallback={<div>Loading...</div>}>
        <Dashboard />
      </Suspense>

      <Suspense fallback={<div>Loading Settings...</div>}>
        <Settings />
      </Suspense>
    </div>
  );
}

ルートベースのコード分割

最も効果的なコード分割ポイントは、ルート（ページ）レベルです。React Routerと組み合わせて使用します。

React Routerとの組み合わせ 
import { lazy, Suspense } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

// 各ページを遅延読み込み
const HomePage = lazy(() => import('./pages/HomePage'));
const AboutPage = lazy(() => import('./pages/AboutPage'));
const ProductsPage = lazy(() => import('./pages/ProductsPage'));

// ローディングコンポーネント
function LoadingSpinner() {
  return (
    <div className="flex justify-center items-center h-screen">
      <div className="animate-spin rounded-full h-12 w-12 border-b-2"></div>
    </div>
  );
}

function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<LoadingSpinner />}>
        <Routes>
          <Route path="/" element={<HomePage />} />
          <Route path="/about" element={<AboutPage />} />
          <Route path="/products" element={<ProductsPage />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}

マウスホバーでプリロード

const HeavyComponent = lazy(() => import('./HeavyComponent'));

9.8 Suspenseの活用

Suspenseは、コンポーネントの読み込み待ちを宣言的に処理する機能です。React 18では、データフェッチングにも対応しています。

Suspenseの階層化

細かい粒度のSuspense 
import { lazy, Suspense } from 'react';

const Header = lazy(() => import('./Header'));
const Sidebar = lazy(() => import('./Sidebar'));
const MainContent = lazy(() => import('./MainContent'));

function App() {
  return (
    <div>
      {/* ヘッダーは常に表示 */}
      <Suspense fallback={<div className="h-16 bg-gray-200" />}>
        <Header />
      </Suspense>

      <div className="flex">
        {/* サイドバーとメインコンテンツは独立して読み込み */}
        <Suspense fallback={<div className="w-64 bg-gray-100" />}>
          <Sidebar />
        </Suspense>

        <Suspense fallback={<div className="flex-1 p-4">Loading...</div>}>
          <MainContent />
        </Suspense>
      </div>
    </div>
  );
}

ErrorBoundaryとの組み合わせ

エラーハンドリング 
import { Component, lazy, Suspense } from 'react';

const LazyComponent = lazy(() => import('./LazyComponent'));

// エラーバウンダリ
class ErrorBoundary extends Component<
  { children: React.ReactNode },
  { hasError: boolean }
> {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return <div>読み込みエラーが発生しました</div>;
    }
    return this.props.children;
  }
}

// 使用例
function App() {
  return (
    <ErrorBoundary>
      <Suspense fallback={<div>Loading...</div>}>
        <LazyComponent />
      </Suspense>
    </ErrorBoundary>
  );
}

9.10 Web Vitalsの改善

Web Vitalsは、Googleが提唱するWebサイトのユーザー体験を測定する指標です。ReactアプリケーションでもこれらのメトリクスをImprovementすることが重要です。

主要なCore Web Vitals

LCP

Largest Contentful Paint
最大コンテンツの描画時間

目標: 2.5秒以内

FID

First Input Delay
初回入力遅延

目標: 100ms以内

CLS

Cumulative Layout Shift
累積レイアウトシフト

目標: 0.1以内

実習 9-1: パフォーマンス最適化の実践

React.memo、useMemo、useCallbackを組み合わせて、大量のデータを扱うリストコンポーネントを最適化しましょう。

実装コード

import { useState, useMemo, useCallback, memo } from 'react';

interface Item {
  id: number;
  name: string;
  price: number;
}

// メモ化された子コンポーネント
const ItemRow = memo(({
  item,
  onRemove
}: {
  item: Item;
  onRemove: (id: number) => void;
}) => {
  console.log('ItemRow rendered:', item.id);
  return (
    <li className="flex justify-between items-center p-2 border-b">
      <span>{item.name} - ¥{item.price}</span>
      <button onClick={() => onRemove(item.id)}>削除</button>
    </li>
  );
});

function OptimizedItemList() {
  const [items, setItems] = useState<Item[]>(() =>
    Array.from({ length: 1000 }, (_, i) => ({
      id: i,
      name: `商品${i}`,
      price: Math.floor(Math.random() * 10000),
    }))
  );
  const [filter, setFilter] = useState('');

  // フィルタリングをメモ化
  const filteredItems = useMemo(() => {
    console.log('Filtering items...');
    return items.filter(item =>
      item.name.toLowerCase().includes(filter.toLowerCase())
    );
  }, [items, filter]);

  // 合計金額を計算（メモ化）
  const total = useMemo(() => {
    return filteredItems.reduce((sum, item) => sum + item.price, 0);
  }, [filteredItems]);

  // 削除ハンドラーをメモ化
  const handleRemove = useCallback((id: number) => {
    setItems(prev => prev.filter(item => item.id !== id));
  }, []);

  return (
    <div>
      <input
        type="text"
        placeholder="商品を検索..."
        value={filter}
        onChange={(e) => setFilter(e.target.value)}
        className="border p-2 mb-4 w-full"
      />
      <p>合計: ¥{total.toLocaleString()}</p>
      <ul>
        {filteredItems.map(item => (
          <ItemRow key={item.id} item={item} onRemove={handleRemove} />
        ))}
      </ul>
    </div>
  );
}

確認ポイント

  • • React DevToolsでレンダリング回数を確認
  • • フィルター入力時に全アイテムが再レンダリングされないこと
  • • 削除ボタンクリック時に該当アイテムのみ更新されること

理解度確認クイズ

  1. Q1. React.memoが効果を発揮しない状況はどれですか？

    回答を表示

    回答: propsとして渡す関数やオブジェクトが毎回新しく生成される場合、参照が変わるためReact.memoは再レンダリングを防げません。useCallbackやuseMemoと組み合わせて使用する必要があります。

  2. Q2. useMemoとuseCallbackの違いは何ですか？

    回答を表示

    回答: useMemoは計算結果（値）をメモ化し、useCallbackは関数自体をメモ化します。useCallback(fn, deps)はuseMemo(() => fn, deps)と等価です。

  3. Q3. コード分割が最も効果的なポイントはどこですか？

    回答を表示

    回答: ルート（ページ）レベルでのコード分割が最も効果的です。各ページを訪れた時だけそのページのコードを読み込むことで、初回読み込み時間を大幅に短縮できます。

  4. Q4. 仮想化（react-window）が必要になるのはどのような場合ですか？

    回答を表示

    回答: 数百から数千以上の要素を持つリストを表示する場合に有効です。仮想化により、画面に表示されている要素だけをDOMにレンダリングし、メモリ使用量とレンダリング時間を削減できます。

  • Web Vitals（LCP、FID、CLS）を測定し、ユーザー体験を継続的に改善
