カスタムフック - ロジックの再利用

カスタムフックの設計原則、命名規則、実装パターン、実用的なカスタムフックの作成、テスト方法、パッケージ化と共有について実践的に学びます。

  • useDebounce、useThrottleを実装できる
  • useFetchでデータ取得を抽象化できる
  • useLocalStorageで永続化ロジックを再利用できる
  • useMediaQueryでレスポンシブ対応ができる

  • フックをパッケージ化して共有できる

8.1 カスタムフックとは - 設計原則と命名規則

カスタムフックは、複数のReact Hooksを組み合わせて再利用可能なロジックとして抽出したものです。コンポーネント間でロジックを共有し、コードの重複を減らすことができます。

カスタムフックの設計原則

単一責任の原則

1つのカスタムフックは1つの明確な目的を持つべきです。複数の機能を詰め込むと、再利用性が下がります。

命名規則（必須）

カスタムフックの名前は必ずuseで始める必要があります。これはReactがHooksのルールを適用するために必要です。

カスタムフックの基本構造

useCounter.ts（基本的なカスタムフック） 
import { useState } from 'react';

// カスタムフック: カウンター機能を提供
function useCounter(initialValue: number = 0) {
  const [count, setCount] = useState(initialValue);

  // カウントを増やす関数
  const increment = () => setCount(c => c + 1);

  // カウントを減らす関数
  const decrement = () => setCount(c => c - 1);

  // カウントをリセットする関数
  const reset = () => setCount(initialValue);

  // 使用側に必要な値と関数を返す
  return { count, increment, decrement, reset };
}

// 使用例
function CounterComponent() {
  const { count, increment, decrement, reset } = useCounter(10);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
}

カスタムフックの利点

  • ロジックの再利用: 同じ機能を複数のコンポーネントで使用可能
  • テストの容易さ: ロジックを独立してテストできる
  • 関心の分離: UIとロジックを分離できる
  • コードの整理: コンポーネントがシンプルになる

8.2 useDebounce - 入力遅延処理

useDebounceは、ユーザーの入力が一定時間止まるまで処理を遅延させるカスタムフックです。検索フィールドなどで、入力のたびにAPIを呼び出さないようにする際に便利です。

500ms待機 U->>D: ab Note over D: タイマーリセット
500ms待機 U->>D: abc Note over D: タイマーリセット
500ms待機 Note over U: 入力停止 Note over D: 500ms経過 D->>A: abc で検索
useDebounce.ts 
import { useState, useEffect } from 'react';

// Debounceカスタムフック
function useDebounce<T>(value: T, delay: number = 500): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value);

  useEffect(() => {
    // 遅延タイマーを設定
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    // クリーンアップ関数でタイマーをキャンセル
    return () => {
      clearTimeout(timer);
    };
  }, [value, delay]);

  return debouncedValue;
}

// 使用例: 検索フィールド
function SearchComponent() {
  const [searchTerm, setSearchTerm] = useState('');
  const debouncedSearch = useDebounce(searchTerm, 500);

  useEffect(() => {
    if (debouncedSearch) {
      // 入力が500ms止まったらAPI呼び出し
      console.log('API呼び出し:', debouncedSearch);
    }
  }, [debouncedSearch]);

  return (
    <input
      type="text"
      value={searchTerm}
      onChange={(e) => setSearchTerm(e.target.value)}
      placeholder="検索..."
    />
  );
}

8.3 useThrottle - 関数実行頻度制御

useThrottleは、指定した時間内に最大1回だけ処理を実行するカスタムフックです。スクロールイベントやウィンドウリサイズイベントなど、高頻度で発生するイベント処理に使用します。

DebounceとThrottleの違い

Debounce

最後のイベントから一定時間後に実行

用途: 検索入力、フォーム入力の確定

Throttle

一定時間ごとに定期的に実行

用途: スクロール、ウィンドウリサイズ

useThrottle.ts 
import { useState, useEffect, useRef } from 'react';

// Throttleカスタムフック
function useThrottle<T>(value: T, limit: number = 500): T {
  const [throttledValue, setThrottledValue] = useState<T>(value);
  const lastRan = useRef(Date.now());

  useEffect(() => {
    const handler = setTimeout(() => {
      if (Date.now() - lastRan.current >= limit) {
        setThrottledValue(value);
        lastRan.current = Date.now();
      }
    }, limit - (Date.now() - lastRan.current));

    return () => {
      clearTimeout(handler);
    };
  }, [value, limit]);

  return throttledValue;
}

// 使用例: スクロール位置の監視
function ScrollComponent() {
  const [scrollY, setScrollY] = useState(0);
  const throttledScrollY = useThrottle(scrollY, 200);

  useEffect(() => {
    const handleScroll = () => {
      setScrollY(window.scrollY);
    };

    window.addEventListener('scroll', handleScroll);
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);

  return <div>スクロール位置: {throttledScrollY}px</div>;
}

8.4 useFetch - データ取得カスタムフック

useFetchは、APIからのデータ取得ロジックを抽象化するカスタムフックです。ローディング状態、エラーハンドリング、データの管理を一元化します。

useFetch.ts 
import { useState, useEffect } from 'react';

interface FetchState<T> {
  data: T | null;
  loading: boolean;
  error: Error | null;
}

// データ取得カスタムフック
function useFetch<T>(url: string): FetchState<T> {
  const [state, setState] = useState<FetchState<T>>({
    data: null,
    loading: true,
    error: null,
  });

  useEffect(() => {
    // AbortControllerで競合状態を防ぐ
    const controller = new AbortController();

    const fetchData = async () => {
      setState({ data: null, loading: true, error: null });

      try {
        const response = await fetch(url, {
          signal: controller.signal,
        });

        if (!response.ok) {
          throw new Error(`HTTP error: ${response.status}`);
        }

        const data = await response.json();
        setState({ data, loading: false, error: null });
      } catch (error) {
        // AbortErrorは無視
        if (error instanceof Error && error.name !== 'AbortError') {
          setState({ data: null, loading: false, error });
        }
      }
    };

    fetchData();

    // クリーンアップでリクエストをキャンセル
    return () => {
      controller.abort();
    };
  }, [url]);

  return state;
}
useFetch使用例 
interface User {
  id: number;
  name: string;
  email: string;
}

function UserList() {
  const { data, loading, error } = useFetch<User[]>(
    'https://api.example.com/users'
  );

  if (loading) return <div>読み込み中...</div>;
  if (error) return <div>エラー: {error.message}</div>;
  if (!data) return null;

  return (
    <ul>
      {data.map(user => (
        <li key={user.id}>{user.name} - {user.email}</li>
      ))}
    </ul>
  );
}

実務での注意

プロダクションでは、TanStack Query（React Query）などの専用ライブラリを使用することをお勧めします。キャッシング、再検証、楽観的更新などの高度な機能が必要になるためです。

8.5 useLocalStorage - ローカルストレージ連携

useLocalStorageは、ブラウザのローカルストレージと同期する状態を提供するカスタムフックです。ユーザー設定やテーマ、一時的なデータの永続化に便利です。

useLocalStorage.ts 
import { useState, useEffect } from 'react';

// ローカルストレージ連携カスタムフック
function useLocalStorage<T>(
  key: string,
  initialValue: T
): [T, (value: T | ((val: T) => T)) => void] {
  // 初期値を取得（ローカルストレージから）
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error('Error loading from localStorage:', error);
      return initialValue;
    }
  });

  // 値を更新する関数
  const setValue = (value: T | ((val: T) => T)) => {
    try {
      // 関数の場合は実行して値を取得
      const valueToStore =
        value instanceof Function ? value(storedValue) : value;

      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error('Error saving to localStorage:', error);
    }
  };

  return [storedValue, setValue];
}

// 使用例: テーマの保存
function ThemeToggle() {
  const [theme, setTheme] = useLocalStorage<'light' | 'dark'>(
    'theme',
    'light'
  );

  const toggleTheme = () => {
    setTheme(current => (current === 'light' ? 'dark' : 'light'));
  };

  return (
    <button onClick={toggleTheme}>
      現在のテーマ: {theme}
    </button>
  );
}

8.6 useMediaQuery - レスポンシブ対応

useMediaQueryは、CSSメディアクエリをJavaScriptで監視するカスタムフックです。画面サイズに応じたUIの切り替えや、ダークモードの検出に使用します。

useMediaQuery.ts 
import { useState, useEffect } from 'react';

// メディアクエリ監視カスタムフック
function useMediaQuery(query: string): boolean {
  const [matches, setMatches] = useState(() => {
    // SSR対応: windowが存在しない場合はfalse
    if (typeof window === 'undefined') return false;
    return window.matchMedia(query).matches;
  });

  useEffect(() => {
    const mediaQuery = window.matchMedia(query);

    // 現在の値を設定
    setMatches(mediaQuery.matches);

    // 変更を監視
    const handler = (event: MediaQueryListEvent) => {
      setMatches(event.matches);
    };

    // イベントリスナーを追加
    mediaQuery.addEventListener('change', handler);

    // クリーンアップ
    return () => {
      mediaQuery.removeEventListener('change', handler);
    };
  }, [query]);

  return matches;
}
useMediaQuery使用例 
function ResponsiveComponent() {
  const isMobile = useMediaQuery('(max-width: 768px)');
  const isTablet = useMediaQuery('(min-width: 769px) and (max-width: 1024px)');
  const isDesktop = useMediaQuery('(min-width: 1025px)');
  const isDarkMode = useMediaQuery('(prefers-color-scheme: dark)');

  return (
    <div>
      <h2>デバイス情報</h2>
      <p>モバイル: {isMobile ? 'はい' : 'いいえ'}</p>
      <p>タブレット: {isTablet ? 'はい' : 'いいえ'}</p>
      <p>デスクトップ: {isDesktop ? 'はい' : 'いいえ'}</p>
      <p>ダークモード: {isDarkMode ? '有効' : '無効'}</p>

      {isMobile && <MobileMenu />}
      {!isMobile && <DesktopMenu />}
    </div>
  );
}

8.7 カスタムフックのテスト

カスタムフックは、React Testing LibraryのrenderHookユーティリティを使用してテストします。コンポーネントから独立してロジックをテストできます。

useCounter.test.ts 
import { renderHook, act } from '@testing-library/react';
import { useCounter } from './useCounter';

describe('useCounter', () => {
  test('初期値が正しく設定される', () => {
    const { result } = renderHook(() => useCounter(10));
    expect(result.current.count).toBe(10);
  });

  test('incrementでカウントが増える', () => {
    const { result } = renderHook(() => useCounter(0));

    act(() => {
      result.current.increment();
    });

    expect(result.current.count).toBe(1);
  });

  test('decrementでカウントが減る', () => {
    const { result } = renderHook(() => useCounter(5));

    act(() => {
      result.current.decrement();
    });

    expect(result.current.count).toBe(4);
  });

  test('resetで初期値に戻る', () => {
    const { result } = renderHook(() => useCounter(10));

    act(() => {
      result.current.increment();
      result.current.increment();
    });

    expect(result.current.count).toBe(12);

    act(() => {
      result.current.reset();
    });

    expect(result.current.count).toBe(10);
  });
});

非同期フックのテスト

useFetch.test.ts 
import { renderHook, waitFor } from '@testing-library/react';
import { useFetch } from './useFetch';

// fetchをモック
global.fetch = jest.fn();

describe('useFetch', () => {
  beforeEach(() => {
    jest.clearAllMocks();
  });

  test('データ取得成功時', async () => {
    const mockData = { id: 1, name: 'Test' };

    (global.fetch as jest.Mock).mockResolvedValueOnce({
      ok: true,
      json: async () => mockData,
    });

    const { result } = renderHook(() =>
      useFetch<typeof mockData>('https://api.example.com/data')
    );

    // 初期状態: loading
    expect(result.current.loading).toBe(true);
    expect(result.current.data).toBeNull();

    // データ取得完了を待つ
    await waitFor(() => {
      expect(result.current.loading).toBe(false);
    });

    expect(result.current.data).toEqual(mockData);
    expect(result.current.error).toBeNull();
  });

  test('エラー発生時', async () => {
    (global.fetch as jest.Mock).mockRejectedValueOnce(
      new Error('Network error')
    );

    const { result } = renderHook(() =>
      useFetch('https://api.example.com/data')
    );

    await waitFor(() => {
      expect(result.current.loading).toBe(false);
    });

    expect(result.current.error?.message).toBe('Network error');
    expect(result.current.data).toBeNull();
  });
});

実習 8-1: useToggleカスタムフックの作成

ON/OFF状態を切り替えるuseToggleカスタムフックを実装し、モーダルやドロワーの開閉に使用してみましょう。

要件

  • 初期値を受け取れること
  • toggle、setTrue、setFalse関数を提供すること
  • TypeScriptで型安全に実装すること

実装例

import { useState } from 'react';

function useToggle(initialValue: boolean = false) {
  const [value, setValue] = useState(initialValue);

  const toggle = () => setValue(v => !v);
  const setTrue = () => setValue(true);
  const setFalse = () => setValue(false);

  return { value, toggle, setTrue, setFalse };
}

// 使用例
function ModalExample() {
  const modal = useToggle(false);

  return (
    <>
      <button onClick={modal.setTrue}>モーダルを開く</button>

      {modal.value && (
        <div className="modal">
          <h2>モーダル</h2>
          <button onClick={modal.setFalse}>閉じる</button>
        </div>
      )}
    </>
  );
}

理解度確認クイズ

  1. Q1. カスタムフックの命名規則として正しいものはどれですか？

    回答を表示

    回答: カスタムフックの名前は必ずuseで始める必要があります。例: useDebounce、useFetch。これはReactがHooksのルールを適用するために必須です。

  2. Q2. DebounceとThrottleの違いは何ですか？

    回答を表示

    回答: Debounceは最後のイベントから一定時間後に実行（検索入力などに使用）、Throttleは一定時間ごとに定期的に実行（スクロールイベントなどに使用）します。

  3. Q3. カスタムフックをテストする際に使用するユーティリティは何ですか？

    回答を表示

    回答: React Testing LibraryのrenderHookユーティリティを使用します。コンポーネントから独立してフックのロジックをテストできます。

  4. Q4. useLocalStorageで注意すべき点は何ですか？

    回答を表示

    回答: ローカルストレージは文字列しか保存できないため、JSON.stringifyとJSON.parseでシリアライズ/デシリアライズする必要があります。また、エラーハンドリングも重要です。

  • useLocalStorageでブラウザストレージと状態を同期
