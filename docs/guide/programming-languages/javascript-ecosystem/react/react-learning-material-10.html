10.1 React Routerとは - なぜルーティングが必要か

React RouterはReactアプリケーションにルーティング機能を提供する公式ライブラリです。SPA（Single Page Application）では、ページ遷移時にサーバーへリクエストを送らず、クライアント側でコンテンツを切り替えます。React Routerはこのクライアント側のルーティングを効率的に実装するための強力なツールです。

ルーティングの役割

URLとUIの同期

URLに応じて適切なコンポーネントを表示します。ブラウザの戻る/進むボタンや、ブックマーク、URLの直接入力にも対応できます。

履歴管理

ブラウザの履歴APIと統合し、ユーザーが期待する「戻る」「進む」の動作を実現します。

ネストルート

親子関係を持つルートを定義し、レイアウトやナビゲーションを階層的に構成できます。

アクセス制御

認証状態に応じてページへのアクセスを制限し、未認証ユーザーをログインページへリダイレクトできます。

React Router v6の主な変更点

React Router v6は、v5から大幅に改善され、よりシンプルで直感的なAPIになりました。

  • Routesコンポーネント: Switchの代わりに、より明確なRoutes/Route構造

10.2 React Router v6の基本セットアップ

まず、React Routerのインストールと基本的なセットアップから始めましょう。

インストール

React Routerのインストール

npm install react-router-dom

# yarnの場合

# pnpmの場合
pnpm add react-router-dom

基本的なルーター設定

main.tsx（エントリーポイント） 
import React from 'react';
import ReactDOM from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import App from './App';
import './index.css';

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </React.StrictMode>
);
App.tsx（ルート定義） 
import { Routes, Route, Link } from 'react-router-dom';

// ページコンポーネント
function Home() {
  return <h1>ホームページ</h1>;
}

function About() {
  return <h1>Aboutページ</h1>;
}

function Contact() {
  return <h1>お問い合わせページ</h1>;
}

function App() {
  return (
    <div>
      <nav>
        <Link to="/">ホーム</Link> |
        <Link to="/about">About</Link> |
        <Link to="/contact">お問い合わせ</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/contact" element={<Contact />} />
      </Routes>
    </div>
  );
}

export default App;

主要コンポーネント

  • BrowserRouter: HTML5 History APIを使用するルーター
  • Routes: ルート定義のコンテナ（v5のSwitchの代替）
  • Route: パスとコンポーネントをマッピング
  • Link: ページ遷移を行うリンクコンポーネント

10.3 ルート定義とネスト

React Router v6では、ルートのネスト（入れ子構造）が非常に簡単になりました。レイアウトコンポーネントを使って、共通のヘッダーやサイドバーを持つページ構成を実現できます。

ネストルートの構造

flowchart TB Root["ルート /"] Layout["Layout
共通レイアウト"] Dashboard["Dashboard /dashboard"] Profile["Profile /dashboard/profile"] Settings["Settings /dashboard/settings"] Root --> Layout Layout --> Dashboard Dashboard --> Profile Dashboard --> Settings
ネストルートの実装例 
import { Routes, Route, Outlet, Link } from 'react-router-dom';

// レイアウトコンポーネント
function Layout() {
  return (
    <div>
      <header>
        <h1>アプリケーション</h1>
        <nav>
          <Link to="/">ホーム</Link> |
          <Link to="/dashboard">ダッシュボード</Link>
        </nav>
      </header>
      {/* 子ルートがここに表示される */}
      <Outlet />
      <footer>© 2025 アプリケーション</footer>
    </div>
  );
}

function App() {
  return (
    <Routes>
      <Route path="/" element={<Layout />}>
        <Route index element={<Home />} />
        <Route path="dashboard" element={<Dashboard />}>
          <Route index element={<DashboardHome />} />
          <Route path="profile" element={<Profile />} />
          <Route path="settings" element={<Settings />} />
        </Route>
      </Route>
    </Routes>
  );
}

Outletコンポーネント

Outletは、親ルートのコンポーネント内で子ルートをレンダリングする場所を指定します。レイアウトパターンで非常に重要な役割を果たします。

10.4 動的ルーティングとパラメータ

URLパラメータを使用すると、動的なルート定義が可能になります。ブログの記事詳細ページや、ユーザープロフィールページなど、IDに基づいて異なるコンテンツを表示する際に使用します。

URLパラメータの種類

種類 例 用途
パスパラメータ /users/:id リソースの識別
クエリパラメータ /search?q=react 検索・フィルタリング
オプショナル /posts/:id? 任意パラメータ
パスパラメータの使用 
import { useParams, Link } from 'react-router-dom';

// ユーザー詳細ページ
function UserDetail() {
  const { userId } = useParams<{ userId: string }>();

  return (
    <div>
      <h1>ユーザーID: {userId}</h1>
      <p>ユーザー詳細情報をここに表示...</p>
    </div>
  );
}

function App() {
  return (
    <Routes>
      <Route path="/users/:userId" element={<UserDetail />} />
    </Routes>
  );
}

クエリパラメータの取得

useSearchParamsの使用 
import { useSearchParams } from 'react-router-dom';

function SearchPage() {
  const [searchParams, setSearchParams] = useSearchParams();
  const query = searchParams.get('q') || '';
  const category = searchParams.get('category') || 'all';

  const handleSearch = (newQuery: string) => {
    // クエリパラメータを更新
    setSearchParams({ q: newQuery, category });
  };

  return (
    <div>
      <h1>検索結果: {query}</h1>
      <p>カテゴリ: {category}</p>
    </div>
  );
}

10.5 ナビゲーションとリンク

React Routerは、ページ遷移のために複数の方法を提供しています。宣言的なLinkコンポーネントと、プログラマティックなuseNavigateフックを適切に使い分けましょう。

Linkコンポーネント

基本的なリンク 
import { Link } from 'react-router-dom';

function Navigation() {
  return (
    <nav>
      {/* 通常のリンク */}
      <Link to="/about">About</Link>

      {/* 相対パス */}
      <Link to="../parent">親ページへ</Link>

      {/* stateを渡す */}
      <Link to="/profile" state={{ from: 'dashboard' }}>
        プロフィール
      </Link>
    </nav>
  );
}

NavLinkコンポーネント

NavLinkは、アクティブなリンクにスタイルを適用できる特別なLinkコンポーネントです。

NavLinkの使用 
import { NavLink } from 'react-router-dom';

function Sidebar() {
  return (
    <nav>
      <NavLink
        to="/dashboard"
        className={({ isActive }) =>
          isActive ? 'nav-link active' : 'nav-link'
        }
      >
        ダッシュボード
      </NavLink>

      <NavLink
        to="/settings"
        style={({ isActive }) => ({
          color: isActive ? 'red' : 'black',
          fontWeight: isActive ? 'bold' : 'normal'
        })}
      >
        設定
      </NavLink>
    </nav>
  );
}

10.6 プログラマティックナビゲーション

フォーム送信後のリダイレクトや、条件に応じたページ遷移など、プログラム的にページ遷移を行う場合は、useNavigateフックを使用します。

useNavigateの基本 
import { useNavigate } from 'react-router-dom';

function LoginForm() {
  const navigate = useNavigate();

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();

    // ログイン処理...
    const success = await login(username, password);

    if (success) {
      // ログイン成功後、ダッシュボードへ遷移
      navigate('/dashboard');

      // または、stateを渡す
      navigate('/dashboard', { state: { message: 'ログイン成功' } });

      // 履歴を残さずに遷移（replaceオプション）
      navigate('/dashboard', { replace: true });
    }
  };

  return <form onSubmit={handleSubmit}>...</form>;
}

履歴操作

戻る・進む操作 
function NavigationButtons() {
  const navigate = useNavigate();

  return (
    <div>
      {/* 1つ前のページに戻る */}
      <button onClick={() => navigate(-1)}>戻る</button>

      {/* 1つ先のページに進む */}
      <button onClick={() => navigate(1)}>進む</button>

      {/* 2つ前のページに戻る */}
      <button onClick={() => navigate(-2)}>2つ前へ</button>
    </div>
  );
}

10.7 ルートガードと認証

認証が必要なページへのアクセス制御を実装するために、保護されたルート（Protected Route）パターンを使用します。

ProtectedRouteコンポーネント 
import { Navigate, useLocation } from 'react-router-dom';

interface ProtectedRouteProps {
  children: React.ReactNode;
  isAuthenticated: boolean;
}

function ProtectedRoute({ children, isAuthenticated }: ProtectedRouteProps) {
  const location = useLocation();

  if (!isAuthenticated) {
    // 未認証の場合、ログインページへリダイレクト
    // 元のURLをstateに保存し、ログイン後に戻れるようにする
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return <>{children}</>;
}

function App() {
  const isAuthenticated = useAuth(); // 認証状態を取得

  return (
    <Routes>
      <Route path="/login" element={<Login />} />
      <Route
        path="/dashboard"
        element={
          <ProtectedRoute isAuthenticated={isAuthenticated}>
            <Dashboard />
          </ProtectedRoute>
        }
      />
    </Routes>
  );
}

セキュリティ注意

クライアント側のルートガードは、UIの保護のみを目的としています。機密データの保護には、必ずサーバー側での認証・認可を実装してください。

10.8 遅延読み込みとSuspense

コード分割を使用すると、初回読み込みのバンドルサイズを削減し、パフォーマンスを向上できます。React.lazyとSuspenseを組み合わせて、ルートごとのコンポーネントを遅延読み込みします。

遅延読み込みの仕組み

flowchart LR A["初回読み込み
main.js"] --> B{"ページ遷移"} B -->|Dashboard| C["dashboard.js
動的にロード"] B -->|Settings| D["settings.js
動的にロード"] B -->|Profile| E["profile.js
動的にロード"]
React.lazyとSuspenseの使用 
import { lazy, Suspense } from 'react';
import { Routes, Route } from 'react-router-dom';

// コンポーネントを遅延読み込み
const Dashboard = lazy(() => import('./pages/Dashboard'));
const Settings = lazy(() => import('./pages/Settings'));
const Profile = lazy(() => import('./pages/Profile'));

// ローディングコンポーネント
function LoadingFallback() {
  return (
    <div className="loading-container">
      <div className="spinner"></div>
      <p>読み込み中...</p>
    </div>
  );
}

function App() {
  return (
    <Suspense fallback={<LoadingFallback />}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/settings" element={<Settings />} />
        <Route path="/profile" element={<Profile />} />
      </Routes>
    </Suspense>
  );
}

パフォーマンス向上のコツ

すべてのルートを遅延読み込みする必要はありません。ホームページなど頻繁にアクセスされるページは通常読み込みし、大きなコンポーネントや使用頻度の低いページを遅延読み込みすると効果的です。

実習 10-1: ブログアプリのルーティング実装

React Routerを使って、ブログアプリケーションのルーティングを実装してみましょう。

要件

  • • ホームページ（/）: 記事一覧表示
  • • 記事詳細ページ（/posts/:id）: 個別記事の表示
  • • カテゴリページ（/category/:name）: カテゴリ別記事
  • • ダッシュボード（/dashboard）: 認証が必要

実装コード例

import { Routes, Route, Link, useParams } from 'react-router-dom';

// ホームページ
function Home() {
  return (
    <div>
      <h1>ブログ記事一覧</h1>
      <ul>
        <li><Link to="/posts/1">記事1</Link></li>
        <li><Link to="/posts/2">記事2</Link></li>
      </ul>
    </div>
  );
}

// 記事詳細ページ
function PostDetail() {
  const { id } = useParams<{ id: string }>();
  return <h1>記事 {id} の詳細</h1>;
}

// カテゴリページ
function Category() {
  const { name } = useParams<{ name: string }>();
  return <h1>カテゴリ: {name}</h1>;
}

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/posts/:id" element={<PostDetail />} />
      <Route path="/category/:name" element={<Category />} />
    </Routes>
  );
}

理解度確認クイズ

  1. Q1. React Router v6でネストルートの子要素を表示するコンポーネントは？

    回答を表示

    回答: Outletコンポーネントです。親ルートのコンポーネント内でOutletを使用すると、そこに子ルートのコンポーネントが表示されます。

  2. Q2. URLパラメータを取得するために使用するフックは？

    回答を表示

    回答: useParamsフックを使用します。例: const { id } = useParams()

  3. Q3. プログラム的にページ遷移を行うために使用するフックは？

    回答を表示

    回答: useNavigateフックです。navigate('/path')のように使用してページ遷移を実行します。

  4. Q4. コード分割で遅延読み込みするReactの機能は？

    回答を表示

    回答: React.lazyとSuspenseを組み合わせて使用します。lazyでコンポーネントを遅延読み込みし、Suspenseでローディング中の表示を制御します。
