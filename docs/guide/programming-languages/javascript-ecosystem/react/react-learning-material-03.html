第3章: PropsとState - データの管理

Reactのデータフロー - PropsとState

この章で学ぶこと
  • Propsの概念と使い方を理解する
  • 親コンポーネントから子コンポーネントへのデータ渡しを学ぶ
  • Propsのデフォルト値と型チェック（PropTypes）を設定する
  • Stateの概念とuseStateフックの基本を習得する
  • Stateの更新方法とイミュータブルな更新を理解する
  • 複数のStateを管理する方法を学ぶ
  • PropsとStateの違いと使い分けを理解する

3.1 Propsの概念

Props（プロップス）は「properties（プロパティ）」の略で、親コンポーネントから子コンポーネントにデータを渡すための仕組みです。Propsは読み取り専用であり、子コンポーネント内で変更することはできません。

Propsの特徴

  • 親から子への一方向データフロー
  • 読み取り専用（イミュータブル）
  • HTMLの属性のように渡す
  • 任意の型のデータを渡せる（文字列、数値、オブジェクト、関数など）
flowchart TD A["親コンポーネント
（データの送信元）"] -->|Props| B["子コンポーネント
（データの受信先）"] B -->|Props| C["孫コンポーネント"]

3.2 Propsの基本的な使い方

実習 3-1: Propsでデータを渡す

親コンポーネントから子コンポーネントにデータを渡す基本的な方法を学びます。

基本的なProps
// 子コンポーネント
function Greeting(props) {
  return <h1>こんにちは、{props.name}さん！</h1>;
}

// 親コンポーネント
function App() {
  return (
    <div>
      <Greeting name="太郎" />
      <Greeting name="花子" />
    </div>
  );
}
分割代入を使用した書き方（推奨）
// 分割代入で直接取得
function Greeting({ name }) {
  return <h1>こんにちは、{name}さん！</h1>;
}

// 複数のPropsを受け取る
function UserCard({ name, age, email }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>年齢: {age}歳</p>
      <p>メール: {email}</p>
    </div>
  );
}

// 使用例
function App() {
  return (
    <UserCard
      name="山田太郎"
      age={30}
      email="taro@example.com"
    />
  );
}
重要なポイント
  • 文字列以外のデータ（数値、真偽値、オブジェクト等）は中括弧{}で囲む
  • 分割代入を使うとコードがシンプルで読みやすくなる

3.3 様々な型のPropsを渡す

Propsには、文字列、数値、真偽値、配列、オブジェクト、関数など、あらゆるJavaScriptのデータ型を渡すことができます。

function Product({ name, price, inStock, tags, specs }) {
  return (
    <div>
      <h3>{name}</h3>
      <p>価格: {price}円</p>
      <p>在庫: {inStock ? "あり" : "なし"}</p>
      <p>タグ: {tags.join(", ")}</p>
      <p>重さ: {specs.weight}g</p>
    </div>
  );
}

// 使用例
function App() {
  return (
    <Product
      name="ノートパソコン"
      price={89800}
      inStock={true}
      tags={["電子機器", "PC", "新商品"]}
      specs={{ weight: 1200, color: "シルバー" }}
    />
  );
}

3.4 Propsのデフォルト値

Propsが渡されなかった場合に使用するデフォルト値を設定できます。

// 方法1: デフォルトパラメータを使用
function Button({ text = "クリック", color = "blue" }) {
  return <button style={{ backgroundColor: color }}>{text}</button>;
}

// 方法2: defaultPropsを使用（古い方法）
function Button({ text, color }) {
  return <button style={{ backgroundColor: color }}>{text}</button>;
}

Button.defaultProps = {
  text: "クリック",
  color: "blue"
};

// 使用例
function App() {
  return (
    <div>
      <Button />  {/* デフォルト値が使用される */}
      <Button text="送信" color="green" />
    </div>
  );
}

3.5 Stateの概念

State（ステート）は、コンポーネント内で管理される動的なデータです。Propsとは異なり、Stateはコンポーネント内で変更することができます。Stateが変更されると、Reactは自動的にコンポーネントを再レンダリングします。

Stateの特徴

  • コンポーネント内部で管理される
  • 変更可能（ミュータブル）
  • State変更時に自動的に再レンダリングされる
  • ユーザーの操作やデータの取得などに応じて変化する
flowchart LR A["ユーザーの操作"] --> B["State変更"] B --> C["コンポーネント再レンダリング"] C --> D["UIの更新"]

3.6 useStateフックの基本

関数コンポーネントでStateを使用するには、useStateフックを使用します。これはReact Hooksの中で最も基本的で頻繁に使用されるものです。

実習 3-2: useStateでカウンターを作成

useStateを使用してシンプルなカウンターアプリケーションを作成します。

useStateの基本構文
import { useState } from 'react';

function Counter() {
  // useState(初期値) は [現在の値, 更新関数] を返す
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        増やす
      </button>
    </div>
  );
}
useStateの仕組み
  • useState(0)：初期値を0に設定
  • count：現在のStateの値
  • setCount：Stateを更新する関数
  • setCount(count + 1)が呼ばれると、Reactはコンポーネントを再レンダリングする
注意：直接変更してはいけない

Stateを直接変更することはできません。必ず更新関数（setCount等）を使用してください。

// ❌ 間違い
count = count + 1;

// ✅ 正しい
setCount(count + 1);

3.7 様々な型のState

Stateには、数値、文字列、真偽値、配列、オブジェクトなど、あらゆるデータ型を使用できます。

function Example() {
  // 文字列のState
  const [name, setName] = useState("太郎");

  // 真偽値のState
  const [isVisible, setIsVisible] = useState(true);

  // 配列のState
  const [items, setItems] = useState(["りんご", "バナナ"]);

  // オブジェクトのState
  const [user, setUser] = useState({
    name: "山田太郎",
    age: 30
  });

  return <div>...</div>;
}

3.8 複数のStateの管理

1つのコンポーネントで複数のuseStateを使用することができます。関連するデータは1つのオブジェクトにまとめることも可能です。

実習 3-3: フォームで複数のStateを管理

複数のuseStateを使用してフォーム入力を管理します。

方法1: 個別のState
function UserForm() {
  const [firstName, setFirstName] = useState("");
  const [lastName, setLastName] = useState("");
  const [age, setAge] = useState(0);

  return (
    <form>
      <input
        value={firstName}
        onChange={(e) => setFirstName(e.target.value)}
        placeholder="名"
      />
      <input
        value={lastName}
        onChange={(e) => setLastName(e.target.value)}
        placeholder="姓"
      />
      <input
        type="number"
        value={age}
        onChange={(e) => setAge(Number(e.target.value))}
        placeholder="年齢"
      />
    </form>
  );
}
方法2: オブジェクトで管理（関連データをまとめる）
function UserForm() {
  const [user, setUser] = useState({
    firstName: "",
    lastName: "",
    age: 0
  });

  const handleChange = (field, value) => {
    setUser({
      ...user,  // 既存のプロパティをコピー
      [field]: value  // 特定のフィールドのみ更新
    });
  };

  return (
    <form>
      <input
        value={user.firstName}
        onChange={(e) => handleChange('firstName', e.target.value)}
        placeholder="名"
      />
      <input
        value={user.lastName}
        onChange={(e) => handleChange('lastName', e.target.value)}
        placeholder="姓"
      />
      <input
        type="number"
        value={user.age}
        onChange={(e) => handleChange('age', Number(e.target.value))}
        placeholder="年齢"
      />
    </form>
  );
}

3.9 イミュータブルなState更新

Reactでは、Stateを更新する際、元のオブジェクトや配列を直接変更せず、新しいコピーを作成する必要があります。これを「イミュータブル（不変）な更新」と呼びます。

配列のイミュータブル更新

function TodoList() {
  const [todos, setTodos] = useState(["買い物", "掃除"]);

  // ❌ 間違い：直接変更
  const addTodoWrong = (newTodo) => {
    todos.push(newTodo);  // 元の配列を変更している
    setTodos(todos);  // 再レンダリングされない可能性
  };

  // ✅ 正しい：新しい配列を作成
  const addTodo = (newTodo) => {
    setTodos([...todos, newTodo]);  // スプレッド構文で新配列作成
  };

  // 削除
  const removeTodo = (index) => {
    setTodos(todos.filter((_, i) => i !== index));
  };

  return <div>...</div>;
}

オブジェクトのイミュータブル更新

function Profile() {
  const [user, setUser] = useState({
    name: "太郎",
    age: 25,
    address: { city: "東京", zip: "123-4567" }
  });

  // ❌ 間違い：直接変更
  const updateAgeWrong = () => {
    user.age = 26;  // 直接変更している
    setUser(user);
  };

  // ✅ 正しい：新しいオブジェクトを作成
  const updateAge = () => {
    setUser({ ...user, age: 26 });
  };

  // ネストされたオブジェクトの更新
  const updateCity = (newCity) => {
    setUser({
      ...user,
      address: {
        ...user.address,  // addressもコピー
        city: newCity
      }
    });
  };

  return <div>...</div>;
}
なぜイミュータブルな更新が必要なのか？

Reactは、Stateが変更されたかを参照の比較（===）で判断します。元のオブジェクトを直接変更すると、参照が変わらないため、Reactが変更を検出できず、再レンダリングされません。

3.10 PropsとStateの違い

PropsとStateは、どちらもコンポーネントのレンダリングに影響を与えるデータですが、重要な違いがあります。

比較項目 Props State
データの所有者 親コンポーネント コンポーネント自身
変更可能性 読み取り専用（イミュータブル） 変更可能（更新関数を使用）
データの流れ 親から子へ（上から下） コンポーネント内部
使用目的 コンポーネントの設定・カスタマイズ 動的なデータの管理
flowchart TB A["親コンポーネント
（State保持）"] -->|Props| B["子コンポーネント"] A -->|Props| C["子コンポーネント"] B -->|State| B C -->|State| C
理解度確認クイズ
  1. Propsとは何ですか？その主な特徴を説明してください。
  2. 子コンポーネント内でPropsを変更できますか？なぜですか？
  3. useStateフックは何を返しますか？
  4. なぜStateを直接変更してはいけないのですか？
  5. 配列のStateに新しい要素を追加する正しい方法を示してください。
  6. PropsとStateの主な違いを3つ挙げてください。
  7. オブジェクトのStateで特定のプロパティだけを更新する方法を説明してください。
解答例
  1. Propsは親コンポーネントから子コンポーネントにデータを渡すための仕組み。読み取り専用で、一方向データフローを実現する
  2. できません。Propsは読み取り専用（イミュータブル）であり、変更しようとするとエラーになります
  3. 配列を返す。1番目の要素は現在の値、2番目の要素はStateを更新する関数
  4. Reactが変更を検出できないため。参照が変わらないと再レンダリングされない
  5. setArray([...array, newElement]) のようにスプレッド構文で新しい配列を作成する
  6. ①Propsは親から渡される、Stateはコンポーネント内部で管理 ②Propsは読み取り専用、Stateは変更可能 ③Propsは設定に使用、Stateは動的データに使用
  7. スプレッド構文を使用: setUser({ ...user, propertyName: newValue })
