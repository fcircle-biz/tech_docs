3.1 関数コンポーネントの基本構造

関数コンポーネントは、Reactにおける最も基本的なコンポーネント形式です。クラスコンポーネントと比べてシンプルで、React Hooksの登場により、状態管理や副作用の処理も関数コンポーネントで完結できるようになりました。

基本的な関数コンポーネント

Greeting.tsx（基本的な関数コンポーネント） 
// 最もシンプルな関数コンポーネント
function Greeting() {
  return <h1>こんにちは！</h1>;
}

// アロー関数を使った書き方
const Greeting = () => {
  return <h1>こんにちは！</h1>;
};

// return を省略した書き方
const Greeting = () => <h1>こんにちは！</h1>;

// TypeScriptの型定義を含む書き方
const Greeting: React.FC = () => {
  return <h1>こんにちは！</h1>;
};

export default Greeting;

関数コンポーネントの命名規則

コンポーネント名は必ず大文字で始める必要があります（PascalCase）。これにより、ReactはHTMLタグと区別できます。例: Button、UserProfile

3.2 Propsの型定義と使い方

Props（プロパティ）は、親コンポーネントから子コンポーネントへデータを渡すための仕組みです。TypeScriptを使うことで、Propsの型を明確に定義し、型安全なコンポーネントを作成できます。

Propsのデータフロー

flowchart TD A["親コンポーネント
App"] -->|Props渡し| B["子コンポーネント
UserCard"] B -->|Props渡し| C["孫コンポーネント
Avatar"] style A fill:#e0f2fe style B fill:#dbeafe style C fill:#bfdbfe

Propsの重要な特性

Propsは読み取り専用（イミュータブル）です。子コンポーネント内でPropsの値を直接変更することはできません。これにより、データフローが一方向に保たれ、予測可能なコードになります。

Propsの型定義パターン

UserCard.tsx（Props型定義の基本） 
// インターフェースでProps型を定義
interface UserCardProps {
  name: string;
  age: number;
  email: string;
  isActive: boolean;
}

// 関数コンポーネントでPropsを受け取る
function UserCard(props: UserCardProps) {
  return (
    <div className="card">
      <h2>{props.name}</h2>
      <p>年齢: {props.age}</p>
      <p>メール: {props.email}</p>
      <p>ステータス: {props.isActive ? '有効' : '無効'}</p>
    </div>
  );
}

// 分割代入を使った書き方（推奨）
function UserCard({ name, age, email, isActive }: UserCardProps) {
  return (
    <div className="card">
      <h2>{name}</h2>
      <p>年齢: {age}</p>
      <p>メール: {email}</p>
      <p>ステータス: {isActive ? '有効' : '無効'}</p>
    </div>
  );
}

複雑なProps型の定義

ProductCard.tsx（複雑なProps型） 
// オブジェクトや配列を含むProps型
interface ProductCardProps {
  product: {
    id: number;
    name: string;
    price: number;
    tags: string[];
  };
  onAddToCart: (productId: number) => void;
  discount?: number; // オプショナル
}

function ProductCard({ product, onAddToCart, discount }: ProductCardProps) {
  const finalPrice = discount
    ? product.price * (1 - discount / 100)
    : product.price;

  return (
    <div className="product-card">
      <h3>{product.name}</h3>
      <p>価格: ¥{finalPrice.toLocaleString()}</p>
      <div>
        {product.tags.map(tag => (
          <span key={tag} className="tag">{tag}</span>
        ))}
      </div>
      <button onClick={() => onAddToCart(product.id)}>
        カートに追加
      </button>
    </div>
  );
}

3.3 オプショナルプロパティとデフォルト値

すべてのPropsを必須にすると、コンポーネントの使い勝手が悪くなることがあります。オプショナルプロパティとデフォルト値を活用することで、柔軟で使いやすいコンポーネントを設計できます。

オプショナルプロパティの定義

Button.tsx（オプショナルプロパティ） 
interface ButtonProps {
  children: React.ReactNode;    // 必須
  onClick?: () => void;          // オプショナル（?マーク）
  variant?: 'primary' | 'secondary' | 'danger';
  size?: 'small' | 'medium' | 'large';
  disabled?: boolean;
}

// デフォルト値を設定する方法1: 分割代入時に設定
function Button({
  children,
  onClick,
  variant = 'primary',   // デフォルト値
  size = 'medium',       // デフォルト値
  disabled = false       // デフォルト値
}: ButtonProps) {
  const baseStyle = 'px-4 py-2 rounded font-medium transition-colors';

  const variantStyle = {
    primary: 'bg-blue-500 text-white hover:bg-blue-600',
    secondary: 'bg-gray-200 text-gray-800 hover:bg-gray-300',
    danger: 'bg-red-500 text-white hover:bg-red-600'
  }[variant];

  const sizeStyle = {
    small: 'text-sm px-3 py-1',
    medium: 'text-base px-4 py-2',
    large: 'text-lg px-6 py-3'
  }[size];

  return (
    <button
      className={`${baseStyle} ${variantStyle} ${sizeStyle}`}
      onClick={onClick}
      disabled={disabled}
    >
      {children}
    </button>
  );
}

方法1: 分割代入でデフォルト値 

function Button({
  size = 'medium'
}: ButtonProps) {
  // ...
}

シンプルで読みやすい。最も推奨される方法。

方法2: defaultPropsを使用 

Button.defaultProps = {
  variant: 'primary',
  size: 'medium'
};

古い書き方。関数コンポーネントでは非推奨。

3.4 コンポーネントの合成パターン

コンポーネントの合成（Composition）は、小さなコンポーネントを組み合わせて、より複雑なUIを構築する手法です。継承ではなく合成を使うことで、柔軟で再利用可能なコンポーネントを作成できます。

childrenプロパティを使った合成

Card.tsx（childrenを使った合成） 
// カードコンポーネント（コンテナ）
interface CardProps {
  children: React.ReactNode;
  title?: string;
}

function Card({ children, title }: CardProps) {
  return (
    <div className="border rounded-lg p-4 shadow-md">
      {title && <h3 className="text-lg font-bold mb-2">{title}</h3>}
      {children}
    </div>
  );
}

// 使用例
function App() {
  return (
    <div>
      <Card title="ユーザー情報">
        <p>名前: 田中太郎</p>
        <p>メール: tanaka@example.com</p>
      </Card>

      <Card title="商品情報">
        <img src="/product.jpg" alt="商品" />
        <p>価格: ¥10,000</p>
      </Card>
    </div>
  );
}

名前付きスロットパターン

Dialog.tsx（名前付きスロット） 
// 複数のスロットを持つダイアログ
interface DialogProps {
  header: React.ReactNode;
  content: React.ReactNode;
  footer: React.ReactNode;
  isOpen: boolean;
  onClose: () => void;
}

function Dialog({ header, content, footer, isOpen, onClose }: DialogProps) {
  if (!isOpen) return null;

  return (
    <div className="fixed inset-0 bg-black bg-opacity-50 flex items-center justify-center">
      <div className="bg-white rounded-lg p-6 max-w-md w-full">
        <div className="border-b pb-3 mb-4">
          {header}
        </div>
        <div className="mb-4">
          {content}
        </div>
        <div className="border-t pt-3 flex justify-end gap-2">

}

        <>
          <button onClick={() => setIsOpen(false)}>キャンセル</button>
          <button onClick={handleConfirm}>OK</button>

      }
    />
  );
}

3.5 レンダープロップスパターン

レンダープロップスは、関数をPropsとして受け取り、その関数を呼び出してコンポーネントをレンダリングするパターンです。ロジックを共有しながら、レンダリング内容を柔軟に変更できます。

MouseTracker.tsx（レンダープロップス） 
// マウス位置を追跡するコンポーネント
interface MouseTrackerProps {
  render: (position: { x: number; y: number }) => React.ReactNode;
}

function MouseTracker({ render }: MouseTrackerProps) {
  const [position, setPosition] = useState({ x: 0, y: 0 });

  useEffect(() => {
    const handleMouseMove = (e: MouseEvent) => {
      setPosition({ x: e.clientX, y: e.clientY });
    };

    window.addEventListener('mousemove', handleMouseMove);
    return () => window.removeEventListener('mousemove', handleMouseMove);
  }, []);

  return <>{render(position)}</>;
}

// 使用例
function App() {
  return (
    <div>
      <h1>マウストラッキング</h1>

      {/* パターン1: 座標を表示 */}
      <MouseTracker
        render={({ x, y }) => (
          <p>マウス位置: ({x}, {y})</p>
        )}
      />

      {/* パターン2: 追従する要素を表示 */}
      <MouseTracker
        render={({ x, y }) => (
          <div
            style={{
              position: 'fixed',
              left: x,
              top: y,
              width: 20,
              height: 20,
              background: 'blue',
              borderRadius: '50%'
            }}
          />
        )}
      />
    </div>
  );
}

レンダープロップス vs カスタムフック

現代のReactでは、レンダープロップスよりもカスタムフックを使う方が一般的です。上記の例は、カスタムフック useMousePosition() として実装する方がシンプルになります。レンダープロップスは、レガシーコードで見かけることがあるため、理解しておくことが重要です。

3.6 Higher-Order Components（HOC）

Higher-Order Component（高階コンポーネント、HOC）は、コンポーネントを受け取り、新しいコンポーネントを返す関数です。コンポーネント間でロジックを共有するための古典的なパターンですが、現在はカスタムフックに置き換えられることが多くなっています。

withLoading.tsx（HOCの例） 
// ローディング機能を追加するHOC
interface WithLoadingProps {
  isLoading: boolean;
}

function withLoading<P extends object>(
  Component: React.ComponentType<P>
) {
  return function WithLoadingComponent(
    props: P & WithLoadingProps
  ) {
    const { isLoading, ...restProps } = props;

    if (isLoading) {
      return <div>読み込み中...</div>;
    }

    return <Component {...(restProps as P)} />;
  };
}

// 通常のコンポーネント
interface UserListProps {
  users: Array<{ id: number; name: string }>;
}

function UserList({ users }: UserListProps) {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

// HOCでラップ
const UserListWithLoading = withLoading(UserList);

// 使用例
function App() {
  const [isLoading, setIsLoading] = useState(true);
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetchUsers().then(data => {
      setUsers(data);
      setIsLoading(false);
    });
  }, []);

  return <UserListWithLoading isLoading={isLoading} users={users} />;
}

HOCの注意点

HOCは便利ですが、以下の問題があります：
• ネストが深くなり「ラッパー地獄」になりやすい
• Propsの名前衝突が起きやすい
• TypeScriptの型推論が難しい

現代のReactでは、カスタムフックを使う方が推奨されています。

3.7 再利用可能なコンポーネント設計のベストプラクティス

優れたコンポーネント設計は、保守性と再利用性を高めます。ここでは、実践的なベストプラクティスを紹介します。

設計原則

単一責任の原則

1つのコンポーネントは1つの責任だけを持つべきです。複数の機能を持つ場合は、小さなコンポーネントに分割しましょう。

抽象化のレベル

適切な抽象化レベルを保ちます。汎用的すぎると使いにくく、具体的すぎると再利用できません。

疎結合

コンポーネント間の依存関係を最小限にします。Propsを通じてのみデータをやり取りします。

合成可能性

小さなコンポーネントを組み合わせて、複雑なUIを構築できるようにします。

実践例: 再利用可能なInputコンポーネント

Input.tsx（再利用可能なInput） 
interface InputProps extends React.InputHTMLAttributes<HTMLInputElement> {
  label?: string;
  error?: string;
  helperText?: string;
}

function Input({
  label,
  error,
  helperText,
  className = '',
  ...rest
}: InputProps) {
  return (
    <div className="mb-4">
      {label && (
        <label className="block text-sm font-medium mb-1">
          {label}
        </label>
      )}
      <input
        className={`
          w-full px-3 py-2 border rounded-md
          ${error ? 'border-red-500' : 'border-gray-300'}
          focus:outline-none focus:ring-2
          ${error ? 'focus:ring-red-500' : 'focus:ring-blue-500'}
          ${className}
        `}
        {...rest}
      />
      {error && (
        <p className="text-red-500 text-sm mt-1">{error}</p>
      )}
      {helperText && !error && (
        <p className="text-gray-500 text-sm mt-1">{helperText}</p>
      )}
    </div>
  );
}

// 使用例
function LoginForm() {
  return (
    <form>
      <Input
        label="メールアドレス"
        type="email"
        placeholder="user@example.com"
        helperText="ログインに使用するメールアドレスを入力してください"
      />

      <Input
        label="パスワード"
        type="password"
        error="パスワードは8文字以上である必要があります"
      />
    </form>
  );
}

実習 3-1: 再利用可能なCardコンポーネントの作成

これまで学んだProps、オプショナルプロパティ、コンポーネント合成の知識を使って、柔軟で再利用可能なCardコンポーネントを作成しましょう。

要件

  • タイトル（オプショナル）を表示できる
  • 画像（オプショナル）を表示できる
  • childrenで任意のコンテンツを挿入できる
  • フッター（オプショナル）を表示できる

実装のヒント

interface CardProps {
  title?: string;
  image?: string;
  imageAlt?: string;
  children: React.ReactNode;
  footer?: React.ReactNode;
}

function Card({ title, image, imageAlt, children, footer }: CardProps) {
  return (
    <div className="border rounded-lg overflow-hidden shadow-md">
      {/* 画像エリア */}
      {image && (
        <img src={image} alt={imageAlt || ''} className="w-full h-48 object-cover" />
      )}

      {/* コンテンツエリア */}
      <div className="p-4">
        {title && <h3 className="text-xl font-bold mb-2">{title}</h3>}
        {children}
      </div>

      {/* フッターエリア */}
      {footer && (
        <div className="border-t p-4 bg-gray-50">
          {footer}
        </div>
      )}
    </div>
  );
}

// 使用例
function App() {
  return (
    <div className="grid grid-cols-3 gap-4">
      <Card
        title="商品A"
        image="/product-a.jpg"
        imageAlt="商品A"
        footer={<button className="btn-primary">購入</button>}
      >
        <p>価格: ¥10,000</p>
        <p>在庫: 5個</p>
      </Card>

      <Card title="お知らせ">
        <p>新機能をリリースしました！</p>
      </Card>
    </div>
  );
}

理解度確認クイズ

  1. Q1. Propsの重要な特性として正しいものはどれですか？

    回答を表示

    回答: Propsは読み取り専用（イミュータブル）です。子コンポーネント内でPropsの値を直接変更することはできません。これにより、データフローが一方向に保たれ、予測可能なコードになります。

  2. Q2. オプショナルプロパティを定義する方法として正しいものはどれですか？

    回答を表示

    回答: プロパティ名の後に ? を付けます。例: title?: string

  3. Q3. コンポーネントの合成（Composition）の主な利点は何ですか？

    回答を表示

    回答: 小さなコンポーネントを組み合わせることで、柔軟で再利用可能なUIを構築できます。継承よりも合成を使うことで、コードの保守性と拡張性が向上します。

  4. Q4. 現代のReactでHOCやレンダープロップスの代わりに推奨されるパターンは何ですか？

    回答を表示

    回答: カスタムフック（Custom Hooks）が推奨されます。カスタムフックは、ロジックの再利用をよりシンプルかつ型安全に実現でき、ネストが深くならないという利点があります。

第3章のまとめ

  • 関数コンポーネントはReactの基本的な構成要素で、シンプルで再利用可能
  • Propsは親から子へデータを渡す仕組みで、読み取り専用（イミュータブル）
  • TypeScriptでProps型を定義することで、型安全なコンポーネントを作成できる
  • オプショナルプロパティとデフォルト値で、柔軟なコンポーネントを設計できる
  • コンポーネントの合成により、小さな部品を組み合わせて複雑なUIを構築できる
  • レンダープロップスやHOCは、現代ではカスタムフックに置き換えられることが多い
