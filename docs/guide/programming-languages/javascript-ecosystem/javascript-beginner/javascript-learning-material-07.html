第7章

処理をまとめよう（関数の基礎）

よく使う処理を「関数」としてまとめ、必要な時に呼び出して使う方法を、料理のレシピに例えながら理解します。

この章で学ぶこと

  • 関数とは何か、なぜ必要なのかを理解する
  • 関数の基本的な書き方と呼び出し方を学ぶ
  • 引数を使って値を関数に渡す方法を理解する
  • 戻り値を使って関数から値を受け取る方法を学ぶ

7.1 関数とは何か

関数（かんすう）とは、よく使う処理をまとめて、名前をつけて保存したものです。 一度作った関数は、何度でも呼び出して使うことができます。

料理のレシピで考えてみよう

カレーを作るとき、毎回「野菜を切る → 炒める → 煮込む → ルウを入れる」という手順を覚えておく必要はありません。 レシピ本を開けば、「カレーの作り方」という名前で、必要な手順がまとめて書いてあります。 関数は、プログラムの世界の「レシピ」のようなものです。

例えば、「Hello!」と画面に表示する処理を何度も書くのは大変です。 そこで、この処理を関数としてまとめておけば、必要なときに簡単に呼び出せます。

flowchart LR A["関数を作る
処理をまとめて名前をつける"] --> B["関数を呼び出す
何度でも使える"] B --> C["結果が返ってくる"]

7.2 関数を使う理由

なぜ関数を使うのでしょうか？理由は大きく3つあります。

再利用できる

一度作った関数は、プログラムの色々な場所で何度でも使えます。 同じコードを何度も書く必要がありません。

読みやすい

処理に名前がついているので、プログラムを読んだときに 「何をしているか」がすぐにわかります。

修正しやすい

処理を変更したいとき、関数の中身を1箇所修正するだけで、 その関数を使っている全ての場所に反映されます。

重要ポイント

プログラムが長くなればなるほど、関数の便利さが実感できます。 最初は「わざわざ関数にしなくても」と思うかもしれませんが、 関数を使う習慣をつけることが、良いプログラマーへの第一歩です。

7.3 関数の基本的な書き方

JavaScriptで関数を作る（定義する）には、functionという言葉を使います。

関数の基本形 
// 関数を定義する（作る）
function あいさつ() {
    console.log("こんにちは!");
}

// 関数を呼び出す（使う）
あいさつ();

コードの読み方

  • function 「これから関数を定義します」という合図
  • あいさつ 関数の名前（好きな名前をつけられます）
  • () 関数には必ずつける記号（後で説明します）
  • {...} 波括弧の中に、実行したい処理を書く
  • あいさつ(); 関数を呼び出す（関数名の後に()をつけて実行）

関数の名前のつけ方

関数名は、変数名と同じルールで付けます。英語の小文字で始め、 複数の単語をつなげるときは2つ目以降の単語の最初を大文字にします（キャメルケース）。 例: sayHello, calculateTotal

自己紹介をする関数を作って実行してみましょう。

7.4 引数を使おう

引数（ひきすう）とは、関数に渡す値のことです。 引数を使うと、状況に応じて処理を変えることができます。

料理のレシピで考えると

「カレーを作る」というレシピがあるとき、「何人分作るか」を指定できると便利ですよね。 この「何人分」が引数です。同じレシピでも、引数（人数）を変えれば、材料の量が変わります。

引数を使った関数 
// 名前を受け取って挨拶する関数
function aisatsu(namae) {
    console.log("こんにちは、" + namae + "さん!");
}

// 関数を呼び出すときに値を渡す
aisatsu("太郎");  // 結果: こんにちは、太郎さん!
aisatsu("花子");  // 結果: こんにちは、花子さん!

引数の仕組み

  • function aisatsu(namae) 関数定義の()の中に、引数の名前を書く
  • aisatsu("太郎") 関数を呼び出すときに()の中に値を書く
  • namae 関数の中では、引数の名前で値を使える（変数のようなもの）

引数は複数使うこともできます。複数の引数を使うときは、カンマ（,）で区切ります。

複数の引数を使う 
// 2つの引数を受け取る関数
function tashizan(kazu1, kazu2) {
    let kotae = kazu1 + kazu2;
    console.log(kazu1 + " + " + kazu2 + " = " + kotae);
}

// 関数を呼び出す
tashizan(3, 5);   // 結果: 3 + 5 = 8
tashizan(10, 20); // 結果: 10 + 20 = 30

実習 7-2: 引数を使った計算関数

2つの数を受け取って掛け算をする関数を作りましょう。

kakezan.html 
// 掛け算をする関数
function kakezan(kazu1, kazu2) {
    let kotae = kazu1 * kazu2;
    console.log(kazu1 + " × " + kazu2 + " = " + kotae);
}

// 色々な数で試してみる
kakezan(3, 4);
kakezan(7, 8);
kakezan(12, 5);

7.5 戻り値を使おう

戻り値（もどりち）とは、関数が計算した結果を返す値のことです。 returnという言葉を使って、値を返します。

自動販売機で考えると

自動販売機にお金を入れて（引数）、ボタンを押すと、飲み物が出てきます（戻り値）。 関数も同じで、値を渡すと、計算した結果が返ってきます。

戻り値を使った関数 
// 2つの数を足して結果を返す関数
function tasu(a, b) {
    let kekka = a + b;
    return kekka;  // 結果を返す
}

// 関数の戻り値を変数に入れる
let kotae1 = tasu(5, 3);
console.log("5 + 3 = " + kotae1);  // 結果: 5 + 3 = 8

// 戻り値をそのまま使うこともできる
console.log("10 + 20 = " + tasu(10, 20));  // 結果: 10 + 20 = 30

重要ポイント

returnを使うと、その時点で関数の処理が終了します。 returnの後ろに書いたコードは実行されません。

戻り値を使うと、関数を組み合わせて複雑な計算ができます。

関数を組み合わせる 
// 平方（2乗）を計算する関数
function heiho(kazu) {
    return kazu * kazu;
}

// 2つの数の平方を足す
let kekka = heiho(3) + heiho(4);
console.log("3の2乗 + 4の2乗 = " + kekka);  // 結果: 9 + 16 = 25

実習 7-3: 消費税計算関数を作ろう

価格を渡すと、消費税込みの金額を返す関数を作りましょう。

shouhizei.html 
// 消費税込みの金額を計算する関数
function shouhizeiKomi(kakaku) {
    let zeikomi = kakaku * 1.1;  // 消費税10%を加算
    return zeikomi;
}

// 色々な価格で試してみる
console.log("1000円の税込価格: " + shouhizeiKomi(1000) + "円");
console.log("500円の税込価格: " + shouhizeiKomi(500) + "円");

// 変数に結果を入れて使う
let goukei = shouhizeiKomi(3000);
console.log("3000円の商品の税込価格: " + goukei + "円");

7.6 実践例：複数の関数を組み合わせる

ここまで学んだことを使って、複数の関数を組み合わせたプログラムを作ってみましょう。

買い物合計計算プログラム 
// 消費税込みの金額を計算する関数
function zeiKomi(kakaku) {
    return kakaku * 1.1;
}

// 送料を計算する関数（5000円未満は500円、以上は無料）
function souryo(kingaku) {
    if (kingaku < 5000) {
        return 500;
    } else {
        return 0;
    }
}

// 最終的な支払い金額を計算する関数
function shiharaiKingaku(shouhinKakaku) {
    let zeikomi = zeiKomi(shouhinKakaku);
    let sosouryo = souryo(zeikomi);
    let goukei = zeikomi + sosouryo;
    return goukei;
}

// 実際に使ってみる
console.log("3000円の商品の支払い金額: " + shiharaiKingaku(3000) + "円");
console.log("6000円の商品の支払い金額: " + shiharaiKingaku(6000) + "円");

このプログラムのポイント

  • それぞれの機能を別々の関数に分けている
  • 関数の中で別の関数を呼び出している
  • 1つの関数は1つの仕事だけをする（わかりやすい）

第7章のまとめ

  • 関数は、よく使う処理をまとめて名前をつけたもの
  • 関数を使うと、再利用できる・読みやすい・修正しやすいというメリットがある
  • 引数を使って、関数に値を渡すことができる
  • 戻り値を使って、関数から計算結果を受け取ることができる
  • 複数の関数を組み合わせて、複雑なプログラムを作ることができる

理解度確認クイズ

  1. Q1. 関数を使う3つのメリットを答えてください。

    回答を表示

    ① 再利用できる（同じコードを何度も書かなくて良い）
    ② 読みやすい（処理に名前がついているので何をしているかわかりやすい）
    ③ 修正しやすい（1箇所変更すれば全ての場所に反映される）

  2. Q2. 引数とは何ですか？簡単に説明してください。

    回答を表示

    引数とは、関数に渡す値のことです。関数を呼び出すときに()の中に値を書くことで、 その値を関数の中で使うことができます。

  3. Q3. 次のコードの実行結果は何ですか？

    function kakeru(a, b) {
    return a * b;
}
console.log(kakeru(6, 7));
    回答を表示

    42
    （関数kakeruは2つの引数を掛け算した結果を返すので、6 × 7 = 42が表示されます）

  4. Q4. returnを書くと、どうなりますか？

    回答を表示

    returnを書くと、その時点で関数の処理が終了し、returnの後ろに書いた値が 関数を呼び出した側に返されます（戻り値）。 returnの後ろに書いたコードは実行されません。

トラブルシューティング

関数を呼び出しても何も起きない

原因: 関数名の後ろに()を付け忘れている可能性があります

解決方法: 関数を呼び出すときは、必ず関数名の後ろに()を付けます。myFunction()のように書いてください。

引数が undefined になる

原因: 関数を呼び出すときに引数を渡し忘れています

解決方法: 関数定義で引数を指定している場合は、呼び出し時も同じ数の値を渡す必要があります。myFunction(10, 20)のように書いてください。

戻り値が使えない

原因: 関数の中でreturnを書いていない

解決方法: 関数から値を返したい場合は、必ずreturn 値;を書いてください。returnがないと、関数は何も返しません（undefinedになります）。
