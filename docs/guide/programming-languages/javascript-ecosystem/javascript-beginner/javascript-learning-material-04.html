第4章

変数で情報を記憶しよう

変数という「箱」を使って、数値や文字などの情報を記憶・管理する方法を、買い物リストなどの身近な例で学びます。

この章で学ぶこと

  • 変数とは何か、なぜ必要なのかを理解する
  • 変数の宣言方法（let、const）を習得する
  • JavaScriptの基本的なデータ型（文字列、数値、真偽値）を知る
  • 変数の命名規則と良い変数名の付け方を学ぶ

4.1 変数とは何か

変数とは、データを入れておく「箱」のようなものです。 プログラムの中で、数値や文字などの情報を一時的に保存しておき、必要なときに取り出して使うことができます。

身近な例で考えてみよう

買い物に行くとき、「りんご」と書いたメモを持っていくことを想像してください。 この「りんご」というメモが「変数」です。メモには「りんご」という情報が保存されていて、 お店に着いたらメモを見て「りんごを買わなきゃ」と思い出すことができます。 プログラムでも同じように、変数に情報を保存して、後で使うことができます。

変数の仕組みを図で見てみましょう：

変数には名前を付けます。この名前を「変数名」と呼びます。 例えば、「りんごの個数」を保存する変数には appleCount という名前を付けたり、 「ユーザーの名前」を保存する変数には userName という名前を付けたりします。

4.2 変数の宣言方法

JavaScriptで変数を作ることを「変数を宣言する」と言います。 変数を宣言するには、主に let と const という2つのキーワードを使います。

1 let（レット）- 変更できる変数

let で宣言した変数は、後から値を変更することができます。 

// 変数を宣言して、値を代入する
let age = 25;
console.log(age); // 25 と表示される

// 値を変更する
age = 26;
console.log(age); // 26 と表示される

いつ使う？

カウンターの数値や、ユーザーが入力した内容など、後で変わる可能性がある値を保存するときに使います。

2 const（コンスト）- 変更できない変数

const で宣言した変数は、後から値を変更することができません（定数）。 

// 変数を宣言して、値を代入する
const birthYear = 2000;
console.log(birthYear); // 2000 と表示される

// 値を変更しようとするとエラーになる
birthYear = 2001; // エラー！

いつ使う？

生年月日や税率など、一度決めたら変わらない値を保存するときに使います。 間違って値を変更してしまうことを防げるので、安全です。

どちらを使えばいい？

迷ったら、まず const を使うことをおすすめします。 後から「やっぱり値を変更したい」となったら、その時に let に変えれば大丈夫です。 const を使うことで、意図しない変更を防ぎ、バグ（プログラムの間違い）を減らすことができます。

var は使わない

古いJavaScriptのコードでは var というキーワードも使われていますが、 現代のJavaScriptでは使わないようにしましょう。 var には予期しない動作をすることがあるため、let や const を使うのが推奨されています。

4.3 データ型の基本

変数に入れることができるデータには、いくつかの「型（かた）」があります。 これを「データ型」と呼びます。まずは、最もよく使う3つの型を覚えましょう。

1 文字列（String）

文字や文章を表すデータ型です。シングルクォート（'）またはダブルクォート（"）で囲みます。 

// 文字列の例
const name = "太郎";
const message = '今日はいい天気ですね';
const emoji = "😊";

注意点

数字でも、クォートで囲むと文字列になります。
例: "123" は文字列、123 は数値です。

2 数値（Number）

数字を表すデータ型です。クォートは付けません。 

// 数値の例
const age = 25;
const price = 1980;
const temperature = -5.5;

計算ができる

数値型の変数は、足し算（+）、引き算（-）、掛け算（*）、割り算（/）などの計算ができます。

3 真偽値（Boolean）

「本当（true）」か「嘘（false）」の2つの値だけを持つデータ型です。 

// 真偽値の例
const isRaining = true;  // 雨が降っている
const isSunny = false;   // 晴れていない

いつ使う？

「ログイン済みかどうか」「エラーが発生したかどうか」など、 Yes/Noで答えられることを表すときに使います。

4.4 変数を使ってみよう

実際に変数を使ったプログラムを書いてみましょう。

実習：自己紹介プログラムを作ろう

変数を使って、自己紹介のメッセージを表示するプログラムを作ってみましょう。 

// 自己紹介の情報を変数に保存
const name = "太郎";
const age = 25;
const hobby = "プログラミング";

// 変数を使ってメッセージを作る
console.log("はじめまして！");
console.log(name + "です。");
console.log(age + "歳です。");
console.log("趣味は" + hobby + "です。");

実行結果

はじめまして！
太郎です。
25歳です。
趣味はプログラミングです。

変数を使う利点

例えば、名前を「太郎」から「花子」に変えたいとき、変数を使っていれば const name = "花子"; の1箇所を変えるだけで、 すべての表示が「花子」に変わります。変数を使わないと、何箇所も書き換える必要があります。

計算にも使える

数値型の変数を使えば、計算もできます。 

// 商品の値段を計算する
const applePrice = 150;  // りんご1個の値段
const appleCount = 3;    // 買う個数

// 合計金額を計算
const totalPrice = applePrice * appleCount;

console.log("りんごを" + appleCount + "個買います。");
console.log("合計金額は" + totalPrice + "円です。");
// 実行結果:
// りんごを3個買います。
// 合計金額は450円です。

4.5 変数の命名規則

変数名は自由に付けられますが、いくつかのルールとマナーがあります。

必須のルール（守らないとエラー）

  • 使える文字は、英字（a-z、A-Z）、数字（0-9）、アンダースコア（_）、ドル記号（$）のみ
  • 数字で始めることはできない（例: 1name はNG、name1 はOK）
  • 予約語（JavaScriptで特別な意味を持つ言葉）は使えない（例: let、const、if など）
  • 大文字と小文字は区別される（name と Name は別の変数）

推奨のルール（わかりやすいコードを書くために）

  • 英語で書く（日本語も使えますが、英語が一般的）
  • 意味がわかる名前にする（x より age の方が良い）
  • キャメルケースを使う（複数の単語をつなげるとき、2単語目以降の先頭を大文字にする）

キャメルケースの例

// 良い例（キャメルケース）
const userName = "太郎";
const totalPrice = 1000;
const isLoggedIn = true;

// 避けた方が良い例
const username = "太郎";      // 読みにくい
const total_price = 1000;     // JavaScriptでは一般的でない
const IsLoggedIn = true;      // 先頭が大文字（別の用途で使う書き方）

良い変数名を付けるコツ

変数名を見ただけで「何を保存しているか」がわかるようにしましょう。
例えば、ユーザーの年齢を保存するなら a より userAge の方が、 後で見たときに理解しやすいです。

第4章のまとめ

  • 変数は、データを入れておく「箱」のようなもの
  • letは変更可能な変数、constは変更不可の変数（定数）
  • 主なデータ型は文字列（String）、数値（Number）、真偽値（Boolean）
  • 変数名は意味がわかる英語で、キャメルケースを使うのが推奨
  • 迷ったらまず const を使い、変更が必要なら let に変える

理解度確認クイズ

  1. Q1. 変数とは何ですか？簡単に説明してください。

    回答を表示

    変数とは、データを入れておく「箱」のようなものです。 数値や文字などの情報を一時的に保存しておき、必要なときに取り出して使うことができます。

  2. Q2. let と const の違いを説明してください。

    回答を表示

    let は後から値を変更できる変数を宣言します。
    const は後から値を変更できない変数（定数）を宣言します。
    迷ったら、まず const を使い、変更が必要になったら let に変えるのが推奨されています。

  3. Q3. 次のデータ型のうち、正しいものをすべて選んでください。
    ① 文字列（String）
    ② 数値（Number）
    ③ 真偽値（Boolean）
    ④ 配列（Array）

    回答を表示

    すべて正しいです。①②③はこの章で学んだ基本的なデータ型です。
    ④の配列（Array）も重要なデータ型ですが、これは後の章で学習します。

  4. Q4. 次のうち、正しい変数名はどれですか？
    ① 1userName
    ② user-name
    ③ userName
    ④ let

    回答を表示

    正解は ③ userName です。
    ① 数字で始まるのでエラーになります
    ② ハイフン（-）は使えません
    ④ let は予約語なので変数名として使えません

  5. Q5. 次のコードの実行結果を答えてください。

    const price = 100;
const count = 5;
const total = price * count;
console.log(total);
    回答を表示

    500 が表示されます。
    price（100）× count（5）= total（500）という計算が行われます。
