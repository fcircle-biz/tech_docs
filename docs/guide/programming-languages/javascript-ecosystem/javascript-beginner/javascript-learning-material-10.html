ユーザーの操作に反応しよう（イベント処理）

ボタンクリックやキー入力などのユーザー操作に反応するインタラクティブなプログラムの作り方を実践します。

この章で学ぶこと

  • イベントとは何か、どんな種類があるかを理解する
  • イベントリスナーを使って、ユーザー操作に反応するプログラムを書けるようになる
  • クリック、入力、キーボード、マウス操作など、様々なイベントを使いこなせる
  • インタラクティブな（対話的な）Webページを作成できる

10.1 イベントとは何か

イベントとは、Webページ上で起こる「出来事」のことです。 例えば、「ボタンがクリックされた」「キーボードのキーが押された」「マウスが動いた」などが、すべてイベントです。

日常生活でのイベントに例えると

現実世界でも、ドアベルが鳴ったら出る、電話が鳴ったら出る、という風に「何かが起こったら（イベント）→それに対応する（処理）」という行動をしています。 プログラミングでも同じように、「ボタンが押されたら（イベント）→画面に文字を表示する（処理）」という仕組みを作ることができます。

JavaScriptでは、イベントが発生したときに自動的に実行される処理を設定できます。 この処理のことをイベントハンドラー（またはイベントリスナー）と呼びます。

ボタンクリック"] --> B["イベント発生

主なイベントの種類

クリックイベント

click, dblclick（ダブルクリック）

キーボードイベント

keydown（押した時）, keyup（離した時）

マウスイベント

mouseover（乗った時）, mouseout（離れた時）

入力イベント

input（入力中）, change（変更完了時）

10.2 イベントリスナーの基本

イベントに反応する処理を設定するには、addEventListenerというメソッドを使います。 これは「イベントを監視する人（リスナー）を追加する」という意味です。

基本的な書き方 
// 要素を取得
const button = document.querySelector('#myButton');

// イベントリスナーを追加
button.addEventListener('click', function() {
    // ボタンがクリックされたときに実行される処理
    console.log('ボタンが押されました！');
});

addEventListenerの3つの要素

  1. 対象の要素：どの要素のイベントを監視するか（button）
  2. イベントの種類：どんなイベントを監視するか（'click'）
  3. 実行する関数：イベントが起きたら何をするか（function() {...}）

10.3 クリックイベントを使ってみよう

最も基本的なイベントであるclickイベントを使って、ボタンを押したら文字が表示される仕組みを作ってみましょう。

実習 10-1: クリックで文字を表示する

ボタンをクリックすると、画面に「こんにちは！」と表示されるプログラムを作ります。

HTMLファイル (click-demo.html)

click-demo.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>クリックイベント</title>
</head>
<body>
    <!-- ボタン要素 -->
    <button id="greetBtn">挨拶する</button>

    <!-- メッセージ表示エリア -->
    <p id="message"></p>

    <script src="script.js"></script>
</body>
</html>

JavaScriptファイル (script.js)

script.js 
// ボタン要素を取得
const greetBtn = document.querySelector('#greetBtn');
// メッセージ表示エリアを取得
const message = document.querySelector('#message');

// クリックイベントリスナーを追加
greetBtn.addEventListener('click', function() {
    // ボタンがクリックされたら、メッセージを表示
    message.textContent = 'こんにちは！';
});

何度でもクリックできる

イベントリスナーは一度設定すると、何度でも反応します。上のボタンを何回クリックしても、毎回「こんにちは！」が表示されます。 クリックするたびにカウントを増やしたり、色を変えたりすることもできます。

10.4 入力イベントを使おう

入力フォーム（テキストボックス）に文字を入力したときに反応するinputイベントを使ってみましょう。

リアルタイムで入力を表示 
// テキストボックスを取得
const nameInput = document.querySelector('#nameInput');
// 表示エリアを取得
const displayArea = document.querySelector('#displayArea');

// inputイベントリスナーを追加
nameInput.addEventListener('input', function() {
    // 入力された内容をリアルタイムで表示
    displayArea.textContent = 'あなたの名前: ' + nameInput.value;
});

inputとchangeの違い

  • input：文字を入力するたびに（リアルタイムで）反応する
  • change：入力が完了してフォーカスが外れたときに反応する

リアルタイムで表示したいときはinput、入力が完了したときだけ処理したいときはchangeを使います。

実用例：文字数カウンター

文字数をカウント 
const textArea = document.querySelector('#textArea');
const counter = document.querySelector('#counter');

textArea.addEventListener('input', function() {
    // 入力された文字数を計算
    const count = textArea.value.length;
    // 文字数を表示
    counter.textContent = count + '文字';
});

10.5 キーボードイベント

キーボードのキーが押されたり離されたりしたときに反応するkeydownイベントとkeyupイベントを使ってみましょう。

Enterキーで送信 
const inputField = document.querySelector('#inputField');

// keydownイベントリスナーを追加
inputField.addEventListener('keydown', function(event) {
    // 押されたキーがEnterキーかどうかを確認
    if (event.key === 'Enter') {
        console.log('Enterキーが押されました！');
        console.log('入力内容: ' + inputField.value);
    }
});

eventオブジェクトとは

イベントリスナーの関数には、eventという特別な情報が自動的に渡されます。 この中には「どのキーが押されたか」「マウスの位置はどこか」などの詳しい情報が入っています。 event.keyで押されたキーの名前がわかります（'Enter', 'a', 'ArrowUp'など）。

よく使うキーの名前

Enter → 'Enter'

スペース → ' '

矢印キー → 'ArrowUp', 'ArrowDown'

文字キー → 'a', 'b', '1'

10.6 マウスイベント

マウスが要素の上に乗ったり離れたりしたときに反応するmouseoverイベントとmouseoutイベントを使ってみましょう。

マウスで色を変える 
const box = document.querySelector('#box');

// マウスが乗ったとき
box.addEventListener('mouseover', function() {
    box.style.backgroundColor = 'lightblue';
    box.textContent = 'マウスが乗っています';
});

// マウスが離れたとき
box.addEventListener('mouseout', function() {
    box.style.backgroundColor = 'lightgray';
    box.textContent = 'マウスを乗せてください';
});

その他の便利なマウスイベント

1

mousedown / mouseup

マウスボタンを押した瞬間 / 離した瞬間

2

mousemove

マウスが動いている間ずっと

3

dblclick

ダブルクリックされたとき

10.7 実践演習：カウンターアプリを作ろう

これまで学んだイベント処理を組み合わせて、ボタンをクリックすると数が増えたり減ったりするカウンターアプリを作りましょう。

実習 10-2: カウンターアプリ

「増やす」ボタンと「減らす」ボタン、「リセット」ボタンを持つカウンターアプリを作成します。

HTMLファイル

counter.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>カウンターアプリ</title>
</head>
<body>
    <h1>カウンター</h1>
    <p id="count">0</p>
    <button id="increaseBtn">増やす</button>
    <button id="decreaseBtn">減らす</button>
    <button id="resetBtn">リセット</button>

    <script src="counter.js"></script>
</body>
</html>

JavaScriptファイル

counter.js 
// 要素を取得
const countDisplay = document.querySelector('#count');
const increaseBtn = document.querySelector('#increaseBtn');
const decreaseBtn = document.querySelector('#decreaseBtn');
const resetBtn = document.querySelector('#resetBtn');

// カウントを保持する変数
let count = 0;

// 増やすボタンのクリックイベント
increaseBtn.addEventListener('click', function() {
    count++;  // countを1増やす
    countDisplay.textContent = count;
});

// 減らすボタンのクリックイベント
decreaseBtn.addEventListener('click', function() {
    count--;  // countを1減らす
    countDisplay.textContent = count;
});

// リセットボタンのクリックイベント
resetBtn.addEventListener('click', function() {
    count = 0;  // countを0に戻す
    countDisplay.textContent = count;
});

チャレンジ問題

カウンターアプリに以下の機能を追加してみましょう：

  • カウントが0より小さくならないようにする（0未満にならない制限）
  • カウントが10になったら「目標達成！」と表示する
  • カウントが5以上になったら数字の色を赤にする

トラブルシューティング

ボタンをクリックしても何も起こらない

原因: イベントリスナーが正しく設定されていないか、要素の取得に失敗しています。

解決方法: querySelector()で要素が正しく取得できているか確認しましょう。console.log()で要素を出力して、nullになっていないかチェックしてください。また、HTMLファイルでJavaScriptファイルが正しく読み込まれているか確認しましょう。

「Cannot read property 'addEventListener' of null」というエラーが出る

原因: querySelector()で要素が見つからず、nullになっています。

解決方法: IDやクラス名のスペルが正しいか確認しましょう。また、JavaScriptコードがHTMLより先に実行されていないか確認してください。scriptタグは</body>の直前に置くのがおすすめです。

イベントが何度も実行されてしまう

原因: 同じ要素に同じイベントリスナーを複数回追加している可能性があります。

解決方法: addEventListener()は一度だけ実行されるように注意しましょう。ループの中で実行していないか確認してください。
