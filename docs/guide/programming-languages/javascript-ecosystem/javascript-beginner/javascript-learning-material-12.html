  • 自分でプログラムを設計・実装する力を身につける

12.1 ToDoアプリとは

ToDoアプリ（やることリストアプリ）は、やるべきタスクを管理するためのアプリです。 タスクの追加、完了マーク、削除などの機能があり、日常生活や仕事で広く使われています。

完成するアプリの機能

  • 新しいタスクを追加できる
  • タスクを完了済みにマークできる
  • 不要なタスクを削除できる
  • タスクを「全て」「未完了」「完了済み」でフィルター表示できる
  • 残りのタスク数を表示できる

12.5 タスクを追加する機能を作る

それでは、JavaScriptでタスクを追加する機能を実装していきます。 まず、必要な変数と基本的な仕組みを作ります。

タスク追加の流れ：

flowchart LR A["ユーザーが
テキスト入力"] --> B["追加ボタンを
クリック"] B --> C["入力内容を
取得"] C --> D{入力は空？} D -->|はい| E["何もしない
警告を表示"] D -->|いいえ| F["タスク配列に
追加"] F --> G["画面に
タスク表示"] G --> H["入力欄を
クリア"]
script.js（scriptタグ内に記述） 
// タスクを保存する配列
let tasks = [];

// DOM要素を取得
const taskInput = document.getElementById('taskInput');
const addButton = document.getElementById('addButton');
const taskList = document.getElementById('taskList');
const taskCount = document.getElementById('taskCount');

// タスクを追加する関数
function addTask() {
    const taskText = taskInput.value.trim(); // 前後の空白を削除

    // 入力が空なら何もしない
    if (taskText === '') {
        alert('タスクを入力してください');
        return;
    }

    // タスクオブジェクトを作成
    const task = {
        id: Date.now(), // ユニークなIDとして現在時刻を使用
        text: taskText,
        completed: false // 最初は未完了
    };

    // タスク配列に追加
    tasks.push(task);

    // 入力欄をクリア
    taskInput.value = '';

    // 画面を更新
    renderTasks();
}

// 追加ボタンをクリックしたときの処理
addButton.addEventListener('click', addTask);

コードの解説

  • tasks配列: すべてのタスクを保存する場所
  • trim(): 文字列の前後の空白を削除するメソッド
  • Date.now(): 現在の時刻をミリ秒で取得（ユニークなIDとして使用）
  • taskオブジェクト: id、text、completedの3つの情報を持つ
  • renderTasks(): タスクを画面に表示する関数（次で作成）

12.6 タスクを画面に表示する

タスク配列の内容を画面に表示する`renderTasks()`関数を作成します。 この関数は、タスクが追加・削除・更新されるたびに呼び出されます。

renderTasks関数の追加 
// タスクを画面に表示する関数
function renderTasks() {
    // リストを一度空にする
    taskList.innerHTML = '';

    // タスク配列をループして、1つずつ表示
    tasks.forEach(task => {
        // li要素を作成
        const li = document.createElement('li');
        li.className = 'task-item';
        if (task.completed) {
            li.classList.add('completed'); // 完了済みならクラス追加
        }

        // チェックボックスを作成
        const checkbox = document.createElement('input');
        checkbox.type = 'checkbox';
        checkbox.checked = task.completed;
        checkbox.addEventListener('change', () => toggleTask(task.id));

        // タスクテキストを作成
        const span = document.createElement('span');
        span.textContent = task.text;

        // 削除ボタンを作成
        const deleteBtn = document.createElement('button');
        deleteBtn.textContent = '削除';
        deleteBtn.className = 'delete-btn';
        deleteBtn.addEventListener('click', () => deleteTask(task.id));

        // li要素に追加
        li.appendChild(checkbox);
        li.appendChild(span);
        li.appendChild(deleteBtn);

        // リストに追加
        taskList.appendChild(li);
    });

    // 残りタスク数を更新
    updateTaskCount();
}

renderTasks()の重要性

この関数は、tasksデータ配列の内容を元に、画面を毎回再構築します。 データが変わったら画面も更新する、というデータ駆動の考え方です。 これにより、データと表示を一致させることができます。

12.7 タスクの完了と削除機能

チェックボックスをクリックしたときの完了マーク機能と、削除ボタンをクリックしたときの削除機能を実装します。

完了と削除の関数 
// タスクの完了状態を切り替える関数
function toggleTask(id) {
    // 指定されたIDのタスクを見つける
    const task = tasks.find(t => t.id === id);
    if (task) {
        // completedを反転（trueならfalse、falseならtrue）
        task.completed = !task.completed;
        // 画面を更新
        renderTasks();
    }
}

// タスクを削除する関数
function deleteTask(id) {
    // 指定されたID以外のタスクだけを残す
    tasks = tasks.filter(t => t.id !== id);
    // 画面を更新
    renderTasks();
}

// 残りタスク数を更新する関数
function updateTaskCount() {
    // 未完了のタスクだけをカウント
    const activeCount = tasks.filter(t => !t.completed).length;
    taskCount.textContent = activeCount;
}

配列メソッドの活用

  • find(): 条件に合う最初の要素を見つける
  • filter(): 条件に合う要素だけを抽出した新しい配列を作る
  • length: 配列の要素数を取得する

12.8 フィルター機能を追加

「全て」「未完了」「完了済み」のボタンをクリックすると、表示するタスクを絞り込む機能を追加します。

フィルター機能 
// 現在のフィルター状態
let currentFilter = 'all';

// フィルターボタンの処理
const filterButtons = document.querySelectorAll('.filter-btn');
filterButtons.forEach(btn => {
    btn.addEventListener('click', () => {
        // すべてのボタンからactiveクラスを削除
        filterButtons.forEach(b => b.classList.remove('active'));
        // クリックされたボタンにactiveクラスを追加
        btn.classList.add('active');
        // フィルター状態を更新
        currentFilter = btn.dataset.filter;
        // 画面を更新
        renderTasks();
    });
});

// renderTasks関数を修正（フィルター対応）
function renderTasks() {
    taskList.innerHTML = '';

    // フィルターに応じてタスクを絞り込む
    let filteredTasks = tasks;
    if (currentFilter === 'active') {
        filteredTasks = tasks.filter(t => !t.completed);
    } else if (currentFilter === 'completed') {
        filteredTasks = tasks.filter(t => t.completed);
    }

    // 絞り込まれたタスクを表示
    filteredTasks.forEach(task => {
        // （以前と同じli要素作成のコード）
    });

    updateTaskCount();
}

理解度確認クイズ

  1. Q1. renderTasks()関数の役割は何ですか？

    回答を表示

    tasksデータ配列の内容を元に、画面のタスクリストを再構築する関数です。 タスクが追加・削除・更新されるたびに呼び出され、データと表示を一致させます。

  2. Q2. タスクを削除する際に使った配列メソッドは何ですか？

    回答を表示

    filter()メソッドです。指定されたID以外のタスクだけを抽出した新しい配列を作成しました。
    例: tasks = tasks.filter(t => t.id !== id)

  3. Q3. Date.now()を使った理由は何ですか？

    回答を表示

    各タスクにユニーク（一意）なIDを割り当てるためです。 Date.now()は現在時刻をミリ秒で返すため、同じ値が重複する可能性がほぼありません。

  4. Q4. フィルター機能で「未完了」タスクだけを表示するには、どのような条件で絞り込みますか？

    回答を表示

    tasks.filter(t => !t.completed) のように、 completedプロパティがfalseのタスクだけを抽出します。
