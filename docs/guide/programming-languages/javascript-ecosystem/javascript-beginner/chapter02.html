第2章: 数字と文字を扱おう

簡単な計算機を作成します

変数を使って数値を保存し、計算結果を表示する電卓プログラムを作ってみましょう！

変数って何？

2.1 変数は「箱」のようなもの

変数とは、データを入れておく「箱」のようなものです。 この箱には名前を付けて、後で中身を取り出したり、新しい値を入れたりできます。

例えば：

  • 「名前」という箱に「太郎」を入れる
  • 「年齢」という箱に「25」を入れる
  • 「好きな食べ物」という箱に「ラーメン」を入れる

2.2 変数を作ってみよう

JavaScriptでは、

let
という言葉を使って変数を作ります。

実習 2-1: 最初の変数

開発者ツールのコンソールに以下を入力してください：

let myName = "太郎";
console.log(myName);

「太郎」と表示されましたか？これで「myName」という箱に「太郎」を入れて、それを表示しました。

変数の作り方の基本形：

let 変数名 = 値;

2.3 const - 変わらない値を扱おう

let
の他に、
const
という言葉もあります。 constは「定数（constant）」の略で、一度値を入れたら変更できない箱です。

実習 2-2: constを使う

開発者ツールのコンソールに以下を入力してください：

const PI = 3.14159;
console.log(PI);

// これを試すとエラーになります
PI = 3.14;  // エラー！constで宣言した値は変更できません

エラーメッセージが表示されましたか？constで宣言した値は変更できないことがわかります。

💡 letとconstの使い分け
宣言方法 再代入 使う場面
let できる ✓ 値が変わる可能性がある場合（カウンター、計算途中の値など）
const できない ✗ 値が変わらない場合（税率、円周率、設定値など）
実習 2-3: letとconstの違いを確認

以下を試して、違いを確認しましょう：

// letは値を変更できる
let age = 25;
console.log(age);  // 25
age = 26;          // 変更できる！
console.log(age);  // 26

// constは値を変更できない
const birthYear = 1998;
console.log(birthYear);  // 1998
birthYear = 1999;        // エラー！変更できない
🎯 プロのアドバイス

基本的にはconstを使い、値を変更する必要がある場合だけletを使うのが推奨されています。 これにより、意図しない値の変更を防ぎ、バグを減らすことができます。

数値を扱ってみよう

2.4 数値の基本

JavaScriptでは、数値をそのまま書くことができます。 小数点も使えます。

実習 2-4: 数値を扱う

以下を試してみましょう：

let age = 25;
let height = 170.5;
let temperature = -3;

console.log(age);
console.log(height);
console.log(temperature);

2.5 四則演算をしてみよう

JavaScriptでは、数学で使う計算記号を使って計算ができます。

計算 記号 例
足し算 + 5 + 3 = 8
引き算 - 10 - 4 = 6
掛け算 * 6 * 7 = 42
割り算 / 15 / 3 = 5
実習 2-5: 計算してみよう

以下を試してみましょう：

let a = 10;
let b = 3;

console.log(a + b);  // 13
console.log(a - b);  // 7
console.log(a * b);  // 30
console.log(a / b);  // 3.333...

2.6 計算結果を変数に保存

計算結果を変数に保存して、後で使うことができます。

実習 2-6: 計算結果を保存
let price = 100;
let quantity = 5;
let total = price * quantity;

console.log("商品の値段: " + price + "円");
console.log("数量: " + quantity + "個");
console.log("合計: " + total + "円");

文字列を扱ってみよう

2.7 文字列の基本

文字列は、文字や文章を表すデータです。 JavaScriptでは、文字列を

"
（ダブルクォート）や
'
（シングルクォート）で囲みます。

実習 2-7: 文字列を作る
let firstName = "太郎";
let lastName = "田中";
let hobby = 'プログラミング';

console.log(firstName);
console.log(lastName);
console.log(hobby);

2.8 文字列を繋げてみよう

+
記号を使って、文字列を繋げることができます。

実習 2-8: 文字列を繋げる
let firstName = "太郎";
let lastName = "田中";
let fullName = lastName + firstName;

console.log(fullName);  // 田中太郎

let greeting = "こんにちは、" + fullName + "さん";
console.log(greeting);  // こんにちは、田中太郎さん

2.9 テンプレートリテラル（便利な文字列）

バッククォート（`）を使うと、文字列の中に変数を直接埋め込めます。

実習 2-9: テンプレートリテラル
let name = "太郎";
let age = 25;

let message = `私の名前は${name}で、年齢は${age}歳です`;
console.log(message);

こちらの方が読みやすくて便利ですね！

実践プロジェクト：簡単な計算機

🎯 プロジェクト: 簡単な計算機を作ろう

変数と四則演算を使って、簡単な計算機プログラムを作りましょう。

ステップ1: 基本的な計算機

2つの数値を使って、四則演算を行います：

let num1 = 15;
let num2 = 4;

console.log("=== 簡単な計算機 ===");
console.log(`${num1} + ${num2} = ${num1 + num2}`);
console.log(`${num1} - ${num2} = ${num1 - num2}`);
console.log(`${num1} * ${num2} = ${num1 * num2}`);
console.log(`${num1} / ${num2} = ${num1 / num2}`);
ステップ2: 買い物の計算

実際の買い物を想定した計算をしてみましょう：

let itemName = "りんご";
let price = 150;
let quantity = 5;
let tax = 0.1;  // 10%の税金

let subtotal = price * quantity;
let taxAmount = subtotal * tax;
let total = subtotal + taxAmount;

console.log("=== 買い物計算 ===");
console.log(`商品名: ${itemName}`);
console.log(`単価: ${price}円`);
console.log(`数量: ${quantity}個`);
console.log(`小計: ${subtotal}円`);
console.log(`税額: ${taxAmount}円`);
console.log(`合計: ${total}円`);
ステップ3: 身長とBMIの計算

身長と体重からBMIを計算してみましょう：

let name = "太郎";
let height = 170; // cm
let weight = 65;  // kg

let heightInMeters = height / 100;
let bmi = weight / (heightInMeters * heightInMeters);

console.log("=== BMI計算 ===");
console.log(`名前: ${name}`);
console.log(`身長: ${height}cm`);
console.log(`体重: ${weight}kg`);
console.log(`BMI: ${bmi.toFixed(1)}`);

デバッグツールでステップ実行をマスターしよう

2.10 デバッガーって何？

デバッガーは、プログラムの動きを1行ずつ確認できる強力なツールです。 プログラムがどのように動いているか、変数にどんな値が入っているかを確認しながら実行できます。

💡 デバッガーでできること
  • プログラムを1行ずつ実行できる（ステップ実行）
  • 任意の場所でプログラムを一時停止できる（ブレークポイント）
  • 変数の値をリアルタイムで確認できる
  • エラーの原因を特定しやすくなる

2.11 Sourcesタブを開こう

まず、開発者ツールの「Sources」タブを開きます。ここがデバッガーの操作場所です。

実習 2-10: Sourcesタブを開く
  1. ブラウザでF12キーを押して開発者ツールを開く
  2. 上部のタブから「Sources」をクリック
  3. 左側にファイル一覧、中央にコード、右側にデバッグ情報が表示される

2.12 ブレークポイントを設定しよう

ブレークポイントは、プログラムを一時停止する場所を指定する機能です。 行番号をクリックすると、その行で実行が止まります。

実習 2-11: ブレークポイントの設定

以下のHTMLファイルを作成して試してみましょう：

<!DOCTYPE html>
<html>
<head>
    <title>デバッグ練習</title>
</head>
<body>
    <h1>デバッグツールの練習</h1>
    <script>
        let a = 10;
        let b = 20;
        let sum = a + b;
        console.log("合計: " + sum);
    </script>
</body>
</html>

ブレークポイントを設定する手順：

  1. 上記のHTMLファイルをブラウザで開く
  2. F12で開発者ツールを開き、「Sources」タブへ
  3. 左側のファイル一覧から、作成したHTMLファイルを選択
  4. コードエディタで「let sum = a + b;」の行番号をクリック
  5. 青い印がつけばブレークポイント設定完了
  6. ページをリロード（F5）すると、その行で実行が止まります

2.13 ステップ実行の基本操作

ブレークポイントで止まった状態から、プログラムを1行ずつ実行していきます。 右側のデバッグパネルに表示されるボタンを使います。

ボタン 名前 ショートカット 動作
▶️ (Resume) 再開 F8 次のブレークポイントまで実行を続ける
⤵️ (Step Over) ステップオーバー F10 次の行へ進む（関数は飛ばす）
⬇️ (Step Into) ステップイン F11 次の行へ進む（関数の中に入る）
⬆️ (Step Out) ステップアウト Shift + F11 現在の関数から抜ける
実習 2-12: ステップオーバーを使う

先ほどのHTMLファイルで、ブレークポイントで止まった状態から：

  1. F10キーを押す（またはステップオーバーボタンをクリック）
  2. 次の行「console.log("合計: " + sum);」に黄色いハイライトが移動する
  3. 右側の「Scope」セクションで、変数a、b、sumの値を確認できる
  4. もう一度F10を押すと、プログラムが終了する

2.14 変数の値を確認する

ステップ実行中は、変数の値をいくつかの方法で確認できます。

変数の値を確認する3つの方法
  1. Scopeパネル：右側の「Scope」セクションに全変数が表示される
  2. マウスオーバー：コード上の変数名にマウスを乗せると値が表示される
  3. Watchパネル：監視したい変数を登録して常に表示できる
実習 2-13: 変数を監視する（Watch）

より複雑なコードでWatchを使ってみましょう：

<!DOCTYPE html>
<html>
<head>
    <title>Watch機能の練習</title>
</head>
<body>
    <h1>Watch機能を使おう</h1>
    <script>
        let price = 1000;
        let quantity = 3;
        let tax = 0.1;

        let subtotal = price * quantity;  // ブレークポイントをここに設定
        let taxAmount = subtotal * tax;
        let total = subtotal + taxAmount;

        console.log("合計金額: " + total + "円");
    </script>
</body>
</html>

Watch機能を使う手順：

  1. 「let subtotal = price * quantity;」の行にブレークポイントを設定
  2. ページをリロードして、ブレークポイントで停止
  3. 右側の「Watch」セクションの「+」ボタンをクリック
  4. 「subtotal」と入力してEnter
  5. F10でステップ実行すると、subtotalの値が更新されるのが確認できる
  6. 同様に「taxAmount」「total」も追加して監視してみましょう

2.15 実践：買い物計算をデバッグ

これまでに作った買い物計算のコードを、デバッガーで1行ずつ確認してみましょう。

🎯 デバッグ実践課題

以下のコードをHTMLファイルに保存し、デバッガーで動きを確認しましょう：

<!DOCTYPE html>
<html>
<head>
    <title>買い物計算デバッグ</title>
</head>
<body>
    <h1>買い物計算をデバッグしよう</h1>
    <script>
        // 商品情報
        let itemName = "りんご";
        let price = 150;
        let quantity = 5;
        let tax = 0.1;

        // 計算処理
        let subtotal = price * quantity;  // ブレークポイント1
        let taxAmount = subtotal * tax;   // ブレークポイント2
        let total = subtotal + taxAmount; // ブレークポイント3

        // 結果表示
        console.log("=== 買い物計算 ===");
        console.log("商品名: " + itemName);
        console.log("単価: " + price + "円");
        console.log("数量: " + quantity + "個");
        console.log("小計: " + subtotal + "円");
        console.log("税額: " + taxAmount + "円");
        console.log("合計: " + total + "円");
    </script>
</body>
</html>

やってみよう：

  1. コメントで「ブレークポイント1」と書かれた行にブレークポイントを設定
  2. ページをリロードして実行を開始
  3. F10を使って1行ずつ実行
  4. 各行が実行された後、変数の値がどう変わるか確認
  5. Watch機能で「subtotal」「taxAmount」「total」を監視

確認すべきポイント：

  • ☑ subtotalが750（150 × 5）になっているか
  • ☑ taxAmountが75（750 × 0.1）になっているか
  • ☑ totalが825（750 + 75）になっているか
🔍 デバッグのコツ
  • 計算が合わない時：ブレークポイントを設定して、各変数の値を確認
  • エラーが出た時：エラーが出る直前の行にブレークポイントを設定
  • 複雑な処理：Watch機能を使って重要な変数を常に監視
  • 条件分岐や繰り返し：ステップ実行で処理の流れを追う（次章以降で学習）
