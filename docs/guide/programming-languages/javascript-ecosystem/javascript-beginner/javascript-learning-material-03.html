はじめてのプログラムを書こう

「Hello World!」という文字を画面に表示する最初のプログラムを作成し、プログラムの基本的な書き方と実行方法を学びます。

この章で学ぶこと

  • HTMLファイルにJavaScriptを書く方法を理解する
  • 「Hello World!」を表示する3つの方法を実践する
  • ブラウザで実行結果を確認する方法を身につける
  • 自分のコードを書いて動作させる楽しさを体験する

3.1 Hello Worldとは

プログラミングを学ぶとき、「Hello World!」という文字を画面に表示することから始めるのが伝統です。 これは、世界中のプログラマーが最初に書くプログラムで、「こんにちは、世界！」という意味です。

なぜ「Hello World」なのか？

「Hello World」プログラムは、環境が正しく動いているか確認するための、 一番シンプルなプログラムです。これが動けば、あなたの開発環境は正常に動作していることがわかります。 また、プログラムの基本的な書き方を学ぶのにも最適です。

JavaScriptでは、画面に文字を表示する方法がいくつかあります。 この章では、代表的な3つの方法を学びます。

3.2 HTMLファイルを作成しよう

JavaScriptのプログラムを動かすために、まずHTMLファイルを作成します。 HTMLは、Webページの土台となる言語です。

実習1: HTMLファイルを作成する

1

Visual Studio Codeを開きます

2

「ファイル」→「新しいファイル」をクリックします

3

「ファイル」→「名前を付けて保存」で、hello.html という名前で保存します

ファイルを作成したら、次のコードを入力してみましょう。 一文字ずつ丁寧に入力することが大切です。

hello.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>はじめてのJavaScript</title>
</head>
<body>
    <h1>JavaScriptを学ぼう</h1>

    <!-- ここにJavaScriptを書きます -->
    <script>
        // JavaScriptのコードをここに書きます
    </script>
</body>
</html>

コードの解説

  • <!DOCTYPE html> : これはHTMLファイルですよ、という宣言
  • <html lang="ja"> : 日本語のページですよ、という指定
  • <meta charset="UTF-8"> : 文字化けを防ぐおまじない
  • <script>タグ : この中にJavaScriptを書きます
  • <!-- --> : HTMLのコメント（メモ書き）
  • // : JavaScriptのコメント

3.3 console.logで表示しよう

console.log()は、開発者ツールのコンソール画面にメッセージを表示する命令です。 プログラムが正しく動いているか確認するときに、よく使います。

console.logの使い方 
<script>
    // コンソールに「Hello World!」と表示
    console.log("Hello World!");
</script>

実習2: console.logを実行してみよう

1

先ほど作った hello.html の <script> タグの中に、 console.log("Hello World!"); と書きます

2

ファイルを保存します（Ctrl+S または Cmd+S）

3

hello.html をダブルクリックして、ブラウザで開きます

4

ブラウザで F12キー を押して、開発者ツールを開きます

5

「Console（コンソール）」タブをクリックします

6

「Hello World!」というメッセージが表示されていれば成功です！

コンソールって何？

コンソールは、プログラマーがプログラムの動作を確認するための裏側の画面です。 普通のユーザーには見えませんが、開発者ツールを開くと見ることができます。 プログラムの間違い（エラー）もここに表示されるので、とても重要です。

3.4 alertで表示しよう

alert()は、ポップアップウィンドウでメッセージを表示する命令です。 画面の真ん中に小さな窓が出てきて、「OK」ボタンを押すまで他の操作ができなくなります。

alertの使い方 
<script>
    // ポップアップで「Hello World!」と表示
    alert("Hello World!");
</script>

実習3: alertを実行してみよう

1

hello.html の <script> タグの中に、 alert("こんにちは！"); と書きます

2

ファイルを保存して、ブラウザで開きます（またはリロードします）

3

ポップアップが表示されたら成功です！「OK」ボタンを押して閉じます

alertの注意点

alertは便利ですが、「OK」を押すまで画面がフリーズしてしまいます。 そのため、実際のWebサイトではあまり使われません。学習用や簡単な確認には便利です。

3.5 document.writeで表示しよう

document.write()は、Webページに直接文字を書き込む命令です。 画面に表示されるので、結果がすぐにわかります。

document.writeの使い方 
<script>
    // ページに「Hello World!」と表示
    document.write("Hello World!");
</script>

実習4: 3つの方法を組み合わせてみよう

ここまで学んだ3つの方法を全部使ってみましょう！

hello.html（完成版） 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>はじめてのJavaScript</title>
</head>
<body>
    <h1>JavaScriptを学ぼう</h1>

    <script>
        // 1. コンソールに表示
        console.log("コンソールに表示: Hello World!");

        // 2. ポップアップで表示
        alert("ポップアップ: こんにちは！");

        // 3. ページに表示
        document.write("<p>ページに表示: Hello World!</p>");
    </script>
</body>
</html>

動作確認のポイント

  • ページを開くとすぐに alert のポップアップが出る
  • 「OK」を押すと、ページに document.write で書いた文字が表示される
  • F12で開発者ツールを開き、コンソールタブで console.log のメッセージを確認

3.6 3つの方法の使い分け

ここまで、画面に文字を表示する3つの方法を学びました。 それぞれ使う場面が異なります。

console.log()

開発中の確認に使う

  • プログラムの途中経過を確認
  • エラーの原因を調べる
  • 一番よく使う方法

alert()

重要な通知に使う

  • ユーザーに確実に伝えたいとき
  • 簡単な確認・注意喚起
  • 学習用の動作確認

document.write()

学習用として使う

  • 結果をページに直接表示
  • 初心者の練習に便利
  • 実際のサイトではあまり使わない

実務では？

実際のWebサイト開発では、console.log()を一番よく使います。 プログラムの動作確認やエラー調査に欠かせません。 alert()は、ユーザー体験を損なうので避けられることが多いです。 document.write()は、今ではほとんど使われず、もっと安全で柔軟な方法（DOM操作）を使います。

3.7 よくあるエラーと対処法

プログラミングでは、エラーが出るのは当たり前です。 エラーメッセージを読んで、原因を見つける力が大切です。

トラブルシューティング

エラー1: 何も表示されない

原因：ファイルを保存し忘れている、またはブラウザをリロードしていない

対処法：

  • ファイルを保存（Ctrl+S または Cmd+S）
  • ブラウザで F5 を押してリロード

エラー2: 「Uncaught SyntaxError」と表示される

原因：書き方が間違っている（カッコやクォーテーションの閉じ忘れなど）

対処法：

  • console.log("テキスト") の " が閉じているか確認
  • () のカッコが両方あるか確認
  • 最後の ;（セミコロン）があるか確認

エラー3: 日本語が文字化けする

原因：文字コードの設定が正しくない

対処法：

  • <meta charset="UTF-8"> が <head> 内にあるか確認
  • ファイルを保存するときの文字コードを「UTF-8」にする

第3章のまとめ

  • JavaScriptは <script> タグの中に書く
  • console.log() は開発者ツールのコンソールに表示する（開発中の確認用）
  • alert() はポップアップで表示する（重要な通知用）
  • document.write() はページに直接表示する（学習用）
  • エラーが出たら、エラーメッセージを読んで原因を探すことが大切

理解度確認クイズ

  1. Q1. JavaScriptのコードを書くのは、HTMLのどのタグの中ですか？

    回答を表示

    <script> タグの中に書きます。

  2. Q2. 開発者ツールのコンソールにメッセージを表示するには、どのコマンドを使いますか？

    回答を表示

    console.log() を使います。
    例: console.log("メッセージ");

  3. Q3. ポップアップウィンドウでメッセージを表示する方法は何ですか？

    回答を表示

    alert() を使います。
    例: alert("こんにちは");

  4. Q4. 次のコードには間違いがあります。どこを直せばよいでしょうか？
    console.log("Hello World!

    回答を表示

    ダブルクォーテーション " が閉じられていません。また、最後にカッコ ) とセミコロン ; が必要です。
    正解: console.log("Hello World!");
