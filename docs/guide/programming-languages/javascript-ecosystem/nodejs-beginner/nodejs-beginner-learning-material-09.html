HTTPサーバーを作ってみよう

Node.jsの標準モジュールを使って、シンプルなWebサーバーを構築します。HTTPの基礎知識も合わせて学習します。

この章で学ぶこと

  • HTTPプロトコルの基礎知識を理解する
  • Node.jsのhttpモジュールを使ってWebサーバーを構築できるようになる
  • リクエストとレスポンスの処理方法を習得する
  • 静的ファイルの配信と簡易ルーティングを実装できるようになる

9.1 HTTPの基礎知識

Webサーバーを構築する前に、まずHTTPの基本を理解しましょう。HTTP（HyperText Transfer Protocol）は、WebブラウザとWebサーバーがデータをやり取りするための約束事（プロトコル）です。

わかりやすい例え

HTTPは、レストランでの注文に似ています。お客様（ブラウザ）が注文（リクエスト）を出すと、店員（サーバー）が料理（レスポンス）を提供します。この注文と提供のやり取りを規定したルールがHTTPです。

HTTPリクエストとレスポンス

sequenceDiagram participant Browser as ブラウザ全文クライアント全文 participant Server as Webサーバー Browser->>Server: HTTPリクエスト全文例: GET /index.html全文 Note over Server: ファイルを読み込む Server-->>Browser: HTTPレスポンス全文HTMLコンテンツ全文 Note over Browser: HTMLを表示

主なHTTPメソッド

HTTPメソッドは、サーバーに対して何をしたいのかを伝えるための動詞のようなものです。

メソッド 意味 用途
GET 取得 ページやデータの取得
POST 送信 フォームデータの送信
PUT 更新 データの更新
DELETE 削除 データの削除

HTTPステータスコード

ステータスコードは、リクエストの処理結果をクライアントに伝える3桁の数字です。

2xx 成功

  • 200 OK: 成功
  • 201 Created: 作成成功

3xx リダイレクト

  • 301: 永続的な移動
  • 302: 一時的な移動

4xx クライアントエラー

  • 400 Bad Request: 不正なリクエスト
  • 404 Not Found: 見つからない

5xx サーバーエラー

  • 500: サーバー内部エラー
  • 503: サービス利用不可

9.2 はじめてのHTTPサーバー

Node.jsのhttpモジュールを使って、最もシンプルなWebサーバーを構築してみましょう。

basic-server.js 
// httpモジュールを読み込む
const http = require('http');

// サーバーを作成
const server = http.createServer((req, res) => {
  // レスポンスヘッダーを設定
  res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });

  // レスポンスボディを送信
  res.end('<h1>Hello, Node.js Server!</h1>');
});

// ポート3000でリクエストを待ち受ける
server.listen(3000, () => {
  console.log('サーバーが起動しました: http://localhost:3000');
});

実習 9-1: 基本的なHTTPサーバーを起動

上記のコードを実行して、ブラウザで確認してみましょう。

手順

  1. 1 上記のコードを basic-server.js として保存
  2. 2 ターミナルで node basic-server.js を実行
  3. 3 ブラウザで http://localhost:3000 にアクセス
  4. 4 サーバーを停止するには、ターミナルで Ctrl + C

コードの解説

1. httpモジュールの読み込み

const http = require('http') で、Node.js標準のhttpモジュールを使えるようにします。

2. サーバーの作成

http.createServer() は、リクエストを受け取るたびに実行されるコールバック関数を登録します。

  • req (request): リクエスト情報（URL、メソッドなど）
  • res (response): レスポンスを作成するオブジェクト

3. レスポンスの送信

res.writeHead() でステータスコードとヘッダーを設定し、res.end() で本文を送信します。

4. ポートの待ち受け

server.listen(3000) で、ポート3000番でリクエストを待ち受けます。

ポート番号とは？

ポート番号は、コンピュータ上で複数のサービスを区別するための番号です。Webサーバーは通常80番（HTTP）や443番（HTTPS）を使いますが、開発時は3000番や8080番などを使うのが一般的です。

9.3 リクエストとレスポンスの処理

リクエストからURLやメソッドを取得し、それに応じた処理を行う方法を学びましょう。

request-handler.js 
const http = require('http');

const server = http.createServer((req, res) => {
  // リクエスト情報を取得
  const method = req.method;  // HTTPメソッド
  const url = req.url;        // リクエストURL

  // リクエスト情報をログ出力
  console.log(`${method} ${url}`);

  // レスポンスを作成
  res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });
  res.end(`
    <h1>リクエスト情報</h1>
    <p>メソッド: ${method}</p>
    <p>URL: ${url}</p>
  `);
});

server.listen(3000, () => {
  console.log('サーバー起動: http://localhost:3000');
});

URLのパース

URLからパス名やクエリパラメータを取り出すには、Node.js標準のurlモジュールを使います。

url-parse.js 
const http = require('http');
const url = require('url');

const server = http.createServer((req, res) => {
  // URLをパース
  const parsedUrl = url.parse(req.url, true);
  const pathname = parsedUrl.pathname;     // /users
  const query = parsedUrl.query;           // { id: '123' }

  console.log('パス名:', pathname);
  console.log('クエリ:', query);

  res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });
  res.end(`
    <h1>URL解析結果</h1>
    <p>パス: ${pathname}</p>
    <p>クエリ: ${JSON.stringify(query)}</p>
  `);
});

server.listen(3000);

重要ポイント

url.parse(req.url, true) の第2引数に true を指定すると、クエリパラメータがオブジェクトとして自動的にパースされます。

動作確認

サーバーを起動して、以下のURLにアクセスしてみましょう：

  • http://localhost:3000/users?id=123&name=taro
  • http://localhost:3000/products?category=books

期待される結果 

パス名: /users
クエリ: { id: '123', name: 'taro' }

9.4 静的ファイルの配信

HTMLファイルやCSSファイルなどの静的ファイルをサーバーから配信する方法を学びましょう。

プロジェクト構成 

myserver/
├── server.js
└── public/
    ├── index.html
    └── style.css
public/index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>Node.js Server</title>
  <link rel="stylesheet" href="/style.css">
</head>
<body>
  <h1>Welcome to Node.js Server!</h1>
  <p>これは静的HTMLファイルです。</p>
</body>
</html>
server.js（静的ファイル配信） 
const http = require('http');
const fs = require('fs');
const path = require('path');

const server = http.createServer((req, res) => {
  // ルートパスは index.html に変換
  let filePath = req.url === '/' ? '/index.html' : req.url;

  // publicディレクトリ内のファイルパスを構築
  filePath = path.join(__dirname, 'public', filePath);

  // ファイルを非同期で読み込む
  fs.readFile(filePath, (err, data) => {
    if (err) {
      // ファイルが見つからない場合
      res.writeHead(404, { 'Content-Type': 'text/html; charset=utf-8' });
      res.end('<h1>404 Not Found</h1>');
    } else {
      // ファイルの拡張子からContent-Typeを判定
      const ext = path.extname(filePath);
      const contentType = getContentType(ext);

      res.writeHead(200, { 'Content-Type': contentType });
      res.end(data);
    }
  });
});

// Content-Type判定関数
function getContentType(ext) {
  switch (ext) {
    case '.html': return 'text/html; charset=utf-8';
    case '.css':  return 'text/css';
    case '.js':   return 'application/javascript';
    case '.json': return 'application/json';
    default:      return 'text/plain';
  }
}

server.listen(3000, () => {
  console.log('サーバー起動: http://localhost:3000');
});

Content-Typeとは？

Content-Typeは、送信するデータの種類をブラウザに伝えるHTTPヘッダーです。これが正しく設定されていないと、ブラウザが適切にファイルを処理できません。

  • text/html: HTMLファイル
  • text/css: CSSファイル
  • application/javascript: JavaScriptファイル

9.5 簡易ルーティングの実装

URLに応じて異なるレスポンスを返す、簡易的なルーティング機能を実装してみましょう。

routing-server.js 
const http = require('http');
const url = require('url');

const server = http.createServer((req, res) => {
  const parsedUrl = url.parse(req.url, true);
  const pathname = parsedUrl.pathname;
  const method = req.method;

  // ルーティング処理
  if (pathname === '/' && method === 'GET') {
    // トップページ
    res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });
    res.end('<h1>ホームページ</h1><p>ようこそ！</p>');

  } else if (pathname === '/about' && method === 'GET') {
    // aboutページ
    res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });
    res.end('<h1>Aboutページ</h1><p>このサイトについて</p>');

  } else if (pathname === '/api/users' && method === 'GET') {
    // API: ユーザー一覧
    const users = [
      { id: 1, name: '太郎' },
      { id: 2, name: '花子' }
    ];
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify(users));

  } else {
    // 404 Not Found
    res.writeHead(404, { 'Content-Type': 'text/html; charset=utf-8' });
    res.end('<h1>404 Not Found</h1><p>ページが見つかりません</p>');
  }
});

server.listen(3000, () => {
  console.log('サーバー起動: http://localhost:3000');
});

トラブルシューティング

Error: listen EADDRINUSE :::3000

原因: ポート3000番が既に使用中です。前回のサーバープロセスが残っているか、別のアプリが使用しています。

解決方法:

  • Ctrl+C でサーバーを停止してから再起動
  • 別のポート番号（例: 3001, 8080）を使用する
  • プロセスを強制終了: pkill node（Windowsは taskkill /f /im node.exe）
ブラウザに「接続できません」と表示される

原因: サーバーが起動していないか、別のポートで起動しています。

解決方法:

  • ターミナルで「サーバー起動」メッセージが出ているか確認
  • URLのポート番号が正しいか確認（例: http://localhost:3000）
  • node server.js でサーバーを起動
CSSが適用されない

原因: Content-Typeが正しく設定されていない、またはファイルパスが間違っています。

解決方法:

  • ブラウザの開発者ツール（F12）のネットワークタブで、CSSファイルが読み込まれているか確認
  • Content-Typeが text/css になっているか確認
  • CSSファイルのパスが正しいか確認（例: /style.css）

理解度確認クイズ

  1. Q1. HTTPステータスコードの「404」と「500」の違いは何ですか？

    回答を表示

    404 Not Foundは、クライアントが存在しないページやリソースをリクエストしたときに返されるエラーです（クライアント側の問題）。一方、500 Internal Server Errorは、サーバー側でプログラムのエラーが発生したときに返されるエラーです（サーバー側の問題）。

  2. Q2. http.createServer() のコールバック関数の2つの引数（reqとres）は、それぞれ何を表しますか？

    回答を表示

    req（request）は、クライアントからのリクエスト情報を含むオブジェクトです（URL、メソッド、ヘッダーなど）。res（response）は、クライアントに送り返すレスポンスを作成するためのオブジェクトです（ステータスコード、ヘッダー、本文を設定する）。

  3. Q3. Content-Typeヘッダーはなぜ重要ですか？具体例を挙げて説明してください。

    回答を表示

    Content-Typeヘッダーは、送信するデータの種類をブラウザに伝えるために重要です。例えば、CSSファイルを送る際に Content-Type: text/html と設定してしまうと、ブラウザはそれをHTMLとして解釈してしまい、スタイルが適用されません。正しく Content-Type: text/css を設定することで、ブラウザが適切に処理できます。
