モジュールシステムを理解しよう

Node.jsのモジュールシステム（CommonJSとESモジュール）を学びます。コードを分割・再利用する方法を、実践的な例を通じて理解します。

この章で学ぶこと

  • モジュールとは何か、なぜ必要なのかを理解する
  • CommonJSモジュール（require/module.exports）の書き方を学ぶ
  • ESモジュール（import/export）の書き方を学ぶ
  • 2つのモジュールシステムの違いと使い分けを理解する

4.1 モジュールとは何か

モジュールとは、機能ごとにコードを分割して整理する仕組みです。大きなプログラムを1つのファイルに書くと読みにくく、メンテナンスが難しくなります。モジュールを使うことで、コードを再利用しやすく、管理しやすくなります。

わかりやすい例え

レゴブロックを想像してください。大きな作品を作るとき、全てを一体として作るのではなく、パーツ（モジュール）に分けて作り、後から組み合わせます。プログラムも同じで、機能ごとにモジュール化することで、組み合わせて大きなアプリケーションを作れます。

モジュールのメリット

再利用性

一度書いたコードを他のプログラムでも使える

名前空間の分離

変数名の衝突を避けられる

可読性

機能ごとにファイルが分かれて読みやすい

保守性

修正や追加がしやすくなる

4.2 CommonJSモジュール

CommonJSは、Node.jsで最初に採用されたモジュールシステムです。require() で読み込み、module.exports でエクスポートします。

基本的な使い方

mathUtils.js（モジュール側） 
// 足し算関数
function add(a, b) {
    return a + b;
}

// 引き算関数
function subtract(a, b) {
    return a - b;
}

// 関数をエクスポート
module.exports = {
    add: add,
    subtract: subtract
};
app.js（使う側） 
// モジュールを読み込む
const mathUtils = require('./mathUtils');

// 関数を使う
console.log(mathUtils.add(5, 3));      // 8
console.log(mathUtils.subtract(10, 4)); // 6

重要ポイント

require() でモジュールを読み込むとき、相対パス（./ や ../）を忘れないようにしましょう。拡張子 .js は省略できます。

エクスポートの方法

方法1: オブジェクトでまとめてエクスポート 
// 複数の関数をまとめてエクスポート
module.exports = {
    greet: function(name) {
        return `こんにちは、${name}さん！`;
    },
    farewell: function(name) {
        return `さようなら、${name}さん！`;
    }
};
方法2: 1つの関数だけエクスポート 
// 1つの関数をそのままエクスポート
module.exports = function(message) {
    console.log('メッセージ: ' + message);
};

// 使う側
const showMessage = require('./showMessage');
showMessage('テスト'); // メッセージ: テスト

4.3 ESモジュール（ES6 Modules）

ESモジュールは、JavaScriptの標準仕様（ECMAScript）で定義されたモジュールシステムです。import で読み込み、export でエクスポートします。

Node.jsでESモジュールを使うには

Node.jsでESモジュールを使うには、以下のいずれかの方法が必要です：
1. package.json に "type": "module" を追加
2. ファイル拡張子を .mjs にする

基本的な使い方

mathUtils.js（モジュール側） 
// 名前付きエクスポート
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

// 定数もエクスポートできる
export const PI = 3.14159;
app.js（使う側） 
// 必要な関数だけをインポート
import { add, subtract } from './mathUtils.js';

console.log(add(5, 3));      // 8
console.log(subtract(10, 4)); // 6

// すべてをインポート
import * as math from './mathUtils.js';
console.log(math.add(5, 3));  // 8
console.log(math.PI);         // 3.14159

デフォルトエクスポート

logger.js 
// デフォルトエクスポート（モジュールから1つだけ）
export default function log(message) {
    console.log(`[LOG] ${message}`);
}
app.js 
// デフォルトエクスポートは好きな名前でインポートできる
import logger from './logger.js';

logger('アプリケーション起動'); // [LOG] アプリケーション起動

4.4 CommonJSとESモジュールの違い

主な違い

項目 CommonJS ESモジュール
読み込み require() import
エクスポート module.exports export
読み込みタイミング 実行時（動的） コンパイル時（静的）
ファイル拡張子 .js .js (type: module) または .mjs
ブラウザ対応 非対応（Bundlerが必要） モダンブラウザで対応

モジュール読み込みの流れ

flowchart LR subgraph CommonJS["CommonJS（動的）"] C1["コード実行中に
require呼び出し"] C2["その場でファイル読込"] C3["実行して値を返す"] C1 --> C2 --> C3 end subgraph ESM["ESモジュール（静的）"] E1["コード解析時に
import検出"] E2["実行前に全モジュール
を読み込み"] E3["実行開始"] E1 --> E2 --> E3 end

どちらを使えばいい？

新しいプロジェクトでは、ESモジュールの使用が推奨されます。ただし、既存のNode.jsプロジェクトやnpmパッケージの多くはまだCommonJSを使っているため、両方を理解しておくことが重要です。

4.5 実践例：ユーティリティモジュール作成

実習 4-1: 文字列操作モジュールを作ろう

文字列を操作する便利な関数をまとめたモジュールを作成してみましょう。

手順

  1. 1 stringUtils.js を作成し、文字列操作関数を定義する
  2. 2 app.js から読み込んで使ってみる
stringUtils.js（CommonJS版） 
// 文字列を大文字に変換
function toUpperCase(str) {
    return str.toUpperCase();
}

// 文字列を逆順にする
function reverse(str) {
    return str.split('').reverse().join('');
}

// 文字数をカウント
function countChars(str) {
    return str.length;
}

// エクスポート
module.exports = {
    toUpperCase,
    reverse,
    countChars
};
app.js 
// モジュールを読み込む
const strUtils = require('./stringUtils');

// 使用例
const text = 'Hello Node.js';

console.log(strUtils.toUpperCase(text)); // HELLO NODE.JS
console.log(strUtils.reverse(text));     // sj.edoN olleH
console.log(strUtils.countChars(text));  // 13

動作確認

ターミナルで以下のコマンドを実行してください。

node app.js

期待される結果 

HELLO NODE.JS
sj.edoN olleH
13

トラブルシューティング

Error: Cannot find module './xxx'

原因: ファイルパスが間違っているか、ファイルが存在しません。

解決方法: 相対パス（./）を正しく指定しているか、ファイル名のスペルミスがないか確認してください。

SyntaxError: Cannot use import statement outside a module

原因: ESモジュールの設定をしていないのに import を使おうとしています。

解決方法: package.json に "type": "module" を追加するか、ファイル拡張子を .mjs に変更してください。

TypeError: xxx is not a function

原因: エクスポートした関数名と、インポート側で使っている名前が一致していません。

解決方法: エクスポートとインポートで同じ名前を使っているか確認してください。デストラクチャリング（{ add }）の書き方も確認しましょう。

理解度確認クイズ

  1. Q1. モジュールを使う主なメリットを3つ挙げてください。

    回答を表示
    • 再利用性: 一度書いたコードを他のプログラムでも使える
    • 可読性: 機能ごとにファイルが分かれて読みやすい
    • 保守性: 修正や追加がしやすくなる

  2. Q2. CommonJSで複数の関数をエクスポートする方法を書いてください。

    回答を表示 
    module.exports = {
    functionA: functionA,
    functionB: functionB
};

  3. Q3. ESモジュールとCommonJSの主な違いを2つ説明してください。

    回答を表示
    • 構文: CommonJSはrequire/module.exports、ESモジュールはimport/export
    • 読み込みタイミング: CommonJSは実行時（動的）、ESモジュールはコンパイル時（静的）に読み込まれる

  4. Q4. Node.jsでESモジュールを使うための方法を2つ挙げてください。

    回答を表示
    • package.json に "type": "module" を追加する
    • ファイル拡張子を .mjs にする

  • モジュールはコードを機能ごとに分割して整理する仕組み
  • CommonJSはrequireとmodule.exportsを使う
  • ESモジュールはimportとexportを使う
  • CommonJSは動的、ESモジュールは静的に読み込まれる
  • 新しいプロジェクトではESモジュールの使用が推奨される
