Expressフレームワーク入門

人気のWebフレームワークExpressを使って、効率的にWebアプリケーションを開発する方法を学びます。ルーティングとミドルウェアの基礎を習得します。

この章で学ぶこと

  • Expressフレームワークとは何か、なぜ使われるのかを理解する
  • Expressのインストールと基本的なサーバー構築方法を習得する
  • ルーティングの概念と基本的な使い方をマスターする
  • ミドルウェアの仕組みと活用方法を理解する
  • 静的ファイルの配信とテンプレートエンジンの基礎を学ぶ

10.1 Expressとは

Expressは、Node.jsで最も人気のあるWebアプリケーションフレームワークです。前章で学んだ標準のhttpモジュールよりも、シンプルで分かりやすいコードでWebサーバーを構築できます。

わかりやすい例え

Webサーバー構築を「料理」に例えると、httpモジュールは「素材から全て手作り」、Expressは「便利な調理器具や半調理食材を使う」ようなものです。同じ料理（Webサーバー）を作るにしても、Expressを使えばずっと簡単に、早く、美味しく作れます。

Expressの特徴

シンプル・軽量

最小限の機能のみを提供し、必要に応じてミドルウェアで拡張できます

強力なルーティング

URLパターンに応じた柔軟なリクエスト処理が簡単に書けます

豊富なミドルウェア

認証、ログ、セッション管理など、多くの機能を追加できます

活発なコミュニティ

豊富なドキュメント、チュートリアル、サードパーティ製パッケージ

10.2 Expressのインストールと基本構成

Expressのインストール

Expressはnpmパッケージとして提供されているので、npm installで簡単にインストールできます。

Terminal 
# プロジェクトディレクトリを作成
mkdir my-express-app
cd my-express-app

# package.jsonを初期化
npm init -y

# Expressをインストール
npm install express

最もシンプルなExpressサーバー

以下は、わずか数行で動作するExpressサーバーです。前章のhttpモジュールと比べて、非常にシンプルになっています。

app.js 
// Expressをインポート
const express = require('express');

// Expressアプリケーションを作成
const app = express();

// ルート（/）へのGETリクエストに応答
app.get('/', (req, res) => {
  res.send('Hello Express!');
});

// サーバーを起動（ポート3000で待機）
app.listen(3000, () => {
  console.log('サーバーが起動しました: http://localhost:3000');
});

重要ポイント

express() を呼び出すとExpressアプリケーションのインスタンスが作成されます。このインスタンス（app）を使って、ルーティングやミドルウェアを設定します。

動作確認

ターミナルで node app.js を実行し、ブラウザで http://localhost:3000 にアクセスしてください。

期待される結果 

ブラウザに「Hello Express!」と表示されます

10.3 ルーティングの基礎

ルーティングとは、URLパス（ルート）に応じて異なる処理を実行する仕組みです。レストランのメニューのようなもので、「この注文にはこの料理を出す」というルールを定義します。

基本的なルーティング

routes.js 
const express = require('express');
const app = express();

// トップページ
app.get('/', (req, res) => {
  res.send('トップページ');
});

// aboutページ
app.get('/about', (req, res) => {
  res.send('このサイトについて');
});

// contactページ
app.get('/contact', (req, res) => {
  res.send('お問い合わせ');
});

app.listen(3000);

HTTPメソッド別のルーティング

Expressでは、GET、POST、PUT、DELETEなど、HTTPメソッドごとに異なるルーティングを定義できます。

http-methods.js 
// GETリクエスト: データの取得
app.get('/users', (req, res) => {
  res.send('ユーザー一覧を取得');
});

// POSTリクエスト: データの作成
app.post('/users', (req, res) => {
  res.send('新しいユーザーを作成');
});

// PUTリクエスト: データの更新
app.put('/users/123', (req, res) => {
  res.send('ユーザー123を更新');
});

// DELETEリクエスト: データの削除
app.delete('/users/123', (req, res) => {
  res.send('ユーザー123を削除');
});

ルートパラメータ

URLの一部を変数として受け取ることができます。例えば、/users/123の「123」をユーザーIDとして取得できます。

params.js 
// :idがルートパラメータ
app.get('/users/:id', (req, res) => {
  const userId = req.params.id;  // URLからidを取得
  res.send(`ユーザーID: ${userId} の情報`);
});

// 複数のパラメータ
app.get('/posts/:year/:month', (req, res) => {
  const { year, month } = req.params;
  res.send(`${year}年${month}月の投稿一覧`);
});

10.4 ミドルウェアとは

ミドルウェアとは、リクエストとレスポンスの間で実行される処理です。工場の組み立てラインのように、複数のミドルウェアを順番に通過させることができます。

わかりやすい例え

空港のセキュリティチェックを想像してください。乗客（リクエスト）は、荷物検査 → 金属探知機 → パスポート確認と、複数のチェックポイント（ミドルウェア）を通過して搭乗ゲート（最終処理）に到達します。各チェックポイントで問題があれば、そこで止まります（エラー処理）。

全文ログ記録全文"] M1 --> M2["ミドルウェア2
全文認証チェック全文"] M2 --> M3["ミドルウェア3
全文データ解析全文"] M3 --> Route["ルートハンドラ
全文最終処理全文"] Route --> Res["レスポンス"]

基本的なミドルウェア

middleware.js 
const express = require('express');
const app = express();

// すべてのリクエストで実行されるミドルウェア
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);  // ログ出力
  next();  // 次の処理へ
});

// ルート定義
app.get('/', (req, res) => {
  res.send('トップページ');
});

app.listen(3000);

重要ポイント

next() を呼び出さないと、次のミドルウェアやルートハンドラに処理が進みません。必ず next() を呼ぶか、res.send() でレスポンスを返してください。

よく使われるミドルウェア

common-middleware.js 
// JSONリクエストボディを解析
app.use(express.json());

// URLエンコードされたデータを解析（フォーム送信）
app.use(express.urlencoded({ extended: true }));

// POSTリクエストでJSONを受け取る例
app.post('/api/data', (req, res) => {
  console.log(req.body);  // JSONデータが自動で解析される
  res.json({ message: '受信しました', data: req.body });
});

10.5 静的ファイルの配信

画像、CSS、JavaScriptファイルなどの静的ファイルを配信するには、express.staticミドルウェアを使います。

static-files.js 
const express = require('express');
const app = express();

// publicフォルダ内のファイルを静的配信
app.use(express.static('public'));

// 例: public/style.css → http://localhost:3000/style.css
// 例: public/images/logo.png → http://localhost:3000/images/logo.png

app.listen(3000);

推奨フォルダ構成

    ├── style.css
    └── images/
        └── logo.png

10.6 テンプレートエンジン（EJS）の基礎

テンプレートエンジンを使うと、HTMLの中にJavaScriptの変数や処理を埋め込んで、動的なページを生成できます。ここでは、人気のEJS（Embedded JavaScript）を紹介します。

EJSのインストールと設定

Terminal 
# EJSをインストール
npm install ejs
app.js 
const express = require('express');
const app = express();

// テンプレートエンジンとしてEJSを設定
app.set('view engine', 'ejs');

// viewsフォルダにテンプレートファイルを配置
app.set('views', './views');

// ルート定義
app.get('/', (req, res) => {
  // テンプレートにデータを渡してレンダリング
  res.render('index', { title: 'ホーム', name: '太郎' });
});

app.listen(3000);

EJSテンプレートファイル

views/index.ejs 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title><%= title %></title>
</head>
<body>
  <h1>こんにちは、<%= name %>さん！</h1>
  <p>現在時刻: <%= new Date().toLocaleString() %></p>
</body>
</html>

EJSの基本構文

  • <%= 変数 %> - 変数の値を出力（HTMLエスケープあり）
  • <%- 変数 %> - 変数の値を出力（HTMLエスケープなし）
  • <% コード %> - JavaScriptコードを実行（出力なし）

実習 10-1: シンプルなExpressアプリを作ろう

これまで学んだ内容を使って、複数ページを持つExpressアプリケーションを作成しましょう。

手順

  1. 1 新しいフォルダを作成し、npm init -yとnpm install expressを実行
  2. 2 app.jsを作成し、/、/about、/contactの3つのルートを定義
  3. 3 すべてのリクエストをログに記録するミドルウェアを追加
  4. 4 サーバーを起動してブラウザで動作確認

理解度確認クイズ

  1. Q1. Expressフレームワークを使う主なメリットは何ですか？3つ挙げてください。

    回答を表示
    • シンプルで読みやすいコードでWebサーバーを構築できる
    • 強力なルーティング機能により、URLパターンごとの処理が簡単に書ける
    • 豊富なミドルウェアエコシステムで機能拡張が容易

  2. Q2. ミドルウェアの役割を説明し、next()の重要性を述べてください。

    回答を表示

    ミドルウェアは、リクエストとレスポンスの間で実行される処理です。ログ記録、認証、データ解析などの共通処理を行います。next()を呼び出すことで次のミドルウェアやルートハンドラに処理を渡します。next()を呼ばないと、処理がそこで止まりレスポンスが返らなくなります。

  3. Q3. ルートパラメータとは何ですか？具体例を挙げて説明してください。

    回答を表示

    ルートパラメータは、URLの一部を変数として受け取る仕組みです。例えば、app.get('/users/:id', ...)と定義すると、/users/123へのアクセス時にreq.params.idで「123」を取得できます。これにより、動的なURLパターンを扱えます。

  • ミドルウェアを使って、共通処理を効率的に実装できる
  • express.staticで静的ファイルを簡単に配信できる
  • テンプレートエンジン（EJS）で動的なHTMLページを生成できる
