Node.jsとは何か

Node.jsとは何か、ブラウザのJavaScriptとの違いは何かを、身近な例を使って分かりやすく説明します。サーバーサイドJavaScriptの世界への第一歩を踏み出しましょう。

この章で学ぶこと

  • Node.jsとは何か、どのような技術なのかを理解する
  • ブラウザのJavaScriptとNode.jsの違いを説明できるようになる
  • Node.jsの特徴（イベント駆動、ノンブロッキングI/O）を理解する
  • Node.jsが得意なこと・苦手なことを把握する

1.1 Node.jsとは

Node.js（ノードジェイエス）は、JavaScriptをサーバー側（バックエンド）で実行するためのプラットフォームです。これまでJavaScriptはブラウザ上でのみ動作する言語でしたが、Node.jsの登場により、サーバーサイドでもJavaScriptを使った開発が可能になりました。

わかりやすい例え

レストランで例えると、従来のJavaScriptは「フロア担当（お客様対応）」だけを担当していました。Node.jsは、そのJavaScriptが「キッチン（裏方）」でも働けるようにした技術です。つまり、同じ言語でフロントエンド（画面）もバックエンド（サーバー）も書けるようになったのです。

1.2 ブラウザJavaScriptとNode.jsの違い

同じJavaScriptでも、ブラウザで動くJavaScriptとNode.jsには重要な違いがあります。

主な違い

項目 ブラウザ JavaScript Node.js
実行環境 ブラウザ（Chrome、Firefoxなど） サーバー、コマンドライン
できること DOM操作、画面の動的変更 ファイル操作、サーバー構築
グローバルオブジェクト window global
用途 ユーザーインターフェース API、ツール、サーバー

重要ポイント

ブラウザJavaScriptには document や window がありますが、Node.jsにはありません。逆に、Node.jsには fs（ファイル操作）や http（サーバー構築）などのモジュールがあります。

1.3 Node.jsの特徴

V8エンジン

Node.jsは、Google Chromeと同じV8 JavaScriptエンジンを使用しています。V8は非常に高速で、JavaScriptコードを機械語に変換して実行するため、パフォーマンスが優れています。

イベント駆動・ノンブロッキングI/O

Node.jsの最大の特徴は、イベント駆動とノンブロッキングI/Oというアーキテクチャです。

ノンブロッキングI/Oとは？

レストランで例えると、従来の方式（ブロッキング）は、1つの注文の料理が完成するまで次の注文を受け付けません。一方、Node.js（ノンブロッキング）は、調理中でも次々と注文を受け付け、完成した順に提供します。

つまり、処理の完了を待たずに次の処理を開始できるため、多数の同時接続を効率的に処理できます。

シングルスレッド

Node.jsはシングルスレッドで動作します。これは「1つの作業者（スレッド）がすべてのリクエストを処理する」という意味です。しかし、非同期処理により、その1つのスレッドで多数のリクエストを並行して処理できます。

1.4 Node.jsが得意なこと・苦手なこと

得意なこと

  • リアルタイム通信：チャット、通知システム
  • REST API：モバイルアプリのバックエンド
  • I/O集約型処理：ファイル操作、データベース問い合わせ
  • マイクロサービス：軽量なサービス構築
  • ストリーミング：動画配信、大容量ファイル処理

苦手なこと

  • CPU集約型処理：画像処理、動画エンコード、暗号化
  • 複雑な計算：科学技術計算、機械学習
  • メモリ集約型処理：大量データを一度にメモリに載せる処理

シングルスレッドのため、重い計算でメインスレッドがブロックされると、他のリクエスト処理が遅延します。

理解度確認クイズ

  1. Q1. Node.jsとブラウザのJavaScriptの最大の違いは何ですか？

    回答を表示

    実行環境と用途が異なります。ブラウザJavaScriptはブラウザ上でDOM操作やUI制御を行いますが、Node.jsはサーバー上でファイル操作やサーバー構築を行います。また、ブラウザには window や document がありますが、Node.jsには fs や http などのモジュールがあります。

  2. Q2. Node.jsの「ノンブロッキングI/O」とは何ですか？具体例で説明してください。

    回答を表示

    処理の完了を待たずに次の処理を開始できる仕組みです。例えば、データベースへの問い合わせを開始したら、その結果を待たずに次のリクエスト処理を開始できます。レストランの例では、料理の調理中でも次々と注文を受け付けられるようなものです。これにより、多数の同時接続を効率的に処理できます。

  3. Q3. Node.jsが得意なことと苦手なことをそれぞれ2つずつ挙げてください。

    回答を表示

    得意なこと:

    • リアルタイム通信（チャット、通知システム）
    • REST API開発やI/O集約型処理

    苦手なこと:

    • CPU集約型処理（画像処理、動画エンコード）
    • 複雑な計算（科学技術計算、機械学習）

この章のまとめ

  • ブラウザJavaScriptとは実行環境と用途が異なる
  • V8エンジンを使用し、イベント駆動・ノンブロッキングI/Oが特徴
  • リアルタイム通信やI/O集約型処理が得意、CPU集約型処理は苦手
  • Netflix、PayPal、LinkedInなど大手企業でも使用されている
