Node.js環境のセットアップ

Node.jsのインストール方法をOS別に丁寧に解説します。nvmを使ったバージョン管理や、VS Codeの便利な拡張機能も紹介します。

この章で学ぶこと

  • OS別のNode.jsインストール方法を理解する
  • nvmを使ったバージョン管理の方法を習得する
  • Node.jsとnpmのインストール確認方法を学ぶ
  • VS Codeの便利な拡張機能をセットアップできる

2.1 インストール方法の選択肢

Node.jsをインストールする方法は大きく分けて2つあります。それぞれの方法には特徴があるため、自分に合った方法を選びましょう。

推奨事項

実務ではnvm（Node Version Manager）の使用が推奨されます。プロジェクトごとに異なるNode.jsバージョンを簡単に切り替えられるため、開発現場では必須ツールとなっています。

2.2 公式インストーラーによるインストール

まずは最もシンプルな方法から見ていきましょう。公式サイトからインストーラーをダウンロードしてインストールする方法です。

どのバージョンを選ぶべきか

Node.js公式サイトには2つのバージョンが表示されています：

LTS（Long Term Support）

長期サポート版

  • 安定性重視
  • 本番環境向け
  • 初心者に推奨

Current

最新機能版

  • 最新機能を試せる
  • 実験的な機能も含む
  • 不具合の可能性あり

インストール手順

Windows

  1. Node.js公式サイトにアクセス
  2. 「LTS（推奨版）」のインストーラーをダウンロード
  3. ダウンロードした .msi ファイルを実行
  4. インストールウィザードに従って進める（デフォルト設定でOK）
  5. 「Install」をクリックしてインストール完了

macOS

  1. Node.js公式サイトにアクセス
  2. 「LTS（推奨版）」のインストーラーをダウンロード
  3. ダウンロードした .pkg ファイルを実行
  4. インストールウィザードに従って進める
  5. 管理者パスワードを入力してインストール完了

Linux（Ubuntu/Debian）

パッケージマネージャーを使ってインストールします：

ターミナル 
# NodeSource リポジトリを追加
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -

# Node.js をインストール
sudo apt-get install -y nodejs

2.3 nvmを使ったインストール（推奨）

nvm（Node Version Manager）は、複数のNode.jsバージョンを管理できるツールです。プロジェクトごとに異なるバージョンを簡単に切り替えられるため、実務では必須のツールです。

nvmのインストール

Windows：nvm-windows

  1. nvm-windowsのリリースページにアクセス
  2. 最新版の nvm-setup.exe をダウンロード
  3. ダウンロードしたファイルを実行してインストール
  4. コマンドプロンプトを管理者権限で開く
  5. インストール確認：nvm version

macOS / Linux

ターミナルで以下のコマンドを実行：

ターミナル 
# nvmインストールスクリプトを実行
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# シェル設定を再読み込み
source ~/.bashrc  # bashの場合
# または
source ~/.zshrc   # zshの場合

nvmを使ってNode.jsをインストール

nvmがインストールできたら、Node.jsをインストールしましょう。

基本的なnvmコマンド 
# 利用可能なNode.jsバージョンを確認
nvm list available

# 最新のLTS版をインストール
nvm install --lts

# 特定のバージョンをインストール（例：v20.10.0）
nvm install 20.10.0

# インストール済みバージョンを一覧表示
nvm list

# 使用するバージョンを切り替え
nvm use 20.10.0

# デフォルトバージョンを設定
nvm alias default 20.10.0

ヒント

nvm install --lts コマンドを使えば、常に最新のLTS版をインストールできます。バージョン番号を覚える必要がないので便利です。

2.4 インストール確認

Node.jsが正しくインストールされたか確認しましょう。ターミナル（またはコマンドプロンプト）を開いて、以下のコマンドを実行します。

動作確認

バージョン確認コマンド 
# Node.jsのバージョンを確認
node --version
# または
node -v

# npmのバージョンを確認
npm --version
# または
npm -v

期待される出力例：

$ node -v
v20.10.0

$ npm -v
10.2.3

npmとは？

npm（Node Package Manager）は、Node.jsのパッケージ管理ツールです。Node.jsと一緒に自動的にインストールされます。第6章で詳しく学習します。

2.5 VS Codeのセットアップ

Visual Studio Code（VS Code）は、Node.js開発に最も人気のあるエディタです。無料で使えて、拡張機能が豊富です。

VS Codeのインストール

  1. VS Code公式サイトにアクセス
  2. お使いのOSに合ったインストーラーをダウンロード
  3. インストーラーを実行し、ウィザードに従ってインストール

推奨拡張機能

Node.js開発を効率化する拡張機能をインストールしましょう。VS Codeの拡張機能パネル（Ctrl+Shift+X）から検索してインストールできます。

ESLint

JavaScriptのコード品質チェックツール。文法エラーやコーディング規約違反を自動検出します。

dbaeumer.vscode-eslint

Prettier

コードの自動フォーマッター。保存時に自動的にコードを整形してくれます。

esbenp.prettier-vscode

Node.js Extension Pack

Node.js開発に便利な拡張機能がまとめてインストールされるパック。

waderyan.nodejs-extension-pack

JavaScript Debugger

デバッグ機能を強化。ブレークポイントを設定して、コードの実行を追跡できます。

ms-vscode.js-debug

2.6 トラブルシューティング

インストール時によくある問題と解決方法をまとめました。

よくある問題と解決策

❌ 「node」コマンドが認識されない

原因：環境変数PATHが正しく設定されていない

解決策：

  • ターミナル/コマンドプロンプトを再起動
  • PCを再起動してパスを再読み込み
  • 再インストールして「Add to PATH」にチェック

❌ nvmコマンドが使えない（macOS/Linux）

原因：シェル設定ファイルが読み込まれていない

解決策：

# bashの場合
source ~/.bashrc

# zshの場合
source ~/.zshrc

# または新しいターミナルを開く

❌ 権限エラーが発生する（Windows）

原因：管理者権限が必要な操作を実行している

解決策：

  • コマンドプロンプトを管理者として実行
  • nvm-windowsは必ず管理者権限で操作する

❌ バージョンが古いまま変わらない

原因：既存のNode.jsと競合している

解決策：

  • 既存のNode.jsをアンインストール
  • nvmで管理する場合は公式インストーラー版を削除
  • which node で実行ファイルの場所を確認

理解度確認クイズ

  1. Q1. Node.jsのLTS版とCurrent版の違いは何ですか？どちらを選ぶべきですか？

    回答を表示

    LTS版は長期サポート版で安定性重視、本番環境向け。Current版は最新機能版で実験的な機能も含まれます。初心者や本番環境ではLTS版を選ぶべきです。最新機能を試したい場合のみCurrent版を使用します。

  2. Q2. nvmを使うメリットは何ですか？

    回答を表示

    nvmを使うと、複数のNode.jsバージョンを簡単に管理できます。プロジェクトごとに異なるバージョンを使い分けたり、新しいバージョンを試したりする際に便利です。nvm use コマンドで瞬時にバージョンを切り替えられるため、実務では必須のツールです。

  3. Q3. Node.jsと一緒にインストールされるnpmとは何ですか？

    回答を表示

    npm（Node Package Manager）は、Node.jsのパッケージ管理ツールです。JavaScriptライブラリやツールをインストール・管理するために使用します。Node.jsをインストールすると自動的に一緒にインストールされ、npm -v コマンドでバージョンを確認できます。

この章のまとめ

  • Node.jsのインストール方法には「公式インストーラー」と「nvm」の2種類がある
  • 実務ではバージョン管理が容易なnvmの使用が推奨される
  • LTS版は安定性重視で本番環境向け、Current版は最新機能を試したい場合に使用
  • インストール後は node -v と npm -v で確認
  • VS CodeにESLint、Prettier、Node.js Extension Packなどの拡張機能を導入すると開発効率が向上
