組み込みモジュールを活用しよう

Node.jsに標準で搭載されている便利なモジュール（fs、path、os、url）の使い方を学び、Node.jsの強力な機能を体験します。

この章で学ぶこと

  • Node.jsの組み込みモジュールとは何かを理解する
  • fsモジュールでファイル操作の基礎を習得する
  • pathモジュールで安全なパス操作を実践する
  • osモジュールとurlモジュールの活用方法を学ぶ

5.1 組み込みモジュールとは

組み込みモジュール（Built-in Modules）とは、Node.jsに最初から搭載されている機能群のことです。追加のインストール不要で、すぐに使えます。

わかりやすい例え

スマートフォンで例えると、組み込みモジュールは「最初からインストールされているカメラアプリや電卓アプリ」のようなものです。追加ダウンロード不要で、すぐに使えます。一方、npmパッケージは「App Storeからインストールするアプリ」に相当します。

主要な組み込みモジュール

fs（File System）

ファイルやディレクトリの操作

path

ファイルパスの操作・結合

os

オペレーティングシステム情報の取得

url

URL文字列の解析と操作

http / https

Webサーバーとクライアントの構築

stream

データのストリーム処理

5.2 fsモジュール（ファイルシステム）の基礎

fsモジュールは、ファイルやディレクトリを操作するためのモジュールです。ファイルの読み書き、削除、ディレクトリの作成などができます。

基本的な使い方

fsモジュールは require('fs') で読み込みます。最近では、Promiseベースの fs/promises も推奨されています。

fs-basic.js 
// 従来の方法（コールバック）
const fs = require('fs');

// Promiseベースの方法（推奨）
const fs = require('fs').promises;

// または
const fsPromises = require('fs/promises');

ファイルの読み込み

read-file.js 
// async/awaitを使った読み込み
const fs = require('fs').promises;

async function readFile() {
  try {
    // ファイルを読み込む
    const data = await fs.readFile('sample.txt', 'utf8');
    console.log('ファイル内容:', data);
  } catch (error) {
    console.error('エラー:', error.message);
  }
}

readFile();

ファイルへの書き込み

write-file.js 
const fs = require('fs').promises;

async function writeFile() {
  try {
    // ファイルに書き込む（上書き）
    await fs.writeFile('output.txt', 'Hello Node.js!', 'utf8');
    console.log('ファイルを作成しました');
  } catch (error) {
    console.error('エラー:', error.message);
  }
}

writeFile();

重要ポイント

readFile と writeFile の第2引数 'utf8' を指定すると、文字列として扱われます。指定しない場合はBufferオブジェクトが返されます。

実習 5-1: テキストファイルの読み書き

fsモジュールを使って、テキストファイルを読み込み、内容を加工して新しいファイルに保存してみましょう。

手順

  1. 1 message.txt を作成し、「Node.jsは楽しい」と記述
  2. 2 fsモジュールでファイルを読み込み、内容を大文字に変換
  3. 3 変換した内容を output.txt に書き込む

5.3 pathモジュールでパス操作

pathモジュールは、ファイルパスの操作を安全かつ簡単に行うためのモジュールです。OSごとに異なるパス区切り文字（Windows: \、macOS/Linux: /）を自動的に処理してくれます。

よくある間違い

パスを文字列連結（'folder' + '/' + 'file.txt'）で作成すると、Windowsで動作しない場合があります。必ず path.join() を使いましょう。

主要なメソッド

path-examples.js 
const path = require('path');

// パスを結合（OS依存の区切り文字を自動使用）
const filePath = path.join('folder', 'subfolder', 'file.txt');
console.log(filePath); // folder/subfolder/file.txt

// ファイル名を取得
console.log(path.basename(filePath)); // file.txt

// ディレクトリ名を取得
console.log(path.dirname(filePath)); // folder/subfolder

// 拡張子を取得
console.log(path.extname(filePath)); // .txt

絶対パスと相対パス

absolute-path.js 
const path = require('path');

// 相対パスを絶対パスに変換
const absolutePath = path.resolve('data', 'users.json');
console.log(absolutePath);
// 例: /home/user/project/data/users.json

// 現在のディレクトリの絶対パス
console.log(__dirname); // 実行中ファイルのディレクトリ
console.log(__filename); // 実行中ファイルのフルパス

5.4 osモジュールでシステム情報取得

osモジュールは、実行中のオペレーティングシステムに関する情報を取得できます。OS判定、メモリ容量、CPUコア数などが取得可能です。

os-info.js 
const os = require('os');

// OS名とバージョン
console.log('OS:', os.type()); // 例: Linux, Darwin, Windows_NT
console.log('プラットフォーム:', os.platform()); // win32, darwin, linux

// メモリ情報（バイト単位）
const totalMem = os.totalmem() / (1024 ** 3); // GBに変換
const freeMem = os.freemem() / (1024 ** 3);
console.log(`メモリ: ${freeMem.toFixed(2)}GB / ${totalMem.toFixed(2)}GB`);

// CPU情報
console.log('CPUコア数:', os.cpus().length);

活用例

osモジュールは、OS別に処理を分岐させたり、システムリソースを監視するツールを作る際に便利です。例えば、Windows専用の処理を os.platform() === 'win32' で判定できます。

5.5 urlモジュールでURL解析

urlモジュールは、URL文字列を解析したり、クエリパラメータを取得したりできます。最近では URL クラス（グローバル）も使えます。

url-parse.js 
// URLクラスを使用（推奨）
const myURL = new URL('https://example.com/search?q=nodejs&page=1');

console.log('プロトコル:', myURL.protocol); // https:
console.log('ホスト:', myURL.host); // example.com
console.log('パス:', myURL.pathname); // /search

// クエリパラメータを取得
console.log('検索キーワード:', myURL.searchParams.get('q')); // nodejs
console.log('ページ番号:', myURL.searchParams.get('page')); // 1

URLの構築

url-build.js 
const baseURL = 'https://api.example.com/users';
const params = new URLSearchParams({
  page: 1,
  limit: 10,
  sort: 'name'
});

// URLを構築
const apiURL = `${baseURL}?${params.toString()}`;
console.log(apiURL);
// https://api.example.com/users?page=1&limit=10&sort=name

理解度確認クイズ

  1. Q1. 組み込みモジュールとnpmパッケージの違いは何ですか?

    回答を表示

    組み込みモジュールはNode.jsに最初から搭載されており、追加インストール不要ですぐに使えます。一方、npmパッケージは外部ライブラリで、npm installでインストールする必要があります。組み込みモジュールの例: fs、path、os、url。npmパッケージの例: express、lodash、axios。

  2. Q2. パスを文字列連結ではなく path.join() を使う理由は?

    回答を表示

    OSによってパス区切り文字が異なるためです。Windowsは \（バックスラッシュ）、macOS/Linuxは /（スラッシュ）を使います。path.join() を使えば、実行環境に応じて自動的に正しい区切り文字を使ってくれるため、クロスプラットフォーム対応のコードが書けます。

  3. Q3. fs.readFile() で 'utf8' を指定しない場合、何が返されますか?

    回答を表示

    Bufferオブジェクトが返されます。Bufferはバイナリデータを扱うためのオブジェクトです。テキストファイルを読み込む場合は、文字エンコーディング（通常は 'utf8'）を指定することで、文字列として取得できます。バイナリファイル（画像など）を扱う場合は、エンコーディングを指定せずBufferとして扱います。
