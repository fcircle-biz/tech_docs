非同期処理をマスターしよう

Node.jsの核心である非同期処理を深く理解します。コールバック、Promise、async/awaitの使い分けを、実践的な例で学びます。

この章で学ぶこと

  • 同期処理と非同期処理の違いを理解する
  • コールバック関数の仕組みとコールバック地獄を回避する方法を学ぶ
  • Promiseの基本と使い方をマスターする
  • async/awaitを使った最新の非同期処理パターンを習得する
  • 非同期処理のエラーハンドリング手法を学ぶ

7.1 同期処理と非同期処理

プログラミングには、同期処理（Synchronous）と非同期処理（Asynchronous）の2つの実行パターンがあります。Node.jsでは非同期処理が中心となるため、この違いをしっかり理解することが重要です。

同期処理とは

同期処理は、1つの処理が完了するまで次の処理を待つ実行方式です。処理は上から順番に実行され、前の処理が終わらない限り次に進みません。

sync-example.js 
// 同期処理の例
console.log('開始');
console.log('処理1');
console.log('処理2');
console.log('終了');

// 出力:
// 開始
// 処理1
// 処理2
// 終了

非同期処理とは

非同期処理は、処理の完了を待たずに次の処理を開始できる実行方式です。時間のかかる処理（ファイル読み込み、データベースアクセス、HTTP通信など）を待っている間に、他の処理を実行できます。

async-example.js 
// 非同期処理の例
console.log('開始');

// setTimeoutは非同期関数
setTimeout(() => {
  console.log('処理1（2秒後）');
}, 2000);

console.log('処理2');
console.log('終了');

// 出力:
// 開始
// 処理2
// 終了
// 処理1（2秒後）← 最後に実行される

重要ポイント

非同期処理では、処理の開始順序と完了順序が異なることがあります。Node.jsでは、I/O操作のほとんどが非同期で実行されるため、この動作を理解することが重要です。

7.2 コールバック関数

コールバック関数は、非同期処理が完了したときに呼び出される関数です。Node.jsの伝統的な非同期処理パターンで、特にファイル操作やネットワーク通信で広く使われています。

基本的なコールバック

callback-basic.js 
const fs = require('fs');

// ファイルを非同期で読み込む
fs.readFile('sample.txt', 'utf8', (err, data) => {
  // コールバック関数
  if (err) {
    console.error('エラー:', err);
    return;
  }
  console.log('ファイル内容:', data);
});

console.log('ファイル読み込みを開始しました');

Node.jsのエラーファーストコールバック

Node.jsのコールバック関数は、最初の引数がエラーオブジェクトという規約があります。エラーがない場合は null が渡されます。

コールバック地獄

複数の非同期処理を順番に実行する場合、コールバック関数をネストしていくと、コードが読みにくくなります。これを「コールバック地獄（Callback Hell）」と呼びます。

callback-hell.js 
// ❌ コールバック地獄の例（読みにくい）
fs.readFile('user.json', 'utf8', (err, userData) => {
  if (err) return console.error(err);

  const user = JSON.parse(userData);
  fs.readFile(`${user.id}.txt`, 'utf8', (err, profile) => {
    if (err) return console.error(err);

    fs.readFile('settings.json', 'utf8', (err, settings) => {
      if (err) return console.error(err);

      // さらにネストが深くなる...
    });
  });
});

つまずきポイント

コールバックのネストが深くなると、コードの可読性が低下し、エラーハンドリングも複雑になります。この問題を解決するために、Promise や async/await が導入されました。

7.3 Promise

Promiseは、非同期処理をより扱いやすくするために導入されたJavaScriptの機能です。Promiseを使うと、コールバック地獄を回避し、より読みやすいコードを書けます。

Promiseの3つの状態

  • ⏳
    Pending（待機中）

    非同期処理が実行中の状態

  • ✓
    Fulfilled（成功）

    非同期処理が正常に完了した状態

  • ✗
    Rejected（失敗）

    非同期処理がエラーで終了した状態

Promiseの基本構文

promise-basic.js 
// Promiseを作成
const myPromise = new Promise((resolve, reject) => {
  const success = true;

  if (success) {
    resolve('成功しました'); // 成功時
  } else {
    reject('失敗しました');   // 失敗時
  }
});

// Promiseを使用
myPromise
  .then(result => {
    console.log(result); // "成功しました"
  })
  .catch(error => {
    console.error(error);
  });

Promiseチェーン

.then() を連結することで、複数の非同期処理を順番に実行できます。

promise-chain.js 
const fs = require('fs').promises;

// ✅ Promiseチェーンで順次実行（読みやすい）
fs.readFile('user.json', 'utf8')
  .then(userData => {
    const user = JSON.parse(userData);
    return fs.readFile(`${user.id}.txt`, 'utf8');
  })
  .then(profile => {
    console.log('プロフィール:', profile);
    return fs.readFile('settings.json', 'utf8');
  })
  .then(settings => {
    console.log('設定:', settings);
  })
  .catch(err => {
    console.error('エラーが発生:', err);
  });

Promise.all（並列実行）

複数の非同期処理を同時に実行し、すべて完了するまで待つには Promise.all() を使います。

promise-all.js 
const fs = require('fs').promises;

// 複数のファイルを同時に読み込む
Promise.all([
  fs.readFile('file1.txt', 'utf8'),
  fs.readFile('file2.txt', 'utf8'),
  fs.readFile('file3.txt', 'utf8')
])
  .then(([data1, data2, data3]) => {
    console.log('ファイル1:', data1);
    console.log('ファイル2:', data2);
    console.log('ファイル3:', data3);
  })
  .catch(err => {
    console.error('エラー:', err);
  });

7.4 async/await

async/awaitは、Promiseをより読みやすく書くための構文です。非同期コードを同期処理のように書けるため、現在のNode.js開発では最も推奨される方法です。

基本構文

async-await-basic.js 
const fs = require('fs').promises;

// async関数を定義
async function readFiles() {
  try {
    // awaitでPromiseの完了を待つ
    const data1 = await fs.readFile('file1.txt', 'utf8');
    console.log('ファイル1:', data1);

    const data2 = await fs.readFile('file2.txt', 'utf8');
    console.log('ファイル2:', data2);

    const data3 = await fs.readFile('file3.txt', 'utf8');
    console.log('ファイル3:', data3);

  } catch (err) {
    console.error('エラー:', err);
  }
}

readFiles();

重要ポイント

await は async 関数の中でしか使えません。また、async 関数は常にPromiseを返します。

3つの書き方の比較

同じ処理の3パターン

❌ コールバック（読みにくい）

fs.readFile('user.json', 'utf8', (err, data) => {
  if (err) return console.error(err);
  console.log(data);
});

⚠️ Promise（やや読みやすい）

fs.promises.readFile('user.json', 'utf8')
  .then(data => console.log(data))
  .catch(err => console.error(err));

✅ async/await（最も読みやすい）

async function readUser() {
  try {
    const data = await fs.promises.readFile('user.json', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

並列実行とasync/await

async/awaitでも Promise.all() を使うことで並列実行できます。

async-parallel.js 
const fs = require('fs').promises;

async function readFilesParallel() {
  try {
    // 並列実行（速い）
    const [data1, data2, data3] = await Promise.all([
      fs.readFile('file1.txt', 'utf8'),
      fs.readFile('file2.txt', 'utf8'),
      fs.readFile('file3.txt', 'utf8')
    ]);

    console.log('ファイル1:', data1);
    console.log('ファイル2:', data2);
    console.log('ファイル3:', data3);

  } catch (err) {
    console.error('エラー:', err);
  }
}

readFilesParallel();

パフォーマンスのヒント

順次実行と並列実行の違いを理解しましょう。独立した複数の非同期処理は Promise.all() で並列実行することで、処理時間を大幅に短縮できます。

7.5 エラーハンドリング

非同期処理では、エラーハンドリングが特に重要です。それぞれの方式で適切にエラーを処理する方法を学びましょう。

async/awaitのエラーハンドリング

error-handling.js 
const fs = require('fs').promises;

async function processFile(filename) {
  try {
    const data = await fs.readFile(filename, 'utf8');
    const parsed = JSON.parse(data);
    return parsed;

  } catch (err) {
    // エラーの種類に応じた処理
    if (err.code === 'ENOENT') {
      console.error('ファイルが見つかりません:', filename);
    } else if (err instanceof SyntaxError) {
      console.error('JSON解析エラー:', err.message);
    } else {
      console.error('予期しないエラー:', err);
    }
    throw err; // 上位の呼び出し元にエラーを伝播
  }
}

// 呼び出し側でもエラーハンドリング
async function main() {
  try {
    const result = await processFile('data.json');
    console.log('処理成功:', result);
  } catch (err) {
    console.error('処理失敗');
    process.exit(1);
  }
}

main();

Promise.allのエラーハンドリング

Promise.all() は、1つでもエラーが発生するとすべての処理が中断されます。すべての処理を完了させたい場合は Promise.allSettled() を使います。

promise-allsettled.js 
const fs = require('fs').promises;

async function readMultipleFiles() {
  // すべての処理を実行し、成功・失敗の両方の結果を取得
  const results = await Promise.allSettled([
    fs.readFile('file1.txt', 'utf8'),
    fs.readFile('notexist.txt', 'utf8'), // エラー
    fs.readFile('file3.txt', 'utf8')
  ]);

  results.forEach((result, index) => {
    if (result.status === 'fulfilled') {
      console.log(`ファイル${index + 1} 成功:`, result.value);
    } else {
      console.error(`ファイル${index + 1} 失敗:`, result.reason);
    }
  });
}

readMultipleFiles();

実習 7-1: async/awaitで複数のファイルを処理する

async/awaitを使って、複数のテキストファイルを読み込み、内容を結合して新しいファイルに書き出すプログラムを作成しましょう。

手順

  1. 1 3つのテキストファイル（part1.txt、part2.txt、part3.txt）を作成し、それぞれに好きな文章を書く
  2. 2 async関数を使って、3つのファイルを順番に読み込む
  3. 3 読み込んだ内容を結合し、combined.txtに書き出す
  4. 4 エラーハンドリングを追加する（try-catch）
解答例を表示 
const fs = require('fs').promises;

async function combineFiles() {
  try {
    // 3つのファイルを順番に読み込む
    const part1 = await fs.readFile('part1.txt', 'utf8');
    const part2 = await fs.readFile('part2.txt', 'utf8');
    const part3 = await fs.readFile('part3.txt', 'utf8');

    // 内容を結合
    const combined = part1 + '\n' + part2 + '\n' + part3;

    // 新しいファイルに書き出す
    await fs.writeFile('combined.txt', combined, 'utf8');

    console.log('ファイルの結合が完了しました');

  } catch (err) {
    console.error('エラーが発生しました:', err.message);
  }
}

combineFiles();

理解度確認クイズ

  1. Q1. 同期処理と非同期処理の違いを説明してください。

    回答を表示

    同期処理は、1つの処理が完了するまで次の処理を待つ方式です。処理は順番に実行されます。非同期処理は、処理の完了を待たずに次の処理を開始できる方式で、時間のかかる処理（I/O操作など）を待っている間に他の処理を実行できます。

  2. Q2. コールバック地獄を解決するために、どのような方法がありますか？

    回答を表示

    コールバック地獄を解決する方法は主に2つあります：
    1. Promiseを使う：.then() を連結することで、ネストを浅くできます。
    2. async/awaitを使う：非同期コードを同期処理のように書けるため、最も読みやすくなります。

  3. Q3. Promise.all() と Promise.allSettled() の違いは何ですか？

    回答を表示

    Promise.all() は、すべてのPromiseが成功した場合のみ結果を返します。1つでも失敗するとすぐにエラーで終了します。
    Promise.allSettled() は、すべてのPromiseが完了する（成功または失敗）まで待ち、それぞれの結果（成功/失敗）をすべて返します。

  4. Q4. async関数が常に返すものは何ですか？

    回答を表示

    async関数は常にPromiseを返します。関数内で値を return した場合、その値で解決されたPromiseが返されます。エラーが発生した場合は、エラーで拒否されたPromiseが返されます。

この章のまとめ

  • 非同期処理は、処理の完了を待たずに次の処理を開始できる仕組み
  • コールバック関数は伝統的な非同期パターンだが、ネストが深くなると読みにくい
  • Promiseは非同期処理を扱いやすくし、3つの状態（Pending、Fulfilled、Rejected）を持つ
  • async/awaitは最も読みやすい非同期パターンで、現在のNode.js開発で推奨される
  • Promise.all()で並列実行、Promise.allSettled()ですべての結果を取得できる
  • 非同期処理のエラーハンドリングはtry-catchで行うのが基本
