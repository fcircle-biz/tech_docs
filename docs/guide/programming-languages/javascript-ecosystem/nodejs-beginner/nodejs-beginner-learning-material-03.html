はじめてのNode.jsプログラム

ターミナルからNode.jsプログラムを実行する方法を学びます。「Hello World」から始めて、対話モード（REPL）の使い方も習得します。

この章で学ぶこと

  • ターミナル（コマンドライン）からNode.jsを実行する方法
  • 初めてのNode.jsプログラム「Hello World」を作成する
  • REPL（対話モード）を使ってJavaScriptコードを試す
  • Node.jsスクリプトファイルの作成と実行方法を習得する

3.1 Node.jsの実行方法

Node.jsプログラムを実行するには、主に2つの方法があります。

（対話モード）"] A --> C["スクリプト実行モード"] B --> B1["コマンド: node"] B --> B2["1行ずつコードを試せる"] B --> B3["学習や動作確認に便利"] C --> C1["コマンド: node ファイル名.js"] C --> C2["ファイルに書いたプログラムを実行"] C --> C3["実際の開発で使う"]

ターミナルとは？

ターミナル（コマンドライン）は、テキストコマンドを使ってコンピュータを操作するツールです。Windowsでは「コマンドプロンプト」や「PowerShell」、macOS/Linuxでは「ターミナル」と呼ばれます。Node.jsはこのターミナルから実行します。

3.2 REPL（対話モード）を使ってみよう

REPL（レプル）は「Read-Eval-Print-Loop」の略で、コードを1行ずつ入力して即座に結果を確認できる対話モードです。JavaScriptの文法や関数の動作を手軽に試すのに便利です。

実習: REPLモードを起動する

ターミナルを開いて、以下のコマンドを入力しましょう。

ターミナル 
node

成功すると、以下のようなプロンプトが表示されます。

Welcome to Node.js v18.17.0.
Type ".help" for more information.
>

> プロンプトが表示されたら、Node.jsのREPLモードが起動しています。

実習: REPLで簡単な計算をしてみる

REPLモードで以下のコードを1行ずつ入力してみましょう。

REPL 
// 基本的な計算
1 + 2

// 変数の宣言
const message = "Hello, Node.js!"

// 変数の表示
message

// 配列の操作
const numbers = [1, 2, 3, 4, 5]
numbers.map(n => n * 2)

各コードを入力してEnterキーを押すと、即座に結果が表示されます。

REPL の便利なコマンド

コマンド 説明
.help 使用可能なコマンド一覧を表示
.exit REPLモードを終了（Ctrl+C を2回押しても終了）
.clear コンテキストをクリア（変数などをリセット）
.save ファイル名 現在のセッションをファイルに保存
.load ファイル名 ファイルからコードを読み込んで実行

3.3 初めてのNode.jsプログラム「Hello World」

それでは、Node.jsで最も基本的なプログラム「Hello World」を作成しましょう。REPLモードを終了して、実際のスクリプトファイルを作成します。

実習: Hello Worldプログラムを作成する

ステップ1: ファイルを作成する

任意のフォルダに hello.js というファイルを作成します。

hello.js 
// 初めてのNode.jsプログラム
console.log("Hello, Node.js!");
console.log("Node.jsの世界へようこそ！");

ステップ2: プログラムを実行する

ターミナルでファイルを作成したフォルダに移動し、以下のコマンドを実行します。

ターミナル 
node hello.js

実行結果

Hello, Node.js!
Node.jsの世界へようこそ！

重要ポイント

console.log() はブラウザJavaScriptと同じようにNode.jsでも使えます。これは標準出力（ターミナル）にメッセージを表示する関数です。

3.4 プログラムに機能を追加してみよう

Hello Worldに少し機能を追加して、Node.jsプログラムの基本を学びましょう。

実習: 変数と関数を使ったプログラム

greeting.js という新しいファイルを作成します。

greeting.js 
// 挨拶関数を定義
function greet(name) {
  return `こんにちは、${name}さん！`;
}

// 変数を定義
const userName = "太郎";

// 関数を呼び出して結果を表示
const message = greet(userName);
console.log(message);

// 複数のユーザーに挨拶
const users = ["花子", "次郎", "三郎"];
users.forEach(user => {
  console.log(greet(user));
});

実行してみる

ターミナル 
node greeting.js

実行結果

こんにちは、太郎さん！
こんにちは、花子さん！
こんにちは、次郎さん！
こんにちは、三郎さん！

3.5 コマンドライン引数を使ってみよう

Node.jsプログラムは、実行時にコマンドライン引数を受け取ることができます。これにより、同じプログラムでも異なる入力で動作させることが可能になります。

process.argvとは？

process.argv は、Node.jsで実行時の引数にアクセスするための組み込み変数です。配列形式でコマンドライン引数が格納されています。

実習: コマンドライン引数を使う

args.js を作成します。

args.js 
// コマンドライン引数を表示
console.log("全ての引数:", process.argv);

// ユーザーが入力した引数のみ取得（最初の2つはNode.jsとファイル名）
const userArgs = process.argv.slice(2);
console.log("ユーザー引数:", userArgs);

// 引数がある場合は挨拶、ない場合はデフォルトメッセージ
if (userArgs.length > 0) {
  const name = userArgs[0];
  console.log(`こんにちは、${name}さん！`);
} else {
  console.log("名前を引数として渡してください。");
}

実行例

ターミナル 
# 引数なしで実行
node args.js

# 引数ありで実行
node args.js 太郎

# 複数の引数
node args.js 太郎 25 東京

動作確認

以下の項目を確認しましょう。

  • REPLモードを起動して、簡単な計算ができる
  • hello.jsを作成し、ターミナルから実行できる
  • greeting.jsで変数と関数を使ったプログラムが動作する
  • args.jsでコマンドライン引数を受け取れる

トラブルシューティング

エラー: 'node' は内部コマンドまたは外部コマンドとして認識されていません

原因: Node.jsがインストールされていないか、環境変数PATHが設定されていません。

解決策: Node.jsを公式サイトからインストールし、ターミナルを再起動してください。

エラー: Cannot find module 'hello.js'

原因: ターミナルの現在位置（カレントディレクトリ）がファイルのある場所と異なります。

解決策: cd コマンドでファイルのあるフォルダに移動してから実行してください。

REPLを終了できない

解決策: .exit と入力するか、Ctrl+C を2回押してください。

理解度確認クイズ

  1. Q1. REPLモードとスクリプト実行モードの違いは何ですか？それぞれどのような場面で使いますか？

    回答を表示

    REPLモード: コマンド node で起動し、1行ずつコードを入力して即座に結果を確認できます。JavaScriptの文法や関数の動作を試す学習や動作確認に便利です。

    スクリプト実行モード: node ファイル名.js でファイルに書いたプログラムを実行します。実際の開発では主にこちらを使います。

  2. Q2. process.argv の最初の2つの要素には何が格納されていますか？

    回答を表示

    process.argv[0] にはNode.jsの実行パス、process.argv[1] には実行されたスクリプトファイルのパスが格納されています。ユーザーが渡した引数は process.argv[2] 以降に格納されます。

  3. Q3. Node.jsプログラムで標準出力にメッセージを表示するには、どの関数を使いますか？

    回答を表示

    console.log() 関数を使います。これはブラウザJavaScriptと同じで、ターミナルにメッセージを出力します。例: console.log("Hello, Node.js!")

この章のまとめ

  • Node.jsプログラムはREPLモード（対話モード）とスクリプト実行モードの2つの方法で実行できる
  • REPLモードは node コマンドで起動し、1行ずつコードを試せる
  • スクリプトファイルは node ファイル名.js で実行する
  • console.log() でターミナルにメッセージを出力できる
  • process.argv を使ってコマンドライン引数を受け取れる
