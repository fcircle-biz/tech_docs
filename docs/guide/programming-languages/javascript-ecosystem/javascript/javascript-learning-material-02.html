第2章

開発環境を準備しよう

プログラムを書くための道具（エディタ）の準備と、ブラウザの開発者ツールの使い方を画像付きで丁寧に解説します。

この章で学ぶこと

  • 開発環境とは何か、なぜ必要なのかを理解する
  • Visual Studio Codeのインストールと基本的な使い方を習得する
  • ブラウザの開発者ツールとは何か、どんなことができるのかを知る
  • 開発者ツールの基本的な使い方を実際に試してみる

2.1 開発環境とは

プログラミングを始めるには、開発環境と呼ばれる道具を準備する必要があります。 開発環境とは、プログラムを書いたり、動かしたり、問題を見つけたりするための道具一式のことです。

身近な例で考えてみよう

料理をするには、包丁、まな板、フライパンなどの道具が必要ですよね。 プログラミングも同じで、プログラムを書くための専用の道具が必要なのです。 その道具の中で最も重要なのが、テキストエディタと開発者ツールです。

テキストエディタ

プログラムのコード（命令文）を書くための道具です。メモ帳のようなものですが、プログラミング専用の便利な機能がたくさん付いています。

開発者ツール

ブラウザに組み込まれている、プログラムの動きを確認したり、問題を見つけたりするための道具です。

この章では、初心者に最もおすすめのVisual Studio Code（ビジュアル スタジオ コード）というテキストエディタと、 ブラウザの開発者ツールの準備と使い方を学びます。

2.2 Visual Studio Codeのインストール

Visual Studio Code（略して「VS Code」と呼びます）は、Microsoftが作った無料のテキストエディタです。 世界中のプログラマーに愛用されており、初心者にもとても使いやすいのが特徴です。

VS Codeが初心者におすすめな理由

  • 完全無料で、Windows、Mac、Linuxのどれでも使える
  • 日本語に対応していて、メニューも全部日本語で読める
  • コードの色分け表示やミスを教えてくれる機能が充実している
  • 世界中で使われているので、困ったときの情報が豊富

インストール手順

実習 2-1: VS Codeをインストールしよう

実際に手を動かして、VS Codeをインストールしてみましょう。

手順

  1. 1 Webブラウザで以下のURLにアクセスします。
    https://code.visualstudio.com/
  2. 2 画面中央に大きく表示されている「Download」ボタンをクリックします。
    お使いのパソコン（Windows、Mac、Linux）が自動的に判別されます。
  3. 3 ダウンロードしたファイルを開きます。
    Windows: ダウンロードフォルダの「VSCodeUserSetup-*.exe」をダブルクリック
    Mac: ダウンロードフォルダの「VSCode-*.zip」を解凍して、Visual Studio Code.appをアプリケーションフォルダに移動
  4. 4 インストーラーの指示に従って進めます。
    特に理由がなければ、全て「次へ」で進めてOKです。
  5. 5 インストールが完了したら、VS Codeを起動してみましょう。

注意: Macをお使いの方へ

初めて起動すると「開発元が未確認のため開けません」というメッセージが出る場合があります。 その場合は、「システム環境設定」→「セキュリティとプライバシー」で「このまま開く」をクリックしてください。

2.3 VS Codeの日本語化と基本的な使い方

VS Codeは初期状態では英語で表示されていますが、拡張機能を入れることで簡単に日本語化できます。

実習 2-2: VS Codeを日本語化しよう

拡張機能を使って、VS Codeのメニューを日本語にしましょう。

手順

  1. 1 VS Codeを起動します。
  2. 2 左側のサイドバーにある四角いブロックのアイコン（拡張機能）をクリックします。
    または、キーボードで「Ctrl + Shift + X」（Macは「Cmd + Shift + X」）を押します。
  3. 3 検索ボックスに「Japanese」と入力します。
  4. 4 「Japanese Language Pack for Visual Studio Code」という拡張機能が表示されるので、 「Install」（インストール）ボタンをクリックします。
  5. 5 インストール後、右下に「再起動が必要です」というメッセージが表示されるので、 「Restart」をクリックしてVS Codeを再起動します。

VS Codeの画面構成

日本語化が完了したら、VS Codeの画面構成を確認しましょう。主に以下の部分で構成されています。

① エクスプローラー（左サイドバー）

ファイルやフォルダを表示・管理する場所です。プロジェクト内のファイル一覧が表示されます。

② エディタエリア（中央）

実際にコードを書く場所です。複数のファイルをタブで切り替えて編集できます。

③ パネル（下部）

ターミナル（コマンドを入力する場所）や、エラーメッセージなどが表示される場所です。

④ アクティビティバー（最左端）

エクスプローラー、検索、拡張機能など、様々な機能に素早くアクセスできるアイコンが並んでいます。

最初は触って慣れよう

最初のうちは、全ての機能を覚える必要はありません。 使っているうちに自然と覚えていくので、まずは実際にファイルを作って、コードを書いてみることから始めましょう。

2.4 ブラウザの開発者ツールとは

JavaScriptはWebブラウザで動くプログラムなので、ブラウザに組み込まれている開発者ツールを使って、 プログラムが正しく動いているかを確認します。

開発者ツールでできること

  • JavaScriptのコードを実行して、結果をすぐに確認できる
  • プログラムがエラーで止まったとき、どこで問題が起きたかを教えてくれる
  • Webページの構造（HTML、CSS）を確認・変更できる
  • ネットワーク通信の内容を見ることができる

どのブラウザにも開発者ツールは付いていますが、この教材ではGoogle Chromeを例に説明します。 Firefox、Edge、Safariでも基本的な使い方は同じです。

2.5 開発者ツールの基本的な使い方

それでは、実際に開発者ツールを開いて、使い方を確認してみましょう。

実習 2-3: 開発者ツールを開いてみよう

ブラウザで開発者ツールを開く練習をしましょう。

手順

  1. 1 Google Chromeを起動します。
  2. 2 どんなWebページでも良いので開きます（例：Yahoo! JAPANやGoogleのトップページ）。
  3. 3 キーボードで「F12」キーを押します。
    または、右クリックして「検証」を選択します。
    （Macの場合は「Cmd + Option + I」でも開けます）
  4. 4 ブラウザの右側または下側に、開発者ツールのパネルが表示されます。

開発者ツールの主要なタブ

開発者ツールには、たくさんのタブがありますが、初心者がまず覚えておくべきタブは次の2つです。

1 Consoleタブ（コンソール）

JavaScriptのコードを実際に実行して、結果を確認できる場所です。
また、プログラムにエラーがあると、ここにエラーメッセージが表示されます。

試しに入力してみよう：

// Consoleタブで試してみてください
console.log("こんにちは、JavaScript!");
2 + 3

上のコードを、Consoleタブに1行ずつ入力してEnterを押すと、結果が表示されます。

2 Elementsタブ（要素）

Webページの構造（HTML）とスタイル（CSS）を確認できる場所です。
JavaScriptでページの要素を操作するときに、どの部分が変更されたかを視覚的に確認できます。

今はまだ使いませんが、第9章「Webページを動かそう」で活躍します。

動作確認

Consoleタブで簡単な計算をしてみましょう。

  1. 1. 開発者ツールのConsoleタブを開く
  2. 2. 「10 + 20」と入力してEnterを押す
  3. 3. 「"Hello" + "World"」と入力してEnterを押す

期待される結果 

10 + 20
→ 30

"Hello" + "World"
→ "HelloWorld"

第2章のまとめ

  • 開発環境とは、プログラムを書いたり動かしたりするための道具一式のこと
  • Visual Studio Codeは無料で使える初心者向けのテキストエディタ
  • VS Codeは拡張機能を使って簡単に日本語化できる
  • 開発者ツールは、ブラウザに組み込まれている、プログラムの動作確認やエラー発見のための道具
  • 開発者ツールのConsoleタブで、JavaScriptのコードをすぐに試すことができる

理解度確認クイズ

  1. Q1. 開発環境とは何ですか？簡単に説明してください。

    回答を表示

    開発環境とは、プログラムを書いたり、動かしたり、問題を見つけたりするための道具一式のことです。 主にテキストエディタ（VS Codeなど）と開発者ツールが含まれます。

  2. Q2. Visual Studio Codeの日本語化には、何を使いますか？

    回答を表示

    拡張機能（Extension）を使います。具体的には「Japanese Language Pack for Visual Studio Code」という拡張機能をインストールすることで、日本語化できます。

  3. Q3. ブラウザの開発者ツールを開くには、どのキーを押しますか？（Windows/Linuxの場合）

    回答を表示

    F12キーを押します。または、Webページ上で右クリックして「検証」を選択する方法もあります。

  4. Q4. 開発者ツールで、JavaScriptのコードを試したり、エラーメッセージを確認したりできるタブはどれですか？

    回答を表示

    Consoleタブ（コンソール）です。このタブでコードを入力して実行したり、エラーメッセージを確認したりできます。

トラブルシューティング

VS Codeが日本語化されない

原因: 拡張機能のインストール後に再起動していない可能性があります。

解決方法: VS Codeを完全に終了してから、もう一度起動してみてください。それでも日本語化されない場合は、拡張機能が正しくインストールされているか確認しましょう。

開発者ツールが開かない

原因: F12キーが別の機能に割り当てられている、またはブラウザが古い可能性があります。

解決方法: Webページ上で右クリックして「検証」を選択してください。または、ブラウザを最新版にアップデートしてから再度試してみてください。

Consoleタブが見つからない

原因: 開発者ツールの表示が小さくて、タブが隠れている可能性があります。

解決方法: 開発者ツールの上部にあるタブの右側に「≫」のようなアイコンがある場合、そこをクリックするとConsoleタブが選択できます。または、開発者ツールのウィンドウを大きくしてみてください。
