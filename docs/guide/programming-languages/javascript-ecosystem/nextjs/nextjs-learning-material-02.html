第2章

ファイルベースルーティングを使いこなそう

Next.jsのApp Routerでは、フォルダ構造がそのままURLになる直感的なルーティングシステムを採用しています。動的ルート、レイアウト、エラーハンドリングなど、ルーティングの全機能を学びます。

この章で学ぶこと

  • App Routerのディレクトリ構造とファイル規約
  • 動的ルーティング（[slug]）とネストされたルート
  • ルートグループ、並列ルート、インターセプトルート
  • not-foundとerrorページの実装、リンクとメタデータ管理

2.1 App Routerのディレクトリ構造

Next.jsのApp Routerでは、appディレクトリ内のフォルダ構造がそのままURLルートになります。この直感的な設計により、ルーティング設定を別途記述する必要がありません。

特別なファイル規約

  • page.tsx：そのルートで表示されるページコンポーネント
  • layout.tsx：子ルートに共通するレイアウト（ヘッダー、サイドバー等）
  • loading.tsx：ページ読み込み中に表示するUI
  • error.tsx：エラー発生時に表示するUI
  • not-found.tsx：404ページ
  • route.tsx：API Routeハンドラ

App Routerの基本構造 

app/
├── page.tsx                 → /  (トップページ)
├── layout.tsx               → 全ページ共通レイアウト
├── about/
│   └── page.tsx             → /about
├── blog/
│   ├── page.tsx             → /blog  (ブログ一覧)
│   ├── [slug]/
│   │   └── page.tsx         → /blog/hello-world  (動的ルート)
│   └── layout.tsx           → /blog/* 共通レイアウト
└── dashboard/
    ├── page.tsx             → /dashboard
    ├── settings/
    │   └── page.tsx         → /dashboard/settings
    └── profile/
        └── page.tsx         → /dashboard/profile

2.2 ページとレイアウトの作成

page.tsxがそのルートの実際のコンテンツを定義し、layout.tsxが複数のページで共有されるUIを提供します。

実習 2-1: ページとレイアウトを作成する

ブログセクションにページとレイアウトを作成してみましょう。

app/blog/layout.tsx

TypeScript 
export default function BlogLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <div className="max-w-4xl mx-auto">
      {/* ブログ共通ヘッダー */}
      <header className="py-6 border-b">
        <h1 className="text-3xl font-bold">Tech Blog</h1>
      </header>
      {/* 子ページのコンテンツ */}
      <main className="py-8">{children}</main>
    </div>
  )
}

app/blog/page.tsx

TypeScript 
export default function BlogPage() {
  return (
    <div>
      <h2 className="text-2xl font-semibold mb-4">記事一覧</h2>
      <ul className="space-y-4">
        <li>記事1</li>
        <li>記事2</li>
      </ul>
    </div>
  )
}

レイアウトの重要ポイント

  • レイアウトは子ルート全てで共有される（ルートレイアウトは全ページで共有）
  • レイアウトはページ遷移時に再レンダリングされない（状態が保持される）
  • ネストされたレイアウトは内側に向かって入れ子になる

2.3 動的ルーティング（[slug]）

フォルダ名を[パラメータ名]で囲むと、動的なURLパラメータとして扱えます。ブログ記事やユーザープロフィールなど、IDに基づくページを作成できます。

実習 2-2: 動的ルートを作成する

app/blog/[slug]/page.tsx

TypeScript 
// paramsで動的パラメータを受け取る
export default function BlogPostPage({
  params,
}: {
  params: { slug: string }
}) {
  return (
    <article>
      <h1 className="text-3xl font-bold">記事: {params.slug}</h1>
      <p className="mt-4">ここに記事の内容が入ります。</p>
    </article>
  )
}

アクセス例

  • /blog/hello-world → params.slug = "hello-world"
  • /blog/nextjs-tutorial → params.slug = "nextjs-tutorial"

Catch-all Segments: [...slug]

[...slug]を使うと、複数のパスセグメントをまとめて受け取れます。 

app/docs/[...slug]/page.tsx
→ /docs/a  (params.slug = ['a'])
→ /docs/a/b  (params.slug = ['a', 'b'])
→ /docs/a/b/c  (params.slug = ['a', 'b', 'c'])

2.4 ルートグループと高度なルーティング

App Routerでは、URLに影響を与えずにフォルダを整理できるルートグループや、高度な並列表示が可能な機能があります。

ルートグループ: (フォルダ名)

フォルダ名を(name)で囲むと、URLパスに含まれません。レイアウトをグループごとに分けたい場合に有用です。 

app/
├── (marketing)/
│   ├── layout.tsx     → マーケティング用レイアウト
│   ├── page.tsx       → /
│   └── about/
│       └── page.tsx   → /about
└── (shop)/
    ├── layout.tsx     → ショップ用レイアウト
    ├── products/
    │   └── page.tsx   → /products
    └── cart/
        └── page.tsx   → /cart

(marketing)と(shop)は異なるレイアウトを持ちながら、URLにはフォルダ名が現れません。

並列ルート & インターセプトルート

並列ルート（@slot）：同じレイアウト内で複数のページを同時に表示できます（ダッシュボードなど）。

インターセプトルート（(..)folder）：別のルートのコンテンツをモーダルなどで表示できます。 

app/
├── @analytics/
│   └── page.tsx
├── @team/
│   └── page.tsx
└── layout.tsx  → {children} {analytics} {team} を同時表示

2.5 not-foundとerrorページの実装

Next.jsでは、404エラーやその他のエラーを美しく処理するための専用ファイルを用意できます。

not-found.tsx - 404ページ

app/not-found.tsx 
import Link from 'next/link'

export default function NotFound() {
  return (
    <div className="text-center py-20">
      <h2 className="text-4xl font-bold">404 - ページが見つかりません</h2>
      <Link href="/" className="text-blue-500">トップに戻る</Link>
    </div>
  )
}

error.tsx - エラーページ

エラーバウンダリとして機能し、子コンポーネントのエラーをキャッチします。

app/error.tsx 
'use client' // エラーコンポーネントはクライアントコンポーネント

export default function Error({
  error,
  reset,
}: {
  error: Error
  reset: () => void
}) {
  return (
    <div className="text-center py-20">
      <h2 className="text-2xl font-bold">エラーが発生しました</h2>
      <button onClick={reset}>もう一度試す</button>
    </div>
  )
}

2.6 リンクとナビゲーション

Next.jsでは、next/linkのLinkコンポーネントを使用してページ間を移動します。プリフェッチングにより高速なナビゲーションを実現します。

実習 2-3: Linkコンポーネントを使う

TypeScript 
import Link from 'next/link'

export default function Navigation() {
  return (
    <nav>
      <Link href="/">ホーム</Link>
      <Link href="/about">概要</Link>
      <Link href="/blog/hello-world">ブログ記事</Link>
    </nav>
  )
}

プログラマティックナビゲーション

フォーム送信後などにプログラムでページ遷移する場合は、useRouterを使います。

TypeScript 
'use client'
import { useRouter } from 'next/navigation'

export default function MyButton() {
  const router = useRouter()

  return (
    <button onClick={() => router.push('/dashboard')}>
      ダッシュボードへ
    </button>
  )
}

Linkのメリット

  • リンク先ページを事前に読み込むプリフェッチング
  • ページ全体をリロードせず高速な画面遷移
  • ブラウザの履歴が正しく管理される

2.7 メタデータの管理

Next.jsでは、SEOのために重要なメタデータ（title、description、OGタグなど）を簡単に設定できます。

静的メタデータの設定

app/about/page.tsx 
import { Metadata } from 'next'

export const metadata: Metadata = {
  title: '会社概要',
  description: '私たちの会社について',

理解度確認クイズ

  1. Q1. page.tsxとlayout.tsxの役割の違いを説明してください。

    回答を表示

    page.tsxはそのルートで表示される実際のコンテンツを定義します。layout.tsxは子ルート全てで共有されるUI（ヘッダー、サイドバー等）を定義し、ページ遷移時に再レンダリングされません。

  2. Q2. 動的ルート [slug] と Catch-all Segments [...slug] の違いは何ですか？

    回答を表示

    [slug]は単一のパスセグメントのみを受け取りますが、[...slug]は複数のパスセグメントを配列として受け取ることができます。例: /docs/a/b/c の場合、[...slug]でparams.slug = ['a', 'b', 'c']になります。

  3. Q3. ルートグループ (folder) を使う利点を2つ挙げてください。

    回答を表示

    ①URLパスに影響を与えずにルートを論理的に整理できる。②異なるレイアウトをグループごとに適用できる（例：マーケティングページとショップページで異なるデザイン）。

この章のまとめ

  • App Routerでは、appディレクトリのフォルダ構造がそのままURLになる直感的なルーティングを実現
  • page.tsx、layout.tsx、loading.tsx、error.tsx、not-found.tsxなどの特別なファイルで機能を定義
  • [slug]で動的ルート、[...slug]でCatch-all Segmentsを実装でき、柔軟なURL設計が可能
  • Linkコンポーネントとメタデータ管理により、高速でSEOに強いナビゲーションが実現できる
