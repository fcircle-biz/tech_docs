画像とフォントの最適化

next/imageとnext/fontを使った画像・フォントの自動最適化により、ページ速度とユーザー体験を大幅に改善する手法を学びます。

この章で学ぶこと

  • next/imageコンポーネントの基本的な使い方と自動最適化の仕組み
  • レスポンシブ画像、遅延読み込み、プレースホルダーの実装方法
  • 外部画像の扱いとリモート画像パターンの設定
  • next/fontによるフォント最適化とカスタムフォントの追加方法

8.1 next/imageによる画像最適化

Webサイトのパフォーマンスにおいて、画像は最大のボトルネックとなることが多いです。Next.jsのnext/imageコンポーネントは、以下の最適化を自動的に実行します：

next/imageの自動最適化機能

  • 自動リサイズ：デバイスに応じて適切なサイズの画像を配信
  • フォーマット変換：WebP/AVIFなどのモダンフォーマットに自動変換
  • 遅延読み込み：ビューポートに入るまで画像をロードしない
  • レイアウトシフト防止：CLS（Cumulative Layout Shift）を最小化
  • キャッシュ最適化：CDNとブラウザキャッシュを効率的に利用
  • 優先度制御：LCP（Largest Contentful Paint）画像の優先読み込み

8.2 next/imageの基本的な使い方

next/imageコンポーネントの基本的な使い方を理解しましょう。

実習 8-1: 基本的な画像表示

next/imageコンポーネントを使って、ローカル画像を表示します。

手順

  1. 1 public/imagesフォルダに画像を配置
  2. 2 next/imageをインポートしてコンポーネントで使用

コード例

app/page.tsx 
import Image from 'next/image';

export default function HomePage() {
  return (
    <div>
      <h1>画像最適化のデモ</h1>

      {/* 基本的な使い方 */}
      <Image
        src="/images/hero.jpg"  // publicフォルダからの相対パス
        alt="ヒーロー画像"
        width={800}              // 必須: 元画像の幅
        height={600}             // 必須: 元画像の高さ
      />
    </div>
  );
}

widthとheightは必須

next/imageではwidth/heightの指定が必須です。これにより、ブラウザは画像読み込み前にスペースを確保し、レイアウトシフトを防ぎます。

8.3 レスポンシブ画像の実装

画像をレスポンシブにする主な方法として、fillとsizesプロパティを使った実装があります。

app/components/HeroImage.tsx 
import Image from 'next/image';

export default function HeroImage() {
  return (
    <div className="relative w-full h-96">
      {/* fillを使った親要素いっぱいに表示 */}
      <Image
        src="/images/hero.jpg"
        alt="ヒーロー画像"
        fill                    // width/heightの代わりにfillを使用
        style={{ objectFit: 'cover' }}  // カバー表示
        priority                // LCP画像なので優先読み込み
      />
    </div>
  );
}

Imageコンポーネント主要プロパティ

プロパティ 説明 使用例
src 画像のパス（ローカル/リモート） /images/hero.jpg
width/height 画像の幅と高さ（px） 800 / 600
fill 親要素いっぱいに表示 true
priority 優先的に読み込む（LCP画像向け） true
loading 読み込み方式（lazy/eager） "lazy"
sizes レスポンシブ時の画像サイズ指定 "(max-width: 768px) 100vw"

sizesプロパティによる最適化

sizesプロパティを使うと、ブラウザが適切な画像サイズを選択できるようになります。

例: レスポンシブ画像 
<Image
  src="/images/product.jpg"
  alt="商品画像"
  width={1200}
  height={800}
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
  // モバイル: 画面幅いっぱい
  // タブレット: 画面幅の50%
  // デスクトップ: 画面幅の33%
/>

8.4 プレースホルダーとBlur効果

画像読み込み中にプレースホルダーを表示することで、より良いユーザー体験を提供できます。

app/components/BlurImage.tsx 
import Image from 'next/image';
import productImage from '@/public/images/product.jpg';

export default function BlurImage() {
  return (
    <Image
      src={productImage}
      alt="商品画像"
      placeholder="blur"  // Blur効果を有効化
      // ローカル画像（import）の場合、blurDataURLは自動生成される
    />
  );
}

プレースホルダーの種類

placeholder="blur"

低解像度のぼかし画像を表示（ローカル画像は自動生成）

placeholder="empty"

空白（デフォルト）

blurDataURL="..."

カスタムプレースホルダー（リモート画像用）

リモート画像でのBlur効果

外部URLから取得する画像の場合は、blurDataURLを手動で指定します。

リモート画像のBlur 
<Image
  src="https://example.com/image.jpg"
  alt="リモート画像"
  width={800}
  height={600}
  placeholder="blur"
  blurDataURL="data:image/jpeg;base64,/9j/4AAQSkZJRg..." // Base64エンコード
/>

8.5 外部画像の設定

外部URLから画像を取得する場合、セキュリティのためnext.config.jsで許可ドメインを設定する必要があります。

next.config.js 
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'example.com',
        port: '',              // 空文字 = デフォルトポート
        pathname: '/images/**', // パスパターン
      },
      {
        protocol: 'https',
        hostname: 'cdn.example.com', // 複数ドメイン可
      },
    ],
  },
};

module.exports = nextConfig;

セキュリティ上の注意

信頼できるドメインのみを許可リストに追加してください。悪意のあるサイトからの画像読み込みを防ぐため、ワイルドカード（*）の使用は避けましょう。

8.6 next/fontによるフォント最適化

next/fontを使うと、フォントをビルド時に自動最適化し、外部リクエストなしでセルフホストできます。これにより、レイアウトシフトを完全に防ぎ、プライバシーも向上します。

flowchart LR A["従来のGoogle Fonts"] --> B["問題点"] B --> B1["外部リクエスト
速度低下"] B --> B2["FOUT/FOIT
レイアウトシフト"] B --> B3["プライバシー懸念
外部サーバーへ情報送信"] C["next/font"] --> D["利点"] D --> D1["ビルド時に
ローカル化"] D --> D2["自動サイズ調整
レイアウトシフトゼロ"] D --> D3["プライバシー保護
外部送信なし"]

Google Fontsの使用

app/layout.tsx 
import { Inter, Noto_Sans_JP } from 'next/font/google';

// Interフォントの読み込み
const inter = Inter({
  subsets: ['latin'],
  display: 'swap',  // フォント読み込み戦略
});

// Noto Sans JPの読み込み
const notoSansJP = Noto_Sans_JP({
  weight: ['400', '700'],  // 必要なウェイトのみ
  subsets: ['latin'],
  display: 'swap',
});

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="ja" className={notoSansJP.className}>
      <body>{children}</body>
    </html>
  );
}

displayプロパティ

display: 'swap'を指定すると、フォント読み込み中でもフォールバックフォントでテキストを表示し、読み込み完了後にスムーズに切り替わります。

8.7 カスタムフォントの追加

Google Fonts以外のカスタムフォントも、next/fontを使って最適化できます。

実習 8-2: ローカルフォントの追加

プロジェクトにカスタムフォントを追加して使用します。

手順

  1. 1 public/fonts/にフォントファイル（.woff2等）を配置
  2. 2 next/font/localを使ってフォントを読み込む

コード例

app/fonts.ts 
import localFont from 'next/font/local';

// カスタムフォントの読み込み
export const myFont = localFont({
  src: [
    {
      path: '../public/fonts/MyFont-Regular.woff2',
      weight: '400',
      style: 'normal',
    },
    {
      path: '../public/fonts/MyFont-Bold.woff2',
      weight: '700',
      style: 'normal',
    },
  ],
  display: 'swap',
  variable: '--font-my-font', // CSS変数として使用
});

使用例

app/layout.tsx 
import { myFont } from './fonts';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="ja" className={myFont.variable}>
      <body>{children}</body>
    </html>
  );
}

8.8 パフォーマンス改善のベストプラクティス

画像とフォント最適化のベストプラクティスをまとめます。

画像最適化のポイント

  • ✅ LCP画像にはpriorityを指定
  • ✅ レスポンシブ画像にはsizesを設定
  • ✅ ローカル画像はimportして使用
  • ✅ プレースホルダーで読み込み体験を改善
  • ✅ 適切な画像フォーマット（WebP/AVIF）を使用

フォント最適化のポイント

  • ✅ next/fontでフォントをセルフホスト
  • ✅ 必要なウェイトのみを読み込む
  • ✅ display: 'swap'でFOUTを防ぐ
  • ✅ CSS変数を使って柔軟に適用
  • ✅ preloadでクリティカルフォントを優先

Core Web Vitals改善のチェックリスト

最適化チェックリスト

  • LCP（Largest Contentful Paint）：ファーストビューの大きな画像にpriority指定
  • CLS（Cumulative Layout Shift）：width/heightを必ず指定してレイアウトシフトを防ぐ
  • FID（First Input Delay）：画像の遅延読み込みでJavaScriptの実行を妨げない
  • 帯域幅削減：適切なsizes指定で無駄な大きい画像をダウンロードさせない

理解度確認クイズ

  1. Q1. next/imageコンポーネントの主な利点を3つ挙げてください。

    回答を表示

    ①デバイスに応じた自動リサイズとフォーマット変換（WebP/AVIF）、②遅延読み込みによる初回ロード高速化、③width/height指定によるレイアウトシフト防止。これらにより、Core Web Vitalsが改善されます。

  2. Q2. priorityプロパティはどのような場合に使用すべきですか？

    回答を表示

    ファーストビュー（初期表示範囲）に表示される大きな画像、特にLCP（Largest Contentful Paint）を構成する重要な画像に使用します。priorityを指定すると遅延読み込みが無効化され、即座に読み込まれます。

  3. Q3. next/fontを使うことで得られるメリットは何ですか？

    回答を表示

    ①ビルド時にフォントをセルフホスト化することで外部リクエストを削減、②自動的にサイズ調整属性を追加してレイアウトシフトを防止、③Google Fontsへのリクエストがないためプライバシー保護になる、④パフォーマンスとセキュリティが向上します。

  • next/imageは画像の自動最適化（リサイズ、フォーマット変換、遅延読み込み）を提供し、Core Web Vitalsを改善する
  • レスポンシブ画像ではfillとsizesを活用し、デバイスごとに最適なサイズを配信する
  • プレースホルダーとBlur効果により、画像読み込み中のユーザー体験を向上させる
  • next/fontでフォントをセルフホスト化し、レイアウトシフトを防ぎながらプライバシーとパフォーマンスを向上させる
