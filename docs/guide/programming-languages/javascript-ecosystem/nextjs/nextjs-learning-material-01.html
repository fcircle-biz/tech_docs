Next.jsとは？フレームワークの全体像を理解しよう

Reactをベースとしたフルスタックフレームワーク「Next.js」の概要、解決する課題、コア機能、そしてモダンWeb開発における位置づけを学びます。

この章で学ぶこと

  • Next.jsが解決する従来のReactアプリケーションの課題
  • レンダリング戦略（CSR、SSR、SSG、ISR）の違いと使い分け
  • Pages RouterとApp Routerの比較とNext.jsの進化
  • Next.jsプロジェクトの初期セットアップと基本構成

1.1 Next.jsとは何か？

Next.jsは、Vercel社が開発するReactベースのフルスタックWebフレームワークです。2016年の登場以来、モダンWeb開発のデファクトスタンダードとして急速に普及してきました。

Reactが提供するUI構築の柔軟性に加えて、以下の機能を「ゼロコンフィグ」で利用できることが最大の特徴です：

Next.jsの主要機能

  • ファイルベースルーティング：フォルダ構造がそのままURLになる直感的な設計
  • サーバーサイドレンダリング（SSR）：初回ロード時にサーバーでHTMLを生成してSEO向上
  • 静的サイト生成（SSG）：ビルド時に静的HTMLを事前生成して高速配信
  • API Routes：フロントエンドと同じリポジトリでバックエンドAPIを構築
  • 画像・フォント最適化：自動的にリサイズ、フォーマット変換、遅延読み込みを実行
  • TypeScript完全サポート：型安全性とDXを標準で提供

1.2 Next.jsが解決する課題

従来のReact（CRA: Create React App）では、以下のような課題がありました：

従来のReact（CRA）の課題

  • 初回ロードが遅い（JavaScript読み込み後にレンダリング）
  • SEOに弱い（クローラーがJavaScriptを実行しにくい）
  • ルーティングライブラリの手動設定が必要
  • バックエンドは別途構築が必要
  • パフォーマンス最適化に専門知識が必要

Next.jsによる解決

  • SSR/SSGで初回ロード高速化 & SEO対応
  • 完全なHTMLをクローラーに提供可能
  • ファイルベースルーティングで設定不要
  • API Routesでフルスタック開発が可能
  • 画像最適化やコード分割が自動適用

1.3 レンダリング戦略の理解

Next.jsの最大の強みは、ページごとに最適なレンダリング戦略を選択できる点です。主要な4つの戦略を理解しましょう。

レンダリング戦略比較表

戦略 レンダリングタイミング 速度 SEO 適用例
CSR ブラウザ実行時 ⚠️ 初回遅い ❌ 弱い 管理画面、ダッシュボード
SSR リクエスト毎 ⚡ 速い ✅ 強い ニュース、ユーザー専用ページ
SSG ビルド時 🚀 最速 ✅ 強い ブログ、ドキュメント、LP
ISR ビルド時 + 定期更新 🚀 最速 ✅ 強い Eコマース商品ページ、記事一覧
CSR（Client-Side Rendering）

ブラウザ上でJavaScriptを実行してHTMLを生成する従来のReactの方式。初回ロードでは空のHTMLが返され、その後JavaScriptがダウンロード・実行されてUIが描画されます。

使い所：管理画面やダッシュボードなど、SEOが不要で認証後にアクセスするページ。

SSR（Server-Side Rendering）

リクエスト毎にサーバーでHTMLを生成してクライアントに返す方式。ユーザーがページにアクセスするたびに最新データを使ってHTMLが作られます。

使い所：ニュースサイト、SNSのタイムラインなど、頻繁に更新されるコンテンツ。

SSG（Static Site Generation）

ビルド時に全てのページをHTMLとして事前生成し、CDNから高速配信する方式。コンテンツが変わらない限り再ビルド不要。

使い所：ブログ、ドキュメント、ランディングページなど、変更頻度が低いコンテンツ。

ISR（Incremental Static Regeneration）

SSGの進化版で、ビルド時に静的生成しつつ、指定した間隔（例：60秒）でバックグラウンドで再生成する方式。全体を再ビルドせずに、個別ページだけを更新できます。

使い所：Eコマースの商品ページ、ブログ記事一覧など、定期的に更新されるが毎リクエストでの再生成は不要なコンテンツ。

1.4 Pages Router vs App Router

Next.js 13以降では、新しいApp Routerが導入されました。従来のPages Routerも引き続き利用可能ですが、これから学習する場合はApp Routerを推奨します。

Pages Router（従来） 

pages/
├── index.js         → /
├── about.js         → /about
└── blog/
    ├── [id].js      → /blog/:id
    └── index.js     → /blog
  • ✅ シンプルで学習しやすい
  • ✅ 豊富な実例とコミュニティ
  • ⚠️ React Server Componentsが使えない
  • ⚠️ レイアウト共有が煩雑

App Router（推奨・最新） 

app/
├── page.js          → /
├── about/
│   └── page.js      → /about
└── blog/
    ├── [id]/
    │   └── page.js  → /blog/:id
    └── page.js      → /blog
  • ✅ React Server Components対応
  • ✅ レイアウト・ローディング状態を簡単に共有
  • ✅ ストリーミングSSRでパフォーマンス向上
  • ✅ Next.jsの将来性を見据えた設計

このガイドでの方針

本教材では、App Routerを中心に学習を進めます。ただし、既存プロジェクトではPages Routerも多く使われているため、両者の違いを理解することが重要です。

1.5 Next.jsエコシステムと主要機能

Next.jsは単なるReactフレームワークではなく、フルスタック開発を支える豊富なエコシステムを提供しています。

パフォーマンス最適化

  • next/image: 自動リサイズ、WebP変換、遅延読み込み
  • next/font: フォントの最適読み込み、レイアウトシフト防止
  • next/script: サードパーティスクリプト最適化
  • 自動コード分割とTree Shaking

バックエンド機能

  • API Routes: RESTful API構築
  • Server Actions: フォーム送信をサーバー関数で処理
  • Middleware: 認証、リダイレクト、ヘッダー操作
  • Edge Runtime: エッジロケーションでの高速実行

1.6 プロジェクトの初期セットアップ

Next.jsプロジェクトを作成してみましょう。公式のcreate-next-appを使うと、TypeScript、ESLint、Tailwind CSSなどの設定が自動的に完了します。

実習 1-1: Next.jsプロジェクトを作成する

Next.jsの公式CLIツールを使って、新しいプロジェクトを作成します。

手順

  1. 1 ターミナルで以下のコマンドを実行してプロジェクトを作成
  2. 2 対話形式で表示される質問に答える（TypeScript: Yes、Tailwind CSS: Yesを推奨）
  3. 3 プロジェクトフォルダに移動して開発サーバーを起動

実行例

terminal 
# プロジェクト作成
npx create-next-app@latest my-next-app

# プロジェクトフォルダに移動
cd my-next-app

# 開発サーバー起動
npm run dev

期待される結果

開発サーバーが起動し、ブラウザで http://localhost:3000 にアクセスすると、Next.jsのウェルカムページが表示されます。

生成されるプロジェクト構成（App Router） 

my-next-app/
├── app/                    # App Router のルートフォルダ
│   ├── layout.tsx          # ルートレイアウト（全ページ共通）
│   ├── page.tsx            # トップページ（/）
│   └── globals.css         # グローバルCSS
├── public/                 # 静的ファイル（画像、フォント等）
├── node_modules/           # 依存パッケージ
├── package.json            # プロジェクト設定
├── tsconfig.json           # TypeScript設定
├── next.config.js          # Next.js設定
└── tailwind.config.ts      # Tailwind CSS設定

重要なファイル

  • app/page.tsx: 各フォルダの page.tsx がルートになります
  • app/layout.tsx: レイアウトを定義し、子ページに共通UIを提供
  • next.config.js: Next.jsの設定ファイル（画像ドメイン許可など）

理解度確認クイズ

  1. Q1. Next.jsの主要な利点を3つ挙げてください。

    回答を表示

    ①ファイルベースルーティングによる直感的なページ管理、②SSR/SSGによる初回ロード高速化とSEO対応、③API Routesによるフルスタック開発が可能。

  2. Q2. SSRとSSGの違いを説明してください。

    回答を表示

    SSR（Server-Side Rendering）はリクエスト毎にサーバーでHTMLを生成し、常に最新データを表示できます。一方、SSG（Static Site Generation）はビルド時に全てのページを事前生成し、CDNから高速配信しますが、コンテンツ更新には再ビルドが必要です。

  3. Q3. App RouterとPages Routerの主な違いは何ですか？

    回答を表示

    App RouterはReact Server Componentsに対応し、レイアウトやローディング状態の共有が容易で、ストリーミングSSRによるパフォーマンス向上が可能です。Pages Routerはシンプルで学習しやすいですが、これらの機能は利用できません。Next.js 13以降ではApp Routerが推奨されます。

この章のまとめ

  • Next.jsはReactベースのフルスタックフレームワークで、SEO、パフォーマンス、開発体験を大幅に向上させる
  • CSR、SSR、SSG、ISRの4つのレンダリング戦略を理解し、適切に使い分けることが重要
  • App Routerは最新のNext.jsの中核機能で、React Server Componentsやストリーミングに対応
  • create-next-appを使えば、TypeScriptやTailwind CSSを含む最適な開発環境を即座に構築できる
