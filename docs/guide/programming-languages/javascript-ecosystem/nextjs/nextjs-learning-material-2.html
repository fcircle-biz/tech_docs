第2章: Reactの基礎とコンポーネント設計

Reactコンポーネントの世界

この章で学ぶこと

  • 関数コンポーネントとJSX構文の使い方を習得する
  • React Hooks（useState、useEffect）の基礎を学ぶ
  • 再利用可能なコンポーネントの設計方法を理解する
  • 条件付きレンダリングとリストの表示方法を習得する

2.1 Reactの基本概念

2.1.1 Reactとは

Reactは、Facebookが開発したJavaScriptライブラリで、ユーザーインターフェース（UI）を構築するための部品化を中心とした考え方を持っています。

flowchart TB A["Webページ"] --> B["ヘッダー
コンポーネント"] A --> C["サイドバー
コンポーネント"] A --> D["メイン
コンポーネント"] A --> E["フッター
コンポーネント"] D --> F["カード
コンポーネント"] D --> G["ボタン
コンポーネント"]

Reactの3つの核心概念：

  • コンポーネント：UIを部品として分割し、再利用可能にする
  • Props：親コンポーネントから子コンポーネントへデータを渡す仕組み
  • State：コンポーネント内で管理される動的なデータ

2.1.2 コンポーネントとは

コンポーネントは、UIの一部を表す再利用可能な部品です。ボタン、カード、フォームなど、あらゆるUIをコンポーネントとして作成できます。

コンポーネントの利点
  • 再利用性：一度作成すれば、複数の場所で使える
  • 保守性：コードを部品に分割することで、管理しやすくなる
  • 独立性：各コンポーネントは独立して動作する

2.2 関数コンポーネントとJSX

2.2.1 最もシンプルなコンポーネント

Reactコンポーネントは、JavaScriptの関数として定義します：

// 最もシンプルなコンポーネント
function Welcome() {
  return <h1>Hello, React!</h1>;
}

export default Welcome;

このコードの要素：

  • function Welcome()：関数コンポーネントの定義（関数名は大文字で始める）
  • return：このコンポーネントが表示する内容を返す
  • <h1>...</h1>：JSX（JavaScript XML）構文
  • export default：他のファイルから使用できるようにエクスポート

2.2.2 JSX構文とは

JSXは、JavaScriptの中にHTMLのような構文を書ける特別な記法です。実際にはJavaScriptにコンパイルされます。 

// JSXの例
const element = <h1>Hello, World!</h1>;

// これは以下のJavaScriptコードにコンパイルされる
const element = React.createElement('h1', null, 'Hello, World!');

2.2.3 JSXの重要なルール

ルール 説明 例
単一ルート要素 returnは1つの要素だけを返す <div>...</div>または<>...</>
タグは閉じる すべてのタグは閉じる必要がある <img />、<br />
classはclassName HTMLのclass属性はclassNameで指定 <div className="box">
{}で式を埋め込む JavaScriptの式を{}で囲んで埋め込める {name}、{2 + 3}

2.3 Props：コンポーネント間のデータ受け渡し

2.3.1 Propsとは

Propsは、親コンポーネントから子コンポーネントへデータを渡す仕組みです。HTMLの属性のように値を渡します。

flowchart LR A["親コンポーネント"] -->|"Props
(name, age)"| B["子コンポーネント"] B --> C["表示"]

2.3.2 Propsの基本的な使い方

// Propsを受け取るコンポーネント
function Greeting(props) {
  return <h1>こんにちは、{props.name}さん！</h1>;
}

// 使用例
<Greeting name="太郎" />
<Greeting name="花子" />

// 出力：
// こんにちは、太郎さん！
// こんにちは、花子さん！

2.3.3 分割代入でよりシンプルに

ES6の分割代入を使うと、propsからプロパティを直接取り出せます： 

// 分割代入を使った書き方（推奨）
function Greeting({ name }) {
  return <h1>こんにちは、{name}さん！</h1>;
}

// 複数のPropsを受け取る
function UserCard({ name, age, job }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>年齢: {age}歳</p>
      <p>職業: {job}</p>
    </div>
  );
}

2.4 State：動的なデータ管理

2.4.1 Stateとは

Stateは、コンポーネント内で管理される変化するデータです。Stateが変更されると、Reactは自動的にコンポーネントを再レンダリング（再描画）します。

特徴 Props State
データの流れ 親から子へ コンポーネント内部
変更可能性 読み取り専用 変更可能
用途 コンポーネントの設定 動的なデータ管理

2.4.2 useStateフックの基本

Stateを使用するには、ReactのuseStateフックを使います： 

import { useState } from 'react';

function Counter() {
  // State変数の宣言
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        +1
      </button>
    </div>
  );
}

useStateの構文：

  • useState(初期値)：Stateフックを呼び出す
  • count：現在のState値
  • setCount：Stateを更新する関数
  • 0：Stateの初期値
flowchart LR A["ボタンクリック"] --> B["setCount実行"] B --> C["Stateが更新"] C --> D["コンポーネント
再レンダリング"] D --> E["画面更新"]
実習 2-3: カウンターコンポーネントの作成

useStateを使った簡単なカウンターを作成します。

// app/components/Counter.js
'use client';  // クライアントコンポーネントとして指定

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div style={{
      border: '2px solid #667eea',
      borderRadius: '8px',
      padding: '20px',
      maxWidth: '300px',
      textAlign: 'center'
    }}>
      <h2>カウンター</h2>
      <p style={{ fontSize: '48px', margin: '20px 0' }}>{count}</p>
      <button
        onClick={() => setCount(count + 1)}
        style={{ marginRight: '10px' }}
      >
        +1
      </button>
      <button
        onClick={() => setCount(count - 1)}
        style={{ marginRight: '10px' }}
      >
        -1
      </button>
      <button onClick={() => setCount(0)}>
        リセット
      </button>
    </div>
  );
}

export default Counter;
重要：'use client'ディレクティブ

Next.jsのApp Routerでは、useStateなどのフックを使用するコンポーネントには、ファイル先頭に'use client'を記述する必要があります。これについては第4章で詳しく学びます。

2.5 useEffectフック：副作用の処理

2.5.1 useEffectとは

useEffectは、コンポーネントのレンダリング後に副作用（データ取得、タイマー、DOM操作など）を実行するためのフックです。 

import { useState, useEffect } from 'react';

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    // 1秒ごとにsecondsを+1
    const interval = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);

    // クリーンアップ関数（コンポーネントがアンマウントされる時に実行）
    return () => clearInterval(interval);
  }, []); // 空の依存配列 = 初回のみ実行

  return <p>経過時間: {seconds}秒</p>;
}

2.5.2 useEffectの依存配列

依存配列 動作 用途
なし 毎回のレンダリング後に実行 あまり推奨されない
[]（空配列） 初回レンダリング後のみ実行 データ取得、タイマー設定
[value] valueが変更された時に実行 特定の値に応じた処理

2.6 条件付きレンダリング

2.6.1 if文を使った条件分岐

function LoginButton({ isLoggedIn }) {
  if (isLoggedIn) {
    return <button>ログアウト</button>;
  }
  return <button>ログイン</button>;
}

2.6.2 三項演算子を使った条件分岐

function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? (
        <h1>おかえりなさい！</h1>
      ) : (
        <h1>ログインしてください</h1>
      )}
    </div>
  );
}

2.6.3 &&演算子を使った条件表示

function Notification({ hasNewMessage, messageCount }) {
  return (
    <div>
      {hasNewMessage && (
        <p>新しいメッセージが{messageCount}件あります</p>
      )}
    </div>
  );
}

2.7 リストのレンダリング

2.7.1 mapを使った配列の表示

function FruitList() {
  const fruits = ['りんご', 'バナナ', 'オレンジ', 'ぶどう'];

  return (
    <ul>
      {fruits.map((fruit, index) => (
        <li key={index}>{fruit}</li>
      ))}
    </ul>
  );
}
keyプロパティの重要性

リストをレンダリングする際は、必ずkeyプロパティを指定してください。keyはReactがどの要素が変更されたかを識別するために使用します。可能であれば、インデックスではなく一意のIDを使用することが推奨されます。

2.7.2 オブジェクト配列の表示

function UserList() {
  const users = [
    { id: 1, name: '太郎', age: 25 },
    { id: 2, name: '花子', age: 23 },
    { id: 3, name: '次郎', age: 28 }
  ];

  return (
    <div>
      {users.map(user => (
        <div key={user.id} style={{ margin: '10px', padding: '10px', border: '1px solid #ccc' }}>
          <h3>{user.name}</h3>
          <p>年齢: {user.age}歳</p>
        </div>
      ))}
    </div>
  );
}
理解度確認クイズ
  1. PropsとStateの違いを説明してください。
    回答例：Propsは親コンポーネントから受け取る読み取り専用のデータ。Stateはコンポーネント内部で管理される変更可能なデータで、更新されると再レンダリングが発生する。
  2. useStateフックで返される配列の2つの要素は何ですか？
    回答：第1要素は現在のState値、第2要素はStateを更新する関数
  3. リストをレンダリングする際に必要な属性は何ですか？
    回答：key属性（各要素を一意に識別するため）
