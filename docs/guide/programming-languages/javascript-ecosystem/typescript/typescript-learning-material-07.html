繰り返し処理：同じことを効率的にやろう

for文やwhile文を使った繰り返し処理の方法を学びます。型を使って配列の要素を安全に処理する方法や、カウンター変数の適切な使い方を実践的に習得しましょう。

この章で学ぶこと

  • for文とwhile文を使った繰り返し処理の基本を理解する
  • 型を使って配列の要素を安全に処理する方法を学ぶ
  • カウンター変数に適切な型を指定し、安全なループを書けるようになる
  • break/continue文を使ったループ制御の方法を習得する

7.1 繰り返し処理って何だろう？

プログラミングでは、同じ処理を何度も繰り返すことがよくあります。 例えば、「1から10までの数字をすべて表示する」「リスト内のすべての商品に対して処理を行う」といった場合です。

日常生活でも繰り返し処理は身近です。例えば：

  • 洗濯物を一枚ずつ干していく
  • 試験問題を1問ずつ解いていく
  • 友達全員にメッセージを送る

こうした「繰り返し」をコンピュータに効率的にやってもらうのが繰り返し処理（ループ）です。

繰り返し処理の威力

手作業で100回同じことを書く代わりに、たった数行のコードで済むのが繰り返し処理の魅力です。 コンピュータは疲れないので、何千回・何万回でも正確に繰り返せます。

7.2 for文の基本：決まった回数繰り返す

for文は、「あらかじめ決まった回数だけ繰り返す」ときに使います。 例えば、「1から5まで数える」「配列の要素をすべて処理する」といった場合です。

for文の基本構造

for (初期化; 条件式; 更新式) {

繰り返したい処理

}

基本的なfor文の例 
// 1から5まで表示する
for (let i: number = 1; i <= 5; i++) {
  console.log(i);
}
// 出力: 1, 2, 3, 4, 5

コードの詳細解説

  • let i: number = 1 カウンター変数iを数値型で初期化（1から開始）
  • i <= 5 iが5以下の間、繰り返しを続ける条件
  • i++ ループごとにiを1増やす（i = i + 1と同じ）

型指定のメリット

カウンター変数iにnumber型を指定することで、 間違って文字列を代入したり、計算ミスをしたりした場合にエラーで教えてくれます。

7.3 配列と繰り返し処理：要素を一つずつ処理する

配列（複数のデータをまとめたもの）の各要素に対して処理を行うとき、for文は非常に便利です。 TypeScriptでは、配列に型を指定することで、安全に要素を扱えます。

配列の繰り返し処理 
// 果物のリスト（文字列配列）
let fruits: string[] = ["りんご", "バナナ", "オレンジ"];

// 配列の要素をすべて表示
for (let i: number = 0; i < fruits.length; i++) {
  console.log(`${i + 1}番目: ${fruits[i]}`);
}

// 出力:
// 1番目: りんご
// 2番目: バナナ
// 3番目: オレンジ

for...of文：より簡潔な書き方

配列のすべての要素を順番に処理したい場合、for...of文を使うとより簡潔に書けます。

for...of文の例 
let fruits: string[] = ["りんご", "バナナ", "オレンジ"];

// for...of文：インデックス番号を気にせず要素を取得
for (let fruit of fruits) {
  console.log(`好きな果物: ${fruit}`);
}

// 出力:
// 好きな果物: りんご
// 好きな果物: バナナ
// 好きな果物: オレンジ

7.4 while文：条件が満たされる間繰り返す

while文は、「条件が満たされている間ずっと繰り返す」ときに使います。 for文は「何回繰り返すか」が明確な場合に向いていますが、while文は「いつ終わるか分からないけど条件を満たす間続ける」という場合に便利です。

while文の基本構造

while (条件式) {

繰り返したい処理

}

while文の例 
// カウントダウンの例
let count: number = 5;

while (count > 0) {
  console.log(`カウント: ${count}`);
  count--; // countを1減らす
}

console.log("発射！");

// 出力:
// カウント: 5
// カウント: 4
// カウント: 3
// カウント: 2
// カウント: 1
// 発射！

無限ループに注意！

while文では、条件式が永遠に真(true)にならないよう注意が必要です。 例えばcount--を忘れると、カウントが減らないため永遠にループし続けます。

do...while文：最低1回は実行する

do...while文は、条件チェックの前に必ず1回は処理を実行します。

do...while文の例 
let answer: number = 0;

do {
  console.log("もう一度挑戦しますか？");
  answer++;
} while (answer < 3);

console.log(`${answer}回実行しました`);
// 出力: 3回実行しました

7.5 break と continue：ループを制御する

繰り返し処理の途中で「ループを抜けたい」「この回はスキップしたい」という場合があります。 そんなときに使うのがbreakとcontinueです。

break

ループを完全に抜ける（終了する）

continue

この回の処理をスキップして、次のループへ進む

breakの例：特定の値が見つかったら終了 
let numbers: number[] = [1, 3, 5, 7, 8, 9, 11];

// 偶数を探す（最初に見つかったら終了）
for (let num of numbers) {
  if (num % 2 === 0) {
    console.log(`偶数が見つかりました: ${num}`);
    break; // ここでループを抜ける
  }
}

// 出力: 偶数が見つかりました: 8
continueの例：特定の値をスキップ 
let numbers: number[] = [1, 2, 3, 4, 5];

// 偶数だけスキップして奇数だけ表示
for (let num of numbers) {
  if (num % 2 === 0) {
    continue; // 偶数の場合、この回はスキップ
  }
  console.log(`奇数: ${num}`);
}

// 出力:
// 奇数: 1
// 奇数: 3
// 奇数: 5

よくあるエラーと対処法

エラー: 無限ループでプログラムが止まらない

原因: ループの条件が永遠に真(true)のまま
対処: カウンターの更新（i++やcount--）を忘れていないか確認

エラー: 配列の範囲外にアクセスしている

原因: インデックスが配列の長さを超えている
対処: 条件式をi < array.lengthにする

エラー: 型エラーが出る

原因: カウンター変数や配列の型が正しくない
対処: カウンターはnumber型、配列は型[]で指定

理解度確認クイズ

  1. Q1. for文とwhile文の使い分けは？

    回答を表示

    for文: 繰り返す回数が明確な場合（例：配列の全要素を処理、1から10まで表示）
    while文: 条件が満たされる間繰り返す場合（例：目標値に達するまで、ユーザーが終了するまで）

  2. Q2. breakとcontinueの違いは？

    回答を表示

    break: ループを完全に抜けて終了する
    continue: 現在の回の処理をスキップして、次のループへ進む

  3. Q3. 配列の繰り返し処理で型を指定するメリットは？

    回答を表示

    配列にstring[]やnumber[]のように型を指定すると、 取り出した要素が必ずその型であることが保証されます。これにより、間違った型の操作（例：数値配列に文字列メソッドを使う）を防げます。

この章のまとめ

  • for文は決まった回数繰り返す場合に使い、カウンター変数に型を指定できる
  • while文は条件が満たされる間繰り返し、無限ループに注意が必要
  • 配列と繰り返し処理を組み合わせることで、複数のデータを効率的に処理できる
  • breakでループを終了、continueで現在の処理をスキップできる
  • 型指定により、配列の要素やカウンター変数を安全に扱える
