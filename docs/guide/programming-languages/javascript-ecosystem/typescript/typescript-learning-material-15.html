これまで学んだ知識を総動員して、型安全なタスク管理アプリケーションを段階的に作成します。インターフェース定義から実装、エラー処理まで、実践的な開発の流れを体験しましょう。

15.1 アプリケーションの設計：何を作るか

この章では、タスク管理アプリを作成します。このアプリでは、以下の機能を実装します：

タスク管理アプリの機能

  • タスクの追加 - 新しいタスクを作成できる
  • タスクの一覧表示 - すべてのタスクを確認できる
  • タスクの完了/未完了の切り替え - タスクの状態を管理できる
  • タスクの削除 - 不要なタスクを削除できる
  • 優先度の設定 - タスクに優先度を付けられる

設計を先に考える重要性

コードを書き始める前に、「どんなデータが必要か」「どんな操作が必要か」を考えることで、 型定義を使った安全な設計ができます。これが TypeScript の強みです。

15.2 データ構造の設計：型定義から始めよう

まず、タスクのデータ構造を型として定義します。これにより、どんな情報を扱うかが明確になります。

優先度の型定義 
// 優先度を表す型（ユニオン型を使用）
type Priority = "low" | "medium" | "high";

// 優先度は3つの値のいずれかしか取れない
// これにより、タイプミスや間違った値の使用を防げる
タスクのインターフェース定義 
// タスクの構造を定義するインターフェース
interface Task {
  id: number;          // タスクを識別するユニークなID
  title: string;       // タスクのタイトル
  completed: boolean;  // 完了状態（true=完了、false=未完了）
  priority: Priority;  // 優先度（low, medium, high）
  createdAt: Date;     // 作成日時
}

型定義の利点

インターフェースを定義しておくことで、タスクを扱うすべてのコードで同じ構造を保証できます。 また、エディタの補完機能も使えるようになります。

15.3 タスク管理クラスの実装

次に、タスクを管理するクラスを実装します。このクラスは、タスクの追加、削除、完了状態の変更などの機能を持ちます。

TaskManagerクラスの実装 
class TaskManager {
  // タスクを保存する配列（privateで外部から直接アクセスできない）
  private tasks: Task[] = [];
  // 次に割り当てるID（自動インクリメント）
  private nextId: number = 1;

  // タスクを追加するメソッド
  addTask(title: string, priority: Priority): Task {
    const newTask: Task = {
      id: this.nextId++,
      title: title,
      completed: false,
      priority: priority,
      createdAt: new Date()
    };
    this.tasks.push(newTask);
    return newTask; // 追加したタスクを返す
  }
}

このクラスのaddTaskメソッドは、 タイトルと優先度を受け取り、新しいタスクを作成して配列に追加します。

15.4 その他の機能を追加する

タスクの取得、完了状態の変更、削除などの機能を追加していきます。

TaskManagerクラスに機能を追加 
class TaskManager {
  // ... 前のコードに続けて追加 ...

  // すべてのタスクを取得
  getAllTasks(): Task[] {
    return [...this.tasks]; // 配列のコピーを返す
  }

  // 完了状態を切り替える
  toggleTask(id: number): boolean {
    const task = this.tasks.find(t => t.id === id);
    if (task) {
      task.completed = !task.completed;
      return true; // 成功
    }
    return false; // タスクが見つからない
  }

  // タスクを削除
  deleteTask(id: number): boolean {
    const index = this.tasks.findIndex(t => t.id === id);
    if (index !== -1) {
      this.tasks.splice(index, 1);
      return true; // 成功
    }
    return false; // タスクが見つからない
  }
}
メソッド 機能 戻り値
getAllTasks() すべてのタスクを取得 Task[]
toggleTask(id) 完了状態を切り替え boolean（成功/失敗）
deleteTask(id) タスクを削除 boolean（成功/失敗）

配列のコピーを返す理由

getAllTasks()で [...this.tasks] としているのは、元の配列を外部から直接変更されないようにするためです。これを「カプセル化」と呼びます。

15.5 エラーハンドリングの追加

実際のアプリケーションでは、エラーが発生することを想定し、適切に処理する必要があります。 タスクの追加時に、タイトルが空でないかチェックする機能を追加しましょう。

エラーハンドリングの実装 
class TaskManager {
  // ... 前のコードに続けて書き換え ...

  // タスクを追加（エラーチェック付き）
  addTask(title: string, priority: Priority): Task {
    // タイトルが空またはスペースのみの場合はエラー
    if (!title.trim()) {
      throw new Error("タスクのタイトルを入力してください");
    }

    const newTask: Task = {
      id: this.nextId++,
      title: title.trim(), // 前後の空白を削除
      completed: false,
      priority: priority,
      createdAt: new Date()
    };
    this.tasks.push(newTask);
    return newTask;
  }
}

エラーハンドリングの重要性

ユーザーの入力を受け付ける場合、予期しない値が来る可能性を常に考慮しましょう。 適切なエラーチェックを行うことで、バグを防ぎ、ユーザーにわかりやすいメッセージを提供できます。

15.6 フィルター機能の追加

完了済みのタスクだけ、未完了のタスクだけ、特定の優先度のタスクだけを取得できるフィルター機能を追加します。

フィルター機能の実装 
class TaskManager {
  // ... 前のコードに続けて追加 ...

  // 完了したタスクのみ取得
  getCompletedTasks(): Task[] {
    return this.tasks.filter(task => task.completed);
  }

  // 未完了のタスクのみ取得
  getIncompleteTasks(): Task[] {
    return this.tasks.filter(task => !task.completed);
  }

  // 特定の優先度のタスクを取得
  getTasksByPriority(priority: Priority): Task[] {
    return this.tasks.filter(task => task.priority === priority);
  }
}

配列のfilterメソッド

filterは、 条件に合う要素だけを抽出した新しい配列を返すメソッドです。元の配列は変更されません。

15.7 完全なコード例と使い方

これまで作成したクラスを実際に使ってみましょう。

タスク管理アプリの使用例 
// タスク管理アプリのインスタンスを作成
const manager = new TaskManager();

// タスクを追加
try {
  manager.addTask("TypeScriptの学習", "high");
  manager.addTask("買い物に行く", "medium");
  manager.addTask("部屋の掃除", "low");
  console.log("タスクを3つ追加しました");
} catch (error) {
  console.error("エラー:", error);
}

// すべてのタスクを表示
const allTasks = manager.getAllTasks();
console.log("\n=== すべてのタスク ===");
allTasks.forEach(task => {
  console.log(`[${task.id}] ${task.title} (優先度: ${task.priority})`);
});

// タスクを完了にする
manager.toggleTask(1);
console.log("\nタスク1を完了にしました");

// 未完了のタスクを表示
const incompleteTasks = manager.getIncompleteTasks();
console.log("\n=== 未完了のタスク ===");
incompleteTasks.forEach(task => {
  console.log(`[${task.id}] ${task.title}`);
});

15.8 発展：LocalStorageへの保存

ブラウザで動かす場合、LocalStorageを使ってタスクを保存・読み込みできるようにすると、 ページをリロードしてもデータが残るようになります。

LocalStorageへの保存機能 
class TaskManager {
  // ... 前のコードに続けて追加 ...

  // タスクをLocalStorageに保存
  saveTasks(): void {
    const tasksJSON = JSON.stringify(this.tasks);
    localStorage.setItem("tasks", tasksJSON);
    console.log("タスクを保存しました");
  }

  // LocalStorageからタスクを読み込み
  loadTasks(): void {
    const tasksJSON = localStorage.getItem("tasks");
    if (tasksJSON) {
      this.tasks = JSON.parse(tasksJSON);
      // 最大IDを復元
      const maxId = Math.max(...this.tasks.map(t => t.id), 0);
      this.nextId = maxId + 1;
      console.log("タスクを読み込みました");
    }
  }
}

LocalStorageとは

LocalStorageは、ブラウザにデータを保存できる機能です。 JSON.stringify()でオブジェクトを文字列に変換して保存し、 JSON.parse()で読み込み時に元に戻します。

よくあるエラーと対処法

エラー: "Property 'xxx' does not exist"

原因：インターフェースで定義されていないプロパティにアクセスしようとしています。
対処：インターフェース定義を確認し、必要なプロパティを追加してください。

エラー: "Cannot read property of undefined"

原因：存在しないタスクにアクセスしようとしています。
対処：find()の結果が undefinedでないか確認してから使用してください。

理解度確認クイズ

  1. Q1. インターフェースを使ってデータ構造を定義する利点は何ですか？

    回答を表示

    すべてのコードで同じ構造を保証でき、型チェックによって間違いを早期に発見できます。 また、エディタの補完機能が使えるようになり、開発効率が向上します。

  2. Q2. privateを使う目的は何ですか？

    回答を表示

    クラスの内部データを外部から直接変更できないようにすることです（カプセル化）。 これにより、データの整合性を保ち、予期しない変更を防ぐことができます。

  3. Q3. エラーハンドリングが重要な理由は何ですか？

    回答を表示

    予期しない入力やエラーが発生した場合でも、アプリケーションが適切に動作し続けるためです。 ユーザーにわかりやすいエラーメッセージを提供することで、使いやすいアプリケーションになります。
