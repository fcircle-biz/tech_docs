クラス：設計図から実体を作ろう

オブジェクト指向プログラミングの基礎となるクラスの概念を学びます。プロパティとメソッドに型をつけて、より構造的なプログラムを作る方法を実践しましょう。

この章で学ぶこと

  • クラスとは何か、設計図としての役割を理解する
  • プロパティとメソッドに型をつけてクラスを定義する方法
  • コンストラクタとインスタンス化の仕組みを理解する
  • アクセス修飾子を使ってデータを適切に保護する方法

11.1 クラスって何だろう？

クラスとは、「もの」の設計図のようなものです。 家を建てる時に設計図があるように、プログラムで「もの」を作る時にもクラスという設計図を使います。

例えば、「ユーザー」という概念を考えてみましょう。ユーザーには以下のような情報があります：

  • 名前（田中太郎、佐藤花子など）
  • 年齢（25歳、30歳など）
  • メールアドレス（xxx@example.comなど）

クラスは、この「ユーザーというものには、名前、年齢、メールアドレスがある」という共通の構造を定義します。 そして、この設計図から実際のユーザー（田中太郎さん、佐藤花子さんなど）を作り出すことができます。

クラスと実体（インスタンス）

クラスは設計図、インスタンスは設計図から作った実体です。 たい焼きで例えると、クラスは「たい焼きの型」、インスタンスは「実際に焼いたたい焼き」です。

11.2 プロパティとメソッド：データと機能

クラスには2種類の要素があります：

プロパティ（データ）

そのものが持っている「情報」です。

例: 名前、年齢、メールアドレス

メソッド（機能）

そのものが「できること」です。

例: 自己紹介する、誕生日を祝う

基本的なクラスの定義 
// Userクラス（ユーザーという「もの」の設計図）
class User {
  // プロパティ（データ）
  name: string;    // 名前
  age: number;     // 年齢
  email: string;   // メールアドレス

  // メソッド（できること）
  greet(): string {
    return `こんにちは、${this.name}です。`;
  }
}

thisキーワード

thisは「自分自身」を指します。メソッドの中でthis.nameと書くと、 「この人（インスタンス）の名前」という意味になります。

11.3 コンストラクタ：初期化のための特別なメソッド

コンストラクタは、クラスから実体（インスタンス）を作る時に最初に実行される特別なメソッドです。 初期値を設定するために使います。

コンストラクタの定義 
class User {
  name: string;
  age: number;
  email: string;

  // コンストラクタ：初期値を設定する
  constructor(name: string, age: number, email: string) {
    this.name = name;
    this.age = age;
    this.email = email;
  }

  greet(): string {
    return `こんにちは、${this.name}です。${this.age}歳です。`;
  }
}

設計図ができたので、実際にユーザーを作ってみましょう：

インスタンスの作成 
// new キーワードでインスタンスを作成
const tanaka = new User("田中太郎", 25, "tanaka@example.com");
const sato = new User("佐藤花子", 30, "sato@example.com");

// メソッドを呼び出す
console.log(tanaka.greet()); // こんにちは、田中太郎です。25歳です。
console.log(sato.greet());   // こんにちは、佐藤花子です。30歳です。

// プロパティにアクセス
console.log(tanaka.name);  // 田中太郎
console.log(sato.age);     // 30

11.4 アクセス修飾子：情報の公開範囲を制御する

アクセス修飾子は、プロパティやメソッドを「誰に見せるか」を決める仕組みです。 TypeScriptには3種類あります：

修飾子 意味 アクセス範囲
public 公開 どこからでもアクセス可能（デフォルト）
private 非公開 クラスの中からのみアクセス可能
protected 保護 クラスと継承先からアクセス可能

実際の例を見てみましょう：

アクセス修飾子の例 
class BankAccount {
  public accountNumber: string;      // 口座番号（公開）
  private balance: number;           // 残高（非公開）

  constructor(accountNumber: string, initialBalance: number) {
    this.accountNumber = accountNumber;
    this.balance = initialBalance;
  }

  // 残高を確認する（公開メソッド）
  public getBalance(): number {
    return this.balance;
  }

  // お金を預ける（公開メソッド）
  public deposit(amount: number): void {
    if (amount > 0) {
      this.balance += amount;
      console.log(`${amount}円を入金しました。`);
    }
  }

  // お金を引き出す（公開メソッド）
  public withdraw(amount: number): boolean {
    if (this.canWithdraw(amount)) {
      this.balance -= amount;
      console.log(`${amount}円を出金しました。`);
      return true;
    }
    return false;
  }

  // 引き出し可能かチェック（非公開メソッド）
  private canWithdraw(amount: number): boolean {
    return this.balance >= amount;
  }
}

このクラスを使ってみましょう：

アクセス修飾子の動作 
const myAccount = new BankAccount("123-456", 10000);

// publicなプロパティにはアクセス可能
console.log(myAccount.accountNumber);  // 123-456

// publicなメソッドは呼び出し可能
console.log(myAccount.getBalance());   // 10000
myAccount.deposit(5000);               // 5000円を入金しました。
myAccount.withdraw(3000);              // 3000円を出金しました。

// privateなプロパティには直接アクセス不可（エラー）
// console.log(myAccount.balance);     // エラー！

// privateなメソッドは呼び出し不可（エラー）
// myAccount.canWithdraw(1000);        // エラー！

privateを使う理由

残高（balance）をprivateにすることで、外部から勝手に値を変更されることを防ぎます。 必ずdeposit()やwithdraw()メソッドを通すことで、正しい処理が保証されます。

実習：Bookクラスを作ってみよう

書籍を表すBookクラスを作成してみましょう。

要件：

  • プロパティ: title（タイトル）, author（著者）, pages（ページ数）
  • コンストラクタで初期値を設定
  • getInfo()メソッドで書籍情報を返す
  • 適切な型を指定する
解答例を表示
解答例 
class Book {
  title: string;
  author: string;
  pages: number;

  constructor(title: string, author: string, pages: number) {
    this.title = title;
    this.author = author;
    this.pages = pages;
  }

  getInfo(): string {
    return `『${this.title}』 著者: ${this.author}, ${this.pages}ページ`;
  }
}

// 使用例
const book1 = new Book("TypeScript入門", "山田太郎", 300);
const book2 = new Book("プログラミングの基礎", "鈴木花子", 250);

console.log(book1.getInfo());
// 『TypeScript入門』 著者: 山田太郎, 300ページ
console.log(book2.getInfo());
// 『プログラミングの基礎』 著者: 鈴木花子, 250ページ

11.5 簡潔な書き方：コンストラクタ引数でプロパティを定義

TypeScriptでは、コンストラクタの引数にアクセス修飾子をつけることで、 プロパティの宣言と初期化を同時に行える便利な書き方があります。

通常の書き方

class User {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

簡潔な書き方

class User {
  constructor(
    public name: string,
    public age: number
  ) {
    // 自動的にプロパティが作られる
  }
}

どちらも同じ動作をしますが、簡潔な書き方の方がコードが短くなります。

実務でよく使われる書き方

実際のプロジェクトでは、簡潔な書き方がよく使われます。 コードが短くなり、同じことを二度書く必要がなくなるためです。

11.6 クラスを使った実践例：タスク管理システム

これまで学んだクラスの知識を使って、簡単なタスク管理システムを作ってみましょう。

タスク管理システムの実装 
// タスクを表すクラス
class Task {
  constructor(
    public id: number,
    public title: string,
    private completed: boolean = false
  ) {}

  // タスクを完了にする
  public complete(): void {
    this.completed = true;
    console.log(`タスク "${this.title}" を完了しました！`);
  }

  // 完了状態を確認
  public isCompleted(): boolean {
    return this.completed;
  }

  // タスク情報を表示
  public getInfo(): string {
    const status = this.completed ? "✓完了" : "未完了";
    return `[${this.id}] ${this.title} - ${status}`;
  }
}

// タスクリストを管理するクラス
class TaskList {
  private tasks: Task[] = [];
  private nextId: number = 1;

  // タスクを追加
  public addTask(title: string): void {
    const task = new Task(this.nextId, title);
    this.tasks.push(task);
    this.nextId++;
    console.log(`タスク "${title}" を追加しました。`);
  }

  // すべてのタスクを表示
  public showAllTasks(): void {
    console.log("=== タスク一覧 ===");
    this.tasks.forEach(task => {
      console.log(task.getInfo());
    });
  }

  // タスクを完了にする
  public completeTask(id: number): void {
    const task = this.tasks.find(t => t.id === id);
    if (task) {
      task.complete();
    } else {
      console.log("タスクが見つかりません。");
    }
  }
}

実際に使ってみましょう：

使用例 
const myTasks = new TaskList();

// タスクを追加
myTasks.addTask("TypeScriptを学ぶ");
myTasks.addTask("買い物に行く");
myTasks.addTask("レポートを書く");

// すべてのタスクを表示
myTasks.showAllTasks();
// === タスク一覧 ===
// [1] TypeScriptを学ぶ - 未完了
// [2] 買い物に行く - 未完了
// [3] レポートを書く - 未完了

// タスクを完了にする
myTasks.completeTask(1);
// タスク "TypeScriptを学ぶ" を完了しました！

// 再度表示
myTasks.showAllTasks();
// === タスク一覧 ===
// [1] TypeScriptを学ぶ - ✓完了
// [2] 買い物に行く - 未完了
// [3] レポートを書く - 未完了

理解度確認クイズ

  1. Q1. クラスとインスタンスの違いは何ですか？

    回答を表示

    クラスは「設計図」、インスタンスは「設計図から作った実体」です。 例えば、たい焼きの型がクラスで、実際に焼いたたい焼きがインスタンスです。 1つのクラスから複数のインスタンスを作ることができます。

  2. Q2. privateアクセス修飾子を使う理由は何ですか？

    回答を表示

    クラスの外部から直接アクセスされたくないプロパティやメソッドを保護するためです。 これにより、不正な値の設定を防ぎ、必ずメソッドを通して適切な処理を行うことが保証されます。 例えば、銀行口座の残高を勝手に変更されないようにする場合などに使います。

  3. Q3. コンストラクタの役割は何ですか？

    回答を表示

    コンストラクタは、クラスからインスタンスを作る時に最初に実行される特別なメソッドで、 プロパティの初期値を設定する役割があります。newキーワードでインスタンスを作る時に自動的に呼び出されます。

この章のまとめ

  • クラスは「もの」の設計図で、プロパティ（データ）とメソッド（機能）を持つ
  • コンストラクタはインスタンス作成時に実行され、初期値を設定する
  • アクセス修飾子（public/private/protected）でデータの公開範囲を制御できる
  • privateを使うことで、外部からの不正なアクセスを防ぎ、安全性を高められる
  • クラスを使うことで、関連するデータと機能をまとめて管理できる
