関数：処理をパッケージ化しよう

よく使う処理を「関数」としてまとめ、引数と返り値に型をつける方法を学びます。型があることで、関数の使い方が明確になり、間違った使い方を防げることを体験します。

この章で学ぶこと

  • 関数とは何か、なぜ便利なのかを理解する
  • 引数と返り値に型をつける方法を理解する
  • 型による関数の安全性を体験する

8.1 関数って何だろう？

関数（かんすう）とは、よく使う処理をひとまとめにして、名前をつけたものです。 一度作っておけば、何度でも呼び出して使うことができます。

日常生活で例えると、「電子レンジで温める」という行動を考えてみましょう：

  1. 1. 食べ物を皿に入れる
  2. 2. 電子レンジに入れる
  3. 3. 時間を設定する
  4. 4. スタートボタンを押す
  5. 5. 温まったら取り出す

これらの手順を毎回考えるのは大変ですよね。だから私たちは「電子レンジで温める」という一つの塊として覚えています。 プログラミングの関数も同じで、複雑な処理を名前付きの塊にまとめることで、何度でも簡単に使えるようにするのです。

関数のメリット

関数を使うことで、同じコードを何度も書く必要がなくなり、プログラムが読みやすく、 管理しやすくなります。また、修正が必要な時も一箇所直せば良いので便利です。

8.2 関数の基本的な書き方

TypeScriptで関数を定義（作成）するには、functionというキーワードを使います。 基本的な構造を見てみましょう。

関数の基本構造 
// 基本的な関数の定義
function greet(): void {
    console.log("こんにちは！");
}

// 関数を呼び出す（使う）
greet();

コードの解説

function - これから関数を定義しますという意味

greet - 関数の名前（好きな名前をつけられます）

() - 引数（パラメータ）を書く場所（今回は空）

: void - 返り値の型（voidは「何も返さない」という意味）

{ } - 波括弧の中に処理を書く

greet() - 関数を呼び出す（実行する）

  4. 4. 「こんにちは！」と表示されることを確認

8.3 引数：関数にデータを渡そう

引数（ひきすう）とは、関数に渡すデータのことです。 引数を使うことで、同じ関数を異なるデータで実行できます。

料理で例えると、「電子レンジで温める」という関数に対して、「何を」「何分」という情報を渡すイメージです。

引数のある関数 
// 名前を受け取って挨拶する関数
function greetPerson(name: string): void {
    console.log(`こんにちは、${name}さん！`);
}

// 異なる名前で関数を呼び出す
greetPerson("太郎");  // こんにちは、太郎さん！
greetPerson("花子");  // こんにちは、花子さん！

引数の型指定

name: stringという部分で、 「name という引数は string 型（文字列）ですよ」と宣言しています。

型を指定することで、間違って数値を渡そうとした時にエディタがエラーで教えてくれます。

複数の引数を持つ関数 
// 2つの数値を受け取って足し算する関数
function add(a: number, b: number): void {
    const result = a + b;
    console.log(`${a} + ${b} = ${result}`);
}

add(5, 3);   // 5 + 3 = 8
add(10, 20); // 10 + 20 = 30

引数は複数指定できる

カンマ,で区切ることで、 好きなだけ引数を追加できます。それぞれに型を指定しましょう。

8.4 返り値：関数から結果を受け取ろう

返り値（かえりち）とは、関数が処理を終えた後に返してくれる結果のことです。 これまでの例では結果をconsole.logで表示していましたが、 返り値を使うことで、その結果を別の場所で使うことができます。

返り値のある関数 
// 2つの数値を受け取って、その合計を返す関数
function add(a: number, b: number): number {
    return a + b;  // 結果を返す
}

// 関数の返り値を変数に代入できる
const sum = add(5, 3);
console.log(sum); // 8

// 返り値を直接使うこともできる
console.log(add(10, 20)); // 30

返り値の型指定

): number - この関数は number 型（数値）を返しますという宣言

return a + b - 計算結果を呼び出し元に返す

返り値の型を指定することで、間違った型を返そうとした時にエラーが出ます

返り値の型 意味 使用例
void 何も返さない 画面に表示するだけの関数
number 数値を返す 計算結果を返す関数
string 文字列を返す メッセージを生成する関数
boolean 真偽値を返す 条件判定の結果を返す関数
様々な返り値の型 
// 文字列を返す関数
function createMessage(name: string): string {
    return `こんにちは、${name}さん！`;
}

// 真偽値を返す関数
function isAdult(age: number): boolean {
    return age >= 20;
}

// 使用例
const message = createMessage("太郎");
console.log(message); // こんにちは、太郎さん！

const adult = isAdult(25);
console.log(adult); // true

8.5 型がある関数の安全性

TypeScriptで関数に型をつけることで、どのような安全性が得られるのか見てみましょう。

型が守ってくれる例 
// 正しく型付けされた関数
function multiply(a: number, b: number): number {
    return a * b;
}

// 正しい使い方
const result1 = multiply(5, 3); // OK: 15

// 間違った使い方（エラーになる）
// const result2 = multiply("5", 3);
// エラー: 引数の型が 'string' ですが、'number' が期待されています

// const result3 = multiply(5);
// エラー: 2つの引数が必要ですが、1つしか渡されていません

型による関数の安全性

  • 引数の型チェック - 間違った型のデータを渡すとエラーで教えてくれる
  • 引数の数チェック - 必要な引数が足りないとエラーで教えてくれる
  • 返り値の型チェック - 宣言した型と異なる値を返そうとするとエラーで教えてくれる
  • エディタの支援 - 何が渡せて何が返ってくるか、エディタが自動で表示してくれる

課題：消費税込み価格を計算する関数

  • • 関数名：calculateWithTax

  • • 返り値：数値型（消費税10%を加えた価格）

8.6 アロー関数：もう一つの書き方

TypeScriptには、アロー関数という、より短く書ける関数の記法があります。 =>という矢印を使うことから「アロー関数」と呼ばれます。

アロー関数の書き方 
// 通常の関数
function add1(a: number, b: number): number {
    return a + b;
}

// アロー関数（同じ意味）
const add2 = (a: number, b: number): number => {
    return a + b;
};

// さらに短く書ける（処理が1行の場合）
const add3 = (a: number, b: number): number => a + b;

// すべて同じ動作
console.log(add1(5, 3)); // 8
console.log(add2(5, 3)); // 8
console.log(add3(5, 3)); // 8

アロー関数のポイント

const 関数名 = - 関数を変数に代入する形で書く

=> - 矢印の後に処理を書く

処理が1行だけなら、波括弧{ }とreturnを省略できる

型の指定方法は通常の関数と同じ

どちらを使えばいい？

どちらを使っても構いません。ただし、最近のJavaScript/TypeScriptではアロー関数がよく使われます。 最初は通常の関数で慣れて、慣れてきたらアロー関数も使ってみましょう。

理解度確認クイズ

  1. Q1. 関数を使うメリットは何ですか？

    回答を表示

    よく使う処理をまとめて名前をつけることで、同じコードを何度も書く必要がなくなります。 また、プログラムが読みやすく管理しやすくなり、修正が必要な時も一箇所直せば良いため便利です。

  2. Q2. 引数と返り値の違いは何ですか？

    回答を表示

    引数は関数に渡すデータ（入力）で、返り値は関数が返す結果（出力）です。 引数を使うことで同じ関数を異なるデータで実行でき、返り値を使うことで処理結果を別の場所で使うことができます。

  3. Q3. 関数に型をつけることで、どのような安全性が得られますか？

    回答を表示

    引数の型・数をチェックしてくれるので、間違った使い方をするとエラーで教えてくれます。 また、返り値の型も保証されるため、関数の使い方が明確になり、実行前に間違いに気づくことができます。

この章のまとめ

  • 関数は、よく使う処理をまとめて名前をつけたもの
  • 引数を使うことで、関数にデータを渡すことができる
  • 返り値を使うことで、関数から結果を受け取ることができる
  • 型をつけることで、引数と返り値の安全性が保たれる
  • アロー関数を使うと、より簡潔に関数を書くことができる
