非同期処理：時間のかかる処理を扱おう

Promise と async/await を使った非同期処理の基礎を学び、型を使って非同期処理の結果を安全に扱う方法を実践的に習得しましょう。

この章で学ぶこと

  • 同期処理と非同期処理の違いを理解する
  • Promiseの概念と基本的な使い方を学ぶ
  • async/awaitを使って非同期処理を分かりやすく書く方法を習得する
  • 非同期処理での型の扱い方とエラーハンドリングを理解する

14.1 同期処理と非同期処理の違い

プログラムの処理には、同期処理と非同期処理の2つの種類があります。 この違いを理解することが、効率的なプログラムを作る第一歩です。

同期処理とは

同期処理は、1つの処理が終わってから次の処理に進む、順番に実行される処理です。 レストランで料理を注文する場合を考えてみましょう。

同期処理の例：1つずつ順番に調理

1. カレーを作る（20分待つ）
2. カレーが完成してから、サラダを作る（5分待つ）
3. サラダが完成してから、デザートを作る（3分待つ）
合計時間：20 + 5 + 3 = 28分

非同期処理とは

非同期処理は、ある処理の完了を待たずに次の処理を始められる処理です。 複数の料理を同時に調理する場合を考えてみましょう。

非同期処理の例：同時に調理

1. カレーを火にかける（調理開始）
2. カレーが煮込まれている間に、サラダを作る
3. サラダを作りながら、デザートも準備する
4. カレーが完成したら、すべての料理が揃う
合計時間：約20分（最も時間がかかる処理に合わせる）

同期処理と非同期処理の違い

非同期処理の重要性

非同期処理を使うと、時間のかかる処理（サーバーとの通信、ファイルの読み込みなど）を待っている間に 他の処理を実行でき、プログラム全体が効率的に動きます。

14.2 Promiseの基本

Promise（プロミス）は、「将来的に値が得られることを約束するオブジェクト」です。 非同期処理の結果を扱うための仕組みで、TypeScriptでは型を使って安全に扱えます。

Promiseの3つの状態

Promiseには以下の3つの状態があります：

Pending

処理が実行中の状態

「待っている」状態

Fulfilled

処理が成功した状態

「成功」状態

Rejected

処理が失敗した状態

「失敗」状態

Promiseの基本的な使い方

Promiseの基本例 
// 時間のかかる処理をシミュレートする関数
function fetchData(): Promise<string> {
  // Promiseオブジェクトを返す
  return new Promise((resolve, reject) => {
    // 2秒後に成功する処理
    setTimeout(() => {
      resolve("データの取得に成功しました");
    }, 2000);
  });
}

// Promiseの使い方
fetchData()
  .then((data: string) => {
    // 成功した場合の処理
    console.log(data);
  })
  .catch((error) => {
    // 失敗した場合の処理
    console.error("エラー:", error);
  });

Promiseの型定義

Promise<string> は 「将来的にstring型の値が得られる」ことを示しています。 型パラメータに成功時の値の型を指定します。

エラーハンドリング

エラーを含むPromise 
// 成功または失敗する処理
function fetchUserData(userId: number): Promise<string> {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (userId > 0) {
        // 成功
        resolve(`ユーザーID ${userId} のデータ`);
      } else {
        // 失敗
        reject(new Error("無効なユーザーIDです"));
      }
    }, 1000);
  });
}

// 使用例
fetchUserData(1)
  .then((data) => console.log("成功:", data))
  .catch((error) => console.error("失敗:", error.message));

fetchUserData(-1)
  .then((data) => console.log("成功:", data))
  .catch((error) => console.error("失敗:", error.message));

14.3 async/awaitでスッキリ書こう

async/awaitは、Promiseをより読みやすく、分かりやすく書くための構文です。 非同期処理を同期処理のように書けるため、初心者にとっても理解しやすいです。

asyncキーワード

関数の前に async をつけると、 その関数は自動的にPromiseを返す関数になります。

async関数の基本 
// async関数は自動的にPromiseを返す
async function greet(): Promise<string> {
  return "こんにちは";
}

// 使用例
greet().then((message) => {
  console.log(message); // "こんにちは"
});

awaitキーワード

await は、Promiseの結果が得られるまで待つキーワードです。 async関数の中でのみ使用できます。

awaitの使用例 
// データを取得する非同期関数
async function getUserName(userId: number): Promise<string> {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve(`ユーザー${userId}`);
    }, 1000);
  });
}

// awaitを使って分かりやすく書く
async function showUserInfo(): Promise<void> {
  console.log("データを取得中...");

  // awaitで結果を待つ（同期処理のように書ける）
  const userName: string = await getUserName(123);

  console.log(`ユーザー名: ${userName}`);
  console.log("処理完了");
}

// 実行
showUserInfo();

thenとasync/awaitの比較

then/catchを使った書き方 

fetchUserData(1)
  .then((data) => {
    console.log(data);
    return fetchOrders(1);
  })
  .then((orders) => {
    console.log(orders);
  })
  .catch((error) => {
    console.error(error);
  });

async/awaitを使った書き方 

async function process() {
  try {
    const data = await fetchUserData(1);
    console.log(data);
    const orders = await fetchOrders(1);
    console.log(orders);
  } catch (error) {
    console.error(error);
  }
}

async/awaitの利点

async/awaitを使うと、非同期処理のコードが同期処理のように読みやすくなります。 特に複数の非同期処理を順番に実行する場合、コードがシンプルで分かりやすくなります。

14.4 非同期処理でのエラーハンドリング

非同期処理では、エラーが発生する可能性が高いため、適切なエラーハンドリングが重要です。 TypeScriptの型を使って、安全にエラーを処理する方法を学びましょう。

try-catchでエラーを捕捉する

try-catchでのエラーハンドリング 
// APIからデータを取得する関数（型定義付き）
async function fetchAPI(url: string): Promise<any> {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (url.startsWith("https://")) {
        resolve({ data: "APIレスポンス" });
      } else {
        reject(new Error("無効なURLです"));
      }
    }, 1000);
  });
}

// エラーハンドリング付きの処理
async function getData(): Promise<void> {
  try {
    // 成功するケース
    const result = await fetchAPI("https://api.example.com");
    console.log("成功:", result);

    // エラーになるケース
    const errorResult = await fetchAPI("http://api.example.com");
    console.log("成功:", errorResult);

  } catch (error) {
    // エラーの型を確認して安全に扱う
    if (error instanceof Error) {
      console.error("エラーが発生しました:", error.message);
    } else {
      console.error("予期しないエラー:", error);
    }
  }
}

getData();

型安全なエラーハンドリング

カスタムエラー型の定義 
// カスタムエラー型
class ApiError extends Error {
  constructor(
    message: string,
    public statusCode: number
  ) {
    super(message);
    this.name = "ApiError";
  }
}

// 型安全な非同期関数
async function fetchUser(userId: number): Promise<{ id: number; name: string }> {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (userId > 0) {
        resolve({ id: userId, name: "太郎" });
      } else {
        reject(new ApiError("ユーザーが見つかりません", 404));
      }
    }, 1000);
  });
}

// 使用例
async function displayUser(userId: number): Promise<void> {
  try {
    const user = await fetchUser(userId);
    console.log(`ユーザー名: ${user.name}`);
  } catch (error) {
    if (error instanceof ApiError) {
      console.error(`API Error [${error.statusCode}]: ${error.message}`);
    } else {
      console.error("予期しないエラー:", error);
    }
  }
}

エラーハンドリングを忘れずに

非同期処理では必ずエラーハンドリングを行いましょう。エラーを処理しないと、 プログラムが予期せず停止したり、ユーザーに適切なメッセージを表示できなくなります。

14.5 複数の非同期処理を扱う

実際の開発では、複数の非同期処理を同時に実行したり、順番に実行したりする必要があります。 TypeScriptでは型を使って、これらを安全に扱えます。

Promise.all - 複数の処理を同時に実行

Promise.all を使うと、 複数の非同期処理を同時に実行し、すべて完了するまで待つことができます。

Promise.allの使用例 
// 複数のデータ取得関数
async function getUser(id: number): Promise<string> {
  return new Promise((resolve) => {
    setTimeout(() => resolve(`ユーザー${id}`), 1000);
  });
}

async function getProduct(id: number): Promise<string> {
  return new Promise((resolve) => {
    setTimeout(() => resolve(`商品${id}`), 1500);
  });
}

async function getOrder(id: number): Promise<string> {
  return new Promise((resolve) => {
    setTimeout(() => resolve(`注文${id}`), 800);
  });
}

// すべての処理を同時に実行
async function fetchAllData(): Promise<void> {
  console.log("データ取得開始...");
  const startTime = Date.now();

  // Promise.allで同時実行（型も自動で推論される）
  const [user, product, order] = await Promise.all([
    getUser(1),
    getProduct(100),
    getOrder(5)
  ]);

  const endTime = Date.now();

  console.log("ユーザー:", user);
  console.log("商品:", product);
  console.log("注文:", order);
  console.log(`実行時間: ${endTime - startTime}ms`);
  // 約1500ms（最も遅い処理に合わせる）
}

fetchAllData();

順番に実行する場合

順次実行の例 
// 前の処理の結果を使う場合は順次実行
async function processInOrder(): Promise<void> {
  console.log("順次処理開始...");
  const startTime = Date.now();

  // 1つずつ順番に実行
  const user = await getUser(1);
  console.log("1. ユーザー取得完了:", user);

  const product = await getProduct(100);
  console.log("2. 商品取得完了:", product);

  const order = await getOrder(5);
  console.log("3. 注文取得完了:", order);

  const endTime = Date.now();
  console.log(`実行時間: ${endTime - startTime}ms`);
  // 約3300ms（すべての処理時間の合計）
}

実習：天気情報取得アプリを作ろう

非同期処理を使って、複数の都市の天気情報を取得するプログラムを作成してみましょう。

天気情報取得プログラム 
// 天気情報の型定義
interface WeatherInfo {
  city: string;
  temperature: number;
  condition: string;
}

// 天気情報を取得する非同期関数（シミュレーション）
async function getWeather(city: string): Promise<WeatherInfo> {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      // ランダムで天気を生成
      const conditions = ["晴れ", "曇り", "雨", "雪"];
      const randomTemp = Math.floor(Math.random() * 30) + 5;
      const randomCondition = conditions[Math.floor(Math.random() * conditions.length)];

      if (city) {
        resolve({
          city: city,
          temperature: randomTemp,
          condition: randomCondition
        });
      } else {
        reject(new Error("都市名が指定されていません"));
      }
    }, 1000);
  });
}

// 複数の都市の天気を取得
async function getAllWeather(): Promise<void> {
  try {
    console.log("天気情報を取得中...\n");

    // 複数の都市の天気を同時に取得
    const cities = ["東京", "大阪", "札幌", "福岡"];
    const weatherPromises = cities.map(city => getWeather(city));
    const weatherData: WeatherInfo[] = await Promise.all(weatherPromises);

    // 結果を表示
    console.log("=== 天気情報 ===");
    weatherData.forEach(weather => {
      console.log(`${weather.city}: ${weather.temperature}°C ${weather.condition}`);
    });

  } catch (error) {
    if (error instanceof Error) {
      console.error("エラー:", error.message);
    }
  }
}

// 実行
getAllWeather();

理解度確認クイズ

  1. Q1. 同期処理と非同期処理の違いは何ですか？

    回答を表示

    同期処理は1つの処理が終わってから次の処理に進む順次実行、 非同期処理は処理の完了を待たずに次の処理を始められる実行方式です。 非同期処理を使うと、時間のかかる処理を待っている間に他の処理を実行できます。

  2. Q2. Promiseの3つの状態とは何ですか？

    回答を表示

    Pending（処理中）、Fulfilled（成功）、Rejected（失敗）の3つの状態があります。 Promiseは必ずこれらのいずれかの状態になり、一度状態が決まると変わりません。

  3. Q3. async/awaitを使うメリットは何ですか？

    回答を表示

    非同期処理を同期処理のように読みやすく書けることです。 thenチェーンよりもコードがシンプルになり、エラーハンドリングもtry-catchで統一的に書けます。

  4. Q4. Promise.allを使うのはどんな時ですか？

    回答を表示

    複数の非同期処理を同時に実行して、すべての処理が完了するまで待ちたい時に使います。 処理同士が独立している場合、Promise.allを使うことで実行時間を短縮できます。

よくあるエラーと対処法

"await is only valid in async function"

原因：async関数の外でawaitを使っています

対処法：awaitを使う関数の前にasyncキーワードを追加してください

エラーが捕捉されずプログラムが停止する

原因：try-catchでエラーをキャッチしていません

対処法：async関数内でtry-catchを使ってエラーを適切に処理してください

Promiseの結果がundefinedになる

原因：awaitを忘れているか、Promiseがresolveする前に値を使っています

対処法：Promiseを返す関数の前にawaitを付けて、結果を待ってから使用してください
