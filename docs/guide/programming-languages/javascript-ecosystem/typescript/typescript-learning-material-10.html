オブジェクトとインターフェース：データを構造化しよう

関連する情報をまとめるオブジェクトと、その構造を定義するインターフェースの作り方を学びます。型で構造を定義することで、チーム開発でも安心してコードが書けることを理解しましょう。

この章で学ぶこと

  • オブジェクトを使って関連する情報をまとめる方法を理解する
  • インターフェースでオブジェクトの構造を定義する方法を習得する
  • オプショナルプロパティと読み取り専用プロパティの使い方を学ぶ
  • 実践的なデータモデリングの基礎を身につける

10.1 オブジェクトって何だろう？

これまで学んできた変数は、1つの値を保存する「箱」のようなものでした。 しかし、実際のプログラムでは、関連する複数の情報をまとめて扱いたい場面が頻繁にあります。

例えば、ある人の情報を扱うとき、以下のようなデータが必要になります：

個別の変数で管理する場合（非効率） 

// 個別に変数を作ると管理が大変
let userName: string = "田中太郎";
let userAge: number = 28;
let userEmail: string = "tanaka@example.com";
let userIsStudent: boolean = false;

このように個別の変数で管理すると、変数の数が増えて管理が大変になります。 そこでオブジェクトを使うと、関連する情報を1つにまとめることができます。

オブジェクトとは

オブジェクトは、関連する複数のデータを1つのまとまりとして扱うための仕組みです。 複数の引き出しを持つ収納ボックスのようなイメージです。

10.2 オブジェクトの基本構文

オブジェクトは波括弧 { } を使って作成します。 オブジェクトの中には、プロパティ（「名前: 値」のペア）を複数持つことができます。

オブジェクトの基本構文 
// ユーザー情報をオブジェクトにまとめる
let user = {
  name: "田中太郎",
  age: 28,
  email: "tanaka@example.com",
  isStudent: false
};

// プロパティにアクセス（ドット記法）
console.log(user.name);  // "田中太郎"
console.log(user.age);   // 28

オブジェクトの各プロパティにはドット記法（.）を使ってアクセスできます。

プロパティの命名規則

プロパティ名は通常、小文字で始め、複数の単語を組み合わせる場合はキャメルケース（camelCase）を使います。 例：userName, emailAddress, isActive

10.3 オブジェクトに型をつけよう

TypeScriptでは、オブジェクトにも型を指定できます。 これにより、どんなプロパティがあるか、それぞれの型は何かを明確にできます。

型を明示したオブジェクト 
// オブジェクトの型を定義
let user: {
  name: string;
  age: number;
  email: string;
  isStudent: boolean;
} = {
  name: "田中太郎",
  age: 28,
  email: "tanaka@example.com",
  isStudent: false
};

// 型が合わないとエラーになる
// user.age = "28歳";  // エラー！ numberを期待

しかし、このように毎回型を書くのは大変です。 そこでインターフェースという仕組みを使うと、型定義を再利用できるようになります。

10.4 インターフェースで型を定義しよう

インターフェースは、オブジェクトの「設計図」のようなものです。 どんなプロパティがあり、それぞれの型は何かを定義します。

インターフェースの定義 
// Userインターフェースを定義
interface User {
  name: string;
  age: number;
  email: string;
  isStudent: boolean;
}

// インターフェースを使って変数を定義
let user: User = {
  name: "田中太郎",
  age: 28,
  email: "tanaka@example.com",
  isStudent: false
};

// 別のユーザーも同じ型で定義可能
let anotherUser: User = {
  name: "佐藤花子",
  age: 25,
  email: "sato@example.com",
  isStudent: true
};

インターフェースのメリット

インターフェースを使うと、同じ構造を持つオブジェクトを何度も作るときに、 型定義を再利用でき、コードが読みやすく保守しやすくなります。

10.5 オプショナルプロパティ：省略可能な項目

すべてのプロパティが常に必要とは限りません。 例えば、電話番号は持っている人もいれば持っていない人もいます。

このような「あってもなくても良い」プロパティには、?（クエスチョンマーク）をつけます。

オプショナルプロパティ 
// 電話番号は省略可能（?をつける）
interface User {
  name: string;
  age: number;
  email: string;
  phone?: string;  // オプショナル
}

// 電話番号がなくてもOK
let user1: User = {
  name: "田中太郎",
  age: 28,
  email: "tanaka@example.com"
};

// 電話番号があってもOK
let user2: User = {
  name: "佐藤花子",
  age: 25,
  email: "sato@example.com",
  phone: "090-1234-5678"
};

オプショナルプロパティの活用

オプショナルプロパティを使うと、柔軟なデータ構造を作れます。 ただし、使う前に値が存在するかチェックすることをお勧めします。

10.6 読み取り専用プロパティ：変更禁止の項目

一度設定したら変更してはいけないデータもあります。 例えば、ユーザーIDは一度付与されたら変更できないようにしたいですね。

そのようなプロパティにはreadonly（読み取り専用）を指定します。

読み取り専用プロパティ 
// ユーザーIDは読み取り専用
interface User {
  readonly id: number;  // 変更禁止
  name: string;
  age: number;
}

let user: User = {
  id: 12345,
  name: "田中太郎",
  age: 28
};

// 名前は変更できる
user.name = "田中次郎";  // OK

// IDは変更できない
// user.id = 67890;  // エラー！readonlyプロパティ

readonlyの使いどころ

IDや作成日時など、一度設定したら変更すべきでないデータにはreadonlyを使います。 これにより、誤って値を上書きするミスを防げます。

10.7 ネストしたオブジェクト：オブジェクトの中のオブジェクト

オブジェクトのプロパティには、別のオブジェクトを持たせることもできます。 これをネスト（入れ子）構造と呼びます。

ネストしたオブジェクト 
// 住所の型を定義
interface Address {
  postalCode: string;
  prefecture: string;
  city: string;
}

// ユーザーの型（住所を含む）
interface User {
  name: string;
  age: number;
  address: Address;  // オブジェクト型のプロパティ
}

// ネストしたオブジェクトを作成
let user: User = {
  name: "田中太郎",
  age: 28,
  address: {
    postalCode: "123-4567",
    prefecture: "東京都",
    city: "渋谷区"
  }
};

// ネストしたプロパティへのアクセス
console.log(user.address.prefecture);  // "東京都"

実習：書籍管理システムを作ろう

インターフェースを使って、書籍の情報を管理するシステムを作成しましょう。

ステップ1：インターフェースを定義

// 書籍の型を定義
interface Book {
  readonly isbn: string;  // ISBN（変更不可）
  title: string;          // タイトル
  author: string;         // 著者
  publishedYear: number;  // 出版年
  price: number;          // 価格
  description?: string;   // 説明（省略可）
}

ステップ2：書籍オブジェクトを作成

// 書籍データを作成
let book1: Book = {
  isbn: "978-4-123-45678-9",
  title: "TypeScript入門",
  author: "田中太郎",
  publishedYear: 2024,
  price: 2800,
  description: "初心者向けTypeScript解説書"
};

let book2: Book = {
  isbn: "978-4-987-65432-1",
  title: "プログラミングの基礎",
  author: "佐藤花子",
  publishedYear: 2023,
  price: 3200
  // descriptionは省略可能
};

ステップ3：書籍情報を表示する関数

// 書籍情報を表示する関数
function displayBook(book: Book): void {
  console.log(`タイトル: ${book.title}`);
  console.log(`著者: ${book.author}`);
  console.log(`出版年: ${book.publishedYear}`);
  console.log(`価格: ¥${book.price}`);

  // オプショナルプロパティのチェック
  if (book.description) {
    console.log(`説明: ${book.description}`);
  }
}

// 関数を実行
displayBook(book1);

よくあるエラーと対処法

Property 'xxx' is missing

原因：必須プロパティが抜けている

解決：インターフェースで定義されたすべての必須プロパティを指定するか、 省略可能にする場合は?をつけてオプショナルにします。

Cannot assign to 'xxx' because it is a read-only property

原因：readonlyプロパティを変更しようとしている

解決：readonlyプロパティは作成時にのみ値を設定でき、後から変更できません。 変更が必要な場合はreadonlyを外します。

Property 'xxx' does not exist on type

原因：インターフェースに定義されていないプロパティにアクセスしている

解決：プロパティ名のスペルミスを確認するか、 必要なプロパティをインターフェースに追加します。

理解度確認クイズ

  1. Q1. オブジェクトを使うメリットは何ですか？

    回答を表示

    関連する複数のデータを1つのまとまりとして管理できることです。 個別の変数で管理するよりも、データの関係性が明確になり、コードが整理されます。

  2. Q2. インターフェースとは何ですか？

    回答を表示

    オブジェクトの構造を定義する「設計図」です。 どんなプロパティがあり、それぞれの型は何かを定義することで、同じ構造を持つ複数のオブジェクトを作成できます。

  3. Q3. オプショナルプロパティとreadonlyプロパティの違いは何ですか？

    回答を表示

    オプショナルプロパティ（?）は「あってもなくても良い」プロパティで、省略可能です。 readonlyプロパティは「一度設定したら変更できない」プロパティで、作成時にのみ値を設定できます。
