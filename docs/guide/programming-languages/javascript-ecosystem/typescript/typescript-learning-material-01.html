プログラミングとTypeScriptの世界へようこそ

プログラミングとは何か、TypeScriptがどのように「安全」なプログラムを書く手助けをしてくれるのか、身近な例を使って分かりやすく学びましょう。

この章で学ぶこと

  • プログラミングとは何か、その基本概念を理解する
  • TypeScriptの「型」がプログラムを安全にする仕組みを理解する
  • コンピュータと対話するための基本的な考え方を身につける

1.1 プログラミングって何だろう？

「プログラミング」という言葉を聞くと、難しそう、専門家だけのもの、と感じる方もいるかもしれません。 でも、実はプログラミングは「コンピュータに指示を出して、自動的に仕事をしてもらう」ための方法です。

例えば、朝起きてから家を出るまでの行動を考えてみましょう：

  1. 1. 目覚まし時計が鳴ったら起きる
  2. 2. 顔を洗う
  3. 3. 朝食を食べる
  4. 4. 着替える
  5. 5. 家を出る

これらは「順番に実行する手順」です。プログラミングも同じで、コンピュータに「何を」「どの順番で」「どのように」実行するかを指示するのです。

プログラミングの本質

プログラミングとは、コンピュータに「手順」を伝えることです。 人間が理解できる言葉（プログラミング言語）で書き、コンピュータが実行できる形に変換します。

1.2 レシピで学ぶプログラミングの考え方

プログラミングは「料理のレシピ」に似ています。レシピには以下のような情報が書かれていますね：

料理のレシピの例：カレーライス

材料（データ）

  • じゃがいも 2個
  • にんじん 1本
  • 玉ねぎ 1個
  • カレールー 1箱

手順（処理）

  1. 野菜を切る
  2. 鍋に油を熱する
  3. 野菜を炒める
  4. 水を加えて煮る
  5. カレールーを入れる

プログラミングでも同じように考えます：

データ（材料）

プログラムで扱う情報。数字、文字、リストなど、様々な形があります。

処理（手順）

データをどう加工するか。計算したり、表示したり、保存したりします。

1.3 TypeScriptとは？JavaScriptとの違い

TypeScriptは、JavaScriptというプログラミング言語に「型」という仕組みを追加したものです。

JavaScriptは、Webサイトを動かすために広く使われている言語です。しかし、JavaScriptには「型」がないため、 間違った使い方をしてもすぐには気づけず、プログラムが動いてから初めてエラーが出ることがあります。

特徴 JavaScript TypeScript
型の有無 型がない（自由） 型がある（安全）
エラー発見 実行時にエラーが出る 書いている時にエラーが分かる
学習難易度 やや簡単 少し時間がかかる
大規模開発 管理が難しい 管理しやすい

TypeScriptはJavaScriptの進化版

TypeScriptで書いたコードは、最終的にJavaScriptに変換されて実行されます。 つまり、TypeScriptを学ぶことで、JavaScriptも使えるようになります。

1.4 「型」って何？なぜ必要なの？

「型」とは、データの種類を決めるラベルのようなものです。

例えば、引き出しを想像してください。引き出しには「文房具」「書類」「薬」などのラベルが貼ってありますよね。 このラベルのおかげで、間違って文房具の引き出しに薬を入れることを防げます。

数値型 (number)

数字のデータ

例: 42, 3.14, -100

文字列型 (string)

文字のデータ

例: "こんにちは", "TypeScript"

真偽値型 (boolean)

はい/いいえのデータ

例: true, false

TypeScriptでは、変数（データを入れる箱）に型を指定します。こうすることで、間違った種類のデータを入れようとすると、 プログラムを書いている段階でエディタが「これは違いますよ！」と教えてくれます。

型の例 
// 数値型の変数
let age: number = 25;

// 文字列型の変数
let name: string = "太郎";

// 真偽値型の変数
let isStudent: boolean = true;

型があると何が良いの？

  • 間違いに早く気づける - プログラムを実行する前にエラーが分かる
  • エディタが賢く手伝ってくれる - 何が使えるか提案してくれる
  • コードが読みやすくなる - 何のデータか一目で分かる

1.5 TypeScriptで学ぶメリット

プログラミング初心者にとって、TypeScriptから学び始めることには多くのメリットがあります。

1

安全に学習できる

間違いをすぐに指摘してくれるので、正しい書き方が身につきやすい

2

構造的に考える力がつく

型を意識することで、データの構造や関係性を整理する習慣が身につく

3

エディタの支援が手厚い

Visual Studio Codeなどのエディタが、コードの補完や説明を自動で表示してくれる

4

実務で求められるスキル

近年、多くの企業がTypeScriptを採用しており、就職・転職にも有利

理解度確認クイズ

  1. Q1. プログラミングとは何ですか？

    回答を表示

    コンピュータに「何を」「どの順番で」「どのように」実行するかを指示することです。 料理のレシピのように、手順を明確に伝える作業だと言えます。

  2. Q2. TypeScriptの「型」とは何ですか？

    回答を表示

    データの種類を決めるラベルのようなものです。数値型(number)、文字列型(string)、真偽値型(boolean)などがあり、 間違った種類のデータを入れようとするとエラーで教えてくれます。

  3. Q3. TypeScriptをJavaScriptと比較した時の最大のメリットは何ですか？

    回答を表示

    型があることで、プログラムを実行する前に間違いに気づけることです。 これにより、安全で信頼性の高いコードを書くことができます。

  • プログラミングは、コンピュータに手順を伝える作業である
  • TypeScriptは、JavaScriptに「型」という仕組みを追加したもの
  • 「型」は、データの種類を明確にし、間違いを早期に発見できるようにする
  • TypeScriptで学ぶことで、安全で構造的なプログラミングの習慣が身につく
