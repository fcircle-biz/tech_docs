第13章

ジェネリクス：汎用的な型を作ろう

さまざまな型で使い回せる汎用的な関数やクラスを作るジェネリクスの基礎を学び、型の再利用性を高める方法を実例を通じて理解しましょう。

この章で学ぶこと

  • ジェネリクスとは何か、なぜ必要なのかを理解する
  • ジェネリック関数とジェネリッククラスの作り方を学ぶ
  • 型制約を使った安全なジェネリクスの使い方を習得する

13.1 ジェネリクスとは？

ジェネリクス（Generics）とは、「型をパラメータとして扱う」仕組みです。 これまでは関数に値を渡していましたが、ジェネリクスでは「型」自体を渡すことができます。

例えば、「箱に入れる」という処理を考えてみましょう。箱の中身が数値の場合もあれば、文字列の場合もあります。 ジェネリクスを使うと、どんな型にも対応できる汎用的な「箱」を作ることができます。

身近な例で理解：万能な収納ボックス

収納ボックスには、本を入れることもできれば、服を入れることも、おもちゃを入れることもできます。 ボックス自体は同じですが、中に入れるものの種類だけが違うのです。

服を入れた箱

おもちゃを入れた箱

ジェネリクスの本質

ジェネリクスを使うと、型を決めずに関数やクラスを定義し、 使うときに型を指定できます。これにより、同じコードを様々な型で再利用できます。

13.2 なぜジェネリクスが必要なのか

ジェネリクスがないと、同じような処理でも型ごとに別々の関数を作る必要があります。 これは非効率で、コードの重複が増えてしまいます。

❌ ジェネリクスを使わない場合（非効率） 
// 数値用の関数
function getFirstNumber(items: number[]): number {
  return items[0];
}

// 文字列用の関数
function getFirstString(items: string[]): string {
  return items[0];
}

// 真偽値用の関数
function getFirstBoolean(items: boolean[]): boolean {
  return items[0];
}

上記のコードでは、型が違うだけで同じ処理を3回も書いています。 ジェネリクスを使えば、これを1つの関数にまとめることができます。

✅ ジェネリクスを使う場合（効率的） 
// どんな型でも使える汎用関数
function getFirst(items: T[]): T {
  return items[0];
}

// 使用例：型を指定して使う
const firstNumber = getFirst([1, 2, 3]); // number型
const firstName = getFirst(["太郎", "花子"]); // string型
const firstFlag = getFirst([true, false]); // boolean型

<T>は、 「型のプレースホルダー（型変数）」です。Tは慣習的によく使われる名前で、 "Type"の頭文字を取っています。他の名前（E, K, Vなど）を使うこともできます。

13.3 ジェネリック関数の基本

ジェネリック関数は、<型変数名> を関数名の後に付けて定義します。

ジェネリック関数の構文 

function 関数名<T>(引数: T): T {
  // 処理
  return 戻り値;
}
ジェネリック関数の実例 
// 配列の最後の要素を取得する汎用関数
function getLast(items: T[]): T {
  return items[items.length - 1];
}

// 使用例
const numbers = [10, 20, 30, 40];
const lastNumber = getLast(numbers); // 40（number型）

const fruits = ["りんご", "バナナ", "みかん"];
const lastFruit = getLast(fruits); // "みかん"（string型）

// 型を明示的に指定することもできる
const lastValue = getLast(["A", "B", "C"]); // "C"

型推論が働く

TypeScriptは賢いので、多くの場合<型>を省略しても、 引数から型を自動的に推論してくれます。ただし、明示的に書いた方が分かりやすい場合もあります。

13.4 複数の型パラメータを使う

ジェネリクスでは、複数の型変数を使うこともできます。これにより、より複雑なパターンに対応できます。

複数の型パラメータを使う例 
// キーと値のペアを作る関数
function createPair(key: K, value: V): { key: K; value: V } {
  return { key, value };
}

// 使用例
const pair1 = createPair("名前", "太郎"); // { key: string, value: string }
const pair2 = createPair("年齢", 25); // { key: string, value: number }
const pair3 = createPair(1, true); // { key: number, value: boolean }

console.log(pair1); // { key: "名前", value: "太郎" }
console.log(pair2); // { key: "年齢", value: 25 }

この例では、K（Key）と V（Value）という 2つの型変数を使っています。それぞれ独立した型を表します。

実習：スワップ関数を作ろう

2つの値を入れ替えるswap関数を ジェネリクスを使って作成してみましょう。 

function swap(a: T, b: U): [U, T] {
  return [b, a];
}

// テスト
const result1 = swap("こんにちは", 123);
console.log(result1); // [123, "こんにちは"]

const result2 = swap(true, "TypeScript");
console.log(result2); // ["TypeScript", true]

13.5 ジェネリッククラスを作ろう

関数だけでなく、クラスもジェネリクスにすることができます。 これにより、様々な型のデータを扱えるクラスを作成できます。

ジェネリッククラスの例：データ保管庫 
// どんな型のデータでも保管できるクラス
class Box {
  private content: T;

  constructor(value: T) {
    this.content = value;
  }

  // データを取得
  getValue(): T {
    return this.content;
  }

  // データを更新
  setValue(value: T): void {
    this.content = value;
  }
}
ジェネリッククラスの使用例 
// 数値を保管する箱
const numberBox = new Box(100);
console.log(numberBox.getValue()); // 100
numberBox.setValue(200);
console.log(numberBox.getValue()); // 200

// 文字列を保管する箱
const stringBox = new Box("こんにちは");
console.log(stringBox.getValue()); // "こんにちは"

// オブジェクトを保管する箱
interface User {
  name: string;
  age: number;
}

const userBox = new Box({ name: "太郎", age: 25 });
console.log(userBox.getValue()); // { name: "太郎", age: 25 }

13.6 型制約：制限をつけて安全にする

ジェネリクスはどんな型でも受け入れられるので便利ですが、時には「特定の条件を満たす型だけ」に制限したい場合があります。 これを型制約（Type Constraints）と呼びます。

❌ 問題のあるコード 
// どんな型でもOKだが、問題が起きる可能性がある
function getLength(item: T): number {
  return item.length; // エラー！Tにlengthプロパティがあるとは限らない
}

上記のコードでは、T型が lengthプロパティを持っている保証がありません。 型制約を使って解決しましょう。

✅ 型制約を使った解決策 
// lengthプロパティを持つ型だけを受け入れる
interface HasLength {
  length: number;
}

function getLength(item: T): number {
  return item.length; // これで安全！
}

// 使用例
console.log(getLength("こんにちは")); // 5（文字列はlengthを持つ）
console.log(getLength([1, 2, 3])); // 3（配列もlengthを持つ）
// console.log(getLength(123)); // エラー！数値にはlengthがない

<T extends HasLength> という書き方で、「TはHasLengthインターフェースを 満たす型でなければならない」という制約をつけています。

型制約の実用例 
// 数値または文字列だけを受け入れる
function printValue(value: T): void {
  console.log(`値: ${value}`);
}

printValue(100); // OK
printValue("こんにちは"); // OK
// printValue(true); // エラー！boolean型は受け入れない

// 特定のプロパティを持つオブジェクトだけを受け入れる
function printName(obj: T): void {
  console.log(`名前: ${obj.name}`);
}

printName({ name: "太郎", age: 25 }); // OK（nameがある）
// printName({ age: 25 }); // エラー！nameがない

型制約で安全性を高める

型制約を使うと、ジェネリクスの柔軟性を保ちながら、 必要な条件を満たす型だけを受け入れることができます。 これにより、より安全なコードが書けます。

13.7 実践例：汎用的なデータ処理

ジェネリクスの実践的な使い方として、配列を扱うユーティリティ関数を作成してみましょう。

汎用的な配列ユーティリティ関数 
// 配列をシャッフルする汎用関数
function shuffle(array: T[]): T[] {
  const result = [...array]; // 元の配列を変更しないようコピー
  for (let i = result.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [result[i], result[j]] = [result[j], result[i]]; // 要素を入れ替え
  }
  return result;
}

// 配列から重複を除去する汎用関数
function unique(array: T[]): T[] {
  return [...new Set(array)];
}

// 配列を指定サイズに分割する汎用関数
function chunk(array: T[], size: number): T[][] {
  const result: T[][] = [];
  for (let i = 0; i < array.length; i += size) {
    result.push(array.slice(i, i + size));
  }
  return result;
}
ユーティリティ関数の使用例 
// 数値配列をシャッフル
const numbers = [1, 2, 3, 4, 5];
const shuffledNumbers = shuffle(numbers);
console.log(shuffledNumbers); // 例: [3, 1, 5, 2, 4]

// 文字列配列から重複を除去
const fruits = ["りんご", "バナナ", "りんご", "みかん", "バナナ"];
const uniqueFruits = unique(fruits);
console.log(uniqueFruits); // ["りんご", "バナナ", "みかん"]

// 配列を2つずつに分割
const items = [1, 2, 3, 4, 5, 6, 7];
const chunked = chunk(items, 2);
console.log(chunked); // [[1, 2], [3, 4], [5, 6], [7]]

実習：汎用的なフィルタ関数を作ろう

配列から条件に合う要素だけを取り出すfilter関数を ジェネリクスを使って作成してみましょう。 

// 汎用的なフィルタ関数
function filter(
  array: T[],
  predicate: (item: T) => boolean
): T[] {
  const result: T[] = [];
  for (const item of array) {
    if (predicate(item)) {
      result.push(item);
    }
  }
  return result;
}

// テスト：偶数だけを取り出す
const nums = [1, 2, 3, 4, 5, 6];
const evens = filter(nums, (n) => n % 2 === 0);
console.log(evens); // [2, 4, 6]

// テスト：3文字以上の文字列だけを取り出す
const words = ["犬", "ねこ", "うさぎ", "鳥"];
const longWords = filter(words, (w) => w.length >= 3);
console.log(longWords); // ["うさぎ"]

動作確認：総合演習

これまで学んだジェネリクスの知識を使って、以下のコードを実行して動作を確認しましょう。 

// ジェネリッククラス：スタック（後入れ先出し）
class Stack {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }
}

// 使用例
const numberStack = new Stack();
numberStack.push(10);
numberStack.push(20);
numberStack.push(30);

console.log(numberStack.peek()); // 30
console.log(numberStack.pop());  // 30
console.log(numberStack.pop());  // 20
console.log(numberStack.isEmpty()); // false

理解度確認クイズ

  1. Q1. ジェネリクスとは何ですか？

    回答を表示

    型をパラメータとして扱う仕組みです。これにより、どんな型でも使える汎用的な関数やクラスを作成できます。 型を決めずに定義し、使用時に型を指定することで、コードの再利用性が高まります。

  2. Q2. 型制約（Type Constraints）とは何ですか？

    回答を表示

    ジェネリクスで受け入れる型に制限をつけることです。 <T extends 型>という書き方で、 特定の条件を満たす型だけを受け入れるようにできます。

  3. Q3. ジェネリクスを使うメリットは何ですか？

    回答を表示

    主なメリットは以下の3つです：
    1. コードの重複を減らせる（型ごとに関数を作らなくてよい）
    2. 型安全性を保ちながら柔軟性を持たせられる
    3. 再利用可能なコードを書ける

よくあるエラーと対処法

エラー：Property 'length' does not exist on type 'T'

原因：型Tが必ずlengthプロパティを持つとは限らない

対処法：型制約を使って <T extends { length: number }> のように制限をつける

エラー：Type 'number' does not satisfy the constraint 'string'

原因：型制約に合わない型を指定している

対処法：型制約の条件を満たす型を使うか、型制約を緩める

この章のまとめ

  • ジェネリクスは、型をパラメータとして扱い、様々な型で使い回せるコードを書く仕組み
  • ジェネリック関数とジェネリッククラスを使うことで、型安全性を保ちながらコードを再利用できる
  • 型制約（<T extends 型>）を使うことで、受け入れる型に条件をつけられる
  • ジェネリクスを使うことで、型ごとに同じコードを書く必要がなくなり、保守性が向上する
