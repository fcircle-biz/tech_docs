条件分岐：場合に応じて処理を変えよう

「もし〜なら」という条件分岐を使って、状況に応じて異なる処理を行う方法を学びます。型と組み合わせることで、より安全な条件分岐が書けることを理解しましょう。

この章で学ぶこと

  • if文を使った基本的な条件分岐の方法を理解する
  • 比較演算子と論理演算子を使って複雑な条件を作れるようになる
  • 型を活用した安全な条件分岐の書き方を身につける

6.1 条件分岐って何だろう？

日常生活では、状況に応じて異なる行動をとることがよくあります。

日常の例：天気による行動の変化

  • もし晴れなら → 洗濯物を外に干す
  • もし雨なら → 洗濯物を室内に干す
  • それ以外なら → 様子を見る

プログラミングでも同じように、条件によって異なる処理を実行することができます。 これを「条件分岐」と呼びます。

条件分岐の本質

条件分岐は「もし〜なら」という判断をプログラムに組み込むことです。 状況に応じて異なる処理を実行できるようになります。

6.2 if文の基本

TypeScriptで条件分岐を行うには、if文を使います。 if文は「もし〜なら」という意味です。

if文の基本構文 
// 基本的なif文
let age: number = 18;

if (age >= 18) {
  console.log("成人です");
}

この例では、ageが18以上なら、 「成人です」と表示します。条件を満たさない場合は、何も実行されません。

if文の構造

1. if キーワード

「もし」という意味を表します

2. 条件（丸括弧の中）

判断する条件を書きます。結果は真（true）か偽（false）

3. 処理（波括弧の中）

条件が真の場合に実行される処理を書きます

温度による判断の例 
// 温度による服装の提案
let temperature: number = 28;

if (temperature >= 25) {
  console.log("暑いです！半袖がおすすめです");
}

6.3 else文：条件を満たさない場合の処理

条件を満たさない場合にも別の処理を実行したいときは、else文を使います。 elseは「そうでなければ」という意味です。

if-else文の例 
// 年齢による判定
let age: number = 15;

if (age >= 18) {
  console.log("成人です");
} else {
  console.log("未成年です");
}
// 出力: 未成年です

6.4 else if文：複数の条件を判定する

3つ以上の場合分けをしたいときは、else if文を使います。 これを使うと、複数の条件を順番にチェックできます。

成績評価の例 
// テストの点数による評価
let score: number = 75;

if (score >= 90) {
  console.log("優秀です！");
} else if (score >= 70) {
  console.log("良好です");
} else if (score >= 50) {
  console.log("もう少し頑張りましょう");
} else {
  console.log("復習が必要です");
}
// 出力: 良好です

条件は上から順番にチェックされる

else if文では、条件が上から順番にチェックされます。 最初に真（true）になった条件の処理が実行され、それ以降の条件はチェックされません。

6.5 比較演算子：条件を作る

条件分岐では、値を比較するための比較演算子を使います。

演算子 意味 例 結果
=== 等しい 5 === 5 true
!== 等しくない 5 !== 3 true
> より大きい 5 > 3 true
< より小さい 5 < 3 false
>= 以上（等しいか大きい） 5 >= 5 true
<= 以下（等しいか小さい） 5 <= 3 false
比較演算子の使用例 
// 様々な比較
let price: number = 1000;

if (price === 1000) {
  console.log("価格は1000円です");
}

if (price > 500) {
  console.log("500円より高いです");
}

if (price <= 2000) {
  console.log("2000円以下です");
}

== と === の違いに注意

TypeScriptでは、===（厳密等価演算子）を使うことが推奨されます。 ==は型が異なっても比較してしまうため、意図しない結果になることがあります。

6.6 論理演算子：複数の条件を組み合わせる

複数の条件を組み合わせたいときは、論理演算子を使います。

&&

AND（かつ）

両方の条件が真の場合に真

a && b

||

OR（または）

どちらかの条件が真なら真

a || b

!

NOT（否定）

条件の真偽を反転させる

!a

AND演算子（&&）の例 
// 年齢と身長の両方をチェック
let age: number = 20;
let height: number = 165;

if (age >= 18 && height >= 160) {
  console.log("ジェットコースターに乗れます");
} else {
  console.log("ジェットコースターには乗れません");
}
OR演算子（||）の例 
// 曜日による判定
let day: string = "土曜日";

if (day === "土曜日" || day === "日曜日") {
  console.log("休日です");
} else {
  console.log("平日です");
}
NOT演算子（!）の例 
// ログイン状態の確認
let isLoggedIn: boolean = false;

if (!isLoggedIn) {
  console.log("ログインしてください");
} else {
  console.log("ようこそ！");
}

論理演算子の真理値表

AND（&&）の結果

  • true && true → true
  • true && false → false
  • false && true → false
  • false && false → false

OR（||）の結果

  • true || true → true
  • true || false → true
  • false || true → true
  • false || false → false

6.7 型を活用した安全な条件分岐

TypeScriptの型システムを活用することで、より安全な条件分岐を書くことができます。

型による安全な条件分岐 
// 型を明示することで安全性向上
let status: "success" | "error" | "loading" = "success";

if (status === "success") {
  console.log("処理が成功しました");
} else if (status === "error") {
  console.log("エラーが発生しました");
} else if (status === "loading") {
  console.log("読み込み中です");
}
// statusには3つの値しか入らないことが保証される
typeof による型チェック 
// typeofを使った型の判定
function processInput(input: string | number): void {
  if (typeof input === "string") {
    // この中では input は string 型として扱われる
    console.log("文字列の長さ: " + input.length);
  } else {
    // この中では input は number 型として扱われる
    console.log("数値の2倍: " + (input * 2));
  }
}

processInput("こんにちは");  // 文字列の長さ: 5
processInput(10);           // 数値の2倍: 20

型ガードのメリット

typeofで型をチェックすると、 TypeScriptがif文の中で変数の型を自動的に絞り込んでくれます。 これにより、型安全なコードが書けます。

実習：成績判定プログラムを作ろう

課題内容

テストの点数を受け取り、以下のルールで評価を表示するプログラムを作成してください：

  • 90点以上：「A評価：優秀です！」
  • 80点以上90点未満：「B評価：良好です」
  • 70点以上80点未満：「C評価：合格です」
  • 60点以上70点未満：「D評価：もう少し頑張りましょう」
  • 60点未満：「F評価：不合格です」
解答例を表示
成績判定プログラム 
// 成績判定プログラム
let score: number = 85;

if (score >= 90) {
  console.log("A評価：優秀です！");
} else if (score >= 80) {
  console.log("B評価：良好です");
} else if (score >= 70) {
  console.log("C評価：合格です");
} else if (score >= 60) {
  console.log("D評価：もう少し頑張りましょう");
} else {
  console.log("F評価：不合格です");
}

// 出力: B評価：良好です

よくあるエラーと解決方法

エラー1：条件式で = を使ってしまう

if (age = 18)は代入になってしまいます

解決策： 比較には===を使います

エラー2：波括弧を忘れる

処理が1行でも波括弧{}を書くことを推奨します

理由： あとで処理を追加する時に間違いを防げます

エラー3：論理演算子の優先順位

複雑な条件では、括弧()を使って明確にしましょう

if ((age >= 18 && hasLicense) || isSpecialCase)

理解度確認クイズ

  1. Q1. 以下のコードの出力は何ですか？

    let age: number = 17;
if (age >= 18) {
  console.log("成人");
} else {
  console.log("未成年");
}
    回答を表示

    「未成年」が出力されます。ageが17なので、 age >= 18は偽（false）となり、else文の処理が実行されます。

  2. Q2. &&演算子と||演算子の違いは何ですか？

    回答を表示

    &&（AND）は両方の条件が真の場合に真を返します。 ||（OR）はどちらかの条件が真なら真を返します。

  3. Q3. TypeScriptで型を活用した条件分岐のメリットは何ですか？

    回答を表示

    typeofなどで型をチェックすることで、 TypeScriptが条件分岐の中で変数の型を自動的に絞り込んでくれます。 これにより、型に応じた安全な処理が書けます。

  • if文、else文、else if文を使って条件に応じた処理ができる
  • 比較演算子（===, !==, >, <, >=, <=）で値を比較できる
  • 論理演算子（&&, ||, !）で複数の条件を組み合わせられる
  • TypeScriptの型システムを活用することで、より安全な条件分岐が書ける
