画像とアセットの扱い

ローカル画像、ネットワーク画像の表示方法、アセットの管理方法を学びます。

この章で学ぶこと

  • アセットフォルダの構成とpubspec.yamlの設定方法
  • ローカル画像（プロジェクト内画像）の表示方法
  • ネットワーク画像（Web上の画像）の表示と読み込み処理
  • 画像のサイズ調整、フィット方法、装飾の実装

14.1 アセットとは

アセット（Assets）とは、アプリに含める静的なファイルのことです。画像、フォント、JSONファイル、音声ファイルなどが該当します。

代表的なアセットの種類

  • 画像ファイル: PNG、JPG、GIF、SVG など
  • フォントファイル: TTF、OTF など
  • データファイル: JSON、CSV など
  • 音声・動画: MP3、MP4 など

典型的なアセットフォルダ構成

プロジェクト構成 
my_flutter_app/
├── lib/
│   └── main.dart
├── assets/
│   ├── images/
│   │   ├── logo.png
│   │   ├── profile.jpg
│   │   └── background.jpg
│   ├── fonts/
│   │   └── custom_font.ttf
│   └── data/
│       └── config.json
└── pubspec.yaml

14.2 pubspec.yamlでのアセット登録

アセットをアプリで使用するには、pubspec.yamlファイルにアセットのパスを登録する必要があります。

pubspec.yamlの設定

pubspec.yaml 
flutter:
  assets:
    # フォルダ全体を指定（推奨）
    - assets/images/
    # 個別ファイルを指定
    - assets/images/logo.png
    - assets/data/config.json

重要ポイント

  • インデント: スペース2つでインデント（タブ不可）
  • フォルダ指定: フォルダ指定の場合は最後に/を付ける
  • 変更後: pubspec.yamlを編集したらflutter pub getを実行
  • ホットリロード不可: アセット追加時はアプリを再起動する必要があります

14.3 ローカル画像の表示

プロジェクト内に配置した画像を表示するにはImage.asset()を使用します。

基本的な画像表示

Dart 
import 'package:flutter/material.dart';

class MyImagePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('ローカル画像の表示'),
      ),
      body: Center(
        child: Image.asset(
          'assets/images/logo.png', // アセットパス
          width: 200, // 幅を指定
          height: 200, // 高さを指定
        ),
      ),
    );
  }
}

画像のフィット方法（BoxFit）

Dart 
Image.asset(
  'assets/images/background.jpg',
  width: 300,
  height: 200,
  fit: BoxFit.cover, // フィット方法を指定
)

主なBoxFitの種類

BoxFit.cover

枠全体を画像で覆う（はみ出した部分は切り取られる）

BoxFit.contain

画像全体を枠内に収める（余白ができる可能性）

BoxFit.fill

枠にぴったり合わせる（アスペクト比が変わる）

BoxFit.fitWidth

幅を枠に合わせる（高さは自動調整）

14.4 ネットワーク画像の表示

インターネット上の画像を表示するにはImage.network()を使用します。

ネットワーク画像の表示

Dart 
Image.network(
  'https://picsum.photos/300/200',
  width: 300,
  height: 200,
  fit: BoxFit.cover,
)

ローディング表示の追加

Dart 
Image.network(
  'https://picsum.photos/300/200',
  loadingBuilder: (context, child, loadingProgress) {
    if (loadingProgress == null) {
      return child; // 読み込み完了
    }
    return Center(
      child: CircularProgressIndicator(), // 読み込み中
    );
  },
  errorBuilder: (context, error, stackTrace) {
    return Icon(Icons.error); // エラー時
  },
)

14.5 画像の装飾

Containerで画像を囲むことで、枠線、角丸、影などの装飾を追加できます。

角丸画像（ClipRRect）

Dart 
ClipRRect(
  borderRadius: BorderRadius.circular(16), // 角丸半径
  child: Image.asset(
    'assets/images/profile.jpg',
    width: 100,
    height: 100,
    fit: BoxFit.cover,
  ),
)

円形画像（CircleAvatar）

Dart 
CircleAvatar(
  radius: 50, // 半径
  backgroundImage: AssetImage('assets/images/profile.jpg'),
)

枠線と影の追加

Dart 
Container(
  decoration: BoxDecoration(
    border: Border.all(color: Colors.grey, width: 2),
    borderRadius: BorderRadius.circular(12),
    boxShadow: [
      BoxShadow(
        color: Colors.black26,
        blurRadius: 10,
        offset: Offset(0, 4),
      ),
    ],
  ),
  child: ClipRRect(
    borderRadius: BorderRadius.circular(10),
    child: Image.asset(
      'assets/images/logo.png',
      width: 150,
      height: 150,
      fit: BoxFit.cover,
    ),
  ),
)

実習: 画像ギャラリーアプリ

ローカル画像とネットワーク画像を組み合わせた簡単なギャラリーアプリを作成します。

ステップ1: アセットの準備

  1. プロジェクトルートにassets/images/フォルダを作成
  2. 任意の画像ファイル（3枚程度）を配置
  3. pubspec.yamlにassets/images/を登録
  4. flutter pub getを実行

ステップ2: ギャラリーページの作成

lib/gallery_page.dart 
import 'package:flutter/material.dart';

class GalleryPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('画像ギャラリー')),
      body: GridView.count(
        crossAxisCount: 2, // 2列
        padding: EdgeInsets.all(16),
        crossAxisSpacing: 12,
        mainAxisSpacing: 12,
        children: [
          // ローカル画像
          _buildImageCard(
            Image.asset('assets/images/image1.jpg', fit: BoxFit.cover),
            'ローカル画像1',
          ),
          // ネットワーク画像
          _buildImageCard(
            Image.network('https://picsum.photos/200', fit: BoxFit.cover),
            'ネットワーク画像',
          ),
        ],
      ),
    );
  }

  Widget _buildImageCard(Widget image, String label) {
    return Column(
      children: [
        Expanded(
          child: ClipRRect(
            borderRadius: BorderRadius.circular(12),
            child: image,
          ),
        ),
        SizedBox(height: 8),
        Text(label, style: TextStyle(fontSize: 12)),
      ],
    );
  }
}

ステップ3: 動作確認

  1. アプリを再起動（アセット追加時は再起動必須）
  2. ギャラリーページを表示し、画像が正しく表示されることを確認
  3. ネットワーク画像の読み込み動作を確認

よくあるトラブルと対処法

画像が表示されない

  • pubspec.yamlに正しくアセットが登録されているか確認
  • flutter pub getを実行したか確認
  • アプリを再起動（ホットリロードでは反映されない）
  • ファイルパスの大文字小文字が正しいか確認

ネットワーク画像が表示されない

  • インターネット接続を確認
  • URLが正しいか確認
  • Android: AndroidManifest.xmlにINTERNET権限があるか確認
  • iOS: HTTP接続の場合はInfo.plistにNSAppTransportSecurity設定が必要

画像のアスペクト比が崩れる

  • fit: BoxFit.containまたはBoxFit.coverを使用
  • widthとheightのどちらか一方のみを指定してアスペクト比を保つ

理解度チェッククイズ

Q1

プロジェクト内の画像を表示するために正しいウィジェットはどれですか？

解答を見る
正解: B
プロジェクト内のアセット画像を表示する場合はImage.asset()を使用します。Image.network()はURLから画像を読み込む場合、Image.file()は端末のファイルシステムから読み込む場合に使用します。
Q2

画像の角を丸くするために使用するウィジェットはどれですか？

解答を見る
正解: B
ClipRRectウィジェットを使うと、子ウィジェット（画像など）の角を丸くできます。BorderRadiusは角丸の設定値で、ウィジェットではありません。
Q3

BoxFit.coverとBoxFit.containの違いは何ですか？

解答を見る
正解: B
BoxFit.coverは枠全体を画像で覆います（はみ出した部分は切り取られます）。BoxFit.containは画像全体を枠内に収めます（余白ができる可能性があります）。

第14章のまとめ

  • アセットはアプリに含める静的ファイル（画像、フォント、データなど）
  • pubspec.yamlでアセットを登録し、flutter pub getを実行
  • ローカル画像はImage.asset()、ネットワーク画像はImage.network()で表示
  • BoxFitで画像のフィット方法を制御（cover、contain、fill等）
  • ClipRRectで角丸、CircleAvatarで円形画像を作成
  • ネットワーク画像ではloadingBuilderとerrorBuilderで状態管理
