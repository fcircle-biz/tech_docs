ローカルデータの保存

shared_preferencesを使った簡易データ保存、アプリ設定の永続化方法を学びます。

この章で学ぶこと

  • ローカルデータ保存の必要性と種類を理解する
  • shared_preferencesパッケージの導入と使い方を学ぶ
  • 文字列、数値、真偽値、リストデータの保存と読み込みを実装する
  • アプリ設定の永続化（ダークモード、言語設定など）を実践する

18.1 ローカルデータ保存の必要性

モバイルアプリでは、ユーザーの設定や操作履歴などをアプリを閉じても保持しておきたいケースがよくあります。これを実現するのがローカルデータ保存です。

アプリ設定の保存

ダークモード、言語設定、通知ON/OFF

ユーザー情報

ログイン状態、ユーザー名、最終ログイン日時

操作履歴

検索履歴、閲覧履歴、お気に入りアイテム

一時データ

入力途中のフォーム、下書き保存

この章で学ぶ shared_preferences

shared_preferencesは、キーバリュー形式で簡易データを保存できるパッケージです。アプリ設定やユーザー情報など、複雑でないデータの永続化に最適です。

18.2 shared_preferencesの導入

shared_preferencesパッケージをプロジェクトに追加する手順を解説します。

実習: shared_preferencesパッケージの追加

手順1: pubspec.yamlにパッケージを追加

pubspec.yaml 
dependencies:
  flutter:
    sdk: flutter
  shared_preferences: ^2.2.2  # 最新版を確認してください

手順2: パッケージのインストール

Dartファイルでshared_preferencesをインポートします。

import 'package:shared_preferences/shared_preferences.dart';

18.3 基本的なデータの保存と読み込み

shared_preferencesでは、文字列、数値、真偽値、リストなど様々な型のデータを保存できます。

データ型 保存メソッド 読み込みメソッド
文字列（String） setString() getString()
整数（int） setInt() getInt()
小数（double） setDouble() getDouble()
真偽値（bool） setBool() getBool()
文字列リスト（List<String>） setStringList() getStringList()

基本的な使い方

Dart 
// データを保存する
Future<void> saveData() async {
  // SharedPreferencesインスタンスを取得
  final prefs = await SharedPreferences.getInstance();

  // 文字列を保存
  await prefs.setString('username', '太郎');

  // 整数を保存
  await prefs.setInt('age', 25);

  // 真偽値を保存
  await prefs.setBool('isDarkMode', true);

  // リストを保存
  await prefs.setStringList('favorites', ['Apple', 'Banana', 'Orange']);
}

// データを読み込む
Future<void> loadData() async {
  final prefs = await SharedPreferences.getInstance();

  // データを読み込み（存在しない場合はnullまたはデフォルト値）
  String? username = prefs.getString('username');
  int? age = prefs.getInt('age');
  bool isDarkMode = prefs.getBool('isDarkMode') ?? false;  // デフォルトfalse
  List<String>? favorites = prefs.getStringList('favorites');
}

ポイント

  • SharedPreferences.getInstance()は非同期処理なのでawaitが必要
  • 保存メソッド（setXxx()）も非同期なのでawaitで完了を待つ
  • データが存在しない場合はnullが返るので、??でデフォルト値を指定すると安全

18.4 実践例 - ダークモード設定の保存

ユーザーが設定したダークモードのON/OFFをshared_preferencesで保存し、アプリ起動時に復元する実装を学びます。

実習: ダークモード設定保存アプリ

main.dart 
import 'package:flutter/material.dart';
import 'package:shared_preferences/shared_preferences.dart';

void main() => runApp(MyApp());

class MyApp extends StatefulWidget {
  @override
  _MyAppState createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  bool _isDarkMode = false;  // ダークモード状態

  @override
  void initState() {
    super.initState();
    _loadDarkModeSetting();  // アプリ起動時に設定を読み込み
  }

  // ダークモード設定を読み込む
  Future<void> _loadDarkModeSetting() async {
    final prefs = await SharedPreferences.getInstance();
    setState(() {
      // 保存されている設定を読み込み（デフォルトはfalse）
      _isDarkMode = prefs.getBool('isDarkMode') ?? false;
    });
  }

  // ダークモード設定を保存する
  Future<void> _saveDarkModeSetting(bool value) async {
    final prefs = await SharedPreferences.getInstance();
    await prefs.setBool('isDarkMode', value);
    setState(() {
      _isDarkMode = value;
    });
  }

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      // ダークモードに応じてテーマを切り替え
      theme: _isDarkMode ? ThemeData.dark() : ThemeData.light(),
      home: SettingsPage(
        isDarkMode: _isDarkMode,
        onToggleDarkMode: _saveDarkModeSetting,
      ),
    );
  }
}

設定画面の実装

Dart 
class SettingsPage extends StatelessWidget {
  final bool isDarkMode;
  final Function(bool) onToggleDarkMode;

  SettingsPage({required this.isDarkMode, required this.onToggleDarkMode});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('設定')),
      body: ListView(
        children: [
          SwitchListTile(
            title: Text('ダークモード'),
            subtitle: Text('外観を暗くします'),
            value: isDarkMode,
            onChanged: (value) {
              onToggleDarkMode(value);  // 保存処理を呼び出す
            },
          ),
        ],
      ),
    );
  }
}

動作確認

  1. アプリを起動してダークモードをONに切り替える
  2. アプリを完全に終了する（タスクキルする）
  3. 再度アプリを起動すると、ダークモードがONのまま維持されている

18.5 その他の操作

shared_preferencesには、保存・読み込み以外にも便利な機能があります。

データの削除

Dart 
// 特定のキーのデータを削除
Future<void> removeData() async {
  final prefs = await SharedPreferences.getInstance();
  await prefs.remove('username');  // 'username'キーのデータを削除
}

// すべてのデータを削除
Future<void> clearAllData() async {
  final prefs = await SharedPreferences.getInstance();
  await prefs.clear();  // 保存されているすべてのデータを削除
}

データの存在確認

Dart 
// 特定のキーが存在するか確認
Future<void> checkDataExists() async {
  final prefs = await SharedPreferences.getInstance();

  if (prefs.containsKey('username')) {
    print('usernameは保存されています');
  } else {
    print('usernameは保存されていません');
  }
}

保存されているすべてのキーを取得

Dart 
// 保存されているすべてのキーを取得
Future<void> getAllKeys() async {
  final prefs = await SharedPreferences.getInstance();
  Set<String> keys = prefs.getKeys();

  print('保存されているキー一覧: $keys');
}

18.6 注意点とベストプラクティス

セキュリティに注意

shared_preferencesに保存されたデータは暗号化されません。以下のデータは保存しないでください。

  • パスワード、PINコード
  • クレジットカード情報
  • 個人を特定できる機密情報

機密情報はflutter_secure_storageなど暗号化されたストレージを使用してください。

ベストプラクティス

  • キー名は定数化する: タイポを防ぐため、キー名を定数として管理する 
    class PrefsKeys {
  static const String isDarkMode = 'isDarkMode';
  static const String username = 'username';
}
  • デフォルト値を設定する: ??演算子でnull時のデフォルト値を指定する
  • 大量データは保存しない: shared_preferencesは小規模データ向け。大量のデータはSQLiteやファイルストレージを使用

理解度チェッククイズ

Q1

shared_preferencesで保存できないデータ型はどれですか？

解答を見る
正解: C
shared_preferencesはString、int、double、bool、List<String>の5つの型のみ対応しています。Mapオブジェクトは直接保存できません（JSON文字列に変換すれば保存可能）。
Q2

shared_preferencesに保存してはいけないデータはどれですか？

解答を見る
正解: A
shared_preferencesのデータは暗号化されないため、パスワードなどの機密情報を保存してはいけません。機密データにはflutter_secure_storageなど暗号化されたストレージを使用します。
Q3

次のコードの空欄に入る適切なコードはどれですか？

final prefs = await SharedPreferences.getInstance();
int count = prefs.getInt('count') _____;
解答を見る
正解: B
??はnull合体演算子で、左側がnullの場合に右側の値を返します。データが存在しない場合のデフォルト値を設定する際によく使用されます。

第18章のまとめ

  • shared_preferencesは、キーバリュー形式で簡易データを永続化できるパッケージ
  • String、int、double、bool、List<String>の5つの型に対応
  • setXxx()でデータを保存、getXxx()でデータを読み込む
  • データが存在しない場合はnullが返るため、??でデフォルト値を指定
  • ダークモード設定などアプリ設定の永続化に最適
  • データは暗号化されないため、パスワードなど機密情報は保存しない
