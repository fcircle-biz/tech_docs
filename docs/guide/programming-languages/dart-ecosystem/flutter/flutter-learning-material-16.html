非同期処理の基礎

Future、async/awaitを使った非同期処理の基本を学びます。API通信の準備として、待機処理の概念を理解します。

この章で学ぶこと

  • 同期処理と非同期処理の違いを理解する
  • Futureの概念と使い方を学ぶ
  • async/awaitキーワードで非同期処理を記述する
  • 待機処理とローディング表示を実装する

16.1 非同期処理とは

非同期処理（ひどうきしょり）とは、ある処理の完了を待たずに次の処理を実行する仕組みです。モバイルアプリでは、ネットワーク通信やファイル読み込みなど、時間のかかる処理を行う際に必須の概念です。

同期処理

1つの処理が完了するまで次の処理を待機

  • • 処理Aが終わるまで処理Bを開始できない
  • • 時間のかかる処理でアプリがフリーズする
  • • ユーザー操作が受け付けられない

非同期処理

処理の完了を待たずに次の処理を実行

  • • 処理Aの完了を待たずに処理Bを実行
  • • アプリが応答性を保つ
  • • ユーザー操作を継続できる

同期処理 vs. 非同期処理の流れ

graph TB subgraph 同期処理 A1["処理開始"] --> A2["処理A（3秒）"] A2 --> A3["待機中..."] A3 --> A4["処理B（1秒）"] A4 --> A5["完了（4秒後）"] end subgraph 非同期処理 B1["処理開始"] --> B2["処理A（3秒）
バックグラウンド実行"] B1 --> B3["処理B（1秒）
すぐ実行"] B2 --> B4["処理A完了時に通知"] B3 --> B5["完了（1秒後）"] end

実生活での例

同期処理: レストランで注文した料理が完成するまで、他の客の注文を一切受け付けない（非効率）

非同期処理: 複数の注文を並行して調理し、完成したものから順に提供する（効率的）

16.2 Futureとは

Futureは、「未来に得られる値」を表すDartのクラスです。非同期処理の結果をラップ（包む）し、処理が完了したときに値を受け取ることができます。

Futureの概念

flowchart LR A["非同期処理を開始"] --> B["Future<String>を返す"] B --> C{"処理完了?"} C -->|完了| D["値を取得"] C -->|未完了| E["待機中..."] E --> C
basic_future.dart 
// 3秒後に文字列を返す非同期関数
Future<String> fetchData() {
  // Future.delayed: 指定時間後に値を返す
  return Future.delayed(
    Duration(seconds: 3), // 3秒待機
    () => 'データ取得完了!', // 完了時に返す値
  );
}

void main() {
  print('処理開始');

  // fetchDataを呼び出し（すぐに返る）
  fetchData().then((data) {
    print(data); // 3秒後に表示: データ取得完了!
  });

  print('処理継続中...');
}
// 実行結果の順序:
// 処理開始
// 処理継続中...
// （3秒待機）
// データ取得完了!

ポイント

Future.delayed() は指定した時間後に処理を実行する関数です。API通信のシミュレーションに便利です。

16.3 async/awaitの使い方

async/awaitは、非同期処理をより直感的に記述するためのキーワードです。.then()を使うよりも読みやすく、同期処理のように書けます。

async

非同期関数であることを宣言するキーワード

Future<void> funcName() async { }

await

Futureの完了を待つキーワード（async内でのみ使用可能）

var data = await fetchData();
async_await_example.dart 
// ユーザー情報を取得する非同期関数
Future<String> fetchUserName() async {
  // 2秒待機してユーザー名を返す
  await Future.delayed(Duration(seconds: 2));
  return '山田太郎';
}

// メイン処理
void main() async {
  print('ユーザー情報取得開始');

  // awaitで完了を待つ
  String userName = await fetchUserName();

  print('ユーザー名: $userName'); // 2秒後に表示
  print('処理完了');
}
// 実行結果:
// ユーザー情報取得開始
// （2秒待機）
// ユーザー名: 山田太郎
// 処理完了

重要ポイント

awaitを使うことで、非同期処理を同期処理のように上から順に読めるコードで記述できます。

16.4 複数の非同期処理を扱う

実際のアプリでは、複数のAPIを呼び出したり、複数のデータを並行して取得することがあります。DartではFuture.wait()を使って複数の非同期処理を効率的に実行できます。

順次実行（遅い）

// 処理Aが完了してから処理Bを実行
var dataA = await fetchA(); // 2秒
var dataB = await fetchB(); // 2秒
// 合計: 4秒

並列実行（速い）

// 処理AとBを同時に実行
var results = await Future.wait([
  fetchA(), // 2秒
  fetchB(), // 2秒
]);
// 合計: 2秒
future_wait_example.dart 
// ユーザー情報を取得
Future<String> fetchUserName() async {
  await Future.delayed(Duration(seconds: 2));
  return '山田太郎';
}

// 投稿件数を取得
Future<int> fetchPostCount() async {
  await Future.delayed(Duration(seconds: 2));
  return 42;
}

void main() async {
  print('データ取得開始');

  // 並列実行
  var results = await Future.wait([
    fetchUserName(),
    fetchPostCount(),
  ]);

  String userName = results[0] as String;
  int postCount = results[1] as int;

  print('ユーザー名: $userName');
  print('投稿件数: $postCount');
}
// 実行時間: 約2秒（並列実行のため）

16.5 Flutterでの非同期処理とローディング表示

Flutterアプリでは、データ取得中にローディングインジケーター（読み込み中の表示）を表示することで、ユーザーに処理中であることを伝えます。

実習 16-1: ローディング表示を伴うデータ取得

3秒後にメッセージを表示するアプリを作成し、待機中はローディングインジケーターを表示します。

手順

  1. 1 新しいFlutterプロジェクトを作成または既存プロジェクトのlib/main.dartを開く
  2. 2 以下のコードを入力
main.dart 
import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: AsyncLoadingPage(),
    );
  }
}

class AsyncLoadingPage extends StatefulWidget {
  @override
  _AsyncLoadingPageState createState() => _AsyncLoadingPageState();
}

class _AsyncLoadingPageState extends State<AsyncLoadingPage> {
  String _message = ''; // 取得したメッセージ
  bool _isLoading = false; // ローディング状態

  // データ取得をシミュレート
  Future<String> fetchMessage() async {
    await Future.delayed(Duration(seconds: 3));
    return 'データ取得完了しました！';
  }

  // ボタン押下時の処理
  void _loadData() async {
    setState(() {
      _isLoading = true; // ローディング開始
      _message = '';
    });

    // データ取得（3秒待機）
    String result = await fetchMessage();

    setState(() {
      _isLoading = false; // ローディング終了
      _message = result;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('非同期処理デモ'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // ローディング中はCircularProgressIndicator表示
            if (_isLoading)
              CircularProgressIndicator()
            else if (_message.isNotEmpty)
              Text(
                _message,
                style: TextStyle(fontSize: 18),
              )
            else
              Text('ボタンを押してデータ取得'),
            SizedBox(height: 30),
            ElevatedButton(
              onPressed: _isLoading ? null : _loadData,
              child: Text('データ取得'),
            ),
          ],
        ),
      ),
    );
  }
}

動作確認

アプリを実行し、「データ取得」ボタンを押します。

期待される結果

  • 1. ボタンを押すと円形のローディングインジケーターが表示される
  • 2. 3秒間待機（ボタンは無効化される）
  • 3. 「データ取得完了しました！」というメッセージが表示される
  • 4. 再度ボタンを押すと同じ動作を繰り返す

ポイント

  • • CircularProgressIndicator(): 円形のローディング表示
  • • _isLoadingでローディング状態を管理
  • • ローディング中はボタンを無効化（onPressed: _isLoading ? null : _loadData）

16.6 エラーハンドリング

非同期処理では、ネットワークエラーやタイムアウトなど、様々なエラーが発生する可能性があります。try-catch文を使ってエラーを適切に処理します。

error_handling.dart 
// エラーが発生する可能性がある非同期関数
Future<String> fetchDataWithError() async {
  await Future.delayed(Duration(seconds: 2));

  // エラーを発生させる
  throw Exception('サーバーに接続できません');
}

void main() async {
  try {
    // エラーが発生する可能性がある処理
    String data = await fetchDataWithError();
    print('データ: $data');
  } catch (e) {
    // エラーが発生した場合の処理
    print('エラーが発生しました: $e');
  } finally {
    // 成功・失敗に関わらず実行される処理
    print('処理終了');
  }
}
// 実行結果:
// （2秒待機）
// エラーが発生しました: Exception: サーバーに接続できません
// 処理終了

注意

実際のアプリでは、エラー発生時にユーザーにわかりやすいメッセージを表示することが重要です。「エラーが発生しました」だけでなく、「ネットワークに接続できません。Wi-Fiまたはモバイルデータ通信を確認してください」のような具体的な案内を心がけましょう。
