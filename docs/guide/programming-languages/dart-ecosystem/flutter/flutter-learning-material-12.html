画面遷移（ナビゲーション）

Navigatorを使った画面遷移の基本、pushとpopの概念、画面間のデータ受け渡しを学びます。

この章で学ぶこと

  • Navigatorクラスを使った画面遷移の基本
  • pushとpopメソッドの違いとスタック構造の理解
  • 画面間でデータを渡す方法と受け取る方法
  • 名前付きルートを使った画面遷移の管理

12.1 Navigatorとは

Navigatorは、Flutterで画面遷移を管理するためのクラスです。複数の画面を持つアプリでは、ユーザーが「次の画面へ進む」「前の画面に戻る」といった操作を行います。

Navigatorはスタック構造で画面を管理します。新しい画面を積み重ねるように追加し、前の画面に戻るときは一番上の画面を取り除きます。

スタック構造とは

本や書類を積み重ねるイメージです。新しいページを一番上に乗せ、不要になったら一番上のページから取り除きます。

  • push（積む）: 新しい画面を一番上に追加
  • pop（取り除く）: 一番上の画面を削除して前の画面に戻る

12.2 基本的な画面遷移（push と pop）

Navigatorの基本的な使い方を学びます。Navigator.push()で新しい画面に進み、Navigator.pop()で前の画面に戻ります。

基本的な画面遷移の例 
import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: HomePage(),
    );
  }
}

// ホーム画面
class HomePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('ホーム画面')),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            // 詳細画面へ遷移（push）
            Navigator.push(
              context,
              MaterialPageRoute(builder: (context) => DetailPage()),
            );
          },
          child: Text('詳細画面へ'),
        ),
      ),
    );
  }
}

// 詳細画面
class DetailPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('詳細画面')),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            // 前の画面に戻る（pop）
            Navigator.pop(context);
          },
          child: Text('戻る'),
        ),
      ),
    );
  }
}

コードの解説

  • Navigator.push()で新しい画面（DetailPage）に遷移
  • MaterialPageRouteは画面遷移のアニメーションを提供
  • builderパラメータで遷移先の画面を指定
  • Navigator.pop()でスタックから現在の画面を削除し、前の画面に戻る
  • AppBarの戻るボタンは自動的に追加され、Navigator.pop()が呼ばれる

12.3 データを渡す画面遷移

画面遷移時にデータを次の画面に渡すことができます。例えば、リストアイテムをタップして詳細画面に遷移するとき、そのアイテムの情報を詳細画面に渡します。

データを渡す画面遷移 
// ホーム画面
class HomePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('商品リスト')),
      body: ListView(
        children: [
          ListTile(
            title: Text('りんご'),
            subtitle: Text('¥150'),
            onTap: () {
              // 詳細画面にデータを渡す
              Navigator.push(
                context,
                MaterialPageRoute(
                  builder: (context) => DetailPage(
                    productName: 'りんご',
                    price: 150,
                  ),
                ),
              );
            },
          ),
          ListTile(
            title: Text('バナナ'),
            subtitle: Text('¥100'),
            onTap: () {
              Navigator.push(
                context,
                MaterialPageRoute(
                  builder: (context) => DetailPage(
                    productName: 'バナナ',
                    price: 100,
                  ),
                ),
              );
            },
          ),
        ],
      ),
    );
  }
}

// 詳細画面（コンストラクタでデータを受け取る）
class DetailPage extends StatelessWidget {
  final String productName;
  final int price;

  // コンストラクタでデータを受け取る
  DetailPage({required this.productName, required this.price});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('商品詳細')),
      body: Padding(
        padding: EdgeInsets.all(16),
        child: Column(
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            Text('商品名: $productName', style: TextStyle(fontSize: 24)),
            SizedBox(height: 16),
            Text('価格: ¥$price', style: TextStyle(fontSize: 20)),
          ],
        ),
      ),
    );
  }
}

データ渡しのポイント

  • 遷移先の画面クラスにコンストラクタでパラメータを定義
  • requiredを付けると必須パラメータになる
  • 受け取ったデータはfinal変数に格納
  • 複雑なデータはクラスにまとめると管理しやすい

12.4 データを返す画面遷移

画面を閉じるときに、前の画面にデータを返すこともできます。例えば、設定画面でユーザーが選択した内容を前の画面に返すケースです。

データを返す画面遷移 
// ホーム画面（データを受け取る側）
class HomePage extends StatefulWidget {
  @override
  _HomePageState createState() => _HomePageState();
}

class _HomePageState extends State<HomePage> {
  String selectedColor = '色を選択してください';

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('ホーム画面')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text('選択した色: $selectedColor', style: TextStyle(fontSize: 20)),
            SizedBox(height: 20),
            ElevatedButton(
              onPressed: () async {
                // 色選択画面へ遷移し、結果を待つ
                final result = await Navigator.push(
                  context,
                  MaterialPageRoute(builder: (context) => ColorSelectPage()),
                );
                // 結果を受け取って状態を更新
                if (result != null) {
                  setState(() {
                    selectedColor = result;
                  });
                }
              },
              child: Text('色を選択'),
            ),
          ],
        ),
      ),
    );
  }
}

// 色選択画面（データを返す側）
class ColorSelectPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('色を選択')),
      body: Column(
        children: [
          ListTile(
            title: Text('赤'),
            onTap: () {
              // 選択した色を返して画面を閉じる
              Navigator.pop(context, '赤');
            },
          ),
          ListTile(
            title: Text('青'),
            onTap: () {
              Navigator.pop(context, '青');
            },
          ),
          ListTile(
            title: Text('緑'),
            onTap: () {
              Navigator.pop(context, '緑');
            },
          ),
        ],
      ),
    );
  }
}

データを返す仕組み

  • await Navigator.push()で遷移結果を待機
  • Navigator.pop(context, 値)で値を返して画面を閉じる
  • 返された値はresult変数に格納される
  • nullチェックで「キャンセル」と「選択」を区別

12.5 名前付きルート

アプリの規模が大きくなると、画面遷移を管理するために名前付きルートを使うと便利です。各画面にパス（URLのようなもの）を割り当てて、名前で遷移先を指定します。

名前付きルートの定義と使用 
class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      // ルート定義
      initialRoute: '/',
      routes: {
        '/': (context) => HomePage(),
        '/detail': (context) => DetailPage(),
        '/settings': (context) => SettingsPage(),
      },
    );
  }
}

// ホーム画面
class HomePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('ホーム画面')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            ElevatedButton(
              onPressed: () {
                // 名前で画面遷移
                Navigator.pushNamed(context, '/detail');
              },
              child: Text('詳細画面へ'),
            ),
            ElevatedButton(
              onPressed: () {
                Navigator.pushNamed(context, '/settings');
              },
              child: Text('設定画面へ'),
            ),
          ],
        ),
      ),
    );
  }
}

名前付きルートの利点

  • 一元管理: 全ての画面がMaterialAppのroutesで定義され、管理しやすい
  • コードの可読性: '/settings'のように直感的な名前で遷移
  • ディープリンク対応: 外部からURLで特定の画面を開く場合に便利

12.6 特殊な画面遷移メソッド

pushとpop以外にも、特定のシナリオで便利な画面遷移メソッドがあります。

Navigator.pushReplacement

現在の画面を新しい画面で置き換えるメソッドです。スタックから現在の画面を削除し、新しい画面を追加します。

使用例: ログイン画面からホーム画面へ遷移する際、ログイン画面に戻れないようにする 
Navigator.pushReplacement(
  context,
  MaterialPageRoute(builder: (context) => HomePage()),
);

Navigator.pushAndRemoveUntil

新しい画面に遷移し、スタック内の古い画面を全て削除するメソッドです。

使用例: ログアウト時に全ての画面を削除してログイン画面に戻る 
Navigator.pushAndRemoveUntil(
  context,
  MaterialPageRoute(builder: (context) => LoginPage()),
  (route) => false, // falseを返すと全ての画面を削除
);

  • Navigatorはスタック構造で画面遷移を管理
  • Navigator.push()で新しい画面に進み、Navigator.pop()で戻る
