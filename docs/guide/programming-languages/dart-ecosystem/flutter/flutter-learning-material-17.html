HTTP通信とAPI連携

httpパッケージを使ったAPI通信、JSONデータの取得と表示、エラーハンドリングを学びます。

この章で学ぶこと

  • httpパッケージのインストールと基本的な使い方を理解する
  • GETリクエストでAPIからデータを取得する方法を学ぶ
  • JSONデータの解析（パース）と表示方法をマスターする
  • POST/PUT/DELETEリクエストとエラーハンドリングを実装する

17.1 HTTP通信とは

HTTP（HyperText Transfer Protocol）は、インターネット上でデータをやり取りするための通信規約です。Flutterアプリでは、このHTTPを使って外部のサーバー（Web API）からデータを取得したり、データを送信したりします。

GETリクエスト

サーバーからデータを取得する（例: ユーザー情報の取得）

POSTリクエスト

サーバーに新しいデータを作成する（例: 新規登録）

PUTリクエスト

サーバーのデータを更新する（例: プロフィール編集）

DELETEリクエスト

サーバーのデータを削除する（例: 投稿削除）

HTTP通信の流れ

sequenceDiagram participant App as Flutterアプリ participant Server as Webサーバー App->>Server: HTTPリクエスト送信 Note over App,Server: GET/POST/PUT/DELETE Server->>Server: データ処理 Server->>App: HTTPレスポンス返却 Note over App,Server: JSON形式のデータ App->>App: データ解析・表示

17.2 httpパッケージのインストール

FlutterでHTTP通信を行うには、httpパッケージを使用します。まずはこのパッケージをプロジェクトに追加します。

実習: httpパッケージのインストール

手順1: pubspec.yamlファイルを開き、dependenciesセクションにhttpパッケージを追加します。

pubspec.yaml 
dependencies:
  flutter:
    sdk: flutter
  http: ^1.1.0  # この行を追加

手順2: ターミナルで以下のコマンドを実行してパッケージをインストールします。

ターミナル 
flutter pub get

インストール後の確認

パッケージが正しくインストールされたかは、pubspec.lockファイルにhttpパッケージの情報が追加されていることで確認できます。

17.3 GETリクエストの実装

まずは最も基本的なGETリクエストを実装して、外部APIからデータを取得してみましょう。ここでは公開されているテスト用API「JSONPlaceholder」を使用します。

実習: ユーザー情報を取得するアプリ

JSONPlaceholder APIからユーザー情報を取得して表示するアプリを作成します。

lib/main.dart 
import 'package:flutter/material.dart';
import 'package:http/http.dart' as http; // httpパッケージをインポート
import 'dart:convert'; // JSON解析用

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: UserListScreen(),
    );
  }
}

class UserListScreen extends StatefulWidget {
  @override
  _UserListScreenState createState() => _UserListScreenState();
}
lib/main.dart（続き） 
class _UserListScreenState extends State<UserListScreen> {
  List users = []; // ユーザー情報を格納するリスト
  bool isLoading = false; // ローディング状態

  // GETリクエストでデータを取得する関数
  Future<void> fetchUsers() async {
    setState(() => isLoading = true);

    final url = Uri.parse('https://jsonplaceholder.typicode.com/users');
    final response = await http.get(url);

    if (response.statusCode == 200) {
      setState(() {
        users = json.decode(response.body); // JSON文字列を配列に変換
        isLoading = false;
      });
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('ユーザー一覧')),
      body: isLoading
          ? Center(child: CircularProgressIndicator())
          : ListView.builder(
              itemCount: users.length,
              itemBuilder: (context, index) {
                return ListTile(
                  title: Text(users[index]['name']),
                  subtitle: Text(users[index]['email']),
                );
              },
            ),
      floatingActionButton: FloatingActionButton(
        onPressed: fetchUsers,
        child: Icon(Icons.refresh),
      ),
    );
  }
}

コード解説

  • import 'package:http/http.dart' as http - httpパッケージをインポート
  • Uri.parse() - URLを解析してUri型に変換
  • http.get(url) - GETリクエストを送信
  • response.statusCode - HTTPステータスコード（200=成功）
  • json.decode() - JSON文字列をDartのデータ型に変換

17.4 JSONデータの解析とモデルクラス

実際のアプリ開発では、JSONデータをモデルクラスに変換して扱うのが一般的です。これにより、型安全性が高まり、コードの可読性も向上します。

APIが返すJSONデータの例

JSON 
{
  "id": 1,
  "name": "Leanne Graham",
  "username": "Bret",
  "email": "Sincere@april.biz"
}

実習: モデルクラスの作成

lib/models/user.dart 
class User {
  final int id;
  final String name;
  final String username;
  final String email;

  User({
    required this.id,
    required this.name,
    required this.username,
    required this.email,
  });

  // JSONからUserオブジェクトを作成するファクトリコンストラクタ
  factory User.fromJson(Map<String, dynamic> json) {
    return User(
      id: json['id'],
      name: json['name'],
      username: json['username'],
      email: json['email'],
    );
  }
}
モデルクラスを使ったfetchUsers関数 
List<User> users = []; // User型のリストに変更

Future<void> fetchUsers() async {
  setState(() => isLoading = true);

  final url = Uri.parse('https://jsonplaceholder.typicode.com/users');
  final response = await http.get(url);

  if (response.statusCode == 200) {
    final List jsonList = json.decode(response.body);
    setState(() {
      users = jsonList.map((json) => User.fromJson(json)).toList();
      isLoading = false;
    });
  }
}

17.5 POSTリクエストの実装

POSTリクエストは、サーバーに新しいデータを送信する際に使用します。フォームの送信や新規登録などでよく使われます。

実習: 投稿データの送信

POSTリクエストの例 
Future<void> createPost(String title, String body) async {
  final url = Uri.parse('https://jsonplaceholder.typicode.com/posts');

  final response = await http.post(
    url,
    headers: {'Content-Type': 'application/json'},
    body: json.encode({
      'title': title,
      'body': body,
      'userId': 1,
    }),
  );

  if (response.statusCode == 201) {
    print('投稿作成成功！');
    print(response.body);
  }
}

POSTリクエストのポイント

  • headers - リクエストヘッダー（データ形式などを指定）
  • body - 送信するデータ（JSON形式にエンコード）
  • statusCode 201 - リソース作成成功のステータスコード

17.6 エラーハンドリング

ネットワーク通信では、接続エラーやサーバーエラーなど様々な問題が発生する可能性があります。適切なエラーハンドリングを実装しましょう。

実習: エラーハンドリングの実装

エラーハンドリング付きfetchUsers 
Future<void> fetchUsers() async {
  setState(() => isLoading = true);

  try {
    final url = Uri.parse('https://jsonplaceholder.typicode.com/users');
    final response = await http.get(url);

    if (response.statusCode == 200) {
      final List jsonList = json.decode(response.body);
      setState(() {
        users = jsonList.map((json) => User.fromJson(json)).toList();
        isLoading = false;
      });
    } else {
      throw Exception('サーバーエラー: ${response.statusCode}');
    }
  } catch (e) {
    setState(() => isLoading = false);
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(content: Text('データ取得に失敗しました: $e')),
    );
  }
}

主なエラーパターン

  • 400番台 クライアントエラー（例: 401 認証エラー、404 Not Found）
  • 500番台 サーバーエラー（例: 500 Internal Server Error）
  • Exception ネットワーク接続エラー、タイムアウトなど

トラブルシューティング

問題: SocketException が発生する

原因: インターネット接続権限が不足している、または実機でHTTP通信が許可されていない。

解決策: AndroidManifest.xmlにINTERNET権限を追加。iOS 9以降でHTTP通信する場合はInfo.plistにATS設定を追加。

問題: FormatException が発生する

原因: JSON解析に失敗している（不正なJSON形式）。

解決策: response.bodyを出力してJSON形式が正しいか確認。APIのドキュメントを確認。
