はじめてのFlutterアプリ

最初のFlutterプロジェクトを作成し、アプリを実行します。プロジェクト構造の理解と、ホットリロードの便利さを体験します。

この章で学ぶこと

  • Flutterプロジェクトの新規作成方法を理解する
  • プロジェクトのフォルダ構造と役割を把握する
  • アプリの実行方法とホットリロード機能を体験する
  • シンプルなコード変更を行い、変更が即座に反映されることを確認する

3.1 Flutterプロジェクトの作成

Visual Studio Codeまたはターミナル（コマンドプロンプト）を使って、最初のFlutterプロジェクトを作成します。

プロジェクト作成前の確認

第2章で環境構築が完了していることを確認してください。以下のコマンドで確認できます。

環境確認コマンド 
flutter doctor

すべてのチェック項目が緑色のチェックマークであることを確認してください。

方法1: コマンドライン（推奨）

ターミナルまたはコマンドプロンプトを開き、プロジェクトを作成したい場所に移動してから以下のコマンドを実行します。

プロジェクト作成コマンド 
# プロジェクト作成
flutter create my_first_app

# プロジェクトフォルダに移動
cd my_first_app

コマンドの解説

  • flutter create - 新しいFlutterプロジェクトを作成するコマンド
  • my_first_app - プロジェクト名（自由に変更可能、英数字とアンダースコアのみ使用可）
  • cd - ディレクトリ移動コマンド（Change Directoryの略）

方法2: Visual Studio Code

Visual Studio Codeを使ってGUI操作でプロジェクトを作成することもできます。

  1. VS Codeで Ctrl+Shift+P（macOSは Cmd+Shift+P）でコマンドパレットを開く
  2. 「Flutter: New Project」と入力して選択
  3. 「Application」を選択
  4. プロジェクトを作成する場所を選択
  5. プロジェクト名を入力（例: my_first_app）

3.2 プロジェクト構造の理解

作成されたプロジェクトには、多くのフォルダとファイルが含まれています。主要なものを理解しましょう。

プロジェクトのフォルダ構造 

my_first_app/
├── android/          # Androidプラットフォーム固有の設定
├── ios/              # iOSプラットフォーム固有の設定
├── lib/              # Dartコードを配置する主要フォルダ（最重要）
│   └── main.dart    # アプリのエントリーポイント（起動ファイル）
├── test/             # テストコードを配置するフォルダ
├── web/              # Webプラットフォーム固有の設定
├── pubspec.yaml     # プロジェクトの設定ファイル（依存関係、アセット管理）
├── README.md         # プロジェクトの説明ファイル
└── .gitignore        # Gitで管理しないファイルの設定

lib/フォルダ

Dartコードを配置する場所。ほとんどの開発作業はここで行います。main.dartがアプリの起動ファイルです。

pubspec.yaml

プロジェクト設定ファイル。外部パッケージの追加、アセット（画像、フォント等）の管理を行います。

android/フォルダ

Android固有の設定。アプリアイコン、パーミッション設定など、Android向けカスタマイズに使用します。

ios/フォルダ

iOS固有の設定。アプリアイコン、パーミッション設定など、iOS向けカスタマイズに使用します。

3.3 アプリの実行

作成したプロジェクトを実際に実行してみましょう。初回は少し時間がかかります。

実行前の準備

アプリを実行するには、以下のいずれかが必要です:

  • エミュレータ: Android StudioまたはXcodeのエミュレータ（仮想端末）
  • 実機: USBケーブルで接続したスマートフォン
  • Chromeブラウザ: Web版として実行

実行手順（コマンドライン）

プロジェクトフォルダ内で以下のコマンドを実行します。

アプリ実行コマンド 
# デバイスの一覧を確認
flutter devices

# アプリを実行（利用可能なデバイスが自動選択される）
flutter run
初回実行時の注意: 初めてアプリを実行する際は、各プラットフォームの依存関係をダウンロード・ビルドするため、5〜10分程度かかる場合があります。2回目以降は数十秒で起動します。

実行手順（Visual Studio Code）

  1. VS Codeでプロジェクトフォルダを開く
  2. 右下のステータスバーからデバイスを選択（例: Chrome、Android Emulator等）
  3. F5キーを押す、またはメニューから「実行」→「デバッグの開始」を選択

3.4 ホットリロードの体験

Flutterの最大の魅力の1つであるホットリロード機能を体験しましょう。コードを変更すると、アプリを再起動せずに即座に変更が反映されます。

実習: タイトルを変更してみよう

lib/main.dartファイルを開いて、アプリのタイトルを変更してみましょう。

手順

  1. VS Codeでlib/main.dartを開く
  2. 20行目付近のtitle: 'Flutter Demo'を探す
  3. 'Flutter Demo'を'はじめてのアプリ'に変更
  4. ファイルを保存（Ctrl+S / Cmd+S）
変更前（デフォルト） 
// 変更前
title: 'Flutter Demo',
変更後 
// 変更後
title: 'はじめてのアプリ',

ホットリロードの実行方法

ファイルを保存すると自動的にホットリロードが実行されますが、手動で実行することもできます。

コマンドライン

アプリ実行中のターミナルで以下を入力:

  • r - ホットリロード
  • R - ホットリスタート（完全再起動）
  • q - アプリ終了

Visual Studio Code

デバッグツールバーのボタンをクリック:

  • ホットリロードボタン
  • ホットリスタートボタン
  • 保存時に自動でホットリロード（推奨）

ホットリロード vs ホットリスタート

機能 ホットリロード（r） ホットリスタート（R）
速度 非常に高速（1秒未満） やや遅い（数秒）
状態の保持 保持される リセットされる
使用場面 UIの微調整、色の変更等 大きな構造変更、初期化処理の変更等

3.5 main.dartの構造を理解する

lib/main.dartファイルはFlutterアプリのエントリーポイント（起動ファイル）です。詳細な理解は後の章で行いますが、ここでは大まかな構造を把握しましょう。

main.dartの簡略版 
import 'package:flutter/material.dart';

// アプリの起動ポイント
void main() {
  runApp(const MyApp()); // MyAppウィジェットを実行
}

// アプリ全体の設定を行うウィジェット
class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'はじめてのアプリ', // アプリのタイトル
      home: MyHomePage(),       // ホーム画面のウィジェット
    );
  }
}

1 import文

import 'package:flutter/material.dart';
Material Design用のウィジェットをインポート（読み込み）します。

2 main関数

void main() {'{'}...{'}'}
アプリの起動ポイント。すべてのDartアプリはmain関数から始まります。

3 runApp()

runApp(const MyApp());
指定したウィジェット（MyApp）をアプリのルート（最上位）として実行します。

4 MaterialApp

MaterialApp(...)
Material Designをベースとしたアプリの設定を行うウィジェットです。

解答を見る

正解: B

解答を見る
正解: C

よくあるトラブルと対処法

トラブルシューティング

デバイスが見つからない

flutter devicesで何も表示されない場合:

  • エミュレータが起動しているか確認
  • 実機の場合、USBデバッグが有効になっているか確認
  • Chromeブラウザがインストールされているか確認

ビルドに失敗する

ビルドエラーが発生した場合:

クリーンビルドコマンド 
flutter clean       # ビルドキャッシュをクリア
flutter pub get     # 依存関係を再取得
flutter run         # 再度実行

ホットリロードが効かない

ホットリロードで変更が反映されない場合:

  • ファイルが正しく保存されているか確認
  • ホットリスタート（R）を試す
  • main関数や初期化処理の変更にはホットリロードが効かない場合があります

第3章のまとめ

  • flutter createコマンドで新しいプロジェクトを作成できる
  • lib/フォルダにDartコードを配置し、lib/main.dartがエントリーポイント
  • pubspec.yamlでパッケージやアセットを管理する
  • flutter runでアプリを実行できる
  • ホットリロード機能により、コード変更が即座に反映される
  • main()関数からrunApp()でアプリが起動する
