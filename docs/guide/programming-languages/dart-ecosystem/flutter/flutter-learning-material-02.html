開発環境の準備

Flutter SDKのインストール、Visual Studio Codeの設定、エミュレータの準備を行います。Windows、macOS、Linuxそれぞれの環境構築手順を解説します。

この章で学ぶこと

  • 自分のOS（Windows/macOS/Linux）に合わせたFlutter SDKのインストール方法
  • Visual Studio Codeのインストールと拡張機能の設定
  • エミュレータ/シミュレータのセットアップ方法
  • flutter doctorコマンドによる環境確認の手順

2.1 開発に必要なもの

Flutterでアプリ開発を始めるには、以下の3つが必要です。すべて無料でインストールできます。

Flutter SDK

Flutter本体（開発キット）

Visual Studio Code

コードエディタ（推奨）

エミュレータ

アプリを動かす仮想端末

システム要件

  • OS: Windows 10/11、macOS 10.15以降、またはLinux
  • ディスク: 最低10GB以上の空き容量（推奨20GB以上）
  • RAM: 8GB以上推奨（4GBでも可能だが動作が遅い）
  • インターネット: 初回セットアップ時に必要

2.2 Windows環境の構築

1 Flutter SDKのダウンロード

  1. Flutter公式サイトにアクセス
  2. 「Get the Flutter SDK」セクションから最新版のZIPファイルをダウンロード
  3. ダウンロードしたZIPファイルを展開（例: C:\src\flutter）
注意: パスにスペースや日本語が含まれないフォルダに展開してください。例: C:\flutter や C:\dev\flutter

2 環境変数Pathの設定

  1. 「Windowsキー」を押して「環境変数」と検索
  2. 「システム環境変数の編集」を開く
  3. 「環境変数」ボタンをクリック
  4. 「ユーザー環境変数」の「Path」を選択して「編集」
  5. 「新規」をクリックし、C:\src\flutter\bin を追加
  6. 「OK」で閉じる
確認方法: コマンドプロンプトを開いて flutter --version と入力してバージョンが表示されればOK

3 Android Studio（エミュレータ用）のインストール

  1. Android Studio公式サイトからダウンロード
  2. インストーラーを実行し、デフォルト設定でインストール
  3. 初回起動時に「Android SDK」のインストールが始まるので完了まで待つ
  4. 「AVD Manager」から仮想端末（エミュレータ）を作成
推奨設定: エミュレータは「Pixel 5」または「Pixel 6」、OSバージョンは最新の安定版を選択

2.3 macOS環境の構築

1 Xcodeのインストール（iOS開発用）

  1. App Storeを開く
  2. 「Xcode」を検索してインストール（10GB以上、時間がかかります）
  3. インストール完了後、Xcodeを起動してライセンスに同意
  4. ターミナルで sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer を実行

2 Flutter SDKのインストール

以下のコマンドをターミナルで実行します。

ターミナル 
# Homebrewがない場合は先にインストール
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Flutterのインストール
brew install --cask flutter

# バージョン確認
flutter --version

3 iOS Simulatorの起動

ターミナル 
# iOS Simulatorを起動
open -a Simulator

起動後、メニューから「File → Open Simulator → iPhone 14」などを選択して端末を起動します。

2.4 Visual Studio Codeの設定

Visual Studio Code（VS Code）は軽量で高機能なコードエディタです。Flutter開発に最適です。

1 VS Codeのインストール

  1. VS Code公式サイトからダウンロード
  2. インストーラーを実行してデフォルト設定でインストール
  3. VS Codeを起動

2 Flutter拡張機能のインストール

  1. VS Codeの左側メニューから「拡張機能」アイコン（四角が4つ）をクリック
  2. 検索ボックスに「Flutter」と入力
  3. 「Flutter」（Dart Code提供）をインストール
  4. 同時に「Dart」拡張機能も自動的にインストールされます
その他の推奨拡張機能:
  • Awesome Flutter Snippets（コード補完）
  • Bracket Pair Colorizer（括弧の色分け）
  • Error Lens（エラー表示の強化）

2.5 環境確認（flutter doctor）

すべてのセットアップが完了したら、flutter doctorコマンドで環境を確認します。

flutter doctorの実行

ターミナル / コマンドプロンプト 
flutter doctor

出力例（すべて正常な場合）

Doctor summary (to see all details, run flutter doctor -v):
[✓] Flutter (Channel stable, 3.16.0, on macOS 14.0)
[✓] Android toolchain - develop for Android devices (Android SDK version 34.0.0)
[✓] Xcode - develop for iOS and macOS (Xcode 15.0)
[✓] Chrome - develop for the web
[✓] Android Studio (version 2023.1)
[✓] VS Code (version 1.85)
[✓] Connected device (2 available)
[✓] Network resources

• No issues found!

エラーや警告が出た場合

  • [✗] マークの項目を確認し、指示に従って修正してください
  • 多くの場合、コマンドを実行してライセンスに同意する必要があります
  • 例: flutter doctor --android-licenses を実行してすべて「y」で同意

2.6 よくあるトラブルと解決方法

トラブル1: flutterコマンドが認識されない

原因: 環境変数Pathが正しく設定されていない

解決方法:

  • 環境変数Pathを再確認し、flutter\binのパスが正しいか確認
  • ターミナル/コマンドプロンプトを一度閉じて再起動
  • PCを再起動してみる

トラブル2: エミュレータが起動しない

原因: 仮想化が有効になっていない、またはメモリ不足

解決方法:

  • BIOSで仮想化（VT-x/AMD-V）を有効にする
  • Windowsの場合、Hyper-Vを有効にする
  • メモリを増やす、または他のアプリを閉じる

トラブル3: Android licensesのエラー

原因: Androidライセンスに同意していない

解決方法:

flutter doctor --android-licenses

すべての質問に「y」で答えてライセンスに同意します。

重要ポイント

  • flutter doctorですべての項目が[✓]になるまで確認
  • 環境構築は一度だけ、最初に丁寧に行うことが大切
  • エラーが出ても焦らず、エラーメッセージをよく読んで対処する
  • わからない場合は公式ドキュメントやコミュニティに質問する

第2章のまとめ

  • Flutter開発にはFlutter SDK、エディタ、エミュレータの3つが必要
  • WindowsではAndroid Studio、macOSではXcodeも必要
  • Visual Studio CodeにFlutter拡張機能をインストールすると便利
  • flutter doctorで環境が正しくセットアップされているか確認
  • エラーが出た場合は、メッセージをよく読んで一つずつ解決していく
