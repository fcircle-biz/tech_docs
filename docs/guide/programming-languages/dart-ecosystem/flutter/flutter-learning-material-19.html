パッケージの活用

pub.devからのパッケージ追加方法、人気パッケージの紹介と使い方を学びます。

この章で学ぶこと

  • pub.devからパッケージを検索・追加する方法を理解する
  • pubspec.yamlファイルの役割と編集方法を学ぶ
  • 実用的な人気パッケージの使い方を実践的に学ぶ
  • パッケージのバージョン管理とアップデート方法を知る

19.1 パッケージとは

パッケージ（Package）とは、他の開発者が作成した再利用可能なコードの集まりです。Flutterでは、世界中の開発者が作成したパッケージをpub.devというプラットフォームで公開・共有しています。

開発時間の短縮

既存のパッケージを使うことで、ゼロから実装する必要がなくなります

品質の保証

多くの開発者に使われ、テストされた信頼性の高いコードを利用できます

コミュニティのサポート

問題が発生しても、コミュニティのサポートを受けられます

機能の拡張

標準機能にない高度な機能を簡単に追加できます

pub.dev とは

pub.devは、Dart/Flutterパッケージの公式リポジトリです。現在、4万個以上のパッケージが公開されており、日々新しいパッケージが追加されています。

  • パッケージの検索・閲覧ができます
  • 各パッケージのドキュメント、使用例、人気度を確認できます
  • Dart/Flutter公式チームによる品質スコア（Pub Pointsなど）が表示されます

1 pub.devでパッケージを検索

https://pub.devにアクセスし、検索バーでパッケージを検索します。

例: 「http」と検索すると、HTTP通信用パッケージが見つかります

2 pubspec.yamlに追加

プロジェクトルートのpubspec.yamlファイルを開き、dependenciesセクションにパッケージを追加します。 

name: my_app
description: My Flutter app

dependencies:
  flutter:
    sdk: flutter

  # パッケージをここに追加
  http: ^1.1.0  # HTTP通信用パッケージ
  intl: ^0.18.1  # 日付・数値フォーマット用

バージョン指定の記号

  • ^1.1.0 → 1.1.0以上、2.0.0未満（推奨）
  • 1.1.0 → 正確に1.1.0のみ
  • >=1.0.0 → 1.0.0以上

3 flutter pub getを実行

ターミナルで以下のコマンドを実行し、パッケージをダウンロードします。 

flutter pub get
補足: Visual Studio Codeでは、pubspec.yamlを保存すると自動的にflutter pub getが実行されます

4 importして使用

Dartファイルの先頭でimport文を使ってパッケージをインポートし、使用します。 

import 'package:http/http.dart' as http;

// httpパッケージを使用
final response = await http.get(
  Uri.parse('https://api.example.com/data')
);

19.3 実用的な人気パッケージ

Flutterアプリ開発でよく使われる、人気の高い実用的なパッケージを紹介します。

provider

状態管理

アプリ全体で状態を共有・管理するための最も人気のある状態管理パッケージです。 

# pubspec.yaml
dependencies:
  provider: ^6.1.1
使用例を見る 
import 'package:provider/provider.dart';

// カウンターモデル
class Counter with ChangeNotifier {
  int _count = 0;
  int get count => _count;

  void increment() {
    _count++;
    notifyListeners(); // リスナーに通知
  }
}

// アプリで使用
ChangeNotifierProvider(
  create: (context) => Counter(),
  child: MyApp(),
)

dio

HTTP通信

httpパッケージより高機能なHTTPクライアント。インターセプター、タイムアウト設定、ファイルアップロードなど充実した機能を持ちます。 

# pubspec.yaml
dependencies:
  dio: ^5.4.0
使用例を見る 
import 'package:dio/dio.dart';

// Dioインスタンス作成
final dio = Dio();

// GETリクエスト
try {
  final response = await dio.get(
    'https://api.example.com/users'
  );
  print(response.data);
} catch (e) {
  print('エラー: $e');
}

cached_network_image

画像キャッシュ

ネットワーク画像を自動的にキャッシュし、ローディング中のプレースホルダーやエラー表示も簡単に設定できます。 

# pubspec.yaml
dependencies:
  cached_network_image: ^3.3.0
使用例を見る 
import 'package:cached_network_image/cached_network_image.dart';

CachedNetworkImage(
  imageUrl: 'https://example.com/image.jpg',
  placeholder: (context, url) => CircularProgressIndicator(),
  errorWidget: (context, url, error) => Icon(Icons.error),
)

flutter_bloc

状態管理

BLoC（Business Logic Component）パターンを実装するためのパッケージ。ビジネスロジックとUIを明確に分離できます。 

# pubspec.yaml
dependencies:
  flutter_bloc: ^8.1.3

go_router

ルーティング

宣言的なルーティングを実現するパッケージ。ディープリンク、リダイレクト、ネストされたルートに対応しています。 

# pubspec.yaml
dependencies:
  go_router: ^13.0.0

google_fonts

フォント

Google Fontsの1000種類以上のフォントを簡単にFlutterアプリで使えるようにするパッケージです。 

# pubspec.yaml
dependencies:
  google_fonts: ^6.1.0
使用例を見る 
import 'package:google_fonts/google_fonts.dart';

Text(
  'こんにちは',
  style: GoogleFonts.notoSansJP(
    fontSize: 24,
    fontWeight: FontWeight.bold,
  ),
)

19.4 実習：パッケージを使った名言表示アプリ

実習内容

httpとgoogle_fontsパッケージを使って、APIから名言を取得して表示するアプリを作成します。

無料の名言API（Quotable API）を使って、ボタンを押すたびにランダムな名言を表示します。

手順1: パッケージを追加

# pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  http: ^1.1.0
  google_fonts: ^6.1.0

保存後、flutter pub getを実行してください。

手順2: コードを実装

import 'package:flutter/material.dart';
import 'package:http/http.dart' as http;
import 'dart:convert';
import 'package:google_fonts/google_fonts.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '名言アプリ',
      home: QuotePage(),
    );
  }
}

class QuotePage extends StatefulWidget {
  @override
  _QuotePageState createState() => _QuotePageState();
}

class _QuotePageState extends State<QuotePage> {
  String quote = '名言を取得してください';
  String author = '';
  bool isLoading = false;

  // API呼び出し関数
  Future<void> fetchQuote() async {
    setState(() => isLoading = true);

    try {
      final response = await http.get(
        Uri.parse('https://api.quotable.io/random')
      );

      if (response.statusCode == 200) {
        final data = jsonDecode(response.body);
        setState(() {
          quote = data['content'];
          author = data['author'];
          isLoading = false;
        });
      }
    } catch (e) {
      setState(() {
        quote = 'エラーが発生しました';
        author = '';
        isLoading = false;
      });
    }
  }
}

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('名言アプリ'),
        backgroundColor: Colors.purple,
      ),
      body: Padding(
        padding: EdgeInsets.all(16),
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // 名言表示
            Card(
              child: Padding(
                padding: EdgeInsets.all(20),
                child: Column(
                  children: [
                    Text(
                      quote,
                      style: GoogleFonts.lato(
                        fontSize: 20,
                        fontWeight: FontWeight.w500,
                      ),
                      textAlign: TextAlign.center,
                    ),
                    if (author.isNotEmpty) ...[
                      SizedBox(height: 16),
                      Text(
                        '- $author -',
                        style: GoogleFonts.lato(
                          fontSize: 16,
                          fontStyle: FontStyle.italic,
                        ),
                      ),
                    ],
                  ],
                ),
              ),
            ),
            SizedBox(height: 24),
            ElevatedButton(
              onPressed: isLoading ? null : fetchQuote,
              child: isLoading
                  ? CircularProgressIndicator(color: Colors.white)
                  : Text('新しい名言を取得'),
            ),
          ],
        ),
      ),
    );
  }
}

動作確認

  • ✓ ボタンを押すと英語の名言が表示される
  • ✓ 名言の著者名が表示される
  • ✓ Google Fontsのフォントが適用されている
  • ✓ ローディング中はボタンが無効になる

19.5 パッケージ管理のベストプラクティス

パッケージ選びのポイント

  • Pub Points: 100点に近いほど品質が高い
  • Popularity: 多くの開発者に使われているか
  • Likes: コミュニティの評価が高いか
  • 最終更新日: メンテナンスされているか（3ヶ月以内が理想）
  • Null Safety: Null Safetyに対応しているか

バージョン管理

  • 通常は^を使ったバージョン指定を推奨
  • 定期的にflutter pub upgradeでアップデート
  • メジャーバージョンアップ時は変更履歴（CHANGELOG）を確認

注意すべき点

  • 必要最小限のパッケージのみ追加（アプリサイズ増加を防ぐ）
  • 似た機能のパッケージを複数追加しない
  • メンテナンスされていないパッケージは避ける
  • パッケージのライセンスを確認（商用利用可能か）

よくあるトラブルと解決方法

パッケージが見つからないエラー

Error: Could not find a file named "pubspec.yaml"

原因: プロジェクトルートでコマンドを実行していない

解決: cdコマンドでプロジェクトルートに移動してから実行

バージョン競合エラー

Because package_a depends on package_b ^1.0.0...

原因: パッケージ間で依存バージョンが競合している

解決: flutter pub upgradeまたはflutter pub get --verboseで詳細確認
