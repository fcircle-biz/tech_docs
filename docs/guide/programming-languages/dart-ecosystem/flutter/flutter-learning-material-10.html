状態管理の基礎

StatefulWidgetを使った状態管理の基本を学びます。setStateによる画面更新の仕組みを理解します。

この章で学ぶこと

  • StatefulWidgetとStatelessWidgetの違いを理解する
  • setState()を使った状態更新の仕組みを学ぶ
  • 状態を持つウィジェットを作成して画面を動的に更新する
  • カウンターアプリやトグルスイッチなど実用的な例を実装する

10.1 状態とは何か

状態（State）とは、アプリケーションが持つデータや情報のことです。状態が変わることで、画面表示も変化します。

カウンターアプリ

数値（カウント）が状態。ボタンを押すと数値が増減し画面更新

スイッチ

ON/OFF（true/false）が状態。トグルすると表示が切り替わる

テキスト入力

入力されたテキストが状態。文字を入力すると画面に反映

ToDoリスト

タスクのリストが状態。追加・削除するとリスト表示が更新

10.2 StatefulWidgetとStatelessWidgetの違い

StatelessWidget

状態を持たないウィジェット。一度表示されたら、外部から新しいデータが渡されるまで変化しません。

使用例:

  • 固定テキストの表示
  • アイコン表示
  • 静的な画像
  • 変化しないレイアウト

StatefulWidget

状態を持つウィジェット。内部にデータを保持し、そのデータが変わると自動的に画面が再描画されます。

使用例:

  • カウンター
  • フォーム入力
  • チェックボックス/スイッチ
  • アニメーション

10.3 setState()の仕組み

StatefulWidgetで状態を更新するには、setState()メソッドを使います。setState()内で状態を変更すると、Flutterが自動的に画面を再描画します。

重要ポイント

  • setState()内で状態変数を変更する必要がある
  • setState()を呼ぶと、build()メソッドが自動的に再実行される
  • setState()を呼ばないと、変数は変わっても画面は更新されない

setState()の動作フロー

sequenceDiagram participant User as ユーザー participant Button as ボタン participant setState as setState() participant Build as build() participant Screen as 画面 User->>Button: クリック Button->>setState: 状態変更を通知 setState->>setState: count++ setState->>Build: 再描画を要求 Build->>Screen: 新しい値で描画 Screen->>User: 更新された画面を表示

基本的なsetState()の使い方

基本構文 
// 状態変数を定義
int _counter = 0;

// ボタン押下時の処理
void _incrementCounter() {
  setState(() {
    // この中で状態を更新
    _counter++;
  });
}

// build()メソッド内で状態を表示
Text('カウント: $_counter')

よくある間違い

❌ 間違い: setState()を使わない

_counter++;  // 変数は変わるが画面更新されない

✅ 正しい: setState()内で変更

setState(() {
  _counter++;  // 変数が変わり画面も更新される
});

10.4 カウンターアプリの実装

実習: カウンターアプリを作ろう

ボタンを押すとカウントが増える基本的なカウンターアプリを作成します。StatefulWidgetの基礎を実践的に学びましょう。

完成イメージ

カウント:

0

カウンターアプリの完全なコード

lib/main.dart 
import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'カウンターアプリ',
      home: CounterPage(),
    );
  }
}
CounterPage（StatefulWidget） 
class CounterPage extends StatefulWidget {
  @override
  _CounterPageState createState() => _CounterPageState();
}

class _CounterPageState extends State<CounterPage> {
  // 状態変数: カウント
  int _counter = 0;

  // カウントアップのメソッド
  void _incrementCounter() {
    setState(() {
      _counter++;  // カウントを1増やす
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('カウンターアプリ'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text(
              'カウント:',
              style: TextStyle(fontSize: 24),
            ),
            Text(
              '$_counter',  // 状態変数を表示
              style: TextStyle(
                fontSize: 72,
                fontWeight: FontWeight.bold,
                color: Colors.blue,
              ),
            ),
            SizedBox(height: 32),
            ElevatedButton(
              onPressed: _incrementCounter,  // ボタン押下時
              child: Text('カウントアップ'),
            ),
          ],
        ),
      ),
    );
  }
}

コード解説

1. StatefulWidgetの定義

CounterPageはStatefulWidgetを継承し、createState()で状態クラスを返します。

2. 状態クラス

_CounterPageStateが実際の状態（_counter）を保持します。

3. setState()で更新

_incrementCounter()内でsetState()を呼び、カウントを増やして画面を再描画します。

4. 状態の表示

Text('$_counter')で状態変数を表示。setState()後に自動的に更新されます。

10.5 複数の状態を管理する

1つのStatefulWidgetで複数の状態変数を管理できます。トグルスイッチとカウンターを組み合わせた例を見てみましょう。

実習: スイッチ付きカウンター

スイッチがONの時だけカウントアップできるアプリを作成します。

複数状態の管理例 
class _CounterPageState extends State<CounterPage> {
  // 状態1: カウント
  int _counter = 0;
  // 状態2: スイッチのON/OFF
  bool _isEnabled = true;

  void _incrementCounter() {
    // スイッチがONの時のみカウントアップ
    if (_isEnabled) {
      setState(() {
        _counter++;
      });
    }
  }

  void _toggleSwitch(bool value) {
    setState(() {
      _isEnabled = value;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('スイッチ付きカウンター')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text('カウント: $_counter', style: TextStyle(fontSize: 36)),
            SizedBox(height: 32),
            // スイッチ
            Row(
              mainAxisAlignment: MainAxisAlignment.center,
              children: [
                Text('カウント有効'),
                Switch(
                  value: _isEnabled,
                  onChanged: _toggleSwitch,
                ),
              ],
            ),
            SizedBox(height: 16),
            ElevatedButton(
              onPressed: _isEnabled ? _incrementCounter : null,
              child: Text('カウントアップ'),
            ),
          ],
        ),
      ),
    );
  }
}

ポイント

  • 複数の状態変数を1つのStateクラスで管理できる
  • setState()内で複数の変数を同時に変更してもOK
  • 状態間で条件判定（if文）を使って動作を制御できる

10.6 状態管理のベストプラクティス

1 状態変数は必要最小限に

計算で求められる値は状態として持たず、build()内で計算します。例: _total = _price * _quantityは状態変数にせず、build()内で_price * _quantityと計算。

2 状態変数には「_」プレフィックス

状態変数名の先頭に「_（アンダースコア）」を付けると、プライベート変数（そのクラス内でのみアクセス可能）になります。例: _counter、_isEnabled

3 setState()の範囲を最小限に

setState()内では状態変更のみを行い、重い処理（API通信など）は外で実行してから結果をsetState()で反映します。

4 適切なWidget分割

画面全体を1つのStatefulWidgetにせず、状態を持つ部分だけをStatefulWidgetにすると効率的です（次章以降で詳しく学習）。

動作確認

  • ✅ カウンターアプリを実行し、ボタンを押すと数字が増えることを確認
  • ✅ スイッチ付きカウンターでON/OFFによる動作変化を確認
  • ✅ ホットリロードで画面が更新されることを体験
  • ✅ setState()を削除してボタンを押すと画面が更新されないことを確認（その後元に戻す）

解答を見る

Q2

setState()の役割として正しいのはどれですか？

解答を見る
正解: A

Q3

次のコードで画面が更新されない理由は何ですか？
_counter++;

解答を見る
正解: A
setState()を使わずに状態変数を変更しても、Flutterは「画面を再描画する必要がある」ことを知ることができません。変数の値は変わっていますが、画面の表示は古いままになります。

トラブルシューティング

よくあるエラーと解決方法

❌ エラー: setState() called after dispose()

原因: ウィジェットが破棄された後にsetState()を呼んでいる

解決: 非同期処理の前にif (mounted)でチェック

❌ 問題: ボタンを押しても画面が更新されない

原因: setState()を使わずに変数を変更している

解決: setState(() { _counter++; });のようにsetState()内で変更

❌ エラー: type 'int' is not a subtype of type 'String'

原因: 型が一致していない（例: int型をString型に代入）

解決: '$_counter'のように文字列補間を使うか、_counter.toString()で変換

第10章のまとめ

  • 状態（State）とは、アプリが保持するデータで、変化すると画面も更新される
  • StatelessWidgetは状態を持たず、StatefulWidgetは状態を持つ
  • setState()を使って状態を更新すると、自動的にbuild()が再実行され画面が更新される
  • setState()を使わずに変数を変更しても画面は更新されない
  • 複数の状態変数を1つのStatefulWidgetで管理できる
  • 状態変数は必要最小限にし、計算可能な値は状態にしない
