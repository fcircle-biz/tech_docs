第8章

メソッドの基本 - 処理を部品化して再利用しよう

よく使う処理を「メソッド」としてまとめ、何度でも簡単に呼び出せるようにする方法を学びます。プログラムを部品化して、効率的に開発する技術を習得します。

この章で学ぶこと

  • メソッドとは何か、「名前を付けた処理のまとまり」という概念を理解する
  • メソッドの基本的な書き方（def...end）をマスターする
  • 引数と戻り値の使い方を学び、より柔軟なメソッドを作成する
  • デフォルト引数を使って、省略可能な引数を持つメソッドを作る
  • メソッドを使うことで得られる3つのメリット（再利用性・保守性・可読性）を体感する

8.1 メソッドとは何か？

これまでに学んだ変数、条件分岐、繰り返しを使えば、様々なプログラムを作ることができます。しかし、同じような処理を何度も書くのは面倒ですし、プログラムも長くなってしまいます。

メソッドとは、よく使う処理に「名前」を付けて、いつでも簡単に呼び出せるようにしたものです。まるでレシピブックに料理の手順を書いておいて、「今日はカレーを作ろう」と言えばその手順が実行されるようなイメージです。

flowchart TD A([処理を書く]) --> B{何度も使う？} B -->|はい| C[メソッドとして定義] B -->|いいえ| D[そのまま書く] C --> E[名前を付けて保存] E --> F[必要なときに呼び出す] F --> G([効率的なプログラム]) D --> H([プログラム]) style A fill:#fee2e2 style C fill:#dbeafe style G fill:#d1fae5

レシピブックの例え

料理のレシピブックには「カレー」「ハンバーグ」「オムライス」など、料理ごとに手順が書かれています。メソッドも同じで、「挨拶する」「合計を計算する」「データを保存する」など、処理ごとに名前を付けて保存しておきます。

8.2 メソッドの基本的な書き方

Rubyでは、def（define：定義するの略）というキーワードを使ってメソッドを作ります。

基本的な構文

メソッドの基本構文 
# メソッドの定義
def メソッド名
  # ここに処理を書く
end

# メソッドの呼び出し
メソッド名

実例：挨拶するメソッド

「こんにちは！」と表示するメソッドを作ってみましょう。

greeting.rb 
# 挨拶するメソッドを定義
def greet
  puts "こんにちは！"
  puts "Rubyの世界へようこそ！"
end

# メソッドを呼び出す
greet
greet  # 何度でも呼び出せる

実行結果

こんにちは！
Rubyの世界へようこそ！
こんにちは！
Rubyの世界へようこそ！

greetというメソッド名を呼び出すだけで、何度でも同じ処理が実行できます。処理を何度も書く必要がなくなりました。

メソッド名の付け方

メソッド名は小文字で始め、単語の区切りはアンダースコア（_）を使います（例：calculate_total、show_message）。変数名と同じルールです。

8.3 引数を持つメソッド

先ほどのメソッドは、呼び出すたびに同じメッセージを表示するだけでした。しかし、引数（ひきすう）を使えば、メソッドに「材料」を渡して、もっと柔軟な処理ができるようになります。

これは、レシピに「材料」を渡すようなイメージです。「カレーを作る」というレシピに、「鶏肉」を渡せばチキンカレー、「牛肉」を渡せばビーフカレーができるように、引数によって動作が変わります。

greet_user全角（佐藤全角）"] --> B["引数
name = 佐藤"] B --> C["メソッド内の処理
こんにちは、佐藤さん！"] C --> D([表示される]) style A fill:#fee2e2 style B fill:#dbeafe style D fill:#d1fae5

実例：名前を受け取って挨拶

greet_with_name.rb 
# 名前を受け取って挨拶するメソッド
def greet_user(name)
  puts "こんにちは、#{name}さん！"
end

# メソッドを呼び出す（引数を渡す）
greet_user("佐藤")
greet_user("田中")
greet_user("鈴木")

実行結果

こんにちは、佐藤さん！
こんにちは、田中さん！
こんにちは、鈴木さん！

複数の引数を受け取る

メソッドは複数の引数を受け取ることもできます。引数はカンマ（,）で区切ります。

calculate.rb 
# 2つの数値を足すメソッド
def add(a, b)
  sum = a + b
  puts "#{a} + #{b} = #{sum}"
end

# メソッドを呼び出す
add(5, 3)
add(10, 20)
add(100, 200)

実行結果

5 + 3 = 8
10 + 20 = 30
100 + 200 = 300

8.4 戻り値 - メソッドから結果を受け取る

これまでのメソッドは、結果を画面に表示するだけでした。しかし、戻り値（もどりち）を使えば、メソッドの処理結果を受け取って、変数に保存したり、他の処理で使ったりできます。

これは、自動販売機のようなイメージです。お金を入れて（引数）、ボタンを押すと（メソッド呼び出し）、飲み物が出てきます（戻り値）。その飲み物は受け取って、飲んだり、誰かに渡したりできます。

flowchart LR A["呼び出し側"] -->|引数を渡す| B["メソッド"] B -->|戻り値を返す| A B --> C["処理実行"] C --> D["結果を計算"] D --> B style A fill:#fee2e2 style B fill:#dbeafe style D fill:#fef3c7

returnキーワード

returnキーワードを使うと、明示的に値を返すことができます。

return_example.rb 
# 2つの数値を足した結果を返すメソッド
def add(a, b)
  result = a + b
  return result  # 結果を返す
end

# 戻り値を変数に保存
answer = add(10, 5)
puts "答えは #{answer} です"

# 戻り値を直接使う
puts "別の計算: #{add(20, 30)}"

実行結果

答えは 15 です
別の計算: 50

Rubyの便利な機能：暗黙の戻り値

Rubyでは、returnを書かなくても、メソッドの最後に評価された式の値が自動的に戻り値になります。これはRubyの便利な特徴です。

implicit_return.rb 
# returnを省略した書き方（Rubyらしい）
def multiply(a, b)
  a * b  # 最後の式の値が自動的に戻り値になる
end

result = multiply(7, 8)
puts "7 × 8 = #{result}"

Rubyらしい書き方

Rubyプログラマーは、returnを省略することが多いです。シンプルで読みやすいコードが書けるからです。ただし、途中で処理を終了したい場合はreturnを明示的に書きます。

8.5 デフォルト引数 - 省略可能な引数

メソッドの引数にデフォルト値を設定すると、その引数を省略して呼び出すことができます。これは、「特に指定がなければ、この値を使う」という意味です。

これは、コーヒーショップの注文に似ています。「コーヒーをください」と言えばホットコーヒーが出てきて（デフォルト）、「アイスコーヒーをください」と言えばアイスが出てくるようなイメージです。

default_argument.rb 
# デフォルト引数を持つメソッド
def greet(name, greeting = "こんにちは")
  puts "#{greeting}、#{name}さん！"
end

# 引数を2つ渡す場合
greet("佐藤", "おはようございます")

# 2つ目の引数を省略（デフォルト値が使われる）
greet("田中")

# 1つ目の引数だけ指定
greet("鈴木")

実行結果

おはようございます、佐藤さん！
こんにちは、田中さん！
こんにちは、鈴木さん！

実用例：消費税の計算

tax_calculator.rb 
# 消費税込みの金額を計算するメソッド
def calculate_with_tax(price, tax_rate = 0.1)
  total = price * (1 + tax_rate)
  total.round  # 小数点以下を四捨五入
end

# 通常の消費税10%で計算
puts "1000円の税込価格: #{calculate_with_tax(1000)}円"

# 軽減税率8%で計算
puts "1000円の税込価格（軽減税率）: #{calculate_with_tax(1000, 0.08)}円"

実行結果

1000円の税込価格: 1100円
1000円の税込価格（軽減税率）: 1080円

8.6 メソッドを使う3つのメリット

再利用性

一度定義したメソッドは、プログラムの中で何度でも呼び出せます。同じコードを繰り返し書く必要がなくなります。

保守性

処理を修正する必要が出たとき、メソッドの定義部分だけを直せば、その処理を使っている全ての箇所に反映されます。

可読性

メソッド名を見れば何をしているかが分かるため、プログラム全体が読みやすくなります。長い処理も名前を付けることで整理できます。

比較：メソッドを使わない場合と使う場合

以下の2つのプログラムを比較してみましょう。

without_method.rb（メソッドを使わない） 
# メソッドを使わない例 - 繰り返しが多く読みにくい
price1 = 1000
total1 = price1 * 1.1
puts "商品A: #{total1.round}円"

price2 = 2000
total2 = price2 * 1.1
puts "商品B: #{total2.round}円"

price3 = 3000
total3 = price3 * 1.1
puts "商品C: #{total3.round}円"
with_method.rb（メソッドを使う） 
# メソッドを使う例 - シンプルで読みやすい
def with_tax(price)
  (price * 1.1).round
end

puts "商品A: #{with_tax(1000)}円"
puts "商品B: #{with_tax(2000)}円"
puts "商品C: #{with_tax(3000)}円"

メソッドを使うことで、コードが短く、読みやすくなりました。さらに、消費税率が変わったときも、メソッドの定義を1箇所直すだけで全体に反映されます。

実習：メソッドを使ってみよう

課題1：割引価格を計算するメソッド

元の価格と割引率を受け取って、割引後の価格を返すメソッドdiscount_priceを作成してください。デフォルトの割引率は20%とします。

呼び出し例：

puts discount_price(5000) # 4000
puts discount_price(5000, 0.3) # 3500

課題2：合格判定メソッド

点数を受け取って、60点以上なら「合格」、未満なら「不合格」と表示するメソッドjudgeを作成してください。

呼び出し例：

judge(75) # 合格
judge(45) # 不合格

課題3：配列の平均値を計算するメソッド

数値の配列を受け取って、平均値を返すメソッドaverageを作成してください。

呼び出し例：

scores = [80, 90, 70]
puts average(scores) # 80.0

理解度確認クイズ

  1. Q1. メソッドとは何ですか？レシピブックに例えて説明してください。

    回答を表示

    メソッドとは、よく使う処理に「名前」を付けて、いつでも簡単に呼び出せるようにしたものです。レシピブックに料理の手順を書いておいて、「今日はカレーを作ろう」と言えばその手順が実行されるように、処理をまとめて名前を付けることで、何度でも再利用できます。

  2. Q2. 引数と戻り値の違いを説明してください。

    回答を表示

    引数は、メソッドに渡す「材料」のようなもので、メソッドの動作を柔軟に変えることができます。戻り値は、メソッドが処理した結果を呼び出し元に返す値です。自動販売機で例えると、お金が引数、出てくる飲み物が戻り値に相当します。

  3. Q3. 次のメソッド定義で、引数を省略して呼び出すと何が表示されますか？

    def greet(name = "ゲスト")
    puts "ようこそ、#{name}さん"
    end

    greet()
    回答を表示

    「ようこそ、ゲストさん」と表示されます。引数を省略すると、デフォルト値の"ゲスト"が使われます。

  4. Q4. メソッドを使う3つのメリットを挙げてください。

    回答を表示

    ①再利用性：一度定義すれば何度でも使える。②保守性：修正が必要なとき、メソッドの定義部分だけ直せば全体に反映される。③可読性：メソッド名を見れば何をしているか分かり、プログラムが読みやすくなる。

よくあるエラーと対処法

❌ エラー：wrong number of arguments

wrong number of arguments (given 1, expected 2)

原因：引数の数が合っていません。

対処法：メソッド定義で指定した引数の数と、呼び出し時の引数の数を確認しましょう。デフォルト引数を使うと引数を省略可能にできます。

❌ エラー：undefined method

undefined method `greet' for main:Object (NoMethodError)

原因：メソッドが定義される前に呼び出されているか、メソッド名が間違っています。

対処法：メソッドの定義（def...end）を、呼び出しより前に書きましょう。メソッド名のスペルミスも確認してください。

❌ 戻り値がnilになってしまう

原因：メソッドの最後にputs文があると、nilが戻り値になります（putsの戻り値はnilのため）。

対処法：値を返したい場合は、最後の行に返したい値を書くか、returnを使いましょう。表示だけしたい場合はputsでOKです。

まとめ

この章では、処理を部品化するメソッドについて学びました。

  • メソッドとは、処理に名前を付けて再利用できるようにしたもの
  • def メソッド名...endで定義し、メソッド名で呼び出す
  • 引数でメソッドに値を渡し、戻り値で結果を受け取る
  • デフォルト引数を使うと、省略可能な引数を作れる
  • Rubyでは最後の式が自動的に戻り値になる（returnは省略可能）
  • メソッドを使うことで、再利用性・保守性・可読性が向上する
