第5章

条件分岐 - プログラムに判断をさせよう

「もし〜なら〜する」という条件判断を使って、状況に応じて動作を変えるプログラムを作ります。

この章で学ぶこと

  • 条件分岐の考え方を日常の例（傘を持っていくかなど）を通じて理解する
  • if文、elsif、elseを使った条件判断の書き方を習得する
  • 比較演算子（==, !=, <, >など）を使ってプログラムで判断する方法を学ぶ
  • 論理演算子（&&, ||）を使って複数の条件を組み合わせる方法を理解する

5.1 条件分岐とは？

私たちは日常生活で、状況によって判断を変えています。たとえば、「もし雨が降っていたら傘を持っていく」「もしお腹が空いていたらご飯を食べる」といった具合です。

プログラムでも同じように、「もし〜なら〜する」という条件判断を使うことができます。これを条件分岐と呼びます。

flowchart TD A([朝起きた]) --> B{雨が降っている?} B -->|はい| C[傘を持っていく] B -->|いいえ| D[傘は持たない] C --> E([出発]) D --> E style A fill:#fee2e2 style B fill:#fef3c7 style E fill:#d1fae5

条件分岐の仕組み

条件分岐では、まず「条件」を調べます（雨が降っているか？）。その結果が「真（true）」なら一つの処理を、「偽（false）」なら別の処理を実行します。

5.2 if文の基本

Rubyでは、if文を使って条件分岐を行います。基本的な書き方を見てみましょう。

if_basic.rb 
# 天気によって行動を変える
weather = "雨"

if weather == "雨"
  puts "傘を持っていきます"
end

このプログラムの意味は「もし天気が『雨』なら、『傘を持っていきます』と表示する」です。ifで条件を書き、最後にendで閉じます。

flowchart LR A["if 条件"] --> B["条件がtrueなら
処理を実行"] B --> C["end"] A -.->|条件がfalse| C style A fill:#fef3c7 style B fill:#dbeafe style C fill:#fee2e2

Rubyのif文の構造

if文は「if 条件」で始まり、「end」で終わります。条件が真（true）のときだけ、ifとendの間の処理が実行されます。

5.3 else - 「そうでなければ」の処理

条件が偽（false）のときに別の処理をしたい場合は、elseを使います。

if_else.rb 
# 天気によって行動を変える（elseあり）
weather = "晴れ"

if weather == "雨"
  puts "傘を持っていきます"
else
  puts "傘は必要ありません"
end

この例では、天気が「雨」でない場合（晴れ、曇りなど）は「傘は必要ありません」と表示されます。

5.4 elsif - 複数の条件を判定する

3つ以上の場合分けをしたいときは、elsif（else if の略）を使います。

if_elsif.rb 
# 点数によってメッセージを変える
score = 75

if score >= 80
  puts "素晴らしい！"
elsif score >= 60
  puts "合格です"
elsif score >= 40
  puts "もう少し頑張りましょう"
else
  puts "復習が必要です"
end

このプログラムは、点数によって4つのメッセージを出し分けます。条件は上から順番にチェックされ、最初に真（true）になった条件の処理だけが実行されます。

elsifの判定順序

elsifは複数書くことができますが、上から順番に判定され、最初に真になった条件の処理だけが実行されます。それ以降の条件は判定されません。

5.5 比較演算子 - 値を比べる

条件判断では、値を比較する比較演算子がよく使われます。以下が主な比較演算子です。

主な比較演算子

== 等しい（同じ値かどうか）

例: age == 20

!= 等しくない（異なる値かどうか）

例: age != 20

< より小さい（未満）

例: age < 20

> より大きい（超）

例: age > 20

<= 以下（その数を含む）

例: age <= 20

>= 以上（その数を含む）

例: age >= 20

comparison.rb 
# 年齢による判定
age = 18

if age >= 20
  puts "成人です"
else
  puts "未成年です"
end

# 実行結果: 未成年です

よくある間違い

等しいかどうかを調べるときは ==（イコール2つ）を使います。=（イコール1つ）は代入を意味するので、条件式では使えません。

5.6 論理演算子 - 複数の条件を組み合わせる

「雨が降っている、かつ、気温が低い」のように、複数の条件を組み合わせたい場合は論理演算子を使います。

主な論理演算子

&& AND（かつ）- 両方とも真のときだけ真

例: age >= 20 && has_license

|| OR（または）- どちらか一方でも真なら真

例: weather == "雨" || weather == "雪"

! NOT（否定）- 真偽を反転させる

例: !is_raining

AND演算子（&&）の例

and_example.rb 
# 映画館の年齢制限チェック
age = 25
has_id = true

if age >= 18 && has_id
  puts "入場できます"
else
  puts "入場できません"
end

# 実行結果: 入場できます

OR演算子（||）の例

or_example.rb 
# 休日判定
day = "土曜日"

if day == "土曜日" || day == "日曜日"
  puts "今日は休日です"
else
  puts "今日は平日です"
end

# 実行結果: 今日は休日です

論理演算子の使い分け

&&は「AかつB」両方の条件を満たす必要がある場合に、||は「AまたはB」どちらか一つでも満たせばよい場合に使います。

5.7 実用例：割引判定プログラム

ここまで学んだ条件分岐を使って、実用的なプログラムを作ってみましょう。年齢と会員種別によって割引率を決定するプログラムです。

discount.rb 
# 映画チケット割引判定
age = 15
is_member = true

if age < 12
  discount = 50  # 子供料金：50%割引
  puts "子供料金が適用されます（#{discount}%割引）"
elsif age >= 65
  discount = 40  # シニア料金：40%割引
  puts "シニア料金が適用されます（#{discount}%割引）"
elsif is_member
  discount = 20  # 会員割引：20%割引
  puts "会員割引が適用されます（#{discount}%割引）"
else
  discount = 0   # 割引なし
  puts "通常料金です"
end

このプログラムは、上から順に条件をチェックし、最初に当てはまった条件の処理を実行します。子供（12歳未満）とシニア（65歳以上）が優先され、それ以外は会員かどうかで判定されます。

実習 5-1: テスト点数判定プログラム

テストの点数を入力して、評価を表示するプログラムを作ってみましょう。

手順

  1. 1 変数scoreに点数（0〜100）を代入してください
  2. 2 90点以上なら「優秀」、80点以上なら「良い」、70点以上なら「普通」、60点以上なら「もう少し」、60点未満なら「要復習」と表示してください
  3. 3 いくつかの点数で試して、正しく判定されることを確認してください

トラブルシューティング

条件が真のはずなのに処理が実行されない

原因: 等しいかどうかの判定に = を使っている可能性があります。

解決方法: 等しいかどうかを調べるときは ==（イコール2つ）を使います。= は代入を意味します。

elsifの後の条件が判定されない

原因: elsifは上から順に判定され、最初に真になった条件だけが実行されます。

解決方法: 条件の順序を見直してください。より限定的な条件を先に書くのが一般的です。例：「80点以上」の前に「90点以上」を判定する。

ANDやOR条件がうまく動かない

原因: &&や||の使い方が間違っているか、括弧が必要な場合があります。

解決方法: 複雑な条件式は括弧で囲んで優先順位を明確にします。例：(age >= 18 && has_id) || is_vip

理解度確認クイズ

  1. Q1. 以下のコードの実行結果は何ですか？
    age = 16
    if age >= 18
    puts "成人"
    else
    puts "未成年"
    end

    回答を表示

    「未成年」が表示されます。ageは16なので、age >= 18は偽（false）となり、elseの処理が実行されます。

  2. Q2. ==（イコール2つ）と=（イコール1つ）の違いは何ですか？

    回答を表示

    ==は「等しいかどうかを比較する」演算子で、=は「値を代入する」演算子です。条件分岐では==を使います。

  3. Q3. &&（AND）と||（OR）の違いを説明してください。

    回答を表示

    &&は「両方とも真のときだけ真」になります（例：18歳以上かつ免許あり）。||は「どちらか一方でも真なら真」になります（例：土曜日または日曜日）。

まとめ

この章で学んだこと

  • 条件分岐は「もし〜なら〜する」という判断をプログラムで実現する機能
  • if文、elsif、elseを使って複数の条件を判定できる
  • 比較演算子（==, !=, <, >, <=, >=）で値を比較する
  • 論理演算子（&&, ||）で複数の条件を組み合わせる
