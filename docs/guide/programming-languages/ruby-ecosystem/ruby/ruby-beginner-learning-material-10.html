クラスとオブジェクト入門 - Rubyらしいプログラミングの第一歩

「オブジェクト」という考え方と、その設計図となる「クラス」の基礎を学び、Rubyの特徴である「すべてがオブジェクト」という概念を理解しましょう。

この章で学ぶこと

  • クラスとは何か、設計図に例えて理解する
  • オブジェクトとは何か、設計図から作られる実体として理解する
  • クラスの定義方法と、initializeメソッドの役割を学ぶ
  • インスタンス変数とインスタンスメソッドの使い方を習得する
  • Rubyの「すべてがオブジェクト」という哲学を理解する

10.1 クラスとオブジェクトの概念

これまでに学んできた変数、配列、ハッシュ、メソッドなどは、すべて「オブジェクト指向プログラミング」の一部です。オブジェクト指向とは、「物（オブジェクト）」を中心にプログラムを組み立てる考え方です。

例えば、犬を飼うことを考えてみましょう。犬には「名前」「年齢」「犬種」などの情報（データ）があり、「吠える」「走る」「食べる」などの動作（機能）があります。

classDiagram class 犬 { +名前 +年齢 +犬種 +吠える() +走る() +食べる() }

このように、データと機能をひとまとめにしたものが「オブジェクト」です。そして、オブジェクトを作るための設計図が「クラス」です。

設計図と実体

クラスは「犬という設計図」、オブジェクトは「ポチ、タロウ、ハチなど具体的な犬」というイメージです。設計図（クラス）は1つですが、そこから何匹でも犬（オブジェクト）を作ることができます。

10.2 クラスの定義方法

Rubyでクラスを定義するには、classキーワードを使います。クラス名は必ず大文字で始めるルールがあります。

dog.rb 
# 犬クラスの定義
class Dog
  # ここにクラスの内容を書く
end

# クラスからオブジェクトを作成
pochi = Dog.new
puts "オブジェクトを作成しました: #{pochi}"

Dog.newで、Dogクラスの設計図から「新しい犬」を作成しています。これを「インスタンス化」と呼び、作られたオブジェクトを「インスタンス」と呼びます。

10.3 initializeメソッド - オブジェクト作成時の初期設定

オブジェクトを作成する際に、最初に実行される特別なメソッドがinitializeです。これは、オブジェクトに名前や年齢などの初期値を設定するために使います。

dog_with_initialize.rb 
# initializeメソッドを持つDogクラス
class Dog
  # オブジェクト作成時に自動実行される
  def initialize(name, age)
    @name = name  # インスタンス変数に保存
    @age = age
    puts "#{@name}という犬が生まれました（#{@age}歳）"
  end
end

# オブジェクト作成（initializeが自動実行される）
pochi = Dog.new("ポチ", 3)
taro = Dog.new("タロウ", 5)

実行結果

ポチという犬が生まれました（3歳）
タロウという犬が生まれました（5歳）

initializeの仕組み

Dog.new("ポチ", 3)と書くと、自動的にinitialize("ポチ", 3)が実行されます。newメソッドが内部でinitializeを呼び出すため、明示的に呼ぶ必要はありません。

10.4 インスタンス変数 - オブジェクト固有のデータ

@で始まる変数をインスタンス変数と呼びます。これは、そのオブジェクト専用のデータを保存する変数です。

インスタンス変数は、クラス内のどのメソッドからでもアクセスできるのが特徴です。ポチの@nameとタロウの@nameは別々の変数です。

instance_variable.rb 
class Dog
  def initialize(name, age)
    @name = name  # インスタンス変数（ポチ専用）
    @age = age
  end

  def introduce
    # 同じクラス内なら、どのメソッドからでも@nameにアクセスできる
    puts "私の名前は#{@name}です。#{@age}歳です。"
  end

  def birthday
    @age += 1  # 年齢を1つ増やす
    puts "#{@name}は#{@age}歳になりました！"
  end
end

pochi = Dog.new("ポチ", 3)
pochi.introduce  # 私の名前はポチです。3歳です。
pochi.birthday   # ポチは4歳になりました！

ローカル変数との違い

nameのような普通の変数（ローカル変数）は、そのメソッド内でしか使えません。一方、@nameのようなインスタンス変数は、クラス内のどのメソッドからでもアクセスできます。

10.5 インスタンスメソッド - オブジェクトの動作

クラス内で定義したメソッドをインスタンスメソッドと呼びます。これは、オブジェクトができること（動作）を定義します。

dog_methods.rb 
class Dog
  def initialize(name, breed)
    @name = name
    @breed = breed
  end

  # 吠えるメソッド
  def bark
    puts "#{@name}: ワンワン！"
  end

  # 走るメソッド
  def run
    puts "#{@name}が元気に走っています！"
  end

  # 自己紹介メソッド
  def introduce
    puts "名前: #{@name}, 犬種: #{@breed}"
  end
end

# 2匹の犬を作成
pochi = Dog.new("ポチ", "柴犬")
taro = Dog.new("タロウ", "ゴールデンレトリバー")

# それぞれのメソッドを実行
pochi.bark       # ポチ: ワンワン！
taro.run         # タロウが元気に走っています！
pochi.introduce  # 名前: ポチ, 犬種: 柴犬

pochi.barkのように、オブジェクト名.メソッド名の形式でメソッドを呼び出します。これまで使ってきた配列.pushや文字列.upcaseも、実は同じ仕組みでした。

実習：本クラスを作ってみよう

「本」を表すBookクラスを作成しましょう。本には「タイトル」「著者」「ページ数」があり、「情報を表示する」動作ができます。

book.rb 
class Book
  def initialize(title, author, pages)
    @title = title    # タイトル
    @author = author  # 著者
    @pages = pages    # ページ数
  end

  # 本の情報を表示
  def display_info
    puts "「#{@title}」"
    puts "著者: #{@author}"
    puts "ページ数: #{@pages}ページ"
  end

  # 読むメソッド
  def read
    puts "#{@title}を読んでいます..."
  end
end

# 本のオブジェクトを作成
ruby_book = Book.new("Ruby入門", "山田太郎", 300)
ruby_book.display_info
ruby_book.read

10.6 Rubyの「すべてがオブジェクト」

Rubyでは、数値、文字列、配列など、すべてがオブジェクトです。これまで何気なく使っていた.lengthや.upcaseなどは、実はインスタンスメソッドだったのです。

everything_is_object.rb 
# 数値もオブジェクト（Integerクラスのインスタンス）
num = 5
puts num.class  # Integer
puts num.even?  # false（偶数かチェックするメソッド）

# 文字列もオブジェクト（Stringクラスのインスタンス）
str = "hello"
puts str.class     # String
puts str.upcase    # HELLO（大文字に変換するメソッド）
puts str.length    # 5（長さを返すメソッド）

# 配列もオブジェクト（Arrayクラスのインスタンス）
arr = [1, 2, 3]
puts arr.class    # Array
puts arr.sum      # 6（合計を計算するメソッド）

Rubyの一貫性

すべてがオブジェクトなので、5.times { ... }や"hello".reverseのように、数値や文字列に対してもメソッドを呼び出せます。これがRubyの「一貫性のある美しい文法」と言われる理由です。

10.7 実践例 - 人物管理プログラム

これまでに学んだことを組み合わせて、人物を管理するPersonクラスを作成してみましょう。

person.rb 
class Person
  def initialize(name, age, hobby)
    @name = name
    @age = age
    @hobby = hobby
  end

  # 自己紹介メソッド
  def introduce
    puts "こんにちは！#{@name}です。"
    puts "#{@age}歳で、趣味は#{@hobby}です。"
  end

  # 誕生日メソッド
  def birthday
    @age += 1
    puts "#{@name}の誕生日！#{@age}歳になりました🎂"
  end

  # 趣味を変更するメソッド
  def change_hobby(new_hobby)
    puts "#{@name}の趣味が#{@hobby}から#{new_hobby}に変わりました"
    @hobby = new_hobby
  end
end

# 人物オブジェクトを作成
alice = Person.new("アリス", 25, "読書")
bob = Person.new("ボブ", 30, "ジョギング")

# メソッドを実行
alice.introduce
alice.birthday
alice.change_hobby("プログラミング")

puts "\n---\n"

bob.introduce
bob.birthday

10.8 クラス設計のポイント

良いクラスを作るためには、以下のポイントを意識しましょう。

1. 1つのクラスには1つの責任

Dogクラスは「犬」に関することだけを扱い、Bookクラスは「本」に関することだけを扱います。1つのクラスに何でも詰め込むと、管理が難しくなります。

2. データは隠す（カプセル化）

インスタンス変数は@nameのように外部から直接触れません。データを変更したい場合は、メソッドを通して行います。これにより、予期しない変更を防げます。

3. 分かりやすい名前をつける

クラス名、メソッド名、変数名は、何をするものかが一目で分かる名前にしましょう。Dog、bark、@nameのように、英語でも意味が通じる名前が理想です。

理解度確認クイズ

  1. Q1. クラスとオブジェクトの関係を、身近な例を使って説明してください。

    回答を表示

    クラスは「設計図」、オブジェクトは「設計図から作られた実体」です。例えば、Dogクラスは「犬という設計図」であり、そこからポチ、タロウ、ハチなどの具体的な犬（オブジェクト）を何匹でも作ることができます。

  2. Q2. initializeメソッドの役割は何ですか？また、どのタイミングで実行されますか？

    回答を表示

    initializeメソッドは、オブジェクトに初期値を設定する役割を持ちます。Dog.newなどでオブジェクトを作成する際に、自動的に実行されます。名前や年齢などの初期データを設定するために使います。

  3. Q3. インスタンス変数（@で始まる変数）の特徴を2つ挙げてください。

    回答を表示

    ①そのオブジェクト専用のデータを保存する（ポチの@nameとタロウの@nameは別々）、②クラス内のどのメソッドからでもアクセスできる、などが挙げられます。

  4. Q4. Rubyの「すべてがオブジェクト」とはどういう意味ですか？具体例を挙げて説明してください。

    回答を表示

    Rubyでは、数値、文字列、配列など、すべてのデータがクラスから作られたオブジェクトです。例えば、5はIntegerクラスのインスタンスであり、"hello"はStringクラスのインスタンスです。そのため、5.even?や"hello".upcaseのように、すべてのデータに対してメソッドを呼び出すことができます。

まとめ

  • クラスは設計図、オブジェクトは設計図から作られた実体
  • initializeメソッドは、オブジェクト作成時に自動実行される特別なメソッド
  • インスタンス変数（@で始まる変数）は、オブジェクト専用のデータを保存する
  • インスタンスメソッドは、オブジェクトができること（動作）を定義する
  • Rubyではすべてがオブジェクトなので、一貫性のある文法でプログラムを書ける
