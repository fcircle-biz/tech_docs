変数とデータ型 - 値を入れる「箱」の使い方

プログラムで扱うデータを保存する「変数」という箱の概念を学びます。数値、文字列、真偽値など、基本的なデータの種類と使い方を身近な例で理解しましょう。

この章で学ぶこと

  • 変数とは何か、「名前付きの箱」として理解する
  • 変数の宣言と代入の仕組みを理解し、実際に使えるようになる
  • 文字列、数値、真偽値などの基本的なデータ型の違いを知る
  • 型変換の方法を学び、データ型を適切に扱えるようになる

3.1 変数とは何か？

「変数」という言葉は、数学の授業で聞いたことがあるかもしれません。プログラミングにおける変数とは、データを入れておく「名前付きの箱」のようなものです。

例えば、実生活でラベルを貼った収納ボックスを想像してください。「文房具」とラベルを貼った箱には文房具を入れ、「書類」とラベルを貼った箱には書類を入れますよね。プログラミングの変数も同じように、名前を付けた箱に値を入れて、後で取り出すことができます。

なぜ変数が必要なのか？

変数を使うことで、計算結果を保存したり、ユーザーの入力を受け取ったり、何度も同じ値を使い回したりできます。変数がないと、プログラムで複雑な処理を行うことはほぼ不可能です。

3.2 変数の宣言と代入

Rubyでは、変数に値を入れることを「代入」と呼びます。代入には=という記号（代入演算子）を使います。

variable_basic.rb 
# 変数に値を代入する
name = "太郎"
age = 25
height = 175.5

# 変数の値を表示する
puts name    # => 太郎
puts age     # => 25
puts height  # => 175.5

変数名の付け方のルール

変数名を付けるときには、いくつかのルールがあります。

変数名のルール

  • 小文字のアルファベットで始める（user_name、age など）
  • アルファベット、数字、アンダースコア（_）が使える
  • 複数の単語を組み合わせるときはアンダースコアでつなぐ（user_name、total_price など）
  • 数字から始めることはできない（1st_name は×）
  • Rubyの予約語（if、end、class など）は使えない
variable_naming.rb 
# 良い変数名の例
user_name = "山田太郎"
total_price = 1000
is_student = true

# 避けるべき変数名の例
x = "山田太郎"       # 意味が分かりにくい
totalprice = 1000   # 単語の区切りが分かりにくい
# 1st_name = "太郎" # エラー：数字で始まっている

分かりやすい変数名を付けよう

変数名は、後で見返したときに「何を入れているか」がすぐ分かるような名前にしましょう。user_name（ユーザー名）のように、英語の意味が分かる名前を付けると良いでしょう。

3.3 データ型の種類

変数に入れることができる値には、いくつかの「型」（データ型）があります。Rubyの基本的なデータ型を見ていきましょう。

文字列（String）

文字や文章を扱う型です。"または'で囲みます。

name = "太郎"

整数（Integer）

小数点を含まない数値です。計算に使います。

age = 25

小数（Float）

小数点を含む数値です。より精密な計算に使います。

height = 175.5

真偽値（Boolean）

「真」（true）か「偽」（false）のどちらかです。条件判定に使います。

is_student = true
data_types.rb 
# 様々なデータ型の例
name = "山田太郎"       # 文字列（String）
age = 25              # 整数（Integer）
height = 175.5        # 小数（Float）
is_student = true     # 真偽値（Boolean）

# データ型を確認する
puts name.class       # => String
puts age.class        # => Integer
puts height.class     # => Float
puts is_student.class # => TrueClass

3.4 文字列型（String）

文字列は、テキストを扱うためのデータ型です。ダブルクォート（"）またはシングルクォート（'）で囲んで作ります。

string_example.rb 
# ダブルクォートとシングルクォート
message1 = "こんにちは"
message2 = 'こんにちは'

puts message1  # => こんにちは
puts message2  # => こんにちは

式展開（しきてんかい）

ダブルクォートで囲んだ文字列では、#{}を使って変数の値を埋め込むことができます。これを式展開と呼びます。

string_interpolation.rb 
# 式展開を使った例
name = "太郎"
age = 25

# #{} の中に変数を書くと、その値が埋め込まれる
puts "私の名前は#{name}です"          # => 私の名前は太郎です
puts "#{name}さんは#{age}歳です"      # => 太郎さんは25歳です

# シングルクォートでは式展開できない
puts '私の名前は#{name}です'          # => 私の名前は#{name}です（そのまま表示）

ダブルクォートとシングルクォートの使い分け

変数を埋め込みたいときはダブルクォート（"）、単純な文字列だけならシングルクォート（'）を使うのが一般的です。

3.5 数値型（Integer / Float）

数値型には、整数（Integer）と小数（Float）の2種類があります。

number_example.rb 
# 整数（Integer）
age = 25
count = 100
temperature = -5

puts age         # => 25
puts count       # => 100
puts temperature # => -5

# 小数（Float）
height = 175.5
price = 1980.0
rate = 0.05

puts height # => 175.5
puts price  # => 1980.0
puts rate   # => 0.05

整数と小数の計算の違い

整数同士の割り算では、小数点以下が切り捨てられます。小数の結果が欲しい場合は、どちらかを小数にする必要があります。

number_calculation.rb 
# 整数同士の割り算（小数点以下は切り捨て）
puts 10 / 3      # => 3

# 小数を含む割り算（小数点以下も計算される）
puts 10.0 / 3    # => 3.3333333333333335
puts 10 / 3.0    # => 3.3333333333333335

整数の割り算に注意

初心者がよくつまずくポイントです。10 / 3は3になります（3.333...ではない）。小数の結果が欲しい場合は、10.0 / 3のように小数を使いましょう。

3.6 真偽値型（Boolean）

真偽値型は、true（真、正しい）かfalse（偽、間違い）のどちらかの値しか持ちません。主に条件判定で使います。

boolean_example.rb 
# 真偽値の例
is_student = true
is_adult = false

puts is_student  # => true
puts is_adult    # => false

# 比較演算で真偽値が返される
age = 25
puts age > 20    # => true（25は20より大きい）
puts age < 18    # => false（25は18より小さくない）

真偽値の活用

真偽値は、第5章で学ぶ「条件分岐」で大活躍します。「もし〜が正しければ（trueなら）こうする」といった判断をプログラムに任せることができます。

3.7 nilと型変換

nil（何も入っていない）

nilは、「何も入っていない」「値が存在しない」ことを表す特別な値です。空の箱のようなイメージです。

nil_example.rb 
# nilの例
result = nil
puts result  # => （何も表示されない）

# nilかどうかを確認する
puts result.nil?  # => true

型変換（データ型を変える）

異なるデータ型同士を組み合わせたいとき、型変換（型を変えること）が必要になります。Rubyには型変換のための便利なメソッドが用意されています。

主な型変換メソッド

  • to_s 文字列（String）に変換
  • to_i 整数（Integer）に変換
  • to_f 小数（Float）に変換
type_conversion.rb 
# 型変換の例

# 数値を文字列に変換
age = 25
age_str = age.to_s
puts "私は" + age_str + "歳です"  # => 私は25歳です

# 文字列を数値に変換
price_str = "1000"
price = price_str.to_i
puts price + 500  # => 1500（数値として計算される）

# 整数を小数に変換
num = 10
num_float = num.to_f
puts num_float / 3  # => 3.3333333333333335

型変換が必要な場面

文字列と数値を+で直接つなぐことはできません。「私は」+「25」+「歳です」のように文字列をつなぎたい場合は、数値を.to_sで文字列に変換する必要があります。

実習 3-1: 自己紹介プログラムを作ろう

変数とデータ型を使って、自己紹介を表示するプログラムを作ってみましょう。

手順

  1. 1 新しいファイルself_introduction.rbを作成します
  2. 2 自分の名前、年齢、身長を変数に代入します
  3. 3 式展開を使って、「私の名前は○○です。年齢は○○歳で、身長は○○cmです。」と表示します
self_introduction.rb（例） 
# 自己紹介プログラム
name = "太郎"
age = 25
height = 175.5

puts "私の名前は#{name}です。"
puts "年齢は#{age}歳で、身長は#{height}cmです。"

理解度確認クイズ

  1. Q1. 変数とは何ですか？身近な例を使って説明してください。

    回答を表示

    変数とは、データを入れておく「名前付きの箱」のようなものです。実生活で言えば、「文房具」とラベルを貼った収納ボックスに文房具を入れるように、プログラミングでは名前を付けた箱に値を入れて、後で取り出すことができます。

  2. Q2. 以下のコードの出力結果は何ですか？
    puts 10 / 3

    回答を表示

    答えは3です。整数同士の割り算では小数点以下が切り捨てられます。3.333...という結果が欲しい場合は、10.0 / 3のように小数を使う必要があります。

  3. Q3. 数値age = 25を文字列に変換するメソッドは何ですか？

    回答を表示

    to_sメソッドです。age.to_sとすることで、数値25を文字列"25"に変換できます。文字列と数値をつなげたいときに便利です。

  4. Q4. ダブルクォート（"）とシングルクォート（'）の違いは何ですか？

    回答を表示

    ダブルクォート（"）で囲んだ文字列では、#{}を使った式展開ができます。シングルクォート（'）で囲んだ文字列では式展開はできず、そのまま表示されます。変数を埋め込みたいときはダブルクォートを使いましょう。
