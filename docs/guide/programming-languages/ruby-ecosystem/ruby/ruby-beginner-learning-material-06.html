繰り返し処理 - 同じ作業を自動化しよう

eachメソッド、timesメソッド、whileループを使って、繰り返し処理を自動化する方法を学びます。Rubyならではの直感的な繰り返し処理を体験しましょう。

この章で学ぶこと

  • 繰り返し処理とは何か、日常の例を通じて理解する
  • timesメソッドで決まった回数だけ繰り返す方法を習得する
  • eachメソッドで配列の要素を1つずつ処理する方法を学ぶ
  • whileループで条件が成り立つ間繰り返す方法を理解する

6.1 繰り返し処理とは何か？

「繰り返し処理」とは、同じ作業を何度も自動的に実行することです。これは、日常生活でもよく行っている作業です。

たとえば、先生が生徒30人分のテストの採点をするとき、「①答案用紙を見る」「②正解と比較する」「③点数をつける」という作業を30回繰り返します。この「繰り返し」をプログラムで実現するのが繰り返し処理です。

プログラミングにおける繰り返しの威力

人間が手作業で30回同じことをするのは大変ですが、プログラムなら一瞬で何百回、何千回でも繰り返せます。これがプログラミングの大きな利点です。

6.2 timesメソッド - 決まった回数だけ繰り返す

timesメソッドは、「○回繰り返す」という処理を実現する最もシンプルな方法です。Rubyならではの読みやすい書き方ができます。

基本的な使い方

times_basic.rb 
# 「こんにちは」を3回表示する
3.times do
  puts "こんにちは"
end

このコードを実行すると、以下のように3回繰り返し表示されます。

実行結果

こんにちは
こんにちは
こんにちは

カウント付きの繰り返し

timesメソッドには、何回目の繰り返しかを示す番号を取得する機能もあります。この番号は0から始まることに注意してください。

times_with_index.rb 
# 1から5までの数字を表示する
5.times do |i|
  puts "#{i + 1}回目の繰り返しです"
end

実行結果

1回目の繰り返しです
2回目の繰り返しです
3回目の繰り返しです
4回目の繰り返しです
5回目の繰り返しです

|i| の意味

|i| は、繰り返しの回数を受け取る変数です。0から始まるので、1回目は i=0、2回目は i=1 となります。人間が数える「1回目、2回目」にしたい場合は i + 1 を使います。

6.3 eachメソッド - 配列の要素を1つずつ処理

eachメソッドは、配列（リスト）の中身を1つずつ取り出して処理するときに使います。買い物リストの商品を1つずつ確認するイメージです。

基本的な使い方

each_basic.rb 
# 果物のリストを1つずつ表示する
fruits = ["りんご", "バナナ", "オレンジ"]

fruits.each do |fruit|
  puts "#{fruit}が好きです"
end

実行結果

りんごが好きです
バナナが好きです
オレンジが好きです

実践例：合計点の計算

eachメソッドを使って、テストの点数の合計を計算してみましょう。

each_sum.rb 
# テストの点数リスト
scores = [80, 75, 90, 65, 88]

# 合計点を計算
total = 0
scores.each do |score|
  total = total + score  # または total += score
end

puts "合計点: #{total}点"
puts "平均点: #{total / scores.length}点"

実行結果

合計点: 398点
平均点: 79点

6.4 whileループ - 条件が成り立つ間繰り返す

whileは、「ある条件が成り立っている間、ずっと繰り返す」というループです。「お腹がいっぱいになるまで食べ続ける」のような繰り返しに使います。

基本的な使い方

while_basic.rb 
# 1から5までカウントアップ
count = 1

while count <= 5
  puts "現在のカウント: #{count}"
  count = count + 1  # または count += 1
end

puts "カウント終了！"

実行結果

現在のカウント: 1
現在のカウント: 2
現在のカウント: 3
現在のカウント: 4
現在のカウント: 5
カウント終了！

無限ループに注意！

whileでは、条件が永遠に真のままだと、プログラムが止まらなくなります（無限ループ）。必ず「ループを終わらせる条件」を設定し、カウンタを更新することを忘れないでください。

実践例：ユーザー入力の繰り返し

whileは、ユーザーが「終了」と入力するまで繰り返すような処理に向いています。

while_input.rb 
# 「終了」と入力されるまで繰り返す
puts "メッセージを入力してください（終了するには「終了」と入力）"

input = ""
while input != "終了"
  print "> "
  input = gets.chomp  # ユーザーの入力を取得
  puts "あなたの入力: #{input}"
end

puts "プログラムを終了します"

6.5 forループ - 範囲を指定して繰り返す

forは、範囲や配列を指定して繰り返す方法です。ただし、Rubyではeachメソッドの方が好まれます。

for_basic.rb 
# forループで1から5まで表示
for i in 1..5
  puts "数字: #{i}"
end

# eachの方がRubyらしい書き方（推奨）
(1..5).each do |i|
  puts "数字: #{i}"
end

eachの方が好まれる理由

Rubyでは、forよりもeachの方が「Rubyらしい」書き方とされています。理由は、eachの方が柔軟で、Rubyの「メソッドチェーン」という強力な機能と組み合わせやすいためです。

6.6 繰り返し処理の使い分け

Rubyには複数の繰り返し方法がありますが、状況に応じて適切なものを選ぶことが大切です。

繰り返し方法 使うとき 具体例
times 決まった回数だけ繰り返したいとき 「10回繰り返す」
each 配列の要素を1つずつ処理したいとき 「買い物リストの全商品を確認」
while 条件が成り立つ間繰り返したいとき 「正しい入力があるまで繰り返す」
for （eachを使う方が推奨） 範囲指定の繰り返し

実習: 九九表を作ってみよう

timesメソッドを2重に使って、九九表を作成してみましょう。

multiplication_table.rb 
# 九九表を表示する
9.times do |i|
  row = i + 1  # 1の段から9の段まで

  9.times do |j|
    col = j + 1  # 1から9まで
    result = row * col
    print "#{result}\t"  # タブ区切りで表示
  end

  puts ""  # 改行
end

チャレンジ: eachメソッドを使って、配列 [1, 2, 3, 4, 5] の各要素を2倍にして表示するプログラムを作ってみましょう。

理解度確認クイズ

  1. Q1. 「こんにちは」を5回表示するプログラムを、timesメソッドを使って書いてください。

    回答を表示 
    5.times do
  puts "こんにちは"
end

  2. Q2. eachメソッドとtimesメソッドの違いは何ですか？

    回答を表示

    timesメソッドは「決まった回数だけ繰り返す」ときに使い、eachメソッドは「配列の要素を1つずつ取り出して処理する」ときに使います。timesは回数が重要で、eachは配列の中身が重要です。

  3. Q3. whileループで無限ループにならないために注意すべきことは何ですか？

    回答を表示

    ①ループを終わらせる条件を必ず設定すること、②ループ内で条件を変化させる処理（カウンタの更新など）を忘れないことです。条件が永遠に真のままだと、プログラムが止まらなくなります。
