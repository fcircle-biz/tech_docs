配列とハッシュ - 複数のデータをまとめて管理しよう

買い物リストや成績表のように、関連するデータをまとめて扱う方法を学び、配列とハッシュの使い分けを習得しましょう。

この章で学ぶこと

  • 配列を「番号付きの引き出し」として理解し、基本操作を習得する
  • ハッシュを「ラベル付きの引き出し」として理解し、基本操作を習得する
  • 要素の追加、削除、検索などの操作を具体例で学ぶ
  • 配列とハッシュの使い分けを理解し、実践的なプログラムを作る

7.1 配列とは？ - 番号付きの引き出し

これまでは、変数1つにつき値1つを保存してきました。しかし、「買い物リストの品物」や「クラスの生徒の名前」など、関連するデータを複数まとめて扱いたい場合はどうすればいいでしょうか？

そこで登場するのが配列（Array）です。配列は、「番号付きの引き出し」のようなものと考えるとわかりやすいでしょう。

flowchart LR subgraph Array["配列: fruits"] direction TB A["[0] りんご"] B["[1] バナナ"] C["[2] みかん"] D["[3] ぶどう"] end style Array fill:#fee2e2 style A fill:#fef3c7 style B fill:#fef3c7 style C fill:#fef3c7 style D fill:#fef3c7

インデックス（添字）とは

配列の各引き出しには「番号」が付いています。この番号をインデックス（添字）と呼びます。重要なのは、インデックスは0から始まるということです。最初の要素は[0]、2番目の要素は[1]となります。

配列の作り方

配列は、角カッコ[]の中に値をカンマで区切って並べます。

array_basic.rb 
# 配列の作成
fruits = ["りんご", "バナナ", "みかん", "ぶどう"]

# 配列の中身を表示
puts fruits
# 出力:
# りんご
# バナナ
# みかん
# ぶどう

# 配列全体を確認
p fruits
# 出力: ["りんご", "バナナ", "みかん", "ぶどう"]

配列の要素にアクセスする

配列の特定の引き出しを開くには、配列名[インデックス]と書きます。

array_access.rb 
fruits = ["りんご", "バナナ", "みかん", "ぶどう"]

# 最初の要素（インデックス0）を取得
puts fruits[0]  # => りんご

# 2番目の要素（インデックス1）を取得
puts fruits[1]  # => バナナ

# 最後の要素を取得（-1は最後を表す）
puts fruits[-1] # => ぶどう

# 存在しないインデックスを指定すると nil が返る
puts fruits[10] # => （何も表示されない）

マイナスインデックス

Rubyでは、-1で最後の要素、-2で最後から2番目の要素にアクセスできます。「後ろから数える」と考えるとわかりやすいです。

7.2 配列の基本操作

要素を追加する

配列に新しい要素を追加するには、pushメソッドや<<演算子を使います。

array_add.rb 
fruits = ["りんご", "バナナ"]

# 最後に要素を追加（pushメソッド）
fruits.push("みかん")
p fruits # => ["りんご", "バナナ", "みかん"]

# 最後に要素を追加（<<演算子）
fruits << "ぶどう"
p fruits # => ["りんご", "バナナ", "みかん", "ぶどう"]

# 最初に要素を追加（unshiftメソッド）
fruits.unshift("いちご")
p fruits # => ["いちご", "りんご", "バナナ", "みかん", "ぶどう"]

要素を削除する

array_delete.rb 
fruits = ["いちご", "りんご", "バナナ", "みかん", "ぶどう"]

# 最後の要素を削除（popメソッド）
last_fruit = fruits.pop
puts "削除された要素: #{last_fruit}" # => ぶどう
p fruits # => ["いちご", "りんご", "バナナ", "みかん"]

# 最初の要素を削除（shiftメソッド）
first_fruit = fruits.shift
puts "削除された要素: #{first_fruit}" # => いちご
p fruits # => ["りんご", "バナナ", "みかん"]

# 値を指定して削除（deleteメソッド）
fruits.delete("バナナ")
p fruits # => ["りんご", "みかん"]

要素を検索する

array_search.rb 
fruits = ["りんご", "バナナ", "みかん", "ぶどう"]

# 配列に要素が含まれているかチェック（include?メソッド）
if fruits.include?("バナナ")
  puts "バナナがあります！"
else
  puts "バナナはありません"
end

# 要素のインデックスを取得（indexメソッド）
index = fruits.index("みかん")
puts "みかんは#{index}番目にあります" # => みかんは2番目にあります

# 配列の要素数を取得（sizeメソッド）
puts "果物は全部で#{fruits.size}種類です" # => 果物は全部で4種類です

7.3 ハッシュとは？ - ラベル付きの引き出し

配列は「番号」で引き出しを管理していましたが、ハッシュ（Hash）は「ラベル」で引き出しを管理する仕組みです。

たとえば、「太郎さんの年齢は20歳、花子さんの年齢は25歳」というデータを保存したい場合、配列だとインデックス番号で管理するため分かりにくいですが、ハッシュなら「太郎」「花子」というラベルで管理できます。

flowchart LR subgraph Hash["ハッシュ: ages"] direction TB A["[太郎] => 20"] B["[花子] => 25"] C["[次郎] => 18"] end style Hash fill:#fee2e2 style A fill:#e9d5ff style B fill:#e9d5ff style C fill:#e9d5ff

キーと値

ハッシュでは、ラベルのことをキー（key）、そのラベルに対応するデータを値（value）と呼びます。辞書で「単語（キー）」を引いて「意味（値）」を調べるイメージです。

ハッシュの作り方

ハッシュは、波カッコ{}の中にキー => 値の形式で書きます。

hash_basic.rb 
# ハッシュの作成
ages = {
  "太郎" => 20,
  "花子" => 25,
  "次郎" => 18
}

# ハッシュ全体を確認
p ages
# 出力: {"太郎"=>20, "花子"=>25, "次郎"=>18}

# キーを指定して値を取得
puts "太郎さんの年齢: #{ages["太郎"]}歳" # => 太郎さんの年齢: 20歳
puts "花子さんの年齢: #{ages["花子"]}歳" # => 花子さんの年齢: 25歳

シンボルを使ったハッシュ

Rubyでは、キーにシンボルを使うことが一般的です。シンボルは:名前のように、コロンで始まる識別子です。

hash_symbol.rb 
# シンボルをキーにしたハッシュ（旧書式）
person = {
  :name => "太郎",
  :age => 20,
  :city => "東京"
}

# シンボルをキーにしたハッシュ（新書式・推奨）
person = {
  name: "太郎",
  age: 20,
  city: "東京"
}

# 値を取得
puts person[:name]  # => 太郎
puts person[:age]   # => 20
puts person[:city]  # => 東京

シンボルと文字列の違い

シンボルは文字列より効率的で、ハッシュのキーとして使うのに最適です。name: "太郎"という書き方は、:name => "太郎"と同じ意味です。

7.4 ハッシュの基本操作

要素を追加・更新する

hash_add.rb 
person = {
  name: "太郎",
  age: 20
}

# 新しいキーと値を追加
person[:city] = "東京"
p person # => {:name=>"太郎", :age=>20, :city=>"東京"}

# 既存のキーの値を更新
person[:age] = 21
p person # => {:name=>"太郎", :age=>21, :city=>"東京"}

要素を削除する

hash_delete.rb 
person = {
  name: "太郎",
  age: 21,
  city: "東京"
}

# キーを指定して削除（deleteメソッド）
deleted_value = person.delete(:city)
puts "削除された値: #{deleted_value}" # => 東京
p person # => {:name=>"太郎", :age=>21}

要素を検索する

hash_search.rb 
person = {
  name: "太郎",
  age: 21,
  city: "東京"
}

# キーが存在するかチェック（has_key?メソッド）
if person.has_key?(:age)
  puts "年齢の情報があります"
end

# 全てのキーを取得（keysメソッド）
p person.keys # => [:name, :age, :city]

# 全ての値を取得（valuesメソッド）
p person.values # => ["太郎", 21, "東京"]

# 要素数を取得（sizeメソッド）
puts "情報は#{person.size}項目です" # => 情報は3項目です

ハッシュの繰り返し処理

hash_each.rb 
person = {
  name: "太郎",
  age: 21,
  city: "東京"
}

# eachメソッドで全ての要素を処理
person.each do |key, value|
  puts "#{key}: #{value}"
end

# 出力:
# name: 太郎
# age: 21
# city: 東京

7.5 配列とハッシュの使い分け

配列とハッシュ、どちらを使うべきか迷ったときは、以下の基準で判断しましょう。

配列を使う場面

  • 順番が重要な場合（買い物リスト、待機列など）
  • 同じ種類のデータをまとめる場合（果物の名前、数値のリストなど）
  • インデックス番号で管理しやすい場合

ハッシュを使う場面

  • ラベル名で管理したい場合（名前と年齢、商品名と価格など）
  • 異なる種類のデータをまとめる場合（ユーザー情報など）
  • キーで検索することが多い場合

7.6 実践：買い物リストを作ろう

配列とハッシュを組み合わせて、商品名と価格を管理する買い物リストプログラムを作ってみましょう。

実習 7-1: 買い物リストプログラム

ハッシュの配列を使って、買い物リストを管理するプログラムを作りましょう。

shopping_list.rb 
# 買い物リスト（ハッシュの配列）
shopping_list = [
  { name: "りんご", price: 150, quantity: 3 },
  { name: "バナナ", price: 100, quantity: 5 },
  { name: "牛乳", price: 200, quantity: 1 }
]

puts "=== 買い物リスト ==="
total = 0

# 各商品の情報を表示し、合計金額を計算
shopping_list.each do |item|
  subtotal = item[:price] * item[:quantity]
  puts "#{item[:name]}: #{item[:price]}円 × #{item[:quantity]}個 = #{subtotal}円"
  total += subtotal
end

puts "-------------------"
puts "合計金額: #{total}円"

動作確認

ターミナルでruby shopping_list.rbを実行してください。

期待される結果

バナナ: 100円 × 5個 = 500円
牛乳: 200円 × 1個 = 200円
-------------------
合計金額: 1150円

配列とハッシュの組み合わせ

このプログラムでは、ハッシュ（1つの商品情報）を配列（買い物リスト）にまとめています。このように、配列とハッシュを組み合わせることで、複雑なデータ構造を表現できます。

理解度確認クイズ

  1. Q1. 配列のインデックスは何番から始まりますか？また、最後の要素にアクセスするマイナスインデックスは何番ですか？

    回答を表示

    配列のインデックスは0番から始まります。最後の要素にアクセスするには-1を使います。-2は最後から2番目の要素、-3は最後から3番目の要素となります。

  2. Q2. 配列に要素を追加する方法を2つ挙げてください。

    回答を表示

    ①pushメソッド: fruits.push("りんご")
    ②<<演算子: fruits << "りんご"
    どちらも配列の最後に要素を追加します。最初に追加したい場合はunshiftメソッドを使います。

  3. Q3. 配列とハッシュの違いを説明してください。それぞれどんな場面で使うべきですか？

    回答を表示

    配列は「番号付きの引き出し」で、インデックス（0, 1, 2...）で管理します。順番が重要な場合や、同じ種類のデータをまとめる場合に使います。

    ハッシュは「ラベル付きの引き出し」で、キーと値のペアで管理します。ラベル名で管理したい場合や、異なる種類のデータをまとめる場合に使います。

    例: 買い物リスト（順番がある）→配列、ユーザー情報（名前、年齢、住所など）→ハッシュ
