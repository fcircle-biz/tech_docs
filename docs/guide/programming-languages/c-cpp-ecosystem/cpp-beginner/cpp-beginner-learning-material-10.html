第10章

構造体とメモリ管理

関連するデータをまとめる「構造体」について学びます。また、malloc/freeを使った動的メモリ確保の基本を理解し、プログラム実行中にメモリを柔軟に扱う方法を習得します。

この章で学ぶこと

  • 構造体（struct）を使って関連するデータをまとめる方法を理解する
  • mallocとfreeを使った動的メモリ確保と解放の仕組みを学ぶ
  • メモリリークを防ぐための正しいメモリ管理の方法を習得する

10.1 構造体とは - 関連するデータをまとめる

これまで変数や配列を使ってデータを扱ってきましたが、**実際のプログラムでは複数の種類のデータを一つのまとまりとして管理**したいことがよくあります。 たとえば「学生」のデータなら、名前（文字列）、年齢（整数）、成績（浮動小数点数）など、異なる型のデータが必要です。

このように**関連する複数のデータを一つの型としてまとめる仕組みが「構造体（struct）」**です。 構造体を使うことで、複雑なデータを整理して扱いやすくできます。

構造体の定義

構造体は以下のように定義します：

student.c 
#include <stdio.h>

// 構造体の定義
struct Student {
    char name[50];  // 名前
    int age;        // 年齢
    float score;    // 成績
};

int main() {
    // 構造体変数の宣言と初期化
    struct Student s1 = {"田中太郎", 20, 85.5};

    // メンバへのアクセス
    printf("名前: %s\n", s1.name);
    printf("年齢: %d歳\n", s1.age);
    printf("成績: %.1f点\n", s1.score);

    return 0;
}

ポイント

構造体のメンバにアクセスするには「.（ドット）」演算子を使います。s1.nameのように書きます。

10.2 typedefで構造体を使いやすくする

毎回struct Studentと書くのは面倒です。 typedefを使うことで、型名を短く書けるようになります。

typedef_example.c 
#include <stdio.h>

// typedefを使った構造体定義
typedef struct {
    char name[50];
    int age;
    float score;
} Student;  // 型名をStudentにする

int main() {
    // struct キーワード不要で宣言できる
    Student s1 = {"山田花子", 22, 90.0};

    printf("名前: %s, 年齢: %d, 成績: %.1f\n",
           s1.name, s1.age, s1.score);

    return 0;
}

重要ポイント

typedefを使うと、構造体変数を宣言するときにStudent s1のように短く書けます（structキーワード不要）。

10.3 構造体の応用 - 配列・ポインタ・関数

構造体の配列

複数の学生データを管理したい場合、**構造体の配列**を使います。

struct_array.c 
#include <stdio.h>

typedef struct {
    char name[50];
    int age;
} Student;

int main() {
    // 構造体配列の宣言と初期化
    Student students[3] = {
        {"田中太郎", 20},
        {"山田花子", 22},
        {"佐藤次郎", 21}
    };

    // 全員のデータを表示
    for (int i = 0; i < 3; i++) {
        printf("%d人目: %s (%d歳)\n",
               i+1, students[i].name, students[i].age);
    }
    return 0;
}

構造体ポインタ

構造体へのポインタを使う場合、メンバアクセスには->（アロー演算子）を使います。

struct_pointer.c 
#include <stdio.h>

typedef struct {
    char name[50];
    int age;
} Student;

int main() {
    Student s1 = {"田中太郎", 20};
    Student *p = &s1;  // ポインタで構造体を指す

    // アロー演算子でメンバアクセス
    printf("名前: %s\n", p->name);
    printf("年齢: %d歳\n", p->age);

    return 0;
}

ポインタでのメンバアクセス

p->nameは(*p).nameと同じ意味です。アロー演算子の方が読みやすいので一般的に使われます。

10.4 動的メモリ確保 - mallocとfree

これまで使ってきた変数や配列は、**コンパイル時にサイズが決まる「静的なメモリ」**でした。 しかし実際のプログラムでは、「実行中にユーザーが入力したデータ数だけメモリを確保したい」など、実行時に柔軟にメモリを確保する必要があります。

このために使うのが**malloc（メモリアロケート）**です。mallocは**ヒープ領域**という場所からメモリを確保します。

mallocの使い方

mallocはstdlib.hに含まれる関数で、指定したバイト数のメモリを確保し、その先頭アドレスを返します。

malloc_basic.c 
#include <stdio.h>
#include <stdlib.h>  // malloc, free

int main() {
    int n = 5;

    // int型5個分のメモリを確保
    int *arr = (int *)malloc(n * sizeof(int));

    // メモリ確保の成功確認
    if (arr == NULL) {
        printf("メモリ確保に失敗しました\n");
        return 1;
    }

    // 配列として使用
    for (int i = 0; i < n; i++) {
        arr[i] = i * 10;
        printf("arr[%d] = %d\n", i, arr[i]);
    }

    // メモリ解放（必須）
    free(arr);
    return 0;
}

重要: メモリ解放を忘れない

mallocで確保したメモリは、使い終わったら必ずfree関数で解放してください。解放しないと「メモリリーク」が発生し、長時間動作するプログラムではメモリ不足になる可能性があります。

callocとの違い

callocもメモリ確保の関数ですが、確保したメモリを**0で初期化**してくれるという違いがあります。

calloc_example.c 
#include <stdio.h>
#include <stdlib.h>

int main() {
    // calloc: 要素数とサイズを指定、0で初期化
    int *arr = (int *)calloc(5, sizeof(int));

    if (arr == NULL) {
        return 1;
    }

    // 確保したメモリは全て0
    for (int i = 0; i < 5; i++) {
        printf("arr[%d] = %d\n", i, arr[i]);  // 全て0
    }

    free(arr);
    return 0;
}

10.5 構造体と動的メモリ確保を組み合わせる

構造体の配列を動的に確保することで、実行時に必要な数だけデータを管理できます。

struct_malloc.c 
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
    char name[50];
    int age;
} Student;

int main() {
    int n = 3;

    // Student型の配列を動的確保
    Student *students = (Student *)malloc(n * sizeof(Student));

    if (students == NULL) {
        printf("メモリ確保失敗\n");
        return 1;
    }

    // データ設定
    strcpy(students[0].name, "田中太郎");
    students[0].age = 20;

    strcpy(students[1].name, "山田花子");
    students[1].age = 22;

    strcpy(students[2].name, "佐藤次郎");
    students[2].age = 21;

    // 表示
    for (int i = 0; i < n; i++) {
        printf("%s (%d歳)\n", students[i].name, students[i].age);
    }

    // メモリ解放
    free(students);
    return 0;
}

実用的な使い方

この方法を使えば、「学生が何人いるかユーザーに入力してもらい、その数だけメモリを確保する」といった柔軟なプログラムが作れます。

10.6 メモリリークと注意点

動的メモリ確保を使う場合、以下の点に注意しないと**メモリリーク（メモリの無駄遣い）**や**セグメンテーション違反（クラッシュ）**が発生します。

メモリリーク

mallocで確保したメモリをfreeせずにプログラムを終了すると、そのメモリは解放されずに残ります（メモリリーク）。長時間動作するプログラムでは深刻な問題になります。

二重解放

同じポインタに対してfreeを2回呼ぶと、プログラムがクラッシュします。freeした後は、ポインタにNULLを代入する習慣をつけましょう。

ダングリングポインタ

free後のポインタにアクセスすると、予期しない動作やクラッシュが起きます。free後は必ずNULLを代入しましょう。

NULLチェック

mallocはメモリ確保に失敗するとNULLを返します。使用前に必ずNULLチェックを行いましょう。

良い例と悪い例

悪い例（メモリリーク） 

int *p = malloc(100);
// 何か処理
// freeを忘れた！
return 0;  // メモリリーク

良い例（正しい解放） 

int *p = malloc(100);
// 何か処理
free(p);     // 必ず解放
p = NULL;    // 安全のためNULL代入
return 0;

実習 10-1: 本の管理プログラムを作ろう

構造体と動的メモリ確保を使って、複数の本のデータを管理するプログラムを作成してみましょう。

手順

  1. 1 Book構造体を定義する（タイトル、著者名、価格のメンバを持つ）
  2. 2 ユーザーに本の冊数を入力してもらい、その数だけ動的にメモリを確保する
  3. 3 各本のデータを入力し、全ての本の情報を表示する
  4. 4 最後に必ずメモリを解放する

理解度確認クイズ

  1. Q1. 構造体を使う主なメリットは何ですか？

    回答を表示

    構造体を使うと、名前、年齢、成績など**異なる型の関連するデータを一つのまとまりとして管理**できます。これにより、複雑なデータを整理して扱いやすくなり、コードの可読性と保守性が向上します。

  2. Q2. 構造体ポインタでメンバにアクセスする場合、どの演算子を使いますか？

    回答を表示

    アロー演算子（->）を使います。例: p->name。これは(*p).nameと同じ意味ですが、アロー演算子の方が読みやすいです。

  3. Q3. mallocとcallocの違いは何ですか？

    回答を表示

    どちらも動的メモリを確保しますが、**callocは確保したメモリを0で初期化**してくれます。mallocは初期化しないため、確保したメモリには不定値（ゴミ）が入っています。初期化が必要ならcalloc、そうでなければmallocを使います。

  4. Q4. mallocで確保したメモリを解放しないとどうなりますか？

    回答を表示

    **メモリリーク**が発生します。確保したメモリが解放されずに残り続け、長時間動作するプログラムではメモリ不足になる可能性があります。必ずfree関数で解放してください。

  5. Q5. free後のポインタにNULLを代入する理由は何ですか？

    回答を表示

    free後のポインタは**ダングリングポインタ（無効なメモリを指すポインタ）**になります。NULLを代入しておくことで、誤ってそのポインタにアクセスした場合にエラーが明確になり、デバッグしやすくなります。

  • typedefを使うことで、構造体の型名を短く書ける
  • 構造体ポインタでメンバにアクセスするには「->」演算子を使う
  • mallocで動的にメモリを確保し、freeで必ず解放する
  • メモリリークやダングリングポインタを避けるため、適切なメモリ管理が重要
