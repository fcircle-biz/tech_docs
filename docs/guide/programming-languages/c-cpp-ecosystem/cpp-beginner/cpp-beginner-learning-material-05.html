第5章

条件分岐 - プログラムに判断させる

「もし〜なら〜する」というif文を使って、条件によって動作を変えるプログラムを作ります。switch文も含め、年齢判定や成績評価など実践的な例で学習します。

この章で学ぶこと

  • if文を使った条件分岐の基本を理解する
  • if-else、if-else if-elseを使った複雑な条件分岐ができるようになる
  • switch文を使った多方向分岐を習得する
  • 実践的な例（年齢判定、成績評価など）でプログラミングできるようになる

5.1 条件分岐とは何か

プログラムは通常、上から下へ順番に実行されます。しかし、実際の世界では「もし雨が降っていたら傘を持つ」「もし晴れていたら外で遊ぶ」のように、状況に応じて異なる行動を選ぶ必要があります。

プログラムでもこの考え方を実現するのが「条件分岐」です。条件分岐を使うと、変数の値や計算結果によってプログラムの動作を変えることができます。

flowchart TD Start["プログラム開始"] --> Input["変数の値をチェック"] Input --> Condition{"条件を満たす?"} Condition -->|はい| Action1["処理A を実行"] Condition -->|いいえ| Action2["処理B を実行"] Action1 --> End["次の処理へ"] Action2 --> End

日常生活との対比

自動販売機を想像してください。お金を入れた金額が商品の値段以上なら商品が出てきて、足りなければ「お金が足りません」と表示されます。これが条件分岐です。

5.2 if文の基本

C言語で条件分岐を実現する最も基本的な構文がif文です。if文は「もし〜なら〜する」という意味を持ちます。

if文の基本構文

C 
if (条件式) {
    // 条件がtrueのときに実行される処理
}

実例: 年齢判定

C 
#include <stdio.h>

int main() {
    int age = 20;  // 年齢を保存する変数

    // もし年齢が18歳以上なら
    if (age >= 18) {
        printf("あなたは成人です。\n");
    }

    return 0;
}

重要ポイント

条件式の結果がtrue（真）のとき、{}の中の処理が実行されます。false（偽）のときは何も実行されず、次の行へ進みます。

5.3 if-else文 - 2つの選択肢

条件が満たされないときに別の処理を実行したい場合は、else（そうでなければ）を使います。

if-else文の構文

C 
if (条件式) {
    // 条件がtrueのときの処理
} else {
    // 条件がfalseのときの処理
}

実例: 成年判定

C 
#include <stdio.h>

int main() {
    int age = 15;  // 年齢を保存

    if (age >= 18) {
        // 18歳以上の場合
        printf("あなたは成人です。\n");
    } else {
        // 18歳未満の場合
        printf("あなたは未成年です。\n");
    }

    return 0;
}

5.4 if-else if-else文 - 複数の選択肢

3つ以上の選択肢がある場合は、else ifを使って条件を追加できます。

else ifの構文

C 
if (条件式1) {
    // 条件1がtrueのときの処理
} else if (条件式2) {
    // 条件1がfalseで、条件2がtrueのときの処理
} else {
    // どの条件もfalseのときの処理
}

実例: 成績評価

C 
#include <stdio.h>

int main() {
    int score = 75;  // テストの点数

    if (score >= 90) {
        // 90点以上
        printf("評価: A（優秀）\n");
    } else if (score >= 70) {
        // 70点以上90点未満
        printf("評価: B（良好）\n");
    } else if (score >= 50) {
        // 50点以上70点未満
        printf("評価: C（合格）\n");
    } else {
        // 50点未満
        printf("評価: D（不合格）\n");
    }

    return 0;
}

条件の評価順序

if-else if-else文は上から順に条件を評価します。どれか1つの条件が真になったら、その処理を実行して終了します。残りの条件は評価されません。

実習: 温度判定プログラムを作ろう

気温に応じてメッセージを表示するプログラムを作成しましょう。

条件:

  • • 30度以上: 「猛暑です。水分補給を忘れずに！」
  • • 20度以上30度未満: 「過ごしやすい気温です。」
  • • 10度以上20度未満: 「少し肌寒いです。」
  • • 10度未満: 「寒いです。暖かくしてください。」
解答例を表示
C 
#include <stdio.h>

int main() {
    int temp = 28;  // 気温（摂氏）

    if (temp >= 30) {
        printf("猛暑です。水分補給を忘れずに！\n");
    } else if (temp >= 20) {
        printf("過ごしやすい気温です。\n");
    } else if (temp >= 10) {
        printf("少し肌寒いです。\n");
    } else {
        printf("寒いです。暖かくしてください。\n");
    }

    return 0;
}

5.5 ネストされた条件分岐

if文の中に別のif文を入れることを「ネスト（入れ子）」と呼びます。複雑な条件判定が必要なときに使います。

実例: 年齢と会員判定

C 
#include <stdio.h>

int main() {
    int age = 25;          // 年齢
    int is_member = 1;     // 会員かどうか（1=会員, 0=非会員）

    if (age >= 18) {
        // 成人の場合
        if (is_member == 1) {
            // さらに会員の場合
            printf("割引価格: 800円\n");
        } else {
            // 会員でない場合
            printf("通常価格: 1000円\n");
        }
    } else {
        // 未成年の場合
        printf("学生価格: 500円\n");
    }

    return 0;
}

注意

ネストが深くなりすぎるとコードが読みにくくなります。3段階以上のネストは避け、論理演算子（&& や ||）を使って条件を組み合わせることを検討しましょう。

5.6 switch文 - 多方向分岐

1つの変数の値によって処理を分岐させる場合、switch文を使うとコードがすっきりします。

switch文の構文

C 
switch (変数) {
    case 値1:
        // 変数が値1のときの処理
        break;
    case 値2:
        // 変数が値2のときの処理
        break;
    default:
        // どのcaseにも該当しないときの処理
        break;
}

実例: 曜日判定

C 
#include <stdio.h>

int main() {
    int day = 3;  // 曜日（1=月, 2=火, ..., 7=日）

    switch (day) {
        case 1:
            printf("今日は月曜日です。\n");
            break;
        case 2:
            printf("今日は火曜日です。\n");
            break;
        case 3:
            printf("今日は水曜日です。\n");
            break;
        case 6:
        case 7:
            // case 6とcase 7で同じ処理
            printf("今日は週末です！\n");
            break;
        default:
            printf("無効な曜日です。\n");
            break;
    }

    return 0;
}

breakの重要性

各caseの最後にはbreak;を書きます。これがないと次のcaseの処理も実行されてしまいます（フォールスルー）。意図的にフォールスルーを使う場合もありますが、基本的にはbreakを忘れないようにしましょう。

実習: 簡単な計算機メニュー

switch文を使って、四則演算を選択できる計算機プログラムを作りましょう。

条件:

  • • 1を選ぶと足し算（10 + 5 = 15を表示）
  • • 2を選ぶと引き算（10 - 5 = 5を表示）
  • • 3を選ぶと掛け算（10 * 5 = 50を表示）
  • • 4を選ぶと割り算（10 / 5 = 2を表示）
  • • それ以外は「無効な選択です」と表示
解答例を表示
C 
#include <stdio.h>

int main() {
    int choice = 3;  // 選択肢
    int a = 10, b = 5;  // 計算する数値

    switch (choice) {
        case 1:
            printf("%d + %d = %d\n", a, b, a + b);
            break;
        case 2:
            printf("%d - %d = %d\n", a, b, a - b);
            break;
        case 3:
            printf("%d * %d = %d\n", a, b, a * b);
            break;
        case 4:
            printf("%d / %d = %d\n", a, b, a / b);
            break;
        default:
            printf("無効な選択です。\n");
            break;
    }

    return 0;
}

5.7 if文とswitch文の使い分け

if文とswitch文はどちらも条件分岐を実現しますが、状況によって使い分けることで読みやすいコードになります。

if文が適している場合

  • 範囲指定の条件（x >= 10 など）
  • 複数の変数を組み合わせた条件
  • 小数や文字列の比較

switch文が適している場合

  • 1つの変数の値による分岐
  • 選択肢が多い（3つ以上）
  • 整数や文字での完全一致判定

理解度確認クイズ

  1. Q1. 次のコードの出力結果は何ですか？

    int x = 15;
if (x > 20) {
    printf("A\n");
} else if (x > 10) {
    printf("B\n");
} else {
    printf("C\n");
}
    回答を表示

    答え: B
    理由: x = 15なので、最初の条件（x > 20）はfalseですが、2番目の条件（x > 10）がtrueなので「B」が出力されます。

  2. Q2. switch文で各caseの最後にbreakを書かないとどうなりますか？

    回答を表示

    breakを書かないと、次のcaseの処理も続けて実行されてしまいます（フォールスルー）。これは意図しない動作の原因になるため、基本的には各caseの最後にbreakを書くべきです。

  3. Q3. if文とswitch文の使い分けの基準を説明してください。

    回答を表示

    if文が適している場合: 範囲指定（x >= 10など）、複数の変数を組み合わせた条件、小数や文字列の比較。

    switch文が適している場合: 1つの変数の値による分岐、選択肢が多い（3つ以上）、整数や文字での完全一致判定。

  4. Q4. 次のプログラムのバグを見つけてください。

    int score = 85;
if (score >= 90)
    printf("優秀\n");
    printf("よくできました\n");  // これは常に実行される！
else
    printf("頑張りましょう\n");
    回答を表示

    バグ: if文に{}（波括弧）がないため、「よくできました」は常に実行されます。その後のelseは構文エラーになります。

    修正方法: if文の処理を{}で囲むか、1行だけにする必要があります。

この章のまとめ

  • if文は「もし〜なら〜する」という条件分岐を実現する基本構文
  • if-else文で2つの選択肢、if-else if-else文で複数の選択肢を扱える
  • switch文は1つの変数の値による多方向分岐に適している
  • switch文の各caseの最後にはbreakを忘れずに書く
  • 条件分岐を使うことで、状況に応じた柔軟なプログラムが作れる
