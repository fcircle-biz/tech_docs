ポインタの基礎

C/C++の核心であり、多くの初心者がつまずく「ポインタ」を丁寧に解説します。メモリアドレスとは何か、ポインタ変数の使い方、配列との関係を、住所と建物のたとえでわかりやすく説明します。

この章で学ぶこと

  • メモリとアドレスの概念を理解し、コンピュータがデータをどこに保存しているかを知る
  • ポインタ変数の宣言と初期化、&演算子と*演算子の使い方を習得する
  • ポインタと配列の密接な関係、ポインタを使った配列操作ができるようになる

はじめに

ポインタはC/C++の中でも特に難しいと言われる部分です。しかし、住所と建物のたとえを使えば理解しやすくなります。焦らず、何度も読み返し、実際にコードを書いて実験しながら学びましょう。一度理解できれば、C/C++の強力な機能を使いこなせるようになります。

9.1 メモリとアドレスの基本

メモリとは

コンピュータのメモリ（RAM）は、プログラムが実行中にデータを一時的に保存する場所です。 メモリは、たくさんの小さな「箱」が並んでいるようなもので、それぞれの箱に1バイト（8ビット）のデータを入れることができます。

アドレスとは

それぞれの箱（メモリの場所）にはアドレスと呼ばれる番号が振られています。 これは街の住所と同じで、「このデータはメモリのどこにあるか」を示す識別番号です。

データ: 10"] B["アドレス 0x1004
データ: 20"] C["アドレス 0x1008
データ: 30"] end

たとえ話

住所と建物のたとえで考えましょう。
- 建物 = データ（変数の中身）
- 住所 = アドレス（メモリ上の場所）
「東京都渋谷区〇〇1-2-3」という住所があれば、その場所にある建物を見つけられますね。同じように、アドレスがわかればメモリ上のデータを見つけられます。

変数とメモリアドレス

変数を宣言すると、コンピュータはメモリ上に適切なサイズの領域を確保し、そこにデータを保存します。 私たちは変数名でアクセスしますが、内部的にはアドレスを使って管理されています。

address_example.c 
#include <stdio.h>

int main() {
    int num = 42;  // int型変数を宣言・初期化

    // 変数の値を表示
    printf("変数numの値: %d\n", num);

    // 変数のアドレスを表示（&演算子を使う）
    printf("変数numのアドレス: %p\n", (void*)&num);

    return 0;
}

実行結果

変数numの値: 42
変数numのアドレス: 0x7ffe5c8b4a3c

アドレスは実行するたびに変わる可能性があります。重要なのは「変数には値だけでなく、メモリ上の場所（アドレス）もある」ということです。

9.2 ポインタとは何か

ポインタ（pointer）は、「他の変数のアドレスを保存する変数」です。 つまり、住所を書いたメモのようなものです。

通常の変数は「データそのもの」を保存しますが、ポインタ変数は「データがある場所（アドレス）」を保存します。

flowchart LR A["通常の変数 num
値: 42"] B["ポインタ変数 ptr
値: 0x1000（numのアドレス）"] B -.->|参照先| A

住所のたとえ

友達に「私の家に遊びに来て」と伝えるとき、
① 家そのものを持っていく（無理！） → 通常の変数
② 住所を書いたメモを渡す → ポインタ
ポインタは「住所メモ」なので、実際のデータを渡さなくても、その場所を教えることができます。

9.3 ポインタ変数の宣言と使い方

ポインタ変数の宣言

ポインタ変数を宣言するには、データ型の後ろに*（アスタリスク）を付けます。

pointer_declaration.c 
int num = 42;      // 通常のint型変数
int *ptr;          // int型のポインタ変数（*が付く）

ptr = #        // numのアドレスをptrに代入（&演算子）

重要な演算子

&演算子（アドレス演算子）

変数のアドレスを取得する

int num = 10;
int *ptr = #  // numのアドレスを取得してptrに代入

*演算子（間接参照演算子）

ポインタが指す先の値を取得する

int value = *ptr;  // ptrが指す先の値を取得（10が入る）

ポインタの完全な例

pointer_basic.c 
#include <stdio.h>

int main() {
    int num = 100;      // 通常の変数
    int *ptr = #    // ポインタ変数（numのアドレスを保存）

    // 値とアドレスを表示
    printf("numの値: %d\n", num);
    printf("numのアドレス: %p\n", (void*)&num);
    printf("\n");

    printf("ptrの値（numのアドレス）: %p\n", (void*)ptr);
    printf("ptrが指す先の値: %d\n", *ptr);  // 間接参照
    printf("\n");

    // ポインタを使って値を変更
    *ptr = 200;  // numの値が200になる
    printf("変更後のnumの値: %d\n", num);

    return 0;
}

実行結果

numの値: 100
numのアドレス: 0x7ffe5c8b4a3c

ptrの値（numのアドレス）: 0x7ffe5c8b4a3c
ptrが指す先の値: 100

変更後のnumの値: 200

*ptr = 200; とすることで、ポインタ経由でnumの値を変更できます。これがポインタの強力な機能です。

よくある間違い

間違い: int *ptr = num; ← 値を直接代入してはいけない！
正しい: int *ptr = # ← アドレスを代入する（&を付ける）

9.4 ポインタと配列の関係

C/C++では、配列名はその配列の先頭要素へのポインタとして扱われます。 つまり、配列とポインタは密接に関係しています。

pointer_array.c 
#include <stdio.h>

int main() {
    int arr[5] = {10, 20, 30, 40, 50};  // 配列
    int *ptr = arr;  // 配列名はポインタとして使える

    // 配列を普通に表示
    printf("arr[0] = %d\n", arr[0]);
    printf("arr[1] = %d\n", arr[1]);
    printf("\n");

    // ポインタで表示（同じ結果）
    printf("*ptr = %d\n", *ptr);       // arr[0]と同じ
    printf("*(ptr+1) = %d\n", *(ptr+1));  // arr[1]と同じ
    printf("*(ptr+2) = %d\n", *(ptr+2));  // arr[2]と同じ

    return 0;
}

実行結果

arr[0] = 10
arr[1] = 20

*ptr = 10
*(ptr+1) = 20
*(ptr+2) = 30

30"] end ptr["ポインタ ptr"] -.->|指す| A0 ptr1["ptr+1"] -.->|指す| A1 ptr2["ptr+2"] -.->|指す| A2

ポイント

arr[i] と *(arr + i) は同じ意味です。 配列とポインタは裏側で同じ仕組みで動いています。

ポインタで配列を走査する

pointer_loop.c 
#include <stdio.h>

int main() {
    int arr[5] = {10, 20, 30, 40, 50};
    int *ptr = arr;  // ポインタに配列の先頭アドレスを代入

    // ポインタを使って配列を走査
    for (int i = 0; i < 5; i++) {
        printf("arr[%d] = %d (アドレス: %p)\n",
               i, *(ptr + i), (void*)(ptr + i));
    }

    return 0;
}

9.5 よくある間違いと注意点

トラブルシューティング

未初期化ポインタを使ってしまった

原因: ポインタを宣言しただけで、どこも指していない状態で使用した

int *ptr;      // 初期化していない
*ptr = 100;    // 危険！どこを指しているか不明

解決方法: 必ず有効なアドレスを代入するか、NULLで初期化する

int *ptr = NULL;  // NULLで初期化
int num = 10;
ptr = #       // 有効なアドレスを代入してから使う
&と*の使い分けがわからない

覚え方:

  • & = 「住所を教えて」演算子（アドレス取得）
  • * = 「その住所の中身を見せて」演算子（間接参照）
int num = 10;
int *ptr = #  // numの住所を取得してptrに保存
int value = *ptr;  // ptrが指す住所の中身を取得（10）
ポインタの型が一致していない

問題: int型の変数に対してdouble型のポインタを使おうとした

int num = 10;
double *ptr = #  // 警告！型が一致しない

解決方法: ポインタの型は、指す変数の型と一致させる

int num = 10;
int *ptr = #     // 正しい！両方int型

安全なポインタの使い方

  • ポインタは必ず初期化する（NULLまたは有効なアドレス）
  • 使う前にNULLチェックを行う
  • ポインタの型と変数の型を一致させる
  • 配列の範囲外を指さないよう注意する

実習 9-1: ポインタで2つの変数を交換する

ポインタを使って、2つの変数の値を入れ替える関数を作りましょう。

手順

  1. 1 swap関数を作成し、int型のポインタを2つ受け取るようにする
  2. 2 一時変数を使って、2つのポインタが指す値を交換する
  3. 3 main関数から呼び出して、交換前後の値を表示する
解答例を表示
swap_example.c 
#include <stdio.h>

// 2つの整数をポインタで交換する関数
void swap(int *a, int *b) {
    int temp = *a;  // aが指す値を一時保存
    *a = *b;        // aにbの値を代入
    *b = temp;      // bに一時保存した値を代入
}

int main() {
    int x = 10, y = 20;

    printf("交換前: x = %d, y = %d\n", x, y);

    swap(&x, &y);  // xとyのアドレスを渡す

    printf("交換後: x = %d, y = %d\n", x, y);

    return 0;
}

理解度確認クイズ

  1. Q1. ポインタとは何ですか？

    回答を表示

    ポインタは「他の変数のアドレス（メモリ上の場所）を保存する変数」です。住所メモのように、データそのものではなく、データがある場所を記録します。

  2. Q2. &演算子と*演算子の違いを説明してください。

    回答を表示

    &演算子（アドレス演算子）: 変数のアドレスを取得する。&num = 「numの住所」
    *演算子（間接参照演算子）: ポインタが指す先の値を取得・変更する。*ptr = 「ptrが指す場所の中身」

  3. Q3. 配列名とポインタの関係を説明してください。

    回答を表示

    C/C++では、配列名はその配列の先頭要素へのポインタとして扱われます。したがって、arr[i] と *(arr + i) は同じ意味です。ポインタを使って配列の要素にアクセスできます。

  4. Q4. ポインタを使う際の注意点を3つ挙げてください。

    回答を表示

    ① ポインタは必ず初期化する（NULLまたは有効なアドレス）
    ② ポインタの型と変数の型を一致させる
    ③ 配列の範囲外を指さないよう注意する
    ※ その他、「使う前にNULLチェックを行う」なども正解です。

この章のまとめ

  • メモリには番地（アドレス）があり、変数はメモリ上の特定の場所にデータを保存する
  • ポインタは「他の変数のアドレスを保存する変数」で、住所メモのような役割を果たす
  • &演算子でアドレスを取得、*演算子でポインタが指す先の値にアクセスできる
  • 配列名はポインタとして扱われ、ポインタを使って配列を操作できる
  • ポインタは必ず初期化し、型を一致させ、範囲外アクセスに注意する
