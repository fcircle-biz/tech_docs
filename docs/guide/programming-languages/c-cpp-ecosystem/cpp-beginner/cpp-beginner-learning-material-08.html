関数の基本

よく使う処理をまとめて名前をつける「関数」について学びます。引数と戻り値の概念、関数の宣言と定義、ローカル変数とスコープを理解します。レシピのように一度書けば何度でも使える便利な仕組みを習得します。

この章で学ぶこと

  • 関数とは何か、なぜ関数が必要なのかを理解する
  • 関数の宣言と定義、引数と戻り値の使い方を学ぶ
  • ローカル変数とスコープの概念を理解する

8.1 関数とは何か

プログラミングでの関数（function）とは、特定の処理をまとめて名前をつけたものです。 料理でたとえると、「カレーの作り方」というレシピのようなものです。一度レシピを作っておけば、何度でも「カレーを作る」という処理を実行できます。

関数を使うと、同じコードを何度も書く必要がなくなり、プログラムが読みやすく、管理しやすくなります。 また、バグが見つかったときも、関数の中身だけを修正すればよいので、メンテナンスが楽になります。

ポイント

実は、これまでに何度も使ってきた printf() も関数です。 誰かが作ってくれた「画面に文字を表示する」という処理を、私たちは関数として利用していたのです。

8.2 関数の基本構造

C言語の関数は、以下の4つの要素から構成されています：

関数の基本構造 
// 戻り値の型 関数名（引数リスト）
int add(int a, int b) {
    // 関数本体（処理内容）
    int result = a + b;
    return result;  // 戻り値を返す
}

1. 戻り値の型

関数が処理結果として返すデータの型を指定します。上の例では int 型の値を返します。

2. 関数名

関数を識別するための名前です。分かりやすい名前をつけることが重要です。上の例では add です。

3. 引数（パラメータ）

関数に渡すデータです。上の例では int a, int b という2つの整数を受け取ります。

4. 関数本体

{} で囲まれた部分で、実際の処理内容を記述します。

重要ポイント

return 文は、関数の処理結果を呼び出し元に返すために使います。 return が実行されると、その時点で関数の実行は終了します。

8.3 関数の宣言と定義

C言語では、関数を使う前に「こんな関数がありますよ」とコンパイラに伝える必要があります。これを関数宣言（プロトタイプ宣言）といいます。

example.c 
#include <stdio.h>

// 関数宣言（プロトタイプ）
int add(int a, int b);

int main() {
    int x = 10;
    int y = 20;
    int sum = add(x, y);  // 関数呼び出し
    printf("合計: %d\n", sum);
    return 0;
}

// 関数定義（実装）
int add(int a, int b) {
    return a + b;
}
flowchart TD A["main関数開始"] --> B["変数 x = 10, y = 20"] B --> C["add関数を呼び出し"] C --> D["add関数内で a + b を計算"] D --> E["結果をmain関数に返す"] E --> F["sum に結果を代入"] F --> G["printf で表示"] G --> H["main関数終了"]

ヒント

関数宣言は通常、main() 関数の前に書きます。 こうすることで、main() の中で関数を使えるようになります。

8.4 引数と戻り値

引数（ひきすう）

引数は、関数に渡すデータです。料理でいえば「材料」のようなものです。 引数は複数指定でき、それぞれの型と名前を , で区切って書きます。

複数の引数を持つ関数 
// 3つの整数の平均を計算する関数
double average(int a, int b, int c) {
    double sum = a + b + c;
    return sum / 3.0;  // 実数で割る
}

戻り値（もどりち）

戻り値は、関数の処理結果として返されるデータです。料理でいえば「完成した料理」です。 return 文で値を返します。

戻り値の例 
// 円の面積を計算する関数
double circle_area(double radius) {
    return 3.14159 * radius * radius;
}

int main() {
    double r = 5.0;
    double area = circle_area(r);  // 戻り値を変数に代入
    printf("半径 %.1f の円の面積: %.2f\n", r, area);
    return 0;
}

戻り値が不要な関数（void型）

関数によっては、戻り値が不要な場合もあります。そのような場合は void 型を使います。

void型の関数 
// 挨拶を表示する関数（戻り値なし）
void greet(char name[]) {
    printf("こんにちは、%sさん！\n", name);
    // return文は不要（書いてもreturn;のみ）
}

重要ポイント

引数も戻り値も不要な関数は、void func(void) と書きます。

8.5 ローカル変数とスコープ

関数の中で宣言された変数をローカル変数といいます。 ローカル変数は、その関数の中でしか使えません。この「変数が有効な範囲」をスコープ（scope）といいます。

スコープの例 
#include <stdio.h>

void funcA() {
    int x = 10;  // funcA のローカル変数
    printf("funcA の x: %d\n", x);
}

void funcB() {
    int x = 20;  // funcB のローカル変数（funcAのxとは別）
    printf("funcB の x: %d\n", x);
}

int main() {
    funcA();  // 出力: funcA の x: 10
    funcB();  // 出力: funcB の x: 20
    return 0;
}
flowchart TD subgraph main["main関数のスコープ"] M1["main内の変数"] end subgraph funcA["funcA関数のスコープ"] A1["funcA内の変数 x = 10"] end subgraph funcB["funcB関数のスコープ"] B1["funcB内の変数 x = 20"] end M1 -.-> A1 M1 -.-> B1

ポイント

関数ごとにスコープが独立しているため、同じ名前の変数を別の関数で使っても問題ありません。 これにより、複数の人が共同でプログラムを書くときも、変数名の衝突を気にせずに済みます。

注意

ローカル変数は関数が終了すると消えてしまいます。次回その関数を呼び出したときは、また新しく作られます。

8.6 関数を使う利点

1. コードの再利用

一度作った関数は何度でも呼び出せます。同じコードを繰り返し書く必要がなくなります。

2. 可読性の向上

処理に名前をつけることで、プログラムが何をしているのか理解しやすくなります。

3. 保守性の向上

バグ修正や機能追加が必要なとき、関数の中身だけを修正すればよいので、メンテナンスが楽になります。

4. 分割統治

複雑な問題を小さな関数に分割することで、一つひとつの処理が単純になり、開発しやすくなります。

実習 8-1: 関数を作って使ってみよう

2つの整数の最大値を返す関数を作成し、実際に使ってみましょう。

手順

  1. 1 新しいファイル max_function.c を作成します
  2. 2 以下のコードを入力してください
max_function.c 
#include <stdio.h>

// 関数宣言
int max(int a, int b);

int main() {
    int num1 = 15;
    int num2 = 23;
    int result = max(num1, num2);
    printf("%d と %d の最大値: %d\n", num1, num2, result);
    return 0;
}

// 関数定義
int max(int a, int b) {
    if (a > b) {
        return a;
    } else {
        return b;
    }
}
  1. 3 コンパイルして実行します
コンパイルと実行 
gcc max_function.c -o max_function
./max_function

期待される結果 

15 と 23 の最大値: 23

理解度確認クイズ

  1. Q1. 関数を使う利点を3つ挙げてください。

    回答を表示

    ① コードの再利用：一度作った関数は何度でも使える
    ② 可読性の向上：処理に名前をつけることで分かりやすくなる
    ③ 保守性の向上：修正が必要なとき、関数の中身だけを直せばよい
    （④ 分割統治：複雑な問題を小さな部分に分けられる、も正解）

  2. Q2. 関数の4つの構成要素を答えてください。

    回答を表示

    ① 戻り値の型
    ② 関数名
    ③ 引数（パラメータ）
    ④ 関数本体（処理内容）

  3. Q3. ローカル変数とは何ですか？その特徴を説明してください。

    回答を表示

    ローカル変数とは、関数の中で宣言された変数のことです。特徴は以下の通りです：
    - その関数の中でしか使えない（スコープが限定される）
    - 関数が終了すると消える
    - 別の関数で同じ名前の変数を使っても問題ない

  4. Q4. 戻り値が不要な関数はどのように定義しますか？

    回答を表示

    戻り値が不要な関数は、戻り値の型に void を指定します。
    例: void greet(char name[])
    このような関数では return 文は不要です（書く場合は return; のみ）。

この章のまとめ

  • 関数とは、特定の処理をまとめて名前をつけたもので、何度でも再利用できる
  • 関数は「戻り値の型」「関数名」「引数」「関数本体」の4つの要素で構成される
  • 関数宣言（プロトタイプ）を書くことで、main関数の前に関数を定義できる
  • ローカル変数はその関数の中でのみ有効で、関数が終了すると消える
  • 関数を使うことで、コードの再利用性、可読性、保守性が向上する
