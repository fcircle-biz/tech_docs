クラスとオブジェクト指向の基礎

C++の核心である「オブジェクト指向」の入り口として、クラス（設計図）とオブジェクト（実体）の概念を学びます。コンストラクタ、デストラクタ、メンバ変数とメンバ関数を理解します。

この章で学ぶこと

  • オブジェクト指向プログラミングの基本概念を理解する
  • クラスの定義方法と、クラスからオブジェクトを生成する方法を学ぶ
  • コンストラクタとデストラクタの役割を理解する
  • メンバ変数とメンバ関数を使ってデータと機能をまとめる方法を習得する

12.1 オブジェクト指向プログラミングとは

オブジェクト指向プログラミング（OOP: Object-Oriented Programming）は、プログラムを「モノ（オブジェクト）の集まり」として考える方法です。

例えば、車を考えてみましょう。車には「色」「速度」「燃料」などのデータ（属性）と、「アクセルを踏む」「ブレーキをかける」「給油する」などの動作（機能）があります。 オブジェクト指向では、このように「データ」と「それを操作する機能」をひとつにまとめて扱います。

ポイント

C言語では、データ（構造体）と機能（関数）は別々に管理していました。C++のクラスは、これらを一体化して管理できる強力な仕組みです。

クラスとオブジェクトの違い

クラスは「設計図」、オブジェクトは「実体」です。

クラス（設計図）

「車」という概念の設計図。どんなデータを持ち、どんな機能があるかを定義します。たとえ話でいえば「たい焼きの型」です。

オブジェクト（実体）

クラスから作られた実際のモノ。「赤い車」「青い車」など、具体的な実体です。たとえ話でいえば「焼き上がったたい焼き」です。

12.2 クラスの定義

C++でクラスを定義するには、classキーワードを使います。

基本的なクラスの書き方

car.cpp 
#include <iostream>
#include <string>
using namespace std;

// Carクラスの定義
class Car {
public:
    // メンバ変数（データ）
    string color;  // 色
    int speed;     // 速度

    // メンバ関数（機能）
    void showInfo() {
        cout << "色: " << color << endl;
        cout << "速度: " << speed << " km/h" << endl;
    }
};

重要ポイント

クラス定義の最後にはセミコロン（;）が必要です！忘れやすいので注意してください。

オブジェクトの作成と使用

main.cpp 
int main() {
    // Carクラスからcar1オブジェクトを生成
    Car car1;
    car1.color = "赤";
    car1.speed = 60;

    // メンバ関数を呼び出す
    car1.showInfo();

    return 0;
}

実行結果

色: 赤
速度: 60 km/h

12.3 アクセス指定子（public と private）

クラスのメンバ（変数や関数）には、アクセス権限を設定できます。これにより、外部から自由に触れる部分と、触れない部分を分けることができます。

public（公開）

クラスの外からアクセスできます。ユーザーが自由に使える「公開インターフェース」です。

private（非公開）

クラスの外からはアクセスできません。クラス内部でのみ使える「内部データ」です。

アクセス指定子を使った例

bank_account.cpp 
#include <iostream>
using namespace std;

class BankAccount {
private:
    // 残高は外部から直接変更できないようにする
    int balance;

public:
    // 初期残高を設定
    void setBalance(int amount) {
        balance = amount;
    }

    // 預金
    void deposit(int amount) {
        balance += amount;
        cout << amount << "円を預金しました" << endl;
    }

    // 残高表示
    void showBalance() {
        cout << "現在の残高: " << balance << "円" << endl;
    }
};

なぜprivateを使うのか？

銀行残高を外部から直接書き換えられたら困りますよね？privateにすることで、決められた方法（deposit関数など）を通してのみ変更できるようになり、データの安全性が高まります。これをカプセル化といいます。

12.4 コンストラクタとデストラクタ

オブジェクトが「生まれる時」と「消える時」に自動的に実行される特別な関数があります。

コンストラクタ（初期化）

コンストラクタは、オブジェクトが作られた瞬間に自動的に呼ばれる関数です。メンバ変数の初期化によく使われます。

student.cpp 
#include <iostream>
#include <string>
using namespace std;

class Student {
private:
    string name;
    int age;

public:
    // コンストラクタ（クラス名と同じ名前、戻り値なし）
    Student(string n, int a) {
        name = n;
        age = a;
        cout << "生徒 " << name << " が登録されました" << endl;
    }

    void showInfo() {
        cout << "名前: " << name << endl;
        cout << "年齢: " << age << "歳" << endl;
    }
};
使用例 
int main() {
    // オブジェクト生成時に自動的にコンストラクタが呼ばれる
    Student s1("太郎", 16);
    s1.showInfo();

    return 0;
}

デストラクタ（終了処理）

デストラクタは、オブジェクトが消える瞬間に自動的に呼ばれる関数です。メモリ解放やファイルクローズなどの後片付けに使われます。

destructor_example.cpp 
class Student {
private:
    string name;

public:
    // コンストラクタ
    Student(string n) {
        name = n;
        cout << name << " 登録" << endl;
    }

    // デストラクタ（~クラス名）
    ~Student() {
        cout << name << " 削除" << endl;
    }
};
flowchart LR A["オブジェクト生成"] --> B["コンストラクタ呼び出し"] B --> C["オブジェクトを使用"] C --> D["オブジェクト破棄"] D --> E["デストラクタ呼び出し"]

重要ポイント

コンストラクタはクラス名と同じ名前で、デストラクタは~クラス名です。どちらも戻り値はありません。

実習 12-1: 本クラスを作ろう

書籍を管理するBookクラスを作成して、オブジェクト指向の理解を深めましょう。

要件

  • メンバ変数: タイトル（string）、著者（string）、価格（int）
  • コンストラクタで3つの値を受け取り、初期化する
  • showInfo()関数で書籍情報を表示する
解答例を見る 
#include <iostream>
#include <string>
using namespace std;

class Book {
private:
    string title;
    string author;
    int price;

public:
    // コンストラクタ
    Book(string t, string a, int p) {
        title = t;
        author = a;
        price = p;
    }

    // 書籍情報表示
    void showInfo() {
        cout << "タイトル: " << title << endl;
        cout << "著者: " << author << endl;
        cout << "価格: " << price << "円" << endl;
    }
};

int main() {
    Book book1("C++入門", "山田太郎", 2500);
    book1.showInfo();

    return 0;
}

12.5 クラスを使う利点

1. カプセル化

データと機能をひとつにまとめて管理できます。内部データを外部から勝手に変更されないように守ることができます。

2. 再利用性

一度クラスを作れば、何度でも同じ設計図からオブジェクトを生成できます。コードの重複を減らせます。

3. 保守性

クラス内部の実装を変更しても、外部に影響を与えにくい構造にできます。バグの混入を防ぎやすくなります。

4. 直感的な設計

現実世界の「モノ」をそのままプログラムで表現できるため、設計が直感的で分かりやすくなります。

理解度確認クイズ

  1. Q1. クラスとオブジェクトの違いを説明してください。

    回答を表示

    クラスは「設計図」で、オブジェクトは「実体」です。クラスはデータと機能の構造を定義し、オブジェクトはその設計図から実際に作られた具体的なモノです。たとえば、「車」というクラスから「赤い車」「青い車」などのオブジェクトを作ることができます。

  2. Q2. publicとprivateの違いは何ですか？

    回答を表示

    publicはクラスの外からアクセスできる「公開」メンバ、privateはクラス内部でのみアクセスできる「非公開」メンバです。privateを使うことで、重要なデータを外部から勝手に変更されないように保護できます（カプセル化）。

  3. Q3. コンストラクタの役割は何ですか？

    回答を表示

    コンストラクタは、オブジェクトが生成される瞬間に自動的に呼ばれる特別な関数です。主にメンバ変数の初期化に使われます。クラス名と同じ名前で、戻り値はありません。

  4. Q4. クラスを使う利点を2つ挙げてください。

    回答を表示

    ① カプセル化: データと機能をまとめて管理し、内部データを保護できる
    ② 再利用性: 一度作ったクラスから何度でもオブジェクトを生成でき、コードの重複を減らせる
    ※ その他、保守性、直感的な設計なども正解です。

この章のまとめ

  • オブジェクト指向は「モノ」を中心にプログラムを設計する考え方である
  • クラスは設計図、オブジェクトは実体である
  • publicは外部公開、privateは内部限定のアクセス権限を表す
  • コンストラクタは生成時、デストラクタは破棄時に自動実行される
  • クラスを使うことで、カプセル化、再利用性、保守性が向上する
