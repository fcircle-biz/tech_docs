第2章

最初のプログラム - Hello World

伝統的な「Hello, World!」プログラムを作成し、コンパイルと実行の仕組みを理解します。ソースコードが実行可能なプログラムになるまでの流れを、料理のレシピにたとえて解説します。

この章で学ぶこと

  • 伝統的な「Hello, World!」プログラムの意味と役割を理解する
  • C言語のソースコードを書き、コンパイルして実行できるようになる
  • コンパイルエラーの読み方と基本的な対処法を身につける

2.1 Hello Worldプログラムとは

「Hello, World!」は、プログラミング学習の最初に書く伝統的なプログラムです。 画面に「Hello, World!」という文字を表示するだけのシンプルなプログラムですが、その中にはプログラミングの基本要素がすべて詰まっています。

この「Hello, World!」という伝統は、1978年に出版された伝説的な本「プログラミング言語C」（通称K&R本）で紹介されたのが始まりです。 以来、ほぼすべてのプログラミング言語で、学習の最初のステップとして採用されています。

なぜHello Worldから始めるのか

① 環境が正しく動作するか確認できる
② プログラムの基本構造を理解できる
③ コンパイル→実行の流れを体験できる
④ 成功体験を積んでモチベーションが上がる

2.2 最初のC言語プログラムを書く

ステップ1: ファイルを作成する

まず、VSCodeで新しいフォルダを作成し、その中にhello.cという名前のファイルを作ります。 .cという拡張子は「C言語のソースコードですよ」という目印です。

ステップ2: コードを入力する

以下のコードを、コピー＆ペーストせず、自分の手で入力してください。打ち間違えても大丈夫です！エラーと向き合うことも大切な学習です。

hello.c 
#include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}

コードの解説

このたった5行のコードには、重要な要素が詰まっています。一つずつ見ていきましょう。

1行目: #include <stdio.h>

stdio.h（スタンダード・アイ・オー）は「標準入出力ヘッダファイル」です。 printf関数など、画面に文字を表示する機能を使うための「道具箱」を読み込んでいます。
#includeは「この道具箱を使わせてください」という命令です。

3行目: int main(void)

main関数（メイン関数）は、プログラムの入り口です。どんなC言語プログラムも、必ずmainから実行が始まります。
intは「整数を返す」という意味で、voidは「引数（ひきすう：渡すデータ）はない」という意味です。
料理のレシピでいえば「1. 調理開始」のようなものです。

4行目: printf("Hello, World!\n");

printf（プリントエフ）は「画面に文字を出力する」関数です。
"Hello, World!\n"の\nは「改行（エンター）」を意味します。
行末のセミコロン（;）は「この命令はここまでですよ」という区切りの印で、必ず必要です！忘れるとエラーになります。

5行目: return 0;

return 0は「プログラムが正常に終了しました」という意味です。
0は成功を表し、エラーがあった場合は0以外の値を返すのが慣習です。
料理のレシピでいえば「調理完了」のようなものです。

重要ポイント

C言語では、大文字と小文字は別物です。printfとPrintfは違う意味になります。また、セミコロン（;）は命令の区切りとして必須です。

2.3 コンパイルと実行の仕組み

書いたソースコード（hello.c）は、そのままでは実行できません。コンパイラという翻訳ソフトを使って、コンピュータが理解できる「機械語」に変換する必要があります。 これをコンパイルと呼びます。

flowchart LR A["hello.c
（ソースコード）"] --> B["コンパイラ
（gcc/clang）"] B --> C["hello.exe または a.out
（実行ファイル）"] C --> D["実行すると
画面に表示"]

料理のレシピにたとえると

ソースコードは「料理のレシピ（日本語で書かれた手順）」、コンパイラは「翻訳機」、実行ファイルは「完成した料理」に相当します。

ソースコード

人間が書いたレシピ（hello.c）

コンパイラ

レシピを翻訳する翻訳機（gcc）

実行ファイル

完成した料理（hello.exe）

実際にコンパイルしてみよう

実習: Hello Worldをコンパイルして実行する

Windows (MinGW/MSYS2)

# コンパイル（hello.exeを作成）
gcc hello.c -o hello.exe

# 実行
hello.exe

macOS / Linux

# コンパイル（helloという実行ファイルを作成）
gcc hello.c -o hello

# 実行
./hello

成功すると、画面にこう表示されます：

Hello, World!

コマンドの意味

gccはコンパイラの名前（GNU C Compiler）
hello.cはコンパイルするソースファイル
-o helloは「出力ファイル名をhelloにする」という指示
./は「カレントディレクトリ（今いる場所）のファイルを実行する」という意味

2.4 エラーの読み方と対処法

プログラミングを始めたばかりの頃は、エラーが出るのは当たり前です。むしろ、エラーメッセージは「ここが違うよ」と教えてくれる親切なアドバイスです。 恐れずに、一つずつ読んで対処しましょう。

よくあるエラーと対処法

トラブルシューティング

エラー1: expected ';' before 'return' 

// 間違い（セミコロンがない）
printf("Hello, World!\n")
return 0;

原因: printfの行末にセミコロン（;）がない 

// 正しい
printf("Hello, World!\n");
return 0;

エラー2: implicit declaration of function 'printf' 

// 間違い（#includeがない）
int main(void) {
    printf("Hello, World!\n");
    return 0;
}

原因: #include <stdio.h>がない 

// 正しい
#include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}

エラー3: 'Main' undeclared 

// 間違い（大文字のMain）
int Main(void) {
    printf("Hello, World!\n");
    return 0;
}

原因: mainがMainになっている（大文字・小文字の違い） 

// 正しい（小文字のmain）
int main(void) {
    printf("Hello, World!\n");
    return 0;
}

エラー対処の心構え

① エラーメッセージの行番号を確認する
② セミコロン（;）の忘れがないかチェック
③ スペルミス（特に大文字・小文字）を確認
④ 波括弧{}の対応が取れているか確認

2.5 C++版のHello World

C++でも同じ「Hello, World!」を書いてみましょう。C言語版と少し違いますが、より現代的で読みやすい書き方になっています。

hello.cpp 
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

C言語版との違い

C言語版

  • #include <stdio.h>
  • printf("Hello, World!\n");
  • main(void)と書く

C++版

  • #include <iostream>
  • std::cout << "Hello, World!" << std::endl;
  • main()だけでOK

C++版のコンパイルと実行

Windows 
g++ hello.cpp -o hello.exe
hello.exe
macOS / Linux 
g++ hello.cpp -o hello
./hello

gccとg++の違い

gccはC言語用、g++はC++用のコンパイラです。
ファイル名も.c（C言語）と.cpp（C++）で区別します。

理解度確認クイズ

  1. Q1. コンパイルとは何ですか？なぜ必要ですか？

    回答を表示

    コンパイルとは、人間が書いたソースコード（hello.c）を、コンピュータが理解できる機械語（実行ファイル）に変換する作業です。C/C++はコンパイル型言語なので、実行前にこの変換が必要です。コンパイラ（gcc/g++）がこの翻訳を行います。

  2. Q2. printf("Hello!\n"); の\nは何を意味しますか？

    回答を表示

    \nは「改行（エンターキーを押す）」を意味します。これがあると、文字を表示した後に次の行に移ります。逆に\nがないと、続けて同じ行に表示されます。

  3. Q3. C言語プログラムに必ず必要な関数は何ですか？

    回答を表示

    main関数です。どんなC/C++プログラムも、必ずint main()から実行が始まります。これはプログラムの入り口（エントリーポイント）と呼ばれます。

  4. Q4. セミコロン（;）を忘れるとどうなりますか？

    回答を表示

    コンパイルエラーになります。セミコロン（;）は「この命令はここまで」という区切りの印なので、忘れるとコンパイラが次の行まで1つの命令だと勘違いして、エラーメッセージが表示されます。例: expected ';' before 'return'

  5. Q5. C言語とC++のHello Worldの違いを1つ挙げてください。

    回答を表示

    ① C言語はprintf、C++はstd::coutを使う
    ② C言語は#include <stdio.h>、C++は#include <iostream>
    ③ ファイル拡張子がC言語は.c、C++は.cpp
    ※ いずれか1つ答えられればOKです

この章のまとめ

  • 「Hello, World!」は最初に書く伝統的なプログラムで、環境確認と基本構造の理解に役立つ
  • C言語プログラムには#include、main関数、printf、returnが含まれる
  • コンパイルは「人間のコード → 機械語」への翻訳作業で、gccやg++コンパイラを使う
  • エラーは恐れず、行番号とメッセージを確認してセミコロンやスペルミスをチェックする
  • C++版はiostreamとstd::coutを使い、より現代的な書き方ができる
