第4章

変数と型：情報に名前と種類をつけよう

変数という「ラベル付きの箱」と、その箱に入れるものの「種類（型）」を指定する方法を学びます。数字専用の箱、文字専用の箱など、整理整頓されたプログラムの書き方を身につけましょう。

この章で学ぶこと

  • 変数とは何か、なぜ必要なのかを理解する
  • 変数の宣言と初期化、型の指定方法を身につける
  • let、const、varの違いと使い分けを理解する
  • 分かりやすい変数名の付け方を学ぶ

4.1 変数って何だろう？

変数（へんすう）とは、データを入れておくための「箱」のようなものです。 この箱には、名前（ラベル）をつけることができ、後から中身を取り出したり、変更したりできます。

例えば、日常生活で考えてみましょう：

日常生活の「箱」の例

お菓子の缶

「お菓子の缶」というラベルが貼ってあれば、中にお菓子が入っていることが分かります。

手紙の封筒

「手紙」と書いてあれば、中に手紙が入っていることが分かります。

貯金箱

「貯金箱」とラベルがあれば、中にお金が入っていることが分かります。

プログラミングでも同じように、データに名前をつけて管理します。 名前をつけることで、後から「あの箱の中身を使いたい」「あの箱の中身を変更したい」と簡単に指定できるのです。

変数の役割

変数は、データを一時的に保存し、名前をつけて管理するための仕組みです。 名前をつけることで、プログラムのどこからでもそのデータを呼び出すことができます。

4.2 変数の宣言と初期化

変数を使うためには、まず「宣言（せんげん）」と「初期化（しょきか）」という2つのステップが必要です。

宣言（Declaration）

「この名前の箱を用意します」と宣言すること。

let age;

初期化（Initialization）

箱に最初の値を入れること。

age = 25;

多くの場合、宣言と初期化は同時に行います：

変数の宣言と初期化 
// 数値を入れる変数を宣言して初期化
let age = 25;

// 文字列を入れる変数を宣言して初期化
let name = "太郎";

// 真偽値を入れる変数を宣言して初期化
let isStudent = true;

この例では、letというキーワードを使って変数を宣言しています。 =の右側の値が、箱に入れる最初のデータです。

4.3 変数に型をつけよう

TypeScriptでは、変数に型（type）をつけることができます。 型をつけることで、「この箱には数字しか入れられません」「この箱には文字しか入れられません」と制限できます。

型をつける書き方は次の通りです：

型の指定方法 
// 変数名: 型 = 値
let age: number = 25;
let name: string = "太郎";
let isStudent: boolean = true;

変数名の後に: 型と書くことで、型を指定します。

型名 説明 例
number 数値型（整数・小数） 42, 3.14, -100
string 文字列型 "Hello", 'こんにちは'
boolean 真偽値型（true/false） true, false
any 何でも入れられる型 任意の値

型が違うとエラーになる

数値型の変数に文字列を入れようとすると、TypeScriptがエラーを出してくれます。 これにより、間違いを事前に防ぐことができます。

型エラーの例 
// 正しい例
let age: number = 25;

// エラーになる例（数値型の変数に文字列を入れようとしている）
let height: number = "180cm";  // ❌ エラー！

// 正しくは、文字列型で宣言する
let heightText: string = "180cm";  // ✅ OK

型推論（がたすいろん）

実は、型を書かなくてもTypeScriptが自動で型を推測してくれます。 let age = 25と書くだけで、 TypeScriptは「これは数値型だ」と理解してくれます。 ただし、明示的に型を書くと、コードが読みやすくなります。

4.4 let、const、varの違い

変数を宣言するキーワードには、let、 const、 varの3種類があります。 それぞれ特徴が違うので、使い分けを理解しましょう。

キーワード 再代入 おすすめ度 使い方
const ❌ できない ⭐⭐⭐ 最優先 変わらない値に使う
let ✅ できる ⭐⭐ 推奨 変わる値に使う
var ✅ できる ❌ 非推奨 古い書き方、使わない

constの使い方

constは「定数（ていすう）」を宣言するキーワードです。 一度値を設定したら、後から変更できません。

constの使い方 
// 変わらない値はconstで宣言
const pi: number = 3.14;
const greeting: string = "こんにちは";

// エラー：constで宣言した変数は変更できない
pi = 3.15;  // ❌ エラー！

letの使い方

letは値を後から変更できる変数を宣言するキーワードです。

letの使い方 
// 変わる値はletで宣言
let age: number = 25;
let score: number = 80;

// OK：letで宣言した変数は変更できる
age = 26;  // ✅ 誕生日が来たので年齢を更新
score = 90;  // ✅ 点数が上がった

使い分けのコツ

基本はconstを使いましょう。後から値を変更する必要がある場合のみletを使います。 varは古い書き方なので、使わないようにしましょう。

4.5 変数名の付け方のルール

変数名は自由につけられますが、いくつかのルールとマナーがあります。 分かりやすい名前をつけることで、後から見返したときに理解しやすくなります。

文法的なルール（必須）

1

使える文字

英字（a-z, A-Z）、数字（0-9）、アンダースコア（_）、ドル記号（$）

2

数字で始められない

1ageはNG、 age1はOK

3

予約語は使えない

let、 const、 ifなどのキーワードは変数名に使えない

命名規則（マナー）

一般的に使われる命名規則に従うと、コードが読みやすくなります。

キャメルケース（camelCase）

最も一般的な書き方。最初の単語は小文字、2つ目以降の単語の最初は大文字にします。

✅ userName

✅ totalPrice

✅ isLoggedIn

意味が分かる名前にする

❌ 悪い例

let a = 25;

let x = "太郎";

let tmp = true;

✅ 良い例

let age = 25;

let userName = "太郎";

let isActive = true;

真偽値はis/hasで始める

真偽値（boolean）の変数は、「〜かどうか」を表す名前にします。

✅ isStudent（学生かどうか）

✅ hasPermission（権限があるかどうか）

✅ isVisible（表示されているかどうか）

日本語は使える？

技術的には日本語も使えますが、一般的には英語で書くのがマナーです。 国際的なチームで開発する場合や、ライブラリ・フレームワークとの統一感を出すために、英語で書くことが推奨されます。

実習：変数を使ってみよう

実際に変数を宣言して、型をつけてみましょう。

課題

  1. 1. あなたの年齢を格納する変数ageを、数値型で宣言してください
  2. 2. あなたの名前を格納する変数myNameを、文字列型で宣言してください
  3. 3. 学生かどうかを格納する変数isStudentを、真偽値型で宣言してください
  4. 4. 適切なキーワード（let/const）を選んで宣言してください
解答例を表示
解答例 
// 年齢（変わる可能性があるのでlet）
let age: number = 25;

// 名前（変わらないのでconst）
const myName: string = "太郎";

// 学生かどうか（状況が変わる可能性があるのでlet）
let isStudent: boolean = true;

理解度確認クイズ

  1. Q1. 変数とは何ですか？

    回答を表示

    データを入れておくための「箱」のようなもので、名前（ラベル）をつけて管理できます。 後から中身を取り出したり、変更したりすることができます。

  2. Q2. constとletの違いは何ですか？

    回答を表示

    constは一度値を設定したら変更できない定数を宣言します。 letは後から値を変更できる変数を宣言します。 基本はconstを使い、変更が必要な場合のみletを使います。

  3. Q3. 次のうち、正しい変数名はどれですか？

    • A. 1age
    • B. user-name
    • C. userName
    • D. let
    回答を表示

    C. userNameが正しいです。
    A. 数字で始まっているのでNG
    B. ハイフン（-）は使えないのでNG
    D. letは予約語なのでNG

この章のまとめ

  • 変数は、データを入れておく「箱」で、名前をつけて管理できる
  • 型をつけることで、箱に入れられるデータの種類を制限できる
  • constは変更できない定数、letは変更できる変数を宣言する
  • 変数名は意味が分かる名前にし、キャメルケースで書くのが一般的
