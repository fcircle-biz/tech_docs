第9章

配列とタプル：複数のデータを扱おう

同じ種類のデータをまとめて扱う配列と、決まった構成のデータを扱うタプルの使い方を学びます。型を使って配列の中身を保証する方法を理解しましょう。

この章で学ぶこと

  • 配列とは何か、どのように複数のデータをまとめて扱うかを理解する
  • 配列に型をつけて、安全にデータを操作する方法を学ぶ
  • タプルという固定された構造のデータ型の使い方を習得する
  • 配列とタプルの違いを理解し、適切に使い分けられるようになる

9.1 配列とは？身近な例で理解しよう

これまで学んできた変数は、一つの値を入れる「箱」でした。でも、実際のプログラムでは、 複数のデータをまとめて扱いたい場面がたくさんあります。

例えば、クラスの生徒の名前を管理したいとき、一人ひとりに別々の変数を作るのは大変ですよね：

非効率な方法（おすすめしません） 
// 生徒が増えるたびに変数を追加...非効率
let student1: string = "田中太郎";
let student2: string = "佐藤花子";
let student3: string = "鈴木一郎";
// 100人いたら100個の変数を作る？大変！

そこで登場するのが「配列（はいれつ）」です。配列は、同じ種類のデータを順番に並べて 一つのまとまりとして扱うことができる仕組みです。

配列とは

配列は、同じ型のデータを順番に並べて保管できる「棚」のようなものです。 それぞれの棚には番号（インデックス）が付いていて、0から数えます。

配列のイメージ（生徒名簿の例）

配列を使えば、たくさんのデータを一つの変数名で管理でき、「何番目のデータ」という形で 簡単にアクセスできます。

9.2 配列の作り方と型の指定

TypeScriptで配列を作るには、主に2つの方法があります。どちらも配列の中身の型を明確にすることで、 間違ったデータを入れることを防げます。

方法1: 角カッコ記法

配列の作成と型指定 
// 文字列の配列（型は string[]）
let students: string[] = ["田中太郎", "佐藤花子", "鈴木一郎"];

// 数値の配列（型は number[]）
let scores: number[] = [85, 92, 78, 88];

// 空の配列を作ってから追加することもできる
let colors: string[] = [];
colors.push("赤");   // 要素を追加
colors.push("青");

方法2: Array型を使う（読み方が分かりやすい）

Array型を使った配列の作成 
// どちらの書き方も同じ意味
let students: Array<string> = ["田中太郎", "佐藤花子"];
let scores: Array<number> = [85, 92, 78];

どちらの書き方を使えばいい？

string[] の方がシンプルで一般的です。 Array<string> は型が複雑な場合に読みやすくなることがあります。 どちらを使っても機能は同じなので、好きな方を選んでOKです。

配列の要素にアクセスする

配列の各要素には、インデックスと呼ばれる番号を使ってアクセスします。 重要なのは、インデックスは0から始まるという点です。

配列の要素へのアクセス 
let students: string[] = ["田中太郎", "佐藤花子", "鈴木一郎"];

// 最初の要素（インデックス0）
console.log(students[0]);  // 「田中太郎」

// 2番目の要素（インデックス1）
console.log(students[1]);  // 「佐藤花子」

// 要素を変更することもできる
students[1] = "佐藤美咲";
console.log(students);     // ["田中太郎", "佐藤美咲", "鈴木一郎"]

インデックスは0から始まる

配列の最初の要素は students[0] です。 students[1] は2番目の要素になるので注意しましょう。 存在しないインデックスにアクセスすると undefined が返ります。

9.3 配列の便利な操作メソッド

配列には、データを操作するための便利なメソッド（機能）がたくさん用意されています。 よく使うものを見ていきましょう。

配列の基本メソッド 
let fruits: string[] = ["りんご", "バナナ", "みかん"];

// 配列の長さを取得
console.log(fruits.length);  // 3

// 末尾に要素を追加
fruits.push("ぶどう");
console.log(fruits);  // ["りんご", "バナナ", "みかん", "ぶどう"]

// 末尾の要素を削除して取り出す
let last = fruits.pop();
console.log(last);    // "ぶどう"
console.log(fruits);  // ["りんご", "バナナ", "みかん"]

// 先頭に要素を追加
fruits.unshift("いちご");
console.log(fruits);  // ["いちご", "りんご", "バナナ", "みかん"]

// 先頭の要素を削除して取り出す
let first = fruits.shift();
console.log(first);   // "いちご"
メソッド 説明 例
push() 末尾に要素を追加 arr.push("新要素")
pop() 末尾の要素を削除して返す arr.pop()
unshift() 先頭に要素を追加 arr.unshift("新要素")
shift() 先頭の要素を削除して返す arr.shift()
length 配列の長さを取得 arr.length
includes() 要素が含まれるか確認 arr.includes("値")

9.4 タプルとは？配列との違い

配列は「同じ型の要素を好きなだけ入れられる箱」でしたが、タプルは 「決まった位置に、決まった型のデータを入れる箱」です。

例えば、座標を表すとき「横の位置（数値）」と「縦の位置（数値）」の2つの数字がセットで必要ですよね。 また、人の情報を「名前（文字列）」と「年齢（数値）」のペアで管理したいこともあります。

タプルの作成 
// タプル：[文字列, 数値] の順番で固定
let person: [string, number] = ["田中太郎", 25];

// 1番目は名前（文字列）、2番目は年齢（数値）
console.log(person[0]);  // "田中太郎"
console.log(person[1]);  // 25

// 座標を表すタプル：[x座標, y座標]
let position: [number, number] = [10, 20];
console.log(position);   // [10, 20]

タプルの特徴

タプルは要素の数と型が固定されています。 1番目は必ず文字列、2番目は必ず数値、というように決まっているので、 順番を間違えたり、違う型のデータを入れようとするとエラーになります。

タプルの型安全性

タプルの型チェック 
let person: [string, number] = ["田中太郎", 25];

// エラー：1番目は文字列なのに数値を代入しようとしている
// person[0] = 30;  ← コンパイルエラー

// 正しい：1番目は文字列なのでOK
person[0] = "佐藤花子";

// エラー：順番が逆
// let wrongPerson: [string, number] = [25, "田中"];  ← エラー

配列とタプルの違い

9.5 タプルの実践的な使い方

タプルは、関連する複数の値をセットで返したいときに特に便利です。

タプルを返す関数の例 
// 生徒の情報を取得する関数（名前と点数を返す）
function getStudentInfo(id: number): [string, number] {
    // 実際はDBから取得する想定
    if (id === 1) {
        return ["田中太郎", 85];  // [名前, 点数]
    } else {
        return ["佐藤花子", 92];
    }
}

// タプルを受け取って使う
let [name, score] = getStudentInfo(1);
console.log(`生徒名: ${name}, 点数: ${score}点`);
// 出力: 「生徒名: 田中太郎, 点数: 85点」

分割代入（Destructuring）

let [name, score] = getStudentInfo(1); のように、 タプルの要素を一度に複数の変数に代入できます。これを「分割代入」と呼びます。 とても便利なので、ぜひ覚えておきましょう。

ラベル付きタプル（TypeScript 4.0+）

最近のTypeScriptでは、タプルの各要素に名前（ラベル）をつけることができ、 コードがより読みやすくなります。

ラベル付きタプル 
// ラベルがない場合（何が何だか分かりにくい）
let person1: [string, number] = ["田中太郎", 25];

// ラベル付き（意味が明確で分かりやすい）
let person2: [name: string, age: number] = ["佐藤花子", 30];

// 関数の返り値にも使える
function getCoordinates(): [x: number, y: number] {
    return [100, 200];
}

let [posX, posY] = getCoordinates();
console.log(`X座標: ${posX}, Y座標: ${posY}`);

9.6 配列とタプルの使い分け

配列とタプル、どちらを使えばいいのか迷うこともあるかもしれません。 以下の基準で判断してみましょう。

状況 使うべきもの 理由
同じ種類のデータをたくさん扱う 配列 要素数が可変で、すべて同じ型だから
関連する異なる型のデータをセットで扱う タプル 各位置に意味があり、型が異なるから
何個データがあるか分からない 配列 可変長に対応できるから
必ず2つや3つなど、数が決まっている タプル 固定長で型安全性が高いから
forループで繰り返し処理したい 配列 要素が同じ型なので処理しやすいから

配列が適している例

  • • クラスの生徒名簿
  • • 買い物リスト
  • • テストの点数一覧
  • • 検索結果（数が変動）

タプルが適している例

  • • 座標（x, y）のペア
  • • 人の情報（名前, 年齢）
  • • 日付（年, 月, 日）
  • • RGB色（赤, 緑, 青）
配列とタプルの使い分け実例 
// 配列：同じ型のデータを複数
let students: string[] = ["太郎", "花子", "一郎", "次郎"];

// タプル：異なる型のデータを組み合わせ
let studentInfo: [name: string, age: number, score: number] =
    ["田中太郎", 15, 85];

// タプルの配列：複数の生徒情報を管理
let allStudents: [string, number, number][] = [
    ["田中太郎", 15, 85],
    ["佐藤花子", 16, 92],
    ["鈴木一郎", 15, 78]
];

// 配列をループで処理
for (let student of allStudents) {
    let [name, age, score] = student;
    console.log(`${name}さん（${age}歳）: ${score}点`);
}

理解度確認クイズ

  1. Q1. 配列のインデックスは何から始まりますか？

    回答を表示

    配列のインデックスは0から始まります。最初の要素は arr[0]、 2番目は arr[1] となります。

  2. Q2. 配列とタプルの最大の違いは何ですか？

    回答を表示

    配列は同じ型の要素を可変長で扱いますが、タプルは決まった位置に決まった型のデータを固定長で扱います。 配列はすべての要素が同じ型ですが、タプルは各位置で異なる型を持つことができます。

  3. Q3. 次のコードで正しいのはどれですか？

    let colors: string[] = ["赤", "青", "緑"];
// A) colors.length の値は？
// B) colors[1] の値は？
    回答を表示

    A) colors.length は 3（要素が3つあるため）
    B) colors[1] は "青"（インデックス0が"赤"、1が"青"）

  4. Q4. 座標 (x, y) を表すデータには、配列とタプルのどちらが適していますか？

    回答を表示

    タプルが適しています。座標は常に2つの数値のペアであり、 1番目がx座標、2番目がy座標という意味が固定されているからです。 let position: [x: number, y: number] = [100, 200]; のように書けます。

この章のまとめ

  • 配列は同じ型のデータを順番に並べて保管できる仕組みで、インデックスは0から始まる
  • 配列には push()、pop()、shift()、unshift() などの便利なメソッドがある
  • タプルは決まった位置に決まった型のデータを入れる固定長の構造
  • 配列は可変長で同じ型、タプルは固定長で異なる型を組み合わせられる
  • 分割代入を使うことで、タプルや配列の要素を一度に複数の変数に代入できる
