型の応用：ユニオン型と型ガード

複数の型を扱えるユニオン型と、実行時に型を判定する型ガードの使い方を学び、柔軟性と安全性を両立させる TypeScript の強力な機能を理解しましょう。

この章で学ぶこと

  • ユニオン型を使って複数の型を扱う方法を理解する
  • 型ガードを使って実行時に型を安全に判定する方法を学ぶ
  • 柔軟性と型安全性を両立させたプログラムを書けるようになる

12.1 複数の型を扱える「ユニオン型」とは？

これまで学んできた変数や関数のパラメータでは、1つの型しか指定できませんでした。 しかし、実際のプログラムでは「数値または文字列」「成功の結果またはエラー」など、複数の型のいずれかになるケースがよくあります。

ユニオン型（Union Types）は、このような「AまたはB」という型を表現する仕組みです。 |（パイプ記号）を使って、複数の型を組み合わせます。

日常生活での例：飲み物の注文

カフェで飲み物を注文するとき、「コーヒーまたは紅茶」のどちらかを選びますよね。 このように「どちらか一方」を表すのがユニオン型です。

お客様の注文:

コーヒー または 紅茶

ユニオン型の意味

ユニオン型は「複数の型のうち、どれか1つ」という意味です。 これにより、柔軟性を保ちながら型安全性も確保できます。

12.2 ユニオン型の基本的な使い方

ユニオン型は、型A | 型Bという形で定義します。 まずはシンプルな例を見てみましょう。

ユニオン型の基本例 
// 数値または文字列を受け取る変数
let id: number | string;

id = 123;           // OK: 数値を代入
console.log(id);    // 123

id = "user-456";    // OK: 文字列を代入
console.log(id);    // user-456

// id = true;       // エラー: booleanは許可されていない

上記の例では、id変数は number型または string型のどちらでも受け取れます。

関数パラメータでの使用 
// 結果を表示する関数（成功の文字列またはエラーコード）
function displayResult(result: string | number): void {
  console.log("結果:", result);
}

displayResult("成功しました");  // OK
displayResult(404);             // OK

12.3 型ガードとは？実行時に型を判定する

ユニオン型を使うと柔軟になりますが、1つ問題があります。 「今、この変数はnumberなのか stringなのか？」を プログラム実行中に判断しないと、安全に処理できません。

型ガード（Type Guard）は、実行時に型を判定して、 その型に応じた処理を安全に実行するための仕組みです。

型を判定しないと何が問題？

例えば、数値型にしかないtoFixed()メソッドを、 文字列型の値に対して呼び出そうとするとエラーになります。

型判定が必要な例 
function formatValue(value: number | string): string {
  // このままでは、valueがnumberかstringか分からない
  // return value.toFixed(2);  // エラー: stringにtoFixed()は無い

  // 型を判定する必要がある
  return "処理が必要";
}

型ガードを使うことで、TypeScriptは「この条件の中では、この変数はこの型である」と理解し、 その型専用のメソッドやプロパティを安全に使えるようになります。

12.4 typeofを使った型ガード

最もシンプルな型ガードは、typeof演算子を使う方法です。 typeofは、 値の型を文字列で返してくれます。

typeofを使った型ガード 
function formatValue(value: number | string): string {
  // typeofで型を判定
  if (typeof value === "number") {
    // この中では、valueは確実にnumber型
    return value.toFixed(2);  // OK: toFixed()が使える
  } else {
    // この中では、valueは確実にstring型
    return value.toUpperCase();  // OK: toUpperCase()が使える
  }
}

console.log(formatValue(3.14159));  // "3.14"
console.log(formatValue("hello"));  // "HELLO"

typeofの返す値

typeofは以下のような文字列を返します： "number"、 "string"、 "boolean"、 "object"など。

複数の型を安全に処理 
// 数値、文字列、真偽値のいずれかを受け取る
function describe(value: number | string | boolean): string {
  if (typeof value === "number") {
    return `数値: ${value}`;
  } else if (typeof value === "string") {
    return `文字列: ${value}`;
  } else {
    return `真偽値: ${value}`;
  }
}

console.log(describe(42));       // "数値: 42"
console.log(describe("Hello"));  // "文字列: Hello"
console.log(describe(true));     // "真偽値: true"

12.5 inを使った型ガード

オブジェクトの型を判定するときは、in演算子を使います。 inは、 「あるプロパティがオブジェクトに存在するか」を確認できます。

インターフェース定義 
// 犬のインターフェース
interface Dog {
  name: string;
  bark(): void;  // 吠えるメソッド
}

// 猫のインターフェース
interface Cat {
  name: string;
  meow(): void;  // 鳴くメソッド
}
in演算子を使った型ガード 
// 犬または猫を受け取って鳴かせる関数
function makeSound(animal: Dog | Cat): void {
  // barkプロパティがあるかチェック
  if ("bark" in animal) {
    // この中では、animalは確実にDog型
    animal.bark();
  } else {
    // この中では、animalは確実にCat型
    animal.meow();
  }
}

const myDog: Dog = {
  name: "ポチ",
  bark: () => console.log("ワンワン！")
};

const myCat: Cat = {
  name: "タマ",
  meow: () => console.log("ニャー！")
};

makeSound(myDog);  // "ワンワン！"
makeSound(myCat);  // "ニャー！"

in演算子の仕組み

"プロパティ名" in オブジェクトという形式で、 そのプロパティが存在すればtrue、 存在しなければfalseを返します。

12.6 実習：ユニオン型と型ガードを使ったプログラム

実習 12-1: 支払い処理プログラム

現金またはクレジットカードで支払う処理を、ユニオン型と型ガードを使って実装してみましょう。

手順

  1. 1 以下のコードをコピーして、payment.tsという名前で保存します
  2. 2 ターミナルでtsc payment.tsを実行してコンパイルします
  3. 3 node payment.jsで実行して、結果を確認します
payment.ts 
// 現金支払いの情報
interface CashPayment {
  method: "cash";
  amount: number;
}

// クレジットカード支払いの情報
interface CreditPayment {
  method: "credit";
  cardNumber: string;
  amount: number;
}

// ユニオン型: 現金またはクレジット
type Payment = CashPayment | CreditPayment;

// 支払いを処理する関数
function processPayment(payment: Payment): void {
  console.log(`支払い金額: ${payment.amount}円`);

  // 型ガード: methodプロパティで判定
  if (payment.method === "cash") {
    console.log("現金でお支払いいただきます");
  } else {
    // この中では、paymentはCreditPayment型
    console.log(`カード番号: ${payment.cardNumber}`);
    console.log("クレジットカードで決済します");
  }
}

// 現金で支払い
const cashPayment: CashPayment = {
  method: "cash",
  amount: 1500
};

// クレジットカードで支払い
const creditPayment: CreditPayment = {
  method: "credit",
  cardNumber: "1234-5678-9012-3456",
  amount: 3000
};

processPayment(cashPayment);
processPayment(creditPayment);

動作確認

プログラムを実行すると、以下のような出力が表示されます：

期待される結果 

支払い金額: 1500円
現金でお支払いいただきます
支払い金額: 3000円
カード番号: 1234-5678-9012-3456
クレジットカードで決済します

このプログラムのポイント

  • Payment型は、CashPaymentまたはCreditPaymentのユニオン型です
  • methodプロパティの値で型を判定しています（判別ユニオン型）
  • 型ガードにより、elseブロック内ではcardNumberプロパティに安全にアクセスできます

理解度確認クイズ

  1. Q1. ユニオン型とは何ですか？

    回答を表示

    複数の型のうち、どれか1つを取ることができる型です。 型A | 型Bのように、パイプ記号（|）で型を組み合わせます。 これにより、柔軟性を保ちながら型安全性も確保できます。

  2. Q2. 型ガードはなぜ必要ですか？

    回答を表示

    ユニオン型の変数は複数の型を取る可能性があるため、実行時にどの型なのかを判定する必要があります。 型ガードを使うことで、その型に応じた安全な処理（特定の型にしかないメソッドの呼び出しなど）ができるようになります。

  3. Q3. typeofとinの使い分けは？

    回答を表示

    typeofは、プリミティブ型（number、string、booleanなど）を判定するときに使います。 inは、オブジェクトに特定のプロパティが存在するかを確認して、 インターフェース型を判定するときに使います。

この章のまとめ

  • ユニオン型は、型A | 型Bの形式で複数の型を組み合わせる
  • 型ガードを使うことで、実行時に型を判定し、安全に処理できる
  • typeofはプリミティブ型の判定、inはオブジェクトのプロパティ存在確認に使う
  • ユニオン型と型ガードにより、柔軟性と型安全性を両立できる
