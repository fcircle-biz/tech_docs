第3章

はじめてのTypeScriptプログラム

「Hello, TypeScript!」という文字を表示する最初のプログラムを作成し、TypeScriptファイルの作り方、実行方法、そして「型」という概念の初歩を体験しましょう。

この章で学ぶこと

  • TypeScriptファイルの作成方法と基本構造を理解する
  • コンパイルの概念とTypeScriptの実行方法を学ぶ
  • 型の基本的な書き方と型エラーの見方を体験する

3.1 プロジェクトフォルダを作ろう

まず、TypeScriptのプログラムを書くための専用のフォルダ（ディレクトリ）を作成しましょう。 これは、料理を始める前にキッチンを片付けるようなものです。

実習：プロジェクトフォルダの作成

以下の手順でフォルダを作成し、Visual Studio Codeで開いてみましょう。

  1. ステップ1: デスクトップやドキュメントフォルダに「typescript-practice」という名前のフォルダを作成
  2. ステップ2: Visual Studio Codeを起動し、「ファイル」→「フォルダーを開く」から作成したフォルダを選択
  3. ステップ3: VS Code内で「ターミナル」→「新しいターミナル」を選択（画面下部にターミナルが開きます）

ターミナルとは？

ターミナルは、キーボードで文字を入力してコンピュータに命令を出すツールです。 マウスでクリックする代わりに、文字で指示を出すイメージです。最初は不慣れかもしれませんが、すぐに慣れます！

3.2 最初のTypeScriptファイルを作ろう

それでは、記念すべき最初のTypeScriptプログラムを書きましょう！

実習：hello.tsファイルを作成する

  1. ステップ1: VS Codeの左側のエクスプローラーで、プロジェクトフォルダ名の横にある「新しいファイル」アイコンをクリック
  2. ステップ2: ファイル名に「hello.ts」と入力してEnterキーを押す

ファイル名の「.ts」って何？

.tsは「TypeScriptファイル」を表す拡張子です。 写真が.jpgで終わるように、TypeScriptのプログラムは.tsで終わります。 これにより、エディタが「このファイルはTypeScriptだ」と認識してくれます。

作成したhello.tsファイルに、以下のコードを入力してみましょう：

hello.ts 
// 最初のTypeScriptプログラム
console.log("Hello, TypeScript!");

このコードの意味を一行ずつ見ていきましょう：

  • //
    コメント（注釈）
    //から始まる行は、プログラムの実行に影響しないメモです。人間が読むための説明を書きます。
  • console.log()
    コンソールへの出力
    画面（コンソール）に文字を表示する命令です。カッコの中の文字が表示されます。
  • "Hello, TypeScript!"
    文字列（テキスト）
    ダブルクォート（"）で囲まれた部分は、そのまま表示される文字列です。

3.3 プログラムを実行してみよう

TypeScriptのコードは、そのままでは実行できません。 JavaScriptに「コンパイル（変換）」してから実行する必要があります。

TypeScriptの実行までの流れ

コンパイルって何？

コンパイルとは、人間が読みやすい形で書かれたプログラム（TypeScript）を、 コンピュータが実行できる形（JavaScript）に変換することです。 翻訳作業のようなイメージです。

実習：プログラムを実行する

VS Code内のターミナルで、以下のコマンドを順番に実行してみましょう。

  1. ステップ1: TypeScriptをJavaScriptにコンパイルする
    ターミナル 
    tsc hello.ts
  2. ステップ2: 生成されたJavaScriptファイルを実行する
    ターミナル 
    node hello.js

動作確認

ターミナルに以下のように表示されれば成功です！

Hello, TypeScript!

おめでとうございます！あなたは初めてのTypeScriptプログラムを書いて実行できました。

うまく動かない場合

  • 「tscコマンドが見つからない」と表示される → 第2章でNode.jsとTypeScriptをインストールできているか確認しましょう
  • 「hello.jsが見つからない」と表示される → まず「tsc hello.ts」でコンパイルしてからnodeコマンドを実行しましょう
  • 何も表示されない → ファイル名やコードのスペルミスがないか確認しましょう

3.4 型を使ってみよう

次は、TypeScriptの最大の特徴である「型」を実際に使ってみましょう。

hello.tsファイルを以下のように書き換えてみましょう：

hello.ts（更新版） 
// 型を使った変数の宣言
let userName: string = "太郎";
let age: number = 20;

// 画面に表示
console.log("こんにちは、" + userName + "さん！");
console.log("あなたは" + age + "歳ですね。");

このコードの重要なポイント：

変数名: 型

userName: string

変数名の後ろにコロン（:）と型名を書くことで、この変数には文字列しか入れられないと宣言します。

文字列の連結

"こんにちは、" + userName

プラス記号（+）で文字列をつなげることができます。

実習：型付きプログラムを実行してみよう

  1. 1. 上記のコードをhello.tsに書き換える
  2. 2. コンパイルする：tsc hello.ts
  3. 3. 実行する：node hello.js

実行すると、以下のように表示されます：

こんにちは、太郎さん！
あなたは20歳ですね。

3.5 型エラーを体験してみよう

TypeScriptの最大の強みは、間違いを事前に教えてくれることです。 わざと間違ったコードを書いて、どんなエラーが出るか見てみましょう。

安心してください！

これから出すエラーは、学習のためにわざと間違えるものです。 エラーメッセージは怖いものではなく、「ここを直してね」と教えてくれる親切な先生だと思ってください。

以下のコードをhello.tsに書いてみましょう（間違ったコードです）：

hello.ts（エラーを含む例） 
// 数値型の変数を宣言
let age: number = 20;

// わざと文字列を代入してみる（エラーになる）
age = "二十歳";

console.log(age);

このコードをコンパイルしようとすると、VS Codeのエディタ上でエラーが表示されます：

Type 'string' is not assignable to type 'number'.

日本語訳： 型'string'（文字列）は型'number'（数値）に代入できません。

これが型の力！

age変数はnumber型（数値専用）と宣言しているのに、"二十歳"という文字列を入れようとしたため、 TypeScriptが「それは違うよ！」と教えてくれました。 これにより、プログラムを実行する前に間違いに気づけます。

実習：エラーを修正してみよう

エラーを修正する方法は2つあります。どちらか試してみましょう：

方法1：数値を代入する

let age: number = 20;
age = 25; // 数値なのでOK
console.log(age);

方法2：型をstring型に変更する

let age: string = "20歳";
age = "二十歳"; // 文字列なのでOK
console.log(age);

3.6 型推論：型を書かなくてもいい場合

実は、TypeScriptは賢いので、値を見て自動的に型を推測してくれることがあります。 これを「型推論」と呼びます。

型推論の例 
// 型を明示的に書く場合
let userName: string = "太郎";

// 型推論：TypeScriptが自動的にstring型だと判断
let userName2 = "次郎"; // 型を書かなくてもOK

// 数値も同様
let age = 20; // 自動的にnumber型になる

型を書かなくても、TypeScriptは値を見て「これは文字列だな」「これは数値だな」と判断してくれます。

型を書く？書かない？

初心者のうちは、型を明示的に書くことをおすすめします。 そうすることで、「この変数はどんなデータを入れる予定か」を意識する習慣がつきます。 慣れてきたら、型推論を活用してコードを短くすることもできます。

理解度確認クイズ

  1. Q1. TypeScriptファイルの拡張子は何ですか？

    回答を表示

    .tsです。TypeScriptファイルは必ず.tsで終わります。

  2. Q2. TypeScriptをJavaScriptに変換することを何と言いますか？

    回答を表示

    コンパイルと言います。tscコマンドを使ってコンパイルします。

  3. Q3. 変数に型を指定するにはどうしますか？

    回答を表示

    変数名の後ろにコロン（:）と型名を書きます。例：let age: number = 20;

  4. Q4. 次のコードはエラーになりますか？
    let score: number = "100";

    回答を表示

    はい、エラーになります。score変数はnumber型（数値専用）と宣言されているのに、 文字列"100"を代入しようとしているためです。正しくはlet score: number = 100;とします。

この章のまとめ

  • TypeScriptファイルは.ts拡張子で作成し、tscコマンドでコンパイルする
  • 変数に型を指定することで、間違ったデータの代入を防げる
  • 型エラーはプログラムを実行する前に教えてくれる親切な先生
  • 型推論により、型を書かなくても自動的に型が決まることがある
