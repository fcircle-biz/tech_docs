第2章: 基本構文とプログラム構造

COBOLプログラムの基本構造を理解する

2.1 COBOLの4つのDIVISION

COBOLプログラムは、必ず4つのDIVISION（部）で構成されます。

COBOLプログラムの構造
IDENTIFICATION DIVISION
プログラムの識別情報
ENVIRONMENT DIVISION
実行環境の設定
DATA DIVISION
データの定義
PROCEDURE DIVISION
実行する処理

各DIVISIONの詳細

IDENTIFICATION DIVISION

役割: プログラムの識別情報

必須項目: PROGRAM-ID

例: PROGRAM-ID. SAMPLE-PROG.

ENVIRONMENT DIVISION

役割: 実行環境の設定

内容: ファイルの指定など

例: 簡単なプログラムでは省略可能

DATA DIVISION

役割: データの定義

内容: 変数、ファイル構造など

例: 01 WS-NAME PIC X(20).

PROCEDURE DIVISION

役割: 実行する処理

内容: 実際のプログラムロジック

例: DISPLAY, MOVE, ADD など

2.2 固定形式の記述ルール

COBOLは固定形式で記述する必要があります。各行は80桁で構成され、用途が決まっています。

123456789012345678901234567890123456789012345678901234567890123456789012345678901234567890
^ ^ ^ ^
7 12 72 80
| | | |
| A領域 B領域 | 識別エリア
| |
行番号エリア コメントエリア
重要な記述規則
  • 1-6桁目: 行番号エリア（通常は空白）
  • 7桁目: 標識エリア（継続行は'-'、コメント行は'*'）
  • 8-11桁目: A領域（DIVISION、SECTION、段落名など）
  • 12-72桁目: B領域（一般的な命令文）
  • 73-80桁目: 識別エリア（通常は空白）

記述例

IDENTIFICATION DIVISION.
PROGRAM-ID. SAMPLE-PROG.

ENVIRONMENT DIVISION.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 WS-NAME PIC X(20).

PROCEDURE DIVISION.
MAIN-PARA.
DISPLAY "Hello, COBOL!".
STOP RUN.

2.3 基本的な命令文

COBOLでよく使用される基本的な命令文を学習します。

DISPLAY文

画面に文字列や変数の値を表示

DISPLAY "Hello World"
DISPLAY WS-NAME
MOVE文

データを変数に代入

MOVE "田中" TO WS-NAME
MOVE 100 TO WS-NUMBER
ACCEPT文

キーボードから入力を受け取る

ACCEPT WS-NAME
ACCEPT WS-AGE
STOP RUN文

プログラムを終了

STOP RUN

2.4 プログラムの記述規則

COBOLプログラムを記述する際の重要な規則について学習します。

命名規則

  • プログラム名: 最大30文字、英数字とハイフンのみ
  • 変数名: 最大30文字、英数字とハイフンのみ
  • 段落名: 英数字とハイフンで構成

文の終わり

  • COBOLの文は必ずピリオド（.）で終わる
  • ピリオドの後は改行するか、空白を入れる
IDENTIFICATION DIVISION.
PROGRAM-ID. BASIC-STRUCTURE.

ENVIRONMENT DIVISION.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 WS-MESSAGE PIC X(30).
01 WS-NAME PIC X(20).

PROCEDURE DIVISION.
MAIN-PARA.
MOVE "こんにちは、" TO WS-MESSAGE.
MOVE "COBOL世界" TO WS-NAME.
DISPLAY WS-MESSAGE WS-NAME.
STOP RUN.
