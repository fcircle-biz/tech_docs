ビジネス文書の基本構造

ビジネス文書の基本的な構成要素とフォーマット、5W1Hを活用した情報整理、結論先行型の書き方、読みやすいレイアウトの工夫を学びます。

この章で学ぶこと

  • ビジネス文書の基本的な構成要素(宛名、日付、件名、本文など)を理解する
  • 社内文書と社外文書のフォーマットの違いを知る
  • 5W1H(いつ・どこで・誰が・何を・なぜ・どのように)で情報を整理する方法を習得する
  • 結論先行型の書き方(PREP法の基礎)を身につける
  • 箇条書きと具体例を活用し、読みやすいレイアウトを作成できるようになる

3.1 ビジネス文書の構成要素

ビジネス文書には、基本的な構成要素があります。これらを正しく理解し使うことで、相手に伝わりやすい文書を作成できます。

ビジネス文書の主な構成要素

  • 日付 - 作成日または発信日を記載
  • 宛名 - 文書を送る相手の名前・所属
  • 差出人 - 自分の名前・所属・連絡先
  • 件名(タイトル) - 文書の内容を一目で分かるように
  • 本文 - 前文(挨拶)、主文(用件)、末文(結び)
  • 結語 - 「以上」「敬具」など
flowchart TD A[ビジネス文書] --> B[ヘッダー部分] A --> C[本文部分] A --> D[結語部分] B --> B1[日付] B --> B2[宛名] B --> B3[差出人] B --> B4[件名] C --> C1[前文/挨拶] C --> C2[主文/用件] C --> C3[末文/結び] D --> D1[結語/署名]

3.2 社内文書と社外文書のフォーマットの違い

ビジネス文書は社内文書(同じ会社内でのやり取り)と社外文書(お客様や取引先とのやり取り)で、書き方やフォーマットが異なります。

項目 社内文書 社外文書
敬語レベル 丁寧語中心、簡潔に 尊敬語・謙譲語を使用、丁寧に
宛名の書き方 「営業部 各位」「山田課長」 「株式会社〇〇 △△様」
挨拶文 「お疲れ様です」または省略可 「平素よりお世話になっております」必須
文書の長さ 簡潔・要点のみ 丁寧な説明、配慮のある表現
結語 「以上」「よろしくお願いします」 「敬具」「今後ともよろしくお願い申し上げます」

ポイント

社内文書は効率重視で簡潔に、社外文書は丁寧さと配慮を重視して書きます。相手との関係性によって使い分けましょう。

3.3 5W1Hで情報を整理する

5W1Hは、ビジネス文書で必要な情報を整理するための基本的なフレームワークです。これを使うことで、抜け漏れなく情報を伝えることができます。

When
(いつ)

日時・期間・締切

例: 1月15日(月)15時～

Where
(どこで)

場所・会場・環境

例: 3階会議室A

Who
(誰が)

対象者・担当者

例: 営業部全員

What
(何を)

内容・対象物

例: 売上報告会議

Why
(なぜ)

理由・目的・背景

例: 売上目標の進捗確認

How
(どのように)

方法・手段

例: Zoom会議で実施

5W1Hを使った例文

件名: 営業会議のご案内

営業部各位

下記の通り営業会議を開催いたします。

  • • 日時(When): 1月15日(月) 15:00～16:00
  • • 場所(Where): 3階会議室A
  • • 対象(Who): 営業部全員
  • • 議題(What): 第4四半期売上報告
  • • 目的(Why): 目標達成に向けた進捗確認
  • • 方法(How): 各チームリーダーから報告

3.4 結論を先に書く習慣をつける

ビジネス文書では結論を先に書くことが基本です。相手は忙しいことが多いため、最初に要点を伝えることで、効率的にコミュニケーションできます。

PREP法(プレップ法)の基本構造

P

Point(結論)

最初に結論や要点を伝える

R

Reason(理由)

なぜその結論になったのか、理由を説明

E

Example(具体例)

具体的な例やデータで補足

P

Point(結論の再確認)

最後にもう一度結論を強調

悪い例(結論が最後)

「先週からA社との商談を進めており、何度か打ち合わせを重ねてきました。価格についても折り合いがつき、納期も問題ないことを確認しました。よって、契約を締結したいと思います。」

良い例(結論が最初)

「A社との契約を締結したいと思います。先週から商談を進めており、価格・納期ともに条件が整いました。複数回の打ち合わせを経て、双方が合意に至っています。」

3.5 箇条書きで分かりやすく

複数の項目や情報を伝える場合、箇条書きを使うと格段に読みやすくなります。文章で書くよりも、視覚的に情報を整理できます。

悪い例(すべて文章)

「会議の準備として、会議室の予約と参加者への連絡、資料の印刷、プロジェクターの動作確認をお願いします。」

良い例(箇条書き)

会議の準備をお願いします:

  • 会議室の予約
  • 参加者への連絡
  • 資料の印刷(各10部)
  • プロジェクターの動作確認

箇条書きを使うべき場面

  • 複数の項目をリストアップするとき
  • 手順やステップを説明するとき
  • 重要なポイントを強調したいとき
  • 比較や対比を示すとき

3.6 数字と具体例で説得力アップ

具体的な数字や実例を使うことで、文書の説得力が大きく向上します。曖昧な表現ではなく、定量的なデータを示しましょう。

曖昧な表現

「売上が大きく増加しました」

具体的な数字

「売上が前月比25%増加し、500万円に達しました」

曖昧な表現

「多くのお客様から好評をいただいています」

具体的な数字

「アンケートで95%のお客様から『満足』以上の評価をいただきました」

数字を使う際のポイント

  • 「多い」「少ない」より「50%」「10件」など具体的に
  • 「すぐに」「早く」より「3日以内」「1時間で」など明確に
  • 比較するときは基準を明示(前年比、前月比など)

3.7 読みやすいレイアウトの工夫

内容が良くても、レイアウトが悪いと読んでもらえません。視覚的に読みやすいレイアウトを心がけましょう。

読みやすいレイアウトのチェックリスト

段落の工夫

  • 適度に段落を分ける
  • 1段落は3～5行程度に
  • 段落間に空行を入れる

見出しの活用

  • 章・節で見出しをつける
  • 見出しで内容が分かるように
  • 階層構造を意識する

文字の強調

  • 重要な箇所は太字に
  • 強調しすぎない(2～3箇所)
  • 下線や色は控えめに

余白の確保

  • 適度な行間を保つ
  • 左右の余白を確保
  • 詰め込みすぎない

実習 3-1: 5W1Hを使って文書を改善しよう

以下の文章を、5W1Hを意識して書き直してみましょう。

改善前の文章

「来週、新しいプロジェクトのキックオフミーティングを行います。関係者は必ず出席してください。」

改善例を見る

5W1Hを明記した改善例:

件名: 【必須出席】新プロジェクトキックオフミーティングのご案内

下記の通りキックオフミーティングを開催いたします。

  • • 日時(When): 1月20日(月) 10:00～12:00
  • • 場所(Where): 本社3階 大会議室
  • • 対象(Who): 開発チーム全員、営業部リーダー
  • • 議題(What): プロジェクト概要説明、役割分担、スケジュール確認
  • • 目的(Why): プロジェクトの方向性を全員で共有し、スムーズな立ち上げを実現するため
  • • 準備(How): 各自、担当業務の現状を整理してご参加ください

理解度確認クイズ

  1. Q1. 5W1Hとは何ですか？それぞれの意味を答えてください。

    回答を表示

    5W1Hとは、情報を整理するためのフレームワークです。
    When(いつ)、Where(どこで)、Who(誰が)、What(何を)、Why(なぜ)、How(どのように)の6つの要素で構成されます。これらを明確にすることで、抜け漏れのない情報伝達が可能になります。

  2. Q2. PREP法の「PREP」は何の略ですか？それぞれ説明してください。

    回答を表示

    P = Point(結論): 最初に結論や要点を伝える
    R = Reason(理由): なぜその結論になったのか理由を説明
    E = Example(具体例): 具体的な例やデータで補足
    P = Point(結論再確認): 最後にもう一度結論を強調

  3. Q3. 社内文書と社外文書の大きな違いは何ですか？

    回答を表示

    社内文書は効率重視で簡潔に書き、丁寧語中心で挨拶文を省略することもあります。
    社外文書は丁寧さと配慮を重視し、尊敬語・謙譲語を使用し、挨拶文は必ず入れます。相手との関係性によって、敬語のレベルや文書の長さ、フォーマットが異なります。

この章のまとめ

  • ビジネス文書には日付、宛名、差出人、件名、本文、結語などの基本構成要素がある
  • 社内文書は簡潔に、社外文書は丁寧に書く。相手との関係性で使い分ける
  • 5W1H(いつ・どこで・誰が・何を・なぜ・どのように)を使うと情報の抜け漏れを防げる
  • 結論を先に書く(PREP法)ことで、相手に要点が伝わりやすくなる
  • 箇条書きを使うと、複数の項目を視覚的に整理でき、読みやすさが向上する
  • 具体的な数字や実例を使うことで、説得力のある文書になる
  • 適度な段落分け、見出し、余白を確保することで、視覚的に読みやすいレイアウトになる
