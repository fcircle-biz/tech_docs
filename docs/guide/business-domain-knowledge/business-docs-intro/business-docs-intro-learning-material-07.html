報告書の作成

報告書の目的と種類、基本構成、事実と意見の書き分け、PREP法を使った報告書作成について学びます。実践的な出張報告書の作成演習も行います。

この章で学ぶこと

  • 報告書の目的と種類（業務報告、出張報告、調査報告、トラブル報告）
  • 報告書の基本構成（表題、報告日、報告者、概要、詳細、結論）
  • 事実と意見を明確に分けて記述する方法
  • 時系列報告と項目別報告の使い分け
  • PREP法を使った分かりやすい報告書作成
  • データや根拠を効果的に提示する方法

7.1 報告書の目的と種類

報告書は、業務の進捗や結果、調査内容、発生したトラブルなどを上司や関係者に伝えるための重要な文書です。組織の意思決定や今後の方針決定の基礎となる情報を提供します。

flowchart TD A[報告書を作成する] --> B[目的] B --> C[業務の状況を共有] B --> D[意思決定の材料を提供] B --> E[記録として残す] C --> F[上司・関係者が現状を把握] D --> F E --> F

報告書の主な種類

  • 業務報告
    日常業務の進捗や完了状況を報告。日報・週報・月報など
  • 出張報告
    出張先での活動内容、訪問先との協議結果などを報告
  • 調査報告
    市場調査、競合分析、顧客ニーズ調査などの結果を報告
  • トラブル報告
    発生した問題、原因、対応状況、再発防止策を報告

7.2 報告書の基本構成

報告書には定型的な構成があります。この構成に沿って書くことで、読み手が必要な情報をすぐに見つけられます。

1

表題

報告書のタイトル。何についての報告かを明確に

2

報告日・報告者

いつ、誰が報告したのかを明記

3

概要（要約）

結論や重要なポイントを冒頭で簡潔に

4

詳細（本文）

経緯、内容、データ、分析などを詳しく記載

5

結論・所見

分析結果、意見、提案、今後の対応など

6

添付資料

参考資料、データ、写真などがあれば添付

ポイント

概要で結論を先に伝えることで、忙しい上司でも要点をすぐに把握できます。詳細は後から読めばよい構成にしましょう。

7.3 事実と意見の書き分け

報告書では「事実」と「意見・所見」を明確に区別して書くことが非常に重要です。混同すると、読み手が正しい判断をできなくなります。

区分 説明 例
事実 客観的に確認できること、数値データ、発生した出来事 「売上は前年比20%増の500万円だった」
意見 報告者の考え、解釈、推測、提案 「新商品の効果と考えられます」

悪い例：事実と意見が混在

「A社訪問では非常に良い反応があり、おそらく来月には契約が成立すると思われる。」

→ 「良い反応」「おそらく」「思われる」はすべて主観的な意見・推測です

良い例：事実と意見を分けて記述

【事実】
A社訪問時、先方担当者より「興味がある」「前向きに検討したい」との発言があった。具体的な契約時期については「来月中旬までに社内で判断する」との回答を得た。

【所見】
先方の反応から判断すると、契約成立の可能性は高いと考えられます。来月上旬にフォローアップを行うことを提案します。

使い分けのコツ

「【事実】」「【所見】」のように見出しを付けて明確に区別しましょう。読み手が客観的な情報と主観的な判断を混同せずに理解できます。

7.4 時系列報告と項目別報告の使い分け

報告書の詳細部分を書く際、時系列報告と項目別報告の2つの方法があります。内容によって使い分けましょう。

時系列報告

時間の流れに沿って記述

適している場合：

  • 出張報告
  • トラブル対応報告
  • イベント報告

例：

9:00 A社到着
9:30 製品説明
11:00 価格交渉
12:00 昼食会

項目別報告

テーマごとに整理して記述

適している場合：

  • 業務報告
  • 調査報告
  • 複数の議題がある場合

例：

1. 価格について
2. 納期について
3. サポートについて
4. 契約条件について

選び方のポイント

「何が起きたか」を伝えたいなら時系列、「何について話したか」を伝えたいなら項目別を選びましょう。

7.5 PREP法を使った報告書作成

PREP法は、分かりやすく説得力のある文章を書くための型です。報告書の概要や結論部分で活用すると効果的です。

flowchart TD A[PREP法] --> B["P: Point (結論)"] A --> C["R: Reason (理由)"] A --> D["E: Example (具体例)"] A --> E["P: Point (結論の再確認)"] B --> F[分かりやすく説得力のある報告] C --> F D --> F E --> F
P

Point（結論・要点）

まず結論を述べる

例：「A社との契約は成立する見込みです」
R

Reason（理由）

なぜそう言えるのかを説明

例：「先方担当者が『ぜひ導入したい』と明言し、予算も確保済みとのことです」
E

Example（具体例・事実）

具体的なデータや事例を示す

例：「契約書のドラフトを確認済みで、来週の取締役会で最終決裁予定とのことです」
P

Point（結論の再確認）

最後にもう一度結論を強調

例：「以上の状況から、来月中の契約成立は確実と判断しています」

7.6 データと根拠の示し方

報告書の説得力を高めるには、数値データや具体的な根拠を示すことが重要です。

悪い例

「顧客からの問い合わせが増えています」

→ 曖昧で具体性がない

良い例

「顧客からの問い合わせは先月比30%増の150件でした」

→ 具体的な数値で明確

データを示す際のポイント

  • 数値を明記：「多い」ではなく「150件」と具体的に
  • 比較を示す：「前年比」「先月比」など基準を明確に
  • 出典を記載：「社内アンケート結果より」など情報源を明示
  • グラフ・表を活用：視覚的に分かりやすく提示

実習 7-1: 出張報告書を書いてみよう

以下のシナリオに基づいて、簡単な出張報告書を作成してみましょう。

シナリオ

あなたは1月15日に大阪のB社を訪問し、新製品「SmartPad Pro」のプレゼンテーションを行いました。 先方の担当者（田中部長、鈴木課長）は製品に強い関心を示し、「社内で検討して2週間以内に回答する」と約束してくれました。 価格については「予算内で問題ない」との回答を得ました。

作成する報告書の構成

  1. 表題：「B社訪問報告」
  2. 報告日・報告者
  3. 概要：結論を簡潔に
  4. 詳細：訪問の経緯と内容
  5. 所見：今後の対応提案
記入例を見る

B社訪問報告

報告日：2025年1月15日

報告者：営業部 山田太郎

【概要】

B社（大阪）を訪問し、新製品「SmartPad Pro」のプレゼンテーションを実施しました。先方は強い関心を示し、2週間以内に導入可否を回答する予定です。

【詳細】

・訪問日時：2025年1月15日 14:00～16:00

・訪問先：B社（大阪市北区）

・先方出席者：田中部長、鈴木課長

・内容：新製品「SmartPad Pro」の機能説明と価格提示

・先方の反応：「社内で前向きに検討したい」「予算は問題ない」との発言あり

【所見】

先方の反応から判断すると、導入の可能性は高いと考えられます。2週間後にフォローアップの連絡を行い、進捗を確認することを提案します。

理解度確認クイズ

  1. Q1. 報告書で最も重要な原則は何ですか？

    回答を表示

    事実と意見を明確に分けて記述することです。事実は客観的なデータや出来事、意見は報告者の考えや推測です。これらを混同すると、読み手が正しい判断をできなくなります。

  2. Q2. PREP法の「PREP」は何の略ですか？

    回答を表示

    Point（結論）、Reason（理由）、Example（具体例）、Point（結論の再確認）の略です。この順番で書くことで、分かりやすく説得力のある報告書になります。

  3. Q3. 出張報告に適しているのは、時系列報告と項目別報告のどちらですか？

    回答を表示

    時系列報告が適しています。出張では「何時に何をしたか」という時間の流れが重要なため、訪問の順序や活動の流れに沿って記述すると分かりやすくなります。ただし、複数の議題について協議した場合は、項目別報告を併用することもあります。

この章のまとめ

  • 報告書は、業務の状況を共有し、意思決定の材料を提供する重要な文書
  • 基本構成は表題、報告日・報告者、概要、詳細、結論、添付資料の順
  • 事実と意見は明確に区別し、【事実】【所見】などの見出しを付けて分ける
  • 時系列報告は出張報告やトラブル報告に、項目別報告は業務報告や調査報告に適している
  • PREP法（結論→理由→具体例→結論）を使うと分かりやすい報告書になる
  • 説得力を高めるには数値データや具体的な根拠を示すことが重要
