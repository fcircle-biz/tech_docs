第3章: 報告書の書き方

論理的で説得力のある報告書を作成する

この章で学ぶこと
  • 報告書の目的と種類を理解する
  • 報告書の基本構成を習得する
  • 事実と意見を明確に区別する方法を学ぶ
  • PREP法を活用した論理的な文章構成を身につける
  • データと図表を効果的に活用する技術を習得する

3.1 報告書の目的と種類

報告書は、業務の進捗や結果、調査内容などを上司や関係者に伝えるための文書です。正確な情報提供と意思決定のサポートが主な目的です。

flowchart TD A[報告書の種類] --> B[業務報告書] A --> C[調査報告書] A --> D[出張報告書] A --> E[トラブル報告書] A --> F[プロジェクト報告書] B --> B1["日常業務の
進捗・結果報告"] C --> C1["市場調査・
競合分析の報告"] D --> D1["出張目的・
成果・経費報告"] E --> E1["問題発生時の
原因・対策報告"] F --> F1["プロジェクトの
進捗・課題報告"]

3.2 報告書の基本構成

報告書には定型的な構成があります。この構成に従うことで、読み手が必要な情報を素早く見つけられます。

構成要素内容ポイント
表題報告書のタイトル内容が一目で分かるタイトルをつける
報告日時報告書作成日正確な日付を記載
報告者作成者の氏名・所属責任の所在を明確にする
概要要点の要約結論を先に述べる（結論先行型）
詳細具体的な内容5W1Hを意識して記載
結論まとめと今後の方針次のアクションを明示
添付資料関連データ・参考資料本文を補足する資料を添付

3.3 事実と意見の書き分け

報告書では、客観的な事実と主観的な意見を明確に区別することが極めて重要です。

区分説明記述例
事実 客観的に確認できる情報 「売上は前月比10%増加した」「会議には15名が出席した」
意見・考察 主観的な判断や分析 「〜と考えられる」「〜の可能性がある」「〜と推測される」
事実と意見の混同に注意
  • 悪い例: 「このシステムは使いにくい」→ 主観的意見のみ
  • 良い例: 「操作手順が5ステップ必要で、他社製品（3ステップ）より多い。このため使いにくいと考えられる」→ 事実+意見

3.4 結論先行型の書き方

ビジネス報告書では、結論を最初に述べる構成が効果的です。忙しい読み手は要点だけを素早く把握したいからです。

flowchart TD A[結論先行型の構成] --> B["結論
最も伝えたいこと"] B --> C["理由
なぜそう言えるか"] C --> D["根拠・データ
事実・数値・事例"] D --> E["まとめ
再度結論を強調"]

3.5 PREP法の活用

PREP法は、論理的で説得力のある文章を書くためのフレームワークです。報告書やプレゼンテーションで広く使われています。

要素英語内容記述例
P Point 結論・主張 「新システムの導入を推奨します」
R Reason 理由 「なぜなら、業務効率が30%向上するからです」
E Example 具体例・根拠 「試験導入した営業部では、事務作業時間が1日2時間削減されました」
P Point 結論の再主張 「以上の理由から、新システム導入を提案します」
実習 3-1: PREP法で文章作成

テーマ: 社内研修の実施を報告する

作成例
【P: 結論】
本日実施したビジネスマナー研修は、非常に有意義でした。

【R: 理由】
なぜなら、参加者の満足度が高く、実践的なスキルを習得できたからです。

【E: 具体例】
研修後のアンケートでは、参加者30名のうち28名が「満足」または「非常に満足」と回答しました。
また、「明日から実践したい」というコメントが20件以上寄せられました。

【P: 結論の再主張】
以上の結果から、今回の研修は成功であり、今後も定期的に実施すべきと考えます。

3.6 データと根拠の提示方法

報告書の信頼性を高めるには、具体的な数値データや客観的な根拠を示すことが重要です。

  • 定量データ: 売上高、達成率、件数、時間など数値で示す
  • 比較データ: 前年比、前月比、目標比など比較して示す
  • 出典明記: データの出所を明確にする
  • 視覚化: グラフや表で分かりやすく表現する

3.7 図表・グラフの効果的な使用

複雑なデータや大量の情報は、図表やグラフを使って視覚的に表現すると理解しやすくなります。

図表の種類適した用途使用例
棒グラフ数値の比較月別売上比較、部門別実績比較
折れ線グラフ時系列の変化売上推移、アクセス数の変動
円グラフ構成比・割合市場シェア、予算配分
表詳細データの整理商品一覧、スケジュール
フローチャートプロセスの説明業務フロー、システム構成

3.8 時系列報告と項目別報告

報告書の記述方法には、時系列報告と項目別報告の2つのアプローチがあります。

方式特徴適した場面
時系列報告 時間の流れに沿って記述 出張報告、イベント報告、トラブル報告
項目別報告 テーマごとに整理して記述 プロジェクト報告、調査報告、業務報告

3.9 問題点と改善策の記述

報告書で問題点を指摘する際は、必ず改善策もセットで提案します。問題提起だけでは建設的ではありません。

flowchart TD A[問題報告の流れ] --> B["現状
何が起きているか"] B --> C["問題点
何が課題か"] C --> D["原因分析
なぜ起きたか"] D --> E["改善策
どう解決するか"] E --> F["実行計画
いつ、誰が、何を"]
実習 3-2: 業務報告書の作成

状況: 新製品発表会を実施した結果を報告する

作成例
業務報告書

報告日: 2025年10月30日
報告者: 営業企画部 山田太郎

件名: 新製品発表会 実施報告

1. 概要
新製品「XYZ-2000」の発表会を予定通り実施し、目標を上回る成果を得ました。

2. 実施内容
日時: 2025年10月28日(月) 14:00~17:00
場所: 東京国際フォーラム
参加者: 150名(目標100名に対し150%)

3. 成果
- 商談希望: 45社(目標30社を50%上回る)
- 即日受注: 8件、合計1,200万円
- メディア掲載: 業界誌3社、Webメディア5社

4. 課題と対策
【課題】会場が手狭で一部立ち見が発生
【対策】次回は収容人数200名の会場を確保

5. 今後の予定
- フォローアップ商談: 11月中旬までに完了予定
- 次回発表会: 2026年4月を計画

3.10 報告書作成のチェックリスト

報告書を提出する前に、以下のチェックリストで確認しましょう。

チェック項目確認内容
5W1Hいつ、どこで、誰が、何を、なぜ、どのように が明確か
事実と意見事実と意見を明確に区別しているか
データ具体的な数値データを示しているか
結論結論や今後の方針が明確か
誤字脱字誤字脱字がないか
固有名詞人名、会社名、商品名などが正確か
日付・数値日付や数値に誤りがないか
図表図表が適切に配置され、見やすいか
理解度確認クイズ
  1. 報告書の基本構成要素を5つ以上挙げてください。
  2. PREP法の4つの要素とは何ですか？
  3. 事実と意見を区別する重要性を説明してください。
  4. 結論先行型の書き方とは何ですか？なぜ有効ですか？
  5. 時系列報告と項目別報告の使い分けを説明してください。
解答例
  1. 表題、報告日時、報告者、概要、詳細、結論、添付資料
  2. Point(結論)、Reason(理由)、Example(具体例)、Point(結論の再主張)
  3. 客観的な事実と主観的な意見を区別することで、報告書の信頼性が高まり、読み手が正確な判断ができる
  4. 最初に結論を述べる書き方。忙しい読み手が要点を素早く把握でき、効率的なコミュニケーションができる
  5. 時系列報告は出張報告など時間の流れが重要な場合に使用。項目別報告はプロジェクト報告など複数テーマを整理する場合に使用
この章のまとめ
  • 報告書は業務報告、調査報告、出張報告など目的に応じて種類がある
  • 表題、日時、報告者、概要、詳細、結論、添付資料の構成が基本
  • 事実と意見を明確に区別することが信頼性を高める
  • PREP法を使うと論理的で説得力のある文章が書ける
  • データと図表を活用して視覚的に分かりやすく表現する
  • 問題報告では必ず改善策もセットで提案する
