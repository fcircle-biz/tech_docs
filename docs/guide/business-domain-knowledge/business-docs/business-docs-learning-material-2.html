第2章: 社内文書の作成基礎

社内コミュニケーションを円滑にする文書作成

この章で学ぶこと
  • 社内文書の種類と特徴を理解する
  • 通知文・連絡文の効果的な書き方を習得する
  • 業務報告書の基本構成を身につける
  • 回覧文書と依頼文の作成方法を学ぶ
  • 定型文書とテンプレートの活用法を知る

2.1 社内文書の種類と特徴

社内文書は、組織内での情報共有や業務連絡を目的とした文書です。社外文書と比べて簡潔で実用的な表現が求められます。

flowchart TD A[社内文書] --> B[連絡系文書] A --> C[報告系文書] A --> D[記録系文書] A --> E[申請系文書] B --> B1[通知文] B --> B2[案内文] B --> B3[回覧文] C --> C1[業務報告] C --> C2[進捗報告] C --> C3[完了報告] D --> D1[議事録] D --> D2[作業記録] D --> D3[引継書] E --> E1[稟議書] E --> E2[申請書] E --> E3[届出書]

2.2 社内通知・連絡文の書き方

社内通知や連絡文は、社内の構成員に情報を伝達するための文書です。要点を明確に、誤解のない表現を心がけます。

社内通知文の基本構造

flowchart TD A[社内通知文の構成] --> B[件名] A --> C[日付] A --> D[宛先] A --> E[差出人] A --> F[本文] A --> G[補足情報] B --> B1["具体的で分かりやすく"] F --> F1[前文: 通知の目的] F --> F2[主文: 具体的内容] F --> F3[末文: 依頼・締めの言葉] G --> G1[問い合わせ先]
実習 2-1: 社内通知文の作成

状況: システムメンテナンスのため、社内ネットワークが一時停止することを全社員に通知する

作成例
件名: システムメンテナンスに伴うネットワーク停止のお知らせ

日付: 2025年10月30日
宛先: 全社員各位
差出人: 情報システム部

いつもお世話になっております。

下記の日程でシステムメンテナンスを実施いたします。
メンテナンス中は社内ネットワークが使用できなくなりますので、ご注意ください。

記

1. メンテナンス日時
   2025年11月5日(水) 22:00 ~ 翌6:00

2. 影響範囲
   - 社内LAN/Wi-Fi
   - 社内共有サーバー
   - 社内メールシステム

3. お願い事項
   - 重要なファイルは事前にローカル保存してください
   - 緊急の連絡は各部署の携帯電話をご利用ください

ご不便をおかけしますが、ご理解とご協力をお願いいたします。

問い合わせ先: 情報システム部 田中(内線:1234)

2.3 社内報告書の基本構成

社内報告書は、業務の進捗や完了を上司や関係者に報告するための文書です。事実を正確に、簡潔に記載することが重要です。

報告書の種類 目的 主な内容
日報 日々の業務進捗を報告 実施した業務、進捗状況、課題
週報 1週間の業務をまとめて報告 週間実績、達成事項、次週予定
月報 月単位の業務実績を報告 月間実績、KPI達成状況、分析
プロジェクト報告 プロジェクトの状況を報告 進捗率、課題、リスク、対策

2.4 回覧文書の作成方法

回覧文書は、複数の関係者に順次回覧して情報を共有する文書です。回覧ルートと期限を明記することが重要です。

実習 2-2: 回覧文書の作成

状況: 新しい勤怠管理システムの導入について、各部門の責任者に回覧する

作成例
回覧

件名: 新勤怠管理システム導入のご案内
発行日: 2025年10月30日
発行者: 人事部 山田太郎

各部門長各位

いつもお世話になっております。

このたび、新しい勤怠管理システムを導入することになりました。
詳細は別紙の資料をご確認いただき、各部門での説明会実施についてご検討ください。

回覧期限: 2025年11月10日まで
回覧ルート: 営業部長 → 開発部長 → 総務部長 → 経理部長

※ご確認後、押印の上、次の部門へ回覧願います。

2.5 業務連絡メモの書き方

業務連絡メモは、日常的な業務連絡や簡単な依頼を行う際に使用する簡易的な文書です。

  • 簡潔性: 必要最小限の情報のみ記載
  • 明確性: 誰が、何を、いつまでに、を明示
  • 完結性: 1枚で完結するようにまとめる

2.6 社内依頼文の作成

社内依頼文は、他の部署や同僚に協力や支援を依頼する際に使用します。依頼内容を具体的に、かつ丁寧に伝えることが大切です。

社内依頼文作成の注意点
  • 依頼の背景と理由を明確に説明する
  • 期限と優先度を明示する
  • 相手の負担を考慮した依頼内容にする
  • 協力への感謝の気持ちを表す

2.7 社内案内文の作成

社内イベントや研修などの案内文は、参加を促すために魅力的で分かりやすい内容にします。

実習 2-3: 社内研修案内文の作成

状況: 全社員向けにビジネスマナー研修を開催する

作成例
件名: ビジネスマナー研修のご案内

日付: 2025年10月30日
宛先: 全社員各位
差出人: 人材開発部

いつもお世話になっております。

下記のとおりビジネスマナー研修を実施いたします。
お客様対応の基本から電話応対、名刺交換まで、実践的な内容です。
ぜひ奮ってご参加ください。

記

1. 研修テーマ
   「実践ビジネスマナー ~信頼される社会人になるために~」

2. 日時
   2025年11月20日(水) 14:00~17:00

3. 場所
   本社2階 大会議室

4. 対象者
   全社員(特に入社3年目までの方を推奨)

5. 申込方法
   11月10日までに人材開発部・鈴木まで社内メールでお申し込みください

6. 持ち物
   筆記用具、名刺10枚

皆様のご参加をお待ちしております。

問い合わせ先: 人材開発部 鈴木(内線:2345)

2.8 定型文書とテンプレートの活用

頻繁に作成する文書は、テンプレート化することで作成時間を大幅に短縮できます。Wordのテンプレート機能を活用しましょう。

flowchart LR A[定型文書の作成手順] --> B[基本フォーマット作成] B --> C[必要項目を定義] C --> D[テンプレート保存] D --> E[繰り返し利用] E --> F[定期的な見直し]

テンプレート化すべき文書例

  • 日報・週報・月報
  • 業務引継書
  • 出張報告書
  • 休暇申請書
  • 会議室予約申請
  • 社内案内文

2.9 社内文書の保管と管理

作成した社内文書は、後から参照できるように適切に保管・管理することが重要です。

管理項目 ポイント
ファイル命名規則 日付_文書種類_件名.docx（例: 20251030_報告書_営業実績.docx）
フォルダ構成 年度別→文書種類別→月別に整理
バージョン管理 改訂版は_v2、_v3などを付与
保存場所 共有サーバーまたはクラウドストレージ

2.10 メール vs 書面の使い分け

社内コミュニケーションでは、メールと書面（文書ファイル）を適切に使い分けることが重要です。

特徴 メール 書面（文書ファイル）
適した用途 日常的な連絡、簡単な依頼 正式な報告、提案、申請
作成時間 短時間 やや時間がかかる
形式 カジュアル フォーマル
記録性 メール保存 ファイルサーバーに保管
承認フロー 不向き 適している

2.11 社内文書作成時の注意点

  • 社内用語の統一: 組織内で共通の用語を使用する
  • 略語の説明: 新入社員でも理解できるよう、初出時に説明を加える
  • 機密情報の扱い: 機密レベルを明示し、適切に管理する
  • 期限の明示: 対応期限や提出期限を明確にする
  • 問い合わせ先の記載: 担当者と連絡先を必ず記載する
理解度確認クイズ
  1. 社内文書と社外文書の主な違いは何ですか？
  2. 社内通知文に必ず含めるべき基本要素を5つ挙げてください。
  3. 回覧文書で特に重要な記載事項は何ですか？
  4. 業務報告書の種類を3つ挙げ、それぞれの目的を説明してください。
  5. 社内文書のファイル命名規則の例を示してください。
解答例
  1. 社内文書は比較的カジュアルで簡潔な表現が可能。社外文書はフォーマルで丁寧な敬語が必須。
  2. 件名、日付、宛先、差出人、本文（問い合わせ先も含めると6つ）
  3. 回覧ルート（誰から誰へ）と回覧期限
  4. 日報（日々の業務進捗報告）、週報（1週間の業務まとめ）、月報（月間実績とKPI報告）
  5. 20251030_報告書_営業実績.docx （日付_文書種類_件名.docx）
この章のまとめ
  • 社内文書は連絡系、報告系、記録系、申請系に分類される
  • 通知文や案内文は要点を明確に、誤解のない表現で記載する
  • 報告書は事実を正確に、簡潔に記載することが重要
  • 回覧文書では回覧ルートと期限を明記する
  • 頻繁に作成する文書はテンプレート化して効率化する
  • メールと書面を用途に応じて適切に使い分ける
