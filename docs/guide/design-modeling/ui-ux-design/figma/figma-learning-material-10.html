他の人と一緒に作業しよう

Figmaの強みである「共同編集」機能を学びます。ファイルの共有方法、コメント機能、リアルタイムでの共同作業について覚えましょう。

この章で学ぶこと

  • ファイルを他の人と共有する方法を理解する
  • 閲覧のみ・編集可能の権限の違いを知る
  • リアルタイム共同編集とコメント機能の使い方をマスターする
  • 変更履歴の確認とデザインのエクスポート方法を習得する

10.1 ファイルを他の人と共有する

Figmaの最大の強みは、複数人で同じファイルを同時に編集できることです。従来のデザインツールでは、ファイルをメールで送ったり、クラウドストレージに保存したりする必要がありましたが、Figmaではリンクを共有するだけで、すぐに共同作業を始められます。

共同編集可能"]

実習 10-1: ファイルを共有する

Figmaファイルを他の人と共有してみましょう。

手順

  1. 1 共有したいFigmaファイルを開きます。
  2. 2 画面右上の「Share」（共有）ボタンをクリックします。
  3. 3 共有ダイアログが開きます。「Copy link」（リンクをコピー）をクリックします。
  4. 4 リンクがクリップボードにコピーされます。このリンクをメール、チャット、Slackなどでチームメンバーやクライアントに送信します。
  5. 5 リンクを受け取った人がそのリンクをクリックすると、ブラウザでFigmaファイルが開きます。

期待される結果

✅ リンクをコピーできれば成功です。共有相手がそのリンクを開けば、あなたのデザインを見ることができます。

重要ポイント

Figmaの共有機能は非常にシンプルです。リンクを共有するだけで、相手はすぐにあなたのデザインを見られるようになります。ファイルを添付したり、ダウンロードしたりする必要はありません。

10.2 閲覧のみ・編集可能の違い

Figmaでファイルを共有する際、権限（アクセス権）を設定できます。主な権限は以下の2つです。

閲覧のみ（View）

デザインを見ることだけができます。編集はできません。

  • • デザインを見る
  • • コメントを残す
  • • 画像をエクスポートする

編集可能（Edit）

デザインを自由に編集できます。

  • • デザインを見る
  • • デザインを編集する
  • • コメントを残す
  • • 画像をエクスポートする

ヒント：権限の使い分け

閲覧のみは、クライアントや上司など「デザインを確認してフィードバックをもらいたい人」に適しています。編集可能は、チームメンバーなど「一緒にデザインを作り上げていく人」に適しています。

権限の設定方法

共有ダイアログの「Copy link」の隣にあるドロップダウンメニューから、権限を選択できます。

  • Anyone with the link can view（リンクを知っている人は閲覧のみ）
  • Anyone with the link can edit（リンクを知っている人は編集可能）

10.3 リアルタイムで一緒に編集する

Figmaでは、複数の人が同時に同じファイルを編集できます。これをリアルタイム共同編集といいます。Google Docsのように、誰かが編集しているのをリアルタイムで見ることができます。

リアルタイム共同編集とは？

複数の人が同じFigmaファイルを開いていると、画面右上に他のユーザーのアイコンが表示されます。また、誰かがオブジェクトを選択すると、その人の名前と色が表示され、どこを編集しているかがわかります。

flowchart LR A["デザイナーA"] --> B["Figmaファイル"] C["デザイナーB"] --> B D["デザイナーC"] --> B B --> E["全員が同時に
編集できる"] E --> F["変更が即座に
全員に反映"]

共同編集のメリット

  • バージョン管理が不要：ファイルを何度もやり取りする必要がありません
  • 効率的な作業：複数人で分担して作業できます
  • 即座のフィードバック：その場で確認しながら作業を進められます
  • 透明性の向上：誰が何を編集しているかがわかります

注意：編集権限には気を付けて

「編集可能」権限を持つ人は、デザインを自由に変更できます。重要なファイルを共有する際は、信頼できる人にのみ編集権限を与えましょう。クライアントなど外部の人には「閲覧のみ」権限が安全です。

10.4 コメントを付ける

Figmaにはコメント機能があり、デザインの特定の部分にメモやフィードバックを残すことができます。これは、クライアントや上司からのフィードバックを受け取るときにとても便利です。

実習 10-2: コメントを付ける

デザインにコメントを付けてみましょう。

手順

  1. 1 Figmaファイルを開き、左上のツールバーから「コメントツール」（吹き出しアイコン）をクリックします。ショートカットキーは「C」です。
  2. 2 コメントを付けたい場所をクリックします。例えば、ボタンの上やテキストの隣など、気になる部分をクリックしましょう。
  3. 3 コメント入力ボックスが表示されます。「この色をもう少し明るくしてください」「このテキストのサイズを大きくしたいです」など、具体的なフィードバックを入力します。
  4. 4 コメントを入力したら、「Post」（投稿）ボタンをクリックします。
  5. 5 コメントが表示されます。デザイン上にピンアイコンが表示され、クリックするとコメント内容が見られます。

期待される結果

✅ コメントが投稿され、デザイン上にピンアイコンが表示されればOKです。

コメントの便利な機能

  • @メンション：コメント内で「@ユーザー名」と入力すると、特定の人に通知を送れます
  • スレッド機能：コメントに返信でき、会話形式でやり取りできます
  • 解決済みマーク：対応が完了したコメントは「Resolve」（解決済み）ボタンで非表示にできます

10.5 コメントに返信する

コメントには返信することができます。これにより、デザインレビューやフィードバックを会話形式で進められます。

実習 10-3: コメントに返信する

既存のコメントに返信してみましょう。

手順

  1. 1 返信したいコメントのピンアイコンをクリックします。
  2. 2 コメントパネルが開き、既存のコメント内容が表示されます。
  3. 3 コメントの下にある「Reply」（返信）ボックスに、返信内容を入力します。例: 「わかりました！すぐに修正します」
  4. 4 「Post」（投稿）ボタンをクリックして返信を送信します。
  5. 5 返信が投稿され、元のコメントの下にスレッド形式で表示されます。

期待される結果

✅ 返信が投稿され、元のコメントと一緒にスレッド形式で表示されればOKです。

コメントを解決済みにする

フィードバックに対応したら、コメントを「解決済み」にして整理しましょう。

  1. 対応が完了したコメントを開きます
  2. 「Resolve」（解決済み）ボタンをクリックします
  3. コメントが非表示になります（後で「Show resolved comments」で確認可能）

10.6 変更履歴を確認する

Figmaは、ファイルの変更履歴を自動的に保存しています。過去のバージョンに戻したり、誰がいつ何を変更したかを確認したりできます。

実習 10-4: 変更履歴を確認する

ファイルの変更履歴を確認してみましょう。

手順

  1. 1 Figmaファイルを開き、画面上部のファイル名の横にある「▼」（下矢印）をクリックします。
  2. 2 メニューが開くので、「Show version history」（バージョン履歴を表示）をクリックします。
  3. 3 画面右側に変更履歴パネルが表示されます。日時順にバージョンが並んでいます。
  4. 4 任意のバージョンをクリックすると、その時点のデザインを確認できます。
  5. 5 過去のバージョンに戻したい場合は、「Restore this version」（このバージョンを復元）をクリックします。

期待される結果

✅ 変更履歴パネルが表示され、過去のバージョンを確認できればOKです。

ヒント：無料プランの制限

無料プラン（Starter）では、変更履歴は30日間保存されます。有料プラン（Professional）では無制限に保存されます。重要なマイルストーンは「Save to version history」でバージョンに名前を付けて保存しておくと便利です。

10.7 デザインをエクスポート（書き出し）する

完成したデザインを画像ファイルとして書き出すことを「エクスポート」といいます。SNSに投稿したり、プレゼン資料に使ったり、開発者に渡したりする際に必要です。

実習 10-5: デザインを画像として書き出す

フレームやオブジェクトを画像ファイルとしてエクスポートしてみましょう。

手順

  1. 1 書き出したいフレームやオブジェクトを選択します。
  2. 2 画面右側のプロパティパネルを下にスクロールし、「Export」（エクスポート）セクションを探します。
  3. 3 「+」（プラス）ボタンをクリックして、エクスポート設定を追加します。
  4. 4 ファイル形式（PNG、JPG、SVG、PDFなど）を選択します。一般的にはPNGが推奨されます。
  5. 5 「Export [オブジェクト名]」ボタンをクリックします。
  6. 6 画像ファイルがダウンロードされます。ファイルを開いて確認してみましょう。

期待される結果

✅ 画像ファイル（PNG、JPGなど）がダウンロードされ、開いて確認できればOKです。

主なファイル形式

形式 特徴 用途
PNG 背景透過可能、高画質 SNS投稿、Web用画像、アイコン
JPG ファイルサイズが小さい 写真、Web用画像
SVG 拡大しても劣化しない ロゴ、アイコン、イラスト
PDF 印刷に適している 印刷物、プレゼン資料

ヒント：書き出しサイズの調整

Exportセクションでは、「1x」「2x」「3x」など、書き出しサイズを倍率で指定できます。例えば「2x」にすると、元のサイズの2倍の解像度で書き出されます。高解像度ディスプレイ（Retina）用の画像を作る際に便利です。

理解度確認クイズ

  1. Q1. Figmaでファイルを共有する最も簡単な方法は何ですか？

    回答を表示

    正解：「Share」ボタンをクリックして、リンクをコピーして共有する
    Figmaでは、Shareボタンからリンクをコピーして相手に送るだけで、簡単にファイルを共有できます。メールやチャットでリンクを送るだけでOKです。

  2. Q2. 「閲覧のみ」権限と「編集可能」権限の違いは何ですか？

    回答を表示

    正解：閲覧のみはデザインを見るだけ、編集可能はデザインを変更できる
    「閲覧のみ」はデザインを見てコメントを残すことができますが、デザインの編集はできません。「編集可能」はデザインを自由に変更できます。クライアントには閲覧のみ、チームメンバーには編集可能を与えることが一般的です。

  3. Q3. コメント機能で、特定の人に通知を送るにはどうすればよいですか？

    回答を表示

    正解：コメント内で「@ユーザー名」とメンションする
    コメント入力時に「@」を入力すると、ユーザー名の候補が表示されます。特定のユーザーを選択すると、その人に通知が届きます。

  4. Q4. 完成したデザインを画像ファイルとして保存するには、どの機能を使いますか？

    回答を表示

    正解：Export（エクスポート）機能
    デザインを画像ファイル（PNG、JPG、SVG、PDFなど）として書き出すには、プロパティパネルの「Export」セクションを使います。ファイル形式を選択して「Export」ボタンをクリックするだけです。

トラブルシューティング

Q: 共有リンクを開いても「アクセス権がない」と表示される

A: 共有リンクの権限設定を確認してください。「Anyone with the link can view/edit」に設定されているか確認しましょう。もしくは、相手のメールアドレスを直接招待することもできます。

Q: コメントツールが見つからない

A: 左上のツールバーにある吹き出しアイコンがコメントツールです。見つからない場合は、キーボードの「C」キーを押すと、コメントツールに切り替わります。

Q: エクスポートしたPNG画像が低画質に見える

A: エクスポート設定で「2x」や「3x」などの倍率を選択すると、高解像度で書き出せます。Retina対応ディスプレイで見る場合は、2x以上を推奨します。

Q: 変更履歴が見つからない

A: 画面上部のファイル名の横にある「▼」（下矢印）をクリックし、「Show version history」を選択してください。無料プランでは30日間の履歴が保存されます。

第10章のまとめ

  • Figmaの共有機能は、リンクをコピーして送るだけで簡単にファイルを共有できる
  • 閲覧のみと編集可能の権限を使い分けることで、適切なアクセス管理ができる
  • リアルタイム共同編集により、複数人が同時に同じファイルを編集できる
  • コメント機能で、デザインの特定の部分にフィードバックを残せる
  • 変更履歴を確認して、過去のバージョンに戻すことができる（無料プランは30日間）
  • エクスポート機能で、デザインをPNG、JPG、SVG、PDFなどの画像ファイルとして書き出せる
