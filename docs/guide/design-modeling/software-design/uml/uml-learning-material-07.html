シーケンス図の応用 - 条件分岐とループの表現

シーケンス図でより複雑な処理を表現する方法を学びます。「もし～なら」という条件分岐（alt/opt）、「繰り返し」のループ（loop）、並行処理（par）などの複合フラグメントを使いこなす方法を習得します。

この章で学ぶこと

  • 複合フラグメントとは何かを理解する
  • 条件分岐（alt/opt）の表現方法を習得する
  • 繰り返し（loop）と並行処理（par）の書き方を学ぶ

7.1 複合フラグメントとは

第6章で学んだ基本的なシーケンス図では、メッセージが順番に流れる単純な処理を表現しました。 しかし実際のシステムでは、「条件によって処理が分岐する」「同じ処理を繰り返す」「複数の処理を並行して行う」といった複雑な流れが必要になります。

複合フラグメント（Combined Fragment）は、こうした複雑な制御構造をシーケンス図で表現するための仕組みです。 四角い枠で囲み、左上に演算子（operator）を記述することで、どのような制御を表すかを示します。

主な複合フラグメント演算子

  • alt：条件分岐（if-else）
  • opt：オプション（if のみ）
  • loop：繰り返し（for/while）
  • par：並行処理（parallel）
  • ref：他のシーケンス図の参照

7.2 条件分岐の表現（alt）

alt（alternative）は、「もし～なら○○、そうでなければ××」というif-else構造を表現します。 枠内を点線で区切り、各領域にガード条件（条件式）を記述します。

例：ログイン処理の条件分岐

ユーザーがログイン情報を入力し、正しければトップページへ、間違っていればエラーメッセージを表示する処理を考えます。

sequenceDiagram autonumber participant U as ユーザー participant W as Web画面 participant S as 認証サービス participant DB as ユーザーDB U->>W: ID・パスワード入力 W->>S: 認証要求 S->>DB: ユーザー情報取得 DB-->>S: ユーザー情報 alt パスワード一致 S-->>W: 認証成功 W-->>U: トップページ表示 else パスワード不一致 S-->>W: 認証失敗 W-->>U: エラーメッセージ表示 end

ポイント

altフラグメントは、点線で区切られた各領域に「どの条件のときか」を明記します。 プログラミングのif-else文と同じように、どちらか一方の処理だけが実行されます。

7.3 オプション処理の表現（opt）

opt（option）は、「もし～なら○○する」というif文（elseなし）を表現します。 条件が満たされたときだけ処理を実行し、満たされなければ何もしません。

例：商品在庫確認

商品の在庫が少なくなったときだけ、管理者に通知を送る処理を考えます。

sequenceDiagram autonumber participant C as 顧客 participant O as 注文システム participant I as 在庫管理 participant N as 通知サービス C->>O: 商品購入 O->>I: 在庫減算 I-->>O: 在庫更新完了 opt 在庫が閾値以下 I->>N: 在庫アラート送信 N-->>I: 通知完了 end O-->>C: 購入完了

altとoptの使い分け

alt：条件が真の場合と偽の場合で異なる処理をする（if-else）
opt：条件が真の場合だけ処理をする（if のみ）

7.4 繰り返し処理の表現（loop）

loopは、「○○を繰り返す」というfor文やwhile文を表現します。 ガード条件に繰り返しの条件（回数や条件式）を記述します。

例：メール一括送信

顧客リストを取得し、各顧客に対してメールを送信する処理を考えます。

sequenceDiagram autonumber participant A as 管理者 participant S as メール送信システム participant DB as 顧客DB participant M as メールサーバー A->>S: 一括送信開始 S->>DB: 顧客リスト取得 DB-->>S: 顧客リスト loop 各顧客に対して S->>M: メール送信 M-->>S: 送信完了 end S-->>A: 全件送信完了

繰り返し条件の書き方

  • loop [for each 顧客] - 各要素に対して
  • loop [1..10] - 10回繰り返し
  • loop [while 在庫 > 0] - 条件が真の間

7.5 並行処理の表現（par）

par（parallel）は、「複数の処理を同時に実行する」という並行処理を表現します。 枠内を点線で区切り、各領域の処理が並行して行われることを示します。

例：商品注文時の並行処理

注文確定後、在庫確保と決済処理を同時に開始する処理を考えます。

sequenceDiagram autonumber participant C as 顧客 participant O as 注文システム participant I as 在庫システム participant P as 決済システム C->>O: 注文確定 par 並行処理 O->>I: 在庫確保 I-->>O: 確保完了 and O->>P: 決済実行 P-->>O: 決済完了 end O-->>C: 注文完了通知

注意

並行処理では、どちらの処理が先に終わるかは保証されません。 両方の処理が完了してから次の処理に進む場合は、parフラグメントの外に記述します。

7.6 複合フラグメントの組み合わせ

複合フラグメントは入れ子にして組み合わせることができます。 例えば、ループの中に条件分岐を含めたり、並行処理の中に繰り返しを含めたりすることが可能です。

例：バッチ処理での組み合わせ

複数のファイルを処理する際に、各ファイルに対してエラーチェックを行い、エラーがあれば通知する処理を考えます。

sequenceDiagram autonumber participant B as バッチシステム participant F as ファイルシステム participant V as 検証サービス participant N as 通知サービス B->>F: ファイルリスト取得 F-->>B: ファイルリスト loop 各ファイルに対して B->>F: ファイル読み込み F-->>B: ファイルデータ B->>V: データ検証 V-->>B: 検証結果 alt エラーあり B->>N: エラー通知 N-->>B: 通知完了 else エラーなし B->>B: データ処理 end end B->>N: 処理完了通知

重要ポイント

複合フラグメントを組み合わせることで、実際のビジネスロジックに近い複雑な処理フローを表現できます。 ただし、複雑になりすぎると読みにくくなるため、必要に応じて複数のシーケンス図に分割することも検討しましょう。

実習 7-1: ATMでの出金処理シーケンス図

ATMで現金を引き出す処理を、複合フラグメントを使ってシーケンス図で表現してみましょう。 以下の条件を含めてください：

  • 暗証番号が正しいかチェック（alt）
  • 残高が出金額以上あるかチェック（opt）
  • 正しければ現金を払い出す

手順

  1. 1 draw.ioまたはPlantUMLを開く
  2. 2 参加者を配置：ユーザー、ATM、銀行システム、口座DB
  3. 3 altフラグメントで暗証番号チェックの分岐を表現
  4. 4 optフラグメントで残高チェックを表現
  5. 5 完成した図を保存

理解度確認クイズ

  1. Q1. 複合フラグメントとは何ですか？

    回答を表示

    複合フラグメントは、シーケンス図で条件分岐、繰り返し、並行処理などの複雑な制御構造を表現するための仕組みです。 四角い枠で囲み、左上に演算子（alt、opt、loop、parなど）を記述して使用します。

  2. Q2. altとoptの違いは何ですか？

    回答を表示

    altは条件が真の場合と偽の場合で異なる処理を行う（if-else）構造を表します。
    optは条件が真の場合だけ処理を行う（if のみ）構造を表します。

  3. Q3. 並行処理（par）を使う場面の例を挙げてください。

    回答を表示

    例：ECサイトで注文確定後、在庫確保と決済処理を同時に開始する場合。 データ処理で複数のAPIを同時に呼び出す場合。 メール送信とログ記録を並行して行う場合など。 parは、処理の順序が保証されない、または同時に実行される処理を表現するときに使います。

第7章のまとめ

  • 複合フラグメントを使うと、条件分岐・繰り返し・並行処理を表現できます
  • alt（条件分岐）はif-else、opt（オプション）はif のみを表します
  • loop（繰り返し）でfor/while文を表現できます
  • par（並行処理）で同時実行される処理を表現できます
  • 複合フラグメントは入れ子にして組み合わせることができます
