1.1 UMLとは何か

UML（Unified Modeling Language：統一モデリング言語）は、ソフトウェアシステムの設計を視覚的に表現するための標準的な記法です。1990年代に開発され、現在はObject Management Group（OMG）によって標準化されています。

UMLの主な特徴：

  • 視覚的表現：複雑なシステムを図として表現
  • 標準化：業界標準として広く受け入れられている
  • 多目的：分析から実装まで幅広い用途に対応
  • オブジェクト指向：オブジェクト指向設計の概念をサポート
flowchart TD A["ソフトウェア開発の課題"] --> B["複雑性の増大"] A --> C["チーム間の意思疎通"] A --> D["設計の文書化"] B --> E["UMLによる視覚化"] C --> E D --> E E --> F["理解しやすい設計"] E --> G["効率的な開発"] E --> H["保守性の向上"]

1.3 モデリングの重要性

ソフトウェア開発においてモデリングが重要な理由：

モデリングの効果
  • 理解の促進：複雑な概念を視覚的に表現
  • コミュニケーション向上：チーム間での共通認識の形成
  • 設計品質の向上：事前の設計検討による品質確保
  • 開発効率の向上：早期の問題発見と修正

1.5 基本記法ルールと命名規則

UMLを正しく使うための基本ルール：

命名規則

  • クラス名：PascalCase（例：CustomerOrder）
  • 属性名：camelCase（例：firstName）
  • 操作名：camelCase（例：calculateTotal()）
  • 定数：ALL_CAPS（例：MAX_COUNT）

可視性の記法

  • + public（公開）
  • - private（非公開）
  • # protected（保護）
  • ~ package（パッケージ）
classDiagram class Customer { -customerId : int -firstName : String -lastName : String #email : String +getName() : String +setEmail(email : String) : void -validateEmail(email : String) : boolean }

1.6 モデリングツールの選定

UML図を作成するためのツールは多数存在します。用途に応じて適切なツールを選択することが重要です。

主要なUMLツール
無料ツール
  • PlantUML：テキストベースの図作成、プログラマー向け
  • draw.io（diagrams.net）：Web-based、初心者向け
  • StarUML：本格的な機能、商用制限あり
商用ツール
  • Enterprise Architect：エンタープライズ向け、高機能
  • Visual Paradigm：幅広い図に対応
  • Lucidchart：クラウドベース、チーム協作
実習 1-2: ツールの比較と選択

手順
  1. draw.ioにアクセス
  2. 新しい図を作成（UMLテンプレートを選択）
  3. 簡単なクラス図を作成してみる

PlantUMLの例

1.7 UMLの活用場面

UMLは開発の様々な段階で活用できます：

flowchart LR A[要件分析] --> B[設計] B --> C[実装] C --> D[テスト] D --> E[保守] A -.-> F[ユースケース図] B -.-> G[クラス図] B -.-> H[シーケンス図] C -.-> I[コンポーネント図] D -.-> J[アクティビティ図] E -.-> K[デプロイメント図]
注意点

UMLは万能ではありません。以下の点に注意しましょう：

  • すべてを詳細に記述する必要はない
  • 目的に応じて適切な図を選択する
  • 図の更新を忘れないよう管理する
  • コミュニケーションツールとして活用する
