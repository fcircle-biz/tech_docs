第3章

ユースケース図入門 - システムの「使われ方」を描く

最も直感的で入門しやすいユースケース図から学習を始めます。「誰が」「何をするか」を図で表現する方法を、ATM、ECサイト、図書館システムなど身近な例を使って習得します。

この章で学ぶこと

  • ユースケース図の役割と基本構成要素を理解する
  • アクター、ユースケース、システム境界の概念を理解する
  • 身近なシステムをユースケース図で表現できるようになる

3.1 ユースケース図とは

ユースケース図（Use Case Diagram）は、システムが「誰に」「どのように使われるか」を視覚化したUML図です。 システムの外部から見た機能を、利用者の視点で整理し表現します。

開発の早い段階、特に要件定義のフェーズで活用され、システムが提供すべき機能の全体像を把握するのに役立ちます。 技術者だけでなく、顧客やビジネス部門の担当者にも理解しやすい図として重宝されます。

ユースケース図の目的

  • 利用者の視点：システムを「使う人」の立場で機能を整理
  • 機能の洗い出し：システムが提供すべき機能を漏れなく把握
  • 関係者との共有：非技術者にも分かりやすい形でシステムを説明

3.2 基本要素

ユースケース図は、以下の3つの基本要素で構成されています：

1. アクター（Actor）

システムを利用する人や外部システムを表します。棒人間の図形で表現されます。

例：顧客、管理者、決済システム、メール配信サービス

2. ユースケース（Use Case）

システムが提供する機能を表します。楕円形で表現し、動詞で命名します。

例：商品を検索する、注文を確定する、在庫を管理する

3. システム境界（System Boundary）

システムの範囲を明確にする四角形の枠です。どこまでがシステムの責任範囲かを示します。

例：ECサイトシステム、図書館管理システム

3.3 基本的な記法

ユースケース図では、アクターとユースケースを線で結び、関係性を表現します。

関連（Association）

アクターとユースケースを直線で結びます

顧客 ──→ 商品を検索する

多重度（Multiplicity）

利用回数や関係性の数を示します（省略可）

1人の顧客が0回以上利用

重要ポイント

ユースケースの命名は「〜する」という動詞形で表現します。 これにより、システムが提供する「機能」であることが明確になります。 例：「商品」ではなく「商品を検索する」

実習 3-1: ユースケース図を描いてみよう

draw.ioまたはPlantUMLを使って、以下のシステムのユースケース図を描いてみましょう。

課題：レストラン予約システム

システムの概要：

  • 顧客は、空席を検索し、予約を作成できる
  • 顧客は、予約を変更・キャンセルできる
  • 店員は、予約を確認し、来店処理を行う
  • 店員は、テーブル配置を管理する
  • 予約確認メールが自動送信される（メール配信サービス）

手順

  1. 1 draw.io（https://app.diagrams.net/）を開き、新規ファイルを作成
  2. 2 左側のパネルから「UML」→「Use Case」を選択
  3. 3 システム境界の四角形を配置し、「レストラン予約システム」と名前を付ける
  4. 4 アクターを配置する（顧客、店員、メール配信サービス）
  5. 5 ユースケースを楕円で配置し、それぞれに名前を付ける（動詞形で）
  6. 6 アクターとユースケースを線で結ぶ

理解度確認クイズ

  1. Q1. ユースケース図の3つの基本要素は何ですか？

    回答を表示

    1. アクター（Actor）：システムを利用する人や外部システム
    2. ユースケース（Use Case）：システムが提供する機能（楕円形）
    3. システム境界（System Boundary）：システムの範囲を示す枠

  2. Q2. アクターは「人」だけですか？それとも他のものも含まれますか？

    回答を表示

    アクターは「人」だけでなく、外部システムも含まれます。 例えば、決済サービス、メール配信システム、他社のAPIなども、システムと相互作用する存在としてアクターになります。

  3. Q3. ユースケースの命名は、なぜ動詞形にする必要がありますか？

    回答を表示

    ユースケースはシステムが提供する「機能」や「動作」を表すためです。 動詞形（「〜する」）で表現することで、「何ができるか」が明確になります。 例：「商品」ではなく「商品を検索する」「商品を登録する」など。

  4. Q4. ユースケース図は開発のどのフェーズで最も活用されますか？

    回答を表示

    主に要件定義フェーズで活用されます。 システムが「誰に」「どのように使われるか」を整理し、必要な機能を洗い出すために使用します。 顧客やビジネス部門との認識合わせにも有効です。

第3章のまとめ

  • ユースケース図は、システムの「使われ方」を視覚化したUML図です
  • 3つの基本要素：アクター、ユースケース、システム境界で構成されます
  • アクターは人だけでなく、外部システムも含まれます
  • ユースケースは「〜する」という動詞形で命名します
  • ATM、ECサイト、図書館などの身近なシステムをユースケース図で表現できます
  • 要件定義フェーズで、関係者との認識合わせに活用されます
