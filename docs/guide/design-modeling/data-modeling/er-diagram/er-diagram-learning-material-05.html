リレーションシップ入門 - エンティティ同士のつながりを描く

ER図の核心である「リレーションシップ（関連）」を理解し、draw.ioで実際に描いてみましょう。

この章で学ぶこと

  • リレーションシップ（関連）とは何かを理解する
  • エンティティ間の関係性を線で表現する方法を習得する
  • リレーションシップの見つけ方と名前の付け方を学ぶ
  • draw.ioでリレーションシップを描く実践練習を行う

5.1 リレーションシップとは何か

これまでの章で、エンティティ（管理したい「もの」や「こと」）と属性（エンティティが持つ詳細情報）を学びました。 第5章では、ER図の最も重要な要素である「リレーションシップ（Relationship）」を学びます。

リレーションシップ（Relationship）とは

エンティティ同士の「関連性」「つながり」を表すものです。 例えば、「顧客が商品を注文する」「社員が部署に所属する」といった関係性を視覚化します。

なぜリレーションシップが重要なのか

エンティティと属性だけでは、データの「静的な構造」しか表現できません。 リレーションシップがあることで、「どのデータとどのデータが関連しているか」という動的な関係性を明確にできます。

リレーションシップがない場合

顧客と注文が別々に存在するだけで、「どの顧客がどの注文をしたか」が分かりません。

リレーションシップがある場合

顧客と注文の間に「注文する」という関連が明確になり、業務の流れが理解できます。

5.2 リレーションシップの表現方法

ER図では、リレーションシップを線（エッジ）で表現します。 エンティティ同士を線でつなぐことで、その関係性を視覚化します。

基本的なリレーションシップの例

erDiagram CUSTOMER ||--o{ ORDER : places CUSTOMER { string customer_id PK string name string email } ORDER { string order_id PK date order_date string customer_id FK }

顧客（CUSTOMER）エンティティと注文（ORDER）エンティティの間に「places（注文する）」というリレーションシップがあります。

リレーションシップの要素

1

接続する2つのエンティティ

例：「顧客」エンティティと「注文」エンティティ

2

リレーションシップ名（動詞）

例：「注文する」「所属する」「含む」など

3

カーディナリティ（次章で詳しく学びます）

例：「1対1」「1対多」「多対多」などの数量関係

5.3 リレーションシップの見つけ方

業務要件からリレーションシップを見つけるには、「エンティティ同士の動作や関係を表す動詞」に着目します。

リレーションシップを見つけるヒント

  • 「〜が〜を〜する」という文章を作ってみる
  • 「〜が〜に〜される」という受け身の関係も考える
  • 「〜が〜に所属する」「〜が〜を含む」といった包含関係に着目する

具体例で見るリレーションシップの抽出

業務要件の例

「オンライン書店では、顧客が書籍を購入します。 各書籍は著者が執筆しており、一人の著者が複数の書籍を執筆することもあります。 また、書籍はカテゴリーに分類されています。」

抽出されるエンティティとリレーションシップ

エンティティ1 リレーションシップ エンティティ2
顧客 購入する 書籍
著者 執筆する 書籍
書籍 分類される カテゴリー

オンライン書店のER図

erDiagram CUSTOMER ||--o{ PURCHASE : makes PURCHASE }o--|| BOOK : includes AUTHOR ||--o{ BOOK : writes BOOK }o--|| CATEGORY : belongs_to CUSTOMER { string customer_id PK string name string email } BOOK { string book_id PK string title decimal price string author_id FK string category_id FK } AUTHOR { string author_id PK string author_name } CATEGORY { string category_id PK string category_name } PURCHASE { string purchase_id PK date purchase_date string customer_id FK string book_id FK }

5.4 リレーションシップ名の付け方

リレーションシップには、その関係性を表す名前（動詞）を付けることが推奨されます。 名前を付けることで、エンティティ間の関係がより明確になります。

良いリレーションシップ名の例

「顧客」が「注文」を 「注文する（places）」

→ 主語と目的語の関係が明確

「社員」が「部署」に 「所属する（belongs_to）」

→ 包含関係が分かりやすい

「注文」が「商品」を 「含む（contains）」

→ 構成関係が理解しやすい

避けるべきリレーションシップ名

「関連」「関係」など抽象的すぎる名前

→ 具体的な業務の流れが伝わらない

名詞形（「注文」「所属」など）

→ エンティティ名と混同しやすい

「has」「have」など英語の所有を表す動詞

→ 日本語で「持つ」は曖昧なので、より具体的な動詞を使う

双方向の読み方を考える

リレーションシップは両方向から読めるように名前を付けると良いです。 例：「顧客が注文を注文する」「注文が顧客に注文される」

5.5 draw.ioでリレーションシップを描く実習

それでは、draw.ioを使って実際にリレーションシップを描いてみましょう。 ここでは、「社員管理システム」のER図を作成します。

実習：社員管理システムのER図を作成する

業務要件

  • 会社には複数の部署があります
  • 社員は必ず1つの部署に所属します
  • 社員は複数のプロジェクトに参加できます
  • プロジェクトには複数の社員が参加します

手順1: エンティティを配置する

  1. draw.ioを開く（https://app.diagrams.net/）
  2. 左側のシェイプパネルから「Entity Relation」を選択
  3. 「Entity」シェイプを3つ配置し、それぞれ「社員」「部署」「プロジェクト」と名前を付ける

手順2: リレーションシップを描く

  1. 「社員」エンティティから「部署」エンティティに線を引く（所属する）
  2. 「社員」エンティティと「プロジェクト」エンティティの間に線を引く（参加する）
  3. 線をダブルクリックし、リレーションシップ名を入力（例：「所属する」「参加する」）

手順3: 属性を追加する（オプション）

  1. 各エンティティに主要な属性を追加（社員ID、部署ID、プロジェクトIDなど）
  2. 主キー（PK）を設定する

完成例

erDiagram EMPLOYEE }o--|| DEPARTMENT : belongs_to EMPLOYEE }o--o{ PROJECT : participates_in EMPLOYEE { string employee_id PK string name date hire_date string department_id FK } DEPARTMENT { string department_id PK string department_name string location } PROJECT { string project_id PK string project_name date start_date date end_date }

社員は1つの部署に所属し、複数のプロジェクトに参加できる関係性を表現しています。

draw.ioでのリレーションシップ描画のコツ

  • 線は直線だけでなく、曲線や折れ線も使えます（右クリック → スタイル）
  • 線の太さや色を変えて、重要なリレーションシップを強調できます
  • リレーションシップ名は線の中央に配置すると見やすくなります
  • 複雑なER図では、エンティティの配置を工夫して線が交差しないようにしましょう

5.6 よくあるリレーションシップのパターン

実務でよく見られるリレーションシップのパターンをいくつか紹介します。

パターン1: 所属・階層関係

「〜が〜に所属する」という関係。組織構造やカテゴリー分類でよく使われます。

例 エンティティ1 リレーションシップ エンティティ2
組織 社員 所属する 部署
商品分類 商品 属する カテゴリー

パターン2: 発注・取引関係

「〜が〜を注文/購入/発注する」という関係。ECサイトや受発注システムで頻出します。

例 エンティティ1 リレーションシップ エンティティ2
ECサイト 顧客 注文する 商品
在庫管理 仕入先 納品する 商品

パターン3: 作成・執筆関係

「〜が〜を作成/執筆/投稿する」という関係。コンテンツ管理システムでよく見られます。

例 エンティティ1 リレーションシップ エンティティ2
ブログ 著者 投稿する 記事
SNS ユーザー 投稿する コメント

理解度確認クイズ

  1. Q1. リレーションシップとは何ですか？簡潔に説明してください。

    回答を表示

    リレーションシップとは、エンティティ同士の関連性やつながりを表すものです。

    例えば、「顧客が商品を注文する」「社員が部署に所属する」といった業務上の関係性を、ER図上で線（エッジ）を使って視覚化します。

  2. Q2. 以下の業務要件から、エンティティとリレーションシップを抽出してください。
    「学生は複数の授業を履修できる。授業には複数の学生が参加する。」

    回答を表示

    エンティティ：

    • 学生（STUDENT）
    • 授業（COURSE）

    リレーションシップ：

    • 学生が授業を「履修する（enrolls_in）」
    • または、授業に学生が「参加する（attended_by）」

  3. Q3. リレーションシップ名を付ける際に、避けるべきものはどれですか？

    1. 動詞を使う（「注文する」「所属する」など）
    2. 名詞を使う（「注文」「所属」など）
    3. 業務の流れが伝わる具体的な言葉を使う
    4. 双方向から読めるように配慮する
    回答を表示

    2. 名詞を使うが避けるべきものです。

    リレーションシップ名は動詞で表現することが推奨されます。名詞を使うとエンティティ名と混同しやすく、関係性が明確に伝わりません。

第5章のまとめ

  • リレーションシップは、エンティティ同士の関連性を表す重要な要素である
  • ER図では、リレーションシップを線（エッジ）で表現する
  • 業務要件から「〜が〜を〜する」という動詞を抽出することでリレーションシップを見つけられる
  • リレーションシップ名は動詞形で付け、具体的で分かりやすい名前を選ぶ
  • draw.ioでは、エンティティ間に線を引き、ダブルクリックで名前を付けることができる
  • 「所属」「発注」「作成」などのパターンを理解することで、リレーションシップの抽出が容易になる
