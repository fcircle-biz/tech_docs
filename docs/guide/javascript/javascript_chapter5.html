第5章：非同期処理（Promise・async/await）

現代のWebアプリケーションでは、サーバーからデータを取得したり、時間のかかる処理を行う際に非同期処理が欠かせません。この章では、JavaScriptの非同期処理を効率的に扱う方法を学習し、実際にAPIを呼び出すアプリケーションを作成します。

5.1 非同期処理の基礎

非同期処理とは、時間のかかる処理を実行している間に、他の処理を並行して実行できる仕組みです。

同期処理 vs 非同期処理

図5.1: 同期処理と非同期処理の違い

JavaScriptにおける非同期処理の例

// 同期処理の例 console.log('処理1開始'); console.log('処理2開始'); console.log('処理3開始'); // 非同期処理の例 console.log('処理1開始'); setTimeout(() => { console.log('2秒後に実行される処理'); }, 2000); console.log('処理3開始（すぐに実行される）');
実行結果：
処理1開始
処理3開始（すぐに実行される）
（2秒後）
2秒後に実行される処理
実習 5.1.1: setTimeoutによる非同期処理の体験
  1. ブラウザのコンソールを開く
  2. 上記のコードを実行し、実行順序を確認する
  3. 以下のコードも試してみる：
console.log('開始'); setTimeout(() => console.log('1000ms後'), 1000); setTimeout(() => console.log('500ms後'), 500); setTimeout(() => console.log('0ms後'), 0); console.log('終了');

5.2 コールバック地獄とPromise

非同期処理を連続して行う場合、従来のコールバック関数では「コールバック地獄」という問題が発生していました。

コールバック地獄の例

// ユーザー情報を取得→プロフィールを取得→投稿を取得 getUserData(userId, (userData) => { getProfile(userData.id, (profile) => { getPosts(profile.id, (posts) => { getComments(posts[0].id, (comments) => { // ネストが深くなりすぎて読みにくい console.log('コメント:', comments); }); }); }); });

Promiseによる解決

ES6で導入されたPromiseは、非同期処理をより読みやすく記述できる仕組みです。

// Promiseを使った書き方 getUserData(userId) .then(userData => getProfile(userData.id)) .then(profile => getPosts(profile.id)) .then(posts => getComments(posts[0].id)) .then(comments => { console.log('コメント:', comments); }) .catch(error => { console.error('エラーが発生:', error); });
Promiseの3つの状態
  • Pending（保留中）：処理が実行中の状態
  • Fulfilled（成功）：処理が成功して完了した状態
  • Rejected（失敗）：処理が失敗した状態

5.3 Promiseの基本

Promiseの基本的な使い方とメソッドについて学習します。

Promiseの作成

// 基本的なPromiseの作成 const myPromise = new Promise((resolve, reject) => { // 非同期処理をシミュレート setTimeout(() => { const success = Math.random() > 0.5; if (success) { resolve('処理が成功しました！'); } else { reject('処理が失敗しました...'); } }, 1000); }); // Promiseの使用 myPromise .then(result => { console.log('成功:', result); }) .catch(error => { console.log('エラー:', error); });

Promise.allとPromise.race

// 複数のPromiseを並行実行 const promise1 = fetch('/api/user'); const promise2 = fetch('/api/posts'); const promise3 = fetch('/api/comments'); // すべてのPromiseが完了するまで待つ Promise.all([promise1, promise2, promise3]) .then(responses => { console.log('すべての処理が完了:', responses); }) .catch(error => { console.log('いずれかの処理が失敗:', error); }); // 最初に完了したPromiseの結果を取得 Promise.race([promise1, promise2, promise3]) .then(firstResponse => { console.log('最初に完了した処理:', firstResponse); });
実習 5.3.1: Promiseの作成と使用

以下のPromiseベースの関数を作成し、使用してみましょう：

// ランダムな時間後に数値を返すPromise function getRandomNumber(max) { return new Promise((resolve) => { const delay = Math.random() * 2000; // 0-2秒のランダム setTimeout(() => { const number = Math.floor(Math.random() * max); resolve(number); }, delay); }); } // 使用例 getRandomNumber(100) .then(number => { console.log('取得した数値:', number); return number * 2; // 次のthenに値を渡す }) .then(doubled => { console.log('2倍した値:', doubled); });

5.4 Fetch APIの使い方

Fetch APIは、HTTP通信を行うためのモダンなJavaScript APIです。XMLHttpRequestの後継として開発され、Promiseベースで使いやすく設計されています。

基本的なFetch API

// 基本的なGETリクエスト fetch('https://jsonplaceholder.typicode.com/posts/1') .then(response => { console.log('レスポンス受信:', response); return response.json(); // JSONとして解析 }) .then(data => { console.log('データ:', data); }) .catch(error => { console.error('エラー:', error); });

レスポンスの状態確認

fetch('https://jsonplaceholder.typicode.com/posts/1') .then(response => { console.log('ステータスコード:', response.status); console.log('成功:', response.ok); if (!response.ok) { throw new Error('ネットワークエラー: ' + response.status); } return response.json(); }) .then(data => { console.log('投稿データ:', data); }) .catch(error => { console.error('エラーが発生:', error); });

POSTリクエストの送信

// データを送信する例 const postData = { title: 'JavaScript学習', body: 'Fetch APIの使い方を学習中です', userId: 1 }; fetch('https://jsonplaceholder.typicode.com/posts', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify(postData) }) .then(response => response.json()) .then(data => { console.log('投稿完了:', data); }) .catch(error => { console.error('投稿エラー:', error); });
メソッド 用途 データ送信
GET データの取得 URL パラメータ
POST 新しいデータの作成 リクエストボディ
PUT データの更新（置換） リクエストボディ
DELETE データの削除 通常なし
実習 5.4.1: JSONPlaceholder APIの利用

無料のテスト用API「JSONPlaceholder」を使って、データの取得と投稿を試してみましょう：

// ユーザー一覧を取得 function getUsers() { fetch('https://jsonplaceholder.typicode.com/users') .then(response => response.json()) .then(users => { console.log('ユーザー数:', users.length); users.forEach(user => { console.log(`${user.id}: ${user.name} (${user.email})`); }); }) .catch(error => console.error('エラー:', error)); } getUsers();

5.5 async/await

ES2017で導入されたasync/awaitは、非同期処理をより直感的に記述できる構文です。Promiseの上に構築されており、同期処理のような書き方で非同期処理を扱えます。

async関数の基本

// async関数の宣言 async function fetchUserData() { // await キーワードでPromiseの完了を待つ const response = await fetch('https://jsonplaceholder.typicode.com/users/1'); const user = await response.json(); console.log('ユーザー情報:', user); return user; } // async関数は常にPromiseを返す fetchUserData() .then(user => console.log('関数の戻り値:', user)) .catch(error => console.error('エラー:', error));

Promiseチェーンとasync/awaitの比較

// Promiseチェーンでの記述 function getUserPosts(userId) { return fetch(`https://jsonplaceholder.typicode.com/users/${userId}`) .then(response => response.json()) .then(user => { console.log('ユーザー:', user.name); return fetch(`https://jsonplaceholder.typicode.com/posts?userId=${userId}`); }) .then(response => response.json()) .then(posts => { console.log('投稿数:', posts.length); return posts; }); } // async/awaitでの記述（より読みやすい） async function getUserPostsAsync(userId) { const userResponse = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`); const user = await userResponse.json(); console.log('ユーザー:', user.name); const postsResponse = await fetch(`https://jsonplaceholder.typicode.com/posts?userId=${userId}`); const posts = await postsResponse.json(); console.log('投稿数:', posts.length); return posts; }

複数の非同期処理の並行実行

// 順次実行（遅い） async function sequentialFetch() { const user = await fetch('https://jsonplaceholder.typicode.com/users/1').then(r => r.json()); const posts = await fetch('https://jsonplaceholder.typicode.com/posts?userId=1').then(r => r.json()); const albums = await fetch('https://jsonplaceholder.typicode.com/albums?userId=1').then(r => r.json()); return { user, posts, albums }; } // 並行実行（速い） async function concurrentFetch() { const [user, posts, albums] = await Promise.all([ fetch('https://jsonplaceholder.typicode.com/users/1').then(r => r.json()), fetch('https://jsonplaceholder.typicode.com/posts?userId=1').then(r => r.json()), fetch('https://jsonplaceholder.typicode.com/albums?userId=1').then(r => r.json()) ]); return { user, posts, albums }; }

5.6 エラーハンドリング

非同期処理では、ネットワークエラーやAPIエラーなど様々な問題が発生する可能性があります。適切なエラーハンドリングを行うことが重要です。

try-catch文によるエラー処理

async function fetchUserWithErrorHandling(userId) { try { const response = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`); // レスポンスの状態をチェック if (!response.ok) { throw new Error(`HTTP Error: ${response.status} ${response.statusText}`); } const user = await response.json(); // データの妥当性をチェック if (!user.id || !user.name) { throw new Error('不正なユーザーデータです'); } return user; } catch (error) { // エラーの種類に応じた処理 if (error instanceof TypeError) { console.error('ネットワークエラー:', error.message); } else { console.error('その他のエラー:', error.message); } // エラーを再度throw（呼び出し元に伝播） throw error; } }

カスタムエラークラス

// カスタムエラークラスの定義 class APIError extends Error { constructor(message, status, response) { super(message); this.name = 'APIError'; this.status = status; this.response = response; } } class NetworkError extends Error { constructor(message) { super(message); this.name = 'NetworkError'; } } // エラークラスを使用した関数 async function fetchWithCustomErrors(url) { try { const response = await fetch(url); if (!response.ok) { throw new APIError( `API Error: ${response.status}`, response.status, response ); } return await response.json(); } catch (error) { if (error instanceof TypeError) { throw new NetworkError('ネットワーク接続に失敗しました'); } throw error; // その他のエラーはそのまま再throw } }

リトライ機能の実装

// 指定回数まで再試行する関数 async function fetchWithRetry(url, maxRetries = 3, delay = 1000) { for (let attempt = 1; attempt <= maxRetries; attempt++) { try { console.log(`試行 ${attempt}/${maxRetries}: ${url}`); const response = await fetch(url); if (response.ok) { return await response.json(); } // 5xx エラーの場合はリトライ if (response.status >= 500 && attempt < maxRetries) { console.log(`サーバーエラー (${response.status}), ${delay}ms後に再試行...`); await new Promise(resolve => setTimeout(resolve, delay)); continue; } // 4xx エラーの場合はリトライしない throw new Error(`HTTP Error: ${response.status}`); } catch (error) { if (attempt === maxRetries) { throw new Error(`${maxRetries}回試行しましたが失敗しました: ${error.message}`); } console.log(`エラー発生: ${error.message}, ${delay}ms後に再試行...`); await new Promise(resolve => setTimeout(resolve, delay)); } } } // 使用例 fetchWithRetry('https://api.example.com/data', 3, 2000) .then(data => console.log('データ取得成功:', data)) .catch(error => console.error('最終的にエラー:', error));
エラーハンドリングのベストプラクティス
  • すべての非同期処理でエラーハンドリングを行う
  • ユーザーにわかりやすいエラーメッセージを表示する
  • ネットワークエラーとAPIエラーを区別して処理する
  • 重要でない処理のエラーはアプリケーションを停止させない
  • ログ出力で問題の特定を容易にする

} initializeElements() { this.cityInput = document.getElementById('cityInput'); this.searchBtn = document.getElementById('searchBtn'); this.loading = document.getElementById('loading'); this.weatherResult = document.getElementById('weatherResult'); this.errorMessage = document.getElementById('errorMessage'); this.cityName = document.getElementById('cityName'); this.temperature = document.getElementById('temperature'); this.description = document.getElementById('description'); this.humidity = document.getElementById('humidity'); } bindEvents() { this.searchBtn.addEventListener('click', () => this.searchWeather()); this.cityInput.addEventListener('keypress', (e) => { if (e.key === 'Enter') { this.searchWeather(); } }); } async searchWeather() { const city = this.cityInput.value.trim(); if (!city) { this.showError('都市名を入力してください'); return; } try { this.showLoading(); const weatherData = await this.fetchWeatherData(city); this.displayWeatherData(weatherData); } catch (error) { this.showError(error.message); } finally { this.hideLoading(); } } async fetchWeatherData(city) { const url = `${this.apiUrl}?q=${encodeURIComponent(city)}&appid=${this.apiKey}&units=metric&lang=ja`; try { const response = await fetch(url); if (!response.ok) { if (response.status === 404) { throw new Error('指定された都市が見つかりません'); } else if (response.status === 401) { throw new Error('APIキーが無効です'); } else { throw new Error(`API Error: ${response.status}`); } } const data = await response.json(); return data; } catch (error) { if (error instanceof TypeError) { throw new Error('ネットワーク接続エラーが発生しました'); } throw error; } } displayWeatherData(data) { this.cityName.textContent = `${data.name}, ${data.sys.country}`; 第5章の理解度チェック

以下の問題に答えて、非同期処理の理解度を確認しましょう。

問題1: Promiseの3つの状態として正しくないものはどれですか？

問題2: async/await構文について正しい説明はどれですか？

問題3: Fetch APIでJSONデータを取得する正しい手順はどれですか？

問題4: Promise.allの特徴として正しいものはどれですか？

問題5: 非同期処理でのエラーハンドリングに関して、適切でない記述はどれですか？

正解:
  1. d（Completed（完了））
  2. b（Promiseの上に構築された、より読みやすい構文糖）
  3. c（fetch() → then(response.json()) → then()）
  4. c（すべてのPromiseが成功した場合のみ結果を返す）
  5. c（ネットワークエラーはfetch()で自動的に処理される）
