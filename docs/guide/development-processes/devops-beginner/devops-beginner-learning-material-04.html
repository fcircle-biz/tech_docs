コマンドラインの基礎 - 文字でコンピュータを操作する

普段マウスでクリックして操作するパソコンを、キーボードで文字を入力して操作する方法を学びます。エンジニアが日常的に使う「コマンドライン」の基本を、初心者にも分かりやすく解説します。

この章で学ぶこと

  • コマンドラインとは何か、なぜエンジニアが使うのか
  • Windows（コマンドプロンプト/PowerShell）とMac/Linux（ターミナル）の違い
  • 基本的なコマンド（フォルダ移動、ファイル操作、内容確認など）
  • 実際に手を動かして、コマンドを使ってみる練習

4.1 コマンドラインとは何か

普段パソコンを使うとき、私たちはマウスでアイコンをクリックしたり、メニューをプルダウンしたりして操作します。 これはGUI（グラフィカル・ユーザー・インターフェース）と呼ばれる操作方法です。

一方、コマンドラインは、黒い（または白い）画面に文字を入力してコンピュータを操作する方法です。 見た目は古臭く感じるかもしれませんが、実はとても強力で効率的な操作方法なのです。

コマンドラインの別名

コマンドラインは、環境によって呼び方が異なります：
Windows: コマンドプロンプト、PowerShell
Mac/Linux: ターミナル、シェル、コンソール
本質的にはどれも「文字で操作する画面」です。

4.2 なぜエンジニアはコマンドラインを使うのか

「マウスでクリックする方が簡単なのに、なぜ文字で操作するの？」と疑問に思うかもしれません。 実は、コマンドラインには以下のような大きなメリットがあります。

圧倒的に速い

例えば「100個のファイルの名前を一括変更する」といった作業は、マウスでは何十分もかかりますが、コマンド1行なら数秒で完了します。

自動化できる

コマンドをファイルに保存しておけば、繰り返し作業を自動化できます。毎回同じクリックを繰り返す必要がありません。

リモート操作が可能

サーバーなどの遠隔地のコンピュータを操作するとき、GUIは使えないことがほとんどです。コマンドラインなら遠隔操作が簡単です。

記録が残る

実行したコマンドは履歴に残ります。「どんな操作をしたか」を後から確認したり、同じコマンドを再利用したりできます。

4.3 コマンドラインを起動してみよう

お使いのOSによって、起動方法が異なります。以下の手順で開いてみましょう。

Windows の場合

方法1: PowerShell を開く（推奨）

  1. スタートボタンを右クリック
  2. 「Windows PowerShell」または「ターミナル」を選択

方法2: コマンドプロンプトを開く

  1. Win + R キーを押す
  2. 「cmd」と入力してEnter

macOS の場合

方法1: Spotlightから開く（推奨）

  1. Command + Space キーを押す
  2. 「ターミナル」と入力してEnter

方法2: アプリケーションから開く

  1. Finder → アプリケーション → ユーティリティ
  2. 「ターミナル.app」をダブルクリック

Linux の場合

一般的な方法

  1. Ctrl + Alt + T キーを押す
  2. または、アプリケーションメニューから「端末」「ターミナル」を検索

4.4 基本的なコマンドを覚えよう

コマンドラインでは、コマンドと呼ばれる命令を入力して操作します。 まずは、最もよく使う基本コマンドを覚えましょう。

WindowsとMac/Linuxでコマンドが違う

Windowsのコマンドプロンプトと、Mac/Linuxのターミナルでは、一部コマンドが異なります。 PowerShellを使えば、Windowsでも多くのLinux風コマンドが使えます。 ここでは両方のコマンドを紹介します。

よく使う基本コマンド一覧

操作 Windows (cmd) Mac/Linux 説明
現在地を表示 cd pwd 今どのフォルダにいるか表示
ファイル一覧 dir ls 現在のフォルダ内のファイル・フォルダを一覧表示
フォルダ移動 cd フォルダ名 cd フォルダ名 指定したフォルダに移動
親フォルダへ cd .. cd .. 1つ上のフォルダに移動
フォルダ作成 mkdir フォルダ名 mkdir フォルダ名 新しいフォルダを作成
ファイル作成 type nul > ファイル名 touch ファイル名 空のファイルを作成
ファイル内容表示 type ファイル名 cat ファイル名 ファイルの中身を表示
ファイルコピー copy 元 先 cp 元 先 ファイルを複製
ファイル移動 move 元 先 mv 元 先 ファイルを移動（名前変更にも使える）
ファイル削除 del ファイル名 rm ファイル名 ファイルを削除（注意！復元できません）

実習: 実際にコマンドを試してみよう

ここでは、実際にコマンドラインを開いて、基本操作を体験してみましょう。 安心してください。この実習で何か壊れることはありません。

ステップ1: 現在地を確認

まず、今自分がどこにいるか確認しましょう。

Windows (PowerShell/cmd) 
# Windows (コマンドプロンプト)
cd

# または PowerShell
pwd
Mac/Linux 
# 現在のディレクトリパスを表示
pwd

実行すると、現在いるフォルダのパスが表示されます。

ステップ2: ファイル一覧を表示

現在のフォルダに何があるか見てみましょう。

Windows 
# ファイル一覧を表示
dir
Mac/Linux 
# ファイル一覧を表示
ls

# より詳しい情報を表示
ls -l

ステップ3: 練習用フォルダを作成

「test」という名前のフォルダを作ってみましょう。

すべてのOS共通 
# testフォルダを作成
mkdir test

フォルダエクスプローラーで確認すると、実際に「test」フォルダが作られています！

ステップ4: フォルダに移動

作ったtestフォルダの中に入ってみましょう。

すべてのOS共通 
# testフォルダに移動
cd test

# 移動できたか確認（Windows）
cd

# 移動できたか確認（Mac/Linux）
pwd

ステップ5: ファイルを作成

空のテキストファイルを作ってみましょう。

Windows 
# hello.txtを作成
type nul > hello.txt
Mac/Linux 
# hello.txtを作成
touch hello.txt

ステップ6: 元のフォルダに戻る

1つ上のフォルダ（親フォルダ）に戻りましょう。

すべてのOS共通 
# 親フォルダに戻る
cd ..

「..」は「1つ上のフォルダ」を意味する特別な記号です。

4.5 覚えておくと便利な機能

Tabキーでの補完

ファイル名やフォルダ名の途中まで入力してTabキーを押すと、残りを自動で補完してくれます。 

# "Doc" まで入力して Tab キーを押す
cd Doc[Tab]
# → cd Documents/ と自動補完される

↑↓キーでコマンド履歴

↑キーを押すと、以前実行したコマンドを呼び出せます。 同じコマンドを何度も入力する必要がありません。
↓キーで新しい履歴へ移動できます。

画面をクリア

画面がごちゃごちゃしてきたら、クリアして綺麗にできます。 

# Windows
cls

# Mac/Linux
clear

コマンドの使い方を調べる

コマンドの詳しい使い方が分からないときは、ヘルプを見ることができます。 

# Windows
コマンド名 /?
# 例: dir /?

# Mac/Linux
man コマンド名
# 例: man ls

4.6 よくあるエラーと対処法

トラブルシューティング

「コマンドが見つかりません」と表示される

原因: コマンドのスペルミス、または存在しないコマンドを入力している

対処: スペルを確認する。WindowsとMac/Linuxでコマンドが違う場合があるので、自分のOSに合ったコマンドを使う。

「ファイルが見つかりません」と表示される

原因: 指定したファイルやフォルダが存在しない、または現在のフォルダにない

対処: まず dir や ls でファイル一覧を確認。ファイル名のスペルミスにも注意。

「アクセスが拒否されました」と表示される

原因: システムフォルダなど、変更権限がない場所を操作しようとしている

対処: 自分のホームフォルダ（ドキュメントなど）で練習する。システムフォルダは触らない。

コマンドが途中で止まって何も反応しない

原因: 長時間かかる処理を実行中、または入力待ち状態

対処: Ctrl + C キーを押すと、実行中のコマンドを中断できます。

注意: 削除コマンドは取り消せません！

del や rm でファイルを削除すると、ゴミ箱を経由せず完全に削除されます。 大切なファイルを誤って削除しないよう、特に注意してください。 練習では、専用の練習フォルダを作って、そこで実験しましょう。

コマンドライン操作の基本フロー

flowchart TD A([コマンドラインを開く]) --> B[現在地を確認
pwd または cd] B --> C{何をしたい？} C -->|見る| D[ファイル一覧
ls または dir] C -->|移動| E[フォルダ移動
cd フォルダ名] C -->|作成| F[フォルダ/ファイル作成
mkdir または touch] C -->|削除| G[ファイル削除
rm または del] D --> H[次の操作へ] E --> H F --> H G --> H H --> C

コマンドライン操作は、「現在地確認 → 操作 → 次の操作」の繰り返しです。

理解度確認クイズ

  1. Q1. コマンドラインを使うメリットを3つ挙げてください。

    回答を表示

    回答例:
    1. 圧倒的に速い（大量のファイル操作が一瞬）
    2. 自動化できる（繰り返し作業を楽にできる）
    3. リモート操作が可能（遠隔地のサーバーを操作できる）
    4. 記録が残る（履歴を確認・再利用できる）
    ※ 3つ以上あれば正解です。

  2. Q2. 現在いるフォルダを確認するコマンドは何ですか？（WindowsとMac/Linux両方答えてください）

    回答を表示

    回答:
    ・Windows (cmd): cd
    ・Windows (PowerShell) / Mac / Linux: pwd

  3. Q3. ファイルを削除するとき、なぜ注意が必要ですか？

    回答を表示

    回答: コマンドラインで削除したファイルは、ゴミ箱を経由せず完全に削除されるため、元に戻すことができません。 そのため、削除コマンド（del や rm）を使うときは、 ファイル名をよく確認し、大切なファイルを誤って削除しないよう注意が必要です。

この章のまとめ

  • コマンドラインは、文字を入力してコンピュータを操作する方法
  • 速い、自動化できる、リモート操作可能、記録が残る、という4つのメリットがある
  • WindowsとMac/Linuxでコマンドが異なる場合がある（例: dir と ls）
  • 基本コマンドは10個程度覚えれば、日常的な操作ができる
  • Tabキーでの補完、↑キーでの履歴呼び出しなど、便利な機能がある
  • 削除コマンドは取り消せないので、特に注意が必要
