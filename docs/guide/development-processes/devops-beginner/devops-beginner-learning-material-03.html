コンピュータの仕組み - ハードウェアとソフトウェア

コンピュータを構成するハードウェア(機械部品)とソフトウェア(プログラム)の役割を、身近なたとえで理解します。

この章で学ぶこと

  • ハードウェアとソフトウェアの違いと役割
  • CPU・メモリ・ストレージの役割(頭脳・作業机・本棚にたとえて理解)
  • OS(オペレーティングシステム)の役割と重要性
  • アプリケーションがどのように動いているのか

3.1 ハードウェアとソフトウェアの違い

コンピュータは、ハードウェア(Hardware)とソフトウェア(Software)という2つの要素で成り立っています。

ハードウェアとは？

ハードウェアは、コンピュータの「物理的な部品」のことです。 触ることができ、目に見えるものです。例: CPU、メモリ、ディスク、キーボード、ディスプレイ

ソフトウェアとは？

ソフトウェアは、コンピュータに「何をするか」を指示する「プログラム」です。 目に見えず、触ることもできません。例: Windows、Chrome、Excel、アプリ

たとえ話: ハードウェアが「楽器(ピアノ)」なら、ソフトウェアは「楽譜」です。 どちらか一方だけでは音楽を演奏できません。楽器(ハードウェア)があっても楽譜(ソフトウェア)がなければ何も弾けず、 楽譜があっても楽器がなければ音は鳴りません。

ソフトウェア(特にOS)がハードウェアを制御することで、コンピュータは私たちの指示通りに動きます。

3.2 ハードウェアの主要部品

コンピュータを構成する主要な部品を、身近なたとえで理解しましょう。

CPU(Central Processing Unit) - 頭脳

CPUは、コンピュータの「頭脳」です。 すべての計算や判断を行う部品で、「1+1=2」という計算から、「このボタンがクリックされたら次の画面を表示する」という複雑な処理まで担当します。

  • たとえ: 料理人の頭脳(レシピを見て手順を考える)
  • 役割: プログラムの命令を実行する
  • 性能指標: GHz(ギガヘルツ)で表される処理速度

メモリ(RAM) - 作業机

メモリは、「作業机」のようなものです。 いま作業中のデータやプログラムを一時的に置いておく場所です。 作業机が広いほど、多くの書類(プログラム)を同時に広げて作業できます。

  • たとえ: 作業机(広いほど効率的)
  • 役割: 現在実行中のプログラムとデータを保存
  • 性能指標: GB(ギガバイト)で表される容量
  • 特徴: 電源を切ると中身が消える(揮発性)

ストレージ(SSD/HDD) - 本棚・倉庫

ストレージは、「本棚」や「倉庫」のようなものです。 写真、動画、ドキュメント、アプリなど、すべてのデータを長期保存します。 電源を切っても中身は消えません。

  • たとえ: 本棚・倉庫(大きいほど多く保存できる)
  • 役割: データやプログラムを長期保存
  • 性能指標: GB/TB(ギガバイト/テラバイト)で表される容量
  • 特徴: 電源を切っても中身が残る(不揮発性)

メモリとストレージの違い

メモリ: 作業中の一時的な場所(作業机)。速いが、電源を切ると消える。
ストレージ: 長期保存の場所(本棚)。遅いが、電源を切っても残る。

CPU・メモリ・ストレージの連携

sequenceDiagram participant U as ユーザー participant A as アプリ participant M as メモリ participant C as CPU participant S as ストレージ U->>A: アプリを起動 A->>S: プログラムを読み込み S->>M: プログラムをメモリに展開 M->>C: 命令を渡す C->>C: 計算・処理 C->>M: 結果を保存 M->>S: ファイルとして保存(必要に応じて) S->>U: 作業完了

アプリを起動すると、ストレージからプログラムが読み込まれ、メモリに展開され、CPUが処理を実行します。

3.3 OS(オペレーティングシステム)の役割

OS(Operating System)は、ハードウェアとアプリケーションの「仲介役」です。 Windows、macOS、Linuxなどが代表的なOSです。

OSがない世界を想像してみましょう

もしOSがなかったら、Wordを起動するたびに「CPUに命令を送り、メモリを確保し、ディスクから読み込み…」という 複雑な作業を人間が手動でやらなければなりません。OSがあるからこそ、私たちは「アイコンをクリックするだけ」でアプリを使えます。

OSの主な役割

ハードウェア管理

CPU、メモリ、ディスクなどを管理し、アプリが効率よく使えるように調整します。

ファイル管理

フォルダやファイルを整理し、どこに何が保存されているかを管理します。

アプリ実行

アプリケーションを起動・実行し、必要なメモリやCPUを割り当てます。

ユーザーインターフェース

私たちが操作しやすいように、デスクトップ画面やウィンドウを提供します。

たとえ話: OSは「会社の総務部」のようなものです。 総務部が設備(ハードウェア)を管理し、従業員(アプリ)が働きやすい環境を整えてくれるおかげで、 各部署(アプリ)は自分の仕事に集中できます。

3.4 アプリケーションとは

アプリケーション(略してアプリ)は、特定の目的のために作られたソフトウェアです。

代表的なアプリケーション

  • ワープロソフト: Word、Google Docs(文書作成)
  • 表計算ソフト: Excel、Google Sheets(計算・グラフ作成)
  • Webブラウザ: Chrome、Edge、Safari(Webサイト閲覧)
  • メールソフト: Outlook、Gmail(メール送受信)
  • 画像編集ソフト: Photoshop、GIMP(写真編集)
  • SNSアプリ: LINE、Twitter、Instagram(コミュニケーション)

アプリはOSの上で動く

アプリは、OSがなければ動きません。Windowsで動くアプリはmacOSではそのまま動かないことが多いのは、 OSごとに「話す言葉(API)」が違うからです。

3.5 DevOpsとの関連 - なぜコンピュータの仕組みを知る必要があるのか

「DevOpsを学ぶのに、なぜCPUやメモリの話を知る必要があるの？」と疑問に思うかもしれません。 実は、DevOpsエンジニアはサーバーを扱う機会が非常に多いのです。

サーバーもコンピュータの一種

サーバーは、私たちが使うパソコンと基本的な構造は同じです。CPU、メモリ、ストレージ、OSがあります。 DevOpsエンジニアは、サーバーのCPU使用率やメモリ使用量を監視し、適切に管理する必要があります。

DevOpsでよくある場面

  • 「サーバーのメモリが足りなくてアプリが落ちた」→ メモリを増やす
  • 「CPUが100％になってサーバーが重い」→ CPUを増強するか、負荷分散する
  • 「ディスク容量が満杯になった」→ ストレージを拡張する
  • 「OSのバージョンが古くてセキュリティリスクがある」→ OSをアップデートする

この章で学んだハードウェアとソフトウェアの基礎知識は、 後の章で「サーバー」「クラウド」「コンテナ」を学ぶときの土台になります。

理解度確認クイズ

  1. Q1. ハードウェアとソフトウェアの違いを説明してください。

    回答を表示

    回答:
    ・ハードウェア: コンピュータの物理的な部品(触れる、見える)。例: CPU、メモリ、ディスク
    ・ソフトウェア: プログラムや命令(触れない、見えない)。例: OS、アプリケーション
    両方が揃って初めてコンピュータは動きます。

  2. Q2. メモリとストレージの違いを、「作業机」と「本棚」のたとえで説明してください。

    回答を表示

    回答:
    ・メモリ(作業机): いま作業中のものを一時的に置く場所。速いが、電源を切ると消える。
    ・ストレージ(本棚): データを長期保存する場所。遅いが、電源を切っても残る。
    作業机が広いほど多くのアプリを同時に開けます。本棚が大きいほど多くのファイルを保存できます。

  3. Q3. OS(オペレーティングシステム)の役割を3つ挙げてください。

    回答を表示

    回答例:
    1. ハードウェア管理(CPU、メモリ、ディスクなどを管理)
    2. ファイル管理(フォルダやファイルの整理)
    3. アプリ実行(アプリケーションを起動・実行)
    4. ユーザーインターフェース提供(デスクトップ画面など)
    OSがあることで、私たちは複雑な操作をせずにコンピュータを使えます。

この章のまとめ

  • ハードウェアは物理的な部品、ソフトウェアはプログラム
  • CPU(頭脳)、メモリ(作業机)、ストレージ(本棚)がコンピュータの主要部品
  • OSはハードウェアとアプリケーションの仲介役
  • アプリケーションはOSの上で動く特定目的のソフトウェア
  • DevOpsではサーバー(コンピュータの一種)を扱うため、この知識が必須
