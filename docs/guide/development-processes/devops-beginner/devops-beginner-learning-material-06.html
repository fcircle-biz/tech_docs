インターネットの仕組み - Webサービスが届くまで

スマートフォンでSNSを開くと、なぜ投稿が表示されるのでしょうか？インターネットの基本的な仕組みを「郵便システム」に例えて学びます。

この章で学ぶこと

  • インターネットとは何か、どのような仕組みで世界中のコンピュータがつながっているか
  • IPアドレス（住所）とDNS（電話帳）の役割
  • HTTP/HTTPS（通信ルール）の基本と、データが安全に届く仕組み
  • Webサイトを開いたときに裏側で何が起きているか

6.1 インターネットとは何か

インターネット（Internet）とは、世界中のコンピュータやスマートフォンを「ネットワーク」でつないだ巨大なシステムです。 「ネット」という略称で呼ばれることもあります。

インターネットを郵便システムに例えると

郵便で手紙を送るには、「送り先の住所」が必要ですよね。インターネットも同じで、 各コンピュータには「IPアドレス」という住所があります。手紙が郵便配達員によって運ばれるように、 データは「ルーター」という装置を経由して目的地に届けられます。

私たちがスマートフォンでWebサイトを開くと、裏側では「自分のスマホ → インターネット → サーバー」という通信が行われています。 サーバーから返ってきた情報（HTMLやCSS、画像など）が、画面に表示される仕組みです。

インターネット通信の基本構造

flowchart LR A[あなたのスマホ
192.168.1.10] -->|リクエスト| B{インターネット} B --> C[Webサーバー
203.0.113.50] C -->|レスポンス
HTMLやデータ| B B --> A

スマホやPCがインターネットを経由してWebサーバーと通信し、情報をやり取りします。

6.2 IPアドレス - コンピュータの住所

IPアドレス（Internet Protocol Address）は、インターネットに接続された機器すべてに割り当てられる「住所」です。 例えば、192.168.1.1 や 203.0.113.50 のような数字の並びです。

IPアドレスの種類

IPv4（主流）

192.168.1.1 のような形式

4つの数字（0〜255）をドット（.）でつないだもの。現在最も使われているバージョンです。

IPv6（次世代）

2001:0db8::1 のような形式

IPv4のアドレスが不足してきたため、より多くのアドレスを割り当てられる新しいバージョンです。

重要ポイント

IPアドレスは「数字の羅列」なので人間には覚えにくいです。そこで、次に説明する「DNS」という仕組みで、 分かりやすい名前（ドメイン名）に変換できるようになっています。

6.3 DNS - インターネットの電話帳

DNS（Domain Name System）は、人間が覚えやすい「ドメイン名」を、 コンピュータが理解できる「IPアドレス」に変換する仕組みです。

DNSを電話帳に例えると

友達に電話するとき、電話番号を覚えていなくても、名前を検索すれば番号が分かりますよね。 DNSも同じで、「www.google.com」という名前を入力すれば、自動的にGoogleのIPアドレスを調べてくれます。

DNSの仕組み

あなたがブラウザで「www.example.com」と入力すると、裏側では以下のような処理が行われます：

  1. ブラウザがDNSサーバーに「www.example.comのIPアドレスは何？」と問い合わせる
  2. DNSサーバーが「192.0.2.1だよ」と返事をする
  3. ブラウザはそのIPアドレス宛にリクエストを送る

DNS名前解決の流れ

sequenceDiagram autonumber participant U as ユーザー
ブラウザ participant DNS as DNSサーバー participant WEB as Webサーバー
192.0.2.1 U->>DNS: www.example.comのIPは？ DNS-->>U: 192.0.2.1です U->>WEB: HTMLをください
宛先: 192.0.2.1 WEB-->>U: HTMLを返送

DNSサーバーが「ドメイン名 → IPアドレス」の変換を行い、通信が可能になります。

6.4 HTTP/HTTPS - データのやり取りのルール

ブラウザとサーバーがデータをやり取りするときには、HTTP（HyperText Transfer Protocol）という「通信ルール」が使われます。 これは「こういう形式でリクエストを送る」「こういう形式で返事をする」という約束事です。

HTTPリクエストとレスポンス

Webサイトを開くとき、以下のような流れでデータがやり取りされます：

  • リクエスト（Request）: ブラウザがサーバーに「このページのデータをください」と依頼する
  • レスポンス（Response）: サーバーが「はい、どうぞ」と言ってHTMLや画像を返す

HTTPS - 安全な通信

HTTPS（HTTP Secure）は、HTTPに「暗号化」を加えた安全な通信方式です。 パスワードやクレジットカード情報など、重要な情報をやり取りするときに使われます。

HTTPSの見分け方

ブラウザのアドレスバーに「鍵マーク🔒」が表示されていれば、そのサイトは暗号化されています。 安心して個人情報を入力できます。

主なHTTPメソッド

GET データを取得する

Webページを表示するときに使います。「このページを見せてください」というリクエストです。

POST データを送信する

フォームで入力した内容を送るときに使います。「このデータを登録してください」というリクエストです。

PUT データを更新する

既存のデータを書き換えるときに使います。「この情報を更新してください」というリクエストです。

DELETE データを削除する

データを削除するときに使います。「この情報を消してください」というリクエストです。

6.5 Webページが表示されるまでの流れ

ここまで学んだIPアドレス、DNS、HTTPを使って、Webページが表示されるまでの全体像を見てみましょう。

Webページが表示されるまでの流れ

flowchart TD A([ユーザーがURLを入力
www.example.com]) --> B[DNS問い合わせ] B --> C[IPアドレス取得
192.0.2.1] C --> D[HTTPリクエスト送信
GET /index.html] D --> E{サーバー処理} E --> F[HTMLファイル生成] F --> G[HTTPレスポンス返送
200 OK] G --> H[ブラウザがHTMLを解析] H --> I[CSS・画像の追加取得] I --> J[JavaScriptの実行] J --> K([画面に表示完了])

ブラウザでURLを入力してから画面が表示されるまで、複数のステップが自動的に実行されています。

1

URLの入力

ユーザーがブラウザのアドレスバーに「www.example.com」と入力します。

2

DNS問い合わせ

ブラウザがDNSサーバーに「www.example.comのIPアドレスは？」と問い合わせます。

3

IPアドレス取得

DNSサーバーが「192.0.2.1です」と回答します。

4

HTTPリクエスト送信

ブラウザがIPアドレス宛に「GET /index.html」というHTTPリクエストを送ります。

5

サーバー処理

Webサーバーがリクエストを受け取り、HTMLファイルを準備します。

6

HTTPレスポンス返送

サーバーがHTMLファイルを「200 OK」というステータスコードとともに返送します。

7

ブラウザがHTMLを解析

ブラウザがHTMLを読み取り、画面に表示する準備をします。

8

追加リソースの取得

HTMLに記載されたCSS、画像、JavaScriptファイルを追加で取得します。

9

画面に表示完了

すべてのリソースが揃ったら、ブラウザが画面に表示します。

6.6 DevOpsとネットワークの関係

DevOpsエンジニアにとって、ネットワークの知識は非常に重要です。なぜなら、以下のような業務でネットワークを扱うからです：

  • サーバーの構築: サーバーにIPアドレスを設定し、外部からアクセスできるようにする
  • セキュリティ対策: ファイアウォールでアクセス制限をかけ、不正なアクセスを防ぐ
  • ロードバランサーの設定: 複数のサーバーにアクセスを分散し、負荷を軽減する
  • DNS設定: ドメイン名とIPアドレスを紐付け、ユーザーが簡単にアクセスできるようにする
  • SSL証明書の設定: HTTPSで安全な通信を実現する

理解度確認クイズ

  1. Q1. IPアドレスとは何ですか？身近な例で説明してください。

    回答を表示

    回答: IPアドレスは、インターネットに接続された機器すべてに割り当てられる「住所」です。 郵便で手紙を送るときに住所が必要なのと同じように、データを送るときにはIPアドレスが必要です。 例: 192.168.1.1

  2. Q2. DNSの役割を説明してください。

    回答を表示

    回答: DNSは、人間が覚えやすい「ドメイン名」（例: www.google.com）を、 コンピュータが理解できる「IPアドレス」（例: 172.217.175.46）に変換する仕組みです。 電話帳のように、名前から番号を調べる役割を果たします。

  3. Q3. HTTPとHTTPSの違いは何ですか？

    回答を表示

    回答: HTTPは通常の通信方式で、データが暗号化されていません。 HTTPSは暗号化を加えた安全な通信方式で、パスワードやクレジットカード情報など重要な情報をやり取りするときに使われます。 HTTPSを使っているサイトは、ブラウザのアドレスバーに鍵マーク🔒が表示されます。

  4. Q4. Webページを開いたときに裏側で起きる流れを簡単に説明してください。

    回答を表示

    回答:
    1. ブラウザがDNSサーバーにドメイン名を問い合わせ、IPアドレスを取得
    2. HTTPリクエストをサーバーに送信
    3. サーバーがHTMLファイルを返送
    4. ブラウザがHTMLを解析し、CSS・画像・JavaScriptを追加取得
    5. 画面に表示
