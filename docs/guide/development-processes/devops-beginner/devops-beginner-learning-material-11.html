クラウドサービス入門 - 誰でも使えるコンピュータリソース

かつてはサーバーを使うために高価なコンピュータを購入し、専用の部屋に設置する必要がありました。しかし今では、インターネット経由で「クラウド」からコンピュータリソースを借りることができます。

この章で学ぶこと

  • クラウドサービスとは何か、なぜ便利なのか（電気や水道のたとえ）
  • オンプレミスとクラウドの違い、それぞれのメリット・デメリット
  • 主要クラウドサービス（AWS、Azure、Google Cloud）の特徴
  • IaaS、PaaS、SaaSという3つのクラウドサービスモデル
  • クラウドがDevOpsとどのように結びついているか

11.1 クラウドとは何か - 電気や水道のようなIT資源

「クラウド」という言葉を聞いたことがあるでしょうか？ 正確にはクラウドコンピューティング（Cloud Computing）と呼ばれます。

クラウドコンピューティングとは？

インターネット経由で、サーバー、ストレージ、データベースなどのコンピュータリソースを「必要なときに、必要なだけ」借りられるサービスです。

身近な例で考えてみましょう。電気を使うとき、私たちは発電所を自分で持っていません。 電力会社から電気を「借りて」使い、使った分だけ料金を支払います。

クラウドも同じです。高価なサーバーを自分で購入・管理する代わりに、 インターネット経由でコンピュータリソースを借り、使った分だけ支払います。

オンプレミスとクラウドの比較

graph TB subgraph オンプレミス["オンプレミス（自社設置）"] A1[サーバーを購入] A2[専用部屋を準備] A3[空調・電源を整備] A4[保守・運用を自社で実施] A1 --> A2 --> A3 --> A4 end subgraph クラウド["クラウド（レンタル）"] B1[必要なリソースを選択] B2[即座に利用開始] B3[使った分だけ支払い] B4[保守はクラウド事業者が実施] B1 --> B2 --> B3 --> B4 end

オンプレミスは自社でサーバーを所有・管理する方式、クラウドは借りる方式です。

11.2 オンプレミスとクラウドの違い

クラウドが登場する前は、オンプレミス（On-Premises）という方式が一般的でした。 これは、サーバーを自社で購入し、社内に設置して運用する方式です。

項目 オンプレミス クラウド
初期費用 高額（数百万円〜） なし（または少額）
導入スピード 遅い（数週間〜数ヶ月） 速い（数分〜数時間）
拡張性 低い（機器購入が必要） 高い（即座に拡張可能）
保守・管理 自社で実施（人件費が必要） 事業者が実施
コスト構造 固定費（使っても使わなくても同じ） 従量課金（使った分だけ）
物理的管理 自社で設置場所・電源・空調を管理 不要（事業者が管理）

クラウドのメリット

  • 初期投資が少なく、すぐに始められる
  • 必要に応じて柔軟にリソースを増減できる
  • 世界中のデータセンターを利用できる
  • 保守・運用の手間が減る

クラウドのデメリット

  • 長期利用では割高になる場合がある
  • インターネット接続が必須
  • データを外部に預けることになる
  • 設定ミスによるコスト増加のリスク

11.3 主要クラウドサービスプロバイダー

世界には多くのクラウドサービスがありますが、特に有名な3つのサービスを「三大クラウド」と呼びます。

AWS（Amazon Web Services）

Amazon社が提供する世界最大のクラウドサービス。 2006年にスタートし、現在も世界シェアNo.1です。

特徴：

  • • サービスの種類が非常に豊富（200種類以上）
  • • 世界中に多数のデータセンター
  • • 先進的な機能をいち早く提供

Microsoft Azure（アジュール）

Microsoft社が提供するクラウドサービス。 Windows ServerやOffice 365との連携が強みです。

特徴：

  • • Microsoftの製品・サービスとの親和性が高い
  • • 企業向け機能が充実
  • • ハイブリッドクラウド（オンプレミスとクラウドの併用）に強い

Google Cloud Platform（GCP）

Google社が提供するクラウドサービス。 Googleの検索やYouTubeを支える技術が基盤です。

特徴：

  • • データ分析・AI/機械学習に強い
  • • Googleの高速ネットワークを利用可能
  • • Kubernetes（コンテナ管理ツール）発祥の地

11.4 クラウドサービスのモデル - IaaS、PaaS、SaaS

クラウドサービスは、提供される機能のレベルによって大きく3つに分類されます。 これを「サービスモデル」と呼びます。

クラウドサービスモデルの階層

Software as a Service"] S1[アプリケーション] end subgraph PaaS["PaaS
Platform as a Service"] P1[実行環境・開発ツール] P2[アプリケーション] end subgraph IaaS["IaaS
Infrastructure as a Service"] I1[サーバー・ストレージ] I2[OS・ミドルウェア] I3[実行環境・開発ツール] I4[アプリケーション] end style SaaS fill:#e0f2fe style PaaS fill:#ddd6fe style IaaS fill:#fce7f3

下に行くほど、利用者が管理する範囲が広くなります。

IaaS（Infrastructure as a Service）

インフラストラクチャー・アズ・ア・サービス

イメージ： 「土地と建物だけ借りて、内装や家具は自分で用意する」

サーバー（仮想マシン）やストレージなど、インフラ基盤だけを提供するサービスです。 OS、ミドルウェア、アプリケーションは利用者が自分で設定します。

代表的なサービス：

  • • AWS EC2（Elastic Compute Cloud）
  • • Azure Virtual Machines
  • • Google Compute Engine

PaaS（Platform as a Service）

プラットフォーム・アズ・ア・サービス

イメージ： 「キッチン付きの賃貸住宅。家具や調理器具が揃っていて、すぐに料理ができる」

インフラに加えて、アプリケーションの実行環境や開発ツールまで提供されます。 利用者はアプリケーションのコード開発に集中できます。

代表的なサービス：

  • • AWS Elastic Beanstalk
  • • Azure App Service
  • • Google App Engine
  • • Heroku（サードパーティ）

SaaS（Software as a Service）

ソフトウェア・アズ・ア・サービス

イメージ： 「レストランで食事をする。料理も調理器具も用意されていて、食べるだけ」

完成したアプリケーションをインターネット経由で提供するサービスです。 利用者はWebブラウザやスマホアプリで使うだけです。

代表的なサービス：

  • • Gmail（メールサービス）
  • • Google Workspace（旧G Suite）
  • • Microsoft 365（旧Office 365）
  • • Salesforce（営業支援ツール）
  • • Slack（チャットツール）

11.5 クラウドとDevOpsの関係

クラウドとDevOpsは、非常に相性が良い組み合わせです。 DevOpsの「素早く、柔軟に、自動的に」という理念を、クラウドが技術的に支えています。

クラウドがDevOpsを加速する理由

1. 環境構築が迅速

オンプレミスでは数週間かかるサーバー準備が、クラウドなら数分で完了。 開発・テスト・本番環境をすぐに用意できます。

2. 自動化が容易

クラウドのAPIを使って、インフラ構築・デプロイ・監視を全て自動化できます。 コードでインフラを管理する「Infrastructure as Code（IaC）」が実現しやすくなります。

3. スケーラビリティ

アクセス増加時には自動でサーバーを増やし、減少時には減らすことができます。 常に最適なリソースで運用できます。

4. 世界展開が簡単

世界中のデータセンターを利用できるため、グローバルサービスを低コストで展開できます。

クラウド上でのDevOpsパイプライン

クラウド上でCI/CDパイプラインを構築すれば、コード変更から本番リリースまで全て自動化できます。

11.6 クラウドを学ぶ第一歩

クラウドサービスは、多くの場合無料枠（Free Tier）を提供しています。 実際にアカウントを作成して、小規模な環境を触ってみることをお勧めします。

主要クラウドの無料枠

AWS無料利用枠

12ヶ月間、一定量のEC2やS3などを無料で利用できます。 初心者向けチュートリアルも豊富です。

Azure無料アカウント

12ヶ月間の無料サービスと、$200分のクレジットが提供されます。

Google Cloud無料枠

90日間の$300クレジットと、永続的な無料枠（Always Free）があります。

クラウド利用時の注意点

無料枠を超えると課金されます。初めて使う場合は、必ず料金アラートを設定し、 使わないリソースはすぐに削除する習慣をつけましょう。 設定ミスによる高額請求を避けるため、最初は少額のクレジットカードを登録することをお勧めします。

理解度確認クイズ

  1. Q1. オンプレミスとクラウドの主な違いを3つ挙げてください。

    回答を表示

    回答例:
    1. 初期費用: オンプレミスは高額、クラウドは少額またはなし
    2. 導入スピード: オンプレミスは数週間〜数ヶ月、クラウドは数分〜数時間
    3. 拡張性: オンプレミスは機器購入が必要、クラウドは即座に拡張可能

  2. Q2. IaaS、PaaS、SaaSの違いを説明してください。

    回答を表示

    回答:
    • IaaS: インフラ基盤（サーバー・ストレージ）のみ提供
    • PaaS: インフラ + 実行環境・開発ツールを提供
    • SaaS: 完成したアプリケーションを提供（GmailやSlackなど）
    IaaS → PaaS → SaaSの順で、利用者が管理する範囲が狭くなります。

  3. Q3. クラウドがDevOpsを加速する理由を2つ挙げてください。

    回答を表示

    回答例:
    1. 環境構築が迅速で、数分でサーバーを用意できる
    2. APIを使った自動化が容易で、CI/CDパイプラインを構築しやすい
    その他: スケーラビリティ、世界展開の容易さなども正解です。
