ブラックボックステスト技法

内部構造を知らなくても実施できるテスト技法を学びます。同値分割法、境界値分析、デシジョンテーブル、状態遷移テストなど、効率的にテストケースを作成する方法を習得します。

この章で学ぶこと

  • ブラックボックステストとは何か、その特徴と利点
  • 同値分割法によるテストケース削減の考え方
  • 境界値分析でバグを見つけやすい箇所を特定する方法
  • デシジョンテーブルで複雑な条件を整理する技法
  • 状態遷移テストでシステムの動作を確認する方法

6.1 ブラックボックステストとは

ブラックボックステストは、ソフトウェアの内部構造やコードを知らなくても実施できるテスト技法です。「入力」に対して「期待される出力」が得られるかを確認します。プログラミング知識がなくても実施できるため、テストの初心者でも取り組みやすい技法です。

（内部構造不明）"] B --> C[出力結果] style A fill:#dbeafe style B fill:#1e293b,color:#fff style C fill:#d1fae5

ブラックボックステストの特徴

  • 仕様ベース - 要求仕様や設計書を基にテストを設計
  • ユーザー視点 - エンドユーザーの使い方に近い観点でテスト
  • 独立性 - コードの実装方法に依存しない
  • 再利用性 - 内部実装が変わってもテストケースを再利用可能

6.2 同値分割法

同値分割法（Equivalence Partitioning）は、入力データの範囲を「同じ動作をするグループ（同値クラス）」に分け、各グループから代表値を1つ選んでテストする技法です。全パターンをテストする代わりに、効率的にテストケースを削減できます。

具体例: 年齢入力フィールドのテスト

「年齢は1歳から120歳まで入力可能」という仕様の場合、以下のように分割します。

1 無効クラス（範囲外: 小さすぎる）

0歳以下（例: -5, 0）

2 有効クラス（正常範囲）

1歳〜120歳（例: 25, 60, 100）

3 無効クラス（範囲外: 大きすぎる）

121歳以上（例: 150, 200）

同値分割法のメリット

1歳から120歳まで120個すべてをテストする必要がなく、各クラスから代表値を1つずつ選ぶだけで、効率的かつ網羅的にテストできます。

6.3 境界値分析

境界値分析（Boundary Value Analysis）は、入力範囲の「境界」に注目してテストする技法です。バグは境界付近で発生しやすいという経験則に基づいています。同値分割法と組み合わせて使用することが一般的です。

ポイント"] --> B["最小値"] A --> C["最小値+1"] A --> D["最大値-1"] A --> E["最大値"] A --> F["最小値-1"] A --> G["最大値+1"] style A fill:#fef3c7 style B fill:#d1fae5 style C fill:#d1fae5 style D fill:#d1fae5 style E fill:#d1fae5 style F fill:#fecaca style G fill:#fecaca

具体例: 年齢入力フィールドの境界値テスト

「年齢は1歳から120歳まで」という仕様の場合、以下の値をテストします。

テストケース 入力値 期待結果
最小値未満 0 エラー
最小値 1 正常
最小値より1大きい 2 正常
最大値より1小さい 119 正常
最大値 120 正常
最大値超過 121 エラー

なぜ境界でバグが起きやすいのか

プログラマーは条件分岐を書く際に「以上(≧)」「より大きい(>)」などを間違えやすく、「1以上」のつもりが「0以上」になっていたり、「120以下」が「121以下」になっていたりすることがあります。境界値をテストすることでこうしたミスを発見できます。

6.4 デシジョンテーブル

デシジョンテーブル（決定表）は、複数の条件の組み合わせとそれに対応する処理を表形式で整理する技法です。条件が複雑に絡み合うビジネスロジックのテストに有効です。

具体例: 割引適用ルールのテスト

ECサイトで以下のような割引ルールがあるとします。

  • • 会員なら10%割引
  • • 5000円以上の購入でさらに5%割引
  • • 両方の条件を満たせば15%割引
条件 ケース1 ケース2 ケース3 ケース4
会員か？ はい はい いいえ いいえ
5000円以上？ はい いいえ はい いいえ
割引率 15% 10% 5% 0%

デシジョンテーブルのメリット

条件の組み合わせを漏れなく整理できるため、複雑なビジネスルールをテストする際に非常に有効です。テストケースの抜け漏れを防ぐことができます。

6.5 状態遷移テスト

状態遷移テストは、システムの状態が変化していく過程をテストする技法です。ログイン/ログアウト、注文処理、ワークフロー管理など、複数の状態を持つシステムのテストに有効です。

stateDiagram-v2 [*] --> ログアウト中 ログアウト中 --> ログイン中 : ログイン成功 ログアウト中 --> ログアウト中 : ログイン失敗 ログイン中 --> ログアウト中 : ログアウト ログイン中 --> [*]

具体例: ログイン機能の状態遷移

ログイン機能には「ログアウト中」「ログイン中」という2つの状態があります。

状態1: ログアウト中

• 正しいIDとパスワード → ログイン中へ遷移

• 誤ったIDまたはパスワード → ログアウト中のまま

状態2: ログイン中

• ログアウトボタンをクリック → ログアウト中へ遷移

状態遷移図を使ったテストケース設計

状態遷移図を描くことで、すべての遷移パターンを視覚的に把握できます。各遷移経路をテストすることで、想定外の状態になったり、エラーが発生したりしないかを確認できます。

6.6 ブラックボックステスト技法の使い分け

各技法には適した場面があります。以下の表を参考に、状況に応じて使い分けましょう。

技法 適した場面 具体例
同値分割法 入力範囲が明確な場合 年齢、金額、個数入力
境界値分析 範囲の端でバグを疑う場合 最小値・最大値のチェック
デシジョンテーブル 複数条件の組み合わせがある場合 割引ルール、料金計算
状態遷移テスト システムの状態が変化する場合 ログイン、注文処理、ワークフロー

実習: 境界値分析を使ったテストケース作成

課題

オンラインショップで「商品個数は1個から99個まで購入可能」という仕様があります。境界値分析を使って、テストすべき入力値を列挙してください。

解答例を表示

テストケース:

  • • 0個（最小値未満）→ エラー
  • • 1個（最小値）→ 正常
  • • 2個（最小値+1）→ 正常
  • • 98個（最大値-1）→ 正常
  • • 99個（最大値）→ 正常
  • • 100個（最大値超過）→ エラー

  2. Q2. 同値分割法の目的は何ですか？

    回答を表示

    回答:
    入力データの範囲を「同じ動作をするグループ（同値クラス）」に分け、各グループから代表値を1つ選ぶことで、効率的にテストケースを削減することが目的です。全パターンをテストする代わりに、少ないテストケースで網羅的にテストできます。

  3. Q3. 境界値分析で重点的にテストする値はどこですか？なぜそこを重点的にテストするのですか?

    回答を表示

    回答:
    入力範囲の「境界」（最小値、最大値、その前後の値）を重点的にテストします。
    理由: プログラマーが条件分岐を書く際に「以上」「より大きい」などを間違えやすく、バグが境界付近で発生しやすいためです。

  4. Q4. デシジョンテーブルが有効な場面を説明してください。

    回答を表示

    回答:
    複数の条件の組み合わせとそれに対応する処理を整理する必要がある場合に有効です。例えば、会員種別と購入金額による割引ルールなど、条件が複雑に絡み合うビジネスロジックのテストで、条件の組み合わせを漏れなく整理できます。

第6章のまとめ

  • ブラックボックステストは内部構造を知らなくても実施でき、仕様ベースでユーザー視点のテストができます
  • 同値分割法により、入力範囲を同じ動作をするグループに分けて効率的にテストケースを削減できます
  • 境界値分析により、バグが発生しやすい境界付近の値を重点的にテストできます
  • デシジョンテーブルにより、複数条件の組み合わせを表形式で整理し、漏れなくテストできます
  • 状態遷移テストにより、システムの状態変化を視覚的に把握し、各遷移経路をテストできます
