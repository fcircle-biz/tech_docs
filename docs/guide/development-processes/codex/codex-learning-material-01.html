Codexとは

Codex は、ターミナル上で動作する OpenAI 公式の「エージェント型コーディングツール」です。この章では、Codex が何で、何ができ、従来のツールとどう違うのか、そして AI がどのように作業を進めるのかという全体像をつかみます。

この章で学ぶこと

  • Codex とは何か、誰が提供しているのかを理解する
  • 従来のコード補完ツールやチャット型 AI との違いを説明できる
  • Codex でできることの具体例を知る
  • 「計画→編集→検証」というエージェントの働き方の全体像をつかむ

1.1 Codex とは何か

Codex は、AI 企業 OpenAI が提供する公式のコーディング支援ツールです。最大の特徴は、ターミナル（コマンドライン）上で動作し、自然言語（ふだんの日本語や英語）で指示するだけで、AI が自分でコードを読み、考え、ファイルを編集し、コマンドを実行して、開発作業を進めてくれる点にあります。

単にコードの続きを提案したり（補完）、質問に答えたり（チャット）するだけのツールとは異なり、Codex は「やってほしいこと」を伝えると、その完了に向けて一連の作業を主体的に進めることができます。このような AI を「エージェント（agent）」と呼びます。本ガイドでは、ターミナルで動く Codex CLI を中心に学んでいきます。

ターミナルで動く

使い慣れたコマンドラインから起動。特別な画面に切り替える必要がありません。

自然言語で指示

「このバグを直して」のように、ふだんの言葉で依頼できます。

作業を完了させる

調査・編集・実行・確認まで、一連の作業を自分で進めます。

用語: 「Codex」という名前について

「Codex」はもともと OpenAI が2021年に公開したコード生成モデルの名前でしたが、本ガイドで学ぶ Codex は、最新の AI モデルを「開発作業のためのエージェント」として使えるようにしたツール（Codex CLI）を指します。ターミナルで codex コマンドとして利用します。

1.2 従来ツールとの違い

AI を使った開発支援ツールには、大きく3つのタイプがあります。それぞれの違いを理解すると、Codex の立ち位置がはっきりします。

タイプ 代表例 できること
コード補完型 GitHub Copilot など 入力中のコードの続きを提案する。あくまで「補助」。
チャット型 ブラウザで使う対話 AI（ChatGPT など） 質問に答える・コード例を出す。ファイルの編集や実行は自分で行う必要がある。
エージェント型 Codex 指示に基づき、自分でファイルを読み・編集し、コマンドを実行し、結果を確認して作業を完了させる。

つまり Codex は、「提案してくれる」「教えてくれる」を超えて、実際に手を動かして作業を進めてくれるところに大きな違いがあります。もちろん、変更を適用したりコマンドを実行したりする前に承認を求める仕組み（承認モード・サンドボックス）も備わっているため、安心して使えます（詳しくは第7章で学びます）。

関連: Claude Code との関係

エージェント型コーディング CLI には、Codex のほかに Anthropic の Claude Code もあります。考え方や使い勝手はよく似ており、本ガイドで学ぶ概念の多くは両者に共通します。違いを意識して両方を使い分けられるようになると、より強力です。

1.3 Codex でできること

Codex は、開発に関わるさまざまな作業を任せられます。代表的な活用例は次のとおりです。

  • 新機能の実装 — 「ログイン画面を追加して」のような依頼からコードを作成
  • バグ修正 — エラーの原因を調べて修正
  • リファクタリング — 動作を変えずにコードを整理・改善
  • コードの調査・説明 — 「この処理の流れを説明して」に回答
  • テストの作成・実行 — テストコードを書き、実行して結果を確認
  • Git 操作 — 変更のコミットやプルリクエストの作成（第10章）

実際のセッション（やり取り）は、次のようなイメージで進みます。

terminal — Codex セッション例 
$ codex
> ログイン処理にバグがあるようなので、原因を調べて直してほしい

● src/auth/login.js を読み込んでいます…
● 原因はトークンの有効期限チェックが抜けていることでした。修正します。
● テストを実行して動作を確認しました（5件すべて成功）。
● 修正が完了しました。変更内容を確認しますか？

1.4 エージェントの働き方（全体像）

Codex は、指示を受け取ってから完了するまで、人間の開発者と同じように「計画 → 調査 → 編集・実行 → 検証」というステップを繰り返します。検証で問題が見つかれば、計画に戻ってやり直します。このループを理解しておくと、AI が何をしているのかが見えやすくなります。

flowchart TD A["あなたの指示（自然言語）"] --> B["計画を立てる"] B --> C["コードベースを読む・調べる"] C --> D["ファイルを編集する／コマンドを実行する"] D --> E["結果を検証する（テスト・確認）"] E -->|うまくいった| F["変更内容を報告する"] E -->|問題があった| B

重要ポイント

Codex は「魔法の箱」ではなく、あなたの指示と確認のもとで動く協働パートナーです。良い指示を出し、変更内容を確認しながら使うことで、その力を最大限に引き出せます。

やってみよう 1-1: Codex を使える場面を考える

インストール前のこの段階では、まず「自分のどの作業に Codex を活かせそうか」をイメージしてみましょう。手を動かす前に活用イメージを持つことが、上達の近道です。

手順

  1. 1 自分が普段行っている開発・作業を3つ書き出してみる（例: バグ修正、ドキュメント作成、調査）。
  2. 2 そのうち、1.3 の活用例に当てはまり「Codex に任せられそう」なものに印を付ける。
  3. 3 第3章でインストールしたら、その作業を実際に依頼してみることを目標にする。

理解度確認クイズ

  1. Q1. Codex が、コード補完ツールやチャット型 AI と最も異なる点は何ですか？

    回答を表示

    指示に基づいて、自分でファイルを読み・編集し、コマンドを実行し、結果を確認するところまで「作業を完了させる」エージェント型である点。補完は入力の提案、チャットは質問への回答が中心で、編集や実行は人が行います。

  2. Q2. Codex は何の上で（どこで）動作しますか？

    回答を表示

    ターミナル（コマンドライン）上で動作します。使い慣れたターミナルから codex コマンドで起動します。

  3. Q3. エージェントの基本的な働き方のループを、順番に説明してください。

    回答を表示

    「計画を立てる → コードベースを調べる → ファイルを編集・コマンドを実行する → 結果を検証する」というステップを繰り返します。検証で問題があれば計画に戻り、うまくいけば変更内容を報告します。
