基本フローチャート

この章では、これまで学んだ基本構造とフローチャートの記号を使って、実際に図を描く練習をします。偶数判定、点数判定、1〜10表示の3題を通して、分岐とループの表現に慣れていきましょう。

この章で学ぶこと

  • 偶数判定・点数判定・1〜10表示をフローチャートで表現できる
  • 分岐の書き方と、条件ごとの流れの分け方を実践できる
  • ループの戻り矢印と終了条件の描き方を理解する
  • 図を描く前に、文章で手順を整理する習慣を身につける

4.1 演習の進め方

ここからは、実際にフローチャートを描く練習に入ります。いきなり図を描き始めるのではなく、まず文章で手順を整理し、そのあとで図形に置き換える流れで進めるのがコツです。

今回は「分岐のある問題」と「ループのある問題」をそれぞれ扱うので、どこで判断するか、どこで繰り返すかを意識しながら描いてみましょう。

重要ポイント

描く前に「入力は何か」「条件は何か」「どこで終わるか」を文章で決めておくと、フローチャートがぶれにくくなります。

演習の基本ステップ

flowchart LR A[問題文を読む] --> B[文章で手順を書く] B --> C[必要な記号に分ける] C --> D[矢印でつなぐ] D --> E[読み直して修正する] style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style E fill:#f3e8ff,stroke:#a855f7,color:#581c87

文章で考えてから図にすると、手順の漏れに気づきやすい

4.2 課題1: 偶数判定フローチャート

最初の課題は、整数を1つ受け取り、それが偶数か奇数かを判定するフローチャートです。ここでは 分岐 を正しく使えるかがポイントになります。

問題

  • 入力: 整数を1つ受け取る
  • 判定: 2で割った余りが0かどうかを調べる
  • 出力: 偶数または奇数を表示する

文章での手順

  1. 1. 整数を入力する
  2. 2. 2で割った余りが0か調べる
  3. 3. 0なら偶数と表示する
  4. 4. 0でなければ奇数と表示する

参考例: 偶数判定フローチャート

flowchart TD A([開始]) --> B[/整数を入力する/] B --> C{2で割った余りは0か} C -- はい --> D[偶数と表示する] C -- いいえ --> E[奇数と表示する] D --> F([終了]) E --> F style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style F fill:#f3e8ff,stroke:#a855f7,color:#581c87

分岐の線に「はい」「いいえ」を書くのを忘れない

4.3 課題2: 点数判定

次の課題は、点数によって「優・良・不可」を判定するフローチャートです。ここでは 分岐が連続する場合の描き方 を練習します。

条件 結果
80点以上 優
60点以上 80点未満 良
60点未満 不可

参考例: 点数判定フローチャート

flowchart TD A([開始]) --> B[/点数を入力する/] B --> C{80点以上か} C -- はい --> D[優と表示する] C -- いいえ --> E{60点以上か} E -- はい --> F[良と表示する] E -- いいえ --> G[不可と表示する] D --> H([終了]) F --> H G --> H style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style E fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style H fill:#f3e8ff,stroke:#a855f7,color:#581c87

分岐が1回で足りないときは、分岐をつなげて考える

ヒント

条件が3つ以上あるときは、いきなり全部を図にしようとせず、「最初に何を聞くか」「次に何を聞くか」を順番に決めると整理しやすくなります。

4.4 課題3: 1〜10表示（ループ）

最後の課題は、1から10までの数を順番に表示するフローチャートです。ここでは ループの戻り矢印 と 終了条件 の描き方が重要になります。

文章での手順

  1. 1. 数字を入れる箱に 1 を入れる
  2. 2. その数字が 10 以下か調べる
  3. 3. 10 以下なら表示する
  4. 4. 数字を 1 増やして、もう一度判定に戻る
  5. 5. 10 を超えたら終了する

注目ポイント

  • 最初の数をどこで決めるか
  • どの条件でループを終えるか
  • 戻り矢印がどこへ戻るか

参考例: 1〜10表示フローチャート

flowchart TD A([開始]) --> B[i を 1 にする] B --> C{i は 10 以下か} C -- はい --> D[i を表示する] D --> E[i を 1 増やす] E --> C C -- いいえ --> F([終了]) style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style F fill:#f3e8ff,stroke:#a855f7,color:#581c87

戻り矢印は「次にどこをもう一度見るか」をはっきり示す

つまずきポイント

ループでは「増やす処理」を入れ忘れると、同じ数をずっと表示し続ける図になってしまいます。戻る前に何を更新するのかを必ず確認しましょう。

4.5 仕上げの見直しポイント

フローチャートを描き終えたら、そのまま終わりにせず必ず見直しをしましょう。自分ではわかっていても、図にすると抜けや矛盾が見つかることがあります。

見直しチェック

  • 開始と終了があるか
  • 分岐に「はい」「いいえ」があるか
  • ループに終了条件があるか
  • 1箱1処理になっているか

よくある修正点

  • 条件文をもっと具体的に書き直す
  • 分岐の出口がどこへ進むか整理する
  • ループ内の更新処理を書き足す
  • 矢印の交差を減らして読みやすくする

ミニ演習（15分）: 自分で1題選んで描いてみよう

次の3題のうち、まだ自信がないものを1つ選び、紙に最初からフローチャートを描いてみましょう。

選べる課題

  1. 1 偶数判定
  2. 2 点数判定
  3. 3 1〜10表示

コツ: 描き終えたら、線を指で追いながら「本当にこの順番で動くか」を声に出して確認すると、抜けや矛盾に気づきやすくなります。

第4章のまとめ

  • フローチャートを描く前に、まず文章で手順を整理すると進めやすい
  • 偶数判定では、分岐の条件と「はい / いいえ」の流れが重要
  • 点数判定では、分岐を連続して使う考え方が必要
  • 1〜10表示では、戻り矢印と終了条件を正しく描くことが大切
  • 描いたあとは、開始から終了まで線を追って見直すとミスを見つけやすい

理解度確認クイズ

  1. Q1. 偶数判定のフローチャートで、分岐の条件には何を書けばよいですか？

    回答例を表示

    「2で割った余りは0か」のように、偶数かどうかを判断できる具体的な条件を書きます。

  2. Q2. 1〜10表示のループで、戻り矢印の前に必要な処理は何ですか？

    回答例を表示

    数字を1増やす処理です。これがないと同じ数のまま繰り返されてしまいます。

  3. Q3. 点数判定で「80点以上」「60点以上」「それ未満」を表すには、なぜ分岐が2回必要ですか？

    回答例を表示

    3つの結果に分ける必要があるからです。最初に80点以上かを調べ、違った場合に次の条件として60点以上かを調べます。
