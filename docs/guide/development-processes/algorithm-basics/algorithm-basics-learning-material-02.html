順次・分岐・繰り返し

アルゴリズムを形づくる3つの基本構造を学びます。どんな手順も「順番に進む」「条件で分かれる」「同じ処理をくり返す」の組み合わせで表現できることを、身近な例で確認していきましょう。

この章で学ぶこと

  • 順次・分岐・繰り返しがそれぞれどんな役割を持つか説明できる
  • 問題文を読んで、どの基本構造を使えばよいか判断できる
  • 簡単な処理を文章とフローチャートで表現できる
  • 偶数・奇数判定や 1 から 100 の合計を手順として整理できる

2.1 基本構造とは何か

どんなアルゴリズムでも、よく見ると 順次・分岐・繰り返し の3つに分解できます。長く見える手順でも、この3つのどれが使われているかを意識すると、ぐっと読みやすくなります。

まずは「順番に進む流れ」を作り、必要な場所だけ「条件による分かれ道」と「くり返し」を加える、という考え方が基本です。

基本構造 役割 身近な例
順次 上から順番に処理を進める 顔を洗う → 歯をみがく → 家を出る
分岐 条件によって次の処理を変える 雨なら傘を持つ、晴れなら持たない
繰り返し 同じ処理を必要な回数だけ行う 1 から 10 まで数える、全員の点数を見る

重要ポイント

難しそうな問題でも、まず「順番どおりに進む部分」を見つけると整理しやすくなります。そのあとで分岐や繰り返しが必要な場所を見つけましょう。

3つの基本構造の関係

アルゴリズムは3つの基本構造を組み合わせて作られる

2.2 順次: 上から順に実行する

順次は、「最初の処理が終わったら次へ、その次も終わったらさらに次へ」 と進む、もっとも基本的な流れです。迷う場面がなく、一本道で進むのが特徴です。

朝の支度や料理の準備のように、手順を入れ替えると困る作業では順次の考え方がそのまま使えます。

順次の例: 家を出るまでの流れ

flowchart TD A([開始]) --> B[顔を洗う] B --> C[歯をみがく] C --> D[かばんを持つ] D --> E[家を出る] E --> F([完了]) style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style F fill:#f3e8ff,stroke:#a855f7,color:#581c87

処理の順番がそのまま結果に影響する

順次が向いている場面

  • 手順の順番が決まっているとき
  • 途中で判断する必要がないとき
  • 1回ずつ確実に進めたいとき

見分けるコツ

  • 「次に」「そのあと」「最後に」という言葉が多い
  • Yes / No の判断が登場しない
  • 同じ手順を何度も戻って実行しない

2.3 分岐: 条件によって処理を変える

分岐は、ある条件を満たすかどうかで次の処理を変える構造です。日常生活では「雨なら傘を持つ」「お金が足りなければ買わない」のような判断が分岐にあたります。

分岐で大切なのは、条件をはっきり書くことです。「高ければ」「たぶん多ければ」ではなく、「60 点以上なら」「在庫が 1 個以上なら」のように判断基準を具体的にします。

分岐の例: 点数で合否を判定する

flowchart TD A([開始]) --> B[点数を確認する] B --> C{60点以上か} C -- はい --> D[合格と表示する] C -- いいえ --> E[再挑戦と表示する] D --> F([終了]) E --> F style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style F fill:#f3e8ff,stroke:#a855f7,color:#581c87

条件によって進む道が変わるのが分岐

ヒント

分岐は「質問」と考えるとわかりやすくなります。質問に対して「はい」「いいえ」のどちらかが返り、その答えに応じて処理が分かれます。

よくある分岐の例

  • 点数が 60 点以上なら合格
  • 数字を 2 で割った余りが 0 なら偶数
  • 在庫があれば注文を受け付ける

分岐を書くときの注意

  • 条件をあいまいな言葉で書かない
  • 「はい」と「いいえ」の両方の流れを考える
  • 判断する前に必要な情報がそろっているか確認する

2.4 繰り返し: 同じ処理を何度も行う

繰り返しは、同じ処理を必要な回数だけ実行する構造です。「1 から 10 まで表示する」「全員の名前を順番に確認する」のような場面で使います。

繰り返しでは、いつ終わるのかを必ず決めておくことが重要です。終わりの条件がないと、処理がずっと止まらなくなってしまいます。

繰り返しの例: 1 から 5 まで表示する

flowchart TD A([開始]) --> B[i を 1 にする] B --> C{i は 5 以下か} C -- はい --> D[i を表示する] D --> E[i を 1 増やす] E --> C C -- いいえ --> F([終了]) style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style F fill:#f3e8ff,stroke:#a855f7,color:#581c87

条件を満たす間だけ同じ流れをくり返す

つまずきポイント

繰り返しは便利ですが、終了条件を書き忘れると無限に続いてしまいます。「何回くり返すか」または「どんな条件で止めるか」を必ずセットで考えましょう。

繰り返しの種類 考え方 例
回数が決まっている 何回行うかを先に決める 1 から 100 まで足す
条件で止まる 条件を満たしたら終了する 正しい答えが出るまで入力を続ける

2.5 3つを組み合わせると実用的になる

実際の問題では、順次・分岐・繰り返しをひとつだけで使うことはあまりありません。たとえば「3 人分の点数を順番に見て、60 点以上なら合格、そうでなければ再挑戦と表示する」という処理では、3つすべてが登場します。

組み合わせ例: 3人分の点数を確認する

flowchart TD A([開始]) --> B[1人目から順に点数を見る] B --> C[点数を読み取る] C --> D{60点以上か} D -- はい --> E[合格と表示する] D -- いいえ --> F[再挑戦と表示する] E --> G{次の人がいるか} F --> G G -- はい --> C G -- いいえ --> H([終了]) style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style D fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style G fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style H fill:#f3e8ff,stroke:#a855f7,color:#581c87

順次の流れの中に分岐と繰り返しが入ると考えると整理しやすい

順次

1人目から順番に確認する、という土台の流れを作っています。

分岐

60 点以上かどうかで「合格」か「再挑戦」かを決めています。

繰り返し

次の人がいる間は、同じ確認を何度もくり返しています。

考える順番のコツ

まず順次で全体の流れを作り、次に「どこで判断するか」を探し、最後に「どこを何回くり返すか」を決めると、複雑な問題でも手順を整理しやすくなります。

ミニ演習（15分）: 2つの問題を手順にしてみよう

次の2問について、まずは日本語の文章で手順を書いてみましょう。慣れてきたらフローチャートにも直してみてください。

課題

  1. 1 偶数・奇数判定: 数字を 1 つ受け取り、偶数か奇数かを表示する
  2. 2 1 から 100 の合計: 1 から 100 までの数を順番に足して合計を表示する

考え方のヒント: 偶数・奇数判定は「分岐」が中心、1 から 100 の合計は「繰り返し」が中心です。どちらも、最初に入力や初期値を決める部分では順次が使われています。

第2章のまとめ

  • 順次は、上から順に処理を進める基本の流れ
  • 分岐は、条件によって次の処理を変えるしくみ
  • 繰り返しは、同じ処理を必要な回数だけ行うしくみ
  • 実際のアルゴリズムは、3つの基本構造を組み合わせて作られる
  • 問題を見たら、まず順次・分岐・繰り返しのどれが必要かを考えると整理しやすい

理解度確認クイズ

  1. Q1. 順次・分岐・繰り返しの3つは、それぞれどんな役割を持っていますか？

    回答例を表示

    順次は上から順に処理を進める流れ、分岐は条件によって進む道を変える流れ、繰り返しは同じ処理を何度も実行する流れです。

  2. Q2. 「点数が 60 点以上なら合格、そうでなければ不合格」は、どの基本構造が中心ですか？

    回答例を表示

    分岐が中心です。点数という条件を見て、合格と不合格のどちらの処理に進むかを決めているからです。

  3. Q3. 繰り返しを書くときに、必ず考えなければならないことは何ですか？

    回答例を表示

    いつ繰り返しを終えるか、つまり終了条件です。回数や停止条件がないと、処理が終わらなくなることがあります。
