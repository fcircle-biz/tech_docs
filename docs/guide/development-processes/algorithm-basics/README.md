# アルゴリズム入門 学習ガイドライン

このガイドラインでは、アルゴリズムの基礎をプログラミング初心者向けに240分で学習するカリキュラムを提供しています。「アルゴリズムとは何か」という基本的な疑問から始め、順次・分岐・繰り返しの3つの基本構造、フローチャートによる表現、そして演習を通じた実践的な理解まで、身近な例を交えながらわかりやすく解説します。

## 前提条件

### 必要な環境
- Webブラウザ（Chrome、Firefox、Edge など）
- 筆記用具・ノート（フローチャートの手書き練習に使用）

### 前提知識
- **必須**: コンピュータの基本操作（ファイル作成、Webブラウジング等）
- **不要**: プログラミング経験は一切不要。本ガイドは完全な初心者を対象としています。

## 学習コンテンツ

### [① 導入（20分） - アルゴリズムとは](https://fcircle-biz.github.io/tech_docs/guide/development-processes/algorithm-basics/algorithm-basics-learning-material-01.html)

アルゴリズム = 「誰がやっても同じ結果になる手順」という核心的な定義から学習をスタートします。曖昧な指示と明確な手順の違いを「悪い例 vs 良い例」で対比しながら理解します。

**ミニ演習（5分）**: 「カップラーメンの作り方」を文章で書いてみる。どこが曖昧か気づくことで、アルゴリズムの本質（明確さ・一貫性）を体感します。

### [② 基本構造（40分） - 順次・分岐・繰り返し](https://fcircle-biz.github.io/tech_docs/guide/development-processes/algorithm-basics/algorithm-basics-learning-material-02.html)

アルゴリズムを構成する3つの柱を学びます。

1. **順次** — 上から順に実行する基本の流れ
2. **分岐** — 条件によって処理を変える（例：「点数が60以上なら合格」）
3. **繰り返し** — 同じ処理を何度も行う（例：「1〜10まで表示」）

**ミニ演習（15分）**: 以下を文章で書く
- 偶数・奇数判定
- 1〜100の合計

### [③ フローチャート基礎（30分） - 記号とルール](https://fcircle-biz.github.io/tech_docs/guide/development-processes/algorithm-basics/algorithm-basics-learning-material-03.html)

アルゴリズムを視覚的に表現するフローチャートの書き方を学びます。

**基本記号**: 開始/終了・処理・分岐（ひし形）・入出力

**基本ルール**:
- 上 → 下に流れる
- 分岐にはYes/Noを書く
- 1箱1処理

**NG例も学ぶ（重要）**: 条件が曖昧・矢印が交差・処理が複数混在している悪い例を見ることで、正しい書き方の理由を理解します。

### [④ 演習①（40分） - 基本フローチャート](https://fcircle-biz.github.io/tech_docs/guide/development-processes/algorithm-basics/algorithm-basics-learning-material-04.html)

学んだ知識をフローチャートで表現する練習をします。

**課題1（基礎） — 偶数判定フローチャート**
- 入力：整数
- 出力：偶数 or 奇数

**課題2 — 点数判定**
- 80以上 → 優
- 60以上 → 良
- それ未満 → 不可

**課題3 — 1〜10表示（ループ）**
- 繰り返しの戻り矢印の書き方を練習

**指導ポイント**: 分岐の書き方の徹底、ループの戻り矢印の描き方

### [⑤ 応用（30分） - 線形探索を体験する](https://fcircle-biz.github.io/tech_docs/guide/development-processes/algorithm-basics/algorithm-basics-learning-material-05.html)

アルゴリズムの「応用」を軽く体験します。ここでは探索アルゴリズムの中でも最もシンプルな「線形探索」を扱います。

**線形探索（超シンプル）**: 順番に探すだけ

**例**: 「リストの中から5を探す」
- [3, 8, 1, 5, 9] の中から5を探す手順をフローチャートで表現する

難しい理論ではなく、「今まで学んだ順次・繰り返し・分岐の組み合わせで作れる」という感覚を掴みます。

### [⑥ 演習②（60分） - 総合課題](https://fcircle-biz.github.io/tech_docs/guide/development-processes/algorithm-basics/algorithm-basics-learning-material-06.html)

ここが最重要。演習を通じて理解を定着させます。

**総合課題A（必須） — 数当てゲーム**

仕様：
- 正解は固定（例：7）
- ユーザーが数字を入力する
- 当たり → 「正解！」と表示して終了
- 違う → 「もう一度」と表示して繰り返す

**総合課題B — レベル別課題**

| レベル | 課題 | 内容 |
|-------|------|------|
| 初級 | 合計計算（0で終了） | 数字を入力し続け、0で終了して合計を表示 |
| 中級 | 最大値探索 | 複数の数値入力から最大値を求める |
| 上級（時間あれば） | 簡易ログイン | IDとパスワード判定、3回失敗で終了 |

**指導のコツ**:
- いきなりフローチャートを書かせない → まず文章で手順を書く
- 詰まりやすいポイント：ループの終了条件・分岐の整理

### [⑦ 振り返り（20分） - 解説・まとめ](https://fcircle-biz.github.io/tech_docs/guide/development-processes/algorithm-basics/algorithm-basics-learning-material-07.html)

今日学んだことを振り返り、理解度を確認します。

**確認するポイント**:
- アルゴリズムとは何か、自分の言葉で説明できるか
- 分岐と繰り返しを使い分けられるか
- フローチャートでアルゴリズムを表現できるか

## 学習の進め方

1. **①から順番に進める**: 各セクションは前のセクションの知識を前提としています。
2. **まず文章で書く**: フローチャートを書く前に、手順を日本語の文章で書く習慣をつけましょう。
3. **手書きで練習する**: フローチャートはまず紙に手書きで描くことで理解が深まります。
4. **詰まったら前に戻る**: わからなくなったら②の基本構造に戻って確認しましょう。

## 学習目標

このガイドを完了すると、以下のスキルを身につけることができます：

- アルゴリズムとは何かを自分の言葉で説明できる
- 順次・分岐・繰り返しの3つの基本構造を理解し、使い分けられる
- フローチャートの基本記号と書き方のルールを守って作成できる
- 簡単な問題（数当てゲーム、合計計算など）をフローチャートで表現できる
- 線形探索の仕組みを理解し、フローチャートで表現できる
