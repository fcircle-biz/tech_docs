線形探索を体験する

この章では、アルゴリズムの応用として「線形探索」を体験します。難しい理論ではなく、リストの先頭から順番に調べていくシンプルな流れを、これまで学んだ基本構造の組み合わせとして理解していきましょう。

この章で学ぶこと

  • 線形探索が「順番に探すだけ」のシンプルな探索方法だと理解する
  • 順次・分岐・繰り返しの組み合わせで探索の流れを説明できる
  • リストの中から目的の値を探す手順を文章で整理できる
  • [3, 8, 1, 5, 9] の中から 5 を探す流れをフローチャートで表現できる

5.1 線形探索とは何か

線形探索は、先頭から順番に1つずつ調べて、目的の値を見つける方法です。特別な準備がいらず、とてもわかりやすい探索アルゴリズムなので、応用の最初の題材としてぴったりです。

たとえば、紙に書かれた数字の列から「5」を探すとき、多くの人は左から順番に見ていくはずです。その自然な探し方が、そのまま線形探索です。

重要ポイント

線形探索は新しい種類の魔法のような処理ではありません。これまで学んだ「順次」「分岐」「繰り返し」を組み合わせるだけで作れます。

線形探索のイメージ

順番に見て、違ったら次へ進むという流れをくり返す

5.2 どんな流れで探すのか

線形探索の流れはとても単純です。先頭から順番に値を見て、探したいものと同じかどうかを確認します。同じなら終了し、違うなら次の値を見ます。

使われる基本構造 役割
順次 最初の要素から順番に見る
分岐 今見ている値が目的の値か判定する
繰り返し 見つかるまで、または最後まで同じ確認を続ける

ヒント

線形探索は「1つ調べる → 違ったら次へ」をくり返すだけです。新しい考え方を無理に覚えるより、基本構造の組み合わせとして見ると理解しやすくなります。

5.3 例: [3, 8, 1, 5, 9] から 5 を探す

実際に、[3, 8, 1, 5, 9] の中から 5 を探してみましょう。線形探索では、左から順番に 3、8、1、5 と見ていき、5 を見つけた時点で終了します。

文章での手順

  1. 1. 先頭の値から調べ始める
  2. 2. 今見ている値が 5 か確認する
  3. 3. 5 なら「見つかった」とする
  4. 4. 5 でなければ次の値に進む
  5. 5. 最後まで見てもなければ「見つからない」とする

実際の確認順

3 8 1 5 9

5 を見つけた時点で止まるので、9 までは調べません。

5.4 フローチャートにしてみる

ここまでの流れを、実際にフローチャートにしてみましょう。ポイントは、「今見ている値が 5 か」を分岐で判断し、「違ったら次へ進む」をループで表すことです。

線形探索の参考フローチャート

flowchart TD A([開始]) --> B[先頭の値を見る] B --> C{今の値は5か} C -- はい --> D[見つかったと表示する] C -- いいえ --> E{次の値があるか} E -- はい --> F[次の値へ進む] F --> C E -- いいえ --> G[見つからないと表示する] D --> H([終了]) G --> H style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style E fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style H fill:#f3e8ff,stroke:#a855f7,color:#581c87

見つかれば終了、見つからなければ次へ進む、最後までなければ終了

つまずきポイント

「見つからない場合」を忘れやすいので注意しましょう。最後の値まで見ても見つからなかったとき、どこで終了するかを必ず入れてください。

5.5 応用として見る意味

線形探索を学ぶ価値は、「探索」という少し実践的な問題でも、基本構造の組み合わせで対応できると実感できることにあります。これがわかると、新しい問題に出会っても「まずは順次・分岐・繰り返しに分けてみよう」と考えられるようになります。

順次

リストの先頭から順番に要素を見る流れを作る。

分岐

今見ている値が目的の値かどうかを判断する。

繰り返し

見つかるまで、または最後まで調べる処理をくり返す。

ミニ演習（10分）: 別の値を探してみよう

同じリスト [3, 8, 1, 5, 9] を使って、今度は 7 を探す手順を文章で書き、そのあとフローチャートにしてみましょう。

確認ポイント

  1. 1 最後まで見ても見つからない場合の流れを書いているか
  2. 2 次の要素へ進むループが入っているか
  3. 3 「見つかった」と「見つからない」の両方で終了できるか

コツ: 見つかる例だけでなく、見つからない例も考えると、アルゴリズムの終わり方をきちんと設計できます。

第5章のまとめ

  • 線形探索は、先頭から順番に値を調べるシンプルな探索方法
  • 線形探索は 順次・分岐・繰り返し の組み合わせで表現できる
  • [3, 8, 1, 5, 9] から 5 を探す例では、5 を見つけた時点で終了する
  • 探索では「見つかった場合」だけでなく、見つからない場合の流れも必要
  • 少し応用的な問題でも、基本構造に分けることで理解しやすくなる

理解度確認クイズ

  1. Q1. 線形探索とは、どんな探し方ですか？

    回答例を表示

    リストの先頭から順番に1つずつ値を調べて、目的の値と同じかどうかを確認していく探し方です。

  2. Q2. 線形探索で「次の値へ進む」処理は、どの基本構造と関係が深いですか？

    回答例を表示

    繰り返しです。値を見つけるまで、または最後まで、同じ確認を何度も続けるからです。

  3. Q3. 線形探索のフローチャートで「見つからない場合」の流れも必要なのはなぜですか？

    回答例を表示

    探したい値が必ず存在するとは限らないからです。最後まで見てもなかったときに、どのように終了するかを決めておく必要があります。
