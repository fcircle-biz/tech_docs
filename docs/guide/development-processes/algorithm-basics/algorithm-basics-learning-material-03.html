記号とルール

この章では、アルゴリズムを図で表す「フローチャート」の基本を学びます。開始・終了、処理、分岐、入出力の記号と、読みやすく描くためのルールを整理していきましょう。

この章で学ぶこと

  • フローチャートがアルゴリズムを視覚的に表現する道具だと理解する
  • 開始・終了、処理、分岐、入出力の基本記号を見分けられる
  • 上から下へ流す、分岐に Yes/No を書く、1箱1処理などの基本ルールを説明できる
  • NG例を見て「どこが読みにくいのか」を指摘できる

3.1 フローチャートとは何か

フローチャートは、アルゴリズムの流れを図形と矢印で表したものです。文章だけでは見落としやすい順番や分岐を、ひと目で確認できるのが大きな強みです。

特に初心者のうちは、いきなりプログラムを書くよりも、先にフローチャートで整理すると「何をどんな順番で行うのか」がはっきりします。

重要ポイント

フローチャートは「きれいな絵」を描くものではなく、「処理の流れを誰でも読める形にする」ための道具です。読み手が迷わないことが最優先です。

文章で考えた手順を、図形と矢印で読みやすく整理する

3.2 基本記号を覚える

フローチャートでは、処理の種類ごとに使う図形がほぼ決まっています。まずはよく使う4つの記号を押さえましょう。

記号 意味 書く内容
開始 / 終了 処理のスタートと終わり 開始、終了、完了 など
処理 実際に行う作業 足す、表示する、入力を保存する など
分岐 条件による判断 60点以上か、0かどうか など
入出力 入力を受け取る、結果を表示する 数字を入力、結果を表示 など

基本記号を使った簡単な例

flowchart TD A([開始]) --> B[/数字を入力する/] B --> C{0以上か} C -- はい --> D[正の数または0と表示] C -- いいえ --> E[負の数と表示] D --> F([終了]) E --> F style A fill:#f3e8ff,stroke:#a855f7,color:#581c87 style C fill:#e9d5ff,stroke:#7e22ce,color:#4c1d95 style F fill:#f3e8ff,stroke:#a855f7,color:#581c87

開始・入出力・分岐・処理・終了の5種類が入っている

3.3 フローチャートの基本ルール

記号を覚えただけでは、まだ読みやすいフローチャートにはなりません。図を見た人が迷わないように、いくつかの基本ルールを守る必要があります。

1. 上から下へ流す

基本は上から下、必要に応じて左から右へ補助的に使います。流れの向きがそろうと読みやすくなります。

2. 分岐に Yes / No を書く

分岐の後ろには、どちらが「はい」か「いいえ」かを明記します。判断結果がわからないと図を正しく読めません。

3. 1箱1処理にする

1つの箱にたくさんの動作を詰め込まないようにします。1回読んで意味がわかる量に分けるのがコツです。

ヒント

手書きで描くときも、まず「開始」と「終了」を置いてから途中をつなぐと、全体の流れを見失いにくくなります。

3.4 NG例から学ぶ

良いフローチャートを描く近道は、読みにくい例を知ることです。どこが問題なのかを意識すると、正しい書き方の理由が見えてきます。

読みにくい例

  • 1. 分岐の後ろに Yes / No が書かれていない
  • 2. 1つの処理箱に「入力して計算して表示」と詰め込みすぎている
  • 3. 矢印が交差していて、どこへ進むか追いにくい

読みやすく直すには

  • 1. 分岐の線に Yes / No（または はい / いいえ）を書く
  • 2. 処理を 1 箱 1 動作に分ける
  • 3. 矢印が交差しないように上から下へ並べ直す

注意

条件が曖昧だったり、処理が詰め込まれていたりすると、描いた本人はわかっても他の人は読めません。フローチャートは共有するための図だという意識が大切です。

3.5 描くときの手順

初めてフローチャートを描くときは、いきなり図にしようとせず、短い文章で手順を書いてから図形に置き換えるとスムーズです。

1. 手順を書く

まず文章で「何をするか」を順番に書き出します。

2. 記号に分ける

開始、処理、分岐、入出力に分けて図形を選びます。

3. 矢印でつなぐ

基本は上から下へ流れるように配置します。

4. 読み直す

他の人が読んで迷わないかを確認します。

例: 偶数・奇数判定の流れ

次の章の演習につながる、基本的なフローチャートの例

ミニ演習（10分）: 合否判定をフローチャートにしてみよう

「点数を入力し、60点以上なら合格、そうでなければ不合格と表示する」という手順を、フローチャートで描いてみましょう。

確認ポイント

  1. 1 開始と終了の記号が入っているか
  2. 2 分岐のひし形に「60点以上か」という条件を書けているか
  3. 3 分岐の線に Yes / No か、はい / いいえ が書かれているか

コツ: まずは文章で「入力する → 判定する → 表示する」と並べ、そのあとで記号に置き換えると描きやすくなります。

第3章のまとめ

  • フローチャートは、アルゴリズムを図形と矢印で表す方法
  • 基本記号は 開始 / 終了・処理・分岐・入出力 の4種類をまず覚える
  • 基本ルールは 上から下へ流す・分岐に Yes / No を書く・1箱1処理
  • NG例を見ると、読みやすい図にするためのポイントがよくわかる
  • 描くときは、先に文章で手順を書いてから図にすると整理しやすい

理解度確認クイズ

  1. Q1. ひし形の記号は、どんな場面で使いますか？

    回答例を表示

    条件を判断して、次に進む処理を分けるときに使います。たとえば「60点以上か」「0かどうか」などです。

  2. Q2. 分岐のあとに Yes / No を書くのは、なぜ大切ですか？

    回答例を表示

    どちらの線がどんな条件の結果なのかを明確にするためです。書かれていないと、読み手が進み方を誤解する可能性があります。

  3. Q3. 1つの処理箱に複数の動作を詰め込むと、どんな問題がありますか？

    回答例を表示

    何をしている箱なのかが一目でわかりにくくなり、修正や説明もしづらくなります。処理はできるだけ1箱1動作に分けた方が読みやすくなります。
