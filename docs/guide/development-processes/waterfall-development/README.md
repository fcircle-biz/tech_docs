# ウォーターフォール開発入門 学習ガイドライン

このガイドラインでは、ウォーターフォール開発の基礎を**プログラミング未経験者**向けに段階的に学習するためのカリキュラムを提供しています。専門用語をできるだけ噛み砕いて説明し、身近な例えを使って分かりやすく解説します。

## 前提条件

### 必要な環境
- パソコン（Windows、Mac、Linuxいずれでも可）
- インターネット接続環境
- メモ帳やテキストエディタ（文字を書けるアプリなら何でもOK）
- 紙とペン（メモ用）

### 参考リソース
- [独立行政法人 情報処理推進機構（IPA）](https://www.ipa.go.jp/) - IT資格や開発手法の公式情報
- [ITパスポート試験 シラバス](https://www.ipa.go.jp/shiken/syllabus/ps6vr70000012o8a-att/syllabus_ip_ver6_2.pdf) - IT基礎知識の学習範囲

### 前提知識
- **必須**: 特になし（完全な初心者から始められます）
- **推奨**: パソコンの基本操作ができること

## 学習コンテンツ

### [1. そもそも「システム開発」とは何か？](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-01.html)
「システム」や「開発」という言葉の意味から丁寧に解説します。身近な例（スマホのアプリ、銀行のATM、コンビニのレジなど）を使って、システム開発が私たちの生活にどう関わっているかを学びます。なぜ「開発手法」が必要なのか、その理由も理解します。

### [2. ウォーターフォール開発とは？〜水が流れるように進む開発〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-02.html)
ウォーターフォール（waterfall＝滝）という名前の由来と、開発が「上から下へ」順番に流れていく仕組みを解説します。家を建てることに例えながら、なぜこの順番が大切なのかを分かりやすく説明します。他の開発手法との違いもシンプルに紹介します。

### [3. 要件定義〜「何を作るか」を決める〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-03.html)
システム開発の最初のステップ「要件定義」を学びます。お客様の「こんなものが欲しい」という希望を聞き取り、「何を作るのか」を明確にする作業です。ケーキの注文に例えながら、なぜ最初にしっかり決めることが重要なのかを理解します。

### [4. 基本設計〜「どんな形にするか」を決める〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-04.html)
要件定義で決めた「作るもの」を、どんな形・見た目にするかを決める「基本設計」を学びます。家の間取り図を描くような作業です。画面のレイアウトやボタンの配置など、ユーザーから見える部分の設計について説明します。

### [5. 詳細設計〜「どうやって動かすか」を決める〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-05.html)
基本設計をさらに細かく、プログラマーが作業できるレベルまで具体化する「詳細設計」を学びます。家で例えると、電気の配線図や水道管の設計図を描くような作業です。この設計書があれば、誰でも同じものを作れるようになります。

### [6. 実装（プログラミング）〜設計図通りに作る〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-06.html)
設計書に基づいて実際にプログラムを書く「実装」フェーズを学びます。プログラミングとは何か、どんな作業をするのかを概念的に理解します。専門的なコードは書きませんが、プログラマーがどんな仕事をしているかがイメージできるようになります。

### [7. テスト〜ちゃんと動くか確認する〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-07.html)
作ったプログラムが正しく動くかをチェックする「テスト」を学びます。単体テスト、結合テスト、総合テストという3つの段階があることを説明します。車の検査に例えながら、なぜ何度もテストするのかを理解します。

### [8. リリースと運用〜完成品を届けて使ってもらう〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-08.html)
完成したシステムをお客様に届ける「リリース」と、その後の「運用・保守」について学びます。家が完成した後の引き渡しと、その後の点検・修理にあたる作業です。システムは作って終わりではないことを理解します。

### [9. ウォーターフォール開発のメリットとデメリット](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-09.html)
ここまで学んだウォーターフォール開発の良いところと課題を整理します。どんなプロジェクトに向いているのか、逆にどんな場合は別の方法が良いのかを学びます。現場でよく使われる理由も解説します。

### [10. プロジェクト管理の基本〜チームで開発を成功させる〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-10.html)
システム開発は一人で行うものではありません。チームでプロジェクトを進めるための基本的な考え方を学びます。スケジュール管理、役割分担、進捗報告など、プロジェクトを成功に導くポイントを紹介します。

### [11. ドキュメント（文書）の重要性](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-11.html)
ウォーターフォール開発では「ドキュメント」（設計書や報告書などの文書）がとても重要です。なぜ文書を残すのか、どんな文書があるのか、文書があることで何が助かるのかを学びます。

### [12. ウォーターフォール開発の実践例〜架空のプロジェクトで体験〜](https://fcircle-biz.github.io/tech_docs/guide/development-processes/waterfall-development/waterfall-development-learning-material-12.html)
学んだ知識を使って、架空の「図書館の本の貸出システム」を開発するプロジェクトをシミュレーションします。各フェーズでどんな作業をするのか、具体的な例を通じて理解を深めます。

## 学習の進め方

1. **順番通りに学習する**: 第1章から順番に進めてください。ウォーターフォール開発自体が「順番」を大切にする手法なので、学習も順番が重要です
2. **身近な例で考える**: 各章で紹介する例（家づくり、ケーキ注文など）を自分の経験に置き換えて考えてみましょう
3. **分からない言葉はメモする**: 初めて聞く用語が出てきたらメモして、次の章で理解が深まるか確認しましょう
4. **完璧を目指さない**: 最初から全てを理解する必要はありません。まずは全体の流れを掴むことを目標にしましょう
5. **復習する**: 第12章の実践例に取り組む前に、第1章〜第11章を軽く見直すと効果的です

## 学習目標

このガイドを完了すると、以下のスキルを身につけることができます：

- 「システム開発」とは何かを人に説明できる
- ウォーターフォール開発の5つのフェーズ（要件定義、設計、実装、テスト、リリース・運用）を説明できる
- 各フェーズで「何をするのか」「なぜそれが必要なのか」を理解できる
- システム開発のプロジェクトに参加したとき、今どのフェーズにいるかが分かる
- ウォーターフォール開発が向いているプロジェクト、向いていないプロジェクトの違いが分かる
- 開発現場でよく使われる基本的な用語（要件、設計、テスト、リリースなど）を理解できる
- チームでシステム開発を進めるときの基本的な考え方を理解できる
- ドキュメント（文書）の重要性を理解し、なぜ記録を残すのかを説明できる
