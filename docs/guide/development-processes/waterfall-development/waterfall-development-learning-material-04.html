基本設計〜「どんな形にするか」を決める〜

要件定義で決めた「作るもの」を、どんな形・見た目にするかを決める「基本設計」を学びます。家の間取り図を描くような作業です。

この章で学ぶこと

  • 基本設計とは何かを理解する
  • 画面設計・データベース設計・インターフェース設計の役割を知る
  • 基本設計で作成される設計書の種類を理解する

4.1 基本設計とは？

第3章で学んだ「要件定義」では、「何を作るか」を決めました。次の段階である基本設計では、「どんな形にするか」を決めます。

家を建てることに例えると、要件定義は「3LDKで庭付きの家が欲しい」と決めることでした。基本設計は、その希望をもとに間取り図を描く作業に相当します。

基本設計の例（家づくり）

  • リビングの広さ: 20畳
  • キッチンの配置: 対面式
  • 窓の位置: 南側に大きな窓
  • 収納スペース: 各部屋にクローゼット

システム開発における基本設計も同じです。ユーザーの目に見える部分（画面の配置やボタン、データの流れなど）を具体的に決めていきます。

4.2 要件定義と基本設計の違い

要件定義と基本設計は混同しやすいですが、明確な違いがあります。

要件定義

「何を」作るか

  • • 機能を決める
  • • 目的を明確にする
  • • 顧客の要望を整理

基本設計

「どんな形で」作るか

  • • 画面レイアウト
  • • データ構造
  • • システム構成

4.3 基本設計の主な内容

基本設計では、主に以下の3つの設計を行います。

4.3.1 画面設計（UI設計）

ユーザーが実際に見る画面のレイアウトやデザインを決めます。

画面設計で決めること

  • • ボタンの配置と大きさ
  • • 入力欄の位置と種類
  • • メニューの構成
  • • 画面遷移（どのボタンを押すとどの画面に移動するか）
  • • エラーメッセージの表示方法

例: ログイン画面の設計

  • • 画面の上部にタイトル「ログイン」を表示
  • • ユーザー名入力欄（幅300px）
  • • パスワード入力欄（幅300px、伏字表示）
  • • ログインボタン（青色、幅150px）
  • • 「パスワードを忘れた方はこちら」のリンク

4.3.2 データベース設計

システムが扱うデータをどう保存するかを決めます。データベースは、情報を整理して保管する「電子的な棚」のようなものです。

データベース設計で決めること

  • • どんな情報を保存するか（顧客名、住所、電話番号など）
  • • データの種類（文字、数字、日付など）
  • • データ同士の関係性
  • • 検索しやすい構造

4.3.3 インターフェース設計（外部連携設計）

他のシステムとどうやってデータをやり取りするかを決めます。

インターフェース設計の例

  • • ECサイトから決済システムへ金額データを送信
  • • 在庫管理システムから商品データを取得
  • • メール配信システムに顧客情報を連携

4.4 基本設計で作る設計書

基本設計では、次のような設計書（ドキュメント）を作成します。

設計書の種類 内容
画面設計書 各画面のレイアウト、ボタン配置、画面遷移図
データベース設計書 テーブル構造、データ項目、データ型、関連図
システム構成図 サーバーの構成、ネットワーク構成、機器配置
インターフェース仕様書 外部システムとのデータ連携方法

4.5 基本設計のポイント

良い基本設計には、以下のポイントがあります。

使いやすさ

ユーザーが直感的に操作できる画面設計

拡張性

将来の機能追加に対応できる設計

セキュリティ

データを安全に保護する仕組み

性能

多くのユーザーが同時に使っても遅くならない

重要: 顧客レビュー

基本設計が完成したら、顧客に確認してもらうことが大切です。この段階で修正するのは比較的簡単ですが、プログラムを作った後の修正はとても大変です。

4.6 基本設計と詳細設計の関係

基本設計の次は「詳細設計」です。この2つの違いを理解しておきましょう。

理解度確認クイズ

  1. Q1. 基本設計では「何を」決めますか？

    回答例を表示

    「どんな形にするか」を決めます。具体的には、画面のレイアウト、データベースの構造、外部システムとの連携方法などを設計します。

  2. Q2. 基本設計の3つの主な内容を挙げてください。

    回答例を表示

    ①画面設計（UI設計）、②データベース設計、③インターフェース設計（外部連携設計）の3つです。

  3. Q3. 基本設計が完成したら、なぜ顧客に確認してもらうことが重要ですか？

    回答例を表示

    この段階で修正するのは比較的簡単ですが、プログラムを作った後の修正はとても大変で時間もコストもかかるからです。早い段階で確認することで、顧客の期待と実際の設計のズレを防げます。

第4章のまとめ

  • 基本設計は、要件定義で決めた「何を作るか」を「どんな形にするか」に具体化する段階
  • 主な内容は、画面設計、データベース設計、インターフェース設計の3つ
  • 画面設計書、データベース設計書、システム構成図などの設計書を作成する
  • 使いやすさ、拡張性、セキュリティ、性能を考慮した設計が重要
  • 基本設計完成後は顧客に確認してもらい、早期に修正することがポイント
