実装（プログラミング）〜設計図通りに作る〜

設計書に基づいて実際にプログラムを書く「実装」フェーズを学びます。プログラミングとは何か、どんな作業をするのかを概念的に理解します。

この章で学ぶこと

  • 実装（プログラミング）フェーズの役割と重要性を理解する
  • プログラミングとは何かを身近な例で理解する
  • 実装作業での注意点とコーディング規約の重要性を学ぶ

6.1 実装（プログラミング）とは？

前の章までで、システムの「設計図」が完成しました。いよいよ「実装」のフェーズです。

実装とは、詳細設計書をもとに、実際にプログラムコードを書く作業のことです。

家づくりで例えると

詳細設計書は「建築図面」、実装は「実際に家を建てる工事」にあたります。 設計図があっても、それを実際に形にするには職人の技術が必要です。 プログラミングも同様に、設計書を見ながらコンピューターが理解できる言葉（プログラム言語）で書いていきます。

6.2 プログラミングとは何か？

プログラミングと聞くと難しく感じるかもしれませんが、本質的には「コンピューターへの指示書を書くこと」です。

人間の言葉

「ユーザーがボタンを押したら、データを保存して画面に『保存しました』と表示してください」

プログラムの言葉

if (button.clicked) {
  data.save();
  show("保存しました");
}

人間の言葉をコンピューターが理解できる形に翻訳していく作業が、プログラミングです。

6.3 プログラミング言語

コンピューターへの指示を書くために使う言葉を「プログラミング言語」と呼びます。

人間の言語に日本語、英語、中国語などがあるように、プログラミング言語にも様々な種類があります。

6.4 実装作業の流れ

実際のプログラミング作業は、以下のようなステップで進みます。

各ステップの詳細

1

詳細設計書を読む

どんな機能を実装するか、どのように動作させるかを理解します

2

コードを書く

プログラミング言語を使って、設計書の内容を実際のコードにします

3

動作確認する

書いたコードが正しく動くかをテストします

4

バグを修正

もし間違いがあれば修正して、再度確認します

6.5 コーディング規約の重要性

複数のプログラマーがチームで開発する場合、「コーディング規約」というルールを決めておくことが重要です。

コーディング規約とは？

プログラムの書き方に関する共通ルールのことです。変数の命名方法、インデント（字下げ）の仕方、コメントの書き方などを統一します。

規約がないと...

  • • 他の人が書いたコードが読みにくい
  • • バグを見つけにくい
  • • 修正や変更が難しい
  • • チーム全体の効率が下がる

規約があると...

  • • 誰が書いても読みやすい
  • • 品質が安定する
  • • 引き継ぎがスムーズ
  • • 開発スピードが上がる

6.6 バージョン管理の重要性

プログラムを書いているとき、「以前の状態に戻したい」ということがよくあります。 そのために「バージョン管理ツール」を使います。

代表的なバージョン管理ツール: Git

現在、最も広く使われているバージョン管理ツールが「Git（ギット）」です。

Gitでできること

  • • 変更履歴を自動的に記録できる
  • • 過去の任意の時点に簡単に戻せる
  • • 複数人が同時に作業できる
  • • 誰がいつ何を変更したかが分かる

文書作成で例えると

Word文書で「上書き保存」する前に「別名で保存」して、複数のバージョンを残しておくのに似ています。 Gitを使えば、これが自動的にできるだけでなく、変更の差分も簡単に確認できます。

6.7 プログラマーの心構え

実装フェーズでは、以下のような心構えが大切です。

設計書を正確に読む

思い込みや自己判断で進めず、設計書通りに実装する

こまめにコメントを書く

コードに説明を書いて、後から見ても分かるようにする

小まめにテストする

書いたらすぐ動作確認して、問題を早期発見する

分からないことは相談する

一人で悩まず、チームメンバーに相談して解決する

重要ポイント

実装フェーズは、ウォーターフォール開発において「設計を実際の形にする」重要なフェーズです。 設計書通りに正確に作ることと、チームでのルール（コーディング規約）を守ることが成功の鍵です。

理解度確認クイズ

  1. Q1. 「実装」とは何をすることですか？簡単に説明してください。

    回答例を表示

    詳細設計書をもとに、実際にプログラムコードを書く作業のこと。コンピューターが理解できる言葉（プログラミング言語）で、設計書の内容を実現します。

  2. Q2. コーディング規約がなぜ重要なのですか？

    回答例を表示

    複数のプログラマーがチームで開発する際に、コードの書き方を統一することで、誰が書いても読みやすく、品質が安定し、開発効率が上がるため。

  3. Q3. バージョン管理ツール（Git）を使うメリットを2つ挙げてください。

    回答例を表示

    ①変更履歴を自動記録し、過去の状態に簡単に戻せる、②複数人が同時に作業でき、誰がいつ何を変更したかが明確になる、など。

第6章のまとめ

  • 実装は、詳細設計書をもとに実際にプログラムコードを書く作業
  • プログラミングとは、コンピューターへの指示書を書くこと
  • プログラミング言語には、Python、JavaScript、Javaなど様々な種類がある
  • コーディング規約を守ることで、チーム開発が円滑に進む
  • バージョン管理ツール（Git）で、変更履歴を管理し、安全に開発を進める
  • 設計書通りに正確に実装し、こまめにテストすることが重要
