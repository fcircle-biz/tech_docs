9.1 ソフトウェア品質管理の基本

品質管理は、ソフトウェアが要求される品質基準を満たすことを保証する活動です。ウォーターフォール開発では、各フェーズで体系的に品質を確保します。

品質保証（QA）と品質管理（QC）

観点品質保証（QA）品質管理（QC）
焦点プロセス成果物
目的欠陥を予防する欠陥を検出する
活動プロセス改善、標準化テスト、レビュー
時期プロジェクト全体各フェーズの成果物完成時

9.2 品質特性（ISO/IEC 25010）

ISO/IEC 25010では、ソフトウェア品質を8つの特性で定義しています。

graph TD A["ソフトウェア
品質特性"] --> B["機能適合性"] A --> C["性能効率性"] A --> D["互換性"] A --> E["使用性"] A --> F["信頼性"] A --> G["セキュリティ"] A --> H["保守性"] A --> I["移植性"]

各品質特性の詳細

  • 機能適合性：要求された機能を提供する
  • 性能効率性：応答時間、スループット等の性能
  • 互換性：他システムとの共存・相互運用
  • 使用性：ユーザーが容易に使える
  • 信頼性：障害なく安定動作する
  • セキュリティ：不正アクセスから保護
  • 保守性：修正・拡張が容易
  • 移植性：異なる環境で動作する

9.3 レビュー技法

レビューは、成果物の欠陥を早期に発見する重要な品質管理活動です。

1. インスペクション（Inspection）

インスペクションの特徴
  • 形式的で厳格なレビュー
  • チェックリストを使用
  • モデレーター（司会者）が主導
  • 欠陥の定量的な記録
手順
  1. 計画：レビュー対象と参加者を決定
  2. 概要説明：作成者が概要を説明
  3. 準備：各自が事前にレビュー
  4. 検査会議：欠陥を指摘・記録
  5. 修正：指摘事項を修正
  6. フォローアップ：修正確認

2. ウォークスルー（Walkthrough）

作成者が主導して、成果物を順番に説明しながらレビューします。インスペクションより形式的でなく、知識共有の側面も強いです。

9.4 品質メトリクス

品質を定量的に測定する指標を設定します。

品質メトリクスの例
メトリクス計算式目標値例
不具合密度不具合数 / KLOC（1000行あたり）3件以下/KLOC
テストカバレッジ実行された行数 / 全行数80%以上
レビュー効率レビューで検出した欠陥数 / レビュー時間2件以上/時間
コード品質循環的複雑度10以下/関数

9.5 リスク管理の基本

リスク管理は、プロジェクトの脅威を事前に識別し、対策を講じるプロセスです。

リスク管理プロセス

graph LR A[リスク識別] --> B[リスク分析] B --> C[リスク対応計画] C --> D[リスク監視] D --> A

9.6 リスク識別と分析

プロジェクトで起こりうるリスクを洗い出し、影響度と発生確率を分析します。

リスク登録簿の例
リスク影響度発生確率優先度対応策
キーメンバーの退職高中高知識共有、ドキュメント化
外部API仕様変更高低中代替API検討
テスト環境不足中高高早期環境準備
要件追加要求中中中変更管理プロセス徹底

9.7 リスク対応戦略

識別したリスクに対して、以下の4つの戦略で対応します。

4つのリスク対応戦略

  • 回避（Avoid）：リスクの原因を除去する（例：実績のない新技術を使わない）
  • 軽減（Mitigate）：発生確率や影響度を下げる（例：複数ベンダーから見積もり）
  • 転嫁（Transfer）：他者にリスクを移す（例：保険加入、外部委託）
  • 受容（Accept）：リスクを受け入れる（例：コンティンジェンシープラン準備）

理解度確認クイズ
  1. 品質保証（QA）と品質管理（QC）の違いを説明してください。
  2. ISO/IEC 25010で定義される品質特性を5つ挙げてください。
  3. インスペクションとウォークスルーの違いは何ですか？
  4. リスク管理プロセスの4つのステップを挙げてください。
  5. 4つのリスク対応戦略を挙げてください。
解答例
  1. QAはプロセスに焦点を当て欠陥を予防し、QCは成果物に焦点を当て欠陥を検出します。
  2. 例：機能適合性、性能効率性、互換性、使用性、信頼性
  3. インスペクションは形式的で厳格、チェックリスト使用。ウォークスルーは作成者主導で比較的非形式的。
  4. リスク識別、リスク分析、リスク対応計画、リスク監視
  5. 回避、軽減、転嫁、受容
第9章のまとめ
  • QAはプロセス改善、QCは成果物検査
  • ISO/IEC 25010で品質を8つの特性で定義
  • インスペクションとウォークスルーで早期に欠陥発見
  • 品質メトリクスで定量的に品質を測定
  • リスク管理は識別→分析→対応計画→監視のサイクル
  • リスク対応は回避・軽減・転嫁・受容の4戦略
