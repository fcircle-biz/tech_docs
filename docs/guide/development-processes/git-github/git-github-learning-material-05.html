コマンドライン入門 - ターミナルの基本操作

コマンドライン（ターミナル）の基本を一から学びます。カレントディレクトリやパスの概念、基本コマンドの使い方など、Gitを使う上で必要な知識を習得しましょう。

この章で学ぶこと

  • コマンドラインとは何か、なぜ必要かを理解する
  • ターミナル/Git Bashの起動方法を知る
  • カレントディレクトリとパスの概念を理解する
  • 基本コマンド（cd, ls, pwd, mkdir等）を使いこなす
  • Tab補完とコマンド履歴を活用できる

5.1 コマンドラインとは何か

コマンドライン（Command Line）は、キーボードで文字を入力してコンピュータを操作する方法です。マウスでアイコンをクリックするのではなく、コマンド（命令）を文字で入力してファイルを操作したり、プログラムを実行したりします。

GUI vs CUI（コマンドライン）

なぜコマンドラインが必要なのか？

  • Gitはコマンドラインベース - Gitの多くの機能はコマンドで使用します
  • 効率的な操作 - 慣れればマウスより速く操作できます
  • 自動化が可能 - コマンドを組み合わせて作業を自動化できます
  • リモートサーバー操作 - サーバー管理では必須のスキルです
  • 開発者の標準ツール - 多くのプログラマーが日常的に使用します

主なコマンドライン環境

本教材での推奨環境

本教材では、WindowsユーザーはGit Bashを、macOS/Linuxユーザーはターミナルをメインとして説明します。Git Bashを使えば、OSに関わらず同じコマンドが使えます。

5.2 ターミナル/Git Bashの起動方法

OSごとにターミナル（コマンドライン）の起動方法が異なります。以下を参考に、自分の環境で起動してみましょう。

Windows - Git Bashの起動

  1. スタートメニューから起動: 「スタート」→「Git」→「Git Bash」をクリック
  2. 右クリックメニューから起動: エクスプローラーで任意のフォルダを右クリック→「Git Bash Here」を選択
  3. 検索から起動: Windowsキーを押して「git bash」と入力→Enter

起動すると、黒い画面に緑色の文字で「ユーザー名@コンピュータ名」が表示されます。

macOS - ターミナルの起動

  1. Launchpadから起動: 「Launchpad」→「その他」→「ターミナル」をクリック
  2. Spotlightから起動: Command + Spaceキーを押して「ターミナル」または「terminal」と入力→Enter
  3. Finderから起動: Finder→「アプリケーション」→「ユーティリティ」→「ターミナル」をダブルクリック

起動すると、白い画面に黒い文字で「コンピュータ名:カレントディレクトリ ユーザー名$」が表示されます。

Linux - ターミナルの起動

  • ショートカットキー: Ctrl + Alt + T（多くのディストリビューション共通）
  • アプリケーションメニュー: 「アプリケーション」→「アクセサリ」または「システムツール」→「ターミナル」
  • デスクトップ環境の検索: 検索バーで「terminal」と入力

プロンプトとは？

ターミナルを起動すると表示される「$」や「>」の記号をプロンプトと呼びます。これが表示されている状態が「コマンド入力待ち」の状態です。プロンプトの左側には、現在のユーザー名やディレクトリが表示されます。

5.3 カレントディレクトリとパスの概念

コマンドラインでは、常に「今どこにいるか」を意識する必要があります。この「今いる場所」をカレントディレクトリ（作業ディレクトリ）と呼びます。

パス（Path）とは

パスは、ファイルやフォルダの場所を示す「住所」のようなものです。パスには2種類あります：

絶対パス（Absolute Path）

ルート（最上位）からの完全な道筋を示すパスです。

絶対パスの例 
# Windows
C:\Users\username\Documents\project

# macOS/Linux
/home/username/Documents/project

相対パス（Relative Path）

カレントディレクトリを基準とした道筋を示すパスです。

相対パスの例 
# 現在の場所からの相対位置
./subfolder/file.txt  # 現在のディレクトリ内
../parent/file.txt    # 1つ上の階層
../../grandparent/    # 2つ上の階層

特殊なディレクトリ記号

  • . （ドット）- 現在のディレクトリ
  • .. （ドットドット）- 1つ上の親ディレクトリ
  • ~ （チルダ）- ホームディレクトリ（ユーザーのホームフォルダ）
  • / （スラッシュ）- ルートディレクトリ（最上位）

ディレクトリ構造の例

階層構造を持つディレクトリツリー。各フォルダは親子関係で繋がっています。

5.4 基本コマンド一覧

Gitを使う前に、まずは基本的なコマンドラインの操作に慣れましょう。以下は、ファイルやディレクトリを操作するための基本コマンドです。

pwd 現在のディレクトリを表示

pwd（Print Working Directory）は、現在いる場所（カレントディレクトリ）の絶対パスを表示します。

pwdコマンドの使用例 
# 現在のディレクトリを確認
$ pwd
/home/username/Documents

ls ファイル・ディレクトリ一覧を表示

ls（List）は、現在のディレクトリ内のファイルとフォルダの一覧を表示します。

lsコマンドの使用例 
# 基本的な使い方
$ ls
Desktop  Documents  Downloads  Pictures

# 詳細情報を表示（-l オプション）
$ ls -l

# 隠しファイルも表示（-a オプション）
$ ls -a

# 詳細情報＋隠しファイル（オプション組み合わせ）
$ ls -la

cd ディレクトリを移動

cd（Change Directory）は、カレントディレクトリを変更します。最もよく使うコマンドの1つです。

cdコマンドの使用例 
# Documentsフォルダに移動
$ cd Documents

# 1つ上の階層に移動
$ cd ..

# ホームディレクトリに移動
$ cd ~

# 絶対パスで移動
$ cd /home/username/Documents/project

mkdir ディレクトリを作成

mkdir（Make Directory）は、新しいディレクトリ（フォルダ）を作成します。

mkdirコマンドの使用例 
# 新しいフォルダを作成
$ mkdir my-project

# 複数のフォルダを一度に作成
$ mkdir folder1 folder2 folder3

# 親ディレクトリも含めて作成（-p オプション）
$ mkdir -p parent/child/grandchild

rm ファイル・ディレクトリを削除

rm（Remove）は、ファイルやディレクトリを削除します。削除したファイルは復元できないので注意が必要です。

rmコマンドの使用例 
# ファイルを削除
$ rm file.txt

# ディレクトリを削除（-r オプション: 再帰的削除）
$ rm -r my-folder

# 確認なしで削除（-f オプション: 強制削除）
$ rm -rf my-folder  # 注意: 取り消せません！

警告: rmコマンドで削除したファイルはゴミ箱に移動せず、完全に消去されます。特に「rm -rf」は非常に強力なので、実行前に必ず確認しましょう。

cp ファイル・ディレクトリをコピー

cp（Copy）は、ファイルやディレクトリをコピーします。

cpコマンドの使用例 
# ファイルをコピー
$ cp file.txt file_backup.txt

# ディレクトリをコピー（-r オプション）
$ cp -r folder1 folder2

mv ファイル・ディレクトリを移動/名前変更

mv（Move）は、ファイルやディレクトリを移動したり、名前を変更したりします。

mvコマンドの使用例 
# ファイル名を変更
$ mv old_name.txt new_name.txt

# ファイルを別のディレクトリに移動
$ mv file.txt Documents/

# ディレクトリを移動
$ mv folder1 Documents/folder1

基本コマンド早見表

コマンド 意味 例
pwd 現在のディレクトリを表示 pwd
ls ファイル一覧を表示 ls -la
cd ディレクトリを移動 cd Documents
mkdir ディレクトリを作成 mkdir my-project
rm ファイル・ディレクトリを削除 rm file.txt
cp ファイルをコピー cp src.txt dst.txt
mv ファイルを移動/名前変更 mv old.txt new.txt

5.5 コマンドのオプションと引数

多くのコマンドは、オプションと引数を使って動作をカスタマイズできます。

コマンドの構造

コマンドの基本構造 
# 基本形
コマンド [オプション] [引数]

# 例: lsコマンドの場合
ls -la Documents/
│  │  └─ 引数（対象となるディレクトリ）
│  └─ オプション（-l: 詳細表示、-a: 隠しファイルも表示）
└─ コマンド本体

オプション（Options）

コマンドの動作を変更する設定です。通常、ハイフン（-）で始まります。

  • -a : 短いオプション（1文字）
  • --all : 長いオプション（単語）
  • 複数指定可能: -la = -l -a

引数（Arguments）

コマンドが操作する対象（ファイル名やディレクトリ名など）です。

  • ファイル名: file.txt
  • ディレクトリ名: Documents/
  • 複数指定可能: file1 file2 file3
オプションと引数の使用例 
# オプションなし
$ ls
Desktop  Documents  Downloads

# オプションあり（-l: 詳細表示）
$ ls -l

# オプションと引数
$ ls -l Documents/

# 複数オプション
$ ls -la  # -l と -a を組み合わせ

# 長いオプション
$ ls --all --human-readable

5.6 Tab補完とコマンド履歴の活用

コマンドラインには、入力を効率化するための便利な機能があります。

Tab補完（Tab Completion）

Tabキーを押すことで、ファイル名やディレクトリ名を自動補完できます。長い名前を最後まで入力する必要がなくなり、入力ミスも防げます。

使い方：

  1. コマンドやファイル名の最初の数文字を入力
  2. Tabキーを押す
  3. 候補が1つなら自動補完、複数ならリストが表示される
  4. もう一度Tabを押すと候補が表示される（環境により動作が異なる）
Tab補完の例 
# "Doc" と入力してTabキーを押すと...
$ cd Doc[Tab]
$ cd Documents/  # 自動補完される

# 候補が複数ある場合は、もう一度Tabで一覧表示
$ cd D[Tab][Tab]
Desktop/  Documents/  Downloads/

コマンド履歴（Command History）

過去に実行したコマンドを矢印キーで呼び出すことができます。長いコマンドを何度も入力する手間が省けます。

主なショートカット：

  • ↑（上矢印） - 1つ前のコマンドを表示
  • ↓（下矢印） - 1つ後のコマンドを表示
  • Ctrl + R - コマンド履歴を検索（インクリメンタルサーチ）
  • history - 過去のコマンド履歴を一覧表示
コマンド履歴の例 
# 履歴を表示
$ history
  1  pwd
  2  ls -la
  3  cd Documents
  4  mkdir my-project

# 上矢印キーで過去のコマンドを呼び出し
$ [↑キー] → mkdir my-project が表示される

その他の便利なキーボードショートカット

ショートカット 動作
Ctrl + C 実行中のコマンドを中止
Ctrl + L 画面をクリア（clearコマンドと同じ）
Ctrl + A カーソルを行頭に移動
Ctrl + E カーソルを行末に移動
Ctrl + U カーソル位置から行頭まで削除
Ctrl + K カーソル位置から行末まで削除

5.7 よくあるエラーと対処法

コマンドラインを使い始めると、様々なエラーメッセージに遭遇します。ここでは、よくあるエラーとその対処法を紹介します。

「command not found」エラー

bash: git: command not found

原因: コマンドが見つからない、または入力ミス

対処法:

  • コマンドのスペルミスを確認
  • 必要なソフトウェアがインストールされているか確認
  • PATHが正しく設定されているか確認

「No such file or directory」エラー

cd: Documents: No such file or directory

原因: 指定したファイルやディレクトリが存在しない

対処法:

  • ファイル名やディレクトリ名のスペルミスを確認
  • lsコマンドで存在を確認
  • カレントディレクトリが正しいか確認（pwd）
  • 大文字小文字を正確に入力（Unix系は区別する）

「Permission denied」エラー

rm: file.txt: Permission denied

原因: ファイルやディレクトリへのアクセス権限がない

対処法:

  • 管理者権限で実行（sudo コマンド - Linux/macOS）
  • ファイルの所有者やパーミッションを確認
  • 書き込み権限のある場所で作業する

コマンドが終わらない（固まった）

症状: プロンプトが表示されず、コマンドが実行され続ける

対処法:

  • Ctrl + Cを押して処理を中断
  • 長時間かかる処理の場合は、完了を待つ
  • 入力待ちの場合は、必要な入力を行う

実習: コマンドライン操作を試してみよう

以下の手順で、コマンドライン操作を実際に体験してみましょう。

  1. ターミナルを起動する
    • Windows: Git Bashを起動
    • macOS/Linux: ターミナルを起動
  2. 現在の場所を確認 
    $ pwd
  3. ファイル一覧を表示 
    $ ls
$ ls -la
  4. 練習用フォルダを作成 
    $ mkdir git-practice
$ ls  # フォルダが作成されたか確認
  5. 作成したフォルダに移動 
    $ cd git-practice
$ pwd  # 場所を確認
  6. Tab補完を試す
    • 「cd g」と入力してTabキーを押してみる
    • 自動補完されることを確認
  7. コマンド履歴を使う
    • ↑キーを何度か押して、過去のコマンドを表示
    • historyコマンドで履歴一覧を表示
  8. 1つ上のディレクトリに戻る 
    $ cd ..
$ pwd  # 場所を確認

理解度確認クイズ

  1. Q1. コマンドラインとは何ですか？GUIとの違いを説明してください。

    回答を表示

    コマンドラインは、キーボードで文字（コマンド）を入力してコンピュータを操作する方法です。GUI（グラフィカルユーザーインターフェース）はマウスでアイコンをクリックして操作しますが、コマンドラインは文字だけで操作します。コマンドラインは効率的で自動化が可能なため、開発者の間で広く使われています。

  2. Q2. カレントディレクトリとは何ですか？確認するコマンドは？

    回答を表示

    カレントディレクトリ（作業ディレクトリ）は、現在コマンドラインで操作している場所（フォルダ）のことです。pwdコマンドで確認できます。pwdは「Print Working Directory」の略で、現在のディレクトリの絶対パスを表示します。

  3. Q3. 絶対パスと相対パスの違いを説明してください。

    回答を表示

    絶対パスは、ルート（最上位）からの完全な道筋を示すパスです（例: /home/username/Documents）。

    相対パスは、カレントディレクトリを基準とした道筋を示すパスです（例: ../parent/file.txt）。絶対パスはどこから実行しても同じ場所を指しますが、相対パスは実行場所によって指す場所が変わります。

  4. Q4. 以下のコマンドの意味を説明してください: ls -la

    回答を表示

    lsは、現在のディレクトリ内のファイルとフォルダの一覧を表示するコマンドです。
    -lオプションは詳細情報を表示（ファイルサイズ、日付など）、
    -aオプションは隠しファイルも含めて表示します。
    したがって、ls -laは「隠しファイルも含めて詳細情報付きで一覧表示」という意味です。

  5. Q5. Tab補完を使うメリットを2つ挙げてください。

    回答を表示

    Tab補完のメリット：

    • 入力の効率化 - 長いファイル名やディレクトリ名を最後まで入力する必要がなくなる
    • 入力ミスの防止 - スペルミスを防ぎ、正確な入力ができる
    • 候補の確認 - 複数候補がある場合、どんなファイルがあるか確認できる

  • カレントディレクトリは現在の作業場所で、pwdで確認できる
  • 基本コマンド（pwd, ls, cd, mkdir, rm, cp, mv）の使い方を習得
  • オプションと引数でコマンドの動作をカスタマイズできる
  • Tab補完とコマンド履歴で効率的に操作できる
