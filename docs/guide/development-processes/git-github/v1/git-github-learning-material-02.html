環境構築 - GitとGitHub Desktopのインストール

GitとGitHub Desktopを自分のパソコンにインストールし、GitHubアカウントを作成して、開発環境を整えましょう。

この章で学ぶこと

  • Windows/Mac/LinuxそれぞれでGitをインストールする方法
  • GitHub Desktopのインストールと初期設定
  • GitHubアカウントの作成方法
  • Gitの初期設定（ユーザー名とメールアドレス）
  • SSH鍵の生成とGitHubへの登録
  • VS Codeのインストールとgit連携

2.1 環境構築の全体像

Git/GitHubを使い始めるには、いくつかのソフトウェアをインストールする必要があります。 まずは全体像を理解してから、順番にセットアップしていきましょう。

環境構築の流れ

flowchart LR A([開始]) --> B[Gitインストール] B --> C[GitHub Desktop
インストール] C --> D[GitHubアカウント
作成] D --> E[初期設定] E --> F[VS Code
インストール] F --> G([完了])

インストールするソフトウェア

ソフトウェア 役割 必須/推奨
Git バージョン管理の本体（コマンドライン） 必須
GitHub Desktop GUIでGitを操作するツール 推奨
VS Code コードエディタ（Git統合機能付き） 推奨

用語解説：GUI（グーイ）

GUIは「Graphical User Interface」の略で、マウスやタッチでボタンをクリックして操作する画面のことです。 一方、キーボードでコマンド（命令文）を入力して操作する方式をCUI（Character User Interface）またはCLI（Command Line Interface）と呼びます。

2.2 GitHubアカウントの作成

まずはGitHubのアカウントを作成しましょう。GitHubは無料で利用でき、個人プロジェクトには十分な機能が揃っています。

実習 2-1: GitHubアカウントを作成しよう

GitHubの公式サイトでアカウントを作成します。

手順

  1. 1 ブラウザで https://github.com にアクセス
  2. 2 「Sign up」ボタンをクリック
  3. 3 メールアドレス、パスワード、ユーザー名を入力
  4. 4 メールで届く認証コードを入力して完了

ユーザー名のコツ：ユーザー名は後から変更できますが、URLの一部になるので、覚えやすく短いものがおすすめです。本名やニックネーム、会社名などが一般的です。

パスワードについて

GitHubのパスワードは、8文字以上で数字と小文字を含む必要があります。 セキュリティのため、他のサービスとは異なる強力なパスワードを設定しましょう。

2.3 Gitのインストール（Windows）

Windowsでは「Git for Windows」をインストールします。これにはGit本体と、Git Bash（コマンドライン環境）が含まれています。

実習 2-2: WindowsにGitをインストール

Git for Windowsをダウンロードしてインストールします。

手順

  1. 1 https://git-scm.com/download/win にアクセス
  2. 2 「Click here to download」をクリックしてインストーラーをダウンロード
  3. 3 ダウンロードしたファイル（Git-X.XX.X-64-bit.exe）をダブルクリック
  4. 4 インストーラーの指示に従い、基本的には「Next」をクリックして進める
  5. 5 「Install」をクリックしてインストール開始、完了したら「Finish」

インストール時の重要な設定項目

基本的にはデフォルト設定のままで問題ありませんが、以下の設定は確認しておきましょう：

  • デフォルトエディタ：VS Codeを使う場合は「Use Visual Studio Code as Git's default editor」を選択
  • PATH環境変数：「Git from the command line and also from 3rd-party software」を選択（推奨）
  • 改行コード：「Checkout Windows-style, commit Unix-style line endings」を選択（推奨）

用語解説：Git Bash（ギットバッシュ）

Git for Windowsをインストールすると、Git Bashというアプリケーションも一緒にインストールされます。 これはLinuxやMacで使われる「Bash」というコマンドライン環境をWindows上で再現したもので、 Gitコマンドだけでなく、基本的なLinuxコマンドも使えます。

2.4 Gitのインストール（Mac）

MacではXcode Command Line Toolsをインストールする方法と、Homebrewを使う方法があります。 初心者には前者がおすすめです。

実習 2-3: MacにGitをインストール

ターミナルからXcode Command Line Toolsをインストールします。

手順

  1. 1 「アプリケーション」→「ユーティリティ」→「ターミナル」を開く
  2. 2 以下のコマンドを入力してEnter
Terminal 
# Xcode Command Line Toolsのインストール
xcode-select --install
  1. 3 ダイアログが表示されたら「インストール」をクリック
  2. 4 インストール完了まで数分〜十数分待つ

代替方法：Homebrewを使う場合

すでにHomebrewがインストールされている場合は、以下のコマンドでGitをインストールできます：

Terminal 
# Homebrewを使ったGitインストール
brew install git

2.5 Gitのインストール（Linux）

Linuxでは、ディストリビューション（種類）に応じたパッケージマネージャーを使ってインストールします。

Ubuntu / Debian系 
# パッケージリストを更新
sudo apt update

# Gitをインストール
sudo apt install git
Fedora / CentOS / RHEL系 
# Fedoraの場合
sudo dnf install git

# CentOS/RHEL 7以前の場合
sudo yum install git

用語解説：sudo（スードゥー）

sudoは「Super User DO」の略で、管理者（root）権限でコマンドを実行するときに使います。 ソフトウェアのインストールなど、システムに変更を加える操作には管理者権限が必要なため、sudoを付けて実行します。 実行時にパスワードを求められることがあります。

2.6 インストールの確認

Gitが正しくインストールされたかを確認しましょう。 どのOSでも同じコマンドで確認できます。

実習 2-4: Gitのバージョンを確認しよう

ターミナル（Windows: Git Bash、Mac/Linux: ターミナル）を開いて確認します。

実行コマンド

Terminal / Git Bash 
# Gitのバージョンを確認
git --version

期待される結果

git version 2.43.0

バージョン番号は異なっていても問題ありません。数字が表示されればインストール成功です。

トラブルシューティング

「git」は認識されていません / command not found

原因: GitがPATHに登録されていない、またはインストールが完了していません。

解決方法:

  • Windowsの場合：PCを再起動してみる
  • Git for Windowsを再インストールし、PATH設定を確認
  • Git Bashを使ってコマンドを実行する

2.7 GitHub Desktopのインストール

GitHub Desktopは、GitHubが公式に提供するGUIツールです。 コマンドを覚える前に、視覚的にGitの操作を学ぶことができます。

実習 2-5: GitHub Desktopをインストールしよう

Windows/Macに対応しています（Linux版は公式には提供されていません）。

手順

  1. 1 https://desktop.github.com にアクセス
  2. 2 「Download for Windows」または「Download for macOS」をクリック
  3. 3 ダウンロードしたファイルを実行してインストール
  4. 4 起動したら「Sign in to GitHub.com」をクリック
  5. 5 ブラウザが開くので、先ほど作成したGitHubアカウントでログイン
  6. 6 「Authorize desktop」をクリックして連携を許可

Linuxユーザーの方へ

GitHub Desktopの公式Linux版はありませんが、コミュニティが開発したフォーク版が存在します。 ただし、本ガイドではLinuxユーザーはコマンドライン（第5章〜）から学習を始めることをおすすめします。 VS CodeのGit統合機能もGUIとして活用できます。

2.8 Gitの初期設定

Gitを使い始める前に、ユーザー名とメールアドレスを設定する必要があります。 これは、「誰が変更を行ったか」を記録するための重要な設定です。

なぜ設定が必要？

Gitでは、変更を記録（コミット）するたびに「誰が」「いつ」変更したかが自動的に記録されます。 チーム開発では、誰がどの変更をしたかを追跡するために重要です。 個人開発でも、GitHubに履歴として残るため、正しく設定しておきましょう。

実習 2-6: ユーザー名とメールアドレスを設定しよう

ターミナル（Git Bash）で以下のコマンドを実行します。

Terminal / Git Bash 
# ユーザー名を設定（あなたの名前に変更してください）
git config --global user.name "Taro Yamada"

# メールアドレスを設定（GitHubのメールアドレスと同じにする）
git config --global user.email "taro@example.com"

設定の確認

Terminal / Git Bash 
# 設定内容を確認
git config --global --list

期待される結果

user.name=Taro Yamada
user.email=taro@example.com

--globalオプションとは？

--globalは「このパソコン全体で使う設定」という意味です。 Gitの設定には3つのレベルがあります：

  • --system システム全体（すべてのユーザー）に適用
  • --global 現在のユーザーのすべてのプロジェクトに適用（通常はこれを使用）
  • --local 現在のプロジェクト（リポジトリ）のみに適用

メールアドレスについて

GitHubで使用しているメールアドレスと同じものを設定してください。 異なるメールアドレスを使うと、GitHubの貢献グラフ（草）に反映されなかったり、 コミットがあなたのアカウントに紐づかない場合があります。

2.9 SSH鍵の生成と設定

GitHubへのアクセス方法にはHTTPSとSSHの2種類があります。 SSHを使うと、毎回パスワードを入力せずに安全にGitHubと通信できます。 ここでは、SSH鍵を生成してGitHubに登録する方法を説明します。

HTTPSとSSHの比較

項目 HTTPS SSH
認証方法 ユーザー名 + Personal Access Token SSH鍵（公開鍵/秘密鍵）
セットアップ 簡単（トークン生成のみ） やや複雑（鍵の生成・登録が必要）
日常の操作 認証情報の入力が必要な場合あり パスワード不要で便利
セキュリティ 高い 非常に高い
推奨度 初心者向け おすすめ

用語解説：SSH鍵（公開鍵暗号方式）

SSH鍵は「公開鍵」と「秘密鍵」のペアで構成されます。 秘密鍵は自分のPCに保存し、絶対に他人に見せてはいけません。 公開鍵はGitHubに登録します。 この仕組みにより、パスワードを使わずに安全に本人確認ができます。

実習 2-8: SSH鍵を生成しよう

ターミナル（Git Bash）でSSH鍵を生成します。

ステップ1: SSH鍵の生成

Terminal / Git Bash 
# SSH鍵を生成（メールアドレスはGitHubと同じものに変更）
ssh-keygen -t ed25519 -C "your_email@example.com"

コマンドを実行すると、いくつか質問されます：

「Enter file in which to save the key」

→ そのままEnterを押す（デフォルトの場所に保存）

「Enter passphrase」

→ パスフレーズを設定（任意。セキュリティ向上のため推奨）

「Enter same passphrase again」

→ 同じパスフレーズを再入力

ステップ2: SSH-Agentの起動と鍵の登録

Terminal / Git Bash 
# SSH-Agentを起動
eval "$(ssh-agent -s)"

# 秘密鍵をSSH-Agentに登録
ssh-add ~/.ssh/id_ed25519

ステップ3: 公開鍵をコピー

Terminal / Git Bash（Windows） 
# 公開鍵をクリップボードにコピー（Windows Git Bash）
cat ~/.ssh/id_ed25519.pub | clip
Terminal（Mac） 
# 公開鍵をクリップボードにコピー（Mac）
pbcopy < ~/.ssh/id_ed25519.pub
Terminal（Linux） 
# 公開鍵を表示してコピー（Linux）
cat ~/.ssh/id_ed25519.pub
# 表示された内容を手動でコピー

実習 2-9: GitHubに公開鍵を登録しよう

コピーした公開鍵をGitHubに登録します。

手順

  1. 1 GitHubにログインし、右上のプロフィールアイコンをクリック
  2. 2 「Settings」を選択
  3. 3 左側メニューの「SSH and GPG keys」をクリック
  4. 4 「New SSH key」ボタンをクリック
  5. 5 「Title」に識別しやすい名前を入力（例：My Laptop、Work PC など）
  6. 6 「Key」欄にコピーした公開鍵を貼り付け
  7. 7 「Add SSH key」をクリック

接続確認

Terminal / Git Bash 
# GitHubへのSSH接続をテスト
ssh -T git@github.com

期待される結果

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

このメッセージが表示されれば、SSH接続の設定は完了です！

秘密鍵の取り扱いについて

~/.ssh/id_ed25519（拡張子なし）は秘密鍵です。 これは絶対に他人に見せたり、メールで送ったり、GitHubにアップロードしたりしないでください。 GitHubに登録するのはid_ed25519.pub（.pubがついた公開鍵）です。

SSHのトラブルシューティング

「Permission denied (publickey)」エラー

原因: SSH鍵が正しく設定されていないか、SSH-Agentに登録されていません。

解決方法:

  • ssh-add -lで鍵が登録されているか確認
  • 登録されていなければssh-add ~/.ssh/id_ed25519を実行
  • GitHubに正しい公開鍵が登録されているか確認
「Host key verification failed」エラー

原因: 初回接続時にGitHubのホスト鍵を承認していません。

解決方法: 接続時に「Are you sure you want to continue connecting?」と聞かれたら「yes」と入力してEnterを押してください。

古いシステムで「ed25519」が使えない場合

原因: 古いバージョンのOpenSSHではed25519がサポートされていません。

解決方法: RSA鍵を使用してください：

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

Windows、Mac、Linuxで同じように使える

2.11 環境構築の確認

すべてのインストールと設定が完了したか、最終確認をしましょう。

環境構築チェックリスト

理解度確認クイズ

  1. Q1. Git for Windowsをインストールすると一緒にインストールされる、コマンドライン環境の名前は何ですか？

    回答を表示

    Git Bashです。LinuxやMacのBashシェル環境をWindows上で再現したもので、Gitコマンドや基本的なLinuxコマンドを使用できます。

  2. Q2. Gitの初期設定で、ユーザー名とメールアドレスを設定する理由は何ですか？

    回答を表示

    Gitでは、変更を記録（コミット）するたびに「誰が」変更したかが記録されます。チーム開発で変更の追跡をするため、また個人の貢献をGitHubに正しく反映させるために、ユーザー名とメールアドレスの設定が必要です。

  3. Q3. git config --globalの「--global」オプションは何を意味しますか？

    回答を表示

    「現在のユーザーのすべてのプロジェクトに適用される設定」を意味します。--systemはシステム全体、--localは現在のプロジェクトのみに適用されます。通常は--globalを使用します。
