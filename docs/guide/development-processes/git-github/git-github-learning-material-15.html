第15章

VS CodeでのGitHub連携 - リモート操作とプルリクエスト

VS CodeからGitHubとシームレスに連携する方法を学びます。GitHub認証の設定からプルリクエストの作成・レビューまで、日常的な開発ワークフローを効率化しましょう。

この章で学ぶこと

  • VS Code内でGitHub認証を設定する方法
  • VS CodeからリモートリポジトリへのPush/Pull操作
  • GitHub Pull Requests拡張機能の導入と設定
  • VS Code内でプルリクエストを作成・レビューする
  • 効率的な開発ワークフローのベストプラクティス

15.1 GitHub認証の設定

VS CodeからGitHubにアクセスするには、最初にGitHubアカウントとの連携が必要です。VS Codeは直接GitHubにサインインできるため、Personal Access Token（PAT）を手動で設定する必要はありません。

GitHub認証の手順

1

VS Codeでアカウント管理を開く

左下の「アカウント」アイコン（人のマーク）をクリックします。

または、コマンドパレット（Cmd/Ctrl+Shift+P）で「GitHub: Sign In」と入力します。

2

GitHubでサインインを選択

「GitHub でサインイン」または「GitHub with GitHub」を選択します。

ブラウザが開き、GitHubの認証ページが表示されます。

3

GitHubで認証を許可

GitHubアカウントでログインし、VS Codeへのアクセスを許可します。

  • リポジトリへのアクセス権限
  • プルリクエストの作成・編集権限
  • Issuesへのアクセス権限
4

認証完了の確認

VS Codeに戻ると、左下のアカウントアイコンにGitHubアカウント名が表示されます。

これで、VS CodeからGitHubへのアクセスが可能になりました。

ポイント

一度認証すれば、VS Codeを再起動しても認証状態は保持されます。複数のGitHubアカウントを切り替えることも可能です。

15.2 リモートリポジトリへのPush/Pull操作

GitHub認証が完了したら、VS CodeのSource Controlパネルからリモートリポジトリへの操作ができるようになります。

Push操作 - ローカルの変更をGitHubへアップロード

方法1: Source Controlパネルから

  1. Source Controlパネルを開く（アイコン：分岐マーク）
  2. コミット済みの変更がある場合、上部に「同期の変更」ボタンが表示される
  3. 「同期の変更」または「↑（Push）」アイコンをクリック
  4. 確認ダイアログが表示されたら「はい」をクリック

方法2: コマンドパレットから

  1. Cmd/Ctrl+Shift+P でコマンドパレットを開く
  2. 「Git: Push」と入力して選択
  3. Pushが実行される

方法3: ステータスバーから

左下のステータスバーに「↑1」のように表示されている場合、これをクリックすると即座にPushされます。

Pull操作 - GitHubから最新の変更を取得

方法1: Source Controlパネルから

  1. Source Controlパネルの右上メニュー（…）をクリック
  2. 「プル」を選択
  3. リモートの最新変更が取り込まれる

方法2: コマンドパレットから

  1. Cmd/Ctrl+Shift+P でコマンドパレットを開く
  2. 「Git: Pull」と入力して選択
  3. Pullが実行される

注意

Pushする前に必ずPullしましょう。他のメンバーが変更をプッシュしている可能性があります。Pullせずにプッシュするとコンフリクトが発生する場合があります。

15.3 GitHub Pull Requests拡張機能のインストール

VS Codeには、プルリクエストを作成・レビューするための公式拡張機能があります。これをインストールすると、エディタから離れずにプルリクエストの全操作が可能になります。

実習: GitHub Pull Requests拡張機能のインストール

手順1: 拡張機能パネルを開く

左サイドバーの「拡張機能」アイコン（四角が4つのマーク）をクリック、または Cmd/Ctrl+Shift+X を押します。

手順2: 検索して拡張機能を見つける

検索ボックスに「GitHub Pull Requests」と入力します。

公式拡張機能: GitHub Pull Requests and Issues（GitHub社提供）

手順3: インストール

「インストール」ボタンをクリックします。インストール完了後、自動的にGitHub認証が求められる場合があります。

手順4: 認証とアクティベート

拡張機能がアクティベートされると、左サイドバーに「GitHub」アイコンが追加されます。ここからプルリクエストやIssuesにアクセスできます。

拡張機能の主な機能

  • VS Code内でプルリクエストを作成・編集
  • プルリクエストの一覧表示とフィルタリング
  • コードレビュー（コメント追加、変更要求）
  • プルリクエストのマージ操作
  • GitHubのIssues管理

15.4 VS Code内でプルリクエストを作成する

GitHub Pull Requests拡張機能を使うと、ブラウザを開かずにVS Code内で直接プルリクエストを作成できます。

プルリクエスト作成の前提条件

  • 機能ブランチ（feature/new-featureなど）で作業済み
  • 変更をコミット済み
  • ブランチをリモートにPush済み（origin/feature/new-feature）

プルリクエスト作成手順

1

GitHubビューを開く

左サイドバーの「GitHub」アイコン（拡張機能インストール後に追加）をクリックします。

2

「Create Pull Request」を選択

GitHubビューの上部に「Create Pull Request」ボタンが表示される場合があります。

または、コマンドパレットから「GitHub Pull Requests: Create Pull Request」を実行します。

3

マージ先ブランチとマージ元ブランチを選択

通常は、以下の設定になります：

  • ベースブランチ（マージ先）: main または master
  • 比較ブランチ（マージ元）: 現在の機能ブランチ
4

タイトルと説明を入力

エディタが開き、プルリクエストのタイトルと本文を入力できます。

プルリクエストの例 
# タイトル
新しいログイン機能を追加

# 説明
## 変更内容
- ユーザー認証フォームを追加
- JWT トークンによる認証を実装
- ログイン後のリダイレクト処理を追加

## テスト方法
1. ログインページにアクセス
2. 有効な認証情報を入力
3. ダッシュボードにリダイレクトされることを確認
5

レビュアーとラベルを設定（オプション）

必要に応じて、レビュアー（チームメンバー）やラベル（bug、featureなど）を設定できます。

6

プルリクエストを作成

「Create」ボタンをクリックすると、GitHubにプルリクエストが作成されます。作成後、VS Code内でプルリクエストの詳細を確認できます。

15.5 VS Codeでプルリクエストをレビューする

GitHub Pull Requests拡張機能を使うと、チームメンバーが作成したプルリクエストをVS Code内でレビューできます。

レビュー対象のプルリクエストを開く

  1. 左サイドバーの「GitHub」アイコンをクリック
  2. 「Pull Requests」セクションに、リポジトリのプルリクエスト一覧が表示される
  3. レビューしたいプルリクエストをクリックして展開
  4. 「Checkout」をクリックすると、そのプルリクエストのブランチに切り替わる

変更内容の確認

ファイルごとの差分表示

プルリクエストを展開すると、変更されたファイルの一覧が表示されます。

  • ファイルをクリックすると、差分ビューが開く
  • 左側に元のコード、右側に変更後のコードが表示される
  • 追加行は緑、削除行は赤でハイライトされる

コメントを追加する

1

行にコメントを追加

差分ビューで、コメントしたい行にマウスを合わせると、「+」アイコンが表示されます。

クリックすると、コメント入力欄が開きます。

2

コメント内容を入力

具体的なフィードバックを記述します：

  • 質問: 「この処理の意図を教えてください」
  • 提案: 「このロジックは関数に切り出した方が良いかもしれません」
  • 承認: 「良い実装ですね！」
3

コメントを投稿

「Add Comment」をクリックすると、コメントがGitHubに投稿されます。すぐに表示され、作成者に通知が送られます。

レビューステータスの設定

レビュー完了時の選択肢

  • Comment - 一般的なコメントのみ（承認なし）
  • Approve - 変更を承認（マージ可能）
  • Request Changes - 変更を要求（修正が必要）

15.6 効率的な開発ワークフローのベストプラクティス

VS CodeとGitHubを連携させた効率的な開発ワークフローを構築するためのベストプラクティスを紹介します。

1. 小さく頻繁にコミットする

大きな変更を一度にコミットするのではなく、意味のある単位で小さくコミットします。

  • 1つのコミットには1つの論理的な変更だけを含める
  • 後から変更履歴を追いやすくなる
  • 問題が発生した時に切り戻しやすい

2. 定期的にPull（同期）する

作業開始時と作業中は定期的にリモートから最新の変更をPullします。

  • 朝の作業開始時に必ずPull
  • Pushする前にも必ずPull
  • コンフリクトの発生を最小限に抑える

3. 機能ごとにブランチを作成する

mainブランチで直接作業せず、必ず機能ブランチを作成します。 

# 良い例
git checkout -b feature/add-user-profile
git checkout -b fix/login-bug

# 避けるべき
git checkout main  # mainで直接作業

4. プルリクエストで必ずレビューを受ける

個人開発であっても、プルリクエストを経由してマージする習慣をつけます。

  • 変更内容を整理して説明する習慣が身につく
  • 後から変更理由を振り返りやすい
  • チーム開発に移行しやすい

5. GitHub Copilotとの連携（紹介）

GitHub CopilotはAIがコードを補完してくれる有料サービスです。VS Codeと統合されており、効率的な開発をサポートします。

  • コード補完の提案
  • コメントからコードを自動生成
  • テストコードの生成

学生や一部のオープンソースプロジェクトでは無料で利用できます。

トラブルシューティング

Q: GitHub認証がうまくいかない

A: 左下のアカウントアイコンから「Sign out」し、再度「Sign in with GitHub」を実行してください。ブラウザで認証が完了しても、VS Codeに戻らない場合は、VS Codeを再起動してください。

Q: プルリクエストが作成できない

A: ブランチがリモートにPushされているか確認してください。また、GitHub Pull Requests拡張機能が正しくインストールされているか確認してください。

Q: プルリクエストの一覧が表示されない

A: GitHubビューで右上の更新ボタン（↻）をクリックしてください。また、GitHub認証が有効か確認してください。

Q: Pushしようとすると「認証に失敗しました」と表示される

A: GitHub認証の有効期限が切れている可能性があります。再度GitHub認証を行ってください。または、SSHキーを設定してSSH接続に切り替えることも検討してください。

  • Source ControlパネルやコマンドパレットからPush/Pull操作が簡単にできる
