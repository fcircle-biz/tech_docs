GitHubの基本 - アカウント作成とWebインターフェース

GitHubのWebサイトの使い方と基本機能を学びます。アカウント作成からリポジトリの作成、他の人のプロジェクトの閲覧まで、GitHubの基本を習得しましょう。

この章で学ぶこと

  • GitHubアカウントの作成とプロフィール設定
  • GitHubのダッシュボードと画面構成の理解
  • 新しいリポジトリの作成（Webから）
  • README.mdファイルの作成と編集
  • パブリック/プライベートリポジトリの違い
  • 他の人のリポジトリを見る・フォークする方法
  • Starとウォッチ機能の活用

3.1 GitHubアカウントの作成

GitHubを使い始めるには、まずアカウントを作成する必要があります。個人利用であれば無料で使用できます。

実習: GitHubアカウントを作成しよう

手順1: GitHubのWebサイトにアクセス

ブラウザで https://github.com にアクセスします。

手順2: サインアップをクリック

右上の「Sign up」ボタンをクリックします。

手順3: 必要情報を入力

  • メールアドレス: 受信可能なメールアドレス
  • パスワード: 15文字以上、または8文字以上で数字と小文字を含む
  • ユーザー名: GitHubでの表示名（後から変更可能）

手順4: メール認証

登録したメールアドレスに認証コードが送られてきます。そのコードを入力して認証を完了します。

手順5: プランの選択

個人利用なら「Free」プランを選択します（無料でも十分な機能が使えます）。

ユーザー名の選び方のヒント

  • ・本名でも、ニックネームでもOK
  • ・将来、仕事でも使うことを考えて、プロフェッショナルな名前が推奨
  • ・ハイフン（-）は使えますが、アンダースコア（_）は使えません
  • ・後から変更可能ですが、URLも変わるので注意

プロフィールの設定

アカウント作成後、プロフィールを設定しましょう。プロフィールは他の開発者に見られるため、丁寧に設定しておくと良いでしょう。

プロフィール設定手順 
1. 右上のプロフィールアイコンをクリック
2. 「Settings」を選択
3. 「Public profile」で以下を設定:
   - Name: 表示名（本名またはニックネーム）
   - Bio: 自己紹介文（簡潔に）
   - Location: 所在地（任意）
   - Website: Webサイト（任意）
   - Profile picture: プロフィール画像（任意）

3.3 新しいリポジトリの作成（Webから）

GitHubのWeb画面から、新しいリポジトリを作成してみましょう。

実習: 最初のリポジトリを作成しよう

手順1: 新規リポジトリ作成画面へ

右上の「+」アイコンをクリックし、「New repository」を選択します。

手順2: リポジトリ情報を入力

  • Owner: 自分のアカウント（自動選択）
  • Repository name: 例えば「my-first-repo」
  • Description: 「初めてのGitHubリポジトリ」（任意だが推奨）
  • Public/Private: 「Public」を選択（誰でも閲覧可能）
  • Initialize this repository with: 「Add a README file」にチェック

手順3: リポジトリを作成

下部の「Create repository」ボタンをクリックします。

作成完了！

リポジトリが作成され、リポジトリのトップページが表示されます。URLは「https://github.com/ユーザー名/リポジトリ名」の形式になります。

リポジトリ名の命名規則

リポジトリ名のベストプラクティス

  • 小文字を使用（my-project、not My-Project）
  • 単語の区切りはハイフン「-」を使う（my-awesome-project）
  • 短く、わかりやすい名前にする
  • プロジェクトの内容を表す名前にする
  • 特殊文字やスペースは使えない

3.4 README.mdファイルの作成と編集

README.mdは、リポジトリのトップページに表示される説明ファイルです。Markdown記法で書かれています。

Markdown記法とは

Markdownは、シンプルな記号を使って文書を装飾する軽量マークアップ言語です。「#」で見出し、「-」でリスト、「**」で太字など、簡単な記号で読みやすい文書が作成できます。

実習: README.mdを編集しよう

手順1: 編集ボタンをクリック

リポジトリのトップページで、README.mdの右上にある鉛筆アイコン（編集ボタン）をクリックします。

手順2: 内容を編集

以下のような内容に書き換えてみましょう：

README.md 
# 私の最初のリポジトリ

これはGitHubの学習用リポジトリです。

## 学習内容

- GitHubアカウントの作成
- リポジトリの作成
- README.mdの編集

## 参考資料

- [GitHub公式ドキュメント](https://docs.github.com/ja)

手順3: プレビューで確認

「Preview」タブをクリックして、表示を確認します。

手順4: コミットで保存

下部の「Commit changes」をクリックし、コミットメッセージを入力します：

  • 短いメッセージ: 「Update README.md」
  • 詳細（任意）: 「学習内容と参考資料を追加」

「Commit changes」ボタンをクリックして保存します。

3.5 パブリックリポジトリとプライベートリポジトリの違い

GitHubのリポジトリには、パブリック（公開）とプライベート（非公開）の2種類があります。

項目 パブリック プライベート
閲覧権限 誰でも閲覧可能 招待した人のみ
検索 GitHub検索に表示される 検索には表示されない
費用 無料 無料（個人アカウント）
用途 オープンソースプロジェクト
ポートフォリオ		 個人プロジェクト
会社の機密コード
フォーク 誰でもフォーク可能 権限のある人のみ

パブリックがおすすめの場合

  • オープンソースプロジェクト
  • 学習用のサンプルコード
  • 技術ブログの記事データ
  • 就職活動でのポートフォリオ

プライベートがおすすめの場合

  • 会社の業務コード
  • 公開前のプロジェクト
  • 個人的なメモやドキュメント
  • APIキーなど機密情報を含むコード

後から変更できます

パブリック↔プライベートの切り替えは、リポジトリの設定からいつでも変更できます。ただし、パブリックにしていた期間にフォークされたリポジトリは、プライベートに変更しても削除されません。

3.6 他の人のリポジトリを見る・フォークする

GitHubには世界中の開発者が公開している無数のリポジトリがあります。他の人のコードを見て学ぶことは、スキル向上に非常に有効です。

リポジトリの検索

実習: 人気のリポジトリを探してみよう

手順1: 検索バーで検索

GitHubのトップページの検索バーに「python」や「javascript」など、興味のあるキーワードを入力します。

手順2: フィルターで絞り込み

検索結果で以下のフィルターが使えます：

  • Most stars: Star数が多い順
  • Most forks: フォーク数が多い順
  • Recently updated: 最近更新されたもの

手順3: リポジトリを閲覧

興味のあるリポジトリをクリックして、README、コード、コミット履歴などを見てみましょう。

フォーク（Fork）とは

フォークは、他の人のリポジトリを自分のアカウントにコピーすることです。コピー後は自由に変更でき、元のプロジェクトに影響を与えません。

フォークした後、変更をプルリクエストで元のプロジェクトに提案することもできます（後の章で学習）。

フォークの手順 
1. フォークしたいリポジトリのページを開く
2. 右上の「Fork」ボタンをクリック
3. 自分のアカウントを選択
4. 数秒で自分のアカウントにコピーされる

3.7 Starとウォッチ機能

GitHubには、気に入ったリポジトリを管理するための機能があります。

Star機能

リポジトリに「いいね」を付ける機能です。

  • 気に入ったリポジトリをブックマーク
  • Star一覧からいつでもアクセス
  • プロジェクトへの評価・応援の意味
  • Star数は人気の指標になる

Watch機能

リポジトリの更新を通知してもらう機能です。

  • 更新情報をメールで受け取る
  • Issue、プルリクエストの通知
  • アクティブに追いかけたい時に使う
  • 通知頻度をカスタマイズ可能

使い分けのポイント

状況 Star Watch
後で参考にしたいリポジトリ ✓ -
プロジェクトの応援 ✓ -
開発に参加している ✓ ✓
最新情報を追いたい - ✓

初心者へのアドバイス

最初は気軽にStarを付けてみましょう。Starしたリポジトリは「Your stars」から一覧で見られるので、自分だけの技術参考書棚を作るイメージです。Watchは通知が多くなるので、本当に追いかけたいプロジェクトだけに使うのがおすすめです。

理解度確認クイズ

  1. Q1. README.mdファイルの役割は何ですか？

    回答を表示

    README.mdは、リポジトリのトップページに表示される説明ファイルです。プロジェクトの概要、使い方、インストール方法などを記述します。Markdown記法で書かれており、訪問者が最初に読むドキュメントとして重要です。

  2. Q2. パブリックリポジトリとプライベートリポジトリの主な違いを説明してください。

    回答を表示

    パブリックリポジトリは誰でも閲覧でき、GitHub検索にも表示されます。オープンソースプロジェクトやポートフォリオに適しています。

    プライベートリポジトリは招待した人のみが閲覧できます。会社の業務コードや公開前のプロジェクトに適しています。

    どちらも個人アカウントでは無料で使用できます。

  3. Q3. 「フォーク（Fork）」とは何ですか？どのような場合に使いますか？

    回答を表示

    フォークは、他の人のリポジトリを自分のアカウントにコピーすることです。コピー後は元のプロジェクトに影響を与えずに、自由に変更できます。

    使用例：
    ・オープンソースプロジェクトに貢献したい時
    ・他の人のコードをベースに自分のプロジェクトを作りたい時
    ・学習のために他の人のコードを試してみたい時

  4. Q4. StarとWatchの違いを説明してください。

    回答を表示

    Starは、気に入ったリポジトリに「いいね」を付ける機能です。ブックマーク的な使い方ができ、プロジェクトへの評価・応援の意味もあります。

    Watchは、リポジトリの更新を通知してもらう機能です。Issue、プルリクエストなどの活動をメールで受け取れます。

    参考にしたいリポジトリにはStar、アクティブに追いかけたいプロジェクトにはWatchを使います。

  5. Q5. Markdown記法で太字を表示する方法を書いてください。

    回答を表示

    **太字にしたいテキスト** と書くと、太字にしたいテキスト と表示されます。

    または __太字にしたいテキスト__ （アンダースコア2つ）でも同じ結果になります。

第3章のまとめ

  • GitHubアカウントは無料で作成でき、メール認証で簡単に始められる
  • ダッシュボードから、自分のリポジトリやフォロー中の活動を確認できる
  • Webインターフェースから直接リポジトリを作成し、README.mdを編集できる
  • パブリックは公開、プライベートは非公開で、用途に応じて使い分ける
  • フォークで他の人のリポジトリをコピーし、自由に改変できる
  • StarとWatchで、気になるプロジェクトを管理・追跡できる
