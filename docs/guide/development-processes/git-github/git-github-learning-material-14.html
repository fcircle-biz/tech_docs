VS Codeでのブランチとマージ - GUIでの分岐作業

VS Codeでブランチ操作とマージを行う方法を学びます。ステータスバーでのブランチ確認と切り替え、コマンドパレットでの操作、そしてコンフリクトの視覚的な解決方法を習得しましょう。

この章で学ぶこと

  • VS Codeのステータスバーでブランチを確認・切り替える方法
  • コマンドパレットを使ったブランチ作成と管理
  • VS Codeでのブランチマージ操作
  • 3-wayマージエディタを使ったコンフリクト解決
  • GitLens拡張機能の導入と基本的な活用方法

14.1 VS Codeのブランチ機能概要

VS Codeは、ブランチの作成・切り替え・マージといった操作をGUIで直感的に行える機能を提供しています。コマンドラインで学んだ操作を、視覚的に確認しながら実行できます。

VS Codeのブランチ操作の特徴

  • ステータスバーで現在のブランチを常に確認できる
  • ブランチの切り替えがワンクリックで可能
  • コンフリクトが視覚的に表示され、解決しやすい
  • コマンドラインとGUIを併用できる

VS Codeでのブランチ操作の主な機能

機能 操作方法 対応コマンド
ブランチ確認 ステータスバー表示 git branch
ブランチ切り替え ステータスバーをクリック git switch
ブランチ作成 コマンドパレット git switch -c
ブランチマージ コマンドパレット git merge
コンフリクト解決 3-wayマージエディタ 手動編集

14.2 ステータスバーでのブランチ確認と切り替え

VS Codeの画面下部にあるステータスバーには、現在のブランチ名が常に表示されています。このブランチ名をクリックすることで、簡単にブランチを切り替えることができます。

ステータスバーの見方

main ← 現在のブランチ名

ステータスバーの左側に、ブランチアイコン（）とともに現在のブランチ名が表示されます。この部分をクリックすると、ブランチ選択メニューが開きます。

実習: ステータスバーでブランチを切り替える

  1. ステータスバーのブランチ名をクリック

    画面下部のステータスバーにある現在のブランチ名（例: main）をクリックします。

  2. ブランチリストを確認

    画面上部にブランチの一覧が表示されます。リモートブランチも含めて全て表示されます。

  3. 切り替え先のブランチを選択

    移動したいブランチ名をクリックすると、即座にブランチが切り替わります。

  4. ステータスバーで確認

    ステータスバーのブランチ名が変わったことを確認します。

ポイント

ステータスバーからのブランチ切り替えは、git switch ブランチ名コマンドと同じ動作です。未コミットの変更がある場合は、先にコミットまたはstashしておく必要があります。

14.3 コマンドパレットでのブランチ作成と管理

VS Codeのコマンドパレット（Ctrl + Shift + P / Cmd + Shift + P）を使うと、ブランチの作成や削除など、より詳細なブランチ操作ができます。

実習: 新しいブランチを作成する

  1. コマンドパレットを開く

    Ctrl + Shift + P（Mac: Cmd + Shift + P）を押します。

  2. ブランチ作成コマンドを検索

    「Git: Create Branch」または「Git: ブランチの作成」と入力して選択します。

  3. ブランチ名を入力

    例えば「feature/new-feature」のように、わかりやすい名前を入力します。

  4. 自動的に切り替わる

    新しいブランチが作成され、自動的にそのブランチに切り替わります。

よく使うブランチ関連のコマンド

コマンド名 説明
Git: Create Branch 新しいブランチを作成
Git: Checkout to... ブランチを切り替え
Git: Merge Branch... ブランチをマージ
Git: Delete Branch... ブランチを削除
Git: Rename Branch... ブランチ名を変更

Tips

コマンドパレットは「Git」と入力するだけで、Git関連のコマンドがすべて表示されます。コマンド名を覚えていなくても、リストから選択できるので便利です。

14.4 VS Codeでのブランチマージ操作

VS Codeでブランチをマージする際も、コマンドパレットを使います。コマンドラインでのgit mergeと同じ動作ですが、視覚的に確認しながら操作できます。

実習: ブランチをマージする

  1. マージ先のブランチに切り替え

    例えば、feature/new-featureをmainにマージする場合、まずmainブランチに切り替えます。

  2. コマンドパレットを開く

    Ctrl + Shift + P（Mac: Cmd + Shift + P）を押します。

  3. マージコマンドを選択

    「Git: Merge Branch」と入力して選択します。

  4. マージ元のブランチを選択

    リストから「feature/new-feature」を選択します。

  5. マージ完了を確認

    コンフリクトがなければ、自動的にマージが完了します。

マージ方式の選択

VS Codeでのマージは、コマンドラインと同様に、状況に応じてFast-forwardまたは3-wayマージが自動的に選択されます。

flowchart LR A[main] --> B[Commit 1] B --> C[Commit 2] C --> D[feature作成] D --> E[Commit 3
feature] E --> F[Commit 4
feature] F --> G[mainにマージ] C -.-> G style A fill:#fbbf24,stroke:#f59e0b style D fill:#10b981,stroke:#059669 style G fill:#8b5cf6,stroke:#7c3aed

ブランチの分岐後、mainに変更がなければFast-forwardマージが使われます。

14.5 3-wayマージエディタでのコンフリクト解決

マージ時にコンフリクト（競合）が発生した場合、VS Codeは3-wayマージエディタという視覚的なツールを提供します。これにより、コンフリクトの内容を簡単に確認し、解決できます。

コンフリクト発生時のVS Codeの表示

コンフリクトが発生すると、以下のような表示になります：

<<<<<<< HEAD（現在のブランチ）
const message = "Hello from main";
=======
const message = "Hello from feature";
>>>>>>> feature（マージ元のブランチ）

VS Codeでは、コンフリクトしている箇所の上に、以下のようなボタンが表示されます：

Accept Current Change

現在のブランチ（HEAD）の変更を採用

Accept Incoming Change

マージ元ブランチの変更を採用

Accept Both Changes

両方の変更を残す

Compare Changes

3-wayマージエディタで詳細に確認

実習: コンフリクトを解決する

  1. コンフリクトが発生したファイルを開く

    Source Controlパネルに「Merge Changes」として表示されたファイルをクリックします。

  2. 3-wayマージエディタを開く

    コンフリクト箇所の上にある「Compare Changes」をクリックします。

  3. 変更内容を確認

    左側に現在のブランチ、右側にマージ元ブランチ、下側にマージ結果が表示されます。

  4. 採用する変更を選択

    「Accept Current」「Accept Incoming」「Accept Both」のいずれかをクリックするか、手動で編集します。

  5. コンフリクトマーカーを削除

    <<<<<<<、=======、>>>>>>>のマーカーが全て削除されたことを確認します。

  6. ファイルを保存

    Ctrl + S（Mac: Cmd + S）で保存します。

  7. ステージングしてコミット

    解決したファイルをステージングし、「Merge branch 'feature'」のようなメッセージでコミットします。

ポイント

3-wayマージエディタは、コマンドラインでの手動編集よりも視覚的でわかりやすいため、特に初心者におすすめです。左右の変更を見比べながら、適切な解決策を選択できます。

14.6 GitLens拡張機能の導入と活用

GitLensは、VS CodeでGitをより強力に使うための拡張機能です。ブランチの視覚化、コミット履歴の詳細表示、行ごとの変更履歴（Blame）など、多くの便利な機能を提供します。

GitLensのインストール

実習: GitLensをインストールする

  1. 拡張機能ビューを開く

    Ctrl + Shift + X（Mac: Cmd + Shift + X）を押します。

  2. GitLensを検索

    検索ボックスに「GitLens」と入力します。

  3. インストール

    「GitLens — Git supercharged」を見つけて「Install」ボタンをクリックします。

  4. インストール完了

    インストールが完了すると、サイドバーにGitLensのアイコンが追加されます。

GitLensの主な機能

Git Blame（行ごとの履歴）

各行の最終変更者とコミット情報が表示される

ブランチの可視化

ブランチのツリー構造をグラフィカルに表示

コミット検索

過去のコミットを素早く検索できる

ファイル履歴

ファイル単位で変更履歴を確認できる

GitLensの使い方の例

行末のBlame情報

const message = "Hello"; // 山田太郎, 2日前 · feat: メッセージ機能を追加

GitLensを有効にすると、エディタの各行の末尾に、その行を最後に変更した人とコミット情報が表示されます。これをクリックすると、そのコミットの詳細が確認できます。

Tips

GitLensは機能が豊富なため、最初は少し圧倒されるかもしれません。まずは「Git Blame」機能だけを使ってみて、徐々に他の機能も試していくのがおすすめです。

14.7 その他のおすすめGit関連拡張機能

GitLens以外にも、VS CodeでGitをより便利に使える拡張機能がいくつかあります。

拡張機能名 説明
Git Graph ブランチのグラフを視覚的に表示する
Git History ファイルやブランチの履歴を詳細に確認できる
GitHub Pull Requests VS Code内でプルリクエストを作成・レビューできる
gitignore .gitignoreファイルのテンプレートを自動生成

ポイント

拡張機能は一度にたくさん入れすぎると、VS Codeが重くなる可能性があります。まずは必要なものから試して、自分のワークフローに合ったものを選びましょう。

理解度確認クイズ

  1. Q1. VS Codeでブランチを切り替える最も簡単な方法は何ですか？

    回答を表示

    画面下部のステータスバーにある現在のブランチ名をクリックし、表示されたリストから切り替え先のブランチを選択する方法です。

  2. Q2. VS Codeで新しいブランチを作成するには、コマンドパレットで何と検索すればよいですか？

    回答を表示

    「Git: Create Branch」または「Git: ブランチの作成」と検索します。

  3. Q3. コンフリクトが発生した時、VS Codeが提供する選択肢を3つ挙げてください。

    回答を表示

    以下の選択肢があります：

    • Accept Current Change（現在のブランチの変更を採用）
    • Accept Incoming Change（マージ元ブランチの変更を採用）
    • Accept Both Changes（両方の変更を残す）
    • Compare Changes（3-wayマージエディタで詳細に確認）

  4. Q4. 3-wayマージエディタで、どのように画面が分割されていますか？

    回答を表示

    左側に現在のブランチ（HEAD）の内容、右側にマージ元ブランチの内容、下側にマージ後の結果が表示されます。

  5. Q5. GitLens拡張機能の主な機能を2つ挙げてください。

    回答を表示

    以下のような機能があります（2つ挙げる）：

    • Git Blame（各行の最終変更者とコミット情報の表示）
    • ブランチの視覚化（ツリー構造のグラフィカル表示）
    • コミット検索（過去のコミットの素早い検索）
    • ファイル履歴（ファイル単位での変更履歴確認）

  • VS Codeでのマージは「Git: Merge Branch」コマンドで実行できる
  • 3-wayマージエディタを使えば、コンフリクトを視覚的に解決できる
