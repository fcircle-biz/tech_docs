環境構築 - GitとVS Codeのインストール

Gitと開発環境を自分のパソコンにインストールします。Windows、Mac、Linuxそれぞれの環境でのインストール手順を詳しく解説します。

この章で学ぶこと

  • Windows、Mac、Linuxそれぞれの環境でGitをインストールできる
  • GitHubアカウントを作成し、基本設定を行える
  • Visual Studio Code（VS Code）をインストールし、基本設定ができる
  • Git BashやターミナルでGitコマンドを実行できる環境を整える
  • Gitの初期設定（ユーザー名とメールアドレス）を完了できる

2.1 環境構築の全体像

Git/GitHubを使い始めるには、いくつかのツールをインストールし、設定する必要があります。この章では、以下の3つのステップで環境を整えます。

環境構築の所要時間

環境によって異なりますが、一般的に30分〜1時間程度で完了します。インターネット接続が安定している環境で作業してください。

2.2 Gitのインストール

お使いのオペレーティングシステムに応じて、Gitをインストールします。以下、Windows、Mac、Linuxそれぞれの手順を説明します。

Windows環境でのインストール

Windows用Gitのインストール手順

  1. Git公式サイト（https://git-scm.com/）にアクセス
  2. 「Download for Windows」ボタンをクリックして、インストーラーをダウンロード
  3. ダウンロードした「Git-x.xx.x-64-bit.exe」をダブルクリックして起動
  4. ライセンス条項を確認し、「Next」をクリック
  5. インストール先フォルダを確認（デフォルトのままで問題なし）して「Next」
  6. コンポーネント選択画面で、デフォルト設定のまま「Next」
  7. スタートメニューフォルダ名を確認して「Next」
  8. デフォルトエディタの選択（「Use Visual Studio Code as Git's default editor」を推奨）
  9. 以降の設定画面は、基本的にデフォルトのまま「Next」を選択
  10. 「Install」をクリックしてインストール開始
  11. インストール完了後、「Finish」をクリック

macOS環境でのインストール

macOS用Gitのインストール手順

macOSには複数のインストール方法がありますが、Homebrewを使う方法が最も簡単です。

方法1: Homebrewを使う（推奨）

  1. ターミナルを起動（Spotlight検索で「ターミナル」と入力）
  2. Homebrewがインストールされていない場合、以下のコマンドを実行
Homebrewのインストール 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  1. Homebrewのインストール後、以下のコマンドでGitをインストール
Gitのインストールコマンド 
brew install git

方法2: Xcodeコマンドラインツールを使う

  1. ターミナルで以下のコマンドを実行
コマンドラインツールのインストール 
xcode-select --install
  1. ポップアップウィンドウで「インストール」をクリック

Linux環境でのインストール

Linux用Gitのインストール手順

ディストリビューションごとのパッケージマネージャーを使用してインストールします。

Ubuntu/Debian系

apt-getでインストール 
sudo apt-get update
sudo apt-get install git

Fedora/Red Hat系

dnfでインストール 
sudo dnf install git

Arch Linux系

pacmanでインストール 
sudo pacman -S git

インストールの確認

Gitが正しくインストールされたか確認しましょう。ターミナル（WindowsではGit Bash）を開いて、以下のコマンドを実行します。

Gitバージョン確認コマンド 
git --version

正常にインストールされていれば、以下のようにバージョン番号が表示されます：

実行結果の例 
git version 2.43.0

インストール成功

バージョン番号が表示されれば、Gitのインストールは成功です。次のステップに進みましょう。

2.3 GitHubアカウントの作成

GitHubは、Gitのリモートリポジトリをホスティングする最も人気のあるサービスです。無料でアカウントを作成できます。

GitHubアカウント作成手順

  1. GitHubのWebサイト（https://github.com/）にアクセス
  2. 右上の「Sign up」ボタンをクリック
  3. メールアドレスを入力して「Continue」をクリック
  4. パスワードを設定（8文字以上、数字と小文字を含む）
  5. ユーザー名を設定（英数字とハイフン、3〜39文字）
  6. 製品アップデート情報のメール受信可否を選択（「y」または「n」）
  7. 人間であることを確認するパズルを完了
  8. 「Create account」をクリック
  9. 登録したメールアドレスに届いた確認コードを入力
  10. 簡単なアンケートに回答（スキップも可能）
  11. 無料プラン「Free」を選択

ユーザー名の選び方

GitHubのユーザー名は、あなたのプロフィールURLや他のユーザーからの見え方に影響します。プロフェッショナルな印象を与える名前を選ぶことをお勧めします。後から変更も可能ですが、リポジトリのURLが変わるため、最初から考えて決めましょう。

2.4 Visual Studio Code（VS Code）のインストール

Visual Studio Code（VS Code）は、Microsoft社が提供する無料のテキストエディタです。Gitとの統合機能が優れており、初心者にも扱いやすいため、本教材で使用します。

VS Codeのインストール手順

  1. VS Code公式サイト（https://code.visualstudio.com/）にアクセス
  2. 「Download for Windows/Mac/Linux」ボタンをクリック（自動でOSを判定）
  3. ダウンロードしたインストーラーを実行
  4. ライセンス条項に同意して「次へ」
  5. インストール先フォルダを確認（デフォルトのままでOK）
  6. 追加タスクの選択で、以下にチェックを入れることを推奨：
    • 「デスクトップ上にアイコンを作成する」
    • 「PATHへの追加」
    • 「Codeで開く」をエクスプローラーのコンテキストメニューに追加
  7. 「インストール」をクリック
  8. インストール完了後、「完了」をクリック

VS Codeの基本設定

VS Codeを初めて起動したら、以下の設定を行うと使いやすくなります。

日本語化

拡張機能から「Japanese Language Pack」をインストール

  1. 左側の拡張機能アイコンをクリック
  2. 検索欄に「japanese」と入力
  3. 「Japanese Language Pack」をインストール
  4. VS Codeを再起動

テーマ設定

好みのカラーテーマを選択

  1. Ctrl+K, Ctrl+T（Mac: Cmd+K, Cmd+T）
  2. 好みのテーマを選択
  3. 人気のテーマ: Dark+、Light+、Monokai

自動保存の有効化

ファイルを自動的に保存する設定

  1. ファイル → 自動保存 をクリック
  2. チェックマークが付けばOK

Git拡張機能

GitLensなどの便利な拡張機能をインストール

  1. 拡張機能アイコンをクリック
  2. 「GitLens」を検索してインストール
  3. Gitの履歴が見やすくなる

2.5 Git BashとTerminalの基本

Gitコマンドを実行するには、ターミナル（コマンドライン）を使います。Windowsでは「Git Bash」、macOS/Linuxでは「ターミナル」を使用します。

Windows - Git Bash

Gitインストール時に一緒にインストールされるターミナル

起動方法：

  • スタートメニューから「Git Bash」を検索
  • または、フォルダ内で右クリック → 「Git Bash Here」

macOS - ターミナル

macOSに標準でインストールされているアプリ

起動方法：

  • Spotlight検索（Cmd+Space）で「ターミナル」と入力
  • または、アプリケーション → ユーティリティ → ターミナル

VS Codeの統合ターミナル

VS Code内でもターミナルを使用できます。メニューの「ターミナル」→「新しいターミナル」で開くか、ショートカットキー Ctrl+`（Mac: Cmd+`）で表示できます。これにより、コードとターミナルを同じ画面で操作できて便利です。

2.6 Gitの初期設定

Gitを使い始める前に、ユーザー名とメールアドレスを設定する必要があります。これらの情報は、コミット履歴に記録されます。

ユーザー名とメールアドレスの設定

ターミナル（またはGit Bash）を開いて、以下のコマンドを実行します。[あなたの名前]と[あなたのメール]の部分は、実際の名前とメールアドレスに置き換えてください。

ユーザー名の設定 
git config --global user.name "あなたの名前"
メールアドレスの設定 
git config --global user.email "your.email@example.com"

重要な注意事項

  • メールアドレスは、GitHubアカウントで使用しているものと同じにすることを推奨します
  • これらの情報は公開リポジトリでは誰でも見ることができます
  • プライバシーが心配な場合は、GitHubが提供する「no-reply」メールアドレスを使用できます

設定の確認

設定が正しく行われたか確認しましょう。

設定内容の確認 
git config --global --list

または、個別に確認する場合：

個別確認コマンド 
# ユーザー名の確認
git config user.name

# メールアドレスの確認
git config user.email

その他の推奨設定

以下の設定も行っておくと便利です。

デフォルトブランチ名を「main」に設定 
git config --global init.defaultBranch main
改行コードの自動変換設定（Windows） 
git config --global core.autocrlf true
改行コードの自動変換設定（Mac/Linux） 
git config --global core.autocrlf input

環境構築の最終確認

以下の項目がすべて完了していることを確認してください。

よくあるトラブルと解決方法

Q: Gitコマンドが認識されない（Windows）

解決策: システム環境変数のPATHにGitが追加されていない可能性があります。Gitを再インストールし、インストール時に「Git from the command line and also from 3rd-party software」を選択してください。

Q: GitHubのメール確認が届かない

解決策: 迷惑メールフォルダを確認してください。それでも見つからない場合は、GitHubの設定から確認メールを再送信できます。

Q: VS Codeでターミナルが開かない

解決策: 設定でデフォルトのシェルが正しく設定されているか確認してください。Windowsの場合は、VS Codeの設定で「Terminal › Integrated › Default Profile: Windows」を「Git Bash」に変更してみてください。

Q: 日本語が文字化けする

解決策: Git Bashで以下のコマンドを実行してください：
git config --global core.quotepath false
