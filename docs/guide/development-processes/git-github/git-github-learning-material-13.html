13.1 VS CodeのSource Control（ソース管理）パネル

Visual Studio Code（VS Code）には、Gitと統合された強力なソース管理機能が備わっています。コマンドラインで学んだ操作を、GUIで直感的に実行できます。

GUIとコマンドラインの使い分け

VS CodeのGUIは、日常的な操作を効率化するのに最適です。しかし、コマンドラインでの操作を理解していることで、「何が起きているか」を正確に把握でき、トラブル時にも対処できます。

Source Controlパネルを開く

Source Controlパネルを開く方法は3つあります：

方法1

左側のアクティビティバーにあるブランチアイコンをクリック

方法2

キーボードショートカット
Ctrl+Shift+G (Windows/Linux)
Cmd+Shift+G (Mac)

方法3

メニューから
View → Source Control

13.2 ファイルの変更状態の確認 - 色分け表示の意味

VS Codeでは、ファイルの変更状態が色とアイコンで一目でわかるように表示されます。これにより、git statusコマンドを実行しなくても、現在の作業状態を把握できます。

ファイルの状態表示（エクスプローラー）

状態 色 記号 意味
Modified M 変更されたファイル
Untracked U 新規作成された、まだ追跡されていないファイル
Deleted D 削除されたファイル
Renamed R 名前が変更されたファイル
Conflict C マージ時にコンフリクトが発生したファイル

コマンドラインとの対応

これらの色分けは、git statusコマンドの出力と同じ情報を視覚的に表示しています。

  • オレンジ色 (M) → modified（変更されたファイル）
  • 緑色 (U) → untracked（追跡されていないファイル）
  • 赤色 (D) → deleted（削除されたファイル）

ステータスバーでの表示

VS Code画面下部のステータスバーにも、Git関連の情報が表示されます：

ブランチ名

現在のブランチ（例: main）

同期状態

↓2 ↑3 のような表示（プル/プッシュ待ちのコミット数）

変更数

未コミットの変更ファイル数

13.3 ステージングとコミットをGUIで実行

VS Codeでは、git addやgit commitコマンドを入力せずに、マウス操作だけでステージングとコミットができます。

実習: ファイルをステージングしてコミット

ステップ1: ファイルを変更する

  1. VS Codeでプロジェクトフォルダを開く
  2. 既存のファイル（例: index.html）を編集する
  3. ファイル名がオレンジ色（M）で表示されることを確認

ステップ2: 変更をステージングする

  1. Source Controlパネルを開く（Ctrl+Shift+G）
  2. 「Changes」セクションに変更ファイルが表示される
  3. ファイル名の右側にある＋ボタンをクリック
  4. ファイルが「Staged Changes」セクションに移動する

全ファイルを一括ステージング

「Changes」セクションの右側にある＋ボタンをクリックすると、全ての変更を一度にステージングできます（git add .と同じ）。

ステップ3: コミットする

  1. コミットメッセージ入力欄にメッセージを入力（例: 「レイアウトを修正」）
  2. 上部の✓ Commitボタンをクリック
  3. ステージングエリアが空になることを確認

キーボードショートカット

Ctrl+Enter (Windows/Linux) または Cmd+Enter (Mac) でもコミットできます。

ステージングの取り消し

間違ってステージングしたファイルを元に戻すには：

ステージングを取り消す方法

  1. 「Staged Changes」セクションで、取り消したいファイルを選択
  2. ファイル名の右側にある−ボタンをクリック
  3. ファイルが「Changes」セクションに戻る

これはgit restore --staged <ファイル名>コマンドと同じ操作です。

変更の破棄

注意: 変更の破棄は元に戻せません

ファイルの変更を完全に破棄する場合、ファイルを右クリック → 「Discard Changes」を選択します。これはgit restore <ファイル名>と同じで、元に戻せないので注意してください。

13.4 変更差分（Diff）の確認と比較

VS Codeでは、git diffコマンドを実行しなくても、変更内容を視覚的に確認できます。

Diffビューを開く方法

方法1: Source Controlから

Source Controlパネルで変更ファイルをクリック

方法2: エクスプローラーから

エクスプローラーで変更ファイル（オレンジ色）をクリック

Diffビューの見方

Diffビューは左右2分割で表示されます：

領域 内容
左側 変更前の状態（前回のコミット時点）
右側 現在の作業ディレクトリの状態（変更後）
赤色の行 削除された行（左側に表示）
緑色の行 追加された行（右側に表示）

Diffビューの便利な機能

  • 行番号: 変更箇所の行番号が表示される
  • インラインDiff: 同じ行内の変更箇所が強調表示される
  • ナビゲーション: 上下の変更箇所にジャンプできる矢印ボタン
  • 折りたたみ: 変更のない部分は自動的に折りたたまれる

13.5 行単位でのステージング

VS Codeでは、ファイル全体ではなく、特定の行だけをステージングすることもできます。これは、1つのファイルに複数の異なる変更が含まれている場合に便利です。

実習: 行単位でのステージング

ステップ1: Diffビューを開く

  1. 複数箇所を変更したファイルをSource Controlパネルで選択
  2. Diffビューが開く

ステップ2: 特定の行をステージング

  1. 右側（変更後）のエディタで、ステージングしたい変更箇所にカーソルを置く
  2. 変更箇所の左側にある「Stage Change」ボタン（＋アイコン）をクリック
  3. その変更だけがステージングされる

使い分けの例

例えば、バグ修正とリファクタリングを同時に行った場合、それぞれを別々のコミットに分けることで、履歴がわかりやすくなります。

コマンドラインとの対応

これはgit add -p（パッチモード）と同じ機能ですが、VS CodeではGUIで直感的に操作できます。

13.6 VS Codeのターミナル統合機能

VS Codeには統合ターミナルが内蔵されており、エディタとターミナルを切り替えることなく、Gitコマンドを実行できます。

統合ターミナルを開く

キーボード

Ctrl+` (バッククォート)

メニュー

View → Terminal

パネル

下部パネルの「Terminal」タブ

統合ターミナルの利点

  • 作業ディレクトリが自動設定 - 開いているプロジェクトのルートフォルダで起動する
  • エディタとの連携 - ターミナルからファイルをすぐに開ける
  • 分割表示 - エディタとターミナルを同時に見ながら作業できる
  • 複数ターミナル - 複数のターミナルを開いて切り替えられる
統合ターミナルでのGitコマンド例 
# 統合ターミナルは既にプロジェクトフォルダで開かれている
# cdコマンドで移動する必要なし

git status
git log --oneline
git branch -a

GUIとコマンドラインの使い分け

日常的な操作（add、commit、diff）はGUIで効率的に行い、複雑な操作（rebase、cherry-pick等）はターミナルでコマンドを実行するのがおすすめです。

13.7 Git関連の設定カスタマイズ

VS Codeには、Git操作を快適にするための設定がいくつかあります。設定画面を開くには、Ctrl+,（Windows/Linux）またはCmd+,（Mac）を押します。

おすすめのGit設定

設定項目 説明 おすすめ
git.enabled Git機能の有効/無効 true（有効）
git.autofetch 定期的に自動でfetch true（便利）
git.confirmSync 同期時に確認ダイアログを表示 true（安全）
git.enableSmartCommit ステージング不要でコミット可能 false（明示的に）
git.ignoreLimitWarning 大量ファイル時の警告を無視 false（警告を表示）

設定方法

settings.json での設定例 
{
  "git.enabled": true,
  "git.autofetch": true,
  "git.confirmSync": true,
  "git.enableSmartCommit": false
}

settings.jsonの開き方

  1. Ctrl+Shift+P（Windows/Linux）またはCmd+Shift+P（Mac）でコマンドパレットを開く
  2. 「Preferences: Open Settings (JSON)」と入力
  3. settings.jsonファイルが開く

  • ファイルの変更状態が色とアイコンで表示される（M, U, D, R, C）
  • ステージング・コミットはマウス操作で簡単に実行できる
  • Diffビューで変更内容を左右比較できる
