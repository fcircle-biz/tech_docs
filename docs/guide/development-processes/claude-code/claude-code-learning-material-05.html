ファイルの読み書きとコード編集

Claude Code がプロジェクト内のファイルをどのように読み、新規作成・編集するかを学びます。変更内容（差分）の確認方法、複数ファイルにまたがる修正の進め方、そして AI に編集を任せながら内容を自分で確認・承認する大切さを身につけましょう。

この章で学ぶこと

  • Claude Code がファイルを読み込み、内容を把握する仕組みを理解する
  • 新規ファイルの作成・既存ファイルの編集を依頼し、差分を確認する流れを身につける
  • 「@」記号でファイルやディレクトリを指定して文脈に含める方法を知る
  • 変更を任せながら自分で確認・承認する安全な使い方の心構えを理解する

5.1 Claude Code はどうファイルを「読む」のか

Claude Code に何かを依頼すると、AI はまずプロジェクト内のファイルを読み込んで内容を把握するところから始めます。これは人間の開発者が作業前にコードを確認するのと同じです。

Claude Code が読む対象は、依頼の内容や関連するファイルに応じて自動的に判断されます。たとえば「greeting.js のバグを直して」と伝えれば、Claude Code はまずそのファイルを開いて中身を確認してから作業を始めます。

コンテキストとファイル参照

Claude Code が読めるのは現在の作業ディレクトリ（プロジェクトフォルダ）内のファイルが中心です。ファイルを明示的に指定したいときは、プロンプト内に @ファイル名 と書くと、そのファイルを文脈として読み込ませることができます。詳しくは 5.4 節で説明します。

Claude Code がファイルを読んで作業を進めるフロー

flowchart TD A["あなたの指示"] --> B["関連ファイルを自動で特定"] B --> C["ファイルの内容を読み込む"] C --> D["内容を理解して計画を立てる"] D --> E["編集・作成を実行する"] E --> F["変更内容を提示する"]

Claude Code は指示を受けたとき、必要なファイルを自ら探して読む能力を持っています。ただし、読めるファイルの量には限りがあるため（コンテキストウィンドウの制限）、大きなプロジェクトでは「どのファイルが関係するか」を指示で補足すると、より正確な作業ができます。

5.2 新規ファイルの作成と既存ファイルの編集

Claude Code は、新しいファイルを作成することも、既存のファイルを編集することも自然言語の指示だけで行えます。

新規ファイルの作成

「〇〇.js を新しく作って」と依頼すると、内容を含む新しいファイルを生成してくれます。

「utils.js を作って、数値をゼロ埋めする関数を書いて」

既存ファイルの編集

ファイルを読んだ上で、必要な箇所だけを変更します。ファイル全体を書き直すのではなく、差分を最小にするよう動作します。

「greeting.js のバグを修正して（ほかの部分は変えないで）」

「必要な箇所だけを変更する」という動作は重要なポイントです。たとえば100行のファイルにある3行のバグを直す場合、Claude Code はその3行のみを変更するように処理します。これにより、意図しない範囲まで書き換えてしまうリスクが下がります。

重要ポイント: 変更範囲を指定する

「他の部分は変えないで」「〇〇関数だけ直して」のように変更範囲を明示した指示を書くと、意図しない編集を防げます。曖昧な指示は広範な変更につながりやすいため、範囲を絞って伝えることを意識しましょう。

5.3 差分の確認と承認の流れ

Claude Code がファイルを編集しようとするとき、変更内容（差分）をターミナルに表示して、あなたの承認を求めることができます。これにより、AI が何を変えるのかを確認してから適用できる仕組みになっています。

実際のやり取りのイメージ（疑似ターミナル）:

terminal — ファイル編集の依頼と差分確認の例 
$ claude
> greeting.js の greet 関数が空文字を渡すとエラーになるので直して

● greeting.js を読み込んでいます…
● 問題を確認しました。空文字チェックが欠けています。
  次の変更を加えてよいですか？

  ─ greeting.js ──────────────────
  - function greet(name) {
  -   return "こんにちは、" + name + "さん！";
  + function greet(name) {
  +   if (!name) return "こんにちは！";  // 空文字ガード追加
  +   return "こんにちは、" + name + "さん！";
  ────────────────────────────────

  [y/n] > y
● 変更を適用しました。

差分表示では、-（赤）が削除される行、+（緑）が追加される行を示します。これは Git の差分表示と同じ形式で、何が変わるかを一目で把握できます。

注意: 承認前に必ず差分を読む習慣を

確認画面が出ても「y」をすぐ押さず、変更内容をひととおり目で追ってから承認する癖をつけましょう。特に削除行（-）は大切なコードが消える場合があります。疑問があれば一度「n」で断り、指示を修正して再度依頼できます。

変更内容を確認・承認する基本フロー

flowchart TD A["編集を依頼する"] --> B["Claude Codeが差分を提示"] B --> C{"内容を確認する"} C -->|"問題なし → 承認（y）"| D["変更がファイルに適用される"] C -->|"問題あり → 拒否（n）"| E["指示を修正して再依頼"] E --> A

5.4 「@」でファイル・ディレクトリを指定する（ファイル参照）

プロンプトの中で @ファイル名 や @ディレクトリ名 と書くと、そのファイルやフォルダの内容を文脈に含めて Claude Code に渡すことができます。「このファイルを参照しながら作業してほしい」という意図を明確に伝えられる便利な記法です。

記法 意味 使用例
@utils.js 指定ファイルを文脈に含める 「@utils.js の関数を使って処理を書いて」
@src/ ディレクトリ内を文脈に含める 「@src/ 全体を踏まえてリファクタリング案を出して」

特に複数のファイルが絡む作業では、Claude Code が自動で関連ファイルを特定しきれないこともあります。そのときは @ 記法でファイルを明示的に指定することで、より正確な文脈を AI に渡せます。

ファイル参照のコツ

ファイル数が多すぎると読み込み量がコンテキストウィンドウを超える場合があります。作業に必要な最小限のファイルを指定するのがベストプラクティスです。ディレクトリ全体より特定のファイルを指定するほうが、精度の高い結果につながります。

5.5 複数ファイルにまたがる修正

現実の開発では、1つの変更が複数のファイルに影響することがよくあります。たとえば関数名を変えると、その関数を使っている別のファイルも修正が必要になります。Claude Code は複数ファイルの変更を一連の作業として進めることができます。

terminal — 複数ファイルにまたがる修正の例 
$ claude
> formatDate 関数を dateFormatter に名前変更して。
  使っているファイルもすべて更新して

● utils.js, main.js, report.js を読み込んでいます…
● 3つのファイルで変更が必要です。

  1. utils.js  ─ 関数名を変更（1箇所）
  2. main.js   ─ 呼び出し箇所を更新（2箇所）
  3. report.js ─ 呼び出し箇所を更新（1箇所）

  変更を適用してよいですか？ [y/n] > y
● すべてのファイルを更新しました。

このように、指示1つで複数ファイルを横断した変更を一度に実行できます。ただし変更ファイルが多いほど差分の確認に注意が必要です。1つひとつの差分を確認してから承認するようにしましょう。

変更後は動作確認を忘れずに

複数ファイルを変更したあとは、アプリが正しく動くかを実際に確認する習慣をつけましょう。Claude Code に「変更後にテストを実行して」と依頼すると、自動でテストを走らせて結果を報告してくれます。

5.6 変更を任せながら安全に使う心構え

Claude Code に編集を任せることで作業が効率化されますが、最終的に何が変わったかを確認するのは人間であるあなたの役割です。AI は高い精度で作業しますが、指示の解釈がずれることや、意図しない副作用が生じることがゼロではありません。

差分を読む

承認前に変更内容をひととおり目で確認する。削除行（-）に特に注意。

Git を活用する

変更前にコミットしておけば、問題があっても即座に元の状態に戻せる。

動作を確認する

変更が適用されたあと、テストや手動確認でアプリが正しく動くかを検証する。

Claude Code はあくまで強力な協働パートナーです。指示を出し、変更を確認し、承認する主体は常にあなた自身です。差分を読む、Git で保護する、動作確認する——この3つの習慣が、Claude Code を安心して使い続けるための基盤になります。

やってみよう 5-1: ファイル作成 → 編集依頼 → 差分確認

Claude Code でファイルを新規作成し、続けて編集を依頼して差分を確認する一連の流れを体験しましょう。

手順

  1. 1 適当な作業フォルダに移動し、claude でセッションを開始する。
  2. 2 「hello.js を新規作成して。名前を引数に取り挨拶文字列を返す sayHello 関数を書いて」と依頼する。ファイルが作成されることを確認する。
  3. 3 次に「@hello.js の sayHello 関数に、名前が空文字だったとき "ゲスト" を使うガードを追加して」と依頼する。
  4. 4 提示された差分（変更内容）を読み、追加行（+）と削除行（-）を確認してから y で承認する。
  5. 5 実際に hello.js を開いて変更が正しく反映されているか目で確認する。

確認ポイント

  • 差分表示で変更箇所のみが変わっていた（ほかの行は変わっていない）
  • @hello.js の指定によって、Claude Code が正しいファイルを参照できた

理解度確認クイズ

  1. Q1. Claude Code が既存ファイルを編集するとき、ファイル全体を書き直すのではなく「必要な箇所だけを変更する」のはなぜですか？

    回答を表示

    変更範囲を最小限に抑えることで、意図しない箇所まで書き換えてしまうリスクを下げるためです。差分が小さいほど人間がレビューしやすくなる、という安全上の理由もあります。

  2. Q2. プロンプトで @utils.js のように「@」を使うと、何が起こりますか？

    回答を表示

    指定したファイル（この場合 utils.js）の内容が Claude Code の文脈（コンテキスト）に明示的に含まれ、そのファイルを参照した上で作業が進められるようになります。Claude Code が自動でファイルを特定しにくいときに特に有効です。

  3. Q3. 変更内容を承認する前に行うべき最も重要な確認は何ですか？ また、誤った変更が適用された場合に元に戻しやすくする方法は？

    回答を表示

    最も重要な確認は、提示された差分（変更内容）を目で読んで意図どおりかを確認することです。特に削除行（-）を見落とさないよう注意します。元に戻しやすくする方法としては、作業前に Git でコミットしておくことが最もシンプルかつ確実な手段です。
