スラッシュコマンドと便利機能

会話中に / で始まる「スラッシュコマンド」を使うと、会話のリセット・設定確認・モデル切り替えなど、Claude Code の各種機能を素早く呼び出せます。この章では代表的な組み込みコマンドを整理し、日々の作業を効率化する使い方を身につけます。

この章で学ぶこと

  • スラッシュコマンドとは何か、どのように入力して使うかを理解する
  • 代表的な組み込みコマンド（/help・/clear・/config・/model など）の役割を説明できる
  • カスタムスラッシュコマンドを定義できることを知り、その概要を理解する
  • 実際に /help・/clear・/config を試して、コマンドの動作を自分で確認できる

7.1 スラッシュコマンドとは

Claude Code のセッション中、プロンプト（入力欄）に / で始まる文字列を入力すると、通常の会話とは別に「機能を直接呼び出す」ことができます。これがスラッシュコマンドです。

通常のメッセージが「AI に依頼する言葉」であるのに対し、スラッシュコマンドは「ツール自体の動作を制御する命令」です。たとえば会話履歴をリセットしたい、現在の設定を確認したい、使用するモデルを変えたい、といったときに活躍します。

入力方法

セッション中のプロンプトに /コマンド名 と入力してEnterを押すだけです。引数が必要なコマンドは /model claude-opus-4-5 のようにスペース区切りで続けて入力します。

7.2 代表的な組み込みコマンド一覧

Claude Code にはあらかじめ用意された組み込みスラッシュコマンドがあります。よく使う代表的なものを整理しておきましょう。

コマンド 役割 使いどころ
/help 使えるコマンドの一覧・ヘルプを表示する コマンドを忘れたとき・初めて使うとき
/clear 現在の会話履歴（コンテキスト）をリセットする 別のタスクに切り替えるとき・会話が長くなったとき
/config 現在の設定を確認・変更する 設定値を確認したいとき・設定を変更したいとき
/model 使用する Claude モデルを切り替える 重いタスクに Opus を使いたいとき・高速応答が欲しいとき
/init プロジェクト用の CLAUDE.md 雛形を生成する 新しいプロジェクトで最初に設定ファイルを作りたいとき（詳細は第8章）
/review 現在の差分（変更内容）のコードレビューを依頼する コミット前に変更内容をレビューさせたいとき
/status 現在のセッション状態（モデル・コンテキスト使用量など）を表示する トークン消費量やモデル名を確認したいとき
/exit セッションを終了する 作業を終えてターミナルに戻るとき（Ctrl+C でも可）

最新のコマンド一覧は /help で確認

Claude Code はアップデートにより新しいコマンドが追加されることがあります。上記はよく使う代表例ですが、セッション中に /help を実行すると、その時点で使えるすべてのコマンドを確認できます。

7.3 よく使うコマンドの使い方

/help — ヘルプを表示する

最初に覚えるべきコマンドです。現在使えるスラッシュコマンドの一覧と、各コマンドの簡単な説明が表示されます。

terminal — /help の実行例 
# セッション中にそのまま入力するだけ
/help
# → 使えるコマンドの一覧と説明が表示される

/clear — 会話履歴をリセットする

現在の会話履歴（コンテキスト）をすべてリセットして、新しい会話を始めます。別のタスクに切り替えるときや、会話が長くなってきたときに使います。

terminal — /clear の実行例 
/clear
# → 「会話履歴をクリアしました」と表示され、
#    コンテキストが空の状態から再スタートできる

/clear と --continue の違い

/clear はセッション中に履歴を消去します。一方、claude --continue（第4章）は前回のセッションを継続して再開するコマンドです。使い分けを意識しましょう。

/config — 設定を確認・変更する

現在の設定（デフォルトのモデル、テーマ、言語設定など）を確認したり、対話的に変更したりできます。設定ファイル（settings.json）の内容を直接編集しなくても、よく使う設定はここから変更できます。

terminal — /config の実行例 
/config
# → 現在の設定一覧が表示される
# → 対話的なメニューから設定値を変更できる

/model — 使用モデルを切り替える

Claude Code が使用する Claude モデルをセッション中に切り替えます。引数なしで実行すると現在のモデルが確認でき、モデル名を指定すると切り替わります。

terminal — /model の使い方 
/model
# → 現在使用中のモデル名が表示される

/model claude-opus-4-5
# → 指定したモデルに切り替わる

7.5 カスタムスラッシュコマンド（ユーザー定義）

Claude Code では、自分でオリジナルのスラッシュコマンドを定義することもできます。たとえば「テストを実行してレビューするまでの一連の手順」を /my-review という名前でまとめておくと、毎回同じ手順を書かずに済みます。

プロジェクト共有コマンド

.claude/commands/ フォルダにMarkdownファイルを置くと、プロジェクト内でチーム共有できるコマンドになります。

個人用コマンド

~/.claude/commands/ フォルダに置くと、自分だけが使えるコマンドになります。どのプロジェクトでも使えます。

カスタムコマンドの作り方（概要）

コマンドの内容（実行してほしい指示）をMarkdownファイルとして保存するだけです。ファイル名がそのままコマンド名になります。例: my-review.md → /my-review。詳細な権限や設定の管理については第10章で学びます。

7.6 各種モードと実行制御（概要）

Claude Code には、AI がどのように作業を進めるかを制御する「モード」や「実行オプション」があります。権限や設定の詳細は第10章で扱いますが、ここでは代表的なものを紹介します。

モード / オプション 特徴
通常モード（デフォルト） ファイル変更やコマンド実行の前に確認を求める。安全で推奨される使い方。
プランモード（--plan） 実際の変更はせず、「何をどう変えるか」の計画だけを提示する。第13章で詳しく扱う。
プリント（非対話）モード（-p） スクリプトや自動化で使用。結果をテキストで出力してセッションを終了する。

通常モードが基本

初心者のうちはデフォルトの通常モード（確認つき実行）で使いましょう。AI が何かを変更・実行しようとするたびに確認を求めるため、意図しない変更を防げます。自動化やバッチ処理に必要なモードは、慣れてから学ぶのが安全です。

やってみよう 7-1: 主要コマンドを実際に試す

Claude Code を起動して、/help・/clear・/config の3つを実際に入力し、それぞれの動作を確認しましょう。

手順

  1. 1 ターミナルで claude を実行してセッションを開始する。
  2. 2 /help と入力してEnterを押す。表示されたコマンド一覧をざっと眺めておく。
  3. 3 適当なメッセージ（「テストファイルの構成を説明して」など）を送って会話履歴を作ったあと、/clear でリセットされることを確認する。
  4. 4 /config を実行して、現在のモデルや設定の内容を確認する。

確認ポイント

  • /help でコマンド一覧が表示されたか
  • /clear 後に「新しい会話」の状態になったか
  • /config で現在の設定が確認できたか

理解度確認クイズ

  1. Q1. スラッシュコマンドと通常のメッセージの違いを説明してください。

    回答を表示

    通常のメッセージは「AI に依頼する言葉」であり、Claude がその内容を解釈して作業を行います。スラッシュコマンド（/clear など）は Claude Code ツール自体の動作を制御する命令で、会話の内容とは切り離された特別な操作です。

  2. Q2. 別のタスクに切り替えるとき、どのスラッシュコマンドを使えばよいですか？その理由も説明してください。

    回答を表示

    /clear を使います。これまでの会話履歴（コンテキスト）がリセットされるため、前のタスクの情報が新しいタスクに混入せず、AI が新しい依頼に集中できます。

  3. Q3. カスタムスラッシュコマンドを「プロジェクトメンバー全員で共有したい」場合、どこにファイルを置けばよいですか？

    回答を表示

    プロジェクトルートの .claude/commands/ フォルダにMarkdownファイルを置きます。このフォルダをGit管理に含めることで、チーム全員が同じカスタムコマンドを使えます（個人用は ~/.claude/commands/）。
