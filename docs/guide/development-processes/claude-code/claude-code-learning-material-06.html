コードベースの探索と理解

既存のプロジェクトを Claude Code に調べさせて、コードの全体像や個々の処理の流れを素早く把握する方法を学びます。引き継いだコードや初めて触るコードの理解に特に力を発揮する「調査専用の使い方」を身につけましょう。

この章で学ぶこと

  • Claude Code がコードベースを探索する仕組みと、それを活用してプロジェクト全体の構造を把握させる方法を理解する
  • 「この関数はどこで使われている？」「この処理の流れを教えて」という調査依頼の出し方を習得する
  • 効果的な質問のコツ（具体的に・対象を絞る・段階的に深掘りする）を学ぶ
  • 編集せずに「調べるだけ」という使い方も有効であることを知り、安心して活用できるようになる

6.1 Claude Code がコードを「読む」仕組み

Claude Code は、ターミナルを起動したディレクトリをカレントディレクトリとして認識し、そこからファイルの読み込み・検索・一覧取得といったツールを自分で選んで使います。あなたが「このプロジェクトの構成を説明して」と頼むと、Claude Code は次のような手順で動きます。

flowchart TD A["あなたの調査依頼"] --> B["ディレクトリ一覧を取得する"] B --> C["重要そうなファイルを特定する"] C --> D["ファイルを読み込む・コードを検索する"] D --> E["内容を整理して説明する"] E --> F["あなたへ回答"]

調査の段階では、Claude Code は基本的に読み取り中心で動きます。あなたが明示的に「編集して」「修正して」と指示しない限り、調査だけでファイルが書き換わることは通常ありません。なお、編集を確実に避けたい場合は、後の章で学ぶプランモード（変更せず計画だけ提示するモード／第13章）や権限設定（第10章）を使うと、読み取り専用の状態を明示的に保てます。

調査依頼はファイルを変更しない

Claude Code が使える主なツールには「ファイルを読む」「ディレクトリを一覧する」「コードを検索する」などがあります。調査のみの依頼では原則これらの読み取り系ツールだけが使われ、プロジェクトの内容は書き換わりません。ファイルの変更が発生するのは、あなたが明示的に編集・作成・削除を依頼したときです。変更を確実に避けたい場合は、プランモード（第13章）や権限設定（第10章）で読み取り専用を明示しておくとより安全です。

6.2 プロジェクト全体の構造を把握させる

初めて触るプロジェクトや引き継いだコードを理解するとき、まず「プロジェクト全体の構成を教えて」と依頼するのが効果的です。Claude Code はディレクトリ構成を調べ、主要なファイルを読んで、役割を整理してくれます。

terminal — プロジェクト構成の調査例 
$ claude
> このプロジェクトの全体構成を説明してください

● ディレクトリ構成を確認しています…
● src/ フォルダ配下のファイルを読み込んでいます…
● package.json を確認しています…

このプロジェクトは Node.js 製の Web API です。
src/routes/ にエンドポイント定義、src/services/ にビジネスロジック、
src/models/ にデータ定義が配置されています。

一度プロジェクトの全体像を把握させると、その後の会話でも文脈が引き継がれるため、より具体的な調査へスムーズに移行できます。

代表的な調査依頼のパターン

構造の把握

「このプロジェクトの全体構成を教えて」

フォルダ構成・主要ファイルの役割を整理してもらう

使われ方の調査

「getUserById 関数はどこで使われている？」

特定の関数・変数の参照箇所をすべて洗い出す

処理フローの説明

「ログイン処理の流れを順番に説明して」

コードを追いながら処理の順序をわかりやすく整理してもらう

ファイル・関数の説明

「auth.js の中身を説明して」

特定ファイルの目的・各関数の役割を要約してもらう

6.3 調査依頼のセッション例

実際にどのようなやり取りになるか、疑似セッション例で流れを確認しましょう。引き継いだプロジェクトで、認証まわりの処理を理解したい場面です。

terminal — 調査セッション例（引き継ぎコードの理解） 
$ claude
> このプロジェクトの認証処理の流れを教えてください

● src/ のディレクトリ構成を確認しています…
● src/middleware/auth.js を読み込んでいます…
● src/routes/auth.js を読み込んでいます…

認証の流れは次のとおりです：
1. POST /login でユーザー名・パスワードを受け取る
2. auth.js の verifyPassword() でハッシュ照合する
3. 成功時は JWT を生成して Cookie にセットする
4. 以降のリクエストは authMiddleware で JWT を検証します

> verifyPassword はどこで定義されていますか？

● src/utils/crypto.js を読み込んでいます…

verifyPassword は src/utils/crypto.js の 23 行目に定義されています。
bcrypt.compare() を使ったハッシュ比較処理です。

最初に全体の流れを聞き、次に気になった関数の場所を深掘りするというやり取りが典型的なパターンです。Claude Code は前の回答で読んだファイルの情報を会話内で保持しているため、「verifyPassword は？」と短く聞くだけでも文脈を理解して答えてくれます。

6.4 効果的な質問のコツ

Claude Code の調査能力を最大限に引き出すには、質問の仕方が重要です。3つのコツを意識するだけで、回答の精度が大幅に上がります。

1

具体的に聞く

「コードを説明して」より「ログイン処理の流れを説明して」のように、何について知りたいかを明示します。

曖昧な例:

「このコードを説明して」

具体的な例:

「src/auth.js の verifyToken 関数の動きを説明して」

2

対象を絞る

プロジェクト全体ではなく、特定のファイル・機能・モジュールに絞ることで、より正確な回答が得られます。巨大なプロジェクトでも「まず○○の部分だけ教えて」と段階的に聞くのが効果的です。

例: 「まず src/models/ フォルダにあるファイルの役割だけ教えて」
3

段階的に深掘りする

全体→特定機能→個別関数という順で掘り下げると、理解が積み上がりやすくなります。Claude Code は会話内の文脈を保持するため、「その中の〇〇はどうなっている？」と続けて聞けます。

例: 全体構成 → 認証フロー → JWT の生成処理 → 署名アルゴリズムの詳細

重要ポイント: 調査だけの使い方も立派な活用

Claude Code は「コードを書かせる」だけのツールではありません。コードを理解するための相棒としても非常に有効です。引き継いだプロジェクト、大きなコードベース、複雑なライブラリ——いずれも「まず全体を教えて」と頼むところから始めるのが近道です。

6.5 調査が特に役立つ場面

Claude Code のコードベース探索機能は、次のような場面で特に大きな力を発揮します。

引き継ぎコード

前任者が書いたコードや、ドキュメントが少ないレガシーコードの構造をすばやく把握できます。

OSSの調査

使いたいライブラリやフレームワークの内部実装を調べて、どう動いているかを理解するのに役立ちます。

バグ調査の前段

修正に入る前に「この処理がどう動いているか」を把握することで、原因を絞り込みやすくなります。

注意: コンテキストウィンドウの上限に注意

プロジェクトが非常に大きい場合、一度にすべてのファイルを読み込むことは難しい場合があります（コンテキストウィンドウに上限があるため）。「まず src/ だけ」「このフォルダのみ」と対象を絞って依頼するか、調査を複数の会話に分けて進めると効果的です。コンテキストウィンドウについては第2章で学んだ内容を思い出してください。

やってみよう 6-1: 自分のプロジェクトを調査させてみよう

手元にある自分のプロジェクト（またはOSSのリポジトリをクローンしたもの）を題材に、調査依頼を試してみましょう。

手順

  1. 1 ターミナルでプロジェクトのルートディレクトリへ移動し、claude を起動する。
  2. 2 「このプロジェクトの全体構成を教えてください」と入力してEnterを押す。回答を読みながら、知らなかった部分や気になった部分をメモする。
  3. 3 メモした気になる機能や関数を1つ選び、「〇〇の処理の流れを詳しく教えて」と続けて質問する。段階的に深掘りしてみよう。
  4. 4 今回の調査でファイルが変更されていないことを git status で確認する（変更なしであることを確かめる）。

確認ポイント: 質問を具体的にするほど回答が詳しくなる実感をつかむ。また、調査だけではファイルが変わらないことを体験する。

理解度確認クイズ

  1. Q1. Claude Code に調査依頼（「このプロジェクトの構成を教えて」など）をしたとき、ファイルは変更されますか？ その理由も答えてください。

    回答を表示

    通常は変更されません。調査依頼では Claude Code は「ファイルを読む」「ディレクトリを一覧する」「コードを検索する」といった読み取り系のツールを中心に使うためです。ファイルの変更が発生するのは、明示的に編集・作成・削除を指示したときです。変更を確実に避けたい場合は、プランモード（第13章）や権限設定（第10章）で読み取り専用を明示できます。

  2. Q2. 効果的な調査依頼の3つのコツを挙げてください。

    回答を表示

    ①具体的に聞く（何についての情報か明示する）、②対象を絞る（特定ファイル・機能・モジュールに限定する）、③段階的に深掘りする（全体→特定機能→個別関数の順で掘り下げていく）の3つです。

  3. Q3. 非常に大きなプロジェクトで一度にすべてを調査させることが難しい場合、どのように対処するとよいですか？

    回答を表示

    調査対象を特定のフォルダや機能に絞って依頼するか、調査を複数の会話セッションに分けて進めると効果的です。コンテキストウィンドウには上限があるため、「まず src/ だけ」「この機能まわりのみ」と対象を限定することで正確な回答を得やすくなります。
