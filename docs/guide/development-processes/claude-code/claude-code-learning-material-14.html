14.1 よくあるトラブルと対処法

Claude Code を使い始めると、いくつかのつまずきポイントがあります。ここでは頻出のトラブルと、その解決策を整理します。焦らず一つひとつ確認してください。

トラブル 1: インストール後に claude コマンドが見つからない

claude: command not found のようなエラーが表示される場合、主に3つの原因が考えられます。

原因A: PATH が通っていない

npm でグローバルインストールした場合、npm bin -g のディレクトリが PATH に含まれていないことがあります。

terminal — PATHの確認と追加 
# npm グローバルインストール先を確認する
npm bin -g
# 出力されたパスを ~/.bashrc や ~/.zshrc の PATH に追加する
# 例: export PATH="$HOME/.npm-global/bin:$PATH"
source ~/.zshrc  # 設定を即時反映する

原因B: Node.js のバージョンが古い

Claude Code は Node.js 18 以上が必要です。node -v でバージョンを確認し、古ければ更新してください。

原因C: インストールに失敗している

インストール時にエラーが出ていなかった場合でも、再インストールで解決することがあります。

terminal — 再インストール 
# アンインストールして再インストールする
npm uninstall -g @anthropic-ai/claude-code
npm install -g @anthropic-ai/claude-code

トラブル 2: ログイン・認証に失敗する

起動後に認証エラーが出る、またはログインループが続く場合の対処法です。

対処A: 再ログインを試みる

既存の認証情報をクリアして、再度ログインします。

terminal — 再ログイン 
# ログアウトして再ログインする
claude logout
claude login

対処B: API キーを確認する

API キーで利用している場合は、ANTHROPIC_API_KEY 環境変数が正しく設定されているか確認してください。キーが有効期限切れや無効化されていないかも Anthropic Console で確認します。

対処C: Claude.ai アカウントのプランを確認する

Claude Code は Claude.ai の Pro / Max プラン、または Anthropic Console の API キーが必要です。無料プランでは利用できません。

トラブル 3: 権限まわりで操作が進まない

ファイル編集やコマンド実行の都度、確認ダイアログが出て作業が止まる、または操作が拒否される場合です。

対処A: 権限設定を見直す

settings.json の permissions セクションで、許可するツールと拒否するツールを設定します。詳細は 第10章「権限と設定」 を参照してください。

settings.json — 権限設定の例 
{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  }
}

対処B: 確認の頻度を調整する

確認ダイアログが多い場合は /config コマンドや settings.json で設定を調整できます。ただし、危険なコマンドへの制限は維持することを推奨します。

トラブル 4: 応答が意図とずれる・見当違いな作業をする

Claude Code が意図と異なる方向に作業を進める場合の対処法です。

対処A: 指示をより具体的にする

「直して」ではなく「○○ファイルの△△関数で、□□という条件のときに××が起きるバグを修正して」のように、対象・条件・期待する結果を明確に伝えます（第13章参照）。

対処B: /clear でコンテキストを整理する

会話が長くなるとコンテキストが蓄積し、以前のやり取りが影響することがあります。/clear で会話をリセットして、新しいセッションとして指示し直します。

terminal — コンテキストのリセット 
# 会話履歴をクリアして新しいセッションを開始する
/clear

対処C: CLAUDE.md に方針を明記する

プロジェクト固有のルール・制約・方針を CLAUDE.md に書いておくことで、Claude Code の行動を一貫させられます（第8章参照）。

14.2 環境診断: claude doctor

問題の原因が特定しにくいときは、claude doctor コマンドを使うと便利です。このコマンドは Claude Code の環境を診断し、よくある設定の問題を検出して報告します。

terminal — 環境診断の実行 
# 環境診断を実行する
claude doctor

claude doctor で確認できること

  • Node.js のバージョンが要件を満たしているか
  • インストールされている Claude Code のバージョン
  • 認証状態（ログイン済みかどうか）
  • 設定ファイルの読み込み状況

問題が発生したときは、次のフローで原因を切り分けると効率的です。

flowchart TD A["問題発生"] --> B["claude doctor を実行する"] B --> C{"エラーや警告はあるか？"} C -->|ある| D["診断結果に従って修正する"] C -->|ない| E{"claudeコマンドが見つからないか？"} E -->|はい| F["PATH / Node.js / 再インストールを確認する"] E -->|いいえ| G{"認証エラーか？"} G -->|はい| H["claude logout → claude login を実行する"] G -->|いいえ| I{"応答がずれているか？"} I -->|はい| J["/clear でリセット、指示を具体的にする"] I -->|いいえ| K["公式ドキュメントや GitHub Issues を確認する"]

自走するための心構え

Claude Code は日々進化するツールです。エラーが出たときは「公式ドキュメントで最新情報を確認する」「claude doctor で環境を診断する」「GitHub Issues で既知の問題を探す」という順番で対処すれば、ほとんどの問題は解決できます。困ったら一次情報に当たる習慣を身につけましょう。

理解度確認クイズ

  1. Q1. インストール後に claude コマンドが見つからない場合、最初に確認すべき3つの原因は何ですか？

    回答を表示

    ①PATH が通っていない（npm グローバルインストール先が PATH に含まれていない）、②Node.js のバージョンが古い（18以上が必要）、③インストール自体が失敗している（再インストールで解消することがある）の3点です。

  2. Q2. Claude Code の応答が意図とずれるとき、会話の蓄積が原因の場合はどうすればよいですか？

    回答を表示

    /clear コマンドで会話履歴をリセットし、新しいセッションとして指示し直します。合わせて指示内容をより具体的にし、プロジェクトの方針は CLAUDE.md に明記することで再発を防げます。

  3. Q3. このガイドを修了した後、さらに学びを深めるためにまず確認すべき一次情報源はどこですか？

    回答を表示

    Anthropic が提供する Claude Code の公式ドキュメント（https://docs.anthropic.com/en/docs/claude-code）です。最新の機能・設定・コマンドの正確な情報はここに集約されています。不具合や既知の問題は GitHub Issues（https://github.com/anthropics/claude-code）も参照します。
