基本操作と最初のセッション

Claude Code の実際の使い方を体験します。インストールが完了したら、次は「対話の流れ」を身につける番です。プロンプトの送り方、応答の読み方、処理の中断、セッションの継続・再開など、日々の操作に必要な基本をすべて網羅します。

この章で学ぶこと

  • セッションを開始し、自然言語で依頼を送る基本操作を習得する
  • Claude の応答を読み、AI が何をしているかを把握できるようになる
  • 処理の中断（Esc）、セッションの継続・再開の方法を理解する
  • ワンショット実行（print mode）の使いどころを知る

4.1 セッションを開始する

Claude Code との対話は「セッション」という単位で行います。セッションとは、一連の会話のやり取りのことです。作業フォルダ（プロジェクトのディレクトリ）に移動してから claude コマンドを実行すると、対話型の REPL（Read-Eval-Print Loop）が起動します。

terminal — セッション開始 
# 作業フォルダへ移動してから起動する
cd ~/my-project
claude

# 起動すると対話プロンプトが表示される
# > （ここに指示を入力してEnterキーで送信）

なぜ作業フォルダで起動するのか？

Claude Code は起動したフォルダをプロジェクトのルートとして認識します。ここから相対パスでファイルを読み書きするため、作業するプロジェクトのフォルダに移動してから claude を実行することが大切です。

セッション全体の流れは次のようになります。起動から終了まで、一つのまとまった会話として記録・管理されます。

flowchart LR A["ターミナルで
cd 作業フォルダ"] --> B["claude コマンドを実行"] B --> C["対話プロンプト起動
「>」が表示される"] C --> D["指示を入力して Enter"] D --> E["Claude が作業を実行"] E --> F["応答・結果を表示"] F --> G{"続けるか？"} G -->|次の指示を送る| D G -->|終了する| H["Ctrl+D または /exit"]

4.2 プロンプト（指示）の送り方

Claude Code への指示は自然言語で入力します。特別な構文や形式は不要です。対話プロンプト（>）にやりたいことを書いて Enter キーを押すだけです。

ファイル関連

「README.md を作って」「app.py の中身を確認して」

バグ修正・調査

「エラーの原因を調べて直して」「この関数の動きを説明して」

作成・追加

「Hello World を出力する Python スクリプトを作って」

良い指示の書き方

指示は具体的に書くほど精度が上がります。「やってほしいこと」「対象のファイルや範囲」「期待する結果」を含めると、Claude は意図をより正確に理解できます。

あいまいな指示（避ける） 具体的な指示（推奨）
「直して」 「src/main.py の TypeError を調べて修正して」
「説明して」 「utils.py の parse_csv 関数の処理の流れを日本語で説明して」
「追加して」 「app.py にログイン失敗時のエラーメッセージ表示を追加して」

重要ポイント

最初から完璧な指示を目指さなくて大丈夫です。Claude の応答を見ながら「もう少し詳しくして」「別のファイルも見て」と追加指示を送れます。会話を重ねるにつれ、意図が伝わるようになります。

4.3 応答の読み方

Claude Code が作業を進めるとき、画面には何をしているのかをリアルタイムで表示します。「ファイルを読んでいる」「編集した」「コマンドを実行した」などのステップが次々と出てきます。これを読み解くことが、AI の動作を把握する第一歩です。

terminal — 応答の例 
● src/main.py を読み込んでいます…        ← ファイルを読んでいる
● 問題箇所を特定しました。修正します。   ← 計画を立てた
● src/main.py を編集しました。            ← ファイルを編集した
● python -m pytest を実行しています…     ← コマンドを実行している
● テスト 3件 すべて成功しました。         ← 結果を確認した
修正が完了しました。変更内容をご確認ください。

「● 〜を読み込んでいます」

ファイルの内容を取得して分析しているステップ。どのファイルを参照しているか確認できます。

「● 〜を編集しました」

ファイルへの書き込みが行われたステップ。変更内容は後で diff 表示で確認できます。

「● 〜を実行しています」

コマンドを実行しているステップ。何のコマンドを実行しているか確認しましょう。

最後のメッセージ

作業の結果サマリーや次のアクションを案内します。ここを読んで作業完了か確認します。

確認のタイミング

ファイル編集やコマンド実行の前に、Claude が「この操作を行っていいですか？」と確認を求めることがあります。これは意図しない変更を防ぐ安全機能です。許可・拒否をその場で選択できます（権限設定の詳細は第10章で学びます）。

4.4 処理を中断する（Esc キー）

Claude が作業を進めている最中に「やっぱり止めたい」「別の指示に切り替えたい」と思ったときは、Esc キーを押すと実行中の処理を即座に中断できます。

注意: 中断後はファイルを確認しよう

Esc で中断した場合、すでに書き込まれたファイルへの変更は元に戻りません。途中まで編集されたファイルが残る可能性があります。中断後は対象ファイルの内容を確認しましょう。

中断後は、対話プロンプト（>）が再び表示されます。そのまま別の指示を入力したり、追加情報を渡してやり直したりできます。

terminal — 中断の操作例 
> 全ファイルをリファクタリングして
● src/ フォルダを読み込んでいます…
^[   ← ここで Esc キーを押した（Ctrl+C ではなく Esc）

中断しました。

> まず src/main.py だけリファクタリングして
         ↑ 範囲を絞り直して再指示

4.5 セッションの継続・再開

Claude Code では、過去のセッションを記憶として引き継いで作業を続けることができます。セッションを終了して後で戻ってきたときも、会話の流れをそのまま再開できます。

コマンド 動作 使いどころ
claude 新しいセッションを開始する まったく新しい作業を始めるとき
claude --continue 直前のセッションを自動的に再開する さっきの作業をすぐ続けたいとき
claude --resume 過去のセッション一覧を表示して選択する 少し前のセッションに戻りたいとき
terminal — セッション再開の例 
# 直前のセッションをそのまま再開（最速・最もよく使う）
claude --continue

# 過去のセッション一覧から選んで再開
claude --resume
# → 対話的な一覧が表示されるので矢印キーで選択して Enter

セッション管理の全体像

flowchart TD A["claude で新しいセッション開始"] --> B["会話・作業を進める"] B --> C["終了 (Ctrl+D / /exit)"] C --> D["セッション履歴に保存される"] D --> E{"再開方法を選ぶ"} E -->|"--continue"| F["直前のセッションを即再開"] E -->|"--resume"| G["一覧から選んで再開"] E -->|"claude のみ"| H["新しいセッションを開始"] F --> B G --> B H --> B

コンテキストが引き継がれる

--continue や --resume で再開すると、以前の会話の内容（コンテキスト）が引き継がれます。「さっき話していた auth.py の件なんですが…」のように、前の会話を前提にした指示がそのまま通じます。

4.6 ワンショット実行（print mode）

セッションを開かずに、1回だけ質問して結果を受け取る「print mode（-p オプション）」という使い方もあります。シェルスクリプトやコマンドパイプラインに組み込みたいときや、「ちょっと聞くだけ」のような場面で便利です。

terminal — ワンショット実行 
# -p オプションで1回だけ実行して終了
claude -p "今のフォルダにある Python ファイルを一覧して"

# パイプと組み合わせることもできる
cat error.log | claude -p "このエラーログの原因を教えて"

セッションとの違い

print mode は1回のやり取りで完結し、結果をそのままターミナルに出力して終了します。会話の継続や前後のコンテキストは保持されません。対話が不要な場面では積極的に使いましょう。

4.7 最初の依頼を出してみる

ここでは、実際に最初の依頼を出して Claude がどのように作業を進めるかを観察する流れを確認します。シンプルな「Hello World スクリプトの作成」を例に、セッションの一連の流れを見てみましょう。

terminal — 最初のセッション例（全体） 
$ cd ~/hello-demo    # 作業フォルダへ移動
$ claude             # セッション開始

> 「こんにちは！」と出力する Python スクリプトを hello.py として作って

# Claude の応答（リアルタイム表示）
● hello.py を作成しています…
● hello.py を書き込みました。

スクリプトを作成しました。
`python hello.py` で実行できます。

> 実際に動かして結果を確認して

● python hello.py を実行しています…
出力: こんにちは！

正常に動作しています。

観察ポイント

  • 「● 〜を作成しています」「● 〜を書き込みました」というステップ表示で、今何をしているか分かる
  • 最初の依頼でファイルを作り、次の依頼で実行するという連続した会話が成立している
  • Claude は自分でコマンドを実行して結果を確認し、「正常に動作しています」と報告している

やってみよう 4-1: 最初のセッション体験

起動・依頼・応答観察・中断体験・セッション継続の一連の流れを実際に体験してみましょう。

手順

  1. 1 ターミナルで適当な作業フォルダ（例: ~/sandbox）に移動して claude を実行する。
  2. 2 「今日の日付を表示するスクリプトを date.py として作って」と入力して Enter を押す。
  3. 3 応答の「● 〜しています」という表示を観察し、何をしているか確認する。
  4. 4 次に「全ファイルを一覧して」と入力して Enter を押し、処理が始まったら Esc キーで中断する体験をする。
  5. 5 Ctrl+D または /exit でセッションを終了した後、claude --continue で再開してみる。

確認ポイント: --continue で再開したとき、前の会話（date.py を作った話）を Claude が覚えているか確認してみましょう。

理解度確認クイズ

  1. Q1. Claude Code のセッション中に処理を即座に中断するにはどのキーを押しますか？

    回答を表示

    Esc キーを押すと、実行中の処理を即座に中断できます。ただし、中断前にすでに書き込まれたファイルへの変更は元に戻らないため、中断後はファイルの内容を確認することを習慣にしましょう。

  2. Q2. 直前のセッションを会話の流れを引き継いで再開するコマンドを答えてください。

    回答を表示

    claude --continue を使います。直前のセッションのコンテキスト（会話の内容）を引き継いで、そのまま作業を続けられます。一覧から選んで再開したい場合は claude --resume を使います。

  3. Q3. セッションを開かずに1回だけ質問して結果を受け取るオプション（print mode）は何ですか？

    回答を表示

    claude -p "質問内容" を使います（-p は print mode の略）。1回のやり取りで完結し、結果をそのままターミナルに出力して終了します。シェルスクリプトとの組み合わせやパイプ処理でも活用できます。
