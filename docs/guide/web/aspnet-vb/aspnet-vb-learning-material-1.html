第1章: ASP.NET基礎とセットアップ

ASP.NET Web Formsの基礎と開発環境構築

1.1 ASP.NET Web Formsとは

ASP.NET Web Formsは、Microsoftが開発したWebアプリケーション開発フレームワークです。Windowsアプリケーションのようなイベント駆動型のプログラミングモデルをWebに持ち込んだ技術です。

ASP.NET Web Formsの主な特徴

  • イベント駆動モデル: ボタンクリックなどのイベントを処理
  • サーバーコントロール: 再利用可能なUI部品
  • ViewState: ページ状態の自動保持
  • ページライフサイクル: 明確なページ処理フロー
  • RAD開発: Visual Studioによる高速開発

1.2 開発環境の準備

ASP.NET開発を始めるために必要なツールとソフトウェアを準備します。

必要なソフトウェア

  • Visual Studio 2019/2022: Community Edition（無料）で十分
  • .NET Framework 4.7.2以上: Visual Studioに含まれる
  • IIS Express: Visual Studioに含まれるWeb サーバー
  • SQL Server Express LocalDB: データベース学習用（オプション）
実習 1-1: Visual Studioのインストールと設定

Visual Studio Community Editionをインストールし、ASP.NET開発環境を構築します。

手順
  1. Visual Studio ダウンロードページからCommunity Editionをダウンロード
  2. インストーラーを実行し、以下のワークロードを選択：
    • ASP.NET と Web 開発
    • .NET デスクトップ開発
  3. インストール完了後、Visual Studioを起動
  4. Microsoft アカウントでサインイン（推奨）
期待される結果

Visual Studioが正常に起動し、新しいプロジェクト作成画面が表示されることを確認します。

1.3 初回ASP.NETプロジェクトの作成

Visual Studioを使用して最初のASP.NET Web Formsプロジェクトを作成します。

実習 1-2: Hello World Webアプリケーションの作成

シンプルなWebアプリケーションを作成し、ASP.NETの基本構造を理解します。

手順
  1. Visual Studioで「ファイル」→「新規作成」→「プロジェクト」を選択
  2. 「ASP.NET Web アプリケーション (.NET Framework)」を選択
  3. プロジェクト名：「HelloWorldApp」、場所を指定して「作成」
  4. テンプレートで「Web Forms」を選択
  5. 「作成」をクリック
プロジェクト構造の確認
  • Default.aspx: デフォルトのWebページ
  • Default.aspx.vb: VB.NETコードビハインドファイル
  • Web.config: アプリケーション設定ファイル
  • Global.asax: アプリケーションレベルのイベント処理
期待される結果

プロジェクトが正常に作成され、ソリューションエクスプローラーに各ファイルが表示されます。

1.4 初回ページの編集とVB.NETコード作成

作成したプロジェクトのDefault.aspxページを編集し、VB.NETでサーバーサイド処理を追加します。

実習 1-3: Hello Worldページの実装

ASP.NETページにVB.NETコードを追加し、動的コンテンツを表示します。

手順
  1. Default.aspxファイルをダブルクリックしてデザイナーを開く
  2. ツールボックスから「Label」コントロールと「Button」コントロールを配置
  3. Buttonをダブルクリックしてコードビハインドファイル（Default.aspx.vb）を開く
  4. 以下のVB.NETコードを追加
実行例

Default.aspx (デザイン部分)

<%@ Page Title="Home Page" Language="VB" MasterPageFile="~/Site.Master" AutoEventWireup="true" CodeBehind="Default.aspx.vb" Inherits="HelloWorldApp._Default" %>

<asp:Content ID="BodyContent" ContentPlaceHolderID="MainContent" runat="server">
    <div class="row">
        <div class="col-md-4">
            <h2>ASP.NET Hello World</h2>
            <asp:Label ID="lblMessage" runat="server" Text="こんにちは！" Font-Size="Large"></asp:Label>
            <br /><br />
            <asp:Button ID="btnClick" runat="server" Text="クリックしてください" CssClass="btn btn-primary" />
            <br /><br />
            <asp:Label ID="lblResult" runat="server" Font-Bold="True" ForeColor="Blue"></asp:Label>
        </div>
    </div>
</asp:Content>

Default.aspx.vb (コードビハインド)

Public Class _Default
    Inherits System.Web.UI.Page

    Protected Sub Page_Load(ByVal sender As Object, ByVal e As System.EventArgs) Handles Me.Load
        ' ページが初めて読み込まれる時のみ実行
        If Not IsPostBack Then
            lblResult.Text = "ページが読み込まれました: " & DateTime.Now.ToString("yyyy/MM/dd HH:mm:ss")
        End If
    End Sub

    Protected Sub btnClick_Click(sender As Object, e As EventArgs) Handles btnClick.Click
        ' ボタンクリック時の処理
        Static clickCount As Integer = 0
        clickCount += 1
        
        lblResult.Text = String.Format("ボタンが {0} 回クリックされました！現在時刻: {1}", 
                                      clickCount, DateTime.Now.ToString("HH:mm:ss"))
        
        ' メッセージを変更
        Select Case clickCount
            Case 1
                lblMessage.Text = "初回クリックありがとうございます！"
            Case 5
                lblMessage.Text = "5回クリックされました！"
            Case 10
                lblMessage.Text = "10回達成！ASP.NET楽しいですね！"
            Case Else
                lblMessage.Text = "ASP.NET Web Forms with VB.NET"
        End Select
    End Sub
End Class
期待される結果

F5キーでアプリケーションを実行し、ブラウザでページが表示され、ボタンクリックで動的にコンテンツが変更されることを確認します。

1.5 ASP.NETページライフサイクルの基本

ASP.NETページの処理フローとライフサイクルイベントについて理解します。

主要なページライフサイクルイベント

イベント タイミング 用途
Page_Load ページ読み込み時 初期化処理、データバインド
Page_PreRender HTML出力前 最終的な変更処理
Button.Click コントロールイベント ユーザーアクション処理
Page_Unload ページ処理完了後 リソース解放
実習 1-4: ページライフサイクルの観察

各ライフサイクルイベントでログを出力し、実行順序を確認します。

手順
  1. 新しいWebフォーム「LifeCycle.aspx」を追加
  2. ページにLabelコントロールとButtonコントロールを配置
  3. 各ライフサイクルイベントにログ出力処理を追加
実行例

LifeCycle.aspx.vb

Public Class LifeCycle
    Inherits System.Web.UI.Page
    
    Private Sub AddLog(message As String)
        lblLog.Text += DateTime.Now.ToString("HH:mm:ss.fff") & " - " & message & "<br/>"
    End Sub

    Protected Sub Page_Init(sender As Object, e As EventArgs) Handles Me.Init
        AddLog("Page_Init: ページ初期化")
    End Sub

    Protected Sub Page_Load(sender As Object, e As EventArgs) Handles Me.Load
        AddLog("Page_Load: ページ読み込み (IsPostBack: " & IsPostBack.ToString() & ")")
        
        If Not IsPostBack Then
            AddLog("初回アクセスです")
        Else
            AddLog("ポストバック処理です")
        End If
    End Sub

    Protected Sub btnTest_Click(sender As Object, e As EventArgs) Handles btnTest.Click
        AddLog("btnTest_Click: ボタンクリックイベント")
    End Sub

    Protected Sub Page_PreRender(sender As Object, e As EventArgs) Handles Me.PreRender
        AddLog("Page_PreRender: HTML出力前")
    End Sub

    Protected Sub Page_Unload(sender As Object, e As EventArgs) Handles Me.Unload
        ' Unloadイベントではページに出力できないため、デバッグ出力のみ
        System.Diagnostics.Debug.WriteLine("Page_Unload: ページ処理完了")
    End Sub
End Class
ASPX ページ
<asp:Label ID="lblLog" runat="server" Text=""></asp:Label>
<br /><br />
<asp:Button ID="btnTest" runat="server" Text="ライフサイクルテスト" CssClass="btn btn-info" />
期待される結果

ページの読み込みとボタンクリック時に、各イベントの実行順序とタイミングが表示されます。

1.6 デバッグとトラブルシューティング

Visual Studioのデバッグ機能を使用して、ASP.NET アプリケーションの問題を解決する方法を学びます。

デバッグの基本操作

  • ブレークポイント: F9キーで設定/解除
  • ステップ実行: F10（ステップオーバー）、F11（ステップイン）
  • 変数監視: ローカルウィンドウ、ウォッチウィンドウ
  • 即座ウィンドウ: デバッグ中の式評価
よくある初心者のエラー
  • コンパイルエラー: 構文エラー、タイプミス
  • NullReferenceException: オブジェクトの初期化忘れ
  • ポストバック問題: EnableViewStateの理解不足
  • パス問題: 相対パスと絶対パスの混同
理解度確認クイズ
  1. ASP.NET Web Formsの主な特徴を3つ挙げてください。
  2. Visual Studioで新しいASP.NET Web Forms プロジェクトを作成する手順を説明してください。
  3. 「Default.aspx」と「Default.aspx.vb」ファイルの関係と役割を説明してください。
  4. Page_Loadイベントはいつ発生し、どのような用途で使用しますか？
  5. IsPostBackプロパティの役割と使用場面を説明してください。
  6. ASP.NETでデバッグを行う際の基本的な操作を3つ挙げてください。
解答例
  1. 1)イベント駆動モデル、2)サーバーコントロール、3)ViewStateによる状態管理
  2. Visual Studio起動→新規プロジェクト→ASP.NET Web アプリケーション→Web Formsテンプレート選択
  3. Default.aspxはUIデザイン、Default.aspx.vbはVB.NETによるサーバーサイド処理を記述
  4. ページ読み込み時に発生。初期化処理やデータバインドに使用
  5. 初回アクセスかポストバックかを判定。初回のみ実行したい処理の制御に使用
  6. 1)ブレークポイント設定、2)ステップ実行、3)変数値の監視
