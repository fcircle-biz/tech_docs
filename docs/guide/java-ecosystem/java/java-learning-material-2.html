2.1 変数とデータ型

プログラミングにおける変数とは、データを格納するための「箱」のような存在です。現実世界でラベル付きの箱に物を入れて管理するように、プログラムでは変数に値を入れて管理します。

変数とは何か？

変数は以下の3つの要素から構成されます：

  • 名前（変数名）：変数を識別するための名称
  • 型（データ型）：格納できるデータの種類
  • 値（データ）：実際に格納されている内容
graph TD A[変数の概念] --> B[名前] A --> C[型] A --> D[値] B --> E[age] B --> F[name] B --> G[isStudent] C --> H[int 整数] C --> I[String 文字列] C --> J[boolean 真偽値] D --> K[25] D --> L["田中太郎"] D --> M[true]

Javaの基本データ型

Javaには以下の基本データ型（プリミティブ型）があります。これらは最も基本的で効率的なデータ型です：

分類 データ型 サイズ 値の範囲 用途
整数 byte 8ビット -128 ～ 127 小さな整数、バイト配列
short 16ビット -32,768 ～ 32,767 中程度の整数
int 32ビット 約-21億 ～ 21億 一般的な整数（最頻使用）
long 64ビット 約-922京 ～ 922京 大きな整数
小数 float 32ビット 約±3.4×10³⁸（精度7桁） 単精度浮動小数点
double 64ビット 約±1.8×10³⁰⁸（精度15桁） 倍精度浮動小数点（最頻使用）
文字 char 16ビット 0 ～ 65,535 単一文字（Unicode）
真偽値 boolean 1ビット true または false 論理値、条件判定

参照型（Reference Type）

基本データ型とは別に、参照型があります。最も重要なのが String 型（文字列型）です：

  • String：文字列を格納（「こんにちは」「Hello World」など）
  • 配列：同じ型の複数の値を順序付けて格納
  • クラス型：ユーザー定義のデータ型

変数名の命名規則
  • アルファベット、数字、アンダースコア（_）、ドル記号（$）が使用可能
  • 数字で始めることはできない
  • 予約語（if、for、classなど）は使用できない
  • キャメルケース記法を推奨：firstName、totalPrice
  • 意味のある名前を付ける：x より studentCount

2.2 演算子

演算子は、変数や値に対して計算や比較などの操作を行うための記号です。Javaには多くの演算子があり、それぞれ異なる目的で使用されます。

算術演算子

数値に対する基本的な計算を行います：

演算子 名称 例 結果 説明
+ 加算 5 + 3 8 足し算
- 減算 5 - 3 2 引き算
* 乗算 5 * 3 15 掛け算
/ 除算 10 / 3 3（整数除算） 割り算
% 剰余 10 % 3 1 余り
整数除算の注意点

Javaでは、整数同士の割り算は整数結果となります。10 / 3 は 3.33... ではなく 3 になります。小数点以下も得たい場合は、どちらか一方をdouble型にキャストするか、double型の値を使用してください。

int a = 10, b = 3;
System.out.println(a / b);           // 3 (整数除算)
System.out.println((double)a / b);   // 3.3333... (小数除算)

比較演算子

値を比較してtrue または false を返します：

演算子 名称 例 意味
== 等価 a == b aとbが等しい
!= 不等価 a != b aとbが等しくない
> より大きい a > b aがbより大きい
< より小さい a < b aがbより小さい
>= 以上 a >= b aがb以上
<= 以下 a <= b aがb以下

論理演算子

boolean値（trueまたはfalse）に対する論理操作を行います：

演算子 名称 例 結果
&& 論理AND true && false false（両方がtrueの時のみtrue）
|| 論理OR true || false true（どちらかがtrueならtrue）
! 論理NOT !true false（真偽値を反転）
実習 2-2: 演算子の動作確認

各種演算子の動作を確認しましょう。

public class OperatorExample {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;
        
        // 算術演算子
        System.out.println("=== 算術演算子 ===");
        System.out.println(a + " + " + b + " = " + (a + b));
        System.out.println(a + " - " + b + " = " + (a - b));
        System.out.println(a + " * " + b + " = " + (a * b));
        System.out.println(a + " / " + b + " = " + (a / b));
        System.out.println(a + " % " + b + " = " + (a % b));
        
        // 比較演算子
        System.out.println("\n=== 比較演算子 ===");
        System.out.println(a + " == " + b + " : " + (a == b));
        System.out.println(a + " != " + b + " : " + (a != b));
        System.out.println(a + " > " + b + " : " + (a > b));
        System.out.println(a + " < " + b + " : " + (a < b));
        
        // 論理演算子
        boolean x = true;
        boolean y = false;
        System.out.println("\n=== 論理演算子 ===");
        System.out.println(x + " && " + y + " = " + (x && y));
        System.out.println(x + " || " + y + " = " + (x || y));
        System.out.println("!" + x + " = " + (!x));
    }
}

2.3 条件分岐

プログラムは通常、上から下へ順番に実行されますが、条件分岐を使うことで、特定の条件に応じて異なる処理を実行できます。これは人間の思考過程と同じです：「もし雨が降っているなら傘を持っていく、そうでなければ傘は不要」

flowchart TD A[プログラム開始] --> B{条件判定} B -->|true| C[条件が真の時の処理] B -->|false| D[条件が偽の時の処理] C --> E[プログラム続行] D --> E

if文の基本形

最も基本的な条件分岐は if文 です：

if (条件式) {
    // 条件がtrueの時に実行される処理
}

if-else文

条件が真の場合と偽の場合で異なる処理を行う場合：

if (条件式) {
    // 条件がtrueの時の処理
} else {
    // 条件がfalseの時の処理
}

if-else if-else文

複数の条件を順番にチェックする場合：

if (条件式1) {
    // 条件式1がtrueの時の処理
} else if (条件式2) {
    // 条件式2がtrueの時の処理
} else if (条件式3) {
    // 条件式3がtrueの時の処理
} else {
    // どの条件もfalseの時の処理
}
実習 2-3: 成績判定プログラム

点数に応じて成績を判定するプログラムを作成しましょう。

public class GradeExample {
    public static void main(String[] args) {
        int score = 75;  // ここの値を変更して動作を確認
        String grade;
        String message;
        
        // 成績判定
        if (score >= 90) {
            grade = "A";
            message = "優秀です！";
        } else if (score >= 80) {
            grade = "B";
            message = "良好です。";
        } else if (score >= 70) {
            grade = "C";
            message = "普通です。";
        } else if (score >= 60) {
            grade = "D";
            message = "もう少し頑張りましょう。";
        } else {
            grade = "F";
            message = "再試験が必要です。";
        }
        
        // 結果表示
        System.out.println("点数: " + score + "点");
        System.out.println("成績: " + grade);
        System.out.println("コメント: " + message);
        
        // ボーナス判定
        if (score >= 60) {
            System.out.println("合格です！");
            if (score >= 90) {
                System.out.println("特別賞も受賞できます！");
            }
        } else {
            System.out.println("不合格です。再試験を受けてください。");
        }
    }
}

switch文

一つの変数に対して複数の値をチェックする場合、switch文が便利です：

switch (変数) {
    case 値1:
        // 値1の時の処理
        break;
    case 値2:
        // 値2の時の処理
        break;
    default:
        // どのcaseにも当てはまらない時の処理
        break;
}

2.4 繰り返し処理

繰り返し処理（ループ）は、同じ処理を何度も実行したい場合に使用します。例えば「1から10まで数える」「配列の全要素を処理する」といった場面で威力を発揮します。

flowchart TD A[ループ開始] --> B{継続条件チェック} B -->|true| C[処理実行] C --> D[ループ変数更新] D --> B B -->|false| E[ループ終了]

for文

回数が決まっている繰り返しに最適です：

for (初期化; 継続条件; 更新処理) {
    // 繰り返し実行される処理
}
for文の実行順序
  1. 初期化：最初に1回だけ実行（通常はカウンタ変数の設定）
  2. 継続条件チェック：条件がtrueなら処理続行、falseなら終了
  3. 処理実行：ブロック内の処理を実行
  4. 更新処理：通常はカウンタ変数のインクリメント
  5. 2に戻って繰り返し
実習 2-5: for文による繰り返し処理

for文を使った基本的な繰り返し処理を学習します。

public class ForExample {
    public static void main(String[] args) {
        // 基本的なカウントアップ
        System.out.println("=== 1から10までカウント ===");
        for (int i = 1; i <= 10; i++) {
            System.out.println("カウント: " + i);
        }
        
        // 掛け算表
        System.out.println("\n=== 7の段の掛け算 ===");
        for (int i = 1; i <= 9; i++) {
            System.out.println("7 × " + i + " = " + (7 * i));
        }
        
        // 偶数だけ表示
        System.out.println("\n=== 1から20までの偶数 ===");
        for (int i = 2; i <= 20; i += 2) {
            System.out.print(i + " ");
        }
        System.out.println();  // 改行
        
        // 逆順カウント
        System.out.println("\n=== 10から1へカウントダウン ===");
        for (int i = 10; i >= 1; i--) {
            System.out.println("あと " + i + " 秒");
        }
        System.out.println("時間切れ！");
    }
}

while文

条件が満たされている間、繰り返しを続けます：

while (継続条件) {
    // 条件がtrueの間、繰り返し実行される処理
}

do-while文

最低1回は処理を実行し、その後条件をチェックします：

do {
    // 最低1回は実行される処理
} while (継続条件);
実習 2-6: while文とdo-while文の比較

while文とdo-while文の動作の違いを確認しましょう。

public class WhileExample {
    public static void main(String[] args) {
        // while文の例
        System.out.println("=== while文の例 ===");
        int count = 1;
        while (count <= 5) {
            System.out.println("while: カウント " + count);
            count++;
        }
        
        // do-while文の例
        System.out.println("\n=== do-while文の例 ===");
        count = 1;
        do {
            System.out.println("do-while: カウント " + count);
            count++;
        } while (count <= 5);
        
        // 条件が最初からfalseの場合の違い
        System.out.println("\n=== 条件が最初からfalseの場合 ===");
        
        // while文：処理は1回も実行されない
        int num = 10;
        while (num < 5) {
            System.out.println("while: この文は実行されません");
            num++;
        }
        System.out.println("while文の処理回数: 0回");
        
        // do-while文：処理は1回実行される
        num = 10;
        int doWhileCount = 0;
        do {
            System.out.println("do-while: この文は1回実行されます");
            doWhileCount++;
            num++;
        } while (num < 5);
        System.out.println("do-while文の処理回数: " + doWhileCount + "回");
    }
}

2.5 配列

配列は、同じ型の複数の値をひとまとめにして管理するデータ構造です。例えば、学生5人の点数を管理する場合、5つの変数を個別に作るよりも、1つの配列で管理する方が効率的です。

graph TD A[配列の概念] --> B[配列名: scores] B --> C[インデックス0: 85] B --> D[インデックス1: 92] B --> E[インデックス2: 78] B --> F[インデックス3: 96] B --> G[インデックス4: 83] H[個別変数の場合] --> I[score1 = 85] H --> J[score2 = 92] H --> K[score3 = 78] H --> L[score4 = 96] H --> M[score5 = 83]

配列の宣言と初期化

配列は以下の方法で作成できます：

// 方法1: 宣言と初期化を別々に行う
int[] scores;              // 配列の宣言
scores = new int[5];       // 5要素の配列を作成（初期値は0）

// 方法2: 宣言と同時に初期化
int[] scores = new int[5];

// 方法3: 初期値を指定して作成
int[] scores = {85, 92, 78, 96, 83};

// 方法4: new演算子で初期値指定
int[] scores = new int[]{85, 92, 78, 96, 83};
配列の重要なポイント
  • インデックス：配列の各要素は0から始まる番号（インデックス）でアクセス
  • length：配列名.length で配列の長さ（要素数）を取得
  • 範囲外アクセス：存在しないインデックスにアクセスするとエラー発生
  • 同一型：配列内のすべての要素は同じ型である必要がある
実習 2-7: 配列の基本操作

配列の作成、値の設定、取得、および配列の走査を学習しましょう。

public class ArrayExample {
    public static void main(String[] args) {
        // 学生の点数を管理する配列
        int[] scores = {85, 92, 78, 96, 83};
        String[] names = {"田中", "佐藤", "鈴木", "高橋", "渡辺"};
        
        System.out.println("=== 学生の成績一覧 ===");
        System.out.println("学生数: " + scores.length + "人");
        
        // 各学生の成績を表示
        for (int i = 0; i < scores.length; i++) {
            System.out.println(names[i] + "さんの点数: " + scores[i] + "点");
        }
        
        // 統計情報の計算
        int total = 0;
        int highest = scores[0];
        int lowest = scores[0];
        String topStudent = names[0];
        
        for (int i = 0; i < scores.length; i++) {
            total += scores[i];
            
            if (scores[i] > highest) {
                highest = scores[i];
                topStudent = names[i];
            }
            
            if (scores[i] < lowest) {
                lowest = scores[i];
            }
        }
        
        double average = (double) total / scores.length;
        
        System.out.println("\n=== 統計情報 ===");
        System.out.println("合計点: " + total + "点");
        System.out.println("平均点: " + String.format("%.1f", average) + "点");
        System.out.println("最高点: " + highest + "点（" + topStudent + "さん）");
        System.out.println("最低点: " + lowest + "点");
        
        // 拡張for文（for-each文）の使用
        System.out.println("\n=== 拡張for文による表示 ===");
        for (String name : names) {
            System.out.print(name + "さん ");
        }
        System.out.println("\nの成績を確認しました。");
    }
}

拡張for文（for-each文）

配列のすべての要素を順番に処理する場合、拡張for文が便利です：

for (データ型 変数名 : 配列名) {
    // 各要素に対する処理
}

2.6 コメント

コメントは、プログラムの説明や注釈を記述するための機能です。コンパイラに無視されるため、プログラムの動作には影響しませんが、コードの理解を大幅に助けます。

コメントの種類

種類 記述方法 用途 例
単行コメント // 一行の説明 // これは変数の宣言です
複数行コメント /* */ 複数行にわたる説明 /* これは
複数行の
コメントです */
Javadocコメント /** */ APIドキュメント生成 /** クラスの説明 */
良いコメントの書き方
  • なぜを説明する：「何をしているか」よりも「なぜそうしているか」
  • 簡潔で分かりやすく：冗長すぎず、必要十分な説明
  • 更新を忘れない：コードを変更したらコメントも更新
  • 日本語でOK：チーム開発では統一されたルールに従う

  4. c) 30 - 配列のインデックスは0から始まるため、インデックス2は3番目の要素
  5. b) 5回 - i が 0, 1, 2, 3, 4 の時に実行される

章のまとめ

この章では、Java言語の基本的な構成要素について学習しました：

  • 変数とデータ型：プリミティブ型（int、double、boolean等）と参照型（String、配列等）
  • 演算子：算術演算子、比較演算子、論理演算子による値の操作
  • 条件分岐：if文、else文、switch文による処理の分岐制御
  • 繰り返し処理：for文、while文、do-while文による反復処理
  • 配列：同じ型の複数の値をまとめて管理する仕組み
  • コメント：プログラムの可読性を向上させる説明文の記述方法
