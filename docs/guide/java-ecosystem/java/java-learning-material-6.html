第6章: 例外処理

プログラムの堅牢性を高める

この章で学ぶこと
  • 例外とエラーの概念と違い
  • try-catch-finally 文の使用方法
  • 例外の種類（検査例外と非検査例外）
  • 例外のスローと伝播メカニズム
  • 独自例外クラスの作成方法
  • 例外処理のベストプラクティス

6.1 例外処理の基本概念

例外（Exception）は、プログラムの実行中に発生する予期しない事象です。適切に処理することで、プログラムのクラッシュを防ぎ、ユーザーに適切な情報を提供できます。

classDiagram class Throwable { +getMessage() +printStackTrace() } class Error { システムレベルのエラー 回復困難 } class Exception { アプリケーションレベル 回復可能 } class RuntimeException { 非検査例外 実行時例外 } class IOException { 検査例外 コンパイル時チェック } Throwable <|-- Error Throwable <|-- Exception Exception <|-- RuntimeException Exception <|-- IOException
例外の分類
  • 検査例外（Checked Exception）：コンパイル時に処理が必要（IOException等）
  • 非検査例外（Unchecked Exception）：実行時例外（RuntimeException等）
  • エラー（Error）：システムレベルの重大な問題（OutOfMemoryError等）

6.2 try-catch-finally 文

例外処理の基本構文です。tryブロック内で例外が発生した場合、対応するcatchブロックが実行されます。

実習 6-1: 基本的な例外処理
BasicExceptionExample.java
public class BasicExceptionExample {
    public static void main(String[] args) {
        // 算術例外の処理
        System.out.println("=== 算術例外の処理 ===");
        try {
            int a = 10;
            int b = 0;
            int result = a / b; // ゼロ除算でArithmeticException発生
            System.out.println("結果: " + result);
        } catch (ArithmeticException e) {
            System.out.println("エラー: ゼロで割ることはできません");
            System.out.println("例外メッセージ: " + e.getMessage());
        }
        
        // 配列の範囲外アクセス
        System.out.println("\n=== 配列範囲外アクセス ===");
        int[] numbers = {1, 2, 3, 4, 5};
        try {
            System.out.println("5番目の要素: " + numbers[4]); // OK
            System.out.println("10番目の要素: " + numbers[9]); // 範囲外
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("エラー: 配列の範囲を超えています");
            System.out.println("配列サイズ: " + numbers.length);
        }
        
        // 複数の例外処理
        System.out.println("\n=== 複数例外の処理 ===");
        String[] data = {"10", "20", "abc", "40"};
        for (int i = 0; i < data.length + 1; i++) {
            try {
                String str = data[i];
                int num = Integer.parseInt(str);
                System.out.println("数値: " + num);
            } catch (ArrayIndexOutOfBoundsException e) {
                System.out.println("配列範囲外: インデックス " + i);
            } catch (NumberFormatException e) {
                System.out.println("数値変換エラー: " + data[i] + " は数値ではありません");
            }
        }
        
        System.out.println("プログラム終了");
    }
}

6.3 finally ブロック

finallyブロックは、例外の有無に関わらず必ず実行されます。リソースのクリーンアップに使用されます。

実習 6-2: finally ブロックの使用
FinallyExample.java
import java.io.*;

public class FinallyExample {
    
    public static void readFileExample(String filename) {
        FileReader reader = null;
        try {
            System.out.println("ファイルを開いています: " + filename);
            reader = new FileReader(filename);
            
            // ファイル読み込み処理（簡略化）
            int data = reader.read();
            System.out.println("ファイル読み込み成功");
            
        } catch (FileNotFoundException e) {
            System.out.println("エラー: ファイルが見つかりません - " + filename);
        } catch (IOException e) {
            System.out.println("エラー: ファイル読み込みに失敗しました");
            e.printStackTrace();
        } finally {
            // リソースのクリーンアップ
            System.out.println("リソースをクリーンアップしています");
            if (reader != null) {
                try {
                    reader.close();
                    System.out.println("ファイルを閉じました");
                } catch (IOException e) {
                    System.out.println("ファイルクローズでエラー発生");
                }
            }
        }
    }
    
    public static void main(String[] args) {
        System.out.println("=== 存在するファイルの場合 ===");
        // 存在しないファイルでテスト
        readFileExample("nonexistent.txt");
        
        System.out.println("\n=== 正常終了の場合 ===");
        // try-with-resources（Java 7以降の推奨方法）
        try (FileWriter writer = new FileWriter("test.txt")) {
            writer.write("テストデータ");
            System.out.println("ファイル書き込み完了");
        } catch (IOException e) {
            System.out.println("書き込みエラー: " + e.getMessage());
        }
        // try-with-resourcesでは自動的にclose()が呼ばれる
    }
}

6.4 例外のスローと伝播

throw文を使って意図的に例外を発生させることができます。また、メソッドはthrows文で例外を宣言できます。

実習 6-3: 例外のスローと伝播
ThrowExample.java
public class ThrowExample {
    
    // 例外をスローするメソッド
    public static void validateAge(int age) throws IllegalArgumentException {
        if (age < 0) {
            throw new IllegalArgumentException("年齢は0以上である必要があります: " + age);
        }
        if (age > 150) {
            throw new IllegalArgumentException("年齢は150以下である必要があります: " + age);
        }
        System.out.println("有効な年齢: " + age + "歳");
    }
    
    // 検査例外をスローするメソッド
    public static void processData(String data) throws Exception {
        if (data == null || data.trim().isEmpty()) {
            throw new Exception("データが空です");
        }
        
        if (data.length() < 3) {
            throw new Exception("データが短すぎます: " + data);
        }
        
        System.out.println("データ処理完了: " + data);
    }
    
    // 銀行口座の例
    public static void withdraw(double balance, double amount) {
        try {
            if (amount <= 0) {
                throw new IllegalArgumentException("引き出し額は0より大きくしてください");
            }
            if (amount > balance) {
                throw new RuntimeException("残高不足です。残高: " + balance + ", 引き出し額: " + amount);
            }
            
            double newBalance = balance - amount;
            System.out.println("引き出し成功。新しい残高: " + newBalance);
            
        } catch (IllegalArgumentException e) {
            System.out.println("入力エラー: " + e.getMessage());
        } catch (RuntimeException e) {
            System.out.println("取引エラー: " + e.getMessage());
        }
    }
    
    public static void main(String[] args) {
        System.out.println("=== 年齢検証テスト ===");
        int[] ages = {25, -5, 200, 30};
        
        for (int age : ages) {
            try {
                validateAge(age);
            } catch (IllegalArgumentException e) {
                System.out.println("年齢エラー: " + e.getMessage());
            }
        }
        
        System.out.println("\n=== データ処理テスト ===");
        String[] dataList = {"Hello World", "", null, "Hi"};
        
        for (String data : dataList) {
            try {
                processData(data);
            } catch (Exception e) {
                System.out.println("処理エラー: " + e.getMessage());
            }
        }
        
        System.out.println("\n=== 銀行取引テスト ===");
        withdraw(1000.0, 500.0);  // 正常
        withdraw(1000.0, 1500.0); // 残高不足
        withdraw(1000.0, -100.0); // 不正な金額
    }
}

6.5 独自例外クラス

アプリケーション固有の例外を表現するために、独自の例外クラスを作成できます。

実習 6-4: 独自例外クラスの作成
BankAccount.java（例外クラスも含む）
// 独自例外クラス1: 残高不足例外
class InsufficientFundsException extends Exception {
    private double balance;
    private double requestedAmount;
    
    public InsufficientFundsException(double balance, double requestedAmount) {
        super("残高不足: 残高=" + balance + ", 要求額=" + requestedAmount);
        this.balance = balance;
        this.requestedAmount = requestedAmount;
    }
    
    public double getBalance() { return balance; }
    public double getRequestedAmount() { return requestedAmount; }
    public double getShortfall() { return requestedAmount - balance; }
}

// 独自例外クラス2: 無効な口座操作例外
class InvalidAccountOperationException extends RuntimeException {
    public InvalidAccountOperationException(String message) {
        super(message);
    }
}

// 銀行口座クラス
public class BankAccount {
    private String accountNumber;
    private String accountHolder;
    private double balance;
    private boolean isActive;
    
    public BankAccount(String accountNumber, String accountHolder, double initialBalance) {
        this.accountNumber = accountNumber;
        this.accountHolder = accountHolder;
        this.balance = initialBalance;
        this.isActive = true;
    }
    
    // 預金メソッド
    public void deposit(double amount) throws InvalidAccountOperationException {
        if (!isActive) {
            throw new InvalidAccountOperationException("口座が無効です");
        }
        if (amount <= 0) {
            throw new InvalidAccountOperationException("預金額は0より大きくしてください");
        }
        
        balance += amount;
        System.out.println(amount + "円を預金しました。残高: " + balance + "円");
    }
    
    // 引き出しメソッド
    public void withdraw(double amount) throws InsufficientFundsException, InvalidAccountOperationException {
        if (!isActive) {
            throw new InvalidAccountOperationException("口座が無効です");
        }
        if (amount <= 0) {
            throw new InvalidAccountOperationException("引き出し額は0より大きくしてください");
        }
        if (amount > balance) {
            throw new InsufficientFundsException(balance, amount);
        }
        
        balance -= amount;
        System.out.println(amount + "円を引き出しました。残高: " + balance + "円");
    }
    
    // 口座情報表示
    public void displayInfo() {
        System.out.println("口座番号: " + accountNumber + ", 名義: " + accountHolder + 
                          ", 残高: " + balance + "円, 状態: " + (isActive ? "有効" : "無効"));
    }
    
    // ゲッター
    public double getBalance() { return balance; }
    public void setActive(boolean active) { this.isActive = active; }
    
    public static void main(String[] args) {
        BankAccount account = new BankAccount("123-456", "田中太郎", 10000);
        account.displayInfo();
        
        try {
            // 正常な取引
            account.deposit(5000);
            account.withdraw(3000);
            
            // 残高不足のテスト
            account.withdraw(20000);
            
        } catch (InsufficientFundsException e) {
            System.out.println("取引失敗: " + e.getMessage());
            System.out.println("不足額: " + e.getShortfall() + "円");
            
        } catch (InvalidAccountOperationException e) {
            System.out.println("操作エラー: " + e.getMessage());
        }
        
        try {
            // 無効な口座での操作
            account.setActive(false);
            account.deposit(1000);
            
        } catch (InvalidAccountOperationException e) {
            System.out.println("操作エラー: " + e.getMessage());
        }
    }
}
例外処理のベストプラクティス
  • 具体的な例外をキャッチ：Exception ではなく具体的な例外クラスを使用
  • 例外を無視しない：空の catch ブロックは避ける
  • リソース管理：try-with-resources を活用
  • 例外メッセージ：わかりやすいエラーメッセージを提供
  • ログ出力：適切なログレベルで例外情報を記録
理解度確認クイズ
  1. 検査例外と非検査例外の違いを説明し、それぞれの例を2つずつ挙げてください。
  2. finally ブロックが実行される条件と、主な用途を説明してください。
  3. throw と throws の違いを説明してください。
  4. 独自例外クラスを作成するメリットを3つ挙げてください。
  5. try-with-resources 文の利点と使用場面を説明してください。
