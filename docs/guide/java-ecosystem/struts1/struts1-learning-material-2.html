第2章: 環境構築とプロジェクト作成

Struts 1.x 開発環境のセットアップ

この章で学ぶこと
  • Pleiades Eclipse All in Oneを使った簡単環境構築
  • struts-blank.warを使ったプロジェクト作成
  • EclipseでのWARファイルインポート
  • Tomcatでのアプリケーション実行
  • 開発環境の動作確認
新しいアプローチ

従来の複雑な手動環境構築ではなく、Pleiades Eclipseとstruts-blank.warを使用して、最小限の手順で開発環境を構築します。

2.1 Pleiades Eclipse All-in-One の準備

Pleiades Eclipse All-in-Oneを使えば、Java、Eclipse、Tomcatが一つのパッケージになっているので、面倒な設定なしで開発環境が構築できます。

Pleiades Eclipse All-in-One の特徴
  • 日本語化済みのEclipse IDE
  • Tomcat 9が最初から組み込み済み
  • JDK 17が同梱（別途インストール不要）
  • Mavenも最初から使用可能
  • Zip展開するだけで使える
事前準備
  • ハードディスク容量: 最低5GB以上の空き容量
  • メモリ: 8GB以上推奨（4GBでも動作可能）
  • Windows: Windows 10/11 (64bit)
  • 権限: ファイル展開とフォルダ作成権限
実習 2-1: Pleiades Eclipse All-in-One のダウンロードと設定

すべてが一つになったPleiadesパッケージをダウンロードして設定します。

ステップ 1: Pleiades Eclipse All-in-One のダウンロード
  1. Pleiades All-in-One ダウンロードページにアクセス
  2. 「Eclipse 2023-12 Java Full Edition」を選択
  3. Windows x86_64版をダウンロード
  4. ファイルサイズは約2GB（時間に余裕を持ってダウンロード）
ステップ 2: Pleiades の展開とセットアップ
  1. ダウンロードした.zipファイルを右クリック
  2. 「すべて展開...」を選択
  3. 展開先フォルダを指定（例: C:\pleiades）
  4. 注意: パスに日本語や空白が含まれないフォルダを選択
  5. 展開完了まで待機（5-10分程度）
ステップ 3: Eclipse の起動確認
  1. 展開されたフォルダ内のeclipseフォルダを開く
  2. eclipse.exeをダブルクリックして起動
  3. ワークスペースの場所を聞かれたらデフォルトのまま「起動」をクリック
  4. Eclipseが日本語で起動することを確認

2.2 struts-blank.war の入手

Strutsプロジェクトを手早く作成するために、struts-blank.war（空のStrutsプロジェクトテンプレート）を使用します。

struts-blank.war とは

struts-blank.warは、Strutsの基本設定がすべて完了した「空のプロジェクト」です。以下が含まれています：

  • 必要なStrutsライブラリ（jarファイル）
  • web.xmlの基本設定
  • struts-config.xmlのひな形
  • 基本的なフォルダ構造

これを使えば面倒な設定作業をスキップして、すぐにStrutsアプリ開発に集中できます。

実習 2-2: struts-blank.war のダウンロード

Apache Struts公式サイトからstruts-blank.warをダウンロードします。

ダウンロード手順
  1. Apache Struts公式サイトにアクセス： https://archive.apache.org/dist/struts/binaries/
  2. struts-1.3.10-bin.zipをクリックしてダウンロード
  3. ダウンロードしたstruts-1.3.10-bin.zipを適当なフォルダに解凍
  4. 解凍されたフォルダ内のwebappsフォルダを開く
  5. struts-blank.warファイルを見つける
  6. このファイルを分かりやすい場所（例: デスクトップ）にコピー
ファイル構成の確認

struts-1.3.10-bin.zipを解凍すると以下のような構成になっています：

struts-1.3.10/ ├── lib/ ← Strutsライブラリファイル ├── webapps/ │ ├── struts-blank.war ← これを使います！ │ ├── struts-examples.war │ └── struts-mailreader.war └── docs/ ← ドキュメント

2.3 Eclipse での WAR ファイルインポート

struts-blank.warをEclipseにインポートして、Strutsプロジェクトを作成します。とても簡単で、設定ファイルも自動で用意されます。

Eclipse でのプロジェクトインポートとは

EclipseはWARファイル（Webアプリのパッケージファイル）を展開して、編集可能なプロジェクトとしてインポートできます。struts-blank.warをインポートすることで、すぐに開発を開始できます。

実習 2-3: Eclipse での WAR ファイルインポート

struts-blank.warをEclipseにインポートして、Strutsプロジェクトを作成します。

インポート手順
  1. Eclipseのメニューから「ファイル」→「インポート」を選択
  2. 「Web」フォルダを展開し、「WARファイル」を選択
  3. 「次へ」をクリック
  4. WARファイル欄で「参照」をクリック
  5. 先ほど保存したstruts-blank.warを選択
  6. プロジェクト名を「struts-learning」に変更
  7. ターゲットランタイムで「Tomcat v9.0 Server」を選択
  8. 「完了」をクリック
インポート後のプロジェクト構造

インポートが成功すると、以下のような構造でプロジェクトが作成されます：

struts-learning/ ├── src/ │ └── main/ │ └── webapp/ │ ├── WEB-INF/ │ │ ├── web.xml ← Webアプリ設定 │ │ ├── struts-config.xml ← Struts設定 │ │ └── lib/ ← Strutsライブラリ │ ├── index.jsp ← トップページ │ └── Welcome.do ← サンプルアクション └── WebContent/ (同じ内容)
インポート成功の確認

以下が表示されていれば成功です：

  • プロジェクトエクスプローラーに「struts-learning」プロジェクト
  • WEB-INF/libフォルダ内にStrutsのJARファイル
  • WEB-INF/struts-config.xmlファイル
  • Welcome.jspなどのサンプルファイル

2.4 Tomcat でのアプリケーション実行

struts-blank.warにはサンプルのStrutsアプリが含まれています。まずは既存のアプリを動かして、Strutsの動作を確認しましょう。

struts-blank.war に含まれるサンプル

struts-blank.warには以下のサンプルファイルが含まれています：

  • Welcome.do: サンプルアクション
  • Welcome.jsp: ウェルカムページ
  • WelcomeAction.java: サンプルアクションクラス
  • struts-config.xml: 基本設定済み

これらのファイルを確認して、Strutsの基本的な仕組みを理解しましょう。

実習 2-4: サンプルアプリケーションの実行

struts-blank.warに含まれているサンプルアプリケーションを実行して、Strutsの動作を確認します。

ステップ 1: プロジェクトの確認
  1. プロジェクトエクスプローラーで「struts-learning」プロジェクトを展開
  2. WebContentフォルダを確認
  3. WEB-INF/struts-config.xmlをダブルクリックで開いて内容を確認
  4. Welcome.jspファイルがあることを確認
ステップ 2: Tomcat サーバーでの実行
  1. プロジェクトを右クリック → 「実行」 → 「サーバーで実行」
  2. 「Tomcat v9.0 Server」を選択し、「次へ」
  3. プロジェクトが右側に表示されることを確認
  4. 「完了」をクリック
  5. Tomcatサーバーが起動するまで待機
ステップ 3: ブラウザでの動作確認
  1. Eclipse内蔵ブラウザが自動で開きます
  2. または外部ブラウザで以下にアクセス:
アクセスURL
  • トップページ: http://localhost:8080/struts-learning/
  • ウェルカムアクション: http://localhost:8080/struts-learning/Welcome.do

メモ: 「.do」で終わるのStrutsアクションの特徴です！

成功した場合の表示

以下のページが表示されれStruts環境の構築成功です：

  • 「Welcome!」のメッセージ
  • Strutsのロゴと基本的な説明
  • サンプルリンクが動作する

2.5 プロジェクトのカスタマイズ

サンプルアプリが動作したら、簡単なカスタマイズをしてStrutsの仕組みを理解しましょう。

実習 2-5: サンプルページのカスタマイズ

サンプルページを少し変更してみましょう：

ステップ 1: JSP ファイルの編集
  1. Welcome.jspをダブルクリックで開く
  2. タイトル部分を見つけて「私の最初のStrutsアプリ」に変更
  3. Ctrl+Sで保存
  4. ブラウザでF5を押してリロード
  5. 変更が反映されることを確認
ステップ 2: 設定ファイルの確認
  1. WEB-INF/struts-config.xmlを開いて確認：
    • action-mappings セクション
    • form-beans セクション
    • global-forwards セクション
  2. WEB-INF/web.xmlでStrutsの設定を確認：
    • ActionServlet の設定
    • *.do マッピング
    • タグライブラリの設定
ステップ 3: ライブラリファイルの確認

WEB-INF/libフォルダ内の主要なJARファイルを確認：

  • struts.jar: Strutsコアライブラリ
  • commons-*.jar: Apache Commonsライブラリ群
  • jakarta-*.jar: Jakartaライブラリ

2.6 Eclipse での開発効率化設定

EclipseでStruts開発をより効率的に行うための設定を行いましょう。

実習 2-6: Eclipse の設定最適化

Struts開発をしやすくするためのEclipse設定を行います。

基本設定
  1. 文字エンコーディング:
    • 「ウィンドウ」→「設定」
    • 「一般」→「ワークスペース」
    • 「テキストファイルエンコーディング」を「UTF-8」に設定
  2. フォント設定:
    • 「一般」→「外観」→「色とフォント」
    • 「テキストエディタブロックフォント」で読みやすいフォントを選択
  3. 自動保存:
    • 「一般」→「エディター
    • 「保存時に自動ビルド」を有効化
Tomcatサーバー設定
  1. サーバービューでの設定:
    • Eclipse下部の「サーバー」ビューを確認
    • Tomcatサーバーをダブルクリック
    • 「モジュール」タブで「自動リロード」を有効化
ショートカットキー
機能ショートカット説明
保存Ctrl+Sファイルの保存
実行Ctrl+F11アプリケーションの実行
終了Ctrl+F2サーバーの停止
コメントCtrl+/コメントアウト

2.7 トラブルシューティング

よくある問題と解決方法を確認しておきましょう。

よくある問題と解決方法
Eclipse関連
  • Eclipseが起動しない: パスに日本語や空白が含まれていないか確認。管理者権限で実行してみる。
  • メモリエラー: eclipse.iniで-Xmxの値を小さくする（例: -Xmx1024m）
  • 文字化け: 「設定」→「一般」→「ワークスペース」で文字エンコーディングを「UTF-8」に設定
WARインポート関連
  • 「WARファイル」が表示されない: Eclipse for Enterprise Java版を使用しているか確認
  • Tomcatが選択できない: 先にTomcatサーバーを設定してください
  • プロジェクトにエラーマーク: プロジェクトを右クリック→「リフレッシュ」を実行
アプリ実行関連
  • 404エラー: プロジェクトが正しくデプロイされているかサーバービューで確認
  • 500エラー: Tomcatのログを確認（コンソールビュー）
  • アクセスできない: ポート8080が他のアプリで使用されていないか確認
  • Tomcat起動エラー: サーバーを削除して再作成してみる
パフォーマンス関連
  • 動作が遅い: メモリ設定を確認、不要なプロジェクトを閉じる
  • ファイルの変更が反映されない: プロジェクトをクリーンビルドする
注意点とベストプラクティス
  • Pleiades EclipseにはJDK 17が同梱されていますが、Struts 1.xとの互換性は問題ありません
  • Struts 1.xはメンテナンス終了しているため、新規開発での使用は推奨されません（学習目的のみ）
  • ホットデプロイが有効になっているので、ファイル保存時に自動的に変更が反映されます
  • パスの日本語・空白を避けることで、トラブルを防げます
  • struts-blank.warは古い設定ファイル形式ですが、学習には最適です

解答を確認
