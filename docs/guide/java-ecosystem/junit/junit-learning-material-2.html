第2章: 基本的なテストケースの作成

効果的なテストメソッドの書き方

この章で学ぶこと
  • テストメソッドの基本的な構造を理解する
  • assertEquals、assertTrue、assertFalseなどの基本的なアサーションを使い分ける
  • 意味のあるテスト名の付け方を学ぶ
  • 成功するテストと失敗するテストの両方を作成する
  • テストコードの可読性を向上させるコツを習得する

2.1 テストメソッドの基本構造

良いテストメソッドには、明確な構造があります。前章で学んだAAAパターンを詳しく見てみましょう。

flowchart TD A["テストメソッド開始"] --> B["Arrange（準備）"] B --> C["Act（実行）"] C --> D["Assert（検証）"] B --> E["テストデータ用意"] B --> F["オブジェクト初期化"] C --> G["メソッド呼び出し"] D --> H["期待値と実際値の比較"] D --> I["テスト完了"]

2.1.1 Arrange（準備）段階

テストに必要なデータやオブジェクトを準備します：

@Test
void 商品の税込み価格計算テスト() {
    // Arrange（準備）
    double basePrice = 1000.0;
    double taxRate = 0.10;
    PriceCalculator calculator = new PriceCalculator();
    
    // Act（実行）
    double result = calculator.calculateTaxIncludedPrice(basePrice, taxRate);
    
    // Assert（検証）
    assertEquals(1100.0, result);
}

2.1.2 Act（実行）段階

テスト対象のメソッドを実際に呼び出します。この段階は通常1行で済みます：

// 良い例：Actは1つの処理
double result = calculator.calculateTaxIncludedPrice(basePrice, taxRate);

// 避けるべき例：複数の処理が混在
double result = calculator.calculateTaxIncludedPrice(basePrice, taxRate);
calculator.setDiscount(0.05);  // これは別のテストにすべき

2.1.3 Assert（検証）段階

実行結果が期待通りかを検証します。複数のアサーションがあっても構いません：

// 複数のアサーションの例
@Test
void ユーザー情報登録テスト() {
    User user = userService.registerUser("田中太郎", "tanaka@example.com");
    
    // 複数の検証を行う
    assertNotNull(user);
    assertEquals("田中太郎", user.getName());
    assertEquals("tanaka@example.com", user.getEmail());
    assertTrue(user.getId() > 0);
}

2.2 基本的なアサーションメソッド

JUnit 5では、様々なアサーションメソッドが用意されています。それぞれの特徴と使い分けを理解しましょう。

2.2.1 assertEquals - 値の等価性テスト

最も基本的なアサーションで、期待値と実際の値が等しいかを検証します：

@Test
void 等価性テストの例() {
    // 数値の比較
    assertEquals(10, 5 + 5);
    
    // 文字列の比較
    assertEquals("Hello", "Hell" + "o");
    
    // オブジェクトの比較（equalsメソッドが使用される）
    Person expected = new Person("田中", 30);
    Person actual = personService.findById(1);
    assertEquals(expected, actual);
}
assertEquals の引数順序

assertEquals(expected, actual) の順序で書きます：

  • 第1引数：期待値（expected）
  • 第2引数：実際の値（actual）

この順序を守ることで、エラーメッセージが理解しやすくなります。

2.2.2 assertTrue / assertFalse - boolean値のテスト

条件が真または偽であることを検証します：

@Test
void boolean値のテスト例() {
    String email = "test@example.com";
    
    // 条件が真であることを検証
    assertTrue(email.contains("@"));
    assertTrue(email.length() > 5);
    
    // 条件が偽であることを検証
    assertFalse(email.isEmpty());
    assertFalse(email.startsWith("invalid"));
}

2.2.3 assertNull / assertNotNull - null値のテスト

オブジェクトがnullかどうかを検証します：

@Test
void null値のテスト例() {
    // オブジェクトが存在することを確認
    User user = userService.findById(1);
    assertNotNull(user);
    
    // 存在しないユーザーはnullが返されることを確認
    User nonExistentUser = userService.findById(999);
    assertNull(nonExistentUser);
}

2.2.4 assertSame / assertNotSame - 参照の同一性テスト

オブジェクトの参照が同じかどうかを検証します：

@Test
void 参照の同一性テスト例() {
    // シングルトンパターンのテスト
    DatabaseConnection conn1 = DatabaseConnection.getInstance();
    DatabaseConnection conn2 = DatabaseConnection.getInstance();
    assertSame(conn1, conn2);  // 同じインスタンスを参照
    
    // 異なるインスタンスであることの確認
    List<String> list1 = new ArrayList<>();
    List<String> list2 = new ArrayList<>();
    assertNotSame(list1, list2);  // 異なるインスタンス
}
実習 2-1: 文字列操作クラスのテスト作成

文字列を操作するStringUtilsクラスを作成し、様々なアサーションを使ってテストしてみましょう。

手順1: テスト対象クラスの作成

StringUtils.javaを作成します：

public class StringUtils {
    
    public String reverse(String input) {
        if (input == null) return null;
        return new StringBuilder(input).reverse().toString();
    }
    
    public boolean isPalindrome(String input) {
        if (input == null) return false;
        String normalized = input.toLowerCase().replaceAll("\\s", "");
        return normalized.equals(reverse(normalized));
    }
    
    public String capitalize(String input) {
        if (input == null || input.isEmpty()) return input;
        return input.substring(0, 1).toUpperCase() + 
               input.substring(1).toLowerCase();
    }
    
    public int countWords(String input) {
        if (input == null || input.trim().isEmpty()) return 0;
        return input.trim().split("\\s+").length;
    }
}
手順2: テストクラスの作成

StringUtilsTest.javaを作成します：

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class StringUtilsTest {
    
    @Test
    void reverse文字列を反転するテスト() {
        StringUtils utils = new StringUtils();
        
        assertEquals("olleh", utils.reverse("hello"));
        assertEquals("4321", utils.reverse("1234"));
        assertEquals("", utils.reverse(""));
    }
    
    @Test
    void reverseでnullを渡した場合のテスト() {
        StringUtils utils = new StringUtils();
        
        assertNull(utils.reverse(null));
    }
    
    @Test
    void isPalindromeで回文判定のテスト() {
        StringUtils utils = new StringUtils();
        
        assertTrue(utils.isPalindrome("racecar"));
        assertTrue(utils.isPalindrome("A man a plan a canal Panama"));
        assertFalse(utils.isPalindrome("hello"));
        assertFalse(utils.isPalindrome(null));
    }
    
    @Test
    void capitalizeで先頭文字大文字化のテスト() {
        StringUtils utils = new StringUtils();
        
        assertEquals("Hello", utils.capitalize("hello"));
        assertEquals("Java", utils.capitalize("JAVA"));
        assertEquals("", utils.capitalize(""));
        assertNull(utils.capitalize(null));
    }
    
    @Test
    void countWordsで単語数カウントのテスト() {
        StringUtils utils = new StringUtils();
        
        assertEquals(3, utils.countWords("Hello World Java"));
        assertEquals(1, utils.countWords("Hello"));
        assertEquals(0, utils.countWords(""));
        assertEquals(0, utils.countWords("   "));
        assertEquals(0, utils.countWords(null));
    }
}
期待される結果

すべてのテストが成功することを確認してください。各テストが異なるアサーションメソッドを使用していることに注目してください。

2.3 意味のあるテスト名の付け方

テスト名は、そのテストが何を検証するのかを明確に表現する必要があります。日本語でも英語でも構いませんが、一貫性を保ちましょう。

2.3.1 良いテスト名の例

flowchart TD A["良いテスト名の条件"] --> B["何をテストするかが明確"] A --> C["どんな条件でテストするかが分かる"] A --> D["期待される結果が推測できる"] B --> E["addメソッドのテスト → ×"] B --> F["正の数の加算で正しい結果が返されること → ○"] C --> G["境界値での動作"] C --> H["異常系での動作"] D --> I["成功ケース"] D --> J["失敗ケース"] 
// 良いテスト名の例
@Test
void 正の整数2つを加算すると正しい合計が返される() { }

@Test
void ゼロで除算すると例外が発生する() { }

@Test
void 空の文字列を渡すと空の文字列が返される() { }

@Test
void nullを渡すとNullPointerExceptionが発生する() { }

// 英語の場合
@Test
void shouldReturnCorrectSumWhenAddingTwoPositiveIntegers() { }

@Test
void shouldThrowExceptionWhenDividingByZero() { }

// 悪いテスト名の例（避けるべき）
@Test
void test1() { }  // 何をテストするか不明

@Test
void addTest() { }  // 曖昧すぎる

@Test
void testCalculator() { }  // 具体的でない

2.3.2 テスト名の命名パターン

以下のようなパターンを使うと、一貫性のあるテスト名を作成できます：

  • [メソッド名]で[条件]の場合[期待される結果]
  • [機能]のテスト_[条件]_[結果]
  • should[期待される動作]When[条件]（英語の場合）

2.4 成功テストと失敗テストの作成

包括的なテストスイートには、正常系（成功ケース）と異常系（失敗ケース）の両方のテストが必要です。

2.4.1 正常系テスト（Happy Path）

期待される正常な動作を検証するテストです：

@Test
void 有効なメールアドレスで登録が成功する() {
    // Arrange
    UserService service = new UserService();
    String validEmail = "user@example.com";
    String password = "password123";
    
    // Act
    User user = service.register(validEmail, password);
    
    // Assert
    assertNotNull(user);
    assertEquals(validEmail, user.getEmail());
    assertTrue(user.isActive());
}

2.4.2 異常系テスト（Edge Cases）

エラー条件や境界値での動作を検証するテストです：

@Test
void 無効なメールアドレスで登録が失敗する() {
    UserService service = new UserService();
    String invalidEmail = "invalid-email";
    
    // assertThrowsについては第6章で詳しく学習します
    assertThrows(IllegalArgumentException.class, () -> {
        service.register(invalidEmail, "password123");
    });
}

@Test
void 空のパスワードで登録が失敗する() {
    UserService service = new UserService();
    
    assertThrows(IllegalArgumentException.class, () -> {
        service.register("user@example.com", "");
    });
}

2.4.3 境界値テスト

境界条件での動作を検証するテストです：

@Test
void 年齢制限の境界値テスト() {
    AgeValidator validator = new AgeValidator();
    
    // 境界値での検証
    assertTrue(validator.isValid(18));   // 最小値
    assertTrue(validator.isValid(65));   // 最大値
    assertFalse(validator.isValid(17));  // 最小値未満
    assertFalse(validator.isValid(66));  // 最大値超過
}
実習 2-2: バンキングシステムのテスト作成

銀行口座の基本的な機能（入金、出金、残高照会）をテストしてみましょう。

手順1: テスト対象クラスの作成

BankAccount.javaを作成します：

public class BankAccount {
    private double balance;
    private final String accountNumber;
    
    public BankAccount(String accountNumber, double initialBalance) {
        if (initialBalance < 0) {
            throw new IllegalArgumentException("初期残高は0以上である必要があります");
        }
        this.accountNumber = accountNumber;
        this.balance = initialBalance;
    }
    
    public void deposit(double amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("入金額は正の数である必要があります");
        }
        balance += amount;
    }
    
    public void withdraw(double amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("出金額は正の数である必要があります");
        }
        if (amount > balance) {
            throw new IllegalArgumentException("残高不足です");
        }
        balance -= amount;
    }
    
    public double getBalance() {
        return balance;
    }
    
    public String getAccountNumber() {
        return accountNumber;
    }
}
手順2: 包括的なテストの作成

正常系と異常系の両方をテストするBankAccountTest.javaを作成します：

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class BankAccountTest {
    
    // 正常系テスト
    @Test
    void 有効な初期残高で口座が作成できる() {
        BankAccount account = new BankAccount("12345", 1000.0);
        
        assertEquals("12345", account.getAccountNumber());
        assertEquals(1000.0, account.getBalance());
    }
    
    @Test
    void 正の金額で入金が成功する() {
        BankAccount account = new BankAccount("12345", 1000.0);
        
        account.deposit(500.0);
        
        assertEquals(1500.0, account.getBalance());
    }
    
    @Test
    void 残高の範囲内で出金が成功する() {
        BankAccount account = new BankAccount("12345", 1000.0);
        
        account.withdraw(300.0);
        
        assertEquals(700.0, account.getBalance());
    }
    
    // 異常系テスト
    @Test
    void 負の初期残高で口座作成が失敗する() {
        assertThrows(IllegalArgumentException.class, () -> {
            new BankAccount("12345", -100.0);
        });
    }
    
    @Test
    void 負の金額で入金が失敗する() {
        BankAccount account = new BankAccount("12345", 1000.0);
        
        assertThrows(IllegalArgumentException.class, () -> {
            account.deposit(-100.0);
        });
    }
    
    @Test
    void 残高を超える出金が失敗する() {
        BankAccount account = new BankAccount("12345", 1000.0);
        
        assertThrows(IllegalArgumentException.class, () -> {
            account.withdraw(1500.0);
        });
    }
    
    // 境界値テスト
    @Test
    void ゼロ初期残高で口座作成が成功する() {
        BankAccount account = new BankAccount("12345", 0.0);
        
        assertEquals(0.0, account.getBalance());
    }
    
    @Test
    void 残高ちょうどの金額で出金が成功する() {
        BankAccount account = new BankAccount("12345", 1000.0);
        
        account.withdraw(1000.0);
        
        assertEquals(0.0, account.getBalance());
    }
}
期待される結果

すべてのテストが成功し、正常系・異常系・境界値の各パターンが適切にカバーされていることを確認してください。

2.5 テストコードの可読性向上

テストコードは、本番コードと同様に保守しやすく、理解しやすい必要があります。

2.5.1 わかりやすい変数名の使用

// 良い例：意図が明確な変数名
@Test
void 消費税計算のテスト() {
    double basePrice = 1000.0;
    double taxRate = 0.10;
    double expectedTotalPrice = 1100.0;
    
    PriceCalculator calculator = new PriceCalculator();
    double actualTotalPrice = calculator.calculateTaxIncludedPrice(basePrice, taxRate);
    
    assertEquals(expectedTotalPrice, actualTotalPrice);
}

// 悪い例：意図が不明瞭
@Test
void test() {
    double a = 1000.0;
    double b = 0.10;
    double c = 1100.0;
    
    PriceCalculator calc = new PriceCalculator();
    double result = calc.calculateTaxIncludedPrice(a, b);
    
    assertEquals(c, result);
}

2.5.2 テストデータの明確な準備

@Test
void 商品検索のテスト() {
    // テストデータの準備を明確に分離
    Product laptop = new Product("ノートパソコン", 80000, "電子機器");
    Product mouse = new Product("マウス", 2000, "電子機器");
    Product book = new Product("Java入門書", 3000, "書籍");
    
    ProductService service = new ProductService();
    service.addProduct(laptop);
    service.addProduct(mouse);
    service.addProduct(book);
    
    // 実行
    List<Product> electronicsProducts = service.findByCategory("電子機器");
    
    // 検証
    assertEquals(2, electronicsProducts.size());
    assertTrue(electronicsProducts.contains(laptop));
    assertTrue(electronicsProducts.contains(mouse));
    assertFalse(electronicsProducts.contains(book));
}

2.5.3 エラーメッセージの追加

テストが失敗した時により詳細な情報を提供するため、カスタムメッセージを追加できます：

@Test
void 割引適用後の価格計算テスト() {
    double originalPrice = 1000.0;
    double discountRate = 0.20;
    
    PriceCalculator calculator = new PriceCalculator();
    double discountedPrice = calculator.applyDiscount(originalPrice, discountRate);
    
    assertEquals(800.0, discountedPrice, 
        "20%割引で1000円は800円になるはずです");
}
理解度確認クイズ
  1. AAAパターンの「Assert」段階では何を行いますか？
    • a) テストデータの準備
    • b) メソッドの実行
    • c) 期待値と実際値の比較
    • d) オブジェクトの初期化
  2. assertEquals(expected, actual)の引数順序で正しいのはどれですか？
    • a) assertEquals(actual, expected)
    • b) assertEquals(expected, actual)
    • c) どちらでも構わない
    • d) 型によって変わる
  3. null値をテストする際に使用するアサーションメソッドはどれですか？
    • a) assertEquals(null, object)
    • b) assertTrue(object == null)
    • c) assertNull(object)
    • d) すべて正しい
  4. 良いテスト名の条件として正しいものはどれですか？
    • a) 短くて覚えやすい
    • b) test1, test2のように連番を使う
    • c) 何をテストするかが明確
    • d) 英語でなければならない
解答を見る

1. c) 期待値と実際値の比較

2. b) assertEquals(expected, actual)

3. c) assertNull(object) が最も適切（他も動作はします）

4. c) 何をテストするかが明確
