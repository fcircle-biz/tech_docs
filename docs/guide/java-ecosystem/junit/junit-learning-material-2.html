2.1 テストメソッドの基本構造

JUnit 5でのテストメソッドは、@Testアノテーションを付けたpublicメソッドとして定義します。

テストメソッドの要件

  • @Testアノテーション: メソッドがテストであることを示す
  • voidメソッド: 戻り値を持たない
  • 引数なし: パラメータを取らない（パラメータ化テスト除く）
  • アクセス修飾子: public、protected、package-privateのいずれか
実習 2-1: 基本的なテストメソッドの作成

String操作を行うStringUtilsクラスのテストを作成します。

手順
  1. StringUtilsクラスを作成
  2. StringUtilsTestクラスを作成
  3. 各メソッドのテストを実装
実行例

StringUtils.java

public class StringUtils {
    
    public static boolean isEmpty(String str) {
        return str == null || str.length() == 0;
    }
    
    public static boolean isNotEmpty(String str) {
        return !isEmpty(str);
    }
    
    public static String reverse(String str) {
        if (isEmpty(str)) {
            return str;
        }
        return new StringBuilder(str).reverse().toString();
    }
    
    public static String capitalize(String str) {
        if (isEmpty(str)) {
            return str;
        }
        return str.substring(0, 1).toUpperCase() + str.substring(1).toLowerCase();
    }
}

StringUtilsTest.java

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class StringUtilsTest {
    
    @Test
    void testIsEmpty() {
        assertTrue(StringUtils.isEmpty(null));
        assertTrue(StringUtils.isEmpty(""));
        assertFalse(StringUtils.isEmpty("hello"));
        assertFalse(StringUtils.isEmpty(" "));
    }
    
    @Test
    void testIsNotEmpty() {
        assertFalse(StringUtils.isNotEmpty(null));
        assertFalse(StringUtils.isNotEmpty(""));
        assertTrue(StringUtils.isNotEmpty("hello"));
        assertTrue(StringUtils.isNotEmpty(" "));
    }
}
期待される結果

すべてのテストが成功し、StringUtilsの基本機能が正しく動作することが確認できます。

2.2 基本的なアサーションメソッド

JUnit 5では、Assertionsクラスに様々なアサーションメソッドが用意されています。

主要なアサーションメソッド

メソッド 用途 例
assertEquals(expected, actual) 値の等価性確認 assertEquals(5, result)
assertTrue(condition) 条件がtrueか確認 assertTrue(list.isEmpty())
assertFalse(condition) 条件がfalseか確認 assertFalse(str.isEmpty())
assertNull(actual) 値がnullか確認 assertNull(object)
assertNotNull(actual) 値がnullでないか確認 assertNotNull(result)
実習 2-2: 様々なアサーションの使用

数値計算を行うMathUtilsクラスのテストで、様々なアサーションを使用します。

手順
  1. MathUtilsクラスを作成
  2. 各メソッドに対応するテストメソッドを作成
  3. 適切なアサーションメソッドを選択
実行例

MathUtils.java

public class MathUtils {
    
    public static int factorial(int n) {
        if (n < 0) {
            throw new IllegalArgumentException("Negative number");
        }
        if (n <= 1) {
            return 1;
        }
        return n * factorial(n - 1);
    }
    
    public static boolean isPrime(int n) {
        if (n <= 1) {
            return false;
        }
        for (int i = 2; i <= Math.sqrt(n); i++) {
            if (n % i == 0) {
                return false;
            }
        }
        return true;
    }
    
    public static Integer findMax(int[] numbers) {
        if (numbers == null || numbers.length == 0) {
            return null;
        }
        int max = numbers[0];
        for (int num : numbers) {
            if (num > max) {
                max = num;
            }
        }
        return max;
    }
}

MathUtilsTest.java

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class MathUtilsTest {
    
    @Test
    void testFactorial() {
        assertEquals(1, MathUtils.factorial(0));
        assertEquals(1, MathUtils.factorial(1));
        assertEquals(2, MathUtils.factorial(2));
        assertEquals(6, MathUtils.factorial(3));
        assertEquals(120, MathUtils.factorial(5));
    }
    
    @Test
    void testIsPrime() {
        assertFalse(MathUtils.isPrime(1));
        assertTrue(MathUtils.isPrime(2));
        assertTrue(MathUtils.isPrime(3));
        assertFalse(MathUtils.isPrime(4));
        assertTrue(MathUtils.isPrime(5));
        assertFalse(MathUtils.isPrime(9));
        assertTrue(MathUtils.isPrime(11));
    }
    
    @Test
    void testFindMax() {
        assertEquals(5, MathUtils.findMax(new int[]{1, 3, 5, 2}));
        assertEquals(10, MathUtils.findMax(new int[]{10}));
        assertEquals(-1, MathUtils.findMax(new int[]{-5, -3, -1}));
        assertNull(MathUtils.findMax(new int[]{}));
        assertNull(MathUtils.findMax(null));
    }
}
期待される結果

各テストが適切なアサーションで検証され、期待される結果が得られます。

2.3 テストの失敗と成功

テストの成功と失敗のメカニズムを理解することは重要です。

テストが失敗する条件

  • アサーションの失敗: 期待値と実際の値が一致しない
  • 例外の発生: 予期しない例外がスローされる
  • タイムアウト: 実行時間が制限を超える
実習 2-3: 意図的な失敗テストの作成

失敗するテストを作成し、エラーメッセージを確認します。

手順
  1. 意図的に失敗するテストを作成
  2. テストを実行してエラーメッセージを確認
  3. テストを修正して成功させる
実行例
@Test
void testFailingAssertion() {
    // 意図的に失敗するテスト
    assertEquals(10, 5 + 3, "5 + 3 should equal 10");
}

@Test
void testFailingAssertionFixed() {
    // 修正されたテスト
    assertEquals(8, 5 + 3, "5 + 3 should equal 8");
}

@Test
void testCustomErrorMessage() {
    String result = "Hello";
    assertEquals("Hello World", result, 
        () -> "Expected 'Hello World' but was '" + result + "'");
}
期待される結果

最初のテストは失敗し、詳細なエラーメッセージが表示されます。修正後のテストは成功します。

2.4 テストメソッドの命名規則

読みやすく保守しやすいテストコードのための命名規則を紹介します。

一般的な命名パターン

  • test + メソッド名: testCalculateTotal()
  • should + 期待動作 + when + 条件: shouldReturnZeroWhenEmptyList()
  • given + 前提条件 + when + 動作 + then + 結果: givenEmptyString_whenCheckLength_thenReturnZero()
  • メソッド名 + 条件 + 期待結果: calculateTotal_WithValidItems_ReturnsCorrectSum()
命名のベストプラクティス
  • テストの目的が明確に分かる名前を付ける
  • 日本語での説明も@DisplayNameアノテーションで可能
  • 一つのテストは一つの側面だけを検証する
実習 2-4: 適切な命名規則の適用

@DisplayNameアノテーションを使用して、テストの内容を明確に示します。

手順
  1. 既存のテストに@DisplayNameアノテーションを追加
  2. テストの目的を明確に表現する日本語名を付ける
  3. テスト実行結果での表示を確認
実行例
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

@DisplayName("電卓機能のテスト")
class CalculatorTest {
    
    @Test
    @DisplayName("正の数の加算が正しく動作すること")
    void shouldAddPositiveNumbersCorrectly() {
        Calculator calculator = new Calculator();
        assertEquals(8, calculator.add(3, 5));
    }
    
    @Test
    @DisplayName("ゼロとの加算が正しく動作すること")
    void shouldHandleZeroAdditionCorrectly() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(5, 0));
        assertEquals(5, calculator.add(0, 5));
    }
    
    @Test
    @DisplayName("負の数の加算が正しく動作すること")
    void shouldAddNegativeNumbersCorrectly() {
        Calculator calculator = new Calculator();
        assertEquals(-8, calculator.add(-3, -5));
        assertEquals(2, calculator.add(5, -3));
    }
}
期待される結果

テスト実行時に日本語の説明が表示され、どのテストが何を検証しているかが明確になります。

2.5 テスト結果の読み方

テスト実行結果を正しく解釈することは、効率的なデバッグのために重要です。

成功時の表示

  • 緑色のチェックマーク ✓
  • 実行時間の表示
  • 「All tests passed」メッセージ

失敗時の表示

  • 赤色のXマーク ✗
  • 失敗したテストメソッド名
  • 期待値と実際の値の比較
  • スタックトレース情報
理解度確認クイズ
  1. @Testアノテーションを付けたメソッドの要件を3つ挙げてください。
  2. assertEquals(expected, actual)の第1引数と第2引数には何を指定しますか？
  3. assertTrue()とassertEquals(true, actual)の違いは何ですか？
  4. テストメソッドの適切な命名規則を2つ挙げてください。
  5. @DisplayNameアノテーションの利点は何ですか？
  6. assertNull()とassertEquals(null, actual)のどちらが推奨されますか？その理由も答えてください。
解答例
  1. 1) @Testアノテーションを付ける、2) voidメソッドである、3) 引数を取らない
  2. 第1引数: 期待値、第2引数: 実際の値
  3. assertTrue()はより意味が明確で、失敗時のメッセージも分かりやすい
  4. 1) test + メソッド名、2) should + 期待動作 + when + 条件
  5. テストの目的を日本語で明確に表現でき、テスト結果が読みやすくなる
  6. assertNull()が推奨。より意味が明確で、専用のエラーメッセージが提供される
