第1章: JUnit基礎とセットアップ

JUnitの基礎とプロジェクトセットアップ

この章で学ぶこと
  • JUnitの概要と役割
  • JUnit 5の特徴とアーキテクチャ
  • MavenとGradleでのJUnit導入方法
  • IDEでのJUnitテスト実行環境の構築
  • 初回テストケースの作成と実行

1.1 JUnitとは

JUnitは、Java言語用の単体テストフレームワークです。コードの品質を確保し、リファクタリングを安全に行うために不可欠なツールです。

JUnitの主な役割

  • 単体テストの自動化: メソッドやクラス単位での動作検証
  • 継続的な品質保証: コード変更時の回帰テスト
  • ドキュメント機能: テストケースによる仕様の明確化
  • 開発効率の向上: バグの早期発見と修正

1.2 JUnit 5の特徴

JUnit 5は、JUnit 4から大幅に刷新されたバージョンで、モジュラーアーキテクチャを採用しています。

JUnit 5のアーキテクチャ

  • JUnit Platform: テスト実行の基盤となるランチャー
  • JUnit Jupiter: JUnit 5のテスト記述とエンジン
  • JUnit Vintage: JUnit 3/4との互換性を提供
注意事項

JUnit 5を使用する場合、Java 8以上が必要です。また、アノテーションの名前空間が変更されているため、JUnit 4からの移行時は注意が必要です。

1.3 Mavenプロジェクトでのセットアップ

MavenプロジェクトでJUnit 5を使用する場合の設定方法を説明します。

実習 1-1: Maven依存関係の設定

pom.xmlファイルにJUnit 5の依存関係を追加します。

手順
  1. プロジェクトのpom.xmlファイルを開く
  2. <dependencies>セクションに以下を追加
実行例
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
期待される結果

Maven依存関係が正しく設定され、JUnit 5のライブラリがプロジェクトで利用可能になります。

1.4 Gradleプロジェクトでのセットアップ

Gradleプロジェクトでの設定方法も確認しましょう。

実習 1-2: Gradle依存関係の設定

build.gradleファイルにJUnit 5の依存関係を追加します。

手順
  1. build.gradleファイルを開く
  2. dependenciesブロックに以下を追加
  3. testタスクの設定を追加
実行例
dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}

test {
    useJUnitPlatform()
}
期待される結果

Gradle設定が完了し、./gradlew testコマンドでJUnit 5テストを実行できます。

1.5 初回テストケースの作成

簡単な電卓クラスを作成し、そのテストケースを実装してみましょう。

実習 1-3: 電卓クラスとテストの作成

基本的な計算機能を持つクラスとそのテストを作成します。

手順
  1. src/main/javaにCalculatorクラスを作成
  2. src/test/javaにCalculatorTestクラスを作成
  3. テストメソッドを実装
  4. テストを実行
実行例

Calculator.java

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    
    public int subtract(int a, int b) {
        return a - b;
    }
    
    public int multiply(int a, int b) {
        return a * b;
    }
    
    public double divide(int a, int b) {
        if (b == 0) {
            throw new IllegalArgumentException("Division by zero");
        }
        return (double) a / b;
    }
}

CalculatorTest.java

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    
    @Test
    void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
    
    @Test
    void testSubtract() {
        Calculator calculator = new Calculator();
        int result = calculator.subtract(5, 3);
        assertEquals(2, result);
    }
}
期待される結果

テストが成功し、電卓の基本機能が正しく動作することが確認できます。IDEまたはコマンドラインでテスト実行結果が表示されます。

1.6 IDEでのテスト実行

主要なIDEでのJUnitテスト実行方法を確認します。

IntelliJ IDEAでの実行

  • テストクラス上で右クリック → "Run 'CalculatorTest'"
  • 個別メソッドの実行: メソッド横の緑の矢印をクリック
  • キーボードショートカット: Ctrl+Shift+F10 (Windows/Linux)

Eclipseでの実行

  • テストクラス上で右クリック → "Run As" → "JUnit Test"
  • JUnitビューでテスト結果を確認

VS Codeでの実行

  • Java Test Runner拡張機能をインストール
  • テストエクスプローラーからテストを実行
理解度確認クイズ
  1. JUnit 5の3つのサブプロジェクト（Platform、Jupiter、Vintage）の役割をそれぞれ説明してください。
  2. Mavenプロジェクトでjunit-jupiterの依存関係を追加する際、どのスコープを指定すべきでしょうか？
  3. @Testアノテーションの役割は何ですか？
  4. assertEquals()メソッドの第1引数と第2引数には何を指定しますか？
  5. JUnit 5を使用するために必要な最小のJavaバージョンは何ですか？
解答例
  1. Platform: テスト実行基盤、Jupiter: JUnit 5のテスト記述とエンジン、Vintage: JUnit 3/4互換性
  2. testスコープ（本番コードには不要なため）
  3. そのメソッドがテストメソッドであることを示すマーカー
  4. 第1引数: 期待値、第2引数: 実際の値
  5. Java 8以上
