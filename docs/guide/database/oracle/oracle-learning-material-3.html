第3章: SQL Developerの導入と基本操作

3.1 SQL Developerの概要

Oracle SQL Developerは、Oracleが提供する無料のグラフィカルデータベース開発ツールです。 SQL文の実行、データベースオブジェクトの管理、データの表示・編集など、 Oracleデータベースでの作業に必要な機能が統合されています。

3.2 SQL Developerの主要機能

  • SQLワークシート: SQL文の作成・実行・結果表示
  • オブジェクトブラウザ: テーブル、ビュー、プロシージャなどの管理
  • データベース管理: ユーザー、権限、ストレージの管理
  • インポート/エクスポート: データの取り込み・出力
  • PL/SQLデバッガ: プロシージャ・ファンクションのデバッグ
  • レポート機能: カスタムレポートの作成

3.3 SQL Developerのダウンロードとインストール

実習 3-1: SQL Developerのダウンロード
  1. Oracle公式サイトにアクセス： SQL Developer ダウンロードページ
  2. 「Oracle SQL Developer」を選択
  3. 使用するOSに応じたファイルをダウンロード：
    • Windows: sqldeveloper-xxx-windows.x64.zip
    • macOS: sqldeveloper-xxx-macosx.app.zip
    • Linux: sqldeveloper-xxx-no-jre.zip
  4. Oracleアカウントでログイン（無料登録可能）
  5. ライセンス契約に同意してダウンロード

3.4 SQL Developerの起動と初期設定

実習 3-2: SQL Developerの起動
Windows の場合：
  1. ダウンロードしたzipファイルを解凍
  2. sqldeveloper フォルダ内の sqldeveloper.exe を実行
  3. 初回起動時にJavaパスの設定が求められる場合があります
macOS の場合：
  1. ダウンロードしたzipファイルを解凍
  2. SQLDeveloper.app をアプリケーションフォルダにドラッグ
  3. Launchpadまたはアプリケーションフォルダから起動
Linux の場合：
  1. 事前にJava JDK 8以上をインストール
  2. 解凍したフォルダ内で ./sqldeveloper.sh を実行

3.5 データベース接続の設定

SQL DeveloperからDockerで起動したOracleデータベースに接続してみましょう：

実習 3-3: データベース接続の作成
  1. SQL Developerを起動
  2. 左側の「接続」パネルで右クリック → 「新規接続」
  3. 接続情報を入力：
    • 接続名: Oracle XE Local
    • ユーザー名: sys
    • パスワード: Password123
    • パスワードの保存: チェック（任意）
    • ホスト名: localhost
    • ポート: 1521
    • SID: XE
    • ロール: SYSDBA
  4. 「テスト」ボタンをクリックして接続確認
  5. 「成功」メッセージを確認後、「接続」をクリック
接続エラーの対処法
  • TNS-12541: Dockerコンテナが起動しているか確認（docker ps）
  • ORA-01017: ユーザー名・パスワードを再確認
  • ORA-28009: ロールがSYSDBAに設定されているか確認
  • タイムアウト: ホスト名・ポート番号を確認
  • 接続失敗: SIDがXEに設定されているか確認
接続確認コマンド

コンテナ内でのサービス名確認：

docker exec -it oracle-xe sqlplus sys/Password123@localhost:1521/XE as sysdba SQL> SELECT name FROM v$database; SQL> SHOW PARAMETER service_names;

3.6 SQLワークシートの基本操作

SQLワークシートでSQL文を実行してみましょう：

実習 3-4: SQLワークシートの基本操作
  1. 接続が成功すると自動的にSQLワークシートが開きます
  2. 以下のSQL文を入力してください：
-- 現在のユーザーを確認 SELECT USER FROM DUAL; -- 現在の日時を確認 SELECT SYSDATE FROM DUAL; -- データベースバージョンを確認 SELECT * FROM v$version;
  1. F5キーまたは「実行」ボタン（緑の再生アイコン）をクリック
  2. 「スクリプト出力」タブで結果を確認
  3. 単一のSQL文を実行するには、カーソルを文の上に置いてCtrl+Enter

3.7 オブジェクトブラウザの使い方

オブジェクトブラウザでデータベースオブジェクトを確認しましょう：

実習 3-5: オブジェクトブラウザの操作
  1. 左側の接続ツリーを展開
  2. 「Oracle XE Local」→「テーブル」を展開
  3. システムテーブルを確認（例：DUAL）
  4. テーブルをダブルクリックして詳細表示
  5. 「データ」タブでテーブル内容を確認
  6. 「SQL」タブでCREATE文を確認

3.8 テーブルの作成と管理

GUI操作でテーブルを作成してみましょう：

実習 3-6: GUIでのテーブル作成
  1. 「テーブル」を右クリック → 「新規テーブル」
  2. テーブル情報を入力：
    • スキーマ: LEARNER
    • 名前: EMPLOYEES
  3. 「列」タブで列を追加：
    EMP_IDNUMBERNOT NULL
    FIRST_NAMEVARCHAR2(50)
    LAST_NAMEVARCHAR2(50)
    EMAILVARCHAR2(100)
    HIRE_DATEDATE
    SALARYNUMBER(10,2)
  4. 「主キー」タブでEMP_IDを主キーに設定
  5. 「OK」をクリックしてテーブル作成

3.9 データの挿入と表示

実習 3-7: データの挿入

SQLワークシートで以下のコードを実行：

-- サンプルデータの挿入 INSERT INTO EMPLOYEES VALUES (1, '太郎', '田中', 'taro.tanaka@example.com', DATE '2023-01-15', 350000); INSERT INTO EMPLOYEES VALUES (2, '花子', '佐藤', 'hanako.sato@example.com', DATE '2023-02-01', 380000); INSERT INTO EMPLOYEES VALUES (3, '次郎', '鈴木', 'jiro.suzuki@example.com', DATE '2023-03-10', 320000); -- 変更をコミット COMMIT; -- データの確認 SELECT * FROM EMPLOYEES;

3.10 便利な機能とショートカット

機能 ショートカット 説明
SQL実行 Ctrl+Enter カーソル位置のSQL文を実行
スクリプト実行 F5 全てのSQL文を実行
オートコンプリート Ctrl+Space テーブル名・列名の自動補完
コメント Ctrl+/ 選択行をコメントアウト
フォーマット Ctrl+F7 SQLコードの整形
検索・置換 Ctrl+H テキストの検索・置換

3.11 設定のカスタマイズ

SQL Developerをより使いやすくカスタマイズしましょう：

  • フォント設定: ツール → 設定 → コードエディタ → フォント
  • カラーテーマ: ツール → 設定 → 表示 → 配色
  • 自動保存: ツール → 設定 → データベース → ワークシート
  • 履歴件数: ツール → 設定 → データベース → SQLおよびPL/SQL
