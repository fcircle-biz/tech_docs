1.1 Reactとは何か

React（リアクト）は、Facebook（現Meta）が開発したJavaScriptライブラリで、ユーザーインターフェース（UI）を構築するために設計されています。2013年にオープンソースとして公開されて以来、世界中で最も人気のあるフロントエンド技術の一つとなっています。

なぜReactが人気なのでしょうか？

Reactの人気の理由は、その設計哲学と実用性にあります。Reactは「UI構築のための効率的で柔軟な方法」を提供し、複雑なWebアプリケーションを簡単に開発できるようにします。Facebook、Instagram、Netflix、Airbnbなど、世界中の大規模サービスで使用されています。

Reactの主要な特徴

  • コンポーネントベース：UIを再利用可能な部品（コンポーネント）に分割して構築できる
  • 宣言的UI：アプリケーションの各状態に対してUIがどう見えるかを記述するだけで、Reactが効率的に更新する
  • 仮想DOM：高速なUI更新のために、仮想的なDOMツリーを使用して効率的にレンダリングする
  • 単方向データフロー：データの流れが一方向なので、アプリケーションの動作を理解しやすい
  • 豊富なエコシステム：React Routerやstyled-componentsなど、多数のライブラリと統合できる

1.2 仮想DOMの概念

Reactの最も重要な特徴の一つが、仮想DOM（Virtual DOM）です。この概念を理解することで、なぜReactが高速に動作するのかがわかります。

従来のDOMの問題点

通常のWebページでは、JavaScriptがブラウザのDOM（Document Object Model）を直接操作します。しかし、DOMの操作は比較的コストが高く、特に大量の変更がある場合、ページ全体の再レンダリングが必要になり、パフォーマンスが低下します。

仮想DOMによる解決

Reactは、実際のDOMの「コピー」である仮想DOMをメモリ内に保持します。何か変更があったとき、Reactはまず仮想DOMを更新し、その後、変更が必要な部分だけを効率的に実際のDOMに反映します。

flowchart TB A["状態（State）の変更"] --> B["新しい仮想DOMツリーを作成"] B --> C["前の仮想DOMと比較
（差分検出）"] C --> D["変更された部分のみを特定"] D --> E["実際のDOMを効率的に更新"]
重要なポイント

仮想DOMのおかげで、開発者は「現在の状態でUIがどう見えるべきか」だけを考えればよく、「どのようにDOMを更新するか」という複雑な部分はReactが自動的に処理してくれます。これにより、コードがシンプルになり、パフォーマンスも向上します。

1.3 開発環境のセットアップ

Reactアプリケーションを開発するためには、Node.jsとnpm（またはyarn）が必要です。Node.jsはJavaScript実行環境であり、npmはJavaScriptパッケージマネージャーです。

Node.jsとnpmのインストール

Node.jsをインストールすると、npmも自動的にインストールされます。

実習 1-1: Node.jsの確認とインストール

まず、コンピュータにNode.jsがインストールされているかを確認し、必要に応じてインストールを行います。

手順
  1. コマンドプロンプト（Windows）またはターミナル（Mac/Linux）を開く
  2. 以下のコマンドを入力してNode.jsとnpmのバージョンを確認する
バージョン確認コマンド
# Node.jsのバージョン確認
node -v

# npmのバージョン確認
npm -v
期待される結果

正常にインストールされている場合、以下のような出力が表示されます：

v18.17.0
9.6.7
コマンドが認識されない場合

エラーが出る場合は、Node.js公式サイト（https://nodejs.org/）からLTS（Long Term Support）版をダウンロードしてインストールしてください。インストール後は、ターミナルを再起動してください。

1.4 Create React Appでプロジェクト作成

Create React App（CRA）は、Reactアプリケーションの開発環境を簡単にセットアップできる公式ツールです。Webpack、Babel、ESLintなどの設定が事前に組み込まれているため、初心者でもすぐに開発を始められます。

Create React Appの利点

  • 複雑な設定不要で、すぐに開発を始められる
  • モダンなJavaScript（ES6+）を使用できる
  • ホットリロード機能により、コード変更が即座に反映される
  • 本番環境用の最適化されたビルドが簡単に作成できる
実習 1-2: Create React Appでプロジェクト作成

Create React Appを使用して、最初のReactプロジェクトを作成します。

手順
  1. ターミナルで作業用ディレクトリに移動する
  2. 以下のコマンドを実行してプロジェクトを作成する
プロジェクト作成コマンド
# Create React Appを使用してプロジェクト作成
npx create-react-app my-first-app

# プロジェクトディレクトリに移動
cd my-first-app

# 開発サーバーを起動
npm start
期待される結果

開発サーバーが起動し、ブラウザが自動的に開いて http://localhost:3000 にアクセスします。Reactのロゴが回転する初期画面が表示されます。

補足：npxとは？

npxは、npmパッケージを一時的にダウンロードして実行するツールです。ローカルにインストールすることなく、最新版のツールを使用できます。

1.5 Viteを使用したプロジェクト作成（代替方法）

Vite（ヴィート）は、高速な開発サーバーとビルドツールを提供する新しいフロントエンド開発ツールです。Create React Appよりも起動が速く、最近人気が高まっています。

Viteの利点

  • 開発サーバーの起動が非常に高速
  • ホットモジュールリプレースメント（HMR）が高速
  • モダンなブラウザのネイティブESモジュールを活用
  • 本番ビルドが高速で最適化されている
実習 1-3: Viteでプロジェクト作成

Viteを使用してReactプロジェクトを作成します。

手順
  1. ターミナルで以下のコマンドを実行する
  2. フレームワークとして「React」を選択する
  3. バリアント（言語）として「JavaScript」を選択する
Viteプロジェクト作成コマンド
# Viteを使用してプロジェクト作成
npm create vite@latest my-vite-app

# プロジェクトディレクトリに移動
cd my-vite-app

# 依存関係をインストール
npm install

# 開発サーバーを起動
npm run dev
期待される結果

開発サーバーが起動し、http://localhost:5173 にアクセスすると、Viteのロゴとシンプルなカウンターアプリが表示されます。

1.6 プロジェクト構造の理解

Create React AppまたはViteで作成したプロジェクトには、多くのファイルとフォルダが含まれています。それぞれの役割を理解することが重要です。

Create React Appのプロジェクト構造

my-first-app/
├── node_modules/      # インストールされたパッケージ
├── public/            # 静的ファイル（画像、favicon等）
│   ├── index.html     # HTMLテンプレート
│   └── ...
├── src/               # ソースコード
│   ├── App.js         # メインアプリケーションコンポーネント
│   ├── App.css        # アプリケーションのスタイル
│   ├── index.js       # エントリーポイント
│   └── ...
├── package.json       # プロジェクト情報と依存関係
└── README.md          # プロジェクト説明

重要なファイルの説明

  • package.json：プロジェクトの情報、使用するライブラリ、実行可能なスクリプトが記載されている
  • public/index.html：アプリケーションのHTMLテンプレート。Reactコンポーネントがこのファイルの中に挿入される
  • src/index.js：アプリケーションのエントリーポイント。Reactをブラウザに接続する
  • src/App.js：メインのReactコンポーネント。ここからアプリケーションの開発を始める
  • node_modules/：プロジェクトで使用するすべての外部ライブラリが格納される（Gitには含めない）
重要なポイント

開発中に主に編集するのはsrc/フォルダ内のファイルです。public/index.html は通常編集する必要はありません。

1.7 React Developer Toolsの導入

React Developer Tools は、Reactアプリケーションのデバッグを支援するブラウザ拡張機能です。コンポーネントツリーの確認、PropsとStateの検査、パフォーマンスプロファイリングなどができます。

React Developer Toolsの機能

  • コンポーネントツリーの階層構造を視覚化
  • 各コンポーネントのPropsとStateをリアルタイムで確認
  • コンポーネントの再レンダリングを追跡
  • パフォーマンスの問題を特定
実習 1-4: React Developer Toolsのインストール

ブラウザにReact Developer Toolsをインストールします。

手順
  1. 使用しているブラウザの拡張機能ストアにアクセスする
    • Chrome: Chrome Web Store
    • Firefox: Firefox Add-ons
    • Edge: Microsoft Edge Add-ons
  2. 「React Developer Tools」を検索してインストールする
  3. Reactアプリケーションを開いた状態でブラウザの開発者ツール（F12）を開く
  4. 「Components」と「Profiler」というタブが追加されていることを確認する
期待される結果

開発者ツールに「Components」タブが表示され、Reactコンポーネントの階層構造を確認できるようになります。

理解度確認クイズ
  1. Reactの主な利点を3つ挙げてください。
  2. 仮想DOMとは何ですか？なぜReactは仮想DOMを使用するのですか？
  3. Create React Appの役割は何ですか？
  4. package.jsonファイルの目的は何ですか？
  5. src/index.jsとsrc/App.jsの違いは何ですか？
  6. React Developer Toolsで何ができますか？
解答例
  1. コンポーネントベースのアーキテクチャ、仮想DOMによる高速なレンダリング、豊富なエコシステム
  2. 仮想DOMは実際のDOMのコピーをメモリ内に保持し、変更を効率的に検出して必要な部分だけを更新することで、パフォーマンスを向上させる
  3. 複雑な設定不要でReactアプリケーションの開発環境をすぐにセットアップできるツール
  4. プロジェクトのメタデータ、依存関係、実行可能なスクリプトなどを定義する
  5. index.jsはアプリケーションのエントリーポイントでReactをブラウザに接続する。App.jsはメインのコンポーネントで実際のUIを定義する
  6. コンポーネントツリーの確認、PropsとStateの検査、パフォーマンスプロファイリングなど
