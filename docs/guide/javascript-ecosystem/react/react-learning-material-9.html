9.1 シングルページアプリケーション（SPA）とは

SPA（Single Page Application）は、ページ全体をリロードせず、必要な部分だけを動的に更新するWebアプリケーションです。React Routerを使用することで、URLとコンポーネントを対応付け、SPAのナビゲーションを実現します。

従来のWebサイト vs SPA

  • 従来のWebサイト：ページ遷移のたびにサーバーからHTML全体を取得し、ブラウザがリロード
  • SPA：初回のみHTML/CSS/JSをロード。以降はJavaScriptがDOMを更新し、必要なデータのみをAPIから取得

9.2 React Routerのインストールと設定

実習 9-1: React Routerの基本設定
インストール
npm install react-router-dom
基本的なルーティング設定
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

// ページコンポーネント
function Home() {
  return <h1>ホームページ</h1>;
}

function About() {
  return <h1>Aboutページ</h1>;
}

function Contact() {
  return <h1>お問い合わせ</h1>;
}

// メインアプリ
function App() {
  return (
    <BrowserRouter>
      {/* ナビゲーション */}
      <nav>
        <Link to="/">ホーム</Link>
        <Link to="/about">About</Link>
        <Link to="/contact">お問い合わせ</Link>
      </nav>

      {/* ルート定義 */}
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/contact" element={<Contact />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;

9.3 動的ルーティング（URLパラメータ）

URLパラメータを使用して、動的なページを作成できます。例えば、ユーザーのIDを受け取って該当ユーザーの詳細を表示します。

実習 9-2: 動的ルートの実装
import { BrowserRouter, Routes, Route, Link, useParams } from 'react-router-dom';
import { useState, useEffect } from 'react';

// ユーザー一覧ページ
function UserList() {
  const users = [
    { id: 1, name: "山田太郎" },
    { id: 2, name: "佐藤花子" },
    { id: 3, name: "鈴木一郎" }
  ];

  return (
    <div>
      <h2>ユーザー一覧</h2>
      <ul>
        {users.map(user => (
          <li key={user.id}>
            <Link to={`/users/${user.id}`}>{user.name}</Link>
          </li>
        ))}
      </ul>
    </div>
  );
}

// ユーザー詳細ページ
function UserDetail() {
  const { id } = useParams(); // URLパラメータを取得
  const [user, setUser] = useState(null);

  useEffect(() => {
    // 実際はAPIから取得
    const mockUser = {
      1: { id: 1, name: "山田太郎", email: "yamada@example.com" },
      2: { id: 2, name: "佐藤花子", email: "sato@example.com" },
      3: { id: 3, name: "鈴木一郎", email: "suzuki@example.com" }
    };
    setUser(mockUser[id]);
  }, [id]);

  if (!user) return <p>Loading...</p>;

  return (
    <div>
      <h2>{user.name}の詳細</h2>
      <p>ID: {user.id}</p>
      <p>メール: {user.email}</p>
      <Link to="/users">一覧に戻る</Link>
    </div>
  );
}

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/users" element={<UserList />} />
        <Route path="/users/:id" element={<UserDetail />} />
      </Routes>
    </BrowserRouter>
  );
}

9.4 プログラマティックナビゲーション

useNavigateフックを使用して、JavaScriptのコードからページ遷移を実行できます。

import { useNavigate } from 'react-router-dom';

function LoginForm() {
  const navigate = useNavigate();

  const handleLogin = (e) => {
    e.preventDefault();
    // ログイン処理
    console.log("ログイン成功");

    // ダッシュボードへ遷移
    navigate('/dashboard');

    // 戻るボタンで戻れないようにする場合
    // navigate('/dashboard', { replace: true });
  };

  return (
    <form onSubmit={handleLogin}>
      <button type="submit">ログイン</button>
    </form>
  );
}

9.5 404ページとリダイレクト

実習 9-3: 404ページの実装
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';

function NotFound() {
  return (
    <div>
      <h1>404 - ページが見つかりません</h1>
      <Link to="/">ホームに戻る</Link>
    </div>
  );
}

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />

        {/* 旧URLから新URLへリダイレクト */}
        <Route path="/old-page" element={<Navigate to="/new-page" replace />} />

        {/* 404ページ（一致しないすべてのパス） */}
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}

9.6 ネストされたルート

ルートを入れ子にすることで、レイアウトの共有や階層構造を表現できます。

import { BrowserRouter, Routes, Route, Outlet, Link } from 'react-router-dom';

// レイアウトコンポーネント
function DashboardLayout() {
  return (
    <div>
      <h1>ダッシュボード</h1>
      <nav>
        <Link to="/dashboard/profile">プロフィール</Link>
        <Link to="/dashboard/settings">設定</Link>
      </nav>
      <hr />
      <Outlet />  {/* 子ルートがここに表示される */}
    </div>
  );
}

function Profile() {
  return <h2>プロフィールページ</h2>;
}

function Settings() {
  return <h2>設定ページ</h2>;
}

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/dashboard" element={<DashboardLayout />}>
          <Route path="profile" element={<Profile />} />
          <Route path="settings" element={<Settings />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}

  2. LinkコンポーネントとHTMLの<a>タグの違いは何ですか？
  3. URLパラメータを取得するために使用するフックは何ですか？
  4. プログラマティックナビゲーションを実現するフックは何ですか？
  5. 404ページを実装するためのパスは何ですか？
解答例
  1. ページ全体をリロードせず、必要な部分だけを動的に更新するWebアプリケーション
  2. Linkコンポーネントはページリロードせずにナビゲート。<a>タグはページ全体がリロードされる
  3. useParams()
  4. useNavigate()
  5. path="*" （ワイルドカード）
