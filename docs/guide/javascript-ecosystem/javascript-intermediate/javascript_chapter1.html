第1章：モダン開発環境構築

プロフェッショナルなJavaScript開発を行うために、現代の開発現場で標準的に使用されているツールチェーンを構築します。この章では、基本的なエディタ設定から、Node.js、パッケージマネージャー、モジュールバンドラー、コード品質ツール、バージョン管理システムまで、実際の開発現場で使用される環境を段階的に構築していきます。準備にかかる時間は約4〜5時間を想定しています。

1.1 モダンJavaScript開発環境の概要

現代のJavaScript開発では、単純なテキストエディタとブラウザだけでは不十分です。効率的で保守性の高いコードを書くために、以下のようなツールチェーンを構築します。

モダン開発環境のツール一覧

ツール名 用途 必須レベル 代替ツール
Node.js JavaScriptランタイム・開発ツール実行環境 必須 Deno, Bun
npm / yarn パッケージマネージャー・依存関係管理 必須 pnpm
VS Code 統合開発環境・エディタ 必須 WebStorm, Vim
Vite モジュールバンドラー・開発サーバー 必須 Webpack, Parcel
ESLint JavaScriptコード品質チェック 必須 JSHint, TSLint
Prettier コードフォーマッター 必須 Standard JS
Git 分散バージョン管理システム 必須 SVN, Mercurial
GitHub リモートリポジトリ・コラボレーション 推奨 GitLab, Bitbucket
Chrome DevTools ブラウザデバッグ・パフォーマンス分析 必須 Firefox DevTools

図1.1: モダンJavaScript開発環境のツールチェーン

モダン開発環境の各ツールの役割
  • Node.js：JavaScriptランタイム環境。サーバーサイドでのJavaScript実行、ビルドツールの実行基盤
  • npm/yarn：パッケージマネージャー。ライブラリのインストール、依存関係管理、スクリプト実行
  • VS Code：統合開発環境。コード編集、デバッグ、Git連携、拡張機能で生産性向上
  • Vite：高速なモジュールバンドラー。開発サーバー、ホットリロード、ビルド最適化
  • ESLint：コード品質チェッカー。バグの事前発見、コーディングスタイル統一
  • Prettier：コードフォーマッター。コードの自動整形、可読性向上
  • Git/GitHub：バージョン管理システム。コード履歴管理、チーム開発、バックアップ
  • Browser DevTools：ブラウザの開発者ツール。デバッグ、パフォーマンス解析、ネットワーク監視

1.2 Node.jsとパッケージマネージャーの設定

Node.jsは、JavaScriptをブラウザ以外で実行するためのランタイム環境です。モダンなJavaScript開発では、ビルドツール、パッケージマネージャー、開発サーバーなど、多くのツールがNode.js上で動作します。

Node.jsのインストール

1 公式サイトからダウンロード
  1. Node.js公式サイトにアクセス
  2. LTS（Long Term Support）版を選択してダウンロード
  3. インストーラーを実行し、デフォルト設定でインストール
2 インストールの確認

コマンドラインで以下を実行し、バージョンが表示されることを確認：

# Node.jsのバージョン確認 node --version # 例: v18.17.0 # npmのバージョン確認 npm --version # 例: 9.6.7

パッケージマネージャーの選択と設定

Node.jsにはnpmが標準で付属していますが、より高速で効率的なyarnも選択できます。

パッケージマネージャー 特徴 インストール方法
npm Node.js標準、幅広いサポート Node.jsと同時にインストール済み
yarn 高速、依存関係のロック機能 npm install -g yarn
pnpm ディスク容量節約、高速 npm install -g pnpm
推奨設定

この教材ではyarnを推奨します。以下のコマンドでインストール：

npm install -g yarn

インストール後、以下で確認：

yarn --version

グローバルパッケージの管理

開発に必要なグローバルツールをインストールします：

# 基本ツールのインストール yarn global add @vite/create-app yarn global add create-vite yarn global add eslint yarn global add prettier # インストール済みグローバルパッケージの確認 yarn global list
実習 1.2.1: Node.js環境の確認
  1. コマンドラインでNode.jsとnpmのバージョンを確認
  2. yarnをインストールし、バージョンを確認
  3. グローバルツールをインストール
  4. グローバルパッケージ一覧を表示し、インストールが成功していることを確認

パッケージマネージャーの基本コマンド

操作 npm yarn 説明
プロジェクト初期化 npm init yarn init package.jsonを作成
依存関係インストール npm install yarn install package.jsonの依存関係をインストール
パッケージ追加 npm install package-name yarn add package-name 新しいパッケージを追加
開発用パッケージ追加 npm install -D package-name yarn add -D package-name 開発時のみ使用するパッケージ
スクリプト実行 npm run script-name yarn script-name package.jsonのスクリプトを実行

1.3 VS Codeの高度な設定とワークスペース管理

Visual Studio Code（VS Code）は、Microsoft が開発している無料のコードエディタです。JavaScriptの開発に最適な機能が豊富に揃っています。

VS Codeのインストール手順

1 公式サイトからダウンロード
  1. VS Code公式サイトにアクセス
  2. 「Download for Windows/macOS/Linux」ボタンをクリック
  3. 使用しているOSに対応したインストーラをダウンロード
2 インストーラを実行
  • ダウンロードしたファイルを実行
  • インストール時は基本的にデフォルト設定のまま進行
  • 「PATHへの追加」オプションにチェックを入れる（推奨）
3 初回起動と基本設定

VS Codeを起動し、以下の設定を行います：

// VS Codeの推奨設定（ファイル > 基本設定 > 設定） { "editor.fontSize": 14, "editor.tabSize": 2, "editor.insertSpaces": true, "files.autoSave": "afterDelay", "files.autoSaveDelay": 1000, "files.encoding": "utf8", "emmet.includeLanguages": { "javascript": "javascriptreact" } }

必須拡張機能のインストール

以下の拡張機能をインストールすることで、JavaScript開発がより効率的になります：

拡張機能名 説明 インストール方法
JavaScript (ES6) code snippets JavaScript ES6のコードスニペット 拡張機能タブで「JavaScript ES6」を検索
Prettier - Code formatter コードの自動フォーマット 拡張機能タブで「Prettier」を検索
ESLint JavaScriptのコード品質チェック 拡張機能タブで「ESLint」を検索
Live Server ローカル開発サーバーの起動、リアルタイムプレビュー 拡張機能タブで「Live Server」を検索（必須）
実習 1.2.1: VS Codeの動作確認
  1. VS Codeを起動する
  2. 新しいフォルダ「javascript-practice」を作成し、VS Codeで開く
  3. 「index.html」という名前のファイルを作成
  4. 以下のコードを入力し、保存する：
<!DOCTYPE html> <html lang="ja"> <head> <meta charset="UTF-8"> <title>VS Codeテスト</title> </head> <body> <h1>VS Codeの設定が完了しました！</h1> <script> console.log('VS Codeの設定が完了しました！'); console.log('JavaScript学習を始めましょう。'); </script> </body> </html>

1.3 ブラウザ開発者ツールの使い方

ブラウザの開発者ツールは、JavaScriptのデバッグや動作確認に欠かせない機能です。主要なブラウザ（Chrome、Firefox、Edge）には強力な開発者ツールが搭載されています。

開発者ツールの開き方

ブラウザ 開き方 ショートカットキー
Chrome メニュー → その他のツール → デベロッパー ツール F12 または Ctrl+Shift+I
Firefox メニュー → ウェブ開発 → 開発者ツール F12 または Ctrl+Shift+I
Edge メニュー → その他のツール → 開発者ツール F12 または Ctrl+Shift+I

主要なタブとその機能

  • Elements（要素）：HTMLとCSSの確認・編集
  • Console（コンソール）：JavaScriptコードの実行とエラー確認
  • Sources（ソース）：JavaScriptファイルの表示とデバッグ
  • Network（ネットワーク）：ネットワーク通信の監視
  • Application（アプリケーション）：ローカルストレージなどの確認

コンソールでのJavaScript実行

コンソールタブでは、JavaScriptコードを直接実行できます：

// コンソールで試してみよう console.log('Hello, Console!'); // 計算もできます 2 + 3 // 変数の定義と使用 let name = 'JavaScript'; console.log('私は' + name + 'を学習中です'); // 現在の日時を取得 new Date();
実習 1.3.1: 開発者ツールの基本操作
  1. ブラウザを開き、F12キーで開発者ツールを開く
  2. Consoleタブをクリック
  3. 以下のコードを一行ずつ入力して実行する：
console.log('開発者ツールのテストです'); alert('アラートの動作確認'); document.title = 'タイトルを変更しました';

それぞれの結果を確認し、動作を理解しましょう。

簡単なHTMLファイルの作成と実行

JavaScriptをブラウザで実行するための基本的なHTMLファイルを作成します：

<!DOCTYPE html> <html lang="ja"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>JavaScript学習</title> </head> <body> <h1>JavaScript学習のページ</h1> <script> console.log('HTMLファイルからJavaScriptが実行されました'); console.log('今日の日付:', new Date().toLocaleDateString()); </script> </body> </html>
実習 1.3.2: HTMLファイルでのJavaScript実行
  1. VS Codeで「index.html」ファイルを作成
  2. 上記のHTMLコードを入力して保存
  3. ファイルをブラウザで開く（ファイルをブラウザにドラッグ&ドロップ）
  4. F12で開発者ツールを開き、Consoleタブでメッセージを確認

1.4 UTF-8設定の確認

文字エンコーディングの設定は、日本語を含むプログラムを正しく動作させるために重要です。UTF-8は現在最も広く使用されている文字エンコーディングです。

VS CodeでのUTF-8設定確認

1 エンコーディング設定の確認
  1. VS Codeで任意のファイルを開く
  2. 画面下部のステータスバーを確認
  3. 「UTF-8」と表示されていることを確認
  4. 表示されていない場合は、該当部分をクリックしてUTF-8を選択
2 デフォルトエンコーディングの設定

ファイル → 基本設定 → 設定 で以下を確認：

// settings.jsonに以下の設定があることを確認 { "files.encoding": "utf8", "files.autoGuessEncoding": false }

日本語を含むJavaScriptの動作確認

UTF-8エンコーディングが正しく設定されているかを確認するため、日本語を含むJavaScriptコードを実行します：

<!DOCTYPE html> <html lang="ja"> <head> <meta charset="UTF-8"> <title>日本語テスト</title> </head> <body> <h1>日本語の表示テスト</h1> <p>こんにちは、JavaScriptの世界へようこそ！</p> <script> console.log('こんにちは、JavaScript！'); console.log('文字化けしていませんか？'); let 名前 = '太郎'; let 年齢 = 25; console.log(名前 + 'さんは' + 年齢 + '歳です。'); // 日本語コメントも正しく表示されることを確認 /* 複数行の 日本語コメント */ </script> </body> </html>
実習 1.4.1: 日本語文字エンコーディングの確認
  1. 「japanese-test.html」ファイルを作成
  2. 上記のコードを入力して保存
  3. ファイルをブラウザで開く（Live Serverを使用またはファイルをダブルクリック）
  4. ブラウザの開発者ツールでコンソールメッセージを確認
  5. ページ上の日本語テキストも正しく表示されることを確認

HTMLファイルでの文字エンコーディング設定

HTMLファイルでは、<meta>タグでエンコーディングを明示的に指定します：

<!DOCTYPE html> <html lang="ja"> <head> <meta charset="UTF-8"> <!-- 上記のcharset="UTF-8"が重要 --> <title>日本語テスト</title> </head> <body> <h1>日本語の表示テスト</h1> <p>こんにちは、JavaScriptの世界へようこそ！</p> <script> console.log('HTMLからの日本語出力テスト'); alert('アラートダイアログの日本語表示'); </script> </body> </html>
文字エンコーディングに関する注意事項
  • 古いエディタやシステムでは、Shift_JISやEUC-JPが使用されている場合があります
  • チーム開発では、全員がUTF-8を使用することを確認してください
  • 既存のファイルを編集する際は、元のエンコーディングを確認してから変更してください
