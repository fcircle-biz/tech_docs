9.1 プロジェクト企画と設計

成功するプロジェクトは、しっかりとした企画と設計から始まります。まずは作りたいアプリケーションのアイデアを具体化し、実現可能な計画を立てましょう。

プロジェクト企画の流れ

1 アイデアの発散

日常の課題や興味のある分野から、解決したい問題を見つけます。

2 ターゲットユーザーの設定

誰がこのアプリを使うのか、どのような場面で使うのかを明確にします。

3 機能の整理と優先度付け

必要な機能をリストアップし、重要度順に並べます。

4 技術的実現性の評価

現在のスキルで実装可能かを検討し、必要に応じて機能を調整します。

企画書テンプレート

# プロジェクト企画書 ## アプリケーション名 「シンプル家計簿」 ## コンセプト 日々の収支を簡単に記録・可視化できる家計管理アプリ ## ターゲットユーザー - 家計管理を始めたい20-30代 - 複雑な機能は不要、シンプルさを重視する人 - スマートフォンでの利用が中心 ## 主要機能 ### 必須機能（MVP: Minimum Viable Product） 1. 収支記録の追加・編集・削除 2. カテゴリ別分類（食費、交通費など） 3. 月別収支の表示 4. 簡単なグラフ表示 ### 拡張機能（時間があれば） 1. 予算設定と残額表示 2. レシート撮影による自動入力 3. データのエクスポート機能
実習 9.1.1: プロジェクト企画書の作成
  1. 作りたいアプリケーションのアイデアを3つ考える
  2. 1つを選択し、上記テンプレートを使って企画書を作成
  3. 家族や友人にアイデアを説明し、フィードバックをもらう
  4. フィードバックを反映して企画を改善

この課程では、実装しやすさと学習効果を考慮した3つのテーマを用意しています。各テーマの特徴を理解し、あなたの興味と技術レベルに最適なものを選択しましょう。

概要：Webで見つけた記事をブックマークし、タグ付けや検索ができるシングルページアプリケーション

主要技術：React/Vue、LocalStorage、外部API（記事メタデータ取得）

学習ポイント：SPAの構築、状態管理、外部API連携

難易度：⭐⭐⭐☆☆（中級）

主要技術：Weather API、Geolocation API、カレンダーライブラリ

堅牢で保守しやすいアプリケーションを作るには、適切なアーキテクチャ設計が不可欠です。ここでは、モダンなJavaScriptアプリケーションの標準的な構成を学びます。

simple-household-budget/ ├── public/ │ ├── index.html │ ├── manifest.json │ └── icons/ ├── src/ │ ├── components/ # 再利用可能なコンポーネント │ │ ├── Header.jsx │ │ ├── TransactionForm.jsx │ │ ├── TransactionList.jsx │ │ └── Chart.jsx │ ├── pages/ # ページコンポーネント │ │ ├── Dashboard.jsx │ │ ├── AddTransaction.jsx │ │ └── Statistics.jsx │ ├── hooks/ # カスタムフック │ │ ├── useLocalStorage.js │ │ └── useTransactions.js │ ├── utils/ # ユーティリティ関数 │ │ ├── dateHelper.js │ │ ├── currencyFormat.js │ │ └── apiClient.js │ ├── styles/ # スタイル関連 │ │ ├── globals.css │ │ └── components.css │ ├── App.jsx │ └── index.js ├── tests/ # テストファイル │ ├── components/ │ └── utils/ ├── package.json ├── README.md └── .gitignore

アーキテクチャ設計の考慮事項

コンポーネント設計の原則
  • 単一責任の原則：1つのコンポーネントは1つの責任のみを持つ
  • 再利用性：他の場所でも使えるよう汎用的に設計
  • テスタビリティ：テストしやすい小さな単位で分割

状態管理の設計

アプリケーションの状態をどこで管理するかは重要な設計判断です：

// useTransactions.js - カスタムフック例 import { useState, useEffect } from 'react'; export const useTransactions = () => { const [transactions, setTransactions] = useState([]); const [loading, setLoading] = useState(false); const [error, setError] = useState(null); // ローカルストレージからデータを読み込み useEffect(() => { try { const saved = localStorage.getItem('transactions'); if (saved) { setTransactions(JSON.parse(saved)); } } catch (err) { setError('データの読み込みに失敗しました'); } }, []); // トランザクションを追加 const addTransaction = (transaction) => { const newTransaction = { ...transaction, id: Date.now().toString(), createdAt: new Date().toISOString() }; const updated = [...transactions, newTransaction]; setTransactions(updated); localStorage.setItem('transactions', JSON.stringify(updated)); }; // トランザクションを削除 const deleteTransaction = (id) => { const updated = transactions.filter(t => t.id !== id); setTransactions(updated); localStorage.setItem('transactions', JSON.stringify(updated)); }; return { transactions, loading, error, addTransaction, deleteTransaction }; };

API連携の実装パターン

// apiClient.js - API通信の基本クラス class ApiClient { constructor(baseURL, apiKey) { this.baseURL = baseURL; this.apiKey = apiKey; } async get(endpoint, params = {}) { try { const url = new URL(endpoint, this.baseURL); url.searchParams.append('appid', this.apiKey); Object.keys(params).forEach(key => { url.searchParams.append(key, params[key]); }); const response = await fetch(url); if (!response.ok) { throw new Error(`HTTP error! status: ${response.status}`); } return await response.json(); } catch (error) { console.error('API request failed:', error); throw error; } } } // weatherService.js - 天気API専用サービス export class WeatherService { constructor() { this.client = new ApiClient( 'https://api.openweathermap.org/data/2.5/', process.env.REACT_APP_WEATHER_API_KEY ); } async getCurrentWeather(city) { try { const data = await this.client.get('weather', { q: city, units: 'metric', lang: 'ja' }); return { temperature: Math.round(data.main.temp), description: data.weather[0].description, humidity: data.main.humidity, pressure: data.main.pressure }; } catch (error) { throw new Error('天気情報の取得に失敗しました'); } } }

エラーハンドリングとローディング状態

// useWeather.js - 天気情報取得のカスタムフック import { useState, useEffect } from 'react'; import { WeatherService } from '../utils/weatherService'; export const useWeather = (city) => { const [weather, setWeather] = useState(null); const [loading, setLoading] = useState(false); const [error, setError] = useState(null); useEffect(() => { if (!city) return; const fetchWeather = async () => { setLoading(true); setError(null); try { const weatherService = new WeatherService(); const data = await weatherService.getCurrentWeather(city); setWeather(data); } catch (err) { setError(err.message); } finally { setLoading(false); } }; fetchWeather(); }, [city]); return { weather, loading, error }; };
APIキーの管理に関する重要な注意
  • APIキーは絶対にGitHubにコミットしない
  • 環境変数（.env.local）を使用してキーを管理
  • .gitignoreに.env*を追加
  • 本番環境では環境変数を別途設定

9.6 デプロイとCI/CD

作成したアプリケーションを世界に公開しましょう。GitHub Pages を使った簡単なデプロイから、自動化されたCI/CDパイプラインまでを体験します。

GitHub Pages デプロイ手順

1 ビルド設定の確認
# package.json にビルドスクリプトがあることを確認 { "scripts": { "build": "vite build", "preview": "vite preview" } }
2 GitHub Actions ワークフローの作成
# .github/workflows/deploy.yml name: Deploy to GitHub Pages on: push: branches: [ main ] pull_request: branches: [ main ] jobs: build-and-deploy: runs-on: ubuntu-latest steps: - name: Checkout uses: actions/checkout@v3 - name: Setup Node.js uses: actions/setup-node@v3 with: node-version: '18' cache: 'npm' - name: Install dependencies run: npm ci - name: Build run: npm run build - name: Deploy to GitHub Pages uses: peaceiris/actions-gh-pages@v3 if: github.ref == 'refs/heads/main' with: github_token: ${{ secrets.GITHUB_TOKEN }} publish_dir: ./dist
3 GitHub Pages の有効化
  1. リポジトリの Settings タブを開く
  2. 左サイドバーの「Pages」をクリック
  3. Source を「GitHub Actions」に設定
  4. ワークフローを実行してデプロイ完了を確認
