第1章: Next.js入門と環境構築

Next.jsの世界へようこそ

1.1 Next.jsとは

1.1.1 Next.jsの概要

Next.jsは、ReactをベースにしたフルスタックWebアプリケーション開発フレームワークです。Vercel社が開発・保守しており、モダンなWeb開発に必要な機能を統合的に提供します。

Next.jsの主な特徴：

  • サーバーサイドレンダリング（SSR）：初回表示を高速化し、SEOを改善
  • 静的サイト生成（SSG）：ビルド時にHTMLを生成し、CDNから高速配信
  • ファイルベースルーティング：ディレクトリ構造がそのままURLに対応
  • APIルート：バックエンドAPIをNext.js内に統合可能
  • 画像最適化：自動的に画像を最適化して配信
  • TypeScript対応：型安全な開発をサポート

アプリケーション"] B --> C["サーバーサイド
レンダリング"] B --> D["静的サイト生成"] B --> E["クライアント
サイドルーティング"] C --> F["高速な初回表示"] D --> G["優れたSEO"] E --> H["スムーズな
ページ遷移"]

1.1.2 なぜNext.jsを使うのか

通常のReactアプリケーション（Create React App等で作成）は、すべてがクライアントサイド（ブラウザ側）で動作します。これには以下の課題があります：

  • 初回表示が遅い：JavaScriptのダウンロードと実行が完了するまで画面が表示されない
  • SEO対策が困難：検索エンジンのクローラーがJavaScriptで生成されたコンテンツを読み取れない場合がある
  • ルーティングの手動設定：React Routerなどのライブラリを別途導入・設定する必要がある

Next.jsはこれらの課題を解決します：

ダウンロード"] A2 --> A3["React実行"] A3 --> A4["画面表示"] end subgraph NextJS["Next.js"] B1["ブラウザ"] --> B2["サーバーで
HTML生成"] B2 --> B3["即座に
画面表示"] B3 --> B4["React
ハイドレーション"] end

1.1.3 ReactとNext.jsの関係

Next.jsはReactの上位レイヤーとして機能します。つまり：

  • Next.jsを使うには、Reactの知識が必要です
  • ReactコンポーネントをそのままNext.jsで使用できます
  • Next.jsは、Reactに「サーバーサイド機能」「ルーティング」「最適化」などを追加したものです
例えるなら

React = 車のエンジン（基本的な動力）
Next.js = 完全装備の自動車（エンジン + ナビ + 安全装置 + 快適装備）

1.2 開発環境の構築

1.2.1 必要な環境

Next.jsの開発を始める前に、以下の環境を準備します：

ツール 推奨バージョン 役割
Node.js 18.17以降（LTS推奨） JavaScriptランタイム環境
npm/yarn/pnpm 最新版 パッケージマネージャー
VS Code 最新版 コードエディタ（推奨）
Git 最新版 バージョン管理

1.2.2 Node.jsのインストール確認

ターミナル（コマンドプロンプト）で以下のコマンドを実行して、Node.jsがインストールされているか確認します：

# Node.jsのバージョン確認
node --version
# 出力例: v20.10.0

# npmのバージョン確認
npm --version
# 出力例: 10.2.3
バージョンが表示されない場合

Node.jsがインストールされていない可能性があります。Node.js公式サイトから最新のLTS版をダウンロードしてインストールしてください。

1.3 Next.jsプロジェクトの作成

1.3.1 create-next-appを使った自動セットアップ

Next.jsプロジェクトはcreate-next-appコマンドで簡単に作成できます。このコマンドは、必要なファイルやフォルダを自動的に生成してくれます。

実習 1-1: 最初のNext.jsプロジェクト作成

実際にNext.jsプロジェクトを作成してみましょう。

手順
  1. 任意のディレクトリでターミナルを開く
  2. 以下のコマンドを実行する
# プロジェクトを作成
npx create-next-app@latest my-first-nextjs-app

# 対話形式の質問に答える（以下は推奨設定）
# ✓ Would you like to use TypeScript? ... No
# ✓ Would you like to use ESLint? ... Yes
# ✓ Would you like to use Tailwind CSS? ... Yes
# ✓ Would you like to use `src/` directory? ... No
# ✓ Would you like to use App Router? ... Yes
# ✓ Would you like to customize the default import alias? ... No
コマンドの意味
  • npx：Node.jsに含まれるコマンド実行ツール
  • create-next-app：Next.jsプロジェクト作成ツール
  • @latest：最新版を使用
  • my-first-nextjs-app：プロジェクト名（任意の名前に変更可能）
期待される結果

my-first-nextjs-appという名前のフォルダが作成され、その中にNext.jsプロジェクトの基本構造が生成されます。

1.3.2 開発サーバーの起動

プロジェクトが作成できたら、開発サーバーを起動して確認します：

# プロジェクトディレクトリに移動
cd my-first-nextjs-app

# 開発サーバーを起動
npm run dev

# 以下のメッセージが表示される
# ✓ Ready in 2.3s
# ○ Local: http://localhost:3000

ブラウザでhttp://localhost:3000にアクセスすると、Next.jsのウェルカムページが表示されます。

開発サーバーの特徴
  • ホットリロード：コードを編集すると自動的にブラウザが更新される
  • 高速リフレッシュ：Reactコンポーネントの変更が即座に反映される
  • エラー表示：コードにエラーがある場合、ブラウザ上に詳細が表示される

1.4 Next.jsプロジェクトの基本構造

1.4.1 フォルダ・ファイル構成

作成されたプロジェクトには、以下のような構造があります：

my-first-nextjs-app/
├── app/                 # アプリケーションのメインディレクトリ（App Router）
│   ├── favicon.ico     # ファビコン
│   ├── globals.css     # グローバルスタイル
│   ├── layout.js       # ルートレイアウト（全ページ共通）
│   └── page.js         # ホームページ（/）
├── public/             # 静的ファイル（画像、フォント等）
├── node_modules/       # npmパッケージ（自動生成）
├── .eslintrc.json      # ESLint設定
├── .gitignore          # Gitで無視するファイル
├── next.config.js      # Next.js設定ファイル
├── package.json        # プロジェクト情報と依存関係
├── package-lock.json   # パッケージバージョン固定
└── README.md           # プロジェクト説明

1.4.2 重要なファイルの役割

ファイル/フォルダ 役割
app/page.js トップページ（/）のコンテンツを定義
app/layout.js 全ページで共通するレイアウト（ヘッダー、フッター等）
app/globals.css アプリケーション全体に適用されるCSSスタイル
public/ 画像やファビコンなど、静的に配信するファイルを配置
package.json プロジェクト情報、依存パッケージ、スクリプトを記述
next.config.js Next.jsの動作をカスタマイズする設定ファイル

1.4.3 app/page.jsの中身を見てみよう

app/page.jsを開くと、以下のような内容になっています（簡略版）： 

// app/page.js
export default function Home() {
  return (
    <main>
      <h1>Welcome to Next.js!</h1>
      <p>Get started by editing app/page.js</p>
    </main>
  );
}

このコードの説明：

  • export default function Home()：Reactコンポーネントを定義してエクスポート
  • return (...)：このページに表示するHTMLのような構文（JSX）を返す
  • <main>、<h1>、<p>：標準的なHTML要素

1.5 App RouterとPages Routerの違い

1.5.1 2つのルーティングシステム

Next.jsには2つの異なるルーティングシステムが存在します：

項目 Pages Router（旧） App Router（新・推奨）
導入時期 Next.js 1.0から Next.js 13.4から（安定版）
ディレクトリ pages/ app/
主要機能 getServerSideProps、getStaticProps React Server Components、Server Actions
レイアウト _app.js、_document.js layout.js（ネスト可能）
推奨度 既存プロジェクト向け 新規プロジェクトに推奨

1.5.2 本ガイドの方針

本ガイドでは、最新のApp Routerを中心に学習します。
App RouterはNext.jsの未来であり、React Server Componentsなどの最新機能を活用できます。

（推奨）"] A --> C["Pages Router
（レガシー）"] B --> D["React Server
Components"] B --> E["Server Actions"] B --> F["ネストされた
レイアウト"] C --> G["従来の
SSR/SSG"]

1.6 最初のページをカスタマイズしよう

実習 1-2: ホームページを変更する

app/page.jsを編集して、自分のメッセージを表示してみましょう。

手順
  1. VS Codeなどのエディタでapp/page.jsを開く
  2. 以下のように内容を書き換える
// app/page.js
export default function Home() {
  return (
    <main style={{ padding: '50px', textAlign: 'center' }}>
      <h1>私の最初のNext.jsアプリ</h1>
      <p>Next.jsの学習を始めます！</p>
      <ul style={{ listStyle: 'none', padding: 0 }}>
        <li>✓ 環境構築完了</li>
        <li>✓ プロジェクト作成完了</li>
        <li>✓ 開発サーバー起動完了</li>
      </ul>
    </main>
  );
}
期待される結果

保存すると自動的にブラウザが更新され、新しいメッセージが表示されます。開発サーバーを再起動する必要はありません。

1.7 よくあるトラブルと解決策

症状 原因 解決策
ポート3000が使用中 別のアプリが既に3000番ポートを使用 PORT=3001 npm run devで別のポートを指定
変更が反映されない ブラウザキャッシュ、または開発サーバーの不具合 ブラウザをハードリロード（Ctrl+Shift+R）、または開発サーバー再起動
ESLintエラーが出る コード規約違反 エラーメッセージを読んで該当箇所を修正
モジュールが見つからない node_modulesが破損または未インストール npm installを再実行
理解度確認クイズ
  1. Next.jsの主な利点を3つ挙げてください。
    回答例：（1）サーバーサイドレンダリングによる高速な初回表示、（2）優れたSEO対策、（3）ファイルベースルーティングによる直感的な開発
  2. create-next-appコマンドで作成されたプロジェクトの中で、トップページ（/）を定義しているファイルはどれですか？
    回答：app/page.js
  3. Next.js 13.4以降で推奨されるルーティングシステムは何ですか？
    回答：App Router（app/ディレクトリを使用）
  4. 開発サーバーを起動するコマンドは何ですか？
    回答：npm run dev

1.8 まとめ

この章では、Next.jsの基礎を学びました：

  • Next.jsは、Reactをベースにしたフルスタックフレームワーク
  • SSR、SSG、ファイルベースルーティングなどの強力な機能を提供
  • create-next-appで簡単にプロジェクトを作成できる
  • app/ディレクトリを使用するApp Routerが最新の推奨方式
  • 開発サーバーはホットリロードに対応し、快適な開発体験を提供
