10.1 Vercelへのデプロイ

10.1.1 Vercelデプロイの手順

Next.jsの開発元であるVercelは、最も簡単なデプロイ方法を提供します：

実習 10-1: Vercelへの初回デプロイ
手順
  1. VercelにアクセスしてGitHubアカウントで登録
  2. プロジェクトをGitHubにプッシュ
  3. Vercelダッシュボードで「New Project」をクリック
  4. GitHubリポジトリを選択してインポート
  5. 「Deploy」ボタンをクリック

数分後、プロジェクトが自動的にデプロイされ、URLが発行されます。

10.1.2 自動デプロイ

GitHubにプッシュすると、自動的にVercelがビルド・デプロイします： 

# mainブランチへのプッシュ → 本番デプロイ
git push origin main

# その他のブランチ → プレビューデプロイ
git push origin feature/new-page

10.2 環境変数の管理

10.2.1 ローカル環境変数

# .env.local（Gitにコミットしない）
DATABASE_URL="postgresql://..."
NEXTAUTH_SECRET="your-secret-key"
NEXT_PUBLIC_API_URL="https://api.example.com"
重要：環境変数の種類
  • NEXT_PUBLIC_で始まる変数：クライアント側で使用可能（公開される）
  • それ以外：サーバー側でのみ使用可能（安全）

10.2.2 Vercelでの環境変数設定

  1. Vercelダッシュボードでプロジェクトを選択
  2. 「Settings」→「Environment Variables」
  3. 変数名と値を入力
  4. Production、Preview、Developmentの環境を選択
  5. 「Save」をクリック

10.3 本番ビルド

10.3.1 ビルドコマンド

# 本番用ビルド
npm run build

# ビルド結果を確認
npm run start

10.3.2 ビルド最適化

// next.config.js
module.exports = {
  // 画像最適化
  images: {
    formats: ['image/webp'],
  },
  
  // 厳格モード
  reactStrictMode: true,
  
  // SWC minifier（高速化）
  swcMinify: true,
  
  // バンドル分析
  // npm install @next/bundle-analyzer
  ...(process.env.ANALYZE === 'true' && {
    webpack: (config) => {
      return config;
    },
  }),
};

10.4 Dockerによるデプロイ

# Dockerfile
FROM node:18-alpine AS base

# 依存関係のインストール
FROM base AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci

# ビルド
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# 本番実行
FROM base AS runner
WORKDIR /app
ENV NODE_ENV production
COPY --from=builder /app/public ./public
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static

EXPOSE 3000
CMD ["node", "server.js"]

10.5 CI/CD パイプライン

10.5.1 GitHub Actionsの設定

# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm ci
      - run: npm run lint
      - run: npm run build

10.6 モニタリング

10.6.1 Vercel Analytics

Vercel Analyticsを有効にすると、Core Web Vitalsをリアルタイムで監視できます： 

npm install @vercel/analytics
// app/layout.js
import { Analytics } from '@vercel/analytics/react';

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Analytics />
      </body>
    </html>
  );
}
理解度確認クイズ
  1. 本番用ビルドを作成するコマンドは何ですか？
    回答：npm run build
  2. クライアント側で使用する環境変数は何で始まる必要がありますか？
    回答：NEXT_PUBLIC_
