7.1 CSS Modules

7.1.1 CSS Modulesとは

CSS Modulesは、CSSのスコープをコンポーネント単位で管理できる仕組みです。クラス名の衝突を自動的に防ぎます。 

/* app/components/Button.module.css */
.button {
  background-color: #667eea;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 5px;
  cursor: pointer;
}

.button:hover {
  background-color: #4fa8c5;
}
// app/components/Button.js
import styles from './Button.module.css';

export default function Button({ children }) {
  return (
    <button className={styles.button}>
      {children}
    </button>
  );
}

7.2 Tailwind CSS

7.2.1 Tailwind CSSの特徴

Tailwind CSSは、ユーティリティファーストのCSSフレームワークで、HTMLクラス名を使ってスタイルを適用します。

実習 7-1: Tailwind CSSを使ったカードコンポーネント
// app/components/Card.js
export default function Card({ title, description }) {
  return (
    <div className="max-w-sm rounded-lg overflow-hidden shadow-lg bg-white p-6 hover:shadow-xl transition-shadow">
      <h2 className="text-2xl font-bold text-gray-800 mb-2">
        {title}
      </h2>
      <p className="text-gray-600">
        {description}
      </p>
    </div>
  );
}

7.3 Next.js Imageコンポーネント

7.3.1 Imageコンポーネントの利点

  • 自動的に最適なフォーマット（WebP等）に変換
  • 遅延読み込み（lazy loading）
  • レスポンシブ画像の自動生成
  • CLS（Cumulative Layout Shift）の防止
// app/page.js
import Image from 'next/image';

export default function HomePage() {
  return (
    <div>
      <h1>画像の最適化</h1>
      <Image
        src="/images/hero.jpg"
        alt="ヒーロー画像"
        width={800}
        height={600}
        priority
      />
    </div>
  );
}

7.3.2 外部画像の使用

// next.config.js
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'example.com',
      },
    ],
  },
}
