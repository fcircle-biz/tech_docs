第5章: データフェッチングとレンダリング戦略

パフォーマンスを最大化するレンダリング戦略

5.1 Next.jsのレンダリング戦略概要

5.1.1 3つの主要なレンダリング方式

Next.jsは、ページごとに最適なレンダリング方式を選択できます：

レンダリング方式 タイミング 用途 パフォーマンス
SSG（Static Site Generation） ビルド時 静的コンテンツ ★★★★★
ISR（Incremental Static Regeneration） ビルド時+定期更新 定期的に更新されるコンテンツ ★★★★☆
SSR（Server-Side Rendering） リクエスト時 動的コンテンツ ★★★☆☆

を即座に返す"] B -->|ISR| D["キャッシュ確認
期限切れなら再生成"] B -->|SSR| E["毎回サーバーで
HTMLを生成"] C --> F["超高速"] D --> G["高速"] E --> H["中速"]

5.2 サーバーコンポーネントでのデータフェッチ

5.2.1 基本的なデータ取得

サーバーコンポーネントでは、async/awaitを使って直接データを取得できます： 

// app/posts/page.js
export default async function PostsPage() {
  // サーバーで実行されるデータ取得
  const res = await fetch('https://jsonplaceholder.typicode.com/posts');
  const posts = await res.json();

  return (
    <div>
      <h1>ブログ記事一覧</h1>
      <ul>
        {posts.slice(0, 10).map(post => (
          <li key={post.id}>
            <h2>{post.title}</h2>
            <p>{post.body.substring(0, 100)}...</p>
          </li>
        ))}
      </ul>
    </div>
  );
}

5.2.2 複数のデータソースの並列取得

// Promise.allで並列実行
export default async function DashboardPage() {
  const [posts, users, comments] = await Promise.all([
    fetch('https://api.example.com/posts').then(r => r.json()),
    fetch('https://api.example.com/users').then(r => r.json()),
    fetch('https://api.example.com/comments').then(r => r.json()),
  ]);

  return (
    <div>
      <h1>ダッシュボード</h1>
      <p>投稿数: {posts.length}</p>
      <p>ユーザー数: {users.length}</p>
      <p>コメント数: {comments.length}</p>
    </div>
  );
}

5.3 Static Site Generation（SSG）

5.3.1 SSGの基本

デフォルトでは、fetchはキャッシュされるため、ビルド時にデータを取得してHTMLを生成します： 

// app/about/page.js
// デフォルトでSSG（キャッシュあり）
export default async function AboutPage() {
  const res = await fetch('https://api.example.com/company-info');
  const info = await res.json();

  return (
    <div>
      <h1>{info.name}</h1>
      <p>{info.description}</p>
    </div>
  );
}

5.3.2 動的ルートのSSG：generateStaticParams

動的ルートでSSGを使用する場合、generateStaticParamsで事前生成するパスを指定します：

実習 5-1: ブログ記事の静的生成
// app/blog/[slug]/page.js

// ビルド時に生成するページのパスを指定
export async function generateStaticParams() {
  const posts = await fetch('https://api.example.com/posts').then(r => r.json());

  return posts.map((post) => ({
    slug: post.slug,
  }));
}

// 各ページの内容を生成
export default async function BlogPost({ params }) {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`)
    .then(r => r.json());

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}

この設定により、ビルド時にすべての記事ページが事前生成されます。

5.4 Server-Side Rendering（SSR）

5.4.1 動的レンダリングの有効化

fetchに{ cache: 'no-store' }オプションを指定すると、毎回サーバーで実行されます： 

// app/dashboard/page.js
export default async function DashboardPage() {
  // 毎回サーバーでデータ取得（SSR）
  const res = await fetch('https://api.example.com/dashboard', {
    cache: 'no-store'
  });
  const data = await res.json();

  return (
    <div>
      <h1>ダッシュボード</h1>
      <p>最終更新: {new Date().toLocaleString()}</p>
      <p>データ: {JSON.stringify(data)}</p>
    </div>
  );
}

5.4.2 動的関数を使った強制SSR

以下の関数を使用すると、自動的にSSRになります： 

import { cookies, headers } from 'next/headers';

export default async function Page() {
  // cookiesやheadersを使用するとSSRになる
  const cookieStore = cookies();
  const headersList = headers();

  return <div>動的にレンダリングされます</div>;
}

5.5 Incremental Static Regeneration（ISR）

5.5.1 ISRの基本

ISRは、静的生成の利点と動的更新の柔軟性を組み合わせた戦略です： 

// app/news/page.js
export default async function NewsPage() {
  // 60秒ごとに再検証（revalidate）
  const res = await fetch('https://api.example.com/news', {
    next: { revalidate: 60 }
  });
  const news = await res.json();

  return (
    <div>
      <h1>最新ニュース</h1>
      <p>このページは60秒ごとに更新されます</p>
      {news.map(item => (
        <article key={item.id}>
          <h2>{item.title}</h2>
          <p>{item.summary}</p>
        </article>
      ))}
    </div>
  );
}
ISRの動作
  1. 初回リクエスト：静的に生成されたページを返す（超高速）
  2. revalidate期間内：キャッシュされたページを返す
  3. 期間経過後の次回リクエスト：
    • 古いページを即座に返す
    • バックグラウンドで新しいページを生成
    • 次回以降は新しいページを表示

5.5.2 ルートセグメントでのrevalidate設定

// app/blog/page.js
export const revalidate = 3600; // 1時間ごとに再検証

export default async function BlogPage() {
  const posts = await fetch('https://api.example.com/posts').then(r => r.json());

  return (
    <div>
      {posts.map(post => (
        <article key={post.id}>
          <h2>{post.title}</h2>
        </article>
      ))}
    </div>
  );
}

5.6 SuspenseとStreaming

5.6.1 React Suspenseの活用

Suspenseを使うと、データ読み込み中に代替UIを表示できます： 

// app/page.js
import { Suspense } from 'react';
import Posts from './components/Posts';
import Comments from './components/Comments';

export default function HomePage() {
  return (
    <div>
      <h1>ホームページ</h1>

      {/* 投稿リストが読み込まれるまで「読み込み中...」を表示 */}
      <Suspense fallback={<p>投稿を読み込み中...</p>}>
        <Posts />
      </Suspense>

      {/* コメントは独立して読み込まれる */}
      <Suspense fallback={<p>コメントを読み込み中...</p>}>
        <Comments />
      </Suspense>
    </div>
  );
}
// app/components/Posts.js
export default async function Posts() {
  // 時間のかかるデータ取得
  const posts = await fetch('https://api.example.com/posts').then(r => r.json());

  return (
    <ul>
      {posts.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}

5.6.2 Streamingによる段階的表示

Suspenseを使用すると、ページの一部が準備できた段階で順次表示されます（ストリーミング）：

flowchart LR A["リクエスト"] --> B["ヘッダー
即座に表示"] B --> C["投稿リスト
1秒後に表示"] C --> D["コメント
2秒後に表示"]
理解度確認クイズ
  1. SSG、ISR、SSRの中で、最もパフォーマンスが高いのはどれですか？
    回答：SSG（ビルド時に生成されたHTMLを即座に返すため）
  2. fetchでSSRを有効にするオプションは何ですか？
    回答：{ cache: 'no-store' }
  3. ISRで再検証の間隔を指定するプロパティは何ですか？
    回答：revalidate（秒単位で指定）
