第6章: 稟議書・申請書の作成

承認を得るための効果的な稟議書作成

この章で学ぶこと
  • 稟議書の目的と社内承認フローを理解する
  • 稟議書の基本構成を習得する
  • 説得力のある申請理由の書き方を学ぶ
  • 費用対効果の説明方法を身につける
  • 各種申請書の作成方法を理解する

6.1 稟議書の目的と社内承認フロー

稟議書（りんぎしょ）は、業務上の重要な決定や支出について、上司や経営層の承認を得るための文書です。正式な社内手続きとして、多くの企業で使用されています。

flowchart LR A[稟議書の目的] --> B[承認獲得] A --> C[意思決定の記録] A --> D[責任の明確化] A --> E[予算管理] B --> B1["上司・経営層の
承認を得る"] C --> C1["決定プロセスを
文書化"] D --> D1["誰が何を決定
したか記録"] E --> E1["支出の管理と
統制"]

稟議が必要なケース

  • 物品購入: 一定額以上の備品・設備の購入
  • 契約締結: 業務委託契約、システム導入契約など
  • 人事関連: 採用、昇進、人事異動など
  • 出張・研修: 海外出張、高額な研修受講
  • その他: 社内規程で定められた事項

6.2 稟議書の基本構成

稟議書には標準的なフォーマットがあります。企業によって若干異なりますが、基本的な構成要素は共通しています。

項目内容記載のポイント
件名申請事項のタイトル具体的で分かりやすく
申請日稟議書提出日正確な日付を記載
申請者起案者の氏名・所属連絡先も記載
申請事項何を申請するか簡潔明瞭に
申請理由なぜ必要か説得力のある理由を記載
詳細内容具体的な内容5W1Hで整理
予算・費用必要な金額見積書を添付
期待効果得られる成果定量的に示す
添付資料見積書、仕様書等判断材料となる資料
実習 6-1: 稟議書の作成

状況: 業務用ノートPC 10台の購入申請

作成例
稟議書

件名: 業務用ノートPC購入の件
申請日: 2025年10月30日
申請者: 情報システム部 田中太郎（内線:1234）

【申請事項】
業務用ノートPC 10台の購入

【申請理由】
現在使用中のPCが導入から5年以上経過し、以下の問題が発生しています。
1. 動作が遅く業務効率が低下（起動に5分以上）
2. OSサポート終了によるセキュリティリスク
3. 故障頻度の増加（月2〜3件の修理対応）

【詳細内容】
購入機種: Dell Latitude 5540
台数: 10台
配布先: 営業部 8台、総務部 2台
納期: 2025年12月中旬

【予算】
単価: 15万円 × 10台 = 150万円
設定費用: 5万円
合計: 155万円（税込）

【期待効果】
・業務効率向上: 起動時間5分→30秒（90%短縮）
・修理コスト削減: 年間約30万円
・セキュリティリスク低減

【費用対効果】
投資: 155万円
削減効果: 修理費30万円/年 + 業務効率化50万円/年 = 80万円/年
投資回収期間: 約2年

以上、ご承認のほどよろしくお願いいたします。

添付資料: 
1. 見積書（Dell社）
2. 現行PC仕様書
3. 故障履歴一覧

6.3 承認者が求める情報の理解

稟議書を承認する人（上司、役員など）は、限られた時間で多くの稟議書を確認します。承認者の視点を理解することが重要です。

承認者が確認するポイント

  • 必要性: 本当に必要な支出か
  • 妥当性: 金額や内容は適切か
  • 緊急性: 今すぐ必要か、先延ばし可能か
  • 効果: 会社にとってどんなメリットがあるか
  • リスク: 承認しないとどうなるか
承認されにくい稟議書の特徴
  • 理由が曖昧で説得力がない
  • 費用対効果が不明確
  • 代替案の検討が不十分
  • 情報が不足している（見積書がない等）
  • 申請内容に矛盾がある

6.4 説得力のある申請理由の書き方

申請理由は稟議書の中で最も重要な部分です。なぜ必要かを論理的に説明します。

効果的な申請理由の構成

flowchart TD A[申請理由の構成] --> B["現状の問題点
何が困っているか"] B --> C["問題の影響
放置するとどうなるか"] C --> D["解決方法
なぜこの申請が解決策か"] D --> E["代替案の検討
他の方法は考えたか"]
悪い例良い例
「PCが古くなったので新しいものが欲しいです」 「導入5年経過のPCが動作遅延により業務効率を低下させています。1日あたり30分の時間損失が発生しており、年間では約50万円相当の生産性低下となっています」
「システムを導入したいです」 「現行の手作業による処理では月間80時間を要しており、システム導入により20時間に短縮可能です。人件費換算で年間180万円の削減効果が見込まれます」

6.5 費用対効果の説明方法

稟議書では、投資に見合う効果があることを数値で示すことが重要です。

費用対効果の示し方

  1. 初期投資額を明示する
  2. ランニングコスト（継続的な費用）を示す
  3. 削減効果や増収効果を具体的に算出する
  4. 投資回収期間を計算する
  5. 定性的効果（数値化しにくい効果）も記載する
【費用対効果の記載例】

投資額: 300万円
ランニングコスト: 年間50万円

削減効果:
・業務時間削減: 月200時間 → 年間2,400時間
  人件費換算: 2,400時間 × 3,000円 = 720万円/年
・ミス削減効果: 月10件 → 1件
  手戻り工数削減: 年間約100万円相当

投資回収期間: 
初年度利益 = 820万円 - 350万円 = 470万円
→ 1年以内に回収可能

定性的効果:
・顧客満足度の向上
・社員の業務負担軽減
・コンプライアンス強化

6.6 リスク対策の記述

申請内容を実行する際のリスクと対策を記載することで、承認者の不安を軽減します。

想定リスク対策
予算超過 複数社から相見積もりを取得済み。予備費10%を確保
導入スケジュール遅延 ベンダーと契約書で納期を明記。遅延時のペナルティ条項を設定
システム障害 テスト期間を2週間確保。バックアップ体制を構築
社員の不満 導入前に説明会を実施。サポート窓口を設置

6.7 代替案の提示

複数の選択肢を検討し、なぜこの案が最適かを示すことで、稟議書の説得力が増します。

実習 6-2: 代替案の比較
比較表の例
案メリットデメリット費用評価
案1: 新品購入最新性能、保証付き初期費用高150万円◎推奨
案2: 中古購入初期費用安保証なし、故障リスク80万円△
案3: リース初期費用不要長期的には割高月3万円○
案4: 現状維持費用不要業務効率悪化継続0円×

推奨案: 案1（新品購入）
理由: 初期費用は高いが、保証付きで長期的に最もコスト効率が良い。業務効率向上効果も最大。

6.8 各種申請書の作成

稟議書以外にも、業務上様々な申請書を作成する機会があります。

主な申請書の種類

  • 出張申請書: 出張の目的、日程、費用を申請
  • 購買申請書: 物品やサービスの購入を申請
  • 休暇申請書: 有給休暇や特別休暇を申請
  • 交通費精算書: 業務で使用した交通費を精算
  • 経費精算書: 立替経費の精算を申請

6.9 電子稟議システムの活用

多くの企業では、紙の稟議書から電子稟議システムに移行しています。

電子稟議のメリット

  • 承認スピード向上: メール通知で即座に確認可能
  • 進捗状況の可視化: 誰が承認待ちかすぐ分かる
  • 検索性向上: 過去の稟議を簡単に検索
  • ペーパーレス: 印刷・保管コスト削減
  • テンプレート機能: 標準フォーマットで入力ミス防止

6.10 稟議書作成の実践演習

実習 6-3: 総合演習

状況: あなたの部署で業務効率化のために新しいツールを導入したい。稟議書を作成してください。

考慮すべきポイント
  • 現状の問題点を具体的に（数値で示す）
  • 導入するツールの概要と選定理由
  • 必要な予算（初期費用+ランニングコスト）
  • 期待される効果（定量的+定性的）
  • 実施スケジュール
  • 想定リスクと対策
  • 代替案の検討
理解度確認クイズ
  1. 稟議書の主な目的を3つ挙げてください。
  2. 稟議書の基本構成要素を5つ以上挙げてください。
  3. 承認者が稟議書で確認する主なポイントを3つ挙げてください。
  4. 効果的な申請理由を書くための4つの要素は何ですか？
  5. 電子稟議システムのメリットを3つ挙げてください。
解答例
  1. 承認獲得、意思決定の記録、責任の明確化
  2. 件名、申請日、申請者、申請事項、申請理由、詳細内容、予算、期待効果、添付資料
  3. 必要性、妥当性、緊急性（効果、リスクも可）
  4. 現状の問題点、問題の影響、解決方法、代替案の検討
  5. 承認スピード向上、進捗状況の可視化、検索性向上
この章のまとめ
  • 稟議書は社内承認を得るための正式な文書
  • 申請事項、理由、詳細、予算、効果が基本構成
  • 承認者の視点（必要性・妥当性・効果）を意識する
  • 申請理由は論理的に、データを使って説明する
  • 費用対効果を数値で明確に示す
  • リスクと対策を記載して承認者の不安を軽減する
  • 代替案を比較し、推奨案の妥当性を示す
  • 電子稟議システムで承認プロセスを効率化できる
