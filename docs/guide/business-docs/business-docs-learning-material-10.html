10.1 総合演習：報告書作成

演習 10-1: 業務報告書の作成

シチュエーション

あなたは営業部の社員です。先週、A社との商談があり、その結果を上司に報告する業務報告書を作成してください。

背景情報

  • 日時: 2025年11月8日（金）14:00～15:30
  • 場所: A社本社（東京都千代田区）
  • 訪問者: あなた（営業担当）と上司の田中課長
  • 面談相手: A社 購買部長 山田様、担当者 佐藤様
  • 目的: 新製品XYZ-2000の提案
  • 結果: 好感触。来月中に正式見積を提出することに
  • 懸念事項: 予算が限られており、価格交渉が必要
  • 次回アクション: 11月20日までに見積書提出

作成のポイント

  • 5W1Hを意識した構成
  • 結論先行型（結果を最初に記載）
  • 客観的な事実と主観的な所見を区別
  • 次のアクションを明確に記載

10.2 総合演習：議事録作成

演習 10-2: 会議議事録の作成

シチュエーション

新製品開発プロジェクトの定例会議が開催されました。以下の情報をもとに議事録を作成してください。

会議情報

  • 会議名: 新製品開発プロジェクト定例会議
  • 日時: 2025年11月10日（月）10:00～11:30
  • 場所: 本社3階会議室B
  • 出席者: 企画部 鈴木部長、開発部 高橋課長、営業部 田中課長、あなた（記録係）
  • 欠席者: マーケティング部 伊藤課長（出張中）

議題と決定事項

  1. 開発進捗報告: プロトタイプ完成。テスト段階へ移行（担当: 高橋課長）
  2. 販売価格の検討: 原価を考慮し、希望小売価格を98,000円に設定（決定）
  3. 発売時期: 来年3月の発売を目指す。最終決定は次回会議で（継続審議）
  4. プロモーション戦略: Webサイトとカタログを12月中に準備（担当: 田中課長）

作成のポイント

  • 決定事項と継続審議事項を明確に区別
  • 担当者と期限を明記
  • 次回会議の日程を記載（11月24日 10:00～）
  • 簡潔かつ分かりやすい表現

10.3 総合演習：提案書作成

演習 10-3: 社内業務改善提案書の作成

シチュエーション

あなたは総務部の社員です。現在、社内の承認プロセスが紙ベースで非効率なため、電子承認システムの導入を提案します。

提案内容

  • 現状の課題:
    • 稟議書が紙で回覧され、承認に1週間以上かかる
    • 承認者不在時に業務が停滞
    • 書類の紛失リスク
    • 保管スペースの圧迫
  • 解決策: 電子承認システム導入
  • 期待効果:
    • 承認時間が平均3日に短縮
    • モバイルから承認可能
    • ペーパーレス化で年間コスト50万円削減
    • 承認履歴の自動記録
  • 導入費用: 初期費用200万円、月額利用料5万円
  • 投資回収期間: 約3年

作成のポイント

  • 現状分析→課題の明確化→解決策の提示→効果の定量化
  • 費用対効果を具体的な数字で示す
  • 図表やグラフを効果的に使用
  • 実施スケジュール案を提示

10.4 総合演習：稟議書作成

演習 10-4: 設備購入稟議書の作成

シチュエーション

あなたは情報システム部の担当者です。老朽化したサーバーを更新するため、新サーバー購入の稟議書を作成してください。

申請内容

  • 申請事項: 新サーバー購入
  • 現状: 現行サーバーは導入から7年経過、メーカーサポート終了
  • 必要性:
    • システム停止リスクが高い
    • セキュリティアップデート適用不可
    • 処理速度の低下で業務効率悪化
  • 見積金額: 350万円（本体300万円、設置・設定費用50万円）
  • 導入効果:
    • システム安定性向上
    • 処理速度3倍改善
    • 電気代年間20万円削減
  • 導入時期: 承認後2ヶ月以内

作成のポイント

  • 申請理由を論理的に説明
  • 承認者が判断しやすい情報を提供
  • リスクと対策を明示
  • 代替案も検討したことを示す

10.5 総合演習：ビジネスメール対応

演習 10-5: 複数のメール対応

以下のシチュエーションに対して、適切なビジネスメールを作成してください。

ケース1: 社内への会議招集メール

  • 明日の臨時会議を招集する
  • 議題: 来期予算の見直し
  • 日時: 11月12日（火）15:00～16:00
  • 対象: 営業部全員

ケース2: 取引先への資料送付メール

  • 先方から依頼された製品カタログをPDFで送付
  • ファイルサイズ: 3MB
  • 補足説明として、価格表も同封

ケース3: お客様からのクレーム対応メール

  • 納品した製品に不具合があったとの連絡
  • 誠意ある謝罪と迅速な対応を約束
  • 交換品を翌日発送することを伝える

作成のポイント

  • 件名は具体的で分かりやすく
  • 社内・社外で敬語を使い分ける
  • 必要な情報を漏れなく記載
  • 迅速な対応を心がける

10.6 ケーススタディ：トラブル対応文書

ケーススタディ 1: 納期遅延への対応

状況

あなたの会社は、B社に部品を納品する契約を結んでいます。しかし、製造ラインのトラブルにより、納期が5日遅れることが判明しました。B社は納期を重視しており、遅延により生産計画に影響が出る可能性があります。

課題

  1. B社への連絡をどのタイミングで、どのような形式で行うべきか？
  2. お詫びと状況説明をどのように伝えるか？
  3. 今後の信頼回復のために何を提案すべきか？

検討すべきポイント

  • 迅速な報告（問題発覚後すぐに連絡）
  • 電話→メール→正式な文書という段階的対応
  • 原因の明確な説明
  • 代替案の提示（可能であれば分納など）
  • 再発防止策の説明
  • 補償措置（割引、優先対応など）

演習

この状況に対する、①緊急連絡メール、②正式なお詫び状、③再発防止策を記載した報告書の3つを作成してください。

10.7 ケーススタディ：新規プロジェクト提案

ケーススタディ 2: 新規事業提案

状況

あなたは企画部に所属しています。最近、環境問題への関心が高まっていることに注目し、自社の強みを活かしたエコ製品の開発を提案したいと考えています。

背景情報

  • 自社は電子機器メーカー
  • 省エネ技術には定評がある
  • しかし、環境配慮型製品のラインナップは少ない
  • 競合他社は既にエコ製品を展開
  • 顧客調査では「環境に優しい製品」への関心が高い

提案内容

  • リサイクル素材を使用した新製品ライン
  • 既存製品の環境性能向上
  • エコマーク認証取得

検討すべきポイント

  • 市場動向の分析（データに基づく説明）
  • 競合分析（他社の動向）
  • 自社の強みと弱みの分析
  • 投資額と売上予測
  • 実施スケジュール
  • リスクと対策

演習

経営陣に提出する新規事業提案書を作成してください。PowerPoint形式で5～10スライド程度を想定。

10.8 ケーススタディ：お客様対応文書

ケーススタディ 3: 製品不具合への対応

状況

自社製品に設計上の欠陥が見つかり、すでに販売した1000台についてリコールを実施することになりました。お客様への告知と回収の案内文書を作成する必要があります。

対応方針

  • 全顧客に書面で通知
  • Webサイトでも告知
  • 無償で点検・修理または交換
  • お詫びの品を送付

法的要件

  • 消費者安全法に基づく報告義務
  • 不具合の内容を正確に説明
  • 危険性の有無を明示
  • 対応方法を明確に案内

演習

以下の文書を作成してください。

  1. 顧客向けリコール通知書
  2. プレスリリース（報道機関向け）
  3. 社内向け対応マニュアル
  4. コールセンター用Q&A集

10.9 文書作成スピードアップのテクニック

ビジネスの現場では、質を保ちながらスピーディーに文書を作成することが求められます。

flowchart TD A[文書作成の高速化] --> B[準備段階の工夫] A --> C[作成段階の工夫] A --> D[仕上げ段階の工夫] B --> B1[テンプレート活用] B --> B2[情報収集の効率化] B --> B3[構成の事前設計] C --> C1[一気に下書き] C --> C2[完璧を求めない] C --> C3[ショートカット活用] D --> D1[チェックリスト使用] D --> D2[他者レビュー依頼] D --> D3[推敲の時間確保]

スピードアップテクニック10選

No テクニック 説明
1 テンプレート活用 頻出文書はテンプレート化し、空欄を埋めるだけに
2 定型文の登録 よく使う挨拶文や表現を単語登録
3 構成の型を決める 結論→理由→詳細→結論などパターン化
4 箇条書きを活用 長文よりも箇条書きで要点を整理
5 一気に下書き 細部にこだわらず、まず全体を書き上げる
6 ショートカットキー Ctrl+C（コピー）、Ctrl+V（貼り付け）などを習得
7 音声入力の活用 長文はキーボードより音声入力が速い
8 参考文書の収集 過去の優れた文書を参考にする
9 集中時間の確保 メールやチャットを閉じて集中する
10 完璧を求めすぎない 80%の完成度で一度レビューを受ける

10.10 文書の推敲とレビュー技術

文書作成後の推敲（すいこう）は、品質を高める重要なプロセスです。

推敲の3段階

段階 チェック項目 方法
第1段階
（構成）		 ・論理展開は適切か
・結論は明確か
・必要な情報は全て含まれているか 		全体を俯瞰して読み直す
第2段階
（表現）		 ・分かりやすい表現か
・冗長な部分はないか
・適切な敬語が使われているか 		段落ごとに精読
第3段階
（形式）		 ・誤字脱字はないか
・数字や日付は正確か
・フォーマットは統一されているか 		音読または逆順チェック

効果的なレビューのポイント

  • 時間を置く: 作成直後ではなく、少し時間を置いてから見直す
  • 印刷して確認: 画面と紙では見え方が異なる
  • 音読する: 声に出して読むと不自然な箇所に気づく
  • 第三者に依頼: 他者の視点でチェックしてもらう
  • チェックリスト使用: 確認項目を漏れなくチェック
文書レビューチェックリスト
  • □ 件名・タイトルは内容を適切に表しているか
  • □ 宛先・差出人は正確か
  • □ 日付は正しいか
  • □ 結論が明確に述べられているか
  • □ 5W1Hが明示されているか
  • □ 論理展開に矛盾はないか
  • □ 事実と意見が区別されているか
  • □ 数字・固有名詞は正確か
  • □ 敬語は適切に使われているか
  • □ 誤字脱字はないか
  • □ フォーマットは統一されているか
  • □ 添付ファイルは正しいか

10.11 プロフェッショナルな文書作成者への道

ビジネス文書作成のスキルは、継続的な学習と実践によって磨かれます。

flowchart LR A[初心者] --> B[基礎習得] B --> C[実践経験] C --> D[スキル向上] D --> E[プロフェッショナル] B --> B1[テンプレート使用] C --> C1[多様な文書作成] D --> D1[独自スタイル確立] E --> E1[指導・改善提案]

成長段階と目標

レベル 特徴 目標
初心者 基本的なルールを学習中 テンプレートを使って正確に作成できる
中級者 様々な文書を作成できる 状況に応じて適切な文書を選択・作成できる
上級者 迅速かつ高品質な作成 他者の文書を改善できる
プロ 組織の文書品質向上に貢献 ルールや テンプレートを作成できる

10.12 継続的なスキル向上のためのヒント

日常的に実践すべきこと

  1. 良い文書を収集する
    • 社内の優れた文書を保存
    • 上司や先輩の文書を参考にする
    • 表現方法や構成を学ぶ
  2. フィードバックを求める
    • 上司や同僚にレビューを依頼
    • 改善点を素直に受け入れる
    • 同じミスを繰り返さない
  3. 定期的に見直す
    • 過去の文書を振り返る
    • 改善できる点を考える
    • テンプレートを更新する
  4. 関連書籍を読む
    • 文章術の本を定期的に読む
    • ビジネスマナー本で敬語を復習
    • 業界の動向を把握する
  5. 実践の機会を増やす
    • 積極的に文書作成を引き受ける
    • 新しいタイプの文書に挑戦
    • 社内勉強会を開催する
プロフェッショナルへの10のステップ
  1. 基本ルールを完璧にマスターする
  2. テンプレートを使いこなす
  3. 多様な文書作成を経験する
  4. フィードバックを積極的に求める
  5. 良い文書のサンプルを収集する
  6. 推敲の習慣を身につける
  7. 時間管理とスピードを意識する
  8. 他者の文書を添削する
  9. 組織の文書品質向上に貢献する
  10. 後輩の指導とテンプレート改善を行う
