第9章: 実践的なプログラム作成

これまでの学習内容を統合した実践プログラム

この章で学ぶこと
  • これまでの学習内容を統合したプログラムを作成する
  • 実践的なビジネスロジックを実装する
  • エラーハンドリングを含む堅牢なプログラムを作成する
  • 実際のビジネス要件に基づいたプログラムを設計する

9.1 総合的なプログラム設計

売上管理システムを例に、実践的なプログラムを作成します。

IDENTIFICATION DIVISION. PROGRAM-ID. SALES-MANAGEMENT-SYSTEM. AUTHOR. COBOL-LEARNER. DATE-WRITTEN. 2025-01-01. ENVIRONMENT DIVISION. DATA DIVISION. WORKING-STORAGE SECTION. 01 WS-SALES-DATA. 05 WS-SALES-RECORD OCCURS 100 TIMES. 10 WS-PRODUCT-CODE PIC X(5). 10 WS-PRODUCT-NAME PIC X(20). 10 WS-QUANTITY PIC 9(4). 10 WS-UNIT-PRICE PIC 9(6)V9(2). 10 WS-TOTAL-AMOUNT PIC 9(8)V9(2). 01 WS-SUMMARY-DATA. 05 WS-TOTAL-SALES PIC 9(10)V9(2) VALUE 0. 05 WS-RECORD-COUNT PIC 9(4) VALUE 0. 05 WS-AVERAGE-SALE PIC 9(8)V9(2) VALUE 0. 05 WS-MAX-SALE PIC 9(8)V9(2) VALUE 0. 05 WS-MIN-SALE PIC 9(8)V9(2) VALUE 99999999.99. 01 WS-CONTROL-VARS. 05 WS-INDEX PIC 9(4). 05 WS-MENU-CHOICE PIC 9(1). 05 WS-CONTINUE-FLAG PIC X(1) VALUE "Y". PROCEDURE DIVISION. MAIN-PARA. DISPLAY "売上管理システム開始". PERFORM INIT-SAMPLE-DATA. PERFORM UNTIL WS-CONTINUE-FLAG = "N" PERFORM DISPLAY-MENU PERFORM PROCESS-MENU-CHOICE END-PERFORM. DISPLAY "売上管理システム終了". STOP RUN. INIT-SAMPLE-DATA. MOVE "P0001" TO WS-PRODUCT-CODE(1). MOVE "商品A" TO WS-PRODUCT-NAME(1). MOVE 50 TO WS-QUANTITY(1). MOVE 1500.00 TO WS-UNIT-PRICE(1). PERFORM CALC-TOTAL-AMOUNT VARYING WS-INDEX FROM 1 BY 1 UNTIL WS-INDEX > 5. MOVE 5 TO WS-RECORD-COUNT. CALC-TOTAL-AMOUNT. COMPUTE WS-TOTAL-AMOUNT(WS-INDEX) = WS-QUANTITY(WS-INDEX) * WS-UNIT-PRICE(WS-INDEX). DISPLAY-MENU. DISPLAY "=== メニュー ===". DISPLAY "1. 売上一覧表示". DISPLAY "2. 売上統計". DISPLAY "3. 終了". DISPLAY "選択してください (1-3): ". ACCEPT WS-MENU-CHOICE. PROCESS-MENU-CHOICE. EVALUATE WS-MENU-CHOICE WHEN 1 PERFORM DISPLAY-SALES-LIST WHEN 2 PERFORM DISPLAY-STATISTICS WHEN 3 MOVE "N" TO WS-CONTINUE-FLAG WHEN OTHER DISPLAY "無効な選択です" END-EVALUATE.

9.2 エラーハンドリング

堅牢なプログラムのためのエラーハンドリングを実装します。

01 WS-ERROR-HANDLING. 05 WS-ERROR-CODE PIC 9(4). 05 WS-ERROR-MESSAGE PIC X(50). 05 WS-ERROR-FLAG PIC X(1) VALUE "N". VALIDATE-INPUT. MOVE "N" TO WS-ERROR-FLAG. IF WS-QUANTITY(WS-INDEX) <= 0 MOVE 1001 TO WS-ERROR-CODE MOVE "数量は正の値である必要があります" TO WS-ERROR-MESSAGE MOVE "Y" TO WS-ERROR-FLAG END-IF. IF WS-UNIT-PRICE(WS-INDEX) <= 0 MOVE 1002 TO WS-ERROR-CODE MOVE "単価は正の値である必要があります" TO WS-ERROR-MESSAGE MOVE "Y" TO WS-ERROR-FLAG END-IF. HANDLE-ERROR. IF WS-ERROR-FLAG = "Y" DISPLAY "エラー [" WS-ERROR-CODE "]: " WS-ERROR-MESSAGE DISPLAY "処理をスキップします" END-IF.

9.3 実践課題

学習内容を活用した実践的な課題に取り組みます。

総合演習: 在庫管理システム

これまでの学習内容を活用して、在庫管理システムを作成してください。

要件
  • 商品マスタ（商品コード、商品名、在庫数、単価）
  • 入庫・出庫処理
  • 在庫一覧表示
  • 在庫統計（合計在庫数、平均単価等）
  • エラーハンドリング（負の数値チェック等）
使用する技術
  • 配列（OCCURS句）
  • 条件分岐（IF文、EVALUATE文）
  • ループ処理（PERFORM文）
  • 計算処理（COMPUTE文）
  • サブプログラム（段落の分割）
理解度確認クイズ
  1. 実践的なプログラムを作成する際の重要な要素を3つ挙げてください。
  2. エラーハンドリングが必要な理由を説明してください。
  3. プログラムの保守性を高めるための方法を説明してください。
この章のまとめ

この章では、これまでの学習内容を統合した実践的なプログラムの作成方法を学習しました。実際のビジネス要件に基づいたシステム設計、エラーハンドリング、保守性の高いプログラムの作成方法を理解しました。最終章では、学習内容の総復習と今後の学習方向性について学習します。
