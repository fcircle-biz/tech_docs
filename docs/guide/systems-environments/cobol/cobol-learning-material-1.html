1.1 COBOLとは

COBOL（Common Business Oriented Language）は、1959年に開発された、ビジネス分野向けのプログラミング言語です。

COBOLの名前の由来

COBOL = COmmon Business Oriented Language

「共通のビジネス指向言語」という意味で、ビジネス処理に特化した言語として設計されました。

1.2 COBOLの歴史と特徴

歴史

  • 1959年: アメリカ国防総省の主導で開発開始
  • 1960年: 初版仕様書が発表
  • 1968年: ANSI規格として標準化
  • 現在: 金融機関、政府機関で広く使用

主な特徴

🔍 英語に近い構文

人間が読みやすい英語に近い記述方式

💰 ビジネス処理に特化

金融計算、データ処理に最適化

🛡️ 高い信頼性

長期間の安定動作が可能

📊 大量データ処理

膨大なデータの処理に優れている

1.3 VSCode環境の設定

Visual Studio CodeでCOBOLプログラムを開発するための環境を設定します。

VSCode COBOL開発環境の設定手順

  1. Visual Studio Codeをダウンロード・インストール
  2. VSCode拡張機能で「COBOL」を検索し、bitlang.cobolをインストール
  3. GnuCOBOLをインストール（Windows: 公式サイトからダウンロード）
  4. 新しいCOBOLファイル（.cob または .cbl）を作成
COBOL開発時の注意点
  • COBOLは固定形式で記述する必要があります
  • 7桁目から72桁目の間に命令を記述します
  • 1-6桁目は行番号、73-80桁目は識別エリアです
  • VSCodeのCOBOL拡張機能により、構文ハイライトとエラーチェックが可能
GnuCOBOLの基本的な使用方法
  • コンパイル: cobc -x ファイル名.cob
  • 実行: ./ファイル名 (Windows: ファイル名.exe)
  • デバッグ: cobc -x -g ファイル名.cob でデバッグ情報付きコンパイル

1.4 初めてのCOBOLプログラム

最初のCOBOLプログラムとして、「Hello, World!」を出力するプログラムを作成します。

IDENTIFICATION DIVISION. PROGRAM-ID. HELLO-WORLD. ENVIRONMENT DIVISION. DATA DIVISION. PROCEDURE DIVISION. MAIN-PARA. DISPLAY "Hello, World!". STOP RUN.

プログラムの構造説明

  • IDENTIFICATION DIVISION: プログラムの識別情報
  • PROGRAM-ID: プログラム名を定義
  • ENVIRONMENT DIVISION: 環境設定（今回は空）
  • DATA DIVISION: データ定義（今回は空）
  • PROCEDURE DIVISION: 実行する処理
  • DISPLAY: 文字列を画面に表示
  • STOP RUN: プログラムの終了
実習 1-1: Hello Worldプログラムの実行

上記のプログラムをVSCodeで実際に作成・実行してみましょう。

手順
  1. VSCodeで新しいファイルを作成し、「hello-world.cob」として保存
  2. 上記のコードをエディタに入力する
  3. ターミナルで「cobc -x hello-world.cob」を実行してコンパイル
  4. 「./hello-world」（Windows: hello-world.exe）を実行して結果を確認
期待される結果
Hello, World!
実習 1-2: メッセージを変更してみよう

DISPLAYで表示するメッセージを変更してみましょう。

課題
  1. 「Hello, World!」を「Hello, COBOL!」に変更
  2. 自分の名前を表示するプログラムを作成
手順
  1. 新しいファイル「hello-cobol.cob」を作成
  2. DISPLAY文の文字列部分を変更
  3. コンパイル・実行して結果を確認
ヒント

DISPLAY文の文字列部分を変更するだけです。VSCodeの拡張機能により、構文エラーがあれば赤い波線で表示されます。
