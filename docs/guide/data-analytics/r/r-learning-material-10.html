10.1 R Markdownとは

R Markdownは、テキスト、Rコード、出力結果を一つの文書に統合できる強力なツールです。

R Markdownの利点

  • 再現可能性: コードと結果が一体化
  • 効率性: 分析からレポートまで一貫した作業流れ
  • 柔軟性: 複数の出力形式に対応
  • 共有性: バージョン管理が容易

必要パッケージのインストール

# R Markdownと関連パッケージ
install.packages(c("rmarkdown", "knitr", "tinytex"))

# PDF出力用（LaTeX環境）
tinytex::install_tinytex()

# 表作成用
install.packages(c("kableExtra", "DT", "flextable"))

10.3 Rコードチャンク


チャンクオプション

# コードを表示せず、結果のみ表示
```{r echo=FALSE}
plot(iris$Sepal.Length, iris$Sepal.Width)
```



実習 10-2: 美しい表とグラフの作成

kableExtraパッケージを使用して、プロフェッショナルな表を作成しましょう。

実行例
```{r}
library(kableExtra)
library(dplyr)

# 美しい表の作成
mtcars %>%
  head(10) %>%
  kable(caption = "自動車データ（上位10台）", digits = 2) %>%
  kable_styling(bootstrap_options = c("striped", "hover", "condensed")) %>%
  add_header_above(c(" " = 1, "燃費・性能" = 4, "車体" = 3, "その他" = 3)) %>%
  column_spec(2, bold = T, color = "white", background = "dodgerblue")
```

```{r fig.cap="車重と燃費の関係"}
library(ggplot2)
ggplot(mtcars, aes(x = wt, y = mpg)) +
  geom_point(aes(color = factor(cyl)), size = 3) +
  geom_smooth(method = "lm", se = FALSE) +
  labs(title = "Vehicle Weight vs Fuel Efficiency",
       x = "Weight (1000 lbs)", y = "Miles per Gallon",
       color = "Cylinders") +
  theme_minimal()
```

10.4 出力形式のカスタマイズ

YAML ヘッダーの設定

---
title: "データ分析レポート"
author: "分析者名"

output:
  html_document:
    toc: true
    toc_float: true
    theme: flatly
    highlight: tango
    fig_caption: yes
    code_folding: hide
  pdf_document:
    toc: true
    number_sections: true
    fig_caption: yes
  word_document:
    toc: true
    fig_caption: yes
---
