第1章: ASP.NET Web Forms入門と開発環境構築

ASP.NET Web Forms入門と開発環境構築

この章で学ぶこと

  • コンソールアプリケーションとWebアプリケーションの違いを学ぶ
  • Visual StudioでのWeb Forms開発環境を構築する

📋 前提条件の確認

この章を始める前に、VB.NET学習ガイドラインを完了していることを確認してください。特に以下の知識が必要です：

  • VB.NETの基本文法とオブジェクト指向プログラミング
  • 例外処理とファイル操作
  • クラス、プロパティ、メソッドの理解

1.1 ASP.NET Web Formsとは

ASP.NET Web Formsは、Microsoftが開発したWebアプリケーション開発フレームワークです。VB.NETで学習したオブジェクト指向プログラミングの知識を活用して、動的なWebサイトを構築できます。

Web Formsの特徴

  • イベントドリブンモデル: Windows Formsと同様のイベント処理方式
  • Webコントロール: HTML要素を抽象化したサーバーサイドコントロール
  • PostBack機能: サーバーとの双方向通信を自動化
  • 状態管理: ViewState等によるサーバー状態の保持

1.2 コンソールアプリケーションとWebアプリケーションの比較

VB.NETで学習したコンソールアプリケーションとWebアプリケーションの主な違いを理解しましょう。

比較項目 コンソールアプリケーション Webアプリケーション
実行環境 ローカルコンピュータ Webサーバー（IIS等）
ユーザーインターフェース コマンドライン Webブラウザ
入出力方式 Console.ReadLine/WriteLine HTML要素（フォーム等）
プロセスモデル 単一プロセス、連続実行 リクエスト/レスポンス、状態なし
配布方法 実行ファイル(.exe) Webサーバーへのデプロイ
アクセス方式 直接実行 URL経由でアクセス

1.3 Web開発の基本概念

Webアプリケーション開発で理解すべき基本概念を学習します。

クライアント・サーバーモデル

  • クライアント（ブラウザ）: ユーザーが操作するWebブラウザ
  • サーバー（IIS + .NET）: VB.NETコードが実行されるWebサーバー
  • HTTP通信: クライアントとサーバー間の通信プロトコル

リクエスト/レスポンスサイクル

  1. ユーザーがブラウザでURLにアクセス（HTTPリクエスト）
  2. サーバーでVB.NETコードが実行
  3. HTMLが生成されブラウザに送信（HTTPレスポンス）
  4. ブラウザがHTMLを解釈して画面に表示

1.5 初回Web Formsプロジェクトの作成

Visual StudioでASP.NET Web Formsプロジェクトを作成し、基本的な仕組みを理解しましょう。

実習 1-2: Hello World Web Formsアプリケーションの作成

VB.NETで作成した初回のコンソールアプリケーションと同様に、Web版の「Hello World」を作成します。

プロジェクト作成手順
  1. Visual Studioを起動し、「新しいプロジェクトの作成」を選択
  2. 「ASP.NET Web アプリケーション (.NET Framework)」を選択
  3. プロジェクト名: HelloWorldWebApp
  4. 場所: 適切なフォルダを選択
  5. フレームワーク: .NET Framework 4.7.2 以上を選択
  6. 「Web Forms」テンプレートを選択
  7. 「認証なし」を選択
  8. 「作成」をクリック
プロジェクト構造の確認

作成されたプロジェクトの主要ファイルを確認しましょう：

  • Default.aspx: Webページのマークアップ（HTML + ASP.NET）
  • Default.aspx.vb: VB.NETコードビハインド
  • Site.Master: マスターページ（共通レイアウト）
  • Global.asax: アプリケーション全体の設定
  • Web.config: Webアプリケーションの設定ファイル
Hello Worldページの作成
  1. Default.aspxを開く
  2. 以下のコードに修正：
Default.aspx（マークアップ）
<%@ Page Title="Hello World" Language="VB" MasterPageFile="~/Site.Master" AutoEventWireup="true" CodeBehind="Default.aspx.vb" Inherits="HelloWorldWebApp._Default" %>

<asp:Content ID="BodyContent" ContentPlaceHolderID="MainContent" runat="server">
    <div class="jumbotron">
        <h1>ASP.NET Web Forms へようこそ！</h1>
        <p>
            <asp:Label ID="lblMessage" runat="server" Text="" CssClass="lead"></asp:Label>
        </p>
        <p>
            <asp:Button ID="btnShowMessage" runat="server" Text="メッセージを表示" CssClass="btn btn-primary btn-lg" />
        </p>
        <hr />
        <p>
            お名前: <asp:TextBox ID="txtName" runat="server" placeholder="お名前を入力してください"></asp:TextBox>
            <br /><br />
            <asp:Button ID="btnGreet" runat="server" Text="挨拶する" CssClass="btn btn-success" />
        </p>

Default.aspx.vb（コードビハインド）
Public Class _Default
    Inherits System.Web.UI.Page

    Protected Sub Page_Load(ByVal sender As Object, ByVal e As System.EventArgs) Handles Me.Load
        ' ページが読み込まれたときの処理
        If Not IsPostBack Then
            lblMessage.Text = "VB.NETからWeb Formsへようこそ！現在時刻: " & DateTime.Now.ToString("yyyy年MM月dd日 HH:mm:ss")
        End If
    End Sub

    Protected Sub btnShowMessage_Click(sender As Object, e As EventArgs) Handles btnShowMessage.Click
        ' メッセージ表示ボタンがクリックされた時の処理
        lblMessage.Text = "ボタンがクリックされました！時刻: " & DateTime.Now.ToString("HH:mm:ss")
        lblMessage.CssClass = "lead text-success"
    End Sub

    Protected Sub btnGreet_Click(sender As Object, e As EventArgs) Handles btnGreet.Click
        ' 挨拶ボタンがクリックされた時の処理
        Dim name As String = txtName.Text.Trim()
        
        If String.IsNullOrEmpty(name) Then
            lblGreeting.Text = "お名前を入力してください。"
            lblGreeting.CssClass = "alert alert-warning"
        Else
            lblGreeting.Text = $"こんにちは、{name}さん！Web Formsへようこそ！"
            lblGreeting.CssClass = "alert alert-success"
        End If
    End Sub
End Class
実行とテスト
  1. F5キーを押してデバッグ実行
  2. ブラウザが自動的に開き、Webページが表示される
  3. 「メッセージを表示」ボタンをクリックして動作確認
  4. 名前を入力して「挨拶する」ボタンをクリック
  5. ブラウザの開発者ツール（F12）でHTMLソースを確認
期待される結果

  • ボタンクリックでメッセージが動的に変更される
  • テキストボックスに入力した名前を使った挨拶が表示される

1.6 Web Forms固有の概念

コンソールアプリケーションにはない、Web Forms固有の重要な概念を理解しましょう。

ページライフサイクル

Web Formsページは、リクエストごとに以下のライフサイクルを経ます：

  1. Page_PreInit: ページの初期化前
  2. Page_Init: ページの初期化
  3. Page_Load: ページの読み込み（最もよく使用）
  4. Page_PreRender: レンダリング前
  5. Page_Unload: ページのアンロード

IsPostBackプロパティ

' 初回アクセス時のみ実行する処理
If Not IsPostBack Then
    ' 初期データの設定等
    txtName.Text = ""
    lblMessage.Text = "ページが初回読み込みされました"
End If

サーバーコントロール vs HTMLコントロール

サーバーコントロール HTMLコントロール
<asp:Label runat="server" /> <span></span>
<asp:TextBox runat="server" /> <input type="text" />
<asp:Button runat="server" /> <input type="submit" />
VB.NETで直接操作可能 JavaScriptまたはrunat="server"が必要

1.7 デバッグとトラブルシューティング

Web Formsアプリケーションのデバッグ方法を学習します。

Visual Studioでのデバッグ

  • ブレークポイント: コードビハインドにF9でブレークポイント設定
  • ステップ実行: F10（ステップオーバー）、F11（ステップイントゥ）
  • 変数の監視: ウォッチウィンドウで変数の値を確認
  • 即座ウィンドウ: デバッグ中にVB.NETコードを実行

よくあるエラーと対処法

  • 「System.NullReferenceException」: コントロールがNull（Nothing）
  • 「コンパイルエラー」: VB.NET構文エラー
  • 「HTTP 404 エラー」: ページが見つからない
  • 「HTTP 500 エラー」: サーバーサイドでの実行時エラー
⚠️ Web開発での注意点
  • 各リクエストで新しいページインスタンスが作成される
  • 変数の値はリクエスト間で保持されない（状態管理が必要）
  • Console.WriteLineは使用できない（デバッグ出力にはDebug.WriteLineを使用）
  • ブラウザのキャッシュが原因で変更が反映されない場合がある
