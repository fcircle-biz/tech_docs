8.1 データベース

関係データベース（RDB）

関係データベースは、データを表（テーブル）形式で管理するデータベースです。

基本用語
  • テーブル（表）：データを格納する表
  • 行（レコード、タプル）：1件のデータ
  • 列（フィールド、属性）：データ項目
  • 主キー：レコードを一意に識別する列
  • 外部キー：他のテーブルの主キーを参照する列

SQL（Structured Query Language）

SQLは、データベースを操作するための言語です。

主なSQL文
分類 命令 用途
データ操作 SELECT、INSERT、UPDATE、DELETE データの検索・追加・更新・削除
データ定義 CREATE、ALTER、DROP テーブルの作成・変更・削除
データ制御 GRANT、REVOKE 権限の付与・剥奪

トランザクション処理

トランザクションは、複数の処理をまとめた作業単位です。

ACID特性
  • 原子性（Atomicity）：全て成功するか、全て失敗するか
  • 一貫性（Consistency）：データの整合性が保たれる
  • 独立性（Isolation）：他のトランザクションの影響を受けない
  • 永続性（Durability）：確定したデータは永続的に保存される
トランザクション制御
  • COMMIT：変更を確定
  • ROLLBACK：変更を取り消し

データベースの正規化

正規化は、データの冗長性を排除し、整合性を保つためにテーブルを分割する手法です。

  • 第1正規形：繰り返し項目をなくす
  • 第2正規形：部分関数従属をなくす
  • 第3正規形：推移的関数従属をなくす

8.2 ネットワーク

ネットワークの種類

  • LAN（Local Area Network）：建物内などの狭い範囲のネットワーク
  • WAN（Wide Area Network）：広範囲のネットワーク
  • インターネット：世界規模のネットワーク

ネットワークトポロジ

  • スター型：中心のハブに全端末を接続
  • バス型：1本のケーブルに複数の端末を接続
  • リング型：端末を環状に接続

IPアドレス

IPアドレスは、ネットワーク上の機器を識別するための番号です。

IPv4

32ビット（4バイト）、ドット区切りの10進数で表記（例：192.168.1.1）

IPv6

128ビット（16バイト）、コロン区切りの16進数で表記（例：2001:0db8:85a3:0000:0000:8a2e:0370:7334）

プライベートIPアドレス
  • 10.0.0.0 ～ 10.255.255.255
  • 172.16.0.0 ～ 172.31.255.255
  • 192.168.0.0 ～ 192.168.255.255

サブネットマスク

サブネットマスクは、IPアドレスのネットワーク部とホスト部を区別するためのマスクです。

例：255.255.255.0（ネットワーク部24ビット、ホスト部8ビット）

プロトコル

プロトコルは、通信の規約です。

プロトコル 用途
HTTP/HTTPS Webページの閲覧
FTP ファイル転送
SMTP メール送信
POP3/IMAP メール受信
DNS ドメイン名とIPアドレスの変換
DHCP IPアドレスの自動割り当て

ネットワーク機器

  • リピータ：信号を増幅
  • ハブ：複数の機器を接続（全ポートに転送）
  • スイッチ：MACアドレスで宛先を判断（該当ポートのみ転送）
  • ルータ：異なるネットワークを接続、IPアドレスで経路制御
  • ゲートウェイ：異なるプロトコル間の変換

無線LAN

  • Wi-Fi：無線LAN規格（IEEE 802.11a/b/g/n/ac/ax）
  • SSID：無線LANのネットワーク識別子
  • 暗号化：WPA2、WPA3（WEPは脆弱で非推奨）

ゼロトラストネットワーク

ゼロトラストネットワークは、「信頼しない、必ず検証する（Never Trust, Always Verify）」を基本原則とした、最新のセキュリティモデルです。

ゼロトラストとは

従来の「社内ネットワークは安全」という前提を捨て、すべてのアクセスを疑い、常に認証・認可を行うセキュリティアプローチです。リモートワークやクラウド利用の増加により、境界型セキュリティだけでは不十分になったことが背景にあります。

ゼロトラストの基本原則
  • すべてのリソースへのアクセスを認証・認可する：社内からでも外部からでも、すべてのアクセスを検証
  • 最小権限の原則：必要最小限の権限のみを付与
  • 侵害を前提とした設計：攻撃者の侵入を前提に、被害を最小化
  • 継続的な監視と検証：アクセス後も継続的に行動を監視
ゼロトラストの主要コンポーネント

  1. ID認証とアクセス制御
    • 多要素認証（MFA）の導入
    • シングルサインオン（SSO）
    • ID・アクセス管理（IAM）システム
    • ユーザーごとの詳細な権限設定
  2. デバイス認証
    • デバイスの正当性確認（証明書ベース）
    • セキュリティ状態のチェック（パッチ適用状況など）
    • マネージドデバイスの登録管理
  3. マイクロセグメンテーション
    • ネットワークを細かく分割
    • セグメント間の通信を厳密に制御
    • 侵害された場合の影響範囲を限定
  4. 継続的な監視と検証
    • リアルタイムでのログ収集・分析
    • 異常なアクセスパターンの検知
    • 定期的な権限の見直し
    • 行動分析による脅威検出
ゼロトラストのメリット
メリット 説明
セキュリティの強化 すべてのアクセスを検証するため、不正アクセスのリスクが低減
リモートワーク対応 場所を問わず安全にアクセス可能（VPN不要でもセキュア）
内部脅威対策 社内からのアクセスも検証するため、内部不正を防止
侵害範囲の限定 マイクロセグメンテーションにより、攻撃者の横展開を防ぐ
クラウド対応 クラウドサービス利用時も一貫したセキュリティポリシーを適用
コンプライアンス 詳細なログと監視により、規制要件への対応が容易
実装の課題
ゼロトラスト導入時の主な課題
  • 導入コスト：新しいシステムやツールの導入に初期投資が必要
  • 複雑性の増加：認証・認可の仕組みが複雑になり、管理負担が増加
  • ユーザー体験：頻繁な認証がユーザーの利便性を低下させる可能性
  • レガシーシステムの対応：古いシステムがゼロトラストに対応していない場合がある
  • 段階的な移行：一度に全てを変更するのは困難なため、段階的なアプローチが必要
  • スキル不足：専門知識を持つ人材の確保・育成が課題
ITパスポート試験での重要ポイント
  • ゼロトラストの基本原則「信頼しない、必ず検証する」を理解する
  • 従来の境界型セキュリティとの違いを説明できる
  • リモートワークやクラウド環境に適したモデルであることを理解する
  • 主要な技術要素（多要素認証、マイクロセグメンテーション、継続的監視）を把握する

8.3 ヒューマンインターフェース

GUI（Graphical User Interface）

GUIは、アイコンやウィンドウを使った直感的な操作画面です。

GUIの要素
  • ウィンドウ：作業領域
  • アイコン：機能を表す小さな絵
  • メニュー：機能の一覧
  • ダイアログボックス：ユーザーとの対話

ユニバーサルデザイン

ユニバーサルデザインは、すべての人が使いやすいデザインです。

7原則
  1. 公平な利用
  2. 利用における柔軟性
  3. 簡単で直感的な利用
  4. 分かりやすい情報
  5. 間違いに対する寛大さ
  6. 少ない身体的負担
  7. 接近や使用のためのサイズと空間

アクセシビリティ

アクセシビリティは、高齢者や障がい者も利用しやすくする配慮です。

  • 音声読み上げ：視覚障がい者向け
  • 字幕表示：聴覚障がい者向け
  • 配色の工夫：色覚特性への配慮
  • 拡大表示：視力低下への対応

マルチメディア

画像データ
  • ビットマップ（ラスタ）：ピクセルの集合（JPEG、PNG、GIF）
  • ベクタ：図形の数式で表現（SVG）
画像の解像度

解像度は、画像の細かさを表す（dpi：dots per inch）

音声データ
  • サンプリング：アナログ音声をデジタル化
  • サンプリング周波数：1秒間に何回測定するか（Hz）
  • 量子化ビット数：音の大きさの精度

    • A. 信号の増幅

  6. ゼロトラストネットワークの基本原則として正しいものはどれか？
    • A. 社内ネットワークは常に信頼する
    • B. ファイアウォールの内側は安全とみなす
    • C. すべてのアクセスを疑い、常に検証する
    • D. 一度認証すれば再認証は不要
    解答を見る

    正解：C
    ゼロトラストは「信頼しない、必ず検証する（Never Trust, Always Verify）」が基本原則です。社内・社外を問わず、すべてのアクセスを検証します。

  7. ゼロトラストネットワークの主要コンポーネントでないものはどれか？
    • A. 多要素認証
    • B. マイクロセグメンテーション
    • C. 継続的な監視と検証
    • D. 境界ファイアウォールのみでの防御
    解答を見る

    正解：D
