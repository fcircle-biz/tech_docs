第5章

システムの導入

システム構成、導入の流れ、障害対策、システム監査について学習します。

この章で学ぶこと

  • デュアルシステム、デュプレックスシステム、仮想化などのシステム構成を理解する
  • RFI、RFPなどシステム導入の流れと用語を学ぶ
  • 稼働率、MTBF、MTTRの計算方法とバックアップ戦略を習得する
  • システム監査の目的と監査プロセスを理解する

5.1 システム構成

システムの信頼性と可用性を高めるため、様々なシステム構成方法があります。ここでは代表的な構成方式を学習します。

デュアルシステム

デュアルシステムは、同じ処理を2つのシステムで同時に実行し、結果を照合する方式です。一方のシステムに障害が発生しても、もう一方のシステムで処理を継続できます。主に金融機関など高い信頼性が求められる分野で使用されます。

graph LR Input["入力"] --> SystemA["システムA"] Input --> SystemB["システムB"] SystemA --> Compare["結果照合"] SystemB --> Compare Compare --> Output["出力"]

デュプレックスシステム

デュプレックスシステムは、通常時は主系システムが稼働し、障害発生時に待機系システムに切り替わる方式です。待機系システムには、常時稼働しているホットスタンバイと、障害時に起動するコールドスタンバイがあります。

方式 特徴 切替時間
ホットスタンバイ 待機系システムも常時稼働している 短い
コールドスタンバイ 障害発生時に待機系システムを起動 長い

仮想化

仮想化は、1台の物理サーバ上で複数の仮想サーバを稼働させる技術です。ハードウェア資源を効率的に利用でき、サーバの集約やコスト削減が可能になります。代表的な仮想化技術には、ハイパーバイザ型とコンテナ型があります。

重要ポイント

仮想化により、サーバ台数の削減、リソースの有効活用、柔軟なシステム構成変更が可能になります。

5.2 システム導入の流れ

企業がシステムを導入する際には、計画的なプロセスを経る必要があります。ここでは、システム導入の主要なステップと関連用語を学習します。

（提案依頼）"] C --> D["ベンダ選定"] D --> E["契約"] E --> F["システム構築"] F --> G["導入・稼働"]

RFI（Request for Information）

RFI（情報提供依頼）は、システム導入の初期段階でベンダに対して製品情報や技術情報の提供を求める文書です。市場調査や要件定義の参考情報を収集するために使用されます。

RFP（Request for Proposal）

RFP（提案依頼書）は、具体的なシステム要件をベンダに示し、提案を求める文書です。RFPには、業務要件、機能要件、非機能要件、予算、スケジュールなどが記載されます。

RFPの主な記載項目

  • システムの目的と背景
  • 現状の課題と要求事項
  • 機能要件・非機能要件
  • 予算とスケジュール
  • 評価基準

SLA（Service Level Agreement）

SLA（サービスレベル合意書）は、サービス提供者と利用者の間でサービスレベルを合意する文書です。稼働率、応答時間、障害対応時間などの指標が定められます。

5.3 システムの障害対策

システムの信頼性を確保するため、障害対策と性能評価の指標を理解することが重要です。

稼働率

稼働率は、システムが正常に稼働している時間の割合を示す指標です。次の式で計算されます。

稼働率の計算式

稼働率 = 稼働時間 ÷ (稼働時間 + 停止時間)

例: 1ヶ月（720時間）のうち、2時間停止した場合
稼働率 = 718 ÷ 720 = 0.9972 = 99.72%

複数のシステムを直列接続する場合、全体の稼働率は各システムの稼働率の積になります。並列接続する場合は、全体の稼働率は高くなります。

接続方式 計算式 例
直列接続 R = R1 × R2 0.9 × 0.9 = 0.81（81%）
並列接続 R = 1 - (1-R1) × (1-R2) 1 - 0.1 × 0.1 = 0.99（99%）

MTBF（Mean Time Between Failures）

MTBF（平均故障間隔）は、システムが故障してから次の故障までの平均時間を表します。MTBFが長いほど、システムの信頼性が高いことを示します。

MTTR（Mean Time To Repair）

MTTR（平均修理時間）は、システムが故障してから修復するまでの平均時間を表します。MTTRが短いほど、障害からの復旧が速いことを示します。

稼働率とMTBF、MTTRの関係

稼働率 = MTBF ÷ (MTBF + MTTR)

バックアップ

バックアップは、データの消失や破損に備えてデータの複製を作成することです。主なバックアップ方式には以下があります。

方式 特徴 時間
フルバックアップ すべてのデータをバックアップ 長い
差分バックアップ 前回のフルバックアップからの変更分 中程度
増分バックアップ 前回のバックアップからの変更分 短い

5.4 システム監査

システム監査は、情報システムの信頼性、安全性、効率性を客観的に評価する活動です。

システム監査の目的

システム監査の主な目的は以下の通りです。

システム監査の目的

  • 情報システムの信頼性の確保
  • 情報セキュリティの確保
  • 業務の効率性と有効性の評価
  • 法令遵守の確認

監査人の独立性

システム監査では、監査人の独立性が重要です。監査対象部門から独立した立場で、客観的かつ公正に監査を実施する必要があります。

注意

監査人が監査対象部門の業務に関与している場合、独立性が損なわれます。

監査のプロセス

システム監査は、計画、実施、報告、フォローアップの4つのフェーズから成ります。

flowchart LR A["監査計画"] --> B["監査実施"] B --> C["監査報告"] C --> D["フォローアップ"]
フェーズ 内容
監査計画 監査の目的、範囲、スケジュールを決定
監査実施 証拠の収集、分析、評価を実施
監査報告 監査結果を報告書にまとめて経営層に報告
フォローアップ 指摘事項の改善状況を確認

重要ポイント

フォローアップは、監査で指摘された問題が適切に改善されたかを確認する重要なプロセスです。

理解度確認クイズ

  1. Q1. 同じ処理を2つのシステムで同時に実行し、結果を照合する方式は何か。

    回答を表示

    解答: デュアルシステム
    解説: デュアルシステムは、同じ処理を2つのシステムで同時に実行し、結果を照合することで高い信頼性を実現します。

  2. Q2. システム導入時に、ベンダに対して具体的な要件を示して提案を求める文書は何か。

    回答を表示

    解答: RFP（Request for Proposal / 提案依頼書）
    解説: RFPは、具体的なシステム要件をベンダに示し、提案を求める文書です。RFIは情報提供依頼で、初期段階で使用されます。

  3. Q3. システムが故障してから次の故障までの平均時間を表す指標は何か。

    回答を表示

    解答: MTBF（Mean Time Between Failures / 平均故障間隔）
    解説: MTBFが長いほど、システムの信頼性が高いことを示します。MTTRは平均修理時間です。

  4. Q4. 稼働率0.9のシステムを直列接続した場合の全体の稼働率はいくつか。

    回答を表示

    解答: 0.81（81%）
    解説: 直列接続の稼働率は各システムの稼働率の積なので、0.9 × 0.9 = 0.81です。

  5. Q5. システム監査で最も重要な原則は何か。

    回答を表示

    解答: 監査人の独立性
    解説: 監査人は監査対象部門から独立した立場で、客観的かつ公正に監査を実施する必要があります。
