情報セキュリティ

この章で学ぶこと

  • 脆弱性診断とペネトレーションテストの違いを知る

9.1 情報セキュリティの基礎

情報セキュリティの3要素（CIA）

情報セキュリティは、以下の3つの要素で構成されます。

  • 機密性（Confidentiality）：許可された者だけがアクセスできる
  • 完全性（Integrity）：情報が正確で改ざんされていない
  • 可用性（Availability）：必要な時に情報にアクセスできる
真正性と否認防止

3要素に加えて、以下も重要です：

  • 真正性（Authenticity）：本人であることを証明
  • 否認防止（Non-repudiation）：後から否定できない

9.2 脅威と脆弱性

マルウェア（Malware）

マルウェアは、悪意のあるソフトウェアの総称です。

種類 特徴
ウイルス 他のプログラムに寄生して自己増殖
ワーム 単独で自己増殖、ネットワーク経由で拡散
トロイの木馬 正常なソフトに偽装、バックドアを作成
ランサムウェア データを暗号化して身代金を要求
スパイウェア 情報を盗み出す
ボット 外部から遠隔操作される

サイバー攻撃

DoS攻撃・DDoS攻撃

DoS（Denial of Service）攻撃は、大量のアクセスでサーバを停止させる攻撃です。

  • DoS攻撃：1台から攻撃
  • DDoS攻撃：複数台から同時攻撃
SQLインジェクション

入力フォームに不正なSQL文を入力して、データベースを不正操作する攻撃です。

クロスサイトスクリプティング（XSS）

Webサイトに悪意のあるスクリプトを埋め込み、訪問者の情報を盗む攻撃です。

フィッシング

本物そっくりの偽サイトに誘導し、個人情報を入力させる詐欺です。

標的型攻撃

特定の組織や個人を狙った攻撃です。

ゼロデイ攻撃

脆弱性が発見されてから修正されるまでの間に行われる攻撃です。

ソーシャルエンジニアリング

技術ではなく、人間の心理的な隙をついて情報を盗む手法です。

  • ショルダーハッキング：後ろから覗き見
  • なりすまし：関係者を装う
  • ごみあさり：廃棄物から情報を入手

脆弱性診断

脆弱性診断は、システムやネットワークに存在するセキュリティ上の弱点（脆弱性）を発見し、評価する活動です。

脆弱性診断の種類
診断の種類 対象 内容
ネットワーク診断 ネットワーク機器、サーバ ポートスキャン、既知の脆弱性確認
Webアプリケーション診断 Webアプリケーション SQLインジェクション、XSSなどの脆弱性確認
プラットフォーム診断 OS、ミドルウェア 設定不備、パッチ未適用の確認
ペネトレーションテストとの違い
  • 脆弱性診断：既知の脆弱性を網羅的に検出（広く浅く）
  • ペネトレーションテスト：実際の攻撃を模擬して侵入を試みる（深く掘り下げる）
脆弱性診断の実施タイミング
  • システム構築後の本番リリース前
  • 大規模なシステム変更後
  • 定期的な診断（年1回など）
  • 重大な脆弱性が公表された際

9.3 セキュリティ技術と対策

暗号化技術

共通鍵暗号方式

暗号化と復号に同じ鍵を使用する方式です。

  • メリット：処理が高速
  • デメリット：鍵の安全な配送が困難
  • 例：AES、DES
公開鍵暗号方式

暗号化と復号に異なる鍵（公開鍵と秘密鍵）を使用する方式です。

  • 公開鍵：誰でも入手可能、暗号化に使用
  • 秘密鍵：本人のみが保持、復号に使用
  • メリット：鍵配送問題が解決
  • デメリット：処理が遅い
  • 例：RSA

ハイブリッド暗号

共通鍵暗号と公開鍵暗号を組み合わせた方式です。

  1. 共通鍵でデータを暗号化（高速）
  2. 公開鍵で共通鍵を暗号化（安全な配送）

デジタル署名

デジタル署名は、電子文書の作成者を証明し、改ざんを検知する技術です。

  1. 送信者が秘密鍵で署名を作成
  2. 受信者が送信者の公開鍵で署名を検証

ハッシュ関数

ハッシュ関数は、任意のデータから固定長の値（ハッシュ値）を生成します。

  • 特徴：一方向性（元データに戻せない）、衝突困難性
  • 用途：パスワード保存、改ざん検知
  • 例：SHA-256、MD5（現在は脆弱）

認証技術

認証の3要素

認証は、以下の3つの要素に分類されます。

認証要素 説明 例
知識認証（Something you know） 本人が知っている情報 パスワード、PIN、秘密の質問
所持認証（Something you have） 本人が所有している物 ICカード、スマートフォン、ハードウェアトークン
生体認証（Something you are） 本人の身体的特徴 指紋、顔、虹彩、静脈、声紋
多要素認証（MFA: Multi-Factor Authentication）

多要素認証（MFA）は、2つ以上の異なる要素を組み合わせて認証する方法です。セキュリティを大幅に向上させます。

MFAの仕組みと例
  • SMS認証：パスワード（知識） + SMSで送信されるワンタイムパスワード（所持）
  • 認証アプリ：パスワード（知識） + Google Authenticatorなどのアプリで生成されるコード（所持）
  • ハードウェアトークン：パスワード（知識） + YubiKeyなどの物理デバイス（所持）
  • 生体認証組み合わせ：指紋（生体） + PINコード（知識）
MFAのメリット
  • パスワードが漏洩しても、第2要素がないとログインできない
  • フィッシング攻撃への耐性が高まる
  • 不正アクセスのリスクを大幅に低減
  • ITパスポート試験でも頻出の重要概念
注意：2段階認証と多要素認証の違い

2段階認証：2回認証を行うが、同じ要素でもよい（例：パスワード + セキュリティ質問 = 両方とも知識）

多要素認証：必ず異なる要素を組み合わせる（例：パスワード + SMS = 知識 + 所持）

PKI（Public Key Infrastructure）

PKIは、公開鍵暗号を安全に運用するための基盤です。

  • 認証局（CA）：デジタル証明書を発行
  • デジタル証明書：公開鍵の正当性を証明

セキュリティ対策機器

ファイアウォール

ネットワークの境界で不正アクセスを防ぐ機器です。

  • パケットフィルタリング
  • アプリケーションゲートウェイ
IDS/IPS
  • IDS（侵入検知システム）：不正アクセスを検知して通知
  • IPS（侵入防止システム）：不正アクセスを検知して遮断
WAF（Web Application Firewall）

Webアプリケーションへの攻撃を防ぐファイアウォールです。

SOC（セキュリティオペレーションセンター）

SOC（Security Operation Center）は、組織のセキュリティを監視・分析・対処する専門チームおよび設備です。

SOCの役割と機能
  • セキュリティ監視：ネットワークやシステムを24時間365日リアルタイム監視
  • ログ分析：各種ログを収集・分析し、異常を検知
  • インシデント検知：不審な活動やセキュリティイベントを早期発見
  • 初動対応：インシデント発生時の初期対応と報告
SOCの主な業務
  1. 監視（Monitoring）：セキュリティアラートの監視
  2. 分析（Analysis）：脅威の分析と評価
  3. 対応（Response）：インシデント対応と報告
  4. 改善（Improvement）：セキュリティ対策の継続的改善

CSIRT（シーサート）

CSIRT（Computer Security Incident Response Team）は、コンピュータセキュリティインシデントに対応する組織内の専門チームです。

CSIRTの定義と役割
  • インシデント対応の中核：セキュリティインシデント発生時の対応を統括
  • 再発防止策の策定：根本原因分析と再発防止のための対策立案
  • 情報共有：関連部署や外部機関との連携・情報共有
  • 予防活動：セキュリティ教育や脆弱性管理
SOCとCSIRTの違い
観点 SOC CSIRT
主な役割 監視・検知・初動対応 インシデント対応・復旧・再発防止
活動時期 平常時の継続的監視 インシデント発生時に中心的に活動
焦点 技術的な監視・分析 組織的な対応・管理
体制 24時間365日運用が基本 インシデント時に招集されることも
補足：連携関係

SOCが異常を検知し、CSIRTに引き継ぐことで、検知から対応・復旧まで一貫したインシデント対応が可能になります。

セキュリティインシデント対応プロセス

セキュリティインシデントが発生した際の標準的な対応プロセスは、以下の6ステップで構成されます。

各ステップの詳細
ステップ 主な活動内容
1. 検知（Detection） IDS/IPS、SOCによる異常検知、ログ分析
2. 初動対応（Triage） インシデントの確認、影響範囲の特定、関係者への連絡
3. 封じ込め（Containment） 被害拡大の防止、感染端末の隔離、ネットワーク遮断
4. 根絶（Eradication） マルウェアの駆除、脆弱性の修正、不正アカウントの削除
5. 復旧（Recovery） システムの復旧、正常性の確認、監視強化
6. 事後対応（Post-Incident） 原因分析、報告書作成、再発防止策の策定と実施

セキュリティプロトコル

  • SSL/TLS：通信の暗号化（HTTPS）
  • IPsec：IP層での暗号化（VPN）

ISMS（情報セキュリティマネジメントシステム）

ISMSは、組織全体で情報セキュリティを管理する仕組みです。

ISMS の PDCAサイクル
  • Plan（計画）：リスク分析、対策計画
  • Do（実施）：対策の実施、運用
  • Check（点検）：監視、見直し
  • Act（改善）：是正措置、継続的改善

リスクマネジメント

リスク = 脅威 × 脆弱性 × 資産価値

リスク対応
  • リスク回避：リスクの原因を除去
  • リスク低減：対策でリスクを下げる
  • リスク移転：保険など第三者に転嫁
  • リスク受容：リスクを受け入れる

ゼロトラストモデル

ゼロトラストセキュリティは、「信頼せず、必ず検証する（Never Trust, Always Verify）」という原則に基づくセキュリティモデルです。

ゼロトラストの原則
  • 全てを疑う：社内ネットワークでも信頼しない、常に認証・認可を確認
  • 最小権限の原則：必要最小限のアクセス権限のみ付与
  • 継続的な検証：一度認証しても、継続的に監視・検証
  • セグメンテーション：ネットワークを細かく分割し、侵害の拡大を防ぐ
従来型セキュリティとの違い
観点 従来型（境界防御） ゼロトラスト
考え方 社内は信頼、外部は不信 全て不信、常に検証
境界 明確な境界（ファイアウォール） 境界なし、全てを保護対象
認証 一度認証すればOK 継続的に認証・認可
適用場面 オフィス中心の環境 リモートワーク、クラウド環境
ゼロトラスト実装のポイント
  • 多要素認証（MFA）の導入：全てのアクセスで本人確認
  • アイデンティティ管理：ユーザー・デバイスの厳格な管理
  • マイクロセグメンテーション：ネットワークを細分化
  • ログ監視と分析：全ての活動を記録・分析
  • エンドポイント保護：デバイスレベルでのセキュリティ強化
ITパスポート試験での重要ポイント

ゼロトラストは、クラウドやリモートワークの普及に伴い重要性が高まっている最新のセキュリティ概念です。試験でも頻出しています。

9.4 セキュリティ運用と最新動向

IPA「情報セキュリティ10大脅威」

情報セキュリティ10大脅威は、IPA（情報処理推進機構）が毎年発表する、その年に特に注意すべきセキュリティ脅威のランキングです。

情報セキュリティ10大脅威とは
  • 発表元：IPA（独立行政法人 情報処理推進機構）
  • 更新頻度：毎年1回（年初に前年度版を発表）
  • 目的：企業や個人が直面する最新のセキュリティ脅威を周知
  • 分類：「組織」向けと「個人」向けに分けてランキング
最新の脅威動向の重要性

サイバー攻撃の手法は日々進化しています。最新の脅威動向を把握することで：

  • 効果的なセキュリティ対策を講じることができる
  • 経営層や関係者へのセキュリティ意識の啓発に活用できる
  • ITパスポート試験でも最新の脅威が出題される
2024年版 情報セキュリティ10大脅威（組織）の主な例
脅威 説明
ランサムウェアによる被害 データを暗号化して身代金を要求する攻撃。近年最も深刻な脅威の一つ
標的型攻撃による機密情報の窃取 特定の組織を狙った高度で執拗な攻撃
サプライチェーンの弱点を悪用した攻撃 取引先や関連企業を経由して本命の組織を攻撃
内部不正による情報漏えい 従業員や関係者による意図的な情報持ち出し
テレワーク等のニューノーマルな働き方を狙った攻撃 リモートワーク環境の脆弱性を狙った攻撃
修正プログラムの公開前を狙う攻撃（ゼロデイ攻撃） 脆弱性が公表されてから修正されるまでの間を狙う攻撃
IPAのWebサイトで最新情報をチェック

公式サイト：https://www.ipa.go.jp/security/10threats/

毎年更新される最新の脅威情報は、ITパスポート試験の学習および実務において非常に重要です。定期的に確認することをお勧めします。

試験対策のポイント
  • 試験前に最新年度版の10大脅威をチェック
  • 上位にランクインしている脅威は特に重点的に学習
  • 各脅威の対策方法も合わせて理解する
  • 組織向けと個人向けの違いを把握する
実習 9-1: セキュリティ対策の選択

以下の脅威に対する適切な対策を考えましょう。

  1. フィッシング詐欺 → 社員教育、多要素認証、URLの確認
  2. ランサムウェア → 定期的なバックアップ、ウイルス対策ソフト
  3. SQLインジェクション → 入力値のチェック、WAF
  4. DoS攻撃 → IPS、負荷分散
理解度確認クイズ
  1. 情報セキュリティの3要素でないものは？
    • A. 機密性
    • B. 完全性
    • C. 可用性
    • D. 効率性
    解答を見る

    正解：D
    効率性は情報セキュリティの3要素ではありません。

  2. 公開鍵暗号方式の特徴は？
    • A. 暗号化と復号に同じ鍵を使う
    • B. 暗号化と復号に異なる鍵を使う
    • C. 鍵が不要
    • D. 処理が最も高速
    解答を見る

    正解：B
    公開鍵暗号方式は暗号化と復号に異なる鍵を使います。

  3. ランサムウェアの特徴は？
    • A. データを暗号化して身代金を要求
    • B. 単独で自己増殖する
    • C. バックドアを作成する
    • D. 情報を盗み出す
    解答を見る

    正解：A
    ランサムウェアはデータを暗号化して身代金を要求します。

  4. IDSの役割は？
    • A. 不正アクセスを検知して通知
    • B. 不正アクセスを検知して遮断
    • C. ウイルスを駆除
    • D. データを暗号化
    解答を見る

    正解：A
    IDSは不正アクセスを検知して通知します（遮断はIPS）。

  5. デジタル署名の目的は？
    • A. データの暗号化
    • B. 作成者の証明と改ざん検知
    • C. ウイルス対策
    • D. バックアップ
    解答を見る

    正解：B
    デジタル署名は作成者の証明と改ざん検知に使用されます。
