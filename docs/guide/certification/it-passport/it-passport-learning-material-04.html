セキュリティ

情報資産への脅威、攻撃手法、脅威への対策、ネットワークセキュリティ、暗号化技術、情報セキュリティマネジメントについて学習します。

この章で学ぶこと

  • 情報資産への脅威（マルウェア、脆弱性）を理解する
  • 攻撃手法（フィッシング、SQLインジェクション、ブルートフォース攻撃、プロンプトインジェクション）を学ぶ
  • 脅威への対策（ファイアウォール、WAF、IDS/IPS）を理解する
  • ネットワークセキュリティ（DMZ、VPN）の仕組みを学ぶ
  • 暗号化技術（共通鍵・公開鍵暗号方式、デジタル署名、PKI）を習得する

4.1 情報資産への脅威

情報資産を脅かす要因には、マルウェアと脆弱性があります。これらは企業や個人のデータを危険にさらす可能性があります。

マルウェア（悪意のあるソフトウェア）

マルウェア（Malware）は、コンピュータに害を与える悪意のあるソフトウェアの総称です。代表的なマルウェアには以下があります。

種類 特徴
ウイルス 他のプログラムに寄生し、自己増殖する
ワーム 自己完結型で、ネットワークを通じて拡散する
トロイの木馬 正規のプログラムに偽装し、バックドアを仕掛ける
ランサムウェア データを暗号化し、身代金を要求する
スパイウェア ユーザーの行動を監視し、情報を外部に送信する

脆弱性

脆弱性（Vulnerability）は、ソフトウェアやシステムの設計・実装上の欠陥で、攻撃者に悪用される可能性があるセキュリティ上の弱点です。脆弱性を放置すると、攻撃者に侵入経路を提供してしまいます。

重要ポイント

OSやソフトウェアの更新（パッチ適用）は、脆弱性を修正する重要な対策です。定期的にアップデートを行いましょう。

4.2 攻撃手法

攻撃者は様々な手法を用いて、システムやユーザーを狙います。代表的な攻撃手法を理解しましょう。

フィッシング

フィッシング（Phishing）は、正規の企業やサービスを装った偽のメールやWebサイトを使い、ユーザーから個人情報やパスワードを盗み取る攻撃です。

注意

不審なメールのリンクはクリックせず、送信者のメールアドレスやURLを確認しましょう。

SQLインジェクション

SQLインジェクションは、Webアプリケーションの入力フォームに悪意のあるSQL文を入力し、データベースを不正に操作する攻撃です。データの漏洩や改ざんが発生する可能性があります。

ブルートフォース攻撃

ブルートフォース攻撃（総当たり攻撃）は、パスワードやIDを片っ端から試して不正ログインを試みる攻撃です。

プロンプトインジェクション

プロンプトインジェクションは、AIチャットボットや生成AIに対して、不正な指示を含むプロンプトを入力し、意図しない動作をさせる攻撃です。近年、生成AIの普及により注目されています。

4.3 脅威への対策

セキュリティ脅威に対する技術的な対策には、ファイアウォール、WAF、IDS/IPSなどがあります。

ファイアウォール

ファイアウォール（Firewall）は、ネットワークの境界で通信を監視し、不正なアクセスを遮断するセキュリティシステムです。送信元IPアドレスやポート番号などの情報をもとに、通信の許可・拒否を制御します。

WAF（Web Application Firewall）

WAFは、Webアプリケーションに特化したファイアウォールで、SQLインジェクションやクロスサイトスクリプティング（XSS）などのWebアプリケーションへの攻撃を検知・防御します。

IDS/IPS

IDS（Intrusion Detection System：侵入検知システム）は、不正な通信やアクセスを検知して管理者に通知するシステムです。IPS（Intrusion Prevention System：侵入防止システム）は、IDSの機能に加えて、不正な通信を自動的に遮断します。

対策技術 機能
ファイアウォール ネットワーク境界での通信制御
WAF Webアプリケーションへの攻撃を防御
IDS 不正アクセスを検知・通知
IPS 不正アクセスを検知・自動遮断

4.4 ネットワークセキュリティ

ネットワークのセキュリティを高めるために、DMZやVPNといった技術が活用されています。

DMZ（DeMilitarized Zone：非武装地帯）

DMZは、インターネットと内部ネットワークの間に設置される緩衝地帯です。Webサーバーやメールサーバーなど、外部からアクセスされる必要があるサーバーをDMZに配置することで、内部ネットワークへの直接アクセスを防ぎます。

VPN（Virtual Private Network：仮想プライベートネットワーク）

VPNは、インターネットなどの公衆ネットワーク上に、暗号化された専用回線のような仮想的なプライベートネットワークを構築する技術です。リモートワークや拠点間通信で安全に情報をやり取りできます。

VPNの主な用途

  • リモートアクセスVPN: 在宅勤務等で、自宅から社内ネットワークに安全にアクセス
  • 拠点間VPN: 本社と支社間など、離れた拠点間を安全に接続

4.5 暗号化技術

暗号化は、データを第三者に読まれないように変換する技術です。主に共通鍵暗号方式と公開鍵暗号方式があります。

共通鍵暗号方式

共通鍵暗号方式は、暗号化と復号化に同じ鍵を使う方式です。暗号化・復号化の処理が高速ですが、鍵の配送や管理が課題となります。代表的なアルゴリズムに、AES（Advanced Encryption Standard）があります。

公開鍵暗号方式

公開鍵暗号方式は、暗号化と復号化に異なる鍵（公開鍵と秘密鍵）を使う方式です。公開鍵は誰でも使えるように公開し、秘密鍵は受信者のみが保持します。代表的なアルゴリズムに、RSAがあります。

方式 鍵の種類 特徴
共通鍵暗号 1つの鍵（共通鍵） 高速だが鍵の配送が課題
公開鍵暗号 2つの鍵（公開鍵・秘密鍵） 鍵配送が容易だが処理が遅い

デジタル署名

デジタル署名は、電子文書の作成者を証明し、改ざんされていないことを保証する技術です。公開鍵暗号方式を利用し、送信者の秘密鍵で署名を作成し、受信者は送信者の公開鍵で署名を検証します。

PKI（Public Key Infrastructure：公開鍵基盤）

PKIは、公開鍵暗号方式を安全に利用するための仕組みです。認証局（CA：Certificate Authority）が、公開鍵の持ち主を証明するデジタル証明書を発行します。

HTTPS通信の仕組み

HTTPS（Hypertext Transfer Protocol Secure）は、SSL/TLSという暗号化技術を使ってWebサーバーとブラウザ間の通信を保護します。公開鍵暗号方式と共通鍵暗号方式を組み合わせて安全な通信を実現しています。

4.6 情報セキュリティマネジメント

組織全体で情報セキュリティを確保するためには、技術的対策だけでなく、管理的・人的対策も重要です。

情報セキュリティの三要素（CIA）

情報セキュリティは、次の3つの要素を維持することで実現されます。

情報セキュリティの三要素（CIA）

  • 機密性（Confidentiality）: 許可された者のみが情報にアクセスできる
  • 完全性（Integrity）: 情報が改ざんされずに正確である
  • 可用性（Availability）: 必要なときに情報やシステムが利用できる

ISMS（Information Security Management System）

ISMSは、組織が情報セキュリティを継続的に管理・改善するための仕組みです。国際規格ISO/IEC 27001に準拠した認証制度があり、企業の信頼性向上につながります。

リスクアセスメント

リスクアセスメントは、情報資産に対する脅威を特定し、そのリスクを評価・分析するプロセスです。リスクの大きさに応じて、リスク対応（回避、低減、移転、受容）を決定します。

重要ポイント

情報セキュリティは、技術・管理・人の3つの側面からバランス良く対策を講じることが重要です。定期的な教育や訓練も欠かせません。

理解度確認クイズ

  1. Q1. データを暗号化し、身代金を要求するマルウェアはどれか。

    回答を表示

    解答: ランサムウェア
    解説: ランサムウェアは、感染したコンピュータのデータを暗号化し、復号と引き換えに身代金（Ransom）を要求するマルウェアです。

  2. Q2. Webアプリケーションへの攻撃を防御する技術はどれか。

    回答を表示

    解答: WAF（Web Application Firewall）
    解説: WAFは、SQLインジェクションやXSSなどのWebアプリケーション特有の攻撃を検知・防御します。

  3. Q3. 暗号化と復号化に同じ鍵を使う暗号方式はどれか。

    回答を表示

    解答: 共通鍵暗号方式
    解説: 共通鍵暗号方式は、送信者と受信者が同じ鍵を使って暗号化・復号化を行います。代表的なアルゴリズムにAESがあります。

  4. Q4. 情報セキュリティの三要素（CIA）に含まれないものはどれか。

    回答を表示

    解答: 認証性は三要素に含まれない
    解説: 情報セキュリティの三要素は、機密性（Confidentiality）、完全性（Integrity）、可用性（Availability）です。

  5. Q5. インターネットと内部ネットワークの間に設置される緩衝地帯はどれか。

    回答を表示

    解答: DMZ（DeMilitarized Zone：非武装地帯）
    解説: DMZは、Webサーバーなど外部公開サーバーを配置し、内部ネットワークへの直接アクセスを防ぐ緩衝地帯です。
