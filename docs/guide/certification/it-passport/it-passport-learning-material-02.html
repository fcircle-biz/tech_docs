データベース

データベースの基本概念、DBMS、RDBMS、NoSQL、データベース設計、E-R図、正規化、データベース操作、主キー、外部キー、関係演算について学習します。

この章で学ぶこと

  • データベースの基本概念とDBMS、RDBMS、NoSQLの違いを理解する
  • データベース設計の基礎（E-R図、正規化）を学ぶ
  • 主キー、外部キーの役割とデータベース操作の基本を理解する
  • 抽出条件の指定と関係演算（選択、射影、結合）を習得する

2.1 データベースの基本概念

データベースとは、大量のデータを整理して保存し、必要なときに検索・更新できるようにしたデータの集まりです。企業の顧客情報、商品在庫、Webサービスのユーザーデータなど、あらゆる情報を効率的に管理するために使われます。

DBMS（データベース管理システム）

DBMS（Database Management System）は、データベースを効率的に管理するためのソフトウェアです。DBMSを使用することで、データの追加・更新・削除・検索といった操作を安全かつ効率的に行うことができます。

DBMSの主な機能

  • データの一元管理: 複数のユーザーが同時にアクセスできる
  • データの整合性維持: 矛盾のないデータを保証
  • セキュリティ管理: アクセス権限の設定
  • バックアップとリカバリ: データ損失を防ぐ

RDBMS（リレーショナルデータベース管理システム）

RDBMS（Relational Database Management System）は、データを「表（テーブル）」の形式で管理するDBMSです。複数のテーブルを関連付けてデータを管理することで、効率的なデータ操作が可能になります。代表的なRDBMSには、MySQL、PostgreSQL、Oracle Database、Microsoft SQL Serverなどがあります。

製品名 開発元 特徴
MySQL Oracle（旧Sun Microsystems） オープンソース、高速
PostgreSQL PostgreSQL Global Development Group オープンソース、高機能
Oracle Database Oracle 商用、大規模システム向け
Microsoft SQL Server Microsoft 商用、Windows環境との親和性

NoSQL（非リレーショナルデータベース）

NoSQLは、RDBMSの表形式とは異なる方法でデータを管理するデータベースです。大量のデータや高速な読み書きが必要なWebサービスなどで広く利用されています。代表的なNoSQLには、MongoDB（ドキュメント型）、Redis（キーバリュー型）、Cassandra（列指向型）などがあります。

RDBMSとNoSQLの使い分け

RDBMS: 構造化されたデータ、複雑な検索、トランザクションが必要な場合（会計システム、顧客管理等）
NoSQL: 柔軟なデータ構造、高速な読み書き、大量データの処理が必要な場合（SNS、ログ収集、IoT等）

2.2 データベース設計

データベース設計は、データをどのようにテーブルに分けて保存するかを決める重要なプロセスです。適切な設計により、データの整合性を保ち、効率的な検索が可能になります。

E-R図（Entity-Relationship Diagram）

E-R図は、データベース設計の初期段階で使用される図です。エンティティ（実体）とリレーションシップ（関連）を視覚的に表現します。エンティティは「顧客」「商品」「注文」などのデータのまとまりを表し、リレーションシップはそれらの関係を表します。

erDiagram CUSTOMER ||--o{ ORDER : places ORDER ||--|{ ORDER_DETAIL : contains PRODUCT ||--o{ ORDER_DETAIL : referenced_by CUSTOMER { int customer_id PK string name string email string phone } ORDER { int order_id PK int customer_id FK date order_date string status } ORDER_DETAIL { int detail_id PK int order_id FK int product_id FK int quantity decimal unit_price } PRODUCT { int product_id PK string product_name decimal price int stock }

カーディナリティ（多重度）の表記

  • 1:1 - 1対1の関係（例: 社員と社員証）
  • 1:N - 1対多の関係（例: 顧客と注文）
  • N:M - 多対多の関係（例: 学生と授業）

正規化

正規化は、データの重複を排除し、整合性を保つためにテーブルを分割する手法です。正規化には段階があり、第1正規形、第2正規形、第3正規形などがあります。ITパスポート試験では、第3正規形までの理解が求められます。

正規形 条件
第1正規形 繰り返し項目をなくし、各項目が単一の値を持つ
第2正規形 第1正規形かつ、主キーに完全に依存する（部分関数従属を排除）
第3正規形 第2正規形かつ、主キー以外の項目に依存しない（推移的関数従属を排除）

正規化のメリット

データの重複を排除することで、更新時の不整合を防ぎ、ストレージ容量を節約できます。ただし、過度な正規化はテーブルの結合が増え、検索速度が低下する場合もあります。

2.3 データベース操作の基礎

データベースを操作するためには、主キーと外部キーの概念を理解する必要があります。

主キー（Primary Key）

主キーは、テーブル内の各レコード（行）を一意に識別するための項目です。主キーには、以下の特徴があります。

主キーの特徴

  • 一意性: 同じ値が複数のレコードに存在しない
  • 非NULL: 必ず値を持つ（空白は許されない）
  • 不変性: 一度設定したら変更しない

例: 社員テーブルの「社員番号」、商品テーブルの「商品コード」など

外部キー（Foreign Key）

外部キーは、別のテーブルの主キーを参照する項目です。外部キーを使用することで、テーブル間の関連を表現し、データの整合性を保つことができます。

参照整合性

外部キーは、参照先のテーブルに存在する値でなければなりません。存在しない顧客IDを注文テーブルに登録しようとすると、エラーになります。

2.4 抽出条件の指定と関係演算

データベースから必要なデータを取り出すための操作として、関係演算があります。ITパスポート試験では、選択、射影、結合の3つの基本演算を理解しておきましょう。

選択（Selection）

選択は、テーブルから条件に合う行（レコード）を抽出する操作です。

選択の例

「価格が1000円以上の商品」を抽出

元のテーブル:
商品ID | 商品名    | 価格
--------|-----------|------
1      | りんご    | 800
2      | バナナ    | 1200
3      | ぶどう    | 1500

選択後:
商品ID | 商品名    | 価格
--------|-----------|------
2      | バナナ    | 1200
3      | ぶどう    | 1500

射影（Projection）

射影は、テーブルから必要な列（フィールド）だけを抽出する操作です。

射影の例

「商品名と価格」の列だけを抽出

元のテーブル:
商品ID | 商品名    | 価格 | 在庫
--------|-----------|------|-----
1      | りんご    | 800  | 50
2      | バナナ    | 1200 | 30

射影後:
商品名    | 価格
-----------|------
りんご    | 800
バナナ    | 1200

結合（Join）

結合は、複数のテーブルを共通の列をキーにして結合し、新しいテーブルを作成する操作です。最も一般的な結合は「内部結合」で、両方のテーブルに存在するデータのみを結合します。

結合の例

「顧客テーブル」と「注文テーブル」を顧客IDで結合

顧客テーブル:
顧客ID | 顧客名
--------|--------
1      | 田中
2      | 佐藤

注文テーブル:
注文ID | 顧客ID | 商品
--------|--------|--------
101    | 1      | りんご
102    | 1      | バナナ
103    | 2      | ぶどう

結合後:
注文ID | 顧客名 | 商品
--------|--------|--------
101    | 田中   | りんご
102    | 田中   | バナナ
103    | 佐藤   | ぶどう

関係演算の組み合わせ

実際のデータベース操作では、選択・射影・結合を組み合わせて使用します。例えば、「特定の顧客が購入した商品の名前と価格のリスト」を得るには、結合→選択→射影の順で演算を行います。

理解度確認クイズ

  1. Q1. データベースを管理するソフトウェアを何というか。

    回答を表示

    解答: DBMS（Database Management System）
    解説: DBMSは、データの追加・更新・削除・検索などを効率的に行うためのソフトウェアです。

  2. Q2. データを表（テーブル）の形式で管理するDBMSを何というか。

    回答を表示

    解答: RDBMS（Relational Database Management System）
    解説: RDBMSは、データを複数の表（テーブル）として管理し、それらを関連付けることでデータを効率的に操作します。

  3. Q3. テーブル内の各レコードを一意に識別するための項目を何というか。

    回答を表示

    解答: 主キー（Primary Key）
    解説: 主キーは、テーブル内で重複せず、必ず値を持つ項目です。例えば、社員番号、商品コードなどが主キーとして使われます。

  4. Q4. テーブルから条件に合う行を抽出する関係演算を何というか。

    回答を表示

    解答: 選択（Selection）
    解説: 選択は、「価格が1000円以上の商品」のように、条件に合致する行だけを抽出する操作です。

  5. Q5. データの重複を排除し、整合性を保つためにテーブルを分割する手法を何というか。

    回答を表示

    解答: 正規化
    解説: 正規化により、データの重複をなくし、更新時の不整合を防ぐことができます。第1正規形、第2正規形、第3正規形などの段階があります。
