6.1 2進数と基数変換

基数（n進数）

コンピュータは2進数で情報を表現します。

基数 使用する数字 用途
2進数 0, 1 コンピュータ内部の表現
8進数 0～7 2進数の簡略表記
10進数 0～9 人間が日常使う数
16進数 0～9, A～F メモリアドレス、色コード

2進数から10進数への変換

例：2進数 1011 を10進数に変換

1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8 + 0 + 2 + 1 = 11

10進数から2進数への変換

例：10進数 13 を2進数に変換（2で割り続ける）

  • 13 ÷ 2 = 6 余り 1
  • 6 ÷ 2 = 3 余り 0
  • 3 ÷ 2 = 1 余り 1
  • 1 ÷ 2 = 0 余り 1
  • 余りを逆順に並べる：1101

16進数

16進数では、10～15をA～Fで表現します。

  • A = 10, B = 11, C = 12, D = 13, E = 14, F = 15
  • 例：16進数 2F = 2×16¹ + 15×16⁰ = 32 + 15 = 47

ビットとバイト

  • ビット（bit）：情報の最小単位、0または1
  • バイト（byte）：8ビット
  • キロバイト（KB）：1,024バイト（2¹⁰）
  • メガバイト（MB）：1,024KB（2²⁰）
  • ギガバイト（GB）：1,024MB（2³⁰）
  • テラバイト（TB）：1,024GB（2⁴⁰）

6.3 データ構造とアルゴリズム

基本的なデータ構造

配列（Array）

同じ型のデータを連続したメモリ領域に格納するデータ構造です。

  • 特徴：インデックスで要素に直接アクセス可能
  • 用途：固定サイズのデータ管理
リスト（List）

要素を順番に連結したデータ構造です。

  • 特徴：要素の追加・削除が容易
  • 用途：可変長のデータ管理
スタック（Stack）

LIFO（Last In, First Out：後入れ先出し）のデータ構造です。

  • 操作：プッシュ（追加）、ポップ（取り出し）
  • 用途：関数呼び出しの管理、戻るボタンの履歴
キュー（Queue）

FIFO（First In, First Out：先入れ先出し）のデータ構造です。

  • 操作：エンキュー（追加）、デキュー（取り出し）
  • 用途：印刷待ち行列、メッセージキュー

代表的なアルゴリズム

線形探索

データを先頭から順番に探索する方法です。

  • 計算量：O(n)
  • 特徴：データがソートされていなくても使用可能
2分探索

ソート済みデータを半分ずつ絞り込んで探索する方法です。

  • 計算量：O(log n)
  • 特徴：高速だが、データのソートが必要
バブルソート

隣接する要素を比較して交換を繰り返すソート方法です。

  • 計算量：O(n²)
  • 特徴：実装は簡単だが効率は悪い

プログラミング言語

言語の分類
分類 特徴 例
機械語 コンピュータが直接実行できる言語 0と1の羅列
アセンブリ言語 機械語を人間が読みやすくした言語 LOAD、STORE、ADD など
高級言語 人間が理解しやすい言語 Python、Java、C、JavaScript
コンパイラとインタプリタ
  • コンパイラ：ソースコードを機械語に一括変換（C、Java）
  • インタプリタ：ソースコードを1行ずつ実行（Python、JavaScript）

マークアップ言語

  • HTML：Webページの構造を記述
  • XML：データの構造を記述
  • JSON：データ交換フォーマット

6.4 AIの基礎

AI（人工知能）とは

AI（Artificial Intelligence：人工知能）は、人間の知的活動をコンピュータで実現する技術です。

AIの分類
分類 説明 例
強いAI 人間と同等かそれ以上の知能を持つAI（汎用人工知能：AGI） 現在は実現していない
弱いAI 特定の分野に特化したAI（特化型人工知能） 画像認識、音声認識、チャットボット

機械学習（Machine Learning）

データからパターンや規則性を自動的に学習し、予測や判断を行う技術です。

機械学習の種類
種類 説明 活用例
教師あり学習 正解データ（ラベル）を使って学習 スパムメール判定、画像分類
教師なし学習 正解データなしでデータの特徴を発見 顧客グループ分け、異常検知
強化学習 試行錯誤を通じて報酬を最大化する行動を学習 ゲームAI、ロボット制御

ディープラーニング（Deep Learning）

ディープラーニングは、多層のニューラルネットワークを使った機械学習の手法です。

ニューラルネットワークとは

人間の脳の神経回路を模倣した情報処理モデルです。

  • 入力層：データを受け取る層
  • 隠れ層：データを処理する層（複数の層がある場合はディープラーニング）
  • 出力層：結果を出力する層
ディープラーニングの特徴
  • 特徴量の自動抽出：データから重要な特徴を自動的に見つける
  • 高い精度：大量のデータを使うことで高精度の予測が可能
  • 計算リソース：大量の計算が必要（GPU使用）

AIの活用事例

分野 活用例
画像認識 顔認証、医療画像診断、自動運転
自然言語処理 機械翻訳、チャットボット、文章要約
音声認識 音声アシスタント、議事録作成
推薦システム 商品レコメンド、動画配信サービス
予測分析 需要予測、故障予知保全

6.5 データサイエンス

データサイエンスとは

データサイエンスは、大量のデータから有益な知見を引き出し、ビジネス上の意思決定に活用する学問・技術分野です。

ビッグデータ

従来の手法では処理が困難な巨大で複雑なデータの集合をビッグデータと呼びます。

ビッグデータの3V（Three Vs）
特徴 英語 説明
量（Volume） Volume データ量が膨大（テラバイト、ペタバイト級）
速度（Velocity） Velocity データが高速に生成・更新される
多様性（Variety） Variety データの種類が多様（構造化、非構造化データ）
補足：5Vモデル

3Vに加えて、以下の2つを含めることもあります。

  • 正確性（Veracity）：データの信頼性と品質
  • 価値（Value）：データから得られるビジネス価値

データ分析プロセス

CRISP-DM（Cross-Industry Standard Process for Data Mining）

データマイニングプロジェクトの標準的なプロセスモデルです。

  1. ビジネス理解：目的と要件の明確化
  2. データ理解：データの収集と特性の把握
  3. データ準備：データのクリーニングと加工
  4. モデリング：分析手法の選択と適用
  5. 評価：モデルの精度とビジネス目標の達成度確認
  6. 展開：分析結果の実用化

データマイニング

大量のデータから有用なパターンや知識を発見する技術です。

主なデータマイニング手法
手法 説明 用途
分類 データをカテゴリに分類 顧客の購買傾向分類
クラスタリング 類似データをグループ化 顧客セグメンテーション
相関ルール データ間の関連性を発見 併売分析（バスケット分析）
回帰分析 数値の予測 売上予測、需要予測

データ可視化

データを図表やグラフで視覚的に表現することで、理解しやすくする技術です。

データ可視化の重要性
  • 直感的な理解：複雑なデータを一目で把握できる
  • パターンの発見：データの傾向や異常値を見つけやすい
  • コミュニケーション：分析結果を効果的に伝達できる
  • 意思決定支援：データに基づく判断を促進
主な可視化手法
  • 棒グラフ・円グラフ：数量の比較
  • 折れ線グラフ：時系列の変化
  • 散布図：2変数間の相関関係
  • ヒートマップ：データの分布や密度
  • ダッシュボード：複数の指標を統合表示

6.6 量子コンピュータの概要

量子コンピュータとは

量子力学の原理を利用して計算を行うコンピュータです。従来のコンピュータとは根本的に異なる仕組みで動作します。

古典コンピュータと量子コンピュータの違い

項目 古典コンピュータ 量子コンピュータ
情報の単位 ビット（0または1） 量子ビット（qubit：0と1の重ね合わせ）
状態 確定的（0か1のどちらか） 確率的（0と1が同時に存在）
計算方式 逐次処理が基本 並列処理（重ね合わせによる）
得意な問題 一般的な計算全般 特定の複雑な組み合わせ問題

量子ビット（Qubit）の特徴

量子の性質
  • 重ね合わせ（Superposition）：0と1の状態を同時に持つことができる
  • 量子もつれ（Entanglement）：複数の量子ビットが相互に関連する状態
  • 量子干渉：確率振幅を強め合ったり弱め合ったりして、正しい答えを導く
量子ビットの利点

n個の量子ビットは、2n個の状態を同時に表現できます。

  • 2量子ビット：4つの状態（2²）
  • 10量子ビット：1,024の状態（2¹⁰）
  • 100量子ビット：約10³⁰の状態（2¹⁰⁰）

これにより、膨大な並列計算が可能になります。

量子コンピュータの期待される応用分野

分野 応用例
暗号解読 現在の暗号技術（RSA暗号）の解読が可能になる可能性
新素材開発 分子・原子レベルのシミュレーションによる新材料の設計
創薬 タンパク質の立体構造解析、薬剤候補の探索
金融 ポートフォリオ最適化、リスク分析
機械学習 量子機械学習による高速な学習と予測
最適化問題 物流ルート最適化、スケジューリング
量子コンピュータの現状

量子コンピュータは研究開発段階にあり、以下の課題があります。

  • エラー率：量子ビットは環境ノイズに弱く、エラーが発生しやすい
  • 動作環境：超低温（絶対零度近く）が必要
  • 量子ビット数：実用的な計算には多数の量子ビットが必要
  • 得意分野の限定：すべての問題で古典コンピュータより速いわけではない

現在は、古典コンピュータと量子コンピュータを組み合わせた「ハイブリッド計算」が実用化に向けた主要アプローチです。

実習 6-1: 基数変換の練習
問題
  1. 2進数 10110 を10進数に変換しなさい
  2. 10進数 25 を2進数に変換しなさい
  3. 16進数 1A を10進数に変換しなさい
解答
  1. 1×16 + 0×8 + 1×4 + 1×2 + 0×1 = 22
  2. 11001（25÷2=12余1、12÷2=6余0、6÷2=3余0、3÷2=1余1、1÷2=0余1）
  3. 1×16 + 10×1 = 26
