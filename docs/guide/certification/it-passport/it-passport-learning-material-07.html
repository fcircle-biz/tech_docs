プロジェクトマネジメント

プロジェクトマネジメントの基礎、アローダイアグラム、順列と組合せ、サービスマネジメントについて学習します。

7.1 プロジェクトマネジメントの基礎

プロジェクトマネジメントとは、特定の目標を達成するために、限られた期間内でリソース（人、時間、予算）を効果的に管理する活動です。

PMBOK（Project Management Body of Knowledge）

PMBOKは、プロジェクトマネジメント協会（PMI）が発行するプロジェクトマネジメントの知識体系です。プロジェクトマネジメントのベストプラクティスや標準的な手法がまとめられています。

PMBOKの主な知識エリア

  • 統合マネジメント: プロジェクト全体の統合と調整
  • スコープマネジメント: プロジェクトの範囲を定義・管理
  • タイムマネジメント: スケジュールの計画と管理
  • コストマネジメント: 予算の計画と管理
  • 品質マネジメント: 品質基準の設定と管理
  • リソースマネジメント: 人的リソースの管理
  • コミュニケーションマネジメント: 情報共有とコミュニケーション
  • リスクマネジメント: リスクの識別と対応
  • 調達マネジメント: 外部リソースの調達
  • ステークホルダーマネジメント: 利害関係者の管理

WBS（Work Breakdown Structure）

WBSは、プロジェクトの作業を階層的に分解し、管理可能な単位に分割する手法です。大きなプロジェクトを小さなタスクに分解することで、作業の見積もりや進捗管理が容易になります。

graph TB A["プロジェクト全体"] B["フェーズ1: 計画"] C["フェーズ2: 設計"] D["フェーズ3: 実装"] E["タスク1.1: 要件定義"] F["タスク1.2: スケジュール作成"] G["タスク2.1: 基本設計"] H["タスク2.2: 詳細設計"] I["タスク3.1: 開発"] J["タスク3.2: テスト"] A --> B A --> C A --> D B --> E B --> F C --> G C --> H D --> I D --> J

重要ポイント

WBSを作成することで、作業の抜け漏れを防ぎ、正確な見積もりと効率的な進捗管理が可能になります。

7.2 アローダイアグラム

アローダイアグラムは、プロジェクトの作業順序と所要時間を矢印と丸（ノード）で表現する図です。作業の依存関係を視覚化し、プロジェクト全体の所要時間を計算できます。

アローダイアグラムの基本

アローダイアグラムでは、矢印が作業を表し、丸（ノード）が作業の開始点または終了点を表します。各作業の所要時間を矢印の上に記載します。

graph LR A((開始)) B((2)) C((3)) D((4)) E((終了)) A -->|作業A:3日| B A -->|作業B:5日| C B -->|作業C:4日| D C -->|作業D:2日| D D -->|作業E:3日| E

クリティカルパス

クリティカルパスは、プロジェクトの開始から終了までの最も時間がかかる経路のことです。クリティカルパス上の作業が遅れると、プロジェクト全体の納期が遅れます。

クリティカルパスの計算例

上記のアローダイアグラムのクリティカルパスを求めます：

  • 経路1: 開始 → A（3日） → C（4日） → E（3日） = 10日
  • 経路2: 開始 → B（5日） → D（2日） → E（3日） = 10日

クリティカルパス: 経路1と経路2（両方とも10日）

重要ポイント

クリティカルパス上の作業を優先的に管理することで、プロジェクト全体の納期遅延を防ぐことができます。

7.3 順列と組合せ

順列と組合せは、確率・統計の基礎となる数学的概念です。ITパスポート試験では、選択肢の数を数える問題などで出題されます。

順列（Permutation）

順列は、複数の要素から一部を選び、順序を考慮して並べる方法の総数を計算します。nPr（n個からr個を選んで並べる）で表します。

順列の計算式と例

nPr = n! / (n - r)!

例題: A、B、C、Dの4人から3人を選んで並べる方法は何通りか。

4P3 = 4! / (4 - 3)! = 24 / 1 = 24通り

組合せ（Combination）

組合せは、複数の要素から一部を選ぶ方法の総数を計算します。順序は考慮しません。nCr（n個からr個を選ぶ）で表します。

組合せの計算式と例

nCr = n! / (r! × (n - r)!)

例題: A、B、C、Dの4人から3人を選ぶ方法は何通りか。

4C3 = 4! / (3! × 1!) = 24 / (6 × 1) = 4通り

項目 順列（Permutation） 組合せ（Combination）
順序 考慮する 考慮しない
記号 nPr nCr
例 パスワードの組み合わせ 委員会メンバーの選出

7.4 サービスマネジメント

サービスマネジメントは、ITサービスを効果的に提供・運用するための管理活動です。

ITIL（IT Infrastructure Library）

ITILは、ITサービスマネジメントのベストプラクティス集です。サービスの計画、提供、サポート、改善のプロセスを体系化しています。

SLA（Service Level Agreement）

SLAは、サービス提供者と顧客との間で合意されたサービスレベル（品質、可用性、応答時間など）を定めた契約書です。SLAを明確にすることで、顧客の期待値とサービス提供者の責任範囲が明確になります。

SLAに含まれる主な項目

  • 可用性: システムが稼働している時間の割合（例: 99.9%以上）
  • 応答時間: 問い合わせへの回答時間（例: 1時間以内）
  • 復旧時間: 障害発生時の復旧目標時間（例: 4時間以内）
  • サポート時間: サポート対応可能な時間帯（例: 24時間365日）

サービスデスク

サービスデスクは、ITサービスの利用者からの問い合わせや障害報告を受け付ける窓口です。ユーザーの問題を迅速に解決し、満足度を向上させる役割を担います。

種類 特徴
ヘルプデスク 技術的な問題の解決に特化
サービスデスク 問い合わせ全般を受け付ける総合窓口
コールセンター 電話による問い合わせ対応に特化

サービスデスクの役割

サービスデスクは単なる問い合わせ窓口ではなく、インシデント管理、問題管理、変更管理など、ITサービス全体の品質向上に貢献します。

理解度確認クイズ

  1. Q1. プロジェクトの作業を階層的に分解し、管理可能な単位に分割する手法は何か。

    回答を表示

    解答: WBS（Work Breakdown Structure）
    解説: WBSは、大きなプロジェクトを小さなタスクに分解することで、作業の見積もりや進捗管理を容易にします。

  2. Q2. プロジェクトの開始から終了までの最も時間がかかる経路を何というか。

    回答を表示

    解答: クリティカルパス
    解説: クリティカルパス上の作業が遅れると、プロジェクト全体の納期が遅れるため、優先的に管理する必要があります。

  3. Q3. 5人の候補者から3人の委員を選ぶ方法は何通りか。（順序は考慮しない）

    回答を表示

    解答: 10通り
    解説: 組合せ（nCr）を使って計算します。5C3 = 5! / (3! × 2!) = 120 / (6 × 2) = 10通りです。

  4. Q4. サービス提供者と顧客との間で合意されたサービスレベルを定めた契約書を何というか。

    回答を表示

    解答: SLA（Service Level Agreement）
    解説: SLAには、可用性、応答時間、復旧時間、サポート時間などの項目が含まれます。

  5. Q5. ITサービスマネジメントのベストプラクティス集を何というか。

    回答を表示

    解答: ITIL（IT Infrastructure Library）
    解説: ITILは、サービス戦略、サービス設計、サービス移行、サービス運用、継続的サービス改善の5つのライフサイクルで構成されています。
