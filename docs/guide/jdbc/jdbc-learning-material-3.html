第3章：基本的なCRUD操作

データベースアプリケーションの基本となるのは、CRUD（Create, Read, Update, Delete）操作です。これらの操作はそれぞれ、データの作成、読み取り、更新、削除を表しています。JDBCを使用することで、JavaプログラムからこれらのCRUD操作を簡単に実行することができます。この章では、それぞれの操作について詳しく学んでいきましょう。

3.1 SELECT文を使用したデータの取得と処理

データベースからデータを取得するには、SQL SELECT文を使用します。JDBCでは、Statement, PreparedStatement, CallableStatementのいずれかを使用してSELECT文を実行し、ResultSetオブジェクトを取得してデータを処理します。

基本的なSELECT操作

以下は、基本的なSELECT操作の例です：

import java.sql.*; public class SelectExample { public static void main(String[] args) { // データベース接続情報 String url = "jdbc:postgresql://localhost:5432/jdbctest"; String user = "postgres"; String password = "password"; // 実際のパスワードに置き換えてください // 自動リソース管理を使用した接続とSELECT操作 try ( Connection conn = DriverManager.getConnection(url, user, password); Statement stmt = conn.createStatement(); ResultSet rs = stmt.executeQuery("SELECT id, name, email, age FROM users") ) { // 結果セットのデータを表示 System.out.println("取得したユーザー情報:"); System.out.println("--------------------------------------------------"); System.out.printf("%-5s %-15s %-25s %-5s%n", "ID", "名前", "メール", "年齢"); System.out.println("--------------------------------------------------"); // 各行のデータを処理 while (rs.next()) { int id = rs.getInt("id"); String name = rs.getString("name"); String email = rs.getString("email"); int age = rs.getInt("age"); System.out.printf("%-5d %-15s %-25s %-5d%n", id, name, email, age); } } catch (SQLException e) { System.err.println("データベースエラー: " + e.getMessage()); e.printStackTrace(); } } }

ResultSetの基本操作

ResultSetオブジェクトは、SELECT文の実行結果を保持し、行単位でデータを取得するためのメソッドを提供します。主な操作方法は以下の通りです：

メソッド 説明 使用例
next() 次の行に移動し、行が存在すればtrueを返す while (rs.next()) { ... }
getXXX(インデックス) 列インデックス（1から始まる）を指定してデータを取得 int id = rs.getInt(1);
getXXX(カラム名) 列名を指定してデータを取得 String name = rs.getString("name");
isFirst(), isLast() 現在の位置が最初/最後の行かどうかを確認 if (rs.isFirst()) { ... }
beforeFirst(), afterLast() カーソルを最初の行の前/最後の行の後に移動 rs.beforeFirst();
absolute(行番号) 指定した行番号に移動 rs.absolute(5); // 5行目に移動
wasNull() 最後に読み取った値がNULLだったかを確認 int age = rs.getInt("age"); boolean isNull = rs.wasNull();
ResultSetの型とスクロール可能性

デフォルトのResultSetは前方向にのみスクロール可能で、読み取り専用です。より高度な操作が必要な場合は、Statementを作成する際に以下のようにタイプを指定できます：

// スクロール可能で更新可能なResultSetを取得 Statement stmt = conn.createStatement( ResultSet.TYPE_SCROLL_INSENSITIVE, // スクロール可能 ResultSet.CONCUR_UPDATABLE // 更新可能 );

条件付きSELECT文

WHERE句を使用して、特定の条件に合致するデータのみを取得することができます：

// 年齢が30歳以上のユーザーを検索 String sql = "SELECT id, name, email, age FROM users WHERE age >= 30"; ResultSet rs = stmt.executeQuery(sql);

ResultSetMetaDataの活用

ResultSetMetaDataを使用すると、結果セットの構造（列数、列名、データ型など）を動的に取得できます：

// ResultSetMetaDataを使用して列情報を取得 ResultSetMetaData metaData = rs.getMetaData(); int columnCount = metaData.getColumnCount(); // 列名を表示 for (int i = 1; i <= columnCount; i++) { System.out.printf("%-20s", metaData.getColumnName(i)); } System.out.println(); // データを表示（動的に列数に対応） while (rs.next()) { for (int i = 1; i <= columnCount; i++) { System.out.printf("%-20s", rs.getString(i)); } System.out.println(); }

3.2 PreparedStatementを使用したデータの挿入

データベースに新しいデータを挿入する際は、SQLのINSERT文を使用します。特に、ユーザー入力値をデータベースに挿入する場合は、セキュリティ上の理由からPreparedStatementを使用することが推奨されます。

PreparedStatementの利点

  • SQLインジェクション攻撃からの保護：ユーザー入力値が自動的にエスケープされる
  • パフォーマンスの向上：SQLがプリコンパイルされるため、繰り返し実行時に効率的
  • 複雑なデータ型のサポート：バイナリデータなど特殊なデータ型の扱いが容易
  • コードの可読性向上：SQLとパラメータが分離され、コードが読みやすくなる
SQLインジェクションの危険性

通常のStatement使用時に文字列連結でSQLを構築すると、悪意のある入力によってSQLインジェクション攻撃が可能になります。

// 危険な例（絶対に使用しないでください！） String name = request.getParameter("username"); // ユーザー入力 String sql = "SELECT * FROM users WHERE username = '" + name + "'"; // 悪意のあるユーザーが name に "' OR '1'='1" を入力すると // "SELECT * FROM users WHERE username = '' OR '1'='1'" となり、 // すべてのユーザー情報が漏洩する可能性がある

基本的なINSERT操作

PreparedStatementを使用したINSERT操作の基本的な例：

import java.sql.*; public class InsertExample { public static void main(String[] args) { // データベース接続情報 String url = "jdbc:postgresql://localhost:5432/jdbctest"; String user = "postgres"; String password = "password"; // 実際のパスワードに置き換えてください // 挿入するユーザーデータ String userName = "伊藤誠"; String email = "ito@example.com"; int age = 28; // 自動リソース管理を使用した接続とINSERT操作 try (Connection conn = DriverManager.getConnection(url, user, password)) { // SQLステートメントの準備（?はプレースホルダー） String sql = "INSERT INTO users (name, email, age) VALUES (?, ?, ?)"; try (PreparedStatement pstmt = conn.prepareStatement(sql)) { // プレースホルダに値をセット pstmt.setString(1, userName); // 1番目の?に名前をセット pstmt.setString(2, email); // 2番目の?にメールアドレスをセット pstmt.setInt(3, age); // 3番目の?に年齢をセット // SQLを実行して影響を受けた行数を取得 int rowsAffected = pstmt.executeUpdate(); // 結果の表示 System.out.println(rowsAffected + "行のデータが挿入されました。"); } // 挿入されたデータを確認 try (Statement stmt = conn.createStatement(); ResultSet rs = stmt.executeQuery("SELECT * FROM users WHERE email = '" + email + "'")) { if (rs.next()) { System.out.println("挿入されたユーザー情報:"); System.out.println("ID: " + rs.getInt("id")); System.out.println("名前: " + rs.getString("name")); System.out.println("メール: " + rs.getString("email")); System.out.println("年齢: " + rs.getInt("age")); } } } catch (SQLException e) { System.err.println("データベースエラー: " + e.getMessage()); e.printStackTrace(); } } }

生成されたキー値の取得

多くのデータベースでは、主キーとして自動採番される列（PostgreSQLではSERIAL型など）が使用されます。INSERTで生成された値を取得するには：

// 生成されたキー値を取得するためのSQL String sql = "INSERT INTO users (name, email, age) VALUES (?, ?, ?)"; try (PreparedStatement pstmt = conn.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS)) { // プレースホルダに値をセット pstmt.setString(1, userName); pstmt.setString(2, email); pstmt.setInt(3, age); // SQLを実行 int rowsAffected = pstmt.executeUpdate(); // 生成されたキー値を取得 try (ResultSet generatedKeys = pstmt.getGeneratedKeys()) { if (generatedKeys.next()) { int generatedId = generatedKeys.getInt(1); System.out.println("生成されたID: " + generatedId); } } }

バッチ処理によるデータ挿入

大量のデータを効率的に挿入するには、バッチ処理が有効です：

// バッチ処理による複数行のデータ挿入 String sql = "INSERT INTO users (name, email, age) VALUES (?, ?, ?)"; try (PreparedStatement pstmt = conn.prepareStatement(sql)) { // トランザクションの手動制御の開始 conn.setAutoCommit(false); // ユーザーデータの配列（実際のアプリケーションでは外部から取得） String[][] userData = { {"山本浩二", "yamamoto@example.com", "33"}, {"中村美咲", "nakamura@example.com", "27"}, {"小林健太", "kobayashi@example.com", "41"} }; // バッチにデータを追加 for (String[] user : userData) { pstmt.setString(1, user[0]); // 名前 pstmt.setString(2, user[1]); // メール pstmt.setInt(3, Integer.parseInt(user[2])); // 年齢 // バッチに追加 pstmt.addBatch(); } // バッチを実行して結果を取得 int[] results = pstmt.executeBatch(); // コミット conn.commit(); // 結果の表示 int totalInserted = 0; for (int count : results) { totalInserted += count; } System.out.println("バッチ処理により " + totalInserted + " 行のデータが挿入されました。"); // 自動コミットモードに戻す conn.setAutoCommit(true); }

3.3 データの更新と削除

データベース内の既存データを変更するには、UPDATE文を使用します。また、不要になったデータを削除するには、DELETE文を使用します。いずれの操作もPreparedStatementを使用することで、安全かつ効率的に実行できます。

基本的なUPDATE操作

以下は、PreparedStatementを使用したUPDATE操作の基本的な例です：

import java.sql.*; public class UpdateExample { public static void main(String[] args) { // データベース接続情報 String url = "jdbc:postgresql://localhost:5432/jdbctest"; String user = "postgres"; String password = "password"; // 実際のパスワードに置き換えてください // 更新するデータ int userId = 2; String newEmail = "updated_sato@example.com"; int newAge = 26; // 自動リソース管理を使用した接続とUPDATE操作 try (Connection conn = DriverManager.getConnection(url, user, password)) { // 更新前のデータを表示 try (PreparedStatement pstmtSelect = conn.prepareStatement("SELECT * FROM users WHERE id = ?")) { pstmtSelect.setInt(1, userId); try (ResultSet rs = pstmtSelect.executeQuery()) { if (rs.next()) { System.out.println("更新前のユーザー情報:"); System.out.println("ID: " + rs.getInt("id")); System.out.println("名前: " + rs.getString("name")); System.out.println("メール: " + rs.getString("email")); System.out.println("年齢: " + rs.getInt("age")); } else { System.out.println("指定されたIDのユーザーが見つかりません: " + userId); return; } } } // UPDATE操作 String sql = "UPDATE users SET email = ?, age = ? WHERE id = ?"; try (PreparedStatement pstmt = conn.prepareStatement(sql)) { // プレースホルダに値をセット pstmt.setString(1, newEmail); pstmt.setInt(2, newAge); pstmt.setInt(3, userId); // SQLを実行して影響を受けた行数を取得 int rowsAffected = pstmt.executeUpdate(); // 結果の表示 System.out.println(rowsAffected + "行のデータが更新されました。"); } // 更新後のデータを表示 try (PreparedStatement pstmtSelect = conn.prepareStatement("SELECT * FROM users WHERE id = ?")) { pstmtSelect.setInt(1, userId); try (ResultSet rs = pstmtSelect.executeQuery()) { if (rs.next()) { System.out.println("更新後のユーザー情報:"); System.out.println("ID: " + rs.getInt("id")); System.out.println("名前: " + rs.getString("name")); System.out.println("メール: " + rs.getString("email")); System.out.println("年齢: " + rs.getInt("age")); } } } } catch (SQLException e) { System.err.println("データベースエラー: " + e.getMessage()); e.printStackTrace(); } } }

基本的なDELETE操作

PreparedStatementを使用したDELETE操作の例：

import java.sql.*; public class DeleteExample { public static void main(String[] args) { // データベース接続情報 String url = "jdbc:postgresql://localhost:5432/jdbctest"; String user = "postgres"; String password = "password"; // 実際のパスワードに置き換えてください // 削除するユーザーのID int userId = 3; // 自動リソース管理を使用した接続とDELETE操作 try (Connection conn = DriverManager.getConnection(url, user, password)) { // 削除前のデータを表示 try (PreparedStatement pstmtSelect = conn.prepareStatement("SELECT * FROM users WHERE id = ?")) { pstmtSelect.setInt(1, userId); try (ResultSet rs = pstmtSelect.executeQuery()) { if (rs.next()) { System.out.println("削除対象のユーザー情報:"); System.out.println("ID: " + rs.getInt("id")); System.out.println("名前: " + rs.getString("name")); System.out.println("メール: " + rs.getString("email")); System.out.println("年齢: " + rs.getInt("age")); } else { System.out.println("指定されたIDのユーザーが見つかりません: " + userId); return; } } } // DELETE操作 String sql = "DELETE FROM users WHERE id = ?"; try (PreparedStatement pstmt = conn.prepareStatement(sql)) { // プレースホルダに値をセット pstmt.setInt(1, userId); // SQLを実行して影響を受けた行数を取得 int rowsAffected = pstmt.executeUpdate(); // 結果の表示 System.out.println(rowsAffected + "行のデータが削除されました。"); } // 削除後に確認 try (PreparedStatement pstmtSelect = conn.prepareStatement("SELECT COUNT(*) FROM users WHERE id = ?")) { pstmtSelect.setInt(1, userId); try (ResultSet rs = pstmtSelect.executeQuery()) { if (rs.next()) { int count = rs.getInt(1); if (count == 0) { System.out.println("ID: " + userId + " のユーザーは正常に削除されました。"); } else { System.out.println("削除操作が失敗しました。ユーザーはまだ存在しています。"); } } } } // 残りのユーザー数を表示 try (Statement stmt = conn.createStatement(); ResultSet rs = stmt.executeQuery("SELECT COUNT(*) FROM users")) { if (rs.next()) { System.out.println("現在のユーザー数: " + rs.getInt(1)); } } } catch (SQLException e) { System.err.println("データベースエラー: " + e.getMessage()); e.printStackTrace(); } } }

条件付き更新と削除

複数の条件を組み合わせたUPDATEとDELETE操作も可能です：

// 複数条件を使用したUPDATE String updateSql = "UPDATE users SET status = ? WHERE age >= ? AND last_login_date < ?"; try (PreparedStatement pstmt = conn.prepareStatement(updateSql)) { pstmt.setString(1, "inactive"); pstmt.setInt(2, 18); pstmt.setDate(3, java.sql.Date.valueOf("2024-01-01")); int rowsAffected = pstmt.executeUpdate(); System.out.println(rowsAffected + "人のユーザーが非アクティブに設定されました。"); } // 複数条件を使用したDELETE String deleteSql = "DELETE FROM temporary_users WHERE created_at < ? OR status = ?"; try (PreparedStatement pstmt = conn.prepareStatement(deleteSql)) { // 30日前の日付を計算 long thirtyDaysAgo = System.currentTimeMillis() - (30 * 24 * 60 * 60 * 1000L); pstmt.setTimestamp(1, new java.sql.Timestamp(thirtyDaysAgo)); pstmt.setString(2, "expired"); int rowsAffected = pstmt.executeUpdate(); System.out.println(rowsAffected + "人の一時ユーザーが削除されました。"); }
トランザクション管理の重要性

更新や削除操作を複数まとめて実行する場合、トランザクション管理が重要です。以下は基本的なトランザクション管理の例です：

// トランザクション管理の例 try (Connection conn = DriverManager.getConnection(url, user, password)) { // 自動コミットを無効化（トランザクション開始） conn.setAutoCommit(false); try { // 1つ目の操作: アカウント残高の減額 String sql1 = "UPDATE accounts SET balance = balance - ? WHERE account_id = ?"; try (PreparedStatement pstmt = conn.prepareStatement(sql1)) { pstmt.setDouble(1, 1000.00); pstmt.setString(2, "A123"); pstmt.executeUpdate(); } // 2つ目の操作: 別アカウントの残高増額 String sql2 = "UPDATE accounts SET balance = balance + ? WHERE account_id = ?"; try (PreparedStatement pstmt = conn.prepareStatement(sql2)) { pstmt.setDouble(1, 1000.00); pstmt.setString(2, "B456"); pstmt.executeUpdate(); } // すべての操作が成功したらコミット conn.commit(); System.out.println("送金処理が正常に完了しました。"); } catch (SQLException e) { // エラーが発生した場合はロールバック conn.rollback(); System.err.println("送金処理がロールバックされました: " + e.getMessage()); throw e; } finally { // 自動コミットモードを元に戻す conn.setAutoCommit(true); } }

図3.3: UPDATE操作とトランザクション管理

WHERE句のない更新・削除操作の危険性

WHERE句を指定せずにUPDATEやDELETE文を実行すると、テーブル内のすべての行が更新または削除されます。これは、特に本番環境では非常に危険な操作です。常にWHERE句を指定し、必要であれば事前にSELECT文で影響を受ける行数を確認することをお勧めします。

// 危険な例（絶対に実行しないでください！） stmt.executeUpdate("DELETE FROM users"); // すべてのユーザーデータが削除される！ // 安全な例 try (PreparedStatement pstmt = conn.prepareStatement("DELETE FROM users WHERE id = ?")) { pstmt.setInt(1, userId); int rowsAffected = pstmt.executeUpdate(); // 削除された行数を確認 if (rowsAffected != 1) { System.out.println("警告: 予期しない数の行が削除されました: " + rowsAffected); } }
練習問題 3.3.1: ユーザー情報更新機能の実装

次の要件を満たすユーザー情報更新機能を実装してください：

  1. コンソールからユーザーIDを入力し、そのユーザーの情報を表示する
  2. 各項目（名前、メール、年齢）ごとに更新するかどうかを選択できる
  3. 更新する項目だけを含むUPDATE文を動的に構築する

import java.sql.*; import java.util.ArrayList; import java.util.List; public class ContactDAO { // データベース接続情報 private static final String URL = "jdbc:postgresql://localhost:5432/jdbctest"; private static final String USER = "postgres"; private static final String PASSWORD = "password"; // 実際のパスワードに置き換えてください // 接続を取得するメソッド private Connection getConnection() throws SQLException { return DriverManager.getConnection(URL, USER, PASSWORD); } // すべての連絡先を取得 public List getAllContacts() throws SQLException { List contacts = new ArrayList<>(); String sql = "SELECT * FROM contacts ORDER BY last_name, first_name"; try (Connection conn = getConnection(); Statement stmt = conn.createStatement(); ResultSet rs = stmt.executeQuery(sql)) { while (rs.next()) { contacts.add(extractContactFromResultSet(rs)); } } return contacts; } // IDで連絡先を検索 public Contact getContactById(int id) throws SQLException { String sql = "SELECT * FROM contacts WHERE id = ?"; try (Connection conn = getConnection(); PreparedStatement pstmt = conn.prepareStatement(sql)) { pstmt.setInt(1, id); try (ResultSet rs = pstmt.executeQuery()) { if (rs.next()) { return extractContactFromResultSet(rs); } } } return null; // 該当する連絡先がない場合 } // 名前またはメールアドレスで連絡先を検索 public List searchContacts(String searchTerm) throws SQLException { List contacts = new ArrayList<>(); String sql = "SELECT * FROM contacts WHERE " + "LOWER(first_name) LIKE LOWER(?) OR " + "LOWER(last_name) LIKE LOWER(?) OR " + "LOWER(email) LIKE LOWER(?) " + "ORDER BY last_name, first_name"; try (Connection conn = getConnection(); PreparedStatement pstmt = conn.prepareStatement(sql)) { String searchPattern = "%" + searchTerm + "%"; pstmt.setString(1, searchPattern); pstmt.setString(2, searchPattern); pstmt.setString(3, searchPattern); try (ResultSet rs = pstmt.executeQuery()) { while (rs.next()) { contacts.add(extractContactFromResultSet(rs)); } } } return contacts; } // カテゴリで連絡先を検索 public List getContactsByCategory(String category) throws SQLException { List contacts = new ArrayList<>(); String sql = "SELECT * FROM contacts WHERE category = ? ORDER BY last_name, first_name"; try (Connection conn = getConnection(); PreparedStatement pstmt = conn.prepareStatement(sql)) { pstmt.setString(1, category); try (ResultSet rs = pstmt.executeQuery()) { while (rs.next()) { contacts.add(extractContactFromResultSet(rs)); } } } return contacts; } // 新しい連絡先を追加 public int addContact(Contact contact) throws SQLException { String sql = "INSERT INTO contacts (first_name, last_name, email, phone, address, birthday, category) " + "VALUES (?, ?, ?, ?, ?, ?, ?) RETURNING id"; try (Connection conn = getConnection(); PreparedStatement pstmt = conn.prepareStatement(sql)) { pstmt.setString(1, contact.getFirstName()); pstmt.setString(2, contact.getLastName()); pstmt.setString(3, contact.getEmail()); pstmt.setString(4, contact.getPhone()); pstmt.setString(5, contact.getAddress()); pstmt.setDate(6, contact.getBirthday()); pstmt.setString(7, contact.getCategory()); try (ResultSet rs = pstmt.executeQuery()) { if (rs.next()) { return rs.getInt(1); // 生成されたIDを返す } } } return -1; // 失敗した場合 } // 連絡先を更新 public boolean updateContact(Contact contact) throws SQLException { String sql = "UPDATE contacts SET first_name = ?, last_name = ?, email = ?, " + "phone = ?, address = ?, birthday = ?, category = ?, updated_at = CURRENT_TIMESTAMP " + "WHERE id = ?"; try (Connection conn = getConnection(); PreparedStatement pstmt = conn.prepareStatement(sql)) { pstmt.setString(1, contact.getFirstName()); pstmt.setString(2, contact.getLastName()); pstmt.setString(3, contact.getEmail()); pstmt.setString(4, contact.getPhone()); pstmt.setString(5, contact.getAddress()); pstmt.setDate(6, contact.getBirthday()); pstmt.setString(7, contact.getCategory()); pstmt.setInt(8, contact.getId()); int rowsAffected = pstmt.executeUpdate(); return rowsAffected > 0; } } // 連絡先を削除 public boolean deleteContact(int id) throws SQLException { String sql = "DELETE FROM contacts WHERE id = ?"; try (Connection conn = getConnection(); PreparedStatement pstmt = conn.prepareStatement(sql)) { pstmt.setInt(1, id); int rowsAffected = pstmt.executeUpdate(); return rowsAffected > 0; } } // ResultSetからContactオブジェクトを抽出するユーティリティメソッド private Contact extractContactFromResultSet(ResultSet rs) throws SQLException { Contact contact = new Contact(); contact.setId(rs.getInt("id")); contact.setFirstName(rs.getString("first_name")); contact.setLastName(rs.getString("last_name")); contact.setEmail(rs.getString("email")); contact.setPhone(rs.getString("phone")); contact.setAddress(rs.getString("address")); contact.setBirthday(rs.getDate("birthday")); contact.setCategory(rs.getString("category")); contact.setCreatedAt(rs.getTimestamp("created_at")); contact.setUpdatedAt(rs.getTimestamp("updated_at")); return contact; } }
