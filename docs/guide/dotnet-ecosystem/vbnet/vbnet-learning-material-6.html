6.1 継承

継承は、既存のクラス（親クラス・基底クラス）の機能を受け継いで新しいクラス（子クラス・派生クラス）を作成する仕組みです。

基本的な継承

' 基底クラス（親クラス）
Public Class Animal
    Protected name As String
    Protected age As Integer
    
    Public Sub New(name As String, age As Integer)
        Me.name = name
        Me.age = age
    End Sub
    
    Public Overridable Sub MakeSound()
        Console.WriteLine(name & "が音を出しています")
    End Sub
    
    Public Sub ShowInfo()
        Console.WriteLine("名前: " & name)
        Console.WriteLine("年齢: " & age & "歳")
    End Sub
End Class

' 派生クラス（子クラス）
Public Class Dog
    Inherits Animal
    
    Private breed As String
    
    Public Sub New(name As String, age As Integer, breed As String)
        MyBase.New(name, age)  ' 親クラスのコンストラクタを呼び出し
        Me.breed = breed
    End Sub
    
    ' メソッドのオーバーライド
    Public Overrides Sub MakeSound()
        Console.WriteLine(name & "が ワンワン と鳴いています")
    End Sub
    
    ' 新しいメソッドの追加
    Public Sub Fetch()
        Console.WriteLine(name & "がボールを取ってきます")
    End Sub
    
    Public Sub ShowDogInfo()
        ShowInfo()  ' 親クラスのメソッドを呼び出し
        Console.WriteLine("犬種: " & breed)
    End Sub
End Class

継承の使用例

Module Module1
    Sub Main()
        ' 親クラスのオブジェクト
        Dim animal As New Animal("動物A", 5)
        animal.ShowInfo()
        animal.MakeSound()
        Console.WriteLine()
        
        ' 子クラスのオブジェクト
        Dim dog As New Dog("ポチ", 3, "柴犬")
        dog.ShowDogInfo()
        dog.MakeSound()  ' オーバーライドされたメソッド
        dog.Fetch()      ' 子クラス独自のメソッド
        
        Console.ReadLine()
    End Sub
End Module
実習 6-1: 従業員管理システム

継承を使用して、基本従業員クラスから正社員と契約社員クラスを作成しましょう。

プログラムコード
Public Class Employee
    Protected employeeId As Integer
    Protected name As String
    Protected department As String
    Protected baseSalary As Decimal
    
    Public Sub New(id As Integer, name As String, dept As String, salary As Decimal)
        employeeId = id
        Me.name = name
        department = dept
        baseSalary = salary
    End Sub
    
    Public Overridable Function CalculateSalary() As Decimal
        Return baseSalary
    End Function
    
    Public Overridable Sub ShowEmployeeInfo()
        Console.WriteLine("=== 従業員情報 ===")
        Console.WriteLine("ID: " & employeeId)
        Console.WriteLine("名前: " & name)
        Console.WriteLine("部署: " & department)
        Console.WriteLine("基本給: " & baseSalary.ToString("C"))
    End Sub
End Class

Public Class FullTimeEmployee
    Inherits Employee
    
    Private bonus As Decimal
    Private overtime As Integer
    
    Public Sub New(id As Integer, name As String, dept As String, salary As Decimal, bonus As Decimal)
        MyBase.New(id, name, dept, salary)
        Me.bonus = bonus
        overtime = 0
    End Sub
    
    Public Sub SetOvertime(hours As Integer)
        overtime = hours
    End Sub
    
    Public Overrides Function CalculateSalary() As Decimal
        Dim overtimePay As Decimal = overtime * 2000  ' 時給2000円
        Return baseSalary + bonus + overtimePay
    End Function
    
    Public Overrides Sub ShowEmployeeInfo()
        MyBase.ShowEmployeeInfo()
        Console.WriteLine("雇用形態: 正社員")
        Console.WriteLine("賞与: " & bonus.ToString("C"))
        Console.WriteLine("残業時間: " & overtime & "時間")
        Console.WriteLine("総支給額: " & CalculateSalary().ToString("C"))
    End Sub
End Class

Public Class ContractEmployee
    Inherits Employee
    
    Private contractPeriod As Integer
    Private hourlyRate As Decimal
    Private workedHours As Integer
    
    Public Sub New(id As Integer, name As String, dept As String, rate As Decimal, period As Integer)
        MyBase.New(id, name, dept, 0)  ' 基本給は0
        hourlyRate = rate
        contractPeriod = period
        workedHours = 0
    End Sub
    
    Public Sub SetWorkedHours(hours As Integer)
        workedHours = hours
    End Sub
    
    Public Overrides Function CalculateSalary() As Decimal
        Return hourlyRate * workedHours
    End Function
    
    Public Overrides Sub ShowEmployeeInfo()
        Console.WriteLine("=== 従業員情報 ===")
        Console.WriteLine("ID: " & employeeId)
        Console.WriteLine("名前: " & name)
        Console.WriteLine("部署: " & department)
        Console.WriteLine("雇用形態: 契約社員")
        Console.WriteLine("時給: " & hourlyRate.ToString("C"))
        Console.WriteLine("契約期間: " & contractPeriod & "ヶ月")
        Console.WriteLine("勤務時間: " & workedHours & "時間")
        Console.WriteLine("総支給額: " & CalculateSalary().ToString("C"))
    End Sub
End Class

' 使用例
Module Module1
    Sub Main()
        ' 正社員
        Dim fullTime As New FullTimeEmployee(1001, "田中太郎", "開発部", 300000, 50000)
        fullTime.SetOvertime(20)
        fullTime.ShowEmployeeInfo()
        Console.WriteLine()
        
        ' 契約社員
        Dim contract As New ContractEmployee(2001, "佐藤花子", "営業部", 1500, 12)
        contract.SetWorkedHours(160)
        contract.ShowEmployeeInfo()
        Console.WriteLine()
        
        Console.ReadLine()
    End Sub
End Module

6.2 ポリモーフィズム

ポリモーフィズムは、同じインターフェースで異なるオブジェクトに対して異なる処理を実行する仕組みです。

' ポリモーフィズムの例
Module Module1
    Sub Main()
        ' 異なるクラスのオブジェクトを同じ配列に格納
        Dim employees() As Employee = {
            New FullTimeEmployee(1, "田中", "開発", 300000, 50000),
            New ContractEmployee(2, "佐藤", "営業", 1500, 12),
            New FullTimeEmployee(3, "鈴木", "総務", 280000, 40000)
        }
        
        ' 正社員の残業と契約社員の勤務時間を設定
        DirectCast(employees(0), FullTimeEmployee).SetOvertime(15)
        DirectCast(employees(1), ContractEmployee).SetWorkedHours(150)
        DirectCast(employees(2), FullTimeEmployee).SetOvertime(25)
        
        ' ポリモーフィズムを使用して同じメソッドで異なる処理
        For Each emp As Employee In employees
            emp.ShowEmployeeInfo()  ' 各クラスのオーバーライドされたメソッドが呼ばれる
            Console.WriteLine()
        Next
        
        ' 給与計算も同様
        Dim totalSalary As Decimal = 0
        For Each emp As Employee In employees
            totalSalary += emp.CalculateSalary()
        Next
        Console.WriteLine("総人件費: " & totalSalary.ToString("C"))
        
        Console.ReadLine()
    End Sub
End Module

6.3 抽象クラス（MustInherit）

抽象クラスは、インスタンス化できないクラスで、派生クラスでの実装を強制する仕組みです。

' 抽象クラスの定義
Public MustInherit Class Shape
    Protected name As String
    
    Public Sub New(name As String)
        Me.name = name
    End Sub
    
    ' 抽象メソッド（派生クラスで必ず実装する必要がある）
    Public MustOverride Function CalculateArea() As Double
    Public MustOverride Function CalculatePerimeter() As Double
    
    ' 通常のメソッド（派生クラスで共通して使用）
    Public Overridable Sub ShowInfo()
        Console.WriteLine("図形名: " & name)
        Console.WriteLine("面積: " & CalculateArea().ToString("F2"))
        Console.WriteLine("周囲長: " & CalculatePerimeter().ToString("F2"))
    End Sub
End Class

' 抽象クラスを継承した具象クラス
Public Class Rectangle
    Inherits Shape
    
    Private width As Double
    Private height As Double
    
    Public Sub New(width As Double, height As Double)
        MyBase.New("長方形")
        Me.width = width
        Me.height = height
    End Sub
    
    ' 抽象メソッドの実装（必須）
    Public Overrides Function CalculateArea() As Double
        Return width * height
    End Function
    
    Public Overrides Function CalculatePerimeter() As Double
        Return 2 * (width + height)
    End Function
End Class

Public Class Circle
    Inherits Shape
    
    Private radius As Double
    Private Const PI As Double = 3.14159265359
    
    Public Sub New(radius As Double)
        MyBase.New("円")
        Me.radius = radius
    End Sub
    
    Public Overrides Function CalculateArea() As Double
        Return PI * radius * radius
    End Function
    
    Public Overrides Function CalculatePerimeter() As Double
        Return 2 * PI * radius
    End Function
End Class

6.4 インターフェース

インターフェースは、クラスが実装すべきメソッドの仕様を定義する契約です。

' インターフェースの定義
Public Interface IDrawable
    Sub Draw()
    Sub SetColor(color As String)
End Interface

Public Interface IMovable
    Sub Move(x As Integer, y As Integer)
    ReadOnly Property Position() As String
End Interface

' インターフェースを実装するクラス
Public Class GraphicRectangle
    Implements IDrawable, IMovable
    
    Private width As Integer
    Private height As Integer
    Private color As String
    Private x As Integer
    Private y As Integer
    
    Public Sub New(w As Integer, h As Integer)
        width = w
        height = h
        color = "黒"
        x = 0
        y = 0
    End Sub
    
    ' IDrawableインターフェースの実装
    Public Sub Draw() Implements IDrawable.Draw
        Console.WriteLine($"{color}色の長方形({width}×{height})を({x}, {y})に描画")
    End Sub
    
    Public Sub SetColor(color As String) Implements IDrawable.SetColor
        Me.color = color
    End Sub
    
    ' IMovableインターフェースの実装
    Public Sub Move(x As Integer, y As Integer) Implements IMovable.Move
        Me.x = x
        Me.y = y
        Console.WriteLine($"位置を({x}, {y})に移動しました")
    End Sub
    
    Public ReadOnly Property Position() As String Implements IMovable.Position
        Get
            Return $"({x}, {y})"
        End Get
    End Property
End Class

Public Class GraphicCircle
    Implements IDrawable, IMovable
    
    Private radius As Integer
    Private color As String
    Private x As Integer
    Private y As Integer
    
    Public Sub New(r As Integer)
        radius = r
        color = "黒"
        x = 0
        y = 0
    End Sub
    
    Public Sub Draw() Implements IDrawable.Draw
        Console.WriteLine($"{color}色の円(半径{radius})を({x}, {y})に描画")
    End Sub
    
    Public Sub SetColor(color As String) Implements IDrawable.SetColor
        Me.color = color
    End Sub
    
    Public Sub Move(x As Integer, y As Integer) Implements IMovable.Move
        Me.x = x
        Me.y = y
        Console.WriteLine($"位置を({x}, {y})に移動しました")
    End Sub
    
    Public ReadOnly Property Position() As String Implements IMovable.Position
        Get
            Return $"({x}, {y})"
        End Get
    End Property
End Class
実習 6-2: グラフィック描画システム

インターフェースとポリモーフィズムを使用したグラフィック描画システムを作成しましょう。

プログラムコード
Module Module1
    Sub Main()
        ' 異なるクラスのオブジェクトを同じインターフェース型で管理
        Dim drawableObjects As New List(Of IDrawable)()
        
        Dim rect As New GraphicRectangle(100, 50)
        Dim circle As New GraphicCircle(30)
        
        drawableObjects.Add(rect)
        drawableObjects.Add(circle)
        
        ' 初期描画
        Console.WriteLine("=== 初期描画 ===")
        For Each obj As IDrawable In drawableObjects
            obj.Draw()
        Next
        Console.WriteLine()
        
        ' 色を変更して描画
        Console.WriteLine("=== 色変更後 ===")
        rect.SetColor("赤")
        circle.SetColor("青")
        
        For Each obj As IDrawable In drawableObjects
            obj.Draw()
        Next
        Console.WriteLine()
        
        ' 移動操作（IMovableとしてキャスト）
        Console.WriteLine("=== 移動操作 ===")
        DirectCast(rect, IMovable).Move(10, 20)
        DirectCast(circle, IMovable).Move(50, 100)
        
        Console.WriteLine()
        Console.WriteLine("=== 移動後の描画 ===")
        For Each obj As IDrawable In drawableObjects
            obj.Draw()
            ' 位置情報も表示
            Dim movable As IMovable = DirectCast(obj, IMovable)
            Console.WriteLine("現在位置: " & movable.Position)
        Next
        
        Console.ReadLine()
    End Sub
End Module

6.5 メソッドのオーバーロード

同じ名前のメソッドで、異なる引数を持つメソッドを複数定義することができます。

Public Class Calculator
    ' 整数の加算
    Public Function Add(a As Integer, b As Integer) As Integer
        Console.WriteLine("整数の加算を実行")
        Return a + b
    End Function
    
    ' 小数の加算
    Public Function Add(a As Double, b As Double) As Double
        Console.WriteLine("小数の加算を実行")
        Return a + b
    End Function
    
    ' 3つの整数の加算
    Public Function Add(a As Integer, b As Integer, c As Integer) As Integer
        Console.WriteLine("3つの整数の加算を実行")
        Return a + b + c
    End Function
    
    ' 文字列の連結（加算のオーバーロード）
    Public Function Add(a As String, b As String) As String
        Console.WriteLine("文字列の連結を実行")
        Return a & b
    End Function
    
    ' オプション引数を使用したオーバーロード
    Public Function Multiply(a As Integer, b As Integer, Optional c As Integer = 1) As Integer
        Console.WriteLine($"乗算を実行: {a} × {b} × {c}")
        Return a * b * c
    End Function
End Class

' 使用例
Module Module1
    Sub Main()
        Dim calc As New Calculator()
        
        ' コンパイラが引数の型と数によって適切なメソッドを選択
        Console.WriteLine("結果: " & calc.Add(5, 3))           ' 整数版
        Console.WriteLine("結果: " & calc.Add(5.5, 3.2))       ' 小数版
        Console.WriteLine("結果: " & calc.Add(1, 2, 3))        ' 3引数版
        Console.WriteLine("結果: " & calc.Add("Hello", "World")) ' 文字列版
        
        Console.WriteLine("結果: " & calc.Multiply(2, 3))       ' オプション引数省略
        Console.WriteLine("結果: " & calc.Multiply(2, 3, 4))    ' オプション引数指定
        
        Console.ReadLine()
    End Sub
End Module

6.6 抽象クラスとインターフェースの使い分け

適切なデザインパターンを選択することが重要です。

項目 抽象クラス（MustInherit） インターフェース
継承 単一継承のみ 複数実装可能
実装 具象メソッドも持てる メソッドの実装は含まない
フィールド インスタンスフィールドを持てる フィールドは持てない
アクセス修飾子 Public, Protected, Private使用可 Publicのみ
使用場面 関連するクラス群の共通基盤 異なるクラス間の共通契約
設計のガイドライン
  • 抽象クラスを使う場合：関連するクラス群で共通のコードがある場合
  • インターフェースを使う場合：異なるクラスに同じ機能を提供させたい場合
  • 複数のインターフェースを実装することで、多重継承に似た効果が得られる
理解度確認クイズ
  1. 継承で親クラスのコンストラクタを呼び出すキーワードは何ですか？
  2. 親クラスのメソッドを子クラスで再定義することを何と呼びますか？
  3. 同じインターフェースで異なる処理を実行する仕組みを何と呼びますか？
  4. インスタンス化できないクラスを定義するキーワードは何ですか？
  5. クラスが実装すべきメソッドの仕様を定義する仕組みは何ですか？
  6. 同じ名前で異なる引数を持つメソッドを定義することを何と呼びますか？
  7. インターフェースを実装する際に使用するキーワードは何ですか？
  8. 仮想メソッドを定義する際に使用するキーワードは何ですか？
