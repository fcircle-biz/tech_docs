5.1 オブジェクト指向プログラミングとは

オブジェクト指向プログラミング（OOP: Object-Oriented Programming）は、現実世界の「もの」をプログラムで表現する考え方です。例えば、車、人、銀行口座など、私たちの周りにある様々な「もの」をプログラム内でオブジェクトとして表現します。

オブジェクト指向の3つの基本概念

  • カプセル化: データと処理をまとめて隠蔽する
  • 継承: 既存のクラスを元に新しいクラスを作る
  • ポリモーフィズム: 同じインターフェースで異なる動作を実現

5.2 クラスとオブジェクト

クラスは、オブジェクトの設計図や型です。オブジェクトは、クラスから作られた実際のインスタンス（実体）です。例えば、「車」がクラスだとすると、「田中さんのプリウス」や「佐藤さんのアクア」がオブジェクトです。

クラスの基本構造

' 基本的なクラス定義
Public Class Car
    ' フィールド（内部データ）
    Private brand As String
    Private model As String
    Private year As Integer
    
    ' プロパティ（外部からのアクセス）
    Public Property Brand As String
        Get
            Return brand
        End Get
        Set(value As String)
            brand = value
        End Set
    End Property
    
    ' メソッド（動作・処理）
    Public Sub StartEngine()
        Console.WriteLine("エンジンが始動しました")
    End Sub
End Class
実習 5-1: 学生クラスの作成

学生の情報を管理するクラスを作成し、オブジェクトを使ってみましょう。

サンプルコード
Public Class Student
    ' プライベートフィールド
    Private _name As String
    Private _age As Integer
    Private _studentId As String
    Private _grade As Double
    
    ' パブリックプロパティ
    Public Property Name As String
        Get
            Return _name
        End Get
        Set(value As String)
            If Not String.IsNullOrEmpty(value) Then
                _name = value
            End If
        End Set
    End Property
    
    Public Property Age As Integer
        Get
            Return _age
        End Get
        Set(value As Integer)
            If value >= 0 Then
                _age = value
            End If
        End Set
    End Property
    
    Public Property StudentId As String
        Get
            Return _studentId
        End Get
        Set(value As String)
            _studentId = value
        End Set
    End Property
    
    Public Property Grade As Double
        Get
            Return _grade
        End Get
        Set(value As Double)
            If value >= 0 And value <= 100 Then
                _grade = value
            End If
        End Set
    End Property
    
    ' メソッド
    Public Sub ShowInfo()
        Console.WriteLine("=== 学生情報 ===")
        Console.WriteLine($"名前: {_name}")
        Console.WriteLine($"年齢: {_age}歳")
        Console.WriteLine($"学籍番号: {_studentId}")
        Console.WriteLine($"成績: {_grade}点")
    End Sub
    
    Public Function GetGradeLevel() As String
        If _grade >= 90 Then
            Return "A"
        ElseIf _grade >= 80 Then
            Return "B"
        ElseIf _grade >= 70 Then
            Return "C"
        ElseIf _grade >= 60 Then
            Return "D"
        Else
            Return "F"
        End If
    End Function
End Class

Module Program
    Sub Main(args As String())
        ' オブジェクトの作成
        Dim student1 As New Student()
        Dim student2 As New Student()
        
        ' データの設定
        student1.Name = "田中太郎"
        student1.Age = 20
        student1.StudentId = "S001"
        student1.Grade = 85
        
        student2.Name = "佐藤花子"
        student2.Age = 19
        student2.StudentId = "S002"
        student2.Grade = 92
        
        ' 情報の表示
        student1.ShowInfo()
        Console.WriteLine($"成績レベル: {student1.GetGradeLevel()}")
        
        Console.WriteLine()
        
        student2.ShowInfo()
        Console.WriteLine($"成績レベル: {student2.GetGradeLevel()}")
        
        Console.ReadKey()
    End Sub
End Module

5.3 コンストラクタ

コンストラクタは、オブジェクトが作成される際に自動的に実行される特別なメソッドです。オブジェクトの初期化処理を行い、必要なデータを設定します。

コンストラクタの種類

  • デフォルトコンストラクタ: パラメータなしのコンストラクタ
  • パラメータ付きコンストラクタ: 初期値を受け取るコンストラクタ
  • オーバーロードコンストラクタ: 複数の異なるコンストラクタ
Public Class BankAccount
    Private _accountNumber As String
    Private _balance As Decimal
    Private _ownerName As String
    
    ' デフォルトコンストラクタ
    Public Sub New()
        _accountNumber = "000000"
        _balance = 0
        _ownerName = "Unknown"
        Console.WriteLine("新しい口座が作成されました（デフォルト）")
    End Sub
    
    ' パラメータ付きコンストラクタ
    Public Sub New(accountNumber As String, ownerName As String)
        _accountNumber = accountNumber
        _ownerName = ownerName
        _balance = 0
        Console.WriteLine($"{ownerName}さんの口座（{accountNumber}）が作成されました")
    End Sub
    
    ' 初期残高付きコンストラクタ
    Public Sub New(accountNumber As String, ownerName As String, initialBalance As Decimal)
        _accountNumber = accountNumber
        _ownerName = ownerName
        _balance = initialBalance
        Console.WriteLine($"{ownerName}さんの口座が{initialBalance:C}で開設されました")
    End Sub
    
    ' プロパティ
    Public ReadOnly Property AccountNumber As String
        Get
            Return _accountNumber
        End Get
    End Property
    
    Public ReadOnly Property Balance As Decimal
        Get
            Return _balance
        End Get
    End Property
    
    Public ReadOnly Property OwnerName As String
        Get
            Return _ownerName
        End Get
    End Property
    
    ' メソッド
    Public Sub Deposit(amount As Decimal)
        If amount > 0 Then
            _balance += amount
            Console.WriteLine($"{amount:C}を入金しました。残高: {_balance:C}")
        End If
    End Sub
    
    Public Sub Withdraw(amount As Decimal)
        If amount > 0 AndAlso amount <= _balance Then
            _balance -= amount
            Console.WriteLine($"{amount:C}を出金しました。残高: {_balance:C}")
        Else
            Console.WriteLine("出金できません。残高不足です。")
        End If
    End Sub
End Class

5.4 カプセル化

カプセル化は、オブジェクトの内部データと処理を隠蔽し、外部からの直接アクセスを制限する仕組みです。これにより、データの整合性を保ち、プログラムの安全性を向上させます。

アクセス修飾子

修飾子 アクセス範囲 説明
Private 同じクラス内のみ 最も制限的。内部実装の隠蔽に使用
Protected 同じクラスと継承したクラス 継承関係のあるクラスから使用可能
Friend 同じアセンブリ内 同じプロジェクト内のクラスから使用可能
Public どこからでも 制限なし。外部インターフェースに使用
理解度確認クイズ
  1. クラスとオブジェクトの関係で正しいのは？
    a) クラスはオブジェクトの実体
    b) オブジェクトはクラスの設計図
    c) クラスはオブジェクトの設計図
    d) 同じもの
  2. コンストラクタの役割は？
    a) オブジェクトの破棄
    b) オブジェクトの初期化
    c) データの計算
    d) メソッドの実行
  3. 最も制限的なアクセス修飾子は？
    a) Public
    b) Friend
    c) Protected
    d) Private
  4. カプセル化の利点は？
    a) 実行速度の向上
    b) データの隠蔽と保護
    c) メモリ使用量の削減
    d) コードの短縮
解答を表示
  1. c) クラスはオブジェクトの設計図 - クラスは型定義、オブジェクトはその実体です。
  2. b) オブジェクトの初期化 - コンストラクタはオブジェクト作成時の初期設定を行います。
  3. d) Private - Privateは同じクラス内からのみアクセス可能です。
  4. b) データの隠蔽と保護 - カプセル化により内部データを安全に管理できます。
