第6章: データベース連携（ADO.NET）

データベースとWebアプリケーションの橋渡し

この章で学ぶこと
  • ADO.NETの基本概念と役割について理解する
  • データベース接続の仕組みとConnection文字列の重要性を学ぶ
  • DataSetとDataReaderの違いと使い分けを理解する
  • 基本的なCRUD（作成・読取・更新・削除）操作の実装方法を習得する
  • データベースアクセスにおけるセキュリティとエラーハンドリング

6.1 ADO.NETとは何か

ADO.NET（ActiveX Data Objects .NET）は、.NET Frameworkが提供するデータアクセス技術です。これまでのWindows Formsアプリケーションがローカルのファイルやメモリ内のデータを扱っていたのに対し、Webアプリケーションでは多くの場合、データベースという「データの宝庫」にアクセスする必要があります。

なぜデータベースが必要なのか？

Webアプリケーションの特徴を考えてみましょう：

  • 多数のユーザー：同時に何百、何千人ものユーザーがアクセスする可能性
  • データの永続化：ユーザーが入力した情報を長期間保存する必要
  • データの共有：複数のユーザー間で同じデータを共有
  • データの整合性：矛盾のないデータを維持する必要
sequenceDiagram participant WebForm as Web Form participant ADO as ADO.NET participant DB as データベース WebForm->>ADO: データ要求 ADO->>DB: SQL実行 DB-->>ADO: 結果セット ADO-->>WebForm: DataSetまたはDataReader WebForm->>WebForm: データ表示・処理

6.2 ADO.NETの主要コンポーネント

ADO.NETは、データベースアクセスを効率的に行うための複数のコンポーネントから構成されています。それぞれのコンポーネントには特定の役割があり、適切に使い分けることが重要です。

Connection（接続）

Connectionは、アプリケーションとデータベースを結ぶ「道路」のような存在です。この道路が確立されていないと、データのやり取りはできません。


' SQL Server接続の例
Dim connectionString As String = "Server=localhost;Database=SampleDB;Integrated Security=true"
Dim connection As New SqlConnection(connectionString)

Command（コマンド）

Commandは、データベースに対して「何をしたいか」を伝える指示書です。SELECT文でデータを取得したり、INSERT文でデータを追加したりします。


Dim command As New SqlCommand("SELECT * FROM Users", connection)

DataReader vs DataSet

データの取得方法には2つの主要な選択肢があります：

graph TD A[データ取得方法] --> B[DataReader] A --> C[DataSet] B --> B1["高速・軽量"] B --> B2["順次読み取り専用"] B --> B3["接続型"] C --> C1["柔軟性が高い"] C --> C2["メモリ内テーブル"] C --> C3["非接続型"]

6.3 データベース接続の実装

実際にデータベースに接続するためには、接続文字列（Connection String）と呼ばれる設定情報が必要です。これは、データベースサーバーの住所、使用するデータベース名、認証情報などを含む重要な情報です。

セキュリティに関する重要な注意

接続文字列にはデータベースへのアクセス権限が含まれているため、ソースコード内に直接書くことは避け、設定ファイル（web.config）に記述することが重要です。

実習 6-1: 基本的なデータベース接続

まずは、データベースに接続して、接続が成功するかを確認する簡単なプログラムを作成してみましょう。

手順
  1. web.configファイルに接続文字列を追加
  2. 接続テスト用のWebフォームを作成
  3. 接続の開閉を正しく処理するコードを実装
web.config設定例
<connectionStrings>
  <add name="DefaultConnection" 
       connectionString="Server=(localdb)\MSSQLLocalDB;Database=SampleDB;Integrated Security=true" 
       providerName="System.Data.SqlClient" />
</connectionStrings>
VB.NET実装例

Imports System.Data.SqlClient

Partial Class DatabaseTest
    Inherits System.Web.UI.Page

    Protected Sub Page_Load(sender As Object, e As EventArgs) Handles Me.Load
        TestDatabaseConnection()
    End Sub

    Private Sub TestDatabaseConnection()
        ' 接続文字列を設定ファイルから取得
        Dim connectionString As String = ConfigurationManager.ConnectionStrings("DefaultConnection").ConnectionString
        
        Try
            Using connection As New SqlConnection(connectionString)
                connection.Open()
                Response.Write("データベース接続成功！")
                
                ' 接続状態を確認
                If connection.State = ConnectionState.Open Then
                    Response.Write("<br>接続状態：開いています")
                End If
            End Using ' Usingブロックを出ると自動的にconnection.Close()が呼ばれる
        Catch ex As Exception
            Response.Write("接続エラー：" & ex.Message)
        End Try
    End Sub
End Class
期待される結果

正常に接続できれば「データベース接続成功！」と「接続状態：開いています」が表示されます。エラーが発生した場合は、接続文字列やデータベースの設定を確認してください。

6.4 DataReaderを使用したデータ取得

DataReaderは、データベースからデータを「ストリーミング」方式で読み取るためのコンポーネントです。一度に1行ずつデータを処理するため、メモリ効率が良く、大量のデータを扱う際に適しています。

DataReaderの特徴
  • 高速：データベースから直接データを読み取るため処理が速い
  • 省メモリ：一度に1行のみメモリに保持
  • 順次アクセス：データを先頭から順番にのみ読み取り可能
  • 読み取り専用：データの変更はできない
  • 接続型：データベース接続を維持したままデータを読み取り
実習 6-2: DataReaderを使用したユーザー一覧表示

DataReaderを使用して、データベースからユーザー情報を取得し、Webページに表示してみましょう。

事前準備：テーブル作成

CREATE TABLE Users (
    UserID int IDENTITY(1,1) PRIMARY KEY,
    UserName nvarchar(50) NOT NULL,
    Email nvarchar(100) NOT NULL,
    CreatedDate datetime DEFAULT GETDATE()
)

-- サンプルデータ挿入
INSERT INTO Users (UserName, Email) VALUES 
('田中太郎', 'tanaka@example.com'),
('佐藤花子', 'sato@example.com'),
('鈴木一郎', 'suzuki@example.com')
実装例

Protected Sub btnLoadUsers_Click(sender As Object, e As EventArgs) Handles btnLoadUsers.Click
    Dim connectionString As String = ConfigurationManager.ConnectionStrings("DefaultConnection").ConnectionString
    Dim html As New StringBuilder()
    
    html.Append("<table class='table table-striped'>")
    html.Append("<thead><tr><th>ID</th><th>名前</th><th>メールアドレス</th><th>登録日</th></tr></thead>")
    html.Append("<tbody>")
    
    Try
        Using connection As New SqlConnection(connectionString)
            Dim sql As String = "SELECT UserID, UserName, Email, CreatedDate FROM Users ORDER BY CreatedDate DESC"
            Using command As New SqlCommand(sql, connection)
                connection.Open()
                
                Using reader As SqlDataReader = command.ExecuteReader()
                    While reader.Read()
                        html.Append("<tr>")
                        html.Append("<td>" & reader("UserID").ToString() & "</td>")
                        html.Append("<td>" & reader("UserName").ToString() & "</td>")
                        html.Append("<td>" & reader("Email").ToString() & "</td>")
                        html.Append("<td>" & Convert.ToDateTime(reader("CreatedDate")).ToString("yyyy/MM/dd HH:mm") & "</td>")
                        html.Append("</tr>")
                    End While
                End Using
            End Using
        End Using
        
        html.Append("</tbody></table>")
        divUserList.InnerHtml = html.ToString()
        
    Catch ex As Exception
        divUserList.InnerHtml = "<div class='alert alert-danger'>エラー: " & ex.Message & "</div>"
    End Try
End Sub
期待される結果

ユーザー一覧がテーブル形式で表示され、各ユーザーのID、名前、メールアドレス、登録日が確認できます。

6.5 CRUD操作の実装

CRUDとは、データベース操作の基本となる4つの操作の頭文字を取った言葉です：

  • Create（作成）：新しいデータの追加
  • Read（読取）：既存のデータの取得
  • Update（更新）：既存のデータの修正
  • Delete（削除）：不要なデータの削除
実習 6-3: 新規ユーザーの登録（Create操作）

ユーザーから入力された情報をデータベースに保存する機能を実装してみましょう。

実装例

Protected Sub btnAddUser_Click(sender As Object, e As EventArgs) Handles btnAddUser.Click
    ' 入力値の検証
    If String.IsNullOrEmpty(txtUserName.Text) OrElse String.IsNullOrEmpty(txtEmail.Text) Then
        lblMessage.Text = "名前とメールアドレスを入力してください。"
        lblMessage.CssClass = "text-danger"
        Return
    End If
    
    Dim connectionString As String = ConfigurationManager.ConnectionStrings("DefaultConnection").ConnectionString
    
    Try
        Using connection As New SqlConnection(connectionString)
            ' パラメータ化クエリを使用（SQLインジェクション対策）
            Dim sql As String = "INSERT INTO Users (UserName, Email) VALUES (@UserName, @Email)"
            Using command As New SqlCommand(sql, connection)
                ' パラメータの追加
                command.Parameters.AddWithValue("@UserName", txtUserName.Text.Trim())
                command.Parameters.AddWithValue("@Email", txtEmail.Text.Trim())
                
                connection.Open()
                Dim rowsAffected As Integer = command.ExecuteNonQuery()
                
                If rowsAffected > 0 Then
                    lblMessage.Text = "ユーザーを正常に登録しました。"
                    lblMessage.CssClass = "text-success"
                    
                    ' 入力フィールドをクリア
                    txtUserName.Text = ""
                    txtEmail.Text = ""
                Else
                    lblMessage.Text = "ユーザーの登録に失敗しました。"
                    lblMessage.CssClass = "text-warning"
                End If
            End Using
        End Using
    Catch ex As Exception
        lblMessage.Text = "エラー: " & ex.Message
        lblMessage.CssClass = "text-danger"
    End Try
End Sub
セキュリティのポイント

パラメータ化クエリを使用することで、SQLインジェクション攻撃を防ぐことができます。直接SQL文に値を埋め込む方法は危険なので、必ず@パラメータを使用してください。

実習 6-4: ユーザー情報の更新（Update操作）

既存のユーザー情報を更新する機能を実装してみましょう。

実装例

Protected Sub btnUpdateUser_Click(sender As Object, e As EventArgs) Handles btnUpdateUser.Click
    Dim userID As Integer
    If Not Integer.TryParse(txtUserID.Text, userID) OrElse userID <= 0 Then
        lblUpdateMessage.Text = "有効なユーザーIDを入力してください。"
        lblUpdateMessage.CssClass = "text-danger"
        Return
    End If
    
    Dim connectionString As String = ConfigurationManager.ConnectionStrings("DefaultConnection").ConnectionString
    
    Try
        Using connection As New SqlConnection(connectionString)
            Dim sql As String = "UPDATE Users SET UserName = @UserName, Email = @Email WHERE UserID = @UserID"
            Using command As New SqlCommand(sql, connection)
                command.Parameters.AddWithValue("@UserName", txtUpdateUserName.Text.Trim())
                command.Parameters.AddWithValue("@Email", txtUpdateEmail.Text.Trim())
                command.Parameters.AddWithValue("@UserID", userID)
                
                connection.Open()
                Dim rowsAffected As Integer = command.ExecuteNonQuery()
                
                If rowsAffected > 0 Then
                    lblUpdateMessage.Text = "ユーザー情報を正常に更新しました。"
                    lblUpdateMessage.CssClass = "text-success"
                Else
                    lblUpdateMessage.Text = "指定されたユーザーが見つかりません。"
                    lblUpdateMessage.CssClass = "text-warning"
                End If
            End Using
        End Using
    Catch ex As Exception
        lblUpdateMessage.Text = "エラー: " & ex.Message
        lblUpdateMessage.CssClass = "text-danger"
    End Try
End Sub

6.6 エラーハンドリングとベストプラクティス

データベースアクセスは、ネットワークの問題、データベースサーバーの負荷、SQL構文エラーなど、様々な要因でエラーが発生する可能性があります。適切なエラーハンドリングは、ユーザー体験の向上とシステムの安定性確保に不可欠です。

データベースアクセスのベストプラクティス
  • Usingステートメントの使用：リソースの確実な解放
  • パラメータ化クエリ：SQLインジェクション攻撃の防止
  • 接続の最小化：必要な時だけ接続を開き、速やかに閉じる
  • 例外処理：適切なTry-Catchブロックの使用
  • ログ出力：エラーの原因を特定するための情報記録
理解度確認クイズ
  1. ADO.NETの役割：ADO.NETがWebアプリケーション開発において果たす役割と、従来のファイルベースのデータ管理と比較した利点を3つ説明してください。
  2. DataReaderとDataSetの使い分け：以下のシナリオで、DataReaderとDataSetのどちらを使用すべきか理由と共に答えてください：
    • 10万件のデータを順次処理してレポートを生成する場合
    • 50件程度のデータを画面に表示し、ユーザーが編集できるようにする場合
  3. セキュリティ対策：なぜパラメータ化クエリを使用する必要があるのでしょうか。SQLインジェクション攻撃の危険性と合わせて説明してください。
  4. リソース管理：Usingステートメントを使用する理由と、使用しなかった場合に発生する可能性のある問題について説明してください。
  5. エラーハンドリング：データベースアクセス時に発生する可能性のあるエラーを3つ挙げ、それぞれに対する適切な対処方法を説明してください。
