第2章: Web FormsとWebコントロール

Webコントロールの世界を理解する

この章で学ぶこと
  • HTMLコントロールとサーバーコントロールの違いとそれぞれの特徴
  • 基本的なWebコントロール（Label、TextBox、Button等）の使用方法
  • VB.NETのWindows Formsコントロールとの違いと類似点
  • コントロールのプロパティ設定方法とイベント処理
  • Webコントロールの階層構造と継承関係の理解

2.1 HTMLコントロール vs サーバーコントロール

Web Formsでは、2つの主要なコントロールタイプが存在します。それぞれの特徴と使い分けを理解することが重要です。

graph TD A["Webコントロール"] --> B["HTMLコントロール"] A --> C["サーバーコントロール
(Webコントロール)"] B --> D["<input type='text'>
通常のHTMLタグ"] B --> E["<input type='text' runat='server'>
サーバー処理可能"] C --> F["<asp:TextBox>
ASP.NET専用コントロール"] C --> G["高度な機能
自動ViewState管理"]

HTMLコントロール

通常のHTMLタグをそのまま使用するか、runat="server"属性を追加してサーバーサイドで制御できるようにしたものです。

<!-- 通常のHTMLタグ（サーバーサイドから制御不可） -->
<input type="text" id="txtName" />
<input type="button" value="クリック" onclick="alert('Hello!');" />

<!-- HTMLサーバーコントロール（サーバーサイドから制御可能） -->
<input type="text" id="txtName" runat="server" />
<input type="button" id="btnSubmit" value="送信" runat="server" />

ASP.NETサーバーコントロール（Webコントロール）

ASP.NET専用のコントロールで、より高度な機能と開発支援機能を提供します。

<asp:TextBox ID="txtUserName" runat="server" 
             placeholder="ユーザー名を入力" 
             CssClass="form-control"></asp:TextBox>
             
<asp:Button ID="btnLogin" runat="server" 
            Text="ログイン" 
            CssClass="btn btn-primary"
            OnClick="btnLogin_Click" />
サーバーコントロールの利点
  • IntelliSense支援：Visual Studioでの入力支援が充実
  • プロパティベースの設定：直感的なプロパティ操作
  • 自動ViewState管理：状態が自動的に維持される
  • イベント処理の統合：VB.NETのイベント処理構文をそのまま使用
  • バリデーション統合：検証機能が組み込まれている

2.2 基本的なWebコントロールの詳細

最も頻繁に使用される基本的なWebコントロールについて、その特徴と使用方法を詳しく学習しましょう。

Label コントロール

テキストの表示に使用します。Windows FormsのLabelコントロールと概念は同じですが、HTML出力という点で違いがあります。

<asp:Label ID="lblWelcome" runat="server" 
           Text="ようこそ" 
           ForeColor="Blue" 
           Font-Size="Large"></asp:Label>

コードビハインドでの制御：

' ページ読み込み時にラベルのテキストを設定
Protected Sub Page_Load(sender As Object, e As EventArgs) Handles Me.Load
    lblWelcome.Text = "ようこそ、" & System.Environment.UserName & "さん"
    lblWelcome.ForeColor = System.Drawing.Color.DarkBlue
End Sub

TextBox コントロール

ユーザーからの文字入力を受け取るコントロールです。単一行、複数行、パスワード入力に対応します。

<!-- 単一行テキストボックス -->
<asp:TextBox ID="txtName" runat="server" 
             placeholder="氏名を入力してください"
             MaxLength="50"></asp:TextBox>

<!-- 複数行テキストボックス -->
<asp:TextBox ID="txtComments" runat="server" 
             TextMode="MultiLine" 
             Rows="4" 
             Columns="50"></asp:TextBox>

<!-- パスワード入力 -->
<asp:TextBox ID="txtPassword" runat="server" 
             TextMode="Password"></asp:TextBox>

Button コントロール

ユーザーの操作を受け取り、サーバーサイドでの処理を実行するトリガーとなります。

<asp:Button ID="btnSubmit" runat="server" 
            Text="送信" 
            OnClick="btnSubmit_Click"
            CssClass="btn btn-primary"
            UseSubmitBehavior="true" />
実習 2-1: 基本コントロールを使用したフォーム作成

名前、年齢、コメントを入力できる簡単なフォームを作成し、送信時に入力内容を表示する機能を実装します。

手順
  1. 新しいWeb Formページ「UserInfo.aspx」を作成
  2. 以下のコントロールを配置：
    • Label: "ユーザー情報入力"（タイトル用）
    • Label: "氏名:"
    • TextBox: txtName
    • Label: "年齢:"
    • TextBox: txtAge
    • Label: "コメント:"
    • TextBox: txtComments（MultiLine）
    • Button: btnSubmit（"送信"）
    • Label: lblResult（結果表示用、初期は空）
  3. コードビハインドでボタンクリックイベントを実装
UserInfo.aspx のマークアップ例
<div style="padding: 20px;">
    <h2><asp:Label ID="lblTitle" runat="server" Text="ユーザー情報入力"></asp:Label></h2>
    
    <table>
        <tr>
            <td><asp:Label ID="Label1" runat="server" Text="氏名:"></asp:Label></td>
            <td><asp:TextBox ID="txtName" runat="server"></asp:TextBox></td>
        </tr>
        <tr>
            <td><asp:Label ID="Label2" runat="server" Text="年齢:"></asp:Label></td>
            <td><asp:TextBox ID="txtAge" runat="server"></asp:TextBox></td>
        </tr>
        <tr>
            <td><asp:Label ID="Label3" runat="server" Text="コメント:"></asp:Label></td>
            <td><asp:TextBox ID="txtComments" runat="server" 
                             TextMode="MultiLine" Rows="3"></asp:TextBox></td>
        </tr>
        <tr>
            <td colspan="2">
                <asp:Button ID="btnSubmit" runat="server" Text="送信" />
            </td>
        </tr>
    </table>

    <asp:Label ID="lblResult" runat="server"></asp:Label>
</div>
UserInfo.aspx.vb のコードビハインド例
Protected Sub btnSubmit_Click(sender As Object, e As EventArgs) Handles btnSubmit.Click
    ' 入力値を取得
    Dim name As String = txtName.Text.Trim()
    Dim age As String = txtAge.Text.Trim()
    Dim comments As String = txtComments.Text.Trim()
    
    ' 結果を表示
    Dim result As New System.Text.StringBuilder()
    result.AppendLine("<h3>入力された情報</h3>")
    result.AppendLine("<p><strong>氏名:</strong> " & name & "</p>")
    result.AppendLine("<p><strong>年齢:</strong> " & age & "</p>")
    result.AppendLine("<p><strong>コメント:</strong><br/>" & comments.Replace(vbCrLf, "<br/>") & "</p>")
    
    lblResult.Text = result.ToString()
End Sub

2.3 選択系コントロール

ユーザーが選択肢から値を選ぶためのコントロールについて学習します。

CheckBox コントロール

真偽値（Yes/No、True/False）の選択に使用します。

<asp:CheckBox ID="chkAgree" runat="server" 
              Text="利用規約に同意します" 
              Checked="false" />

<asp:CheckBox ID="chkNewsletter" runat="server" 
              Text="ニュースレターを受信する" />

RadioButton コントロール

複数の選択肢から一つを選ぶ場合に使用します。同じGroupNameを持つRadioButtonは相互排他的になります。

<asp:RadioButton ID="rbMale" runat="server" 
                 GroupName="Gender" 
                 Text="男性" />
<asp:RadioButton ID="rbFemale" runat="server" 
                 GroupName="Gender" 
                 Text="女性" />

DropDownList コントロール

複数の選択肢をドロップダウン形式で提供します。

<asp:DropDownList ID="ddlPrefecture" runat="server">
    <asp:ListItem Text="-- 選択してください --" Value=""></asp:ListItem>
    <asp:ListItem Text="東京都" Value="Tokyo"></asp:ListItem>
    <asp:ListItem Text="大阪府" Value="Osaka"></asp:ListItem>
    <asp:ListItem Text="神奈川県" Value="Kanagawa"></asp:ListItem>
</asp:DropDownList>

コードビハインドでの動的な項目追加：

Protected Sub Page_Load(sender As Object, e As EventArgs) Handles Me.Load
    If Not Page.IsPostBack Then
        ' 初回読み込み時のみ実行
        ddlPrefecture.Items.Clear()
        ddlPrefecture.Items.Add(New ListItem("-- 選択してください --", ""))
        ddlPrefecture.Items.Add(New ListItem("北海道", "Hokkaido"))
        ddlPrefecture.Items.Add(New ListItem("青森県", "Aomori"))
        ddlPrefecture.Items.Add(New ListItem("岩手県", "Iwate"))
        ' 他の都道府県も同様に追加...
    End If
End Sub

2.4 VB.NETのWindows Formsコントロールとの比較

VB.NETのWindows Formsコントロールと比較して、Webコントロールの特徴を理解しましょう。

graph LR A["Windows Forms"] --> B["直接描画
リアルタイム更新"] A --> C["ローカルメモリ
状態維持"] A --> D["イベント即座実行"] E["Web Forms"] --> F["HTML出力
ブラウザ描画"] E --> G["ViewState
状態管理"] E --> H["サーバー往復
PostBack"]
主な違いのポイント
項目 Windows Forms Web Forms
実行場所 クライアントマシン サーバーサイド（結果をHTMLで送信）
状態管理 メモリ内で自動維持 ViewState、Session等で管理
イベント処理 即座に実行 サーバーへのPostBack後実行
表示方法 直接画面描画 HTMLとしてブラウザに送信
実習 2-2: 選択系コントロールを使用した登録フォーム

CheckBox、RadioButton、DropDownListを組み合わせた会員登録フォームを作成します。

手順
  1. 新しいページ「MemberRegistration.aspx」を作成
  2. 以下の要素を含むフォームを作成：
    • 氏名（TextBox）
    • 性別（RadioButton × 2）
    • 居住地（DropDownList）
    • 興味のある分野（CheckBox × 3）
    • メール受信許可（CheckBox）
    • 登録ボタン（Button）
  3. 登録ボタン押下時に全ての入力内容を表示
期待される結果

ユーザーが選択・入力した内容が整理されて表示され、各コントロールの値を正しく取得できることが確認できます。

2.5 コントロールの共通プロパティとメソッド

すべてのWebコントロールが継承する共通の機能について理解しましょう。

重要な共通プロパティ

  • ID：コントロールの一意識別子（コードから参照する際に使用）
  • Visible：コントロールの表示・非表示制御
  • Enabled：コントロールの有効・無効制御
  • CssClass：適用するCSSクラス名
  • ToolTip：マウスオーバー時に表示するヒント
  • Width / Height：コントロールのサイズ
' コードビハインドでの動的制御例
Protected Sub Page_Load(sender As Object, e As EventArgs) Handles Me.Load
    ' コントロールの表示/非表示
    lblMessage.Visible = True
    txtSecret.Visible = False
    
    ' コントロールの有効/無効
    btnSubmit.Enabled = True
    txtReadOnly.Enabled = False
    
    ' CSSクラスの動的適用
    lblError.CssClass = "alert alert-danger"
    lblSuccess.CssClass = "alert alert-success"
    
    ' ツールチップの設定
    txtPassword.ToolTip = "8文字以上で入力してください"
End Sub

重要な共通メソッド

  • FindControl(id)：ID指定でコントロールを検索
  • Focus()：コントロールにフォーカスを設定
  • DataBind()：データバインディングを実行
理解度確認クイズ
  1. HTMLコントロールとASP.NETサーバーコントロールの主な違いは何ですか？
  2. 以下のコントロールの用途を説明してください：
    • CheckBox
    • RadioButton
    • DropDownList
  3. VB.NETのWindows FormsコントロールとWeb Formsコントロールで、イベント処理の大きな違いは何ですか？
  4. コントロールのVisibleプロパティとEnabledプロパティの違いを説明してください。
  5. Page.IsPostBackプロパティを使う理由は何ですか？
解答例：
  1. HTMLコントロールは通常のHTMLタグベース、ASP.NETサーバーコントロールは高度な機能（ViewState管理、豊富なプロパティ、イベント統合等）を提供
  2. CheckBox:真偽値選択、RadioButton:択一選択（同グループ内）、DropDownList:複数選択肢から一つ選択
  3. Windows Formsは即座にイベント実行、Web FormsはサーバーへのPostBackが必要
  4. Visible:表示/非表示、Enabled:有効/無効（無効でも表示される）
  5. ページ初回読み込み時とPostBack時で処理を分けるため（例：初回のみドロップダウンリスト項目設定）
