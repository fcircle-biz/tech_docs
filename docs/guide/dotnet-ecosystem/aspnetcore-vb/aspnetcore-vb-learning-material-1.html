1.1 ASP.NET Coreとは何か

ASP.NET Coreは、Microsoftが開発したモダンでオープンソースのWebフレームワークです。 従来のASP.NET Frameworkの後継として設計されており、パフォーマンス、柔軟性、そしてクロスプラットフォーム対応を重視して開発されました。

sequenceDiagram participant U as ユーザー participant B as ブラウザ participant S as ASP.NET Coreサーバー participant D as データベース U->>B: Webページにアクセス B->>S: HTTPリクエスト送信 S->>D: データ取得 D->>S: データ返却 S->>B: HTMLレスポンス B->>U: Webページ表示

ASP.NET Coreの特徴

  • クロスプラットフォーム対応: Windows、macOS、Linuxで動作
  • 高性能: 従来のASP.NET Frameworkと比べて大幅な性能向上
  • モジュラー設計: 必要な機能のみを選択して軽量化可能
  • オープンソース: GitHubで開発が進められている
  • 依存性注入: DIコンテナが標準で組み込まれている

1.2 従来のWeb Formsとの違い

ASP.NET Web Formsに慣れ親しんだ開発者にとって、ASP.NET Coreへの移行は大きな変化となります。 主な違いを理解することで、新しいパラダイムに適応しやすくなります。

1.3 開発環境のセットアップ

ASP.NET Core開発を始めるために必要な環境を整備しましょう。 以下の手順で開発環境をセットアップします。

必要なソフトウェア

  1. Visual Studio 2022（Community版で十分）
  2. .NET 8 SDK（長期サポート版）
  3. SQL Server Express（データベース用）
  4. Git（ソースコード管理用）
注意事項

Visual Studio 2022をインストールする際は、「ASP.NET とWeb開発」ワークロードを必ず選択してください。 これにより、ASP.NET Core開発に必要なテンプレートとツールが自動的にインストールされます。

実習 1-1: 開発環境の確認

インストールした開発環境が正しく動作するかを確認します。

手順
  1. コマンドプロンプトまたはPowerShellを開く
  2. 以下のコマンドを実行して.NETのバージョンを確認
  3. Visual Studio 2022を起動して新しいプロジェクトダイアログを確認
確認コマンド
# .NETバージョンの確認
dotnet --version

# インストール済みSDKの一覧
dotnet --list-sdks

# 利用可能なプロジェクトテンプレートの確認
dotnet new --list
期待される結果

コマンド実行により、.NET 8.0以上のバージョンが表示されること。 プロジェクトテンプレート一覧に「ASP.NET Core Web App」が含まれていること。

1.4 初めてのASP.NET Coreプロジェクト

実際にASP.NET CoreプロジェクトをVB.NETで作成し、基本的な構造を理解しましょう。 プロジェクト作成から実行まで、段階的に進めていきます。

実習 1-2: 初めてのプロジェクト作成

Visual Studioを使用してASP.NET Core MVCプロジェクトをVB.NETで作成します。

手順
  1. Visual Studio 2022を起動
  2. 「新しいプロジェクトの作成」を選択
  3. 「ASP.NET Core Web アプリ (Model-View-Controller)」を検索
  4. 言語で「Visual Basic」を選択
  5. プロジェクト名を「MyFirstWebApp」に設定
  6. フレームワークは「.NET 8.0 (Long Term Support)」を選択
  7. 認証の種類は「なし」のまま
  8. 「HTTPS用に構成する」をチェック
  9. プロジェクトを作成
プロジェクト構造の確認

作成されたプロジェクトには以下のフォルダ構造が含まれています：

MyFirstWebApp/
├── Controllers/          # コントローラークラス
├── Models/              # データモデル
├── Views/               # Razorビューファイル
├── wwwroot/             # 静的ファイル（CSS、JS、画像）
├── Program.vb           # アプリケーションのエントリーポイント
└── MyFirstWebApp.vbproj # プロジェクトファイル
期待される結果

プロジェクトが正常に作成され、ソリューションエクスプローラーに上記のフォルダ構造が表示されること。

1.5 プロジェクトの基本構造理解

作成したプロジェクトの各ファイルの役割を理解することは、ASP.NET Core開発の基礎となります。 主要ファイルの内容を詳しく見ていきましょう。

Program.vb - アプリケーションのエントリーポイント

ASP.NET Coreアプリケーションの設定と起動を行うメインファイルです。 依存性注入の設定やミドルウェアパイプラインの構成を行います。 

' Program.vb の基本構造
Imports Microsoft.AspNetCore.Builder
Imports Microsoft.Extensions.DependencyInjection
Imports Microsoft.Extensions.Hosting

Module Program
    Sub Main(args As String())
        Dim builder = WebApplication.CreateBuilder(args)

        ' サービスの追加（DIコンテナへの登録）
        builder.Services.AddControllersWithViews()

        Dim app = builder.Build()

        ' HTTP リクエストパイプラインの設定
        If Not app.Environment.IsDevelopment() Then
            app.UseExceptionHandler("/Home/Error")
            app.UseHsts()
        End If

        app.UseHttpsRedirection()
        app.UseStaticFiles()
        app.UseRouting()
        app.UseAuthorization()

        ' ルーティングの設定
        app.MapControllerRoute(
            name:="default",
            pattern:="{controller=Home}/{action=Index}/{id?}")

        ' アプリケーション開始
        app.Run()
    End Sub
End Module

Controllers/HomeController.vb - コントローラーの例

ユーザーからのHTTPリクエストを受け取り、適切な処理を行ってビューに結果を渡すクラスです。 

' HomeController.vb の基本構造
Imports Microsoft.AspNetCore.Mvc

Public Class HomeController
    Inherits Controller

    ' インデックスページの処理
    Public Function Index() As IActionResult
        Return View()
    End Function

    ' Privacyページの処理
    Public Function Privacy() As IActionResult
        Return View()
    End Function

    ' エラーページの処理
    Public Function ErrorAction() As IActionResult
        Return View()
    End Function
End Class
実習 1-3: アプリケーションの実行

作成したプロジェクトを実際に実行して、ブラウザで動作を確認します。

手順
  1. Visual StudioでF5キーを押してデバッグ実行
  2. 自動的に開かれるブラウザでページを確認
  3. ナビゲーションリンク（Home、Privacy）をクリックして動作を確認
  4. 開発者ツール（F12）を開いてネットワークタブでHTTPリクエストを観察
確認ポイント
  • HTTPSでアクセスされていること（URLがhttps://で始まる）
  • ページが正しく表示されること
  • ナビゲーションが機能すること
  • Visual Studioの出力ウィンドウにログが表示されること
期待される結果

ブラウザでASP.NET Coreの初期ページが表示され、「Welcome」メッセージと Bootstrap を使用したレスポンシブなデザインが確認できること。

1.6 VB.NETでのASP.NET Core開発の特徴

VB.NETを使用したASP.NET Core開発には、C#とは異なるいくつかの特徴があります。 言語固有の注意点とベストプラクティスを理解しておきましょう。

VB.NET特有の構文

  • 大文字小文字を区別しない: 変数名やメソッド名で大文字小文字を混在可能
  • End文の使用: ブロックの終了にEnd If、End Functionなどを使用
  • Imports文: 名前空間のインポートにImports文を使用
  • 配列の宣言: Dim arr(10) As Integerの形式
