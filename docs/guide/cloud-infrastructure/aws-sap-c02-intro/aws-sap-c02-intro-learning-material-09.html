ハイブリッドクラウド接続 - オンプレミス統合

オンプレミス環境とAWSを安全に接続するハイブリッドアーキテクチャを設計します。

9.1 ハイブリッドクラウドの重要性

多くの企業では、既存のオンプレミスインフラを維持しながらAWSを活用するハイブリッドクラウド環境を構築しています。 規制要件、レガシーシステムの継続利用、段階的なクラウド移行などの理由から、オンプレミスとAWSを安全かつ効率的に接続することが重要です。

ハイブリッドクラウドのユースケース

レガシーシステムの継続利用

メインフレームやレガシーデータベースをオンプレミスに残し、新しいワークロードをAWSで構築。 両環境間でデータ連携を実現します。

規制・コンプライアンス要件

個人情報や機密データをオンプレミスに保持しつつ、分析・処理基盤をAWSで構築。 データ所在地の要件を満たします。

段階的なクラウド移行

全システムを一度に移行するのではなく、段階的にワークロードをAWSへ移行。 リスクを最小化しながら移行を進めます。

ディザスタリカバリ

オンプレミスの本番環境のバックアップやDR環境としてAWSを活用。 コストを抑えながら高可用性を実現します。

9.2 Site-to-Site VPN - 暗号化接続の基本

冗長化のベストプラクティス

各VPN接続は自動的に2つのトンネルを作成しますが、さらに高可用性を実現するには、 複数のカスタマーゲートウェイを配置し、複数のVPN接続を確立します。 これにより、オンプレミス側のルーター障害にも対応できます。

Transit Gatewayとの統合

複数のVPCやオンプレミスネットワークを接続する場合、Transit Gatewayを使用することで、管理を簡素化できます。

Site-to-Site VPNの制限

  • • 帯域幅: 各トンネル最大1.25 Gbps
  • • レイテンシー: インターネット経由のため変動する
  • • 可用性: インターネット接続に依存
  • • BGPサポート: 動的ルーティング可能

9.3 AWS Direct Connect - 専用線接続

AWS Direct Connectは、オンプレミスからAWSへの専用ネットワーク接続を提供します。 インターネットを経由しないため、安定した帯域幅と低レイテンシーを実現できます。

Direct Connectの接続タイプ

接続タイプ 帯域幅 用途
Dedicated Connection 1 Gbps、10 Gbps、100 Gbps 大容量トラフィック、エンタープライズ
Hosted Connection 50 Mbps～10 Gbps 小～中規模、柔軟な帯域変更

仮想インターフェース（VIF）の種類

1. プライベートVIF (Private Virtual Interface)

VPC内のプライベートリソース（EC2、RDS等）に接続するためのVIFです。

  • • Virtual Private Gateway (VGW) またはDirect Connect Gatewayに接続
  • • プライベートIPアドレス空間でアクセス
  • • BGPによる動的ルーティング

2. パブリックVIF (Public Virtual Interface)

S3、DynamoDB等のAWSパブリックサービスに接続するためのVIFです。

  • • AWSのパブリックIPアドレス空間に接続
  • • インターネットを経由せずアクセス
  • • S3、DynamoDB、CloudFront等に対応

3. Transit VIF (Transit Virtual Interface)

Transit Gatewayに接続し、複数のVPCやVPN接続を統合するためのVIFです。

  • • Transit Gatewayのみに接続可能
  • • 複数VPCの集約接続
  • • Direct Connect Gatewayと組み合わせ可能

Direct Connect Gateway

Direct Connect Gatewayを使用すると、1つのDirect Connect接続から複数リージョンの複数VPCにアクセスできます。

Direct Connect Gatewayのメリット

従来は各リージョン・各VPCごとにDirect Connect接続が必要でしたが、 Direct Connect Gatewayにより1つの接続で複数リージョンのVPCにアクセスできます。 ただし、VPC間の通信はできないため、VPC間通信にはTransit Gatewayが必要です。

9.4 Direct Connect の冗長化とフェイルオーバー

高可用性を実現するため、Direct Connect接続を冗長化し、障害時のフェイルオーバーを設計します。

高可用性構成のパターン

複数DXロケーション

異なるDirect Connectロケーションから2つの接続を確立。 ロケーション障害に対応できます。

複数オンプレミスルーター

オンプレミス側も冗長化し、ルーター障害に対応。 BGPによる自動フェイルオーバーを実現します。

VPN over Direct Connect（暗号化）

Direct Connectは専用線ですが、暗号化されていません。 セキュリティ要件により暗号化が必要な場合、Direct Connect上でVPN接続を確立します。

VPN over Direct Connectの利点

  • • セキュリティ: IPsec暗号化によるデータ保護
  • • パフォーマンス: 専用線の安定した帯域とレイテンシー
  • • コンプライアンス: 暗号化要件への対応
  • • フェイルオーバー: インターネットVPNとの冗長構成

Route 53 Resolverの構成要素

Inbound Endpoint（インバウンドエンドポイント）

オンプレミスからVPCのプライベートホストゾーンを解決するためのエンドポイント。

  • • オンプレミスDNSからの問い合わせを受け付け
  • • VPC内のプライベートDNS名を解決
  • • 各AZに1つのIPアドレスを割り当て

Outbound Endpoint（アウトバウンドエンドポイント）

VPCからオンプレミスDNSへ問い合わせを転送するためのエンドポイント。

  • • 特定ドメインの問い合わせをオンプレミスDNSへ転送
  • • Resolver Ruleで転送ルールを定義
  • • 社内ドメイン名の解決に使用

ハイブリッドDNSのベストプラクティス

オンプレミスとAWSで異なるドメイン名を使用することで、名前解決を明確に分離できます。 例: オンプレミスは「internal.example.com」、AWSは「aws.example.com」。 これにより、転送ルールの設定が簡素化されます。

9.6 帯域幅とレイテンシーの考慮事項

ハイブリッドクラウド接続では、帯域幅とレイテンシーがアプリケーションパフォーマンスに大きく影響します。

接続方式の比較

項目 Site-to-Site VPN Direct Connect
帯域幅 最大1.25 Gbps/トンネル 50 Mbps～100 Gbps
レイテンシー 変動あり（インターネット経由） 安定（専用線）
セットアップ時間 数分～数時間 数週間～数ヶ月
コスト 低（時間課金） 高（月額固定費+転送料）
暗号化 標準（IPsec） オプション（VPN over DX）
推奨用途 初期構築、バックアップ回線 本番環境、大容量データ転送

ハイブリッド接続のベストプラクティス

1 データ転送の最小化

オンプレミス-AWS間のデータ転送を最小限に抑えるため、AWS側でデータ処理を完結させる設計を検討します。 必要なデータのみを転送し、キャッシュを活用します。

2 非同期処理の活用

リアルタイム性が不要な処理は、SQS、SNS、EventBridgeなどを使用して非同期化し、 ネットワーク遅延の影響を軽減します。

3 段階的な移行

最初はVPNで接続を開始し、トラフィック量が増加したらDirect Connectに移行する段階的アプローチが推奨されます。 VPNをバックアップ回線として維持します。

4 監視と最適化

CloudWatch Metricsで帯域使用率、レイテンシー、パケットロスを監視し、 閾値を超えた場合はアラートを発報します。定期的にトラフィックパターンを分析し、最適化します。

実習 9-1: Site-to-Site VPN接続の構築

仮想環境を使用して、オンプレミス環境をシミュレートし、Site-to-Site VPN接続を構築します。

手順

  1. 1 VPCを作成し、Virtual Private Gatewayをアタッチ
  2. 2 別のVPCでオンプレミス環境をシミュレート（EC2でVPNルーター構築）
  3. 3 Customer Gatewayを作成し、VPN Connectionを確立
  4. 4 ルートテーブルを設定し、プライベートサブネット間の通信を確認
  5. 5 CloudWatch Metricsで接続状態とトンネルステータスを監視

ヒント: VPN接続の設定ファイルをダウンロードし、 strongSwanやOpenSwanなどのOSSソフトウェアで設定します。

理解度確認クイズ

  1. Q1. Site-to-Site VPN接続において、1つのVPN Connectionに含まれるIPsecトンネルの数は？

    回答を表示

    2つです。各VPN Connectionは自動的に2つのIPsecトンネルを作成し、冗長性を確保します。 両方のトンネルをアクティブに保つことが推奨されます。

  2. Q2. Direct Connect Gatewayの主な利点は何ですか？

    回答を表示

    1つのDirect Connect接続から複数リージョンの複数VPCにアクセスできることです。 これにより、各リージョン・各VPCごとに個別のDirect Connect接続を確立する必要がなくなり、コストと管理の複雑さを削減できます。

  3. Q3. Route 53 Resolverのアウトバウンドエンドポイントの用途は？

    回答を表示

    VPCからオンプレミスDNSサーバーへDNS問い合わせを転送するために使用されます。 Resolver Ruleで特定のドメイン名（例: internal.example.com）の問い合わせをオンプレミスDNSに転送するよう設定できます。

  4. Q4. Direct Connect接続で暗号化通信を実現する方法は？

    回答を表示

    VPN over Direct Connect（Direct Connect上でIPsec VPNを確立）を使用します。 これにより、専用線の安定性と低レイテンシーを維持しながら、IPsecによる暗号化を実現できます。 Transit Gateway VPN AttachmentまたはVirtual Private GatewayのVPN接続を使用します。

第9章のまとめ

  • ハイブリッドクラウドは、レガシーシステム継続利用、規制要件、段階的移行に有効
  • Site-to-Site VPNは低コストで迅速に構築でき、インターネット経由のIPsec暗号化通信を提供
  • Direct Connectは専用線接続で、安定した帯域幅と低レイテンシーを実現
  • Direct Connect Gatewayにより、1つの接続から複数リージョンのVPCにアクセス可能
  • Route 53 Resolverでハイブリッド環境のDNS名前解決を統合
  • 高可用性のため、複数DXロケーション、複数ルーター、VPNバックアップ回線を構成
