Global Accelerator - グローバルアプリケーションの高速化

Global Acceleratorを使用したグローバルアプリケーションのパフォーマンス最適化を学びます。

7.1 Global Acceleratorとは

AWS Global Acceleratorは、AWSのグローバルネットワークを活用して、アプリケーションの可用性とパフォーマンスを向上させるネットワークサービスです。 グローバルに分散したユーザーに対して、最適なルートでトラフィックを転送します。

Global Acceleratorの主な特徴

  • 静的Anycast IPアドレス: グローバルで固定された2つのIPアドレス
  • AWSグローバルネットワーク: インターネット経由よりも高速・安定
  • 自動フェイルオーバー: リージョン障害時の即座のトラフィック切替
  • クライアントIP保持: エンドポイントで元のクライアントIPを確認可能
  • レイヤ4最適化: TCP/UDPトラフィックの最適化

Global Acceleratorの仕組み

Global Acceleratorは、AWSエッジロケーションでクライアントのトラフィックを受け取り、 AWSのプライベートネットワークを経由して最適なエンドポイントにルーティングします。

Anycast IPアドレスの利点

Anycast IPを使用することで、世界中のどこからでも同じIPアドレスにアクセスでき、最も近いエッジロケーションにトラフィックが自動的にルーティングされます。 DNSキャッシュやIPホワイトリストの問題を回避できます。

7.2 CloudFrontとの違い

Global AcceleratorとCloudFrontはどちらもAWSのエッジロケーションを活用しますが、用途と動作が異なります。

観点 CloudFront Global Accelerator
主な用途 コンテンツキャッシュ・配信（CDN） 動的トラフィックの高速化・ルーティング
対象レイヤ レイヤ7（HTTP/HTTPS） レイヤ4（TCP/UDP）
IPアドレス 変動するエッジロケーションのIP 固定の2つのAnycast IP
キャッシュ コンテンツをエッジでキャッシュ キャッシュなし（プロキシのみ）
適したコンテンツ 静的コンテンツ（画像、動画、JS/CSS） 動的コンテンツ（API、ゲーム、IoT）
フェイルオーバー オリジンフェイルオーバーグループ 即座の自動フェイルオーバー（DNS不要）
クライアントIP X-Forwarded-Forヘッダー クライアントIPアフィニティ機能

CloudFrontが適している場合

  • • 静的コンテンツの配信（画像、動画）
  • • Webサイトの高速化
  • • エッジでのキャッシュが有効
  • • Lambda@Edgeでのカスタム処理
  • • コスト重視のコンテンツ配信

Global Acceleratorが適している場合

  • • 動的コンテンツ（API、リアルタイムアプリ）
  • • 非HTTP/HTTPSプロトコル（TCP/UDP）
  • • 固定IPが必要（ホワイトリスト）
  • • 即座のリージョンフェイルオーバー
  • • オンラインゲーム、VoIP、IoT

併用も可能

CloudFrontとGlobal Acceleratorは併用できます。例えば、CloudFrontの前段にGlobal Acceleratorを配置することで、 静的コンテンツのキャッシュと動的トラフィックの高速化を両立できます。

7.3 エンドポイントグループとトラフィックダイヤル

Global Acceleratorでは、エンドポイントグループを使用してリージョンごとにトラフィックを制御します。

エンドポイントグループの構成

エンドポイントグループは、1つのAWSリージョンに関連付けられたエンドポイント（ALB、NLB、EC2、Elastic IP）の集合です。

トラフィックダイヤルの仕組み

トラフィックダイヤルは、エンドポイントグループに送信されるトラフィックの割合を0〜100%で制御します。

トラフィックダイヤルの使用例

1

ブルーグリーンデプロイ

新リージョンを0%で追加 → 段階的に増加 → 旧リージョンを0%に

2

メンテナンス

メンテナンス対象のリージョンを0%に設定してトラフィックを遮断

3

キャパシティ調整

リソースに応じてトラフィック比率を調整

エンドポイントの重み付け

各エンドポイントには0〜255の重みを設定でき、同一エンドポイントグループ内でトラフィックを分散できます。

エンドポイント重み付けの例 
エンドポイントグループ: us-east-1
├─ ALB-1: 重み 128 → 50%のトラフィック
├─ ALB-2: 重み 128 → 50%のトラフィック
└─ 合計: 256

エンドポイントグループ: eu-west-1
├─ NLB-1: 重み 200 → 78%のトラフィック
├─ NLB-2: 重み 56  → 22%のトラフィック
└─ 合計: 256

7.4 ヘルスチェックと自動フェイルオーバー

Global Acceleratorは、エンドポイントのヘルスチェックを実行し、障害を検出すると自動的にトラフィックを正常なエンドポイントに切り替えます。

ヘルスチェックの仕組み

正常 participant EP2 as エンドポイント2

トラフィック分散再開

ヘルスチェックの設定項目

  • プロトコル: TCP、HTTP、HTTPS
  • ポート: エンドポイントのポート（デフォルトはリスナーのポート）
  • パス: HTTP/HTTPSの場合のヘルスチェックパス（例: /health）
  • 間隔: 30秒または10秒
  • しきい値: 異常判定までの連続失敗回数（デフォルト: 3回）

フェイルオーバーのメリット

高速なフェイルオーバー

DNSベースのフェイルオーバーと異なり、DNSキャッシュの影響を受けずに即座にトラフィックを切り替え。 障害検出から数秒以内にフェイルオーバー完了。

リージョン間フェイルオーバー

リージョン全体の障害に対しても自動的に他のリージョンにフェイルオーバー。 マルチリージョン構成で高可用性を実現。

DRTOとDRPOの向上

Global Acceleratorの自動フェイルオーバーにより、災害復旧目標時間（RTO）を数秒に短縮できます。 データ損失目標（RPO）は、データレプリケーションの設定に依存します。

7.5 クライアントIPアドレスの保持

Global Acceleratorは、クライアントのIPアドレスをエンドポイントに伝える機能を提供しています。

クライアントIPアフィニティ

Global Acceleratorでは、以下の方法でクライアントIPアドレスを保持できます。

1. クライアントIPアドレスの保持（ALB/NLB）

ALBまたはNLBをエンドポイントとする場合、クライアントIPアドレスをヘッダーに追加します。

  • • ALB: X-Forwarded-ForヘッダーにクライアントIPが含まれる
  • • NLB: Proxy Protocolを有効化してクライアントIPを取得

2. クライアントIPアフィニティ

EC2やElastic IPをエンドポイントとする場合、クライアントIPアドレスの保持設定を有効化できます。

  • • 有効時: クライアントの実際のIPアドレスが保持される
  • • 無効時: Global AcceleratorのIPアドレスが送信元として表示される
クライアントIPアフィニティの設定例（AWS CLI） 
# クライアントIPアフィニティを有効化
aws globalaccelerator update-endpoint-group \
  --endpoint-group-arn arn:aws:globalaccelerator::123456789012:... \
  --endpoint-configurations \
    EndpointId=arn:aws:ec2:us-east-1:123456789012:eip-allocation/...,\
    ClientIPPreservationEnabled=true

注意事項

クライアントIPアフィニティを有効にすると、Global Acceleratorのフェイルオーバー機能が一部制限されます。 エンドポイントのセキュリティグループで、クライアントIPアドレスからのトラフィックを許可する必要があります。

7.6 ユースケースと選択基準

Global Acceleratorが最適なユースケースと、他のサービスとの選択基準を理解しましょう。

Global Acceleratorが適したユースケース

オンラインゲーム

低レイテンシーが求められるリアルタイムゲームで、グローバルプレイヤーに最適なパフォーマンスを提供。

  • • UDP/TCPトラフィックの最適化
  • • 固定IPでファイアウォール設定が容易
  • • リージョン障害時の即座のフェイルオーバー

IoT・ストリーミング

IoTデバイスやライブストリーミングアプリケーションで安定した接続を維持。

  • • 非HTTPプロトコル（MQTT、RTMP等）対応
  • • デバイスのホワイトリスト管理が容易
  • • パケットロスの削減

エンタープライズアプリケーション

VoIP、VDI（仮想デスクトップ）など、レイテンシーに敏感なアプリケーション。

  • • グローバルユーザーへの一貫したパフォーマンス
  • • 予測可能な固定IP
  • • DDoS保護（AWS Shield Standard統合）

API・マイクロサービス

動的コンテンツを返すRESTful APIやgRPCサービス。

  • • キャッシュできない動的コンテンツの高速化
  • • マルチリージョンでの高可用性
  • • 地理的に分散したマイクロサービス間の通信最適化

ハイブリッド構成の例

CloudFront（静的コンテンツ）+ Global Accelerator（動的API）の組み合わせで、 Webアプリケーション全体を最適化できます。CloudFrontはS3から画像・JS・CSSを配信し、 Global AcceleratorはバックエンドAPIへのリクエストを高速化します。

実習 7-1: Global Acceleratorの構築

2つのリージョンにALBを配置し、Global Acceleratorでグローバルトラフィックを最適化します。

手順

  1. 1 2つのリージョン（us-east-1、ap-northeast-1）にVPC、サブネット、EC2インスタンス、ALBを作成
  2. 2 Global Acceleratorを作成し、2つの静的Anycast IPを取得
  3. 3 リスナー（ポート80）を作成し、エンドポイントグループを各リージョンに設定
  4. 4 各リージョンのALBをエンドポイントとして追加
  5. 5 トラフィックダイヤルを調整して、トラフィック分散を確認
  6. 6 一方のリージョンのEC2を停止し、自動フェイルオーバーを確認

理解度確認クイズ

  1. Q1. Global AcceleratorとCloudFrontの最も大きな違いは何ですか？

    回答を表示

    CloudFrontはレイヤ7（HTTP/HTTPS）のCDNサービスでコンテンツをキャッシュします。 一方、Global Acceleratorはレイヤ4（TCP/UDP）のネットワークサービスで、キャッシュせずにトラフィックを最適化します。 また、Global Acceleratorは固定の2つのAnycast IPアドレスを提供します。

  2. Q2. トラフィックダイヤルを50%に設定すると、どのような動作になりますか？

    回答を表示

    そのエンドポイントグループに送信されるトラフィックが50%に制限されます。 残りのトラフィックは、他のエンドポイントグループに分散されます。 メンテナンスやブルーグリーンデプロイに有用です。

  3. Q3. Global Acceleratorが適したユースケースをすべて選んでください。

    回答を表示

    • オンラインゲーム（低レイテンシー、UDP/TCP）
    • IoTアプリケーション（非HTTPプロトコル、固定IP）
    • VoIP・VDI（レイテンシーに敏感）
    • 動的API（キャッシュできないコンテンツ）
    静的コンテンツの配信にはCloudFrontの方が適しています。

  4. Q4. Anycast IPアドレスの利点は何ですか？

    回答を表示

    世界中のどこからでも同じIPアドレスでアクセスでき、最も近いエッジロケーションに自動的にルーティングされます。 また、固定IPのためファイアウォールのホワイトリスト設定が容易で、DNSキャッシュの問題を回避できます。

  • Global Acceleratorは、AWSグローバルネットワークを活用してトラフィックを最適化
  • CloudFrontはレイヤ7のCDN、Global Acceleratorはレイヤ4のネットワーク最適化
  • トラフィックダイヤルでリージョンごとのトラフィック比率を制御
  • ヘルスチェックにより自動フェイルオーバーを実現、RTOを数秒に短縮
  • オンラインゲーム、IoT、VoIP、動的APIなどに最適
