Solutions Architect Professionalへの道 - 試験概要と学習戦略

SAP-C02試験の全体像を把握し、効果的な学習計画を立てます。試験範囲、出題傾向、合格に必要なスキルセットを理解します。

1.1 SAP-C02試験の全体像

AWS Certified Solutions Architect - Professional（SAP-C02）は、AWSアーキテクトとして最高レベルの設計スキルを証明する資格です。 複雑なビジネス要件に基づき、スケーラブル、高可用性、耐障害性、セキュアなアプリケーションを設計・実装できる能力が求められます。

試験の基本情報

  • 試験コード: SAP-C02
  • 試験時間: 180分（3時間）
  • 問題数: 75問（多肢選択式、複数選択式）
  • 合格点: 750点/1000点（スコアベース）
  • 受験料: 300 USD（税別）
  • 前提資格: なし（Associate資格推奨）

試験の出題分野

SAP-C02試験は、以下の4つのドメインから構成されています。

ドメイン1: 複雑な組織のソリューション設計 (26%)

  • • マルチアカウント戦略
  • • ハイブリッドアーキテクチャ
  • • ネットワーク接続設計
  • • セキュリティとコンプライアンス

ドメイン2: 新しいソリューションの設計 (29%)

  • • ビジネス要件のアーキテクチャへの変換
  • • 高可用性・耐障害性の設計
  • • セキュリティ制御の設計
  • • コスト最適化の設計

ドメイン3: 既存ソリューションの継続的な改善 (25%)

  • • パフォーマンス改善
  • • 信頼性の向上
  • • セキュリティ強化
  • • デプロイプロセスの改善

ドメイン4: ワークロードの移行と最適化 (20%)

  • • 移行戦略の選択
  • • データ転送ソリューション
  • • アプリケーション移行戦略
  • • コスト最適化の実装

1.2 Associate資格とProfessional資格の違い

Solutions Architect - Associate (SAA-C03) を取得された方は、Professional資格との違いを理解することが重要です。

観点 Associate (SAA-C03) Professional (SAP-C02)
設計の複雑さ 単一アカウント、シンプルなワークロード マルチアカウント、複雑なエンタープライズワークロード
要件の抽象度 技術的要件が明確 ビジネス要件から設計を導出
トレードオフの評価 ベストプラクティスの適用 複数の選択肢を比較し、最適解を選択
対象範囲 AWSコアサービス（EC2, S3, RDS等） 広範なAWSサービス、統合パターン
実務経験 1年以上推奨 2年以上推奨

Professional資格の特徴

Professional試験では「正解がひとつ」ではなく、「状況に応じた最適解」を選ぶ能力が問われます。 コスト、パフォーマンス、セキュリティ、運用性などのトレードオフを理解し、ビジネス要件に最も適した設計を選択できることが重要です。

Well-Architected Framework の5つの柱

1. 運用上の優秀性 (Operational Excellence)

システムを運用・監視し、ビジネス価値を継続的に提供し、プロセスと手順を改善する能力。

  • • Infrastructure as Code (IaC) による自動化
  • • 頻繁で小規模な変更の実施
  • • 運用手順の文書化とテスト

2. セキュリティ (Security)

データ、システム、資産を保護し、リスク評価とリスク軽減戦略を活用してビジネス価値を提供する能力。

  • • 強力なアイデンティティ基盤の実装
  • • トレーサビリティの有効化
  • • 多層防御の適用

3. 信頼性 (Reliability)

ワークロードが意図した機能を正しく一貫して実行し、障害から迅速に復旧する能力。

  • • 障害から自動的に復旧
  • • 水平スケーリングによる可用性の向上
  • • キャパシティの推測を排除

4. パフォーマンス効率 (Performance Efficiency)

コンピューティングリソースを効率的に使用し、要件の変化とテクノロジーの進化に応じて効率性を維持する能力。

  • • 高度なテクノロジーの民主化
  • • グローバル展開を数分で実現
  • • サーバーレスアーキテクチャの活用

5. コスト最適化 (Cost Optimization)

最も低い価格でシステムを実行してビジネス価値を提供する能力。

  • • 消費モデルの実装
  • • 全体的な効率性の測定
  • • マネージドサービスの活用

試験での注意点

試験問題では、これら5つの柱のトレードオフを考慮する必要があります。 例えば、「最もコスト効率が良い」と「最も高可用性」は異なる設計になる可能性があります。 問題文をよく読み、何が優先されているかを見極めましょう。

学習の4ステップ

1 第1段階: 基礎固め（3-4週間）

高可用性設計の原則とコアサービスの深い理解

  • • 第1章-第5章: 高可用性、ELB、Auto Scaling、Route 53
  • • AWS Well-Architected Frameworkの理解
  • • SLA、RTO、RPOなどの指標の理解

2 第2段階: ネットワーク・接続（3-4週間）

グローバル配信とエンタープライズネットワーク設計

  • • 第6章-第9章: CloudFront、Global Accelerator、VPC高度設計、ハイブリッド接続
  • • Transit Gateway、Direct Connectの理解
  • • VPCピアリングとPrivateLinkの使い分け

3 第3段階: データ・DR（2-3週間）

データベース・ストレージの高可用性と災害復旧戦略

  • • 第10章-第12章: データベースHA、ストレージHA、DR戦略
  • • Aurora Global Database、DynamoDB Global Table
  • • バックアップ戦略とリージョン間レプリケーション

4 第4段階: 運用・セキュリティ・総合演習（2-3週間）

エンタープライズセキュリティと監視設計、総合演習

  • • 第13章-第15章: セキュリティ、監視、総合演習
  • • AWS Organizations、Control Tower、Security Hub
  • • CloudWatch、X-Rayによる監視設計
  • • 総合演習でアーキテクチャ設計スキルを統合

学習のポイント

効果的な学習方法

  • • 実践重視: 読むだけでなく、AWSマネジメントコンソールで実際に設定を試す
  • • 図を描く: アーキテクチャ図を自分で描いて理解を深める
  • • 公式ドキュメント参照: 本ガイドで基礎を学んだ後、公式ドキュメントで詳細を確認
  • • 模擬問題: 各章の学習後、関連する試験問題を解いて知識を定着
  • • コスト管理: 実習後は必ずリソースを削除し、予期しない課金を防ぐ

理解度確認クイズ

  1. Q1. SAP-C02試験の合格点は何点ですか？

    回答を表示

    750点/1000点です。スコアベースの評価で、絶対評価ではありません。

  2. Q2. Well-Architected Frameworkの5つの柱をすべて挙げてください。

    回答を表示

    1. 運用上の優秀性 (Operational Excellence)
    2. セキュリティ (Security)
    3. 信頼性 (Reliability)
    4. パフォーマンス効率 (Performance Efficiency)
    5. コスト最適化 (Cost Optimization)

  3. Q3. AssociateとProfessional資格の最も大きな違いは何ですか？

    回答を表示

    Professionalでは、ビジネス要件から設計を導出し、複数の選択肢のトレードオフを評価して最適解を選択する能力が求められます。 また、マルチアカウント、エンタープライズワークロードなど、より複雑な環境への対応が必要です。

  4. Q4. SAP試験の出題分野で最も配点が高いドメインはどれですか？

    回答を表示

    ドメイン2: 新しいソリューションの設計 (29%) が最も配点が高いです。 ビジネス要件をアーキテクチャに変換する能力が重視されています。

第1章のまとめ

  • SAP-C02試験は180分、75問、合格点750点/1000点のProfessional資格
  • 試験はWell-Architected Frameworkの5つの柱に基づいた設計思考が求められる
  • Associateとの違いは、複雑な要件とトレードオフの評価能力
  • 10-14週間の体系的な学習計画で、4つの段階に分けて学習を進める
  • AWS Skill Builder、公式ドキュメント、Whitepapers、Well-Architected Labsを活用
