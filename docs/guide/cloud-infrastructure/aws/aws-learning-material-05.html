ファイルの保管場所を作ろう - S3入門

写真や文書ファイルを保存できる「S3」というサービスを使ってみます。Dropboxのようなものですが、もっと大きなファイルもたくさん保存できます。

この章で学ぶこと

  • S3は「インターネット上のファイル置き場」
  • 「バケット」は「フォルダ」のようなもの
  • バケットの作り方
  • ファイルのアップロード・ダウンロード
  • ファイルを誰でも見られるようにする方法（Webサイト公開）
  • ファイルを見られないように守る方法

5.1 S3とは何か？

S3（Simple Storage Service）は、AWSが提供する「ファイル保存サービス」です。 インターネット上に写真、動画、文書など、どんなファイルでも保存できます。

身近な例え

S3 = トランクルーム（いくらでも荷物を入れられる貸し倉庫）
バケット = 倉庫の中の棚（整理用）
オブジェクト = 棚に入れる荷物（ファイル）

DropboxやGoogleドライブのようなサービスと似ていますが、S3の方が大容量で、企業システムに適した設計になっています。 また、S3はファイルを保存するだけでなく、Webサイトのホスティング（公開）にも使えます。

重要ポイント

S3の保存容量は無制限です。何テラバイト（TB）でも保存できます。料金は保存したデータ量とダウンロード量に応じて発生します。

5.2 バケットとオブジェクト

S3では、ファイルを「バケット」という入れ物に保存します。

バケット（Bucket）

バケットは、ファイルを整理するための「フォルダのようなもの」です。 ただし、通常のフォルダと違い、バケット名は世界中で唯一の名前である必要があります。

注意

バケット名は世界中のAWSユーザーと共有されるため、「test」や「my-bucket」のような一般的な名前はすでに使われています。 「会社名-用途-日付」のような形式にすると重複しにくくなります。
例: mycompany-photos-2025

オブジェクト（Object）

バケットに保存された個々のファイルを「オブジェクト」と呼びます。 画像、動画、PDF、HTMLファイルなど、どんな種類のファイルでも保存できます。

バケットとオブジェクトの関係

バケット

  • • ファイルを整理する入れ物
  • • 世界で唯一の名前が必要
  • • 1つのAWSアカウントで最大100個まで
  • • リージョンを指定して作成

オブジェクト

  • • 実際に保存されるファイル
  • • 1つのオブジェクトは最大5TB
  • • 各オブジェクトにURLが付与される
  • • メタデータ（付加情報）も保存可能

5.3 バケットを作ってみよう

実際にS3バケットを作成してみましょう。手順は簡単です。

ハンズオン: S3バケットの作成

ステップ1: S3ダッシュボードを開く

  1. AWSマネジメントコンソールにログイン
  2. 画面上部の検索バーに「S3」と入力
  3. 「S3」をクリックしてダッシュボードを開く

ステップ2: バケットを作成

  1. 「バケットを作成」ボタンをクリック
  2. バケット名を入力（例: yourname-test-bucket-2025）
  3. AWSリージョンで「アジアパシフィック（東京）ap-northeast-1」を選択
  4. 「このバケットのパブリックアクセスをすべてブロック」にチェックを入れたまま（セキュリティのため）
  5. その他はデフォルト設定のまま、画面下部の「バケットを作成」をクリック

ステップ3: 作成完了を確認

バケット一覧に、今作成したバケット名が表示されれば成功です！

ヒント

バケット名に使える文字は、小文字のアルファベット、数字、ハイフン（-）のみです。 大文字やアンダースコア（_）は使えないので注意しましょう。

5.4 ファイルをアップロードしよう

作成したバケットにファイルをアップロード（保存）してみましょう。

ハンズオン: ファイルのアップロード

ステップ1: バケットを開く

S3ダッシュボードで、先ほど作成したバケット名をクリック

ステップ2: ファイルをアップロード

  1. 「アップロード」ボタンをクリック
  2. 「ファイルを追加」をクリックして、適当な画像や文書ファイルを選択
  3. （または、ファイルをブラウザにドラッグ&ドロップ）
  4. 画面下部の「アップロード」ボタンをクリック

ステップ3: 完了を確認

「アップロード成功」と表示され、バケット内にファイル名が表示されれば完了です！

5.5 ファイルのアクセス権限

S3に保存したファイルは、デフォルトで「非公開」です。 つまり、あなただけがアクセスできる状態になっています。

パブリックアクセス vs プライベートアクセス

パブリックアクセス

誰でもインターネット経由でファイルにアクセスできる状態。

用途: Webサイトの公開、誰でもダウンロードできるファイル

プライベートアクセス

あなた（AWSアカウント所有者）だけがアクセスできる状態。

用途: 個人情報、社内文書、バックアップデータ

重要な注意事項

パブリックアクセスを有効にすると、ファイルのURLを知っている人なら誰でもダウンロードできるようになります。 個人情報や機密データを間違って公開しないよう、十分注意してください。

静的Webサイトホスティング

S3の便利な機能の1つが「静的Webサイトホスティング」です。 HTMLファイルをアップロードして公開設定をすれば、誰でもWebサイトとして閲覧できるようになります。

ヒント

「静的」とは、ログイン機能やデータベース連携のない、シンプルなWebサイトのことです。 ポートフォリオサイトやランディングページに最適です。

5.6 S3の料金体系

S3の料金は主に以下の3つから構成されます。

S3の料金体系

1. ストレージ料金

保存したデータ量に応じて課金。最初の50TBまでは約0.025ドル/GB/月（東京リージョン）

2. リクエスト料金

アップロード、ダウンロード、削除などの操作回数に応じて課金。1,000リクエストあたり数セント

3. データ転送料金

インターネットへのダウンロード量に応じて課金。月1GBまでは無料、それ以降は約0.114ドル/GB

無料利用枠

新規ユーザーは12ヶ月間、以下の無料枠があります：
• ストレージ: 5GB
• GETリクエスト: 20,000回/月
• PUTリクエスト: 2,000回/月
• データ転送: 15GB/月（アウト）

5.7 お片付け - バケットとファイルの削除

練習が終わったら、料金が発生しないよう、バケットとファイルを削除しましょう。

バケットとファイルの削除手順

ステップ1: ファイルを削除

  1. バケットを開く
  2. 削除したいファイルにチェックを入れる
  3. 「削除」ボタンをクリック
  4. 確認画面で「完全に削除」と入力して削除を確定

ステップ2: バケットを削除

  1. S3ダッシュボードに戻る
  2. 削除したいバケットにチェックを入れる
  3. 「削除」ボタンをクリック
  4. 確認画面でバケット名を入力して削除を確定

注意

バケット内にファイルが残っていると、バケットは削除できません。 必ず先にファイルを全て削除してから、バケットを削除してください。

理解度確認クイズ

  1. Q1. S3のバケット名はどのように決めるべきですか？

    回答を表示

    世界中で唯一の名前である必要があります。会社名や用途、日付などを組み合わせた名前にすると重複しにくくなります。 また、小文字のアルファベット、数字、ハイフンのみが使用可能です。

  2. Q2. S3に保存したファイルは、デフォルトでどうなっていますか？

    回答を表示

    デフォルトでは「非公開（プライベート）」です。あなた（AWSアカウント所有者）だけがアクセスできます。 パブリックアクセスを有効にすると、誰でもインターネット経由でアクセスできるようになります。

  3. Q3. S3バケットを削除する前に何をする必要がありますか？

    回答を表示

    バケット内のすべてのファイル（オブジェクト）を削除する必要があります。 ファイルが残っていると、バケットは削除できません。

この章のまとめ

  • S3はインターネット上のファイル保存サービス。容量は無制限で、どんなファイルでも保存できる。
  • バケットはファイルを整理する入れ物。世界で唯一の名前が必要。
  • オブジェクトはバケットに保存された個々のファイル。各オブジェクトにURLが付与される。
  • デフォルトは非公開。パブリックアクセスを有効にすると誰でも閲覧可能になる（個人情報は注意）。
  • S3は静的Webサイトのホスティングにも使える。HTMLファイルをアップロードして公開可能。
  • 使い終わったらファイルとバケットを削除して、料金発生を防ごう。
