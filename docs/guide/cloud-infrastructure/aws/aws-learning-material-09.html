第9章

サーバーの状態を見守ろう - CloudWatch入門

サーバーが正常に動いているか監視する方法を学びます。車のダッシュボードのように、いろいろな情報を見ることができます。

この章で学ぶこと

  • CloudWatchは「AWSの健康診断ツール」
  • メトリクスは「健康診断の数値」
  • ダッシュボードで複数の情報を一覧表示
  • アラーム（警告）を設定して異常を知らせる
  • ログを保存して後から確認する

9.1 CloudWatchとは何か？

CloudWatchは、AWSのリソース（サーバー、データベースなど）を監視するサービスです。 車のダッシュボードのように、さまざまな「健康状態」を数値やグラフで表示してくれます。

身近な例え

CloudWatch = 車のダッシュボード
メトリクス = スピードメーター、燃料計、水温計
アラーム = 警告ランプ（異常があると光る）
ダッシュボード = 計器類をまとめた表示板

CloudWatchを使うと、以下のようなことが分かります：

  • CPUの使用率 - サーバーがどれくらい働いているか
  • メモリの使用量 - 作業スペースがどれくらい使われているか
  • ネットワークトラフィック - どれくらい通信しているか
  • ディスクの読み書き - データの保存・読み込み頻度

9.2 メトリクスとは？

メトリクスとは、監視対象の「測定値」のことです。 健康診断で測る「体温」「血圧」「体重」のようなものと考えてください。

重要ポイント

初心者はまず「CPUの使用率」だけ見ておけばOK。サーバーが忙しすぎないか確認するための最も基本的な指標です。

主要なメトリクス

CPUUtilization

サーバーのCPU使用率。70%以上が続くと負荷が高い状態。

NetworkIn/NetworkOut

ネットワーク通信量。急増すると攻撃を受けている可能性も。

DiskReadBytes/DiskWriteBytes

ディスクの読み書き量。データベース操作が多いと増える。

StatusCheckFailed

サーバーの正常性チェック。失敗すると問題あり。

9.3 ダッシュボードを見てみよう

ダッシュボードは、複数のメトリクスをまとめて表示する画面です。 車の計器盤のように、一目で全体の状態を把握できます。

実習: EC2のメトリクスを見てみよう

  1. 1 AWSマネジメントコンソールにログイン
  2. 2 検索バーで「CloudWatch」と入力し、サービスを開く
  3. 3 左メニューから「メトリクス」→「すべてのメトリクス」を選択
  4. 4 「EC2」→「インスタンス別メトリクス」を選択
  5. 5 CPUUtilizationにチェックを入れてグラフを表示

ヒント

グラフの期間は右上のドロップダウンで変更できます。「1時間」「3時間」「1日」などから選べます。

9.4 アラームを設定しよう

アラームは、設定した条件を超えたときに通知してくれる機能です。 車の警告ランプのように、問題が起きたら自動で知らせてくれます。

重要ポイント

アラームを設定しておけば、サーバーに問題が起きてもすぐに気づけます。例: CPU使用率が80%を超えたらメール通知。

実習: CPU使用率のアラームを作成

  1. 1 CloudWatchの左メニューから「アラーム」→「すべてのアラーム」を選択
  2. 2 「アラームの作成」ボタンをクリック
  3. 3 「メトリクスの選択」をクリック → EC2 → インスタンス別メトリクス
  4. 4 CPUUtilizationを選択し、「メトリクスの選択」をクリック
  5. 5 しきい値の設定: 「80」と入力（80%を超えたら警告）
  6. 6 通知設定: メールアドレスを登録（任意）
  7. 7 アラーム名を入力（例: EC2-CPU-High）
  8. 8 「アラームの作成」をクリック

9.5 ログを確認しよう

CloudWatch Logsは、アプリケーションやサーバーの「記録（ログ）」を保存・確認できるサービスです。 日記のように、いつ何が起きたかを後から確認できます。

ログの例

2025-12-24 10:30:15 INFO サーバー起動完了
2025-12-24 10:35:22 WARN データベース接続遅延
2025-12-24 10:40:03 ERROR ファイル読み込み失敗

ログの活用例

  • エラー調査 - エラーが発生した原因を特定
  • セキュリティ監査 - 不正アクセスの痕跡を確認
  • アクセス分析 - どのページが人気か分析
  • 過去の記録 - 問題が起きた時刻を特定

注意

ログは長期間保存すると料金がかかります。不要なログは削除するか、保存期間を設定しましょう。

9.6 初心者がつまずきやすいポイント

よくある問題と解決方法

問題1: グラフにデータが表示されない

原因: 時間範囲の設定ミス、またはリソースが停止中

解決: グラフ右上の期間を「1時間」に変更。EC2が起動しているか確認。

問題2: アラームが発動しない

原因: しきい値の設定が高すぎる

解決: テストとして低めの値（例: CPU 10%）で試してみる。

問題3: ログがどこにあるか分からない

原因: ログ収集の設定が必要

解決: EC2にCloudWatch Agentをインストールする必要があります（中級編で解説）。

動作確認チェックリスト

  • CloudWatchでEC2のCPU使用率グラフが表示できた
  • CPU使用率のアラームを作成できた
  • グラフの表示期間を変更できた
  • アラームの状態（OK/アラーム）が確認できた

理解度確認クイズ

  1. Q1. CloudWatchを車に例えると何ですか？

    回答を表示

    車のダッシュボード。スピードメーター、燃料計、水温計などの計器類をまとめた表示板。

  2. Q2. 初心者がまず見るべき基本的なメトリクスは何ですか？

    回答を表示

    CPUUtilization（CPU使用率）。サーバーがどれくらい働いているかを示す最も基本的な指標。

  3. Q3. アラームとは何をするための機能ですか？

    回答を表示

    設定した条件（例: CPU使用率80%以上）を超えたときに、管理者に通知する機能。車の警告ランプのようなもの。

この章のまとめ

  • CloudWatchは車のダッシュボードのように、AWSリソースの状態を監視するサービス。
  • メトリクスは「測定値」。初心者はまずCPU使用率を見ればOK。
  • アラームを設定すると、問題が起きたときに自動で通知してくれる。
  • ログを保存すると、後から「いつ何が起きたか」を確認できる。
  • グラフの期間設定を間違えるとデータが見えないことがあるので注意。
