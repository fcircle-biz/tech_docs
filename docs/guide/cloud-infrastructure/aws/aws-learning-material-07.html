データベースを使ってみよう - RDS入門

たくさんのデータを整理して保存する「データベース」を作ります。Excelの表をもっと便利にしたものと考えてください。

この章で学ぶこと

  • データベースは「超高機能な表計算」
  • RDSは「管理おまかせのデータベース」
  • よく使うデータベースの種類（MySQL、PostgreSQL）
  • データベースの作り方
  • データベースに接続する方法
  • バックアップ（自動保存）の仕組み

7.1 データベースとは何か？

データベースとは、たくさんの情報を整理して保存し、簡単に検索・更新できるようにした「超高機能な表計算」のようなものです。

身近な例え

データベース = 図書館の蔵書目録
Excelで本のリストを管理するより、図書館の検索システムの方が速く・正確に探せますよね。

Excelとデータベースの違い

項目 Excel データベース
データ量 数万行まで 数億行でもOK
同時利用 1人 数千人でも同時アクセス可能
検索速度 遅い（全行スキャン） 高速（インデックス利用）
データの整合性 手動管理 自動チェック機能あり

データベースの基本構造

flowchart TB DB[(データベース)] DB --> T1[テーブル1: 顧客情報] DB --> T2[テーブル2: 注文履歴] DB --> T3[テーブル3: 商品マスタ] T1 --> R1["レコード1: 山田太郎"] T1 --> R2["レコード2: 佐藤花子"] T2 --> R3["レコード1: 注文#001"] T2 --> R4["レコード2: 注文#002"]

重要ポイント

データベース = 複数のテーブル（表）の集まり
テーブル = Excelシートに似た表形式
レコード = 表の1行（1件のデータ）

7.2 RDSとは何か？

RDS（Relational Database Service）は、AWSが提供するデータベースサービスです。「面倒な管理作業はAWSにおまかせ」できます。

身近な例え

RDS = 司書さんが管理してくれる図書館
自分でデータベースを管理する = 自分で本棚を整理する
RDSを使う = 司書さんにおまかせして、本を読むことに集中

RDSが自動でやってくれること

自動バックアップ

毎日自動でデータのコピーを保存。うっかり削除しても復旧可能。

自動更新

セキュリティパッチやバージョンアップを自動適用。

監視機能

データベースの状態を常に監視。異常があればアラート。

スケール変更

容量や性能を後から変更できる。最初は小さく始めてOK。

RDSで使えるデータベースの種類

MySQL

世界で最も使われているデータベース。無料で利用可能。

PostgreSQL

高機能で信頼性が高い。企業システムでよく使われる。

MariaDB

MySQLから派生。互換性が高い。

Aurora

AWS独自の高性能データベース。MySQL/PostgreSQL互換。

ヒント

初めての方はMySQLがおすすめ。情報が多く、わからないことがあっても調べやすいです。

7.3 RDSデータベースを作成しよう

実際にRDSでデータベースを作成してみましょう。手順通りに進めれば、15分程度で完成します。

重要な注意事項

  • RDSは起動しているだけで課金されます
  • 無料枠は月750時間まで（1台なら1ヶ月フル稼働でOK）
  • 練習が終わったら必ず削除してください
  • db.t2.micro または db.t3.micro を選べば無料枠内で利用可能

実習 7-1: RDSデータベースの作成

MySQLデータベースを作成し、無料枠内で動作させます。

手順

  1. 1 AWSマネジメントコンソールにログインし、検索バーに「RDS」と入力して選択します。
  2. 2 画面右上のリージョンが「東京（ap-northeast-1）」になっていることを確認します。
  3. 3 左サイドバーから「データベース」を選択し、「データベースの作成」ボタンをクリックします。
  4. 4 エンジンのオプション: 「MySQL」を選択します。
  5. 5 テンプレート: 「無料利用枠」を選択します（重要！）。
  6. 6 設定: DB インスタンス識別子に「my-first-db」と入力します。
  7. 7 認証情報: マスターユーザー名は「admin」のまま。マスターパスワードは複雑なものを入力し、必ずメモします。
  8. 8 インスタンスの設定: 「db.t3.micro」が選択されていることを確認します（無料枠対象）。
  9. 9 ストレージ: 汎用SSD（gp2）、20GBのまま変更不要です。
  10. 10 接続: 「EC2コンピューティングリソースに接続しない」を選択します（今回は単独で作成）。
  11. 11 パブリックアクセス: 「いいえ」を選択（セキュリティのため）。
  12. 12 その他の設定はデフォルトのままで、画面下部の「データベースの作成」ボタンをクリックします。
  13. 13 作成には5〜10分かかります。ステータスが「作成中」→「利用可能」になるまで待ちます。

ヒント

パスワードは大文字・小文字・数字・記号を含む8文字以上にしましょう。例: MyDB2025!Pass

RDSの重要な概念

DBインスタンス

データベースが動作している「仮想マシン」のこと。EC2と同じように、スペック（メモリ、CPU）を選べます。

エンドポイント

データベースに接続するための「住所」。例: my-first-db.abc123.ap-northeast-1.rds.amazonaws.com

セキュリティグループ

「誰がデータベースに接続できるか」を制御するファイアウォール。VPCの章で学んだ仕組みです。

自動バックアップ

毎日決まった時間にデータの「スナップショット（写真）」を自動作成。過去7日分まで保存されます。

重要ポイント

RDSはデータベース本体だけでなく、バックアップ、監視、セキュリティも含めて管理してくれます。

7.5 データベースの接続確認

作成したデータベースに接続できるか確認しましょう。通常はEC2インスタンスから接続しますが、ここでは概念を理解することが目的です。

接続に必要な情報

データベース接続情報の確認方法

  1. 1. RDSダッシュボードで「my-first-db」をクリックします。
  2. 2. 「接続とセキュリティ」タブを開きます。
  3. 3. 以下の情報をメモします：
  • エンドポイント: my-first-db.xxxxx.ap-northeast-1.rds.amazonaws.com
  • ポート: 3306（MySQLのデフォルトポート）
  • ユーザー名: admin
  • パスワード: （作成時に設定したもの）

注意

「パブリックアクセス:いいえ」で作成したため、インターネットから直接接続できません。これはセキュリティのための正しい設定です。 実際の接続は、同じVPC内のEC2インスタンスから行います。

データベース接続の流れ

7.6 バックアップと復元

RDSの強力な機能の一つが「自動バックアップ」です。データを間違えて削除しても、過去に戻すことができます。

バックアップの種類

自動バックアップ

毎日決まった時間に自動実行。

  • 保持期間: 7日間（設定で最大35日まで延長可能）
  • ポイントインタイムリカバリ対応（任意の時点に復元）
  • 追加料金なし（無料枠内）

手動スナップショット

自分で好きなタイミングで作成。

  • 保持期間: 無制限（削除するまで残る）
  • 大きな変更前に作成しておくと安心
  • 他リージョンへのコピーも可能

ヒント

「バックアップウィンドウ」は深夜の時間帯を選ぶのがおすすめ。バックアップ中は少し性能が落ちるためです。

7.7 お片付け: データベースの削除

重要: RDSは起動しているだけで課金されます。練習が終わったら必ず削除しましょう。

削除前の確認事項

  • 必要なデータはすべてバックアップ済みですか？
  • アプリケーションから接続していませんか？
  • 最終スナップショットを作成しますか？（残したい場合のみ）

実習 7-2: RDSデータベースの削除

手順

  1. 1 RDSダッシュボードで「my-first-db」を選択します。
  2. 2 画面右上の「アクション」→「削除」をクリックします。
  3. 3 「最終スナップショットを作成しますか？」→ 練習用なので「いいえ」を選択します。
  4. 4 「自動バックアップを保持しますか？」→ 「いいえ」を選択します。
  5. 5 確認のため「delete me」と入力します（大文字小文字も正確に）。
  6. 6 「削除」ボタンをクリックします。
  7. 7 削除には数分かかります。ステータスが「削除中」→消えるまで待ちます。

削除完了の確認

データベース一覧から「my-first-db」が完全に消えたことを確認してください。これで課金が止まります。

理解度確認クイズ

  1. Q1. RDSとは何ですか？

    回答を表示

    AWSが提供するデータベースサービスで、バックアップ、更新、監視などの管理作業をAWSがおまかせしてくれるサービスです。

  2. Q2. なぜRDSは「起動しているだけで課金」されるのですか？

    回答を表示

    データベースは24時間稼働する「仮想マシン」上で動いているため、使っていなくてもCPUやメモリのリソースを消費しているからです。EC2と同じ仕組みです。

  3. Q3. 自動バックアップと手動スナップショットの違いは何ですか？

    回答を表示

    自動バックアップは毎日決まった時間に自動実行され7日間保持されます。手動スナップショットは自分で作成し、削除するまで無期限に保持されます。

この章のまとめ

  • データベースは「超高機能な表計算」で、大量データの保存・検索に優れています。
  • RDSは管理作業をAWSがおまかせしてくれる便利なサービス。バックアップ、更新、監視が自動化されます。
  • 初心者にはMySQLがおすすめ。db.t3.microを選べば無料枠内で利用可能。
  • 自動バックアップは毎日実行され、過去7日分まで保存。データを間違えて削除しても復元できます。
  • RDSは起動中は課金されるので、練習後は必ず削除すること。
