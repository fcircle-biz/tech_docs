総合演習 - アーキテクチャ設計の実践

これまで学んだ知識を統合し、実際のシナリオに基づいたアーキテクチャ設計を実践します。要件分析からWell-Architectedレビュー、コスト最適化まで、SAA-C03試験に向けた総仕上げを行います。

12.1 要件分析と設計プロセス

アーキテクチャ設計の第一歩は、要件を正しく理解し整理することです。SAA-C03試験でも、シナリオ問題で「何を重視すべきか」を見極める力が問われます。

設計プロセスの5ステップ

flowchart LR A([1. 要件収集]) --> B([2. 優先順位決定]) B --> C([3. サービス選択]) C --> D([4. 構成設計]) D --> E([5. レビュー]) E --> F{改善点
あり?} F -->|はい| C F -->|いいえ| G([設計完成])
1

要件収集

可用性、パフォーマンス、セキュリティ、コスト、コンプライアンスなど、全ての要件を網羅的に収集します。

2

優先順位決定

相反する要件がある場合、ビジネス影響度を基準に優先順位をつけます。

3

サービス選択

要件を満たすAWSサービスを選択し、代替案も検討します。

4

構成設計

選択したサービスを組み合わせ、全体アーキテクチャを設計します。

5

レビュー

Well-Architectedの6つの柱で設計を評価し、改善点を洗い出します。

12.2 Well-Architectedに基づく設計レビュー

設計したアーキテクチャをWell-Architectedフレームワークの6つの柱で評価し、改善点を特定します。

6つの柱でのチェックリスト

運用上の優秀性

  • CloudWatchでメトリクス監視
  • SNSでアラート通知設定
  • インフラのコード化（IaC）

セキュリティ

  • IAMロールによる最小権限設定
  • データの暗号化（保管時・転送時）
  • WAFでアプリケーション保護

信頼性

  • マルチAZ構成で単一障害点排除
  • Auto Scalingで負荷対応
  • RDSの自動バックアップ設定

パフォーマンス効率

  • CloudFrontでコンテンツ配信
  • ElastiCacheでDB負荷軽減
  • 適切なインスタンスタイプ選択

コスト最適化

  • Savings Plans活用
  • S3ライフサイクルポリシー設定
  • Cost Explorerで定期レビュー

サステナビリティ

  • リソース使用率の最適化
  • 未使用リソースの削除
  • Gravitonプロセッサ検討

12.3 総合演習: 3層Webアプリケーションの設計

実際のビジネスシナリオに基づいて、高可用性・高セキュリティ・コスト最適化されたアーキテクチャを設計します。

シナリオ: ECサイトのリニューアル

要件

  • 可用性: 99.9%以上のSLA、RTO 1時間、RPO 15分
  • パフォーマンス: ページ読み込み3秒以内、月間100万PV対応
  • セキュリティ: PCI DSS準拠、個人情報暗号化
  • コスト: 月額20万円以内（データ転送費含む）
  • 拡張性: 将来的にグローバル展開を想定

各レイヤーの設計判断

プレゼンテーション層

  • CloudFront: グローバル配信、HTTPS対応、静的コンテンツキャッシュ
  • WAF: SQLインジェクション、XSS対策
  • Route 53: DNSフェイルオーバー、ヘルスチェック

アプリケーション層

  • ALB: パスベースルーティング、スティッキーセッション
  • Auto Scaling: ターゲット追跡スケーリング（CPU 70%）
  • ElastiCache: セッション管理、DBクエリキャッシュ

データ層

  • RDS Multi-AZ: 自動フェイルオーバー、暗号化ストレージ
  • S3: 静的コンテンツ保管、バージョニング有効化
  • バックアップ: 自動バックアップ保持期間7日、スナップショット毎日取得

12.4 コスト見積もりと最適化

設計したアーキテクチャの月額コストを見積もり、最適化の余地を検討します。

ALB 1台、処理時間100万LCU/月 ¥30,000
RDS db.m5.large Multi-AZ（RI 1年） ¥50,000
ElastiCache cache.t3.small × 2ノード ¥15,000
CloudFront 100GB転送/月 ¥10,000
S3 Standard 500GB、リクエスト10万回/月 ¥5,000

WAF ルール5個、リクエスト100万件/月 ¥8,000

コスト最適化の施策

Savings Plans活用

EC2とRDSで1年または3年のコミットメントで最大72%削減

S3ライフサイクル

30日経過後はStandard-IA、90日後はGlacierに自動移行

Auto Scalingの調整

夜間・休日は最小台数を1台に縮小、トラフィック増加時のみスケールアウト

Cost Explorerで定期レビュー

月次でコスト推移を確認、予想外の増加を早期発見

12.5 試験対策のポイント

SAA-C03試験で合格点を取るための実践的なアドバイスをまとめます。

試験の攻略法

頻出パターンと対策

パターン1: 「最も」「最適な」選択を問う問題

4つの選択肢がすべて技術的に可能だが、要件に最も合致するものを選ぶ。消去法と優先順位付けが重要。

パターン2: トラブルシューティング問題

「接続できない」「パフォーマンスが悪い」などの状況で、原因と解決策を特定する。セキュリティグループ、NACLが頻出。

パターン3: 複数選択問題（2つまたは3つ選択）

部分点なし。すべて正解しないと得点にならないため、慎重に。

よくある誤答パターン

注意: これらの選択肢は一見正しそうだが誤り

  • • 「CloudWatch Logsでネットワークトラフィック分析」→ VPC Flow Logsが正解
  • • 「パブリックサブネットにNATゲートウェイ配置」→ プライベートサブネットから使うためパブリックに配置するのが正解
  • • 「RDSリードレプリカで自動フェイルオーバー」→ Multi-AZが正解（リードレプリカは手動昇格）
  • • 「S3でブロックストレージ」→ EBSが正解（S3はオブジェクトストレージ）

試験当日のTips

時間配分

130分÷65問=2分/問。わからない問題はフラグ立てて後回し。

問題文は丁寧に

「最小のコスト」「最小の運用負荷」など、問われているポイントを見落とさない。

消去法活用

明らかに間違っている選択肢を2つ消せば、正解率50%→50%に改善。

見直し時間確保

最後の15分はフラグ付き問題の見直しと、ケアレスミスチェック。

理解度確認クイズ

  1. Q1. 3層Webアプリケーションで、99.9%のSLAを実現するために必須の構成はどれですか？

    回答を表示

    マルチAZ構成です。単一のAZに障害が発生した場合でも、別のAZで継続稼働することで高可用性を実現します。ELB、EC2 Auto Scaling、RDS Multi-AZの組み合わせが基本パターンです。

  2. Q2. コスト最適化のために、アクセス頻度が低くなった古いログデータに対して実施すべきS3の機能はどれですか？

    回答を表示

    S3ライフサイクルポリシーです。作成から30日経過後はStandard-IAへ、90日経過後はGlacierへ自動移行することで、ストレージコストを大幅に削減できます。

  3. Q3. RDSでデータベースのフェイルオーバーを自動化するために必要な設定はどれですか？

    回答を表示

    Multi-AZデプロイです。プライマリDBに障害が発生すると、スタンバイDBに自動的にフェイルオーバーされます。リードレプリカは手動昇格が必要なため、自動フェイルオーバーには使用できません。
