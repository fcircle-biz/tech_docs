サーバーレスとアプリケーション統合

サーバーレスアーキテクチャとアプリケーション統合サービスを活用し、疎結合で拡張性の高いシステムを設計する方法を学びます。

この章で学ぶこと

  • Lambda の設計パターンとユースケース
  • API Gateway によるAPI設計
  • SQSとSNSによる疎結合アーキテクチャ
  • Step Functions によるワークフロー設計
  • EventBridge によるイベント駆動設計
  • CloudFront と Global Accelerator の使い分け

11.1 AWS Lambda - サーバーレスコンピューティング

AWS Lambdaは、サーバーの管理なしにコードを実行できるサーバーレスコンピューティングサービスです。インフラ管理が不要で、実行時間に応じた課金のため、コスト効率が高いのが特徴です。

Lambdaの特徴とメリット

サーバー管理不要

プロビジョニング、スケーリング、パッチ適用が自動化

従量課金

リクエスト数と実行時間のみ課金

自動スケーリング

リクエストに応じて自動的にスケール

統合性

多数のAWSサービスと連携可能

重要なLambdaの制約

  • • 最大実行時間: 15分
  • • メモリ: 128MB～10,240MB
  • • デプロイパッケージサイズ: 50MB（圧縮時）、250MB（展開時）
  • • 同時実行数: デフォルト1,000（引き上げ可）

11.2 API Gateway - RESTful API設計

Amazon API Gatewayは、RESTful APIやWebSocket APIを作成、公開、管理するフルマネージドサービスです。Lambdaと組み合わせることで、完全なサーバーレスAPIを構築できます。

API Gatewayの3つのタイプ

REST API

RESTful API向け。キャッシング、認証、アクセス制御など豊富な機能

ユースケース: 高度な機能が必要なAPI

HTTP API

シンプルで低コスト。REST APIより機能は少ないがパフォーマンス重視

ユースケース: コスト重視の単純なAPI

WebSocket API

双方向通信対応。チャットやリアルタイム通知に最適

ユースケース: リアルタイム通信

11.3 SQS と SNS - メッセージングサービス

Amazon SQS（Simple Queue Service）とAmazon SNS（Simple Notification Service）は、疎結合なアーキテクチャを実現するマネージドメッセージングサービスです。

SQS と SNS の違い

項目 SQS（キュー） SNS（Pub/Sub）
通信モデル ポイント to ポイント Publish/Subscribe
配信先 1つのコンシューマー 複数のサブスクライバー
メッセージ保持 最大14日間 保持しない（即配信）
ユースケース ジョブキュー、負荷平準化 イベント通知、ファンアウト

11.4 Step Functions と EventBridge

AWS Step Functionsはワークフローオーケストレーション、Amazon EventBridgeはイベント駆動アーキテクチャを実現するサービスです。

Step Functions - ワークフロー設計

Step Functionsの特徴

  • 複数のLambda関数を順序立てて実行
  • 条件分岐、並列実行、エラーハンドリングをビジュアルに定義
  • 長時間実行ワークフロー（最大1年）に対応

EventBridge - イベント駆動設計

EventBridgeの特徴

  • AWSサービスやカスタムアプリケーションからのイベントをルーティング
  • イベントパターンによるフィルタリング
  • スケジュール実行（cron、rate式）

11.5 CloudFront と Global Accelerator の使い分け

Amazon CloudFrontとAWS Global Acceleratorは、どちらもグローバルなコンテンツ配信を高速化しますが、用途が異なります。

項目 CloudFront Global Accelerator
主な用途 コンテンツ配信（CDN） アプリケーション高速化
対象トラフィック HTTP/HTTPS TCP/UDP
キャッシング あり（重要） なし
典型例 静的サイト、動画配信 ゲーム、VoIP、IoT

理解度確認クイズ

  1. Q1. Lambdaの最大実行時間は何分ですか？

    回答を表示

    15分です。長時間実行が必要な処理は、Step FunctionsやECS/Fargateの使用を検討します。

  2. Q2. 1つのメッセージを複数のサービスに配信したい場合、SQSとSNSのどちらを使いますか？

    回答を表示

    SNS（Simple Notification Service）を使います。SNSはPub/Subモデルで、複数のサブスクライバーに同じメッセージを配信できます。

  3. Q3. CloudFrontとGlobal Acceleratorの最大の違いは何ですか？

    回答を表示

    CloudFrontはキャッシング機能を持つCDNで、主にHTTP/HTTPS向け。Global Acceleratorはキャッシングなしでネットワークレベルで高速化し、TCP/UDPトラフィックに対応します。

この章のまとめ

  • Lambdaはサーバーレスコンピューティングの中核で、15分以内の処理に最適
  • API Gatewayには REST API、HTTP API、WebSocket API の3タイプがあり、用途に応じて選択
  • SQSはキュー、SNSはPub/Subモデルで、疎結合アーキテクチャを実現
  • Step Functionsはワークフローオーケストレーション、EventBridgeはイベント駆動設計に使用
  • CloudFrontはCDN、Global Acceleratorはアプリケーション高速化と用途が異なる
