Solutions Architect Associateへの道 - 試験概要と設計思考

AWS Certified Solutions Architect - Associate（SAA-C03）試験の全体像を理解し、ソリューションアーキテクトとしての設計思考の基礎を学びます。

この章で学ぶこと

  • SAA-C03試験の構成と4つの出題ドメイン
  • 試験で問われる設計判断のポイント
  • AWS Well-Architectedフレームワークの6つの柱
  • Cloud Practitionerとの違い
  • 効果的な学習ロードマップの作成

1.1 SAA-C03試験の概要

AWS Certified Solutions Architect - Associate（SAA-C03）は、AWSクラウド上でセキュアでスケーラブルなアプリケーションを設計する能力を証明する資格試験です。この資格は、AWSの幅広いサービスに対する知識と、それらを組み合わせて最適なソリューションを設計できる力を求められます。

試験の基本情報

試験時間

130分

問題数

65問（選択問題）

合格ライン

720点/1000点満点

受験料

15,000円（税別）

4つの出題ドメイン

SAA-C03試験は、以下の4つのドメインで構成されています。各ドメインの出題比率を理解し、重点的に学習する領域を計画しましょう。

1

セキュアなアーキテクチャの設計

出題比率: 30%

  • セキュアなアクセス制御
  • ワークロードのセキュリティ
2

弾力性に優れたアーキテクチャの設計

出題比率: 26%

  • スケーラブルなアーキテクチャ
  • 高可用性設計
3

高性能アーキテクチャの設計

出題比率: 24%

  • ストレージソリューションの選択
  • データベース設計
4

コスト最適化アーキテクチャの設計

出題比率: 20%

  • コスト効率の良いストレージ
  • コンピューティングリソースの最適化

1.2 ソリューションアーキテクトに求められる思考法

SAA-C03試験では、単にAWSサービスの機能を暗記するだけでは合格できません。重要なのは「なぜこの設計が最適なのか」を判断できる思考力です。

設計判断の3つの軸

要件の理解

セキュリティ要件、可用性要件、パフォーマンス要件、コスト制約など、複数の要件を正しく読み取り、優先順位をつける能力が求められます。

トレードオフの判断

コストと性能、セキュリティと利便性など、相反する要素のバランスを取りながら、最適解を導き出す思考力が必要です。

サービスの組み合わせ

複数のAWSサービスを適切に組み合わせ、全体として最適なアーキテクチャを構成する設計力が評価されます。

1.3 AWS Well-Architectedフレームワーク

AWS Well-Architectedフレームワークは、クラウドアーキテクチャを評価・改善するためのベストプラクティス集です。SAA-C03試験では、この6つの柱に基づいた設計判断が頻繁に問われます。

運用上の優秀性

システムの運用と監視、継続的な改善

セキュリティ

データ保護、権限管理、セキュリティイベント検知

信頼性

障害からの回復、需要変化への対応

パフォーマンス効率

効率的なリソース使用、需要に応じた最適化

コスト最適化

不要なコストの排除、費用対効果の最大化

サステナビリティ

環境への影響最小化、エネルギー効率

重要ポイント

試験では、これら6つの柱のうち複数を同時に考慮する必要がある問題が多く出題されます。単一の視点だけでなく、総合的な判断力が求められます。

1.4 Cloud Practitionerとの違い

AWS Certified Cloud Practitioner（CLF）との違いを理解することで、SAA-C03で求められるレベル感を把握しましょう。

項目 Cloud Practitioner Solutions Architect
対象者 AWS初学者 実務経験者
知識レベル 基本的な用語・サービス理解 サービスの組み合わせ設計
問題の特徴 「このサービスは何か」 「どう組み合わせるか」
合格に必要な経験 6ヶ月のAWS使用経験推奨 1年以上の設計経験推奨

理解度確認クイズ

  1. Q1. SAA-C03試験で最も出題比率が高いドメインはどれですか？

    回答を表示

    「セキュアなアーキテクチャの設計」（30%）です。セキュリティは全てのアーキテクチャ設計の基礎となるため、最も重要視されています。

  2. Q2. AWS Well-Architectedフレームワークの6つの柱をすべて挙げてください。

    回答を表示

    1. 運用上の優秀性、2. セキュリティ、3. 信頼性、4. パフォーマンス効率、5. コスト最適化、6. サステナビリティの6つです。

  3. Q3. Cloud Practitioner試験とSolutions Architect試験の最大の違いは何ですか？

    回答を表示

    Cloud Practitionerは「AWSサービスの基本知識」を問うのに対し、Solutions Architectは「複数のサービスを組み合わせて最適なアーキテクチャを設計する能力」を問います。

この章のまとめ

  • SAA-C03試験は4つのドメインで構成され、セキュリティが最重要（30%）
  • ソリューションアーキテクトには、要件理解・トレードオフ判断・サービス組み合わせの3つの思考力が必要
  • Well-Architectedの6つの柱（運用・セキュリティ・信頼性・パフォーマンス・コスト・サステナビリティ）が設計の基礎
  • Cloud Practitionerは「知識」、Solutions Architectは「設計力」を問う
