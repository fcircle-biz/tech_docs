9.1 S3ストレージクラスとライフサイクル管理

Amazon S3は複数のストレージクラスを提供し、アクセス頻度とコストのバランスに応じて最適な保存方法を選択できます。ライフサイクルポリシーを活用することで、データの経年変化に応じた自動的なコスト最適化が可能です。

S3ストレージクラスの種類

ストレージクラス 用途 取り出し時間 コスト
S3 Standard 頻繁にアクセスするデータ ミリ秒 高
S3 Intelligent-Tiering アクセスパターンが変化するデータ ミリ秒 中（自動最適化）
S3 Standard-IA アクセス頻度低いデータ ミリ秒 中
S3 One Zone-IA 再作成可能な低頻度データ ミリ秒 低
S3 Glacier Instant Retrieval 四半期に1回程度のアクセス ミリ秒 低
S3 Glacier Flexible Retrieval 長期アーカイブ（即時不要） 分〜時間 非常に低
S3 Glacier Deep Archive 年単位の長期保存 12時間 最低

ライフサイクルポリシーの活用

ライフサイクルポリシーを設定することで、オブジェクトの経過日数に応じて自動的にストレージクラスを変更したり、削除したりできます。

Flexible Retrieval] D -->|365日後| E[S3 Glacier
Deep Archive] E -->|7年後| F[自動削除] style B fill:#fff7ed style C fill:#fed7aa style D fill:#fdba74 style E fill:#fb923c style F fill:#c2410c,color:#fff

ヒント: ストレージクラス選択の基準

アクセス頻度が予測できない場合はS3 Intelligent-Tieringを選択することで、自動的にコスト最適化できます。ただし、小さいオブジェクト（128KB未満）には不向きです。

JSON 
{
  "Rules": [
    {
      "Id": "ログファイルのアーカイブ",
      "Status": "Enabled",
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER"
        }
      ],
      "Expiration": {
        "Days": 2555
      }
    }
  ]

9.2 S3の暗号化とセキュリティ

S3では複数の暗号化オプションを提供しており、データの機密性要件に応じて適切な方法を選択できます。

暗号化オプション

SSE-S3

AWS管理キー

  • 最も簡単な設定
  • 追加コストなし
  • AES-256暗号化

SSE-KMS

KMS管理キー

  • キーの監査ログ
  • キーローテーション
  • 細かいアクセス制御

SSE-C

顧客管理キー

  • 完全な鍵管理
  • HTTPS必須
  • 鍵の自己管理

バージョニングとレプリケーション

S3バージョニングを有効にすることで、オブジェクトの過去のバージョンを保持し、誤削除やデータ破損から保護できます。クロスリージョンレプリケーション（CRR）を使用すると、別のリージョンにデータを自動複製できます。

重要ポイント

クロスリージョンレプリケーションを使用する場合、ソースバケットとデスティネーションバケットの両方でバージョニングを有効にする必要があります。

9.3 EBSボリュームタイプと選択基準

Amazon EBS（Elastic Block Store）は、EC2インスタンスにアタッチする永続的なブロックストレージです。ワークロードの特性に応じて最適なボリュームタイプを選択することが重要です。

EBSボリュームタイプの比較

タイプ 用途 性能指標 最大IOPS
gp3（汎用SSD） 一般的なワークロード IOPS独立設定可 16,000
gp2（汎用SSD） 一般的なワークロード 容量とIOPS連動 16,000
io2（プロビジョンドIOPS SSD） 高IOPS要求アプリ 高耐久性 64,000
io1（プロビジョンドIOPS SSD） 高IOPS要求アプリ プロビジョンドIOPS 64,000
st1（スループット最適化HDD） ビッグデータ、ログ処理 スループット重視 500
sc1（コールドHDD） アクセス頻度低いデータ 最低コスト 250

ヒント: gp3とgp2の選択

新規作成の場合はgp3を選択することで、IOPS・スループット・容量を独立して設定でき、コスト効率が良くなります。gp2からgp3への変更も可能です。

EBSスナップショットとバックアップ

EBSスナップショットは増分バックアップとしてS3に保存され、リージョン間でコピーすることもできます。

9.4 EFSとFSxの使い分け

共有ファイルストレージが必要な場合、Amazon EFS（Elastic File System）やAmazon FSxを使用します。それぞれの特性を理解し、要件に応じて選択しましょう。

EFSの特徴

NFSプロトコル

Linux系OSからマウント可能な共有ファイルシステム。複数のEC2インスタンスから同時アクセス可能。

自動スケーリング

ファイルの追加・削除に応じて自動的に容量が増減し、事前のプロビジョニングが不要。

ストレージクラス

Standard（頻繁アクセス）とInfrequent Access（低頻度アクセス）の2つのクラスを提供。

FSxの種類と用途

FSx for Windows File Server

  • Windows環境向け
  • SMBプロトコル
  • Active Directory統合

FSx for Lustre

  • 高性能コンピューティング
  • 機械学習、動画処理
  • S3と統合可能

9.5 Storage Gateway とハイブリッドストレージ

AWS Storage Gatewayは、オンプレミス環境とAWSクラウドストレージをシームレスに統合するハイブリッドクラウドストレージサービスです。

Storage Gatewayの種類

File Gateway

NFS/SMBプロトコルでS3バケットにアクセス。ファイルはオブジェクトとしてS3に保存され、頻繁にアクセスするファイルはローカルキャッシュに保持。

用途: ファイルバックアップ、アーカイブ、データ移行

Volume Gateway

iSCSIプロトコルでブロックストレージを提供。Cached Volume（S3が主、ローカルがキャッシュ）とStored Volume（ローカルが主、S3がバックアップ）の2モード。

用途: 仮想マシンのバックアップ、ディザスタリカバリ

Tape Gateway

仮想テープライブラリ（VTL）インターフェースを提供。既存のバックアップソフトウェアとの互換性を保ちながら、テープをS3/Glacierに置き換え。

用途: テープバックアップの代替、長期アーカイブ

9.6 AWS Backup による統合バックアップ

AWS Backupは、複数のAWSサービスのバックアップを一元管理できるフルマネージドサービスです。バックアップポリシーを一箇所で定義し、自動化できます。

サポートされるサービス

RDS

EBS

EFS

FSx

Storage
Gateway

DynamoDB

EC2

S3

バックアッププランの設定

1

バックアップルール

バックアップの頻度（毎日、毎週など）、開始時間、保持期間を定義します。

2

リソースの割り当て

タグまたはリソースIDでバックアップ対象を指定します。

3

ライフサイクル管理

一定期間後にコールドストレージ（低コスト）に移行するルールを設定できます。

重要ポイント

AWS Backupを使用することで、各サービスごとに個別に設定していたバックアップポリシーを統一的に管理でき、コンプライアンス要件への対応が容易になります。

実習: S3ライフサイクルポリシーの設定

  1. 1 S3コンソールで新しいバケットを作成します（例: my-lifecycle-demo-bucket-[yourname]）
  2. 2 「管理」タブからライフサイクルルールを作成します
  3. 3 30日後にStandard-IAに移行、90日後にGlacierに移行するルールを設定
  4. 4 テストファイルをアップロードし、オブジェクトのストレージクラスを確認
  5. 5 実習後は必ずバケットとオブジェクトを削除してください

理解度確認クイズ

  1. Q1. アクセス頻度が予測できないデータに最適なS3ストレージクラスはどれですか？

    回答を表示

    S3 Intelligent-Tieringです。アクセスパターンを自動的に監視し、30日間アクセスがないオブジェクトを低頻度アクセス層に移動することでコストを最適化します。

  2. Q2. データベースに高いIOPSが必要な場合、どのEBSボリュームタイプを選択すべきですか？

    回答を表示

    io2（またはio1）プロビジョンドIOPS SSDを選択します。最大64,000 IOPSまで対応でき、データベースなど高IOPS要求のワークロードに最適です。

  3. Q3. 複数のEC2インスタンスから同時にアクセス可能な共有ファイルシステムが必要な場合、どのサービスを使用しますか？

    回答を表示

    Linux環境の場合はAmazon EFS、Windows環境の場合はAmazon FSx for Windows File Serverを使用します。EFSはNFS、FSx for WindowsはSMBプロトコルをサポートします。
