3.1 ハンドラー関数の設計パターン

Lambda関数のエントリーポイントとなる**ハンドラー関数**は、イベントを受け取り、処理結果を返す役割を持ちます。ハンドラーの設計によって、Lambda関数の保守性と拡張性が大きく変わります。

Python Lambda ハンドラーの基本構造

Python 
# lambda_function.py
import json

def lambda_handler(event, context):
    """
    Lambda関数のメインハンドラー
    event: イベントデータ（dict型）
    context: Lambda実行環境情報
    """
    # イベントからデータを取得
    body = json.loads(event.get('body', '{}'))
    name = body.get('name', 'Guest')

    # レスポンスを返す
    return {
        'statusCode': 200,
        'body': json.dumps({
            'message': f'Hello, {name}!'
        })
    }

重要ポイント

ハンドラー関数は**event**と**context**の2つの引数を受け取ります。eventにはトリガー元から渡されるデータが格納され、contextにはリクエストID、実行環境のメタデータが含まれます。

Node.js Lambda ハンドラーの基本構造

JavaScript 
// index.js
exports.handler = async (event, context) => {
    // イベントからデータを取得
    const body = JSON.parse(event.body || '{}');
    const name = body.name || 'Guest';

    // レスポンスを返す
    return {
        statusCode: 200,
        body: JSON.stringify({
            message: `Hello, ${name}!`
        })
    };
};

3.2 環境変数と設定管理

環境変数を使用することで、**ハードコードを避け**、Lambda関数を環境ごとに柔軟に設定できます。

環境変数の設定例

Python 
import os

def lambda_handler(event, context):
    # 環境変数から設定を読み込む
    db_host = os.environ.get('DB_HOST')
    db_name = os.environ.get('DB_NAME')
    api_key = os.environ.get('API_KEY')

    # 環境変数が設定されていない場合のエラーハンドリング
    if not all([db_host, db_name, api_key]):
        return {
            'statusCode': 500,
            'body': '環境変数が設定されていません'
        }

    # ビジネスロジック...
    return {'statusCode': 200}

注意事項

機密情報（データベースパスワード、APIキー）は環境変数に直接保存せず、**AWS Secrets Manager**または**Parameter Store**を使用して暗号化して管理しましょう。

3.3 AWS SAM（Serverless Application Model）によるローカル開発

AWS SAMは、サーバーレスアプリケーションの定義、ローカルテスト、デプロイを簡素化するオープンソースフレームワークです。SAMを使用することで、**ローカル環境でLambda関数をテスト**できます。

SAMテンプレートの基本構造

YAML 
# template.yaml
AWSTemplateFormatVersion: '2010-09-09'
Transform: AWS::Serverless-2016-10-31

Resources:
  HelloWorldFunction:
    Type: AWS::Serverless::Function
    Properties:
      Handler: app.lambda_handler
      Runtime: python3.11
      MemorySize: 128
      Timeout: 3
      Environment:
        Variables:
          DB_HOST: localhost
      Events:
        ApiEvent:
          Type: Api
          Properties:
            Path: /hello
            Method: get

SAM CLIによるローカルテスト

Bash 
# Lambda関数をローカルで起動
sam local start-api

# または特定の関数を直接実行
sam local invoke HelloWorldFunction

3.4 Lambda関数のパッケージング

Lambda関数のデプロイには、**ZIPアーカイブ**と**コンテナイメージ**の2つの方法があります。それぞれの特徴を理解し、適切な方法を選択しましょう。

ZIPアーカイブによるデプロイ

Bash 
# 依存関係を含めてパッケージ化
pip install -r requirements.txt -t package/
cd package
zip -r ../lambda_function.zip .
cd ..
zip -g lambda_function.zip lambda_function.py

# Lambda関数を更新
aws lambda update-function-code \
    --function-name my-function \
    --zip-file fileb://lambda_function.zip

コンテナイメージによるデプロイ

Dockerfile 
# Dockerfileの例
FROM public.ecr.aws/lambda/python:3.11

# 依存関係をインストール
COPY requirements.txt ${LAMBDA_TASK_ROOT}
RUN pip install -r requirements.txt

# Lambda関数コードをコピー
COPY app.py ${LAMBDA_TASK_ROOT}

# ハンドラーを指定
CMD ["app.lambda_handler"]

3.5 バージョニングとエイリアス

Lambda関数の**バージョニング**と**エイリアス**を使用することで、安全なデプロイと段階的なリリースが可能になります。

バージョンの発行

Bash 
# 新しいバージョンを発行
aws lambda publish-version \
    --function-name my-function \
    --description "Initial release"

エイリアスの作成

Bash 
# prodエイリアスを作成してバージョン1を指定
aws lambda create-alias \
    --function-name my-function \
    --name prod \
    --function-version 1

3.6 トラフィックシフティング（カナリアデプロイ）

エイリアスを使用して、**段階的にトラフィックを新バージョンに移行**するカナリアデプロイを実装できます。これにより、新バージョンのリスクを最小化できます。

flowchart LR A["API Gateway"] --> B["Alias: prod"] B --> |"90% トラフィック"| C["Version 1
安定版"] B --> |"10% トラフィック"| D["Version 2
新バージョン"]

カナリアデプロイの設定

Bash 
# トラフィックの10%を新バージョンに向ける
aws lambda update-alias \
    --function-name my-function \
    --name prod \
    --function-version 2 \
    --routing-config '{"AdditionalVersionWeights": {"1": 0.9}}'

ヒント: ベストプラクティス

カナリアデプロイを実施する際は、CloudWatch Alarmsを設定し、エラー率が閾値を超えた場合に自動的に前のバージョンにロールバックする仕組みを構築しましょう。

3.7 AWS CDKによるInfrastructure as Code

AWS CDK（Cloud Development Kit）を使用すると、TypeScriptやPythonなどのプログラミング言語でインフラを定義できます。SAMよりも柔軟性が高く、複雑なサーバーレスアプリケーションに適しています。

CDKによるLambda関数の定義

TypeScript 
import * as cdk from 'aws-cdk-lib';
import * as lambda from 'aws-cdk-lib/aws-lambda';

export class MyLambdaStack extends cdk.Stack {
  constructor(scope: cdk.App, id: string) {
    super(scope, id);

    // Lambda関数を定義
    const myFunction = new lambda.Function(this, 'MyFunction', {
      runtime: lambda.Runtime.PYTHON_3_11,
      handler: 'app.lambda_handler',
      code: lambda.Code.fromAsset('lambda'),
      environment: {
        DB_HOST: 'localhost'
      }
    });
  }
}

CDKのデプロイ

Bash 
# CDKスタックをデプロイ
cdk deploy

理解度確認クイズ

  1. Q1. Lambda関数のハンドラーが受け取る2つの引数は何ですか?

    回答を表示

    eventとcontextです。eventにはトリガー元から渡されるデータが格納され、contextにはリクエストID、実行環境のメタデータが含まれます。

  2. Q2. 機密情報をLambda関数で管理する際のベストプラクティスは何ですか?

    回答を表示

    環境変数に直接保存せず、AWS Secrets ManagerまたはParameter Storeを使用して暗号化して管理することがベストプラクティスです。

  3. Q3. カナリアデプロイの目的は何ですか?

    回答を表示

    新バージョンへのトラフィックを段階的に増やすことで、リスクを最小化しながら安全にデプロイすることが目的です。問題が発生した場合、影響を受けるユーザーを最小限に抑えられます。
