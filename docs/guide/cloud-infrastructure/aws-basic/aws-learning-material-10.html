サーバーなしでプログラムを動かそう - Lambda入門

「サーバーレス」という新しい仕組みを体験します。サーバーを持たなくても、必要なときだけプログラムを動かせます。

この章で学ぶこと

  • サーバーレスは「必要なときだけ動く自販機」
  • Lambdaは「呼ばれたときだけ働くプログラム」
  • 簡単なプログラムの作り方(プログラミング経験不要)
  • 自動実行の設定(毎日決まった時間に動かす)
  • S3にファイルが追加されたら自動で処理する

10.1 サーバーレスとは何か？

これまでの章では、EC2でサーバーを起動してプログラムを動かしていました。しかし、「サーバーレス」では、サーバーを起動しなくてもプログラムを実行できます。

身近な例え

Lambda = 出前サービス(注文があったときだけ調理・配達)
EC2 = 常設レストラン(お客さんがいなくても店を開けておく)
トリガー = 注文ボタン(押されたら調理開始)

サーバーレスの最大のメリットは、「プログラムが動いた分だけ料金が発生する」こと。サーバーを常時起動させておく必要がないため、コストを大幅に削減できます。

EC2とLambdaの比較

flowchart LR subgraph A["EC2 (従来型)"] A1["サーバー常時起動"] A2["待機中も課金"] A3["手動でスケール調整"] end subgraph B["Lambda (サーバーレス)"] B1["必要なときだけ実行"] B2["実行時間分だけ課金"] B3["自動でスケール調整"] end A --> C{コストと管理} B --> C C --> D["Lambdaは小規模な処理に最適
待機時間が不要"]

重要ポイント

Lambdaは月100万リクエストまで無料！小規模なアプリケーションなら実質タダで運用できます。

10.2 Lambda関数を作成しよう

それでは実際にLambda関数を作ってみましょう。「Hello, Lambda!」と表示するシンプルなプログラムを作成します。

実習: 最初のLambda関数を作る

1 AWSコンソールでLambdaサービスを開く

検索ボックスに「Lambda」と入力し、サービスを選択します。

2 「関数の作成」をクリック

  • 「一から作成」を選択
  • 関数名: my-first-lambda
  • ランタイム: Python 3.12 を選択

3 「関数の作成」をクリック

デフォルト設定のまま作成します。

Lambda関数のコードを書く

関数が作成されると、コードエディタ画面が表示されます。以下のコードに書き換えましょう。

lambda_function.py 
# Lambda関数のメインハンドラー
def lambda_handler(event, context):
    # メッセージを定義
    message = "Hello, Lambda!"

    # 結果を返す
    return {
        'statusCode': 200,  # 成功ステータス
        'body': message      # 返却メッセージ
    }

コードの説明

  • lambda_handler: Lambdaが最初に呼び出す関数(エントリーポイント)
  • event: トリガーから渡されるデータ
  • context: 実行環境の情報
  • statusCode: HTTP 200 は成功を意味します

10.3 Lambda関数をテストしよう

コードを書いたら、実際に動作するか確認しましょう。

動作確認: Lambda関数を実行する

1. 「Deploy」ボタンをクリック

コードを保存・デプロイします。

2. 「Test」ボタンをクリック

初回はテストイベント設定画面が表示されます。

  • イベント名: test-event
  • デフォルトのJSONのまま「保存」

3. もう一度「Test」をクリック

実行結果が表示されます。

期待される結果 

{
  "statusCode": 200,
  "body": "Hello, Lambda!"
}

成功！ この結果が表示されればLambda関数は正常に動作しています。

10.4 トリガーを設定しよう(自動実行)

Lambdaは「トリガー」を設定することで、さまざまなイベントに反応して自動実行できます。

代表的なトリガー

EventBridge(CloudWatch Events)

定期実行(毎日9時、1時間ごとなど)

S3

ファイルアップロード時に自動実行

API Gateway

HTTPリクエスト受信時に実行

DynamoDB

データベース変更時に実行

例: 毎日9時に実行する設定

EventBridge(CloudWatch Events)を使って、毎朝9時に自動実行する設定をしてみましょう。

実習: 定期実行トリガーを設定

1 Lambda関数画面で「トリガーを追加」

2 トリガー設定

  • トリガーを選択: EventBridge(CloudWatch Events)
  • 「新しいルールの作成」を選択
  • ルール名: daily-9am-trigger
  • ルールタイプ: スケジュール式
  • スケジュール式: cron(0 0 * * ? *)

3 「追加」をクリック

cron式の読み方

cron(0 0 * * ? *) = 毎日午前0時(UTC)
日本時間(JST)は+9時間なので、午前9時に実行されます。

10.5 S3トリガーで画像処理を自動化

最後に、実用的な例として「S3にファイルがアップロードされたら自動で処理する」仕組みを作ってみましょう。

シナリオ

S3バケット my-upload-bucket に画像がアップロードされたら、 自動でログにファイル名を記録する。

S3トリガーの仕組み

flowchart LR A["ユーザー"] --> B["S3バケット"] B --> C{"ファイルアップロード
イベント発生"} C --> D["Lambda関数
自動起動"] D --> E["処理実行
ログ出力等"] style C fill:#fef3c7 style D fill:#dbeafe style E fill:#d1fae5
s3_trigger_lambda.py 
import json

def lambda_handler(event, context):
    # S3イベント情報を取得
    for record in event['Records']:
        # バケット名とファイル名を取得
        bucket = record['s3']['bucket']['name']
        key = record['s3']['object']['key']

        # ログ出力
        print(f"新しいファイルがアップロードされました!")
        print(f"バケット: {bucket}")
        print(f"ファイル名: {key}")

    return {
        'statusCode': 200,
        'body': json.dumps('処理完了!')
    }

注意事項

  • S3トリガーを設定する前に、Lambda関数にS3読み取り権限を付与する必要があります
  • トリガー設定時にバケット名とイベントタイプ(PUT, POST等)を指定します
  • 無限ループに注意!(Lambda → S3書き込み → 再びLambda起動)

10.6 Lambdaの制限と料金

実行時間制限

Lambdaには実行時間の上限があります。デフォルトは3秒、最大15分まで設定可能です。

ヒント

15分以上かかる処理は、ECSやBatch、Step Functionsなど別のサービスを検討しましょう。

料金体系

無料枠(毎月)

  • 100万リクエストまで無料
  • 40万GB秒の実行時間まで無料

有料の場合(無料枠超過後)

  • リクエスト料金: 100万リクエストあたり $0.20
  • 実行時間料金: メモリサイズと実行時間で計算

10.7 お片付け: Lambda関数を削除

Lambda関数は実行時のみ課金されますが、練習用の関数は削除しておきましょう。

削除手順

  1. Lambda管理画面で削除したい関数を選択
  2. 「アクション」→「削除」をクリック
  3. 関数名を入力して確認
  4. 「削除」をクリック

重要ポイント

トリガー(EventBridge等)も一緒に削除されるか確認しましょう。残っているとイベントが発生し続けることがあります。

初心者がつまずきやすいポイント

「ハンドラー」はプログラムの入り口

lambda_handlerという関数名は変更できますが、 設定で「ハンドラー名」も合わせて変更する必要があります。

タイムアウト設定に注意

デフォルト3秒で時間切れになると、処理が途中で止まります。 長い処理の場合は設定画面でタイムアウト時間を延長しましょう。

テスト実行で動作確認してから本番利用

いきなりトリガーを設定せず、まずは「Test」ボタンで正常動作を確認しましょう。

理解度確認クイズ

  1. Q1. Lambdaのメリットは何ですか？

    回答を表示

    サーバーを常時起動させる必要がなく、プログラムが動いた分だけ料金が発生するため、コストを削減できます。また、月100万リクエストまで無料です。

  2. Q2. Lambda関数の実行時間の上限は？

    回答を表示

    最大15分です。デフォルトは3秒なので、長い処理の場合は設定変更が必要です。

  3. Q3. S3トリガーで注意すべきことは？

    回答を表示

    無限ループに注意が必要です。Lambda関数がS3にファイルを書き込み、それがまたトリガーとなってLambdaが起動する、という循環を避ける設計が大切です。

この章のまとめ

  • サーバーレスは「必要なときだけ動く」仕組み。Lambdaは実行時間分だけ課金される。
  • Lambda関数はlambda_handlerから実行開始。月100万リクエストまで無料。
  • トリガーを設定することで、S3アップロード、定期実行、APIリクエストなど様々なイベントで自動実行できる。
  • 実行時間の上限は15分。タイムアウト設定に注意し、テスト実行で動作確認してから本番利用。
