11.1 これまで学んだサービスの全体像

10章に渡って学んだAWSの主要サービス。ここで一度全体像を整理しましょう。 各サービスは単独でも使えますが、組み合わせることでより強力なシステムが構築できます。

理解のポイント

IAMはすべてのサービスへのアクセスを管理する「門番」。CloudWatchはシステム全体を見守る「監視カメラ」。 VPCはサービスを安全に包む「ネットワークの壁」です。

各サービスの役割と使い分け

EC2 vs Lambda

EC2: 常時稼働が必要なアプリ向け

Lambda: たまに実行する処理向け

S3 vs RDS

S3: 画像・動画などファイル保存

RDS: 構造化データ（表形式）保存

11.2 実践プロジェクト: 自己紹介Webサイトを公開しよう

これまで学んだ知識を使って、S3で静的Webサイトを公開します。 プログラミング不要で、HTMLファイルをアップロードするだけで世界中からアクセスできるサイトが完成します。

実習: S3で自己紹介ページ公開

ステップ1: HTMLファイルを準備

以下の内容をメモ帳やテキストエディタにコピーし、index.htmlとして保存します。 

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>私の自己紹介</title>
</head>
<body>
    <h1>こんにちは！</h1>
    <p>AWSの学習を完了しました。</p>
</body>
</html>

ステップ2: S3バケットを作成

  1. AWSコンソールで「S3」を検索して開く
  2. 「バケットを作成」をクリック
  3. バケット名を入力（例: my-website-20250101）
  4. 「パブリックアクセスをすべてブロック」のチェックを外す
  5. 警告が出たら「承諾する」にチェック
  6. 「バケットを作成」をクリック

ステップ3: HTMLファイルをアップロード

  1. 作成したバケットをクリック
  2. 「アップロード」ボタンをクリック
  3. 先ほど作った「index.html」を選択
  4. 「アップロード」をクリック

ステップ4: Webサイトとして公開

  1. バケットの「プロパティ」タブを開く
  2. 一番下までスクロールして「静的ウェブサイトホスティング」を探す
  3. 「編集」をクリック
  4. 「有効にする」を選択
  5. インデックスドキュメントに「index.html」と入力
  6. 「変更を保存」をクリック
  7. 「アクセス許可」タブで「バケットポリシー」を編集し、以下を貼り付け（バケット名は自分のものに変更）
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::バケット名/*"
  }]
}

動作確認

  1. 「プロパティ」タブの「静的ウェブサイトホスティング」にあるURLをクリック
  2. ブラウザで自己紹介ページが表示されることを確認
  3. URLを友達にシェアしてアクセスできるか試してみよう

おめでとうございます！これであなたのWebサイトが世界中に公開されました！

11.3 お片付け: リソースの削除

学習が完了したら、課金を防ぐために作成したリソースを必ず削除しましょう。 削除の順番が重要です。

重要な注意

リソースを削除し忘れると、無料枠を超えて課金される可能性があります。 特にEC2インスタンスとRDSは停止しても料金がかかる場合があるので、必ず削除（終了）してください。

削除チェックリスト

削除する順番（上から順に）

  1. Lambda関数
    • Lambdaコンソールで関数を選択 → 「アクション」→「削除」
  2. RDSデータベース
    • RDSコンソールでインスタンスを選択 → 「アクション」→「削除」
    • 「最終スナップショットを作成しますか？」→「いいえ」
    • 確認のため「delete me」と入力
  3. EC2インスタンス
    • EC2コンソールでインスタンスを選択 → 「インスタンスの状態」→「インスタンスの終了」
  4. S3バケット
    • S3コンソールでバケットを選択 → まずバケット内のファイルを全削除
    • その後バケット自体を削除
  5. CloudWatchアラーム
    • CloudWatchコンソールで「アラーム」→ 作成したアラームを選択 → 削除
  6. IAMユーザー（練習用）
    • IAMコンソールで練習用ユーザーを選択 → 削除
    • ルートユーザーは削除しないこと！

確認方法

すべて削除したら、各サービスのコンソールを開いて「リソースなし」と表示されることを確認しましょう。 また、Billing（請求）ダッシュボードで現在の料金を確認することもおすすめです。

総まとめクイズ

  1. Q1. EC2とLambdaの使い分けを説明してください。

    回答を表示

    EC2は常時稼働が必要なWebアプリケーションなどに適しています。 一方、Lambdaは定期的なバッチ処理や、イベント駆動型の処理（ファイルアップロード時など）に適しており、 実行時間だけ課金されるためコスト効率が良いです。

  2. Q2. IAMがなぜ重要なのか説明してください。

    回答を表示

    IAMはAWSリソースへのアクセスを制御する仕組みです。 適切に設定することで、不正アクセスを防ぎ、各ユーザーやサービスに必要最小限の権限だけを与えることができます。 ルートユーザーを日常的に使わず、IAMユーザーを使うことがセキュリティのベストプラクティスです。

  3. Q3. 無料枠を超えて課金されないために、どんな対策が必要ですか？

    回答を表示

    料金アラートを設定して予算を超えたら通知を受け取る。 使わないリソースは必ず停止または削除する。 EC2やRDSは停止しても料金がかかる場合があるので、学習完了後は削除（終了）する。 定期的にBillingダッシュボードで料金を確認する。
