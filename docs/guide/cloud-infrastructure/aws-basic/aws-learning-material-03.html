AWSの画面に慣れよう - 管理画面の歩き方

AWSにログインすると、たくさんのメニューがあって最初は迷います。この章では、管理画面（マネジメントコンソール）の基本的な使い方を覚えます。

この章で学ぶこと

  • ログイン画面の種類（ルートユーザーとIAMユーザー）
  • 管理画面のメニュー構成
  • サービスの探し方（検索機能の使い方）
  • 「東京リージョン」の選び方（日本から使うときは東京がおすすめ）
  • よく使う画面をお気に入り登録する方法

3.1 ログイン画面の種類を理解しよう

AWSにログインする方法は主に2種類あります。「ルートユーザー」と「IAMユーザー」です。

身近な例え

ルートユーザー = 家の「マスターキー」（すべての部屋に入れる大家さんの鍵）
IAMユーザー = 「部屋の鍵」（必要な部屋だけ入れる住人の鍵）

ルートユーザー

AWSアカウント作成時に登録したメールアドレスとパスワードでログインします。 すべての機能にアクセスできる「最強の権限」を持っていますが、普段は使わないことが推奨されます。

警告: ルートユーザーの取り扱い注意

  • 普段の作業では使わない（乗っ取られると全てを失う）
  • 初期設定や課金設定など、特別な作業のときだけ使用
  • 必ずMFA（多要素認証）を設定する

IAMユーザー

ルートユーザーが作成する「日常業務用のアカウント」です。 必要な権限だけを与えることができるため、セキュリティ上安全です。実際の作業はこちらを使います。

3.2 管理画面（マネジメントコンソール）の構成

AWSにログインすると、「AWSマネジメントコンソール」という管理画面が表示されます。 最初は情報が多くて圧倒されるかもしれませんが、基本的な構成を理解すれば大丈夫です。

身近な例え

管理画面 = ショッピングモールの案内図
サービス検索 = フロアガイドで目的のお店を探す
リージョン選択 = どの支店を利用するか選ぶ

画面の主要エリア

1. 検索バー（画面上部）

「EC2」「S3」など、使いたいサービス名を入力して検索できます。 メニューを探し回るより、検索した方が早いです。

2. リージョン選択（画面右上）

「東京」「オレゴン」など、どの地域のデータセンターを使うか選択します。 日本から使う場合は「アジアパシフィック（東京）」がおすすめ。

3. サービスメニュー（画面左上）

すべてのAWSサービスがカテゴリ別に整理されています。 「コンピューティング」「ストレージ」など、種類ごとに分類されています。

4. アカウントメニュー（画面右上）

自分のアカウント情報、セキュリティ設定、サポート情報にアクセスできます。 料金・請求情報もここから確認します。

3.3 サービスの探し方・アクセス方法

AWSには200種類以上のサービスがありますが、探し方を知っていればすぐに目的のサービスにアクセスできます。

方法1: 検索バーを使う（推奨）

最も早く確実な方法です。画面上部の検索バーにサービス名を入力するだけです。

重要ポイント

検索は日本語でも英語でもOKです。 「EC2」「S3」などの略称でも、「仮想サーバー」のような説明文でも検索できます。

検索例 
# 検索バーでの入力例（どれでもOK）
EC2
S3
仮想サーバー
ストレージ
データベース

方法2: サービスメニューから探す

画面左上の「サービス」メニューをクリックすると、カテゴリ別に分類されたサービス一覧が表示されます。

主要カテゴリ

コンピューティング

EC2、Lambdaなど

ストレージ

S3、EBSなど

データベース

RDS、DynamoDBなど

ネットワーキング

VPC、Route 53など

セキュリティ

IAM、Cognitoなど

管理とガバナンス

CloudWatch、CloudTrailなど

方法3: お気に入り登録を活用する

よく使うサービスは「お気に入り」に登録しておくと便利です。

お気に入り登録の手順

  1. 1. サービス名の横にある「星マーク」をクリック
  2. 2. 検索バー下の「お気に入り」タブに追加される
  3. 3. 次回から「お気に入り」から素早くアクセス可能

3.4 リージョンの選択と変更

リージョンとは、AWSのデータセンターがある地域のことです。 画面右上のドロップダウンメニューから選択できます。

ヒント

日本から使う場合は「アジアパシフィック（東京）ap-northeast-1」を選びましょう。 通信速度が速く、料金も比較的安いです。

主要リージョン一覧

アジアパシフィック（東京）

ap-northeast-1

日本から最速・おすすめ

米国東部（バージニア北部）

us-east-1

最も古く、サービスが充実

アジアパシフィック（シンガポール）

ap-southeast-1

アジア全域向け

欧州（フランクフルト）

eu-central-1

ヨーロッパ向け

注意点

  • リージョンを変更すると、それまで作ったリソース（サーバーなど）は見えなくなります
  • リソースは各リージョンごとに独立して管理されます
  • 作業前に必ず正しいリージョンが選択されているか確認しましょう

3.5 初心者がつまずきやすいポイント

トラブルシューティング

問題: 作ったサーバーが見つからない

原因: リージョンが違う可能性があります。 画面右上のリージョン表示を確認し、サーバーを作成したリージョンに切り替えてください。

問題: 英語の画面が表示される

解決策: 画面下部の言語設定アイコンをクリックし、「日本語」を選択してください。 ブラウザの設定で日本語を優先言語にするのもおすすめです。

問題: サービスが多すぎて何を使えばいいかわからない

解決策: 最初は以下の5つだけ覚えればOKです。 EC2（サーバー）、S3（ファイル保存）、RDS（データベース）、IAM（権限管理）、CloudWatch（監視）。

理解度確認クイズ

  1. Q1. ルートユーザーとIAMユーザーの違いは何ですか？

    回答を表示

    ルートユーザーはすべての機能にアクセスできる「マスターキー」で、普段は使いません。 IAMユーザーは日常作業用のアカウントで、必要な権限だけを付与できます。

  2. Q2. 日本から使う場合、どのリージョンを選ぶべきですか？

    回答を表示

    「アジアパシフィック（東京）ap-northeast-1」を選びます。 通信速度が速く、料金も比較的安いためです。

  3. Q3. サービスを素早く見つける最も効率的な方法は？

    回答を表示

    画面上部の検索バーにサービス名を入力する方法が最も早いです。 日本語でも英語でも検索できます。

この章のまとめ

  • ルートユーザーは初期設定のみ使用、日常作業はIAMユーザーを使う。
  • 管理画面は「検索バー」「リージョン選択」「サービスメニュー」「アカウントメニュー」で構成。
  • サービス検索は画面上部の検索バーが最速。お気に入り登録も活用しよう。
  • 日本から使う場合は「東京リージョン（ap-northeast-1）」を選択。
  • リソースが見つからない場合は、リージョンが正しいか確認する。
