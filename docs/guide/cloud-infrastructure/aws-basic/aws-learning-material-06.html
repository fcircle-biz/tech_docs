インターネットの通り道を作ろう - ネットワーク入門

サーバーがインターネットとつながるためには「ネットワーク」の設定が必要です。VPCという仕組みを使って、安全な通信環境を作ります。

この章で学ぶこと

  • ネットワークは「通信の道路」
  • VPCは「自分専用の敷地」
  • サブネットは「敷地内の区画」
  • インターネットゲートウェイは「正門」
  • セキュリティグループは「警備員」

6.1 ネットワークとは何か？

ネットワークとは、複数のコンピュータやサーバーが情報をやり取りするための「通信の道路」です。

身近な例え

ネットワーク = 道路網（車が通るための道）
サーバー = 家や建物（道でつながっている）
データ = 車や宅配便（道路を通って届く）

AWSでは、サーバー（EC2）を起動しても、ネットワークが正しく設定されていないとインターネットに接続できません。 この章では、AWSのネットワーク設定の基本を学びます。

6.2 VPCとは？ - 自分専用の敷地を作る

VPC（Virtual Private Cloud）は、AWS上に作る「自分専用の仮想ネットワーク空間」です。

重要ポイント

VPCは「塀で囲まれた自分の家の敷地」のようなもの。外からは見えず、中に何を置くか自由に決められます。

VPCの主な特徴

分離された空間

他の人のVPCとは完全に分離されているので安全です。

自由にカスタマイズ

IPアドレス範囲、サブネット構成などを自分で設定できます。

6.3 サブネットとは？ - 敷地内の区画を作る

サブネットは、VPC内をさらに細かく区切った「区画」のことです。

ヒント

サブネット = 敷地内の「家」「庭」「車庫」のような区画分け。用途ごとに分けて管理します。

サブネットの種類

パブリックサブネット vs プライベートサブネット

パブリックサブネット

インターネットからアクセスできる区画

例：Webサーバー、APIサーバー

プライベートサブネット

インターネットから直接アクセスできない区画

例：データベースサーバー、内部処理サーバー

6.4 インターネットゲートウェイ - 外の世界への門

インターネットゲートウェイ（IGW）は、VPCとインターネットをつなぐ「正門」です。

重要ポイント

インターネットゲートウェイがないと、VPC内のサーバーはインターネットに接続できません。正門がないと敷地から外に出られないのと同じです。

インターネットゲートウェイの設定手順

設定ステップ

  1. IGWを作成 - VPCコンソールから「インターネットゲートウェイ」を作成
  2. VPCにアタッチ - 作成したIGWを自分のVPCに接続
  3. ルートテーブルに設定 - パブリックサブネットからインターネット宛の通信がIGWを通るよう設定

6.5 セキュリティグループ - 通信を守る警備員

セキュリティグループは、サーバーへの通信を制御する「仮想ファイアウォール」です。

身近な例え

セキュリティグループ = 玄関の警備員
「誰が（どのIPアドレス）」「どこに（どのポート番号）」アクセスできるかをチェックします。

セキュリティグループのルール

セキュリティグループには、インバウンドルール（受信）とアウトバウンドルール（送信）があります。

よく使うルールの例

タイプ プロトコル ポート 送信元 用途
HTTP TCP 80 0.0.0.0/0 Webサイト公開
HTTPS TCP 443 0.0.0.0/0 暗号化Web
SSH TCP 22 自宅IP サーバー管理

注意事項

すべてのIPアドレス（0.0.0.0/0）を許可すると、世界中から接続できるようになります。セキュリティリスクがあるため、本番環境では特定のIPのみに制限しましょう。

6.6 IPアドレスとは？ - インターネット上の住所

IPアドレスは、インターネット上での「住所」のようなものです。

ヒント

IPアドレス = 「〒123-4567 東京都〇〇区...」のような住所
データはこの住所を頼りに正しい場所へ届けられます。

IPアドレスの種類

パブリックIP

インターネットからアクセスできる住所

例: 203.0.113.10

プライベートIP

VPC内だけで使える住所

例: 10.0.1.50

初心者は詳細な数値を覚える必要はありません。「インターネットからアクセスするにはパブリックIPが必要」ということだけ理解しておけば十分です。

実習: デフォルトVPCを確認しよう

AWSアカウントを作成すると、各リージョンに「デフォルトVPC」が自動的に作成されています。まずはこれを確認してみましょう。

  1. 1. AWSマネジメントコンソールにログイン
  2. 2. 検索バーに「VPC」と入力してサービスを開く
  3. 3. 左メニューから「VPC」を選択し、デフォルトVPCを確認
  4. 4. 「サブネット」メニューから、デフォルトサブネットの数を確認（通常3〜4個）
  5. 5. 「インターネットゲートウェイ」メニューから、デフォルトIGWが接続されていることを確認

動作確認

以下の項目が確認できればOKです：

  • VPCコンソールが開ける
  • デフォルトVPCが1つ表示されている
  • サブネットが複数（3〜4個）表示されている
  • インターネットゲートウェイが表示されている

トラブルシューティング

Q. デフォルトVPCが見つからない

A. リージョンを確認してください。東京リージョン（ap-northeast-1）が選択されているか確認しましょう。

Q. VPCの設定を間違えて変更してしまった

A. デフォルトVPCは削除して再作成できます。VPCコンソールの「アクション」→「デフォルトVPCを作成」で復元できます。

Q. サブネットの数字の意味がわからない

A. 最初は気にしなくてOKです。AWSが自動的に適切な設定をしています。慣れてきたら学習しましょう。

理解度確認クイズ

  1. Q1. VPCを身近なもので例えると何ですか？

    回答を表示

    塀で囲まれた自分の家の敷地。外からは見えず、中に何を置くか自由に決められる空間です。

  2. Q2. パブリックサブネットとプライベートサブネットの違いは？

    回答を表示

    パブリックサブネット: インターネットからアクセスできる（Webサーバーなど）
    プライベートサブネット: インターネットから直接アクセスできない（データベースなど）

  3. Q3. セキュリティグループの役割は？

    回答を表示

    サーバーへの通信を制御する仮想ファイアウォール。玄関の警備員のように、「誰が」「どこに」アクセスできるかをチェックします。

この章のまとめ

  • ネットワークは「通信の道路」。サーバー同士がデータをやり取りするために必要。
  • VPCは「自分専用の敷地」。AWS上に作る仮想ネットワーク空間。
  • サブネットは「敷地内の区画」。パブリック（外部アクセス可）とプライベート（内部専用）がある。
  • インターネットゲートウェイは「正門」。VPCとインターネットをつなぐ。
  • セキュリティグループは「警備員」。通信を制御して安全を守る。
  • デフォルトVPCは自動作成されるので、初心者はそれを使えばOK。
