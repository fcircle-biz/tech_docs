第4章

最初のサーバーを作ってみよう - EC2入門

いよいよサーバー（インターネット上のパソコン）を作ります。「サーバー」と聞くと難しそうですが、手順通りに進めれば15分で完成します。

この章で学ぶこと

  • サーバーとは「24時間動いているパソコン」
  • EC2は「レンタルパソコン」のサービス
  • 無料で使えるサーバーの選び方（t2.micro / t3.micro）
  • サーバーの作り方（ステップバイステップ）
  • サーバーに接続する方法
  • 使い終わったらサーバーを止める方法（課金を防ぐ）

4.1 サーバーって何？

サーバーとは、一言で表すと「24時間動き続けているパソコン」です。 普段使っているパソコンとの違いは、常に電源が入っていて、いつでもインターネット経由でアクセスできることです。

身近な例え

サーバー = コンビニ（24時間営業、いつでも利用できる）
普通のパソコン = 自宅の冷蔵庫（自分の家族だけが使う）

サーバーの役割

Webサイト配信

インターネット上のWebサイトは、すべてサーバーに保存されています。

データ保存

ユーザー情報、商品情報などを保存する場所として使われます。

プログラム実行

検索機能や計算処理など、様々なプログラムを実行します。

メール送受信

メールサーバーとして、メールの送受信を管理します。

4.2 EC2とは？

EC2（Elastic Compute Cloud）は、AWSが提供する「レンタルパソコン」のサービスです。 「Elastic（弾力的）」という名前の通り、必要に応じてサーバーのサイズを増やしたり減らしたりできます。

重要ポイント

EC2 = レンタカーのようなもの。必要なときだけ借りて、使い終わったら返す。借りている間だけ料金が発生します。

EC2の仕組み

（レンタルサーバー）"] EC2 -->|保存| EBS["EBSストレージ
（ハードディスク）"] EC2 -->|通信| VPC["VPC
（ネットワーク）"] AWS["AWS データセンター"] -.->|物理的に配置| EC2

EC2の主要概念

インスタンス

レンタルしたサーバーそのもの。「車」のようなもの。

キーペア

サーバーに接続するための「鍵」。絶対になくさないこと！

セキュリティグループ

サーバーへのアクセスを制限する「警備員」のようなもの。

EBS（Elastic Block Store）

サーバーのハードディスク。データを保存する場所。

4.3 無料で使えるインスタンスタイプ

AWSには「無料利用枠」があり、初めてアカウントを作成してから12ヶ月間、一部のサービスを無料で使えます。 EC2では、t2.microまたはt3.microというタイプのサーバーを、月750時間まで無料で使えます。

ヒント

月750時間 = 約31日 × 24時間。つまり、サーバーを1台だけ動かし続けても1ヶ月フル稼働で無料！

インスタンスタイプの見方

t2.micro の意味

t2

インスタンスファミリー（汎用タイプ）

micro

サイズ（最小サイズ）

スペック:

  • CPU: 1コア
  • メモリ: 1GB
  • ネットワーク: 低〜中程度

注意

t2.small、t2.medium など、「micro」以外のサイズを選ぶと課金されます！必ず「t2.micro」または「t3.micro」を選んでください。

4.4 実習: EC2インスタンスを起動してみよう

それでは、実際にサーバーを作ってみましょう！画面の指示に従って進めてください。

実習 4-1: EC2インスタンスの起動

無料利用枠でサーバーを起動し、接続できる状態にします。

手順

  1. 1 AWSマネジメントコンソールにログイン
  2. 2 画面右上のリージョンが「東京」になっているか確認
  3. 3 検索バーに「EC2」と入力し、「EC2」サービスをクリック
  4. 4 「インスタンスを起動」ボタンをクリック
  5. 5 インスタンス名を入力（例: my-first-server）
  6. 6 「Amazon Linux 2023 AMI」を選択（デフォルトで選ばれている）
  7. 7 インスタンスタイプで「t2.micro」または「t3.micro」を選択
  8. 8 キーペアで「新しいキーペアを作成」をクリック
    - キーペア名: my-key
    - キーペアタイプ: RSA
    - 秘密鍵ファイル形式: .pem（Macの場合）または .ppk（Windowsの場合）
    - 「キーペアを作成」をクリック → ファイルがダウンロードされます（大切に保管！）
  9. 9 ネットワーク設定はデフォルトのまま（後の章で詳しく学びます）
  10. 10 画面右側の「概要」で、無料利用枠の対象であることを確認
  11. 11 「インスタンスを起動」ボタンをクリック
  12. 12 「インスタンスを表示」をクリックして、起動状態を確認

起動完了の確認

インスタンスの「インスタンスの状態」が「実行中」になり、「ステータスチェック」が「2/2のチェックに合格しました」と表示されれば完了です。

キーペアは絶対になくさない！

  • ダウンロードしたキーペアファイル（.pem または .ppk）は、再発行できません
  • なくすとサーバーに接続できなくなります
  • 安全な場所（USBメモリ、クラウドストレージなど）にバックアップしましょう

4.5 EC2インスタンスに接続してみよう

サーバーが起動したら、次は接続してみましょう。AWSには「EC2 Instance Connect」という、ブラウザから簡単に接続できる機能があります。

実習 4-2: ブラウザからサーバーに接続

EC2 Instance Connectを使って、コマンドラインでサーバーを操作します。

手順

  1. 1 EC2ダッシュボードで、先ほど作成したインスタンスにチェックを入れる
  2. 2 画面上部の「接続」ボタンをクリック
  3. 3 「EC2 Instance Connect」タブを選択（デフォルトで選ばれている）
  4. 4 「接続」ボタンをクリック
  5. 5 新しいタブでターミナル（黒い画面）が開きます

試しにコマンドを実行してみよう

以下のコマンドを入力して、Enterキーを押してみてください。

コマンド 
# 現在のディレクトリを表示
pwd

# ファイル一覧を表示
ls -la

# OSのバージョンを確認
cat /etc/os-release

ヒント

ターミナルは「コマンドライン」や「シェル」とも呼ばれます。文字でコンピュータを操作するための画面です。最初は難しく感じるかもしれませんが、基本的なコマンドだけ覚えれば大丈夫！

4.6 インスタンスの「停止」と「終了」の違い

EC2には「停止」と「終了（削除）」という2つの操作があります。この違いを理解しておくことが、課金を防ぐために非常に重要です。

停止と終了の違い

flowchart LR A["インスタンス
（実行中）"] -->|停止| B["インスタンス
（停止中）"] B -->|再起動| A A -->|終了| C["インスタンス
（削除）"] B -.->|データは残る
EBS課金あり| D[EBSストレージ] C -.->|完全削除
課金なし| E[なし]

停止（Stop）

  • サーバーを一時停止
  • データは残る
  • 再起動できる
  • EBS（ストレージ）の課金は続く

終了（Terminate）

  • サーバーを完全削除
  • データも全部消える
  • 元に戻せない
  • 課金は完全に止まる

重要ポイント

停止 = レンタカーを駐車場に停めておく（また使える）
終了 = レンタカーを返却する（もう使えない）

4.7 お片付け: インスタンスを停止・終了する

練習が終わったら、必ずインスタンスを停止または終了しましょう。予期しない課金を防ぐためです。

実習 4-3: インスタンスの停止

また使う予定がある場合は「停止」を選びます。

手順

  1. 1 EC2ダッシュボードで、インスタンスにチェックを入れる
  2. 2 「インスタンスの状態」→「インスタンスを停止」をクリック
  3. 3 確認画面で「停止」をクリック
  4. 4 「インスタンスの状態」が「停止中」→「停止済み」になることを確認

実習 4-4: インスタンスの終了（削除）

もう使わない場合は「終了」を選びます。完全に削除されます。

手順

  1. 1 EC2ダッシュボードで、インスタンスにチェックを入れる
  2. 2 「インスタンスの状態」→「インスタンスを終了」をクリック
  3. 3 確認画面で「終了」をクリック（慎重に！）
  4. 4 「インスタンスの状態」が「終了中」→「終了済み」になることを確認

注意: 終了すると元に戻せません！本当に削除して良いか、よく確認してから実行してください。

理解度確認クイズ

  1. Q1. EC2とは何ですか？

    回答を表示

    AWSが提供する「レンタルパソコン」のサービスです。必要なときだけサーバーを借りて、使い終わったら返すことができます。

  2. Q2. 無料利用枠で使えるインスタンスタイプは何ですか？

    回答を表示

    t2.micro または t3.micro です。月750時間まで無料で使えます。

  3. Q3. インスタンスの「停止」と「終了」の違いは何ですか？

    回答を表示

    「停止」は一時停止でデータは残り、再起動できます。「終了」は完全削除で、データも消えて元に戻せません。停止中もEBS（ストレージ）の課金は続きますが、終了すれば課金は完全に止まります。

  4. Q4. キーペアファイルをなくすとどうなりますか？

    回答を表示

    キーペアファイルは再発行できないため、なくすとサーバーに接続できなくなります。必ず安全な場所にバックアップを取りましょう。
