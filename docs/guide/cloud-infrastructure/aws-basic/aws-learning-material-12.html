12.1 AWS資格でスキルを証明しよう

AWSには公式認定資格があります。資格を取得すると、AWSの知識を客観的に証明できます。

  • 試験時間: 90分
  • 問題数: 65問（選択式）
  • 合格ライン: 700点/1000点

12.5 よくある質問

Q1. このガイドを終えたら次に何をすればいい？

A: 以下の順番で進めるのがおすすめです。

  1. AWS Skill Builderで「AWS Cloud Practitioner Essentials」を受講
  2. AWS公式ハンズオンチュートリアルで実践練習
  3. クラウドプラクティショナー資格取得にチャレンジ
  4. 興味のある分野（インフラ/開発/AI）の専門サービスを深掘り
Q2. 無料利用枠が終わったらどうなる？

A: 無料利用枠は12ヶ月間の期間限定分と、永続的に使える分があります。

  • 12ヶ月後も、EC2 t2.micro 750時間/月などは使えます
  • Lambda 100万リクエスト/月、S3 5GBなどは永続無料
  • 期間限定分が終了すると、通常料金が発生
  • CloudWatchで料金アラートを設定しておけば安心
Q3. AWSを仕事で使えるレベルになるには？

A: レベル別の目安は以下の通りです。

  • 初級レベル（このガイド完了後）: 基本サービスの理解、簡単な環境構築
  • 中級レベル（3〜6ヶ月）: アソシエイトレベル資格取得、実務プロジェクト参加可能
  • 上級レベル（1〜2年）: アーキテクチャ設計、本番環境運用
  • エキスパート（3年以上）: プロフェッショナル資格、大規模システム設計
Q4. Azure や GCP など他のクラウドも学ぶべき？

A: まずはAWSをしっかり習得することをおすすめします。

  • AWSは世界シェアNo.1で、求人も最も多い
  • 1つのクラウドをマスターすれば、他のクラウドも理解しやすい
  • 基本概念（仮想サーバー、ストレージ、ネットワーク）は共通
  • 企業によってはマルチクラウド運用もあるため、将来的に複数習得も有益
Q5. プログラミングができないとAWSは使えない？

A: いいえ、プログラミング不要でも十分使えます。

  • このガイドで学んだサービスは、ほとんどマウス操作だけで使える
  • インフラエンジニア職であれば、GUIでの操作がメイン
  • ただし、Lambdaや自動化を使うにはプログラミングスキルが必要
  • Python や Bash を学ぶと、さらに便利に使えるようになります

12.6 学習継続のコツ

AWSは機能が多く、すべてを覚えるのは困難です。継続的に学習を続けるためのヒントを紹介します。

定期的な学習習慣

毎日15分でもOK。朝起きたら、通勤中、昼休みなど、決まった時間に学習すると継続しやすい。

小さな目標設定

「今週はS3を完璧にする」など、週単位で達成可能な目標を立てる。達成感がモチベーションになる。

学んだことをアウトプット

ブログ、Qiita、Zennなどで学習メモを公開。誰かの役に立つと実感できて継続しやすい。

仲間と一緒に学ぶ

勉強会やオンラインコミュニティに参加。同じ目標を持つ仲間がいると挫折しにくい。

ヒント

完璧を目指さない。最初は「なんとなく分かる」レベルでOK。実際に手を動かしながら、徐々に理解を深めていきましょう。
