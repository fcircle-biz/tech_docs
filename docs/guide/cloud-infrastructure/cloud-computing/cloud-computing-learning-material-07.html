7.1 クラウドの料金体系を理解する

クラウドコンピューティングの大きな特徴の一つが従量課金制（Pay-as-you-go）です。従来のオンプレミス方式では、サーバーを購入する際に数百万円〜数千万円の初期投資が必要でしたが、クラウドでは使った分だけ料金を払います。

身近な例で理解しよう

携帯電話の料金プランを考えてみましょう。
・定額プラン：月額5,000円でデータ通信無制限
・従量課金プラン：使ったデータ量に応じて1GB = 1,000円

クラウドも同じように、使ったリソース（コンピュータ、ストレージ、通信）に応じて料金が発生します。

料金が発生する主な要素

クラウドでは、以下の要素に対して料金が発生します：

1. コンピューティング（仮想マシン）

課金単位：時間単位（または秒単位）
料金例：小型サーバー 約5円/時間〜
注意：起動している間ずっと課金される

💡 使わない時は停止すれば課金されない（電気を消すのと同じ）

2. ストレージ（データ保管）

課金単位：GB/月単位
料金例：約2〜3円/GB/月
注意：保存している間ずっと課金される

💡 不要なファイルは削除しよう（倉庫の整理と同じ）

3. データ転送（通信量）

課金単位：GB単位
料金例：約10円/GB（外部への送信）
注意：受信は無料、送信が有料のことが多い

💡 大きな動画ファイルの配信は要注意（転送量が多い）

4. その他のサービス

課金単位：サービスごとに異なる
例：データベース、ロードバランサー、監視サービス等
注意：サービスごとに料金体系が違う

7.2 3大プロバイダーの料金比較

AWS、Azure、GCPの料金体系は基本的に似ていますが、それぞれ特徴があります。また、初めて使う人向けに無料利用枠（Free Tier）が用意されています。

無料利用枠の活用方法

AWS 無料利用枠

期間：12ヶ月間（一部サービスは永久無料）
内容：EC2 750時間/月、S3 5GB、RDS 750時間/月 など
開始条件：新規アカウント作成時にクレジットカード登録

注意：無料枠を超えると自動的に課金されます。予算アラート設定が重要！

Azure 無料アカウント

期間：12ヶ月間 + 初回200ドルクレジット（30日間）
内容：仮想マシン 750時間/月、Blob Storage 5GB など
開始条件：新規アカウント作成時にクレジットカード登録

特徴：初回200ドルクレジットで、有料サービスも試せる

GCP 無料トライアル

期間：90日間 + 300ドルクレジット
内容：全サービスで使える300ドル分のクレジット
開始条件：新規アカウント作成時にクレジットカード登録

特徴：90日間または300ドルを使い切るまで無料（どちらか先に来た方）

・リージョン（地域）によって料金が異なることに注意

7.3 コストが膨らむ原因

クラウドは便利ですが、知らないうちに料金が高額になってしまうことがあります。よくある原因を理解しておきましょう。

1. 使っていないリソースの放置

最も多い原因：テストで作った仮想マシンを削除し忘れて、何ヶ月も起動したまま
影響：月額数万円〜数十万円の無駄なコスト

対策：
・週に一度、使っていないリソースをチェックする習慣をつける
・タグ機能で「テスト用」「本番用」などラベル付けして管理
・自動削除スケジュールを設定する

2. 過剰なスペックの選択

よくあるミス：「念のため」と大きなサーバーを選んでしまう
影響：実際に必要なスペックの2〜3倍の料金を払う

対策：
・まず小さなサイズから始める（後から拡張できる）
・監視ツールでCPU・メモリ使用率をチェック
・使用率が低ければ、小さなサイズに変更する

3. データ転送料金の見落とし

盲点：データ転送料金は意外と高額（1GB = 約10円）
影響：大量のデータ配信で数十万円の請求が来ることも

対策：
・CDN（コンテンツ配信ネットワーク）を活用してデータ転送を削減
・画像や動画を圧縮してファイルサイズを小さくする
・同じリージョン内の通信は無料なので、構成を工夫

4. 不要なスナップショット・バックアップの蓄積

問題：自動バックアップが溜まり続けて、ストレージ料金が増加
影響：月数千円〜数万円の無駄なストレージコスト

対策：
・バックアップの保持期間を設定（例：30日で自動削除）
・古いスナップショットは定期的に手動削除
・ライフサイクルポリシーで自動削除を設定

5. 無限ループなどの設定ミス

危険：プログラムのバグで同じ処理が無限に実行される
影響：数時間で数万円〜数十万円の請求

対策：
・予算アラートを必ず設定（後述）
・テスト環境では利用上限を設定
・異常な課金があったらすぐにメール通知されるようにする

7.4 コスト最適化の基本テクニック

クラウドのコストを最適化するための具体的なテクニックを学びましょう。

1. リソースの適正サイズ選択（Rightsizing）

必要以上に大きなサーバーを使わず、実際の負荷に合ったサイズを選ぶことが重要です。

具体例：

  • ・CPU使用率が常に10%以下 → 1つ小さいサイズに変更
  • ・メモリ使用率が80%以上 → 1つ大きいサイズに変更
  • ・夜間は使わない → スケジュールで自動停止

2. リザーブドインスタンス（Reserved Instances）

仕組み：1年または3年の長期契約をすることで、最大75%割引
向いているケース：24時間365日稼働するサーバー

例：通常料金10,000円/月のサーバー → リザーブドで3,000円/月に
注意：途中解約できないので、確実に使い続けるリソースだけに適用

3. スポットインスタンス（Spot Instances）

仕組み：クラウド事業者の余っているサーバーを格安で使う（最大90%割引）
注意：いつでも停止される可能性がある

向いている用途：
・データ分析のバッチ処理（途中で止まっても再開できる）
・開発・テスト環境（本番環境には不向き）
不向き：Webサーバーなど、常時稼働が必要なもの

4. 自動停止・開始のスケジュール設定

開発環境など、業務時間外は使わないサーバーを自動で停止・起動する。

例：平日9時に起動、18時に停止するスケジュール
→ 稼働時間が1/3になり、コストも1/3に削減
実現方法：AWS Lambda、Azure Automation、Cloud Scheduler等

5. ストレージクラスの最適化

アクセス頻度に応じて、ストレージクラスを使い分ける。

ストレージクラスの例（AWS S3）：

  • ・Standard：頻繁にアクセスするデータ（高い）
  • ・Infrequent Access：月1回程度のアクセス（中）
  • ・Glacier：年1回程度、アーカイブ用（安い）

古いログファイルなどはGlacierに移動すると、コストを1/10に削減できる

7.5 コスト管理ツールの紹介

各クラウドプロバイダーは、コストを管理するための便利なツールを提供しています。これらを活用することで、予期しない高額請求を防げます。

予算アラートの設定（最重要）

クラウドを使い始めたら、必ず最初に設定すべきなのが予算アラートです。

予算アラート設定は必須！

予算アラートを設定しないと、気づかないうちに数十万円の請求が来ることがあります。
必ず、クラウド利用開始の初日に設定しましょう。

AWS Budgets（予算管理）

機能：月の予算を設定し、超えそうになったらメール通知
設定例：「月5,000円を超えたらアラート」「80%到達でメール」

設定方法：AWSマネジメントコンソール → Billing → Budgets → Create budget

Azure Cost Management（コスト管理）

機能：コスト分析、予算設定、推奨事項の提示
特徴：AIがコスト削減の提案をしてくれる

設定方法：Azureポータル → Cost Management + Billing → Budgets

Google Cloud Billing（予算アラート）

機能：予算しきい値の設定、超過時のメール・Pub/Sub通知
特徴：自動停止のトリガーとしても使える

設定方法：Google Cloudコンソール → Billing → Budgets & alerts

コスト分析ダッシュボード

各プロバイダーは、コストを可視化するダッシュボードを提供しています。

コスト分析でできること

  • サービスごとのコスト内訳を確認
  • 日別・週別・月別のコスト推移
  • リソースタグによるグループ化
  • 予測：現在の使い方なら月末にいくらになるか

定期的なチェックが重要

  • 毎日：異常な増加がないか確認
  • 毎週：不要なリソースを探して削除
  • 毎月：コスト最適化の施策を検討
  • 四半期：リザーブド購入を検討

7.6 FinOps（クラウド財務管理）の考え方

FinOps（Financial Operations）とは、クラウドコストの管理を技術部門だけでなく、財務・経営部門も含めた組織全体で行う考え方です。

FinOpsが重要な理由

クラウドでは、エンジニアが自由にリソースを作成できるため、コストが急増しやすいです。
FinOpsの考え方を取り入れることで、コストの可視化・最適化・説明責任を明確にし、ビジネス価値を最大化します。

FinOpsの3つの原則

1. チーム間の協力

技術・財務・ビジネスの各チームが協力してコストを管理する。

2. ビジネス価値の最大化

コスト削減だけでなく、ビジネスへの貢献を重視する。

3. コストの可視化と説明責任

誰がいくら使っているかを明確にし、各チームが責任を持つ。

初心者でもできるFinOps実践

・タグ付け：リソースに「プロジェクト名」「担当者」などのタグをつける
・定例会議：週1回、コストレポートをチーム内で共有
・目標設定：「今月は先月より10%コスト削減」など具体的な目標を持つ
・インセンティブ：コスト削減に成功したチームを表彰する

実践演習：コスト最適化シナリオ

シナリオ：

あなたの会社は、AWSでWebサービスを運営しています。今月の請求額が予想の2倍（20万円）になってしまいました。
以下の状況を確認したところ、いくつかの問題が見つかりました。

  • ・開発用サーバー（5台）が24時間稼働している
  • ・古いスナップショット（100個）が削除されずに残っている
  • ・本番サーバーが過剰スペック（CPU使用率10%）
  • ・データ転送量が先月の3倍に増加

課題：各問題に対して、どのような対策を取りますか？

解答例を表示

1. 開発用サーバーの対策：

→ 業務時間外（平日18時〜翌9時、土日）は自動停止するスケジュールを設定
→ 効果：稼働時間が約1/3になり、月6万円削減（20万円 × 30%）

2. 古いスナップショットの対策：

→ 30日以上前のスナップショットを削除
→ ライフサイクルポリシーで、今後は自動削除されるように設定
→ 効果：月1万円削減

3. 本番サーバーの過剰スペック：

→ 1つ小さいインスタンスタイプに変更（CPU使用率が30%程度になるサイズ）
→ 効果：月4万円削減

4. データ転送量の増加：

→ CloudFront（CDN）を導入して、画像配信を最適化
→ 画像を圧縮して、ファイルサイズを削減
→ 効果：月5万円削減

合計削減額：約16万円/月

20万円 → 4万円に改善（元の予算10万円を大幅に下回る）

理解度確認クイズ

  1. Q1. クラウドの従量課金制において、料金が発生する主な要素を3つ挙げてください。

    回答を表示

    1. コンピューティング（仮想マシン） - 時間単位で課金
    2. ストレージ（データ保管） - GB/月単位で課金
    3. データ転送（通信量） - GB単位で課金（外部への送信）
    その他、データベース、ロードバランサーなど各種サービスでも料金が発生します。

  2. Q2. クラウドのコストが予想外に膨らむ原因を2つ挙げてください。

    回答を表示

    代表的な原因：
    1. 使っていないリソースの放置 - テスト用サーバーを削除し忘れて課金され続ける
    2. 過剰なスペックの選択 - 必要以上に大きなサーバーを使ってしまう
    3. データ転送料金の見落とし - 大量のデータ配信で高額請求
    4. 不要なバックアップの蓄積 - 古いスナップショットを削除せずに溜め続ける
    このうち2つ以上答えられれば正解です。

  3. Q3. リザーブドインスタンスとスポットインスタンスの違いを説明してください。

    回答を表示

    リザーブドインスタンス：
    ・1年または3年の長期契約で、最大75%割引
    ・24時間365日稼働するサーバーに向いている
    ・途中解約できないが、安定して使える

    スポットインスタンス：
    ・余っているサーバーを格安で使う（最大90%割引）
    ・いつでも停止される可能性がある
    ・バッチ処理や開発環境など、途中で止まっても大丈夫な用途に向いている

  4. Q4. クラウドを使い始めたら、必ず最初に設定すべきことは何ですか？

    回答を表示

    予算アラートの設定です。

    予算額を設定し、その額を超えそうになったらメールで通知されるようにします。
    例：「月5,000円を超えたらアラート」「80%到達でメール」

    これを設定しないと、予期しない高額請求に気づけず、数十万円の請求が来ることもあります。

  5. Q5. FinOps（クラウド財務管理）とは何ですか？簡潔に説明してください。

    回答を表示

    FinOps（Financial Operations）は、クラウドコストの管理を技術部門だけでなく、財務・経営部門も含めた組織全体で行う考え方です。

    3つの原則：
    1. チーム間の協力（技術・財務・ビジネスが協力）
    2. ビジネス価値の最大化（コスト削減だけでなく貢献度を重視）
    3. コストの可視化と説明責任（誰がいくら使っているかを明確に）

    これにより、無駄なコストを削減しながら、ビジネス価値を高めることができます。

第7章のまとめ

クラウドは従量課金制で、コンピューティング・ストレージ・データ転送などの使用量に応じて料金が発生する

AWS、Azure、GCPはそれぞれ無料利用枠を提供しており、料金計算ツールで事前見積もりが可能

コストが膨らむ主な原因は、リソースの放置・過剰スペック・データ転送量の見落とし・不要なバックアップ蓄積など

コスト最適化のテクニック：適正サイズ選択、リザーブドインスタンス、スポットインスタンス、自動停止、ストレージクラス最適化

予算アラートの設定は必須。コスト分析ダッシュボードで定期的にチェックする習慣をつける

FinOpsの考え方を取り入れ、組織全体でコスト管理を行い、ビジネス価値を最大化する
