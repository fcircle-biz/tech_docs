クラウドを始めよう - 最初の一歩と学習ロードマップ

実際にクラウドを始めるための具体的な手順と、今後の学習の進め方を解説します。

この章で学ぶこと

  • AWS、Azure、GCPの無料アカウント作成手順

8.1 無料アカウントの作成手順

それでは、実際にクラウドサービスを使い始めましょう。3大クラウドプロバイダー（AWS、Azure、GCP）はいずれも無料利用枠を提供しています。

重要な注意事項

無料枠を超えると課金される可能性があります。アカウント作成時にクレジットカードの登録が必要ですが、予算アラート（使用量通知）を必ず設定し、使い終わったリソースは必ず削除しましょう。

AWS無料利用枠の活用

AWSでは、アカウント作成から12ヶ月間、主要サービスを無料で利用できます。

無料利用枠の例：

  • EC2（仮想サーバー）: t2.micro インスタンス 750時間/月
  • S3（ストレージ）: 5GB の標準ストレージ
  • RDS（データベース）: db.t2.micro インスタンス 750時間/月

公式サイト：
https://aws.amazon.com/jp/free/

Azure無料アカウントの作成

Microsoftアカウントで登録でき、200ドル分のクレジット（30日間有効）と、12ヶ月間の無料サービスが利用できます。

無料利用枠の例：

  • Virtual Machines（仮想マシン）: B1S インスタンス 750時間/月
  • Blob Storage（オブジェクトストレージ）: 5GB
  • SQL Database（データベース）: 250GB

公式サイト：
https://azure.microsoft.com/ja-jp/free/

GCP無料トライアルの開始

Googleアカウントで登録でき、300ドル分のクレジット（90日間有効）と、常時無料の枠が利用できます。

無料利用枠の例：

  • Compute Engine（仮想マシン）: e2-micro インスタンス 1台（常時無料）
  • Cloud Storage（ストレージ）: 5GB（常時無料）
  • BigQuery（データ分析）: 1TB のクエリ処理/月（常時無料）

公式サイト：
https://cloud.google.com/free

8.2 最初に試してほしいこと

アカウントを作成したら、まずは以下の3つのステップを試してみましょう。

1

管理コンソールを触ってみる

まずは、Webブラウザから管理コンソール（コントロールパネル）にログインして、どんなサービスがあるか眺めてみましょう。

やってみること：
・サービス一覧を見て、どんなカテゴリがあるか確認
・ダッシュボードで現在の使用状況を確認
・料金ページで現在のコストを確認（最初は0円のはず）

2

簡単なリソースを作成してみる

無料枠の範囲で、簡単なリソース（仮想マシンやストレージ）を作成してみましょう。

おすすめの最初のリソース：

  • AWS: S3バケット（ストレージ）を作成して、ファイルをアップロード
  • Azure: Blob Storageコンテナーを作成して、画像をアップロード
  • GCP: Cloud Storageバケットを作成して、テキストファイルをアップロード
3

作成したリソースを削除する（重要！）

これが最も重要です。使い終わったリソースは必ず削除しましょう。放置すると予期しない課金が発生する可能性があります。

削除のポイント：
・ストレージは中身のファイルも削除する
・仮想マシンは「停止」ではなく「削除」する（停止でも一部課金される）
・削除後、数時間してから料金ページで課金がないか確認

ハンズオンチュートリアルを活用しよう

各クラウドプロバイダーは、初心者向けの手順書（ハンズオンチュートリアル）を公開しています。これに従って操作すれば、安全に学習できます。

レベル1: 初心者（このガイドで学習済み）

クラウドとは何か、どんな種類があるか、メリット・デメリットは何かを理解する段階です。

学習内容：
・クラウドの基本概念（IaaS、PaaS、SaaS）
・主要プロバイダーの特徴（AWS、Azure、GCP）
・セキュリティ、コスト管理の基礎
👉 このガイドで完了しています！

2

レベル2: 基礎（数週間〜数ヶ月）

実際に管理コンソールを操作し、主要サービスを触って理解を深める段階です。

学習内容：
・仮想マシンの作成・削除
・ストレージへのファイルアップロード
・データベースの作成
・ネットワーク設定の基礎
推奨リソース： 公式ハンズオンラボ、オンライン学習プラットフォーム

3

レベル3: 実践（数ヶ月〜半年）

複数のサービスを組み合わせて、実際に動くシステムを構築する段階です。

学習内容：
・Webアプリケーションのデプロイ
・データベースと連携したシステム構築
・自動スケーリングの設定
・監視・ログ収集の設定
推奨リソース： 個人プロジェクト、入門資格（AWS CCP、AZ-900等）

4

レベル4: 専門分野（継続的）

得意分野を見つけて、専門性を深める段階です。

専門分野の例：
・インフラエンジニア： ネットワーク、セキュリティ設計
・DevOpsエンジニア： CI/CD、Infrastructure as Code
・データエンジニア： データパイプライン、AI/ML
・ソリューションアーキテクト： システム設計、コスト最適化

8.4 役立つ学習リソースの紹介

クラウド学習を進めるために役立つリソースを紹介します。すべて無料または低コストで利用できます。

公式トレーニング（無料）

AWS Skill Builder

AWSの公式学習プラットフォーム。動画、ハンズオンラボ、クイズなど豊富なコンテンツ。

explore.skillbuilder.aws

Microsoft Learn

Azureの公式学習サイト。学習パスに沿って体系的に学習できます。

learn.microsoft.com

Google Cloud Skills Boost

GCPの公式学習プラットフォーム。実際のGCP環境でハンズオンを体験できます。

cloudskillsboost.google

オンライン学習プラットフォーム

Udemy

クラウド関連の動画コースが豊富。セール時は1,500円程度で購入可能。日本語コースも多数。

Coursera

大学や企業が提供する高品質なコース。一部無料で視聴可能。

ドットインストール

日本語の動画学習サイト。クラウドの基礎を短い動画で学習できます。

ハンズオンラボ（実践演習）

AWS Hands-On Tutorials

AWSの公式ハンズオンチュートリアル。ステップバイステップで操作を学べます。

aws.amazon.com/jp/getting-started/hands-on/

Qwiklabs（Google Cloud）

時間制限付きの実践環境でGCPを学習。料金を気にせず試せます。

8.5 クラウド関連の資格紹介

クラウドの知識を証明する資格は、就職・転職やキャリアアップに役立ちます。各プロバイダーの入門資格から紹介します。

AWS認定資格の体系

AWS Certified Cloud Practitioner（入門）

対象：クラウドの基礎知識を証明したい初学者
難易度：初級
試験時間：90分（100ドル）

AWSの基本サービス、料金体系、セキュリティの基礎を問う試験。このガイドの知識を深めれば合格可能。

AWS Certified Solutions Architect - Associate（中級）

実際にシステムを設計できる能力を証明。クラウドエンジニアの登竜門として人気。

Microsoft Azure認定資格

Microsoft Azure Fundamentals（AZ-900）（入門）

対象：Azureの基礎知識を証明したい初学者
難易度：初級
試験時間：60分（12,500円）

クラウドの基本概念、Azureのコアサービス、セキュリティ、価格設定を問う試験。

Azure Administrator Associate（AZ-104）（中級）

Azure環境の管理・運用能力を証明。実務で役立つ資格。

Google Cloud認定資格

Google Cloud Digital Leader（入門）

対象：クラウドのビジネス活用を理解したい方
難易度：初級
試験時間：90分（99ドル）

技術的な知識だけでなく、クラウドのビジネス価値やデジタルトランスフォーメーションも問われます。

Associate Cloud Engineer（中級）

GCPでアプリケーションをデプロイ・運用できる能力を証明。

資格取得のタイミング

入門資格は、このガイドを学習し、数週間ハンズオンを経験すれば合格可能です。資格勉強を通じて体系的に知識を整理できるため、学習の区切りとしておすすめです。

8.6 今後のキャリアパス

クラウドのスキルを活かせるキャリアパスはさまざまです。自分の興味や適性に合った道を選びましょう。

クラウドエンジニア

クラウドインフラの構築・運用を担当。サーバー、ネットワーク、ストレージの設計と管理が主な仕事。

向いている人：
インフラやシステム構築に興味がある、安定したシステム運用にやりがいを感じる

ソリューションアーキテクト

顧客の課題をクラウドで解決するシステムを設計。技術とビジネスの橋渡し役。

向いている人：
顧客とのコミュニケーションが得意、全体を見渡す設計が好き

DevOpsエンジニア

開発と運用を自動化し、継続的にサービスを改善。CI/CD、Infrastructure as Codeなどを駆使。

向いている人：
自動化が好き、開発と運用の両方に興味がある

データエンジニア

データ基盤の構築・運用、データパイプラインの設計。ビッグデータ、AI/MLを支える。

向いている人：
データ分析に興味がある、大規模データ処理に興味がある

クラウドスキルの市場価値

クラウドエンジニアの需要は年々増加しており、給与水準も高い傾向にあります。今からクラウドを学ぶことは、将来のキャリアにおいて大きなアドバンテージになります。

    ・GCP：300ドルのクレジット（90日間）＋常時無料の枠（e2-micro 1台など）。

  3. Q3. クラウド学習のロードマップで、レベル2（基礎）で学ぶべき内容を3つ挙げてください。

    ・仮想マシンの作成・削除
    ・ストレージへのファイルアップロード

  4. Q4. クラウドの入門資格を3つ挙げてください。

    回答を表示

    ・AWS Certified Cloud Practitioner - AWSの入門資格
    ・Microsoft Azure Fundamentals（AZ-900） - Azureの入門資格

  5. Q5. クラウドスキルを活かせるキャリアパスを2つ挙げて、それぞれの特徴を説明してください。
