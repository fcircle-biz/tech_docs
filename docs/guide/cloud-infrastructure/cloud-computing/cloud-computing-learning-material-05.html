クラウドの基本構成要素 - 仮想マシン、ストレージ、データベース

クラウドを構成する主要な要素を、「オフィスビル」の例えを使って分かりやすく解説します。

この章で学ぶこと

  • クラウドを構成する3つの基本要素の理解
  • コンピューティング（仮想マシン）- 仕事をする人（CPU/メモリ）の役割
  • ストレージ - 書類を保管する倉庫の種類と使い分け
  • データベース - 整理された情報の保管庫の考え方
  • 3要素の連携イメージを理解する

5.1 クラウドを構成する3つの基本要素

クラウドサービスは、大きく分けて3つの基本要素で構成されています。これらは「オフィスビル」に例えると分かりやすいです。

オフィスには「働く人（社員）」「書類を保管する倉庫」「整理された情報を保管するキャビネット」が必要です。
クラウドも同じで、「処理をする仮想マシン」「データを保管するストレージ」「構造化されたデータを管理するデータベース」の3つが基本です。

5.2 コンピューティング（仮想マシン）- 仕事をする人（CPU/メモリ）

コンピューティングとは、実際にプログラムを実行したり、計算処理を行ったりする部分です。クラウドでは仮想マシン（Virtual Machine）として提供されます。

仮想化技術とは - 1台のコンピュータを複数に分ける

「仮想化」とは、1台の物理的なサーバーを複数の仮想的なサーバーに分割する技術です。

マンションの部屋に例えると

1棟のマンション（物理サーバー）を複数の部屋（仮想マシン）に分けて、それぞれ別の住人（利用者）に貸し出すイメージです。
各部屋は独立していて、隣の部屋の影響を受けません。

インスタンスタイプの考え方（小さい部屋から大きな部屋まで）

クラウドでは、様々なスペック（CPUの性能、メモリ容量など）の仮想マシンが用意されています。これをインスタンスタイプと呼びます。

💡 中小企業の業務システム向け

メモリ: 128GB

主要クラウドのコンピューティングサービス

  • AWS: EC2（Elastic Compute Cloud）
  • Azure: Virtual Machines
  • GCP: Compute Engine

5.3 ストレージ - 書類を保管する倉庫

ストレージは、データを保管する場所です。クラウドでは、用途に応じて3種類のストレージが提供されています。

オブジェクトストレージ - 写真や動画の保管庫

オブジェクトストレージは、写真、動画、ドキュメントなど、あらゆるファイルを保管できるストレージです。

代表的なサービス

  • AWS: S3（Simple Storage Service）
  • Azure: Blob Storage
  • GCP: Cloud Storage

Amazon倉庫に例えると

Amazonの倉庫に商品（ファイル）を預けるイメージです。どの商品も同じように保管され、商品ID（URL）で取り出せます。
容量は無制限に近く、ファイル数も制限がありません。

ブロックストレージ - 仮想マシンのハードディスク

ブロックストレージは、仮想マシンに接続して使うハードディスクのようなストレージです。

代表的なサービス

  • AWS: EBS（Elastic Block Store）
  • Azure: Managed Disks
  • GCP: Persistent Disk

ファイルストレージ - 共有フォルダ

ファイルストレージは、複数のサーバーから同時にアクセスできる共有フォルダのようなストレージです。

代表的なサービス

  • AWS: EFS（Elastic File System）
  • Azure: Files
  • GCP: Filestore

ストレージクラスと料金の考え方

オブジェクトストレージでは、アクセス頻度に応じて複数のストレージクラスを選べます。

頻繁にアクセス

保存料金: 高い
取り出し料金: 安い

例: 毎日使う業務ファイル

たまにアクセス

保存料金: 中
取り出し料金: 中

例: 月に数回見る資料

ほぼアクセスなし

保存料金: 安い
取り出し料金: 高い

例: 法律で保管義務がある古い記録

5.4 データベース - 整理された情報の保管庫

データベースは、データを整理して保管し、必要な時に素早く検索・取り出しできる仕組みです。

図書館に例えると

ストレージが「倉庫に本を積み上げる」だとすれば、データベースは「図書館のように本を分類して整理し、検索しやすくする」仕組みです。
「著者名で検索」「出版年で並び替え」といった高度な操作ができます。

リレーショナルデータベース（RDB）

リレーショナルデータベースは、データを表（テーブル）形式で管理する、最も一般的なデータベースです。

例: 顧客テーブル

顧客ID 名前 メール 登録日
001 山田太郎 yamada@example.com 2024-01-15
002 佐藤花子 sato@example.com 2024-02-20
003 鈴木一郎 suzuki@example.com 2024-03-10

代表的なサービス

  • AWS: RDS（MySQL、PostgreSQL、SQL Serverなど）、Aurora
  • Azure: Azure SQL Database、Azure Database for MySQL/PostgreSQL
  • GCP: Cloud SQL、Cloud Spanner

NoSQLデータベース

NoSQLデータベースは、表形式にとらわれない柔軟なデータ構造を持つデータベースです。

高速なアクセスが必要

SNSのタイムライン、リアルタイムゲームのスコアなど、大量のデータに高速アクセスする用途に適しています。

柔軟なデータ構造

商品カタログのように、商品ごとに異なる属性を持つデータを扱う場合に便利です。

代表的なサービス

  • AWS: DynamoDB（キーバリュー型）
  • Azure: Cosmos DB（多モデル対応）
  • GCP: Firestore（ドキュメント型）、Bigtable（ワイドカラム型）

マネージドデータベースのメリット

クラウドのデータベースサービスはマネージド（管理された）サービスです。

自動バックアップ

毎日自動でデータのバックアップを取ってくれます。データが消えても復元できます。

自動アップデート

データベースソフトウェアのバージョンアップやセキュリティパッチ適用を自動で行ってくれます。

簡単なスケーリング

データ量が増えたら、ボタン1つでストレージ容量や性能を増やせます。

高可用性

自動的に複数のデータセンターにデータを複製し、障害時も自動で切り替わります。

コンピューティング、ストレージ、データベースは、それぞれ独立したサービスですが、連携して動作します。

ECサイトの例

例えばECサイトでは、以下のように3要素が連携します：

  • 仮想マシン: Webサーバーとしてアプリケーションを実行
  • オブジェクトストレージ: 商品画像を保存
  • リレーショナルDB: 顧客情報、商品マスター、注文履歴を管理
  • NoSQL DB: ショッピングカート情報、アクセスログを記録

理解度確認クイズ

  1. Q1. クラウドの3つの基本構成要素を挙げてください。

    回答を表示

    1. コンピューティング（仮想マシン） - プログラムを実行する
    2. ストレージ - データを保管する
    3. データベース - データを整理して管理する

  2. Q2. 仮想化技術とは何ですか？簡単に説明してください。

    回答を表示

    1台の物理的なサーバーを複数の仮想的なサーバーに分割する技術です。
    マンション1棟を複数の部屋に分けて、それぞれ別の人に貸し出すイメージです。各仮想マシンは独立して動作し、互いに干渉しません。

  3. Q3. ストレージの3種類（オブジェクト、ブロック、ファイル）の違いを説明してください。

    回答を表示

    ・オブジェクトストレージ: 写真や動画など、あらゆるファイルを保管。容量無制限に近い（S3、Blob Storage、Cloud Storage）
    ・ブロックストレージ: 仮想マシンに接続するハードディスク。高速アクセスが必要なデータ向け（EBS、Managed Disks、Persistent Disk）
    ・ファイルストレージ: 複数のサーバーから同時にアクセスできる共有フォルダ（EFS、Azure Files、Filestore）

  4. Q4. リレーショナルデータベース（RDB）とNoSQLデータベースの違いは何ですか？

    回答を表示

    ・リレーショナルDB: データを表（テーブル）形式で管理。顧客情報や注文データなど、構造が決まったデータ向け。
    ・NoSQL DB: 表形式にとらわれない柔軟な構造。SNSのタイムライン、ログデータなど、高速アクセスや柔軟なデータ構造が必要な場合に適している。

  5. Q5. マネージドデータベースのメリットを3つ以上挙げてください。

    回答を表示

    ・自動バックアップ: 毎日自動でデータをバックアップ
    ・自動アップデート: ソフトウェア更新やセキュリティパッチを自動適用
    ・簡単なスケーリング: 容量や性能を簡単に増減できる
    ・高可用性: 自動で複数拠点にデータを複製し、障害時も自動復旧

第5章のまとめ

クラウドは「コンピューティング（仮想マシン）」「ストレージ」「データベース」の3つの基本要素で構成される

仮想化技術により、1台の物理サーバーを複数の仮想マシンに分割して効率的に利用できる

ストレージには用途に応じて3種類（オブジェクト、ブロック、ファイル）がある

データベースはリレーショナルDBとNoSQLの2種類があり、用途によって使い分ける

マネージドサービスにより、バックアップやアップデートなどの運用作業を自動化できる
