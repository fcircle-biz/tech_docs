クラウドサービスの3つの種類 - IaaS、PaaS、SaaSを理解する

クラウドサービスの3つの提供形態を、「ピザ」の例え（宅配ピザ、冷凍ピザ、手作りピザ）を使って分かりやすく解説します。

この章で学ぶこと

  • サービスモデルとは - なぜ種類を知る必要があるのか
  • IaaS（Infrastructure as a Service）の概念と具体例（AWS EC2、Azure VMなど）
  • PaaS（Platform as a Service）の概念と具体例（Heroku、AWS Elastic Beanstalkなど）
  • SaaS（Software as a Service）の概念と具体例（Gmail、Microsoft 365など）
  • 3つのサービスモデルの比較と選び方のポイント
  • その他のサービスモデル（FaaS、DaaS、BaaSなど）の紹介

2.1 サービスモデルとは - なぜ種類を知る必要があるのか

第1章でクラウドの基本概念を学びましたが、実はクラウドサービスには3つの主要な提供形態（サービスモデル）があります。

この違いを理解することで、「自分たちのビジネスにはどのクラウドが合っているのか」を適切に判断できるようになります。

ピザの例えで理解しよう

クラウドのサービスモデルを理解するには、ピザの例えが分かりやすいです：

1. 自宅で一から作るピザ → オンプレミス（従来の方式）
2. 材料だけ買ってきて自宅で焼く（冷凍ピザ） → IaaS
3. ピザ屋で焼いて持ち帰る → PaaS
4. 宅配ピザで完成品を届けてもらう → SaaS

右に行くほど「クラウド事業者がやってくれること」が増え、利用者の管理負担が減ります。逆に、カスタマイズの自由度は下がります。

2.2 IaaS（Infrastructure as a Service）- 土地と建物を借りる

IaaS（イアース）は、クラウド事業者がインフラ（サーバー、ストレージ、ネットワーク）を提供し、利用者がその上に自由にシステムを構築できるサービスです。

不動産に例えると

土地と建物を借りるイメージです。建物の中をどう使うか（店舗にするか、オフィスにするか）は借り手の自由。でも、建物自体のメンテナンス（耐震補強、外壁塗装など）は大家さん（クラウド事業者）がやってくれます。

IaaSの具体例

AWS EC2

Amazon Web ServicesのElastic Compute Cloud。仮想サーバーを数分で起動できます。

💡 世界中のデータセンターから選択可能

Azure VM

MicrosoftのAzure Virtual Machines。Windows環境との相性が抜群です。

💡 Active Directoryとの統合が簡単

GCE

Google Cloud PlatformのCompute Engine。Googleのネットワークを活用できます。

💡 コスト最適化機能が充実

IaaSが向いている用途

既存システムのクラウド移行

オンプレミスで動いているシステムを、そのままクラウドに持っていきたい場合

完全にカスタマイズしたい環境

OSレベルから細かく設定を変更したい、特殊なミドルウェアを使いたい場合

大規模なシステム

数百〜数千台のサーバーが必要な大規模システムを構築する場合

注意点

IaaSは自由度が高い反面、OSの管理、セキュリティパッチ適用、バックアップ設定など、すべて自分でやる必要があります。運用の知識と手間が必要です。

2.3 PaaS（Platform as a Service）- 店舗を借りて商売する

PaaS（パース）は、アプリケーション開発・実行環境を提供するサービスです。OS、ミドルウェア、開発ツールなどが最初から用意されています。

レストランに例えると

厨房設備が整った店舗を借りるイメージです。ガスコンロ、冷蔵庫、調理器具は全部揃っているので、あなたは料理（アプリケーション開発）に集中できます。設備のメンテナンスは大家さん（クラウド事業者）がやってくれます。

PaaSの具体例

Heroku

開発者に人気のPaaS。コードをアップロードするだけで、アプリが動きます。

💡 git pushでデプロイ完了

AWS EB

AWSのPaaSサービス。Java、PHP、Python、Rubyなど多言語対応。

💡 自動スケーリング機能内蔵

GAE

Google App Engine。Googleのインフラでアプリを実行できます。

💡 トラフィックに応じて自動拡張

PaaSが向いている用途

Webアプリケーション開発

開発に集中したい。インフラ管理に時間を使いたくない

スタートアップ・小規模チーム

インフラエンジニアがいない。すぐにサービスを立ち上げたい

プロトタイプ・MVP開発

アイデアを素早く形にして検証したい

PaaSのメリット

OS、ミドルウェア、セキュリティパッチ、バックアップなどをクラウド事業者が管理してくれるので、開発者はコードを書くことだけに集中できます。インフラの知識が少なくても、本格的なアプリが作れます。

2.4 SaaS（Software as a Service）- 完成品サービスを利用する

SaaS（サース）は、完成したソフトウェアをインターネット経由で提供するサービスです。ユーザーは何もインストールせずに、すぐに使い始められます。

宅配ピザに例えると

宅配ピザのイメージです。材料を買う必要も、調理する必要もありません。電話（またはアプリ）で注文すれば、完成したピザが届きます。あとは食べるだけ。

SaaSの具体例

Gmail

Googleのメールサービス。アカウントを作成すれば、すぐに使えます。

💡 サーバー設定不要、無料で15GB

Microsoft 365

Word、Excel、PowerPointをブラウザで利用。ソフトのインストール不要。

💡 複数人での同時編集可能

Salesforce

世界No.1のCRM（顧客管理）システム。営業支援機能が充実。

💡 カスタマイズ・拡張機能も豊富

Slack

ビジネスチャットツール。チームコミュニケーションを効率化します。

💡 他サービスとの連携が豊富

Zoom

オンライン会議サービス。アカウント作成後、すぐに会議を開催できます。

💡 録画機能、画面共有も標準装備

Dropbox

ファイル保管・共有サービス。複数デバイスで自動同期されます。

💡 リンク共有で簡単ファイル送信

SaaSが向いている用途

標準的な業務ツール

メール、チャット、オフィスソフトなど、多くの人が使う一般的なツール

すぐに使い始めたい

開発やカスタマイズは不要。アカウント作成後、即利用開始

運用負担をゼロにしたい

アップデート、バックアップ、障害対応などをすべて事業者に任せたい

SaaSのメリット

ユーザーは「使う」ことだけに専念できます。インフラ、OS、ミドルウェア、アプリケーション、すべての管理をクラウド事業者が行います。IT知識がなくても、ビジネスツールを活用できます。

2.5 3つのサービスモデルの比較と選び方

IaaS、PaaS、SaaSの違いを表で整理してみましょう。

項目 IaaS PaaS SaaS
提供されるもの サーバー、ストレージ、ネットワーク 開発・実行環境（OS、ミドルウェア含む） 完成したソフトウェア
利用者がやること OS、ミドルウェア、アプリの管理 アプリケーション開発・管理 ログインして使うだけ
カスタマイズ性 ⭐⭐⭐ 高い ⭐⭐ 中程度 ⭐ 低い
運用負荷 ⭐⭐⭐ 高い ⭐⭐ 中程度 ⭐ 低い（ほぼゼロ）
必要な技術知識 インフラ、OS、ネットワーク プログラミング 不要（一般ユーザー向け）
代表例 AWS EC2、Azure VM Heroku、GAE Gmail、Salesforce

実際は組み合わせて使うことが多い

多くの企業では、IaaS、PaaS、SaaSを組み合わせて使っています。
例：
・メールはGmail（SaaS）
・Webアプリ開発はHeroku（PaaS）
・データベースサーバーはAWS EC2（IaaS）
このように、用途に応じて最適なサービスモデルを選択することが重要です。

2.6 その他のサービスモデル（FaaS、DaaS、BaaSなど）

IaaS、PaaS、SaaSが3大サービスモデルですが、最近では他にも様々な「〇〇aaS」が登場しています。代表的なものを紹介します。

FaaS（Function as a Service）

関数（プログラムの一部分）だけを実行する環境。サーバーの管理は一切不要で、実行した時間だけ課金されます。
サーバーレスとも呼ばれます。

例：AWS Lambda、Azure Functions、Google Cloud Functions
用途：画像処理、データ変換、APIのバックエンドなど

DaaS（Desktop as a Service）

仮想デスクトップをクラウドで提供。どのデバイスからでも同じデスクトップ環境にアクセスできます。

例：Amazon WorkSpaces、Windows 365
用途：リモートワーク環境、セキュアなデスクトップ利用

BaaS（Backend as a Service）

モバイルアプリやWebアプリのバックエンド機能（ユーザー認証、データベース、プッシュ通知など）を提供。

例：Firebase、AWS Amplify、Supabase
用途：モバイルアプリ開発、Webアプリのバックエンド構築

STaaS（Storage as a Service）

ストレージ（データ保管場所）だけをクラウドで提供。容量を自由に増減できます。

例：AWS S3、Google Cloud Storage、Dropbox
用途：ファイルバックアップ、画像・動画の保管

「〇〇aaS」の共通点

すべてに共通するのは、「as a Service（サービスとして提供）」という考え方です。
自分で所有・管理せず、必要な機能だけをサービスとして借りる。これがクラウドの基本コンセプトです。

理解度確認クイズ

  1. Q1. IaaS、PaaS、SaaSを、運用負荷が低い順に並べてください。

    回答を表示

    SaaS → PaaS → IaaS（運用負荷が低い順）

    ・SaaS：運用負荷ほぼゼロ（すべて事業者が管理）
    ・PaaS：アプリケーションの管理のみ
    ・IaaS：OS、ミドルウェア、アプリすべて自分で管理

  2. Q2. ピザの例えで、「冷凍ピザを買ってきて自宅で焼く」はどのサービスモデルに該当しますか？

    回答を表示

    IaaSです。

    材料（インフラ）は用意されているが、焼く（設定・運用）のは自分でやる必要があります。
    一方、宅配ピザはSaaS、ピザ屋で焼いて持ち帰るはPaaSに該当します。

  3. Q3. 以下のサービスをIaaS、PaaS、SaaSに分類してください：AWS EC2、Gmail、Heroku

    回答を表示

    ・AWS EC2 → IaaS（仮想サーバーを提供）
    ・Gmail → SaaS（完成したメールソフトウェア）
    ・Heroku → PaaS（アプリ開発・実行環境を提供）

  4. Q4. 「すぐにWebアプリを作りたいが、インフラ管理には時間をかけたくない」という場合、どのサービスモデルが適していますか？

    回答を表示

    PaaSが最適です。

    PaaSは開発環境が整っており、インフラやOS、ミドルウェアの管理をクラウド事業者が行ってくれます。開発者はアプリケーションのコードを書くことだけに集中できます。
    例：Heroku、AWS Elastic Beanstalk、Google App Engine

  5. Q5. FaaS（Function as a Service）の特徴を説明してください。

    回答を表示

    FaaSは、関数（プログラムの一部）だけを実行する環境を提供するサービスです。

    特徴：
    ・サーバーの管理が一切不要（サーバーレス）
    ・実行した時間だけ課金される（使わなければ料金ゼロ）
    ・必要な時だけ瞬時に起動

    例：AWS Lambda、Azure Functions、Google Cloud Functions

第2章のまとめ

クラウドサービスには3つの主要な提供形態がある：IaaS、PaaS、SaaS

IaaS：インフラを提供（例：AWS EC2、Azure VM）。カスタマイズ性が高いが運用負荷も高い

PaaS：開発環境を提供（例：Heroku、GAE）。開発に集中でき、運用負荷は中程度

SaaS：完成品ソフトウェアを提供（例：Gmail、Salesforce）。すぐ使えて運用負荷ほぼゼロ

用途に応じて最適なサービスモデルを選ぶことが重要。実際は組み合わせて使うことが多い

FaaS、DaaS、BaaSなど、他にも様々な「〇〇aaS」が存在し、進化を続けている
