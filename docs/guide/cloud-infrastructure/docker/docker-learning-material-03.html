第3章

はじめてのDockerコマンド - Hello Worldを動かそう

ターミナルの基本操作から始めて、最初のDockerコマンドを実行します。コンテナが動く仕組みを体験しましょう。

この章で学ぶこと

  • ターミナル（コマンドライン）の基本的な使い方を理解する
  • 最初のDockerコマンド「docker run hello-world」を実行する
  • 基本的なDockerコマンド（run, ps, stop, rm）を習得する

3.1 ターミナル（コマンドライン）の基本

Dockerを使うには、ターミナルというツールを使ってコマンドを入力します。初めての方でも安心してください。ここではターミナルの基本的な使い方を学びます。

ターミナルとは？

ターミナルは、文字でコンピュータに命令を出すためのツールです。マウスを使わず、キーボードで「コマンド」と呼ばれる命令文を入力します。プログラミングや開発作業では、ターミナルを使うことが一般的です。

ターミナルの開き方

Windows

「PowerShell」または「コマンドプロンプト」を使用します。

  • Windowsキー + R → 「powershell」と入力 → Enter
  • またはスタートメニューから「PowerShell」を検索

macOS

「ターミナル」アプリを使用します。

  • Command + Space → 「ターミナル」と入力 → Enter
  • または「アプリケーション」→「ユーティリティ」→「ターミナル」

3.2 はじめてのDockerコマンド - Hello World

それでは、実際に最初のDockerコマンドを実行してみましょう。プログラミングの世界では、最初に「Hello World」を表示するのが伝統です。

実習: Hello Worldコンテナを起動しよう

ターミナルを開いて、以下のコマンドを入力してください。

ターミナル 
docker run hello-world

このコマンドを実行すると、Dockerが自動的に「hello-world」というイメージをダウンロードし、コンテナを起動します。

実行結果の解説

コマンドを実行すると、以下のようなメッセージが表示されます。一行ずつ意味を見ていきましょう。

出力例 
Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
c1ec31eb5944: Pull complete
Digest: sha256:53f1bbee2f52c39e41682ee1d388285290c5c8a76cc92b42687eecf38e0af3f0
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
This message shows that your installation appears to be working correctly.

メッセージの意味

  • 1
    Unable to find image 'hello-world:latest' locally
    ローカル（あなたのPC）に「hello-world」イメージが見つからないという意味です。
  • 2
    Pulling from library/hello-world
    Docker Hubから「hello-world」イメージをダウンロードしています。
  • 3
    Pull complete
    ダウンロードが完了しました。
  • 4
    Hello from Docker!
    コンテナが起動し、アプリケーションが実行された結果のメッセージです。成功！

内部で何が起きているのか？

sequenceDiagram
    participant User as あなた
    participant Docker as Docker Engine
    participant Local as ローカルPC
    participant Hub as Docker Hub

    User->>Docker: docker run hello-world
    Docker->>Local: イメージを探す
    Local-->>Docker: 見つからない
    Docker->>Hub: hello-worldをダウンロード
    Hub-->>Docker: イメージを送信
    Docker->>Docker: コンテナを起動
    Docker->>User: Hello from Docker!

3.3 基本的なDockerコマンド

コンテナを操作する基本的なコマンドを学びましょう。これらのコマンドは今後頻繁に使用します。

docker ps - コンテナの一覧を表示

現在実行中のコンテナを確認できます。

ターミナル 
# 実行中のコンテナを表示
docker ps

# すべてのコンテナを表示（停止中も含む）
docker ps -a

ヒント

「hello-world」コンテナは実行後すぐに終了するため、「docker ps」では表示されません。「docker ps -a」で確認してください。

docker run - コンテナを起動

イメージからコンテナを作成して起動します。最も基本的なコマンドです。

基本構文 
docker run [オプション] イメージ名

よく使うオプション

  • -d バックグラウンドで実行（デタッチモード）
  • --name コンテナに名前を付ける
  • -p ポートを公開（次章で詳しく学習）
  • --rm コンテナ終了時に自動削除

docker stop - コンテナを停止

実行中のコンテナを停止します。

ターミナル 
# コンテナIDまたはコンテナ名で停止
docker stop [コンテナIDまたは名前]

# 例: コンテナID "abc123" を停止
docker stop abc123

docker rm - コンテナを削除

停止したコンテナを削除します。

ターミナル 
# コンテナを削除
docker rm [コンテナIDまたは名前]

# 実行中のコンテナを強制削除
docker rm -f [コンテナIDまたは名前]

3.4 実習: コマンドを組み合わせて使ってみよう

ハンズオン練習

以下の手順で、学んだコマンドを実際に使ってみましょう。

ステップ1: コンテナを起動

ターミナル 
docker run hello-world

ステップ2: コンテナ一覧を確認

ターミナル 
# すべてのコンテナ（停止中も含む）を表示
docker ps -a

コンテナIDと状態（STATUS）を確認してください。

ステップ3: コンテナを削除

ターミナル 
# コンテナIDの最初の数文字を使用（例: abc123）
docker rm [コンテナID]

ステップ2で確認したコンテナIDを使って削除してください。

ステップ4: 削除を確認

ターミナル 
docker ps -a

先ほど削除したコンテナが一覧に表示されないことを確認してください。

動作確認

以下の項目を確認できれば成功です！

  • 「Hello from Docker!」というメッセージが表示された
  • 「docker ps -a」でコンテナ一覧を確認できた
  • コンテナを削除できた

重要ポイント

  • 「docker run」でイメージがローカルになければ自動的にダウンロードされる
  • 「docker ps」で実行中のコンテナ、「docker ps -a」ですべてのコンテナを表示
  • コンテナIDは最初の数文字だけでも識別可能

理解度確認クイズ

  1. Q1. 「docker run hello-world」を実行したとき、最初に何が起きますか？

    回答を表示

    答え: Dockerはまずローカル（あなたのPC）にhello-worldイメージがあるか探します。見つからない場合、Docker Hubから自動的にダウンロード（pull）します。その後、イメージからコンテナを作成して起動します。

  2. Q2. 「docker ps」と「docker ps -a」の違いは何ですか？

    回答を表示

    答え: 「docker ps」は現在実行中のコンテナのみを表示します。「docker ps -a」は、停止中のコンテナも含めてすべてのコンテナを表示します。hello-worldコンテナは実行後すぐに終了するため、docker ps -aで確認する必要があります。

  3. Q3. コンテナを削除する前に必ず停止する必要がありますか？

    回答を表示

    答え: 通常は停止してから削除しますが、「docker rm -f」コマンドを使えば、実行中のコンテナを強制的に停止して削除できます。ただし、hello-worldコンテナのように既に終了しているコンテナは、停止コマンドなしで直接削除できます。

第3章のまとめ

  • ターミナルは文字でコンピュータに命令を出すツールで、Dockerを使うために必要
  • 「docker run」でイメージからコンテナを起動できる（イメージがなければ自動ダウンロード）
  • 「docker ps」で実行中のコンテナ、「docker ps -a」ですべてのコンテナを確認
  • 「docker stop」でコンテナを停止、「docker rm」で削除できる
