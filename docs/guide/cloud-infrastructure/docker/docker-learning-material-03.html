第3章

はじめてのDockerコマンド - Hello Worldを動かそう

ターミナル（コマンドライン）の基本的な使い方から始め、最初のDockerコマンドを実行します。出力の意味を一行ずつ丁寧に解説し、基本的なDockerコマンドを実践します。

この章で学ぶこと

  • ターミナル（コマンドライン）の基本的な使い方を理解する
  • docker run hello-worldを実行し、出力の意味を理解する
  • 基本的なDockerコマンド（ps, images, rm, rmi）を使えるようになる
  • コンテナが動く仕組みを視覚的に理解する

3.1 ターミナル（コマンドライン）の基本

Dockerコマンドを実行する前に、ターミナル（コマンドライン）の基本的な使い方を確認しましょう。ターミナルは、キーボードでコマンドを入力してコンピュータに指示を出すツールです。

ターミナルとは？

ターミナルは、コンピュータとの対話ツールです。マウスでアイコンをクリックする代わりに、文字で指示を出します。

例：

  • マウス操作： フォルダのアイコンをダブルクリックして開く
  • ターミナル： cd フォルダ名 と入力して移動する

ターミナルの開き方

Windows

  1. スタートメニューをクリック
  2. 「PowerShell」または「ターミナル」と入力
  3. 「Windows PowerShell」または「ターミナル」をクリック

macOS

  1. Cmd + スペース キーを押す
  2. 「ターミナル」と入力
  3. 「ターミナル.app」を選択

コマンドの入力方法

ターミナルでは、以下の手順でコマンドを実行します：

  1. コマンドを入力する（例：docker --version）
  2. Enterキーを押す
  3. 結果が表示される

ヒント： コマンドをコピー＆ペーストすることもできます。間違いを防ぐため、長いコマンドはコピーして貼り付けることをお勧めします。

3.2 Hello Worldコンテナを実行しよう

それでは、最初のDockerコマンドを実行してみましょう。docker run hello-worldは、Dockerの動作確認に使われる有名なコマンドです。

実習 3-1: Hello Worldコンテナの実行

ターミナルを開いて、以下のコマンドを実行してみましょう。

実行コマンド

Terminal / PowerShell 
docker run hello-world

期待される結果

以下のようなメッセージが表示されます：

3.3 出力メッセージを理解しよう

Hello Worldコンテナが表示したメッセージには、Dockerの動作の仕組みが説明されています。一行ずつ見ていきましょう。

メッセージの詳細解説

1 The Docker client contacted the Docker daemon.

意味： あなたが入力したコマンド（docker run）が、Dockerを管理するプログラム（Docker daemon、デーモン）に伝わりました。

Docker clientは「あなたが入力したコマンド」、Docker daemonは「実際にコンテナを動かすプログラム」です。

2 The Docker daemon pulled the "hello-world" image from the Docker Hub.

意味： Docker daemonが、Docker Hub（インターネット上のコンテナイメージ置き場）から「hello-world」というイメージをダウンロードしました。

イメージは「コンテナの設計図」のようなものです。アプリを動かすために必要な全てが詰まっています。

3 The Docker daemon created a new container from that image.

意味： ダウンロードしたイメージから、新しいコンテナを作成しました。

設計図（イメージ）から実際の実行環境（コンテナ）を作り出すイメージです。

4 The Docker daemon streamed that output to the Docker client.

意味： コンテナが実行した結果（このメッセージ）が、あなたのターミナルに表示されました。

コンテナが出力した内容を、ターミナルで見ることができます。

Dockerの動作フロー

flowchart LR A[あなた
docker run hello-world] -->|1. コマンド送信| B[Docker daemon] B -->|2. イメージ取得| C[Docker Hub] C -->|hello-worldイメージ| B B -->|3. コンテナ作成| D[Hello Worldコンテナ] D -->|4. 出力| A style A fill:#e0f2fe,stroke:#0ea5e9 style B fill:#bae6fd,stroke:#0284c7 style C fill:#7dd3fc,stroke:#0369a1 style D fill:#38bdf8,stroke:#0284c7

このフローが数秒で自動的に行われます。これがDockerの基本的な動作です。

3.4 基本的なDockerコマンドを覚えよう

Hello Worldを実行したので、次はコンテナやイメージを確認・削除するコマンドを学びましょう。

コマンド 説明
docker ps 実行中のコンテナ一覧を表示
docker ps -a 全てのコンテナ（停止中も含む）を表示
docker images ダウンロード済みのイメージ一覧を表示
docker rm [コンテナID] 指定したコンテナを削除
docker rmi [イメージ名] 指定したイメージを削除

実習 3-2: コンテナとイメージの確認

先ほど実行したHello Worldコンテナを確認してみましょう。

1. 全てのコンテナを表示

Terminal / PowerShell 
docker ps -a

期待される結果

CONTAINER ID   IMAGE         COMMAND    CREATED          STATUS                      PORTS     NAMES
a1b2c3d4e5f6   hello-world   "/hello"   2 minutes ago    Exited (0) 2 minutes ago              clever_name

STATUSがExited (0)になっています。これは「正常に終了した」という意味です。Hello Worldコンテナは、メッセージを表示したら自動的に終了します。

2. イメージ一覧を表示

Terminal / PowerShell 
docker images

期待される結果

REPOSITORY    TAG       IMAGE ID       CREATED        SIZE
hello-world   latest    d2c94e258dcb   7 months ago   13.3kB

hello-worldイメージがダウンロードされていることが確認できます。サイズはわずか13.3KB（非常に小さい）です。

IDの短縮形について

コンテナIDやイメージIDは、実際には長い文字列（例：a1b2c3d4e5f6a7b8c9d0e1f2...）ですが、docker psやdocker imagesでは最初の12文字だけが表示されます。

コマンドで指定する際は、最初の数文字だけでも識別できます。例えば、a1b2c3d4e5f6の代わりにa1b2だけでも大丈夫です。

3.5 コンテナとイメージを削除しよう

不要になったコンテナやイメージは削除できます。削除方法を学びましょう。

削除の順番に注意

イメージを削除する前に、そのイメージから作られたコンテナを先に削除する必要があります。

正しい順番：

  1. コンテナを削除（docker rm）
  2. イメージを削除（docker rmi）

実習 3-3: コンテナとイメージの削除

1. コンテナIDを確認

まず、削除したいコンテナのIDを確認します。

Terminal / PowerShell 
docker ps -a

表示されたCONTAINER IDをメモします（例：a1b2c3d4e5f6）

2. コンテナを削除

確認したIDを使って、コンテナを削除します。

Terminal / PowerShell 
# 例: コンテナIDがa1b2c3d4e5f6の場合
docker rm a1b2

注： a1b2の部分は、あなたの環境で表示されたコンテナIDの最初の数文字に置き換えてください。

3. イメージを削除

コンテナを削除したら、イメージも削除できます。

Terminal / PowerShell 
docker rmi hello-world

4. 削除の確認

Terminal / PowerShell 
# コンテナが削除されたか確認
docker ps -a

# イメージが削除されたか確認
docker images

hello-worldが表示されなければ、正常に削除されています。

3.6 もう一度Hello Worldを実行してみよう

イメージを削除したので、もう一度docker run hello-worldを実行してみましょう。今度は何が起こるか観察してください。

動作確認: 2回目の実行

Terminal / PowerShell 
docker run hello-world

今回表示される追加メッセージ

Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
c1ec31eb5944: Pull complete
Digest: sha256:...
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
...

1回目との違い： イメージを削除したので、再度Docker Hubからダウンロード（Pull）されます。この追加メッセージが表示されるのが確認できます。

Dockerのキャッシュの仕組み

Dockerは賢く動作します：

  • 1回目： イメージがないので、Docker Hubからダウンロード
  • 2回目（削除前）： ローカルにイメージがあるので、すぐに実行
  • 3回目（削除後）： イメージを削除したので、再度ダウンロード

このように、必要なときだけダウンロードするので、効率的です。

理解度確認クイズ

  1. Q1. docker run hello-worldを実行したとき、Dockerは何をしましたか？4つのステップを順番に説明してください。

    回答を表示

    1. Docker clientがDocker daemonにコマンドを送信
    2. Docker daemonがDocker Hubから「hello-world」イメージを取得（Pull）
    3. そのイメージから新しいコンテナを作成
    4. コンテナの出力をDocker client（ターミナル）に表示

  2. Q2. docker psとdocker ps -aの違いは何ですか？

    回答を表示

    docker psは実行中のコンテナのみを表示します。docker ps -aは、実行中・停止中を含む全てのコンテナを表示します。-aは「all（全て）」の意味です。Hello Worldコンテナはメッセージを表示したらすぐ終了するため、docker psでは表示されず、docker ps -aで確認できます。

  3. Q3. コンテナとイメージを削除する正しい順番と、そのコマンドを答えてください。

    回答を表示

    正しい順番：
    1. まずコンテナを削除: docker rm [コンテナID]
    2. 次にイメージを削除: docker rmi [イメージ名]

    この順番が必要な理由は、イメージから作られたコンテナが残っている状態では、イメージを削除できないからです。設計図（イメージ）を消す前に、それから作られた実体（コンテナ）を先に削除する必要があります。

第3章のまとめ

  • ターミナルはコンピュータと対話するツールで、コマンドを入力してEnterキーで実行する
  • docker run hello-worldで最初のコンテナを実行し、Dockerの動作フローを体験した
  • docker ps -aで全てのコンテナ、docker imagesでイメージ一覧を確認できる
  • コンテナ削除（docker rm）→イメージ削除（docker rmi）の順番で削除する
  • Dockerはイメージをキャッシュし、必要なときだけダウンロードする
