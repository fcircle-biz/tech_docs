第3章

はじめてのDockerコマンド - Hello Worldを動かそう

ターミナル（コマンドライン）の基本的な使い方から始め、最初のDockerコマンドを実行します。docker run hello-worldを実行して、その出力の意味を一行ずつ丁寧に解説し、基本的なDockerコマンドを実践します。

この章で学ぶこと

  • ターミナル（コマンドライン）の基本的な使い方を理解する
  • docker run hello-worldを実行し、出力の意味を理解する
  • コンテナが動く仕組みを視覚的に理解する
  • 基本的なDockerコマンド（run, ps, stop, rm）を実践する

3.1 ターミナル（コマンドライン）とは

Dockerを使うには、ターミナル（コマンドライン）を使います。ターミナルとは、文字だけでパソコンに指示を出すためのツールです。

マウスでアイコンをクリックする代わりに、キーボードで「こうしてください」とコマンド（命令）を打ち込みます。最初は慣れないかもしれませんが、慣れればマウスより速く、正確に操作できます。

ターミナルの開き方

Windows

  1. 1. スタートメニューを開く
  2. 2. 「PowerShell」または「コマンドプロンプト」と検索
  3. 3. アプリをクリックして起動
ヒント: Rancher Desktopを使っている場合は、PowerShellがおすすめです

macOS

  1. 1. Finderを開く
  2. 2. 「アプリケーション」→「ユーティリティ」
  3. 3. 「ターミナル」をダブルクリック
ヒント: Cmd + Spaceで検索して「Terminal」と入力しても起動できます

ターミナルの画面の見方

ターミナルを開くと、以下のような画面が表示されます：

# Windowsの場合（PowerShell）
PS C:\Users\YourName>

# macOS/Linuxの場合
username@computer ~ %

各要素の意味：

  • PS (Windows) / username@computer (Mac): コンピュータ情報
  • C:\Users\YourName / ~: 現在いるフォルダ（ディレクトリ）
  • > / % / $: コマンド入力待ち状態（プロンプト）

このプロンプトの後に、コマンドを入力します

3.2 はじめてのDockerコマンド - Hello World

それでは、実際に最初のDockerコマンドを実行してみましょう。プログラミングの世界では、新しい技術を学ぶとき、まず「Hello World」と表示するのが伝統です。

実習: Hello Worldコンテナを実行

ターミナルに以下のコマンドを入力して、Enterキーを押してください：

docker run hello-world

入力のポイント

  • 半角英数字で正確に入力してください
  • 大文字小文字を区別します（すべて小文字）
  • スペースは1つだけ（dockerとrunの間、runとhello-worldの間）

実行結果の出力を理解しよう

コマンドを実行すると、以下のような文字が表示されます。一行ずつ見ていきましょう： 

Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
c1ec31eb5944: Pull complete
Digest: sha256:1234abcd...
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
 3. The Docker daemon created a new container from that image...
 4. The Docker daemon streamed that output to the Docker client...

1行目 Unable to find image 'hello-world:latest' locally

意味： あなたのPCに「hello-world」というイメージ（設計図）が見つからなかったという報告です。初めて実行するときは必ずこのメッセージが出ます。

2〜4行目 latest: Pulling from library/hello-world...

意味： Dockerが自動的にインターネット（Docker Hub）から「hello-world」イメージをダウンロードしています。「Pull complete」と表示されたらダウンロード完了です。

中盤 Hello from Docker!

意味： コンテナが起動し、実際に実行された結果がこの「Hello from Docker!」というメッセージです。これが表示されたら成功です！

後半 To generate this message, Docker took the following steps...

意味： Dockerが内部でどのような処理を行ったかの説明です。初心者の方はまだ完全に理解する必要はありませんが、「4つのステップで動いているんだな」ということだけ覚えておきましょう。

動作確認のポイント

「Hello from Docker!」が表示されたら成功です
エラーが出た場合は、Rancher Desktopが起動しているか確認してください
2回目以降は「Pulling from...」が表示されません（既にダウンロード済みのため）

3.4 基本的なDockerコマンドを使ってみよう

「docker run」以外にも、コンテナを管理するための基本的なコマンドがあります。一つずつ実践してみましょう。

コマンド1: docker ps - 実行中のコンテナを確認

docker ps は、今動いているコンテナの一覧を表示します。 

docker ps

実行結果（例）：

CONTAINER ID   IMAGE     COMMAND   CREATED   STATUS    PORTS     NAMES

何も表示されない（ヘッダー行だけ）場合、現在実行中のコンテナがないという意味です。hello-worldコンテナは実行後すぐに停止するため、通常は表示されません。

コマンド2: docker ps -a - すべてのコンテナを確認

docker ps -a は、停止中のコンテナも含めて、すべてのコンテナを表示します。 

docker ps -a

実行結果（例）：

CONTAINER ID   IMAGE         COMMAND    CREATED         STATUS                     NAMES
abc123def456   hello-world   "/hello"   2 minutes ago   Exited (0) 2 minutes ago   romantic_euler

hello-worldコンテナが「Exited」（終了済み）のステータスで表示されます。CONTAINER IDとNAMESは毎回ランダムに生成されます。

各列の意味：

  • CONTAINER ID: コンテナの識別番号（ユニークなID）
  • IMAGE: このコンテナの元になったイメージ名
  • COMMAND: コンテナ起動時に実行されたコマンド
  • CREATED: コンテナが作成された日時
  • STATUS: コンテナの状態（実行中/停止など）
  • NAMES: コンテナに自動でつけられた名前（指定も可能）

コマンド3: docker stop - コンテナを停止

docker stop は、実行中のコンテナを停止します。 

# 書式
docker stop [CONTAINER IDまたはNAME]

# 例（CONTAINER IDで指定）
docker stop abc123def456

# 例（NAMEで指定）
docker stop romantic_euler
注意: hello-worldコンテナは実行後すぐに自動停止するため、docker stopを使う必要はありません。第5章で実行し続けるWebサーバーコンテナを起動したときに使います。

コマンド4: docker rm - コンテナを削除

docker rm は、停止中のコンテナを削除します。 

# 書式
docker rm [CONTAINER IDまたはNAME]

# 例（CONTAINER IDで指定 - 最初の3文字だけでもOK）
docker rm abc

# 例（NAMEで指定）
docker rm romantic_euler

実際に試してみましょう：

  1. 1. docker ps -a で停止中のhello-worldコンテナを確認
  2. 2. CONTAINER IDの最初の3文字をメモ（例: abc）
  3. 3. docker rm abc でコンテナを削除
  4. 4. もう一度 docker ps -a を実行して、削除されたことを確認
成功: コンテナIDが返されたら削除成功です。PCのディスク容量も少し節約できました。

基本コマンドまとめ

コマンド 説明 例
docker run コンテナを作成して起動 docker run hello-world
docker ps 実行中のコンテナ一覧 docker ps
docker ps -a 全てのコンテナ一覧（停止中含む） docker ps -a
docker stop 実行中のコンテナを停止 docker stop abc123
docker rm 停止中のコンテナを削除 docker rm abc123

3.5 よくあるエラーと対処法

初めてDockerコマンドを実行すると、エラーが出ることがあります。よくあるエラーと対処法を紹介します。

トラブルシューティング

エラー: "Cannot connect to the Docker daemon"

原因: Rancher Desktopが起動していない

対処法: Rancher Desktopを起動してから、もう一度コマンドを実行してください

エラー: "docker: command not found"

原因: Dockerコマンドが見つからない

対処法: Rancher Desktopが正しくインストールされているか確認してください。ターミナルを再起動することで解決することもあります

ネットワークエラー

原因: インターネット接続の問題、またはDocker Hubにアクセスできない

対処法: インターネット接続を確認してください。会社のネットワークではプロキシ設定が必要な場合があります

理解度確認クイズ

  1. Q1. 「docker run hello-world」を実行したとき、最初に何が起こりますか?

    回答を表示

    Dockerはまずあなたのパソコンに「hello-world」イメージがあるか確認します。ない場合は、自動的にDocker Hub（インターネット上のイメージ保管場所）からダウンロード（Pull）します。イメージがダウンロードされたら、そのイメージからコンテナを作成して実行します。

  2. Q2. 「docker ps」と「docker ps -a」の違いは何ですか?

    回答を表示

    「docker ps」は実行中（Running）のコンテナだけを表示します。一方、「docker ps -a」は停止中（Exited）のコンテナも含めて、すべてのコンテナを表示します。hello-worldコンテナは実行後すぐに停止するため、「docker ps」では表示されず、「docker ps -a」で確認できます。

  3. Q3. コンテナを削除するには、どのコマンドを使いますか?また、削除する前に必要な条件は何ですか?

    回答を表示

    「docker rm [CONTAINER IDまたはNAME]」を使います。ただし、削除できるのは停止中のコンテナだけです。実行中のコンテナを削除したい場合は、まず「docker stop」で停止してから「docker rm」で削除する必要があります。

第3章のまとめ

  • ターミナル（コマンドライン）は、文字でパソコンに指示を出すツール
  • 「docker run hello-world」で最初のコンテナを実行できる
  • イメージがない場合は自動でDocker Hubからダウンロードされる
  • 基本コマンド: docker run（起動）、docker ps（一覧）、docker stop（停止）、docker rm（削除）
  • 1つのイメージから何個でもコンテナを作成できる
