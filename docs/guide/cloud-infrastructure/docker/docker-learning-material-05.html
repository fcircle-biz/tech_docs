第5章

Webサーバーを動かしてみよう - Nginxで学ぶポート設定

実際にNginx（エンジンエックス）というWebサーバーをコンテナで起動し、ブラウザからアクセスしてみます。「ポート」の概念を建物の部屋番号の例えで理解し、コンテナのポートとホストマシンのポートの関係を図解で学びます。

この章で学ぶこと

  • Nginxとは何か、Webサーバーの役割を理解する
  • ポートの概念を日常の例えで理解する
  • コンテナのポートマッピング（-pオプション）を習得する
  • 自分のHTMLファイルをNginxで表示する

5.1 Webサーバーとは何か - Nginxの紹介

Webサーバーとは、ブラウザからのリクエスト（要求）を受け取って、HTMLファイルや画像などを返してくれるソフトウェアです。

Nginx（エンジンエックス）は、世界中で広く使われているWebサーバーソフトウェアの一つです。高速で軽量なことが特徴で、多くの人気Webサイトで採用されています。

Webサーバーの役割

sequenceDiagram participant B as ブラウザ participant W as Webサーバー
(Nginx) participant F as HTMLファイル B->>W: ページを見せて! W->>F: ファイルを読み込む F-->>W: HTMLファイル W-->>B: ページを表示

ブラウザからのリクエストに応じて、WebサーバーがHTMLファイルを返します

5.2 「ポート」を理解しよう - 建物の部屋番号の例え

コンピュータの世界で「ポート」とは、建物の部屋番号のようなものです。

建物の部屋番号の例

1
建物 = コンピュータ本体

あなたのパソコンは、一つの大きなマンションだと考えてください

2
部屋番号 = ポート番号

各サービスは、それぞれ異なる部屋番号を持っています

3
訪問者 = 外部からのアクセス

Webブラウザなどは、特定の部屋番号を訪ねてきます

たとえば、Webサーバーは通常80番ポート（HTTP）や443番ポート（HTTPS）を使います。これは「Web用の部屋は80号室ですよ」と決まっているようなものです。

一つのコンピュータ上で、複数のサービスが異なるポート番号で動作します

5.3 Nginxコンテナを起動する

それでは実際に、NginxのWebサーバーをDockerコンテナで起動してみましょう。

実習 5-1: Nginxコンテナの起動

ターミナルで以下のコマンドを実行してください

# Nginxコンテナを起動（ポート80を公開）
docker run -d -p 80:80 --name my-nginx nginx

コマンドの意味

  • -d: バックグラウンドで実行（デタッチモード）
  • -p 80:80: ポートマッピング（後で詳しく解説）
  • --name my-nginx: コンテナに「my-nginx」という名前をつける
  • nginx: 使用するイメージの名前

動作確認

Webブラウザで以下のURLにアクセスしてください

http://localhost

期待される結果

「Welcome to nginx!」というページが表示されれば成功です。Nginxのデフォルトページが表示されています。

5.4 ポートマッピングの仕組み - -p ホスト:コンテナ の意味

先ほど使った-p 80:80というオプションについて、詳しく見ていきましょう。

ポートマッピングとは、「ホストマシン（あなたのパソコン）のポート」と「コンテナ内のポート」を橋渡しすることです。

ポートマッピング: -p 80:80 の仕組み

にアクセス| host[ホストマシン
80番ポート] host -->|マッピング| container[Dockerコンテナ

Webサーバー] style browser fill:#fef3c7,stroke:#f59e0b style host fill:#bae6fd,stroke:#0ea5e9 style container fill:#e0f2fe,stroke:#0ea5e9 style nginx fill:#d1fae5,stroke:#10b981

ホストの80番ポートへのアクセスが、コンテナ内の80番ポートに転送されます

-pオプションの書き方

基本形式: -p ホストのポート:コンテナのポート

  • -p 80:80 - ホストの80番とコンテナの80番をつなぐ
  • -p 8080:80 - ホストの8080番とコンテナの80番をつなぐ
  • -p 3000:80 - ホストの3000番とコンテナの80番をつなぐ

試してみよう: 異なるポート番号でアクセス

既存のコンテナを停止して、別のポート番号でNginxを起動してみましょう

# 既存のコンテナを停止・削除
docker stop my-nginx
docker rm my-nginx

# 今度は8080番ポートで起動
docker run -d -p 8080:80 --name my-nginx nginx

ブラウザでhttp://localhost:8080にアクセスすると、同じNginxページが表示されます。

5.5 自分のHTMLを表示する - ボリュームマウント

Nginxのデフォルトページではなく、自分で作成したHTMLページを表示してみましょう。そのために-vオプションを使います。

実習 5-2: 自分のHTMLページを表示する

手順

  1. 1 自分のパソコンに、HTMLファイル用のフォルダを作成します
  2. 2 そのフォルダ内に、簡単なHTMLファイルを作成します
  3. 3 そのフォルダをコンテナにマウントしてNginxを起動します

1. HTMLファイルの作成

# ホームディレクトリに移動
cd ~

# HTMLファイル用のフォルダを作成
mkdir my-html
cd my-html

# HTMLファイルを作成（エディタで以下の内容を保存）
# ファイル名: index.html

2. index.html の内容

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>私のページ</title>
</head>
<body>
    <h1>こんにちは！Dockerで動いています</h1>
    <p>これは私が作成したHTMLページです。</p>
</body>
</html>

3. コンテナの起動（ボリュームマウント）

# 既存のコンテナがあれば停止・削除
docker stop my-nginx
docker rm my-nginx

# ボリュームマウントしてNginxを起動
docker run -d -p 80:80 \
  -v ~/my-html:/usr/share/nginx/html \
  --name my-nginx nginx

-vオプションで、ホストの~/my-htmlフォルダを、コンテナ内の/usr/share/nginx/htmlにマウント（接続）しています。

ボリュームマウントの仕組み

index.html] nginx[Nginx] end folder -.マウント.-> nginx_folder nginx_folder --> nginx style host fill:#bae6fd,stroke:#0ea5e9 style container fill:#e0f2fe,stroke:#0ea5e9 style nginx fill:#d1fae5,stroke:#10b981

ホストのフォルダがコンテナ内にマウントされ、Nginxがそのファイルを配信します

動作確認

ブラウザでhttp://localhostにアクセスしてください

期待される結果

「こんにちは！Dockerで動いています」というあなたのHTMLページが表示されれば成功です。

ヒント: ファイルをリアルタイムで更新

ホストのindex.htmlを編集して保存すると、コンテナを再起動しなくてもブラウザをリロードするだけで変更が反映されます。これは、ホストとコンテナでファイルが共有されているためです。

理解度確認クイズ

  1. Q1. コマンドdocker run -p 3000:80 nginxを実行した場合、ブラウザでNginxにアクセスするにはどのURLを使いますか？

    回答を表示

    http://localhost:3000です。-p 3000:80は「ホストの3000番ポート」を「コンテナの80番ポート」にマッピングするという意味なので、ブラウザからはホスト側のポート番号（3000）でアクセスします。

  2. Q2. 「ポート」を建物の例えで説明したとき、部屋番号に相当するのは何ですか？

    回答を表示

    ポート番号です。建物（コンピュータ）の中の各部屋（サービス）にアクセスするために、部屋番号（ポート番号）を指定します。例えば、Webサーバーは通常80番や443番という「部屋」で待機しています。

  3. Q3. -v ~/my-html:/usr/share/nginx/htmlというオプションは何をしていますか？

    回答を表示

    ホストマシンの~/my-htmlフォルダを、コンテナ内の/usr/share/nginx/htmlにマウント（接続）しています。これにより、ホストで編集したHTMLファイルが、コンテナ内のNginxで配信されるようになります。ボリュームマウントと呼ばれる機能です。

第5章のまとめ

  • Webサーバー（Nginx）をDockerコンテナで起動できるようになった
  • ポートとは建物の部屋番号のようなもので、サービスごとに異なる番号を使う
  • -pオプションでホストとコンテナのポートをマッピングできる
  • -vオプションでホストのフォルダをコンテナにマウントし、自分のHTMLを配信できる
