Webサーバーを動かしてみよう - Nginxで学ぶポート設定

Nginxコンテナを起動してブラウザからアクセスし、ポートマッピングの仕組みを理解します。

この章で学ぶこと

  • Nginxコンテナを起動してWebサーバーとして動かす
  • ポートマッピングの概念を理解する
  • 自分だけのHTMLページをコンテナで表示する

5.1 Nginxとは - 世界中で使われるWebサーバー

Nginx（エンジンエックス）は、世界中で広く使われているWebサーバーソフトウェアです。軽量で高速、かつ安定しているため、多くのWebサイトで採用されています。

Webサーバーとは？

Webブラウザからのリクエストを受け取り、HTMLファイルや画像などを返す役割を持つソフトウェアです。レストランで例えると、お客さんの注文を受けて料理を提供するウェイターのような存在です。

Nginxの特徴

  • 高速: 多数のリクエストを効率的に処理できます
  • 軽量: メモリ使用量が少なく済みます
  • 多機能: リバースプロキシ、ロードバランサーとしても使えます

5.2 ポートとは - 建物の部屋番号のようなもの

ポートは、コンピュータ上のサービスを識別するための番号です。建物に例えると、IPアドレスが住所、ポート番号が部屋番号のようなものです。

よく使われるポート番号

  • 80: HTTP（Webサイト）
  • 443: HTTPS（セキュアなWebサイト）
  • 3306: MySQL（データベース）
  • 5432: PostgreSQL（データベース）

5.3 Nginxコンテナを起動してみよう

それでは、実際にNginxコンテナを起動してブラウザからアクセスしてみましょう。

実習: Nginxコンテナの起動

以下のコマンドを実行して、Nginxコンテナを起動します。 

# Nginxコンテナを起動（ポート8080でアクセス可能にする）
docker run -d -p 8080:80 --name my-nginx nginx

コマンドの意味：

  • -d: バックグラウンドで実行（デタッチモード）
  • -p 8080:80: ホストの8080ポートをコンテナの80ポートにマッピング
  • --name my-nginx: コンテナに「my-nginx」という名前をつける
  • nginx: 使用するイメージ名

5.4 ブラウザからアクセスしてみよう

動作確認

  1. ステップ1: ブラウザを開く
  2. ステップ2: アドレスバーに以下を入力してアクセス 
    http://localhost:8080
  3. ステップ3: 「Welcome to nginx!」というページが表示されればOK

成功した場合

「Welcome to nginx!」というタイトルと、「If you see this page, the nginx web server is successfully installed and working.」というメッセージが表示されます。

何が起こっているのか？

  1. ブラウザが localhost:8080 にアクセス
  2. ホストマシン（あなたのPC）の8080ポートがリクエストを受け取る
  3. Dockerがそのリクエストをコンテナの80ポートに転送
  4. Nginxコンテナが80ポートでリクエストを受け取り、HTMLを返す
  5. ブラウザにNginxのウェルカムページが表示される

5.5 自分だけのHTMLページを表示しよう

デフォルトのウェルカムページではなく、自分で作成したHTMLファイルを表示してみましょう。

実習: カスタムHTMLの表示

ステップ1: HTMLファイルを作成する 

# ホームディレクトリにhtmlフォルダを作成
mkdir ~/my-html
cd ~/my-html

# index.htmlを作成
echo '<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>私のWebサイト</title>
</head>
<body>
    <h1>Dockerで動くWebサイト！</h1>
    <p>これは自分で作成したHTMLページです。</p>
</body>
</html>' > index.html

ステップ2: 既存のコンテナを停止・削除 

# コンテナを停止
docker stop my-nginx

# コンテナを削除
docker rm my-nginx

ステップ3: HTMLフォルダをマウントしてコンテナを起動 

# カスタムHTMLをマウントして起動
docker run -d -p 8080:80 \
  -v ~/my-html:/usr/share/nginx/html \
  --name my-nginx nginx

-vオプションの意味：

-v ~/my-html:/usr/share/nginx/htmlは、ホストマシンの~/my-htmlフォルダをコンテナの/usr/share/nginx/htmlにマウント（接続）することを意味します。これにより、ホスト側のHTMLファイルがコンテナ内から見えるようになります。

ステップ4: ブラウザで http://localhost:8080 にアクセスして確認

5.6 ポートの競合とトラブルシューティング

よくあるエラー

「Error starting userland proxy: listen tcp4 0.0.0.0:8080: bind: address already in use」

このエラーは、指定したポート（8080）が既に他のプログラムに使われていることを意味します。

解決方法

方法1: 別のポート番号を使う

# ポート9090を使用する例
docker run -d -p 9090:80 --name my-nginx nginx

この場合は http://localhost:9090 でアクセスします。

方法2: 既存のコンテナを停止する

# 実行中のコンテナを確認
docker ps

# 該当コンテナを停止
docker stop my-nginx

5.7 コンテナのログを確認する

コンテナがどのようなリクエストを受け取っているかは、ログで確認できます。 

# コンテナのログを表示
docker logs my-nginx

# リアルタイムでログを表示（Ctrl+Cで終了）
docker logs -f my-nginx

ブラウザでアクセスすると、以下のようなログが表示されます。 

172.17.0.1 - - [30/Dec/2025:10:30:15 +0000] "GET / HTTP/1.1" 200 615
172.17.0.1 - - [30/Dec/2025:10:30:16 +0000] "GET /favicon.ico HTTP/1.1" 404 153

ログの読み方

  • 172.17.0.1: アクセス元のIPアドレス
  • GET /: ルートパスへのGETリクエスト
  • 200: HTTPステータスコード（200は成功）
  • 615: レスポンスのサイズ（バイト）

理解度確認クイズ

  1. Q1. docker run -p 8080:80 nginx というコマンドで、ブラウザからアクセスするURLはどれですか？

    回答を表示

    答え: http://localhost:8080 です。ホスト側のポート8080をコンテナの80ポートにマッピングしているため、ブラウザからはホスト側のポート番号（8080）を指定してアクセスします。

  2. Q2. ポートが既に使用中というエラーが出た場合、どのように対処すればよいですか？

    回答を表示

    答え: 2つの方法があります。(1) 別のポート番号を使用する（例: -p 9090:80）、(2) 既存のコンテナを停止する（docker stop コンテナ名）。一般的には別のポート番号を使用する方が安全です。

  3. Q3. -v ~/my-html:/usr/share/nginx/html というオプションの役割は何ですか？

    回答を表示

    答え: ホストマシンの~/my-htmlフォルダを、コンテナ内の/usr/share/nginx/htmlにマウント（接続）します。これにより、ホスト側で編集したHTMLファイルがコンテナ内から見えるようになり、カスタムコンテンツを表示できます。

第5章のまとめ

  • Nginxは軽量で高速なWebサーバーソフトウェア
  • ポートはサービスを識別するための番号で、建物の部屋番号のようなもの
  • -pオプションでホストのポートとコンテナのポートをマッピングできる
  • -vオプションでホストのフォルダをコンテナにマウントできる
  • docker logsコマンドでコンテナのログを確認できる
