第5章

Webサーバーを動かしてみよう - Nginxで学ぶポート設定

実際にNginx（エンジンエックス）というWebサーバーをコンテナで起動し、ブラウザからアクセスしてみます。「ポート」の概念を建物の部屋番号の例えで理解し、コンテナのポートとホストマシンのポートの関係を図解で学びます。

この章で学ぶこと

  • Webサーバー（Nginx）とは何か、その役割を理解する
  • 「ポート」の概念を建物の部屋番号の例えで理解する
  • コンテナのポートとホストマシンのポートの関係を図解で学ぶ
  • 自分だけのWebページを表示する体験をする

5.1 Webサーバーとは何か - Nginxの紹介

Webサーバーとは、ブラウザからの「このページを見せてください」というリクエストを受け取り、HTMLファイルや画像などを返してくれるソフトウェアです。

Nginx（エンジンエックス）は、世界中で最も人気のあるWebサーバーソフトウェアの一つです。高速で軽量なことが特徴で、YouTubeやNetflixなどの大規模サイトでも使われています。

Webサーバーの役割

sequenceDiagram participant U as ユーザー participant B as Webブラウザ participant W as Webサーバー
（Nginx） participant F as HTMLファイル U->>B: URLを入力 B->>W: ページを見せて! W->>F: ファイルを読み込む F-->>W: HTMLファイル W-->>B: ページを返す B-->>U: ページを表示

ブラウザからのリクエストに応じて、WebサーバーがHTMLファイルを返す仕組み

Nginxの読み方

Nginxは「エンジンエックス」と読みます。「Engine X」を略してNginxと書くようになりました。初心者がよく「エヌジンクス」と読み間違えますが、正しくは「エンジンエックス」です。

5.2 「ポート」を理解しよう - 建物の部屋番号の例え

Dockerを使う上で避けて通れない概念が「ポート」です。難しく聞こえるかもしれませんが、日常の例えで理解すれば簡単です。

建物の部屋番号で理解するポートの概念

1
建物 = コンピュータ本体

あなたのパソコンは、大きなマンションだと考えてください。このマンションには様々なサービスが住んでいます。

2
部屋番号 = ポート番号

各サービス（Webサーバー、データベース、メールサーバーなど）は、それぞれ異なる部屋番号（ポート番号）を持っています。

3
訪問者 = 外部からのアクセス

Webブラウザやアプリは訪問者として、特定の部屋番号（ポート番号）を訪ねてきます。「80番の部屋のWebサーバーさんに会いたいです」という感じです。

たとえば、Webサーバーは伝統的に80番ポート（HTTP）や443番ポート（HTTPS）を使います。データベース（MySQL）は3306番ポート、メールサーバーは25番ポートといった具合に、サービスごとに「定番の部屋番号」が決まっています。

重要ポイント

ポート番号は0〜65535の範囲で指定できますが、0〜1023番は「ウェルノウンポート」と呼ばれ、有名なサービス専用に予約されています。自分で自由に使えるのは1024番以降です。

5.3 Nginxコンテナを起動してみよう

ポートの概念を理解したところで、実際にNginxのWebサーバーをDockerコンテナで起動してみましょう。

実習 5-1: Nginxコンテナの起動

ターミナルを開いて、以下のコマンドを実行してください

# Nginxコンテナを起動（ポート80を公開）
docker run -d -p 80:80 --name my-nginx nginx

コマンドの意味を一つずつ解説

  • docker run - コンテナを起動するコマンド
  • -d - バックグラウンドで実行（デタッチモード）。ターミナルを占有せずに裏で動きます
  • -p 80:80 - ポートマッピング（次のセクションで詳しく解説）
  • --name my-nginx - コンテナに「my-nginx」という覚えやすい名前をつける
  • nginx - 使用するイメージの名前（Docker Hubから自動ダウンロード）

動作確認

Webブラウザを開いて、以下のURLにアクセスしてください

http://localhost

期待される結果

以下のような「Welcome to nginx!」というページが表示されれば成功です。

Welcome to nginx!

If you see this page, the nginx web server is successfully installed and working...

コンテナが起動しているか確認する方法

以下のコマンドで、起動中のコンテナ一覧を確認できます

# 起動中のコンテナを表示
docker ps

「my-nginx」というコンテナが表示され、STATUSが「Up」になっていればOKです。

5.4 ポートマッピングの仕組み - コンテナとホストをつなぐ橋

先ほど使った-p 80:80というオプションについて、詳しく見ていきましょう。これがポートマッピングの設定です。

ポートマッピングとは、「ホストマシン（あなたのパソコン）のポート」と「コンテナ内のポート」を橋渡しすることです。これがないと、コンテナの中で動いているWebサーバーに外から誰もアクセスできません。

ホストの80番ポートへのアクセスが、コンテナ内の80番ポートに転送されます

-pオプションの書き方

基本形式: -p ホストのポート:コンテナのポート

  • -p 80:80

    → ホストの80番とコンテナの80番をつなぐ（同じ番号）

  • -p 8080:80

    → ホストの8080番とコンテナの80番をつなぐ（異なる番号）

  • -p 3000:80

    → ホストの3000番とコンテナの80番をつなぐ

試してみよう: 異なるポート番号でアクセス

既存のコンテナを停止して、別のポート番号でNginxを起動してみましょう

# 既存のコンテナを停止・削除
docker stop my-nginx
docker rm my-nginx

# 今度は8080番ポートで起動
docker run -d -p 8080:80 --name my-nginx nginx

ブラウザでhttp://localhost:8080にアクセスすると、同じNginxページが表示されます。ホスト側のポート番号だけが変わり、コンテナ内では引き続き80番ポートで動いています。

なぜ異なるポート番号を使うのか？

同じホストで複数のWebサーバーを起動したい場合、ホスト側のポート番号を変える必要があります。例えば、プロジェクトAは80番、プロジェクトBは8080番、プロジェクトCは3000番というように使い分けます。

5.5 自分だけのWebページを表示しよう

Nginxのデフォルトページではなく、自分で作成したHTMLページを表示してみましょう。そのためにボリュームマウントという機能を使います。

実習 5-2: 自分のHTMLページを表示する

ステップ1: HTMLファイル用のフォルダを作成

# ホームディレクトリに移動
cd ~

# HTMLファイル用のフォルダを作成
mkdir my-website
cd my-website

ステップ2: HTMLファイルを作成

テキストエディタ（メモ帳など）で以下の内容を保存してください（ファイル名: index.html）

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>私の初めてのWebページ</title>
</head>
<body>
    <h1>🎉 こんにちは！Dockerで動いています</h1>
    <p>これは私が作成したHTMLページです。</p>
    <p>Dockerコンテナの中でNginxが配信しています。</p>
</body>
</html>

ステップ3: ボリュームマウントでNginxを起動

# 既存のコンテナがあれば停止・削除
docker stop my-nginx
docker rm my-nginx

# ボリュームマウントしてNginxを起動
docker run -d -p 80:80 \
  -v ~/my-website:/usr/share/nginx/html \
  --name my-nginx nginx

-vオプションで、ホストの~/my-websiteフォルダを、コンテナ内の/usr/share/nginx/htmlにマウントしています。

動作確認

ブラウザでhttp://localhostにアクセスしてください

期待される結果

「🎉 こんにちは！Dockerで動いています」というあなたのHTMLページが表示されれば成功です。

リアルタイムで更新を確認できる

ホストのindex.htmlを編集して保存すると、コンテナを再起動しなくてもブラウザをリロードするだけで変更が反映されます。これは、ホストとコンテナでファイルが共有されているためです。試しに、HTMLを編集して保存し、ブラウザをリロード（F5キー）してみてください。

5.6 よくあるトラブルと解決方法

Nginxコンテナを起動する際によくあるトラブルと、その解決方法を紹介します。

トラブル1: ポートがすでに使用されている

エラーメッセージ:

Error: bind: address already in use

原因: ホストの80番ポートが、既に別のプログラムで使われています。

解決方法:

# 方法1: 別のポート番号を使う
docker run -d -p 8080:80 --name my-nginx nginx

# 方法2: 使用中のコンテナを停止する
docker ps  # 起動中のコンテナを確認
docker stop <コンテナID>

トラブル2: 同じ名前のコンテナが既に存在する

エラーメッセージ:

Error: Conflict. The container name "/my-nginx" is already in use

原因: 「my-nginx」という名前のコンテナが既に存在します。

解決方法:

# 既存のコンテナを削除
docker rm my-nginx

# または、停止してから削除
docker stop my-nginx
docker rm my-nginx

トラブル3: ブラウザに何も表示されない

原因: いくつかの可能性があります。

確認手順:

  1. 1. コンテナが起動しているか確認: docker ps
  2. 2. 正しいURLでアクセスしているか確認: http://localhost（httpsではない）
  3. 3. ファイアウォールがブロックしていないか確認
  4. 4. コンテナのログを確認: docker logs my-nginx

理解度確認クイズ

  1. Q1. コマンドdocker run -p 3000:80 nginxを実行した場合、ブラウザでNginxにアクセスするにはどのURLを使いますか？

    回答を表示

    http://localhost:3000です。-p 3000:80は「ホストの3000番ポート」を「コンテナの80番ポート」にマッピングするという意味なので、ブラウザからはホスト側のポート番号（3000）でアクセスします。コンテナ内では80番で動いていますが、外からは3000番経由でアクセスすることになります。

  2. Q2. 「ポート」を建物の例えで説明したとき、部屋番号に相当するのは何ですか？また、建物自体は何に相当しますか？

    回答を表示

    部屋番号はポート番号に、建物自体はコンピュータ本体に相当します。建物（コンピュータ）の中の各部屋（サービス）にアクセスするために、部屋番号（ポート番号）を指定します。例えば、Webサーバーは通常80番や443番という「部屋」で待機していて、訪問者（ブラウザ）がそこを訪ねてきます。

  3. Q3. -v ~/my-website:/usr/share/nginx/htmlというオプションは何をしていますか？この機能の名前は何ですか？

    回答を表示

    ホストマシンの~/my-websiteフォルダを、コンテナ内の/usr/share/nginx/htmlにマウント（接続）しています。これをボリュームマウントと呼びます。これにより、ホストで編集したHTMLファイルが、コンテナ内のNginxでリアルタイムに配信されるようになります。ファイルを共有する仕組みです。

第5章のまとめ

  • Webサーバー（Nginx）をDockerコンテナで起動し、ブラウザからアクセスできるようになった
  • ポートとは建物の部屋番号のようなもので、サービスごとに異なる番号を使う
  • -pオプションでホストとコンテナのポートをマッピングし、外部からアクセスできるようにする
  • -vオプションでホストのフォルダをコンテナにマウントし、自分のHTMLを配信できる
  • よくあるトラブル（ポート競合、コンテナ名重複）の解決方法を理解した
