実践プロジェクト - シンプルなWebアプリをコンテナ化しよう

これまで学んだ知識を総動員して、フロントエンド、バックエンド、データベースの3層構成をDockerで構築します。

この章で学ぶこと

  • 3層構成のWebアプリケーションの構造を理解する
  • フロントエンド・バックエンド・データベースをそれぞれコンテナ化する
  • Docker Composeで複数コンテナを連携させて動作させる
  • 開発環境としてのDockerの便利さを体感する

9.1 今回作るアプリケーション - シンプルなTodoアプリ

9.2 プロジェクトの準備 - フォルダ構成を作ろう

まずは、プロジェクトのフォルダ構成を作成します。フロントエンド、バックエンド、データベースの設定を分けて管理します。

実習 9-1: プロジェクトフォルダを作成

フォルダ構成を作成して、各層のファイルを配置できるようにします。

手順

  1. 1 ターミナルで作業用フォルダを作成し、移動します
  2. 2 フロントエンド、バックエンド用のフォルダを作成します
ターミナル 
# プロジェクトフォルダを作成
mkdir todo-app

# 各層のフォルダを作成

9.3 フロントエンドの作成 - Nginxで静的ファイルを配信

まずはフロントエンドを作成します。シンプルなHTML/CSS/JavaScriptでTodoアプリのUIを作り、Nginxコンテナで配信します。

実習 9-2: フロントエンドのファイルを作成

TodoアプリのUIを作成します。

1. frontend/index.html を作成

frontend/index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>シンプルTodoアプリ</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <div class="container">
        <h1>Todo リスト</h1>
        <input type="text" id="todo-input" placeholder="新しいタスクを入力">
        <button onclick="addTodo()">追加</button>
        <ul id="todo-list"></ul>
    </div>
    <script src="app.js"></script>
</body>
</html>

2. frontend/app.js を作成

frontend/app.js 
// APIのベースURL (バックエンドのアドレス)
const API_URL = 'http://localhost:3000';

// ページ読み込み時にTodoリストを取得
window.onload = () => {
    loadTodos();
};

// Todoリストを取得して表示
async function loadTodos() {
    const response = await fetch(`${API_URL}/todos`);
    const todos = await response.json();
    displayTodos(todos);
}

3. frontend/Dockerfile を作成

frontend/Dockerfile 
# Nginxの公式イメージを使用
FROM nginx:alpine

# HTMLファイルをNginxのドキュメントルートにコピー
COPY index.html /usr/share/nginx/html/
COPY style.css /usr/share/nginx/html/
COPY app.js /usr/share/nginx/html/

# ポート80を公開
EXPOSE 80

フロントエンドのポイント

フロントエンドはNginxコンテナで静的ファイル(HTML/CSS/JS)を配信します。JavaScriptからバックエンドAPIにアクセスしてデータを取得・更新します。

9.4 バックエンドの作成 - Node.js APIサーバー

次にバックエンドAPIを作成します。Node.js + ExpressでシンプルなREST APIを作り、PostgreSQLデータベースに接続します。

実習 9-3: バックエンドAPIを作成

Node.js + ExpressでTodoのCRUD APIを作成します。

1. backend/package.json を作成

backend/package.json 
{
  "name": "todo-backend",
  "version": "1.0.0",
  "main": "server.js",
  "dependencies": {
    "express": "^4.18.0",
    "pg": "^8.11.0",
    "cors": "^2.8.5"
  }
}

2. backend/server.js を作成

backend/server.js 
const express = require('express');
const cors = require('cors');
const { Pool } = require('pg');

const app = express();
app.use(cors());
app.use(express.json());

// PostgreSQL接続設定 (環境変数から取得)
const pool = new Pool({
  host: process.env.DB_HOST || 'db',
  database: process.env.DB_NAME || 'todos',
  user: process.env.DB_USER || 'postgres',
  password: process.env.DB_PASS || 'password',
  port: 5432
});

3. backend/Dockerfile を作成

backend/Dockerfile 
# Node.jsの公式イメージを使用
FROM node:18-alpine

# 作業ディレクトリを作成
WORKDIR /app

# package.jsonをコピーして依存関係をインストール
COPY package.json .
RUN npm install

# アプリケーションファイルをコピー
COPY server.js .

# ポート3000を公開
EXPOSE 3000

# サーバー起動
CMD ["node", "server.js"]

9.5 Docker Composeで3層を連携 - すべてを一度に起動

最後に、フロントエンド・バックエンド・データベースの3つのコンテナをDocker Composeで連携させます。

実習 9-4: docker-compose.yml を作成

3層すべてを定義して、一度に起動できるようにします。

docker-compose.yml 
version: '3.8'

services:
  # データベース (PostgreSQL)
  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: todos
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
    volumes:
      - db-data:/var/lib/postgresql/data

  # バックエンド (Node.js API)
  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      DB_HOST: db
      DB_NAME: todos
    depends_on:
      - db

  # フロントエンド (Nginx)
  frontend:
    build: ./frontend
    ports:
      - "8080:80"
    depends_on:
      - backend

volumes:
  db-data:

Docker Composeの重要ポイント

depends_onで起動順序を制御しています。データベースが起動してからバックエンドが起動し、バックエンドが起動してからフロントエンドが起動します。

9.7 コンテナの連携を理解する - ネットワークと通信

Docker Composeは、各コンテナを同じネットワークに配置して相互通信を可能にします。コンテナ名がホスト名として使えるため、簡単に接続できます。

コンテナ名でアクセスできる理由

Docker Composeは、サービス名(frontend, backend, db)を自動的にDNSとして解決します。そのため、コンテナ内から「backend」「db」という名前でアクセスできるのです。

9.8 開発環境としての便利さを体感する

Dockerを使った開発環境の便利さを体感してみましょう。

1コマンドで全環境起動

docker-compose upだけで、データベースからWebサーバーまですべてが起動します。

チームメンバーも同じ環境

docker-compose.ymlを共有すれば、誰でも同じ環境を再現できます。

簡単なクリーンアップ

docker-compose downで全コンテナを停止・削除。PCが汚れません。

複数プロジェクトの切り替え

異なるバージョンのデータベースを使うプロジェクトも、コンテナで簡単に切り替えられます。

開発時によく使うコマンド

ターミナル 
# すべてのコンテナを起動
docker-compose up

# バックグラウンドで起動
docker-compose up -d

# ログを表示
docker-compose logs -f

# 特定サービスのログを表示
docker-compose logs -f backend

# すべて停止・削除
docker-compose down

# ボリュームも含めて削除
docker-compose down -v

トラブルシューティング

フロントエンドからバックエンドに接続できない

原因: APIのURLが間違っているか、バックエンドが起動していない可能性があります。

解決方法: docker-compose psでバックエンドが起動しているか確認。ログをdocker-compose logs backendで確認してエラーをチェック。

データベースに接続できない

原因: データベースの起動が遅く、バックエンドが接続を試みた時にまだ準備ができていない可能性があります。

解決方法: バックエンドに再接続ロジックを追加するか、depends_onに加えてヘルスチェックを設定します。

ポート8080が既に使用されている

原因: 他のアプリケーションがポート8080を使用している可能性があります。

解決方法: docker-compose.ymlのフロントエンドのポートを"8081:80"などに変更し、ブラウザでhttp://localhost:8081にアクセスします。

理解度確認クイズ

  1. Q1. この実践プロジェクトでは何層構成のアプリケーションを作りましたか？それぞれの役割を説明してください。

    回答を表示

    答え: 3層構成です。①フロントエンド層(Nginx): 静的ファイル(HTML/CSS/JS)を配信しユーザーインターフェースを提供。②バックエンド層(Node.js): REST APIを提供しビジネスロジックを実装。③データベース層(PostgreSQL): Todoデータを永続化して保存。

  2. Q2. Docker Composeを使うメリットは何ですか？

    回答を表示

    答え: 複数のコンテナを一つのファイル(docker-compose.yml)で管理できるため、①1コマンドで全環境を起動・停止できる、②依存関係を明確に定義できる(depends_on)、③ネットワーク設定が自動で行われる、④チーム内で同じ環境を簡単に共有できる、などのメリットがあります。

  3. Q3. コンテナ間で通信する際、なぜコンテナ名(backend, dbなど)でアクセスできるのですか？

    回答を表示

    答え: Docker Composeは、同じComposeファイルで定義されたすべてのコンテナを同じDockerネットワークに配置し、サービス名を自動的にDNS名として登録します。そのため、IPアドレスを知らなくても、サービス名(backend, dbなど)をホスト名として使ってアクセスできます。
