第8章

Docker Composeで複数のコンテナを管理しよう

複数のコンテナを連携させる必要性を理解し、Docker Composeの基本を学びます。docker-compose.ymlファイルの書き方を丁寧に解説し、Webアプリケーション + データベースという実践的な構成を一つのコマンドで起動する方法を習得します。

この章で学ぶこと

  • Docker Composeとは何かを理解する
  • なぜ複数のコンテナを管理する必要があるのかを知る
  • docker-compose.ymlファイルの基本的な書き方を学ぶ
  • Webアプリケーション + データベースという実践的な構成を構築する
  • docker-composeコマンドの基本的な使い方を習得する

8.1 複数コンテナの管理が必要な理由

これまでの章では、一度に1つのコンテナを起動・停止してきました。しかし、実際のWebアプリケーションでは、複数のコンテナを同時に動かす必要があります。

実際のWebアプリケーションの構成

React/Vue.js等] backend[バックエンド
Python/Node.js等] database[データベース
MySQL/PostgreSQL等] cache[キャッシュ

1つのWebアプリケーションでも、役割ごとに複数のコンテナが必要になります

これらのコンテナを1つずつdocker runコマンドで起動するのは大変です。さらに、コンテナ同士を連携させる設定も必要になります。

docker runコマンドで複数コンテナを起動する問題点

  • 長いコマンドを何度も実行 - オプションが多く、タイプミスしやすい
  • 起動順序を管理できない - データベースが起動する前にアプリが起動するとエラーになる
  • ネットワーク設定が複雑 - コンテナ同士を接続する設定が面倒
  • チームメンバーに共有しづらい - 手順書に書いてもミスが起こりやすい

8.2 Docker Composeとは何か

Docker Compose（ドッカーコンポーズ）は、複数のコンテナをまとめて管理するためのツールです。1つの設定ファイル（docker-compose.yml）に、必要なコンテナの情報をすべて書いておくことで、たった1つのコマンドですべてのコンテナを起動・停止できます。

Docker Composeの仕組み

設定ファイル] -->|docker compose up| start{Docker Composeが実行} start -->|自動的に| web[Webコンテナ起動] start -->|自動的に| db[DBコンテナ起動] start -->|自動的に| network[ネットワーク作成] start -->|自動的に| volume[ボリューム作成] web -.連携.- db style compose fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style start fill:#e0f2fe,stroke:#0ea5e9 style web fill:#d1fae5,stroke:#10b981 style db fill:#ddd6fe,stroke:#8b5cf6 style network fill:#fecaca,stroke:#ef4444 style volume fill:#fed7aa,stroke:#fb923c

docker-compose.ymlに書いた内容をもとに、Docker Composeがすべてを自動でセットアップしてくれます

日常生活の例え

Docker Composeは「レシピ本」のようなもの

  • レシピ本（docker-compose.yml） - 料理の作り方が書いてある
  • レシピに従う（docker compose up） - レシピ通りに料理を作る
  • 完成（複数のコンテナが起動） - 同じ料理が誰でも作れる

Docker Composeのメリット

簡単起動

1つのコマンドで全コンテナを起動

設定を保存

YMLファイルで設定を管理

チームで共有

ファイルを共有すれば誰でも同じ環境

自動連携

コンテナ間の接続を自動設定

8.3 docker-compose.ymlファイルの基本構造

Docker Composeの設定ファイルはdocker-compose.ymlという名前のYAML形式のファイルです。まずは、シンプルな例で基本構造を理解しましょう。

最もシンプルなdocker-compose.ymlの例 

# バージョンの指定（省略可能）
version: '3.8'

# サービス（コンテナ）の定義
services:
  # サービス名（自由に決められる）
  web:
    # 使用するイメージ
    image: nginx:latest
    # ポートのマッピング（ホスト:コンテナ）
    ports:
      - "8080:80"

重要: YAMLファイルはインデント（字下げ）が重要です。スペース2つでインデントします（タブは使用しません）。

主要な項目の説明

version

Docker Composeファイルの形式のバージョンを指定します。最新版では省略可能です。

services

起動するコンテナ（サービス）を定義します。複数のサービスを定義できます。

image

使用するDockerイメージを指定します（Docker Hubのイメージ名）。

ports

ポートのマッピングを指定します。ホストポート:コンテナポートの形式で書きます。

8.4 実践：Nginx + MySQLの構成を作ってみよう

それでは、実際にWebサーバー（Nginx）とデータベース（MySQL）を組み合わせた構成を作ってみましょう。この構成は、多くのWebアプリケーションで使われる基本的なパターンです。

今回作る構成

flowchart TB user[ブラウザ] -->|http://localhost:8080| nginx[Nginxコンテナ
Webサーバー] nginx -.内部ネットワーク.- mysql[MySQLコンテナ
データベース] mysql -->|データ永続化| volume[ボリューム] style user fill:#f1f5f9,stroke:#64748b style nginx fill:#d1fae5,stroke:#10b981,stroke-width:2px style mysql fill:#ddd6fe,stroke:#8b5cf6,stroke-width:2px style volume fill:#fed7aa,stroke:#fb923c

実習：docker-compose.ymlを作成しよう

ステップ1: 作業フォルダを作成

# 作業フォルダを作成
mkdir docker-compose-practice
cd docker-compose-practice

ステップ2: docker-compose.ymlファイルを作成

version: '3.8'

services:
  # Webサーバー（Nginx）
  web:
    image: nginx:latest
    ports:
      - "8080:80"
    # データベースの起動を待つ
    depends_on:
      - db

  # データベース（MySQL）
  db:
    image: mysql:8.0
    # 環境変数でパスワードなどを設定
    environment:
      MYSQL_ROOT_PASSWORD: mysecretpassword
      MYSQL_DATABASE: myapp
    # データを永続化するボリューム
    volumes:
      - mysql-data:/var/lib/mysql

# ボリュームの定義
volumes:
  mysql-data:

新しく登場した項目の説明

depends_on（起動順序の制御）

データベースが起動してからWebサーバーを起動するように指定します。ただし、完全に起動完了を待つわけではなく、コンテナが開始されればOKです。

environment（環境変数）

コンテナ内で使う環境変数を設定します。MySQLの場合、rootパスワードやデータベース名を指定します。

volumes（ボリューム）

データを永続化するためのボリュームを指定します。ボリューム名:コンテナ内のパスの形式で書きます。

8.5 docker-composeコマンドの基本

docker-compose.ymlファイルを作成したら、docker composeコマンドで操作します。よく使うコマンドを覚えましょう。

実習：コンテナを起動してみよう

1. すべてのコンテナを起動

# バックグラウンドで起動（-d オプション）
docker compose up -d

初回は必要なイメージをダウンロードするため、少し時間がかかります。

2. 起動しているコンテナを確認

# 起動中のコンテナを表示
docker compose ps

webとdbの2つのコンテナが起動していることを確認できます。

3. ログを確認

# すべてのコンテナのログを表示
docker compose logs

# 特定のサービスのログのみ表示
docker compose logs web

動作確認

ブラウザで以下のURLにアクセスして、Nginxのウェルカムページが表示されることを確認しましょう。

http://localhost:8080

「Welcome to nginx!」というページが表示されればOKです！

主要なdocker composeコマンド一覧

コマンド 説明
docker compose up -d すべてのコンテナをバックグラウンドで起動
docker compose down すべてのコンテナを停止して削除
docker compose ps 起動中のコンテナを表示
docker compose logs すべてのコンテナのログを表示
docker compose stop コンテナを停止（削除はしない）
docker compose start 停止したコンテナを再開
docker compose restart コンテナを再起動

重要：upとdownの違い

docker compose up - コンテナを作成して起動します

docker compose down - コンテナを停止して削除します（ボリュームは削除されません）

注意： downすると、コンテナ内の一時的なデータは消えますが、ボリュームに保存したデータは残ります。

8.6 コンテナ間のネットワーク

Docker Composeで起動したコンテナは、自動的に同じネットワーク内に配置されます。これにより、コンテナ同士が簡単に通信できるようになります。

Docker Composeのネットワーク

Nginx] db[dbコンテナ
MySQL] end web -->|サービス名で接続
mysql://db:3306| db style network fill:#e0f2fe,stroke:#0ea5e9,stroke-width:2px style web fill:#d1fae5,stroke:#10b981 style db fill:#ddd6fe,stroke:#8b5cf6

コンテナはサービス名を使ってお互いに通信できます

サービス名で接続できる仕組み

Docker Composeは、各サービス名をホスト名として自動的に解決してくれます。

例：Webコンテナからデータベースに接続する場合

接続先: db:3306

「db」というのは、docker-compose.ymlで定義したサービス名です。IPアドレスを指定する必要はありません。

日常生活の例え

Docker Composeのネットワークは「社内LANシステム」のようなもの

  • 社内LAN - 同じオフィス内のパソコン同士は簡単に通信できる
  • 名前で呼べる - IPアドレスを覚えなくても「田中さんのPC」で接続できる
  • 外部からは見えない - 社外からは直接アクセスできない（セキュリティ）

8.7 実践的なWebアプリケーション構成

最後に、もう少し実践的な例として、Webアプリケーション（PHP） + データベース（MySQL）の構成を作ってみましょう。これは、WordPressのような一般的なWebアプリケーションと同じ構成です。

実習：PHPアプリケーションの環境を作ろう

ステップ1: 新しいフォルダを作成

# 新しいフォルダを作成
mkdir php-mysql-app
cd php-mysql-app

ステップ2: docker-compose.ymlを作成

version: '3.8'

services:
  # PHPアプリケーション
  web:
    image: php:8.2-apache
    ports:
      - "8080:80"
    volumes:
      # 現在のフォルダをコンテナにマウント
      - ./html:/var/www/html
    depends_on:
      - db

  # MySQLデータベース
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootpass
      MYSQL_DATABASE: myapp
      MYSQL_USER: appuser
      MYSQL_PASSWORD: apppass
    volumes:
      - db-data:/var/lib/mysql

volumes:
  db-data:

ステップ3: htmlフォルダを作成してindex.phpを作成

# htmlフォルダを作成
mkdir html

html/index.php ファイルを作成して、以下の内容を記述します：

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>PHPアプリケーション</title>
</head>
<body>
    <h1>Docker Composeで動くPHPアプリケーション</h1>
    <p>PHPバージョン: <?php echo phpversion(); ?></p>
    <p>データベース接続: 準備完了！</p>
</body>
</html>

ステップ4: コンテナを起動

# コンテナを起動
docker compose up -d

動作確認

ブラウザで以下のURLにアクセスして、PHPのバージョンが表示されることを確認しましょう。

http://localhost:8080

PHPバージョンが表示されれば成功です！これで、PHPアプリケーションとMySQLデータベースが連携する開発環境が完成しました。

作成したファイルの構造

php-mysql-app/
├── docker-compose.yml    # Docker Composeの設定ファイル
└── html/                 # Webサーバーのドキュメントルート
    └── index.php         # PHPファイル

htmlフォルダの中のファイルは、Webサーバーから直接見えるようになっています。ファイルを追加・編集すると、すぐにブラウザで確認できます。

8.8 トラブルシューティング

Docker Composeを使っていてよく起こる問題と、その解決方法を紹介します。

よくある問題と解決方法

問題1：ポートが既に使用されている

エラー例： Bind for 0.0.0.0:8080 failed: port is already allocated

原因： 8080ポートが既に他のアプリケーションで使われています。

解決方法：

  • • docker-compose.ymlのportsを変更（例："8081:80"）
  • • または、既存のコンテナを停止する

問題2：YAMLファイルの文法エラー

エラー例： yaml: line X: mapping values are not allowed in this context

原因： インデント（字下げ）が間違っています。

解決方法：

  • • スペース2つでインデント（タブは使わない）
  • • 各項目の階層を正しく揃える
  • • コピーペーストする際はインデントが崩れないように注意

問題3：コンテナが起動しない

解決方法：

# ログを確認してエラーを特定
docker compose logs

# 特定のサービスのログのみ確認
docker compose logs db

ログを読むと、エラーの原因が分かることが多いです。

問題4：設定を変更したのに反映されない

解決方法：

# 一度停止して削除してから再起動
docker compose down
docker compose up -d

docker-compose.ymlを変更したら、コンテナを作り直す必要があります。

お役立ちコマンド

すべてのコンテナ、ネットワーク、ボリュームを削除 
# ボリュームも含めて完全に削除
docker compose down -v

注意: -vオプションを付けると、データベースのデータも削除されます。

イメージを再ビルドしてから起動 
# Dockerfileを変更した場合に使用
docker compose up -d --build

理解度確認クイズ

  1. Q1. Docker Composeを使う最大のメリットは何ですか？

    回答を表示

    複数のコンテナを1つのコマンドで管理できることです。docker-compose.ymlファイルに設定を書いておけば、docker compose upだけですべてのコンテナ、ネットワーク、ボリュームが自動的にセットアップされます。また、チームメンバーと設定ファイルを共有すれば、誰でも同じ環境を簡単に構築できます。

  2. Q2. docker-compose.ymlファイルで、コンテナ間の起動順序を制御するために使う項目は何ですか？

    回答を表示

    depends_onです。例えば、Webアプリケーションがデータベースに依存している場合、depends_on: - dbと書くことで、データベースコンテナが起動してからWebコンテナが起動するようになります。ただし、これは起動開始のタイミングを制御するだけで、完全に起動完了するまで待つわけではありません。

  3. Q3. Docker Composeで起動したコンテナ同士が通信するときに使うホスト名は何ですか？

    回答を表示

    サービス名です。docker-compose.ymlのservicesセクションで定義した名前（例：web、db）がそのままホスト名として使えます。例えば、webコンテナからdbコンテナのMySQLに接続する場合、db:3306という形で接続できます。IPアドレスを調べる必要はありません。

第8章のまとめ

  • Docker Composeは複数のコンテナをまとめて管理するツール
  • docker-compose.ymlファイルに設定を書き、1つのコマンドで起動・停止できる
  • コンテナ同士はサービス名を使って簡単に通信できる
  • ボリュームを使うことで、データを永続化できる
  • チーム開発では、docker-compose.ymlを共有することで全員が同じ環境を構築できる
