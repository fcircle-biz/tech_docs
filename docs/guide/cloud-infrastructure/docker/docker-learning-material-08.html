Docker Composeで複数のコンテナを管理しよう

複数のコンテナを連携させる必要性を理解し、Docker Composeの基本を学びます。docker-compose.ymlファイルの書き方を丁寧に解説し、Webアプリケーション + データベースという実践的な構成を一つのコマンドで起動する方法を習得します。

この章で学ぶこと

  • Docker Composeとは何か、なぜ必要なのかを理解する
  • docker-compose.ymlファイルの基本的な書き方を習得する
  • Webアプリ+データベースの構成を一括管理できるようになる

8.1 Docker Composeとは - 複数コンテナを楽に管理する

これまで学んできたように、docker runコマンドで1つのコンテナを起動できます。しかし、実際のWebアプリケーションでは複数のコンテナが連携して動くことが一般的です。

よくある複数コンテナ構成

  • Webサーバー + データベース: NginxとMySQLを連携させる
  • フロントエンド + バックエンド + DB: React、Node.js、PostgreSQLの3層構成
  • アプリ + キャッシュサーバー + DB: WebアプリとRedis、MySQLを組み合わせる

これらをdocker runコマンドで1つ1つ起動するのは大変です。起動順序、ネットワーク設定、ポート番号、ボリューム設定...覚えるべきオプションがたくさんあります。

Docker Composeの役割

Docker Composeを使えば、複数のコンテナ構成を1つのファイル（docker-compose.yml）にまとめて定義し、1つのコマンドで一括起動・停止できます。レストランの定食セットのように、必要なものがまとめて出てくるイメージです。

8.2 Docker Composeのメリット

簡単な起動・停止

複数コンテナをdocker-compose upで一括起動、docker-compose downで一括停止できます。

設定の可視化

YAMLファイルでシステム構成が一目瞭然。チームメンバーとの共有も簡単です。

自動ネットワーク構築

コンテナ間の通信用ネットワークを自動で作成してくれます。

環境の再現性

誰がどのマシンで実行しても同じ環境が立ち上がります。

8.3 docker-compose.ymlの基本構造

Docker Composeの設定ファイルはYAML形式で記述します。基本的な構造を見てみましょう。

docker-compose.yml（基本構造） 
# Composeファイルのバージョン
version: '3.8'

# サービス（コンテナ）の定義
services:
  # サービス名1
  service_name1:
    image: nginx:latest    # 使用するイメージ
    ports:                 # ポートマッピング
      - "8080:80"

  # サービス名2
  service_name2:
    image: mysql:8.0
    environment:           # 環境変数
      MYSQL_ROOT_PASSWORD: password123

主要な設定項目

項目 説明
version Composeファイルのフォーマットバージョン（'3.8'が推奨）
services 起動するコンテナの定義（複数指定可能）
image 使用するDockerイメージ
build Dockerfileからビルドする場合のパス
ports ホスト:コンテナのポートマッピング
environment コンテナ内の環境変数
volumes データの永続化設定

8.4 実習 - Nginx + MySQL構成を作ってみよう

実習 8-1: Webサーバー + データベース構成

NginxとMySQLの2つのコンテナをDocker Composeで起動してみましょう。

手順

  1. 1 プロジェクト用のディレクトリを作成します
  2. 2 docker-compose.ymlファイルを作成します
  3. 3 Docker Composeで起動・確認します

ステップ1: プロジェクトディレクトリの作成

ターミナル 
# プロジェクトフォルダを作成
mkdir my-webapp
cd my-webapp

ステップ2: docker-compose.ymlを作成

以下の内容でdocker-compose.ymlファイルを作成します。

docker-compose.yml 
version: '3.8'

services:
  # Webサーバー（Nginx）
  web:
    image: nginx:latest
    ports:
      - "8080:80"
    depends_on:
      - db

  # データベース（MySQL）
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: mydb
      MYSQL_USER: user
      MYSQL_PASSWORD: password
    volumes:
      - db_data:/var/lib/mysql

# ボリューム定義
volumes:
  db_data:

各項目の説明

  • web: Nginxコンテナ。ポート8080でアクセス可能にします
  • db: MySQLコンテナ。データベース名やパスワードを環境変数で設定
  • depends_on: webコンテナはdbコンテナの後に起動します
  • volumes: MySQLのデータを永続化するためのボリューム

ステップ3: Docker Composeで起動

ターミナル 
# バックグラウンドで起動
docker-compose up -d

# 起動確認
docker-compose ps

動作確認

ブラウザで http://localhost:8080 にアクセスすると、Nginxのウェルカムページが表示されます。

期待される結果 

NAME                STATUS      PORTS
my-webapp-web-1     running     0.0.0.0:8080->80/tcp
my-webapp-db-1      running     3306/tcp

8.5 Docker Composeの基本コマンド

よく使うコマンド一覧

コマンド 説明
docker-compose up サービスを起動（フォアグラウンド）
docker-compose up -d サービスをバックグラウンドで起動
docker-compose down サービスを停止して削除
docker-compose ps 実行中のサービス一覧を表示
docker-compose logs すべてのサービスのログを表示
docker-compose logs -f ログをリアルタイムで表示
docker-compose restart サービスを再起動
docker-compose exec [サービス名] bash コンテナ内でコマンド実行
コマンド例 
# webサービスのログを表示
docker-compose logs web

# dbサービスに接続してMySQLコマンドを実行
docker-compose exec db mysql -u root -p

# すべてのサービスを停止・削除・ボリュームも削除
docker-compose down -v

8.6 サービス間通信の仕組み

Docker Composeで起動したコンテナは、サービス名を使って相互に通信できます。IPアドレスを気にする必要がありません。 

flowchart LR
    A[Webアプリ
サービス名: web] -->|db:3306で接続| B[(MySQL
サービス名: db)]

    style A fill:#e0f2fe,stroke:#0ea5e9,stroke-width:2px
    style B fill:#d1fae5,stroke:#10b981,stroke-width:2px

サービス名での名前解決

例えば、webコンテナからdbコンテナのMySQLに接続する場合、ホスト名としてdbを指定するだけでOKです。Docker Composeが自動的にIPアドレスを解決してくれます。

接続例（Pythonアプリから） 
import mysql.connector

# サービス名'db'を使って接続
connection = mysql.connector.connect(
    host='db',        # ← docker-compose.ymlのサービス名
    user='user',
    password='password',
    database='mydb'
)

トラブルシューティング

docker-compose: command not found

原因: Docker Composeがインストールされていない、またはパスが通っていません。

解決方法: Rancher DesktopやDocker Desktopをインストールしていれば、docker compose（ハイフンなし）コマンドを使用してください。V2ではdocker composeが推奨されています。

ポートがすでに使用されている

原因: 指定したポート（例: 8080）が別のプロセスで使用中です。

解決方法: docker-compose.ymlのportsセクションで、ホスト側のポート番号を変更してください（例: "8081:80"）。

YAMLのインデントエラー

原因: YAMLファイルのインデント（字下げ）が間違っています。

解決方法: YAMLはスペース2個でインデントするのが一般的です。タブ文字は使わず、スペースで統一してください。エディタのYAML検証機能を使うと便利です。

理解度確認クイズ

  1. Q1. Docker Composeを使う最大のメリットは何ですか？

    回答を表示

    答え: 複数のコンテナを1つのファイル（docker-compose.yml）で定義し、docker-compose upという1つのコマンドで一括起動・停止できることです。また、チーム全体で同じ環境を簡単に共有できます。

  2. Q2. docker-compose.ymlで定義したサービス名は、どのように使われますか？

    回答を表示

    答え: サービス名は、他のコンテナから接続する際のホスト名として使われます。例えばサービス名が「db」なら、他のコンテナから「db:3306」という形でMySQLに接続できます。Docker Composeが自動的に名前解決してくれます。

  3. Q3. docker-compose up -d の「-d」オプションは何を意味しますか？

    回答を表示

    答え: 「-d」はdetached（デタッチド）モードの略で、コンテナをバックグラウンドで起動することを意味します。このオプションを付けると、ターミナルがコンテナのログに占有されず、他のコマンドを実行できます。

第8章のまとめ

  • Docker Composeは複数コンテナを1つのファイルで管理できるツール
  • docker-compose.ymlにサービス定義、ポート、環境変数、ボリュームなどを記述
  • docker-compose upで一括起動、docker-compose downで一括停止
  • サービス名を使ってコンテナ間で通信できる
