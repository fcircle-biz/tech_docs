Dockerイメージを理解しよう - コンテナの設計図

「イメージ」と「コンテナ」の関係を料理のレシピと料理の例えで分かりやすく解説します。Docker Hubから様々なイメージを取得して実行する方法を学び、イメージの管理方法を実践します。

この章で学ぶこと

  • 「イメージ」と「コンテナ」の違いを理解する
  • Docker Hubからイメージを検索・取得する方法を学ぶ
  • イメージの一覧表示、削除などの管理方法を習得する
  • 公式イメージの見分け方と安全な選び方を知る

4.1 「イメージ」と「コンテナ」の違い

第3章では「Hello World」を動かして、初めてコンテナを体験しました。しかし、その際に「イメージ」という言葉が出てきましたね。「イメージ」と「コンテナ」は何が違うのでしょうか？

最も分かりやすい例えは、料理のレシピと実際の料理です。

料理のレシピと料理の関係

レシピ（イメージ）

  • 作り方が書かれた設計図
  • 何度でも使える
  • 読むだけでは食べられない
  • 人に共有できる

料理（コンテナ）

  • レシピから作られた実物
  • 同じレシピから何個でも作れる
  • 実際に使える・食べられる
  • 使い終わったら捨てられる

1つのイメージから、複数のコンテナを作ることができます

重要ポイント

  • イメージは、コンテナを作るための「設計図」や「レシピ」
  • コンテナは、イメージから作られた「実行可能なアプリケーション」
  • 1つのイメージから、何個でもコンテナを作れる

4.2 Docker Hub - イメージの宝庫

Docker Hub（ドッカーハブ）は、世界中の人々が作ったDockerイメージを共有・公開しているイメージの図書館のような場所です。

スマートフォンの「アプリストア」を想像してください。Docker Hubも同じように、必要なソフトウェアのイメージを検索してダウンロード（pull）できます。

Docker Hubでできること

イメージの検索

nginx、MySQL、Pythonなど、欲しいソフトウェアのイメージを探せます

イメージのダウンロード

見つけたイメージを自分のPCにダウンロードして使えます

公式イメージの利用

Docker社や各プロジェクトが公式に提供する安全なイメージが利用できます

自作イメージの公開

自分で作ったイメージを世界中に共有することもできます

ヒント：Docker Hubにアクセスしてみよう

ブラウザで https://hub.docker.com/ にアクセスすると、Docker Hubのウェブサイトが表示されます。

検索ボックスで「nginx」や「mysql」を検索して、どんなイメージがあるか眺めてみましょう。

4.3 イメージを取得してみよう

実際にDocker Hubからイメージをダウンロード（pull）してみましょう。今回は、nginx（エンジンエックス）という人気のWebサーバーのイメージを取得します。

実習：nginxイメージを取得する

ターミナルを開いて、以下のコマンドを実行してください。

コマンド 
# nginxイメージをDocker Hubから取得
docker pull nginx

実行すると、以下のような出力が表示されます：

実行結果 
Using default tag: latest
latest: Pulling from library/nginx
a480a496ba95: Pull complete
f3ace1b8ce45: Pull complete
7c72d17c6984: Pull complete
...
Status: Downloaded newer image for nginx:latest
docker.io/library/nginx:latest

出力の意味を理解しよう

1

Using default tag: latest

タグを指定しなかったので、最新版（latest）を取得します

2

Pulling from library/nginx

Docker Hubの公式ライブラリからnginxイメージをダウンロードしています

3

a480a496ba95: Pull complete

イメージは複数の層（レイヤー）で構成されており、それぞれをダウンロードしています

4

Status: Downloaded newer image

ダウンロードが成功しました！

イメージの「タグ」とは？

イメージには「タグ」というバージョンラベルが付いています。タグを指定することで、特定のバージョンを取得できます。

タグ指定の例：

# 最新版（デフォルト）
docker pull nginx:latest

# バージョン1.25を取得
docker pull nginx:1.25

# Alpine Linuxベースの軽量版
docker pull nginx:alpine

タグを省略すると自動的にlatestが指定されます。

4.4 取得したイメージを確認しよう

自分のPCにどんなイメージがあるか確認してみましょう。

実習：イメージ一覧を表示する

コマンド 
# イメージの一覧を表示
docker images

実行結果の例：

実行結果 
REPOSITORY    TAG       IMAGE ID       CREATED       SIZE
nginx         latest    a6bd71f48f68   2 days ago    187MB
hello-world   latest    9c7a54a9a43c   4 months ago  13.3kB
列名 意味
REPOSITORY イメージ名（例：nginx、hello-world）
TAG バージョンを示すタグ（例：latest、1.25）
IMAGE ID イメージを一意に識別するID（先頭12文字のみ表示）
CREATED イメージが作成された日時
SIZE イメージのディスク使用量

4.5 不要なイメージを削除しよう

イメージは何個でもダウンロードできますが、ディスク容量を消費します。不要になったイメージは削除しましょう。

注意：コンテナを先に削除

イメージを削除する前に、そのイメージから作られたコンテナをすべて削除する必要があります。実行中のコンテナがある場合は、先に停止・削除してください。

実習：イメージを削除する

コマンド 
# 方法1：イメージ名とタグで削除
docker rmi nginx:latest

# 方法2：イメージIDで削除
docker rmi a6bd71f48f68

# 複数のイメージを一度に削除
docker rmi nginx:latest hello-world:latest

rmiは「remove image」の略です。

便利なコマンド：使われていないイメージをまとめて削除

どのコンテナからも使われていない（ダングリング）イメージをまとめて削除できます。

コマンド 
# 使われていないイメージを削除
docker image prune

# 確認なしで削除（-fオプション）
docker image prune -f

4.6 安全なイメージの選び方

Docker Hubには世界中の人が作ったイメージが公開されていますが、中には古いものや信頼性の低いものもあります。安全なイメージを選ぶ方法を学びましょう。

安全なイメージの見分け方

1. 公式イメージ（Official Image）

  • Docker社や各プロジェクトが公式に提供
  • 定期的にセキュリティアップデートされる
  • Docker Hubで「OFFICIAL IMAGE」バッジが表示される
  • 例：nginx、mysql、python、node

2. 検証済みイメージ（Verified Publisher）

  • Docker社が身元を確認した企業・組織が提供
  • 「VERIFIED PUBLISHER」バッジが表示される
  • 例：microsoft、oracle、redhat

3. コミュニティイメージ

  • 一般ユーザーが作成したイメージ
  • 品質やセキュリティはバラバラ
  • 使う場合は、ダウンロード数やスター数を確認

イメージを選ぶときのチェックポイント

  1. 1
    公式イメージか？

    可能な限り公式イメージを使いましょう

  2. 2
    最終更新日は新しいか？

    数年前で止まっているイメージは避けましょう

  3. 3
    ダウンロード数は多いか？

    多くの人に使われているイメージは信頼性が高い傾向があります

  4. 4
    ドキュメントが充実しているか？

    説明や使用例が豊富なイメージを選びましょう

4.7 コマンドラインでイメージを検索しよう

Docker Hubのウェブサイトだけでなく、コマンドラインからもイメージを検索できます。

実習：イメージを検索する

コマンド 
# pythonイメージを検索
docker search python

# 公式イメージのみ検索
docker search --filter is-official=true python

実行結果の例（上位5件のみ）：

実行結果 
NAME          DESCRIPTION                          OFFICIAL
python        Python is an interpreted...          [OK]
pypy          PyPy is a fast, compliant...         [OK]
nikolaik/...  Python with Node.js
bitnami/...   Bitnami Python Docker Image

OFFICIAL列に[OK]と表示されているのが公式イメージです。

理解度確認クイズ

  1. Q1. イメージとコンテナの関係を料理に例えると、イメージは何に相当しますか？

    回答を表示

    レシピ（設計図）に相当します。レシピには料理の作り方が書かれており、何度でも使えますが、レシピ自体は食べられません。同じように、イメージはコンテナを作るための設計図で、1つのイメージから何個でもコンテナを作れます。

  2. Q2. docker pull nginxコマンドは何をしますか？

    回答を表示

    Docker Hubからnginxイメージをダウンロード（pull）します。タグを指定していないので、自動的に最新版（latest）が取得されます。ダウンロードしたイメージは、docker imagesコマンドで確認できます。

  3. Q3. 安全なイメージを選ぶために、最も確実な方法は何ですか？

    回答を表示

    公式イメージ（Official Image）を使うことです。公式イメージはDocker社や各プロジェクトが公式に提供しており、定期的にセキュリティアップデートされます。Docker Hubで「OFFICIAL IMAGE」バッジが付いているイメージを選びましょう。

第4章のまとめ

  • イメージはコンテナを作るための設計図・レシピで、コンテナはイメージから作られた実行可能なアプリケーション
  • Docker Hubは世界中のDockerイメージが公開されている「イメージの図書館」
  • docker pullでイメージを取得、docker imagesで一覧表示、docker rmiで削除できる
  • 公式イメージは安全性が高いので、可能な限り公式イメージを使うべき
