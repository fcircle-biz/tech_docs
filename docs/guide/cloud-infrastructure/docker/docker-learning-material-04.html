Dockerイメージを理解しよう - コンテナの設計図

「イメージ」と「コンテナ」の関係を料理のレシピと料理の例えで分かりやすく解説し、Docker Hubから様々なイメージを取得する方法を学びます。

この章で学ぶこと

  • Dockerイメージとコンテナの関係を料理の例えで理解する
  • Docker Hubから様々なイメージを取得（pull）して実行する方法を学ぶ
  • イメージの一覧表示、削除などの管理方法を実践する
  • 公式イメージの見分け方と安全な選び方を理解する

4.1 イメージとコンテナの関係 - レシピと料理の例え

第1章でも触れましたが、ここではもう少し詳しくイメージとコンテナの関係を見ていきましょう。

料理で例えると...

イメージはレシピ本のレシピです。レシピには材料や作り方が書いてあります。
コンテナはそのレシピに従って実際に作った料理です。
1つのレシピから何度でも同じ料理を作れるように、1つのイメージから何個でもコンテナを起動できます。

イメージの特徴

  • 読み取り専用: 一度作られたイメージは書き換えられない
  • 再利用可能: 何度でも使い回せる
  • 配布可能: Docker Hubで共有できる
  • バージョン管理: タグで複数バージョンを管理

コンテナの特徴

  • 読み書き可能: 実行中にファイルを作成・変更できる
  • 独立稼働: 各コンテナは独立して動く
  • 削除しても安心: 元のイメージには影響なし
  • 停止・再開可能: 必要なときだけ起動できる

4.2 Docker Hub - イメージの巨大な図書館

Docker Hubは、世界中のDockerイメージが集まる公開リポジトリです。レシピ本の図書館のようなもので、誰でも自由にイメージを取得できます。

Docker Hubの特徴

  • 公式イメージ: Nginx、MySQL、Pythonなど、Docker社が認証した安全なイメージ
  • コミュニティイメージ: 世界中の開発者が作成したイメージ
  • プライベートリポジトリ: 自分だけが使えるイメージも保存可能

公式イメージの見分け方

Docker Hubで「nginx」と検索すると、一番上に「DOCKER OFFICIAL IMAGE」というバッジが付いたイメージが表示されます。これが公式イメージです。初心者の方は公式イメージを選ぶと安心です。

4.3 イメージの取得 - docker pull コマンド

Docker Hubからイメージを取得するにはdocker pullコマンドを使います。

実習: Ubuntuイメージを取得しよう

Ubuntu（Linux OS）のイメージを取得してみましょう。

ターミナル 
# Ubuntuイメージを取得
docker pull ubuntu

# 最新版のubuntuイメージがダウンロードされます

実行すると以下のような出力が表示されます：

Using default tag: latest
latest: Pulling from library/ubuntu
01007420e9b0: Pull complete
Digest: sha256:xxxxx...
Status: Downloaded newer image for ubuntu:latest
docker.io/library/ubuntu:latest

解説: latestというタグが自動的に選ばれます。これは「最新版」という意味です。特定のバージョンを指定したい場合はdocker pull ubuntu:22.04のようにタグを指定します。

4.4 イメージの管理 - 一覧表示と削除

取得したイメージを管理する基本的なコマンドを見ていきましょう。

イメージ一覧の表示

あなたのPCにあるすべてのイメージを表示します。

ターミナル 
# イメージ一覧を表示
docker images

# または
docker image ls

出力例：

REPOSITORY    TAG       IMAGE ID       CREATED       SIZE
ubuntu        latest    3b418d7b466a   2 weeks ago   77.8MB
hello-world   latest    feb5d9fea6a5   2 years ago   13.3kB

各列の意味

  • REPOSITORY イメージの名前
  • TAG バージョンやタイプを示すタグ（latest = 最新版）
  • IMAGE ID イメージを識別するユニークなID
  • CREATED イメージが作成された日時
  • SIZE イメージのサイズ

イメージの削除

不要になったイメージを削除してディスク容量を節約します。

ターミナル 
# イメージを削除（名前:タグで指定）
docker rmi ubuntu:latest

# または IMAGE ID で指定
docker rmi 3b418d7b466a

# 強制削除（コンテナが使用中でも削除）
docker rmi -f ubuntu:latest

注意点

そのイメージから起動したコンテナが存在する場合、削除できません。先にコンテナを削除してから（docker rm）、イメージを削除してください。または-fオプションで強制削除できますが、実行中のコンテナがある場合は注意が必要です。

イメージにはタグという仕組みがあり、同じイメージの異なるバージョンを管理できます。

実習: 異なるバージョンのPythonイメージを取得

Pythonの異なるバージョンのイメージを取得してみましょう。

ターミナル 
# Python 3.9 を取得
docker pull python:3.9

# Python 3.11 を取得
docker pull python:3.11

# イメージ一覧を確認
docker images python

出力例：

REPOSITORY   TAG    IMAGE ID       CREATED       SIZE
python       3.9    a5d7930b60cc   3 weeks ago   997MB
python       3.11   fc7a60e86bae   3 weeks ago   1.01GB

ポイント: 同じPythonイメージでも、タグが異なれば別のイメージとして扱われます。これにより、プロジェクトごとに異なるバージョンのPythonを使い分けることができます。

よく使うタグのパターン

  • latest 最新版（ただし、本番環境では避けるべき - バージョンが突然変わるため）
  • 3.11 メジャーバージョン指定（3.11.x系の最新）
  • 3.11.5 完全バージョン指定（最も安全）
  • 3.11-alpine 軽量版（Alpine Linuxベース、サイズが小さい）

4.6 イメージからコンテナを起動してみよう

取得したイメージを使って、実際にコンテナを起動してみましょう。

実習: Ubuntuコンテナを起動してみよう

Ubuntuイメージから対話型のコンテナを起動してみます。

ターミナル 
# Ubuntuコンテナを対話モードで起動
docker run -it ubuntu bash

# コンテナの中に入った状態になります
# プロンプトが「root@xxxxx:/#」のように変わります

コンテナの中で操作してみましょう：

コンテナ内 
# 現在のディレクトリを表示
pwd

# ファイル一覧を表示
ls

# Ubuntuのバージョンを確認
cat /etc/os-release

# コンテナから出る
exit

オプション解説:
-it: 対話モード（interactive + tty）でコンテナに入る
ubuntu: 使用するイメージ名
bash: コンテナ内で実行するコマンド（シェルを起動）

動作確認

以下を確認してみましょう：

  • コンテナ内でlsを実行すると、Ubuntu Linux のディレクトリ構造が見えること
  • exitでコンテナを抜けて、元のターミナルに戻れること
  • docker ps -aで、停止したUbuntuコンテナが表示されること

理解度確認クイズ

  1. Q1. Docker Hubから「nginx」の公式イメージを取得するコマンドは？

    回答を表示

    答え: docker pull nginx です。タグを指定しない場合、自動的にlatestタグが選ばれます。特定のバージョンを指定する場合はdocker pull nginx:1.25のようにタグを付けます。

  2. Q2. イメージとコンテナの違いを説明してください。

    回答を表示

    答え: イメージは「読み取り専用の設計図」で、何度でも使い回せます。コンテナは「イメージから起動した実行環境」で、読み書き可能です。料理に例えると、イメージはレシピ、コンテナはそのレシピから作った料理です。1つのイメージから複数のコンテナを起動できます。

  3. Q3. あなたのPCにあるすべてのDockerイメージを一覧表示するコマンドは？

    回答を表示

    答え: docker images または docker image ls です。どちらのコマンドも同じ結果を表示します。出力には、リポジトリ名、タグ、イメージID、作成日時、サイズが表示されます。

第4章のまとめ

  • イメージは「読み取り専用の設計図」、コンテナは「起動した実行環境」という関係
  • Docker Hubは世界中のDockerイメージが集まる公開リポジトリ
  • docker pullでイメージを取得、docker imagesで一覧表示、docker rmiで削除
  • タグを使って同じイメージの異なるバージョンを管理できる
  • 公式イメージ（DOCKER OFFICIAL IMAGE）を選ぶと安心
