第4章

Dockerイメージを理解しよう - コンテナの設計図

「イメージ」と「コンテナ」の関係を料理のレシピと料理の例えで分かりやすく解説します。Docker Hubから様々なイメージを取得して実行する方法を学びます。

この章で学ぶこと

  • 「イメージ」と「コンテナ」の違いを理解する
  • Docker Hubからイメージを取得（pull）できる
  • イメージの一覧表示と削除ができる
  • 公式イメージの見分け方と安全な選び方を知る

4.1 「イメージ」と「コンテナ」の関係

第3章でコンテナを動かしましたが、実はその前にイメージというものをダウンロードしていました。この「イメージ」と「コンテナ」の違いを理解することが、Dockerを使いこなす第一歩です。

料理のレシピと料理の例え

1
イメージ = 料理のレシピ

「カレーの作り方」が書かれたレシピ本のようなもの。何度でも読めて、同じ料理を再現できます

2
コンテナ = 実際に作った料理

レシピを見て実際に作ったカレー。食べたり、アレンジしたり、捨てたりできます

3
1つのレシピから何度でも作れる

同じレシピから、今日も明日も何度でもカレーを作れます。コンテナも同じです

イメージとコンテナの関係

1つのイメージから、何個でもコンテナを作れます

重要なポイント

  • イメージは読み取り専用 - 変更できません（レシピ本に書き込まない）
  • コンテナは変更できる - 実行中に変更可能ですが、削除すると元に戻ります
  • 1つのイメージから複数のコンテナ - 同じイメージから何個でもコンテナを作れます

4.2 Docker Hub - イメージの図書館

Docker Hubは、世界中の人が作ったDockerイメージを共有・公開するためのオンライン図書館のような場所です。無料で誰でも利用できます。

豊富なイメージ

Python、Node.js、MySQL、Nginxなど、様々なソフトウェアのイメージが公開されています

公式イメージ

Docker社が認定した信頼できる公式イメージが多数あります

無料で利用

ほとんどのイメージは無料でダウンロード（pull）して使用できます

コミュニティ

世界中の開発者がイメージを公開し、共有しています

4.3 イメージの取得（docker pull）

Docker Hubからイメージをダウンロードするには、docker pullコマンドを使います。第3章では自動的にダウンロードされましたが、今度は明示的に取得してみましょう。

実習 4-1: Ubuntuイメージを取得する

UbuntuというLinuxのイメージをDocker Hubから取得してみましょう。

実行コマンド

Terminal / PowerShell 
# Ubuntuの最新版イメージを取得
docker pull ubuntu

# 特定のバージョンを取得する場合（タグを指定）
docker pull ubuntu:22.04

期待される結果

Using default tag: latest
latest: Pulling from library/ubuntu
445a6a12be2b: Pull complete
Digest: sha256:xxx...
Status: Downloaded newer image for ubuntu:latest
docker.io/library/ubuntu:latest

イメージがDocker Hubからダウンロードされ、ローカルに保存されます。

docker pullコマンドの構造

docker pull [イメージ名]:[タグ]
要素 説明 例
イメージ名 取得したいソフトウェアの名前 ubuntu, nginx, python
タグ バージョンを指定（省略時はlatest） 22.04, 3.9, alpine

タグとは？

タグはイメージのバージョンを識別するための目印です。latestは最新版を意味しますが、本番環境では具体的なバージョン番号（例：ubuntu:22.04）を指定することが推奨されます。これにより、予期しない変更を防げます。

4.4 ローカルイメージの確認（docker images）

ダウンロード済みのイメージを確認するには、docker imagesコマンドを使います。

実習 4-2: ローカルイメージの一覧表示

実行コマンド

Terminal / PowerShell 
# イメージ一覧を表示
docker images

# 別の書き方（同じ結果）
docker image ls

期待される結果

REPOSITORY    TAG       IMAGE ID       CREATED        SIZE
ubuntu        latest    08d22c0ceb15   2 weeks ago    77.8MB
ubuntu        22.04     08d22c0ceb15   2 weeks ago    77.8MB
hello-world   latest    9c7a54a9a43c   6 months ago   13.3kB

出力内容の説明

列名 意味
REPOSITORY イメージの名前（リポジトリ名）
TAG イメージのバージョンやバリエーション
IMAGE ID イメージを一意に識別するID（短縮形）
CREATED イメージが作成された日時
SIZE イメージのディスク使用量

4.5 不要なイメージの削除（docker rmi）

使わなくなったイメージは、docker rmiコマンドで削除できます。ディスク容量を節約するために、定期的に整理しましょう。

削除前の注意点

イメージを削除する前に、そのイメージから作られたコンテナがすべて停止・削除されている必要があります。実行中または停止中のコンテナが残っていると、エラーになります。

実習 4-3: イメージを削除する

hello-worldイメージを削除してみましょう。まず、このイメージから作られたコンテナを削除します。

手順

Terminal / PowerShell 
# 1. すべてのコンテナを確認（停止中も含む）
docker ps -a

# 2. hello-worldコンテナを削除（コンテナIDまたは名前を指定）
docker rm [コンテナID]

# 3. イメージを削除
docker rmi hello-world

# イメージIDで削除する場合
docker rmi 9c7a54a9a43c

期待される結果

Untagged: hello-world:latest
Untagged: hello-world@sha256:xxx...
Deleted: sha256:9c7a54a9a43c...
Deleted: sha256:xxx...

イメージ削除のコマンドバリエーション

Terminal / PowerShell 
# イメージ名で削除
docker rmi ubuntu

# イメージ名:タグで削除
docker rmi ubuntu:22.04

# イメージIDで削除（短縮形でもOK）
docker rmi 08d22c0ceb15

# 複数のイメージを一度に削除
docker rmi ubuntu nginx python

# 使われていないイメージをすべて削除
docker image prune

4.6 公式イメージの見分け方と安全な選び方

Docker Hubには誰でもイメージを公開できるため、安全で信頼できるイメージを選ぶことが重要です。

信頼できるイメージの見分け方

1. 公式イメージ（Official Image）

Docker社が認定したイメージ。名前に「/」がなく、シンプルです。
例：ubuntu, python, nginx, mysql

2. 検証済みパブリッシャー（Verified Publisher）

企業や組織が公式に公開しているイメージ。
例：microsoft/dotnet, oracle/mysql

3. コミュニティイメージ

個人や団体が公開しているイメージ。使用前に以下を確認しましょう：

  • ダウンロード数が多いか
  • スター（評価）が多いか
  • 最近更新されているか
  • 説明が詳しく書かれているか

実習 4-4: Docker Hubでイメージを検索する

  1. 1 ブラウザでhttps://hub.docker.com/を開きます
  2. 2 検索ボックスに「nginx」と入力して検索します
  3. 3 検索結果から「nginx」（公式イメージ）を選択します
  4. 4 イメージの詳細ページで、以下を確認します：
    • 「Docker Official Image」のバッジがあるか
    • ダウンロード数（Pulls）
    • タグ一覧（バージョン）
    • 使い方の説明

初心者へのアドバイス

最初は公式イメージを使うことを強くお勧めします。公式イメージは定期的に更新され、セキュリティも保たれています。慣れてきたら、コミュニティイメージも活用できます。

理解度確認クイズ

  1. Q1. イメージとコンテナの関係を、料理で例えた場合、それぞれ何に相当しますか？

    回答を表示

    イメージ = 料理のレシピ、コンテナ = 実際に作った料理です。レシピ（イメージ）は何度でも読めて変更できませんが、実際に作った料理（コンテナ）は食べたり、アレンジしたり、捨てたりできます。1つのレシピから何度でも同じ料理を作れるように、1つのイメージから何個でもコンテナを作れます。

  2. Q2. Docker Hubで公式イメージを見分けるポイントは何ですか？

    回答を表示

    公式イメージの特徴は、「Docker Official Image」のバッジが表示されていることと、イメージ名に「/」が含まれないことです。例えば、ubuntu、python、nginxなどはすべて公式イメージです。一方、ユーザー名/イメージ名の形式（例：myuser/myapp）はコミュニティイメージです。

  3. Q3. イメージを削除しようとしたら「image is being used by stopped container」というエラーが出ました。どうすればいいですか？

    回答を表示

    このエラーは、削除しようとしているイメージから作られた停止中のコンテナが残っていることを示しています。対処方法は以下の通りです：

    1. docker ps -aで停止中を含むすべてのコンテナを確認
    2. 該当するコンテナをdocker rm [コンテナID]で削除
    3. その後、docker rmi [イメージ名]でイメージを削除

    または、docker image pruneコマンドを使って、使われていないイメージをまとめて削除することもできます。

第4章のまとめ

  • イメージは「コンテナの設計図」、コンテナは「実際に動いているアプリケーション」
  • Docker Hubは無料で使えるイメージの図書館
  • docker pullでイメージをダウンロード、docker imagesで一覧確認、docker rmiで削除
  • 公式イメージは信頼性が高く、初心者にもおすすめ
  • タグを使ってバージョンを指定できる（例：python:3.9）
