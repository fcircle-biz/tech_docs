第2章

Rancher Desktopのインストールと初期設定

Rancher Desktopのダウンロードからインストール、初期設定までを詳しく解説します。Windowsの場合はWSL2のセットアップ方法も含めて、つまずきやすいポイントを丁寧にフォローします。

この章で学ぶこと

  • システム要件の確認方法を理解する
  • Windows環境でWSL2をセットアップできる
  • Rancher Desktopをインストールできる
  • Dockerが正常に動作することを確認できる

2.1 Rancher Desktopとは

Rancher Desktopは、WindowsやmacOSでDockerコンテナを実行するための無料のオープンソースツールです。以前はDocker Desktopが主流でしたが、商用利用に制限がかかったため、Rancher Desktopが人気の選択肢になっています。

完全無料

個人利用でも商用利用でも、費用は一切かかりません

簡単導入

インストーラーをダウンロードして実行するだけで使えます

互換性

Dockerコマンドがそのまま使えるため、学習資料がそのまま活用できます

2.2 システム要件の確認

Rancher Desktopをインストールする前に、お使いのパソコンが要件を満たしているか確認しましょう。

項目 最小要件 推奨
OS Windows 10 Home
macOS 10.15 		Windows 11
macOS 12以降
CPU 2コア 4コア以上
メモリ 4GB 8GB以上
ストレージ 10GB 20GB以上

Windows環境の追加要件

Windows環境では、WSL2（Windows Subsystem for Linux 2）のインストールが必要です。次のセクションで詳しく説明します。

2.3 WSL2のセットアップ（Windows環境の方のみ）

WSL2（Windows Subsystem for Linux 2）は、Windows上でLinux環境を動かすための機能です。Rancher DesktopはこのWSL2を使ってDockerコンテナを実行します。

WSL2の役割

Linux環境] B --> C[Rancher Desktop] C --> D[Dockerコンテナ] style A fill:#dbeafe,stroke:#3b82f6 style B fill:#e0f2fe,stroke:#0ea5e9 style C fill:#bae6fd,stroke:#0284c7 style D fill:#7dd3fc,stroke:#0369a1

WindowsとDockerコンテナの間でWSL2が橋渡しをします

実習 2-1: WSL2のインストール

PowerShellまたはコマンドプロンプトを使って、WSL2をインストールします。

手順

  1. 1 スタートメニューを右クリックし、「Windows PowerShell（管理者）」または「ターミナル（管理者）」を選択します
  2. 2 ユーザーアカウント制御（UAC）のダイアログが表示されたら、「はい」をクリックします
  3. 3 以下のコマンドを入力し、Enterキーを押します

実行コマンド

PowerShell（管理者） 
wsl --install
  1. 4 インストールが完了するまで待ちます（5〜10分程度かかります）
  2. 5 インストール完了後、パソコンを再起動します
  3. 6 再起動後、Ubuntuのセットアップ画面が自動的に表示されます。ユーザー名とパスワードを設定してください

期待される結果

インストールが成功すると、以下のようなメッセージが表示されます：

インストールが正常に完了しました。
Ubuntu がインストールされました。
変更を有効にするには、コンピューターを再起動してください。

WSL2のインストール確認

再起動後、WSL2が正しくインストールされたか確認しましょう。

PowerShell 
wsl --list --verbose

期待される結果 

  NAME      STATE           VERSION
* Ubuntu    Running         2

VERSIONの列に2と表示されていれば、WSL2が正しく動作しています。

macOSユーザーの方へ

macOSの場合、WSL2のインストールは不要です。次のセクション「Rancher Desktopのインストール」に進んでください。

2.4 Rancher Desktopのダウンロード

Rancher Desktop公式サイトから、お使いのOSに合ったインストーラーをダウンロードします。

実習 2-2: インストーラーのダウンロード

  1. 1 ブラウザでRancher Desktop公式サイトを開きます：
    https://rancherdesktop.io/
  2. 2 トップページの「Download」ボタンをクリックします
  3. 3 お使いのOSに合ったインストーラーを選択します：
    • Windows: Rancher Desktop Setup X.X.X.exe
    • macOS (Intel): Rancher.Desktop-X.X.X.x86_64.dmg
    • macOS (Apple Silicon): Rancher.Desktop-X.X.X.aarch64.dmg
  4. 4 ダウンロードが完了するまで待ちます（数分程度）

macOSのチップ確認方法

Macをお使いの方で、IntelチップかApple Siliconか分からない場合は、以下の手順で確認できます：

  1. 画面左上のAppleメニュー（リンゴマーク）をクリック
  2. 「このMacについて」を選択
  3. 「チップ」または「プロセッサ」の項目を確認
    • 「Apple M1」「Apple M2」などと表示 → Apple Silicon
    • 「Intel」と表示 → Intel

2.5 Rancher Desktopのインストール

ダウンロードしたインストーラーを使って、Rancher Desktopをインストールします。

Windowsの場合

  1. 1 ダウンロードしたRancher Desktop Setup X.X.X.exeをダブルクリックして実行します
  2. 2 ユーザーアカウント制御（UAC）のダイアログが表示されたら、「はい」をクリックします
  3. 3 セットアップウィザードが起動します。「Next」をクリックして進みます
  4. 4 ライセンス規約を確認し、同意する場合は「I accept the agreement」を選択して「Next」をクリックします
  5. 5 インストール先フォルダを確認し（通常は変更不要）、「Next」をクリックします
  6. 6 「Install」をクリックして、インストールを開始します
  7. 7 インストールが完了したら、「Finish」をクリックします

macOSの場合

  1. 1 ダウンロードしたRancher.Desktop-X.X.X.dmgをダブルクリックして開きます
  2. 2 表示されるウィンドウで、Rancher DesktopのアイコンをApplicationsフォルダにドラッグ＆ドロップします
  3. 3 コピーが完了したら、dmgファイルを閉じます（ウィンドウを閉じるか、デスクトップのディスクアイコンを右クリックして「取り出す」を選択）
  4. 4 FinderでApplicationsフォルダを開き、Rancher Desktopをダブルクリックして起動します
  5. 5 「開発元が未確認のため開けません」と表示された場合は、右クリック（またはControlキーを押しながらクリック）して「開く」を選択します

2.6 Rancher Desktopの初期設定

Rancher Desktopを初めて起動すると、初期設定画面が表示されます。基本的な設定を行いましょう。

実習 2-3: 初期設定の実施

  1. 1
    Welcome画面が表示されます。内容を確認して「OK」をクリックします
  2. 2
    Container Engineの選択画面が表示されます。以下の設定を行います：
    • Container Engine: dockerd (moby)を選択（Dockerコマンドを使うため）
    • Kubernetes: 初心者の方は無効（Disable）にしておきましょう（後で有効にできます）
  3. 3
    設定を確認して「Apply」または「OK」をクリックします
  4. 4
    Rancher Desktopの起動が始まります。初回は数分かかることがあります。画面右下（macOSの場合はメニューバー）のアイコンが緑色になるまで待ちます

Kubernetesを無効にする理由

Kubernetesは高度なコンテナ管理ツールですが、このガイドでは使いません。有効にするとメモリを多く消費するため、初心者の方は無効にしておくことをお勧めします。

2.7 動作確認

Rancher Desktopが正しくインストールされ、Dockerが使える状態になっているか確認します。

実習 2-4: Dockerコマンドの確認

ターミナル（Windowsの場合はPowerShell、macOSの場合はターミナル）を開いて、Dockerが使えるか確認します。

手順

  1. 1 Windows: スタートメニューから「PowerShell」を検索して起動
    macOS: Spotlight検索（Cmd + スペース）で「ターミナル」を検索して起動
  2. 2 以下のコマンドを入力し、Enterキーを押します

実行コマンド

Terminal / PowerShell 
docker --version

期待される結果

Docker version 24.0.6, build ed223bc

※ バージョン番号は異なる場合がありますが、エラーが表示されなければ成功です。

追加の動作確認

さらに詳しい情報を確認するには、以下のコマンドを実行してみましょう。

Terminal / PowerShell 
docker info

期待される結果 

Client:
 Version:    24.0.6
 Context:    rancher-desktop
 ...
Server:
 Containers: 0
  Running: 0
  Paused: 0
  Stopped: 0
 Images: 0
 ...

Docker環境の詳細情報が表示されれば、正常に動作しています。

2.8 よくあるトラブルと対処法

インストール中によくあるトラブルと、その対処法を紹介します。

トラブルシューティング

「docker: command not found」と表示される

原因： Rancher Desktopが起動していない、またはインストールが完了していません。

対処法：

  • Rancher Desktopが起動しているか確認してください（タスクバーまたはメニューバーにアイコンがあり、緑色になっているか）
  • ターミナルを一度閉じて、再度開いてください
  • それでも解決しない場合は、PCを再起動してください

Rancher Desktopが起動しない（Windows）

原因： WSL2が正しくインストールされていない可能性があります。

対処法：

  • PowerShellでwsl --list --verboseを実行し、WSL2がインストールされているか確認してください
  • VERSIONが1の場合は、wsl --set-default-version 2を実行してください
  • WSL2がインストールされていない場合は、セクション2.3に戻ってWSL2をインストールしてください

メモリ不足のエラーが表示される

原因： PCのメモリが不足しています。

対処法：

  • 不要なアプリケーションを閉じてください
  • Rancher Desktopの設定で、割り当てメモリを減らしてください（最低2GB必要）
  • Kubernetesを無効にしてください（設定画面でDisableを選択）

「開発元が未確認」と表示される（macOS）

原因： macOSのセキュリティ機能が動作しています。

対処法：

  • Rancher Desktopのアイコンを右クリック（またはControlキーを押しながらクリック）
  • 「開く」を選択
  • 警告が表示されたら「開く」をクリック

理解度確認クイズ

  1. Q1. Windows環境でRancher Desktopを使用する場合、事前にインストールが必要なものは何ですか？

    回答を表示

    WSL2（Windows Subsystem for Linux 2）です。Rancher DesktopはWSL2を使ってLinux環境を動かし、その上でDockerコンテナを実行します。PowerShellでwsl --installコマンドを実行することで簡単にインストールできます。

  2. Q2. Rancher Desktopの初期設定で、Container Engineとして何を選択すべきですか？その理由も答えてください。

    回答を表示

    dockerd (moby)を選択すべきです。理由は、このガイドで学習するDockerコマンド（docker run、docker psなど）を使用するためです。もう一つの選択肢であるcontainerdは、より低レベルのコンテナランタイムで、Dockerコマンドとは互換性がありません。

  3. Q3. Dockerが正しくインストールされたことを確認するために実行するコマンドは何ですか？

    回答を表示

    docker --versionです。このコマンドを実行すると、インストールされているDockerのバージョン情報が表示されます。「Docker version X.X.X」のような出力が表示されれば、Dockerが正しくインストールされています。エラーメッセージが表示される場合は、Rancher Desktopが起動していないか、インストールに問題がある可能性があります。

第2章のまとめ

  • Rancher Desktopは、無料でDockerを使える便利なツール
  • Windows環境では、WSL2のインストールが必要
  • Container Engineは「dockerd (moby)」を選択する
  • docker --versionコマンドで正常にインストールされたことを確認できる
  • トラブルが発生したら、Rancher Desktopの起動状態とWSL2の設定を確認する
