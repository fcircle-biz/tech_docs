第2章

Rancher Desktopのインストールと初期設定

Rancher Desktopのダウンロードからインストール、初期設定までを詳しく解説します。Windowsの場合はWSL2のセットアップ方法も含めて、確実にDocker環境を構築します。

この章で学ぶこと

  • Rancher Desktopとは何か、なぜ選ぶのかを理解する
  • Windows/macOSへのインストール手順を習得する
  • WSL2のセットアップ（Windows）と動作確認方法を学ぶ

2.1 Rancher Desktopとは - 無料で使えるDocker実行環境

Rancher Desktopは、WindowsやmacOSでDockerコンテナを実行するための完全無料のオープンソースツールです。

なぜRancher Desktopなのか？

従来はDocker Desktopが主流でしたが、2021年以降、大企業での商用利用は有料化されました。Rancher Desktopは完全無料で、Dockerコマンドもそのまま使えるため、学習・個人開発・商用利用すべてにおいて安心して利用できます。

Rancher Desktopの主な特徴

  • 完全無料: 個人・商用問わず無料で使えます
  • Dockerコマンド互換: Docker公式ドキュメントの知識がそのまま使えます
  • 簡単インストール: ダウンロードして実行するだけで使えます
  • Kubernetesも使える: 将来の学習にも対応（本ガイドでは扱いません）

2.2 システム要件 - 必要なハードウェアとソフトウェア

Windows環境

  • OS: Windows 10/11（Home版OK）
  • CPU: 2コア以上
  • メモリ: 8GB以上推奨
  • ストレージ: 20GB以上の空き
  • WSL2のインストールが必須

macOS環境

  • OS: macOS 10.15以降
  • CPU: Intel / Apple Silicon（M1/M2）対応
  • メモリ: 8GB以上推奨
  • ストレージ: 20GB以上の空き
  • 追加セットアップ不要

メモリについて

最低4GBでも動作しますが、8GB以上を強く推奨します。複数のコンテナを同時に起動する場合、メモリ不足でパフォーマンスが低下します。

2.3 Windows環境の準備 - WSL2のインストール

Windows環境でRancher Desktopを使用するには、まずWSL2（Windows Subsystem for Linux 2）をインストールする必要があります。

WSL2とは？

WindowsでLinux環境を動かすための仕組みです。Dockerコンテナは元々Linux技術なので、Windows上でもLinux環境（WSL2）を経由して動作します。

実習: WSL2のインストール

ステップ1: PowerShellを管理者権限で開く

  1. 1. Windowsキーを押して「PowerShell」と入力
  2. 2. 「Windows PowerShell」を右クリック → 「管理者として実行」

ステップ2: 以下のコマンドを実行

PowerShell（管理者） 
# WSL2とデフォルトのLinuxディストリビューション（Ubuntu）をインストール
wsl --install

ステップ3: PCを再起動

インストール完了後、必ず再起動してください。再起動後、自動的にUbuntuのセットアップ画面が表示されます。

ステップ4: Ubuntuのユーザー設定

ユーザー名とパスワードを設定します（Windowsとは別のユーザーです）。
※ パスワード入力時、画面には何も表示されませんが正常に入力されています。

WSL2インストールの確認

PowerShellで以下のコマンドを実行し、バージョン2が表示されればOKです。 

wsl --list --verbose

「VERSION」列に「2」と表示されていることを確認してください。

2.4 Rancher Desktopのダウンロード

実習: Rancher Desktopのダウンロード

ステップ1: 公式サイトにアクセス

ブラウザで以下のURLにアクセスします：
https://rancherdesktop.io/

ステップ2: お使いのOSに合わせてダウンロード

  • Windows: 「Download for Windows」ボタンをクリック
  • macOS: 「Download for macOS」ボタンをクリック（Intel/Apple Siliconは自動判定されます）

ステップ3: ダウンロード完了を待つ

ファイルサイズは約500MB～1GB程度です。ダウンロードが完了するまで数分かかる場合があります。

2.5 Rancher Desktopのインストール

Windowsでのインストール手順

  1. 1
    ダウンロードしたRancher.Desktop.Setup.exeをダブルクリック
  2. 2
    「このアプリがデバイスに変更を加えることを許可しますか？」→ 「はい」をクリック
  3. 3
    インストーラーが起動するので、「Next」をクリックして進める
  4. 4
    インストール先はデフォルトのままで「Install」をクリック
  5. 5
    インストール完了後、「Finish」をクリック

macOSでのインストール手順

  1. 1
    ダウンロードしたRancher.Desktop.dmgをダブルクリック
  2. 2
    表示されたウィンドウで、Rancher Desktopアイコンを「Applications」フォルダにドラッグ&ドロップ
  3. 3
    「アプリケーション」フォルダからRancher Desktopを起動
  4. 4
    初回起動時、「開発元が未確認のため開けません」と表示される場合：
    → システム環境設定 → セキュリティとプライバシー → 「このまま開く」をクリック

2.6 Rancher Desktopの初期設定

初回起動時、いくつかの設定を行います。基本的にはデフォルトのままで問題ありませんが、重要な部分を確認していきましょう。

実習: 初期設定を行う

ステップ1: Welcome画面

初回起動時に表示されるウェルカム画面で「Continue」をクリックします。

ステップ2: Kubernetesの設定（無効化を推奨）

「Enable Kubernetes」のチェックボックスを外すことを推奨します。
※ このガイドではDockerのみを使用し、Kubernetesは使用しません。

ステップ3: Container Engineの選択

「Container Engine」は dockerd (moby) を選択します。
※ これが標準的なDockerエンジンです。

ステップ4: リソース設定

CPUとメモリの割り当ては、デフォルト設定のままでOKです。
（後から変更も可能です）

ステップ5: 設定完了

「Finish」をクリックすると、Rancher Desktopが起動します。
初回起動時は環境の準備に数分かかる場合があります。

2.7 動作確認 - Dockerコマンドを試してみよう

インストールが正しく完了しているか、ターミナル（コマンドプロンプト）でDockerコマンドを実行して確認しましょう。

動作確認: Dockerバージョンの確認

ステップ1: ターミナルを開く

  • Windows: 「PowerShell」または「コマンドプロンプト」を開く
  • macOS: 「ターミナル」アプリを開く（アプリケーション → ユーティリティ → ターミナル）

ステップ2: バージョン確認コマンドを実行

ターミナル 
# Dockerのバージョンを確認
docker --version

期待される出力例： 

Docker version 24.0.7, build afdd53b

ステップ3: Docker Composeの確認

ターミナル 
# Docker Composeのバージョンを確認
docker compose version

両方のコマンドでバージョン番号が表示されれば、インストール成功です！

トラブルシューティング

Q. 「docker: command not found」と表示される

原因: Rancher Desktopが正しく起動していないか、パスが通っていません。
対処: Rancher Desktopアプリを起動し、画面左下のステータスが緑色（Running）になるまで待ってから再度試してください。

Q. 「Cannot connect to the Docker daemon」エラー

原因: Dockerエンジンが起動していません。
対処: Rancher Desktopアプリを起動し、画面左下のステータスが「Running」になるまで待ってください。

Q. （Windows）WSL2のインストールでエラーが出る

対処: Windowsのバージョンが古い可能性があります。Windows Updateで最新版に更新してから再度お試しください。

第2章のまとめ

  • Rancher Desktopは完全無料でDockerを実行できるツール
  • Windows環境ではWSL2のインストールが必須
  • 初期設定ではKubernetesを無効化し、Container Engineは「dockerd (moby)」を選択
  • 「docker --version」コマンドでインストール成功を確認できる
