第2章

Rancher Desktopのインストールと初期設定

Rancher Desktopのダウンロードからインストール、初期設定までを詳しく解説します。Windowsの場合はWSL2のセットアップ方法も含めて、つまずきやすいポイントを丁寧にフォローします。

この章で学ぶこと

  • Rancher Desktopとは何か、なぜDocker Desktopの代わりに使うのかを理解する
  • 自分のPCがDocker環境を動かせるかシステム要件を確認する
  • Windows環境でWSL2をセットアップする方法を習得する
  • Rancher Desktopをインストールし、初期設定を行う
  • インストールが正常に完了したか動作確認を行う

2.1 Rancher Desktopとは

Rancher Desktopは、WindowsやmacOSでDockerコンテナを実行するための無料のオープンソースツールです。

Dockerを使うための公式ツールに「Docker Desktop」がありますが、商用利用には有償ライセンスが必要な場合があります。Rancher Desktopは完全に無料で使えるため、初心者から企業まで幅広く利用されています。

なぜRancher Desktopを選ぶ?

本ガイドでは、ライセンスの心配なく使えるRancher Desktopを採用しています。Dockerコマンドは同じなので、公式ドキュメントの知識もそのまま活用できます。

Windows追加要件

Windowsの場合、WSL2（Windows Subsystem for Linux 2）が必要です。

  • Windows 10 バージョン 2004以降（ビルド19041以降）
  • または Windows 11
  • 仮想化機能が有効（通常は自動で有効）

実習 2-1: Windowsバージョンを確認しよう

自分のWindowsのバージョンを確認してみましょう。

手順

  1. 1 キーボードで Win + R を押す
  2. 2 「ファイル名を指定して実行」に winver と入力してEnter
  3. 3 表示されたウィンドウで「バージョン」と「OSビルド」を確認

確認ポイント

  • ✅ バージョン 2004以降、またはWindows 11
  • ✅ OSビルド 19041以降

2.3 Windowsの準備（WSL2セットアップ）

Windowsユーザーのみ、この手順を実施してください。macOSユーザーは次のセクション（2.4）へ進んでください。

WSL2とは?

WSL2は、Windows上でLinuxを動かすための技術です。DockerはもともとLinux向けの技術なので、Windowsで動かすにはWSL2が必要になります。難しく聞こえますが、数分で簡単にセットアップできます。

実習 2-2: WSL2をインストールしよう

PowerShellを使って、WSL2を簡単にインストールします。

手順

  1. 1 スタートメニューから「PowerShell」を検索
  2. 2 「Windows PowerShell」を右クリック → 「管理者として実行」を選択
  3. 3 以下のコマンドを入力してEnterキーを押す
PowerShell（管理者権限） 
# WSL2をインストール
wsl --install

つまずきポイント

「このコマンドを実行するには管理者として実行する必要があります」と表示された場合は、PowerShellを管理者として実行していません。手順2をもう一度確認してください。

コマンドを実行すると、WSL2と既定のLinuxディストリビューション（Ubuntu）が自動的にインストールされます。数分かかる場合があります。

重要: 再起動が必要

インストール完了後、PCを再起動してください。再起動しないとWSL2が正しく動作しません。

動作確認: WSL2が正しくインストールされたか確認

再起動後、PowerShellで以下のコマンドを実行してください（管理者権限不要）。

PowerShell 
# WSL2のバージョン確認
wsl --list --verbose

期待される結果 

NAME      STATE           VERSION
* Ubuntu    Running         2

VERSIONが「2」になっていればWSL2が正しく動作しています。

2.4 Rancher Desktopをダウンロードしよう

公式サイトから、あなたのOSに合ったRancher Desktopをダウンロードします。

実習 2-3: Rancher Desktopをダウンロードしよう

公式サイトから最新版をダウンロードします。

手順

  1. 1 ブラウザで https://rancherdesktop.io/ を開く
  2. 2 「Download」ボタンをクリック
  3. 3 あなたのOSに合ったインストーラーをダウンロード
    • • Windows: Rancher.Desktop.Setup.X.X.X.exe
    • • macOS (Intel): Rancher.Desktop-X.X.X.x86_64.dmg
    • • macOS (Apple Silicon): Rancher.Desktop-X.X.X.aarch64.dmg

macOSのチップの見分け方

画面左上のAppleメニュー → 「このMacについて」を開き、「チップ」または「プロセッサ」の項目を確認してください。
• M1/M2/M3などと表示されていればApple Silicon
• Intel Coreと表示されていればIntel

実習 2-4: Rancher Desktopをインストール（Windows）

  1. 1 ダウンロードした .exe ファイルをダブルクリック
  2. 2 「このアプリがデバイスに変更を加えることを許可しますか?」→ 「はい」をクリック
  3. 3 インストーラーが起動したら、「Install」をクリック
  4. 4 インストールが完了するまで待つ（3〜5分程度）
  5. 5 「Finish」をクリックして完了

実習 2-5: Rancher Desktopをインストール（macOS）

  1. 1 ダウンロードした .dmg ファイルをダブルクリック
  2. 2 開いたウィンドウで、Rancher Desktopアイコンを「アプリケーション」フォルダにドラッグ
  3. 3 「アプリケーション」フォルダからRancher Desktopを起動
  4. 4 「"Rancher Desktop"はインターネットからダウンロードされたアプリケーションです」→ 「開く」をクリック

つまずきポイント（macOS）

「開発元を検証できないため開けません」と表示された場合:
システム設定 → プライバシーとセキュリティ → 「このまま開く」をクリックしてください。

2.6 初期設定を行おう

Rancher Desktopを初めて起動すると、初期設定画面が表示されます。基本的にはデフォルト設定のままでOKです。

初期設定の各項目

1. コンテナエンジン（Container Engine）

推奨: dockerd（moby）を選択

dockerdは、Dockerの標準的なエンジンです。本ガイドではこちらを使用します。

2. Kubernetes

推奨: 無効化（チェックを外す）

Kubernetesは高度なコンテナ管理ツールですが、Docker入門では不要です。メモリ使用量を抑えるため無効化します。

3. PATH設定（Automatic）

推奨: 有効のまま

自動的にDockerコマンドを使えるようにしてくれます。有効のままにしてください。

初めての方へ

設定がよく分からなくても大丈夫です。上記の推奨設定のままにして、「Accept」または「OK」をクリックすれば問題ありません。後から変更することもできます。

設定を完了すると、Rancher Desktopが起動を開始します。初回起動は5〜10分程度かかることがあります。しばらくお待ちください。

2.7 動作確認をしよう

Rancher Desktopが正しくインストールされ、Dockerコマンドが使えるか確認しましょう。

ターミナル（コマンドライン）の開き方

Windows: スタートメニュー → 「PowerShell」または「コマンドプロンプト」

macOS: アプリケーション → ユーティリティ → 「ターミナル」

動作確認: Dockerコマンドが使えるか確認

ターミナルを開いて、以下のコマンドを実行してください。

ターミナル 
# Dockerのバージョンを確認
docker --version

期待される結果 

Docker version 24.0.X, build XXXXXXX

バージョン番号が表示されればOKです（数字は異なる場合があります）。

トラブルシューティング

「dockerは認識されていません」と表示される

原因: Rancher Desktopが起動していないか、PATHが通っていません

解決方法:

  1. Rancher Desktopが起動しているか確認
  2. ターミナルを一度閉じて、再度開く
  3. それでも解決しない場合は、PCを再起動
「Cannot connect to the Docker daemon」と表示される

原因: Dockerエンジンが起動していません

解決方法:

  1. Rancher Desktopを起動
  2. ウィンドウ右下のステータスが「Running」になるまで待つ（数分かかる場合あり）
  3. 再度コマンドを実行
Windows: WSL2エラーが表示される

原因: WSL2が正しくインストールされていないか、起動していません

解決方法:

  1. セクション2.3の手順を再度確認
  2. PowerShellで wsl --list --verbose を実行し、WSL2が動作しているか確認
  3. PCを再起動してから再度試す

理解度確認クイズ

  1. Q1. Rancher DesktopとDocker Desktopの大きな違いは何ですか?

    回答を表示

    ライセンス（料金）の違いです。Docker Desktopは商用利用に条件がありますが、Rancher Desktopは完全無料で商用利用も可能です。機能的にはどちらもDockerコマンドが使えるため、学習や開発には大きな差はありません。

  2. Q2. Windows環境でRancher Desktopを動かすために必要なものは何ですか?

    回答を表示

    WSL2（Windows Subsystem for Linux 2）が必要です。DockerはもともとLinux向けの技術なので、Windowsで動かすにはWSL2を使ってLinux環境を用意する必要があります。WSL2は wsl --install コマンドで簡単にインストールできます。

  3. Q3. 初期設定でKubernetesを無効化する理由は何ですか?

    回答を表示

    Kubernetesは高度なコンテナ管理ツールですが、Docker入門では使用しません。有効にすると多くのメモリを消費するため、学習段階では無効化してリソースを節約します。必要になったら後から有効化できます。

第2章のまとめ

  • Rancher Desktopは完全無料のDocker環境で、Docker Desktopの代替として使える
  • Windows環境ではWSL2のインストールが必須（wsl --installで簡単にセットアップ可能）
  • 初期設定はデフォルトのままでOK（dockerd選択、Kubernetes無効化を推奨）
  • docker --versionコマンドでインストールが成功したか確認できる
