データを保存しよう - ボリュームの基本

コンテナを削除するとデータが消えてしまう問題を理解し、データを永続化する方法(ボリューム)を学びます。

この章で学ぶこと

  • コンテナのデータ消失問題と永続化の必要性を理解する
  • ボリュームとバインドマウントの違いと使い分けを学ぶ
  • MySQLコンテナでデータ永続化を実践する

7.1 コンテナを削除するとデータが消える問題

これまで学んできたように、コンテナは起動・停止・削除が簡単にできます。しかし、コンテナを削除すると、その中に保存されていたデータもすべて消えてしまいます。

コンテナのデータは一時的

データベースコンテナを削除すると、登録したユーザー情報や投稿データもすべて失われます。これではアプリケーションとして使えません。

この問題を解決するのがボリューム(Volume)という仕組みです。ボリュームを使えば、コンテナを削除してもデータは残り続けます。

7.2 ボリュームとは - データを保存するUSBメモリのようなもの

ボリュームは、コンテナの外にデータを保存する仕組みです。USBメモリを例に考えてみましょう。

USBメモリの例え

パソコン(コンテナ)が壊れても、USBメモリ(ボリューム)に保存したデータは無事です。別のパソコンに差し替えても、同じデータを使えます。

ボリュームの特徴

  • 永続性: コンテナを削除してもデータは残る
  • 共有可能: 複数のコンテナで同じボリュームを使える
  • Docker管理: Dockerが保存場所を自動管理してくれる

7.3 ボリュームの基本コマンド

ボリュームを操作する基本的なコマンドを覚えましょう。

ボリュームを作成する

コマンド 
# my-dataという名前のボリュームを作成
docker volume create my-data

ボリューム一覧を確認する

コマンド 
# 作成済みのボリュームを一覧表示
docker volume ls

ボリュームの詳細情報を確認する

コマンド 
# ボリュームの詳細情報(保存場所など)を表示
docker volume inspect my-data

ボリュームを削除する

コマンド 
# ボリュームを削除(使用中のボリュームは削除できません)
docker volume rm my-data

7.4 実習: MySQLコンテナでデータを永続化しよう

実際にMySQLデータベースのコンテナを起動し、ボリュームを使ってデータを永続化してみましょう。

実習: MySQLでボリュームを使う

ステップ1: ボリュームを作成

コマンド 
# MySQLのデータ保存用ボリュームを作成
docker volume create mysql-data

ステップ2: MySQLコンテナをボリューム付きで起動

コマンド 
# -v でボリュームをマウント
# mysql-data:/var/lib/mysql は「ボリューム名:コンテナ内のパス」
docker run -d \
  --name my-mysql \
  -e MYSQL_ROOT_PASSWORD=mypassword \
  -v mysql-data:/var/lib/mysql \
  mysql:8.0

-v mysql-data:/var/lib/mysql でボリュームをコンテナにマウントしています

ステップ3: データベースを作成

コマンド 
# MySQLコンテナに接続
docker exec -it my-mysql mysql -uroot -pmypassword

# MySQLコンソールで実行
CREATE DATABASE testdb;
SHOW DATABASES;
exit;

ステップ4: コンテナを削除して確認

コマンド 
# コンテナを停止・削除
docker stop my-mysql
docker rm my-mysql

# 同じボリュームで再起動
docker run -d \
  --name my-mysql-new \
  -e MYSQL_ROOT_PASSWORD=mypassword \
  -v mysql-data:/var/lib/mysql \
  mysql:8.0

# データが残っているか確認
docker exec -it my-mysql-new mysql -uroot -pmypassword
SHOW DATABASES;

testdbが残っていれば成功です！

バインドマウントの使用例

開発中のHTMLファイルをリアルタイムでNginxコンテナに反映させたい場合は、バインドマウントが便利です。

コマンド 
# カレントディレクトリの html フォルダをマウント
# $(pwd) は現在のディレクトリの絶対パスを取得
docker run -d -p 8080:80 \
  -v $(pwd)/html:/usr/share/nginx/html \
  nginx

# htmlフォルダ内のファイルを編集すると即座に反映される

7.6 ボリュームとバインドマウントの使い分け

ボリュームを使う場合

  • データベースのデータ保存
  • 本番環境でのデータ永続化
  • 複数コンテナでのデータ共有

バインドマウントを使う場合

  • 開発中のソースコード
  • 設定ファイルの直接編集
  • ログファイルの確認

理解度確認クイズ

  1. Q1. コンテナを削除したときにデータを残すためには何を使いますか？

    回答を表示

    答え: ボリューム(Volume)を使います。ボリュームはコンテナの外にデータを保存するため、コンテナを削除してもデータは残り続けます。

  2. Q2. ボリュームとバインドマウントの主な違いは何ですか？

    回答を表示

    答え: ボリュームはDockerが保存場所を自動管理し、本番環境でのデータ永続化に適しています。バインドマウントはホストの任意のフォルダを指定でき、開発中のソースコードなどファイルを直接編集したい場合に便利です。

  3. Q3. MySQLコンテナでボリュームを使う場合、マウント先のパスはどこですか？

    回答を表示

    答え: /var/lib/mysql です。MySQLはこのディレクトリにデータベースファイルを保存するため、ここにボリュームをマウントします。

第7章のまとめ

  • コンテナを削除するとデータも消えるため、データ永続化にはボリュームが必要
  • ボリュームはDockerが管理する専用領域で、本番環境のデータ保存に適している
  • バインドマウントはホストのフォルダを直接使うため、開発中のファイル編集に便利
  • 用途に応じてボリュームとバインドマウントを使い分けることが重要
