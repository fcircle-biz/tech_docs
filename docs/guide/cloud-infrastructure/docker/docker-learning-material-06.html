Dockerfileを書いてみよう - オリジナルイメージの作成

Dockerfileとは何か、なぜ必要なのかを解説し、シンプルなDockerfileを一から作成します。FROM、COPY、RUNなどの基本的な命令を一つずつ丁寧に解説し、自分だけのオリジナルイメージを作成する喜びを体験します。

この章で学ぶこと

  • Dockerfileとは何か、なぜ必要なのかを理解する
  • Dockerfileの基本的な命令（FROM、COPY、RUN、CMD）を学ぶ
  • オリジナルのWebサーバーイメージを作成して実行する

6.1 Dockerfileとは - イメージの設計図を自分で書く

これまでDocker Hubから既存のイメージを使ってきましたが、Dockerfileを使えば、自分だけのオリジナルイメージを作成できます。

Dockerfileとは？

Dockerfileは、Dockerイメージの設計図を記述したテキストファイルです。料理で例えると、レシピを自分で書くようなものです。「どのベースイメージを使うか」「どんなファイルをコピーするか」「どんなコマンドを実行するか」といった手順を記述します。

6.2 なぜDockerfileが必要なのか

再現性

手順をコードで記述するため、誰でも同じイメージを作成できます。

バージョン管理

Dockerfileをgitで管理すれば、イメージの変更履歴を追跡できます。

共有が簡単

チームメンバーにテキストファイルを渡すだけで環境を共有できます。

カスタマイズ自由

既存イメージをベースに必要な環境を自由に構築できます。

6.3 Dockerfileの基本構造

Dockerfileは命令の集まりです。主要な命令を見てみましょう。

主要な命令一覧

命令 説明 例
FROM ベースとなるイメージを指定 FROM nginx:alpine
COPY ホストからコンテナへファイルコピー COPY index.html /usr/share/nginx/html/
RUN イメージビルド時にコマンド実行 RUN apt-get update
CMD コンテナ起動時のデフォルトコマンド CMD ["nginx", "-g", "daemon off;"]
WORKDIR 作業ディレクトリを設定 WORKDIR /app
EXPOSE 公開するポート番号を宣言 EXPOSE 80

6.4 シンプルなDockerfileを書いてみよう

オリジナルのHTMLページを表示するNginxサーバーを作成してみます。

実習: オリジナルWebサーバーイメージを作成

手順1: 作業ディレクトリの作成

Terminal 
# 作業用のディレクトリを作成
mkdir my-web-server
cd my-web-server

手順2: HTMLファイルを作成

お好きなテキストエディタでindex.htmlを作成します。

index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>私のDockerサーバー</title>
</head>
<body>
    <h1>こんにちは、Docker！</h1>
    <p>これは私が作った初めてのDockerイメージです。</p>
</body>
</html>

手順3: Dockerfileを作成

同じディレクトリにDockerfileという名前のファイルを作成します（拡張子なし）。

Dockerfile 
# ベースイメージとしてNginx（Alpine版）を使用
FROM nginx:alpine

# 作成したHTMLファイルをNginxのドキュメントルートにコピー
COPY index.html /usr/share/nginx/html/

# ポート80を公開（ドキュメント目的）
EXPOSE 80

# Nginxをフォアグラウンドで起動（デフォルトのCMDを使用）

各行の解説:

  • FROM nginx:alpine: 軽量なNginxイメージをベースにします
  • COPY index.html ...: 作成したHTMLをNginxが読める場所にコピー
  • EXPOSE 80: ポート80を使うことを明示（情報提供のみ）

6.5 イメージをビルドする - docker build

Dockerfileからイメージを作成するにはdocker buildコマンドを使います。

イメージをビルド 
# -t でイメージに名前とタグを付ける
# . はDockerfileがあるディレクトリ（カレントディレクトリ）
docker build -t my-web-server:1.0 .

-tオプションで「名前:タグ」を指定します。最後の.はDockerfileがあるディレクトリのパス（ここではカレントディレクトリ）を示します。

6.6 作成したイメージを実行する

コンテナを起動 
# 作成したイメージからコンテナを起動
docker run -d -p 8080:80 --name my-server my-web-server:1.0

動作確認

ブラウザで以下にアクセスしてみましょう：

http://localhost:8080

あなたが作成したHTMLページが表示されれば成功です！

コンテナを停止・削除 
# コンテナを停止
docker stop my-server

# コンテナを削除
docker rm my-server

6.7 もう少し実用的なDockerfile

RUN命令を使って、イメージビルド時にパッケージをインストールする例を見てみましょう。

Python Webアプリのイメージ例 
# Python 3.11をベースイメージに
FROM python:3.11-slim

# 作業ディレクトリを設定
WORKDIR /app

# 依存関係ファイルをコピー
COPY requirements.txt .

# 依存パッケージをインストール
RUN pip install --no-cache-dir -r requirements.txt

# アプリケーションコードをコピー
COPY . .

# ポート8000を公開
EXPOSE 8000

# アプリケーションを起動
CMD ["python", "app.py"]

ベストプラクティス

  • 軽量なベースイメージを選ぶ: alpineやslim版を使うとイメージサイズが小さくなります
  • レイヤーを意識する: 変更頻度が低いものを先に書くとビルドが速くなります
  • .dockerignoreを活用: 不要なファイルをビルドコンテキストから除外します
  • 1つの命令に1つの役割: わかりやすく保守しやすいDockerfileになります

6.8 ビルドキャッシュを理解する

Dockerはビルド時にレイヤーキャッシュという仕組みを使います。変更がない命令は再利用されるので、2回目以降のビルドが高速化されます。

キャッシュを活かすコツ

理解度確認クイズ

  1. Q1. Dockerfileの役割は何ですか？

    回答を表示

    答え: Dockerfileは、Dockerイメージを作成するための設計図（レシピ）です。ベースイメージの指定、ファイルのコピー、コマンドの実行など、イメージをどのように構築するかを記述したテキストファイルです。

  2. Q2. FROM命令とCOPY命令の違いは何ですか？

    回答を表示

    答え: FROM命令はベースとなるイメージを指定します（例: FROM nginx:alpine）。COPY命令はホストマシン上のファイルをコンテナ内にコピーします（例: COPY index.html /usr/share/nginx/html/）。FROMは必ず最初に書き、COPYはファイルを追加したいときに使います。

  3. Q3. Dockerfileからイメージを作成するコマンドは何ですか？

    回答を表示

    答え: docker buildコマンドです。例: docker build -t イメージ名:タグ . のように使います。-tオプションでイメージに名前を付け、最後の.はDockerfileがあるディレクトリを指定します。

第6章のまとめ

  • Dockerfileは、Dockerイメージを作成するためのテキストベースの設計図
  • FROM、COPY、RUN、CMD、WORKDIR、EXPOSEなどの命令を使って構成する
  • docker buildコマンドでDockerfileからイメージを作成できる
  • ビルドキャッシュを意識した書き方で、効率的にイメージを構築できる
  • オリジナルのWebサーバーイメージを作成し、実際に動かすことができた
