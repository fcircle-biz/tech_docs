第6章

Dockerfileを書いてみよう - オリジナルイメージの作成

Dockerfileとは何か、なぜ必要なのかを理解し、シンプルなDockerfileを一から作成します。HTMLページを表示するWebサーバーのオリジナルイメージを作成する喜びを体験しましょう。

この章で学ぶこと

  • Dockerfileとは何か、なぜ必要なのかを理解する
  • Dockerfileの基本的な命令（FROM、COPY、RUN、CMD、EXPOSE）を学ぶ
  • HTMLを表示するシンプルなWebサーバーイメージを自作する
  • イメージのビルドプロセスを理解する

6.1 Dockerfileとは何か

これまでの章では、Docker Hubから既存のイメージ（nginxなど）を取得して使ってきました。しかし、実際の開発では自分のアプリケーションを含んだオリジナルのイメージを作る必要があります。

Dockerfile（ドッカーファイル）は、Dockerイメージを作るための設計図・レシピです。料理のレシピに「材料」と「手順」が書かれているように、Dockerfileには「ベースとなるイメージ」と「イメージを作る手順」が書かれています。

料理のレシピの例え

料理のレシピ

  • • 材料: 米、水
  • • 手順1: 米を洗う
  • • 手順2: 水を加える
  • • 手順3: 炊飯器で炊く
  • • 完成: ご飯ができる

Dockerfile

  • • FROM: ベースイメージ
  • • COPY: ファイルをコピー
  • • RUN: コマンド実行
  • • CMD: 起動コマンド
  • • 完成: イメージができる

Dockerfileを書けば、誰でも同じイメージを作れます。チーム全員が同じDockerfileを使うことで、全員が同じ環境を手に入れることができます。

6.2 なぜDockerfileが必要なのか

バージョン管理できる

Dockerfileはテキストファイルなので、Gitでバージョン管理できます。「誰が」「いつ」「何を変更したか」が記録されます。

チームで共有できる

Dockerfileをチームで共有すれば、誰でも同じイメージを作れます。環境構築の手順書を書く必要がなくなります。

再現性が高い

同じDockerfileからは常に同じイメージが作られます。「私のPCでは動くのに」という問題がなくなります。

ドキュメントになる

Dockerfileを読めば、そのイメージがどう作られているかが分かります。環境構築の手順がコードとして残ります。

6.3 Dockerfileの基本的な命令

Dockerfileには様々な命令がありますが、まずは基本的な5つの命令を覚えましょう。

Dockerfileの基本構造

flowchart TD A[FROM: ベースイメージを指定] --> B[COPY: ファイルをコピー] B --> C[RUN: コマンドを実行] C --> D[EXPOSE: ポートを公開] D --> E[CMD: 起動コマンドを指定] E --> F[完成: イメージができる] style A fill:#e0f2fe,stroke:#0ea5e9 style B fill:#e0f2fe,stroke:#0ea5e9 style C fill:#e0f2fe,stroke:#0ea5e9 style D fill:#e0f2fe,stroke:#0ea5e9 style E fill:#e0f2fe,stroke:#0ea5e9 style F fill:#d1fae5,stroke:#10b981

FROM - ベースイメージを指定

どのイメージを土台にするかを指定します。必ず最初に書く必要があります。

例 
FROM nginx:latest

COPY - ファイルをコピー

PC上のファイルをイメージの中にコピーします。自分が作ったHTMLやプログラムをイメージに入れるときに使います。

例 
COPY index.html /usr/share/nginx/html/

※ 左側がPC上のファイル、右側がイメージ内のコピー先

RUN - コマンドを実行

イメージを作る過程でコマンドを実行します。ソフトウェアのインストールや設定に使います。

例 
# パッケージの更新
RUN apt-get update

EXPOSE - ポートを公開

コンテナがどのポート番号で通信するかを宣言します（第5章で学んだポートです）。

例 
EXPOSE 80

CMD - 起動コマンドを指定

コンテナが起動したときに実行するコマンドを指定します。Dockerfileの最後に書きます。

例 
CMD ["nginx", "-g", "daemon off;"]

6.4 実習: オリジナルのWebサーバーイメージを作ろう

それでは、実際に自分だけのDockerイメージを作ってみましょう。HTMLページを表示するシンプルなWebサーバーを作ります。

実習 6-1: オリジナルWebサーバーイメージの作成

自分だけのHTMLページを表示するWebサーバーイメージを作成します

手順1: 作業フォルダを作成

まず、作業用のフォルダを作ります。

ターミナル 
# フォルダを作成して移動
mkdir my-website
cd my-website

手順2: HTMLファイルを作成

次に、表示するHTMLページを作ります。

index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>私の最初のDockerイメージ</title>
</head>
<body>
    <h1>こんにちは、Docker!</h1>
    <p>これは私が作った最初のDockerイメージです。</p>
</body>
</html>

手順3: Dockerfileを作成

同じフォルダに「Dockerfile」という名前のファイルを作ります（拡張子なし）。

Dockerfile 
# ベースイメージとしてNginxを使用
FROM nginx:latest

# 自分のHTMLをコンテナ内にコピー
COPY index.html /usr/share/nginx/html/

# ポート80を公開
EXPOSE 80

# Nginxを起動（デフォルトで実行されるのでCMDは省略可能）

現在のフォルダ構成

my-website/
├── Dockerfile        ← イメージの設計図
└── index.html        ← 表示するHTMLページ

手順4: イメージをビルド

Dockerfileからイメージを作成します。これを「ビルド」と言います。

ターミナル 
# イメージをビルド
# docker build -t [イメージ名]:[タグ] [Dockerfileがあるフォルダ]
docker build -t my-website:1.0 .

説明:
• -t my-website:1.0 : イメージに「my-website」という名前と「1.0」というタグをつける
• . : 現在のフォルダ（Dockerfileがある場所）

手順5: イメージが作成されたか確認

ターミナル 
# イメージ一覧を表示
docker images

「my-website」という名前のイメージが表示されれば成功です！

手順6: コンテナを起動

ターミナル 
# 自分が作ったイメージからコンテナを起動
docker run -d -p 8080:80 --name my-site my-website:1.0

手順7: ブラウザで確認

ブラウザで以下のURLにアクセスしてみましょう。

http://localhost:8080

自分が作ったHTMLページが表示されれば大成功です！🎉

6.5 イメージのビルドプロセスを理解しよう

docker buildコマンドを実行すると、Dockerは何をしているのでしょうか？そのプロセスを図で理解しましょう。

イメージビルドの流れ

重要ポイント

Dockerはレイヤー（層）という仕組みを使ってイメージを作ります。各命令が1つのレイヤーになり、重なってイメージができます。これにより、変更があった部分だけを作り直せるので効率的です。

各命令が1つのレイヤーとなり、積み重なってイメージができます

6.6 よくあるつまずきポイント

トラブルシューティング

Dockerfileが見つからない

原因: Dockerfileが正しい場所にないか、ファイル名が間違っている

解決方法:

  • ファイル名は「Dockerfile」（拡張子なし、大文字のD）
  • docker buildを実行するフォルダにDockerfileがあることを確認
  • lsコマンドでファイルがあるか確認
COPYでファイルがコピーされない

原因: コピー元のファイルパスが間違っている

解決方法:

  • COPYの左側は、Dockerfileがあるフォルダからの相対パス
  • ファイルがDockerfileと同じフォルダにあることを確認
  • ファイル名のスペルミスに注意
ビルドは成功したが起動しない

原因: CMDの指定が間違っているか、ポート番号が合っていない

解決方法:

  • docker logsコマンドでエラーメッセージを確認
  • ポート番号の対応を確認（-p 8080:80など）
  • 別のコンテナが同じポートを使っていないか確認

理解度確認クイズ

  1. Q1. Dockerfileとは何ですか？料理のレシピに例えて説明してください。

    回答を表示

    Dockerfileは、Dockerイメージを作るための設計図・レシピです。料理のレシピに「材料」と「手順」が書かれているように、Dockerfileには「ベースとなるイメージ（FROM）」と「イメージを作る手順（COPY、RUN、CMDなど）」が書かれています。このDockerfileがあれば、誰でも同じイメージを再現できます。

  2. Q2. FROM、COPY、RUN、EXPOSE、CMDの5つの命令は、それぞれ何をするものですか？

    回答を表示

    • FROM: ベースとなるイメージを指定する（土台を決める）
    • COPY: PCのファイルをイメージ内にコピーする
    • RUN: イメージを作る過程でコマンドを実行する（ソフトインストールなど）
    • EXPOSE: コンテナが使うポート番号を宣言する
    • CMD: コンテナ起動時に実行するコマンドを指定する

  3. Q3. docker build -t my-app:1.0 .コマンドの意味を説明してください。

    回答を表示

    • docker build: イメージをビルド（作成）する
    • -t my-app:1.0: イメージに「my-app」という名前と「1.0」というタグをつける
    • .: 現在のフォルダにあるDockerfileを使う

    このコマンドは、現在のフォルダにあるDockerfileを使って「my-app:1.0」という名前のイメージを作成します。

第6章のまとめ

  • Dockerfileは、イメージを作るための設計図・レシピ
  • 基本的な命令は FROM、COPY、RUN、EXPOSE、CMD の5つ
  • docker buildコマンドでDockerfileからイメージを作成できる
  • イメージはレイヤー構造で、各命令が1つのレイヤーになる
  • 自分だけのオリジナルイメージを作成できるようになった
