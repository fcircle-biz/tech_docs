第6章

Dockerfileを書いてみよう - オリジナルイメージの作成

Dockerfileとは何か、なぜ必要なのかを解説し、シンプルなDockerfileを一から作成します。FROM、COPY、RUNなどの基本的な命令を一つずつ丁寧に解説し、自分だけのオリジナルイメージを作成する喜びを体験します。

この章で学ぶこと

  • Dockerfileとは何か、なぜ必要なのかを理解する
  • FROM, COPY, RUN, CMD などの基本命令を習得する
  • 自分だけのオリジナルイメージを作成する
  • docker buildコマンドでイメージをビルドする

6.1 Dockerfileとは何か

Dockerfile（ドッカーファイル）は、Dockerイメージを作成するための「レシピ」です。料理のレシピと同じように、材料（ベースイメージ）と手順（命令）を記述することで、誰でも同じイメージを作ることができます。

第4章・第5章では、Docker Hubにある既存のイメージを使ってきました。しかし、実際の開発では、自分のアプリケーションを含むオリジナルのイメージを作る必要があります。そのときにDockerfileが必要になります。

Dockerfileは料理のレシピと同じ

1
材料を用意（FROM）

ベースとなるイメージを指定します（例: nginx, python, node.js）

2
材料を追加（COPY）

自分のファイル（HTMLやソースコード）をコンテナ内にコピーします

3
調理する（RUN）

パッケージのインストールなど、必要な準備作業を実行します

4
完成したら実行（CMD）

コンテナ起動時にどのコマンドを実行するかを指定します

Dockerfileを一度作っておけば、チームメンバーや将来の自分が同じイメージを簡単に再現できます。これがDockerfileの最大のメリットです。

6.2 基本的な命令を学ぶ

Dockerfileで使える命令はたくさんありますが、まずは以下の6つの基本命令を覚えましょう。

FROM

ベースとなるイメージを指定します。必ず最初に記述する必要があります。

FROM nginx:latest

COPY

ホストマシンのファイルをコンテナ内にコピーします。

COPY index.html /usr/share/nginx/html/

RUN

イメージ作成時にコマンドを実行します（パッケージインストールなど）。

RUN apt-get update && apt-get install -y curl

CMD

コンテナ起動時に実行するデフォルトコマンドを指定します。

CMD ["nginx", "-g", "daemon off;"]

WORKDIR

作業ディレクトリを指定します。以降の命令はこのディレクトリで実行されます。

WORKDIR /app

EXPOSE

コンテナが使用するポート番号を宣言します（ドキュメントとしての役割）。

EXPOSE 80

重要ポイント

RUNはイメージ作成時、CMDはコンテナ起動時に実行されます。この違いをしっかり理解しましょう。

6.3 はじめてのDockerfileを作成する

それでは、実際にシンプルなDockerfileを作成してみましょう。ここでは、自分のHTMLページを表示するWebサーバーイメージを作成します。

実習 6-1: オリジナルWebページイメージの作成

自分のHTMLページを表示するDockerイメージを作成します。

手順

  1. 1 作業用フォルダを作成します
  2. 2 HTMLファイルを作成します
  3. 3 Dockerfileを作成します

ステップ1: 作業用フォルダを作成

まず、作業用のフォルダを作成して移動します。

ターミナル 
# 作業用フォルダを作成
mkdir my-web-app
cd my-web-app

ステップ2: HTMLファイルを作成

次に、表示したいHTMLファイルを作成します。テキストエディタでindex.htmlというファイルを作成し、以下の内容を保存してください。

index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>私のDockerページ</title>
</head>
<body>
    <h1>Dockerfileで作った私のWebページ！</h1>
    <p>Dockerfileを使って、オリジナルイメージを作成しました。</p>
</body>
</html>

ステップ3: Dockerfileを作成

次に、Dockerfileを作成します。ファイル名は必ず「Dockerfile」（拡張子なし）にしてください。

Dockerfile 
# ベースイメージとしてNginxを使用
FROM nginx:latest

# 作成したHTMLファイルをNginxのドキュメントルートにコピー
COPY index.html /usr/share/nginx/html/

# ポート80を公開
EXPOSE 80

# Nginxを起動（デフォルトのままでOK）
CMD ["nginx", "-g", "daemon off;"]

ヒント

Dockerfileの各行は、上から順番に実行されます。#で始まる行はコメントです。

現在のフォルダ構成 

my-web-app/
├── Dockerfile        ← イメージのレシピ
└── index.html        ← 表示したいHTMLファイル

6.4 docker buildでイメージを作成する

Dockerfileを作成したら、docker buildコマンドでイメージをビルドします。

ターミナル 
# イメージをビルド（-t でイメージ名を指定、. は現在のフォルダを指定）
docker build -t my-web-app .

ビルドが成功すると、以下のような出力が表示されます。

docker imagesコマンドで、イメージが作成されたことを確認できます。

6.5 作成したイメージを実行する

それでは、作成したイメージからコンテナを起動してみましょう。

ターミナル 
# コンテナを起動（ポート8080で公開）
docker run -d -p 8080:80 --name my-app my-web-app

動作確認

ブラウザでhttp://localhost:8080にアクセスしてください。

期待される結果

自分が作成したHTMLページが表示されます。「Dockerfileで作った私のWebページ！」という見出しが表示されれば成功です。

後片付け

確認が終わったら、コンテナを停止・削除しましょう。

ターミナル

docker rm my-app

6.6 Dockerfileのベストプラクティス（基本）

Dockerfileを書くときに覚えておきたい基本的なベストプラクティスを紹介します。

1. 軽量なベースイメージを選ぶ

できるだけ小さいイメージを使うと、ダウンロードやビルドが速くなります。例: nginx:alpineはnginx:latestより小さいです。

2. .dockerignoreファイルを活用する

不要なファイル（.git、node_modulesなど）をコピーしないように、.dockerignoreファイルに記述します。イメージサイズの削減につながります。

3. レイヤーを意識する

Dockerfileの各命令（FROM、COPY、RUNなど）は1つのレイヤーを作ります。変更の少ない命令を先に書くと、ビルドが速くなります。

4. 1つのRUNで複数のコマンドを実行する

&&で繋げることで、レイヤー数を減らせます。例: RUN apt-get update && apt-get install -y curl

良い例と悪い例

悪い例 

# レイヤーが増える
RUN apt-get update
RUN apt-get install -y curl
RUN apt-get install -y git

良い例 

# 1つのレイヤーで完結
RUN apt-get update && \
    apt-get install -y curl git

理解度確認クイズ

  1. Q1. Dockerfileの最初に必ず書かなければならない命令は何ですか？

    回答を表示

    FROM命令です。FROMはベースとなるイメージを指定する命令で、Dockerfileの最初に記述する必要があります。これがないとイメージを作成できません。

  2. Q2. RUN命令とCMD命令の違いは何ですか？

    回答を表示

    RUNはイメージをビルドする際（docker build時）に実行され、CMDはコンテナを起動する際（docker run時）に実行されます。RUNはパッケージのインストールなど環境準備に使い、CMDはアプリケーションの起動コマンドを指定します。

  3. Q3. docker build -t my-app . コマンドの最後の「.（ドット）」は何を意味しますか？

    回答を表示

    「.（ドット）」は現在のディレクトリを意味します。つまり、「現在のディレクトリにあるDockerfileを使ってイメージをビルドする」という意味になります。また、このディレクトリがビルドコンテキスト（COPYやADD命令でコピーできるファイルの範囲）になります。

第6章のまとめ

  • Dockerfileはイメージを作成するための「レシピ」で、料理のレシピのように手順を記述する
  • 基本命令: FROM（ベースイメージ）、COPY（ファイルコピー）、RUN（ビルド時実行）、CMD（起動時実行）など
  • docker build -t イメージ名 . でイメージをビルドできる
  • ベストプラクティスを意識することで、効率的で保守しやすいDockerfileが書ける
