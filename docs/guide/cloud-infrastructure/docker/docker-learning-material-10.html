トラブルシューティングと次のステップ

Docker使用中によくあるエラーとその解決方法をまとめます。「コンテナが起動しない」「ポートが使用中」「ディスク容量不足」などの典型的な問題への対処法を学びます。また、この先の学習パス（Kubernetes、クラウドへのデプロイなど）も紹介します。

この章で学ぶこと

  • よくあるDockerエラーの原因を理解する
  • 典型的なトラブルの解決方法を習得する
  • デバッグの基本的なアプローチを学ぶ
  • 次の学習ステップ（Kubernetes、クラウドなど）を知る

10.1 トラブルシューティングの基本姿勢

Dockerを使っていると、必ずエラーに遭遇します。しかし、エラーは成長のチャンスです。エラーメッセージを読み、原因を探り、解決することで、Dockerへの理解が深まります。

トラブルシューティングの3つの基本

1
エラーメッセージを最後まで読む

赤い文字に驚かず、落ち着いてメッセージを読みましょう。多くの場合、何が原因かヒントが書かれています。

2
ログを確認する

docker logs コンテナ名でコンテナの出力を確認します。エラーの詳細が記録されています。

3
一つずつ確認する

複数の原因が考えられる場合、一つずつ切り分けて確認します。推測だけで進めず、実際に確認しましょう。

10.2 よくあるエラーと解決方法

Docker使用中によく遭遇するエラーとその対処法を紹介します。これらを知っておくことで、トラブルが起きても慌てずに対処できます。

エラー1: コンテナが起動しない

症状:

# docker psで確認しても何も表示されない
$ docker ps
CONTAINER ID   IMAGE     COMMAND   CREATED   STATUS    PORTS     NAMES

原因と対処法:

  • 原因1: コンテナがすぐに終了している - docker ps -aで終了したコンテナも表示し、docker logs コンテナ名でエラーを確認します。
  • 原因2: イメージの指定ミス - イメージ名のスペルミスやタグの間違いがないか確認します。
  • 原因3: コマンドが正しくない - DockerfileのCMDやrunコマンドで指定したコマンドが間違っていないか確認します。

エラー2: ポートが使用中 (port already in use)

症状:

Error: Bind for 0.0.0.0:8080 failed: port is already allocated

原因と対処法:

# 使用中のポートを確認
$ docker ps

# 該当ポートを使用中のコンテナを停止
$ docker stop コンテナ名

# または、別のポートを使用
$ docker run -p 8081:80 nginx  # 8080の代わりに8081を使用

エラー3: ディスク容量不足

症状:

Error: no space left on device

原因と対処法:

# ディスク使用量を確認
$ docker system df

# 不要なイメージ、コンテナ、ボリュームを削除
$ docker system prune -a

# ボリュームも含めて全削除（注意: データが消えます）
$ docker system prune -a --volumes

エラー4: パーミッションエラー (permission denied)

症状:

Got permission denied while trying to connect to the Docker daemon socket

原因と対処法:

Rancher Desktopが起動していない、またはユーザーに権限がない可能性があります。

  • 対処1: Rancher Desktopが起動しているか確認します。
  • 対処2: Rancher Desktopを再起動します。
  • 対処3: ターミナルを再起動して、環境変数を再読み込みします。

エラー5: ネットワーク接続エラー

症状:

# コンテナからインターネットに接続できない
Could not resolve host: example.com

原因と対処法:

# コンテナのネットワーク設定を確認
$ docker network ls

# コンテナのネットワーク接続を確認
$ docker exec コンテナ名 ping -c 3 google.com

# ネットワークを再作成
$ docker network create my-network
$ docker run --network my-network イメージ名

10.3 デバッグに便利なコマンド

トラブル発生時に状況を把握するための便利なコマンドを紹介します。これらのコマンドを使いこなすことで、問題の原因を素早く特定できます。

docker logs

コンテナの標準出力とエラー出力を表示します。アプリケーションのログやエラーメッセージを確認できます。 

# 最新のログを表示
docker logs コンテナ名

# リアルタイムでログを追跡
docker logs -f コンテナ名

docker exec

実行中のコンテナ内でコマンドを実行します。コンテナの中に入って状況を直接確認できます。 

# コンテナ内でbashを起動
docker exec -it コンテナ名 bash

# ファイルの内容を確認
docker exec コンテナ名 cat /etc/hosts

docker inspect

コンテナやイメージの詳細情報をJSON形式で表示します。設定やネットワーク情報を詳しく確認できます。 

# コンテナの詳細情報を表示
docker inspect コンテナ名

# IPアドレスだけを表示
docker inspect -f '{{.NetworkSettings.IPAddress}}' コンテナ名

docker stats

コンテナのリソース使用状況（CPU、メモリ、ネットワーク）をリアルタイムで表示します。 

# 全コンテナの統計情報を表示
docker stats

# 特定コンテナのみ表示
docker stats コンテナ名

10.4 リソースのクリーンアップ

Dockerを使い続けると、不要なイメージ、コンテナ、ボリュームが溜まってディスク容量を圧迫します。定期的にクリーンアップしましょう。

docker system pruneコマンド

docker system pruneは、使用していないDockerリソースを一括で削除する強力なコマンドです。 

# 停止中のコンテナ、未使用のネットワーク、ダングリングイメージを削除
$ docker system prune

# 未使用のイメージも全て削除（より徹底的に）
$ docker system prune -a

# ボリュームも含めて全削除（注意: データが消えます）
$ docker system prune -a --volumes

# 確認なしで実行（自動化スクリプトで使用）
$ docker system prune -f

注意: --volumesオプションを使うと、ボリューム内のデータも削除されます。重要なデータは事前にバックアップしてください。

個別に削除する方法

# 停止中のコンテナを全て削除
$ docker container prune

# 未使用のイメージを削除
$ docker image prune

# 未使用のボリュームを削除
$ docker volume prune

# 未使用のネットワークを削除
$ docker network prune

10.5 次の学習ステップ

この学習ガイドでDockerの基礎を習得したあなたは、さらに高度な技術へステップアップする準備ができています。次に学ぶべき技術を紹介します。

Kubernetesへのステップアップ

Kubernetes（K8s）は、複数のコンテナを大規模に管理・運用するためのオーケストレーションツールです。Dockerで1つのコンテナを動かせるようになったら、次は数百、数千のコンテナを管理する方法を学びましょう。

学べること: コンテナの自動スケーリング、負荷分散、自動復旧、ローリングアップデートなど

クラウドへのデプロイ

ローカルで動かしたコンテナを、実際のWebサービスとして世界に公開しましょう。主要なクラウドサービスはすべてDockerコンテナに対応しています。

  • AWS ECS/Fargate: Amazonのコンテナサービス。サーバーレスでコンテナを実行できます。
  • Google Cloud Run: Googleのサーバーレスコンテナプラットフォーム。自動スケールが特徴。
  • Azure Container Instances: Microsoftのコンテナサービス。Windows Serverコンテナにも対応。

CI/CDパイプラインとの連携

コードをGitにプッシュしたら自動でDockerイメージをビルドし、テストして、デプロイする仕組みを作りましょう。開発効率が劇的に向上します。

  • GitHub Actions: GitHubに統合されたCI/CDツール。Dockerイメージの自動ビルドが簡単。
  • GitLab CI/CD: GitLabの組み込みCI/CD。Dockerとの親和性が高い。
  • Jenkins: オープンソースの定番CI/CDツール。高度なカスタマイズが可能。
