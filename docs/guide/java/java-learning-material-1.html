第1章：Java基礎

この章では、Javaプログラミングの基礎知識から開発環境の構築、最初のプログラム作成までを学びます。Javaの特徴を理解し、プログラミングの第一歩を踏み出しましょう。

1.1 Javaの概要と特徴

Javaは1995年にSun Microsystems社（現Oracle社）によって開発されたプログラミング言語です。「Write Once, Run Anywhere（一度書けば、どこでも動く）」という設計思想のもと、様々なプラットフォームで動作する汎用性の高い言語として広く普及しています。

Javaの主な特徴

プラットフォーム非依存

Java仮想マシン(JVM)上で動作するため、異なるOS上でも同じように動作します。

オブジェクト指向

クラスとオブジェクトを用いた設計ができ、再利用性と保守性に優れたコードを書けます。

自動メモリ管理

ガベージコレクションにより、プログラマがメモリ管理を意識する必要がありません。

堅牢性と安全性

厳格な型チェックと例外処理機構により、安全で堅牢なプログラムを作成できます。

1.2 JDK (Java Development Kit)のインストール

Javaでプログラミングを行うには、まずJDK（Java Development Kit）をインストールする必要があります。JDKには、Javaプログラムの開発に必要なコンパイラやツール、ライブラリが含まれています。

JDKのバージョン選択

JDKには様々なバージョンがありますが、初心者の方は最新のLTS（Long-Term Support）バージョンを選ぶことをお勧めします。現在のLTSバージョンは、Java 17およびJava 21です。

JDKの構成要素
  • JRE (Java Runtime Environment): Javaプログラムを実行するための環境
  • Java Compiler (javac): Java言語のソースコードをバイトコードにコンパイルするツール
  • 開発ツール: デバッガやドキュメント生成ツールなど
  • 基本的なライブラリ: Java APIなど

インストール手順

  1. Oracle公式サイトからJDKをダウンロード: https://www.oracle.com/java/technologies/downloads/
  2. ダウンロードしたインストーラーを実行し、画面の指示に従ってインストール
  3. 環境変数の設定：
    • 「システムのプロパティ」→「環境変数」を開く
    • 「システム環境変数」に「JAVA_HOME」を追加し、JDKのインストールパスを設定
    • 「Path」変数に「%JAVA_HOME%\bin」を追加
  4. コマンドプロンプトで「java -version」と入力し、正しくインストールされたか確認

  1. Oracle公式サイトからmacOS用JDKをダウンロード
  2. ダウンロードしたDMGファイルを開き、インストーラーを実行
  3. ターミナルで「java -version」と入力し、正しくインストールされたか確認

Ubuntu/Debianの場合：

sudo apt update sudo apt install openjdk-17-jdk

CentOS/Fedoraの場合：

sudo dnf install java-17-openjdk-devel

インストール確認：

java -version

1.4 最初のJavaプログラム "Hello World"

プログラミング学習の伝統として、最初に「Hello World」と出力するプログラムを作成してみましょう。このシンプルなプログラムを通して、Javaの基本的な構文とプログラムの実行方法を学びます。

Hello Worldプログラムの作成

以下のコードを新しいファイル「HelloWorld.java」として保存します：

public class HelloWorld { public static void main(String[] args) { // コンソールに「Hello, World!」と出力する System.out.println("Hello, World!"); } }
ファイル名に関する注意点

Javaでは、publicクラスのファイル名は、クラス名と完全に一致する必要があります。大文字と小文字も区別されるので注意しましょう。

プログラムの実行方法

  1. IDEを使う場合：
    • プロジェクトにHelloWorld.javaを追加
    • 「実行」ボタンをクリックするか、ショートカットキー（IDEによって異なる）を押す
  2. コマンドラインを使う場合：
    • コマンドプロンプト（Windows）またはターミナル（macOS/Linux）を開く
    • HelloWorld.javaが保存されているディレクトリに移動
    • 以下のコマンドを実行してコンパイル：
      javac HelloWorld.java
    • 以下のコマンドを実行して実行：
      java HelloWorld

  3. メッセージの間に区切り線（例：--------）を表示する

1.5 Javaプログラムの構造

Hello Worldプログラムを見てきましたが、ここではJavaプログラムの基本的な構造について詳しく学びましょう。

クラスの構造

Javaはオブジェクト指向言語であり、すべてのコードはクラスの中に記述します。

public class クラス名 { // クラスの内容 }

mainメソッド

Javaプログラムのエントリーポイント（開始点）は、mainメソッドです。

public static void main(String[] args) { // プログラムのメイン処理 }

mainメソッドの各部分の意味：

  • public: このメソッドが他のクラスからアクセス可能であることを示す
  • static: このメソッドがクラスのインスタンスを作成せずに呼び出せることを示す
  • void: このメソッドが値を返さないことを示す
  • main: メソッド名（Javaの実行環境が最初に呼び出すメソッド）
  • String[] args: コマンドライン引数を受け取るためのパラメータ

コメント

Javaでは以下の方法でコメントを記述できます：

// 1行コメント /* 複数行コメント 複数行にわたって記述できます */ /** * JavaDocコメント * ドキュメント生成用のコメントです * @param args コマンドライン引数 */

パッケージ

Javaではクラスをパッケージという単位でグループ化します。パッケージはクラスの名前空間を提供し、名前の衝突を避けるのに役立ちます。

package com.example.myapp; public class MyClass { // クラスの内容 }

1.6 コンパイルと実行の仕組み

Javaプログラムは「コンパイル」と「実行」の2つのステップで動作します。これはJavaの「Write Once, Run Anywhere」という特性を支える重要な仕組みです。

Javaプログラムの動作フロー

  1. ソースコードの作成: .javaファイルとしてプログラムを記述
  2. コンパイル: javacコマンドでソースコードをバイトコード（.classファイル）に変換
  3. 実行: javaコマンドでJVM上でバイトコードを実行

図1.3: Javaプログラムのコンパイルと実行フロー

JVMの役割

Java Virtual Machine (JVM) は、Javaプログラムの実行環境です。JVMの主な役割は：

  • バイトコードを実行時に解釈し、実際のコンピュータが理解できる機械語に変換
  • メモリ管理とガベージコレクション（不要になったオブジェクトの自動回収）
  • セキュリティ管理（Javaプログラムが実行できる操作の制限）
  • プラットフォーム非依存性の提供（異なるOS上で同じバイトコードを実行可能）
コンパイル言語 vs インタプリタ言語

Javaは「コンパイル言語」と「インタプリタ言語」の特性を兼ね備えています：

  • コンパイル言語としての側面: ソースコードをバイトコードにコンパイルする
  • インタプリタ言語としての側面: JVMがバイトコードを実行時に解釈する

この「ハイブリッド」なアプローチにより、開発の容易さとパフォーマンスのバランスを取っています。
