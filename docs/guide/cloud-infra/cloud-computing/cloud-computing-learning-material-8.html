8.1 サーバーレスコンピューティングの概念

サーバーレスとは

サーバーレスコンピューティングは、サーバーのプロビジョニングや管理を開発者が意識せずに、コードの実行のみに集中できるモデルです。

項目従来型（サーバーベース）サーバーレス
サーバー管理開発者が管理クラウドプロバイダーが管理
スケーリング手動または設定が必要完全自動
課金稼働時間ベース実行時間・リクエスト数ベース
アイドル時コスト発生ゼロ
起動時間常時稼働コールドスタートあり

サーバーレスのメリット

  • 運用負荷の削減: サーバー管理、パッチ適用、スケーリングが不要
  • コスト効率: 使用した分だけ課金、アイドル時はコストゼロ
  • 自動スケーリング: トラフィックに応じて自動的にスケール
  • 開発速度の向上: インフラではなくビジネスロジックに集中

実習 8-1: AWS Lambdaの作成とデプロイ
# lambda_function.py
import json

def lambda_handler(event, context):
    name = event.get('name', 'World')
    message = f'Hello, {name}!'

    return {
        'statusCode': 200,
        'body': json.dumps({
            'message': message
        })
    }
# Lambda関数の作成（AWS CLI）
# まずIAMロールを作成（Lambda実行用）
aws iam create-role --role-name lambda-ex --assume-role-policy-document '{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {"Service": "lambda.amazonaws.com"},
    "Action": "sts:AssumeRole"
  }]
}'

# 基本実行ポリシーをアタッチ
aws iam attach-role-policy \
    --role-name lambda-ex \
    --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

# コードをZIP化
zip function.zip lambda_function.py

# Lambda関数を作成
aws lambda create-function \
    --function-name my-function \
    --runtime python3.11 \
    --role arn:aws:iam::ACCOUNT_ID:role/lambda-ex \
    --handler lambda_function.lambda_handler \
    --zip-file fileb://function.zip

# 関数を実行
aws lambda invoke \
    --function-name my-function \
    --payload '{"name": "Claude"}' \
    response.json

cat response.json

8.4 コンテナの基本概念

コンテナとは

コンテナは、アプリケーションとその依存関係をパッケージ化し、どの環境でも一貫して実行できるようにする技術です。

項目仮想マシン（VM）コンテナ
起動時間数分数秒
サイズGB単位MB単位
リソース効率低い高い
分離レベル完全分離プロセスレベル
ポータビリティ低い非常に高い

Dockerの基本コマンド

# イメージの検索
docker search nginx

# イメージのダウンロード
docker pull nginx

# コンテナの実行
docker run -d -p 80:80 --name my-nginx nginx

# 実行中のコンテナ一覧
docker ps

# コンテナのログ確認
docker logs my-nginx

# コンテナの停止
docker stop my-nginx

# コンテナの削除
docker rm my-nginx

8.6 Kubernetesの基本概念

Kubernetesとは

Kubernetes（K8s）は、コンテナオーケストレーションプラットフォームで、コンテナの自動デプロイ、スケーリング、管理を行います。

主要なKubernetesリソース

リソース説明用途
Pod1つ以上のコンテナのグループKubernetesの最小デプロイ単位
DeploymentPodのレプリカセットを管理アプリケーションのデプロイと更新
ServicePodへのネットワークアクセスを提供ロードバランシング、サービスディスカバリ
Ingress外部からのHTTP/HTTPSアクセスを管理URLベースのルーティング
ConfigMap設定データの保存環境変数、設定ファイル
Secret機密データの保存パスワード、APIキー、証明書
実習 8-2: Kubernetesでアプリケーションをデプロイ
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: LoadBalancer
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
# GKEクラスターの作成
gcloud container clusters create my-cluster \
    --num-nodes=3 \
    --zone=asia-northeast1-a

# 認証情報の取得
gcloud container clusters get-credentials my-cluster --zone=asia-northeast1-a

# デプロイ
kubectl apply -f deployment.yaml

# 確認
kubectl get deployments
kubectl get pods
kubectl get services

8.7 サーバーレス vs コンテナ

観点サーバーレスコンテナ
適している用途イベント駆動、短時間処理、スパイク的トラフィック長時間実行、複雑なアプリ、ステートフル
起動時間コールドスタートあり（数百ms〜数秒）非常に高速（数秒）
実行時間制限あり（15分程度）なし
コスト実行時のみ課金稼働時間で課金
スケーリング完全自動、瞬時自動だが設定が必要
状態管理ステートレスステートフル可能

8.8 本章のまとめ

重要ポイント
  • サーバーレスはサーバー管理不要で、使用した分だけ課金される
  • Lambda、Azure Functions、Cloud Functionsで関数ベースの開発が可能
  • API Gatewayでサーバーレスアプリケーションを公開
  • コンテナは軽量で起動が速く、高いポータビリティを持つ
  • ECS、AKS、GKEでマネージドコンテナ運用が可能
  • Kubernetesはコンテナオーケストレーションの標準
  • サーバーレスとコンテナは用途に応じて使い分ける
