2.1 3大クラウドプロバイダーの概要と市場動向

クラウド市場は現在、AWS、Microsoft Azure、Google Cloud Platform（GCP）の3社が大きなシェアを占めています。これらを「3大クラウドプロバイダー」と呼び、それぞれ異なる強みと特徴を持っています。

市場シェアと成長率（2024年時点）

pie title クラウド市場シェア（2024年第3四半期） "AWS" : 32 "Azure" : 23 "GCP" : 11 "その他" : 34
プロバイダー 市場シェア サービス開始 強み
AWS 約32%（第1位） 2006年 最多のサービス数、最大のグローバル展開
Azure 約23%（第2位） 2010年 Microsoftエコシステムとの統合
GCP 約11%（第3位） 2008年 データ分析・AI、コンテナ技術

2.2 AWS（Amazon Web Services）の詳細

AWS - クラウドのパイオニア

歴史と背景

AWSは2006年にAmazon EC2とS3を提供開始し、現代的なクラウドコンピューティングの先駆者となりました。元々はAmazon.comの内部インフラを外部に提供したことが始まりです。

主な特徴
  • サービス数: 200以上のサービスを提供（最多）
  • グローバル展開: 33のリージョン、105のアベイラビリティゾーン（2024年）
  • 顧客層: スタートアップから大企業まで幅広い（Netflix、Airbnb、NASAなど）
  • エコシステム: 最大のコミュニティとサードパーティツール
強みと適している用途
  • 豊富なサービスラインナップで、ほぼすべてのニーズに対応可能
  • 最も成熟したドキュメント、トレーニング、認定プログラム
  • グローバル展開が必要なサービスに最適
  • サーバーレス（Lambda）、コンテナ（ECS/EKS）が充実
代表的なサービス
カテゴリ サービス名 概要
コンピューティング EC2 仮想サーバー
ストレージ S3 オブジェクトストレージ
データベース RDS リレーショナルデータベース
サーバーレス Lambda 関数実行環境
コンテナ ECS/EKS コンテナオーケストレーション
料金体系の特徴
  • 従量課金: 使った分だけ支払い（秒単位課金も多い）
  • リザーブドインスタンス: 1年〜3年契約で最大72%割引
  • スポットインスタンス: オンデマンドの最大90%割引（中断可能性あり）
  • 無料利用枠: 12ヶ月間の期限付き無料枠 + 永続的な無料枠

2.3 Microsoft Azureの詳細

Azure - エンタープライズ統合の強み

歴史と背景

Azureは2010年に「Windows Azure」として開始され、後にMicrosoft Azureに改名されました。Microsoftのエコシステムとの緊密な統合が最大の特徴です。

主な特徴
  • エンタープライズ統合: Windows Server、Active Directory、Microsoft 365との親和性
  • ハイブリッドクラウド: Azure Arc、Azure Stackでオンプレミスとの統合が容易
  • グローバル展開: 60以上のリージョン（最多）
  • 顧客層: 大企業・公共機関に強い（Fortune 500の95%が利用）
強みと適している用途
  • 既にMicrosoft製品を使用している企業に最適
  • Active Directoryとのシームレスな統合
  • .NET開発に最適な環境
  • ハイブリッドクラウド戦略を採用する企業向け
  • エンタープライズ向けサポート・セキュリティ・コンプライアンス
代表的なサービス
カテゴリ サービス名 概要
コンピューティング Virtual Machines 仮想サーバー
ストレージ Blob Storage オブジェクトストレージ
データベース Azure SQL Database マネージドSQL Server
サーバーレス Azure Functions 関数実行環境
コンテナ AKS Kubernetesサービス
料金体系の特徴
  • 従量課金: 使用量に応じた課金
  • 予約インスタンス: 1年〜3年契約で割引
  • ハイブリッド特典: 既存のWindowsライセンスを持ち込み可能
  • 無料アカウント: $200分のクレジット（30日間） + 12ヶ月無料サービス

2.4 Google Cloud Platform（GCP）の詳細

GCP - データとAIの最先端

歴史と背景

GCPは2008年にGoogle App Engineからスタートし、Googleが自社サービス（検索、YouTube、Gmail等）で使用している技術を提供しています。

主な特徴
  • データ分析・AI: BigQuery、Vertex AI、TensorFlowなど業界をリード
  • コンテナ技術: Kubernetesの開発元、GKEが高性能
  • ネットワーク: Googleの高速グローバルネットワーク
  • イノベーション: 最先端技術の早期導入
強みと適している用途
  • ビッグデータ分析が必要なプロジェクト（BigQueryは他社より高速・安価）
  • 機械学習・AIを活用したアプリケーション
  • Kubernetesを使ったコンテナアプリケーション
  • データサイエンティスト・エンジニア向け
  • シンプルで使いやすいUI・UX
代表的なサービス
カテゴリ サービス名 概要
コンピューティング Compute Engine 仮想サーバー
ストレージ Cloud Storage オブジェクトストレージ
データベース Cloud SQL リレーショナルデータベース
データ分析 BigQuery ビッグデータ分析
コンテナ GKE Kubernetesサービス
料金体系の特徴
  • 従量課金: 使用量に応じた課金
  • 継続利用割引: 自動的に割引適用（月間使用率に応じて最大30%）
  • コミット利用割引: 1年〜3年契約で割引
  • 無料トライアル: $300分のクレジット（90日間） + Always Free（永続無料枠）

2.6 無料利用枠と無料トライアルの比較

学習やテストのために、各クラウドプロバイダーは無料利用枠を提供しています。これらを活用することで、コストをかけずにクラウドを体験できます。

項目 AWS無料利用枠 Azure無料アカウント GCP無料トライアル
初回クレジット なし $200（30日間） $300（90日間）
期限付き無料枠 12ヶ月間
（EC2: 750時間/月、S3: 5GB等）		 12ヶ月間
（VM: 750時間/月、Blob: 5GB等）		 12ヶ月間
（一部サービス）
永続的な無料枠 あり
（Lambda: 100万リクエスト/月、DynamoDB: 25GB等）		 一部サービスで提供 Always Free
（Compute Engine: f1-micro 1台等）
クレジットカード 必須 必須 必須
自動課金 無料枠超過時に課金 クレジット消費後に課金 手動でアップグレードしない限り課金されない
無料枠使用時の重要な注意点
  • クレジットカード登録が必須: 無料枠でもカード情報の登録が必要です
  • 無料枠を超えると課金される: リソース使用量に注意し、予算アラートを設定しましょう
  • リソースの削除を忘れない: 使用後は必ずリソースを停止・削除してください
  • リージョンに注意: 一部のサービスは特定リージョンでのみ無料枠が適用されます
  • 期限を把握する: 12ヶ月の期限がある無料枠は、期限後に自動的に有料に切り替わります

2.7 AWSアカウントの作成手順

それでは実際にAWSアカウントを作成し、基本的なセキュリティ設定を行いましょう。

実習 2-1: AWSアカウントの作成とセキュリティ設定
手順1: AWSアカウントの作成
  1. AWS公式サイト（https://aws.amazon.com/jp/）にアクセス
  2. 「無料でアカウント作成」をクリック
  3. Eメールアドレス、アカウント名を入力
  4. 連絡先情報（氏名、住所、電話番号）を入力
  5. クレジットカード情報を入力（本人確認のため）
  6. 電話による本人確認（SMSまたは音声通話）
  7. サポートプランを選択（無料の「ベーシックサポート」を選択）
手順2: ルートアカウントの保護（MFA有効化）

作成したアカウントは「ルートアカウント」と呼ばれ、すべての権限を持つため、保護が重要です。

  1. AWSマネジメントコンソールにサインイン
  2. 右上のアカウント名をクリック → 「セキュリティ認証情報」を選択
  3. 「多要素認証（MFA）」セクションで「MFAの有効化」をクリック
  4. 「仮想MFAデバイス」を選択（スマートフォンアプリを使用）
  5. Google AuthenticatorやMicrosoft Authenticatorアプリをインストール
  6. QRコードをスキャンし、2つの連続したMFAコードを入力
手順3: IAMユーザーの作成

日常的な操作にはルートアカウントを使わず、IAMユーザーを作成します。

  1. AWSマネジメントコンソールで「IAM」を検索
  2. 左メニューから「ユーザー」→「ユーザーを追加」をクリック
  3. ユーザー名を入力（例: admin-user）
  4. 「AWSマネジメントコンソールへのアクセスを提供する」を選択
  5. 「カスタムパスワード」を設定
  6. 「アクセス許可の設定」で「ポリシーを直接アタッチ」を選択
  7. 「AdministratorAccess」ポリシーを選択（学習用）
  8. ユーザーを作成し、サインインURLをメモ
手順4: 請求アラートの設定
  1. AWSマネジメントコンソールで「Billing」を検索
  2. 「請求設定」→「請求アラートを受け取る」を有効化
  3. CloudWatchで予算アラートを作成（例: $10を超えたら通知）
期待される結果
  • AWSアカウントが作成され、MFAが有効化されている
  • 日常使用するIAMユーザーが作成されている
  • 請求アラートが設定され、予期しない課金を防げる
ベストプラクティス: ルートアカウントの使用を最小限に

ルートアカウントはすべての権限を持つため、以下の場合のみ使用し、日常的にはIAMユーザーを使いましょう：

  • IAMユーザーの初回作成時
  • アカウントの請求情報の変更
  • アカウントの削除
  • サポートプランの変更

2.8 Azureアカウントの作成手順

実習 2-2: Azure無料アカウントの作成
手順1: Azureアカウントの作成
  1. Azure公式サイト（https://azure.microsoft.com/ja-jp/）にアクセス
  2. 「無料で始める」をクリック
  3. Microsoftアカウントでサインイン（なければ作成）
  4. 電話番号による本人確認
  5. クレジットカード情報を入力（本人確認のため、$200クレジット期間中は課金されない）
  6. 契約に同意し、「サインアップ」をクリック
手順2: Azureポータルへのアクセス
  1. Azure Portal（https://portal.azure.com/）にアクセス
  2. 作成したMicrosoftアカウントでサインイン
  3. ダッシュボードが表示されることを確認
手順3: サブスクリプションとリソースグループの理解

Azureでは、リソースは以下の階層構造で管理されます：

graph TD A[管理グループ] --> B[サブスクリプション] B --> C[リソースグループ] C --> D[リソース
VM、Storage等]
  • サブスクリプション: 請求の単位。無料アカウントでは「無料試用版」サブスクリプションが作成される
  • リソースグループ: 関連するリソースをまとめるコンテナ（例: プロジェクトごとにグループ化）
  • リソース: 仮想マシン、ストレージなどの個別のサービス
手順4: リソースグループの作成
  1. Azure Portalで「リソースグループ」を検索
  2. 「+ 作成」をクリック
  3. サブスクリプションを選択
  4. リソースグループ名を入力（例: learning-rg）
  5. リージョンを選択（例: Japan East）
  6. 「確認および作成」→「作成」をクリック

2.9 GCPアカウントの作成手順

実習 2-3: GCP無料トライアルの開始
手順1: GCPアカウントの作成
  1. Google Cloud公式サイト（https://cloud.google.com/）にアクセス
  2. 「無料で開始」をクリック
  3. Googleアカウントでサインイン（なければ作成）
  4. 国と利用規約に同意
  5. アカウントタイプを選択（「個人」または「ビジネス」）
  6. クレジットカード情報を入力（手動アップグレードまで課金されない）
  7. 「無料トライアルを開始」をクリック
手順2: Google Cloud Consoleへのアクセス
  1. Google Cloud Console（https://console.cloud.google.com/）にアクセス
  2. 「My First Project」が自動的に作成される
手順3: プロジェクトの理解と作成

GCPでは、リソースは「プロジェクト」という単位で管理されます。プロジェクトは請求、権限、リソースの境界となります。

graph TD A[組織] --> B[フォルダ] B --> C[プロジェクト] C --> D[リソース
VM、Storage等]
手順4: 新しいプロジェクトの作成
  1. Google Cloud Consoleの上部にあるプロジェクト選択ドロップダウンをクリック
  2. 「新しいプロジェクト」をクリック
  3. プロジェクト名を入力（例: cloud-learning）
  4. 「作成」をクリック
  5. 作成されたプロジェクトに切り替え
手順5: 課金アラートの設定
  1. 左メニューから「お支払い」を選択
  2. 「予算とアラート」→「予算を作成」をクリック
  3. 予算額を設定（例: $50）
  4. アラートのしきい値を設定（例: 50%、90%、100%）
  5. メール通知先を設定
GCPの自動課金について

GCPの無料トライアルは、手動でアップグレードしない限り自動的に有料アカウントに移行しません。これはAWSやAzureと異なる点で、より安全です。ただし、無料枠を超えて使用すると、サービスが停止する可能性があるため注意が必要です。

2.10 CLIツールの紹介と基本操作

クラウドの操作には、Webコンソールだけでなく、コマンドラインインターフェース（CLI）も使用できます。CLIは自動化やスクリプト作成に便利です。

AWS CLI

インストール方法（例: macOS/Linux）:

# AWS CLIのインストール（pip使用）
pip install awscli

# インストール確認
aws --version

# 認証情報の設定
aws configure
# AWS Access Key ID: （IAMユーザーのアクセスキー）
# AWS Secret Access Key: （IAMユーザーのシークレットキー）
# Default region name: ap-northeast-1（東京リージョン）
# Default output format: json

# 動作確認
aws s3 ls  # S3バケット一覧を表示

Azure CLI

# Azure CLIのインストール（macOS）
brew install azure-cli

# インストール確認
az --version

# ログイン
az login  # ブラウザが開いてログイン

# サブスクリプション確認
az account list

# 動作確認
az group list  # リソースグループ一覧を表示

Google Cloud CLI（gcloud）

# Google Cloud SDKのインストール
# https://cloud.google.com/sdk/docs/install からダウンロード

# インストール確認
gcloud --version

# 認証
gcloud auth login  # ブラウザが開いてログイン

# プロジェクトの設定
gcloud config set project プロジェクトID

# 動作確認
gcloud compute instances list  # VMインスタンス一覧を表示
CLIツールのメリット
  • 自動化: スクリプトで繰り返し作業を自動化
  • 効率性: Webコンソールより高速な操作
  • Infrastructure as Code: コードでインフラを管理
  • バッチ処理: 複数のリソースを一括操作
理解度確認クイズ
  1. AWS、Azure、GCPの主な強みをそれぞれ1つずつ挙げてください。
  2. AWSの無料利用枠とGCPの無料トライアルの違いは何ですか？
  3. ルートアカウントとIAMユーザーの違いは何ですか？なぜIAMユーザーを作成すべきですか？
  4. MFA（多要素認証）はなぜ重要ですか？どのように機能しますか？
  5. Azureの「リソースグループ」とは何ですか？その目的を説明してください。
  6. GCPの「プロジェクト」とは何ですか？その役割を説明してください。
  7. CLIツール（AWS CLI、Azure CLI、gcloud）を使用するメリットを3つ挙げてください。
  8. 無料枠を使用する際の注意点を3つ挙げてください。

2.11 本章のまとめ

重要ポイント
  • AWS、Azure、GCPはそれぞれ異なる強みを持つ（AWS: 豊富なサービス、Azure: Microsoft統合、GCP: データ/AI）
  • 各プロバイダーは無料利用枠を提供しており、学習に活用できる
  • ルートアカウントは保護し、日常的にはIAMユーザーを使用する
  • MFAの有効化はセキュリティの基本
  • 予算アラートの設定で予期しない課金を防ぐ
  • CLIツールは自動化と効率化に不可欠
  • Azureはリソースグループ、GCPはプロジェクトでリソースを管理する
