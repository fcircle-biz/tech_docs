8.1 オブザーバビリティとは

オブザーバビリティ（Observability）は、システムの内部状態を外部から理解できる度合いを表します。モニタリングがあらかじめ定義された問題を検出するのに対し、オブザーバビリティは未知の問題も発見できます。

モニタリング vs オブザーバビリティ

4つのゴールデンシグナル

Google SREが提唱する、システム監視の4つの重要なメトリクスです。

4つのゴールデンシグナル
  1. レイテンシ（Latency）: リクエストの応答時間
  2. トラフィック（Traffic）: システムへのリクエスト数
  3. エラー（Errors）: 失敗したリクエストの割合
  4. 飽和度（Saturation）: リソースの使用率（CPU、メモリ、ディスク）

8.2 PrometheusとGrafana

Prometheusとは

Prometheusは、時系列データベースを内蔵したオープンソースのモニタリングシステムです。Pull型でメトリクスを収集します。

flowchart LR A["アプリケーション
（Exporter）"] -->|メトリクス公開| B["Prometheus
（定期的にScrape）"] B --> C["時系列DB"] C --> D["Grafana
（可視化）"] C --> E["AlertManager
（アラート）"]

Prometheusのインストールと設定

# docker-compose.yml
version: '3.8'

services:
  prometheus:
    image: prom/prometheus:latest
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus_data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    volumes:
      - grafana_data:/var/lib/grafana
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin

  node-exporter:
    image: prom/node-exporter:latest
    ports:
      - "9100:9100"

volumes:
  prometheus_data:
  grafana_data:
# prometheus.yml
global:
  scrape_interval: 15s  # 15秒ごとにメトリクスを収集

scrape_configs:
  # Prometheus自体のメトリクス
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  # Node Exporter（システムメトリクス）
  - job_name: 'node'
    static_configs:
      - targets: ['node-exporter:9100']

  # アプリケーションのメトリクス
  - job_name: 'myapp'
    static_configs:
      - targets: ['myapp:8080']

PromQL（Prometheusクエリ言語）

# CPU使用率
100 - (avg by (instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# メモリ使用率
(1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100

# HTTPリクエスト率（毎秒）
rate(http_requests_total[5m])

# エラー率
rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m]) * 100

# レスポンスタイムのP95
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))

Grafanaダッシュボード

Grafanaは、Prometheusのメトリクスを視覚化するダッシュボードツールです。

Grafanaダッシュボードの作成手順
  1. Grafanaにアクセス（http://localhost:3000）
  2. データソースとしてPrometheusを追加
  3. 新しいダッシュボードを作成
  4. パネルを追加してPromQLクエリを設定
  5. グラフタイプを選択（折れ線、棒グラフ、ゲージ等）
  6. アラートルールを設定（オプション）

8.3 ロギング戦略

集中ログ管理の重要性

マイクロサービスや分散システムでは、複数のサービスからログを集中管理し、検索・分析できることが重要です。

ELKスタック（Elasticsearch, Logstash, Kibana）

# docker-compose.yml（ELKスタック）
version: '3.8'

services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.10.0
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
    ports:
      - "9200:9200"

  logstash:
    image: docker.elastic.co/logstash/logstash:8.10.0
    ports:
      - "5000:5000"
    volumes:
      - ./logstash.conf:/usr/share/logstash/pipeline/logstash.conf
    depends_on:
      - elasticsearch

  kibana:
    image: docker.elastic.co/kibana/kibana:8.10.0
    ports:
      - "5601:5601"
    depends_on:
      - elasticsearch

構造化ログ

JSON形式の構造化ログにより、検索・分析が容易になります。

// 構造化ログの例（Node.js + Winston）
const winston = require('winston');

const logger = winston.createLogger({
  format: winston.format.json(),
  transports: [
    new winston.transports.Console(),
    new winston.transports.File({ filename: 'app.log' })
  ]
});

// ログ出力
logger.info('User login', {
  user_id: 12345,
  ip_address: '192.168.1.1',
  timestamp: new Date().toISOString(),
  action: 'login'
});

// 出力（JSON）
// {"level":"info","message":"User login","user_id":12345,"ip_address":"192.168.1.1","timestamp":"2025-01-15T10:30:00.000Z","action":"login"}

分散トレーシング（Jaeger）

分散システムでのリクエストフローを追跡します。

// OpenTelemetryを使った分散トレーシング（Node.js）
const { trace } = require('@opentelemetry/api');
const tracer = trace.getTracer('my-service');

async function processOrder(orderId) {
  const span = tracer.startSpan('process-order');
  
  try {
    span.setAttribute('order.id', orderId);
    
    // データベース操作
    await database.getOrder(orderId);
    
    // 外部API呼び出し
    await paymentService.charge(orderId);
    
    span.setStatus({ code: SpanStatusCode.OK });
  } catch (error) {
    span.recordException(error);
    span.setStatus({ code: SpanStatusCode.ERROR });
  } finally {
    span.end();
  }
}

理解度確認クイズ
  1. Q: オブザーバビリティの3本柱は何ですか？
    A: メトリクス（Metrics）、ログ（Logs）、トレース（Traces）の3つです。
  2. Q: 4つのゴールデンシグナルを挙げてください。
    A: レイテンシ（応答時間）、トラフィック（リクエスト数）、エラー（失敗率）、飽和度（リソース使用率）です。
  3. Q: PrometheusとGrafanaの役割の違いは何ですか？
    A: Prometheusはメトリクスの収集・保存を行い、Grafanaはそのデータを可視化してダッシュボードを作成します。
  4. Q: 構造化ログのメリットは何ですか？
    A: JSON形式で出力することで、検索・フィルタリング・集計が容易になり、機械的な処理（自動アラート等）がしやすくなります。
  5. Q: 分散トレーシングの目的は何ですか？
    A: マイクロサービスなど分散システムでのリクエストの流れを追跡し、どのサービスでボトルネックやエラーが発生しているかを特定するためです。
