MySQLの環境構築 - インストールとMySQL Workbenchの使い方

MySQLのダウンロードからインストール、初期設定までを画面キャプチャ付きで丁寧に解説します。MySQL Workbenchの基本操作（サーバー接続、データベース作成、SQLエディタの使い方）を習得します。

この章で学ぶこと

  • MySQLのダウンロードとインストール方法（Windows/Mac）
  • MySQLの初期設定とrootパスワードの設定
  • MySQL Workbenchのインストールと基本操作
  • データベース作成、SQLエディタでのクエリ実行

2.1 MySQLをダウンロードしよう

まず、MySQL公式サイトからMySQLをダウンロードします。MySQLには大きく分けて「Community Edition（無料）」と「Commercial Edition（有料）」がありますが、学習用途ではCommunity Editionで十分です。

ダウンロードページへのアクセス

MySQL公式ダウンロードページ: https://dev.mysql.com/downloads/mysql/

このページから、Windows、Mac、Linux用のインストーラーをダウンロードできます。ご自分のOS（オペレーティングシステム）に合ったものを選択してください。

にアクセス] --> B{OSを選択} B -->|Windows| C[Windows
インストーラー] B -->|Mac| D[macOS
DMGファイル] B -->|Linux| E[Linux
パッケージ] C --> F[ダウンロード完了] D --> F E --> F

ポイント

ダウンロード時、Oracleアカウントのログインを求められることがありますが、ページ下部の「No thanks, just start my download.」をクリックすれば、アカウント登録なしでダウンロードできます。

2.2 MySQLをインストールしよう

ダウンロードしたインストーラーを実行して、MySQLをインストールします。WindowsとMacでは手順が若干異なりますが、基本的な流れは同じです。

Windowsの場合

  1. ダウンロードしたmysql-installer-web-community-x.x.xx.msiをダブルクリック
  2. 「Choosing a Setup Type」で「Developer Default」を選択（開発に必要なツールがすべてインストールされます）
  3. 「Check Requirements」で不足しているコンポーネントがあれば「Execute」でインストール
  4. 「Installation」画面で「Execute」をクリックし、インストールを開始
  5. 「Product Configuration」でMySQLサーバーの設定を行う（次のセクションで詳細解説）

Macの場合

  1. ダウンロードしたmysql-x.x.xx-macos-xxx.dmgをダブルクリック
  2. DMGファイルが開いたら、パッケージファイル（.pkg）をダブルクリック
  3. インストーラーの指示に従い、「続ける」をクリック
  4. 「インストール」をクリックし、管理者パスワードを入力
  5. 初期設定で「Use Strong Password Encryption」を選択
  6. rootユーザーのパスワードを設定（後で使うので必ず覚えておきましょう）

注意

rootパスワードは、MySQLの管理者アカウントのパスワードです。忘れるとデータベースにアクセスできなくなるため、必ずメモしておきましょう。開発環境では「root123」など簡単なパスワードでも構いませんが、本番環境では強力なパスワードを設定してください。

2.3 MySQLの初期設定

インストールが完了したら、MySQLサーバーの初期設定を行います。主に以下の項目を設定します。

主な設定項目

  • Type and Networking - デフォルトのポート番号（3306）を確認
  • Authentication Method - 「Use Strong Password Encryption」を選択（推奨）
  • Accounts and Roles - rootパスワードを設定
  • Windows Service - Windowsサービスとして登録（自動起動）

設定内容の詳細

1. Type and Networking

MySQLサーバーが使用するポート番号を設定します。デフォルトは3306で、通常はそのままで問題ありません。「Open Windows Firewall port for network access」にチェックを入れると、外部からの接続を許可します（学習用途では不要）。

2. Authentication Method

パスワードの暗号化方式を選択します。「Use Strong Password Encryption」を選択すると、MySQL 8.0以降の強力な暗号化方式（caching_sha2_password）が使われます。古いアプリケーションとの互換性が必要な場合のみ「Use Legacy Authentication Method」を選択します。

3. Accounts and Roles

rootユーザーのパスワードを設定します。また、必要に応じて新しいユーザーアカウントを作成することもできます。学習段階では、rootユーザーだけで十分です。

2.4 MySQLが正しくインストールされたか確認しよう

MySQLが正しくインストールされ、サーバーが起動しているかを確認しましょう。コマンドラインから確認する方法を紹介します。

Windowsの場合

コマンドプロンプト 
# MySQLのバージョンを確認
mysql --version

# MySQLサーバーに接続（パスワード入力を求められます）
mysql -u root -p

Macの場合

ターミナル 
# MySQLのバージョンを確認
mysql --version

# MySQLサーバーに接続（パスワード入力を求められます）
mysql -u root -p

動作確認

正しくインストールされていれば、以下のような出力が表示されます。

期待される結果 

$ mysql --version
mysql  Ver 8.0.35 for macos13 on arm64 (MySQL Community Server - GPL)

$ mysql -u root -p
Enter password: ******* （rootパスワードを入力）
Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 8
Server version: 8.0.35 MySQL Community Server - GPL

Type 'help;' or '\h' for help. Type '\c' to clear the current input statement.

mysql>

ヒント

mysql> プロンプトが表示されれば、MySQLサーバーに正常に接続できています。exit; または quit; と入力すると、MySQLから抜けることができます。

2.5 MySQL Workbenchをインストールしよう

MySQL WorkbenchはMySQLの公式GUIツールです。コマンドラインよりも視覚的にデータベースを操作でき、テーブル設計、SQLクエリ実行、データのインポート/エクスポートなど、さまざまな作業を効率的に行えます。

MySQL Workbenchの特徴

  • ビジュアルDB設計 - ER図を描きながらテーブル設計ができる
  • SQLエディタ - シンタックスハイライト、自動補完機能付き
  • データ管理 - GUIでデータの追加・編集・削除が可能
  • サーバー管理 - ユーザー管理、バックアップ、パフォーマンス監視など

ダウンロードとインストール

MySQL Workbench公式ダウンロードページ: https://dev.mysql.com/downloads/workbench/

  1. 上記のページから、ご自分のOS用のインストーラーをダウンロード
  2. インストーラーを実行し、画面の指示に従ってインストール
  3. インストールが完了したら、MySQL Workbenchを起動

ポイント

Windowsの場合、MySQLインストーラーで「Developer Default」を選択していれば、MySQL Workbenchも同時にインストールされています。その場合は、別途ダウンロードする必要はありません。

2.6 MySQL Workbenchの基本操作

MySQL Workbenchを起動したら、まずMySQLサーバーに接続しましょう。以下の手順で接続を設定します。

サーバーへの接続

  1. MySQL Workbenchを起動すると、ホーム画面が表示されます
  2. 「MySQL Connections」セクションに、デフォルトで「Local instance MySQL」などの接続が表示されています
  3. この接続をクリックすると、パスワード入力ダイアログが表示されます
  4. インストール時に設定したrootパスワードを入力し、「OK」をクリック
  5. 接続が成功すると、SQLエディタ画面が開きます

ヒント

「Store password in vault」または「Store password in keychain」にチェックを入れると、次回からパスワード入力を省略できます。ただし、セキュリティを重視する場合はチェックを外しておきましょう。

データベースの作成

接続が成功したら、実際にデータベースを作成してみましょう。MySQL Workbenchでは、GUIまたはSQLコマンドの両方でデータベースを作成できます。

方法1: GUIで作成

  1. 左側の「Navigator」パネルで「Schemas」タブを選択
  2. 空白部分を右クリックし、「Create Schema...」を選択
  3. 「Name」にtest_dbと入力
  4. 「Charset/Collation」はutf8mb4 / utf8mb4_general_ciを選択（日本語対応）
  5. 「Apply」をクリックすると、SQLプレビューが表示されます
  6. 「Apply」をもう一度クリックして実行

方法2: SQLコマンドで作成

SQL 
-- データベースの作成
CREATE DATABASE test_db
CHARACTER SET utf8mb4
COLLATE utf8mb4_general_ci;

-- 作成されたデータベースの確認
SHOW DATABASES;

SQLエディタ（中央のテキストエリア）に上記のコマンドを入力し、「⚡」アイコン（Execute）をクリックするか、Ctrl+Enter（Mac: Cmd+Enter）を押すと実行されます。

2.7 SQLエディタの使い方

MySQL WorkbenchのSQLエディタは、SQLクエリを記述・実行するための強力な機能を備えています。ここでは基本的な使い方を学びましょう。

SQLエディタの主な機能

  • シンタックスハイライト - SQLキーワードが色分けされて見やすい
  • 自動補完 - テーブル名やカラム名を途中まで入力すると候補が表示される
  • 複数クエリ実行 - 複数のSQL文をまとめて実行できる
  • クエリ履歴 - 実行したクエリの履歴を確認・再実行できる
  • 結果表示 - クエリ実行結果が下部のパネルに表示される

実際にクエリを実行してみよう

作成したtest_dbデータベースに、簡単なテーブルを作成し、データを挿入・取得してみましょう。

SQL 
-- データベースを選択
USE test_db;

-- テーブルの作成
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- データの挿入
INSERT INTO users (name, email) VALUES
('山田太郎', 'yamada@example.com'),
('佐藤花子', 'sato@example.com'),
('鈴木一郎', 'suzuki@example.com');

-- データの取得
SELECT * FROM users;

ヒント

SQLエディタで特定のSQL文だけを実行したい場合は、そのSQL文を選択（ドラッグ）してから「⚡」アイコンをクリックするか、Ctrl+Shift+Enter（Mac: Cmd+Shift+Enter）を押します。

動作確認

上記のクエリを実行すると、以下のような結果が表示されます。

期待される結果

| id | name         | email                 | created_at          |

|  1 | 山田太郎      | yamada@example.com    | 2025-01-15 10:30:00 |
|  2 | 佐藤花子      | sato@example.com      | 2025-01-15 10:30:00 |
|  3 | 鈴木一郎      | suzuki@example.com    | 2025-01-15 10:30:00 |

実習 2-1: 自分用のデータベースを作ろう

MySQL Workbenchを使って、自分用のデータベースを作成し、簡単なテーブルを作ってデータを登録してみましょう。

手順

  1. 1 my_first_dbという名前のデータベースを作成する（文字コード: utf8mb4）
  2. 2 productsテーブルを作成する（カラム: id, product_name, price, stock）
  3. 3 3件以上の商品データを挿入する
  4. 4 SELECT * FROM products;で全データを取得し、結果を確認する

トラブルシューティング

「Access denied for user 'root'@'localhost'」エラーが出る

原因: パスワードが間違っている、または設定されていません。

解決方法: インストール時に設定したrootパスワードを再確認してください。パスワードを忘れた場合は、MySQLを再インストールするか、rootパスワードをリセットする必要があります。

MySQL Workbenchで接続が「Can't connect to MySQL server」エラーになる

原因: MySQLサーバーが起動していません。

解決方法: Windowsの場合は「サービス」アプリでMySQLサービスを開始してください。Macの場合は「システム環境設定」→「MySQL」でサーバーを起動します。

日本語が文字化けする

原因: データベースまたはテーブルの文字コードがutf8mb4に設定されていません。

解決方法: データベース作成時にCHARACTER SET utf8mb4 COLLATE utf8mb4_general_ciを指定してください。既存のデータベースの場合はALTER DATABASE test_db CHARACTER SET utf8mb4;で変更できます。

理解度確認クイズ

  1. Q1. MySQLのデフォルトのポート番号は何番ですか?

    回答を表示

    3306番です。MySQLサーバーはデフォルトでポート3306を使用してクライアントからの接続を待ち受けます。

  2. Q2. MySQL Workbenchでデータベースを作成するGUI操作の手順を説明してください。

    回答を表示

    ①左側の「Navigator」パネルで「Schemas」タブを選択、②空白部分を右クリックして「Create Schema...」を選択、③データベース名と文字コード（utf8mb4推奨）を設定、④「Apply」をクリックして実行します。

  3. Q3. 日本語を正しく扱うために推奨される文字コードは何ですか?

    回答を表示

    utf8mb4が推奨されます。従来のutf8は一部の文字（絵文字など4バイト文字）をサポートしていませんが、utf8mb4はすべてのUnicode文字をサポートしています。

  • データベース作成時は文字コードutf8mb4を指定すると日本語や絵文字を正しく扱える
