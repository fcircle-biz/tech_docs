実践！MySQLでWebアプリのDBを設計しよう

これまで学んだ知識を活用して、実際のWebアプリケーション（ECサイトやブログシステム）を想定したデータベースを設計・構築します。テーブル設計、インデックス設計、文字コード設定、ビュー作成までを総合的に実践します。

この章で学ぶこと

  • ECサイトを想定したテーブル設計（ユーザー、商品、注文、レビューなど）
  • 適切なデータ型・制約・インデックスの選定
  • 文字コード・照合順序の設定
  • ビューとストアドプロシージャを活用した運用

12.1 要件定義と設計の流れ

実際のWebアプリケーション開発では、まず「どんな機能が必要か」を整理してからデータベース設計に入ります。今回は、シンプルなECサイトを想定して設計してみましょう。

ECサイトに必要な主な機能

  • 会員管理 - ユーザー登録、ログイン、プロフィール編集
  • 商品管理 - 商品一覧、商品詳細、在庫管理
  • カート機能 - カート追加、数量変更、削除
  • 注文機能 - 注文確定、注文履歴、注文状態管理
  • レビュー機能 - 商品レビュー投稿、評価

12.2 テーブル設計（エンティティ定義）

ECサイトに必要なエンティティ（テーブル）を洗い出します。今回は以下の5つのテーブルを作成します。

erDiagram USERS ||--o{ ORDERS : places USERS ||--o{ REVIEWS : writes USERS ||--o{ CART_ITEMS : has PRODUCTS ||--o{ CART_ITEMS : contains PRODUCTS ||--o{ ORDER_ITEMS : includes PRODUCTS ||--o{ REVIEWS : receives ORDERS ||--o{ ORDER_ITEMS : contains USERS { INT user_id PK VARCHAR email VARCHAR password_hash VARCHAR name DATETIME created_at DATETIME updated_at } PRODUCTS { INT product_id PK VARCHAR name TEXT description DECIMAL price INT stock_quantity DATETIME created_at DATETIME updated_at } ORDERS { INT order_id PK INT user_id FK DECIMAL total_amount VARCHAR status DATETIME ordered_at DATETIME updated_at } ORDER_ITEMS { INT order_item_id PK INT order_id FK INT product_id FK INT quantity DECIMAL price_at_order } REVIEWS { INT review_id PK INT user_id FK INT product_id FK INT rating TEXT comment DATETIME created_at } CART_ITEMS { INT cart_item_id PK INT user_id FK INT product_id FK INT quantity DATETIME added_at }

テーブル概要

  • users - 会員情報（ユーザーID、メールアドレス、パスワード、名前など）
  • products - 商品情報（商品ID、商品名、説明、価格、在庫数など）
  • orders - 注文情報（注文ID、ユーザーID、合計金額、状態など）
  • order_items - 注文明細（注文ID、商品ID、数量、注文時の価格）
  • reviews - 商品レビュー（レビューID、ユーザーID、商品ID、評価、コメント）
  • cart_items - カート情報（ユーザーID、商品ID、数量、追加日時）

12.3 データベースとテーブルの作成

まず、文字コードをutf8mb4に設定したデータベースを作成し、各テーブルを定義していきます。

実習: データベースの作成

まず、ECサイト用のデータベースを作成します。

データベース作成（SQL） 
-- ECサイト用データベース作成
CREATE DATABASE ec_site
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_general_ci;

-- データベースを選択
USE ec_site;

usersテーブルの作成

usersテーブル作成（SQL） 
-- 会員情報テーブル
CREATE TABLE users (
    user_id INT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(100) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

productsテーブルの作成

productsテーブル作成（SQL） 
-- 商品情報テーブル
CREATE TABLE products (
    product_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    price DECIMAL(10, 2) NOT NULL,
    stock_quantity INT NOT NULL DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

ordersテーブルの作成

ordersテーブル作成（SQL） 
-- 注文情報テーブル
CREATE TABLE orders (
    order_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    total_amount DECIMAL(10, 2) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    ordered_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id),
    INDEX idx_user_id (user_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

order_itemsテーブルの作成

order_itemsテーブル作成（SQL） 
-- 注文明細テーブル
CREATE TABLE order_items (
    order_item_id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL,
    price_at_order DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (order_id) REFERENCES orders(order_id),
    FOREIGN KEY (product_id) REFERENCES products(product_id),
    INDEX idx_order_id (order_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

reviewsテーブルの作成

reviewsテーブル作成（SQL） 
-- 商品レビューテーブル
CREATE TABLE reviews (
    review_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    product_id INT NOT NULL,
    rating INT NOT NULL CHECK (rating BETWEEN 1 AND 5),
    comment TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id),
    FOREIGN KEY (product_id) REFERENCES products(product_id),
    INDEX idx_product_id (product_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

cart_itemsテーブルの作成

cart_itemsテーブル作成（SQL） 
-- カート情報テーブル
CREATE TABLE cart_items (
    cart_item_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL DEFAULT 1,
    added_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id),
    FOREIGN KEY (product_id) REFERENCES products(product_id),
    INDEX idx_user_id (user_id),
    UNIQUE KEY uk_user_product (user_id, product_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

12.5 ビューとストアドプロシージャの作成

複雑な集計や頻繁に使うクエリはビューやストアドプロシージャとして保存しておくと便利です。

ビューの作成例：商品レビュー一覧

ビュー作成（SQL） 
-- 商品レビュー一覧ビュー
CREATE VIEW product_reviews AS
SELECT
    p.product_id,
    p.name AS product_name,
    u.name AS reviewer_name,
    r.rating,
    r.comment,
    r.created_at
FROM
    reviews r
    JOIN users u ON r.user_id = u.user_id
    JOIN products p ON r.product_id = p.product_id
ORDER BY
    r.created_at DESC;

ストアドプロシージャの作成例：注文作成

ストアドプロシージャ作成（SQL） 
DELIMITER //

CREATE PROCEDURE create_order(IN p_user_id INT)
BEGIN
    DECLARE v_total DECIMAL(10, 2);
    DECLARE v_order_id INT;

    -- カート内の合計金額を計算
    SELECT SUM(p.price * c.quantity) INTO v_total
    FROM cart_items c
    JOIN products p ON c.product_id = p.product_id
    WHERE c.user_id = p_user_id;

    -- 注文レコードを作成
    INSERT INTO orders (user_id, total_amount) VALUES (p_user_id, v_total);
    SET v_order_id = LAST_INSERT_ID();

    -- カート内容を注文明細に移動
    INSERT INTO order_items (order_id, product_id, quantity, price_at_order)
    SELECT v_order_id, c.product_id, c.quantity, p.price
    FROM cart_items c
    JOIN products p ON c.product_id = p.product_id
    WHERE c.user_id = p_user_id;

    -- カートをクリア
    DELETE FROM cart_items WHERE user_id = p_user_id;
END //

DELIMITER ;

12.7 パフォーマンスチューニングのポイント

実際の運用では、アクセス数が増えた際のパフォーマンスも考慮する必要があります。

インデックス設計のポイント

  • ・WHERE句で頻繁に使われるカラムにはインデックスを設定
  • ・外部キーには必ずインデックスを設定（JOIN時に高速化）
  • ・複合インデックスは検索条件の組み合わせを考慮
  • ・EXPLAINコマンドで実行計画を確認
実行計画の確認（SQL） 
-- 実行計画を表示してインデックスが使われるか確認
EXPLAIN SELECT * FROM orders WHERE user_id = 1;

理解度確認クイズ

  1. Q1. ECサイトのテーブル設計で、注文時の商品価格を保存する理由は何ですか？

    回答を表示

    商品テーブルの価格は変動する可能性があるため、注文時の価格を別途保存しておかないと、過去の注文の金額が変わってしまいます。order_itemsテーブルのprice_at_orderカラムに注文時の価格を記録することで、注文履歴を正確に保持できます。

  2. Q2. ビューとテーブルの違いは何ですか？

    回答を表示

    テーブルは実際にデータを保存するのに対し、ビューは複数のテーブルを結合したクエリを「仮想的なテーブル」として保存したものです。ビュー自体はデータを持たず、参照時に元のテーブルからデータを取得して表示します。

  3. Q3. 外部キー制約を設定するメリットは何ですか？

    回答を表示

    外部キー制約により、存在しないユーザーIDや商品IDを誤って登録することを防げます。また、参照整合性が保たれるため、データの矛盾を防ぐことができます。例えば、注文が残っているユーザーを削除しようとするとエラーになり、データの一貫性が保たれます。

トラブルシューティング

外部キー制約エラーが出る

参照先のテーブルが存在しない、または参照先のカラムにデータがない可能性があります。先に親テーブル（users、productsなど）にデータを投入してから、子テーブル（orders、reviewsなど）にデータを追加しましょう。

ストアドプロシージャが作成できない

DELIMITERを設定せずにストアドプロシージャを作成しようとしている可能性があります。DELIMITER //を設定してから作成し、最後にDELIMITER ;で元に戻しましょう。
