MySQLの便利な関数 - 文字列・日付・数値の操作

MySQLが提供する豊富な組み込み関数を学習し、文字列の加工、日付の計算、数値の処理など実践的なデータ操作の方法を習得します。

この章で学ぶこと

  • 文字列関数（CONCAT、SUBSTRING、TRIMなど）の使い方
  • 日付・時刻関数（NOW、DATE_ADD、DATE_FORMATなど）の活用方法
  • 数値関数（ROUND、CEIL、FLOORなど）による計算処理
  • 集約関数（COUNT、SUM、AVGなど）の実践的な使用法

5.1 文字列関数 - テキストデータを自由に加工する

MySQLは文字列を操作するための豊富な関数を提供しています。データベースに格納されたテキストデータを加工・結合・抽出する際に、これらの関数を活用することで柔軟な処理が可能になります。

文字列の結合 - CONCAT関数

複数の文字列やカラムを結合する際に使用します。姓と名を結合してフルネームを作成したり、URLを動的に生成したりする場合に便利です。

文字列の結合 
-- 姓と名を結合してフルネームを作成
SELECT CONCAT(last_name, ' ', first_name) AS full_name
FROM employees;

-- カラムと固定文字列を結合
SELECT CONCAT('ID: ', employee_id, ' - ', last_name) AS employee_info
FROM employees;

-- NULL値の扱い（CONCAT_WS関数の方が安全）
SELECT CONCAT_WS(' ', last_name, first_name) AS full_name
FROM employees;
-- CONCAT_WSはNULL値を無視して結合してくれる

文字列の抽出 - SUBSTRING関数

文字列の一部を取り出す際に使用します。郵便番号の前半3桁を取得したり、電話番号の市外局番を抽出したりする場合に活用できます。

文字列の抽出 
-- 文字列の3文字目から5文字を取得
SELECT SUBSTRING('Hello World', 3, 5) AS result;
-- 結果: 'llo W'

-- 郵便番号の前半3桁を取得
SELECT SUBSTRING(postal_code, 1, 3) AS postal_prefix
FROM addresses;

-- 後ろから2文字を取得
SELECT SUBSTRING('Hello', -2) AS result;
-- 結果: 'lo'

空白の削除 - TRIM関数

文字列の前後の空白を削除する際に使用します。ユーザー入力データのクリーンアップに役立ちます。

空白の削除 
-- 前後の空白を削除
SELECT TRIM('  Hello World  ') AS result;
-- 結果: 'Hello World'

-- 前方の空白だけ削除
SELECT LTRIM('  Hello World') AS result;

-- 後方の空白だけ削除
SELECT RTRIM('Hello World  ') AS result;

大文字・小文字変換

文字列を大文字または小文字に統一する際に使用します。メールアドレスの正規化などに活用できます。

大文字・小文字変換 
-- 大文字に変換
SELECT UPPER('hello world') AS upper_text;
-- 結果: 'HELLO WORLD'

-- 小文字に変換
SELECT LOWER('HELLO WORLD') AS lower_text;
-- 結果: 'hello world'

-- メールアドレスを小文字に統一
SELECT LOWER(email) AS normalized_email
FROM users;

その他の便利な文字列関数

関数 説明 使用例
LENGTH 文字列のバイト数を取得 LENGTH('Hello')
CHAR_LENGTH 文字列の文字数を取得 CHAR_LENGTH('こんにちは')
REPLACE 文字列を置換 REPLACE('Hello', 'e', 'a')
REVERSE 文字列を逆順にする REVERSE('Hello')
LOCATE 部分文字列の位置を検索 LOCATE('lo', 'Hello')

5.2 日付・時刻関数 - 時間を自由に操る

MySQLは日付・時刻を扱うための強力な関数群を提供しています。現在時刻の取得、日付の加減算、フォーマット変換など、時間に関するあらゆる操作が可能です。

現在日時の取得

現在の日付や時刻を取得する関数は、データ登録時のタイムスタンプ記録などで頻繁に使用します。

現在日時の取得 
-- 現在の日付と時刻を取得
SELECT NOW() AS current_datetime;
-- 結果例: '2025-01-15 14:30:45'

-- 現在の日付のみ取得
SELECT CURDATE() AS current_date;
-- 結果例: '2025-01-15'

-- 現在の時刻のみ取得
SELECT CURTIME() AS current_time;
-- 結果例: '14:30:45'

日付の加減算 - DATE_ADD/DATE_SUB関数

日付に指定した期間を加算・減算できます。有効期限の計算、契約満了日の算出などに活用できます。

日付の加減算 
-- 現在日時から7日後を計算
SELECT DATE_ADD(NOW(), INTERVAL 7 DAY) AS one_week_later;

-- 3ヶ月前の日付を計算
SELECT DATE_SUB(NOW(), INTERVAL 3 MONTH) AS three_months_ago;

-- 注文日から30日後の配送予定日を算出
SELECT
    order_id,
    order_date,
    DATE_ADD(order_date, INTERVAL 30 DAY) AS estimated_delivery
FROM orders;

日付のフォーマット - DATE_FORMAT関数

日付を指定した書式で文字列に変換できます。画面表示用の日付整形に便利です。

日付のフォーマット 
-- 日本形式で表示（2025年01月15日）
SELECT DATE_FORMAT(NOW(), '%Y年%m月%d日') AS jp_date;

-- スラッシュ区切りで表示（2025/01/15）
SELECT DATE_FORMAT(NOW(), '%Y/%m/%d') AS slash_date;

-- 曜日付きで表示
SELECT DATE_FORMAT(NOW(), '%Y-%m-%d (%W)') AS date_with_day;

DATE_FORMATの主なフォーマット指定子

  • %Y - 4桁の年（2025）
  • %y - 2桁の年（25）
  • %m - 2桁の月（01-12）
  • %d - 2桁の日（01-31）
  • %H - 24時間制の時（00-23）
  • %i - 分（00-59）
  • %s - 秒（00-59）
  • %W - 曜日名（Sunday-Saturday）

日付の差分計算 - DATEDIFF関数

2つの日付の差を日数で取得できます。期間の計算や経過日数の算出に使用します。

日付の差分計算 
-- 2つの日付の差を日数で取得
SELECT DATEDIFF('2025-01-15', '2025-01-01') AS days_diff;
-- 結果: 14

-- 注文から発送までの日数を計算
SELECT
    order_id,
    DATEDIFF(shipped_date, order_date) AS processing_days
FROM orders
WHERE shipped_date IS NOT NULL;

日付の部分抽出

日付データから年、月、日、曜日などの要素を個別に取り出すことができます。

日付の部分抽出 
-- 年、月、日を個別に取得
SELECT
    YEAR(NOW()) AS year,
    MONTH(NOW()) AS month,
    DAY(NOW()) AS day;

-- 月別の売上集計
SELECT
    YEAR(order_date) AS year,
    MONTH(order_date) AS month,
    SUM(total_amount) AS monthly_sales
FROM orders
GROUP BY YEAR(order_date), MONTH(order_date);

5.3 数値関数 - 計算と加工を行う

MySQLは数値の計算や加工を行うための様々な関数を提供しています。四捨五入、切り上げ、切り捨てなどの基本的な操作から、べき乗、平方根などの数学的な計算まで対応しています。

四捨五入・切り上げ・切り捨て

金額計算や統計処理で頻繁に使用する数値の丸め処理を行います。

数値の丸め処理 
-- 四捨五入
SELECT ROUND(123.456, 2) AS rounded;
-- 結果: 123.46

-- 切り上げ
SELECT CEIL(123.1) AS ceiling;
-- 結果: 124

-- 切り捨て
SELECT FLOOR(123.9) AS floor_val;
-- 結果: 123

-- 小数点以下を切り捨て
SELECT TRUNCATE(123.456, 2) AS truncated;
-- 結果: 123.45

絶対値・符号・べき乗

数値の絶対値を取得したり、べき乗計算を行ったりする関数です。

数値の基本演算 
-- 絶対値を取得
SELECT ABS(-123) AS absolute_value;
-- 結果: 123

-- 符号を取得（正なら1、負なら-1、0なら0）
SELECT SIGN(-123) AS sign_value;

-- べき乗（2の3乗）
SELECT POWER(2, 3) AS power_result;
-- 結果: 8

-- 平方根
SELECT SQRT(16) AS square_root;
-- 結果: 4

乱数・余り

ランダムな値の生成や割り算の余りを求める関数です。

乱数と余り 
-- 0から1の間の乱数を取得
SELECT RAND() AS random_value;

-- 1から100の整数乱数を取得
SELECT FLOOR(RAND() * 100) + 1 AS random_int;

-- 余りを計算（割り算の余り）
SELECT MOD(10, 3) AS remainder;
-- 結果: 1

5.4 集約関数 - データを集計する

集約関数は複数行のデータをまとめて1つの結果を返す関数です。データ分析やレポート作成に欠かせない機能です。

基本的な集約関数

集約関数の使用例 
-- レコード数をカウント
SELECT COUNT(*) AS total_orders
FROM orders;

-- 合計を計算
SELECT SUM(amount) AS total_sales
FROM orders;

-- 平均を計算
SELECT AVG(price) AS average_price
FROM products;

-- 最大値・最小値を取得
SELECT
    MAX(price) AS max_price,
    MIN(price) AS min_price
FROM products;

GROUP BYと組み合わせた集計

集約関数はGROUP BY句と組み合わせることで、グループごとの集計が可能になります。

グループ別集計 
-- カテゴリ別の商品数と平均価格
SELECT
    category_id,
    COUNT(*) AS product_count,
    AVG(price) AS avg_price,
    MAX(price) AS max_price
FROM products
GROUP BY category_id;

-- 月別売上集計
SELECT
    YEAR(order_date) AS year,
    MONTH(order_date) AS month,
    COUNT(*) AS order_count,
    SUM(total_amount) AS monthly_sales
FROM orders
GROUP BY YEAR(order_date), MONTH(order_date)
ORDER BY year, month;

COUNT(*)とCOUNT(カラム名)の違い

COUNT(*)は全ての行をカウントしますが、COUNT(カラム名)はそのカラムがNULLでない行のみをカウントします。NULLを含むデータを集計する際は注意が必要です。

実習: 様々な関数を試してみよう

演習1: 文字列関数を使ってフルネームを作成

-- employeesテーブルで姓名を結合し、すべて大文字で表示
SELECT UPPER(CONCAT(last_name, ' ', first_name)) AS full_name_upper
FROM employees
LIMIT 5;

演習2: 日付関数で期限を計算

-- 現在日時、7日後、30日後を計算
SELECT
    NOW() AS current_time,
    DATE_ADD(NOW(), INTERVAL 7 DAY) AS one_week_later,
    DATE_ADD(NOW(), INTERVAL 30 DAY) AS one_month_later;

演習3: 数値関数で価格を加工

-- 価格に1.1を掛けて税込価格を計算し、小数点第1位で四捨五入
SELECT
    product_name,
    price AS original_price,
    ROUND(price * 1.1, 1) AS tax_included_price
FROM products
LIMIT 5;

理解度確認クイズ

  1. Q1. CONCAT関数とCONCAT_WS関数の違いは何ですか？

    回答を表示

    CONCAT関数は単純に文字列を結合しますが、いずれかの引数がNULLの場合は結果もNULLになります。CONCAT_WS関数（With Separator）は第1引数に区切り文字を指定し、NULL値を無視して結合してくれるため、より安全に使用できます。

  2. Q2. DATE_ADD関数で使用できるINTERVALの単位を3つ挙げてください。

    回答を表示

    DAY（日）、MONTH（月）、YEAR（年）のほか、HOUR（時）、MINUTE（分）、SECOND（秒）、WEEK（週）などが使用できます。

  3. Q3. ROUND、CEIL、FLOORの違いを説明してください。

    回答を表示

    ROUNDは四捨五入、CEILは切り上げ（天井）、FLOORは切り捨て（床）を行います。例えば3.4に対して、ROUND(3.4)=3、CEIL(3.4)=4、FLOOR(3.4)=3となります。

  • 日付・時刻関数（NOW、DATE_ADD、DATE_FORMATなど）で時間データを操作・計算できる
  • 数値関数（ROUND、CEIL、FLOORなど）で様々な数値計算や丸め処理ができる
  • 集約関数（COUNT、SUM、AVGなど）をGROUP BYと組み合わせることでグループ別集計が可能
