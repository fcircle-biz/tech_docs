同じ処理を繰り返そう - LOOP文で効率的なプログラム

基本LOOP、FOR LOOP、WHILE LOOPの3種類の繰り返し処理を学習します。1から100までの合計計算、テーブルの全データ処理など、繰り返しが必要な場面での使い方を習得します。

この章で学ぶこと

  • 繰り返し処理とは何か、なぜ必要なのかを理解する
  • 基本LOOP、FOR LOOP、WHILE LOOPの3種類の書き方を習得する
  • 繰り返し処理の使い分け（どの場面でどのLOOPを使うか）を知る
  • 実用的な例（合計計算、データ一括処理）を実装できる

6.1 繰り返し処理とは - 同じ作業を自動化しよう

日常生活でも「繰り返し」はたくさんあります。例えば、「1から100までの数字を紙に書く」という作業を考えてみましょう。
手作業なら100回書く必要がありますが、プログラミングでは「繰り返し処理」を使えば自動化できます。

繰り返し処理が役立つ場面

  • 1から100までの合計を計算する
  • テーブルの全データを1件ずつ処理する
  • 全社員の給与に5%の昇給を適用する
  • 在庫が10個以下の商品を全て発注する

PL/SQLの3種類の繰り返し処理

種類 特徴 使う場面
基本LOOP 自分で終了条件を書く 柔軟な制御が必要なとき
FOR LOOP 回数が決まっている 1〜100のように範囲が明確なとき
WHILE LOOP 条件を満たす間だけ繰り返す 終了条件が複雑なとき

6.2 基本LOOP - 自分で終了条件を決める

基本LOOPは、最もシンプルな繰り返し処理です。
LOOPで始まり、END LOOP;で終わります。
終了条件はEXITまたはEXIT WHENで指定します。

基本LOOPの書き方

基本LOOPの構文 
DECLARE
    v_counter NUMBER := 1;  -- カウンター変数（1から開始）
BEGIN
    LOOP
        -- 繰り返す処理をここに書く
        DBMS_OUTPUT.PUT_LINE('カウント: ' || v_counter);

        -- カウンターを1増やす
        v_counter := v_counter + 1;

        -- 終了条件（10を超えたら終了）
        EXIT WHEN v_counter > 10;
    END LOOP;

    DBMS_OUTPUT.PUT_LINE('繰り返し終了');
END;
/

重要ポイント

  • EXIT WHEN 条件 で終了条件を指定
  • カウンター変数を忘れずに増やす（無限ループに注意）
  • EXIT; だけ書くと、その時点で即終了

実習 6-1: 1から5までの数字を表示する

実習 6-1: 基本LOOPで数字を表示

基本LOOPを使って、1から5までの数字を順番に表示するプログラムを作成しましょう。

basic_loop_sample.sql 
DECLARE
    v_num NUMBER := 1;  -- 1から開始
BEGIN
    LOOP
        -- 現在の数字を表示
        DBMS_OUTPUT.PUT_LINE(v_num);

        -- 次の数字へ
        v_num := v_num + 1;

        -- 5を超えたら終了
        EXIT WHEN v_num > 5;
    END LOOP;
END;

6.3 FOR LOOP - 回数が決まっている繰り返し

FOR LOOPは、「1から10まで」のように繰り返し回数が決まっている場合に便利です。
カウンター変数を自動で作成・更新してくれるため、基本LOOPよりシンプルに書けます。

FOR LOOPの書き方

FOR LOOPの構文 
BEGIN
    -- 1から10まで繰り返す
    FOR i IN 1..10 LOOP
        DBMS_OUTPUT.PUT_LINE('カウント: ' || i);
    END LOOP;

    DBMS_OUTPUT.PUT_LINE('繰り返し終了');
END;
/

FOR LOOPの特徴

  • i は自動で作成されるカウンター変数（DECLARE不要）
  • 1..10 は「1から10まで」を意味する（両端含む）
  • カウンターは自動で1ずつ増える
  • カウンター変数は読み取り専用（変更できない）

逆順に繰り返す（REVERSE）

REVERSEキーワードを使うと、逆順（大きい数から小さい数へ）に繰り返せます。

REVERSE FOR LOOP 
BEGIN
    -- 10から1まで逆順に表示
    FOR i IN REVERSE 1..10 LOOP
        DBMS_OUTPUT.PUT_LINE(i);
    END LOOP;
END;
/

実習 6-2: 1から100までの合計を計算する

実習 6-2: FOR LOOPで合計計算

FOR LOOPを使って、1から100までの数字を全て足した合計を計算しましょう。

for_loop_sum.sql 
DECLARE
    v_total NUMBER := 0;  -- 合計を入れる変数（0から開始）
BEGIN
    -- 1から100まで繰り返す
    FOR i IN 1..100 LOOP
        -- 現在の数を合計に加える
        v_total := v_total + i;
    END LOOP;

    -- 結果を表示
    DBMS_OUTPUT.PUT_LINE('1から100までの合計: ' || v_total);
END;
/

期待される出力

1から100までの合計: 5050

6.4 WHILE LOOP - 条件を満たす間だけ繰り返す

WHILE LOOPは、「条件がTRUE（真）の間だけ繰り返す」処理です。
基本LOOPと似ていますが、ループの最初に条件をチェックするため、最初から条件が満たされていなければ1回も実行されません。

WHILE LOOPの書き方

WHILE LOOPの構文 
DECLARE
    v_counter NUMBER := 1;  -- カウンター変数
BEGIN
    -- v_counterが10以下の間だけ繰り返す
    WHILE v_counter <= 10 LOOP
        DBMS_OUTPUT.PUT_LINE('カウント: ' || v_counter);

        -- カウンターを1増やす（これを忘れると無限ループ）
        v_counter := v_counter + 1;
    END LOOP;

    DBMS_OUTPUT.PUT_LINE('繰り返し終了');
END;
/

無限ループに注意

WHILE LOOPでは、必ずカウンター変数を更新してください。
更新を忘れると、条件が永遠にTRUEのままになり、プログラムが終わらなくなります（無限ループ）。
無限ループになったら、SQL Developerの「停止」ボタンで止めましょう。

実習 6-3: WHILE LOOPで偶数だけ表示

実習 6-3: WHILE LOOPで条件付き表示

WHILE LOOPを使って、1から20までの偶数だけを表示するプログラムを作成しましょう。

while_loop_even.sql 
DECLARE
    v_num NUMBER := 2;  -- 2から開始（最初の偶数）
BEGIN
    -- 20以下の間だけ繰り返す
    WHILE v_num <= 20 LOOP
        -- 偶数を表示
        DBMS_OUTPUT.PUT_LINE(v_num);

        -- 次の偶数へ（2ずつ増やす）
        v_num := v_num + 2;
    END LOOP;
END;

6.5 3種類のLOOPの使い分け - どれを選ぶべき?

3種類のLOOPには、それぞれ得意な場面があります。適切なLOOPを選ぶことで、コードがシンプルで読みやすくなります。

使い分けの目安

FOR LOOPを選ぶとき

繰り返し回数が明確なとき（例: 1から100まで、配列の全要素を処理）
最もシンプルで読みやすいため、使える場面では優先的に使いましょう。

WHILE LOOPを選ぶとき

条件を満たす間だけ繰り返すとき（例: 残高が1000円以上の間、データが見つかる間）
最初から条件が満たされていない場合、1回も実行されない点が特徴です。

基本LOOPを選ぶとき

柔軟な制御が必要なとき（例: 途中で複数の終了条件がある、特定の場所で抜けたい）
FOR LOOPやWHILE LOOPで書きにくい場合に使います。

6.6 実用例 - データベースのデータを一括処理

実際の業務では、テーブルのデータを繰り返し処理することがよくあります。
例えば、「全商品の在庫が10個以下なら発注する」といった処理を実装してみましょう。

例: 在庫チェックと発注処理

stock_check_loop.sql 
DECLARE
    v_product_id NUMBER;
    v_stock NUMBER;
    v_count NUMBER := 0;  -- 発注した商品数
BEGIN
    -- 1から10までの商品IDを順番に処理
    FOR v_product_id IN 1..10 LOOP
        -- 商品の在庫数を取得
        SELECT stock INTO v_stock
        FROM products
        WHERE product_id = v_product_id;

        -- 在庫が10個以下なら発注
        IF v_stock <= 10 THEN
            -- 発注テーブルに追加
            INSERT INTO orders (product_id, quantity, order_date)
            VALUES (v_product_id, 50, SYSDATE);

            v_count := v_count + 1;
            DBMS_OUTPUT.PUT_LINE('商品ID ' || v_product_id || ' を発注しました');
        END IF;
    END LOOP;

    DBMS_OUTPUT.PUT_LINE('合計 ' || v_count || ' 件の商品を発注しました');
    COMMIT;  -- データベースに反映
END;
/

プログラムの流れ

  1. 商品ID 1から10まで順番に処理（FOR LOOP）
  2. 各商品の在庫数をSELECT文で取得
  3. 在庫が10個以下ならIF文で判定
  4. 条件に合う商品は発注テーブルにINSERT
  5. 発注した商品数を数えて最後に表示

理解度確認クイズ

  1. Q1. 1から50まで繰り返す処理に最も適したLOOPはどれですか？

    回答を表示

    FOR LOOPが最適です。
    繰り返し回数が明確（1から50まで）なので、FOR i IN 1..50 LOOPとシンプルに書けます。

  2. Q2. WHILE LOOPと基本LOOPの違いは何ですか？

    回答を表示

    WHILE LOOPは、ループの最初に条件をチェックします。条件が最初から満たされていなければ、1回も実行されません。
    基本LOOPは、少なくとも1回は実行され、途中でEXIT WHENをチェックします。

  3. Q3. FOR LOOPのカウンター変数を変更できますか？

    回答を表示

    いいえ、変更できません。
    FOR LOOPのカウンター変数（例: i）は読み取り専用です。
    値を変更しようとするとエラーになります。カウンターの値を変更したい場合は、基本LOOPを使います。

第6章のまとめ

  • PL/SQLには基本LOOP、FOR LOOP、WHILE LOOPの3種類の繰り返し処理がある
  • FOR LOOPは繰り返し回数が決まっているときに最もシンプル
  • WHILE LOOPは条件を満たす間だけ繰り返す（最初に条件チェック）
  • 基本LOOPは柔軟な制御が必要なときに使う
  • 無限ループを防ぐため、カウンター変数の更新を忘れずに
  • 繰り返し処理を使えば、テーブルのデータを一括処理できる
