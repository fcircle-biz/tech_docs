複数のデータをまとめて処理 - カーソルで行ごとの処理

SELECT文で複数行を取得して一行ずつ処理するカーソルの使い方を学習します。明細データの合計計算、全社員の給与更新など、実践的な例で理解を深めます。

この章で学ぶこと

  • カーソルとは何か、なぜ必要なのかを理解する
  • 基本カーソルの使い方（DECLARE、OPEN、FETCH、CLOSE）を習得する
  • FOR LOOPカーソルによる簡単な書き方を学ぶ
  • 実務での活用例（明細データの合計計算、全社員の給与更新）を実装する

7.1 カーソルとは - 複数のデータを1行ずつ処理する仕組み

これまで学習したSELECT INTO文は、1行だけのデータを取得して変数に格納する方法でした。
しかし、実際の業務では「全社員の給与を1人ずつ確認して更新する」「注文明細を1行ずつ処理して合計金額を計算する」といった複数行のデータを処理する場面がよくあります。

カーソル（Cursor）は、SELECT文で取得した複数行のデータを、1行ずつ順番に処理するための仕組みです。
身近な例で言えば、「本の栞（しおり）」のようなものです。栞を使って1ページずつ読み進めるように、カーソルを使ってデータを1行ずつ読み取ります。

カーソルが必要な場面

  • SELECT文が複数行の結果を返す場合（社員一覧、商品一覧など）
  • 1行ずつ異なる処理を実行したい場合（条件によって更新内容を変える）
  • データを集計しながら処理したい場合（合計金額を計算しながら進める）

7.2 基本カーソルの使い方 - 4つのステップ

カーソルを使うには、4つのステップを順番に実行します。

カーソルの4ステップ

① DECLARE（宣言）- カーソルを定義する

どのデータを取得するか（SELECT文）を定義します。
このステップでは、まだデータは取得されません（定義だけ）。

② OPEN（開く）- カーソルを開いてデータを取得

SELECT文を実行して、実際にデータを取得します。
栞を本に挟む（読み始める準備をする）イメージです。

③ FETCH（読み取り）- 1行ずつデータを取り出す

カーソルから1行ずつデータを取り出して変数に格納します。
本のページを1ページずつ読むイメージです。

④ CLOSE（閉じる）- カーソルを閉じてリソースを解放

全てのデータを処理し終えたら、カーソルを閉じます。
本を閉じて本棚に戻すイメージです。

基本カーソルのコード例

基本カーソルの例 
DECLARE
    -- ① カーソルを宣言（社員情報を取得するSELECT文を定義）
    CURSOR emp_cursor IS
        SELECT employee_id, first_name, salary
        FROM employees
        WHERE department_id = 10;

    -- 取得したデータを格納する変数
    v_emp_id employees.employee_id%TYPE;
    v_name employees.first_name%TYPE;
    v_salary employees.salary%TYPE;

BEGIN
    -- ② カーソルを開く（SELECT文を実行）
    OPEN emp_cursor;

    -- ③ LOOP文で1行ずつデータを取得
    LOOP
        -- FETCHで1行取得して変数に格納
        FETCH emp_cursor INTO v_emp_id, v_name, v_salary;

        -- データがなくなったらループを抜ける
        EXIT WHEN emp_cursor%NOTFOUND;

        -- 取得したデータを表示
        DBMS_OUTPUT.PUT_LINE('社員ID: ' || v_emp_id ||
                           ', 名前: ' || v_name ||
                           ', 給与: ' || v_salary);
    END LOOP;

    -- ④ カーソルを閉じる
    CLOSE emp_cursor;

    DBMS_OUTPUT.PUT_LINE('全ての社員データを処理しました');
END;
/

%NOTFOUND属性

カーソル名%NOTFOUNDは、FETCHで取得するデータがもうない場合にTRUEを返します。
この属性を使って「データがなくなったらループを終了」という制御を実現します。

7.3 FOR LOOPカーソル - もっと簡単な書き方

基本カーソルは4つのステップ（DECLARE、OPEN、FETCH、CLOSE）が必要でしたが、FOR LOOPカーソルを使うと、これらのステップを自動で実行してくれます。
初心者には、こちらの書き方がおすすめです。

FOR LOOPカーソルの例 
DECLARE
    -- カーソルを宣言（基本カーソルと同じ）
    CURSOR emp_cursor IS
        SELECT employee_id, first_name, salary
        FROM employees
        WHERE department_id = 10;

BEGIN
    -- FOR LOOPで自動的にOPEN、FETCH、CLOSEを実行
    FOR emp_rec IN emp_cursor LOOP
        -- emp_recはレコード型変数（自動で作成される）
        DBMS_OUTPUT.PUT_LINE('社員ID: ' || emp_rec.employee_id ||
                           ', 名前: ' || emp_rec.first_name ||
                           ', 給与: ' || emp_rec.salary);
    END LOOP;

    -- CLOSEは自動で実行されるので不要
    DBMS_OUTPUT.PUT_LINE('全ての社員データを処理しました');
END;
/

FOR LOOPカーソルのメリット

項目 基本カーソル FOR LOOPカーソル
OPEN/CLOSE 明示的に書く必要あり 自動実行（書かなくてOK）
FETCH LOOPの中に書く必要あり 自動実行（書かなくてOK）
変数宣言 1列ごとに変数を宣言 レコード型変数が自動作成
コード量 多い 少ない（シンプル）

レコード型変数とは

FOR LOOPカーソルでは、emp_recのようなレコード型変数が自動で作られます。
これは、SELECT文で取得した複数の列をまとめて格納できる「箱のセット」のようなものです。
emp_rec.employee_id、emp_rec.first_nameのように列名でアクセスできます。

7.4 実践例1: 注文明細の合計金額を計算する

注文明細テーブル（ORDER_LINES）から、特定の注文IDの全明細を取得して、合計金額を計算するプログラムを作ります。

実習 7-1: 注文明細の合計計算プログラム

注文ID=1001の全明細を取得して、小計（単価×数量）を計算し、最後に合計金額を表示します。

サンプルコード

order_total.sql 
DECLARE
    -- カーソル宣言（注文明細を取得）
    CURSOR order_cursor IS
        SELECT product_name, unit_price, quantity
        FROM order_lines
        WHERE order_id = 1001;

    -- 合計金額を格納する変数
    v_total NUMBER := 0;  -- 初期値0
    v_subtotal NUMBER;    -- 小計（単価×数量）

BEGIN
    DBMS_OUTPUT.PUT_LINE('=== 注文ID: 1001 の明細 ===');
    DBMS_OUTPUT.PUT_LINE('');

    -- FOR LOOPで1行ずつ処理
    FOR rec IN order_cursor LOOP
        -- 小計を計算（単価×数量）
        v_subtotal := rec.unit_price * rec.quantity;

        -- 明細を表示
        DBMS_OUTPUT.PUT_LINE(rec.product_name || ': ' ||
                           rec.unit_price || '円 × ' ||
                           rec.quantity || '個 = ' ||
                           v_subtotal || '円');

        -- 合計に加算
        v_total := v_total + v_subtotal;
    END LOOP;

    DBMS_OUTPUT.PUT_LINE('');
    DBMS_OUTPUT.PUT_LINE('=== 合計金額: ' || v_total || '円 ===');
END;
/

動作確認

SQL Developerで実行してみましょう。

期待される結果

ノートPC: 120000円 × 2個 = 240000円
マウス: 1500円 × 3個 = 4500円
キーボード: 8000円 × 2個 = 16000円

=== 合計金額: 260500円 ===

7.5 実践例2: 全社員の給与を昇給させる

全社員の給与を勤続年数に応じて昇給させるプログラムを作ります。
カーソルで社員データを1人ずつ取得し、UPDATE文で給与を更新します。

実習 7-2: 勤続年数による昇給プログラム

昇給ルール
- 勤続3年以上: 給与10%アップ
- 勤続1年以上3年未満: 給与5%アップ
- 勤続1年未満: 昇給なし

サンプルコード

salary_update.sql 
DECLARE
    -- カーソル宣言（全社員の情報を取得）
    CURSOR emp_cursor IS
        SELECT employee_id, first_name, salary, hire_date
        FROM employees;

    v_years NUMBER;      -- 勤続年数
    v_new_salary NUMBER; -- 新しい給与
    v_rate NUMBER;       -- 昇給率

BEGIN
    DBMS_OUTPUT.PUT_LINE('=== 給与昇給処理開始 ===');
    DBMS_OUTPUT.PUT_LINE('');

    -- FOR LOOPで1人ずつ処理
    FOR emp IN emp_cursor LOOP
        -- 勤続年数を計算（現在日付 - 入社日）
        v_years := FLOOR(MONTHS_BETWEEN(SYSDATE, emp.hire_date) / 12);

        -- 勤続年数に応じて昇給率を決定
        IF v_years >= 3 THEN
            v_rate := 1.10;  -- 10%アップ
            DBMS_OUTPUT.PUT_LINE(emp.first_name || '（勤続' || v_years || '年）: 10%昇給');
        ELSIF v_years >= 1 THEN
            v_rate := 1.05;  -- 5%アップ
            DBMS_OUTPUT.PUT_LINE(emp.first_name || '（勤続' || v_years || '年）: 5%昇給');
        ELSE
            v_rate := 1.00;  -- 昇給なし
            DBMS_OUTPUT.PUT_LINE(emp.first_name || '（勤続' || v_years || '年）: 昇給なし');
        END IF;

        -- 新しい給与を計算
        v_new_salary := emp.salary * v_rate;

        -- 給与を更新
        UPDATE employees
        SET salary = v_new_salary
        WHERE employee_id = emp.employee_id;

        DBMS_OUTPUT.PUT_LINE('  ' || emp.salary || '円 → ' || v_new_salary || '円');
    END LOOP;

    -- 変更をコミット
    COMMIT;

    DBMS_OUTPUT.PUT_LINE('');
    DBMS_OUTPUT.PUT_LINE('=== 全社員の給与更新が完了しました ===');
END;
/

COMMITを忘れずに

UPDATE文でデータを変更した場合、必ずCOMMIT;を実行してください。
COMMITしないと、変更がデータベースに反映されません。
また、テストの場合はROLLBACK;で変更を取り消すこともできます。

7.6 カーソルの注意点とパフォーマンス

カーソルは便利ですが、使い方を間違えるとプログラムが遅くなる原因になります。

避けるべき使い方

  • 大量データ（数万行以上）を1行ずつ処理
  • カーソルの中でさらにカーソルを使う（ネスト）
  • 単純なUPDATE/DELETEなのにカーソルを使う

推奨される使い方

  • 1行ごとに異なる処理が必要な場合のみ使う
  • データ量が少ない（数百〜数千行）場合
  • FOR LOOPカーソルを優先的に使う

カーソルよりSQLで一括処理を優先

例えば「全社員の給与を10%アップ」は、カーソルを使わなくても
UPDATE employees SET salary = salary * 1.10;
この1行のSQLで実行できます。カーソルは「1行ごとに違う処理」が必要な場合のみ使いましょう。

理解度確認クイズ

  1. Q1. カーソルとは何ですか？

    回答を表示

    カーソルは、SELECT文で取得した複数行のデータを1行ずつ順番に処理するための仕組みです。
    本の栞（しおり）のように、データを1行ずつ読み進めることができます。

  2. Q2. 基本カーソルの4つのステップは何ですか？

    回答を表示

    ① DECLARE（宣言）- カーソルを定義する
    ② OPEN（開く）- カーソルを開いてデータを取得
    ③ FETCH（読み取り）- 1行ずつデータを取り出す
    ④ CLOSE（閉じる）- カーソルを閉じてリソースを解放

  3. Q3. FOR LOOPカーソルのメリットは何ですか？

    回答を表示

    FOR LOOPカーソルは、OPEN、FETCH、CLOSEを自動で実行してくれます。
    また、レコード型変数が自動で作成されるため、変数宣言も不要です。
    コードがシンプルになり、初心者でも使いやすいです。

  4. Q4. カーソルを使わない方が良い場面はありますか？

    回答を表示

    はい、あります。全行に対して同じ処理を行う場合は、カーソルを使わずにSQLの一括UPDATE/DELETEを使う方が高速です。
    例: 全社員の給与を10%アップ → UPDATE employees SET salary = salary * 1.10;
    カーソルは「1行ごとに違う処理」が必要な場合のみ使いましょう。
