PL/SQLってなに？ - プログラミングでデータベース処理を自動化しよう

SQLだけでは難しい「もし〜なら」「〜を繰り返す」といった処理を可能にするPL/SQLの役割と特徴を、身近な例を使って分かりやすく説明します。

この章で学ぶこと

  • PL/SQLとは何か、SQLとの違いを理解する
  • なぜデータベース処理にプログラミングが必要なのかを知る
  • PL/SQLで実現できる身近な例（請求書作成、在庫チェック）を知る
  • PL/SQLプログラムの基本構造を理解する

1.1 SQLとPL/SQLの違い - なぜプログラミングが必要？

これまで学習したSQLは、データベースに対して「データを見せて」「データを追加して」といった命令を1つずつ実行する言語でした。
例えば、「商品テーブルから在庫が10個以下の商品を探す」というSQLは1回で実行できます。

flowchart LR A[SQL学習者] -->|SELECT文を実行| B[データベース] B -->|結果を返す| A style A fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style B fill:#fecaca,stroke:#ef4444,stroke-width:2px

しかし、実際の業務では「もし在庫が10個以下なら警告メールを送る」「全商品を1つずつチェックして処理する」といった複雑な処理が必要になります。
PL/SQL（Procedural Language/SQL）は、SQLに「プログラミングの力」を追加した言語です。

PL/SQLの正式名称

PL/SQL = Procedural Language（手続き型言語） + SQL
SQLに「手順を書ける機能」を追加したものです。

SQLとPL/SQLの違い（比較表）

項目 SQL PL/SQL
できること データの検索・追加・更新・削除 SQL + 条件分岐・繰り返し・変数・関数
実行方法 1つずつ命令を実行 複数の処理をまとめて実行
例 SELECT * FROM products; IF 在庫 < 10 THEN 警告表示

1.2 PL/SQLで実現できること - 身近な例で理解しよう

例1: 請求書の自動作成

あなたが書店の店員で、毎月の売上データから請求書を作成するとします。

SQLだけの場合（手作業が多い）

  1. SELECT文で売上データを取得
  2. Excelで合計金額を計算
  3. 消費税を手動で計算（10%掛ける）
  4. 請求書テーブルに手動でINSERT

PL/SQLを使うと（自動化）

  1. 売上データを自動で取得
  2. 合計金額を自動計算
  3. 消費税を自動計算
  4. 請求書テーブルに自動でINSERT
  5. ボタン1つで完了！

例2: 在庫チェックと自動発注

在庫が少なくなった商品を自動でチェックし、発注する処理を考えます。

このように、「もし〜なら」（条件分岐）や「繰り返し」といった処理が必要な場面で、PL/SQLが活躍します。

1.3 PL/SQLプログラムの基本構造 - 3つのブロック

PL/SQLプログラムは、3つのブロックで構成されています。
料理のレシピに例えると分かりやすいです。

PL/SQL基本構造 
DECLARE
    -- ① 宣言部（材料を準備）
    -- 変数や定数を宣言する場所
    v_stock_count NUMBER;  -- 在庫数を入れる変数

BEGIN
    -- ② 実行部（調理する）
    -- 実際の処理を書く場所（必須）
    SELECT COUNT(*) INTO v_stock_count FROM products;
    DBMS_OUTPUT.PUT_LINE('在庫件数: ' || v_stock_count);

EXCEPTION
    -- ③ 例外処理部（失敗したときの対処）
    -- エラーが起きたときの処理を書く場所
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('エラーが発生しました');
END;
/

各ブロックの役割

① DECLARE（宣言部）- 材料を準備

変数（データを入れる箱）や定数を準備する場所です。
省略可能（変数を使わない場合）

② BEGIN〜END（実行部）- 調理する

実際の処理（SELECT、INSERT、IF文など）を書く場所です。
必須（プログラムの中心部分）

③ EXCEPTION（例外処理部）- 失敗したときの対処

エラーが起きたときの対処を書く場所です。
省略可能（後の章で詳しく学習）

最後のスラッシュ（/）を忘れずに

PL/SQLプログラムの最後には、必ずEND;の後に/（スラッシュ）を書きます。
これは「プログラムの終わり」をOracleに伝える合図です。

1.4 PL/SQLが使われる場面 - 実務での活用例

PL/SQLは、企業の基幹システム（会計、販売、在庫管理など）で幅広く使われています。

会計システム

月次決算の自動計算、伝票の自動作成、税額計算

販売管理システム

注文処理、在庫更新、売上集計、顧客ポイント計算

在庫管理システム

入庫・出庫処理、棚卸し計算、発注点チェック

人事給与システム

給与計算、勤怠集計、社会保険料計算

理解度確認クイズ

  1. Q1. SQLとPL/SQLの最大の違いは何ですか？

    回答を表示

    SQLは「1つずつ命令を実行する言語」ですが、PL/SQLは「条件分岐（IF文）や繰り返し（LOOP文）などのプログラミング機能を持つ言語」です。
    PL/SQLを使うことで、複雑な処理を自動化できます。

  2. Q2. PL/SQLプログラムの3つのブロックは何ですか？

    回答を表示

    ① DECLARE（宣言部）- 変数や定数を準備
    ② BEGIN〜END（実行部）- 実際の処理を書く（必須）
    ③ EXCEPTION（例外処理部）- エラー時の処理を書く

  3. Q3. PL/SQLプログラムの最後に必ず書く記号は何ですか？

    回答を表示

    スラッシュ（/）です。
    END;の後に必ず/を書きます。これは「プログラムの終わり」をOracleに伝える合図です。

第1章のまとめ

  • PL/SQLは、SQLに「プログラミングの力」を追加した言語
  • 「もし〜なら」（条件分岐）や「繰り返し」といった処理が可能
  • PL/SQLプログラムは、DECLARE、BEGIN〜END、EXCEPTIONの3つのブロックで構成
  • 請求書作成、在庫チェック、給与計算など、実務で幅広く活用されている
  • プログラムの最後には必ずEND;の後に/を書く
