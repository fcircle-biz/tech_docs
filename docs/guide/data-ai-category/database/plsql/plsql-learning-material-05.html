条件によって処理を変えよう - IF文で賢いプログラム

IF-THEN-ELSE文を使った条件分岐を学習します。年齢による割引計算、在庫チェック、成績評価など、実用的な例を通じて条件による処理の切り替え方を理解します。

この章で学ぶこと

  • IF-THEN構文による基本的な条件分岐を理解する
  • IF-THEN-ELSE構文で「そうでない場合」の処理を書く
  • IF-THEN-ELSIF構文で複数の条件を扱う
  • CASE文を使ったスッキリした条件分岐を習得する

5.1 条件分岐とは - 状況に応じて処理を変える仕組み

プログラムでは、状況に応じて異なる処理を実行する必要があります。
例えば「もし在庫が10個以下なら発注する」「60歳以上なら割引料金を適用する」といった場面です。
これを実現するのが条件分岐です。

日常生活での条件分岐

「もし雨が降っていたら傘を持っていく、そうでなければ持っていかない」
「もし50円以上お釣りがあればガムを買う、そうでなければ何も買わない」
私たちは日常生活で自然に条件分岐を使っています。

5.2 IF-THEN構文 - 「もし〜なら」の基本

最も基本的な条件分岐がIF-THEN構文です。
「もし（条件）なら（処理）する」という形で書きます。

基本構文

IF-THEN構文 
IF 条件 THEN
    -- 条件が真のときに実行される処理
END IF;

例1: 在庫チェック

在庫が少ないときに警告を表示 
DECLARE
    v_stock NUMBER := 8;  -- 在庫数
BEGIN
    -- もし在庫が10個以下なら
    IF v_stock <= 10 THEN
        DBMS_OUTPUT.PUT_LINE('警告: 在庫が少なくなっています');
        DBMS_OUTPUT.PUT_LINE('現在の在庫: ' || v_stock || '個');
    END IF;

    DBMS_OUTPUT.PUT_LINE('在庫チェック完了');
END;
/

実行結果

5.3 IF-THEN-ELSE構文 - 「そうでなければ」を追加

条件が真（TRUE）でない場合の処理も書きたいときは、ELSEを使います。
「もし〜なら（処理A）する、そうでなければ（処理B）する」という形です。

基本構文

IF-THEN-ELSE構文 
IF 条件 THEN
    -- 条件が真のときの処理
ELSE
    -- 条件が偽のときの処理
END IF;

例2: シニア割引の判定

年齢によって料金を変える 
DECLARE
    v_age NUMBER := 65;      -- 年齢
    v_price NUMBER := 1000;  -- 基本料金
    v_final_price NUMBER;    -- 最終料金
BEGIN
    -- もし60歳以上なら
    IF v_age >= 60 THEN
        v_final_price := v_price * 0.8;  -- 20%割引
        DBMS_OUTPUT.PUT_LINE('シニア割引適用');
    ELSE
        v_final_price := v_price;  -- 通常料金
        DBMS_OUTPUT.PUT_LINE('通常料金適用');
    END IF;

    DBMS_OUTPUT.PUT_LINE('お支払い金額: ' || v_final_price || '円');
END;
/

実行結果

シニア割引適用
お支払い金額: 800円

5.4 IF-THEN-ELSIF構文 - 複数の条件を扱う

3つ以上の条件を扱う場合は、ELSIF（エルスイフ）を使います。
上から順番に条件をチェックし、最初に真になった条件の処理だけが実行されます。

ELSIFのスペルに注意

他のプログラミング言語では「ELSEIF」と書くこともありますが、PL/SQLではELSIF（Eが1つ）です。
間違えると構文エラーになります。

基本構文

IF-THEN-ELSIF構文 
IF 条件1 THEN
    -- 条件1が真のときの処理
ELSIF 条件2 THEN
    -- 条件2が真のときの処理
ELSIF 条件3 THEN
    -- 条件3が真のときの処理
ELSE
    -- どの条件も満たさないときの処理
END IF;

例3: 成績評価

点数に応じて評価を判定 
DECLARE
    v_score NUMBER := 75;  -- 点数
    v_grade VARCHAR2(1);   -- 評価（A, B, C, D, F）
BEGIN
    -- 点数に応じて評価を判定
    IF v_score >= 90 THEN
        v_grade := 'A';
    ELSIF v_score >= 80 THEN
        v_grade := 'B';
    ELSIF v_score >= 70 THEN
        v_grade := 'C';
    ELSIF v_score >= 60 THEN
        v_grade := 'D';
    ELSE
        v_grade := 'F';
    END IF;

    DBMS_OUTPUT.PUT_LINE('点数: ' || v_score || '点');
    DBMS_OUTPUT.PUT_LINE('評価: ' || v_grade);
END;
/

実行結果

点数: 75点
評価: C

5.5 論理演算子 - 複数の条件を組み合わせる

「〜かつ〜」「〜または〜」といった複数の条件を組み合わせる場合は、論理演算子を使います。

論理演算子の種類

演算子 意味 例
AND かつ（両方とも真） IF v_age >= 18 AND v_age < 60 THEN
OR または（どちらか一方が真） IF v_day = '土' OR v_day = '日' THEN
NOT 否定（真偽を反転） IF NOT v_is_member THEN

例4: 複数条件の組み合わせ

会員割引とシニア割引の判定 
DECLARE
    v_age NUMBER := 35;           -- 年齢
    v_is_member BOOLEAN := TRUE;  -- 会員フラグ
    v_discount_rate NUMBER := 0;  -- 割引率
BEGIN
    -- 会員かつシニアの場合（最大割引）
    IF v_is_member AND v_age >= 60 THEN
        v_discount_rate := 0.3;  -- 30%割引
        DBMS_OUTPUT.PUT_LINE('会員シニア割引: 30%');
    -- 会員のみの場合
    ELSIF v_is_member THEN
        v_discount_rate := 0.1;  -- 10%割引
        DBMS_OUTPUT.PUT_LINE('会員割引: 10%');
    -- シニアのみの場合
    ELSIF v_age >= 60 THEN
        v_discount_rate := 0.2;  -- 20%割引
        DBMS_OUTPUT.PUT_LINE('シニア割引: 20%');
    -- どちらでもない場合
    ELSE
        DBMS_OUTPUT.PUT_LINE('割引なし');
    END IF;

    DBMS_OUTPUT.PUT_LINE('割引率: ' || (v_discount_rate * 100) || '%');
END;
/

5.6 CASE文 - スッキリ書ける条件分岐

同じ変数に対して複数の値を比較する場合、CASE文を使うとスッキリ書けます。
IF-ELSIF文よりも読みやすくなる場合があります。

基本構文（単純CASE式）

単純CASE式 
CASE 変数
    WHEN 値1 THEN 処理1
    WHEN 値2 THEN 処理2
    WHEN 値3 THEN 処理3
    ELSE 処理4
END CASE;

例5: 曜日による営業時間の表示

曜日別の営業時間 
DECLARE
    v_day_of_week VARCHAR2(10) := '土';  -- 曜日
    v_hours VARCHAR2(50);                 -- 営業時間
BEGIN
    -- 曜日に応じて営業時間を設定
    CASE v_day_of_week
        WHEN '月' THEN v_hours := '9:00-18:00';
        WHEN '火' THEN v_hours := '9:00-18:00';
        WHEN '水' THEN v_hours := '9:00-18:00';
        WHEN '木' THEN v_hours := '9:00-18:00';
        WHEN '金' THEN v_hours := '9:00-20:00';
        WHEN '土' THEN v_hours := '10:00-17:00';
        WHEN '日' THEN v_hours := '定休日';
        ELSE v_hours := '不明';
    END CASE;

    DBMS_OUTPUT.PUT_LINE(v_day_of_week || '曜日の営業時間: ' || v_hours);
END;
/

検索CASE式（複雑な条件）

範囲チェックなど、より複雑な条件を扱う場合は検索CASE式を使います。

検索CASE式 
CASE
    WHEN 条件1 THEN 処理1
    WHEN 条件2 THEN 処理2
    WHEN 条件3 THEN 処理3
    ELSE 処理4
END CASE;

例6: BMIによる体型判定

BMI値による判定 
DECLARE
    v_bmi NUMBER := 23.5;  -- BMI値
    v_result VARCHAR2(20); -- 判定結果
BEGIN
    -- BMI値による体型判定
    CASE
        WHEN v_bmi < 18.5 THEN v_result := '低体重';
        WHEN v_bmi < 25 THEN v_result := '普通体重';
        WHEN v_bmi < 30 THEN v_result := '肥満（1度）';
        ELSE v_result := '肥満（2度以上）';
    END CASE;

    DBMS_OUTPUT.PUT_LINE('BMI: ' || v_bmi);
    DBMS_OUTPUT.PUT_LINE('判定: ' || v_result);
END;
/

実習: 電気料金の計算

電力使用量に応じて料金単価が変わる電気料金計算プログラムを作成してみましょう。

料金体系

  • 120kWh以下: 1kWhあたり20円
  • 121〜300kWh: 1kWhあたり25円
  • 301kWh以上: 1kWhあたり30円
解答例を表示
電気料金計算 
DECLARE
    v_usage NUMBER := 250;  -- 電力使用量（kWh）
    v_price NUMBER;         -- 料金
BEGIN
    -- 使用量に応じて料金を計算
    IF v_usage <= 120 THEN
        v_price := v_usage * 20;
        DBMS_OUTPUT.PUT_LINE('料金区分: 第1段階');
    ELSIF v_usage <= 300 THEN
        v_price := v_usage * 25;
        DBMS_OUTPUT.PUT_LINE('料金区分: 第2段階');
    ELSE
        v_price := v_usage * 30;
        DBMS_OUTPUT.PUT_LINE('料金区分: 第3段階');
    END IF;

    DBMS_OUTPUT.PUT_LINE('使用量: ' || v_usage || 'kWh');
    DBMS_OUTPUT.PUT_LINE('電気料金: ' || v_price || '円');
END;
/

理解度確認クイズ

  1. Q1. IF文の最後に書く必要があるのは何ですか？

    回答を表示

    END IF;です。
    IF文は必ずEND IF;で終わる必要があります。セミコロンも忘れずに。

  2. Q2. 「そうでない場合」の処理を書くときに使うキーワードは？

    回答を表示

    ELSEです。
    IF-THEN-ELSE構文で、条件が偽（FALSE）のときの処理を書きます。

  3. Q3. 「60歳以上かつ会員」という条件を書くときの正しい書き方は？

    回答を表示

    IF v_age >= 60 AND v_is_member THEN
    複数の条件を「かつ」で結ぶときはAND演算子を使います。

第5章のまとめ

  • IF-THEN構文で「もし〜なら」という条件分岐を実現できる
  • IF-THEN-ELSE構文で「そうでなければ」の処理を追加できる
  • IF-THEN-ELSIF構文で3つ以上の条件を扱える（ELSIFのスペルに注意）
  • 論理演算子（AND, OR, NOT）で複数の条件を組み合わせられる
  • CASE文を使うと、同じ変数の複数値チェックをスッキリ書ける
  • 条件分岐を使うことで、状況に応じた賢いプログラムを作成できる
