第8章

エラーに備えよう - 例外処理で安全なプログラム

プログラム実行中のエラーに対処する例外処理を学習します。TRY-CATCHのような仕組みで、エラーが起きても止まらないプログラムを作ります。

この章で学ぶこと

  • 例外処理（EXCEPTION）とは何か、なぜ必要かを理解する
  • よく使う定義済み例外（NO_DATA_FOUND、TOO_MANY_ROWSなど）の使い方を学ぶ
  • すべてのエラーをキャッチするWHEN OTHERSの使い方を習得する
  • ユーザー定義例外を作成して、独自のエラー処理を実装する

8.1 例外処理とは何か - エラーが起きたときの対処

プログラムを実行していると、「データが見つからない」「ゼロで割り算してしまった」といったエラー（例外）が発生することがあります。
例外処理（EXCEPTION）は、こうしたエラーが起きたときに「プログラムを止めずに、適切に対処する仕組み」です。

例外処理がない場合

  1. エラーが発生
  2. プログラムが途中で止まる
  3. エラーメッセージだけが表示される
  4. ユーザーに何が起きたか分からない

例外処理がある場合

  1. エラーが発生
  2. 例外処理部がエラーをキャッチ
  3. 分かりやすいメッセージを表示
  4. プログラムは正常に終了

例外処理の例え

車の運転で言えば、「タイヤがパンクしたら（エラー発生）、安全な場所に停車して（例外処理）、対処する」という感じです。
例外処理がないと、パンクしたまま走り続けて大事故になってしまいます。

例外処理の基本構文

PL/SQLの例外処理は、EXCEPTIONブロックに書きます。

例外処理の基本構造 
BEGIN
    -- 通常の処理
    SELECT 商品名 INTO v_product FROM products WHERE 商品ID = 999;

EXCEPTION
    -- エラーが起きたときの処理
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('エラー: 商品が見つかりませんでした');
END;
/

8.2 よく使う定義済み例外 - NO_DATA_FOUND と TOO_MANY_ROWS

PL/SQLには、よくあるエラーに対応する定義済み例外が用意されています。
ここでは、SELECT文でよく使う2つの例外を学習します。

NO_DATA_FOUND（データが見つからない）

SELECT INTO文でデータが1件も見つからなかったときに発生します。

NO_DATA_FOUND の例 
DECLARE
    v_emp_name VARCHAR2(50);  -- 社員名を入れる変数
BEGIN
    -- 存在しない社員IDで検索（エラーが発生する）
    SELECT ename INTO v_emp_name FROM emp WHERE empno = 9999;

    DBMS_OUTPUT.PUT_LINE('社員名: ' || v_emp_name);

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('該当する社員が見つかりませんでした');
END;
/

TOO_MANY_ROWS（データが複数件返ってきた）

SELECT INTO文で、2件以上のデータが返ってきたときに発生します。
SELECT INTOは「1件だけ取得する」構文なので、複数件だとエラーになります。

TOO_MANY_ROWS の例 
DECLARE
    v_dept_name VARCHAR2(50);
BEGIN
    -- 部署10には複数の社員がいるため、複数行が返る（エラー）
    SELECT dname INTO v_dept_name FROM dept WHERE deptno = 10;

EXCEPTION
    WHEN TOO_MANY_ROWS THEN
        DBMS_OUTPUT.PUT_LINE('複数の結果が返されました');
        DBMS_OUTPUT.PUT_LINE('カーソルを使用してください');
END;
/

SELECT INTOは1件だけ

SELECT INTO文は「必ず1件だけ返ってくる」ことが前提です。
複数行を処理したい場合は、第7章で学んだカーソルを使いましょう。

8.3 その他のよく使う定義済み例外

主な定義済み例外一覧

例外名 発生するタイミング 例
NO_DATA_FOUND SELECT INTOでデータが0件 存在しない商品IDで検索
TOO_MANY_ROWS SELECT INTOで複数行が返る 部署番号で社員を検索（複数人）
ZERO_DIVIDE ゼロで割り算 100 / 0
VALUE_ERROR データ型や長さが合わない VARCHAR2(10)に20文字代入
INVALID_NUMBER 文字列を数値に変換できない TO_NUMBER('ABC')
DUP_VAL_ON_INDEX 一意制約違反（重複） 既に存在するIDでINSERT

ZERO_DIVIDE の実例

ゼロ除算エラーの処理 
DECLARE
    v_result NUMBER;
    v_dividend NUMBER := 100;  -- 割られる数
    v_divisor NUMBER := 0;     -- 割る数（ゼロ）
BEGIN
    v_result := v_dividend / v_divisor;  -- ゼロ除算エラー
    DBMS_OUTPUT.PUT_LINE('結果: ' || v_result);

EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('エラー: ゼロで割り算はできません');
        DBMS_OUTPUT.PUT_LINE('割られる数: ' || v_dividend);
        DBMS_OUTPUT.PUT_LINE('割る数: ' || v_divisor);
END;
/

8.4 WHEN OTHERS - すべてのエラーをキャッチ

WHEN OTHERSは、「それ以外のすべてのエラー」をキャッチする特別な例外処理です。
予期しないエラーが起きても、プログラムが止まらないようにできます。

WHEN OTHERS の例 
DECLARE
    v_emp_name VARCHAR2(50);
BEGIN
    -- 何らかの処理（エラーが起きるかもしれない）
    SELECT ename INTO v_emp_name FROM emp WHERE empno = 9999;

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('データが見つかりませんでした');
    WHEN TOO_MANY_ROWS THEN
        DBMS_OUTPUT.PUT_LINE('データが複数件見つかりました');
    WHEN OTHERS THEN
        -- 上記以外のすべてのエラーをキャッチ
        DBMS_OUTPUT.PUT_LINE('予期しないエラーが発生しました');
END;
/

WHEN OTHERSは最後に書く

WHEN OTHERSは、必ず例外処理の最後に書きます。
先に書くと、他の例外処理が実行されなくなってしまいます。

8.5 例外情報の取得 - SQLCODE と SQLERRM

エラーが起きたとき、「どんなエラーか」を知りたい場合があります。
PL/SQLには、エラー情報を取得する関数が用意されています。

関数 説明 戻り値の例
SQLCODE エラーコード（数値） 100（NO_DATA_FOUND）、-1476（ZERO_DIVIDE）
SQLERRM エラーメッセージ（文字列） "ORA-01476: divisor is equal to zero"
エラー情報を取得する例 
DECLARE
    v_result NUMBER;
BEGIN
    v_result := 100 / 0;  -- ゼロ除算エラー

EXCEPTION
    WHEN OTHERS THEN
        -- エラー情報を詳しく出力
        DBMS_OUTPUT.PUT_LINE('エラーが発生しました');
        DBMS_OUTPUT.PUT_LINE('エラーコード: ' || SQLCODE);
        DBMS_OUTPUT.PUT_LINE('エラーメッセージ: ' || SQLERRM);
END;
/

エラーログに記録しよう

実務では、SQLCODEとSQLERRMを使って、エラー内容をログテーブルに記録することが多いです。
後で「いつ、どんなエラーが起きたか」を確認できるようにします。

8.6 ユーザー定義例外 - 独自のエラーを作る

定義済み例外だけでは足りない場合、ユーザー定義例外を作成できます。
例えば、「在庫が足りない」「営業時間外」といった業務上のエラーを表現できます。

ユーザー定義例外の作り方

① 例外を宣言する（DECLAREセクション）

例外名 EXCEPTION; という形式で宣言します。

② 例外を発生させる（BEGINセクション）

条件に合ったら、RAISE 例外名; で例外を発生させます。

③ 例外を処理する（EXCEPTIONセクション）

WHEN 例外名 THEN で例外をキャッチして処理します。

在庫不足エラーの例 
DECLARE
    -- ① ユーザー定義例外を宣言
    e_stock_shortage EXCEPTION;

    v_stock NUMBER := 5;   -- 現在の在庫数
    v_order NUMBER := 10;  -- 注文数
BEGIN
    DBMS_OUTPUT.PUT_LINE('在庫チェック中...');

    -- 在庫が足りないか確認
    IF v_stock < v_order THEN
        -- ② 例外を発生させる
        RAISE e_stock_shortage;
    END IF;

    DBMS_OUTPUT.PUT_LINE('在庫は十分です。出荷処理を実行します。');

EXCEPTION
    -- ③ 例外を処理する
    WHEN e_stock_shortage THEN
        DBMS_OUTPUT.PUT_LINE('エラー: 在庫が不足しています');
        DBMS_OUTPUT.PUT_LINE('現在の在庫: ' || v_stock);
        DBMS_OUTPUT.PUT_LINE('注文数: ' || v_order);
END;
/

実習 8-1: 給与更新プログラムに例外処理を追加

社員の給与を更新するプログラムに、例外処理を追加しましょう。

手順

  1. 1 社員番号を入力して、該当社員の給与を10%アップするプログラムを作成
  2. 2 社員が見つからない場合のエラー処理を追加（NO_DATA_FOUND）
  3. 3 その他のエラーもキャッチ（WHEN OTHERS）してエラー情報を表示
解答例を表示 
DECLARE
    v_empno NUMBER := 9999;  -- 存在しない社員番号
    v_old_sal NUMBER;
    v_new_sal NUMBER;
BEGIN
    -- 現在の給与を取得
    SELECT sal INTO v_old_sal FROM emp WHERE empno = v_empno;

    -- 10%アップした給与を計算
    v_new_sal := v_old_sal * 1.1;

    -- 給与を更新
    UPDATE emp SET sal = v_new_sal WHERE empno = v_empno;
    COMMIT;

    DBMS_OUTPUT.PUT_LINE('給与更新完了');
    DBMS_OUTPUT.PUT_LINE('旧給与: ' || v_old_sal);
    DBMS_OUTPUT.PUT_LINE('新給与: ' || v_new_sal);

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('エラー: 社員番号 ' || v_empno || ' は存在しません');
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('予期しないエラーが発生しました');
        DBMS_OUTPUT.PUT_LINE('エラーコード: ' || SQLCODE);
        DBMS_OUTPUT.PUT_LINE('エラーメッセージ: ' || SQLERRM);
        ROLLBACK;  -- エラー時は変更を取り消す
END;
/

8.7 実践：在庫チェックプログラムに例外処理を追加

第7章で学んだカーソルと、今回の例外処理を組み合わせた実践的なプログラムを作成しましょう。

在庫チェックプログラム（例外処理付き） 
DECLARE
    -- カーソル定義
    CURSOR c_products IS
        SELECT product_id, product_name, stock
        FROM products;

    -- ユーザー定義例外
    e_low_stock EXCEPTION;

    v_product c_products%ROWTYPE;
    v_min_stock CONSTANT NUMBER := 10;  -- 最低在庫数
BEGIN
    DBMS_OUTPUT.PUT_LINE('=== 在庫チェック開始 ===');

    OPEN c_products;
    LOOP
        FETCH c_products INTO v_product;
        EXIT WHEN c_products%NOTFOUND;

        BEGIN
            -- 在庫チェック
            IF v_product.stock < v_min_stock THEN
                RAISE e_low_stock;
            END IF;

            DBMS_OUTPUT.PUT_LINE('OK: ' || v_product.product_name ||
                               ' (在庫: ' || v_product.stock || ')');

        EXCEPTION
            WHEN e_low_stock THEN
                DBMS_OUTPUT.PUT_LINE('警告: ' || v_product.product_name ||
                                   ' の在庫が不足しています (現在: ' ||
                                   v_product.stock || ')');
        END;
    END LOOP;
    CLOSE c_products;

    DBMS_OUTPUT.PUT_LINE('=== 在庫チェック完了 ===');

EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('システムエラーが発生しました');
        DBMS_OUTPUT.PUT_LINE('エラー: ' || SQLERRM);
        IF c_products%ISOPEN THEN
            CLOSE c_products;  -- カーソルを閉じる
        END IF;
END;
/

ネストした例外処理

このプログラムでは、LOOPの中にも例外処理（BEGIN〜EXCEPTION〜END）を入れています。
こうすることで、1つの商品でエラーが起きても、他の商品のチェックを続けられます。

理解度確認クイズ

  1. Q1. SELECT INTO文でデータが見つからなかったときに発生する例外は何ですか？

    回答を表示

    NO_DATA_FOUNDです。
    SELECT INTO文で0件のデータが返されたときに発生します。

  2. Q2. すべてのエラーをキャッチする例外処理は何ですか？

    回答を表示

    WHEN OTHERSです。
    定義済み例外では捕捉できないすべてのエラーをキャッチします。
    必ず例外処理の最後に書く必要があります。

  3. Q3. エラーコードを取得する関数と、エラーメッセージを取得する関数は何ですか？

    回答を表示

    エラーコード：SQLCODE
    エラーメッセージ：SQLERRM
    これらを使って、エラーログテーブルに記録することが多いです。

第8章のまとめ

  • 例外処理（EXCEPTION）は、エラーが起きたときにプログラムを止めずに適切に対処する仕組み
  • よく使う定義済み例外：NO_DATA_FOUND、TOO_MANY_ROWS、ZERO_DIVIDEなど
  • WHEN OTHERSで、すべてのエラーをキャッチできる（必ず最後に書く）
  • SQLCODE（エラーコード）とSQLERRM（エラーメッセージ）でエラー情報を取得できる
  • ユーザー定義例外を作成して、業務固有のエラー（在庫不足など）を表現できる
