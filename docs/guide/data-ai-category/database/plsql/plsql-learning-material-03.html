変数を使ってデータを覚えよう - 値を保存する箱の使い方

変数という「データを入れる箱」の概念を理解し、数値、文字、日付などのデータ型の選び方、代入の方法、定数の使い方を、電卓や住所録の例で学習します。

この章で学ぶこと

  • 変数とは何か、なぜ必要なのかを理解する
  • データ型（NUMBER、VARCHAR2、DATE、BOOLEAN）の使い分けを学ぶ
  • 変数の宣言方法と代入方法を習得する
  • 定数の使い方と変数との違いを知る

3.1 変数とは何か？ - データを入れる「箱」

変数（へんすう）とは、データを入れておく箱のようなものです。
第2章で学んだプログラムでは、計算結果や文字列を直接表示していました。しかし、実際の業務では「一度計算した値を覚えておいて、後で使いたい」という場面がたくさんあります。

身近な例：電卓の「メモリー機能」

電卓には「M+」「M-」「MR」といったボタンがあります。これは計算結果を一時的に記憶しておく機能です。
変数も同じで、データを一時的に記憶しておく場所なのです。

変数が必要な理由

以下のような場面で、変数が必要になります：

場面 変数の役割
計算結果を保存 消費税額を計算して、変数に保存。後で合計金額の計算に使う
データベースから取得した値を保存 SELECT文で取得した顧客名を変数に保存。後で表示に使う
途中経過を保存 ループ処理で合計を計算するとき、途中の合計値を変数に保存

3.2 データ型の種類 - 箱の種類を選ぼう

変数（箱）には入れるデータの種類によって、いくつかの「型（かた）」があります。
これは、物を整理するときに「本は本棚に」「服はタンスに」と使い分けるのと同じです。

NUMBER

数値を入れる箱

例: 100、3.14、-50

VARCHAR2

文字列を入れる箱

例: '東京都'、'山田太郎'

DATE

日付・時刻を入れる箱

例: 2025-12-08 14:30:00

BOOLEAN

真偽値を入れる箱

例: TRUE、FALSE

重要ポイント

データ型を間違えると、エラーになります。
例: NUMBER型の変数に文字列'ABC'を入れようとすると、エラーが発生します。

3.3 変数の宣言と代入 - 箱を準備してデータを入れる

変数の宣言（箱を準備）

変数を使う前に、まず「どんな箱を使うか」を宣言（せんげん）します。
宣言は、DECLARE部分で行います。

変数宣言の基本形 
DECLARE
    -- 変数名 データ型;
    v_price NUMBER;        -- 価格（数値）
    v_name VARCHAR2(50);   -- 商品名（文字列、最大50文字）
    v_order_date DATE;     -- 注文日（日付）
    v_is_available BOOLEAN; -- 在庫有無（真偽値）
BEGIN
    -- ここで変数を使う処理を書く
    NULL;
END;
/

変数名の付け方のルール

  • 分かりやすい名前にする（v_price、v_nameなど）
  • 「v_」を付けると「これは変数だ」と分かりやすい（習慣）
  • 日本語は使えない（半角英数字とアンダースコアのみ）
  • 予約語（SELECT、BEGINなど）は使えない

代入（箱にデータを入れる）

変数にデータを入れることを代入（だいにゅう）といいます。
PL/SQLでは、:=（コロンとイコール）を使って代入します。

代入の例 
DECLARE
    v_price NUMBER;      -- 価格
    v_tax NUMBER;        -- 消費税
    v_total NUMBER;      -- 合計
BEGIN
    -- 代入: 変数名 := 値;
    v_price := 1000;     -- 価格に1000を代入
    v_tax := v_price * 0.1;  -- 消費税を計算して代入
    v_total := v_price + v_tax; -- 合計を計算

    DBMS_OUTPUT.PUT_LINE('価格: ' || v_price || '円');
    DBMS_OUTPUT.PUT_LINE('消費税: ' || v_tax || '円');
    DBMS_OUTPUT.PUT_LINE('合計: ' || v_total || '円');
END;
/

実行結果

価格: 1000円
消費税: 100円
合計: 1100円

宣言と同時に初期化する

変数を宣言するときに、同時に値を入れることもできます。

宣言と初期化 
DECLARE
    v_discount_rate NUMBER := 0.2;  -- 割引率20%を最初から設定
    v_shop_name VARCHAR2(50) := 'Fサークル書店'; -- 店舗名を設定
BEGIN
    DBMS_OUTPUT.PUT_LINE('店舗名: ' || v_shop_name);
    DBMS_OUTPUT.PUT_LINE('割引率: ' || (v_discount_rate * 100) || '%');
END;
/

3.4 定数の使い方 - 変わらない値を保存

定数（ていすう）は、一度値を入れたら変更できない変数です。
消費税率や円周率のように「プログラム中で変わらない値」を定数として定義します。

定数の宣言 
DECLARE
    -- CONSTANT キーワードを付ける
    c_tax_rate CONSTANT NUMBER := 0.1;  -- 消費税率10%（定数）
    v_price NUMBER := 5000;             -- 価格（変数）
    v_total NUMBER;                     -- 合計（変数）
BEGIN
    v_total := v_price * (1 + c_tax_rate);  -- 定数を使って計算
    DBMS_OUTPUT.PUT_LINE('合計金額: ' || v_total || '円');

    -- これはエラーになる（定数は変更できない）
    -- c_tax_rate := 0.08;  -- ← エラー！
END;
/

定数を使う理由

  1. プログラム中で何度も使う値を1箇所で管理できる
  2. 誤って値を変更してしまうミスを防げる
  3. 値の意味が分かりやすくなる（0.1より「c_tax_rate」の方が分かりやすい）

変数と定数の使い分け

項目 変数 定数
値の変更 何度でも変更可能 変更不可
宣言方法 v_price NUMBER; c_tax_rate CONSTANT NUMBER := 0.1;
使う場面 計算結果、データベースから取得した値 消費税率、円周率、会社名など

実習 3-1: 変数を使った簡単な計算

変数を使って、商品の合計金額と割引後の金額を計算しましょう。

手順

  1. 1 SQL Developerを起動し、「ワークシート」を開く
  2. 2 以下のPL/SQLコードを入力する
実習コード 
DECLARE
    -- 定数: 割引率
    c_discount_rate CONSTANT NUMBER := 0.15;  -- 15%割引

    -- 変数
    v_item_name VARCHAR2(50) := 'プログラミング入門書';
    v_price NUMBER := 3000;         -- 定価
    v_discount NUMBER;              -- 割引額
    v_final_price NUMBER;           -- 割引後価格
BEGIN
    -- 割引額を計算
    v_discount := v_price * c_discount_rate;

    -- 割引後の価格を計算
    v_final_price := v_price - v_discount;

    -- 結果を表示
    DBMS_OUTPUT.PUT_LINE('商品名: ' || v_item_name);
    DBMS_OUTPUT.PUT_LINE('定価: ' || v_price || '円');
    DBMS_OUTPUT.PUT_LINE('割引額: ' || v_discount || '円');
    DBMS_OUTPUT.PUT_LINE('割引後価格: ' || v_final_price || '円');
END;
/
  1. 3 「スクリプトの実行」ボタン（F5）をクリックして実行
  2. 4 「スクリプト出力」タブで結果を確認

期待される出力

商品名: プログラミング入門書
定価: 3000円
割引額: 450円
割引後価格: 2550円

理解度確認クイズ

  1. Q1. 変数とは何ですか？

    回答を表示

    変数とは、データを一時的に保存しておく「箱」のようなものです。
    計算結果やデータベースから取得した値を保存して、後で使うことができます。

  2. Q2. PL/SQLで代入を行うときに使う記号は何ですか？

    回答を表示

    :=（コロンとイコール）です。
    例: v_price := 1000;

  3. Q3. 定数と変数の違いは何ですか？

    回答を表示

    定数は一度値を設定すると変更できませんが、変数は何度でも値を変更できます。
    定数はCONSTANTキーワードを使って宣言し、消費税率など「変わらない値」を保存するときに使います。

  4. Q4. 文字列を保存するデータ型は何ですか？

    回答を表示

    VARCHAR2です。
    例: v_name VARCHAR2(50);（最大50文字の文字列）

第3章のまとめ

  • 変数は、データを一時的に保存する「箱」のようなもの
  • データ型にはNUMBER（数値）、VARCHAR2（文字列）、DATE（日付）、BOOLEAN（真偽値）がある
  • 変数の宣言はDECLARE部分で行い、変数名 データ型;の形式で書く
  • 代入は:=を使い、変数名 := 値;の形式で書く
  • 定数はCONSTANTキーワードを使い、値を変更できないようにする
  • 変数名には分かりやすい名前を付け、「v_」を付けると変数だと分かりやすい
