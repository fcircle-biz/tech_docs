第4章

データベースからデータを取り出そう - SELECTとINTO句

SQLのSELECT文をPL/SQL内で使って、データベースから値を取得して変数に格納する方法を学習します。1件のデータ取得、%TYPE属性を使った安全な変数宣言も習得します。

この章で学ぶこと

  • PL/SQL内でSELECT文とINTO句を使う方法を理解する
  • データベースから1件のデータを取得して変数に格納する
  • %TYPE属性を使った安全な変数宣言を習得する
  • NO_DATA_FOUNDエラーに対処する方法を知る

4.1 SELECT INTO句の基本 - データベースから値を取り出す

これまで学習した変数には、直接値を代入していました（v_name := '田中';）。
しかし、実際の業務ではデータベースに保存されているデータを取得して変数に格納することが多いです。

SELECT INTO句とは？

PL/SQL内でSELECT文を使うときは、INTO句を追加して、取得した値を変数に格納します。
SELECT 列名 INTO 変数名 FROM テーブル名;

基本構文

SELECT INTO句の基本構文 
DECLARE
    -- 変数を宣言（データを入れる箱を準備）
    v_product_name VARCHAR2(100);  -- 商品名を入れる変数
    v_price        NUMBER;         -- 価格を入れる変数

BEGIN
    -- データベースから値を取得して変数に格納
    SELECT product_name, price
    INTO   v_product_name, v_price
    FROM   products
    WHERE  product_id = 101;

    -- 取得した値を表示
    DBMS_OUTPUT.PUT_LINE('商品名: ' || v_product_name);
    DBMS_OUTPUT.PUT_LINE('価格: ' || v_price || '円');
END;
/

重要ポイント

  • SELECT句の列数とINTO句の変数の数は一致させる必要があります
  • SELECT文は必ず1件のデータを返す必要があります（0件や2件以上はエラー）
  • 変数のデータ型は、列のデータ型と互換性がある必要があります

4.2 %TYPE属性で安全な変数宣言 - テーブル定義と同じ型を使う

これまでは変数のデータ型を手動で指定していました（VARCHAR2(100)など）。
しかし、テーブルの列と同じデータ型を使いたい場合、%TYPE属性を使うと便利で安全です。

%TYPE属性とは？

%TYPEは、「テーブルの列と同じデータ型を使う」という指定方法です。
v_name products.product_name%TYPE;と書くと、productsテーブルのproduct_name列と同じ型になります。

%TYPE属性の使い方

%TYPE属性を使った変数宣言 
DECLARE
    -- テーブル定義と同じデータ型を自動的に使用
    v_product_name products.product_name%TYPE;  -- VARCHAR2(100)と同じ
    v_price        products.price%TYPE;         -- NUMBER(10,2)と同じ
    v_stock        products.stock%TYPE;         -- NUMBER(5)と同じ

BEGIN
    SELECT product_name, price, stock
    INTO   v_product_name, v_price, v_stock
    FROM   products
    WHERE  product_id = 101;

    DBMS_OUTPUT.PUT_LINE('商品名: ' || v_product_name);
    DBMS_OUTPUT.PUT_LINE('価格: ' || v_price || '円');
    DBMS_OUTPUT.PUT_LINE('在庫: ' || v_stock || '個');
END;
/

%TYPE属性のメリット（比較表）

項目 手動で型指定 %TYPE属性を使用
変数宣言 v_name VARCHAR2(100); v_name products.product_name%TYPE;
テーブル変更時 PL/SQLも修正が必要（エラーになる可能性） 自動的に追従（修正不要）
安全性 型の不一致が起きやすい 常に一致するので安全
メンテナンス 管理が煩雑 管理が楽

実務での推奨

データベースの列と同じ値を扱う変数は、%TYPE属性を使うことが推奨されています。
テーブル定義が変更されても、PL/SQLプログラムを修正する必要がなくなります。

4.3 複数列の取得 - まとめて複数のデータを取り出す

SELECT INTO句では、複数の列を一度に取得して、それぞれ別の変数に格納できます。
列の順番と変数の順番を一致させることが重要です。

複数列を取得する例 
DECLARE
    -- 社員情報を格納する変数（%TYPE属性を使用）
    v_emp_name    employees.employee_name%TYPE;
    v_department  employees.department%TYPE;
    v_salary      employees.salary%TYPE;
    v_hire_date   employees.hire_date%TYPE;

BEGIN
    -- 社員ID 1001の情報を取得
    SELECT employee_name, department, salary, hire_date
    INTO   v_emp_name, v_department, v_salary, v_hire_date
    FROM   employees
    WHERE  employee_id = 1001;

    -- 取得した情報を表示
    DBMS_OUTPUT.PUT_LINE('=== 社員情報 ===');
    DBMS_OUTPUT.PUT_LINE('氏名: ' || v_emp_name);
    DBMS_OUTPUT.PUT_LINE('部署: ' || v_department);
    DBMS_OUTPUT.PUT_LINE('給与: ' || v_salary || '円');
    DBMS_OUTPUT.PUT_LINE('入社日: ' || v_hire_date);
END;
/

列と変数の順番を一致させる

SELECT句の列の順番とINTO句の変数の順番は、必ず一致させてください。
順番が違うと、データが正しく格納されません（型エラーになることも）。

4.4 集計関数の利用 - COUNT、SUM、AVGの結果を取得

SELECT INTO句では、集計関数（COUNT、SUM、AVGなど）の結果も変数に格納できます。
データ件数の取得や、合計・平均の計算結果を使った処理が可能です。

例1: データ件数を取得（COUNT）

COUNT関数で件数を取得 
DECLARE
    v_product_count NUMBER;  -- 商品件数を入れる変数

BEGIN
    -- 在庫が10個以下の商品件数を取得
    SELECT COUNT(*)
    INTO   v_product_count
    FROM   products
    WHERE  stock <= 10;

    -- 結果を表示
    DBMS_OUTPUT.PUT_LINE('在庫切れ間近の商品: ' || v_product_count || '件');

    -- 条件分岐で警告を出す
    IF v_product_count > 0 THEN
        DBMS_OUTPUT.PUT_LINE('警告: 発注が必要です！');
    END IF;
END;
/

例2: 合計と平均を取得（SUM、AVG）

SUM、AVG関数で合計・平均を取得 
DECLARE
    v_total_sales  NUMBER;  -- 売上合計
    v_avg_sales    NUMBER;  -- 売上平均

BEGIN
    -- 2024年1月の売上合計と平均を取得
    SELECT SUM(sales_amount), AVG(sales_amount)
    INTO   v_total_sales, v_avg_sales
    FROM   sales
    WHERE  sales_date BETWEEN DATE '2024-01-01' AND DATE '2024-01-31';

    -- 結果を表示
    DBMS_OUTPUT.PUT_LINE('=== 2024年1月 売上集計 ===');
    DBMS_OUTPUT.PUT_LINE('合計: ' || v_total_sales || '円');
    DBMS_OUTPUT.PUT_LINE('平均: ' || ROUND(v_avg_sales, 0) || '円');
END;
/

集計関数は必ず1件を返す

COUNT、SUM、AVGなどの集計関数は、必ず1件の結果を返します（データが0件でもNULLが返る）。
そのため、NO_DATA_FOUNDエラーは発生しません。

4.5 データが見つからない場合の対処 - NO_DATA_FOUNDエラー

SELECT INTO句で該当データが0件の場合、NO_DATA_FOUNDエラーが発生します。
例外処理を使って、エラーを適切に処理しましょう。

NO_DATA_FOUNDエラーの処理 
DECLARE
    v_product_name products.product_name%TYPE;
    v_price        products.price%TYPE;

BEGIN
    -- 存在しない商品IDを検索（エラーになる可能性）
    SELECT product_name, price
    INTO   v_product_name, v_price
    FROM   products
    WHERE  product_id = 99999;  -- 存在しないID

    DBMS_OUTPUT.PUT_LINE('商品名: ' || v_product_name);

EXCEPTION
    -- データが見つからない場合の処理
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('該当する商品が見つかりませんでした');

    -- その他のエラー
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('エラーが発生しました: ' || SQLERRM);
END;
/

エラーの種類

  • NO_DATA_FOUND: SELECT文で該当データが0件のとき
  • TOO_MANY_ROWS: SELECT文で該当データが2件以上のとき

SELECT INTO句は必ず1件のデータを返す必要があります。

実習: データ取得プログラムを作ろう

以下の要件を満たすPL/SQLプログラムを作成してください。

要件

  1. 社員ID 1005の社員情報を取得する
  2. 氏名、部署、給与を変数に格納する（%TYPE属性を使用）
  3. 給与が30万円以上なら「高給与」、未満なら「標準給与」と表示する
  4. 社員が見つからない場合は「該当社員なし」と表示する
解答例を表示
解答例 
DECLARE
    v_emp_name   employees.employee_name%TYPE;
    v_department employees.department%TYPE;
    v_salary     employees.salary%TYPE;

BEGIN
    SELECT employee_name, department, salary
    INTO   v_emp_name, v_department, v_salary
    FROM   employees
    WHERE  employee_id = 1005;

    DBMS_OUTPUT.PUT_LINE('氏名: ' || v_emp_name);
    DBMS_OUTPUT.PUT_LINE('部署: ' || v_department);
    DBMS_OUTPUT.PUT_LINE('給与: ' || v_salary || '円');

    IF v_salary >= 300000 THEN
        DBMS_OUTPUT.PUT_LINE('判定: 高給与');
    ELSE
        DBMS_OUTPUT.PUT_LINE('判定: 標準給与');
    END IF;

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('該当社員なし');
END;
/

理解度確認クイズ

  1. Q1. SELECT INTO句で気をつけることは何ですか？

    回答を表示

    SELECT文は必ず1件のデータを返す必要があります。0件の場合はNO_DATA_FOUNDエラー、2件以上の場合はTOO_MANY_ROWSエラーが発生します。
    また、SELECT句の列数とINTO句の変数の数を一致させる必要があります。

  2. Q2. %TYPE属性を使うメリットは何ですか？

    回答を表示

    テーブルの列と同じデータ型を自動的に使用できるため、型の不一致を防ぐことができます。
    また、テーブル定義が変更されても、PL/SQLプログラムを修正する必要がありません（自動的に追従）。

  3. Q3. COUNT関数でNO_DATA_FOUNDエラーが発生しない理由は？

    回答を表示

    COUNT、SUM、AVGなどの集計関数は、必ず1件の結果を返すためです。
    データが0件の場合でも、COUNTは「0」を返し、SUMやAVGはNULLを返します。

第4章のまとめ

  • SELECT INTO句を使って、データベースから値を取得して変数に格納できる
  • SELECT文は必ず1件のデータを返す必要がある（0件や2件以上はエラー）
  • %TYPE属性を使うと、テーブルの列と同じデータ型を安全に使用できる
  • 集計関数（COUNT、SUM、AVG）の結果も変数に格納できる
  • NO_DATA_FOUNDエラーは例外処理で適切に対処する
