複数の処理をパッケージ化 - 関連する機能をまとめて管理

関連するプロシージャやファンクションをパッケージとしてまとめる方法を学習します。「社員管理パッケージ」「在庫管理パッケージ」など、機能別に整理する方法を習得します。

この章で学ぶこと

  • パッケージとは何か、なぜ必要なのかを理解する
  • パッケージ仕様部とパッケージ本体部の違いを知る
  • パッケージの作成方法と使用方法を習得する
  • 関連する機能を整理して管理できるようになる

11.1 パッケージとは - 関連する処理を一つにまとめる

これまでに学習したプロシージャやファンクションは、それぞれ個別に作成・管理していました。
しかし、実際の業務システムでは「社員管理」「在庫管理」「売上管理」など、関連する処理が複数存在します。
パッケージ（Package）は、関連するプロシージャやファンクションを一つのまとまり（グループ）として管理する仕組みです。

パッケージのイメージ

パッケージは「道具箱」のようなものです。
例えば、大工さんの道具箱には「ノコギリ」「カンナ」「カナヅチ」など、木工作業に必要な道具がまとめて入っています。
同様に、PL/SQLのパッケージには「社員登録プロシージャ」「社員更新プロシージャ」「社員給与計算ファンクション」など、社員管理に必要な処理がまとめて入っています。

パッケージを使わない場合の問題点

個別にプロシージャやファンクションを作成すると、以下のような問題が発生します：

個別管理の問題点

  • どの処理がどの業務に関連しているか分かりにくい
  • 処理の数が増えると管理が大変
  • 同じ機能を持つ処理を見つけにくい
  • 変更時に影響範囲が把握しづらい

11.2 パッケージの構造 - 仕様部と本体部

PL/SQLのパッケージは、2つの部分で構成されています：

① パッケージ仕様部（Package Specification）

パッケージの「外から見える部分」を定義します。
どのようなプロシージャやファンクションが使えるかを宣言する「目次」や「カタログ」のようなものです。

② パッケージ本体部（Package Body）

仕様部で宣言したプロシージャやファンクションの実際の処理内容を記述します。
「目次」に対する「本文」のようなものです。

本に例えると

パッケージ仕様部 = 本の「目次」（どんな章があるか）
パッケージ本体部 = 本の「本文」（各章の詳しい内容）

パッケージの作成順序

flowchart LR A[1. パッケージ仕様部を作成] --> B[2. パッケージ本体部を作成] B --> C[3. パッケージを使用] style A fill:#dbeafe,stroke:#3b82f6,stroke-width:2px style B fill:#d1fae5,stroke:#10b981,stroke-width:2px style C fill:#fef3c7,stroke:#f59e0b,stroke-width:2px

重要な順序

パッケージは必ず仕様部から作成します。
本体部だけを先に作ることはできません（目次のない本は作れないのと同じ）。

11.3 パッケージの作成と使用 - 社員管理パッケージの例

実際に簡単な社員管理パッケージを作成してみましょう。
このパッケージには、社員情報を表示するプロシージャと、給与を計算するファンクションを含めます。

手順1: パッケージ仕様部の作成

まず、パッケージの「目次」となる仕様部を作成します。

パッケージ仕様部の作成 
-- パッケージ仕様部の作成
CREATE OR REPLACE PACKAGE employee_pkg AS
    -- 社員情報を表示するプロシージャ
    PROCEDURE show_employee_info(p_emp_id IN NUMBER);

    -- 給与を計算するファンクション（基本給 + 手当）
    FUNCTION calculate_salary(
        p_base_salary IN NUMBER,
        p_allowance IN NUMBER
    ) RETURN NUMBER;
END employee_pkg;
/

仕様部のポイント

  • CREATE OR REPLACE PACKAGE でパッケージ仕様部を作成
  • プロシージャやファンクションの名前とパラメータだけを宣言（処理内容は書かない）
  • AS と END パッケージ名; で囲む

手順2: パッケージ本体部の作成

次に、仕様部で宣言したプロシージャとファンクションの実際の処理を記述します。

パッケージ本体部の作成 
-- パッケージ本体部の作成
CREATE OR REPLACE PACKAGE BODY employee_pkg AS

    -- 社員情報を表示するプロシージャの実装
    PROCEDURE show_employee_info(p_emp_id IN NUMBER) IS
        v_emp_name VARCHAR2(100);
        v_salary NUMBER;
    BEGIN
        -- 社員情報を取得（仮のデータ）
        SELECT 'サンプル太郎', 300000 INTO v_emp_name, v_salary
        FROM DUAL;

        DBMS_OUTPUT.PUT_LINE('社員ID: ' || p_emp_id);
        DBMS_OUTPUT.PUT_LINE('社員名: ' || v_emp_name);
        DBMS_OUTPUT.PUT_LINE('給与: ' || v_salary || '円');
    EXCEPTION
        WHEN NO_DATA_FOUND THEN
            DBMS_OUTPUT.PUT_LINE('社員が見つかりません');
    END show_employee_info;

    -- 給与計算ファンクションの実装
    FUNCTION calculate_salary(
        p_base_salary IN NUMBER,
        p_allowance IN NUMBER
    ) RETURN NUMBER IS
        v_total_salary NUMBER;
    BEGIN
        v_total_salary := p_base_salary + p_allowance;
        RETURN v_total_salary;
    END calculate_salary;

END employee_pkg;
/

本体部のポイント

  • CREATE OR REPLACE PACKAGE BODY でパッケージ本体部を作成
  • 仕様部で宣言したプロシージャ・ファンクションの実際の処理を記述
  • 各プロシージャ・ファンクションは個別に BEGIN〜END; を持つ

手順3: パッケージの使用

作成したパッケージのプロシージャやファンクションを呼び出すには、パッケージ名.プロシージャ名 の形式を使います。

パッケージの使用例 
-- パッケージの使用例
DECLARE
    v_total_salary NUMBER;
BEGIN
    -- プロシージャの呼び出し
    employee_pkg.show_employee_info(101);

    -- ファンクションの呼び出し
    v_total_salary := employee_pkg.calculate_salary(250000, 50000);
    DBMS_OUTPUT.PUT_LINE('合計給与: ' || v_total_salary || '円');
END;
/

呼び出しの形式

パッケージ内の処理を呼び出すときは、パッケージ名とプロシージャ名をドット（.）でつなぎます。
例: employee_pkg.show_employee_info(101)

11.4 パッケージの利点 - なぜパッケージを使うのか

パッケージを使うことで、以下のような多くの利点があります：

1. 整理整頓

関連する処理をグループ化することで、プログラムが整理され、理解しやすくなります

2. カプセル化

仕様部に公開したい処理だけを宣言し、内部処理を隠すことができます

3. パフォーマンス向上

パッケージは一度メモリに読み込まれると、複数の処理を高速に実行できます

4. チーム開発

機能ごとにパッケージを分けることで、複数人での開発がしやすくなります

実務での活用例

大規模なシステムでは、以下のようにパッケージを分けて管理します：
・EMPLOYEE_PKG - 社員管理関連の処理
・INVENTORY_PKG - 在庫管理関連の処理
・ORDER_PKG - 注文管理関連の処理
・UTIL_PKG - 共通ユーティリティ処理

実習 11-1: 在庫管理パッケージを作ろう

在庫数を確認するファンクションと、在庫を更新するプロシージャを含む、在庫管理パッケージを作成してみましょう。

手順

  1. 1 パッケージ仕様部を作成（INVENTORY_PKG）
  2. 2 在庫確認ファンクション（check_stock）を仕様部に宣言
  3. 3 在庫更新プロシージャ（update_stock）を仕様部に宣言
  4. 4 パッケージ本体部を作成し、各処理の実装を記述
  5. 5 作成したパッケージを使って在庫を確認・更新
実習のヒント: パッケージ仕様部 
-- 在庫管理パッケージ仕様部
CREATE OR REPLACE PACKAGE inventory_pkg AS
    -- 在庫数を確認するファンクション
    FUNCTION check_stock(p_product_id IN NUMBER) RETURN NUMBER;

    -- 在庫を更新するプロシージャ
    PROCEDURE update_stock(
        p_product_id IN NUMBER,
        p_quantity IN NUMBER
    );
END inventory_pkg;
/

動作確認

作成したパッケージを実際に使ってみましょう。

パッケージ使用例 
-- パッケージの使用例
DECLARE
    v_current_stock NUMBER;
BEGIN
    -- 在庫確認
    v_current_stock := inventory_pkg.check_stock(101);
    DBMS_OUTPUT.PUT_LINE('現在の在庫: ' || v_current_stock);

    -- 在庫更新（10個追加）
    inventory_pkg.update_stock(101, 10);
    DBMS_OUTPUT.PUT_LINE('在庫を10個追加しました');

    -- 更新後の在庫確認
    v_current_stock := inventory_pkg.check_stock(101);
    DBMS_OUTPUT.PUT_LINE('更新後の在庫: ' || v_current_stock);
END;
/

期待される結果 

現在の在庫: 100
在庫を10個追加しました
更新後の在庫: 110

トラブルシューティング

パッケージ本体部でエラー「識別子が無効です」

原因: 仕様部で宣言した名前と、本体部の名前が一致していません

解決方法: パッケージ仕様部と本体部で、プロシージャ・ファンクション名、パラメータ名、パラメータの型が完全に一致しているか確認してください

パッケージが見つからない（存在しないオブジェクト）

原因: パッケージ仕様部が作成されていない、または作成に失敗しています

解決方法: まず、パッケージ仕様部を作成してから本体部を作成してください。パッケージの作成順序を確認しましょう

呼び出し時「メソッドが見つかりません」

原因: パッケージ名またはプロシージャ/ファンクション名の指定が間違っています

解決方法: パッケージ名.プロシージャ名 の形式が正しいか、スペルミスがないか確認してください

理解度確認クイズ

  1. Q1. パッケージの2つの構成要素は何ですか？

    回答を表示

    ① パッケージ仕様部（Package Specification） - 外から見える部分を定義する「目次」
    ② パッケージ本体部（Package Body） - 実際の処理内容を記述する「本文」

  2. Q2. パッケージ内のプロシージャを呼び出すときの形式は？

    回答を表示

    パッケージ名.プロシージャ名(パラメータ) の形式です。
    例: employee_pkg.show_employee_info(101);

  3. Q3. パッケージを使う利点を3つ挙げてください

    回答を表示

    ① 整理整頓 - 関連する処理をグループ化して管理しやすくなる
    ② カプセル化 - 公開する処理だけを仕様部に宣言し、内部処理を隠せる
    ③ パフォーマンス向上 - 一度メモリに読み込まれると高速に実行できる

第11章のまとめ

  • パッケージは、関連するプロシージャやファンクションを一つのまとまりとして管理する仕組み
  • パッケージは仕様部（目次）と本体部（本文）の2つで構成される
  • パッケージを作成するときは、必ず仕様部から作成する
  • パッケージ内の処理を呼び出すときは パッケージ名.プロシージャ名 の形式を使う
  • パッケージを使うことで、整理整頓・カプセル化・パフォーマンス向上・チーム開発などの利点が得られる
