PostgreSQLの環境構築 - インストールとpgAdminの使い方

PostgreSQLのダウンロードからインストール、初期設定までを画面キャプチャ付きで丁寧に解説します。pgAdminの基本操作（サーバー接続、データベース作成、SQLエディタの使い方）を習得します。

この章で学ぶこと

  • PostgreSQLのインストール方法（Windows/Mac/Linux）
  • 初期設定（ポート番号、スーパーユーザーパスワード）の方法
  • pgAdminの基本操作（サーバー接続、データベース作成）
  • SQLエディタでクエリを実行する方法

2.1 PostgreSQLのインストール

PostgreSQLは、Windows、Mac、Linuxの各OSに対応しています。 公式サイトから各OS用のインストーラーをダウンロードしてインストールできます。 ここでは、各OSごとのインストール手順を解説します。

全apt/yum/dnf全"] Win --> Download["公式サイトからダウンロード"] Mac --> Download Linux --> PackageManager["パッケージマネージャーで

ダウンロード先

PostgreSQL公式ダウンロードページ: https://www.postgresql.org/download/
お使いのOSを選択して、最新の安定版（Stable Release）をダウンロードしてください。

2.2 Windowsへのインストール手順

Windows環境では、EnterpriseDB社が提供するインストーラーを使用します。 GUIベースのウィザード形式で、簡単にインストールできます。

1

インストーラーのダウンロード

公式サイトから「Windows x86-64」版のインストーラーをダウンロードします。 バージョンは最新の安定版（例: PostgreSQL 16.x）を選択してください。

2

インストーラーの実行

ダウンロードしたインストーラー（postgresql-16.x-windows-x64.exe）をダブルクリックして実行します。 「このアプリがデバイスに変更を加えることを許可しますか？」と表示されたら「はい」をクリックします。

3

インストール先の選択

インストール先ディレクトリを指定します。特に理由がなければデフォルト（C:\Program Files\PostgreSQL\16）のまま「Next」をクリックします。

4

コンポーネントの選択

インストールするコンポーネントを選択します。以下をすべてチェックして「Next」をクリック：
・PostgreSQL Server（本体）
・pgAdmin 4（管理ツール）
・Command Line Tools（コマンドラインツール）
・Stack Builder（追加機能インストーラー、任意）

5

スーパーユーザーパスワードの設定

PostgreSQLの管理者（postgres）のパスワードを設定します。 このパスワードは忘れないようにメモしておいてください。

重要: このパスワードは後で使用するため、必ず覚えておくか安全な場所にメモしてください。

6

ポート番号の設定

PostgreSQLが使用するポート番号を指定します。デフォルトは5432です。 特に理由がなければデフォルトのまま「Next」をクリックします。

7

ロケールの設定

文字コードとロケールを設定します。デフォルト（Default locale）のまま「Next」をクリックします。 日本語環境では自動的に「Japanese_Japan」が選択されます。

8

インストール実行

設定内容を確認して「Next」をクリックすると、インストールが開始されます。 インストールには数分かかります。完了したら「Finish」をクリックします。

2.3 macOSへのインストール手順

macOS環境では、公式インストーラーまたはHomebrewを使用してインストールできます。 ここでは両方の方法を紹介しますが、初心者には公式インストーラーをお勧めします。

方法1: 公式インストーラー（推奨）

GUIベースで簡単にインストールできます。pgAdminも同時にインストールされます。

方法2: Homebrew

コマンドラインで素早くインストールできます。開発者向けの方法です。

方法1: 公式インストーラーを使用

Windows版とほぼ同じ手順でインストールできます。公式サイトから「macOS」版インストーラーをダウンロードして実行してください。

方法2: Homebrewを使用

Homebrewがインストールされている場合、以下のコマンドでPostgreSQLをインストールできます。

ターミナル 
# PostgreSQLのインストール
brew install postgresql@16

# サービスとして起動
brew services start postgresql@16

# pgAdminのインストール（別途）
brew install --cask pgadmin4

2.4 pgAdminの起動と初期設定

pgAdminは、PostgreSQLを視覚的に操作できる管理ツールです。 データベースの作成、テーブルの管理、SQLクエリの実行など、様々な操作をGUIで行えます。

flowchart TD Start["pgAdminを起動"] --> Browser["Webブラウザで
管理画面が開く"] Browser --> MasterPW["マスターパスワード設定
全初回のみ全"] MasterPW --> AddServer["サーバー接続設定"] AddServer --> Connect["PostgreSQLに接続"] Connect --> Ready["管理画面を使用可能"]
1

pgAdminの起動

Windows: スタートメニューから「pgAdmin 4」を検索して起動
macOS: Launchpadまたはアプリケーションフォルダから「pgAdmin 4」を起動
起動すると、Webブラウザが自動的に開き、pgAdminの管理画面が表示されます。

2

マスターパスワードの設定（初回のみ）

初回起動時、pgAdminのマスターパスワードを設定するよう求められます。 このパスワードは、pgAdminを起動するたびに入力が必要になります。 覚えやすいパスワードを設定してください。

3

サーバー接続設定

左側のツリーから「Servers」を展開し、「PostgreSQL 16」（バージョン番号は環境により異なる）をクリックします。 パスワードを求められたら、インストール時に設定したスーパーユーザー（postgres）のパスワードを入力して「OK」をクリックします。

接続成功

正常に接続されると、左側のツリーに「Databases」「Login/Group Roles」などの項目が表示されます。 これでPostgreSQLの管理ができる状態になりました。

2.5 pgAdminでデータベースを作成する

pgAdminを使って、練習用のデータベースを作成してみましょう。 データベースは、テーブルやデータを格納する箱のようなものです。

1

Databasesを右クリック

左側のツリーから「Servers」→「PostgreSQL 16」→「Databases」を右クリックして、「Create」→「Database...」を選択します。

2

データベース名を入力

「Database」欄に、作成したいデータベース名を入力します。例: practice_db
その他の設定（Owner、Encoding等）はデフォルトのままで構いません。

3

Saveをクリック

右下の「Save」ボタンをクリックすると、データベースが作成されます。 左側のツリーに作成したデータベース名が表示されれば成功です。

SQL（参考: コマンドラインで作成する場合） 
-- データベースの作成
CREATE DATABASE practice_db;

2.6 SQLエディタでクエリを実行する

pgAdminのSQLエディタを使って、SQLクエリを実行してみましょう。 ここでは、簡単なテーブルを作成し、データを挿入・取得する操作を行います。

1

Query Toolを開く

左側のツリーから作成したデータベース（practice_db）を選択し、上部メニューの「Tools」→「Query Tool」をクリックします。 またはデータベースを選択した状態で、ツールバーの「Query Tool」アイコンをクリックします。

2

SQLを入力

Query Toolの上部エリアに、実行したいSQLを入力します。 複数のSQL文を入力できますが、セミコロン（;）で区切る必要があります。

3

実行ボタンをクリック

ツールバーの「Execute/Refresh」ボタン（▶アイコン）をクリックすると、SQLが実行されます。 実行結果は下部の「Data Output」タブに表示されます。

実習: テーブルを作成してデータを操作する

以下のSQLを実行して、テーブルの作成とデータ操作を体験しましょう。

SQL 
-- テーブルの作成
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- データの挿入
INSERT INTO users (name, email) VALUES
('山田太郎', 'yamada@example.com'),
('佐藤花子', 'sato@example.com'),
('鈴木一郎', 'suzuki@example.com');

-- データの取得
SELECT * FROM users;

Query Toolに上記のSQLをコピー&ペーストして、▶ボタンをクリックしてください。 下部に3件のユーザーデータが表示されれば成功です。

2.7 トラブルシューティング

インストールや接続時によくあるエラーとその対処法を紹介します。

よくあるエラーと対処法

エラー1: pgAdminでサーバーに接続できない

原因: PostgreSQLサービスが起動していない可能性があります。

対処法:
・Windows: サービス（services.msc）から「postgresql-x64-16」を確認し、停止していれば起動
・macOS: ターミナルで brew services start postgresql@16 を実行

エラー2: パスワード認証に失敗する

原因: パスワードが間違っている、またはユーザー名が間違っている可能性があります。

対処法:
・ユーザー名が「postgres」になっているか確認
・インストール時に設定したパスワードを再確認
・どうしても思い出せない場合は、PostgreSQLを再インストール

エラー3: ポート5432が既に使用されている

原因: 他のアプリケーションがポート5432を使用しています。

対処法:
・PostgreSQLの設定ファイル（postgresql.conf）でポート番号を変更（例: 5433）
・または、ポートを使用している他のアプリケーションを停止

エラー4: pgAdminが起動しない（Webページが開かない）

原因: pgAdminのサービスが起動していない可能性があります。

対処法:
・pgAdmin 4を再起動
・ブラウザで手動でアクセス: http://127.0.0.1:5050
・それでもダメな場合は、pgAdmin 4を再インストール

理解度確認クイズ

  1. Q1. PostgreSQLのデフォルトポート番号は何番ですか？

    回答を表示

    答え: 5432番
    PostgreSQLはデフォルトで5432番ポートを使用します。インストール時に変更することもできますが、特に理由がなければデフォルトのまま使用します。

  2. Q2. pgAdminとは何ですか？その役割を説明してください。

    回答を表示

    答え: pgAdminは、PostgreSQLを視覚的に操作できるGUI管理ツールです。
    主な役割は、データベースの作成・削除、テーブルの管理、SQLクエリの実行、データのインポート/エクスポート、ユーザー権限の管理など、PostgreSQLの様々な操作をGUIで簡単に行えるようにすることです。コマンドラインに不慣れな初心者でも直感的に操作できます。

  3. Q3. PostgreSQLのスーパーユーザーのデフォルトユーザー名は何ですか？

    回答を表示

    答え: postgres
    PostgreSQLをインストールすると、デフォルトで「postgres」という名前のスーパーユーザーが作成されます。このユーザーはすべての権限を持ち、データベースの作成・削除、ユーザー管理など、すべての管理操作を実行できます。pgAdminでの初回接続時にもこのユーザー名を使用します。

第2章のまとめ

  • PostgreSQLは公式インストーラーまたはパッケージマネージャーで簡単にインストールできる
  • インストール時にスーパーユーザー（postgres）のパスワードを設定し、必ずメモしておく
  • pgAdminはWebブラウザベースのGUI管理ツールで、直感的にデータベースを操作できる
  • Query ToolでSQLを実行し、テーブルの作成やデータ操作ができる
  • 接続エラーやパスワード認証エラーは、サービスの起動状態や設定を確認して対処する
