4.1 DB2のデータ型

DB2は標準SQLのデータ型をサポートしつつ、独自の拡張データ型も提供しています。適切なデータ型の選択はパフォーマンスとストレージ効率に影響します。

数値データ型

データ型 サイズ 範囲 用途
SMALLINT 2バイト -32,768 ～ 32,767 小さな整数
INTEGER (INT) 4バイト -2,147,483,648 ～ 2,147,483,647 一般的な整数
BIGINT 8バイト -9,223,372,036,854,775,808 ～ 9,223,372,036,854,775,807 大きな整数
DECIMAL(p,s) 可変 精度とスケール指定 正確な小数
REAL 4バイト 単精度浮動小数点 近似値
DOUBLE 8バイト 倍精度浮動小数点 高精度近似値

文字列データ型

データ型 最大長 格納方法 用途
CHAR(n) 254文字 固定長 固定長文字列
VARCHAR(n) 32,672文字 可変長 一般的な文字列
CLOB 2GB LOB 大きなテキスト
実習 4-1: DB2データ型の使用

様々なDB2データ型を使用してテーブルを作成し、特性を確認します。

実行するSQL
-- データ型テスト用テーブルの作成
CREATE TABLE datatype_test (
    id INTEGER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    small_num SMALLINT,
    big_num BIGINT,
    precise_num DECIMAL(10,2),
    float_num REAL,
    fixed_text CHAR(10),
    var_text VARCHAR(50),
    created_date DATE,
    created_time TIMESTAMP
);

-- データの挿入
INSERT INTO datatype_test (
    small_num, big_num, precise_num, float_num, 
    fixed_text, var_text, created_date, created_time
) VALUES (
    100, 1234567890123, 123.45, 123.456789,
    'FIXED', 'Variable length text', CURRENT DATE, CURRENT TIMESTAMP
);

-- データの確認
SELECT * FROM datatype_test;

-- 文字列長の確認
SELECT 
    LENGTH(fixed_text) as fixed_len,
    LENGTH(var_text) as var_len,
    LENGTH(RTRIM(fixed_text)) as fixed_trimmed_len
FROM datatype_test;
確認ポイント
  • CHAR型は指定長まで空白でパディングされる
  • VARCHAR型は実際の文字数のみ格納される
  • DECIMAL型は正確な小数値を保持する
  • REAL型は近似値として格納される

4.2 スキーマの概念と管理

DB2のスキーマは、データベースオブジェクトを論理的にグループ化するための名前空間です。適切なスキーマ設計により、オブジェクトの整理と権限管理が効率化されます。

実習 4-2: スキーマの作成と使用

新しいスキーマを作成し、スキーマ内でのオブジェクト管理を学習します。

実行するSQL
-- 新しいスキーマの作成
CREATE SCHEMA sales_dept;

-- スキーマを指定してテーブルを作成
CREATE TABLE sales_dept.products (
    product_id INTEGER NOT NULL PRIMARY KEY,
    product_name VARCHAR(100) NOT NULL,
    category VARCHAR(50),
    price DECIMAL(10,2),
    stock_quantity INTEGER DEFAULT 0
);

-- スキーマを指定してデータを挿入
INSERT INTO sales_dept.products VALUES
(1, 'Laptop Computer', 'Electronics', 899.99, 50),
(2, 'Office Chair', 'Furniture', 199.99, 25),
(3, 'Desk Lamp', 'Furniture', 49.99, 100);

-- 現在のスキーマの確認
VALUES CURRENT SCHEMA;

-- スキーマのパスを設定
SET SCHEMA sales_dept;

-- スキーマ設定後は修飾名不要
SELECT * FROM products;

-- 元のスキーマに戻す
SET SCHEMA DB2INST1;

-- スキーマ内のテーブル一覧を確認
SELECT TABSCHEMA, TABNAME 
FROM SYSCAT.TABLES 
WHERE TABSCHEMA = 'SALES_DEPT';
期待される結果
  • SALES_DEPTスキーマが作成される
  • スキーマ内にproductsテーブルが作成される
  • SET SCHEMA文でカレントスキーマを変更できる
  • システムカタログからスキーマ情報を取得できる

4.3 制約の実装

DB2は標準SQLの制約をサポートし、データの整合性を保証します。適切な制約設定により、不正なデータの挿入を防止できます。

実習 4-3: 制約の作成と確認

様々な制約を設定したテーブルを作成し、制約の動作を確認します。

実行するSQL
-- 制約を含むテーブルの作成
CREATE TABLE sales_dept.customers (
    customer_id INTEGER NOT NULL PRIMARY KEY,
    email VARCHAR(100) NOT NULL UNIQUE,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    age INTEGER CHECK (age >= 18 AND age <= 120),
    status VARCHAR(10) DEFAULT 'ACTIVE' CHECK (status IN ('ACTIVE', 'INACTIVE', 'PENDING')),
    created_date DATE NOT NULL DEFAULT CURRENT DATE
);

-- 注文テーブル（外部キー制約付き）
CREATE TABLE sales_dept.orders (
    order_id INTEGER NOT NULL PRIMARY KEY,
    customer_id INTEGER NOT NULL,
    order_date DATE NOT NULL DEFAULT CURRENT DATE,
    total_amount DECIMAL(10,2) CHECK (total_amount > 0),
    FOREIGN KEY (customer_id) REFERENCES sales_dept.customers(customer_id)
);

-- 正常なデータの挿入
INSERT INTO sales_dept.customers (customer_id, email, first_name, last_name, age) 
VALUES (1, 'john@example.com', 'John', 'Doe', 30);

INSERT INTO sales_dept.orders (order_id, customer_id, total_amount) 
VALUES (1001, 1, 150.00);

-- 制約違反のテスト（エラーになることを確認）
-- CHECK制約違反のテスト
-- INSERT INTO sales_dept.customers (customer_id, email, first_name, last_name, age) 
-- VALUES (2, 'jane@example.com', 'Jane', 'Smith', 15);

-- UNIQUE制約違反のテスト
-- INSERT INTO sales_dept.customers (customer_id, email, first_name, last_name, age) 
-- VALUES (3, 'john@example.com', 'Another', 'John', 25);

-- 制約情報の確認
SELECT 
    CONSTNAME,
    COLNAME,
    TYPE,
    TEXT
FROM SYSCAT.COLCHECKS
WHERE TABSCHEMA = 'SALES_DEPT' AND TABNAME = 'CUSTOMERS';
制約の種類と効果
  • PRIMARY KEY: 主キー制約（NOT NULL + UNIQUE）
  • NOT NULL: NULL値を禁止
  • UNIQUE: 重複値を禁止
  • CHECK: 値が指定条件を満たすことを保証
  • FOREIGN KEY: 参照整合性を保証
  • DEFAULT: デフォルト値を設定

4.4 シーケンスの作成と使用

DB2のシーケンスは、一意の連続番号を生成するためのオブジェクトです。主キーの値生成や、アプリケーションでの連番管理に使用されます。

実習 4-4: シーケンスの作成と使用

シーケンスを作成し、テーブルのデータ挿入で活用する方法を学習します。

実行するSQL
-- シーケンスの作成
CREATE SEQUENCE sales_dept.customer_seq
    START WITH 1000
    INCREMENT BY 1
    MINVALUE 1000
    MAXVALUE 999999
    CYCLE
    CACHE 10;

-- シーケンスを使用するテーブルの作成
CREATE TABLE sales_dept.customer_records (
    record_id INTEGER DEFAULT NEXTVAL FOR sales_dept.customer_seq PRIMARY KEY,
    customer_name VARCHAR(100) NOT NULL,
    registration_date TIMESTAMP DEFAULT CURRENT TIMESTAMP
);

-- シーケンスを使用したデータ挿入
INSERT INTO sales_dept.customer_records (customer_name) 
VALUES ('Alice Johnson');

INSERT INTO sales_dept.customer_records (customer_name) 
VALUES ('Bob Wilson');

-- 次の値を直接取得
VALUES NEXTVAL FOR sales_dept.customer_seq;

-- 現在の値を確認
VALUES PREVVAL FOR sales_dept.customer_seq;

-- シーケンス情報の確認
SELECT 
    SEQSCHEMA,
    SEQNAME,
    START,
    INCREMENT,
    MINVALUE,
    MAXVALUE,
    CYCLE,
    CACHE
FROM SYSCAT.SEQUENCES
WHERE SEQSCHEMA = 'SALES_DEPT';

-- データの確認
SELECT * FROM sales_dept.customer_records;
シーケンスのオプション
  • START WITH: 開始値
  • INCREMENT BY: 増分値
  • MINVALUE/MAXVALUE: 最小値/最大値
  • CYCLE: 最大値到達後に最小値から再開
  • CACHE: パフォーマンス向上のためのキャッシュサイズ

4.5 DB2特有のSQL構文

DB2には標準SQLを拡張した独自の構文や機能があります。これらを理解することで、より効率的なSQLを記述できます。

実習 4-5: DB2特有のSQL機能

DB2固有のSQL構文と機能を実践的に学習します。

実行するSQL
-- VALUES文の使用（単発の値を生成）
VALUES CURRENT TIMESTAMP;
VALUES USER;
VALUES CURRENT SCHEMA;

-- WITH句を使用した共通テーブル式（CTE）
WITH monthly_sales AS (
    SELECT 
        MONTH(order_date) as month,
        SUM(total_amount) as monthly_total
    FROM sales_dept.orders
    GROUP BY MONTH(order_date)
)
SELECT * FROM monthly_sales;

-- CASE式の使用
SELECT 
    customer_id,
    first_name,
    last_name,
    age,
    CASE 
        WHEN age < 25 THEN 'Young'
        WHEN age BETWEEN 25 AND 50 THEN 'Middle'
        ELSE 'Senior'
    END as age_group
FROM sales_dept.customers;

-- ROW_NUMBER()ウィンドウ関数
SELECT 
    ROW_NUMBER() OVER(ORDER BY total_amount DESC) as rank,
    order_id,
    customer_id,
    total_amount
FROM sales_dept.orders;

-- MERGE文（UPSERT操作）
MERGE INTO sales_dept.customers AS target
USING (VALUES (1, 'john.doe@email.com', 'John', 'Doe', 31)) 
    AS source(customer_id, email, first_name, last_name, age)
ON target.customer_id = source.customer_id
WHEN MATCHED THEN 
    UPDATE SET age = source.age
WHEN NOT MATCHED THEN 
    INSERT (customer_id, email, first_name, last_name, age)
    VALUES (source.customer_id, source.email, source.first_name, source.last_name, source.age);

-- 結果の確認
SELECT * FROM sales_dept.customers;
DB2特有の機能
  • VALUES文: 単一の値や行を生成
  • WITH句: 共通テーブル式でサブクエリを再利用
  • MERGE文: INSERT/UPDATE/DELETEを統合
  • ウィンドウ関数: 分析的な処理に使用
  • GENERATED ALWAYS AS IDENTITY: 自動採番列

4.6 DB2と標準SQLの違い

DB2は標準SQLに準拠していますが、独自の拡張や制限があります。他のデータベースから移行する際は、これらの違いを理解することが重要です。

主な違いと注意点
  • 終端記号: DB2ではセミコロン（;）が必須
  • 引用符: 識別子には二重引用符（"）を使用
  • NULL処理: COALESCE関数を推奨
  • 日付関数: CURRENT DATE, CURRENT TIME, CURRENT TIMESTAMP
  • 文字列連結: ||演算子または CONCAT関数
  • LIMIT句: FETCH FIRST n ROWS ONLYを使用
⚠️ 移植性に関する注意

DB2固有の機能を使用する際は、他のデータベースシステムとの互換性を考慮してください。可能な限り標準SQLの機能を使用することで、移植性を保つことができます。

理解度確認クイズ
  1. DB2でスキーマを変更するSQL文はどれですか？
    1. USE SCHEMA schema_name
    2. SET SCHEMA schema_name
    3. CHANGE SCHEMA schema_name
    4. SELECT SCHEMA schema_name
  2. DB2でシーケンスの次の値を取得する関数はどれですか？
    1. NEXT VALUE FOR sequence_name
    2. NEXTVAL FOR sequence_name
    3. SEQUENCE.NEXTVAL
    4. GET NEXT sequence_name
  3. DB2で現在のタイムスタンプを取得するにはどの関数を使用しますか？
    1. NOW()
    2. GETDATE()
    3. CURRENT TIMESTAMP
    4. SYSDATE
  4. DB2でLIMIT句と同等の機能を提供するのはどれですか？
    1. TOP n
    2. ROWNUM <= n
    3. FETCH FIRST n ROWS ONLY
    4. LIMIT n
解答を見る

解答:

  1. b) SET SCHEMA schema_name
  2. b) NEXTVAL FOR sequence_name
  3. c) CURRENT TIMESTAMP
  4. c) FETCH FIRST n ROWS ONLY
