1.1 IBM DB2とは

IBM DB2は、IBM社が開発したリレーショナルデータベース管理システム（RDBMS）です。1982年に最初にリリースされ、現在では世界中の企業で使用されているエンタープライズグレードのデータベースシステムです。

DB2の主な特徴

  • エンタープライズグレード: 大規模な企業環境での使用を想定した高い信頼性とパフォーマンス
  • マルチプラットフォーム対応: Linux、Windows、AIX、z/OSなど様々なプラットフォームで動作
  • 高度なセキュリティ機能: データ暗号化、行レベルセキュリティ、監査機能など
  • スケーラビリティ: 小規模から大規模システムまで対応可能
  • 高可用性: クラスタリング、レプリケーション、自動フェイルオーバー機能

1.2 DB2のアーキテクチャ

DB2は、以下の主要コンポーネントで構成されています：

データベースインスタンス

DB2インスタンスは、DB2データベースマネージャーの論理的な環境です。1つのシステム上で複数のインスタンスを実行でき、各インスタンスは独立して動作します。

データベース

データベースは、関連するデータを格納する論理的なコンテナです。1つのインスタンスで複数のデータベースを管理できます。

テーブルスペース

テーブルスペースは、データベース内でテーブル、インデックス、一時的なデータを格納する論理的な単位です。物理的な記憶域の管理を抽象化します。

バッファープール

バッファープールは、データベースページをメモリ内に保持するための領域で、I/Oパフォーマンスの向上に重要な役割を果たします。

1.3 DB2 Community Edition

DB2 Community Editionは、開発者や学習者向けに無料で提供されているDB2のバージョンです。商用版の主要な機能を含みながら、制限事項があります。

Community Editionの制限事項

  • メモリ使用量: 最大16GB
  • CPU: 4コアまで
  • 商用利用の制限
  • 一部の高度な機能が制限される場合がある
⚠️ ライセンスに関する注意

DB2 Community Editionは開発・学習目的で使用できますが、商用環境での使用には制限があります。本格的な商用利用を検討する場合は、適切なライセンスを購入してください。

1.4 学習環境の準備

このコースでは、Docker環境を使用してDB2 Community Editionを実行します。以下の準備が必要です：

必要なシステム要件

  • Docker Desktop: 最新版のインストール
  • メモリ: 8GB以上（推奨12GB以上）
  • ディスク容量: 25GB以上の空き容量
  • IBM Developerアカウント: Docker Hubからのイメージ取得に必要
実習 1-1: IBM Developerアカウントの作成

DB2のDockerイメージを取得するために、IBM DeveloperアカウントとDocker Hubアカウントが必要です。

手順
  1. IBM Developer サイト（https://developer.ibm.com/）にアクセス
  2. 「Create an IBMid」をクリックしてアカウントを作成
  3. メール認証を完了
  4. Docker Hub（https://hub.docker.com/）でアカウントを作成（未作成の場合）
  5. Docker HubでIBM DB2ページ（https://hub.docker.com/r/ibmcom/db2）にアクセス
  6. 利用規約に同意してイメージへのアクセス権を取得
確認事項
  • IBM DeveloperアカウントでログインできることConfirm
  • Docker Hubでibmcom/db2イメージが表示されることを確認
実習 1-2: Docker環境の確認

DB2を実行するためのDocker環境が適切に動作することを確認します。

手順
  1. Docker Desktopを起動
  2. ターミナル（コマンドプロンプト）を開く
  3. 以下のコマンドでDockerが動作していることを確認：
実行例
docker --version
docker run hello-world
期待される結果
  • Dockerのバージョン情報が表示される
  • hello-worldコンテナが正常に実行される
  • "Hello from Docker!" のメッセージが表示される
