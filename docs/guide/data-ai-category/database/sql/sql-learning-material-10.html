データをまとめて集計しよう

COUNT・SUM・AVG・MAX・MINといった集計関数の使い方と、GROUP BY句でカテゴリごとに集計する方法を、売上分析などの実例で学習します。

この章で学ぶこと

  • 集計関数（COUNT, SUM, AVG, MAX, MIN）の使い方を理解する
  • GROUP BY句でカテゴリ別に集計する方法を学ぶ
  • HAVING句で集計結果を絞り込む方法を習得する
  • 売上分析など実務で役立つ集計テクニックを実践する

10.1 集計関数とは？

集計関数とは、複数の行のデータをまとめて計算する機能のことです。

例えば、「全商品の売上合計はいくらか？」「今月の注文件数は何件か？」「商品の平均価格はいくらか？」といった質問に答えるために使います。Excelの「SUM関数」や「AVERAGE関数」と同じように、SQLでもデータをまとめて計算できます。

ポイント

集計関数を使うと、手作業で計算する必要がなく、一瞬で結果が得られます。大量のデータを扱う場合は特に便利です。

10.2 基本的な集計関数

よく使う5つの集計関数を紹介します。まずは、練習用のテーブルを作成しましょう。

練習用テーブルの作成

売上データを管理する「sales」テーブルを作成します。

テーブル作成とデータ挿入 
-- 売上テーブルの作成
CREATE TABLE sales (
    sale_id SERIAL PRIMARY KEY,      -- 売上ID
    product_name VARCHAR(100),       -- 商品名
    category VARCHAR(50),            -- カテゴリ
    price INTEGER,                   -- 価格
    quantity INTEGER,                -- 数量
    sale_date DATE                   -- 販売日
);

-- データの挿入
INSERT INTO sales (product_name, category, price, quantity, sale_date) VALUES
('ノートパソコン', '電化製品', 80000, 2, '2025-01-05'),
('マウス', '電化製品', 1500, 5, '2025-01-06'),
('キーボード', '電化製品', 3000, 3, '2025-01-07'),
('デスク', '家具', 25000, 1, '2025-01-08'),
('チェア', '家具', 15000, 2, '2025-01-09');

COUNT関数 - 件数を数える

データの件数を数えます。

COUNT関数の例 
-- 全レコードの件数を数える
SELECT COUNT(*) FROM sales;

-- 特定の列（商品名）の件数を数える（NULLは除外される）
SELECT COUNT(product_name) FROM sales;

実行結果： 全レコード数は5件と表示されます。

SUM関数 - 合計を計算する

数値データの合計を計算します。

SUM関数の例 
-- 全商品の価格合計を計算
SELECT SUM(price) AS total_price FROM sales;

-- 全商品の数量合計を計算
SELECT SUM(quantity) AS total_quantity FROM sales;

実行結果： 価格合計は124,500円、数量合計は13個と表示されます。

AVG関数 - 平均を計算する

数値データの平均値を計算します。

AVG関数の例 
-- 商品の平均価格を計算
SELECT AVG(price) AS average_price FROM sales;

-- 平均数量を計算
SELECT AVG(quantity) AS average_quantity FROM sales;

実行結果： 平均価格は24,900円、平均数量は2.6個と表示されます。

MAX関数 / MIN関数 - 最大値・最小値を見つける

データの中から最大値または最小値を見つけます。

MAX関数・MIN関数の例 
-- 最も高い価格を見つける
SELECT MAX(price) AS max_price FROM sales;

-- 最も安い価格を見つける
SELECT MIN(price) AS min_price FROM sales;

実行結果： 最高価格は80,000円、最低価格は1,500円と表示されます。

集計関数のまとめ

  • • COUNT: 件数を数える
  • • SUM: 合計を計算する
  • • AVG: 平均を計算する
  • • MAX: 最大値を見つける
  • • MIN: 最小値を見つける

10.3 GROUP BY句でカテゴリ別に集計

GROUP BY句を使うと、データをグループ（カテゴリ）ごとにまとめて集計できます。

例えば、「カテゴリごとの売上合計」や「商品ごとの販売件数」など、グループ単位で集計したい場合に使います。

カテゴリごとの集計

各カテゴリの売上件数と価格合計を計算します。

GROUP BYの基本 
-- カテゴリごとの件数と価格合計を集計
SELECT
    category,                       -- グループ化する列
    COUNT(*) AS sale_count,         -- 件数
    SUM(price) AS total_price       -- 合計価格
FROM sales
GROUP BY category;                  -- カテゴリでグループ化

実行結果：

category sale_count total_price
電化製品 3 84,500
家具 2 40,000

複数の集計を組み合わせる

カテゴリごとに、件数、合計、平均、最大、最小を同時に計算できます。

複数の集計関数 
-- カテゴリごとの詳細な集計
SELECT
    category,
    COUNT(*) AS count,              -- 件数
    SUM(price) AS total,            -- 合計
    AVG(price) AS average,          -- 平均
    MAX(price) AS max_price,        -- 最高価格
    MIN(price) AS min_price         -- 最低価格
FROM sales
GROUP BY category;

GROUP BYの重要ルール

SELECT句に書く列は、GROUP BY句に指定した列か、集計関数のどちらかでなければなりません。それ以外の列を指定するとエラーになります。

10.4 HAVING句で集計結果を絞り込む

HAVING句は、集計した結果に対して条件を指定するときに使います。

WHERE句との違いは、WHERE句は「集計前のデータ」を絞り込むのに対し、HAVING句は「集計後の結果」を絞り込む点です。

HAVING句の使用例

売上件数が2件以上のカテゴリだけを表示します。

HAVING句の例 
-- 売上件数が2件以上のカテゴリを集計
SELECT
    category,
    COUNT(*) AS sale_count,
    SUM(price) AS total_price
FROM sales
GROUP BY category
HAVING COUNT(*) >= 2;               -- 集計結果を絞り込む

実行結果：

category sale_count total_price
電化製品 3 84,500
家具 2 40,000

※両方とも2件以上なので、どちらも表示されます

WHERE句とHAVING句を組み合わせる

価格が10,000円以上の商品だけを対象に、カテゴリごとの件数を集計し、合計価格が50,000円以上のカテゴリを表示します。

WHERE句とHAVING句の組み合わせ 
-- WHERE句で集計前のデータを絞り込み、HAVING句で集計後の結果を絞り込む
SELECT
    category,
    COUNT(*) AS sale_count,
    SUM(price) AS total_price
FROM sales
WHERE price >= 10000              -- 集計前に絞り込む（10,000円以上の商品のみ）
GROUP BY category
HAVING SUM(price) >= 50000;       -- 集計後に絞り込む（合計50,000円以上のカテゴリのみ）

よくある間違い

集計関数の結果で絞り込みたいときに、WHERE句を使ってしまうことがあります。集計結果の絞り込みには必ずHAVING句を使います。

10.5 実践例：売上分析

集計関数とGROUP BYを使った実務的な売上分析の例を見てみましょう。

実習1: カテゴリ別売上ランキング

各カテゴリの売上を多い順に並べて表示します。

売上ランキング 
-- カテゴリ別売上を多い順に表示
SELECT
    category,
    COUNT(*) AS product_count,      -- 商品数
    SUM(price * quantity) AS total_sales,  -- 売上合計（価格×数量）
    AVG(price) AS avg_price         -- 平均価格
FROM sales
GROUP BY category
ORDER BY total_sales DESC;          -- 売上合計の降順

実習2: 月別売上集計

月ごとの売上件数と売上合計を集計します。

月別売上集計 
-- 月ごとに売上を集計（日付から月を抽出）
SELECT
    TO_CHAR(sale_date, 'YYYY-MM') AS sale_month,  -- 年月を抽出
    COUNT(*) AS sale_count,
    SUM(price * quantity) AS total_sales
FROM sales
GROUP BY TO_CHAR(sale_date, 'YYYY-MM')
ORDER BY sale_month;

TO_CHAR関数は、日付を指定した形式の文字列に変換する関数です。'YYYY-MM'は「年-月」の形式を表します。

実習3: 高額売上カテゴリの抽出

平均価格が20,000円以上のカテゴリを抽出します。

高額カテゴリ抽出 
-- 平均価格が20,000円以上のカテゴリを表示
SELECT
    category,
    COUNT(*) AS product_count,
    AVG(price) AS avg_price,
    MAX(price) AS max_price,
    MIN(price) AS min_price
FROM sales
GROUP BY category
HAVING AVG(price) >= 20000;         -- 平均価格が20,000円以上

理解度確認クイズ

  1. Q1. COUNT関数とSUM関数の違いを説明してください。

    回答を表示

    COUNT関数は「データの件数」を数えます。一方、SUM関数は「数値データの合計」を計算します。例えば、COUNT(*)は全レコード数、SUM(price)は価格の合計を求めます。

  2. Q2. GROUP BY句は何のために使いますか？

    回答を表示

    GROUP BY句は、データをカテゴリ（グループ）ごとにまとめて集計するために使います。例えば、「カテゴリごとの売上合計」や「月ごとの注文件数」など、グループ単位で集計したいときに使用します。

  3. Q3. WHERE句とHAVING句の違いを説明してください。

    回答を表示

    WHERE句は「集計前のデータ」を絞り込むために使います。HAVING句は「集計後の結果」を絞り込むために使います。例えば、WHERE price >= 10000は集計前に10,000円以上の商品だけを抽出し、HAVING SUM(price) >= 50000は集計後に合計が50,000円以上のグループだけを表示します。

  4. Q4. 次のSQL文は何を求めていますか？
    SELECT AVG(price) FROM products;

    回答を表示

    productsテーブルの全商品の平均価格（price列の平均値）を求めています。AVG関数は平均を計算する関数です。
