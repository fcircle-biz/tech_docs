第9章

複数のテーブルを組み合わせよう（JOIN入門）

テーブルの結合の基礎を学び、「顧客テーブル」と「注文テーブル」を例に、INNER JOINの基本的な使い方を理解します。

この章で学ぶこと

  • なぜ複数のテーブルが必要なのかを理解する
  • テーブル間の関連付けの概念を学ぶ
  • INNER JOINの基本的な書き方を習得する
  • 実際のビジネスシーンでのJOINの活用例を知る

9.1 なぜ複数のテーブルが必要なのか？

これまで、1つのテーブルでデータを管理する方法を学んできました。しかし、実際の業務では、情報を複数のテーブルに分けて保存することがほとんどです。なぜでしょうか？

例：オンラインショップの情報管理

オンラインショップで商品を購入する場合を考えてみましょう。必要な情報は以下のようなものです：

  • 顧客情報：名前、住所、電話番号、メールアドレスなど
  • 商品情報：商品名、価格、在庫数など
  • 注文情報：注文日、注文者、購入商品、数量など

もしこれらの情報をすべて1つのテーブルにまとめてしまうと、以下のような問題が起こります：

1つのテーブルにまとめた場合の問題点

  • • 同じ顧客が複数回注文すると、顧客情報が何度も重複する
  • • 顧客の住所が変わったとき、すべての注文記録を更新しなければならない
  • • データの整合性が保ちにくく、間違いが起こりやすい
  • • データ量が無駄に増えてしまう

複数のテーブルに分けるメリット

  • • 情報の重複を避けられる（顧客情報は1回だけ保存）
  • • データの更新が簡単（住所変更は1箇所だけ）
  • • データの整合性が保たれる
  • • ディスク容量を節約できる

9.2 テーブルの関連付けとは

複数のテーブルに分けたデータは、キー（鍵）を使って関連付けます。これにより、別々のテーブルに保存された情報を結びつけることができます。

テーブルの関連イメージ

erDiagram CUSTOMER ||--o{ ORDER : places CUSTOMER { int customer_id PK string name string email string address } ORDER { int order_id PK int customer_id FK date order_date decimal total_amount }

主キー（Primary Key）と外部キー（Foreign Key）

  • 主キー（PK）：各テーブルで、行を一意に識別するための列。例えば、顧客テーブルの customer_id は、各顧客を区別するためのIDです。
  • 外部キー（FK）：他のテーブルの主キーを参照する列。例えば、注文テーブルの customer_id は、顧客テーブルの customer_id を参照しています。

具体的な例を見てみましょう。以下は、顧客テーブルと注文テーブルの簡単なサンプルです。

顧客テーブル（customers）

customer_id（主キー） name email
1 田中太郎 tanaka@example.com
2 佐藤花子 sato@example.com
3 鈴木一郎 suzuki@example.com

注文テーブル（orders）

order_id（主キー） customer_id（外部キー） order_date total_amount
101 1 2025-01-10 5000
102 2 2025-01-12 3000
103 1 2025-01-15 7000

注文テーブルの customer_id は、顧客テーブルの customer_id を参照しています。例えば、注文ID 101の customer_id が 1 なので、この注文は 田中太郎さんのものだとわかります。

9.3 INNER JOINの基本

JOIN（ジョイン）は、複数のテーブルを結合して、1つの結果として取得するためのSQL構文です。その中でも最もよく使われるのがINNER JOIN（インナージョイン）です。

INNER JOINとは？

INNER JOINは、両方のテーブルに共通するデータだけを取得します。つまり、外部キーと主キーが一致する行だけが結果に含まれます。

INNER JOINの基本構文

SQL構文 
SELECT 列名1, 列名2, ...
FROM テーブル1
INNER JOIN テーブル2
ON テーブル1.キー列 = テーブル2.キー列;

実際の例を見てみましょう。顧客テーブルと注文テーブルを結合して、「誰がいつ注文したか」を一覧で取得します。

例：顧客名と注文情報を一緒に取得

SQL例 
-- 顧客名と注文日、注文金額を一緒に表示
SELECT
    customers.name,         -- 顧客名
    orders.order_date,      -- 注文日
    orders.total_amount     -- 注文金額
FROM customers
INNER JOIN orders
ON customers.customer_id = orders.customer_id;

このSQL文は、顧客テーブルと注文テーブルを customer_id で結合し、顧客名、注文日、注文金額を取得します。

実行結果

name order_date total_amount
田中太郎 2025-01-10 5000
佐藤花子 2025-01-12 3000
田中太郎 2025-01-15 7000

田中太郎さんは2回注文しているので、2行表示されています。このように、JOINを使うことで、別々のテーブルの情報を1つの結果として取得できます。

テーブル名の指定について

列名の前に customers.name のようにテーブル名を付けるのは、どのテーブルの列かを明確にするためです。両方のテーブルに同じ名前の列がある場合は、必ずテーブル名を付けましょう。

9.4 JOINの実践例

JOINは、様々な条件と組み合わせることで、より実用的なデータ抽出ができます。いくつかの実践例を見てみましょう。

実習 9-1: 特定の顧客の注文履歴を取得

田中太郎さんの注文履歴だけを取得してみましょう。

SQL例 
-- 田中太郎さんの注文履歴を取得
SELECT
    customers.name,
    orders.order_date,
    orders.total_amount
FROM customers
INNER JOIN orders
ON customers.customer_id = orders.customer_id
WHERE customers.name = '田中太郎';

実習 9-2: 高額注文を新しい順に並べる

5000円以上の注文を、新しい順に表示します。

SQL例 
-- 5000円以上の注文を新しい順に取得
SELECT
    customers.name,
    orders.order_date,
    orders.total_amount
FROM customers
INNER JOIN orders
ON customers.customer_id = orders.customer_id
WHERE orders.total_amount >= 5000
ORDER BY orders.order_date DESC;

実習 9-3: テーブルに別名（エイリアス）を付ける

テーブル名が長い場合、別名（エイリアス）を付けるとSQL文が短くなり読みやすくなります。

SQL例 
-- テーブルに別名（c, o）を付ける
SELECT
    c.name,
    o.order_date,
    o.total_amount
FROM customers AS c
INNER JOIN orders AS o
ON c.customer_id = o.customer_id;

AS c で customers テーブルに「c」という別名を付けています。AS は省略可能です。

ビジネスシーンでの活用例

  • • 売上レポート：顧客ごとの購入金額を集計
  • • 在庫管理：商品と在庫情報を結合して在庫切れ商品を検出
  • • 受注管理：顧客情報と注文情報を結合して納期を確認
  • • 会員管理：会員情報とポイント履歴を結合してポイント残高を表示

9.5 この章のまとめ

第9章で学んだこと

  • データを複数のテーブルに分けることで、情報の重複を避け、整合性を保てる
  • 主キー（PK）は各テーブルで行を一意に識別し、外部キー（FK）は他のテーブルを参照する
  • INNER JOINは、両方のテーブルに共通するデータだけを結合する
  • JOINはWHERE、ORDER BYなどと組み合わせて、より実用的な検索ができる
  • テーブルに別名（エイリアス）を付けることで、SQL文を短く読みやすくできる

理解度確認クイズ

  1. Q1. 複数のテーブルに情報を分けるメリットを2つ挙げてください。

    回答を表示

    【例】
    1. 情報の重複を避けられる（同じ情報を何度も保存しなくて済む）
    2. データの更新が簡単（1箇所を変更すればよい）
    3. データの整合性が保たれる
    4. ディスク容量を節約できる

  2. Q2. 主キー（PK）と外部キー（FK）の違いを説明してください。

    回答を表示

    主キー（PK）：各テーブルで行を一意に識別するための列。例：顧客テーブルの customer_id
    外部キー（FK）：他のテーブルの主キーを参照する列。例：注文テーブルの customer_id（顧客テーブルを参照）

  3. Q3. INNER JOINはどのようなデータを取得しますか？

    回答を表示

    INNER JOINは、両方のテーブルに共通するデータだけを取得します。つまり、外部キーと主キーが一致する行だけが結果に含まれます。

  4. Q4. 次のSQL文の空欄を埋めて、顧客テーブルと注文テーブルを結合してください。
    SELECT c.name, o.order_date FROM customers AS c _____ orders AS o _____ c.customer_id = o.customer_id;

    回答を表示

    SELECT c.name, o.order_date FROM customers AS c INNER JOIN orders AS o ON c.customer_id = o.customer_id;
