第4章

データを入れてみよう（INSERT文）

INSERT文を使って、テーブルに実際にデータを追加する方法を学習します。

この章で学ぶこと

  • INSERT文の基本的な書き方を理解する
  • 1件ずつデータを追加する方法を習得する
  • 複数のデータをまとめて追加する方法を学ぶ
  • INSERT文のよくあるエラーと対処法を知る

4.1 INSERT文とは？

INSERT文は、テーブルに新しいデータ（行）を追加するためのSQL文です。第3章で作成した「連絡先」テーブルに、実際に名前や電話番号などのデータを入れていきます。

例えば、紙のノートに新しい友達の連絡先を書き込むように、データベースにもINSERT文を使って情報を登録します。

INSERT文の役割

flowchart LR A[INSERT文を実行] --> B[テーブルに新しい行を追加] B --> C[データが保存される] style A fill:#dbeafe style B fill:#bfdbfe style C fill:#93c5fd

ポイント

INSERT文は「追加（挿入）」専用のSQL文です。データの追加が成功すると、テーブルに新しい行が1つ（または複数）増えます。

4.2 基本的なINSERT文の書き方

INSERT文の基本的な構文は以下の通りです。

INSERT文の基本構文 
INSERT INTO テーブル名 (列名1, 列名2, 列名3, ...)
VALUES (値1, 値2, 値3, ...);

各部分の意味

  • INSERT INTO テーブル名：どのテーブルにデータを追加するかを指定
  • (列名1, 列名2, ...)：どの列にデータを入れるかを指定
  • VALUES (値1, 値2, ...)：実際に入れるデータを指定

重要ポイント

列名の順序と値の順序は必ず一致させてください。1番目の列名には1番目の値、2番目の列名には2番目の値が対応します。

4.3 1件ずつデータを追加してみよう

前章で作成した「contacts」（連絡先）テーブルに、実際にデータを追加してみましょう。

実習 4-1: 1件のデータを追加する

contactsテーブルに、1人分の連絡先データを追加してみましょう。

手順

  1. 1 pgAdminまたはpsqlを開いて、学習用データベースに接続します
  2. 2 以下のINSERT文を入力して実行します
1件のデータを追加 
-- 田中太郎さんの連絡先を追加
INSERT INTO contacts (name, email, phone)
VALUES ('田中太郎', 'tanaka@example.com', '090-1234-5678');

このINSERT文の解説

  • contacts：データを追加するテーブル名
  • (name, email, phone)：データを入れる列名を指定
  • VALUES (...)：実際のデータ（文字列はシングルクォート ' で囲む）

動作確認

データが正しく追加されたか確認してみましょう。

確認用SELECT文 
-- テーブルの全データを表示
SELECT * FROM contacts;

期待される結果

id name email phone
1 田中太郎 tanaka@example.com 090-1234-5678

ヒント

id列には値を指定していませんが、自動で連番（1, 2, 3...）が振られます。これは第3章でテーブル作成時にSERIAL型を指定したためです。

4.4 複数のデータをまとめて追加してみよう

1つのINSERT文で、複数のデータを一度に追加することもできます。VALUESの後に、カンマ区切りで複数のデータセットを並べます。

複数行を一度に追加 
-- 3人分の連絡先をまとめて追加
INSERT INTO contacts (name, email, phone)
VALUES
    ('佐藤花子', 'sato@example.com', '090-2345-6789'),
    ('鈴木一郎', 'suzuki@example.com', '080-3456-7890'),
    ('高橋美咲', 'takahashi@example.com', '070-4567-8901');

書き方のポイント

  • 各データセットは(...)で囲む
  • データセット同士はカンマ,で区切る
  • 最後のデータセットの後ろはセミコロン;で終わる
  • 見やすくするために改行やインデントを入れても問題ありません

実習 4-2: 複数のデータを追加する

上記のSQL文を実行して、3人分のデータを一度に追加してみましょう。

確認

実行後、SELECT * FROM contacts;で全データを表示して、4件のデータが登録されていることを確認してください。

まとめて追加のメリット

1つのINSERT文で複数のデータを追加すると、1件ずつ実行するよりも処理が速くなります。大量のデータを登録するときに便利です。

4.5 NULLと列の省略

すべての列にデータがあるとは限りません。例えば、メールアドレスがわからない人や、電話番号を登録したくない場合があります。そんなときは、その列をNULL（ヌル）にするか、列自体を省略します。

NULLとは？

NULL（ヌル）は「データが存在しない」「不明」を表す特別な値です。0（ゼロ）や空文字（''）とは異なります。

電話番号がわからない場合 
-- 方法1: 明示的にNULLを指定
INSERT INTO contacts (name, email, phone)
VALUES ('山田次郎', 'yamada@example.com', NULL);

-- 方法2: phone列を省略（自動的にNULLになる）
INSERT INTO contacts (name, email)
VALUES ('伊藤三郎', 'ito@example.com');

ポイント

ただし、テーブル作成時にNOT NULL制約を付けた列は、必ず値を指定する必要があります。省略やNULLを指定するとエラーになります。

4.6 よくあるエラーと対処法

INSERT文を実行する際に、よく遭遇するエラーとその対処法を紹介します。

トラブルシューティング

エラー: 列の数と値の数が一致しない

エラーメッセージ例:

ERROR: INSERT has more expressions than target columns

原因: 列名の数と値の数が合っていません。

解決方法: 列名と値の数を揃えます。列名が3つなら、値も3つ指定してください。

エラー: データ型が一致しない

エラーメッセージ例:

ERROR: invalid input syntax for type integer

原因: 数値型の列に文字列を入れようとしたなど、データ型が合っていません。

解決方法: 列のデータ型に合った値を指定します。文字列はシングルクォートで囲み、数値はそのまま書きます。

エラー: シングルクォートの閉じ忘れ

エラーメッセージ例:

ERROR: unterminated quoted string

原因: 文字列を囲むシングルクォート'を閉じ忘れています。

解決方法: 文字列は必ず'で開始し、'で閉じてください。

エラー: 存在しないテーブルや列名

エラーメッセージ例:

ERROR: relation "contact" does not exist

原因: テーブル名や列名のスペルミスです。

解決方法: テーブル名・列名を正しく入力します。大文字小文字やスペルを確認してください。

つまずきポイント

エラーメッセージが英語で表示されると焦るかもしれませんが、落ち着いて読んでみましょう。エラーメッセージには「どこで」「何が」問題なのかが書かれています。

4.7 この章のまとめ

第4章で学んだこと

  • INSERT文は、テーブルに新しいデータを追加するためのSQL文
  • 基本構文はINSERT INTO テーブル名 (列名...) VALUES (値...);
  • 1つのINSERT文で複数のデータをまとめて追加できる
  • データがない列はNULLにするか、列を省略できる
  • エラーメッセージを読んで、問題箇所を特定する習慣が大切

理解度確認クイズ

  1. Q1. INSERT文の役割は何ですか？

    回答を表示

    INSERT文は、テーブルに新しいデータ（行）を追加するためのSQL文です。

  2. Q2. 以下のINSERT文の誤りを指摘してください。
    INSERT INTO users (id, name) VALUES ('山田太郎');

    回答を表示

    列名が2つ（id, name）なのに、値が1つしかありません。列の数と値の数を一致させる必要があります。

  3. Q3. 1つのINSERT文で3件のデータを追加する場合、どのように書きますか？

    回答を表示

    VALUES句の後に、カンマ区切りで3つのデータセットを並べます。
    例: VALUES (値1, 値2), (値3, 値4), (値5, 値6);

  4. Q4. NULLとは何ですか？

    回答を表示

    NULL（ヌル）は「データが存在しない」「不明」を表す特別な値です。0（ゼロ）や空文字（''）とは異なります。
