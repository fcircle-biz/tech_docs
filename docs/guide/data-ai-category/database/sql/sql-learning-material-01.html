データベースってなに？

プログラム未経験者でも分かるデータベースの基礎を、身近な例を使って丁寧に解説します。

この章で学ぶこと

  • データベースとは何か、なぜ必要なのかを理解する

  • SQLという言語の役割を学ぶ
  • PostgreSQLの特徴を把握する

1.1 データベースとは何か？

データベースとは、大量の情報を整理して保存し、必要なときに素早く取り出せるようにする仕組みのことです。

例えば、図書館を思い浮かべてみてください。図書館には膨大な数の本が並んでいますが、本のジャンル、著者名、出版年などで分類され、整理されています。だから、読みたい本を探すときに、すべての本棚を端から端まで探す必要はありません。データベースも同じように、情報を整理して保存することで、必要な情報をすぐに見つけられるようにしています。

1.2 身近な例で考えるデータベース

データベースは、私たちの日常生活のあらゆる場面で使われています。以下のような場面を思い浮かべてみてください。

電話帳

名前、電話番号、住所などの情報を整理して保存。名前や番号から相手を探せます。

オンラインショッピング

商品情報、在庫数、注文履歴などを保存。検索やフィルターで商品を探せます。

銀行口座

預金額、取引履歴、顧客情報などを管理。残高照会や振込ができます。

SNS

ユーザー情報、投稿、友達リストなどを保存。タイムラインを表示したり、検索できます。

ポイント

これらの例に共通するのは、「たくさんの情報を整理して保存」し、「必要なときに素早く取り出せる」という点です。これがデータベースの基本的な役割です。

1.3 ExcelとデータベースのL違い

「Excelでも表を作ってデータを管理できるのに、なぜデータベースが必要なの？」と思うかもしれません。確かに、Excelも便利なツールですが、データベースには以下のような違いがあります。

まとめ

Excelは手軽で便利ですが、データ量が増えたり、複数人で使ったり、複雑な処理が必要になったりすると、データベースが威力を発揮します。用途に応じて使い分けることが大切です。

1.4 SQLとは何か？

SQL（エスキューエル）とは、Structured Query Languageの略で、データベースに命令を出すための言語です。

例えば、図書館で本を探すときに司書さんに「〇〇という本はどこにありますか？」と尋ねるように、データベースにも「こういうデータがほしい」「このデータを追加して」といった命令を出す必要があります。その命令を書くための言語がSQLです。

SQLの例

例えば、「顧客テーブルから名前が『田中』さんの情報を探す」という命令は、SQLで次のように書きます。

SQL例 
SELECT * FROM 顧客テーブル WHERE 名前 = '田中';

これだけで、データベースから条件に合う情報が取り出せます。次の章以降で、このようなSQL文を実際に書いて学んでいきます。

1.5 PostgreSQLについて

データベースには様々な種類がありますが、このガイドではPostgreSQL（ポストグレスキューエル）というデータベースを使用します。

PostgreSQLは、無料で使えて、高機能で、信頼性が高いデータベースとして世界中で利用されています。

無料で使える

オープンソースなので、商用利用も含めて完全に無料で使えます。

高い信頼性

データの整合性を守る仕組みが充実しており、企業でも広く使われています。

学習に最適

標準的なSQLを学べるので、他のデータベースでも応用が効きます。

Windows/Mac/Linux対応

どのOSでも動作するので、環境を選びません。

他のデータベースとの関係

データベースには、PostgreSQLの他にもMySQL、Oracle、SQL Serverなど様々な種類があります。それぞれ特徴がありますが、基本的なSQLの書き方はほぼ共通しています。

PostgreSQLで学んだSQLの知識は、他のデータベースでもそのまま活かせるので、安心して学習を進めてください。
