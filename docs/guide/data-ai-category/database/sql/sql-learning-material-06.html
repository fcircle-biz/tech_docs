第6章

データを変更・削除してみよう（UPDATE・DELETE）

UPDATE文でデータを修正する方法、DELETE文でデータを削除する方法を学習します。間違えて全データを消してしまわないための注意点も含め、安全な操作方法を身につけます。

この章で学ぶこと

  • UPDATE文を使ったデータの修正方法
  • DELETE文を使ったデータの削除方法
  • WHERE句を使った安全な更新・削除の実践
  • 間違えて全データを変更・削除してしまわないための注意点
  • トランザクションの基礎（COMMIT・ROLLBACK）

6.1 UPDATEとDELETEの重要性

これまでの章では、データを追加（INSERT）する方法と、データを取り出す（SELECT）方法を学びました。しかし、実際の業務では、すでに登録されているデータを修正したり、不要なデータを削除したりする必要がよくあります。

例えば、以下のような場面を考えてみましょう。

データの修正（UPDATE）

顧客が引っ越して住所が変わった、電話番号が変わった、商品の価格を変更したい、など

データの削除（DELETE）

退会した会員のデータを削除、期限切れのキャンペーン情報を削除、重複データを削除、など

重要な注意事項

UPDATE文とDELETE文は、間違った使い方をすると大量のデータを誤って変更・削除してしまう可能性があります。この章では、安全に操作するための方法をしっかり学習しましょう。

6.2 UPDATE文の基礎

UPDATE文は、テーブルに保存されているデータを修正（更新）するためのSQL文です。

UPDATE文の基本構文

SQL構文 
-- 基本構文
UPDATE テーブル名
SET 列名1 = 新しい値1,
    列名2 = 新しい値2
WHERE 条件;

重要： WHERE句を使って、どのデータを更新するかを明確に指定します。WHERE句がない場合、テーブル内の全データが更新されてしまいます。

各部分の説明

flowchart TD A[UPDATE文] --> B[UPDATE テーブル名
どのテーブルを更新するか] A --> C[SET 列名 = 値
どの列をどんな値に変更するか] A --> D[WHERE 条件
どのデータを更新するか指定] style A fill:#dbeafe style D fill:#fee2e2

ポイント

UPDATE文は、既存のデータを変更するだけで、新しいデータを追加することはありません。新しいデータを追加する場合は、INSERT文を使います。

6.3 UPDATE文の実践

実際の例を使って、UPDATE文の使い方を学習しましょう。以下のような連絡先テーブルがあるとします。

id 名前 電話番号 メール
1 田中太郎 090-1234-5678 tanaka@example.com
2 佐藤花子 080-9876-5432 sato@example.com
3 鈴木一郎 070-1111-2222 suzuki@example.com

実習：電話番号を更新してみよう

田中太郎さん（id = 1）の電話番号が変わったので、新しい番号「090-9999-8888」に更新します。

SQL 
-- 田中太郎さんの電話番号を更新
UPDATE 連絡先
SET 電話番号 = '090-9999-8888'
WHERE id = 1;

解説： WHERE句で「id = 1」と指定することで、田中太郎さん（id = 1）のデータだけが更新されます。

複数の列を同時に更新する

カンマ（,）で区切ることで、複数の列を同時に更新できます。

SQL 
-- 佐藤花子さんの電話番号とメールアドレスを同時に更新
UPDATE 連絡先
SET 電話番号 = '080-5555-6666',
    メール = 'sato.hanako@newmail.com'
WHERE id = 2;

複数の行を同時に更新する

WHERE句の条件に複数のデータが該当する場合、それらすべてが更新されます。

SQL 
-- 名前に「田中」を含む人全員のメールアドレスを更新
UPDATE 連絡先
SET メール = '更新中@example.com'
WHERE 名前 LIKE '%田中%';

注意： この例では、「田中」を含む人が複数いれば、全員のメールアドレスが更新されます。

6.4 DELETE文の基礎

DELETE文は、テーブルからデータを削除するためのSQL文です。削除されたデータは基本的に復元できないため、特に注意が必要です。

DELETE文の基本構文

SQL構文 
-- 基本構文
DELETE FROM テーブル名
WHERE 条件;

最重要： WHERE句を使って、削除するデータを明確に指定します。WHERE句がない場合、テーブル内の全データが削除されてしまいます。

各部分の説明

flowchart TD A[DELETE文] --> B[DELETE FROM テーブル名
どのテーブルから削除するか] A --> C[WHERE 条件
どのデータを削除するか指定] style A fill:#dbeafe style C fill:#fee2e2

警告

DELETE文は元に戻せません。削除する前に、必ずSELECT文で対象データを確認してから実行しましょう。

6.5 DELETE文の実践

実際の例を使って、DELETE文の安全な使い方を学習しましょう。

実習：安全にデータを削除する手順

鈴木一郎さん（id = 3）のデータを削除します。安全に削除するため、以下の手順で進めます。

ステップ1: まずSELECT文で確認

SQL 
-- 削除対象のデータを確認
SELECT * FROM 連絡先 WHERE id = 3;

削除したいデータが表示されることを確認します。

ステップ2: DELETE文を実行

SQL 
-- データを削除
DELETE FROM 連絡先 WHERE id = 3;

ステップ3: 削除結果を確認

SQL 
-- 削除されたことを確認
SELECT * FROM 連絡先;

id = 3のデータが削除され、残り2件だけが表示されることを確認します。

複数の行を削除する

WHERE句の条件に複数のデータが該当する場合、それらすべてが削除されます。

SQL 
-- 名前に「田中」を含む人全員を削除
-- まず確認
SELECT * FROM 連絡先 WHERE 名前 LIKE '%田中%';

-- 削除実行
DELETE FROM 連絡先 WHERE 名前 LIKE '%田中%';

6.6 安全な更新・削除のベストプラクティス

データの更新や削除を安全に行うために、以下のベストプラクティスを守りましょう。

1

必ずWHERE句を付ける

WHERE句がないと全データが対象になります。必ず条件を指定しましょう。

2

実行前にSELECTで確認

UPDATE/DELETE実行前に、同じWHERE条件でSELECT文を実行し、対象データを確認しましょう。

SQL 
-- まず確認
SELECT * FROM 連絡先 WHERE id = 1;

-- 確認後に実行
UPDATE 連絡先 SET 電話番号 = '090-9999-8888' WHERE id = 1;
3

影響行数を確認する

UPDATE/DELETE実行後、何行が影響を受けたか確認しましょう。想定外の数なら問題がある可能性があります。

4

バックアップを取る

重要なデータを削除・更新する前には、データベース全体またはテーブルのバックアップを取りましょう。

5

トランザクションを利用する

BEGIN、COMMIT、ROLLBACKを使って、間違えた場合に元に戻せるようにしましょう（次のセクションで解説）。

6.7 トランザクションの基礎（COMMIT・ROLLBACK）

トランザクションとは、複数のSQL文をひとまとまりの処理として扱う仕組みです。UPDATE文やDELETE文を実行する前に、トランザクションを開始することで、間違えた場合に元に戻すことができます。

トランザクションの流れ

flowchart LR A[BEGIN
トランザクション開始] --> B[UPDATE/DELETE
データ操作] B --> C{結果は正しい?} C -->|正しい| D[COMMIT
確定] C -->|間違えた| E[ROLLBACK
取り消し] style A fill:#dbeafe style D fill:#dcfce7 style E fill:#fee2e2

トランザクションの基本的な使い方

SQL 
-- トランザクション開始
BEGIN;

-- データを更新
UPDATE 連絡先 SET 電話番号 = '090-9999-8888' WHERE id = 1;

-- 結果を確認
SELECT * FROM 連絡先 WHERE id = 1;

-- 問題なければ確定
COMMIT;

-- または、間違えた場合は取り消し
-- ROLLBACK;

実習：トランザクションで安全に削除

トランザクションを使って、安全にデータを削除してみましょう。

SQL 
-- トランザクション開始
BEGIN;

-- データを削除
DELETE FROM 連絡先 WHERE id = 2;

-- 結果確認
SELECT * FROM 連絡先;

-- もし間違えていたら
ROLLBACK;  -- これを実行すると削除が取り消される

-- 問題なければ
-- COMMIT;  -- これを実行すると削除が確定される

重要： ROLLBACKを実行すれば、BEGINの時点の状態に戻ります。COMMITを実行すると、変更が確定され、元に戻せなくなります。

まとめ

トランザクションを使えば、「試しに実行してみて、結果を見てから確定するか取り消すか決める」ことができます。初心者の方は、最初のうちはすべてのUPDATE/DELETE操作をトランザクション内で行うことをお勧めします。

6.8 よくあるミスと対処法

トラブルシューティング

ミス1: WHERE句を忘れて全データを更新・削除してしまった

対処法： トランザクション内で実行していた場合、すぐにROLLBACKを実行しましょう。トランザクションを使っていなかった場合、バックアップから復元する必要があります。

ミス2: 間違った条件で削除してしまった

対処法： 削除前にSELECT文で確認する習慣をつけましょう。また、トランザクションを使えば、間違いに気づいた時点でROLLBACKできます。

ミス3: 更新後の値が想定と違う

対処法： UPDATE実行後、すぐにSELECTで結果を確認しましょう。トランザクション内なら、おかしければROLLBACKできます。

ミス4: 影響を受けた行数が想定と違う

対処法： WHERE条件が正しいか確認しましょう。特に「=」と「LIKE」、「AND」と「OR」の使い分けに注意が必要です。

6.9 この章のまとめ

第6章で学んだこと

  • UPDATE文は、既存のデータを修正するために使う
  • DELETE文は、データを削除するために使う
  • WHERE句は必須。ないと全データが対象になってしまう
  • 実行前にSELECT文で確認する習慣が重要
  • トランザクション（BEGIN/COMMIT/ROLLBACK）を使えば、間違えた場合に元に戻せる
  • 重要なデータ操作の前にはバックアップを取る

理解度確認クイズ

  1. Q1. UPDATE文でWHERE句を書かないとどうなりますか？

    回答を表示

    WHERE句がない場合、テーブル内の全データが更新されてしまいます。必ずWHERE句で条件を指定して、更新対象を限定しましょう。

  2. Q2. DELETE文を実行する前に、必ずやるべきことは何ですか？

    回答を表示

    DELETE文を実行する前に、同じWHERE条件でSELECT文を実行し、削除対象のデータを確認することが重要です。また、トランザクション（BEGIN）を開始してから実行すれば、間違えた場合にROLLBACKで元に戻せます。

  3. Q3. 複数の列を同時に更新するには、どうすればよいですか？

    回答を表示

    SET句で、列名と値のペアをカンマ（,）で区切って複数指定します。
    例：UPDATE テーブル SET 列1 = 値1, 列2 = 値2 WHERE 条件;

  4. Q4. トランザクションのCOMMITとROLLBACKの違いは何ですか？

    回答を表示

    COMMITは変更を確定（保存）するコマンドで、実行後は元に戻せません。ROLLBACKは変更を取り消すコマンドで、BEGIN実行時の状態に戻ります。

  5. Q5. 以下のSQL文は何をしますか？
    UPDATE 商品 SET 価格 = 価格 * 1.1 WHERE カテゴリ = '食品';

    回答を表示

    商品テーブルの中で、カテゴリが「食品」のデータすべてについて、価格を現在の1.1倍（10%値上げ）に更新します。このように、現在の値を使って新しい値を計算することもできます。
