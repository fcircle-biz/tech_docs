PostgreSQLをインストールしてみよう

PostgreSQLのダウンロードからインストールまで、画面キャプチャを使いながら一つ一つ丁寧に説明します。

この章で学ぶこと

  • PostgreSQLのダウンロード方法を知る
  • 各OS（Windows/Mac/Linux）でのインストール手順を理解する
  • pgAdminの基本的な使い方を学ぶ
  • データベースに初めて接続する方法を体験する

2.1 インストールの準備

PostgreSQLをインストールする前に、必要な準備を確認しましょう。この章では、実際にパソコンにPostgreSQLをインストールして、データベースを使える環境を整えます。

インストール前のチェックリスト

インストールにかかる時間

ダウンロードからインストール完了まで、約15〜30分かかります（インターネット速度による）。時間に余裕があるときに作業を始めましょう。

2.2 PostgreSQLのダウンロード

まず、PostgreSQLの公式サイトからインストーラーをダウンロードします。公式サイトは日本語版も用意されているので、安心してください。

実習 2-1: PostgreSQLのダウンロード

公式サイトから最新版のPostgreSQLをダウンロードします。

手順

  1. 1 Webブラウザで PostgreSQL公式ダウンロードページ にアクセス
  2. 2 お使いのOS（Windows/Mac/Linux）を選択
  3. 3 「Download the installer」をクリック（EnterpriseDB提供版を推奨）
  4. 4 最新の安定版（バージョン16.x以降）をダウンロード

ヒント

PostgreSQLには様々なバージョンがありますが、初心者の方は最新の安定版（Latest Stable）を選んでおけば間違いありません。バージョン番号が大きいものほど新しいバージョンです。

2.3 Windowsへのインストール手順

Windowsユーザーの方は、以下の手順でPostgreSQLをインストールします。画面の指示に従って進めていけば大丈夫です。

実習 2-2: Windows版インストール

ダウンロードしたインストーラーを実行します。

手順

  1. 1 ダウンロードした postgresql-XX-windows-x64.exe をダブルクリック
  2. 2 セットアップウィザードが起動したら「Next」をクリック
  3. 3 インストール先フォルダはデフォルトのまま「Next」（例: C:\Program Files\PostgreSQL\16）
  4. 4 インストールするコンポーネントを選択（全てチェックのまま推奨）
  5. 5 データディレクトリもデフォルトのまま「Next」
  6. 6 重要: スーパーユーザー（postgres）のパスワードを設定 → 必ずメモしておく！
  7. 7 ポート番号はデフォルトの 5432 のまま「Next」
  8. 8 ロケール（言語設定）は「Japanese, Japan」または「Default locale」を選択
  9. 9 設定内容を確認して「Next」→ インストール開始
  10. 10 インストール完了後、Stack Builderは「スキップ」して「Finish」

重要：パスワードを忘れないで！

手順6で設定したパスワードは、データベースに接続するたびに必要になります。忘れると再インストールが必要になる場合もあるので、必ず安全な場所にメモしておきましょう。

2.4 Mac/Linuxへのインストール手順

MacやLinuxユーザーの方は、パッケージマネージャーやGUIインストーラーを使ってインストールできます。

Macの場合（Homebrewを使う方法）

Homebrewがインストールされている場合、ターミナルで以下のコマンドを実行します。

ターミナル 
# Homebrewを使ってPostgreSQLをインストール
brew install postgresql@16

# PostgreSQLを起動
brew services start postgresql@16

# バージョン確認
psql --version

Linuxの場合（Ubuntu/Debian）

apt-getを使ってインストールします。

ターミナル 
# パッケージリストを更新
sudo apt update

# PostgreSQLをインストール
sudo apt install postgresql postgresql-contrib

# PostgreSQLサービスの状態を確認
sudo systemctl status postgresql

# バージョン確認
psql --version

GUIインストーラーも利用可能

コマンドライン操作に不安がある方は、Windows版と同じGUIインストーラー（.dmgや.runファイル）も公式サイトからダウンロードできます。操作方法はWindows版とほぼ同じです。

2.5 pgAdminの起動と基本操作

pgAdminは、PostgreSQLをグラフィカルに操作できるツールです。インストール時に自動的に含まれているので、新たにインストールする必要はありません。

pgAdminでできること

実習 2-3: pgAdminを起動する

pgAdminを初めて起動してみましょう。

手順

  1. 1 Windows: スタートメニューから「pgAdmin 4」を検索して起動
  2. 2 Mac: アプリケーションフォルダから「pgAdmin 4」を起動
  3. 3 Webブラウザが自動的に開き、pgAdminの画面が表示されます
  4. 4 初回起動時は、マスターパスワードの設定を求められる場合があります（任意のパスワードを設定）

pgAdminはWebアプリ

pgAdminはWebブラウザ上で動作するアプリケーションです。起動すると自動的にブラウザが開きますが、これは正常な動作です。

2.6 初めてのデータベース接続

pgAdminを起動したら、実際にデータベースに接続してみましょう。インストール時に作成されたデフォルトのサーバーに接続します。

実習 2-4: データベースに接続する

デフォルトのサーバーに接続して、データベース一覧を確認します。

手順

  1. 1 左側のサイドバーに「Servers」というグループが表示されています
  2. 2 「Servers」の左の三角アイコンをクリックして展開
  3. 3 「PostgreSQL 16」（バージョンは異なる場合あり）をダブルクリック
  4. 4 パスワード入力ダイアログが表示されたら、インストール時に設定したパスワードを入力
  5. 5 接続が成功すると、「Databases」フォルダが表示されます
  6. 6 「Databases」を展開すると、デフォルトで「postgres」というデータベースがあることを確認

動作確認

接続が正しくできているか、簡単なSQLクエリを実行して確認しましょう。

手順

  1. 1 「postgres」データベースを右クリック → 「Query Tool」を選択
  2. 2 クエリエディタが開いたら、以下のSQLを入力
SQL 
-- PostgreSQLのバージョンを確認
SELECT version();
  1. 3 実行ボタン（▶アイコン）をクリック、またはF5キーを押す

期待される結果 

PostgreSQL 16.x on x86_64-pc-linux-gnu, compiled by gcc...
（バージョン情報が表示されればOK）

2.7 よくあるトラブルと解決方法

トラブルシューティング

インストーラーが起動しない

原因: 管理者権限がない、ウイルス対策ソフトがブロックしている

解決方法: インストーラーを右クリック → 「管理者として実行」を選択。ウイルス対策ソフトの除外設定も確認してください。

パスワードを忘れてしまった

原因: インストール時に設定したパスワードを記録していなかった

解決方法: 残念ながら、パスワードのリセットは初心者には難しい作業です。PostgreSQLをアンインストールして、再度インストールすることをお勧めします。今度は必ずパスワードをメモしてください。

pgAdminが起動しない

原因: ポート番号の競合、ブラウザの問題

解決方法: 別のブラウザでアクセスしてみる（Chrome、Firefox、Edgeなど）。それでも解決しない場合は、パソコンを再起動してから再度試してください。

「サーバーに接続できません」エラー

原因: PostgreSQLサービスが起動していない、ファイアウォールがブロックしている

解決方法:

  • Windows: サービス管理を開き、「postgresql-x64-16」サービスが「実行中」になっているか確認
  • Mac: ターミナルで brew services list を実行し、PostgreSQLが起動しているか確認
  • ファイアウォール設定で、ポート5432の通信を許可

理解度確認クイズ

  1. Q1. PostgreSQLをインストールする際に、必ずメモしておく必要がある情報は何ですか？

    回答を表示

    スーパーユーザー（postgres）のパスワードです。このパスワードを忘れると、データベースに接続できなくなり、再インストールが必要になる場合があります。

  2. Q2. pgAdminとは何ですか？

    回答を表示

    pgAdminは、PostgreSQLをグラフィカルに操作できる管理ツールです。SQLクエリの実行、データベースの作成、テーブルの編集などをWebブラウザ上で行うことができます。

  3. Q3. PostgreSQLがデフォルトで使用するポート番号は何番ですか？

    回答を表示

    5432番です。インストール時に特に変更しなければ、このポート番号が使用されます。

  4. Q4. インストール後、PostgreSQLのバージョンを確認するSQLクエリは何ですか？

    回答を表示

    SELECT version(); です。このクエリを実行すると、インストールされているPostgreSQLのバージョン情報が表示されます。
