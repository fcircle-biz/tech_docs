12.1 今回作るシステムの概要

この章では、図書館の図書管理システムを例に、実践的なデータベースを設計・構築していきます。

図書管理システムでは、「どの本があるか」「誰が借りているか」「いつ返却されるか」といった情報を管理します。これらの情報を効率的に扱うために、複数のテーブルを関連付けて使用します。

実際のシステムでは、このように「複数の種類の情報」を「複数のテーブル」に分けて管理し、それらを関連付けて使います。第9章で学んだJOINが重要な役割を果たします。

12.2 テーブル設計

まず、必要なテーブルを設計します。今回は以下の3つのテーブルを作成します。

テーブル構成とリレーションシップ

テーブル1: 図書テーブル（books）

図書館にある本の情報を保存します。各本には一意のIDを付けて管理します。

図書テーブル作成 
-- 図書テーブルの作成
CREATE TABLE books (
    book_id SERIAL PRIMARY KEY,           -- 図書ID（自動採番）
    title VARCHAR(200) NOT NULL,          -- タイトル
    author VARCHAR(100) NOT NULL,         -- 著者
    published_year INT,                   -- 出版年
    isbn VARCHAR(20) UNIQUE               -- ISBN（重複不可）
);

テーブル2: 会員テーブル（members）

図書館の会員情報を保存します。本を借りることができる人の情報です。

会員テーブル作成 
-- 会員テーブルの作成
CREATE TABLE members (
    member_id SERIAL PRIMARY KEY,         -- 会員ID（自動採番）
    name VARCHAR(100) NOT NULL,           -- 氏名
    email VARCHAR(150) UNIQUE NOT NULL,   -- メールアドレス（重複不可）
    joined_date DATE DEFAULT CURRENT_DATE -- 入会日（デフォルト：今日）
);

テーブル3: 貸出テーブル（loans）

「誰が」「どの本を」「いつ借りて」「いつ返すか」という貸出情報を保存します。

貸出テーブル作成 
-- 貸出テーブルの作成
CREATE TABLE loans (
    loan_id SERIAL PRIMARY KEY,           -- 貸出ID（自動採番）
    book_id INT NOT NULL,                 -- 図書ID
    member_id INT NOT NULL,               -- 会員ID
    loan_date DATE DEFAULT CURRENT_DATE,  -- 貸出日（デフォルト：今日）
    due_date DATE NOT NULL,               -- 返却予定日
    return_date DATE,                     -- 返却日（未返却ならNULL）
    status VARCHAR(20) DEFAULT '貸出中',  -- 状態
    FOREIGN KEY (book_id) REFERENCES books(book_id),
    FOREIGN KEY (member_id) REFERENCES members(member_id)
);

重要ポイント：外部キー制約

FOREIGN KEY（外部キー）は、他のテーブルとの関連を定義します。これにより、「存在しない図書ID」や「存在しない会員ID」が登録されるのを防ぎます。

12.4 実践的な検索クエリ

実際の業務で必要になる、様々な検索を実行してみましょう。

例1: 現在貸出中の本を一覧表示

「今どの本が借りられているか」を知りたいとき

貸出中の本を検索 
-- 現在貸出中の本を検索（JOIN使用）
SELECT
    b.title AS 書籍名,
    b.author AS 著者,
    m.name AS 借りている人,
    l.loan_date AS 貸出日,
    l.due_date AS 返却予定日
FROM loans l
INNER JOIN books b ON l.book_id = b.book_id
INNER JOIN members m ON l.member_id = m.member_id
WHERE l.status = '貸出中'
ORDER BY l.due_date;

例2: 返却期限が過ぎている本を検索

延滞している本を探す（督促が必要）

延滞図書を検索 
-- 返却期限を過ぎている貸出中の本
SELECT
    b.title AS 書籍名,
    m.name AS 借りている人,
    m.email AS メールアドレス,
    l.due_date AS 返却予定日,
    CURRENT_DATE - l.due_date AS 延滞日数
FROM loans l
INNER JOIN books b ON l.book_id = b.book_id
INNER JOIN members m ON l.member_id = m.member_id
WHERE l.status = '貸出中'
  AND l.due_date < CURRENT_DATE
ORDER BY l.due_date;

ポイント

CURRENT_DATE - l.due_dateで延滞日数を計算しています。このように、日付型のカラムは四則演算ができます。

例3: 特定の会員の貸出履歴を表示

「田中太郎」さんがこれまでに借りた本の履歴

貸出履歴検索 
-- 特定会員の貸出履歴
SELECT
    b.title AS 書籍名,
    l.loan_date AS 貸出日,
    l.return_date AS 返却日,
    l.status AS 状態
FROM loans l
INNER JOIN books b ON l.book_id = b.book_id
INNER JOIN members m ON l.member_id = m.member_id
WHERE m.name = '田中太郎'
ORDER BY l.loan_date DESC;

例4: 人気の本ランキング（貸出回数順）

どの本がよく借りられているかを集計

人気ランキング 
-- 貸出回数が多い本ランキング（TOP 3）
SELECT
    b.title AS 書籍名,
    b.author AS 著者,
    COUNT(*) AS 貸出回数
FROM loans l
INNER JOIN books b ON l.book_id = b.book_id
GROUP BY b.book_id, b.title, b.author
ORDER BY 貸出回数 DESC
LIMIT 3;

12.5 サブクエリを使った高度な検索

第11章で学んだサブクエリを使って、より複雑な検索にも挑戦してみましょう。

例: まだ一度も借りられていない本を探す

未貸出図書の検索 
-- 一度も借りられていない本を検索（サブクエリ使用）
SELECT
    title AS 書籍名,
    author AS 著者,
    published_year AS 出版年
FROM books
WHERE book_id NOT IN (
    SELECT DISTINCT book_id
    FROM loans
);

サブクエリの使い方

NOT IN (SELECT ...)を使うことで、「貸出テーブルに存在しない本」を探すことができます。サブクエリで全貸出履歴から図書IDを取り出し、それに含まれない本を抽出しています。

12.6 集計と分析

第10章で学んだGROUP BYを使って、様々な角度からデータを集計してみましょう。

例1: 会員ごとの貸出冊数

会員別貸出冊数 
-- 会員ごとの貸出冊数を集計
SELECT
    m.name AS 会員名,
    COUNT(*) AS 貸出冊数,
    SUM(CASE WHEN l.status = '貸出中' THEN 1 ELSE 0 END) AS 現在貸出中
FROM members m
LEFT JOIN loans l ON m.member_id = l.member_id
GROUP BY m.member_id, m.name
ORDER BY 貸出冊数 DESC;

例2: 月別の貸出件数

月別貸出件数 
-- 月別の貸出件数を集計
SELECT
    TO_CHAR(loan_date, 'YYYY-MM') AS 年月,
    COUNT(*) AS 貸出件数
FROM loans
GROUP BY TO_CHAR(loan_date, 'YYYY-MM')
ORDER BY 年月;

日付関数の活用

TO_CHAR(loan_date, 'YYYY-MM')で日付を「年-月」の形式に変換しています。これにより月ごとにグループ化できます。

12.7 データの更新操作

実際の運用では、データの更新も頻繁に行います。

例: 本を返却する処理

貸出中の本が返却されたときの処理

返却処理 
-- 貸出ID=2の本を返却（返却日を今日に、状態を返却済みに更新）
UPDATE loans
SET return_date = CURRENT_DATE,
    status = '返却済み'
WHERE loan_id = 2;

例: 新規貸出の登録

新しく本が借りられたときの処理（返却予定日は14日後に設定）

新規貸出登録 
-- 会員ID=3が図書ID=2を借りる

実務での注意点

理解度確認クイズ

  1. Q1. 外部キー制約（FOREIGN KEY）の役割は何ですか？

    回答を表示

    外部キー制約は、他のテーブルとの関連を定義し、データの整合性を保つ役割があります。例えば、loansテーブルのbook_idには、booksテーブルに実際に存在するbook_idしか登録できなくなります。これにより、「存在しない本」が貸出データに登録されるのを防ぎます。

  2. Q2. 3つのテーブルをJOINするとき、どのように記述しますか？

    回答を表示

    FROM句で最初のテーブルを指定し、INNER JOINを2回使います。
    例: FROM loans l INNER JOIN books b ON l.book_id = b.book_id INNER JOIN members m ON l.member_id = m.member_id
    このように、JOINを連続して記述することで、3つ以上のテーブルを結合できます。

  3. Q3. 「現在貸出中の本」を検索する際、WHERE句にはどんな条件を書きますか？

    回答を表示

    WHERE status = '貸出中'のように、statusカラムが「貸出中」のレコードだけを抽出する条件を書きます。また、返却日がNULLであることを条件にすることもできます（WHERE return_date IS NULL）。

  4. Q4. GROUP BYを使って会員ごとの貸出冊数を集計する場合、SELECT句には何を書きますか？

    回答を表示

    SELECT句には、GROUP BY句で指定したカラム（会員名など）と集計関数（COUNT(*)など）を書きます。
    例: SELECT m.name, COUNT(*) AS 貸出冊数 ... GROUP BY m.member_id, m.name
