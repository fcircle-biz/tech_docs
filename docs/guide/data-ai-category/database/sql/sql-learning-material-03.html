はじめてのテーブル作成 - データの入れ物を作ってみよう

テーブルの概念を理解し、実際に簡単な「連絡先」テーブルを作成します。

この章で学ぶこと

  • テーブルとは何か、なぜ必要なのかを理解する
  • 列（カラム）と行（レコード）の概念を学ぶ
  • 基本的なデータ型（文字、数値、日付）を知る
  • CREATE TABLE文で実際にテーブルを作成する

3.1 テーブルとは何か？

データベースの中で、データを整理して保存するための「表」のような入れ物をテーブルと呼びます。

Excelの表（スプレッドシート）を思い浮かべてください。横の行と縦の列があって、それぞれのマス目にデータが入っています。データベースのテーブルも同じように、行と列で構成されています。

テーブルのイメージ（連絡先テーブルの例）

ID（列1） 名前（列2） 電話番号（列3） メールアドレス（列4）
1 田中 太郎 090-1234-5678 tanaka@example.com
2 佐藤 花子 080-9876-5432 sato@example.com
3 鈴木 一郎 070-5555-6666 suzuki@example.com

この表全体が「連絡先テーブル」です。

ポイント

テーブルは、Excelの表と同じように考えるとイメージしやすいです。ただし、データベースのテーブルには「データ型」や「制約」といった、Excelにはない便利な仕組みがあります。

3.2 列（カラム）と行（レコード）の概念

テーブルを理解するには、「列（カラム）」と「行（レコード）」という2つの概念を理解することが大切です。

列（カラム）と行（レコード）

flowchart TD A[テーブル] --> B[列カラム] A --> C[行レコード] B --> B1[項目の種類を定義] B --> B2[例: ID、名前、電話番号] B --> B3[データ型を持つ] C --> C1[1件分のデータ] C --> C2[例: 1, 田中 太郎, 090-1234-5678] C --> C3[複数の列の値をまとめたもの] style A fill:#dbeafe style B fill:#fef3c7 style C fill:#d1fae5

列（カラム）

縦方向のデータの区分けです。「どんな情報を保存するか」を表します。

  • • 例: ID列、名前列、電話番号列
  • • テーブル作成時に定義
  • • データ型を指定する

行（レコード）

横方向のデータのまとまりです。「1件分の情報」を表します。

  • • 例: 田中さんの情報（1行）
  • • データを追加するたびに増える
  • • 各列に対応する値を持つ

覚え方のコツ

列（カラム）は「設計図」、行（レコード）は「実際のデータ」と考えると分かりやすいです。テーブルを作る時には列を定義し、データを入れる時には行を追加します。

3.3 データ型とは？

データ型とは、その列にどんな種類のデータを保存するかを指定するものです。

例えば、「名前」は文字、「年齢」は数値、「生年月日」は日付というように、データにはさまざまな種類があります。データベースでは、それぞれの列にどんなデータを入れるかを決めておく必要があります。

データ型 説明 具体例
INTEGER 整数（小数点なしの数値） 1, 100, -5
VARCHAR(n) 文字列（最大n文字） 田中 太郎、東京都
TEXT 長い文字列（文字数制限なし） コメント、説明文など
DATE 日付 2025-12-06
TIMESTAMP 日付と時刻 2025-12-06 14:30:00
BOOLEAN 真偽値（trueまたはfalse） TRUE, FALSE
DECIMAL(p,s) 小数点を含む数値
p: 全体の桁数、s: 小数点以下の桁数		 1234.56, 99.99

データ型を決める理由

  • • データの整合性: 数値の列に文字を入れられないようにする
  • • 効率的な保存: データに合ったサイズで保存できる
  • • 正確な計算: 数値型なら合計や平均を計算できる
  • • 検索の最適化: データ型に応じた高速な検索が可能

3.4 テーブルを作成してみよう

それでは、実際に「連絡先」テーブルを作成してみましょう。テーブルを作成するには、CREATE TABLE文を使います。

CREATE TABLE文の基本構文

構文 
CREATE TABLE テーブル名 (
    列名1 データ型,
    列名2 データ型,
    列名3 データ型
);

列の定義はカンマ（,）で区切り、最後の列の後にはカンマは付けません。

実習 3-1: 連絡先テーブルを作成

ID、名前、電話番号、メールアドレスを保存できる「連絡先」テーブルを作成します。

SQL 
-- 連絡先テーブルを作成
CREATE TABLE contacts (
    id INTEGER,              -- ID（整数）
    name VARCHAR(100),       -- 名前（最大100文字）
    phone VARCHAR(20),       -- 電話番号（最大20文字）
    email VARCHAR(100)       -- メールアドレス（最大100文字）
);

各列の説明

  • • id: 各連絡先を区別するための番号（INTEGER型）
  • • name: 名前（VARCHAR型、最大100文字）
  • • phone: 電話番号（VARCHAR型、最大20文字）
  • • email: メールアドレス（VARCHAR型、最大100文字）

注意事項

  • • テーブル名や列名には、日本語も使えますが、英数字とアンダースコア（_）を使うのが一般的です
  • • テーブル名や列名は大文字・小文字を区別しませんが、小文字で統一するのが推奨されます
  • • SQLのキーワード（CREATE、TABLEなど）は大文字で書くと読みやすくなります
  • • --（ハイフン2つ）の後ろはコメントです。実行には影響しません

3.5 pgAdminでテーブルを作成する

それでは、pgAdminを使って実際にテーブルを作成してみましょう。

手順

  1. 1 pgAdminを起動し、PostgreSQLサーバーに接続します
  2. 2 左側のツリーから「Databases」→「postgres」を選択します
  3. 3 上部メニューの「Tools」→「Query Tool」をクリックしてクエリエディタを開きます
  4. 4 クエリエディタに上記のCREATE TABLE文を入力（またはコピー＆ペースト）します
  5. 5 再生ボタン（▶）または「F5」キーを押してSQL文を実行します
  6. 6 「Query returned successfully」と表示されれば成功です！

テーブルが作成されたか確認する方法

左側のツリーで「Databases」→「postgres」→「Schemas」→「public」→「Tables」を展開すると、作成した「contacts」テーブルが表示されます。

postgres
└── Schemas
    └── public
        └── Tables
            └── contacts  ← ここに表示されます

3.6 テーブル構造を確認する

作成したテーブルの構造（どんな列があるか）を確認するには、いくつかの方法があります。

方法1: pgAdminで確認

左側のツリーで「contacts」テーブルを右クリック→「Properties」→「Columns」タブで列の一覧を確認できます。

方法2: SQLで確認

PostgreSQLの「\d」コマンド（psqlツール用）や、次のSQL文でテーブル構造を確認できます。

SQLでテーブル構造を確認する

SQL 
-- テーブルの列情報を表示
SELECT column_name, data_type, character_maximum_length
FROM information_schema.columns
WHERE table_name = 'contacts';

このSQL文を実行すると、contactsテーブルの列名、データ型、文字列の最大長が表示されます。

理解度確認クイズ

  1. Q1. テーブルとは何ですか？簡単に説明してください。

    回答を表示

    テーブルとは、データベースの中でデータを整理して保存するための「表」のような入れ物です。Excelの表と同じように、行と列で構成されています。

  2. Q2. 列（カラム）と行（レコード）の違いを説明してください。

    回答を表示

    列（カラム）は縦方向のデータの区分けで、「どんな情報を保存するか」を表します（例: ID列、名前列）。行（レコード）は横方向のデータのまとまりで、「1件分の情報」を表します（例: 田中さんの情報1行）。

  3. Q3. データ型とは何ですか？なぜ必要ですか？

    回答を表示

    データ型とは、その列にどんな種類のデータを保存するかを指定するものです。例えば、名前は文字、年齢は数値、生年月日は日付というように指定します。データ型を指定することで、データの整合性を保ち、効率的な保存と正確な計算が可能になります。

  4. Q4. 次のテーブルを作成するSQL文を考えてください。テーブル名「students」、列：id（整数）、name（最大50文字）、age（整数）

    回答を表示 
    CREATE TABLE students (
    id INTEGER,
    name VARCHAR(50),
    age INTEGER
);
