第5章

データを見てみよう（SELECT文の基礎）

テーブルに保存したデータを表示する基本的な方法を学びます。全データの表示、特定の列の選択、列名の変更方法を習得しましょう。

この章で学ぶこと

  • SELECT文の基本構文を理解する
  • テーブルの全データを表示する方法（SELECT *）
  • 特定の列だけを選択して表示する
  • 列名に別名（エイリアス）をつける方法を学ぶ
  • 重複を除いてデータを表示する方法（DISTINCT）

5.1 SELECT文とは何か？

SELECT文は、データベースからデータを検索（取得）するためのSQL文です。

図書館で例えると、「〇〇というタイトルの本を探してください」「著者が××さんの本を全部見たい」とお願いするのと同じです。SELECT文を使うことで、テーブルに保存されているデータを必要な形で取り出すことができます。

SELECT文の役割

flowchart LR A[ユーザー] -->|SELECT文を実行| B[データベース] B -->|データを返す| A C[テーブル
大量のデータ] -->|必要なデータだけ| D[検索結果
欲しいデータ] style A fill:#dbeafe style B fill:#fef3c7 style C fill:#fee2e2 style D fill:#d1fae5

ポイント

SELECT文は、データを読み取るだけで、データを変更したり削除したりしません。安心して何度でも実行できます。

5.2 全データを表示してみよう（SELECT * FROM）

まず、テーブルに保存されているすべてのデータを表示する方法を学びましょう。

基本構文

SQL 
SELECT * FROM テーブル名;

SELECT：「データを選択する」という意味

*（アスタリスク）：「すべての列」を表す記号

FROM：「～から」という意味

テーブル名：データを取り出すテーブルの名前

実習：連絡先テーブルのデータをすべて表示

前の章で作成した「連絡先」テーブルのデータを表示してみましょう。

SQL 
-- 連絡先テーブルのすべてのデータを表示
SELECT * FROM 連絡先;

実行結果のイメージ：

ID 名前 電話番号 メール
1 山田太郎 090-1234-5678 yamada@example.com
2 佐藤花子 080-9876-5432 sato@example.com
3 鈴木一郎 070-1111-2222 suzuki@example.com

重要ポイント

*（アスタリスク）を使うと、テーブルのすべての列が表示されます。データの全体像を確認したいときに便利です。

5.3 特定の列だけを表示してみよう

すべての列が必要ない場合は、必要な列だけを指定して表示することができます。

基本構文

SQL 
SELECT 列名1, 列名2, 列名3 FROM テーブル名;

列名をカンマ（,）で区切って指定します。指定した順番で列が表示されます。

実習：名前と電話番号だけを表示

連絡先テーブルから、名前と電話番号の2つの列だけを表示してみましょう。

SQL 
-- 名前と電話番号だけを表示
SELECT 名前, 電話番号 FROM 連絡先;

実行結果のイメージ：

名前 電話番号
山田太郎 090-1234-5678
佐藤花子 080-9876-5432
鈴木一郎 070-1111-2222

列の順番を変える

列の順番を変えて表示することもできます。

SQL 
-- 電話番号、メール、名前の順で表示
SELECT 電話番号, メール, 名前 FROM 連絡先;

このように、列を指定する順番を変えると、表示される順番も変わります。

ポイント

必要な列だけを指定することで、表示が見やすくなるだけでなく、データ取得も高速になるというメリットがあります。

5.4 列名に別名（エイリアス）をつける

検索結果の列名を分かりやすい名前に変更することができます。この機能をエイリアス（別名）と呼びます。

基本構文

SQL 
SELECT 列名 AS 別名 FROM テーブル名;

AS：「～として」という意味のキーワード（省略可能）

別名：表示したい列名

実習：列名を分かりやすく変更

列名を日本語や英語で分かりやすい名前に変更してみましょう。

SQL 
-- 列名を分かりやすく変更
SELECT
    名前 AS 氏名,
    電話番号 AS 連絡先電話,
    メール AS メールアドレス
FROM 連絡先;

実行結果のイメージ：

氏名 連絡先電話 メールアドレス
山田太郎 090-1234-5678 yamada@example.com
佐藤花子 080-9876-5432 sato@example.com
鈴木一郎 070-1111-2222 suzuki@example.com

ASキーワードは省略できる

ASキーワードは省略しても同じ結果になります。

SQL 
-- ASを省略した書き方
SELECT
    名前 氏名,
    電話番号 連絡先電話,
    メール メールアドレス
FROM 連絡先;

どちらの書き方でも結果は同じですが、ASをつけた方が読みやすいのでおすすめです。

重要ポイント

エイリアスは検索結果の表示名を変えるだけで、テーブルの列名そのものは変わりません。また、レポートや集計結果を分かりやすくするために頻繁に使われます。

5.5 重複を除いてデータを表示する（DISTINCT）

同じ値が複数回出てくる場合、DISTINCTキーワードを使うことで重複を除いた結果を表示できます。

基本構文

SQL 
SELECT DISTINCT 列名 FROM テーブル名;

DISTINCTをSELECTの直後に記述します。

使用例

「商品テーブル」から、重複しないカテゴリの一覧を取得する場合：

SQL 
-- 重複しないカテゴリの一覧を取得
SELECT DISTINCT カテゴリ FROM 商品;

DISTINCTなし（重複あり）：

カテゴリ
文房具
文房具
食品
雑貨
食品

DISTINCTあり（重複除去）：

カテゴリ
文房具
食品
雑貨

実習：複数列での重複除去

複数の列を指定することもできます。その場合、すべての列の組み合わせが重複しているデータだけが除外されます。

SQL 
-- カテゴリとメーカーの組み合わせを重複なく取得
SELECT DISTINCT カテゴリ, メーカー FROM 商品;

ポイント

DISTINCTは、「どんな種類のデータがあるか」を確認したい場合に便利です。例えば、「どんなカテゴリの商品があるか」「どの都道府県の顧客がいるか」などを調べるときに使います。

5.6 この章のまとめ

第5章で学んだこと

  • SELECT文は、データベースからデータを検索（取得）するためのSQL文
  • SELECT * FROM テーブル名で、すべての列のデータを表示できる
  • 列名をカンマ区切りで指定することで、必要な列だけを表示できる
  • ASキーワードを使って、列名を分かりやすい別名（エイリアス）に変更できる
  • DISTINCTを使うと、重複を除いた結果を表示できる

SELECT文の基本パターンまとめ

SQL 
-- パターン1: すべての列を表示
SELECT * FROM テーブル名;

-- パターン2: 特定の列だけ表示
SELECT 列名1, 列名2 FROM テーブル名;

-- パターン3: 列名を別名で表示
SELECT 列名1 AS 別名1, 列名2 AS 別名2 FROM テーブル名;

-- パターン4: 重複を除いて表示
SELECT DISTINCT 列名 FROM テーブル名;

理解度確認クイズ

  1. Q1. SELECT文は何をするためのSQL文ですか？

    回答を表示

    SELECT文は、データベースからデータを検索（取得）するためのSQL文です。データを読み取るだけで、変更や削除はしません。

  2. Q2. 「商品」テーブルのすべてのデータを表示するSQL文を書いてください。

    回答を表示

    SELECT * FROM 商品;

  3. Q3. 「顧客」テーブルから「名前」と「住所」の2つの列だけを表示するSQL文を書いてください。

    回答を表示

    SELECT 名前, 住所 FROM 顧客;

  4. Q4. エイリアス（別名）を使う目的は何ですか？

    回答を表示

    検索結果の列名を分かりやすい名前に変更するためです。レポートや集計結果を見やすくしたり、複雑な計算結果に意味のある名前をつけたりするときに使います。

  5. Q5. 「商品」テーブルから重複しない「カテゴリ」の一覧を取得するSQL文を書いてください。

    回答を表示

    SELECT DISTINCT カテゴリ FROM 商品;
