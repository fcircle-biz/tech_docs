第7章

条件を付けてデータを探そう（WHERE句）

WHERE句を使って特定の条件に合うデータだけを取り出す方法を、様々な実例で学習します。

この章で学ぶこと

  • WHERE句を使った絞り込み検索の基本を理解する
  • 比較演算子（=、<、>、<=、>=、<>）の使い方を学ぶ
  • LIKE演算子を使ったあいまい検索を習得する
  • 複数条件を組み合わせた検索（AND、OR）ができるようになる

7.1 WHERE句とは？

これまで学んだSELECT文では、テーブルのすべてのデータを取り出していました。しかし、実際の業務では「特定の条件に合うデータだけ」を取り出したいことがほとんどです。

例えば、「東京都に住んでいる顧客だけを表示したい」「価格が1000円以上の商品を探したい」「名前に『田中』が含まれる人を検索したい」といった場合、WHERE句を使います。

WHERE句で絞り込むイメージ

WHERE句の基本構文

SELECT 列名 FROM テーブル名 WHERE 条件;

7.2 比較演算子を使った条件指定

WHERE句では、比較演算子を使って条件を指定します。以下の演算子が使えます。

演算子 意味 例
= 等しい 年齢 = 30
<> または != 等しくない 県名 <> '東京都'
< より小さい（未満） 価格 < 1000
> より大きい 在庫数 > 0
<= 以下 年齢 <= 65
>= 以上 売上 >= 10000

実習 7-1: 比較演算子を使った検索

顧客テーブルから特定の条件に合うデータを検索してみましょう。

サンプルデータ

テーブル作成とデータ挿入 
-- 顧客テーブル作成
CREATE TABLE 顧客 (
    顧客ID INT PRIMARY KEY,
    名前 VARCHAR(50),
    年齢 INT,
    都道府県 VARCHAR(20),
    購入金額 INT
);

-- データ挿入
INSERT INTO 顧客 VALUES
(1, '田中太郎', 28, '東京都', 15000),
(2, '佐藤花子', 35, '大阪府', 8000),
(3, '鈴木一郎', 42, '東京都', 25000),
(4, '高橋美咲', 29, '神奈川県', 12000),
(5, '渡辺健太', 51, '愛知県', 18000);

例1: 年齢が30歳以上の顧客を検索

SQL 
SELECT * FROM 顧客 WHERE 年齢 >= 30;

結果: 佐藤花子（35歳）、鈴木一郎（42歳）、渡辺健太（51歳）が表示されます。

例2: 購入金額が10000円未満の顧客を検索

SQL 
SELECT 名前, 購入金額 FROM 顧客 WHERE 購入金額 < 10000;

結果: 佐藤花子（8000円）のみが表示されます。

7.3 LIKE演算子によるあいまい検索

「名前に『田中』が含まれる人を検索したい」「メールアドレスが『gmail.com』で終わる顧客を探したい」といった場合、LIKE演算子を使います。

LIKE演算子では、ワイルドカードと呼ばれる特殊な記号を使ってパターンを指定します。

ワイルドカード 意味 例 マッチする例
% 0文字以上の任意の文字列 '田中%' 田中、田中太郎、田中花子
_ 任意の1文字 '田中_郎' 田中太郎、田中一郎

LIKEパターンの使い方

名前 LIKE '田中%'

→ 「田中」で始まる名前（田中太郎、田中花子など）

名前 LIKE '%太郎'

→ 「太郎」で終わる名前（田中太郎、山田太郎など）

名前 LIKE '%太%'

→ 「太」を含む名前（田中太郎、山田太一など）

名前 LIKE '田中_郎'

→ 「田中」+任意の1文字+「郎」（田中太郎、田中一郎など、田中二三郎は×）

実習 7-2: LIKE演算子を使った検索

例1: 名前に「田中」を含む顧客を検索

SQL 
SELECT * FROM 顧客 WHERE 名前 LIKE '%田中%';

例2: 都道府県名が「東京」で始まる顧客を検索

SQL 
SELECT 名前, 都道府県 FROM 顧客 WHERE 都道府県 LIKE '東京%';

注意点

LIKE演算子では、%や_を文字として検索したい場合はエスケープが必要です（例: 'abc\%' で「abc%」という文字列を検索）。また、大量データでLIKE検索すると遅くなることがあります。

7.4 その他の便利な条件指定

BETWEEN演算子（範囲指定）

BETWEEN演算子を使うと、数値や日付の範囲を簡単に指定できます。

BETWEEN例 
-- 年齢が30歳以上40歳以下の顧客を検索
SELECT * FROM 顧客 WHERE 年齢 BETWEEN 30 AND 40;

-- 上記は以下と同じ意味
SELECT * FROM 顧客 WHERE 年齢 >= 30 AND 年齢 <= 40;

IN演算子（複数の値を指定）

IN演算子を使うと、複数の値のいずれかに一致するデータを検索できます。

IN例 
-- 東京都、大阪府、愛知県のいずれかに住む顧客を検索
SELECT * FROM 顧客 WHERE 都道府県 IN ('東京都', '大阪府', '愛知県');

-- 上記は以下と同じ意味
SELECT * FROM 顧客
WHERE 都道府県 = '東京都'
   OR 都道府県 = '大阪府'
   OR 都道府県 = '愛知県';

IS NULL演算子（NULL値の検索）

データが入っていない（NULL）かどうかを調べるには、IS NULLまたはIS NOT NULLを使います。

IS NULL例 
-- 電話番号が登録されていない顧客を検索
SELECT * FROM 顧客 WHERE 電話番号 IS NULL;

-- 電話番号が登録されている顧客を検索
SELECT * FROM 顧客 WHERE 電話番号 IS NOT NULL;

重要ポイント

NULL値の比較では、= NULL ではなく IS NULLを使います。これは、NULLは「値が不明」という特別な状態なので、通常の比較演算子では正しく判定できないためです。

7.5 複数の条件を組み合わせる（AND、OR、NOT）

実際の業務では、複数の条件を組み合わせて検索することがよくあります。AND、OR、NOTを使って条件を組み合わせることができます。

論理演算子 意味 例
AND すべての条件を満たす 年齢 >= 30 AND 都道府県 = '東京都'
OR いずれかの条件を満たす 都道府県 = '東京都' OR 都道府県 = '大阪府'
NOT 条件を満たさない NOT 都道府県 = '東京都'

実習 7-3: 複数条件の組み合わせ

例1: 東京都在住で年齢が30歳以上の顧客（AND）

SQL 
SELECT * FROM 顧客
WHERE 都道府県 = '東京都' AND 年齢 >= 30;

結果: 鈴木一郎（42歳、東京都）のみが表示されます。

例2: 東京都または大阪府在住の顧客（OR）

SQL 
SELECT * FROM 顧客
WHERE 都道府県 = '東京都' OR 都道府県 = '大阪府';

結果: 田中太郎、佐藤花子、鈴木一郎が表示されます。

例3: 複雑な条件の組み合わせ

SQL 
-- 「東京都在住で購入金額が20000円以上」または「年齢が50歳以上」
SELECT * FROM 顧客
WHERE (都道府県 = '東京都' AND 購入金額 >= 20000)
   OR 年齢 >= 50;

結果: 鈴木一郎（東京都、25000円）と渡辺健太（51歳）が表示されます。

注意: ANDとORの優先順位

ANDはORより優先されます。例えば「A OR B AND C」は「A OR (B AND C)」として解釈されます。意図した通りに動作させるため、カッコ ( ) を使って優先順位を明示することをお勧めします。

7.6 この章のまとめ

第7章で学んだこと

  • WHERE句を使うと、特定の条件に合うデータだけを取り出せる
  • 比較演算子（=、<、>、<=、>=、<>）で数値や文字列の比較ができる
  • LIKE演算子とワイルドカード（%、_）であいまい検索ができる
  • BETWEENで範囲指定、INで複数値指定、IS NULLでNULL判定ができる
  • AND、OR、NOTで複数の条件を組み合わせられる

理解度確認クイズ

  1. Q1. WHERE句は何のために使いますか？

    回答を表示

    WHERE句は、特定の条件に合うデータだけを絞り込んで取り出すために使います。例えば、「年齢が30歳以上」「東京都在住」といった条件を指定できます。

  2. Q2. 「価格が1000円以上3000円以下」の商品を検索するSQL文を2通りの書き方で書いてください。

    回答を表示

    【書き方1】BETWEEN演算子を使う
    SELECT * FROM 商品 WHERE 価格 BETWEEN 1000 AND 3000;

    【書き方2】比較演算子を使う
    SELECT * FROM 商品 WHERE 価格 >= 1000 AND 価格 <= 3000;

  3. Q3. LIKE演算子のワイルドカード「%」と「_」の違いを説明してください。

    回答を表示

    %は「0文字以上の任意の文字列」を表します。例: '田中%'は「田中」「田中太郎」「田中花子」などにマッチします。

    _は「任意の1文字」を表します。例: '田中_郎'は「田中太郎」「田中一郎」にマッチしますが、「田中二三郎」（4文字）にはマッチしません。

  4. Q4. NULL値を検索するときに「= NULL」ではなく「IS NULL」を使う理由は何ですか？

    回答を表示

    NULLは「値が不明」という特別な状態を表すため、通常の比較演算子（=）では正しく判定できません。IS NULLを使うことで、正確にNULL値を判定できます。

トラブルシューティング

WHERE句で文字列を検索しても結果が0件になる

原因: 文字列の比較では、全角・半角、大文字・小文字が区別されます。また、前後の空白も影響します。

解決方法: データに含まれる正確な文字列を確認してください。PostgreSQLでは、大文字・小文字を区別しない検索にはILIKE演算子が使えます（例: WHERE 名前 ILIKE 'tanaka'）。

ANDとORを組み合わせたら意図しない結果になる

原因: ANDはORより優先順位が高いため、意図した条件と異なる組み合わせになっています。

解決方法: カッコ ( ) を使って優先順位を明示してください。例: (A OR B) AND C

LIKE検索で%や_を文字として検索したい

原因: %や_はワイルドカードとして解釈されるため、そのままでは文字として検索できません。

解決方法: エスケープ文字（\）を使います。例: WHERE 商品名 LIKE '%10\%%'（「10%」を含む商品名を検索）
