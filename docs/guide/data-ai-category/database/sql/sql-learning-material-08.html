第8章

データを並び替えて見やすくしよう

ORDER BY句でデータを昇順・降順に並び替える方法、LIMIT句で表示件数を制限する方法を学習します。「売上トップ10」「最新の5件」といった実用的な使い方も紹介します。

この章で学ぶこと

  • ORDER BY句を使ってデータを昇順・降順に並び替える方法を理解する
  • 複数の列を使った並び替えの方法を学ぶ
  • LIMIT句を使って表示件数を制限する方法を習得する
  • ORDER BYとLIMITを組み合わせた実用的な使い方を身につける

8.1 なぜデータを並び替える必要があるのか？

データベースから取り出したデータは、登録された順番で表示されることが多いです。しかし、実際の業務では「売上が高い順に並べたい」「最新の注文から見たい」「名前順に整理したい」といったニーズがよくあります。

例えば、以下のような場面を思い浮かべてみてください。

売上ランキング

売上トップ10の商品を確認したい → 売上額の高い順に並び替えて、上位10件だけ表示

最新の投稿

最新の5件の投稿を表示したい → 投稿日時の新しい順に並び替えて、5件だけ表示

五十音順リスト

顧客名簿を見やすくしたい → 名前を五十音順（昇順）に並び替え

価格の安い順

予算内の商品を探したい → 価格の安い順（昇順）に並び替え

ポイント

このような「データを見やすく整理する」機能を実現するのが、ORDER BY句（並び替え）とLIMIT句（件数制限）です。

8.2 ORDER BY句の基本 - 昇順と降順

ORDER BY句は、SELECT文の結果を指定した列の値に基づいて並び替えるための構文です。

基本構文

SQL構文 
-- 基本形
SELECT 列名1, 列名2, ...
FROM テーブル名
ORDER BY 列名 [ASC | DESC];

-- ASC: 昇順（小さい順、古い順）← デフォルト
-- DESC: 降順（大きい順、新しい順）

昇順（ASC）と降順（DESC）の違い

flowchart LR A[データの並び順] --> B[ASC 昇順] A --> C[DESC 降順] B --> D["数値: 1, 2, 3, 4, 5..."] B --> E["日付: 古い → 新しい"] B --> F["文字: あ, い, う, え, お..."] C --> G["数値: 100, 90, 80, 70..."] C --> H["日付: 新しい → 古い"] C --> I["文字: わ, ら, や, ま..."] style A fill:#dbeafe style B fill:#dcfce7 style C fill:#fef3c7

昇順の例（ASC） - 安い順

商品テーブルから価格の安い順に商品を取り出す例です。

SQL例 
-- 価格の安い順に並び替え（昇順）
SELECT 商品名, 価格
FROM 商品
ORDER BY 価格 ASC;

-- ASCは省略可能（デフォルトが昇順なので）
SELECT 商品名, 価格
FROM 商品
ORDER BY 価格;

実行結果のイメージ

商品名         | 価格
---------------+------
ボールペン     |   80
消しゴム       |  100
ノート         |  150
シャープペンシル|  200
クリアファイル |  250

降順の例（DESC） - 高い順

価格の高い順に並び替える例です。売上ランキングなどでよく使います。

SQL例 
-- 価格の高い順に並び替え（降順）
SELECT 商品名, 価格
FROM 商品
ORDER BY 価格 DESC;

実行結果のイメージ

商品名         | 価格
---------------+------
クリアファイル |  250
シャープペンシル|  200
ノート         |  150
消しゴム       |  100
ボールペン     |   80

覚え方のコツ

ASCは「Ascending（上昇）」、DESCは「Descending（下降）」の略です。
・ASC（昇順）: 1 → 2 → 3（数が上がっていく）
・DESC（降順）: 3 → 2 → 1（数が下がっていく）

8.3 複数の列で並び替える

ORDER BY句では、複数の列を指定して、優先順位をつけて並び替えることができます。例えば「部署ごとにまとめて、その中で給料の高い順に並べる」といった使い方ができます。

複数列の並び替え構文

SQL構文 
-- 複数列での並び替え
SELECT 列名1, 列名2, ...
FROM テーブル名
ORDER BY 列名1 [ASC|DESC], 列名2 [ASC|DESC], ...;

-- 最初の列で並び替え → 同じ値の場合は次の列で並び替え

実例: 部署別・給料順

社員テーブルから、部署ごとにまとめて、各部署内では給料の高い順に表示する例です。

SQL例 
-- 部署名で昇順に並び替え、同じ部署内では給料の降順
SELECT 社員名, 部署名, 給料
FROM 社員
ORDER BY 部署名 ASC, 給料 DESC;

実行結果のイメージ

社員名   | 部署名   | 給料
---------+----------+--------
佐藤太郎 | 営業部   | 450000
鈴木花子 | 営業部   | 400000
田中次郎 | 営業部   | 350000
高橋三郎 | 経理部   | 420000
山田美咲 | 経理部   | 380000
渡辺健一 | 開発部   | 500000
伊藤優子 | 開発部   | 480000

まず部署名で並び、同じ部署内では給料の高い順に表示されています。

並び替えの優先順位

カンマで区切った順番が優先度になります。
1. まず最初に指定した列で並び替え
2. 同じ値がある場合、次の列で並び替え
3. それでも同じなら、その次の列で並び替え...

8.4 LIMIT句で表示件数を制限する

LIMIT句は、検索結果の表示件数を制限するための構文です。「上位5件だけ見たい」「最初の10件だけ表示」といった場面で使います。

基本構文

SQL構文 
-- 基本形
SELECT 列名1, 列名2, ...
FROM テーブル名
LIMIT 件数;

-- 例: 最初の5件だけ表示
SELECT * FROM 商品 LIMIT 5;

実例: 最初の3件を表示

SQL例 
-- 商品テーブルから最初の3件を取得
SELECT 商品名, 価格
FROM 商品
LIMIT 3;

実行結果のイメージ

商品名         | 価格
---------------+------
ボールペン     |   80
消しゴム       |  100
ノート         |  150

（それ以降のデータは表示されない）

注意点

LIMIT句だけでは「どの順番で最初の何件」なのか不明確です。ORDER BY句と組み合わせて使うことで、「価格の安い順で最初の5件」のように意図が明確になります。

8.5 ORDER BYとLIMITの組み合わせ - 実用的な使い方

ORDER BYとLIMITを組み合わせることで、「売上トップ10」「最新の5件」のような実用的なクエリが書けます。

書く順番（重要）

SQL構文 
-- 正しい順番
SELECT 列名1, 列名2, ...
FROM テーブル名
WHERE 条件
ORDER BY 列名 [ASC|DESC]
LIMIT 件数;

-- ポイント: ORDER BY → LIMIT の順番で書く

例1: 売上トップ10

売上が高い順に並び替えて、上位10件だけを表示する例です。

SQL例 
-- 売上トップ10の商品を取得
SELECT 商品名, 売上金額
FROM 売上
ORDER BY 売上金額 DESC
LIMIT 10;

実行結果のイメージ

商品名         | 売上金額
---------------+----------
商品A          | 1500000
商品B          | 1200000
商品C          |  980000
...（中略）...
商品J          |  650000

（上位10件のみ表示）

例2: 最新の5件の投稿

投稿日時の新しい順に並び替えて、最新5件を表示する例です。

SQL例 
-- 最新の5件の投稿を取得
SELECT タイトル, 投稿者, 投稿日時
FROM 投稿
ORDER BY 投稿日時 DESC
LIMIT 5;

例3: 価格が安い順で3件

予算内で買える商品を探すときに便利です。

SQL例 
-- 1000円以下の商品を価格の安い順に3件取得
SELECT 商品名, 価格
FROM 商品
WHERE 価格 <= 1000
ORDER BY 価格 ASC
LIMIT 3;

実行順序の理解

SQLの実行順序は次の通りです：
1. FROM: テーブルを指定
2. WHERE: 条件で絞り込み
3. ORDER BY: 並び替え
4. LIMIT: 件数制限

実習: ORDER BYとLIMITを使ってみよう

これまで使ってきた「連絡先」テーブルや「商品」テーブルで、ORDER BYとLIMITを試してみましょう。

練習1: 名前順に並び替え

連絡先テーブルから、名前を五十音順（昇順）に並び替えて全件表示してください。

SQL例を表示
SQL例 
SELECT * FROM 連絡先
ORDER BY 名前 ASC;

練習2: 最新の3件を取得

登録日時が新しい順に並び替えて、最新3件を表示してください。

SQL例を表示
SQL例 
SELECT 名前, 登録日時
FROM 連絡先
ORDER BY 登録日時 DESC
LIMIT 3;

練習3: 複数条件での並び替え

カテゴリ別に分類し、同じカテゴリ内では価格の高い順に表示してください。

SQL例を表示
SQL例 
SELECT 商品名, カテゴリ, 価格
FROM 商品
ORDER BY カテゴリ ASC, 価格 DESC;

8.6 この章のまとめ

第8章で学んだこと

  • ORDER BY句を使うと、データを昇順（ASC）または降順（DESC）に並び替えられる
  • ASCは小さい順・古い順、DESCは大きい順・新しい順（ASCはデフォルトなので省略可能）
  • 複数列での並び替えも可能で、カンマで区切った順番が優先度になる
  • LIMIT句を使うと、表示する件数を制限できる
  • ORDER BYとLIMITを組み合わせることで、「売上トップ10」「最新の5件」のような実用的なクエリが書ける
  • SQL文の書く順番はWHERE → ORDER BY → LIMITの順

理解度確認クイズ

  1. Q1. ORDER BY句で昇順に並び替えるときのキーワードは何ですか？また、省略した場合はどうなりますか？

    回答を表示

    キーワードは「ASC」です。省略した場合もデフォルトで昇順（ASC）になります。

  2. Q2. 売上金額の高い順に上位5件を表示するSQL文を書いてください。

    回答を表示

    SELECT * FROM テーブル名 ORDER BY 売上金額 DESC LIMIT 5;

  3. Q3. 複数列で並び替える場合、優先順位はどのように決まりますか？

    回答を表示

    ORDER BYの後にカンマで区切って列名を指定した順番が優先度になります。最初に指定した列で並び替え、同じ値の場合は次の列で並び替えます。

  4. Q4. WHERE、ORDER BY、LIMITを使う場合、正しい書く順番は何ですか？

    回答を表示

    WHERE → ORDER BY → LIMIT の順番です。
    SELECT ... FROM ... WHERE ... ORDER BY ... LIMIT ...;
