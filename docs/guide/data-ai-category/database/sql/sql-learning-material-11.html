第11章

より複雑な検索をしてみよう（サブクエリ入門）

SQLの中にSQLを書く「サブクエリ」の基本概念を学習し、平均値以上の売上がある店舗のような、少し複雑な条件での検索方法を段階的に理解していきます。

この章で学ぶこと

  • サブクエリとは何か、その役割を理解する
  • WHERE句でのサブクエリの使い方を学ぶ
  • FROM句でのサブクエリの使い方を学ぶ
  • IN演算子とサブクエリの組み合わせを習得する
  • 実務でよく使うサブクエリのパターンを理解する

11.1 サブクエリとは何か？

サブクエリ（副問い合わせ）とは、SQL文の中に別のSQL文を入れ子にする仕組みのことです。「クエリの中のクエリ」とも言えます。

これまで学んできたSELECT文、WHERE句、集計関数などを組み合わせて、より複雑な条件での検索を実現できます。

サブクエリの構造イメージ

サブクエリが役立つ場面

「平均値以上のデータを抽出したい」「特定の条件に合う顧客の注文だけを見たい」など、一度の問い合わせだけでは表現しにくい条件を扱うときにサブクエリが活躍します。

11.2 サブクエリの基本構文

サブクエリは、必ずカッコ ( ) で囲みます。基本的な構文は以下の通りです。

サブクエリの基本形 
-- 基本構文
SELECT 列名
FROM テーブル名
WHERE 列名 = (サブクエリ);

-- 具体例：平均価格以上の商品を検索
SELECT 商品名, 価格
FROM 商品テーブル
WHERE 価格 >= (SELECT AVG(価格) FROM 商品テーブル);

重要なルール

  • • サブクエリは必ずカッコ ( ) で囲む
  • • サブクエリはSELECT文を使う
  • • サブクエリは先に実行され、その結果をメインクエリが利用する

11.3 WHERE句でのサブクエリ

WHERE句でサブクエリを使うと、動的な条件での絞り込みが可能になります。最もよく使われるパターンです。

例1：平均値以上のデータを抽出

まず、テーブルを準備します。

テーブル準備 
-- 売上テーブルを作成
CREATE TABLE 売上 (
    店舗ID INT,
    店舗名 VARCHAR(100),
    月間売上 INT
);

-- データを挿入
INSERT INTO 売上 (店舗ID, 店舗名, 月間売上) VALUES
(1, '新宿店', 500000),
(2, '渋谷店', 800000),
(3, '池袋店', 450000),
(4, '品川店', 700000),
(5, '横浜店', 600000);

「平均売上以上の店舗」を検索する場合、サブクエリを使うと以下のように書けます。

平均以上の店舗を検索 
-- 平均売上以上の店舗を検索
SELECT 店舗名, 月間売上
FROM 売上
WHERE 月間売上 >= (SELECT AVG(月間売上) FROM 売上)
ORDER BY 月間売上 DESC;

実行結果

店舗名 月間売上
渋谷店 800000
品川店 700000
横浜店 600000

平均売上（610000円）以上の3店舗が抽出されました。

例2：最大値を持つデータを抽出

最高売上の店舗を検索 
-- 最高売上の店舗を検索
SELECT 店舗名, 月間売上
FROM 売上
WHERE 月間売上 = (SELECT MAX(月間売上) FROM 売上);

ポイント

サブクエリで集計関数（AVG, MAX, MIN, SUM, COUNTなど）を使うことで、動的に計算された値を条件に使えるのが大きな利点です。

11.4 IN演算子とサブクエリ

サブクエリが複数の値を返す場合は、IN演算子と組み合わせます。

まず、顧客テーブルと注文テーブルを準備します。

テーブル準備 
-- 顧客テーブルを作成
CREATE TABLE 顧客 (
    顧客ID INT PRIMARY KEY,
    顧客名 VARCHAR(100),
    地域 VARCHAR(50)
);

-- 注文テーブルを作成
CREATE TABLE 注文 (
    注文ID INT PRIMARY KEY,
    顧客ID INT,
    注文金額 INT,
    注文日 DATE
);

-- データを挿入
INSERT INTO 顧客 (顧客ID, 顧客名, 地域) VALUES
(1, '田中商店', '東京'),
(2, '鈴木物産', '大阪'),
(3, '佐藤商会', '東京'),
(4, '高橋貿易', '福岡'),
(5, '伊藤工業', '東京');

INSERT INTO 注文 (注文ID, 顧客ID, 注文金額, 注文日) VALUES
(101, 1, 50000, '2025-01-10'),
(102, 3, 80000, '2025-01-15'),
(103, 2, 30000, '2025-01-20'),
(104, 1, 45000, '2025-01-25');

例：東京の顧客の注文だけを抽出

IN演算子でサブクエリを使用 
-- 東京の顧客の注文を検索
SELECT 注文ID, 顧客ID, 注文金額, 注文日
FROM 注文
WHERE 顧客ID IN (SELECT 顧客ID FROM 顧客 WHERE 地域 = '東京')
ORDER BY 注文日;

実行の流れ

sequenceDiagram autonumber participant M as メインクエリ participant S as サブクエリ participant D as データベース M->>S: サブクエリを実行 S->>D: 東京の顧客IDを取得 D-->>S: 顧客ID: 1, 3, 5 S-->>M: 結果を返す M->>D: 顧客ID IN (1, 3, 5) で注文を検索 D-->>M: 該当する注文データ

IN演算子の使いどころ

サブクエリが1つの値を返す場合は「=」、複数の値を返す場合は「IN」を使います。

11.5 FROM句でのサブクエリ（インラインビュー）

FROM句にサブクエリを書くこともできます。この場合、サブクエリの結果を一時的なテーブル（インラインビュー）として扱います。

FROM句でのサブクエリ 
-- 各顧客の総注文金額を計算し、10万円以上の顧客を抽出
SELECT 顧客名, 総注文金額
FROM (
    SELECT
        顧客.顧客ID,
        顧客.顧客名,
        SUM(注文.注文金額) AS 総注文金額
    FROM 顧客
    INNER JOIN 注文 ON 顧客.顧客ID = 注文.顧客ID
    GROUP BY 顧客.顧客ID, 顧客.顧客名
) AS 顧客別集計
WHERE 総注文金額 >= 100000
ORDER BY 総注文金額 DESC;

エイリアスが必須

FROM句のサブクエリには、必ずエイリアス（別名）を付ける必要があります。上記の例では「AS 顧客別集計」がエイリアスです。

11.6 SELECT句でのサブクエリ（スカラーサブクエリ）

SELECT句にサブクエリを書くこともできます。この場合、サブクエリは1つの値を返す必要があります（スカラーサブクエリ）。

SELECT句でのサブクエリ 
-- 各店舗の売上と、全体平均との差を表示
SELECT
    店舗名,
    月間売上,
    (SELECT AVG(月間売上) FROM 売上) AS 平均売上,
    月間売上 - (SELECT AVG(月間売上) FROM 売上) AS 平均との差
FROM 売上
ORDER BY 月間売上 DESC;

注意

SELECT句のサブクエリは、必ず1つの値を返すようにしてください。複数の行や列を返すとエラーになります。

11.7 実践例：実務でよく使うサブクエリパターン

パターン1：トップN件を除外

トップ1以外の店舗を抽出 
-- 売上トップ以外の店舗を抽出
SELECT 店舗名, 月間売上
FROM 売上
WHERE 月間売上 < (SELECT MAX(月間売上) FROM 売上)
ORDER BY 月間売上 DESC;

パターン2：存在チェック（EXISTS）

EXISTS演算子は、サブクエリが1件でもデータを返すかどうかをチェックします。

注文のある顧客のみを抽出 
-- 注文実績のある顧客のみを抽出
SELECT 顧客名, 地域
FROM 顧客
WHERE EXISTS (
    SELECT 1
    FROM 注文
    WHERE 注文.顧客ID = 顧客.顧客ID
);

パターン3：NOTを使った除外

注文のない顧客を抽出 
-- 注文実績のない顧客を抽出
SELECT 顧客名, 地域
FROM 顧客
WHERE NOT EXISTS (
    SELECT 1
    FROM 注文
    WHERE 注文.顧客ID = 顧客.顧客ID
);

EXISTSとINの使い分け

EXISTSは「存在するかどうか」をチェックするだけなので、大量データでも高速です。INは実際の値を比較するので、小規模なデータ向きです。

これまで作成したテーブルを使って、サブクエリの練習をしましょう。

課題

トラブルシューティング

エラー「subquery must return only one column」

原因: WHERE句やIN演算子で使うサブクエリが複数の列を返しています。

解決方法: サブクエリのSELECT句を1つの列だけにしてください。例：「SELECT 顧客ID FROM ...」

エラー「more than one row returned by a subquery」

原因: 「=」演算子で使うサブクエリが複数の行を返しています。

解決方法: 複数の行を返す場合は「IN」演算子を使うか、サブクエリにLIMIT 1を追加してください。

エラー「subquery in FROM must have an alias」

原因: FROM句のサブクエリにエイリアスが付いていません。

解決方法: サブクエリの後ろに「AS 別名」を追加してください。例：「) AS 集計結果」

理解度確認クイズ

  1. Q1. サブクエリとは何ですか？簡単に説明してください。

    回答を表示

    サブクエリとは、SQL文の中に別のSQL文を入れ子にする仕組みです。「クエリの中のクエリ」とも呼ばれ、動的な条件での検索や複雑な処理を実現できます。

  2. Q2. サブクエリを書くときに必ず守るべきルールは何ですか？

    回答を表示

    サブクエリは必ずカッコ ( ) で囲む必要があります。また、FROM句でサブクエリを使う場合は、必ずエイリアス（別名）を付ける必要があります。

  3. Q3. サブクエリが複数の値を返す場合、どの演算子を使いますか？

    回答を表示

    IN演算子を使います。例：WHERE 列名 IN (サブクエリ)。単一の値を返す場合は「=」を使います。

  4. Q4. EXISTS演算子の役割は何ですか？

    回答を表示

    EXISTS演算子は、サブクエリが1件でもデータを返すかどうか（存在するかどうか）をチェックします。大量データでも高速に動作するのが特徴です。
