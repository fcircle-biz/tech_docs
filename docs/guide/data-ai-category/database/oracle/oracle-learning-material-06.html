Oracle基本SQL（テーブル作成・CRUD操作）

CREATE TABLE、INSERT、SELECT、UPDATE、DELETEといった基本的なSQL文を学習し、データベース操作の基礎を身に付けます。

この章で学ぶこと

  • CREATE TABLE文でテーブルを作成する方法を理解する
  • INSERT文を使ってデータを挿入する方法を習得する
  • SELECT文でデータを検索・取得する方法を学ぶ
  • UPDATE文でデータを更新する方法を理解する
  • DELETE文でデータを削除する方法を習得する

6.1 CRUDとは

CRUD（クラッド）とは、データベースやアプリケーションで行う基本的な4つの操作の頭文字を取った用語です。データベースを扱う上で最も基礎となる操作群です。

flowchart LR C[Create
作成] --> I[INSERT文] R[Read
読取] --> S[SELECT文] U[Update
更新] --> UP[UPDATE文] D[Delete
削除] --> DEL[DELETE文]
操作 意味 SQL文 説明
Create 作成 INSERT 新しいデータをテーブルに追加
Read 読取 SELECT テーブルからデータを取得
Update 更新 UPDATE 既存のデータを変更
Delete 削除 DELETE テーブルからデータを削除

重要ポイント

これらの基本操作を組み合わせることで、複雑なデータベースアプリケーションを構築できます。まずはこの4つの操作を確実にマスターしましょう。

6.2 CREATE TABLE - テーブルの作成

CREATE TABLE文は、新しいテーブルを作成するためのSQL文です。テーブル名、カラム名、データ型、制約などを定義します。

基本構文

CREATE TABLE構文 
-- テーブル作成の基本構文
CREATE TABLE テーブル名 (
    カラム名1 データ型 [制約],
    カラム名2 データ型 [制約],
    ...
);

主なデータ型

データ型 説明 例
NUMBER(p, s) 数値型（整数・小数） NUMBER(10, 2)
VARCHAR2(n) 可変長文字列（最大4000バイト） VARCHAR2(100)
CHAR(n) 固定長文字列 CHAR(10)
DATE 日付型（時刻含む） DATE
TIMESTAMP 日付時刻型（マイクロ秒まで） TIMESTAMP

主な制約

制約 説明
PRIMARY KEY 主キー（一意で NULL 不可）
NOT NULL NULL（空値）を許可しない
UNIQUE 一意制約（重複不可）
DEFAULT デフォルト値を設定

実践例：社員テーブルの作成

employees.sql 
-- 社員テーブル（EMPLOYEES）の作成
CREATE TABLE employees (
    employee_id   NUMBER(6)      PRIMARY KEY,  -- 社員ID（主キー）
    first_name    VARCHAR2(50)   NOT NULL,     -- 名
    last_name     VARCHAR2(50)   NOT NULL,     -- 姓
    email         VARCHAR2(100)  UNIQUE,       -- メールアドレス（一意）
    hire_date     DATE           DEFAULT SYSDATE, -- 入社日（デフォルトは今日）
    salary        NUMBER(10, 2),                -- 給与
    department_id NUMBER(4)                      -- 部署ID
);

NUMBER型の精度指定

NUMBER(p, s) の p は全体の桁数、s は小数点以下の桁数です。例えば NUMBER(10, 2) は「全体10桁、小数点以下2桁」を意味し、最大 99999999.99 まで格納できます。

6.3 INSERT - データの挿入

INSERT文は、テーブルに新しいデータ（行）を追加するSQL文です。1行ずつ挿入する方法と、複数行を一度に挿入する方法があります。

基本構文

INSERT構文 
-- 基本構文（全カラム指定）
INSERT INTO テーブル名 (カラム1, カラム2, ...)
VALUES (値1, 値2, ...);

-- 全カラムに挿入する場合はカラム名省略可
INSERT INTO テーブル名
VALUES (値1, 値2, ...);

実践例：社員データの挿入

insert_employee.sql 
-- 1件のデータを挿入
INSERT INTO employees (employee_id, first_name, last_name, email, hire_date, salary, department_id)
VALUES (1001, '太郎', '山田', 'yamada@example.com', DATE '2024-01-15', 350000, 10);

-- 一部のカラムのみ挿入（hire_dateはデフォルト値が使われる）
INSERT INTO employees (employee_id, first_name, last_name, email, salary)
VALUES (1002, '花子', '佐藤', 'sato@example.com', 380000);

-- 複数行を一度に挿入（Oracle 23c以降）
INSERT INTO employees (employee_id, first_name, last_name, email, salary, department_id)
VALUES
    (1003, '次郎', '鈴木', 'suzuki@example.com', 320000, 20),
    (1004, '美咲', '田中', 'tanaka@example.com', 340000, 10),
    (1005, '健太', '高橋', 'takahashi@example.com', 310000, 30);

ヒント

文字列は必ずシングルクォート（'）で囲みます。日付はDATE '2024-01-15'のように指定するか、TO_DATE関数を使用します。数値はクォート不要です。

注意

PRIMARY KEY制約があるカラム（employee_idなど）に同じ値を挿入しようとするとエラーになります。また、NOT NULL制約のあるカラムを省略することもできません。

6.4 SELECT - データの検索

SELECT文は、テーブルからデータを取得するSQL文です。最もよく使われる文で、条件指定、並び替え、グループ化など多彩な機能があります。

基本構文

SELECT構文 
-- 基本構文
SELECT カラム1, カラム2, ...
FROM テーブル名
WHERE 条件
ORDER BY カラム名 [ASC|DESC];

SELECT文の主要要素

flowchart TD A[SELECT文] --> B[SELECT句
取得するカラムを指定] A --> C[FROM句
対象テーブルを指定] A --> D[WHERE句
条件を指定] A --> E[ORDER BY句
並び順を指定] A --> F[GROUP BY句
グループ化]

実践例：様々なSELECT文

select_examples.sql 
-- 1. 全カラム、全行を取得
SELECT * FROM employees;

-- 2. 特定のカラムのみ取得
SELECT employee_id, first_name, last_name FROM employees;

-- 3. 条件を指定して取得（WHERE句）
SELECT * FROM employees WHERE department_id = 10;

-- 4. 給与が35万円以上の社員
SELECT first_name, last_name, salary
FROM employees
WHERE salary >= 350000;

-- 5. 複数条件（AND、OR）
SELECT * FROM employees
WHERE department_id = 10 AND salary > 300000;

-- 6. あいまい検索（LIKE）
SELECT * FROM employees WHERE email LIKE '%example.com';

-- 7. 並び替え（ORDER BY）- 給与の降順
SELECT first_name, last_name, salary
FROM employees
ORDER BY salary DESC;

WHERE句の主な演算子

演算子 説明 例
= 等しい department_id = 10
<> または != 等しくない salary <> 0
>, <, >=, <= 大小比較 salary >= 300000
LIKE あいまい検索 email LIKE '%@example.com'
IN 複数の値のいずれか department_id IN (10, 20, 30)
BETWEEN 範囲指定 salary BETWEEN 300000 AND 400000
IS NULL NULLかどうか department_id IS NULL

重要ポイント

LIKEで使うワイルドカード：% は「任意の0文字以上」、_ は「任意の1文字」を意味します。例：'%田%' は「田を含む」、'山_' は「山で始まる2文字」

6.5 UPDATE - データの更新

UPDATE文は、既存のデータを変更するSQL文です。WHERE句で対象行を指定し、SET句で変更後の値を指定します。

基本構文

UPDATE構文 
-- 基本構文
UPDATE テーブル名
SET カラム1 = 値1, カラム2 = 値2, ...
WHERE 条件;

実践例：社員データの更新

update_employee.sql 
-- 1. 特定の社員の給与を更新
UPDATE employees
SET salary = 400000
WHERE employee_id = 1001;

-- 2. 複数のカラムを同時に更新
UPDATE employees
SET salary = 420000, department_id = 20
WHERE employee_id = 1002;

-- 3. 条件に合う複数行を一括更新
UPDATE employees
SET salary = salary * 1.1  -- 給与を10%アップ
WHERE department_id = 10;

-- 4. 計算式を使った更新
UPDATE employees
SET salary = salary + 50000
WHERE hire_date < DATE '2024-01-01';

つまずきポイント

WHERE句を忘れると、テーブルの全行が更新されてしまいます。必ずWHERE句で対象を限定しましょう。更新前にSELECT文で対象行を確認する習慣をつけると安全です。

実習 6-1: UPDATE文の安全な実行

更新前に対象データを確認してからUPDATEを実行する練習をしましょう。

手順

  1. 1 まずSELECT文で更新対象を確認: SELECT * FROM employees WHERE employee_id = 1001;
  2. 2 UPDATE文を実行: UPDATE employees SET salary = 450000 WHERE employee_id = 1001;
  3. 3 更新後に再度SELECT文で確認: 給与が変更されたことを確認

6.6 DELETE - データの削除

DELETE文は、テーブルからデータ（行）を削除するSQL文です。UPDATEと同様に、WHERE句で削除対象を指定します。

基本構文

DELETE構文 
-- 基本構文
DELETE FROM テーブル名
WHERE 条件;

実践例：社員データの削除

delete_employee.sql 
-- 1. 特定の社員を削除
DELETE FROM employees
WHERE employee_id = 1001;

-- 2. 条件に合う複数行を削除
DELETE FROM employees
WHERE department_id = 99;

-- 3. 入社日が古いデータを削除
DELETE FROM employees
WHERE hire_date < DATE '2020-01-01';

重要な注意

WHERE句を省略すると、テーブルの全データが削除されます。DELETE文は特に慎重に実行してください。本番環境では必ず事前にSELECT文で対象を確認し、可能であればバックアップを取得してから実行しましょう。

DELETE vs TRUNCATE

項目 DELETE TRUNCATE
用途 条件に合う行を削除 テーブルの全データを削除
WHERE句 使用可能 使用不可
ロールバック 可能 不可（即座に確定）
速度 遅い 高速
TRUNCATE例 
-- テーブルの全データを高速削除（ロールバック不可）
TRUNCATE TABLE employees;

6.7 トランザクション基礎（COMMIT / ROLLBACK）

OracleではINSERT、UPDATE、DELETEなどのデータ変更は、COMMIT文を実行するまで確定しません。間違えた場合はROLLBACK文で取り消すことができます。

flowchart LR A[データ変更
INSERT/UPDATE/DELETE] --> B{確認} B -->|正しい| C[COMMIT
変更を確定] B -->|間違い| D[ROLLBACK
変更を取消] C --> E[データベースに反映] D --> F[変更前の状態に戻る]
トランザクション例 
-- データを挿入
INSERT INTO employees (employee_id, first_name, last_name, email, salary)
VALUES (1006, '一郎', '中村', 'nakamura@example.com', 360000);

-- 確認のため検索
SELECT * FROM employees WHERE employee_id = 1006;

-- 問題なければ確定
COMMIT;

-- もし間違えていたら取り消し（COMMITする前のみ有効）
-- ROLLBACK;

ヒント

SQL Developerでは、画面上の「コミット」ボタンまたは「ロールバック」ボタンでも操作できます。練習時はROLLBACKを活用して、何度でもやり直しながら学習しましょう。

実習 6-2: CRUD操作の総合演習

社員テーブルを使って、一連のCRUD操作を実践しましょう。

手順

  1. 1 CREATE: employeesテーブルを作成（カラム：employee_id, first_name, last_name, email, salary）
  2. 2 INSERT: 3件以上の社員データを挿入
  3. 3 SELECT: 全件検索、条件検索、並び替えを試す
  4. 4 UPDATE: 特定の社員の給与を変更し、COMMITで確定
  5. 5 DELETE: 1件削除し、SELECTで削除されたことを確認

理解度確認クイズ

  1. Q1. CRUDのそれぞれに対応するSQL文を答えてください。

    回答を表示

    回答: Create → INSERT文、Read → SELECT文、Update → UPDATE文、Delete → DELETE文。これらはデータベース操作の基本4操作で、すべてのデータベースアプリケーションで使用されます。

  2. Q2. UPDATE文やDELETE文でWHERE句を省略するとどうなりますか？

    回答を表示

    回答: テーブルの全行が対象となります。UPDATE文では全行が更新され、DELETE文では全行が削除されます。これは意図しない結果を招く可能性が高いため、必ずWHERE句で対象を限定することが重要です。実行前にSELECT文で対象を確認する習慣をつけましょう。

  3. Q3. COMMITとROLLBACKの違いを説明してください。

    回答を表示

    回答: COMMITは、INSERT/UPDATE/DELETEなどのデータ変更を確定してデータベースに反映させます。ROLLBACKは、COMMITする前の変更を取り消し、変更前の状態に戻します。間違えた操作をした場合、COMMITする前であればROLLBACKで元に戻せるため、慎重に確認してからCOMMITすることが大切です。
