PL/SQL入門

Oracleの手続き型言語PL/SQLの基本構文を学習します。ストアドプロシージャやファンクションの作成方法を理解します。

この章で学ぶこと

  • PL/SQLとは何か、なぜ使うのかを理解する
  • PL/SQLブロック構文と基本的な変数宣言を習得する
  • IF文やループ文などの制御構文を使えるようになる
  • ストアドプロシージャとファンクションの作成方法を学ぶ
  • 例外処理（エラーハンドリング）の基本を理解する

8.1 PL/SQLとは

PL/SQL（Procedural Language/SQL）は、Oracle Databaseに搭載されているプログラミング言語です。通常のSQLに「手続き的なプログラミング機能」を追加したもので、変数、条件分岐、ループ、関数など、プログラミング言語の標準的な機能を使ってデータベース操作を記述できます。

PL/SQLの特徴

  • データベース内で実行: プログラムがデータベースサーバー内で実行されるため、ネットワーク通信のオーバーヘッドが少なく高速
  • 複数のSQL文をまとめて実行: 複雑なビジネスロジックを一つのまとまりとして実行できる
  • 再利用性が高い: ストアドプロシージャやファンクションとして保存し、繰り返し呼び出せる
  • セキュリティ向上: アプリケーションがテーブルに直接アクセスせず、プロシージャ経由でアクセスすることで権限管理を厳密化

SQLとPL/SQLの違い

SQLは「何をしたいか」を記述する宣言的な言語ですが、PL/SQLは「どのように処理するか」を記述できる手続き型言語です。例えば、SQLでは一度に一つのSELECT文を実行しますが、PL/SQLでは複数のSQL文を組み合わせて、条件によって処理を分岐させたり、繰り返し処理を行ったりできます。

8.2 PL/SQLブロック構文の基本

PL/SQLコードは「ブロック」という単位で記述します。ブロックは以下の3つのセクションで構成されます。

graph TD A[PL/SQLブロック] --> B["DECLAREセクション（変数宣言）"] A --> C["BEGINセクション（実行部）"] A --> D["EXCEPTIONセクション（例外処理）"] style B fill:#e0f2fe,stroke:#0284c7 style C fill:#dcfce7,stroke:#16a34a style D fill:#fef3c7,stroke:#ca8a04

基本的なPL/SQLブロックの構造

基本的なPL/SQLブロック 
-- 匿名ブロック（名前のないPL/SQLブロック）
DECLARE
    -- 変数宣言セクション（省略可能）
    v_message VARCHAR2(100);
    v_counter NUMBER := 0;
BEGIN
    -- 実行セクション（必須）
    v_message := 'Hello, PL/SQL!';
    v_counter := v_counter + 1;
    DBMS_OUTPUT.PUT_LINE(v_message);
    DBMS_OUTPUT.PUT_LINE('カウンター: ' || v_counter);
EXCEPTION
    -- 例外処理セクション（省略可能）
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('エラーが発生しました');
END;
/

重要ポイント

  • • BEGINセクションは必須、DECLAREとEXCEPTIONは省略可能
  • • 各SQL文の末尾にはセミコロン（;）を付ける
  • • ブロック全体の末尾に「END;」を記述し、「/」で実行
  • • 変数名の先頭に「v_」を付けるのが慣例（テーブルカラムと区別するため）

8.3 変数宣言とデータ型

PL/SQLでは、DECLAREセクションで変数を宣言します。変数にはデータ型を指定し、初期値を設定することもできます。

主なデータ型

データ型 説明 例
VARCHAR2(n) 可変長文字列（最大n文字） VARCHAR2(100)
NUMBER 数値型 NUMBER(10,2)
DATE 日付型 DATE
BOOLEAN 真偽値（TRUE/FALSE/NULL） BOOLEAN
%TYPE 既存カラムと同じ型 employees.name%TYPE
%ROWTYPE テーブル行全体の型 employees%ROWTYPE

変数宣言の例

変数宣言のサンプル 
DECLARE
    -- 基本的な変数宣言
    v_name VARCHAR2(50);
    v_age NUMBER := 25;  -- 初期値を設定
    v_salary NUMBER(10,2) DEFAULT 50000;  -- DEFAULTで初期値
    v_hire_date DATE := SYSDATE;  -- 現在日付を初期値に
    v_is_active BOOLEAN := TRUE;

    -- %TYPEを使った宣言（テーブル定義と同期）
    v_emp_name employees.name%TYPE;

    -- %ROWTYPEを使った宣言（行全体）
    v_emp_record employees%ROWTYPE;

    -- 定数の宣言
    c_tax_rate CONSTANT NUMBER := 0.1;
BEGIN
    DBMS_OUTPUT.PUT_LINE('変数宣言完了');
END;
/

ヒント: %TYPEと%ROWTYPEの使い分け

%TYPEは特定のカラムと同じデータ型を持つ変数を宣言するときに便利です。テーブル定義が変更されても、PL/SQLコードを修正する必要がありません。%ROWTYPEは、SELECT文の結果をまるごと格納したいときに使用します。

8.4 制御構文（IF文、LOOP文）

IF文（条件分岐）

IF文を使用して、条件に応じて処理を分岐させることができます。

IF文のサンプル 
DECLARE
    v_score NUMBER := 85;
    v_grade VARCHAR2(10);
BEGIN
    -- 単純なIF文
    IF v_score >= 80 THEN
        v_grade := '優';
        DBMS_OUTPUT.PUT_LINE('成績: ' || v_grade);
    ELSIF v_score >= 60 THEN
        v_grade := '良';
        DBMS_OUTPUT.PUT_LINE('成績: ' || v_grade);
    ELSE
        v_grade := '可';
        DBMS_OUTPUT.PUT_LINE('成績: ' || v_grade);
    END IF;
END;
/

LOOP文（繰り返し処理）

PL/SQLには3種類のループがあります：基本LOOP、WHILE LOOP、FOR LOOP

LOOP文のサンプル 
DECLARE
    v_counter NUMBER := 1;
BEGIN
    -- 基本LOOP（明示的にEXITで抜ける）
    LOOP
        DBMS_OUTPUT.PUT_LINE('カウント: ' || v_counter);
        v_counter := v_counter + 1;
        EXIT WHEN v_counter > 5;  -- 条件でループを抜ける
    END LOOP;

    -- WHILE LOOP
    v_counter := 1;
    WHILE v_counter <= 5 LOOP
        DBMS_OUTPUT.PUT_LINE('WHILEカウント: ' || v_counter);
        v_counter := v_counter + 1;
    END LOOP;

    -- FOR LOOP（最もシンプル）
    FOR i IN 1..5 LOOP
        DBMS_OUTPUT.PUT_LINE('FORカウント: ' || i);
    END LOOP;
END;
/

8.5 ストアドプロシージャの作成

ストアドプロシージャは、名前を付けて保存し、繰り返し呼び出すことができるPL/SQLブロックです。パラメータを受け取り、データベース操作を行えます。

基本構文

プロシージャ作成の基本構文 
CREATE OR REPLACE PROCEDURE プロシージャ名 (
    パラメータ1 IN データ型,
    パラメータ2 OUT データ型,
    パラメータ3 IN OUT データ型
) IS
    -- 変数宣言
    v_変数名 データ型;
BEGIN
    -- 処理内容
    NULL;
EXCEPTION
    -- 例外処理
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('エラー: ' || SQLERRM);
END;
/

実践例

社員情報を登録するプロシージャ 
-- 社員登録プロシージャの作成
CREATE OR REPLACE PROCEDURE add_employee (
    p_name IN VARCHAR2,
    p_age IN NUMBER,
    p_department IN VARCHAR2,
    p_employee_id OUT NUMBER
) IS
    v_max_id NUMBER;
BEGIN
    -- 最大IDを取得
    SELECT NVL(MAX(employee_id), 0) + 1
    INTO v_max_id
    FROM employees;

    -- 新規社員を挿入
    INSERT INTO employees (employee_id, name, age, department)
    VALUES (v_max_id, p_name, p_age, p_department);

    -- 生成されたIDを返す
    p_employee_id := v_max_id;

    COMMIT;
    DBMS_OUTPUT.PUT_LINE('社員登録完了: ID=' || v_max_id);
EXCEPTION
    WHEN OTHERS THEN
        ROLLBACK;
        DBMS_OUTPUT.PUT_LINE('エラー発生: ' || SQLERRM);
END;
/

プロシージャの呼び出し

プロシージャの実行例 
-- プロシージャを呼び出す
DECLARE
    v_new_id NUMBER;
BEGIN
    add_employee('山田太郎', 30, '営業部', v_new_id);
    DBMS_OUTPUT.PUT_LINE('新規社員ID: ' || v_new_id);
END;
/

パラメータモードの種類

  • IN: 呼び出し元から値を受け取る（読み取り専用）
  • OUT: 呼び出し元に値を返す（書き込み専用）
  • IN OUT: 値を受け取り、変更して返す（読み書き可能）

8.6 ファンクションの作成

ファンクションは、必ず値を返すPL/SQLサブプログラムです。SQL文の中で直接使用できるため、SELECT文やWHERE句の中でも呼び出せます。

プロシージャとファンクションの違い

項目 プロシージャ ファンクション
戻り値 なし（OUTパラメータで返す） 必ず1つの値を返す
SQL文内での使用 使用不可 使用可能
主な用途 データ更新、複雑な処理 計算、変換、判定

ファンクションの作成例

税込価格を計算するファンクション 
-- 税込価格を計算するファンクション
CREATE OR REPLACE FUNCTION calc_tax_included (
    p_price IN NUMBER,
    p_tax_rate IN NUMBER DEFAULT 0.1
) RETURN NUMBER IS
    v_tax_included NUMBER;
BEGIN
    -- 税込価格を計算
    v_tax_included := p_price * (1 + p_tax_rate);
    RETURN v_tax_included;
EXCEPTION
    WHEN OTHERS THEN
        RETURN NULL;  -- エラー時はNULLを返す
END;
/

ファンクションの使用例

SQL文内でファンクションを使用 
-- SQL文の中で直接呼び出せる
SELECT
    product_name,
    price,
    calc_tax_included(price) AS tax_included_price,
    calc_tax_included(price, 0.08) AS reduced_tax_price
FROM products;

-- PL/SQLブロック内での使用
DECLARE
    v_result NUMBER;
BEGIN
    v_result := calc_tax_included(1000);
    DBMS_OUTPUT.PUT_LINE('税込価格: ' || v_result || '円');
END;
/

8.7 エラーハンドリング（例外処理）

例外処理を使うことで、実行時エラーが発生してもプログラムを適切に終了させたり、エラー情報をログに記録したりできます。

主な定義済み例外

例外名 発生条件
NO_DATA_FOUND SELECT INTOで行が見つからない
TOO_MANY_ROWS SELECT INTOで複数行が返される
ZERO_DIVIDE ゼロで除算
DUP_VAL_ON_INDEX 一意制約違反
OTHERS すべての例外（キャッチオール）

例外処理の記述例

例外処理のサンプル 
DECLARE
    v_salary NUMBER;
    v_name VARCHAR2(100);
BEGIN
    -- 社員情報を取得
    SELECT name, salary
    INTO v_name, v_salary
    FROM employees
    WHERE employee_id = 999;  -- 存在しないID

    DBMS_OUTPUT.PUT_LINE(v_name || 'の給与: ' || v_salary);
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('該当する社員が見つかりません');
    WHEN TOO_MANY_ROWS THEN
        DBMS_OUTPUT.PUT_LINE('複数の社員がヒットしました');
    WHEN OTHERS THEN
        -- SQLERRMで詳細なエラーメッセージを取得
        DBMS_OUTPUT.PUT_LINE('予期しないエラー: ' || SQLERRM);
END;
/

実習: エラーハンドリング付きプロシージャ

社員の給与を更新するプロシージャを作成し、適切な例外処理を実装してみましょう。

給与更新プロシージャ 
CREATE OR REPLACE PROCEDURE update_salary (
    p_employee_id IN NUMBER,
    p_new_salary IN NUMBER
) IS
    v_old_salary NUMBER;
BEGIN
    -- 現在の給与を取得
    SELECT salary INTO v_old_salary
    FROM employees
    WHERE employee_id = p_employee_id;

    -- 給与を更新
    UPDATE employees
    SET salary = p_new_salary
    WHERE employee_id = p_employee_id;

    COMMIT;
    DBMS_OUTPUT.PUT_LINE('給与更新完了: ' || v_old_salary || '円 → ' || p_new_salary || '円');
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('エラー: 社員ID ' || p_employee_id || ' は存在しません');
    WHEN OTHERS THEN
        ROLLBACK;
        DBMS_OUTPUT.PUT_LINE('予期しないエラー: ' || SQLERRM);
END;
/

トラブルシューティング

Q. DBMS_OUTPUT.PUT_LINEの結果が表示されない

SQL Developerでは、実行前に「DBMS出力」ウィンドウを開き、接続を有効化する必要があります。または、スクリプト実行前にSET SERVEROUTPUT ONを実行してください。

Q. プロシージャのコンパイルエラーが出る

SHOW ERRORSコマンドで詳細なエラー情報を確認できます。セミコロンの位置、スペルミス、データ型の不一致などを確認しましょう。

Q. OUTパラメータの値が取得できない

OUTパラメータを受け取るには、呼び出し側で変数を宣言してから渡す必要があります。匿名ブロック内で変数を宣言し、プロシージャ呼び出し後にその変数を参照してください。

理解度確認クイズ

  1. Q1. PL/SQLブロックで必須のセクションはどれですか？

    回答を表示

    回答: BEGINセクションが必須です。DECLAREセクション（変数宣言）とEXCEPTIONセクション（例外処理）は省略可能です。ただし、変数を使う場合はDECLAREセクションが、エラーハンドリングを行う場合はEXCEPTIONセクションが必要になります。

  2. Q2. ストアドプロシージャとファンクションの最も大きな違いは何ですか？

    回答を表示

    回答: ファンクションは必ず1つの値を返しますが、プロシージャは戻り値を持ちません（OUTパラメータで値を返すことは可能）。また、ファンクションはSQL文の中で直接使用できますが、プロシージャはSQL文内では使用できません。計算や判定にはファンクション、データ更新や複雑な処理にはプロシージャを使うのが一般的です。

  3. Q3. %TYPEを使った変数宣言のメリットは何ですか？

    回答を表示

    回答: %TYPEを使うと、テーブルカラムと同じデータ型を持つ変数を宣言できます。テーブル定義が変更されても（例えば、VARCHAR2(50)からVARCHAR2(100)に変更されても）、PL/SQLコードを修正する必要がありません。これにより、保守性が向上し、データ型の不一致によるエラーを防ぐことができます。
