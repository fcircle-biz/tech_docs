SQL Developerの導入と接続設定

Oracle SQL Developerのインストールから、データベースへの接続設定までを実践的に学習します。接続エラーの対処法も詳しく解説します。

この章で学ぶこと

  • SQL Developerとは何か、その役割と特徴を理解する
  • JDKのインストールとSQL Developerのセットアップ方法を習得する
  • データベースへの接続設定（Windows/Mac両環境対応）ができるようになる
  • 接続エラーが発生した場合の原因切り分けと対処法を知る

4.1 SQL Developerとは

Oracle SQL Developer（オラクル エスキューエル デベロッパー）は、Oracle Corporationが無償で提供しているデータベース開発・管理ツールです。グラフィカルなユーザーインターフェース（GUI）を通じて、Oracleデータベースへの接続、SQL文の実行、テーブルの作成・編集などをマウス操作で直感的に行うことができます。

SQL Developerの主な機能

データベース接続管理

複数のデータベースへの接続情報を保存し、簡単に切り替えて操作できます。

SQLエディタ

構文ハイライトや自動補完機能を備えたエディタでSQL文を作成・実行できます。

オブジェクト管理

テーブル、ビュー、インデックスなどのデータベースオブジェクトをGUIで作成・編集できます。

データのエクスポート/インポート

データをCSV、Excel、SQL形式などで簡単に出力・取り込みできます。

コマンドライン（SQL*Plus）との違い

OracleにはSQL*Plusというコマンドラインツールもありますが、SQL Developerはグラフィカルな画面で直感的に操作できるため、初心者にも扱いやすくなっています。一方、サーバー上での自動化処理や軽量な操作ではSQL*Plusが使われることもあります。

4.2 JDK（Java Development Kit）のインストール

SQL DeveloperはJava言語で開発されているため、実行にはJDK（Java Development Kit）が必要です。まずJDKをインストールします。

バージョンによってはJDKが不要な場合があります

SQL Developerのバージョンによっては、JDKが同梱されている場合があります（「JDK同梱版」）。その場合はこの手順をスキップできます。ダウンロードページで「JDK同梱版」があればそちらを選択してください。

JDKのダウンロード手順

  1. 1
    Oracle JDKダウンロードページにアクセス

    ブラウザで以下のURLにアクセスします：
    https://www.oracle.com/java/technologies/downloads/

  2. 2
    Java 11以上のバージョンを選択

    Java 11、Java 17、Java 21のいずれかを選択します（推奨: Java 17）

  3. 3
    OSに応じたインストーラーをダウンロード

    Windows: jdk-17_windows-x64_bin.exe
    macOS: jdk-17_macos-x64_bin.dmg（Intel Mac）または
    jdk-17_macos-aarch64_bin.dmg（Apple Silicon Mac）

  4. 4
    ダウンロードしたインストーラーを実行

    ダウンロードしたファイルをダブルクリックし、インストールウィザードに従って進めます。

  5. 5
    インストールの完了を確認

    コマンドプロンプト（Windows）またはターミナル（Mac）で以下のコマンドを実行してバージョンが表示されればOKです。

動作確認

コマンドプロンプトまたはターミナルで以下を実行します：

Bash 
java -version

以下のような出力が表示されれば成功です：
java version "17.0.9" 2023-10-17 LTS

4.3 SQL Developerのダウンロードとインストール

ダウンロード手順

  1. 1
    SQL Developerダウンロードページにアクセス

    https://www.oracle.com/database/sqldeveloper/technologies/download/

  2. 2
    OSに応じたバージョンを選択

    - Windows: 「Windows 64-bit with JDK included」を選択（JDK同梱版推奨）
    - Mac: 「macOS」を選択（Intel版またはApple Silicon版）

  3. 3
    Oracle ライセンスに同意してダウンロード

    「I accept the license agreement」にチェックを入れてDownloadボタンをクリック

  4. 4
    Oracleアカウントでサインイン

    アカウントがない場合は無料で作成できます

インストール手順（Windows）

  1. 1
    ダウンロードしたZIPファイルを任意のフォルダに解凍します
  2. 2
    解凍したフォルダ内の sqldeveloper.exe をダブルクリックして起動
  3. 3
    初回起動時は少し時間がかかります（1〜2分程度）

インストール手順（Mac）

  1. 1
    ダウンロードしたDMGファイルをダブルクリック
  2. 2
    「SQL Developer」アプリケーションをApplicationsフォルダにドラッグ&ドロップ
  3. 3
    Launchpadまたはアプリケーションフォルダから「SQL Developer」を起動
  4. 4
    初回起動時にセキュリティ警告が表示された場合は、システム環境設定の「セキュリティとプライバシー」で「開く」を許可

ショートカット作成のヒント（Windows）

sqldeveloper.exeを右クリックして「ショートカットの作成」を選ぶと、デスクトップやタスクバーに配置して簡単に起動できるようになります。

4.4 データベースへの接続設定（Windows環境）

第2章でインストールしたOracle Database FreeにSQL Developerから接続します。

実習: 接続設定を作成する

  1. 1. SQL Developerを起動

    sqldeveloper.exeをダブルクリックして起動します

  2. 2. 新しい接続の作成

    画面左上の「接続」パネルで、緑色の「+」アイコンをクリック → 「新規接続」を選択

  3. 3. 接続情報を入力

    以下の情報を入力します：

    • 接続名: LOCAL_FREE（任意の名前）
    • ユーザー名: system
    • パスワード: インストール時に設定したパスワード
    • ホスト名: localhost
    • ポート: 1521
    • SID: FREE
  4. 4. 接続のテスト

    「テスト」ボタンをクリックして、「ステータス: 成功」と表示されることを確認

  5. 5. 保存して接続

    「保存」ボタンをクリックして設定を保存し、「接続」ボタンで接続を開始

4.5 データベースへの接続設定（Mac/Docker環境）

第3章でDocker上に構築したOracle DatabaseにSQL Developerから接続します。基本的な手順はWindows環境と同じですが、接続情報が若干異なります。

実習: Docker環境への接続設定

  1. 1. Dockerコンテナが起動していることを確認
    Bash 
    docker ps

    oracleコンテナが「Up」状態であることを確認

  2. 2. SQL Developerで新規接続を作成

    接続情報を以下のように入力します：

    • 接続名: DOCKER_ORACLE（任意の名前）
    • ユーザー名: system
    • パスワード: コンテナ起動時に設定したパスワード
    • ホスト名: localhost
    • ポート: 1521
    • サービス名: FREEPDB1
  3. 3. テストと接続

    「テスト」ボタンで接続確認後、「保存」→「接続」の順にクリック

重要ポイント

Docker環境では、接続タイプで「SID」ではなく「サービス名」を選び、FREEPDB1 を入力します。これはDockerコンテナのデフォルト設定です。

4.6 接続エラーのトラブルシューティング

よくあるエラーと対処法

エラー: IO Error: The Network Adapter could not establish the connection

原因:

  • データベースサービスが起動していない
  • ホスト名やポート番号が間違っている
  • ファイアウォールで通信がブロックされている

対処法:

  • Windows: サービス一覧で「OracleServiceFREE」が起動しているか確認
  • Mac/Docker: docker ps でコンテナが起動しているか確認
  • ホスト名を 127.0.0.1 に変更してみる

エラー: ORA-01017: invalid username/password; logon denied

原因:

  • ユーザー名またはパスワードが間違っている

対処法:

  • パスワードを再入力（大文字小文字を区別します）
  • ユーザー名が正しいか確認（通常は system）

エラー: ORA-12505: TNS:listener does not currently know of SID given

原因:

  • SIDまたはサービス名が間違っている

対処法:

  • Windows環境: SIDに FREE を指定
  • Mac/Docker環境: 「サービス名」に FREEPDB1 を指定

エラー: java.lang.ClassNotFoundException: oracle.jdbc.OracleDriver

原因:

  • JDBCドライバが見つからない

対処法:

  • SQL Developerを再起動してみる
  • JDK同梱版のSQL Developerを使用する

4.7 接続確認とテスト

接続が成功したら、簡単なSQL文を実行してデータベースが正常に動作していることを確認しましょう。

動作確認: 簡単なSQLを実行

  1. 1. SQLワークシートを開く

    接続を右クリック → 「SQLワークシートを開く」を選択

  2. 2. 現在時刻を取得するSQLを実行
    SQL 
    -- 現在時刻を取得
SELECT SYSDATE FROM DUAL;

    F5キーまたは画面上部の▶実行ボタンをクリック

  3. 3. データベースバージョンを確認
    SQL 
    -- Oracleバージョン確認
SELECT * FROM V$VERSION;
  4. 4. 結果が表示されることを確認

    画面下部の「結果」タブに現在時刻やバージョン情報が表示されればOKです

理解度確認クイズ

  1. Q1. SQL Developerを使用するために必要な前提ソフトウェアは何ですか？

    回答を表示

    回答: JDK（Java Development Kit）が必要です。ただし、SQL DeveloperにJDKが同梱されているバージョン（JDK同梱版）を選択した場合は、別途JDKをインストールする必要はありません。JDK同梱版がダウンロードできる場合は、そちらを選択すると手順が簡単になります。

  2. Q2. Windows環境とMac/Docker環境で、データベース接続設定の違いは何ですか？

    回答を表示

    回答: Windows環境では接続タイプに「SID」を選択し、値に「FREE」を指定します。一方、Mac/Docker環境では接続タイプに「サービス名」を選択し、値に「FREEPDB1」を指定します。これはDockerコンテナのデフォルト設定によるものです。その他の設定（ホスト名、ポート、ユーザー名）は基本的に同じです。

  3. Q3. 「IO Error: The Network Adapter could not establish the connection」というエラーが表示されました。最初に確認すべきことは何ですか？

    回答を表示

    回答: まず、Oracleデータベースサービスが起動しているかを確認します。Windows環境では、サービス一覧で「OracleServiceFREE」が起動していることを確認します。Mac/Docker環境では、ターミナルで「docker ps」コマンドを実行し、oracleコンテナが「Up」状態であることを確認します。サービスやコンテナが停止している場合は、起動してから再度接続テストを行います。
