SQL Developerの基本操作

SQL Developerの画面構成と基本的な操作方法を習得します。SQLの実行、結果の確認、テーブル情報の閲覧など、日常的に使用する機能を網羅します。

この章で学ぶこと

  • SQL Developerの画面構成を理解する
  • SQLワークシートの基本操作を習得する
  • SQLの実行と結果確認方法を学ぶ
  • テーブル情報の閲覧方法を習得する

5.1 SQL Developerの起動と画面構成

SQL Developerを起動すると、いくつかのパネルで構成されたメイン画面が表示されます。まず、この画面構成を理解することが重要です。

画面の主要構成

flowchart TD A[SQL Developer メイン画面] --> B[接続ナビゲータ
左側パネル] A --> C[ワークスペース
中央エリア] A --> D[プロパティパネル
下部または右側] B --> B1[データベース接続一覧] B --> B2[スキーマオブジェクト
テーブル/ビュー等] C --> C1[SQLワークシート] C --> C2[データグリッド] C --> C3[スクリプト出力] D --> D1[オブジェクトプロパティ] D --> D2[DDL/制約情報]

各パネルの役割

パネル名 役割 主な機能
接続ナビゲータ データベース接続の管理 接続の追加、切断、スキーマオブジェクトの参照
SQLワークシート SQL文の編集と実行 SQLコードの入力、実行、履歴管理
データグリッド クエリ結果の表示 SELECTの実行結果、データの編集
スクリプト出力 実行結果のメッセージ表示 エラーメッセージ、実行ログ
プロパティパネル オブジェクトの詳細情報表示 テーブル定義、制約、インデックス情報

パネルのカスタマイズ

各パネルは自由にサイズ変更や移動ができます。境界線をドラッグしてサイズを調整したり、パネルのタブをドラッグして位置を変更することができます。自分の作業スタイルに合わせてカスタマイズしましょう。

5.2 接続ナビゲータの使い方

接続ナビゲータは、SQL Developerの左側に配置されているパネルで、データベース接続の管理とスキーマオブジェクトの参照を行います。

接続の展開とオブジェクトの参照

接続名の横にある「▶」アイコンをクリックすると、接続が展開され、スキーマオブジェクトのカテゴリが表示されます。

flowchart TD A["Oracle接続名
SYSTEM"] --> B[テーブル] A --> C[ビュー] A --> D[索引] A --> E[シーケンス] A --> F["プロシージャ/ファンクション"] B --> B1["EMPLOYEES
社員テーブル"] B --> B2["DEPARTMENTS
部署テーブル"] B1 --> B1A[列: カラム定義] B1 --> B1B[制約: PK/FK] B1 --> B1C[索引: INDEX定義] B1 --> B1D["データ: 実データ閲覧"]

主なスキーマオブジェクトカテゴリ

  • テーブル: データを格納するテーブルの一覧。展開すると列、制約、索引などが確認できる
  • ビュー: 仮想テーブル（SELECT文を保存したもの）の一覧
  • シーケンス: 自動採番用のシーケンスオブジェクト
  • プロシージャ/ファンクション: PL/SQLで作成されたストアドプロシージャや関数
  • パッケージ: 関連するプロシージャやファンクションをまとめたもの

ヒント

テーブル名を右クリックすると、コンテキストメニューから「データの問合せ」を選択でき、簡単にSELECT文を実行できます。また、「編集」を選択すると、テーブル定義を変更することも可能です。

5.3 SQLワークシートの基本操作

SQLワークシートは、SQL文を入力・実行するためのエディタです。新しいワークシートを開くには、ツールバーの「SQLワークシート」アイコンをクリックするか、接続名を右クリックして「SQLワークシートを開く」を選択します。

ワークシートの基本操作

基本的な操作手順

  1. 1
    ワークシートを開く

    ツールバーから「SQLワークシート」アイコンをクリック

  2. 2
    接続を選択

    ワークシート上部のドロップダウンから接続を選択

  3. 3
    SQL文を入力

    エディタ部分にSQL文を入力（複数のSQL文も可能）

  4. 4
    実行

    F5キーまたは実行ボタンをクリックしてSQL文を実行

コード補完機能

SQL Developerは強力なコード補完機能を備えています。テーブル名やカラム名の一部を入力して「Ctrl + Space」を押すと、候補リストが表示されます。

実習 5-1: SQLワークシートで簡単なSQLを実行

SQLワークシートを使って、簡単なSQLを実行してみましょう。

手順

  1. 1 SQL Developerで新しいSQLワークシートを開く
  2. 2 接続ドロップダウンから接続を選択
  3. 3 以下のSQL文を入力して実行してみる
example.sql 
-- 現在の日時を取得
SELECT SYSDATE FROM DUAL;

-- 接続中のユーザー名を取得
SELECT USER FROM DUAL;

5.4 SQLの実行方法

SQL Developerには、SQL文を実行する方法が複数あります。それぞれの違いを理解して、適切な方法を選択しましょう。

実行方法の種類

実行方法 ショートカット 説明 用途
文の実行 F9 カーソル位置の1つのSQL文を実行 単一のSELECT文を実行
スクリプトの実行 F5 ワークシート内の全てのSQL文を順次実行 複数のSQL文をまとめて実行
選択範囲の実行 範囲選択 + F9 選択した部分のみ実行 一部のSQL文だけ実行したい場合
COMMIT なし 変更を確定 INSERT/UPDATE/DELETE後
ROLLBACK なし 変更を取り消し 間違えた変更をキャンセル

重要ポイント

F9（文の実行）は結果をデータグリッドに表示し、データの編集も可能です。F5（スクリプトの実行）は結果をテキスト形式でスクリプト出力パネルに表示します。SELECT文を1つだけ実行する場合はF9、複数のSQL文やDDL文を実行する場合はF5を使います。

注意

INSERT、UPDATE、DELETEなどのデータ変更文を実行した後は、必ずCOMMITまたはROLLBACKを実行してください。COMMITしないまま接続を閉じると、変更が破棄される可能性があります。また、自動コミットの設定を確認しておきましょう（デフォルトはオフです）。

5.5 テーブル情報の確認

SQL Developerでは、テーブルの構造や制約、索引などの情報を視覚的に確認できます。これにより、SQLを書かなくてもテーブルの詳細を把握できます。

テーブル情報の確認方法

操作手順

  1. 1
    接続ナビゲータでテーブルを展開

    「テーブル」カテゴリを展開し、確認したいテーブルを選択

  2. 2
    テーブルをクリック

    右側のワークスペースにテーブルの詳細情報が表示される

  3. 3
    各タブで情報を確認

    列、データ、制約、索引などのタブを切り替えて詳細を確認

タブ名 表示内容

索引 作成されているインデックスの一覧と対象カラム

5.6 スクリプトの保存と読み込み

作成したSQL文は、ファイルとして保存しておくことができます。これにより、繰り返し使うSQLや重要なクエリを管理しやすくなります。

スクリプトの保存

保存手順

  1. 1
    メニューから「ファイル」→「保存」を選択

    または Ctrl+S（Windows）/ Cmd+S（Mac）

  2. 2
    保存先とファイル名を指定

    拡張子は .sql が推奨

  3. 3
    「保存」ボタンをクリック

    ファイルが保存され、次回から再利用できる

スクリプトの読み込み

読み込み手順

  1. 1
    メニューから「ファイル」→「開く」を選択

    または Ctrl+O（Windows）/ Cmd+O（Mac）

  2. 2
    開きたいSQLファイルを選択

    .sql ファイルを選択

  3. 3
    ファイルの内容が新しいワークシートに読み込まれる

    そのまま実行可能

ヒント

よく使うSQLスクリプトは、プロジェクト別やテーマ別にフォルダ分けして整理しておくと便利です。また、スクリプトの先頭にコメントで作成日や目的を記載しておくと、後から見返したときに分かりやすくなります。

5.7 便利なショートカットキー

SQL Developerには多くのショートカットキーが用意されています。よく使うものを覚えておくと、作業効率が大幅に向上します。

主要ショートカットキー一覧

操作 Windows Mac 説明
文の実行 F9 F9 カーソル位置のSQL文を実行
スクリプトの実行 F5 F5 全てのSQL文を実行
コード補完 Ctrl + Space Ctrl + Space 候補リストを表示
コメント化 Ctrl + / Cmd + / 選択行を -- でコメント化
保存 Ctrl + S Cmd + S スクリプトを保存
開く Ctrl + O Cmd + O スクリプトファイルを開く
新しいワークシート Ctrl + Shift + N Cmd + Shift + N 新しいSQLワークシートを開く
検索 Ctrl + F Cmd + F テキスト検索
置換 Ctrl + H Cmd + H テキスト置換
フォーマット Ctrl + F7 Ctrl + F7 SQL文を整形

重要ポイント

特に覚えておきたいショートカットは、F9（文の実行）、F5（スクリプトの実行）、Ctrl+Space（コード補完）の3つです。これだけでも作業効率が大幅に向上します。その他のショートカットは、必要に応じて徐々に覚えていきましょう。

理解度確認クイズ

  1. Q1. F9とF5の違いは何ですか？

    回答を表示

    回答: F9は「文の実行」で、カーソル位置の1つのSQL文を実行し、結果をデータグリッドに表示します。主にSELECT文で使用します。F5は「スクリプトの実行」で、ワークシート内の全てのSQL文を順次実行し、結果をスクリプト出力パネルにテキスト形式で表示します。複数のSQL文やDDL文を実行する場合に使用します。

  2. Q2. テーブルの列定義を確認するには、どのタブを見れば良いですか？

    回答を表示

    回答: 「列」タブを確認します。接続ナビゲータでテーブルを選択すると、右側のワークスペースに複数のタブが表示されます。「列」タブには、カラム名、データ型、NULL許可、デフォルト値などのカラム定義情報が表示されます。

  3. Q3. INSERT文を実行した後、変更を確定するために実行する必要があるコマンドは何ですか？

    回答を表示

    回答: COMMITコマンドを実行する必要があります。INSERT、UPDATE、DELETEなどのデータ変更文を実行しただけでは、変更は一時的なものです。COMMITを実行することで変更が確定され、データベースに永続的に保存されます。間違えた場合はROLLBACKで変更を取り消すこともできます。
