Windows環境でのOracleインストール

Windows環境でのOracle Database Freeのインストール手順を詳しく解説します。初心者が躓きやすいポイントを丁寧に説明し、確実に環境構築できるよう導きます。

この章で学ぶこと

  • Oracle Database Freeのダウンロード方法を理解する
  • Windows環境でのインストール前の準備と確認事項を学ぶ
  • インストールウィザードを使用してOracleを正しく設定する
  • インストール後の確認方法と典型的なトラブルの対処法を習得する

2.1 インストール前の準備

システム要件の確認

Oracle Database Freeをインストールする前に、お使いのWindowsコンピューターが以下の要件を満たしているか確認しましょう。

項目 要件 推奨
OS Windows 10/11 (64bit) Windows 11
RAM 最低 4GB 8GB以上
ディスク空き容量 10GB以上 20GB以上
管理者権限 必要（インストール時のみ）
インターネット接続 必要（ダウンロード時のみ）

重要ポイント

インストールには管理者権限が必要です。標準ユーザーアカウントでログインしている場合は、管理者権限のあるアカウントに切り替えるか、インストーラーを右クリックして「管理者として実行」を選択してください。

システム情報の確認方法

お使いのWindowsコンピューターのスペックを確認するには、以下の手順を実行します。

確認手順

  1. 1
    設定を開く

    Windowsキー + I を押して「設定」を開きます

  2. 2
    システム情報を表示

    「システム」→「バージョン情報」を選択

  3. 3
    スペックを確認

    実装RAM、システムの種類（64bitか確認）、ディスク容量を確認

2.2 Oracle Database Freeのダウンロード

Oracle Database Freeは、Oracle公式サイトから無料でダウンロードできます。

ダウンロード手順

  1. 1
    Oracle公式サイトにアクセス

    ブラウザで以下のURLを開きます

    https://www.oracle.com/database/technologies/xe-downloads.html

  2. 2
    Windows版を選択

    「Oracle Database 23ai Free - Download for Windows x64」を探してクリック

  3. 3
    ライセンス同意とログイン

    ライセンスに同意し、Oracleアカウントでログイン（未登録の場合は無料で作成）

  4. 4
    ダウンロード開始

    インストーラー（約2.5GB）のダウンロードが開始されます

ヒント

Oracleアカウントは無料で作成できます。登録時には正確なメールアドレスを入力し、確認メールを受信してアクティベーションを完了してください。このアカウントは、今後のサポートフォーラムやドキュメントへのアクセスにも利用できます。

2.3 インストールの実行

ダウンロードしたインストーラーを使用してOracle Database Freeをインストールします。インストールには約20〜30分かかります。

インストール手順

実習 2-1: Oracle Database Freeのインストール

インストールウィザードを使用して、Oracle Database Freeをシステムにインストールします。

手順

  1. 1 ダウンロードしたインストーラー（.exeファイル）を右クリックし、「管理者として実行」を選択
  2. 2 Oracle Database Setup ウィザードが起動したら「次へ」をクリック
  3. 3 インストール先フォルダを確認（デフォルト: C:\app\[ユーザー名]\product\23.0.0\dbhomeFree）。特別な理由がなければデフォルトのまま「次へ」
  4. 4 パスワード設定画面で、SYSおよびSYSTEM管理者アカウントのパスワードを設定（英大文字・小文字・数字を含む8文字以上）。このパスワードは必ず記録してください
  5. 5 コンテナデータベース（Container Database）のオプションはデフォルトのまま「次へ」
  6. 6 前提条件チェックが実行されます。すべてグリーンになったら「インストール」をクリック
  7. 7 インストールが開始されます（約20〜30分）。進行状況バーが100%になるまで待ちます
  8. 8 データベース作成が自動的に実行されます（さらに10〜15分）
  9. 9 「完了」画面が表示されたら「閉じる」をクリック

パスワードについての重要な注意

設定したパスワードは、SYSユーザー（最高管理者）とSYSTEMユーザー（管理者）の両方に適用されます。このパスワードを忘れると、データベースの管理操作ができなくなります。必ず安全な場所にメモして保管してください。推奨形式: Oracle123! のような英大文字・小文字・数字・記号を含む8文字以上

2.4 インストール後の確認

インストールが完了したら、Oracle Databaseが正常に起動しているか確認しましょう。

サービスの確認

動作確認: Oracleサービスの状態確認

Windowsサービスマネージャーを使用して、Oracleのサービスが正常に起動しているか確認します。

確認手順

  1. 1 Windowsキー + R を押して「ファイル名を指定して実行」を開く
  2. 2 services.msc と入力してEnterキーを押す
  3. 3 サービス一覧から「OracleServiceFREE」と「OracleOraDB23ai_home1TNSListener」を探す
  4. 4 両方のサービスの「状態」列が「実行中」となっていることを確認

期待される結果

  • OracleServiceFREE: 状態「実行中」、スタートアップの種類「自動」
  • OracleOraDB23ai_home1TNSListener: 状態「実行中」、スタートアップの種類「自動」

SQL*Plusでの接続確認

コマンドラインツールであるSQL*Plusを使用して、データベースに接続できるか確認します。

動作確認: SQL*Plusでの接続テスト

SQL*Plusを使用してデータベースに接続し、正常に動作していることを確認します。

確認手順

  1. 1 コマンドプロンプトを管理者として開く（検索で「cmd」と入力し、右クリックして「管理者として実行」）
  2. 2 以下のコマンドを入力してSQL*Plusを起動
コマンドプロンプト 
sqlplus sys as sysdba
  1. 3 パスワード入力を求められたら、インストール時に設定したパスワードを入力
  2. 4 接続成功後、以下のSQLを実行してデータベース名を確認
SQL*Plus 
-- データベース名を確認
SELECT name, open_mode FROM v$database;

期待される結果 

NAME      OPEN_MODE
--------- ----------
FREE      READ WRITE

この結果が表示されれば、データベースは正常に動作しています。

SQL*Plusを終了するには以下のコマンドを実行します：

SQL*Plus 
EXIT;

2.5 トラブルシューティング

インストール中や接続時によくあるトラブルとその対処法をまとめました。

トラブルシューティング

インストール中に「前提条件チェックに失敗しました」と表示される

原因: システム要件を満たしていない、または他のソフトウェアと競合している可能性があります。

解決方法:

  • RAMとディスク容量が要件を満たしているか再確認
  • 他のOracleソフトウェアがインストールされている場合はアンインストール
  • ウイルス対策ソフトを一時的に無効化してから再試行
  • Windowsの更新プログラムをすべて適用してから再実行
サービスが「開始」状態にならない

原因: ポート番号の競合、またはインストールが不完全な可能性があります。

解決方法:

  • コンピューターを再起動してからサービスを手動で開始
  • 他のアプリケーションがポート1521を使用していないか確認
  • イベントビューアーでエラーログを確認（Windowsキー + X → イベントビューアー）
  • 問題が解決しない場合は、Oracleを完全にアンインストールして再インストール
SQL*Plusで「ORA-12560: TNS:protocol adapter error」が発生する

原因: Oracleリスナーサービスが起動していない、または環境変数が正しく設定されていません。

解決方法:

  • サービス一覧で「OracleOraDB23ai_home1TNSListener」が実行中か確認し、停止している場合は開始
  • 環境変数ORACLE_SIDが「FREE」に設定されているか確認
  • コマンドプロンプトで set ORACLE_SID=FREE を実行してから再度SQL*Plusを起動
パスワードを忘れてしまった

原因: インストール時に設定したSYS/SYSTEMアカウントのパスワードを記録し忘れた場合です。

解決方法:

  • SYSユーザーのパスワードはコマンドラインから変更できます
  • 管理者権限でコマンドプロンプトを開き、以下を実行：
sqlplus / as sysdba
ALTER USER sys IDENTIFIED BY 新しいパスワード;

新しいパスワードは必ず安全な場所に記録してください。

インストール中にディスク容量不足エラーが出る

原因: Cドライブ（または指定したドライブ）の空き容量が不足しています。

解決方法:

  • 不要なファイルやアプリケーションを削除して20GB以上の空き容量を確保
  • ディスククリーンアップツールを実行（検索で「ディスククリーンアップ」と入力）
  • Tempフォルダ（C:\Users\[ユーザー名]\AppData\Local\Temp）内の不要なファイルを削除
  • 容量に余裕のある別のドライブにインストール先を変更

理解度確認クイズ

  1. Q1. Oracle Database FreeをWindowsにインストールするために必要な最低限のシステム要件として正しいものはどれですか？

    回答を表示

    回答: 最低限の要件は、Windows 10/11（64bit）、RAM 4GB以上、ディスク空き容量10GB以上です。ただし、推奨はRAM 8GB以上、ディスク空き容量20GB以上です。また、インストールには管理者権限が必要です。

  2. Q2. インストール時に設定するパスワードの要件について説明してください。

    回答を表示

    回答: パスワードは英大文字・小文字・数字を含む8文字以上で設定する必要があります。このパスワードはSYSユーザー（最高管理者）とSYSTEMユーザー（管理者）の両方に適用されます。パスワードを忘れるとデータベース管理ができなくなるため、必ず安全な場所に記録してください。

  3. Q3. インストール後に確認すべき2つのWindowsサービス名は何ですか？また、これらのサービスはどのような状態であるべきですか？

    回答を表示

    回答: 確認すべきサービスは「OracleServiceFREE」（データベース本体）と「OracleOraDB23ai_home1TNSListener」（ネットワークリスナー）の2つです。両方とも「状態」が「実行中」、「スタートアップの種類」が「自動」になっている必要があります。これらが実行中でない場合、データベースへの接続ができません。
