1.1 Oracleデータベースの概要

Oracle Database（通称Oracle）は、Oracle Corporationが開発・販売しているリレーショナルデータベース管理システム（RDBMS）です。 1979年に初版がリリースされ、40年以上にわたって企業の基幹システムを支える世界有数のデータベースシステムとして発展してきました。

1.2 Oracleの特徴

  • 高い可用性: RAC（Real Application Clusters）による冗長化
  • スケーラビリティ: 大規模データ処理に対応
  • セキュリティ: 企業レベルのセキュリティ機能
  • 標準SQL準拠: ANSI/ISO SQL標準に準拠
  • 豊富な機能: PL/SQL、XML、JSON、分析関数など
  • エンタープライズサポート: 24時間365日のサポート体制

1.3 他のデータベースとの比較

データベース 用途 特徴 ライセンス
Oracle 大規模企業システム 高機能・高性能・高可用性 商用
MySQL Webアプリケーション 軽量・高速 オープンソース/商用
PostgreSQL 汎用的な用途 高機能・標準準拠 オープンソース
SQL Server Microsoft環境 Windowsとの親和性 商用

1.4 Oracleのアーキテクチャ

Oracleデータベースは以下の主要コンポーネントから構成されています：

インスタンス（Instance）

  • SGA（System Global Area）: 共有メモリ領域
  • バックグラウンドプロセス: データベース処理を行うプロセス群

データベース（Database）

  • データファイル: 実際のデータを格納
  • 制御ファイル: データベースの構造情報
  • REDOログファイル: トランザクションログ

1.5 Oracle Databaseのエディション

エディション 特徴 用途
Enterprise Edition 全機能・無制限 大規模企業システム
Standard Edition 基本機能・制限あり 中小企業システム
Express Edition (XE) 無料・機能制限 開発・学習・小規模利用
注意事項

本コースでは学習目的でOracle Database Express Edition (XE) をDockerで使用します。 商用利用の場合は適切なライセンスの確認が必要です。

1.6 学習環境の準備

Oracle学習を始める前に、以下の環境を準備しましょう：

必要なソフトウェア

  1. Docker Desktop: Oracle Databaseをコンテナで実行
  2. SQL Developer: Oracleの公式開発ツール
  3. Webブラウザ: Oracle Enterprise Managerアクセス用

システム要件

  • メモリ: 8GB以上（推奨：16GB）
  • ディスク容量: 20GB以上の空き容量
  • OS: Windows 10/11、macOS、Linux
実習 1-1: Docker Desktopの準備確認

以下のコマンドでDocker環境を確認してみましょう：

# Docker バージョン確認
docker --version

# Docker が動作しているか確認
docker ps

# 利用可能なメモリを確認
docker system info | grep "Total Memory"
