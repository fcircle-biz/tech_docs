Windows環境でのOracleインストール（Rancher Desktop利用）

Windows環境でのRancher Desktop導入とOracleコンテナの構築手順を解説します。Docker未経験者でも理解できるよう、基本的なコマンドから説明します。

この章で学ぶこと

  • Rancher Desktopの概要と必要性を理解する
  • Rancher DesktopをWindowsにインストールする方法を習得する
  • Oracle Databaseコンテナイメージの取得と起動方法を学ぶ
  • Dockerコンテナの基本的な操作（起動・停止・確認）ができる

3.1 Rancher Desktopとは

Rancher Desktop（ランチャー デスクトップ）は、WindowsやmacOS上でDockerコンテナを簡単に動かすためのオープンソースアプリケーションです。Docker Desktopの代替として利用でき、完全無料で商用利用も可能です。第2章ではOracle Database FreeをWindowsにネイティブインストールする方法を学びましたが、コンテナを使えばより簡単に環境構築ができ、環境の分離も可能になります。

Dockerコンテナとは

コンテナは「アプリケーションとその実行環境を一つのパッケージにまとめた仮想環境」です。仮想マシン（VM）と似ていますが、より軽量で起動が速いのが特徴です。

重要ポイント

Rancher Desktopを使うことで、Windows上でLinux版のOracle Databaseを動かすことができます。また、コンテナは簡単に削除・再作成できるため、環境を壊してしまっても安心です。Rancher Desktopはオープンソースで完全無料なので、ライセンスを気にせず利用できます。

3.2 Rancher Desktopのインストール

必要なシステム要件

  • Windows 10/11（64ビット版、Pro/Enterprise/Education推奨）
  • WSL 2（Windows Subsystem for Linux 2）が有効であること
  • 8GB以上のRAM（推奨：16GB）
  • 30GB以上の空きディスク容量
  • BIOSで仮想化機能（Intel VT-x/AMD-V）が有効であること

実習 3-1: Rancher Desktopのインストール

Rancher DesktopをWindowsにインストールします。

手順

  1. 1 公式サイトにアクセス: https://rancherdesktop.io/
  2. 2 「Download for Windows」ボタンをクリック
  3. 3 ダウンロードした「Rancher.Desktop.Setup.x.x.x.msi」をダブルクリックして実行
  4. 4 インストールウィザードに従い「Next」→「Install」をクリック
  5. 5 インストールが完了したら「Finish」をクリック
  6. 6 スタートメニューから「Rancher Desktop」を起動
  7. 7 初回起動時の設定画面で「Container Engine」に「dockerd（moby）」を選択
  8. 8 「Accept」をクリックして設定を完了

初回起動時の注意

Rancher Desktopの初回起動には数分かかります。タスクトレイにRancher Desktopアイコンが表示され、緑色の状態になるまで待ちましょう。WSL 2のインストールを促すメッセージが表示された場合は、指示に従ってインストールしてください。

Container Engineの選択について

Rancher Desktopでは「dockerd（moby）」または「containerd」を選択できます。本ガイドではdockerコマンドをそのまま使用するため、「dockerd（moby）」を選択してください。containerdを選択した場合はnerdctlコマンドを使用することになります。

3.3 Dockerの動作確認

Rancher Desktopが正常にインストールされたか、コマンドを実行して確認します。

動作確認

コマンドプロンプトまたはPowerShellを開き、以下のコマンドを実行します。

コマンドプロンプト / PowerShell 
docker --version

期待される結果 

Docker version 24.0.6, build ed223bc

※バージョン番号は異なる場合があります

ヒント

コマンドプロンプトまたはPowerShellは、スタートメニューで「cmd」または「powershell」と検索して起動できます。または、Windowsキー + R を押して「cmd」と入力しても開けます。

3.4 Oracle Databaseコンテナイメージの取得

Oracle公式が提供しているDockerイメージ（コンテナの設計図）を取得します。

Dockerイメージとは

Dockerイメージは、コンテナを作成するための設計図（テンプレート）です。Oracle公式のイメージには、Linux版Oracle Databaseがあらかじめ設定された状態で含まれています。

イメージ取得コマンド

以下のコマンドを実行して、Oracle Database Freeのイメージをダウンロードします（初回は10〜15分程度かかります）。

コマンドプロンプト / PowerShell 
# Oracle Database Free 23ai イメージを取得
docker pull container-registry.oracle.com/database/free:latest

ダウンロード時間について

イメージサイズは約4〜5GBあります。インターネット接続速度によっては、ダウンロードに15〜30分程度かかる場合があります。完了するまで気長に待ちましょう。

イメージ取得の確認

ダウンロードが完了したら、以下のコマンドでイメージが取得されたか確認します。

コマンドプロンプト / PowerShell 
# ダウンロード済みイメージの一覧を表示
docker images

期待される出力 

REPOSITORY                                      TAG       IMAGE ID       SIZE
container-registry.oracle.com/database/free     latest    a1b2c3d4e5f6   4.8GB

3.5 Oracleコンテナの起動と設定

取得したイメージからOracleコンテナを起動します。

コンテナ起動コマンド

以下のコマンドでOracleコンテナを起動します。初回起動時は、データベースの初期化に15〜20分程度かかります。

コマンドプロンプト / PowerShell 
# Oracleコンテナを起動（コンテナ名: oracle-free）
docker run -d --name oracle-free -p 1521:1521 -p 5500:5500 -e ORACLE_PWD=OraclePass123 container-registry.oracle.com/database/free:latest

コマンドオプションの説明

オプション 説明
-d バックグラウンドで起動（デタッチドモード）
--name oracle-free コンテナ名を「oracle-free」に設定
-p 1521:1521 ポート1521をマッピング（データベース接続用）
-p 5500:5500 ポート5500をマッピング（Enterprise Manager用）
-e ORACLE_PWD=... SYS/SYSTEMユーザーのパスワードを設定

パスワードについて

ORACLE_PWD=OraclePass123 で設定したパスワードは、SYSユーザーとSYSTEMユーザーの両方に適用されます。このパスワードは後でSQL Developerから接続する際に使用するので、忘れないようにメモしておきましょう。

起動ログの確認

コンテナが正常に起動しているか、ログを確認します。

コマンドプロンプト / PowerShell 
# コンテナのログを表示（Ctrl+Cで終了）
docker logs -f oracle-free

ログに「DATABASE IS READY TO USE!」と表示されたら、データベースの起動完了です。

3.6 コンテナの状態確認

起動したコンテナの状態を確認するコマンドを学習します。

基本的なDockerコマンド

起動中のコンテナ一覧

コマンドプロンプト / PowerShell 
docker ps

コンテナの停止

コマンドプロンプト / PowerShell 
docker stop oracle-free

コンテナの起動（既存コンテナ）

コマンドプロンプト / PowerShell 
docker start oracle-free

コンテナの再起動

コマンドプロンプト / PowerShell 
docker restart oracle-free

ヒント

一度コンテナを作成（docker run）したら、次回からは docker start oracle-free で起動できます。docker run を再度実行すると、新しいコンテナが作成されてエラーになるので注意しましょう。

3.7 接続情報の確認

次章でSQL Developerから接続する際に必要な情報をまとめておきます。

Oracle接続情報

項目 値
ホスト名 localhost
ポート 1521
サービス名 FREEPDB1
ユーザー名 SYSTEM
パスワード OraclePass123

FREEPDBとは

Oracle 12c以降では「マルチテナント・アーキテクチャ」が導入され、1つのコンテナデータベース（CDB）の中に複数のプラガブルデータベース（PDB）を持つ構造になっています。FREEPDB1は、Oracle Database Freeで最初から用意されているPDBの名前です。

トラブルシューティング

Rancher Desktopが起動しない

原因: WSL 2が有効化されていない、またはBIOSで仮想化が無効になっている

解決方法: PowerShellを管理者権限で開き、wsl --install を実行してWSL 2を有効化します。また、BIOSで仮想化機能（Intel VT-x/AMD-V）が有効になっているか確認してください。

docker pullが途中で失敗する

原因: ネットワーク接続が不安定、またはディスク容量不足

解決方法: 安定したネットワーク環境で再度実行するか、不要なDockerイメージを削除してディスク容量を確保してください（docker image prune）。

コンテナが起動しない（"Conflict" エラー）

原因: 同じ名前のコンテナがすでに存在している

解決方法: 既存のコンテナを削除してから再度作成します。

コマンドプロンプト / PowerShell 
# 既存コンテナを削除（停止状態でなければ先に停止）
docker rm -f oracle-free

# 再度コンテナを作成
docker run -d --name oracle-free -p 1521:1521 -p 5500:5500 -e ORACLE_PWD=OraclePass123 container-registry.oracle.com/database/free:latest
ログに「DATABASE IS READY TO USE!」が表示されない

原因: 初回起動時のデータベース初期化は時間がかかる

解決方法: 初回起動は15〜30分程度かかる場合があります。気長に待ちましょう。ログにエラーメッセージが出ていなければ正常に処理中です。

理解度確認クイズ

  1. Q1. Windows環境でDockerを使ってOracleを動かすメリットは何ですか？

    回答を表示

    回答: Dockerコンテナを使うことで、Windowsのシステムに影響を与えずにOracleを動作させることができます。環境の分離ができるため、本番環境とは別のテスト環境を簡単に作成でき、コンテナを削除すればクリーンな状態に戻せます。また、第2章のネイティブインストールと比較して、環境構築が簡単で、チーム間で同じ環境を共有しやすいというメリットもあります。

  2. Q2. docker run と docker start の違いは何ですか？

    回答を表示

    回答: docker run は新しいコンテナを作成して起動するコマンドです。一度作成したコンテナを再度起動する場合は docker start コンテナ名 を使います。同じコンテナ名で docker run を複数回実行するとエラーになります。

  3. Q3. コンテナ起動時に設定したパスワード「OraclePass123」はどのユーザーのパスワードですか？

    回答を表示

    回答: ORACLE_PWD 環境変数で設定したパスワードは、SYSユーザーとSYSTEMユーザーの両方に適用されます。このパスワードは次章でSQL Developerから接続する際に使用します。
