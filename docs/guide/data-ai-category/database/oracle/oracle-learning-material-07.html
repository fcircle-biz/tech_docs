Oracle SQL実践（結合・サブクエリ）

JOIN操作やサブクエリを使った実践的なデータ取得方法を学習します。実際のビジネスシーンで使用される複雑なクエリの作成方法を習得します。

この章で学ぶこと

  • テーブル結合（JOIN）の概念と種類を理解する
  • INNER JOIN、OUTER JOINを使い分けられる
  • サブクエリの基本と相関サブクエリを理解する
  • EXISTS/IN演算子を使った効率的なクエリを作成できる

7.1 テーブル結合とは

実際のデータベースでは、情報を効率的に管理するために複数のテーブルに分けてデータを保存します。例えば、「顧客情報」と「注文情報」は別々のテーブルに格納されます。テーブル結合（JOIN）は、これらの関連するデータを組み合わせて1つの結果として取得する操作です。

なぜテーブルを分けるのか

  • データの重複を避ける: 同じ情報を何度も保存しない（正規化）
  • データの整合性: 1箇所を更新すれば全体に反映される
  • 柔軟性: 新しい情報を追加しやすい
erDiagram CUSTOMERS ||--o{ ORDERS : "1人の顧客は複数の注文ができる" CUSTOMERS { int customer_id PK string name string email } ORDERS { int order_id PK int customer_id FK date order_date number total_amount }

主キー（PK）と外部キー（FK）

主キー（Primary Key）: テーブル内の各行を一意に識別するカラム（例: customer_id）

外部キー（Foreign Key）: 他のテーブルの主キーを参照するカラム（例: ordersテーブルのcustomer_id）

7.2 内部結合（INNER JOIN）

INNER JOIN（内部結合）は、両方のテーブルに一致するデータがある行だけを取得する結合方法です。最も基本的で、よく使われる結合タイプです。

基本構文

SQL 
-- 基本的なINNER JOIN構文
SELECT テーブル1.カラム名, テーブル2.カラム名
FROM テーブル1
INNER JOIN テーブル2
ON テーブル1.結合キー = テーブル2.結合キー;

具体例：顧客と注文の結合

SQL 
-- 顧客名と注文情報を一緒に取得
SELECT
    c.customer_id,    -- 顧客ID
    c.name,           -- 顧客名
    o.order_id,       -- 注文ID
    o.order_date,     -- 注文日
    o.total_amount    -- 注文金額
FROM customers c      -- customersテーブルに別名「c」を付与
INNER JOIN orders o   -- ordersテーブルに別名「o」を付与
ON c.customer_id = o.customer_id;

実習：INNER JOINを試してみよう

  1. 1. SQL Developerで新しいSQLワークシートを開きます
  2. 2. サンプルテーブルを作成し、データを挿入します
  3. 3. INNER JOINを使って顧客と注文情報を結合してみましょう

重要ポイント

INNER JOINは、両方のテーブルに一致するデータがある場合のみ結果に含まれます。どちらかのテーブルに一致するデータがない場合、その行は結果に表示されません。

7.3 外部結合（OUTER JOIN）

OUTER JOIN（外部結合）は、一方のテーブルに一致するデータがない場合でも、結果に含める結合方法です。LEFT、RIGHT、FULL OUTER JOINの3種類があります。

LEFT OUTER JOIN（左外部結合）

左側のテーブル（FROM句で指定したテーブル）のすべての行を取得し、右側のテーブルに一致するデータがない場合はNULLで埋めます。

SQL 
-- 全顧客と、注文がある場合は注文情報も取得
SELECT
    c.customer_id,
    c.name,
    o.order_id,
    o.total_amount
FROM customers c
LEFT OUTER JOIN orders o
ON c.customer_id = o.customer_id;

RIGHT OUTER JOIN（右外部結合）

右側のテーブル（JOIN句で指定したテーブル）のすべての行を取得します。実務ではLEFT JOINの方が多く使われます。

SQL 
-- 全注文と、顧客情報があれば取得
SELECT
    c.customer_id,
    c.name,
    o.order_id,
    o.total_amount
FROM customers c
RIGHT OUTER JOIN orders o
ON c.customer_id = o.customer_id;

FULL OUTER JOIN（完全外部結合）

両方のテーブルのすべての行を取得します。一致しない場合は両方向でNULLが設定されます。

SQL 
-- すべての顧客とすべての注文を取得
SELECT
    c.customer_id,
    c.name,
    o.order_id,
    o.total_amount
FROM customers c
FULL OUTER JOIN orders o
ON c.customer_id = o.customer_id;

ヒント

「注文していない顧客を見つけたい」場合はLEFT JOINを使い、WHERE句で「o.order_id IS NULL」という条件を追加します。これにより、右側のテーブルに一致しない行だけを抽出できます。

7.4 自己結合とクロス結合

自己結合（Self Join）

同じテーブルを自分自身と結合する操作です。例えば、社員テーブルで「上司と部下の関係」を表現する場合に使います。

SQL 
-- 社員と上司の情報を一緒に取得
SELECT
    e.employee_id,
    e.name AS employee_name,    -- 社員名
    m.name AS manager_name       -- 上司名
FROM employees e                 -- 社員テーブル（別名e）
LEFT JOIN employees m            -- 同じテーブルを別名mで参照
ON e.manager_id = m.employee_id;

クロス結合（CROSS JOIN）

2つのテーブルのすべての組み合わせを作成する結合です。結合条件（ON句）を指定しません。結果の行数は「テーブル1の行数 × テーブル2の行数」になります。

SQL 
-- すべての商品とすべての色の組み合わせを生成
SELECT
    p.product_name,
    c.color_name
FROM products p
CROSS JOIN colors c;

注意

CROSS JOINは結果の行数が非常に多くなる可能性があります。例えば、1000行のテーブル同士をクロス結合すると100万行になります。意図しない使い方をするとシステムに負荷がかかるため、注意が必要です。

7.5 サブクエリの基本

サブクエリ（副問い合わせ）は、SQL文の中に別のSQL文を埋め込む記法です。複雑な条件でのデータ抽出や、計算結果を使った絞り込みに使用します。

WHERE句でのサブクエリ

最も基本的な使い方は、WHERE句の条件としてサブクエリを使用する方法です。

SQL 
-- 平均注文金額より高い注文を抽出
SELECT
    order_id,
    customer_id,
    total_amount
FROM orders
WHERE total_amount > (
    SELECT AVG(total_amount)  -- サブクエリで平均を計算
    FROM orders
);

FROM句でのサブクエリ

サブクエリの結果を仮想的なテーブルとして扱い、FROM句に指定できます。インラインビューとも呼ばれます。

SQL 
-- 顧客ごとの注文合計金額を計算し、上位3名を抽出
SELECT
    customer_id,
    total_sales
FROM (
    SELECT
        customer_id,
        SUM(total_amount) AS total_sales
    FROM orders
    GROUP BY customer_id
) order_summary
WHERE ROWNUM <= 3
ORDER BY total_sales DESC;

SELECT句でのサブクエリ

SELECT句にサブクエリを記述することで、計算結果を追加カラムとして表示できます。スカラーサブクエリと呼ばれます。

SQL 
-- 各顧客の注文件数を一緒に表示
SELECT
    customer_id,
    name,
    (SELECT COUNT(*)
     FROM orders o
     WHERE o.customer_id = c.customer_id) AS order_count
FROM customers c;

7.6 相関サブクエリ

相関サブクエリは、外側のクエリの各行に対してサブクエリが実行されるタイプのサブクエリです。外側のテーブルの値を参照できるのが特徴です。

相関サブクエリの例

SQL 
-- 各顧客の平均注文額より高い注文を抽出
SELECT
    o1.order_id,
    o1.customer_id,
    o1.total_amount
FROM orders o1
WHERE o1.total_amount > (
    SELECT AVG(o2.total_amount)
    FROM orders o2
    WHERE o2.customer_id = o1.customer_id  -- 外側のクエリを参照
);

このクエリでは、外側のordersテーブル（o1）の各行について、その顧客の平均注文額をサブクエリで計算し、比較しています。

通常のサブクエリと相関サブクエリの違い

通常のサブクエリ: 1回だけ実行され、その結果が外側のクエリで使用される

相関サブクエリ: 外側のクエリの各行ごとに実行される（パフォーマンスに注意）

7.7 EXISTS/IN演算子の使い方

IN演算子

IN演算子は、サブクエリの結果に含まれる値と一致するかをチェックします。

SQL 
-- 注文履歴がある顧客を抽出
SELECT
    customer_id,
    name,
    email
FROM customers
WHERE customer_id IN (
    SELECT DISTINCT customer_id
    FROM orders
);

EXISTS演算子

EXISTS演算子は、サブクエリが1行以上の結果を返すかどうかをチェックします。データの存在確認に効率的です。

SQL 
-- 注文履歴がある顧客を抽出（EXISTS版）
SELECT
    customer_id,
    name,
    email
FROM customers c
WHERE EXISTS (
    SELECT 1
    FROM orders o
    WHERE o.customer_id = c.customer_id
);

NOT EXISTS/NOT IN

NOT EXISTSやNOT INを使うと、「該当しないデータ」を抽出できます。

SQL 
-- 注文履歴がない顧客を抽出
SELECT
    customer_id,
    name,
    email
FROM customers c
WHERE NOT EXISTS (
    SELECT 1
    FROM orders o
    WHERE o.customer_id = c.customer_id
);
項目 IN演算子 EXISTS演算子
用途 値のリストとの一致確認 データの存在確認
パフォーマンス サブクエリの結果を全て取得 1行見つかれば処理終了
推奨ケース 結果が少ない場合 大量データの存在確認

重要ポイント

一般的に、大量データの存在確認にはEXISTSの方が効率的です。INは結果セット全体を取得するのに対し、EXISTSは1行でも見つかれば処理を終了するためです。

7.8 実践：複数テーブルを使ったクエリ作成

これまで学んだJOINとサブクエリを組み合わせて、実際のビジネスシーンで使える複雑なクエリを作成してみましょう。

実習：複雑なクエリを作成しよう

以下のビジネス要件を満たすSQLを作成してください。

要件：

  • • 2024年1月以降に注文した顧客の一覧を取得
  • • 各顧客の注文件数と合計金額を表示
  • • 合計金額が10,000円以上の顧客のみ抽出
  • • 合計金額の降順で並び替え

解答例

SQL 
-- 複数テーブルを使った実践的なクエリ
SELECT
    c.customer_id,
    c.name,
    COUNT(o.order_id) AS order_count,
    SUM(o.total_amount) AS total_sales
FROM customers c
INNER JOIN orders o
    ON c.customer_id = o.customer_id
WHERE o.order_date >= TO_DATE('2024-01-01', 'YYYY-MM-DD')
GROUP BY c.customer_id, c.name
HAVING SUM(o.total_amount) >= 10000
ORDER BY total_sales DESC;

サブクエリを使った別解

SQL 
-- サブクエリを使ったアプローチ
SELECT
    customer_id,
    name,
    order_count,
    total_sales
FROM (
    SELECT
        c.customer_id,
        c.name,
        COUNT(o.order_id) AS order_count,
        SUM(o.total_amount) AS total_sales
    FROM customers c
    INNER JOIN orders o
        ON c.customer_id = o.customer_id
    WHERE o.order_date >= TO_DATE('2024-01-01', 'YYYY-MM-DD')
    GROUP BY c.customer_id, c.name
) order_summary
WHERE total_sales >= 10000
ORDER BY total_sales DESC;

よくあるエラーと対処法

ORA-00918: 列の定義が未確定です

原因: テーブル別名を使わずにカラム名だけを指定している
対処: SELECT句やWHERE句で「テーブル別名.カラム名」の形式で指定する

ORA-00979: GROUP BYの式ではありません

原因: SELECT句のカラムがGROUP BYに含まれていない
対処: 集計関数でないカラムは全てGROUP BYに含める

理解度確認クイズ

  1. Q1. INNER JOINとLEFT OUTER JOINの違いを説明してください。

    回答を表示

    回答: INNER JOINは両方のテーブルに一致するデータがある行のみを取得します。LEFT OUTER JOINは、左側のテーブルのすべての行を取得し、右側のテーブルに一致するデータがない場合はNULLで埋めます。「すべての顧客と、注文があれば注文情報も表示」のような要件ではLEFT JOINを使います。

  2. Q2. 相関サブクエリと通常のサブクエリの違いは何ですか？

    回答を表示

    回答: 通常のサブクエリは1回だけ実行され、その結果が外側のクエリで使用されます。相関サブクエリは外側のクエリの各行ごとに実行され、外側のテーブルの値を参照できます。相関サブクエリはより柔軟ですが、パフォーマンスに注意が必要です。

  3. Q3. EXISTS演算子がIN演算子より効率的な場合を説明してください。

    回答を表示

    回答: EXISTSは、サブクエリが1行でも該当するデータを見つけた時点で処理を終了します。一方、INはサブクエリの結果を全て取得してから比較します。そのため、大量のデータから「存在するかどうか」を確認する場合は、EXISTSの方が効率的です。特にNOT EXISTSで「該当しないデータ」を探す場合に顕著な差が出ます。
