プロンプトの基礎 - AIに伝わる指示の書き方

効果的なプロンプト（指示文）の書き方、具体的で明確な指示の重要性、基本的なプロンプトテンプレートを習得します。

この章で学ぶこと

  • プロンプト（指示文）とは何か、その重要性
  • 良いプロンプトと悪いプロンプトの違い
  • 具体的で明確な指示の書き方の基本原則
  • すぐに使える基本的なプロンプトテンプレート

3.1 プロンプトとは何か

プロンプト（Prompt）とは、生成AIに対して出す「指示文」や「質問文」のことです。 プロンプトの書き方によって、AIからの回答の質や精度が大きく変わります。

プロンプトの例

  • 「Pythonとは何ですか?」← シンプルな質問
  • 「Pythonのリスト操作について、初心者にもわかりやすく説明してください」← 具体的な指示
  • 「ビジネスメールの文例を作成してください。相手は取引先の担当者で、納期延長をお願いする内容です」← 詳細な条件指定

良いプロンプトを書くことを「プロンプトエンジニアリング」と呼びます。この章では、誰でも使える基本的なプロンプトの書き方を学びます。

3.2 良いプロンプトの5つの特徴

効果的なプロンプトには、以下の5つの特徴があります。

1. 明確性（Clarity）

何を求めているのか、曖昧さなく明確に伝える。

悪い例: 「Pythonについて教えて」

良い例: 「Python初心者向けに、Pythonの基本構文（変数、ループ、関数）を例を交えて説明してください」

2. 具体性（Specificity）

詳細な条件や背景情報を含める。

悪い例: 「レポートを書いて」

良い例: 「ビジネスプレゼンテーション用に、AI活用による業務効率化の事例を3つ挙げ、それぞれ200字程度で説明してください」

3. 文脈提供（Context）

誰に向けたものか、どのような状況で使うのかなど背景情報を伝える。

悪い例: 「SQLを説明して」

良い例: 「プログラミング未経験の営業部門スタッフ向けに、SQLとは何か、なぜビジネスで重要なのかを簡単に説明してください」

4. 形式指定（Format）

箇条書き、表、コードブロックなど、出力形式を明示する。

悪い例: 「Webフレームワークを比較して」

良い例: 「Django、Flask、FastAPIの3つのPython Webフレームワークを、表形式で比較してください。項目は「特徴」「メリット」「デメリット」「適したプロジェクト」の4つでお願いします」

5. 制約条件（Constraints）

文字数、行数、トーン（フォーマル/カジュアル）など、制約を設ける。

悪い例: 「会社紹介文を書いて」

良い例: 「IT企業向けの会社紹介文を、300字以内でフォーマルなトーンで作成してください。強調すべき点は『AI技術』『クラウドサービス』『24時間サポート』です」

3.3 悪いプロンプトと改善例

悪いプロンプトの共通点は、曖昧さ、情報不足、目的不明確です。 以下に、よくある悪いプロンプトとその改善例を示します。

悪い例1: 曖昧すぎる質問

「AIについて教えて」

問題点: AIという広大なテーマのどの部分を知りたいのか不明。初心者向けか専門家向けかも不明。

改善例

「ビジネスパーソン向けに、AI（人工知能）とは何か、どのような種類があるのか、ビジネスでどう活用されているのかを、初心者にもわかりやすく300字程度で説明してください」

悪い例2: 情報不足

「コードを書いて」

問題点: どのプログラミング言語で、何をするコードなのか、どんな入出力を想定しているのかが全く不明。

改善例

「Pythonで、CSVファイルを読み込み、特定の列（'売上'列）の合計値を計算して表示するコードを書いてください。pandasライブラリを使い、コメントも入れてください」

悪い例3: 目的不明確

「会議のメールを作って」

問題点: 会議の招集メールなのか、議事録なのか、フォローアップメールなのか不明。相手や内容も不明。

改善例

「社内の営業チーム向けに、来週火曜日10時からの『第2四半期売上報告会』の招集メールを作成してください。会議室はA会議室、所要時間は1時間、参加者は営業部全員で、事前準備として各自の売上データを持参してもらいたいです。フォーマルなトーンでお願いします」

3.4 すぐに使える基本的なプロンプトテンプレート

以下のテンプレートをコピーして、[ ]内を自分の内容に置き換えるだけで、効果的なプロンプトが作成できます。

テンプレート1: 文章作成 

[対象読者]向けに、[トピック]について[文字数]程度で[トーン（フォーマル/カジュアル）]に説明してください。

重要ポイント:
- [ポイント1]
- [ポイント2]
- [ポイント3]

使用例: 「新入社員向けに、ビジネスメールのマナーについて500字程度でフォーマルに説明してください。重要ポイント: 件名の書き方、適切な敬語、結びの言葉」

テンプレート2: 要約 

以下の文章を[文字数]程度で要約してください。特に[強調したい点]を重点的に含めてください。

[要約したい文章]

使用例: 「以下の文章を200字程度で要約してください。特に『AI活用のメリット』を重点的に含めてください。[長い記事テキスト]」

テンプレート3: 比較分析 

[項目A]と[項目B]を比較して、表形式で示してください。

比較観点:
- [観点1]
- [観点2]
- [観点3]
- [観点4]

使用例: 「ChatGPTとClaudeを比較して、表形式で示してください。比較観点: 料金プラン、処理可能なトークン数、得意分野、無料プランの有無」

テンプレート4: コード生成 

[プログラミング言語]で、[機能]を実装するコードを書いてください。

要件:
- [要件1]
- [要件2]
- [要件3]

コメントを日本語で入れてください。

使用例: 「Pythonで、リストから重複を削除する関数を書いてください。要件: 順序を保持する、空リストにも対応する、コメントを日本語で入れてください」

テンプレート5: アイデア出し 

[目的]のために、[制約条件]を考慮して[数]個のアイデアを提案してください。

各アイデアには以下を含めてください:
- タイトル
- 概要（100字程度）
- メリット
- デメリット

使用例: 「社員のモチベーション向上のために、予算50万円以内で実施できる施策を5個提案してください。各アイデアにはタイトル、概要、メリット、デメリットを含めてください」

3.5 プロンプト作成の実習

実習: 効果的なプロンプトを作成してみよう

シナリオ1: ビジネスメールの作成

取引先に新製品の紹介メールを送りたい。製品名は「CloudSync Pro」で、クラウドストレージとファイル同期機能を持つ。無料トライアルを提案したい。

この情報を元に、効果的なプロンプトを作成してください。

例示回答を表示

プロンプト例:

「取引先の企業担当者向けに、新製品『CloudSync Pro』の紹介メールを作成してください。製品の特徴はクラウドストレージとファイル同期機能で、30日間の無料トライアルを提案します。フォーマルなビジネストーンで、300字程度でお願いします。」

シナリオ2: 技術用語の説明

非エンジニアの営業チーム向けに、「API」とは何か、なぜビジネスで重要なのかを説明する資料を作りたい。

この情報を元に、効果的なプロンプトを作成してください。

例示回答を表示

プロンプト例:

「プログラミング知識のない営業チーム向けに、『API（Application Programming Interface）』とは何か、なぜビジネスで重要なのかを、具体的な事例を交えて400字程度でわかりやすく説明してください。専門用語は避け、日常的な例え話を使ってください。」

シナリオ3: データ分析レポート

売上データから、上半期の売上傾向を分析し、要約レポートを作成したい。ポイントは「前年比」「地域別動向」「主力商品」の3つ。

この情報を元に、効果的なプロンプトを作成してください。

例示回答を表示

プロンプト例:

「上半期の売上データを分析した要約レポートを作成してください。重点項目は『前年比の推移』『地域別の売上動向』『主力商品の販売状況』の3つです。経営陣向けにフォーマルなトーンで、箇条書き形式で500字程度にまとめてください。」

理解度確認クイズ

  1. Q1. 良いプロンプトの5つの特徴を挙げてください。

    回答を表示

    ① 明確性（Clarity）: 何を求めているか明確に伝える、② 具体性（Specificity）: 詳細な条件や背景情報を含める、③ 文脈提供（Context）: 誰に向けたものか、どのような状況で使うのか背景を伝える、④ 形式指定（Format）: 箇条書き、表、コードブロックなど出力形式を明示する、⑤ 制約条件（Constraints）: 文字数、行数、トーンなど制約を設ける。

  2. Q2. 「Pythonについて教えて」というプロンプトを、具体的で効果的なものに改善してください。

    回答を表示

    例: 「プログラミング初心者向けに、Pythonとは何か、どのような分野で使われているのか、学習を始めるメリットを、400字程度でわかりやすく説明してください」など。重要なのは、対象読者、知りたい内容の範囲、文字数、トーンを明確にすることです。

  3. Q3. プロンプトに「制約条件」を含めることが重要な理由は何ですか?

    回答を表示

    制約条件（文字数、行数、トーン、形式など）を含めることで、生成AIの出力を自分の目的に合った形に絞り込むことができます。制約がないと、AIは自由に回答を生成するため、期待とは異なる長さや形式の回答が返ってくる可能性があります。

  4. Q4. 「文脈提供（Context）」がなぜ重要か、例を挙げて説明してください。

    回答を表示

    文脈提供により、AIは回答の対象読者やレベルを適切に調整できます。例えば「SQLを説明して」だけでは、専門的な説明になる可能性がありますが、「プログラミング未経験の営業部門スタッフ向けに」と文脈を加えることで、専門用語を避けた初心者向けの説明が得られます。

この章のまとめ

  • プロンプトとは、生成AIに対する「指示文」や「質問文」で、その書き方が回答の質を大きく左右する
  • 良いプロンプトの5つの特徴: 明確性、具体性、文脈提供、形式指定、制約条件
  • 悪いプロンプトの共通点は、曖昧さ、情報不足、目的不明確
  • 基本的なプロンプトテンプレートを活用することで、初心者でも効果的な指示を作成できる
  • 実際にプロンプトを書いて試行錯誤することが、上達への近道
