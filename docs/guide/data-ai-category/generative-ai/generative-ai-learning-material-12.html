APIとプログラミング連携の基礎

OpenAI API、Claude APIの概要と基本的な使い方、簡単なプログラミング連携の入門を学びます。

この章で学ぶこと

  • 生成AI API（OpenAI API、Claude API）の基本概念と仕組み
  • APIキーの取得方法とセキュリティの基礎
  • Pythonを使った簡単なAPI呼び出しの実装

12.1 APIとは何か - プログラムから生成AIを使う仕組み

API（Application Programming Interface）は、異なるソフトウェア同士がやり取りするための「窓口」のことです。 生成AI APIを使うことで、WebブラウザのChatGPTではなく、プログラムから直接生成AIの機能を呼び出すことができます。

APIを使うメリット

  • 自動化: プログラムで大量のテキストを一括処理できる
  • カスタマイズ: 独自のアプリケーションに生成AI機能を組み込める
  • 効率化: 手動でコピー&ペーストする手間が不要
  • 統合: データベースや他のシステムと連携できる

API通信の基本的な流れ

sequenceDiagram participant A as あなたのプログラム participant B as 生成AI API
（OpenAI/Claude） participant C as AIモデル
（GPT-4/Claude等） A->>B: リクエスト送信
（プロンプト＋設定） B->>C: モデル呼び出し C->>B: 応答生成 B->>A: レスポンス返却
（生成されたテキスト）

具体例: ブログ記事の自動生成

例えば、商品情報のデータベースから複数の商品説明を取り出し、それぞれについて生成AIで紹介記事を自動生成する、といったことがAPIで可能になります。

12.2 OpenAI API - ChatGPTをプログラムから使う

OpenAI APIは、ChatGPTやDALL-Eなどの機能をプログラムから利用できるサービスです。 GPT-4、GPT-3.5など、最新モデルにアクセスできます。

APIキーの取得手順

  1. OpenAI Platform にアクセスし、アカウントを作成（既存のChatGPTアカウントとは別）
  2. ダッシュボードの「API keys」セクションへ移動
  3. 「Create new secret key」をクリックしてAPIキーを生成
  4. 生成されたキーをコピー（二度と表示されないので注意）
  5. 支払い方法を登録（無料枠あり、従量課金制）

APIキーの取り扱い注意

  • APIキーは他人に見せない、共有しない
  • GitHubなどのリポジトリに直接書き込まない（環境変数を使う）
  • 定期的にキーをローテーション（再生成）する
  • 不要になったキーは削除する

Pythonでの基本的な使用例

以下は、OpenAI APIを使って簡単なテキスト生成を行うPythonコードの例です。

openai_example.py 
# OpenAI APIのライブラリをインストール: pip install openai
import openai
import os

# APIキーを環境変数から読み込む
openai.api_key = os.getenv("OPENAI_API_KEY")

# APIリクエストを送信
response = openai.ChatCompletion.create(
    model="gpt-3.5-turbo",  # 使用するモデル
    messages=[
        {"role": "user", "content": "生成AIとは何か、簡潔に説明してください"}
    ],
    max_tokens=200,  # 最大トークン数
    temperature=0.7  # 創造性（0〜1、高いほど多様な出力）
)

# 応答を表示
print(response.choices[0].message.content)

コードの解説

  • model: 使用するAIモデル（gpt-3.5-turbo、gpt-4など）
  • messages: 会話の履歴（roleは「user」「assistant」「system」）
  • max_tokens: 生成する最大トークン数（コスト管理に重要）
  • temperature: 応答の多様性（0に近いほど決定的、1に近いほど創造的）

12.3 Claude API - Anthropic AIをプログラムから使う

Claude APIは、Anthropic社が提供する高品質な生成AIのAPIです。 長文処理能力が高く、安全性と正確性に優れています。

Claude APIキーの取得手順

  1. Anthropic Console にアクセスし、アカウントを作成
  2. 「API Keys」セクションへ移動
  3. 「Create Key」をクリックしてAPIキーを生成
  4. 生成されたキーをコピー（再表示不可）
  5. 支払い方法を登録（従量課金制）

Pythonでの基本的な使用例

claude_example.py 
# Claude APIのライブラリをインストール: pip install anthropic
import anthropic
import os

# APIキーを環境変数から読み込む
client = anthropic.Anthropic(
    api_key=os.getenv("ANTHROPIC_API_KEY")
)

# APIリクエストを送信
message = client.messages.create(
    model="claude-3-5-sonnet-20241022",  # 使用するモデル
    max_tokens=300,  # 最大トークン数
    messages=[
        {"role": "user", "content": "生成AIとは何か、簡潔に説明してください"}
    ]
)

# 応答を表示
print(message.content[0].text)

Claude APIの特徴

  • 長文処理能力が高い（最大200,000トークン）
  • 安全で正確な応答（Constitutional AI技術）
  • ビジネス文書や専門的な内容に強い
  • 日本語の処理品質が高い

12.4 料金体系と使用量管理

生成AI APIは従量課金制で、使用したトークン数に応じて料金が発生します。 適切に管理することでコストを抑えながら効果的に活用できます。

サービス モデル 料金目安（2024年時点）
OpenAI API GPT-3.5-turbo $0.0015/1Kトークン（入力）、$0.002/1Kトークン（出力）
OpenAI API GPT-4 $0.03/1Kトークン（入力）、$0.06/1Kトークン（出力）
Claude API Claude 3.5 Sonnet $3/MTok（入力）、$15/MTok（出力）
Claude API Claude 3 Haiku $0.25/MTok（入力）、$1.25/MTok（出力）

トークンとは

トークンは、テキストを小さな単位に分割したものです。日本語の場合、1文字が約2〜3トークンに相当します。

例: 「生成AI」は約4〜6トークン、400文字の日本語文章は約800〜1200トークンです。

コスト管理のベストプラクティス

  • max_tokensを設定: 出力トークン数の上限を設定してコスト予測を容易に
  • 適切なモデル選択: 簡単なタスクには低コストモデル（GPT-3.5、Claude Haiku）
  • 使用量モニタリング: APIダッシュボードで定期的に使用状況を確認
  • 予算アラート設定: 使いすぎ防止のため予算上限を設定

12.5 APIキーの安全な管理 - 環境変数の使い方

APIキーをコードに直接書き込むのは危険です。環境変数を使って安全に管理しましょう。

環境変数の設定方法

Windows（コマンドプロンプト）

set OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
set ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxxxx

Mac/Linux（ターミナル）

export OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
export ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxxxx

.envファイルを使う方法（推奨）

プロジェクトルートに.envファイルを作成:

.env 
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxxxx

Pythonで読み込む（python-dotenvをインストール: pip install python-dotenv）:

from dotenv import load_dotenv
load_dotenv()  # .envファイルを読み込む

重要: .envファイルは公開しない

.envファイルをGitリポジトリにコミットしないよう、.gitignoreに必ず追加してください。

.gitignore 
.env

実際にOpenAI APIまたはClaude APIを使って、簡単なテキスト生成プログラムを作成してみましょう。

  1. APIキーを取得（OpenAIまたはAnthropic）
  2. 環境変数にAPIキーを設定

理解度確認クイズ

  1. Q1. APIとは何ですか？生成AI APIを使うメリットを3つ挙げてください。

    回答を表示

    APIは、異なるソフトウェア同士がやり取りするための窓口です。メリットは、①自動化（大量処理が可能）、②カスタマイズ（独自アプリに組み込める）、③統合（他システムと連携できる）などがあります。

  2. Q2. APIキーを安全に管理するために、やってはいけないことは何ですか？

    回答を表示

    ①APIキーをコードに直接書き込む、②GitHubなど公開リポジトリにコミットする、③他人に共有する、④不要なキーをそのまま放置する、などです。環境変数や.envファイルを使い、.gitignoreに.envを追加するのがベストプラクティスです。

  3. Q3. 生成AI APIの料金は何に基づいて計算されますか？コストを抑える方法を2つ挙げてください。

    回答を表示

    料金は使用したトークン数に基づいて計算されます（従量課金制）。コストを抑える方法は、①max_tokensで出力上限を設定する、②簡単なタスクには低コストモデル（GPT-3.5、Claude Haiku）を使う、などです。

  4. Q4. OpenAI APIとClaude APIの主な違いは何ですか？

    回答を表示

    OpenAI APIはChatGPT（GPT-4、GPT-3.5など）を提供し、幅広い用途に対応します。Claude APIはAnthropicのClaude（Claude 3.5 Sonnetなど）を提供し、長文処理能力が高く（最大200,000トークン）、安全性と正確性に優れています。用途や予算に応じて使い分けることが重要です。

この章のまとめ

  • APIは異なるソフトウェア同士がやり取りする窓口で、生成AIをプログラムから利用できる
  • OpenAI APIとClaude APIを使えば、Pythonなどから簡単に生成AI機能を呼び出せる
  • APIキーは環境変数や.envファイルで安全に管理し、公開リポジトリにコミットしない
  • 料金は使用トークン数に基づく従量課金制で、max_tokensや適切なモデル選択でコスト管理が可能
  • 実際にコードを書いて試すことで、APIの使い方とプログラミング連携の基礎を習得できる
