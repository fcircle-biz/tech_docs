第5章

文章作成への活用 - ビジネス文書・メール・レポート

ビジネスメール、報告書、議事録、プレゼン資料など、実務での文章作成に生成AIを活用する方法を学びます。

この章で学ぶこと

  • ビジネスメールの作成と効果的なプロンプト設計
  • 報告書・議事録の作成における生成AI活用のポイント
  • プレゼン資料の構成案作成とスライド原稿の生成
  • 文章の添削・校正・リライトの実践方法

5.1 ビジネスメールの作成

生成AIは、ビジネスメールの下書き作成、書き直し、添削に非常に有効です。 相手や目的に合わせた適切なトーンで、効率的にメールを作成できます。

ビジネスメール作成のプロンプト要素

  • 宛先: 社内・社外、上司・同僚・取引先など関係性を明示
  • 目的: お礼、依頼、報告、謝罪、提案など
  • トーン: 丁寧、カジュアル、フォーマルなど
  • 内容: 伝えたい具体的な情報や背景

実例1: 会議調整の依頼メール

プロンプト例 
取引先の田中様に、来週火曜日または水曜日での打ち合わせ調整をお願いするメールを作成してください。

【背景】
- 新商品のプレゼンテーションを行いたい
- 所要時間は1時間程度
- 当方は午後であればいつでも可能

【トーン】丁寧でフォーマル
【文字数】200文字程度

生成AIの回答例

件名: 新商品プレゼンテーションのご相談

田中様

いつも大変お世話になっております。

この度、弊社の新商品につきまして、ぜひ田中様にプレゼンテーションをさせていただきたく、ご連絡申し上げました。

つきましては、来週火曜日または水曜日の午後のお時間で、1時間程度お時間をいただけますでしょうか。ご都合の良い日時をお知らせいただけますと幸いです。

お忙しいところ恐縮ですが、何卒よろしくお願い申し上げます。

実例2: イベント参加後のお礼メール

プロンプト例 
昨日の展示会に参加した佐藤様へ、お礼メールを作成してください。

【背景】
- 弊社ブースへご来場いただいた
- 製品について熱心に質問していただいた
- 後日、詳細資料を送付する予定

【トーン】丁寧で親しみやすい
【文字数】150文字程度

メール作成のコツ

  • まずAIに下書きを作成させ、それを自分の言葉で調整する
  • 相手の名前や固有名詞は必ず自分で確認・修正する
  • 最終的な送信前に必ず読み返す（誤情報の可能性があるため）

5.2 報告書・議事録の作成

会議の内容やプロジェクトの進捗を報告する際、生成AIを活用することで、効率的かつ分かりやすい文書を作成できます。

議事録作成フロー

flowchart LR A["会議メモ・録音"] --> B["要点を箇条書きで整理"] B --> C["生成AIに整形を依頼"] C --> D["構造化された議事録"] D --> E["人間による確認・調整"] E --> F["最終版の完成"]

実例: 会議議事録の作成

プロンプト例 
以下の会議メモを、分かりやすい議事録の形式に整形してください。

【会議メモ】
- 日時: 2026年1月10日 14:00-15:00
- 参加者: 山田部長、佐藤課長、田中、鈴木
- テーマ: 新商品のマーケティング戦略

討議内容:
- ターゲット層は30-40代の働く女性
- SNS広告とインフルエンサーマーケティングを併用
- 予算は500万円で検討
- 2月中旬にキャンペーン開始予定
- 次回会議は1月20日 10:00

【形式】
- タイトル、日時、参加者、議題、決定事項、次回アクション

実例: プロジェクト進捗報告書

プロンプト例 
以下の情報をもとに、上司向けのプロジェクト進捗報告書を作成してください。

【プロジェクト情報】
- プロジェクト名: 新ECサイト構築
- 報告期間: 2026年1月1日〜1月7日
- 進捗率: 45% (予定より5%遅延)
- 完了したタスク: デザインカンプ作成、DB設計
- 進行中のタスク: フロントエンド実装
- 課題: 決済APIの選定が遅れている
- 次週の予定: 決済API選定完了、ユーザー登録機能実装

【形式】
- エグゼクティブサマリー、進捗状況、課題、今後の対応、次週の予定

注意点

数値や固有名詞、日付などは必ず自分で確認してください。生成AIが誤った情報を補完する可能性があります。

5.3 プレゼン資料の構成案作成

プレゼン資料の作成では、まず全体の構成（ストーリー）を決めることが重要です。 生成AIは、テーマに合わせた論理的な流れを提案してくれます。

実例: プレゼン構成案の作成

プロンプト例 
以下のテーマでプレゼンテーション資料の構成案を作成してください。

【テーマ】生成AI導入による業務効率化の提案
【対象】経営層（IT知識は中程度）
【時間】15分
【目的】生成AI導入の承認を得る
【スライド枚数】10-12枚

【含めたい内容】
- 生成AIとは何か
- 導入のメリット（業務効率化、コスト削減）
- 他社の成功事例
- 導入プラン（スケジュール、予算）
- セキュリティ対策

各スライドのタイトルと簡単な内容説明を提案してください。

生成AIの回答例

  1. スライド1: タイトル - 「生成AI導入による業務効率化の提案」
  2. スライド2: 現状の課題 - 業務負荷の高いタスク、残業時間の増加
  3. スライド3: 生成AIとは - 基本概念と主要サービス紹介
  4. スライド4: 導入メリット① - 業務効率化（文書作成、メール対応）
  5. スライド5: 導入メリット② - コスト削減（時間削減による人件費削減）
  6. スライド6: 他社成功事例 - A社: 30%の業務時間削減、B社: 顧客対応速度2倍
  7. スライド7: 導入プラン - 3ヶ月のパイロット導入 → 全社展開
  8. スライド8: スケジュール - 月別のタスクとマイルストーン
  9. スライド9: セキュリティ対策 - データ管理、利用ガイドライン
  10. スライド10: 予算 - 初期費用、月額費用の内訳
  11. スライド11: 期待効果 - 定量的な効果予測（時間削減率、ROI）
  12. スライド12: まとめ・次のアクション - 導入承認のお願い

スライドの詳細原稿作成

プロンプト例 
先ほどの構成案の「スライド4: 導入メリット① 業務効率化」について、
詳細な内容を箇条書きで作成してください。

【要件】
- 具体的な業務例を含める
- 数値で効果を示す
- 3-5個の箇条書き
- 各項目は15文字程度

プレゼン資料作成のポイント

  • まず全体構成を作り、それから各スライドの詳細を作成
  • 1スライド1メッセージの原則を守る
  • 聴衆に合わせた言葉遣いと専門用語の説明を意識
  • ビジュアル要素（図表、グラフ）の配置もAIに提案させる

5.4 文章の添削・校正・リライト

生成AIは、既存の文章を改善する用途でも非常に有効です。誤字脱字のチェック、表現の改善、トーンの調整などが可能です。

文章リライトの種類

flowchart TD A["元の文章"] --> B["誤字脱字チェック"] A --> C["表現の改善"] A --> D["トーン調整"] A --> E["簡潔化・要約"] B --> F["校正済み文章"] C --> F D --> F E --> F

実例1: 文章の校正

プロンプト例 
以下の文章を校正してください。誤字脱字、文法の誤り、不自然な表現を修正し、
改善点を説明してください。

【文章】
弊社は新商品の開発に成功しました。これにより、お客様により良いサービスを提供できると
考えております。今後とも、弊社製品を御愛顧頂けますよう、宜しくお願い致します。

実例2: トーン調整（カジュアル→フォーマル）

プロンプト例 
以下の文章を、よりフォーマルな表現に書き換えてください。

【文章】
来週の会議なんですが、ちょっと時間変更してもいいですか？
急な用事が入っちゃって、14時からだと厳しいです。
できれば16時からにしてもらえると助かります。

実例3: 文章の簡潔化

プロンプト例 
以下の文章をより簡潔に、分かりやすく書き換えてください。

【文章】
本プロジェクトにつきましては、当初の計画通りに進行しておりまして、
現時点におきましては特段の問題は発生しておらず、予定されているスケジュールに
従って、着実に進捗している状況でございます。

文章改善のベストプラクティス

  • 改善前後を比較し、変更理由を説明させると学びになる
  • 複数のバージョンを提案させ、最適なものを選ぶ
  • トーンや文体の指定は具体的に（「丁寧」「親しみやすい」「専門的」など）

実習: 自分の業務で使う文書を作成してみよう

  1. ステップ1: 自分の業務で頻繁に作成する文書を1つ選ぶ（メール、報告書など）
  2. ステップ2: その文書を作成するためのプロンプトを設計する（背景、目的、トーンを含める）
  3. ステップ3: 生成AIに下書きを作成させる
  4. ステップ4: 生成された文書を読み、修正が必要な箇所をメモする
  5. ステップ5: プロンプトを改善し、再度生成してみる（反復して精度を上げる）

理解度確認クイズ

  1. Q1. ビジネスメール作成時に生成AIに指定すべき要素を3つ挙げてください。

    回答を表示

    ① 宛先・関係性（社内・社外、上司・同僚など）、② 目的（お礼、依頼、報告など）、③ トーン（丁寧、フォーマル、カジュアルなど）。これらを明確に指定することで、適切なメールが生成されます。

  2. Q2. 議事録を生成AIで作成する際の注意点は何ですか?

    回答を表示

    数値、固有名詞、日付などの事実情報は必ず人間が確認することが重要です。生成AIが誤った情報を補完したり、推測で内容を追加する可能性があるため、最終チェックは必須です。

  3. Q3. プレゼン資料作成における「1スライド1メッセージの原則」とは何ですか?

    回答を表示

    1つのスライドには1つの主要メッセージだけを載せるという原則です。情報を詰め込みすぎず、聴衆が理解しやすいように、各スライドで伝えたいことを1つに絞ることが重要です。

  4. Q4. 文章のリライトで生成AIに依頼できる改善の種類を3つ挙げてください。

    回答を表示

    ① 誤字脱字・文法チェック、② トーン調整（フォーマル⇔カジュアル）、③ 簡潔化・要約。その他にも、表現の改善、専門用語の平易化なども可能です。

この章のまとめ

  • ビジネスメール作成では、宛先・目的・トーンを明確に指定することが重要
  • 議事録や報告書作成では、要点を整理してから生成AIに整形を依頼すると効率的
  • プレゼン資料は、まず構成案を作り、その後各スライドの詳細を作成する
  • 文章の添削・リライトでは、改善前後を比較し学びを得ることが大切
  • 生成された文書は必ず人間が最終確認し、誤情報がないかチェックする
