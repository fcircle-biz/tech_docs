ハッシュオブジェクト（Hash Objects）

メモリ上でデータを結合する、SAS最強の高速処理手法を学びます。ハッシュオブジェクトの基本構造（DefineKey, DefineData）、巨大なマスタデータとの高速マッチング（Merge文との速度比較）、動的なルックアップテーブルの作成を習得します。

この章で学ぶこと

  • ハッシュオブジェクトの基本構造（DefineKey, DefineData）とメモリ上でのデータ操作
  • 巨大なマスタデータとの高速マッチング手法と、Merge文との速度比較
  • 動的なルックアップテーブルの作成と実務での活用方法

3.1 ハッシュオブジェクトとは

ハッシュオブジェクトは、メモリ上にデータをキー・バリュー形式で格納し、超高速でデータを検索・結合できるSASの強力な機能です。従来のMerge文やPROC SQLによる結合では、データセット全体を読み込んで処理しますが、ハッシュオブジェクトを使用すると、メモリ内での即座の検索が可能になります。

特に、数百万行のトランザクションデータと数千行のマスタデータを結合する場合、Merge文では両方を並べ替える必要がありますが、ハッシュオブジェクトならマスタデータをメモリに一度読み込むだけで、トランザクションの各行に対して高速にマッチングできます。

ハッシュオブジェクトの特徴

  • 高速検索: キーによるハッシュ検索（O(1)）で即座にデータを取得
  • メモリ効率: 小さなマスタデータを一度だけメモリに読み込む
  • 柔軟性: 動的にデータを追加・更新・削除可能
  • ソート不要: Merge文と違い、事前にソートする必要がない

3.2 基本構造：DefineKeyとDefineData

ハッシュオブジェクトは、DECLARE文で宣言し、DefineKeyメソッドでキー列を、DefineDataメソッドでデータ列を定義します。その後、DefineDoneメソッドで定義を完了します。

ハッシュオブジェクトの基本メソッド

  • DECLARE HASH - ハッシュオブジェクトを宣言
  • DefineKey - キー列を定義
  • DefineData - データ列を定義
  • DefineDone - 定義完了
  • FIND - キーで検索（見つかれば0、見つからなければ非0）

例1: 商品マスタのルックアップ

売上明細データに、商品マスタから商品名と価格を追加する例です。

ハッシュオブジェクト基本.sas 
/* 商品マスタデータ */
DATA products;
    INPUT product_id $ product_name $ price;
    DATALINES;
P001 Laptop 1200
P002 Mouse 25
P003 Keyboard 75
;
RUN;

/* 売上明細データ */
DATA sales;
    INPUT sale_id product_id $ quantity;
    DATALINES;
S001 P001 2
S002 P002 5
S003 P003 3
S004 P001 1
;
RUN;

/* ハッシュオブジェクトでルックアップ */
DATA sales_enriched;
    /* ハッシュオブジェクトを宣言（データセット=productsから読込） */
    IF _N_ = 1 THEN DO;
        DECLARE HASH h(dataset: 'products');
        h.DefineKey('product_id');         /* キー列 */
        h.DefineData('product_name', 'price'); /* データ列 */
        h.DefineDone();
    END;

    SET sales;  /* 売上明細を1行ずつ読込 */

    /* product_idで検索して、product_nameとpriceを取得 */
    rc = h.FIND();

    /* 見つかった場合（rc=0）、金額を計算 */
    IF rc = 0 THEN
        total_amount = quantity * price;
    ELSE
        total_amount = .;  /* 見つからない場合は欠損値 */

    DROP rc;  /* 戻り値は不要なので削除 */
RUN;

PROC PRINT DATA=sales_enriched;
RUN;

実行結果

Obs  sale_id  product_id  quantity  product_name  price  total_amount
 1    S001     P001           2     Laptop       1200       2400
 2    S002     P002           5     Mouse          25        125
 3    S003     P003           3     Keyboard       75        225
 4    S004     P001           1     Laptop       1200       1200

_N_=1の条件でハッシュを初期化

IF _N_ = 1の条件により、ハッシュオブジェクトは最初の行のみで初期化されます。2行目以降は既に作成されたハッシュを使い回すため、高速に検索できます。

3.3 巨大なマスタデータとの高速マッチング

ハッシュオブジェクトの真価は、大量のトランザクションデータと小～中規模のマスタデータを結合する場合に発揮されます。特に以下のケースで劇的な速度改善が見込めます：

  • トランザクション: 数百万～数千万行
  • マスタ: 数千～数万行（メモリに収まるサイズ）
  • 結合キーが一致しない行が多い（LEFT JOINのような場合）

例2: 大量取引データの顧客情報付加

大量データ処理.sas 
/* 顧客マスタ（10,000件） */
DATA customers;
    LENGTH customer_id $10 customer_name $50 region $20;
    /* 実際は外部ファイルから読み込むことが多い */
    /* ここでは簡略化のため省略 */
RUN;

/* 取引データ（100万件） */
DATA transactions;
    LENGTH transaction_id $15 customer_id $10;
    /* 実際は外部ファイルから読み込む */
RUN;

/* ハッシュオブジェクトで結合 */
DATA transactions_with_customer;
    IF _N_ = 1 THEN DO;
        DECLARE HASH h(dataset: 'customers');
        h.DefineKey('customer_id');
        h.DefineData('customer_name', 'region');
        h.DefineDone();
    END;

    SET transactions;

    /* 顧客情報をルックアップ */
    rc = h.FIND();

    /* 見つからない場合は「不明」を設定 */
    IF rc NE 0 THEN DO;
        customer_name = '不明';
        region = '不明';
    END;

    DROP rc;
RUN;

メモリ使用量の注意点

ハッシュオブジェクトはマスタデータ全体をメモリに読み込むため、マスタが数百万行を超える場合はメモリ不足になる可能性があります。その場合は、Merge文やPROC SQLの使用を検討してください。一般的には、マスタが10万行以下であればハッシュオブジェクトが適しています。

3.4 Merge文との速度比較

実際にハッシュオブジェクトとMerge文で同じ処理を行い、速度を比較してみましょう。OPTIONS FULLSTIMER;を使用することで、詳細な実行時間を測定できます。

例3: 速度比較コード

速度比較.sas 
/* 実行時間を測定 */
OPTIONS FULLSTIMER;

/* 【方法1】Merge文による結合 */
PROC SORT DATA=sales;
    BY product_id;
RUN;

PROC SORT DATA=products;
    BY product_id;
RUN;

DATA sales_merge;
    MERGE sales(IN=a) products(IN=b);
    BY product_id;
    IF a;  /* LEFT JOIN */
RUN;

/* 【方法2】ハッシュオブジェクトによる結合 */
DATA sales_hash;
    IF _N_ = 1 THEN DO;
        DECLARE HASH h(dataset: 'products');
        h.DefineKey('product_id');
        h.DefineData('product_name', 'price');
        h.DefineDone();
    END;

    SET sales;
    rc = h.FIND();
    DROP rc;
RUN;

OPTIONS NOFULLSTIMER;

速度比較結果（例）

【Merge文】
- PROC SORT (sales):     2.5秒
- PROC SORT (products):  0.3秒
- DATA Merge:            1.2秒
合計: 4.0秒

【ハッシュオブジェクト】
- DATA Hash:             0.8秒
合計: 0.8秒

速度改善: 約5倍高速化

ハッシュオブジェクトが有利なケース

  • トランザクションデータが既にソート済みでない場合（PROC SORTの時間を節約）
  • マスタデータが小さく、トランザクションが大量にある場合
  • 同じマスタに対して複数回ルックアップする場合

実習 3-1: 複数キーによるハッシュオブジェクト

複数の列をキーにしてハッシュオブジェクトを使用し、地域別・商品別の価格マスタからデータを取得してください。

手順

  1. 1 地域別商品価格マスタを作成します（region, product_id, special_price列）。
  2. 2 売上データに対して、地域とproduct_idの両方をキーとしてハッシュオブジェクトを作成します。
  3. 3 FINDメソッドで該当する特別価格を取得し、計算に利用します。

ヒント

/* 複数キーの定義 */
h.DefineKey('region', 'product_id');
h.DefineData('special_price');
h.DefineDone();

理解度確認クイズ

  1. Q1. ハッシュオブジェクトがMerge文よりも高速になる理由は何ですか？

    回答を表示

    ハッシュオブジェクトは、マスタデータをメモリ上に一度だけ読み込み、キーによるハッシュ検索（O(1)）で即座にデータを取得します。一方、Merge文は両方のデータセットをソートし、順次照合する必要があるため、ソート時間とI/O時間がかかります。

  2. Q2. DefineKeyとDefineDataメソッドの役割の違いは何ですか？

    回答を表示

    DefineKeyは検索に使用するキー列を指定し、DefineDataは取得したいデータ列を指定します。FINDメソッドはDefineKeyで指定した列の値で検索し、DefineDataで指定した列の値を取得します。

  3. Q3. ハッシュオブジェクトの使用が適さないケースは何ですか？

    回答を表示

    マスタデータが数百万行以上と非常に大きい場合、全てをメモリに読み込むとメモリ不足になる可能性があります。このような場合は、Merge文やPROC SQLを使用する方が適切です。一般的には、マスタが10万行以下の場合にハッシュオブジェクトが有効です。

第3章のまとめ

  • ハッシュオブジェクトは、メモリ上でキー・バリュー形式でデータを保持し、超高速検索を実現する。
  • DefineKey、DefineData、DefineDoneでハッシュ構造を定義し、FINDメソッドで検索する。
  • 大量のトランザクションと小～中規模のマスタを結合する場合、Merge文より大幅に高速化できる。
  • マスタが10万行以下の場合に最適。それ以上の場合はメモリ不足に注意が必要。
