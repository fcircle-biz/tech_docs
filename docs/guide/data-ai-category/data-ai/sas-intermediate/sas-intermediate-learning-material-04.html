高度なPROC SQL

データステップでは記述が複雑になる処理をSQLでスマートに解決します。相関サブクエリを用いた高度な抽出、Dictionaryテーブルの活用（SASシステム内のメタデータ・変数一覧の取得）、SQL内でのマクロ変数生成（INTO句の活用）を学びます。

この章で学ぶこと

  • 相関サブクエリを使った高度なデータ抽出テクニック
  • Dictionaryテーブルによるメタデータ（テーブル一覧・列情報）の活用
  • INTO句を使ったSQL実行結果のマクロ変数への格納

4.1 相関サブクエリによる高度なデータ抽出

相関サブクエリ（Correlated Subquery）とは、外側のクエリの各行ごとに実行されるサブクエリです。外側のクエリの列を参照しながら、条件を動的に変化させることができるため、「各グループの平均を超える値」「各カテゴリの最新レコード」といった複雑な抽出が簡潔に記述できます。

相関サブクエリの特徴

  • 外側のクエリの各行ごとにサブクエリが実行される
  • サブクエリ内で外側のテーブルのエイリアスを参照できる
  • 通常のサブクエリに比べて処理時間がかかる場合がある

例1: 各部門の平均給与を超える従業員を抽出

各部門ごとに平均給与を計算し、その平均を超える従業員のみを抽出します。

相関サブクエリ_平均超え.sas 
/* サンプルデータ */
DATA employees;
    INPUT emp_id dept $ salary;
    DATALINES;
1 Sales 50000
2 Sales 60000
3 Sales 55000
4 IT 70000
5 IT 80000
6 IT 65000
7 HR 45000
8 HR 50000
;
RUN;

/* 各部門の平均給与を超える従業員を抽出 */
PROC SQL;
    SELECT emp_id, dept, salary,
           /* 各部門の平均給与もあわせて表示 */
           (SELECT AVG(salary)
            FROM employees AS e2
            WHERE e2.dept = e1.dept) AS dept_avg
    FROM employees AS e1
    WHERE salary >
          (SELECT AVG(salary)
           FROM employees AS e2
           WHERE e2.dept = e1.dept);
QUIT;

実行結果

emp_id    dept      salary    dept_avg
  2       Sales      60000      55000
  4       IT         70000      71667
  5       IT         80000      71667
  8       HR         50000      47500

エイリアスの使い分け

相関サブクエリでは、外側のテーブルをAS e1、内側のテーブルをAS e2のようにエイリアスで区別することで、どちらのテーブルを参照しているかが明確になります。

例2: 各カテゴリの最新レコードを抽出

各商品カテゴリごとに、最も新しい更新日のレコードのみを抽出します。

相関サブクエリ_最新レコード.sas 
/* サンプルデータ */
DATA products;
    INPUT product_id category $ price update_date :YYMMDD10.;
    FORMAT update_date YYMMDD10.;
    DATALINES;
101 Electronics 29999 2024-01-15
102 Electronics 19999 2024-02-20
103 Books 1500 2024-01-10
104 Books 2500 2024-03-01
105 Clothing 5000 2024-02-10
106 Clothing 7000 2024-03-15
;
RUN;

/* 各カテゴリの最新レコードのみ抽出 */
PROC SQL;
    SELECT product_id, category, price, update_date
    FROM products AS p1
    WHERE update_date =
          (SELECT MAX(update_date)
           FROM products AS p2
           WHERE p2.category = p1.category);
QUIT;

実行結果

product_id    category        price    update_date
   102       Electronics      19999    2024-02-20
   104       Books             2500    2024-03-01
   106       Clothing          7000    2024-03-15

4.2 Dictionaryテーブルでメタデータを活用

SASは、Dictionaryテーブルというメタデータテーブルを提供しています。これにより、ライブラリに存在するデータセット一覧、各データセットの列情報、マクロ変数の一覧など、SASシステム内の情報をSQLで取得できます。プログラムを動的に生成する際に非常に有用です。

主要なDictionaryテーブル

  • DICTIONARY.TABLES - データセット一覧と属性
  • DICTIONARY.COLUMNS - 列（変数）の詳細情報
  • DICTIONARY.MACROS - マクロ変数の一覧
  • DICTIONARY.OPTIONS - SASオプション設定情報

例3: WORKライブラリのデータセット一覧を取得

Dictionary_テーブル一覧.sas 
/* WORKライブラリのデータセット一覧を取得 */
PROC SQL;
    SELECT memname AS dataset_name,
           nobs AS num_rows,
           nvar AS num_vars,
           crdate AS created_date FORMAT=DATETIME20.
    FROM DICTIONARY.TABLES
    WHERE libname = 'WORK'
    ORDER BY memname;
QUIT;

実行結果例

dataset_name    num_rows    num_vars    created_date
EMPLOYEES           8           3        15MAR2024:10:30:45
PRODUCTS            6           4        15MAR2024:10:35:12
SALES_DATA        150           5        15MAR2024:09:15:30

例4: 特定データセットの列情報を取得

DICTIONARY.COLUMNSを使って、特定のデータセットにどのような列（変数）が存在するかを調べることができます。

Dictionary_列情報.sas 
/* EMPLOYEESデータセットの列情報を取得 */
PROC SQL;
    SELECT name AS column_name,
           type AS data_type,  /* char=文字型, num=数値型 */
           length,
           label
    FROM DICTIONARY.COLUMNS
    WHERE libname = 'WORK'
      AND memname = 'EMPLOYEES'
    ORDER BY varnum;  /* 定義順にソート */
QUIT;

Dictionaryテーブルの活用シーン

複数のデータセットに対して同じ処理を自動化したい場合や、プログラムで列名を動的に取得して処理を行いたい場合に、Dictionaryテーブルとマクロを組み合わせることで強力な自動化が実現できます。

4.3 INTO句によるマクロ変数の生成

PROC SQLのINTO句を使用すると、クエリの実行結果をマクロ変数に直接格納できます。これにより、SQLで取得した値を後続の処理で使用したり、動的にプログラムを制御することが可能になります。

INTO句の構文パターン

  • INTO :変数名 - 1つの値を格納
  • INTO :変数1, :変数2 - 複数の列をそれぞれ格納
  • INTO :変数1-:変数n - 複数行を連番マクロ変数に格納
  • INTO :変数 SEPARATED BY ' ' - 複数行をスペース区切りで1つの変数に格納

例5: 単一値の取得

集計結果など、1つの値をマクロ変数に格納します。

INTO句_単一値.sas 
/* 従業員の平均給与をマクロ変数に格納 */
PROC SQL NOPRINT;  /* NOPRINTオプションで結果を画面に表示しない */
    SELECT AVG(salary)
    INTO :avg_salary
    FROM employees;
QUIT;

/* マクロ変数を使用 */
%PUT 平均給与: &avg_salary;

DATA high_earners;
    SET employees;
    WHERE salary > &avg_salary;
RUN;

例6: 複数の値をリスト形式で取得

SEPARATED BY句を使って、複数の値をカンマ区切りなどで1つのマクロ変数にまとめます。

INTO句_リスト形式.sas 
/* すべての部門名をカンマ区切りで取得 */
PROC SQL NOPRINT;
    SELECT DISTINCT dept
    INTO :dept_list SEPARATED BY ', '
    FROM employees
    ORDER BY dept;
QUIT;

%PUT 部門リスト: &dept_list;
/* 出力例: 部門リスト: HR, IT, Sales */

例7: 複数行を連番マクロ変数に格納

各行を別々のマクロ変数に格納し、ループ処理で使用します。

INTO句_連番変数.sas 
/* 各部門を連番マクロ変数に格納 */
PROC SQL NOPRINT;
    SELECT DISTINCT dept
    INTO :dept1-:dept999  /* 最大999個の変数を用意 */
    FROM employees
    ORDER BY dept;

    /* 実際の部門数をカウント */
    %LET dept_count = &SQLOBS;  /* SQLOBSは抽出行数 */
QUIT;

/* 各部門ごとに処理を実行 */
%MACRO process_departments;
    %DO i = 1 %TO &dept_count;
        %PUT 処理中の部門: &&dept&i;

        /* 各部門のデータを出力 */
        DATA dept_&&dept&i;
            SET employees;
            WHERE dept = "&&dept&i";
        RUN;
    %END;
%MEND;

%process_departments;

自動マクロ変数SQLOBS

PROC SQL実行後、&SQLOBSには最後に実行されたクエリの行数が格納されます。INTO句で連番変数に格納した際の実際の個数を知るために活用できます。

実用例: Dictionaryテーブル + INTO句の組み合わせ

WORKライブラリのすべてのデータセット名をリストで取得し、それぞれに対して処理を実行します。

Dictionary_INTO組み合わせ.sas 
/* WORKライブラリのすべてのデータセット名を取得 */
PROC SQL NOPRINT;
    SELECT memname
    INTO :dataset1-:dataset999
    FROM DICTIONARY.TABLES
    WHERE libname = 'WORK'
    ORDER BY memname;

    %LET dataset_count = &SQLOBS;
QUIT;

/* 各データセットの行数をログに出力 */
%MACRO show_dataset_info;
    %DO i = 1 %TO &dataset_count;
        PROC SQL NOPRINT;
            SELECT COUNT(*) INTO :row_count
            FROM &&dataset&i;
        QUIT;

        %PUT データセット: &&dataset&i (行数: &row_count);
    %END;
%MEND;

%show_dataset_info;

実習 4-1: 相関サブクエリとINTO句の組み合わせ

各カテゴリの平均価格を超える商品の数をカウントし、マクロ変数に格納してログに出力してください。

手順

  1. 1 商品データ（カテゴリ、価格）を作成します。
  2. 2 相関サブクエリで各カテゴリの平均価格を超える商品を抽出します。
  3. 3 その件数をINTO句でマクロ変数に格納し、%PUTで表示します。

ヒント

/* ヒント: COUNT(*)とINTOの組み合わせ */
PROC SQL NOPRINT;
    SELECT COUNT(*) INTO :above_avg_count
    FROM products AS p1
    WHERE price > (SELECT AVG(price)
                   FROM products AS p2
                   WHERE p2.category = p1.category);
QUIT;

%PUT 平均超え商品数: &above_avg_count;

理解度確認クイズ

  1. Q1. 相関サブクエリと通常のサブクエリの違いは何ですか？

    回答を表示

    相関サブクエリは外側のクエリの各行ごとに実行され、外側のテーブルの列を参照できます。通常のサブクエリは1回だけ実行され、外側のテーブルの値を参照できません。

  2. Q2. DICTIONARY.COLUMNSで取得できる情報は何ですか？

    回答を表示

    データセットに含まれる列（変数）の名前、データ型（文字型/数値型）、長さ、ラベル、フォーマットなどの詳細情報を取得できます。

  3. Q3. INTO句で複数の値をカンマ区切りのリストにする構文は何ですか？

    回答を表示

    INTO :変数名 SEPARATED BY ', 'のように記述します。SEPARATED BY句で区切り文字を指定できます。

第4章のまとめ

  • 相関サブクエリにより、各グループの集計値と比較するなど、動的な条件抽出が可能になる。
  • Dictionaryテーブルを使うことで、データセット一覧や列情報などのメタデータをSQLで取得できる。
  • INTO句により、SQLの実行結果をマクロ変数に格納し、後続の処理で活用できる。
  • DictionaryテーブルとINTO句を組み合わせることで、完全自動化されたバッチ処理を実現できる。
