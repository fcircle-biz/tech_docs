配列（Array）と反復処理の応用

大量の変数を一括で変換するためのテクニックを磨きます。多次元配列の利用方法、DO WHILE / DO UNTILによる動的なループ制御、配列を用いた横持ちデータから縦持ちデータ（転置）への高度な変換を学習します。

この章で学ぶこと

  • 配列の基本構文と多次元配列の定義・参照方法
  • DO WHILE / DO UNTILによる動的なループ制御とその使い分け
  • 配列を用いた横持ちデータから縦持ちデータへの変換テクニック

2.1 配列の基本と多次元配列

SASの配列（ARRAY）は、同じ型の複数の変数をまとめて参照し、ループ処理で一括操作するための仕組みです。配列を使うことで、数十個、数百個の変数に対する同じ処理を、たった数行のコードで記述できます。

配列の基本構文

ARRAY 配列名[要素数] 変数リスト;
配列名の後に要素数を指定し、対象となる変数をリストで指定します。配列要素は配列名[インデックス]で参照します。

例1: 複数の変数を一括で変換

以下のコードは、4つの試験科目の点数を配列で一括処理し、欠損値を0に置き換える例です。

配列で欠損値を0に変換.sas 
/* サンプルデータ（一部欠損値あり） */
DATA test_scores;
    INPUT student_id math science english history;
    DATALINES;
101 80 . 75 90
102 . 85 80 .
103 90 95 . 88
;
RUN;

/* 配列を使って欠損値を0に一括変換 */
DATA test_scores_fixed;
    SET test_scores;

    /* 4科目を配列として定義 */
    ARRAY scores[4] math science english history;

    /* DOループで各科目をチェック */
    DO i = 1 TO 4;
        IF MISSING(scores[i]) THEN scores[i] = 0;
    END;

    DROP i;  /* ループカウンタは出力データセットから除外 */
RUN;

PROC PRINT DATA=test_scores_fixed;
RUN;

実行結果

Obs    student_id    math    science    english    history
  1         101         80        0         75         90
  2         102          0       85         80          0
  3         103         90       95          0         88

配列の便利な機能

配列の要素数を省略してARRAY scores[*]と書くと、SASが自動的に変数の個数をカウントします。また、DIM(配列名)関数で配列の要素数を取得できます。

多次元配列の活用

2次元以上の配列を定義することで、行列形式のデータを扱えます。たとえば、複数の学生の複数科目の点数を一括処理する場合に便利です。

多次元配列.sas 
/* 3人の学生、4科目のデータを2次元配列で処理 */
DATA test_matrix;
    /* 2次元配列: 3行（学生） x 4列（科目） */
    ARRAY scores[3,4] _TEMPORARY_;

    /* データを配列に格納（例） */
    scores[1,1] = 80; scores[1,2] = 75; scores[1,3] = 90; scores[1,4] = 85;
    scores[2,1] = 70; scores[2,2] = 80; scores[2,3] = 75; scores[2,4] = 90;
    scores[3,1] = 95; scores[3,2] = 90; scores[3,3] = 88; scores[3,4] = 92;

    /* 全体の平均を計算 */
    total = 0;
    count = 0;
    DO i = 1 TO 3;
        DO j = 1 TO 4;
            total + scores[i,j];
            count + 1;
        END;
    END;

    average = total / count;
    PUT "全体の平均点: " average 5.2;
RUN;

2.2 DO WHILE / DO UNTILによる動的なループ制御

DO WHILEとDO UNTILは、条件式の真偽に基づいてループを制御します。回数が事前に決まっていない繰り返し処理に適しています。

DO WHILEとDO UNTILの違い

  • DO WHILE (条件式); - 条件が真の間ループを続ける（条件を先にチェック）
  • DO UNTIL (条件式); - 条件が真になるまでループを続ける（条件を後でチェック）
  • DO UNTILは必ず1回は実行される点が異なります

例2: 複利計算（DO WHILE）

元本が2倍になるまでの年数を計算する例です。

複利計算.sas 
/* 元本が2倍になるまでの年数を計算 */
DATA compound_interest;
    principal = 1000000;  /* 元本100万円 */
    rate = 0.05;          /* 年利5% */
    target = principal * 2;

    balance = principal;
    year = 0;

    /* 残高が目標額に達するまでループ */
    DO WHILE (balance < target);
        year + 1;
        balance = balance * (1 + rate);
    END;

    PUT "元本が2倍になるまでの年数: " year "年";
    PUT "最終残高: " balance COMMA12.;
RUN;

例3: データ読み込み（DO UNTIL）

グループごとの合計を計算する際に、DO UNTILを使ってグループの最後まで読み込む例です。

DO UNTILでグループ集計.sas 
/* サンプルデータ */
DATA sales;
    INPUT customer_id amount;
    DATALINES;
101 1000
101 1500
102 2000
102 3000
102 500
103 1200
;
RUN;

/* 顧客ごとの合計をDO UNTILで計算 */
DATA customer_total;
    /* 最初の行を読み込まずにEOFフラグをチェック */
    DO UNTIL (last_customer);
        SET sales END=eof;
        BY customer_id;

        total + amount;

        /* グループの最後で出力 */
        IF LAST.customer_id THEN DO;
            OUTPUT;
            total = 0;
        END;

        last_customer = eof;
    END;
    STOP;
RUN;

無限ループに注意

DO WHILEやDO UNTILでは、条件式が必ず変化するようにコードを書く必要があります。条件が永久に真（または偽）のままだと、プログラムが終了しなくなります。

2.3 配列による横持ちデータから縦持ちデータへの変換

横持ち（Wide形式）のデータは、複数の列に値が並んでいる形式です（例: 月ごとの売上が横に並ぶ）。縦持ち（Long形式）では、項目名と値のペアを縦に並べます。配列とDOループを組み合わせることで、この変換を簡潔に記述できます。

データ形式の違い

横持ち（Wide形式）: 1行に複数の値が横に並ぶ
ID Jan Feb Mar
1 100 200 150

縦持ち（Long形式）: 項目名と値のペアで複数行に展開
ID Month Sales
1 Jan 100
1 Feb 200
1 Mar 150

例4: 月次売上データの変換

横持ち→縦持ち変換.sas 
/* 横持ちデータ（月ごとの売上が横に並ぶ） */
DATA sales_wide;
    INPUT store_id Jan Feb Mar Apr;
    DATALINES;
1 100 200 150 180
2 120 180 160 200
3 90  110 140 130
;
RUN;

/* 縦持ちデータに変換 */
DATA sales_long;
    SET sales_wide;

    /* 月別売上を配列として定義 */
    ARRAY monthly[4] Jan Feb Mar Apr;
    /* 月名を配列として定義 */
    ARRAY month_names[4] $ _TEMPORARY_ ('Jan' 'Feb' 'Mar' 'Apr');

    /* 各月のデータを1行ずつ出力 */
    DO i = 1 TO 4;
        month = month_names[i];
        sales = monthly[i];
        OUTPUT;  /* 月ごとに1行出力 */
    END;

    KEEP store_id month sales;
RUN;

PROC PRINT DATA=sales_long;
RUN;

  2         1        Feb      200
  3         1        Mar      150
  4         1        Apr      180
  5         2        Jan      120

  7         2        Mar      160
  8         2        Apr      200
  9         3        Jan       90

 12         3        Apr      130

PROC TRANSPOSEとの使い分け

SASにはPROC TRANSPOSEという転置専用のプロシジャがあります。しかし、配列を使った変換では、変換と同時に条件分岐や計算を加えることができ、より柔軟な処理が可能です。

実習 2-1: 配列を使った標準化処理

複数の試験科目のデータを配列で処理し、各科目の点数を標準化（平均0、標準偏差1）してください。

手順

  1. 1 複数の学生の複数科目の点数データを作成します。
  2. 2 PROC MEANSで各科目の平均と標準偏差を計算し、マクロ変数に格納します。
  3. 3 配列とDOループを使い、各科目の点数を標準化（z = (x - 平均) / 標準偏差）します。

ヒント

/* ヒント: 配列を2つ定義 */
ARRAY original[3] math science english;
ARRAY standardized[3] math_z science_z english_z;

DO i = 1 TO 3;
    standardized[i] = (original[i] - mean) / std;
END;

理解度確認クイズ

  1. Q1. 配列の要素数を自動的に決定させるには、どう記述しますか？

    回答を表示

    ARRAY 配列名[*] 変数リスト;のように、要素数の部分に*を指定します。SASが自動的に変数の個数をカウントして配列を定義します。

  2. Q2. DO WHILEとDO UNTILの違いは何ですか？

    回答を表示

    DO WHILEは条件が真の間ループを続け、条件を先にチェックするため、最初から条件が偽の場合は1回も実行されません。DO UNTILは条件が真になるまでループを続け、条件を後でチェックするため、必ず1回は実行されます。

  3. Q3. 横持ちデータを縦持ちデータに変換する際、OUTPUT文はどのタイミングで実行しますか？

    回答を表示

    DOループの中で、各配列要素を処理した後にOUTPUT文を実行します。これにより、1つの横持ちレコードから複数の縦持ちレコードが生成されます。

第2章のまとめ

  • 配列を使うことで、大量の変数に対する同じ処理を簡潔に記述できる。
  • DO WHILE / DO UNTILにより、条件に基づく動的なループ制御が可能になる。
  • 配列とDOループを組み合わせることで、横持ちデータから縦持ちデータへの変換を柔軟に実装できる。
  • 多次元配列を活用することで、行列形式のデータを効率的に処理できる。
