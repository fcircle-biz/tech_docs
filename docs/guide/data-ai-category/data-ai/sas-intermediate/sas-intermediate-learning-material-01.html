DATAステップの高度な制御

初級で学んだ内容を深掘りし、行をまたぐ処理や複雑な保持変数を学びます。RETAIN文と合計変数の高度な活用、LAG関数・DIF関数による前行データとの比較・演算、FIRST.変数とLAST.変数を用いたグループごとの複雑なフラグ立てと集計を習得します。

この章で学ぶこと

  • RETAIN文による変数値の保持と、累積合計・ランニング平均の計算方法
  • LAG関数・DIF関数を用いた前行データとの比較演算と変化量の算出
  • FIRST.変数とLAST.変数によるグループごとのフラグ立て・集計処理

1.1 RETAIN文と累積処理

DATAステップの通常の動作では、各オブザベーション（行）の処理が終わるごとに変数の値はリセットされます。しかし、RETAIN文を使用することで、特定の変数の値を次の行でも保持し続けることができます。これにより、累積合計やランニング平均など、行をまたいだ計算が可能になります。

RETAIN文の基本構文

RETAIN 変数名 初期値;
変数名を指定し、必要に応じて初期値を設定します。初期値を省略した場合、欠損値（.）で初期化されます。

例1: 売上の累積合計

以下のコードは、日ごとの売上データから累積売上を計算する例です。

累積売上計算.sas 
/* サンプルデータ */
DATA sales;
    INPUT date :YYMMDD10. amount;
    FORMAT date YYMMDD10.;
    DATALINES;
2024-01-01 1000
2024-01-02 1500
2024-01-03 2000
2024-01-04 1200
;
RUN;

/* 累積売上を計算 */
DATA sales_cumulative;
    SET sales;
    RETAIN cumulative_sales 0;  /* 累積売上の初期値を0に設定 */
    cumulative_sales + amount;  /* 累積加算（SUM文と同じ効果） */
RUN;

PROC PRINT DATA=sales_cumulative;
RUN;

実行結果

cumulative_sales列に、各日の累積売上が記録されます。 

Obs    date          amount    cumulative_sales
  1    2024-01-01     1000           1000
  2    2024-01-02     1500           2500
  3    2024-01-03     2000           4500
  4    2024-01-04     1200           5700

SUM文による簡潔な記述

変数 + 値; という記法は、自動的にRETAINを適用し、累積加算を行います。明示的にRETAINを書かなくても、この記法だけで累積計算が可能です。

1.2 LAG関数とDIF関数による前行データとの比較

LAG関数は、前のオブザベーションの変数値を取得します。DIF関数は、現在の値と前の値の差分を計算します。これらを使用することで、前日比の売上増減や、株価の変動など、時系列データの変化を分析できます。

LAG関数とDIF関数の構文

  • LAG(変数) - 1つ前の行の変数値を返す
  • LAG2(変数) - 2つ前の行の変数値を返す（最大LAG100まで）
  • DIF(変数) - 現在の値 - LAG(変数)

例2: 売上の前日比計算

前日比計算.sas 
/* 前日の売上と比較 */
DATA sales_comparison;
    SET sales;
    previous_amount = LAG(amount);      /* 前日の売上 */
    sales_diff = DIF(amount);           /* 売上の変化量 */

    /* パーセント変化率を計算（前日がある場合のみ） */
    IF NOT MISSING(previous_amount) THEN
        pct_change = (sales_diff / previous_amount) * 100;
RUN;

PROC PRINT DATA=sales_comparison;
    VAR date amount previous_amount sales_diff pct_change;
    FORMAT pct_change 8.2;
RUN;

実行結果

Obs    date          amount    previous_amount    sales_diff    pct_change
  1    2024-01-01     1000              .               .              .
  2    2024-01-02     1500            1000             500          50.00
  3    2024-01-03     2000            1500             500          33.33
  4    2024-01-04     1200            2000            -800         -40.00

LAG関数使用時の注意点

LAG関数はDATAステップ内で1回だけ呼び出すようにしてください。IF文の中で条件付きで呼び出すと、意図しない結果になることがあります。LAG関数の呼び出し結果を変数に代入し、その変数を条件判定で使用する方法が安全です。

1.3 FIRST.変数とLAST.変数によるグループ処理

BY文でデータをグループ化すると、SASは自動的にFIRST.変数とLAST.変数という一時変数を生成します。これらは、グループの最初と最後の行を識別するフラグとして機能し、グループごとの集計処理を効率的に実装できます。

FIRST.変数とLAST.変数

  • FIRST.変数名 - グループの最初の行で1、それ以外で0
  • LAST.変数名 - グループの最後の行で1、それ以外で0
  • BY文でソートされたデータが必要（PROC SORTまたはBY文にNOSORT EDオプション）

例3: 顧客ごとの購入金額合計

顧客別集計.sas 
/* サンプルデータ */
DATA purchases;
    INPUT customer_id purchase_amount;
    DATALINES;
101 1000
101 1500
102 2000
102 3000
102 500
103 1200
;
RUN;

/* 顧客IDでソート */
PROC SORT DATA=purchases;
    BY customer_id;
RUN;

/* 顧客ごとの合計購入額を計算 */
DATA customer_summary;
    SET purchases;
    BY customer_id;

    /* 各グループの最初で合計をリセット */
    IF FIRST.customer_id THEN total_amount = 0;

    /* 累積加算 */
    total_amount + purchase_amount;

    /* 各グループの最後の行のみ出力 */
    IF LAST.customer_id THEN OUTPUT;

    /* total_amountを保持 */
    RETAIN total_amount;
RUN;

PROC PRINT DATA=customer_summary;
    VAR customer_id total_amount;
RUN;

実行結果

Obs    customer_id    total_amount
  1         101            2500
  2         102            5500
  3         103            1200

グループ処理の典型パターン

グループごとの集計では、FIRST.で集計変数を初期化し、各行で累積計算を行い、LAST.で結果を出力する、という3ステップが基本パターンです。

実習 1-1: 月次売上データの移動平均計算

RETAIN文とLAG関数を組み合わせて、過去3か月の移動平均を計算してください。

手順

  1. 1 月ごとの売上データを作成します（データは自由に設定してください）。
  2. 2 LAG関数を使って過去2か月の売上を取得します。
  3. 3 当月と過去2か月の平均を計算し、moving_avg変数に代入します。

ヒント

/* ヒント: LAG1とLAG2を使用 */
lag1_sales = LAG(sales);
lag2_sales = LAG2(sales);

/* 3か月分のデータがそろったら平均を計算 */
IF NOT MISSING(lag2_sales) THEN
    moving_avg = MEAN(sales, lag1_sales, lag2_sales);

理解度確認クイズ

  1. Q1. RETAIN文を使わずに累積合計を計算する方法は何ですか？

    回答を表示

    SUM文（変数 + 値;）を使用すると、自動的にRETAINが適用され、累積加算が行われます。

  2. Q2. FIRST.変数が1になるのはどのタイミングですか？

    回答を表示

    BY文で指定した変数のグループが変わり、そのグループの最初のオブザベーション（行）が処理される時に1になります。それ以外の行では0です。

  3. Q3. LAG関数を条件分岐（IF文）の中で使用すると危険な理由は何ですか？

    回答を表示

    LAG関数は呼び出されるたびに内部キューを更新するため、条件によって呼び出し回数が変わると、期待した前の値が取得できなくなります。常に無条件で1回呼び出し、その結果を変数に保存してから条件判定を行うべきです。

第1章のまとめ

  • RETAIN文により、変数の値を次の行でも保持し、累積計算が可能になる。
  • LAG関数・DIF関数を使うことで、前行データとの比較や変化量の計算ができる。
  • FIRST.変数・LAST.変数により、グループごとの集計処理を効率的に記述できる。
  • これらの機能を組み合わせることで、複雑な時系列分析やグループ集計が実現できる。
