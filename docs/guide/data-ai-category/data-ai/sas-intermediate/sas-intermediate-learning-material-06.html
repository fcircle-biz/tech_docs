カスタムフォーマットとラベル（PROC FORMAT）

データの見せ方をプログラムから切り離して管理します。PROC FORMATによる独自フォーマットの作成（値の範囲によるグループ化）、外部データ（CSV/Excel）からフォーマットを自動生成する方法（CNTLIN=オプション）、マルチラベルフォーマットの活用を学びます。

この章で学ぶこと

  • PROC FORMATによる独自フォーマットの作成と、値の範囲によるグループ化
  • 外部データ（CSV/Excel）からフォーマットを自動生成する方法（CNTLIN=オプション）
  • マルチラベルフォーマットを用いた柔軟なデータ分類と集計

6.1 PROC FORMATの基礎

PROC FORMATは、データの表示形式をカスタマイズするための専用プロシジャです。値を見やすいラベルに変換したり、数値範囲を意味のあるカテゴリにグループ化したりできます。フォーマットはデータとは別に定義・保存されるため、同じフォーマットを複数のプログラムで再利用でき、データの見せ方を一元管理できます。

フォーマットを使う利点

  • コードと表示ラベルの分離（保守性向上）
  • 複数のプログラムでフォーマットを再利用可能
  • データを変更せずに表示だけを変更できる
  • 複雑な分類ルールを一箇所で管理

例1: 基本的なフォーマット作成

以下のコードは、性別コード（1/2）を「男性」「女性」というラベルに変換するフォーマットを作成します。

性別フォーマット.sas 
/* 性別フォーマットを作成 */
PROC FORMAT;
    VALUE genderfmt  /* 数値フォーマット（VALUE） */
        1 = '男性'
        2 = '女性'
        . = '不明';  /* 欠損値も定義可能 */
RUN;

/* サンプルデータ */
DATA employees;
    INPUT emp_id gender age;
    DATALINES;
101 1 35
102 2 28
103 1 42
104 2 31
105 . 29
;
RUN;

/* フォーマットを適用して表示 */
PROC PRINT DATA=employees;
    FORMAT gender genderfmt.;  /* フォーマット名の後にピリオド必須 */
RUN;

実行結果

Obs    emp_id    gender    age
  1      101      男性      35
  2      102      女性      28
  3      103      男性      42
  4      104      女性      31
  5      105      不明      29

文字フォーマットとの違い

VALUEは数値用、VALUE $は文字変数用です。文字変数の場合は、フォーマット名の前に$を付けます（例：VALUE $status_fmt 'A' = 'アクティブ';）。

6.2 値の範囲によるグループ化フォーマット

数値データを範囲でグループ化することで、年齢層、売上ランクなどのカテゴリを自動的に割り当てられます。範囲の境界は「-」で指定し、片側だけの指定（「LOW」「HIGH」）も可能です。

例2: 年齢層フォーマット

年齢層フォーマット.sas 
/* 年齢層フォーマット作成 */
PROC FORMAT;
    VALUE agefmt
        LOW-<20   = '10代以下'
        20-<30    = '20代'
        30-<40    = '30代'
        40-<50    = '40代'
        50-<60    = '50代'
        60-HIGH   = '60代以上';  /* HIGH = 最大値 */
RUN;

/* サンプルデータ */
DATA customers;
    INPUT customer_id age sales;
    DATALINES;
1001 18 15000
1002 25 23000
1003 32 45000
1004 47 38000
1005 55 52000
1006 68 21000
;
RUN;

/* フォーマット適用 */
PROC PRINT DATA=customers;
    FORMAT age agefmt.;
RUN;

/* 年齢層別の集計 */
PROC FREQ DATA=customers;
    TABLES age;
    FORMAT age agefmt.;
RUN;

範囲指定の記法

範囲の境界設定

記法 意味 例
a-b a以上b以下（両端含む） 20-29 → 20, 21, ..., 29
a-<b a以上b未満 20-<30 → 20～29.999...
LOW-a 最小値からa LOW-19 → 19以下すべて
a-HIGH aから最大値 60-HIGH → 60以上すべて

範囲の重複に注意

範囲が重複していると、最初にマッチした定義が適用されます。20-30と30-40のように境界が重なる場合は、20-<30と30-<40のように明示的に分けることを推奨します。

6.3 外部データからフォーマット生成（CNTLIN=オプション）

フォーマットの定義をExcelやCSVで管理している場合、CNTLIN=オプションを使って外部データからフォーマットを自動生成できます。大量のコード変換ルールがある場合や、非プログラマーがメンテナンスする場合に非常に便利です。

CNTLIN用データセットの構造

必須変数

  • FMTNAME: フォーマット名（最大32文字）
  • START: 範囲の開始値（または個別の値）
  • END: 範囲の終了値（個別の値の場合はSTARTと同じ）
  • LABEL: 表示ラベル（最大256文字）
  • TYPE: 'N'（数値）または 'C'（文字）

例3: ExcelからフォーマットをインポートExcelからフォーマットをインポート

外部フォーマット生成.sas 
/* ステップ1: フォーマット定義データの作成（実際はExcel/CSVから読み込み） */
DATA format_def;
    LENGTH FMTNAME $32 START $16 END $16 LABEL $256 TYPE $1;
    INPUT FMTNAME $ START $ END $ LABEL $ TYPE $;
    DATALINES;
DEPTFMT 100 100 営業部 N
DEPTFMT 200 200 開発部 N
DEPTFMT 300 300 総務部 N
DEPTFMT 400 400 人事部 N
RANKFMT A A S級社員 C
RANKFMT B B A級社員 C
RANKFMT C C B級社員 C
;
RUN;

/* ステップ2: CNTLIN=でフォーマット生成 */
PROC FORMAT CNTLIN=format_def;
RUN;

/* ステップ3: 生成されたフォーマットを確認 */
PROC FORMAT FMTLIB;
    SELECT DEPTFMT RANKFMT;
RUN;

/* サンプルデータでフォーマット適用 */
DATA staff;
    INPUT emp_id dept rank $;
    DATALINES;
1 100 A
2 200 B
3 300 C
4 400 A
;
RUN;

PROC PRINT DATA=staff;
    FORMAT dept DEPTFMT. rank $RANKFMT.;
RUN;

実際のExcelファイルからの読み込み例

Excelからフォーマット.sas 
/* Excelファイルからフォーマット定義を読み込み */
PROC IMPORT DATAFILE='/path/to/format_definitions.xlsx'
    OUT=format_source
    DBMS=XLSX REPLACE;
    SHEET="FormatDef";
    GETNAMES=YES;
RUN;

/* フォーマット生成 */
PROC FORMAT CNTLIN=format_source;
RUN;

フォーマットの保存と再利用

生成したフォーマットはPROC FORMAT LIBRARY=オプションで永続カタログに保存できます。これにより、他のSASセッションでも再利用可能になります。

6.4 マルチラベルフォーマット

マルチラベルフォーマット（MLF）は、1つの値を複数のグループに同時に分類できる高度な機能です。例えば、「25歳」という値を「20代」と「若年層」の両方に同時に分類することができます。これにより、同じデータを異なる視点で柔軟に集計できます。

マルチラベルフォーマットの用途

  • 年齢を「10代」「20代」と「若年層」「中年層」の両方で分類
  • 商品を「カテゴリ」と「価格帯」で同時に分類
  • 地域を「都道府県」と「エリア（東日本/西日本）」で分類

例4: 年齢のマルチラベル分類

マルチラベルフォーマット.sas 
/* マルチラベルフォーマット定義 */
PROC FORMAT;
    VALUE age_decade (MULTILABEL)  /* MULTILABELオプション */
        10-<20 = '10代'
        20-<30 = '20代'
        30-<40 = '30代'
        40-<50 = '40代'
        50-<60 = '50代'
        60-HIGH = '60代以上'

        /* 同じ値に別のラベルを追加 */
        10-<30 = '若年層'
        30-<50 = '中年層'
        50-HIGH = 'シニア層';
RUN;

/* サンプルデータ */
DATA survey;
    INPUT respondent_id age score;
    DATALINES;
1 18 75
2 25 82
3 32 68
4 47 91
5 55 78
6 68 85
;
RUN;

/* マルチラベルフォーマットで集計（PROC TABULATEで使用） */
PROC TABULATE DATA=survey;
    CLASS age / MLF;  /* MLFオプションでマルチラベル有効化 */
    VAR score;
    TABLE age, score*(MEAN*F=5.1);
    FORMAT age age_decade.;
RUN;

実行結果イメージ

----------------------------------------
| 年齢層        | 平均スコア         |
|---------------|-------------------|
| 10代          | 75.0              |
| 20代          | 82.0              |
| 30代          | 68.0              |
| 40代          | 91.0              |
| 50代          | 78.0              |
| 60代以上      | 85.0              |
| 若年層        | 78.5  ← 10代+20代 |
| 中年層        | 79.5  ← 30代+40代 |
| シニア層      | 81.5  ← 50代+60代 |
----------------------------------------

マルチラベルフォーマットの制約

マルチラベルフォーマットはPROC TABULATEおよびPROC MEANSでのみ使用可能です。PROC FREQやPROC PRINTでは通常のフォーマットとして動作します（最初にマッチしたラベルのみ表示）。

実習 6-1: 売上ランクフォーマットの作成

売上金額に応じて「S」「A」「B」「C」のランクを自動で割り当てるフォーマットを作成し、店舗別の売上ランク集計を行ってください。

手順

  1. 1 PROC FORMATで売上ランクフォーマットを定義します（S: 100万以上、A: 50万以上、B: 20万以上、C: 20万未満）。
  2. 2 店舗ごとの売上データを作成します（データは自由に設定してください）。
  3. 3 PROC FREQでランクごとの店舗数を集計します。

ヒント

/* ヒント: 範囲指定フォーマット */
PROC FORMAT;
    VALUE salesrank
        1000000-HIGH = 'Sランク'
        500000-<1000000 = 'Aランク'
        200000-<500000 = 'Bランク'
        LOW-<200000 = 'Cランク';
RUN;

理解度確認クイズ

  1. Q1. 文字変数用のフォーマットを定義する際、VALUE文にどのような記号を付ける必要がありますか？

    回答を表示

    $記号を付けます。例：VALUE $genderfmt 'M' = '男性' 'F' = '女性';

  2. Q2. 外部データからフォーマットを生成する際に使用するオプション名は何ですか？

    回答を表示

    CNTLIN=オプションです。フォーマット定義を含むデータセットを指定します（例：PROC FORMAT CNTLIN=format_def;）。

  3. Q3. マルチラベルフォーマットを有効に活用できるプロシジャは何ですか？

    回答を表示

    PROC TABULATEとPROC MEANSです。これらのプロシジャでCLASS文にMLFオプションを指定することで、マルチラベルフォーマットが有効になります。

第6章のまとめ

  • PROC FORMATにより、データの表示形式をプログラムと分離して管理できる。
  • 範囲指定フォーマットにより、数値を意味のあるカテゴリに自動分類できる。
  • CNTLIN=オプションにより、ExcelやCSVからフォーマットを動的に生成できる。
  • マルチラベルフォーマットにより、1つの値を複数の視点で同時に分類・集計できる。
