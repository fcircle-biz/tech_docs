統計解析の実践：モデル構築

予測や因果関係の特定に挑戦します。単回帰分析と重回帰分析（PROC REG）、ロジスティック回帰分析（PROC LOGISTIC）、モデルの適合度（AIC, R2乗値）の確認と変数選択法を習得します。

この章で学ぶこと

  • 単回帰分析と重回帰分析（PROC REG）による予測モデルの構築方法
  • ロジスティック回帰分析（PROC LOGISTIC）による分類モデルの構築と予測確率の算出
  • モデルの適合度（AIC, R2乗値）の確認と変数選択法（ステップワイズ法など）

9.1 単回帰分析と重回帰分析（PROC REG）

回帰分析は、目的変数（従属変数）を説明変数（独立変数）によって予測するための統計手法です。単回帰分析は1つの説明変数を用いる最もシンプルな形式で、重回帰分析は複数の説明変数を用いることで、より複雑な関係を表現できます。

回帰分析の基本式

単回帰: Y = β0 + β1 * X + ε

重回帰: Y = β0 + β1 * X1 + β2 * X2 + ... + βn * Xn + ε

Y: 目的変数、X: 説明変数、β: 回帰係数、ε: 誤差

例1: 単回帰分析（広告費と売上の関係）

広告費（説明変数）が売上（目的変数）にどの程度影響を与えるかを分析します。

単回帰分析.sas 
/* サンプルデータ */
DATA sales_ad;
    INPUT ad_cost sales;
    DATALINES;
10 150
15 180
20 230
25 250
30 300
35 320
40 380
;
RUN;

/* 単回帰分析: 売上 = f（広告費） */
PROC REG DATA=sales_ad PLOTS=ALL;
    MODEL sales = ad_cost;  /* 目的変数 = 説明変数 */
    TITLE "単回帰分析: 広告費と売上の関係";
RUN;
QUIT;

出力結果の読み方

PROC REGの出力には、以下の重要な指標が含まれます：

  • R-Square（決定係数）: モデルが目的変数の変動をどの程度説明できるか（0-1の範囲、1に近いほど良い）
  • Parameter Estimates（回帰係数）: 切片（Intercept）と各説明変数の係数
  • p-Value: 各係数が統計的に有意かどうか（0.05未満なら有意）
  • F-Value: モデル全体の有意性を示す検定統計量

例2: 重回帰分析（複数の説明変数）

売上を、広告費・店舗面積・立地スコアの3つの変数で説明するモデルを構築します。

重回帰分析.sas 
/* サンプルデータ */
DATA store_sales;
    INPUT ad_cost store_size location_score sales;
    DATALINES;
10 50 7 150
15 60 8 180
20 55 6 230
25 70 9 250
30 80 8 300
35 75 7 320
40 90 9 380
;
RUN;

/* 重回帰分析: 売上 = f（広告費, 店舗面積, 立地スコア） */
PROC REG DATA=store_sales;
    MODEL sales = ad_cost store_size location_score;
    TITLE "重回帰分析: 売上予測モデル";
RUN;
QUIT;

多重共線性に注意

説明変数同士が強く相関している場合、多重共線性の問題が発生します。VIF（分散拡大係数）オプションで診断できます。VIF > 10 の場合は注意が必要です。 

MODEL sales = ad_cost store_size location_score / VIF;

9.2 ロジスティック回帰分析（PROC LOGISTIC）

ロジスティック回帰分析は、目的変数が2値（0 or 1、成功 or 失敗、購入 or 非購入など）の場合に使用する分類モデルです。通常の回帰分析と異なり、確率（0～1の範囲）を予測します。

ロジスティック回帰の応用例

  • 顧客の購入有無を予測（マーケティング）
  • 病気の発症リスクを予測（医療）
  • ローンの返済可否を予測（与信審査）
  • メールがスパムかどうかを分類（セキュリティ）

例3: 顧客の購入予測モデル

年齢・収入・過去の購入回数から、新商品を購入するかどうか（1=購入、0=非購入）を予測します。

ロジスティック回帰.sas 
/* サンプルデータ */
DATA customer_purchase;
    INPUT age income past_purchases purchased;
    DATALINES;
25 300 1 0
30 450 3 1
35 500 5 1
40 350 2 0
45 600 8 1
50 700 10 1
28 320 0 0
;
RUN;

/* ロジスティック回帰分析 */
PROC LOGISTIC DATA=customer_purchase DESCENDING;
    MODEL purchased = age income past_purchases;
    TITLE "ロジスティック回帰分析: 顧客購入予測モデル";
RUN;

DESCENDINGオプションの重要性

DESCENDINGオプションを指定すると、目的変数の値が大きい方（1）を「イベント発生」として扱います。省略すると0をイベントとして扱うため、オッズ比の解釈が逆になってしまいます。

オッズ比の解釈

ロジスティック回帰の出力には、オッズ比（Odds Ratio）が表示されます。オッズ比が1より大きい場合、その変数が増加するとイベント発生確率が上昇することを意味します。

  • オッズ比 = 1.5: 説明変数が1単位増加すると、購入オッズが1.5倍になる
  • オッズ比 = 0.8: 説明変数が1単位増加すると、購入オッズが0.8倍（20%減少）になる
  • オッズ比 = 1.0: その変数は購入確率に影響を与えない

9.3 モデルの適合度評価と変数選択法

どんなに複雑なモデルを作っても、過学習（Overfitting）により、新しいデータに対する予測精度が低くなることがあります。適切な変数を選択し、モデルの適合度を評価することが重要です。

主な適合度指標

  • AIC（赤池情報量規準）: 小さいほど良いモデル。モデルの複雑さを考慮した評価指標
  • BIC（ベイズ情報量規準）: AICより変数の数に対してペナルティが大きい
  • Adjusted R-Square（自由度調整済み決定係数）: 説明変数の数が増えても安易に上昇しない
  • C統計量（ロジスティック）: 0.5～1.0の範囲で、0.7以上なら許容できるモデル

変数選択法

多数の候補変数から最適な組み合わせを選ぶ方法として、以下の自動選択法があります。

ステップワイズ法.sas 
/* 重回帰分析でステップワイズ法を適用 */
PROC REG DATA=store_sales;
    MODEL sales = ad_cost store_size location_score
                  / SELECTION=STEPWISE SLE=0.05 SLS=0.05;
    TITLE "ステップワイズ法による変数選択";
RUN;
QUIT;

/* ロジスティック回帰でステップワイズ法を適用 */
PROC LOGISTIC DATA=customer_purchase DESCENDING;
    MODEL purchased = age income past_purchases
                      / SELECTION=STEPWISE SLE=0.05 SLS=0.05;
    TITLE "ステップワイズ法による変数選択（ロジスティック）";
RUN;

変数選択法の種類

  • SELECTION=FORWARD - 前進選択法（変数を1つずつ追加）
  • SELECTION=BACKWARD - 後退選択法（全変数から1つずつ削除）
  • SELECTION=STEPWISE - ステップワイズ法（追加と削除を両方実施）

モデルの比較

複数のモデルを構築した場合、AICやBICを比較して最適なモデルを選択します。

ホールドアウト検証のすすめ

データを「訓練データ」と「テストデータ」に分割し、訓練データでモデルを構築、テストデータで予測精度を検証する方法（ホールドアウト検証）が推奨されます。これにより過学習を防げます。

実習 9-1: 住宅価格予測モデルの構築

住宅の価格を、築年数・面積・駅からの距離の3つの変数で予測する重回帰モデルを構築してください。

手順

  1. 1 築年数・面積・駅距離・価格のデータを作成します（10件以上推奨）。
  2. 2 PROC REGで重回帰分析を実行し、R-Squareと各係数のp-Valueを確認します。
  3. 3 ステップワイズ法を用いて、最適な変数の組み合わせを探します。

ヒント

/* ヒント: PROC REGのステップワイズ法 */
PROC REG DATA=housing;
    MODEL price = age area distance
          / SELECTION=STEPWISE SLE=0.05 SLS=0.05;
RUN;
QUIT;

理解度確認クイズ

  1. Q1. 重回帰分析のR-Squareが0.85だった場合、どのように解釈しますか？

    回答を表示

    目的変数の変動の85%がモデルの説明変数によって説明できることを意味します。一般的に0.7以上なら良好なモデルとされますが、分野や目的により異なります。

  2. Q2. ロジスティック回帰でDESCENDINGオプションを付ける理由は何ですか？

    回答を表示

    目的変数の値が大きい方（通常は1）を「イベント発生」として扱うためです。省略すると0をイベントとして扱い、オッズ比の解釈が逆になってしまいます。

  3. Q3. AICとBICの違いは何ですか？

    回答を表示

    どちらもモデルの適合度を評価する指標ですが、BICはAICよりも変数の数に対するペナルティが大きいため、よりシンプルなモデルを選択する傾向があります。どちらも小さい値が良いモデルを示します。

第9章のまとめ

  • PROC REGにより、単回帰・重回帰分析を実行し、目的変数を説明変数で予測するモデルを構築できる。
  • PROC LOGISTICにより、2値の目的変数を予測するロジスティック回帰分析が可能になる。
  • AIC、BIC、R-Squareなどの適合度指標により、モデルの良し悪しを評価できる。
  • ステップワイズ法などの変数選択法により、最適な説明変数の組み合わせを自動選択できる。
