第5章

SASマクロ：自動化と動的生成

「プログラムがプログラムを書く」段階へ進みます。マクロ関数（%SCAN, %SUBSTR, %INDEX）による文字列操作、CALL SYMPUTXによるデータの内容に応じたマクロ変数の動的生成、%DOループと%IF-%THENを用いた条件による処理の切り替えを習得します。

この章で学ぶこと

  • マクロ変数の定義と参照、マクロ関数（%SCAN, %SUBSTR, %INDEX）による文字列操作
  • CALL SYMPUTXによるデータの内容からマクロ変数を動的に生成する方法
  • %MACROプロシジャによるマクロプログラムの作成と%DOループ・%IF-%THEN条件分岐

5.1 マクロ変数の基本とマクロ関数

マクロ変数は、プログラム実行前にテキストとして置き換えられる変数です。データステップの変数とは異なり、マクロ変数はプログラムコード自体を動的に生成するために使用されます。これにより、同じ処理を複数のデータセットに対して繰り返す、条件によってコードを切り替えるなど、高度な自動化が実現できます。

マクロ変数の定義と参照

  • %LET 変数名 = 値; - マクロ変数を定義
  • &変数名 - マクロ変数を参照（値に置き換えられる）
  • &変数名.. - 後続の文字と区別するためのピリオド

例1: マクロ変数の基本的な使い方

マクロ変数基本.sas 
/* マクロ変数の定義 */
%LET year = 2024;
%LET month = 03;
%LET dataset_name = sales_data;

/* マクロ変数を使用してデータセットを読み込む */
DATA work.report;
    SET mylib.&dataset_name.;
    WHERE year = &year AND month = &month;
    /* &year は 2024 に、&month は 03 に置き換えられる */
RUN;

/* タイトルにもマクロ変数を使用可能 */
TITLE "&year年&month月の売上レポート";
PROC PRINT DATA=work.report;
RUN;

主要なマクロ関数

マクロ関数は、マクロ変数の内容を加工するための関数です。文字列操作に特化しており、%で始まります。

代表的なマクロ関数

関数 説明 例
%SCAN(文字列, n) n番目の単語を抽出 %SCAN(A B C, 2) → B
%SUBSTR(文字列, 位置, 長さ) 部分文字列を抽出 %SUBSTR(ABCDE, 2, 3) → BCD
%INDEX(文字列, 検索文字列) 検索文字列の位置を返す %INDEX(ABCDE, CD) → 3
%UPCASE(文字列) 大文字に変換 %UPCASE(abc) → ABC
%LENGTH(文字列) 文字列の長さを返す %LENGTH(ABCDE) → 5

例2: マクロ関数による文字列操作

マクロ関数活用.sas 
/* ファイルパスからファイル名を抽出 */
%LET filepath = /data/sales/monthly/2024_03_sales.csv;

/* 最後のスラッシュ以降を抽出してファイル名を取得 */
%LET filename = %SCAN(&filepath, -1, /);  /* 2024_03_sales.csv */
%PUT ファイル名: &filename;

/* ファイル名から年月を抽出 */
%LET year_month = %SUBSTR(&filename, 1, 7);  /* 2024_03 */
%PUT 年月: &year_month;

/* 拡張子を確認 */
%LET ext_pos = %INDEX(&filename, .csv);
%PUT 拡張子位置: &ext_pos;

%PUT文でデバッグ

%PUT文を使うと、マクロ変数の内容をログに出力できます。マクロプログラムのデバッグに非常に有効です。

5.2 CALL SYMPUTXによる動的マクロ変数生成

CALL SYMPUTXは、DATAステップの中でマクロ変数を作成・更新するルーチンです。これにより、データの内容に基づいてマクロ変数を動的に生成し、後続の処理で活用できます。例えば、「最新の日付」「レコード件数」「条件に合致したグループ名」などをマクロ変数に格納し、次のプロシジャやDATAステップで参照できます。

CALL SYMPUTXの構文

CALL SYMPUTX('マクロ変数名', 値);
第1引数にマクロ変数名（文字列）、第2引数に代入する値を指定します。

例3: 最新日付をマクロ変数に格納

動的マクロ変数生成.sas 
/* サンプルデータ */
DATA transactions;
    INPUT transaction_date :YYMMDD10. amount;
    FORMAT transaction_date YYMMDD10.;
    DATALINES;
2024-01-15 1000
2024-02-20 1500
2024-03-10 2000
;
RUN;

/* 最新の取引日をマクロ変数に格納 */
DATA _NULL_;
    SET transactions END=last_row;
    /* END=オプションで最後の行を識別 */
    IF last_row THEN DO;
        CALL SYMPUTX('latest_date', transaction_date);
        CALL SYMPUTX('latest_amount', amount);
    END;
RUN;

/* マクロ変数を確認 */
%PUT 最新取引日: &latest_date;
%PUT 最新取引金額: &latest_amount;

例4: レコード件数をマクロ変数に格納

件数カウント.sas 
/* レコード件数をマクロ変数に格納 */
DATA _NULL_;
    SET transactions NOBS=total_count;
    /* NOBS=オプションで総件数を取得 */
    CALL SYMPUTX('record_count', total_count);
    STOP;  /* 1行目だけ処理して終了 */
RUN;

%PUT データセットのレコード件数: &record_count;

/* 件数によって処理を分岐 */
%MACRO process_data;
    %IF &record_count > 100 %THEN %DO;
        %PUT 大量データのため、サンプリング処理を実行します;
    %END;
    %ELSE %DO;
        %PUT 全件処理を実行します;
    %END;
%MEND;

%process_data;

CALL SYMPUTとCALL SYMPUTXの違い

古いCALL SYMPUTは前後の空白をトリミングしませんが、CALL SYMPUTXは自動的にトリミングします。通常はCALL SYMPUTXを使用することを推奨します。

5.3 マクロプログラム（%MACRO）の作成

%MACROプロシジャを使うと、繰り返し使用する処理を1つのマクロとしてまとめることができます。パラメータを受け取って処理内容を変えたり、%DOループで反復処理を行ったり、%IF-%THEN文で条件分岐を実装できます。これにより、同じパターンの処理を複数のデータセットに適用する、といった高度な自動化が実現します。

マクロプログラムの基本構文

%MACRO マクロ名(パラメータ1, パラメータ2);
    /* マクロの処理内容 */
%MEND マクロ名;

/* マクロの呼び出し */
%マクロ名(値1, 値2);

例5: パラメータ付きマクロの作成

パラメータ付きマクロ.sas 
/* データセットの基本統計を出力するマクロ */
%MACRO basic_stats(dataset, var);
    TITLE "データセット &dataset の変数 &var の統計量";
    PROC MEANS DATA=&dataset N MEAN STD MIN MAX;
        VAR &var;
    RUN;
    TITLE;  /* タイトルをクリア */
%MEND basic_stats;

/* マクロの実行例 */
%basic_stats(sashelp.class, height);
%basic_stats(sashelp.class, weight);

例6: %DOループによる反復処理

DOループマクロ.sas 
/* 複数年のデータを連続処理するマクロ */
%MACRO process_years(start_year, end_year);
    %DO year = &start_year %TO &end_year;
        DATA sales_&year;
            SET raw_data;
            WHERE year_col = &year;
        RUN;

        PROC PRINT DATA=sales_&year;
            TITLE "Year &year Sales Report";
        RUN;
    %END;
%MEND process_years;

/* 2020年から2024年まで処理 */
%process_years(2020, 2024);

例7: %IF-%THENによる条件分岐

条件分岐マクロ.sas 
/* 環境に応じて処理を切り替えるマクロ */
%MACRO export_data(environment);
    %IF %UPCASE(&environment) = DEV %THEN %DO;
        %PUT 開発環境: ローカルディレクトリに出力;
        LIBNAME outlib "C:/dev/output";
    %END;
    %ELSE %IF %UPCASE(&environment) = PROD %THEN %DO;
        %PUT 本番環境: サーバーディレクトリに出力;
        LIBNAME outlib "/prod/output";
    %END;
    %ELSE %DO;
        %PUT エラー: 未知の環境 (&environment);
        %RETURN;  /* マクロを終了 */
    %END;

    /* データのエクスポート */
    DATA outlib.result;
        SET work.processed_data;
    RUN;
%MEND export_data;

%export_data(DEV);

マクロの再利用性を高めるコツ

  • パラメータにデフォルト値を設定する（例: %MACRO test(var=age)）
  • 処理の開始・終了をログに出力する（%PUTを活用）
  • エラーチェックを入れる（%IF %SYSFUNC(EXIST(&dataset)) = 0 など）

実習 5-1: 複数データセットの一括処理マクロ

マクロを使って、複数のデータセット（sales_2023, sales_2024など）に対して同じ処理を一括実行してください。

手順

  1. 1 サンプルデータとして sales_2023, sales_2024 の2つのデータセットを作成します。
  2. 2 データセット名をパラメータとして受け取り、基本統計量を出力するマクロを作成します。
  3. 3 %DOループで2023年から2024年まで処理を繰り返します。

ヒント

%MACRO analyze_sales(start, end);
    %DO year = &start %TO &end;
        PROC MEANS DATA=sales_&year;
            TITLE "Sales Analysis for &year";
        RUN;
    %END;
%MEND;

%analyze_sales(2023, 2024);

動作確認: マクロ変数の展開確認

マクロプログラムが正しく動作しているか確認しましょう。

確認項目

  • ログに MPRINT オプションを有効にして、展開されたコードを確認しましたか？
  • %PUT でマクロ変数の値が期待通りか確認しましたか？
  • エラーや警告がログに出力されていないか確認しましたか？

MPRINT有効化: OPTIONS MPRINT; をプログラムの先頭に記述すると、マクロが展開された実際のSASコードがログに表示されます。

理解度確認クイズ

  1. Q1. マクロ変数を参照する際に使用する記号は何ですか？

    回答を表示

    &（アンパサンド）を変数名の前に付けます。例: &year

  2. Q2. %SCANマクロ関数の第2引数に負の値を指定するとどうなりますか？

    回答を表示

    負の値を指定すると、末尾からカウントされます。%SCAN(A B C, -1) は最後の単語「C」を返します。

  3. Q3. CALL SYMPUTXを使う主なメリットは何ですか？

    回答を表示

    DATAステップの処理結果（最新日付、レコード件数など）をマクロ変数に格納し、後続のプログラムで参照できる点です。データの内容に応じて動的にプログラムの挙動を変更できます。

  4. Q4. %MACROと%MENDの間に記述できる文は何ですか？

    回答を表示

    通常のSASプログラム（DATAステップ、プロシジャ）に加え、マクロ専用の制御文（%DO, %IF-%THEN-%ELSE, %LET, %PUT など）を記述できます。

第5章のまとめ

  • マクロ変数とマクロ関数により、プログラムコード自体を動的に生成できる。
  • CALL SYMPUTXで、データの内容からマクロ変数を動的に生成し、後続処理に活用できる。
  • %MACROプロシジャにより、再利用可能なマクロプログラムを作成できる。
  • %DOループと%IF-%THENにより、反復処理と条件分岐を実装し、複雑な自動化を実現できる。
