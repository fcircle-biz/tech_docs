SAS Viyaの概要とアーキテクチャ

SAS Viyaとは何か、従来のSASとの違い、クラウドネイティブアーキテクチャの特徴、CASエンジンの基本概念と分散処理の仕組みを理解します。

この章で学ぶこと

  • 従来のSAS（SAS 9.x）との主な違いを把握する
  • クラウドネイティブアーキテクチャの特徴を学ぶ

1.1 SAS Viyaとは

SAS Viyaは、SAS Instituteが開発した次世代のクラウドベース分析プラットフォームです。2016年に登場し、ビッグデータ、AI/機械学習、リアルタイム分析、データ可視化を統合的に扱える環境を提供します。

従来のSAS 9.xがクライアント/サーバーモデルで動作していたのに対し、SAS Viyaはクラウドネイティブ設計を採用し、マイクロサービスアーキテクチャとコンテナ技術（Kubernetes）をベースに構築されています。

SAS Viyaの名前の由来

「Viya」は「見る（View）」と「ビジョン（Vision）」を組み合わせた造語で、データから洞察を「見る」力を象徴しています。

SAS Viyaの主な特徴

  • クラウドネイティブ: AWS、Azure、GCPなどのクラウド環境に最適化
  • 高速処理: インメモリ処理により従来の数十倍〜数百倍の速度を実現
  • スケーラビリティ: データ量やユーザー数に応じて自動的にリソース拡張
  • 多言語対応: SAS、Python、R、Lua、Javaなど多様な言語をサポート
  • ビジュアル操作: ノーコード/ローコードで高度な分析が可能
  • API駆動: REST APIによる外部システムとの連携が容易

1.2 従来のSAS（SAS 9.x）との違い

SAS Viyaは従来のSAS 9.xから大きく進化しました。以下の表で主な違いを比較します。

項目 従来のSAS（SAS 9.x） SAS Viya
アーキテクチャ クライアント/サーバー クラウドネイティブ、マイクロサービス
処理エンジン SAS Kernel CAS（Cloud Analytic Services）
データ処理 ディスクベース インメモリ、分散処理
スケーラビリティ 垂直スケール 水平スケール（ノード追加）
インターフェース SAS Studio、Enterprise Guide Web UI、REST API、多言語対応
言語サポート SAS SAS、Python、R、Lua、Java
機械学習 Enterprise Miner Model Studio（AutoML対応）
可視化 PROC SGPLOT等 Visual Analytics（インタラクティブ）
デプロイメント オンプレミス中心 クラウド/オンプレミス/ハイブリッド

重要ポイント

SAS Viyaは従来のSASコードとの互換性を保ちながら、クラウド時代の要求に応える新世代プラットフォームです。既存のSASプログラムの多くはViya上でも動作しますが、CASエンジンを活用することで劇的な性能向上が得られます。

1.3 クラウドネイティブアーキテクチャ

SAS Viyaはクラウドネイティブアーキテクチャを採用し、現代のクラウド環境に最適化されています。

マイクロサービスアーキテクチャ

SAS Viyaは機能ごとに独立したマイクロサービスで構成されています。各サービスは個別にスケールでき、障害が発生しても他のサービスに影響を与えません。

コンテナ技術とKubernetes

SAS Viyaの各マイクロサービスはDockerコンテナとして動作し、Kubernetesによってオーケストレーションされます。これにより以下のメリットがあります。

  • 自動スケーリング: 負荷に応じてリソースを自動調整
  • 高可用性: コンテナが停止しても自動再起動
  • ポータビリティ: 異なるクラウド環境間で容易に移行
  • DevOps統合: CI/CDパイプラインとの連携が容易

1.4 CAS（Cloud Analytic Services）エンジン

CAS（Cloud Analytic Services）は、SAS Viyaの心臓部となるインメモリ分散処理エンジンです。従来のSASカーネルに代わる、次世代の高速処理エンジンです。

CASの主な機能

  • インメモリ処理: データを全てメモリ上に保持し高速処理
  • 分散コンピューティング: 複数ノードで並列処理を実行
  • マルチユーザー対応: 複数ユーザーが同時にアクセス可能
  • 多様なデータソース: Hadoop、クラウドストレージ、DBなど幅広く対応

CASアクションとは

CASエンジンでは、従来のSASプロシジャに代わりCASアクションという単位で処理を実行します。アクションはAPI経由で呼び出され、SAS、Python、R、Javaなど多様な言語から利用できます。

cas_action_example.sas 
/* CASセッションの開始 */
cas mysession;

/* データをCASにロード */
proc casutil;
  load data=sashelp.cars casout="cars";
quit;

/* CASアクションで集計処理 */
proc cas;
  simple.summary /
    table={name="cars"}
    inputs={"msrp", "horsepower"}
    casout={name="car_summary", replace=true};
quit;

/* 結果を表示 */
proc print data=car_summary;
run;

/* CASセッションの終了 */
cas mysession terminate;

ヒント

CASアクションは従来のPROCよりも高速に動作します。特に大量データの処理では、数十倍〜数百倍の速度向上が期待できます。第3章でCASの詳細を学びます。

1.5 SAS Viyaのエコシステムと主要コンポーネント

SAS Viyaは多数のコンポーネントで構成され、包括的な分析プラットフォームを提供します。

SAS Studio

WebベースのSAS開発環境。SASコードの記述、実行、デバッグが可能。Jupyter Notebookのような操作感。

Visual Analytics

ノーコードでインタラクティブなダッシュボードを作成。ドラッグ＆ドロップで高度な可視化が可能。

Model Studio

機械学習モデルの構築・評価・デプロイを一元管理。AutoML機能により自動でモデル選択・チューニング。

SAS Drive

コンテンツ管理システム。レポート、データセット、プログラムを一元管理し、チームで共有可能。

理解度確認クイズ

  1. Q1. SAS Viyaの「Viya」という名前の由来は何ですか？

    回答を表示

    「見る（View）」と「ビジョン（Vision）」を組み合わせた造語で、データから洞察を「見る」力を象徴しています。

  2. Q2. SAS Viyaの心臓部となる処理エンジンの名前は何ですか？

    回答を表示

    CAS（Cloud Analytic Services）エンジンです。インメモリ分散処理により高速な分析を実現します。

  3. Q3. 従来のSAS 9.xと比較したSAS Viyaの主な利点を3つ挙げてください。

    回答を表示

    ①インメモリ処理による高速性、②水平スケーリングによる拡張性、③REST APIと多言語対応による柔軟性（他にも複数の正解があります）

  4. Q4. SAS Viyaのアーキテクチャで採用されているコンテナオーケストレーション技術は何ですか？

    回答を表示

    Kubernetesです。各マイクロサービスはDockerコンテナとして動作し、Kubernetesが自動スケーリングや障害回復を管理します。
