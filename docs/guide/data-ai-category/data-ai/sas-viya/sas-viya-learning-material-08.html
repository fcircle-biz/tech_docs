SAS Model Studioによる機械学習（基礎）

ノーコード/ローコードで機械学習モデルを構築する方法を学びます。自動機械学習（AutoML）、データパイプライン、モデル比較の基本を習得します。

この章で学ぶこと

  • SAS Model Studioの概要とインターフェースを理解する
  • プロジェクトの作成とデータソースの設定方法を学ぶ
  • 自動機械学習（AutoML）を使ったモデル構築を体験する
  • データパイプラインの基本構造を理解する

8.1 SAS Model Studioとは

SAS Model Studioは、SAS Viyaに統合された機械学習モデル開発環境です。コードを書かずに、視覚的なパイプラインを構築しながらモデルを作成できます。

従来のSAS Enterprise Minerに代わる次世代ツールとして、AutoML（自動機械学習）機能を備えており、データサイエンスの専門知識がなくても高品質なモデルを構築できます。

Model Studioの主な機能

  • AutoML: 自動的に複数のアルゴリズムを試行し、最適なモデルを選択
  • パイプライン構築: ドラッグ＆ドロップで前処理から評価までを一貫構築
  • モデル比較: 複数モデルの性能を自動的に比較・ランキング
  • モデル解釈: LIME、SHAPなどの説明可能AI手法をサポート
  • モデル展開: ワンクリックで本番環境へデプロイ可能
  • コード生成: パイプラインからSAS/Pythonコードを自動生成

Model Studioの位置づけ

Model Studioは「市民データサイエンティスト」向けのツールで、コーディングスキルがなくても機械学習モデルを構築できます。一方で、SAS StudioやPythonを使ったカスタムコーディングも可能で、柔軟性も確保されています。

Model Studioは3つの主要エリアで構成されています。

各エリアの役割

ノードパレット

パイプラインに追加できるノード（処理単位）が一覧表示されます。カテゴリ別に整理されており、ドラッグ＆ドロップでキャンバスに配置します。

パイプラインキャンバス

データフローを視覚的に設計するエリアです。ノードを配置して矢印で接続し、データ処理の流れを構築します。パイプライン全体を実行することもできます。

プロパティペイン

選択したノードの詳細設定を行うエリアです。パラメータの調整、データプレビュー、実行結果の確認ができます。

8.3 プロジェクトの作成

Model Studioでは、機械学習プロジェクトを作成して、複数のパイプラインを管理します。

プロジェクト作成の手順

実習 8-1: 初めてのModel Studioプロジェクト

顧客解約予測のプロジェクトを作成します。

手順

  1. 1 SAS Viyaホーム画面から「Model Studio」アプリを開きます
  2. 2 「New Project」ボタンをクリック
  3. 3 プロジェクト名: 「Customer Churn Prediction」、説明: 「顧客解約予測モデルの構築」を入力
  4. 4 データソースとして「Public/CHURN」テーブルを選択
  5. 5 ターゲット変数: 「Churn」（Yes/No）を選択
  6. 6 プロジェクトタイプ: 「Classification」（分類）を選択
  7. 7 「Create」をクリックしてプロジェクトを作成

重要ポイント

プロジェクト作成時にターゲット変数を指定することで、Model Studioは自動的に問題のタイプ（分類/回帰）を判定し、適切なアルゴリズムと評価指標を推奨します。

8.4 AutoML（自動機械学習）の実行

AutoMLは、複数のアルゴリズムを自動的に試行し、最適なモデルを見つける機能です。Model Studioには「Autotuned Pipeline」という名前でAutoML機能が組み込まれています。

AutoMLの仕組み

AutoMLは以下のプロセスを自動実行します。

AutoMLの処理フロー

flowchart LR A["データ読込"] --> B["前処理
（欠損値処理等）"] B --> C["特徴量選択
（重要度分析）"] C --> D["アルゴリズム試行
（複数モデル）"] D --> E["ハイパーパラメータ
チューニング"] E --> F["モデル評価
（CV検証）"] F --> G["最適モデル選択"]

AutoMLで試行されるアルゴリズム

  • 決定木: Decision Tree（CART、CHAID）
  • アンサンブル: Random Forest、Gradient Boosting（GBM）
  • ニューラルネット: Neural Network（MLP）
  • 回帰/分類: Logistic Regression、Linear Regression
  • SVM: Support Vector Machine

実習 8-2: AutoMLパイプラインの実行

自動機械学習で最適なモデルを見つけます。

手順

  1. 1 作成したプロジェクト内で「Pipelines」タブを開く
  2. 2 「Autotuned Pipeline」を選択（デフォルトで作成済み）
  3. 3 パイプライン画面で「Run Pipeline」ボタンをクリック
  4. 4 処理の進行状況を確認（各ノードが順次実行される）
  5. 5 実行完了後、「Model Comparison」タブを開く
  6. 6 各モデルの精度、F1スコア、AUCを比較して最適モデルを確認

ヒント

AutoMLの実行時間は、データサイズと試行するアルゴリズム数に依存します。小規模データでは数分、大規模データでは数十分かかる場合があります。処理は並列実行されるため、CASの分散処理が活用されます。

Model Studioのパイプラインは、データの流れに沿った処理ノードの連鎖です。

標準的なパイプラインの構成

主要ノードの役割

ノード 役割 主な設定
Data Source データの読み込み CASテーブル選択、変数の役割設定
Data Partition Train/Validation/Test分割 分割比率（70/15/15など）
Imputation 欠損値の補完 補完方法（平均値、中央値、定数など）
Transform Variables 変数変換 正規化、標準化、対数変換など
Feature Machine 自動特徴量生成 交互作用項、多項式など
Forest Random Forestモデル 木の数、深さ、分岐基準
Model Comparison 複数モデルの比較 評価指標の選択

重要ポイント

パイプラインはノードをドラッグ＆ドロップして作成しますが、AutoMLパイプラインは自動的に最適なノードを配置してくれます。初心者はAutoMLから始めて、徐々にカスタムパイプラインに挑戦するのがおすすめです。

8.6 モデル比較と評価

Model Studioでは、複数のモデルを自動的に比較し、最適なモデルを選択できます。

評価指標の種類

分類問題では以下の指標が主に使用されます。

  • Accuracy（精度）: 全体の正答率
  • F1 Score: 適合率と再現率の調和平均
  • AUC（ROC曲線下面積）: 分類性能の総合指標
  • Precision（適合率）: 予測がPositiveの場合の正答率
  • Recall（再現率）: 実際のPositiveを正しく予測できた割合

モデル比較の見方

「Model Comparison」タブでは、各モデルが以下のように表示されます。

モデル比較結果の例

AutoML実行後、以下のような結果が表示されます。

Model Name              | AUC   | F1 Score | Accuracy
------------------------|-------|----------|----------
Gradient Boosting       | 0.892 | 0.845    | 0.867
Random Forest           | 0.881 | 0.831    | 0.854
Neural Network          | 0.874 | 0.820    | 0.848
Logistic Regression     | 0.835 | 0.789    | 0.812

✅ Champion Model: Gradient Boosting（最高AUC）

ヒント

最高スコアのモデルが「Champion」として自動的にマークされます。ただし、業務要件（例: 解釈性重視ならシンプルなモデル）に応じて、手動で別のモデルをChampionに設定することも可能です。

  • SAS Model Studioはノーコード/ローコードで機械学習モデルを構築できる統合環境

理解度確認クイズ

  1. Q1. SAS Model StudioのAutoML機能の正式名称は何ですか？

    回答を表示

    「Autotuned Pipeline」です。複数のアルゴリズムとハイパーパラメータを自動的に試行し、最適な組み合わせを見つけます。

  2. Q2. パイプラインの「Data Partition」ノードの役割は何ですか？

    回答を表示

    データをTrain（学習）、Validation（検証）、Test（テスト）セットに分割する役割を持ちます。一般的には70:15:15の比率で分割されます。

  3. Q3. 分類モデルの性能を総合的に評価する指標として最も一般的なものは何ですか？

    回答を表示

    AUC（ROC曲線下面積）です。0から1の値を取り、1に近いほど高性能です。クラス不均衡にも強く、ビジネスでよく使われます。

  4. Q4. Model Studioで最高スコアのモデルは何と呼ばれますか？

    回答を表示

    「Champion Model」と呼ばれます。このモデルが本番環境へのデプロイ候補として推奨されます。
