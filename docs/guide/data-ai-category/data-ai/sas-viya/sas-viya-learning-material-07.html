第7章

SAS Visual Analyticsによるデータ可視化（応用）

高度な可視化テクニックを習得します。地理空間分析、ネットワーク図、カスタムグラフ、レポートの共有・埋め込み方法を学びます。

この章で学ぶこと

  • 地理空間データの可視化（地図オブジェクト・ヒートマップ）
  • ネットワーク図による関係性の可視化
  • カスタムグラフビルダーによる独自のビジュアライゼーション
  • レポートの共有・埋め込み・パブリッシュ方法
  • パラメータ・プロンプトによる動的レポート作成

7.1 地理空間データの可視化

SAS Visual Analyticsには、地理空間データを効果的に可視化する強力な機能が備わっています。地図オブジェクトを使用することで、位置情報に基づいたデータ分析が可能になります。

地図オブジェクトの種類

  • バブルマップ: 地点の大きさで数値データを表現
  • ヒートマップ: 密度や頻度を色の濃淡で表現
  • 塗り分けマップ: 地域ごとにKPIを色分け表示
  • 座標マップ: 緯度経度による正確な位置表示

サポートされる地理情報

SAS Visual Analyticsは、住所、郵便番号、都道府県、国名などのテキスト情報を自動的にジオコーディングする機能を持ちます。また、緯度・経度の数値データも直接利用可能です。

実習: 売上バブルマップの作成

  1. Visual Analyticsで新規レポートを作成
  2. データペインから「店舗売上データ」を選択
  3. オブジェクトパネルから「地理的バブルマップ」をドラッグ
  4. 地理ロールに「店舗住所」、サイズに「売上金額」を割り当て
  5. 色ロールに「前年比成長率」を割り当て、カラーマップを調整
  6. マップのズームレベルとベースマップスタイルをカスタマイズ

ヒートマップによる密度分析

ヒートマップは、顧客分布や事故発生地点など、点データの密集度を視覚化するのに最適です。色の濃淡で密度が高いエリアを一目で把握できます。

7.2 ネットワーク図による関係性の可視化

ネットワーク図は、ノード（点）とエッジ（線）で構成され、エンティティ間の関係性を視覚化します。顧客間の紹介関係、組織図、サプライチェーンなどの表現に適しています。

ネットワーク図の活用シーン

  • ソーシャルネットワーク分析: 顧客やインフルエンサーの関係性
  • 組織分析: 部署間の連携度や報告ライン
  • サプライチェーン: 仕入先と販売先の流通経路
  • 不正検知: 異常な取引パターンや関係性の発見

ネットワーク図のデータ構造

ネットワーク図には、ノードテーブルとリンクテーブルの2つのデータセットが必要です。

テーブル 必須列 説明
ノードテーブル ノードID、ラベル 各ノードの情報（顧客ID、名前等）
リンクテーブル FROM_ノードID、TO_ノードID ノード間の関係（誰が誰を紹介したか等）
network_data.sas 
/* ノードテーブル作成 */
data nodes;
  input NodeID $ Name $ Department $;
  datalines;
N001 Tanaka Sales
N002 Suzuki Marketing
N003 Sato IT
N004 Takahashi Finance
;
run;

/* リンクテーブル作成 */
data links;
  input FromNode $ ToNode $ Interaction;
  datalines;
N001 N002 5
N002 N003 3
N001 N004 2
N003 N004 4
;
run;

実習: 組織ネットワーク図の作成

  1. ノードテーブルとリンクテーブルをCASにロード
  2. Visual Analyticsで「ネットワーク図」オブジェクトを配置
  3. データロールに「From」「To」を割り当て
  4. ノードサイズに「インタラクション回数」を設定
  5. ノード色に「部署」を割り当ててグループ化
  6. レイアウトアルゴリズムを「力指向」に設定

7.3 カスタムグラフビルダー

標準の可視化オブジェクトでは表現できない独自のグラフが必要な場合、カスタムグラフビルダーを使用して柔軟なビジュアライゼーションを作成できます。

カスタムグラフでできること

  • 軸とマークの自由な組み合わせ: X軸、Y軸、色、形、サイズを細かく制御
  • レイヤーの重ね合わせ: 複数のグラフタイプを1つのキャンバスに表示
  • 細かいスタイル調整: 線の太さ、透明度、アノテーション追加
  • 計算フィールドの活用: その場で新しいメトリクスを作成

カスタムグラフ作成のステップ

graph TD A["カスタムグラフビルダー起動"] --> B["データソース選択"] B --> C["マークタイプ選択
（点・線・棒・面）"] C --> D["ロール割り当て
（X軸・Y軸・色・サイズ）"] D --> E["レイヤー追加
（複数グラフ重ねる場合）"] E --> F["スタイルとアノテーション"] F --> G["プレビュー・調整"] G --> H["レポートに配置"]

重要ポイント

カスタムグラフビルダーは、TableauやPower BIの「カスタムビジュアル」に相当する機能です。標準オブジェクトで限界を感じたら、カスタムグラフビルダーを試してみましょう。

7.4 レポートの共有・埋め込み・パブリッシュ

作成したレポートを組織内外に配布し、意思決定に活用してもらうための方法を学びます。SAS Visual Analyticsには多様な共有オプションがあります。

共有方法の種類

直接共有

SAS Viya内のユーザー・グループにレポートを共有。閲覧権限や編集権限を細かく設定可能。

使用例: 社内ダッシュボードの配布

PDF/PPT書き出し

静的なレポートとしてPDFやPowerPoint形式でエクスポート。メールやプレゼンで活用。

使用例: 経営会議資料の作成

Web埋め込み

iframeタグで外部Webサイトやイントラネットにレポートをインタラクティブに埋め込み。

使用例: 顧客ポータルサイトへの統計表示

スケジュール配信

定期的にレポートを自動生成し、メール添付やSAS Driveに保存。毎朝自動配信も可能。

使用例: 週次売上レポートの自動配信

Web埋め込みの手順

embed_report.html 
<!-- SAS Visual Analyticsレポートの埋め込み例 -->
<iframe
  src="https://your-viya-server.com/SASVisualAnalytics/?reportUri=/reports/your-report"
  width="100%"
  height="800px"
  frameborder="0"
  allowfullscreen>
</iframe>

<!-- パラメータ付きで特定のセクションを表示 -->
<iframe
  src="https://your-viya-server.com/SASVisualAnalytics/?reportUri=/reports/your-report&sectionIndex=0&page=1"
  width="100%"
  height="800px">
</iframe>

ヒント

埋め込み時には、SAS Viyaのゲストアクセス機能またはOAuth認証を設定することで、外部ユーザーにも安全にレポートを公開できます。

7.5 パラメータ・プロンプトによる動的レポート

レポートにパラメータやプロンプトを追加することで、ユーザーがフィルター条件を選択して柔軟にデータを探索できる動的レポートを作成できます。

パラメータの種類

  • リストプロンプト: ドロップダウンリストから値を選択
  • 日付プロンプト: カレンダーUIで期間を選択
  • テキスト入力: 自由にキーワードや数値を入力
  • スライダー: 範囲を視覚的に調整

実習: 地域選択プロンプトの追加

  1. 既存のレポートを編集モードで開く
  2. 左パネルから「プロンプト」タブを選択
  3. 「リスト選択」プロンプトをキャンバスにドラッグ
  4. データアイテムに「地域」列を割り当て
  5. プロンプトのプロパティで「複数選択可能」を有効化
  6. 既存のグラフオブジェクトにプロンプトを適用
  7. プレビューして動作を確認

プロンプトによるレポートフィルタリング

ベストプラクティス

プロンプトは多すぎるとユーザーが混乱します。3〜5個程度の主要なフィルターに絞り、初期値を設定しておくと使いやすいレポートになります。

7.6 レポートのパフォーマンス最適化

大量データを扱うレポートでは、応答速度が重要です。パフォーマンスを最適化するためのテクニックを学びます。

最適化のポイント

データ集約の事前実行

頻繁にアクセスされるレポートは、データを事前に集約してCASテーブルに保存しておくと高速化できます。

フィルターの活用

初期フィルターを設定し、必要最小限のデータだけをロードすることで、レポート表示速度が向上します。

キャッシュの利用

CASエンジンはメモリ上にデータをキャッシュします。同じデータを使うレポートは高速にロードされます。

オブジェクト数の最適化

1ページに10個以上のグラフを配置すると遅くなります。セクション分割やタブを活用しましょう。

理解度確認クイズ

  1. Q1. 地理空間データを密度で可視化する場合、最適なマップタイプは何ですか？

    回答を表示

    ヒートマップです。点データの密集度を色の濃淡で表現し、密度が高いエリアを一目で把握できます。

  2. Q2. ネットワーク図を作成するために必要な2つのデータセットは何ですか？

    回答を表示

    ノードテーブル（各ノードの情報）とリンクテーブル（ノード間の関係）の2つが必要です。

  3. Q3. レポートを外部Webサイトにインタラクティブに埋め込む方法は何ですか？

    回答を表示

    iframeタグを使用してレポートのURLを埋め込みます。ゲストアクセスまたはOAuth認証を設定することで外部ユーザーにも公開できます。

  4. Q4. レポートのパフォーマンスを向上させるための手法を2つ挙げてください。

    回答を表示

    ①データの事前集約とCASへの保存、②初期フィルターの設定による必要最小限のデータロード（他にも複数の正解があります）
