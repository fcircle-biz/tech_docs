第11章

テキストアナリティクスと自然言語処理

SAS Visual Text Analyticsを使ったテキストマイニング、感情分析、トピックモデリング、エンティティ抽出の実践方法を学びます。

この章で学ぶこと

  • SAS Visual Text Analyticsの概要と活用シーンを理解する
  • テキストデータの前処理とクレンジング手法を習得する
  • 感情分析による顧客意見の定量化方法を学ぶ
  • トピックモデリングによる主要テーマ抽出を実践する
  • 固有表現抽出により重要な情報を自動識別する

11.1 SAS Visual Text Analyticsの概要

SAS Visual Text Analyticsは、非構造化テキストデータから洞察を抽出する強力なツールです。顧客レビュー、SNS投稿、コールセンターログ、アンケート自由記述など、様々なテキストデータを分析できます。

主な機能

  • 感情分析: ポジティブ・ネガティブ・ニュートラルの分類
  • トピックモデリング: 文書集合から主要テーマを自動抽出
  • カテゴリ分類: 文書を事前定義されたカテゴリに自動分類
  • エンティティ抽出: 人名、組織名、地名、製品名などの固有表現を識別
  • 概念抽出: 文書内の主要概念と関連性を可視化
  • 多言語対応: 日本語、英語を含む31言語に対応

テキストアナリティクスのワークフロー

flowchart TD A["テキストデータ収集

活用シーン

カスタマーサポートの品質向上、製品開発への顧客の声の反映、ブランドモニタリング、リスク検知など、幅広い用途で活用されています。

11.2 テキストデータの準備と前処理

精度の高いテキスト分析には、適切なデータ前処理が不可欠です。SAS Viyaでは自動前処理機能も備えていますが、データの特性に応じたカスタマイズも可能です。

主な前処理ステップ

1. ノイズ除去

HTML タグ、URL、特殊記号、絵文字などの不要な要素を除去します。

2. 正規化

大文字小文字の統一、全角半角の統一、表記ゆれの修正を行います。

3. トークン化

文章を単語やフレーズに分割します。日本語の場合は形態素解析を使用。

4. ストップワード除去

「は」「が」などの頻出語で意味の薄い語を除去します。

text_preprocessing.sas 
/* テキストデータをCASにロード */
proc casutil;
  load data=work.customer_reviews
       casout="reviews" replace;
quit;

/* テキスト前処理のカスタマイズ */
proc cas;
  textManagement.tmParse /
    table={name="reviews", vars={"review_text"}}
    language="ja"  /* 日本語指定 */
    casOut={name="reviews_parsed", replace=true};
quit;

11.3 感情分析（Sentiment Analysis）

感情分析は、テキストから書き手の感情や意見を自動判定する技術です。SAS Viyaでは、機械学習モデルによる高精度な感情分析が可能です。

感情分析の種類

  • 極性分析: ポジティブ・ネガティブ・ニュートラルの3分類
  • 感情分類: 喜び・怒り・悲しみ・恐怖などの細かい感情分類
  • スコアリング: -1.0（最もネガティブ）〜 +1.0（最もポジティブ）の連続値

実習 11-1: 顧客レビューの感情分析

製品レビューデータを使って感情分析を実行し、ポジティブ・ネガティブの傾向を把握します。

手順

  1. 1 Visual Text Analyticsを起動し、新しいプロジェクトを作成
  2. 2 レビューデータをインポート（テキスト列を指定）
  3. 3 「分析」タブで感情分析を選択し、実行
  4. 4 感情スコア分布を確認し、ネガティブレビューの内容を詳細分析
sentiment_analysis.sas 
/* 感情分析の実行 */
proc cas;
  textManagement.tmSentiment /
    table={name="reviews_parsed"}
    docId="review_id"
    text="review_text"
    language="ja"
    casOut={name="sentiment_results", replace=true};
quit;

/* 結果の確認 */
proc print data=sentiment_results (obs=10);
  var review_id sentiment_score sentiment_label;
run;

重要ポイント

感情分析の精度は業界・製品カテゴリに依存します。独自の感情辞書を作成してカスタマイズすることで精度向上が可能です。

11.4 トピックモデリング（Topic Modeling）

トピックモデリングは、大量の文書から主要なテーマ（トピック）を自動的に発見する手法です。顧客の関心事や問題点を包括的に把握できます。

SAS Viyaのトピック抽出手法

  • LDA（Latent Dirichlet Allocation）: 確率的トピックモデル
  • 概念リンク: 単語間の関連性を可視化
  • 階層的トピック: 大テーマから小テーマへの階層構造を抽出

トピックモデリングの仕組み（LDA）

実習 11-2: トピック抽出とトピック別分析

コールセンターログからトピックを抽出し、問い合わせ傾向を分析します。

手順

  1. 1 Visual Text Analyticsでコールログをインポート
  2. 2 「概念」タブでトピック数を5に設定し、実行
  3. 3 抽出されたトピックごとに代表的なキーワードと文書を確認
  4. 4 トピック別に感情分析を実施し、問題の深刻度を評価

ヒント

トピック数の決定は試行錯誤が必要です。少なすぎると粗く、多すぎると解釈が困難になります。5〜15個から始めるのが一般的です。

11.5 エンティティ抽出と固有表現認識

エンティティ抽出（固有表現認識、NER: Named Entity Recognition）は、テキストから特定の種類の情報（人名、組織名、地名、製品名など）を自動的に識別する技術です。

抽出可能なエンティティタイプ

エンティティタイプ 説明 活用例
PERSON 人名 インフルエンサー特定、顧客名抽出
ORGANIZATION 組織名・企業名 競合他社言及分析
LOCATION 地名・場所 地域別トレンド分析
PRODUCT 製品名・ブランド 製品別評価分析
DATE 日付・時刻 時系列イベント分析
MONEY 金額 価格感度分析
entity_extraction.sas 
/* エンティティ抽出の実行 */
proc cas;
  textManagement.tmEntity /
    table={name="reviews_parsed"}
    docId="review_id"
    text="review_text"
    language="ja"
    entityTypes={"PERSON", "PRODUCT", "LOCATION"}
    casOut={name="entity_results", replace=true};
quit;

/* 抽出結果の集計 */
proc freq data=entity_results;
  tables entity_type entity_value / nocum nopercent;
  where entity_type in ("PRODUCT", "LOCATION");
run;

11.6 実践例：VOC分析ダッシュボードの構築

これまで学んだテキストアナリティクス技術を組み合わせて、VOC（Voice of Customer：顧客の声）分析ダッシュボードを構築します。

ダッシュボードの構成要素

  • 感情分布グラフ: ポジティブ/ネガティブ/ニュートラルの割合
  • トピック別件数: 主要トピックの出現頻度
  • 時系列トレンド: 感情スコアの推移
  • 頻出エンティティランキング: 製品名・地名のトップ10
  • ワードクラウド: 頻出単語の可視化

実習 11-3: VOC分析ダッシュボード構築

Visual Analyticsを使って、テキスト分析結果を統合したダッシュボードを作成します。

手順

  1. 1 テキスト分析結果のテーブルをCASにロード
  2. 2 Visual Analyticsで新規レポートを作成
  3. 3 感情分布用の円グラフ、トピック別棒グラフを配置
  4. 4 時系列グラフでトレンドを可視化
  5. 5 ワードクラウドオブジェクトで重要単語を表示
  6. 6 フィルターを追加してインタラクティブ分析を可能にする

理解度確認クイズ

  1. Q1. テキスト分析における「ストップワード」とは何ですか？また、なぜ除去する必要がありますか？

    回答を表示

    ストップワードは「は」「が」などの頻出語で意味の薄い語のことです。これらを除去することで、より重要な単語に焦点を当て、分析精度を向上させることができます。

  2. Q2. 感情分析のスコアリングで -1.0 と +1.0 はそれぞれ何を意味しますか？

    回答を表示

    -1.0は最もネガティブな感情を示し、+1.0は最もポジティブな感情を示します。0付近はニュートラル（中立）を意味します。

  3. Q3. トピックモデリングで使用されるLDAアルゴリズムの「L」は何の略ですか？

    回答を表示

    Latent（潜在的な）の略です。LDA（Latent Dirichlet Allocation）は、文書集合に潜在する（隠れた）トピックを発見する確率的モデルです。

  4. Q4. エンティティ抽出で「PERSON」「ORGANIZATION」「LOCATION」以外に抽出できるエンティティタイプを2つ挙げてください。

    回答を表示

    PRODUCT（製品名）、DATE（日付）、MONEY（金額）などがあります。（この中から2つを回答）
