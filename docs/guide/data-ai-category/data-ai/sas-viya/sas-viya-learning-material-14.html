14.1 SAS Viya REST APIの基礎

SAS Viya REST APIは、HTTP/HTTPSプロトコルを通じてSAS Viyaの全機能にアクセスできる統一されたインターフェースです。JSON形式でデータをやり取りし、あらゆるプログラミング言語から利用可能です。

API エンドポイント構造

SAS Viya APIは、機能ごとに整理されたエンドポイント群で構成されています。

SAS Viya REST API構造

認証方式

SAS Viya APIは、OAuth 2.0による認証を標準サポートしています。以下の2つの主要な方式があります。

  • パスワードグラント: ユーザー名・パスワードで直接トークン取得（開発・テスト用）
  • クライアントクレデンシャル: サービスアカウントによる認証（本番推奨）
viya_auth.py 
import requests

# SAS Viya認証情報
base_url = "https://your-viya.server.com"
username = "your_username"
password = "your_password"
# OAuth 2.0トークン取得（パスワードグラント）
auth_url = f"{base_url}/SASLogon/oauth/token"
auth_data = {
    "grant_type": "password",
    "username": username,
    "password": password
}
response = requests.post(auth_url, data=auth_data, auth=("sas.cli", ""))
token = response.json()["access_token"]

# 以降のAPI呼び出しでトークンを使用
headers = {"Authorization": f"Bearer {token}"}
print("認証成功！")

重要ポイント

取得したアクセストークンは一定時間（通常1時間）有効です。長時間実行するプロセスでは、リフレッシュトークンを使って自動更新する仕組みを組み込みましょう。

14.2 APIを使った基本操作

APIを使って、SAS Viyaの主要な操作をプログラムから実行できます。ここでは、ファイルアップロード、ジョブ実行、レポート取得の3つの基本操作を学びます。

①ファイルアップロード（Files API）

CSVファイルなどのデータファイルをSAS Driveにアップロードします。

upload_file.py 
import requests

# ファイルをSAS Driveにアップロード
upload_url = f"{base_url}/files/files"
file_path = "data/sales_data.csv"
with open(file_path, "rb") as f:
    files = {"file": f}
    response = requests.post(
        upload_url,
        headers=headers,
        files=files
    )

file_id = response.json()["id"]
print(f"ファイルID: {file_id}")

②ジョブ実行（Job Execution API）

SASプログラムをジョブとして実行し、結果を取得します。

run_job.py 
import requests
import time

# ジョブ定義を作成
job_url = f"{base_url}/jobDefinitions/definitions"
job_def = {
    "name": "DailySalesAnalysis",
    "type": "Compute",
    "code": "proc means data=sales; run;"
}
response = requests.post(job_url, headers=headers, json=job_def)
job_def_id = response.json()["id"]

# ジョブを実行
exec_url = f"{job_url}/{job_def_id}/executions"
exec_response = requests.post(exec_url, headers=headers)
job_id = exec_response.json()["id"]
print(f"ジョブID: {job_id} - 実行中...")

③レポート生成（Reports API）

Visual Analyticsで作成したレポートをPDF形式でエクスポートします。

export_report.py 
import requests

# レポートをPDF形式でエクスポート
report_id = "your-report-id"
export_url = f"{base_url}/reports/reports/{report_id}/content"
params = {"format": "pdf"}

response = requests.get(export_url, headers=headers, params=params)
# PDFファイルを保存
with open("report.pdf", "wb") as f:
    f.write(response.content)
print("レポート保存完了: report.pdf")

14.3 ジョブスケジューリングと自動化

定期的に実行するバッチ処理は、SAS Viyaのジョブスケジューリング機能またはcron/タスクスケジューラと組み合わせて自動化できます。

SAS Viyaスケジューリングサービス

SAS Viyaには、ジョブを定期実行するためのスケジューリング機能が組み込まれています。

ジョブスケジューリングフロー

flowchart LR A["ジョブ定義作成"] --> B["スケジュール設定"] B --> C["トリガー登録"] C --> D{"スケジュール実行"} D -->|毎日 午前2時| E["ジョブ実行"] D -->|データ更新時| F["イベントトリガー実行"] E --> G["結果保存"] F --> G G --> H["通知送信（成功/失敗）"]
schedule_job.py 
import requests

# ジョブスケジュールを作成（毎日午前2時に実行）
schedule_url = f"{base_url}/scheduler/schedules"
schedule_def = {
    "name": "DailyDataRefresh",
    "jobDefinitionUri": f"/jobDefinitions/definitions/{job_def_id}",
    "cronExpression": "0 2 * * *",  # 毎日2:00 AM
    "enabled": True,
    "emailOnFailure": True,
    "emailAddress": "admin@company.com"
}

response = requests.post(schedule_url, headers=headers, json=schedule_def)
print(f"スケジュールID: {response.json()['id']}")

外部スケジューラとの連携

cron（Linux）やタスクスケジューラ（Windows）から、APIを呼び出すPythonスクリプトを定期実行する方法も有効です。

ヒント

スケジューリングには、ジョブ実行履歴の記録、エラー時の通知、再試行ロジックを組み込むと安定性が向上します。SAS Viyaのジョブ実行APIは実行履歴を自動保存します。

14.4 CI/CDパイプラインとの統合

SAS Viyaのモデルやレポートを、GitHub Actions、GitLab CI、Jenkins等のCI/CDパイプラインに統合することで、開発・テスト・デプロイを自動化できます。

GitHub Actionsワークフロー例

GitHub Actionsを使って、SAS Viyaへのモデルデプロイを自動化する例です。

.github/workflows/deploy.yml 
name: Deploy to SAS Viya
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: pip install requests pyyaml
      - name: Deploy to SAS Viya
        env:
          VIYA_URL: ${{ secrets.VIYA_URL }}
          VIYA_USER: ${{ secrets.VIYA_USER }}
          VIYA_PASS: ${{ secrets.VIYA_PASS }}
        run: python deploy_model.py
deploy_model.py 
import requests
import os

base_url = os.environ["VIYA_URL"]
username = os.environ["VIYA_USER"]
password = os.environ["VIYA_PASS"]

# 認証
auth_url = f"{base_url}/SASLogon/oauth/token"
response = requests.post(auth_url, data={
    "grant_type": "password",
    "username": username,
    "password": password
}, auth=("sas.cli", ""))
token = response.json()["access_token"]
headers = {"Authorization": f"Bearer {token}"}

# モデルをアップロード＆公開
model_url = f"{base_url}/modelPublish/models"
with open("models/customer_churn.pmml", "rb") as f:
    response = requests.post(model_url, headers=headers, files={"file": f})
print(f"デプロイ完了: {response.json()['name']}")

重要ポイント

CI/CD統合では、認証情報をGitHub SecretsやAWS Secrets Managerなどのシークレット管理サービスに保存し、コードに直接記述しないようにしましょう。

APIを使って、Visual Analyticsレポートを毎朝自動でPDF出力し、メール送信するスクリプトを作成します。

①Tableau連携

TableauからSAS Viyaの予測モデルをREST API経由で呼び出し、リアルタイム予測を可視化します。

tableau_integration.py 
import requests

# Tableauからスコアリング要求を受け取る
def score_customer(customer_data):
    # SAS Viyaモデルエンドポイント
    score_url = f"{base_url}/microanalyticScore/modules/churn_model/steps/score"

    response = requests.post(
        score_url,
        headers=headers,
        json={"inputs": customer_data}
    )
    return response.json()["outputs"]["churn_probability"]

②Salesforce連携

Salesforceのリード（見込み客）データをSAS Viyaで分析し、スコアをSalesforceに書き戻す双方向連携です。

salesforce_sync.py 
from simple_salesforce import Salesforce
import requests

# Salesforce認証
sf = Salesforce(username="user@company.com", password="pass", security_token="token")
# リードデータ取得
leads = sf.query("SELECT Id, Name, Industry FROM Lead WHERE Status='New'")

for lead in leads["records"]:
    # SAS Viyaでリードスコアリング
    score = score_lead(lead)
    # Salesforceにスコアを書き戻し
    sf.Lead.update(lead["Id"], {"Lead_Score__c": score})

print(f"{len(leads['records'])}件のリードをスコアリング完了")

③Snowflake連携

Snowflakeのデータウェアハウスから大量データをCASにロードし、分析後に結果をSnowflakeに書き戻します。

snowflake_integration.sas 
/* Snowflakeからデータをロード */
libname snow odbc
  datasrc="SnowflakeDSN"
  schema=PUBLIC;

/* CASセッション開始 */
cas mysess;

/* SnowflakeテーブルをCASにロード */
proc casutil;
  load data=snow.sales_data
    casout="sales" replace;
quit;

/* 集計処理（CAS高速処理） */
proc cas;
  aggregation.aggregate /
    table={name="sales"}
    varSpecs={{name="revenue", summarySubset="SUM"}}
    casout={name="aggregated", replace=true};
quit;

/* 結果をSnowflakeに書き戻し */
proc casutil;
  save casdata="aggregated"
    outcaslib="snow"
    casout="sales_aggregated" replace;
quit;

ヒント

外部システム連携では、APIレート制限、データ形式の変換、エラーハンドリングを考慮しましょう。特にSalesforceは1日あたりのAPI呼び出し数に上限があります。

SAS ViyaのREST APIを効率的に使うための公式ドキュメントとツールを紹介します。

理解度確認クイズ

  1. Q1. SAS Viya REST APIで使用される標準的な認証方式は何ですか?

    回答を表示

    OAuth 2.0です。パスワードグラント（ユーザー名・パスワード）またはクライアントクレデンシャルグラント（サービスアカウント）により、アクセストークンを取得します。

  2. Q2. ジョブを毎日午前2時に実行するcron式はどれですか?

    回答を表示

    0 2 * * * です。フォーマットは「分 時 日 月 曜日」で、毎日2時0分を意味します。

  3. Q3. SAS ViyaをCI/CDパイプラインに統合する主なメリットを2つ挙げてください。

    回答を表示

    ①コード変更から本番デプロイまでの自動化による開発速度向上、②自動テストによる品質保証とエラー早期発見（他にも複数の正解があります）

  4. Q4. TableauからSAS Viyaのモデルをリアルタイムで呼び出す際に使用するAPIは何ですか?

    回答を表示

    Micro Analytic Score API（/microanalyticScore）です。低レイテンシのリアルタイムスコアリング用に最適化されています。
