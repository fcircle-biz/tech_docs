データの準備と管理

SAS Viyaへのデータ取り込み、データ品質チェック、データプロファイリング、様々なデータソース（CSV、Excel、データベース、クラウドストレージ）との接続方法を習得します。

この章で学ぶこと

  • SAS Viyaへの多様なデータ取り込み方法を理解する
  • データ品質チェックとデータプロファイリングの実施方法を習得する
  • CSV、Excel、データベースからのデータ接続方法を学ぶ
  • クラウドストレージ（S3、Azure Blob）との連携方法を理解する
  • データクレンジングと変換の基本テクニックを習得する

5.1 SAS Viyaへのデータ取り込みの概要

SAS Viyaでは、多様なデータソースから効率的にデータを取り込むことができます。データ取り込みには主に以下の方法があります。

主なデータ取り込み方法

  • GUIによるアップロード: SAS Driveから直接ファイルをアップロード
  • PROC CASUTILによる取り込み: SASコードでプログラム的にデータをロード
  • データコネクタ: データベースやクラウドストレージへの直接接続
  • REST API: 外部アプリケーションからのデータ送信

CASテーブルとは

CASテーブルは、CASエンジンのメモリ上に展開されたデータです。従来のSASデータセットと異なり、複数ノードに分散されインメモリで高速処理されます。

5.2 CSV/Excelファイルからの取り込み

方法1: GUIによるアップロード

最も簡単な方法は、SAS DriveのGUIを使ったファイルアップロードです。

実習: GUIからのCSVアップロード

手順:

  1. SAS Viyaにログイン後、左メニューから「Data」を選択
  2. 「Import Data」ボタンをクリック
  3. ローカルPCからCSVまたはExcelファイルを選択
  4. プレビュー画面でデータ型を確認・調整
  5. 「Import to」で保存先（Work/Casuser等）を選択
  6. 「Import」をクリックして取り込み完了

方法2: PROC CASUTILによる取り込み

プログラムで自動化する場合は、PROC CASUTILを使用します。

load_csv_to_cas.sas 
/* CASセッション開始 */
cas mysession;

/* CSVファイルをCASにロード */
proc casutil;
  load file="/path/to/data/customers.csv"
       casout="customers" /* CASテーブル名 */
       outcaslib="casuser" /* CASライブラリ */
       replace; /* 既存テーブルを上書き */
quit;

/* データ確認 */
proc print data=casuser.customers(obs=10);
run;
load_excel_to_cas.sas 
/* Excelファイルの特定シートをロード */
proc casutil;
  load file="/path/to/data/sales.xlsx"
       casout="sales_data"
       outcaslib="casuser"
       importoptions=(sheet="Sales2024") /* シート名指定 */
       replace;
quit;

5.3 データベースからの取り込み

SAS Viyaは主要なデータベース（Oracle、SQL Server、PostgreSQL等）からの直接データロードをサポートします。

LIBNAMEステートメントとPROC CASUTILの組み合わせ

データベース接続にはLIBNAMEを使用し、PROC CASUTILでCASにロードします。

load_from_postgres.sas 
/* PostgreSQLへの接続 */
libname pglib postgres
  server="mydb.example.com"
  port=5432
  database="sales_db"
  user="sasuser"
  password="password123";

/* テーブルをCASにロード */
proc casutil;
  load data=pglib.customer_orders /* PostgreSQLテーブル */
       casout="customer_orders"
       outcaslib="casuser"
       replace;
quit;

/* 接続解除 */
libname pglib clear;

セキュリティ上の注意

パスワードをコードに直接記述しないでください。環境変数やSAS Viyaの認証情報管理機能を使用することを推奨します。

SQL Pass-Through機能

データベース側でSQLクエリを実行し、結果のみをCASにロードする方法です。大量データの絞り込みに有効です。

sql_passthrough.sas 
/* SQL Pass-Throughで絞り込んでロード */
proc sql;
  connect to postgres (
    server="mydb.example.com"
    database="sales_db"
    user="sasuser"
    password=&db_password /* マクロ変数を使用 */
  );

  create table work.filtered_data as
  select * from connection to postgres (
    SELECT customer_id, order_date, total_amount
    FROM orders
    WHERE order_date >= '2024-01-01'
      AND status = 'completed'
  );

  disconnect from postgres;
quit;

/* WorkテーブルをさらにCASにロード */
proc casutil;
  load data=work.filtered_data
       casout="filtered_orders"
       outcaslib="casuser";
quit;

5.4 クラウドストレージからの取り込み

SAS ViyaはAWS S3、Azure Blob Storage、Google Cloud Storageなど主要なクラウドストレージと統合できます。

AWS S3からのデータ取り込み

AWS S3からデータを読み込むには、アクセスキーとシークレットキーの設定が必要です。

load_from_s3.sas 
/* S3バケットへの接続 */
filename s3file s3
  "s3://mybucket/data/transactions.csv"
  access_key_id="YOUR_ACCESS_KEY"
  secret_access_key="YOUR_SECRET_KEY"
  region="ap-northeast-1";

/* CSVファイルをCASにロード */
proc casutil;
  load file=s3file
       casout="s3_transactions"
       outcaslib="casuser"
       replace;
quit;

filename s3file clear;

Azure Blob Storageからのデータ取り込み

load_from_azure.sas 
/* Azure Blob Storageへの接続 */
filename azblob azblob
  "container-name/path/to/file.csv"
  storage_account="mystorageaccount"
  storage_account_key="YOUR_STORAGE_KEY";

/* ファイルをCASにロード */
proc casutil;
  load file=azblob
       casout="azure_data"
       outcaslib="casuser"
       replace;
quit;

filename azblob clear;

ベストプラクティス

クラウド認証情報は、SAS Viyaの「Environment Manager」で管理し、マクロ変数や環境変数経由で参照することを推奨します。これによりコードの安全性が向上します。

5.5 データ品質チェックとプロファイリング

データを取り込んだら、品質チェックとプロファイリングを実施します。SAS Viyaでは以下の方法があります。

PROC CARDINALITYによる基本統計

data_profiling.sas 
/* 各列の基本統計情報を取得 */
proc cardinality data=casuser.customers outcard=work.card_summary;
run;

/* 結果表示 */
proc print data=work.card_summary;
run;

/* 欠損値チェック */
proc cas;
  simple.summary /
    table={name="customers", caslib="casuser"}
    inputs={"age", "income", "credit_score"}
    casout={name="missing_check", replace=true};
quit;

データ品質チェックの主なポイント

  • 欠損値: NULL、空文字、特殊値（999等）の確認
  • 重複データ: 主キーの重複チェック
  • データ型: 数値が文字列で保存されていないか等
  • 外れ値: 異常に大きい/小さい値の検出
  • 一貫性: 日付の前後関係、計算項目の整合性

Visual Analyticsでのデータプロファイリング

SAS Visual Analyticsでは、GUIでデータプロファイリングレポートを自動生成できます。

実習: Visual Analyticsでのプロファイリング

  1. Visual Analyticsを起動し、「New Report」を選択
  2. 左ペインの「Data」から対象テーブルを選択
  3. 「Data Pane」で各列を右クリック → 「Properties」
  4. 「Distribution」タブで分布グラフと統計量を確認
  5. 欠損値の割合、最頻値、外れ値候補を特定

5.6 データクレンジングと変換

データ品質の問題を特定したら、クレンジングと変換を行います。

欠損値の処理

handle_missing_values.sas 
/* 欠損値を平均値で補完 */
data casuser.customers_cleaned;
  set casuser.customers;

  /* 年齢の欠損を平均値で補完 */
  if missing(age) then age = 45; /* 事前計算した平均値 */

  /* 収入の欠損をカテゴリ別中央値で補完 */
  if missing(income) then do;
    if job_category = 'Engineer' then income = 65000;
    else if job_category = 'Manager' then income = 80000;
    else income = 50000;
  end;
run;

データ型変換と正規化

data_transformation.sas 
data casuser.orders_transformed;
  set casuser.orders_raw;

  /* 文字列日付を日付型に変換 */
  order_date_fmt = input(order_date_str, yymmdd10.);
  format order_date_fmt date9.;

  /* 数値の正規化（0-1スケール） */
  amount_normalized = (amount - 1000) / (50000 - 1000);

  /* カテゴリ変数のエンコーディング */
  if status = 'pending' then status_code = 1;
  else if status = 'shipped' then status_code = 2;
  else if status = 'delivered' then status_code = 3;
  else status_code = 0;
run;

重複データの削除

remove_duplicates.sas 
/* 重複行を削除（最初の行のみ保持） */
proc sort data=casuser.customers
          out=casuser.customers_dedup
          nodupkey;
  by customer_id; /* キー列 */
run;

理解度確認クイズ

  1. Q1. CASテーブルと従来のSASデータセットの主な違いは何ですか？

    回答を表示

    CASテーブルは複数ノードに分散され、インメモリで高速処理されます。従来のSASデータセットはディスクベースで単一サーバー処理です。

  2. Q2. PROC CASUTILの主な用途は何ですか？

    回答を表示

    データをCASエンジンにロードしたり、CASからデータを保存したりする際に使用します。LOADやSAVE、DROPTABLEなどのアクションを提供します。

  3. Q3. データベースからデータを取り込む際のSQL Pass-Throughの利点は何ですか？

    回答を表示

    データベース側でSQLクエリを実行し、必要なデータのみを取得するため、ネットワーク転送量が削減され、パフォーマンスが向上します。

  4. Q4. データ品質チェックで確認すべき主なポイントを3つ挙げてください。

    回答を表示

    ①欠損値の確認、②重複データの検出、③外れ値の特定（他にも複数の正解があります）
