実践プロジェクト：顧客分析ダッシュボード構築

これまで学んだ知識を総合して、顧客セグメンテーション、解約予測モデル、インタラクティブダッシュボードを構築する実践プロジェクトに取り組みます。

この章で学ぶこと

  • 実践プロジェクトの全体像とビジネス背景を理解する
  • データ準備とEDAを実施し、データの特性を把握する
  • K-Meansクラスタリングで顧客セグメンテーションを実施する
  • 解約予測モデルを構築し、モデルを評価・デプロイする
  • Visual Analyticsで包括的なダッシュボードを構築する

15.1 プロジェクト概要

本章では、これまで学習したSAS Viyaの機能を統合して、顧客分析ダッシュボードを構築します。実際のビジネス課題に即した実践的なプロジェクトを通じて、データ準備から可視化まで一連のワークフローを体験します。

ビジネス背景

あなたは通信会社のデータアナリストとして、顧客維持率の向上を目指すプロジェクトに参画しています。以下の課題に取り組みます。

  • 顧客セグメンテーション: 顧客を特性に基づいてグループ化し、適切なマーケティング戦略を立案
  • 解約予測: 解約リスクの高い顧客を早期に特定し、リテンション施策を実施
  • インサイト可視化: 経営層が理解しやすいダッシュボードでKPIを監視

プロジェクトの成果物

顧客セグメント

K-Meansクラスタリングにより顧客を4-5のセグメントに分類し、各セグメントの特徴を分析

解約予測モデル

勾配ブースティングやニューラルネットワークで構築した高精度な解約予測モデル

ダッシュボード

顧客KPI、セグメント分析、解約予測結果を統合したインタラクティブダッシュボード

プロジェクト全体フロー

15.2 データ準備と探索的データ分析（EDA）

まず、顧客データをSAS Viyaに取り込み、データの品質確認と探索的分析を行います。

使用するデータセット

サンプルとして、通信業界の顧客データを使用します。データには以下の情報が含まれています。

カテゴリ 変数名 説明
顧客属性 CustomerID, Age, Gender 顧客ID、年齢、性別
契約情報 Tenure, Contract, PlanType 契約期間、契約タイプ、プランタイプ
利用状況 MonthlyCharge, TotalCharge 月額料金、総支払額
サービス利用 InternetService, StreamingTV インターネット、ストリーミングTV
目的変数 Churn 解約フラグ（0:継続、1:解約）

実習 15-1: データ読み込みとEDA

CSVデータをCASにロードし、Data Explorerで基本統計量を確認します。

手順

  1. 1 SAS Studioから「データ」タブを開き、「ローカルファイル」を選択
  2. 2 customer_data.csvをアップロードし、CASライブラリに保存
  3. 3 Data Explorerを開き、欠損値・外れ値・分布を確認
  4. 4 目的変数（Churn）の分布を確認し、不均衡データかどうか判定
data_preparation.sas 
/* CASセッション開始 */
cas mysession;
caslib _all_ assign;

/* データをCASにロード */
proc casutil;
  load data=work.customer_data
    casout="customer_data" replace;
quit;

/* 基本統計量の確認 */
proc cas;
  simple.summary /
    table={name="customer_data"}
    inputs={"Age", "Tenure", "MonthlyCharge", "TotalCharge"};
quit;

ヒント

Data Explorerの「プロファイル」機能を使うと、全変数の分布やデータ品質を一覧で確認できます。欠損値が多い変数や、外れ値の処理方針を検討しましょう。

15.3 顧客セグメンテーション

K-Meansクラスタリングを使って、顧客を類似した特性を持つグループに分類します。

セグメンテーションの流れ

クラスタリングワークフロー

flowchart LR A[データ準備] --> B[変数選択・標準化] B --> C[最適クラスタ数決定] C --> D[K-Meansクラスタリング実行] D --> E[セグメント特性分析] E --> F[セグメント命名・解釈]

実習 15-2: SAS Studioでクラスタリング

PROC CLUSTERとPROC FASTCLUSを使って顧客をセグメント化します。

手順

  1. 1 セグメンテーションに使う変数を選択（Age、Tenure、MonthlyChargeなど）
  2. 2 PROC STANDARDで変数を標準化
  3. 3 エルボー法でクラスタ数を決定（3-6の範囲で試行）
  4. 4 PROC FASTCLUSで最終クラスタリング実行
clustering.sas 
/* 変数の標準化 */
proc standard data=customer_data
  mean=0 std=1
  out=customer_std;
  var Age Tenure MonthlyCharge TotalCharge;
run;

/* K-Meansクラスタリング（4クラスタ） */
proc fastclus data=customer_std
  maxclusters=4
  out=customer_clustered;
  var Age Tenure MonthlyCharge TotalCharge;
run;

/* セグメント別集計 */
proc means data=customer_clustered;
  class cluster;
  var Age Tenure MonthlyCharge Churn;
run;

セグメント特性の解釈例

セグメント 特徴 解約率 推奨施策
ロイヤル顧客 長期契約、高額利用 低（5%） VIPプログラム、アップセル
新規顧客 契約期間短、様子見層 中（20%） オンボーディング強化
価格重視層 低額プラン、長期契約 中（15%） 割引キャンペーン
離反リスク層 短期契約、不満多 高（40%） リテンション集中施策

15.4 解約予測モデル構築

SAS Model Studioを使って、高精度な解約予測モデルを構築します。AutoML機能を活用して、複数のアルゴリズムを自動で比較評価します。

実習 15-3: Model Studioで解約予測モデル構築

AutoMLパイプラインを使って、最適な解約予測モデルを自動選択します。

手順

  1. 1 Model Studioで新しいプロジェクト「顧客解約予測」を作成
  2. 2 データソース: customer_data、目的変数: Churn（バイナリ分類）
  3. 3 「AutoML」パイプラインを選択し、実行
  4. 4 モデル比較画面でAUC、Accuracy、Precisionを確認
  5. 5 最良モデルを選択し、Model Managerに登録

モデル評価指標

解約予測では、以下の指標を重視します。

  • AUC（Area Under Curve）: 0.85以上を目標。解約/継続の識別能力を示す
  • Precision（適合率）: 解約予測の精度。施策コストに直結
  • Recall（再現率）: 実際の解約者をどれだけ捕捉できたか
  • F1 Score: PrecisionとRecallの調和平均

重要ポイント

解約予測では、偽陽性（解約しないのに解約と予測）と偽陰性（解約するのに継続と予測）のコストが異なります。ビジネス要件に応じて、閾値を調整しましょう。

モデル解釈性

Model StudioのSHAP値やPartial Dependence Plotを使って、どの変数が解約に影響しているかを分析します。

解約に影響する主要変数（例）

  • 契約期間（Tenure）: 短いほど解約リスク高
  • 月額料金（MonthlyCharge）: 高額層は解約リスク高
  • 契約タイプ（Contract）: 月次契約は解約リスク高
  • カスタマーサポート利用回数: 多いほど不満の兆候

実習 15-4: ダッシュボード作成

Visual Analyticsで4ページ構成のインタラクティブダッシュボードを構築します。

手順

  1. 1 Visual Analyticsで新規レポート作成
  2. 2 データソースにcustomer_clustered（セグメント付き）とモデルスコアを結合
  3. 3 4つのページを作成し、各ページに必要なビジュアライゼーションを配置

理解度確認クイズ

  1. Q1. K-Meansクラスタリングで最適なクラスタ数を決定する際に使用する手法は何ですか？

    回答を表示

    エルボー法（Elbow Method）です。クラスタ数を変えながらSSE（Sum of Squared Errors）をプロットし、曲線が「肘」のように曲がるポイントを最適クラスタ数とします。

  2. Q2. 解約予測モデルで、実際の解約者をどれだけ捕捉できたかを示す指標は何ですか？

    回答を表示

    Recall（再現率）です。全解約者のうち、モデルが正しく解約と予測できた割合を示します。Recall = TP / (TP + FN)

  3. Q3. Visual Analyticsで複数ページ間でフィルター情報を共有するために使う機能は何ですか？

    回答を表示

    「アクション」機能です。これにより、あるページで選択したフィルターやセグメントを別のページにも自動的に適用できます。

  4. Q4. プロジェクトで構築したモデルを本番環境に展開する際に使うSAS Viyaのコンポーネントは何ですか？

    回答を表示

    SAS Model Managerです。モデルのバージョン管理、デプロイ、モニタリング、再学習など、MLOpsのライフサイクル全体を管理できます。
