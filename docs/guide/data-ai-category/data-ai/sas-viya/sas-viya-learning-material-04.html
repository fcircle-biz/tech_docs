SAS Studioでのプログラミング（Viya版）

SAS Viya上でのSASプログラミングを学びます。従来のSASコードとの互換性、CASアクションの使い方、PROC CASによるCAS操作を習得します。

この章で学ぶこと

  • SAS Studio（Viya版）の基本操作と画面構成を理解する
  • 従来のSASコードとViya版の互換性を把握する
  • CASアクションとPROC CASの基本的な使い方を習得する
  • 従来のPROCとCASアクションの使い分けを学ぶ
  • 実用的なプログラム例を通じてViya版SASプログラミングをマスターする

4.1 SAS Studio（Viya版）の基本操作

SAS Studio（Viya版）は、Webブラウザから利用できるSASの統合開発環境です。従来のSAS Studioと似たインターフェースですが、Viyaのクラウドネイティブ機能とCASエンジンをフル活用できます。

画面構成

SAS Studioの画面は以下の領域で構成されています。

  • コードエディタ: SASコードの記述・編集（シンタックスハイライト対応）

  • 保存: ツールバーの保存アイコンまたはCtrl+S

  • 検索/置換: Ctrl+F（検索）、Ctrl+H（置換）

4.2 従来のSASコードとの互換性

SAS Viyaは従来のSASコードとの高い互換性を保ちながら、CASエンジンの利点を活かせる設計になっています。

互換性のあるコード

以下のような従来のSASコードは、そのままViya上で動作します。

compatible_code.sas 
/* データステップ */
data work.customers;
  set sashelp.class;
  bmi = weight / (height**2) * 703; /* BMI計算 */
  if bmi > 25 then status = '肥満傾向';
  else status = '標準';
run;

/* PROC MEANS（集計） */
proc means data=work.customers mean std;
  var age weight;
run;

/* PROC FREQ（度数分布） */
proc freq data=work.customers;
  tables status;
run;

重要ポイント

データステップ、PROC MEANS、PROC FREQ、PROC SQL、PROC PRINT等の基本プロシジャは、従来通り使用できます。既存のSASプログラムの多くは修正不要で動作します。

CASエンジンを活用したコード

同じ処理をCASエンジンで実行すると、大量データでも高速に処理できます。

cas_optimized.sas 
/* CASセッション開始 */
cas mysession;
caslib _all_ assign;

/* データをCASにロード */
proc casutil;
  load data=sashelp.class casout="customers";
quit;

/* CASテーブルでデータ加工 */
data casuser.customers;
  set casuser.customers;
  bmi = weight / (height**2) * 703;
  if bmi > 25 then status = '肥満傾向';
  else status = '標準';
run;

/* CASアクションで集計（高速） */
proc cas;
  simple.summary /
    table={name="customers"}
    inputs={"age", "weight"}
    casout={name="summary", replace=true};
quit;

互換性モード vs CASモード

従来のSASコードはWORKライブラリで動作します（互換性モード）。CASエンジンを活用する場合は、CASUSERやCASLIBにデータをロードします（CASモード）。データ量や処理内容に応じて使い分けましょう。

4.3 CASアクションの基本

CASアクションは、CASエンジンで処理を実行する基本単位です。従来のPROCに相当しますが、より柔軟で高速な処理が可能です。

主なCASアクションカテゴリ

simple（基本統計）

平均、標準偏差、最小値、最大値など基本的な統計量を計算

simple.summary, simple.freq

table（テーブル操作）

テーブルの読み込み、保存、削除、結合など

table.loadTable, table.fetch

dataPreprocess（前処理）

欠損値処理、正規化、カテゴリカル変数のエンコーディング

dataPreprocess.impute

percentile（分位数）

中央値、四分位数、百分位数の計算

percentile.assess

CASアクションの基本構文

cas_action_syntax.sas 
proc cas;
  /* アクションセット名.アクション名 / パラメータ; */
  simple.summary /
    table={name="customers"}        /* 入力テーブル */
    inputs={"age", "weight"}        /* 対象変数 */
    casout={name="stats", replace=true};  /* 出力テーブル */
quit;

構文のポイント：

  • アクションセット.アクション名で処理を指定
  • table=で入力テーブルを指定（CASテーブル名）
  • inputs=で分析対象の変数を指定
  • casout=で結果を保存するCASテーブルを指定

4.4 PROC CASによるCAS操作

PROC CASは、SASプログラムからCASアクションを実行するためのプロシジャです。

実践例1: データ集計

data_aggregation.sas 
/* CASセッション開始 */
cas mysession;

/* データロード */
proc casutil;
  load data=sashelp.cars casout="cars";
quit;

/* 統計量の計算 */
proc cas;
  simple.summary /
    table={name="cars"}
    inputs={"msrp", "horsepower", "mpg_city"}
    casout={name="car_stats", replace=true};
quit;

/* 結果を確認 */
proc print data=casuser.car_stats;
run;

実践例2: データフィルタリングと集約

data_filtering.sas 
/* 高級車のみフィルタリング */
proc cas;
  table.fetch /
    table={name="cars", where="msrp > 50000"}
    to=100
    casout={name="luxury_cars", replace=true};
quit;

/* メーカー別の集計 */
proc cas;
  aggregation.aggregate /
    table={name="luxury_cars"}
    varSpecs={{name="msrp", summarySubset={"MEAN", "MAX"}}}
    casout={name="by_make", replace=true};
quit;

ヒント

CASアクションは分散処理されるため、数百万〜数億行のデータでも高速に処理できます。従来のPROCでは時間がかかる処理も、CASなら数秒〜数分で完了します。

4.5 従来のPROCとCASアクションの使い分け

従来のPROCとCASアクションは、データ量や処理内容に応じて使い分けることが重要です。

項目 従来のPROC CASアクション
推奨データ量 小〜中規模（〜数百万行） 中〜大規模（数百万行以上）
処理速度 標準 高速（インメモリ＆分散処理）
利用可能環境 WORKライブラリ CASライブラリ（CASUSER等）
機能の豊富さ 非常に豊富 主要機能をカバー（拡張中）
学習コスト 低い（従来知識が活用可能） 中程度（新しい概念の習得必要）
推奨用途 探索的分析、プロトタイピング 本番処理、大規模データ分析

使い分けの指針

  • 従来のPROCを使う場合: データ量が少ない、複雑な統計手法が必要、既存のSASプログラムを再利用
  • CASアクションを使う場合: データ量が多い、処理速度が重要、本番環境での定期実行

実習 4-1: CASアクションで顧客データ分析

CASアクションを使って、顧客データの集計と分析を行います。

手順

  1. 1 SAS Studio（Viya版）で新規プログラムを作成
  2. 2 CASセッションを開始し、sashelp.carsをCASにロード
  3. 3 simple.summaryアクションでMSRP、Horsepower、MPG_Cityの統計量を算出
  4. 4 結果をPROC PRINTで確認
解答例を表示
exercise_4_1.sas 
/* CASセッション開始 */
cas mysession;

/* データをCASにロード */
proc casutil;
  load data=sashelp.cars casout="cars";
quit;

/* 統計量の計算 */
proc cas;
  simple.summary /
    table={name="cars"}
    inputs={"msrp", "horsepower", "mpg_city"}
    casout={name="car_summary", replace=true};
quit;

/* 結果を表示 */
proc print data=casuser.car_summary;
run;

/* CASセッション終了 */
cas mysession terminate;

理解度確認クイズ

  1. Q1. SAS Studio（Viya版）でコードを実行するショートカットキーは何ですか？

    回答を表示

    F3キーです。実行ボタンをクリックする方法もあります。

  2. Q2. 従来のSASコードをそのままViya上で動作させる場合、どのライブラリが使用されますか？

    回答を表示

    WORKライブラリです。CASエンジンを活用する場合は、CASUSERやCASLIBにデータをロードします。

  3. Q3. CASアクションの基本構文で、結果を保存するCASテーブルを指定するパラメータは何ですか？

    回答を表示

    casout=パラメータです。例: casout={name="result", replace=true}

  4. Q4. 大量データ（数百万行以上）を高速に処理する場合、従来のPROCとCASアクションのどちらを使うべきですか？

    回答を表示

    CASアクションを使うべきです。インメモリ分散処理により、大量データでも高速に処理できます。
