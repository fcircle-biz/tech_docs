モデルの展開と運用（MLOps）

作成したモデルを本番環境に展開する方法を学びます。SAS Model Managerによるモデル管理、バージョニング、モニタリング、再学習の仕組みを習得します。

12.1 MLOpsとは

MLOps（Machine Learning Operations）は、機械学習モデルの開発・デプロイ・運用を自動化・標準化するための手法とツールの総称です。DevOpsの考え方を機械学習に適用したもので、モデルの継続的な改善と安定運用を実現します。

なぜMLOpsが必要なのか？

機械学習モデルは一度作ったら終わりではありません。データの変化によりモデル性能は劣化し、定期的な再学習・更新が必要です。MLOpsにより、この運用プロセスを自動化・効率化できます。

SAS ViyaにおけるMLOps

SAS Viyaは包括的なMLOpsプラットフォームを提供します。

  • Model Studio: モデル開発・評価
  • Model Manager: モデル管理・バージョニング
  • MAS（Model Analytics Service）: 推論サービス
  • Model Monitor: 性能監視・ドリフト検出

12.2 SAS Model Managerの基礎

SAS Model Managerは、モデルのライフサイクル全体を管理する中央管理システムです。モデルの登録、バージョン管理、承認フロー、デプロイメント、モニタリングを一元化します。

Model Managerの主な機能

モデルリポジトリ

全てのモデルを一元管理。メタデータ、バージョン履歴、性能指標、入出力スキーマを保存。

バージョン管理

モデルの各バージョンを追跡。チャンピオン/チャレンジャーモデルの比較とロールバック機能。

ガバナンス

承認ワークフロー、アクセス制御、監査ログによりモデルの適切な管理を実現。

デプロイメント

開発環境から本番環境へのモデル移行を自動化。REST API、バッチスコアリングなど複数の形式でデプロイ。

モデル登録の流れ

Model Studioで作成したモデルをModel Managerに登録する基本的な手順を見ていきます。

12.3 モデルのバージョニングとガバナンス

モデルのバージョン管理は、MLOpsの基盤となる重要な機能です。SAS Model Managerでは、チャンピオン/チャレンジャーモデルの概念により、複数バージョンの比較・管理を行います。

チャンピオン/チャレンジャーモデル

  • チャンピオンモデル: 現在本番環境で稼働中のモデル（最も性能が良いとされるモデル）
  • チャレンジャーモデル: チャンピオンを超える可能性のある候補モデル

チャンピオン/チャレンジャー比較プロセス

flowchart LR A[チャンピオンモデル v1.0] --> B{性能比較} C[チャレンジャーモデル v1.1] --> B D[チャレンジャーモデル v1.2] --> B B -->|v1.2が最良| E[v1.2を新チャンピオンに昇格] E --> F[本番環境に展開] B -->|v1.0が最良| G[現状維持]

モデルガバナンス

モデルのガバナンスは、規制遵守、リスク管理、透明性確保のために不可欠です。

重要ポイント

金融・医療などの規制業界では、モデルの監査証跡、説明責任、公平性評価が法的に要求されます。SAS Model Managerはこれらの要件に対応するガバナンス機能を提供します。

ガバナンス機能 説明
承認ワークフロー モデルの登録・変更・デプロイに承認プロセスを設定
監査ログ 全ての操作履歴を自動記録（誰が、いつ、何を変更したか）
モデルカード モデルの目的、データ、性能、制限事項を文書化
公平性評価 バイアス検出・緩和ツールによる公平性チェック
アクセス制御 役割ベースのアクセス権限管理（RBAC）

12.4 モデルのデプロイメントと推論API

モデルをModel Managerに登録したら、次はデプロイメントです。SAS Viyaでは、モデルをMAS（Micro Analytic Service）として公開し、REST APIで推論を実行できます。

デプロイメント先の選択肢

  • MAS（Micro Analytic Service）: リアルタイム推論REST API
  • CAS: バッチスコアリング（大量データの一括予測）
  • 外部システム: PMML、PythonPickleなどの形式でエクスポート

MASへのデプロイ手順

実習 12-1: モデルのMASデプロイ

Model ManagerからモデルをMASにデプロイし、REST APIで推論を実行します。

手順

  1. 1 Model Managerでデプロイ対象モデルを選択
  2. 2 「Publish」→「Publish to Destination」をクリック
  3. 3 Destination Typeで「Micro Analytic Service」を選択
  4. 4 サービス名を入力し、「Publish」をクリック
  5. 5 生成されたREST APIエンドポイントURLをコピー

REST APIによる推論実行

MASにデプロイしたモデルは、HTTP POSTリクエストで推論を実行できます。

model_inference.py 
import requests
import json

# MAS APIエンドポイント
url = "https://your-viya-server.com/microanalyticScore/modules/churn_model/steps/score"

# 推論リクエストデータ
payload = {
    "inputs": [
        {"name": "age", "value": 45},
        {"name": "income", "value": 65000},
        {"name": "tenure", "value": 24}
    ]
}

# API呼び出し
response = requests.post(url, json=payload, auth=("username", "password"))

# 結果表示
if response.status_code == 200:
    result = response.json()
    print("予測確率:", result["outputs"][0]["value"])
else:
    print("エラー:", response.status_code)

ヒント

MAS APIは1秒あたり数千件の推論をサポートします。高トラフィックな本番システムで利用できる、エンタープライズグレードの推論サービスです。

12.5 モデル性能のモニタリングと再学習

本番環境にデプロイしたモデルは、時間とともに性能が劣化します。これをモデルドリフトと呼び、定期的なモニタリングと再学習が必要です。

モデルドリフトの種類

  • データドリフト（Concept Drift）: 入力データの分布が変化（例: 顧客の行動パターン変化）
  • 性能劣化（Performance Drift）: 予測精度の低下（例: 正解率の減少）

モデルドリフト検出と対応

flowchart TD A[本番モデル稼働] --> B[モニタリング開始] B --> C{ドリフト検出?} C -->|No| B C -->|Yes| D[アラート発報] D --> E[原因分析] E --> F{対応方法決定} F -->|軽微| G[チューニング] F -->|深刻| H[再学習実行] G --> I[更新モデルデプロイ] H --> I I --> A

SAS Model Monitorの機能

SAS Model Monitorは、モデルの性能を継続的に追跡し、ドリフトを自動検出します。

監視項目 内容
予測精度 Accuracy、F1スコア、ROC-AUCなどの時系列推移
データ品質 欠損値率、外れ値検出、データ型不整合
データ分布 学習時と推論時のデータ分布の比較（KLダイバージェンス等）
バイアス 特定グループに対する予測偏り（公平性指標）
推論頻度 API呼び出し回数、レスポンス時間、エラー率

自動再学習パイプライン

SAS Viyaでは、ドリフト検出時に自動的に再学習を実行するパイプラインを構築できます。

auto_retrain.sas 
/* 自動再学習パイプラインのサンプル */

/* ドリフト検出結果を確認 */
proc cas;
  action modelPublishing.getPerformance /
    modelName="churn_model"
    metricName="F1"
    threshold=0.85
    result=r;
run;

/* 性能が閾値以下なら再学習 */
%if &r.drift_detected = 1 %then %do;

  /* 最新データで再学習 */
  proc autotune data=mycas.latest_data;
    target churn / level=binary;
    input age income tenure / level=interval;
    code file="/models/retrained_model.sas";
  run;

  /* 新モデルを登録 */
  proc cas;
    modelPublishing.registerModel /
      modelName="churn_model"
      version="auto_retrain_2025-12-19"
      modelFile="/models/retrained_model.sas";
  run;

%end;

注意

自動再学習は便利ですが、必ず人間によるレビューと承認プロセスを組み込みましょう。再学習後のモデルが意図しない挙動をする可能性があります。

12.6 MLOpsのベストプラクティス

効果的なMLOpsを実現するための推奨事項をまとめます。

1. モデル開発からデプロイまでの自動化

  • CI/CDパイプラインにモデル学習・評価・デプロイを統合
  • 単体テスト、統合テストを自動実行

2. 再現性の確保

  • 乱数シードの固定、データバージョン管理
  • 学習時の環境（ライブラリバージョン等）を記録

3. モニタリングとアラート

  • 性能劣化の閾値を設定し、超過時に通知
  • ダッシュボードで主要指標をリアルタイム可視化

4. ドキュメンテーション

  • モデルカードで意図・制限・性能を明記
  • データセット仕様、前処理ロジックを文書化

5. セキュリティと権限管理

  • 最小権限の原則に基づくアクセス制御
  • 機密データの暗号化、個人情報の匿名化

理解度確認クイズ

  1. Q1. チャンピオンモデルとチャレンジャーモデルの違いを説明してください。

    回答を表示

    チャンピオンモデルは現在本番環境で稼働中の最良モデルです。チャレンジャーモデルはチャンピオンを超える可能性のある候補モデルで、性能比較の結果、より優れていればチャンピオンに昇格します。

  2. Q2. モデルドリフトの2つの主な種類は何ですか？

    回答を表示

    ①データドリフト（Concept Drift）: 入力データの分布変化、②性能劣化（Performance Drift）: 予測精度の低下です。

  3. Q3. SAS ViyaでモデルをREST APIとして公開するサービスの名前は何ですか？

    回答を表示

    MAS（Micro Analytic Service）です。リアルタイム推論APIとして高速な予測を提供します。

  4. Q4. モデルガバナンスで重要な機能を3つ挙げてください。

    回答を表示

    ①承認ワークフロー、②監査ログ、③アクセス制御などです（他にもモデルカード、公平性評価など多数あります）。
