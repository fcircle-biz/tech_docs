ワークフロー入門 - ビジュアルで処理を設計する

Difyワークフローの基本概念を理解し、ドラッグ&ドロップで処理の流れを設計するビジュアルプログラミングを体験します。

この章で学ぶこと

  • Difyワークフローとは何か、その基本概念を理解する
  • ワークフローエディタの基本操作（ノードの追加、接続、削除）
  • 最初のシンプルなワークフローを構築する実践
  • ワークフローのテスト実行とデバッグ方法

9.1 ワークフローとは何か

ワークフロー（Workflow）は、複数の処理ステップを順番に実行する自動化の仕組みです。 Difyでは、ビジュアルエディタ上でノード（処理単位）をドラッグ&ドロップで配置し、それらを線で接続することで処理フローを設計します。

従来、こうした処理フローを実装するにはプログラミング言語でコードを書く必要がありましたが、 Difyのワークフローを使えば、ビジュアルプログラミングにより、直感的に複雑な処理を設計できます。

ワークフローのメリット

  • 視覚的理解: 処理の流れが図で一目瞭然、チームで共有しやすい
  • 再利用性: ノードを組み合わせて別のワークフローにも応用可能
  • デバッグ容易: 各ノードの入出力を確認しながら問題箇所を特定できる
  • 迅速な修正: ノードの設定変更や接続変更が即座に反映される

ワークフローの基本構造

各ノードが1つの処理を担当し、左から右へデータが流れていきます。

9.2 ワークフローエディタの基本操作

Difyのワークフローエディタは、直感的に操作できるように設計されています。 ここでは、基本的な操作方法を理解しましょう。

ワークフローエディタの構成

左サイドバー: ノードパレット

利用可能なノードが種類別に分類されています。ドラッグしてキャンバスに配置します。

中央: キャンバス

ノードを配置してワークフローを設計する作業領域。ズームやパンが可能です。

右サイドバー: ノード設定パネル

選択したノードの詳細設定を行います。パラメータ、入出力変数などを設定します。

上部ツールバー: 実行・保存・テスト

ワークフローの実行、保存、デバッグ実行などの操作ボタンがあります。

操作 方法
ノードを追加 左サイドバーからノードをドラッグしてキャンバスにドロップ
ノード同士を接続 ノードの出力ポート（右側）から次のノードの入力ポート（左側）へドラッグ
ノードを削除 ノードを選択してDeleteキーまたは右クリックメニューから削除
ノードを設定 ノードをクリックして右サイドバーで各種パラメータを編集
ワークフローをテスト 上部ツールバーの「実行」ボタンをクリック

9.3 最初のワークフローを作成する

それでは、実際にシンプルなワークフローを作成してみましょう。 ここでは「ユーザーの入力文を受け取り、LLMで要約を生成し、結果を出力する」という基本的なワークフローを構築します。

実習: テキスト要約ワークフローを作成

ステップ1: ワークフローアプリを新規作成

  1. Difyダッシュボードから「新しいアプリを作成」→「ワークフロー」を選択
  2. アプリ名を「テキスト要約」と入力して作成
  3. ワークフローエディタが開きます

ステップ2: 開始ノードを設定

  1. 既に配置されている「開始」ノードをクリック
  2. 右サイドバーで「入力変数を追加」をクリック
  3. 変数名: original_text、型: テキスト、必須: ON
  4. ラベル: 「要約したい文章」、説明: 「要約対象のテキストを入力してください」

ステップ3: LLMノードを追加

  1. 左サイドバーから「LLM」ノードをキャンバスにドラッグ
  2. 開始ノードの出力ポートからLLMノードの入力ポートへ接続
  3. LLMノードをクリックして設定を開く
  4. モデル: GPT-4o mini（またはお好みのモデル）
  5. システムプロンプト: 「あなたは要約の専門家です。以下のテキストを3文以内で簡潔に要約してください。」
  6. ユーザープロンプト: {{#start.original_text#}}（開始ノードの変数を参照）

ステップ4: 終了ノードを追加

  1. 左サイドバーから「終了」ノードをキャンバスにドラッグ
  2. LLMノードの出力ポートから終了ノードの入力ポートへ接続
  3. 終了ノードをクリックして設定を開く
  4. 出力変数を追加: 変数名 summary、値 {{#llm.text#}}

9.4 ワークフローのテスト実行

ワークフローを作成したら、正しく動作するかテスト実行して確認しましょう。

動作確認: ワークフローを実行

  1. 上部ツールバーの「実行」ボタンをクリック

    右サイドバーにテスト実行パネルが表示されます。

  2. 入力変数に値を入力

    例: 「人工知能（AI）は、機械が人間のような知的な振る舞いをする技術です。機械学習、深層学習、自然言語処理など多様な分野を含み、 画像認識、音声認識、自動運転などに応用されています。近年の発展により、ビジネスや医療、教育など様々な領域で活用が進んでいます。」

  3. 「実行」ボタンをクリック

    ワークフローが実行され、各ノードの実行状況がリアルタイムで表示されます。

  4. 実行結果を確認

    終了ノードの出力に要約結果が表示されます。

    例: 「AIは機械が知的に振る舞う技術で、機械学習や自然言語処理を含みます。画像・音声認識や自動運転に応用され、ビジネスや医療など多分野で活用が拡大しています。」

デバッグ機能の活用

実行中、各ノードをクリックすると、そのノードの入力値と出力値を詳細に確認できます。 期待通りの結果が得られない場合は、ノードごとにデータを追跡して問題箇所を特定しましょう。

9.5 ワークフローの保存と公開

ワークフローが正しく動作することを確認したら、保存して公開します。

ワークフローの保存

Difyは自動保存機能がありますが、重要な変更後は手動で「保存」ボタンをクリックして確実に保存しましょう。

ワークフローの公開

右上の「公開」ボタンをクリックすると、ワークフローがアプリとして利用可能になります。 公開後は、Webアプリとしてアクセスしたり、APIとして呼び出したりできます。

  • • Webアプリ: ブラウザから直接利用できるUIが生成されます
  • • API: 他のシステムから呼び出せるREST APIが提供されます

ベストプラクティス

  • • ノードには分かりやすい名前をつける（後で見返したときに理解しやすい）
  • • 複雑なワークフローは、論理的なブロックに分けて配置する
  • • こまめにテスト実行して、各ステップが正しく動作するか確認する
  • • エラーが発生したら、ノードの入出力を確認してデバッグする

9.6 ワークフローで使える主要なノード（概要）

次章で詳しく学びますが、ここではDifyワークフローで利用できる主要なノードの種類を簡単に紹介します。

LLMノード

プロンプトを指定してLLMに推論させます。文章生成、要約、翻訳、抽出など、LLMの基本機能を利用します。

条件分岐ノード

IF文のように、条件によって処理フローを分岐させます。データの内容に応じて異なる処理を実行できます。

HTTPリクエストノード

外部APIを呼び出します。天気情報、データベース検索、他のサービスとの連携などに使用します。

コードノード

Pythonコードを直接記述して、複雑なデータ処理や計算を実行します。ノーコードの範囲を超えた柔軟な処理が可能です。

ナレッジベースノード

ナレッジベースから関連ドキュメントを検索します。RAG機能をワークフローに組み込むために使用します。

ループノード

配列データを繰り返し処理します。複数のデータに対して同じ処理を一括実行する場合に使用します。

これらのノードの詳しい使い方は、第10章「ワークフローノードの活用」で学習します。

理解度確認クイズ

  1. Q1. Difyワークフローの主なメリットは何ですか？

    回答を表示

    視覚的に処理の流れを理解できる、ノードを組み合わせて再利用できる、各ノードの入出力を確認しながらデバッグできる、変更が即座に反映されるなどのメリットがあります。

  2. Q2. ワークフローエディタの構成要素を3つ挙げてください。

    回答を表示

    左サイドバー（ノードパレット）、中央（キャンバス）、右サイドバー（ノード設定パネル）、上部ツールバー（実行・保存・テスト）などがあります。

  3. Q3. 実習で作成したテキスト要約ワークフローは、どのノードで構成されていましたか？

    回答を表示

    開始ノード（入力変数original_textを定義）、LLMノード（要約生成）、終了ノード（出力変数summaryを定義）の3つのノードで構成されていました。
