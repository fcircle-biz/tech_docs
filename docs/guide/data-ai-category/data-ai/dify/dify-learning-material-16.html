実践プロジェクト1 - FAQ対応チャットボット

これまで学んだ技術を統合して、ナレッジベースを活用したFAQ対応チャットボットを構築します。企業での実用を想定した設計と実装を行います。

この章で学ぶこと

  • 企業向けFAQチャットボットの要件定義と設計プロセス
  • ナレッジベースの構築とRAG設定の最適化手法
  • エスカレーション機能と回答品質の向上テクニック
  • 実用レベルのチャットボットのテストと改善サイクル

16.1 プロジェクト概要と要件定義

この実践プロジェクトでは、社内ヘルプデスク向けFAQ対応チャットボットを構築します。 社員からのIT関連の問い合わせに自動で回答し、ヘルプデスク担当者の負担を軽減することを目指します。

プロジェクト要件

1. 機能要件

  • • 社員からのIT関連質問に自動回答（VPN接続、パスワードリセット、ソフトウェア使用方法など）
  • • 社内ドキュメントを参照した正確な回答
  • • 回答不可の場合はヘルプデスクへのエスカレーション機能
  • • 日本語での自然な対話

2. 非機能要件

  • • 回答精度: 80%以上の正答率
  • • 応答速度: 5秒以内
  • • 24時間365日稼働
  • • セキュアなアクセス制御（社内からのみアクセス可能）

3. ナレッジソース

  • • 社内ITマニュアル（PDF）
  • • よくある質問FAQ（テキスト）
  • • トラブルシューティングガイド（Word）

16.2 ナレッジベースの構築

高品質なFAQチャットボットには、適切に構造化されたナレッジベースが不可欠です。 ドキュメントの準備、アップロード、最適化を段階的に進めます。

実習: ナレッジベースの作成とドキュメントアップロード

Step 1: ドキュメントの準備

以下のようなドキュメントを準備します：

  • ITマニュアル.pdf: VPN接続手順、メール設定、ソフトウェアインストール方法
  • FAQ.txt: よくある質問と回答（パスワードリセット、アカウントロック解除など）
  • トラブルシューティング.docx: 一般的な問題と解決策

Step 2: ナレッジベースの作成

  1. Difyダッシュボードで「ナレッジベース」→「ナレッジベースを作成」
  2. 名前: 「社内ITヘルプデスクFAQ」
  3. 説明: 「社員向けIT関連FAQ、マニュアル、トラブルシューティングガイド」
  4. 「作成」をクリック

Step 3: ドキュメントのアップロード

  1. 「ドキュメントを追加」をクリック
  2. 準備した3つのファイルをドラッグ&ドロップ
  3. チャンク設定: 自動（推奨）または カスタム（チャンクサイズ: 500トークン、オーバーラップ: 50トークン）
  4. 「アップロードして処理」をクリック
  5. インデックス作成完了まで待機（数分）

ナレッジベース構築のベストプラクティス

  • • 構造化: カテゴリごとにドキュメントを分類（ネットワーク、メール、ソフトウェアなど）
  • • 最新性: 定期的にドキュメントを更新し、古い情報を削除
  • • 明確性: 曖昧な表現を避け、手順は番号付きリストで記載
  • • メタデータ: ドキュメントにタグを付けて検索精度を向上（「VPN」「パスワード」など）

16.3 チャットアプリの構築と設定

ナレッジベースを活用したチャットアプリを作成し、プロンプトとRAG設定を最適化します。

実習: FAQチャットアプリの作成

Step 1: アプリの作成

  1. 「スタジオ」→「アプリを作成」→「チャットアシスタント」
  2. アプリ名: 「社内ITヘルプデスク」
  3. アイコン: （ヘッドセット）

Step 2: モデル設定

モデル: GPT-4o-mini（コストと性能のバランスが良い）

Temperature: 0.2（正確性重視）

Max Tokens: 1000

Step 3: システムプロンプトの設定

システムプロンプト 
あなたは社内ITヘルプデスクのアシスタントです。
社員からのIT関連の質問に、丁寧かつ正確に回答してください。

## 対応範囲
- VPN接続、リモートアクセス
- パスワードリセット、アカウント管理
- メール設定、Outlook関連
- ソフトウェアのインストールと使用方法
- ネットワーク接続トラブル

## 回答方針
1. 提供されたナレッジベースの情報に基づいて回答してください
2. 手順は番号付きリストで明確に説明してください
3. 専門用語は必要に応じて簡単に説明してください
4. 複雑な問題や情報が不足している場合は、ヘルプデスクへの問い合わせを促してください

## エスカレーション基準
以下の場合はヘルプデスクへのエスカレーションを案内してください：
- ナレッジベースに該当情報がない
- ハードウェア障害の可能性がある
- セキュリティインシデントの可能性がある
- 緊急性が高い問題

エスカレーション時のメッセージ：
「この問題は専門のヘルプデスク担当者による対応が必要です。
以下のリンクからチケットを作成してください：[チケット作成URL]
または内線XXXXにお電話ください。」

実習: RAG設定の最適化

Step 1: ナレッジベースの追加

  1. 「コンテキスト」セクションで「ナレッジベース」を追加
  2. 「社内ITヘルプデスクFAQ」を選択

Step 2: 検索設定

検索モード: ハイブリッド検索（セマンティック + キーワード）

Top K: 3（上位3件の関連チャンクを取得）

スコア閾値: 0.7（関連性の低い結果を除外）

リランキング: 有効（取得結果を再ランク付け）

Step 3: 引用の有効化

「引用と帰属を有効化」をONにして、回答の根拠となったドキュメント情報を表示

16.4 エスカレーション機能の実装

チャットボットが回答できない質問を検出し、人間のヘルプデスク担当者にエスカレーションする仕組みを実装します。

実装: エスカレーション判定の追加

システムプロンプトに既に基本的なエスカレーション基準を含めました。 さらに高度な制御が必要な場合は、ワークフローを使用して条件分岐を実装できます。

ワークフローによる高度なエスカレーション（オプション）:

  1. チャットアプリをワークフローモードに変換
  2. RAG検索ノード追加 → スコア取得
  3. 条件分岐ノード: スコア < 0.7 → エスカレーション
  4. HTTPノード: チケット管理APIにPOSTリクエスト

16.5 テストと継続的な改善

構築したチャットボットの品質を検証し、実際の運用を想定したテストと改善を行います。

テストシナリオ

1. 基本的なFAQ質問

  • • 「VPNに接続する方法を教えてください」
  • • 「パスワードを忘れた場合の対処法は？」
  • • 「Outlookでメール送信できません」

期待結果: マニュアルに基づいた明確な手順を回答

2. 曖昧な質問

  • • 「ネットワークがつながらない」
  • • 「PCの調子が悪い」

期待結果: 状況を確認するための質問を返す

3. 範囲外の質問

  • • 「PCが物理的に壊れた」
  • • 「新しいソフトウェアの購入申請方法は？」

期待結果: ヘルプデスクへエスカレーション

継続的な改善サイクル

  1. ログ分析: Difyの「ログと注釈」機能で実際の質問と回答を確認
  2. 問題の特定: 回答精度が低い質問パターンを抽出
  3. ナレッジ追加: 不足している情報をナレッジベースに追加
  4. プロンプト調整: システムプロンプトを微調整
  5. 再テスト: 同じ質問で改善を確認

運用KPI

指標 目標値 測定方法
回答精度 80%以上 ユーザーフィードバック（👍👎）
自動解決率 70%以上 エスカレーション率（100% - 自動解決率）
応答速度 5秒以内 Difyログの応答時間
ユーザー満足度 4.0/5.0以上 アンケート結果

16.6 公開と運用開始

テストが完了したら、社内向けにチャットボットを公開し、運用を開始します。

公開手順

Step 1: Webアプリとして公開

  1. 「公開」ボタンをクリック
  2. 「Webアプリ」を選択
  3. アクセス制御: 「パスワード保護」または「IPアドレス制限」を設定
  4. 公開URLをコピー

Step 2: 社内ポータルへの統合（オプション）

Step 3: 社員への案内

社内メールやSlackで以下を案内：

  • • チャットボットのURL
  • • 対応可能な質問範囲
  • • 使い方のガイド
  • • フィードバック方法

運用監視のポイント

  • • 毎日: エラーログを確認、緊急度の高い問題がないか
  • • 毎週: ユーザーフィードバック（👍👎）を集計、低評価の質問を分析
  • • 毎月: KPIレポート作成、ナレッジベースの更新
  • • 四半期: プロンプトとモデルの見直し、コスト分析

理解度確認クイズ

  1. Q1. FAQチャットボットで回答精度を高めるために重要な設定は何ですか？

    回答を表示

    RAG設定の最適化（ハイブリッド検索、適切なTop K、スコア閾値の設定）、構造化されたナレッジベース、 低いTemperature値（0.2など）での正確性重視、明確なシステムプロンプトによる振る舞い定義が重要です。

  2. Q2. エスカレーション機能はなぜ必要ですか？

    回答を表示

    チャットボットが回答できない質問や、ハードウェア障害・セキュリティインシデントなど専門的な対応が必要な問題を、 人間の担当者に適切に引き継ぐために必要です。これにより、ユーザーが適切なサポートを受けられ、チャットボットの信頼性が向上します。

  3. Q3. 運用開始後に継続的に改善するためにはどのような指標を追跡すべきですか？

    回答を表示

    回答精度（ユーザーフィードバック）、自動解決率（エスカレーション率の逆）、応答速度、ユーザー満足度を追跡すべきです。 また、ログ分析により回答精度が低い質問パターンを特定し、ナレッジベースの追加やプロンプトの調整を行います。

第16章のまとめ

  • 企業向けFAQチャットボットは、要件定義、ナレッジベース構築、RAG設定の最適化、エスカレーション機能の実装という段階で構築する
  • ナレッジベースは構造化し、カテゴリ分類、メタデータ付与、定期的な更新を行うことで検索精度が向上する
  • ハイブリッド検索、適切なTop K、スコア閾値、リランキングによりRAGの回答品質を最適化できる
  • エスカレーション機能により、チャットボットが対応できない質問を人間の担当者に適切に引き継ぐことができる
  • 運用開始後は、ログ分析、KPI追跡、ユーザーフィードバックに基づく継続的な改善サイクルが重要
