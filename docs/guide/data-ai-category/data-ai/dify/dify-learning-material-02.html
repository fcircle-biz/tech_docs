Difyの環境構築 - Dify Cloudとセルフホスト

Dify Cloudへのサインアップ、APIキーの設定、ワークスペース管理の基本と、Dockerによるセルフホスト環境構築を学びます。

この章で学ぶこと

  • Dify CloudとDifyセルフホスト版の違いを理解する
  • Dify Cloudへのサインアップとワークスペースの作成方法
  • LLMプロバイダー（OpenAI等）のAPIキー設定方法
  • Dockerを使ったセルフホスト環境の構築手順

2.1 Dify CloudとDifyセルフホストの違い

Difyには、クラウド版（Dify Cloud）とセルフホスト版の2つの利用形態があります。 どちらを選ぶべきかは、組織の要件によって異なります。

観点 Dify Cloud Difyセルフホスト
初期構築 サインアップのみで即利用可 Docker環境構築が必要
運用コスト インフラ管理不要、メンテナンスフリー サーバー管理・アップデート作業が必要
データ管理 Dify社のクラウド上に保存 自社サーバー内で完結
カスタマイズ性 提供される機能のみ ソースコード変更可能
コスト 無料プラン・有料プランあり サーバー費用のみ（ライセンス無料）
推奨ケース まず試したい、メンテナンスを避けたい 機密情報を外部に出せない、カスタマイズしたい

初学者への推奨

まずはDify Cloudで学習を始めることをお勧めします。 環境構築の手間なく、すぐにDifyの機能を体験できます。 本番運用で機密性が求められる場合は、後からセルフホストに移行することも可能です。

2.2 Dify Cloudへのサインアップ

Dify Cloudを使い始めるための手順を順番に見ていきましょう。

実習: Dify Cloudアカウント作成

  1. 1. https://cloud.dify.ai/ にアクセスします。
  2. 2. 「Sign Up」または「今すぐ始める」ボタンをクリック。
  3. 3. 以下のいずれかの方法でサインアップ：
    • • GitHubアカウント
    • • Googleアカウント
    • • メールアドレス＋パスワード
  4. 4. メールアドレス認証（メール登録の場合）を完了します。
  5. 5. ワークスペース名を入力（例：「My AI Workspace」）して作成。

2.3 LLMプロバイダーのAPIキー設定

DifyでAIアプリを動かすには、LLM（大規模言語モデル）プロバイダーのAPIキーが必要です。 OpenAIを例に、APIキーの取得と設定方法を見ていきましょう。

APIキーについて

APIキーは、LLMサービスを利用するための認証情報です。 OpenAIの場合、APIキーを取得するにはクレジットカード登録が必要で、使用量に応じて課金されます。 無料プランでは利用できないため、注意してください。

実習: OpenAI APIキーの取得と設定

【ステップ1】OpenAI APIキーの取得

  1. 1. https://platform.openai.com/ にアクセスし、サインイン。
  2. 2. 右上のアカウントメニューから「View API keys」を選択。
  3. 3. 「Create new secret key」をクリック。
  4. 4. キー名を入力（例：「Dify Test」）して、「Create secret key」をクリック。
  5. 5. 表示されたAPIキー（sk-で始まる文字列）をコピーして保存。
    ※ この画面を閉じると二度と表示されません

【ステップ2】DifyへのAPIキー設定

  1. 1. Difyダッシュボードの右上にあるワークスペース名をクリック。
  2. 2. 「Settings」→「Model Provider」を選択。
  3. 3. 「OpenAI」のカードを探して「Setup」または「設定」をクリック。
  4. 4. 「API Key」欄にコピーしたOpenAI APIキーを貼り付け。
  5. 5. 「Save」または「保存」をクリック。

APIキーの取り扱い注意

  • • APIキーは絶対に公開しないでください（GitHubへのコミットなど）
  • • 不正利用されると高額な課金が発生する可能性があります
  • • 使用量上限を設定しておくことを推奨します（OpenAI → Billing → Usage limits）
  • • 漏洩した疑いがある場合は、すぐにAPIキーを無効化（Revoke）してください

その他のLLMプロバイダー

Difyは以下のようなプロバイダーにも対応しています：

  • Anthropic Claude: Claude 3.5 Sonnetなどが利用可能。高度な推論能力が特徴。
  • Google Gemini: Google AI StudioでAPIキー取得。無料枠あり。
  • Azure OpenAI: Azureサブスクリプション経由でOpenAIモデルを利用。企業向け。
  • Ollama（ローカル）: セルフホスト環境でローカルLLMを使用。無料だが性能は制限的。

2.4 ワークスペースの基本操作

Difyのワークスペースは、アプリやナレッジベースを管理する作業場所です。 主要なメニューと機能を理解しておきましょう。

Apps（アプリ）

作成したAIアプリケーションの一覧と管理。 チャットボット、ワークフロー、エージェントなど、全てのアプリがここに表示されます。

Datasets（ナレッジベース）

RAGで使用するドキュメントを管理。 PDF、Word、テキストファイルをアップロードし、AIが参照可能なナレッジベースを構築します。

Tools（ツール）

ビルトインツールとカスタムツールの管理。 Web検索、画像生成、計算など、AIが利用可能なツールを設定します。

Settings（設定）

ワークスペース設定、Model Provider（LLMプロバイダー）の設定、メンバー管理などを行います。

ワークスペースの切り替え

Difyでは複数のワークスペースを作成できます。 個人用と仕事用、プロジェクトごとに分けて管理するとわかりやすくなります。 左上のワークスペース名をクリックすると、ワークスペース切り替えやメンバー招待が可能です。

2.5 Dockerによるセルフホスト環境構築（参考）

機密情報を扱う場合や完全なカスタマイズが必要な場合は、Difyをセルフホストで構築します。 ここでは、Dockerを使った基本的なセットアップ方法を紹介します。

前提条件

  • • Docker Desktop（Windows/Mac）またはDocker Engine（Linux）がインストール済み
  • • Docker Composeがインストール済み（Docker Desktop含む場合は不要）
  • • 最低4GB以上のメモリを持つサーバー環境
  • • Git（ソースコードのクローンに使用）

実習: Difyセルフホスト環境構築

【ステップ1】リポジトリのクローン

bash 
# Difyリポジトリをクローン
git clone https://github.com/langgenius/dify.git
cd dify/docker

【ステップ2】環境変数ファイルのコピー

bash 
# .envファイルをコピー
cp .env.example .env

【ステップ3】Docker Composeで起動

bash 
# コンテナをバックグラウンドで起動
docker compose up -d

【ステップ4】起動確認

ブラウザで以下にアクセス：

  • • http://localhost:80 - Difyダッシュボード（初回アクセス時にアカウント作成）
  • • http://localhost:5001 - API（オプション確認）

トラブルシューティング

Q. ポート80が既に使われている

A. `.env`ファイルで`EXPOSE_NGINX_PORT`を変更（例：8080）

Q. コンテナ起動に失敗する

A. `docker compose logs`でログ確認。メモリ不足の場合はDocker Desktopの設定でメモリを増やす。

Q. データが消えてしまった

A. `docker/volumes`ディレクトリが永続化されているか確認。`docker compose down -v`は使わない（ボリューム削除される）。

セルフホストのメリット・デメリット

メリット：

  • • データを完全に自社管理できる（セキュリティ・機密性）
  • • ソースコード変更でカスタマイズ可能
  • • インターネット接続不要なオフライン環境でも利用可能

デメリット：

  • • サーバー管理・メンテナンスの手間
  • • アップデート作業が必要（セキュリティパッチなど）
  • • 初期構築の技術的ハードル

理解度確認クイズ

  1. Q1. Dify CloudとDifyセルフホストの主な違いは何ですか？

    回答を表示

    Dify Cloudはサインアップのみで即利用可能でメンテナンスフリーですが、データはDify社のクラウドに保存されます。 セルフホストは自社サーバーで構築するため初期構築とメンテナンスの手間はかかりますが、データを完全に自社管理でき、カスタマイズも可能です。

  2. Q2. DifyでLLMを利用するために必要な設定は何ですか？

    回答を表示

    LLMプロバイダー（OpenAI、Anthropic、Googleなど）のAPIキーが必要です。 Difyの「Settings」→「Model Provider」からAPIキーを設定することで、そのプロバイダーのモデルを利用できるようになります。

  3. Q3. OpenAI APIキーの取り扱いで注意すべきことは何ですか？

    回答を表示

    APIキーは絶対に公開してはいけません（GitHubへのコミット等）。 不正利用されると高額な課金が発生する可能性があるため、使用量上限を設定し、漏洩の疑いがある場合はすぐに無効化（Revoke）する必要があります。

第2章のまとめ

  • Dify Cloudは即座に利用開始でき、セルフホストはデータを自社管理できる特徴がある
  • Dify Cloudへのサインアップは、GitHub/Google/メールアドレスで簡単に行える
  • LLMプロバイダーのAPIキーを設定することで、OpenAI、Claude、Geminiなどのモデルを利用可能
  • セルフホスト環境はDockerで構築でき、完全な自社管理とカスタマイズが可能
