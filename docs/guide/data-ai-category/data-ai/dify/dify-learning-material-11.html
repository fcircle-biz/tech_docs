エージェント機能 - AIに自律的に行動させる

Difyのエージェント機能を使って、AIが状況に応じてツールを選択・実行する自律的なアプリケーションを構築します。Function CallingとReActの違いを理解します。

この章で学ぶこと

  • エージェントとは何か、ワークフローとの違いを理解する
  • Function CallingとReActの2つのエージェントモードの違いと使い分け
  • エージェント機能を使った自律的なAIアプリケーションの構築方法
  • エージェントのプロンプト設計とツール選択精度の向上テクニック

11.1 エージェントとは何か

エージェント（Agent）は、AIが状況に応じて自律的にツールを選択し、実行する仕組みです。 ワークフローが「事前に定義された手順を順番に実行する」のに対し、エージェントは「目的を達成するために、AIが最適なツールと実行順序を判断する」点が大きく異なります。

観点 ワークフロー エージェント
実行フロー 事前定義された固定フロー AIが動的に判断
柔軟性 条件分岐のみ 状況に応じて柔軟に対応
予測可能性 高い（必ず同じ流れ） 低い（AIの判断で変化）
向いている用途 定型処理の自動化 多様な質問への対応
コスト 低い 高い（複数回のLLM呼び出し）

エージェントの動作概念

「東京の天気と最適な服装を教えて」"] --> B[エージェント: LLMが分析] B --> C{必要なツールを判断} C -->|ツール1| D["Web検索ツール:
東京の天気を検索"] C -->|ツール2| E["計算ツール:

エージェントの本質

エージェントは「人間のように考えて行動する」AI です。ユーザーの質問を理解し、「この質問に答えるにはどのツールが必要か？」を自分で考え、実行し、結果を統合して回答します。

11.2 Function CallingとReActの違い

Difyのエージェント機能には、Function CallingとReActという2つのモードがあります。それぞれの特徴と使い分けを理解しましょう。

Function Calling（関数呼び出し）

LLMが「どの関数を、どんなパラメータで呼び出すべきか」を判断し、構造化されたJSON形式で指示を返します。 OpenAIのGPT-4、Anthropic Claudeなど、多くのモデルがネイティブでサポートしています。

動作の流れ:

  1. ユーザーの質問を分析
  2. 必要なツールとパラメータをJSON形式で出力
  3. Difyがツールを実行
  4. 結果をLLMに渡して最終回答を生成

メリット: 精度が高く、並列実行可能、構造化された出力

デメリット: 対応モデルが限られる

ReAct（Reasoning + Acting）

LLMが「思考（Thought）→ 行動（Action）→ 観察（Observation）」のサイクルを繰り返し、段階的に問題を解決します。 プロンプトベースの手法なので、どのLLMでも利用可能です。

動作の流れ:

  1. Thought: 「まず天気を調べる必要がある」
  2. Action: Web検索ツールを実行
  3. Observation: 「晴れ、25℃」
  4. Thought: 「次に服装を判定する」
  5. Action: 計算ツールを実行
  6. Observation: 「半袖が適切」
  7. Final Answer: 最終回答を生成

メリット: どのモデルでも使える、推論過程が可視化

デメリット: 精度がやや低い、並列実行不可、トークン消費が多い

Function Calling vs ReAct

JSON出力"] FC2 --> FC3["並列実行"] FC3 --> FC4["結果統合"] end subgraph RA["ReAct"] RA1["Thought 1"] --> RA2["Action 1"] RA2 --> RA3["Observation 1"] RA3 --> RA4["Thought 2"] RA4 --> RA5["Action 2"] RA5 --> RA6["Final Answer"] end FC4 -.速い.- X[ ] RA6 -.柔軟.- Y[ ]

使い分けの目安

Function Calling: GPT-4、Claudeなど対応モデルを使う場合、精度重視、並列処理が必要な場合
ReAct: 対応モデルがない場合、推論過程を可視化したい場合、複雑な多段階推論が必要な場合

11.3 エージェントアプリの作成

実際にエージェント機能を使ったアプリケーションを作成してみましょう。ここでは、Web検索と計算ツールを組み合わせた簡単なエージェントを構築します。

実習 11-1: 情報検索エージェントの作成

Web検索ツールを使って、ユーザーの質問に自律的に答えるエージェントを作成します。

手順

  1. 1 Difyダッシュボードで「新規アプリを作成」→「エージェント」を選択
  2. 2 モデル設定で「GPT-4」または「Claude 3.5」など、Function Calling対応モデルを選択
  3. 3 エージェントモードで「Function Calling」を選択
  4. 4 「ツールを追加」→「Web Search」を選択（SerpAPI、Google Search APIなど）
  5. 5 システムプロンプトを設定
  6. 6 「公開」ボタンをクリックして保存
システムプロンプト例 
あなたは情報検索のエキスパートです。
ユーザーの質問に答えるために、Web検索ツールを使って最新の情報を調べてください。

【重要な指示】
- 検索結果は必ず複数のソースを確認し、信頼性の高い情報を優先してください
- 検索キーワードは、質問の意図を正確に反映したものを選んでください
- 最終回答には、参照したソースのURLを含めてください
- 検索結果が見つからない場合は、その旨を正直に伝えてください

動作確認

以下のような質問を試してみましょう：

  • • 「2024年のノーベル物理学賞の受賞者を教えて」
  • • 「最新のiPhoneモデルの特徴は？」
  • • 「東京の明日の天気予報は？」

期待される動作

1. エージェントがWeb検索ツールを自動的に選択
2. 適切なキーワードで検索を実行
3. 検索結果を分析して回答を生成
4. 参照元のURLと共に回答を表示

11.4 複数ツールを組み合わせたエージェント

エージェントの真価は、複数のツールを組み合わせて使う場面で発揮されます。 Web検索、計算、データベース検索などを状況に応じて使い分けるエージェントを構築してみましょう。

実習 11-2: 多機能リサーチエージェント

Web検索、計算、知識ベース検索を組み合わせた高度なエージェントを作成します。

追加するツール

  1. 1 Web Search: 最新情報の検索
  2. 2 Calculator: 数値計算、単位変換
  3. 3 Knowledge Base: 社内文書の参照（事前に作成したナレッジベースを指定）
  4. 4 Current Time: 現在時刻の取得

多機能エージェントのシステムプロンプト例 
あなたは多機能なリサーチアシスタントです。
ユーザーの質問に答えるために、以下のツールを適切に使い分けてください。

【利用可能なツール】
1. Web Search: 最新のニュース、製品情報、一般的な知識を検索
2. Calculator: 数値計算、単位変換、日数計算などを実行
3. Knowledge Base: 社内文書、マニュアル、規定を検索
4. Current Time: 現在の日時を取得

【ツール選択のガイドライン】
- 「最新」「今年」「現在」などのキーワードがあればWeb Search
- 数値計算、パーセント、金額計算などはCalculator
- 社内の規定、手順、ポリシーに関する質問はKnowledge Base
- 日付や時刻が必要な場合はCurrent Time
- 複数のツールが必要な場合は、順番に実行してください

【回答スタイル】
- 使用したツールと取得した情報を明示
- 計算結果は式と答えの両方を示す
- 不確かな情報は推測せず、その旨を伝える

テスト質問例

  • 複合質問: 「今日から1週間後の日付と、その日の東京の天気予報は？」
    → Current Time + Calculator + Web Search を組み合わせる
  • 計算+検索: 「最新のiPhoneの価格を米ドルから日本円に換算して」
    → Web Search + Calculator を組み合わせる
  • 社内情報+計算: 「有給休暇の規定と、今年の残日数から使用率を計算して」
    → Knowledge Base + Calculator を組み合わせる

11.5 エージェントのプロンプト設計

エージェントの性能は、プロンプト設計で大きく変わります。ツール選択の精度を高め、適切な回答を生成するためのプロンプト設計テクニックを学びましょう。

1. ツールの使用条件を明確に記述する

「いつ、どのツールを使うべきか」を具体的に指示することで、ツール選択の精度が向上します。

例: 「ユーザーが『最新の』『今年の』『現在の』というキーワードを使った場合は、必ずWeb Searchツールを使用してください」

2. ツール実行結果の扱い方を指示する

ツールが返した結果をどう処理し、どう回答に反映するかを明示します。

例: 「Web検索の結果は、必ず参照元のURLと共に提示してください。複数の情報源がある場合は、それぞれを比較してください」

3. エラーハンドリングを定義する

ツール実行に失敗した場合や、期待する結果が得られなかった場合の対応を指示します。

例: 「検索結果が見つからない場合は、別のキーワードで再検索を試みてください。3回試しても見つからない場合は、その旨を正直に伝えてください」

4. 回答フォーマットを指定する

一貫性のある回答を生成するために、フォーマットを明示します。

例: 「回答は以下の形式で提示してください：
【使用ツール】: [ツール名]
【取得情報】: [結果の要約]
【回答】: [最終回答]」

過度な制約に注意

プロンプトが詳細すぎると、AIの柔軟性が失われます。基本的なガイドラインを示しつつ、エージェントが状況に応じて判断できる余地を残すことが重要です。

11.6 エージェントのデバッグとモニタリング

エージェントは自律的に動作するため、デバッグやモニタリングが重要です。Difyの機能を使って、エージェントの動作を確認し、改善する方法を学びましょう。

ログとトレースの確認

Difyの「ログ」タブでは、エージェントがどのツールを選択し、どのような結果を得たかを時系列で確認できます。

確認すべきポイント:

  • ユーザーの質問に対して、適切なツールが選択されているか
  • ツールに渡されたパラメータは正確か
  • ツールの実行結果は期待通りか
  • 最終回答は質問に適切に答えているか
  • トークン消費量は許容範囲内か

よくある問題と対処法

問題: エージェントが適切なツールを選択しない

原因: システムプロンプトでツールの使用条件が不明確

解決方法: プロンプトに「いつ、どのツールを使うか」の具体例を追加。ツール名だけでなく、ツールの目的と使用シーンを明記する。

問題: ツールの実行結果が不正確

原因: ツールに渡すパラメータが不適切

解決方法: プロンプトで「パラメータの抽出方法」を指示。例：「検索キーワードはユーザーの質問から重要な名詞と動詞を抽出して作成してください」

問題: 回答生成に時間がかかりすぎる

原因: ReActモードで不必要に多くの思考ステップを実行している

解決方法: Function Callingモードに切り替える。または、プロンプトで最大ステップ数を制限する指示を追加。

問題: トークン消費量が予算を超える

原因: 複数ツールの結果を全て含めた長大なコンテキスト

解決方法: プロンプトで「ツール実行結果は要約して使用してください」と指示。または、より小さいモデル（GPT-3.5など）を試す。

理解度確認クイズ

  1. Q1. エージェントとワークフローの最大の違いは何ですか？

    回答を表示

    ワークフローは事前に定義された固定的な処理フローを順番に実行しますが、エージェントはAIが状況に応じて動的にツールを選択し、実行順序を判断します。エージェントは柔軟性が高い一方、予測可能性は低くなります。

  2. Q2. Function CallingとReActの違いと、それぞれの使い分けを説明してください。

    回答を表示

    Function CallingはLLMがツールとパラメータをJSON形式で出力し、並列実行可能で精度が高いですが、対応モデルが限られます。ReActは思考→行動→観察のサイクルを繰り返すプロンプトベース手法で、どのモデルでも使えますが、精度がやや低くトークン消費が多くなります。GPT-4やClaudeを使う場合はFunction Calling、それ以外や推論過程を可視化したい場合はReActを選びます。

  3. Q3. エージェントのプロンプト設計で重要なポイントは何ですか？

    回答を表示

    ツールの使用条件を明確にする、ツール実行結果の扱い方を指示する、エラーハンドリングを定義する、回答フォーマットを指定する、といった点が重要です。ただし、過度に詳細な指示はAIの柔軟性を損なうため、基本ガイドラインを示しつつ判断の余地を残すバランスが大切です。

第11章のまとめ

  • エージェントはAIが状況に応じて自律的にツールを選択・実行する仕組みで、ワークフローとは異なる柔軟性を持つ
  • Function Calling（高精度・並列実行可能）とReAct（どのモデルでも使える・推論過程可視化）の2つのモードがある
  • 複数のツールを組み合わせることで、Web検索、計算、知識ベース参照などを統合した高度なエージェントを構築できる
  • 適切なプロンプト設計とログモニタリングにより、エージェントの精度と効率を継続的に改善できる
