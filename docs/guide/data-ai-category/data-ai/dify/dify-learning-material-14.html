アプリの公開と共有 - 作ったアプリを届ける

作成したDifyアプリをWebアプリとして公開する方法、APIとして提供する方法、iframe埋め込みで既存サイトに統合する方法を学びます。アクセス制御とセキュリティ設定も理解します。

この章で学ぶこと

  • Difyアプリの3つの公開方法（Webアプリ、API、iframe埋め込み）
  • 公開設定とアクセス制御（パブリック/プライベート）
  • APIキーの管理とセキュリティのベストプラクティス
  • カスタムドメインとブランディング設定

14.1 Difyアプリの公開方法概要

Difyで作成したアプリケーションは、用途に応じて3つの方法で公開・共有できます。 それぞれの方法には特徴と適切な利用シーンがあります。

14.2 Webアプリとして公開する

最もシンプルな公開方法は、DifyがホストするWebアプリとしてアプリを公開することです。 URLを共有するだけで、ユーザーはブラウザからアプリにアクセスできます。

実習: Webアプリとして公開

  1. 1. 公開設定画面を開く
    • • アプリ編集画面の右上「公開」ボタンをクリック
    • • 「Webアプリ」タブを選択
  2. 2. 公開範囲を設定
    • • パブリック: 誰でもアクセス可能（URLを知っていれば誰でも）
    • • プライベート: パスワードやログインが必要
  3. 3. ブランディング設定（オプション）
    • • アプリアイコンを設定
    • • アプリ名と説明文をカスタマイズ
    • • 初期メッセージや例示質問を設定
  4. 4. 公開URLを取得
    • • 「公開」ボタンをクリック
    • • 生成されたURLをコピー
    • • ブラウザで開いてテスト

公開されたWebアプリのURLの例

公開URL 
# Dify Cloudの場合
https://udify.app/chat/[app-id]

# セルフホストの場合
https://your-domain.com/chat/[app-id]

# パブリックアクセス（例）
https://udify.app/chat/abc123xyz456

# プライベートアクセス（パスワード保護）
https://udify.app/chat/abc123xyz456?protected=true

Webアプリ公開のメリット

• 技術的な知識不要で即座に公開可能
• Difyがホスティングとメンテナンスを管理
• モバイル対応のレスポンシブデザイン
• リアルタイムでプロンプトや設定の変更が反映される

14.3 APIとして提供する

DifyアプリをAPIとして公開すると、外部のプログラムやシステムから呼び出すことができます。 これにより、既存のアプリケーションやワークフローにAI機能を統合できます。

API公開の流れ

sequenceDiagram participant Dev as 開発者 participant Dify as Dify管理画面 participant Client as クライアント
プログラム participant API as Dify API participant App as Difyアプリ Dev->>Dify: APIキーを生成 Dify-->>Dev: APIキーを取得 Dev->>Client: APIキーを設定 Client->>API: リクエスト送信
（APIキー付き） API->>App: アプリ実行 App-->>API: 結果を返す API-->>Client: レスポンス

実習: APIとして公開

  1. 1. APIキーを生成
    • • アプリ編集画面の「アクセス」タブを開く
    • • 「APIキーを作成」をクリック
    • • キーの名前と権限（読み取りのみ/書き込み可能）を設定
  2. 2. APIエンドポイントを確認
    • • APIドキュメントタブでエンドポイントを確認
    • • リクエストフォーマット、パラメータを把握
  3. 3. APIを呼び出す
    • • Pythonやcurl等でAPIをテスト
    • • APIキーをAuthorizationヘッダーに含める

PythonでDify APIを呼び出す例

call_dify_api.py 
import requests
import json

# APIエンドポイントとAPIキー
API_URL = "https://api.dify.ai/v1/chat-messages"
API_KEY = "your-api-key-here"

# リクエストヘッダー
headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

# リクエストボディ
data = {
    "inputs": {},
    "query": "Difyの主な機能を教えてください",
    "response_mode": "blocking",  # ストリーミングなしの場合
    "user": "user-123"
}

# APIリクエスト送信
response = requests.post(API_URL, headers=headers, json=data)
result = response.json()

# レスポンス表示
print("回答:", result["answer"])

curlでDify APIを呼び出す例

bash 
curl -X POST "https://api.dify.ai/v1/chat-messages" \
  -H "Authorization: Bearer your-api-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": {},
    "query": "Difyの主な機能を教えてください",
    "response_mode": "blocking",
    "user": "user-123"
  }'

API利用時の重要ポイント

  • • APIキーは秘密情報: Gitリポジトリにコミットしない、環境変数で管理
  • • レート制限: 短時間に大量のリクエストを送ると制限される場合がある
  • • ストリーミングモード: response_mode=streamingで、リアルタイムに回答を受信可能
  • • 会話履歴: conversation_idを指定することで会話を継続できる

14.4 iframe埋め込みで既存サイトに統合

iframe埋め込みを使うと、既存のWebサイトにDifyアプリを組み込むことができます。 HTMLコードを数行追加するだけで、企業サイトにAIチャットボットを設置できます。

実習: iframe埋め込みを設定

  1. 1. 埋め込みコードを取得
    • • 公開設定画面の「埋め込み」タブを選択
    • • 埋め込みスタイル（フルスクリーン/ポップアップ/固定ウィジェット）を選択
    • • 生成されたHTMLコードをコピー
  2. 2. WebサイトにHTMLコードを貼り付け
    • • body閉じタグの直前に貼り付け
    • • カスタマイズ（サイズ、配置、色など）を設定
  3. 3. 動作確認
    • • Webサイトを開いてチャットが表示されるか確認
    • • レスポンシブデザインが正しく動作するか確認

iframe埋め込みのHTMLコード例

index.html 
<!-- ページ右下に固定チャットウィジェット -->
<script>
  window.difyChatbotConfig = {
    token: 'your-token-here',
    baseUrl: 'https://api.dify.ai',
    position: 'bottom-right',  // 表示位置
    theme: 'light',            // lightまたはdark
    buttonColor: '#6366f1'     // ボタンの色
  }
</script>
<script
  src="https://udify.app/embed.min.js"
  defer>
</script>
スタイル 説明 適した用途
フルスクリーン ページ全体に表示 専用ページでのチャット体験
ポップアップ クリックで開閉するウィンドウ サイト訪問者への積極的サポート
固定ウィジェット 画面隅に常時表示のアイコン 控えめなヘルプチャット

iframe埋め込みのカスタマイズ

• デザインの統一: 企業ブランドカラーに合わせてボタンの色を変更
• 初期メッセージ: ユーザーが開いた時に表示されるウェルカムメッセージをカスタマイズ
• 言語設定: 多言語サイトの場合、ユーザーの言語に応じて自動切り替え可能

14.5 アクセス制御とセキュリティ設定

アプリを公開する際には、適切なアクセス制御とセキュリティ対策が重要です。 用途に応じて適切な設定を行いましょう。

1. パブリック/プライベート設定

パブリック: URLを知っている人なら誰でもアクセス可能
プライベート: パスワードまたはログイン認証が必要

推奨: 社外向けはパブリック、社内ツールはプライベート

2. APIキーの管理

• 用途別にAPIキーを分けて管理（本番用、開発用など）
• 不要になったAPIキーは即座に削除
• APIキーの権限を最小限に設定（必要な機能のみ許可）

推奨: 環境変数やシークレット管理ツール（AWS Secrets Manager等）で管理

3. レート制限（Rate Limiting）

• 1ユーザーあたりの1分/1時間あたりのリクエスト数を制限
• 悪意ある大量リクエストからの保護
• コストの異常増加を防止

推奨: 通常利用の10倍程度を上限に設定

4. センシティブデータの除外

• ナレッジベースに個人情報を含めない
• ユーザー入力の内容をログに記録する際の配慮
• 機密情報がプロンプトに含まれないようチェック

推奨: GDPRやPマークなどのコンプライアンスに準拠

公開前のセキュリティチェックリスト

  • APIキーがコードにハードコードされていないか確認
  • ナレッジベースに機密情報が含まれていないか確認
  • プライベートアプリには認証を設定したか確認
  • レート制限を適切に設定したか確認
  • ログやフィードバックに個人情報が残らないか確認

セルフホスト版でのフルカスタマイズ

セルフホスト版を利用すれば、完全なカスタマイズが可能です：
• UIの完全なカスタマイズ（CSS、HTMLの編集）
• 独自の認証システムの統合（SAML、LDAP等）
• オンプレミス環境での運用（データを外部に出さない）

実践問題: アプリを3つの方法で公開してみよう

課題1: Webアプリとして公開

これまでに作成したアプリをWebアプリとして公開し、URLをブラウザで開いて動作確認してください。 パブリック設定とプライベート設定の両方を試してみましょう。

課題2: PythonでAPIを呼び出す

APIキーを生成し、PythonまたはcurlでDify APIを呼び出してください。 質問を変えて複数回リクエストを送り、会話履歴が保持されるか確認しましょう。

課題3: シンプルなHTMLページに埋め込み

iframe埋め込みコードを取得し、シンプルなHTMLページに貼り付けてください。 ローカルでHTMLファイルを開いてチャットが動作するか確認しましょう。

理解度確認クイズ

  1. Q1. Difyアプリの公開方法3つを挙げ、それぞれの適した用途を説明してください。

    回答を表示

    1. Webアプリ: 専用URLで誰でもアクセス可能。社内ツールやデモに最適。
    2. API: プログラムから呼び出し可能。既存システムとの連携や自動化に最適。
    3. iframe埋め込み: 既存Webページに組み込み。企業サイトのヘルプチャットに最適。

  2. Q2. API公開時のセキュリティで特に注意すべき点は何ですか？

    回答を表示

    • APIキーをコード内にハードコードせず、環境変数で管理すること
    • 不要なAPIキーは即座に削除すること
    • APIキーの権限を最小限に設定すること
    • レート制限を設定して大量リクエストからの保護を行うこと

  3. Q3. iframe埋め込みでカスタマイズできる主な項目は何ですか？

    回答を表示

    • 埋め込みスタイル（フルスクリーン、ポップアップ、固定ウィジェット）
    • 表示位置（右下、左下、中央など）
    • テーマ（ライト、ダーク）
    • ボタンの色（企業ブランドカラーに合わせる）
    • 初期メッセージの内容

第14章のまとめ

  • DifyアプリはWebアプリ、API、iframe埋め込みの3つの方法で公開できる
  • Webアプリ公開はURLを共有するだけで誰でもアクセス可能、技術的な知識不要
  • API公開により既存システムとの連携や自動化が実現でき、Pythonやcurlで簡単に呼び出し可能
  • iframe埋め込みで既存サイトにチャット機能を統合でき、デザインのカスタマイズも可能
  • APIキーの適切な管理、レート制限、センシティブデータの除外などセキュリティ対策が重要
  • Enterprise版やセルフホスト版では独自ドメインや完全なブランディングカスタマイズが可能
