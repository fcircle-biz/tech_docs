はじめてのアプリ作成 - チャットボットを作る

Difyで最初のAIアプリケーションを作成します。シンプルなチャットボットを構築しながら、アプリタイプの選択、モデル設定、プロンプト設計の基本を体験します。

この章で学ぶこと

  • Difyのアプリタイプ（チャットアシスタント、テキスト生成、エージェント、ワークフロー）の違い
  • 新規アプリの作成手順とインターフェースの基本操作
  • モデル設定（LLMプロバイダー、モデル選択、パラメータ調整）
  • システムプロンプトの設計とテスト実行の方法

3.1 Difyのアプリタイプを理解する

Difyでは、目的に応じて4つのアプリタイプから選択できます。それぞれの特徴と使い分けを理解しましょう。

アプリタイプ 特徴 用途例
チャットアシスタント 会話履歴を保持する対話型アプリ カスタマーサポート、FAQ対応
テキスト生成 単発の入力に対して結果を生成 文書要約、翻訳、文章生成
エージェント ツールを自律的に選択・実行 Web検索、計算、API呼び出し
ワークフロー 複数ステップを組み合わせた処理 データ処理パイプライン、自動化

初めてのアプリには「チャットアシスタント」がおすすめ

会話形式で試行錯誤しやすく、プロンプトの調整やモデルの切り替えの効果がすぐに確認できます。 この章では「チャットアシスタント」を作成します。

3.2 新規アプリを作成する

Difyで初めてのチャットアシスタントを作成してみましょう。実際の操作手順を確認しながら進めます。

実習: シンプルなチャットボットを作成

ステップ1: アプリ作成画面を開く

  1. Difyにログイン後、ダッシュボードで「新しいアプリを作成」ボタンをクリック
  2. アプリタイプ選択画面が表示される

ステップ2: チャットアシスタントを選択

  1. 「チャットアシスタント（Chatbot）」をクリック
  2. アプリ名を入力（例: 「はじめてのチャットボット」）
  3. 説明（オプション）を入力（例: 「学習用のシンプルなチャットボット」）
  4. 「作成」ボタンをクリック

ステップ3: アプリエディタが開く

アプリ編集画面が表示されます。左側に設定パネル、右側にプレビュー画面が配置されています。

3.3 モデルを設定する

チャットボットが使用するLLM（大規模言語モデル）を選択し、パラメータを調整します。

モデル設定の手順

1. モデルプロバイダーを選択

「モデル」セクションで、使用するLLMプロバイダーを選択します（OpenAI、Anthropic Claude、Google Geminiなど）。 APIキーを設定済みのプロバイダーが表示されます。

2. 具体的なモデルを選択

初心者向けの推奨モデル：

  • • GPT-4o-mini（OpenAI）: コスパに優れた高性能モデル
  • • Claude 3.5 Haiku（Anthropic）: 高速で正確な応答
  • • Gemini 1.5 Flash（Google）: 無料枠が大きく、学習に最適

3. モデルパラメータを調整（任意）

最初はデフォルト設定のままでOKです。後で調整できます。

パラメータ 説明 推奨設定
Temperature（温度） 出力のランダム性を調整（0〜1） 0.7（バランス型）
Max Tokens（最大トークン数） 生成する応答の最大長 1000〜2000
Top P 単語選択の多様性を調整（0〜1） 0.9（デフォルト）

Temperature（温度）の使い分け

低い（0.0〜0.3）: 正確で一貫性のある回答が必要な場合（技術サポート、データ抽出）
中程度（0.5〜0.7）: バランスの取れた会話（カスタマーサポート、FAQ）
高い（0.8〜1.0）: 創造的な内容生成（アイデア出し、物語生成）

3.4 システムプロンプトを設計する

システムプロンプトは、AIの振る舞い・役割・トーンを定義する重要な要素です。 チャットボットの性格や回答スタイルはここで決まります。

良いシステムプロンプトの要素

  • 役割の定義: AIが何者であるか（例: カスタマーサポート担当、技術アドバイザー）
  • トーン・口調: 丁寧語、フレンドリー、専門的など
  • 回答の制約: 何を答えて、何を答えないか
  • フォーマット指示: 箇条書き、構造化された回答など

システムプロンプトの例

システムプロンプト（学習サポートチャットボット） 
あなたはDifyを学習中の初心者向けのサポートアシスタントです。

## 役割
- Difyの機能や操作方法について、分かりやすく丁寧に説明する
- 専門用語を使う場合は、必ず簡単な言葉で補足する
- 実例を交えて説明することを心がける

## 回答スタイル
- 丁寧で親しみやすい口調
- 箇条書きや段落分けで読みやすく
- 複雑な説明は、ステップごとに区切る

## 制約
- Difyに関係のない質問には「Difyに関する質問にお答えします」と返答
- 不明な点は推測せず、「公式ドキュメントをご確認ください」と案内
- コードを提示する場合は、日本語コメントを必ず含める

実習: システムプロンプトを設定してテスト

1. システムプロンプトを入力

左パネルの「プロンプト」セクションで、上記の例を参考にシステムプロンプトを入力します。

2. プレビューでテスト

右パネルのプレビュー画面で、以下の質問を試してみましょう：

  • 「Difyとは何ですか？」
  • 「RAGアプリを作る手順を教えてください」
  • 「今日の天気は？」（制約のテスト）

3. プロンプトを調整

回答が意図通りでない場合、システムプロンプトを修正して再度テストします。

3.5 テスト実行と改善サイクル

Difyの強みは、プロンプトやモデル設定を変更後、すぐにテスト実行できる点です。 この反復改善サイクルを回すことで、高品質なAIアプリが完成します。

テストのベストプラクティス

  • 様々な質問パターンを試す: 想定される質問を幅広くテスト
  • エッジケースを確認: 不適切な質問や範囲外の質問への対応
  • 会話の流れをチェック: 複数回のやり取りで文脈が維持されるか
  • 複数モデルで比較: 同じプロンプトで異なるLLMの挙動を比較

よくある問題と対処法

問題: AIが質問の範囲外のことまで答えてしまう

対処法: システムプロンプトに「制約」セクションを追加し、回答範囲を明確に指定する

問題: 回答が冗長すぎる、または簡潔すぎる

対処法: プロンプトで「200文字程度で回答」などの長さ指定、またはMax Tokensパラメータを調整

問題: 回答の一貫性がない

対処法: Temperature（温度）を下げる（0.3〜0.5）、プロンプトで具体的な回答フォーマットを指定

3.6 アプリを保存・公開する

チャットボットが完成したら、保存して公開しましょう。Difyでは複数の公開方法が用意されています。

ステップ1: アプリを保存

画面右上の「保存」ボタンをクリックして、設定を保存します。 自動保存機能がありますが、明示的に保存することを推奨します。

ステップ2: 公開方法を選択

画面右上の「公開」ボタンから、以下の公開方法を選択できます：

  • Webアプリ: URLでアクセス可能なチャット画面を提供
  • API: 外部システムから呼び出せるAPIとして公開
  • 埋め込み: iframeで既存サイトに統合

ステップ3: 公開URLを取得

Webアプリとして公開すると、専用URLが発行されます。 このURLを共有することで、誰でもチャットボットを利用できるようになります（アクセス制御の設定も可能）。

動作確認: 公開されたチャットボットにアクセス

  1. 発行された公開URLにブラウザでアクセス
  2. チャット画面が表示されることを確認
  3. いくつか質問を投げて、想定通りの回答が返ることを確認
  4. 会話履歴が保持されていることを確認

理解度確認クイズ

  1. Q1. 会話履歴を保持するチャットボットを作りたい場合、どのアプリタイプを選ぶべきですか？

    回答を表示

    「チャットアシスタント（Chatbot）」を選びます。このタイプは会話履歴を自動的に保持し、文脈を考慮した応答が可能です。

  2. Q2. システムプロンプトで必ず定義すべき要素を3つ挙げてください。

    回答を表示

    1) 役割の定義（AIが何者か）、2) トーン・口調（丁寧語、フレンドリーなど）、3) 回答の制約（何を答えて何を答えないか）の3つが重要です。

  3. Q3. AIの回答が冗長すぎる場合、どのような対処法がありますか？

    回答を表示

    システムプロンプトで「200文字程度で簡潔に回答してください」などの長さ指定を追加するか、 モデルパラメータのMax Tokens（最大トークン数）を減らすことで対処できます。

第3章のまとめ

  • Difyには4つのアプリタイプがあり、目的に応じて選択する（チャットアシスタント、テキスト生成、エージェント、ワークフロー）
  • モデル設定では、プロバイダー選択、モデル選択、パラメータ調整を行う。初心者には「GPT-4o-mini」や「Gemini 1.5 Flash」が推奨
  • システムプロンプトは、AIの役割・トーン・制約を定義する重要な要素で、回答品質を大きく左右する
  • Difyではプロンプト修正後すぐにテスト実行できるため、反復改善サイクルを素早く回せる
  • 公開方法は、Webアプリ、API、埋め込みの3つから選択でき、URLで簡単に共有可能
