ツールの活用 - AIの能力を拡張する

Difyに組み込まれたビルトインツールの使い方と、カスタムツールの作成方法を学びます。

この章で学ぶこと

  • Difyのツールとは何か、エージェントやワークフローでどう活用されるかを理解する
  • Web検索、画像生成、計算など主要なビルトインツールの使い方を習得する
  • カスタムツールの作成方法とOpenAPI仕様による外部API統合
  • MCPツールの概念と標準化された外部ツール連携の仕組み

12.1 Difyのツールとは

ツール（Tool）とは、LLMが実行できる外部機能のことです。 LLM自体は「テキスト生成」に特化していますが、ツールを使うことで「Web検索」「計算」「画像生成」「API呼び出し」など、多様な処理を実行できるようになります。

Difyでは、エージェント機能やワークフローノード内でツールを利用することで、AIアプリケーションの能力を大幅に拡張できます。

ツールの仕組み

graph LR A[ユーザー入力] --> B[LLM] B -->|ツール呼び出し判断| C{必要なツールは？} C -->|Web検索| D[Google検索ツール] C -->|計算| E[計算機ツール] C -->|画像生成| F[DALL-E ツール] D --> G[結果を取得] E --> G F --> G G --> B B --> H[最終回答生成]

ツールの役割

ツールは、LLMができないこと（最新情報の取得、計算、外部システム連携）を補完します。 エージェントモードでは、LLMが自律的にどのツールを使うか判断し、ワークフローではノードとして明示的にツールを配置します。

Difyで利用できるツールの種類

  • ビルトインツール: Difyに標準搭載された50以上の機能（Google検索、画像生成、計算など）
  • カスタムツール: OpenAPI仕様で独自のAPIエンドポイントを登録
  • MCPツール: Model Context Protocol準拠の標準化されたツール

12.2 主要なビルトインツール

Difyには50種類以上のビルトインツールが搭載されています。ここでは、特に使用頻度の高い主要ツールを紹介します。

Web検索・情報取得ツール

Google検索

Googleで検索し、検索結果を取得。最新情報やリアルタイムデータの取得に最適。

用途: ニュース検索、事実確認、最新技術情報の取得

Webスクレイピング

指定したURLのWebページからテキスト情報を抽出。

用途: 特定サイトの情報取得、競合分析、コンテンツ収集

Wikipedia検索

Wikipedia記事を検索し、要約や詳細情報を取得。

用途: 用語解説、歴史情報、百科事典的な知識取得

画像・動画処理ツール

DALL-E画像生成

OpenAIのDALL-Eを使ってテキストから画像を生成。

用途: マーケティング素材、説明図、アイコン生成

Vision分析

画像を解析し、内容を説明。Vision対応モデルが必要。

用途: 画像の説明文生成、OCR、画像分類

データ処理・計算ツール

計算機（Calculator）

数式を解析し、計算結果を返す。LLMの計算ミスを防ぐ。

用途: 価格計算、統計処理、単位換算

Pythonコード実行

Pythonコードを実行し、結果を取得（サンドボックス環境）。

用途: データ分析、複雑な計算、グラフ生成

コミュニケーション・通知ツール

メール送信

SMTP経由でメールを送信。

用途: 自動通知、レポート配信、アラート送信

Slack通知

SlackチャンネルにメッセージをPOST。

用途: チームへの通知、アラート、ワークフロー完了通知

実習: Google検索ツールを使ったエージェント

エージェントにGoogle検索ツールを追加し、最新情報を取得できるAIアシスタントを作成しましょう。

  1. 1. Difyで新しいエージェントアプリを作成
  2. 2. 「ツール」セクションから「Google検索」を追加
  3. 3. システムプロンプトに「あなたは最新情報を提供するアシスタントです。必要に応じてGoogle検索ツールを使用してください。」と設定
  4. 4. 「2024年のノーベル賞受賞者は？」と質問してテスト

確認ポイント: AIが自動的にGoogle検索ツールを使用し、検索結果に基づいて回答しているか確認しましょう。

12.3 カスタムツールの作成

ビルトインツールで対応できない独自の処理が必要な場合、カスタムツールを作成できます。 カスタムツールは、OpenAPI仕様（Swagger）に基づいてAPIエンドポイントを定義します。

カスタムツール作成の流れ

graph LR A[外部APIを用意] --> B[OpenAPI仕様を作成] B --> C[Difyにインポート] C --> D[ツール設定] D --> E[エージェント/ワークフローで利用]

OpenAPI仕様の例

以下は、天気情報APIをカスタムツールとして登録するためのOpenAPI仕様の例です。

yaml 
openapi: 3.0.0
info:
  title: 天気情報API
  version: 1.0.0
servers:
  - url: https://api.weather-example.com
paths:
  /weather:
    get:
      summary: 現在の天気を取得
      description: 指定した都市の現在の天気情報を取得します
      parameters:
        - name: city
          in: query
          required: true
          schema:
            type: string
          description: 都市名（例: Tokyo）
      responses:
        '200':
          description: 成功
          content:
            application/json:
              schema:
                type: object
                properties:
                  temperature:
                    type: number
                  condition:
                    type: string
                  humidity:
                    type: number

Difyでのカスタムツール追加手順

  1. 1. ツール管理画面へ: Difyのメニューから「ツール」→「カスタムツール」→「ツールを追加」
  2. 2. OpenAPI仕様をインポート: 上記のYAMLファイルをアップロードまたは貼り付け
  3. 3. 認証情報を設定: APIキーやベアラートークンが必要な場合は設定
  4. 4. テスト実行: ツールの動作確認
  5. 5. エージェント/ワークフローに追加: 作成したカスタムツールを選択

OpenAPI仕様のポイント

LLMがツールを正しく呼び出せるよう、descriptionフィールドに明確な説明を記載することが重要です。 パラメータの意味、期待される入力形式、レスポンスの内容を具体的に記述しましょう。

12.4 MCPツールの活用

MCP（Model Context Protocol）は、Anthropic社が提唱するLLMツール連携の標準規格です。 Difyは2024年末からMCPに対応し、標準化されたツールを簡単に統合できるようになりました。

MCPのメリット

  • 標準化: 統一された仕様でツールを定義・共有
  • エコシステム: 公開されているMCPツールを簡単にインポート
  • 相互運用性: Dify、Claude Desktop、LangChainなど複数のプラットフォームで利用可能

DifyでのMCPツール追加

  1. 1. MCPツールを探す: MCP公式リポジトリやAnthropic MCP紹介ページで利用可能なツールを確認
  2. 2. ツール設定画面へ: Difyの「ツール」→「MCPツール」→「ツールを追加」
  3. 3. サーバー設定: MCPサーバーのエンドポイントを入力（例: http://localhost:3000/mcp）
  4. 4. 認証情報: 必要に応じてAPIキーやトークンを設定
  5. 5. 利用開始: エージェントやワークフローでMCPツールを選択

代表的なMCPツールの例

  • filesystem MCP: ファイルシステムへの読み書き
  • sqlite MCP: SQLiteデータベース操作
  • github MCP: GitHub API経由でのリポジトリ操作
  • google-drive MCP: Google Driveファイル操作
  • slack MCP: Slackメッセージ送信・チャンネル情報取得

MCP連携の仕組み

graph LR A[Difyエージェント] -->|MCP Protocol| B[MCPサーバー] B --> C[filesystem] B --> D[GitHub API] B --> E[Slack API] C --> F[ローカルファイル] D --> G[GitHubリポジトリ] E --> H[Slackワークスペース]

12.5 ツール利用時のベストプラクティス

適切なツールを選択する

エージェントに複数のツールを追加すると、LLMがどのツールを使うか迷う場合があります。 システムプロンプトで「いつどのツールを使うべきか」を明確に指示しましょう。

ツールのコストを把握する

DALL-Eやプレミアム検索APIなど、一部のツールは使用量に応じて課金されます。 APIキーの利用制限や料金体系を確認し、不要な実行を防ぎましょう。

エラーハンドリングを考慮する

ツール実行が失敗した場合（APIエラー、タイムアウト）の対処をシステムプロンプトに記載しましょう。 例: 「検索に失敗した場合は、既存の知識で回答してください」

セキュリティを考慮する

外部APIと連携する際は、APIキーの管理に注意しましょう。 Difyの環境変数機能を使い、ハードコードを避けることが推奨されます。

実習: 複数ツールを活用したリサーチアシスタント

Google検索、Wikipedia、計算機ツールを組み合わせた高度なリサーチアシスタントを作成しましょう。

  1. 1. 新しいエージェントを作成し、以下のツールを追加:
    • • Google検索
    • • Wikipedia検索
    • • 計算機（Calculator）
  2. 2. システムプロンプト設定: 
    あなたは高度なリサーチアシスタントです。
- 最新情報が必要な場合はGoogle検索を使用
- 歴史や定義を調べる場合はWikipedia検索を使用
- 計算が必要な場合は計算機ツールを使用
複数のツールを組み合わせて、正確で包括的な回答を提供してください。
  3. 3. テスト質問例:
    • • 「2024年の世界人口は何人で、前年比何パーセント増加しましたか？」
    • • 「Pythonとは何ですか？最新バージョンはいつリリースされましたか？」

確認ポイント: AIが複数のツールを組み合わせて使用し、総合的な回答を生成しているか確認しましょう。

理解度確認クイズ

  1. Q1. Difyのツールとは何ですか？またどのような種類がありますか？

    回答を表示

    ツールは、LLMが実行できる外部機能で、テキスト生成以外の処理（Web検索、計算、画像生成など）を可能にします。 Difyには、ビルトインツール（標準搭載の50以上の機能）、カスタムツール（OpenAPI仕様で独自APIを登録）、MCPツール（標準化された外部ツール）の3種類があります。

  2. Q2. カスタムツールを作成する際に必要なものは何ですか？

    回答を表示

    カスタムツールを作成するには、OpenAPI仕様（Swagger）に基づくAPI定義が必要です。 エンドポイントURL、パラメータ、レスポンス形式を記述し、LLMが正しく呼び出せるよう詳細な説明を含めます。

  3. Q3. MCP（Model Context Protocol）のメリットは何ですか？

    回答を表示

    MCPは標準化されたツール連携プロトコルで、以下のメリットがあります： 統一された仕様でツールを定義・共有できる、公開されているMCPツールを簡単にインポートできる、Dify以外のプラットフォーム（Claude Desktop、LangChainなど）でも同じツールを利用できる相互運用性。

第12章のまとめ

  • ツールは、LLMの能力を拡張し、Web検索、計算、画像生成、外部API連携などを可能にする
  • Difyには50以上のビルトインツールが搭載され、Google検索、DALL-E、計算機、メール送信などが利用可能
  • カスタムツールはOpenAPI仕様で独自APIを定義し、Difyに統合できる
  • MCP（Model Context Protocol）により、標準化されたツールを複数のプラットフォームで共有・利用できる
  • ツール利用時は、適切な選択、コスト把握、エラーハンドリング、セキュリティ考慮が重要
