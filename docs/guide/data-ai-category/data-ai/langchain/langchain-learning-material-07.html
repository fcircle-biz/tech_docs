LCEL（LangChain Expression Language）入門

LangChainの新しい標準であるLCELを使った、シンプルで直感的なチェーン構築方法を習得します。パイプ演算子を使った処理の連結を体験します。

この章で学ぶこと

  • LCELとは何か、従来のChainとの違いを理解する
  • パイプ演算子（|）を使った処理の連結方法を習得する
  • LCELの基本構文とRunnableインターフェースを学ぶ
  • LCELの実用的なパターンとストリーミング処理を体験する

7.1 LCELとは何か

LCEL（LangChain Expression Language）は、LangChain v0.1.0（2024年1月）で正式に導入された、チェーン構築のための新しい標準構文です。従来のLLMChainやSequentialChainよりも、シンプルで直感的にコードを書けるように設計されています。

LCELの最大の特徴は、パイプ演算子（|）を使って、まるでUnixのシェルコマンドのように処理を繋げられることです。これにより、コードの可読性が向上し、保守性も高まります。

LCELの位置づけ

LCELは従来のChain（LLMChain、SequentialChainなど）を置き換える新標準です。今後のLangChainアプリ開発では、LCELの使用が推奨されています。

7.2 LCELの基本構文

LCELでは、コンポーネントをパイプ演算子（|）で繋ぐことでチェーンを構築します。各コンポーネントはRunnableインターフェースを実装しており、統一的な方法で実行できます。

最もシンプルなLCEL

lcel_basic.py 
from langchain.prompts import ChatPromptTemplate
from langchain.chat_models import ChatOpenAI
from langchain.schema.output_parser import StrOutputParser

# コンポーネントの定義
prompt = ChatPromptTemplate.from_template("{topic}について3行で説明してください。")
model = ChatOpenAI(model="gpt-3.5-turbo")
output_parser = StrOutputParser()

# LCELでチェーンを構築（パイプ演算子で連結）
chain = prompt | model | output_parser

# チェーンの実行
result = chain.invoke({"topic": "LangChain"})
print(result)

LCELの読み方

prompt | model | output_parser は、「プロンプトに入力を渡し、その結果をモデルに渡し、最後にパーサーで整形する」という処理の流れを表現しています。

Runnableインターフェース

LCELで使用できるコンポーネントは、すべてRunnableインターフェースを実装しています。これにより、以下のメソッドが統一的に使えます。

invoke()

単一入力を処理し、結果を返す。最も基本的な実行メソッド。

stream()

結果をストリーミングで返す。長い応答をリアルタイムで受け取る。

batch()

複数の入力を一括処理。効率的な並列実行が可能。

ainvoke()

非同期実行版。async/await構文で使用。

7.3 LCELの実用パターン

LCELでは、様々な処理パターンを簡潔に記述できます。以下、実用的なパターンを紹介します。

パターン1: 複数ステップの連結

multi_step_chain.py 
from langchain.prompts import ChatPromptTemplate
from langchain.chat_models import ChatOpenAI
from langchain.schema.output_parser import StrOutputParser

# ステップ1: トピックから質問を生成
question_prompt = ChatPromptTemplate.from_template(
    "{topic}について、よくある質問を1つ作ってください。"
)

# ステップ2: 質問に回答
answer_prompt = ChatPromptTemplate.from_template(
    "以下の質問に簡潔に答えてください:\n{question}"
)

model = ChatOpenAI(model="gpt-3.5-turbo")

# LCELで多段チェーンを構築
chain = (
    question_prompt
    | model
    | StrOutputParser()
    | {"question": lambda x: x}  # 辞書に変換
    | answer_prompt
    | model
    | StrOutputParser()
)

result = chain.invoke({"topic": "Python"})
print(result)

パターン2: 並列実行

parallel_chain.py 
from langchain.prompts import ChatPromptTemplate
from langchain.chat_models import ChatOpenAI
from langchain.schema.runnable import RunnableParallel

model = ChatOpenAI(model="gpt-3.5-turbo")

# 3つの異なる視点で分析
pros_chain = (
    ChatPromptTemplate.from_template("{topic}のメリットを3つ挙げてください。")
    | model
)
cons_chain = (
    ChatPromptTemplate.from_template("{topic}のデメリットを3つ挙げてください。")
    | model
)
summary_chain = (
    ChatPromptTemplate.from_template("{topic}を1行で要約してください。")
    | model
)

# RunnableParallelで並列実行
parallel_chain = RunnableParallel(
    pros=pros_chain,
    cons=cons_chain,
    summary=summary_chain
)

result = parallel_chain.invoke({"topic": "リモートワーク"})
print(result)  # {'pros': ..., 'cons': ..., 'summary': ...}

パターン3: 条件分岐（RunnableBranch）

branching_chain.py 
from langchain.schema.runnable import RunnableBranch
from langchain.prompts import ChatPromptTemplate
from langchain.chat_models import ChatOpenAI

model = ChatOpenAI(model="gpt-3.5-turbo")

# 技術的な質問とそれ以外で処理を分岐
technical_chain = (
    ChatPromptTemplate.from_template("技術的な質問: {question}\n専門的に回答:")
    | model
)
general_chain = (
    ChatPromptTemplate.from_template("一般的な質問: {question}\n簡潔に回答:")
    | model
)

# 条件分岐
branch = RunnableBranch(
    (lambda x: "コード" in x["question"], technical_chain),
    general_chain  # デフォルト
)

result = branch.invoke({"question": "Pythonのコード最適化方法は？"})
print(result)

7.4 ストリーミング処理

LCELのstream()メソッドを使うと、LLMの応答をリアルタイムで受け取れます。長い応答でもユーザーに即座にフィードバックを提供できるため、UX向上に効果的です。

streaming_chain.py 
from langchain.prompts import ChatPromptTemplate
from langchain.chat_models import ChatOpenAI
from langchain.schema.output_parser import StrOutputParser

prompt = ChatPromptTemplate.from_template(
    "{topic}について詳しく説明してください。"
)
model = ChatOpenAI(model="gpt-3.5-turbo", streaming=True)

# LCELチェーン
chain = prompt | model | StrOutputParser()

# ストリーミングで実行
for chunk in chain.stream({"topic": "機械学習"}):
    print(chunk, end="", flush=True)  # リアルタイム出力

ストリーミングのメリット

チャットボットなど対話型アプリでは、応答が長い場合でもストリーミングを使うことで、ユーザーを待たせずに段階的に情報を提供できます。

実習 7-1: LCELで翻訳チェーンを作成

LCELを使って、日本語から英語への翻訳と、その英語を再度日本語に戻すチェーンを構築してみましょう。

手順

  1. 1 日本語→英語のプロンプトと、英語→日本語のプロンプトを作成
  2. 2 LCELでパイプ演算子を使って2段階の翻訳チェーンを構築
  3. 3 任意の日本語文章を入力し、英語→日本語の変換結果を確認
  4. 4 元の文章と再翻訳後の文章を比較し、意味が保たれているか検証

7.5 LCELのメリットと従来のChainとの使い分け

LCELのメリット

LCELの主なメリット

  • 可読性の向上: パイプ演算子で処理の流れが一目瞭然
  • シンプルな記述: クラスのインスタンス化が不要、コード量が削減
  • 統一的なインターフェース: すべてのコンポーネントがRunnableを実装
  • ストリーミング対応: リアルタイム処理が標準サポート
  • 並列実行の容易さ: RunnableParallelで簡単に並列化

従来のChainとの比較

項目 従来のChain LCEL
記述方法 クラスベース パイプ演算子
コード量 多い 少ない
可読性 ネストが深い フラットで直感的
ストリーミング 手動実装 標準サポート
推奨度 レガシー 新規開発推奨

注意

従来のChain（LLMChain、SequentialChainなど）は引き続き使用可能ですが、LangChain公式では今後の開発ではLCELの使用を推奨しています。

この章のまとめ

  • LCELは、LangChainの新しい標準であり、パイプ演算子でチェーンを構築できる
  • Runnableインターフェースにより、invoke、stream、batchなど統一的な実行方法が提供される
  • RunnableParallelで並列実行、RunnableBranchで条件分岐を簡潔に記述できる
  • ストリーミング処理が標準サポートされており、リアルタイム応答が容易に実装できる
  • LCELは従来のChainよりもコード量が少なく、可読性と保守性が向上する

理解度確認クイズ

  1. Q1. LCELの最大の特徴は何ですか？

    回答を表示

    LCELの最大の特徴は、パイプ演算子（|）を使って処理を連結できることです。これにより、Unixシェルコマンドのように直感的に処理の流れを記述でき、コードの可読性が大幅に向上します。

  2. Q2. Runnableインターフェースで使用できる主なメソッドを4つ挙げてください。

    回答を表示

    ①invoke()（単一入力の処理）、②stream()（ストリーミング実行）、③batch()（複数入力の一括処理）、④ainvoke()（非同期実行）の4つです。これらは全てのRunnableコンポーネントで統一的に使用できます。

  3. Q3. LCELで並列実行を行うにはどのクラスを使用しますか？また、その利点は何ですか？

    回答を表示

    RunnableParallelクラスを使用します。利点は、複数のチェーンを同時に実行できることで、例えば同じ入力に対して「メリット」「デメリット」「要約」など複数の視点から分析する処理を効率的に行えます。結果は辞書形式で返されるため、各チェーンの出力を簡単に取得できます。
