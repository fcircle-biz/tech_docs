第14章

Memory - 会話履歴の管理

ConversationBufferMemory、ConversationSummaryMemoryなど、LLMとの会話履歴を管理する機能を学びます。文脈を保持した対話システムを構築します。

この章で学ぶこと

  • Memoryの基本概念と会話履歴管理の重要性を理解する
  • ConversationBufferMemoryで全履歴を保持する方法を習得する
  • ConversationSummaryMemoryで要約による効率化を学ぶ
  • Memory付きチェーンで文脈保持型チャットボットを構築する

14.1 Memoryとは - 会話の文脈を保つ技術

LLMは基本的にステートレス（状態を持たない）であり、各リクエストは独立して処理されます。Memory（メモリ）は、過去の会話履歴を管理し、LLMに文脈を提供する機能です。

flowchart LR User[ユーザー] Memory[(Memory
会話履歴)] LLM[LLM] User -->|新しい質問| Memory Memory -->|履歴＋質問| LLM LLM -->|回答| User LLM -->|回答を保存| Memory

図14-1: Memoryの仕組み

Memoryが必要な理由

「それについて詳しく教えて」「前に言った内容を要約して」といった、過去の会話を参照する質問に対応するには、会話履歴を保持する仕組みが必須です。

LangChainのMemoryタイプ

ConversationBufferMemory

全ての会話履歴をそのまま保存。シンプルだが、長い会話ではトークン数が増大

ConversationSummaryMemory

会話履歴を要約して保存。トークン節約に有効だが、詳細情報が失われる可能性

ConversationBufferWindowMemory

最新のN件のメッセージのみ保持。長期会話で一定サイズを維持

ConversationSummaryBufferMemory

古い履歴は要約、最新履歴は完全保持。柔軟なトークン管理

14.2 ConversationBufferMemory - 全履歴の保存

最もシンプルなMemoryで、全ての会話をそのまま保存します。短い会話や、詳細な文脈が必要な場合に適しています。

基本的な使い方

# ConversationBufferMemoryの基本
from langchain.memory import ConversationBufferMemory

# Memoryの初期化
memory = ConversationBufferMemory()

# 会話を記録
memory.save_context(
    {"input": "こんにちは！私の名前は太郎です"},
    {"output": "こんにちは太郎さん！何かお手伝いできることはありますか？"}
)

memory.save_context(
    {"input": "Pythonについて教えてください"},
    {"output": "Pythonは初心者にも学びやすいプログラミング言語です..."}
)

# 履歴を確認
print(memory.load_memory_variables({})["history"])
# Human: こんにちは！私の名前は太郎です
# AI: こんにちは太郎さん！何かお手伝いできることはありますか？
# Human: Pythonについて教えてください
# AI: Pythonは初心者にも学びやすいプログラミング言語です...

Chainとの統合

# Memory付きチェーンの構築
from langchain.chains import ConversationChain
from langchain_openai import ChatOpenAI

# LLMの初期化
llm = ChatOpenAI(temperature=0.7)

# Memory付きConversationChain
conversation = ConversationChain(
    llm=llm,
    memory=ConversationBufferMemory(),
    verbose=True  # デバッグ情報を表示
)

# 会話1
response1 = conversation.predict(input="私は東京に住んでいます")
print(response1)

# 会話2（前の会話を覚えている）
response2 = conversation.predict(input="私が住んでいる場所の特徴を教えて")
print(response2)  # 「東京」を参照した回答が返る

ポイント

verbose=Trueを設定すると、LLMに送信されるプロンプト全体が表示され、Memoryがどのように機能しているかを確認できます。

14.3 ConversationSummaryMemory - 要約による効率化

長い会話ではトークン数が増大し、コストや処理時間の問題が発生します。ConversationSummaryMemoryは、会話履歴をLLMで要約することでトークン数を削減します。

flowchart LR Conv1[会話履歴
100ターン] Summary[LLMで要約] Compact[要約版
10ターン相当] Conv1 --> Summary Summary --> Compact

図14-2: 要約による履歴圧縮

# ConversationSummaryMemoryの使用
from langchain.memory import ConversationSummaryMemory
from langchain_openai import ChatOpenAI

# LLM（要約にも使用）
llm = ChatOpenAI(temperature=0)

# 要約型Memory
memory = ConversationSummaryMemory(llm=llm)

# 会話を保存
memory.save_context(
    {"input": "私はデータサイエンティストとして5年働いています"},
    {"output": "素晴らしいご経歴ですね！どのような分野を専門にされていますか？"}
)

memory.save_context(
    {"input": "主に機械学習モデルの構築と最適化です"},
    {"output": "機械学習の専門家なのですね。どのようなフレームワークを使用されていますか？"}
)

# 要約された履歴を確認
print(memory.load_memory_variables({})["history"])
# 「ユーザーは5年経験のデータサイエンティストで、機械学習モデルの構築と最適化を専門としている」

注意点

要約にはLLMの呼び出しが必要なため、追加コストと時間がかかります。また、詳細な情報が失われる可能性があるため、用途に応じて選択してください。

14.4 その他のMemoryタイプ

ConversationBufferWindowMemory

最新のN件のメッセージのみを保持します。長期会話でトークン数を一定に保ちたい場合に有効です。 

# 最新5件のみ保持
from langchain.memory import ConversationBufferWindowMemory

memory = ConversationBufferWindowMemory(k=5)  # 最新5ターン

# 10回会話しても、最新5ターンのみ保持される
for i in range(10):
    memory.save_context(
        {"input": f"質問{i+1}"},
        {"output": f"回答{i+1}"}
    )

# 最新5ターンのみ表示される
print(memory.load_memory_variables({})["history"])

ConversationSummaryBufferMemory

古い履歴は要約し、最新の履歴は完全な形で保持します。柔軟なトークン管理が可能です。 

# ハイブリッド型Memory
from langchain.memory import ConversationSummaryBufferMemory

memory = ConversationSummaryBufferMemory(
    llm=llm,
    max_token_limit=200  # この閾値を超えたら要約
)

# 会話が増えると、古い部分は自動的に要約される
# 最新の会話は完全な形で保持

14.5 Memoryの永続化 - 会話の保存と復元

デフォルトではMemoryはメモリ上に保存されるため、プログラム終了で消失します。会話を保存・復元するには、永続化が必要です。

Chat Message Historyによる永続化

# ファイルへの保存
from langchain.memory import FileChatMessageHistory

# ファイルに保存
history = FileChatMessageHistory("chat_history.json")

# Memoryに統合
memory = ConversationBufferMemory(
    chat_memory=history,
    return_messages=True
)

# 会話は自動的にファイルに保存される
# プログラム再起動後も履歴が維持される

他の永続化オプション

  • • RedisChatMessageHistory: Redis（高速キャッシュDB）に保存
  • • MongoChatMessageHistory: MongoDBに保存
  • • PostgresChatMessageHistory: PostgreSQLに保存

実習: Memory付きチャットボットの構築

ConversationBufferMemoryを使って、文脈を保持した対話型チャットボットを作成しましょう。 

# Memory付きチャットボット
from langchain.chains import ConversationChain
from langchain.memory import ConversationBufferMemory
from langchain_openai import ChatOpenAI

# 初期化
llm = ChatOpenAI(temperature=0.7)
memory = ConversationBufferMemory()

conversation = ConversationChain(
    llm=llm,
    memory=memory
)

# チャットループ
print("チャットボット起動（終了は'quit'）")
while True:
    user_input = input("あなた: ")
    if user_input.lower() == 'quit':
        break

    response = conversation.predict(input=user_input)
    print(f"ボット: {response}\n")

# 会話履歴を確認
print("\n=== 会話履歴 ===")
print(memory.load_memory_variables({})["history"])

実行例：

あなた: 私は昨日、京都に旅行に行きました
ボット: 京都旅行、素敵ですね！どこを訪れましたか？

あなた: 清水寺と金閣寺を見てきました
ボット: 京都の代表的な観光地ですね。どちらも歴史ある美しい寺院です。

あなた: 旅行先でおすすめのお土産はありますか？
ボット: 京都でしたら、八つ橋や抹茶スイーツが定番のお土産として人気です。

※ ボットは「京都旅行」という文脈を保持しています

14.6 Memoryのベストプラクティス

用途に応じたMemory選択

  • • 短い会話: ConversationBufferMemory
  • • 長い会話: ConversationSummaryMemory
  • • 最近の文脈重視: BufferWindowMemory

トークン数の監視

履歴が増えるとトークン消費が増大。定期的に履歴をクリアするか、要約型Memoryを検討

ユーザー別の管理

複数ユーザーがいる場合は、ユーザーIDごとにMemoryインスタンスを分ける

プライバシーとセキュリティ

個人情報を含む会話は暗号化や適切なアクセス制御を実施

この章のまとめ

  • Memoryは、ステートレスなLLMに会話履歴を提供し、文脈保持を可能にする
  • ConversationBufferMemoryは全履歴を保存、シンプルだが長期会話ではトークン増大
  • ConversationSummaryMemoryは要約によりトークン節約、詳細情報は失われる
  • ConversationBufferWindowMemoryは最新N件のみ保持、一定サイズを維持
  • FileChatMessageHistoryなどで会話を永続化し、プログラム再起動後も復元可能

理解度確認クイズ

  1. Q1. LangChainのMemoryが必要な理由を説明してください。

    回答を表示

    LLMは基本的にステートレス（状態を持たない）で、各リクエストは独立して処理されます。そのため、「それについて教えて」「前に言ったことを要約して」といった過去の会話を参照する質問に対応するには、会話履歴を保持・管理するMemoryが必要です。Memoryにより、文脈を保った自然な対話が実現できます。

  2. Q2. ConversationBufferMemoryとConversationSummaryMemoryの違いと、それぞれの適用シーンを説明してください。

    回答を表示

    ConversationBufferMemoryは全ての会話をそのまま保存します。シンプルで詳細な文脈を保持できますが、長い会話ではトークン数が増大します。短い会話や詳細な文脈が必要な場合に適しています。

    ConversationSummaryMemoryは会話履歴をLLMで要約して保存します。トークン数を節約できますが、要約時に詳細情報が失われる可能性があり、要約にもLLM呼び出しコストがかかります。長期会話やコスト削減が重要な場合に適しています。

  3. Q3. Memoryを永続化する方法と、その利点を説明してください。

    回答を表示

    FileChatMessageHistory、RedisChatMessageHistory、MongoChatMessageHistoryなどを使用することで、会話履歴をファイルやデータベースに保存できます。永続化により、プログラム再起動後も会話履歴が維持され、ユーザーが前回の会話を継続できます。また、複数のサーバーで履歴を共有したり、分析用にデータを蓄積したりする用途にも活用できます。
