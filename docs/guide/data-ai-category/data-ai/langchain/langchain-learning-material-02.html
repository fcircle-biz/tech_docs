第2章

環境構築とはじめてのLangChain

LangChainのインストール、APIキーの設定、最初のLLM呼び出しを行います。開発環境を整え、基本的な動作確認を通じてLangChainの使い方を体験します。

この章で学ぶこと

  • Pythonの仮想環境を作成し、LangChainをインストールする
  • OpenAI APIキーを取得・設定する方法を習得する
  • LangChainを使って初めてのLLM呼び出しを実行する
  • 環境変数の管理と.envファイルの安全な使い方を理解する

2.1 開発環境の準備

LangChainを始める前に、Pythonの開発環境を整えましょう。仮想環境を使うことで、プロジェクトごとにパッケージを管理でき、依存関係の問題を回避できます。

前提条件の確認

以下のソフトウェアがインストールされていることを確認してください。

必要な環境

  • • Python 3.9以上（推奨: Python 3.10または3.11）
  • • pip（Pythonと一緒にインストールされます）
  • • テキストエディタまたはIDE（VS Code推奨）
Pythonバージョン確認 
# Pythonのバージョンを確認
python --version
# または
python3 --version

# pip のバージョンを確認
pip --version

仮想環境の作成

プロジェクト専用の仮想環境を作成します。これにより、システム全体のPython環境に影響を与えずに、LangChainと依存パッケージをインストールできます。

仮想環境の作成と有効化 
# プロジェクトフォルダを作成して移動
mkdir langchain-project
cd langchain-project

# 仮想環境を作成
python -m venv venv

# 仮想環境を有効化
# Windowsの場合
venv\Scripts\activate
# macOS/Linuxの場合
source venv/bin/activate

# 有効化されると (venv) がプロンプトに表示される

仮想環境のメリット

仮想環境を使うと、プロジェクトごとに異なるバージョンのパッケージを使い分けられ、依存関係の競合を防げます。また、requirements.txtでパッケージ一覧を管理でき、チーム開発や本番環境への展開が容易になります。

2.2 LangChainのインストール

仮想環境が有効化された状態で、LangChainと関連パッケージをインストールします。

必要なパッケージ

LangChainのインストール 
# LangChainとOpenAI統合パッケージをインストール
pip install langchain langchain-openai

# 環境変数管理用パッケージ
pip install python-dotenv

# インストール確認
pip list | grep langchain

パッケージの説明

  • • langchain: LangChainのコアライブラリ
  • • langchain-openai: OpenAI GPT-4などとの統合パッケージ
  • • python-dotenv: .envファイルから環境変数を読み込むライブラリ

2.3 OpenAI APIキーの取得と設定

LangChainでOpenAIのモデル（GPT-4など）を使うには、APIキーが必要です。

APIキー取得手順

  1. 1 OpenAI Platformにアクセスし、Googleアカウントなどでサインアップ
  2. 2 ダッシュボードから「API keys」セクションに移動
  3. 3 「Create new secret key」ボタンをクリック
  4. 4 表示されたキーをコピー（このキーは一度しか表示されません！）

重要: APIキーの安全管理

APIキーは絶対にコードに直接書かず、環境変数やenvファイルで管理してください。GitHubなどにアップロードすると、第三者に悪用される危険があります。.gitignoreに必ず.envを追加しましょう。

.envファイルの作成

プロジェクトフォルダ直下に.envファイルを作成し、APIキーを保存します。

.env 
# OpenAI APIキー
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
.gitignore（重要） 
# 仮想環境
venv/

# 環境変数ファイル（絶対にアップロード禁止）
.env

# Python キャッシュ
__pycache__/
*.pyc

2.4 はじめてのLangChain実行

環境構築が完了したら、実際にLangChainを使ってLLMを呼び出してみましょう。

最初のLLM呼び出し

hello_langchain.py 
from dotenv import load_dotenv  # 環境変数読み込み
from langchain_openai import ChatOpenAI  # OpenAI Chat Models

# .envファイルから環境変数を読み込む
load_dotenv()

# ChatOpenAIモデルの初期化
llm = ChatOpenAI(
    model="gpt-4o-mini",  # 使用するモデル
    temperature=0  # 出力のランダム性（0=決定的、1=創造的）
)

# LLMに質問を送信
response = llm.invoke("LangChainとは何ですか？簡潔に説明してください。")

# 結果を表示
print(response.content)
実行コマンド 
python hello_langchain.py

コードの解説

load_dotenv()

.envファイルからOPENAI_API_KEY環境変数を読み込みます。

ChatOpenAI

OpenAIのチャットモデル（GPT-4など）を使うためのクラスです。

temperature

0に近いほど決定的な出力、1に近いほど創造的な出力になります。

invoke()

LLMに質問を送信して、回答を取得するメソッドです。

2.5 動作確認とトラブルシューティング

動作確認チェックリスト

  • 仮想環境が有効化されている（プロンプトに(venv)表示）
  • pip listでlangchainとlangchain-openaiが表示される
  • .envファイルにOPENAI_API_KEYが正しく設定されている
  • スクリプト実行時にLLMからの応答が表示される

よくあるエラーと対処法

❌ ModuleNotFoundError: No module named 'langchain'

原因: 仮想環境が有効化されていない、またはlangchainがインストールされていない
対処: source venv/bin/activateで有効化後、pip install langchain langchain-openaiを実行

❌ AuthenticationError: No API key provided

原因: .envファイルが読み込まれていない、または環境変数名が間違っている
対処: .envファイルの存在確認、OPENAI_API_KEY=のスペルチェック、load_dotenv()呼び出し確認

❌ RateLimitError: Rate limit exceeded

原因: OpenAI APIの利用制限を超過した
対処: OpenAI Platformで利用状況を確認、無料枠の場合は有料プランへの移行を検討

❌ InvalidRequestError: The model does not exist

原因: 指定したモデル名が存在しない、またはアカウントでアクセスできない
対処: model="gpt-4o-mini"など、利用可能なモデル名に変更

実習: 様々な質問を試してみよう

以下の質問をLLMに投げて、異なるtemperature値（0、0.5、1）で結果がどう変わるか観察してください。

  • 「Pythonとは何ですか？」（factual question）
  • 「未来のAI技術について想像してください」（creative question）
  • 「2 + 2は何ですか？」（simple calculation）

この章のまとめ

  • Pythonの仮想環境を使うことで、プロジェクトごとにパッケージを分離管理できる
  • LangChainはpip install langchain langchain-openaiでインストール可能
  • OpenAI APIキーは.envファイルで管理し、絶対にコードに直接書かない
  • ChatOpenAIクラスとinvoke()メソッドで簡単にLLMを呼び出せる
  • temperatureパラメータで出力の決定性・創造性を調整できる

理解度確認クイズ

  1. Q1. Pythonで仮想環境を使うメリットは何ですか？

    回答を表示

    仮想環境を使うと、①プロジェクトごとに異なるバージョンのパッケージを使い分けられる、②システム全体のPython環境を汚さない、③依存関係をrequirements.txtで管理しチーム共有が容易、④本番環境への展開時に必要なパッケージのみをインストールできる、などのメリットがあります。

  2. Q2. .envファイルにAPIキーを保存する理由は何ですか？

    回答を表示

    APIキーをコードに直接書くと、GitHubなどのリポジトリにアップロードした際に第三者に公開され、不正利用されるリスクがあります。.envファイルは.gitignoreに追加することでバージョン管理から除外でき、APIキーを安全に管理できます。また、環境ごと（開発・本番）に異なるキーを使い分けやすくなります。

  3. Q3. temperatureパラメータを0に設定した場合と1に設定した場合、出力はどう変わりますか？

    回答を表示

    temperature=0の場合、LLMは最も確率の高いトークンを選択するため、出力が決定的で一貫性があります（同じ質問には毎回ほぼ同じ答え）。temperature=1の場合、より多様なトークンが選択されるため、創造的で多様な出力が得られますが、一貫性は低下します。事実確認にはtemperature=0、創作にはtemperature=0.7～1が適しています。
