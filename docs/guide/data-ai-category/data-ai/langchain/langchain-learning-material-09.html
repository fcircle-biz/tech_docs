Text Splitters - ドキュメントの分割技術

長いドキュメントをLLMが処理しやすいサイズに分割する技術を学びます。文字数、トークン数、セマンティックな分割方法を理解します。

この章で学ぶこと

  • なぜドキュメント分割が必要なのか、その背景と目的を理解する
  • 文字数ベースとトークン数ベースの分割方法とその使い分けを学ぶ
  • RecursiveCharacterTextSplitterの仕組みと設定方法を習得する
  • 用途別のText Splitter（Code、Markdown、HTML等）の活用方法を理解する
  • chunk_size、chunk_overlap、separatorsなど、重要なパラメータの調整方法を学ぶ

9.1 なぜドキュメント分割が必要なのか

Document Loadersで読み込んだドキュメントは、そのままではLLMに渡せない場合が多くあります。その理由と、Text Splittersがどのように問題を解決するのかを理解しましょう。

主な課題

トークン数の制限

LLMには入力できる最大トークン数（文字数の目安）が決まっています。例：GPT-4は128Kトークン（約10万文字）。長文書はそのまま入力できません。

コストと処理時間

入力トークン数が増えるとAPI利用料金が高くなり、応答速度も遅くなります。必要な部分だけを効率的に処理することが重要です。

精度の低下

LLMに大量のテキストを与えると、重要な情報を見落としたり、関連性の低い情報に引きずられる可能性があります。

検索効率の問題

RAGシステムでは、小さな単位（チャンク）でベクトル化・検索することで、ユーザーの質問に最も関連する部分を正確に取得できます。

チャンク（Chunk）とは

ドキュメントを分割した各部分を「チャンク」と呼びます。適切なサイズのチャンクに分割することで、LLMが効率的に処理でき、検索精度も向上します。

9.2 基本的なText Splitters

LangChainは複数のText Splitterを提供しています。まずは最も基本的な2つを理解しましょう。

CharacterTextSplitter - 文字数ベースの分割

指定した文字数で単純に分割します。最もシンプルですが、文の途中で切れる可能性があります。

Python 
# CharacterTextSplitterの基本的な使い方
from langchain.text_splitter import CharacterTextSplitter

# テキストを準備
text = """LangChainは、大規模言語モデルを活用したアプリケーション開発を支援するフレームワークです。
プロンプト管理、データ処理、処理の連結など、開発における課題を統一的に解決します。
Models、Prompts、Chains、Memory、Agents、Indexesの6つの主要モジュールで構成されています。"""

# Splitterを初期化
splitter = CharacterTextSplitter(
    separator="\n",        # 分割の区切り文字
    chunk_size=100,        # チャンクの最大文字数
    chunk_overlap=20,      # チャンク間の重複文字数
    length_function=len,   # 長さを測る関数
)

# 分割を実行
chunks = splitter.split_text(text)

# 結果を確認
for i, chunk in enumerate(chunks):
    print(f"チャンク{i+1}: {chunk}")
    print(f"  文字数: {len(chunk)}\n")

TokenTextSplitter - トークン数ベースの分割

LLMが実際に処理する「トークン」単位で分割します。より正確なコスト計算と制限管理が可能です。

Python 
# TokenTextSplitterの使い方
from langchain.text_splitter import TokenTextSplitter

text = """長いドキュメントの例..."""

# Splitterを初期化（トークン単位）
splitter = TokenTextSplitter(
    chunk_size=100,       # 最大トークン数
    chunk_overlap=10,     # 重複トークン数
)

chunks = splitter.split_text(text)
print(f"分割数: {len(chunks)}")

トークンとは

LLMが処理する最小単位。日本語では1文字が2-3トークン程度、英単語は1トークン程度です。トークン数がAPI利用料金や処理時間に直接影響します。

9.3 RecursiveCharacterTextSplitter - 推奨される分割方法

RecursiveCharacterTextSplitterは、最も汎用的で推奨されるText Splitterです。複数の区切り文字を優先度順に試し、意味のある単位で分割します。

動作の仕組み

デフォルトでは、以下の順序で区切り文字を試します：

  1. "\n\n"（段落区切り） - 最優先
  2. "\n"（改行） - 段落区切りで失敗したら試す
  3. " "（スペース） - 改行で失敗したら試す
  4. ""（文字単位） - 最後の手段
Python 
# RecursiveCharacterTextSplitterの使い方
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 長いテキストを準備
text = """第1章：LangChainとは

LangChainは、大規模言語モデルを活用したアプリケーション開発を支援するフレームワークです。

第2章：環境構築

Pythonとpipを使ってインストールします。
APIキーの設定も必要です。

第3章：プロンプトテンプレート

PromptTemplateを使って再利用可能なプロンプトを作成できます。"""

# Splitterを初期化
splitter = RecursiveCharacterTextSplitter(
    chunk_size=100,          # チャンクの最大文字数
    chunk_overlap=20,        # 重複文字数
    separators=["\n\n", "\n", " ", ""],  # 区切り文字の優先順位
    length_function=len,
)

# 分割実行
chunks = splitter.split_text(text)

for i, chunk in enumerate(chunks):
    print(f"--- チャンク {i+1} ({len(chunk)}文字) ---")
    print(chunk)
    print()

9.4 重要なパラメータの調整

Text Splitterの性能は、パラメータの設定によって大きく変わります。主要なパラメータを理解しましょう。

chunk_size

各チャンクの最大サイズ（文字数またはトークン数）

推奨値:
  • 一般的なテキスト: 500-1000文字
  • コード: 200-500文字
  • 短い質問応答: 200-300文字

chunk_overlap

隣接するチャンク間で重複させる文字数

推奨値:
  • chunk_sizeの10-20%程度
  • 例: chunk_size=500なら50-100
  • 文脈の連続性を保つために重要

separators

分割に使用する区切り文字のリスト（優先度順）

デフォルト: ["\n\n", "\n", " ", ""]
カスタマイズ例: ["。", "、", " "]（日本語）

length_function

テキストの長さを計測する関数

デフォルト: len（文字数）
トークン数で計測: tiktoken等のライブラリを使用

chunk_overlapの重要性

重複がないと、文脈の境界で情報が失われる可能性があります。例えば、「東京は日本の首都です。人口は1400万人です。」という文が「東京は日本の首都です。」と「人口は1400万人です。」に分かれると、後者だけでは何の人口か分かりません。適切な重複により、文脈を保持できます。

9.5 用途別のText Splitters

LangChainは、特定の形式に特化したText Splittersも提供しています。

PythonCodeTextSplitter - コード専用

Pythonの文法を理解し、関数やクラスの単位で分割します。

Python 
# PythonCodeTextSplitterの使い方
from langchain.text_splitter import PythonCodeTextSplitter

python_code = """
def hello():
    print("Hello")

class MyClass:
    def __init__(self):
        self.value = 0
"""

splitter = PythonCodeTextSplitter(
    chunk_size=100,
    chunk_overlap=0,
)

chunks = splitter.split_text(python_code)
print(f"分割数: {len(chunks)}")

MarkdownTextSplitter - Markdown専用

見出しやリストなど、Markdownの構造を保持しながら分割します。

Python 
# MarkdownTextSplitterの使い方
from langchain.text_splitter import MarkdownTextSplitter

markdown_text = """
# タイトル

## セクション1
内容1

## セクション2
内容2
"""

splitter = MarkdownTextSplitter(
    chunk_size=100,
    chunk_overlap=0,
)

chunks = splitter.split_text(markdown_text)

実践演習：最適な分割パラメータを見つける

様々なパラメータ設定を試して、違いを確認しましょう。

Python 
# パラメータの比較実験
from langchain.text_splitter import RecursiveCharacterTextSplitter

text = """LangChainは大規模言語モデルを活用したアプリケーション開発を支援するフレームワークです。
プロンプト管理、データ処理、処理の連結など、開発における課題を統一的に解決します。
Models、Prompts、Chains、Memory、Agents、Indexesの6つの主要モジュールで構成されています。
チャットボット、RAGシステム、コード生成、文書要約など、様々な用途で活用されています。"""

# パターン1: 小さいチャンク、重複なし
splitter1 = RecursiveCharacterTextSplitter(chunk_size=50, chunk_overlap=0)
chunks1 = splitter1.split_text(text)
print(f"パターン1: {len(chunks1)}個のチャンク")

# パターン2: 中サイズチャンク、適度な重複
splitter2 = RecursiveCharacterTextSplitter(chunk_size=100, chunk_overlap=20)
chunks2 = splitter2.split_text(text)
print(f"パターン2: {len(chunks2)}個のチャンク")

# パターン3: 大きいチャンク、多めの重複
splitter3 = RecursiveCharacterTextSplitter(chunk_size=150, chunk_overlap=50)
chunks3 = splitter3.split_text(text)
print(f"パターン3: {len(chunks3)}個のチャンク")

# 各チャンクの内容を比較
for i, chunk in enumerate(chunks2):
    print(f"\nチャンク{i+1}:\n{chunk}")

演習のポイント

  • • chunk_sizeを変えると分割数がどう変わるか観察する
  • • chunk_overlapがある場合とない場合で、チャンク間の連続性を確認する
  • • 自分のドキュメントに適したパラメータを見つける

この章のまとめ

  • LLMのトークン制限、コスト、精度の観点から、ドキュメント分割は必須の技術
  • RecursiveCharacterTextSplitterは、段落→改行→スペース→文字の順で分割を試みる推奨方式
  • chunk_size（チャンクサイズ）とchunk_overlap（重複）の設定が分割品質を左右する
  • コード、Markdown、HTML等、形式別の専用Splittersを活用すると構造を保持できる
  • 適切な分割により、RAGシステムの検索精度とLLMの応答品質が大きく向上する

理解度確認クイズ

  1. Q1. なぜドキュメントを分割する必要があるのですか？主な理由を3つ挙げてください。

    回答を表示

    ①LLMの入力トークン数制限を超えないため、②API利用料金と処理時間を削減するため、③検索精度を向上させ、必要な情報だけを効率的に取得するため、の3つが主な理由です。大きなドキュメントをそのまま入力すると、これらの問題が発生します。

  2. Q2. RecursiveCharacterTextSplitterが「Recursive（再帰的）」と呼ばれる理由は何ですか？

    回答を表示

    複数の区切り文字を優先度順に試し、各レベルで分割できるかを再帰的に判断するためです。まず段落区切り（\n\n）を試し、それで分割できなければ改行（\n）、スペース（ ）、最後に文字単位、と段階的に試していきます。この再帰的な処理により、意味のある単位で分割できます。

  3. Q3. chunk_overlapを設定する理由と、推奨される値の目安を説明してください。

    回答を表示

    chunk_overlapは、隣接するチャンク間で重複させる文字数です。設定する理由は、分割境界で文脈が失われるのを防ぐためです。重複がないと、文の途中で切れて意味が通じなくなる可能性があります。推奨値はchunk_sizeの10-20%程度で、例えばchunk_size=500なら50-100文字の重複が適切です。
