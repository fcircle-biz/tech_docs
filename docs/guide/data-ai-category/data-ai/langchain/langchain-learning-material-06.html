Chains入門 - 処理の連結と自動化

複数の処理ステップを連結するChainの基本概念を理解し、LLMChain、SequentialChainなどの基本的なチェーンの構築方法を学びます。

この章で学ぶこと

  • Chainの基本概念と必要性を理解する
  • LLMChainを使った基本的な処理フローを実装する
  • SequentialChainによる複数ステップの連結方法を習得する
  • RouterChainを使った条件分岐処理を実装する

6.1 Chainとは - 処理の連結という思想

Chain（チェーン）は、LangChainの名前の由来にもなっている中核的な概念です。複数の処理ステップを連結（チェーン化）して、複雑なワークフローを構築するための仕組みです。

LLMを使ったアプリケーションでは、「プロンプトを生成→LLMに送信→結果を解析→次の処理に渡す」といった複数のステップが必要になります。Chainはこれらのステップを統一的に管理し、メンテナンス性の高いコードを実現します。

Chainの基本思想

Chainは「入力を受け取り、処理を行い、出力を返す」というシンプルなインターフェースを持ちます。この統一性により、複数のChainを組み合わせて、さらに複雑なChainを構築できます。

プロンプト生成] Chain2[Chain 2
LLM呼び出し] Chain3[Chain 3
出力解析] Output[最終結果] Input --> Chain1 Chain1 --> Chain2 Chain2 --> Chain3 Chain3 --> Output

6.2 LLMChain - 最もシンプルなChain

LLMChainは、PromptTemplateとLLMを組み合わせた最もシンプルなChainです。プロンプト生成→LLM呼び出し→結果取得の基本的な流れを一つのChainにまとめます。

LLMChainの基本構造

変数を含むテンプレート] LLM[LLM
言語モデル] PT --> LLM end Input[入力変数
topic: Python] Output[出力結果
生成されたテキスト] Input --> LLMChain LLMChain --> Output

図6-2: LLMChainの構造

LLMChainの実装例

basic_llm_chain.py 
# LLMChainの基本的な使い方
from langchain.prompts import PromptTemplate
from langchain.chains import LLMChain
from langchain_openai import ChatOpenAI

# PromptTemplateの定義
prompt = PromptTemplate(
    input_variables=["topic"],
    template="{topic}について初心者向けに3行で説明してください。"
)

# LLMの準備
llm = ChatOpenAI(temperature=0.7, model="gpt-4")

# LLMChainの作成
chain = LLMChain(llm=llm, prompt=prompt)

# Chainの実行
result = chain.run(topic="LangChainのChain")
print(result)

LLMChainの利点

  • • プロンプトとLLMを一体化し、再利用性を向上
  • • 入力変数の管理が簡潔になる
  • • 他のChainと組み合わせやすい標準的なインターフェース

6.3 SequentialChain - 複数ステップの連結

SequentialChainは、複数のChainを順番に実行するChainです。あるChainの出力を次のChainの入力として渡し、段階的な処理フローを実現します。

SimpleSequentialChain vs SequentialChain

SimpleSequentialChain

各Chainが単一の文字列を入出力する場合に使用

シンプルだが、各ステップで1つの値しか扱えない

SequentialChain

各Chainが複数の変数を入出力する場合に使用

複雑だが、複数の値を受け渡しできる

テーマ: AI技術] subgraph Chain1[Chain 1: タイトル生成] P1[PromptTemplate] L1[LLM] P1 --> L1 end subgraph Chain2[Chain 2: 本文生成] P2[PromptTemplate] L2[LLM] P2 --> L2 end subgraph Chain3[Chain 3: 要約生成] P3[PromptTemplate] L3[LLM] P3 --> L3 end End[最終出力
タイトル・本文・要約] Start --> Chain1 Chain1 --> |title| Chain2 Chain2 --> |content| Chain3 Chain3 --> End

図6-3: SequentialChainの処理フロー

SequentialChainの実装例

sequential_chain_example.py 
# SequentialChainで段階的な処理を実現
from langchain.prompts import PromptTemplate
from langchain.chains import LLMChain, SequentialChain
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(temperature=0.7, model="gpt-4")

# Chain 1: タイトル生成
title_template = PromptTemplate(
    input_variables=["topic"],
    template="{topic}に関する記事のタイトルを1つ提案してください。"
)
title_chain = LLMChain(llm=llm, prompt=title_template, output_key="title")

# Chain 2: 本文生成
content_template = PromptTemplate(
    input_variables=["title"],
    template='タイトル「{title}」の記事本文を200文字程度で書いてください。'
)
content_chain = LLMChain(llm=llm, prompt=content_template, output_key="content")

# Chain 3: 要約生成
summary_template = PromptTemplate(
    input_variables=["content"],
    template="以下の文章を50文字以内で要約してください。\n\n{content}"
)
summary_chain = LLMChain(llm=llm, prompt=summary_template, output_key="summary")

# SequentialChainで連結
overall_chain = SequentialChain(
    chains=[title_chain, content_chain, summary_chain],
    input_variables=["topic"],
    output_variables=["title", "content", "summary"],
    verbose=True
)

# 実行
result = overall_chain({"topic": "生成AIの未来"})
print("タイトル:", result["title"])
print("本文:", result["content"])
print("要約:", result["summary"])

ハンズオン: ブログ記事生成システムの構築

SequentialChainを使って、キーワードから完全なブログ記事を生成するシステムを作成しましょう。

処理フロー

  1. 1. Chain 1: キーワードから記事のアウトライン（見出し）を生成
  2. 2. Chain 2: アウトラインから本文を生成
  3. 3. Chain 3: 本文からSEO用のメタディスクリプションを生成
blog_generator.py 
# ブログ記事生成システム
from langchain.prompts import PromptTemplate
from langchain.chains import LLMChain, SequentialChain
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(temperature=0.7, model="gpt-4")

# Chain 1: アウトライン生成
outline_prompt = PromptTemplate(
    input_variables=["keyword"],
    template="""キーワード「{keyword}」に関するブログ記事のアウトラインを作成してください。
以下の形式で出力してください：
1. 導入
2. [小見出し1]
3. [小見出し2]
4. まとめ"""
)
outline_chain = LLMChain(llm=llm, prompt=outline_prompt, output_key="outline")

# Chain 2: 本文生成
content_prompt = PromptTemplate(
    input_variables=["outline"],
    template="""以下のアウトラインに基づいて、各セクション100文字程度の本文を書いてください。
アウトライン:
{outline}"""
)
content_chain = LLMChain(llm=llm, prompt=content_prompt, output_key="content")

# Chain 3: メタディスクリプション生成
meta_prompt = PromptTemplate(
    input_variables=["content"],
    template="以下の記事内容から、SEO用のメタディスクリプション（120文字以内）を生成してください。\n\n{content}"
)
meta_chain = LLMChain(llm=llm, prompt=meta_prompt, output_key="meta_description")

# 全てを連結
blog_chain = SequentialChain(
    chains=[outline_chain, content_chain, meta_chain],
    input_variables=["keyword"],
    output_variables=["outline", "content", "meta_description"],
    verbose=True
)

# 実行
result = blog_chain({"keyword": "LangChainのChain機能"})
print("=== アウトライン ===")
print(result["outline"])
print("\n=== 本文 ===")
print(result["content"])
print("\n=== メタディスクリプション ===")
print(result["meta_description"])

6.5 Chainのベストプラクティス

適切な粒度で分割

各Chainは単一の責任を持つように設計。再利用性とテストのしやすさが向上します。

出力キーの明確化

output_keyを明示的に指定し、変数名を分かりやすくすることで、デバッグが容易になります。

verboseモードの活用

開発時はverbose=Trueで各Chainの入出力を確認。処理フローの把握に役立ちます。

エラーハンドリング

各Chainで例外処理を実装し、一部のChainが失敗しても全体が停止しないように設計します。

よくあるトラブルと解決法

変数名の不一致エラー

症状: 前のChainのoutput_keyと次のChainのinput_variablesが一致せず、エラーが発生

解決策: 各Chainのoutput_keyとinput_variablesを確認し、名前を統一します。verbose=Trueで実行して変数の流れを追跡しましょう。

LLMの応答が期待と異なる

症状: Chainは動作するが、生成される内容が期待と大きく異なる

解決策: PromptTemplateの指示を明確にし、出力形式の例を示します。temperatureを下げて出力の安定性を向上させることも検討しましょう。

処理時間が長すぎる

症状: SequentialChainで複数のLLM呼び出しが連続し、全体の処理時間が長い

解決策: 並列実行できる部分は別のアプローチで実装するか、より高速なモデル（gpt-4o-miniなど）を一部のChainで使用します。

この章のまとめ

  • Chainは複数の処理ステップを連結し、複雑なワークフローを構築する仕組み
  • LLMChainは、PromptTemplateとLLMを組み合わせた最もシンプルなChain
  • SequentialChainは、複数のChainを順番に実行し、段階的な処理を実現
  • RouterChainは、入力内容に応じて異なるChainに処理を振り分ける
  • 適切な粒度での分割、明確な変数名、verboseモードの活用がベストプラクティス

理解度確認クイズ

  1. Q1. LLMChainの構成要素は何ですか？2つ答えてください。

    回答を表示

    LLMChainは、①PromptTemplate（プロンプトテンプレート）と②LLM（言語モデル）の2つの要素で構成されます。PromptTemplateで変数を含むプロンプトを定義し、LLMがそれを実行して結果を生成します。

  2. Q2. SequentialChainとSimpleSequentialChainの違いは何ですか？

    回答を表示

    SimpleSequentialChainは各Chainが単一の文字列を入出力する場合に使用し、シンプルですが1つの値しか扱えません。一方、SequentialChainは各Chainが複数の変数を入出力でき、より複雑なデータの受け渡しが可能です。

  3. Q3. RouterChainはどのような場面で有用ですか？具体例を1つ挙げてください。

    回答を表示

    RouterChainは、入力内容に応じて異なる処理フローを選択する場合に有用です。例えば、カスタマーサポートシステムで、ユーザーの質問が「技術的な質問」「価格に関する質問」「サポート依頼」のどれかを判定し、各カテゴリに特化したChainに振り分けることで、より適切な回答を生成できます。
