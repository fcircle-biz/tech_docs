LLMとChat Models - モデルの使い分け

LangChainで利用できる様々なLLM（OpenAI、Anthropic、ローカルモデル等）の種類と特徴を理解し、用途に応じた使い分け方を学びます。

この章で学ぶこと

  • LLMとChat Modelsの違いを理解する
  • 主要なLLMプロバイダー（OpenAI、Anthropic、Google等）の特徴を把握する
  • 用途に応じたモデル選択の基準を学ぶ
  • ローカルモデルとクラウドモデルの使い分けを理解する

4.1 LLMとChat Modelsの違い

LangChainでは、言語モデルを大きく2つのカテゴリーに分類しています。それぞれ異なるインターフェースと用途を持つため、使い分けが重要です。

LLM（テキスト補完型）

入力されたテキストの続きを生成するモデル。プロンプト全体を1つの文字列として処理します。

入力例:
"次の質問に答えてください：東京の首都は？"
出力:
"東京の首都は東京です。"

Chat Models（会話型）

メッセージのリストを入力として、会話形式で応答を生成するモデル。役割（system/user/assistant）を指定可能。

入力例:
[{role: "user", content: "東京の首都は？"}]
出力:
"東京は日本の首都です。"

推奨される使い分け

Chat Models（ChatOpenAI、ChatAnthropic等）が推奨されています。会話履歴の管理がしやすく、システムメッセージで振る舞いを制御できるため、より柔軟なアプリケーション開発が可能です。

4.2 主要なLLMプロバイダー

LangChainは、複数のLLMプロバイダーに対応しており、統一的なインターフェースで切り替えることができます。主要なプロバイダーの特徴を見ていきましょう。

1. OpenAI（ChatOpenAI）

主要モデル

GPT-4o, GPT-4 Turbo, GPT-3.5 Turbo

特徴

高精度、Function Calling対応、豊富なエコシステム

用途

汎用的なタスク、複雑な推論、コード生成

Python 
# OpenAIモデルの使用例
from langchain_openai import ChatOpenAI

# モデルの初期化
llm = ChatOpenAI(
    model="gpt-4o",  # 使用するモデル
    temperature=0.7,  # 生成の多様性（0～1）
    max_tokens=500    # 最大生成トークン数
)

# メッセージ生成
response = llm.invoke("AIの未来について説明してください")
print(response.content)

2. Anthropic（ChatAnthropic）

主要モデル

Claude 3.5 Sonnet, Claude 3 Opus, Claude 3 Haiku

特徴

長文処理に強い、安全性重視、高度な推論

用途

長文分析、複雑な指示の理解、安全性が重要なタスク

Python 
# Anthropicモデルの使用例
from langchain_anthropic import ChatAnthropic

# モデルの初期化
llm = ChatAnthropic(
    model="claude-3-5-sonnet-20241022",
    temperature=0.7,
    max_tokens=1000
)

# メッセージ生成
response = llm.invoke("長文の契約書を要約してください...")
print(response.content)

3. Google（ChatGoogleGenerativeAI）

主要モデル

Gemini 1.5 Pro, Gemini 1.5 Flash

特徴

マルチモーダル対応、大規模コンテキスト、高速

用途

画像・動画分析、大量データ処理、リアルタイム応答

Python 
# Googleモデルの使用例
from langchain_google_genai import ChatGoogleGenerativeAI

# モデルの初期化
llm = ChatGoogleGenerativeAI(
    model="gemini-1.5-pro",
    temperature=0.5
)

# メッセージ生成
response = llm.invoke("画像認識の技術について解説してください")
print(response.content)

4. ローカルモデル（Ollama等）

主要モデル

Llama 3, Mistral, Phi-3等

特徴

オフライン動作、データ外部流出なし、コスト削減

用途

機密情報処理、開発環境、実験用途

Python 
# Ollamaでローカルモデルを使用
from langchain_ollama import ChatOllama

# モデルの初期化（事前にOllamaでモデルをpull必要）
llm = ChatOllama(
    model="llama3.1",
    temperature=0.7
)

# メッセージ生成
response = llm.invoke("Pythonの基礎を教えてください")
print(response.content)

4.4 重要なモデルパラメータ

モデルの挙動を制御するパラメータを理解することで、より適切な出力を得られます。

temperature（0～2）

出力の多様性を制御。低いほど確定的、高いほど創造的。

0.0～0.3: 分析・分類など正確性重視
0.7～1.0: 汎用的なタスク（デフォルト）
1.5～2.0: 創作・アイデア生成

max_tokens

生成する最大トークン数。コスト管理と出力長の制御。

100～500: 短い回答（FAQ等）
500～1000: 標準的な説明
2000+: 長文生成（記事・レポート）

top_p（0～1）

ニュークリアスサンプリング。確率の高いトークンから選択。

0.9～1.0: バランスの良い出力
0.5～0.7: より確実性の高い出力
※ temperatureとの併用は推奨されない

streaming

生成中のトークンを逐次受信。UX向上に有効。

True: チャットUIなどリアルタイム表示
False: バッチ処理など完了待ち
Python 
# パラメータの設定例
from langchain_openai import ChatOpenAI

# 正確性重視（分類・分析タスク）
llm_precise = ChatOpenAI(
    model="gpt-4o",
    temperature=0.0,  # 確定的な出力
    max_tokens=500
)

# 創造性重視（アイデア生成）
llm_creative = ChatOpenAI(
    model="gpt-4o",
    temperature=1.5,  # 多様な出力
    max_tokens=1000
)

# ストリーミング有効化
llm_stream = ChatOpenAI(
    model="gpt-4o",
    streaming=True
)

実習: 複数モデルで同じタスクを実行

同じプロンプトを異なるモデルに送信し、出力の違いを比較してみましょう。

Python 
# 複数モデルの比較実験
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

# タスク: 技術記事のタイトル生成
prompt = "Pythonで機械学習を始めるための入門記事のタイトルを5つ提案してください"

# GPT-3.5で実行
gpt35 = ChatOpenAI(model="gpt-3.5-turbo", temperature=0.8)
response_gpt35 = gpt35.invoke(prompt)
print("=== GPT-3.5 Turbo ===")
print(response_gpt35.content)

# GPT-4で実行
gpt4 = ChatOpenAI(model="gpt-4o", temperature=0.8)
response_gpt4 = gpt4.invoke(prompt)
print("\n=== GPT-4o ===")
print(response_gpt4.content)

# Claude 3.5で実行
claude = ChatAnthropic(model="claude-3-5-sonnet-20241022", temperature=0.8)
response_claude = claude.invoke(prompt)
print("\n=== Claude 3.5 Sonnet ===")
print(response_claude.content)

この章のまとめ

  • LLMはテキスト補完型、Chat Modelsは会話型で、Chat Modelsが推奨される
  • 主要プロバイダー: OpenAI（汎用）、Anthropic（長文・安全性）、Google（マルチモーダル）、ローカルモデル（機密性）
  • 用途に応じてコスト・精度・速度・機密性を考慮してモデルを選択する
  • temperature、max_tokens、top_p等のパラメータで出力を細かく制御可能
  • 同じプロンプトでも、モデルによって出力スタイルと品質が異なる

理解度確認クイズ

  1. Q1. LLMとChat Modelsの違いは何ですか？なぜChat Modelsが推奨されますか？

    回答を表示

    LLMはテキストの続きを生成する補完型、Chat Modelsは役割（system/user/assistant）を指定できる会話型です。Chat Modelsが推奨される理由は、①会話履歴の管理がしやすい、②システムメッセージで振る舞いを制御できる、③多くの最新モデルがChat形式を前提としているためです。

  2. Q2. 機密性の高い社内文書を処理する場合、どのモデルを選ぶべきですか？その理由は？

    回答を表示

    ローカルモデル（Llama 3、Mistral等）をOllama等で動作させるべきです。理由は、データが外部のクラウドサービスに送信されず、オンプレミスまたはローカル環境で完結するため、機密情報の漏洩リスクを最小化できるからです。

  3. Q3. temperatureパラメータとは何ですか？どのように使い分けますか？

    回答を表示

    temperatureは出力の多様性を制御するパラメータです（0～2）。0に近いほど確定的で一貫性のある出力、高いほど創造的で多様な出力になります。使い分け: 0.0～0.3（分類・分析など正確性重視）、0.7～1.0（汎用タスク）、1.5～2.0（創作・アイデア生成）。
