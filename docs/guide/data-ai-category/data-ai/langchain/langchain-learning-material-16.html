第16章

Tools - Agentの能力を拡張する

Web検索、計算機、カスタムツールなど、Agentが使用できるToolの作成方法を学びます。LLMの能力を外部機能で拡張する技術を習得します。

この章で学ぶこと

  • LangChainのToolの基本概念と役割を理解する
  • 組み込みTools（Web検索、計算機、Python REPL等）の使い方を習得する
  • カスタムToolの作成方法（@toolデコレーター、StructuredTool）を学ぶ
  • ToolとAgentを組み合わせた実践的なアプリケーションを構築する

16.1 LangChainにおけるToolsとは

Toolは、LangChainのAgentが外部機能を実行するためのインターフェースです。LLMは言語処理に優れていますが、計算、Web検索、データベースアクセスなどの処理は直接実行できません。Toolを使うことで、LLMに「手足」を与えることができます。

Toolの3要素

  • name: Toolの名前（AgentがToolを選択する際の識別子）
  • description: Toolの説明（LLMが適切なToolを選ぶための情報）
  • func: 実際に実行する関数（Pythonの関数として定義）

16.2 組み込みToolsの活用

LangChainには、すぐに使える便利な組み込みToolsが多数用意されています。代表的なものを見ていきましょう。

主要な組み込みTools

SerpAPIWrapper

Google検索結果を取得。最新情報や一般知識を補完（APIキー必要）

LLMMathChain

数式を評価して計算結果を返す。複雑な数値計算に対応

PythonREPLTool

Pythonコードを実行。データ処理やグラフ作成など

WikipediaQueryRun

Wikipedia情報を検索。百科事典的な知識を提供

実習: Web検索Toolを使ったAgent

SerpAPIを使って、最新情報を検索できるAgentを作成します。

search_agent.py 
# SerpAPI Toolを使ったAgent（SerpAPI APIキーが必要）
from langchain.agents import initialize_agent, AgentType
from langchain.tools import Tool
from langchain_community.utilities import SerpAPIWrapper
from langchain_openai import ChatOpenAI

# Web検索Toolの初期化
search = SerpAPIWrapper()  # 環境変数 SERPAPI_API_KEY を設定
search_tool = Tool(
    name="Google検索",
    func=search.run,
    description="最新の情報や一般的な知識を検索する際に使用します"
)

# LLMの初期化
llm = ChatOpenAI(temperature=0, model="gpt-3.5-turbo")

# Agentの作成
agent = initialize_agent(
    tools=[search_tool],
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION
)

# 実行
result = agent.run("2025年のノーベル物理学賞は誰が受賞しましたか？")
print(result)

16.3 カスタムToolの作成

独自の機能をToolとして実装することで、Agentの能力を自由に拡張できます。LangChainでは、@toolデコレーターを使った簡単な方法と、StructuredToolを使った高度な方法の2つがあります。

方法1: @toolデコレーター（推奨）

custom_tool_decorator.py 
from langchain.agents import tool

# @toolデコレーターで簡単にToolを作成
@tool
def get_word_length(word: str) -> int:
    """文字列の長さを返すTool"""
    return len(word)

@tool
def multiply_numbers(a: float, b: float) -> float:
    """2つの数を掛け算するTool"""
    return a * b

# Toolの情報確認
print(f"名前: {get_word_length.name}")
print(f"説明: {get_word_length.description}")

# Toolを直接実行
print(get_word_length.invoke("LangChain"))  # 9
print(multiply_numbers.invoke({"a": 3.5, "b": 2}))  # 7.0

@toolデコレーターのポイント

  • 関数のdocstringが自動的にdescriptionになる
  • 型ヒントから入力パラメータが自動推論される
  • 最もシンプルで推奨される方法

方法2: StructuredTool（高度な制御）

custom_tool_structured.py 
from langchain.tools import StructuredTool
from pydantic import BaseModel, Field

# 入力スキーマ定義
class CalculatorInput(BaseModel):
    a: float = Field(description="1つ目の数値")
    b: float = Field(description="2つ目の数値")
    operation: str = Field(description="演算子（+, -, *, /）")

# Tool関数定義
def calculator_func(a: float, b: float, operation: str) -> float:
    """四則演算を実行する"""
    if operation == "+":
        return a + b
    elif operation == "-":
        return a - b
    elif operation == "*":
        return a * b
    elif operation == "/":
        return a / b
    else:
        return 0

# StructuredToolの作成
calculator_tool = StructuredTool.from_function(
    func=calculator_func,
    name="計算機",
    description="四則演算（+, -, *, /）を実行します",
    args_schema=CalculatorInput
)

# 実行
result = calculator_tool.invoke({"a": 10, "b": 3, "operation": "*"})
print(result)  # 30.0

16.4 複数Toolsを組み合わせたAgent

複数のToolを組み合わせることで、より高度なタスクを自動化できます。以下の例では、計算機ToolとWeb検索Toolを両方持つAgentを作成します。

実習: 複数Toolsを持つ万能Agent

multi_tools_agent.py 
from langchain.agents import initialize_agent, AgentType, tool
from langchain_openai import ChatOpenAI

# カスタムTool: 文字列処理
@tool
def reverse_string(text: str) -> str:
    """文字列を逆順にするTool"""
    return text[::-1]

# カスタムTool: データベース風の検索
@tool
def search_user_info(user_id: str) -> str:
    """ユーザー情報を検索するTool（サンプルデータ）"""
    users = {
        "001": "山田太郎, 営業部, 東京",
        "002": "佐藤花子, 開発部, 大阪",
        "003": "鈴木一郎, 総務部, 名古屋"
    }
    return users.get(user_id, "該当するユーザーが見つかりません")

# カスタムTool: 簡易計算機
@tool
def calculate(expression: str) -> float:
    """数式を評価して結果を返すTool（例: 2+3*4）"""
    try:
        return eval(expression)  # 本番では安全な評価器を使用
    except:
        return 0

# Agentの作成
llm = ChatOpenAI(temperature=0, model="gpt-3.5-turbo")
tools = [reverse_string, search_user_info, calculate]

agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True  # 実行プロセスを表示
)

# テスト実行
print("=== テスト1: 文字列処理 ===")
result1 = agent.run("「LangChain」を逆順にしてください")
print(result1)

print("\n=== テスト2: ユーザー情報検索 ===")
result2 = agent.run("ユーザーID 002の情報を教えて")
print(result2)

print("\n=== テスト3: 計算 ===")
result3 = agent.run("15の2乗を計算してください")
print(result3)

16.5 Tool設計のベストプラクティス

効果的なTool設計の原則

  • 明確なdescription: LLMがToolを正しく選択できるよう、用途を具体的に記述

    ❌ "データを取得する" → ✅ "ユーザーIDから氏名・部署・勤務地を検索します"

  • 単一責任: 1つのToolは1つの機能に集中（複数機能は複数Toolに分割）
  • エラーハンドリング: 例外が発生しても適切なエラーメッセージを返す
  • 型ヒント: 入力・出力の型を明示してAgentが正しく使えるようにする
  • テスト: Agentに組み込む前に、Tool単体で動作確認する

セキュリティ上の注意

  • PythonREPLTool: 任意のコード実行は危険。信頼できる環境でのみ使用
  • eval()の使用: 外部入力を直接評価しない（サンプルコード以外では避ける）
  • APIキー: 環境変数で管理し、コードにハードコードしない

16.6 実践例: データ分析アシスタント

これまで学んだ知識を統合して、CSVデータを分析するAgentを構築します。

実習: CSVデータ分析Agent

data_analysis_agent.py 
import pandas as pd
from langchain.agents import initialize_agent, AgentType, tool
from langchain_openai import ChatOpenAI

# サンプルCSVデータを作成
df = pd.DataFrame({
    "商品名": ["りんご", "バナナ", "オレンジ", "ぶどう"],
    "価格": [150, 80, 120, 300],
    "在庫数": [50, 100, 30, 20]
})
df.to_csv("products.csv", index=False)

# Tool: CSVデータ読み込み
@tool
def load_csv_data(file_path: str) -> str:
    """CSVファイルを読み込んでデータフレームの内容を返すTool"""
    df = pd.read_csv(file_path)
    return df.to_string()

# Tool: 統計情報取得
@tool
def get_statistics(file_path: str, column: str) -> str:
    """指定したカラムの統計情報（平均、合計等）を返すTool"""
    df = pd.read_csv(file_path)
    if column in df.columns:
        stats = df[column].describe().to_string()
        return f"{column}の統計情報:\n{stats}"
    return f"カラム「{column}」が見つかりません"

# Tool: 最高値・最低値検索
@tool
def find_extremes(file_path: str, column: str) -> str:
    """指定カラムの最大値・最小値を持つ行を返すTool"""
    df = pd.read_csv(file_path)
    if column in df.columns:
        max_row = df.loc[df[column].idxmax()]
        min_row = df.loc[df[column].idxmin()]
        return f"最大値: {max_row.to_dict()}\n最小値: {min_row.to_dict()}"
    return f"カラム「{column}」が見つかりません"

# Agentの作成
llm = ChatOpenAI(temperature=0, model="gpt-3.5-turbo")
tools = [load_csv_data, get_statistics, find_extremes]

agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True
)

# 質問実行
print("=== 質問1: データ概要 ===")
result1 = agent.run("products.csvのデータを見せてください")
print(result1)

print("\n=== 質問2: 統計分析 ===")
result2 = agent.run("価格の平均と合計を教えてください")
print(result2)

print("\n=== 質問3: 最高値・最低値 ===")
result3 = agent.run("最も高い商品と最も安い商品を教えて")
print(result3)

この章のまとめ

  • ToolはAgentが外部機能を実行するためのインターフェース（name, description, func）
  • 組み込みTools（SerpAPI、計算機、PythonREPL等）ですぐに高度な機能が使える
  • @toolデコレーターで簡単にカスタムToolを作成できる
  • 複数Toolsを組み合わせることで、Agentの能力を大幅に拡張できる
  • 明確なdescription、単一責任、エラーハンドリングが効果的なTool設計の鍵

理解度確認クイズ

  1. Q1. LangChainのToolを構成する3つの要素は何ですか？

    回答を表示

    ①name（Toolの名前）、②description（Toolの説明）、③func（実際に実行する関数）の3つです。descriptionはLLMがToolを選択する際の重要な判断材料となるため、用途を具体的に記述することが重要です。

  2. Q2. @toolデコレーターを使ったカスタムToolの作成方法を説明してください。

    回答を表示

    Python関数に@toolデコレーターを付けるだけで作成できます。関数のdocstringが自動的にdescriptionになり、型ヒントから入力パラメータが推論されます。例: @tool def my_tool(text: str) -> str: """説明""" return text.upper()

  3. Q3. 効果的なTool設計のベストプラクティスを3つ挙げてください。

    回答を表示

    ①明確なdescription: LLMが正しくToolを選べるよう用途を具体的に記述、②単一責任の原則: 1つのToolは1つの機能に集中、③エラーハンドリング: 例外が発生しても適切なエラーメッセージを返す、などが挙げられます。その他、型ヒントの明示やTool単体でのテストも重要です。
