Agents入門 - LLMに意思決定させる

LLMが状況に応じてツールを選択・実行するAgentの基本概念を理解します。ReActフレームワークとZero-shot Agentの実装を体験します。

この章で学ぶこと

  • Agentとは何か、ChainやMemoryとの違いを理解する
  • ReActフレームワークの仕組みと動作原理を学ぶ
  • Zero-shot ReAct Agentの実装方法を習得する
  • Agentの実行プロセスとデバッグ方法を理解する

15.1 Agentとは何か

これまで学んできたChainは、事前に定義された処理フローを順番に実行する仕組みでした。一方、Agentは、LLMが状況を判断し、動的に次のアクションを決定する仕組みです。

Agentは、ユーザーの質問を受け取ると、「何を知る必要があるか」「どのツールを使うべきか」「どの順番で実行するか」をLLM自身が考え、自律的にタスクを実行します。

ChainとAgentの違い

Chainは「決められた手順の実行」、Agentは「LLMによる意思決定と実行」です。Chainは確実性が高く、Agentは柔軟性が高いという特性があります。

flowchart LR subgraph Chain["Chain（固定フロー）"] C1[ステップ1] --> C2[ステップ2] --> C3[ステップ3] --> C4[完了] end subgraph Agent["Agent（動的判断）"] A1[質問受付] --> A2{LLMが判断} A2 -->|ツールA必要| A3[ツールA実行] A2 -->|ツールB必要| A4[ツールB実行] A2 -->|十分な情報| A5[回答生成] A3 --> A2 A4 --> A2 end

図15-1: ChainとAgentの実行フローの違い

Agentの主な用途

情報収集タスク

Web検索、API呼び出しなど、必要な情報を複数ソースから収集して統合

計算・分析タスク

計算機ツール、データベースクエリなど、複雑な計算や分析を実行

マルチステップ実行

複数のツールを組み合わせて、手順が不明確なタスクを段階的に解決

自律的な意思決定

状況に応じて最適なアクションを判断し、人間の介入なしで実行

15.2 ReActフレームワーク

LangChainのAgentは、ReAct（Reasoning and Acting）という思考フレームワークに基づいています。ReActは、LLMが「推論（Reasoning）」と「行動（Acting）」を繰り返すことで、複雑な問題を解決する手法です。

ReActの実行サイクル

ReActの3ステップ

1 Thought（思考）

LLMが現在の状況を分析し、「次に何をすべきか」を考えます。

2 Action（行動）

使用するツールとその入力を決定します（例: 検索ツールで「東京 天気」を検索）。

3 Observation（観察）

ツールの実行結果を受け取り、次の思考に活かします。回答可能であれば終了、不足があれば再度Thoughtへ。

flowchart TD Start[質問受付] --> Thought{Thought
思考} Thought -->|情報不足| Action[Action
ツール実行決定] Action --> Execute[ツール実行] Execute --> Observation[Observation
結果を観察] Observation --> Thought Thought -->|回答可能| Answer[最終回答生成]

図15-2: ReActフレームワークの実行サイクル

ReActの実行例

質問: 「東京の明日の天気と、傘が必要かどうか教えて」

Thought: 天気情報が必要なので、検索ツールを使おう

Action: 検索ツール(query="東京 明日 天気")

Observation: 「明日は雨、降水確率80%」という結果を取得

Thought: 降水確率80%なら傘が必要だと判断できる

Answer: 「明日は雨です。傘を持っていくことをお勧めします。」

15.3 Zero-shot ReAct Agentの実装

LangChainで最も基本的なAgentはZero-shot ReAct Agentです。「Zero-shot」とは、事前に具体例を与えず、LLMが一般的な知識だけでツールを使いこなすという意味です。

必要なコンポーネント

LLM

意思決定を行うLLM（GPT-4推奨）

Tools

Agentが使用できるツールのリスト

Agent

LLMとToolsを統合する実行エンジン

基本的な実装例

simple_agent.py 
# LangChainとOpenAIのインポート
from langchain.agents import AgentType, initialize_agent, load_tools
from langchain.chat_models import ChatOpenAI

# LLMの初期化（GPT-4推奨）
llm = ChatOpenAI(
    model="gpt-4",
    temperature=0  # 一貫性のある判断のため0に設定
)

# 組み込みツールを読み込み
tools = load_tools(
    ["serpapi", "llm-math"],  # Web検索と計算ツール
    llm=llm
)

# Agentの初期化
agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True  # 実行プロセスを表示
)

# Agentの実行
response = agent.run(
    "東京タワーの高さは何メートル？それを2倍にするといくつ？"
)
print(response)

verboseパラメータ

verbose=Trueにすると、AgentのThought、Action、Observationプロセスがコンソールに出力され、どのように判断しているかを確認できます。

15.4 Agentの実行プロセスを観察する

verbose=Trueで実行すると、以下のような詳細なログが出力されます。これを読み解くことで、Agentの思考プロセスを理解できます。

実行ログの例

> Entering new AgentExecutor chain...

Thought: 東京タワーの高さを知る必要があります。検索ツールを使います。

Action: serpapi
Action Input: "東京タワー 高さ メートル"

Observation: 東京タワーの高さは333メートルです。

Thought: 333メートルを2倍する必要があります。計算ツールを使います。

Action: llm-math
Action Input: 333 * 2

Observation: 666

Thought: 最終的な回答が得られました。

Final Answer: 東京タワーの高さは333メートルです。それを2倍にすると666メートルになります。

実行ログの読み方

  • Thought: LLMが現在の状況をどう解釈し、何を考えているか
  • Action: どのツールを選択したか
  • Action Input: ツールに渡す入力パラメータ
  • Observation: ツールから返された結果
  • Final Answer: 最終的な回答（ユーザーに返される）

15.5 Agentの重要なパラメータ

Agentの動作は、いくつかのパラメータで制御できます。適切に設定することで、安定性と柔軟性のバランスを調整できます。

パラメータ 説明 推奨値
max_iterations 最大実行回数（無限ループ防止） 10～15
max_execution_time 最大実行時間（秒） 60～120
early_stopping_method 上限到達時の停止方法 "generate"
handle_parsing_errors LLM出力の解析エラー処理 True

パラメータを設定した実装例

agent_with_params.py 
# パラメータ設定付きAgent
agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True,
    max_iterations=10,              # 最大10回まで実行
    max_execution_time=60,          # 60秒でタイムアウト
    early_stopping_method="generate", # 上限時に回答生成
    handle_parsing_errors=True       # エラー時に再試行
)

# 実行
response = agent.run("複雑な質問をここに入力")

注意: Agentの実行コスト

AgentはThought→Action→Observationを繰り返すため、1つの質問で複数回LLMを呼び出します。GPT-4を使う場合、コストが高くなる可能性があるため、max_iterationsとmax_execution_timeを適切に設定してください。

実習: 最初のAgentを作る

計算と検索ができるシンプルなAgentを作成してみましょう。

my_first_agent.py 
import os
from langchain.agents import AgentType, initialize_agent, load_tools
from langchain.chat_models import ChatOpenAI

# APIキーの設定（環境変数から読み込み）
os.environ["OPENAI_API_KEY"] = "your-api-key"

# LLMの初期化
llm = ChatOpenAI(model="gpt-4", temperature=0)

# ツールの読み込み（計算ツールのみ）
tools = load_tools(["llm-math"], llm=llm)

# Agentの初期化
agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True,
    max_iterations=5
)

# 実行
print("=== 質問1 ===")
result1 = agent.run("12の2乗と15の2乗を足すといくつ？")
print(f"\n回答: {result1}\n")

print("=== 質問2 ===")
result2 = agent.run("円周率の3倍を計算して")
print(f"\n回答: {result2}\n")

動作確認

  1. 1 実行ログに「Thought」「Action」「Observation」が表示されることを確認
  2. 2 計算ツール（llm-math）が呼び出されていることを確認
  3. 3 正しい計算結果が「Final Answer」として返されることを確認

この章のまとめ

  • Agentは、LLMが状況を判断して動的にツールを選択・実行する仕組み
  • ReActフレームワークは、Thought（思考）→Action（行動）→Observation（観察）を繰り返す
  • Zero-shot ReAct Agentは、事前の具体例なしでツールを使いこなす基本的なAgent
  • verbose=Trueで実行プロセスを可視化し、Agentの判断を観察できる
  • max_iterationsやmax_execution_timeで実行回数とコストを制御できる

理解度確認クイズ

  1. Q1. ChainとAgentの違いは何ですか？

    回答を表示

    Chainは事前に定義された固定的な処理フローを順番に実行します。一方、Agentは、LLMが状況を判断して動的に次のアクションを決定し、必要なツールを選択・実行します。Chainは確実性が高く、Agentは柔軟性が高いという特徴があります。

  2. Q2. ReActフレームワークの3つのステップを説明してください。

    回答を表示

    ①Thought（思考）: LLMが現在の状況を分析し、次に何をすべきか考える。②Action（行動）: 使用するツールとその入力を決定する。③Observation（観察）: ツールの実行結果を受け取り、次の思考に活かす。この3つを繰り返し、回答可能になったら終了します。

  3. Q3. Agentのmax_iterationsパラメータはなぜ重要ですか？

    回答を表示

    max_iterationsは、Agentの最大実行回数を制限するパラメータです。LLMの判断ミスや複雑すぎる質問により無限ループに陥るのを防ぎ、APIコストとレスポンス時間を適切に制御できます。一般的には10～15程度に設定することが推奨されます。
