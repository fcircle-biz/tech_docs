Embeddings - テキストのベクトル化

テキストを数値ベクトルに変換するEmbeddingsの概念と使い方を学びます。OpenAI Embeddings、ローカルモデルなど様々な選択肢を理解します。

この章で学ぶこと

  • Embeddingsの基本概念とベクトル空間の理解
  • OpenAI Embeddingsモデルの使い方
  • ベクトル類似度（コサイン類似度）の計算方法
  • HuggingFaceやローカルモデルなど、OpenAI以外の選択肢

10.1 Embeddingsとは何か

Embeddings（エンベディング）とは、テキストを数値ベクトルに変換する技術です。人間が理解できる自然言語を、コンピュータが計算できる数値の配列に変換することで、テキストの意味的な類似性を数学的に測定できるようになります。

例えば「犬」と「猫」は意味的に近い概念なので、それぞれのEmbeddingsベクトルも近い位置に配置されます。一方、「犬」と「自動車」は意味的に離れているため、ベクトルも離れた位置になります。

Embeddingsの次元数

OpenAIのtext-embedding-3-smallは1536次元、text-embedding-3-largeは3072次元のベクトルを生成します。つまり、1つのテキストが1536個または3072個の数値で表現されます。

Embeddingsの主な用途

意味的検索（Semantic Search）

10.2 OpenAI Embeddingsの使い方

LangChainでは、OpenAIのEmbeddingsモデルを簡単に利用できます。まず基本的な使い方を見ていきましょう。

基本的なEmbeddings生成

from langchain_openai import OpenAIEmbeddings
import numpy as np

# Embeddingsモデルの初期化
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small"  # またはtext-embedding-3-large
)

# 単一テキストのEmbeddings生成
text = "LangChainはLLMアプリ開発のフレームワークです"
embedding_vector = embeddings.embed_query(text)

# ベクトルの確認
print(f"ベクトルの次元数: {len(embedding_vector)}")
print(f"最初の5要素: {embedding_vector[:5]}")
# 出力例:
# ベクトルの次元数: 1536
# 最初の5要素: [0.023, -0.015, 0.042, -0.008, 0.031]

複数ドキュメントの一括Embeddings生成

from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()

# 複数のテキストを一括処理
texts = [
    "LangChainはLLMアプリ開発フレームワーク",
    "Pythonはプログラミング言語です",
    "機械学習はAIの一分野です"
]

# embed_documentsで複数のテキストを一度に処理（効率的）
embedding_vectors = embeddings.embed_documents(texts)

print(f"生成されたベクトル数: {len(embedding_vectors)}")
print(f"各ベクトルの次元数: {len(embedding_vectors[0])}")
# 出力:
# 生成されたベクトル数: 3
# 各ベクトルの次元数: 1536

embed_queryとembed_documentsの使い分け

embed_query(): 検索クエリなど、単一テキストのベクトル化に使用
embed_documents(): 保存用の複数ドキュメントを一括処理（効率的）

10.3 ベクトル類似度の計算

Embeddingsの最も重要な応用は、2つのテキストの意味的類似度を計算することです。代表的な指標がコサイン類似度です。

コサイン類似度とは

2つのベクトルのなす角度の余弦（cos）を計算する指標。-1～1の範囲で、1に近いほど類似、-1に近いほど反対の意味を持ちます。通常、0.7以上で類似と判断します。

コサイン類似度の計算例

from langchain_openai import OpenAIEmbeddings
import numpy as np

# コサイン類似度を計算する関数
def cosine_similarity(vec1, vec2):
    """2つのベクトルのコサイン類似度を計算"""
    dot_product = np.dot(vec1, vec2)  # 内積
    norm1 = np.linalg.norm(vec1)      # ベクトル1のノルム
    norm2 = np.linalg.norm(vec2)      # ベクトル2のノルム
    return dot_product / (norm1 * norm2)

# Embeddings生成
embeddings = OpenAIEmbeddings()

text1 = "犬は忠実な動物です"
text2 = "猫は気ままな動物です"
text3 = "自動車は便利な乗り物です"

vec1 = embeddings.embed_query(text1)
vec2 = embeddings.embed_query(text2)
vec3 = embeddings.embed_query(text3)

# 類似度を計算
sim_1_2 = cosine_similarity(vec1, vec2)
sim_1_3 = cosine_similarity(vec1, vec3)

print(f"「犬」と「猫」の類似度: {sim_1_2:.4f}")
print(f"「犬」と「自動車」の類似度: {sim_1_3:.4f}")
# 出力例:
# 「犬」と「猫」の類似度: 0.8521
# 「犬」と「自動車」の類似度: 0.6237

実習: 類似度判定システムを作る

複数の質問と回答例を用意し、ユーザーの質問に最も近い回答例を自動的に見つけるシステムを作成してみましょう。 

from langchain_openai import OpenAIEmbeddings
import numpy as np

def find_most_similar(query, documents, embeddings):
    """クエリに最も類似したドキュメントを見つける"""
    query_vec = embeddings.embed_query(query)
    doc_vecs = embeddings.embed_documents(documents)

    similarities = [
        np.dot(query_vec, doc_vec) /
        (np.linalg.norm(query_vec) * np.linalg.norm(doc_vec))
        for doc_vec in doc_vecs
    ]

    most_similar_idx = np.argmax(similarities)
    return documents[most_similar_idx], similarities[most_similar_idx]

# FAQ例
faq_questions = [
    "パスワードを忘れました",
    "返品したいです",
    "配送状況を知りたい"
]

# ユーザーの質問
user_query = "ログインできません"

embeddings = OpenAIEmbeddings()
result, score = find_most_similar(user_query, faq_questions, embeddings)

print(f"最も近いFAQ: {result}")
print(f"類似度: {score:.4f}")

10.4 OpenAI以外のEmbeddingsモデル

LangChainは、OpenAI以外にも多様なEmbeddingsモデルをサポートしています。コスト削減やプライバシー要件に応じて選択できます。

HuggingFace Embeddings（無料・オープンソース）

from langchain_huggingface import HuggingFaceEmbeddings

# 日本語対応の多言語モデル
embeddings = HuggingFaceEmbeddings(
    model_name="intfloat/multilingual-e5-base"
)

# 使い方はOpenAIと同じ
text = "LangChainを学習しています"
vector = embeddings.embed_query(text)

print(f"ベクトル次元数: {len(vector)}")
# 出力: ベクトル次元数: 768

Google Vertex AI Embeddings

from langchain_google_vertexai import VertexAIEmbeddings

# Google Cloudのプロジェクト設定が必要
embeddings = VertexAIEmbeddings(
    model_name="textembedding-gecko@003"
)

vector = embeddings.embed_query("テスト文章")
print(f"次元数: {len(vector)}")

Cohere Embeddings

from langchain_cohere import CohereEmbeddings

embeddings = CohereEmbeddings(
    model="embed-multilingual-v3.0",  # 多言語対応
    cohere_api_key="your-api-key"
)

vector = embeddings.embed_query("テスト")

Embeddingsモデルの比較表

モデル 次元数 コスト 特徴
OpenAI text-embedding-3-small 1536 $0.02 / 1M tokens 高性能・高速
OpenAI text-embedding-3-large 3072 $0.13 / 1M tokens 最高精度
HuggingFace multilingual-e5-base 768 無料 オープンソース・ローカル実行可
Cohere embed-multilingual-v3.0 1024 $0.10 / 1M tokens 多言語性能に強み

モデル変更時の注意点

Embeddingsモデルを変更すると、既存のベクトルデータと互換性がなくなります。Vector Storeのデータを全て再生成する必要があるため、本番環境での変更は慎重に行ってください。

10.5 キャッシュによる最適化

同じテキストのEmbeddingsを何度も生成するのは無駄です。LangChainには、計算結果をキャッシュして再利用する機能があります。

CacheBackedEmbeddingsの使用

from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore
from langchain_openai import OpenAIEmbeddings

# ローカルファイルストアを設定
store = LocalFileStore("./cache/embeddings_cache")

# キャッシュ機能を追加したEmbeddings
underlying_embeddings = OpenAIEmbeddings()
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    underlying_embeddings,
    store,
    namespace="openai-embeddings"
)

# 1回目: APIを呼び出して生成（遅い）
vector1 = cached_embeddings.embed_query("LangChainを学習中")

# 2回目: キャッシュから取得（高速）
vector2 = cached_embeddings.embed_query("LangChainを学習中")

# 結果は同じ
print(vector1 == vector2)  # True

キャッシュのメリット

✓ コスト削減: API呼び出し回数を削減
✓ 高速化: ネットワーク通信が不要
✓ 開発効率向上: テスト時の実行時間短縮

理解度確認クイズ

  1. Q1. Embeddingsとは何ですか？なぜ必要なのか説明してください。

    回答を表示

    Embeddingsは、テキストを数値ベクトル（数値の配列）に変換する技術です。人間が理解できる自然言語を、コンピュータが計算できる形式に変換することで、テキスト間の意味的な類似性を数学的に測定できるようになります。RAGシステムでの関連ドキュメント検索や、レコメンデーション、クラスタリングなど幅広い用途で必須の技術です。

  2. Q2. embed_query()とembed_documents()の違いは何ですか？

    回答を表示

    embed_query()は、検索クエリなど単一のテキストをベクトル化するメソッドです。一方、embed_documents()は複数のドキュメントを一括でベクトル化するメソッドで、内部でバッチ処理を行うため効率的です。保存用のドキュメントは複数まとめてembed_documents()で処理し、検索時のクエリはembed_query()で処理するのが一般的です。

  3. Q3. コサイン類似度とは何ですか？どのような値の範囲を取りますか？

    回答を表示

    コサイン類似度は、2つのベクトルのなす角度の余弦（cos）を計算する指標で、ベクトル間の類似性を測定します。値の範囲は-1～1で、1に近いほど類似、0は無関係、-1に近いほど反対の意味を持ちます。通常、0.7以上で類似と判断することが多いです。計算式は「内積 ÷ (ベクトル1のノルム × ベクトル2のノルム)」です。
