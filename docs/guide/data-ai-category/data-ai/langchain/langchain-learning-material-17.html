第17章

Callbacks - 実行の監視とデバッグ

LangChainの処理をリアルタイムで監視するCallbackシステムを学びます。ログ出力、ストリーミング、LangSmith連携によるデバッグ手法を習得します。

この章で学ぶこと

  • Callbackシステムの基本概念と目的を理解する
  • 標準Callbackを使ったログ出力とデバッグ方法を習得する
  • ストリーミング出力でリアルタイム表示を実装する
  • LangSmith連携で本格的なモニタリング環境を構築する

17.1 Callbacksとは何か

Callbacksは、LangChainの処理実行中に発生する様々なイベント（LLM呼び出し開始、トークン生成、チェーン実行など）をフックして、ログ出力やカスタム処理を実行できる仕組みです。

Callbackを使うことで、LangChainアプリケーションの内部動作を詳細に把握でき、デバッグやパフォーマンスチューニングが容易になります。

Callbacksの主な用途

  • • LLM呼び出しのログ記録（プロンプト、応答、トークン数など）
  • • 処理時間の計測とパフォーマンス分析
  • • リアルタイムストリーミング表示
  • • エラートラッキングとデバッグ
  • • 使用コストの監視（API呼び出し回数、トークン消費量）

17.2 標準Callbackの使用

LangChainには、すぐに使える標準Callbackハンドラーが用意されています。最もシンプルなのはStdOutCallbackHandlerで、処理の詳細を標準出力に表示します。

基本的なログ出力

basic_callback.py 
from langchain_openai import ChatOpenAI
from langchain.callbacks import StdOutCallbackHandler
from langchain.prompts import ChatPromptTemplate

# ログ出力用のCallbackハンドラー
handler = StdOutCallbackHandler()

# LLMの初期化（Callbackを指定）
llm = ChatOpenAI(
    model="gpt-4",
    callbacks=[handler]
)

# プロンプトテンプレート
prompt = ChatPromptTemplate.from_messages([
    ("system", "あなたは親切なアシスタントです。"),
    ("user", "{question}")
])

# チェーンの実行
chain = prompt | llm
response = chain.invoke({"question": "LangChainとは何ですか？"})

出力例

on_llm_start: prompts=['System: あなたは親切なアシスタントです。\nHuman: LangChainとは何ですか？']
on_llm_end: token_usage={'prompt_tokens': 25, 'completion_tokens': 120, 'total_tokens': 145}

詳細ログの取得

verbose_callback.py

17.3 ストリーミング出力

LLMの応答をトークンごとにリアルタイム表示するには、ストリーミングCallbackを使用します。チャットUIで応答が徐々に表示される体験を実装できます。

streaming_callback.py 
from langchain_openai import ChatOpenAI
from langchain.callbacks.streaming_stdout import StreamingStdOutCallbackHandler

# ストリーミング表示用のCallback
llm = ChatOpenAI(
    model="gpt-4",
    streaming=True,  # ストリーミングを有効化
    callbacks=[StreamingStdOutCallbackHandler()]
)

# トークンが生成されるたびに即座に表示される
response = llm.invoke("東京の観光名所を3つ教えて")

# 出力例: 「東」「京」「の」「観光」... と1トークンずつ表示

カスタムストリーミングCallback

custom_streaming.py 
from langchain.callbacks.base import BaseCallbackHandler
from langchain_openai import ChatOpenAI

class CustomStreamingHandler(BaseCallbackHandler):
    """カスタムストリーミングハンドラー"""

    def on_llm_new_token(self, token: str, **kwargs):
        """新しいトークンが生成されたとき"""
        print(f"[TOKEN] {token}", end="", flush=True)

    def on_llm_end(self, response, **kwargs):
        """LLM呼び出しが終了したとき"""
        print("\n[COMPLETED]")

# カスタムハンドラーの使用
llm = ChatOpenAI(
    model="gpt-4",
    streaming=True,
    callbacks=[CustomStreamingHandler()]
)

response = llm.invoke("Pythonとは何ですか？")

実習: ストリーミングチャットボット

ストリーミングCallbackを使って、応答をリアルタイム表示するチャットボットを作成してみましょう。

streaming_chat.py 
from langchain_openai import ChatOpenAI
from langchain.callbacks.streaming_stdout import StreamingStdOutCallbackHandler
from langchain.prompts import ChatPromptTemplate
from langchain.memory import ConversationBufferMemory

# ストリーミング対応チャットボット

    model="gpt-4",
    streaming=True,
    callbacks=[StreamingStdOutCallbackHandler()]
)

prompt = ChatPromptTemplate.from_messages([
    ("system", "あなたは親切なアシスタントです。"),
    ("user", "{input}")

print("チャットボット起動（'quit'で終了）\n")
while True:

    if user_input.lower() == "quit":
        break

    print("Bot: ", end="")
    response = chain.invoke({"input": user_input})
    print()  # 改行

17.4 カスタムCallbackの作成

独自のロジックを実装するには、BaseCallbackHandlerを継承してカスタムCallbackを作成します。

主要なCallbackメソッド

メソッド タイミング 用途
on_llm_start LLM呼び出し開始 プロンプトのログ記録
on_llm_new_token トークン生成時 ストリーミング表示
on_llm_end LLM呼び出し終了 トークン数、処理時間記録
on_chain_start チェーン実行開始 処理フロー追跡
on_chain_end チェーン実行終了 結果のログ記録
on_tool_start ツール実行開始 Agent動作の追跡
on_agent_action Agentアクション実行 意思決定の記録

コスト監視Callback

cost_tracking_callback.py 
from langchain.callbacks.base import BaseCallbackHandler
from typing import Any, Dict

class CostTrackingHandler(BaseCallbackHandler):
    """コスト追跡用Callback"""

    def __init__(self):
        self.total_tokens = 0
        self.prompt_tokens = 0
        self.completion_tokens = 0

    def on_llm_end(self, response, **kwargs):
        """LLM終了時にトークン数を集計"""
        if hasattr(response, "llm_output"):
            token_usage = response.llm_output.get("token_usage", {})
            self.prompt_tokens += token_usage.get("prompt_tokens", 0)
            self.completion_tokens += token_usage.get("completion_tokens", 0)
            self.total_tokens += token_usage.get("total_tokens", 0)

    def get_cost_summary(self):
        """コスト概算を返す（GPT-4の場合）"""
        prompt_cost = self.prompt_tokens * 0.03 / 1000
        completion_cost = self.completion_tokens * 0.06 / 1000
        total_cost = prompt_cost + completion_cost

        return {
            "prompt_tokens": self.prompt_tokens,
            "completion_tokens": self.completion_tokens,
            "total_tokens": self.total_tokens,
            "estimated_cost_usd": total_cost
        }

# 使用例
cost_handler = CostTrackingHandler()

llm = ChatOpenAI(
    model="gpt-4",
    callbacks=[cost_handler]
)

# 複数回の呼び出し
llm.invoke("こんにちは")
llm.invoke("Python について教えて")

# コスト集計
summary = cost_handler.get_cost_summary()
print(f"総トークン数: {summary['total_tokens']}")
print(f"推定コスト: ${summary['estimated_cost_usd']:.4f}")

17.5 LangSmith連携

LangSmithは、LangChain公式のモニタリング・デバッグプラットフォームです。すべての実行履歴、プロンプト、応答、トークン数、処理時間などを可視化できます。

LangSmithの主な機能

  • • 実行トレース: すべてのChain/Agent実行の詳細を記録
  • • プロンプト管理: バージョン管理と共有
  • • パフォーマンス分析: レイテンシ、トークン数の可視化
  • • エラートラッキング: 失敗したリクエストの分析
  • • 評価機能: LLM出力の品質評価

LangSmithのセットアップ

setup_langsmith.py 
import os

# 環境変数でLangSmithを有効化
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_ENDPOINT"] = "https://api.smith.langchain.com"
os.environ["LANGCHAIN_API_KEY"] = "your-langsmith-api-key"
os.environ["LANGCHAIN_PROJECT"] = "my-project"  # プロジェクト名

# これ以降のLangChain実行は自動的にLangSmithに記録される

Agent実行の可視化

debug_agent.py 
from langchain.agents import AgentExecutor, create_react_agent
from langchain_openai import ChatOpenAI
from langchain.tools import Tool
from langchain.callbacks.base import BaseCallbackHandler

class AgentDebugHandler(BaseCallbackHandler):
    """Agent実行の詳細を出力"""

    def on_agent_action(self, action, **kwargs):
        print(f"\n[ACTION] Tool: {action.tool}")
        print(f"[INPUT] {action.tool_input}")

    def on_tool_start(self, tool_name, input_str, **kwargs):
        print(f"\n[TOOL START] {tool_name}")

    def on_tool_end(self, output, **kwargs):
        print(f"[TOOL OUTPUT] {output}")

# デバッグハンドラー付きでAgent実行
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    callbacks=[AgentDebugHandler()],
    verbose=True
)

result = agent_executor.invoke({
    "input": "2024年の東京の人口を調べて"
})

トラブルシューティング

問題: ストリーミングが表示されない

原因: streaming=Trueの設定忘れ、またはモデルがストリーミング非対応

解決: LLM初期化時にstreaming=Trueを設定。一部ローカルモデルは非対応の場合あり

問題: LangSmithにトレースが表示されない

原因: APIキーの設定ミス、環境変数の読み込み順序

解決: LANGCHAIN_TRACING_V2=trueとLANGCHAIN_API_KEYが正しく設定されているか確認。load_dotenv()をLangChainインポート前に実行

問題: Callbackが呼ばれない

原因: Callbackの登録場所が誤っている

解決: LLM初期化時のcallbacksパラメータ、またはinvoke()時のconfig={"callbacks": [handler]}で指定

この章のまとめ

  • Callbacksは、LangChainの処理実行中のイベントをフックしてログやカスタム処理を実行する仕組み
  • StdOutCallbackHandlerやStreamingStdOutCallbackHandlerなど、標準ハンドラーをすぐに利用可能
  • ストリーミング出力により、LLM応答をリアルタイムで表示可能
  • BaseCallbackHandlerを継承してカスタムCallbackを作成し、コスト監視などの独自処理を実装
  • LangSmithと連携することで、すべての実行履歴を可視化し、本格的なモニタリング環境を構築可能

理解度確認クイズ

  1. Q1. Callbacksの主な用途を3つ挙げてください。

    回答を表示

    ①LLM呼び出しのログ記録（プロンプト、応答、トークン数など）、②リアルタイムストリーミング表示、③処理時間の計測とパフォーマンス分析、④エラートラッキングとデバッグ、⑤使用コストの監視などです。これらにより、LLMアプリケーションの内部動作を詳細に把握できます。

  2. Q2. ストリーミング出力を実装するには、どのような設定が必要ですか？

    回答を表示

    LLM初期化時にstreaming=Trueを設定し、StreamingStdOutCallbackHandlerまたはon_llm_new_tokenメソッドを実装したカスタムCallbackをcallbacksパラメータで指定します。これにより、トークンが生成されるたびにリアルタイムで表示されます。

  3. Q3. LangSmithの主な機能と、開発においてどのような価値を提供するか説明してください。

    回答を表示

    LangSmithは、実行トレース（すべてのChain/Agent実行の詳細記録）、プロンプト管理、パフォーマンス分析（レイテンシ・トークン数の可視化）、エラートラッキング、評価機能などを提供します。これにより、LLMアプリケーションの動作を可視化し、デバッグ、最適化、品質向上を効率的に行えます。本番環境での問題調査にも有効です。
