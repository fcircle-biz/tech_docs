Output Parsers - LLMの出力を構造化する

LLMの自然言語出力をJSON、リスト、Pydanticモデルなど、プログラムで扱いやすい構造化データに変換する方法を習得します。

この章で学ぶこと

  • Output Parserの必要性と役割を理解する
  • PydanticOutputParserを使った型安全な出力の実装
  • StructuredOutputParser、CommaSeparatedListOutputParserなど様々なParserの使い分け
  • パース失敗時の自動リトライとエラーハンドリング

5.1 Output Parserとは何か

LLMは基本的に自然言語のテキストを出力します。しかし、プログラムで扱うには、JSON、リスト、辞書などの構造化されたデータが必要です。

Output Parserは、LLMの出力を解析（パース）し、プログラムが扱いやすい形式に変換する機能です。LangChainでは、様々な形式に対応したParserが用意されています。

Output Parserの2つの役割

①プロンプトに「出力形式の指示」を自動追加、②LLMの出力を指定した型にパース（変換）

5.2 なぜOutput Parserが必要なのか

Output Parserなしの場合の課題

output_parser不使用の例 
# LLMにJSONを出力させる（Output Parserなし）
prompt = "次の商品情報をJSON形式で出力してください。商品名: MacBook Air, 価格: 150000"
result = llm.predict(prompt)
print(result)
# 出力例: "商品情報は以下の通りです。\n```json\n{\"name\": \"MacBook Air\", \"price\": 150000}\n```"

# 手動でパースする必要がある
import json
import re
json_match = re.search(r'\{.*\}', result, re.DOTALL)
data = json.loads(json_match.group())  # エラーが起きやすい

Output Parserを使った場合

StructuredOutputParserを使った例 
from langchain.output_parsers import StructuredOutputParser, ResponseSchema

# スキーマ定義
response_schemas = [
    ResponseSchema(name="name", description="商品名"),
    ResponseSchema(name="price", description="価格（数値）")
]
output_parser = StructuredOutputParser.from_response_schemas(response_schemas)

# プロンプトに形式指示を自動追加
format_instructions = output_parser.get_format_instructions()
prompt = f"次の商品情報を抽出してください。\n商品名: MacBook Air, 価格: 150000\n\n{format_instructions}"

# LLM実行と自動パース
result = llm.predict(prompt)
data = output_parser.parse(result)  # 自動的に辞書に変換
print(data)  # {'name': 'MacBook Air', 'price': 150000}

Output Parserの利点

  • 形式指示の自動生成: get_format_instructions()でLLMへの指示を自動作成
  • 型安全性: Pydanticモデルで厳密な型チェックが可能
  • エラーハンドリング: パース失敗時の自動リトライ機能
  • コードの簡潔性: 手動パース処理が不要

5.3 主要なOutput Parserの種類

LangChainには、用途に応じた様々なOutput Parserが用意されています。

PydanticOutputParser

Pydanticモデルを使った型安全な出力。最も推奨される方法

StructuredOutputParser

ResponseSchemaで定義した辞書形式のデータを出力

CommaSeparatedListOutputParser

カンマ区切りのリストを出力（例: "リンゴ, バナナ, オレンジ"）

JsonOutputParser

JSON形式のデータを辞書に変換（シンプルなJSON用）

5.4 PydanticOutputParserの実装

PydanticOutputParserは、型安全性とバリデーション機能を備えた最も強力なParserです。Pydanticモデルで出力の構造を定義します。

基本的な使い方

PydanticOutputParserの基本実装 
from langchain.output_parsers import PydanticOutputParser
from pydantic import BaseModel, Field

# Pydanticモデルの定義
class ProductInfo(BaseModel):
    name: str = Field(description="商品名")
    price: int = Field(description="価格（円）")
    category: str = Field(description="カテゴリ")
    in_stock: bool = Field(description="在庫あり/なし")

# Parserの作成
parser = PydanticOutputParser(pydantic_object=ProductInfo)

# プロンプト作成
from langchain.prompts import PromptTemplate
prompt = PromptTemplate(
    template="次のテキストから商品情報を抽出してください。\n{format_instructions}\n\nテキスト: {text}",
    input_variables=["text"],
    partial_variables={"format_instructions": parser.get_format_instructions()}
)

# LLM実行
from langchain.chat_models import ChatOpenAI
llm = ChatOpenAI(temperature=0)
chain = prompt | llm | parser

# 実行
result = chain.invoke({"text": "MacBook Air、価格は150000円、ノートPCカテゴリ、在庫あり"})
print(type(result))  # 
print(result.name)   # MacBook Air
print(result.price)  # 150000

get_format_instructions()の出力例

Parserは以下のような形式指示をLLMに自動追加します:
"The output should be formatted as a JSON instance that conforms to the JSON schema below..."

ネストした構造のパース

複雑なデータ構造のパース 
from typing import List

# ネストした構造のモデル
class Author(BaseModel):
    name: str = Field(description="著者名")
    nationality: str = Field(description="国籍")

class Book(BaseModel):
    title: str = Field(description="書籍タイトル")
    authors: List[Author] = Field(description="著者リスト")
    publication_year: int = Field(description="出版年")
    isbn: str = Field(description="ISBN番号")

# Parserとチェーン作成
parser = PydanticOutputParser(pydantic_object=Book)
# （プロンプト・LLM・chain設定は同様）

# 実行
text = "『罪と罰』はドストエフスキー（ロシア）が1866年に出版した小説。ISBN: 978-4-10-600001-9"
result = chain.invoke({"text": text})
print(result.authors[0].name)  # ドストエフスキー

実習: Output Parserを使った情報抽出

以下の課題に取り組み、Output Parserの使い方を体験しましょう。

課題

ニュース記事から「タイトル」「日付」「カテゴリ」「要約（3文以内）」を抽出するPydanticOutputParserを実装してください。

解答例を表示
解答例 
from langchain.output_parsers import PydanticOutputParser
from pydantic import BaseModel, Field
from langchain.prompts import PromptTemplate
from langchain.chat_models import ChatOpenAI

class NewsArticle(BaseModel):
    title: str = Field(description="記事のタイトル")
    date: str = Field(description="日付（YYYY-MM-DD形式）")
    category: str = Field(description="カテゴリ（政治/経済/スポーツ等）")
    summary: str = Field(description="記事の要約（3文以内）")

parser = PydanticOutputParser(pydantic_object=NewsArticle)

prompt = PromptTemplate(
    template="以下のニュース記事から情報を抽出してください。\n{format_instructions}\n\n記事:\n{article}",
    input_variables=["article"],
    partial_variables={"format_instructions": parser.get_format_instructions()}
)

llm = ChatOpenAI(temperature=0)
chain = prompt | llm | parser

article_text = """
2025年1月3日 - 経済ニュース
日本銀行は本日、金融政策決定会合を開催し、現行の金利政策を維持することを決定した。
インフレ率の動向を注視しつつ、経済成長を支援する姿勢を継続する。
市場関係者からは、今後の政策変更の可能性について様々な見方が示されている。
"""

result = chain.invoke({"article": article_text})
print(f"タイトル: {result.title}")
print(f"日付: {result.date}")
print(f"カテゴリ: {result.category}")
print(f"要約: {result.summary}")

5.5 エラーハンドリングとリトライ

LLMが指定された形式で出力しない場合、パースエラーが発生します。LangChainには、エラー時に自動でLLMにフィードバックして修正させるOutputFixingParserがあります。

OutputFixingParserを使った自動リトライ 
from langchain.output_parsers import OutputFixingParser

# 元のParser
base_parser = PydanticOutputParser(pydantic_object=ProductInfo)

# エラー修正機能付きParser
fixing_parser = OutputFixingParser.from_llm(parser=base_parser, llm=llm)

# 不正な出力でもリトライして修正
bad_output = "商品名はMacBook Airで、価格は高いです。"  # priceが数値でない
result = fixing_parser.parse(bad_output)  # LLMに修正を依頼して再パース

注意: OutputFixingParserのコスト

OutputFixingParserは追加のLLM呼び出しを行うため、コストが増加します。本番環境では、プロンプトの改善でエラーを減らすことが推奨されます。

5.6 その他の便利なParser

CommaSeparatedListOutputParser

カンマ区切りリストのパース 
from langchain.output_parsers import CommaSeparatedListOutputParser

parser = CommaSeparatedListOutputParser()
format_instructions = parser.get_format_instructions()

prompt = f"5つのプログラミング言語を列挙してください。\n{format_instructions}"
result = llm.predict(prompt)
parsed = parser.parse(result)
print(parsed)  # ['Python', 'JavaScript', 'Java', 'C++', 'Go']

DatetimeOutputParser

日付データのパース 
from langchain.output_parsers import DatetimeOutputParser

parser = DatetimeOutputParser()
prompt = f"2025年1月3日をパースしてください。\n{parser.get_format_instructions()}"
result = llm.predict(prompt)
parsed = parser.parse(result)
print(type(parsed))  # 
print(parsed)        # 2025-01-03 00:00:00

この章のまとめ

  • Output Parserは、LLMの自然言語出力を構造化データに変換する機能
  • 形式指示の自動生成とパース処理により、コードの簡潔性と信頼性が向上
  • PydanticOutputParserは型安全性とバリデーション機能を備え、最も推奨される
  • 用途に応じてStructured、List、JSON、Datetimeなど様々なParserを使い分ける
  • OutputFixingParserでパースエラー時の自動リトライが可能

理解度確認クイズ

  1. Q1. Output Parserの主な役割を2つ説明してください。

    回答を表示

    ①プロンプトに出力形式の指示を自動追加する、②LLMの自然言語出力を指定した型（JSON、リスト、Pydanticモデルなど）にパース（変換）する。これにより、手動でのパース処理が不要になり、型安全性が向上します。

  2. Q2. PydanticOutputParserが他のParserより推奨される理由は何ですか？

    回答を表示

    Pydanticモデルを使用することで、①型安全性（str、int、boolなど厳密な型定義）、②バリデーション機能（Fieldによる制約チェック）、③ネストした複雑な構造のサポート、④IDEの補完・型ヒントが効く、などのメリットがあり、堅牢なアプリケーション開発に適しているためです。

  3. Q3. OutputFixingParserの役割と注意点を説明してください。

    回答を表示

    OutputFixingParserは、パースエラーが発生した際に、LLMにエラー内容をフィードバックして自動的に修正させる機能です。役割: 不正な出力を自動修正してパース成功率を向上。注意点: 追加のLLM呼び出しが発生するためコストが増加するので、本番環境ではプロンプトの改善でエラーを減らすことが推奨されます。
