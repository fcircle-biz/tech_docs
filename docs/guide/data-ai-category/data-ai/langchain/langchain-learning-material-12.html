第12章

Retrievers - 関連情報の検索

Vector StoreをベースにしたRetrieverの構築と、検索精度を向上させるテクニック（MMR、セルフクエリなど）を習得します。

この章で学ぶこと

  • Retrieverの基本概念とVector Storeとの関係を理解する
  • 類似度検索とMMR（最大限界関連性）検索の違いを学ぶ
  • 検索パラメータ（k、score_threshold）のチューニング方法を習得する
  • Self-Query Retrieverやアンサンブル検索など高度な検索手法を理解する

12.1 Retrieverとは何か

Retrieverは、ユーザーのクエリ（質問）に対して、関連性の高いドキュメントを取得するための抽象レイヤーです。前章で学んだVector Storeの検索機能をより使いやすく、柔軟に拡張したものと考えることができます。

Vector Storeは「ベクトル空間での類似検索」という低レベルの操作を提供しますが、Retrieverはそれを「自然言語クエリから関連ドキュメントを取得」という高レベルのインターフェースに変換します。

Vector Store vs Retriever

Vector Storeは「データベース」、Retrieverは「検索エンジン」に例えられます。同じデータでも、検索方法や結果の返し方によって使い分けができます。

12.2 基本的なRetrieverの構築

Vector Storeから最もシンプルにRetrieverを作成する方法は、as_retriever()メソッドを使うことです。

basic_retriever.py 
# Vector StoreからRetrieverを作成
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings

# 既存のVector Store（前章で作成済みと仮定）
vectorstore = FAISS.load_local("db/faiss_index", OpenAIEmbeddings(), allow_dangerous_deserialization=True)

# Retrieverに変換
retriever = vectorstore.as_retriever()

# 検索実行
docs = retriever.get_relevant_documents("LangChainでRAGを構築する方法は？")
print(f"検索結果: {len(docs)}件")
for i, doc in enumerate(docs):
    print(f"\n--- ドキュメント {i+1} ---")
    print(doc.page_content[:200])

検索パラメータのカスタマイズ

as_retriever()には検索条件を細かく指定できます。

customized_retriever.py 
# パラメータをカスタマイズしたRetriever
retriever = vectorstore.as_retriever(
    search_type="similarity",  # 検索タイプ: similarity, mmr, similarity_score_threshold
    search_kwargs={
        "k": 3,  # 取得するドキュメント数（デフォルト: 4）
        "score_threshold": 0.7  # 類似度の閾値（0.0-1.0）
    }
)

# 検索実行
docs = retriever.get_relevant_documents("Embeddingsとは何ですか？")
for i, doc in enumerate(docs):
    print(f"ドキュメント{i+1}: {doc.page_content[:100]}...")

主要なRetrieverパラメータ

search_type

  • similarity: 類似度順（デフォルト）
  • mmr: 多様性を考慮した検索
  • similarity_score_threshold: 閾値以上のみ

search_kwargs

  • k: 取得件数（推奨: 3-5件）
  • score_threshold: 最小スコア（0.0-1.0）
  • fetch_k: MMR用の候補数

12.3 MMR検索 - 多様性を重視した検索

MMR（Maximal Marginal Relevance）は、検索結果の「関連性」と「多様性」のバランスを取る手法です。通常の類似度検索では、似たような内容のドキュメントばかりが返されることがありますが、MMRはそれを防ぎます。

MMRの実装

mmr_retriever.py 
# MMR検索を使ったRetriever
mmr_retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={
        "k": 5,  # 最終的に返す件数
        "fetch_k": 20,  # MMR計算用の候補数（多いほど多様性が高まる）
        "lambda_mult": 0.5  # 関連性と多様性のバランス（0=多様性重視、1=関連性重視）
    }
)

# 検索実行
docs = mmr_retriever.get_relevant_documents("機械学習のアルゴリズム")
for i, doc in enumerate(docs):
    print(f"\n【ドキュメント{i+1}】")
    print(doc.page_content[:150])

lambda_multパラメータの使い分け

lambda_mult=0.0: 多様性最優先（異なるトピックを網羅）
lambda_mult=0.5: バランス型（デフォルト）
lambda_mult=1.0: 関連性最優先（通常の類似度検索と同じ）

12.4 スコア閾値検索 - 低品質な結果を排除

類似度が低いドキュメント（関連性が薄い結果）を自動的にフィルタリングする方法です。「関連するドキュメントが存在しない場合は何も返さない」という挙動が必要な場合に有効です。

threshold_retriever.py 
# スコア閾値を設定したRetriever
threshold_retriever = vectorstore.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={
        "score_threshold": 0.8,  # 0.8以上のスコアのみ取得
        "k": 5
    }
)

# 関連性が高いクエリ
docs = threshold_retriever.get_relevant_documents("LangChainのDocument Loaders")
print(f"高関連クエリ結果: {len(docs)}件")

# 関連性が低いクエリ
docs = threshold_retriever.get_relevant_documents("明日の天気予報")
print(f"低関連クエリ結果: {len(docs)}件")  # 閾値未満なら0件

注意: Vector Storeごとにスコアの範囲が異なる

FAISSはユークリッド距離（小さいほど類似）、Chromaはコサイン類似度（大きいほど類似）を使用します。閾値設定時は使用しているVector Storeのスコアリング方法を確認してください。

12.5 高度なRetriever - Self-Queryとアンサンブル

Self-Query Retriever

ユーザーのクエリからメタデータフィルタ条件を自動抽出して検索する高度なRetrieverです。「2023年以降の記事で、著者がJohnのもの」といった複雑な条件を自然言語で指定できます。

self_query_retriever.py 
# Self-Query Retrieverの構築
from langchain.retrievers.self_query.base import SelfQueryRetriever
from langchain.chains.query_constructor.base import AttributeInfo
from langchain_openai import ChatOpenAI

# メタデータの定義
metadata_field_info = [
    AttributeInfo(name="source", description="ドキュメントの出典", type="string"),
    AttributeInfo(name="year", description="作成年", type="integer"),
]

# Self-Query Retrieverの作成
llm = ChatOpenAI(temperature=0)
self_query_retriever = SelfQueryRetriever.from_llm(
    llm=llm,
    vectorstore=vectorstore,
    document_contents="技術文書",
    metadata_field_info=metadata_field_info
)

# 自然言語での複雑な検索
docs = self_query_retriever.get_relevant_documents(
    "2023年以降に作成されたPythonに関する記事を探して"
)

アンサンブルRetriever

複数の異なるRetrieverの結果を組み合わせて、検索精度を向上させる手法です。例えば、「ベクトル検索」と「キーワード検索（BM25）」を組み合わせることができます。

ensemble_retriever.py 
# アンサンブルRetrieverの構築
from langchain.retrievers import EnsembleRetriever
from langchain_community.retrievers import BM25Retriever

# BM25Retriever（キーワードベース）の準備
documents = ["LangChainの概要", "Retrieverの使い方", "RAGの実装方法"]
bm25_retriever = BM25Retriever.from_texts(documents)
bm25_retriever.k = 3

# ベクトル検索Retriever
vector_retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

# アンサンブル（重み付け平均）
ensemble_retriever = EnsembleRetriever(
    retrievers=[bm25_retriever, vector_retriever],
    weights=[0.4, 0.6]  # BM25:40%, ベクトル検索:60%
)

docs = ensemble_retriever.get_relevant_documents("RAGの実装")

12.6 Retrieverのベストプラクティス

検索精度を高めるためのTips

1 k（取得件数）は少なめから始める

k=3〜5件から始め、必要に応じて増やす。多すぎるとノイズが混入し、LLMへの負荷も増加します。

2 用途に応じて検索タイプを使い分ける

FAQ検索：similarity、要約用検索：mmr、厳密な検索：similarity_score_threshold

3 メタデータを積極的に活用

Document作成時にsource、author、dateなどのメタデータを付与しておくと、Self-Query Retrieverで高度な検索が可能になります。

4 アンサンブルで精度向上

重要なアプリケーションでは、複数のRetrieverを組み合わせることで、単一手法の弱点を補完できます。

検索タイプ選択ガイド

用途 推奨検索タイプ 推奨k値
FAQ回答 similarity_score_threshold 1-3
文書要約のための情報収集 mmr（lambda=0.5） 5-10
社内文書検索 similarity 3-5
高精度な情報抽出 ensemble 3-5

この章のまとめ

  • RetrieverはVector Storeの検索機能を抽象化し、柔軟な検索を可能にする
  • MMR検索を使うと、関連性と多様性のバランスが取れた結果が得られる
  • スコア閾値検索で低品質な結果を自動的にフィルタリングできる
  • Self-Query RetrieverやアンサンブルRetrieverで高度な検索が実現できる
  • 用途に応じて検索タイプとパラメータを最適化することが重要

理解度確認クイズ

  1. Q1. RetrieverとVector Storeの違いは何ですか？

    回答を表示

    Vector Storeは「ベクトルデータを保存・検索するデータベース」であり、低レベルの類似検索APIを提供します。一方、Retrieverは「自然言語クエリから関連ドキュメントを取得する抽象レイヤー」で、検索タイプの切り替え、スコアフィルタリング、複数検索の組み合わせなど、高レベルな検索機能を提供します。

  2. Q2. MMR検索が通常の類似度検索より優れている点は何ですか？

    回答を表示

    MMR検索は「関連性」と「多様性」のバランスを取ることができます。通常の類似度検索では似たような内容のドキュメントばかりが返されることがありますが、MMRは異なる側面の情報を含む結果を返すため、文書要約や幅広い情報収集に適しています。lambda_multパラメータで関連性と多様性のバランスを調整できます。

  3. Q3. アンサンブルRetrieverの利点と、組み合わせる検索手法の例を挙げてください。

    回答を表示

    アンサンブルRetrieverは複数の検索手法の結果を統合することで、単一手法の弱点を補い、検索精度を向上させます。代表的な組み合わせは「BM25Retriever（キーワードベース検索）」と「Vector Retriever（意味ベース検索）」です。前者は完全一致や固有名詞に強く、後者は意味の類似性を捉えるため、両者を組み合わせることで幅広いクエリに対応できます。
