第8章

Document Loaders - 様々なデータソースの読み込み

PDF、Web、CSV、データベースなど、多様なデータソースからドキュメントを読み込む方法を学びます。RAG構築の基礎となるデータ取得を習得します。

この章で学ぶこと

  • Document Loaderの基本概念とRAGにおける役割を理解する
  • PDF、Web、CSV、Markdownなど様々なデータソースの読み込み方法を習得する
  • Documentオブジェクトの構造とメタデータの活用方法を学ぶ
  • 複数ファイルの一括読み込みとディレクトリ処理の実装を理解する

8.1 Document Loaderとは

Document Loaderは、様々なデータソース(PDF、Web、データベースなど)からドキュメントを読み込み、LangChainで扱えるDocument形式に変換するコンポーネントです。RAG(検索拡張生成)システムにおいて、外部データを取り込む最初のステップを担います。

RAGデータパイプラインにおける位置づけ

Document Loaderは、RAG実装の起点です。ここで読み込んだデータが、Text Splitters(分割)→ Embeddings(ベクトル化)→ Vector Stores(保存)と流れ、最終的にLLMが参照するナレッジベースになります。

Documentオブジェクトの構造

すべてのDocument Loaderは、データをDocumentオブジェクトのリストに変換します。

document_structure.py 
from langchain.schema import Document

# Documentオブジェクトの構造
doc = Document(
    page_content="ここに本文テキストが入る",  # 必須
    metadata={                               # オプション
        "source": "example.pdf",
        "page": 1,
        "author": "山田太郎"
    }
)

print(doc.page_content)  # 本文
print(doc.metadata)      # メタデータ辞書

page_content

ドキュメントの本文テキスト。LLMに渡される実際の内容。

metadata

ファイル名、ページ番号、タイムスタンプなどの補足情報。検索フィルタに活用可能。

8.2 PDFファイルの読み込み

PDF形式の文書を読み込む方法を学びます。LangChainでは複数のPDFローダーが提供されており、用途に応じて使い分けます。

PyPDFLoaderを使った基本的な読み込み

pdf_loader.py 
# PyPDFLoaderのインストール
# pip install pypdf

from langchain.document_loaders import PyPDFLoader

# PDFファイルを読み込み
loader = PyPDFLoader("sample.pdf")
documents = loader.load()  # Documentオブジェクトのリスト

# 読み込み結果を確認
print(f"読み込んだページ数: {len(documents)}")
print(f"1ページ目の内容:\n{documents[0].page_content[:200]}")
print(f"メタデータ: {documents[0].metadata}")

PyPDFLoaderの特徴

PyPDFLoaderは、PDFをページ単位で分割します。各ページが1つのDocumentオブジェクトになり、metadata['page']にページ番号が自動設定されます。

複数ページのPDFから特定ページを抽出

pdf_filter.py 
from langchain.document_loaders import PyPDFLoader

loader = PyPDFLoader("manual.pdf")
documents = loader.load()

# 特定ページのみ抽出(例: 3〜5ページ目)
target_pages = [doc for doc in documents if 3 <= doc.metadata['page'] <= 5]

# キーワードを含むページを検索
keyword = "API仕様"
matching_pages = [doc for doc in documents if keyword in doc.page_content]

print(f"3〜5ページの文書数: {len(target_pages)}")
print(f"'{keyword}'を含むページ数: {len(matching_pages)}")

8.3 Webページの読み込み

WebサイトやブログのコンテンツをRAGシステムに取り込む方法を学びます。

WebBaseLoaderでHTMLを読み込み

web_loader.py 
# WebBaseLoaderのインストール
# pip install beautifulsoup4

from langchain.document_loaders import WebBaseLoader

# Webページを読み込み
url = "https://example.com/article"
loader = WebBaseLoader(url)
documents = loader.load()

print(f"タイトル: {documents[0].metadata['title']}")
print(f"本文(最初の300文字):\n{documents[0].page_content[:300]}")
print(f"ソースURL: {documents[0].metadata['source']}")

注意

WebBaseLoaderは、JavaScriptで動的に生成されるコンテンツには対応していません。その場合はSeleniumやPlaywrightを使ったローダーを検討してください。

8.4 CSV・Excelファイルの読み込み

表形式データをRAGシステムに取り込む方法を学びます。各行がDocumentオブジェクトに変換されます。

CSVLoaderの基本的な使い方

csv_loader.py 
from langchain.document_loaders import CSVLoader

# CSVファイルを読み込み
loader = CSVLoader(
    file_path="products.csv",
    encoding="utf-8",
    csv_args={
        'delimiter': ',',  # 区切り文字
    }
)
documents = loader.load()

# 各行がDocumentに変換される
print(f"読み込んだ行数: {len(documents)}")
print(f"1行目のデータ:\n{documents[0].page_content}")
print(f"メタデータ: {documents[0].metadata}")

ヒント

CSVLoaderは各行を「列名: 値」形式の文字列に変換します。これにより、LLMが構造化データを理解しやすくなります。例: "商品名: リンゴ\n価格: 100円\n在庫: 50"

8.5 ディレクトリ一括読み込み

フォルダ内の複数ファイルを一度に読み込む方法を学びます。大量のドキュメントを効率的に処理できます。

DirectoryLoaderで複数ファイルを読み込み

directory_loader.py 
from langchain.document_loaders import DirectoryLoader
from langchain.document_loaders import TextLoader

# docsフォルダ内のすべての.txtファイルを読み込み
loader = DirectoryLoader(
    path="./docs",
    glob="**/*.txt",  # *.txtですべての.txtファイル
    loader_cls=TextLoader,  # テキストローダーを指定
    show_progress=True  # 進捗表示
)
documents = loader.load()

print(f"読み込んだファイル数: {len(documents)}")
for doc in documents:
    print(f"- {doc.metadata['source']}")

複数形式のファイルを同時に読み込み

multi_format_loader.py 
from langchain.document_loaders import DirectoryLoader
from langchain.document_loaders import PyPDFLoader, TextLoader

# PDFファイルとテキストファイルを別々に読み込み
pdf_loader = DirectoryLoader("./docs", glob="**/*.pdf", loader_cls=PyPDFLoader)
txt_loader = DirectoryLoader("./docs", glob="**/*.txt", loader_cls=TextLoader)

# 両方を結合
pdf_docs = pdf_loader.load()
txt_docs = txt_loader.load()
all_documents = pdf_docs + txt_docs

print(f"PDF文書: {len(pdf_docs)}件")
print(f"テキスト文書: {len(txt_docs)}件")
print(f"合計: {len(all_documents)}件")

実習 8-1: 複数形式のファイルを読み込んでメタデータを確認

PDF、テキスト、CSVファイルを読み込み、それぞれのメタデータ構造を確認します。

手順

  1. 1 サンプルファイル(sample.pdf、sample.txt、sample.csv)を用意する
  2. 2 各形式に対応するLoaderで読み込む
  3. 3 metadata辞書の内容を比較し、違いを理解する
exercise_8_1.py 
from langchain.document_loaders import PyPDFLoader, TextLoader, CSVLoader

# PDF読み込み
pdf_loader = PyPDFLoader("sample.pdf")
pdf_docs = pdf_loader.load()

# テキスト読み込み
txt_loader = TextLoader("sample.txt", encoding="utf-8")
txt_docs = txt_loader.load()

# CSV読み込み
csv_loader = CSVLoader("sample.csv", encoding="utf-8")
csv_docs = csv_loader.load()

# メタデータを比較
print("PDF metadata:", pdf_docs[0].metadata)
print("TXT metadata:", txt_docs[0].metadata)
print("CSV metadata:", csv_docs[0].metadata)

この章のまとめ

  • Document Loaderは、様々なデータソースをLangChainで扱えるDocument形式に変換する
  • Documentオブジェクトは、page_content(本文)とmetadata(メタデータ)で構成される
  • PyPDFLoader、WebBaseLoader、CSVLoaderなど、データ形式ごとに専用ローダーが提供されている
  • DirectoryLoaderを使うと、フォルダ内の複数ファイルを一括で読み込める
  • 100種類以上のLoaderが用意されており、Google Drive、Slack、GitHubなど幅広いデータソースに対応

理解度確認クイズ

  1. Q1. Documentオブジェクトの2つの主要な要素は何ですか?

    回答を表示

    ①page_content(本文テキスト)と、②metadata(メタデータ辞書)です。page_contentにはLLMに渡す実際の内容が、metadataにはファイル名やページ番号などの補足情報が格納されます。

  2. Q2. PyPDFLoaderの特徴は何ですか?

    回答を表示

    PyPDFLoaderは、PDFファイルをページ単位で分割します。各ページが1つのDocumentオブジェクトになり、metadata['page']にページ番号が自動的に設定されます。これにより、検索時に「どのページに情報があるか」を把握しやすくなります。

  3. Q3. DirectoryLoaderを使うメリットは何ですか?

    回答を表示

    DirectoryLoaderを使うと、フォルダ内の複数ファイルを一度のコードで読み込めます。glob="**/*.pdf"のようなパターンマッチングで特定の形式のみ読み込むことも可能で、大量のドキュメントを効率的に処理できます。
