プロンプトテンプレート - 再利用可能な指示の設計

PromptTemplateを使って、動的に変数を埋め込める再利用可能なプロンプトを作成します。テンプレート化によるプロンプト管理の効率化を習得します。

この章で学ぶこと

  • PromptTemplateの基本的な使い方を理解する
  • 変数埋め込み、部分テンプレート、複数変数の活用方法を習得する
  • ChatPromptTemplateを使った会話形式のプロンプト設計を学ぶ
  • Few-shot Promptingとテンプレートの組み合わせ技術を実践する

3.1 プロンプトテンプレートとは

PromptTemplateは、LLMに送信するプロンプトを、変数を含むテンプレートとして定義する機能です。同じ構造のプロンプトを繰り返し使用する場合に、変数部分だけを差し替えて効率的に再利用できます。

第2章でLLMを直接呼び出す方法を学びましたが、実際の開発では同じパターンのプロンプトを何度も使うことが一般的です。テンプレート化することで、コードの保守性と再利用性が大幅に向上します。

プロンプトテンプレートの利点

  • 変数を使ってプロンプトを動的に生成
  • 同じパターンのプロンプトを一箇所で管理
  • プロンプトとアプリケーションロジックの分離
  • チームでの共有・再利用が容易

3.2 基本的なPromptTemplateの使い方

まずは、単一の変数を含むシンプルなテンプレートから始めましょう。

最も基本的なテンプレート

basic_prompt_template.py 
# PromptTemplateのインポート
from langchain.prompts import PromptTemplate

# テンプレートの定義
template = "{product}の特徴を3つ教えてください。"

# PromptTemplateオブジェクトの作成
prompt = PromptTemplate(
    input_variables=["product"],  # 変数名を指定
    template=template
)

# テンプレートに値を埋め込む
formatted_prompt = prompt.format(product="スマートフォン")
print(formatted_prompt)
# 出力: スマートフォンの特徴を3つ教えてください。

重要ポイント

{変数名}の形式で変数を定義し、input_variablesで変数名のリストを指定します。format()メソッドで変数に値を代入します。

from_templateメソッドでのシンプルな記述

simple_template.py 
# input_variablesを自動推論する方法
prompt = PromptTemplate.from_template(
    "{product}の特徴を3つ教えてください。"
)

# LLMと組み合わせて使用
from langchain.llms import OpenAI

llm = OpenAI(temperature=0.7)
response = llm(prompt.format(product="ノートパソコン"))
print(response)

from_template()を使うと、テンプレート文字列から自動的に変数を検出してくれるため、コードがシンプルになります。

3.3 複数変数の活用

実際のアプリケーションでは、複数の変数を含むテンプレートを使うことが一般的です。

multiple_variables.py 
from langchain.prompts import PromptTemplate

# 複数の変数を含むテンプレート
prompt = PromptTemplate.from_template(
    """
    あなたは{role}です。
    {topic}について、{audience}向けに{length}文字程度で説明してください。
    """
)

# 複数の変数に値を代入
formatted_prompt = prompt.format(
    role="ITコンサルタント",
    topic="クラウドコンピューティング",
    audience="初心者",
    length="200"
)

print(formatted_prompt)

実習: 翻訳プロンプトを作成する

実習 3-1: 翻訳テンプレートの作成

以下の機能を持つ翻訳プロンプトを作成してください：

  • 変数: source_language, target_language, text
  • LLMに、自然で正確な翻訳を依頼するプロンプトにする
解答例を表示 
prompt = PromptTemplate.from_template(
    """
    以下のテキストを{source_language}から{target_language}に翻訳してください。
    自然で正確な表現を心がけてください。

    テキスト: {text}

    翻訳結果:
    """
)

formatted = prompt.format(
    source_language="英語",
    target_language="日本語",
    text="Machine learning is transforming industries."
)
print(formatted)

3.4 ChatPromptTemplate - 会話形式のテンプレート

ChatGPTやClaude等のチャット型LLMでは、システムメッセージ、ユーザーメッセージ、アシスタントメッセージという役割分担があります。ChatPromptTemplateを使うと、これらを構造的に管理できます。

chat_prompt_template.py 
from langchain.prompts import ChatPromptTemplate

# チャット形式のテンプレート定義
chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "あなたは{role}です。{tone}で回答してください。"),
    ("human", "{question}")
])

# メッセージの生成
messages = chat_prompt.format_messages(
    role="親切なアシスタント",
    tone="丁寧かつ分かりやすく",
    question="Pythonのリスト内包表記とは何ですか？"
)

# LLMに送信
from langchain.chat_models import ChatOpenAI

chat = ChatOpenAI(temperature=0.7)
response = chat(messages)
print(response.content)

メッセージの役割

system

LLMの役割や振る舞いを定義。トーン、制約条件などを指示

human / user

ユーザーからの質問やリクエストを表現

assistant / ai

LLMからの回答を表現（Few-shot例示等で使用）

format_messagesとformat_promptの違い

format_messages()はメッセージオブジェクトのリストを返し、Chat Modelsに直接渡せます。format_prompt()はPromptValueオブジェクトを返します。通常はformat_messagesを使用します。

3.5 Few-shot Prompting - 例示による学習

Few-shot Promptingは、LLMにタスクの例を示すことで、望ましい出力形式や振る舞いを学習させる手法です。LangChainではFewShotPromptTemplateを使って効率的に実装できます。

few_shot_prompt.py 
from langchain.prompts import FewShotPromptTemplate, PromptTemplate

# 例示データ
examples = [
    {"input": "happy", "output": "sad"},
    {"input": "tall", "output": "short"},
    {"input": "fast", "output": "slow"}
]

# 各例を表示するためのテンプレート
example_template = """
入力: {input}
出力: {output}
"""

example_prompt = PromptTemplate(
    input_variables=["input", "output"],
    template=example_template
)

# Few-shotプロンプトの作成
few_shot_prompt = FewShotPromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
    prefix="以下は英単語の反対語を答える例です。",
    suffix="入力: {input}\n出力:",
    input_variables=["input"]
)

# プロンプトの生成
print(few_shot_prompt.format(input="hot"))
# 出力:
# 以下は英単語の反対語を答える例です。
# 入力: happy
# 出力: sad
# 入力: tall
# 出力: short
# 入力: fast
# 出力: slow
# 入力: hot
# 出力:

動的な例示選択

入力に応じて最も関連性の高い例を動的に選択するExample Selectorも利用できます。

semantic_example_selector.py 
from langchain.prompts import SemanticSimilarityExampleSelector
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import FAISS

examples = [
    {"input": "犬", "output": "動物"},
    {"input": "東京", "output": "都市"},
    {"input": "Python", "output": "プログラミング言語"}
]

# セマンティック類似度ベースのセレクター
example_selector = SemanticSimilarityExampleSelector.from_examples(
    examples,
    OpenAIEmbeddings(),
    FAISS,
    k=2  # 上位2件を選択
)

# 「京都」に類似した例を選択
selected = example_selector.select_examples({"input": "京都"})
print(selected)
# 「東京」が最も類似した例として選ばれる

3.6 部分テンプレート - 変数を段階的に埋める

部分テンプレート（Partial Template）は、一部の変数を先に埋め込んでおき、残りを後で埋める機能です。共通の設定を事前に固定する場合に便利です。

partial_template.py 
from langchain.prompts import PromptTemplate

# テンプレートの定義
prompt = PromptTemplate.from_template(
    "あなたは{role}です。{task}を実行してください。"
)

# roleを先に固定（部分テンプレート化）
partial_prompt = prompt.partial(role="データアナリスト")

# taskだけを後で指定
formatted = partial_prompt.format(task="売上データの分析")
print(formatted)
# 出力: あなたはデータアナリストです。売上データの分析を実行してください。

活用シーン

複数のタスクで共通の設定（システムロール、出力形式など）を使う場合、部分テンプレートで一度設定しておくと、コードの重複を減らせます。

3.7 プロンプトテンプレート設計のベストプラクティス

推奨される設計パターン

1. 明確な指示と構造化

タスクの目的、出力形式、制約条件を明確に記述。改行や区切りを使って可読性を高める。

2. 変数名は分かりやすく

x, yよりproduct_name, user_questionのように、役割が分かる名前を使う。

3. テンプレートファイルとして管理

複雑なテンプレートは外部ファイル（.txt, .json）として管理し、コードから分離すると保守性が向上。

4. Few-shotで期待する振る舞いを明示

複雑なタスクでは、数個の例を示すことでLLMの理解精度が大幅に向上する。

実習 3-2: カスタマーサポート向けテンプレート

以下の要件を満たすChatPromptTemplateを作成してください：

  • システムメッセージ: カスタマーサポートの役割、対応方針を定義
  • 変数: company_name, product_name, customer_question
  • ユーザーメッセージ: 顧客の質問を受け取る
解答例を表示 
from langchain.prompts import ChatPromptTemplate

support_prompt = ChatPromptTemplate.from_messages([
    ("system", """
    あなたは{company_name}のカスタマーサポート担当者です。
    {product_name}に関する顧客からの質問に対して、以下の方針で対応してください：
    - 丁寧かつ分かりやすい言葉遣い
    - 具体的な解決策を提示
    - 技術用語は初心者にも理解できるよう説明
    """),
    ("human", "{customer_question}")
])

messages = support_prompt.format_messages(
    company_name="テックサポート株式会社",
    product_name="スマートホームデバイス",
    customer_question="デバイスがWi-Fiに接続できません"
)

# Chat Modelで実行
from langchain.chat_models import ChatOpenAI
chat = ChatOpenAI(temperature=0.7)
response = chat(messages)
print(response.content)

  • from_template()を使うと、テンプレートから自動的に変数を検出できる
  • ChatPromptTemplateで、system/human/assistantメッセージを構造的に管理
  • Few-shot Promptingで、例示を通じてLLMの振る舞いを制御できる

理解度確認クイズ

  1. Q1. PromptTemplateを使う主な利点を3つ挙げてください。

    回答を表示

    ①変数を使ってプロンプトを動的に生成できる、②同じパターンのプロンプトを一箇所で管理できる、③プロンプトとアプリケーションロジックを分離できる、の3つです。これにより、コードの保守性と再利用性が向上します。

  2. Q2. ChatPromptTemplateのfrom_messagesで使える主なメッセージの役割（role）を3つ挙げてください。

    回答を表示

    ①system（LLMの役割や振る舞いを定義）、②human/user（ユーザーからの質問やリクエスト）、③assistant/ai（LLMからの回答、Few-shot例示等で使用）の3つです。これらを組み合わせて、構造的な会話プロンプトを構築します。

  3. Q3. Few-shot Promptingとは何ですか？どのような場面で有効ですか？

    回答を表示

    Few-shot Promptingは、LLMにタスクの例を数個示すことで、望ましい出力形式や振る舞いを学習させる手法です。複雑なタスク、特定の出力形式が求められる場合、一貫性が重要な場合などで有効です。FewShotPromptTemplateを使えば、例示データを構造的に管理できます。
