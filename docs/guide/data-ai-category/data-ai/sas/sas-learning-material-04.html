第4章

データステップの基本

SASプログラムの中核となるデータステップの仕組みを理解し、データの作成・加工の基礎を身につけます。

この章で学ぶこと

  • データステップの役割と基本構造を理解する
  • DATAステートメントとSETステートメントの使い方を学ぶ
  • PDV（Program Data Vector）の仕組みを知る
  • データステップでデータを作成・加工する

4.1 データステップとは

データステップ（DATA Step）は、SASプログラムの中核となる処理単位で、データセットの作成・加工・変換を行います。第2章で学んだPROCステップが「データを分析・表示する」のに対し、データステップは「データを作成・加工する」役割を担います。

SASの2つのステップ

データステップ：データの作成・加工（DATA～RUN;）
PROCステップ：データの分析・表示（PROC～RUN;）

flowchart LR A[元データ] --> B[データステップ
データ加工] B --> C[加工後データ] C --> D[PROCステップ
分析・表示] D --> E[結果出力]

4.2 データステップの基本構造

データステップは、必ずDATAステートメントで始まり、RUNステートメントで終わります。この間に、データの読み込みや加工の処理を記述します。

データステップの基本構造 
/* データステップの基本形 */
DATA 出力データセット名;
    /* ここにデータ処理の命令を記述 */
    SET 入力データセット名;  /* データを読み込む */
    /* データ加工の処理 */
RUN;

各部分の役割

1
DATAステートメント

データステップの開始を宣言し、作成するデータセットの名前を指定します。

2
SETステートメント

既存のデータセットを読み込みます（オプション：新規作成時は不要）。

3
処理ステートメント

変数の計算、条件分岐、データ加工などの処理を記述します。

4
RUNステートメント

データステップの終了を宣言し、処理を実行します。

4.3 実習 4-1: 最初のデータステップ

データステップでデータセットをコピーする

既存のデータセット（SASHELP.CLASS）を読み込み、新しいデータセットを作成します。

データセットのコピー 
/* SASHELP.CLASSをMY_CLASSとしてコピー */
DATA my_class;
    SET sashelp.class;  /* 元データを読み込む */
RUN;

/* 作成したデータセットを表示 */
PROC PRINT DATA=my_class;
RUN;

プログラムの解説

  • DATA my_class;：MY_CLASSという名前のデータセットを作成
  • SET sashelp.class;：SASHELP.CLASSのデータを1行ずつ読み込む
  • RUN;：データステップの処理を実行

動作確認

  • ログウィンドウに「NOTE: データセットWORK.MY_CLASSが作成されました」と表示される
  • 結果ウィンドウにクラスの学生データ（19人分）が表示される

4.4 データステップの動作フロー（PDV）

データステップは、PDV（Program Data Vector）という仮想的なメモリ領域を使って、1行ずつデータを処理します。これは「作業テーブル」のようなもので、ここでデータを加工してから出力データセットに書き込みます。

flowchart TD A[データステップ開始] --> B[PDV初期化] B --> C[1行目を読み込み] C --> D[PDVにデータをセット] D --> E[処理ステートメント実行] E --> F[PDVの内容を出力データセットに書き込み] F --> G{次の行はあるか?} G -->|Yes| C G -->|No| H[データステップ終了]

PDVのイメージ

PDVは、Excelでいう「1行の作業用セル」のようなものです。元データの1行をコピーして、そこで計算や加工を行い、完成したら次のシートに貼り付けて、また次の行を処理する…という流れです。

4.5 実習 4-2: データステップでデータを加工する

身長をセンチメートルからメートルに変換する

SASHELP.CLASSの身長データ（インチ）をセンチメートルに変換し、新しい変数として追加します。

データ加工の例 
/* 身長をインチからセンチメートルに変換 */
DATA class_metric;
    SET sashelp.class;
    /* 1インチ = 2.54cm で変換 */
    height_cm = height * 2.54;
RUN;

/* 結果を表示（変数を絞って表示） */
PROC PRINT DATA=class_metric;
    VAR name height height_cm;
RUN;

プログラムの解説

  • height_cm = height * 2.54;：新しい変数HEIGHT_CMを作成し、計算結果を代入
  • 元のデータ（NAME、HEIGHT等）はそのまま保持される
  • VAR文で表示する変数を限定することで、見やすくなる

確認すべきポイント

  • HEIGHT_CM列が追加されている
  • HEIGHT（例：62.8インチ）がHEIGHT_CM（159.512cm）に正しく変換されている

4.6 RUNステートメントの役割

RUNステートメントは、データステップやPROCステップの終了を明示的に宣言し、実際に処理を実行させる重要な役割を持っています。

RUNステートメントの役割

  • 1
    ステップの終了を宣言

    DATAやPROCで始まったステップがどこで終わるかを明示します

  • 2
    処理の実行

    RUNステートメントが実行された時点で、その前に書かれた処理が実際に実行されます

  • 3
    次のステップとの区切り

    複数のステップを連続して書く場合、RUNで明確に区切ることで可読性が高まります

RUNステートメントを忘れると…

RUNステートメントがない場合、次のDATAやPROCステートメントが現れるまで処理が実行されず、思わぬエラーや予期しない動作の原因になります。必ず各ステップの最後にRUN;を記述する習慣をつけましょう。

4.7 データセット名の命名規則

データステップで作成するデータセット名には、いくつかのルールがあります。

ルール 説明 例
長さ 1～32文字 customer_data
使用可能文字 英字、数字、アンダースコア（_） sales_2024
先頭文字 英字またはアンダースコア _temp, mydata
大文字小文字 区別されない（慣例的に小文字を使用） MyData = mydata
禁止文字 スペース、ハイフン、特殊記号 my-data ✗

命名のベストプラクティス

データセット名は、内容がわかりやすい名前をつけましょう。例：customer_analysis、sales_summaryなど。
単語の区切りにはアンダースコア（_）を使うことが一般的です。

理解度確認クイズ

  1. Q1. データステップとPROCステップの主な違いは何ですか？

    回答を表示

    データステップ：データの作成・加工を行う（DATA～RUN;）
    PROCステップ：データの分析・表示を行う（PROC～RUN;）
    データステップでデータを整えてから、PROCステップで分析するという流れが基本です。

  2. Q2. PDV（Program Data Vector）とは何ですか？

    回答を表示

    PDVは、データステップが1行ずつデータを処理する際に使用する仮想的なメモリ領域です。Excelの「作業用の1行」のようなもので、ここでデータを加工してから出力データセットに書き込みます。

  3. Q3. RUNステートメントを忘れるとどうなりますか？

    回答を表示

    RUNステートメントがないと、次のDATAやPROCステートメントが現れるまで処理が実行されません。これにより、エラーや予期しない動作が発生する可能性があります。必ず各ステップの最後にRUN;を記述しましょう。

第4章のまとめ

  • データステップは、SASプログラムの中核でデータの作成・加工を担当する
  • 基本構造は「DATA データセット名; 処理; RUN;」
  • SETステートメントで既存データを読み込み、加工できる
  • PDV（Program Data Vector）という仮想メモリで1行ずつ処理する
  • RUNステートメントでステップの終了を宣言し、処理を実行する
