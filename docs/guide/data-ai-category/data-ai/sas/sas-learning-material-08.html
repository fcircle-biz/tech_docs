第8章

データの結合と変換

複数のデータセットの結合（マージ、セット、SQL結合）、データの転置と形式変換を学びます。

この章で学ぶこと

  • 複数データセットの結合方法（MERGE文）を理解する
  • SET文による縦方向のデータ結合を習得する
  • PROC SQLによる柔軟なデータ結合を学ぶ
  • PROC TRANSPOSEによるデータの転置を理解する

8.1 データ結合の基本概念

実務では、複数のデータセットを組み合わせて分析することが一般的です。SASでは、データの結合方法として主に3つのアプローチがあります。

flowchart LR A[複数のデータセット] --> B{結合の方向} B -->|横方向| C[MERGE文
共通キーで結合] B -->|縦方向| D[SET文
行を追加] B -->|柔軟な結合| E[PROC SQL
SQLの結合構文]

結合の種類

横方向の結合は「マージ」、縦方向の結合は「アペンド（追加）」と呼ばれます。ExcelのVLOOKUP関数やデータの追加と同じ概念です。

8.2 MERGE文による横方向結合

MERGE文は、共通のキー変数を基準に複数のデータセットを横方向に結合します。まず、事前にBY文で指定するキー変数でソートしておく必要があります。

1対1結合の例

社員マスタと給与データを社員IDで結合する例です。

SAS 
/* 社員マスタデータを作成 */
DATA employee;
    INPUT employee_id name $ 9-20 dept $;
    DATALINES;
101 山田太郎 営業部
102 佐藤花子 総務部
103 鈴木一郎 開発部
;
RUN;

/* 給与データを作成 */
DATA salary;
    INPUT employee_id salary;
    DATALINES;
101 350000
102 320000
103 380000
;
RUN;

/* 両方のデータセットを社員IDでソート */
PROC SORT DATA=employee;
    BY employee_id;
RUN;

PROC SORT DATA=salary;
    BY employee_id;
RUN;

/* MERGE文で結合 */
DATA emp_salary;
    MERGE employee salary;
    BY employee_id;
RUN;

/* 結果を表示 */
PROC PRINT DATA=emp_salary;
RUN;

動作確認

上記のプログラムを実行すると、以下のような結果が得られます。

employee_id  name       dept     salary
-------------------------------------------
    101      山田太郎   営業部   350000
    102      佐藤花子   総務部   320000
    103      鈴木一郎   開発部   380000

重要な注意点

MERGE文を使う前に、必ずPROC SORTでキー変数を基準にソートしてください。ソートしないとエラーまたは予期しない結果になります。

8.3 1対多結合

1対多の関係（例：1人の顧客が複数の注文を持つ）でも、MERGE文を使用できます。

SAS 
/* 顧客マスタデータ */
DATA customer;
    INPUT customer_id name $ 12-25;
    DATALINES;
1001 田中商店
1002 山本物産
;
RUN;

/* 注文データ（1人の顧客が複数注文） */
DATA orders;
    INPUT customer_id order_id amount;
    DATALINES;
1001 5001 15000
1001 5002 23000
1002 5003 42000
;
RUN;

/* ソート */
PROC SORT DATA=customer; BY customer_id; RUN;
PROC SORT DATA=orders; BY customer_id; RUN;

/* 結合 */
DATA customer_orders;
    MERGE customer orders;
    BY customer_id;
RUN;

8.4 SET文による縦方向結合

SET文は、複数のデータセットを縦方向に追加（アペンド）します。同じ構造のデータを連結する場合に使用します。

SAS 
/* 東京店の売上データ */
DATA sales_tokyo;
    INPUT store $ product $ sales;
    DATALINES;
東京 商品A 100000
東京 商品B 150000
;
RUN;

/* 大阪店の売上データ */
DATA sales_osaka;
    INPUT store $ product $ sales;
    DATALINES;
大阪 商品A 120000
大阪 商品B 180000
;
RUN;

/* SET文で縦に結合 */
DATA all_sales;
    SET sales_tokyo sales_osaka;
RUN;

/* 結果を確認 */
PROC PRINT DATA=all_sales;
RUN;

SET文のポイント

SET文は単純に行を追加します。ソート不要で、データセットの順序通りに連結されます。月次データの統合などに便利です。

8.5 PROC SQLによる結合

PROC SQLを使うと、SQLの標準構文でデータを結合できます。事前のソートが不要で、柔軟な結合が可能です。

INNER JOIN（内部結合）

両方のデータセットに存在するキーのみを結合します。

SAS 
/* PROC SQLでINNER JOIN */
PROC SQL;
    CREATE TABLE result AS
    SELECT e.employee_id,
           e.name,
           e.dept,
           s.salary
    FROM employee AS e
    INNER JOIN salary AS s
    ON e.employee_id = s.employee_id;
QUIT;

LEFT JOIN（左外部結合）

左側のデータセットの全レコードを残し、右側と一致するものを結合します。

SAS 
/* LEFT JOINの例 */
PROC SQL;
    CREATE TABLE result AS
    SELECT e.employee_id,
           e.name,
           s.salary
    FROM employee AS e
    LEFT JOIN salary AS s
    ON e.employee_id = s.employee_id;
QUIT;

SQLの結合タイプ

1
INNER JOIN

両方のテーブルに存在するレコードのみ

2
LEFT JOIN

左テーブルの全レコード + 右テーブルの一致分

3
RIGHT JOIN

右テーブルの全レコード + 左テーブルの一致分

4
FULL JOIN

両テーブルの全レコード

8.6 PROC TRANSPOSEによるデータの転置

PROC TRANSPOSEは、データの行と列を入れ替える（転置する）プロシジャです。縦型データを横型に、横型データを縦型に変換できます。

SAS 
/* 縦型の売上データ */
DATA sales_long;
    INPUT month $ product $ sales;
    DATALINES;
1月 商品A 100
1月 商品B 150
2月 商品A 120
2月 商品B 180
;
RUN;

/* 転置（商品を列にする） */
PROC TRANSPOSE DATA=sales_long OUT=sales_wide;
    BY month;
    ID product;
    VAR sales;
RUN;

/* 結果を確認 */
PROC PRINT DATA=sales_wide;
RUN;

PROC TRANSPOSEの主なオプション

  • BY グループ化する変数を指定
  • ID 列名になる変数を指定
  • VAR 値として使う変数を指定
  • PREFIX= 新しい列名の接頭辞を指定

実習 8-1: データ結合の実践

以下のデータを作成し、様々な方法で結合してみましょう。

課題1: MERGE文による結合

商品マスタと在庫データを商品コードで結合し、商品名と在庫数を表示してください。

SAS 
/* 商品マスタ */
DATA product;
    INPUT product_id name $ 12-25 price;
    DATALINES;
P001 ノートPC 98000
P002 マウス 2500
P003 キーボード 5800
;
RUN;

/* 在庫データ */
DATA stock;
    INPUT product_id stock_qty;
    DATALINES;
P001 15
P002 42
P003 28
;
RUN;

課題2: PROC SQLによる結合

上記データをPROC SQLのINNER JOINを使って結合し、在庫金額（price × stock_qty）を計算してください。

理解度確認クイズ

  1. Q1. MERGE文を使う前に必須の処理は何ですか？

    回答を表示

    PROC SORTによるソートです。MERGE文を使う場合、BY文で指定するキー変数で両方のデータセットを事前にソートしておく必要があります。

  2. Q2. 縦方向にデータを追加する場合、どの構文を使いますか？

    回答を表示

    SET文を使います。SET文は複数のデータセットを縦方向に追加（アペンド）します。月次データの統合などに便利です。

  3. Q3. PROC SQLのLEFT JOINはどのような結合ですか？

    回答を表示

    左外部結合です。左側のテーブルの全レコードを残し、右側のテーブルで一致するレコードがあれば結合します。一致しない場合は欠損値となります。

第8章のまとめ

  • MERGE文は共通キーで横方向にデータを結合する（事前ソート必須）
  • SET文は複数データセットを縦方向に追加する（ソート不要）
  • PROC SQLは柔軟な結合が可能（INNER/LEFT/RIGHT/FULL JOIN）
  • PROC TRANSPOSEは行と列を入れ替える転置処理を行う
  • 実務では用途に応じて最適な結合方法を選択することが重要
