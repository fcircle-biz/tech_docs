ループ処理と配列

DOループによる繰り返し処理、配列を使った効率的なデータ処理を理解します。大量データの一括処理方法を習得します。

この章で学ぶこと

  • DOループを使った繰り返し処理の基本を理解する
  • 様々なDOループのバリエーション（DO WHILE、DO UNTIL）を習得する
  • 配列（ARRAY）を使った効率的なデータ処理方法を学ぶ
  • 配列とループを組み合わせた大量データの一括処理を実践する

7.1 ループ処理とは

ループ処理とは、同じ処理を繰り返し実行するプログラミング技法です。SASでは、大量のデータや複数の変数に対して同じ処理を行う際に、ループを使うことで効率的にコードを記述できます。

なぜループが必要？

例えば、10個の変数に同じ処理を行う場合、ループを使わないと同じコードを10回書く必要があります。ループを使えば、1回のコード記述で済み、メンテナンスも容易になります。

flowchart TD A[開始] --> B{繰り返し条件} B -->|条件満たす| C[処理実行] C --> D[カウンター更新] D --> B B -->|条件満たさない| E[終了]

7.2 DOループの基本

SASのDOループは、指定した回数だけ処理を繰り返します。基本的な構文は以下の通りです。 

/* 基本的なDOループの構文 */
DATA example;
    /* 変数iを1から5まで1ずつ増やしながら繰り返す */
    DO i = 1 TO 5;
        PUT "現在の値は " i;  /* ログに出力 */
    END;
RUN;

構文の読み方

DO 変数 = 開始値 TO 終了値; は「変数を開始値から終了値まで1ずつ増やしながら繰り返す」という意味です。終了のEND;を忘れずに記述しましょう。

実習 7-1: DOループで合計を計算

1から10までの合計をDOループを使って計算してみましょう。

/* 1から10までの合計を計算 */
DATA sum_example;
    total = 0;  /* 合計を保存する変数を初期化 */

    DO i = 1 TO 10;
        total = total + i;  /* 現在の値を加算 */
    END;

    PUT "1から10までの合計は " total;
RUN;

実行結果

ログに「1から10までの合計は 55」と表示されます。

7.3 DOループのバリエーション

SASには、基本的なDOループ以外にも様々なバリエーションがあります。

主なDOループのバリエーション

1
増分指定: DO i = 開始 TO 終了 BY 増分;

増分を自由に指定できます。例: DO i = 1 TO 10 BY 2; (1, 3, 5, 7, 9)

2
DO WHILE(条件);

条件が真の間、繰り返します（条件を最初にチェック）。

3
DO UNTIL(条件);

条件が真になるまで繰り返します（条件を最後にチェック、最低1回は実行）。

/* DO WHILEの例 */
DATA while_example;
    i = 1;
    total = 0;

    /* totalが50未満の間繰り返す */
    DO WHILE(total < 50);
        total = total + i;
        i = i + 1;
    END;

    PUT "合計が50を超えたとき i = " i " total = " total;
RUN;
flowchart LR A[DO WHILE] --> B[条件チェック→処理] C[DO UNTIL] --> D[処理→条件チェック] B --> E[最低0回実行] D --> F[最低1回実行]

7.4 配列（ARRAY）の基本

配列（ARRAY）は、複数の変数をまとめて扱うためのSASの機能です。同じ種類の複数の変数（例: score1, score2, score3...）に対して同じ処理を行う際に非常に便利です。 

/* 配列の基本構文 */
DATA array_example;
    /* 4科目の点数を変数として定義 */
    score1 = 85;
    score2 = 90;
    score3 = 78;
    score4 = 92;

    /* 配列を定義（scores配列に4つの変数をまとめる） */
    ARRAY scores{4} score1-score4;
RUN;

配列の構文解説

ARRAY 配列名{要素数} 変数リスト;
- 配列名: 任意の名前
- 要素数: 含まれる変数の数
- 変数リスト: score1-score4 のように範囲指定可能

flowchart LR A[配列 scores] --> B["scores{1} = score1"] A --> C["scores{2} = score2"] A --> D["scores{3} = score3"] A --> E["scores{4} = score4"]

7.5 配列とループの組み合わせ

配列とループを組み合わせることで、大量のデータを効率的に処理できます。これがSASの真の力を発揮する場面です。

実習 7-2: 配列とループで平均点を計算

4科目の点数から平均点を計算し、50点未満の科目を0点に変換してみましょう。

/* 配列とループを組み合わせた処理 */
DATA student_scores;
    /* 学生の点数データを作成 */
    INPUT name $ score1 score2 score3 score4;
    DATALINES;
太郎 85 45 90 78
花子 92 88 76 95
次郎 68 72 55 80
;

DATA processed_scores;
    SET student_scores;

    /* 配列を定義 */
    ARRAY scores{4} score1-score4;

    /* 合計と平均を計算 */
    total = 0;
    DO i = 1 TO 4;
        total = total + scores{i};
        /* 50点未満は0点に変換 */
        IF scores{i} < 50 THEN scores{i} = 0;
    END;

    average = total / 4;  /* 平均点を計算 */

    /* 不要な変数を削除 */
    DROP i;
RUN;

/* 結果を表示 */
PROC PRINT DATA=processed_scores;
RUN;

配列のメリット

配列を使わない場合、score1, score2, score3, score4それぞれに同じIF文を4回書く必要があります。配列とループを使えば、コードがシンプルになり、変数が増えても簡単に対応できます。

7.6 実践演習: 月次データの集計

12ヶ月分の売上データから、年間合計と月平均を計算してみましょう。 

/* 月次売上データの集計 */
DATA monthly_sales;
    /* 各月の売上を設定 */
    jan=1200; feb=1350; mar=1480; apr=1220;
    may=1560; jun=1680; jul=1750; aug=1820;
    sep=1650; oct=1590; nov=1720; dec=1980;

    /* 配列を定義（12ヶ月分） */
    ARRAY sales{12} jan feb mar apr may jun
                    jul aug sep oct nov dec;

    /* 年間合計を計算 */
    yearly_total = 0;
    DO month = 1 TO 12;
        yearly_total = yearly_total + sales{month};
    END;

    /* 月平均を計算 */
    monthly_avg = yearly_total / 12;

    /* 最高売上月を特定 */
    max_sales = 0;
    max_month = 0;
    DO month = 1 TO 12;
        IF sales{month} > max_sales THEN DO;
            max_sales = sales{month};
            max_month = month;
        END;
    END;

    DROP month;
RUN;

動作確認

以下のコードを追加して、結果を確認しましょう。

PROC PRINT DATA=monthly_sales;
    VAR yearly_total monthly_avg max_month max_sales;
RUN;

理解度確認クイズ

  1. Q1. DO WHILEとDO UNTILの違いは何ですか？

    回答を表示

    DO WHILEは条件を先にチェックするため、最初から条件が偽の場合は1回も実行されません。
    DO UNTILは処理を実行してから条件をチェックするため、最低1回は必ず実行されます。

  2. Q2. 配列を使うメリットは何ですか？

    回答を表示

    複数の変数に対して同じ処理を行う際に、コードを簡潔に記述できることが最大のメリットです。また、変数の数が増えてもループの範囲を変更するだけで対応でき、保守性が向上します。

  3. Q3. ARRAY scores{5} score1-score5; の意味を説明してください。

    回答を表示

    scoresという名前の配列を定義し、5つの要素（score1, score2, score3, score4, score5）をまとめています。scores{1}でscore1、scores{2}でscore2...というようにアクセスできます。

  • 配列とループを組み合わせることで、大量データの一括処理が簡単に実現できる
  • 実務では月次データの集計など、繰り返し処理が必須の場面で活用される
