第12章

SASマクロの基礎

マクロ変数の使い方、簡単なマクロプログラムの作成方法を学び、プログラムの再利用性を高めます。

この章で学ぶこと

  • SASマクロの基本概念と利点を理解する
  • マクロ変数の定義・参照方法を習得する
  • 簡単なマクロプログラムを作成できるようになる
  • マクロによるコードの再利用方法を学ぶ

12.1 SASマクロとは何か

SASマクロは、プログラムコードの一部を「テンプレート化」し、繰り返し利用できるようにする機能です。同じ処理を何度も書く手間を省き、コードの保守性と再利用性を大幅に向上させます。

マクロの役割

マクロは「プログラムを生成するプログラム」です。実行時に、マクロコードが通常のSASコードに展開されてから実行されます。

flowchart LR A[マクロコード] --> B[マクロプロセッサ] B --> C[通常のSASコード] C --> D[SASコンパイラ] D --> E[実行結果]

マクロを使う3つのメリット

再利用性

一度作ったマクロは何度でも使い回せます。同じ処理を毎回書く必要がありません。

保守性

修正が必要な時、マクロ定義部分だけを変更すれば、全ての利用箇所に反映されます。

生産性

長いコードを何度も書く時間が不要になり、開発効率が大幅に向上します。

12.2 マクロ変数の基本

マクロ変数は、値を保存して後で参照できる「変数」です。通常のSAS変数とは異なり、データステップの外でも使えます。

マクロ変数 vs 通常の変数

比較項目 マクロ変数 通常の変数
定義記号 %LET データステップ内で定義
参照記号 &変数名 変数名のみ
使用可能範囲 プログラム全体 データステップ内のみ
データ型 文字列のみ 文字/数値

重要ポイント

マクロ変数は、プログラムのどこからでもアクセスできる「グローバル変数」です。これが通常の変数との最大の違いです。

12.3 マクロ変数の定義と参照

マクロ変数の定義（%LET文）

マクロ変数は%LET文で定義します。基本構文は以下の通りです。

基本構文 
%LET 変数名 = 値;
例: マクロ変数の定義 
/* マクロ変数の定義例 */
%LET year = 2024;
%LET company_name = ABC商事;
%LET file_path = /home/data/sales.csv;

マクロ変数の参照（&記号）

定義したマクロ変数は、&変数名で参照します。

例: マクロ変数の参照 
/* マクロ変数を定義 */
%LET year = 2024;

/* データステップで使用 */
DATA sales_&year;
    SET original_sales;
    /* 2024年のデータのみ抽出 */
    WHERE sales_year = &year;
RUN;

実行時の動作

上記のコードは、実行時に DATA sales_2024; と WHERE sales_year = 2024; に展開されます。

実習 12-1: マクロ変数で年度別レポート作成

マクロ変数を使って、年度を切り替えるだけで異なる年のレポートを作成できるようにします。

手順

  1. 1 マクロ変数で年度を定義します
  2. 2 テストデータを作成し、マクロ変数を使ってフィルタリングします
  3. 3 年度別の売上レポートを出力します
macro_variable_example.sas 
/* マクロ変数で年度を定義 */
%LET target_year = 2024;

/* テストデータ作成 */
DATA sales;
    INPUT year product $ revenue;
    DATALINES;
2023 商品A 1000
2024 商品A 1200
2024 商品B 1500
2023 商品B 1300
;
RUN;

/* 指定年度のデータのみ抽出 */
DATA sales_&target_year;
    SET sales;
    WHERE year = &target_year;
RUN;

/* レポート出力 */
PROC PRINT DATA=sales_&target_year;
    TITLE "&target_year年度 売上レポート";
RUN;

試してみよう

%LET target_year = 2023; に変更すると、2023年のレポートが作成されます。

12.4 マクロプログラムの基礎

マクロ変数だけでなく、マクロプログラム（マクロ定義）を使うと、複雑な処理を一つの名前で呼び出せるようになります。

マクロプログラムの構文

基本構文 
%MACRO マクロ名(パラメータ1, パラメータ2, ...);
    /* ここに実行したい処理を記述 */
%MEND マクロ名;

マクロの実行（呼び出し）

マクロの呼び出し 
%マクロ名(引数1, 引数2, ...);
flowchart LR A[マクロ定義
%MACRO...%MEND] --> B[マクロ呼び出し
%マクロ名] B --> C[マクロ展開] C --> D[SASコード実行]

実習 12-2: 簡単なマクロプログラムを作成

商品別の売上合計を表示するマクロを作成します。

simple_macro.sas 
/* テストデータ作成 */
DATA sales;
    INPUT product $ revenue;
    DATALINES;
商品A 1200
商品B 1500
商品A 800
商品B 1100
商品A 900
;
RUN;

/* マクロ定義: 商品別売上集計 */
%MACRO product_summary(prod_name);
    PROC MEANS DATA=sales SUM MEAN;
        VAR revenue;
        WHERE product = "&prod_name";
        TITLE "&prod_name の売上サマリー";
    RUN;
%MEND product_summary;

/* マクロを実行 */
%product_summary(商品A);
%product_summary(商品B);

動作確認

マクロを実行すると、商品A、商品Bそれぞれの売上合計・平均が自動的に表示されます。

期待される結果 

商品A の売上サマリー
合計: 2900、平均: 966.67

商品B の売上サマリー
合計: 2600、平均: 1300

12.5 パラメータ付きマクロの活用

マクロにパラメータを渡すことで、さらに柔軟な処理が可能になります。パラメータを使うと、1つのマクロで様々なデータや条件に対応できます。

複数パラメータの使用例

例: 年度・商品フィルタリングマクロ 
/* マクロ定義: 年度と商品で絞り込みレポート */
%MACRO sales_report(year, product);
    PROC PRINT DATA=sales;
        WHERE sales_year = &year AND product_name = "&product";
        TITLE "&year年度 &product 売上レポート";
    RUN;
%MEND sales_report;

/* マクロ実行例 */
%sales_report(2024, 商品A);
%sales_report(2023, 商品B);

重要ポイント

文字列パラメータを参照する際は、ダブルクォーテーションで囲む必要があります（例: "&product"）。

12.6 マクロのデバッグ技術

マクロが期待通りに動かない時、以下のオプションを使ってデバッグできます。

便利なデバッグオプション

オプション 説明 使用例
MPRINT マクロが展開されたSASコードを表示 OPTIONS MPRINT;
MLOGIC マクロの実行フローを表示 OPTIONS MLOGIC;
SYMBOLGEN マクロ変数の値を表示 OPTIONS SYMBOLGEN;
デバッグオプションの使用例 
/* デバッグオプションを有効化 */
OPTIONS MPRINT MLOGIC SYMBOLGEN;

%LET year = 2024;
%product_summary(商品A);

/* デバッグオプションを無効化 */
OPTIONS NOMPRINT NOMLOGIC NOSYMBOLGEN;

トラブルシューティング

マクロ変数が展開されない（&year がそのまま表示される）

原因: シングルクォーテーション内でマクロ変数を参照している

解決方法: ダブルクォーテーションを使用してください。WHERE year = "&year";

マクロ名のスペルミスによるエラー

原因: マクロ定義名と呼び出し時の名前が一致していない

解決方法: %MACRO と %MEND、呼び出し時のマクロ名がすべて一致しているか確認してください。

パラメータの数が合わない

原因: マクロ定義のパラメータ数と、呼び出し時の引数の数が一致していない

解決方法: マクロ定義と呼び出しのパラメータ数を合わせてください。足りない場合はデフォルト値を設定することもできます。

理解度確認クイズ

  1. Q1. マクロ変数を定義する際に使用するステートメントは何ですか？

    回答を表示

    %LETステートメントを使用します。例: %LET year = 2024;

  2. Q2. マクロ変数を参照する際に使用する記号は何ですか？

    回答を表示

    &（アンパサンド）記号を使用します。例: &year

  3. Q3. マクロプログラムの定義を終了する際に使用するステートメントは何ですか？

    回答を表示

    %MENDステートメントを使用します。%MACRO マクロ名; ... %MEND マクロ名; という形式で使います。

  4. Q4. マクロが生成したSASコードを確認するためのオプションは何ですか？

    回答を表示

    MPRINTオプションを使用します。OPTIONS MPRINT; と指定すると、マクロが展開されたSASコードがログに表示されます。

第12章のまとめ

  • SASマクロは、コードの再利用性・保守性・生産性を高める重要な機能
  • マクロ変数は%LETで定義、&記号で参照する
  • マクロプログラムは%MACRO...%MENDで定義、%マクロ名で呼び出す
  • パラメータを使うことで、柔軟で汎用的なマクロを作成できる
  • MPRINT、MLOGIC、SYMBOLGENオプションでマクロをデバッグできる
