レポート作成の基本

PROC PRINTやPROC REPORTを使った表形式レポートの作成、ODS（Output Delivery System）によるExcel/PDF出力を学習します。

この章で学ぶこと

  • PROC PRINTを使った基本的なレポート作成
  • PROC REPORTによる高度な表形式レポートの作成
  • ODS（Output Delivery System）によるExcel/PDF出力
  • レポートの見た目のカスタマイズ方法

12.1 レポート作成の重要性

データ分析の結果を他者に伝えるには、見やすく分かりやすいレポートが欠かせません。SASには、データを表形式で出力したり、Excel/PDF形式で保存したりする強力なレポート機能が備わっています。

レポート作成の目的

分析結果を関係者に共有し、意思決定をサポートするために、視覚的に分かりやすく整理された情報提供が重要です。

flowchart LR A[生データ] --> B[SASプログラム
データ処理・集計] B --> C[レポート作成
PROC PRINT/REPORT] C --> D[ODS出力
Excel/PDF] D --> E[意思決定者へ共有]

12.2 PROC PRINTの基本

PROC PRINTは、データセットの内容を表形式で表示する最もシンプルなプロシジャです。データの確認やちょっとしたレポート作成に便利です。

実習 12-1: PROC PRINTで基本レポート作成

売上データをPROC PRINTで表示してみましょう。

手順

  1. 1 サンプルデータセットを作成します
  2. 2 PROC PRINTで全データを表示します
  3. 3 オプションを追加して表示をカスタマイズします
sales_report.sas 
/* サンプル売上データの作成 */
DATA sales;
    INPUT name $ product $ amount;
    DATALINES;
田中 ノートPC 120000
佐藤 マウス 3000
鈴木 キーボード 8000
山田 ノートPC 130000
中村 モニター 25000
;
RUN;

/* 基本的なPROC PRINT */
PROC PRINT DATA=sales;
    TITLE '売上レポート（基本）';
RUN;

/* オプション付きPROC PRINT */
PROC PRINT DATA=sales NOOBS; /* 行番号を非表示 */
    VAR name product amount; /* 表示列を指定 */
    TITLE '売上レポート（カスタマイズ版）';
RUN;

PROC PRINTの主なオプション

オプション 説明 使用例
NOOBS 行番号を非表示 PROC PRINT NOOBS;
VAR 表示する変数を指定 VAR name amount;
SUM 数値列の合計を表示 SUM amount;
WHERE 条件に合うデータのみ表示 WHERE amount > 10000;

12.3 PROC REPORTによる高度なレポート

PROC REPORTは、PROC PRINTよりも高機能なレポート作成プロシジャです。グループ化、集計、小計、書式設定などが柔軟にできます。

flowchart LR A[PROC PRINT
シンプル表示] -->|より高度に| B[PROC REPORT
グループ化・集計] B --> C[見やすいレポート
小計・合計付き]

実習 12-2: PROC REPORTでグループ集計レポート

商品カテゴリ別に売上を集計し、小計・合計を表示するレポートを作成します。

sales_report_advanced.sas 
/* カテゴリ別売上データの作成 */
DATA sales2;
    INPUT category $ product $ amount;
    DATALINES;
PC ノートPC 120000
周辺機器 マウス 3000
周辺機器 キーボード 8000
PC デスクトップPC 95000
周辺機器 モニター 25000
PC ノートPC 130000
;
RUN;

/* PROC REPORTでカテゴリ別集計 */
PROC REPORT DATA=sales2;
    COLUMN category product amount; /* 列を定義 */
    DEFINE category / GROUP 'カテゴリ'; /* グループ化 */
    DEFINE product / DISPLAY '商品名';
    DEFINE amount / SUM '金額' FORMAT=comma12.; /* 合計計算 */
    TITLE 'カテゴリ別売上レポート';
    RBREAK AFTER / SUMMARIZE; /* 全体合計を追加 */
RUN;

ポイント

DEFINEステートメントで各列の役割を定義します。GROUP（グループ化）、DISPLAY（表示のみ）、SUM（合計）などが指定できます。

PROC REPORTのDEFINE用法

  • 1
    GROUP（グループ化）

    同じ値をまとめて表示し、グループごとに集計を行います

  • 2
    DISPLAY（表示）

    データをそのまま表示します（集計はしない）

  • 3
    SUM（合計）

    数値列の合計を計算します

  • 4
    MEAN（平均）

    数値列の平均を計算します

12.4 ODS（Output Delivery System）によるExcel/PDF出力

ODS（Output Delivery System）は、SASの出力を様々な形式（Excel、PDF、HTML、RTFなど）で保存できる強力な機能です。レポートを他者に配布する際に非常に便利です。

ODSの基本構文

ODS EXCEL FILE='ファイルパス'; で開始し、ODS EXCEL CLOSE; で終了します。その間に実行したプロシジャの結果が指定形式で出力されます。

flowchart LR A[ODS EXCEL FILE=...] --> B[PROC PRINT/REPORT
実行] B --> C[ODS EXCEL CLOSE] C --> D[Excelファイル
自動生成]

実習 12-3: Excel形式でレポートを出力

作成したレポートをExcelファイルに保存してみましょう。

output_excel.sas 
/* Excel出力の開始 */
ODS EXCEL FILE='/home/sasdemo/sales_report.xlsx'
    OPTIONS(SHEET_NAME='売上データ');

/* レポート作成 */
PROC PRINT DATA=sales2 NOOBS;
    VAR category product amount;
    TITLE '売上レポート';
RUN;

/* Excel出力の終了 */
ODS EXCEL CLOSE;

ヒント

SHEET_NAMEオプションでシート名を指定できます。複数のプロシジャを実行すれば、複数シートのExcelブックも作成できます。

実習 12-4: PDF形式でレポートを出力

同じレポートをPDFファイルに保存してみましょう。

output_pdf.sas 
/* PDF出力の開始 */
ODS PDF FILE='/home/sasdemo/sales_report.pdf';

/* レポート作成 */
PROC REPORT DATA=sales2;
    COLUMN category product amount;
    DEFINE category / GROUP 'カテゴリ';
    DEFINE product / DISPLAY '商品名';
    DEFINE amount / SUM '金額' FORMAT=comma12.;
    TITLE 'カテゴリ別売上レポート';
    RBREAK AFTER / SUMMARIZE;
RUN;

/* PDF出力の終了 */
ODS PDF CLOSE;

主なODS出力形式

形式 構文 用途
Excel ODS EXCEL FILE='...'; データの加工・再利用
PDF ODS PDF FILE='...'; 印刷・配布用レポート
HTML ODS HTML FILE='...'; Web公開用レポート
RTF ODS RTF FILE='...'; Word文書として編集

12.5 レポートの見た目のカスタマイズ

レポートの見栄えを良くするために、フォーマット（書式）やスタイルを設定できます。

実習 12-5: 数値フォーマットとスタイル設定

カンマ区切りや通貨フォーマット、色付けを行ってみましょう。

styled_report.sas 
/* スタイル付きレポート */
ODS PDF FILE='/home/sasdemo/styled_report.pdf'
    STYLE=journal; /* スタイルテンプレートを指定 */

PROC REPORT DATA=sales2;
    COLUMN category product amount;
    DEFINE category / GROUP 'カテゴリ';
    DEFINE product / DISPLAY '商品名';
    DEFINE amount / SUM '金額（円）' FORMAT=comma12.0; /* カンマ区切り */
    TITLE '売上レポート（スタイル付き）';
    RBREAK AFTER / SUMMARIZE;
RUN;

ODS PDF CLOSE;

重要ポイント

STYLE=オプションで、様々な見た目のテンプレートを適用できます（journal、minimal、sasweb、printer等）。

よく使う数値フォーマット

  • 1
    comma12.（カンマ区切り）

    12345 → 12,345

  • 2
    dollar12.2（ドル表示）

    12345.67 → $12,345.67

  • 3
    percent8.2（パーセント表示）

    0.456 → 45.60%

  • 4
    date9.（日付表示）

    SAS日付値 → 01JAN2025

理解度確認クイズ

  1. Q1. PROC PRINTで行番号を非表示にするオプションは何ですか？

    回答を表示

    NOOBSオプションです。PROC PRINT DATA=dataset NOOBS; のように指定します。OBS（Observation Number、行番号）を表示しない、という意味です。

  2. Q2. PROC REPORTでグループ化を行うDEFINEステートメントのキーワードは何ですか？

    回答を表示

    GROUPです。DEFINE category / GROUP; のように指定すると、categoryの値ごとにグループ化され、集計が行われます。

  3. Q3. レポートをExcelファイルに出力するODSコマンドの開始と終了の構文を答えてください。

    回答を表示

    開始: ODS EXCEL FILE='ファイルパス';
    終了: ODS EXCEL CLOSE;
    この2つのコマンドの間に実行したプロシジャの結果がExcelファイルに出力されます。

第12章のまとめ

  • PROC PRINTは、シンプルな表形式レポート作成に便利（NOOBS、VAR、SUM等のオプション）
  • PROC REPORTは、グループ化・集計・小計など高度なレポート作成が可能
  • ODS（Output Delivery System）で、Excel/PDF/HTML/RTF形式に出力できる
  • 数値フォーマット（comma、dollar、percentなど）でレポートを見やすくできる
  • STYLE=オプションで、レポート全体のデザインテンプレートを適用できる
