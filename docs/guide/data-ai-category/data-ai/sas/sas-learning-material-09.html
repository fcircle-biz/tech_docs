PROC SQLによるデータ操作

SQLを使ったデータ抽出・結合・集計の方法を学びます。データステップとの使い分けや、複雑なクエリの書き方を習得します。

この章で学ぶこと

  • PROC SQLの基本構文とデータステップとの違いを理解する
  • SELECT文によるデータ抽出と条件指定を習得する
  • JOIN句を使った複数テーブルの結合を学ぶ
  • GROUP BYによる集計とHAVING句による絞り込みを理解する

9.1 PROC SQLとは

PROC SQLは、SAS内でSQL（Structured Query Language）を使用するプロシジャです。データベースで標準的なSQLの構文を使って、データの抽出・結合・集計ができます。

flowchart LR A[データステップ] -->|逐次処理| B[データセット] C[PROC SQL] -->|SQL構文| B D[選択のポイント] --> E[複雑な結合はPROC SQL] D --> F[行単位処理はデータステップ]

PROC SQLの特徴

PROC SQLは事前のソートが不要で、複数テーブルの結合が1文で記述できます。RDBMSの経験がある方には馴染みやすい構文です。

データステップとPROC SQLの比較

項目 データステップ PROC SQL
結合前のソート 必須（BY変数でソート） 不要
処理スタイル 逐次（行単位） セット指向（まとめて処理）
複数テーブル結合 複数回のMERGE 1文で記述可能
集計処理 PROC MEANS等と組み合わせ GROUP BY句で簡単

9.2 PROC SQLの基本構文

PROC SQLはPROC SQL;で始まり、QUIT;で終了します。

SAS 
/* PROC SQLの基本構文 */
PROC SQL;
    SELECT 列1, 列2, ...
    FROM テーブル名
    WHERE 条件
    ORDER BY ソート列;
QUIT;

実習 9-1: 基本的なSELECT文

サンプルデータを作成し、PROC SQLでデータを抽出してみましょう。

basic_select.sas 
/* サンプルデータの作成 */
DATA employees;
    INPUT emp_id name $ 9-20 dept $ salary;
    DATALINES;
101 山田太郎 営業部 350000
102 佐藤花子 総務部 320000
103 鈴木一郎 開発部 380000
104 田中健二 営業部 360000
105 高橋美咲 開発部 400000
;
RUN;

/* PROC SQLでデータ抽出 */
PROC SQL;
    SELECT emp_id, name, salary
    FROM employees
    WHERE salary >= 360000
    ORDER BY salary DESC;
QUIT;

実行結果

給与36万円以上の社員が、給与の高い順に表示されます（高橋、田中、鈴木の3名）。

9.3 CREATE TABLEとデータ操作

CREATE TABLEを使うと、SELECT結果を新しいデータセットとして保存できます。

SAS 
/* CREATE TABLEでデータセット作成 */
PROC SQL;
    CREATE TABLE high_salary AS
    SELECT emp_id, name, dept, salary
    FROM employees
    WHERE salary >= 360000;
QUIT;

/* 結果を確認 */
PROC PRINT DATA=high_salary; RUN;

計算列の追加

SELECT句で計算式を使い、新しい列を作成できます。

SAS 
/* 計算列と別名（AS）の使用 */
PROC SQL;
    SELECT emp_id,
           name,
           salary,
           salary * 12 AS annual_salary, /* 年収を計算 */
           salary * 0.1 AS bonus          /* ボーナスを計算 */
    FROM employees;
QUIT;

9.4 JOIN句による複数テーブルの結合

PROC SQLでは、JOIN句を使って複数のテーブルを結合できます。事前のソートは不要です。

graph TD A[INNER JOIN] --> B[両テーブルに存在するキーのみ] C[LEFT JOIN] --> D[左テーブル全件 + 右の一致分] E[RIGHT JOIN] --> F[右テーブル全件 + 左の一致分] G[FULL JOIN] --> H[両テーブル全件]

実習 9-2: テーブル結合

社員データと部門データを結合してみましょう。

join_example.sas 
/* 部門マスタの作成 */
DATA departments;
    INPUT dept $ dept_name $ 11-25 location $;
    DATALINES;
営業部 セールス部門 東京
総務部 管理部門 東京
開発部 技術部門 大阪
;
RUN;

/* INNER JOINで結合 */
PROC SQL;
    SELECT e.emp_id,
           e.name,
           e.dept,
           d.dept_name,
           d.location,
           e.salary
    FROM employees AS e
    INNER JOIN departments AS d
    ON e.dept = d.dept;
QUIT;

ポイント

ASでテーブルに別名（e、d）を付けると、列名の参照が簡潔になります。ON句で結合条件を指定します。

LEFT JOINの例

SAS 
/* LEFT JOIN：左テーブル（employees）の全件を残す */
PROC SQL;
    SELECT e.emp_id,
           e.name,
           e.dept,
           d.dept_name
    FROM employees AS e
    LEFT JOIN departments AS d
    ON e.dept = d.dept;
QUIT;

注意

LEFT JOINで右テーブルに一致するデータがない場合、その列は欠損値（NULL）になります。

9.5 集計関数とGROUP BY句

PROC SQLでは、集計関数を使ってデータを集計できます。GROUP BY句と組み合わせると、グループごとの集計が可能です。

主な集計関数

関数 説明 使用例
COUNT(*) 行数をカウント COUNT(*) AS cnt
SUM() 合計を計算 SUM(salary) AS total
AVG() 平均を計算 AVG(salary) AS avg_sal
MAX() / MIN() 最大値 / 最小値 MAX(salary) AS max_sal

実習 9-3: GROUP BYによる集計

部門別の給与集計を行ってみましょう。

group_by_example.sas 
/* 部門別の給与統計 */
PROC SQL;
    SELECT dept,
           COUNT(*) AS emp_count,
           SUM(salary) AS total_salary,
           AVG(salary) AS avg_salary FORMAT=comma10.,
           MAX(salary) AS max_salary,
           MIN(salary) AS min_salary
    FROM employees
    GROUP BY dept
    ORDER BY avg_salary DESC;
QUIT;

期待される結果 

dept       emp_count  total_salary  avg_salary    max_salary  min_salary
--------   ---------  -----------   ----------    ----------  ----------
開発部         2         780000       390,000       400000      380000
営業部         2         710000       355,000       360000      350000
総務部         1         320000       320,000       320000      320000

HAVING句による集計結果のフィルタリング

WHERE句は集計前のフィルタリング、HAVING句は集計後のフィルタリングに使います。

SAS 
/* HAVING句：平均給与35万円以上の部門のみ表示 */
PROC SQL;
    SELECT dept,
           COUNT(*) AS emp_count,
           AVG(salary) AS avg_salary FORMAT=comma10.
    FROM employees
    GROUP BY dept
    HAVING AVG(salary) >= 350000
    ORDER BY avg_salary DESC;
QUIT;

9.6 サブクエリの活用

サブクエリ（副問合せ）は、クエリの中にクエリを入れ子にする手法です。複雑な条件指定に便利です。

SAS 
/* 平均給与以上の社員を抽出 */
PROC SQL;
    SELECT emp_id, name, dept, salary
    FROM employees
    WHERE salary >= (SELECT AVG(salary) FROM employees);
QUIT;

/* 特定部門の社員のみ抽出（IN句） */
PROC SQL;
    SELECT emp_id, name, dept, salary
    FROM employees
    WHERE dept IN (SELECT dept FROM departments WHERE location = '東京');
QUIT;

サブクエリのポイント

サブクエリは括弧で囲み、外側のクエリより先に実行されます。単一値を返すサブクエリは比較演算子（=, >=等）で、複数値を返すサブクエリはIN句で使います。

理解度確認クイズ

  1. Q1. PROC SQLを終了するステートメントは何ですか？

    回答を表示

    QUIT;です。PROC SQLはQUIT;で終了し、RUN;ではありません。

  2. Q2. LEFT JOINとINNER JOINの違いは何ですか？

    回答を表示

    INNER JOINは両テーブルに存在するキーのみを結合します。LEFT JOINは左テーブルの全レコードを残し、右テーブルで一致するものを結合します（一致しない場合は欠損値）。

  3. Q3. 集計結果に対してフィルタリングする句は何ですか？

    回答を表示

    HAVING句です。WHERE句は集計前のフィルタリング、HAVING句は集計後のフィルタリングに使います。

第9章のまとめ

  • PROC SQLはSQLの標準構文でデータ操作ができる（事前ソート不要）
  • SELECT文でデータ抽出、CREATE TABLE ASで結果をデータセットに保存
  • JOIN句（INNER/LEFT/RIGHT/FULL）で複数テーブルを結合
  • GROUP BY句と集計関数（COUNT, SUM, AVG等）でグループ集計
  • HAVING句は集計後のフィルタリング、サブクエリで複雑な条件指定
