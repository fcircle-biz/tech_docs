第6章

条件処理とデータの抽出

IF-THEN文による条件分岐、WHERE句によるデータ抽出、サブセット作成の方法を学習します。

この章で学ぶこと

  • IF-THEN文による条件分岐の基本を理解する
  • WHERE句を使ったデータの抽出方法を習得する
  • 複合条件（AND、OR）の使い方を理解する
  • サブセット（部分データセット）の作成方法を学ぶ

6.1 条件処理とは

データ分析では、「特定の条件を満たすデータだけを処理したい」という場面が頻繁にあります。例えば、「売上が100万円以上の顧客だけを抽出する」「年齢が30歳未満の場合は『若年層』と分類する」といった処理です。

SASでは、このような条件に応じた処理をIF-THEN文やWHERE句で実現できます。

flowchart TD A[全データ] --> B{条件を満たす?} B -- はい --> C[条件に合致した処理] B -- いいえ --> D[別の処理またはスキップ] C --> E[結果データ] D --> E

6.2 IF-THEN文の基本

IF-THEN文は、「もし～ならば、～する」という条件分岐を実現する構文です。

基本構文

/* IF-THEN文の基本構文 */
IF 条件 THEN 処理;

/* 例 */
IF age >= 20 THEN category = '成人';

実習 6-1: IF-THEN文の基本

年齢に応じた年齢区分を作成してみましょう。

コード

/* 実習6-1: IF-THEN文でカテゴリ分類 */
DATA age_category;
    INPUT name $ age;
    /* 年齢が20歳未満なら「未成年」 */
    IF age < 20 THEN category = '未成年';
    /* 年齢が60歳以上なら「シニア」 */
    IF age >= 60 THEN category = 'シニア';
    DATALINES;
田中 18
佐藤 35
鈴木 65
高橋 25
;
RUN;

/* データを表示 */
PROC PRINT DATA=age_category;
RUN;

注意点

上記のコードでは、20歳以上60歳未満の人はcategoryが未定義（欠損値）になります。この問題はIF-THEN-ELSEで解決できます。

6.3 IF-THEN-ELSE文

ELSEを追加することで、条件に合致しない場合の処理を指定できます。

IF-THEN-ELSE構文

/* IF-THEN-ELSE構文 */
IF 条件 THEN 処理1;
ELSE 処理2;

/* 例 */
IF score >= 60 THEN result = '合格';
ELSE result = '不合格';

実習 6-2: IF-THEN-ELSE文

試験の合否判定を行うプログラムを作成します。

/* 実習6-2: 合否判定プログラム */
DATA exam_result;
    INPUT name $ score;
    /* 60点以上なら合格、未満なら不合格 */
    IF score >= 60 THEN result = '合格';
    ELSE result = '不合格';
    DATALINES;
山田 75
伊藤 58
渡辺 82
中村 45
小林 61
;
RUN;

PROC PRINT DATA=exam_result;
RUN;

ELSE IFによる複数条件

3段階以上の条件分岐にはELSE IFを使います。 

/* 複数条件の例 */
IF score >= 80 THEN grade = 'A';
ELSE IF score >= 60 THEN grade = 'B';
ELSE grade = 'C';

6.4 WHERE句によるデータ抽出

WHERE句は、条件に合致するデータ行だけを読み込む機能です。IF-THEN文との違いは、WHERE句はデータステップやプロシジャで使用でき、処理速度が速いことです。

機能 IF-THEN WHERE
使用場所 DATAステップのみ DATAステップ、PROC両方
処理タイミング 全行読み込み後に判定 読み込み前に絞り込み
処理速度 やや遅い 速い
新変数の参照 可能 不可

実習 6-3: WHERE句でデータ抽出

売上が100万円以上の顧客データだけを抽出します。

/* 実習6-3: WHERE句による抽出 */
DATA sales;
    INPUT customer_id name $ sales;
    DATALINES;
101 A商事 1200
102 B物産 850
103 C興業 950
104 D産業 1500
105 E商会 750
;
RUN;

/* 売上100万円以上の顧客を抽出 */
DATA high_sales;
    SET sales;
    WHERE sales >= 1000;
RUN;

PROC PRINT DATA=high_sales;
    TITLE '売上100万円以上の顧客';
RUN;

6.5 複合条件（AND、OR）

複数の条件を組み合わせる場合は、AND（かつ）やOR（または）を使います。

AND演算子（両方の条件を満たす）

/* AND演算子の例 */
WHERE age >= 20 AND sales >= 1000;

→ 年齢が20歳以上かつ売上が1000以上のデータのみ

OR演算子（どちらかの条件を満たす）

/* OR演算子の例 */
WHERE category = 'A' OR category = 'B';

→ カテゴリがAまたはBのデータのみ

実習 6-4: 複合条件の使用

年齢が30歳以上かつ売上が800以上の顧客を抽出します。

/* 実習6-4: 複合条件 */
DATA customers;
    INPUT id name $ age sales;
    DATALINES;
1 田中 35 1200
2 佐藤 28 950
3 鈴木 42 600
4 高橋 31 850
5 伊藤 25 1100
;
RUN;

/* 年齢30歳以上かつ売上800以上 */
DATA target_customers;
    SET customers;
    WHERE age >= 30 AND sales >= 800;
RUN;

PROC PRINT DATA=target_customers;
    TITLE '対象顧客（30歳以上かつ売上800以上）';
RUN;

6.6 IN演算子

IN演算子は、複数の値のいずれかに該当するかを判定する便利な機能です。

IN演算子の使い方

/* IN演算子の基本構文 */
WHERE 変数 IN (値1, 値2, 値3);

/* 例1: 文字変数 */
WHERE city IN ('東京', '大阪', '名古屋');

/* 例2: 数値変数 */
WHERE age IN (20, 30, 40);

/* 冗長なOR演算子と同じ */
WHERE city = '東京' OR city = '大阪' OR city = '名古屋';

IN演算子の利点

OR演算子を複数回書くより、IN演算子を使う方がコードが読みやすく、記述ミスも減ります。

6.7 サブセッティング（部分データセット作成）

サブセッティングとは、大きなデータセットから条件に合致する行だけを抽出して、新しいデータセットを作成することです。WHERE句やIF-THEN文がよく使われます。

flowchart LR A[元データセット
全1000件] --> B[WHERE句で抽出] B --> C[サブセット
条件に合致する200件]

実習 6-5: サブセット作成

元データから地域ごとのサブセットを作成します。

/* 実習6-5: 地域別サブセット */
DATA all_customers;
    INPUT id name $ region $ sales;
    DATALINES;
1 田中 東京 1200
2 佐藤 大阪 950
3 鈴木 東京 800
4 高橋 福岡 1100
5 伊藤 大阪 750
6 渡辺 東京 1300
;
RUN;

/* 東京の顧客のみ抽出 */
DATA tokyo_customers;
    SET all_customers;
    WHERE region = '東京';
RUN;

/* 大阪の顧客のみ抽出 */
DATA osaka_customers;
    SET all_customers;
    WHERE region = '大阪';
RUN;

/* 結果を表示 */
PROC PRINT DATA=tokyo_customers;
    TITLE '東京の顧客';
RUN;

PROC PRINT DATA=osaka_customers;
    TITLE '大阪の顧客';
RUN;

6.8 よく使う比較演算子

条件処理では、以下の比較演算子がよく使われます。

演算子 意味 例
= 等しい age = 30
^= または ~= 等しくない status ^= 'OK'
> より大きい sales > 1000
< より小さい age < 20
>= 以上 score >= 60
<= 以下 age <= 65
IN リスト内にある city IN ('東京', '大阪')

理解度確認クイズ

  1. Q1. IF-THEN文とWHERE句の違いを説明してください。

    回答を表示

    IF-THEN文：DATAステップで使用。全データを読み込んでから条件判定。新変数の参照が可能。
    WHERE句：DATAステップとPROCの両方で使用可能。データ読み込み前に絞り込むため処理が高速。既存変数のみ参照可能。

  2. Q2. 次のコードは何を実行しますか？
    WHERE age >= 20 AND age <= 30;

    回答を表示

    年齢（age）が20歳以上かつ30歳以下のデータのみを抽出します。両方の条件を満たす必要があります（AND条件）。

  3. Q3. IN演算子を使うメリットは何ですか？

    回答を表示

    複数のOR条件を簡潔に記述できるため、コードが読みやすく、記述ミスが減ります。例えば、
    WHERE city IN ('東京', '大阪', '名古屋');
    は、OR演算子を2回繰り返すより簡潔です。

第6章のまとめ

  • IF-THEN文で条件に応じた処理を実行できる（ELSE、ELSE IFで複数分岐も可能）
  • WHERE句は高速なデータ抽出が可能で、DATAステップとPROCの両方で使える
  • AND演算子で複数条件の「かつ」、OR演算子で「または」を表現できる
  • IN演算子を使えば、複数の値のいずれかに該当するかを簡潔に判定できる
  • サブセッティングで大きなデータから必要な部分だけを抽出し、新しいデータセットを作成できる
