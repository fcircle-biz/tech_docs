AIに「グループ分け」させてみよう

顧客セグメンテーションや商品カテゴリ分けなど、「似たものを集める」クラスタリングの考え方を学びます。正解がない状態からパターンを発見するAIの能力を理解します。

この章で学ぶこと

  • クラスタリング（グループ分け）の基本概念を理解する
  • 分類とクラスタリングの違いを理解する
  • ビジネスでのクラスタリング活用例を知る
  • クラスタリングの代表的な手法を概念レベルで理解する

5.1 クラスタリングとは何か？

クラスタリング（Clustering）とは、似た特徴を持つデータを自動的にグループにまとめる技術のことです。「分類」と似ていますが、大きな違いがあります。

分類は「正解のラベル」があらかじめ分かっている状態で学習しますが、クラスタリングは正解がない状態からデータの中のパターンを見つけ出します。

図5.1: 分類とクラスタリングの違い

キーポイント

クラスタリングは「教師なし学習」と呼ばれます。人間が正解を教えなくても、AIがデータの中から似ているもの同士を自動的にグループにまとめます。

5.2 身近なクラスタリングの例

実は、私たちの身の回りでもクラスタリングは活用されています。以下のような場面で使われています。

顧客セグメンテーション

ECサイトで、購買履歴から顧客を「頻繁に買う人」「たまに買う人」「最近買わない人」などにグループ分けします。それぞれのグループに合わせた販促施策を行います。

商品のカテゴリ分け

大量の商品を、価格や評価、購入層などの特徴から似たグループにまとめます。新商品がどのカテゴリに属するかを自動判定できます。

ニュース記事の自動分類

大量のニュース記事を、内容の類似度から「政治」「経済」「スポーツ」などのグループに自動的に振り分けます。人間が一つひとつラベル付けする必要がありません。

異常検知

クレジットカードの不正利用を検知します。通常の利用パターンとは大きく異なる取引を「異常なグループ」として検出し、アラートを出します。

5.3 クラスタリングの仕組み

クラスタリングは、データの「類似度」を計算して、似ているものをグループにまとめます。最も基本的な方法であるK-means法を例に説明します。

図5.2: K-means法のステップ

わかりやすい例え

教室で友達グループを作る場面を想像してください。最初は適当に3つのグループを作り、話が合う人同士で調整していくと、自然と似た趣味や性格の人が集まります。K-means法も同じように、何度も調整を繰り返して最適なグループ分けを見つけます。

5.4 グループ数の決め方

クラスタリングで難しいのは、「いくつのグループに分けるべきか？」という問題です。グループ数が少なすぎると大雑把すぎ、多すぎると細かすぎて意味がなくなります。

図5.3: グループ数による違いの例

最適なグループ数を見つけるには、以下のような方法があります：

1. ビジネス目的から考える

顧客を3段階（VIP、通常、休眠）に分けて対応を変えたいなら、グループ数は3が適切です。

2. エルボー法を使う

グループ数を変えながら試し、「グループ内のバラつき」が急に減らなくなるポイントを見つけます（肘のように曲がったところが最適）。

3. 複数のパターンを試す

3グループ、4グループ、5グループなど複数試して、ビジネス的に意味のあるグループ分けができているか確認します。

5.5 ビジネスでの活用シナリオ

実際のビジネスで、クラスタリングがどのように活用されているか見てみましょう。

ケーススタディ: オンラインショップの顧客分析

課題

10,000人の顧客がいるオンラインショップで、効果的なマーケティングを行いたい。顧客全員に同じメールを送っても反応が悪い。

使用するデータ

  • 購入回数
  • 購入金額の合計
  • 最終購入日からの経過日数
  • 購入した商品カテゴリ

クラスタリングの結果

4つのグループに分類されました：

  • グループA（VIP顧客 - 15%）： 頻繁に高額購入、最近も購入している
  • グループB（優良顧客 - 25%）： 定期的に購入、中程度の金額
  • グループC（休眠顧客 - 35%）： 過去に購入したが、最近は購入なし
  • グループD（新規・低頻度 - 25%）： 購入回数が少ない

対策

  • グループA： 新商品の先行案内、限定特典
  • グループB： ポイント倍増キャンペーン
  • グループC： 再購入を促すクーポン配布
  • グループD： 初心者向けガイド、お試し商品の案内

5.6 クラスタリングの注意点

クラスタリングを活用する際には、以下の点に注意が必要です。

注意点1: グループの解釈は人間が行う

AIは自動的にグループを作りますが、「このグループはVIP顧客だ」といった意味付けは人間が行う必要があります。データを見て、各グループの特徴を理解しましょう。

注意点2: データの前処理が重要

「購入回数」と「購入金額」のように、スケールが大きく異なるデータを扱う場合、標準化（0～1の範囲に変換）などの前処理が必要です。そうしないと、大きな数値の影響が強くなりすぎます。

注意点3: 結果は毎回少し変わる

K-means法は最初の中心点をランダムに配置するため、実行するたびに少し異なる結果になることがあります。複数回実行して、安定した結果を選びましょう。

理解度確認クイズ

  1. Q1. クラスタリングと分類の違いは何ですか？

    回答を表示

    分類は「正解のラベル」があらかじめ分かっている状態で学習しますが、クラスタリングは正解がない状態からデータの中のパターンを見つけ出します。分類は「教師あり学習」、クラスタリングは「教師なし学習」と呼ばれます。

  2. Q2. クラスタリングがビジネスで使われる例を2つ挙げてください。

    回答を表示

    例：顧客セグメンテーション（購買行動でグループ分け）、商品のカテゴリ分け、ニュース記事の自動分類、異常検知（クレジットカード不正利用など）など（2つ以上挙げられればOK）

  3. Q3. K-means法の基本的な手順を説明してください。

    回答を表示

    1. グループ数を決める、2. ランダムに中心点を配置、3. 各データを最も近い中心点に割り当て、4. 各グループの中心を再計算、5. 中心点が変わらなくなるまで3～4を繰り返す、という手順です。

  4. Q4. クラスタリングを行う際の注意点を1つ挙げてください。

    回答を表示

    例：グループの解釈は人間が行う必要がある、データの前処理（標準化など）が重要、実行するたびに結果が少し変わることがある、など（1つ以上挙げられればOK）

第5章のまとめ

  • クラスタリングは、正解がない状態から似たデータを自動的にグループにまとめる技術です
  • 顧客セグメンテーション、商品カテゴリ分け、異常検知など、様々なビジネス場面で活用されています
  • K-means法は最も基本的なクラスタリング手法で、データの類似度を計算してグループ分けします
  • グループ数の決定やグループの解釈には人間の判断が必要です
  • データの前処理を適切に行うことが、良い結果を得るために重要です
