第1章：JavaとWebの基礎理解

Spring Frameworkは、Javaの基礎知識とWebの基本的な仕組みに基づいて構築されています。この章では、すでに履修済みの内容を簡潔に復習し、Spring Frameworkを学ぶための基盤を強化します。

1.1 Javaの基本文法

Javaは静的型付けのオブジェクト指向プログラミング言語で、「Write Once, Run Anywhere（一度書けば、どこでも実行できる）」という設計思想を持っています。Spring Frameworkを効果的に使うには、以下のJavaの基本文法をしっかり理解している必要があります。

変数と基本データ型

Javaの変数は型宣言が必要であり、基本データ型（プリミティブ型）とオブジェクト型があります。

// 基本データ型（プリミティブ型）の例 int number = 10; // 整数型 double price = 29.99; // 倍精度浮動小数点型 boolean isActive = true; // 真偽値型 char grade = 'A'; // 文字型 // オブジェクト型の例 String name = "Spring Framework"; // 文字列オブジェクト Integer count = 42; // 整数のラッパークラス Date today = new Date(); // 日付オブジェクト

条件分岐

Javaの条件分岐には、if-else文と switch文があります。

// if-else文の例 int score = 85; if (score >= 90) { System.out.println("優"); } else if (score >= 80) { System.out.println("良"); } else if (score >= 70) { System.out.println("可"); } else { System.out.println("不可"); } // switch文の例（Java 14以降では新しいswitch式も使用可能） String day = "Monday"; switch (day) { case "Monday": System.out.println("月曜日"); break; case "Tuesday": System.out.println("火曜日"); break; // その他のケース default: System.out.println("その他の曜日"); }

ループ

Javaには主に3種類のループ構造があります：for, while, do-while。

// for文の例 for (int i = 0; i < 5; i++) { System.out.println("インデックス: " + i); } // 拡張for文（for-each） String[] frameworks = {"Spring", "Hibernate", "Struts", "JSF"}; for (String framework : frameworks) { System.out.println(framework); } // while文の例 int count = 0; while (count < 3) { System.out.println("Count: " + count); count++; } // do-while文の例 int x = 0; do { System.out.println("値: " + x); x++; } while (x < 3);

クラスとメソッド

Javaはクラスベースのオブジェクト指向言語です。クラスはオブジェクトの設計図であり、メソッドは動作を定義します。

// クラス定義の例 public class User { // フィールド（インスタンス変数） private String username; private String email; private boolean active; // コンストラクタ public User(String username, String email) { this.username = username; this.email = email; this.active = true; } // メソッド public void deactivate() { this.active = false; System.out.println(username + "のアカウントを無効化しました"); } // ゲッターとセッター public String getUsername() { return username; } public void setUsername(String username) { this.username = username; } // その他のゲッター・セッター } // クラスの使用例 User user1 = new User("yamada", "yamada@example.com"); System.out.println(user1.getUsername()); // "yamada"が出力される user1.deactivate(); // "yamadaのアカウントを無効化しました"が出力される
Spring Frameworkとの関連性

Spring Frameworkはクラスとオブジェクトを中心にしたフレームワークです。アノテーションを多用し、依存性の注入（DI）を通じてクラス間の結合度を下げています。基本的なJavaの文法を理解していることで、Springの複雑な機能も理解しやすくなります。

1.2 オブジェクト指向の概念

オブジェクト指向プログラミング（OOP）はソフトウェア設計の主要なパラダイムの一つで、Spring Frameworkもこの原則に基づいて設計されています。OOPの主要な概念を復習しましょう。

カプセル化（Encapsulation）

カプセル化は、クラス内の実装詳細を隠蔽し、外部からのアクセスを制限する概念です。これにより、内部の変更がクラスの利用者に影響を与えにくくなります。

// カプセル化の例 public class BankAccount { // privateフィールドで直接アクセスを制限 private double balance; private String accountNumber; private String accountHolder; // コンストラクタ public BankAccount(String accountNumber, String accountHolder) { this.accountNumber = accountNumber; this.accountHolder = accountHolder; this.balance = 0.0; } // publicメソッドを通じた制御されたアクセス public void deposit(double amount) { if (amount > 0) { this.balance += amount; System.out.println(amount + "円を入金しました。残高: " + this.balance + "円"); } else { System.out.println("無効な入金額です"); } } public void withdraw(double amount) { if (amount > 0 && amount <= this.balance) { this.balance -= amount; System.out.println(amount + "円を出金しました。残高: " + this.balance + "円"); } else { System.out.println("無効な出金額または残高不足です"); } } // 残高の読み取り専用アクセス public double getBalance() { return this.balance; } // その他のゲッターメソッド public String getAccountNumber() { return this.accountNumber; } public String getAccountHolder() { return this.accountHolder; } }

継承（Inheritance）

継承は、既存のクラス（親クラス）をベースに新しいクラス（子クラス）を作成する仕組みです。子クラスは親クラスの特性を継承し、独自の機能を追加できます。

// 親クラス public class Vehicle { protected String brand; protected String model; protected int year; public Vehicle(String brand, String model, int year) { this.brand = brand; this.model = model; this.year = year; } public void start() { System.out.println("エンジンを始動します"); } public void stop() { System.out.println("エンジンを停止します"); } // ゲッター・セッター（省略） } // 子クラス public class Car extends Vehicle { private int doors; private boolean convertible; public Car(String brand, String model, int year, int doors, boolean convertible) { super(brand, model, year); // 親クラスのコンストラクタを呼び出し this.doors = doors; this.convertible = convertible; } // 親クラスのメソッドをオーバーライド @Override public void start() { System.out.println("キーでエンジンを始動します"); super.start(); // 親クラスの実装も呼び出し } // 子クラス独自のメソッド public void openTrunk() { System.out.println("トランクを開けます"); } // ゲッター・セッター（省略） }

ポリモーフィズム（Polymorphism）

ポリモーフィズム（多態性）は、同じインターフェースを使って異なる型のオブジェクトを扱う能力です。これによりコードの柔軟性と拡張性が向上します。

// インターフェース定義 public interface Shape { double calculateArea(); void draw(); } // インターフェースを実装したクラス1 public class Circle implements Shape { private double radius; public Circle(double radius) { this.radius = radius; } @Override public double calculateArea() { return Math.PI * radius * radius; } @Override public void draw() { System.out.println("円を描画します"); } } // インターフェースを実装したクラス2 public class Rectangle implements Shape { private double width; private double height; public Rectangle(double width, double height) { this.width = width; this.height = height; } @Override public double calculateArea() { return width * height; } @Override public void draw() { System.out.println("長方形を描画します"); } } // ポリモーフィズムの活用 public class Main { public static void main(String[] args) { // 異なる実装を同じインターフェースで扱う Shape circle = new Circle(5.0); Shape rectangle = new Rectangle(4.0, 6.0); // 同じメソッド呼び出しが異なる動作をする System.out.println("円の面積: " + circle.calculateArea()); System.out.println("長方形の面積: " + rectangle.calculateArea()); circle.draw(); rectangle.draw(); // 配列での取り扱いも容易 Shape[] shapes = {circle, rectangle, new Circle(3.0)}; for (Shape shape : shapes) { shape.draw(); System.out.println("面積: " + shape.calculateArea()); } } }

Spring Frameworkとの関連性

Spring Frameworkは、OOPの原則に深く根差しています：

  • カプセル化：Spring Beanはカプセル化されたオブジェクトで、DIコンテナによって管理されます
  • 継承：Springの多くのクラスは継承を活用し、@Override注釈で親クラスのメソッドをカスタマイズします
  • ポリモーフィズム：インターフェースベースの設計が推奨され、様々な実装を柔軟に切り替えられます
  • 抽象化：Spring Frameworkはインターフェースや抽象クラスを多用し、実装の詳細を隠蔽します

1.3 HTTPの基礎

HTTP（Hypertext Transfer Protocol）はWebのバックボーンとなるプロトコルで、クライアントとサーバー間の通信を定義します。Spring MVCなどのWebフレームワークを理解するには、HTTPの基本を把握しておく必要があります。

HTTPリクエストとレスポンス

HTTPはリクエスト・レスポンス型のプロトコルです。クライアント（通常はWebブラウザ）がサーバーにリクエストを送信し、サーバーがレスポンスを返します。

図1.2: HTTPの基本的な通信モデル

HTTPメソッド

HTTPリクエストには、サーバーが実行すべき操作を示すメソッドがあります。主なHTTPメソッドは以下のとおりです：

HTTPメソッド 説明 主な用途 特徴
GET リソースの取得 データの読み取り、検索 URLにパラメータを含む、キャッシュ可能、冪等性あり
POST リソースの作成 フォーム送信、データ作成 リクエストボディにデータを含む、キャッシュ不可、冪等性なし
PUT リソースの更新/置換 データの更新 全体を置換、冪等性あり
DELETE リソースの削除 データの削除 冪等性あり
PATCH リソースの部分更新 データの部分的な更新 部分的な更新、冪等性は実装依存
冪等性（べき等性）とは

冪等性とは、同じリクエストを複数回実行しても結果が変わらない性質のことです。GET、PUT、DELETEは冪等性がありますが、POSTは冪等性がなく、何度も実行すると結果が変わる可能性があります。

HTTPステータスコード

HTTPレスポンスには、リクエストの結果を示すステータスコードが含まれます。主なステータスコードの分類は以下のとおりです：

  • 1xx（情報）: リクエストを受け取り、処理を継続中
  • 2xx（成功）: リクエストが正常に処理された
    • 200 OK: リクエスト成功
    • 201 Created: リソース作成成功
    • 204 No Content: 成功したが返すコンテンツなし
  • 3xx（リダイレクト）: リクエスト完了のために追加のアクションが必要
    • 301 Moved Permanently: 恒久的リダイレクト
    • 302 Found: 一時的リダイレクト
    • 304 Not Modified: キャッシュを使用
  • 4xx（クライアントエラー）: リクエストに問題がある
    • 400 Bad Request: リクエスト構文エラー
    • 401 Unauthorized: 認証が必要
    • 403 Forbidden: アクセス権限なし
    • 404 Not Found: リソースが存在しない
  • 5xx（サーバーエラー）: サーバー側で問題が発生
    • 500 Internal Server Error: サーバー内部エラー
    • 502 Bad Gateway: ゲートウェイエラー
    • 503 Service Unavailable: サービス利用不可

URLの構造

URL（Uniform Resource Locator）はウェブ上のリソースを特定するための住所です。URLの構造は以下のとおりです：

https://www.example.com:8080/products/category/electronics?sort=price&order=asc#description |-----プロトコル-----|-ドメイン名-|ポート|------------パス------------|---------クエリパラメータ---------|フラグメント|
  • プロトコル: http:// や https:// など
  • ドメイン名: サーバーの場所（IPアドレスでも可）
  • ポート: サーバー上のサービスを特定（省略可、デフォルトはhttp=80, https=443）
  • パス: サーバー上のリソースの場所
  • クエリパラメータ: ?で始まり、&で区切られた追加パラメータ
  • フラグメント: #で始まり、ページ内の特定位置を指定
Spring Frameworkとの関連性

Spring MVCでは、HTTPメソッドやURLパスを利用して、適切なコントローラーメソッドにリクエストをルーティングします。例えば：

@RestController @RequestMapping("/api/products") public class ProductController { @GetMapping("/{id}") public ResponseEntity<Product> getProduct(@PathVariable Long id) { // 商品取得処理 } @PostMapping public ResponseEntity<Product> createProduct(@RequestBody ProductDto productDto) { // 商品作成処理 } @PutMapping("/{id}") public ResponseEntity<Product> updateProduct(@PathVariable Long id, @RequestBody ProductDto productDto) { // 商品更新処理 } @DeleteMapping("/{id}") public ResponseEntity<Void> deleteProduct(@PathVariable Long id) { // 商品削除処理 } }

このコードでは、HTTPメソッド（GET、POST、PUT、DELETE）とURLパス（/api/products/{id}など）を使って、異なる操作を実行しています。

1.4 MVCモデルの基本

MVC（Model-View-Controller）は、ユーザーインターフェースを持つアプリケーションを構築するための設計パターンで、Spring MVCの基盤となっています。MVCの主要なコンポーネントとその役割を理解しましょう。

図1.3: MVCアーキテクチャの基本フロー

Modelの役割

Modelはアプリケーションのデータとそのデータを操作するビジネスロジックを担当します。

  • アプリケーションのデータ構造を定義
  • データベースとの通信や永続化ロジックを含む
  • ビジネスルールやデータ検証ロジックを実装
  • Viewやコントローラに依存しない独立したコンポーネント
// Spring Frameworkでのモデルの例 // エンティティクラス @Entity @Table(name = "products") public class Product { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; private String name; private String description; private BigDecimal price; private int stockQuantity; // コンストラクタ、ゲッター、セッター（省略） } // リポジトリ（データアクセス層） @Repository public interface ProductRepository extends JpaRepository { List findByNameContaining(String name); List findByPriceLessThan(BigDecimal price); } // サービス（ビジネスロジック層） @Service public class ProductService { private final ProductRepository productRepository; @Autowired public ProductService(ProductRepository productRepository) { this.productRepository = productRepository; } public List getAllProducts() { return productRepository.findAll(); } public Product getProductById(Long id) { return productRepository.findById(id) .orElseThrow(() -> new ResourceNotFoundException("Product not found with id: " + id)); } public Product saveProduct(Product product) { // ビジネスルールの検証 if (product.getPrice().compareTo(BigDecimal.ZERO) <= 0) { throw new IllegalArgumentException("Product price must be greater than zero"); } return productRepository.save(product); } // その他のビジネスメソッド }

Viewの役割

Viewはユーザーインターフェースを担当し、Modelのデータを視覚的に表現します。

  • ユーザーに情報を表示
  • ユーザー入力を受け付ける
  • Modelのデータを適切にフォーマットして表示
  • 複雑なビジネスロジックは含まない
// Spring FrameworkでのViewの例（Thymeleafテンプレート） <!DOCTYPE html> <html xmlns:th="http://www.thymeleaf.org"> <head> <title>商品一覧</title> <link rel="stylesheet" href="/css/styles.css" /> </head> <body> <h1>商品一覧</h1> <div class="search-form"> <form th:action="@{/products/search}" method="get"> <input type="text" name="keyword" placeholder="商品名で検索" /> <button type="submit">検索</button> </form> </div> <div class="product-list"> <div th:if="${#lists.isEmpty(products)}"> <p>商品がありません。</p> </div> <div th:unless="${#lists.isEmpty(products)}"> <table> <thead> <tr> <th>ID</th> <th>商品名</th> <th>価格</th> <th>在庫数</th> <th>操作</th> </tr> </thead> <tbody> <tr th:each="product : ${products}"> <td th:text="${product.id}"></td> <td th:text="${product.name}"></td> <td th:text="${#numbers.formatCurrency(product.price)}"></td> <td th:text="${product.stockQuantity}"></td> <td> <a th:href="@{/products/{id}(id=${product.id})}">詳細</a> | <a th:href="@{/products/{id}/edit(id=${product.id})}">編集</a> </td> </tr> </tbody> </table> </div> </div> <a th:href="@{/products/new}" class="btn-add">新規商品登録</a> </body> </html>

Controllerの役割

ControllerはModelとViewの間の調整役で、ユーザーの入力に応じて適切なModelの操作とViewの選択を行います。

  • ユーザーリクエストの受け取りと解釈
  • 適切なModelメソッドの呼び出し
  • 処理結果に基づいて表示するViewの選択
  • ModelとViewの連携を管理
// Spring FrameworkでのControllerの例 @Controller @RequestMapping("/products") public class ProductController { private final ProductService productService; @Autowired public ProductController(ProductService productService) { this.productService = productService; } // 商品一覧を表示 @GetMapping public String listProducts(Model model) { List products = productService.getAllProducts(); model.addAttribute("products", products); return "products/list"; // list.htmlというViewテンプレートを返す } // 詳細表示 @GetMapping("/{id}") public String viewProduct(@PathVariable Long id, Model model) { Product product = productService.getProductById(id); model.addAttribute("product", product); return "products/view"; } // 新規作成フォーム表示 @GetMapping("/new") public String newProductForm(Model model) { model.addAttribute("product", new Product()); return "products/form"; } // 商品保存処理 @PostMapping public String saveProduct(@Valid Product product, BindingResult result) { if (result.hasErrors()) { return "products/form"; } productService.saveProduct(product); return "redirect:/products"; } // 検索機能 @GetMapping("/search") public String searchProducts(@RequestParam String keyword, Model model) { List products = productService.searchProducts(keyword); model.addAttribute("products", products); return "products/list"; } // その他の操作（編集、削除など） }
Spring MVCとMVCパターン

Spring MVCはMVCパターンを実装したフレームワークで、以下のように各コンポーネントが対応します：

  • Model:
    • エンティティクラス（@Entity）
    • リポジトリ（@Repository）
    • サービス（@Service）
  • View:
    • Thymeleaf、JSP、Freemarkerなどのテンプレート
    • JSONやXMLのレスポンス（RESTfulアプリケーションの場合）
  • Controller:
    • @Controller、@RestControllerなどのアノテーションを持つクラス
    • @RequestMapping、@GetMapping、@PostMappingなどのアノテーションを持つメソッド
