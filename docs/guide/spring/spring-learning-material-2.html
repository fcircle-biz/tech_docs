第2章：Spring Bootの導入と開発環境構築

Spring Bootは、本格的なSpringアプリケーションを素早く開発するためのフレームワークです。この章では、Spring Boot開発の第一歩として、開発環境の構築からHello Worldアプリケーションの実行までを学びます。

2.1 Eclipseの導入

EclipseはJava開発に広く使われている統合開発環境（IDE）です。Spring Boot開発をスムーズに行うために、Eclipse IDEを導入しましょう。

2.1.1 Eclipseのダウンロードとインストール

最新版のEclipse IDEをダウンロードしてインストールします。

1Eclipse IDEのダウンロード

Eclipse公式サイト（https://www.eclipse.org/downloads/）から「Eclipse IDE for Enterprise Java and Web Developers」をダウンロードします。

2インストーラーの実行

ダウンロードしたインストーラーを実行し、「Eclipse IDE for Enterprise Java and Web Developers」を選択してインストールします。

3ワークスペースの設定

Eclipseを起動すると、ワークスペースの選択画面が表示されます。プロジェクトを保存する場所を指定して「Launch」をクリックします。

図2.1: Eclipseのインストール手順

2.1.2 Spring ツールのインストール

Eclipse IDEにSpring開発のための追加ツールをインストールします。

1Spring Tool Suiteのインストール

Eclipseのメニューから「Help」→「Eclipse Marketplace」を選択します。

2プラグインの検索

検索欄に「Spring Tools」と入力して検索します。

3インストールの実行

「Spring Tools 4」を見つけて「Install」ボタンをクリックし、画面の指示に従ってインストールします。

4再起動

インストールが完了したら、Eclipseを再起動します。

ポイント

Spring Tool Suite (STS) には以下のような便利な機能があります：

  • Spring固有の設定ファイルの編集サポート
  • Spring Bootプロジェクトの作成ウィザード
  • Spring固有のコード補完機能
  • アプリケーションプロパティの編集サポート
  • Spring Boot DevTools連携

2.2 Spring Initializrの使い方とプロジェクト生成

Spring Initializrは、Spring Bootプロジェクトの雛形を簡単に生成できるWebツールです。ここでは、Spring Initializrを使ってプロジェクトを作成する方法を学びます。

2.2.1 Spring Initializrとは

Spring Initializrは、Spring Bootアプリケーションの基本構造と必要な依存関係を含むプロジェクトテンプレートを生成するツールです。以下の方法でアクセスできます：

  • Webブラウザ: https://start.spring.io/
  • Eclipse STS内のウィザード
  • IntelliJ IDEAの新規プロジェクト作成機能
  • Spring CLIツール

2.2.2 Webブラウザでの利用方法

Webブラウザからアクセスする場合の手順を説明します。

1Spring Initializrにアクセス

ブラウザで https://start.spring.io/ にアクセスします。

2プロジェクト設定

以下の項目を設定します：

  • プロジェクトタイプ: Maven Project または Gradle Project
  • 言語: Java, Kotlin, または Groovy
  • Spring Bootバージョン: 最新の安定版を選択（例: 3.2.0）
  • Project Metadata: Group, Artifact, Name, Description, Package nameなど
  • パッケージング: Jar または War
  • Java バージョン: 17など

3依存関係の追加

「ADD DEPENDENCIES」をクリックして、必要な依存関係を追加します。基本的なWebアプリケーションの場合は以下を追加します：

  • Spring Web
  • Spring Boot DevTools（開発効率化）
  • Thymeleaf（テンプレートエンジン）

4プロジェクトのダウンロード

「GENERATE」ボタンをクリックして、プロジェクトZIPファイルをダウンロードします。

5Eclipseへのインポート

ダウンロードしたZIPファイルを解凍し、Eclipseで「File」→「Import」→「Existing Maven Projects」または「Existing Gradle Project」を選択してインポートします。

図2.2: Spring Initializrを使ったプロジェクト生成フロー

2.2.3 Eclipse STSでの利用方法

Spring Tool Suite (STS) がインストールされている場合、Eclipseから直接Spring Bootプロジェクトを作成できます。

1新規プロジェクトの作成

「File」→「New」→「Spring Starter Project」を選択します。

2プロジェクト情報の入力

プロジェクト名、Javaバージョン、グループID、アーティファクトIDなどを入力します。

3依存関係の選択

「Next」をクリックし、必要な依存関係（Spring Web, Thymeleaf, DevToolsなど）を選択します。

4プロジェクト作成の完了

「Finish」をクリックしてプロジェクトを作成します。Eclipseがプロジェクトの依存関係を解決し、準備が整うまで待ちます。

演習2.1: 最初のSpring Bootプロジェクトの作成

以下の条件でSpring Bootプロジェクトを作成してみましょう：

  • プロジェクト名: hello-spring-boot
  • ビルドツール: Maven
  • 言語: Java
  • Spring Bootバージョン: 3.2.0（または最新の安定版）
  • グループID: com.example
  • アーティファクトID: hello-spring-boot
  • パッケージング: Jar
  • Javaバージョン: 17
  • 依存関係: Spring Web, Spring Boot DevTools, Thymeleaf

作成したプロジェクトをEclipseにインポートし、プロジェクト構造を確認してください。

2.3 pom.xml / build.gradle の基本

Spring Bootプロジェクトでは、MavenまたはGradleというビルドツールを使用します。ここでは、それぞれのビルド設定ファイルの基本的な構造と役割を学びます。

2.3.1 Mavenと pom.xml

Mavenは、Javaプロジェクトの依存関係管理とビルドプロセスを自動化するツールです。設定ファイルはpom.xml（Project Object Model）と呼ばれます。

<?xml version="1.0" encoding="UTF-8"?> <project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd"> <modelVersion>4.0.0</modelVersion> <!-- 親プロジェクト定義 --> <parent> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-parent</artifactId> <version>3.2.0</version> <relativePath/> <!-- repository から親POMを検索 --> </parent> <!-- プロジェクト情報 --> <groupId>com.example</groupId> <artifactId>hello-spring-boot</artifactId> <version>0.0.1-SNAPSHOT</version> <name>hello-spring-boot</name> <description>Demo project for Spring Boot</description> <!-- プロパティ定義 --> <properties> <java.version>17</java.version> </properties> <!-- 依存関係 --> <dependencies> <!-- Spring Webスターター --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-web</artifactId> </dependency> <!-- Thymeleafテンプレートエンジン --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-thymeleaf</artifactId> </dependency> <!-- 開発ツール (自動リロードなど) --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-devtools</artifactId> <scope>runtime</scope> <optional>true</optional> </dependency> <!-- テスト用ライブラリ --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-test</artifactId> <scope>test</scope> </dependency> </dependencies> <!-- ビルド設定 --> <build> <plugins> <plugin> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-maven-plugin</artifactId> </plugin> </plugins> </build> </project>
pom.xmlの主要要素
  • parent: Spring Boot親POMの参照（バージョン管理やプラグイン設定を継承）
  • groupId: プロジェクトグループのID（通常は組織のドメイン名の逆順）
  • artifactId: プロジェクト名
  • version: プロジェクトのバージョン
  • properties: プロジェクト全体で使用するプロパティ（Javaバージョンなど）
  • dependencies: プロジェクトが必要とするライブラリの一覧
  • build: ビルドプロセスの設定（プラグインなど）

2.3.2 Gradleと build.gradle

Gradleは、Mavenの代替として使用できるより柔軟なビルドツールです。Groovyまたは Kotlin DSLで記述され、設定ファイルはbuild.gradleと呼ばれます。

// Gradleプラグイン設定 plugins { id 'java' id 'org.springframework.boot' version '3.2.0' id 'io.spring.dependency-management' version '1.1.4' } // プロジェクト情報 group = 'com.example' version = '0.0.1-SNAPSHOT' // Javaバージョン設定 java { sourceCompatibility = '17' } // リポジトリ設定 repositories { mavenCentral() } // 依存関係 dependencies { // Spring Web implementation 'org.springframework.boot:spring-boot-starter-web' // Thymeleafテンプレートエンジン implementation 'org.springframework.boot:spring-boot-starter-thymeleaf' // 開発ツール developmentOnly 'org.springframework.boot:spring-boot-devtools' // テスト用ライブラリ testImplementation 'org.springframework.boot:spring-boot-starter-test' } // テスト設定 tasks.named('test') { useJUnitPlatform() }
build.gradleの主要要素
  • plugins: Gradleプラグインの設定（Spring Boot, Java, 依存関係管理など）
  • group, version: プロジェクトのグループIDとバージョン
  • java: Javaバージョンなどの設定
  • repositories: 依存関係をダウンロードするリポジトリの設定
  • dependencies: プロジェクトが必要とするライブラリの一覧
  • tasks: ビルドプロセスのカスタムタスク設定

2.3.3 依存関係スコープ

依存関係には、異なる「スコープ」が設定できます。これにより、依存関係がどのタイミングで使用されるかを制御できます。

Maven Gradle 説明
compile (デフォルト) implementation コンパイルとランタイムの両方で必要
runtime runtimeOnly 実行時のみ必要（コンパイル時は不要）
provided compileOnly コンパイル時のみ必要（実行時は環境から提供される）
test testImplementation テスト時のみ必要
- developmentOnly 開発時のみ必要（本番環境では含まれない）
演習2.2: 依存関係の追加

作成したSpring Bootプロジェクトに以下の依存関係を追加してみましょう：

  1. Spring Boot Validation: 入力検証用のライブラリ
  2. Lombok: ボイラープレートコードを減らすためのアノテーションライブラリ

Maven（pom.xml）の場合：

<!-- Bean Validation --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-validation</artifactId> </dependency> <!-- Lombok --> <dependency> <groupId>org.projectlombok</groupId> <artifactId>lombok</artifactId> <optional>true</optional> </dependency>

Gradle（build.gradle）の場合：

// Bean Validation implementation 'org.springframework.boot:spring-boot-starter-validation' // Lombok compileOnly 'org.projectlombok:lombok' annotationProcessor 'org.projectlombok:lombok'

依存関係を追加した後、プロジェクトを更新して依存関係が正しく解決されるか確認してください。

2.4 Hello Worldの表示とプロジェクトの実行確認

Spring Bootプロジェクトの基本構造を理解したところで、実際に動作する簡単なWebアプリケーションを作成してみましょう。

2.4.1 プロジェクト構造

Spring Initializrで生成されたプロジェクトには、基本的な構造が既に用意されています。

基本的なディレクトリ構造
src
 ├── main
 │   ├── java
 │   │   └── com
 │   │       └── example
 │   │           └── hellospringboot
 │   │               └── HelloSpringBootApplication.java
 │   └── resources
 │       ├── static    # 静的リソース（CSS, JavaScript, 画像など）
 │       ├── templates # テンプレートファイル（Thymeleafなど）
 │       └── application.properties # アプリケーション設定ファイル
 └── test
     └── java
         └── com
             └── example
                 └── hellospringboot
                     └── HelloSpringBootApplicationTests.java

2.4.2 メインアプリケーションクラス

Spring Bootアプリケーションのエントリーポイントは、@SpringBootApplicationアノテーションが付いたメインクラスです。

package com.example.hellospringboot; import org.springframework.boot.SpringApplication; import org.springframework.boot.autoconfigure.SpringBootApplication; @SpringBootApplication public class HelloSpringBootApplication { public static void main(String[] args) { SpringApplication.run(HelloSpringBootApplication.class, args); } }
@SpringBootApplication アノテーション

この一つのアノテーションは、以下の3つのアノテーションを組み合わせたものです：

  • @Configuration: このクラスがSpringの設定クラスであることを示す
  • @EnableAutoConfiguration: Spring Bootの自動設定機能を有効にする
  • @ComponentScan: このクラスのパッケージとサブパッケージからSpringコンポーネントをスキャンする

2.4.3 コントローラーの作成

「Hello World」を表示するためのコントローラークラスを作成します。

package com.example.hellospringboot.controller; import org.springframework.stereotype.Controller; import org.springframework.ui.Model; import org.springframework.web.bind.annotation.GetMapping; @Controller public class HelloController { @GetMapping("/") public String hello(Model model) { model.addAttribute("message", "Hello, Spring Boot!"); return "hello"; // テンプレート名 (hello.html) } }
コントローラーの主要部分
  • @Controller: このクラスがSpring MVCのコントローラーであることを示す
  • @GetMapping("/"): ルートURL（/）へのGETリクエストをこのメソッドにマッピング
  • Model: ビュー（テンプレート）に渡すデータを格納するオブジェクト
  • return "hello": 使用するテンプレートの名前（hello.html）を指定

2.4.4 テンプレートの作成

Thymeleafテンプレートエンジンを使用して、HTMLテンプレートを作成します。src/main/resources/templates/hello.html を作成します。

<!DOCTYPE html> <html xmlns:th="http://www.thymeleaf.org"> <head> <meta charset="UTF-8"> <title>Hello Spring Boot</title> <style> body { font-family: 'Arial', sans-serif; margin: 0; padding: 0; display: flex; justify-content: center; align-items: center; height: 100vh; background-color: #f5f5f5; } .container { text-align: center; padding: 2rem; background-color: white; border-radius: 8px; box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1); } h1 { color: #28a745; } </style> </head> <body> <div class="container"> <h1 th:text="${message}">Default Message</h1> <p>Welcome to Spring Boot!</p> </div> </body> </html>
Thymeleafの基本
  • xmlns:th="http://www.thymeleaf.org": Thymeleaf名前空間の宣言
  • th:text="${message}": コントローラーから渡された「message」変数の値を表示
  • Default Message: テンプレートをブラウザで直接開いた場合のデフォルト表示（開発時の確認用）

2.4.5 アプリケーションの実行

作成したSpring Bootアプリケーションを実行して、「Hello World」を表示してみましょう。

1Eclipseでのプロジェクト実行

プロジェクトを右クリックして「Run As」→「Spring Boot App」を選択します。

2コンソール出力の確認

Eclipseのコンソールで起動ログを確認します。「Started HelloSpringBootApplication in X.XXX seconds」のようなメッセージが表示されれば、正常に起動しています。

3ブラウザでのアクセス

ブラウザで http://localhost:8080 にアクセスします。

4表示の確認

「Hello, Spring Boot!」というメッセージが表示されれば成功です。

表示結果イメージ

Welcome to Spring Boot!

演習2.3: コントローラーの拡張

HelloControllerに以下のエンドポイントを追加してみましょう：

  1. '/greeting': 現在の日時と挨拶メッセージを表示するページ
  2. '/about': アプリケーションの説明を表示するページ

コントローラーの拡張例：

@GetMapping("/greeting") public String greeting(Model model) { LocalDateTime now = LocalDateTime.now(); String greeting = "こんにちは"; if (now.getHour() < 12) { greeting = "おはようございます"; } else if (now.getHour() >= 18) { greeting = "こんばんは"; } model.addAttribute("currentTime", now.format(DateTimeFormatter.ofPattern("yyyy年MM月dd日 HH:mm:ss"))); model.addAttribute("greeting", greeting); return "greeting"; } @GetMapping("/about") public String about(Model model) { model.addAttribute("appName", "Spring Boot Demo Application"); model.addAttribute("description", "これはSpring Bootを使った最初のアプリケーションです。"); model.addAttribute("version", "1.0.0"); return "about"; }

対応するThymeleafテンプレート（greeting.html, about.html）も作成し、アプリケーションを実行して動作を確認してください。

2.5 理解度確認テスト

第2章の理解度チェック

以下の問題に答えて、第2章の内容の理解度を確認しましょう。

問題1: Spring Bootアプリケーションのエントリーポイントを示すアノテーションは何ですか？

問題2: Spring Initializrでプロジェクトを生成する方法として正しくないものはどれですか？

問題3: Spring BootでWebアプリケーションを開発するために必要な依存関係はどれですか？

問題4: Thymeleafテンプレートでコントローラーから渡された変数「userName」を表示するための正しい構文は？

問題5: Spring MVC コントローラーのメソッドから返される文字列「about」の意味として正しいものは？

正解:
  1. b（@SpringBootApplication）
  2. d（Eclipseの標準機能でプロジェクトを作成してからSpring依存関係を手動で追加する）
  3. a（spring-boot-starter-web）
  4. a（<p th:text="${userName}">デフォルト値</p>）
  5. c（templates/about.htmlというテンプレートが使用される）
