第4章: プロシージャとファンクション

プロシージャとファンクション

この章で学ぶこと

  • Functionプロシージャの作成と戻り値
  • 引数の渡し方（ByVal、ByRef）

4.1 Subプロシージャの基本

Subプロシージャは一連の処理をまとめた再利用可能なコードブロックです。

Sub プロシージャ名(引数1 As データ型, 引数2 As データ型) ' 処理内容 End Sub
実習 4-1: 基本的なSubプロシージャ
Sub 挨拶(名前 As String, 時間帯 As String) Dim メッセージ As String Select Case 時間帯 Case "朝" メッセージ = "おはようございます、" & 名前 & "さん" Case "昼" メッセージ = "こんにちは、" & 名前 & "さん" Case "夜" メッセージ = "こんばんは、" & 名前 & "さん" Case Else メッセージ = "こんにちは、" & 名前 & "さん" End Select MsgBox メッセージ End Sub Sub 挨拶テスト() Call 挨拶("田中", "朝") Call 挨拶("佐藤", "夜") End Sub

4.2 Functionプロシージャの基本

Functionプロシージャは値を返すプロシージャです。

Function ファンクション名(引数1 As データ型) As 戻り値の型 ' 処理内容 ファンクション名 = 戻り値 End Function
実習 4-2: 計算処理のファンクション
Function 消費税込み価格(税抜価格 As Currency) As Currency Const 消費税率 As Double = 0.1 消費税込み価格 = 税抜価格 * (1 + 消費税率) End Function Function 割引価格(元価格 As Currency, 割引率 As Double) As Currency 割引価格 = 元価格 * (1 - 割引率) End Function Sub 価格計算テスト() Dim 税抜 As Currency Dim 税込 As Currency Dim 割引後 As Currency 税抜 = 1000 税込 = 消費税込み価格(税抜) 割引後 = 割引価格(税込, 0.1) MsgBox "税抜価格: " & Format(税抜, "#,##0") & "円" & vbCrLf & _ "税込価格: " & Format(税込, "#,##0") & "円" & vbCrLf & _ "割引後価格: " & Format(割引後, "#,##0") & "円" End Sub

4.3 引数の渡し方

ByVal（値渡し）

引数の値のコピーを渡します。元の変数は変更されません。

ByRef（参照渡し）

引数の参照を渡します。プロシージャ内で変更すると元の変数も変更されます。

実習 4-3: 値渡しと参照渡しの違い
Sub 値渡しテスト(ByVal 値 As Integer) 値 = 値 + 10 MsgBox "プロシージャ内の値: " & 値 End Sub Sub 参照渡しテスト(ByRef 値 As Integer) 値 = 値 + 10 MsgBox "プロシージャ内の値: " & 値 End Sub Sub 引数渡しテスト() Dim 元の値 As Integer 元の値 = 100 MsgBox "元の値: " & 元の値 Call 値渡しテスト(元の値) MsgBox "値渡し後の元の値: " & 元の値 Call 参照渡しテスト(元の値) MsgBox "参照渡し後の元の値: " & 元の値 End Sub

4.4 オプション引数

Optionalキーワードを使用して、省略可能な引数を定義できます。

実習 4-4: オプション引数の使用
Function 文字列フォーマット(文字列 As String, _ Optional 大文字 As Boolean = False, _ Optional 括弧 As Boolean = False) As String Dim 結果 As String 結果 = 文字列 If 大文字 Then 結果 = UCase(結果) End If If 括弧 Then 結果 = "[" & 結果 & "]" End If 文字列フォーマット = 結果 End Function Sub オプション引数テスト() MsgBox 文字列フォーマット("hello world") MsgBox 文字列フォーマット("hello world", True) MsgBox 文字列フォーマット("hello world", True, True) MsgBox 文字列フォーマット("hello world", , True) End Sub
💡 プロシージャ設計のベストプラクティス
  • 1つのプロシージャは1つの機能に集中させる
  • 意味のある名前を付ける
  • 引数の数は最小限に抑える
  • エラーハンドリングを適切に実装する
  • コメントで動作を説明する
