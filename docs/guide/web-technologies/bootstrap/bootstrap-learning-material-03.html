第3章

コンテナとグリッドシステム入門

Bootstrapの基盤となるコンテナの概念と、12カラムグリッドシステムの基本を学び、効率的にレイアウトを作る方法を習得します。

この章で学ぶこと

  • コンテナの役割と種類（.container、.container-fluid）を理解する
  • Bootstrapの12カラムグリッドシステムの仕組みを学ぶ
  • 行（.row）とカラム（.col）の基本的な使い方をマスターする
  • 等幅カラムと明示的な幅指定の違いを実践的に学ぶ

3.1 コンテナとは？

コンテナ（Container）は、Bootstrapのレイアウトの最も基本的な要素です。コンテナは、Webページのコンテンツを包み込み、適切な余白（padding）と幅を提供する役割を持っています。料理でいうと、料理を盛り付ける「お皿」のようなイメージです。

コンテナがないと、コンテンツがブラウザの端にぴったりくっついてしまい、読みにくいレイアウトになってしまいます。コンテナを使うことで、見やすく整ったレイアウトを簡単に実現できます。

コンテナの種類

.container

画面サイズに応じて固定幅を持つコンテナです。左右に適切な余白が自動的に追加されます。

用途：一般的なWebサイトのメインコンテンツ

.container-fluid

ブラウザの幅いっぱいに広がるコンテナです。画面サイズに関わらず、常に100%の幅を持ちます。

用途：ヘッダーやフッター、全幅を使いたいセクション

コンテナの基本コード

.containerの例 
<!-- 固定幅コンテナ -->
<div class="container">
    <h1>こんにちは、Bootstrap！</h1>
    <p>このコンテンツは画面サイズに応じた固定幅で表示されます。</p>
</div>
.container-fluidの例 
<!-- 全幅コンテナ -->
<div class="container-fluid">
    <h1>全幅のコンテンツ</h1>
    <p>このコンテンツは画面の端から端まで広がります。</p>
</div>

初心者向けアドバイス

迷ったら.containerを使いましょう。一般的なWebサイトでは、.containerの方がよく使われます。.container-fluidは、ヒーローイメージ（トップの大きな画像）やギャラリーなど、全幅を活かしたい場合に使用します。

3.2 12カラムグリッドシステムとは？

Bootstrapのグリッドシステムは、画面を水平方向に12等分し、その組み合わせでレイアウトを作成する仕組みです。12という数字は、2、3、4、6で割り切れるため、さまざまなレイアウトパターンを柔軟に作れるという利点があります。

画面を12等分したマス目のイメージです。各カラムはこのマス目の単位で配置されます。

なぜ12カラム？

2等分レイアウト

12 ÷ 2 = 6カラムずつ（例：メインコンテンツとサイドバー）

3等分レイアウト

12 ÷ 3 = 4カラムずつ（例：3つの特徴を並べる）

4等分レイアウト

12 ÷ 4 = 3カラムずつ（例：4つの商品カードを並べる）

不均等レイアウト

8カラム + 4カラム、9カラム + 3カラムなど、自由な組み合わせ

重要ポイント

グリッドシステムでは、1つの行（row）の中のカラム数の合計は12になるように配置するのが基本です。12を超えると、はみ出たカラムは次の行に折り返されます。

3.3 行（.row）とカラム（.col）の基本

Bootstrapのグリッドシステムを使うには、3つの階層構造を理解する必要があります。

グリッドシステムの3階層

コンテナ（.container または .container-fluid）
└── 行（.row）
    └── カラム（.col-*）
1

コンテナ（.container）

最も外側の枠組み。全体を包み込み、適切な余白を提供します。

2

行（.row）

カラムを横一列に並べるための行です。コンテナの中に配置します。1つのコンテナに複数の行を作ることができます。

3

カラム（.col-*）

実際のコンテンツを入れる列です。行の中に配置します。1つの行に複数のカラムを並べることができます。

基本的なグリッドのコード例

等幅カラム（2列） 
<div class="container">
    <div class="row">
        <!-- 自動的に等幅になる2つのカラム -->
        <div class="col">カラム1</div>
        <div class="col">カラム2</div>
    </div>
</div>

用語解説：カラム（Column）

「カラム」は日本語で「列」という意味です。新聞や雑誌のような縦の区切りをイメージしてください。Bootstrapでは.colというクラスを使ってカラムを作成します。

3.4 等幅カラムの実践

.colクラスを使うと、行の中のカラムが自動的に等幅に分割されます。カラムの数に関わらず、常に均等に配置されます。

例1：3つの等幅カラム

3つの等幅カラム 
<div class="container">
    <div class="row">
        <div class="col">特徴1</div>
        <div class="col">特徴2</div>
        <div class="col">特徴3</div>
    </div>
</div>

4つの等幅カラム

        <div class="col">商品C</div>

3.5 明示的な幅指定（.col-数字）

等幅カラムではなく、カラムごとに異なる幅を指定したい場合は、.col-数字の形式を使います。数字は1〜12の範囲で、占めるカラム数を表します。

幅指定の記法

  • .col-6 → 12カラム中の6カラム分（50%の幅）
  • .col-4 → 12カラム中の4カラム分（33.33%の幅）
  • .col-3 → 12カラム中の3カラム分（25%の幅）
  • .col-8 → 12カラム中の8カラム分（66.67%の幅）

例：メインコンテンツとサイドバー

8:4のレイアウト 
<div class="container">
    <div class="row">
        <!-- メインコンテンツ: 8カラム分（66.67%） -->
        <div class="col-8">
            <h2>メインコンテンツ</h2>
            <p>記事本文がここに入ります。</p>
        </div>
        <!-- サイドバー: 4カラム分（33.33%） -->
        <div class="col-4">
            <h3>サイドバー</h3>
            <p>関連リンクなど</p>
        </div>
    </div>
</div>

表示結果のイメージ

メインコンテンツ（8カラム）

記事本文がここに入ります。

サイドバー（4カラム）

関連リンク

8カラム + 4カラム = 12カラムで、合計がぴったり12になっています

注意

1つの行（.row）内のカラム数の合計は12以下にするのが基本です。合計が12を超えると、はみ出たカラムは次の行に自動的に折り返されます。

実習 3-1: 基本的なグリッドレイアウトを作成しよう

3つの等幅カラムと、8:4の不均等なレイアウトを実際に作成して、グリッドシステムの動作を確認しましょう。

手順

  1. 1 新しいHTMLファイル（grid-practice.html）を作成し、Bootstrap 5のCDNを読み込みます
  2. 2 .containerの中に、3つの等幅カラムを持つ.rowを作成します
  3. 3 次に、8カラムと4カラムの不均等なレイアウトを持つ別の.rowを追加します
  4. 4 各カラムに背景色とパディングを追加して、境界が見えるようにします（例：style="background-color: #f0f0f0; padding: 20px;"）
  5. 5 ブラウザで開いて、カラムが正しく配置されているか確認します

第3章のまとめ

  • コンテナはレイアウトの基盤で、.container（固定幅）と.container-fluid（全幅）の2種類があります
  • Bootstrapは12カラムグリッドシステムを採用しており、柔軟なレイアウトが可能です
  • グリッドはコンテナ → 行（.row） → カラム（.col）の3階層構造で構成されます
  • .colは等幅カラム、.col-数字は明示的な幅指定に使います
  • 1つの行内のカラム数の合計は12以下が基本で、超えると次の行に折り返されます

理解度確認クイズ

  1. Q1. .containerと.container-fluidの違いは何ですか？

    回答を表示

    .containerは画面サイズに応じた固定幅を持ち、左右に余白が自動的に追加されます。一方、.container-fluidはブラウザの幅いっぱい（100%の幅）に広がります。一般的なWebサイトでは.containerを使うことが多く、ヘッダーや全幅を活かしたセクションでは.container-fluidを使います。

  2. Q2. Bootstrapのグリッドシステムが12カラムである理由は何ですか？

    回答を表示

    12という数字は、2、3、4、6で割り切れるため、さまざまなレイアウトパターンを柔軟に作成できるからです。例えば、2等分（6+6）、3等分（4+4+4）、4等分（3+3+3+3）、不均等（8+4、9+3など）といった多様な組み合わせが可能になります。

  3. Q3. 以下のコードで、各カラムは何%の幅になりますか？
    <div class="row"><div class="col-3">A</div><div class="col-9">B</div></div>

    回答を表示

    カラムA（.col-3）は25%の幅、カラムB（.col-9）は75%の幅になります。計算方法：カラムA = 3/12 = 25%、カラムB = 9/12 = 75%。合計が12カラムなので、ちょうど1行に収まります。
