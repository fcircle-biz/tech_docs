第9章: カスタマイズとSass

Bootstrapのカスタマイズ方法

この章で学ぶこと
  • カスタムCSSでBootstrapを上書きする方法を理解する
  • Sassの基礎とBootstrapでの活用方法を学ぶ
  • Sass変数を使ったテーマのカスタマイズ方法を習得する
  • カラーパレットとタイポグラフィの変更方法を学ぶ

9.1 カスタムCSSでの上書き

最も簡単なカスタマイズ方法は、Bootstrapの後に独自のCSSファイルを読み込んで、スタイルを上書きすることです。

基本的な上書き方法

<!-- Bootstrap CSS -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- カスタムCSS（Bootstrapの後に読み込む） -->
<link href="custom.css" rel="stylesheet">

custom.css の例

/* プライマリカラーのカスタマイズ */
.btn-primary {
    background-color: #ff6b6b;
    border-color: #ff6b6b;
}

.btn-primary:hover {
    background-color: #ff5252;
    border-color: #ff5252;
}

/* カードのスタイルをカスタマイズ */
.card {
    border-radius: 15px;
    box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}
注意点
  • カスタムCSSは必ずBootstrapの後に読み込む
  • 詳細度を高めるため、必要に応じてクラスを組み合わせる
  • !importantは最終手段として使用する

9.2 Sassとは

Sass（Syntactically Awesome Style Sheets）は、CSSを拡張したプリプロセッサです。変数、ネスト、ミックスインなどの機能を提供します。

Sassの主な機能
  • 変数：色やサイズを変数として定義し、再利用できる
  • ネスト：セレクタを入れ子にして、階層構造を表現できる
  • ミックスイン：スタイルのまとまりを再利用できる
  • 関数：計算や色の操作ができる

Sassの基本構文

// 変数の定義
$primary-color: #3498db;
$font-size-base: 16px;

// ネスト
.navbar {
    background-color: $primary-color;
    
    .nav-link {
        color: white;
        
        &:hover {
            color: lighten($primary-color, 20%);
        }
    }
}

9.3 BootstrapのSassソースを使ったカスタマイズ

より高度なカスタマイズには、BootstrapのSassソースファイルを使用します。

環境構築手順

# Bootstrapをnpmでインストール
npm install bootstrap

# Sassコンパイラをインストール
npm install sass

カスタムSassファイルの作成

プロジェクトにcustom.scssファイルを作成します。

// 1. Bootstrapの変数をインポート
@import "node_modules/bootstrap/scss/functions";
@import "node_modules/bootstrap/scss/variables";

// 2. カスタム変数で上書き
$primary: #ff6b6b;
$secondary: #4ecdc4;
$success: #95e1d3;
$font-family-base: 'Noto Sans JP', sans-serif;
$border-radius: 0.5rem;

// 3. Bootstrapの残りをインポート
@import "node_modules/bootstrap/scss/bootstrap";

コンパイル方法

# Sassをコンパイル
npx sass custom.scss custom.css

# 監視モード（ファイル変更時に自動コンパイル）
npx sass --watch custom.scss:custom.css

9.4 主要な変数のカスタマイズ

カラーパレット

// テーマカラーの変更
$primary: #007bff;
$secondary: #6c757d;
$success: #28a745;
$danger: #dc3545;
$warning: #ffc107;
$info: #17a2b8;

// グレースケール
$white: #fff;
$gray-100: #f8f9fa;
$gray-900: #212529;
$black: #000;

スペーシング

// スペーシングの倍率
$spacer: 1rem;
$spacers: (
    0: 0,
    1: $spacer * 0.25,   // 4px
    2: $spacer * 0.5,    // 8px
    3: $spacer,          // 16px
    4: $spacer * 1.5,    // 24px
    5: $spacer * 3       // 48px
);

タイポグラフィ

// フォント設定
$font-family-sans-serif: 'Noto Sans JP', -apple-system, BlinkMacSystemFont, sans-serif;
$font-size-base: 1rem;
$line-height-base: 1.6;

// 見出しのサイズ
$h1-font-size: 2.5rem;
$h2-font-size: 2rem;
$h3-font-size: 1.75rem;

グリッドシステム

// グリッドのカラム数を変更
$grid-columns: 12;

// ガター（カラム間の余白）
$grid-gutter-width: 1.5rem;

// ブレークポイントのカスタマイズ
$grid-breakpoints: (
    xs: 0,
    sm: 576px,
    md: 768px,
    lg: 992px,
    xl: 1200px,
    xxl: 1600px  // 独自のブレークポイントを追加
);

実習 9-1: オリジナルテーマの作成

Sass変数を使って、独自のカラーテーマを作成してみましょう。

手順
  1. プロジェクトフォルダでnpm install bootstrap sassを実行
  2. custom.scssファイルを作成
  3. 以下の内容を記述
custom.scss
@import "node_modules/bootstrap/scss/functions";
@import "node_modules/bootstrap/scss/variables";

// オリジナルカラーテーマ
$primary: #6366f1;      // インディゴ
$secondary: #a855f7;    // パープル
$success: #10b981;      // エメラルド
$danger: #ef4444;       // レッド
$warning: #f59e0b;      // アンバー
$info: #06b6d4;         // シアン

// フォント設定
$font-family-base: 'Noto Sans JP', sans-serif;
$body-bg: #f9fafb;
$body-color: #1f2937;

@import "node_modules/bootstrap/scss/bootstrap";
コンパイル
npx sass custom.scss custom.css
HTMLで使用
<link href="custom.css" rel="stylesheet">
理解度確認クイズ
  1. カスタムCSSをBootstrapより前に読み込むとどうなりますか？
    解答例

    Bootstrapのスタイルが後から読み込まれるため、カスタムCSSが上書きされてしまい、意図した変更が反映されません。

  2. Sassの主な利点を3つ挙げてください。
    解答例
    • 変数を使って値を再利用できる
    • ネストでスタイルを階層的に記述できる
    • 関数やミックスインで効率的にスタイルを管理できる
  3. Bootstrap変数をカスタマイズする際、なぜ変数のインポート後に上書きするのですか？
    解答例

    Bootstrapのデフォルト変数を読み込んでから上書きすることで、変更していない変数はデフォルト値が使用され、一部のみをカスタマイズできるからです。
