10.4 パフォーマンス最適化

Bootstrapサイトのパフォーマンスを向上させるベストプラクティスを紹介します。

1. 必要なコンポーネントのみを使用

Sassを使用して、必要なコンポーネントのみをインポートすることで、ファイルサイズを削減できます。

// 必要な部分のみインポート
@import "bootstrap/scss/functions";
@import "bootstrap/scss/variables";
@import "bootstrap/scss/mixins";
@import "bootstrap/scss/root";
@import "bootstrap/scss/reboot";
@import "bootstrap/scss/grid";
@import "bootstrap/scss/buttons";
// 他の不要なコンポーネントは除外

2. 画像の最適化

  • 適切なフォーマット（WebP、AVIF）を使用する
  • レスポンシブ画像（srcset属性）を活用する
  • 遅延読み込み（lazy loading）を実装する
<img src="image.jpg" loading="lazy" class="img-fluid" alt="説明">

3. CSSとJavaScriptの最適化

  • 本番環境では圧縮版（.min.css、.min.js）を使用する
  • 未使用のCSSを削除する（PurgeCSSなどのツールを活用）
  • 必要なJavaScriptコンポーネントのみを読み込む

10.5 ベストプラクティス

アクセシビリティ

  • 適切なセマンティックHTMLを使用する
  • ARIA属性を適切に付与する
  • 十分なコントラスト比を確保する
  • キーボードナビゲーションをサポートする

レスポンシブデザイン

  • モバイルファーストで設計する
  • すべてのブレークポイントでテストする
  • タッチフレンドリーなUI（ボタンサイズ44×44px以上）

コードの整理

  • 一貫した命名規則を使用する
  • コンポーネント単位でコードを分割する
  • コメントで複雑な部分を説明する
最終課題: オリジナルWebサイトの構築

これまで学んだ内容を活用して、オリジナルのWebサイトを作成しましょう。

要件
  • レスポンシブなナビゲーションバー
  • グリッドシステムを使ったレイアウト
  • カード、ボタン、フォームなどのコンポーネントを活用
  • カスタムカラーテーマ（SassまたはカスタムCSS）
  • モーダルまたはアコーディオンなどのインタラクティブ要素
推奨テーマ
  • ポートフォリオサイト
  • 企業のコーポレートサイト
  • サービス紹介ランディングページ
  • ブログサイト
理解度確認クイズ
  1. ランディングページで最も重要なセクションは何ですか？
    解答例

    ヒーローセクション（メインビジュアル）です。訪問者が最初に目にする部分で、製品・サービスの価値を瞬時に伝える必要があります。

  2. 画像の最適化方法を3つ挙げてください。
    解答例
    • 適切なフォーマット（WebP、AVIF）を使用する
    • レスポンシブ画像（srcset）を活用する
    • 遅延読み込み（lazy loading）を実装する
