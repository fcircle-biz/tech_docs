カスタマイズとテーマ作成の基礎

Bootstrapのデフォルトスタイルを自分好みにカスタマイズする方法を学び、独自のテーマを作成する基礎を習得します。

この章で学ぶこと

  • Bootstrapのカスタマイズ方法の基本を理解する
  • カスタムCSSを追加してBootstrapのスタイルを上書きする方法を学ぶ
  • 色、フォント、サイズなどの変数を変更する方法を習得する
  • 独自のテーマを作成するための基礎知識を身につける

14.1 Bootstrapのカスタマイズとは？

Bootstrapは非常に便利なフレームワークですが、デフォルトのまま使うと他のサイトと似たようなデザインになってしまいます。カスタマイズとは、Bootstrapのデフォルトスタイルを変更し、自分のプロジェクトに合った独自のデザインを作り出すことです。

カスタマイズすることで、ブランドカラーを反映したボタン、独自のフォント、特定のサイズ感など、プロジェクトの個性を表現できます。この章では、Bootstrapを活かしながら独自性を出す方法を学びます。

カスタマイズの重要性

企業サイトやブランドサイトでは、独自のデザインアイデンティティが重要です。Bootstrapの便利さを保ちながら、ブランドイメージに合ったデザインを実現することで、プロフェッショナルなWebサイトを効率的に構築できます。

カスタマイズの3つのアプローチ

カスタムCSS

Bootstrapの後に独自のCSSを読み込み、スタイルを上書きする方法（初心者向け）

CSS変数

BootstrapのCSS変数を変更して、色やサイズを一括調整する方法

Sassカスタマイズ

ソースコードレベルでBootstrapをビルドする方法（上級者向け）

本章では、最も簡単で実用的なカスタムCSSとCSS変数を使った方法を中心に学習します。

14.2 カスタムCSSの追加方法

最も基本的なカスタマイズ方法は、カスタムCSSを追加してBootstrapのスタイルを上書きすることです。重要なのは、BootstrapのCSSファイルの後にカスタムCSSを読み込むことです。

正しいファイル読み込み順序

index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <!-- 1. Bootstrap CSS（先に読み込む） -->
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
          rel="stylesheet">

    <!-- 2. カスタムCSS（後に読み込む） -->
    <link href="custom.css" rel="stylesheet">
</head>
<body>
    <!-- コンテンツ -->
</body>
</html>

重要ポイント

CSSは後から読み込まれたものが優先されます（カスケード）。カスタムCSSをBootstrapの後に読み込むことで、Bootstrapのスタイルを上書きできます。

実例：ボタンの色をカスタマイズ

【HTML】

index.html 
<button class="btn btn-primary">標準ボタン</button>
<button class="btn btn-primary custom-btn">カスタムボタン</button>

【カスタムCSS】

custom.css 
/* カスタムクラスで新しいスタイルを定義 */
.custom-btn {
    background-color: #ff6b6b;
    border-color: #ff6b6b;
}

.custom-btn:hover {
    background-color: #ff5252;
    border-color: #ff5252;
}

ベストプラクティス

Bootstrapのクラスを直接上書きするのではなく、custom-btnのように独自のクラスを追加する方法がおすすめです。これにより、Bootstrapの標準スタイルとカスタムスタイルを使い分けられます。

14.3 色のカスタマイズ

Bootstrapの色システム（Primary、Success、Dangerなど）を自社のブランドカラーに変更する方法を学びます。

Bootstrapのデフォルトカラー

Primary (#0d6efd)

Success (#198754)

Danger (#dc3545)

Warning (#ffc107)

CSS変数で色を変更

Bootstrap 5では、CSS変数（カスタムプロパティ）を使って簡単に色を変更できます。

custom.css 
/* ブランドカラーに変更 */
:root {
    --bs-primary: #6366f1;      /* インディゴ */
    --bs-primary-rgb: 99, 102, 241;
    --bs-success: #10b981;      /* エメラルド */
    --bs-success-rgb: 16, 185, 129;
    --bs-danger: #ef4444;       /* レッド */
    --bs-danger-rgb: 239, 68, 68;
}

CSS変数の便利さ

:rootで定義したCSS変数を変更すると、.btn-primary、.bg-primary、.text-primaryなど、すべてのPrimary色が一括で変わります！

独自カラーの追加

custom.css 
/* 独自のブランドカラー定義 */
.btn-brand {
    background-color: #8b5cf6;
    border-color: #8b5cf6;
    color: white;
}

.btn-brand:hover {
    background-color: #7c3aed;
    border-color: #7c3aed;
}

14.4 フォントのカスタマイズ

サイトの雰囲気を大きく左右するフォントをカスタマイズして、ブランドイメージに合った表現を実現します。

Google Fontsの導入

【HTMLにフォント読み込みを追加】

index.html 
<head>
    <!-- Google Fonts -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;700&display=swap"
          rel="stylesheet">
</head>

【CSSでフォントを適用】

custom.css 
/* 全体のフォントを変更 */
body {
    font-family: 'Noto Sans JP', sans-serif;
}

/* 見出しのフォントを変更 */
h1, h2, h3, h4, h5, h6 {
    font-family: 'Noto Sans JP', sans-serif;
    font-weight: 700;
}

日本語サイトにおすすめのフォント

  • • Noto Sans JP - ゴシック体、読みやすさ重視
  • • M PLUS Rounded 1c - 丸ゴシック、やわらかい印象
  • • Zen Kaku Gothic New - モダンなゴシック体

14.5 スペーシングとサイズのカスタマイズ

余白やサイズ感を調整することで、サイト全体の印象を大きく変えることができます。

余白（マージン・パディング）の基準値を変更

custom.css 
/* より広めの余白設定 */
:root {
    --bs-spacer: 1.5rem;    /* デフォルトは1rem */
}

/* 特定の要素の余白をカスタマイズ */
.card {
    padding: 2rem;          /* カードの余白を広げる */
}

.container {
    max-width: 1200px;      /* コンテナの最大幅を変更 */
}

フォントサイズのカスタマイズ

custom.css 
/* 全体のフォントサイズを調整 */
body {
    font-size: 18px;        /* デフォルトは16px */
}

/* 見出しのサイズを調整 */
h1 { font-size: 3rem; }
h2 { font-size: 2.5rem; }
h3 { font-size: 2rem; }

14.6 独自テーマの作成基礎

ここまで学んだカスタマイズ技術を組み合わせて、統一感のある独自テーマを作成しましょう。

テーマの設計手順

1

ブランドカラーを決める

メインカラー、アクセントカラー、背景色など、サイトで使う色を決定します。

2

フォントを選定する

サイトの雰囲気に合ったフォントを選び、見出しと本文のフォントを決定します。

3

余白とサイズ感を調整する

サイト全体のゆとりを決め、余白やフォントサイズを調整します。

4

コンポーネントをカスタマイズする

ボタン、カード、ナビゲーションなど、主要なコンポーネントのスタイルを整えます。

完成例：カスタムテーマCSS

custom-theme.css 
/* カスタムテーマ - モダンビジネス */

/* 色のカスタマイズ */
:root {
    --bs-primary: #2563eb;
    --bs-primary-rgb: 37, 99, 235;
    --bs-success: #10b981;
    --bs-success-rgb: 16, 185, 129;
}

/* フォント */
body {
    font-family: 'Noto Sans JP', sans-serif;
    font-size: 16px;
    line-height: 1.7;
}

/* ボタンのカスタマイズ */
.btn {
    border-radius: 8px;
    font-weight: 600;
}

/* カードのカスタマイズ */
.card {
    border-radius: 12px;
    box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}

実習：独自テーマの作成

課題

自分のプロジェクト用に独自のカスタムテーマを作成してみましょう。

  1. 1. Bootstrapの基本テンプレートHTMLファイルを作成する
  2. 2. custom-theme.cssファイルを作成し、以下をカスタマイズ：
    • • ブランドカラー（Primary, Success, Danger）
    • • フォント（Google Fontsから選択）
    • • ボタンとカードのスタイル
  3. 3. ボタン、カード、ナビゲーションバーなどのコンポーネントを配置し、カスタムテーマが適用されることを確認する

ヒント

  • • Adobe ColorやCoolorsなどのカラーパレット生成ツールを使うと便利です
  • • フォントは1〜2種類に絞るとデザインがまとまります
  • • ブラウザの開発者ツールでリアルタイムに調整しながら作業しましょう

第14章のまとめ

  • カスタマイズにより、Bootstrapの便利さを保ちながら独自のデザインを実現できます
  • カスタムCSSをBootstrapの後に読み込むことで、スタイルを上書きできます
  • CSS変数を使うと、色やサイズを一括で変更できます
  • ブランドカラー、フォント、余白、コンポーネントを調整して独自テーマを作成できます
  • カスタマイズすることで、プロフェッショナルで個性的なWebサイトを構築できます

理解度確認クイズ

  1. Q1. カスタムCSSファイルは、BootstrapのCSSファイルの前に読み込むべきか、後に読み込むべきか、その理由とともに答えてください。

    回答を表示

    後に読み込むべきです。
    理由：CSSはカスケード（段階的適用）の仕組みにより、後から読み込まれたスタイルが優先されます。カスタムCSSをBootstrapの後に読み込むことで、Bootstrapのデフォルトスタイルを上書きして独自のスタイルを適用できます。

  2. Q2. CSS変数を使って、BootstrapのPrimaryカラーを#8b5cf6に変更するコードを書いてください。

    回答を表示 
    :root {
    --bs-primary: #8b5cf6;
    --bs-primary-rgb: 139, 92, 246;
}

  3. Q3. Bootstrapをカスタマイズする3つのアプローチを挙げてください。

    回答を表示

    1. カスタムCSS - 独自のCSSファイルを作成してBootstrapのスタイルを上書き（初心者向け）
    2. CSS変数 - BootstrapのCSS変数を変更して色やサイズを一括調整
    3. Sassカスタマイズ - ソースコードレベルでBootstrapをビルドして詳細にカスタマイズ（上級者向け）
