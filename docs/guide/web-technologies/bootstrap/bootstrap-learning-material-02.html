第2章

Bootstrapの導入と基本テンプレート

CDNを使った最も簡単な導入方法から始めます。Bootstrapの基本的なHTMLテンプレート構造、必要なメタタグの意味、CSSとJavaScriptファイルの読み込み順序など、最初の一歩を確実に理解します。

この章で学ぶこと

  • CDNとは何か、その仕組みとメリットを理解する
  • Bootstrapの導入方法（CDN / ダウンロード）を学ぶ
  • Bootstrap用の基本HTMLテンプレートの構造を理解する
  • 必須メタタグの役割とCSSJavaScript読み込み順序を学ぶ
  • 最初のBootstrapページを実際に作成する

2.1 CDNとは？

CDN（Content Delivery Network、コンテンツ配信ネットワーク）とは、Web上のファイルを高速に配信するための仕組みです。Bootstrapを使う際、自分のサーバーにファイルを置くのではなく、CDN経由で読み込むことで、簡単かつ高速に利用できます。

料理に例えると、自宅で調味料を作る（ファイルをダウンロードする）のではなく、近所のスーパー（CDN）から必要な時に取り寄せる感覚です。CDNは世界中にサーバーを持っているため、ユーザーに最も近いサーバーから配信され、読み込み速度が向上します。

CDNを使うメリット

セットアップが簡単

ファイルをダウンロードする必要がなく、HTMLに1行追加するだけで使えます。

高速な読み込み

世界中に配置されたサーバーから、ユーザーに最も近い場所から配信されます。

自動更新対応

バージョン番号を変更するだけで、最新版のBootstrapに更新できます。

キャッシュ効果

他のサイトで同じCDNを使っていれば、ブラウザのキャッシュが利用され高速化します。

用語解説：キャッシュ

「キャッシュ」とは、一度ダウンロードしたファイルをブラウザが保存しておく機能です。次回同じファイルが必要になった時、再度ダウンロードせずに保存されたファイルを使うため、表示が高速になります。

2.2 Bootstrapの導入方法

Bootstrapを使用する方法は主に2つあります。初心者にはCDN経由での導入をお勧めします。

方法1: CDN経由での導入（推奨）

最も簡単で、初心者に推奨される方法です。BootstrapのCSSとJavaScriptファイルをHTMLの<head>タグと<body>タグの最後に追加するだけで使えます。

CDN経由の導入例 
<!-- head内にCSSを追加 -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
      rel="stylesheet">

<!-- body閉じタグの直前にJavaScriptを追加 -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js">
</script>

おすすめポイント

CDN経由なら、ファイルのダウンロードやフォルダ管理が不要です。HTMLファイル1つだけで学習を始められるため、初心者に最適です。

方法2: ファイルをダウンロードして使う

Bootstrapの公式サイトからファイルをダウンロードし、自分のプロジェクトフォルダに配置して使用する方法です。インターネット接続がない環境でも動作しますが、ファイル管理が必要になります。

ダウンロード版の導入例 
<!-- ローカルファイルを参照 -->
<link href="css/bootstrap.min.css" rel="stylesheet">

<script src="js/bootstrap.bundle.min.js"></script>

ダウンロード版のフォルダ構成例 

my-project/
├── index.html
├── css/
│   └── bootstrap.min.css
└── js/
    └── bootstrap.bundle.min.js

ダウンロード方法

公式サイト（https://getbootstrap.com/）の「Download」ボタンからZIPファイルをダウンロードし、解凍して使います。

2.3 Bootstrap用の基本HTMLテンプレート

Bootstrapを使用するための基本的なHTMLテンプレートをご紹介します。このテンプレートをベースに、あらゆるBootstrapページを作成できます。

bootstrap-template.html 
<!DOCTYPE html>
<html lang="ja">
<head>
  <!-- 必須メタタグ -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <!-- Bootstrap CSS -->
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
        rel="stylesheet">

  <title>Bootstrapサンプルページ</title>
</head>
<body>
  <h1>Hello, Bootstrap!</h1>

  <!-- Bootstrap JavaScript -->
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js">
  </script>
</body>
</html>

テンプレートの各部分の役割

1. DOCTYPE宣言

<!DOCTYPE html> - HTML5文書であることをブラウザに伝えます

2. 言語設定

<html lang="ja"> - 日本語のページであることを指定

3. 文字エンコーディング

<meta charset="UTF-8"> - 日本語など多言語対応のUTF-8を指定

4. ビューポート設定（重要！）

<meta name="viewport"...> - レスポンシブデザインに必須の設定

5. Bootstrap CSS読み込み

<link>タグで、head内でCSSを読み込みます

6. Bootstrap JavaScript読み込み

<script>タグで、body終了タグ直前でJSを読み込みます

2.4 ビューポートメタタグの重要性

レスポンシブデザインを実現するために、ビューポートメタタグは必須です。このタグがないと、スマートフォンで見た時にPC版が縮小表示されてしまい、非常に見づらくなります。

ビューポート設定（必須） 
<meta name="viewport" content="width=device-width, initial-scale=1">

width=device-width

ページの幅をデバイスの画面幅に合わせます。スマホならスマホの幅、タブレットならタブレットの幅に自動調整されます。

initial-scale=1

ページを読み込んだ時の初期ズームレベルを1倍（等倍）に設定します。自動的に拡大・縮小されることを防ぎます。

注意：このタグを忘れると…

ビューポートメタタグがないと、スマートフォンでPC用のデスクトップ版が縮小表示され、文字が小さすぎて読めなくなります。Bootstrapを使う場合は必ず設定してください。

2.5 CSSとJavaScriptの読み込み順序

BootstrapのCSSとJavaScriptは、決められた順序で読み込む必要があります。順序を間違えると、スタイルが正しく適用されなかったり、動的な機能が動作しなかったりします。

正しい読み込み例

正しい順序 
<head>
  <!-- 1. 必須メタタグ -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <!-- 2. Bootstrap CSS（先に読み込む） -->
  <link href="bootstrap.min.css" rel="stylesheet">

  <!-- 3. カスタムCSS（後に読み込む） -->
  <link href="my-custom.css" rel="stylesheet">
</head>
<body>
  <!-- コンテンツ -->

  <!-- 4. Bootstrap JavaScript（body終了直前） -->
  <script src="bootstrap.bundle.min.js"></script>
</body>

重要ポイント

  • • CSSは<head>内で読み込む（スタイルを先に適用するため）
  • • JavaScriptは<body>終了タグ直前で読み込む（HTML要素を全て読み込んでから実行するため）
  • • カスタムCSSはBootstrap CSSの後に読み込む（Bootstrapのスタイルを上書きするため）

用語解説：bootstrap.bundle.min.js

「bundle」は「まとめたもの」という意味で、bootstrap.bundle.min.jsには、Bootstrapの動的機能（モーダル、ドロップダウンなど）に必要なJavaScriptがすべて含まれています。このファイル1つを読み込めば、Bootstrapの全機能が使えます。

2.6 初めてのBootstrapページを作成しよう

実際にBootstrapを使った簡単なページを作成してみましょう。ボタンとアラートメッセージを表示するサンプルです。

実習：最初のBootstrapページ

以下のコードを入力して、HTMLファイルを作成しましょう。

first-bootstrap.html 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
        rel="stylesheet">
  <title>初めてのBootstrap</title>
</head>
<body>
  <div class="container mt-5">
    <h1 class="text-primary">Hello, Bootstrap!</h1>
    <p class="lead">これが最初のBootstrapページです。</p>

    <!-- ボタン -->
    <button class="btn btn-primary">プライマリボタン</button>
    <button class="btn btn-success">成功ボタン</button>

    <!-- アラート -->
    <div class="alert alert-info mt-3">
      これはBootstrapのアラートメッセージです！
    </div>
  </div>

  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js">
  </script>
</body>
</html>

動作確認

  1. 1. テキストエディタで新しいファイルを作成し、first-bootstrap.htmlという名前で保存します
  2. 2. 上記のコードをコピー＆ペーストします
  3. 3. ブラウザでファイルを開きます（ダブルクリック、またはブラウザにドラッグ＆ドロップ）
  4. 4. 青い見出し、2つのボタン、青い背景のアラートメッセージが表示されればOKです！

コードの解説

class="container mt-5"

containerはコンテンツを中央に配置し、適切な余白を設定します。mt-5は上マージン（Margin Top）を追加します。

class="text-primary"

テキストをBootstrapのプライマリカラー（青）にします。

class="btn btn-primary"

btnでボタンスタイルを適用し、btn-primaryで青いボタンにします。

class="alert alert-info"

情報メッセージ用のアラートボックスを表示します。mt-3で上に余白を追加しています。

Bootstrapのすごさを実感！

通常のHTMLならCSSを何十行も書く必要がある見栄えの良いページが、クラス名を指定するだけで簡単に作成できました。これがBootstrapの力です！

トラブルシューティング

スタイルが適用されない

  • • CDNのURLが正しいか確認してください
  • • インターネット接続を確認してください
  • • ブラウザのキャッシュをクリアして再読み込みしてください

スマホで表示が崩れる

  • • ビューポートメタタグが正しく設定されているか確認してください
  • • <meta name="viewport"...>がhead内にあることを確認

文字が文字化けする

  • • <meta charset="UTF-8">がhead内にあることを確認
  • • ファイル保存時のエンコーディングがUTF-8になっているか確認

第2章のまとめ

  • CDNを使えば、ファイルをダウンロードせずにBootstrapを簡単に導入できます
  • Bootstrapの基本テンプレートには、必須メタタグ、CSS、JavaScriptの読み込みが含まれます
  • ビューポートメタタグはレスポンシブデザインに必須で、忘れずに設定しましょう
  • CSSはhead内、JavaScriptはbody終了タグ直前で読み込むのが正しい順序です
  • クラス名を指定するだけで、美しいボタンやアラートが簡単に作成できます

理解度確認クイズ

  1. Q1. CDNとは何ですか？CDNを使うメリットを1つ挙げてください。

    回答を表示

    CDN（Content Delivery Network）とは、Web上のファイルを高速に配信するための仕組みです。

    メリット（例）：
    ・セットアップが簡単（ファイルをダウンロードせず、HTMLに1行追加するだけ）
    ・高速な読み込み（世界中のサーバーから最も近い場所から配信される）
    ・自動更新対応（バージョン番号を変えるだけで最新版に更新できる）

  2. Q2. ビューポートメタタグが必要な理由を説明してください。

    回答を表示

    ビューポートメタタグ（<meta name="viewport"...>）は、レスポンシブデザインを実現するために必須です。

    このタグがないと、スマートフォンで見た時にPC版が縮小表示されてしまい、文字が小さすぎて非常に見づらくなります。width=device-widthでデバイスの画面幅に合わせ、initial-scale=1で初期ズームレベルを等倍に設定します。

  3. Q3. BootstrapのCSSとJavaScriptは、HTMLのどこで読み込むべきですか？

    回答を表示

    Bootstrap CSS：<head>内で読み込みます（スタイルを先に適用するため）

    Bootstrap JavaScript：<body>終了タグ直前で読み込みます（HTML要素を全て読み込んでから実行するため）

    また、カスタムCSSを使う場合は、Bootstrap CSSの後に読み込むことで、Bootstrapのスタイルを上書きできます。
