第1章: Bootstrap入門

Bootstrapとは何か

この章で学ぶこと
  • Bootstrapの概念とフレームワークとしての役割を理解する
  • Bootstrapの歴史と進化、現在のバージョン（Bootstrap 5）の特徴を学ぶ
  • CDNとnpmを使った環境構築の方法を習得する
  • Bootstrapの基本的なテンプレート構成と仕組みを理解する

1.1 Bootstrapとは

Bootstrapは、世界中で最も人気のあるCSSフレームワークです。 Webサイトやアプリケーションを素早く美しく構築するための、事前に用意されたCSSとJavaScriptのコンポーネント集です。

フレームワークとは？

フレームワークは、ソフトウェア開発における「骨組み」や「枠組み」のことです。 建築に例えると、家を建てる際に使う「プレハブ工法」のようなもので、 基本的な構造が既に用意されているため、ゼロから作るよりも効率的に開発できます。

Bootstrapを使う理由

  • 開発速度の向上：CSSを一から書く必要がなく、クラス名を指定するだけで美しいデザインが適用される
  • レスポンシブデザイン：スマートフォン、タブレット、PCなど、様々な画面サイズに自動で対応
  • 一貫性のあるデザイン：統一されたデザインルールに基づいているため、見た目の一貫性が保たれる
  • ブラウザ互換性：主要なブラウザでの動作が保証されている
  • 豊富なコンポーネント：ボタン、フォーム、ナビゲーションバーなど、よく使うUI要素が用意されている

1.2 Bootstrapの歴史と進化

Bootstrapは2011年にTwitter社で開発され、「Twitter Bootstrap」として公開されました。 当初は社内ツールの統一を目的としていましたが、オープンソースとして公開されると、 瞬く間に世界中で使われるフレームワークとなりました。

バージョンの変遷

  • Bootstrap 1-2（2011-2012）：基本的なグリッドシステムとコンポーネント
  • Bootstrap 3（2013）：モバイルファーストの設計思想を導入
  • Bootstrap 4（2018）：Sass（CSSプリプロセッサ）の採用、Flexboxの導入
  • Bootstrap 5（2021）：jQuery依存の削除、カスタマイズ性の向上、ユーティリティAPIの追加
Bootstrap 5の主な変更点
  • jQueryの削除：軽量化とモダンなJavaScriptの採用
  • Internet Explorer 11サポート終了：モダンブラウザへの最適化
  • カスタムCSSプロパティの活用：より柔軟なカスタマイズが可能
  • オフキャンバス：新しいコンポーネントの追加

1.3 環境構築方法（CDN）

Bootstrapを使い始める最も簡単な方法は、CDN（Content Delivery Network）を利用することです。 CDNは、インターネット上に公開されているBootstrapのファイルを直接読み込む方法です。

CDNとは？

CDN（Content Delivery Network）は、世界中に配置されたサーバーから、 ユーザーに最も近い場所からファイルを配信する仕組みです。 メリットとして、ダウンロード不要、高速配信、キャッシュの活用などがあります。

CDNを使った基本テンプレート

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Bootstrap サンプル</title>

    <!-- Bootstrap CSS CDN -->
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
    <div class="container">
        <h1>Hello, Bootstrap!</h1>
        <p>これはBootstrapを使った最初のページです。</p>
    </div>

    <!-- Bootstrap JavaScript CDN -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
重要なポイント
  • <meta name="viewport">タグは必須です。これによりモバイル端末での表示が最適化されます
  • CSSファイルは<head>内に、JavaScriptファイルは<body>の終了タグ直前に配置します
  • bootstrap.bundle.min.jsには、Popperライブラリ（ツールチップやポップオーバーに必要）が含まれています

1.4 環境構築方法（npm）

より本格的なプロジェクトでは、npm（Node Package Manager）を使ってBootstrapをインストールします。 この方法では、Bootstrapのソースファイルを直接編集してカスタマイズすることができます。

npmを使ったインストール手順

# 1. プロジェクトフォルダを作成
mkdir my-bootstrap-project
cd my-bootstrap-project

# 2. package.jsonを作成
npm init -y

# 3. Bootstrapをインストール
npm install bootstrap@5.3.0

# 4. (オプション) Sassコンパイラをインストール
npm install sass

インストール後のファイル構成

my-bootstrap-project/
├── node_modules/
│   └── bootstrap/      # Bootstrapのファイルがここに保存される
│       ├── dist/       # コンパイル済みファイル
│       └── scss/       # Sassソースファイル
├── package.json
└── index.html

HTMLからBootstrapを読み込む

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Bootstrap npm版</title>

    <!-- ローカルのBootstrap CSS -->
    <link href="node_modules/bootstrap/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
    <div class="container">
        <h1>Hello, Bootstrap (npm版)!</h1>
    </div>

    <!-- ローカルのBootstrap JavaScript -->
    <script src="node_modules/bootstrap/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

1.5 Bootstrapの基本構成と仕組み

Bootstrapは、大きく分けて以下の3つの要素から構成されています。

1. CSS（スタイルシート）

Bootstrapの核となる部分です。グリッドシステム、タイポグラフィ、コンポーネントのスタイルなどが定義されています。

  • Reboot：ブラウザのデフォルトスタイルをリセットし、一貫性を持たせる
  • グリッドシステム：12カラムベースのレスポンシブレイアウト
  • コンポーネント：ボタン、カード、ナビゲーションバーなどのスタイル
  • ユーティリティ：マージン、パディング、色などの調整用クラス

2. JavaScript

インタラクティブなコンポーネント（モーダル、ドロップダウン、カルーセルなど）の動作を制御します。

3. アイコン（別パッケージ）

Bootstrap Icons は別パッケージとして提供されており、1,800以上のアイコンが利用可能です。

実習 1-1: 最初のBootstrapページを作成する

CDNを使って、Bootstrapを導入した最初のHTMLページを作成しましょう。 この実習では、Bootstrapの基本的な使い方を体験します。

手順
  1. テキストエディタで新しいHTMLファイル（index.html）を作成します
  2. 上記の「CDNを使った基本テンプレート」をコピーして貼り付けます
  3. <div class="container">内に、追加のコンテンツを記述します
  4. ファイルを保存し、Webブラウザで開いて確認します
追加するコンテンツ例
<div class="container mt-5">
    <h1 class="text-primary">Bootstrap入門</h1>
    <p class="lead">これは私の最初のBootstrapページです。</p>
    <button class="btn btn-primary">クリックしてください</button>
    <button class="btn btn-success">成功ボタン</button>
</div>
期待される結果
  • ページの中央に配置されたコンテナが表示される
  • 青色の見出し（h1）が表示される
  • 大きめのフォントサイズの段落（lead）が表示される
  • 青色と緑色のスタイル付きボタンが表示される
実習 1-2: Bootstrapのクラス名を理解する

Bootstrapでは、HTMLの要素に特定のクラス名を付けることでスタイルを適用します。 いくつかの基本的なクラス名を試してみましょう。

試してみるクラス名
<div class="container">
    <!-- テキストの色 -->
    <p class="text-primary">プライマリ色のテキスト</p>
    <p class="text-success">成功色のテキスト</p>
    <p class="text-danger">危険色のテキスト</p>

    <!-- テキストの配置 -->
    <p class="text-start">左揃え</p>
    <p class="text-center">中央揃え</p>
    <p class="text-end">右揃え</p>

    <!-- 背景色 -->
    <div class="bg-primary text-white p-3">プライマリ背景</div>
    <div class="bg-warning p-3">警告背景</div>
</div>
クラス名の命名規則
  • text-で始まるクラス：テキストに関するスタイル
  • bg-で始まるクラス：背景に関するスタイル
  • btn-で始まるクラス：ボタンに関するスタイル
  • p-、m-で始まるクラス：パディング、マージンの調整
理解度確認クイズ
  1. Bootstrapとは何ですか？ 一言で説明してください。
    解答例

    WebサイトやWebアプリケーションを素早く美しく構築するための、CSSとJavaScriptのフレームワークです。

  2. CDNを使う利点を2つ挙げてください。
    解答例
    • ダウンロードやインストールが不要で、すぐに使い始められる
    • 世界中に配置されたサーバーから高速に配信される
    • （その他）ブラウザのキャッシュが活用される
  3. Bootstrap 5の大きな変更点として、削除されたライブラリは何ですか？
    解答例

    jQuery（ジェイクエリー）が削除されました。これにより、ファイルサイズが軽量化され、モダンなJavaScriptが採用されています。

  4. <meta name="viewport">タグは何のために必要ですか？
    解答例

    スマートフォンやタブレットなどのモバイル端末で、Webページを正しく表示するために必要です。これがないと、レスポンシブデザインが正しく動作しません。

  5. Bootstrapの構成要素として、主なものを3つ挙げてください。
    解答例
    • CSS（グリッドシステム、コンポーネント、ユーティリティ）
    • JavaScript（インタラクティブなコンポーネントの制御）
    • アイコン（Bootstrap Icons - 別パッケージ）

1.6 まとめ

この章では、Bootstrapの基本概念と環境構築方法を学びました。

  • Bootstrapは、Web開発を効率化するCSSフレームワークである
  • レスポンシブデザイン、ブラウザ互換性、豊富なコンポーネントが特徴
  • CDNを使えば、すぐに使い始めることができる
  • npmを使うことで、より本格的なカスタマイズが可能
  • Bootstrap 5では、jQueryが削除され、よりモダンなフレームワークになった
