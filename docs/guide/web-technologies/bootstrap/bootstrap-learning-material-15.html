実践プロジェクト - レスポンシブWebサイトの構築

これまで学んだBootstrapの知識を総合して、実際のWebサイトを一から作成します。ヘッダー、ヒーローセクション、特徴紹介、お問い合わせフォームなど、実践的なWebサイトを完成させます。

この章で学ぶこと

  • 実際のWebサイトを構築する流れと設計の考え方を理解する
  • これまで学んだBootstrapのコンポーネントを組み合わせて使う技術を習得する
  • レスポンシブデザインを考慮した実践的なレイアウト構築方法を学ぶ
  • プロフェッショナルなWebサイトに必要な要素を網羅的に実装する

15.1 プロジェクトの概要と計画

この章では、架空の企業「Tech Solutions」のコーポレートサイトを構築します。一般的な企業サイトに必要な要素を網羅し、Bootstrapを使った実践的なWeb開発の流れを体験します。

構築するWebサイトの構成

サイト構成 

Tech Solutions コーポレートサイト
├── ヘッダー（固定ナビゲーションバー）

15.2 基本テンプレートとヘッダーの実装

まずは、Bootstrapの基本テンプレートを作成し、サイト全体で使用する固定ヘッダーを実装します。

基本HTMLテンプレート

index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tech Solutions - 技術で未来を創る</title>
  <!-- Bootstrap 5 CDN -->
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
  <!-- Bootstrap Icons -->
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.0/font/bootstrap-icons.css">
</head>
<body>
  <!-- コンテンツはここに追加 -->

  <!-- Bootstrap 5 JavaScript -->
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

固定ヘッダーナビゲーション

ヘッダー部分 
<nav class="navbar navbar-expand-lg navbar-dark bg-dark fixed-top">
  <div class="container">
    <a class="navbar-brand fw-bold" href="#">
      <i class="bi bi-laptop"></i> Tech Solutions
    </a>
    <button class="navbar-toggler" type="button" data-bs-toggle="collapse"
            data-bs-target="#navbarNav">
      <span class="navbar-toggler-icon"></span>
    </button>
    <div class="collapse navbar-collapse" id="navbarNav">
      <ul class="navbar-nav ms-auto">
        <li class="nav-item"><a class="nav-link" href="#services">サービス</a></li>
        <li class="nav-item"><a class="nav-link" href="#portfolio">実績</a></li>
        <li class="nav-item"><a class="nav-link" href="#contact">お問い合わせ</a></li>
      </ul>
    </div>
  </div>
</nav>

重要ポイント

fixed-topクラスでナビゲーションバーを画面上部に固定します。navbar-expand-lgで大画面では横並び、小画面ではハンバーガーメニューに自動切り替えします。

15.8 レスポンシブ対応の確認とブラッシュアップ

作成したWebサイトが、さまざまな画面サイズで正しく表示されるか確認し、必要に応じて調整します。

確認すべきブレークポイント

デバイス 画面幅 確認ポイント
スマートフォン ~575px ハンバーガーメニュー、1列表示
タブレット 576px~991px 2列表示、余白調整
PC 992px~ 3列表示、フル機能

実習 15-1: 開発者ツールでレスポンシブ確認

ブラウザの開発者ツールを使って、作成したサイトをさまざまな画面サイズで確認します。

手順

  1. 1 Chrome開発者ツールを開く（F12キー、または右クリック→検証）
  2. 2 デバイスツールバーをクリック（Ctrl+Shift+M）
  3. 3 プリセットのデバイスを選択、または画面幅を手動で変更
  4. 4 各セクションのレイアウト、余白、文字サイズが適切か確認

よくある調整ポイント

スマホでの余白調整

スマホ画面では余白が大きすぎることがあります。py-5 py-md-6のように、画面サイズによって余白を変更できます。

文字サイズの調整

大きな見出しはスマホで画面からはみ出すことがあります。display-4 display-md-3で画面サイズに応じたサイズ調整が可能です。

要素の非表示

特定の画面サイズで要素を非表示にする場合は、d-none d-md-blockのようなクラスを使用します。

15.9 カスタムCSSでのブラッシュアップ

Bootstrapだけでは実現できないデザインは、カスタムCSSを追加して調整します。

custom.css 
/* ヒーローセクションの背景グラデーション */
.hero-section {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  min-height: 75vh;
}

/* カードのホバーエフェクト */
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.card:hover {
  transform: translateY(-5px);
  box-shadow: 0 10px 20px rgba(0,0,0,0.1);
}

/* スムーススクロール */
html {
  scroll-behavior: smooth;
}

/* セクション間の余白を統一 */
section {
  padding: 80px 0;
}

@media (max-width: 767px) {
  section {
    padding: 60px 0;
  }
}

重要ポイント

カスタムCSSは、Bootstrapの後に読み込むことで、Bootstrapのスタイルを上書きできます。ただし、過度なカスタマイズはBootstrapの利点を失うため、バランスが重要です。

理解度確認クイズ

  1. Q1. 固定ナビゲーションバーを作成するために使用するBootstrapクラスは何ですか？

    回答を表示

    fixed-topクラスを使用します。これにより、ナビゲーションバーが画面上部に固定され、スクロールしても常に表示されます。また、コンテンツが隠れないように適切なマージンやパディングの調整も必要です。

  2. Q2. カードの高さを揃えるために使用するクラスは何ですか？また、なぜこれが重要ですか？

    回答を表示

    h-100クラスを使用します。このクラスは要素の高さを100%にし、並んだカードの高さを揃えることができます。これにより、レイアウトが美しく整い、プロフェッショナルな印象を与えることができます。

  3. Q3. レスポンシブデザインを確認する際に使用する、主要な3つのブレークポイント（画面サイズ）を挙げてください。

    回答を表示

    主要な3つのブレークポイント：
    1. スマートフォン: ~575px（小画面）
    2. タブレット: 576px~991px（中画面）
    3. PC: 992px以上（大画面）

    これらの画面サイズで適切に表示されるか確認することで、あらゆるデバイスで快適に閲覧できるサイトになります。
