第4章

レスポンシブグリッドの実践

ブレークポイントを使ってデバイスごとに最適化されたレイアウトを作成し、スマートフォン、タブレット、PCで異なるカラム配置を実現する方法を学びます。

この章で学ぶこと

  • Bootstrapのブレークポイント（画面サイズの区切り）を理解する
  • レスポンシブグリッドクラス（col-sm、col-md、col-lg等）の使い方を習得する
  • デバイスごとに異なるカラム配置を実現する実践的なレイアウトを作成する
  • 開発者ツールを使ったレスポンシブデザインの確認方法を学ぶ

4.1 ブレークポイントとは？

ブレークポイントとは、レスポンシブデザインにおいて「画面サイズの区切り」のことです。スマートフォン、タブレット、PC など、異なるデバイスの画面サイズに応じてレイアウトを切り替える基準となる幅の値を指します。

Bootstrapでは、画面幅に応じて5つのブレークポイントが定義されており、それぞれに対応したクラス名が用意されています。これにより、「スマホでは1カラム、タブレットでは2カラム、PCでは3カラム」といった柔軟なレイアウトを簡単に実現できます。

用語解説：レスポンシブデザイン

レスポンシブデザインとは、画面サイズに応じて自動的にレイアウトが最適化されるデザイン手法です。1つのHTMLで、スマホ、タブレット、PCなど、あらゆるデバイスに対応できます。HTML/CSS学習ガイド第13章で学んだメディアクエリの概念を、Bootstrapではより簡単に実現できます。

Bootstrapの5つのブレークポイント

クラス接頭辞 画面幅 対象デバイス 例
col- <576px 極小画面（スマホ縦向き） col-12
col-sm- ≥576px 小画面（スマホ横向き） col-sm-6
col-md- ≥768px 中画面（タブレット） col-md-4
col-lg- ≥992px 大画面（デスクトップPC） col-lg-3
col-xl- ≥1200px 超大画面（大型ディスプレイ） col-xl-2

重要ポイント

ブレークポイントは「以上」の意味です。例えば col-md-6 は「画面幅が768px以上のとき6カラム幅にする」という意味になります。小さい画面では自動的に12カラム（全幅）になります。

4.2 レスポンシブグリッドクラスの基本

レスポンシブグリッドを実現するには、同じ要素に複数のブレークポイントクラスを組み合わせて指定します。画面サイズごとに異なるカラム幅を設定することで、デバイスに最適なレイアウトを実現できます。

基本的な記法

基本構文 
<div class="col-12 col-md-6 col-lg-4">
  <!-- 内容 -->
</div>

上記の例は、以下のように動作します：

  • 576px未満（スマホ縦向き）: col-12 → 12カラム幅（全幅）
  • 768px以上（タブレット）: col-md-6 → 6カラム幅（半分の幅、2列表示）
  • 992px以上（PC）: col-lg-4 → 4カラム幅（1/3の幅、3列表示）

4.3 実践例1 - 商品カードのレスポンシブレイアウト

ECサイトでよく見る商品カードのレイアウトを、レスポンシブグリッドで実装してみましょう。

実習: 商品カードグリッドを作ろう

以下のHTMLコードを作成し、ブラウザで表示を確認してみましょう。画面サイズを変えて、レイアウトがどのように変化するか観察してください。

product-grid.html 
<div class="container mt-5">
  <div class="row">
    <!-- 商品カード1 -->
    <div class="col-12 col-sm-6 col-md-4 col-lg-3 mb-4">
      <div class="card">
        <div class="card-body">
          <h5 class="card-title">商品A</h5>
          <p class="card-text">¥1,000</p>
        </div>
      </div>
    </div>
    <!-- 商品カード2〜4も同様 -->
  </div>
</div>

このレイアウトの動作：

画面サイズ 適用クラス カラム幅 表示列数
576px未満 col-12 12カラム 1列（縦並び）
576px〜767px col-sm-6 6カラム 2列
768px〜991px col-md-4 4カラム 3列
992px以上 col-lg-3 3カラム 4列

モバイルファーストの考え方

Bootstrapは「モバイルファースト」設計です。まず小さい画面（スマホ）向けのレイアウトを定義し、画面が大きくなるにつれて段階的にカラム数を増やしていきます。col-12（全幅）を基準に、col-sm-6、col-md-4 と順に指定していくのが一般的なパターンです。

4.4 実践例2 - 2カラムレイアウト（サイドバー付き）

ブログサイトなどでよく見る、メインコンテンツとサイドバーの2カラムレイアウトを作成してみましょう。

sidebar-layout.html 
<div class="container">
  <div class="row">
    <!-- メインコンテンツ -->
    <div class="col-12 col-md-8">
      <h2>メインコンテンツ</h2>
      <p>記事本文がここに入ります。</p>
    </div>

    <!-- サイドバー -->
    <div class="col-12 col-md-4">
      <h3>サイドバー</h3>
      <ul>
        <li>カテゴリー</li>
        <li>最新記事</li>
      </ul>
    </div>
  </div>
</div>

このレイアウトの動作：

  • 768px未満（スマホ）: 両方とも全幅（col-12）で縦に並ぶ
  • 768px以上（タブレット・PC）: メインが8カラム、サイドバーが4カラムで横並び

4.5 カラムの順序変更（Order）

画面サイズによって、カラムの表示順序を変更することもできます。order-* クラスを使用します。

order-example.html 
<div class="container">
  <div class="row">
    <!-- スマホでは2番目、PCでは1番目 -->
    <div class="col-12 col-md-8 order-2 order-md-1">
      <h2>メインコンテンツ</h2>
    </div>

    <!-- スマホでは1番目、PCでは2番目 -->
    <div class="col-12 col-md-4 order-1 order-md-2">
      <h3>サイドバー</h3>
    </div>
  </div>
</div>

この例では、スマホでサイドバーが先に表示され、PCではメインコンテンツが先に表示されます。

便利な使い方

スマホでは「重要な情報を上に」表示し、PCでは「視覚的なバランスを優先」した配置にする、といった柔軟なレイアウトが可能です。

4.6 カラムのオフセット（余白）

カラムの左側に余白を作りたい場合、offset-* クラスを使用します。

offset-example.html 
<div class="container">
  <div class="row">
    <!-- 左に3カラム分の余白、6カラム幅 -->
    <div class="col-6 offset-3">
      <p>中央寄せされたコンテンツ</p>
    </div>
  </div>
</div>

offset-3 は左側に3カラム分の余白を作ります（余白3 + 内容6 + 余白3 = 12カラム）。

クラス 説明 用途例
offset-1 左に1カラム分の余白 微調整
offset-3 左に3カラム分の余白 6カラム幅を中央寄せ
offset-md-2 タブレット以上で左に2カラム分の余白 レスポンシブ対応

4.7 開発者ツールでのレスポンシブ確認

作成したレスポンシブレイアウトが正しく動作しているか、開発者ツールで確認する方法を学びましょう。

動作確認手順

  1. Google Chromeでページを開く
  2. F12キーを押して開発者ツールを開く
  3. デバイスツールバーボタン（スマホとタブレットのアイコン）をクリック、またはCtrl + Shift + M
  4. 上部のデバイス選択で「iPhone SE」「iPad」「Responsive」などを選択
  5. 画面幅を変更しながら、レイアウトの変化を確認

確認すべきポイント

  • 576px未満: カラムが縦に並んでいるか
  • 768px前後: タブレット用のレイアウトに切り替わるか
  • 992px以上: PC用のレイアウトに切り替わるか
  • テキストの読みやすさ: すべての画面サイズで文字が読みやすいか
  • 横スクロールが発生していないか: 画面からはみ出す要素がないか

よくあるミス

ブレークポイントクラスの指定順序を間違えると、意図しない動作になります。必ず「小さい画面 → 大きい画面」の順で指定しましょう（col-12 col-md-6 col-lg-4）。

第4章のまとめ

  • ブレークポイントは画面サイズの区切りで、Bootstrap は5つのサイズ（極小、小、中、大、超大）を提供します
  • レスポンシブグリッドクラス（col-sm-、col-md-、col-lg-等）を組み合わせて、デバイスごとに異なるレイアウトを実現できます
  • モバイルファーストの考え方で、まず小さい画面向けを定義し、画面が大きくなるにつれて段階的に調整します
  • order-* で表示順序を変更、offset-* で余白を作成できます
  • 開発者ツールのデバイスモードを使って、複数の画面サイズで動作確認することが重要です

理解度確認クイズ

  1. Q1. ブレークポイントとは何ですか？Bootstrapではいくつのブレークポイントが定義されていますか？

    回答を表示

    ブレークポイントとは、レスポンシブデザインにおける「画面サイズの区切り」のことです。Bootstrapでは5つのブレークポイントが定義されています：極小（<576px）、小（≥576px）、中（≥768px）、大（≥992px）、超大（≥1200px）。

  2. Q2. class="col-12 col-md-6 col-lg-4" の意味を説明してください。

    回答を表示

    このクラスは、画面サイズに応じて以下のように動作します：
    ・768px未満（スマホ）: 12カラム幅（全幅、1列表示）
    ・768px以上（タブレット）: 6カラム幅（半分の幅、2列表示）
    ・992px以上（PC）: 4カラム幅（1/3の幅、3列表示）

  3. Q3. レスポンシブデザインを確認するために、どのツールを使いますか？

    回答を表示

    Chrome開発者ツールの「デバイスツールバー」機能を使います。F12キーで開発者ツールを開き、Ctrl + Shift + M（またはスマホ/タブレットアイコン）でデバイスモードに切り替えます。様々な画面サイズをシミュレートして、レイアウトの変化を確認できます。
