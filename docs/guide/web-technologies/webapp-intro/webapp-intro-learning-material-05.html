第5章

HTTPプロトコルの基本

HTTP（HyperText Transfer Protocol）の基本を学び、Web通信の仕組みを理解します。

この章で学ぶこと

  • HTTPとは何かを説明できるようになる
  • リクエストとレスポンスの構造を理解する
  • 主要なステータスコードの意味を把握する

5.1 HTTPとは

HTTP（HyperText Transfer Protocol）は、WebブラウザとWebサーバーが通信するための「ルール」（プロトコル）です。 「どのような形式でデータを要求し、どのような形式で返すか」を決めています。

身近な例で考えてみよう

電話での会話を想像してください。「もしもし」で始めて、用件を伝え、「さようなら」で終わる。 これは電話の「プロトコル（決まり事）」です。HTTPも同じで、Web通信の「決まり事」を定めています。

5.2 HTTPリクエストの構造

HTTPリクエストは、いくつかの部分で構成されています。

HTTPリクエストの例

GET /products/123 HTTP/1.1
Host: www.example.com
User-Agent: Chrome/91.0
Accept: text/html

リクエストライン: GET /products/123 HTTP/1.1（メソッド、パス、HTTPバージョン）

ヘッダー: Host、User-Agent、Acceptなどの追加情報

5.3 HTTPレスポンスの構造

HTTPレスポンスの例

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<!DOCTYPE html>
<html>...</html>

ステータスライン: HTTP/1.1 200 OK（HTTPバージョン、ステータスコード、理由句）

ヘッダー: Content-Type、Content-Lengthなどの情報

ボディ: 実際のコンテンツ（HTML、画像など）

5.4 ステータスコード

ステータスコードは、サーバーの処理結果を3桁の数字で表したものです。

コード 分類 意味
200 成功 リクエスト成功
301 リダイレクト 永続的に移動
400 クライアントエラー 不正なリクエスト
404 クライアントエラー ページが見つからない
500 サーバーエラー サーバー内部エラー

覚えておきたい分類

  • 2xx: 成功（OK!）
  • 3xx: リダイレクト（移動したよ）
  • 4xx: クライアントエラー（あなたの問題）
  • 5xx: サーバーエラー（サーバーの問題）

5.5 この章のまとめ

1

HTTPはWebブラウザとサーバーが通信するためのルール

2

リクエストはリクエストライン、ヘッダー、ボディで構成

3

ステータスコードで処理結果が分かる（200=成功、404=見つからない）

理解度確認クイズ

  1. Q1. ステータスコード404は何を意味しますか？

    回答を表示

    「ページが見つからない」という意味です。URLが間違っているか、ページが削除された場合に返されます。

  2. Q2. ステータスコードが5xxで始まる場合、問題はどちら側にありますか？

    回答を表示

    サーバー側に問題があります。5xxはサーバーエラーを意味します。
