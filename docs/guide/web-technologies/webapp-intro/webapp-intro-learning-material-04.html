クライアントとサーバー

Webアプリケーションの基本となるクライアント・サーバーモデルを理解しましょう。

この章で学ぶこと

  • クライアントとサーバーの役割を理解する
  • リクエストとレスポンスの概念を把握する
  • Webサーバーとは何かを説明できるようになる

4.1 クライアント・サーバーモデルとは

クライアント・サーバーモデルは、コンピュータ同士が通信するときの基本的な形です。 「サービスを要求する側」と「サービスを提供する側」に役割を分けて通信します。

クライアント

サービスを要求する側

  • - Webブラウザ
  • - スマートフォンアプリ
  • - デスクトップアプリ

サーバー

サービスを提供する側

  • - Webサーバー
  • - データベースサーバー
  • - メールサーバー

身近な例で考えてみよう

レストランを想像してください。
お客さん（クライアント）が「カレーをください」と注文し、
キッチン（サーバー）が料理を作って提供します。
お客さんは「要求する側」、キッチンは「提供する側」です。

4.2 リクエストとレスポンス

クライアントとサーバーの間でやり取りされるデータは、リクエスト（要求）とレスポンス（応答）と呼ばれます。

リクエストとレスポンスの流れ

sequenceDiagram participant C as クライアント participant S as サーバー C->>S: リクエスト（「このページをください」） Note over S: リクエストを処理 S->>C: レスポンス（ページのデータ） Note over C: 受け取ったデータを表示

リクエスト（Request）

クライアントからサーバーへの「要求」。「このページが欲しい」「このデータを保存して」などの指示を含みます。

レスポンス（Response）

サーバーからクライアントへの「応答」。要求されたデータや、処理の結果を返します。

4.3 Webサーバーとは

Webサーバーは、Webサイトやアプリケーションを公開するための特別なコンピュータ（またはソフトウェア）です。 24時間365日稼働し、世界中からのアクセスに応答します。

Webサーバーの主な機能

静的ファイルの配信

HTML、CSS、画像などをそのまま返す

動的コンテンツの生成

プログラムを実行して、その場でページを生成する

セキュリティ

不正なアクセスからシステムを守る

代表的なWebサーバーソフトウェア

  • Apache: 長い歴史を持つ定番のWebサーバー
  • Nginx: 高速で軽量、最近人気が高い
  • IIS: Microsoft製のWebサーバー

4.4 この章のまとめ

1

クライアントはサービスを要求する側、サーバーは提供する側

2

リクエストはクライアントからの要求、レスポンスはサーバーからの応答

3

WebサーバーはWebサイトを公開するためのコンピュータ/ソフトウェア

理解度確認クイズ

  1. Q1. Webブラウザはクライアント、サーバーのどちらですか？

    回答を表示

    クライアントです。サービス（Webページ）を要求する側だからです。

  2. Q2. クライアントからサーバーへの「要求」を何と呼びますか？

    回答を表示

    リクエスト（Request）と呼びます。
