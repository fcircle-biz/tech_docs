ローカル開発環境

localhost（127.0.0.1）の意味、ポート番号の概念、複数のサーバーを同時に起動する方法について学びます。開発環境の基礎知識を身につけます。

この章で学ぶこと

  • localhost（127.0.0.1）の意味を理解する
  • ポート番号の概念と役割を把握する
  • 複数のサーバーを同時に起動する方法を学ぶ

11.1 localhost（ローカルホスト）とは

localhost（ローカルホスト）は、あなたが今使っているコンピュータ自身を指す特別な名前です。 Webアプリの開発では、インターネット上のサーバーにアクセスする前に、まず自分のパソコン上でアプリを動かして確認します。

身近な例で考えてみよう

localhostは「鏡の前の自分」のようなものです。 料理を作った時に、お客さんに出す前に自分で味見をしますよね。 同じように、Webアプリを公開する前に、自分のパソコン上で動かして確認できます。

localhostには、127.0.0.1という数字のアドレスが割り当てられています。 これは「自分自身」を表す特別なIPアドレスです。

重要ポイント

localhost = 127.0.0.1 = 自分のパソコン
この3つはすべて同じものを指しています。ブラウザで「http://localhost」と入力するのと「http://127.0.0.1」と入力するのは同じ意味です。

11.2 ポート番号とは

1台のコンピュータ上では、複数のアプリケーションが同時に動いています。 それぞれのアプリケーションを区別するために使われるのがポート番号です。

身近な例で考えてみよう

ポート番号は「マンションの部屋番号」のようなものです。 マンション全体が「IPアドレス（localhost/127.0.0.1）」で、各部屋が「ポート番号」です。 郵便物（データ）を配達するとき、マンションの住所だけでなく部屋番号も必要ですよね。

URLとポート番号の関係

URLの形式 
http://localhost:3000/
       ↑        ↑     ↑
    プロトコル ホスト名  ポート番号

ポート番号は「:（コロン）」の後に書きます。上の例では、ポート番号3000で動いているサーバーにアクセスしています。

よく使われるポート番号

80 HTTP

通常のWebサイトのデフォルトポート

443 HTTPS

暗号化されたWebサイトのデフォルトポート

3000 開発用

React、Node.jsなどの開発サーバー

8000 開発用

Django、Pythonの開発サーバー

5432 データベース

PostgreSQLデータベース

3306 データベース

MySQLデータベース

ヒント

ポート番号80と443は、Webサイトにアクセスするときに省略できます。「http://example.com」は実際には「http://example.com:80」の意味で、「https://example.com」は「https://example.com:443」の意味です。

11.3 複数のサーバーを同時に起動する

Webアプリの開発では、フロントエンド、バックエンド、データベースなど、複数のサーバーを同時に起動することがよくあります。 それぞれ異なるポート番号を使えば、同時に動かすことができます。

異なるポート番号を使えば、同時に複数のサーバーを起動できる

典型的な開発環境の例

1
フロントエンド（React）を起動

ターミナル1で実行

コマンド 
npm start
# → http://localhost:3000 で起動
2
バックエンド（FastAPI）を起動

ターミナル2で実行

コマンド 
uvicorn main:app --reload --port 8000
# → http://localhost:8000 で起動
3
データベース（PostgreSQL）を起動

ターミナル3で実行（Dockerを使う場合）

コマンド 
docker run -p 5432:5432 postgres
# → localhost:5432 で起動

重要ポイント

各サーバーは異なるポート番号を使う必要があります。
同じポート番号で2つのサーバーを起動しようとすると、「ポートがすでに使用されています」というエラーになります。

11.4 ポート番号の確認と管理

開発中に「どのポート番号が使われているか」を確認したり、不要なサーバーを停止したりする方法を知っておくと便利です。

使用中のポート番号を確認する方法

Windows
コマンドプロンプト / PowerShell 
# すべての使用中ポートを表示
netstat -ano

# 特定のポート（例: 3000）を使っているプログラムを確認
netstat -ano | findstr :3000
macOS / Linux
ターミナル 
# すべての使用中ポートを表示
lsof -i -P

# 特定のポート（例: 3000）を使っているプログラムを確認
lsof -i :3000

よくあるトラブルと対処法

「ポート番号がすでに使用されています」エラー

原因: 同じポート番号で別のプログラムがすでに起動しています。

解決方法:

  • 上記のコマンドでポート番号を使っているプログラムを特定し、停止する
  • または、別のポート番号を使ってサーバーを起動する（例: --port 3001）
localhostに接続できない

原因: サーバーが起動していないか、間違ったポート番号にアクセスしています。

解決方法:

  • サーバーが正常に起動しているか確認する
  • サーバーの起動ログでポート番号を確認する
  • ブラウザのURLが正しいか確認する（例: http://localhost:3000）
複数のターミナルを開くのが面倒

解決方法: 複数のプロセスを管理するツールを使いましょう。

  • Docker Compose: 複数のコンテナを一度に起動・停止できる
  • Concurrently（Node.js）: 複数のnpmスクリプトを同時実行
  • tmux/screen: ターミナルを分割して表示

11.5 外部からのアクセスについて

localhostは「自分のパソコンだけ」がアクセスできます。 他の人のパソコンやスマートフォンからはアクセスできません。

localhostは自分のパソコン内でのみアクセス可能

注意

開発中のアプリを他の人に見せたい場合は、ngrokなどのトンネリングツールを使うか、クラウド上にデプロイする必要があります。localhostのまま公開しようとしても、他の人には見えません。

ヒント: 同じネットワーク内でのアクセス

自分のパソコンと同じWi-Fiに接続されているスマートフォンからアクセスしたい場合は、localhostの代わりに自分のパソコンのIPアドレス（例: 192.168.1.5）を使います。ただし、サーバーの起動時に「0.0.0.0」で起動する必要があります。

11.6 この章のまとめ

1

localhost（127.0.0.1）は自分のパソコン自身を指す特別な名前

2

ポート番号は同じコンピュータ上で動く複数のアプリケーションを区別するための番号

3

異なるポート番号を使えば、複数のサーバーを同時に起動できる

4

localhostは自分のパソコン内でのみアクセス可能で、外部からは直接アクセスできない

理解度確認クイズ

  1. Q1. localhost、127.0.0.1、自分のパソコン。この3つの関係は？

    回答を表示

    この3つはすべて同じものを指しています。localhostは「自分のパソコン」を表す名前で、127.0.0.1はそのIPアドレスです。ブラウザで「http://localhost」と「http://127.0.0.1」のどちらを入力しても、同じ結果になります。

  2. Q2. ポート番号は何のために使われますか？

    回答を表示

    ポート番号は、1台のコンピュータ上で動く複数のアプリケーション（サーバー）を区別するために使われます。マンションの部屋番号のように、同じIPアドレス（マンション）内でどのサービス（部屋）にアクセスするかを指定します。

  3. Q3. フロントエンド（ポート3000）とバックエンド（ポート8000）を同時に起動できますか？

    回答を表示

    はい、できます。異なるポート番号を使っていれば、同じコンピュータ上で複数のサーバーを同時に起動できます。それぞれ別のターミナルウィンドウで起動し、http://localhost:3000とhttp://localhost:8000のように、ポート番号を指定してアクセスします。
