データ形式(JSON)

現代のWebアプリケーションで広く使われているJSONフォーマットについて学びます。JSONの構造と書き方、なぜAPIでJSONが使われるのかを理解します。

この章で学ぶこと

  • JSONとは何か、なぜ広く使われているのかを理解する
  • JSONの基本的な構造とデータ型を学ぶ
  • 実際のJSONデータの読み方を身につける

7.1 JSONとは何か

JSON（JavaScript Object Notation）は、データをやり取りするときの「書式」の一つです。 コンピュータ同士がデータを送受信する際、人間にも読みやすく、機械にも処理しやすい形式として広く使われています。

身近な例で考えてみよう

JSONは「荷物の送り状」のようなものです。 郵便物を送るときに「送り先」「送り主」「内容物」を決まった書式で書くように、 データも決まった書式で書くことで、誰が読んでもすぐに理解できるようになります。

JSONの特徴

人間が読みやすい

テキスト形式なので、目で見てデータの内容を理解できます

軽量でシンプル

余計な記号が少なく、データサイズが小さくて済みます

言語に依存しない

JavaScript、Python、Java、C#など、ほとんどのプログラミング言語で使えます

Web APIの標準

現代のほとんどのWeb APIがJSONを使用しています

7.2 JSONの基本構造

JSONは主に2つの構造から成り立っています。

オブジェクト

{ }で囲まれた「名前と値のペア」の集まり

{
  "name": "田中太郎",
  "age": 25
}

配列

[ ]で囲まれた「値の順序付きリスト」

[
  "りんご",
  "バナナ",
  "みかん"
]

オブジェクトの書き方

オブジェクトは「キー(名前)」と「値」のペアを:で結び、複数のペアを,で区切ります。

JSON オブジェクトの例 
{
  "name": "田中太郎",
  "age": 25,
  "email": "tanaka@example.com",
  "isStudent": false
}

解説:

  • "name"というキーに"田中太郎"という値が対応
  • "age"というキーに25という数値が対応
  • キーは必ずダブルクォート""で囲む

配列の書き方

配列は複数の値を順番に並べたリストです。値は,で区切ります。

JSON 配列の例 
[
  "りんご",
  "バナナ",
  "みかん",
  "ぶどう"
]

解説:

  • 角括弧[ ]で囲む
  • 値をカンマで区切る
  • 順番が保証される(最初の要素は「りんご」)

7.3 JSONのデータ型

JSONでは、以下の6種類のデータ型を使うことができます。

1

文字列(String)

ダブルクォートで囲まれたテキスト

"こんにちは" "123"
2

数値(Number)

整数または小数点を含む数値

123 3.14 -10
3

真偽値(Boolean)

真(true)か偽(false)の2つの値のみ

true false
4

ヌル(Null)

「値が存在しない」ことを表す特殊な値

null
5

オブジェクト(Object)

キーと値のペアの集まり

{"name": "太郎", "age": 25}
6

配列(Array)

値の順序付きリスト

["りんご", "バナナ", "みかん"]

7.4 実践的なJSONの例

実際のWebアプリケーションでは、オブジェクトと配列を組み合わせて複雑なデータを表現します。

例1: ユーザー情報

ユーザー情報のJSON 
{
  "id": 12345,
  "name": "田中太郎",
  "email": "tanaka@example.com",
  "age": 25,
  "isActive": true,
  "address": {
    "zipCode": "123-4567",
    "prefecture": "東京都",
    "city": "渋谷区"
  }
}

ポイント: "address"の値は、さらにオブジェクトになっています(入れ子構造)

例2: 商品リスト

商品リストのJSON 
{
  "products": [
    {
      "id": 1,
      "name": "ノートパソコン",
      "price": 120000,
      "inStock": true
    },
    {
      "id": 2,
      "name": "ワイヤレスマウス",
      "price": 3500,
      "inStock": false
    },
    {
      "id": 3,
      "name": "外付けキーボード",
      "price": 8000,
      "inStock": true
    }
  ]
}

ポイント: "products"キーの値は配列で、配列の各要素がオブジェクトになっています

例3: APIレスポンス

天気APIのレスポンス 
{
  "location": "東京",
  "date": "2025-01-11",
  "weather": {
    "condition": "晴れ",
    "temperature": 12,
    "humidity": 45
  },
  "forecast": [
    {"day": "明日", "condition": "曇り"},
    {"day": "明後日", "condition": "雨"}
  ]
}

ポイント: 実際のAPIは、このようにオブジェクトと配列を組み合わせて複雑なデータを返します

7.5 JSONを書くときの注意点

よくある間違い

❌ キーをシングルクォートで囲む

{'name': '太郎'}

→ JSONでは必ずダブルクォート"を使う

❌ 最後の要素にカンマをつける

{"name": "太郎", "age": 25,}

→ 最後の要素の後ろにカンマは不要

❌ コメントを書く

// これはコメント

→ JSONにはコメント機能がありません

❌ 未定義の値を使う

{"value": undefined}

→ 値がない場合はnullを使う

正しいJSON

{
  "name": "田中太郎",
  "age": 25,
  "email": "tanaka@example.com",
  "address": null
}
  • キーはダブルクォートで囲む
  • 文字列値もダブルクォートで囲む
  • 最後の要素の後ろにカンマなし
  • 値がない場合はnullを使用

7.6 なぜJSONが広く使われるのか

JSONが現代のWebアプリケーションで標準的に使われる理由を見ていきましょう。

他のデータ形式との比較 (JSON vs XML)

以前はXML(Extensible Markup Language)という形式が使われていましたが、現在はJSONが主流です。

JSON
{
  "name": "田中太郎",
  "age": 25
}
  • シンプルで読みやすい
  • データサイズが小さい
  • JavaScriptと親和性が高い
XML
<person>
  <name>田中太郎</name>
  <age>25</age>
</person>
  • タグが冗長
  • データサイズが大きい
  • 処理が複雑

JSONが活躍する場面

  • Web API - サーバーとクライアントのデータ交換
  • 設定ファイル - アプリケーションの設定を保存
  • NoSQLデータベース - MongoDBなどで直接JSON形式で保存
  • ログファイル - 構造化されたログデータの記録

    回答を表示

    (2) 軽量でデータサイズが小さい
    (3) ほとんどのプログラミング言語で使える

  2. Q2. 次のJSONの間違いを指摘してください:
    {'name': '太郎', 'age': 25,}

    回答を表示

  3. Q3. JSONで使える6つのデータ型を挙げてください。

    回答を表示

    (1) 文字列(String)
    (2) 数値(Number)
    (3) 真偽値(Boolean)
    (4) null
    (5) オブジェクト(Object)
    (6) 配列(Array)
