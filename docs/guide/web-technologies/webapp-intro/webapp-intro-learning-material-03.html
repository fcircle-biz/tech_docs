URLの構造を理解しよう

普段何気なく使っているURLの各部分が何を意味しているのか、詳しく解説します。

この章で学ぶこと

  • URLの各構成要素（プロトコル、ドメイン、パス等）を理解する
  • クエリパラメータの意味と使い方を把握する
  • ポート番号の役割を理解する

3.1 URLとは

URL（Uniform Resource Locator）は、インターネット上のリソース（Webページ、画像、動画など）の「場所」を示す文字列です。 日本語では「統一資源位置指定子」と訳されますが、単に「Webアドレス」と呼ばれることも多いです。

URLの例

https://www.example.com:443/products/item?id=123&color=red#details

このURLには多くの情報が含まれています。一つずつ見ていきましょう。

プロトコル（スキーム）

通信方法を指定します。https://やhttp://が一般的です。

  • http: 暗号化なしの通信
  • https: 暗号化された安全な通信（現在の主流）

ドメイン（ホスト名）

アクセス先のサーバーを指定します。前章で学んだように、DNSによってIPアドレスに変換されます。

ポート番号

サーバー内のどの「窓口」にアクセスするかを指定します。通常は省略されます。

  • 80: HTTPの標準ポート
  • 443: HTTPSの標準ポート

パス

サーバー内のどのリソース（ページ）にアクセスするかを指定します。 パソコンのフォルダ構造のようなものです。

クエリパラメータ

?の後に続く部分。サーバーに追加情報を渡します。

?id=123&color=red

キー=値 の形式で、& で区切って複数指定可能

フラグメント（アンカー）

#の後に続く部分。ページ内の特定の位置を指定します。 目次から特定のセクションにジャンプするときなどに使われます。

身近な例で考えてみよう

URLは「住所」のようなものです。
プロトコル = 交通手段（車、電車など）
ドメイン = 建物名（〇〇ビル）
ポート = 入口（正面玄関、裏口など）
パス = フロアと部屋番号（3階301号室）
クエリ = 訪問目的（「田中さんに会いたい」）

3.3 クエリパラメータの実例

クエリパラメータは、検索サイトやECサイトで特によく使われます。実際のURLを見てみましょう。

例1: Google検索

https://www.google.com/search?q=プログラミング

q=プログラミングは「検索キーワードは"プログラミング"」という意味

例2: ECサイトの商品絞り込み

https://shop.example.com/products?category=shoes&size=26&color=black

カテゴリ=靴、サイズ=26cm、色=黒 で絞り込み

3.4 この章のまとめ

1

URLはインターネット上のリソースの場所を示す文字列

2

URLはプロトコル、ドメイン、ポート、パス、クエリ、フラグメントで構成される

3

クエリパラメータは?の後に続く追加情報で、検索や絞り込みに使われる

4

ポート番号は通常省略されるが、HTTPは80、HTTPSは443がデフォルト

理解度確認クイズ

  1. Q1. URLの「https://」の部分は何を表していますか？

    回答を表示

    プロトコル（通信方法）を表しています。httpsは暗号化された安全な通信を意味します。

  2. Q2. ?name=tanaka&age=25は何を表していますか？

    回答を表示

    クエリパラメータです。「name=tanaka」と「age=25」という2つの情報をサーバーに送っています。

  3. Q3. HTTPSの標準ポート番号は何番ですか？

    回答を表示

    443番です。HTTPは80番がデフォルトです。通常は省略されます。
