ネットワークの基礎知識

IPアドレス、ドメイン名、DNS、TCP/IPなど、インターネット通信の根幹となる技術を初心者向けに解説します。

この章で学ぶこと

  • IPアドレスの役割と種類（IPv4/IPv6）を理解する
  • ドメイン名とDNSの仕組みを理解する
  • TCP/IPの基本概念を把握する

2.1 IPアドレスとは

IPアドレスは、インターネットに接続されたコンピュータを識別するための「住所」です。 私たちの住所が「東京都新宿区〇〇町1-2-3」のように場所を特定するように、 IPアドレスはインターネット上のコンピュータの場所を特定します。

身近な例で考えてみよう

手紙を送るとき、宛先の住所がなければ届きません。 IPアドレスは「インターネット上の住所」です。 データを送りたいとき、相手のIPアドレスがなければ届けることができません。

IPv4アドレス

現在最も広く使われているIPアドレスの形式がIPv4（Internet Protocol version 4）です。 4つの数字をピリオドで区切った形式で表されます。

IPv4アドレスの例

192.168.1.1

各数字は0〜255の範囲です。つまり、約43億通りのアドレスが作れます。

IPv6アドレス

インターネットに接続する機器が増えすぎて、IPv4のアドレスが足りなくなりました。 そこで登場したのがIPv6（Internet Protocol version 6）です。

IPv6アドレスの例

2001:0db8:85a3:0000:0000:8a2e:0370:7334

16進数の数字とコロンで構成されます。事実上無限に近い数のアドレスが作れます。

項目 IPv4 IPv6
アドレス長 32ビット 128ビット
アドレス数 約43億 ほぼ無限
表記例 192.168.1.1 2001:db8::1

2.2 ドメイン名とDNS

IPアドレスは数字の羅列なので、人間には覚えにくいですよね。 そこで登場するのがドメイン名です。

IPアドレスとドメイン名の対応

覚えにくい

142.250.196.110

覚えやすい

www.google.com

どちらも同じWebサイトを指しています

DNS（Domain Name System）とは

DNSは、ドメイン名をIPアドレスに変換する仕組みです。 インターネットの「電話帳」のようなものと考えてください。

DNSの動作

sequenceDiagram participant U as あなた participant B as ブラウザ participant D as DNSサーバー participant W as Webサーバー U->>B: www.example.comにアクセス B->>D: www.example.comのIPアドレスは？ D->>B: 93.184.216.34です B->>W: 93.184.216.34に接続 W->>B: Webページのデータ B->>U: Webページを表示

DNSがドメイン名をIPアドレスに変換してくれる

身近な例で考えてみよう

スマートフォンの連絡先アプリを想像してください。 「田中さん」と検索すれば電話番号が出てきます。 DNSも同じで、「google.com」と入力すれば、対応するIPアドレスを教えてくれます。

2.3 TCP/IPの基本

TCP/IPは、インターネットでデータをやり取りするための「ルール」（プロトコル）です。 正式には「Transmission Control Protocol / Internet Protocol」といいます。

IP（Internet Protocol）

データを目的地まで届ける役割。住所を見て荷物を配達する「配送業者」のようなもの。

TCP（Transmission Control Protocol）

データが正しく届いたか確認する役割。荷物が壊れていないか確認する「品質管理」のようなもの。

データはどのように送られるか

大きなデータは小さな「パケット」に分割されて送られ、受信側で再構成されます。

flowchart LR subgraph 送信側["送信側"] A["元のデータ"] A --> B1["パケット1"] A --> B2["パケット2"] A --> B3["パケット3"] end B1 --> C(("インターネット")) B2 --> C B3 --> C subgraph 受信側["受信側"] C --> D1["パケット1"] C --> D2["パケット2"] C --> D3["パケット3"] D1 --> E["再構成されたデータ"] D2 --> E D3 --> E end

なぜパケットに分けるの？

  • 効率的: 大きなデータを一度に送ると、回線を長時間占有してしまう
  • 信頼性: 一部が失われても、その部分だけ再送すればOK
  • 柔軟性: パケットごとに違う経路を通ることができる

理解度確認クイズ

  1. Q1. IPアドレスの役割は何ですか？

    回答を表示

    インターネットに接続されたコンピュータを識別するための「住所」の役割を果たします。データを送りたいとき、相手のIPアドレスがなければ届けることができません。

  2. Q2. DNSは何をするものですか？

    回答を表示

    ドメイン名（例：www.google.com）をIPアドレス（例：142.250.196.110）に変換する仕組みです。インターネットの「電話帳」のような役割を果たします。

  3. Q3. データをパケットに分割して送る理由を1つ挙げてください。

    回答を表示

    以下のいずれかが正解です：
    (1) 効率的: 回線を長時間占有しない
    (2) 信頼性: 一部が失われても、その部分だけ再送すればOK
    (3) 柔軟性: パケットごとに違う経路を通れる
