第6章

HTTPメソッドを理解しよう

GET、POST、PUT、DELETEの4つの主要なHTTPメソッドについて学びます。それぞれのメソッドがどのような場面で使われるのか、実例を通じて理解を深めます。

この章で学ぶこと

  • HTTPメソッドの役割と種類を理解する
  • GET、POST、PUT、DELETEの使い分けを学ぶ
  • 各メソッドの実際の使用例を把握する

6.1 HTTPメソッドとは

HTTPメソッドは、クライアント（ブラウザ）がサーバーに「何をしたいか」を伝えるための指示のことです。 第5章で学んだHTTPリクエストの先頭行に含まれていました。

身近な例で考えてみよう

HTTPメソッドは「注文の仕方」のようなものです。 レストランで店員さんに「見せてください（GET）」「注文します（POST）」「変更してください（PUT）」「取り消してください（DELETE）」と伝えるのと同じように、 ブラウザもサーバーに対して「何をしたいか」を明確に伝える必要があります。

HTTPメソッドには様々な種類がありますが、現代のWebアプリケーション開発では主に以下の4つが使われています。

GET

データを取得する（読み取り専用）

POST

新しいデータを作成する

PUT

既存のデータを更新する

DELETE

既存のデータを削除する

ポイント: CRUDとの対応

これらの4つのメソッドは、データベース操作の基本であるCRUDに対応しています。

Create（作成） → POST

Read（読み取り） → GET

Update（更新） → PUT

Delete（削除） → DELETE

6.2 GETメソッド - データを取得する

GETは、サーバーからデータを取得するためのメソッドです。 私たちが普段Webサイトを閲覧しているときは、ほとんどGETメソッドを使っています。

GETメソッドの特徴

1
データを取得するだけ

サーバーのデータを変更しません。何度実行しても同じ結果が返ってきます（安全性）。

2
URLにパラメータを含む

検索条件などのデータはURLのクエリパラメータとして送られます。

3
ブックマーク可能

URLに情報が含まれているため、ブックマークや共有がしやすいです。

GETリクエストの例 
GET /products?category=books&sort=price HTTP/1.1
Host: example.com

この例では、書籍カテゴリの商品を価格順に取得するリクエストです。 検索条件はURLの?の後ろに付いています。

実際の使用例

  • • Webページの閲覧
  • • 検索結果の表示
  • • 商品リストの取得
  • • ニュース記事の読み込み

6.3 POSTメソッド - データを作成する

POSTは、サーバーに新しいデータを作成するためのメソッドです。 フォームの送信や、新規登録などで使われます。

POSTメソッドの特徴

1
データを作成する

サーバーに新しいデータを追加します。実行するたびに新しいデータが作られます。

2
データはボディに含む

送信するデータはリクエストボディ（本文）に含まれるため、URLには表示されません。

3
機密情報を送れる

パスワードなどの機密情報もURLに表示されないため、比較的安全に送信できます。

POSTリクエストの例 
POST /users HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "田中太郎",
  "email": "tanaka@example.com"
}

この例では、新しいユーザーを作成するリクエストです。 ユーザー情報はリクエストボディにJSON形式で含まれています。

実際の使用例

  • • ユーザー登録
  • • ブログ記事の投稿
  • • コメントの投稿
  • • 注文の確定

6.4 PUTメソッド - データを更新する

PUTは、サーバーの既存のデータを更新するためのメソッドです。 プロフィール編集や設定変更などで使われます。

PUTメソッドの特徴

1
データを更新する

既存のデータを新しい内容で上書きします。

2
対象を特定する

URLに更新対象のIDなどを含めて、どのデータを更新するかを明示します。

3
冪等性（べきとうせい）

同じリクエストを何度実行しても、結果は同じです。

PUTリクエストの例 
PUT /users/123 HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "田中太郎",
  "email": "tanaka.new@example.com"
}

この例では、ID「123」のユーザー情報を更新するリクエストです。 URLに/users/123と指定することで、更新対象を特定しています。

実際の使用例

  • • プロフィール情報の編集
  • • ブログ記事の更新
  • • 設定の変更
  • • パスワードの変更

6.5 DELETEメソッド - データを削除する

DELETEは、サーバーのデータを削除するためのメソッドです。 アカウント削除やデータの取り消しなどで使われます。

DELETEメソッドの特徴

1
データを削除する

指定されたデータをサーバーから削除します。

2
対象を特定する

URLに削除対象のIDを含めて、どのデータを削除するかを明示します。

3
取り消せない操作

一度削除すると元に戻せないため、実装時には確認ダイアログを表示することが多いです。

DELETEリクエストの例 
DELETE /users/123 HTTP/1.1
Host: example.com

この例では、ID「123」のユーザーを削除するリクエストです。 シンプルにURLで削除対象を指定するだけで、ボディは通常不要です。

注意

DELETEメソッドは不可逆的な操作です。実際のWebアプリケーションでは、削除前に必ず確認メッセージを表示したり、 「論理削除」（データを実際には削除せず、削除フラグだけを立てる）を使うことが一般的です。

実際の使用例

  • • アカウントの削除
  • • 投稿の削除
  • • カート内商品の削除
  • • ブックマークの解除

6.6 HTTPメソッドの使い分け

それぞれのHTTPメソッドには明確な役割があります。正しいメソッドを選ぶことで、 Webアプリケーションの設計がわかりやすく、保守しやすくなります。

実例: ブログシステムでのHTTPメソッド使用例

flowchart TD A["ユーザー操作"] --> B{"何をする?"} B -->|記事を読む| C["GET /posts/123"] B -->|新しい記事を書く| D["POST /posts"] B -->|記事を編集する| E["PUT /posts/123"] B -->|記事を削除する| F["DELETE /posts/123"] C --> G["記事データを表示"] D --> H["新規記事を保存"] E --> I["既存記事を更新"] F --> J["記事を削除"]

操作の種類によって適切なHTTPメソッドを選択する

HTTPメソッドの選び方

6.7 この章のまとめ

1

HTTPメソッドは、クライアントがサーバーに「何をしたいか」を伝えるための指示

2

GETはデータを取得、POSTは新規作成、PUTは更新、DELETEは削除

3

これらはデータベース操作のCRUD（Create、Read、Update、Delete）に対応している

4

適切なメソッドを選ぶことで、Webアプリケーションの設計が明確になる

理解度確認クイズ

  1. Q1. Webサイトで商品リストを表示するとき、どのHTTPメソッドを使うべきですか？

    回答を表示

    GETを使います。商品リストの表示は「データを取得する」操作なので、GETメソッドが適切です。GETはサーバーのデータを変更しない安全な操作です。

  2. Q2. 新しくユーザー登録をするとき、どのHTTPメソッドを使うべきですか？

    回答を表示

    POSTを使います。ユーザー登録は「新しいデータを作成する」操作なので、POSTメソッドが適切です。登録情報はリクエストボディに含まれます。

  3. Q3. CRUDのUは何を意味し、どのHTTPメソッドに対応しますか？

    回答を表示

    UはUpdate（更新）を意味し、PUTメソッドに対応します。既存のデータを新しい内容で上書きするときに使用します。
