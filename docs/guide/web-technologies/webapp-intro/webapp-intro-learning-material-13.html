開発者ツールでWebを観察しよう

ブラウザの開発者ツール（DevTools）の使い方を学びます。Networkタブでリクエスト/レスポンスを確認し、実際のWeb通信を観察する方法を習得します。

この章で学ぶこと

  • 開発者ツール（DevTools）を開き、基本的な操作ができるようになる
  • Networkタブでリクエストとレスポンスを確認する方法を理解する
  • 実際のWebサイトの通信内容を観察し、学んだ知識を実践できる

13.1 開発者ツールとは

開発者ツール（DevTools: Developer Tools）は、Webブラウザに標準搭載されている、Web開発者向けのツール群です。 このツールを使うと、Webページの裏側で何が起きているのか、詳しく観察することができます。

開発者ツールでできること

Network（ネットワーク）

HTTPリクエストとレスポンスの詳細を確認できる

Elements（要素）

HTMLとCSSの構造を確認・編集できる

Console（コンソール）

JavaScriptの実行結果やエラーを確認できる

Performance（パフォーマンス）

ページの読み込み速度を測定できる

この章では、特にNetworkタブを中心に学びます。 Networkタブを使うと、これまで学んできたHTTPリクエスト、レスポンス、HTTPメソッド、ステータスコードなどを実際に見ることができます。

13.2 開発者ツールを開く方法

開発者ツールの開き方は、ブラウザによって少し異なります。ここでは主要なブラウザでの開き方を紹介します。

Google Chrome

方法1: キーボードショートカット

  • Windows/Linux: F12 または Ctrl + Shift + I
  • Mac: Command + Option + I

方法2: メニューから

右上の「︙」→「その他のツール」→「デベロッパーツール」

Firefox

方法1: キーボードショートカット

  • Windows/Linux: F12 または Ctrl + Shift + I
  • Mac: Command + Option + I

方法2: メニューから

右上の「≡」→「ウェブ開発」→「開発ツールを表示」

Microsoft Edge

方法1: キーボードショートカット

  • Windows: F12 または Ctrl + Shift + I
  • Mac: Command + Option + I

方法2: メニューから

右上の「…」→「その他のツール」→「開発者ツール」

ヒント

最も簡単なのはF12キーを押すことです。 どのブラウザでも共通して使えるので、覚えておくと便利です。

13.4 Networkタブの使い方

Networkタブは、Webページが読み込まれる際に発生するすべての通信を記録します。 どのファイルがダウンロードされたか、どれくらい時間がかかったか、HTTPステータスコードは何かなど、詳細な情報を確認できます。

実習: Networkタブを開いてみよう

  1. 1 開発者ツールを開く（F12キー）
  2. 2 タブバーの中から「Network」タブをクリック
  3. 3 Webページを再読み込み（F5キーまたはCtrl+R）
  4. 4 リクエストの一覧が表示されることを確認

Networkタブで見られる情報

項目 説明
Name リクエストされたファイル名やURL
Status HTTPステータスコード（200、404、500など）
Type ファイルの種類（document、script、stylesheet、imageなど）
Initiator リクエストを開始したもの（どこからの要求か）
Size ファイルサイズ
Time ダウンロードにかかった時間

ヒント

Networkタブは、ページを開いた後に発生した通信のみを記録します。 既に開いているページの通信を確認したい場合は、ページを再読み込みしてください。

13.5 リクエストの詳細を確認する

Networkタブのリクエスト一覧から、特定のリクエストをクリックすると、より詳細な情報を確認できます。

リクエスト詳細パネルのタブ

Headers（ヘッダー）

リクエストとレスポンスのHTTPヘッダーを確認できます。

確認できる情報：

  • Request URL（リクエストURL）
  • Request Method（GETやPOSTなど）
  • Status Code（ステータスコード）
  • Request Headers（リクエストヘッダー）
  • Response Headers（レスポンスヘッダー）
Preview（プレビュー）

レスポンスの内容を見やすく整形して表示します。 JSONデータは階層構造で表示され、画像は画像として表示されます。

Response（レスポンス）

サーバーから返された生のレスポンスデータを表示します。 HTMLやJSONなどのテキストデータをそのまま確認できます。

Timing（タイミング）

リクエストの各段階にかかった時間を詳細に表示します。 どこで時間がかかっているのか分析できます。

実習: リクエストの詳細を確認してみよう

  1. 1 Networkタブで、一覧の中からHTMLファイル（最初のリクエスト）をクリック
  2. 2 Headersタブで、Request Method（GETなど）とStatus Code（200など）を確認
  3. 3 Responseタブで、サーバーから返されたHTMLを確認
  4. 4 他のリクエスト（画像、CSS、JavaScriptなど）もクリックして確認してみよう

13.6 ステータスコードを確認する

Networkタブでは、各リクエストのHTTPステータスコードが一目で分かります。 第5章で学んだステータスコードが、実際のWebサイトでどのように使われているか確認してみましょう。

ステータスコードの見方

200

成功（2xx系）

リクエストが正常に処理された

304

リダイレクト（3xx系）

キャッシュが使われた、または別のURLへの転送

404

クライアントエラー（4xx系）

リクエストに問題がある（ページが存在しないなど）

500

サーバーエラー（5xx系）

サーバー側で問題が発生した

実習: 404エラーを意図的に発生させてみよう

  1. 1 Networkタブを開いた状態で、存在しないページのURLを入力（例: https://example.com/not-found-page）
  2. 2 Networkタブで、ステータスコード「404」が表示されることを確認
  3. 3 404のリクエストをクリックして、Responseタブで404エラーページの内容を確認

13.7 HTTPメソッドを確認する

Networkタブでは、各リクエストで使われているHTTPメソッド（GET、POST、PUT、DELETEなど）も確認できます。

メソッドの確認方法

1

一覧で確認

Networkタブのリクエスト一覧に「Method」列がある場合、そこで各リクエストのメソッドを確認できます。 （列が表示されていない場合は、列のヘッダーを右クリックして「Method」にチェックを入れます）

2

詳細パネルで確認

リクエストをクリックして詳細パネルを開き、Headersタブの「Request Method」欄で確認できます。

実際のWebサイトでの使われ方

GET: ページを開く、画像を読み込むなど、ほとんどのリクエストはGET

POST: フォーム送信（ログイン、コメント投稿など）、ファイルアップロード

PUT/DELETE: 管理画面やAPIで使われる（一般的なWebページではあまり見ない）

実習: ログインフォームでPOSTメソッドを確認してみよう

  1. 1 Networkタブを開いた状態で、ログインフォームがあるサイト（例: Twitter、Facebook）にアクセス
  2. 2 ログインフォームに情報を入力して送信（実際にログインする必要はありません）
  3. 3 Networkタブで、POSTメソッドのリクエストが発生したことを確認
  4. 4 POSTリクエストをクリックして、Headersタブの「Form Data」欄で送信されたデータを確認（パスワードは暗号化または非表示になっています）

13.8 APIリクエストを観察する

最近のWebアプリケーションは、JavaScriptを使ってバックグラウンドでAPIと通信しています。 Networkタブを使うと、これらのAPIリクエストも観察できます。

APIリクエストの特徴

Type: fetch または xhr

Typeがfetchやxhrのリクエストは、JavaScriptから発行されたAPIリクエストです

レスポンスはJSON形式

Previewタブで、JSONデータが階層構造で見やすく表示されます

ページ遷移なしで実行

ページが再読み込みされずに、データだけが取得されます

実習: TwitterやFacebookでAPIリクエストを観察してみよう

  1. 1 Networkタブを開いた状態で、Twitter（X）やFacebookなどのSNSにアクセス
  2. 2 ページをスクロールすると、新しい投稿が読み込まれる（無限スクロール）
  3. 3 Networkタブで、Typeがfetchやxhrのリクエストが発生することを確認
  4. 4 リクエストをクリックして、PreviewタブでJSON形式のレスポンスを確認

重要なポイント

現代のWebアプリは、ページを開いた後も継続的にAPIと通信しています。 これにより、ページ遷移なしでコンテンツを更新できる「動的なWebアプリ」が実現されています。 Networkタブを使えば、この裏側の通信を観察できます。

13.9 フィルター機能を使いこなす

大きなWebサイトでは、数十〜数百のリクエストが発生します。 Networkタブには、特定の種類のリクエストだけを表示するフィルター機能があります。

主なフィルター

All

すべて

Fetch/XHR

API通信

JS

JavaScript

CSS

スタイルシート

Img

画像

Media

動画・音声

Font

フォント

Doc

HTML

活用のヒント

APIの動作を確認したい: 「Fetch/XHR」フィルターを使う

ページが重い原因を調べたい: 「Img」や「Media」で大きなファイルがないか確認

エラーの原因を探したい: Status列で赤色（4xx、5xx）のリクエストを探す

13.10 開発者ツールを活用した学習

開発者ツールは、プログラミングを学ぶための強力な学習ツールでもあります。 実際のWebサイトがどう作られているか、どう動いているか、自分の目で確認できます。

開発者ツールを使った学習方法

1

気になるWebサイトを分析する

「このサイトはどうやって動いているんだろう？」と思ったら、Networkタブで通信を観察してみましょう。

2

REST APIの実例を見る

TwitterやGitHubなどのサービスでAPIリクエストを観察し、URLの設計やJSONの構造を学びましょう。

3

エラーを理解する

404や500エラーが出たとき、Networkタブで何が起きたのか確認する習慣をつけましょう。

4

自分の作品をテストする

将来、自分でWebアプリを作るときは、Networkタブで動作確認しながら開発しましょう。

プロの開発者も毎日使っています

開発者ツールは、初心者だけでなくプロの開発者も毎日使う必須ツールです。 今のうちから使い慣れておくと、将来の開発作業が格段に楽になります。

理解度確認クイズ

  1. Q1. 開発者ツールを開くための最も簡単なショートカットキーは何ですか？

    回答を表示

    F12キーです。Chrome、Firefox、Edgeなど、ほとんどのブラウザで共通して使えます。

  2. Q2. HTTPリクエストとレスポンスを確認するには、開発者ツールのどのタブを使いますか？

    回答を表示

    Networkタブです。このタブでは、ページが読み込まれる際のすべての通信を確認できます。

  3. Q3. Networkタブで、JavaScriptから発行されたAPI通信だけを表示したい場合、どのフィルターを使いますか？

    回答を表示

    Fetch/XHRフィルターを使います。これにより、API通信だけを絞り込んで表示できます。
