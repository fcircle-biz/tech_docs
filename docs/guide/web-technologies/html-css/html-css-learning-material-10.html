ポジショニング - 要素の配置をコントロール

positionプロパティ（static、relative、absolute、fixed、sticky）を使った要素の配置方法、z-indexによる重なり順の制御について学びます。ヘッダーの固定やポップアップの実装方法を理解します。

この章で学ぶこと

  • positionプロパティの5つの値（static, relative, absolute, fixed, sticky）を理解する
  • 要素の配置位置を正確にコントロールする方法を習得する
  • z-indexを使って要素の重なり順を制御する方法を学ぶ
  • 固定ヘッダーやポップアップなど実用的なレイアウトを実装できるようになる

10.1 positionプロパティとは

positionプロパティは、要素をページ上のどこに配置するかを決める重要なプロパティです。通常の配置ルールから外れて、要素を自由な位置に配置したり、スクロールしても固定したりできます。

positionには5つの主要な値があり、それぞれ異なる配置方法を提供します。

位置指定プロパティ

positionをstatic以外に設定すると、top、right、bottom、leftプロパティで具体的な位置を指定できます。

10.2 position: static（デフォルト）

position: staticは、すべての要素のデフォルト値です。要素は通常のドキュメントフローに従って配置され、top、right、bottom、leftの指定は無視されます。

static-example.css 
/* デフォルトの配置（明示的に指定する必要はありません） */
.box {
    position: static; /* これがデフォルト値 */
    /* top, right, bottom, leftは効きません */
}

特別な配置が不要な場合は、positionプロパティを指定する必要はありません。

10.3 position: relative（相対配置）

position: relativeは、要素を「本来あるべき位置」を基準にして、相対的にずらして配置します。元の場所はそのまま確保されたままになります。

relative-example.html 
<style>
  .box {
    width: 100px;
    height: 100px;
    background-color: #3b82f6;
    margin: 10px;
  }

  /* 元の位置から右に30px、下に20px移動 */
  .box-relative {
    position: relative;
    top: 20px;    /* 上から20px下にずらす */
    left: 30px;   /* 左から30px右にずらす */
  }
</style>

<div class="box">通常の要素</div>
<div class="box box-relative">相対配置</div>
<div class="box">通常の要素</div>

重要ポイント

position: relativeは、他の要素の配置に影響を与えません。移動した要素が他の要素と重なることもあります。また、relativeは子要素のabsolute配置の基準点としてもよく使われます。

10.4 position: absolute（絶対配置）

position: absoluteは、要素を通常のドキュメントフローから完全に取り出し、「positionがstaticでない最も近い祖先要素」を基準に絶対位置で配置します。

absolute-example.html 
<style>
  /* 親要素にrelativeを指定して基準点にする */
  .container {
    position: relative;
    width: 300px;
    height: 200px;
    background-color: #e0e7ff;
    border: 2px solid #6366f1;
  }

  /* 親要素の右上に配置 */
  .badge {
    position: absolute;
    top: 10px;
    right: 10px;
    background-color: #ef4444;
    color: white;
    padding: 5px 10px;
  }
</style>

<div class="container">
  <p>コンテンツ</p>
  <div class="badge">NEW</div>
</div>

基準点の選び方

position: absoluteの要素は、以下の順序で基準点を探します：
1. 親要素にposition（static以外）があればそれを基準
2. なければさらに上の祖先要素を探す
3. 最終的にはブラウザウィンドウ全体が基準になる

10.5 position: fixed（固定配置）

position: fixedは、要素をブラウザウィンドウ（ビューポート）を基準に固定配置します。ページをスクロールしても、要素は常に同じ位置に表示され続けます。

fixed-header.css 
/* 固定ヘッダーの例 */
.fixed-header {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    background-color: #1e40af;
    color: white;
    padding: 20px;
    z-index: 1000; /* 他の要素より前面に表示 */
}

つまずきポイント

固定ヘッダーを使う場合、ヘッダーの下にあるコンテンツが隠れてしまうことがあります。bodyやmainにpadding-topを追加して、ヘッダーの高さ分のスペースを確保しましょう。

固定配置は、ヘッダー、ナビゲーションメニュー、「ページトップへ戻る」ボタンなどでよく使われます。

10.6 position: sticky（粘着配置）

position: stickyは、relativeとfixedのハイブリッドです。スクロール位置に応じて、通常配置と固定配置を自動的に切り替えます。

sticky-example.css 
/* テーブルのヘッダーを固定する例 */
.sticky-header {
    position: sticky;
    top: 0; /* 画面上端からの距離 */
    background-color: #f1f5f9;
    font-weight: bold;
}

/* サイドバーを固定する例 */
.sticky-sidebar {
    position: sticky;
    top: 20px; /* 上から20pxの位置で固定 */
}

stickyの動作

stickyは、指定した位置（例: top: 0）にスクロールで到達すると固定され、親要素の範囲を超えると固定が解除されます。テーブルのヘッダー行やサイドバーナビゲーションでよく使われます。

10.7 z-index - 重なり順の制御

z-indexプロパティは、要素の重なり順（前後関係）を制御します。position（static以外）が指定された要素に対してのみ有効です。

z-index-example.css 
/* 値が大きいほど前面に表示される */
.layer-back {
    position: relative;
    z-index: 1; /* 背面 */
}

.layer-middle {
    position: relative;
    z-index: 2; /* 中間 */
}

.layer-front {
    position: relative;
    z-index: 3; /* 前面 */
}

重要ポイント

一般的なz-indexの使い分け：
• 通常のコンテンツ: 0〜10
• ドロップダウンメニュー: 100〜500
• モーダル・オーバーレイ: 1000〜5000
• トーストメッセージ: 10000以上

実習 10-1: 固定ヘッダーの実装

スクロールしても常に画面上部に表示される固定ヘッダーを作成してみましょう。

手順

  1. 1 HTMLファイルを作成し、headerタグとmainタグを用意する
  2. 2 headerにposition: fixedを設定し、top: 0、width: 100%を指定
  3. 3 z-index: 1000を設定して、他の要素より前面に表示されるようにする
  4. 4 mainにpadding-topを追加して、ヘッダーの高さ分のスペースを確保
  5. 5 長いコンテンツを追加して、スクロールしてもヘッダーが固定されることを確認

実習 10-2: モーダルウィンドウの作成

absoluteとz-indexを使って、中央に表示されるモーダルウィンドウを作成してみましょう。

手順

  1. 1 背景オーバーレイ用のdivを作成し、position: fixed、画面全体を覆うサイズ、半透明の黒背景を設定
  2. 2 モーダルコンテンツ用のdivを作成し、position: fixedを設定
  3. 3 モーダルを画面中央に配置（top: 50%、left: 50%、transform: translate(-50%, -50%)）
  4. 4 z-indexを適切に設定（オーバーレイ: 1000、モーダル: 1001）
  5. 5 閉じるボタンを追加し、JavaScriptで表示/非表示を切り替えられるようにする

トラブルシューティング

absoluteで配置した要素が期待した位置に表示されない

原因: 親要素にpositionが設定されていないため、意図しない祖先要素を基準にしている可能性があります。

解決方法: 基準にしたい親要素にposition: relativeを追加してください。

z-indexを設定しても重なり順が変わらない

原因: z-indexはposition（static以外）が設定された要素にのみ有効です。

解決方法: 対象の要素にposition: relative（または他のposition値）を追加してください。

stickyが効かない

原因: 親要素にoverflow: hiddenやoverflow: autoが設定されていると、stickyが機能しません。

解決方法: 親要素のoverflow設定を確認し、必要に応じて削除するか、構造を見直してください。

理解度確認クイズ

  1. Q1. position: relativeとposition: absoluteの違いは何ですか？

    回答を表示

    relativeは、要素の本来の位置を基準に相対的にずらして配置します。元の場所は確保されたままです。absoluteは、要素を通常のフローから完全に取り出し、positionが指定された親要素（またはブラウザウィンドウ）を基準に絶対位置で配置します。元の場所は詰められます。

  2. Q2. 固定ヘッダーを実装する際、下のコンテンツが隠れないようにするにはどうすればよいですか？

    回答を表示

    bodyやmainタグにpadding-topを追加して、ヘッダーの高さ分のスペースを確保します。例えば、ヘッダーの高さが60pxなら、padding-top: 60pxを設定します。

  3. Q3. z-indexを使う際の注意点は何ですか？

    回答を表示

    z-indexは、position: static以外の要素にのみ有効です。また、値が大きいほど前面に表示されます。モーダルやオーバーレイなど、常に最前面に表示したい要素には大きな値（1000以上）を使うと良いでしょう。

  4. Q4. position: stickyはどのような場面で使うと効果的ですか？

    回答を表示

    テーブルのヘッダー行やサイドバーナビゲーションなど、スクロール時に一定の位置まで来たら固定したい要素に使うと効果的です。fixedと違い、親要素の範囲内でのみ固定されるため、セクションごとに異なるヘッダーを固定する場合などに便利です。

第10章のまとめ

  • positionプロパティには5つの値（static, relative, absolute, fixed, sticky）があり、それぞれ異なる配置方法を提供する
  • relativeは元の位置を基準に相対的に配置し、absoluteは親要素を基準に絶対位置で配置する
  • fixedはブラウザウィンドウを基準に固定配置し、stickyはスクロールに応じて通常配置と固定配置を切り替える
  • z-indexは要素の重なり順を制御し、position（static以外）が設定された要素にのみ有効
  • 固定ヘッダー、モーダルウィンドウ、サイドバーなど、実用的なレイアウトパターンを実装できる
