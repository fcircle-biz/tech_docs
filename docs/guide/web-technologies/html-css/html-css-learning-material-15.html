第15章

実践プロジェクト - シンプルなポートフォリオサイト作成

これまで学んだ知識を総合して、自己紹介やプロフィールを掲載したポートフォリオサイトを作成します。企画から公開までの流れを体験し、実践的なスキルを身につけます。

この章で学ぶこと

  • ポートフォリオサイトの設計と企画を行う方法を習得する
  • これまで学んだHTML/CSSの知識を統合してWebサイトを作成する
  • レスポンシブデザインを実装し、さまざまなデバイスに対応する
  • Webサイトを公開するまでのプロセスを理解する

15.1 ポートフォリオサイトとは

ポートフォリオサイトは、自分の経歴、スキル、作品などを掲載して、自己紹介やアピールを行うためのWebサイトです。特にクリエイターやエンジニアにとって、自分の実力を示すための重要なツールとなっています。

このプロジェクトでは、これまで学んだHTML/CSSの技術をすべて活用して、実際に使えるポートフォリオサイトを作成します。

ポートフォリオサイトの主な要素

セクション 内容 使用する技術
ヘッダー ナビゲーションメニュー Flexbox、固定配置
ヒーローセクション キャッチコピー、プロフィール画像 グラデーション、アニメーション
About（自己紹介） 経歴、スキル、強み テキストスタイリング、リスト
Works（作品） プロジェクト、制作物 Grid、カード、画像
Contact（お問い合わせ） 連絡先、フォーム、SNSリンク フォーム、ボタンスタイル
フッター コピーライト、補足情報 テキスト配置、背景色

ポートフォリオサイトの目的

ポートフォリオサイトは、あなたの「デジタル名刺」です。採用担当者やクライアントが最初に見るものとして、あなたのスキルや個性を効果的に伝える必要があります。

15.2 サイト設計と企画

コードを書き始める前に、まずはサイト全体の設計と企画を行います。どのような情報を掲載するか、どんなデザインにするかを事前に考えることで、効率的に制作を進めることができます。

サイト設計のステップ

1. 目的とターゲットの明確化

まず、サイトの目的（就職活動、フリーランス営業、作品公開など）と、誰に見てもらいたいか（採用担当者、クライアント、同業者など）を明確にします。

2. 必要なセクションの洗い出し

ポートフォリオサイトに含めるセクションを決定します。一般的には以下のセクションが含まれます：

  • ヘッダー（ナビゲーション）
  • ヒーローセクション（メインビジュアル、キャッチコピー）
  • About（自己紹介、スキル）
  • Works（作品・プロジェクト）
  • Contact（お問い合わせ、SNSリンク）
  • フッター

3. コンテンツの準備

各セクションに掲載する実際のテキスト、画像、リンクなどを準備します。後から変更できますが、ある程度準備しておくとイメージがつかみやすくなります。

4. デザインの方向性決定

シンプル、カラフル、ミニマル、ダークテーマなど、サイト全体のデザインの方向性を決めます。自分の個性や業界の雰囲気に合わせて選びましょう。

ヒント

参考になるポートフォリオサイトを複数見て、気に入ったデザインや構成をメモしておきましょう。完全にコピーするのではなく、良いところを自分のサイトに取り入れるイメージです。

15.3 HTMLでの骨組み作成

設計が終わったら、HTMLで全体の骨組みを作成します。まずはCSSを適用せず、構造だけを作ることで、コンテンツの流れを確認できます。

index.html（基本構造） 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Portfolio - あなたの名前</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <!-- ヘッダー -->
    <header>
        <nav>
            <a href="#home">Home</a>
            <a href="#about">About</a>
            <a href="#works">Works</a>
            <a href="#contact">Contact</a>
        </nav>
    </header>
index.html（続き - メインセクション） 
    <!-- ヒーローセクション -->
    <section id="home">
        <h1>あなたの名前</h1>
        <p>Web Developer</p>
    </section>

    <!-- 自己紹介セクション -->
    <section id="about">
        <h2>About Me</h2>
        <p>自己紹介文をここに記載します。</p>
        <h3>Skills</h3>
        <ul>
            <li>HTML</li>
            <li>CSS</li>
            <li>JavaScript</li>
        </ul>
    </section>

重要ポイント

id属性を使うことで、ナビゲーションのリンクをクリックしたときに対応するセクションにスムーズにスクロールできます（ページ内リンク）。

15.4 CSSでのスタイリング

HTMLで骨組みができたら、CSSでデザインを整えます。全体の配色、フォント、レイアウトを統一することで、プロフェッショナルな印象を与えることができます。

全体のスタイル設定

style.css（基本スタイル） 
/* リセットとベーススタイル */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

body {
    font-family: 'Arial', sans-serif;
    line-height: 1.6;
    color: #333;
}

/* ヘッダー（固定ナビゲーション） */
header {
    position: fixed;
    top: 0;
    width: 100%;
    background: #ffffff;
    box-shadow: 0 2px 5px rgba(0,0,0,0.1);
    z-index: 100;
}

ヒーローセクションのスタイリング

style.css（ヒーローセクション） 
/* ヒーローセクション */
#home {
    height: 100vh;
    display: flex;
    flex-direction: column;
    justify-content: center;
    align-items: center;
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    color: white;
    text-align: center;
}

#home h1 {
    font-size: 3rem;
    margin-bottom: 1rem;
    animation: fadeIn 1s ease-in;
}

ヒント

グラデーション背景や、簡単なアニメーション（フェードイン）を追加することで、動きのある魅力的なサイトになります。第14章で学んだアニメーション技術を活用しましょう。

カードレイアウトの作成

作品を紹介するWorksセクションでは、CSS Gridを使ってカードレイアウトを作成します。

style.css（Worksセクション） 
/* Worksセクション */
.works-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
    gap: 2rem;
    padding: 2rem;
}

.work-card {
    background: white;
    border-radius: 8px;
    box-shadow: 0 4px 6px rgba(0,0,0,0.1);
    overflow: hidden;
    transition: transform 0.3s;
}

.work-card:hover {
    transform: translateY(-10px);
}

15.5 レスポンシブ対応

ポートフォリオサイトは、スマートフォンやタブレットなど、さまざまなデバイスで閲覧されます。第13章で学んだレスポンシブデザインの技術を使って、すべてのデバイスで快適に閲覧できるようにします。

メディアクエリの活用

style.css（レスポンシブ対応） 
/* タブレット以下 */
@media (max-width: 768px) {
    #home h1 {
        font-size: 2rem;
    }

    nav {
        flex-direction: column;
        gap: 1rem;
    }

    .works-grid {
        grid-template-columns: 1fr;
    }
}

/* スマートフォン */
@media (max-width: 480px) {
    #home h1 {
        font-size: 1.5rem;
    }
}

動作確認

ブラウザの開発者ツール（F12キー）を使って、さまざまな画面サイズでサイトを確認しましょう。特にスマートフォンサイズでの見やすさをチェックしてください。

確認すべきポイント

  • 文字が小さすぎないか
  • タップしやすいボタンサイズになっているか
  • 横スクロールが発生していないか
  • 画像が画面からはみ出していないか

実習 15-1: ポートフォリオサイトを作成しよう

これまで学んだすべての知識を活用して、あなた自身のポートフォリオサイトを作成してみましょう。

手順

  1. 1 プロジェクトフォルダを作成し、index.htmlとstyle.cssを用意する
  2. 2 HTMLで基本構造を作成する（ヘッダー、ヒーロー、About、Works、Contact、フッター）
  3. 3 CSSで全体のスタイルを設定する（配色、フォント、レイアウト）
  4. 4 Flexbox/Gridを使ってレイアウトを作成する
  5. 5 メディアクエリでレスポンシブ対応を実装する
  6. 6 アニメーションやホバーエフェクトを追加して、動きのあるサイトにする
  7. 7 ブラウザで確認し、さまざまなデバイスサイズで動作をチェックする
  8. 8 （オプション）GitHub Pagesなどで公開してみる

トラブルシューティング

レイアウトが崩れてしまう

原因: 要素のサイズ計算や、Flexbox/Gridの設定が適切でない可能性があります。

解決方法: box-sizing: border-box;が全体に適用されているか確認しましょう。また、ブラウザの開発者ツールで各要素のサイズや余白を確認し、問題箇所を特定します。

画像が表示されない

原因: 画像のパスが間違っているか、画像ファイルが存在しない可能性があります。

解決方法: HTMLファイルから見た画像ファイルの相対パスが正しいか確認します。同じフォルダにある場合はsrc="image.jpg"、サブフォルダにある場合はsrc="images/image.jpg"のように記述します。

スマートフォンで横スクロールが出てしまう

原因: 要素の幅が画面サイズを超えているか、固定幅が設定されている可能性があります。

解決方法: max-width: 100%;を画像やコンテナに設定します。また、overflow-x: hidden;をbodyに設定することで、はみ出た部分を隠すこともできます（ただし根本解決ではないので注意）。

CSSが反映されない

原因: CSSファイルのパスが間違っている、またはブラウザのキャッシュが残っている可能性があります。

解決方法: <link rel="stylesheet" href="style.css">のパスが正しいか確認します。また、ブラウザの強制リロード（Ctrl+F5 / Cmd+Shift+R）を試してキャッシュをクリアしましょう。

理解度確認クイズ

  1. Q1. ポートフォリオサイトに必ず含めるべき基本的なセクションを3つ挙げてください。

    回答を表示

    代表的なセクションは、About（自己紹介）、Works（作品・プロジェクト）、Contact（お問い合わせ）の3つです。これらに加えて、ヒーローセクション（メインビジュアル）やスキル紹介なども含めることが一般的です。

  2. Q2. レスポンシブデザインを実装する際に使う主なCSS機能は何ですか？

    回答を表示

    メディアクエリ（@media）を使います。画面サイズに応じて異なるCSSスタイルを適用することで、スマートフォン、タブレット、PCなどさまざまなデバイスに最適化した表示を実現できます。

  3. Q3. ポートフォリオサイトを公開する前に確認すべきことは何ですか？（3つ以上挙げてください）

    回答を表示

    主な確認事項：1) すべてのリンクが正しく機能しているか、2) 画像が正しく表示されているか、3) さまざまなデバイスサイズで正しく表示されるか、4) 誤字脱字がないか、5) ブラウザ間の互換性、6) ページの読み込み速度などです。

第15章のまとめ

  • ポートフォリオサイトは、自分のスキルや作品を紹介するための重要なツールである
  • サイト制作の前に、目的、ターゲット、必要なセクション、デザイン方針を明確にする
  • HTMLで構造を作り、CSSでスタイリングし、これまで学んだすべての技術を統合する
  • レスポンシブデザインを実装し、すべてのデバイスで快適に閲覧できるようにする
  • 公開前に十分なチェックを行い、GitHub Pagesなどで実際に公開する
  • 実践プロジェクトを通じて、実際に使えるWebサイトを作成するスキルを習得した
