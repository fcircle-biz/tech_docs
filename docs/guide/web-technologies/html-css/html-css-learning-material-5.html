5.1 プロジェクトファイルの整理

推奨されるフォルダ構成

Webサイトを公開する前に、ファイルを適切に整理しましょう。

my-portfolio/
├── index.html          # トップページ
├── about.html          # 自己紹介ページ
├── contact.html        # お問い合わせページ
├── css/
│   └── style.css       # CSSファイル
├── js/
│   └── script.js       # JavaScriptファイル（将来使用）
└── images/
    ├── logo.png        # ロゴ画像
    └── profile.jpg     # プロフィール画像
フォルダ構成のポイント
  • HTMLファイルはルート（最上位）に配置
  • CSS、JS、画像は専用フォルダに整理
  • ファイル名は小文字・英数字・ハイフンを使用
  • スペースは使わない（代わりにハイフンやアンダースコア）

5.2 相対パスと絶対パス

相対パスとは？

現在のファイルから見た、他のファイルへの道順です。 同じサイト内のファイルを参照する際に使用します。 

<!-- 同じフォルダ内のファイル -->
<link rel="stylesheet" href="style.css">

<!-- cssフォルダ内のファイル -->
<link rel="stylesheet" href="css/style.css">

<!-- 一つ上のフォルダ -->
<link rel="stylesheet" href="../css/style.css">

<!-- 画像の参照 -->
<img src="images/logo.png" alt="ロゴ">

絶対パスとは？

完全なURL（http://やhttps://から始まる）です。 外部サイトのファイルを参照する際に使用します。 

<!-- 外部サイトの画像 -->
<img src="https://example.com/images/photo.jpg" alt="写真">

<!-- Google Fonts（外部CSS） -->
<link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">

5.3 GitHubアカウントの作成

実習 5-1: GitHubアカウントの作成
手順
  1. GitHubにアクセス
  2. 「Sign up」ボタンをクリック
  3. メールアドレス、パスワード、ユーザー名を入力
  4. メール認証を完了
  5. プラン選択（無料プランで十分）

アカウント作成が完了したら、ログインしてダッシュボードを確認しましょう。

5.4 Gitの基本操作

Gitとは？

Gitは、ファイルの変更履歴を管理する「バージョン管理システム」です。 コードの変更を記録し、必要なら過去の状態に戻すことができます。

flowchart LR A[作業フォルダ] -->|git add| B[ステージング] B -->|git commit| C[ローカルリポジトリ] C -->|git push| D[GitHub
リモートリポジトリ]
主なGitコマンド
  • git init: 新しいリポジトリを作成
  • git add .: すべての変更をステージングに追加
  • git commit -m "メッセージ": 変更を記録
  • git push: GitHubに送信

5.5 GitHub Pagesでの公開

実習 5-2: GitHub Pagesでサイトを公開
方法1: Webインターフェースから（初心者向け）
  1. GitHubにログイン
  2. 右上の「+」→「New repository」をクリック
  3. リポジトリ名を入力（例: my-portfolio）
  4. 「Public」を選択
  5. 「Create repository」をクリック
  6. 「uploading an existing file」をクリック
  7. index.htmlやその他のファイルをドラッグ＆ドロップ
  8. 「Commit changes」をクリック
  9. 「Settings」→「Pages」に移動
  10. 「Source」で「main」ブランチを選択
  11. 「Save」をクリック
方法2: コマンドラインから（推奨）
# プロジェクトフォルダに移動
cd my-portfolio

# Gitリポジトリを初期化
git init

# すべてのファイルを追加
git add .

# 変更をコミット
git commit -m "Initial commit"

# GitHubのリモートリポジトリを追加
git remote add origin https://github.com/ユーザー名/リポジトリ名.git

# GitHubにプッシュ
git push -u origin main
公開URLの確認

Settings → Pages で公開URLが表示されます：
https://ユーザー名.github.io/リポジトリ名/

5.6 SEOの基礎

SEOとは？

SEO（Search Engine Optimization）は、検索エンジン最適化のことです。 Googleなどの検索結果で上位に表示されやすくするための施策です。

基本的なSEO対策

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- ページタイトル（重要！） -->
    <title>山田太郎のポートフォリオ | Web開発者</title>

    <!-- ページの説明 -->
    <meta name="description" content="Web開発を学習中の山田太郎のポートフォリオサイトです。HTML/CSSの作品を公開しています。">

    <!-- キーワード（オプション） -->
    <meta name="keywords" content="Web開発, HTML, CSS, ポートフォリオ">

    <!-- OGP（SNSシェア用） -->
    <meta property="og:title" content="山田太郎のポートフォリオ">
    <meta property="og:description" content="Web開発を学習中のポートフォリオサイト">
    <meta property="og:type" content="website">
    <meta property="og:url" content="https://example.github.io/portfolio/">
    <meta property="og:image" content="https://example.github.io/portfolio/images/thumbnail.jpg">
</head>
<body>
    <!-- コンテンツ -->
</body>
</html>
SEO対策のポイント
  • titleタグ: 60文字以内で、ページの内容を正確に表現
  • meta description: 160文字以内で、ページの要約を記述
  • 見出しタグ: h1は1ページに1つ、階層を守る
  • 画像のalt属性: 必ず設定する
  • コンテンツの質: オリジナルで有用な内容を書く

5.7 公開後の更新と保守

サイトの更新方法

公開後にサイトを更新する場合の手順：

# ファイルを編集後

# 変更を追加
git add .

# 変更をコミット
git commit -m "プロフィール情報を更新"

# GitHubにプッシュ
git push

数分後、GitHub Pagesに変更が反映されます。

保守のベストプラクティス

  • 定期的にコンテンツを更新する
  • リンク切れをチェックする
  • スマホでの表示を確認する
  • ブラウザのコンソールでエラーをチェック
  • 訪問者のフィードバックを受け入れる

5.8 まとめ

この章で学んだこと
  • プロジェクトファイルは適切なフォルダ構成で整理する
  • 相対パスは同じサイト内、絶対パスは外部サイトの参照に使用
  • GitHubアカウントを作成し、リポジトリを管理する
  • GitHub Pagesで無料でWebサイトを公開できる
  • SEO対策でtitleとmeta descriptionを設定する
  • git add、commit、pushで更新を管理する
理解度確認クイズ
  1. 問題1: 相対パスと絶対パスの違いを説明してください。
    解答例

    相対パスは現在のファイルから見た相対的な道順で、同じサイト内のファイルを参照します。 絶対パスは完全なURL（http://やhttps://から始まる）で、主に外部サイトのファイルを参照します。

  2. 問題2: Gitの基本的なワークフローを説明してください。
    解答例

    1. ファイルを編集
    2. git addでステージングに追加
    3. git commitで変更を記録
    4. git pushでGitHubに送信

  3. 問題3: SEO対策で最も重要なHTMLタグは何ですか？
    解答例

    <title>タグと<meta name="description">タグです。 これらは検索結果に直接表示されるため、SEO対策の最重要要素です。
