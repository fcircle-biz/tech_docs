画像とメディア - ビジュアルコンテンツの追加

画像の表示（`<img>`）、動画や音声の埋め込み（`<video>`、`<audio>`）、YouTubeなど外部コンテンツの埋め込み（`<iframe>`）について学びます。画像の適切なサイズ設定や代替テキストの重要性も理解します。

この章で学ぶこと

  • <img>タグを使った画像の表示方法を理解する
  • 代替テキスト（alt属性）の重要性とアクセシビリティへの配慮を学ぶ
  • <video>と<audio>タグでメディアを埋め込む方法を習得する
  • <iframe>を使ってYouTubeなど外部コンテンツを埋め込む技術を身につける

4.1 画像の表示 - <img>タグ

Webページに画像を表示するには、<img>タグを使用します。このタグは「空要素」と呼ばれ、閉じタグが不要です（</img>は書きません）。

画像は、テキストだけでは伝えにくい情報を視覚的に表現したり、Webページを美しく装飾したりする重要な要素です。

index.html 
<!-- 画像を表示する基本形 -->
<img src="images/sunset.jpg" alt="美しい夕日の写真">

<!-- 幅と高さを指定 -->
<img src="images/logo.png" alt="会社ロゴ" width="200" height="100">

空要素とは？

空要素（または自己閉じタグ）は、開始タグだけで構成され、閉じタグを必要としない要素です。<img>の他に、<br>（改行）や<hr>（水平線）なども空要素です。

4.2 画像の重要な属性

<img>タグには、いくつかの重要な属性があります。これらを正しく使うことで、適切に画像を表示できます。

imgタグの主要属性

属性 説明 必須/任意
src 画像ファイルのパス（URL） 必須
alt 画像が表示できない時に代わりに表示されるテキスト（代替テキスト） 必須
width 画像の幅（ピクセル単位） 任意
height 画像の高さ（ピクセル単位） 任意
title 画像にマウスを乗せた時に表示されるツールチップ 任意

重要ポイント: alt属性は必ず指定しよう

alt属性（代替テキスト）は、画像が読み込めない場合や、視覚障害のある方がスクリーンリーダーを使用する際に非常に重要です。画像の内容を簡潔に説明するテキストを必ず指定しましょう。装飾目的の画像の場合は、alt=""と空にします。

適切なalt属性の例 
<!-- 情報を伝える画像: 内容を説明 -->
<img src="cat.jpg" alt="白い猫がソファで眠っている写真">

<!-- ロゴ画像: 会社名やサービス名 -->
<img src="logo.png" alt="ACME株式会社">

<!-- 装飾目的の画像: altを空にする -->
<img src="decoration.png" alt="">

4.3 動画の埋め込み - <video>タグ

HTML5から、<video>タグを使って動画をWebページに直接埋め込むことができるようになりました。プラグイン（Flash等）を使わずに、ブラウザだけで動画を再生できます。

video-example.html 
<!-- 動画を埋め込む基本形 -->
<video controls width="640" height="360">
    <source src="videos/sample.mp4" type="video/mp4">
    <source src="videos/sample.webm" type="video/webm">
    お使いのブラウザは動画タグに対応していません。
</video>

videoタグの主要属性

  • controls 再生ボタンや音量調整などの操作パネルを表示
  • autoplay ページ読み込み時に自動再生（注意: ユーザー体験を損なう可能性あり）
  • loop 動画を繰り返し再生
  • muted 音声をミュート（無音）にする
  • poster 動画が読み込まれる前に表示するサムネイル画像のパス

ヒント: 複数の動画形式を用意しよう

ブラウザによって対応している動画形式が異なります。MP4、WebM、OGVなど複数の形式を<source>タグで指定すると、ブラウザが対応している形式を自動的に選んで再生します。

4.4 音声の埋め込み - <audio>タグ

<audio>タグは、音声ファイルをWebページに埋め込むために使用します。使い方は<video>タグと非常に似ています。

audio-example.html 
<!-- 音声を埋め込む基本形 -->
<audio controls>
    <source src="audio/bgm.mp3" type="audio/mpeg">
    <source src="audio/bgm.ogg" type="audio/ogg">
    お使いのブラウザは音声タグに対応していません。
</audio>

よく使われる音声形式

  • MP3: 最も広く対応されている音声形式
  • OGG: オープンソースの音声形式
  • WAV: 高品質だがファイルサイズが大きい

4.5 外部コンテンツの埋め込み - <iframe>タグ

<iframe>タグを使うと、他のWebサイトのコンテンツを自分のページ内に埋め込むことができます。YouTube動画、Google Maps、SNSの投稿などを埋め込む際によく使用されます。

YouTube埋め込み例 
<!-- YouTube動画を埋め込む -->
<iframe
    width="560"
    height="315"
    src="https://www.youtube.com/embed/動画ID"
    title="YouTube video player"
    frameborder="0"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    allowfullscreen>
</iframe>
Google Maps埋め込み例 
<!-- Google Mapsを埋め込む -->
<iframe
    src="https://www.google.com/maps/embed?pb=..."
    width="600"
    height="450"
    style="border:0;"
    allowfullscreen=""
    loading="lazy">
</iframe>

注意: iframeのセキュリティ

iframeは信頼できるサイトのコンテンツのみを埋め込むようにしましょう。不審なサイトのコンテンツを埋め込むと、セキュリティリスクが生じる可能性があります。YouTube、Google Maps、Twitterなどメジャーなサービスのコンテンツであれば安全です。

iframeタグの主要属性

  • src 埋め込むページのURL
  • width フレームの幅（ピクセル単位）
  • height フレームの高さ（ピクセル単位）
  • title フレーム内容の説明（アクセシビリティ向上）
  • allowfullscreen 全画面表示を許可
  • loading 遅延読み込み（lazy）を指定してページ速度を改善

実習 4-1: 画像とメディアを使ったページを作成

学んだ内容を活用して、画像、動画、音声、iframeを含むHTMLページを作成してみましょう。

手順

  1. 1 media-page.htmlという名前で新しいHTMLファイルを作成
  2. 2 基本的なHTML構造（DOCTYPE、html、head、body）を記述
  3. 3 <img>タグで画像を2つ以上追加（alt属性を忘れずに！）
  4. 4 YouTube動画を<iframe>で埋め込む（YouTubeの「共有」→「埋め込む」からコードを取得）
  5. 5 ブラウザで開いて、すべての要素が正しく表示されることを確認

理解度確認クイズ

  1. Q1. alt属性はなぜ重要ですか？具体的な理由を2つ挙げてください。

    回答を表示

    理由1: 画像が何らかの理由で読み込めない場合、代替テキストが表示されることでユーザーに画像の内容を伝えることができます。
    理由2: 視覚障害のある方がスクリーンリーダーを使用する際、alt属性のテキストが読み上げられるため、画像の内容を理解できます（アクセシビリティの向上）。

  2. Q2. <video>タグで複数の<source>タグを使う理由は何ですか？

    回答を表示

    ブラウザによって対応している動画形式が異なるため、複数の形式を用意することで、より多くのブラウザで動画を再生できるようにするためです。ブラウザは上から順に確認し、対応している最初の形式を自動的に選んで再生します。

  3. Q3. iframeを使用する際のセキュリティ上の注意点は何ですか？

    回答を表示

    信頼できるサイトのコンテンツのみを埋め込むようにする必要があります。不審なサイトや知らないサイトのコンテンツをiframeで埋め込むと、セキュリティリスク（悪意のあるスクリプトの実行など）が生じる可能性があります。YouTube、Google Maps、主要SNSなど、信頼できるサービスのコンテンツであれば安全です。

第4章のまとめ

  • <img>タグで画像を表示でき、src属性とalt属性は必須
  • alt属性は画像の代替テキストで、アクセシビリティ向上に重要
  • <video>と<audio>タグで動画・音声をブラウザだけで再生できる
  • 複数の<source>タグでブラウザ互換性を向上
  • <iframe>タグでYouTube、Google Mapsなど外部コンテンツを埋め込める
  • iframeは信頼できるサイトのコンテンツのみを埋め込むことが重要
