Webの世界へようこそ - HTMLとCSSの役割

インターネット上のWebページがどのように作られているか、HTMLとCSSがそれぞれどんな役割を持つのかを、身近な例を使って分かりやすく解説します。プログラミング未経験でも理解できる内容からスタートします。

この章で学ぶこと

  • Webページの仕組みと、インターネットの基本的な動作を理解する
  • HTMLとCSSの役割の違いを説明できるようになる
  • ブラウザがWebページを表示する流れを理解する
  • 簡単な例を通じて、HTMLとCSSが実際にどう使われるかを確認する

1.1 Webページとは何か

私たちが毎日使っているWebページ（Webサイト）は、実は「テキストファイル」で作られています。GoogleやYouTube、Amazonなど、どんなに複雑に見えるWebサイトも、基本的には文字で書かれた設計図（コード）がブラウザによって解釈され、私たちが見ている美しいページとして表示されています。

Webページを作るための主な技術が、この章で学ぶ「HTML」と「CSS」です。

Webブラウザとは？

Google Chrome、Firefox、Safari、Microsoft Edgeなど、インターネット上のWebページを見るためのソフトウェアです。ブラウザはHTMLやCSSのコードを読み取り、私たちが見やすい形で画面に表示してくれます。

1.2 HTMLの役割 - 構造を作る

HTML（HyperText Markup Language）は、Webページの「構造」を定義するための言語です。HTMLを使って、見出し、段落、リンク、画像、表などの要素を配置します。

HTMLは「家の骨組み」に例えられます。家を建てるとき、まず柱や壁、屋根などの基本構造を作りますよね。それと同じように、HTMLはWebページの基本的な構造を作ります。

example.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <title>私の最初のWebページ</title>
</head>
<body>
    <h1>ようこそ！</h1>
    <p>これは私の最初のWebページです。</p>
</body>
</html>

このHTMLコードをブラウザで開くと、「ようこそ！」という大きな見出しと、その下に「これは私の最初のWebページです。」という文章が表示されます。

覚えておきたいポイント

HTMLは「タグ」と呼ばれる<タグ名>という記号で要素を定義します。たとえば、<h1>は見出し、<p>は段落を意味します。

1.3 CSSの役割 - 見た目を整える

CSS（Cascading Style Sheets）は、Webページの「見た目」を整えるための言語です。HTMLで作った構造に対して、色、フォント、配置、余白などのデザインを指定します。

CSSは「家の内装・外装」に例えられます。骨組みができた家に、壁紙を貼ったり、床材を敷いたり、ペンキを塗ったりして、美しく仕上げる作業です。

style.css 
/* 見出しのスタイル */
h1 {
    color: #3b82f6;  /* 青色 */
    font-size: 32px;
    text-align: center;
}

/* 段落のスタイル */
p {
    color: #475569;  /* グレー */
    font-size: 16px;
    line-height: 1.6;
}

このCSSを先ほどのHTMLに適用すると、見出しが青色で中央揃えになり、段落の文字色や行間が調整されて、より読みやすくなります。

1.4 HTMLとCSSの関係

HTMLとCSSは、それぞれ独立した役割を持ちながらも、互いに協力してWebページを作ります。HTMLが構造を提供し、CSSがその構造に見た目を与える、という関係です。

HTMLとCSSの比較

特徴 HTML CSS
役割 構造を作る 見た目を整える
例えると 家の骨組み 家の内装・外装
できること 見出し、段落、リンク、画像などの配置 色、フォント、配置、余白の調整
ファイル形式 .html .css

分離の原則

HTMLとCSSを別々のファイルに分けることで、構造とデザインを分離できます。これにより、デザインだけを変更したいときにHTMLを触る必要がなくなり、メンテナンスがしやすくなります。

1.5 ブラウザの役割

Webブラウザは、HTMLとCSSで書かれたコードを読み取り、私たちが見やすい形で画面に表示する「通訳者」のような役割を果たします。

理解度確認クイズ

  1. Q1. HTMLとCSSの役割の違いを説明してください。

    回答を表示

    HTMLはWebページの「構造」を作る言語で、見出し、段落、リンク、画像などの要素を配置します。一方、CSSは「見た目」を整える言語で、色、フォント、配置、余白などのデザインを指定します。HTMLが家の骨組みなら、CSSは内装・外装に例えられます。

  2. Q2. Webブラウザの役割を説明してください。

    回答を表示

    Webブラウザは、HTMLとCSSで書かれたコードを読み取り、私たちが見やすい形で画面に表示する「通訳者」のような役割を果たします。サーバーからHTMLファイルを受信し、CSSを読み込んで、それらを組み合わせて最終的なWebページを表示します。

  3. Q3. なぜHTMLとCSSを別々のファイルに分けることが推奨されるのでしょうか？

    回答を表示

    構造（HTML）とデザイン（CSS）を分離することで、メンテナンスがしやすくなります。例えば、デザインだけを変更したいときにHTMLを触る必要がなくなり、効率的に作業できます。また、複数のHTMLページで同じCSSファイルを共有できるため、サイト全体のデザインを一貫させやすくなります。

第1章のまとめ

  • Webページは、HTMLとCSSで作られたテキストファイルがブラウザで表示されたもの
  • HTMLは「構造」を作る言語で、見出し、段落、リンクなどを定義する
  • CSSは「見た目」を整える言語で、色、フォント、配置などを指定する
  • ブラウザは、HTMLとCSSを読み取り、画面に表示する役割を持つ
  • HTMLとCSSを分離することで、メンテナンスがしやすくなる
