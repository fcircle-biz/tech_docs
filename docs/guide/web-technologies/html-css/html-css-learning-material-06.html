6.1 レスポンシブデザインとは？

レスポンシブデザインは、画面サイズに応じてレイアウトを自動調整する設計手法です。 スマホ、タブレット、PCなど、あらゆるデバイスで最適な表示を実現します。

なぜレスポンシブデザインが重要？
  • スマホからのアクセスが全体の50%以上
  • Googleがモバイル対応を評価基準にしている
  • 1つのHTMLで全デバイスに対応できる
  • ユーザーエクスペリエンスの向上

6.2 viewportメタタグ

スマホでの表示を最適化するため、HTMLの<head>内に以下を必ず記述します。 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

6.3 メディアクエリの基本

メディアクエリは、画面サイズに応じて異なるCSSを適用する機能です。 

/* 基本のスタイル（モバイル） */
.container {
    width: 100%;
    padding: 10px;
}

/* タブレット（768px以上） */
@media (min-width: 768px) {
    .container {
        width: 750px;
        margin: 0 auto;
    }
}

/* PC（1024px以上） */
@media (min-width: 1024px) {
    .container {
        width: 1000px;
    }
}

6.4 主要なブレークポイント

一般的なブレークポイント
  • スマホ: 〜767px
  • タブレット: 768px〜1023px
  • PC: 1024px以上

6.5 モバイルファーストアプローチ

モバイルファーストとは、スマホ向けのデザインを基本とし、 画面が大きくなるにつれて要素を追加していく設計手法です。 

/* モバイル（基本） */
.nav-menu {
    display: flex;
    flex-direction: column;
}

/* タブレット以上 */
@media (min-width: 768px) {
    .nav-menu {
        flex-direction: row;
    }
}
実習 6-1: レスポンシブナビゲーション
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>レスポンシブナビゲーション</title>
    <style>
        * {
            margin: 0;
            padding: 0;
            box-sizing: border-box;
        }

        .navbar {
            background-color: #E44D26;
            padding: 15px;
        }

        .nav-menu {
            display: flex;
            flex-direction: column;
            list-style: none;
        }

        .nav-menu li {
            margin: 5px 0;
        }

        .nav-menu a {
            color: white;
            text-decoration: none;
            padding: 10px;
            display: block;
        }

        /* タブレット以上 */
        @media (min-width: 768px) {
            .nav-menu {
                flex-direction: row;
                justify-content: space-around;
            }

            .nav-menu li {
                margin: 0;
            }
        }
    </style>
</head>
<body>
    <nav class="navbar">
        <ul class="nav-menu">
            <li><a href="#">ホーム</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Services</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>
</body>
</html>

ブラウザの幅を変えて、ナビゲーションが縦並びから横並びに変わることを確認しましょう。

6.6 レスポンシブグリッドレイアウト

/* モバイル: 1カラム */
.grid {
    display: flex;
    flex-direction: column;
    gap: 20px;
}

/* タブレット: 2カラム */
@media (min-width: 768px) {
    .grid {
        flex-direction: row;
        flex-wrap: wrap;
    }

    .grid-item {
        flex: 0 0 calc(50% - 10px);
    }
}

/* PC: 3カラム */
@media (min-width: 1024px) {
    .grid-item {
        flex: 0 0 calc(33.333% - 14px);
    }
}

6.7 まとめ

この章で学んだこと
  • レスポンシブデザインは複数デバイスに対応する設計手法
  • viewportメタタグは必須
  • メディアクエリで画面サイズごとにスタイルを切り替え
  • モバイルファーストで設計するのが推奨
  • Flexboxでレスポンシブレイアウトが簡単に作れる
理解度確認クイズ
  1. 問題1: レスポンシブデザインが重要な理由は？
    解答例

    スマホ、タブレット、PCなど様々なデバイスで最適な表示を実現し、ユーザーエクスペリエンスを向上させるため。また、Googleがモバイル対応をSEO評価基準にしているため。

  2. 問題2: メディアクエリの基本的な書き方は？
    解答例

    @media (min-width: 768px) { ... }のように、画面幅の条件を指定してその中にCSSを記述します。
