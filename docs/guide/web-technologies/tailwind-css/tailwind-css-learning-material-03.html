スペーシングとサイジング - 余白と大きさの調整

padding、margin、width、heightなど、要素の間隔とサイズを制御するクラスを学びます。Tailwindのスペーシングスケールシステムを理解し、一貫性のあるレイアウトを作成します。

この章で学ぶこと

  • Tailwindのスペーシングスケールシステムの理解と活用
  • パディング（padding）とマージン（margin）の使い分け
  • 幅（width）と高さ（height）の指定方法
  • 最大幅・最小幅を使った柔軟なレイアウト作成

3.1 スペーシングシステムの理解

Tailwind CSSは、統一されたスペーシングスケールを提供しており、0から96までの数値で余白やサイズを指定します。 この数値は基本単位（0.25rem = 4px）の倍数に対応しています。

主なスペーシングスケール

クラス値 rem px 用途例
0 0 0px 余白なし
1 0.25rem 4px 極小の余白
2 0.5rem 8px 小さい余白
4 1rem 16px 標準的な余白
6 1.5rem 24px 中程度の余白
8 2rem 32px 大きい余白
12 3rem 48px とても大きい余白

スケールシステムの利点

統一されたスケールを使用することで、デザインに一貫性が生まれます。 「8の倍数ルール」などのデザイン原則に自然と従うことができます。

3.2 パディング（padding）の使い方

パディングは要素の内側の余白を制御します。 背景色や枠線の内側にスペースを作り、コンテンツを見やすくします。

主なパディングクラス

全方向

  • p-4 - 全方向に1rem（16px）
  • p-6 - 全方向に1.5rem（24px）

上下左右個別

  • pt-4 - 上（top）
  • pr-4 - 右（right）
  • pb-4 - 下（bottom）
  • pl-4 - 左（left）

縦横セット

  • px-4 - 左右（x軸）
  • py-4 - 上下（y軸）
padding-example.html 
<!-- パディングの例 -->
<div class="bg-blue-100 p-4">
    全方向に16pxのパディング
</div>

<div class="bg-green-100 px-6 py-3">
    左右24px、上下12pxのパディング
</div>

<div class="bg-purple-100 pt-8 px-4 pb-2">
    上32px、左右16px、下8pxのパディング
</div>

表示例

全方向に16pxのパディング
左右24px、上下12pxのパディング
上32px、左右16px、下8pxのパディング

3.3 マージン（margin）の使い方

マージンは要素の外側の余白を制御します。 要素間のスペースを調整し、レイアウトを整えます。

パディングとマージンの違い

flowchart LR subgraph Element["要素"] subgraph Content["コンテンツエリア"] Text[テキスト] end Padding["← padding: 内側の余白 →"] end Margin1["margin: 外側の余白"] Margin2["margin: 外側の余白"] Margin1 -.-> Element Element -.-> Margin2

マージンの特殊な使い方

中央揃え（mx-auto）

左右のマージンを自動にして、要素を中央に配置します。

mx-auto

負のマージン

要素を重ね合わせたり、レイアウトを調整するために使用します。

-mt-4（上に-16px）、 -ml-2（左に-8px）
margin-example.html 
<!-- 中央揃えカード -->
<div class="max-w-md mx-auto bg-white p-6 rounded-lg shadow">
    この要素は中央に配置されます
</div>

<!-- 要素間の間隔 -->
<div class="space-y-4">
    <div class="bg-blue-100 p-4">要素1</div>
    <div class="bg-blue-100 p-4">要素2</div>
    <div class="bg-blue-100 p-4">要素3</div>
</div>

space-yとspace-xの活用

space-y-4やspace-x-4を使うと、 子要素間に自動的に余白を設定できます。個別にマージンを指定する必要がありません。

3.4 幅（width）と高さ（height）

要素の幅と高さは、固定値、パーセント、画面全体など、さまざまな方法で指定できます。

主な幅（width）クラス

クラス 意味 用途
w-full 100% 親要素の幅いっぱい
w-1/2 50% 親要素の半分
w-1/3 33.33% 3列レイアウト
w-64 16rem（256px） 固定幅
w-screen 100vw 画面幅いっぱい

主な高さ（height）クラス

クラス 意味 用途
h-screen 100vh 画面高さいっぱい
h-64 16rem（256px） 固定高さ
h-full 100% 親要素の高さいっぱい
size-example.html 
<!-- 幅のバリエーション -->
<div class="w-full bg-blue-100 p-4 mb-2">幅100%</div>
<div class="w-1/2 bg-green-100 p-4 mb-2">幅50%</div>
<div class="w-64 bg-purple-100 p-4">幅256px</div>

<!-- 高さの例 -->
<div class="h-32 bg-orange-100 p-4">
    高さ128px
</div>

3.5 最大幅・最小幅の設定

最大幅（max-width）と最小幅（min-width）を使うと、レスポンシブで柔軟なレイアウトを作成できます。

よく使う最大幅クラス

max-w-sm 24rem（384px）- 小さいカード
max-w-md 28rem（448px）- 中サイズカード
max-w-lg 32rem（512px）- 大きいカード
max-w-4xl 56rem（896px）- 記事コンテンツ
max-w-screen-xl 1280px - メインコンテナ
max-width-example.html 
<!-- レスポンシブなコンテナ -->
<div class="max-w-4xl mx-auto px-4">
    <p>
        このコンテンツは最大幅896pxで、
        それを超えると中央に配置されます。
        画面が小さい場合は幅100%になります。
    </p>
</div>

<!-- カードの最大幅制限 -->
<div class="max-w-md mx-auto bg-white p-6 rounded-lg shadow">
    <h2 class="text-xl font-bold">カードタイトル</h2>
    <p>このカードは最大幅448pxに制限されています</p>
</div>

max-w-*とmx-autoの組み合わせ

max-w-4xl mx-autoのように組み合わせると、 最大幅を制限しつつ中央配置できます。レスポンシブなコンテナ作成の定番パターンです。

実習 3-1: スペーシングを活用したカードレイアウト

パディング、マージン、幅の設定を組み合わせて、美しいカードレイアウトを作成しましょう。

手順

  1. 1 新しいHTMLファイルを作成し、Tailwind CDNを読み込みます
  2. 2 max-w-md mx-autoでコンテナを中央配置
  3. 3 カードにbg-white p-6 rounded-lg shadow-lgを適用
  4. 4 見出しとテキストの間にmb-4で余白を追加
  5. 5 複数のカードをspace-y-6で配置してみましょう

期待される結果

整ったカード型レイアウトが表示され、要素間に適切な余白が設定されていることが確認できます。

理解度確認クイズ

  1. Q1. パディング（padding）とマージン（margin）の違いは何ですか？

    回答を表示

    パディングは要素の内側の余白で、背景色や枠線の内側に適用されます。 マージンは要素の外側の余白で、他の要素との間隔を調整します。 例えば、ボタンの文字と枠の間の余白はパディング、ボタンとボタンの間の余白はマージンです。

  2. Q2. px-4とpy-4の違いは何ですか？

    回答を表示

    px-4は左右（x軸）のパディングを1rem（16px）に設定します。 py-4は上下（y軸）のパディングを1rem（16px）に設定します。 組み合わせてpx-4 py-2のように使うと、左右と上下で異なる余白を設定できます。

  3. Q3. 要素を中央に配置するには、どのクラスを組み合わせますか？

    回答を表示

    mx-autoで左右のマージンを自動にして中央配置できます。 ただし、要素に幅の指定（w-*やmax-w-*）が必要です。 例：max-w-md mx-auto

この章のまとめ

  • Tailwindは統一されたスペーシングスケール（0から96まで）を提供し、一貫性のあるデザインを実現します
  • パディング（p-*）は要素内の余白、マージン（m-*）は要素外の余白を制御します
  • px-*、py-*を使えば、左右・上下の余白を個別に設定できます
  • 幅（w-*）と高さ（h-*）は固定値、パーセント、画面サイズなど柔軟に指定できます
  • max-w-*とmx-autoを組み合わせると、レスポンシブな中央配置レイアウトが作れます
