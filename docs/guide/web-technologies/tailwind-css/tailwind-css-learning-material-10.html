10.1 Reactとの統合

ReactプロジェクトでTailwind CSSを使用する方法を学びます。

Create React Appでのセットアップ

# React プロジェクトを作成
npx create-react-app my-app
cd my-app

# Tailwind CSSのインストール
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

設定ファイルの更新

// tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{js,jsx,ts,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

Reactコンポーネントでの使用例

// src/components/Button.jsx
export default function Button({ children, variant = 'primary' }) {
  const baseStyles = "px-6 py-2 rounded-lg font-semibold transition duration-200";

  const variants = {
    primary: "bg-blue-600 hover:bg-blue-700 text-white",
    secondary: "bg-gray-200 hover:bg-gray-300 text-gray-800",
    danger: "bg-red-600 hover:bg-red-700 text-white",
  };

  return (
    <button className={`${baseStyles} ${variants[variant]}`}>
      {children}
    </button>
  );
}

10.4 コンポーネントライブラリの構築

再利用可能なコンポーネントライブラリを作成し、プロジェクト間で共有する方法を学びます。

実習 10-1: デザインシステムの構築
ディレクトリ構造
design-system/
├── src/
│   ├── components/
│   │   ├── Button/
│   │   │   ├── Button.jsx
│   │   │   └── Button.stories.jsx
│   │   ├── Card/
│   │   └── Input/
│   ├── tokens/
│   │   ├── colors.js
│   │   ├── spacing.js
│   │   └── typography.js
│   └── tailwind.config.js
└── package.json
デザイントークンの定義
// src/tokens/colors.js
export const colors = {
  brand: {
    primary: '#0066cc',
    secondary: '#00cc66',
    danger: '#cc0000',
  },
  neutral: {
    50: '#f9fafb',
    100: '#f3f4f6',
    // ...
    900: '#111827',
  },
};
tailwind.config.jsへの統合
// tailwind.config.js
import { colors } from './src/tokens/colors';

module.exports = {
  theme: {
    extend: {
      colors: {
        brand: colors.brand,
      },
    },
  },
};

10.6 トラブルシューティング

よくある問題と解決方法

問題1: スタイルが適用されない

原因：content配列にファイルパスが含まれていない

解決方法：tailwind.config.jsのcontent配列を確認し、すべてのファイルパスを含める

content: [
  "./src/**/*.{js,jsx,ts,tsx}",  // すべてのファイルを含む
]
問題2: ビルドサイズが大きい

原因：未使用スタイルが削除されていない

解決方法：本番ビルドでNODE_ENV=productionを設定

問題3: カスタムクラスが認識されない

原因：JITモードで動的クラス名を使用している

解決方法：完全なクラス名を使用し、動的生成を避ける

// 悪い例（動的生成）
const colors = ['red', 'blue', 'green'];
<div className={`bg-${colors[0]}-500`}>

// 良い例（完全なクラス名）
const colorClasses = {
  red: 'bg-red-500',
  blue: 'bg-blue-500',
  green: 'bg-green-500',
};
<div className={colorClasses[color]}>

10.7 高度なテクニック

1. カスタムバリアントの作成

// tailwind.config.js
const plugin = require('tailwindcss/plugin');

module.exports = {
  plugins: [
    plugin(function({ addVariant }) {
      addVariant('not-last', '&:not(:last-child)');
      addVariant('hocus', ['&:hover', '&:focus']);
    })
  ],
};

使用例

<div class="not-last:mb-4">最後以外にマージン</div>
<button class="hocus:bg-blue-700">ホバーとフォーカスで色変更</button>

2. カスタムユーティリティの追加

// tailwind.config.js
const plugin = require('tailwindcss/plugin');

module.exports = {
  plugins: [
    plugin(function({ addUtilities }) {
      addUtilities({
        '.text-shadow': {
          textShadow: '2px 2px 4px rgba(0,0,0,0.1)',
        },
        '.text-shadow-lg': {
          textShadow: '4px 4px 8px rgba(0,0,0,0.2)',
        },
      })
    })
  ],
};
理解度確認クイズ
  1. ReactやVueなどのフレームワークでTailwind CSSを使用する際の主な利点は何ですか？

    解答：コンポーネントベースの開発と相性が良く、スタイルの再利用性が高まります。コンポーネント内でスタイルが完結するため、スタイルの競合が少なく、メンテナンスが容易になります。

  2. デザインシステムを構築する際の重要なポイントは何ですか？

    解答：デザイントークンの一元管理、再利用可能なコンポーネントの作成、ドキュメント化、チーム全体での一貫性の維持が重要です。
