実践プロジェクト：ランディングページ作成

学んだ知識を総合して、企業や製品のランディングページを作成します。ヘッダー、ヒーローセクション、特徴紹介、料金表、フッターなど、実践的なページを完成させます。

この章で学ぶこと

  • ランディングページの基本構成を理解し、実装できる
  • ヒーローセクション、特徴紹介、料金表などの主要セクションを作成できる
  • レスポンシブデザインとインタラクティブな要素を統合できる
  • これまで学んだTailwind CSSの知識を総合的に活用できる

10.1 ランディングページとは

ランディングページ（LP：Landing Page）とは、訪問者が最初に「着地（ランディング）」するページのことで、 製品やサービスの紹介、コンバージョン（購入・登録など）を目的とした専用ページです。

ランディングページの主要セクション

flowchart TD A[ランディングページ] --> B[ヘッダー/ナビゲーション] A --> C[ヒーローセクション] A --> D[特徴紹介セクション] A --> E[料金プランセクション] A --> F[CTAセクション] A --> G[フッター] C --> C1[キャッチコピー] C --> C2[サブテキスト] C --> C3[CTAボタン] D --> D1[機能1] D --> D2[機能2] D --> D3[機能3] E --> E1[ベーシック] E --> E2[スタンダード] E --> E3[プレミアム]

ランディングページの目的

ランディングページは、訪問者を顧客に変えることが最大の目的です。 そのため、明確な価値提案、わかりやすい特徴、魅力的なデザインが必要です。

10.2 プロジェクトの全体構成

架空のクラウドストレージサービス「CloudBox」のランディングページを作成します。 以下の構成で、完全なランディングページを実装していきます。

ページ構成 

CloudBox ランディングページ
├── ヘッダー（ナビゲーション）
├── ヒーローセクション（メインビジュアル）
├── 特徴紹介セクション（3つの主要機能）
├── 料金プランセクション（3つのプラン）
├── CTAセクション（行動喚起）
└── フッター（リンク・コピーライト）

使用する技術

  • Tailwind CSS（CDN版）
  • Font Awesome（アイコン）
  • Google Fonts

復習する内容

  • Flexbox/Gridレイアウト
  • レスポンシブデザイン
  • ホバー・トランジション

10.3 ヘッダーとナビゲーションの作成

まずは、ページ上部の固定ヘッダーとナビゲーションメニューを作成します。 モバイルでは、ハンバーガーメニューに切り替わる仕様にします。

landing-page.html（ヘッダー部分） 
<!-- ヘッダー（固定） -->
<header class="fixed top-0 left-0 right-0 bg-white shadow-md z-50">
  <nav class="max-w-6xl mx-auto px-4 sm:px-6 py-4">
    <div class="flex items-center justify-between">
      <!-- ロゴ -->
      <div class="flex items-center gap-2">
        <i class="fas fa-cloud text-3xl text-blue-500"></i>
        <span class="text-2xl font-bold text-slate-800">CloudBox</span>
      </div>

      <!-- デスクトップナビゲーション -->
      <div class="hidden md:flex items-center gap-6">
        <a href="#features" class="text-slate-600 hover:text-blue-500">特徴</a>
        <a href="#pricing" class="text-slate-600 hover:text-blue-500">料金</a>
        <a href="#contact" class="text-slate-600 hover:text-blue-500">お問い合わせ</a>
        <button class="bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded-lg">
          無料で始める
        </button>
      </div>
    </div>
  </nav>
</header>

ヒント

fixedでヘッダーを固定し、 z-50で他の要素より前面に表示します。 hidden md:flexでモバイルでは非表示、デスクトップで表示します。

ヒーローセクションは、訪問者が最初に目にする重要なエリアです。 キャッチコピー、説明文、行動喚起ボタンを配置し、サービスの魅力を一目で伝えます。

10.8 カスタマイズのポイント

基本的なランディングページができたら、以下のポイントでカスタマイズしてみましょう。

カラーテーマの変更

blue-500を他の色に変更して、 ブランドカラーに合わせましょう。

bg-blue-500 → bg-purple-500
text-blue-500 → text-purple-500

フォントの変更

Google Fontsで他のフォントを読み込み、デザインの印象を変えましょう。

font-sans → font-serif

画像の追加

ヒーローセクションやカードに実際の画像を追加して、よりリアルなページにしましょう。

<img src="..." alt="..." class="...">

セクションの追加

お客様の声、FAQセクションなどを追加して、コンテンツを充実させましょう。

理解度確認クイズ

  1. Q1. ランディングページに必要な主要セクションを3つ挙げてください。

    回答を表示

    代表的なセクションは以下の通りです：
    1. ヒーローセクション - 第一印象を決める重要なエリア、キャッチコピーとCTAボタンを配置
    2. 特徴紹介セクション - サービスの主要な特徴や利点をわかりやすく説明
    3. 料金プランセクション - 明確な価格設定とプラン比較を提示
    その他、CTAセクション、お客様の声、FAQなども重要です。

  2. Q2. 料金プランのカードで、おすすめプランを目立たせるために使ったTailwindクラスは何ですか？

    回答を表示

    以下のクラスを使用しました：
    - transform md:scale-105 - カードを少し拡大
    - shadow-xl - 大きな影をつけて浮き上がらせる
    - bg-blue-500 - 背景色を青にして他のカードと差別化
    - 「人気」バッジの追加

  3. Q3. レスポンシブデザインで、モバイルでは縦並び、デスクトップでは横並びにするために使用したクラスは何ですか？

    回答を表示

    grid md:grid-cols-3を使用しました。
    - モバイル（md未満）: デフォルトで1列のグリッド（縦並び）
    - デスクトップ（md以上）: 3列のグリッド（横並び）
    また、flex-col sm:flex-rowなども使用し、 画面サイズに応じて配置を切り替えています。
