レスポンシブデザイン - 画面サイズに応じた調整

Tailwindのレスポンシブプレフィックス（sm:、md:、lg:、xl:）を使って、異なる画面サイズに対応したデザインを実装する方法を学びます。モバイルファーストアプローチの実践方法も習得します。

この章で学ぶこと

  • レスポンシブデザインの概念とモバイルファーストアプローチの理解
  • Tailwindのブレークポイント（sm、md、lg、xl、2xl）の使い方を習得
  • 画面サイズごとに異なるレイアウトを実装できる
  • 実際のレスポンシブWebサイトを作成できる

7.1 レスポンシブデザインとは

レスポンシブデザインとは、スマートフォン、タブレット、パソコンなど異なる画面サイズの端末で適切に表示されるWebサイトを作る手法です。 同じHTMLコードでも、画面サイズに応じてレイアウトやスタイルが自動的に調整されます。

デバイス別の表示イメージ

モバイルファーストとは

まずスマートフォン向けのデザインを基本として作成し、画面が大きくなるにつれてレイアウトを調整していく手法です。 現代のWebサイトは多くがスマートフォンからアクセスされるため、この考え方が主流になっています。

7.2 Tailwindのブレークポイント

Tailwind CSSでは、画面サイズを5つのブレークポイントに分けて、それぞれに異なるスタイルを指定できます。 プレフィックスとして sm:、md:、 lg: などを使います。

プレフィックス 最小幅 デバイス例
なし 0px〜 全端末（モバイルファーストの基本）
sm: 640px〜 大きめのスマートフォン（横向き）
md: 768px〜 タブレット
lg: 1024px〜 小型ノートパソコン
xl: 1280px〜 デスクトップ
2xl: 1536px〜 大型ディスプレイ

重要ポイント

プレフィックスなしのクラスは全画面サイズに適用され、sm:、md: などは その幅以上のときに適用されます。小さい画面から順に考えるモバイルファーストの思想です。

7.3 基本的なレスポンシブ指定

レスポンシブプレフィックスは、どのユーティリティクラスにも付けられます。 例えば、文字サイズを画面サイズごとに変えてみましょう。

responsive-text.html 
<h1 class="text-2xl sm:text-3xl md:text-4xl lg:text-5xl xl:text-6xl font-bold">
    レスポンシブな見出し
</h1>

表示例（ブラウザの幅を変えて確認）

レスポンシブな見出し

各画面サイズでの文字サイズ

モバイル（〜640px）: text-2xl = 1.5rem（24px）
sm（640px〜）: text-3xl = 1.875rem（30px）
md（768px〜）: text-4xl = 2.25rem（36px）
lg（1024px〜）: text-5xl = 3rem（48px）
xl（1280px〜）: text-6xl = 3.75rem（60px）

7.4 レイアウトのレスポンシブ対応

最もよく使われるのが、FlexboxやGridのレスポンシブ対応です。 モバイルでは縦並び、タブレット以上では横並びにする、といった実装が簡単にできます。

例1: カードを画面サイズに応じて並べる

responsive-cards.html 
<!-- モバイル: 1列、タブレット: 2列、デスクトップ: 3列 -->
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
    <div class="bg-white p-6 rounded-lg shadow-md">
        <h3 class="font-bold text-lg mb-2">カード1</h3>
        <p class="text-slate-600">内容1</p>
    </div>
    <div class="bg-white p-6 rounded-lg shadow-md">
        <h3 class="font-bold text-lg mb-2">カード2</h3>
        <p class="text-slate-600">内容2</p>
    </div>
    <div class="bg-white p-6 rounded-lg shadow-md">
        <h3 class="font-bold text-lg mb-2">カード3</h3>
        <p class="text-slate-600">内容3</p>
    </div>
</div>

7.5 表示・非表示の切り替え

画面サイズに応じて要素を表示したり非表示にしたりできます。 hiddenとblockを組み合わせます。

responsive-visibility.html 
<!-- モバイルでは非表示、タブレット以上で表示 -->
<div class="hidden md:block bg-green-100 p-4 rounded-lg">
    この要素はタブレット以上でのみ表示されます
</div>

<!-- モバイルでは表示、タブレット以上で非表示 -->
<div class="block md:hidden bg-orange-100 p-4 rounded-lg">
    この要素はモバイルでのみ表示されます
</div>

表示例（ブラウザの幅を変えて確認）

この要素はタブレット以上でのみ表示されます（md:以上）

この要素はモバイルでのみ表示されます（〜md未満）

ヒント

ハンバーガーメニューなど、モバイルとデスクトップで異なるナビゲーションを用意する場合によく使われます。

7.6 レスポンシブなスペーシング

padding、margin、gapなども画面サイズに応じて調整できます。 モバイルでは詰め気味、デスクトップでは余裕を持たせるのが一般的です。

responsive-spacing.html 
<!-- paddingを段階的に大きくする -->
<div class="p-4 md:p-6 lg:p-8 xl:p-10 bg-purple-100 rounded-lg">
    <h3 class="font-bold mb-2">レスポンシブなパディング</h3>
    <p>画面が大きくなるほど余白が広がります</p>
</div>

<!-- gap（間隔）も調整可能 -->
<div class="flex gap-2 md:gap-4 lg:gap-6">
    <div class="flex-1 bg-blue-200 p-4 rounded">Box 1</div>
    <div class="flex-1 bg-blue-200 p-4 rounded">Box 2</div>
</div>

実習 7-1: レスポンシブなカードリストの作成

画面サイズに応じてカラム数が変わるカードリストを作成してみましょう。

手順

  1. 1 新しいHTMLファイル（responsive-practice.html）を作成します
  2. 2 Gridレイアウトで、モバイル: 1列、タブレット: 2列、デスクトップ: 4列になるように設定します
  3. 3 8つのカードを作成し、それぞれに画像、タイトル、説明文を入れます
  4. 4 ブラウザの幅を変えて、レイアウトが切り替わることを確認します

期待される結果

ブラウザの幅を変えると、カードが1列→2列→4列と自動的にレイアウトが変化します。 各画面サイズで適切な見た目になっていることを確認してください。

理解度確認クイズ

  1. Q1. モバイルファーストアプローチとは何ですか？

    回答を表示

    まずスマートフォン向けのデザインを基本として作成し、画面が大きくなるにつれてレイアウトを調整していく手法です。 Tailwindではプレフィックスなしのクラスがモバイル向け、sm:以降が大きな画面向けになります。

  2. Q2. md:grid-cols-2は何を意味しますか？

    回答を表示

    画面幅が768px以上（タブレットサイズ）のときに、グリッドレイアウトを2列にするという意味です。 それより小さい画面では、この指定は適用されず、基本のレイアウトが使われます。

  3. Q3. hidden md:blockの動作を説明してください。

    回答を表示

    基本的に非表示（hidden）ですが、 画面幅が768px以上になると表示（block）されます。 つまり、モバイルでは隠れていて、タブレット以上で表示される要素になります。

この章のまとめ

  • レスポンシブデザインは、異なる画面サイズに対応したWebサイトを作る手法です
  • Tailwindではsm:、md:、lg:、xl:、2xl:のプレフィックスで画面サイズを指定します
  • モバイルファーストアプローチでは、プレフィックスなしがモバイル向け、プレフィックス付きが大画面向けになります
  • レイアウト、文字サイズ、スペーシング、表示・非表示など、あらゆるスタイルをレスポンシブに調整できます
  • ブラウザの開発者ツールでデバイスをエミュレートして、各画面サイズでの表示を確認しましょう
