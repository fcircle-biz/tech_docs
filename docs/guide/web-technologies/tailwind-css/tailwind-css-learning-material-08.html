第8章

ホバーとフォーカス - インタラクティブな要素

hover:、focus:、active:などの状態プレフィックスを使って、ユーザーの操作に反応する要素を作成します。ボタン、リンク、フォーム要素のインタラクティブな動作を実装します。

この章で学ぶこと

  • hover:、focus:、active:などの状態プレフィックスの使い方を理解する
  • ボタンのインタラクティブなスタイリングを実装できる
  • リンクとフォーム要素のホバーエフェクトを作成できる
  • 複数の状態を組み合わせた実践的な実装ができる

8.1 インタラクティブな要素とは

インタラクティブな要素とは、ユーザーの操作（マウスのホバー、クリック、キーボードフォーカスなど）に反応して見た目が変化する要素のことです。 これにより、ユーザーに「操作可能である」ことを視覚的にフィードバックし、使いやすいインターフェースを実現できます。

Tailwind CSSでは、状態プレフィックスを使って、異なる状態でのスタイルを簡単に定義できます。 従来のCSSでは :hover 擬似クラスを使いますが、 Tailwind CSSでは hover: プレフィックスをクラス名の前に付けるだけです。

主な状態プレフィックス

flowchart LR Element[要素] --> Hover[hover:

フォーカス時] Element --> Active[active:

状態プレフィックスの仕組み

hover:bg-blue-600 は、 「マウスホバー時に背景色を青色にする」という意味になります。 通常時のスタイルと組み合わせて使用することで、動的な見た目を実現します。

8.2 hover:プレフィックス - マウスホバー時のスタイル

hover: プレフィックスは、 マウスカーソルが要素の上に乗ったときのスタイルを指定します。 ボタン、リンク、カードなど、あらゆる要素に適用できます。

基本的なボタンのホバー効果

hover-button.html 
<!-- 背景色が変わるボタン -->
<button class="bg-blue-500 hover:bg-blue-600 text-white px-6 py-3 rounded-lg">
    ホバーしてみてください
</button>

<!-- 影が濃くなるボタン -->
<button class="bg-green-500 text-white px-6 py-3 rounded-lg
               shadow-md hover:shadow-xl transition-shadow">
    影が変化
</button>

<!-- 拡大するボタン -->
<button class="bg-purple-500 text-white px-6 py-3 rounded-lg
               transform hover:scale-105 transition-transform">
    拡大効果
</button>

表示例（実際にホバーしてみてください）

リンクのホバー効果

hover-links.html 
<!-- 下線が表示されるリンク -->
<a href="#" class="text-blue-600 hover:underline">
    ホバーで下線表示
</a>

<!-- 色と下線が変わるリンク -->
<a href="#" class="text-blue-600 hover:text-blue-800
                   underline decoration-blue-300 hover:decoration-blue-800
                   transition-colors">
    色と下線が変化
</a>

<!-- アイコン付きリンク（アイコンが移動） -->
<a href="#" class="inline-flex items-center gap-2 text-blue-600 hover:text-blue-800">
    詳細を見る
    <i class="fas fa-arrow-right transition-transform hover:translate-x-1"></i>
</a>

表示例

ヒント: transitionクラス

transition-colors、 transition-transform などを追加すると、 スタイルの変化が滑らかなアニメーションになります。

8.3 focus:プレフィックス - フォーカス時のスタイル

focus: プレフィックスは、 要素がキーボードフォーカスを受けたときのスタイルを指定します。 特にフォーム要素（入力欄、ボタン、セレクトボックスなど）でよく使われます。

フォーカススタイルは、キーボード操作のユーザーにとって「どこを操作しているか」を示す重要な視覚的手がかりです。 アクセシビリティの観点からも必須です。

入力欄のフォーカススタイル

focus-input.html 
<!-- ボーダー色が変わる入力欄 -->
<input type="text"
       placeholder="クリックしてフォーカス"
       class="w-full px-4 py-2 border border-slate-300 rounded-lg
              focus:border-blue-500 focus:outline-none" />

<!-- リング（枠線）が表示される入力欄 -->
<input type="email"
       placeholder="メールアドレス"
       class="w-full px-4 py-2 border border-slate-300 rounded-lg
              focus:ring-2 focus:ring-blue-500 focus:outline-none" />

<!-- 背景色も変わる入力欄 -->
<input type="text"
       placeholder="高度な例"
       class="w-full px-4 py-2 border border-slate-300 rounded-lg
              bg-slate-50 focus:bg-white focus:border-blue-500
              focus:ring-2 focus:ring-blue-200 focus:outline-none
              transition-all" />

表示例（クリックしてフォーカスしてください）

focus:outline-noneの使用

focus:outline-none は、 ブラウザのデフォルトのアウトライン（青い枠線）を消すクラスです。 ただし、この場合は必ず focus:ring-* や focus:border-* で 代替の視覚的フィードバックを提供してください。

テキストエリアとボタンのフォーカス

focus-advanced.html 
<!-- テキストエリア -->
<textarea placeholder="コメントを入力"
          class="w-full px-4 py-2 border border-slate-300 rounded-lg
                 focus:ring-2 focus:ring-purple-500 focus:border-purple-500
                 focus:outline-none transition-all"
          rows="4"></textarea>

<!-- フォーカス時にリングが表示されるボタン -->
<button class="bg-blue-500 hover:bg-blue-600 text-white px-6 py-3 rounded-lg
               focus:ring-4 focus:ring-blue-300 focus:outline-none
               transition-all">
    送信
</button>

表示例

8.4 active:プレフィックス - クリック時のスタイル

active: プレフィックスは、 要素がクリックされている瞬間（マウスボタンが押されている間）のスタイルを指定します。 ボタンを押したときの「押し込まれた」感覚を表現するのに使います。

active-button.html 
<!-- クリック時に縮小するボタン -->
<button class="bg-blue-500 hover:bg-blue-600 active:bg-blue-700
               text-white px-6 py-3 rounded-lg
               transform active:scale-95 transition-all">
    押してみてください
</button>

<!-- クリック時に影が小さくなるボタン -->
<button class="bg-green-500 hover:bg-green-600 active:bg-green-700
               text-white px-6 py-3 rounded-lg
               shadow-lg active:shadow-md transition-all">
    押し込まれる感覚
</button>

<!-- すべての状態を組み合わせたボタン -->
<button class="bg-purple-500 hover:bg-purple-600 active:bg-purple-700
               focus:ring-4 focus:ring-purple-300
               text-white px-6 py-3 rounded-lg
               transform hover:scale-105 active:scale-95
               shadow-md hover:shadow-lg active:shadow
               transition-all duration-200">
    完全な例
</button>

表示例（クリックしてみてください）

重要ポイント: 状態の組み合わせ

hover:、 focus:、 active: は すべて同じ要素に適用できます。 適切に組み合わせることで、操作感の良いインターフェースを作れます。

8.5 その他の状態プレフィックス

disabled:プレフィックス - 無効化状態

disabled: プレフィックスは、 無効化されたフォーム要素のスタイルを指定します。

disabled-example.html 
<!-- 無効化されたボタン -->
<button disabled
        class="bg-blue-500 text-white px-6 py-3 rounded-lg
               disabled:bg-slate-300 disabled:text-slate-500
               disabled:cursor-not-allowed">
    無効化されたボタン
</button>

<!-- 無効化された入力欄 -->
<input type="text"
       disabled
       value="編集できません"
       class="w-full px-4 py-2 border border-slate-300 rounded-lg
              disabled:bg-slate-100 disabled:text-slate-500
              disabled:cursor-not-allowed" />

表示例

group-hover:プレフィックス - 親要素のホバー

group-hover: プレフィックスは、 親要素にホバーしたときに子要素のスタイルを変更します。 親要素に group クラスを追加する必要があります。

group-hover.html 
<!-- カード全体にホバーしたときアイコンが変化 -->
<div class="group p-6 bg-white border border-slate-200 rounded-xl
            hover:shadow-lg transition-all cursor-pointer">
    <div class="flex items-center justify-between">
        <h3 class="text-lg font-semibold text-slate-800">プロジェクトカード</h3>
        <i class="fas fa-arrow-right text-slate-400
                  group-hover:text-blue-500 group-hover:translate-x-2
                  transition-all"></i>
    </div>
    <p class="text-slate-600 mt-2">
        カード全体にホバーするとアイコンが移動します
    </p>
</div>

実習 8-1: インタラクティブなボタンセットの作成

hover、focus、activeの状態を組み合わせた、異なるスタイルのボタンを3つ作成しましょう。

手順

  1. 1 HTMLファイルを作成し、3つのボタンを配置します
  2. 2 1つ目: ホバー時に背景色が変わり、アクティブ時に縮小するボタン
  3. 3 2つ目: ホバー時に影が濃くなり、フォーカス時にリングが表示されるボタン
  4. 4 3つ目: すべての状態（hover、focus、active）を組み合わせたボタン

期待される結果

それぞれのボタンが、マウスホバー、フォーカス、クリック時に 滑らかなアニメーションで適切に反応します。

理解度確認クイズ

  1. Q1. hover:プレフィックスはどのような時にスタイルが適用されますか？

    回答を表示

    マウスカーソルが要素の上に乗ったとき（ホバーしたとき）にスタイルが適用されます。 例えば、hover:bg-blue-600 は、 ホバー時に背景色を青色にします。

  2. Q2. フォーム要素でfocus:プレフィックスを使う理由は何ですか？

    回答を表示

    キーボード操作のユーザーに「どの要素を操作しているか」を視覚的にフィードバックするためです。 特にアクセシビリティの観点から重要で、 focus:ring-* や focus:border-* で 明確な視覚的手がかりを提供する必要があります。

  3. Q3. transition-*クラスを追加する効果は何ですか？

    回答を表示

    スタイルの変化を滑らかなアニメーションにします。 transition-colors は色の変化を、 transition-transform は 変形（拡大・縮小・移動など）を滑らかにします。 これにより、ユーザー体験が向上します。

この章のまとめ

  • hover:プレフィックスでマウスホバー時のスタイルを指定できます
  • focus:プレフィックスでフォーカス時のスタイルを指定でき、アクセシビリティに重要です
  • active:プレフィックスでクリック時のスタイルを指定し、押し込まれた感覚を表現できます
  • disabled:プレフィックスで無効化された要素のスタイルを指定できます
  • group-hover:プレフィックスで親要素のホバー時に子要素のスタイルを変更できます
  • transition-*クラスで状態変化を滑らかなアニメーションにできます
