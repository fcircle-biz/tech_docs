第6章: コンポーネントとパターンの構築

コンポーネントとパターンの構築

この章で学ぶこと
  • 再利用可能なコンポーネントパターンの作成方法を学ぶ
  • ボタン、カード、モーダルなどの一般的なUIコンポーネントを構築する
  • コンポーネントライブラリの設計思想を理解する
  • @applyディレクティブを使用したスタイルの抽出方法を学ぶ
  • 一貫性のあるデザインシステムを構築する

6.1 ボタンコンポーネント

ボタンは最も基本的で頻繁に使用されるUIコンポーネントです。一貫性のあるボタンスタイルを定義することで、UI全体の統一感を保てます。

基本的なボタンパターン

<!-- プライマリーボタン -->
<button class="bg-blue-600 hover:bg-blue-700 text-white font-semibold py-2 px-6 rounded-lg transition duration-200">
  プライマリー
</button>

<!-- セカンダリーボタン -->
<button class="bg-gray-200 hover:bg-gray-300 text-gray-800 font-semibold py-2 px-6 rounded-lg transition duration-200">
  セカンダリー
</button>

<!-- アウトラインボタン -->
<button class="border-2 border-blue-600 text-blue-600 hover:bg-blue-600 hover:text-white font-semibold py-2 px-6 rounded-lg transition duration-200">
  アウトライン
</button>

サイズバリエーション

<!-- 小サイズ -->
<button class="bg-blue-600 text-white py-1 px-3 text-sm rounded">小</button>

<!-- 通常サイズ -->
<button class="bg-blue-600 text-white py-2 px-6 rounded-lg">通常</button>

<!-- 大サイズ -->
<button class="bg-blue-600 text-white py-3 px-8 text-lg rounded-lg">大</button>

6.2 カードコンポーネント

カードは情報をグループ化して表示するための汎用的なコンテナです。

実習 6-1: 多機能カードコンポーネントの作成
<!-- 基本カード -->
<div class="max-w-sm bg-white rounded-lg shadow-md overflow-hidden">
  <!-- カード画像 -->
  <img src="image.jpg" alt="カード画像" class="w-full h-48 object-cover">

  <!-- カードコンテンツ -->
  <div class="p-6">
    <!-- バッジ -->
    <span class="inline-block bg-blue-100 text-blue-800 text-xs px-2 py-1 rounded-full mb-2">
      新着
    </span>

    <!-- タイトル -->
    <h3 class="text-xl font-bold text-gray-800 mb-2">
      カードタイトル
    </h3>

    <!-- 説明文 -->
    <p class="text-gray-600 mb-4">
      カードの説明文がここに入ります。
    </p>

    <!-- アクションボタン -->
    <button class="w-full bg-blue-600 hover:bg-blue-700 text-white py-2 rounded-lg">
      詳細を見る
    </button>
  </div>
</div>

6.3 モーダルコンポーネント

モーダルは重要な情報やアクションをユーザーに提示するためのダイアログです。

<!-- モーダルオーバーレイ -->
<div class="fixed inset-0 bg-black bg-opacity-50 flex items-center justify-center z-50">
  <!-- モーダルコンテンツ -->
  <div class="bg-white rounded-lg shadow-xl max-w-md w-full mx-4">
    <!-- ヘッダー -->
    <div class="flex justify-between items-center p-6 border-b">
      <h3 class="text-xl font-bold">モーダルタイトル</h3>
      <button class="text-gray-400 hover:text-gray-600">
        ✕
      </button>
    </div>

    <!-- ボディ -->
    <div class="p-6">
      <p class="text-gray-600">モーダルの内容がここに入ります。</p>
    </div>

    <!-- フッター -->
    <div class="flex justify-end gap-2 p-6 border-t">
      <button class="px-4 py-2 bg-gray-200 rounded-lg">キャンセル</button>
      <button class="px-4 py-2 bg-blue-600 text-white rounded-lg">確認</button>
    </div>
  </div>
</div>

6.5 @applyディレクティブの活用

同じスタイルの組み合わせを繰り返し使う場合、@applyを使用してカスタムクラスを作成できます。

/* src/input.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer components {
  .btn-primary {
    @apply bg-blue-600 hover:bg-blue-700 text-white font-semibold py-2 px-6 rounded-lg transition duration-200;
  }

  .card {
    @apply bg-white rounded-lg shadow-md overflow-hidden;
  }

  .card-body {
    @apply p-6;
  }
}

使用例

<button class="btn-primary">プライマリーボタン</button>

<div class="card">
  <div class="card-body">
    カードコンテンツ
  </div>
</div>
理解度確認クイズ
  1. @applyディレクティブを使用する利点と欠点は何ですか？

    解答：利点は、繰り返し使用するスタイルの組み合わせを再利用可能なクラスにまとめられることです。欠点は、ユーティリティファーストの原則から逸脱し、CSSファイルが肥大化する可能性があることです。本当に繰り返し使用する場合にのみ使用すべきです。

  2. モーダルにfixed inset-0を使用する理由は何ですか？

    解答：fixedで画面に固定し、inset-0で上下左右すべて0にすることで、画面全体を覆うオーバーレイを作成できます。

6.6 まとめ

  • 一般的なUIコンポーネントはユーティリティクラスの組み合わせで構築できる
  • 一貫性のあるデザインパターンを定義することが重要
  • @applyディレクティブで繰り返し使用するスタイルを抽出可能
  • レスポンシブ対応とアクセシビリティを考慮する
