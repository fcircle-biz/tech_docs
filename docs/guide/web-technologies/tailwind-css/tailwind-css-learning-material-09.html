コンポーネントの作成 - よく使うUIパーツ

ボタン、カード、ナビゲーション、フォーム、モーダルなど、実際のWebサイトでよく使われるUIコンポーネントをTailwind CSSで作成する方法を学びます。

この章で学ぶこと

  • ボタンコンポーネントのバリエーション作成
  • カードコンポーネントの設計と実装
  • ナビゲーションバーの作成
  • フォーム要素のスタイリング
  • モーダルウィンドウの実装
  • 再利用可能なコンポーネント設計の考え方

9.1 コンポーネント設計の考え方

Tailwind CSSでコンポーネントを作成する際は、再利用性と一貫性を意識することが重要です。 同じようなスタイルを何度も書くのではなく、パターン化して繰り返し使えるようにします。

コンポーネント設計の流れ

flowchart LR A[デザイン要件を確認] --> B[基本構造をHTML化] B --> C[Tailwindクラスでスタイリング] C --> D[バリエーションを検討] D --> E[レスポンシブ対応] E --> F[インタラクティブ要素追加] F --> G[完成]

コンポーネント設計のポイント

  • 一貫性: 同じ用途のコンポーネントは同じスタイルパターンを使う
  • 柔軟性: サイズや色のバリエーションを用意する
  • アクセシビリティ: キーボード操作やスクリーンリーダーにも対応
  • レスポンシブ: すべての画面サイズで適切に表示される

9.2 ボタンコンポーネント

ボタンは最も基本的なUIコンポーネントです。用途やデザインに応じて、サイズ・色・スタイルのバリエーションを作成します。

基本のボタン

button-basic.html 
<!-- プライマリーボタン -->
<button class="bg-blue-500 hover:bg-blue-600 text-white font-medium
               px-4 py-2 rounded-lg transition-colors">
    プライマリーボタン
</button>

<!-- セカンダリーボタン -->
<button class="bg-gray-200 hover:bg-gray-300 text-gray-800 font-medium
               px-4 py-2 rounded-lg transition-colors">
    セカンダリーボタン
</button>

<!-- アウトラインボタン -->
<button class="border-2 border-blue-500 text-blue-500 hover:bg-blue-500
               hover:text-white font-medium px-4 py-2 rounded-lg transition-all">
    アウトラインボタン
</button>

9.3 カードコンポーネント

カードは情報をまとめて表示するための汎用的なコンポーネントです。画像、タイトル、説明文、アクションボタンなどを組み合わせて作成します。

基本的なカード

card-basic.html 
<div class="max-w-sm bg-white rounded-xl shadow-lg overflow-hidden">
    <!-- 画像エリア -->
    <div class="h-48 bg-gradient-to-r from-purple-400 to-pink-400"></div>

    <!-- コンテンツエリア -->
    <div class="p-6">
        <h3 class="text-xl font-bold text-gray-800 mb-2">
            カードタイトル
        </h3>
        <p class="text-gray-600 mb-4">
            カードの説明文がここに入ります。Tailwind CSSで簡単に美しいカードを作成できます。
        </p>
        <button class="bg-blue-500 hover:bg-blue-600 text-white px-4 py-2
                       rounded-lg transition-colors">
            詳細を見る
        </button>
    </div>
</div>

9.4 ナビゲーションバー

ナビゲーションバーはWebサイトの主要なメニューを提供するコンポーネントです。ロゴ、メニューリンク、アクションボタンなどを配置します。

navbar.html 
<nav class="bg-white shadow-lg">
    <div class="max-w-6xl mx-auto px-4">
        <div class="flex justify-between items-center py-4">
            <!-- ロゴ -->
            <div class="text-2xl font-bold text-blue-600">
                MyApp
            </div>

            <!-- メニューリンク -->
            <div class="flex gap-6">
                <a href="#" class="text-gray-700 hover:text-blue-600 transition-colors">
                    ホーム
                </a>
                <a href="#" class="text-gray-700 hover:text-blue-600 transition-colors">
                    機能
                </a>
                <a href="#" class="text-gray-700 hover:text-blue-600 transition-colors">
                    料金
                </a>
            </div>

            <!-- アクションボタン -->
            <button class="bg-blue-500 hover:bg-blue-600 text-white px-4 py-2
                           rounded-lg transition-colors">
                ログイン
            </button>
        </div>
    </div>
</nav>

レスポンシブ対応のヒント

スマートフォンでは、メニューリンクをhidden md:flexで隠し、 ハンバーガーメニューボタンを表示するようにします。ボタンクリックでメニューが開閉するようJavaScriptを追加します。

9.5 フォームコンポーネント

フォーム要素は、ユーザーからの入力を受け付けるための重要なコンポーネントです。入力フィールド、選択ボックス、チェックボックスなどをスタイリングします。

form.html 
<form class="max-w-md mx-auto bg-white rounded-xl shadow-lg p-6 space-y-4">
    <!-- テキスト入力 -->
    <div>
        <label class="block text-sm font-medium text-gray-700 mb-2">
            お名前
        </label>
        <input type="text"
               class="w-full px-4 py-2 border border-gray-300 rounded-lg
                      focus:outline-none focus:ring-2 focus:ring-blue-500"
               placeholder="山田太郎">
    </div>

    <!-- メールアドレス -->
    <div>
        <label class="block text-sm font-medium text-gray-700 mb-2">
            メールアドレス
        </label>
        <input type="email"
               class="w-full px-4 py-2 border border-gray-300 rounded-lg
                      focus:outline-none focus:ring-2 focus:ring-blue-500"
               placeholder="example@example.com">
    </div>

    <!-- セレクトボックス -->
    <div>
        <label class="block text-sm font-medium text-gray-700 mb-2">
            お問い合わせ種別
        </label>
        <select class="w-full px-4 py-2 border border-gray-300 rounded-lg
                       focus:outline-none focus:ring-2 focus:ring-blue-500">
            <option>選択してください</option>
            <option>製品について</option>
            <option>サポート</option>
        </select>
    </div>

    <!-- 送信ボタン -->
    <button type="submit"
            class="w-full bg-blue-500 hover:bg-blue-600 text-white font-medium
                   px-4 py-2 rounded-lg transition-colors">
        送信する
    </button>
</form>

表示例

フォーカスリングについて

focus:outline-noneでデフォルトのアウトラインを消し、 focus:ring-2とfocus:ring-blue-500で カスタムのフォーカスリングを表示します。これにより、キーボード操作時の視認性が向上します。

9.6 モーダルウィンドウ

モーダルは、ユーザーの注意を引くために画面上に重ねて表示するダイアログです。 背景を暗くするオーバーレイと、中央に表示されるコンテンツボックスで構成されます。

modal.html 
<!-- モーダルオーバーレイ（背景） -->
<div class="fixed inset-0 bg-black bg-opacity-50 flex items-center justify-center z-50">
    <!-- モーダルコンテンツ -->
    <div class="bg-white rounded-xl shadow-2xl max-w-md w-full mx-4">
        <!-- ヘッダー -->
        <div class="flex items-center justify-between p-6 border-b">
            <h3 class="text-xl font-bold text-gray-800">
                確認
            </h3>
            <button class="text-gray-400 hover:text-gray-600 transition-colors">
                <i class="fas fa-times text-xl"></i>
            </button>
        </div>

        <!-- コンテンツ -->
        <div class="p-6">
            <p class="text-gray-600">
                この操作を実行してもよろしいですか？
            </p>
        </div>

        <!-- フッター -->
        <div class="flex gap-3 p-6 border-t">
            <button class="flex-1 bg-gray-200 hover:bg-gray-300 text-gray-800
                           px-4 py-2 rounded-lg transition-colors">
                キャンセル
            </button>
            <button class="flex-1 bg-blue-500 hover:bg-blue-600 text-white
                           px-4 py-2 rounded-lg transition-colors">
                実行
            </button>
        </div>
    </div>
</div>

9.7 コンポーネントの再利用

同じスタイルのコンポーネントを何度も使う場合、効率的に再利用する方法があります。

再利用の方法

1. HTMLのコピー＆ペースト

最もシンプルな方法。同じHTMLを必要な箇所にコピーします。

メリット：簡単で直感的
デメリット：変更時にすべての箇所を修正する必要がある

2. JavaScriptコンポーネント

React、Vue.jsなどのフレームワークを使ってコンポーネント化します。

メリット：保守性が高い、再利用が容易
デメリット：学習コストがかかる

3. CSSクラスの抽出

よく使うクラスの組み合わせを、カスタムCSSクラスとして定義します。

メリット：HTMLがシンプルになる
デメリット：Tailwindの利点が減る

4. テンプレートエンジン

EJS、Handlebarsなどのテンプレートエンジンで部品化します。

メリット：サーバーサイドで再利用できる
デメリット：環境構築が必要

推奨される方法

小規模なプロジェクトではHTMLのコピー＆ペーストでも十分です。 中〜大規模プロジェクトでは、React・Vue.jsなどのJavaScriptフレームワークと Tailwind CSSを組み合わせることが推奨されます。

実習 9-1: プロフィールカードの作成

学習した内容を活かして、プロフィール情報を表示するカードコンポーネントを作成しましょう。

要件

  • 円形のプロフィール画像エリア（グラデーション背景）
  • 名前、役職、自己紹介文を表示
  • SNSアイコンボタンを3つ配置（Twitter、GitHub、LinkedIn）
  • 「詳細を見る」ボタンを配置

ヒント

  • 円形画像はrounded-fullを使用
  • SNSアイコンはflex gap-3で横並びに
  • 中央揃えにはtext-centerやmx-autoを活用

理解度確認クイズ

  1. Q1. ボタンにホバー効果を追加するには、どのクラスを使用しますか？

    回答を表示

    hover:プレフィックスを使用します。 例: hover:bg-blue-600でホバー時の背景色を変更、 hover:text-whiteでテキスト色を変更できます。 さらにtransition-colorsを追加すると滑らかなアニメーションになります。

  2. Q2. カードコンポーネントで画像がはみ出さないようにするには、どのクラスを使用しますか？

    回答を表示

    overflow-hiddenを親要素に追加します。 これにより、rounded-xlなどで角を丸くした場合でも、 画像が角の外側にはみ出さず、綺麗に表示されます。

  3. Q3. フォーム入力フィールドにフォーカス時のリングを表示するには？

    回答を表示

    focus:outline-noneでデフォルトのアウトラインを消し、 focus:ring-2 focus:ring-blue-500で カスタムのフォーカスリングを表示します。これによりアクセシビリティが向上します。

  4. Q4. モーダルを画面中央に配置するには？

    回答を表示

    オーバーレイ要素にfixed inset-0で全画面に固定し、 flex items-center justify-centerで 子要素を縦横中央に配置します。これでモーダルが画面の中央に表示されます。

この章のまとめ

  • コンポーネント設計では再利用性と一貫性を意識することが重要です
  • ボタンはサイズ、色、スタイルのバリエーションを用意し、用途に応じて使い分けます
  • カードコンポーネントは画像、テキスト、ボタンを組み合わせて情報をまとめて表示します
  • ナビゲーションバーはロゴ、メニュー、アクションボタンで構成され、レスポンシブ対応が必要です
  • フォーム要素はfocus:ringでフォーカス状態を視覚化します
  • モーダルはfixedとflexで画面中央に配置します
  • 大規模プロジェクトでは、React・Vue.jsなどのフレームワークと組み合わせてコンポーネントを再利用します
