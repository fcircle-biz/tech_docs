第4章: レスポンシブデザインの実装

レスポンシブデザインの実装

この章で学ぶこと
  • Tailwind CSSのブレークポイントシステムを理解する
  • モバイルファーストアプローチの概念を習得する
  • 画面サイズごとに異なるスタイルを適用する方法を学ぶ
  • レスポンシブなグリッドレイアウトを作成する
  • 表示/非表示の切り替えをレスポンシブに実装する

4.1 モバイルファーストアプローチ

Tailwind CSSはモバイルファーストの設計哲学を採用しています。これは、最初にモバイル向けのスタイルを定義し、大きな画面向けに徐々にスタイルを追加していくアプローチです。

flowchart LR A["モバイル
(デフォルト)"] --> B["タブレット
(md:以上)"] B --> C["デスクトップ
(lg:以上)"] C --> D["大画面
(xl:以上)"]

従来のデスクトップファーストとの違い

<!-- モバイルファースト（Tailwind CSS） -->
<div class="text-sm md:text-base lg:text-lg">
  レスポンシブテキスト
</div>
<!-- 小さい画面から大きい画面へ段階的にサイズアップ -->

<!-- デスクトップファースト（従来の方法） -->
<div class="text-lg md:text-base sm:text-sm">
  レスポンシブテキスト
</div>
<!-- 大きい画面から小さい画面へ段階的にサイズダウン -->

4.2 ブレークポイントシステム

Tailwind CSSには、5つの標準ブレークポイントが用意されています。

プレフィックス 最小幅 CSS相当 対象デバイス
（なし）0px-モバイル（デフォルト）
sm:640px@media (min-width: 640px)大きめのスマートフォン
md:768px@media (min-width: 768px)タブレット
lg:1024px@media (min-width: 1024px)デスクトップ
xl:1280px@media (min-width: 1280px)大きなデスクトップ
2xl:1536px@media (min-width: 1536px)超大画面

基本的な使い方

<!-- 画面サイズに応じてテキストサイズを変更 -->
<h1 class="text-2xl md:text-4xl lg:text-6xl">
  レスポンシブ見出し
</h1>

<!-- 画面サイズに応じて背景色を変更 -->
<div class="bg-blue-300 md:bg-green-300 lg:bg-purple-300">
  画面サイズで色が変わる
</div>

4.3 レスポンシブなレイアウト

レスポンシブグリッド

画面サイズに応じてカラム数を変更する例：

<!-- モバイル: 1カラム、タブレット: 2カラム、デスクトップ: 3カラム -->
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <div class="bg-blue-200 p-4">アイテム1</div>
  <div class="bg-blue-200 p-4">アイテム2</div>
  <div class="bg-blue-200 p-4">アイテム3</div>
  <div class="bg-blue-200 p-4">アイテム4</div>
  <div class="bg-blue-200 p-4">アイテム5</div>
  <div class="bg-blue-200 p-4">アイテム6</div>
</div>

レスポンシブなスペーシング

<!-- パディングを画面サイズで調整 -->
<div class="p-4 md:p-8 lg:p-12">
  画面が大きくなるほど余白も大きく
</div>

<!-- マージンを画面サイズで調整 -->
<div class="mt-2 md:mt-4 lg:mt-8">
  上マージンがレスポンシブに変化
</div>

Flexboxの方向切り替え

<!-- モバイル: 縦並び、デスクトップ: 横並び -->
<div class="flex flex-col lg:flex-row gap-4">
  <div class="flex-1 bg-blue-200 p-4">左側/上側</div>
  <div class="flex-1 bg-green-200 p-4">右側/下側</div>
</div>
実習 4-1: レスポンシブカードグリッドの作成

画面サイズに応じて最適なレイアウトに変化するカードグリッドを作成します。

要件
  • モバイル（sm未満）: 1カラム
  • タブレット（md）: 2カラム
  • デスクトップ（lg）: 3カラム
  • 大画面（xl）: 4カラム
実装例
<div class="container mx-auto px-4 py-8">
  <h2 class="text-2xl md:text-3xl lg:text-4xl font-bold mb-6">
    レスポンシブカードグリッド
  </h2>

  <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6">
    <!-- カード1 -->
    <div class="bg-white rounded-lg shadow-md overflow-hidden">
      <div class="h-48 bg-gradient-to-br from-blue-400 to-blue-600"></div>
      <div class="p-4 md:p-6">
        <h3 class="text-lg md:text-xl font-bold mb-2">商品タイトル</h3>
        <p class="text-sm md:text-base text-gray-600">商品の説明文</p>
      </div>
    </div>

    <!-- 他のカードも同様に... -->
  </div>
</div>

4.4 表示/非表示の制御

画面サイズに応じて要素を表示・非表示にできます。

基本的な使い方

<!-- モバイルでのみ表示 -->
<div class="block md:hidden">
  モバイル専用コンテンツ
</div>

<!-- タブレット以上で表示 -->
<div class="hidden md:block">
  タブレット・デスクトップ専用コンテンツ
</div>

<!-- デスクトップのみ表示 -->
<div class="hidden lg:block">
  デスクトップ専用コンテンツ
</div>

レスポンシブナビゲーションの例

<nav class="bg-gray-800 text-white">
  <div class="container mx-auto px-4">
    <div class="flex justify-between items-center h-16">
      <!-- ロゴ -->
      <div class="text-xl font-bold">MyWebsite</div>

      <!-- デスクトップメニュー（lg以上で表示） -->
      <div class="hidden lg:flex space-x-4">
        <a href="#">ホーム</a>
        <a href="#">サービス</a>
        <a href="#">お問い合わせ</a>
      </div>

      <!-- モバイルメニューボタン（lg未満で表示） -->
      <button class="lg:hidden">
        <!-- ハンバーガーアイコン -->
        ☰
      </button>
    </div>
  </div>
</nav>

4.5 レスポンシブタイポグラフィ

画面サイズに応じてフォントサイズを調整することで、読みやすさを保ちます。

<!-- 見出しのレスポンシブサイズ -->
<h1 class="text-3xl sm:text-4xl md:text-5xl lg:text-6xl font-bold">
  大見出し
</h1>

<h2 class="text-2xl sm:text-3xl md:text-4xl lg:text-5xl font-bold">
  中見出し
</h2>

<!-- 本文のレスポンシブサイズ -->
<p class="text-sm md:text-base lg:text-lg">
  本文テキストも画面サイズに応じて読みやすいサイズに調整されます。
</p>
理解度確認クイズ
  1. モバイルファーストアプローチとは何ですか？その利点を説明してください。

    解答例：モバイルファーストアプローチは、最初にモバイル向けのスタイルを定義し、大きな画面向けに段階的にスタイルを追加する設計手法です。利点は、モバイルユーザーの体験を優先し、必要なスタイルのみを段階的に追加できるため、パフォーマンスが向上します。

  2. md:text-2xlというクラスはどのような意味ですか？

    解答：画面幅が768px以上（mdブレークポイント）の場合にのみ、フォントサイズを2xlサイズ（1.5rem / 24px）に設定することを意味します。

  3. hidden md:blockとblock md:hiddenの違いを説明してください。

    解答：hidden md:blockは、デフォルト（モバイル）では非表示で、md以上で表示されます。block md:hiddenは、デフォルトで表示され、md以上で非表示になります。

4.6 まとめ

  • Tailwind CSSはモバイルファーストで、小さい画面から大きい画面へスタイルを追加
  • 5つの標準ブレークポイント（sm、md、lg、xl、2xl）を使用
  • プレフィックスを追加するだけで、画面サイズごとにスタイルを変更可能
  • レイアウト、スペーシング、タイポグラフィをレスポンシブに調整できる
  • 表示/非表示の制御で、デバイスごとに最適なUIを提供できる
