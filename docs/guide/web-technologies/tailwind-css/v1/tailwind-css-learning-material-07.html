第7章: フォームとインタラクティブ要素

フォームとインタラクティブ要素

この章で学ぶこと
  • フォーム要素のスタイリング方法を習得する
  • 擬似クラス（hover、focus、active）の使用方法を学ぶ
  • バリデーション表示の実装方法を理解する
  • アクセシブルなフォーム設計の基本を学ぶ
  • インタラクティブなユーザーフィードバックを実装する

7.1 基本的なフォーム要素

テキスト入力フィールド

<!-- 基本的な入力フィールド -->
<div class="mb-4">
  <label for="name" class="block text-sm font-medium text-gray-700 mb-2">
    お名前
  </label>
  <input
    type="text"
    id="name"
    class="w-full px-4 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-transparent"
    placeholder="山田太郎"
  >
</div>

テキストエリア

<div class="mb-4">
  <label for="message" class="block text-sm font-medium text-gray-700 mb-2">
    メッセージ
  </label>
  <textarea
    id="message"
    rows="4"
    class="w-full px-4 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500 resize-none"
    placeholder="メッセージを入力してください"
  ></textarea>
</div>

7.2 擬似クラスの活用

Tailwind CSSでは、擬似クラスを使用してインタラクティブな状態を簡単に制御できます。

Hoverスタイル

<!-- ホバー時に色が変わるボタン -->
<button class="bg-blue-600 hover:bg-blue-700 text-white px-6 py-2 rounded-lg">
  ホバーしてください
</button>

<!-- ホバー時に影が変わるカード -->
<div class="bg-white p-6 rounded-lg shadow-md hover:shadow-xl transition duration-300">
  ホバーで影が大きくなります
</div>

Focusスタイル

<!-- フォーカス時にリングが表示される入力フィールド -->
<input
  type="text"
  class="border border-gray-300 rounded-lg px-4 py-2 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-blue-500"
>

Activeスタイル

<!-- クリック時に色が濃くなるボタン -->
<button class="bg-blue-600 hover:bg-blue-700 active:bg-blue-800 text-white px-6 py-2 rounded-lg">
  押してください
</button>

7.5 バリデーション表示

フォームの検証結果を視覚的にフィードバックすることは、ユーザー体験の向上に重要です。

実習 7-1: バリデーション機能付きフォームの作成
<!-- 成功状態 -->
<div class="mb-4">
  <label class="block text-sm font-medium text-gray-700 mb-2">
    メールアドレス（正常）
  </label>
  <input
    type="email"
    class="w-full px-4 py-2 border-2 border-green-500 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-500"
    value="user@example.com"
  >
  <p class="mt-1 text-sm text-green-600">
    ✓ 有効なメールアドレスです
  </p>
</div>

<!-- エラー状態 -->
<div class="mb-4">
  <label class="block text-sm font-medium text-gray-700 mb-2">
    メールアドレス（エラー）
  </label>
  <input
    type="email"
    class="w-full px-4 py-2 border-2 border-red-500 rounded-lg focus:outline-none focus:ring-2 focus:ring-red-500"
    value="invalid-email"
  >
  <p class="mt-1 text-sm text-red-600">
    ✗ 有効なメールアドレスを入力してください
  </p>
</div>

7.6 完全なフォーム例

<form class="max-w-md mx-auto bg-white p-8 rounded-lg shadow-lg">
  <h2 class="text-2xl font-bold mb-6">お問い合わせフォーム</h2>

  <!-- 名前 -->
  <div class="mb-4">
    <label for="name" class="block text-sm font-medium text-gray-700 mb-2">
      お名前 <span class="text-red-500">*</span>
    </label>
    <input
      type="text"
      id="name"
      required
      class="w-full px-4 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
    >
  </div>

  <!-- メール -->
  <div class="mb-4">
    <label for="email" class="block text-sm font-medium text-gray-700 mb-2">
      メールアドレス <span class="text-red-500">*</span>
    </label>
    <input
      type="email"
      id="email"
      required
      class="w-full px-4 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
    >
  </div>

  <!-- メッセージ -->
  <div class="mb-6">
    <label for="message" class="block text-sm font-medium text-gray-700 mb-2">
      メッセージ
    </label>
    <textarea
      id="message"
      rows="4"
      class="w-full px-4 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
    ></textarea>
  </div>

  <!-- 送信ボタン -->
  <button
    type="submit"
    class="w-full bg-blue-600 hover:bg-blue-700 text-white font-semibold py-3 rounded-lg transition duration-200"
  >
    送信する
  </button>
</form>
理解度確認クイズ
  1. focus:ring-2とfocus:outline-noneを組み合わせる理由は何ですか？

    解答：デフォルトのブラウザアウトラインを非表示にし（outline-none）、代わりにカスタムのリング（ring-2）を表示することで、統一感のあるフォーカススタイルを実現します。

  2. バリデーションエラーを表示する際、色だけでなくアイコンやテキストも使用する理由は？

    解答：色覚障害のあるユーザーでも情報を理解できるようにするためです。アクセシビリティの観点から重要です。

7.7 まとめ

  • Tailwind CSSでフォーム要素を美しくスタイリングできる
  • 擬似クラス（hover、focus、active）でインタラクティブな体験を提供
  • バリデーション表示で適切なフィードバックを実装
  • アクセシビリティを考慮したフォーム設計が重要
