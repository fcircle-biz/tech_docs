第1章: Tailwind CSS入門と環境構築

Tailwind CSS入門と環境構築

この章で学ぶこと
  • Tailwind CSSとは何か、ユーティリティファーストCSSの概念を理解する
  • 従来のCSSアプローチとの違いを理解する
  • 開発環境のセットアップ方法を習得する
  • CDN版とビルド版の違いを理解する
  • 基本的なプロジェクト構成と設定ファイルの役割を学ぶ

1.1 Tailwind CSSとは

Tailwind CSSは、ユーティリティファーストCSSという革新的なアプローチを採用した、モダンなCSSフレームワークです。従来のCSSフレームワーク（BootstrapやFoundationなど）とは異なり、事前定義されたコンポーネント（ボタン、カード、ナビゲーションバーなど）を提供するのではなく、小さな単一目的のユーティリティクラスを提供します。

ユーティリティファーストとは？

ユーティリティファーストとは、HTMLに直接スタイルを適用するための小さなクラス（ユーティリティクラス）を使用するCSSの設計手法です。各クラスは通常1つのスタイルプロパティのみを担当し、それらを組み合わせることで複雑なデザインを構築します。

flowchart LR A["従来のCSSアプローチ"] --> B["独自のCSSクラスを定義"] B --> C["HTMLに適用"] D["Tailwind CSSアプローチ"] --> E["既存のユーティリティクラスを使用"] E --> F["HTMLに組み合わせて適用"]

従来のCSSアプローチとの比較

従来のCSSでは、各コンポーネントに対して独自のCSSクラスを作成し、スタイルを定義していました。

<!-- 従来のアプローチ -->
<style>
  .button-primary {
    background-color: blue;
    color: white;
    padding: 10px 20px;
    border-radius: 5px;
  }
</style>

<button class="button-primary">クリック</button>

一方、Tailwind CSSでは、ユーティリティクラスを組み合わせて同じ結果を実現します。

<!-- Tailwind CSSアプローチ -->
<button class="bg-blue-500 text-white px-5 py-2 rounded">クリック</button>

このアプローチの利点：

  • クラス名を考える必要がない - "button-primary"のような名前を考える時間が不要
  • CSSファイルの肥大化を防ぐ - 新しいスタイルを追加してもCSSファイルが増え続けない
  • 一貫性 - 事前定義された値（色、サイズ、スペーシング）を使用するため、デザインに一貫性が生まれる
  • 高速な開発 - HTMLを編集するだけでスタイルを変更できる

1.2 Tailwind CSSの特徴と利点

主な特徴

flowchart TD A["Tailwind CSSの特徴"] --> B["ユーティリティファースト"] A --> C["高度なカスタマイズ性"] A --> D["レスポンシブデザイン対応"] A --> E["プラグインエコシステム"] A --> F["パフォーマンス最適化"]
  1. 豊富なユーティリティクラス

    色、サイズ、スペーシング、レイアウト、タイポグラフィなど、あらゆるCSSプロパティに対応したユーティリティクラスが用意されています。

  2. レスポンシブデザインの簡単実装

    画面サイズごとに異なるスタイルを適用するためのプレフィックス（sm:、md:、lg:など）が用意されており、レスポンシブデザインを簡単に実装できます。

  3. JITモード（Just-In-Time）

    必要なスタイルのみを動的に生成するため、開発時のビルド速度が速く、ファイルサイズも小さくなります。

  4. カスタマイズ性

    設定ファイル（tailwind.config.js）を使用して、色、スペーシング、フォントなどを自由にカスタマイズできます。

1.3 環境構築：CDN版

Tailwind CSSを使い始める最も簡単な方法は、CDN（Content Delivery Network）を使用することです。これは学習やプロトタイプ作成に最適で、ビルドプロセスやインストールなしでTailwind CSSを試すことができます。

注意

CDN版は学習や簡単なプロトタイプに適していますが、本番環境では推奨されません。本番環境ではビルドプロセスを使用して、未使用のスタイルを削除し、ファイルサイズを最小化する必要があります。

CDN版の実装方法

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tailwind CSS CDN版</title>
  <!-- Tailwind CSS CDN -->
  <script src="https://cdn.tailwindcss.com"></script>
</head>
<body>
  <div class="min-h-screen bg-gray-100 flex items-center justify-center">
    <div class="bg-white p-8 rounded-lg shadow-lg">
      <h1 class="text-3xl font-bold text-blue-600">Hello Tailwind CSS!</h1>
      <p class="mt-4 text-gray-700">CDN版で簡単に始められます</p>
    </div>
  </div>
</body>
</html>

1.4 環境構築：ビルド版（推奨）

本格的なプロジェクトでは、ビルドプロセスを使用したTailwind CSSのセットアップが推奨されます。これにより、未使用のスタイルを自動的に削除し、カスタマイズも可能になります。

必要な環境

  • Node.js（バージョン14.0以上）
  • npm または yarn（パッケージマネージャー）
実習 1-1: Tailwind CSSプロジェクトのセットアップ

ビルド版のTailwind CSSを使用した基本的なプロジェクトを構築します。

手順
  1. プロジェクトディレクトリの作成
# 新しいディレクトリを作成して移動
mkdir my-tailwind-project
cd my-tailwind-project
  1. package.jsonの初期化
# npmプロジェクトを初期化（すべてデフォルト設定でOK）
npm init -y
  1. Tailwind CSSのインストール
# Tailwind CSSと必要な依存パッケージをインストール
npm install -D tailwindcss postcss autoprefixer
  1. Tailwind CSS設定ファイルの生成
# tailwind.config.jsとpostcss.config.jsを自動生成
npx tailwindcss init -p
  1. tailwind.config.jsの設定
// tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{html,js}",  // スキャン対象のファイルパスを指定
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
  1. CSSファイルの作成

src/input.css を作成し、以下を記述：

/* src/input.css */
@tailwind base;
@tailwind components;
@tailwind utilities;
  1. ビルドスクリプトの設定

package.jsonにビルドコマンドを追加：

{
  "scripts": {
    "build": "tailwindcss -i ./src/input.css -o ./dist/output.css",
    "watch": "tailwindcss -i ./src/input.css -o ./dist/output.css --watch"
  }
}
  1. HTMLファイルの作成

src/index.html を作成：

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Tailwind Project</title>
  <link href="../dist/output.css" rel="stylesheet">
</head>
<body>
  <h1 class="text-3xl font-bold underline">
    Hello Tailwind CSS!
  </h1>
</body>
</html>
  1. ビルドの実行
# CSSをビルド
npm run build

# または、自動監視モードで実行（ファイル変更時に自動再ビルド）
npm run watch
期待される結果

dist/output.cssファイルが生成され、HTMLファイルをブラウザで開くと、Tailwind CSSのスタイルが適用された「Hello Tailwind CSS!」という見出しが表示されます。

1.5 プロジェクト構成の理解

Tailwind CSSプロジェクトの基本的なディレクトリ構造を理解しましょう。

flowchart TD A["my-tailwind-project/"] --> B["node_modules/"] A --> C["src/"] A --> D["dist/"] A --> E["tailwind.config.js"] A --> F["postcss.config.js"] A --> G["package.json"] C --> H["input.css"] C --> I["index.html"] D --> J["output.css"]
  • node_modules/ - インストールされたパッケージが格納されるディレクトリ
  • src/ - ソースファイル（HTML、CSS）を配置するディレクトリ
  • dist/ - ビルドされたCSSファイルが出力されるディレクトリ
  • tailwind.config.js - Tailwind CSSの設定ファイル（カスタマイズに使用）
  • postcss.config.js - PostCSS（CSSプロセッサ）の設定ファイル
  • package.json - プロジェクトの依存関係とスクリプトを定義するファイル

1.6 tailwind.config.jsの役割

tailwind.config.jsは、Tailwind CSSのカスタマイズを行うための中心的な設定ファイルです。このファイルで、色、フォント、スペーシング、ブレークポイントなどを自由にカスタマイズできます。

基本的な構造

// tailwind.config.js
module.exports = {
  // スキャン対象のファイルパスを指定
  content: [
    "./src/**/*.{html,js}",
  ],

  // テーマのカスタマイズ
  theme: {
    extend: {
      // ここにカスタム設定を追加
      colors: {
        'brand-blue': '#1e40af',
      },
    },
  },

  // プラグインの追加
  plugins: [],
}

contentの設定

content配列には、Tailwind CSSがスキャンしてクラス名を探すファイルのパスを指定します。これにより、実際に使用されているクラスのみがビルドされます。

themeの設定

themeセクションでは、デフォルトのデザイントークン（色、スペーシング、フォントなど）をカスタマイズできます。extendキーを使用すると、デフォルト値を保持しながら新しい値を追加できます。

理解度確認クイズ
  1. ユーティリティファーストCSSとは何ですか？簡単に説明してください。

    解答例：小さな単一目的のクラス（ユーティリティクラス）をHTMLに直接適用してスタイルを構築するCSSの設計手法です。各クラスは通常1つのスタイルプロパティのみを担当し、それらを組み合わせることで複雑なデザインを作成します。

  2. CDN版とビルド版の主な違いは何ですか？

    解答例：CDN版は簡単に始められますが、すべてのスタイルが含まれるためファイルサイズが大きく、カスタマイズも制限されます。ビルド版は未使用のスタイルを削除でき、カスタマイズも自由に行えるため、本番環境に適しています。

  3. tailwind.config.jsファイルの主な役割は何ですか？

    解答例：Tailwind CSSのカスタマイズを行うための設定ファイルで、色、フォント、スペーシング、ブレークポイントなどのデザイントークンをカスタマイズしたり、スキャン対象のファイルパスを指定したりします。

  4. @tailwind base; @tailwind components; @tailwind utilities; の3行は何を意味しますか？

    解答例：これらはTailwind CSSのディレクティブで、それぞれベーススタイル、コンポーネントスタイル、ユーティリティクラスをCSSに挿入する命令です。ビルドプロセスでこれらが実際のCSSコードに置き換えられます。

1.7 まとめ

この章では、Tailwind CSSの基本概念と環境構築方法を学びました。

  • Tailwind CSSはユーティリティファーストCSSフレームワークで、小さなクラスを組み合わせてスタイルを構築する
  • 従来のCSSアプローチと比較して、クラス名を考える必要がなく、高速な開発が可能
  • CDN版は学習に便利だが、本番環境ではビルド版を使用すべき
  • tailwind.config.jsでカスタマイズを行い、プロジェクトに合わせた設定が可能
