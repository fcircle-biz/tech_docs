第2章: ユーティリティクラスの基礎

ユーティリティクラスの基礎

この章で学ぶこと
  • Tailwind CSSの命名規則とクラス構造を理解する
  • テキストスタイリング（サイズ、太さ、色）の基本を習得する
  • 色の適用方法とカラーパレットシステムを理解する
  • スペーシング（padding、margin）の使い方をマスターする
  • ボーダー、背景、シャドウなどの装飾クラスを学ぶ

2.1 Tailwind CSSの命名規則

Tailwind CSSのユーティリティクラスは、一貫性のある命名規則に従っています。この規則を理解することで、必要なクラス名を直感的に推測できるようになります。

基本的な命名パターン

ユーティリティクラスは通常、以下のパターンで命名されています：

flowchart LR A["プロパティ"] --> B["値"] B --> C["クラス名"] D["例: text"] --> E["例: blue-500"] E --> F["text-blue-500"]
  • text-{size} - テキストサイズ（例：text-sm, text-lg, text-2xl）
  • text-{color} - テキストカラー（例：text-blue-500, text-red-600）
  • bg-{color} - 背景色（例：bg-white, bg-gray-100）
  • p-{size} - パディング（例：p-4, p-8）
  • m-{size} - マージン（例：m-2, m-auto）
  • w-{size} - 幅（例：w-full, w-1/2）
  • h-{size} - 高さ（例：h-screen, h-64）

数値スケールシステム

Tailwind CSSは、一貫した数値スケールを使用しています。基本単位は0.25rem（4px）です。

クラス 値 ピクセル換算
p-00rem0px
p-10.25rem4px
p-20.5rem8px
p-41rem16px
p-82rem32px
p-164rem64px

2.2 テキストスタイリング

Tailwind CSSでは、テキストの見た目を制御するための豊富なユーティリティクラスが用意されています。

フォントサイズ

フォントサイズはtext-{size}クラスで指定します。

<p class="text-xs">極小テキスト (0.75rem / 12px)</p>
<p class="text-sm">小さいテキスト (0.875rem / 14px)</p>
<p class="text-base">通常テキスト (1rem / 16px)</p>
<p class="text-lg">大きいテキスト (1.125rem / 18px)</p>
<p class="text-xl">特大テキスト (1.25rem / 20px)</p>
<p class="text-2xl">2XLテキスト (1.5rem / 24px)</p>
<p class="text-4xl">4XLテキスト (2.25rem / 36px)</p>

フォントウェイト（太さ）

フォントの太さはfont-{weight}クラスで制御します。

<p class="font-thin">極細フォント (100)</p>
<p class="font-light">細いフォント (300)</p>
<p class="font-normal">通常フォント (400)</p>
<p class="font-medium">中太フォント (500)</p>
<p class="font-semibold">半太フォント (600)</p>
<p class="font-bold">太いフォント (700)</p>

テキストの配置

<p class="text-left">左寄せ</p>
<p class="text-center">中央寄せ</p>
<p class="text-right">右寄せ</p>
<p class="text-justify">両端揃え</p>

テキストの装飾

<p class="underline">下線付きテキスト</p>
<p class="line-through">取り消し線テキスト</p>
<p class="no-underline">下線なしテキスト（リンクなど）</p>
<p class="uppercase">大文字変換</p>
<p class="lowercase">小文字変換</p>
<p class="capitalize">各単語の先頭を大文字に</p>

2.3 カラーシステム

Tailwind CSSは、包括的で一貫性のあるカラーパレットを提供しています。各色は50から900までの9段階の濃淡で利用できます。

カラーパレットの構造

flowchart TD A["カラーファミリー"] --> B["Gray"] A --> C["Red"] A --> D["Blue"] A --> E["Green"] A --> F["その他多数"] B --> G["50 - 最も薄い"] B --> H["100, 200, 300..."] B --> I["500 - 標準"] B --> J["900 - 最も濃い"]

テキストカラーの適用

text-{color}-{shade}形式で色を指定します。

<p class="text-gray-500">グレーテキスト</p>
<p class="text-red-600">赤いテキスト</p>
<p class="text-blue-500">青いテキスト</p>
<p class="text-green-600">緑のテキスト</p>
<p class="text-yellow-500">黄色いテキスト</p>
<p class="text-purple-600">紫のテキスト</p>

背景色の適用

bg-{color}-{shade}形式で背景色を指定します。

<div class="bg-blue-100 p-4">薄い青の背景</div>
<div class="bg-blue-500 text-white p-4">標準の青背景</div>
<div class="bg-blue-900 text-white p-4">濃い青の背景</div>

2.4 スペーシング（Padding と Margin）

スペーシングはレイアウトの重要な要素です。Tailwind CSSでは、一貫したスケールでスペーシングを制御できます。

Padding（内側の余白）

flowchart LR A["Paddingクラス"] --> B["p-{size} - 全方向"] A --> C["px-{size} - 左右"] A --> D["py-{size} - 上下"] A --> E["pt-{size} - 上"] A --> F["pr-{size} - 右"] A --> G["pb-{size} - 下"] A --> H["pl-{size} - 左"] 
<!-- 全方向に余白 -->
<div class="p-4 bg-blue-100">全方向に1rem（16px）の内側余白</div>

<!-- 水平方向のみ -->
<div class="px-6 bg-green-100">左右に1.5rem（24px）の内側余白</div>

<!-- 垂直方向のみ -->
<div class="py-3 bg-yellow-100">上下に0.75rem（12px）の内側余白</div>

<!-- 個別指定 -->
<div class="pt-4 pr-6 pb-8 pl-2 bg-purple-100">
  各方向に異なる内側余白
</div>

Margin（外側の余白）

Marginの指定方法はPaddingと同じパターンですが、m-プレフィックスを使用します。

<div class="m-4 bg-blue-500">全方向に1rem（16px）の外側余白</div>
<div class="mx-auto bg-red-500">水平方向に自動マージン（中央寄せ）</div>
<div class="mt-8 bg-green-500">上に2rem（32px）の外側余白</div>

負のマージン

要素を重ねたり、特殊なレイアウトを作る際に便利です。

<div class="-mt-4 bg-blue-500">上に負のマージン</div>
<div class="-ml-2 bg-red-500">左に負のマージン</div>

2.5 サイズ指定（Width と Height）

幅の指定

<!-- 固定幅（rem単位） -->
<div class="w-64 bg-blue-200">幅256px（16rem）</div>

<!-- パーセント指定 -->
<div class="w-1/2 bg-green-200">幅50%</div>
<div class="w-1/3 bg-yellow-200">幅33.33%</div>
<div class="w-full bg-red-200">幅100%</div>

<!-- 特殊な値 -->
<div class="w-screen bg-purple-200">画面幅100vw</div>
<div class="w-auto bg-pink-200">自動幅</div>

高さの指定

<div class="h-32 bg-blue-200">高さ128px（8rem）</div>
<div class="h-screen bg-green-200">画面の高さ100vh</div>
<div class="h-full bg-yellow-200">親要素の高さ100%</div>

2.6 ボーダーとシャドウ

ボーダーの基本

<!-- ボーダーの太さ -->
<div class="border">1pxのボーダー（全方向）</div>
<div class="border-2">2pxのボーダー</div>
<div class="border-4">4pxのボーダー</div>

<!-- 特定の辺のみ -->
<div class="border-t-4">上のみ4pxボーダー</div>
<div class="border-r-2">右のみ2pxボーダー</div>

<!-- ボーダーカラー -->
<div class="border border-blue-500">青いボーダー</div>
<div class="border-2 border-red-600">赤い2pxボーダー</div>

角丸（Border Radius）

<div class="rounded">小さい角丸（4px）</div>
<div class="rounded-md">中程度の角丸（6px）</div>
<div class="rounded-lg">大きい角丸（8px）</div>
<div class="rounded-full">完全な円形（9999px）</div>

<!-- 個別の角 -->
<div class="rounded-t-lg">上の角のみ丸く</div>
<div class="rounded-r-lg">右の角のみ丸く</div>

シャドウ（影）

<div class="shadow-sm">小さい影</div>
<div class="shadow">標準の影</div>
<div class="shadow-md">中程度の影</div>
<div class="shadow-lg">大きい影</div>
<div class="shadow-xl">特大の影</div>
<div class="shadow-2xl">超特大の影</div>
<div class="shadow-none">影なし</div>
実習 2-1: ユーティリティクラスを組み合わせたカード作成

学んだユーティリティクラスを組み合わせて、シンプルなカードコンポーネントを作成します。

手順
  1. HTMLファイルを作成し、Tailwind CSS CDNを読み込む
  2. カードの外枠を作成（背景色、角丸、影）
  3. パディングとマージンでスペーシングを調整
  4. タイトルと本文にテキストスタイルを適用
実装例
<div class="max-w-sm mx-auto mt-8">
  <!-- カード -->
  <div class="bg-white rounded-lg shadow-lg p-6">
    <!-- カードヘッダー -->
    <h2 class="text-2xl font-bold text-gray-800 mb-4">
      カードタイトル
    </h2>

    <!-- カード本文 -->
    <p class="text-gray-600 mb-4">
      これはTailwind CSSのユーティリティクラスを使用して
      作成したシンプルなカードコンポーネントです。
    </p>

    <!-- ボタン -->
    <button class="bg-blue-500 text-white px-6 py-2 rounded hover:bg-blue-600">
      詳細を見る
    </button>
  </div>
</div>
使用したクラスの解説
  • max-w-sm - カードの最大幅を制限（24rem / 384px）
  • mx-auto - 水平方向に中央寄せ
  • mt-8 - 上マージン2rem（32px）
  • bg-white - 白い背景
  • rounded-lg - 大きめの角丸
  • shadow-lg - 大きい影
  • p-6 - 全方向に1.5rem（24px）のパディング
  • hover:bg-blue-600 - ホバー時の背景色（後の章で詳しく学習）
理解度確認クイズ
  1. text-2xlとtext-xl-2の違いは何ですか？

    解答：text-2xlが正しい形式で、2倍のXLサイズ（1.5rem / 24px）を意味します。text-xl-2は存在しない無効なクラス名です。

  2. p-4とm-4の違いを説明してください。

    解答：p-4はpadding（内側の余白）で、要素の内部にスペースを追加します。m-4はmargin（外側の余白）で、要素の外部にスペースを追加します。どちらも1rem（16px）の大きさです。

  3. bg-blue-500のクラス名において、"500"は何を表していますか？

    解答：色の濃淡レベルを表しています。50が最も薄く、900が最も濃い色で、500は標準的な濃さを表します。

  4. w-1/2というクラスは何を意味しますか？

    解答：親要素の幅の50%（1/2）を意味します。分数形式で幅を指定できるTailwindの便利な機能です。

  5. rounded-fullとrounded-lgの使い分けを説明してください。

    解答：rounded-fullは完全な円形または楕円形を作る際に使用し（border-radius: 9999px）、アバター画像などに適しています。rounded-lgは大きめの角丸（8px）で、カードやボタンなどに適しています。

2.7 まとめ

この章では、Tailwind CSSの基本的なユーティリティクラスを学びました。

  • Tailwind CSSは一貫した命名規則に従い、直感的にクラス名を推測できる
  • テキストのサイズ、太さ、色などを簡単に制御できる
  • カラーシステムは50～900の9段階で、一貫性のある配色が可能
  • スペーシングは0.25rem（4px）単位の一貫したスケールで管理される
  • ボーダー、角丸、シャドウなどの装飾も簡単に適用できる
  • これらのクラスを組み合わせることで、複雑なデザインを構築できる

次章では、これらのユーティリティクラスを使用して、実践的なレイアウトを構築する方法を学びます。
