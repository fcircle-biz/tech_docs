第5章: カスタマイズと設定

カスタマイズと設定

この章で学ぶこと

  • カスタムカラーパレットの定義方法を学ぶ
  • スペーシング、フォント、ブレークポイントのカスタマイズを理解する

5.1 tailwind.config.jsの基本構造

tailwind.config.jsは、Tailwind CSSのカスタマイズを行うための中心的な設定ファイルです。このファイルを通じて、プロジェクト固有のデザインシステムを構築できます。

flowchart TD A["tailwind.config.js"] --> B["content - スキャン対象"] A --> C["theme - テーマ設定"] A --> D["plugins - プラグイン"] C --> E["extend - 拡張"] C --> F["デフォルト値の上書き"]

基本構成

// tailwind.config.js
module.exports = {
  // スキャン対象のファイルパス
  content: [
    "./src/**/*.{html,js,jsx,ts,tsx}",
  ],

  // テーマのカスタマイズ
  theme: {
    // デフォルト値を拡張
    extend: {
      // ここにカスタム設定を追加
    },
  },

  // プラグイン
  plugins: [],
}

5.2 カスタムカラーパレットの定義

プロジェクトのブランドカラーや独自のカラーパレットを定義できます。

カラーの追加

// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        // ブランドカラーを追加
        'brand': {
          50: '#f0f9ff',
          100: '#e0f2fe',
          500: '#0ea5e9',  // 主要なブランドカラー
          600: '#0284c7',
          900: '#0c4a6e',
        },
        // 単一カラーの追加
        'primary': '#1e40af',
        'secondary': '#64748b',
        'accent': '#f59e0b',
      },
    },
  },
}

使用例

<!-- カスタムカラーの使用 -->
<div class="bg-brand-500 text-white p-4">
  ブランドカラーの背景
</div>

<button class="bg-primary hover:bg-blue-700 text-white px-4 py-2">
  プライマリーボタン
</button>

5.3 スペーシングのカスタマイズ

独自のスペーシングスケールを定義できます。

module.exports = {
  theme: {
    extend: {
      spacing: {
        '72': '18rem',    // 288px
        '84': '21rem',    // 336px
        '96': '24rem',    // 384px
        '128': '32rem',   // 512px
      },
    },
  },
}

使用例

<div class="mt-72 p-128">
  カスタムスペーシング
</div>

5.4 フォントファミリーのカスタマイズ

プロジェクトで使用するフォントを定義できます。

module.exports = {
  theme: {
    extend: {
      fontFamily: {
        'sans': ['Noto Sans JP', 'sans-serif'],
        'serif': ['Noto Serif JP', 'serif'],
        'mono': ['JetBrains Mono', 'monospace'],
        'display': ['Poppins', 'sans-serif'],
      },
    },
  },
}

使用例

<h1 class="font-display text-4xl">ディスプレイフォント</h1>
<p class="font-sans">サンセリフフォント</p>
<code class="font-mono">モノスペースフォント</code>

5.5 ブレークポイントのカスタマイズ

プロジェクトに最適なブレークポイントを設定できます。

module.exports = {
  theme: {
    screens: {
      'xs': '475px',      // 追加の小画面
      'sm': '640px',
      'md': '768px',
      'lg': '1024px',
      'xl': '1280px',
      '2xl': '1536px',
      '3xl': '1920px',    // 追加の超大画面
    },
  },
}
実習 5-1: カスタムデザインシステムの構築

プロジェクト固有のカラー、フォント、スペーシングを持つデザインシステムを構築します。

実装例
// tailwind.config.js
module.exports = {
  content: ["./src/**/*.{html,js}"],
  theme: {
    extend: {
      // ブランドカラー
      colors: {
        'corporate-blue': {
          50: '#e6f2ff',
          100: '#b3d9ff',
          500: '#0066cc',
          700: '#004d99',
          900: '#003366',
        },
        'corporate-orange': '#ff6600',
      },

      // カスタムフォント
      fontFamily: {
        'heading': ['Montserrat', 'sans-serif'],
        'body': ['Noto Sans JP', 'sans-serif'],
      },

      // カスタムスペーシング
      spacing: {
        '18': '4.5rem',
        '88': '22rem',
      },

      // カスタムブレークポイント
      screens: {
        'tablet': '640px',
        'laptop': '1024px',
        'desktop': '1280px',
      },
    },
  },
}

5.6 プラグインの活用

Tailwind CSSプラグインを使用して、機能を拡張できます。

公式プラグインの導入

# フォームプラグインのインストール
npm install -D @tailwindcss/forms

# タイポグラフィプラグインのインストール
npm install -D @tailwindcss/typography

設定ファイルへの追加

module.exports = {
  plugins: [
    require('@tailwindcss/forms'),
    require('@tailwindcss/typography'),
  ],
}
理解度確認クイズ
  1. theme.extend と theme の違いは何ですか？

    解答：theme.extendはデフォルト値を保持したまま新しい値を追加します。themeを直接設定すると、デフォルト値が完全に上書きされます。通常はextendの使用が推奨されます。

  2. カスタムカラーを定義する際、なぜ50-900の段階的な値を用意するのですか？

    解答：様々な濃淡を用意することで、UI要素に応じて適切な色を選択でき、一貫性のあるデザインを実現できます。hover状態や背景色など、多様な用途に対応できます。

5.7 まとめ

  • tailwind.config.jsでプロジェクト固有のデザインシステムを構築できる
  • theme.extendでデフォルト値を保持しながら拡張可能
  • カラー、フォント、スペーシング、ブレークポイントを自由にカスタマイズ
  • プラグインで機能を拡張し、生産性を向上できる
