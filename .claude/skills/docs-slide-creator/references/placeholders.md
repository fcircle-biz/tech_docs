# index.html プレースホルダーと slides 配列フォーマット

## 1. index.html プレースホルダー置換

`templates/v3/slide/index.html` には全角の `【】` で囲まれたプレースホルダーがある。以下を置換する。

| プレースホルダー | 置換内容 |
|----------------|---------|
| `【タイトル】` | 教材タイトル(例: Excel入門)。`<title>` タグ、ヘッダー見出し、パンくず末尾の3か所に出現する |
| `【カテゴリ】` | 親カテゴリ名(例: Microsoft Office)。ヘッダーのメタ情報に出現する |
| `【親カテゴリ】` | パンくずリスト用の親カテゴリ名(例: Microsoft Office) |
| アイコン | 適切なFont Awesomeアイコン(`references/icon-color-themes.md` 参照) |
| パンくずリストのパス | 相対パスを出力先の階層に合わせて調整する |

補足:
- `【タイトル】` は複数箇所(`<title>` / ヘッダー見出し / パンくず末尾)に現れるため、すべて置換する。
- ヘッダー構造そのもの(レイアウト・要素)は変更しない。

## 2. パンくずリストのパス調整

パンくずリストの親カテゴリへのリンク(`<a href="../../">【親カテゴリ】</a>` 等)は、出力先フォルダの階層の深さに応じて相対パスを調整する。出力先は `docs/slide/[カテゴリパス]/[教材名]/` であり、`[カテゴリパス]` の階層数に合わせる。

## 3. slide-content.js の slides 配列フォーマット

PDFファイル名を解析し、`slides` 配列を生成する。各要素は以下の形式:

```javascript
const slides = [
    { number: 1, title: '第1章タイトル', file: 'pdf/ファイル名1.pdf' },
    { number: 2, title: '第2章タイトル', file: 'pdf/ファイル名2.pdf' },
    // ...
];
```

各キー:
- `number`: スライド番号(1から開始)
- `title`: スライドのタイトル(下記抽出ルールで決定)
- `file`: PDFへの相対パス。`pdf/` ディレクトリ配下を指す

## 4. タイトル抽出ルール

- ファイル名から章番号とタイトルを抽出する。
- 例: `Excel基礎ガイド_第1章Excelの基本操作.pdf` → `title: 'Excelの基本操作'`
- ファイル名のパターンが不規則な場合は、内容から手動でタイトルを推定する。
- `number` はファイルの章番号順、または `work_pdf/` の並び順に基づいて1から連番で付与する。
