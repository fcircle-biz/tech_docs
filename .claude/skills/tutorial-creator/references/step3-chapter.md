# step3: 1ステップ分のHTML生成（生成フェーズ・サブエージェント用）

第1ステップHTMLを構造ベースとして、指定された1ステップ分のHTMLファイルを生成する。生成フェーズでメイン会話ループから `general-purpose` サブエージェントとして並列起動される（1メッセージにまとめて並列発行）。

## 重要な実行ルール（絶対遵守）

**以下をすべて完了するまで終了しないこと:**

1. README.md の読み込みと分析
2. 第1ステップHTMLファイルの読み込み
3. **指定されたステップのHTMLを生成（Write）**
4. 生成されたファイルの確認と報告

途中で「これから生成します」「開始しました」などと報告して終了することは禁止。実際にファイルが生成されるまで処理を継続すること。

## 役割

step2（準備フェーズ）で作成された土台（共通部品＋`sidebar-content.js`＋第1ステップHTML）を前提とし、**第1ステップHTMLをコピーして内容を書き換える**シンプルな作業を行う。

## 入力（呼び出し元から渡される）

- README.md のパス（リポジトリルート相対）
- 第1ステップHTML（`[技術名]-tutorial-01.html`）のパス ← 構造ベース
- 生成対象のステップ番号

## 実行手順

### 1. README分析

README.md を読み込み、指定されたステップ番号の情報（実装目標・内容）を抽出する。

### 2. 既存ファイル確認

技術フォルダに以下が存在することを確認する。

- `sidebar-content.js`
- `styles.css`
- `main.js`
- `drawing-tool.js`
- **`[技術名]-tutorial-01.html`（第1ステップHTML）** ← 必須

存在しない場合は、準備フェーズ（step2）が未完了である旨を報告して終了する。

### 3. 第1ステップHTMLの構造を確認

第1ステップHTMLファイルを読み込み、以下の構造を把握する。

- `<head>` セクション（Tailwind config、CDN読み込み順序）
- `<header>` セクション（ヘッダーのクラス名、構造）
- `<main>` セクション（メインコンテンツの構造、クラス名）
- パンくずリスト、ステップヘッダー、実装目標カードの構造
- `<footer>` セクションとスクリプト読み込み

### 参照ドキュメント（必ず参照）

- コンポーネント: `templates/v2/snippets/components.html`
- Mermaid記法ルール（CLAUDE.md方式）: `mermaid-rules.md`（詳細は `templates/v2/reference/mermaid-patterns.md`）
- カードコンポーネント一覧・テキスト視認性・検証チェックリスト: `components-and-colors.md`

### 4. 指定ステップのHTMLを生成【最重要】

#### 生成手順

1. 第1ステップHTMLファイルをReadツールで読み込み、その内容をベースとする。
2. README.md から該当ステップの情報を読み取る。
3. ステップ番号、タイトル、内容を該当ステップのものに書き換える。
4. **Writeツールでファイルを保存する。**

#### 書き換える箇所（7項目）

1. `<title>` タグ内のステップ番号とタイトル
2. ヘッダー内のステップ表示（あれば）
3. パンくずリストのステップ番号
4. ステップヘッダー（ステップ番号バッジ、`h1` タイトル、説明文）
5. 実装目標カードの内容
6. メインコンテンツのセクション（README.md の該当ステップの内容に基づく）
7. 前ステップ/次ステップナビゲーションのリンクとステップ番号、および完了チェックリストの内容

#### 書き換えてはいけない箇所（構造を維持）

- `<head>` セクションのCDN読み込み、Tailwind config
- `<header>` のクラス名と構造（`header-rich` 等）
- `<main>` のクラス名と構造
- `<footer>` の構造
- スクリプト読み込み部分

## 生成ルール

1. README.md から該当ステップの実装目標・内容を読み取り反映する。
2. 実装手順は番号付きで具体的に記述する。
3. コードブロックは実行可能なサンプル、日本語コメント必須。
4. Mermaid図表はデフォルトテーマを使用（darkテーマ禁止）。記法は `mermaid-rules.md`（CLAUDE.md方式）に従う。
5. 第1ステップHTMLの構造・スタイルを完全に継承すること。

## 重要事項

- `sidebar-content.js`, `styles.css`, `main.js`, `drawing-tool.js` は既存のものを使用する。
- これらの共通部品ファイルを上書きしないこと。
- **第1ステップHTMLの構造から逸脱しないこと。**

## 実践チュートリアル重点事項

- ステップバイステップの実装手順
- 実行可能なコード例
- 各ステップでの成果物確認
- トラブルシューティング情報
- 完了チェックリスト

## 出力

生成完了後、生成したファイルのパス（リポジトリルート相対）を報告する。

**禁止事項:**

- 「これから生成します」と報告して終了すること
- ファイルを生成せずに終了すること
