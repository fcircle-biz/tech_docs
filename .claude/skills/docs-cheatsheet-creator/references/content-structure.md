# コンテンツ構成（content-structure）

チートシートは1ページ完結型のクイックリファレンス。詳細な解説は学習教材に委譲し、実務で即使えるコピー可能な短いコード例に絞る。

## 必須セクション

1. **イントロセクション**
   - 技術の概要説明（2〜3文）
   - 主要機能のタグバッジ（`{{INTRO_*}}` プレースホルダー対応）

2. **基本操作カード**（2カラムグリッド）
   - 最も頻繁に使う操作
   - 各カードにコードブロック + 簡潔な説明

3. **よく使うパターン**
   - 実務で頻出するコード例
   - コピーしてすぐ使える形式

## オプションセクション

4. **実務Tips**（ダーク背景セクション）
   - ベストプラクティス
   - 注意点・落とし穴

5. **処理フロー/実行順序**
   - ステップ表示
   - 視覚的なフロー図（Mermaidを使う場合はCLAUDE.md方式に統一。詳細は `templates/v3/reference/mermaid-patterns.md`、要点は `html-rules.md` の共通HTMLルール参照）

6. **危険な操作**（赤系カード）
   - DELETE, DROP等の破壊的操作
   - 注意喚起アイコン付き（カードカラーは `html-rules.md` の「危険/削除」行）

## コンテンツ作成ルール

- **簡潔さ重視**: 詳細な説明は学習教材に任せる。
- **コピー可能**: すべてのコードにコピーボタン（`copyCode` 構造は `html-rules.md`）。
- **視認性**: 色分けで重要度・カテゴリを区別（半透明背景は禁止 → 共通HTMLルール参照）。
- **実用性**: 実務で即使えるコード例。1ページに収まる分量を厳守する。
