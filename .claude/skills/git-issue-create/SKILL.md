# Git Issue Create

技術ドキュメント（教材）に関する問題・要望を、すぐ動ける日本語のGitHub Issueへ整形して起票するスキル。本スキルは**起票専用**。

## 使用方法

```
/git-issue-create [報告・要望の内容]
```

例:
- `/git-issue-create Dockerガイドの第3章でMermaid図が崩れている`
- `/git-issue-create Reactの新しい学習ガイドを追加したい`

## 前提

- リモート: `origin`（`git@github.com:fcircle-biz/tech_docs.git`）
- GitHub操作には `gh` CLI を用いる（`gh issue create`）。利用不可ならIssue本文と手動起票手順を提示する。
- このリポジトリには固定のIssueテンプレートやラベル定義（labels.yml）は無い。ラベルは既存のものがあれば任意で付け、無ければ付けなくてよい。

## 実行手順

1. ユーザーの報告から、問題/要望の種類・背景・影響範囲・期待結果・（バグなら）実際の結果を抽出する。
2. 不足している情報（再現手順、対象ページのURL/パス、ブラウザ、スクリーンショット等）を確認する。
3. 「Issue本文テンプレート」に沿って、Issueタイトル・本文・受け入れ基準を日本語で作成する。
4. 既存ラベルがあり該当するものがあれば任意で付与する（無ければ省略）。
5. ユーザーが起票まで依頼している場合は、内容を確認してから `gh issue create` で起票する。下書きのみ依頼なら本文を提示して終了する。
6. 完了時にIssue URL（または下書き本文）、付与したラベル、未確定事項を報告する。

## Issue種別（このリポジトリ向け）

教材リポジトリでよくある種別。タイトル冒頭やラベルの目安に使う。

- **不具合**: 表示崩れ（Mermaid/レイアウト/ダークモード）、リンク切れ、コード例の誤り、内容の誤記。
- **改善要望**: 既存教材の加筆・修正、説明の改善、図解追加。
- **新規教材リクエスト**: 新しい学習ガイド/チュートリアル/練習問題/チートシートの追加。
- **保守・運用**: テンプレート/スキル/CI/READMEなどの整備。

## Issue本文テンプレート

```markdown
## 概要
[問題または要望の要約]

## 背景・目的
[なぜ対応が必要か]

## 対象
- 教材種別: 学習ガイド / チュートリアル / 練習問題 / 実践課題 / チートシート / スライド / その他
- 対象パス・URL: [docs/... または GitHub Pages URL]

## 再現手順（不具合の場合）
1. [手順]
2. [手順]

## 期待される結果 / 実際の結果
- 期待: [...]
- 実際: [...]

## 受け入れ基準
- [ ] [検証可能な基準]
- [ ] [検証可能な基準]

## 補足
[関連リンク、スクリーンショット、ログなど]
```

- 種別に応じて不要な節は省略してよい（例: 新規教材リクエストでは「再現手順」を省く）。

## 日本語スタイル

- すべての文面を自然で専門的な日本語で書く。丁寧語を基本に、簡潔で読みやすく。
- 技術用語は一般的な日本語表記を使う。無理に訳すと不自然な語は英語のまま使う。
- 見出しと箇条書きで論理的に整理し、対応者が次の行動を取れる具体性を保つ。

## 品質基準

- タイトルがIssue内容を簡潔に表している。
- 本文がアクション可能で、対応に十分な詳細（対象パス/URLを含む）がある。
- 受け入れ基準が検証可能である。
- 日本語が明確・自然・技術的に正確である。
- 完了報告にIssue URL（または下書き本文）が含まれている。
