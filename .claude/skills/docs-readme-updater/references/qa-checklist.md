# 品質保証チェックリスト

ルート `README.md` を再生成した後、書き込み前後で以下を必ず検証する。1つでも満たさない場合は修正してから完了とする。

## 存在確認(リンク切れ防止)

- [ ] README に列挙したフォルダ・ファイルが、すべてディスク上に実在する。
- [ ] 相対パス(`docs/...`)のリンク先が実在する。
- [ ] 絶対 URL(スライド・チートシート)のパス(`slide/.../index.html`, `cheatsheet/.../index.html`)が実構造のファイルに対応している。
- [ ] 実構造に存在するのに README 未掲載の教材がない(追加漏れなし)。
- [ ] 実構造に存在しないのに README に残っている項目がない(削除漏れなし)。

## セクション順序

- [ ] セクションが固定9順序になっている: 目次 → 概要 → 学習パス → 学習ガイドライン → スライド資料 → チュートリアル → プラクティス → 実践課題 → チートシート。
- [ ] `## 📋 目次` のアンカーリンクが、後続8セクションを正しく指している。

## 技術分野順

- [ ] 各セクション内のサブカテゴリ(`###`)の並びが `tech-knowledge-map.md` の分類順序と一致している。
- [ ] 教材が存在する分野のみ `###` を作っている(空の分野見出しがない)。

## 命名・除外規則

- [ ] 見出し・表示名から「エコシステム」が除去されている(例: 「Java」)。
- [ ] 過去バージョンフォルダ(`v1/` `v2/` 等)が記載されていない。
- [ ] 命名規則・項目フォーマットがセクション内で一貫している。

## リンク形式

- [ ] 学習パス / 学習ガイドライン / チュートリアル / プラクティス / 実践課題 = 相対パス(`docs/...`)。
- [ ] スライド資料 / チートシート = 絶対 GitHub Pages URL(`https://fcircle-biz.github.io/tech_docs/...`)。
- [ ] 環境依存の絶対 FS パス(`/Users/...`, `/home/<user>/...`)が混入していない。

## Markdown 妥当性

- [ ] Markdown 構文が正しい(見出しレベル・リンク記法・テーブル・字下げ)。
- [ ] 文字化け・全角半角の不整合がない。
- [ ] (Mermaid 図を含める場合)CLAUDE.md の Mermaid 記法に準拠している(HTMLエンティティ不使用 / 半角括弧は全角化かダブルクォート / `&` は全角 `＆` / dark テーマ不使用)。詳細は `templates/v3/reference/mermaid-patterns.md` 参照。

## 対象限定

- [ ] 変更したファイルがルート `README.md`(= tech_docs/README.md)のみである。他の README.md を変更していない。
