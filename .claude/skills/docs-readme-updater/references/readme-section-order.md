# README セクション順序と技術分野順

ルート `README.md`(= tech_docs/README.md)を再生成する際の、固定セクション順序・各セクションの内訳・技術分野の並び順を定める。

## 固定セクション順序(必須・この順番を厳守)

以下の9セクションを、必ずこの順番で配置する。

1. `## 📋 目次`
2. `## 📖 概要`
3. `## 🗺️ 学習パス`
4. `## 📖 学習ガイドライン`
5. `## 📖 スライド資料`
6. `## 📖 チュートリアル`
7. `## 📖 プラクティス`
8. `## 📖 実践課題`
9. `## 📖 チートシート`

> 末尾に補助セクション(例: `## 🔗 リンク`)が既存READMEにある場合は、9セクションの後ろに維持してよい。ただし上記9セクションの順序自体は変更しない。

## 各セクションの対応ディレクトリと内訳

| セクション | 真実源ディレクトリ | 単位 |
|------------|--------------------|------|
| 学習パス | `docs/learning-paths/` | 各 `.md` ファイル(学習ロードマップ) |
| 学習ガイドライン | `docs/guide/` | 各技術フォルダの `README.md` |
| スライド資料 | `docs/slide/` | 各教材の `index.html` |
| チュートリアル | `docs/tutorial/` | 各チュートリアルフォルダの `README.md` |
| プラクティス | `docs/practice/` | 各練習問題フォルダ(`README.md` またはトップ) |
| 実践課題 | `docs/assignment/` | 各課題フォルダ(`README.md` またはトップ) |
| チートシート | `docs/cheatsheet/` | 各チートシートの `index.html` |

### 各セクション作成時の要件

- 各セクションには適切な技術分野別のサブカテゴリ(`### 見出し`)を設ける。
- 各項目には簡潔で分かりやすい説明を付ける。
- 適切な Markdown 形式を使用する。
- 実際に存在するフォルダ・ファイルのみを記載する(後述「実在のみ」)。
- 既存 README の説明文は参考程度に留め、実構造に基づいて再生成する。

## 目次(セクション1)の作り方

`## 📋 目次` には、後続8セクションへのアンカーリンクを並べる。アンカーは GitHub の見出しアンカー規則(絵文字・空白の扱い)に従う。既存 README の目次アンカー形式を踏襲する。例:

```markdown
## 📋 目次

-   [📖 概要](#-概要)
-   [🗺️ 学習パス](#%EF%B8%8F-学習パス)
-   [📖 学習ガイドライン](#-学習ガイドライン)
-   [📖 スライド資料](#-スライド資料)
-   [📖 チュートリアル](#-チュートリアル)
-   [📖 プラクティス](#-プラクティス)
-   [📖 実践課題](#-実践課題)
-   [📖 チートシート](#-チートシート)
```

## 技術分野(サブカテゴリ `###`)の表示順序

各セクション内の技術分野サブカテゴリは、リポジトリ直下の **`tech-knowledge-map.md` の分類順序に従う**。

- `tech-knowledge-map.md` を参照し、その表の分類順(プログラミング言語 → Webテクノロジ → … → 資格・認定試験 等)に沿って `###` 見出しを並べる。
- 値(分類名一覧)を本ファイルに丸写ししない。必ず `tech-knowledge-map.md` を参照すること。
- サブカテゴリ見出しは「エコシステム」を除いたシンプルな技術名にする(`references/naming-and-exclusion-rules.md` 参照)。
- 実際に教材が存在する分野のみ `###` を作る(教材ゼロの分野見出しは作らない)。

### 補足: フォルダ命名と表示名の対応(代表例の抜粋)

`docs/.../<ecosystem>/` のフォルダ名と、README に出す `###` 表示名の対応は以下のように「エコシステム」相当を外す。正式な分類名・順序は `tech-knowledge-map.md` を真実源とする。

| フォルダ(ディスク上) | README 表示名(例) |
|------------------------|---------------------|
| `programming-languages/python-ecosystem/` | プログラミング言語 > Python |
| `programming-languages/java-ecosystem/` | プログラミング言語 > Java |
| `programming-languages/javascript-ecosystem/` | プログラミング言語 > JavaScript/Node.js |
| `web-technologies/` | Webテクノロジ |
| `development-processes/` | 開発手法・プロセス |
| `design-modeling/` | 設計手法・モデリング |
| `cloud-infrastructure/` | クラウド/インフラ |
| `data-ai-category/` | データ/AI |
| `business-saas/` | SaaS/PaaS/アプリケーション |
| `certification/` | 資格・認定試験 |

> 上表は代表値の抜粋(参照の便宜)。網羅的な分類名・順序は `tech-knowledge-map.md` を参照すること。
