# step1: README.md（チュートリアルカリキュラム）作成

実践チュートリアル設計の専門家として、specs配下のアプリ仕様を構造化された実践的なチュートリアルカリキュラムに変換し、`docs/tutorial/[分類パス]/[アプリ]/README.md` を作成する。

## 入力

```
[アプリタイプ] [環境] [DB構成]
```

- **アプリタイプ**: specs配下のフォルダ名（例: `todo-app-v1`, `user-management-v2`）。技術スタックはこのフォルダ名に一本化する。
- **環境**: 開発環境（例: `windows環境`, `mac環境`, `linux環境`）
- **DB構成**: DB環境（例: `postgresqlをdockerで構築`, `sqliteを使用`）

入力ファイル: `specs/[アプリタイプ]/README.md`, `tech-knowledge-map.md`

## 実行手順

### 1. 既存ファイル確認とバージョン管理

- 作成対象のフォルダパス（例: `docs/tutorial/web-technologies/nextjs-fastapi/todo-app/`）に既にファイルが存在するか確認する。
- **既存ファイルが存在する場合**:
  - 既存のバージョンフォルダ（`v1/`, `v2/`, `v3/` など）を確認する。
  - 最新のバージョン番号を特定し、次のバージョン番号のフォルダを作成する。
  - 親ディレクトリの既存ファイルを新しいバージョンフォルダに移動する。
- **既存ファイルが存在しない場合**:
  - そのまま新しいファイルを作成する。

### 2. 仕様分析

`specs/[アプリタイプ]/README.md` を注意深く確認し、以下を理解する。

- 技術要件とアーキテクチャパターン
- 実装アプローチと機能要件
- データベース設計とAPI設計

### 3. 分類パスの決定

**必須**: 新しいチュートリアル作成前に、必ず `tech-knowledge-map.md` を参照して、対象技術がどの分類に属するかを確認する。

**分類体系**: `tech-knowledge-map.md` に定義された9つの主要分類のいずれかに配置する（値は `tech-knowledge-map.md` を正とする）。

1. プログラミング言語 (`programming-languages/[エコシステム]/[技術]/`)
2. Web技術 (`web-technologies/[技術]/`)
3. 開発手法・プロセス (`development-processes/[技術]/`)
4. 設計手法・モデリング (`design-modeling/[カテゴリ]/[技術]/`)
5. クラウド／インフラ (`cloud-infrastructure/[技術]/`)
6. データ／AI (`data-ai-category/[カテゴリ]/[技術]/`)
7. 業務SaaS／プラットフォーム (`business-saas/[技術]/`)
8. 資格・認定試験 (`certification/[試験名]/`)
9. 業務知識・スキル (`business-domain-knowledge/[分野]/`)

### 4. README.mdの作成

以下のテンプレート構造を使用する。

```markdown
# [アプリ名] チュートリアル

このチュートリアルでは、[技術スタック]を使って[アプリ名]を構築しながら、実践的なスキルを身につけます。

## 前提条件
### 必要な環境
- [環境要件1]
- [環境要件2]

### 前提知識
- **必須**: [必要な前提知識]
- **推奨**: [推奨される前提知識]

## 学習コンテンツ

### [1. ステップ1タイトル](GitHub Pages URL)
[ステップの概要説明]
- 実装内容: [実装する機能]

### [2. ステップ2タイトル](GitHub Pages URL)
...

## 学習の進め方
1. **環境構築**: まず開発環境をセットアップします
2. **ステップ順に実装**: 各ステップのガイドに沿って実装します
3. **動作確認**: 各ステップの完了条件を満たしているか確認します

## 推奨所要時間

| ステップ | 内容 | 目安時間 |
|----------|------|----------|
| Step 1 | [内容] | X時間 |
| Step 2 | [内容] | X時間 |
| ... | ... | ... |
| **合計** | | **約XX時間** |

## 関連リソース
- [関連する学習ガイドへのリンク]
- [チートシートへのリンク]

## 学習目標
このチュートリアルを完了すると、以下のスキルを身につけることができます：
- [具体的なスキル1]
- [具体的なスキル2]

## 次のステップ
このチュートリアル完了後は、以下の学習に進むことをお勧めします：
- [関連する上級ガイドラインへのリンク]
```

### 5. ファイル命名規則

- **ディレクトリ**: `docs/tutorial/[tech-knowledge-map.mdに基づく分類パス]/[アプリ名]/`
- **ファイル名**: `[技術名]-tutorial-[ステップ番号:2桁].html`
  - **重要**: ステップ番号は必ず2桁ゼロパディング形式を使用（例: `01`, `02`, `03`）。
  - 例: `nextjs-fastapi-tutorial-01.html`, `nextjs-fastapi-tutorial-02.html`
- **GitHub Pages URL**: `https://fcircle-biz.github.io/tech_docs/tutorial/[分類パス]/[アプリ名]/[ファイル名].html`

## 完了後

README.mdを生成したら、本体（メイン会話ループ）は続けて準備フェーズの次工程（`step2-scaffold.md`）へ進む。生成したREADME.mdのフルパス（リポジトリルート相対）を控えておき、第1ステップHTML・共通部品生成、および生成フェーズの各サブエージェントへ渡す。
