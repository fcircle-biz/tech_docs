# Mermaid記法ルール（CLAUDE.md方式に統一）

Mermaid図を書く際は、必ず以下のルールに従う。旧step定義やリポジトリの参照ドキュメント（`templates/v3/reference/mermaid-patterns.md` の「特殊文字のエスケープ」節を含む）にHTMLエンティティ方式（`&#124;` `&#40;` `&amp;` 等）が残っていても、**CLAUDE.md方式を優先**する。詳細パターンは `templates/v3/reference/mermaid-patterns.md` を参照。

## ルール

1. **HTMLエンティティを使わない**: `&#40;` `&#41;` `&#38;` `&#124;` などのHTMLエンティティはMermaidで正しくパースされない。使用しないこと。
2. **括弧の扱い**:
   - 半角括弧 `()` はMermaidで特殊文字として認識されるため、**全角括弧 `（）` を使う**。
   - または、ノードテキスト全体をダブルクォートで囲む: `A["テキスト（括弧入り）"]`
3. **アンパサンド**: `&` ではなく**全角 `＆`** を使う。
4. **改行**: ノード内で `<br/>` を使う場合は、ノードテキストをダブルクォートで囲む: `A["1行目<br/>2行目"]`
5. **特殊文字を含むテキストはダブルクォートで囲む**:
   - スラッシュ（/）を含むパス: `node["/usr/share/nginx/html"]`
   - チルダ（~）を含むパス: `node["~/my-folder"]`
   - 特殊文字なしの場合: `node[通常のラベル]` ← クォート不要
6. **darkテーマ禁止**: Mermaidはデフォルトテーマを使用する。

## 正しい例

```
flowchart TD
    A["第1章: 概要（現在）"] --> B{OS}
    B -->|Windows| C["Windows環境構築"]
```

```
flowchart TD
    A["OR演算子を含む判定"] --> B[結果]
    C["AND条件 ＆ 追加条件"] --> D[処理]
```

## 誤った例（Syntax errorの原因）

```
flowchart TD
    A[第1章: 概要<br/>&#40;現在&#41;] --> B{OS}
    B -->|Windows| C[Windows環境<br/>構築]
```

```
flowchart TD
    A{OR演算 ||} --> B
    C{AND演算 &&} --> D
```

## ベストプラクティス

- 特殊文字を含むテキストは必ず `""` で囲む。
- プログラミング記号は日本語表記も検討する（例: `||` → `OR演算子`、`&&` → `AND演算子`）。
