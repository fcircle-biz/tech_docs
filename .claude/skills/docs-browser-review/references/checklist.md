# 描画QA 詳細チェックリスト

docs-browser-review が実機確認する観点の詳細。CLAUDE.md と `templates/v2/` のテンプレート標準が真実源。

## Chrome MCP 操作の要点

- セッション開始時に `tabs_context_mcp` を1回呼ぶ。タブが無い場合のみ `tabs_create_mcp` で生成する。
- ページ遷移は `navigate`。描画確認は `read_page` とスクリーンショット。
- コンソールは `read_console_messages` を必ず `pattern` 付きで呼ぶ（ノイズ回避。例: `pattern: "error|Error|Mermaid"`）。
- レスポンシブ確認はウィンドウ幅を変更（`resize_window`）して PC（>1024px）/タブレット（768〜1024px）/モバイル（<768px）を見る。
- alert/confirm を誘発する操作は避ける（拡張が応答不能になるため）。

## 観点別チェック

### 1. Mermaid 描画
- [ ] すべての Mermaid 図が図として描画されている（コードのまま残っていない）。
- [ ] 画面に `Syntax error in text` が出ていない。
- [ ] 括弧・`&`・`<br/>` を含むノードが崩れず表示される（CLAUDE.md の Mermaid 記法に準拠）。
- [ ] ダークテーマ指定ではなくデフォルトテーマで描画されている。

### 2. ダークモード
- [ ] ヘッダー右側にダークモード切替ボタンが1つ表示される（HTMLに二重で書かれていない）。
- [ ] 切替でページ全体の配色が変わる。
- [ ] ダーク時もカード内テキスト・コードが判読できる（コントラスト確保）。

### 3. コードブロック
- [ ] highlight.js のシンタックスハイライトが効いている。
- [ ] コードコピーが動作する。
- [ ] CDN デフォルト非対応言語（SAS/Kotlin/Scala 等）も追加読込でハイライトされている。

### 4. サイドバー / ナビゲーション
- [ ] 章/ステップ/回の一覧が全件表示される（sidebar-content.js の定義と一致）。
- [ ] 各リンクが該当ページへ遷移する（相対リンク確認時は静的サーバ経由）。
- [ ] 進捗表示・現在地ハイライトが機能する。
- [ ] 前後ナビ（前の章/次の章）のリンクが正しい。

### 5. レスポンシブ
- [ ] PC/タブレット/モバイル幅で横スクロール・はみ出し・要素重なりが無い。
- [ ] サイドバーがモバイルで適切に開閉する。

### 6. テキスト視認性
- [ ] 半透明背景（`bg-white/70` 等）による判読不能が無い。
- [ ] カード内の内側ボックスが同系色 -100 背景＋-900 テキストで読める。

### 7. 画像 / 図
- [ ] 挿入図（illustration の figure）が表示され、リンク切れが無い。
- [ ] figcaption（図番号・説明）が表示される。

### 8. コンソール / CDN
- [ ] コンソールに JS エラーが無い。
- [ ] Tailwind / highlight.js / Mermaid / Font Awesome / Noto Sans JP の CDN が読み込まれている。

## 重要度の目安

- **高**: 学習を妨げる描画崩れ（Mermaid未描画/Syntax error、ダークモードで判読不能、サイドバーリンク切れ、レイアウト大崩れ、コンソールの致命的エラー）。
- **中**: UX/見た目の問題（軽微な崩れ、ハイライト未適用、コピー不動作、レスポンシブの軽微な崩れ）。
- **低**: 軽微・将来検討（微妙な余白、任意の改善）。
