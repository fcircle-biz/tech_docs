# 命名・除外規則

ルート `README.md` を再生成する際の、表示名の命名規則と、記載から除外する対象を定める。

## 1. 「エコシステム」の除去

セクション名・サブカテゴリ見出しから「エコシステム」という文言を除去し、シンプルな技術名のみを使用する。

- 例: 「Javaエコシステム」→「Java」
- 例: 「Pythonエコシステム」→「Python」
- ディスク上のフォルダ名(`python-ecosystem/` 等)はそのままだが、README に出す**見出し・表示名**からは「エコシステム」を外す。

## 2. 過去バージョンフォルダ(v1/ v2/ 等)の除外

過去バージョンのバックアップフォルダは README に記載しない。走査(Glob/一覧化)からも除外する。

- 例: `docs/guide/programming-languages/java-ecosystem/java/v1/` は記載しない。
- `v1/` `v2/` `v3/` … のような版数バックアップフォルダは一律無視する。
- 真実源は各教材フォルダ直下の最新版(版数フォルダの外にあるアクティブな内容)。

## 3. 実在するパスのみ列挙(リンク切れ禁止)

- README に列挙するフォルダ・ファイルは、すべてディスク上に実在することが前提。
- 存在しないパスへのリンクを書かない(リンク切れ厳禁)。
- 既存 README に載っていても、実構造に存在しないものは削除する(実構造が真実源)。
- 逆に、実構造に存在するが既存 README に未掲載のものは追加する。

## 4. 説明文の方針

- 各項目には簡潔で分かりやすい日本語の説明を付ける。
- 既存 README の説明文があれば参考にしつつ、実態に合わせて調整する。
- 命名規則と表記(項目フォーマット・記号・字下げ)はセクション内で一貫させる。

## 5. 対象ファイルの限定(再掲・重要)

- 書き込み対象はリポジトリ直下の `README.md`(= tech_docs/README.md)のみ。
- 各教材フォルダ内の `README.md` その他は**変更しない**。

## 6. 環境依存パスの禁止(バグ是正)

- 説明・コメント・リンクのいずれにも、`/Users/...` や `/home/<user>/...` 等の環境依存の絶対 FS パスを書かない。
- パスはリポジトリルート相対(`tech_docs/README.md`, `docs/...`)、もしくは公開 URL(`https://fcircle-biz.github.io/tech_docs/...`)で表現する。
