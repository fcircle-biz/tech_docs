# 生成フェーズ: 1回分の練習問題HTML生成（並列サブエージェント用）

旧 practice-creator-step3 の手順を移植。指定された1回分の練習問題HTMLを生成する。**このファイルは生成フェーズで並列起動される general-purpose サブエージェントが読むメイン手順書。**

第1回HTML（土台）をコピーして内容を書き換える、というシンプルな作業を行う。共通ルールは `shared-rules.md`、問題スニペットは `question-components.md` を必ず併読する。

---

## 重要な実行ルール（絶対遵守）

以下をすべて完了するまで終了してはいけない。

1. README.md の読み込みと分析
2. 第1回HTMLファイルの読み込み
3. **指定された回の練習問題HTMLを生成**
4. 生成されたファイルの確認と報告

途中で「これから生成します」「開始しました」等と報告して終了することは禁止。実際にファイルが生成されるまで処理を継続する（`shared-rules.md` のエージェント実行ルール参照）。

---

## 入力（呼び出し元から渡される）

- README.md のパス
- 第1回HTML（`[技術名]-practice-01.html`）のパス
- 生成する回番号

---

## 実行手順

### 1. README分析

README.md を読み込み、指定された回番号の情報を抽出する。

- 回のタイトル
- 難易度
- 問題形式
- 内容の概要

### 2. 既存ファイル確認

練習問題フォルダに以下が存在することを確認する。

- `sidebar-content.js`
- `styles.css`
- `main.js`
- `drawing-tool.js`
- `[技術名]-practice-01.html`（第1回HTML） ← 必須・ベーステンプレート

存在しない場合は、先に準備フェーズ（step2-foundation）が必要な旨を報告して終了する。

### 3. 第1回HTMLの構造を確認

第1回HTMLファイルを読み込み、以下の構造を把握する。

- `<head>` セクション（Tailwind config、CDN読み込み順序）
- `<header>` セクション（ヘッダーのクラス名・構造）
- `<main>` セクション（問題カードの構造・クラス名）
- `<footer>` セクションとスクリプト読み込み

### 4. 指定回のHTMLを生成【最重要ステップ】

#### 生成手順

1. 第1回HTMLファイルを Read で読み込み、その内容をベースとする。
2. README.md から該当回の情報を読み取る。
3. 回番号・タイトル・難易度・内容を該当回のものに書き換える。
4. **Write でファイルを保存する**（ファイル名は `[技術名]-practice-[NN].html`）。

#### 書き換える箇所

1. `<title>` タグ内の回数とタイトル
2. ヘッダー内の回数表示
3. パンくずリストの回数
4. 回ヘッダー（回数バッジ、h1タイトル、難易度表示）
5. 問題セクション（README.md の該当回の内容に基づく問題を作成）
6. 回まとめセクション（学習ポイント）
7. 前回/次回ナビゲーションのリンクと回数

#### 書き換えてはいけない箇所（構造を維持）

- `<head>` セクションのCDN読み込み、Tailwind config
- `<header>` のクラス名と構造
- `<main>` のクラス名と構造
- `<footer>` の構造
- スクリプト読み込み部分（順序: styles.css → sidebar-content.js → main.js → drawing-tool.js）

#### ヘッダー構造の禁止事項

⚠️ 以下は絶対に行わない（詳細は `shared-rules.md`）。

1. **ダークモードボタンをHTMLに追加しない**（main.js が動的生成）。
2. **ヘッダー右側のボタンエリア構造を変更しない。**

### 5. 問題形式のコンポーネント（クリック展開式）

回答は `<details>` 展開式のみ。**textarea・「実行して確認」ボタンは使用禁止。** コーディング問題・選択問題・穴埋め・記述のスニペットは `question-components.md` に集約しているので、必ずそれを使用する。

### 6. 難易度別の問題設計

README で指定された該当回の難易度（★1〜★5）に応じて問題を設計する。

| 難易度 | 問題の特徴 |
|--------|-----------|
| ★☆☆☆☆ | 基本用語の確認、単純な選択問題 |
| ★★☆☆☆ | 基本的な構文の理解、簡単な穴埋め |
| ★★★☆☆ | 複数の概念を組み合わせた問題 |
| ★★★★☆ | 実践的なコーディング問題 |
| ★★★★★ | 応用・発展問題、設計を考える問題 |

### 7. 解説の書き方

- **正解だけでなく「なぜそうなるか」を説明する。**
- **よくある間違いとその理由も記載する。**
- 関連する概念へのリンクを含める。

### 8. 完了報告

HTMLファイル生成が完了したら、生成されたファイルパスを報告する。

```
## 生成完了

以下のファイルを生成しました：
- [フォルダパス]/[技術名]-practice-[NN].html
```

---

## Mermaid図作成時のルール

CLAUDE.md方式に統一する（HTMLエンティティ不可。`&#124;` 等は使わない）。詳細は `shared-rules.md`「Mermaid記法」および `templates/v3/reference/mermaid-patterns.md` を参照。

---

## 重要事項（厳守）

- sidebar-content.js, styles.css, main.js, drawing-tool.js は既存のものを使用する。
- **これらのファイルを上書きしない。**
- **第1回HTMLの構造から逸脱しない。**

## 禁止事項

- 「これから生成します」と報告して終了すること。
- ファイルを生成せずに終了すること。
