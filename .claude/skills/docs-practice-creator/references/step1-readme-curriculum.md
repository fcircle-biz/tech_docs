# 準備フェーズ1: README作成（カリキュラム設計）

旧 practice-creator-step1 の手順を移植。練習問題カリキュラムを README.md として作成する。技術学習用の練習問題設計の専門家として、効果的な問題構成と段階的な難易度設定を行う。

このフェーズはメイン会話ループが直接（逐次）実行する。共通ルールは `shared-rules.md`、パス・命名・分類・バージョン管理は `templates-and-paths.md` を参照。

---

## 1. 既存ファイル確認とバージョン管理

作成対象フォルダ（例: `docs/practice/programming-languages/python-ecosystem/python-basics/`）に既存ファイルがあるか確認し、あれば `templates-and-paths.md`「バージョン管理手順」に従って退避してから新規作成する。

---

## 2. 分類パスの決定

**必須**: 必ず `tech-knowledge-map.md` を参照して、対象技術がどの9分類に属するか確認する（分類一覧は `templates-and-paths.md` 参照）。決定した分類パスが README 内のリンク・URL・出力先ディレクトリに反映される。

---

## 3. 練習問題カリキュラムの設計

以下を考慮して設計する。

- **対象レベル**: 初心者・中級者・上級者
- **問題形式**: 選択問題、穴埋め問題、コーディング問題、記述問題
- **回数構成**: 推奨 6〜15回（各回15〜30分を想定）
- **難易度設計（★カーブ）**: 前半は基礎確認、後半は応用・実践

### 難易度カーブの目安

| 区分 | 回 | 難易度 | 主な問題形式 | 想定時間 |
|------|----|--------|--------------|----------|
| 基礎編 | 前半（第1回〜中盤） | ★☆☆☆☆ 〜 ★★☆☆☆ | 選択問題・穴埋め問題 | 約15〜20分 |
| 中級 | 中盤 | ★★★☆☆ | 概念を組み合わせた問題 | 約25分 |
| 応用編 | 後半（中盤〜最終回） | ★★★★☆ 〜 ★★★★★ | コーディング問題・記述問題 | 約30〜45分 |

難易度は前半から後半へ単調に上げ、急な飛びがないよう滑らかなカーブにする。
