# Todoアプリ 仕様書

この資料は、Todoアプリのチュートリアル作成時の参考として使用します。
特定の言語・フレームワークに依存しない基本設計です。

## 概要

シンプルなTodo管理アプリケーションです。
タスクの作成・表示・更新・削除（CRUD操作）の基本を学習できます。

## 主な機能

### タスク管理機能
- タスクの作成
- タスク一覧の表示
- タスクの編集
- タスクの削除
- タスクの完了/未完了の切り替え

### フィルタリング機能
- 全てのタスクを表示
- 未完了のタスクのみ表示
- 完了済みのタスクのみ表示

## 画面一覧

| 画面名 | 説明 |
|--------|------|
| タスク一覧画面 | メイン画面。タスクの一覧表示、新規作成フォーム、フィルター機能を含む |
| タスク編集画面 | タスクのタイトルを編集（モーダルまたは別画面） |
| 削除確認 | 削除前の確認ダイアログ |

## 学習項目

### フロントエンド
- フォームの作成と送信処理
- 一覧データの表示
- 状態管理の基本
- ユーザー操作へのレスポンス

### バックエンド
- APIエンドポイントの設計
- リクエスト/レスポンスの処理
- データベース操作（CRUD）
- バリデーション

### データベース
- テーブル設計の基本
- SQL操作（SELECT, INSERT, UPDATE, DELETE）

## ファイル構成

- `database.md` - データベース設計

## チュートリアル構成（推奨）

| ステップ | 内容 | 学習ポイント |
|----------|------|--------------|
| 1 | 環境構築 | 開発環境のセットアップ、プロジェクト作成 |
| 2 | データベース設計 | テーブル作成、初期データ投入 |
| 3 | タスク一覧表示 | データ取得、一覧表示の実装 |
| 4 | タスク作成 | フォーム、バリデーション、データ登録 |
| 5 | タスク完了切替 | 状態更新、UI反映 |
| 6 | タスク編集 | 編集フォーム、更新処理 |
| 7 | タスク削除 | 削除確認、削除処理 |
| 8 | フィルター機能 | 条件による絞り込み表示 |

## API設計（REST API の場合）

| メソッド | エンドポイント | 説明 |
|----------|---------------|------|
| GET | /api/todos | タスク一覧取得 |
| POST | /api/todos | タスク作成 |
| PUT | /api/todos/{id} | タスク更新 |
| DELETE | /api/todos/{id} | タスク削除 |

### リクエスト/レスポンス例

#### タスク一覧取得
```
GET /api/todos?status=all
```

レスポンス:
```json
{
  "todos": [
    {
      "id": 1,
      "title": "牛乳を買う",
      "completed": false,
      "created_at": "2025-01-08T10:00:00Z"
    },
    {
      "id": 2,
      "title": "レポートを提出",
      "completed": true,
      "created_at": "2025-01-07T09:00:00Z"
    }
  ]
}
```

#### タスク作成
```
POST /api/todos
Content-Type: application/json

{
  "title": "新しいタスク"
}
```

レスポンス:
```json
{
  "id": 3,
  "title": "新しいタスク",
  "completed": false,
  "created_at": "2025-01-08T12:00:00Z"
}
```

#### タスク更新
```
PUT /api/todos/1
Content-Type: application/json

{
  "title": "牛乳とパンを買う",
  "completed": true
}
```

#### タスク削除
```
DELETE /api/todos/1
```

## バリデーションルール

| 項目 | ルール |
|------|--------|
| タイトル | 必須、1〜200文字 |

## エラーメッセージ例

| 状況 | メッセージ |
|------|------------|
| タイトル未入力 | タイトルを入力してください |
| タイトルが長すぎる | タイトルは200文字以内で入力してください |
| タスクが見つからない | 指定されたタスクは存在しません |

## 対象レベル

プログラミング初心者向け。CRUD操作の基本を実践的に学べます。

## 設計方針（初心者向けのため）

以下の機能は**含めない**：
- ユーザー認証・ログイン
- 複数ユーザー対応
- カテゴリ・タグ機能
- 期限設定
- 優先度設定
- ドラッグ&ドロップによる並び替え
